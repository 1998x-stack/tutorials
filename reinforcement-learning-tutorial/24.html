第二十四章：Stable-Baselines3入门

同学们好！今天我们来聊一个非常实用的话题——Stable-Baselines3（简称 SB3）。在之前的章节中，我们手动实现了 PPO、DQN、SAC 等算法，这有助于大家深入理解强化学习的底层原理。但在实际项目中，我们通常不会从零开始写算法，而是使用经过充分测试的成熟库。SB3 就是目前 Python 生态中最流行的强化学习库之一。

学习目标：学完本章，你将能够使用 SB3 在 5 行代码内训练一个强化学习智能体，掌握回调系统、超参数调优和模型部署等关键技能。

24.1 SB3库概览

Stable-Baselines3 是由 Antonin Raffin 等人维护的开源强化学习库，基于 PyTorch 框架构建。它继承了 Stable-Baselines（基于 TensorFlow 的第一代）的设计理念，但在易用性、可维护性和性能上都有了巨大提升。

SB3 的核心设计哲学是算法无关的统一 API。无论你使用哪种算法，调用方式几乎完全相同：

model = Algo("MlpPolicy", env).learn(total_timesteps=100000)

安装 SB3 非常简单，一行命令即可：

pip install stable-baselines3[extra]

[extra] 会额外安装 TensorBoard、optuna 等可选依赖，强烈建议加上。

在 SB3 的设计中，"Policy"（策略网络）是整个系统的核心概念。SB3 提供了多种策略网络模板：

  • MlpPolicy：多层感知机策略，适用于状态空间为向量的环境（如 CartPole、HalfCheetah）。
  • CnnPolicy：卷积神经网络策略，适用于图像输入的环境（如 Atari 游戏）。
  • MultiInputPolicy：多模态输入策略，适用于字典形式的状态空间。

下表汇总了 SB3 支持的主要算法及其最佳适用场景：

算法 适用场景 动作空间 特点
PPO 通用场景，性价比之王 离散 / 连续 稳定、易调参、适用范围最广
A2C 轻量级任务，快速原型 离散 / 连续 同步 Advantage Actor-Critic
DQN 离散动作空间 离散 经典值函数方法，支持 Double DQN、Dueling
SAC 连续控制，高样本效率 连续 最大熵框架，探索性好
TD3 连续控制，对超参数鲁棒 连续 双 Q 网络 + 延迟更新 + 目标策略平滑
老师的话：初学者总想找到"最好的"算法。但实际项目中，"最合适的"才是最好的。PPO 是绝大多数情况下的安全起点——它的调参难度最低，收敛稳定性最好。只有当 PPO 的样本效率不够时，才考虑 SAC 或 TD3。

24.2 快速上手

我们来看一个完整的例子——使用 PPO 训练 CartPole 游戏。CartPole 是一个经典的离散控制环境，目标是通过左右移动小车，让杆子保持直立。

下面这段代码，就是一个完整可运行的训练脚本：

import gymnasium as gym
from stable_baselines3 import PPO
from stable_baselines3.common.evaluation import evaluate_policy

# 1. 创建环境
env = gym.make("CartPole-v1", render_mode="rgb_array")

# 2. 创建模型 —— MlpPolicy 自动适配 CartPole 的向量状态输入
model = PPO(
    "MlpPolicy",
    env,
    learning_rate=2.5e-4,
    n_steps=2048,
    batch_size=64,
    n_epochs=10,
    gamma=0.99,
    gae_lambda=0.95,
    clip_range=0.2,
    verbose=1  # 打印训练日志
)

# 3. 训练 —— 只需一行
model.learn(total_timesteps=100_000)

# 4. 保存模型
model.save("ppo_cartpole")

# 5. 加载模型
loaded_model = PPO.load("ppo_cartpole")

# 6. 评估
mean_reward, std_reward = evaluate_policy(
    loaded_model,
    env,
    n_eval_episodes=10,
    deterministic=True
)
print(f"平均奖励: {mean_reward:.2f} +/- {std_reward:.2f}")

env.close()

这段代码展示了 SB3 的完整工作流：创建环境 → 创建模型 → 训练 → 保存 → 加载 → 评估。仅需 6 步（实际代码体量不到 20 行），你就能得到一个训练好的强化学习智能体。

关于 render_mode：在 Gymnasium 的新版本中，创建环境时建议显式指定 render_mode。训练时使用 "rgb_array" 或 None（不渲染），评估时如果需要可视化再使用 "human"。

24.3 回调系统 (Callbacks)

回调系统是 SB3 最强大的功能之一。它让你可以在训练过程中插入自定义逻辑，比如定期评估、保存检查点、动态调整学习率等。

SB3 内置了多种实用的回调：

  • EvalCallback：每隔 N 步在验证环境上评估一次模型，自动保存最优模型。
  • CheckpointCallback：每隔 N 步保存一次模型检查点，防止训练意外中断时丢失进度。
  • StopTrainingOnRewardThreshold：当奖励达到预设阈值时自动停止训练。
  • ProgressBarCallback：在 Jupyter Notebook 中显示训练进度条。

下面是一个使用 EvalCallback + StopTrainingOnRewardThreshold 的完整示例：

from stable_baselines3.common.callbacks import (
    EvalCallback,
    StopTrainingOnRewardThreshold
)

# 创建验证环境
eval_env = gym.make("CartPole-v1")

# 设置停止条件：平均奖励达到 500 时停止
stop_callback = StopTrainingOnRewardThreshold(
    reward_threshold=500,
    verbose=1
)

# 设置评估回调
eval_callback = EvalCallback(
    eval_env,
    callback_on_new_best=stop_callback,
    eval_freq=10000,        # 每 10000 步评估一次
    best_model_save_path="./logs/",
    verbose=1,
    deterministic=True      # 使用确定性策略评估
)

# 训练时传入回调列表
model.learn(
    total_timesteps=500_000,
    callback=[eval_callback]
)

如果你需要更定制化的逻辑，可以继承 BaseCallback 并实现以下钩子方法：

  • _on_step()：每一步环境交互后被调用，可获取 obs、reward、done 等信息。
  • _on_rollout_start() / _on_rollout_end()：每个 rollout 阶段开始/结束时调用。
  • _on_training_start() / _on_training_end()：整个训练过程开始/结束时调用。

举个实际例子，假设你想记录训练过程中的每个 episode 奖励：

from stable_baselines3.common.callbacks import BaseCallback

class RewardLoggerCallback(BaseCallback):
    """自定义回调：记录每个 episode 的累计奖励"""
    def __init__(self):
        super().__init__()
        self.episode_rewards = []
        self.current_reward = 0

    def _on_step(self) -> bool:
        # 累计每一步的奖励
        self.current_reward += self.locals["rewards"][0]

        # 如果 episode 结束，记录并重置
        if self.locals["dones"][0]:
            self.episode_rewards.append(self.current_reward)
            self.current_reward = 0

            # 写入 TensorBoard
            self.logger.record("custom/episode_reward",
                               self.episode_rewards[-1])
        return True  # 返回 False 可提前终止训练
踩坑记录 —— SB3 版本兼容性问题：

曾经有一个同学在项目中用 SB3 训练好的模型，换了台服务器就无法加载了。排查了半天，发现是两台机器的 SB3 版本不一致——v1.7.0 保存的模型在 v1.6.0 上无法加载。

教训：SB3 的模型文件是 pickle 格式，与版本强绑定。生产环境中一定要锁死版本号： 
pip install stable-baselines3==2.3.0
此外，保存模型时建议同时保存 model.get_env() 的环境配置信息，确保加载环境与训练环境一致。

24.4 超参数调优

SB3 的默认参数在很多环境上表现不错，但想要达到最优性能，通常需要针对具体任务进行超参数调优。Optuna 是 SB3 生态中最受欢迎的调参工具。

以下是一个完整的 Optuna + PPO 超参数优化示例：

import optuna
from stable_baselines3 import PPO
from stable_baselines3.common.env_util import make_vec_env

def optimize_ppo(trial):
    """一次 trial 的训练和评估"""
    # 建议超参数
    learning_rate = trial.suggest_float(
        "learning_rate", 1e-5, 1e-3, log=True
    )
    n_steps = trial.suggest_int("n_steps", 256, 4096, step=256)
    batch_size = trial.suggest_categorical(
        "batch_size", [32, 64, 128, 256]
    )
    ent_coef = trial.suggest_float(
        "ent_coef", 0.0, 0.1
    )

    # 创建向量化环境
    env = make_vec_env("CartPole-v1", n_envs=4)

    model = PPO(
        "MlpPolicy",
        env,
        learning_rate=learning_rate,
        n_steps=n_steps,
        batch_size=batch_size,
        ent_coef=ent_coef,
        verbose=0
    )

    model.learn(total_timesteps=50_000)

    # 评估
    from stable_baselines3.common.evaluation import evaluate_policy
    mean_reward, _ = evaluate_policy(
        model, env, n_eval_episodes=5, deterministic=True
    )

    env.close()
    return mean_reward

# 创建 Optuna 研究
study = optuna.create_study(
    direction="maximize",
    pruner=optuna.pruners.MedianPruner(
        n_startup_trials=5, n_warmup_steps=20
    ),
    sampler=optuna.samplers.TPESampler(seed=42)
)

# 开始优化
study.optimize(optimize_ppo, n_trials=50)

# 输出最优参数
print("最优参数:", study.best_params)
print("最优奖励:", study.best_value)

调参过程中的几个关键技巧：

  • 先粗后细：先用大范围搜索，找到有潜力的区域后再缩小范围精细搜索。
  • 使用 Pruner：Optuna 的 MedianPruner 可以在早期终止表现不佳的 trial，节省计算资源。
  • 固定随机种子：每次 trial 使用相同的种子，确保参数差异导致性能差异，而非随机波动。
超参数调整的黄金法则：每次只改一个参数，记录下每个参数的影响。很多人同时调整所有参数，最后虽然性能上去了，但完全不知道哪个参数起了作用。这在学术研究或生产环境中是不可接受的。学会使用 Optuna 的可视化工具（如 optuna.visualization.plot_parallel_coordinate）来分析参数交互。

24.5 模型保存与部署

训练完成后，我们需要把模型部署到实际场景中。SB3 提供了多种模型导出方式：

1. 基础保存与加载

# 保存
model.save("my_model.zip")

# 加载
model = PPO.load("my_model.zip")

# 加载时也可传入自定义环境
model = PPO.load("my_model.zip", env=custom_env)

2. 向量化环境

实际项目中，我们通常使用向量化环境来加速训练。SB3 支持两种向量化方式：

  • DummyVecEnv：单进程向量化，适合简单环境。
  • SubprocVecEnv：多进程向量化，利用多个 CPU 核心并行采样，适合计算密集型环境。
from stable_baselines3.common.env_util import make_vec_env
from stable_baselines3.common.vec_env import SubprocVecEnv

# 创建 8 个并行环境
env = make_vec_env(
    "CartPole-v1",
    n_envs=8,
    vec_env_cls=SubprocVecEnv  # 多进程
)

model = PPO("MlpPolicy", env)
model.learn(total_timesteps=1_000_000)

3. 确定性评估

在训练和评估时，一个非常重要的区别是策略的随机性。训练时需要探索（随机策略），但评估时应该关闭探索（确定性策略）。这就是 deterministic=True 的作用：

# 评估时使用确定性策略 —— 每次决策都选概率最大的动作
mean_reward, std_reward = evaluate_policy(
    model, env, n_eval_episodes=10, deterministic=True
)

# 推理时手动使用确定性策略
obs, _ = env.reset()
done = False
while not done:
    action, _ = model.predict(obs, deterministic=True)  # 关键参数！
    obs, reward, terminated, truncated, _ = env.step(action)
    done = terminated or truncated
小贴士：如果你发现模型在训练时表现很好，但部署后效果大打折扣，八成是忘了加 deterministic=True。随机策略会不断探索次优动作，导致表现不稳定。这是新手最容易忽略的细节之一。

4. 导出到 ONNX / TorchScript

在生产环境中，你通常不会直接使用 SB3 的 Python 代码做推理。ONNX 和 TorchScript 是两种主流的模型导出格式，可以脱离 Python 环境运行：

import torch
from stable_baselines3 import PPO
from stable_baselines3.common.policies import ActorCriticPolicy

model = PPO.load("ppo_cartpole.zip")
policy: ActorCriticPolicy = model.policy

# 导出为 TorchScript
# 注意：SB3 的策略网络是 nn.Module，支持 TorchScript 追踪
dummy_obs = torch.randn(1, 4)  # CartPole 状态维度为 4
traced_model = torch.jit.trace(policy, dummy_obs)
traced_model.save("ppo_cartpole.pt")

导出的模型可以在 C++、移动端、嵌入式设备上运行，真正实现从研究到生产的闭环。

从本章到下一章

掌握了 SB3 的基本用法之后，你会发现训练模型变得前所未有的简单。但随之而来的是一个新问题：当训练不收敛时，你该如何调试？SB3 帮你省去了实现算法的繁琐工作，但它不会帮你诊断问题。

在下一章中，我们将深入探讨调试与可视化——如何利用 TensorBoard、自定义日志、奖励分析、值函数诊断等手段，快速定位训练失败的原因。记住，工具越强大，越需要深入了解其内部机制。工具只是加速器，真正的核心竞争力是你对算法本身的理解。

踩坑记录 —— 环境状态范围不一致：

另一个常见问题：在 CartPole 上训练好的 PPO 模型，换到另一个类似但状态范围不同的环境中，性能急剧下降。这是因为神经网络对输入数据的尺度非常敏感。

解决方案：SB3 支持 VecNormalize 包装器，自动对观测值和奖励进行标准化： 
from stable_baselines3.common.vec_env import VecNormalize

env = make_vec_env("CartPole-v1", n_envs=4)
env = VecNormalize(env, norm_obs=True, norm_reward=True)

model = PPO("MlpPolicy", env)
model.learn(total_timesteps=200_000)

# 保存标准化参数
env.save("vec_normalize.pkl")

# 部署时加载
from stable_baselines3.common.vec_env import VecNormalize
env = make_vec_env("CartPole-v1")
env = VecNormalize.load("vec_normalize.pkl", env)
# ❗非常重要：推理时不要更新 running statistics
env.training = False
env.norm_reward = False
注意：VecNormalize 的 running statistics 在推理时应该冻结，否则环境交互会持续改变标准化参数，导致模型行为漂移。

本章小结

  • SB3 是基于 PyTorch 的主流强化学习库，提供统一的算法 API。
  • 5 行代码即可完成一个强化学习智能体的训练。
  • 回调系统让你可以灵活地监控和控制训练过程。
  • Optuna 是 SB3 生态中最推荐的超参数调优工具。
  • 确定性评估和 VecNormalize 是模型部署中容易踩坑但必须掌握的关键点。
  • 工具只是手段，对算法的深入理解才是核心竞争力。
