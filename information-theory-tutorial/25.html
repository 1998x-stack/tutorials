大家好，我是你们的讲师。

从第 1 章到第 24 章，我们一直在讨论信息论的理论。熵、互信息、KL 散度——这些概念在论文中看起来简洁优美，但在真实数据上计算它们却充满了陷阱。连续变量的互信息怎么算？高维互信息能估计吗？离散变量的熵估计有什么偏差？

这些看起来是"工程细节"的问题，实际上直接决定了你论文结果的可信度。我见过不止一个案例：一位同学在研究特征选择时，用简单的直方图 bins 来估计互信息，结果因为 bin 数选得不对，得到了完全相反的结论——"最有信息量的特征"居然是最没用的那个。这并不是工具的错，而是对估计方法偏差的理解不足。

本章就是你的"避坑指南"。我们会逐一看各种常用的信息量估计方法，了解它们的优点和陷阱，并用 Python 库实战。

离散变量的信息度量：直接计数

对于离散变量，如果数据量足够大，信息量的计算是最直接的——用经验频率（empirical frequencies）来近似概率分布，然后代入熵/互信息/KL 散度的公式：

def empirical_entropy(samples, num_bins=None):
    """直接用经验频率估计离散熵"""
    values, counts = np.unique(samples, return_counts=True)
    probs = counts / counts.sum()
    return -np.sum(probs * np.log2(probs))

def empirical_mi_discrete(x, y):
    """经验互信息：直接用离散计数"""
    # 联合分布
    joint, _, _ = np.histogram2d(x, y, bins=10)
    joint_prob = joint / joint.sum()
    # 边缘分布
    px = joint_prob.sum(axis=1)
    py = joint_prob.sum(axis=0)
    # 互信息
    mi = 0
    for i in range(len(px)):
        for j in range(len(py)):
            if joint_prob[i,j] > 0:
                mi += joint_prob[i,j] * np.log2(
                    joint_prob[i,j] / (px[i] * py[j])
                )
    return mi

这个方法看起来简洁直观——但有一个隐藏的偏差。经验熵 Ĥ(p) 是真实熵 H(p) 的有偏估计：它在小样本下系统地低估了真实熵。原因在于：我们没有观察到的类别会被赋予零概率，但它们的真实概率可能不是零。

举个简单的例子：抛一枚公平硬币 10 次，恰好 10 次都是正面朝上的概率是 1/1024。如果真这样了，经验计算出的熵是 0（因为"反面"从未出现），但真实熵是 1 bit。这个偏差在类别数多的场景下更加严重。

几种校正方法：

  • Miller-Madow 校正：Ĥ_mm = Ĥ + (m-1)/(2N)，其中 m 是观测到的类别数，N 是样本数。简单有效。
  • Jackknife（刀切法）：依次去掉一个样本重新估计，用多次估计的变异来校正偏差。计算量稍大，但更鲁棒。
  • 贝叶斯方法：引入 Dirichlet 先验，避免零概率问题（类似于 label smoothing 的思想）。
经验法则：对于离散变量的熵估计，确保每个类别的平均样本数至少为 5-10。如果某个类别只有几个样本，Miller-Madow 校正是最简便的选择。如果类别数量远超样本数（比如用 10000 个类别、1000 个样本），那任何校正方法都救不了你——这时候应该考虑用连续方法或其他聚合策略。

连续变量的MI估计方法

连续变量的互信息估计是实践中最常见、也最具挑战性的问题。连续变量不像离散变量那样可以"直接计数"——因为联合分布 p(x,y) 是一个密度函数，无法通过有限样本直接获得。

主要有以下方法：

方法一：直方图法（Binning）

最直接的方法：把连续变量离散化。把 X 和 Y 各自分成若干个 bins，然后像离散变量一样计算互信息。缺点很明显：

  • bin 数怎么选？太少 → 丢失信息，产生负偏差；太多 → 稀疏数据，产生正偏差
  • 高维时 bins 数量指数增长（"维度灾难"）
  • bin 边界的选择会影响结果

我早年踩过这个坑：在做基因表达数据（约 20000 维）的特征选择时，用了 20×20 的直方图来估计每个特征和目标之间的互信息。结果 MI 值几乎全是 0——因为 20×20=400 个格子，总共只有几十个样本，每个格子里的样本数少得可怜。这就是一个典型的维度灾难案例。

方法二：KSG 估计器

Kraskov-Stölbauer-Grassberger (KSG) 估计器是现阶段中等维度连续变量互信息估计的黄金标准。它基于 k 近邻（kNN）的密度估计思想：

  • 对每个样本点，找到它在联合空间 (X, Y) 中的第 k 个最近邻的距离
  • 分别统计在 X 和 Y 的投影子空间中，有多少点落在这个距离之内
  • 通过这些计数来估计互信息

KSG 的优点是：对 bins 的选择不敏感、自适应数据密度、偏差相对小。它在维度 d ≲ 20 时表现良好，在 d > 50 时效果开始下降。

方法三：KDE 估计

核密度估计（Kernel Density Estimation, KDE）先用核函数拟合密度，再用数值积分近似互信息。计算量很大（O(N²) 或需要近似），在高维时核带宽的选择也很棘手。一般只在低维（d ≤ 5）且要求精度高的时候用。

# 用 sklearn 比较不同的 MI 估计方法
import numpy as np
from sklearn.feature_selection import mutual_info_regression
from sklearn.metrics import mutual_info_score
from sklearn.neighbors import KernelDensity
import warnings
warnings.filterwarnings('ignore')

# 生成已知 MI 的合成数据
np.random.seed(42)
N = 2000
x = np.random.randn(N)
y = 0.5 * x + 0.5 * np.random.randn(N)  # y = 0.5x + noise

# 理论上，X 和 Y 的互信息可以用公式计算
# 对于二元正态分布：MI = -0.5 * log(1 - ρ²)
rho = 0.5 / np.sqrt(1 + 0.5**2)  # 相关系数
mi_true = -0.5 * np.log2(1 - rho**2)
print(f"真实 MI (理论值): {mi_true:.4f} bits")

# 方法 1: sklearn 的 mutual_info_regression (基于 KSG)
mi_ksg = mutual_info_regression(x.reshape(-1, 1), y, n_neighbors=3)[0]
print(f"KSG 估计 (k=3): {mi_ksg:.4f} bits")

# 方法 2: 离散化 + 互信息得分
from sklearn.preprocessing import KBinsDiscretizer
kbd = KBinsDiscretizer(n_bins=10, encode='ordinal', strategy='uniform')
x_disc = kbd.fit_transform(x.reshape(-1, 1)).ravel().astype(int)
y_disc = kbd.fit_transform(y.reshape(-1, 1)).ravel().astype(int)
mi_binning = mutual_info_score(x_disc, y_disc)
print(f"Binning 估计 (10 bins): {mi_binning:.4f} bits")

# 方法 3: KDE 近似 (仅用于 d=1, 低维)
xy = np.column_stack([x, y])
kde = KernelDensity(bandwidth=0.3).fit(xy)
x_kde = KernelDensity(bandwidth=0.3).fit(x.reshape(-1, 1))
y_kde = KernelDensity(bandwidth=0.3).fit(y.reshape(-1, 1))

# 用 Monte Carlo 采样近似积分
sample_pts = np.random.choice(N, size=500, replace=False)
log_pxy = kde.score_samples(xy[sample_pts])
log_px = x_kde.score_samples(x[sample_pts].reshape(-1, 1))
log_py = y_kde.score_samples(y[sample_pts].reshape(-1, 1))
mi_kde = np.mean(log_pxy - log_px - log_py) / np.log(2)
print(f"KDE 估计: {mi_kde:.4f} bits")

# KSG 最接近真实值，bins 方法偏差明显（通常低估），KDE 计算慢且依赖带宽选择

从上面的对比可以看到，在中等维度上，KSG 是效率和精度之间的最佳平衡。binning 方法虽然快，但容易低估。KDE 结果不稳定，且带宽选择是一门"玄学"。

神经估计器：MINE与InfoNCE

2018 年是神经互信息估计的爆发之年。两篇重要论文——MINE (Belghazi et al., 2018) 和 InfoNCE (van den Oord et al., 2018)——分别提出了用神经网络来估计互信息的方法。

MINE（Mutual Information Neural Estimator）的核心思想是利用 Donsker-Varadhan 表示：互信息可以表示为某个函数 f（由神经网络参数化）在所有概率分布上的 supremum：

I(X;Y) = sup_{f} E_{p(x,y)}[f(x,y)] - log(E_{p(x)p(y)}[exp(f(x,y))])

简单来说：训练一个神经网络 f(x,y) 来最大化这个目标，收敛后的 f 给出的值就是互信息的一个估计。

InfoNCE（InfoNCE Loss）则来自对比学习领域。它通过一个"正样本对 vs. 负样本对"的分类任务给出了互信息的下界：

I(X;Y) ≥ E[log(exp(f(x,y)) / ∑ exp(f(x,y_j)))]

这个下界正是 SimCLR、MoCo、CLIP 等对比学习模型的核心损失函数！所以当你用 InfoNCE loss 训练一个对比学习模型时，你实际上是在最大化一个互信息的下界。

常见误区：MINE 和 InfoNCE 是训练代理（training surrogates），不是无偏的 MI 测量工具！如果你在论文中需要报告 MI 值作为"统计量"，用 KSG 或 infomeasure——不要用 MINE。这是 2025 年 infomeasure 论文中明确指出的常见错误。MINE 给出的是下界，而且训练不稳定——不同的随机种子可能得到完全不同的 MI 值。InfoNCE 是下界，严格来说不能作为 MI 的"估计值"。把它们当作优化的目标函数，而不是数据分析的工具。

一个具体的教训：2020 年有一篇论文声称通过 MINE 发现某个深度学习模型的中间层与输出之间的互信息高达 10 bits。但后续研究发现，同样的模型用 KSG 估计只有 2 bits。MINE 的"估计值"高度依赖于网络的容量、训练轮数和随机种子——它反映的是"神经网络能找到多少互信息"而不是"真实互信息有多少"。

那么应该怎么用这些方法？简单的指导原则：

  • 需要报告 MI 数值作为统计量：用 KSG 或 infomeasure
  • 需要最大化互信息作为训练目标：用 InfoNCE（对比学习）或 MINE（作为损失函数的一部分）
  • 需要在论文中展示信息平面：谨慎对待，明确说明估计方法及其局限性

infomeasure库实战

2025 年，一个名为 infomeasure 的 Python 库在 Scientific Reports 上正式发表。它提供了一个统一 API 来估计各种信息论量：熵、互信息、传递熵、KL 散度等，支持多种估计方法，并且提供了 p 值和统计显著性检验。

它的设计哲学非常实用：你不需要自己实现复杂的 KSG 算法，也不需要在不同的库之间反复切换 API。

# 注意：infomeasure 需要单独安装
# pip install infomeasure

import numpy as np
# 尝试导入 infomeasure (如果已安装)
try:
    import infomeasure as im
    HAVE_INFO = True
except ImportError:
    HAVE_INFO = False
    print("infomeasure 未安装，使用 sklearn 的 KSG 作为替代")

# 生成数据：两个相关的时间序列
np.random.seed(42)
N = 1000
t = np.linspace(0, 10, N)
x = np.sin(t) + 0.1 * np.random.randn(N)  # 正弦 + 噪声
y = np.cos(1.5 * t) + 0.1 * np.random.randn(N)  # 余弦 + 噪声
z = x + 0.3 * np.random.randn(N)  # z 与 x 相关

if HAVE_INFO:
    # infomeasure 统一 API
    mi_xy = im.mutual_info(x, y, estimator='ksg', k=5)
    mi_xz = im.mutual_info(x, z, estimator='ksg', k=5)
    entropy_x = im.entropy(x, estimator='ksg', k=5)

    # 传递熵: X→Z 的信息流
    te_x_to_z = im.transfer_entropy(x, z, estimator='ksg', k=5)

    print(f"MI(X;Y): {mi_xy:.4f} bits")
    print(f"MI(X;Z): {mi_xz:.4f} bits")
    print(f"H(X):    {entropy_x:.4f} bits")
    print(f"TE(X→Z): {te_x_to_z:.4f} bits")

    # 局部值（per-sample）分析
    local_mi = im.mutual_info(x, z, estimator='ksg', k=5, local=True)
    print(f"局部 MI 形状: {local_mi.shape}")
    print(f"前 5 个局部 MI: {local_mi[:5].round(3)}")
    # 局部 MI 告诉你：每个样本对互信息的贡献有多大
    # 大值对应"在联合空间中稀疏的区域"——这些样本携带了更多的信息
else:
    from sklearn.feature_selection import mutual_info_regression
    mi_xy = mutual_info_regression(x.reshape(-1,1), y, n_neighbors=5)[0]
    mi_xz = mutual_info_regression(x.reshape(-1,1), z, n_neighbors=5)[0]
    print(f"MI(X;Y) (sklearn KSG): {mi_xy:.4f} bits")
    print(f"MI(X;Z) (sklearn KSG): {mi_xz:.4f} bits")

# 真实场景：特征选择——用 MI 找出最有信息量的特征
from sklearn.datasets import load_diabetes
data = load_diabetes()
X, y = data.data, data.target
feature_names = data.feature_names

# 计算每个特征和目标之间的互信息
mi_scores = []
for i in range(X.shape[1]):
    mi = mutual_info_regression(X[:, i].reshape(-1, 1), y, n_neighbors=5)[0]
    mi_scores.append(mi)

ranked = sorted(zip(feature_names, mi_scores), key=lambda x: x[1], reverse=True)
print("\n特征 MI 排行 (diabetes 数据集):")
for name, score in ranked:
    print(f"  {name}: {score:.4f}")
# 你会发现 'bmi' 和 's5' 的 MI 最高——这和领域知识完全吻合

infomeasure 还有一个很实用的功能——统计检验。当 MI 的计算值很小的时候，你需要区分"真正有相关性"和"纯噪声"：

if HAVE_INFO:
    # 带 p 值的 MI 估计
    result = im.mutual_info(x, y, estimator='ksg', k=5, test=True, n_permutations=100)
    print(f"MI = {result.mi:.4f}, p = {result.p_value:.4f}")
    # 如果 p > 0.05，MI 值可能来自随机波动

总结一下各种 MI 估计方法的特点：

方法 偏差 方差 维度限制 适用场景
经验计数（离散） 负偏差（小样本严重） 低 类别数 ≤ N/10 离散变量，数据量大
直方图（Binning） 可正可负，依赖 bin 数 中等 d ≤ 5 快速粗略估计
KSG (kNN) 小（黄金标准） 中等 d ≤ 20 连续变量默认选择
KDE 依赖带宽 高 d ≤ 5 精度要求高的低维数据
MINE（神经网络） 下界，训练不稳定 高 不限（但训练难） 作为训练目标，不是统计工具
InfoNCE 下界 中等 不限 对比学习训练损失

这一章我们从工程实践的角度梳理了信息量的估计方法。离散变量的经验估计有偏差，需要用 Miller-Madow 等方式校正。连续变量的 KSG 估计器是中等维度的黄金标准。神经估计器 MINE 和 InfoNCE 适合作为训练目标而非统计测量工具。最后，infomeasure 库提供了一个统一便捷的 API。

Shannon 的理论处理的是"语法"层面的信息——0 和 1 的排列、概率分布之间的差异。但信息还有"语义"——它们意味着什么？当我说"猫"时，你脑中浮现的是一只毛茸茸的动物，而不是一串拼音。语义信息论正在重新定义信息论的边界。下一章我们来探索这个激动人心的前沿领域。
