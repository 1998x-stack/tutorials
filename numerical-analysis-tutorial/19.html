第19章 Krylov子空间与共轭梯度法

大家好，我是你们的讲师。今天我们要进入一个让许多数值分析学习者既兴奋又头痛的话题——Krylov子空间与共轭梯度法。为什么说它"既兴奋又头痛"呢？让我先讲个故事。

好几年前，我一个做有限元分析的朋友跑来诉苦。他手里的工程模型有超过一百万自由度，刚度矩阵虽然稀疏但对角占优性很弱。他尝试用Gauss消去法求解Ax=b，结果50GB内存都不够用。换用Jacobi迭代吧，迭代了上万步还不收敛。最后他找到我的时候，整个项目几乎要延期了。"有没有什么方法，能用更少的内存、更快的速度求解这个问题？"我告诉他：你需要的就是Krylov子空间方法——特别是共轭梯度法(CG)，它只需要几个向量的存储空间，理论上最多n步就能收敛到精确解，而且对大规模稀疏系统效果极好。最终他用预处理共轭梯度法，在普通工作站上几分钟就解完了百万自由度的系统。

这个故事引出了本章的核心问题：当我们面对大规模线性系统Ax=b时，有没有比直接分解更聪明的方法？答案是：有。Krylov子空间方法就是现代科学计算中最重要的迭代法家族之一。

19.1 Krylov子空间的概念

我们先来思考一个问题：给定一个n×n的非奇异矩阵A和一个向量b，解x* = A⁻¹b。根据Cayley-Hamilton定理，矩阵A满足它的特征多项式，所以A⁻¹可以表示为A的n-1次多项式。这意味着——

关键洞察：精确解x*一定存在于Krylov子空间 Kn(A, b) = span{b, Ab, A²b, ..., Aⁿ⁻¹b} 中。

一般地，我们定义m维Krylov子空间为：

Krylov子空间的定义

Kk(A, r₀) = span{r₀, Ar₀, A²r₀, ..., Ak-1r₀}

其中r₀ = b - Ax₀是初始残差，k是子空间的维数。Krylov方法的思想是：在第k步，从k维子空间Kk中寻找一个近似解xk。

为什么我们要在"子空间"里找解？有两个原因。第一，对于大规模问题，精确求解需要O(n³)的代价，而我们往往只需求一个足够好的近似解。第二，Krylov子空间有一个优雅的递推结构：Kk+1 = span{Kk, Akr₀}，这意味着我们每次只需要做一个矩阵-向量乘法A·v就能扩展子空间。

不同Krylov方法的区别在于：我们如何从Kk中挑选出"最好"的近似解？这就引出了共轭梯度法。

19.2 共轭梯度法(CG)原理

共轭梯度法(Conjugate Gradient Method, CG)由Hestenes和Stiefel在1952年提出，专门用于求解对称正定(SPD)系统的线性方程组。它的核心思路既简单又优雅。

A-共轭方向：一组向量{p₀, p₁, ..., pk-1}称为A-共轭的，如果对于i≠j有 pᵢT A pⱼ = 0。这实际上是关于内积⟨u, v⟩A = uTAv的正交性。如果我们能用A-共轭方向作为搜索方向，那么每个方向上的极小化可以独立进行！

A-共轭的几何意义

对于二维情况，考虑椭圆形的等值线f(x) = ½xTAx - bTx。最速下降法沿着最陡方向走，往往需要反复横跳（就像在狭长山谷中左右摆动）。而共轭梯度法第一次走的方向是p₀，第二步走的p₁满足p₀TAp₁ = 0，这意味着沿着p₁走不会破坏已在p₀方向上做好的极小化。所以在二维情况下，两步就能到达精确解！

更具体地说，CG方法在每一步k，在Krylov子空间Kk(A, r₀)中极小化A-范数下的误差||x - xk||A。这里的A-范数定义为||v||A = (vTAv)^{½}。这个极小化问题恰好等价于在子空间中求解Galërkin条件：残差rk与Kk正交。

方法搜索方向极小化目标收敛步数(理论)
最速下降法负梯度 -∇f(x)函数值 f(x)无限步（线性收敛）
共轭梯度法A-共轭方向||x-xk||A≤ n步（精确算术下）
Gauss消去法—精确求解1步

19.3 CG算法与实现

CG算法最令人惊叹的是它只有短短的几行代码，却蕴含着深刻的数学思想。整个算法只需要存储4个向量(x, r, p, Ap)，每次迭代只需要一次矩阵-向量乘法！

def conjugate_gradient(A, b, x0, tol=1e-10, max_iter=1000):
    x = x0
    r = b - A @ x       # 初始残差
    p = r               # 初始搜索方向
    rr = r @ r          # r^T r

    for k in range(max_iter):
        Ap = A @ p                  # 唯一一次矩阵-向量乘法
        alpha = rr / (p @ Ap)       # 步长
        x = x + alpha * p           # 更新解
        r_new = r - alpha * Ap      # 更新残差
        rr_new = r_new @ r_new

        if np.sqrt(rr_new) < tol:   # 收敛检查
            return x, k+1

        beta = rr_new / rr          # 共轭方向系数
        p = r_new + beta * p        # 新的搜索方向
        r = r_new
        rr = rr_new

    return x, max_iter

我们来理解一下这个算法中每个变量的含义：

  • r：残差向量 b - Axk，也是负梯度方向。算法的每一步都在隐含地更新残差。
  • p：搜索方向，通过β将当前残差与旧方向组合，确保A-共轭性。
  • α：步长，精确一维搜索的结果——在当前方向p上极小化A-范数误差。
  • β：修正系数，保证pk+1与所有以前的搜索方向A-共轭。

踩坑经验：数值非正交性的陷阱

我在实际使用CG时踩过一个经典的坑。理论上CG在精确算术下最多n步收敛，但我实际运行时到了第n步残差还很大。当时我百思不得其解，检查代码逻辑完全正确。最后发现问题是：当n很大时，舍入误差导致搜索方向之间失去了A-共轭性。残差的数值正交性丢失，导致算法实际上在重复已经搜索过的方向。解决方法有两个：一是使用"重新启动"策略，每几十步用当前残差重置方向向量；二是改用稳定性更好的变体如CGNR。

19.4 收敛性与预处理

CG方法的收敛速度取决于矩阵A的条件数。具体来说，误差在A-范数下的收敛上界为：

CG收敛定理

||ek||A ≤ 2 · ((√κ - 1)/(√κ + 1))k · ||e₀||A

其中κ = cond₂(A) = λmax/λmin是矩阵的谱条件数（对称正定矩阵的2-范数条件数）。

从这个公式可以看到：当条件数κ接近1时，(√κ-1)/(√κ+1)趋近于0，收敛极快；当κ很大时，这个比值接近1，收敛速度急剧下降。例如：

  • κ = 10：比值 ≈ 0.52，每步误差减半，约10步达到10⁻³。
  • κ = 100：比值 ≈ 0.82，约36步达到10⁻³。
  • κ = 10⁶：比值 ≈ 0.998，约3450步才达到10⁻³。

这就是预处理(preconditioning)如此重要的原因。预处理的思路是：找一个矩阵M（称为预处理器），使得M⁻¹A的条件数远小于A本身，然后求解等价系统：

M⁻¹A x = M⁻¹b

好的预处理器M应该满足两个条件：(1) M⁻¹A的条件数小；(2) 求解Mz = r的代价低。常见的预处理器包括：

预处理器类型构造方式特点
Jacobi（对角）预处理M = diag(A)最简单，但对病态问题效果有限
SSOR预处理M对称超松弛因子分解需要选择松弛参数ω
不完全Cholesky分解(IC)M = LLT，其中L是A的近似Cholesky因子效果好但填充控制重要
代数多重网格(AMG)基于矩阵图结构构造多层次网格对椭圆型PDE极高效，可扩展性强

故事：CG+AMG解决百万自由度有限元问题

回到开头我朋友的故事。他求解的是一个三维弹性力学问题的有限元离散系统，刚度矩阵A是SPD的但条件数超过10⁷。直接使用CG迭代，预计要上万步才能收敛。我推荐他使用代数多重网格(AMG)作为预处理器。AMG通过分析矩阵的强弱连接，自动构造粗网格校正算子。仅用代数的信息就能形成多尺度求解策略。结果：只用了不到50次CG迭代就收敛到10⁻⁸的相对精度，总耗时从预计的数小时缩短到几分钟。这就是预处理技术的威力——它把从"不可能"变成了"轻松"。

在实际工程中，选择什么样的Krylov方法和预处理器是一门艺术。对于SPD系统，CG是首选。对于非对称系统，我们有GMRES(广义极小残差法)和BiCGSTAB(双共轭梯度稳定法)等变体，它们的思路不同但共享相同的思想框架。

最后留给大家一个小练习：当矩阵A不是对称正定时，我们能否用CG？答案是"不能直接使用"，但可以通过求解正规方程ATAx = ATb来间接使用（这时得到的系数矩阵ATA是对称正定的）。这就是CGNR方法——虽然会使条件数平方，但有时仍然有效。
