DP 五步法 — 状态定义 → 转移方程 → 初始化 → 遍历顺序 → 举例

大家好，我是你们的讲师。欢迎来到动态规划的世界——这是 LeetCode 面试中最重要、最需要练习、也最让人"闻风丧胆"的一章。

说它重要，是因为 DP 几乎是大厂面试的必考内容；说它难，是因为 DP 没有固定的"模板代码"，每个问题都需要你创造性地定义状态并推导转移方程。但别慌——我总结了一套DP 五步法，只要按这五个步骤走，95% 的 DP 题目都可以系统化地解决：

  1. 状态定义： dp[i] 或 dp[i][j] 表示什么？
  2. 转移方程：如何从前面的状态推导出当前状态？
  3. 初始化：base case 是什么？
  4. 遍历顺序：从小到大？从大到小？从上到下？
  5. 举例验证：拿一个简单的例子，手动演算 dp 数组的值。

五个步骤中，第 1 步（状态定义）是最关键的——状态定得好，转移方程自然就出来了；状态定得不好，后面怎么推都不对。在我的经验里，DP 学不会无非两个原因：要么是没理解状态定义的意义（只想套公式），要么是在错误的状态定义上花太多时间（不敢推倒重来）。

DP 的本质：动态规划是用"空间换时间"的极致体现——把子问题的结果存下来，避免重复计算。你刷过的回溯题，很多都可以用 DP 优化；你即将要学的背包问题，本质上是 DP 的一个特化分支。所以掌握 DP 的意义，不只是解这几道题，而是打通算法面试的"任督二脉"。

线性 DP 基础

爬楼梯（LeetCode 70）

爬楼梯是 DP 的"Hello World"。题目：每次可以爬 1 或 2 个台阶，爬到第 n 阶有多少种不同的方法？

我们按五步法来走：

  • 状态定义： dp[i] 表示爬到第 i 阶的方法数。
  • 转移方程： dp[i] = dp[i-1] + dp[i-2]——因为到第 i 阶前，你只能在第 i-1 阶跨 1 步或在第 i-2 阶跨 2 步。
  • 初始化： dp[1] = 1（第 1 阶只有 1 种方法），dp[2] = 2（1+1 或 2 直接跨）。
  • 遍历顺序：从小到大。
  • 举例： dp[3] = dp[2] + dp[1] = 2 + 1 = 3，确实是 3 种方法（1+1+1, 1+2, 2+1）。
def climb_stairs(n):
    if n <= 2:
        return n
    dp = [0] * (n + 1)
    dp[1], dp[2] = 1, 2
    for i in range(3, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]

这个代码的空间复杂度还可以优化——因为 dp[i] 只依赖前两个状态，所以用三个变量滚动即可（O(1) 空间）。但面试中，建议先写出二维或一维 dp 数组的版本，再主动提出空间优化——这展示了你的思维深度。

面试提示：爬楼梯虽然简单，但它是很多复杂 DP 题目的基础。比如"每次可以爬 1、2、3 阶"就是 dp[i] = dp[i-1] + dp[i-2] + dp[i-3]；"爬楼梯的最小代价"就是 LeetCode 746。从一个最简单的题目发散出去，可以考察出你 DP 基本功的扎实程度。

打家劫舍系列（198 / 213 / 337）

打家劫舍系列是线性 DP 的经典代表。LeetCode 198——打家劫舍（House Robber）：不能偷相邻的两家，求最大偷窃金额。

def rob(nums):
    if not nums:
        return 0
    if len(nums) == 1:
        return nums[0]

    dp = [0] * len(nums)
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])

    for i in range(2, len(nums)):
        dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])

    return dp[-1]

转移方程 dp[i] = max(dp[i-1], dp[i-2] + nums[i]) 的含义：对于第 i 家，要么不偷（金额等于前 i-1 家的最大值），要么偷（金额等于前 i-2 家的最大值 + 当前金额）。这个方程是整个打家劫舍系列的基石。

LeetCode 213 将其扩展到环形社区，解法无非是拆成两个线性问题：偷第一家不偷最后一家 vs 不偷第一家偷最后一家。LeetCode 337 进一步扩展到二叉树结构，用树形 DP + 后序遍历来解决。

最大子数组与最长递增

最大子数组和（LeetCode 53）

这道题要求找出和最大的连续子数组。状态定义：dp[i] 表示以 nums[i] 结尾的最大子数组和。

def max_sub_array(nums):
    dp = [0] * len(nums)
    dp[0] = nums[0]
    max_sum = dp[0]

    for i in range(1, len(nums)):
        dp[i] = max(nums[i], dp[i - 1] + nums[i])
        max_sum = max(max_sum, dp[i])

    return max_sum

转移方程 dp[i] = max(nums[i], dp[i-1] + nums[i]) 的关键在于：要么从当前元素开始一个新的子数组，要么把当前元素接在前面的子数组后面。这道题的 O(1) 空间优化版就是著名的Kadane 算法。

易错点：很多人在定义状态时会写成 dp[i] 表示"前 i 个元素的最大子数组和"。这个定义不是错的，但会导致转移方程非常复杂——因为你不知道最大子数组的结束位置在哪，无法正确决定是否要接上当前元素。所以记住：以…结尾 是线性 DP 中最常见、最好用的状态定义方式。

最长递增子序列（LeetCode 300）

LIS 是另一道线性 DP 经典题。状态定义：dp[i] 表示以 nums[i] 结尾的最长递增子序列的长度。

def length_of_lis(nums):
    dp = [1] * len(nums)
    for i in range(len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)

时间复杂度 O(n²) 的 DP 解法是最基础的版本。进阶版本可以用耐心排序（Patience Sorting）做到 O(n log n)，通过维护一个 tails 数组存储递增序列的尾部最小值。面试中建议先给出 DP 解法，如果能再写出 O(n log n) 的优化版本，加分非常多。

编辑距离

编辑距离和最长公共子序列是二维 DP的经典例题，也是面试中检验 DP 功力的试金石。

编辑距离（LeetCode 72）

题目：将 word1 转换成 word2，每次可以插入、删除、替换一个字符，求最少操作次数。

状态定义：dp[i][j] 表示 word1 的前 i 个字符转换成 word2 的前 j 个字符需要的最少操作数。

def min_distance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # 初始化
    for i in range(m + 1):
        dp[i][0] = i      # word2 为空，删除所有字符
    for j in range(n + 1):
        dp[0][j] = j      # word1 为空，插入所有字符

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]  # 字符相同，无需操作
            else:
                dp[i][j] = min(
                    dp[i - 1][j] + 1,     # 删除 word1[i-1]
                    dp[i][j - 1] + 1,     # 插入 word2[j-1]
                    dp[i - 1][j - 1] + 1   # 替换 word1[i-1] 为 word2[j-1]
                )

    return dp[m][n]

这个三维 min 的选择就是编辑距离的核心：删除、插入、替换三者选最优。很多人在理解"插入"和"删除"是对称操作这一点上有困难——删除 word1 的一个字符，等价于在 word2 中插入一个字符，所以在 dp 表中这两个操作是对称的。

最长公共子序列（LeetCode 1143）

LCS 和编辑距离高度相似，只是操作从"三种"变成了"两种"：匹配和不匹配。

def longest_common_subsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]
问题 状态定义 转移方程 复杂度
爬楼梯（70） dp[i] = 爬到第 i 阶的方法数 dp[i] = dp[i-1] + dp[i-2] O(n)
打家劫舍（198） dp[i] = 前 i 家的最大金额 dp[i] = max(dp[i-1], dp[i-2] + nums[i]) O(n)
最大子数组和（53） dp[i] = 以 i 结尾的最大子数组和 dp[i] = max(nums[i], dp[i-1] + nums[i]) O(n)
最长递增子序列（300） dp[i] = 以 i 结尾的 LIS 长度 dp[i] = max(dp[j] + 1) for j < i O(n²)
编辑距离（72） dp[i][j] = word1[:i] -> word2[:j] 的最小操作 字符相同则不变，不同则 min(删/插/替) O(mn)
最长公共子序列（1143） dp[i][j] = text1[:i] 和 text2[:j] 的 LCS 相同 +1，不同 max(左/上) O(mn)

踩坑与总结

坑 1：dp 数组维度的直觉

很多 DP 新手的问题："我怎么知道状态是二维还是一维？" 答案：看状态的变量个数。如果决策只需要一个变量（比如"爬到第 i 阶"），就是一维；如果需要两个变量（比如"word1 的前 i 个字符匹配 word2 的前 j 个字符"），就是二维。如果两个变量之间有关系（比如"在 i 到 j 的区间内"），就是区间 DP，我们下下章会讲。

坑 2：初始化的陷阱

初始化是 DP 最容易出错的地方。比如打家劫舍中，长度不足 2 时直接处理——很多人在写循环前忘了考虑边界情况。编辑距离中，dp[i][0] = i 和 dp[0][j] = j 的初始化也是最容易忽略的。

分享一个我自己的惨痛教训：有一次在 LeetCode 周赛中，我写了一道二维 DP 题，状态转移想了 40 分钟，转移方程写对了。但初始化少写了一行——结果从第一个测试用例就开始 WA（Wrong Answer）。调试发现后，加上那行初始化，一次性通过。所以我的铁律是：写 DP 代码时，先把初始化写完再写转移，不要反过来。

坑 3：滚动数组优化

滚动数组优化（把二维 dp 压缩到一维）是面试中常被问到的拓展。以编辑距离为例，dp[i][j] 只依赖 dp[i-1][j-1]、dp[i-1][j] 和 dp[i][j-1]——也就是上一行和当前行的前一个元素。所以可以用一维数组 + 一个临时变量来实现空间压缩。

但要注意：一维优化的遍历顺序很重要。如果压缩后的 dp[j] 依赖上一行的 dp[j-1]，那你就需要逆序遍历（或者用临时变量保存被覆盖的值）。这个我们下一章讲背包问题时还会反复强调。

本章心法：学习 DP 没有捷径，只有刻意练习。我把 DP 题目按照"状态定义方式"分为几类：线性 DP（以 i 结尾）、区间 DP（i 到 j）、背包 DP（容量 + 物品）、树形 DP（子树状态）、状态机 DP（持有/不持有）。每类刷 5-8 道题，你就掌握了 DP 的半壁江山。我们从下一章开始，就会逐一深入这些分类。

本章我们学习了 DP 的基础框架和几个经典例题。动态规划的世界远不止于此——下一章我们将进入背包问题，这是 DP 中最重要的一个分支，也是面试中的高频考点。从 01 背包到完全背包，从组合问题到多维背包，你会发现背包问题的统一结构是多么美妙。我们下一章见！
