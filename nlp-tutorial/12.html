第十二章：BERT详解

2018年10月，Google AI发布了一篇论文《BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding》。这篇论文不仅在当时横扫了11项NLP任务的SOTA，更重要的意义在于——它彻底改变了NLP的工作范式。从此，NLP进入了一个新时代："预训练+微调"取代了"从头训练"，BERT成了几乎所有NLP系统的标配。

BERT的影响怎么强调都不过分。SQuAD 1.1上，BERT的F1得分从之前SOTA的87.4提升到93.2，直接提升了近6个点——在NLP基准测试中，这是前所未见的跳跃。GLUE基准上，BERT同样大幅领先。从问答、文本分类到命名实体识别，几乎所有任务都被BERT刷新了一遍。

这一章的安排是这样的：我们先看BERT的架构设计，然后深入两个核心训练任务——掩码语言模型（MLM）和下一句预测（NSP），接着分析它的输入表示和微调范式，最后介绍几个重要的BERT变体。有了上一章Transformer的基础，理解BERT会轻松很多。

12.1 BERT的架构设计

BERT的核心就是多层双向Transformer编码器的堆叠。注意"双向"这个词——这是BERT和GPT最根本的区别。GPT使用单向（从左到右）的Transformer解码器，而BERT使用双向的Transformer编码器，这让BERT能同时利用一个词左右两侧的上下文信息。

Google发布了两个版本：

配置层数 (L)隐藏维度 (H)注意力头数 (A)参数量
BERT-base12768121.1亿
BERT-large241024163.4亿

BERT-base的设计有意和GPT对比：同样是12层、1.1亿参数，但BERT是双向模型。BERT-large则直接堆到了24层，参数量是base的三倍。有意思的是，在大多数任务上，BERT-base的效果已经足够好，large只在部分高难度任务上体现出优势——这对硬件资源有限的团队是个好消息。

每个编码器层由Multi-Head Self-Attention和Feed-Forward Network组成，和Transformer完全一致。总计算量可以粗略估算为：每层O(n²·d + n·d²)，12层下来，处理512个token的序列需要做约12×512²×768 ≈ 24亿次注意力计算。

关键区别：BERT使用Transformer编码器（双向），GPT使用Transformer解码器（单向/因果）。编码器可以看上下文，解码器只能看左边。在实际效果上，这意味着BERT更擅长"理解"任务（文本分类、问答、情感分析），而GPT更擅长"生成"任务（文本续写、对话、翻译）。这个区别贯穿了整个NLP的发展史，后面会详细展开。

12.2 掩码语言模型（Masked Language Model, MLM）

MLM是BERT最核心的训练目标。直觉上很简单：随机遮盖输入句子中15%的词，让模型预测被遮盖的词是什么。听起来像"完形填空"——对，就是中学英语考试那种。

具体做法：对每个被选中的token，有80%的概率替换为[MASK]标记，10%的概率替换为随机词，10%的概率保持不变。为什么要引入随机替换和不变？因为[MASK]标记在微调阶段不会出现，如果模型只在看到[MASK]时才做预测，下游任务中模型就无法适应没有[MASK]的输入。随机替换迫使模型依赖上下文来做判断——如果替换的词和原词不一致但上下文高度一致，模型应该"怀疑"这个替换。保持原词不变则让模型学会即使没有掩码也要输出有意义的表示。

# MLM 训练过程示意
# 原始句子: I [went] to the bank to [deposit] money
#
# Step 1: 随机选择15%的token
# 选中的token: "went", "deposit"
#
# Step 2: 按概率替换
# 80% → [MASK]: I [MASK] to the bank to [MASK] money
# 10% → 随机词: I [eat] to the bank to [run] money
# 10% → 不变:   I [went] to the bank to [deposit] money
#
# Step 3: 模型在每个被遮盖的位置输出一个vocab_size维的分布
# Step 4: 和真实的token计算交叉熵损失
# loss = -∑ log P(correct_token | context)

MLM的一个潜在问题是：训练和微调存在不匹配——[MASK]标记在微调时不会出现。这就是前面说的80/10/10策略的原因。另外，因为只预测15%的词，BERT每个batch只学习到15%的位置信息，所以BERT的训练需要更多的步数。相比之下，GPT预测每个词，学习效率更高，但BERT学到了双向信息——这个权衡是值得的。

12.3 下一句预测（Next Sentence Prediction, NSP）

除了MLM之外，BERT还有一个辅助任务：预测两个句子是否相邻。这对需要理解句子关系的任务（如问答、自然语言推理）非常关键。

# NSP 训练数据构造
# 正样本（50%）：连续的两个句子
# [CLS] I went to the bank. [SEP] I deposited some money. [SEP]
# Label: IsNext (1)
#
# 负样本（50%）：不连续的两个句子（从语料中随机抽取）
# [CLS] I went to the bank. [SEP] The cat sat on the mat. [SEP]
# Label: NotNext (0)
#
# [CLS]位置的输出经过一个二分类层得到预测结果

NSP虽然简单，但它让BERT的[CLS]表示学会了编码句子级别的信息。不过，后来的研究表明NSP并非最优——RoBERTa移除了NSP后效果反而略有提升（这可能是因为移除了NSP后，模型可以把更多容量投入到MLM上）。ALBERT也提出了一种替代方案SOP（Sentence Order Prediction），预测两个句子的顺序而非是否相邻，效果更好。

12.4 BERT的输入表示

BERT的输入并不是简单的词向量，而是三层嵌入的叠加：Token Embedding + Segment Embedding + Position Embedding。这是BERT设计中最优雅的部分之一。

# BERT 输入表示（以"the cat is fat"为例）
# 先经过WordPiece分词:
# ["the", "cat", "is", "fat"]
# 然后加上特殊标记:
# ["[CLS]", "the", "cat", "is", "fat", "[SEP]"]

# 三层Embedding叠加:
# input = TokenEmbedding + SegmentEmbedding + PositionEmbedding

# Token Embedding: 每个token的语义嵌入
#   E_[CLS] E_the  E_cat  E_is   E_fat  E_[SEP]

# Segment Embedding: 区分两个句子（A段 vs B段）
#   E_A     E_A    E_A    E_A    E_A    E_A

# Position Embedding: 位置信息（可学习的，最大511）
#   E_0     E_1    E_2    E_3    E_4    E_5

# output = LayerNorm(input)
# 最终输出维度 [batch_size, seq_len, hidden_dim]

几个设计细节值得注意：

  • [CLS] 标记：放在序列最前面，对应位置的输出被用作整个句子的聚合表示。对于分类任务，直接在[CLS]表示上加一个分类层。之所以放在第一个位置，是因为Self-Attention让每个位置都能看到所有位置——[CLS]位于句首并不妨碍它编码全局信息。
  • [SEP] 标记：句子分隔符，用来区分两个句子。在单句任务中也必须加上。
  • WordPiece分词：BERT的词表大小是30,000，使用WordPiece算法。"playing"会被拆成"play"和"##ing"——这样可以处理未登录词，同时保持词表适中。
  • 最大长度512：由位置嵌入决定（可学习的Position Embedding从0到511）。超过512的文本需要截断或分片处理。这是BERT的一个重要限制。

12.5 微调范式（Fine-tuning）

BERT真正革命性的贡献不是模型本身，而是它开创的"预训练+微调"范式。BERT在通用语料（BookCorpus + English Wikipedia，共33亿词）上完成预训练后，只需要在特定任务的数据上进行少量额外的训练——这就是微调。

不同的下游任务只需要在BERT的输出上做最小的修改：

任务类型典型数据集输入构造输出层
单句分类SST-2 (情感分析)[CLS]句子[SEP][CLS] → Linear → 类别
句对分类MNLI (自然语言推理)[CLS]前提[SEP]假设[SEP][CLS] → Linear → 类别
问答SQuAD[CLS]问题[SEP]文档[SEP]每个token → Linear → 起/止概率
序列标注CoNLL-2003 (NER)[CLS]句子[SEP]每个token → Linear → 标签
# 使用 HuggingFace 快速微调 BERT 进行情感分类
from transformers import BertTokenizer, BertForSequenceClassification

# 加载预训练模型和分词器
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
model = BertForSequenceClassification.from_pretrained(
    'bert-base-uncased',
    num_labels=2   # 二分类：正面/负面
)

# 准备输入
texts = ["This movie was absolutely fantastic!",
         "I've never been so bored in my life."]
inputs = tokenizer(texts, padding=True, truncation=True,
                   return_tensors='pt', max_length=128)

# 前向传播
outputs = model(**inputs)
logits = outputs.logits           # shape: [2, 2]
predictions = logits.argmax(-1)   # [1, 0] (正面/负面)

# 微调时只需调用 model.train()，用标准交叉熵损失即可
# 通常设置：learning_rate=2e-5, batch_size=16/32
# 2-4个epochs就能收敛（因为BERT已经预训练过了）
# optimizer = AdamW(model.parameters(), lr=2e-5, weight_decay=0.01)

微调的关键经验：BERT微调非常"便宜"——不是指计算量，而是指数据需求。在SST-2上，只需要几千条标注数据就能达到很好的效果。这就是预训练的价值：BERT已经学会了通用的语言知识，微调只是让它"适应"特定任务。这个范式让NLP应用的门槛大幅降低。

微调时的超参数选择也很有讲究：学习率通常很小（2e-5到5e-5），因为BERT的权重已经很好了，太大的学习率会破坏预训练学到的知识。Batch size通常在16-32之间。Epoch数2-4即可——太多会导致过拟合。AdamW优化器是标配，因为它比标准Adam更好地处理了权重衰减。

⚠️ 踩坑故事：微调时的"灾难性遗忘"

我第一次用BERT做中文命名实体识别时，遇到了一个非常诡异的bug。我加载了bert-base-chinese，在自己的医疗文本数据集上微调。训练时loss在下降，验证集指标也在上升——一切看起来都很正常。

但当我用训练好的模型做推理时，模型把几乎所有的"发烧"、"头痛"都预测成了"疾病"——看起来没问题对吧？但它把所有"不发烧"、"没有头痛"也预测成了"疾病"！模型完全忽略了否定词。我检查了分词器，没问题。我检查了训练数据，否定句标注也是正确的。

问题的根源在于：医疗文本中否定句的比例远小于肯定句，模型学会了"看到症状词就预测疾病"的捷径。但BERT本身是有能力理解否定关系的——问题出在微调数据的不平衡，而不是BERT的能力限制。解决方案很直接：在训练中过采样否定句样本，或者简单地增加几个包含否定表达的验证集样本。

教训：不要把微调单纯当成"训练"——你的数据集的质量和平衡性决定了微调效果的上限，BERT本身只是提供一个好的起点。微调数据集如果有偏见，BERT会放大这些偏见。

12.6 BERT的变体

BERT之后涌现了大量改进工作，其中三个变体最为重要：RoBERTa、ALBERT和DistilBERT。

模型参数量核心改进训练数据效果
BERT-base1.1亿原始基线BookCorpus + Wiki (16GB)Baseline
RoBERTa1.25亿移除NSP、动态掩码、更大batch、更多数据BookCorpus + Wiki + CC-News + OpenWebText + Stories (160GB)显著优于BERT
ALBERT1200万 (共享参数)跨层参数共享、矩阵分解嵌入、SOP替代NSP同BERT (16GB)略优于BERT，参数量减少18倍
DistilBERT6600万知识蒸馏（从BERT-base蒸馏为6层）同BERT (16GB)保留95%效果，速度快60%

RoBERTa（Facebook, 2019）是全方位的优化版本。它做了几个关键改变：第一，移除了NSP任务（实验证明没有帮助）；第二，使用动态掩码（每个epoch重新生成掩码模式，而不是固定不变）；第三，使用更大的batch size（8K）和更多的训练数据（160GB vs 16GB）。RoBERTa在GLUE和SQuAD上都显著超过了BERT，证明BERT的原始训练策略远未达到最优。

ALBERT（Google, 2019）专注于模型效率。它通过两个技巧大幅减少参数量：跨层参数共享（所有12层共享同一组权重）和嵌入参数矩阵分解（将词嵌入维度从768降低到128，然后在Transformer层之前投影回768）。这些改动让ALBERT在参数量仅为BERT-base十分之一的情况下（1200万 vs 1.1亿），达到了相当甚至更好的效果。ALBERT还提出了SOP（Sentence Order Prediction）替代NSP，效果更好。

DistilBERT（HuggingFace, 2019）使用知识蒸馏技术——用一个更小的学生模型（6层）去模仿教师模型（BERT-base，12层）的行为。蒸馏的关键是让学生模型学习教师模型的输出概率分布（软标签），而不仅仅是ground truth标签。DistilBERT保留了BERT-base约95%的效果，但模型大小减少40%，推理速度快了60%。

# 四种BERT变体的HuggingFace加载方式对比
from transformers import (BertModel, RobertaModel,
                          AlbertModel, DistilBertModel)

# BERT - 原始
bert = BertModel.from_pretrained('bert-base-uncased')

# RoBERTa - Facebook改进版
roberta = RobertaModel.from_pretrained('roberta-base')

# ALBERT - Google轻量版
albert = AlbertModel.from_pretrained('albert-base-v2')

# DistilBERT - HuggingFace蒸馏版
distilbert = DistilBertModel.from_pretrained('distilbert-base-uncased')

# 使用方式完全一致
# output = model(input_ids, attention_mask)
# output.last_hidden_state  # [batch, seq_len, hidden]
# output.pooler_output      # [batch, hidden] — [CLS]经过tanh

选择哪个变体取决于你的场景：如果你追求最佳效果且不在乎计算资源，选RoBERTa-large；如果你需要快速推理或部署在移动端，选DistilBERT；如果你对参数量有严格限制但需要较好的效果，选ALBERT。对于大多数工业场景，DistilBERT是性价比最高的选择。

好了，BERT的内容到这里就差不多了。总结一下：BERT的核心贡献不是模型架构本身——它用的就是我们已经学过的Transformer编码器——而是它的双向预训练策略和微调范式。BERT证明了"预训练+微调"是一条无比有效的路径。

有趣的是，BERT和下一章的GPT形成了一个完美的对称：BERT是双向编码器，擅长理解；GPT是单向解码器，擅长生成。BERT用掩码语言模型预训练，GPT用自回归语言模型预训练。BERT在标注数据上微调，GPT通过提示直接使用。 它们是NLP中"理解"和"生成"两个方向的代表——BERT像一位细心的读者，GPT像一位流利的作者。下一章，我们来深入GPT系列，看看单向模型如何走出了一条完全不同的道路。
