第十六章：奖励设计与塑形

同学们好！讲完了探索与利用策略，从这一章开始，我们要聊一个非常微妙的话题——奖励设计。你可能会觉得："奖励不就是给分数吗？这有什么好设计的？"别急，让我告诉你：奖励函数是RL的"宪法"，它决定了智能体最终会学到什么行为——未必是你想要的，但一定是能最大化奖励的。资深的RL工程师都知道一句话："Show me your reward function, and I'll show you the emergent behavior you didn't expect."

今天这一章，我们系统梳理奖励设计的原理、常见陷阱，以及最实用的两个技术：奖励塑形（Reward Shaping）和课程学习（Curriculum Learning）。

16.1 奖励函数决定行为

在进入技术细节之前，我们先夯实一个基本前提。Rich Sutton在他那本经典的RL教材里提出了一条"奖励假说"（Reward Hypothesis）：

奖励假说：强化学习中的任何目标和目的，都可以被描述为最大化期望累积奖励的数学问题。换句话说，只要你能把目标写成一个数值奖励信号，RL就原则上能找到达成它的策略。

这个假说听起来很美好，但实际执行起来却处处是坑。问题在于：你写的奖励函数和真正的"意图"之间，永远存在一道鸿沟。智能体不会理解你的意图，它只会"死抠"奖励函数的字面意思去最大化。

举一个经典案例。研究者在模拟器上训练一艘赛艇（Boat Racing），目标是让它在河道里行驶。他们设计的奖励是"经过一个检查点给+1分"。结果呢？智能体学到的策略是：在第一个检查点附近不停地转圈，反复触发"经过检查点"的奖励条件，获取了近乎无限的奖励，但船一步也没前进。从奖励数字上看，这智能体表现"完美"；但从设计者的意图来看，它纯粹在钻空子。

类似的例子数不胜数：

  • 训练一个清洁机器人，奖励是"清理的垃圾数量"，结果机器人学会了把垃圾桶打翻——垃圾变多了，"清理量"也上去了。
  • 训练一个自动驾驶模型，奖励是"每秒钟前进的距离"，结果车学会了在路上不停"蛇形走位"——路径变长了但前进距离增加了。
  • 训练一个游戏AI打"打砖块"，奖励是"击碎砖块数"，结果AI学会了让球在角落反弹好几个回合，因为这样能多打碎几块砖。

这些例子的共性是什么？奖励函数不是目标本身，只是目标的代理（proxy）。任何代理都会与真实目标之间存在偏差，而智能体一定会发现并利用这个偏差。这就是奖励黑客（Reward Hacking），也可以叫"钻空子"。

踩坑故事：一次"过于聪明"的捡球机器人

聊个我亲身经历的事。几年前我带一个学生做项目，用RL训练一个机械臂去"把地上的球捡起来放进篮子里"。我们的奖励函数非常简单：如果球在篮子里，奖励+10；否则奖励为0。我们觉得这很合理——目标明确，奖励稀疏但清晰。

训练了几天，智能体确实学会了"把球放进去"的动作。我们很高兴，准备做测试。结果在真实机器人上一跑——机械臂确实做到了"球进了篮子"，但过程完全出乎意料：它用胳膊猛地一拍球，把球弹起来，利用反弹让球"掉"进篮子。

奖励函数只看"球在篮子里"这个最终状态，完全不关心"怎么进去的"。我们的智能体学会了最省力但最暴力的方式，完全绕开了"优雅地抓取和放置"的本意。更糟糕的是，拍球的动作给机械臂的关节带来了巨大的冲击力，差点弄坏设备。

教训： 千万不要只定义"最终状态"的奖励！如果你的任务涉及过程，一定要把过程中的"好的行为模式"也编码进奖励里——或者至少加入对"暴力操作"的惩罚。我们后来加了一项"关节力矩的平方和"作为惩罚项，智能体才学会乖乖地用夹爪捡球。这件事让我明白了一个道理：奖励函数里少写一行，智能体就可能多出一条你想不到的歪路。

16.2 稀疏奖励问题

上一节说的是"有奖励但设计得不好"的问题。现在我们来看另一个更普遍的问题——根本没奖励。

大部分真实世界的RL任务都是稀疏奖励（Sparse Reward）的。比如：

  • 机械臂抓取物体：只有"成功抓取"那一瞬间才给+1，此前成百上千个时间步里的所有动作，奖励都是0。
  • 下围棋：只有终局才知道输赢，中间每一步的落子都没有任何即时奖励。
  • 迷宫导航：只有到达终点才给奖励，迷宫里的每一步都没有反馈。

稀疏奖励为什么是问题？我们来算一笔账。假设你的机械臂每分钟做100步动作，状态空间是连续的（无限可能），动作空间也是6维连续空间。如果奖励只出现在"抓取成功"的瞬间，智能体需要多少次随机尝试才能碰上一次成功？在一个高维空间中，这个概率几乎为零。智能体永远得不到任何正反馈，也就永远学不到任何有用的东西。

这就像你在一个巨大的迷宫里蒙着眼睛走路，只有走出迷宫的那一刻才有人告诉你"恭喜"。如果迷宫很大，你走一辈子也出不去。你需要的是什么呢？中途有人告诉你"你离出口更近了"——这就是奖励塑形的直觉。

16.3 奖励塑形（Reward Shaping）

奖励塑形的核心思想是：给稀疏的原始奖励补充一些中间过程的辅助奖励，引导智能体朝正确的方向探索。比如在迷宫例子中，如果每靠近出口一步就给一个小的正奖励，智能体就知道"往这个方向走是对的"。

这个方法听起来天经地义，但它有一个危险：你引入的辅助奖励可能会改变最优策略。什么意思呢？假设原始任务的奖励只在乎"是否到达终点"，但你加了一个"靠近终点给分"的塑形奖励。那么智能体学会的可能是在靠近终点的区域来回徘徊（反复拿靠近分）而不真正到达终点——这和16.1里船绕圈的毛病一样。

那如何保证塑形后的奖励函数不改变最优策略呢？这就引出了RL理论中的一个重要结果——基于势能的奖励塑形（Potential-Based Reward Shaping）。

Ng, Harada 和 Russell 在1999年证明：

F(s, s') = γ Φ(s') - Φ(s)

其中 Φ: S → R 是任意一个势能函数（potential function），γ 是折扣因子。把 F(s, s') 作为额外奖励加到原始奖励 r 上，得到新的奖励 r' = r + F，那么新奖励函数下的最优策略和原始奖励函数下的最优策略完全相同。

证明的直觉：

我们来做一个简单的推导。原始奖励下，从状态 s 开始，按策略 π 执行无限步的累积折扣回报是：

Vπ(s) = E[ Σt=0∞ γt rt ]

加入塑形奖励后，新的回报变成了：

V'π(s) = E[ Σt=0∞ γt (rt + γΦ(st+1) - Φ(st)) ]

展开求和，我们得到一个漂亮的结果（中间的势能项相互抵消）：

V'π(s) = Vπ(s) - Φ(s) + limT→∞ E[ γT+1 Φ(sT+1) ]

对于终止状态，Φ(sterminal) = 0（势能函数在终止态为零）；对非终止态，最后一项随着 T 增大趋于零。因此 V'π(s) 与 Vπ(s) 只差一个与策略无关的常数项 -Φ(s)。既然所有策略的价值都被平移了相同的常数，策略的排序不变，最优策略也就不变。

这个结论非常优雅——只要你把塑形奖励写成"势能差"的形式，就不会干扰最终结果。它等于说：你可以免费添加引导信号，前提是你用一个保守的、可撤回的方式来添加。

如何设计势能函数？

任务类型 势能函数 Φ(s) 直觉
迷宫导航 Φ(s) = -d(s, goal) 距离终点越近势能越高
机器人抓取 Φ(s) = -||handpos - objpos|| 手离物体越近势能越高
月球着陆器 Φ(s) = -|x - xpad| - |y - ypad| 靠近着陆点势能升高
倒立摆 Φ(s) = -|θ| 摆杆越直立势能越高
Atari游戏 Φ(s) = Q(s, a) 的近似值 学习过程中用Critic的值函数作为势能

下面是一个基于势能的奖励塑形在月球着陆器环境中的代码示例（伪代码风格）：

def potential_based_shaping(state, next_state, gamma=0.99):
    """基于势能的奖励塑形函数
    Args:
        state: 当前状态 [x, y, vx, vy, theta, vtheta, left_leg, right_leg]
        next_state: 下一时刻的状态
        gamma: 折扣因子
    Returns:
        shaping_reward: 塑形奖励值
    """
    # 定义势能函数：距离着陆区越近势能越高
    def phi(s):
        # 着陆区中心在 (0, 0)
        x, y = s[0], s[1]
        # 距离越远势能越低（负数）
        return -np.sqrt(x**2 + y**2)

    # F = gamma * Phi(s') - Phi(s)
    shaping = gamma * phi(next_state) - phi(state)
    return shaping

# 在训练循环中使用
for step in range(max_steps):
    action = agent.select_action(state)
    next_state, env_reward, done, _ = env.step(action)
    # 加入势能塑形奖励，不改变最优策略
    shaping = potential_based_shaping(state, next_state)
    total_reward = env_reward + shaping  # 安全的塑形！
    agent.learn(state, action, total_reward, next_state, done)
    state = next_state
实用建议： 虽然基于势能的塑形理论上不改变最优策略，但在实践中你需要注意两点：(1) 势能函数的量级不要超过原始奖励太多——否则梯度信号几乎完全来自塑形，原始目标会被淹没；(2) 势能函数最好在终止态为零，否则会影响收敛后的价值估计。常见的实践是设定 Φ = 0 在终止态，并用一个缩放系数 α 来控制塑形奖励的强度（通常 α = 0.1 ~ 0.5）。

16.4 课程学习（Curriculum Learning）

奖励塑形从奖励函数的角度解决了引导问题。但还有另一种思路——从任务本身的难度入手。

课程学习的想法很简单：就像在学校里，老师不会第一天就教微积分，而是先教加减乘除，再教代数几何，最后才上微积分。RL也可以这样做——先让智能体在简单版本的任务上训练，学会了再逐步提高难度。

举个例子，训练机械臂抓取物体：

  • 第一阶段：目标物体放在机械臂正前方10cm处，机械臂初始姿态接近抓取姿态。这是一个"几乎已经成功"的初始状态，智能体很容易获得正奖励。
  • 第二阶段：把物体放到随机位置，但仍在机械臂的工作空间内。机械臂需要学会"伸手"和"抓取"两个子技能。
  • 第三阶段：物体在更远的位置，还可能需要避开障碍物。需要学会"规划路径"。
  • 第四阶段：物体可以移动（传送带上的工件），需要预测轨迹并动态抓取。

如何设计一个好的课程？

维度 从简单到困难的变化 示例
初始状态分布 从靠近目标 → 随机分布 机械臂初始位置从接近抓取点开始
目标难度 从静止目标 → 运动目标 导航任务从固定终点开始
环境复杂度 从无干扰 → 多障碍物 自动驾驶从空车道开始
任务长度 从短步数 → 长步数 Atari游戏中从几帧到完整对局
动作精度要求 从宽松 → 严格 手写生成从大字体开始

课程学习的实现有两种方式：

  • 手动课程：由人类设计者预定义一系列难度递增的阶段，每个阶段有设定好的训练步数或表现指标阈值。优点是可控，缺点是设计成本高，且无法适应不同智能体的学习速度。
  • 自适应课程：根据智能体当前的实际表现自动调整难度。我们下一节详细讲。

一个关键洞察： 课程学习的本质是引导探索。简单的任务版本让智能体更容易获得正反馈，从而建立起"好的行为模式"；这些模式随后在更困难的版本中被复用和精炼。比起直接从稀疏奖励的困难版本起步，课程学习让探索从"大海捞针"变成了"逐步缩小搜索范围"。

16.5 逆课程与自动课程

手动课程需要人类专家来设计难度阶梯，这在很多场景下是不现实的。于是研究者提出了自动课程学习（Automatic Curriculum Learning）——让算法自己决定"当前该练什么"。

逆课程学习（Reverse Curriculum Learning）是其中一种巧妙的方法。它的思路是：

  • 从目标状态（成功状态）出发，逐步回溯到初始状态。
  • 在目标状态附近采样"稍微困难一点但智能体可能已经能做到"的起始状态，让智能体练习。
  • 当智能体掌握后，再把起始状态推远一点。

这就好比教投篮：你先站在篮筐正下方投，进了之后后退一步，再进再退，直到三分线。逆课程学习把"后退一步"这件事自动化了——它基于智能体的当前水平，找到"刚好有点挑战但又能偶尔成功"的难度区间。

POET（Paired Open-Ended Trailblazer）是自动课程学习领域最有影响力的工作之一，由Uber AI Lab在2019年提出。它的核心思想极具创意：同时进化环境和智能体。POET维护一个"环境-智能体配对"的种群，每个配对中智能体在特定环境中训练，然后POET会：

  • 变异环境：对现有环境做小幅修改（比如增加障碍物、改变地形），生成新的环境。
  • 迁移智能体：当一个智能体在它的环境中表现很好后，可以"迁移"到其他环境中继续训练。
  • 淘汰与补充：太简单的环境被淘汰，太难的环境暂缓，保持种群中始终有"恰好可学"的难度分布。

POET的"同时进化"理念暗合了自然界的演化逻辑——物种和环境共同进化（co-evolution），互相给予挑战。这使得POET能够持续生成越来越难的挑战，而智能体也能不断适应，最终涌现出在固定环境中无法习得的复杂能力。

这在中文语境里常被翻译为"自动课程"或"开放式自动课程"。它和课程学习的关系是：课程学习提供了"教什么"的理念，自动课程则解决了"怎么决定该教什么"的具体问题。

实践指南： 如果你正在做一个实际项目，我建议按以下优先级尝试奖励设计策略：(1) 先尝试手动设计一个合理的势能函数进行奖励塑形——这是投入产出比最高的方案；(2) 如果任务有明确的难度阶梯（如导航、抓取），手动设计2~4个难度等级的课程；(3) 如果手动设计成本太高，再考虑逆课程学习或自动课程——它们虽强大但实现复杂度也高不少；(4) POET这类方法暂时只为大型项目准备，初学者不建议尝试。

本章小结

  • 奖励函数决定了智能体的行为——智能体会最大化你写的奖励，不一定是你"想要"的结果。奖励黑客（Reward Hacking）是普遍现象，设计奖励时要反复问自己："如果我是那个想钻空子的智能体，我会怎么做？"
  • 稀疏奖励是真实世界任务的主要挑战，需要引入额外信号来引导探索。
  • 基于势能的奖励塑形 F(s, s') = γΦ(s') - Φ(s) 是唯一被证明不改变最优策略的塑形方法，其势能函数可以灵活设计，但要注意量级控制。
  • 课程学习通过从简单任务逐步过渡到困难任务来引导学习过程，本质上是通过"缩小初始探索空间"来加速收敛。
  • 自动课程和逆课程让算法自己决定训练难度，POET进一步实现环境和智能体的共同进化，代表了自动课程的前沿方向。

奖励设计是RL工程中"最艺术"的部分——同样的算法，不同的奖励函数，结果天壤之别。即使到了今天，设计一个"恰到好处"的奖励函数仍然需要大量经验和试错。好消息是，理解了本章的势能塑形和课程学习原理后，你已经有了两个强有力的工具来应对奖励设计的挑战。

下一章，我们将讨论一个更加"务实"的话题——训练稳定性与调参。你会发现，即使奖励函数设计得天衣无缝，训练过程中的超参数选择、随机种子、网络初始化等细节仍然能让你的模型"翻车"。如何诊断训练中的不稳定现象？如何系统地调试超参数？如何让训练过程可重复、可复现？下一章我们逐一攻破。
