网络中的最优路径与最小代价

同学们好。如果说上一章的拓扑排序和并查集处理的是"能否"的问题（能否完成课程？是否连通？），那么本章要处理的是"最优"的问题——哪条路径最短？怎样连接总成本最小？

最短路径和最小生成树是图论中面试频率最高的两类算法题。它们背后有一个共同的思想——贪心：每次选择当前最优的边或节点，逐步构建全局最优解。Dijkstra 用堆维护"当前已知最短距离"，Kruskal 用并查集维护"当前已连接的最小边"——贪心的形式不同，但内核高度一致。

本章路线：Dijkstra（单源正权最短路）→ Floyd（多源最短路）→ Kruskal & Prim（最小生成树）→ 算法对比与选型指南。学完本章，你能解决 LeetCode 上 95% 的最短路径和最小生成树问题。

Dijkstra：堆优化实现

Dijkstra 算法是单源最短路径问题（从一个源点到所有其他节点的最短距离）的经典解法，前提是图中没有负权边。它的核心思想非常优雅：每次从未访问的节点中选出距离源点最近的节点，用它的边去"松弛"其邻居的距离。

朴素 vs 堆优化

朴素版的 Dijkstra 每次找最近节点需要 O(V)，总复杂度 O(V²)。这在稠密图（E ≈ V²）上还可以接受，但对于 LeetCode 上的稀疏图来说，堆优化版（O((V+E)logV)）才是正解。

堆优化版的核心是用优先队列（最小堆）维护"候选节点"，每次 pop 出距离最小的节点。

import heapq

def dijkstra(n, edges, start):
    # 建图
    adj = [[] for _ in range(n)]
    for u, v, w in edges:
        adj[u].append((v, w))
        # 有向图加这一句，无向图再加 adj[v].append((u, w))

    dist = [float('inf')] * n
    dist[start] = 0
    pq = [(0, start)]  # (距离, 节点)

    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]:  # 惰性删除：过期的条目跳过
            continue
        for v, w in adj[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))

    return dist
惰性删除：你可能注意到 if d > dist[u]: continue 这行。由于同一个节点可能被加入优先队列多次（每次发现更短距离时），pop 出来的可能是"过期"的条目。跳过即可，不需要手动删除——这是 Dijkstra 堆优化版的经典写法。

例题：网络延迟时间（LeetCode 743）

题目：n 个网络节点，有向边表示信号传输时间。从节点 k 发送信号，求所有节点都收到信号所需的时间。

直接套 Dijkstra 模板——求出从 k 到所有节点的最短距离，取最大值。如果存在不可达的节点，返回 -1。

def network_delay_time(times, n, k):
    adj = [[] for _ in range(n + 1)]
    for u, v, w in times:
        adj[u].append((v, w))

    dist = [float('inf')] * (n + 1)
    dist[k] = 0
    pq = [(0, k)]

    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]:
            continue
        for v, w in adj[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))

    max_dist = max(dist[1:])  # 节点从 1 开始编号
    return max_dist if max_dist != float('inf') else -1
常见错误：Dijkstra 不能处理负权边！如果图中存在负权边，之前"确定"的最短距离可能在后续被更短的路径推翻，但 Dijkstra 不会回头去更新已确定节点。有负权边时请用 Bellman-Ford 或 SPFA。我有个同事在生产环境的路由系统中用了 Dijkstra，结果一条负权链路导致整个网络的路由表出错——幸好测试阶段发现了。

Floyd 算法：多源最短路

当需要计算任意两点之间的最短距离时，可以运行 n 次 Dijkstra（O(n(E+V)logV)），但更简洁的方式是 Floyd 算法（O(V³)）。Floyd 的核心思想是动态规划：dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])。

def floyd(n, edges):
    INF = 10**9
    dist = [[INF] * n for _ in range(n)]
    for i in range(n):
        dist[i][i] = 0

    for u, v, w in edges:
        dist[u][v] = min(dist[u][v], w)  # 防重边

    for k in range(n):
        for i in range(n):
            for j in range(n):
                if dist[i][k] < INF and dist[k][j] < INF:
                    dist[i][j] = min(dist[i][j],
                                     dist[i][k] + dist[k][j])
    return dist

三层循环的顺序是固定的：最外层是中间节点 k。为什么 k 必须在最外层？因为 Floyd 的本质是"逐步允许使用更多中间节点"——k 从 0 到 n-1 表示每一步允许使用前 k 个节点作为中间点。顺序错了，结果就错了。

例题：阈值距离内邻居最少的城市（LeetCode 1334）

题目：n 个城市，边表示权重（距离）。给定距离阈值，找出在阈值范围内可到达城市数量最少的城市（如果有多个，取编号最大的）。

Floyd 求出所有城市间最短距离后，逐行统计即可。

def find_the_city(n, edges, distanceThreshold):
    INF = 10**9
    dist = [[INF] * n for _ in range(n)]
    for i in range(n):
        dist[i][i] = 0

    for u, v, w in edges:
        dist[u][v] = w
        dist[v][u] = w

    for k in range(n):
        for i in range(n):
            for j in range(n):
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]

    min_count = n
    result = -1
    for i in range(n):
        count = sum(1 for j in range(n)
                    if i != j and dist[i][j] <= distanceThreshold)
        if count <= min_count:
            min_count = count
            result = i
    return result

Floyd 的另一个常见应用是传递闭包：不是求最短距离，而是判断两点是否可达。把 dist[i][j] 改成布尔值，dist[i][j] = dist[i][j] or (dist[i][k] and dist[k][j]) 即可。

传递闭包例题：LeetCode 1462（课程表 IV）——给定先修关系，查询某个课程是否是另一个的先修课。用 Floyd 的传递闭包版本可以一行核心逻辑解决。

最小生成树：Kruskal 与 Prim

最小生成树（MST）问题：在无向带权图中找一棵树，连接所有节点且总边权最小。

Kruskal 算法：排序 + 并查集

Kruskal 的思路非常直观：把所有边按权重排序，从小到大逐一尝试——如果这条边的两端还没有连通（属于不同集合），就把它加入 MST。

def kruskal(n, edges):
    # edges: [(w, u, v)] 或 [(u, v, w)]
    edges.sort(key=lambda x: x[2])
    uf = UnionFind(n)
    total = 0
    count = 0  # 已选边数

    for u, v, w in edges:
        if uf.find(u) != uf.find(v):
            uf.union(u, v)
            total += w
            count += 1
            if count == n - 1:
                break

    return total if count == n - 1 else -1  # -1 表示不连通

时间复杂度 O(ElogE)（排序）+ O(Eα(n))（并查集操作），适合稀疏图。

Prim 算法：堆 + 贪心

Prim 的思路类似 Dijkstra——从一个节点开始，每次选择"到已选集合最近"的边，把对应的节点加入集合。

def prim(n, edges):
    adj = [[] for _ in range(n)]
    for u, v, w in edges:
        adj[u].append((v, w))
        adj[v].append((u, w))

    visited = [False] * n
    pq = [(0, 0)]  # (权重, 节点)
    total = 0
    count = 0

    while pq:
        w, u = heapq.heappop(pq)
        if visited[u]:
            continue
        visited[u] = True
        total += w
        count += 1
        for v, nw in adj[u]:
            if not visited[v]:
                heapq.heappush(pq, (nw, v))

    return total if count == n else -1

时间复杂度 O((V+E)logV)，适合稠密图。

例题：连接所有点的最小费用（LeetCode 1584）

题目：二维平面上 n 个点，连接两点 i 和 j 的费用是曼哈顿距离 |xi - xj| + |yi - yj|，求连接所有点的最小总费用。

def min_cost_connect_points(points):
    n = len(points)
    # 建边：完全图，每个点对之间都有边
    edges = []
    for i in range(n):
        for j in range(i + 1, n):
            d = abs(points[i][0] - points[j][0]) + \
                abs(points[i][1] - points[j][1])
            edges.append((i, j, d))

    return kruskal(n, edges)

对于 n 个点，共有 n(n-1)/2 条边。当 n=1000 时，边数约 50 万，Kruskal 可以接受。但如果 n 更大（如 10⁴），用 Prim 更合适——因为 Prim 不需要显式构建所有边，可以在计算距离的同时进行贪心选择。

Kruskal vs Prim 选择指南：图比较稀疏时用 Kruskal（排序边更划算）；图比较稠密时用 Prim（边太多排序不划算）。LeetCode 上 MST 题目通常 n 不大，两种都能过——选你顺手的那一种即可。

算法对比：选型指南

算法 解决的问题 图的要求 时间复杂度 核心数据结构
Dijkstra 单源最短路径 无负权边 O((V+E)logV) 堆（优先队列）
Floyd 多源最短路径 无负权环（可负权边） O(V³) 二维 DP 数组
Bellman-Ford 单源（可负权） 无负权环 O(VE) 松弛 n-1 轮
Kruskal 最小生成树 无向图 O(ElogE) 并查集
Prim 最小生成树 无向图 O((V+E)logV) 堆 + visited 数组

踩坑与总结

坑1：负权边不能用 Dijkstra

这是最经典的坑，没有之一。Dijkstra 基于这样一个假设：一旦某个节点从优先队列中 pop 出来，它的最短距离就"确定"了。但如果有负权边，后来可能有一条更短的路径更新它。所以面试时如果题目没有明确说"所有边权为正"，一定要先问清楚，或者选择 Bellman-Ford。

坑2：INF 的设定陷阱

初始化最短距离时，很多人用 float('inf')。这在 Python 中没问题，但在 Floyd 的三层循环中，dist[i][k] + dist[k][j] 如果都用 float('inf')，结果不会溢出（inf + 有限 = inf，inf + inf = inf）。但如果你用 10**9 之类的大整数作为 INF，要确保 INF + INF 不会变成负数（整型溢出在 Python 中不会发生，但在其他语言中是个大坑）。

坑3：重边与自环

LeetCode 的测试数据中经常包含重边（两个节点之间有多条边）和自环（节点到自身的边）。建图时一定要取最小值（dist[u][v] = min(dist[u][v], w)），或者在构建邻接表时只保留最短的那条。自环的处理要看题目要求，通常忽略即可。

面试策略：最短路径和 MST 在 LeetCode 上的题目数量不算多，但每道都很有分量。我建议你至少把 Dijkstra（堆优化版）和 Kruskal 的模板练到"闭着眼睛都能写"的程度——面试时考官很可能让你直接在白板上默写其中一个。另外，记得在写 Dijkstra 时主动说明"本题没有负权边，所以用 Dijkstra"——这会加分。
