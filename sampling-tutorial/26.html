第26章 大数据时代的抽样

大家好，我是你们的讲师。这一章我们要讨论一个被问了无数次的问题："现在都有大数据了，抽样还有用吗？"

我每次听到这个问题都觉得很有意思——它隐含了一个假设：大数据 = 全数据。很多人认为"既然我们有了所有数据，就不需要抽样了"。但实际情况远不是这么简单。一个典型例子：某电商平台有 1 亿用户的全量交易数据，但这些用户本身只是全国人口的子集，而且是一个有严重选择偏差的子集（只有上网购物的人）。用这个"全量数据"来推断全国居民的消费水平，结果会严重高估。大数据不是万能药。

大数据 ≠ 全数据，更 ≠ 无偏数据

大数据时代带来了一个令人兴奋的承诺：我们不再需要"用部分推断整体"，因为"整体"就在那里了。但这个承诺在大多数情况下是虚假的。大数据通常存在以下问题：

  • 覆盖偏差（Coverage Bias）：大数据覆盖的往往只是总体中的某个子集。微博数据只能代表活跃用户，移动支付数据只能代表有支付习惯的人群。这些子集与总体的结构存在系统性的差异。
  • 选择偏差（Selection Bias）：大数据是"被动产生"的，而不是"主动收集"的。人们的行为本身是一种选择——更主动的用户产生更多数据，更沉默的用户几乎没有数据。
  • 测量误差（Measurement Error）：大数据的变量定义往往不明确。例如用"搜索次数"来衡量"兴趣程度"，但搜索行为受很多因素影响，并不直接等于兴趣。
  • 相关性不等于因果性：大数据擅长发现相关关系，但对于因果推断，仍然需要精心设计的抽样研究和实验设计。
维度 传统概率抽样 大数据（如行政记录、网络数据）
覆盖范围 通过抽样设计确保对目标总体的覆盖 仅覆盖使用该平台/服务的人群
偏差控制 随机化消除选择偏差 存在固有的选择偏差
样本量 通常较小（几百到几万） 极大（百万到十亿级）
精度量化 严格的标准误和置信区间 难以量化，易产生虚假信心
成本 收集成本高 利用现有数据，收集成本低
时效性 数据收集后可用于分析 实时数据流
变量丰富度 目标明确，变量精准 变量丰富但噪音大、定义模糊

所以，正确的认识是：大数据和抽样不是替代关系，而是互补关系。大数据可以帮我们更好地做抽样，抽样可以帮我们更好地理解和校正大数据的偏差。

利用大数据优化抽样设计

大数据最直接的贡献是让抽样变得更好、更便宜。具体来说：

构建更好的抽样框

传统的抽样框依赖人口普查数据、户籍资料、地图等，更新慢、成本高。大数据的出现提供了大量可用的辅助信息。例如，可以利用手机信令数据来构建流动人口的实时抽样框；利用商业注册数据来更新企业名录。荷兰统计局（Statistics Netherlands）甚至利用税务记录和社保数据来构建"虚拟抽样框"，大幅提高了抽样效率。

更好的分层信息

分层抽样的效率取决于分层变量与目标变量的相关程度。大数据可以提供更丰富、更精准的分层信息。例如，在做居民健康调查时，可以利用医保大数据来构建分层变量——这比传统的按行政区划分层要有效得多，因为医保数据直接与健康指标相关。

ML 优化抽样策略

这是近年来非常活跃的研究方向。核心思路是：利用机器学习模型预测目标变量，然后根据预测结果来优化抽样设计。看一个具体的例子：

# ML 辅助分层抽样示例
import pandas as pd
import numpy as np
from sklearn.ensemble import GradientBoostingRegressor

# 假设有一个大型行政数据库（N=500万）
# 包含一些容易获取的辅助变量
df_all = pd.read_csv('admin_data.csv')

# 从行政数据中抽一个小样本（n=5000）做详细调查
pilot_sample = df_all.sample(5000)

# 用调查结果训练模型，预测目标变量
X = pilot_sample[['age', 'income', 'education', 'region_code']]
y = pilot_sample['health_score']  # 实际调查得到的目标变量
model = GradientBoostingRegressor()
model.fit(X, y)

# 对整个行政数据库做预测
df_all['predicted_score'] = model.predict(
    df_all[['age', 'income', 'education', 'region_code']]
)

# 根据预测值进行最优分层
# 目标是使层内方差最小、层间方差最大
df_all['stratum'] = pd.qcut(
    df_all['predicted_score'],
    q=5,  # 分成 5 层
    labels=['Q1', 'Q2', 'Q3', 'Q4', 'Q5']
)

# 在每层内进行 Neyman 分配
stratum_std = df_all.groupby('stratum')['predicted_score'].std()
stratum_N = df_all.groupby('stratum').size()
total_n = 10000  # 总样本量

# Neyman 分配：n_h = n * (N_h * S_h) / sum(N_h * S_h)
neyman_alloc = (
    total_n * (stratum_N * stratum_std) /
    (stratum_N * stratum_std).sum()
)

print("Neyman 分配结果：")
print(neyman_alloc.round())
关键洞察：ML 辅助抽样的核心优势在于，它允许我们利用大数据中的辅助信息来"指导"抽样，把有限的调查资源集中在信息量最大的样本上。Statistics Netherlands 的研究表明，使用 ML 优化的抽样策略可以在不增加样本量的情况下，将估计精度的提升 20%-40%。这个领域的术语叫做 "Model-Assisted Sampling"（模型辅助抽样）。

非概率样本的校正

大数据带来的另一个挑战是：大量的数据来自非概率样本（例如网络调查、社交媒体数据、APP 用户行为日志）。这些数据虽然量大，但存在严重的自选择偏差。如何利用它们做统计推断？

目前的思路主要有两种：

倾向得分加权（Propensity Score Weighting）

借用因果推断中的倾向得分方法。假设我们有一个"高质量的参考样本"（比如一个小的概率样本），和一个"大的非概率样本"。我们可以估计每个个体同时出现在两个样本中的概率（倾向得分），然后用倾向得分的倒数作为非概率样本的权重。这种方法的核心假设是：给定可观测协变量，样本选择机制是可忽略的。

校准估计（Calibration / GMM）

利用总体层面的已知总量信息（来自人口普查或行政数据）来调整非概率样本的权重，使得加权后的样本矩与总体矩一致。这种方法在官方统计中已经得到广泛应用。例如，如果你知道全国人口的年龄和性别分布，就可以调整非概率样本的权重，使其年龄性别结构与全国一致。

# 校准加权的简化实现
import numpy as np
import pandas as pd
from scipy.optimize import minimize

def calibration_weights(nonprob_sample, pop_totals, X_vars, max_iter=100):
    """
    通过校准使非概率样本的加权总量匹配已知总体总量。

    使用 raking（迭代比例拟合）方法。
    """
    n = len(nonprob_sample)
    # 初始权重：均匀权重
    w = np.ones(n) / n * sum(pop_totals.values())

    X = nonprob_sample[X_vars].values
    # 目标：加权后的各变量总量与总体总量一致
    targets = np.array([pop_totals[v] for v in X_vars])

    def loss(w):
        w = np.clip(w, 0.01, None)  # 权重非负
        weighted_totals = X.T @ w
        return np.sum((weighted_totals - targets) ** 2 / targets)

    # 使用 L-BFGS-B 优化
    result = minimize(loss, w, method='L-BFGS-B',
                      options={'maxiter': max_iter})
    return result.x

# 示例：校准非概率样本的年龄和性别分布
pop_totals = {
    'male_18_35': 150_000_000,
    'male_36_60': 170_000_000,
    'female_18_35': 140_000_000,
    'female_36_60': 165_000_000
}
calibrated_w = calibration_weights(nonprob_sample, pop_totals,
                                   ['male_18_35', 'male_36_60',
                                    'female_18_35', 'female_36_60'])

谷歌流感趋势（Google Flu Trends）的教训

说到大数据与抽样的关系，有一个案例不得不提：Google Flu Trends（GFT）。

2008 年，Google 在《自然》杂志上发表了一篇论文，宣布他们可以利用用户的搜索关键词来"预测"流感爆发。思路很简单：当人们出现流感症状时，会在 Google 上搜索"发烧怎么办""流感症状"等关键词。通过分析 5000 万条搜索词条中的 45 个关键词的时间序列，GFT 成功"预测"了流感的传播趋势。

听起来很完美，对吧？结果呢？

从 2011 年到 2013 年，GFT 持续高估了流感的流行程度。在 2012-2013 年的流感季节，GFT 的估计值比美国 CDC 的实际数据高出了两倍以上。最终 Google 在 2015 年关闭了这个项目。

为什么 GFT 失败了？ 原因正是我们这一章反复强调的——大数据不等于无偏数据。

首先，GFT 的模型存在过拟合：45 个关键词是从 5000 万条搜索历史中选出来的，拟合了太多与流感无关的偶然相关。例如，"高中篮球赛"这个关键词在某些年份与流感传播高度相关——但不是因为篮球导致了流感，而是它们碰巧在同一时间发生。

其次，搜索行为本身在变化：随着 Google 推出自动补全、知识图谱等功能，用户搜索"流感症状"的方式发生了变化，但 GFT 的模型没有捕捉到这种变化。

最重要的教训是：GFT 没有一个概率样本来校正它的估计。它依赖的是一个巨大的、但有严重选择偏差的非概率样本。而 CDC 虽然样本量小得多（几千个哨点医院），但它的估计是通过精心设计的概率抽样得到的，因此是可靠的。

大 + 偏 ≠ 准。这是本课程自第一章《文学文摘》以来就在强调的真理，在大数据时代依然成立。

未来趋势：抽样与大数据的深度融合

那么，抽样技术的未来在哪里？我认为不是什么"被大数据替代"，而是深度融合：

  • 混合估计（Mixed-mode Estimation）：将概率样本的精确性与大数据的丰富性结合起来。例如用小概率样本估计偏差，用大数据推断趋势。
  • 动态抽样（Dynamic Sampling）：利用实时数据流动态调整抽样设计。当某区域的指标发生异常变化时，自动加密该区域的样本。
  • 贝叶斯方法：将大数据信息作为先验分布，概率样本数据作为似然，通过贝叶斯更新得到后验推断。

大数据时代不是抽样的末日，而是抽样的新机遇。工具变了，规模变了，但核心原则没有变——样本的代表性比样本量更重要，偏差控制比数据大小更关键。下一章，我们将用一个完整的综合案例实战，把从第一章到这一章的所有知识串联起来，让你看到一套抽样调查如何从零到一落地实施。
