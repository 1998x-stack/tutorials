第15章 ELECTRA：判别式预训练的效率革命

同学们好。前面几章我们深入研究了BERT、RoBERTa这一系列基于"掩码语言模型"的预训练范式。大家有没有想过一个问题：BERT每次只预测15%的token，剩下85%的计算资源其实都浪费在"看"那些已经知道的token上了？这一章要介绍的ELECTRA，用一种极其巧妙的方式——把生成式任务改成判别式任务——让每一个token都参与损失计算，从而将训练效率提升了3到4倍。这是一篇来自Google和Stanford的工作（ICLR 2020），它的核心思想直接影响了许多后续的高效预训练方案。

核心洞见：与其只预测15%的[MASK]位置，不如让模型学会"揪出被替换的token"——所有token都参与训练，效率自然翻倍。

15.1 从MLM到RTD：一个简单的思想转变

让我们先回顾一下BERT的做法。给定一个句子"我爱吃苹果"，BERT会随机选15%的位置打上[MASK]标记，比如变成"我爱吃[MASK]"，然后让模型预测被遮住的词。这个损失只在被mask的15%位置计算，剩下85%的位置只是被动地提供上下文信息，它们本身不产生梯度。

ELECTRA的论文（arXiv:2003.10555，Clark et al.，ICLR 2020）提出了一个截然不同的思路：用一个Generator（生成器）先把输入中的某些token替换掉，然后让Discriminator（判别器）去判断每一个token到底是"original"还是"replaced"。这样一来，所有N个位置都会参与二分类损失的计算。

名字的由来：ELECTRA全称是"Efficiently Learning an Encoder that Classifies Token Replacements Accurately"，恰好对应"ELECTRA"（希腊神话中的一位女神），与BERT风格一脉相承。

15.2 Generator-Discriminator协作机制

ELECTRA的架构包含两个Transformer Encoder：

  • Generator G：一个小型的BERT（参数约为Discriminator的1/3到1/2），负责做标准的MLM预测，然后用采样结果替换掉被mask的token。
  • Discriminator D：我们的主模型（也就是最终要用的BERT编码器），对每个位置的token做二分类：这个token是原始的，还是被Generator替换过的？

举个例子：原始句子是"the chef cooked the meal"。Generator可能把"cooked"替换成"ate"，于是Discriminator看到的输入变成"the chef ate the meal"。Discriminator的任务是逐token判断：the（原始）、chef（原始）、ate（被替换！）、the（原始）、meal（原始）。

这两个模型在训练过程中联合训练，但共享Embedding层的权重。训练结束后，Generator被丢弃，只保留Discriminator用于下游任务。

15.3 RTD vs MLM：对比分析

对比维度 BERT (MLM) ELECTRA (RTD)
训练目标 生成式：预测被mask的token 判别式：判断token是否被替换
参与损失的token数 15% 100%
预训练/微调一致性 不一致（出现[MASK]符号） 一致（无特殊符号）
训练效率（同计算量） 基准 3-4倍提升
小模型表现 较弱 远优于BERT-Small（接近BERT-Base）
收敛速度 需要更多步数 更快收敛
下游任务效果（GLUE） BERT-Base: 82.2 ELECTRA-Base: 85.0
关键洞察：ELECTRA在GLUE上以不到BERT 1/4的计算量就达到了BERT的效果。对于计算资源有限的团队，ELECTRA是一个非常划算的选择。

15.4 生成器的大小为何有讲究？

一个很自然的问题：既然Discriminator是我们最终要用的模型，那Generator是不是越大越好？毕竟更大的Generator可以生成更"逼真"的替换，给Discriminator制造更难的挑战。

论文通过实验给出了一个反直觉的结论：Generator的参数量大约是Discriminator的1/3到1/2时效果最佳。原因有三：

  1. 难度平衡：如果Generator太强，替换的token过于逼真，Discriminator几乎无法区分，导致学习信号太弱；如果Generator太弱，替换的token太假，Discriminator轻松识别，也学不到什么。
  2. 计算效率：Generator越大，训练总开销越大，会抵消RTD带来的效率优势。
  3. 共享Embedding：Generator和Discriminator共享token embedding，Generator的嵌入表示质量直接影响Discriminator的输入表示。
踩坑实录：我在一次实验中尝试用同等的BERT-Base作为Generator和Discriminator，结果GLUE分数反而比1/3大小的Generator差了1.5个点。训练成本翻倍，效果却更差——Generator太强会让判别任务失去意义，这让我深刻理解了"教学相长，适度为难"的道理。

15.5 训练细节与损失函数

ELECTRA的训练过程包含两个损失函数：

Generator的MLM损失（标准的交叉熵）：

# Generator MLM loss — 只计算被mask的位置
# x_masked: 被mask的输入, y: 原始token
logits_G = generator(x_masked)  # [batch, seq_len, vocab_size]
# 只取被mask位置的logits
masked_logits = logits_G[mask_positions]  # [num_masked, vocab_size]
loss_G = cross_entropy(masked_logits, y[mask_positions])

Discriminator的RTD损失（逐token的二分类）：

# ELECTRA RTD loss — 所有位置都参与
# 1. Generator对masked位置做采样替换
probs = softmax(generator(x_masked), dim=-1)
sampled_tokens = categorical_sample(probs)
# 仅在mask位置做替换，其他位置保留原token
x_corrupted = x_masked.clone()
x_corrupted[mask_positions] = sampled_tokens[mask_positions]

# 2. Discriminator判断每个token是否被替换
logits_D = discriminator(x_corrupted)  # [batch, seq_len, 2]
labels = torch.ones(batch, seq_len)     # 1=original
labels[mask_positions] = 0              # 0=replaced（若采样≠原token）
# 注意：如果采样结果恰好等于原token，label仍为1
labels[mask_positions & (sampled_tokens == y)] = 1

loss_D = binary_cross_entropy(logits_D, labels)

# 3. 联合训练
loss = loss_G + lambda * loss_D  # lambda通常是50，平衡两个损失量级
一个小细节：如果Generator采样出的token恰好和原始token一样（完全有可能，尤其是高频词），那这个位置在Discriminator的label仍然是"original"（1），而不是"replaced"（0）。这个处理防止了Discriminator混淆"未被改变"和"被改成原词"两种情况。

15.6 为什么ELECTRA对小模型特别友好？

ELECTRA一个引人注目的结论是：ELECTRA-Small（12层，512隐藏维）在GLUE上可以达到接近BERT-Base的水平，而参数量只有BERT-Base的1/4。原因在于：

  1. 训练效率高：小模型本身计算量少，加上RTD让每个token都参与学习，相当于从有限的参数中榨取了更多的信号。
  2. 无[MASK]不一致问题：小模型对预训练-微调不一致更敏感，而ELECTRA避免了[MASK]符号，所以微调时没有"域偏移"。
  3. 表示学习更充分：判别式任务迫使Encoder学习所有token的上下文表示，而不只是被mask的那15%。

15.7 ELECTRA的局限与影响

当然，ELECTRA并非完美无缺：

  • 两模型联合训练增加了训练的复杂度，需要同时维护两个Encoder。
  • 在超大模型规模下（如BERT-Large以上），ELECTRA相对于MLM的优势有所缩小。
  • RTD损失虽然训练效率高，但不一定对所有下游任务都最优——比如生成式任务（文本生成）可能更适合生成式预训练。

尽管如此，ELECTRA的判别式预训练思路对后续研究产生了深远影响。许多高效预训练方案都借鉴了"全token参与损失"这一核心思想。如果你想进一步了解后续的融合方案，下一章我们将介绍XLNet——它从另一个角度（排列语言模型）解决了BERT的[MASK]不一致问题。

本章小结：ELECTRA通过将预训练从生成式（预测mask）改为判别式（检测替换），让全部token参与损失计算，在相同计算量下获得了3-4倍的效率提升。Generator-Discriminator的协同训练设计巧妙，尤其适合计算资源有限的场景。对追求效率的团队来说，ELECTRA至今仍是一个非常值得考虑的基线。
