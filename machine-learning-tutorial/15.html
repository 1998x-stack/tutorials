15.1 调参不是玄学，是科学

很多初学者觉得调参就是"随便试，看运气"。这种心态会让你浪费大量时间。调参的本质是在超参数空间中搜索最优组合，这是一个明确的优化问题——只是搜索空间是离散+连续的混合，而且每次"函数评估"（训练一次模型）代价可能很高。

先讲个教训。我早期做比赛的时候，手动调XGBoost的参数——改一个，跑一遍，记一下分数，再改下一个。一个周末调了大概50组参数，最后发现效果最好的那组和我第一次随手设的差不了多少。后来学会了贝叶斯优化（Optuna），同样的时间能搜索500组，效果比手动调高了一大截。所以这件事给我的教训是：把调参这件事交给算法，你的时间应该花在理解问题上。

15.2 三种搜索策略

方法原理何时用
Grid Search穷举所有参数组合参数空间小（≤3个参数，每参数≤5个值）
Random Search随机采样参数组合参数空间大，不知道哪些参数重要
Bayesian Search用概率模型指导搜索训练代价高，想用更少尝试找到更优参数

Grid Search的致命弱点是"维度灾难"——3个参数各10个候选值就是1000种组合。如果有5个参数，各10个值，就是10万种。实践中Grid Search几乎只用于粗搜+细搜的两阶段策略。

💡 Random Search为什么比Grid好？有研究发现，大多数模型只有少数几个参数真正影响效果。Grid Search在每个参数上均匀撒网，但"重要"的参数和"不重要"的参数被同等对待，浪费了很多评估在不重要的参数组合上。Random Search在重要参数上自然尝试了更多不同的值（因为这些参数的值变化会导致结果变化）。再有，你可以随时停止Random Search并得到"目前为止最好"的结果，而Grid Search如果半途停止可能漏掉整个区域。

15.3 Optuna：现代调参利器

Optuna是当前最流行的超参数优化框架。它使用贝叶斯优化（具体是TPE，Tree-structured Parzen Estimator）来智能选择下一组要尝试的参数。核心优势：

  • Define-by-Run：不预先定义搜索空间，在函数内部动态构建
  • 剪枝（Pruning）：训练过程中如果发现某组参数效果明显不好，提前终止，省时
  • 可视化：内置参数重要性、优化历史等可视化工具
import optuna
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score

def objective(trial):
    # 用trial.suggest_*定义搜索空间
    n_estimators = trial.suggest_int('n_estimators', 50, 500)
    max_depth = trial.suggest_int('max_depth', 3, 30)
    min_samples_split = trial.suggest_int('min_samples_split', 2, 20)
    min_samples_leaf = trial.suggest_int('min_samples_leaf', 1, 10)
    max_features = trial.suggest_categorical('max_features', ['sqrt', 'log2', None])

    model = RandomForestClassifier(
        n_estimators=n_estimators,
        max_depth=max_depth,
        min_samples_split=min_samples_split,
        min_samples_leaf=min_samples_leaf,
        max_features=max_features,
        random_state=42, n_jobs=-1
    )

    score = cross_val_score(model, X_train, y_train, cv=5, scoring='f1').mean()
    return score

# 创建study，设方向为最大化
study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=100, show_progress_bar=True)

print(f"最佳参数: {study.best_params}")
print(f"最佳分数: {study.best_value:.4f}")

# 查看参数重要性
optuna.visualization.plot_param_importances(study)

15.4 AutoML：机器学习的"自动驾驶"

AutoML的目标是自动化从数据处理到模型选择的整个流程。目前主流的AutoML工具：

工具特点适用场景
AutoGluonAmazon出品，表格/文本/图像通吃，效果极好快速baseline、中小数据集
FLAMLMicrosoft出品，搜索效率极高，成本低预算受限的调参
PyCaret低代码，流程最完整快速实验和原型
# AutoGluon一行代码搞定表格预测
from autogluon.tabular import TabularPredictor

predictor = TabularPredictor(label='target', eval_metric='f1').fit(
    train_data, time_limit=600  # 10分钟自动搜索
)
predictions = predictor.predict(test_data)
# 自动尝试了RF, GBDT, NN, KNN等多个模型并集成
⚠️ AutoML的边界：AutoML不是万能的！它能帮你找到好的模型和参数组合，但它不能：(1) 帮你定义业务问题；(2) 帮你做数据清洗和理解；(3) 帮你创造有业务含义的特征；(4) 处理数据泄露。我见过有人把所有原始数据扔进AutoGluon然后抱怨效果不好——问题不在AutoML，在数据质量。特征工程和问题定义仍然是人类的核心竞争力。

15.5 调参的实战策略

经过多年踩坑，我的调参策略收敛为以下几步：

  1. 先不调参，建基线：用默认参数跑一个baseline，记下分数。这是你的参照点
  2. 粗调：用Optuna或Random Search，在宽的参数范围搜索50-100次。目标是找到"哪个方向有希望"
  3. 细调：在粗调找到的最优区域附近，缩小搜索范围再搜50次
  4. 固定参数，做集成：不再追求单个模型的极致，把最好的几个模型ensemble起来，通常能再提升1-3%
🔑 关键心态：调参的边际收益是递减的。前20次尝试可能提升5%，后100次可能只提升0.5%。如果你花了三天时间换来0.1%的提升，问问自己：这个时间用来做特征工程是不是性价比更高？好特征 > 好参数 > 好模型——记住这个优先级。
