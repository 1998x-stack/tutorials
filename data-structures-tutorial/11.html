医院急诊室的"优先级"

想象你是一个急诊室的分诊护士。每分钟都有新病人被送来——有轻伤（擦破皮）、有中症（骨折）、也有危重病人（心脏病发作）。你不能按照"先来后到"的顺序处理病人。心脏骤停的病人必须优先于擦破皮的病人——即使擦破皮的人先到。

这就是优先队列（Priority Queue）的概念——每个元素都有一个"优先级"，优先级最高的元素最先被处理，而不考虑入队顺序。

用什么数据结构来实现优先队列？你可能会想到：

  • 数组/链表：插入 O(1)，取出最大/最小值需要 O(n) 扫描。或者保持有序，插入 O(n)，取出 O(1)。都不完美。
  • 平衡BST：插入和取出都是 O(log n)，但实现太复杂了。

有没有一种更轻量级的数据结构，能够以 O(log n) 的复杂度完成插入和取出最大值/最小值，且实现足够简单？有——堆（Heap）。

堆的概念

堆（Heap）是一种特殊的完全二叉树，它满足堆序性质（Heap Property）：

  • 最大堆（Max-Heap）：每个节点的值都 ≥ 其子节点的值。根节点是最大值。
  • 最小堆（Min-Heap）：每个节点的值都 ≤ 其子节点的值。根节点是最小值。

注意堆只要求一个节点和它的子节点之间满足大小关系，不要求左子树和右子树之间有特定的大小关系（这是堆和BST的关键区别）。事实上，在堆中，你甚至不关心"兄弟节点谁大谁小"——只要满足堆序性质就行。

# 这是一个合法的最大堆：
#        100
#       /   \
#      50    30
#     /  \   /
#    20  10 25
#
# 每个节点都比它的子节点大（或等于）
# 但不要求兄弟节点之间有序：
# 50 > 30 或者 50 < 30 都是可以的

# 这也是一个合法的最大堆：
#        100
#       /   \
#      30    50
#     /  \
#    20  10
# 30 < 50? 没问题！堆只要求"父 ≥ 子"

堆的数组实现

由于堆是完全二叉树，我们可以用数组来存储，不需要显式的左右指针。这在第8章已经埋下了伏笔：

  • 根节点存储在 index = 1（方便计算，忽略 index = 0）
  • 对于 index = i 的节点：parent = i // 2
  • 对于 index = i 的节点：left = 2 * i，right = 2 * i + 1
# 数组表示的最大堆
#        100
#       /   \
#      50    30
#     /  \
#    20  10
# 数组: [_, 100, 50, 30, 20, 10]
# index: 0  1    2   3   4   5
# 通常 index=0 留空

# parent(3) = 3//2 = 1 → arr[1] = 100 (节点30的父节点是100) √
# left(2) = 4 → arr[4] = 20 (节点50的左孩子是20) √
# right(2) = 5 → arr[5] = 10 (节点50的右孩子是10) √

上浮（Swim）与下沉（Sink）

堆的两个核心操作是上浮（swim / bubble-up）和下沉（sink / bubble-down）：

  • 上浮：在堆尾插入一个新元素后，它可能比父节点大（最大堆），这时需要不断和父节点交换，直到堆序恢复。
  • 下沉：取出堆顶后，将堆尾元素移到堆顶，它可能比子节点小，这时需要不断和较大的子节点交换，直到堆序恢复。
class MinHeap:
    """最小堆实现"""
    def __init__(self):
        self.heap = [0]  # index从1开始，0占位

    def push(self, val):
        """插入元素"""
        self.heap.append(val)
        self._swim(len(self.heap) - 1)

    def pop(self):
        """弹出最小值（堆顶）"""
        if len(self.heap) <= 1:
            return None
        if len(self.heap) == 2:
            return self.heap.pop()
        min_val = self.heap[1]
        self.heap[1] = self.heap.pop()  # 把堆尾移到堆顶
        self._sink(1)
        return min_val

    def _swim(self, i):
        """上浮：如果节点比父节点小，向上交换"""
        while i > 1 and self.heap[i] < self.heap[i // 2]:
            self.heap[i], self.heap[i // 2] = self.heap[i // 2], self.heap[i]
            i //= 2

    def _sink(self, i):
        """下沉：如果节点比子节点大，向下交换（与较小的子节点交换）"""
        n = len(self.heap) - 1
        while 2 * i <= n:
            j = 2 * i  # 左孩子
            # 如果右孩子更小，用右孩子
            if j + 1 <= n and self.heap[j + 1] < self.heap[j]:
                j = j + 1
            if self.heap[i] <= self.heap[j]:
                break
            self.heap[i], self.heap[j] = self.heap[j], self.heap[i]
            i = j

    def peek(self):
        return self.heap[1] if len(self.heap) > 1 else None

    def __len__(self):
        return len(self.heap) - 1

    def heapify(self, arr):
        """从数组构建堆，O(n)"""
        self.heap = [0] + arr[:]
        n = len(self.heap) - 1
        # 从最后一个非叶子节点开始下沉
        for i in range(n // 2, 0, -1):
            self._sink(i)

# 测试
h = MinHeap()
h.push(3)
h.push(1)
h.push(4)
h.push(1)
h.push(5)
print(h.pop())  # 1
print(h.pop())  # 1
print(h.pop())  # 3
print(h.pop())  # 4
print(h.pop())  # 5
注意 heapify 的时间复杂度是 O(n)，不是 O(n log n)：直觉上你可能觉得对 n 个元素每个执行一次下沉就是 O(n log n)，但仔细分析——堆中第 k 层的节点数约为 n/2ᵏ，这些节点最多下沉 k 次。总操作次数 Σ k·n/2ᵏ = O(n)。这是堆的一个很"反直觉"但非常优雅的性质。

堆排序

堆排序的过程非常简单：先把数组构建成一个最大堆（或最小堆），然后不断取出堆顶元素。每次取出堆顶后，将堆尾移到堆顶并下沉，保持堆序。

def heap_sort(arr):
    """堆排序，原地排序，升序"""
    n = len(arr)

    # 1. 建堆（最大堆），从最后一个非叶子节点开始下沉
    for i in range(n // 2 - 1, -1, -1):
        _sink(arr, n, i)

    # 2. 不断取出堆顶，放到数组末尾
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]  # 堆顶（最大值）放到末尾
        _sink(arr, i, 0)  # 对剩余部分重建堆

    return arr

def _sink(arr, n, i):
    """对 arr[0..n-1] 中的 i 节点下沉"""
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2
    if left < n and arr[left] > arr[largest]:
        largest = left
    if right < n and arr[right] > arr[largest]:
        largest = right
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        _sink(arr, n, largest)

# 测试
print(heap_sort([3, 1, 4, 1, 5, 9, 2, 6]))
# [1, 1, 2, 3, 4, 5, 6, 9]

堆排序的时间复杂度：O(n log n)，空间复杂度：O(1)（原地排序）。但它不是稳定排序——相等的元素可能会改变相对顺序。

Top-K问题

Top-K问题是面试中的高频考点，比如"在100万个元素中找出最大的K个"。堆是解决这类问题最优雅的工具。

考虑LeetCode 215——「数组中的第K个最大元素」：

  • 排序法：对整个数组排序，取倒数第K个。O(n log n)。
  • 堆解法：维护一个大小为K的最小堆，遍历数组，每个元素和堆顶比较，比堆顶大就pop堆顶再push新元素。遍历结束后，堆顶就是第K大的元素。O(n log K)。
  • 快速选择（Quick Select）：类似快排的分区思想，平均O(n)。效率最高但最坏O(n²)。
import heapq

def find_kth_largest(nums, k):
    """第K个最大元素，用最小堆"""
    heap = nums[:k]
    heapq.heapify(heap)  # 建一个大小为k的最小堆
    for num in nums[k:]:
        if num > heap[0]:      # 当前元素比堆顶大
            heapq.heapreplace(heap, num)  # pop堆顶，push新元素
    return heap[0]  # 堆顶就是第K大的

# 对比：如果要求第K个最小元素，用最大堆（或者对负数建最小堆）

def top_k_frequent(nums, k):
    """前K个高频元素，LeetCode 347"""
    from collections import Counter
    count = Counter(nums)
    # 用最小堆维护前K个高频元素
    heap = []
    for num, freq in count.items():
        heapq.heappush(heap, (freq, num))
        if len(heap) > k:
            heapq.heappop(heap)
    return [num for _, num in heap]

为什么堆解法比排序法更好？

  • 当 n 很大而 K 很小时（比如 n=10⁹, K=100），堆的空间只有 O(K)，而排序需要 O(n) 空间。
  • 堆是在线算法——你可以一个一个地接收数据，不需要一开始就看到全部数据。这在处理数据流（data stream）时非常有用。
面试中的Top-K策略：面试官问Top-K时，先答"排序法"（最简单），然后提"堆解法"（常规最优），最后如果能说出"快速选择可以做到平均O(n)，但最坏O(n²)"，面试官会眼前一亮。三种方法展现出你思维的深度。

优先队列的实际应用

Python 的 heapq 模块提供了堆操作，我们可以直接用：

import heapq

# 基本操作
heap = []
heapq.heappush(heap, 3)     # 插入
heapq.heappush(heap, 1)
heapq.heappush(heap, 4)
print(heapq.heappop(heap))  # 1，弹出最小值
print(heap[0])              # 3，peek最小值

# 对已有列表堆化
arr = [5, 2, 8, 1, 9]
heapq.heapify(arr)          # O(n)
print(arr)                  # [1, 2, 8, 5, 9]

# 最大堆：插入负值
max_heap = []
heapq.heappush(max_heap, -3)
heapq.heappush(max_heap, -1)
heapq.heappush(max_heap, -4)
print(-heapq.heappop(max_heap))  # 4

一个经典的堆应用——合并K个有序链表（LeetCode 23）：

def merge_k_lists(lists):
    """合并K个有序链表（每个链表已排序）"""
    import heapq
    dummy = ListNode(0)
    curr = dummy
    heap = []

    for i, lst in enumerate(lists):
        if lst:
            heapq.heappush(heap, (lst.val, i, lst))

    while heap:
        val, i, node = heapq.heappop(heap)
        curr.next = ListNode(val)
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next

堆 vs 有序数组操作复杂度对比：

操作 堆（最小堆） 有序数组
插入 (Insert) O(log n) O(n)（需要移动元素）
获取最小值 (peek) O(1) O(1)
删除最小值 (pop) O(log n) O(n)（需要移动元素）
构建 (Build) O(n) O(n log n)（排序）
查找任意值 O(n) O(log n)（二分查找）
合并两个结构 O(n) O(m + n)

堆的强项在于"频繁插入 + 频繁取最小值"的场景——优先队列、任务调度、Dijkstra算法中的节点选择都是典型的应用。如果主要需求是"快速查找任意元素"，那应该使用平衡BST或哈希表。

本章小结

这一章我们学习了堆与优先队列：

  • 通过医院急诊室的类比理解优先队列的需求，以及为什么堆是实现它的理想结构。
  • 掌握了堆的核心性质——完全二叉树 + 堆序（父≥子或父≤子）。
  • 实现了堆的数组存储（父子下标关系）和两个核心操作上浮（swim）与下沉（sink）。
  • 用堆实现了堆排序——O(n log n) 原地排序，不稳定。
  • 用堆解决Top-K问题和合并K个有序链表，展示了堆在数据流场景的独特优势。
  • 对比了堆与有序数组各操作的复杂度差异。

堆的"完全二叉树 + 数组存储"组合非常优雅，但它的表达能力是有限的——堆只能保证根节点是最大/最小值，无法像BST那样查找任意元素。如果需要"既能快速查找任意元素，又能快速插入删除"，还是要靠平衡树。
