为什么用R做多元分析

大家好，欢迎继续学习！如果你已经跟到了第22章，那么现在你脑中已经有了一个完整的多元分析方法工具箱——PCA、因子分析、聚类、判别、CCA、对应分析、MDS，每一种方法从原理到实战都过了至少两遍。但有一个问题：前面的所有章节，我们都是用手算推导或者伪代码来演示核心逻辑的。从今天开始，我们要动真格的了——用真正的编程语言，把前面学的所有方法跑一遍。

为什么先从R开始？因为在统计学家圈子里，R是当之无愧的"母语"。CRAN（Comprehensive R Archive Network）上托管着超过两万个扩展包，涵盖了从最经典的 Bartlett 球形检验到最深奥的稀疏协方差估计的每一个角落。如果你在学术期刊上看到一篇多元统计的方法论文，十有八九作者会提供一个 R 包来配套实现。加上 RStudio 提供了极舒适的 IDE 体验，以及 R Markdown / Quarto 让可重复性研究报告变得轻而易举——R 在统计分析这个赛道上，至今仍然是王者。

一句话概括：如果你要做严肃的统计分析（发论文、做实验分析、探索性数据分析），R的生态比Python更成熟；如果你要把模型部署到生产环境，Python更合适。两者不是替代关系，是互补关系。

核心包概览

R的包生态系统非常丰富，但我们不需要全部掌握——只需要知道每个多元分析方法对应哪个主力包就够了。下面这个表格是你的"兵器谱"：

方法主力包核心函数补充包
主成分分析statsprcomp(), princomp()factoextra, PCAmixdata
因子分析stats, psychfactanal(), psych::fa()GPArotation, nFactors
聚类分析stats, clusterkmeans(), hclust(), pam()dbscan, mclust, fpc
判别分析MASSlda(), qda()caret, klaR, rpart
典型相关分析CCA, yaccacc(), yacca::cca()candisc, CCP
对应分析FactoMineRCA()ca, ade4
多维标度法statscmdscale(), isoMDS()smacof, vegan

这里有一个重要的区分需要记住：R的stats包是基础包（安装R时就自带了），里面的函数是"标准配置"；而factoextra、psych、cluster等是第三方包，需要单独安装。factoextra尤其值得关注——它的作者是法国统计学家Alboukadel Kassambara，这个包专门为多元分析的结果可视化而生，可以和stats、FactoMineR等包无缝配合。

PCA在R中实现

R里面做PCA有两个原生函数：prcomp() 和 princomp()。有什么区别？prcomp() 使用SVD（奇异值分解）来求解，数值稳定性更好，是推荐选择；princomp() 使用特征值分解，要求协方差矩阵满秩，存在数值精度问题。在实际项目中——永远用 prcomp()。

# 加载数据：R内置的iris数据集（150个样本，4个变量）
data(iris)
X <- iris[, 1:4]  # 取数值变量

# 标准化后做PCA
# scale. = TRUE 等价于在相关矩阵上做PCA
pca_result <- prcomp(X, scale. = TRUE)

# 查看结果
summary(pca_result)
# Importance of components:
#                           PC1    PC2     PC3     PC4
# Standard deviation     1.7084 0.9560 0.38309 0.14393
# Proportion of Variance 0.7296 0.2285 0.03669 0.00518
# Cumulative Proportion  0.7296 0.9581 0.99482 1.00000

# 查看载荷（变量在主成分上的权重）
pca_result$rotation

# 查看得分（每个样本在主成分上的坐标）
head(pca_result$x)

# 碎石图
library(factoextra)
fviz_eig(pca_result, addlabels = TRUE)

# 美观的biplot
fviz_pca_biplot(pca_result,
                col.ind = iris$Species,  # 按物种着色
                palette = "jco",
                repel = TRUE)

这里有两个关键的可视化：scree plot（碎石图）展示了每个主成分解释的方差比例，帮助我们决定保留多少个主成分；biplot同时展示了样本在主成分空间中的分布（散点）和原始变量对主成分的贡献（箭头），一张图就能看出变量之间的相关性、样本的分组模式，以及哪些变量驱动了哪些样本的分布。

小技巧：fviz_pca_biplot() 的 repel = TRUE 参数使用ggrepel避免标签重叠，生成的图形直接就是出版级别的质量。factoextra包的所有函数都返回ggplot2对象，你可以用 + theme()、+ labs() 随心定制。

因子分析在R中实现

R的stats包自带的 factanal() 是一个很好的起点，但它在功能上有些限制——比如不提供KMO和Bartlett检验，也不提供多种旋转方法的选项。这时候就要请出 psych 包了。psych包是芝加哥大学的William Revelle教授开发的，功能非常全面。

library(psych)

# 假设我们有一个心理量表数据，包含20个题目
# 先用KMO和Bartlett检验判断数据是否适合因子分析
# 以iris数据为例（虽然它不太适合因子分析，但方便演示）
KMO(cor(iris[, 1:4]))
# KMO值 > 0.6 勉强可以接受，> 0.8 很好

cortest.bartlett(cor(iris[, 1:4]), n = nrow(iris))
# Bartlett检验显著（p < 0.05）说明相关矩阵不是单位矩阵

# 用psych::fa做因子分析
# nfactors = 2：指定提取2个因子
# rotate = "varimax"：方差最大正交旋转
fa_result <- fa(iris[, 1:4], nfactors = 2, rotate = "varimax", fm = "ml")
print(fa_result, cut = 0.3, sort = TRUE)

# 查看因子载荷矩阵
fa_result$loadings

# 绘制因子载荷图
fa.diagram(fa_result)

# 因子得分
fa_result$scores

这里要特别注意 fm 参数（factor method）。psych::fa支持多种因子提取方法：ml（最大似然法）、pa（主轴法）、ols（最小二乘法）、minres（最小残差法）等。最大似然法有统计推断的优势（可以做显著性检验），但对多元正态性假设比较敏感；主轴法在小样本下更稳健。一般建议先用最大似然法，如果结果不理想再换主轴法。

聚类分析在R中实现

聚类分析在R中的实现家族非常庞大。我们从最经典的开始：

# === 层次聚类 ===
# 计算距离矩阵
dist_mat <- dist(iris[, 1:4], method = "euclidean")

# 使用Ward方法聚类
hc <- hclust(dist_mat, method = "ward.D2")

# 绘制树状图
plot(hc, labels = iris$Species, main = "层次聚类树状图")
rect.hclust(hc, k = 3, border = 2:4)  # 画出三个簇的矩形框

# === K-means聚类 ===
set.seed(42)
km <- kmeans(iris[, 1:4], centers = 3, nstart = 25)

# 查看聚类结果与真实标签的交叉表
table(km$cluster, iris$Species)

# === PAM (Partitioning Around Medoids) ===
library(cluster)
pam_result <- pam(iris[, 1:4], k = 3)
summary(pam_result)

# === 可视化聚类结果 ===
library(factoextra)
fviz_cluster(km, data = iris[, 1:4],
             ellipse = TRUE,
             ellipse.type = "norm",
             palette = "jco",
             ggtheme = theme_minimal())

层次聚类中的 method = "ward.D2" 值得多说一句。Ward方法的目标是使簇内离差平方和最小化，是一种非常有效的层次聚类策略。但R中有两个版本："ward.D"（老版本）和 "ward.D2"（正确版本），区别在于距离的平方处理——几乎在所有情况下你都应该用 "ward.D2"。

判别分析在R中实现

判别分析的主力包是 MASS（Modern Applied Statistics with S），这是Venables和Ripley的经典配套包。Fisher判别（LDA）和二次判别（QDA）的核心函数就是 lda() 和 qda()。

library(MASS)
library(caret)  # 用于交叉验证

# 以iris数据为例：用4个数值变量预测物种
set.seed(42)

# 划分训练集和测试集
train_idx <- sample(1:nrow(iris), size = 100)
train <- iris[train_idx, ]
test <- iris[-train_idx, ]

# 线性判别分析
lda_model <- lda(Species ~ ., data = train)
lda_model
# 可以看到LD1和LD2两个判别函数

# 预测
lda_pred <- predict(lda_model, test)
table(lda_pred$class, test$Species)
# 计算准确率
mean(lda_pred$class == test$Species)

# 可视化判别空间
library(ggplot2)
lda_df <- data.frame(
  LD1 = lda_pred$x[, 1],
  LD2 = lda_pred$x[, 2],
  Species = test$Species
)
ggplot(lda_df, aes(x = LD1, y = LD2, color = Species)) +
  geom_point(size = 3) +
  stat_ellipse() +
  theme_minimal()

# 用caret做5折交叉验证
ctrl <- trainControl(method = "cv", number = 5)
lda_cv <- train(Species ~ ., data = iris,
                method = "lda",
                trControl = ctrl)
lda_cv$results
# Accuracy 通常在 0.97-0.98 之间

注意 predict() 返回的是一个列表，包含三个元素：class（预测类别）、posterior（后验概率矩阵）、x（判别得分）。后验概率非常有用——它不仅告诉你"分到哪一类"，还告诉你"有多大把握"，这对于风险评估类任务（比如信用评分）至关重要。

CCA在R中实现

典型相关分析在R中有多个包可以实现。我们介绍两个最常用的：CCA 和 yacca。

# 安装并加载 CCA 包
# install.packages("CCA")
library(CCA)

# 构造一个两组变量的数据
# 假设我们研究"学习习惯"和"考试成绩"两组变量的关系
# 这里用iris做演示：两组变量分别为花瓣相关和花萼相关
X <- iris[, 1:2]  # 花萼长度、花萼宽度
Y <- iris[, 3:4]  # 花瓣长度、花瓣宽度

# 计算典型相关
cc_result <- cc(X, Y)

# 典型相关系数
cc_result$cor
# [1] 0.9406 0.7771

# 典型载荷（Canonical loadings）
cc_result$xcoef  # X组的典型权重
cc_result$ycoef  # Y组的典型权重

# 显著性检验
library(CCP)
# Wilks' Lambda检验
p.asym(cc_result$cor, n = nrow(iris), p = 2, q = 2)

# 用yacca包
library(yacca)
yacca_result <- cca(X, Y)
summary(yacca_result)
# yacca提供了更详细的输出，包括冗余指数（redundancy index）
# 冗余指数告诉你每个典型变量能解释对方组变量的方差比例

CCA的结果解读需要三个层次：第一，典型相关系数的绝对值（多大程度上相关）；第二，典型相关系数的显著性（Welk's Lambda检验的p值）；第三，冗余指数（实际解释了多少变异）。前两个是"是否相关"，第三个是"相关有多大用"——学界有个共识，仅仅报告典型相关系数是不够的，因为即使相关系数高达0.9，如果冗余指数很低，说明这一对典型变量解释的实际变异非常有限。

踩坑故事：R版本不兼容导致factoextra可视化全崩

最后一节，讲一个让我深夜崩溃的踩坑经历。

去年我做一份市场调研报告，需要用factoextra包画出漂亮的PCA biplot和聚类fviz_cluster图。我的代码在本地R 4.2上跑得好好的，但当我部署到客户的服务器上（R 3.6）时，所有factoextra的可视化都报错——Error: Invalid input: date_trans works with objects of class Date only。我反复检查数据，没有任何Date类型变量。

花了三个小时才发现原因：factoextra底层依赖ggplot2，而ggplot2从3.4版本开始改了scales包中的日期处理逻辑。R 3.6安装的ggplot2版本太旧（3.3.x），与最新版factoextra的某些新特性不兼容。解决方法是在R 3.6上安装旧版本的factoextra：remotes::install_version("factoextra", version = "1.0.7")。

教训：R的包依赖管理是出了名的"牵一发而动全身"。在跨环境部署R项目时，强烈建议使用 renv 包来锁定依赖版本，或者使用Docker容器来保证环境一致性。永远不要假设"同样的代码在另一个机器上也能跑"——R包的版本地狱是真实存在的。

本章小结与衔接

本章我们完整地走了一遍如何在R中实现五大核心多元分析方法：用 prcomp() 做PCA并用factoextra可视化；用 psych::fa() 做主/因子分析；用 hclust()、kmeans() 和 pam() 做聚类分析；用 MASS::lda() 做判别分析；用 CCA::cc() 做典型相关分析。每个方法我们都给出了可复现的代码和关键参数的解释。

从下一章开始，我们将把这些方法迁移到Python生态中——用scikit-learn、statsmodels和factor_analyzer来实现同样的分析流程。如果你是一个Python用户，你会看到scikit-learn的API设计是多么统一和优雅；如果你是一个R用户，你会发现Python在工程化方面的独特优势。
