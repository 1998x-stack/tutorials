第12章 骨干网络对比

大家好，我是你们的讲师。backbone是检测器的骨架——它决定了模型能"看到"什么样的特征。从Darknet到CSPDarknet再到C2f，每一次骨干网络的升级都带来了检测能力的质变。这节课我们就像逛"模型动物园"一样，把YOLO系列中用过的骨干网络逐个过一遍。

骨干网络（backbone）的任务是对输入图像进行特征提取，生成一系列不同分辨率的特征图。它的设计直接决定了模型的计算量、参数量以及特征表示能力。

12.1 Darknet-19：YOLOv2的基石

Darknet-19是YOLOv2使用的骨干网络，由19个卷积层和5个最大池化层组成。它的设计是当时"简单暴力"的典范：

# Darknet-19 核心结构
Layer Type        Filters  Size/Stride  Output
Convolutional     32       3×3          224×224
Maxpool                      2×2/2       112×112
Convolutional     64       3×3          112×112
Maxpool                      2×2/2        56×56
Convolutional     128      3×3           56×56
Convolutional     64       1×1           56×56
Convolutional     128      3×3           56×56
Maxpool                      2×2/2        28×28
# ... 后续层逐渐增加通道数，减少空间尺寸
# 最终输出 a 7×7×1024 的特征图

Darknet-19在设计上的特点是大量使用 1×1卷积来压缩通道数（bottleneck设计），在保持特征表达能力的同时减少计算量。虽然放在今天看它的能力有限，但在2016年，它用不到2000万的参数量就达到了当时领先的检测精度。

12.2 Darknet-53：YOLOv3的跃进

Darknet-53是YOLOv3的骨干网络，名字里的"53"表示它有53个卷积层。它借鉴了ResNet的残差连接思想，解决了深层网络的梯度消失问题。

与Darknet-19相比，Darknet-53的改进体现在：

  • 残差块： 每个卷积组后面跟一个跳跃连接，让梯度可以直接回传到浅层
  • 无池化层： 用步长为2的卷积替代池化进行下采样，让网络自己学习下采样方式
  • 更强的特征表示： 参数量约4100万，是Darknet-19的两倍多

在ImageNet分类任务上，Darknet-53的top-1准确率比ResNet-152略低，但推理速度快了约2倍——这正是YOLO系列追求的核心：在精度和速度之间找到最佳平衡点。

12.3 CSPDarknet-53：YOLOv4的精华

YOLOv4使用的CSPDarknet-53在Darknet-53的基础上引入了Cross Stage Partial（CSP）结构。CSP的核心思想是将特征图分成两部分，一部分经过密集的卷积块处理，另一部分直接与处理后的结果拼接：

# CSP块的核心逻辑
def csp_block(x, num_blocks, filters):
    # 将输入沿通道分成两部分
    x1 = x[:, :filters//2, :, :]    # Part 1: 直接传递
    x2 = x[:, filters//2:, :, :]    # Part 2: 经过密集块处理

    # Part 2经过一系列卷积和残差块
    y2 = dense_blocks(x2, num_blocks)

    # 两部分拼接
    y = torch.cat([x1, y2], dim=1)
    return y

CSP结构的优势：

  • 减少计算量： 只有一半的特征经过密集计算，FLOPs减少了约20%
  • 缓解梯度重复： 标准残差网络中，大量梯度信息在反向传播时被重复使用，CSP通过跨阶段分割避免了这种情况
  • 增强特征复用： 直接传递的那部分特征保留了更原始的梯度信息
骨干网络 参数量 FLOPs（G） ImageNet Top-1 YOLO版本 主要创新
Darknet-19 20.8M 5.6 74.1% v2 1×1 bottleneck
Darknet-53 41.6M 18.7 77.2% v3 残差连接
CSPDarknet-53 27.6M 13.0 79.2% v4 CSP结构
EfficientRep 37.3M 12.4 79.8% v6 重参数化

12.4 EfficientRep：可重参数化的骨干

YOLOv6引入了EfficientRep骨干网络，它的设计目标是训练-部署解耦。核心思路是：训练时使用复杂的多分支结构以获得更好的特征表示，推理时通过结构重参数化将多分支等价转换为单分支卷积。

# RepVGG风格的训练时多分支块
class RepBlock(nn.Module):
    def __init__(self, in_channels, out_channels):
        self.conv_3x3 = Conv2d(in_channels, out_channels, 3, padding=1)
        self.conv_1x1 = Conv2d(in_channels, out_channels, 1)
        self.bn = BatchNorm(out_channels)  # 恒等分支（当in==out时）

    def forward(self, x):
        return self.conv_3x3(x) + self.conv_1x1(x) + self.bn(x)

    # 推理时融合为单个3×3卷积
    def reparameterize(self):
        # 将3×3, 1×1, BN分支融合成一个等效的3×3卷积
        fused_conv = fuse_conv_bn(self.conv_3x3, self.bn)
        return fused_conv

EfficientRep在推理时将所有分支融合成一个3×3卷积，使得部署后的推理速度非常快——在边缘设备上尤为明显。YOLOv6-nano在ARM CPU上的推理速度比同尺寸YOLOv5快了约30%。

12.5 C2f模块：YOLOv8的优雅设计

YOLOv8使用的C2f（Cross Stage Partial with 2 convolutions and f) 是CSP结构的进一步演进。C2f首先将输入特征通过1×1卷积分为两部分，其中一部分经过若干瓶颈块（Bottleneck）处理，最后再融合所有分支：

class C2f(nn.Module):
    def __init__(self, c1, c2, n=1, shortcut=True, e=0.5):
        self.cv1 = Conv(c1, 2 * c2_, 1, 1)     # 1×1卷积分裂通道
        self.cv2 = Conv((2 + n) * c2_, c2, 1)   # 融合所有分支
        self.m = nn.ModuleList([
            Bottleneck(c2_, c2_, shortcut) for _ in range(n)
        ])

    def forward(self, x):
        y = list(self.cv1(x).chunk(2, 1))       # 沿通道分割
        y.extend(m(y[-1]) for m in self.m)      # 每个bottleneck的输出都保留
        return self.cv2(torch.cat(y, 1))         # 拼接所有分支后融合

C2f的关键创新：它将CSP中中间的每一层bottleneck输出都拼接起来（而不仅仅是最后输出），提供了更丰富的梯度流路径。这让网络在参数量基本不变的情况下获得了更强的特征表达能力。

12.6 GELAN：YOLOv9前瞻

YOLOv9提出的GELAN（Generalized Efficient Layer Aggregation Network）是当前YOLO系列中最先进的骨干网络。它将CSPNet和ELAN的思想统一到一个更通用的框架中：

  • 可配置深度和宽度： 通过调整block数量、通道数等参数，可以在不同规模下保持最优的效率
  • 基于梯度路径规划： GELAN的设计考虑到了反向传播过程中梯度的流动路径，确保浅层也能收到有效的梯度
  • 计算效率极高： 在相同的FLOPs限制下，GELAN的检测精度超过了所有之前的YOLO骨干
骨干网络选择的实用建议：
  • 如果你追求部署效率（边缘设备、嵌入式），EfficientRep或轻量版CSPDarknet是最佳选择
  • 如果你追求开箱即用（标准GPU服务器、COCO/LVIS数据集），YOLOv8的C2f-CSPDarknet是成熟稳定的选择
  • 如果你希望达到当前SOTA精度（竞赛、高要求生产环境），YOLOv9的GELAN值得尝试
踩坑实录：骨干网络替换的"兼容性陷阱"。 曾经我尝试将YOLOv8的backbone替换为GELAN，以为直接换掉特征提取部分就能获得精度提升。结果发现：GELAN的下采样倍数和输出通道数与YOLOv8的neck设计不匹配，导致特征金字塔的输入尺寸对不上。修改了neck的通道数和上采样参数后，又发现loss开始震荡——因为不同骨干网络的初始化策略不同，需要重新调整学习率和warmup策略。最终花了3天时间调参才跑通。结论：替换骨干网络不是简单地"拔插"。

12.7 总结

从Darknet-19到GELAN，YOLO骨干网络的发展史可以概括为"更宽、更深、更高效"：更深的层数带来更强的特征抽象能力，CSP和C2f结构提高了计算效率，而重参数化技术则弥合了训练和部署之间的性能差距。选择哪个骨干网络，取决于你的硬件约束、精度要求和部署场景。
