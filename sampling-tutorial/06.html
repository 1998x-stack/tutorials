第6章 系统抽样

大家好，我是你们的讲师。想象一下这个场景：你有一份全市居民的户籍名单，排成了厚厚的一摞，你需要从中选出500人做调查。如果要用简单随机抽样，你需要给每户分配随机数、排序、选前500个——这在没有电脑的年代是一个极其繁琐的过程。但如果用一种更"机械"的方法呢？比如每隔50个名字抽一个？

这就是系统抽样（Systematic Sampling）的核心思想：按照固定的间隔（interval）从有序排列的总体中抽取样本。它比SRS操作起来方便得多，在很多情况下还能比SRS更高效——但它也有一个著名的陷阱，叫做周期性偏差（Periodic Bias）。如果不小心，系统抽样的结果可能比SRS差得多。

在这一章中，我们将详细讨论系统抽样的原理、三种排序方式、以及那个让无数调查人员栽过跟头的周期性偏差问题。

等距抽样的基本原理

操作步骤

系统抽样的操作非常简单，分为三步：

  • 步骤1：将总体N个个体按某种顺序排列（可以是自然顺序，也可以是按某个变量排序后的顺序）
  • 步骤2：计算抽样间隔 k = N/n（通常取整，但实际中可能会有细微调整）
  • 步骤3：在1到k之间随机选择一个起始点r，然后选取第 r, r+k, r+2k, ..., r+(n-1)k 个个体作为样本

例如，某学校有1000名学生，你要抽100人。则k=1000/100=10。在1到10中随机选一个数，比如选到7，则抽取学号为7、17、27、...、997的学生。整个过程不需要复杂的随机数表，执行起来极其简便。

为什么系统抽样如此流行？ 在实际调查中，系统抽样至少有三个优势使得它成为很多调查机构的首选：第一，它不需要完整的抽样框——你只需要知道当前这个位置有没有人、下一步该隔多远就行，不需要预先枚举所有个体（比如在街头拦截调查中，你定好"每隔5个人访问一个"）。第二，它的操作非常简单，访员经过简短培训就能掌握。第三，如果排列顺序与目标变量相关，它还能起到类似分层的效果——我们在下面详细说。

三种排序方式

系统抽样的效果高度依赖于总体是如何排序的。不同的排序方式会导致截然不同的统计性质：

1. 无关标志排序（近似SRS）

如果个体的排列顺序与目标变量完全无关（比如按姓氏拼音排序来调查收入——姓氏和收入没有任何关系），那么系统抽样就近似于SRS。它的方差估算也可以直接套用SRS的公式。这种情况下，系统抽样的主要优势是操作简便，而非统计效率提升。

2. 有关标志排序（隐含分层）

如果个体是按照与目标变量相关的顺序排列的（比如按入学成绩排序来调查大学四年的GPA——入学成绩和毕业GPA通常是正相关的），那么系统抽样就相当于一种"隐含的分层抽样"。

为什么呢？因为当总体按相关变量排序后，每连续的k个个体就构成了一个"隐含层"——它们彼此之间在排序变量上比较接近，因此目标变量的差异也相对较小。系统抽样在每个"隐含层"中恰好抽取一个个体（第r个位置），这和分层抽样中每层抽一个样本的精神非常相似。区别在于：分层抽样需要你明确知道分层变量并对每层独立抽一个样本；而系统抽样通过排序和等距抽取自动完成了这个效果。

对于有关标志排序，系统抽样的方差通常小于SRS。排序变量与目标变量的相关性越强，效率提升越明显。

3. 自然位置排序（便于操作）

很多时候，个体的排列顺序是自然位置——比如门牌号、产品流水线上的位置、地理上的空间顺序。这种排序既有操作上的便利（沿着一条路走、隔几户敲一次门），也可能隐含一些与目标变量的相关性（比如富人往往住在门牌号较大的区域）。自然位置排序的系统抽样在实际操作中最为常见，但它也是最容易出问题的一种——特别是当排序具有周期性的时候。

排序方式 与目标变量的关系 效率 vs SRS 典型应用
无关标志 无关（如姓氏拼音） ≈ SRS 管理名录、电话簿抽样
有关标志 相关（如按成绩排序） 显著优于SRS 教育评估、企业排名调查
自然位置 可能相关（如门牌号） 取决于具体场景 入户调查、田野调查、流水线质检

周期性偏差

如果说系统抽样有什么"阿喀琉斯之踵"，那一定是周期性偏差。当总体的排列存在某种周期性结构，且抽样间隔k恰好是周期长度的整数倍（或约数）时，系统抽样会产生严重的系统性偏差。

让我们通过几个经典案例来理解：

  • 案例1：军队人员调查——某调查机构对军营进行系统抽样，按花名册每隔k人抽一人。但军营的花名册是按班排列的（每班10人），每个班的排头通常是班长，最后一名通常是表现较差的士兵。如果k恰好等于10，那么系统抽样可能全部抽到排头（班长们），或者全部抽到排尾。结果完全无法代表整个部队。
  • 案例2：交通流量调查——如果在一个信号灯周期为60秒的路口，每隔60秒观测一次交通流量——那你可能每次都观测到红灯刚亮时的拥堵，或者每次都是绿灯畅行，取决于起始时间。观测结果会系统性地偏离真实车流状况。
  • 案例3：制造业质量检验——某个流水线上有4台机器轮流工作（按A-B-C-D-A-B-C-D的周期顺序）。如果抽样间隔k正好是4的倍数，那么可能每次都抽到同一台机器的产品——某台机器如果有故障，也永远发现不了。
周期性偏差的隐蔽性：周期性偏差最危险的地方在于——你通常不知道你的数据中是否存在周期性！想想看，如果你在做入户调查，按门牌号每隔5户抽一户——看起来很合理的方案。但如果你所在的城市是单双号分区的（路的一边是双号、另一边是单号），而你的抽样间隔恰好导致你只访问了马路一边的居民——马路两边可能住着完全不同的人群（一边是新建高档小区，一边是老旧小区）。这种偏差不是随机误差，而是系统性偏差——它不会随着样本量的增大而消失。要避免这个问题，最好的办法是：在确定抽样间隔之前，先了解一下总体的排列结构，检查是否存在已知的周期性模式。

方差估计

系统抽样的方差估计比SRS要复杂一些。问题在于：系统抽样只有一个随机要素——起始点r。一旦r确定了，整个样本就确定了。这意味着从重复抽样的角度看，系统抽样实际上只有k个可能的样本（对应r=1,2,...,k），而不是像SRS那样有C(N,n)种可能。

由于只有一个随机要素，系统抽样的方差不能像SRS那样简单地用样本方差除以√n来估算。常用的方法有：

  • 当作SRS处理：如果排序是无关标志的，直接用SRS公式——这是一个保守估计（通常会略微高估实际方差）。
  • 连续差法（Successive Difference Method）：利用相邻样本之间的差异来估计方差。
  • 重叠子样本法：将系统样本随机分成若干组（子样本），用组间差异来估计方差。
  • 重抽样方法：使用bootstrap或jackknife方法从单个系统样本中估计方差。
def systematic_sample(population, k, start):
    """系统抽样：从population中每隔k个抽一个，起始位置为start（0-indexed）"""
    return [population[i] for i in range(start, len(population), k)]

def systematic_variance_approx(sample):
    """
    系统抽样的方差近似（连续差法）
    注意：这只是其中一种近似方法
    """
    n = len(sample)
    if n < 2:
        return None
    # 计算相邻样本的差
    diffs = [sample[i+1] - sample[i] for i in range(n-1)]
    # 连续差法方差估计
    var_est = sum(d**2 for d in diffs) / (2 * (n - 1))
    return var_est / n

# 模拟：总体1000人，按与目标变量相关的顺序排列
import random, numpy as np
random.seed(42)
# 创建一个排序后的总体（与目标变量正相关）
population = sorted([random.gauss(50, 15) for _ in range(1000)])

k = 10
r = random.randint(0, k-1)  # 随机起始点
sample = systematic_sample(population, k, r)
se = np.sqrt(systematic_variance_approx(sample))
print(f"系统样本均值: {np.mean(sample):.2f}")
print(f"近似标准误: {se:.2f}")
print(f"总体真实均值: {np.mean(population):.2f}")
重要提醒：系统抽样的方差估计是抽样理论中较为棘手的领域之一。不要想当然地使用"标准差除以√n"的公式——那个公式只有在SRS下才是精确的。对于系统抽样，特别是当排序与目标变量相关时，用SRS公式通常会高估方差（即你报告的误差会比实际的大），这在统计上是保守的（偏保守通常没问题），但如果你的目标是在严格的预算下达到最小的置信区间宽度，高估方差就意味着你浪费了样本量。更危险的是相反的情况——当存在周期性且k选择不当时，SRS公式会严重低估方差，让你误以为结果比实际可靠得多。

真实案例：门牌号系统抽样的偏差

这是一个来自某城市社区调查的真实案例。调查团队计划在某个街道进行居民健康素养调查，采用系统抽样：按门牌号每隔5户抽取一户。听起来很合理？结果数据出来后，发现样本中的平均收入远高于该街道的整体水平。

调查团队百思不得其解。后来才发现问题出在哪里：该街道的房屋分布是单双号不对称的——路的一侧（双号）主要是2000年以后新建的高档商品房小区，住户多为白领和退休干部；另一侧（单号）主要是1980年代建成的老旧单位宿舍，住户多为退休工人和外来租户。抽样间隔k=5配合从单号开始，结果抽取的样本中大部分来自高档小区一侧。

更隐蔽的是：调查员并没有意识到门牌号两侧的社会经济差异，他们只是忠实地执行了"每隔5户敲一次门"的指令。直到数据分析阶段，当样本的人口结构明显偏离已知的街道人口统计数据时，他们才意识到出了问题。

这个问题的根本原因在于：门牌号的排列并非随机——它包含了地理位置、房屋类型、小区档次等信息。当这些信息与目标变量（健康素养、收入等）相关时，系统抽样就不仅仅是"操作简便的工具"，而是"承载了排序信息的设计"。你必须理解这个排序结构，才能正确地设计和分析系统抽样。

如何避免门牌号问题？ 在实践中，有经验的调查人员会采取这样的策略：如果要在一条路上做系统抽样，可以先将道路两侧视为两个层（分层的思想），然后在每一侧内部独立做系统抽样。或者，在确定抽样间隔后，先随机决定起点在哪一侧。如果你在数据分析阶段才发现这个问题，也可以通过事后加权（Post-stratification weighting）来校正——但这需要你准确知道两侧各自的人口比例，且校正只能消除偏差对均值估计的影响，无法恢复方差估计的准确性。

系统抽样 vs SRS：怎么选？

在实际工作中，系统抽样往往是比SRS更好的选择——前提是：

  • 总体可以按某种顺序排列，且该顺序与目标变量有一定相关性（此时系统抽样效率更高）
  • 操作简便性很重要（现场调查中系统抽样比SRS容易执行得多）
  • 不存在明显的周期性结构，或者你采取了措施避免周期性偏差

系统抽样的主要风险包括：

  • 周期性偏差——当抽样间隔与总体排列的周期重合时
  • 方差估计困难——不像SRS那样有精确的公式
  • 如果总体排列中存在趋势（trend），系统抽样的方差估计可能会偏大或偏小

本章小结

系统抽样是一种操作简便、在实践中被广泛使用的抽样方法。通过不同的排序方式，它可以达到近似SRS的效果（无关标志排序），也可以达到隐含分层的高效效果（有关标志排序）。但它最大的隐患在于周期性偏差——当总体排列存在周期性且抽样间隔选择不当时，系统抽样的结果可能严重失真。在实际使用中，理解你的数据排序结构、检查可能的周期性模式、使用适当的方差估计方法，是做好系统抽样的关键。
