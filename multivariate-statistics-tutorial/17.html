开篇：从"物以类聚"到"按类识别"

大家好，欢迎继续学习。前两章我们一直沉浸在聚类分析的世界里——我们学习了如何在没有标签的情况下发现数据中的自然分组。现在我们要转换视角：如果已知类别标签，如何根据特征变量来预测一个新样本属于哪个类别？这就是判别分析 (Discriminant Analysis) 要解决的问题。

判别分析在现实中的应用极其广泛：银行用它来判断一笔交易是否欺诈（正常 vs 欺诈）；医院用它根据患者的化验指标来诊断疾病（患病 vs 健康）；招聘平台用它预测一个候选人会不会接受offer。本质上，一切"根据已有数据做分类决策"的问题，都可以用判别分析来解决。

判别分析的方法论可以追溯到1936年费希尔 (R. A. Fisher) 的经典论文——他用虹膜（鸢尾花）的数据集，仅凭花瓣长度和宽度两个变量设计了一个"判别函数"，就能几乎完美地区分三种鸢尾花。这个数据集 (iris) 后来成为整个机器学习的"Hello World"。本章我们将从最基础的距离判别开始，逐步深入到贝叶斯判别框架。

判别分析与聚类的区别

先明确一个定位问题。很多初学者分不清"聚类"和"判别"的区别——都是做"分类"，到底有什么不同？下面是两者的根本差异：

维度聚类分析判别分析
学习范式无监督学习 (Unsupervised)有监督学习 (Supervised)
数据要求只需要特征 X，不需要标签 y需要标注好的训练数据 (X, y)
目标发现数据中未知的分组结构学习从 X 到 y 的分类规则
输出簇标签（任意编号）类别预测（对应已知的类别）
评估难以客观验证（无真实标签）可以用验证集计算准确率
典型案例客户分群、基因表达模式发现疾病诊断、信用评分、图像识别

简单说：聚类是"找规律"——没人告诉你怎么分，你自己看着办；判别是"学规矩"——有人给了你一堆"正确答案"，你学会规则后去给新数据做判断。

距离判别

距离判别是最直观的判别方法，它的思想简单到令人发指：一个样本离哪个类的中心最近，就把它判给哪个类。

最小距离分类器

假设有 K 个类，每个类的中心是 μₖ（通常用训练样本的均值估计）。对于一个新样本 x，计算它到每个类中心的欧氏距离：

# 最小距离分类器 (欧氏距离)
dₖ(x) = ||x - μₖ||² = (x - μₖ)ᵀ(x - μₖ)

# 预测类别 = argmin_k dₖ(x)

# 问题：欧氏距离假设所有变量同等重要
# 如果变量量纲不同（如"收入"和"年龄"），
# 量纲大的变量会在距离计算中占据主导

这个方法的致命弱点就是它不考虑变量的方差和相关性。如果变量"收入"的波动范围是 0-10 万，而"年龄"是 0-100，那么距离几乎完全由收入决定。

马氏距离分类器

为了解决这个问题，我们把马氏距离 (Mahalanobis Distance) 引入判别分析。马氏距离通过协方差矩阵对变量进行"白化" (whitening) 处理，使所有变量在同一尺度上可比：

# 马氏距离分类器
dₖ_M(x) = (x - μₖ)ᵀ Σ⁻¹ (x - μₖ)

# 其中 Σ 是合并协方差矩阵（所有类共用一个协方差矩阵）
# 或者各类分别用各自的协方差矩阵 Σₖ

# 展开来看：
# dₖ_M(x) = (x - μₖ)ᵀ Σ⁻¹ (x - μₖ)
#         = xᵀΣ⁻¹x - 2xᵀΣ⁻¹μₖ + μₖᵀΣ⁻¹μₖ
#         = 常数 + 线性项 xᵀΣ⁻¹μₖ

# 忽略与 class 无关的 xᵀΣ⁻¹x 项：
# 判别函数 δₖ(x) = xᵀΣ⁻¹μₖ - (1/2)μₖᵀΣ⁻¹μₖ
# 预测类别 = argmax_k δₖ(x)

马氏距离分类器在数学上等价于线性判别分析 (LDA)，前提是假设各类的协方差矩阵相等。在下一节我们会更详细地推导这一点。

马氏距离的几何直觉：想象数据分布是一个椭圆形（在二维中）。如果某个方向的数据变异更大（椭圆的长轴方向），那么同样沿这个方向偏移一段距离，实际上"更不显著"。马氏距离通过对协方差矩阵求逆来"压扁"高变异方向、"拉伸"低变异方向，从而得到一个各向同性的距离度量。在标准化后的空间中，所有方向同等重要。

线性判别分析（LDA）

线性判别分析 (Linear Discriminant Analysis, LDA) 是判别分析中最经典的方法。费希尔提出的版本是从一个几何问题出发——"找一个投影方向，使得投影后各类的均值尽可能分开、同时各类内部的散布尽可能小"。这个思想后来被称为 Fisher's LDA。

Fisher 的优化目标可以表达为"最大化瑞利商"：

# Fisher LDA 的目标：最大化广义瑞利商
max_a  J(a) = (aᵀ S_B a) / (aᵀ S_W a)

# S_B = 类间散布矩阵 (between-class scatter)
# S_W = 类内散布矩阵 (within-class scatter)
# a = 投影向量

# S_B = Σₖ nₖ (μₖ - μ)(μₖ - μ)ᵀ
# S_W = Σₖ Σ_{i∈Cₖ} (xᵢ - μₖ)(xᵢ - μₖ)ᵀ

# 解：a 是 S_W⁻¹ S_B 的最大特征值对应的特征向量

当只有两个类别时，Fisher LDA 的判别函数是线性的：一条直线（在二维空间中）或一个超平面（在高维空间中）将特征空间分成两部分。

在 LDA 的统计学框架下（各类服从多元正态分布，且协方差矩阵相等），判别函数有简洁的线性形式：

# LDA 判别函数（二分类）
δ₁(x) = xᵀΣ⁻¹μ₁ - (1/2)μ₁ᵀΣ⁻¹μ₁ + ln(π₁)
δ₂(x) = xᵀΣ⁻¹μ₂ - (1/2)μ₂ᵀΣ⁻¹μ₂ + ln(π₂)

# 分类规则：如果 δ₁(x) > δ₂(x)，判为类1，否则判为类2
# 决策边界（令 δ₁(x) = δ₂(x)）是一个超平面：
# xᵀΣ⁻¹(μ₁ - μ₂) = (1/2)(μ₁ᵀΣ⁻¹μ₁ - μ₂ᵀΣ⁻¹μ₂) + ln(π₂/π₁)

# 其中 πₖ 是第 k 类的先验概率

注意决策边界是线性的——这正是"线性判别分析"中"线性"二字的来源。

二次判别分析（QDA）

LDA 有一个关键假设：所有类的协方差矩阵相等，即 Σ₁ = Σ₂ = ... = Σₖ。如果这个假设不成立，LDA 的线性决策边界就不再是最优的。这时我们需要二次判别分析 (Quadratic Discriminant Analysis, QDA)。

QDA 允许每个类别有自己的协方差矩阵 Σₖ。判别函数变为：

# QDA 判别函数
δₖ(x) = -(1/2)ln|Σₖ| - (1/2)(x-μₖ)ᵀΣₖ⁻¹(x-μₖ) + ln(πₖ)

# 去掉二次项后的决策边界是二次曲面（椭圆、抛物线等）
# 分类规则：argmax_k δₖ(x)

# QDA 和 LDA 对比：
# LDA：p 个变量，待估参数数 ≈ Kp + p(p+1)/2
# QDA：p 个变量，待估参数数 ≈ Kp + K·p(p+1)/2
# QDA 的参数是 LDA 的约 K 倍，需要更多训练数据
维度LDAQDA
协方差假设各类协方差矩阵相等各类可以有不同的协方差矩阵
决策边界线性（超平面）二次（椭圆/抛物线）
参数数量少（偏差大、方差小）多（偏差小、方差大）
小样本表现较好（假设强但稳定）较差（参数多，易过拟合）
大样本表现可能欠拟合（太简单）更好（更灵活）

实践中如何选择 LDA 和 QDA？一个简单的规则：如果训练样本量较少（相对于变量数），或者有理由相信各类的分布形状相似，选 LDA。如果样本量充裕，或者散点图显示各类的散布形状明显不同，选 QDA。当然，你也可以用交叉验证来比较两者的表现。

Bayes判别

距离判别和 LDA/QDA 其实都是贝叶斯判别 (Bayes Discriminant) 框架的特殊情况。贝叶斯判别提供了一个统一的概率视角，适用于所有分类问题。

贝叶斯判别的核心是后验概率 (posterior probability)：给定观测数据 x，样本属于第 k 类的概率有多大？

# 贝叶斯公式
P(类k | x) = P(x | 类k) · P(类k) / P(x)

# 其中：
# P(类k) = πₖ 是第 k 类的先验概率
# P(x | 类k) = fₖ(x) 是第 k 类的类条件密度
# P(x) = Σⱼ fⱼ(x)πⱼ 是边缘密度（归一化常数）

# 贝叶斯分类规则：
# 选择后验概率最大的类
# 等价于选择 fₖ(x)πₖ 最大的类

# 如果 fₖ(x) 取多元正态密度 N(μₖ, Σₖ)：
# fₖ(x) = (1/((2π)^{p/2}|Σₖ|^{1/2})) exp(-(1/2)(x-μₖ)ᵀΣₖ⁻¹(x-μₖ))
# 代入取对数，去掉常数项，就得到了 LDA 或 QDA 的判别函数

贝叶斯框架的优雅之处在于它自然地将先验知识和错分代价纳入了分类决策：

  • 先验概率 πₖ：如果你事先知道某个类别的样本更多（比如正常交易占 99%，欺诈仅占 1%），可以在先验中体现这一信息
  • 错分代价：如果第 1 类错分为第 2 类的代价是第 2 类错分为第 1 类的 10 倍，可以调整决策阈值——即使后验概率 P(类1|x) 小于 P(类2|x)，只要损失期望更小，也判为类1

贝叶斯判别规则在理论上是最优的——它最小化了总的期望错分代价。这个结论不依赖于数据的分布假设，但前提是已知真实的类条件密度 fₖ(x) 和先验概率 πₖ。在实践中，fₖ(x) 和 πₖ 都需要从数据中估计，所以实际效果取决于估计的准确性。

理解贝叶斯判别的关键：所有的判别方法——距离判别、LDA、QDA——都可以纳入贝叶斯框架。当你用欧氏距离时，你隐含地假设了各类协方差相等且为球形（单位矩阵）；当你用马氏距离时，你假设各类协方差相等但不一定是球形（LDA）；当各类有自己的协方差时，就得到了 QDA。换句话说，不同判别方法之间的区别，本质上是对类条件密度 fₖ(x) 的不同假设。

判别规则的评估

判别模型建立后，最关键的问题是：这个模型到底有多准？评估分类模型的核心工具是混淆矩阵 (confusion matrix)：

预测为正类预测为负类
实际为正类TP (True Positive)FN (False Negative)
实际为负类FP (False Positive)TN (True Negative)

从混淆矩阵可以派生出多种评估指标：

  • 准确率 (Accuracy) = (TP + TN) / (TP + TN + FP + FN)——整体正确分类比例
  • 精确率 (Precision) = TP / (TP + FP)——预测为正类的样本中有多少是对的
  • 召回率 (Recall / Sensitivity) = TP / (TP + FN)——真实正类中有多少被找出来了
  • 特异度 (Specificity) = TN / (TN + FP)——真实负类中有多少被正确排除
  • F1分数 = 2 · Precision · Recall / (Precision + Recall)——精确率和召回率的调和平均

需要注意的是：不要用训练数据来评估模型的准确率——模型"见过"的样本总是表现得更好，这叫"过拟合乐观"。正确的做法是用交叉验证：将数据分成 K 份，用 K-1 份训练、1 份验证，重复 K 次后取平均准确率。最常用的是留一交叉验证 (LOOCV: Leave-One-Out Cross-Validation)，即每个样本单独留出一次作为验证集。

实战案例：iris物种判别

费希尔经典的鸢尾花 (iris) 数据集包含 150 个样本，3 个物种（setosa、versicolor、virginica），每个样本有 4 个特征（花萼长度、花萼宽度、花瓣长度、花瓣宽度）。我们用 LDA 和 QDA 分别做分类，比较两者的表现。

import numpy as np
import pandas as pd
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis, QuadraticDiscriminantAnalysis
from sklearn.model_selection import cross_val_score
from sklearn.datasets import load_iris

# 加载数据
iris = load_iris()
X, y = iris.data, iris.target

# LDA 分类
lda = LinearDiscriminantAnalysis()
lda_scores = cross_val_score(lda, X, y, cv=10)  # 10折交叉验证
print(f"LDA 平均准确率: {lda_scores.mean():.3f}")

# QDA 分类
qda = QuadraticDiscriminantAnalysis()
qda_scores = cross_val_score(qda, X, y, cv=10)
print(f"QDA 平均准确率: {qda_scores.mean():.3f}")

# 查看 LDA 的判别函数系数
lda.fit(X, y)
print(f"判别函数个数: {lda.n_components}")  # 最多 K-1 = 2 个
print(f"第一个判别函数的系数: {lda.coef_[0]}")

通常情况下，LDA 和 QDA 在 iris 数据上都能达到 97% 以上的准确率——因为三个物种在花瓣维度上分离得很好。唯一的混淆发生在 versicolor 和 virginica 之间，两者有少量重叠区域。

有趣的是，如果只用两个特征（花瓣长度和花瓣宽度），LDA 已经能取得约 96% 的准确率。这说明 Fisher 当年选择的这两个特征确实是区分鸢尾物种的"黄金特征"。这个例子告诉我们：在判别分析中，特征选择比模型选择更重要。好的特征让简单的模型也能取得优秀的表现。

踩坑故事：马氏距离并非万能

几年前我在做一个信用评分项目，需要根据用户的收入、负债、信用历史长度、近期查询次数等变量，用 LDA 区分"好客户"和"坏客户"。工程师团队已经做好了数据预处理和标准化，我心想万事俱备，直接跑了一个 LDA——结果在测试集上准确率只有 63%，几乎和抛硬币差不多。

我开始逐步排查。先检查了各类的均值——好客户和坏客户在大多数变量上确实有统计显著的差异。再检查变量分布——基本对称，没有严重的异常值。最后我画了散点图矩阵——这一看问题就暴露了。好客户和坏客户在各个变量上的分布形状完全不同！比如好客户的"收入"分布是右偏的（大部分人收入中等，少数高收入），而坏客户的收入分布几乎是均匀的。更关键的是，两类客户的协方差矩阵差异很大——好客户群体更"紧凑"，坏客户群体更"分散"。LDA 用一个共享的协方差矩阵去拟合两个形状完全不同的群体，预测效果自然很差。

教训：使用 LDA 前一定要检查各类的协方差矩阵是否大致相等。在这个项目中，我后来改用 QDA（允许各类有自己的协方差矩阵），交叉验证准确率提升到了 82%。LDA 假设各类协方差相等不是一句空话——如果这个假设严重不成立，LDA 的效果会大打折扣。检查方法很简单：分别计算各组的协方差矩阵，比较对角线元素（方差）是否在一个量级。如果有疑问，可以运行 Box's M 检验。记住一句话："LDA 在各类形状相似时才是最佳选择——如果各类的形状截然不同，交给 QDA 吧。"
