大家好，我是你们的讲师。今天我们来处理信息论中一个微妙的转变：从离散到连续。

想一想温度计。如果一支温度计显示 28.7°C，另一支精度更高的显示 28.75°C，再换一支更精密的显示 28.753°C……理论上，一个连续值的精确描述需要无限多位二进制数字。这意味着什么？——如果严格按之前离散熵的定义，任何连续信源的熵都是无穷大。但我们的直觉告诉我们，一个温度为 28°C 的房间和另一个温度为 30°C 的房间，"携带的信息量"显然是不一样的。

香农引入了一个新的概念来解决这个矛盾——微分熵（Differential Entropy）。它不再是"描述一个随机变量所需的最小比特数"（对连续变量来说，这个值是无穷大），而是"相对某一参考精度下的信息量"。它可以为负，可以无穷大，它失去了离散熵的某些直观解释——但它是在连续世界中描述信息流的一个不可或缺的工具。

更重要的是，微分熵是理解高斯信道容量的数学基础。著名的香农公式 C = ½ log₂(1 + P/σ²) 就是从微分熵推导出来的。这些概念在现代通信系统中每时每刻都在被使用——你的手机、Wi-Fi路由器和卫星通信都在这个框架下运作。

微分熵的定义与性质

对于连续随机变量 X，其概率密度函数为 f(x)，微分熵定义为：

h(X) = -∫ f(x) log₂ f(x) dx

形式上，它和离散熵 H(X) = -Σ p(x) log₂ p(x) 完全一样——只是把求和换成了积分。但两者的数学性质有根本性的区别。

最让人困惑的一点是：微分熵可以为负。离散熵永远是 ≥ 0 的，因为 -p log p ≥ 0。但连续变量的概率密度可能大于 1（例如在很窄的区间上的均匀分布），导致 -f(x) log f(x) 为负——积分结果也就可能为负。

举个具体的例子：考虑在区间 [0, ¼] 上的均匀分布 U(0, ¼)，其概率密度 f(x) = 4（因为积分必须为1）。那么：

h(X) = -∫₀^{¼} 4 · log₂(4) dx = -4 · 2 · ¼ = -2 bits

微分熵为 -2 bits！这听起来很奇怪——"负的信息量"？其实，微分熵的基准是"一个在单位区间上的均匀分布"，它的微分熵 h = 0。如果你分布在一个 更窄 的区间（不确定性变小了），微分熵为负；如果分布在一个更宽的区间，微分熵为正。

关键理解：微分熵为负不意味着"负信息"——它只是说在某个精度分辨率下，描述这个连续变量需要的比特数可以小于零（因为连续值本身已经无限精确了，这个"零"是相对于参考精度而言的）。不要试图把离散熵的"最小平均编码长度"解释套用到微分熵上。

微分熵有几个重要的性质：

  • 平移不变性：h(X+c) = h(X) —— 整体平移不改变微分熵
  • 伸缩变换：h(aX) = h(X) + log₂|a| —— 缩放会改变微分熵
  • 联合微分熵：h(X,Y) = -∬ f(x,y) log₂ f(x,y) dx dy
  • 条件微分熵：h(X|Y) = h(X,Y) - h(Y)
  • 链式法则：h(X₁,...,Xₙ) = Σ h(Xᵢ | X₁,...,Xᵢ₋₁)

注意伸缩变换的物理含义：如果你把一个随机变量放大两倍（比如从"米"换成"英尺"作单位，但数值上放大），它的不确定性也增加了——需要更多比特来描述。"你是1.8米"和"你是5.9英尺"说的其实是同一件事，但用英尺测量时数值变化更大（因为单位更小）——这正是 log₂|a| 项在起作用。

最大熵分布

在离散情况下，我们学过：给定一个字母表大小 m，均匀分布的熵最大。在连续情况下，同样的思路——给定一些约束，哪个分布使微分熵最大？——但这个问题的答案更丰富，也更有用。

核心思想是：最大熵分布就是在给定约束下，"最随机"或"最没有偏见的"分布。这也是我们下一章会讨论的最大熵原理的基础。

约束条件 最大熵分布 微分熵 h(X)
支撑集 [a, b] U(a, b) 均匀分布 log₂(b−a)
均值 λ > 0, 支撑集 [0, ∞) Exp(λ) 指数分布 log₂(e·λ) + 1
方差 σ² N(0, σ²) 高斯分布 ½ log₂(2πeσ²)
均值 μ, 方差 σ² N(μ, σ²) 高斯分布 ½ log₂(2πeσ²)
均值 μ, 绝对值一阶矩 E[|X−μ|] = β Laplace 分布 log₂(2eβ)

最经典的结论：在固定方差的约束下，高斯分布具有最大微分熵。这个结果用到了变分法和拉格朗日乘子法，证明思路是在约束 ∫f(x)dx=1 和 ∫x²f(x)dx = σ² 下最大化 -∫f(x)log f(x)dx。

这个结论的重要性体现在哪里？——因为噪声的方差通常是一个自然约束（功率受限），而高斯分布是"最坏的噪声"（即产生最大不确定性的噪声）。这意味着，在设计通信系统时，如果考虑的是高斯噪声的极限情况，你就在考虑最坏情况——如果系统能对抗高斯噪声，它对其他噪声就更不在话下。

用 Python 计算几种常见分布的微分熵：

import numpy as np
from scipy import stats
import math

def differential_entropy(dist, *args):
    """用数值积分计算指定分布的微分熵"""
    if dist == 'uniform':
        a, b = args
        return math.log2(b - a)
    elif dist == 'gaussian':
        mu, sigma = args
        return 0.5 * math.log2(2 * math.pi * math.e * sigma**2)
    elif dist == 'exponential':
        lam = args[0]  # rate parameter
        return math.log2(math.e * (1/lam)) + 1
    elif dist == 'laplace':
        mu, b = args
        return math.log2(2 * math.e * b)

print("=== 不同分布的微分熵 ===")
print(f"U(0,1):  h = {differential_entropy('uniform', 0, 1):.3f} bits")
print(f"U(0,0.1): h = {differential_entropy('uniform', 0, 0.1):.3f} bits")
print(f"N(0,1):  h = {differential_entropy('gaussian', 0, 1):.3f} bits")
print(f"N(0,2):  h = {differential_entropy('gaussian', 0, np.sqrt(2)):.3f} bits")
print(f"Exp(1):  h = {differential_entropy('exponential', 1):.3f} bits")
print(f"Laplace(0,1): h = {differential_entropy('laplace', 0, 1):.3f} bits")
# 输出:
# U(0,1):  h = 0.000 bits
# U(0,0.1): h = -3.322 bits
# N(0,1):  h = 2.047 bits
# N(0,2):  h = 2.547 bits
# Exp(1):  h = 1.443 bits
# Laplace(0,1): h = 2.443 bits

高斯信道与香农极限

终于来到了通信理论中最著名的公式。考虑一个加性高斯白噪声（AWGN）信道：

Y = X + N

其中 X 是发送信号，N 是独立的高斯噪声 N ~ N(0, σ²)，发送功率受约束 E[X²] ≤ P。

信道容量 C 是互信息的最大值：

C = max_{p(x): E[X²] ≤ P} I(X; Y)

由于 I(X;Y) = h(Y) - h(Y|X) = h(Y) - h(N)，而 h(N) = ½ log₂(2πeσ²) 是固定的（与 X 的分布无关）。为了最大化 I(X;Y)，我们需要最大化 h(Y)。

现在，Y = X + N，方差为 Var(Y) = Var(X) + Var(N) = P + σ²（因为 X 和 N 独立）。在固定方差 P + σ² 下，使微分熵最大的是高斯分布。因此最优的 X 分布也是高斯分布 X ~ N(0, P)，使得 Y ~ N(0, P + σ²)。

于是：

C = ½ log₂(2πe(P+σ²)) - ½ log₂(2πeσ²)
  = ½ log₂(1 + P/σ²)   [bits/channel use]

这就是香农公式——信息论最重要的公式之一。它给出了 AWGN 信道上每符号可传输的最大比特数。注意几个关键含义：

  • 容量由信噪比 P/σ² 决定，而不是绝对功率
  • 容量随信噪比对数增长：要增加1 bit/symbol的容量，信噪比需要翻倍
  • 带宽 W 下的总容量：C = W · log₂(1 + P/σ²) bits/second

这个公式解答了无数通信工程师的实际困惑："我的线路噪声很大，理论上还能传多快？"答案：计算信噪比，代入公式，这就是你的上限。任何低于这个速率的传输方案都是可行的（可能很复杂但存在），任何高于这个速率的方案都不可能。

深入思考：香农公式像是一个通信的"热力学第二定律"。它告诉你无论你的编码技术多么巧妙，信噪比决定了你能有多快。但——注意——它没有告诉你"要如何实现"。就像知道光速上限并不能让你自动造出超跑，知道信道容量并不能让你自动设计出逼近容量的编码方案。Turbo码和LDPC码的发明，花了40多年才真正逼近这个极限。

我记得在念研究生的时候，导师给我一个项目：在一个信噪比只有3dB的窄带信道上，设计一个能传 2 Mbps 数据的方案。我兴冲冲地算了一下香农容量：C = ½ log₂(1 + 2) ≈ 0.79 bits/symbol，乘上信道带宽——够了，在理论极限以内。但真正设计时发现，要达到这个极限需要很长的编码块长、复杂的迭代译码器，而我们的嵌入式平台根本跑不动。最后我们选择了一个复杂度更低的方案，工作在 1.5 Mbps——离理论极限还有差距，但符合实际约束。这就是工程：你知道天堂在哪里，但能否到达取决于你带的干粮（计算资源）够不够。

并联高斯信道与注水法

现实中的通信信道往往不是单一的 AWGN 信道，而是多个并联的独立信道。例如 OFDM（正交频分复用）将整个频带分成多个子载波，每个子载波上的噪声水平可能不同。Wi-Fi 和 4G/5G 都在使用这种技术。

问题是：给定总功率预算 P，如何在 k 个噪声方差分别为 σ₁², σ₂², ..., σₖ² 的并联信道上分配功率，使总容量最大？

答案是注水法（Water-filling）：

Pᵢ = max(μ - σᵢ², 0)

其中 μ 由总功率约束 Σ Pᵢ = P 决定

"注水"这个名字的由来非常形象：想象一个容器，底部有高低不同的凸起（代表不同信道的噪声方差 σᵢ²）。你往容器里倒水（分配功率），水面会达到一个统一高度 μ。水深在位置 i 处就是 μ - σᵢ²——也就是分配给该信道的功率。如果某处的凸起太高（噪声太大，σᵢ² > μ），该处不会积水（不分配功率）。

Python 演示注水法：

import numpy as np

def water_filling(noise_variances, total_power, tolerance=1e-6):
    """用二分法求解注水的水位 μ"""
    k = len(noise_variances)
    lo, hi = min(noise_variances), max(noise_variances) + total_power
    while hi - lo > tolerance:
        mid = (lo + hi) / 2
        allocated = np.maximum(mid - noise_variances, 0).sum()
        if allocated < total_power:
            lo = mid
        else:
            hi = mid
    mu = (lo + hi) / 2
    power = np.maximum(mu - noise_variances, 0)
    return mu, power

# 示例：5个并联信道，噪声方差不同
noise_vars = np.array([0.1, 0.3, 0.5, 1.0, 2.0])
total_power = 2.0

mu, P_alloc = water_filling(noise_vars, total_power)

print(f"水位 μ = {mu:.3f}")
print("信道 | 噪声方差 | 分配功率")
for i, (nv, p) in enumerate(zip(noise_vars, P_alloc)):
    if p > 0:
        snr = p / nv
        cap = 0.5 * np.log2(1 + snr)
        print(f"  {i+1}  |   {nv:.2f}    |  {p:.3f}  (SNR={snr:.2f}, C={cap:.3f})")
    else:
        print(f"  {i+1}  |   {nv:.2f}    |  0.000  (不使用)")

total_capacity = sum(0.5 * np.log2(1 + p/nv) if p > 0 else 0
                     for p, nv in zip(P_alloc, noise_vars))
print(f"总容量: {total_capacity:.3f} bits/channel use")

这个算法的优美之处在于：它自动实现了"把功率分配给信噪比好的信道，而舍弃信噪比太差的信道"的直觉。在 OFDM 系统中，当某些子载波由于频率选择性衰落而信噪比极低时，注水法告诉你可以直接不在那些子载波上传数据——把功率集中到更好的子载波上。

注水法不仅是信息论中的一个优化工具，它也是整个通信资源分配领域的基础思想之一。从 Wi-Fi 的速率控制到 5G 的资源调度，处处可以看到它的影子。

这一章我们从离散走向了连续世界。关键 takeaways：

  • 微分熵 h(X) = -∫ f(x) log f(x) dx 是连续信源的基本信息度量
  • 微分熵可以为负，这是它与离散熵的根本性区别
  • 固定方差下，高斯分布具有最大微分熵——这是高斯信道容量推导的关键
  • AWGN 信道容量 C = ½ log₂(1 + P/σ²) 是现代通信的基石
  • 注水法是最优的并联信道功率分配方案
