第19章 随机调度

大家好，我是你们的讲师。如果你从事过一线生产管理，你一定有过这样的经历：你花了三个小时排出了一个完美的调度方案——每个工件的开工时间精确到分钟，每台机器的利用率都拉满了——结果方案下发一小时，现场的班长就来电话了："一号机床主轴断了，得停机四小时"。你的完美方案瞬间变成废纸。

这就是调度的残酷现实：确定性假设在真实世界中几乎永远不成立。加工时间从来就不是一个固定的数——同样一个零件，早上新刀具加工可能要 12 分钟，下午刀具磨损了可能要 15 分钟；机器可能在任何时刻故障；客户的紧急订单可能随时插入。如果我们的调度算法只能处理确定性的输入，那它在工业界的实用价值就非常有限。

从这一章开始，我们正式从确定性调度进入不确定性调度的领域。这一章聚焦于随机调度（Stochastic Scheduling）——我们用概率分布来描述不确定性，然后在期望意义下寻求最优或近似最优的解。

不确定性的三种来源 — 加工时间、机器故障、订单到达

在工业调度场景中，不确定性的来源大致可以分为三类。理解这三类来源，是建立合理随机模型的起点。

第一类：加工时间不确定性。这是最常见、也最容易建模的一类。实际加工时间受刀具磨损、材料硬度波动、操作工熟练度差异、环境温度等多种因素影响。统计分析显示，半导体晶圆制造中同一工序的实际加工时间往往服从 对数正态分布（Log-normal Distribution） 或 Gamma 分布，变异系数（Coefficient of Variation）可能在 0.1 到 0.5 之间。一个变异系数 0.3 的含义是：如果标称加工时间为 10 分钟，那么标准差就是 3 分钟——这意味着约 32% 的工件实际加工时间超过 13 分钟或少于 7 分钟。

第二类：机器故障（Machine Breakdowns）。机器故障更"致命"，因为它不是连续的数值波动，而是系统状态的突然改变。典型建模方式是：每台机器的故障间隔时间服从指数分布（或 Weibull 分布），维修时间也服从某种分布。MTBF（Mean Time Between Failures）和 MTTR（Mean Time To Repair）是衡量机器可靠性的两个核心指标。在调度决策中，我们不仅需要考虑故障发生的概率，还需要考虑故障发生后的重调度策略。

第三类：订单到达不确定性（Arrival Uncertainty）。这在"制造型工厂"和"服务型系统"中表现不同。在面向订单生产（Make-to-Order）的工厂中，客户订单的到达时间、加工要求和交货期都存在不确定性。在服务系统（如外卖配送、急诊分诊）中，任务的到达通常建模为泊松过程（Poisson Process），到达率 λ 可能是时变的。这种不确定性最难处理，因为它意味着我们甚至不知道未来有哪些任务需要调度——问题从"已知任务集"变成了"在线决策"。

一次惨痛的教训：2019年我参与过一个 PCB 钻孔车间的排程项目。我们拿到历史数据，发现加工时间的标准差只有 5%，于是很自信地做了一个确定性排程方案。上线第一周就崩了——问题出在数据采集方式上。MES 系统记录的是"工件在机时间"，但没记录"辅助时间"（上下料、换刀、首件检验）。辅助时间的波动极大，同一批次不同位置的 PCB 板，上下料时间可以相差 4 倍。加上辅助时间后的实际加工时间变异系数高达 0.45，我们只考虑了主轴切削时间的波动，被数据采集的盲区坑了。所以，在建立随机模型之前，先搞清楚你的数据采集口径是否覆盖了所有时间成分。

随机加工时间模型 — 期望值最小化

在随机调度中最自然的建模方式就是把目标函数从"最小化 makespan"改为"最小化期望 makespan"（或期望总完工时间、期望加权延迟等）。这看起来只是一个期望算子的引入，但问题的性质发生了根本变化——决策变量从"确定性的开工时间"变成了"应对随机性的调度策略"。

考虑一个最简单的场景：单机调度，加工时间随机。如果我们知道了每个工件加工时间的分布，目标是最小化期望加权总完工时间 E[ΣwjCj]，那么最优策略是什么？

答案出奇地简洁——WSEPT 规则（Weighted Shortest Expected Processing Time）：按照 wj / E[pj] 的降序排列，即"期望加工时间越短、权重越大的工件越优先"。这个规则的优美之处在于：它只依赖于期望值，与方差无关。这意味着即使两个工件的方差差别很大，只要期望加工时间相同，它们的优先级就一样。

随机动态规划方法

当问题从单机扩展到更复杂的场景时，WSEPT 就不再是最优的了。此时我们通常使用随机动态规划（Stochastic Dynamic Programming, SDP）来解决。SDP 的核心思想是通过状态变量来表示调度系统的当前状态（已完成哪些工件、当前机器状态等），然后反向或正向递推最优决策。

例如对于平行机（Parallel Machines）上的随机调度，状态空间是"当前各台机器的剩余加工时间 + 待调度工件集合"，这是一个高维状态空间，精确求解的复杂度随工件数指数爆炸。实际中通常使用近似动态规划（ADP）或 rollout 策略来逼近最优解。

随机支配与WSEPT规则

一个更深层的理论结果是：如果加工时间分布满足某种随机序（Stochastic Order）关系，我们可以建立"确定性等价"的结论。具体来说，如果两个工件的加工时间分布满足似然比序（Likelihood Ratio Order），那么 WSEPT 排序在更广泛的意义下是最优的。

当分布不满足单调性条件时，情况就复杂了——可能出现"非单调最优策略"，即最优决策取决于分布的具体形状，而不仅仅是期望值。2017 年的一篇 Operations Research 论文证明，即使对于最简单的两工件单机问题，如果加工时间分布是双峰的，最优策略可能违反直觉——在某些状态下，让期望加工时间更长的工件先加工反而更好。这个结论告诉我们要谨慎：不要以为 WSEPT 永远是对的。

马尔可夫决策过程(MDP)在调度中的应用

随机调度问题的标准数学框架是马尔可夫决策过程（Markov Decision Process, MDP）。MDP 为异步到达、随机加工时间、机器故障等问题提供了统一的建模语言。

一个调度 MDP 的四个要素如下：

  • 状态（State）：包括每台机器的状态（空闲/忙碌/故障）、当前待调度工件队列中每个工件的加工进度和剩余加工时间、尚未到达的订单信息等。
  • 动作（Action）：从队列中选择哪个工件分配到哪台机器上加工。当机器空闲时，决策者可以"选择下一个工件"或"等待"。
  • 转移概率（Transition Probability）：从当前状态 s 执行动作 a 后，系统转移到下一个状态 s' 的概率。这由加工时间分布、故障分布和到达过程共同决定。
  • 奖励（Reward）：采取动作后获得的即时收益。对于最小化 makespan 的问题，可以在每个 time step 设置 -1 的奖励（即累积成本最小化）。对于最小化加权延迟的问题，可以在每个工件完工时根据其延迟情况施加惩罚。

MDP 的最优解满足贝尔曼最优方程（Bellman Optimality Equation）：

V*(s) = min_{a ∈ A(s)} { C(s,a) + Σ_{s'} P(s'|s,a) * V*(s') }

在实际应用中，状态空间的维数灾难是最大的挑战。求解大规模调度 MDP 通常需要借助近似动态规划（Approximate Dynamic Programming）、强化学习或策略搜索等方法——这将在后续章节中展开。

Gittins指数与多臂老虎机

如果你之前接触过多臂老虎机（Multi-Armed Bandit, MAB）问题，你会发现调度问题与 MAB 有着深刻的联系。Gittins 在 1979 年提出了一个里程碑式的结果：对于一类特定的调度 MDP（称为"休息式多臂老虎机"，Restless Bandit），最优策略可以通过计算每个工件的Gittins 指数（Gittins Index）并选择指数最大的工件来实现。

Gittins 指数的直观含义是：一个工件的"优先级"可由它在当前状态下的"继续加工的边际收益"来度量。只要我们能独立计算每个工件的 Gittins 指数，调度决策就简化为排序——这保持了 WSEPT 规则的简洁性，但比 WSEPT 更精确，因为它考虑了加工时间分布的完整信息而不仅仅是期望值。

Gittins 指数的一个关键性质是指数可分解性（Indexability）——每个工件的指数计算独立于其他工件，这使得算法具有 O(n) 的每次决策复杂度。近年来，Gittins 指数在云计算任务调度、手术室排程和半导体制造中得到了成功应用。

机会约束规划(Chance-Constrained Programming)

如果你不是一个"期望值最大化者"，而是希望保证"按时完工的概率不低于 95%"，那你需要机会约束规划（Chance-Constrained Programming, CCP）。

"按时完工概率≥95%"的数学表达

CCP 的数学形式非常直观。以单机调度为例，假设我们有一个截止日期 D，则"按时完工概率 ≥ 95%"可以写为：

P( 完工时间 ≤ D ) ≥ 0.95

或者更一般地，对于整个调度方案：

P( makespan ≤ D ) ≥ 0.95
P( 每个工件的延迟 ≥ 0 ) ≥ 0.95     // 多约束同时满足

在数学优化中，CCP 是非常"难搞"的约束——概率约束通常非凸，且计算可行域的边界需要高维积分。对于随机加工时间服从正态分布或对数正态分布的情况，概率约束可以转化为二阶锥约束（Second-Order Cone Constraints），从而可以用现有的求解器处理。但对于更一般的分布，我们需要数值方法。

样本平均近似(SAA)方法

样本平均近似（Sample Average Approximation, SAA）是处理随机优化的最实用方法之一。它的思想出奇地简单：

  • 从加工时间分布中采样 N 个场景（scenarios），每个场景对应一组确定的加工时间样本
  • 将随机机会约束替换为 N 个场景中的确定性约束——"在至少 ⌈0.95·N⌉ 个场景中，调度方案满足截止日期"
  • 求解这个确定性混合整数规划问题

SAA 的美妙之处在于：当 N 足够大时，SAA 的解以概率 1 收敛到真实问题的最优解。而且 SAA 可以和任何商业求解器（Gurobi、CPLEX）无缝集成——你只需要根据场景复制变量和约束。

下面给出一个随机作业车间调度（Stochastic JSP）的 SAA 框架描述：

随机作业车间调度 — SAA 求解框架

输入：
  - N 个场景，每个场景包含所有工序的加工时间样本 pᵢⱼ⁽ˢ⁾
  - 截止日期 D，置信水平 α（如 0.95）

建模：
  1. 对于每个场景 s ∈ {1,...,N}，定义变量 xᵢⱼₖₗ⁽ˢ⁾（工序顺序决策）
  2. 对于每个场景 s，建立标准 JSP 约束（工序先后、机器排他）
  3. 引入辅助变量 z⁽ˢ⁾ = 1 表示场景 s 下 makespan ≤ D
  4. 约束：makespan⁽ˢ⁾ ≤ D + M·(1 - z⁽ˢ⁾)  // big-M 约束
  5. 机会约束：Σ z⁽ˢ⁾ / N ≥ α

求解：
  - 上述问题是一个混合整数规划（MIP），可用 Gurobi/CPLEX 求解
  - 变量规模：O(N · n · m²)，N 通常取 20-200
  - 对于大 N，可用 Benders 分解或场景缩减技术加速

SAA 的核心局限在于：解的质量高度依赖于采样场景的代表性。如果 N 太小，SAA 的"95% 概率保证"在真实分布下可能只有 70%。所以实际应用中，通常需要做外样本验证（Out-of-Sample Validation）——用一批未参与优化的新样本来验证调度方案的可靠程度。

建模方法 对不确定性的处理 输出形式 计算复杂度 典型应用
随机规划（SP） 概率分布已知，优化期望值 场景树上的确定性决策 高（场景数指数增长） 生产计划、库存管理
模糊规划（FP） 用隶属函数描述模糊参数 模糊可行域中的满意解 中等 早期设计阶段，数据极少
鲁棒优化（RO） 不确定集描述参数范围，保护最坏情况 对抗性解 低-中（多可重构为凸优化） 电网调度、供应链
机会约束规划（CCP） 概率分布已知，硬约束放松为概率约束 满足置信水平的"安全解" 高（概率约束非凸） 金融风控、手术室排程

一个来自半导体厂的踩坑实例：2021年，一家 12 英寸晶圆厂的 CMP 工段遇到了一个令人困惑的问题——用确定性最优排程方案时，平均设备利用率达到了 92%，但每周总有那么一两天的实际产出比预期低了 15%-20%。工程师们百思不得其解，以为是设备出了问题。后来分析发现：CMP 机台的加工时间波动非常大（变异系数 0.4），确定性排程假设所有晶圆加工时间都是 3 分钟，但实际有接近 20% 的晶圆需要返工抛光，加工时间会突增到 6-8 分钟。确定性排程在设计时把机器排得满满当当，容不下任何波动——一旦有一个晶圆慢了，后面所有工件的级联延迟就像多米诺骨牌一样倒塌。切换到随机排程后，有意在每台机器前留出 10% 的"松弛时间"作为缓冲，虽然峰值利用率从 92% 降到了 87%，但产出的方差大幅降低，整体周产出反而提升了 5%。这就是"确定性最优解在随机环境下可能表现极差"的经典案例。

好，到了这一章的尾声。我们从真实世界的不确定性出发，讨论了随机调度的三种不确定性来源、随机加工时间模型与 WSEPT 规则、MDP 与 Gittins 指数，以及机会约束规划与 SAA 方法。随机调度给了我们一套在概率框架下做决策的工具，但它有一个局限性：它要求我们知道概率分布。如果分布未知怎么办？下一章我们将进入鲁棒调度（Robust Scheduling）——它不需要精确的分布，而是通过不确定集来保护最坏情况，为"不知道分布长什么样"的现实场景提供了另一条路径。我们下一章见。
