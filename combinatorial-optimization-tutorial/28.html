我在教授这门课的五年里，最常听到的一个问题是："老师，我学了这么多算法，感觉理论知识很扎实，但一到解决实际问题就不知道从何下手。" 这个困惑非常真实。学完第7章的分支定界，不等于会用它来解决一个真实的物流调度问题——中间隔着建模、数据预处理、参数调优、结果分析等一整套工程实践技能。

2023年，我指导的一位研究生参加了一个物流优化竞赛。他的团队在理论上做了非常充分的准备——用Gurobi建模部分设施选址问题，设计了精妙的不等式。但他们在数据预处理上犯了一个低级错误：坐标数据中混入了几个经纬度为0的异常点，导致距离矩阵严重失真，最终的优化方案建议在一个"不存在的位置"建立配送中心。这个教训告诉我们：在组合优化的工程实践中，数据质量和模型接口往往比算法本身更容易出问题。

实战第一课：在组合优化的工程世界里，80%的时间花在数据清洗、参数调试和结果验证上，只有20%的时间真正在"求解"。不要轻视那80%——它往往是项目成功的关键。

精确方法项目：设施选址

问题描述：某连锁便利店品牌计划在N个候选位置中选择p个位置建立配送中心，为M个门店提供配送服务。目标是最小化所有门店到最近配送中心的总距离。

这是一个经典的p-median问题，属于设施选址（Facility Location）家族。它是NP-hard的，但中等规模（N=50, M=200, p=5）下可以用MIP精确求解。

步骤1：数据生成

首先，我们生成模拟数据。为了更真实，我们从实际的城市坐标分布中采样：

import numpy as np
import matplotlib.pyplot as plt
from pulp import *

# ===== 数据生成 =====
np.random.seed(42)
N = 50    # 候选设施数量
M = 200   # 门店数量
p = 5     # 需要选择的设施数量

# 生成候选设施位置（模拟城市中的可用地块）
facility_positions = np.random.uniform(0, 100, (N, 2))

# 生成门店位置（模拟实际门店分布）
store_positions = np.random.uniform(0, 100, (M, 2))

# 计算距离矩阵（欧几里得距离）
dist = np.zeros((N, M))
for i in range(N):
    for j in range(M):
        dist[i, j] = np.linalg.norm(
            facility_positions[i] - store_positions[j]
        )

# 可视化
plt.figure(figsize=(8, 6))
plt.scatter(facility_positions[:, 0], facility_positions[:, 1],
            c='red', marker='s', s=100, label='候选设施')
plt.scatter(store_positions[:, 0], store_positions[:, 1],
            c='blue', s=20, label='门店')
plt.legend()
plt.title(f'设施选址：{N}候选位 × {M}门店 (选p={p}个)')
plt.savefig('facility_layout.png')

步骤2：MIP建模（PuLP）

现在建立整数线性规划模型。我们使用双索引建模——决策变量yᵢ表示是否在位置i建设施，xᵢⱼ表示门店j是否分配给设施i：

# ===== MIP建模 =====
prob = LpProblem("Facility_Location", LpMinimize)

# 决策变量
y = {i: LpVariable(f"y_{i}", cat=LpBinary) for i in range(N)}
x = {(i, j): LpVariable(f"x_{i}_{j}", cat=LpBinary)
     for i in range(N) for j in range(M)}

# 目标：最小化总距离
prob += lpSum(dist[i, j] * x[i, j]
              for i in range(N) for j in range(M))

# 约束1：选p个设施
prob += lpSum(y[i] for i in range(N)) == p

# 约束2：每个门店恰好分配一个设施
for j in range(M):
    prob += lpSum(x[i, j] for i in range(N)) == 1

# 约束3：门店只能分配给已选的设施
for i in range(N):
    for j in range(M):
        prob += x[i, j] <= y[i]

# 求解
prob.solve(PULP_CBC_CMD(msg=True, timeLimit=300))
print(f"状态: {LpStatus[prob.status]}")
print(f"最优总距离: {value(prob.objective):.2f}")

步骤3：Gurobi调优

如果你有Gurobi许可，可以用以下参数调优加速：

# 用Gurobi求解并调优
from gurobipy import Model, GRB, quicksum

m = Model("Facility_Location")

# (变量和目标设置同PuLP部分，此处省略重复代码)

# 参数调优的关键设置
m.setParam("MIPFocus", 1)    # 1=侧重可行解快速找到，2=侧重最优性证明，3=侧重下界
m.setParam("Heuristics", 0.3)  # 启发式强度，默认0.05，调高可以更快找到可行解
m.setParam("Presolve", 2)    # 2=aggressive presolve，减少问题规模

# 利用对称性：给y变量添加优先级（靠近几何中心的候选位优先）
centroid = facility_positions.mean(axis=0)
dist_to_center = [np.linalg.norm(facility_positions[i] - centroid)
                  for i in range(N)]
for i in range(N):
    y[i].setAttr("BranchPriority", int(-dist_to_center[i] * 10))

m.optimize()

我在这个项目上踩过的一个坑是：对称性导致求解缓慢。在p-median中，如果两个候选设施位置非常接近，它们在数学上几乎是"对称"的——交换它们得到的目标值相同。这会导致分支定界大量探索对称的等价解。解决方案是添加对称性破缺约束（例如强制设施编号按坐标排序），或者使用BranchPriority引导求解器优先探索更合理的分支。

小贴士：如果你发现求解器卡住不动了，试试以下几个操作（按效果排序）：(1) 添加最优性割（optimality cuts）收紧下界；(2) 启用对称性破缺；(3) 调整MIPFocus参数；(4) 提供一个好的初始解作为MIP start。不要一上来就加时间限制然后放弃——求解器很多时候会在即将收敛前出现长时间的"沉默"。

启发式项目：TSP求解器对比

问题描述：在TSPLIB的标准实例（如berlin52、eil101）上，实现并对比三种启发式算法——2-opt局部搜索、模拟退火（SA）和遗传算法（GA）。

步骤1：数据加载

TSPLIB格式是TSP问题的事实标准，我们写一个简单的解析器：

def load_tsplib(filename):
    """解析TSPLIB格式文件，返回城市坐标列表"""
    nodes = []
    with open(filename) as f:
        in_section = False
        for line in f:
            if line.startswith("NODE_COORD_SECTION"):
                in_section = True
                continue
            if line.startswith("EOF"):
                break
            if in_section:
                parts = line.strip().split()
                if len(parts) >= 3:
                    nodes.append((float(parts[1]), float(parts[2])))
    return np.array(nodes)

步骤2：实现2-opt

2-opt是最经典的TSP局部搜索操作——交换两条边：

def two_opt(route, dist):
    """2-opt局部搜索，返回改善后的路径"""
    n = len(route)
    improved = True
    while improved:
        improved = False
        for i in range(n - 1):
            for j in range(i + 2, n):
                # 当前边: (i, i+1) 和 (j, (j+1)%n)
                # 新边:   (i, j) 和 (i+1, j+1)
                delta = (
                    -dist[route[i], route[(i+1) % n]]
                    -dist[route[j], route[(j+1) % n]]
                    +dist[route[i], route[j]]
                    +dist[route[(i+1) % n], route[(j+1) % n]]
                )
                if delta < -1e-10:  # 有改进
                    # 反转 i+1 到 j 之间的路径段
                    route[i+1:j+1] = route[i+1:j+1][::-1]
                    improved = True
    return route

# 生成随机初始解
route = np.random.permutation(n).tolist()
route = two_opt(route, dist)
print(f"2-opt结果: {tour_length(route, dist):.2f}")
一个关键优化：上面的双重循环是O(n²)的，对于1000个城市，每次迭代需要检查约50万对边。一个重要的加速技巧是候选邻域剪枝——只检查当前城市最近的K个邻居（K=20-40），而不是所有城市。这在实践中几乎不影响解质量，但能将邻域评估从O(n²)降到O(nK)。这就是著名的3-opt的LN（Limited Neighborhood）技巧。

步骤3：实现模拟退火

def simulated_annealing(cities, dist, T_start=100, T_end=0.01, alpha=0.995):
    n = len(cities)
    route = np.random.permutation(n).tolist()
    best_route = route.copy()
    best_cost = tour_length(route, dist)
    T = T_start
    steps = 0

    while T > T_end:
        # 随机选两个城市翻转
        i, j = sorted(np.random.choice(range(1, n), 2, replace=False))
        # 2-opt move的delta计算
        delta = (
            -dist[route[i-1], route[i]]
            -dist[route[j], route[(j+1) % n]]
            +dist[route[i-1], route[j]]
            +dist[route[i], route[(j+1) % n]]
        )
        if delta < 0 or np.random.random() < np.exp(-delta / T):
            route[i:j+1] = route[i:j+1][::-1]
            cost = tour_length(route, dist)
            if cost < best_cost:
                best_cost = cost
                best_route = route.copy()

        T *= alpha
        steps += 1

    return best_route, best_cost, steps

# 在berlin52上测试
route_sa, cost_sa, _ = simulated_annealing(cities, dist)
print(f"SA结果: {cost_sa:.2f} (最优: 7544.37, gap: {(cost_sa/7544.37-1)*100:.2f}%)")

步骤4：实现遗传算法

def genetic_algorithm(cities, dist, pop_size=100, generations=500,
                     mutation_rate=0.1):
    n = len(cities)

    # 用贪心初始化部分种群（加速收敛）
    def greedy_route(start):
        route = [start]
        unvisited = set(range(n))
        unvisited.remove(start)
        current = start
        while unvisited:
            next_city = min(unvisited, key=lambda c: dist[current, c])
            route.append(next_city)
            unvisited.remove(next_city)
            current = next_city
        return route

    # 种群初始化（混合贪心和随机）
    pop = [greedy_route(i) for i in range(min(20, n))]
    pop += [np.random.permutation(n).tolist()
            for _ in range(pop_size - len(pop))]

    for gen in range(generations):
        # 评估
        costs = [tour_length(ind, dist) for ind in pop]
        # 锦标赛选择
        new_pop = []
        for _ in range(pop_size):
            i, j = np.random.choice(pop_size, 2, replace=False)
            winner = pop[i] if costs[i] < costs[j] else pop[j]
            new_pop.append(winner)
        # 交叉（Order Crossover, OX）
        offspring = []
        for i in range(0, pop_size, 2):
            p1, p2 = new_pop[i], new_pop[(i+1) % pop_size]
            # OX交叉：保留p1的一段子序列，其余位置按p2的顺序填充
            a, b = sorted(np.random.choice(range(n), 2, replace=False))
            child = [-1] * n
            child[a:b] = p1[a:b]
            pos = b
            for city in p2:
                if city not in child:
                    if pos >= n:
                        pos = 0
                    child[pos] = city
                    pos += 1
            offspring.append(child)
        # 变异（随机交换两个城市）
        for ind in offspring:
            if np.random.random() < mutation_rate:
                i, j = np.random.choice(n, 2, replace=False)
                ind[i], ind[j] = ind[j], ind[i]
        pop = offspring

    # 返回最佳解
    costs = [tour_length(ind, dist) for ind in pop]
    best_idx = np.argmin(costs)
    return pop[best_idx], costs[best_idx]

步骤1：数据生成

import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np

def generate_tsp_data(batch_size, n_nodes):
    """生成TSP数据：batch_size个实例，每个n_nodes个城市，坐标在[0,1]²"""
    return torch.FloatTensor(batch_size, n_nodes, 2).uniform_(0, 1)

def tour_length_torch(tour, points):
    """计算批次中每个TSP路径的长度"""
    batch_size, n_nodes, _ = points.shape
    # tour: (batch, n_nodes) 城市的索引
    # 收集坐标
    idx = tour.unsqueeze(-1).expand(-1, -1, 2)
    ordered = points.gather(1, idx)
    # 计算相邻距离并求和（包含回到起点）
    rolled = torch.roll(ordered, shifts=-1, dims=1)
    dists = torch.norm(ordered - rolled, dim=2)
    return dists.sum(dim=1)

步骤2：实现简化Attention Model

class TSPAttentionModel(nn.Module):
    """简化的Attention Model for TSP"""

    def __init__(self, embed_dim=128, n_heads=8, n_layers=3):
        super().__init__()
        self.embed_dim = embed_dim
        # 输入嵌入
        self.init_embed = nn.Linear(2, embed_dim)
        # Transformer编码器层
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=embed_dim, nhead=n_heads, dim_feedforward=512,
            batch_first=True
        )
        self.encoder = nn.TransformerEncoder(encoder_layer, n_layers)

        # 解码器：单层注意力 + 投影到概率
        self.W_query = nn.Linear(embed_dim, embed_dim, bias=False)
        self.W_key = nn.Linear(embed_dim, embed_dim, bias=False)
        self.logit_proj = nn.Linear(embed_dim, 1, bias=False)

    def forward(self, points, decode_mode='greedy'):
        """
        points: (batch, n, 2) 城市坐标
        返回: (batch, n) 路径序列（城市索引）
        """
        batch_size, n_nodes, _ = points.shape

        # 编码器
        h = self.init_embed(points)        # (batch, n, embed_dim)
        h = self.encoder(h)                # (batch, n, embed_dim)

        # 解码（自回归）
        visited = torch.zeros(batch_size, n_nodes, dtype=torch.bool,
                              device=points.device)
        tour = torch.zeros(batch_size, n_nodes, dtype=torch.long,
                          device=points.device)
        log_probs = torch.zeros(batch_size, n_nodes, device=points.device)

        # 随机选起点
        current = torch.randint(0, n_nodes, (batch_size,),
                               device=points.device)
        for step in range(n_nodes):
            visited.scatter_(1, current.unsqueeze(1), True)
            tour[:, step] = current

            # 计算注意力
            query = self.W_query(h[torch.arange(batch_size), current])
            keys = self.W_key(h)                       # (batch, n, embed_dim)
            scores = torch.bmm(keys, query.unsqueeze(2)).squeeze(-1)
            # / sqrt(d_k) 进行缩放
            scores = scores / (self.embed_dim ** 0.5)

            # mask已访问城市
            scores[visited] = -1e9

            # softmax得到概率分布
            probs = F.softmax(scores, dim=1)

            # 选择下一个城市
            if decode_mode == 'greedy':
                current = probs.argmax(dim=1)
            else:  # 采样
                current = torch.multinomial(probs, 1).squeeze(1)

            log_probs[:, step] = torch.log(probs[torch.arange(batch_size),
                                                   current] + 1e-10)

        return tour, log_probs

步骤3：强化学习训练

使用REINFORCE with self-critical baseline：

def train_nco(model, optimizer, n_epochs=100, batch_size=64,
              n_nodes=20):
    losses = []
    gap_history = []

    for epoch in range(n_epochs):
        # 生成训练数据
        points = generate_tsp_data(batch_size, n_nodes)

        # 采样解码（用于训练）
        tour_sample, log_probs = model(points, decode_mode='sample')
        lengths_sample = tour_length_torch(tour_sample, points)

        # 贪心解码（作为baseline）
        with torch.no_grad():
            tour_greedy, _ = model(points, decode_mode='greedy')
            lengths_greedy = tour_length_torch(tour_greedy, points)

        # REINFORCE损失
        advantage = (lengths_sample - lengths_greedy).detach()
        loss = (advantage * log_probs.sum(dim=1)).mean()

        optimizer.zero_grad()
        loss.backward()
        torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        optimizer.step()

        # 评估
        with torch.no_grad():
            avg_length = lengths_greedy.mean().item()
            # 用2-opt作为baseline计算gap
            gap = (avg_length / optimal_length_est - 1) * 100

        if epoch % 10 == 0:
            print(f"Epoch {epoch}: loss={loss.item():.4f}, "
                  f"avg_length={avg_length:.4f}, gap={gap:.2f}%")

        losses.append(loss.item())
        gap_history.append(gap)

    return losses, gap_history

28章知识体系回顾

我们将全部课程分为六个部分，每个部分解决一个核心问题：

第一部分"基础与工具"（第1-6章）回答了"组合优化是什么"的问题。我们学会了用形式化定义⟨C, S, opt, v⟩刻画任何组合优化问题，掌握了图论、计算复杂性、线性规划、对偶理论等核心分析工具，以及贪心和动态规划这两个最基本的算法范式。

第二部分"精确算法"（第7-11章）回答了"怎么找到最优解"的问题。分支定界、网络流、最小割、二分图匹配、一般图匹配——这些算法各有擅长的战场，但共同点是保证找到全局最优解。在计算复杂性理论划定的界限内，它们代表了人类算法智慧的巅峰。

第三部分"整数规划与多面体理论"（第12-16章）回答了"怎么用求解器解决现实问题"的问题。从整数线性规划的标准形式出发，我们深入了分支切割和分支定价这两个求解器的核心算法，学习了多面体组合的理论基础，以及——最重要的——如何把现实问题"翻译"成求解器能理解的数学模型。这部分是理论到工业实践的桥梁。

第四部分"高级理论"（第17-22章）回答了"当我们无法精确求解时怎么办"的问题。约束规划提供了一种不同于MIP的建模范式，Benders和Dantzig-Wolfe分解方法让我们能利用问题结构"分而治之"，拟阵理论统一了贪心算法的理论基础，次模函数优化提供了离散优化的另一个视角，近似算法给出了"有多好"的理论保证，而在线与随机优化处理了决策环境的不确定性。

第五部分"求解器与启发式"（第23-25章）回答了"工业实战怎么打"的问题。我们手把手用Gurobi求解了真实供应链问题，学习了性能调优和debugging的技巧，然后探索了模拟退火、遗传算法、禁忌搜索等元启发式方法——它们不保证最优，但能在"求解时间-解质量"之间提供灵活的折中。

第六部分"前沿与展望"（第26-28章）回答了"未来的方向在哪里"的问题。神经组合优化用深度学习实现了"看一眼就知道怎么走"的直觉推理；GPU加速、Ising机、量子计算从硬件维度突破了传统算法的速度天花板。而本章的三个实战项目，则是你把所有知识转化为实际能力的最后一块拼图。

我的教学理念：如果你的目标是成为组合优化工程师，你需要三样东西：（1）扎实的理论基础——理解算法的"为什么"而不是只记住"是什么"；（2）丰富的实战经验——亲手调试过50个MIP模型、看过100条收敛曲线后，你才能真正理解组合优化的艺术；（3）持续学习的习惯——这个领域每两年就有重要新进展，保持阅读、保持实验、保持好奇。

成为组合优化工程师的能力地图

最后，我想分享一个关于"什么样的工程师是好工程师"的个人观察。过去十年间，我和数十位顶级的组合优化从业者合作过。他们身上有一些共同的品质：

  • 数学直觉：他们不是为了用数学而用数学，而是在面对一个模糊的实际问题时，能用数学的语言精确定义"什么是最优"、"约束是什么"、"决策变量是什么"。这种直觉来自大量的阅读和实践。
  • 系统思维：他们知道一个优化方案不会在真空中运行——数据可能有噪声，约束可能随时间变化，决策需要被业务团队理解。他们设计的不只是一个求解器调用，而是一套端到端的决策系统。
  • 务实主义：他们知道什么时候追求最优解，什么时候"够好就行"。在一个实时推荐场景中，10ms内给出一个gap=5%的解，比10分钟内给出一个最优解有价值得多。他们会在精确性和效率之间做出明智的权衡。
  • 持续学习：组合优化领域在高速发展。一个只懂传统方法而不了解NCO的工程师，和另一个只懂深度学习而不了解分支定界的工程师，在未来都会被淘汰。真正有价值的是那些能跨越经典和前沿、理论和实践的"T型人才"。
