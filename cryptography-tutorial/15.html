第15章 密码分析基础

同学们好。前面十四章我们一直在学"怎么加密"——古典密码、对称加密、公钥密码、哈希函数、数字签名……都是站在密码设计者的角度。但密码学还有另一张面孔：怎么攻破它？这就是本章的主题——密码分析（Cryptanalysis）。

有个道理可能反直觉：一个密码体制真正的强度，不是由它的设计者决定的，而是由全世界最聪明的攻击者决定的。你今天设计了一个自以为无懈可击的加密方案，明天就可能被一篇论文彻底推翻。这种"攻防博弈"正是密码学如此迷人的原因——它是极少数把"敌人"的智慧纳入自身设计框架的学科。

本章我们将系统介绍现代密码分析的主要方法：从差异分析到线性分析，从侧信道攻击到量子威胁。每学会一种攻击方法，你对"什么算安全"的理解就会更深一层。

15.1 攻击模型分类

在讨论具体攻击技术之前，我们先要明确攻击者所处的场景。不同的攻击模型对应着不同的能力假设：

攻击类型 攻击者掌握的信息 典型场景
唯密文攻击（Ciphertext-only） 仅截获密文 窃听者获取加密通信内容
已知明文攻击（Known-plaintext） 若干明文-密文对 邮件头部的固定格式泄露了明文
选择明文攻击（Chosen-plaintext） 可自由选择明文并获取对应密文 攻击者可控制加密输入（如Web表单）
选择密文攻击（Chosen-ciphertext） 可自由选择密文并获取对应明文 攻击者可向解密端发送任意密文
侧信道攻击（Side-channel） 可观测物理泄露（时间、功耗、电磁） 同一台云主机上的其他租户

这里有一个关键点：任何实际可用的密码算法都必须能抵抗选择明文攻击（CPA）。为什么？因为现代密码学中，攻击者通常至少拥有"已知明文"的能力——很多协议的报文头、文件格式的特征都是可预测的。如果连这个都扛不住，这个算法就不配部署到生产环境。

🎯 核心观点：密码分析的目的不是"破解密码"，而是量化安全余量。当我们说AES-128是安全的，意思是：在目前已知的最佳攻击下，攻破它需要的计算量仍然接近2^128——而不是某个聪明的算法能一下子降到2^80。

15.2 差分密码分析

差分密码分析（Differential Cryptanalysis）是现代对称密码分析的开山之作。1990年，以色列密码学家 Eli Biham 和 Adi Shamir 发表了这项技术，震惊了整个密码学界。

核心思想其实很朴素：观察明文对的差异如何通过轮函数传播到密文对的差异。假设我们加密两个明文 P 和 P'，它们只有少数几个比特不同。经过S盒和轮函数后，这两个加密过程产生的中间状态的差异会遵循某种非均匀的概率分布——这种"偏差"就是攻击者可以利用的杠杆。

具体来说，差分攻击寻找的是：给定特定的输入差异 ΔP = P ⊕ P'，经过多个加密轮次后，输出差异 ΔC 并非均匀分布，而是某个特定值出现的概率显著高于其他值。这个"高概率差分特征"就可以用来恢复密钥。

Biham 和 Shamir 用差分攻击证明了：16轮的DES（即标准DES）理论上可以用2^47次选择明文攻击攻破，而当时所有人的认知是：DES唯一的攻击方式就是暴力穷举2^55次。这个发现本身并没有实际破解DES（因为2^47仍然很大），但它揭示了DES的设计并非最优——DES的S盒似乎恰好被设计成能抵抗差分攻击（后来发现，IBM在设计DES时就已知差分攻击，但被要求保密）。

💡 你知道吗？差分密码分析的思想，其实在1970年代IBM设计DES时就已经被NSA知晓。NSA修改了DES的S盒，使之对差分分析有更强的抵抗力。但Biham和Shamir在1990年"重新发现"这项技术时，并不知道NSA已经秘密知道它二十多年了。这就是密码学史上著名的"DES的S盒秘密"——直到1994年，IBM的原设计文档才被解密，证实了这一点。

15.3 线性密码分析

如果说差分分析是"横向比较"，那么线性密码分析（Linear Cryptanalysis）就是"纵向穿透"。1993年，日本密码学家 Mitsuru Matsui 发表了这项技术，并很快用它成功攻击了DES。

线性分析的核心是：寻找明文、密文和密钥之间的线性逼近关系。具体来说，攻击者试图找到一个线性表达式：

P[i1] ⊕ P[i2] ⊕ ... ⊕ P[ia]
⊕ C[j1] ⊕ C[j2] ⊕ ... ⊕ C[jb]
= K[k1] ⊕ K[k2] ⊕ ... ⊕ K[kc]

其中 P[i] 表示明文的第 i 位，C[j]是密文的第j位，K[k]是密钥的第k位。这个表达式以概率 p ≠ 1/2 成立。当 p 偏离 1/2 越远，攻击需要的明文-密文对就越少。

Matsui 使用线性分析攻击 DES，只需要 2^43 个已知明文——比 Biham 和 Shamir 的差分分析（需要2^47个选择明文）更高效。更重要的是，它只需要"已知明文"而无需"选择明文"，这意味着攻击者的能力假设更弱、更现实。

线性分析的深刻启示是：S盒的非线性是最关键的。如果S盒的线性逼近偏差太大，整个密码就会被轻松攻破。AES在设计时就把抵抗线性和差分分析作为首要目标，它的S盒（基于GF(2^8)的乘法逆元）在这两方面都极为优秀。

15.4 侧信道攻击

以上所有攻击都假设攻击者只能看到算法的输入和输出。但如果攻击者可以观察算法执行过程中的物理泄露呢？这就是侧信道攻击的领域——而且它往往比数学攻击有效得多。

侧信道类型 观测对象 攻击方法
时序攻击 执行时间 测量不同输入下的运算耗时差异
功耗分析 芯片功耗 SPA（简单功耗分析）/ DPA（差分功耗分析）
缓存时序攻击 CPU cache访问延迟 Prime+Probe, Flush+Reload, Spectre, Meltdown
电磁辐射 电磁泄露 采集电磁信号重建内部状态
声学攻击 声音 分析CPU或风扇声音推断计算类型

时序攻击是最经典也最容易被忽视的。如果一段比较两个字符串是否相等的代码用了逐字节比较并提前返回：

// 不安全的比较方式 —— 存在时序泄露
bool constant_time_compare_bad(const char *a, const char *b, size_t n) {
    for (size_t i = 0; i < n; i++) {
        if (a[i] != b[i]) return false;  // 一旦不同立即返回
    }
    return true;
}

攻击者可以逐字节猜解密钥：如果第一个字节猜对了，比较时间会略长（因为继续比较第二个字节）；如果猜错了，函数立即返回。对每个字节尝试256种可能，只需要 256 × 字节数 次测量——而不是穷举所有可能的密钥组合。

缓存时序攻击是近年来最受关注的侧信道类型。攻击者利用CPU缓存的访问延迟差异：如果某条数据在缓存中，访问只需几个纳秒；如果不在，则需要几百纳秒。通过精密的计时测量，攻击者可以判断受害者访问了哪些内存地址，从而推断出密钥。

15.5 量子计算威胁

最后，我们要谈谈密码学最深刻的存在威胁——量子计算。这不再是理论推演，而是正在逼近的现实。

量子计算对密码学的冲击可以归结为两个核心算法：

Shor算法（1994年，Peter Shor）能够在多项式时间内解决整数分解和离散对数问题。这意味着：

  • RSA——基于大整数分解的困难性，被Shor算法完全摧毁
  • ECDH / ECDSA——基于椭圆曲线离散对数，同样被摧毁
  • DSA / ElGamal——基于有限域离散对数，也被摧毁

Grover算法（1996年，Lov Grover）对对称密码的影响温和得多：它将密钥搜索的复杂度从 O(2^n) 降至 O(2^(n/2))。所以：

  • AES-128的128位密钥安全性降为64位——不够安全
  • AES-256的安全性降为128位——仍然安全
  • SHA-256的碰撞搜索由2^128降为2^64——不够安全，需用SHA-384或SHA-512
算法 经典安全强度 量子安全强度（Grover） 是否需替换
AES-128 128 bits 64 bits 建议升级到AES-256
AES-256 256 bits 128 bits 仍安全
RSA-2048 112 bits 被Shor完全攻破 立刻替换
ECDSA P-256 128 bits 被Shor完全攻破 立刻替换
SHA-256 128 bits（碰撞） 64 bits（碰撞） 建议用SHA-384/SHA-512
⚠️ 重要提醒："现在的加密数据以后不会被量子计算机破解"是一个常见的误解。正确的做法是先加密后签名，而且如果你的数据需要长期保密（比如保密10年以上），现在就应该考虑采用后量子密码方案。攻击者只需现在收集你的加密流量，等量子计算机成熟后再解密——这就是"现在收集，以后解密"（Harvest Now, Decrypt Later）威胁。

15.6 真实故事：Spectre / Meltdown 与密码泄露

2018年1月，Google Project Zero 的 Jann Horn 与其他团队同时披露了 Spectre（幽灵）和 Meltdown（熔断）两个CPU漏洞。它们不是软件漏洞，而是CPU推测执行和缓存机制本身的设计缺陷——是硬件层面的问题。

这两个漏洞对密码学的冲击是巨大的。原理大致是这样的：CPU为了加速，会"预判"分支的结果并提前执行指令（推测执行）。如果猜错了就回滚——但回滚的是寄存器状态，而不是CPU缓存。攻击者可以利用这个不对称性：让受害者在推测执行中访问某个密钥相关的内存位置，那个位置的数据就会被加载到缓存中。然后攻击者通过缓存时序测量，就能推断出密钥的比特信息。

具体到密码学场景，下面是一个典型的攻击路径：

  • 受害者在进行RSA解密操作，逐位处理密钥比特
  • 攻击者触发Spectre v1，在受害者进程中诱导推测执行
  • 推测执行中，密钥比特决定了某个数组元素的访问
  • 那个数组元素被加载到CPU缓存
  • 攻击者通过Flush+Reload技术测量每个数组位置的访问延迟
  • 哪些位置访问快（在缓存中），就泄露了密钥的哪些比特

Meltdown 的利用更直接——它利用了用户态和内核态隔离的漏洞，让普通用户进程可以直接读取内核内存。如果内核内存中恰好有另一个进程的RSA私钥……

这个故事的教训是深刻的：密码学的安全性不仅取决于数学基础，还取决于硬件的实现。一个在数学上完美安全的算法，跑在不安全的硬件上，就像把保险柜放在纸糊的房间里。Spectre/Meltdown 的发现催生了大量的恒定时间代码（constant-time code）和无分支（branch-free）编程实践——我们将在第18章详细讨论。
