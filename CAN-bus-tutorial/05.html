第5章：CAN总线位时序与同步

各位同学，今天我们来聊聊CAN总线里一个特别关键，但也容易让人犯迷糊的话题——位时序与同步。说实话，我刚开始接触CAN的时候，觉得这部分就是一堆寄存器的配置，没什么大不了的。直到有一次，我在一个车载项目中，总线莫名其妙地丢帧，查了整整两天，最后发现就是同步段配置差了一个Tq。嗯，从那以后，我再也不敢小看这个章节了。

5.1 位时间结构

先说说位时间。一个CAN数据位，在总线上是有固定时间长度的。这个时间被分成了四个小段，就像切蛋糕一样。我个人习惯把这四个段记成：同步段、传播段、相位缓冲段1、相位缓冲段2。

每个段的长度，是用Tq（Time Quantum，时间量子）来衡量的。Tq是CAN控制器的最小时间单位，由系统时钟分频得到。你想想看，一个位时间由多少个Tq组成？这个是可以配置的，通常范围在8到25个Tq之间。

段名称 长度（Tq） 作用
同步段（Sync_Seg） 1 Tq 用于检测总线上的跳变沿
传播段（Prop_Seg） 1~8 Tq 补偿物理延迟
相位缓冲段1（Phase_Seg1） 1~8 Tq 用于重同步，可延长
相位缓冲段2（Phase_Seg2） 1~8 Tq 用于重同步，可缩短

这里有个关键点：采样点。采样点位于相位缓冲段1和相位缓冲段2之间。说白了，CAN控制器就在这个时刻读取总线电平，判断是显性还是隐性。采样点的位置直接影响通信的可靠性。

重要提示：采样点位置建议设置在85%~90%的位时间位置。太靠前或太靠后，都容易采样到错误的电平。我在项目中一般取87.5%，这个值比较稳妥。

5.2 同步段

同步段固定为1个Tq。它的作用很简单——等待总线上的跳变沿。每个CAN节点都在同步段的上升沿或下降沿处，重新对齐自己的时钟。

为什么会这样？因为CAN总线没有独立的时钟线。所有节点都靠总线上的电平变化来同步。你想想看，如果每个节点的晶振都有微小偏差，时间长了就会累积误差。同步段就是用来消除这个误差的。

5.3 传播段

传播段用来补偿信号在总线上的传输延迟。这个延迟包括：驱动器的延迟、接收器的延迟、以及总线电缆的延迟。

我记得有一次，一个学员问我：「老师，传播段设多大合适？」我的回答是：看你的总线长度。总线越长，传播段就要越大。一般来说，1米总线大约对应5ns的延迟。如果你用1Mbps的波特率，一个Tq大约是100ns，那么20米的总线就需要大约1个Tq的传播段。

实战技巧：我建议传播段至少设为2~3个Tq。别抠门，留点余量。我曾经在一个项目中，为了追求高速，把传播段设成了1个Tq，结果总线长度稍微一长就出问题。后来老老实实改回3个Tq，问题就解决了。

5.4 相位缓冲段

相位缓冲段有两个：Phase_Seg1和Phase_Seg2。它们的作用是配合同步机制，动态调整采样点的位置。

Phase_Seg1可以延长，Phase_Seg2可以缩短。延长或缩短的步长，由同步跳转宽度（SJW，Synchronization Jump Width）决定。SJW一般设为1~4个Tq。

这里有个容易踩的坑：SJW不能大于Phase_Seg1和Phase_Seg2的最小值。否则，重同步的时候可能会把采样点调整到错误的位置。嗯，这个细节很多初学者会忽略。

5.5 同步机制

同步机制分为两种：硬同步和重同步。说白了，硬同步是「大调整」，重同步是「微调」。

5.5.1 硬同步

硬同步发生在总线从空闲状态（隐性）变为开始帧（显性）的时刻。也就是SOF（Start of Frame）位。所有节点检测到这个下降沿后，都会把自己的同步段对齐到这个边沿。

硬同步的特点是：一次性调整到位。调整后，所有节点的位时间起点就一致了。

注意：硬同步只在SOF位发生一次。如果在数据帧中间出现硬同步，那说明总线出大问题了。我曾经在调试一个电机控制器时，发现总线上频繁出现硬同步，最后查出是电源干扰导致节点误触发。换了滤波电容后，问题消失。

5.5.2 重同步

重同步发生在数据帧传输过程中。每个节点在同步段检测到跳变沿后，如果发现这个边沿没有落在自己的同步段内，就会进行微调。

调整规则是这样的：

  • 如果边沿落在同步段之前（即相位误差为负），说明本地时钟偏慢，需要延长Phase_Seg1
  • 如果边沿落在同步段之后（即相位误差为正），说明本地时钟偏快，需要缩短Phase_Seg2

调整的幅度不能超过SJW。也就是说，每次最多调整SJW个Tq。

你想想看，如果每个节点都这样微调，整个网络的时钟就能保持同步。这就是CAN总线能在没有独立时钟线的情况下，实现可靠通信的秘密。

5.6 实战配置示例

假设我们要配置一个500kbps的CAN网络，系统时钟为16MHz，我们来看看怎么算。

// 假设使用16MHz晶振，CAN时钟分频为2
// 则Tq = 2 / 16MHz = 125ns
// 位时间 = 1 / 500kbps = 2000ns
// 每个位包含的Tq数 = 2000 / 125 = 16 Tq

// 配置方案（推荐）：
// 同步段：1 Tq
// 传播段：3 Tq
// 相位缓冲段1：5 Tq
// 相位缓冲段2：7 Tq
// 采样点位置 = (1+3+5) / 16 = 56.25%

// 等等，这个采样点太靠前了！
// 我们调整一下：
// 同步段：1 Tq
// 传播段：2 Tq
// 相位缓冲段1：6 Tq
// 相位缓冲段2：7 Tq
// 采样点位置 = (1+2+6) / 16 = 56.25%

// 还是太靠前？那试试这个：
// 同步段：1 Tq
// 传播段：1 Tq
// 相位缓冲段1：7 Tq
// 相位缓冲段2：7 Tq
// 采样点位置 = (1+1+7) / 16 = 56.25%

// 嗯，16个Tq的配置，采样点很难做到85%以上。
// 所以实际项目中，我一般用20个Tq的配置：
// 同步段：1 Tq
// 传播段：3 Tq
// 相位缓冲段1：9 Tq
// 相位缓冲段2：7 Tq
// 采样点位置 = (1+3+9) / 20 = 65%

// 或者用24个Tq：
// 同步段：1 Tq
// 传播段：3 Tq
// 相位缓冲段1：12 Tq
// 相位缓冲段2：8 Tq
// 采样点位置 = (1+3+12) / 24 = 66.7%

关键结论：采样点位置最好在85%~90%。要达到这个目标，位时间至少需要20个Tq以上。我个人推荐24个Tq的配置，这样采样点可以做到87.5%，而且余量充足。

5.7 避坑指南

最后，我总结几个实战中容易踩的坑：

  • 采样点太靠前：我曾经在一个项目中，采样点设在60%的位置，结果总线长度超过10米就丢帧。后来改到87.5%，问题解决。
  • SJW设得太大：SJW不是越大越好。设得太大，重同步时调整幅度过大，反而容易引入抖动。我一般设2个Tq。
  • 忽略晶振误差：不同节点的晶振精度不同。如果晶振误差超过1%，即使配置再完美，也会出问题。我建议使用精度在0.5%以内的晶振。
  • 传播段设得太小：特别是长距离总线，传播段一定要留够余量。我一般按每米5ns计算，再加2个Tq的余量。
