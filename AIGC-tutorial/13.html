第13章 DiT扩散Transformer

大家好，我是你们的讲师。从 DDPM 到 Stable Diffusion，U-Net 一直是扩散模型去噪网络的标准架构。但在 2023 年，一项关键研究改变了这个格局——DiT（Diffusion Transformer）证明了 Transformer 可以取代 U-Net，并且在规模扩大时表现出更好的 Scaling Law（缩放定律）。到了 2024 年，SD 3.0 和 FLUX 全面拥抱 DiT 架构，U-Net 时代正式让位。

这一章，我们将深入分析 DiT 的设计思想：为什么 Transformer 比 U-Net 更适合扩散模型？patchify 如何将潜空间特征图转化为 token 序列？AdaLN 机制如何注入时间步和条件信息？以及，MM-DiT 双流架构（SD3/FLUX 的核心创新）如何分别处理文本和图像特征？

13.1 从 U-Net 到 Transformer：为什么需要 DiT

U-Net 在扩散模型中表现优异，但它有一些本质上的局限性：

  • 局部感受野：卷积操作的感受野受限于卷积核大小，虽然在深层特征图上有全局视野，但信息传递效率不如 Transformer 的全局注意力
  • 缩放不易：增大 U-Net 的参数量（增加通道数、加深层数）带来的收益递减明显，scaling law 不如 Transformer 平坦
  • 条件注入笨重：Cross-Attention 需要被嵌入到 U-Net 的特定层，架构修改不灵活

DiT（Peebles & Xie, 2023）直接回答了这个问题：如果我们将潜空间特征图视为一个 patch 序列，用标准的 Transformer 架构来做扩散，会发生什么？答案是：效果比 U-Net 更好，而且模型越大，优势越明显。

🌟 核心洞察：DiT 的成功不是偶然的。Transformer 的全局注意力机制天然适合扩散模型——因为去噪过程中，图像中相距很远的像素也需要协同决策（比如"一个物体的左右两侧需要对称"）。U-Net 通过下采样-上采样的瓶颈和跳跃连接间接实现全局交互，而 Transformer 通过自注意力直接做到这一点。

13.2 patchify：将潜空间特征图切分为 patch 序列

DiT 的第一步操作是 patchify——将潜空间特征图切分成一个序列的 patch（图像块），类似于 ViT 处理 RGB 图像的方式。

# DiT patchify 过程（简化）
# 输入潜特征图: [B, C, H, W] = [batch, 4, 64, 64]
# patch_size = 2, 则每 2×2 区域被展平为一个 token

# 将 64×64 按 2×2 切分 → 32×32 = 1024 个 patches
# 每个 patch 的特征维度: 4 × 2 × 2 = 16
# 线性投影到 d_model 维度（如 768 或 1152）
# 输出序列: [B, 1024, d_model]

# 再加上位置编码（sin/cos 或可学习的位置嵌入）
x = PatchEmbed(latents, patch_size=2)  # → [B, N, d]
x = x + pos_embedding[:, :N, :]        # 加位置编码

patch size 是一个重要的设计选择：

Patch SizeToken 数量（64×64 潜空间）参数量效果
21024最多最精细的空间控制，计算量大
4256中等效率与效果的良好平衡
864最少高吞吐，但空间细节有损

标准的 DiT 通常使用 patch_size=2 以保留空间细节，同时在计算开销和生成质量之间取得平衡。

13.3 DiT Block 设计：AdaLN 与条件注入

DiT 的核心创新不在于 Transformer 本身，而在于如何将时间步 t 和条件信息（如类别标签或文本嵌入）注入到 Transformer Block 中。DiT 提出了几种设计方案：

设计方案条件注入方式参数量增加效果（FID）
In-context将条件作为额外 token 拼接到序列中微小一般（条件信号不够强）
Cross-Attention在 Self-Attention 后加 Cross-Attention 层中等良好但计算量大
AdaLN（默认方案）用条件向量预测 LayerNorm 的 γ 和 β 参数微小最佳
AdaLN-ZeroAdaLN + 残差连接前的门控缩放（初始为 0）微小最佳（训练更稳定）

AdaLN（Adaptive Layer Normalization）是 DiT 的默认选择。其核心思想是：不用将条件信息"注入"到序列中，而是用条件信息来调整归一化层的参数：

# AdaLN 的计算流程
# 1. 将时间步 t 编码并经过 MLP 得到缩放和偏移参数
t_emb = MLP(SiLU(Linear(TimeEmbed(t))))  # → scale, shift
γ, β = chunk(t_emb, 2, dim=-1)

# 2. 在 LayerNorm 后应用自适应缩放和偏移
x_norm = LayerNorm(x)
x_adapted = (1 + γ) · x_norm + β

# 3. 送入 Transformer 的子层
x = x + Attention(x_adapted)
x = x + MLN(LayerNorm(x))  # 使用另一组 γ, β

AdaLN-Zero 在此基础上增加了一个可学习的零初始化门控参数 α，在训练开始时让 DiT Block 退化为恒等映射（Identity Mapping），从而避免了训练初期的不稳定性。这个技巧借鉴了 ResNet 的"零初始化残差"思想，被证明对稳定深层次 DiT 的训练至关重要。

💡 设计原理：AdaLN 之所以比 Cross-Attention 更有效，原因在于条件信息（时间步 t、类别标签）通常是全局性的——它告诉你"现在要去噪到什么程度"或"生成什么类别的图像"，而不是告诉某个特定位置的像素应该如何变化。用全局条件去调节全局的归一化参数，比用 Cross-Attention 逐位置注入更符合问题的本质。

13.4 Scaling Law：GFLOPs vs FID

DiT 论文中最引人注目的实验结果是它的 Scaling Law 研究。研究者训练了不同大小的 DiT 模型，并测试了计算量（GFLOPs）与生成质量（FID）之间的关系：

模型参数量GFLOPs（单步）ImageNet 256×256 FID
DiT-S33M1.19.62
DiT-B130M2.55.66
DiT-L458M5.43.82
DiT-XL675M8.62.27
DiT-XL（CFG=1.5）675M17.2（两次前向）2.27

实验结果显示，DiT 的计算量（GFLOPs）与生成质量（FID）呈近似对数线性关系——投入越多的计算，质量稳定提升，没有出现明显的"收益递减"。这种优秀的 Scaling Law 是 U-Net 无法达到的——U-Net 增大到一定程度后继续增加参数收益甚微。

🌟 核心洞察：DiT 的 Scaling Law 意味着"只要算力够，质量就能继续提升"。这解释了为什么 SD 3.0（8B 参数）和 FLUX（12B 参数）选择了 DiT 架构——工业界预见到了更大的模型会带来更好的效果，而 U-Net 无法支撑这个 scaling 趋势。

13.5 Class-Conditional DiT

DiT 的训练框架同时支持无条件生成（不需要任何条件信号）和条件生成（给定类别标签或文本）。以 class-conditional 为例：

  • 条件嵌入：类别标签 y 被映射为一个嵌入向量 c，与时间步嵌入 t_emb 相加（或拼接），一起送入 AdaLN 模块
  • 无分类器指导：DiT 同样使用 CFG，训练时有概率将条件标签替换为"空标签"（null class），从而让模型同时具备条件和无条件生成能力
  • Dropout 比例：通常设置为 0.1，即有 10% 的样本在训练时不使用条件标签

DiT 的类条件生成在 ImageNet 上达到了当时最好的 FID=2.27（256×256），大幅超越了之前基于 U-Net 的扩散模型（ADM 的 FID=4.59）。

13.6 MM-DiT 双流架构

DiT 最初是为 class-conditional 图像生成设计的，但 SD 3.0 和 FLUX 需要的是一种能同时处理文本和图像两种模态的架构。这就催生了 MM-DiT（Multimodal DiT）——一个双流（Dual-Stream）的 Transformer 架构。

MM-DiT 的核心设计：

  • 独立编码：文本特征和图像特征各自使用独立的 Transformer 权重进行编码
  • 交互注意力：在每层 Transformer 之后，文本和图像序列之间进行注意力交互（有时称为"双向 Cross-Attention"或"联合注意力"）
  • RoPE 位置编码：使用旋转位置编码（Rotary Position Embedding）替代标准的位置编码，更好地编码相对位置信息
# MM-DiT Block 简化流程
def mm_dit_block(img_tokens, txt_tokens, t_emb):
    # 1. 图像流：Self-Attention + AdaLN
    img = AdaLN(img_tokens, t_emb)
    img = SelfAttention(img) + img_tokens

    # 2. 文本流：Self-Attention + AdaLN
    txt = AdaLN(txt_tokens, t_emb)
    txt = SelfAttention(txt) + txt_tokens

    # 3. 跨模态交互：图像和文本之间做 Cross-Attention
    img = CrossAttention(Q=img, K=txt, V=txt) + img
    txt = CrossAttention(Q=txt, K=img, V=img) + txt

    # 4. FFN
    img = FFN(AdaLN(img, t_emb)) + img
    txt = FFN(AdaLN(txt, t_emb)) + txt

    return img, txt
⚠️ 真实的踩坑故事：我尝试在 8×A100 上从零训练一个 DiT 类模型时，遇到了一个极度隐蔽的问题：训练到第 10 万步后，loss 突然震荡，生成结果开始出现"黑白噪点斑块"。检查了所有超参数和代码逻辑都没有发现问题。最后发现是 AdaLN 中的 LayerNorm 精度不足——当使用 float16 混合精度训练时，LayerNorm 在 fp16 下的数值范围不足以表达某些层的 γ、β 参数的微小变化，导致梯度不稳定。解决方案是在 LayerNorm 操作中使用 fp32 精度（即"mixed precision with fp32 LayerNorm"）。这个问题在 Hugging Face 社区中也多次被报告。教训：大规模 DiT 训练时，LayerNorm 一定要在 fp32 下计算。

DiT 架构将扩散模型带入了一个新的时代——它不仅在质量上超越 U-Net，更重要的是展示了优秀的 Scaling Law，让更大的模型可以持续带来更好的生成效果。下一章，我们将把扩散模型从图像生成扩展到视频生成领域，看看"时序维度"的加入带来了哪些挑战和创新。
