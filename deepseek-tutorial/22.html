第22章 DeepSeek-Math

2023 年，一群来自 MIT 的研究人员做了一个简单实验：他们把 GPT-4 和几个开源模型叫到一起，出了一道小学五年级的数学题——"小明有 15 个苹果，给了小红 3 个，又买了 8 个，他现在有多少个？"结果让人大跌眼镜——多个模型给出了错误的答案，有的甚至做了完全无关的计算。最有趣的是，当你追问这些模型的推理过程时，它们常常给出"正确的推理"但"错误的答案"，或者"错误的推理"但"正确的答案"。

这个实验揭示了一个深刻的真相：大模型并没有真正"理解"数学。它们只是在模仿训练数据中的模式。当问题稍微偏离训练分布时，模型就会暴露出推理能力的脆弱性。DeepSeek 的团队在研究数学推理时也发现了类似的痛点。他们决定从数据的根源入手——构建一个百万级的数学推理语料库，并设计一套针对性的预训练和微调方案。

22.1 数学推理的独特挑战

数学不同于其他 NLP 任务。它有三个层面的挑战：

第一，精确性要求。自然语言生成可以容忍"大致正确"，但数学不行。你算出一个方程的解是 x=3.14159，而正确答案是 x=3.14158——在数学上是错误的。数学推理的每一个步骤都必须精确，任何一步出错都会导致最终答案错误。

第二，多步推理。一个复杂数学问题往往需要 5-10 步甚至更多步的推理。每一步的误差都会累积，步数越多模型越容易"走丢"。DeepSeek 的团队观察到一个有趣现象：模型在 3 步以内的推理中表现良好，但超过 5 步后准确率急剧下降。

第三，符号泛化。数学是高度抽象的。模型需要理解"x+y"中的 x 和 y 是变量，而不是具体的数值。很多模型能够解决"2+3=?"，但面对"a+b=?"时就不知所措了。真正理解数学意味着模型要学会处理符号化的、抽象的表达。

一个真实发现：DeepSeek 团队在分析模型错误时发现了一个"数值偏好"现象——模型在训练数据中见过太多 "3.14" 和 "2.718"，所以当遇到需要输出数学常数的问题时，倾向于输出这两个值，即使题目跟圆周率和自然常数毫无关系。这说明模型学到的不是数学推理，而是"最常见的数字是什么"的统计规律。要打破这种模式，必须在训练数据中加入大量的、多样性的数学推理样本。

22.2 百万级数学语料

DeepSeek-Math 的核心资产是 百万级的高质量数学语料。这个语料的构建过程极具匠心：

数据来源

DeepSeek 的数学语料不仅包含常见的数学教材和考试题，还从多种渠道搜集数据：

  • 教科书和竞赛题——从小学数学到高等数学，从普通考试到国际数学奥林匹克（IMO）
  • 数学论文——arXiv 上的数学论文、定理证明等
  • 数学社区——Math StackExchange、MathOverflow 等问答社区的推理过程
  • 教学材料——Khan Academy、MIT OCW 等教学课程的逐步骤解
  • 定理证明——形式化数学证明（Lean、Coq 等定理证明器的数据）

分步推理标注

DeepSeek 做了一项细致的工作：对语料中的每道数学题，不仅保留"问题-答案"对，更重要的是保留完整的、逐步骤的推理过程。这些推理过程是模型学习数学思维的关键。

数据质量控制的措施包括：

  • 去重——精确去重和语义去重，消除因题目表述不同但本质相同的数据
  • 质量筛选——用答题准确率过滤低质量数据
  • 难度分层——按题目难度分桶，确保各难度级别的数据均衡
难度级别 示例 数据占比 来源
基础算术 四则运算、小数分数计算 15% 教材、习题集
代数与几何 方程求解、几何证明 30% 中学教材、中考题
高等数学 微积分、线性代数、概率统计 30% 大学教材、考研题
竞赛数学 数论、组合数学、不等式 15% IMO、竞赛真题
研究级数学 论文级证明、形式化验证 10% arXiv、定理证明库
深意：为什么要大费周章地覆盖从算术到研究的全难度范围？因为 DeepSeek 团队发现，基础数学能力是高级推理的基石。一个在"2+3"上都会出错的模型，绝不可能做出复杂证明。反之，一个在基础算术上训练扎实的模型，面对复杂问题时更有可能做出正确的分步推理。

22.3 多阶段预训练

DeepSeek-Math 的训练采用了一个三阶段的策略：

第一阶段：通用预训练

使用 DeepSeek-LLM 基座模型（在通用语料上预训练过的 checkpoint）。这个阶段模型已经具备了语言理解、世界知识和基本的推理能力。

第二阶段：数学继续预训练（Math Continue Pretraining）

在通用预训练之后，用百万级数学语料对模型做继续预训练（而不是从零开始）。这一阶段的关键设计是：

  • 使用较低的学习率（约为初始预训练的 1/10），防止灾难性遗忘通用能力
  • 数学语料占比从 10% 逐渐提升到 50%
  • 保持数据多样性的同时，增加数学推理样本的权重

第三阶段：数学微调（Math SFT）

在继续预训练之后，使用高质量的数学推理数据做监督微调。微调数据全部包含完整的分步推理链（Chain-of-Thought）。这一阶段的目标是教会模型"如何正确输出推理过程"。

# 训练数据格式示例
{
  "problem": "一个等差数列的首项为 3，公差为 4，求第 10 项的值。",
  "reasoning": [
    "第一步：等差数列的通项公式为 an = a1 + (n-1) × d",
    "第二步：代入已知条件，a1 = 3, d = 4, n = 10",
    "第三步：a10 = 3 + (10-1) × 4 = 3 + 36 = 39"
  ],
  "answer": "39",
  "difficulty": "easy",
  "source": "math_textbook_v2"
}

这个三阶段策略与"从零开始训练一个数学模型"的区别在于：它充分利用了通用预训练积累的知识和语言能力，再通过数学专项训练来激发推理潜力。DeepSeek 的实验表明，这种方法比从零训练节省了约 60% 的总计算量。

22.4 GSM8K / MATH 基准

DeepSeek-Math 在发布时，在两个最重要的数学基准上取得了开源模型的同期 SOTA：

模型 参数量 GSM8K MATH
LLaMA-2-7B 7B 16.7% 3.9%
LLaMA-2-70B 70B 56.8% 13.7%
Mistral-7B 7B 36.7% 11.8%
DeepSeek-Math-7B 7B 82.7% 46.8%
DeepSeek-Math-67B 67B 87.2% 52.1%
GPT-3.5-Turbo 未知 80.5% 43.5%
GPT-4 未知 94.5% 62.7%

7B 的 DeepSeek-Math 在 GSM8K 上达到了 82.7%，已经超过了 GPT-3.5-Turbo。67B 版本更是在两个基准上都大幅领先同尺寸的 LLaMA-2。虽然与 GPT-4 还有差距，但考虑到 GPT-4 的参数量级和训练成本，DeepSeek-Math 在"性价比"上遥遥领先。

需要注意：GSM8K 是一个针对小学数学的测试集，主要考察多步算术推理。MATH 则涵盖从代数到微积分的更广泛数学领域，难度显著高于 GSM8K。DeepSeek-Math 在两个基准间的一致表现说明，它的数学能力不是"应试技巧"——不是在特定题型上的过拟合，而是真正具备了对不同难度和类型数学问题的泛化推理能力。

22.5 符号推理与形式化数学

DeepSeek-Math 的一个更深入的探索方向是 符号推理和形式化数学。这涉及两个层面的工作：

符号计算——模型学习处理符号化的数学表达式，而不是仅仅依赖数值计算。例如，求解 ∫ x² sin(x) dx，模型需要能够输出符号解 2x sin(x) - (x² - 2) cos(x) + C，而不是输出一个近似数值。

定理证明——DeepSeek-Math 尝试使用 Prover 验证 来提升推理的严谨性。具体来说，模型生成的数学证明被送入定理证明器（如 Lean、Coq）进行形式化验证。如果证明通过，说明推理链是正确的；如果不通过，这可以作为负反馈信号来改进模型。

# 使用 Lean 定理证明器验证模型生成的数学证明
# 模型输出：a² + b² ≥ 2ab 的证明

theorem am_gm_sq (a b : ℝ) : a^2 + b^2 ≥ 2*a*b :=
by
  have h : (a - b)^2 ≥ 0 := by
    apply pow_two_nonneg
  calc
    a^2 + b^2 = (a - b)^2 + 2*a*b := by ring
    _ ≥ 2*a*b := by
      linarith

-- ✅ Lean 验证通过

形式化验证的优势在于它的"零容忍"标准——不存在"大致正确"的证明，要么全对要么全错。这种严格的反馈机制迫使模型生成更严谨、更结构化的推理链。

跨领域影响：形式化数学训练带来的好处不仅限于数学。DeepSeek 发现，经过数学证明训练的模型，在编写代码时的"bug 率"也显著降低。原因很简单——写数学证明和写代码都需要严谨的逻辑、逐步的推理和精确的表达。数学训练提升了模型的整体逻辑链质量，这种能力自然地迁移到了代码生成和其他推理任务中。

22.6 对通用推理的提升

DeepSeek-Math 最有价值的发现之一，可能不是数学本身，而是数学训练对通用推理能力的提升。

团队做了一个消融实验：分别用通用语料和数学语料继续预训练两个同尺寸的模型，然后在非数学的推理任务上做对比。结果令人振奋：

评估任务 通用继续预训练 数学继续预训练 提升
BBH（Big-Bench Hard） 45.2% 52.8% +7.6%
MMLU（通用知识） 62.1% 65.3% +3.2%
HellaSwag（常识推理） 71.5% 74.1% +2.6%
ARC-Challenge（科学推理） 55.8% 61.2% +5.4%
HumanEval（代码生成） 38.5% 42.3% +3.8%

数学继续预训练在几乎所有推理任务上都带来了正面提升。这说明数学推理训练不是狭窄的"偏科"，而是对"逻辑思维能力"的全面提升。这背后的原理是：数学推理涉及大量的模式识别、多步规划、假设验证和回溯纠错，这些能力在本质上适用于一切需要逻辑推理的任务。

给读者的建议：如果你正在训练一个通用大模型，千万别忽略数学数据的价值。DeepSeek 甚至建议在预训练阶段就加入 10-20% 的数学数据——不会降低通用能力，反而会显著提升推理能力。数学是"思维的体操"，这个古老的格言在 LLM 训练中同样成立。

DeepSeek-Math 的成功，不仅让开源模型在数学推理上达到了新的高度，更揭示了一个更深刻的规律：训练数据的多样性和质量，比模型大小更重要。一个 7B 的模型，如果训练数据和策略得当，可以在数学推理上超越数十倍大的模型。

从 DeepSeek-Coder 到 DeepSeek-Math，我们看到了一条清晰的路线：深入理解垂直领域的数据特性，设计针对性的训练策略，然后在特定领域达到顶尖水平。这种"垂直深耕"的思路贯穿了整个 DeepSeek 系列。
