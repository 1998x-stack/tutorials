19.1 从神经元到神经网络

前面学的所有算法——线性回归、决策树、SVM——都是"传统机器学习"。从这一章开始，我们进入"深度学习"。这两者的本质区别是什么？传统ML需要人工做特征工程（人脑提取规律→喂给模型），深度学习让模型自己从原始数据中学习层次化的特征表示。

深度学习的核心构件还是那个最简单的数学模型——感知机。1943年McCulloch和Pitts提出了人工神经元的数学模型，1958年Rosenblatt实现了第一个感知机。直到2006年Hinton提出深度信念网络，神经网络才真正"复兴"。经历了近70年的浮沉，深度学习已经是AI的绝对主流。

19.2 神经元：深度学习的最小单元

一个人工神经元做的事情和逻辑回归几乎一模一样：加权求和 → 过激活函数 → 输出。公式就是：

output = f(w₁x₁ + w₂x₂ + ... + wₙxₙ + b)

一个神经元就是一个微型分类器。把成百上千个神经元堆叠成多层，就构成了神经网络。每一层学习不同抽象层次的特征：底层学边缘/颜色（CNN），中层学形状/纹理，高层学语义/物体。

19.3 激活函数：没有它，再多层也是线性

如果不加激活函数（或激活函数是线性的），无论堆多少层，整个网络等价于一个线性变换——白白浪费深度。激活函数引入非线性，让网络有能力逼近任意复杂函数。

激活函数公式优点缺点
Sigmoid1/(1+e^(-x))输出在(0,1)，可解释为概率梯度消失，输出不以0为中心
Tanh(e^x-e^(-x))/(e^x+e^(-x))输出以0为中心(-1,1)仍有梯度消失
ReLUmax(0, x)计算简单，缓解梯度消失负半区梯度为0（神经元"死亡"）
LeakyReLUmax(0.01x, x)解决了神经元死亡负半区斜率需手动设
GELUx·Φ(x)Transformer标配计算略贵
⚠️ "ReLU死亡"问题：如果学习率太大或梯度更新太猛，可能导致某些神经元的权重被更新到一个状态：对所有输入都输出负值。ReLU对这些负值输出0，梯度为0，这个神经元就永远"死"了——再也不会被更新。使用较小的学习率、LeakyReLU、或Batch Normalization都可以缓解这个问题。

19.4 反向传播：深度学习训练的基石

反向传播（Backpropagation）是训练神经网络的核心算法。网上很多教程把它讲得神乎其神，但本质就是链式法则——从输出层到输入层，逐层计算损失对各参数的梯度。

直觉：你蒙着眼睛，一个朋友告诉你离目标差多远（前向传播计算损失），然后你倒着往回走，每一步感受脚下的坡度（反向传播求梯度），然后调整你的位置（梯度下降更新参数）。反向传播就是那"倒着走"的过程。

19.5 初始化：好的开始是成功的一半

权重初始化的选择直接决定了训练能否成功。全零初始化 → 所有神经元学到相同的东西（对称性问题）。全随机大值 → 激活值进入饱和区，梯度消失。

业内标准做法：

  • Xavier/Glorot初始化：适用于Sigmoid/Tanh激活函数。权重从均值为0、方差为2/(fan_in+fan_out)的分布采样
  • He/Kaiming初始化：适用于ReLU激活函数。方差为2/fan_in。有ReLU的地方就用He初始化
import torch
import torch.nn as nn

# PyTorch默认使用Kaiming初始化用于ReLU层
layer = nn.Linear(128, 64)
nn.init.kaiming_normal_(layer.weight, mode='fan_in', nonlinearity='relu')
nn.init.zeros_(layer.bias)

19.6 Batch Normalization：深度网络的"稳定剂"

Batch Normalization (BN) 在每一层的激活值上做标准化（减去均值，除以标准差），让它保持在梯度不消失、不饱和的区间。效果：可以用更大学习率、对初始化不那么敏感、有轻微正则化效果、训练更快。

BN的原理是：对每个mini-batch，计算均值和方差 → 标准化 → 再用可学习的γ和β做缩放和平移（恢复网络的表达能力）。推理时用的是训练集上的移动平均统计量。

🔑 现代网络标配：Linear → BatchNorm → ReLU → Dropout。这个顺序已经成了大多数网络的默认模块。注意一个小细节：BN在激活函数之前还是之后？原始论文放在激活函数之前（pre-activation），但实践中有人放在之后。两种都有效，保持一致就好。

19.7 Dropout：最简单的正则化

Dropout在训练时随机"关掉"一定比例的神经元（比如50%），强迫网络学习冗余表示——每个神经元不能依赖特定的其他神经元，必须独立地有用。这本质上是一种隐式的模型集成：每次dropout产生一个不同的子网络，测试时取所有子网络的平均效果。

关键操作：训练时dropout（p=0.5），推理时不用dropout但要乘以(1-p)做缩放。PyTorch中nn.Dropout自动处理了训练/推理模式的切换。
