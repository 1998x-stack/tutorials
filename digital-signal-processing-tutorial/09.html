第9章 FIR滤波器设计

你有没有注意过一个细节：当你把音响的高音旋钮拧到最大时，高频的嘶嘶声变得很明显，但音乐中的"空气感"也出来了；当你把低音旋钮拧到最大时，鼓点变得震撼，但人声可能变得模糊含混。音响上的EQ（均衡器）本质上就是一组滤波器——它允许你"选择性地"增强或衰减某些频率分量。

EQ算法在数字域的实现，最常见的就是FIR滤波器（Finite Impulse Response，有限冲激响应滤波器）。那么"有限冲激响应"是什么意思？简单说就是：你给滤波器输入一个单位脉冲（一个1后面全是0），它的输出在有限个样本后一定会归零。这听起来似乎很平凡，但这个"有限"特性却带来了一系列奇妙的工程性质——最诱人的就是绝对稳定和线性相位。

💡 一句话概括：FIR滤波器是所有数字滤波器中最"老实"的一个——它不会震荡、不会发散、而且可以做到所有频率的延迟时间完全一致（线性相位）。代价是：要达到陡峭的过渡带，你需要很长的滤波器阶数，计算量会比较大。

9.1 FIR滤波器的"三大招牌"

为什么FIR滤波器在工程界受到如此尊崇？三个核心原因：

1. 绝对稳定。FIR滤波器没有反馈回路（它是非递归的），系统函数H(z)只有零点没有极点——或者说所有极点都在原点。这意味着无论输入信号多大、多奇怪，输出永远是有界的。对于像航空电子、医疗设备这种"不允许任何不稳定"的场景，FIR是唯一的选择。

2. 线性相位（Linear Phase）。这是FIR最特别的招牌。一个线性相位滤波器对所有频率成分施加相同的群延迟（group delay）。用人话说：信号通过滤波器后，不同频率分量之间的相对时间关系保持不变。对于音频信号来说，这意味着波形形状不被"扯歪"——你听到的声音还是原来的声音，只是某些频率被衰减了。对于通信系统来说，线性相位意味着码间干扰（ISI）最小化。

3. 易于实现。FIR的实现就是"乘加乘加乘加"——一个简单的MAC（Multiply-Accumulate）循环。没有除法、没有迭代、没有稳定性担忧。

代价也很明确：要达到与IIR滤波器相同的过渡带陡峭度，FIR通常需要5-10倍以上的阶数。这意味着更多的乘法器、更高的功耗和更大的延迟。

9.2 线性相位的四种类型

要获得线性相位，滤波器的冲激响应h[n]必须满足某种对称性条件。具体来说有两种对称：

  • 偶对称：h[n] = h[N-1-n]
  • 奇对称：h[n] = -h[N-1-n]

结合阶数N是奇数还是偶数，一共构成四种类型：

类型 对称性 N（阶数+1） 相位响应 适用场景
Type I 偶对称 奇数 β = -(N-1)ω/2 低通、高通、带通、带阻
Type II 偶对称 偶数 β = -(N-1)ω/2 低通、带通（不能做高通/带阻）
Type III 奇对称 奇数 β = -(N-1)ω/2 + π/2 希尔伯特变换器、微分器
Type IV 奇对称 偶数 β = -(N-1)ω/2 + π/2 希尔伯特变换器、微分器

为什么Type II不能做高通和带阻？因为在ω=π（即f_s/2）处，偶对称且N为偶数的频率响应必为零。同理，Type III和IV在ω=0处必为零——所以不能做低通。设计滤波器时，通常首选Type I，因为它最灵活。

9.3 窗函数设计法：从理想走向现实

窗函数法是最直观的FIR设计方法，它的思路是这样的：

  1. 在频域构思一个理想滤波器（比如截止频率为ω_c的理想低通）。
  2. 做IDFT得到无限长的冲激响应h_d[n] = sin(ω_c n) / (π n)。
  3. 截断到有限长度N——相当于用一个矩形窗w[n]截取h_d[n]。
  4. 结果就是FIR系数：h[n] = h_d[n] · w[n]。

听起来很简单对吧？但这里有一个深刻的数学问题：截断（加矩形窗）在频域等价于"理想频率响应与sinc函数的卷积"——而sinc函数的旁瓣在截止频率处产生了著名的Gibbs现象：通带边缘出现约9%的过冲和振铃，而且不管你怎么增加滤波器阶数N，这个过冲都不会消失（只会变得更窄）。

⚠️ 踩坑实录：很多年前我做一个心率监测仪的信号处理部分，需要设计一个0.5Hz-40Hz的带通FIR滤波掉工频干扰和高频噪声。我用矩形窗设计了一个128阶的FIR，仿真时效果非常好。结果上了硬件（16位定点DSP），发现心率波形在R波峰值附近出现了奇怪的"振铃"——每个心跳峰值之后跟着一串小的假峰。排查了整整一天才发现是矩形窗的Gibbs效应在作祟！切换到Blackman窗后，振铃消失了，虽然过渡带稍微宽了一点，但波形质量完全不可同日而语。

9.4 常用窗函数对比

既然矩形窗不行，那用什么窗？工程界已经积累了一整套成熟的窗函数库。它们都是通过对理想冲激响应进行"渐近截断"（边缘平滑衰减到零）来抑制旁瓣的。但天下没有免费的午餐——抑制旁瓣必然以展宽主瓣（即过渡带）为代价。

窗函数 主瓣宽度 旁瓣衰减（dB） 过渡带宽度 特点
矩形（Rectangular） 4π/N -13 0.92/N 最窄主瓣，但旁瓣太高
Hanning 8π/N -31 3.1/N 通用用途，旁瓣衰减好
Hamming 8π/N -41 3.3/N 第一旁瓣极大衰减，常用
Blackman 12π/N -57 5.5/N 旁瓣极低，过渡带宽
Kaiser（β=5） 可调 -40 可调 最灵活，β参数可调旁瓣

选择窗函数的工程原则：

  • 如果你只需要最大的阻带衰减，选Blackman或Kaiser（β大）。
  • 如果你需要最窄的过渡带，接受一些旁瓣，选Hamming。
  • Kaiser窗最灵活——你可以通过调节β参数在过渡带宽度和阻带衰减之间连续折中。一个经验公式：阻带衰减A_s（dB）和β的关系为β = 0.1102(A_s - 8.7)。

9.5 频率采样法

窗函数法是从时域出发的。频率采样法则反过来——先在频域"画"出你想要的频率响应，再通过IDFT得到时域冲激响应。

具体做法：

1. 确定N个均匀分布在[0, 2π)上的频率点 ω_k = 2πk/N
2. 在这些频率点上指定期望的幅度响应 H_d[k]
3. 对 H_d[k] 做 N 点 IDFT，得到 h[n]
4. h[n] 就是 FIR 系数

频率采样法的优点是直观——你可以精确控制某些频点上的响应。但缺点也很明显：

  • 采样点之间的频率响应不受控，可能会出现较大的波纹（尤其是在过渡带附近）。
  • 要减少波纹，一种做法是在过渡带中插入"过渡采样点"（transition samples），用优化方法（如线性规划）调整这些点的值来最小化最大波纹。

频率采样法在1980年代之前曾是主流，但现在大多数工程场景已经被窗函数法或Parks-McClellan算法取代了。

9.6 Parks-McClellan算法简介

如果说窗函数法是"经验派"，那么Parks-McClellan算法（也叫Remez交换算法）就是"优化派"的巅峰。它直接解决了这样一个问题：给定滤波器阶数N、通带边界、阻带边界，在通带和阻带内最小化最大波纹（即等波纹设计）。

简单理解：窗函数法得到的滤波器在通带和阻带内的波纹大小不一致（通常靠近截止频率处的波纹较大），而Parks-McClellan算法会让所有波纹"均匀化"——也就是在最小化最大波纹的意义上做到了最优。这也是为什么这种设计也叫等波纹（Equiripple）FIR。

在Python中，你可以用scipy.signal中的remez或firls来实现：

# 40阶低通FIR，截止频率0.3（归一化），使用Parks-McClellan
from scipy import signal
taps = signal.remez(41, [0, 0.3, 0.35, 0.5], [1, 0], fs=1)

# 或者用最小二乘法（频率采样法的优化版本）
taps = signal.firls(41, [0, 0.3, 0.35, 0.5], [1, 1, 0, 0], fs=1)

两个函数的区别：remez追求"最小化最大误差"（minimax），而firls追求"最小化均方误差"。前者在通带和阻带内波纹均匀但可能在截止频率有较大过冲（对某些应用有风险），后者误差分布更平滑。各有千秋。

🔧 工程建议：当你需要设计一个FIR时，试一下这个优先级：先试scipy.signal.firwin（窗函数法），如果过渡带不够陡峭或阻带衰减不够，升级到firls（最小二乘法），如果还不够好——而且你非常确定不需要IIR——最后再用remez（等波纹法）。你95%的需求firwin加上合适的Kaiser窗就能解决。

9.7 本章小结

这一章我们做了以下几件事：

  • 介绍了FIR滤波器的三大特性（绝对稳定、线性相位、易于实现）及其代价；
  • 详细分析了线性相位的四种类型（Type I-IV）及其适用场景；
  • 推导了窗函数设计法的全流程——从理想低通到截断到加窗，并用一个心率监测的踩坑故事说明了Gibbs现象的后果；
  • 对比了五种常用窗函数（矩形/Hanning/Hamming/Blackman/Kaiser）的主瓣宽度和旁瓣衰减，给出了工程选型建议；
  • 介绍了频率采样法及其优缺点；
  • 简要介绍了Parks-McClellan等波纹最优设计方法和firwin/firls/remez的选型策略。

FIR设计是滤波器设计的"入门必修课"。掌握了FIR之后，下一章我们将进入它的"对手"——IIR滤波器的世界。IIR用更少的阶数实现更陡的过渡带，但代价是非线性相位和稳定性需要验证。FIR和IIR的选型之争，是数字滤波器设计中永恒的议题。
