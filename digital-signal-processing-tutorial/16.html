第16章 高级自适应算法

上一章我们学习了LMS算法。它简单、优雅、实用——但正如我们反复强调的，LMS有一个"硬伤"：收敛速度受输入信号自相关矩阵的特征值散布影响严重。如果你的输入是白噪声，LMS收敛飞快；但如果输入是强相关的有色信号（比如语音），LMS会慢得让你怀疑人生。

这一章我们来升级自适应武器库。我们将学习三种更强大的工具——RLS（递推最小二乘）、Kalman滤波器和MVDR波束形成。它们分别从不同角度突破了LMS的局限，虽然计算代价更高，但收敛速度和跟踪能力也上了一个台阶。

💡 核心认知：从LMS到RLS到Kalman，自适应算法的演进有一条清晰的脉络——对信号/系统的建模越来越精细，获取的性能增益也越来越大。如果你把LMS看作"盲人摸象"，那RLS就是"戴上眼镜看"，而Kalman则是"拿着地图看"。

16.1 RLS算法：最小二乘的递推之美

LMS的核心思想是最小均方——最小化误差的统计期望。而对于RLS（Recursive Least Squares，递推最小二乘），它的优化目标变了：最小化从开始到当前时刻所有误差的加权平方和。

具体来说，RLS的代价函数是：

\(J(n) = \sum_{i=1}^{n} \lambda^{n-i} e^2(i)\)

其中 \(\lambda\) 是遗忘因子（forgetting factor），取值范围通常为 \(0.95 < \lambda < 1\)。\(\lambda\) 越接近1，算法越"记性好"——老的误差样本权重不衰减太多，稳态精度高但跟踪慢；\(\lambda\) 越小，算法越"健忘"——更看重最近的误差，跟踪能力强但稳态噪声大。

RLS的关键步骤是：利用矩阵求逆引理（Woodbury公式），直接递推更新自相关矩阵的逆矩阵 \(\mathbf{P}(n) = \mathbf{R}^{-1}(n)\)，而不用每次都做矩阵求逆。这个技巧是RLS的核心——它将每步计算从 \(O(M^3)\) 降到了 \(O(M^2)\)。虽然比LMS的 \(O(M)\) 高一个量级，但避免了矩阵求逆这个不可接受的操作。

// RLS算法伪代码
初始化: w(0) = 0, P(0) = δ^{-1} * I  (δ 是一个小的正数)
对每个时刻 n = 1, 2, 3, ...:
    // 1. 计算增益向量
    k(n) = P(n-1) * x(n) / (λ + x^T(n) * P(n-1) * x(n))

    // 2. 计算先验误差
    e(n) = d(n) - w^T(n-1) * x(n)

    // 3. 更新系数
    w(n) = w(n-1) + k(n) * e(n)

    // 4. 更新逆相关矩阵
    P(n) = (P(n-1) - k(n) * x^T(n) * P(n-1)) / λ

看到这个流程，你是否注意到一个有趣的相似性？RLS的结构和Kalman滤波器有着惊人的对称性——增益向量 \(k(n)\) 和Kalman增益 \(K(n)\) 扮演的角色几乎相同。事实上，RLS可以看作Kalman滤波器在特定条件下的特例。这点我们在下一节还会提到。

16.2 RLS vs LMS：公平的对比

让我们把两个算法放在一张表格里做对比。这是你在选择自适应算法时必须做的权衡决策。

维度 LMS RLS
优化准则 最小均方（最小化 \(E[e^2]\)） 最小二乘（最小化 \(\sum e^2\)）
每步复杂度 \(O(M)\) \(O(M^2)\)
收敛速度 慢（受特征值散布影响） 快（约5-10倍于LMS）
对输入相关性的敏感度 高（有色输入大幅降速） 低（白化输入后收敛）
稳态失调 可控（通过 \(\mu\) 调节） 理论为零（在时不变系统中）
数值稳定性 好 差（\(\mathbf{P}(n)\) 可能失去正定性）
存储需求 \(O(M)\) \(O(M^2)\)（存储 \(\mathbf{P}\) 矩阵）
🔧 工程选择建议：
  • 选LMS的场景：嵌入式平台资源受限（如MCU、DSP芯片）、滤波器阶数高（M>100）、输入信号近似白噪声；
  • 选RLS的场景：PC或FPGA上有充足算力、需要快速收敛（如信道均衡的训练序列很短）、输入信号相关性很强；
  • 折中方案：仿射投影算法（APA）是LMS和RLS之间的折中——计算量 \(O(KM)\)，K为投影阶数，性能也在两者之间。

16.3 Kalman滤波器：状态空间视角下的最优估计

如果说LMS是"看见什么学什么"，RLS是"记住所有然后找最优"，那么Kalman滤波器就是"根据演化模型预测+用观测数据修正"。

Kalman滤波器由Rudolf Kalman在1960年提出。它的基本思想是把问题建模成状态空间模型：

  • 状态方程： \(\mathbf{x}(n+1) = \mathbf{F} \mathbf{x}(n) + \mathbf{w}(n)\) ——描述了系统状态如何随时间演化；
  • 观测方程： \(\mathbf{z}(n) = \mathbf{H} \mathbf{x}(n) + \mathbf{v}(n)\) ——描述了如何从状态得到观测数据；

其中 \(\mathbf{w}(n) \sim \mathcal{N}(0, \mathbf{Q})\) 是过程噪声，\(\mathbf{v}(n) \sim \mathcal{N}(0, \mathbf{R})\) 是观测噪声。Kalman滤波器给我们的，是在已知这些噪声统计量的前提下，从带噪声的观测中递推估计系统状态。

Kalman滤波器的核心是一个"预测-更新"的两步循环：

// Kalman滤波器核心循环 — 预测-更新

// 预测步骤 (Predict / Time Update):
x_pred = F * x_est              // 先验状态估计
P_pred = F * P_est * F^T + Q    // 先验误差协方差

// 更新步骤 (Update / Measurement Update):
K = P_pred * H^T / (H * P_pred * H^T + R)  // Kalman增益
x_est = x_pred + K * (z - H * x_pred)       // 后验状态估计
P_est = (I - K * H) * P_pred                 // 后验误差协方差

这一步"预测→更新→预测→更新"的循环，就是Kalman滤波器的全部。它优雅、简洁，而且被证明是在高斯噪声假设下的最优线性估计器——在所有线性滤波器中，没有比它更好的。

从自适应滤波的角度来看，Kalman滤波器也可以用来在线估计自适应滤波器的系数。如果我们将待估计的滤波器系数 \(\mathbf{w}(n)\) 视为状态变量，那么RLS实际上就是Kalman滤波器的一个特例：当状态转移矩阵 \(\mathbf{F} = \mathbf{I}\) 且过程噪声协方差 \(\mathbf{Q}\) 取特定值时，两者等价。

16.4 Kalman滤波的经典应用

目标跟踪

想象一个雷达在跟踪一架飞机。每过一秒钟，雷达给出一个带有噪声的飞机位置测量值 \((z_x, z_y)\)。系统的状态可以定义为位置和速度 \([p_x, p_y, v_x, v_y]^T\)，状态转移矩阵 \(\mathbf{F}\) 描述匀速度运动模型：

\(\mathbf{F} = \begin{bmatrix} 1 & 0 & \Delta t & 0 \\ 0 & 1 & 0 & \Delta t \\ 0 & 0 & 1 & 0 \\ 0 & 0 & 0 & 1 \end{bmatrix}\)

Kalman滤波器能在每个时刻给出比原始测量更平滑的位置估计——因为它在"相信测量"和"相信运动模型"之间做了最优加权。一个有意思的调试经验：如果你看到Kalman滤波器的输出有明显的滞后（lag），通常意味着过程噪声协方差 \(\mathbf{Q}\) 设置得太小（太相信模型），需要适当增大。

IMU信号融合

手机、无人机、机器人上都有的IMU（惯性测量单元）包含加速度计和陀螺仪。加速度计低频准确但高频噪声大，陀螺仪高频响应好但存在漂移。Kalman滤波器的经典用法是用加速度计测量来修正陀螺仪的积分漂移——这正是手机定位和无人机姿态估计的基础。

⚠️ 踩坑实录：很多人第一次使用Kalman滤波器时，把过程噪声协方差矩阵 \(\mathbf{Q}\) 随便设了一个对角矩阵就跑。结果在目标突然改变运动方式（比如匀速直线运动的飞机突然拐弯）时，滤波器输出完全跟不上，出现了严重的"发散"现象。后来我才意识到，\(\mathbf{Q}\) 的选取不是"随便填一个数"——它编码了你对系统模型的"信心程度"：如果目标可能突然机动，\(\mathbf{Q}\) 就必须设得足够大。这是一个被称为"Q tuning"的经典难题，没有通用的公式，需要针对具体场景调试。

16.5 阵列信号处理与MVDR波束形成

现在我们进入自适应滤波的另一个分支——空域滤波。如果我们把上一章的自适应滤波器中的"时间延时节拍"替换为"空间位置（不同天线/麦克风）上的信号"，就从时域自适应滤波进入了阵列信号处理。

MVDR（Minimum Variance Distortionless Response，最小方差无失真响应）波束形成，也叫Capon波束形成，是自适应空域滤波的经典算法。

它的核心思想是：对于期望方向的信号，保持无失真通过（增益为1）；与此同时，最小化阵列输出的总功率（即最小化来自其他方向的干扰和噪声的贡献）。数学上，这可以写成一个约束优化问题：

\(\min_{\mathbf{w}} \mathbf{w}^H \mathbf{R} \mathbf{w} \quad \text{s.t.} \quad \mathbf{w}^H \mathbf{a}(\theta_0) = 1\)

其中 \(\mathbf{R}\) 是阵列接收信号的协方差矩阵，\(\mathbf{a}(\theta_0)\) 是期望方向的导向矢量（steering vector）。这个问题的解为：

\(\mathbf{w}_{\text{MVDR}} = \frac{\mathbf{R}^{-1} \mathbf{a}(\theta_0)}{\mathbf{a}^H(\theta_0) \mathbf{R}^{-1} \mathbf{a}(\theta_0)}\)

MVDR波束形成相比传统的延时求和波束形成（Delay-and-Sum），能更有效地抑制干扰。当干扰方向与期望方向接近时，常规波束形成的主瓣会"吞掉"干扰，而MVDR会在干扰方向自动形成一个深陷的零陷（null），这在雷达和智能麦克风阵列中极其有用。

波束形成方法 干扰抑制能力 需要DOA先验 计算量 鲁棒性
延时求和 (DAS) 弱 否（固定波束） \(O(M)\) 非常鲁棒
MVDR (Capon) 强（自适应零陷） 是（需知道期望方向） \(O(M^3)\) 求逆 中等（对导向矢量误差敏感）
LCMV 更强（多约束） 是 \(O(M^3)\) 高（可约束多个方向）
💡 有意思的细节：MVDR的 \(\mathbf{R}\) 矩阵求逆在实时系统中是"不可能完成的任务"。工程上常用的做法是用RLS递归地更新 \(\mathbf{R}^{-1}\)（类似RLS中的 \(\mathbf{P}(n)\)），或者使用LMS-like的梯度下降来渐近逼近MVDR解。另一种巧妙的思路是用GSC（Generalized Sidelobe Canceller）将约束优化转化为无约束优化，然后用LMS来实现。

16.6 从自适应滤波到智能信号处理

回顾第15章和本章，我们实际上走了一条从"手工固定设计"到"在线自适应学习"的进化之路。Wiener滤波器假设已知统计量，LMS勇敢地丢弃了这个假设，RLS又捡回了数据的历史信息，Kalman更进一步引入了状态演化模型，而MVDR则将自适应的思想从时域扩展到了空域。

这种"加入更多先验信息以获得更好性能"的范式，在当今的AI信号处理中同样存在——只是把"先验模型"换成了"训练数据"。你在第26章会看到，深度学习如何把自适应滤波的"在线学习"推向新的高度。

16.7 本章小结

  • RLS算法通过递推更新逆相关矩阵，用\(O(M^2)\)的计算量实现了比LMS快5-10倍的收敛速度，特别适合输入信号相关性强的场景；
  • Kalman滤波器建立了"预测-更新"的双步循环框架，在状态空间模型下给出最优线性估计；
  • Kalman滤波器在目标跟踪和IMU融合中是最经典的应用，过程噪声协方差 \(\mathbf{Q}\) 的调优是最需要经验的环节；
  • MVDR波束形成将自适应滤波从时域扩展到空域，在干扰方向形成零陷，是雷达和麦克风阵列的核心技术；
  • 从LMS到RLS到Kalman，体现了"更多先验知识→更好性能"的算法设计哲学。

下一章我们将转向另一个经典话题——谱估计进阶。你之前学的FFT周期图法只能给出分辨率受数据长度限制的功率谱。而AR模型谱、Burg算法、MUSIC方法能以更高的分辨率揭示信号的频谱结构——在某些条件下，甚至可以分辨频率间隔远小于FFT分辨率的两个正弦分量。
