第26章 2025-2026后训练数据新范式

大家好，我是你们的讲师。走到第26章，我们已经系统性地学习了后训练数据的方方面面——从分类体系到标杆数据集，从偏好对齐到Agent数据，从管线搭建到合规审查。可以说，前25章建立的是"在这个领域我想做什么都可以做到"的工程能力。

但后训练领域有一个让人又爱又恨的特点：变化太快。2023年大家还在纠结"52K够不够"，2024年焦点转向了"DPO vs PPO"，到了2025-2026年，游戏规则几乎被重写了一遍——合成数据从"备选方案"变成了"主流范式"，数据数量从"越多越好"变成了"少即是多"，训练方式从"需要外部数据"变成了"模型自己生成数据再训练自己"。

这一章，我们将聚焦2025-2026年最值得关注的四个新范式，它们正在从根本上改变后训练数据工作的方式。

后训练数据领域正在经历范式转换

在深入具体技术之前，我想先用一张对比表让你直观感受这种变化有多剧烈：

维度 2023-2024 旧范式 2025-2026 新范式
数据来源 以人工标注为主，模型生成为辅 以模型生成为主，人工标注作为种子/验证
数据规模 动辄数百万条 SFT 数据 几万到几十万条精炼数据即可达到或超越
数据获取方式 收集 + 标注 自我博弈 + 自动课程生成 + 数据飞轮
质量管控 人工审核 + 规则过滤 模型自评估 + 可验证奖励信号
迭代速度 周级别（标注周期长） 天级别（合成数据即时生成）
代表作品 Alpaca, ShareGPT, OpenAssistant PiKa, SPIN, DeepSeek-R1, Qwen3

这张表清晰地展示了两个时代的鸿沟。接下来的几个小节，我们将逐一深入这四个关键新范式：PiKa 的小数据奇迹、SPIN 自我博弈、自动课程生成、以及数据飞轮实战。

PiKa范式：3万条精选数据打败1000万条

2025年，一篇名为 PiKa: Unleashing the Potential of Post-Training Data 的论文在整个大模型社区引发了震动。PiKa 团队做了一个简单但极端的实验：他们精心构建了仅包含 30K 样本的多阶段后训练数据（含 SFT、偏好对比对、推理轨迹），训练出来的 8B 模型在 AlpacaEval 2.0 上达到了 42.8% 的 LC win rate——这直接超越了 Llama-3-8B-Instruct 的 34.2%，而后者可是在超过 1000 万条专有数据上训练出来的。

这个结果意味着什么？意味着如果你只关注"我要收集更多数据"，那你的方向可能从一开始就错了。PiKa 的秘诀可以归纳为以下三点：

  • 极致的数据筛选：每条数据都经过质量评分、难度分层和多样性去重。他们的筛选管线对原始数据经过了数十道过滤程序，最终保留率不到 0.5%。
  • 多阶段数据设计：SFT 数据、偏好数据、推理轨迹不是简单堆叠，而是按照训练阶段精心设计配比和顺序。他们发现，SFT + DPO + 推理增强三个阶段的数据如果混在一起训练，效果远不如分阶段、有策略地使用。
  • 合成数据的精准应用：PiKa 的大部分数据是合成的，但他们的合成不是"大模型随便生成"，而是基于难度递进和覆盖度优化的定向生成。
比 LIMA 更进一步：还记得第1章提到的 LIMA（5000条就够）吗？PiKa 在 LIMA 的基础上更进一步——它证明的不是"小数据可行"，而是"小数据可以打败大数据"。LIMA 告诉我们"不要浪费数据"，PiKa 告诉我们"每条数据都应该是特种兵"。

PiKa 给整个行业的最大启示是：后训练数据的瓶颈已经从"数据不够"变成了"数据不够好"。与其花巨资收集百万级数据集，不如投入资源构建一条高质量的小数据管线。

SPIN自我博弈

如果说 PiKa 是在"如何选数据"这个维度上做出了突破，那么 SPIN（Self-Play Fine-Tuning）则是在"数据从哪里来"这个根本问题上彻底打开了新思路。

SPIN 的核心思想非常优雅：让模型自己和自己下棋。具体来说：

第1步：用当前模型（θ_t）对训练集中的每个 prompt 生成回答
第2步：构造一个二元分类任务——
  正样本：人类标注的回答（chosen）
  负样本：模型自己生成的回答（rejected）
第3步：用这对数据做 DPO 训练，得到 θ_{t+1}
第4步：用 θ_{t+1} 重复第1步，进入下一轮自我博弈
...
直到模型无法区分自己的输出和人类输出为止

这个过程的本质是什么？是模型在一场持续的"模仿游戏"中不断提升——它先尝试模仿人类，然后对自己不够像人类的输出进行修正，接着在新的能力水平上再次尝试。每经过一轮自我博弈，模型都会变得更像"它心目中的完美回答者"。

SPIN 最革命性的点在于：它完全不需要外部数据源。不需要更强大的教师模型，不需要人工标注新数据，甚至不需要收集用户反馈。所有的训练数据都是模型自己在迭代过程中生成的。这意味着——一个拥有较好基座能力的团队，理论上可以仅凭借一个种子数据集（几千条高质量 SFT 数据），通过 SPIN 的自我博弈循环，不断自我提升。

SPIN 的后续演进：2025-2026年，SPIN 的思想被多个团队采纳并改进。例如，Qwen 团队提出了 ReST（Reinforced Self-Training），在 SPIN 的基础上加入了基于奖励信号的筛选机制——不是所有模型生成的回答都用作负样本，只有那些奖励模型打分低的才被选入。这种"奖励引导的自我博弈"进一步提升了数据质量。

自动课程生成

2025-2026 年另一个令人兴奋的方向是自动课程生成（Automatic Curriculum Generation）。如果说 SPIN 解决了"数据怎么来"的问题，那自动课程生成解决的是"应该训练什么"的问题。

传统的数据策略是静态的——数据集在训练开始前就确定好，训练过程中不会改变。但自动课程生成的做法完全不同：

1. 评估 → 用当前模型在多个能力维度上做评测，识别薄弱环节
2. 生成 → 针对薄弱环节，设计并生成针对性训练数据
3. 训练 → 用新生成的数据对模型做一轮增量训练
4. 再评估 → 重新评测，看哪些能力提升了，哪些还是短板
5. 循环 → 回到第1步，直到所有能力达标

这个循环的魔力在于：它不需要任何人工干预。模型自己识别自己哪里不够好，自己生成数据来弥补自己的不足，然后自己验证自己是否真的进步了。这是一种完全自动化的、自适应的数据策略。

在实际应用中，自动课程生成有几个关键技术要点：

  • 能力维度的定义：需要一套结构化的能力评估体系。例如，代码能力可以拆分为语法理解、算法设计、Bug修复、代码审查等子维度。
  • 针对性数据的合成策略：识别出某个维度是短板后，如何生成精确命中该短板的数据？常见做法包括模板填充、错误引导（让模型先犯错再纠正）、难度递进等。
  • 遗忘控制：专门针对短板训练可能会导致模型在其他维度上出现灾难性遗忘。需要引入回放机制或正则化策略。
需要注意的风险：自动课程生成虽然强大，但也有"训练过度"的风险。如果循环中识别短板的评测集不够全面，模型可能会在某个狭窄维度上过度优化，反而损害了泛化能力。实践中，建议在自动课程循环中引入至少3个以上的独立评测维度，并且定期做一次全面的多维评测兜底。

数据飞轮实战案例

理论说了这么多，我们来看看几个真实世界中数据飞轮的实践案例：

DeepSeek-R1：冷启动 + RL + 拒绝采样循环

DeepSeek-R1 的数据飞轮是 2025 年最具代表性的案例之一。它的流程大致如下：

第一阶段：冷启动
  少量人工标注的 Chain-of-Thought 数据 → SFT → 获得基础推理能力

第二阶段：RL 探索
  GRPO 训练 → 模型在数学/代码任务上探索多种推理路径
  保留奖励得分高的推理轨迹 → 作为下一轮 SFT 数据

第三阶段：拒绝采样（Rejection Sampling）
  对同一 prompt 采样 N 个回答
  奖励模型筛选 top-K 高质量回答 → 加入训练集

第四阶段：迭代
  SFT(新增强数据) → 再次 GRPO → 再次拒绝采样
  每轮迭代后，模型能力明显提升

DeepSeek-R1 的数据飞轮体现了几个关键设计原则：用 RL 探索多样性（GRPO 的 group sampling 天然产生多样轨迹）、用拒绝采样保证质量（只保留最好的那部分）、用 SFT 巩固能力（把探索结果蒸馏成监督信号）。

UI-TARS-2：轨迹飞轮

UI-TARS-2 是一个 GUI Agent 模型，它的数据飞轮别具一格——从模型的实际使用中收集数据。模型在真实 GUI 操作任务中不断执行，每次成功或失败的轨迹都被记录下来。成功的轨迹直接作为正样本加入 SFT 数据池；失败的轨迹则通过反思机制修正——让模型分析失败原因并生成改正后的正确轨迹，然后加入训练集。

这种飞轮的关键支撑是可验证的奖励信号——GUI 任务中"是否成功完成任务"是一个客观的二元判定。当奖励信号可以自动化时，数据飞轮的运转就不需要任何人工参与。

GLM-5 数据循环

GLM 团队在 2026 年公布的 GLM-5 训练方案中，展示了一套非常成熟的工业级数据循环系统。它的特点是多路数据源并行输入——包括在线用户反馈、离线评测结果、合成数据生成、以及定向采集的数据，四路数据经过统一的质量评分模块打分后，按质量阈值筛选入库，再通过自动配比优化器决定每种数据的采样权重。整个系统由一套数据仪表盘实时监控，数据工程师可以根据仪表盘上的指标随时调整策略。

思考/非思考统一

2025-2026 年还有一个非常有趣的新趋势——思考/非思考数据的统一训练。

传统的做法是将"需要深度推理的任务"和"不需要推理的简单任务"分开处理：推理数据用 CoT 格式训练，简单对话用标准格式训练。但 2025 年底发布的 Qwen3 和 GLM-4.5 采用了一种新方法：在同一个训练集中混合有推理轨迹和无推理轨迹的数据，让模型自己学会根据任务难度自适应地决定是否启动推理过程。

这种混合训练的数据构造方式很巧妙：

{
  "数学推理样本": {
    "prompt": "解方程: 2x + 5 = 13",
    "response": "<think>首先将5移到等式右边...</think>\n解得 x = 4"
  },
  "简单问答样本": {
    "prompt": "中国的首都是哪里？",
    "response": "北京。"
  },
  "混合格式样本": {
    "prompt": "先告诉我3的平方，再解方程 2x + 5 = 13",
    "response": "3的平方是9。<think>对于方程...</think>x = 4"
  }
}

通过在这种混合数据上训练，模型能够在不需要推理的问题上保持高效直接的回答，同时在需要深度推理的问题上自动激活思考过程。这种模式自适应能力，是 2026 年后训练数据设计的核心追求之一。

数据设计启示：如果你正在为自己的模型设计训练数据，不妨在数据集中刻意加入一些"混合格式"的样本——同一个回答中既有直接回答部分也有推理过程部分。这类样本教会模型如何在两种模式之间平滑切换，是提升用户体验的利器。

对未来后训练数据工作的启示

站在 2026 年年中回看，这一节讨论的四个新范式给我们带来了清晰的行动指南：

  1. 拥抱合成数据：人工标注不会消失，但它的角色正在从"数据主要来源"转变为"种子数据和验证数据"。大规模数据的生产方式正在被合成数据接管。
  2. 建立自动评估体系：没有评估就没有迭代。无论是自动课程生成还是数据飞轮，都依赖于快速、准确的能力评估。投资于自动化评估体系，就是投资于数据迭代的速度。
  3. 投资可验证奖励设计：DeepSeek-R1 的成功、UI-TARS-2 的飞轮、SPIN 的自我博弈——所有这些方法的核心都是一个可靠的奖励信号。如果你的任务可以设计出自动化可验证的奖励（数学、代码、GUI 操作、格式合规等），那数据飞轮就能全自动运转。
  4. 质量优于数量，设计优于收集：PiKa 已经用 3 万条数据证明了这件事。后训练数据工作者的核心竞争力，正在从"能收集多少数据"转变为"能设计出多高质量的数据"。
一句话总结：2025-2026 年后训练数据新范式的本质，是从"数据驱动的训练"进化到"数据设计的训练"。前者是有什么数据就学什么，后者是需要什么能力就设计什么数据。这需要数据团队从"数据采集者"转变为"数据架构师"。

了解了前沿趋势之后，一个很自然的问题是：面对这么多选择，我到底该怎么为自己的模型设计数据策略？不同的应用场景需要完全不同的数据组合和配比。下一章，我们将围绕这个问题展开——从场景诊断到数据集组合，从评估体系到迭代优化，建立一套完整的数据策略设计框架。让我们一起进入第27章：数据策略设计与评估体系。
