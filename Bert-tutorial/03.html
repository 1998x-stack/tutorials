第3章 BERT开山论文精读(上)

大家好，我是你们的讲师。上一章我们深入Transformer架构时，你可能已经隐隐感觉到：这么好的并行架构，不做点"大工程"实在可惜。2018年10月11日，Google在arXiv上发布了论文 BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding（编号1810.04805）。这篇论文不仅代码开源，还在11项NLP任务上刷新了SOTA——这个幅度之大，以至于很多研究者第一反应是"不可能"。今天我们就来精读这篇经典论文的上半部分，聚焦架构设计和预训练任务。

还记得我第一次读这篇论文的感受吗？最让我惊讶的是它的"简单"——核心贡献就是两句话：第一，用Masked Language Model实现双向预训练；第二，在微调时用同一个架构适配所有任务。但就是这两点组合起来，产生了1+1>2的效果。有时候，真正伟大的工作不是靠复杂的trick，而是靠戳中问题的本质。

3.1 BERT的核心架构

BERT是Transformer的Encoder部分，去掉了原本Transformer中的Decoder（区别于GPT和后续的T5）。论文提出了两个主力模型规格：

参数 BERT-Base BERT-Large
编码器层数 (L) 12 24
隐藏层维度 (H) 768 1024
注意力头数 (A) 12 16
每头维度 (H/A) 64 64
FFN中间层维度 3072 (4×768) 4096 (4×1024)
参数量 110M 340M
参数量（含嵌入层） ~110M ~340M
训练数据 BooksCorpus + Wikipedia BooksCorpus + Wikipedia
训练时间 4天 (4×TPU v3) 4天 (16×TPU v3)
Batch Size 256 256
训练步数 1,000,000 1,000,000
优化器 Adam (lr=1e-4) Adam (lr=1e-4)

规模对比： BERT-Base的110M参数与GPT的117M大致相当，但BERT使用双向注意力（每个位置能看到所有位置），而GPT使用了单向（从左到右）的自回归注意力。这看起来只是"关注方向"的区别，但在下游任务上带来了质的飞跃——BERT-Base全面超越GPT，证明了双向预训练的巨大优势。

3.2 三层嵌入的叠加

BERT的输入表示是三个嵌入的叠加：

input_embedding = token_embedding + segment_embedding + position_embedding

Token Embedding（词嵌入）： 将subword token映射到768维（Base）或1024维（Large）的向量空间。词表大小为30,000（基于WordPiece）。

Segment Embedding（段嵌入）： 区分输入中的两个句子。A句所有位置加 \(E_A\)，B句所有位置加 \(E_B\)。单句任务中全部加 \(E_A\)。

Position Embedding（位置嵌入）： 可学习的512×768（或512×1024）矩阵，为每个位置分配一个唯一的嵌入向量。

设计考量： 为什么不把三个嵌入合并成一个？分开设计的好处是各部分功能正交——词嵌入负责语义，段嵌入负责句子归属，位置嵌入负责序列顺序。这种分离让模型更容易理解不同类型的输入特征。消融实验表明，去掉任何一个嵌入层都会导致性能下降。

3.3 MLM预训练任务设计

3.3.1 为什么需要Masked Language Model？

语言模型的传统训练方式是预测下一个词（Next Token Prediction），即自回归模式。这种模式的问题很明显：模型只能从左到右看，无法看到右边的上下文。BERT想要做双向预训练，但简单地把双向语言模型拼起来会导致信息泄露（目标词自己看到自己）。MLM的解决方案是：随机遮挡输入中的一部分token，让模型根据上下文预测被遮挡的词。

3.3.2 为什么是15%？为什么80-10-10？

论文实验选择了15%的token被mask——这是一个需要权衡的结果：

  • 如果比例太低（如5%）：预训练任务太简单，模型学不到深度语义
  • 如果比例太高（如30%）：输入被过度破坏，模型无法利用足够的上下文信息

这15%的mask token又按照80-10-10规则进一步细分：

def apply_mlm_mask(tokens, mask_token_id, vocab_size, mask_prob=0.15):
    """BERT MLM mask策略：80-10-10规则"""
    labels = [-100] * len(tokens)  # -100表示不参与损失计算
    masked_indices = []

    for i in range(len(tokens)):
        if random.random() < mask_prob:
            masked_indices.append(i)
            labels[i] = tokens[i]      # 保存原始token作为预测目标
            rand = random.random()
            if rand < 0.8:             # 80%: 替换为[MASK]
                tokens[i] = mask_token_id
            elif rand < 0.9:           # 10%: 替换为随机token
                tokens[i] = random.randint(0, vocab_size - 1)
            else:                      # 10%: 保持不变
                pass                   # 保持原token不变

    return tokens, labels

# 示例
sentence = ["我", "爱", "学", "习", "自", "然", "语", "言", "处", "理"]
# 假设"学"和"语"被选中mask（15%）
# 80%概率 → ["我", "爱", [MASK], "习", "自", "然", [MASK], "言", "处", "理"]
# 10%概率 → ["我", "爱", "吃", "习", "自", "然", "数", "言", "处", "理"]（替换为随机词）
# 10%概率 → ["我", "爱", "学", "习", "自", "然", "语", "言", "处", "理"]（保持不变，但仍需预测）

核心设计理由： 如果100%的被选位置都替换为 [MASK]，那预训练阶段和微调阶段的输入分布就会完全不一致——微调时永远不会看到 [MASK] token。保留10%不变、10%替换为随机词，是为了强迫模型对所有token都产出有意义的上下文表示，而不能"偷懒"地只依赖于[MASK]信号来预测。

3.4 预训练数据

BERT的预训练数据来自两个来源，总量约3.3B个词：

数据源 规模 特点
BooksCorpus 800M 词 约11,000本未出版书籍，长文档连续文本
English Wikipedia 2,500M 词 250万篇英文文章，知识广博，文本质量高
总计 3.3B 词 约16GB 原始文本

论文明确指出：不使用CommonCrawl等网络爬取数据——这是一个有趣的选择。相比后面BERT的变体（如RoBERTa）使用160GB数据，BERT的3.3B词看起来非常"克制"。但这也反映了当时的研究重点：更大的数据不一定带来更好的效果，质量同样重要。

3.5 训练超参数与硬件

论文的训练配置在当时属于豪华配置，但在今天看来已经"入门级"：

# BERT-Base 训练配置（论文附录A.3）
Batch size: 256 sequences (每序列最多512 tokens)
Learning rate: 1e-4 (Adam优化器, β₁=0.9, β₂=0.999)
Learning rate schedule: 前10,000步线性warmup，之后线性衰减
Dropout: 0.1 (所有层)
激活函数: GELU (Gaussian Error Linear Unit)
训练步数: 1,000,000步
硬件: 4×TPU v3 (BERT-Base) / 16×TPU v3 (BERT-Large)
训练时间: ~4天

值得一提的是BERT使用了GELU激活函数而非ReLU。GELU是ReLU的平滑版本，对于负值不是简单截断为0而是保留一个小的负值梯度，这有助于训练的稳定性和表示质量。

2025年的视角看： 如今用同样的数据量和训练配置训练一个110M的BERT-Base，在8×A100上可能只需1天（得益于Flash Attention和混合精度训练）。但BERT原创工作的难度不在于计算量，而在于第一次证明了双向预训练+微调范式可以有效工作——这个洞察的价值远超TPU的账单。

3.6 输出表示与分类架构

BERT的输出与输入形状相同 \([batch, seq\_len, hidden\_size]\)，但不同位置的输出向量各司其职：

  • [CLS] 位置的向量（即 output[:, 0, :]）被设计为"句子级表示"，用于分类任务
  • 其他每个位置的向量对应输入序列中每个token的上下文表示，用于序列标注任务

微调时，只需在BERT顶部添加一个简单的线性分类层。在BERT-Large的GLUE结果中，仅凭[CLS]向量+单个线性层就能达到SOTA——这直观地证明了BERT学到的表示质量有多高。
