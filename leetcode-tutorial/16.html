从依赖关系到连通分量

各位同学好。上一章我们掌握了图的 DFS/BFS 遍历，现在我们来解决两个图论中"最有实际意义"的问题：拓扑排序和并查集。为什么说它们最有实际意义？拓扑排序解决的是"依赖关系"问题——大学课程安排、软件构建顺序、任务调度，本质上都是 DAG（有向无环图）上的拓扑排序。而并查集解决的是"连通关系"问题——社交网络中的朋友圈、计算机网络中的节点连通、地图上的区域划分。

这两个算法有一个共同点：思想极其简单，但应用极其广泛。拓扑排序的核心就两个东西——"入度"和"队列"；并查集的核心就两个操作——"find"和"union"。但正是这两个简单的算法，能派生出无数高级应用。本章我们来把它们吃透。

拓扑排序：入度表 + BFS

拓扑排序（Topological Sorting）是对 DAG 的所有顶点的一种线性排序，使得对每条有向边 u → v，u 在排序中都出现在 v 之前。用人话说就是："先修课"必须先于"后续课"出现。

Kahn 算法

Kahn 算法的思路非常直观：

  1. 统计每个节点的入度
  2. 将所有入度为 0 的节点入队
  3. 依次出队，将出队节点的所有邻居入度减 1，如果某个邻居入度变为 0，则将其入队
  4. 重复直到队列为空
def topological_sort(n, edges):
    # 建图 + 统计入度
    adj = [[] for _ in range(n)]
    indegree = [0] * n
    for u, v in edges:
        adj[u].append(v)
        indegree[v] += 1

    # 入度为 0 的节点入队
    queue = [i for i in range(n) if indegree[i] == 0]
    result = []

    while queue:
        u = queue.pop(0)
        result.append(u)
        for v in adj[u]:
            indegree[v] -= 1
            if indegree[v] == 0:
                queue.append(v)

    # 如果结果长度不等于 n，说明有环
    return result if len(result) == n else []
环检测：Kahn 算法的副产品就是环检测——如果排序结果的长度小于节点总数，说明图中存在环。这是面试中常考的知识点。

例题：课程表（LeetCode 207）

题目：给定 numCourses 门课程和课程间的先修关系 prerequisites（如 [1,0] 表示"学 1 之前必须先学 0"），判断是否可能完成所有课程。

这其实就是拓扑排序的裸题——如果 topological_sort 返回的列表长度等于 numCourses，说明可以完成，否则说明有环。直接套用上面的 Kahn 模板即可。

例题：课程表 II（LeetCode 210）

同 207，但这次不仅要判断能不能完成，还要返回一种可行的学习顺序。这就是拓扑排序返回结果本身——把 Kahn 算法的 result 列表直接返回。

def find_order(numCourses, prerequisites):
    adj = [[] for _ in range(numCourses)]
    indegree = [0] * numCourses
    for v, u in prerequisites:  # 学 v 之前要先学 u
        adj[u].append(v)
        indegree[v] += 1

    queue = [i for i in range(numCourses) if indegree[i] == 0]
    order = []

    while queue:
        u = queue.pop(0)
        order.append(u)
        for v in adj[u]:
            indegree[v] -= 1
            if indegree[v] == 0:
                queue.append(v)

    return order if len(order) == numCourses else []

例题：外星词典（LeetCode 269）

这是一道 Hard 题，也是拓扑排序的经典应用。给定一个按"外星人字典序"排序的单词列表，找出外星字母的字母顺序。

思路：比较相邻两个单词，找到第一个不同的字母，就得到一条有向边（如 "ab" < "ac" 意味着 b → c 的偏序关系）。收集所有偏序关系后，做拓扑排序。

def alien_order(words):
    # 收集所有出现的字母
    chars = set(''.join(words))
    adj = {c: [] for c in chars}
    indegree = {c: 0 for c in chars}

    # 相邻单词比较，提取偏序关系
    for i in range(len(words) - 1):
        w1, w2 = words[i], words[i+1]
        # 特殊情况：w1 是 w2 的前缀但更长 => 非法
        if len(w1) > len(w2) and w1[:len(w2)] == w2:
            return ""
        for c1, c2 in zip(w1, w2):
            if c1 != c2:
                adj[c1].append(c2)
                indegree[c2] += 1
                break

    # Kahn 算法
    queue = [c for c in chars if indegree[c] == 0]
    result = []
    while queue:
        c = queue.pop(0)
        result.append(c)
        for nb in adj[c]:
            indegree[nb] -= 1
            if indegree[nb] == 0:
                queue.append(nb)

    return ''.join(result) if len(result) == len(chars) else ""
外星词典的陷阱：当 w1 是 w2 的前缀但 w1 比 w2 长时（如 ["abc", "ab"]），排序是矛盾的，应直接返回空字符串。这个边界条件非常容易被忽略——我第一次做这题时就漏了，LeetCode 直接 WA。

并查集基础：find + union

并查集（Union-Find / Disjoint Set Union, DSU）解决的核心问题是动态连通性——支持两个操作：查询两个元素是否属于同一集合（find），以及合并两个元素所在的集合（union）。

并查集的实现极度简洁，但优化技巧却非常精妙。我们先从最基础的版本开始。

基础实现

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))

    def find(self, x):
        while self.parent[x] != x:
            x = self.parent[x]
        return x

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx != ry:
            self.parent[ry] = rx

这个版本的问题是：最坏情况下 find 的时间复杂度是 O(n)（树退化成链表）。我们有两个优化来解决这个问题。

优化一：路径压缩（Path Compression）

在 find 的过程中，把沿途所有节点的父节点直接指向根节点。这样下次查找就是 O(1)。

def find(self, x):
    if self.parent[x] != x:
        self.parent[x] = self.find(self.parent[x])  # 路径压缩
    return self.parent[x]

优化二：按秩合并（Union by Rank）

记录每棵树的"高度"（秩），总是把矮树挂到高树上，避免树退化成链表。

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return
        if self.rank[rx] < self.rank[ry]:
            self.parent[rx] = ry
        elif self.rank[rx] > self.rank[ry]:
            self.parent[ry] = rx
        else:
            self.parent[ry] = rx
            self.rank[rx] += 1
复杂度：同时使用路径压缩和按秩合并后，单次操作的均摊时间复杂度为 O(α(n))，其中 α 是阿克曼函数的反函数——在宇宙寿命内可以认为 α(n) ≤ 5。换句话说，就是近乎 O(1)。

例题：冗余连接（LeetCode 684）

题目：一个 n 个节点的树多了一条边（变成图），找出那条多余的边。典型的"检测环"问题——当并查集的 find 发现两个节点已经连通时，这条边就是多余的。

def find_redundant_connection(edges):
    n = len(edges)
    uf = UnionFind(n + 1)  # 节点编号从 1 开始

    for u, v in edges:
        if uf.find(u) == uf.find(v):
            return [u, v]  # 已经连通，这条边冗余
        uf.union(u, v)
    return []

例题：省份数量（LeetCode 547）

题目：n 个城市，isConnected[i][j] = 1 表示城市 i 和 j 直接相连。求省份（连通分量）的数量。

def find_circle_num(isConnected):
    n = len(isConnected)
    uf = UnionFind(n)

    for i in range(n):
        for j in range(i + 1, n):
            if isConnected[i][j]:
                uf.union(i, j)

    # 统计不同根节点的数量
    roots = set(uf.find(i) for i in range(n))
    return len(roots)
DFS 也能做：省份数量用 DFS 遍历连通分量也能解，但并查集写起来更干净，而且可以动态地加边——DFS 每次都要重新遍历整个图。如果题目涉及"多次连接查询"，并查集是更好的选择。

并查集进阶：带权并查集

基础并查集只能维护"连通性"——两个节点在不在一个集合里。但有些问题需要维护节点之间的相对关系，比如"A 和 B 是同一类"或者"A 和 C 不是同一类"。这时就需要带权并查集，在合并时额外维护一个"权值"。

例题：等式方程的可满足性（LeetCode 990）

题目：给定一组形如 "a==b" 或 "a!=b" 的等式，判断它们是否自洽。

思路：先用并查集处理所有 == 关系，把相等的变量合并；然后检查所有 != 关系，如果两个变量已在同一个集合中，则矛盾。

def equations_possible(equations):
    # 26 个小写字母
    uf = UnionFind(26)

    # 第一遍：处理所有相等关系
    for eq in equations:
        if eq[1] == '=':
            uf.union(ord(eq[0]) - ord('a'),
                     ord(eq[3]) - ord('a'))

    # 第二遍：检查所有不等关系
    for eq in equations:
        if eq[1] == '!':
            if uf.find(ord(eq[0]) - ord('a')) == \
               uf.find(ord(eq[3]) - ord('a')):
                return False
    return True
易错点：一定要先处理所有 ==，再检查所有 !=。如果边处理边检查，可能会漏掉一些间接矛盾。比如 ["a==b", "b==c", "a!=c"]——如果按顺序处理，处理到 "a!=c" 时 a 和 c 可能还没合并，就会错误地返回 True。

带权并查集的更高级应用还包括：食物链问题（POJ 1182）、情侣牵手（LeetCode 765）等。核心思想不变：在 union 和 find 时额外维护一个"到根节点的偏移量"，用于表示节点与根的关系。

对比总结：拓扑排序 vs 并查集

这两个算法虽然都处理"关系"，但视角完全不同：

维度 拓扑排序 并查集
处理的关系 有向依赖（u → v 表示 u 先于 v） 无向连通（u 和 v 在同一个集合）
图的方向 有向图（必须是无环的 DAG） 无向图（或有向关系的等价类）
核心操作 入度统计 + BFS find + union（路径压缩）
能检测 有向图中的环 无向图中的环
典型问题 课程安排、编译顺序、任务调度 连通分量、冗余连接、朋友圈
是否有序 输出是线性顺序 输出是集合划分

踩坑与总结

坑1：并查集未路径压缩导致超时

我曾经在一次模拟面试中写并查集，偷懒没写路径压缩，结果被面试官追问"你的 find 最坏情况是什么？"我回答 O(n)，他接着问"如果连续调用 n 次呢？"我沉默了。O(n²) 的复杂度在大数据量下必超时。所以请记住：并查集不加路径压缩，等于没写。代码量只多两行，收益是巨大的。

坑2：拓扑排序的环检测

Kahn 算法结束后，如果 len(result) != n，就说明有环。但很多人会忘记做这个检查，直接返回 result，导致在无解的问题上返回了一个不完整的排序。面试时这属于"功能性错误"，扣分很严重。

坑3：并查集 parent 数组初始化大小

LeetCode 684（冗余连接）的节点编号从 1 到 n，所以 UnionFind 的大小应该是 n + 1。如果你设成 n，所有 1-based 的节点访问都会越界。这类 off-by-one 错误在并查集题目中非常常见。

面试重点：拓扑排序和并查集是图论面试的"必考点"。我建议你把本章的模板代码背下来——不是死记硬背，而是理解后能3 分钟内默写出来。课程表（207）和冗余连接（684）是这两类问题的"最小原型"，搞定它们就等于拿到了钥匙。
