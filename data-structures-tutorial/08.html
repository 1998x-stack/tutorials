从一道"简单"题说起

假设有一天你在刷LeetCode，看到一道标记为"简单"的题——「二叉树的最大深度」。你心想：简单题，一分钟搞定。于是你写出了这样的代码：

def max_depth(root):
    if not root:
        return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))

你点了提交，通过了。用时击败了99%的用户。你满意地关闭了页面。

但你有没有停下来想过——这段只有三行的代码，背后藏着什么？为什么它能计算出深度？为什么递归在这里完美适用？如果我不用递归，用循环怎么写？如果树不是二叉树，是多叉树呢？如果树上有环呢？

这些问题，如果你都能从容回答，那你已经掌握了树的核心思维。如果不能——别担心，这节课就是为你准备的。

树（Tree）是计算机科学中最重要的非线性数据结构之一。从文件系统的目录结构，到浏览器的DOM树，从数据库的B+树索引，到编译器解析源码的AST（抽象语法树）——树无处不在。掌握树，你的"数据结构工具箱"才算真正升级到了 2.0 版本。

生活中的树结构

在正式学习术语之前，我们先从生活中的例子建立直觉。你其实每天都在和树打交道——只是你可能没有意识到。

文件夹目录

你的电脑文件系统是一个典型的树形结构。打开任意一个文件夹：

  • 根目录（比如 C:\ 或 /）是树的根节点。
  • 子文件夹是子节点。
  • 文件是叶节点（没有子节点）。
  • 一个文件夹可以有任意多个子文件夹和文件——这就是"多叉树"。

公司组织架构

CEO 下面是各个 VP，VP 下面是总监，总监下面是经理，经理下面是工程师。每个员工只有一个直接上级（父节点唯一），但可以有多个直接下级（子节点不唯一）。这就是一棵严格树——没有回路，没有交叉汇报（理想情况下）。

HTML 的 DOM 树

<html>
  <head>
    <title>页面标题</title>
  </head>
  <body>
    <div>
      <p>一段文字</p>
      <img src="photo.jpg">
    </div>
  </body>
</html>

HTML 标签嵌套的层级结构本身就是一棵树。CSS 选择器 `div p` 的含义就是"选择所有 div 下层的 p 元素"——这正是树的后代遍历。

你能从这三个例子中找到共同点吗？对了——层次关系。树就是用来表示"一对多"层次关系的最佳结构。

树的定义与术语

形式化定义

一棵树（Tree）是由 n（n ≥ 0）个节点组成的有限集合。当 n = 0 时称为空树。当 n > 0 时：

  • 有且只有一个称为根（Root）的节点。
  • 其余节点可以划分为 m（m ≥ 0）个互不相交的有限集合，每个集合本身就是一棵树，称为根的子树（Subtree）。

注意定义中出现了"树"本身——没错，树的定义是递归的。树由子树构成，子树又由更小的子树构成。这是理解树的关键钥匙。

核心术语一览

术语 英文 含义
根节点 Root 没有父节点的节点，一棵树只有一个根
父节点 Parent 直接指向子节点的节点
子节点 Child 被某个节点直接指向的节点
叶节点 Leaf 没有子节点的节点
兄弟节点 Sibling 拥有相同父节点的节点
祖先 Ancestor 从根到某节点路径上的所有节点（不含自身）
后代 Descendant 某节点子树中的所有节点（不含自身）
深度 Depth 从根到该节点的边数（根的深度为0）
高度 Height 该节点到最远叶节点的边数（叶节点高度为0）
度 Degree 一个节点的子节点个数
深度 vs 高度：很多初学者会混淆这两个概念。一个简单的记忆方法——深度从根往下数（根是0），高度从叶子往上数（叶子是0）。一棵树的高度等于根节点的高度，也等于最深叶节点的深度。

常见的二叉树类型

  • 二叉树（Binary Tree）：每个节点最多有两个子节点，分别称为左孩子和右孩子。
  • 满二叉树（Full Binary Tree）：除了叶节点外，每个节点都有两个子节点。也就是说，所有节点的度要么是0要么是2。
  • 完全二叉树（Complete Binary Tree）：除了最后一层，上面所有层都是满的；最后一层的节点尽可能靠左排列。
  • 完美二叉树（Perfect Binary Tree）：所有内部节点都有两个子节点，且所有叶节点在同一层。这是一种既满又完全的二叉树。

完全二叉树特别重要——下一章的堆就是用完全二叉树来实现的。记住它的特征：节点"从左到右"排列，中间不能有空缺。

二叉树的存储

在代码中，我们有两种方式来存储一棵二叉树：链式存储和顺序存储。

链式存储（最常用）

每个节点是一个对象，包含数据域和左/右指针。这是最直观的方式，像双向链表的"升级版"——一个节点有两个"next"指针。

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left    # 左子节点
        self.right = right  # 右子节点

使用链式存储，构建树就是拼接对象的过程：

# 构建一棵简单的二叉树：
#       1
#      / \
#     2   3
#    / \
#   4   5

root = TreeNode(1)
root.left = TreeNode(2)
root.right = TreeNode(3)
root.left.left = TreeNode(4)
root.left.right = TreeNode(5)

顺序存储（数组实现）

对于完全二叉树，我们可以用数组来节省空间。将节点按层序编号，存储在数组中，利用下标关系定位父子节点：

  • 根节点存储在 index = 1（为了方便计算，通常从1开始）
  • 对于 index = i 的节点：左孩子 index = 2i，右孩子 index = 2i + 1
  • 对于 index = i 的节点（i > 1）：父节点 index = i // 2
# 数组表示的完全二叉树
#       1
#      / \
#     2   3
#    / \  /
#   4   5 6
#
# 数组: [0, 1, 2, 3, 4, 5, 6]
# index: 0  1  2  3  4  5  6
# 从1开始，方便下标计算

arr = [0, 1, 2, 3, 4, 5, 6]

def left_child(i):
    return 2 * i

def right_child(i):
    return 2 * i + 1

def parent(i):
    return i // 2

# arr[left_child(1)] = arr[2] = 2  √
# arr[right_child(1)] = arr[3] = 3 √
特性 链式存储 顺序存储（数组）
适用树类型 任意二叉树 完全二叉树（或满二叉树）
空间效率 需要额外左右指针 无需指针，但可能有空洞
随机访问子节点 O(1)（通过指针） O(1)（通过下标计算）
插入/删除 O(1)（改指针） 可能需要移动大量元素
理解难度 直观 需要下标换算

树的遍历（上）——递归三种遍历

树的遍历，就是用某种顺序"访问"树中的每一个节点。最常见的三种方式是前序、中序、后序遍历。这里的"序"指的是根节点的访问时机。

想象你面前有一棵二叉树，你从根节点出发：

  • 前序遍历（Pre-order）：先访问根，再遍历左子树，最后遍历右子树。→ 根→左→右
  • 中序遍历（In-order）：先遍历左子树，再访问根，最后遍历右子树。→ 左→根→右
  • 后序遍历（Post-order）：先遍历左子树，再遍历右子树，最后访问根。→ 左→右→根

我们可以用"吃水果"来类比。假设你面前有一盘水果：根是一个苹果，左子树是香蕉，右子树是樱桃。而"访问"就是"吃"：

  • 前序："我先吃苹果，再吃左盘香蕉，再吃右盘樱桃。"——先吃根。
  • 中序："我先吃左盘香蕉，再吃苹果，再吃右盘樱桃。"——中间吃根。
  • 后序："我先吃左盘香蕉，再吃右盘樱桃，最后吃苹果。"——最后吃根。

递归实现非常简洁：

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def preorder(root):
    """前序遍历：根 → 左 → 右"""
    if not root:
        return []
    return [root.val] + preorder(root.left) + preorder(root.right)

def inorder(root):
    """中序遍历：左 → 根 → 右"""
    if not root:
        return []
    return inorder(root.left) + [root.val] + inorder(root.right)

def postorder(root):
    """后序遍历：左 → 右 → 根"""
    if not root:
        return []
    return postorder(root.left) + postorder(root.right) + [root.val]

三个函数的高度相似性，恰好验证了它们之间的本质关系——只是访问根节点的时机不同。

如果你对递归还不熟悉：可以先浏览第15章的基础部分再回来看树的遍历。不过坦白说，二叉树的递归遍历是理解递归最好的入门练习——"向左走，走到不能再左，然后回来，向右走一步，继续向左……"这不就是递归吗？

树的遍历（下）——迭代遍历与层序遍历

用栈实现迭代前序遍历

递归虽然简洁，但过深的递归会导致栈溢出。在生产环境中，我们经常使用迭代（显式栈）代替递归：

def preorder_iterative(root):
    """迭代前序遍历：显式栈模拟递归"""
    if not root:
        return []
    result = []
    stack = [root]
    while stack:
        node = stack.pop()
        result.append(node.val)
        # 先右后左，因为栈是后进先出
        if node.right:
            stack.append(node.right)
        if node.left:
            stack.append(node.left)
    return result

def inorder_iterative(root):
    """迭代中序遍历"""
    result = []
    stack = []
    curr = root
    while curr or stack:
        # 一路向左走到底
        while curr:
            stack.append(curr)
            curr = curr.left
        # 弹出，访问，转向右子树
        curr = stack.pop()
        result.append(curr.val)
        curr = curr.right
    return result

def postorder_iterative(root):
    """迭代后序遍历（双栈法或单栈+标记法）"""
    # 法一：前序的变体（根右左 → 反转 → 左右根）
    if not root:
        return []
    stack = [root]
    result = []
    while stack:
        node = stack.pop()
        result.append(node.val)
        if node.left:
            stack.append(node.left)
        if node.right:
            stack.append(node.right)
    return result[::-1]  # 反转即为后序

层序遍历（Level-order / BFS）

层序遍历是"逐层"访问，从根开始，从左到右访问每一层的所有节点。它天然地用到了队列（Queue）：

from collections import deque

def level_order(root):
    """层序遍历：队列实现 BFS"""
    if not root:
        return []
    result = []
    queue = deque([root])
    while queue:
        level_size = len(queue)
        level = []
        for _ in range(level_size):
            node = queue.popleft()
            level.append(node.val)
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        result.append(level)  # 按层分组
    return result

# 输出示例（对前面的树）：
# [[1], [2, 3], [4, 5]]

四种遍历对比

遍历方式 顺序 数据结构 应用场景
前序遍历 (Pre-order) 根 → 左 → 右 栈（显式或隐式递归栈） 树的结构复制、表达式求值的前缀表示、序列化/反序列化
中序遍历 (In-order) 左 → 根 → 右 栈 二叉搜索树的升序输出、BST 验证（下一章核心考点）
后序遍历 (Post-order) 左 → 右 → 根 栈 删除树、计算目录大小（先算子目录）、表达式求值的后缀表示
层序遍历 (Level-order) 从上到下，从左到右 队列 最短路径（无权图）、打印二叉树、找最底层最左节点
关键洞察：遍历顺序的选择不是随意的，它取决于你在什么时候需要用到根节点的信息。如果你需要先知道父节点才能处理子节点（比如复制树结构），用前序。如果你需要先处理完所有子节点才能处理父节点（比如计算树的高度、删除节点），用后序。如果你需要按顺序输出排序结果（BST），用中序。

一个真实的"踩坑"故事

有一次我在处理一个深度超过2000层的JSON数据（某个多年积累的配置文件的解析结构），用递归进行遍历。Python 解释器直接报了个 RecursionError: maximum recursion depth exceeded。我当时的表情是这样的：😅

Python 默认的递归深度限制是 1000（可以通过 sys.setrecursionlimit 调整），但即使你提高了限制，递归超过一定深度仍然是危险的——C 语言的栈空间是有限的，操作系统给每个线程的栈通常只有 8MB，递归太深会导致栈溢出（Segmentation Fault）。从那以后，我在做深度遍历时都会先评估树的深度，深度超过 500 就改用迭代版本。

递归的代价：每次递归调用都会在调用栈上分配空间。对于深度为 h 的树，递归前序遍历的空间复杂度是 O(h)。如果树退化成链表（h = n），空间复杂度就是 O(n)。迭代遍历可以将空间优化到 O(1)（Morris 遍历，第9章会介绍）。
