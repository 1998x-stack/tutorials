📡 第6章 信道编码与信息论基础

大家好，我是你们的讲师。想象一下：你收到了一个包含1000个0/1比特的数据包，但传输过程中受到了干扰——有5个比特被"翻转"了。如果这5个比特恰好是关键的控制信令，你的手机可能就会掉线。如何保证接收端能检测到甚至自动纠正这些错误？

这就是信道编码要做的事。信道编码通过有规律地在原始数据中添加冗余信息，让接收端能够检测和纠正传输中发生的错误。它是通信系统中保障可靠性的最后一道防线。而要理解信道编码为什么能做到这一点，我们需要先了解信息论——这门由香农在1948年创立的学科，定义了通信的终极边界。

6.1 信息论基础：熵、互信息与信道容量

香农在1948年发表的那篇《通信的数学理论》中，提出了几个革命性的概念：

  • 信息熵（Entropy）：衡量一个随机变量"不确定性"的大小。如果一个事件发生的概率是p，它所携带的信息量就是-log₂(p)比特。一个公平硬币投掷的结果（正/反各50%），熵是1比特——因为它确实告诉我们1比特的新信息。但一个"作弊硬币"（99%概率正面），熵只有0.08比特——因为结果几乎可以猜到，提供的信息很少。
  • 互信息（Mutual Information）：衡量一个随机变量包含的关于另一个随机变量的信息量。在通信系统中，就是"接收信号"包含了多少关于"发送信号"的信息——这直接决定了通信能有多可靠。
  • 信道容量：互信息的最大值。香农证明了：只要传输速率低于信道容量，就存在一种编码方案可以实现任意小的误码率——这就是香农第二定理，也被称为"信道编码定理"。

核心思想：信息论的一个核心洞见是——"信息"是可以被量化的。我们通常认为"信息"是抽象的、模糊的，但香农告诉我们："信息就是对不确定性的消除"。你给我发了一条微信，接收前后我的"不确定状态"发生了改变，改变量就是这条微信的信息量。这个量化观点是所有通信理论的基础。

6.2 差错控制：ARQ vs FEC

在实际系统中，差错控制主要有两种策略：

  • ARQ（自动重传请求）：接收端检测到错误后，要求发送端重新发送。优点是实现简单、只在有错时才消耗额外资源。缺点是增加时延——不适合实时业务（如语音通话、视频会议）。
  • FEC（前向纠错）：发送端发送的数据包含足够的冗余信息，接收端可以直接纠正一定数量的错误，无需重传。优点是低时延（一次性传输完成），缺点是编码效率低（总有冗余开销）。

5G系统中采用混合ARQ（HARQ，Hybrid ARQ）策略——结合了FEC和ARQ的优点：第一次传输使用FEC，如果纠错失败则触发重传，重传时发送增量冗余信息（而非整包重传）。HARQ是5G NR实现低时延高可靠（uRLLC）的关键技术之一。

6.3 信道编码的演进：从汉明码到Polar码

信道编码是通信理论中"最接近香农极限"的领域之一。下面这张表记录了编码技术的演进历程：

编码方案 提出年代 逼近香农极限 特点 典型应用
汉明码 1950 远 最简单的纠错码，只能纠正1比特错误 内存ECC、早期通信
卷积码 1955 较远 Viterbi译码，适合约束长度小的场景 GSM、2G/3G语音
Turbo码 1993 接近（约0.7dB） 并行级联卷积码、迭代译码 3G/4G数据信道
LDPC码 1960提出，1996复兴 极接近（约0.1dB） 稀疏校验矩阵、BP译码、可并行 5G NR数据信道、WiFi 6/7
Polar码 2009 可达到（理论上） 信道极化、低复杂度编码、SC/SCL译码 5G NR控制信道

从表中可以看到一个清晰的趋势：编码方案一代比一代更接近香农极限。Turbo码在3G/4G时代"统治"了20年，LDPC码在5G数据信道中接过了接力棒，而Polar码以"首个理论上可达到香农极限"的编码方案成为了5G控制信道的标准。

6.4 LDPC码：迭代的力量

LDPC（低密度奇偶校验码）的核心是一个稀疏校验矩阵——矩阵中"1"的个数远少于"0"的个数。每个码字必须满足该矩阵定义的线性约束。

下面是一个简化的LDPC编码和迭代译码示例：

LDPC码简化示例（4比特信息 + 4比特校验 = 8比特码字）：

校验矩阵 H (4×8 稀疏矩阵，每行仅3个1)：
  H = [1 1 0 1 1 0 0 0   ← 校验方程1: v1+v2+v4+v5=0 (模2)
       0 1 1 0 0 1 1 0   ← 校验方程2: v2+v3+v6+v7=0
       1 0 1 0 0 0 1 1   ← 校验方程3: v1+v3+v7+v8=0
       0 0 0 1 1 1 0 1]  ← 校验方程4: v4+v5+v6+v8=0

编码：给定4个信息比特 s=[1 0 1 1]，求解8个码字比特v
      使得 H·v^T = 0（模2运算）
      得到码字 v = [1 0 1 1 | 0 1 0 1]（前4位=信息位，后4位=校验位）

接收端收到含噪序列 r（假设第3位发生了翻转）：
  r = [1 0 0 1 0 1 0 1]  ← 第3位从1翻转为0

迭代译码（置信传播BP算法，概念流程）：
  第1轮：各校验节点收集变量节点的信息
          → 发现校验方程1、方程3不满足
  第2轮：不满足的方程向关联变量节点发送"纠错消息"
          → 变量节点3收到来自方程1和方程3的"不一致"指示
  第3轮：变量节点3结合多个校验方程的反馈
          → 判断自身最可能是1而非0，翻转
  第4轮：再次检查所有校验方程
          → 全部满足，译码成功！
          输出：v_hat = [1 0 1 1 0 1 0 1] ✓

注：BP译码是多轮迭代的"消息传递"过程，
每轮在变量节点和校验节点之间交换"软信息"（概率值）。
随着迭代进行，判决置信度逐渐提高。

LDPC名字的由来："低密度"指的是校验矩阵中"1"的密度很低。为什么需要低密度？因为稀疏矩阵使得"变量节点-校验节点"之间的连接图（Tanner图）中短环较少，BP迭代译码的性能更好。如果矩阵是稠密的，BP译码会因为信息"打转"而无法收敛——这是LDPC码设计中的核心考虑因素之一。

6.5 Polar码：信道极化的魔力

Polar码是土耳其裔教授Erdal Arikan在2009年提出的。它的理论基础是信道极化（Channel Polarization）——一个非常优美但反直觉的数学现象。

信道极化的直观理解是：假设你有N个相同的独立信道，每个信道的容量都是C。你通过一种巧妙的"合并-分裂"操作，让这些信道中的一部分变得"很好"（容量趋近于1），另一部分变得"很差"（容量趋近于0）。而你只需要在"好信道"上传输信息，在"差信道"上发送固定（收发双方都知道的）比特——这样整体传输速率就接近了总容量。

信道极化示例（N=2的最简情况）：

原始：两个独立的BEC信道（二元删除信道），删除概率ε=0.5
      每个信道的容量 C = 1-ε = 0.5

极化变换：
    输入(u1, u2) → 编码 → 发送(x1=u1⊕u2, x2=u2) → 两个BEC信道
    接收(y1, y2) → 译码

结果：
  - 信道W^-（对应u1）：容量 = 0.25（变差了）
  - 信道W^+（对应u2）：容量 = 0.75（变好了）

当N→∞时（实际Polar码N=2^n，如N=1024）：
  一半信道容量趋近1（好信道），一半趋近0（差信道）
  总容量保持 N·C 不变

5G NR中Polar码参数：N最大支持1024，母码长度2^n
  信息比特放在信道容量高的位置（冻结比特放在低容量位置）

为什么5G控制信道选择Polar码？原因有三：

  • 短码性能优：Polar码在码长较短时（如几十到几百比特）性能优于LDPC码，而控制信令的长度正好是几十到一百比特左右
  • 复杂度可调：SCL（连续消除列表）译码可以通过调整列表大小L来折中性能和复杂度
  • 确定性结构：编码矩阵是确定的（G = F⊗ⁿ），实现简单，没有LDPC的"设计搜索"问题

5G NR的编码分工：5G NR采用"双轨制"——数据信道用LDPC（高性能大块数据传输），控制信道用Polar码（短控制信令高可靠）。这种分工不是技术上的"谁优谁劣"，而是两种码各有最适合的场景。LDPC在大块数据时逼近香农极限且译码可高度并行，Polar在短块时具有理论可达香农极限的优势——互补使用是3GPP标准讨论后业界共识的产物。

6.6 6G编码展望

面向6G时代（2030年商用目标），信道编码面临新的挑战：

  • 太赫兹通信：频率极高（100 GHz以上），信道极不稳定，需要编码方案支持极低时延（<0.1ms）和极高可靠性
  • 海量连接：每平方公里百万级连接，需要超短包（几十比特）的高效编码
  • AI原生编码：基于深度学习的神经网络译码器正在研究当中，有望超越传统迭代译码的性能

几种候选方向包括：增强型LDPC（支持更灵活码率、更低时延）、Polar码的进一步推广（如PAC码）、基于Transformer的神经译码器等。截至2026年，3GPP Release 20已将"AI/ML for信道编码"列为重点研究方向。

6.7 踩坑故事：3GPP标准会上华为力推Polar码

2016年11月，在3GPP RAN1第87次会议（美国雷诺）上，发生了一件在中国通信史上具有标志性意义的事件——Polar码被采纳为5G NR控制信道的编码标准。

这场标准会议的博弈极其激烈。当时LDPC阵营（以高通、三星、爱立信为代表）和Polar阵营（以华为、中国移动为代表）展开了数轮技术角逐。LDPC在数据信道上的优势没有争议——最终大家都同意数据信道用LDPC。但在控制信道（短码场景）上，争论持续了数月之久。

华为提出了大量仿真结果，证明Polar码在短码场景下比LDPC有0.5-1dB的性能增益，且译码复杂度更低。反对意见认为Polar码的专利布局过度集中在华为手中（Arikan的专利已被华为收购），存在"标准必要专利风险"。最终3GPP采取了"双轨制"方案——数据信道LDPC、控制信道Polar——这被普遍视为中国在移动通信核心标准领域的一次重大突破。

从产业格局的角度看，这个决策的影响远超技术本身：它意味着中国企业在3GPP标准制定中从"跟随者"变成了"参与者甚至主导者"。通信标准只有0和1，没有中间态——要么你的技术方案被写进标准，要么你只能交专利费。

好了，这一章我们从信息论的基础概念出发，沿着编码技术演进的时间线走到了5G NR的LDPC+Polar双轨制方案。信道编码是通信系统中"精确与优雅"的代表——用数学的力量对抗物理世界的不确定性。下一章我们将从抽象的理论回到具体的系统架构，看看移动通信网络从2G到5G的物理架构是如何一步步演进的——蜂窝网络架构演进。
