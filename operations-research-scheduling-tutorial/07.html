第07章 单机调度与基本规则

大家好，我是你们的讲师。欢迎来到课程真正的"调度核心"部分——前面六章我们搭建了数学和计算复杂性的理论框架，从本章开始，我们要正式进入调度问题的求解方法。

单机调度（Single Machine Scheduling）是所有调度问题中最简单的形式：只有一个加工资源（一台机器），n 个工件在队列中等待加工。你可能会觉得："单机？不就排个序吗，有什么学问？"

但事实是，单机调度是最被低估的调度问题。首先，很多真实的调度场景本质上就是单机——比如医院的 MRI 检查室、机场的唯一跑道、数据中心里的单一数据库写入节点。其次，单机调度的解法和理论是理解复杂调度（流水车间、作业车间）的基石——多机调度中强大的"分解策略"，就是把多机问题分解成一系列单机子问题。所以，弄懂单机调度，你就走完了调度理论的一半路程。

单机调度的完整分类 — 24 种目标函数的复杂度全貌

单机调度虽然只有一台机器，但目标函数的不同组合产生了截然不同的求解难度。以下是经典分类，按复杂度从低到高排列：

问题 目标函数 最优规则 复杂度
1||ΣCⱼ 总完工时间最小 SPT（最短加工时间优先） O(n log n)
1||ΣwⱼCⱼ 加权总完工时间 WSPT（Smith 规则） O(n log n)
1||Lmax 最大延迟最小 EDD（最早交货期优先） O(n log n)
1|prec|fmax 正规目标通用 Lawler 算法 O(n²)
1|rⱼ|Lmax 带释放时间的最大延迟 Horn 算法 O(n²)
1||ΣUⱼ 延迟工件数最少 Moore-Hodgson 算法 O(n log n)
1||ΣTⱼ 总延迟最小 — NP-Hard
1|rⱼ|ΣCⱼ 带释放时间总完工时间 SRPT（最短剩余时间优先） O(n log n)

注意看：同样是单机，目标从 ΣCⱼ 换成 ΣTⱼ，复杂度就从 O(n log n) 跳到了 NP-Hard。这再次印证了上一章的观点：调度问题的"难"往往来自目标函数。

经典派工规则

单机调度最核心的成果就是几个简洁而优美的"派工规则"。每一个规则都对应一个明确的调度问题，并且有严格的数学证明保证其最优性。

SPT 规则 — 最小化总完工时间 ΣCⱼ

SPT（Shortest Processing Time）：按加工时间从小到大的顺序加工工件。

直觉解释：让短工件先做，可以减少后续工件的等待时间。数学证明使用交换论证。

SPT 最优性的交换证明框架：

目标：最小化 ΣCⱼ（总完工时间）
规则：按 p₁ ≤ p₂ ≤ ... ≤ pₙ 排序

证明思路（交换论证）：
  假设存在一个最优调度，其中某个相邻对 (i, j) 不满足 SPT 顺序：
  即 pᵢ > pⱼ，但 i 排在 j 之前。

  计算交换这对工件对总完工时间的影响：
  当前顺序：... → i → j → ...
  当前部分：Cᵢ = t + pᵢ, Cⱼ = t + pᵢ + pⱼ
  贡献 = Cᵢ + Cⱼ = 2t + 2pᵢ + pⱼ

  交换后：... → j → i → ...
  交换部分：Cⱼ = t + pⱼ, Cᵢ = t + pⱼ + pᵢ
  贡献 = Cⱼ + Cᵢ = 2t + 2pⱼ + pᵢ

  差值 = (2t + 2pᵢ + pⱼ) - (2t + 2pⱼ + pᵢ) = pᵢ - pⱼ > 0

  交换使总完工时间严格减小！与"最优"假设矛盾。
  所以最优调度中不会存在逆序的相邻对 → 整个序列必为 SPT 顺序。
  证毕。

这个交换论证是调度理论中最重要的证明工具之一。它简洁、优雅、无懈可击——你会看到它在很多其他规则中反复出现。

WSPT 规则 — 最小化加权总完工时间 ΣwⱼCⱼ

WSPT（Weighted Shortest Processing Time），也叫 Smith 规则：按 pⱼ / wⱼ 的比值升序排列——即"单位权重的加工时间"越短的工件越优先。

证明思路和 SPT 一样使用交换论证，差别在于比较的指标变为 pᵢ/wᵢ 和 pⱼ/wⱼ。

直觉：一个工件如果加工时间短且权重大（重要），当然应该排在前面。pⱼ/wⱼ 正好量化了这种折中。

EDD 规则 — 最小化最大延迟 Lmax

EDD（Earliest Due Date）：按交货期 dⱼ 从小到大的顺序加工工件。

这个规则几乎不需要复杂证明——直觉上，为了不让任何工件延迟太厉害，你应该优先处理紧急的（交货期最早的）工件。如果有一个最优调度中出现了 dᵢ > dⱼ 但 i 排在 j 前，交换它们不会让最大延迟变大。

EDD 看起来很直观，但注意它只对 Lmax 目标最优——如果想最小化延迟工件数 ΣUⱼ，EDD 就不够了，需要更复杂的 Moore-Hodgson 算法。

Lawler 算法 — 1|prec|fmax 的通用框架

Lawler 算法是单机调度中一个极为优雅的通用方法，适用于带先后约束（precedence constraints）且目标函数为正规目标 fmax（即最小化各工件代价的最大值）的调度问题。

算法流程：

Lawler 算法（1|prec|fmax）：

输入：n 个工件，加工时间 pⱼ，严格递增的费用函数 fⱼ(Cⱼ)，
      先后约束构成有向无环图

输出：最优排序

算法步骤：
  1. 初始化已调度集合 S = ∅
  2. 从尾部向头部构建调度序列（反向贪心）：
     while S ≠ all jobs:
       找出所有"后继全部已在 S 中"的工件集合 V
       从 V 中选择使 fⱼ(T) 最小的工件 j（其中 T = 剩余总加工时间）
       将 j 放在当前序列的最前位置
       将 j 加入 S，剩余总时间 T = T - pⱼ

核心思想：最后一个加工的工件没有后继约束，
所以在所有可能的"最后一个工件"中选择代价最小的。
然后递归处理前 n-1 个工件。

Lawler 算法的精妙之处在于"从后往前"的逆向思维——先确定最后做谁，再逐步前推。这种逆向 DP 的思路在调度中反复出现。

Moore-Hodgson 算法 — 最小化延迟工件数 ΣUⱼ

目标是最小化 ΣUⱼ——即延迟的工件数量。这在现实中非常有意义：比如客服工单处理，你希望尽可能多的工单在 SLA 时间内完成。Moore-Hodgson 算法给出了一个简洁的最优解：

Moore-Hodgson 算法：

输入：n 个工件，加工时间 pⱼ，交货期 dⱼ
输出：最小化延迟工件数的调度方案

算法步骤：
  1. 将所有工件按 EDD 排序（dⱼ 升序）
  2. 初始化序列 π = EDD 排序，延迟集合 L = ∅
  3. 遍历 π 中的工件 j：
     将 j 加入调度序列
     计算当前完工时间 C
     if C > dⱼ（工件 j 延迟了）：
       从当前已调度且尚未标记延迟的工件中，
       选择加工时间 p 最长的那个工件 k，
       将 k 从调度序列中移除，放入延迟集合 L
  4. 最终调度 = (调度序列) → (延迟集合中的工件任意顺序)

示例：
  工件:  1    2    3    4    5
  pⱼ:   7    3    4    8    2
  dⱼ:   9    12    8    17   15

  EDD 排序: 3(p=4,d=8), 1(p=7,d=9), 2(p=3,d=12), 5(p=2,d=15), 4(p=8,d=17)

  遍历：
  加 3: C=4, d=8 → 准时 ✓
  加 1: C=11, d=9 → 延迟！移除最长 p=7 的工件 1，L={1}
  加 2: C=4+3=7, d=12 → 准时 ✓
  加 5: C=7+2=9, d=15 → 准时 ✓
  加 4: C=9+8=17, d=17 → 准时 ✓（边界情况算准时）

  结果：调度顺序 3→2→5→4，延迟工件集 L={1}
  最小延迟工件数 = 1

这个算法的直观理解是：我们尽可能让更多工件准时。如果当前工件延迟了，我们就把已调度工件中"最占时间"的那个踢出去——因为移除它，能为后面的工件争取最多的缓冲时间。

带释放时间和截止时间的单机调度

现实中的工件不是随时可以加工的——原材料可能还没到、前道工序可能还没完成。这个"最早可以开始加工的时间"就是释放时间（Release Time, rⱼ）。

引入释放时间后，问题变得更棘手。即使是 1|rⱼ|Lmax 也需要 Horn 算法（基于 EDD 的启发式加上预占调度）。更复杂的 1|rⱼ|ΣCⱼ 可以用 SRPT（Shortest Remaining Processing Time）规则——这是一个可中断（preemptive）规则，允许加工被打断，优先处理剩余时间最短的工件。

对于同时有释放时间和截止时间的单机调度（1|rⱼ,dⱼ|—），这是一个可行性问题——判断是否存在一个调度使得所有工件在 dⱼ 之前完成。这可以用 Jackson 规则：按 EDD 排序，每次取释放时间已到的工件中交货期最早的加工。

实战踩坑：用 SPT 处理带权重的任务，损失惨重

我第一份工作中负责一个半导体封装车间的排程系统。车间主管告诉我们"先做快的，再做慢的"——听起来就是 SPT 嘛。我做了个 SPT 调度上线，结果一周之后，生产经理怒气冲冲地找上门来。原来，有些大客户的高价值订单（权重高）总是被排在后面，虽然总完工时间看起来不错，但客户满意度直线下降。后来改成了 WSPT，按 pⱼ/wⱼ 来排，重要客户的订单终于被合理提前了。这个教训让我终生难忘：选错目标函数，解出来的"最优"在业务眼里就是"胡来"。排程系统上线前，一定要和业务方确认清楚——你到底要优化什么指标？

派工规则速查表

规则 排序依据 最优目标 何时使用
SPT pⱼ ↑ ΣCⱼ 追求快速周转（在制品最少）
WSPT pⱼ/wⱼ ↑ ΣwⱼCⱼ 重要客户优先 + 快速周转
EDD dⱼ ↑ Lmax 按时交付最重要
Moore-Hodgson EDD + 选择性移除 ΣUⱼ 最大化准时工件数
Lawler 逆向贪心 1\|prec\|fmax 有工序依赖且目标正规
SRPT 剩余时间 ↑ (可抢占) 1|rⱼ,pmtn|ΣCⱼ 带释放时间且允许中断

单机调度看似简单，但它是调度理论的"基本粒子"——所有复杂调度问题最终都可以分解为单机子问题来求解。掌握了 SPT、WSPT、EDD 和背后的交换论证，你就掌握了解读更复杂调度问题的语法。
