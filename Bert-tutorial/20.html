第20章 中文BERT全景

大家好，我是你们的讲师。前面十几章我们一直在讲英文世界的 BERT 生态，但你有没有想过——中文 NLP 其实走出了一条和英文截然不同的道路？当英文社区还在讨论是否应该用更大的词表、更多的数据时，中文 NLP 的研究者们面临的困境完全不同：中文没有天然的空格分隔，一个汉字可能对应多个词，北京这个词拆成"北"和"京"就完全失去了语义。

这一章我们将盘点中文 BERT 生态中最重要的模型：百度 ERNIE 系列（1.0/2.0/3.0）、哈工大的 Whole Word Masking（WWM）、复旦的 ChineseBERT、哈工大讯飞的 MacBERT、华为的 NEZHA 等。最后会介绍中文评测基准 CLUE/FewCLUE/CMRC，帮你选型时心中有数。

📌 本章重点：
  • ERNIE 1.0/2.0/3.0 的核心创新与演进路线
  • 全词掩码（WWM）为何在中文中如此重要
  • ChineseBERT 的字形和拼音特征融合方法
  • MacBERT 的纠错式 MLM 替代方案
  • CLUE 中文评测体系概览

20.1 ERNIE 1.0：实体级知识增强

2019 年 4 月，百度在 ACL 2019 上发布了 ERNIE 1.0（Enhanced Representation through kNowledge IntEgration）。创新点用一个字概括就是——"团"。BERT 的 MLM 是随机 mask 单个 token，但中文里一个完整的语义单位往往是一个词或实体，而不是单个汉字。

举个例子："周杰伦发布了新专辑"。BERT 可能会 mask "杰"和"伦"，让模型通过上下文补全单字。但 ERNIE 1.0 直接 mask 整个"周杰伦"这个实体，迫使模型学会"这个三字组合代表一个歌手"。这就是"实体级掩码"（Entity-level Masking）的核心思想。

掩码策略 原始句子 掩码后 模型需要预测
BERT 随机单字 周杰伦发布了新专辑 周[MASK]伦发布了新专辑 "杰"
ERNIE 实体级 周杰伦发布了新专辑 [MASK][MASK][MASK]发布了新专辑 "周杰伦"
ERNIE 短语级 周杰伦发布了新专辑 [MASK][MASK][MASK]发布了[MASK][MASK] "周杰伦""新专辑"

除此之外，ERNIE 1.0 还引入了短语级掩码（Phrase-level Masking）。短语通过语法分析工具识别，比如"新专辑"是一个偏正短语。掩码短语而非单字让模型学会了更高层级的句法结构。实验证明，在中文 NER、关系抽取、情感分类等任务上，ERNIE 1.0 显著超过了同等规模的 BERT-base。

20.2 ERNIE 2.0：持续多任务增量学习

2020 年 1 月，百度在 AAAI 2020 上发布了 ERNIE 2.0。如果说 ERNIE 1.0 的贡献在于"mask 什么"，ERNIE 2.0 的贡献则是"学什么"——它提出了一个持续多任务增量预训练框架，在预训练阶段加入了大量辅助任务，并且支持随着新任务的出现增量式地扩展。

ERNIE 2.0 的自监督任务分为三大类，总共 7 个任务：

任务类别 具体任务 学习目标
词层面 Knowledge Masking（知识掩码） 继承 ERNIE 1.0 的实体/短语掩码
Capitalization Prediction（大小写预测） 英文词是否大写（中文场景用于识别专名）
Token-Document Relation（词-文档关系） 预测该词是否出现在同一篇文档的其他位置
结构层面 Sentence Reordering（句子重排） 将一个段落内的句子打乱后重新排序
Sentence Distance（句子距离） 判断两个句子是相邻/同文档/不同文档
语义层面 Discourse Relation（篇章关系） 预测句子之间的因果/转折/并列关系
IR Relevance（信息检索相关性） 用搜索点击数据训练 query-文档相关性

这个多任务框架的巧妙之处在于：它不是同时训练所有任务，而是阶段性加入——先训练词层面任务，收敛后加入结构层面任务，再收敛后加入语义层面任务。这种课程学习式（Curriculum Learning）的训练策略让模型的学习更平稳。

💡 实用建议： ERNIE 2.0 的多任务思想后来被广泛应用于很多中文预训练工作中。如果你要预训练自己的中文模型，可以参考这个"先局部后全局、先语法后语义"的课程学习思路，而不是一股脑把所有任务都堆上去。

20.3 ERNIE 3.0：百亿参数的力量

2021 年 7 月，百度发布了 ERNIE 3.0，参数量达到百亿级别（10B），训练数据达到 4TB 规模。ERNIE 3.0 在架构上做了重要改变：不再是单纯的 BERT 编码器，而是采用了统一的生成-理解联合框架，类似于 T5 的 Encoder-Decoder 架构，但通过私有化参数和共享参数的巧妙设计，同时支持 NLU 和 NLG 两种范式。

从 ERNIE 1.0 到 3.0 的演进，清晰地展示了中文预训练模型的"三步走"：先解决"中文的语义单元是什么"（实体掩码），再解决"该学些什么"（多任务），最后解决"模型不够大怎么办"（百亿参数）。

20.4 Whole Word Masking（WWM）

2019 年，哈工大讯飞联合实验室提出了 Whole Word Masking（WWM）中文预训练策略。这个想法异常简洁但极为有效：BERT 在中文上做 MLM 时是按字 mask 的，但"喜"和"欢"分开 mask 根本不合理——因为中文的语义最小单位是词。

WWM 的做法分两步：先对中文句子做分词（用 jieba 或 LTP），然后随机选中一个词，把该词对应的所有汉字全部 mask 掉。

# WWM 掩码逻辑示例
import jieba

def wwm_mask(sentence, mask_prob=0.15):
    # 1. 分词
    words = list(jieba.cut(sentence))
    # 2. 按词概率 mask
    masked_sentence = []
    for word in words:
        if random.random() < mask_prob:
            masked_sentence.append("[MASK]" * len(word))
        else:
            masked_sentence.append(word)
    return "".join(masked_sentence)

# 原始: 我爱北京天安门
# BERT: 我爱[MASK]京天[MASK]门     (单字掩码，需要预测"北""安")
# WWM:  我爱[MASK][MASK][MASK]安门 (全词掩码，需要预测"北京天")
# WWM 让模型的任务更难但更有意义
print(wwm_mask("我爱北京天安门"))
# 输出示例: "我爱[MASK][MASK][MASK]安门"
🔬 效果数据： 在 CLUE 榜单上，BERT-wwm-ext（全词掩码 + 更多训练数据）比原始 BERT-base 在各项任务上平均提升 2-3 个点。尤其在 NER 和阅读理解等需要精确词语边界理解的任务上，提升更加明显。

20.5 ChineseBERT：字形与拼音特征融合

2021 年，复旦大学的 ChineseBERT 独辟蹊径：它认为中文的独特之处不仅在于分词，还在于汉字本身的形态信息和发音信息。ChineseBERT 在 BERT 的标准 token embedding 之外，额外加了两路特征：

  • 字形特征（Glyph Embedding）： 将每个汉字渲染为 24×24 的图片，用 CNN 提取视觉特征。这捕捉了汉字的部首、偏旁、笔画结构等信息（例如"氵"旁的字大多与水有关）。
  • 拼音特征（Pinyin Embedding）： 用 CNN 或 RNN 编码拼音序列（例如"北京"→"bei jing"），捕捉发音信息。这在中文同音词消歧上非常有用（如"琵琶"vs"枇杷"）。

在融合方式上，ChineseBERT 将三路 embedding（词、字形、拼音）拼接后过一个全连接层降维到标准 hidden size，再送入 Transformer：

class ChineseBERTEmbedding(nn.Module):
    def __init__(self, hidden_size=768, glyph_dim=48, pinyin_dim=128):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, hidden_size)
        self.glyph_emb = GlyphCNN(output_dim=glyph_dim)
        self.pinyin_emb = PinyinRNN(output_dim=pinyin_dim)
        self.fusion = nn.Linear(
            hidden_size + glyph_dim + pinyin_dim, hidden_size
        )

    def forward(self, input_ids, pinyin_ids=None, glyph_ids=None):
        tok = self.token_emb(input_ids)
        gly = self.glyph_emb(glyph_ids)   # 字形CNN特征
        pin = self.pinyin_emb(pinyin_ids) # 拼音RNN特征
        combined = torch.cat([tok, gly, pin], dim=-1)
        return self.fusion(combined)

20.6 MacBERT：用近义词替代 [MASK]

MacBERT（MLM as Correction BERT）是哈工大讯飞在 2020 年提出的另一种中文预训练思路。它的核心观察是：预训练阶段的 [MASK] token 在微调阶段永远不会出现，这导致了预训练-微调不匹配（Pretrain-Finetune Discrepancy）。MacBERT 的解决方案是：不直接用 [MASK]，而是用近义词替代被 mask 的词。

具体做法：

  • 对每个被选中的词，用同义词工具（如同义词词林）找一个近义词替换
  • 如果没有合适的近义词，才退化为使用 [MASK]
  • 模型的任务是"纠错"——把替换后的词还原为原文

这种"纠错式"预训练不仅消除了 [MASK] 的预训练-微调 gap，还顺便让模型学会了错别字纠正能力——在真实应用中，用户输入经常包含错别字或同音替代，MacBERT 在这一点上有天然优势。

20.7 NEZHA：相对位置编码的中文 BERT

华为在 2019 年提出了 NEZHA（NEural contextualiZed representation for CHinese lAnguage understanding），核心改进：用相对位置编码替代绝对位置编码。这与我们上一章讲的 DeBERTa 的思路有异曲同工之妙，但 NEZHA 的实现更简洁——它直接修改了自注意力中的 bias 项来注入相对位置信息。NEZHA 在 CLUE 早期榜单上取得过第一名的成绩，也证明了相对位置编码在中文上的有效性。

20.9 中文评测基准：CLUE

CLUE（Chinese Language Understanding Evaluation）是中文 NLP 领域最权威的综合评测基准，涵盖：

  • CLUE 1.0： 包括 9 个任务，涵盖分类（AFQMC、TNEWS、OCNLI等）、阅读理解（CMRC 2018、DRCD）、命名实体识别（MSRA NER）等。
  • FewCLUE： 小样本学习评测，每个任务只提供少量标注样本（几十到几百条）。
  • ZeroCLUE： 零样本评测，测试模型的语义泛化能力。
  • CLUE-GENERAL： 综合榜单，综合各种大小模型的表现排名。
⚠️ 踩坑实录： 在选型中文 BERT 模型时，我经历过一个印象深刻的教训。当时我们做一个法律文书分类项目，贪心地直接选了 CLUE 榜单上排名最高的 RoBERTa-wwm-ext。结果 F1 只有 0.79，反而比 MacBERT 的 0.83 差了一截。原因是法律文书中大量使用"的"字结构（如"…的…的"），而 RoBERTa 使用全词掩码时对高频功能词的掩码效果不好；MacBERT 的近义词替换策略让模型在功能词处理上更鲁棒。所以选型时一定要参考：CLUE 榜单的排名是"平均"排名，不等于在你的特定任务上最优。

20.10 总结与选型指南

中文 BERT 生态虽然百花齐放，但选型时掌握几个基本原则就可以少走弯路：

  • 通用场景（分类、序列标注）： RoBERTa-wwm-ext 或 MacBERT，稳扎稳打
  • 命名实体识别（人名、地名、机构名）： ERNIE 系列（实体掩码的优势）
  • 错别字纠正/用户生成内容： MacBERT（纠错式预训练天然适配）
  • 小样本/零样本场景： 中文 BERT 基座 + FewCLUE 提示微调
  • 计算资源受限： 可以考虑蒸馏后的版本（我们第 23 章会讲）
