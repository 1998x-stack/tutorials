第21章 机器人视觉

大家好，我是你们的讲师。机器人要理解三维世界，靠什么？激光雷达给的是稀疏的点云，IMU给的是加速度和角速度，轮式里程计给的是相对位移——但它们都缺少最丰富的信息：视觉。一张RGB图像包含了颜色、纹理、形状、语义信息，如果能从2D图像中还原出3D世界的结构和语义，那机器人就真正拥有了一双"眼睛"。这就是机器人视觉（Robot Vision）要解决的问题。

2026年的机器人视觉技术已经进入了"多模态大模型时代"：YOLOv10可以在Jetson Orin上以120fps实时检测数百种物体，SAM（Segment Anything Model）可以在零样本下分割任意物体，NeRF和3D Gaussian Splatting让机器人可以从视频流中重建出照片级真实的场景模型。但无论上层AI怎么变，底层的相机几何、特征提取和匹配、3D重建原理始终是根基——不理解这些，你就无法理解什么时候视觉系统会"骗"你。这一章我们从相机模型讲到2026年的前沿3D表示，最后带你搭建一个YOLO+深度相机的视觉抓取管线。

21.1 相机模型与标定

所有机器人视觉的起点，是理解相机如何将3D世界投影到2D图像上。最经典的模型是针孔相机模型（Pinhole Camera Model）。它的原理非常简单：光线通过一个小孔投影到成像平面上，形成倒立的图像。在数学上，从3D世界点 \( (X, Y, Z) \) 到2D像素坐标 \( (u, v) \) 的映射可以写成：

\( s \cdot [u, v, 1]^T = K \cdot [R | t] \cdot [X, Y, Z, 1]^T \)

其中，K是内参矩阵（Intrinsic Matrix），包含了焦距 (f_x, f_y) 和光心 (c_x, c_y)：

\( K = \begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix} \)

[R|t]是外参矩阵（Extrinsic Matrix），表示相机在世界坐标系中的旋转和平移。整个投影过程可以理解为：先通过外参将世界坐标转换到相机坐标系，再通过内参投影到像素平面。

现实中，相机镜头不是完美的针孔，所以存在畸变（Distortion）。径向畸变（Radial Distortion）让直线变弯曲——你看到的"桶形"或"枕形"变形就是径向畸变。切向畸变（Tangential Distortion）则源于镜头和成像平面的安装不平行。经典的张正友标定法（Zhang's Method）通过拍摄不同角度的棋盘格照片，利用棋盘格的已知几何结构同时解出内参和畸变系数。用一个标定板拍15-20张照片，一张棋盘格检测到的角点数据就能建立一组方程——所有照片放到一起就构成了一个超定线性系统，用最小二乘法求解。

实操建议：相机标定是机器人视觉中最容易被忽略但影响最大的步骤之一。我曾经接手一个项目，视觉抓取总是偏了几毫米，调了三个星期的模型和参数都没解决。最后排查到问题出在相机标定——用的还是出厂时默认的内参，但相机在运输过程中受到了振动，内参已经漂移了。重新做了一次标定后，抓取精度立刻恢复。我的建议是：每三个月重新标定一次相机，任何时候更换了相机安装位置都必须重新标定。

21.2 特征提取与匹配

视觉SLAM和视觉里程计的核心是：如何在不同的图像中找到同一个3D点？这需要特征提取（Feature Extraction）和特征匹配（Feature Matching）。

SIFT（Scale-Invariant Feature Transform）是特征提取的"经典之神"。它检测尺度空间中的极值点作为关键点，并为每个关键点计算一个128维的描述子向量。SIFT对缩放、旋转、光照变化和视角变化都有很强的不变性，但计算量巨大——在CPU上处理一张1080p图像可能需要几百毫秒，不适合实时场景。

ORB（Oriented FAST and Rotated BRIEF）是针对实时应用设计的轻量级特征。它用FAST角点检测器找关键点，再用BRIEF描述子（经过旋转校正）描述，结合了速度和一定的旋转不变性。ORB的匹配速度比SIFT快两个数量级，是视觉SLAM系统（如ORB-SLAM3）的首选。

AKAZE是另一种快速特征，采用非线性尺度空间构建，在保持旋转和尺度不变性的同时比SIFT快得多。AKAZE在需要高精度匹配但对计算速度也有要求的场景中非常适用。

特征匹配通常用暴力匹配（Brute-Force Matching）或FLANN（Fast Library for Approximate Nearest Neighbors）。暴力匹配很简单——对每个特征，在另一张图中找描述子距离最近的作为匹配。FLANN则用近似最近邻搜索加速，在特征数量大时效率更高。无论哪种匹配，结果中都会有很多错误匹配（outliers），这时就需要RANSAC（Random Sample Consensus）来剔除。RANSAC随机挑选少量匹配点对估计一个几何变换（如基础矩阵或单应矩阵），然后统计有多少匹配点对符合这个变换——符合最多的那个变换就是最终结果。RANSAC的核心思想是"少数服从多数"——即使50%的匹配都是错的，只要正确的匹配占多数，它就能找到正确模型。

21.3 目标检测与分割

特征提取回答的是"图像中有什么几何结构"，而目标检测回答的是"图像中有什么物体"——这是一个语义层面而不是几何层面的问题。

YOLO（You Only Look Once）系列是实时目标检测的事实标准。2026年，YOLOv8到YOLOv10是主流选择。YOLO将检测视为一个回归问题：将图像划分为网格，每个网格预测多个边界框和类别概率。一次前向传播就可以得到所有检测结果，速度极快（在Jetson Orin上YOLOv8-nano可以达到120fps以上）。YOLOv10引入了"无NMS（Non-Maximum Suppression）训练"的机制，进一步减少了后处理延迟。对于机器人应用，YOLO通常用来检测要抓取的物体、要避让的行人、要识别的标志等。

SAM（Segment Anything Model）是Meta在2023年发布的通用分割模型，2026年已经成为机器人感知管线中不可或缺的组件。SAM可以在零样本下分割任意物体——你只需要提供一个点、一个边界框或一段文字描述作为提示。比如在机器人抓取场景中，你可以用YOLO检测到物体边界框，然后把边界框作为SAM的提示，SAM就会精确分割出物体的像素级轮廓，从而可以计算最佳抓取姿态。SAM的分割质量非常高，但计算量也相应较大，因此在嵌入式平台上通常需要配合TensorRT加速。

2026趋势：YOLO+SAM的组合已经成为机器人视觉感知的标准"双引擎"——YOLO负责快速检测（触发），SAM负责精确分割（精修）。在最新的Isaac ROS中，这两个模型已经被整合为一个硬件加速的管线，可以在Jetson AGX Orin上实现30fps以上的实时运行。

21.4 3D视觉

2D图像虽然信息丰富，但机器人要在三维空间中操作，必须获取深度信息。获取深度信息的主要途径有三种。

立体视觉（Stereo Vision）模仿人类双眼的原理。两个相机之间有一定的基线距离（Baseline），同一个3D点在两个相机上的成像位置有视差（Disparity）。通过三角测量原理，Z = f × B / d，就可以计算出深度。立体视觉的优点是硬件成本低（只需要两个普通相机），但深度精度受基线长度和纹理丰富度的限制，在低纹理区域（如白墙）效果不佳。

深度相机则用主动方式获取深度。Intel RealSense和微软Kinect是典型代表。RealSense D435使用主动立体红外技术——投射不可见的红外纹理图案来增加纹理信息，从而在低纹理区域也能获得可靠的深度。Kinect Azure则采用了ToF（Time of Flight）技术——发射调制红外光，测量反射光的相位差来推算距离。深度相机在1-5米范围内精度很高（毫米到厘米级），但受环境红外光干扰、多机干扰和镜面反射物体的影响。

LiDAR是另一种3D感知方式，用激光束扫描环境直接获取精确的三维点云。机械式LiDAR（如Velodyne、Hesai）提供360度水平视场角，但成本高、体积大。固态LiDAR（如Livox、Ouster）体积小、成本低，已经成为机器人市场上的主流选择。LiDAR的优势是精度高、不受光照影响，但点云稀疏、缺乏纹理信息。

以下是不同3D视觉传感器的对比：

传感器类型 深度原理 量程 精度 帧率 优点 缺点

对于获取的3D点云数据，PCL（Point Cloud Library）提供了丰富的处理算法：体素滤波降采样、统计离群点去除、平面拟合、欧几里得聚类等。ICP（Iterative Closest Point）算法是点云配准的核心算法——给定两组点云，通过迭代寻找最近邻匹配点并求解最优变换，最终将两组点云对齐。ICP是SLAM前端扫描匹配、物体模型配准、手眼标定中最常用的工具。

21.5 新兴3D表示：NeRF与3D Gaussian Splatting

2025-2026年间，两种新的3D表示方法在机器人领域引起了广泛关注。

NeRF（Neural Radiance Fields）用神经网络隐式编码场景的3D几何和外观。给定一组多视角图片，NeRF训练一个MLP网络，输入一个3D位置和观察方向，输出该点的颜色和密度。渲染时从每个像素发射一条射线，沿射线采样网络输出，通过体渲染积分得到像素颜色。NeRF可以生成极其逼真的新视角图像，甚至能重建透明物体和反射表面。在机器人领域的应用包括：场景重建（用于仿真环境构建）、视觉定位（通过NeRF渲染查询位姿）、以及操作规划（在NeRF的隐式几何上规划抓取）。NeRF的缺点是训练时间较长（通常需要数小时到一天），且渲染速度较慢。

3D Gaussian Splatting（3DGS）是2023年提出的新范式，在2026年已经大幅成熟。与NeRF不同，3DGS将场景显式表示为大量3D高斯椭球体——每个高斯体有自己的位置、协方差、不透明度和颜色（用球谐函数表示）。渲染时对所有高斯体按深度排序并投影到图像平面，实现快速的rasterization。3DGS的训练速度比NeRF快数十倍（在GPU上几分钟到几十分钟），渲染速度达到实时（>30fps）。在机器人场景重建中，3DGS已经成为首选方法——波士顿动力的Atlas和宇树H2都已经在演示视频中使用3DGS进行实时环境重建。

踩坑实录——一个"看不见"的玻璃茶几

我在做一个家庭服务机器人项目时，用RealSense D435的深度相机来检测环境中的障碍物。机器人运行了几个月都很正常，直到用户反馈说机器人经常在客厅的某个特定位置"对着空气"绕路。我去现场一看才发现，客厅中央有一张玻璃茶几——透明玻璃在红外深度相机下几乎是完全透明的，深度值直接穿透了玻璃投影到了地板上。这意味着在机器人的障碍物地图中，玻璃茶几完全不存在。更麻烦的是，RGB图像中能看到茶几，但深度图和RGB图的融合逻辑没有为"RGB看到但深度看不到"的物体做特殊处理。这个问题的解法是在融合管线中加入一致性检查：如果RGB检测到边缘但深度没有跳变，对此区域做特殊标注，并用镜面反射模型来估计玻璃位置。从那以后，我深刻认识到：每种传感器都有自己"看不见"的东西——知道传感器的盲区，比知道它的量程更重要。

21.6 实战：YOLO检测 + 深度相机 → 3D抓取位姿

最后我们来搭建一条完整的视觉感知管线：用YOLO在RGB图像中检测目标物体 → 从深度图读取物体区域的深度值 → 结合相机内参计算出3D位置 → 为机械臂输出抓取位姿。这是机器人抓取任务中最经典也最实用的流程。

import cv2
import torch
import numpy as np

# 加载YOLOv8模型（假设已安装ultralytics）
model = torch.hub.load('ultralytics/yolov8', 'yolov8n', pretrained=True)

# 读取RGB图像和深度图（来自RealSense或其他深度相机）
rgb = cv2.imread('scene_rgb.png')
depth = np.load('scene_depth.npy')  # 单位：米

# YOLO检测
results = model(rgb)
detections = results.xyxy[0].cpu().numpy()

# 相机内参（需预先标定）
fx, fy, cx, cy = 615.3, 615.3, 320.0, 240.0

for det in detections:
    x1, y1, x2, y2, conf, cls_id = det
    if conf < 0.5:
        continue  # 低置信度跳过

    # 计算物体中心的像素坐标
    cx_px = int((x1 + x2) / 2)
    cy_px = int((y1 + y2) / 2)

    # 读取中心区域的深度（取中位数以对噪声鲁棒）
    roi = depth[int(y1):int(y2), int(x1):int(x2)]
    valid = roi[roi > 0]
    if len(valid) == 0:
        continue
    Z = np.median(valid)

    # 通过针孔模型反投影到3D世界坐标
    X = (cx_px - cx) * Z / fx
    Y = (cy_px - cy) * Z / fy

    print(f"物体类别: {int(cls_id)}, 3D位置: ({X:.3f}, {Y:.3f}, {Z:.3f})")
    # 输出给MoveIt或ROS 2抓取规划器使用

这段代码虽然简短，但它演示了机器人视觉中最重要的一条思维链：2D检测 → 深度对齐 → 3D反投影。在实际产品中，你还需要考虑：深度图和RGB图的像素对齐（需要相机固件或标定参数）、深度值的时间滤波（减少闪烁）、抓取点的可到达性检查（避免规划目标在关节极限之外）、以及多物体遮挡时的处理逻辑。
