第9章 卷积神经网络

大家好，我是你们的讲师。欢迎回来。

2012 年，Alex Krizhevsky 带着他的 AlexNet 在 ImageNet 图像识别大赛上以碾压性的优势夺冠——错误率从之前的 26% 直接降到 16%。是什么让这个网络如此强大？答案是——卷积层。从那时起，卷积神经网络（CNN）就成了计算机视觉的基石，直到今天，几乎所有图像相关的深度学习模型都离不开卷积操作。

你可能觉得 CNN 在 2026 年已经是"老古董"了——Transformer 不是已经接管一切了吗？但实际上，CNN 远没有过时。在移动端推理、嵌入式设备、实时视频处理等领域，CNN 凭借其高效的局部计算特性依然是标准选择。而且在很多 ViT 模型中，ConvNeXt 等结构证明 CNN 和 Transformer 的融合才是当前的最优解。这一章，我们从最本质的角度来理解卷积。

9.1 卷积的本质：模式匹配

从直观上理解，卷积层做的事情其实很简单——用一个小窗口在图像上滑动，在每个位置检测有没有某种"模式"。

想象你有一张黑白图片，你想找出图片中所有的水平边缘。你可以定义一个 3x3 的"检测器"：

# 水平边缘检测器（Sobel 算子）
kernel = torch.tensor([
    [-1, -1, -1],
    [ 0,  0,  0],
    [ 1,  1,  1]
], dtype=torch.float32)

# 垂直边缘检测器
kernel_v = torch.tensor([
    [-1,  0,  1],
    [-1,  0,  1],
    [-1,  0,  1]
], dtype=torch.float32)

这个 3x3 的矩阵就是 卷积核（kernel）。你把它在图像上从左到右、从上到下地滑动，每次计算核与覆盖区域的逐元素乘积之和。如果某个区域有水平边缘，水平检测器的响应值就会很高；如果是平坦区域，响应值接近零。

这就是卷积的核心直觉：一组可学习的卷积核，每个核负责检测一种特定的局部模式——边缘、纹理、角落、颜色渐变……浅层检测简单模式，深层组合成复杂模式。在 PyTorch 中，nn.Conv2d 就是用来做这件事的，只不过卷积核的值不是手工设计的，而是通过反向传播自动学习的。

9.2 Conv2d 参数详解

import torch.nn as nn

# 定义卷积层
conv = nn.Conv2d(
    in_channels=3,     # 输入通道数（RGB 图像为 3）
    out_channels=64,   # 输出通道数（使用 64 个不同的卷积核）
    kernel_size=3,     # 卷积核大小（3x3）
    stride=1,          # 滑动步长（1 表示每次移动 1 像素）
    padding=1          # 填充像素数（1 保持输入输出尺寸一致）
)
参数 含义 常见选择 备注
in_channels 输入特征图的通道数 RGB 图像为 3，灰度图为 1 在前一层输出通道数变化时相应调整。
out_channels 卷积核的数量（输出特征图的通道数） 64, 128, 256, 512 一般随着网络加深而增加通道数。
kernel_size 卷积核的空间尺寸 3, 5, 7 3x3 是最常用的。两个 3x3 串联等价于一个 5x5 的感受野，但参数更少。
stride 卷积核每次滑动的步长 1 或 2 stride=2 可以替代 MaxPool 来下采样。
padding 输入四周填充 0 的层数 kernel_size//2 padding=kernel_size//2 可保持特征图尺寸不变（当 stride=1 时）。
输出尺寸计算公式：输出高度 = floor((输入高度 + 2×padding - kernel_size) / stride + 1)，宽度同理。这是一个很实用的公式——每次设计网络时先算一下，确保尺寸是整数，否则会报错。

9.3 池化层（Pooling）

池化层的目的是缩小特征图的空间尺寸，同时保留最重要的信息。它就像一个"信息浓缩器"：

pool = nn.MaxPool2d(kernel_size=2, stride=2)
# 输入: [B, C, H, W] → 输出: [B, C, H/2, W/2]
# 每个 2x2 区域取最大值，尺寸缩小为一半

PyTorch 提供了三种池化：

  • MaxPool2d：取窗口内的最大值。保留最显著的特征，丢弃次要信息。最常用。
  • AvgPool2d：取窗口内的平均值。保留区域的整体信息，常用于网络尾部（如 GAP）。
  • AdaptivePool：指定输出尺寸，自动计算 stride 和 kernel_size。输出尺寸固定，不受输入尺寸影响。非常实用，推荐在搭建网络时使用。
# AdaptivePool 可以指定任意输出尺寸
adaptive_pool = nn.AdaptiveAvgPool2d((1, 1))
# 无论输入 H 和 W 是多少，输出都是 [B, C, 1, 1]

AdaptivePool 的妙处在于：它让你的网络可以接受任意尺寸的输入图像。你不需要在输入层强制 resize 到固定大小，AdaptivePool 自动帮你解决这个尺寸不一致的问题。

9.4 BatchNorm2d：训练的稳定器

BatchNorm（批归一化）是 CNN 训练中不可或缺的一环。它的核心思想很简单：把每一层的输出拉回到均值为 0、方差为 1 的标准分布。

bn = nn.BatchNorm2d(num_features=64)  # 64 对应前一层的 out_channels

# 前向传播
output = bn(conv_input)
# 内部计算：
# 1. 计算 batch 的均值 μ 和方差 σ²（在 H, W, N 维度上）
# 2. 归一化：x_norm = (x - μ) / √(σ² + ε)
# 3. 缩放和平移：y = γ * x_norm + β

BatchNorm 有两个模式，它们的表现完全不同：

模式 行为 使用场景
训练模式（model.train()） 使用 当前 batch 的均值和方差来归一化，并更新 running_mean / running_var 训练阶段
推理模式（model.eval()） 使用 累积的 running_mean / running_var（整个训练集的统计量） 验证/测试/推理阶段
极其常见的 Bug：很多人训练完模型后，忘记调用 model.eval() 就做推理。此时 BatchNorm 仍然使用当前 batch 的统计量——如果 batch_size 很小（比如 1），那么均值和方差就是这张图片自身的统计量，毫无意义。结果就是：训练时 loss 正常，推理结果却一团糟。

9.5 关于 BN 和 ReLU 的顺序

这里有一个很多初学者会纠结的问题：Conv → BN → ReLU，还是 Conv → ReLU → BN？

这其实是一个让我们来讲"踩坑"故事的好话题。我之前认识一个做自动驾驶感知的工程师，他的 CNN 模型在验证集上表现一直不太理想。他和团队排查了整整两周，尝试了优化器、学习率、数据增强等各种手段，都没有明显改善。最后是一个新来的实习生偶然发现了一个细节——代码里是 Conv → ReLU → BN 的顺序。

为什么不对？因为 ReLU 会把负值变成 0，而 BatchNorm 的作用是归一化到均值为 0、方差为 1。如果先经过 ReLU，输入到 BN 的特征图已经被"截断"了——全是非负数。此时 BN 学到的均值是一个正的偏移量，和期望的零均值有偏差。正确顺序应该是：

# 正确的顺序：Conv → BN → ReLU
def forward(self, x):
    x = self.conv(x)
    x = self.bn(x)     # 在激活之前归一化
    x = self.relu(x)   # 然后激活
    return x
CNN 组件标准顺序（LeNet 范式）： Conv2d → BatchNorm2d → ReLU → MaxPool2d

这个顺序不是随意定的，背后有数学原因——BN 应该操作在激活函数的"输入空间"而不是"输出空间"上。这是经典论文《Batch Normalization: Accelerating Deep Network Training by Reducing Internal Covariate Shift》中的一个关键设计选择。

9.6 简单 CNN 架构设计

现在我们把所有组件组合成一个经典的 CNN 架构。以下是一个类 LeNet 风格的设计：

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            # 第一组: conv → bn → relu → pool
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),        # 32×32 → 16×16

            # 第二组: conv → bn → relu → pool
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),        # 16×16 → 8×8

            # 第三组
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),        # 8×8 → 4×4
        )

        self.classifier = nn.Sequential(
            nn.AdaptiveAvgPool2d((1, 1)),  # 4×4 → 1×1
            nn.Flatten(),
            nn.Linear(128, num_classes),
        )

    def forward(self, x):
        x = self.features(x)
        x = self.classifier(x)
        return x

这个架构的特征提取部分遵循经典的"卷积→池化"交替模式：每经过一组卷积，特征图的通道数增加（从 3 到 32 到 64 到 128），空间尺寸减半（从 32x32 到 16x16 到 8x8 到 4x4）。分类器部分用 AdaptiveAvgPool 把一个变尺寸的特征图压缩到 1x1，再展平后接全连接层输出分类结果。

9.7 特征图可视化：理解 CNN 在看什么

一个经常被问到的问题是：CNN 的每一层到底学到了什么？答案是：浅层学的是边缘和纹理（Gabor 滤波器状的图案），中层学的是形状和部件（眼睛、轮子、窗户），深层学的是完整语义概念（人脸、汽车、建筑）。

这些特征图可以通过简单的钩子（Hook）机制来提取：

activation = {}

def get_activation(name):
    def hook(model, input, output):
        activation[name] = output.detach()
    return hook

# 在指定层注册钩子
model.features[0].register_forward_hook(get_activation('conv1'))

# 前向传播后查看特征图
_ = model(sample_image)
feature_maps = activation['conv1']  # shape: [1, 32, 32, 32]
# 可以可视化这 32 个特征图，看看卷积核学到了什么

这一章我们深入理解了卷积神经网络的本质——从卷积核的滑动匹配到池化层的浓缩作用，从 BatchNorm 的稳定机制到完整的 CNN 架构设计。CNN 不是深度学习中最"新"的技术，但它是视觉领域最"踏实"的技术——理解 CNN，你就理解了深度学习如何"看"世界。下一章我们进入序列数据的世界——循环神经网络和注意力机制，看看模型如何理解文本和时间序列。
