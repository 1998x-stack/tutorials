第六章：蒙特卡洛方法

同学们好，欢迎来到第六章。前几章我们学了马尔可夫决策过程和贝尔曼方程，然后用动态规划求解了已知模型的MDP。但现实世界有个残酷的问题——大多数情况下你不知道环境的转移概率和奖励函数。你转硬币就知道正反面的概率？你下围棋就知道对手每一步的策略分布？不知道。

这一章讲的方法叫蒙特卡洛（Monte Carlo）方法，简称MC方法。蒙特卡洛是摩纳哥的一个赌城——这个命名暗示了什么？没错，随机采样。MC方法的核心思想是：既然我不知道环境模型，那我就直接和环境交互，拿到完整的一局（episode）数据，然后从这些经验中学习。

6.1 从经验中学习

动态规划（DP）有一个要命的先决条件：你必须知道完整的MDP模型——即状态转移概率 P(s'|s,a) 和奖励函数 R(s,a)。这在很多实际场景中根本不现实。比如你在设计一个机器人走路的控制器，你能精确写出每一步的转移概率吗？几乎不可能。

蒙特卡洛方法完全绕过了这个问题：不需要模型，只需要经验。你让智能体去执行一个策略，记录下一整局游戏的完整轨迹——状态、动作、奖励——然后用这些真实数据来估计值函数。

MC方法的直觉非常朴素：一个状态的价值，就是从这个状态出发所能获得的平均回报。我们不是用公式去算，而是实际去"走"很多遍，然后对回报取平均。就像你想知道"在北京二环里开车到机场要多久"，理论推导再精确也不如实际开车跑几十趟取平均来得实在。

那具体怎么做呢？在MC方法中，我们需要完整的episode（回合）数据。一个episode就是从初始状态到终止状态的完整轨迹：

S₁, A₁, R₂, S₂, A₂, R₃, …, S_T

在每个时间步 t，我们可以计算从这个时间步开始的累积折扣回报：

G_t = R_{t+1} + γR_{t+2} + γ²R_{t+3} + … + γ^{T-t-1}R_T

然后，状态 s 的价值 V(s) 就是所有访问过 s 的episode中 G_t 的平均值。就是这么简单。

这里有一个非常关键的区别，我上课讲到这里的时候，每年都有学生搞混——我们对比一下MC和DP的思路差异：

对比维度 动态规划 (DP) 蒙特卡洛 (MC)
是否需要环境模型 需要（知道P和R） 不需要（从经验学习）
是否使用自举 (Bootstrapping) 是（用其他状态的估计值更新当前状态） 否（用完整的episode回报更新）
更新方式 一步更新 (full sweep) episode结束后更新
估计偏差 有偏（依赖初始估计） 无偏（基于真实采样）
方差 低 高（单次回报波动大）

MC的核心理念：蒙特卡洛 = 完整episode + 平均回报。不需要推断，不需要模型，只需要"跑一遍、记下来、算平均"。这种"从经验中学习"的思路，是强化学习走向实用的第一步。

6.2 MC预测 (MC Prediction)

先讲最简单的任务：给定一个固定的策略 π，用MC方法来估计它的状态值函数 V_π(s)。这个任务叫做"预测"（prediction），也叫"策略评估"（policy evaluation）。

实现上有两种方式：首次访问MC（First-Visit MC）和每次访问MC（Every-Visit MC）。

首次访问MC：在每个episode中，如果一个状态 s 出现了多次，我们只取第一次出现时对应的回报 G_t 来更新。这样做的理由是：回报之间存在相关性（同一个episode后几步的回报嵌套了前几步的回报），只取第一次可以降低估计的有偏性，同时统计性质更好（无偏估计）。

每次访问MC：每次出现状态 s，都把对应的回报 G_t 计入平均。这在实践中更常用，因为利用了更多的数据。

理论上，两种方法都会收敛到 V_π(s)。但当episode数量有限时，首次访问MC的方差通常更小，理论分析也更方便。

下面给出首次访问MC预测的伪代码：

输入：策略 π（待评估），折扣因子 γ
输出：V(s)

初始化：
  V(s) ← 0，∀s ∈ S     （值函数）
  Returns(s) ← []，∀s ∈ S  （存放每个状态的所有回报）

循环（每个episode）：
  用策略 π 生成一个episode：S₁,A₁,R₂,S₂,A₂,R₃,…,S_T
  G ← 0
  对 t = T-1, T-2, …, 1：
    G ← γG + R_{t+1}
    如果 S_t 在本episode中首次出现：
      将 G 追加到 Returns(S_t)
      V(S_t) ← mean( Returns(S_t) )
      （等价于：V(S_t) ← V(S_t) + (G - V(S_t)) / N(S_t)）

返回 V(s)

这段代码可以原地增量更新，不必真的把所有 G 存下来再求平均。只要维护一个计数器 N(s)，每来一个新回报 G，就做：

V(s) ← V(s) + (G - V(s)) / N(s)

这和"每次去新的自助餐厅吃到饱，然后更新你对这家餐厅的评价"是同一个逻辑——第一次去的新体验权重高，去了很多次之后一次体验的影响就变小了。

关键概念澄清：MC不使用自举（bootstrapping）。这是什么意思呢？看看DP的更新公式 V(s) ← Σ π(a|s)[R + γV(s')]——我更新 V(s) 的时候用到了 V(s') 的当前估计值，这叫"用自己的估计来更新自己"。而MC更新 V(s) 时用的是 G_t，这个 G_t 是真实奖励的加总，不依赖于任何其他状态的估计。所以MC是无偏的，换来的代价就是方差大。

6.3 MC控制 (MC Control)

预测是基础，但我们的终极目标是控制——找到最优策略。MC做控制的基本思路和DP的"策略迭代"类似，但有一个关键差异。

思路：

  • 策略评估：用MC方法估计当前策略的动作值函数 Q(s,a)，而不是状态值函数 V(s)。为什么？因为我们要比较在状态 s 下选哪个动作更好，必须知道每个动作的长期价值。
  • 策略改进：基于估计的 Q(s,a)，把策略改进为在每个状态下选择 Q 值最大的动作。

一个关键问题出现了——探索不足。

假设策略是贪心的（总是选当前最优动作），那么从状态 s 出发，智能体永远只走一条路。其他动作从来不被选取，它们的 Q(s,a) 就永远得不到更新。也就是说，你永远无法确定其他动作是不是更好。这个问题在DP中不存在，因为DP能够通过模型"看到"所有动作的结果，而MC只能通过实际采样来获取信息。

解决办法是软策略——最经典的就是 ε-贪心 (ε-greedy)：

  • 以概率 1 - ε 选择当前 Q 值最大的动作（利用 exploitation）
  • 以概率 ε 在所有动作中随机选择（探索 exploration）

这保证了两件事：一是每个状态-动作对都有无限次被采样的机会（保证收敛），二是最终策略可以趋向于最优（不会永远随机乱走）。

在MC控制中，我们还希望探索量随训练过程逐渐减少。这引出GLIE条件：Greedy in the Limit with Infinite Exploration（无限探索下的极限贪心）。GLIE需要满足：

  • 所有状态-动作对都能被无限次探索：即 ε → 0 的速度足够慢
  • 极限情况下策略收敛到贪心策略：即 ε → 0

在实践中，我们通常让 ε = 1/k（其中 k 是episode序号），或者 ε = 1 / log(k+1)。

完整的MC控制（基于 ε-greedy 的首次访问MC）伪代码如下：

输入：折扣因子 γ，探索参数 ε（可随时间衰减）
输出：近似最优策略 π*

初始化：
  Q(s,a) ← 任意值，∀s∈S, a∈A
  Returns(s,a) ← []，∀s∈S, a∈A
  π(s) ← ε-贪心策略基于 Q(s,·)

循环（每个episode）：
  用 π 生成一个episode：S₁,A₁,R₂,S₂,A₂,R₃,…,S_T
  G ← 0
  对 t = T-1, T-2, …, 1：
    G ← γG + R_{t+1}
    如果 (S_t, A_t) 在本episode中首次出现：
      将 G 追加到 Returns(S_t, A_t)
      Q(S_t, A_t) ← mean( Returns(S_t, A_t) )
      更新 π(S_t) 为 ε-贪心（基于更新后的 Q）：
        π(a|S_t) ← 1 - ε + ε/|A|  如果 a = argmax Q(S_t,·)
        π(a|S_t) ← ε/|A|           否则

返回 π* ← 最终策略（ε 衰减到接近0时近似贪心）

看到没？MC控制实际上就是"生成episode → 更新Q → 改进策略 → 再生成episode"的循环。这个框架叫广义策略迭代（GPI），在后续章节中你会不断见到它的影子。

踩坑故事：MC方差的教训。我之前带过一个学生做"21点"的黑杰克游戏AI。他用MC方法，跑了5000个episode后得到了一个策略，看起来很不错——胜率接近42%。他兴奋地直接部署到一个在线模拟器里，结果胜率掉到了35%。为什么？因为5000个episode看起来很多，但在某些边缘状态下（比如手牌15，庄家明牌A），实际只被采样了十几次，Q值的方差极大，估计完全不可靠。这就是MC方法的"阿喀琉斯之踵"——方差。如果你的任务中episode很长（比如1000步），或者奖励稀疏（只有最后一步有奖励），MC的方差会变得非常大。我在实际项目中见过有人用MC做机器人控制，跑了10万个episode，值函数还在剧烈震荡——最后只能换成TD方法。记住：MC用完整收益换来了无偏性，代价就是方差大，数据利用率低。

6.4 MC方法的优缺点

咱们来客观地总结一下MC方法的长处和短处。理解这些，你才能在实际问题中判断"这里该不该用MC"。

优点：

  • 无需环境模型：这是MC最大的卖点。你不知道P和R？没关系，跑起来就行了。这让RL走出"必须建模"的象牙塔，进入了真实应用。
  • 无偏估计 (Unbiased)：MC对 V(s) 的估计是真实值的无偏估计——数学上讲，E[G_t] = v_π(s)。这意味着随着采样次数的增加，估计值会精确收敛到真实值（而不是围绕真实值抖动）。
  • 简单直观：算法逻辑非常直接——跑episode、记回报、取平均。你几乎不需要调试复杂的数学推导。
  • 对状态独立性没有假设：MC不对状态之间的关系做任何假设。值函数中的"干扰"（一个状态估计不准不影响另一个状态）不会像自举方法那样传播误差。
  • 关注有实际意义的状态：MC只评估实际访问过的状态，不会像DP那样在全状态空间上做sweep。在状态空间很大但只有部分状态可到达时，这个特性非常有用。

缺点：

  • 高方差 (High Variance)：这是MC最突出的问题。单个episode的回报波动可能非常大（特别是在随机环境中），需要大量episode才能得到稳定的估计。为了降低方差，你可能需要指数级增加采样数。
  • 只能处理回合制任务 (Episodic Tasks)：MC要求有明确的终止状态。对于continuing tasks（持续任务，没有终止状态），你根本无法定义"完整的episode"。这是MC在工程上一个很大的限制——很多实际问题（如股票交易、工业控制）是持续运行的。
  • 学习效率低：必须等到episode结束才能进行学习更新。如果episode很长（比如一局围棋几百步），你要等很久才能拿到一个更新信号。而且episode内部的信息只被用了一次（只能更新本episode访问过的状态）。
  • 不能利用状态间的结构：因为不做自举（bootstrapping），MC不知道状态之间的关联。它把每个状态当成独立的来估计，这对于状态空间巨大的问题来说非常低效。上面那位学生遇到的"边缘状态采样不足"的问题，本质上就是这个原因。

什么时候用MC？ 四个条件供你判断：(1) 任务是回合制的，有明确终止点；(2) 环境模型未知或极难建模；(3) episode长度适中（几十到几百步）；(4) 你能接受较慢的收敛速度。如果你的任务满足这四点，MC是一个很好的起点。

6.5 MC vs DP vs TD 对比预览

到这里，我们已经学了两大类方法：DP（动态规划）和MC（蒙特卡洛）。下一章我们会学习第三类——TD（时序差分学习）。这三者的关系，构成了整个强化学习算法设计的底层思想框架。这里先做一个总结性的预览：

维度 动态规划 (DP) 蒙特卡洛 (MC) 时序差分 (TD)
是否需要模型 是（必须已知P和R） 否（从经验学习） 否（从经验学习）
是否自举 (Bootstrapping) 是 否 是
更新时机 全状态sweep episode结束后 每一步结束后
是否采样 (Sampling) 否（全期望计算） 是（采样episode） 是（采样步骤）
偏差 有偏（自举引入的初始偏差） 无偏 有偏（自举引入的初始偏差）
方差 低 高 中
数据效率 低（全sweep） 低（只用一个episode） 高（每步都在学）
适用场景 已知模型的小规模MDP 回合制任务、无需模型 通用、最常用

注意看这个二维坐标：DP = Model-based + Bootstrapping，MC = Model-free + No Bootstrapping，TD = Model-free + Bootstrapping。这个观测角度非常重要——三类方法本质上是在"是否需要模型"和"是否使用自举"这两个维度上的不同选择。

DP的思路是"我有模型，我直接计算"。MC的思路是"我没有模型，但是我能收集完整的轨迹，所以我从头算到尾"。TD的思路是"我既没有模型，又不想等那么久，所以我走一步看一步——用下一步的估计来更新当前步"。

下一章我们将正式进入TD学习（时序差分学习）。TD方法结合了MC的"从经验学习"和DP的"自举"优势，是当前强化学习实践中最广泛使用的方法族。你会发现，Q-Learning、SARSA、甚至是Deep Q-Network（DQN）的根基都在TD学习里。好了，本章的蒙特卡洛方法就到这里，咱们第七章再见！
