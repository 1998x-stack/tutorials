第27章 800G/1.6T与400G/通道技术

各位好。上一章我们聊了超以太网（UEC）如何从协议层面解决AI训练的网络瓶颈。但在UEC之下，还有一个更基础的物理层问题——带宽。软件层面的流控和负载均衡再优化，最终也要在物理链路上传输数据。如果链路本身不够快，再聪明的协议也白搭。

今天我们来聊一个与我们的生活距离更近、但又充满技术"黑科技"的话题：以太网物理层的速率演进。从400G到800G，再到1.6T——这些数字是怎么来的？为什么不能用"更快的时钟"简单地翻倍？PAM4、PAM6、LPO、CPO——这些令人眼花缭乱的技术缩写背后，是工程师们在物理极限边缘的精彩博弈。

27.1 为什么需要更高带宽？

简单的驱动因素是：AI集群的规模正在爆炸式增长。2023年大家还在讨论千卡集群，2024年万卡集群已经不再新鲜，到了2025-2026年，十万卡集群开始进入规划阶段。

十万卡是什么概念？假设每张GPU使用400GbE网卡，那就是十万个400G端口。在传统架构中，仅仅是把这些GPU'喂饱'就需要每秒处理PB级别的流量。即使采用前章提到的All-Reduce优化技术，每张GPU在每次迭代中仍然需要收发数百MB的梯度数据。十万张卡的All-Reduce，一秒钟的数据量就是几十TB。

从另一个角度看，当前AI集群的瓶颈已经从"计算力"转移到了"互联力"。很多大模型训练的端到端效率只有30%-50%——换句话说，GPU有一半以上的时间在等待网络传输数据。提升网络带宽，就是直接释放GPU的计算潜力。这解释了两件事：一是为什么业界如此迫切地推进800G和1.6T，二是为什么UEC要把带宽利用率从40%提升到接近100%。

27.2 SerDes速率演进

以太网每向下一代速率迈进，本质上是SerDes（Serializer/Deserializer，串行器/解串器）速率的提升。SerDes是物理层最核心的组件：它把并行的数据流转换成串行信号发送出去，在接收端再把串行信号还原成并行数据。你可以把它想象成一个"高速公路收费站"——并行数据像多车道城市道路，串行信号像单车道高速公路。

SerDes速率的演进历程如下：

世代 单通道速率 调制方式 通道数 典型以太网速率 推出时间 主要挑战
28G SerDes 25-28 Gb/s NRZ（不归零码） 4/8 100GbE / 200GbE 2015 信号反射
56G SerDes 53-58 Gb/s PAM4（4级脉冲幅度调制） 4/8 200GbE / 400GbE 2019 SNR劣化、DSP功耗
112G SerDes 106-116 Gb/s PAM4 4/8 400GbE / 800GbE 2023 信道损耗、串扰
224G SerDes 212-232 Gb/s PAM4 / PAM6 4/8 800GbE / 1.6TbE 2025-2026 超高损耗、DSP复杂度

从这张表中你可以看到一个关键的"跳跃"：从28G到56G，调制方式从NRZ变成了PAM4。这不是巧合。当速率超过30 Gb/s以后，NRZ的信噪比（SNR）瓶颈变得不可逾越——信号频率太高，在PCB和线缆上的衰减过大，接收端已经无法可靠地判断"0"和"1"。PAM4的出现，相当于给SerDes打了一剂"强心针"。

27.3 PAM4与PAM6的调制之争

PAM4（4-Level Pulse Amplitude Modulation）的核心思想很简单：与其用"有信号"和"无信号"两个电平来表示一个bit（NRZ），不如用4个电平来表示两个bit。00、01、10、11分别对应四个不同的电压等级。这样一来，在同样的符号率（baud rate）下，数据传输速率翻倍。

但天下没有免费的午餐。PAM4的代价是信噪比预算的严重劣化。在PAM4中，相邻电平之间的电压差只有NRZ的三分之一。同样的噪声幅度，在NRZ下可能不会导致误码，在PAM4中却会让信号的电压跨过相邻电平的判决阈值，产生误码。换句话说，PAM4系统的信噪比（SNR）要求比NRZ高约9.5 dB——这是一个巨大的差异。

NRZ编码（2个电平，1 bit/符号）：
        电平：   高 ──────────────── 代表"1"
                低 ──────────────── 代表"0"
        电压差：  ████████████████████  （充足）

        PAM4编码（4个电平，2 bit/符号）：
        电平：   高 ──────────────── 代表"11"
                中高 ────────────── 代表"10"
                中低 ────────────── 代表"01"
                低 ──────────────── 代表"00"
        电压差：  ████  （仅为NRZ的1/3）

为了对抗这个SNR劣势，PAM4系统必须依赖DSP（数字信号处理器）。DSP在接收端做均衡、时钟恢复和错误校正，把信号从噪声中"捞"出来。这也是为什么56G和112G SerDes的功耗显著高于28G——DSP的算力需求和功耗随着速率非线性增长。

到了224G SerDes时代，PAM4的调谐余量已经所剩无几。于是PAM6被提上了日程——使用6个电平表示3个bit（实际上是2.5 bit，因为6个电平可以编码3位信息但不完美，需要额外的纠错编码）。与PAM4相比，PAM6的信令效率提升了25%（log2(6)/log2(4) ≈ 1.29，即每个符号多承载29%的信息）。但代价是：相邻电平的间距进一步缩小到PAM4的1/1.5，SNR要求再恶化了约3.5 dB。

PAM6的风险：PAM6在理论上的效率提升是诱人的，但实际工程中，它的DSP复杂度远高于PAM4。目前业界还没有形成统一意见——有些厂商认为PAM6要到400G/通道时才需要，在那之前224G SerDes用PAM4配合更复杂的DSP已经足够；另一些厂商则认为PAM6在224G阶段就应引入，以降低对信道和PCB的苛刻要求。这场争论的本质是"用DSP的复杂度换取信道容量的宽容度"——在摩尔定律放缓的今天，DSP的功耗和成本正在成为越来越重要的约束。

27.4 802.3dj：200G/通道的新标准

2023年，IEEE 802.3启动了802.3dj Task Force，目标是定义200 Gb/s每通道的物理层规范。802.3dj将成为未来5-10年以太网物理层最重要的标准，覆盖以下速率：

  • 200GbE：1通道 x 200G（面向短距服务器接入）
  • 400GbE：2通道 x 200G 或 4通道 x 100G（面向主流数据中心接入层）
  • 800GbE：4通道 x 200G 或 8通道 x 100G（面向AI集群和核心层）
  • 1.6TbE：8通道 x 200G（面向AI超级集群和骨干网）

值得注意的是，802.3dj采用了"灵活的通道化"方式。一个800G端口可以配置为4个200G通道、8个100G通道或16个50G通道。这种灵活性让网络运营商可以根据实际的布线条件选择最优配置——在短距离场景使用更少的通道降低成本，在长距离场景使用更多的通道降低单通道速率保证传输距离。

与802.3dj并行推进的还有802.3dv——定义400G/通道的标准，目标速率3.2 TbE。这将是2028-2030年的技术。从"看远一点"的角度，以太网正在沿着每5-6年翻一倍速率的节奏前进，这个节奏从1980年代保持到了今天，是网络技术史上最惊人的持续演进之一。

27.5 LPO：去掉DSP的革命

说完了调制，再来聊一个有趣的技术路线——LPO（Linear Pluggable Optics，线性可插拔光模块）。它的理念听起来有点反直觉：去掉光模块里的DSP。

传统光模块（如400G DR4/FR4）内部集成了一颗完整的DSP芯片，负责信号调理——均衡、时钟恢复、CDR（时钟数据恢复）、前向纠错（FEC）等。DSP的功耗和成本在光模块中占比很高，以112G SerDes为例，DSP功耗约占模块总功耗的50%-60%。

LPO的思路是：把DSP从光模块中拿掉，将信号调理能力移到交换机的SerDes中。光模块只保留最基本的激光器驱动器和TIA（跨阻放大器），不做任何数字信号处理。这样做的好处很直接：

  • 功耗降低40%-50%：省掉了DSP，光模块功耗大幅下降。对于AI集群这种"万卡交换、百万端口"的场景，功耗节省是千万级别的。
  • 延迟降低：没有DSP的串行化和缓冲，信号路径延迟从纳秒级降到皮秒级。
  • 成本降低：DSP是光模块中最贵的芯片之一，去掉它意味着模块成本降低20%-30%。

但LPO也有明显的代价。因为没有了DSP的均衡和补偿，交换机的SerDes必须直接驱动光模块，对信号的完整性要求更高。这意味着PCB布线、连接器、光纤跳线的质量都必须达到新的标准，任何一个小缺陷都会导致链路不工作。

行业动态：在2024-2025年的OFC（光纤通信大会）上，LPO是当之无愧的热点。多家光模块厂商展示了支持112G SerDes的LPO模块样机。到2026年，支持224G SerDes的LPO模块也开始出现。但LPO的推广仍然面临一个"鸡和蛋"的问题——交换机芯片厂商需要把DSP功能整合到交换芯片中，而交换芯片的设计周期长达3-4年，这使得LPO的规模部署要到2027年以后才能实现。

27.6 CPO与NPO：光引擎的革命

如果说LPO是把DSP从光模块移到交换机，那么CPO（Co-Packaged Optics，共封装光学）就更进一步——把整个光引擎直接封装到交换芯片的基板上。

在传统架构中，交换芯片和光模块是分离的——数据从交换芯片到前面板光模块之间要经过PCB走线、连接器、金手指等多个"关卡"。每经过一个关卡，信号都会劣化。到了112G甚至224G SerDes，这个"芯片到光模块"的物理信道已经变得非常难以设计。

CPO的解决方法是：把光引擎直接做在交换芯片封装里，用非常短的芯片级互连（<5mm）连接交换芯片和光引擎，然后再通过光纤将信号引出到前面板接口。这样一来，电气通道被缩短到极致，信号的完整性挑战大幅降低。

NPO（Near-Package Optics，近封装光学）是CPO的"温和版"——光引擎不在交换芯片封装内部，而是在封装旁边的基板上，通过非常短的基板走线连接。NPO的集成度比CPO低，但制造难度也相对较低，是目前的"折中路线"。

传统架构：
        交换ASIC ── PCB走线(5-15cm) ── 连接器 ── 光模块DSP ── 激光器 ── 光纤
                          ↑信号损耗大、SI挑战大

        LPO架构：
        交换ASIC ── PCB走线(5-15cm) ── 连接器 ── 激光器 ── 光纤
         (含DSP)                    （无DSP）

        CPO架构：
        交换ASIC ── 芯片级互连(<5mm) ── 光引擎 ── 光纤
         (含DSP)                    │
                                    └── 光纤阵列直接引出

CPO的挑战在于：光引擎的制造良率、热管理（激光器对温度非常敏感）和可维护性。目前的CPO产品大多采用"可拆卸光纤连接器"设计——光引擎坏了不用换整个交换机，只需要更换光引擎或重新插拔光纤。但在实际部署中，CPO的可维护性仍然不如传统可插拔光模块。

技术路线之争：2026年的今天，业界对于"LPO vs CPO vs 传统DSP光模块"还没有定论。目前的主流判断是：未来两代（112G和224G SerDes）将是LPO和传统方案的混合部署；而到了400G/通道时代（约2030年），CPO很可能成为主流。因为400G/通道的信号频率已经高到无法在传统PCB上传输——你必须把光引擎和芯片尽可能地拉近。届时，以太网物理层将从"电互联"彻底转向"光互联"。

27.7 残酷的真实故事：400G到800G的升级阵痛

前面讲了很多理论，下面分享一个真实的"踩坑"案例。

2024年，一家大型互联网公司的数据中心决定从400G升级到800G。他们的规划是：将核心交换机的上连从400G x 4升级为800G x 4，采用第一代基于112G SerDes的800G光模块。方案听起来很完美——带宽翻倍，线缆数量不变。

问题出在部署阶段。当他们把第一批800G光模块插入交换机后，发现一半以上的光模块无法稳定建链——链路状态在UP和DOWN之间反复切换，丢包率远高于规格值。工程师们排查了两周才发现问题根源：

第一，原有的单模光纤跳线（G.652.D）在112G SerDes频率下的损耗比预期高出了1-2 dB。400G时代（56G SerDes）勉强可用的光纤链路，到了800G时代就不可用了——频率越高，光纤的弯曲损耗和连接器端面损耗越敏感。

第二，交换机的PCB走线设计在112G频率下出现了严重的谐振问题。400G时代已经验证通过的PCB设计，到了800G时代需要重新设计——走线长度、过孔残留桩（via stub）、耦合间距，每一项都是敏感参数。最终他们不得不召回了一批交换机背板进行返工。

第三，连接器端面污染。112G SerDes对连接器的端面清洁度要求比56G高了一个数量级——一个在400G时代完全可接受的微小灰尘颗粒，在800G时代会导致链路完全中断。为此，他们的运维团队不得不重新制定端面清洁SOP，所有光纤跳线在插拔前必须使用显微镜检查和专用清洁工具处理。

这个故事给我们的教训是：每一次以太网速率的跃升，都是对整个信号链路的全面体检。光纤、连接器、PCB、SFP/QSFP接口——任何一环的微小缺陷都会在更高速率下暴露。这也是为什么网络工程师的"手艺"——端接质量、清洁规范、线缆管理——在高速时代反而变得更加重要。

27.8 下一站：400G/通道与3.2TbE

展望未来，IEEE 802.3dv工作组已经开始定义400G/通道的物理层标准。基于224G SerDes的下一代产品将在2026-2027年进入规模部署，而基于400G/通道的3.2TbE则瞄准2029-2030年。

再往后看，学术界已经在研究太赫兹通信和空芯光纤（Hollow-core Fiber）——前者的目标是突破电子器件的带宽极限，后者的目标是将光纤延迟降低30%（光在真空中比在玻璃中传播速度快）。这些技术一旦成熟，将再次改写以太网的物理边界。

从1973年Metcalfe的2.94 Mbps同轴电缆，到2026年的224G SerDes和800G/1.6T以太网——51年的时间，速率提升了超过50万倍。这个数字背后是几代工程师在材料科学、电磁学和信号处理领域的持续创新。
