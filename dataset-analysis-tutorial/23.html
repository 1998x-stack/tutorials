第23章：数据EDA与可视化工具

假设你刚接手一个从未见过的数据集——不知道有几列、不确定各列的值域范围、不清楚是否存在缺失或重复、也不知道标签分布是均衡还是严重倾斜。这时候你会怎么做？直接送进模型训练？那大概率会踩坑。最佳实践是：先停下来，打开这个数据盒子，仔细看看里面到底有什么。这个过程就是 EDA — Exploratory Data Analysis，探索性数据分析。

EDA 不是一次性的任务，而是贯穿整个数据工程生命周期的基本功。项目初期，你用它理解数据的全貌和潜在问题；预处理阶段，你用它验证清洗和转换的效果；版本迭代时，你用它比较不同版本的数据分布是否发生了偏移。E.D.A. 做得越扎实，后续模型训练中出现的"数据明明没问题但模型就是不收敛"这类玄学问题就越少。

核心理念：EDA 不是给 PM 看的"汇报材料"，而是你做数据工程决策的依据。一个分布图抵得上十次团队讨论。

说一个我亲眼见过的真实案例。2025年我参与的一个NLP项目组拿到了一份约50万条中文电商评论的情感分类数据集。两位同学非常认真，决定先做一轮全面的数据分析。他们各自写了十几个Python脚本，逐列计算统计量、绘制分布图、检查缺失比例和重复条目——整整忙了两天，整理出了一份30多页的PDF报告。

第三天组会上，一个新来的实习生听完汇报后轻声问了一句："我可以用 ydata-profiling 跑一下看看吗？"大家还没反应过来，他已经敲了一行代码。30秒后，一个包含了所有统计指标、缺失值矩阵、相关性热图、异常值检测的交互式HTML报告就生成了——不仅覆盖了两位同学两天手工做的全部内容，还揭示了他们完全没注意到的三个重大问题：某个数值列有超过1000个极端离群值（是正常的200倍）、三个特征之间存在近似完美的多重共线性（R² > 0.98）、以及约2%的样本存在标签矛盾（同一商品的不同评论标注了完全相反的情感极性）。假如团队第一天就跑一下这行代码，能省下整整一天半的时间，并且提前两周发现数据质量问题。

教训：不要用手工脚本重新发明轮子。成熟的EDA工具已经替你想到了你可能根本没想到要检查的问题。先跑工具，再用大脑。

EDA核心分析维度

不管数据是什么形态，EDA都有一些通用的"检查清单"。下面这几项是每次拿到新数据集时必须做的，一个都不能少。

统计摘要与分布概览

第一步永远是 dataset.to_pandas().describe()（或直接Pandas的 .describe()）。这个函数输出每一列数值型特征的计数、均值、标准差、最小值、四分位数和最大值。但这只是最基础的——AI数据集往往需要更深入的统计量：

  • 偏度（Skewness）：衡量分布的不对称程度。偏度绝对值大于1一般视为严重偏斜，说明数据需要变换（对数、Box-Cox等）才能适配正态假设的模型。
  • 峰度（Kurtosis）：衡量分布的"尾部厚度"。高峰度意味着大量离群值，对MSE这类损失函数非常敏感。
  • 缺失率：每列缺失值的百分比。超过5%就需要认真评估是否删除或插补。
  • 唯一值数量：对类别特征尤为重要。如果某个分类列的唯一值接近样本数，那它可能是ID列而非有用特征。
import pandas as pd
import numpy as np
from datasets import load_dataset

dataset = load_dataset("imdb", split="train")
df = dataset.to_pandas()

# 数值统计摘要
print(df.describe(include="all"))

# 自定义统计
summary = {
    "列名": [],
    "类型": [],
    "缺失率(%)": [],
    "唯一值": [],
    "偏度": [],
    "峰度": []
}
for col in df.columns:
    summary["列名"].append(col)
    summary["类型"].append(str(df[col].dtype))
    summary["缺失率(%)"].append(round(df[col].isnull().mean() * 100, 2))
    summary["唯一值"].append(df[col].nunique())
    if np.issubdtype(df[col].dtype, np.number):
        summary["偏度"].append(round(df[col].skew(), 3))
        summary["峰度"].append(round(df[col].kurtosis(), 3))
    else:
        summary["偏度"].append("N/A")
        summary["峰度"].append("N/A")

pd.DataFrame(summary)

缺失值可视化

缺失值不是简单数一下个数就够的。关键问题是"缺失模式"——缺失是随机的，还是有系统性的？如果某个列的缺失值和另一个列高度相关，那很可能存在数据收集环节的系统偏差。

用 missingno 库可以快速生成缺失值矩阵和相关性热图，一眼看出缺失模式：

import missingno as msno

msno.matrix(df)          # 缺失值矩阵：白色条纹就是缺失
msno.heatmap(df)         # 缺失相关性热图：找出共缺失的列对

标签分布与类别平衡

分类任务中，标签分布是第一优先级。一个90%负样本、10%正样本的数据集，不做任何处理直接训练，模型会"学"到永远预测负样本。画一个柱状图或者饼图就一目了然：

import matplotlib.pyplot as plt

df["label"].value_counts().plot(kind="bar")
plt.title("标签分布")
plt.show()
小技巧：对于极度不平衡的数据集（如1000:1），不要只看柱状图，可以计算"不平衡比率"（Imbalance Ratio = 多数类/少数类）。比率超过100意味着你需要采取重采样、cost-sensitive loss 或生成式过采样策略。

Python可视化工具链

Python的可视化生态非常丰富，但不同工具适用于不同场景。下面这个对比可以帮助你做选择：

工具 适合场景 优势 局限性
Matplotlib + Seaborn 静态图表、论文配图、快速探索 API成熟、社区生态强大 交互能力弱、代码量较大
Plotly 交互式探索、仪表盘 缩放/悬停/动画原生支持 大文件渲染较慢
ydata-profiling 一键生成报告、团队协作 自动覆盖所有统计维度 自定义灵活性有限
HF Data Studio HF数据集预览、快速检查 零代码、Web界面 只能处理HF Hub上的数据集

我个人的建议是分层使用：先用 ydata-profiling 扫全貌，然后用 Seaborn 深入看某个具体维度的分布细节，最后用 Plotly 做交互式探索和团队分享。

ydata-profiling的使用极其简单，一行代码生成完整报告：

import pandas as pd
from ydata_profiling import ProfileReport

df = pd.read_csv("my_dataset.csv")
profile = ProfileReport(df, title="EDA报告", explorative=True)
profile.to_file("eda_report.html")  # 生成独立HTML报告

HF Data Studio 预览

如果你使用的是Hugging Face Hub上的数据集，还有一个零代码的选项——Data Studio。在数据集页面上点击"Data Viewer"标签，就可以直接预览数据样本、查看列类型、统计摘要和分布直方图。对于快速了解一个陌生数据集来说，这是最快捷的方式：不需要写任何代码，打开浏览器就能看。

文本数据可视化

文本数据的EDA和表格数据不太一样，你需要关注的是词汇分布、长度特征和语义层面的模式。

词云与词频分布

词云虽然不够"硬核"，但作为沟通工具非常好用——一张图就能让团队直观感受到数据集中哪些词是高频词、领域关键词是什么。

from wordcloud import WordCloud
import matplotlib.pyplot as plt

text = " ".join(df["text"].dropna().tolist())
wordcloud = WordCloud(width=800, height=400,
                      background_color="white",
                      max_words=200).generate(text)
plt.imshow(wordcloud, interpolation="bilinear")
plt.axis("off")
plt.show()

当然，词云只适合定性观察。要定量分析，看词频直方图更有效——尤其要注意那些高频但无意义的功能词（停用词），它们在分类任务中往往是噪音。

文本长度分布

文本长度直接决定了你的 tokenization 策略和模型的最大序列长度（max_seq_length）设置。如果一个数据集里最短的文本只有3个词，最长的有5000个词，而你统一截断到512个token，大部分短文本会被padding浪费，长文本的信息会被截断丢失。

df["text_length"] = df["text"].str.len()
df["word_count"] = df["text"].str.split().str.len()

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
df["text_length"].hist(bins=50, ax=axes[0])
axes[0].set_title("字符长度分布")
df["word_count"].hist(bins=50, ax=axes[1])
axes[1].set_title("词数分布")
plt.show()

percentiles = df["word_count"].quantile([0.5, 0.75, 0.9, 0.95, 0.99])
print(percentiles)  # 看P50/P90/P99来决定截断长度
实践建议：不要用max_length=512这样的默认值。先跑一遍文本长度分布的百分位数。如果P99=128，那设256就绰绰有余，设512只会浪费计算资源。反过来，如果P50就超过了512，那你可能需要考虑分块（chunking）策略而不是简单截断。

主题建模可视化

对于无标注的大规模文本语料，LDA（Latent Dirichlet Allocation）主题建模是一种常用的探索手段。它可以帮助你发现语料中潜藏的主题结构，进而判断数据覆盖是否全面、是否存在某个主题的过度代表。

from sklearn.feature_extraction.text import CountVectorizer
from sklearn.decomposition import LatentDirichletAllocation

vectorizer = CountVectorizer(max_features=1000, stop_words="english")
X = vectorizer.fit_transform(df["text"].sample(10000))
lda = LatentDirichletAllocation(n_components=5, random_state=42)
lda.fit(X)

# 打印每个主题的top词
terms = vectorizer.get_feature_names_out()
for topic_idx, topic in enumerate(lda.components_):
    top_terms = [terms[i] for i in topic.argsort()[:-11:-1]]
    print(f"主题{topic_idx + 1}: {', '.join(top_terms)}")
小技巧：LDA的 n_components 可以用一致性分数（Coherence Score）来调优。遍历3到20个主题，选择一致性分数最高的值。这比拍脑袋定主题数要科学得多。

图像数据可视化

图像数据集的EDA侧重点完全不同——你不能对像素值做描述性统计就完事，必须通过视觉检查才能真正理解数据。

随机样本网格

第一步永远是随机抽样一批图片，排列成网格查看。这看起来简单，但能发现很多元信息层面的问题：图像是否被正确加载？颜色通道顺序是RGB还是BGR？有没有损坏的图片文件？尺寸是否统一？

import matplotlib.pyplot as plt
import numpy as np
from datasets import load_dataset

dataset = load_dataset("cifar10", split="train")
samples = dataset.shuffle(seed=42).select(range(16))

fig, axes = plt.subplots(4, 4, figsize=(8, 8))
for i, ax in enumerate(axes.flat):
    image = samples[i]["img"]
    ax.imshow(image)
    ax.set_title(f"标签: {samples[i]['label']}")
    ax.axis("off")
plt.tight_layout()
plt.show()

图像尺寸分布

图像数据集的尺寸统一性直接影响batch构建策略。如果所有图像都是224×224，那万事大吉；但如果尺寸从32×32到2048×2048不等，你就需要决定是统一resize、padding还是使用多尺度训练策略。

widths, heights = [], []
for sample in dataset:
    w, h = sample["img"].size
    widths.append(w)
    heights.append(h)

plt.scatter(widths, heights, alpha=0.3, s=1)
plt.xlabel("宽度")
plt.ylabel("高度")
plt.title("图像尺寸分布")
plt.show()

print(f"尺寸范围: {min(widths)}x{min(heights)} ~ {max(widths)}x{max(heights)}")
print(f"不同尺寸数量: {len(set(zip(widths, heights)))}")
踩坑预警：如果数据集中混入了灰度图（单通道）和彩色图（三通道），而你的模型输入是3通道，会在训练时报shape mismatch错误。在EDA阶段就应该检查所有图像的通道数，并用 image.convert("RGB") 统一处理。

标注框叠加可视化

对于目标检测数据集，光看图片和标签不够——必须把标注框叠加到图像上，直观确认标注质量：框的位置对不对？有没有漏标的目标？有没有重复标注？

import matplotlib.patches as patches

image = dataset[0]["image"]
boxes = dataset[0]["objects"]["bbox"]  # [x, y, width, height]

fig, ax = plt.subplots(1, 1, figsize=(8, 8))
ax.imshow(image)
for box in boxes:
    x, y, w, h = box
    rect = patches.Rectangle((x, y), w, h,
                             linewidth=2, edgecolor="red", facecolor="none")
    ax.add_patch(rect)
plt.show()

构建EDA报告模板

到这一步，你应该已经意识到：EDA涉及的代码量很大，但大部分代码是重复的。最好的做法是把常用的EDA逻辑封装成可复用的函数库或模块，每次拿到新数据集就调用一下，生成标准化报告。

可复用的EDA函数库

下面是一个轻量级的模板，封装了本章讨论的核心EDA检查项。你可以把它保存为 eda_utils.py，每次在新数据集上运行：

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import missingno as msno
from datasets import Dataset


def run_eda(dataset: Dataset, dataset_name: str = "unnamed"):
    df = dataset.to_pandas()
    print(f"📊 数据集: {dataset_name}")
    print(f"样本数: {len(df)}, 特征数: {len(df.columns)}")

    # 1. 基本统计
    print("\n=== 列级统计 ===")
    info = []
    for col in df.columns:
        info.append({
            "列名": col,
            "类型": str(df[col].dtype),
            "缺失率": f"{df[col].isnull().mean() * 100:.1f}%",
            "唯一值": df[col].nunique(),
        })
    info_df = pd.DataFrame(info)
    print(info_df.to_string(index=False))

    # 2. 缺失值矩阵
    msno.matrix(df)
    plt.suptitle(f"{dataset_name} - 缺失值矩阵")
    plt.show()

    # 3. 数值特征分布
    num_cols = df.select_dtypes(include=np.number).columns
    for col in num_cols[:10]:  # 最多展示10个
        df[col].hist(bins=50)
        plt.title(f"{col} (偏度={df[col].skew():.2f})")
        plt.show()

    # 4. 标签分布（如果存在label列）
    if "label" in df.columns:
        df["label"].value_counts().plot(kind="bar")
        plt.title("标签分布")
        plt.show()

    print("✅ EDA完成")
进阶建议：在这个模板基础上，可以添加红/黄/绿三色标记系统。例如：缺失率 < 1% 标记为绿色、1-5% 为黄色、> 5% 为红色；偏度绝对值 < 0.5 为绿色、0.5-1.0 为黄色、> 1.0 为红色。这样任何人打开报告就能一眼看到需要关注的问题点。

与团队共享EDA结果

EDA报告不是给你一个人看的。推荐的做法是：将生成的HTML报告（ydata-profiling输出或自定义Jupyter Notebook导出）提交到团队的知识库（如Notion、Confluence）或上传到HF Hub的Dataset Card中。这样每个成员——无论是ML工程师、标注团队还是PM——都可以在同一份数据认知基础上讨论问题，而不是各看各的、各说各话。

总结：EDA不是可做可不做的"锦上添花"，而是数据工程的必要前置步骤。用好本章介绍的工具链（ydata-profiling速扫 + Seaborn深挖 + Plotly交互 + 自定义模板复盘），你可以把原本需要几天的手工分析压缩到几十分钟，还能发现那些容易被忽略的系统性问题。

下一章中，我们将从EDA转向另一个同样重要的课题——当你需要构建自己的数据集时该怎么做。自定义数据集构建不只是"把数据塞进Dataset对象"，它还涉及格式设计、Features声明、类型安全、Hub发布等一系列工程决策。而你已经掌握的EDA能力，恰恰就是那个帮助你做出正确决策的指南针。
