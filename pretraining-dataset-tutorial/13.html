给每篇文档打个分：从启发式到模型驱动

大家好，我是你们的讲师。走到这一步，数据已经经过了语言筛选、启发式清洗、多重去重——应该是"干净"了吧？但"干净"不等于"好"。你可能还有大量内容空洞的文章：流水账式的日记、毫无营养的营销软文、AI 自动生成的低质内容。这些文档通过了所有启发式规则，但它们对模型能力的提升贡献接近于零。

2023 年我和团队做了一次消融实验：我们在相同的架构和参数下，分别用"经过启发式过滤的数据"和"额外再用模型评分过滤后的数据"训练了两个 1.3B 模型。结果令人震撼——后者在 MMLU 和 HumanEval 上的分数分别高出 4.2% 和 6.8%。唯一的区别就是：我们用一个 BERT 质量分类器把最低质量的 20% 的文档丢掉了。这证明了质量评分的价值——不是数据越多越好，而是高质量数据越多越好。

这一章我们深入讲解基于模型的文档质量评分方法——从 FineWeb-Edu 和 DCLM 为代表的分类器方案，到基于困惑度（Perplexity）的隐式评分，再到最新的 LLM-as-Judge 和大规模多轴标注体系。

13.1 FineWeb-Edu 分类器

2024 年 Hugging Face 在推出 FineWeb 数据集时，附带发布了一个重要的质量分类器——FineWeb-Edu。它的核心思路很直接：训练一个模型来判断一篇文档的"教育价值"，评分范围为 0-5 分。

FineWeb-Edu 使用的是 distilled BERT（一个经过知识蒸馏的轻量级 BERT 模型），参数量约为 110M。它被训练在约 500,000 条经过人工标注的文档上，标注者根据文档的教育价值从 0（无教育价值）到 5（极高教育价值）打分。

关键发现：只有评分 3 分及以上的文档对模型能力提升有显著正向贡献。FineWeb-Edu 团队通过一系列消融实验证明，当只使用评分 >= 3 的文档训练模型时，在知识密集型 benchmark（如 MMLU、ARC）上的表现显著优于使用全部 FineWeb 数据训练的模型。

from transformers import AutoTokenizer, AutoModelForSequenceClassification
import torch

# 加载 FineWeb-Edu 分类器
model_name = "HuggingFaceTB/fineweb-edu-classifier"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name)

def score_educational_quality(text: str) -> float:
    """返回 0-5 的教育质量评分"""
    inputs = tokenizer(text, truncation=True, max_length=512, return_tensors="pt")
    with torch.no_grad():
        outputs = model(**inputs)
        logits = outputs.logits
        # 模型输出 6 个类别的 logits，映射到 0-5 分
        scores = torch.softmax(logits, dim=-1)
        # 计算加权平均分
        weighted_score = torch.sum(scores * torch.arange(6).float()) / 5.0
    return weighted_score.item()

# 测试
high_quality = "本文介绍微积分基本定理的证明过程..."
low_quality = "今天天气真好哈哈哈大家一起来玩吧"
print(f"高质量文档评分: {score_educational_quality(high_quality):.2f}")
print(f"低质量文档评分: {score_educational_quality(low_quality):.2f}")
实际应用：FineWeb-Edu 分类器在单篇文档上的推理时间约为 5-10ms（GPU）或 50-100ms（CPU）。对于包含 1 亿篇文档的数据集，在单卡 A100 上大约需要 5-6 天完成全量评分。考虑到 FineWeb-Edu 分类器可能偏向英语教育内容，在中文数据集上使用前建议先在小样本上验证其评分分布是否合理。

13.2 DCLM 质量评分

DCLM（DataComp-LM）是另一个重要的数据质量筛选项目，来自多伦多大学和 Together AI 的联合研究。与 FineWeb-Edu 不同，DCLM 使用 fastText 分类器 而非 BERT，因此在推理速度上有显著优势——每秒可以处理数万篇文档。

DCLM 分类器的训练数据是经过精心筛选的：正样本来自维基百科和书籍语料等公认的高质量来源，负样本来自 Common Crawl 中通过启发式规则判断为低质量的文档。通过这个二分类器，DCLM 可以从海量数据中高效筛选出高质量文本。

DCLM 的核心贡献在于它证明了：使用一个简单的 fastText 分类器做质量筛选，就能在大多数 benchmark 上超越更复杂的方案。 关键在于分类器的训练数据质量，而不是模型的复杂程度。

import fasttext

# 加载 DCLM 的 fastText 质量分类器
# 模型可以从 https://huggingface.co/dclm 下载
model = fasttext.load_model('dclm_quality_model.bin')

def score_dclm(text: str) -> float:
    """返回文档属于高质量类别的概率"""
    # DCLM 模型直接输出二分类概率
    labels, scores = model.predict(text.replace('\n', ' '), k=2)
    # 找到 '高质量' 标签对应的概率
    for label, score in zip(labels, scores):
        if label == '__label__high':
            return score
    return 0.0

# 高效地批量处理数百万文档
def batch_filter_dclm(documents: list, threshold: float = 0.5):
    """快速过滤低质量文档"""
    for doc in documents:
        score = score_dclm(doc['text'])
        doc['quality_score'] = score
        if score >= threshold:
            yield doc
对比维度 FineWeb-Edu DCLM
模型类型 Distilled BERT (110M) fastText (~10MB)
输出格式 0-5 多分类 二分类概率
推理速度 ~10ms（GPU） ~0.01ms（CPU）
训练数据 人工标注（50万条） 启发式标注（自动生成）
优势 评分粒度细 速度极快，扩展性好
劣势 推理慢，需 GPU 评分粒度粗

13.3 基于困惑度的质量评分

还有一种更"优雅"的质量评分方法：使用一个小型参考模型来计算文档的困惑度（Perplexity, PPL）。直觉是：高质量的自然语言文本应该具有较低的困惑度——模型能较好地预测其中的 token；而低质量文本（如乱码、无意义重复、机器翻译生硬文本）的困惑度会异常高。

使用困惑度做质量过滤的一个经典案例是 CCNet 工作（2020）：他们使用了一个 1.5B 参数的模型来计算文档的困惑度，然后按困惑度百分位对文档排序，保留困惑度在中间区间的文档（既不高也不低）。为什么丢弃困惑度最低的文档？因为困惑度异常低的文档通常是高度重复的模板化内容。

from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

# 使用一个小型语言模型计算困惑度
model_name = "gpt2"  # 约 120M 参数，速度快
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(model_name)

def compute_perplexity(text: str) -> float:
    inputs = tokenizer(text, return_tensors="pt", truncation=True, max_length=1024)
    with torch.no_grad():
        outputs = model(**inputs, labels=inputs["input_ids"])
        # loss 就是交叉熵损失
        loss = outputs.loss.item()
    return torch.exp(torch.tensor(loss)).item()

def ppl_based_filter(doc: dict) -> bool:
    """基于困惑度的过滤：保留中间 50% 的文档"""
    ppl = compute_perplexity(doc['text'])
    # 实际使用中，你需要先统计 전체数据集的 PPL 分布
    # 然后截取 [P25, P75] 或 [P10, P90] 区间的文档
    return p10 < ppl < p90
需要警惕：困惑度过滤器有一个值得注意的偏见——它倾向于保留模型熟悉领域的文本，可能系统性地丢弃小众领域或罕见表达的内容。例如，一篇关于量子场论的数学推导文章的困惑度可能远高于一篇普通科普文章，但前者的"教育价值"显然更高。因此，困惑度过滤最好与其他质量评分方法结合使用，而不是作为唯一的过滤标准。

13.4 LLM-as-Judge 数据质量评估

2025-2026 年以来，业界开始使用更强的大语言模型本身来评估数据质量——即 LLM-as-Judge 方案。用 GPT-4、Claude 或大型开源模型（如 LLaMA-3-70B）来为训练数据打分，比如：

  • 相关性：文档是否围绕一个主题展开
  • 信息密度：文档中事实性信息与"水词"的比例
  • 可读性：文档的语言是否流畅、逻辑是否清晰
  • 原创性：文档是否包含独特的见解而非拼凑已知信息

LLM-as-Judge 的准确度远高于基于 BERT 或 fastText 的分类器，但成本也非常高——对每篇文档调用一次大模型的 API，对于一个 1 亿文档的数据集来说，费用可能高达数百万美元。因此，实际做法是：用 LLM 标注一小批数据，用它来训练一个轻量级模型做大规模推断。

2025-2026 年的主流做法：大规模多轴标注（Multi-axis Annotation）。不再用一个单一的"质量分"，而是从教育价值、领域术语密度、文本风格、知识层级等多个维度对文档打分。这样，下游的数据配比阶段可以根据每个维度的分数进行精细化的领域混合——例如，在训练数学推理能力时，优先选择"领域术语密度"和"逻辑连贯性"分数高的文档。

13.5 评分阈值的艺术

有了质量评分后，下一个问题是：阈值设在哪里？

对于 FineWeb-Edu 的 0-5 评分，阈值选在 3.0 是最常见的——保留评分 3+ 的文档。这个选择并非随意：消融实验表明，低于 3 分的文档对模型在 benchmark 上的表现贡献很小甚至为负。

但固定阈值的局限性也很明显：不同领域的文档质量分布不同。一篇数学论文可能因为术语密集而被评分较低，但它的教育价值实际上很高。因此，更先进的方案是领域自适应的阈值——按分类（如科技、新闻、文学、学术）分别设定阈值，或者设定每个域保留前百分之多少的文档（如保留每个域的前 30%）。

def adaptive_threshold_filter(documents: list, keep_ratio: float = 0.3):
    """
    领域自适应过滤：按域分别设定保留比例
    每个域保留质量评分最高的 keep_ratio 比例的文档
    """
    from collections import defaultdict
    domain_docs = defaultdict(list)

    for doc in documents:
        domain = doc.get('domain', 'general')
        domain_docs[domain].append(doc)

    filtered = []
    for domain, docs in domain_docs.items():
        # 按评分降序排列
        docs.sort(key=lambda d: d['quality_score'], reverse=True)
        n_keep = max(1, int(len(docs) * keep_ratio))
        filtered.extend(docs[:n_keep])
        print(f"域 {domain}: {len(docs)} -> {n_keep} ({keep_ratio:.0%})")

    return filtered

本章小结

基于模型的质量评分是"从干净到优质"的关键一跃。FineWeb-Edu 的 BERT 分类器提供细致的 0-5 教育价值评分，DCLM 的 fastText 方案追求极致速度，困惑度评分从统计视角筛选异常文本，而 LLM-as-Judge 则代表了质量评估的新边界。将它们组合使用，并配合领域自适应的阈值策略，你可以构建一个精细的数据质量过滤系统。

经过质量评分后，我们留下的文档在信息含量和教育价值上都有了保证。但还有一个重要的安全性关卡必须要过——数据中不能包含有害内容、个人隐私和恶意信息。这就是下一章的主题：有害内容过滤与安全审查。
