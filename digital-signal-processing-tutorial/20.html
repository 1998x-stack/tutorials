第20章 定点DSP实现

上一章我们用Python的浮点运算愉快地做仿真——滤波器响应完美，频谱图漂亮，信噪比精确到小数点后六位。你有没有产生一种错觉："DSP也不过如此嘛，很简单啊。"

好，醒醒。现实世界是残酷的。

你设计的那款优美的32阶FIR滤波器，浮点系数长这样：[0.0034, -0.0098, 0.0245, ...]。但当你要把它烧到一颗没有浮点单元（FPU）的廉价DSP芯片上时，问题来了：这些小数怎么用二进制表示？ 如果每个系数只给16位二进制位，精度损失会不会让滤波器失效？乘法结果的溢出怎么办？

从这一章开始，我们进入DSP开发的"深水区"——定点（fixed-point）实现。这是区分"仿真工程师"和"嵌入式DSP工程师"的分水岭。

20.1 定点数的二进制表示

在开始之前，我们先明确一个基本事实：数字芯片内部只有两种东西——0和1。浮点数（IEEE 754标准）是一种"奢侈"的表示方式，需要专门的浮点运算单元（FPU）来处理，功耗大、面积大、速度慢。而在大多数嵌入式DSP场景中，我们用定点数——一种用固定二进制位数表示小数的方法。

最经典的是二进制补码（two's complement）的Q格式表示。一个Qm.n表示中：

  • m：整数部分的位数（含1位符号位）
  • n：小数部分的位数
  • 总位数 = m + n（不包含额外的符号位——符号位算在m中）

例如：Q15（即Q1.15）表示16位有符号数，1位整数（符号位）+ 15位小数。取值范围：[-1, 1 - 2⁻¹⁵]。精度：2⁻¹⁵ ≈ 3.05 × 10⁻⁵。

Q格式 总位数 整数位 小数位 取值范围 精度
Q15 16 1 (符号) 15 [-1, 0.99997] 3.05e-5
Q31 32 1 (符号) 31 [-1, 0.9999999995] 4.66e-10
Q4.12 16 4 (含符号) 12 [-8, 7.99976] 2.44e-4
Q8.24 32 8 (含符号) 24 [-128, 127.99999994] 5.96e-8
💡 核心认知：Q格式的本质是隐式的小数点位置约定。同样的16位二进制数 0b0100_0000_0000_0000，如果解释为Q15则代表0.5，解释为Q4.12则代表4.0。小数点在哪里，完全是你和代码之间的约定——芯片不会帮你记。这就是定点编程最需要仔细的地方。

20.2 动态范围与精度的权衡

选择Q格式本质上是在做一道取舍题。固定总位宽下，你分给小数的位数越多，精度越高，但整数部分就越少，动态范围就越窄。反之亦然。

公式很简单：

  • 精度 = 2⁻ⁿ（n是小数的位数）
  • 动态范围 ≈ 2ᵐ⁻¹（m是整数位数，去掉符号位）

实际选型时要考虑两个约束：

  • 信号峰值不超过整数部分能表示的最大值——否则溢出，灾难性的结果。
  • 量化噪声不能淹没信号中的微弱分量——否则小信号细节丢失。
# Python中模拟Q格式转换
def to_q15(x_float):
    """将浮点数转换为Q15整数表示"""
    if not -1 <= x_float <= 1:
        x_float = np.clip(x_float, -1, 1 - 2**-15)
    q15_int = int(round(x_float * (2**15)))
    return np.int16(q15_int)

def from_q15(q15_int):
    """将Q15整数转换回浮点数"""
    return float(q15_int) / (2**15)

# 示例：滤波器系数量化
b_float = [0.003418, -0.009827, 0.024536, -0.045679, 0.071234]
b_q15 = [to_q15(x) for x in b_float]
b_recovered = [from_q15(x) for x in b_q15]

print("Float coeffs:  ", [f"{x:.6f}" for x in b_float])
print("Q15 coeffs:    ", [f"{x:6d}" for x in b_q15])
print("Recovered:     ", [f"{x:.6f}" for x in b_recovered])
print("Quantization error (dB):",
      [f"{20*np.log10(abs(a-b)+1e-15):.1f}" for a, b in zip(b_float, b_recovered)])

20.3 量化效应：舍入 vs 截断

从浮点到定点，有一个信息损失的步骤——量化。量化有两种基本模式：

  • 舍入（Round）：四舍五入到最接近的可表示值。误差均匀分布在 ±½ LSB 之间，均值为0。这是默认推荐的做法。
  • 截断（Truncation / Floor）：直接丢弃低位。误差范围在 -1 LSB 到 0 之间，均值为 -½ LSB，会引入直流偏置。

量化噪声的统计模型通常假设为均匀分布的白噪声，方差为 Δ²/12（Δ = 2⁻ⁿ，即LSB步长）。对于16位的Q15格式：Δ = 2⁻¹⁵，噪声方差 ≈ (2⁻¹⁵)²/12 ≈ 8.3 × 10⁻¹⁰。信号功率与量化噪声功率之比 SQNR ≈ 6.02N + 1.76 dB，其中N是位数。对于16位系统，SQNR ≈ 98 dB。这就是CD音质的理论根基。

🔧 实用建议：在Python仿真中，可以用 np.round(x * 2**N) / 2**N 模拟舍入量化效应。如果你想更精确地bit-true仿真（包括饱和、溢出），一定要用整数运算模拟，而不是浮点近似。很多DSP设计团队会用SystemC或Python做bit-accurate的定点模型，这是Level 2验证的核心。

20.4 乘法与累加器（MAC）

定点DSP中最重要的操作是什么？乘累加（Multiply-Accumulate，MAC）。一个FIR滤波器的核心就是一组MAC操作。但这里藏着一个关键细节：乘积的位宽。

两个Q15数相乘：

  • Q15 × Q15 = Q30（15位小数 + 15位小数 = 30位小数）
  • 但最高位是两个符号位的乘积，所以实际的乘积是Q1.30格式，共32位有符号。

如果累加器也是32位的，连续累加时极有可能溢出。因此DSP芯片通常配备40位或64位的累加器——用额外的"守卫位（guard bits）"来防止累加溢出。

/* C语言中模拟定点MAC操作 */
#include <stdint.h>

#define Q15_ONE   ((int16_t)(1 << 15))  /* Q15中的1.0 */

/* 32阶FIR滤波器的定点实现 */
int16_t fir_fixed(int16_t *x, int16_t *coeffs, int taps, int *delay_line) {
    int64_t acc = 0;  /* 64位累加器，带守卫位 */

    for (int i = taps - 1; i > 0; i--)
        delay_line[i] = delay_line[i - 1];
    delay_line[0] = *x;

    for (int i = 0; i < taps; i++)
        acc += (int32_t)delay_line[i] * (int32_t)coeffs[i];

    /* Q15 × Q15 = Q30，右移15位回到Q15 */
    /* 加上饱和保护 */
    acc >>= 15;

    if (acc > INT16_MAX)  return INT16_MAX;
    if (acc < INT16_MIN)  return INT16_MIN;
    return (int16_t)acc;
}

注意上面的关键操作：乘完之后右移15位（acc >>= 15），把Q30格式转换回Q15。如果系数和信号都是Q15，乘积就是Q30，我们需要把它重新归一化到Q15——相当于把乘积的小数点向左移动15位。

很多DSP新手在这里犯错：忘记右移，或者右移位数的计算不对。结果就是滤波器的增益完全错误。

⚠️ 踩坑实录：ADC采样值归一化 有一次我调试一个16位音频ADC的驱动程序。ADC输出的原始值是16位有符号整数，范围是[-32768, 32767]。我直接把它当做Q15输入到FIR滤波器里，结果输出信号小得几乎听不见。排查了半天才发现问题：ADC输出表示的是±Vref的电压，它本质上是一个Q0.15数（0位整数，15位小数），但它的"1.0"对应的是正满量程电压，而不是DSP算法期望的"1.0"。正确做法是先左移1位：q15_input = adc_raw << 1，把范围归一化到[-1, 0.99997]。这个错误在公司review中被抓出来三次——每次不同的人犯。教训是：边界接口的Q格式转换一定要显式写注释，并且做单元测试。

20.5 IIR定点实现的致命陷阱：极限环与溢出振荡

如果说FIR在定点实现中还算"温和"，那么IIR滤波器就是"定点地狱"。原因在于IIR有反馈环路，定点误差会通过反馈不断累积，造成两种独特的破坏性现象。

极限环（Limit Cycle）

当IIR滤波器的输入信号衰减到接近零时，由于量化误差的反馈累积，输出可能陷入一个不衰减的小幅度振荡——这就是极限环。听起来像理想情况下的学术问题？不，在语音通信中，极限环会产生"尾音"——说话人停止说话后，输出端仍然能听到微弱的周期性噪声。

极限环的幅度和频率由滤波器极点位置和量化步长决定。避免的方法包括：

  • 使用溢出后的强制归零（force-to-zero）机制
  • 采用高精度累加器（至少32位用于16位系数）
  • 考虑使用格型结构替代直接型结构，其对极限环更鲁棒

溢出振荡（Overflow Oscillation）

比极限环更可怕的是溢出振荡。当IIR滤波器内部节点因为累加溢出而发生符号翻转时（正数加正数变成负数），反馈环路会把这种错误信号当作"正常"信号继续处理，导致输出产生大幅度的、不可预测的振荡——听起来像爆炸声或啸叫。

溢出振荡的可怕之处在于：它在浮点仿真中完全不会出现。你可以在Python里把滤波器调到完美，然后一烧到定点芯片上，输出就是一片啸叫。

/* 防止溢出振荡的关键：饱和累加而非wrap-around */
int32_t sat_add(int32_t a, int32_t b) {
    int32_t sum = a + b;
    /* 检查溢出：两个正数相加结果为负，或两个负数相加结果为正 */
    if (a > 0 && b > 0 && sum < 0) return INT32_MAX;
    if (a < 0 && b < 0 && sum > 0) return INT32_MIN;
    return sum;
}

/* 二阶IIR节（biquad）的定点实现，带饱和保护 */
int16_t biquad_fixed(int16_t x, int16_t *state, int16_t *coeffs) {
    /* coeffs: [b0, b1, b2, a1, a2] 均为Q15格式 */
    int32_t acc;
    int16_t w;

    acc = (int32_t)x;
    /* 减去反馈部分：w = x - a1*z1 - a2*z2 */
    acc -= (int32_t)coeffs[3] * (int32_t)state[0];
    acc -= (int32_t)coeffs[4] * (int32_t)state[1];
    acc >>= 15;
    acc = (acc > INT16_MAX) ? INT16_MAX : (acc < INT16_MIN) ? INT16_MIN : acc;
    w = (int16_t)acc;

    /* 计算输出：y = b0*w + b1*z1 + b2*z2 */
    acc = (int32_t)coeffs[0] * (int32_t)w;
    acc += (int32_t)coeffs[1] * (int32_t)state[0];
    acc += (int32_t)coeffs[2] * (int32_t)state[1];
    acc >>= 15;
    acc = (acc > INT16_MAX) ? INT16_MAX : (acc < INT16_MIN) ? INT16_MIN : acc;

    /* 更新延迟线 */
    state[1] = state[0];
    state[0] = w;

    return (int16_t)acc;
}

请注意上面代码中的饱和检查（saturation check）——acc > INT16_MAX ? INT16_MAX : ...。这不是可选的优化，而是防止溢出振荡的必需品。

20.6 定点 vs 浮点对比

维度 定点 浮点
运算速度 极快（无FPU开销） 较慢（需FPU，或软件模拟极慢）
功耗 低（逻辑门少） 高（逻辑门多，单指令能耗大）
芯片面积 小 大（FPU占用显著面积）
动态范围 固定，需精心设计 极大（约±10³⁸）
精度 固定，取决于Q格式 可变，随数值大小自适应
开发难度 高，需手动管理标度 低，几乎无标度顾虑
可移植性 不同位宽需重新调参 高度可移植
典型应用 低功耗嵌入式、大规模量产 原型开发、高端音频、基站

20.7 浮点到定点转换流程

最后，我把从浮点算法到定点实现的完整流程总结为以下五个步骤。这是可以应用于任何DSP项目的标准方法论。

  1. Range Estimation（范围估计）： 用浮点仿真跑最大输入信号（正弦满幅、脉冲冲击等），记录所有内部节点的数值范围。找出最大值和最小值。
  2. Q Format Selection（Q格式选择）： 根据估计的范围为每个节点选择合适的Q格式。原则：整数位覆盖max(|min|, |max|)加上¼~½的余量；剩下的位数全部分配给小数。
  3. Quantization Simulation（量化仿真）： 在Python中定点化所有系数和中间变量，做bit-true仿真。对比浮点输出，计算SQNR和最大误差。
  4. Margin Verification（余量验证）： 如果SQNR不满足要求（通常需要 > 60dB），增加总位宽或调整Q格式分配，重复步骤2-4。
  5. Bit-True Verification（Bit-True验证）： 将定点仿真的每个中间结果与硬件（RTL仿真或DSP芯片实测）逐位对比。每一个比特都必须一致——这就是"bit-true"的含义。
🔧 工具推荐：对于范围估计，推荐用 np.finfo(np.float64).eps 级别的浮点精度跑100万以上的样本，确保覆盖所有极值情况。定点仿真可以用Python的 np.int16/np.int32 配合自定义 q_mul/q_add 函数来模拟。商业工具方面，MATLAB Fixed-Point Designer 提供了非常完善的bit-true仿真能力。
