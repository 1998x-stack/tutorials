28.2 区块与链

我们从最基础的东西开始——一个区块（Block）。在比特币中，每个区块包含区块头（前序哈希、时间戳、难度目标、nonce）和交易列表。我们的实现简洁地封装了这些概念：

# === pychain.py — Part 1: Block ===
import hashlib
import json
import time
import os

class Block:
    def __init__(self, index, transactions, timestamp, previous_hash, nonce=0):
        self.index = index
        self.transactions = transactions
        self.timestamp = timestamp
        self.previous_hash = previous_hash
        self.nonce = nonce
        self.hash = self.calculate_hash()

    def calculate_hash(self):
        """计算区块的SHA-256哈希"""
        data = json.dumps({
            "index": self.index,
            "tx": self.transactions,
            "ts": self.timestamp,
            "prev": self.previous_hash,
            "nonce": self.nonce
        }, sort_keys=True)
        return hashlib.sha256(data.encode()).hexdigest()

    def mine_block(self, difficulty):
        """工作量证明：寻找nonce使哈希以difficulty个0开头"""
        target = "0" * difficulty
        while self.hash[:difficulty] != target:
            self.nonce += 1
            self.hash = self.calculate_hash()

    def __repr__(self):
        return f"Block({self.index}, hash={self.hash[:12]}...)"

关键概念：每个区块的哈希依赖于它的所有字段——包括前一个区块的哈希。这意味着任何一个区块的数据被修改，它的哈希就会变化，从而断开与前一个区块的连接（"哈希链"）。这就是区块链不可篡改的根本原因。

28.3 工作量证明（PoW）

工作量证明的核心思想是：要创建一个有效的区块，你必须完成一个计算密集型任务。在我们的实现中，任务是找到一个nonce值，使得区块哈希以特定数量的前导零开头。

当difficulty=4时，目标是以"0000"开头的哈希。每次尝试只有1/65536的概率找到这样的哈希（16^4 = 65536）。这就是"工作"——CPU需要平均进行65,536次哈希计算才能找到一个有效nonce。在比特币中，当前难度要求哈希以约19个前导零开头——找到一次需要约10^19次尝试（全网的ASIC矿机每秒可进行约600 EH/s的计算）。

# === pychain.py — Part 2: Blockchain ===
class Blockchain:
    def __init__(self, difficulty=4):
        self.chain = []
        self.pending_transactions = []
        self.difficulty = difficulty
        self.block_reward = 50  # 挖矿奖励（简化）
        self._create_genesis_block()

    def _create_genesis_block(self):
        """创建创世区块（第0块）"""
        genesis = Block(0, ["Genesis Block"], time.time(), "0")
        self.chain.append(genesis)

    @property
    def latest_block(self):
        return self.chain[-1]

    def add_transaction(self, transaction):
        """将交易添加到待处理池"""
        self.pending_transactions.append(transaction)
        return len(self.chain) + 1  # 返回预计的区块高度

    def mine_pending_transactions(self, miner_address):
        """打包所有待处理交易为一个新区块并挖矿"""
        coinbase_tx = {
            "sender": "COINBASE",
            "receiver": miner_address,
            "amount": self.block_reward,
            "signature": None
        }
        block = Block(
            index=len(self.chain),
            transactions=[coinbase_tx] + self.pending_transactions,
            timestamp=time.time(),
            previous_hash=self.latest_block.hash
        )
        print(f"⛏️  挖矿区块 #{block.index}...")
        block.mine_block(self.difficulty)
        self.chain.append(block)
        self.pending_transactions = []
        print(f"✅ 区块 #{block.index} 已挖出! 哈希: {block.hash[:16]}...")
        return block

    def is_chain_valid(self, chain=None):
        """验证整个链的完整性"""
        chain = chain or self.chain
        for i in range(1, len(chain)):
            current = chain[i]
            previous = chain[i - 1]
            # 验证当前区块的哈希是否正确
            if current.hash != current.calculate_hash():
                print(f"❌ 区块 {i} 哈希不匹配!")
                return False
            # 验证链的连续性
            if current.previous_hash != previous.hash:
                print(f"❌ 区块 {i} 的previous_hash不匹配!")
                return False
        print("✅ 链验证通过!")
        return True

    def to_dict(self):
        """将链转换为字典列表（用于JSON序列化）"""
        result = []
        for block in self.chain:
            block_dict = block.__dict__.copy()
            result.append(block_dict)
        return result
难度调整实验：你可以尝试修改difficulty值来感受挖矿难度的变化。在笔者的MacBook M3上：difficulty=2时几乎瞬间找到nonce；difficulty=4时需要约0.5-2秒；difficulty=6时需要约30秒-2分钟。比特币的全网每2016个区块调整一次难度（约每两周），以确保平均出块时间保持在10分钟。

28.4 交易与钱包

现在我们添加钱包功能。在比特币中，钱包的核心是：私钥（一个256位的随机数）和对应的公钥（通过椭圆曲线secp256k1从私钥推导）。我们使用Python内置库实现一个简化版本：

# === pychain.py — Part 3: Wallet & Transactions ===
def generate_wallet():
    """生成新钱包，返回 (private_key_hex, public_key_hex, address)"""
    private_key = os.urandom(32).hex()
    # 简化：用SHA-256模拟公钥推导（真实比特币使用secp256k1）
    public_key = hashlib.sha256(private_key.encode()).hexdigest()
    # 简化地址：1 + 前34字符的公钥哈希
    address = "1" + hashlib.sha256(public_key.encode()).hexdigest()[:34]
    return private_key, public_key, address


def sign_transaction(private_key, transaction):
    """用私钥对交易进行签名（简化版HMAC-like签名）"""
    message = json.dumps(transaction, sort_keys=True)
    return hashlib.sha256((private_key + message).encode()).hexdigest()


def verify_signature(sender_address, transaction, signature):
    """验证交易签名"""
    if sender_address == "COINBASE":
        return True  # Coinbase交易的签名不需要验证
    tx_copy = {k: v for k, v in transaction.items() if k != "signature"}
    message = json.dumps(tx_copy, sort_keys=True)
    # 简化验证：检查签名字段不为空且sender有效
    # 真实比特币使用ECDSA公钥验证
    return bool(signature) and len(signature) > 0 and len(sender_address) > 0


def create_transaction(sender, receiver, amount, private_key):
    """创建并签名一个交易"""
    tx = {
        "sender": sender,
        "receiver": receiver,
        "amount": amount
    }
    signature = sign_transaction(private_key, tx)
    tx["signature"] = signature
    return tx


def get_balance(blockchain, address):
    """计算指定地址的余额"""
    balance = 0
    for block in blockchain.chain:
        for tx in block.transactions:
            if isinstance(tx, dict):
                if tx.get("receiver") == address:
                    balance += tx.get("amount", 0)
                if tx.get("sender") == address:
                    balance -= tx.get("amount", 0)
    return balance
重要安全提醒：上述签名函数使用了HMAC风格的哈希签名，而不是真正的ECDSA数字签名。这不是生产安全的——真正比特币交易的安全性依赖于secp256k1椭圆曲线数字签名算法。我们的简化版足以演示交易流程的概念，但绝对不能用于任何真实资金。如果你要在生产中使用，请使用成熟的密码学库（如ecdsa或coincurve）。

28.5 P2P网络与Flask API

最后，我们为PyChain添加一个HTTP API层，让多个节点可以通信。使用Flask（一个轻量级Python Web框架）来实现：

# === pychain.py — Part 4: Flask P2P API ===
from flask import Flask, jsonify, request

app = Flask(__name__)
blockchain = Blockchain(difficulty=4)
peers = set()  # 已知的其他节点URL


@app.route("/chain", methods=["GET"])
def get_chain():
    """返回完整区块链"""
    return jsonify({
        "length": len(blockchain.chain),
        "chain": blockchain.to_dict(),
        "pending": blockchain.pending_transactions
    })


@app.route("/mine", methods=["POST"])
def mine():
    """挖矿：将待处理交易打包为新区块"""
    data = request.json or {}
    miner_address = data.get("address", "miner_1")
    block = blockchain.mine_pending_transactions(miner_address)
    return jsonify({
        "message": "挖矿成功",
        "block": {
            "index": block.index,
            "hash": block.hash,
            "transactions": block.transactions,
            "nonce": block.nonce
        }
    })


@app.route("/transactions/new", methods=["POST"])
def new_transaction():
    """提交新交易"""
    data = request.json
    required = ["sender", "receiver", "amount"]
    if not all(k in data for k in required):
        return jsonify({"error": "缺少必填字段"}), 400
    tx = {
        "sender": data["sender"],
        "receiver": data["receiver"],
        "amount": data["amount"],
        "signature": data.get("signature")
    }
    block_index = blockchain.add_transaction(tx)
    return jsonify({"message": f"交易已添加，预计在区块 {block_index}", "tx": tx}), 201


@app.route("/peers/register", methods=["POST"])
def register_peer():
    """注册新的对等节点"""
    data = request.json
    peer_url = data.get("peer")
    if peer_url:
        peers.add(peer_url)
    return jsonify({"peers": list(peers), "count": len(peers)})


@app.route("/peers", methods=["GET"])
def get_peers():
    """获取已注册的节点列表"""
    return jsonify({"peers": list(peers), "count": len(peers)})


@app.route("/validate", methods=["GET"])
def validate():
    """验证区块链的完整性"""
    valid = blockchain.is_chain_valid()
    return jsonify({"valid": valid, "length": len(blockchain.chain)})


@app.route("/balance/
", methods=["GET"])
def balance(address):
    """查询指定地址的余额"""
    bal = get_balance(blockchain, address)
    return jsonify({"address": address, "balance": bal})


@app.route("/consensus", methods=["GET"])
def consensus():
    """简化的共识机制：检查本地区块链"""
    # 在生产环境中，此端点应：
    # 1. 向所有peer请求 /chain
    # 2. 找到最长的有效链
    # 3. 替换本地区块链
    valid = blockchain.is_chain_valid()
    return jsonify({
        "message": "链状态正常" if valid else "链异常",
        "length": len(blockchain.chain),
        "valid": valid
    })

扩展练习

PyChain是一个教学性实现。如果你想进一步挑战自己，以下是一些扩展方向：

  • 真正的ECDSA签名：安装ecdsa库（pip install ecdsa），用secp256k1曲线实现真实的数字签名和验证
  • UTXO模型：将账户模型改为UTXO（未花费交易输出）——就像真正的比特币
  • 难度调整：实现比特币的难度调整算法——每2016个区块重新计算目标难度，保持稳定的出块速度
  • Merkle树：在区块头中存储交易Merkle根，实现SPV（Simplified Payment Verification）验证
  • 内存池：实现交易内存池（mempool）管理——交易优先级、费用市场、孤儿交易处理
  • 完整的P2P共识：实现节点间的真正共识——定期同步、最长链替换、分叉解决
推荐资源：如果你想深入了解比特币核心的实现，推荐阅读bitcoin-core的源代码（C++，github.com/bitcoin/bitcoin），或者阅读《Mastering Bitcoin》第三版（Andreas M. Antonopoulos著）。如果你对以太坊感兴趣，可以去看看Ethereum的Py-EVM实现——一个纯Python的以太坊客户端，理解起来比go-ethereum更容易。

课程结语

到这里，我们的《比特币与加密货币》28章课程正式结束了。

在第一章，我们从7万年前的贝壳货币讲到了中本聪2008年的灵光一现。今天我们亲手构建了一个可以运行的区块链——虽然简化，但它包含了比特币的核心设计：哈希链、工作量证明、数字签名、P2P网络。

让我们回顾一下这28章的旅程：

我们从货币的演化出发，理解了为什么我们需要比特币（第1章）。研读了中本聪的白皮书（第2章），学习了密码学基础（第3章），理解了区块链和去中心化共识的原理（第4-5章）。深入了比特币的网络协议（第6章）和交易机制（第7-9章），掌握了密钥和地址的管理（第10章）。见识了挖矿背后的技术和经济学（第11-12章），理解了分叉和BIP如何推动协议演进（第13-14章）。学习了钱包安全、交易安全、助记词备份和隐私保护（第15-18章）。然后进入更广阔的生态：以太坊与智能合约（第19章）、DeFi（第20章）、Layer 2扩容（第21章）、NFT（第22章）。我们了解了交易所的运作（第23章）、学了链上数据分析（第24章）、动手部署了代币（第25章）。最终，我们站在监管（第26章）和前沿趋势（第27章）的视角展望了未来——然后在这里，用代码践行了这一切（第28章）。

加密货币是一场为期15年（从2009年创世区块算起）并仍在继续的社会实验。它的核心不是价格，不是投机，而是一种基于密码学证明而非信任的价值传输系统——中本聪在2008年写下的那句话，至今仍然是这个领域最精辟的定义。
