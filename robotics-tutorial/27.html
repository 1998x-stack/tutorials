第27章 具身智能与VLA大模型

大家好，我是你们的讲师。上一章我们讨论了人形机器人的"身体"——从ZMP平衡到WBC控制，从四足步态到全身协调。但一个能走能跳的机器人只是一个"健身达人"——它还需要"大脑"来理解世界、理解任务。这就是具身智能（Embodied Intelligence）和VLA大模型的使命。

2026年被认为是"具身智能的ChatGPT时刻"。我不夸张地说，过去两年间机器人领域的最大变革不是硬件突破——硬件在稳步迭代——而是AI模型范式的根本性转变。2023年Google发布了RT-2，第一次证明了一个模型可以同时理解视觉输入、自然语言指令并直接输出机器人关节动作。2024年OpenVLA将这一能力开源。2025年Physical Intelligence的π₀模型用Flow Matching彻底改变了动作生成的精度。2026年SmolVLA在450M参数规模上实现了令人惊讶的灵巧操作能力。这一年一个样，三年大变样——我们正在见证机器人从"被编程"到"被教导"再到"自主学习"的跃迁。

从规则编程到数据驱动

传统的机器人开发管线是这样一个模式：工程师写下感知模块的代码（特征提取、物体检测），然后是规划模块（路径搜索、轨迹优化），最后是控制模块（PID、逆动力学）。每个模块独立设计、独立调试、独立维护。这个管线在过去四十年里非常成功——它让机器人可靠地完成了从汽车焊接芯片封装到仓储分拣的无数任务。但它有一个根本性的局限：每个模块都需要领域专家手工设计，而且面对新的场景、新的物体或新的任务时，手工调整的工作量几乎是线性增长的。

2026年的数据驱动范式完全不同。VLA（Vision-Language-Action）模型直接从像素+文本→动作建立端到端的映射。你不需要写特征提取器——模型的视觉编码器自己学会什么是"抓取点"；你不需要写轨迹规划器——模型的策略头自己学会了什么是"平滑的运动"；你甚至不需要写任务解析器——语言模型部分自己理解了"把红色的方块放到蓝色的圆环里"这样的复杂指令。这不是说传统方法被淘汰了——恰恰相反，传统方法提供的运动学、动力学和安全约束仍然是VLA输出必须对齐的底层基础设施。但决策层面的范式已经完全改变。

为什么2026年是转折点？三个技术条件的成熟推动了这场变革：第一，大规模机器人数据集的积累——Open X-Embodiment数据集包含了22种机器人、60+任务、100万+条轨迹的真实操作数据；第二，预训练模型的迁移能力——互联网规模预训练的大语言模型（如Gemma、Llama-3）为机器人提供了强大的零样本泛化基础；第三，Sim-to-Real的实用化——在仿真中生成无限量训练数据，通过域随机化（Domain Randomization）弥合仿真到真实的差距。

关键洞察：我2019年在斯坦福参加一次机器人研讨会时，一位教授说："如果2020年的AI方法在2025年还没被淘汰，那你的研究可能不够前沿。"当时我觉得这话过于激进。但现在回头看，2023年的RT-1在2024年就已经被RT-2取代，2024年的Octo模型在2025年就被π₀超越，2025年的Flow Matching方案在2026年已经遍地开花。在这个领域，保持学习的速度就是保持竞争力的唯一方式。

VLA模型架构

VLA模型的核心架构可以分解为三个模块：感知编码器（Perception Encoder）→ 语言模型（Language Model）→ 动作解码器（Action Decoder）。

感知编码器接收相机图像（或多视角图像、深度图、点云），将其编码为密集的视觉token序列。常用的视觉backbone包括：SigLIP（Google，基于对比学习的视觉编码器）、DINOv2（Meta，自监督视觉特征）、以及最新的3D Voxel Encoder（直接处理体素数据）。视觉编码器的输出通常是一组空间结构化的特征图——它保留了物体的位置、形状和姿态信息。

语言模型接收用户用自然语言描述的任务指令（如"把桌上的螺丝刀拿起来放到工具箱里"），通过预训练的LLM将语言理解与视觉特征进行跨模态对齐。2026年的主流VLA模型使用视觉-语言联合编码（如LLaVA架构中使用的Q-Former或Cross-Attention机制），让视觉特征和文本嵌入在统一的语义空间中交互作用。语言模型的任务不仅是"理解指令"，还包括生成长时域的任务规划——比如把"泡一杯咖啡"分解为"取杯子→放咖啡粉→按按钮"的子任务序列。

动作解码器是VLA模型中最具机器人特色的部分。它将语言模型输出的隐层表征映射为具体的机器人动作指令。动作空间的定义因任务而异：对于机械臂操作，通常是末端执行器的6D位姿（位置+姿态）+ 夹爪开合度；对于移动操作，还要加上基座的线速度和角速度；对于全身操作，则需要输出所有关节的角度或力矩。π₀模型的一个重要创新是使用Flow Matching替代传统的回归或扩散解码器——动作不是从分布中采样的，而是沿着一个连续流场从噪声平滑变形到目标动作，这使得生成的动作兼具多样性和精准性。

下面是VLA模型推理过程的简化伪代码：

# VLA模型推理：从视觉+语言到动作
import torch
import torch.nn.functional as F

class VLAInference:
    def __init__(self, model, tokenizer, action_dim=7):
        self.model = model.eval()
        self.tokenizer = tokenizer
        self.action_dim = action_dim  # e.g., [x,y,z,roll,pitch,yaw,gripper]

    def predict_action(self, image_tensor, instruction, history=None):
        """
        image_tensor: [3, H, W] RGB图像
        instruction: "pick up the red cube"
        history: 可选的历史observation序列
        """
        # 1. 视觉编码
        visual_tokens = self.model.vision_encoder(image_tensor.unsqueeze(0))

        # 2. 语言编码
        text_tokens = self.tokenizer(instruction, return_tensors="pt")
        text_embeds = self.model.text_encoder(text_tokens.input_ids)

        # 3. 跨模态融合（Q-Former / Cross-Attention）
        fused_features = self.model.fusion_module(
            visual_tokens, text_embeds,
            return_dict=True
        ).last_hidden_state

        # 4. 动作解码（Flow Matching / Diffusion / MLP）
        action_noise = torch.randn(1, self.action_dim)
        with torch.no_grad():
            action = self.model.action_decoder(
                fused_features,
                action_noise,
                timesteps=torch.tensor([0.0])  # Flow Matching终点
            )

        # 输出: [x, y, z, roll, pitch, yaw, gripper]
        return action.squeeze(0).numpy()

实践建议：如果你第一次接触VLA，不要试图从头训练一个模型——那需要数千张GPU、数百万条轨迹数据。正确的做法是使用预训练模型+微调。从HuggingFace下载OpenVLA或SmolVLA的权重，用你自己的机器人收集几百条轨迹数据做LoRA微调。一天之内就能在真实硬件上看到效果。

关键模型盘点

从2023年RT-2发布到现在，VLA模型的发展可以用"日新月异"来形容。我把2026年最值得关注的几个模型整理成了一张全景表：

模型 发布时间 参数量 架构特点 动作解码 开源 核心性能
RT-2 2023.07 55B PaLM-E + PaLI-X融合 离散token分类 否 zero-shot泛化，适合桌面操作
OpenVLA 2024.06 7B SigLIP + Llama 2 离散化+MLP 是 开源标杆，LoRA微调友好
π₀ (pi-zero) 2025.03 3.3B DINOv2 + Flow Matching Flow Matching 是 灵巧操作，接触力隐式建模
SmolVLA 2026.01 450M SigLIP tiny + Gemma 2B蒸馏 扩散+窥视孔 完全开放 边缘端实时推理（35ms）

让我特别解读几个重要的模型：

π₀（Physical Intelligence，2025）是VLA领域的一个里程碑。它抛弃了传统的"先检测物体再规划动作"的两阶段范式，直接用Flow Matching生成平滑的末端轨迹。Flow Matching相比扩散模型的优势在于：它定义了一个确定性的概率路径，从噪声到动作的映射是连续的、可微的，这使得动作生成在时间和空间上都更加稳定。π₀在多种操作任务上（开瓶、叠衣服、使用工具）的成功率超过了之前所有开源模型。

SmolVLA（2026）回答了"VLA模型能不能上边缘端"这个问题。通过知识蒸馏（用7B的OpenVLA教450M的SmolVLA）和架构优化，SmolVLA在Jetson Orin上实现了35ms/step的推理速度——满足实时控制要求的<50ms阈值。这是VLA从"科研演示"走向"工程部署"的关键一步。

DeepSeek-R1 Robotics（2026）将671B参数的MoE大模型与机器人动作空间对齐，最大的亮点是其Chain-of-Thought推理能力：模型在输出动作之前先生成一个自然语言的"思考过程"（"这是一个蓝色方块的场景，我需要先移动到桌子前方，然后调整夹爪到水平姿态……"），然后再生成动作序列。这种显式的推理过程不仅提高了复杂任务的完成率，还让人类可以理解模型"为什么这么做"——这对安全性和可调试性至关重要。

踩坑提醒：我在部署OpenVLA时遇到过一个经典问题。模型在仿真中可以准确地完成"拿起红色方块"的任务，但在真实场景中，同样的指令会导致夹爪直接穿过物体抓空。排查发现：仿真中的相机标定是完美的，而真实相机的内参和外参都有微小误差——视觉编码器对这些误差极其敏感。解决方案是在微调数据中加入随机扰动（平移±2mm、旋转±1°），让模型学会对微小标定误差保持鲁棒。这个技巧叫做视觉域随机化（Visual Domain Randomization），对所有VLA模型都有效。

训练数据与生态

VLA模型的数据基础设施建设在过去两年取得了巨大进展。核心数据集的规模和质量直接决定了模型的泛化能力。

Open X-Embodiment（OXE）是VLA领域的"ImageNet时刻"。这个由Google DeepMind发起、全球33个机构联合贡献的数据集包含了22种不同类型的机器人（从单臂桌面操作到双臂人形）、超过60个任务类型的100万+条遥操作演示轨迹。数据的多样性是OXE最大的价值：一条UR5e机械臂的演示数据可能让Franka Emika的控制器泛化到新的操作技能。但OXE也有明显的短板：数据标注质量参差不齐、动作采样率不统一、传感器配置各异——各机构向模型训练代码都会包含大量的数据清洗和标准化步骤。

LeRobot（HuggingFace维护）提供了统一的数据格式和工具链。它将各种机器人的轨迹数据标准化为统一的Parquet/Arrow格式，并提供了与ROS 2、MuJoCo和Real Robot的接口。2026年LeRobot生态系统已经包含了超过500个公开数据集，覆盖了从简单的pick-and-place到复杂的双臂协调操作。它的最大贡献是让"在自己的机器人上训练VLA模型"的门槛从"全栈工程团队"降低到了"一个硕士生"。

数据采集方面，2026年的主流方案是遥操作+示教学习。操作员通过Apple Vision Pro或VR手柄远程操控机器人完成示范任务，系统记录图像、关节状态和力传感器数据。AgiBot G1在2026年发布了一个开源数据集，包含了100万条高质量的人形机器人操作轨迹——由50个经过培训的操作员在6个月内完成采集。这个数据集的发布极大地推动了中国VLA模型的研究进展。

2026前沿与挑战

2026年的VLA研究正在向更深层次的智能方向推进。这里有几个我特别关注的前沿方向：

人类视频作为演示提示——在ICRA 2026上，多篇论文展示了使用人类操作视频（比如YouTube上的"如何打开瓶盖"视频）直接作为VLA模型的提示（Prompting），而不需要任何机器人演示数据。模型通过观看人类视频理解任务的目标和关键动作阶段（"旋转"而不是"拉"），然后在机器人本体上生成等效的动作序列。这个方向如果成熟，将从根本上改变机器人数据采集的瓶颈——互联网上存在几乎无限量的人类操作视频。

One-shot Policy Learning——传统模仿学习需要数十到数百条演示数据来学习一个策略。2026年的前沿工作正在将这一数字降低到1条。通过在大规模预训练中学习到的"底层技能词汇表"（Skill Vocabulary），模型在看到一个单一演示后，就可以将其分解为已知技能的序列并泛化到新的初始条件。Meta的论文展示了一个双臂机器人仅凭1条演示就学会了"倒水"任务，并在杯子位置偏移20cm的情况下仍然成功。

Kinematic Intelligence（运动智能）——这是一个有趣的概念。研究发现，VLA模型在预训练过程中不需要显式的运动学建模，就自发地学会了理解刚体运动、关节限位和可达空间等概念。当给定一个机械臂的参数时，模型可以预测哪些6D位姿是可到达的、哪些会触发奇异姿态——而它从未被训练过这些知识。这暗示了足够规模的数据中可能隐含了物理世界的深层结构。

Cross-embodiment Transfer（跨本体迁移）——在一个机器人上训练的VLA模型能否直接部署到另一个完全不同的机器人上？2026年的答案是"部分可以"。如果两个机器人在动作空间上具有同构性（比如都是7-DOF机械臂，只是品牌和尺寸不同），迁移的成功率可达85%以上。但如果是从机械臂迁移到人形机器人的全身操作，成功率为20-30%。弥合这个差距需要更通用的动作表征。

当然，VLA模型远未成熟。最大的三个挑战是：长时域推理（当前模型在任务步数超过10步后的成功率急剧下降）、物理常识（模型不理解"杯子放在桌子边缘会掉下来"这样的物理直观）、安全保障（端到端模型可能在学习数据中不包含的边缘情况下产生不可预测的动作）。这些挑战也是未来3-5年最活跃的研究方向。

开源实战：OpenVLA/LeRobot

理论讲了很多，现在动手吧。下面我带你走一遍使用OpenVLA和LeRobot在自己的机器人上训练一个"抓取红色方块"任务的全过程。

# Step 1: 环境准备（需要一台有NVIDIA GPU的电脑）
# conda create -n vla python=3.10
# pip install openvla lerobot torchvision torch

import torch
from transformers import AutoModelForVision2Seq, AutoProcessor
from lerobot import LeRobotDataset

# Step 2: 加载预训练OpenVLA模型（7B参数）
model = AutoModelForVision2Seq.from_pretrained(
    "openvla/openvla-7b",
    torch_dtype=torch.bfloat16,
    device_map="auto"
).eval()

processor = AutoProcessor.from_pretrained("openvla/openvla-7b")

# Step 3: 准备微调数据（假设你已经用遥操作采集了50条轨迹）
# 数据格式: {image, joint_state, action, instruction}
dataset = LeRobotDataset.load(
    "my_robot_pick_red_cube",
    split="train",
    policy="openvla"
)

# Step 4: LoRA微调（只需要训练~0.5%的参数！）
from peft import LoraConfig, get_peft_model

lora_config = LoraConfig(
    r=32,               # LoRA rank
    lora_alpha=64,
    target_modules=["q_proj", "v_proj"],  # 只微调注意力层
    lora_dropout=0.05
)
model = get_peft_model(model, lora_config)

# 训练配置
optimizer = torch.optim.AdamW(model.parameters(), lr=5e-5)
for epoch in range(10):
    for batch in dataset:
        # 预处理: 图像resize + 指令token化
        inputs = processor(
            images=batch["image"],
            text=batch["instruction"],
            return_tensors="pt"
        ).to("cuda")
        # 前向传播 & 损失计算
        outputs = model(**inputs, labels=batch["action"])
        loss = outputs.loss
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()
    print(f"Epoch {epoch}: loss = {loss.item():.4f}")

# Step 5: 保存并部署到机器人
model.save_pretrained("my_openvla_pick_lora")
print("微调完成！现在可以部署到机器人上运行推理了。")

这段代码看起来简单，但每一步都有坑。数据采集质量比数量更重要——50条高质量、覆盖不同初始条件的演示远胜于500条低质量重复数据。Lora的rank参数需要在32-64之间调优：太小了学不到新技能，太大了容易过拟合到50条数据上。推理时的动作平滑也很关键——直接使用模型的单步输出会导致抖动，加一个低通滤波器（指数滑动平均，alpha=0.3）通常就能大幅改善。

重要提醒：VLA模型不会取代你对机器人基础的理解——恰恰相反，它让你的基础更有价值。当你调试VLA模型的失败案例时，80%的问题可以追溯到物理层面的原因：逆运动学解超出关节极限、末端执行器与障碍物碰撞、力传感器的零点漂移。没有扎实的机器人学基础，你连"模型为什么失败"都说不清楚。VLA是锦上添花，不是空中楼阁。
