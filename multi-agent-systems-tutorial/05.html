第5章 CTDE：集中训练分散执行

还记得我们在第3章讨论的第一个MAS核心挑战吗？非平稳性——其他Agent也在学习，这意味着从任何一个Agent的视角看，环境都在不断变化。现在加上第4章博弈论给我们的启示——每个Agent都在追逐自己的利益，均衡极难收敛。面对这双重困难，我们需要的是一种"既要又要"的架构：既要让Agent能利用全局信息做更好的学习，又要让Agent在部署时独立决策、不依赖中央协调。（CNN）

这就是CTDE（Centralized Training with Decentralized Execution）——集中训练，分散执行。CTDE是目前多智能体强化学习（MARL）领域最重要、应用最广泛的范式之一。它的核心理念说起来很简单：训练时让Agent看到完整的全局信息，但执行时每个Agent只根据自己的局部观测来做决策。但把这个理念变成可运行的算法，中间需要跨越一系列精巧的设计。

让我先讲一个故事。2023年我在开发一个多机器人仓库调度模拟系统，最初的方案是"完全分散"——每个机器人只看自己的传感器数据做决策。结果呢？在模拟环境中，两个机器人同时冲向同一个货架——因为从各自的局部视角看，那个货架都是最近的。它们发生了"死锁"——谁也不让谁，因为每个机器人的策略都是"尽快完成任务"，而让路意味着自己的奖励延迟。这是一个典型的协调失败。后来我们改成了CTDE方案：训练时，一个"上帝视角"的评论家能看到所有机器人的位置和任务，由此给出每个机器人行动的全局价值评估；但执行时，每个机器人依然只用局部传感器信息做决策。结果神奇的协同行为出现了——机器人学会了"主动让路"，因为在训练中评论家让它们明白：让路虽然暂时损失个体奖励，但全局上提高了整体效率。

5.1 训练与执行解耦

CTDE的动机可以从"信息可用性"的角度来理解。在训练阶段，Agent通常运行在模拟器或受控环境中，我们可以廉价地获取全局信息——所有Agent的状态、动作和奖励。但在执行阶段，Agent部署在真实环境中，通信有延迟和带宽限制，全局信息不再可用。

CTDE的精髓就是：利用训练阶段的全局信息来指导Agent学习，但最终产出的策略只依赖局部观测。这样既获得了全局信息的好处，又保持了执行时的独立性和可扩展性。

CTDE的数学框架可以通过两种主流方式实现：价值分解（Value Decomposition）和集中评论家（Centralized Critic）。价值分解用于价值基方法（如VDN、QMIX），集中评论家用于策略梯度方法（如MADDPG、MAPPO）。本章聚焦价值分解，下一章聚焦集中评论家策略梯度。

关键理解： CTDE不是一个具体的算法，而是一种设计范式。它的核心假设是：训练和部署的环境条件可以不同——训练时享有"上帝视角"的奢侈，部署时回归"有限视角"的约束。这个假设在很多真实场景中成立（模拟器训练、限制性部署），但在一些特殊场景中（如边缘部署完全不可联网）可能不适用。理解这个假设，你才能判断CTDE是否适合你的场景。

5.2 价值分解的IGM条件

在协同MAS中，我们希望学习一个联合动作价值函数 Qtot(s, a)，它评估在状态s下所有Agent采取联合动作a的全局价值。但部署时，每个Agent只能基于自己的局部观测oi来选择动作ai。问题来了：如何从Qtot推导出每个Agent的个体动作价值函数 Qi(oi, ai)，使得Agent独立执行时选择的动作恰好是全局最优的？

这个问题的数学答案就是IGM条件（Individual-Global-Max）：

IGM条件：全局最优动作 = 每个Agent局部最优动作的联合

形式化定义：
argmax_a Q_tot(s, a) = (argmax_{a_1} Q_1(o_1, a_1),
                        argmax_{a_2} Q_2(o_2, a_2),
                        ...,
                        argmax_{a_N} Q_N(o_N, a_N))

直观理解：如果每个Agent都选择自己的Q_i最大的动作，
那么联合起来的动作一定让Q_tot也最大。
反之亦然：Q_tot的全局最优解，恰好由每个Agent的局部最优解组合而成。

IGM条件听起来很自然，但它实际上是一个非常强的约束。它意味着Agent之间不能有任何"冲突"——一个Agent的行动决策不能依赖于另一个Agent的决策。如果AgentA的最优动作是"向右走"，而AgentB在AgentA向右走后只有"向左走"才最优——这种情况IGM条件就不满足。现实中大量MAS场景都存在这样的"动作依赖"，因此满足IGM条件的算法在设计上有很多学问。

5.3 VDN：加性价值分解

VDN（Value Decomposition Network）是最早尝试实现IGM条件的算法之一，它的思路简单到令人惊讶：

Q_tot(s, a) = sum_i Q_i(o_i, a_i)

即将全局价值函数分解为每个Agent个体价值函数的简单加和。

加性分解的优点是IGM条件自然满足——因为求和运算中，每个分量独立最大化即全局最大化。而且实现非常简单：每个Agent有一个独立的Q网络，输出Qi，将所有Qi相加得到Qtot，然后用标准的DQN损失函数来训练Qtot。

但VDN有一个根本性局限：它只能表示加性可分解的价值函数。如果Agent之间存在非线性交互——比如AgentA和AgentB同时攻击同一个目标的效果远大于各自单独攻击的效果之和——VDN就无法捕捉这种协同增益。换句话说，VDN假设Agent之间是独立的，但MAS中Agent之间恰恰存在复杂的相互依赖。

直觉理解： VDN假设团队的整体价值等于"每个人贡献的简单加总"。这就像一个足球队的教练说"球队的总表现就是每个球员的个人表现之和"——但现实中，好的球队有化学反应、有战术配合、有协同效应，这些都是非线性交互。VDN忽略了这些非线性。

5.4 QMIX：单调混合网络

QMIX在VDN的基础上迈出了一大步。它不假设Qtot是Qi的简单加和，而是用一个混合网络（Mixing Network）来学习Qtot和Qi之间的映射关系，同时施加单调性约束来保证IGM条件。

单调性约束：
∂Q_tot / ∂Q_i >= 0,  ∀i

即：任何单个Agent的Q_i增大，Q_tot不会减小。
这保证了：每个Agent独立最大化Q_i  →  全局Q_tot也最大化。

QMIX的混合网络是一个超网络（Hypernetwork）：它不直接学习混合网络权重，而是用一个单独的网络从全局状态s生成混合网络的权重参数。这样，混合网络的参数会随着环境状态的变化而动态调整，使得Qtot到Qi的映射具有丰富的表达能力——但仍被约束为单调的。

QMIX的架构可以用伪代码简洁地表示：

class QMIX:
    def __init__(self, n_agents, state_dim):
        # 每个Agent的个体Q网络
        self.agent_q_nets = [QNetwork() for _ in range(n_agents)]
        # 超网络：从全局状态生成混合网络权重
        self.hypernet_w1 = nn.Linear(state_dim, n_agents * mixing_embed_dim)
        self.hypernet_w2 = nn.Linear(state_dim, mixing_embed_dim * 1)
        # 确保混合网络权重非负（实现单调性）
        self.abs = nn.ReLU()

    def forward(self, observations, state):
        # 每个Agent基于局部观测计算Q_i
        q_vals = [net(obs_i) for net, obs_i in zip(self.agent_q_nets, observations)]
        q_vals = torch.stack(q_vals, dim=-1)  # [batch, n_agents]

        # 超网络生成混合网络权重（非负）
        w1 = self.abs(self.hypernet_w1(state))   # [batch, n_agents * embed]
        w2 = self.abs(self.hypernet_w2(state))   # [batch, embed * 1]
        b1 = torch.zeros_like(...)               # 偏置项

        # 单调混合：Q_tot = ReLU(q * w1 + b1) * w2
        hidden = F.elu(torch.matmul(q_vals, w1.reshape(...)) + b1)
        q_tot = torch.matmul(hidden, w2.reshape(...))
        return q_tot

    def loss(self, q_tot, rewards, q_tot_next, dones):
        # 标准TD损失：训练Q_tot
        td_target = rewards + gamma * q_tot_next * (1 - dones)
        return F.mse_loss(q_tot, td_target.detach())

QMIX在实践中取得了巨大的成功——在SMAC（星际争霸多Agent挑战）基准上显著超越了VDN。它的成功说明了：更好的价值分解方式直接带来更好的协同行为。但QMIX也有局限：单调性约束虽然保证了IGM，但同时也限制了Qtot的表达能力——有些MAS场景需要非单调的Qtot，比如"AgentA向右走"只在"AgentB也向右走"时才是好动作，否则是坏动作。

5.5 Weighted QMIX与QPLEX

为了突破单调性限制，研究者提出了两个重要的改进方案。

Weighted QMIX（CW-QMIX）： 核心思想是——我们不需要在所有状态和所有动作组合下都满足单调性约束，只需要在当前最优动作的邻域满足单调性就够了。CW-QMIX引入了加权损失函数：对"可能被选中的动作"施加更强的单调性约束，对"不太可能被选中的动作"放松约束。这样既保留了IGM条件，又赋予了模型在非单调区域的学习能力。

QPLEX（Duplex Dueling架构）： QPLEX采用了更革命性的设计——它不依赖单调性约束，而是使用duplex dueling架构来显式地表示Qtot和Qi之间的关系。具体来说，QPLEX将Qtot分解为两个部分：

Q_tot(s, a) = sum_i Q_i(o_i, a_i)  +
               Advantage_{tot}(s, a) - sum_i Advantage_i(o_i, a_i)

其中：
- 第一部分 sum_i Q_i 是VDN风格的加性分解
- 第二部分是"全局优势函数"和"个体优势函数之和"的差值
- 这个差值捕捉了Agent之间的非线性交互效应

通过Transformer风格的注意力机制，QPLEX可以学习复杂的Agent间交互模式，
同时通过结构设计保证IGM条件（不需要单调性约束）。

QPLEX在SMAC的多个困难场景上取得了超越QMIX的结果，特别是在需要Agent间强协调、非单调交互的"走廊"地图上表现尤为突出。

5.6 CTDE的局限与演进

CTDE虽然强大，但并非银弹。这里列出几个值得关注的局限：

集中评论家的信息瓶颈： 在Agent数量较多时（比如超过50个Agent），集中评论家的输入维度（所有Agent的obs + actions）变得非常巨大，学习效率急剧下降。这就好比让一个人同时盯着50个棋手的每一步棋来评估局势——信息过载导致判断失准。

异构Agent的挑战： CTDE框架通常假设Agent是同构的（同样的观测空间和动作空间）。当Agent异构时（比如一个机器人有摄像头，另一个有激光雷达），集中评论家需要处理不同类型的信息输入，混合网络的设计变得更加复杂。

对模拟器的依赖： CTDE的训练阶段要求在可控环境中获取全局信息——这意味着需要模拟器。对于没有高保真模拟器的场景（如金融市场、社交网络），CTDE的可用性受限。这推动了"无模型MARL"和"离线MARL"等替代方案的发展。

扩展至开放世界： CTDE假设Agent数量在训练和部署时是固定的。但在开放世界中，Agent可以加入或离开系统——一个固定的CTDE架构无法优雅地处理这种动态变化。这推动了"图神经网络MARL"和"基于Transformer的价值分解"等新方向。

踩坑实录： 我在使用QMIX训练10个Agent的编队控制任务时遇到过一个奇怪的训练不稳定问题。训练初期效果很好，但大约100万步后Qtot的损失突然飙升，然后所有Agent策略崩溃。排查了很久才发现——问题出在单调性约束上。在编队任务中，当一个Agent偏离编队太远时，最好的策略不是"把它拉回来"而是"其他Agent调整编队来适应它"——这需要非单调的Qtot，而QMIX的单调性约束阻止了这种学习。解决方案：切换到QPLEX后，训练稳定性大幅提升。教训：理解算法的假设边界，才能在正确的场景选择正确的算法。

CTDE作为MAS训练的核心范式，一直是连接博弈论理论和工程实践的桥梁。它告诉我们：训练时的"上帝视角"和执行时的"有限视野"可以精巧地统一在同一个框架中，前提是设计好价值分解的结构。
