第14章 Sampling与生成控制

前几章都在讲怎么让模型跑得快。这一章换个角度——怎么让模型输出得对。采样策略决定了你的输出是"千篇一律"还是"精彩纷呈"，是一堆废话还是精准答案。

14.1 采样参数全解析

参数作用范围推荐值
temperature控制输出的随机性。越高越"放飞"，越低越"保守"0.0 - 2.0创意 0.8-1.0，严谨 0.0-0.3
top_p（核采样）只考虑累积概率达到 p 的 token 集合0.0 - 1.00.9
top_k只考虑概率最高的 k 个 token1 - vocab_size50
max_tokens最大生成 token 数（含 stop tokens）1 - model_max_len按业务设定
stop遇到这些字符串立即停字符串列表["\n\n", "Human:", "</s>"]
repetition_penalty惩罚重复 token，>1.0 减少重复0.0 - 2.01.1（需要时）
presence_penalty惩罚已出现过的 token-2.0 - 2.00.5（鼓励多样性时）
frequency_penalty按出现频率惩罚-2.0 - 2.00.5（减少重复时）

14.2 Temperature 的直观理解

temperature = 0.0 (Greedy, 贪婪解码):
  每一步选概率最高的 token → 输出完全确定
  "今天天气" → 模型每步都选最可能的 → 永远是同一个答案

  适用: 数学计算、代码生成、事实问答

temperature = 0.7 (温和创意):
  "今天天气" → 可能是"很好"、"不错"、"还行"、"晴朗"...
  有变化但不会离谱

  适用: 日常对话、内容总结

temperature = 1.5 (放任自由):
  "今天天气" → 可能是"很好"、"如您所愿"、"量子纠缠"、"打雷下冰雹"...
  极度随机，容易胡说

  适用: 创意写作（不推荐用于严谨场景）

14.3 结构化输出（Guided Decoding）

vLLM 支持强制输出特定格式（JSON、正则等），这对 API 服务至关重要：

from vllm import LLM, SamplingParams
from vllm.sampling_params import GuidedDecodingParams

# JSON 模式
guided = GuidedDecodingParams(json={
    "type": "object",
    "properties": {
        "name": {"type": "string"},
        "age": {"type": "integer"}
    }
})
sampling_params = SamplingParams(guided_decoding=guided)

# 正则模式
guided = GuidedDecodingParams(regex=r"答案: (A|B|C|D)")

# 基于上下文无关文法（CFG）
guided = GuidedDecodingParams(grammar='''
    root ::= answer
    answer ::= "是" | "否"
''')

outputs = llm.generate(prompts, sampling_params)
# 输出保证符合指定格式！

实战技巧：结构化输出不仅让下游代码解析方便，还能减少生成 token 数（模型不用自己组织格式了）。在批量数据提取场景中，结构化解码通常能减少 10-30% 的生成 token 数。

14.4 Logprobs：看模型"想了什么"

sampling_params = SamplingParams(
    temperature=0.7,
    max_tokens=10,
    logprobs=5  # 返回每个位置 top-5 的概率
)
output = llm.generate(["你好"], sampling_params)
# output.outputs[0].logprobs: 每个位置的 log 概率列表
# 适用于: 不确定性检测、黑盒评估、debug

14.5 不同场景的采样方案

场景推荐配置原因
数学/代码生成temperature=0.0, top_p=1.0确定性输出，greedy 最优
客服对话temperature=0.6, top_p=0.9, stop=["\n\n", "用户:"]有点变化但不离谱，防止自问自答
JSON 数据提取temperature=0.0 + GuidedDecoding(json_schema)格式保证 + 确定性内容
创意写作temperature=0.9, top_p=0.95, top_k=100多样性优先
翻译temperature=0.3, top_p=0.9忠实原文为主，少量变化空间
