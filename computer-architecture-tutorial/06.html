第6章 MIPS与x86指令集对比

同学们好！深入学习了RISC-V之后，今天我们把视野拉宽一些。RISC-V不是凭空产生的——它站在MIPS、DEC Alpha、ARM等RISC处理器的肩膀上。与此同时，世界上运行数量最多的指令集架构x86选择了完全不同的进化路线。

理解这些主流ISA的设计哲学和它们的历史轨迹，不仅能帮助你理解计算机生态的形成，更能让你在未来的架构设计中有更丰富的参照系。正如计算机体系结构领域的格言所说："那些不学习历史的人注定要重新发明它——而且往往发明得更差。"

6.1 MIPS：教科书的优雅

MIPS（Microprocessor without Interlocked Pipeline Stages）由斯坦福大学的John Hennessy团队于1985年设计。这个名字本身就揭示了设计哲学——"无互锁流水线级"意味着硬件不做流水线冲突检测，由编译器来保证正确性。这比后来的RISC-V更加"接近金属"。

MIPS的设计特点：

  • 32个寄存器（$0-$31），其中$0硬连接为0（RISC-V借鉴了这一点）。
  • 三种指令格式：R-type（寄存器）、I-type（立即数）、J-type（跳转）。
  • 经典的5级流水线（IF→ID→EX→MEM→WB）就是为MIPS量身定制的——每个流水级恰好完成一个操作。
  • 分支延迟槽（Branch Delay Slot）：分支指令之后的指令总是被执行（无论分支是否跳转）。这是早期MIPS没有硬件互锁的直接后果。

MIPS的优雅使其成为长达30年的教学标准ISA。几乎每一本计算机体系结构教科书（包括Patterson & Hennessy的经典教材）都使用MIPS作为示例。RISC-V从MIPS中借鉴了很多——32个寄存器、3操作数格式、load/store架构——但也改进了MIPS的不足：去掉了分支延迟槽（现代处理器不需要它，反而增加了复杂度），增加了模块化扩展机制。

不过MIPS的商业命运并不顺利。MIPS Technologies先后被SGI、ARM收购，如今MIPS已经转向RISC-V阵营——2021年MIPS宣布放弃自有ISA，全面拥抱RISC-V。这标志着一个时代的结束。

6.2 x86：兼容性的代价与胜利

如果说MIPS是规规矩矩的现代建筑，那么x86就是一座经历了40年不断扩建的历史城堡。它的每一块砖都承载着向后兼容的历史包袱。

进化历程：

  • 1978年：Intel 8086，16位处理器。4个通用寄存器（AX/BX/CX/DX），4个段寄存器（CS/DS/SS/ES），1字节指令。
  • 1985年：Intel 386，首次32位化。寄存器扩展为EAX/EBX/ECX/EDX，新增保护模式和分页机制。
  • 1995年：Pentium Pro，引入微操作（μop）翻译，从此x86处理器内部变成了RISC引擎。
  • 2003年：AMD64（x86-64），将寄存器扩展到64位（RAX/RBX/...），新增8个通用寄存器（R8-R15），废弃了大多数段寄存器的用途。
  • 至今：每年添加新指令集扩展——SSE、AVX、AVX-512、AMX、APX——每一次都增加了译码器的复杂度。

x86的指令长度从1字节到15字节不等，前缀字节可以叠加。一条指令包含可选的前缀（最多4个，包括LOCK、REP、段覆写、操作数大小覆写、地址大小覆写等）、1-3字节的操作码、ModRM字节、SIB字节、位移量和立即数。译码器需要逐字节检查，无法像RISC-V那样"一眼看穿"整条指令。

这种复杂度带来了两方面的代价：

  • 设计代价：x86的译码器消耗大量芯片面积和功耗。Intel的译码器被戏称为"芯片上的芯片"。
  • 胜利：没有其他ISA拥有x86那样庞大的软件生态。Windows、Linux、macOS上的绝大多数桌面和服务器软件都是为x86编译的。这种生态锁定是x86持续统治的终极护城河。

6.3 微码与指令翻译层

这里有一个很多人不知道的事实：现代x86处理器内部并不直接执行x86指令。它们把x86指令翻译成内部的RISC风格微操作（μops），然后在一个高性能的RISC核心上执行这些μops。

这个过程是这样的：

  1. 取指单元从内存中取出1-15字节的x86指令。
  2. 译码器（通常是硬件译码器 + 微码ROM的组合）将x86指令拆分为1到多个μops。简单的指令（如add eax, ebx）直接译码为1个μop；复杂的指令（如rep movs）可能需要几十个μops，存储在微码ROM中。
  3. μops被送入重命名和调度器，进入乱序执行引擎。

Intel的微操作缓存（DSB, Decoded Stream Buffer）可以缓存之前译码过的μops，避免了重复译码的热指令的性能损失。而AMD也有类似的技术（Op Cache）。

这个"CISC前端+RISC后端"的结构是当今x86处理器的标准架构。它完美地说明了ISA和微架构的区别——ISA是对外的"契约"，微架构是内部的实现。RISC-V处理器也可以用微码（尽管很少需要），而x86处理器也可以（并且在事实上）用RISC流水线来执行指令。

💡 设计洞察："x86的复杂指令背后，隐藏着一颗RISC的心。"——这是对现代x86处理器最准确的描述。x86能延续40年，不是因为它作为CISC有多优越，而是因为它有一个能吸收所有复杂性的硬件翻译层。这个翻译层的效率，决定了x86处理器的竞争力。

6.4 三架构全面对比

推出年份201019851978 / 2003
指令数量~40 (基础)~80 (基础)1000+
寄存器（通用）32（x0硬连0）32（$0硬连0）16
指令长度32位固定32位固定1-15字节可变
寻址模式3种3种10+种
条件码寄存器无无有（EFLAGS）
特权级别3级（U/S/M）2级（核心/用户）4级（ring 0-3）

扩展机制模块化扩展（M/A/F/D/C/V…）无模块化扩展指令集扩展（MMX/SSE/AVX）
授权开放标准授权（现转向RISC-V）Intel/AMD交叉授权
应用场景嵌入式到HPC嵌入式/教育桌面/服务器

为了直观地展示差异，让我们用三个版本的汇编实现同一个简单函数——计算数组元素之和：

// C版本：所有三个ISA实现的目标
// int sum_array(int *arr, int n) {
//     int sum = 0;
//     for (int i = 0; i < n; i++) sum += arr[i];
//     return sum;
// }

// RISC-V版本
sum_array:
        li    t0, 0            # sum = 0
        li    t1, 0            # i = 0
.loop:
        bge   t1, a1, .done
        slli  t2, t1, 2
        add   t2, a0, t2
        lw    t3, 0(t2)
        add   t0, t0, t3
        addi  t1, t1, 1
        j     .loop
.done:
        mv    a0, t0
        ret

// MIPS版本（含延迟槽）
sum_array:
        addi  $t0, $0, 0      # sum = 0
        addi  $t1, $0, 0      # i = 0
        addi  $t4, $0, 4      # 常量4
.loop:
        slt   $t2, $t1, $a1   # i < n ?
        beq   $t2, $0, .done
        nop                   # 延迟槽（硬件上先执行）
        mult  $t1, $t4        # i * 4
        mflo  $t2
        add   $t2, $a0, $t2   # &arr[i]
        lw    $t3, 0($t2)
        nop                   # 加载延迟槽（加载后不能立即使用）
        add   $t0, $t0, $t3
        addi  $t1, $t1, 1
        j     .loop
        nop                   # 延迟槽
.done:
        addi  $v0, $t0, 0
        jr    $ra
        nop                   # 延迟槽

// x86-64版本
sum_array:
        xor    eax, eax       # sum = 0 (清空rax)
        xor    ecx, ecx       # i = 0
.loop:
        cmp    ecx, esi       # i < n ?
        jge    .done
        movsxd rdx, ecx       # 符号扩展i到64位
        add    rdx, rcx       # 不，需要乘以4
        lea    rdx, [rdi + 4*rcx]  # &arr[i] (使用x86的基址+索引寻址)
        add    eax, [rdx]     # sum += arr[i]
        inc    ecx            # i++
        jmp    .loop
.done:
        ret

观察差异：MIPS需要手动处理加载和分支延迟槽，这是早期RISC设计的"遗迹"；x86-64可以用lea指令在一个时钟周期内完成地址计算（基址+索引*缩放），而RISC-V需要至少两条指令。但代价是x86的译码器需要识别复杂的LEA编码。

6.5 ARM：移动时代的王者出圈

虽然本章主要聚焦MIPS和x86，但不得不提ARM——它从一个侧面证明了RISC哲学的胜利。

ARM起源于1983年英国Acorn计算机公司的项目，最初叫做Acorn RISC Machine。早期ARM处理器（ARM2/ARM3）用于Acorn的桌面计算机，虽然性能不错，但并未引起世界轰动。

真正的转折发生在移动时代。ARM架构的低功耗特性使其成为手机处理器的首选——从1998年的ARM7TDMI到今天的ARM Cortex-X系列，ARM处理器统治了全球智能手机市场。而近年来，ARM开始向服务器（AWS Graviton）和笔记本电脑（Apple M系列）发起冲击。

ARM与RISC-V一个关键区别是商业模式：ARM采用授权模式（licensing），客户支付前期授权费和每芯片版税；RISC-V则是完全开放的，没有授权费用。这意味着RISC-V在小众市场、学术研究、定制芯片方面有独特的成本优势。

从ISA角度看，ARMv8（2011年）是一次重要的现代化——将指令长度统一为32位（A64），寄存器增加到31个，不再支持ARMv7的thumb指令和条件执行（但通过其他机制弥补）。有趣的是，ARMv8的很多设计与RISC-V不谋而合。这再次说明：好的设计往往会趋同。
