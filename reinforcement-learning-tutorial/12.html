第十二章：Actor-Critic方法

同学们好！上一章我们学习了策略梯度（Policy Gradient）方法，它直接优化策略参数，但有一个显著的缺点——方差太大。你想想，用一整条轨迹的累积回报作为动作好坏的评判标准，相当于一个学生考完期末才知道自己学得怎么样，但中间没有任何反馈。那有没有办法在每一步都给出一个评判信号呢？答案是：给策略网络配一个"教练"——这就是Actor-Critic方法。

12.1 Actor-Critic架构

Actor-Critic的名字很直白："Actor"是演员，负责做动作；"Critic"是评论家，负责评估动作的好坏。两者协同工作，形成一个闭环：Actor在环境中执行动作，Critic根据观察到的奖励和状态给出评价信号，Actor利用这个信号来调整自己的策略。

放在数学框架里看：

  • Actor（策略网络）：参数化为 πθ(a|s)，输入状态 s，输出动作的概率分布（离散）或均值/方差（连续）。
  • Critic（价值网络）：参数化为 Vφ(s) 或 Qφ(s,a)，输入状态（或状态-动作对），输出一个标量价值估计。

Actor的更新不再是像REINFORCE那样用整条轨迹的回报 Gt，而是用Critic提供的TD误差作为权重：

Actor更新：θ ← θ + α × δ × ∇₂ log πₑ(a|s)
Critic更新：φ ← φ + β × δ × ∇₂ Vₑ(s)

其中 δ = r + γ V(s') - V(s) 是 TD 误差

这里的关键改变是：Critic将高方差的蒙特卡洛回报 Gt 替换成了低方差的TD误差 δ。虽然TD误差是有偏的（因为V(s)是自己的估计），但偏差-方差的权衡在这笔交易中通常是划算的——方差大幅降低，收敛速度显著提升。

打个比方：如果把策略梯度比作一个自学跳舞的人，每跳完一整支舞才看录像回放找问题（方差高、反馈慢），那Actor-Critic就是请了一个舞蹈教练。教练（Critic）站在旁边，每做一个动作就给出评价——"这个抬手角度不对""这一步重心偏了"。有了实时反馈，你学舞的速度自然快得多。这就是Actor-Critic的"演员-评论家二重奏"。

直觉理解： 为什么Critic能降低方差？因为Critic提供的评价信号（TD误差）只依赖于单步的随机性，而REINFORCE中整条轨迹的回报累积了多步的随机性。方差与轨迹长度成正比——轨迹越长，方差爆炸得越厉害。Critic的价值函数相当于一个"平滑器"，把长程的随机性压缩到了单步的估计中。

12.2 优势函数（Advantage Function）

刚刚我们用V(s)做Critic，得到的更新权重是 δ = r + γV(s') - V(s)。这个 δ 实际上是对优势函数（Advantage Function）的无偏估计。我们先看定义：

优势函数： A(s,a) = Q(s,a) - V(s)

Q(s,a) 是在状态s下采取动作a能获得的期望回报，V(s) 是在状态s下按当前策略能获得的期望回报。两者的差值A(s,a)衡量的是一个动作相对于平均水平好多少。如果A(s,a) > 0，说明这个动作比平均水平好；如果A(s,a) < 0，说明这个动作比平均水平差。

为什么用优势函数而不是直接用Q(s,a)？原因在于方差。Q(s,a)本身包含了状态价值V(s)的波动，而A(s,a)去掉了这部分"基线"，只保留了动作本身的"优势"信息。这就像考试分数——老师说"这次考试平均分75分"，小明考了90分。用原始分评估，小明的分数受试卷难度影响（相当于状态价值波动），但用"高出平均分15分"来评估，就剔除了试卷难度这个因素。优势函数干的正是这事。

那么问题来了：直接计算A(s,a)需要同时知道Q(s,a)和V(s)——两个网络！这太麻烦了。好在有一个精巧的近似：

TD误差 δ = r + γV(s') - V(s) 是 A(s,a) 的无偏估计。

我们来验证一下。根据定义：A(s,a) = Q(s,a) - V(s) = E[r + γV(s') | s,a] - V(s)。而 δ = r + γV(s') - V(s) 的期望正是 E[δ | s,a] = E[r + γV(s') | s,a] - V(s) = Q(s,a) - V(s) = A(s,a)。所以 δ 的期望等于 A(s,a)——它是一个无偏估计。

关键结论： 在Actor-Critic中，我们不需要显式计算Q函数，只需要用一个V网络，然后计算TD误差 δ = r + γV(s') - V(s)，它就是一个天然的优势函数估计值。这就是"用Critic来估计优势"的实操方法。

当然，单步TD误差虽然无偏，但方差可能受环境单步随机性的影响。在下一节我们会看到更高级的优势估计方法——GAE，它在偏差和方差之间做了更精细的平衡。

12.3 A2C / A3C算法

有了Actor-Critic架构和优势函数，我们就可以搭建更实用的算法了。最具代表性的是A2C（Advantage Actor-Critic）和A3C（Asynchronous Advantage Actor-Critic），它们由DeepMind在2016年提出，在深度学习框架下真正让Actor-Critic方法在大规模问题上发挥了威力。

A2C（同步优势Actor-Critic）：

A2C的流程很直接：维护一个全局的Actor网络和Critic网络，同时启动多个并行的worker（每个worker有自己的环境副本）。在每个训练步中，所有worker各自采集一条轨迹，计算各自的优势估计和梯度，然后同步地将梯度汇总到全局网络，更新参数。关键点是同步——所有worker都计算完了再统一更新。

A3C（异步优势Actor-Critic）：

A3C与A2C的核心区别在于"异步"二字。每个worker独立地与全局网络交互：worker从全局网络拉取最新参数，在自己的环境中跑几步，计算梯度，然后异步地推送到全局网络。没有同步等待，也没有梯度汇总——每个worker各推各的。

A3C在提出时是一个大突破，因为它解决了深度RL中一个关键问题：样本相关性。DQN用经验回放池（replay buffer）来打破样本间的相关性，但A3C用了一种完全不同的方式——多个并行worker各跑各的环境，采集到的经验天然就是去相关的。这意味着A3C不需要replay buffer，内存开销更小，而且支持在线学习。

A3C的历史地位： 在PPO出现之前（2017年），A3C被认为是state-of-the-art的RL算法。它训练速度快（多worker并行）、样本效率高（不需要replay buffer）、实现相对简洁。直到PPO用更稳定的裁剪目标函数取代了A3C的复杂异步机制，A3C才逐渐淡出主流。但A3C"多worker并行去相关"的思想至今仍在使用。

下面是A2C算法的核心伪代码实现：

算法：A2C（同步优势Actor-Critic）

输入：学习率 α、折扣因子 γ、步数 T、worker 数 N

初始化 Actor 网络参数 θ 和 Critic 网络参数 φ
for 每个训练迭代 do:
    重置所有 N 个 worker 的状态
    for 步 t = 1 到 T do:
        每个 worker i:
            a_i ~ πₑ(s_i)              // Actor 采样动作
            执行 a_i, 得到 r_i, s'_i
            计算 δ_i = r_i + γVₑ(s'_i) - Vₑ(s_i)  // Critic 计算 TD 误差
            累积 Actor 梯度: gₑ += δ_i ∇ log πₑ(a_i|s_i)
            累积 Critic 梯度: gₑ += δ_i ∇ Vₑ(s_i)
    // 同步更新
    θ ← θ + α · gₑ
    φ ← φ + α · gₑ
返回 最优策略参数 θ

12.4 GAE（广义优势估计）

前面我们用的优势估计是单步TD误差 δ = r + γV(s') - V(s)。单步TD误差方差低但可能有偏——如果V(s')的估计不准，偏差就会累积。那么有没有一种方法，可以灵活地在"偏差大但方差小"和"方差大但偏差小"之间调节？有的——GAE（Generalized Advantage Estimation），由John Schulman在2015年提出。

先考虑多步优势估计。n步优势估计可以写成：

  • 1步优势：𝜀(1) = δt = rt + γV(st+1) - V(st)
  • 2步优势：𝜀(2) = δt + γδt+1 = rt + γrt+1 + γ²V(st+2) - V(st)
  • k步优势：𝜀(k) = Σl=0k-1 γlδt+l
  • 无穷步优势（蒙特卡洛）：𝜀(∞) = Σl=0∞ γlδt+l = Gt - V(st)

k越小，依赖V(s)的自举越多，方差越低但偏差越大；k越大，依赖实际奖励越多，方差越高但偏差越小。GAE的核心思想是：对所有k步优势进行指数加权平均，用一个参数 λ 来调节权重：

GAE(λ)： 𝜀tGAE(λ) = (1-λ) Σk=1∞ λk-1 𝜀(k)

这个式子展开后可以化简成一个极其优雅的形式：

𝜀tGAE(λ) = Σl=0∞ (γλ)l δt+l

也就是说，GAE就是对一串TD误差进行指数衰减的加权求和！这个形式非常容易递归计算，实际实现中只需要前向算一步存一个变量：

GAE 递归实现（从后往前）：

初始化 g = 0
for t = T-1 到 0:
    δ_t = r_t + γ * V(s_{t+1}) - V(s_t)
    g = δ_t + γ * λ * g
    A_t = g

这个递归公式的含义是：从轨迹的末尾开始反向传播优势估计。越靠近末尾的数据，贡献越直接；越靠近开头的数据，贡献受 (γλ)l 的衰减影响越大。

λ 的选择：

  • λ = 0：退化为单步TD优势，方差低、偏差高（如果V有偏）
  • λ = 1：退化为蒙特卡洛优势 Gt - V(st)，方差高、偏差低
  • λ = 0.95：最常用的默认值，在偏差和方差之间取得良好的平衡
  • λ = 0.97~0.99：用于奖励信号较稀疏或需要长程依赖的任务
踩坑故事：GAE实现中的一个隐秘bug

我有一个学生在实现GAE时，代码几乎逐字照搬了上面的递归公式，但训练就是不收敛——Loss下降极慢，策略几乎不学习。他调了三天学习率、网络宽度、λ值，毫无效果。

最后一行一行debug，发现问题出在一个超级隐蔽的地方：他在计算 δ = r + γ * V(s') - V(s) 时，用的是同一个V网络的输出。这本没错，但他在同一个batch的前向传播中，同时计算了 V(s) 和 V(s')，然后在 δ 的计算中 错误地将 V(s') 对应到了下一个样本的 V(s)——也就是说，他对齐搞错了！在GAE的递归计算中，如果轨迹的最后一个状态 V(s_T) 没有正确处理（比如忽略了terminal状态的V=0），或者backward遍历时索引偏移了一位，整个优势估计序列就会全部偏移，导致策略更新完全错误。

教训： 实现GAE时，一定画张图标清楚每个t对应的s_t、s_{t+1}的索引对应关系。特别是处理terminal状态时，确保 V(terminal) = 0。建议先写一个简单的check：在已知价值的玩具环境中（比如GridWorld），打印出每一步的 δ 和 A_t，和手动计算结果对比一下。花这10分钟能省你3天debug时间。

GAE为什么重要？ 因为它是现代深度RL算法的标配。PPO、TRPO、SAC等主流算法都使用GAE作为优势估计器。它用一个参数 λ 优雅地统一了TD学习和蒙特卡洛方法。你在实现PPO的时候会发现，核心代码很大一部分就是GAE的计算——它直接决定了策略更新的质量。

12.5 Actor-Critic vs 策略梯度 vs 值函数方法

学到现在，我们经历了三种主流RL方法：值函数方法（DQN族）、策略梯度方法（REINFORCE）、以及Actor-Critic方法（本章）。什么时候该用哪种？我们来一个全面对比：

对比维度 值函数方法（DQN） 策略梯度（REINFORCE） Actor-Critic（A2C）
学习对象 Q(s,a) 或 V(s) 直接优化策略 π(a|s) 同时学策略和价值函数
动作空间 离散（DQN需要argmax） 离散和连续都适用 离散和连续都适用
更新方式 异策略（可用replay buffer） 同策略（需当前策略采样） 同策略（需当前策略采样）
方差 低（自举，但可能有偏差） 高（整条轨迹累积回报） 中等（Critic提供低方差信号）
偏差 高（max操作引入过估计偏差） 低（无偏估计） 中等（Critic的V估计引入偏差）
样本效率 高（replay buffer复用数据） 低（用完即弃） 中等（可用多worker提高）
训练稳定性 需要target network等技巧 更新步长敏感 比REINFORCE稳定，但仍需调参
实现复杂度 中等 简单 中等偏上
代表性算法 DQN, Double DQN, Rainbow REINFORCE A2C, A3C, PPO, SAC

选择建议：

  • 如果你的任务是离散动作、状态空间不太大：DQN是稳妥的选择，实现简单、样本效率高。
  • 如果你需要处理连续动作空间：值函数方法用不了（没法对连续动作求argmax），直接上Actor-Critic或其进阶版（PPO、SAC）。
  • 如果你在做机器人控制这类任务：Actor-Critic系方法（PPO/SAC）几乎是标配，它们能处理连续控制，且对超参数相对鲁棒。
  • 如果你只是想做快速原型验证：REINFORCE实现最简单，但你要忍受它的高方差和慢收敛。
一个重要的观察： 现在业界和学术界的主流，几乎全是Actor-Critic系算法。纯值函数方法（DQN）只在特定离散场景下还有竞争力，纯策略梯度（REINFORCE）基本上已经被淘汰。所以，掌握了Actor-Critic的思想，你就拿到了通往现代深度强化学习大门的钥匙。

本章小结

  • Actor-Critic架构将策略网络（Actor）和价值网络（Critic）结合使用，Actor负责决策，Critic负责评价，两者通过TD误差协同更新。
  • 优势函数A(s,a) = Q(s,a) - V(s) 衡量动作相对平均水平的优劣，使用优势可以降低策略梯度的方差。TD误差 δ 是优势的无偏估计。
  • A2C是同步多worker的Actor-Critic实现，各worker独立采集经验后同步更新；A3C采用异步推送梯度，无需replay buffer即可打破样本相关性。
  • GAE(λ)对多步TD误差进行指数加权求和，通过 λ 参数在偏差和方差之间灵活调节。λ=0.95是常见默认值。
  • Actor-Critic方法在偏差、方差、样本效率之间取得了更好的平衡，是现代深度强化学习的基石方法。

同学们，学完本章，你们已经理解了深度强化学习中最核心的算法框架。下一章我们将学习PPO（Proximal Policy Optimization）——它可以说是目前应用最广泛的强化学习算法，没有之一。PPO直接在Actor-Critic的框架下解决了"更新步长到底该迈多大"这个关键问题，用一个裁剪的替代目标函数让策略更新既快又稳。可以说，PPO是Actor-Critic思想的集大成者，也是你开始动手做RL项目的最佳起点。咱们下章见！
