12.1 CNN过时了吗？

2021年，CV领域最热门的话题是Vision Transformer。Swin Transformer在ImageNet上达到了新的SOTA，并且横扫了检测和分割任务的榜单。与此同时，CNN阵营似乎陷入了沉寂——ResNet已经是五年前的架构了，EfficientNet虽然在精度上还有竞争力，但其NAS搜索得到的架构过于复杂，难以分析和改进。

这时，CV圈子里出现了一个尖锐的问题：CNN是不是过时了？从论文发表数量看，2021年顶会中Transformer相关论文的比例从不到5%猛增到了超过40%。不少研究者认为，卷积操作终究会被注意力机制完全取代。

2022年，Meta AI（原Facebook AI Research）发表了一篇引起广泛关注的论文——"A ConvNet for the 2020s"，也就是我们说的ConvNeXt。这篇论文的核心叙事其实是一个精彩的"思想实验"：如果我们把ResNet按照Swin Transformer的设计思路一步步"现代化"，它的性能能否追上Swin？答案是肯定的——经过一系列改造后的ConvNeXt，在ImageNet上达到了与Swin Transformer高度接近甚至略微超越的性能。

ConvNeXt的意义不在于提出多么新颖的操作——实际上它的每个改动都是"已知的"——而在于它用严格的消融实验告诉我们：CNN的性能差距不是因为卷积本身不好，而是因为CNN的工程实践（归一化、激活函数、下采样方式等）没有跟上时代的进步。

12.2 从ResNet到ConvNeXt的7步改造

ConvNeXt从一个标准的ResNet-50出发，逐步应用了7项改造策略。每一步都严格记录了性能变化。下面我们逐一来看。

Step 1: Macro Design——调整各阶段的计算比例

ResNet-50的四个阶段（stage）的块数分布是[3, 4, 6, 3]。Swin Transformer的分布是[1, 1, 9, 1]（第一阶段1个block，第三阶段9个block）。ConvNeXt将ResNet的分布调整为[3, 3, 9, 3]，让计算量向分辨率适中的第三阶段集中——这是因为更深的层有更大的感受野和更丰富的语义信息。

效果：准确率从78.8%提升到79.4%。

Step 2: Patchify Stem——改变下采样方式

ResNet使用7×7卷积+最大池化作为stem（输入层），这在大核卷积后接池化的方式在今天看来效率不高。ConvNeXt借鉴了Swin的做法：使用一个4×4卷积（stride=4）作为stem，直接将输入图像下采样到1/4分辨率。这相当于ViT中的patch embedding——而且更简单。

# ResNet Stem（传统方式）
nn.Sequential(
    nn.Conv2d(3, 64, kernel_size=7, stride=2, padding=3),  # 7×7大核
    nn.BatchNorm2d(64),
    nn.ReLU(),
    nn.MaxPool2d(kernel_size=3, stride=2, padding=1)       # 额外池化层
)
# 输出分辨率: 输入的 1/4

# ConvNeXt Stem（Patchify方式）
nn.Sequential(
    nn.Conv2d(3, 96, kernel_size=4, stride=4),              # 4×4卷积
    nn.LayerNorm(96, eps=1e-6)                              # LayerNorm代替BN
)
# 输出分辨率: 输入的 1/4（一步到位）

效果：准确率从79.4%提升到79.5%。

Step 3: LayerNorm代替BatchNorm

这是最有争议也最有启发性的一步。ConvNeXt用Layer Normalization替代了CNN中使用了近十年的Batch Normalization。原因很简单：Transformer都用LN，而且LN在训练和推理时行为是一致的（不像BN依赖mini-batch统计量），在分布式训练中也更稳定。

ConvNeXt在每个卷积层之前使用LN（pre-normalization），与Transformer的做法一致。

效果：准确率从79.5%提升到79.7%。

Step 4: GELU激活函数

ConvNeXt将ReLU替换为Transformer中使用的GELU（Gaussian Error Linear Unit）。GELU与ReLU类似，但在负值区域有平滑的梯度（而不是完全的零），这有助于训练稳定性。

效果：准确率从79.7%提升到80.0%。

Step 5: 大核卷积——7×7的回归

这是最反直觉的一步。VGG教导我们3×3是最好的，但ConvNeXt发现增大卷积核到7×7能显著提升性能。这背后的逻辑是：既然Transformer的每一层都有全局感受野，CNN也应该在合理的计算开销内尽可能扩大感受野。

为了控制计算开销，ConvNeXt将7×7深度可分离卷积（depthwise convolution）放置在MLP层的"两头"——类似于Transformer中注意力机制放在前馈网络前面的位置。

# ConvNeXt Block 的核心结构
# 输入: [B, C, H, W]
#    ↓
# LayerNorm (Pre-Norm)
#    ↓
# Depthwise Conv2d(7×7)   ← 深度可分离卷积，单个7×7核（每个通道独立）
#    ↓
# LayerNorm (Pre-Norm)
#    ↓
# Conv2d(1×1, C→4C) + GELU    ← 升维 (MLP的第一层)
#    ↓
# Conv2d(1×1, 4C→C)           ← 降维 (MLP的第二层)
#    ↓
# + (残差连接，从输入加到输出)
#    ↓
# 输出: [B, C, H, W]

# 与Swin Transformer Block的类比：
# Swin: LN → Window Attention → LN → MLP
# ConvNeXt: LN → Depthwise Conv (7×7) → LN → MLP
# 两者的结构高度相似！区别只是"全局交互"的方式：
# Swin用窗口自注意力，ConvNeXt用大核深度可分离卷积

效果：准确率从80.0%提升到80.6%。

Step 6: 下采样层改为卷积

ResNet使用stride=2的卷积层进行下采样。ConvNeXt改用独立的2×2卷积（stride=2）来执行下采样，并在之前加上LN。这个设计更接近Swin的patch merging操作。

效果：准确率从80.6%提升到80.9%。

Step 7: 更少的激活函数和归一化层

ConvNeXt在每个block中只保留一个激活函数（GELU放在MLP中的第一个1×1卷积之后），只保留一个LN（在block开始时）。相比之下，ResNet的BasicBlock中每个卷积后面都有BN和ReLU。

效果：准确率从80.9%提升到81.1%。

🔬 改造过程总结：逐项来看，每项改造的提升都在0.2-0.6个百分点之间，单独看都不起眼。但7项改造叠加起来，将ResNet-50从78.8%推到了81.1%，超越了Swin-Tiny的81.2%。这个结果告诉我们：好的架构设计不是靠一个"银弹"创新，而是靠大量小改进的累积。这也正是消融实验的价值所在——让我们理解每个设计决策的真实贡献。

12.4 ConvNeXt的启示：CNN与Transformer正在趋同

ConvNeXt最重要的贡献可能不是架构本身，而是它揭示了一个更宏大的趋势：CNN和Transformer架构正在趋同。从ConvNeXt的结构可以清晰地看到：

  • ConvNeXt Block ≈ Swin Transformer Block：两者的结构完全一致——都是"Pre-Norm → 空间交互模块 → Pre-Norm → MLP → 残差连接"。唯一的区别是"空间交互模块"：Swin用窗口自注意力，ConvNeXt用大核深度可分离卷积。
  • Patchify Stem = Patch Embedding：ConvNeXt的stem和ViT/Swin的patch embedding几乎一样。
  • LayerNorm + GELU：ConvNeXt采用了Transformer阵营的标准组件。
  • 深度可分离卷积 ≈ 多头自注意力：直观上，7×7 depthwise conv是在一个7×7的局部窗口内做"均匀加权求和"（权重由卷积核参数决定），而自注意力是在全局范围内做"自适应加权求和"（权重由QK相似度决定）。两者都是"加权池化聚合邻居信息"的操作，只是权重生成方式不同。

这种趋同意味着：未来的视觉backbone可能不再有明显的"CNN vs Transformer"的分界线，而是一个统一的模块化框架——你可以选择用卷积或者注意力作为空间交互模块，其他组件几乎一样。这种思路在后续的UniRep（统一表示学习）、ConvNeXt V2、以及混合架构中得到了延续。

💡 实践建议：截至2024年，在实际项目中选择backbone时，不必执着于"这是CNN还是Transformer"的标签。ConvNeXt和Swin在ImageNet上的精度几乎相同，在检测/分割任务上的表现也不分伯仲。选择时更多应该考虑：团队对哪种框架更熟悉、推理框架对哪种操作优化得更好（NVIDIA TensorRT对卷积的优化非常成熟，对注意力机制的优化则因模型而异）、以及具体的部署约束。务实比"站队"更重要。

12.5 ConvNeXt V2：进一步的改进

2023年1月，Meta AI发布了ConvNeXt V2，将改进延伸到了自监督学习领域。主要贡献有两点：

  • FCMAE（Fully Convolutional Masked Autoencoder）：将MAE（掩码自编码器）方法适配到ConvNeXt上。传统观点认为MAE只适合ViT，因为ViT处理的是离散的patch token，天然适合做掩码预测。ConvNeXt V2证明了全卷积网络也能做掩码自编码——通过设计一个简单的卷积解码器来重建被掩码的图像区域。
  • GRN（Global Response Normalization，全局响应归一化）：ConvNeXt V2发现，在自监督训练过程中，特征图的通道之间会出现"坍缩"（某些通道的激活值远大于其他通道，导致信息丢失）。GRN通过计算每个通道的全局响应（L2范数），对通道进行归一化和特征竞争，有效缓解了这个问题。
# GRN（全局响应归一化）的数学形式
# 输入: X ∈ [B, C, H, W]
#
# 1. 计算每个通道的全局响应（L2范数的空间聚合）：
#    G_x = ||X_i||_2  (i=1,...,C)  →  shape [B, C]
#
# 2. 通道间的归一化（softmax形式的竞争）：
#    N_x = G_x / (sum_j G_x_j + ε)  →  shape [B, C]
#
# 3. 原始特征 × 归一化后的门控值：
#    X_out = X * (1 + γ * N_x)
#
# 效果：激活值强的通道会被"抑制"，弱通道会被"增强"
# 类似注意力机制中的"通道注意力"

ConvNeXt V2在ImageNet上的自监督预训练结果与MAE（ViT）非常接近，证明了CNN在自监督学习中同样有竞争力。这对工业界来说是个好消息——不需要从CNN迁移到ViT就能享受自监督预训练的好处。

⚠️ 踩坑故事：我复现ConvNeXt时遇到过一个很隐蔽的问题——默认的PyTorch depthwise convolution实现没有optimized kernel，7×7的depthwise conv跑起来比3×3的普通卷积还慢。后来发现需要安装NVIDIA的Apex库或者使用PyTorch 2.0的torch.compile来获得加速。这个教训是：论文报告的速度往往是经过深度优化的，在实际项目中复现时，别忘了检查底层算子的实现效率。

ConvNeXt的故事告诉我们，在深度学习中，历史不是简单的线性进步——有时候，"回头看"的视角同样有价值。接下来，我们将暂时告别架构设计的讨论，进入一个更加实用的主题：训练策略与调参实战。因为无论你选择了AlexNet还是ConvNeXt，如果不掌握正确的训练方法，再好的架构也发挥不出潜力。我们下一章见！
