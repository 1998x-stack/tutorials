第7章 函数逼近基本理论

大家好，我是你们的讲师。今天我们做一个重要的视角切换。

回顾前几章，我们一直在做同一件事：给定一组离散点，找一个多项式穿过它们。这是插值（interpolation）问题。但现实世界还有一种更常见的情形：你的数据点本身有噪声（比如传感器读数），你并不想"完美穿过"每个点——那会把噪声也拟合进去。你想找一个在整体上尽可能接近数据趋势的简单函数。这就是逼近（approximation）问题。

🔍 插值 vs 逼近： 插值是"穿过每一个点"，逼近是"整体上尽可能接近"。如果数据是精确的（比如天文星历表），用插值；如果数据有噪声（比如股票价格），用逼近。两者的数学工具高度重叠，但哲学立场完全不同。

这一章将建立逼近理论的基本框架。我们会在函数空间中定义"距离"（范数），然后讨论"最佳"逼近的存在唯一性。最后，我们将认识一类极其重要的函数族——正交多项式，它们是后续最佳平方逼近和Gauss求积公式的数学基础。

7.1 函数空间与范数

要想讨论"逼近得好不好"，首先得定义"好"的标准。在函数空间中，我们用范数（norm）来度量函数之间的"距离"。

最常见的三种范数：

  • L∞ 范数（Chebyshev范数/一致范数）： ||f||∞ = max|f(x)| over [a, b]。对应的是"最坏情况误差"。
  • L₂ 范数（最小二乘范数）： ||f||₂ = (∫|f(x)|² dx)¹/²。对应的是"整体均方误差"。
  • L₁ 范数： ||f||₁ = ∫|f(x)| dx。对离群值更鲁棒。

选择不同的范数，会得到不同的"最佳逼近"：

范数 度量的是什么 对离群值的敏感度 典型应用
L∞ (Chebyshev) 最大绝对值误差 极高 控制系统、安全关键系统
L₂ (最小二乘) 均方根误差 中等 数据处理、回归分析
L₁ 绝对误差之和 低 鲁棒统计、稀疏恢复

想象一下用这三种不同的"尺子"来逼近一个带尖峰的函数：L∞ 会努力控制尖峰附近的误差（不惜牺牲其他地方），L₂ 会在全局范围内分摊误差，L₁ 则对尖峰几乎"无视"。这不是谁对谁错的问题——而是你的应用需要哪种保证。

7.2 最佳逼近的存在唯一性

有了范数之后，我们可以严格定义最佳逼近问题：给定函数 f ∈ C[a, b] 和一个 n 维子空间 Φ ⊂ C[a, b]（比如次数 ≤ n 的多项式空间），求 φ* ∈ Φ 使得：

||f − φ*|| = min_{φ∈Φ} ||f − φ||

存在性定理：如果 Φ 是有限维子空间，那么对任何 f ∈ C[a, b]，最佳逼近一定存在。

唯一性定理（Chebyshev逼近）：在 L∞ 范数下，如果 Φ 满足Haar条件（即 Φ 中任何非零函数至多有 n-1 个不同的零点），则最佳逼近是唯一的。多项式空间满足Haar条件，所以最佳一致逼近多项式唯一存在。

💡 Haar条件直观理解： 就是要求逼近函数类的"表达能力"不能太差。比如在区间上用多项式逼近时，一个 n 次多项式最多有 n 个零点——这恰好保证了唯一性。如果允许的函数类过于灵活（比如允许任意振荡），最佳逼近就可能不唯一。

7.3 正交多项式族

现在让我们进入逼近论中最优雅的部分——正交多项式。

在函数空间上定义内积：

⟨f, g⟩ = ∫ w(x) f(x) g(x) dx

其中 w(x) ≥ 0 是权函数（weight function）。两个函数正交当且仅当 ⟨f, g⟩ = 0。

正交多项式族 {φ₀, φ₁, φ₂, ...} 满足：

  • φₙ 是 n 次多项式；
  • ⟨φₙ, φₘ⟩ = 0 当 n ≠ m。

所有正交多项式都满足一个三项递推关系（three-term recurrence）：

φₙ₊₁(x) = (x − αₙ) φₙ(x) − βₙ φₙ₋₁(x)

这个递推关系的重要性无论怎么强调都不过分——它意味着我们可以用 O(n) 的时间计算任意阶的正交多项式在任意点的值。

🔍 为什么正交多项式如此重要？ 有三个原因。第一，它们提供了函数逼近的最佳基底——在L₂意义下，最佳逼近的系数就是内积 ⟨f, φₙ⟩/⟨φₙ, φₙ⟩，计算极其简单。第二，它们的零点是最优插值节点（Chebyshev节点就是Chebyshev多项式的零点，能最小化Runge现象）。第三，Gauss求积公式的节点就是正交多项式的零点——把数值积分精度翻倍。

Legendre多项式：定义在 [-1, 1] 上，权函数 w(x) = 1。

  • 递推：P₀ = 1, P₁ = x, (n+1)Pₙ₊₁ = (2n+1)xPₙ − nPₙ₋₁
  • 正交性：∫₋₁¹ Pₙ(x)Pₘ(x) dx = 0 (n≠m), = 2/(2n+1) (n=m)
  • 零点：全部在 (-1, 1) 内，关于原点对称

Chebyshev多项式（第一类）：定义在 [-1, 1] 上，权函数 w(x) = 1/√(1−x²)。

  • 显式形式：Tₙ(x) = cos(n·arccos x)
  • 递推：T₀ = 1, T₁ = x, Tₙ₊₁ = 2xTₙ − Tₙ₋₁
  • 正交性：∫₋₁¹ Tₙ(x)Tₘ(x)/√(1−x²) dx = 0 (n≠m), = π/2 (n=m≠0), = π (n=m=0)
  • 零点：xₖ = cos((2k−1)π/(2n))，k = 1, ..., n
import numpy as np

def legendre(n, x):
    """用三项递推计算n阶Legendre多项式在x处的值"""
    if n == 0:
        return np.ones_like(x)
    elif n == 1:
        return x
    P0 = np.ones_like(x)
    P1 = x
    for k in range(2, n + 1):
        P2 = ((2*k - 1) * x * P1 - (k - 1) * P0) / k
        P0, P1 = P1, P2
    return P1

def chebyshev(n, x):
    """用三项递推计算n阶Chebyshev多项式在x处的值"""
    if n == 0:
        return np.ones_like(x)
    elif n == 1:
        return x
    T0 = np.ones_like(x)
    T1 = x
    for _ in range(2, n + 1):
        T2 = 2 * x * T1 - T0
        T0, T1 = T1, T2
    return T1

# 验证正交性
N = 1000
xs = np.linspace(-1, 1, N)
w_cheb = 1.0 / np.sqrt(1 - xs**2)
dx = 2.0 / N

inner_products = []
for n in range(6):
    for m in range(6):
        Tn = chebyshev(n, xs)
        Tm = chebyshev(m, xs)
        ip = np.sum(Tn * Tm * w_cheb) * dx
        if n == m:
            inner_products.append(f"⟨T{n}, T{n}⟩ ≈ {ip:.4f}")
print("Chebyshev正交性验证：")
for s in inner_products:
    print(f"  {s}")

7.4 Chebyshev多项式与最佳一致逼近

Chebyshev多项式有一个极其重要的极值性质：在所有首项系数为1的 n 次多项式中，Tₙ(x)/2ⁿ⁻¹ 在 [-1, 1] 上的 L∞ 范数最小。

换句话说，max|Tₙ(x)/2ⁿ⁻¹| = 1/2ⁿ⁻¹，且这个最大值是所有首一n次多项式中最小的。这就是最小零偏差（minimax）性质。

这个性质直接导致了Chebyshev交替定理（Alternation Theorem）：f 的最佳一致逼近多项式 p* 在 [-1, 1] 上至少有 n+2 个交替点，在这些点上 f − p* 交替取到最大绝对值 ±||f − p*||∞。

⚠️ 踩坑故事： 某信号处理团队使用等距节点做多项式逼近，在频带边缘总是出现剧烈振荡。他们以为是物理现象，花了一个月调试硬件滤波器。后来一位新来的工程师发现——这是Runge现象的变体，使用Chebyshev节点后振荡完全消失。这就是Chebyshev多项式的最小零偏差性质在实际中最重要的应用：Chebyshev节点 = cos((2k−1)π/(2n)) 是最优插值节点，它能让插值余项在整个区间上均匀分布，而不是集中在端点。

Chebyshev节点的公式为：

xₖ = cos((2k−1)π / (2n)), k = 1, 2, ..., n

注意这些节点在端点附近更密集，在中间更稀疏——正好对抗了Runge现象中误差在端点处放大的趋势。

🔍 Chebyshev节点 vs 等距节点： 用等距节点做 n=20 次多项式插值 Runge 函数，端点误差 ≈ 10⁷；用Chebyshev节点做同样的插值，误差 ≈ 10⁻⁶。这就是13个数量级的差距——仅仅因为选择了不同的节点分布。在数值分析中，节点的选择比多项式的次数更重要。 
import numpy as np

def runge(x):
    return 1.0 / (1.0 + 25.0 * x**2)

# 比较Chebyshev节点和等距节点
n = 15

# 等距节点
x_eq = np.linspace(-1, 1, n)
# Chebyshev节点
x_cheb = -np.cos((2*np.arange(1, n+1) - 1) * np.pi / (2 * n))

# 用两种节点分别做插值，在端点附近测试
x_test = 0.98

# 简单的Lagrange插值
def lagrange_val(xp, yp, x):
    result = 0.0
    for i in range(len(xp)):
        term = yp[i]
        for j in range(len(xp)):
            if i != j:
                term *= (x - xp[j]) / (xp[i] - xp[j])
        result += term
    return result

true_val = runge(x_test)
err_eq = abs(true_val - lagrange_val(x_eq, runge(x_eq), x_test))
err_cheb = abs(true_val - lagrange_val(x_cheb, runge(x_cheb), x_test))

print(f"n={n}, x={x_test}")
print(f"等距节点误差:     {err_eq:.2e}")
print(f"Chebyshev节点误差: {err_cheb:.2e}")
print(f"改进倍数:         {err_eq/err_cheb:.0f}x")
💡 知识延伸： 正交多项式的三项递推公式不仅用于计算多项式值，还用于Gauss求积公式的节点和权重的计算——Golub-Welsch算法正是利用三项递推构造一个三对角矩阵，其特征值就是Gauss求积的节点。我们在第12和14章会详细讨论。

这一章我们从"插值"切换到了"逼近"的视角。你学到了用范数来度量逼近的好坏，知道了正交多项式是逼近计算的利器，特别是Chebyshev多项式的最小零偏差性质让它成为最佳一致逼近的核心工具。

下一章我们将进入最佳平方逼近的具体计算——用L₂范数，配合正交多项式基底，把一个函数的逼近问题简化为一系列内积的计算。你会发现，有了正交多项式这个工具，复杂的逼近问题变得像做算术题一样简单。
