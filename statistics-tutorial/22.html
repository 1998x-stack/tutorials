第22章 贝叶斯统计入门

大家好，我是你们的讲师。走到这一章，我们已经学过了 p 值、置信区间、假设检验——这些都属于频率学派（Frequentist）统计的范畴。但统计学还有另一大流派，近年来在 AI 和机器学习领域全面翻红：贝叶斯统计（Bayesian Statistics）。它的核心思想非常直观：在观测到数据之前，我们对某件事有一个"先验信念"；看到数据后，我们用数据更新这个信念，得到"后验信念"。这个"学习-更新"的循环，和人类认识世界的方式惊人地一致。这一章我们不做深度推导，重在理解贝叶斯思维的哲学和方法。

22.1 贝叶斯定理回顾：P(A|B) = P(B|A)P(A)/P(B)

贝叶斯统计的数学基础是一个简洁的概率公式：

P(A|B) = P(B|A) × P(A) / P(B)

换到统计学的语境中：

  • P(θ) 是先验概率（Prior）：在看到数据之前，我们认为参数 θ 取某个值的概率。
  • P(D|θ) 是似然（Likelihood）：如果参数 θ 是真的，我们观测到当前数据 D 的可能性。
  • P(θ|D) 是后验概率（Posterior）：看到数据后，我们对参数 θ 的更新信念。
  • P(D) 是边际似然（Marginal Likelihood），充当归一化常数。
一句话总结：后验 = 先验 × 似然 / 归一化常数。或者更通俗地：更新后的信念 = 原有信念 × 数据的支持程度。

22.2 先验、似然、后验：贝叶斯更新的三部曲

假设你有一个硬币，想知道它是否公平。贝叶斯方法的做法是：

  1. 设定先验：在没有抛硬币之前，你认为硬币有 99% 的概率是公平的（p ≈ 0.5）——这是你的先验信念。
  2. 收集数据：你抛了 100 次，得到 60 次正面。计算似然——如果 p=0.5，看到 60/100 的概率；如果 p=0.6，看到 60/100 的概率。
  3. 更新后验：结合先验和似然，得到关于 p 的后验分布。后验分布是一个完整的分布，而不是一个单一的点估计——这意味着我们可以说"p 有 95% 的概率落在 0.52 到 0.68 之间"。

频率学派则会说："如果 p 真的等于 0.5，那么看到 60/100 或更极端结果的概率（即 p 值）是 X。如果 X < 0.05，我们拒绝 p=0.5 的假设。"请注意这里的微妙区别：贝叶斯在谈论的是"参数的概率"（p 在某个区间内的概率），而频率学派谈论的是"数据的概率"（在参数为真时看到这些数据的概率）。

22.3 频率学派 vs 贝叶斯学派：两种世界观

维度频率学派贝叶斯学派
概率的定义长期频率：重复实验中事件发生的比例信念程度：对某个命题的信任度
参数固定但未知的常数随机变量，有自己的分布
输出结果点估计 + 置信区间 + p 值后验分布（完整的概率描述）
先验的使用不使用（或尽量避免）明确纳入先验信息
置信区间的含义重复抽样中 95% 的区间包含真值参数有 95% 的概率落在该区间
对小样本的适应性差，依赖大样本渐近性质好，可以通过先验注入信息
哲学小辩论：如果你说"这个置信区间有 95% 的概率包含真值"，频率学派会纠正你——应该说"如果重复实验很多次，95% 的置信区间会包含真值"。这种区别看似咬文嚼字，但反映了深刻的哲学分歧。贝叶斯后验区间（Credible Interval）则可以直接解释为"参数有 95% 的概率在这个区间内"——这是大多数非统计人员直觉上想要的解释。

22.4 MCMC：从后验分布中采样

贝叶斯统计面临一个实际问题：后验分布通常没有解析解。也就是说，你没法用纸笔直接写出后验分布的具体公式。这就引出了 MCMC（Markov Chain Monte Carlo，马尔可夫链蒙特卡洛）方法。

MCMC 的思路是：不需要知道后验的精确函数形式，只需要能从中采样就够了。通过构造一条马尔可夫链，让它的平稳分布等于目标后验分布，然后从链中收集样本。有了足够多的样本，我们就可以计算后验均值、分位数、可信区间等一切我们关心的量。

import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

# 简化版贝叶斯更新：抛硬币问题
# 先验：Beta(2, 2) —— 认为硬币大致公平但不确定
# 似然：二项分布 Binomial(n, p)
# 后验：Beta(α + 正面数, β + 反面数)

prior_a, prior_b = 2, 2  # 先验参数
n_trials = 100
heads = 60
tails = n_trials - heads

# 后验参数
post_a = prior_a + heads
post_b = prior_b + tails

# 生成后验分布样本
posterior_samples = np.random.beta(post_a, post_b, 100000)

print(f"后验均值: {posterior_samples.mean():.3f}")
print(f"后验 95% 可信区间: "
      f"{np.percentile(posterior_samples, 2.5):.3f} — "
      f"{np.percentile(posterior_samples, 97.5):.3f}")

# 可视化：先验 vs 后验
x = np.linspace(0, 1, 1000)
plt.plot(x, stats.beta.pdf(x, prior_a, prior_b),
         label='先验 Beta(2,2)', linestyle='--')
plt.plot(x, stats.beta.pdf(x, post_a, post_b),
         label='后验 Beta(62,42)')
plt.xlabel('硬币正面概率 p')
plt.ylabel('概率密度')
plt.legend()
plt.title('贝叶斯更新：先验 → 后验')
plt.show()
我的踩坑故事：第一次用贝叶斯方法分析一个临床试验数据时，我随手选了一个无信息先验（uniform prior），觉得这样"客观"。结果因为样本量小，后验分布非常宽，结论基本没什么意义。后来在同事的建议下，我加入了合理的"弱信息先验"（weakly informative prior）——从过往文献中提取了先验信息，后验分布立刻变得有说服力了。但我也见过有人恶意选择先验来操纵结果（例如选择极端先验让结果看起来更"显著"）。先验选择是一把双刃剑：合理的先验能注入有效信息，不合理的先验则可能掩盖真相。这就是所谓的 prior sensitivity（先验敏感性）问题——一定要做敏感性分析，看看不同的先验选择是否会导致结论变化。

22.5 为什么贝叶斯在 AI 时代翻红

过去几十年，频率学派统计由于计算简便，在学术界占据统治地位。但近年来贝叶斯方法全面复兴，原因包括：

  • 计算能力的飞跃：MCMC、变分推理（Variational Inference）等方法的成熟，使得贝叶斯计算不再遥不可及。PyMC、Stan、TensorFlow Probability 等工具让贝叶斯建模变得空前简单。
  • 深度学习中的贝叶斯：贝叶斯神经网络（BNN）、贝叶斯优化（Bayesian Optimization）、贝叶斯深度学习方法在不确定性量化方面表现出色，对 AI 安全、主动学习等场景至关重要。
  • A/B 测试与产品决策：贝叶斯方法可以自然地引入业务先验，并给出直观的分位数和概率陈述（例如"方案 A 有 87% 的概率优于方案 B"），而频率学派只能给出 p 值。
  • 小样本学习和迁移学习：在数据稀缺的场景下，贝叶斯方法的先验机制天然适合迁移已有知识。
推荐工具：如果你想深入实践贝叶斯统计，推荐试试 PyMC（Python 生态中的旗舰贝叶斯库）或 Stan（通过 cmdstanpy 调用）。它们的语法比手写 MCMC 友好得多，内部实现了 NUTS（No-U-Turn Sampler）等高效采样算法。PyMC 的官方示例库是一个宝贵的学习资源。
