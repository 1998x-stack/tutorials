第19章 多元线性回归

大家好，我是你们的讲师。在前面的章节中，我们学习了简单线性回归，也就是用一个自变量去预测因变量。但在现实世界中，事情很少这么单纯：房价不只取决于面积，还取决于地段、楼层、房龄、装修程度——任何一个有经验的房产分析师都会告诉你，"Location, location, location" 是金科玉律，但除此之外还有无数因素在同时起作用。多元线性回归（Multiple Linear Regression）就是把简单回归扩展到了多个自变量的场景，让我们能够同时量化多个因素对结果的影响。这一章，我们来拆解这个统计学习中最重要的工具之一。

19.1 从简单回归到多元回归：加入更多自变量

简单线性回归的公式是 y = β₀ + β₁x，而多元回归只是在右边加更多的项：

y = β₀ + β₁x₁ + β₂x₂ + ... + βₚxₚ + ε

这里每个 βⱼ 都是在"控制其他变量不变"的情况下，xⱼ 变化一个单位时 y 的平均变化量。这个"控制其他变量不变"非常关键，也是多元回归相较于简单回归的核心优势：它让我们从"相关"走向更接近"因果"的推断。

教师笔记：很多初学者把多元回归（multiple regression）和多元输出回归（multivariate regression）搞混。前者是多个自变量、一个因变量；后者是多个因变量同时预测。在机器学习术语中，别搞混了。

19.2 回归系数的解释：控制其他变量

多元回归中的系数被称为偏回归系数（partial regression coefficient）。它回答的是这样一个问题："当其他所有变量都保持不变时，这个变量的变化能带来什么效果？"

举个例子。假设我们想预测员工的年薪（万元）：

年薪 = β₀ + β₁ × 工作年限 + β₂ × 教育年限 + β₃ × 每周加班小时 + ε

如果 β₁ = 1.5，那就意味着：在教育程度和加班时间相同的情况下，多工作一年平均多赚 1.5 万元。这比简单回归中只有一个变量时的系数要"干净"得多，因为我们排除了教育水平和加班时间的混淆效应。

19.3 多重共线性：当自变量互相打架

多元回归有一个隐蔽但危险的陷阱：多重共线性（Multicollinearity）。问题出在自变量之间高度相关，使得它们"抢功"——模型无法区分到底哪个变量在贡献预测力。

判断共线性的常用指标是 VIF（Variance Inflation Factor，方差膨胀因子）。一般认为 VIF > 10 时共线性严重，需要处理。

自变量VIF是否严重含义
工作年限2.3否与其他变量相关度低
教育年限1.8否相对独立
每周加班小时1.2否几乎独立
上一次工资14.5是与工作年限强相关
同事评价得分9.8临界可能有问题
我的踩坑故事：几年前我在分析房价数据时，把所有房间数（bedrooms）和总房间数（total_rooms）都放进了模型。结果显示卧室数的系数竟然是负数——卧室越多，房价越低？这显然荒谬。原因是总房间数已经包含了卧室数，两者高度共线，导致系数估计完全翻转。检查 VIF 后发现卧室数的 VIF 高达 18.7。删除总房间数后，卧室数的系数恢复了正号。这就是共线性让系数符号反转的典型案例。

处理共线性的方法包括：删除其中一个高度相关的变量、使用 PCA 降维后建模、或者采用岭回归（Ridge Regression）等正则化方法。

19.4 交互项：效应不是简单相加

多元回归隐含一个假设：各变量的效果是可加的（additive）。但很多时候变量之间存在交互效应（Interaction Effect）——一个变量的效果依赖于另一个变量的取值。

比如，教育年限对收入的影响，可能在高科技行业比在传统行业更大。这就可以建模为：

收入 = β₀ + β₁ × 教育 + β₂ × 行业类型 + β₃ × (教育 × 行业类型) + ε

这里的 β₃ 就是交互项系数。如果它显著为正，说明高科技行业中教育回报率确实更高。

19.5 模型选择：AIC、BIC 与逐步回归的争议

当你有很多潜在自变量时，如何选择最合适的模型？常用的指标包括：

  • AIC（Akaike Information Criterion）：AIC = -2 ln(L) + 2k，其中 L 是似然值，k 是参数个数。AIC 越小越好，偏向选择预测能力强的模型。
  • BIC（Bayesian Information Criterion）：BIC = -2 ln(L) + k ln(n)，惩罚比 AIC 更严厉（尤其当样本量大时），倾向于更简约的模型。

一种常见的做法是逐步回归（Stepwise Regression）：向前选择（逐个加变量）、向后剔除（逐个删变量）或双向选择。但这个方法近年来争议很大，因为它本质上是在做自动化的多重比较，容易过拟合且 p 值不可靠。

经验之谈：我的建议是，不要完全依赖逐步回归的自动选择。先用领域知识和直觉选择候选变量，然后用 AIC/BIC 做有限范围的比较，最后用交叉验证（Cross-validation）来评估真实预测表现。自动工具只是参考，你才是最终决策者。

下面我们用 Python 的 statsmodels 做一个完整的多元回归演示，包括 VIF 计算：

import statsmodels.api as sm
import pandas as pd
from statsmodels.stats.outliers_influence import variance_inflation_factor

# 模拟数据
import numpy as np
np.random.seed(42)
n = 200
data = pd.DataFrame({
    '面积': np.random.normal(100, 20, n),
    '房龄': np.random.uniform(1, 30, n),
    '楼层': np.random.randint(1, 20, n),
    '卧室数': np.random.randint(1, 6, n)
})
# 构造房价：真实关系
data['房价'] = (2.5 * data['面积'] - 1.2 * data['房龄']
                + 0.3 * data['楼层'] + 5 * data['卧室数']
                + np.random.normal(0, 10, n))

X = data[['面积', '房龄', '楼层', '卧室数']]
X = sm.add_constant(X)
y = data['房价']

model = sm.OLS(y, X).fit()
print(model.summary())

# VIF 计算
vif_data = pd.DataFrame()
vif_data["变量"] = X.columns
vif_data["VIF"] = [variance_inflation_factor(X.values, i)
                   for i in range(X.shape[1])]
print(vif_data)

运行这段代码，你会看到每个自变量的系数、p 值和置信区间，以及 VIF 值。注意这里的 卧室数 和 面积 天然有一定相关（更大的房子通常有更多卧室），但因为我们的数据构造时控制得比较好，VIF 应该不会太高。

思考题：如果你在模型中加入了 面积 和 面积²（二次项），它们之间必然高度相关，VIF 会很高。这是共线性问题吗？不一定。多项式项之间的相关是"结构性"的，不是数据问题，通常不需要处理。VIF 工具要结合上下文理解。

好了，这一章我们认识了多元线性回归——从简单回归扩展到多个自变量，理解了偏回归系数的"控制其他变量"的含义，识别了多重共线性的陷阱，知道了如何通过交互项来建模更复杂的关系，以及用 AIC/BIC 来做模型选择。下一章，我们将进入分类问题的世界：当因变量不是连续数值，而是"是/否"这样的二分类变量时，逻辑回归（Logistic Regression）就是我们的利器。
