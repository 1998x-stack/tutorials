第9章 最小二乘曲线拟合

大家好，我是你们的讲师。上一章我们讨论了在连续函数空间中的最佳平方逼近。但现实中我们遇到的情况往往是这样的：手头有一堆离散的观测数据点 $(x_i, y_i)$，这些数据可能有测量误差，我们想找到一个简单模型来刻画它们背后的规律。这就是最小二乘曲线拟合要做的事情。

我想起一次亲身经历。早年我在一家电商公司做数据分析，业务部门拿来了一组销售数据，想让我"准确地"预测下个季度的销售额。我一开始天真地用了 15 次多项式插值——完美地穿过了所有历史数据点！结果预测值是一个天文数字，业务主管看了直摇头。后来我换成了最小二乘拟合，用一个简单的二次多项式，虽然不完美通过每个点，但预测结果反而更合理。这个教训让我永远记住了：拟合不是穿越，是捕捉趋势。

9.1 超定方程组与最小二乘

最小二乘问题的标准形式如下。给定 $m$ 个数据点 $(x_i, y_i)$，我们希望用 $n$ 个基函数的线性组合来拟合（其中 $m > n$）：

y_i ≈ Σ_{j=0}^{n-1} c_j φ_j(x_i),  i = 1, 2, ..., m

写成矩阵形式就是 $A c \approx b$，其中 $A_{ij} = \phi_j(x_i)$，$b_i = y_i$。这是个超定方程组——方程个数多于未知数个数，通常没有精确解。我们的目标是找到使残差 $r = b - Ac$ 的 $L^2$ 范数最小的 $c$：

minimize ||b - Ac||₂² = Σ_{i=1}^m (y_i - Σ_j c_j φ_j(x_i))²

对 $c$ 求导并令梯度为零，得到法方程（normal equations）：

A^T A c = A^T b

这个方程的解就是最小二乘解。

几何解释：$A c$ 是矩阵 $A$ 列空间中的一个向量。最小二乘解 $c$ 使得 $A c$ 是 $b$ 在 $A$ 列空间上的正交投影。换句话说，我们在整个列空间中寻找离 $b$ 最近的那个向量。残差向量 $r = b - Ac$ 垂直于 $A$ 的所有列——所以 $A^T r = 0$，这就是法方程 $A^T(b - Ac) = 0$ 的来源。

9.2 多项式拟合实战

最常用的基函数就是多项式——$\phi_j(x) = x^j$。这时 $A$ 是 Vandermonde 矩阵：

A = [[1, x_1, x_1², ..., x_1^{n-1}],
     [1, x_2, x_2², ..., x_2^{n-1}],
     ...
     [1, x_m, x_m², ..., x_m^{n-1}]]

下面的 Python 代码演示了如何用不同次数的多项式拟合含噪声的观测数据：

import numpy as np
import matplotlib.pyplot as plt

# 生成模拟数据：真实模型是 2 次多项式 + 噪声
np.random.seed(42)
x = np.linspace(0, 5, 50)
true_y = 2 + 1.5*x - 0.3*x**2
y = true_y + 0.5 * np.random.randn(len(x))

# 用不同次数拟合
for deg in [1, 2, 5, 15]:
    c = np.polyfit(x, y, deg)
    y_pred = np.polyval(c, x)
    rmse = np.sqrt(np.mean((y - y_pred)**2))
    print(f"degree={deg:2d}, RMSE={rmse:.4f}, coefficients: {np.array_str(c, precision=3)}")

# 结果分析：
# degree=1 (欠拟合): RMSE较大，模型太简单无法捕捉曲率
# degree=2 (理想):   RMSE最小，正确恢复了真实模型
# degree=5 (过拟合): RMSE虽小，但系数很大，外推会爆炸
# degree=15(严重过拟合): 系数极大，Runge现象出现

踩坑故事——多项式拟合销售数据：我有个朋友做销售预测，用 $n=20$ 的多项式拟合了 21 个季度数据，训练集 RMSE 几乎为零，他非常得意。结果预测的下个季度销售额是负的 3 个亿！这就是过拟合的典型症状——高次多项式为了穿过每一个点，系数变得非常大，在外推区域剧烈振荡。记住一个经验法则：多项式的次数不要超过数据点数的 1/5，而且一定要在外推前检查系数的量级。

拟合程度训练误差测试误差系数大小典型次数
欠拟合 (underfitting)大大偏小1-2
适当拟合 (good fit)中等中等合理2-5
过拟合 (overfitting)很小极大极大>10

9.3 正规方程 vs QR 分解

直接求解法方程 $A^T A c = A^T b$ 是最直观的方法，但有一个严重的问题：$A^T A$ 的条件数是 $A$ 的条件数的平方！

cond(A^T A) = cond(A)²

这意味着如果 $A$ 的条件数是 $10^4$（对于 Vandermonde 矩阵这很容易达到），那么 $A^T A$ 的条件数就是 $10^8$，解法的数值稳定性会大幅下降。

更好的方法——QR 分解：将 $A$ 分解为 $A = QR$，其中 $Q$ 是正交矩阵，$R$ 是上三角矩阵。那么法方程变为 $R^T Q^T Q R c = R^T Q^T b$，即 $R^T R c = R^T Q^T b$，简化后得到 $R c = Q^T b$。QR 分解不需要计算 $A^T A$，条件数保持在 $cond(A)$ 的水平，数值稳定性好得多。再进一步，SVD 分解是最稳健的——MATLAB 的 \\ 运算符和 NumPy 的 lstsq 底层都使用 SVD。

Python 中推荐的做法：

import numpy as np

# 构造数据
x = np.array([0, 1, 2, 3, 4, 5])
y = np.array([2.1, 2.9, 4.0, 5.1, 5.8, 7.2])

# 方法1: 直接用法方程 (不推荐)
A = np.column_stack([np.ones_like(x), x, x**2])
c1 = np.linalg.solve(A.T @ A, A.T @ y)
print("法方程解:", c1)

# 方法2: 用 lstsq (SVD 分解，推荐)
c2, residuals, rank, s = np.linalg.lstsq(A, y, rcond=None)
print("SVD 解:  ", c2)

# 方法3: 用 polyfit (内部调用了 Vandermonde 矩阵的 QR 分解)
c3 = np.polyfit(x, y, 2)
print("polyfit解:", c3)

9.4 非线性最小二乘简介

以上讨论的都是线性最小二乘——模型关于参数 $c_j$ 是线性的。但现实中很多模型是非线性的，比如：

y = a * exp(b * x) + c        # 指数模型
y = a * sin(b * x + c)         # 正弦模型
y = a * x / (b + x)            # Michaelis-Menten 模型

在这些模型中，参数（$a, b, c$）不是线性地出现在方程中。这时不能再用法方程一次性求解，需要迭代方法。

Gauss-Newton 方法的核心思想是：在当前参数估计值处对模型做 Taylor 展开线性化，然后求解线性最小二乘问题得到更新方向。而Levenberg-Marquardt 方法在 Gauss-Newton 和梯度下降之间做了自适应插值——当远离最优解时像梯度下降（稳健但慢），当接近最优解时像 Gauss-Newton（快速收敛）。

实战建议：如果你要用非线性最小二乘，推荐直接用 scipy.optimize.curve_fit，它底层实现的就是 Levenberg-Marquardt 算法。不过要特别注意初值的选择——非线性最小二乘可能有多个局部极小值，不同的初值可能收敛到不同的解。一个实用的策略是先用全局优化方法（如差分进化）找到一个大致的区域，再用 Levenberg-Marquardt 做局部精化。
