什么是图像分割

同学们大家好。从第15章到第18章，我们花了四章的篇幅深入探讨了目标检测——从两阶段的R-CNN家族到单阶段的YOLO和SSD。目标检测告诉我们图像中"有什么、在哪里"，输出的是物体周围的边界框。但有时候，边界框远远不够。想象一下：你正在做医学影像分析，需要精确圈出肿瘤的轮廓；或者你在做自动驾驶，需要区分路面、人行道和车道线——边界框这种"矩形框"过于粗糙，它会把不属于物体的背景也包含进来。这时候，我们就需要进入一个更精细的领域：图像分割（Image Segmentation）。

图像分割的本质是像素级分类。翻译成大白话：对于图像中的每一个像素，我们都给它分配一个类别标签。如果把目标检测比作"在照片上用马克笔画矩形框"，那图像分割就是"沿着物体的边缘，用最细的笔把每个物体精确地涂上颜色"——每一个像素都要被归类。这种精细程度带来的直接好处是：你可以获得物体的精确形状、轮廓和面积，而不只是它的位置范围。

先分享一个亲身踩坑的故事。几年前我参与一个自动驾驶感知项目，初期方案采用的是目标检测模型（YOLOv3）来识别可行驶区域。我们天真地认为，只要检测到前方的障碍物车辆，剩下的路面就是可行驶区域。结果第一次封闭道路测试就出了笑话：检测模型正确识别了前方的车辆，但路中间有一块井盖凸起——它没有被检测为"车辆"或"行人"，系统默认它是路面的一部分，直接规划了一条碾压井盖的路径。虽然测试车速很慢没有出事，但这个教训很深刻：对于需要精细空间理解的场景，边界框太粗了。你需要知道每一个像素到底属于"路面""路沿"还是"障碍物"。从那以后，我们对道路感知的方案全面转向了图像分割。

三大分割任务

你可能经常听到"语义分割""实例分割""全景分割"这些术语乱飞。它们到底有什么区别？我们用一个具体的场景来说清楚。

假设图中有三个人、两辆车和一条狗。不同分割任务的输出截然不同：

分割类型英文名称输出内容是否区分个体覆盖所有像素
语义分割Semantic Segmentation每个像素的类别（人、车、狗、背景）否（所有人都是"人"类）通常不考虑"东西"类
实例分割Instance Segmentation每个物体实例的掩码（人1、人2、人3、车1、车2、狗1）是通常只覆盖"东西"类
全景分割Panoptic Segmentation每个像素的类别+实例ID是（对"东西"类）是（覆盖所有像素）

语义分割是最基础的形式。它把图像分成不同语义区域：所有"人"像素涂成红色、所有"车"涂成蓝色、所有"路面"涂成灰色。但注意，语义分割不区分个体——如果图中有三个人，他们的像素都会标记为"人"，而不是"人1"、"人2"、"人3"。语义分割就像油漆工：每个类别一种颜色，刷过去完事，不管哪里有多少个实例。

实例分割在语义分割的基础上更进一步：它不仅要告诉你是"人"，还要区分出"第一个人""第二个人""第三个人"。实例分割只处理 countable objects（有明确数量的物体，通常称为"things"，如人、车、动物），而不处理 amorphous stuff（无定形区域，如天空、草地、路面，通常称为"stuff"）。Mask R-CNN就是实例分割的代表作。

全景分割由Facebook AI Research在2018年提出，目标是统一语义分割和实例分割——它要求模型同时处理"things"和"stuff"，为图像中的每一个像素都分配一个标签，对于"things"还要附加实例ID。全景分割的完整程度最高，但从实现角度看也最具挑战性。

💡 直观理解：如果图像是一条街道——语义分割告诉你"这个区域是路面，那个区域是人行道，这几个像素是车"；实例分割告诉你"这里有三辆车，这是第一辆的精确轮廓，那是第二辆的"；全景分割在此基础上还要告诉你"天空、建筑墙面、路牌这些区域都是什么"——它想把整个画面全部解释清楚，不留一个未分类的像素。

分割评价指标

做分割和做检测一样，首先需要一把"尺子"来衡量模型的好坏。最核心的指标是IoU（Intersection over Union，交并比）——你可能在检测章节已经见过它了。在分割任务中，IoU的计算方式完全一样：模型预测的像素掩码和真实标注之间的交集面积除以并集面积。

# IoU 的计算伪代码
def compute_iou(pred_mask, gt_mask):
    """计算单个类别的 IoU"""
    intersection = (pred_mask & gt_mask).sum()
    union = (pred_mask | gt_mask).sum()
    if union == 0:
        return 1.0  # 真值和预测都为空的特殊情况
    return intersection / union

# mIoU：所有类别 IoU 的均值
def compute_miou(pred_masks, gt_masks, num_classes):
    ious = []
    for c in range(num_classes):
        iou = compute_iou(pred_masks[c], gt_masks[c])
        ious.append(iou)
    return sum(ious) / len(ious)

# Pixel Accuracy：正确分类的像素数占总像素数的比例
def pixel_accuracy(pred, gt):
    correct = (pred == gt).sum()
    total = pred.numel()
    return correct / total

图像分割中几个关键指标：

  • Pixel Accuracy（PA，像素精度）：最直观的指标——正确分类的像素数除以总像素数。但它有一个严重缺陷：当类别严重不平衡时（比如背景占95%、目标只占5%），模型什么都不做只预测背景就能拿到95%的PA，这显然没有意义。
  • Mean IoU（mIoU，平均交并比）：计算每个类别的IoU，然后对所有类别取平均。这是分割领域最主流、最可靠的指标。mIoU不受类别不平衡的过度影响，因为每个类别在计算IoU时都是独立计算的。
  • Frequency Weighted IoU：在mIoU的基础上加入类别频率权重，让出现频率高的类别贡献更大。在某些场景下更有实际意义。

在实际竞赛和论文中，mIoU几乎是唯一的"标准答案"。如果你看到一个分割论文说"我们的方法达到了78.5%"，通常指的就是mIoU。

⚠️ 踩坑提醒：mIoU的计算有一些"潜规则"需要注意。第一个是在评估时要不要包含背景类——这在不同的数据集和论文中习惯不一致，直接比较数值时要先确认。第二个是当某个类别在图像中没有出现时，这一类的IoU是算0还是忽略——建议在代码中明确处理。我在早期对比不同模型结果时就吃过这个亏，两个模型相差3个点，最后发现是其中一个把未出现类的IoU当0算，另一个直接忽略了——心态直接崩了。

经典分割数据集

数据集的演进史几乎就是分割技术的进步史。早期的PASCAL VOC为分割打下了基础，COCO把任务标准化到新的高度，Cityscapes聚焦于自动驾驶场景，ADE20K挑战模型的细粒度理解能力。

数据集任务图像数类别数特点
PASCAL VOC 2012语义分割2,91321（含背景）最早的标准化分割基准，规模小但标注质量高
MS COCO实例分割 / 全景分割~330K80（things）大规模、多任务、每图平均7.7个实例
Cityscapes语义分割 / 实例分割~25K30（19用于评估）街景驾驶场景、高分辨率、精细标注
ADE20K语义分割 / 全景分割~27K150（things+stuff）场景理解、细粒度、密集标注
BDD100K语义分割~100K40自动驾驶、多样化天气和光照

在这些数据集中，MS COCO是实例分割最重要的基准——几乎每一篇实例分割论文都会报告COCO的mAP（mask AP）。Cityscapes则是自动驾驶场景分割的事实标准，它提供了5000张精细标注图像和20000张粗略标注图像，覆盖了德国多个城市的街道场景。ADE20K由MIT CSAIL发布，以高度多样性和密集标注著称——每张图像平均包含19.5个实例和13.7个语义类别，覆盖了日常生活的方方面面。

分割的难点与挑战

图像分割远不止是"把分类网络改成按像素分类"那么简单。以下几个挑战贯穿了所有分割方法：

边界模糊性。物体之间的边界往往不是清晰的黑白分界线。想一想：一只橘猫趴在橙色地毯上——猫和地毯的边界在哪里？像素级别的标注本身就存在标注者之间的主观差异。这也是为什么分割标注如此昂贵的原因——高质量的标注需要对每一个边界像素仔细确认。

小目标分割。在COCO数据集中，面积小于32×32像素的物体占了约41%的标注实例。对这些小目标做精确分割极其困难，因为随着下采样，小目标的信息在深层特征图中几乎消失殆尽。这也是为什么像U-Net这样的编码-解码结构通过跳跃连接来保留高分辨率细节。

类别不平衡。在一张自然图像中，大面积的背景（天空、地面、墙面）占据了绝大多数像素，而小物体（远处的行人、交通标志）只占极少的像素。如果不做特殊处理，模型会倾向于"忽略"小物体和罕见类，因为反正大多数像素都是背景。常用的解决方案包括加权损失函数（Weighted Cross-Entropy）、Dice Loss、以及Focal Loss（最初为检测设计，但在分割中同样有效）。

尺度多样性。同一个类别在不同图像中可能有完全不同的尺度。比如，一辆近处的大卡车可能占据半幅图像，而远处同一辆车可能只有几十个像素。模型需要同时处理好这两种情况，这对感受野的设计提出了很高要求。

遮挡与截断。现实场景中物体互相遮挡是常态。比如街拍中行人被车辆遮挡了半边身体——分割模型需要"脑补"出被遮挡的部分，这需要很强的上下文推理能力。

# Dice Loss —— 分割中最常用的损失函数之一
# Dice 系数本质上是 IoU 的平滑版本
def dice_loss(pred, target, smooth=1e-6):
    """pred 和 target 都是形状为 [B, C, H, W] 的张量"""
    intersection = (pred * target).sum(dim=(2, 3))
    union = pred.sum(dim=(2, 3)) + target.sum(dim=(2, 3))
    dice = (2.0 * intersection + smooth) / (union + smooth)
    return 1.0 - dice.mean()

分割模型发展脉络

从2015年至今，分割模型的发展大概经历了五代演变。理解这条脉络，能帮助你明白今天的主流方法为什么长这样。

第一代：FCN（Fully Convolutional Network, 2015）。Jonathan Long等人第一次证明了全卷积网络可以做端到端的像素级预测。他们用转置卷积（transposed convolution）代替全连接层进行上采样，并通过跳跃连接融合浅层和深层特征。FCN的意义是开创性的——在此之前，分割通常采用patch-based的方法（滑动窗口+分类器），效率低下且感受野有限。虽然FCN在今天看来很粗糙（上采样结果不够精细），但它是所有后续工作的基石。

第二代：编码-解码结构（2015-2016）。U-Net（2015）和SegNet（2016）将FCN发展为对称的编码-解码结构。编码器逐步提取语义信息并下采样，解码器逐步恢复空间分辨率。U-Net的跳跃连接将编码器每一层的细节特征直接拼接到对应的解码器层，极大地提升了边界精细度。这个设计在医学图像分割中尤其成功，至今仍是许多任务的默认选择。

第三代：空洞卷积与多尺度融合（2016-2018）。DeepLab系列（v1/v2/v3/v3+）引入了空洞卷积（Atrous/Dilated Convolution）和空洞空间金字塔池化（ASPP），在不增加参数量的情况下指数级扩大感受野，解决了下采样导致空间分辨率降低的问题。与此同时，PSPNet（2017）提出了金字塔池化模块（PPM），通过多尺度池化聚合不同区域的上下文信息。

第四代：检测+分割统一（2017-2019）。Mask R-CNN将Faster R-CNN扩展为检测+分割的统一框架，通过RoIAlign解决RoI Pooling的量化误差问题，成为实例分割的事实标准。随后的YOLACT、SOLO等单阶段实例分割方法则在速度上做出了突破。

第五代：Transformer时代（2020-至今）。SETR首次将纯Transformer用于语义分割。SegFormer提出了分层的Transformer编码器+轻量级MLP解码器。DETR中的Transformer架构催生了MaskFormer和Mask2Former，将语义分割、实例分割和全景分割统一在同一个框架之下。而Meta的SAM（Segment Anything）更是开启了分割基础模型的新纪元——用一个模型分割一切。

🔮 趋势观察：从FCN到SAM，分割模型的发展呈现出三条清晰的脉络：一是从粗到精（边界质量不断提升），二是从特定到统一（不同分割任务正在被同一个架构覆盖），三是从监督到通用（从依赖大量标注数据到zero-shot泛化）。理解这些趋势，能帮助你在选择分割方案时做出更有前瞻性的判断。
