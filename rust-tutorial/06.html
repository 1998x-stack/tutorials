引子：当一个结构体拯救了一场Code Review

几年前我review一个用Go写的服务时，看到一段"传了8个参数"的函数签名：

fn create_user(name: &str, age: u8, email: &str, phone: &str,
               address: &str, city: &str, zip: &str, active: bool) -> User

调用者的代码更是壮观——8个参数按顺序传，非常容易搞混参数位置。果不其然，有个调用把city和zip传反了，但同为字符串类型，编译器根本不会报错，直到生产环境才发现地址错误。

如果在Rust中用结构体来重构这个代码，结果会是这样的：

struct CreateUserRequest {
    name: String,
    age: u8,
    email: String,
    phone: String,
    address: String,
    city: String,
    zip: String,
    active: bool,
}

调用时使用字段名初始化：CreateUserRequest { name: ..., age: ..., ... }——参数位置搞混的可能性降到零。更棒的是，你可以在结构体上添加方法、实现trait、做类型安全的扩展。

这就是结构体的魅力：把松散的数据组织成有意义的整体，让类型系统替你做事。

结构体定义

定义与实例化

用struct关键字定义一个结构体：

struct User {
    username: String,
    email: String,
    sign_in_count: u64,
    active: bool,
}

实例化时需要为所有字段提供值：

let user1 = User {
    username: String::from("alice"),
    email: String::from("alice@example.com"),
    sign_in_count: 1,
    active: true,
};

访问字段用点号语法：

println!("用户名: {}", user1.username);
user1.email = String::from("new@example.com");  // 需要user1是mut

注意：Rust的结构体不允许只让部分字段可变——要么整个结构体实例可变（let mut），要么全部不可变。

字段初始化简写

当字段名和变量名一致时，可以用简写：

fn build_user(username: String, email: String) -> User {
    User {
        username,  // 等同于 username: username
        email,     // 等同于 email: email
        sign_in_count: 1,
        active: true,
    }
}

这个语法糖在Rust 2024 Edition中进一步增强了，但基本原则不变：减少重复，提升可读性。

元组结构体

元组结构体（Tuple Struct）有点像是"给元组取了个名字"：

struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

let black = Color(0, 0, 0);
let origin = Point(0, 0, 0);

Color和Point虽然内部都是三个i32，但它们是不同的类型——不会像类型别名那样混淆：

fn print_color(c: Color) { /* ... */ }
print_color(origin);  // 编译错误！期望Color，传了Point

这就是元组结构体的主要价值——当你想给一组数据取一个有意义的名称，但又不想（或不需要）给每个字段命名时，用元组结构体。

newtype模式

元组结构体一个非常常见的用法是newtype模式——包装一个已有类型，赋予它新的语义：

struct UserId(u64);
struct OrderId(u64);

fn get_order(uid: UserId, oid: OrderId) { /* ... */ }

// 调用：不会把参数搞混
get_order(UserId(42), OrderId(100));

// 如果传反了，编译器会报错：
// get_order(OrderId(100), UserId(42)); // 类型不匹配！
newtype模式的妙用：在大型项目中，UserId和OrderId都是u64，但语义完全不同。用newtype包装后，编译器会在类型层面为你做安全检查。这几乎不产生运行时开销——Rust的零成本抽象又立一功。

方法

方法（Method）是与某个类型相关联的函数。在Rust中，方法定义在impl块中：

struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    // &self 是 &Self 的简写，表示"对当前实例的不可变引用"
    fn area(&self) -> u32 {
        self.width * self.height
    }

    // 可变方法：需要修改self
    fn scale(&mut self, factor: u32) {
        self.width *= factor;
        self.height *= factor;
    }
}

fn main() {
    let mut rect = Rectangle {
        width: 30,
        height: 50,
    };
    println!("面积: {}", rect.area());  // 1500

    rect.scale(2);
    println!("缩放后: {}", rect.area());  // 6000
}

方法的三种self形式

签名 含义 所有权
fn foo(self)获取所有权调用后实例被消费
fn foo(&self)不可变借用（最常用）调用后仍可用
fn foo(&mut self)可变借用可以修改字段

如果你有一个方法需要消耗当前实例（比如把结构体转换成另一种类型），用self。如果只是读取数据，用&self。如果需要修改字段，用&mut self。

方法 vs 函数：方法本质上就是函数，只不过第一个参数是self。区别在于调用语法——方法使用object.method()语法，而函数使用function(args)语法。Rust会自动处理引用/解引用：如果你有&T，调用一个需要&self的方法时，编译器会自动帮你处理。这叫做自动引用和解引用（automatic referencing and dereferencing）。

关联函数

关联函数（Associated Function）是定义在impl块中但不以self为参数的函数。它们通常用作"构造函数"：

impl Rectangle {
    // 关联函数：没有self参数
    fn new(width: u32, height: u32) -> Rectangle {
        Rectangle { width, height }
    }

    fn square(size: u32) -> Rectangle {
        Rectangle { width: size, height: size }
    }
}

// 调用关联函数用 :: 语法
let rect = Rectangle::new(30, 50);
let square = Rectangle::square(10);

你最熟悉的关联函数就是String::from("hello")——它是String类型上的一个关联函数。

Rectangle::square是一个很好的设计模式：它给出了一个"从正方形创建矩形"的直观API，而不需要把这种创建逻辑暴露在结构定义的层面。

方法 vs 关联函数：记住，方法一定有self参数，用.语法调用。关联函数没有self参数，用::语法调用。Java/C++中的"静态方法"对应Rust中的关联函数。

结构体更新语法

当你要基于一个已有的结构体实例创建一个新实例，只修改部分字段时，可以用..展开语法：

let user1 = User {
    username: String::from("alice"),
    email: String::from("alice@example.com"),
    sign_in_count: 1,
    active: true,
};

let user2 = User {
    email: String::from("new@example.com"),
    username: String::from("bob"),
    ..user1  // 其余字段从 user1 "继承"
};

// 注意：user1中的username和email是String（堆上数据）
// ..user1 会把剩下的字段"移动"到user2
// 所以 user1 在 .. 之后不能再被整体使用
// 但如果user1的active（bool，实现了Copy）仍然可以单独访问
println!("{}", user1.active);  // 正确，bool是Copy类型
// println!("{user1}");  // 编译错误！部分字段已被move
结构体更新语法与所有权：..user1会把user1中未被显式赋值的字段移动到user2中。所以user1之后不能再作为整体使用。但如果剩下的字段都是Copy类型，user1就还能用——这和变量移动的规则完全一致。理解这一点，你就真正理解了Rust的所有权在结构体层面的应用。

一个完整的示例：构建图书管理系统

让我们把本章学到的知识组合起来，构建一个小型的图书管理模块：

struct Book {
    title: String,
    author: String,
    isbn: String,
    pages: u32,
    available: bool,
}

impl Book {
    // 构造函数（关联函数）
    fn new(title: String, author: String, isbn: String, pages: u32) -> Book {
        Book {
            title,
            author,
            isbn,
            pages,
            available: true,
        }
    }

    // 借出图书
    fn borrow(&mut self) {
        if self.available {
            self.available = false;
            println!("已借出：{}", self.title);
        } else {
            println!("{} 已被借出", self.title);
        }
    }

    // 归还图书
    fn return_book(&mut self) {
        self.available = true;
        println!("已归还：{}", self.title);
    }

    // 获取图书信息
    fn info(&self) -> String {
        format!(
            "《{}》by {} | ISBN: {} | {}页 | {}",
            self.title,
            self.author,
            self.isbn,
            self.pages,
            if self.available { "可借" } else { "已借出" }
        )
    }
}

fn main() {
    let mut book = Book::new(
        String::from("Rust程序设计"),
        String::from("张老师"),
        String::from("978-7-xxx-xxxxx-x"),
        450,
    );

    println!("{}", book.info());
    // 《Rust程序设计》by 张老师 | ISBN: 978-7-xxx-xxxxx-x | 450页 | 可借

    book.borrow();    // 已借出：Rust程序设计
    book.borrow();    // Rust程序设计 已被借出
    book.return_book(); // 已归还：Rust程序设计

    // 用结构体更新语法创建一个续版
    let book2 = Book {
        pages: 520,
        ..book  // 注意：book的title/author/isbn被move到book2
    };
    println!("{}", book2.info());
}

这个示例展示了：结构体定义、关联函数构造、多个impl方法的配合使用、以及结构体更新语法。在实际项目中，你的结构体通常会更加复杂，但基本原则不变——用有意义的类型包装数据，并在impl块中提供操作它们的逻辑。

本章小结

这一章我们学习了Rust的结构体体系：通过命名结构体组织散乱的数据；用元组结构体实现轻量类型包装（newtype模式）；在impl块中定义方法和关联函数；用..语法基于已有实例创建新实例。结构体是你从"使用基本类型"到"定义自己的类型"的第一步，也是Rust类型系统的核心组成部分。
