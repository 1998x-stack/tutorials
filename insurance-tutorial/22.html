第22章 企业风险管理

2019年，我参加了一家保险公司的年度风险管理研讨会。台下坐着董事会成员、高管团队和各部门负责人。首席风险官（CRO）站在台上，屏幕上展示了一张巨大的风险热力图——百余个风险点分布在5×5的矩阵中。董事长指着图说了一句让我至今印象深刻的话："这张图上标了120个风险，但我要的不是120个风险的名字，我要的是——哪3个风险一旦同时发生，我们就完了？"

这个问题的本质，就是企业风险管理（ERM, Enterprise Risk Management）的核心：从"管理每个孤立的风险"升级为"管理风险之间的相互作用和整体暴露"。前面两章讲的精算和偿付能力，解决的分别是"具体险种怎么定价、准备金怎么提"和"满足监管资本要求"。而ERM解决的是"作为一个整体，公司能不能在各种极端情景下活下去、活得更好"。

ERM概述

COSO 2017框架

说到企业风险管理，就绕不开COSO（Committee of Sponsoring Organizations of the Treadway Commission）。2017年，COSO发布了更新的ERM框架——《企业风险管理——与战略和绩效的整合》。这个新框架有五大要素、二十条原则，是当今全球ERM实践的"圣经"。

五大要素：

要素 核心内容 保险公司的具体体现
治理与文化 董事会风控定调、组织架构、人才理念 设置独立CRO、风险管理委员会、三道防线
战略与目标设定 将风险融入战略规划 风险偏好决定产品线和投资策略选择
绩效 风险与绩效的协同管理 风险调整后资本回报率（RAROC）考核
审阅与修订 持续监控和迭代 季度偿付能力报告+年度ORSA
信息、沟通与报告 风险信息的上传下达 风险仪表盘、风险限额报告、压力测试通报

ERM与保险公司的特殊性

保险公司的ERM和其他行业有一个根本差异：保险公司的主营业务就是承担风险。制造业企业的ERM主要关注"如何避免生产中断"；互联网公司的ERM关注"数据安全和业务连续性"。但保险公司的ERM不仅要管理自身运营风险，还要管理承保进来的各类外部风险——而且这些外部风险恰恰是其利润来源。这就产生了一个ERM领域特有的张力：

"承保风险"既是利润来源，也是亏损的根源。一个过于保守的ERM框架会让公司不敢承保高风险业务，失去市场机会；一个过于激进的框架则可能导致偿付能力危机。一家优秀的保险公司，ERM不是在"规避风险"，而是在"选择性地承担风险并获得合理回报"。

风险偏好与风险容忍度

风险偏好声明

风险偏好（Risk Appetite）是ERM体系的"北极星"——它在最高层面回答了"公司愿意承担什么风险、承担多少风险"的问题。每家保险公司都需要一份经董事会审议通过的风险偏好声明（Risk Appetite Statement, RAS）。

一个典型的风险偏好声明可能长这样：

XX保险集团风险偏好声明（节选）

我司在经营活动中遵循"稳妥审慎、风险可控、收益合理"的总原则。

定量偏好：
1. 综合偿付能力充足率始终不低于150%（内部目标，高于监管的120%）
2. 核心偿付能力充足率始终不低于100%
3. 风险综合评级始终维持A类
4. 年度净利润波动率不超过预算的20%
5. 最大单一风险暴露不超过资本的15%

定性偏好：
1. 不承保超出精算定价能力的新险种
2. 不投资于无法穿透底层资产的结构化产品
3. 巨灾风险通过再保险转移比例不低于80%

风险限额体系

风险偏好声明是"顶层大方向"，要落地还需要一套风险限额体系（Risk Limit System）——把宏观的偏好拆解成各业务单元的"硬约束"。

风险限额体系的典型结构：

第一层：公司级偏好（偿付能力 ≥ 150%）
  ↓
第二层：业务条线限额
  ├─ 寿险业务: 新业务价值增长 ≥ 15%, 退保率 ≤ 5%
  ├─ 财险业务: 综合成本率 ≤ 98%, 车险赔付率 ≤ 65%
  └─ 投资业务: 权益类资产 ≤ 总资产20%, 信用债评级 ≥ AA
  ↓
第三层：风险维度限额
  ├─ 保险风险: 单个风险单位自留额 ≤ 资本1%
  ├─ 市场风险: 利率敏感度DV01 ≤ 5亿元
  └─ 信用风险: 单一交易对手集中度 ≤ 资本5%
  ↓
第四层：日常监控指标（月度预警）
  ├─ 流动性覆盖率 ≥ 200%
  ├─ 保费收入达成率 90%-110%
  └─ 投资组合久期缺口 < 2年

实务经验：一个好的风险限额体系不是"条条框框"地限制业务，而是给了前线业务人员一个"自由发挥的跑道"。你知道权益类投资的极限是20%总资产，就可以在这个范围内做最优配置，不需要每笔投资都去请示风控部。限额越清晰，公司的决策越高效。

ORSA（自有风险与偿付能力评估）

监管要求

ORSA（Own Risk and Solvency Assessment，自有风险与偿付能力评估）是偿二代第二支柱的核心要求。根据监管规定，所有保险公司每年至少进行一次ORSA，并形成书面报告存档备查。ORSA不是一般的合规动作——它是董事会和高管层"自我审视"风险状况的年度大考。

评估流程

一个标准的ORSA流程包括以下五个阶段：

  1. 风险识别与更新：过去一年出现了哪些新风险？哪些旧风险的变化超出预期？比如2024-2025年期间，所有保险公司都在做"利率持续下行"的风险识别更新
  2. 风险量化：用内部模型或标准公式量化各风险对偿付能力的影响。这一步需要精算、风控、投资三个部门的深度协作
  3. 前瞻性资本评估：预测未来3-5年，在基准情景和多种不利情景下，公司的资本充足率会如何演变
  4. 压力测试：模拟极端情景（如利率跳降100bp、股市暴跌30%、超大规模巨灾并发）下的资本充足率底线
  5. 报告与决策：形成ORSA报告，提交董事会审议。如果发现资本规划与风险暴露之间存在缺口，需要制定整改计划
ORSA年度时间线（示例）:
3月  → 启动数据收集和风险识别
5月  → 完成风险量化模型运行
7月  → 完成前瞻性资本评估和压力测试
9月  → ORSA报告初稿 -> 管理层审阅
11月 → 董事会审议 -> 终稿存档
次年 → 持续监控并更新假设

压力测试与情景分析

逆情景设计

传统的压力测试是先设定一个假设（比如利率上升1%），然后看后果。但逆情景设计（Reverse Stress Testing）反过来思考——先定义"什么情况下公司会破产"（偿付能力充足率跌破100%），再反推"什么情景组合会导致这个结果"。

"逆情景"比"正情景"更有价值的地方在于：它能帮你发现那些你以为不会同时发生、但实际上可能同时发生的风险组合。比如："利率下行 + 股市暴跌 + 巨灾"在正常年份几乎不可能同时出现，但在2020年新冠疫情爆发时，这三种风险确实在几周内先后冲击了保险公司——利率暴跌、股市熔断、大量意外险赔付涌来。

极端事件模拟

保险公司常用的极端事件模拟包括：

  • 自然巨灾：百年一遇台风登陆长三角、大地震袭击一线城市——不仅评估直接赔付成本，还要考虑经济停摆导致的保费收入下降和退保潮
  • 金融风暴：债券违约潮、股市持续下跌、汇率剧烈波动——保险公司投资端受到的冲击
  • 流行病/公共卫生事件：大规模传染病导致死亡率上升（影响寿险赔付）、经济停滞导致保费收入下降
  • 操作风险事件：核心系统宕机、数据泄露、销售误导导致的集体诉讼

踩坑故事：2020年疫情初期，不少保险公司在压力测试中暴露了严重的"模型漏洞"。过去的情景设计中，没有人把"居家隔离导致新车销量暴跌"和"车险赔付率下降"这两个事情联系起来——结果因为路上车少了，车险赔付率确实下降了（似乎是好事），但保费收入也降了（因为新车卖不出去），而且退保率上升了（因为居民收入下降）。2020年二季度，好几家公司的综合偿付能力充足率"意外"下降了5-8个百分点——原因不是赔付高了，而是保费降了（分母变小了）。这个教训说明：压力测试不能只盯着损失端，还要看收入端。

经济资本模型

经济资本 vs 监管资本

维度 监管资本（Regulatory Capital） 经济资本（Economic Capital）
目的 满足监管最低要求 反映公司真实风险水平
方法 标准公式（监管统一规定） 内部模型（公司自建，需监管审批）
置信水平 隐含在公式中（约99.5%） 公司自定（通常99.5%或99.95%）
风险覆盖 保险风险+市场风险+信用风险（标准模板） 可纳入操作风险、战略风险、声誉风险等
差异化 同质化（所有公司用同一套规则） 个性化（反映公司独特风险特征）
管理层用途 合规报告 战略决策、绩效考核、资本配置

VaR与TVaR

经济资本的核心是风险度量（Risk Measure）——如何用一个数字来概括整个公司的风险暴露。最常用的两个工具是：

VaR（Value at Risk，在险价值）：在给定置信水平和时间窗口下，可能的最大损失。比如"99.5% VaR = 50亿元"意味着："一年内，只有0.5%的概率（200年一遇）亏损超过50亿元。"

TVaR（Tail Value at Risk，尾部在险价值，又称CVaR）：VaR有一个著名的缺陷——它不问"超过VaR以后会怎样"。TVaR弥补了这个缺陷——它计算所有超过VaR的损失的"平均值"：

VaR vs TVaR 的直觉对比：

假设某公司的损失分布（单位: 亿元）:
情景        概率        损失
一般年份    99.0%       10
较坏年份    0.5%        100
极端年份    0.5%        500

VaR(99.5%) = 100亿元 （只看第99.5百分位的值）
TVaR(99.5%) = (100 × 0.25% + 500 × 0.25%)/0.5% = 300亿元

TVaR告诉管理层：在"恶化的0.5%"中，平均损失是300亿，而不仅仅是100亿。
考虑到最坏情景的"数量级"，VaR明显低估了尾部风险。

行业趋势：越来越多的保险公司从VaR转向TVaR作为内部风险管理的核心指标。中国偿二代的标准公式以VaR（75%置信水平，一年期）为基础，但申请使用内部模型的保险公司（如平安、太保）普遍采用了TVaR。原因很简单：保险公司面对的不是正态分布，是厚尾分布——尾部风险才是真正致命的。

ERM最佳实践

平安集团：全面风险管理体系

中国平安的ERM体系被业界公认为标杆。其核心特征包括：

  • 集团统一风控平台：平安建设了覆盖寿险、财险、银行、证券、科技等所有子公司的统一风控平台，实现了风险数据的实时聚合和监控。任何一个子公司的风险事件，在几分钟内就能在集团CRO的仪表盘上反映出来
  • 三年滚动资本规划：平安每年基于集团战略和风险偏好制定三年资本规划，将评价结果与子公司管理层的KPI挂钩
  • 内部模型应用深度：平安是国内最早获批使用内部模型计算最低资本的保险公司之一。其内部模型覆盖了保险风险、市场风险、信用风险和操作风险，置信水平设在99.5%（一年期），以TVaR为主要风险度量
  • 风险文化渗透：平安要求所有中层以上管理者每年参加风险培训，新员工入职必须通过风险合规考试

太平洋保险：RAROC导向的经营模式

太平洋保险的ERM特色是将RAROC（Risk-Adjusted Return on Capital，风险调整后资本回报率）作为核心经营指标。每一笔业务、每一条产品线、每一个子公司的绩效，都用RAROC来衡量——等于说，业务部门不能光看"赚了多少利润"，还要看"占用多少资本"来赚这个利润。

RAROC 计算公式:
RAROC = (净利润 - 预期损失) / 经济资本

举例:
A产品线: 净利润3000万, 经济资本占用1.5亿, 预期损失500万
B产品线: 净利润4000万, 经济资本占用3亿, 预期损失300万

RAROC(A) = (3000 - 500) / 15000 = 16.7%
RAROC(B) = (4000 - 300) / 30000 = 12.3%

结论：虽然B产品线净利润更高，但A产品线的资本使用效率更优。

实务要点：RAROC最大的贡献是解决了保险公司内部"资源分配"的问题。过去业务部门争资源（资本额度）时吵得不可开交；有了RAROC，争论变成了数字之间的比较。A部门说"我的RAROC比你高5个百分点"，那资本就应该优先分配给A部门。这种"用数据说话"的文化，是ERM成熟组织的重要标志。

好了，第二十二章到这里。我们从COSO 2017框架出发，理解了ERM在保险公司的特殊定位，学习了风险偏好和风险限额体系的搭建方法，深入了解了ORSA的年度评估流程，掌握了压力测试和逆情景设计的思路，比较了经济资本和监管资本的差异，最后通过平安和太保的案例看到了ERM最佳实践的落地形态。

ERM是本书目前"最顶层"的视角——它不再关注某一险种的定价或某条产品线的经营，而是站在整个公司的层面审视所有风险的交互作用。它和上一章的偿付能力管理一起，构成了保险公司"守护底线"和"追求上限"的双轮驱动。
