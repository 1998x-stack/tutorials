第六章 CAN控制器硬件：常见芯片与寄存器配置

各位同学，今天我们来聊聊CAN控制器的硬件。说实话，这部分内容在刚入门时容易被忽略——很多人觉得不就是个芯片嘛，接上就能用。但我在项目里吃过不少亏，才明白硬件选型和寄存器配置有多重要。

6.1 主流CAN控制器芯片概览

市面上常见的CAN控制器芯片，我接触过的主要有三类：

  • 独立控制器：SJA1000、MCP2515
  • 集成在MCU内部：STM32的bxCAN、NXP的FlexCAN
  • 带收发器的一体化方案：TJA1043、SN65HVD230

我个人习惯把独立控制器用在需要灵活扩展的场合。比如有一次做车载网关，MCU自带的CAN接口不够用，我就外挂了两个MCP2515，通过SPI扩展。嗯，这里要注意——SPI的速率会直接影响CAN报文的收发效率，别选太慢的MCU。

6.2 SJA1000：经典中的经典

SJA1000是NXP（原Philips）的产品，可以说是CAN控制器的教科书。它支持两种模式：BasicCAN和PeliCAN。BasicCAN是早期标准，现在基本不用了。PeliCAN才是我们常用的，支持11位和29位标识符。

6.2.1 内部寄存器布局

SJA1000的寄存器地址从0x00到0x1F，共32个。我把它分成三组：

地址范围 寄存器组 功能说明
0x00 - 0x0F 控制与状态 模式控制、中断、错误状态
0x10 - 0x1C 验收滤波器 报文ID过滤配置
0x1D - 0x1F 发送/接收缓冲 报文数据暂存

你想想看，如果验收滤波器没配好，总线上的报文会全部进入接收缓冲，CPU根本处理不过来。我曾经在一个项目中，就因为验收滤波器配置错误，导致CAN总线负载率才30%就丢帧了。

6.2.2 关键寄存器配置

配置SJA1000，核心就是几个寄存器：

// 模式寄存器 (MOD) - 地址0x00
// 进入复位模式才能配置
MOD = 0x01;  // 复位模式

// 时钟分频寄存器 (CDR) - 地址0x1F
// 配置CLKOUT输出和CAN模式
CDR = 0x88;  // PeliCAN模式，CLKOUT关闭

// 总线定时寄存器 (BTR0, BTR1)
// 决定波特率，这个我后面会细讲
BTR0 = 0x03;  // SJW=1, BRP=4
BTR1 = 0x1C;  // TSEG1=7, TSEG2=2, SAM=1

// 中断使能寄存器 (IER) - 地址0x04
IER = 0x1F;  // 使能所有中断

// 退出复位模式
MOD = 0x08;  // 正常工作模式
⚠️ 避坑指南
我曾经在配置SJA1000时，忘记先进入复位模式就直接写寄存器，结果芯片死活不工作。查了两天资料才发现——所有配置必须在复位模式下完成，退出复位模式后寄存器就锁定了。

6.3 MCP2515：SPI接口的灵活之选

MCP2515是Microchip的产品，通过SPI接口与MCU通信。它的优势很明显：不占用MCU的CAN外设资源，想扩展几个就扩展几个。

6.3.1 SPI通信协议

MCP2515的SPI指令很简单，就5条：

  • 0x02：写数据到寄存器
  • 0x03：从寄存器读数据
  • 0x05：复位芯片
  • 0x40：加载发送缓冲
  • 0x80：读取接收缓冲

我建议你把这5条指令背下来，调试时经常要用到。有一次我在现场排查问题，示波器抓SPI波形，一眼就看出MCU发的指令是0x02还是0x03，省了不少时间。

6.3.2 内部寄存器结构

MCP2515的寄存器比SJA1000多，地址从0x00到0xFF。但核心的也就那么几个：

// 配置CAN控制寄存器 (CANCTRL) - 地址0x0F
// 模式选择：0x00=正常, 0x20=监听, 0x40=环回
CANCTRL = 0x00;  // 正常模式

// 配置中断使能 (CANINTE) - 地址0x2B
CANINTE = 0x07;  // 使能发送、接收、错误中断

// 配置验收滤波器 (RXF0SIDH) - 地址0x00
// 设置接收ID为0x123
RXF0SIDH = 0x12;
RXF0SIDL = 0x30;  // 注意：标准ID的高3位在低字节

// 配置屏蔽寄存器 (RXM0SIDH) - 地址0x20
// 全匹配，不屏蔽任何位
RXM0SIDH = 0xFF;
RXM0SIDL = 0xE0;
💡 个人经验
MCP2515的验收滤波器配置比SJA1000灵活，但容易搞混。我的习惯是：先配屏蔽寄存器（决定哪些位要比较），再配验收寄存器（决定比较的值）。顺序反了，滤波器可能不生效。

6.4 工作模式详解

不管是SJA1000还是MCP2515，工作模式都差不多。我总结为四种：

6.4.1 正常模式

这是最常用的模式。控制器正常收发报文，参与总线仲裁。说白了，就是车在路上正常跑的状态。

6.4.2 只听模式

只接收不发送。我在做总线监听工具时经常用这个模式。它不会干扰总线，特别适合排查问题。

6.4.3 环回模式

自己发自己收，不经过总线。这个模式用来测试控制器本身是否正常。我记得有一次，客户说CAN发不出去，我切到环回模式一测，发现控制器本身没问题，是收发器坏了。

6.4.4 休眠模式

低功耗模式，用于车载设备的省电设计。唤醒方式有两种：总线活动和外部中断。

6.5 实战：初始化流程

最后，我给出一个通用的初始化流程，SJA1000和MCP2515都适用：

  1. 硬件复位：拉低复位引脚至少5μs
  2. 进入配置模式：设置模式寄存器
  3. 配置波特率：设置BTR0/BTR1或CNF1/CNF2/CNF3
  4. 配置验收滤波器：设置接收ID和屏蔽位
  5. 配置中断：使能需要的中断源
  6. 退出配置模式：进入正常模式
  7. 验证通信：发送一个测试报文，检查是否收到
🔑 核心要点
初始化顺序不能乱。我曾经跳过硬件复位，直接写寄存器，结果芯片状态不确定，折腾了一下午。记住：复位→配置→验证，三步走，稳得很。
