为什么需要在线数据选择

大家好。在之前的章节中，我们花了大量篇幅讨论如何构建一个高质量的静态数据集——从 Common Crawl 采集原始网页，经过层层清洗和过滤，最终产出一份"干净"的训练语料。这个范式隐含了一个假设：一份好的数据，对模型训练的整个阶段都是好的。但 2025-2026 年的研究发现，这个假设并不完全正确。

让我讲一个真实的实验。2025 年，Google DeepMind 的一个团队做了一个很有意思的对比实验。他们准备了两个数据集——A 数据集以高质量教育内容为主，B 数据集以多样化的通用网页为主。用 A 训练的模型在学术基准上表现更好，用 B 训练的模型在对话流畅度上更优。然后他们做了一个混合实验：先让模型用 B 数据训练 30%，再切换到 A 数据训练 70%。结果出人意料——这个"先通用后精调"策略，在学术基准和对话流畅度上都超过了单独使用 A 或 B 的效果。

这个实验揭示了一个重要道理：模型在不同训练阶段需要不同类型的关注。训练早期，模型像一个正在学走路的孩子——需要大量多样化的经验来建立语言的基本感知能力。训练中期，模型开始需要结构化的知识。到了训练后期，模型最需要的是高质量、高难度的样本来"打磨"能力。用一个固定的、静态的数据集来应对所有的训练阶段，本质上是一种次优策略。

这就引出了在线数据选择（Online Data Selection）的核心思想：在训练过程中动态调整数据的采样分布，让模型"在合适的时间看到合适的数据"。

课程学习（Curriculum Learning）

在线数据选择最经典的形式是课程学习（Curriculum Learning）。这个概念在 NLP 领域并不新——早在 2015 年左右就被提出用于机器翻译和语言模型训练——但在 LLM 预训练的语境下，它的系统化应用才刚刚起步。

课程学习在 LLM 预训练中的典型实现分为三个阶段：

  • 第一阶段（~20% 的训练步数）： 以多样化的通用网页数据为主。目标是让模型建立广泛的语言感知——词汇、句法、基本的语义关联。这个阶段不追求深度，追求广度。典型数据来源包括 Wikipedia、Reddit、社交媒体。
  • 第二阶段（~50% 的训练步数）： 逐渐增加中等难度和领域知识的比例。模型开始学习结构化知识——教科书、技术文档、学术论文。数据混合配比向"信息密度更高"的方向倾斜。
  • 第三阶段（~30% 的训练步数）： 以高质量、高难度、专业领域数据为主。这个阶段对应模型能力的"精雕细琢"——用 FineWeb-Edu 风格的高质量筛选数据、专业代码库、多语言并行语料来提升模型在特定任务上的上限。
# 课程学习数据调度伪代码
def curriculum_sampler(training_step, total_steps, data_sources):
    # data_sources: 按难度分级的多个数据源
    # 每个数据源有一个 weight（采样权重）

    progress = training_step / total_steps

    if progress < 0.2:
        # 第一阶段：通用多样化数据为主
        weights = {
            "general_web": 0.7,
            "wikipedia":   0.2,
            "books":       0.05,
            "code":        0.05,
        }
    elif progress < 0.7:
        # 第二阶段：结构化知识比例上升
        weights = {
            "general_web": 0.3,
            "wikipedia":   0.3,
            "books":       0.2,
            "code":        0.2,
        }
    else:
        # 第三阶段：高质量精调数据主导
        weights = {
            "general_web": 0.1,
            "wikipedia":   0.2,
            "books":       0.2,
            "code":        0.3,
            "edu_filtered": 0.2,
        }

    return weighted_sample(data_sources, weights)

注意： 课程学习虽然在直觉上很合理，但在 LLM 预训练中的系统化效果验证仍然是一个开放问题。2025 年的几篇论文显示，精心设计的静态数据混合配比在某些场景下可以取得和课程学习相当的效果。课程学习的最大价值可能不在于"提升最终性能"，而在于"更快达到同等性能"——即加速训练收敛。对于需要训练数万亿 token 的 LLM 来说，哪怕只是 10% 的收敛加速，也意味着数百万美元的算力节省。

基于模型状态的数据选择

课程学习的局限性在于：它的调度规则是预先设定的，并不感知模型的实际学习状态。如果模型在某些领域学得比预期快，课程学习不会自动调整。这促使研究者思考更智能的方案——让数据选择过程感知模型当前的状态。

2026 年，一个名为 FLUX 的系统引发了广泛关注。FLUX 的核心洞察非常简洁而深刻：模型在不同数据上的 loss 变化模式，可以反过来指导数据的选择策略。

具体来说，FLUX 在训练过程中持续跟踪每个数据类别上的 loss 变化趋势：

  • 如果模型在某种数据上的 loss 下降很慢（或几乎不下降），说明模型需要接触更多该类数据来学习——提高这类数据的采样权重。
  • 如果模型在某种数据上的 loss 已经降到了极低水平，说明模型已经"掌握了"这类模式——降低这类数据的采样权重，把宝贵的训练步数让给其他更有挑战的数据。

这种方法的优美之处在于，它不需要对每条数据单独打分，而是基于粗粒度的数据类别进行动态调整。FLUX 采用的类别可以是领域（代码、数学、百科）、难度等级（困惑度区间）、甚至数据来源（如某个网站的爬取内容）。每个类别维护一个滑动窗口的 loss 变化率，系统每隔一定步数重新计算采样权重。

实践洞见： FLUX 的意义不仅在于"提升模型效果"，更在于它打破了"高质量"和"高保留率"之间的传统权衡。传统方法中，如果你想提高数据质量（降低噪声），就必须接受较低的保留率。但 FLUX 的动态调整机制可以在保留大部分数据的同时，通过在线权重分配来达到等效甚至更优的过滤效果。高质量和高保留率可以兼得——关键就在于在线调整的灵活性。

在线选择的工程挑战

在线数据选择虽然在理论上很吸引人，但在落地时面临几个严峻的工程挑战。

计算开销。 评估每条数据对模型的影响是昂贵的。FLUX 的方案通过在类别粒度上跟踪 loss 变化来规避这个问题，但如果想做到更细粒度的选择（比如在文档级别），就需要额外的机制。2026 年的一个主流方案是使用一个小型的 proxy 模型来评估数据质量——例如用一个 1.5B 的模型来为每条数据打分，主训练模型是 70B。proxy 模型的计算开销远小于主模型，而且可以在单独的 GPU 上异步运行。

延迟问题。 在线数据选择不能拖慢主训练进程。这意味着所有选择逻辑必须是轻量级的，并且最好与主训练异步执行。实践中，通常的做法是让数据选择器预先准备好下几轮的采样分布，主训练只需要从预计算好的分布中采样即可。

风险警示： 在线数据选择的最大陷阱是"确认偏误"（Confirmation Bias）。如果选择策略完全基于模型当前的 loss 来调整，就可能产生一个正反馈循环：模型擅长数学推理 → 数学数据的 loss 较低 → 系统降低数学数据的权重 → 模型更少接触数学数据 → 数学能力停滞甚至下降。最终的结果是模型在它已经"擅长"的数据上反复训练，而缺乏探索新领域的机会。这个风险在 FLUX 类系统中尤其需要警惕。缓解方案包括：引入随机探索（类似强化学习中的 epsilon-greedy）、维护每个类别的绝对采样下限、以及人工设置"必须覆盖"的核心数据类别。

复现性挑战。 在线数据选择引入了动态性，使得训练过程的完全可复现变得困难。你无法简单地通过固定一个随机种子来保证每次训练看到相同的数据顺序。对于需要审计的训练项目，这是一个需要认真对待的问题。2026 年的实践方案是将在线选择的决策日志完整记录下来，使得每次训练虽然实际数据不同，但可以事后追溯"模型在每一步看到了什么数据以及为什么"。

2026 年的实践建议

讲完了理论和挑战，最后给出一份 2026 年可落地的实践建议。

推荐方案：混合模式。 不要做纯在线选择（全动态），也不要做纯离线选择（全静态）。业界 2026 年的共识是采用"80% 静态 + 20% 动态"的模式：

  • 80% 静态数据： 通过离线清洗流程（如第 2-13 章所述）准备好一份高质量的静态数据集。这部分数据在训练过程中保持固定的采样分布。它提供了训练的基础稳定性——即使动态部分出了问题，模型依然能在高质量静态数据上有效训练。
  • 20% 动态数据： 保留一部分训练步数用于在线重采样。动态部分可以采用课程学习（预定义调度）或 FLUX（基于模型状态）的策略。这部分提供了训练的灵活性和自适应能力。

更激进的做法（如纯在线选择）虽然在论文中展示了更好的效果，但生产环境的风险较高——一旦动态选择策略出现 bug 或陷入确认偏误，可能整个训练过程都需要回滚。80/20 混合模式在风险可控的前提下，已经可以获得在线选择的大部分收益。
