第18章 信息论与安全

大家好，我是你们的讲师。前几章我们关注的焦点是通信的可靠性——如何在噪声信道中恢复出原始消息。但通信中还有一个同样重要、方向却截然不同的问题：安全性。

如果信道中不仅有噪声，还有窃听者(Eve)呢？你发送的每条消息都被她截获——你该怎么办？一个直觉的反应是"加密"。但这就引出了一个更根本的问题：什么样的加密是'绝对安全'的？

1949年，也就是Shannon发表信息论奠基论文的次年，他发表了另一篇里程碑式的论文——《Communication Theory of Secrecy Systems》。在这篇文章中，Shannon用信息论的概念——熵——完美地回答了这个问题：当且仅当密钥的熵不小于明文的熵时，加密才是完善保密（perfect secrecy）的。

这也解释了为什么一次一密（One-Time Pad, OTP）至今仍是唯一被证明无条件安全的加密方案——但同时，为什么它在实践中如此难以使用。

完善保密性的信息论定义

让我们用信息论的语言来精确描述"安全"。设明文为M，密文为C，密钥为K。直观上，"完善保密"的意思是：看到密文不会给窃听者任何关于明文的信息。

用条件熵来表达：

H(M | C) = H(M)

也就是说，知道密文C之后，明文M的不确定性（熵）和不知道密文时完全一样。另一种等价形式是用互信息：

I(M; C) = H(M) - H(M | C) = 0

明文和密文之间没有互信息——它们统计独立。

Shannon进一步推导出了一个必要条件：

H(K) ≥ H(M)

密钥的熵必须至少等于明文的熵。如果密钥长度小于明文长度（这是大多数实际加密方案的情况），从信息论的意义上说就不可能是"完善保密"的——一定存在某种信息泄漏。

这个定理有一个冰冷而直接的推论：如果密钥比明文短，理论上就存在一种攻击可以获取到关于明文的部分信息。当然，实际的密码学通过将计算复杂性引入等式，绕过了这个限制——但这超出了信息论的范围。

一次一密与Shannon保密定理

一次一密（One-Time Pad）是最简单的加密方案：

c = m ⊕ k

其中m是明文比特串，k是等长的、完全随机且均匀分布的密钥比特串，加密就是按位异或。解密同样：m = c ⊕ k。

我们来验证它是否满足完善保密：

import numpy as np

def generate_key(length):
    """生成均匀随机密钥"""
    return np.random.randint(0, 2, length)

def otp_encrypt(plaintext, key):
    """一次一密加密"""
    return plaintext ^ key

def otp_decrypt(ciphertext, key):
    """一次一密解密"""
    return ciphertext ^ key

# 验证 H(M|C) = H(M)
# 思路：对于任意固定的c，p(m|c) = p(m) —— 因为k均匀随机且与m独立
# 所以 H(M|C) = H(M) 理论上成立
# 这里做数值验证

# 固定明文空间为2比特
all_m = np.array([[0,0], [0,1], [1,0], [1,1]])
n_messages = len(all_m)

# 生成所有可能的密钥
all_k = np.array([[0,0], [0,1], [1,0], [1,1]])

# 对每个明文-密钥对，计算密文
counts = {}  # (c, m) 计数
c_counts = {}  # c 计数
total = 0

for m_bits in all_m:
    for k_bits in all_k:
        c_bits = tuple(otp_encrypt(m_bits, k_bits))
        m_tuple = tuple(m_bits)
        counts[(c_bits, m_tuple)] = counts.get((c_bits, m_tuple), 0) + 1
        c_counts[c_bits] = c_counts.get(c_bits, 0) + 1
        total += 1

# 计算 H(M|C)
import math
h_m_given_c = 0
for c, c_count in c_counts.items():
    p_c = c_count / total
    h_m_given_c_given = 0
    for m_tuple in all_m:
        p_m_given_c = counts.get((c, tuple(m_tuple)), 0) / c_count
        if p_m_given_c > 0:
            h_m_given_c_given -= p_m_given_c * math.log2(p_m_given_c)
    h_m_given_c += p_c * h_m_given_c_given

# 计算 H(M)
h_m = 0
for m_bits in all_m:
    p_m = 1 / n_messages
    h_m -= p_m * math.log2(p_m)

print(f"H(M|C) = {h_m_given_c:.4f} 比特")
print(f"H(M)   = {h_m:.4f} 比特")
print(f"完善保密满足: H(M|C) = H(M)? {abs(h_m_given_c - h_m) < 1e-10}")

理论上的一次一密证明很简洁：对于任意固定的密文c，p(M = m | C = c) = p(K = m ⊕ c | C = c) = p(K = m ⊕ c) = 1/2len = p(M = m)。所以了解c不提供任何关于m的信息。

一次一密有一个著名的真实应用：华盛顿-莫斯科热线。在冷战期间，美苏之间的直接通信线路使用了OTP——载体是磁带卷，由信使物理传递。每次通话前，双方使用同一卷磁带上的同一位置作为密钥。这大概是人类历史上最昂贵的加密方式——但考虑到核威慑的后果，这点成本是可以接受的。

关键警示：一次一密虽然理论上完美，但在实践中极度危险——因为"一次一密"这个名字的含义是：一个密钥只能用一次。如果同一个密钥被用于加密两条不同的消息，那么c1 ⊕ c2 = m1 ⊕ m2，两条明文的信息就会直接暴露。历史上，苏联情报机构的Venona项目正是因为这个原因被破解——他们在冷战期间重用了OTP密钥，导致美国情报机构VA解密了大量苏联间谍通信。

熵在密码学中的角色

除了完善保密的定义之外，熵在密码学中还有更广泛的应用：

  • 密钥强度度量：密钥的熵直接量化了密钥的"不可预测性"。一个128位的均匀随机密钥提供128比特的熵。但如果是用户选择的密码，实际熵远小于密码长度——这也是为什么密码学家建议使用密码管理器生成高熵密码。
  • 最小熵（min-entropy）：对于密码猜测攻击场景，Shannon熵并不总能反映真实的安全性。最小熵H∞(X) = -log maxx p(x)更适合度量最坏情况下的不可预测性。如果某个密码有1%的概率被猜到，它的最小熵大约是6.6比特。
  • 信息泄漏信道：侧信道攻击（时序攻击、功耗分析、电磁泄漏）本质上是信息泄漏的副信道。从信息论视角看，安全系统应该确保所有副信道泄漏的信息量（互信息）接近零。

一个有趣的例子：密码学中的"时间恒定"（constant-time）实现就是为了消除时序副信道中的信息泄漏。如果密钥的某个比特为1时执行了更耗时的操作，那么I(执行时间; 密钥) > 0，熵就会泄漏。

信息论安全的现代发展

Shannon在1949年奠定的信息论安全基础，后来发展出了多个活跃的研究方向。

Wiretap信道（窃听信道模型）：由Wyner在1975年提出。在这个模型中，合法接收者Bob通过主信道接收信号，而窃听者Eve通过一个"更差"的旁路信道接收信号。Wyner证明了：即使Eve能窃听到信号，只要主信道的质量优于窃听信道，就可以实现安全通信而不依赖加密——仅靠信道编码本身。这个"信息论安全"的信道容量被称为保密容量（secrecy capacity）：

Cs = Cmain - Ceve

当窃听信道比主信道差时，存在一种编码方式，使得Bob可以正确译码而Eve得不到任何信息。

差分隐私（Differential Privacy）：这是密码学/信息安全领域近十年最引人注目的概念之一。虽然它不直接源自Shannon信息论，但其核心思想——量化信息泄漏——与信息论深度相通。差分隐私保证：在数据集D中加入或删除一个人的记录，几乎不会改变任何查询结果的分布。这确保了用户级别的隐私——即使攻击者知道其他所有人的数据，也无法推断出特定个人的信息。

安全模型 威胁假设 保证类型 典型构造
完善保密 Eve拥有无限计算能力 I(M;C) = 0 一次一密
计算安全 Eve具有多项式时间计算能力 破解需要不可行计算量 AES、RSA、椭圆曲线
语义安全 Eve具有多项式时间计算能力，选择明文攻击 密文不泄露任何明文信息 概率加密、AES-CBC
信息论安全(Wiretap) Eve信道比Bob信道差 I(M; Eve的接收) → 0 保密编码
差分隐私 攻击者知道所有其他人的信息 单个记录影响有界 拉普拉斯/高斯噪声注入

今天，信息论与安全的交叉依然在蓬勃发展。量子密钥分发（Quantum Key Distribution, QKD）——利用量子力学的不确定性原理来检测窃听——可以被视为Shannon完善保密性在量子域的自然延伸。物理层安全（Physical Layer Security）利用无线信道的空间特性来实现安全通信，是Wiretap信道理论在5G/6G中的应用。

小结与预告

这一章我们看到，信息论不仅关心"能否可靠地通信"，也关心"能否安全地通信"。Shannon的完善保密理论告诉我们：绝对安全是有代价的——密钥至少要和消息一样长。这个结论既是密码学的'不可能性定理'，也是一次一密始终在历史上占有特殊地位的原因。

从下一章开始，我们将进入信息论的另一个重要分支：在"完美不失真"的约束之外，如果我们允许一定程度的失真呢？毕竟，你在手机上看视频时，每个像素与原画面的些许误差是你几乎注意不到的——但数据量可以减少十倍。这就是率失真理论（Rate-Distortion Theory）的故事。
