第11章 数据增强策略

大家好，我是你们的讲师。你可能不会想到，YOLOv4最大的创新不是网络结构，而是数据增强。Mosaic让模型学会了在更小的视野内检测物体——这就是为什么同样的backbone加上Mosaic就能涨点2-3%。在数据量有限的情况下，一个精心设计的数据增强策略，往往比改网络结构更有效。

数据增强不仅是为了"增加数据量"，更关键的是引入有意义的多样性——让模型见过更多样化的场景，从而提升泛化能力。

11.1 基础几何增强

基础增强是所有目标检测训练管线中最基本的组成部分。它们实现简单、计算开销小，但效果显著：

增强方法 实现方式 适用场景 注意事项
随机水平翻转 以50%概率水平镜像图像 几乎所有场景 对文本检测可能有负面影响
随机裁剪 随机裁剪图像的一部分 部分遮挡、局部细节 裁剪后的框需要重新计算坐标
随机缩放 在[0.5, 1.5]范围内随机缩放 多尺度训练 锚框需要覆盖更广的尺度范围
随机旋转 ±10°以内的小角度旋转 非水平场景 大角度旋转会引入黑色填充区域
HSV调整 随机改变色调(H)、饱和度(S)、明度(V) 光照变化大的场景 变化幅度不宜过大，否则颜色失真
灰度化 将图像转为灰度 模拟低质量监控画面 可能丢失颜色关键特征 
# YOLOv5中HSV增强的典型参数
hsv_h = 0.015   # 色调扰动范围
hsv_s = 0.7     # 饱和度扰动范围
hsv_v = 0.4     # 明度扰动范围

# YOLOv5中几何增强的典型参数
degrees = 0.0   # 旋转角度（通常设为0）
translate = 0.1 # 平移比例
scale = 0.5     # 缩放比例
shear = 0.0     # 剪切变换角度

11.2 Mosaic增强

Mosaic增强是YOLOv4提出的最具影响力的数据增强策略。它将4张训练图像拼接成一张大图像，送入网络进行训练：

def mosaic(image1, image2, image3, image4, img_size=640):
    """将4张图拼成一张mosaic图像"""
    # 随机选择拼接中心点
    cx = np.random.randint(img_size // 2, img_size)
    cy = np.random.randint(img_size // 2, img_size)

    # 将4张图分别放在左上、右上、左下、右下
    positions = [
        (cx, cy, img_size, img_size),           # 左上
        (cx - img_size, cy, img_size, img_size), # 右上
        (cx, cy - img_size, img_size, img_size), # 左下
        (cx - img_size, cy - img_size, img_size, img_size), # 右下
    ]

    # 初始化画布

Mosaic为什么有效？

  • 上下文多样性： 一张图中出现4种不同的背景，模型被迫学习在各种上下文中识别物体
  • 小目标增强： 拼接后，每张子图的物体相对尺寸变成了原来的1/2-1/4，相当于强制模型"看见"更小的物体
  • BN层稳定性： 4张完全不同来源的图像在一个batch中拼接，极大地丰富了batch统计量的分布，使Batch Normalization更稳定
实际经验： YOLOv4/v5的官方配置中，Mosaic在训练的前3/4阶段开启，后1/4阶段关闭。原因是Mosaic生成的图像与真实场景差异较大（毕竟正常人不会把4张图拼在一起拍照），持续使用Mosaic训练可能会导致模型在真实分布上过拟合不足。后1/4阶段关闭Mosaic、只使用基础增强，能让模型更好地适应自然图像的分布。

11.3 MixUp与CutMix

MixUp和CutMix是图像分类领域提出的增强策略，后来被YOLOv4和YOLOv5引入到目标检测中。

MixUp

MixUp将两张图像按照一定比例叠加（像素级混合），标签也按照相同比例混合：

mixed_image = λ * image1 + (1 - λ) * image2
mixed_labels = λ * labels1 ∪ (1 - λ) * labels2

# 其中λ ~ Beta(α, α)，通常α=0.2-0.8

在目标检测中，MixUp的标签混合策略是将两张图的所有标注框都保留，但置信度权重分别乘以λ和(1-λ)。这种"软标签"策略提供了更丰富的训练信号。

CutMix

CutMix在MixUp的基础上做了改进——不是像素级混合，而是区域级替换：从一张图中裁剪一块区域，粘贴到另一张图的对应位置：

# CutMix操作
# 1. 从image1中随机裁剪一个矩形区域
# 2. 用image2中对应区域的内容替换
# 3. 标签：使用image1的标签，但如果框位于裁剪区域内，则替换为image2的标签

11.4 自对抗训练（SAT）

YOLOv4引入了自对抗训练（Self-Adversarial Training，SAT）——这是一个让模型自己"找茬"的训练范式。SAT分为两个阶段：

第一阶段（对抗阶段）： 网络冻结检测头，仅更新backbone的权重。但这里的"更新"不是最小化损失，而是最大化损失——让网络生成一个对检测"最具欺骗性"的对抗样本。这个对抗样本对原始图像施加了细微的、有针对性的扰动，使得检测器最容易出错。

第二阶段（训练阶段）： 使用对抗样本作为输入，正常训练整个网络（检测头也参与训练）。模型必须在更难识别的图像上完成检测任务。

SAT的效果： 在YOLOv4的消融实验中，加入SAT后AP提升了约1.5%。更重要的是，SAT训练的模型对噪声、模糊等常见图像退化具有更强的鲁棒性。虽然SAT增加了约30%的训练时间，但推理时没有任何额外开销——它是一种纯训练时的正则化策略。

11.5 Copy-Paste增强

Copy-Paste是近年来比较流行的一种实例级增强方法。它的核心思想非常简单：从一张图中"复制"出物体实例，然后"粘贴"到另一张图像中。

Copy-Paste对于以下场景特别有效：

  • 长尾分布： 罕见类别的样本可以通过Copy-Paste生成更多实例
  • 密集场景： 通过粘贴更多物体模拟拥挤场景，提升模型在密集场景下的表现
  • 遮挡鲁棒性： 粘贴的物体会部分遮挡原有物体，迫使模型学会基于局部特征推理

11.6 实用增强管线推荐

基于YOLOv5/v8的实践经验，以下是一个推荐的数据增强配置：

阶段 增强策略 应用概率 说明
训练前75% Mosaic + MixUp Mosaic: 100%, MixUp: 50% 强增强阶段，快速学习特征
训练后25% HSV + 翻转 + 缩放 HSV: 100%, 翻转: 50%, 缩放: 100% 关闭Mosaic，逼近真实分布
全程可选 Copy-Paste 10-20% 长尾分布场景强烈推荐
踩坑实录：Mosaic调大了导致训练不稳定。 一次项目中，我为了追求数据多样性，将Mosaic从4张拼接改成了6张拼接，同时加大了随机裁剪的比例。结果训练loss在前20个epoch中剧烈震荡，迟迟不收敛。排查发现：6张图拼接导致每个子图的信息量太少（只有原图的1/6），目标被严重截断，大部分标注框都变成了无效的小碎片。最终回到标准的4张Mosaic，loss才恢复正常。数据增强不是越多越好，关键是要保持语义完整性。

11.7 总结

数据增强是YOLO训练管线中性价比最高的优化手段之一。从基础的HSV抖动、翻转，到YOLOv4标志性的Mosaic增强，再到Copy-Paste实例级增强，每一步都在提升模型对多样性的适应能力。一个合理的增强管线，可以在不改变任何网络结构的情况下带来3-5%的AP提升。
