开篇：一个工厂长的烦恼

大家好，我是你们的讲师。这一章我们来聊聊线性规划。

让我从一个亲身经历的故事讲起。2018年，我帮一家小型家具厂做生产计划优化。厂长很困扰：他们有三个车间——木工车间、喷漆车间、组装车间——每种家具（桌子、椅子、柜子）需要的工序时间各不相同。木材每周只有固定配额，油漆也有限。厂长想知道：怎么安排生产才能使利润最大化？

他之前的做法是"经验排产"：优先做利润最高的产品，剩下的产能再安排其他。结果经常出现原材料用完了但高利润产品做不了多少、低利润产品也没法补上的尴尬局面。厂长说："明明感觉产能是够的，但月底一盘算，利润总上不去。"

了解了他的情况后，我花了一个下午把他的问题建成了一个线性规划（Linear Programming, LP）模型。决策变量是每种家具的生产数量；目标函数是总利润；约束条件包括木材配额、油漆配额、各车间的工时上限。然后用单纯形法求解——不到0.1秒就找到了最优方案。

执行新方案后，工厂的月利润提升了23%，而且原材料利用率从71%跃升到了93%。厂长很惊讶："你不是算出来的？你是'变'出来的吧？"我说："不是变，是数学。"

这个故事想表达的是：线性规划不是高高在上的理论，它是直接可以"变现"的工具。在组合优化中，LP 是基础中的基础——整数规划、分支定界、对偶理论都建立在 LP 之上。

LP 是什么

形式化定义

线性规划（Linear Programming）是在一组线性等式/不等式约束下，优化一个线性目标函数的问题。它的标准数学形式是：

最大化（或最小化） c₁x₁ + c₂x₂ + ... + cₙxₙ

满足：
    a₁₁x₁ + a₁₂x₂ + ... + a₁ₙxₙ ≤ b₁
    a₂₁x₁ + a₂₂x₂ + ... + a₂ₙxₙ ≤ b₂
    ...
    aₘ₁x₁ + aₘ₂x₂ + ... + aₘₙxₙ ≤ bₘ
    x₁, x₂, ..., xₙ ≥ 0

用家具厂的案例来理解

家具厂问题可以建模如下：

  • 决策变量：x₁ = 桌子数量，x₂ = 椅子数量，x₃ = 柜子数量
  • 目标：max 利润 = 500x₁ + 200x₂ + 800x₃（单位：元）
  • 木材约束：10x₁ + 3x₂ + 15x₃ ≤ 500（单位：m²）
  • 木工工时：4x₁ + 2x₂ + 6x₃ ≤ 160（单位：小时）
  • 喷漆工时：2x₁ + 1x₂ + 3x₃ ≤ 80
  • 组装工时：1x₁ + 1x₂ + 2x₃ ≤ 50

几何直觉

线性规划有一个极其重要的几何性质：可行域是一个凸多面体（Convex Polytope），最优解一定在某个顶点上。

想象二维的情况：每个不等式对应一条直线，可行域是所有直线围成的多边形。把目标函数想象成一条斜率固定的直线——你在可行域内平移这条直线，直到不能再往目标方向移动了。最后的接触点，就是一个顶点。

这个"最优解在顶点"的性质，就是单纯形法能高效工作的理论基础——你只需要检查有限的几个顶点，而不是无穷多个可行点。

一个关键洞察：线性规划的"线性"二字既是优点也是限制。优点是：线性约束定义了一个凸集，凸集上的线性函数极值一定在边界上，这大大简化了求解。限制是：现实世界的很多关系是非线性的——例如固定成本（要么为零要么为一个固定值）就无法用线性函数表示。这就是为什么我们需要整数线性规划（第12章）来处理离散决策。但先学好 LP，因为它是一切的基础。

LP 的标准型

为了统一分析和算法设计，线性规划有一个标准型（Standard Form）：

max  cᵀx
s.t. Ax ≤ b
     x ≥ 0

其中 x ∈ ℝⁿ 是决策向量，c ∈ ℝⁿ 是目标系数向量，A ∈ ℝᵐˣⁿ 是约束系数矩阵，b ∈ ℝᵐ 是右端项向量。

任意 LP 都可以转化为标准型：

原始形式 转换为标准型的方法
min cᵀx 等价于 max (-cᵀx)
aᵀx ≥ b 两边乘 -1 得到 (-a)ᵀx ≤ -b
aᵀx = b（等式约束） 拆为两个不等式：aᵀx ≤ b 且 aᵀx ≥ b
x 无符号限制（free variable） 拆为 x⁺ - x⁻，其中 x⁺ ≥ 0, x⁻ ≥ 0

你可能觉得这些变换很繁琐，但它们是算法实现的基础。实际使用求解器时（Gurobi / SCIP），求解器会自动做这些转换，但理解转换过程有助于你调试模型和解读求解器输出的诊断信息。

小练习：试着将家具厂问题转化为标准型。注意目标已经是 max，所有约束都是 ≤，变量都是非负的……实际上家具厂问题已经天然符合标准型！但在更复杂的模型中，你可能需要对变量做拆分或对约束做转换。建议你拿一个 min 型 + 含等式约束的例子练练手。

单纯形法

单纯形法（Simplex Method）由 George Dantzig 在1947年发明，是求解线性规划最经典也最广泛使用的算法。虽然理论上是"指数时间"的（某些构造出的反例需要指数次迭代），但在实际问题中它的表现极好——平均迭代次数是 O(m) 到 O(2m) 级别（m 是约束数量）。

顶点 = 基本可行解

标准型 LP 中，我们把不等式加上松弛变量（Slack Variables）变成等式：

原始：Ax ≤ b, x ≥ 0
加入松弛变量 s ≥ 0：Ax + s = b, x ≥ 0, s ≥ 0

现在我们有 n+m 个变量（n个原始 + m个松弛），m个等式约束。
一个基本可行解（Basic Feasible Solution）就是选择 m 个基变量，
令其余 n 个非基变量为 0，求解 m×m 方程组得到的解。

每个基本可行解对应可行域的一个顶点。

Pivoting 操作

单纯形法的核心是一个转轴（Pivot）操作：

  1. 选择一个非基变量"进入"基（使其从0变为正值，改善目标函数）。
  2. 选择一个基变量"离开"基（使其从正值变为0，保持可行性）。
  3. 更新基矩阵，重新计算基本可行解。
  4. 重复直到所有非基变量的"检验数"（Reduced Cost）都不大于0——此时已找到最优解。

你可以把单纯形法想象成：从一个顶点出发，沿着一条边走到相邻的更好的顶点，直到到达最优顶点。每一次 pivot 就是从当前顶点走到一个更好的相邻顶点。

单纯形表

单纯形表是以表格形式组织当前基变量、非基变量、目标函数值等信息的方式：

初始单纯形表示例（家具厂问题简化版）：

基变量 | x₁  x₂  x₃  s₁  s₂  s₃  s₄  | RHS
-------+----------------------------+-----
s₁     | 10   3  15   1   0   0   0 | 500
s₂     |  4   2   6   0   1   0   0 | 160
s₃     |  2   1   3   0   0   1   0 |  80
s₄     |  1   1   2   0   0   0   1 |  50
-------+----------------------------+-----
z      |500 200 800   0   0   0   0 |   0

最下面一行是检验数（Reduced Cost），全部非正时达到最优。
当前解：x₁=x₂=x₃=0（非基），s₁=500, s₂=160, s₃=80, s₄=50，目标值=0

通过一系列 pivot 操作更新表格，最终得到：

最优单纯形表（假设结果）：

基变量 | x₁  x₂  x₃  s₁  s₂  s₃  s₄  | RHS
-------+----------------------------+-----
x₃     | ... ... ... ... ... ... ... |  20
x₂     | ... ... ... ... ... ... ... |  30
s₁     | ... ... ... ... ... ... ... |  20  (松弛=20，木材剩余)
s₄     | ... ... ... ... ... ... ... |   0
-------+----------------------------+-----
z      |  0 -10   0   0 -50 -40   0  |22000

最优解：x₂=30, x₃=20, x₁=0，总利润=22,000元
检验数全部 ≤ 0，达到最优！
历史趣闻：Dantzig 的单纯形法在1947年发表后，虽然实践中表现优异，但直到1972年 Klee 和 Minty 才构造出第一个需要指数次迭代的反例（Klee-Minty 立方体）。也就是说，单纯形法"风光"了25年之后才被发现理论上的弱点。但即便如此，它在实际问题中的平均性能仍然无与伦比——这也是组合优化中一个反复出现的主题：最坏情况分析和实际性能往往大相径庭。

对偶理论入门

每一个线性规划问题都有一个与之配对的对偶问题（Dual Problem）。对偶不是数学游戏——它提供了深刻的洞察，我将在第5章详细展开。这里我们先建立基本概念。

对偶的构造规则

给定一个标准型的"原始"LP：

原始问题（Primal）：
max cᵀx
s.t. Ax ≤ b
     x ≥ 0

对偶问题（Dual）：
min bᵀy
s.t. Aᵀy ≥ c
     y ≥ 0

可以看到，原始问题是最大化，对偶就是最小化；原始是"≤"约束，对偶就是"≥"约束；原始的目标系数 c 变成了对偶的约束右端项，原始的约束右端项 b 变成了对偶的目标系数。

弱对偶定理

弱对偶定理（Weak Duality）：如果 x 是原始问题的可行解，y 是对偶问题的可行解，那么 cᵀx ≤ bᵀy。

也就是说，原始问题的任何可行解的目标值 ≤ 对偶问题任何可行解的目标值。对于最大化问题，对偶提供了上界（upper bound）。

强对偶定理

强对偶定理（Strong Duality）：如果原始问题有最优解 x*，那么对偶问题也有最优解 y*，且 cᵀx* = bᵀy*。

这意味着在最优解处，原始目标值 = 对偶目标值。这个性质极为强大——它意味着你既可以通过求解原始问题来得到答案，也可以通过求解对偶问题来得到答案，而且答案是等价的。

直觉理解：原始问题的每个约束对应一个对偶变量 yᵢ。这些 yᵢ 可以理解为"第 i 种资源的价格"，因此 bᵀy 就是所有资源的总"价值"。弱对偶说：任何可行生产方案的价值 ≤ 所有资源的总价值。强对偶说：在最优生产方案下，两者相等。这个"影子价格"的解释，下一章我们会深入讨论。

互补松弛条件

互补松弛条件（Complementary Slackness）进一步描述了原始和对偶最优解之间的精确关系：

设 x* 是原始最优解，y* 是对偶最优解，s* = b - Ax* ≥ 0 是原始松弛变量。
那么：
    对于所有 j：  x*ⱼ > 0  ⇒  (Aᵀy*)ⱼ = cⱼ
    对于所有 i：  y*ᵢ > 0  ⇒  (Ax*)ᵢ = bᵢ

换句话说：
    - 如果某个原始变量 xⱼ > 0，则对应的对偶约束是紧的（等号成立）
    - 如果某个对偶变量 yᵢ > 0，则对应的原始约束是紧的（等号成立）
    - 反之亦然：不紧的约束对应的变量必须为 0

互补松弛在组合优化中应用广泛——它是很多近似算法设计、验证最优性的核心工具。例如，我们可以猜测一组解，然后用互补松弛条件验证它是否最优。

用法示例：假设我声称家具厂问题的最优解是 x₁=0, x₂=30, x₃=20，利润=22,000。怎么验证？利用互补松弛：x₂>0 且 x₃>0，所以对应的两个对偶约束必须是紧的。我还可以检查哪些原始约束是紧的——如果木材约束不紧（松弛 > 0），则木材的对偶变量 y₁=0。这样就建立了一个关于 y 的线性系统，解出 y 后检查是否对偶可行。如果对偶可行，则 x 和 y 都是一组最优解。这是一个非常优雅的验证方法。
