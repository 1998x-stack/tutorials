第15章 TTS评测体系

大家好，我是你们的讲师。想象一下这样一个场景：你花了两周时间训练了一个新的TTS模型，满心期待地打开生成的音频——听起来还不错！但你老板问了一句："比之前的好在哪？好多少？"你一时语塞。你觉得好，可"好"是个主观感受，怎么用量化的方式告诉别人你的模型确实更优秀？

这一章我们来聊聊TTS评测体系。你可能觉得评测是件枯燥的事——搞个表格贴上MOS分数不就完了吗？但事实上，TTS评测的水远比大多数人想象的深。同样的模型，换一批听评者、换一组测试文本、甚至换一个评测环境，MOS分数都能差出0.5分以上。而0.5分在TTS领域，可能就意味着"显著优于人类"和"还不如拼接合成"的差距。

怎么判断一个声音"好听"？

TTS系统的评测可以分为两大类：主观评测和客观评测。主观评测让真人来听，凭感觉打分；客观评测用算法计算数值指标。理想情况下两者应该一致，但现实往往比这复杂——有时候客观指标很好的模型，听起来反而有问题，反之亦然。

这就引出了TTS评测的第一条原则：最终衡量标准永远是人的听感。客观指标可以作为快速筛选的辅助工具，但要判断一个TTS系统好不好，最终还得让人来听。

主观评测：MOS评分

MOS（Mean Opinion Score，平均意见得分）是TTS领域最经典的评测方式。它遵循ITU-T P.800标准，采用5分制：

评分 等级 含义
5 Excellent（优秀） 完全自然，无法区分是机器还是真人
4 Good（良好） 听起来自然，但偶有不完美
3 Fair（一般） 可以识别为机器语音，但不影响理解
2 Poor（较差） 明显不自然，理解有困难
1 Bad（很差） 极不自然，几乎无法听懂

MOS评测的流程看起来简单——找20-30个人，让他们听一批音频样本，每人给1-5分，最后取平均。但实际操作中有很多坑需要注意：

首先，评测环境需要控制。听评者应该在安静的环境中使用同一副耳机，音量也应当保持一致。如果在嘈杂的咖啡馆或者用笔记本电脑的外放喇叭听，分数的方差会急剧增大。

其次，文本选择非常关键。如果只选简单的短句（比如"今天天气真好"），所有模型的分数都会偏高，拉不开差距。好的评测集应该包含各种难度——长句、数字、人名、中英文混合、情感表达，甚至故意加入一些不常见的地名。

第三，听评者的筛选很重要。理想情况下，应该先给听评者做一次"校准"——播放几个已知质量的样本（一个4.5分的真人和一个2.5分的拼接合成），确保他们能正确区分好坏。如果某个人给真人录音打了2分，这个人的评测结果就应该被剔除。

下面是MOS评分计算的一个简化实现：

import numpy as np
from scipy import stats

def calculate_mos(scores):
    """
    计算MOS分数及置信区间

    Args:
        scores: 所有听评者对单个样本的评分列表
    Returns:
        dict: 包含均值、标准差、95%置信区间
    """
    scores = np.array(scores)
    n = len(scores)
    mean = np.mean(scores)
    std = np.std(scores, ddof=1)
    se = std / np.sqrt(n)  # 标准误

    # 95%置信区间
    t_val = stats.t.ppf(0.975, df=n-1)
    ci_low = mean - t_val * se
    ci_high = mean + t_val * se

    # 去除异常值（3标准差之外）
    clean = scores[np.abs(scores - mean) < 3 * std]

    return {
        "mos": round(mean, 3),
        "std": round(std, 3),
        "ci_95": (round(ci_low, 3), round(ci_high, 3)),
        "valid_responses": n,
        "after_outlier_removal": len(clean)
    }

# 示例：某模型在30个听评者上的评分
synthetic_scores = [4, 5, 4, 3, 4, 5, 4, 4, 3, 5,
                    4, 4, 5, 4, 3, 4, 4, 5, 4, 4,
                    3, 4, 5, 4, 4, 3, 4, 4, 5, 4]
result = calculate_mos(synthetic_scores)
print(f"MOS: {result['mos']} ± {result['std']}")
print(f"95% CI: [{result['ci_95'][0]}, {result['ci_95'][1]}]")
# 输出: MOS: 4.0 ± 0.64
# 95% CI: [3.76, 4.24]

注意到置信区间了吗？即使有30个听评者，95%置信区间也跨度接近0.5分。这意味着如果一个模型得了4.0分、另一个得了4.3分，它们之间可能根本没有统计显著的差异。

CMOS与AB测试

MOS虽然经典，但有一个严重的缺陷：不同评测之间的分数不可比。你今天在实验室里测的MOS 4.2，和上周测的MOS 4.0，差的0.2分到底是因为模型变好了、评测人群变了、还是测试文本不同了？你根本说不清楚。

为了解决这个问题，研究者引入了CMOS（Comparison Mean Opinion Score，比较意见得分）。CMOS让听评者直接对比两个系统A和B，判断"A比B好多少"：

评分 含义
+3 A比B好很多
+2 A比B好一些
+1 A比B好一点点
0 A和B差不多
-1 B比A好一点点
-2 B比A好一些
-3 B比A好很多

CMOS的优势在于：它强迫听评者做出相对判断，而不是绝对判断。人对"好坏"的绝对标准差异很大，但人对"哪个更好"的相对判断通常比较一致。这就消除了不同个体之间打分尺度的差异。

AB测试是比CMOS更简单的形式——不问"好多少"，只问"哪个好"或"哪个更像真人"。有时还会引入ABX测试：先给听评者听一段参考音频（X），然后播放两个样本（A和B），问"哪个和X更相似"。这在说话人相似度评测中非常常用。

实践建议：在工业界的A/B测试中，推荐采用"三明治"呈现方式——A-B交替播放，且每对音频之间加入一段静音（约500ms），让听评者有足够的时间做出判断。评测页面最好设计成"盲测"形式（即隐藏A/B的身份标签），避免先入为主的偏见。

不管用哪种方法，统计显著性检验都是必不可少的。常用的方法包括配对t检验（用于MOS对比）和Wilcoxon符号秩检验（用于AB偏好测试的非参数检验）。一个简单规则：如果p值大于0.05，就说明两个系统的差异不能被认为是可靠的。

客观指标：MCD与频谱距离

主观评测虽然准确，但成本高、周期长。在模型开发过程中，研究者频繁地调整参数、修改架构、重新训练，不可能每次改动都找人听一遍。这时候就需要客观指标来做快速评估。

MCD（Mel Cepstral Distortion，梅尔倒谱距离）是最常用的TTS客观指标。它衡量的是合成语音的梅尔倒谱系数（MCEP）和自然语音之间的差异：

# MCD计算公式
# MCD = (10 / ln(10)) * sqrt(2 * sum((mcep_synth - mcep_natural)^2))

import numpy as np

def compute_mcd(mcep_synth, mcep_natural):
    """
    计算Mel Cepstral Distortion

    Args:
        mcep_synth: 合成语音的MCEP特征，形状 [T, D]
        mcep_natural: 自然语音的MCEP特征，形状 [T, D]
    Returns:
        MCD值（dB），越小越好
    """
    diff = mcep_synth - mcep_natural
    squared = diff ** 2
    frame_mcd = (10.0 / np.log(10)) * np.sqrt(2 * np.sum(squared, axis=1))
    return np.mean(frame_mcd)

# 典型值参考
# 真人录音之间的MCD:  ~2.0 dB
# 优秀TTS模型:        ~3.0 dB
# 可接受TTS模型:      ~5.0 dB
# 较差的TTS:          >7.0 dB

除了MCD之外，还有一些其他客观指标值得了解：

指标 全称 衡量内容 典型值范围
MCD Mel Cepstral Distortion 频谱包络的差异 2.0 - 8.0 dB（越小越好）
F0 RMSE F0 Root Mean Square Error 基频（音高）的准确性 10 - 50 Hz（越小越好）
V/UV Error Voiced/Unvoiced Error 清浊音判断错误率 1% - 10%（越小越好）
STOI Short-Time Objective Intelligibility 语音可懂度 0.5 - 1.0（越大越好）
PESQ Perceptual Evaluation of Speech Quality 感知语音质量 1.0 - 4.5（越大越好）

需要注意的是，客观指标和主观听感之间并非完美的线性关系。有些情况下MCD下降了0.5dB但听感反而变差了，因为MCD只衡量了频谱的全局差异，而人类的听觉系统对一些局部问题（如某个音素的发音错误）特别敏感，这些问题可能在MCD中只贡献了微不足道的数值。

说话人相似度

在多说话人TTS和语音克隆场景中，除了语音的自然度，还需要衡量合成语音听起来像不像目标说话人。这就引出了说话人相似度（Speaker Similarity）的评测。

最常用的方法是使用预训练的说话人嵌入模型（如ECAPA-TDNN或ResNetSE）来提取说话人嵌入向量，然后计算余弦相似度：

import numpy as np

def speaker_similarity(emb1, emb2):
    """
    计算两个说话人嵌入向量的余弦相似度

    Args:
        emb1: 参考语音的嵌入向量
        emb2: 合成语音的嵌入向量
    Returns:
        相似度 [-1, 1]，越大越相似
    """
    dot = np.dot(emb1, emb2)
    norm = np.linalg.norm(emb1) * np.linalg.norm(emb2)
    return dot / norm

def secs(reference_embs, synthetic_embs):
    """
    SECS (Speaker Embedding Cosine Similarity)
    计算多对参考-合成样本的平均相似度
    """
    scores = [speaker_similarity(r, s)
              for r, s in zip(reference_embs, synthetic_embs)]
    return np.mean(scores)

# 典型值：
# 同一个人不同录音之间: 0.85 - 0.95
# 优秀克隆:             0.75 - 0.85
# 可接受克隆:           0.60 - 0.75
# 不同说话人之间:       0.10 - 0.40

SECS（Speaker Embedding Cosine Similarity）是目前最常用的自动相似度指标，但它也有局限性——说话人嵌入模型本身的性能会影响评测结果。如果嵌入模型对某些音色特征不敏感，相似度分数就可能虚高。

因此，工业界通常采用"自动指标 + 人工ABX测试"的组合策略：先用SECS做快速筛选，筛选出的候选模型再用人工ABX测试（让听评者判断"哪个更像目标说话人"）做最终判定。

TTS Arena与排行榜

2024年，Hugging Face推出了TTS Arena——一个众包评测平台，类似于语音领域的Chatbot Arena。任何人可以上传自己的TTS音频或在线评测已有模型，系统采用Elo评分（和棋类比赛一样的机制）来计算模型排名。

TTS Arena的出现是一个重要的里程碑。它解决了传统评测的几个顽疾：

  • 大规模：参与人数越多，统计越可靠。TTS Arena有数千名自愿参与评测的用户。
  • 多样性：评测者来自不同地区、不同语言背景，可以暴露模型在特定口音或语言上的弱点。
  • 动态更新：新模型随时可以加入对比，排名实时变化。
  • 公开透明：所有评测数据公开，任何人都可以复现排名。

但众包评测也有其固有缺陷：评测环境不可控（有人用手机外放、有人用专业监听耳机）、评测者背景不统一（语言学习者可能对某些语音特征过于敏感或过于迟钝）、可能存在刷分或恶意低分的情况。因此TTS Arena的排名可以作为参考，但不能作为学术论文的唯一支撑。

值得关注：如果你去翻看2024-2025年的TTS Arena排名，会发现一个有趣的现象——客观指标最好的模型（MCD最低、SECS最高）往往不在排行榜前列，而拥有更好韵律控制和情感表达的模型反而更受青睐。这说明TTS评测正在从"音质"的单一维度向"表现力"的多维度方向演进。

评测的陷阱

从事TTS评测这么多年，我见过太多因为评测不当而得出误导性结论的案例。这里分享两个最常见的问题：

陷阱一：MOS分数膨胀。2017年Tacotron 2的论文报告的MOS是4.53分，2024年许多新模型的MOS已经逼近4.6甚至4.7。但问题是，这些分数是在不同评测条件下获得的——评测文本越来越简单、听评者越来越熟悉合成语音、评测环境越来越好。实际上，如果你用同样严格的评测条件重新测试Tacotron 2，它可能仍然能拿到4.4分以上。分数膨胀让跨论文的MOS对比几乎失去了意义。

陷阱二：文本选择偏差。有一个著名的故事：某公司声称自己的TTS模型MOS达到了4.6分，"超越人类录音水平"。但仔细一看，他们的评测集只包含5个句子，全都是"今天天气真好"这种简单句。更离谱的是，这5个句子还出现在了训练集中。当用包含数字、专有名词、长句的完整评测集测试时，这个模型的MOS直接掉到了3.8分。

教训：TTS评测是一件极其严肃的事情。一篇负责任的TTS论文应该做到：①评测集公开且包含多样化的文本；②报告置信区间或至少报告标准差；③同时报告主观和客观指标；④和基线系统在相同的评测条件下对比。如果一篇论文的MOS分数没有置信区间、只评测了20个样本、而且评测文本没有公开，那这个分数的可信度就要打一个大大的问号。

总的来说，TTS评测是一个"易学难精"的领域。入门很容易——找几个人听一下打个分就完事了。但要做出可靠、可复现、有说服力的评测，需要仔细设计每一个环节：文本选择、听评者筛选、环境控制、统计分析、结果呈现。好的评测应该像一面显微镜，既能看清模型的全貌，也能发现其中微小的缺陷。
