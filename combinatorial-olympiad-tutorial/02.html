第2章：组合数与恒等式

大家好，我是你们的讲师。上一章我们学习了排列——那些"顺序不同就算不同结果"的计数问题。但现实中有大量的问题并不关心顺序：选5个人组成委员会，张三在前在后有什么关系？从一副牌里拿5张，先拿后拿不都是那5张牌吗？这就引出了我们今天的主题——组合数。

如果说排列是"有序选取"，那么组合就是"无序选取"。但从排列到组合，并不是简单地"去掉顺序"就完事了。围绕组合数展开了一整套恒等式体系——Pascal恒等式、朱世杰恒等式、各类求和公式——它们不仅是竞赛解题的利器，更揭示了组合数学内在的对称与和谐之美。今天我们就来系统地梳理这些内容。

2.1 组合数的三种定义视角

组合数 C(n,k)（也记作 nCk 或 (kn)）表示从 n 个不同的元素中选出 k 个组成一组（不考虑顺序）的方案数。理解它可以从三个不同的视角切入，每个视角都有其独特的价值。

2.1.1 子集视角

这是最直观的理解：一个 n 元集合有多少个 k 元子集？答案就是 C(n,k)。这个视角的好处是——你可以把很多组合问题转化成"子集计数"问题，而子集有天然的包含关系，便于用容斥原理等方法处理。

2.1.2 排列约化视角

排列 P(n,k) 是有序的选取。如果我们先选 k 个元素（组合），然后再把它们排成一排（k! 种排法），就得到了排列。所以：

P(n,k) = C(n,k) × k!

由此得：
C(n,k) = P(n,k) / k!
       = n×(n-1)×...×(n-k+1) / k!
       = n! / (k! × (n-k)!)

这就是组合数的核心公式。这个推导过程也直观地说明了为什么组合数比排列数小——因为顺序不同在组合中被视为相同的。

2.1.3 对称性视角

从公式中可以立即得到一个重要的性质：

C(n,k) = C(n, n-k)

为什么？因为从 n 个元素中选出 k 个，等价于选出 n-k 个"不选"的元素。选哪些人入委员会，就是在决定哪些人"不入委员会"。这个对称性看似简单，但在很多恒等式证明中能发挥意想不到的作用。

💡 对称性的妙用：当你看到一个求和式中有 C(n,k) 出现时，试试把 k 换成 n-k。很多看起来不对称的式子，一换就对称了，求和范围也可以重新理解。比如，∑_{k=0}^{n} C(n,k) 的一半项由对称性自动等于另一半。

2.2 Pascal恒等式与杨辉三角

如果说组合数学有一个"最核心的恒等式"，那非 Pascal 恒等式莫属。它揭示了组合数之间的递推关系：

C(n,k) = C(n-1, k-1) + C(n-1, k)

这个公式的证明非常直观：从 n 个元素中选 k 个，我们可以按"第一个元素是否被选中"来分类：

  • 如果选了第一个元素，那么还需要从剩下的 n-1 个中选 k-1 个：C(n-1, k-1)
  • 如果没选第一个元素，那么需要从剩下的 n-1 个中选 k 个：C(n-1, k)

两类互斥，加起来就是总数 C(n,k)。

这就是著名的杨辉三角的构成规则：每个数等于它上方两数之和。

🔥 关键洞察：Pascal恒等式的力量在于——它把一个大的组合数问题"降解"两个更小的组合数问题。这种"分治"思想在组合数学中反复出现。如果你能熟练地把 C(n,k) 看成 C(n-1,k-1) + C(n-1,k)，很多看似复杂的求和问题会变得无比简单。

2.3 ∑C(n,k) = 2^n 的组合解释

这是组合数学中最基础的求和恒等式：

∑_{k=0}^{n} C(n,k) = C(n,0) + C(n,1) + ... + C(n,n) = 2^n

我们来从三个角度理解它：

2.3.1 组合解释（每个元素选/不选）

这是最直观的解释。n 个元素的集合，它的子集有多少个？每个元素有两种状态——选或不选，所以总共 2^n 个子集。而所有子集可以按大小 k 分类：大小为 0 的有 C(n,0) 个，大小为 1 的有 C(n,1) 个，……，大小为 n 的有 C(n,n) 个。把各类加起来就是子集总数，所以 ∑C(n,k) = 2^n。

2.3.2 二项式解释

由二项式定理，(1+1)^n = ∑C(n,k) × 1^k × 1^{n-k} = ∑C(n,k)。这个思路非常强大——通过选择适当的赋值，可以派生出无数恒等式。

2.3.3 构造性证明

这是竞赛中最灵活的证明方式。构造一个集合 A 和一个集合 B，建立双射（一一对应），证明 |A| = |B|。A 可以选 n 个元素的子集，|A| = ∑C(n,k)；B 可以构造所有的 n 位二进制串，|B| = 2^n。双射就是：每个子集对应一个二进制串——第 i 位为 1 表示第 i 个元素在子集中，为 0 则表示不在。

2.4 朱世杰恒等式与组合恒等式表

除了 ∑C(n,k) = 2^n 之外，竞赛中还有几个极其常用的组合恒等式。

2.4.1 朱世杰恒等式（上指标求和公式）

∑_{i=0}^{m} C(r+i, i) = C(r+m+1, m)

等价形式：
∑_{i=0}^{m} C(r+i, r) = C(r+m+1, r+1)

这个恒等式在中文文献中常被称为"朱世杰恒等式"，以纪念元代数学家朱世杰。它的直观理解可以通过"斜线求和"在杨辉三角上展现——沿着杨辉三角的斜对角线（从 (r,0) 出发往右下）求和，结果等于该斜线末端"再往右下一格"的数。

用 Pascal 恒等式很容易证明这个等式——反复使用 C(n,k) = C(n+1,k+1) - C(n,k+1)，然后裂项相消。

2.4.2 常用组合恒等式速查表

编号恒等式名称/直觉
(1) C(n,k) = C(n, n-k) 对称性
(2) C(n,k) = C(n-1,k-1) + C(n-1,k) Pascal恒等式
(3) ∑_{k=0}^{n} C(n,k) = 2^n 子集总数
(4) ∑_{k=0}^{n} (-1)^k C(n,k) = 0 (n≥1) 交错和为零
(5) ∑_{k=0}^{m} C(r+k, k) = C(r+m+1, m) 朱世杰恒等式
(6) C(n,k) = (n/k) × C(n-1, k-1) 提取因子
(7) C(m+n, k) = ∑_{i=0}^{k} C(m,i) C(n, k-i) Vandermonde卷积
(8) ∑_{k=0}^{n} k×C(n,k) = n×2^{n-1} 加权和
(9) ∑_{k=0}^{n} k²×C(n,k) = n(n+1)×2^{n-2} 二阶加权和

这些恒等式不需要死记硬背——关键是理解它们背后的组合意义。下面我们挑几个重点逐个解释。

2.4.3 Vandermonde卷积

Vandermonde 卷积恒等式：

C(m+n, k) = ∑_{i=0}^{k} C(m, i) × C(n, k-i)

组合解释：从 m+n 个元素中选 k 个。我们把 m+n 个元素分成两堆——一堆 m 个、一堆 n 个。那么从总数中选 k 个，就是：从第一堆中选 i 个（i = 0,1,...,k），同时从第二堆中选 k-i 个。对于每个固定的 i，方案数为 C(m,i) × C(n,k-i)。对所有 i 求和即得总数。

这个恒等式在竞赛中经常出现，尤其是在需要"分堆"或"分类"的场景中。特别地，当 m=n=k 时，我们得到：

C(2n, n) = ∑_{i=0}^{n} C(n,i)²

这是一个美丽的结论——中间二项式系数 C(2n,n) 等于 C(n,i) 的平方和。它在概率论中也有深刻的应用（比如随机游走）。

2.4.4 加权和 ∑k·C(n,k) = n·2^{n-1}

这个恒等式的组合解释也很漂亮：左边 ∑k·C(n,k) 可以理解为"所有子集的大小之和"。也可以理解为：先选一个子集（2^n 种），然后数它的大小（k）。但右边 n·2^{n-1} 是"每个元素被包含的次数之和"——每个元素出现在 2^{n-1} 个子集中（因为固定一个元素被选，其余 n-1 个元素任意），所以总次数为 n×2^{n-1}。两边算的是同一个东西，所以相等。

💡 算两次思想：上面这个证明使用了"算两次"的思想——同一个量从两个不同角度计算，得到的结果必然相等。这是组合数学中最强大的证明工具之一，我们将在第12章中专门讨论。现在先感受一下它的威力。

2.5 组合恒等式的证明方法——给竞赛生的武器库

证明组合恒等式没有"万能钥匙"，但有几种常用的"武器"。竞赛中遇到恒等式时，按以下顺序尝试：

方法一：代数法（用公式展开）

把组合数全部写成阶乘形式 C(n,k) = n!/(k!(n-k)!)，然后进行代数化简。这个方法虽然"暴力"，但可靠性最高——只要代数不过关，一定能证出来。缺点是可能很繁琐。

方法二：组合解释法（构造双射或分类）

给恒等式找一个组合模型。等号两边分别代表"数同一个东西的两种不同方式"，或者"两个集合之间存在双射"。这是竞赛中最高级的证明方法，一旦找到，证明往往极其简洁。

方法三：递推+归纳法

利用 Pascal 恒等式将求和转化为递推关系，然后用数学归纳法证明。适合那些与 n 有明显递推关系的恒等式。

方法四：生成函数法

利用 (1+x)^n 的展开式，通过赋值、求导、积分等操作得到各种恒等式。这是最强大的方法之一——但需要一定的微积分基础。我们会在第10章和第11章详细介绍。

🔥 经典案例：证明 ∑_{k=0}^{n} (-1)^k C(n,k) = 0（n≥1）。

代数法：(1-1)^n = ∑_{k=0}^{n} C(n,k) × 1^{n-k} × (-1)^k = ∑_{k=0}^{n} (-1)^k C(n,k) = 0^n = 0。一步到位！

组合解释：奇子集个数等于偶子集个数。因为固定一个元素，所有子集可以一一配对——"包含该元素的子集"与"不包含该元素的子集"，且两子集大小奇偶性不同。所以奇子集和偶子集一样多。

同一个恒等式，两种完全不同的证明思路——这就是组合数学的魅力。

2.6 实战演练与踩坑故事

说一个我亲身的经历。有一年我在给省队集训时，出了一道题：

"求证：∑_{k=0}^{n} k × C(n,k) × (-1)^k = 0（n≥2）。"

一个很有天赋的学生是这样做的：他看到有 (-1)^k，想到用 (1-1)^n 展开，得 ∑(-1)^k C(n,k) = 0。然后他说，"这里的 k 是外面的系数，而 (1-1)^n 的展开不包含 k，所以不能直接用。但我们可以求导！"

他写出 (1+x)^n = ∑C(n,k)x^k，两边对 x 求导：

n(1+x)^{n-1} = ∑ k·C(n,k)·x^{k-1}

令 x = -1：左边 = n(1-1)^{n-1} = n·0^{n-1}。当 n≥2 时，n-1≥1，所以 0^{n-1}=0，左边为0。右边 = ∑ k·C(n,k)·(-1)^{k-1} = -∑ k·C(n,k)·(-1)^k。

所以 ∑ k·C(n,k)·(-1)^k = 0。证完。

但他的写法里有一步被扣了分——他忘了讨论 n=1 的情况。当 n=1 时，(1+x)^{n-1} = (1+x)^0 = 1，左边 = 1·1 = 1，右边 = ∑ k·C(1,k)·(-1)^{k-1} = 1×C(1,1)×(-1)^0 = 1。结果也是对的，但推导过程中 0^{0} 的出现需要额外说明。数学竞赛中，0^0 的处理和边界条件检查是常见的细节扣分点。

⚠️ 踩坑实录：在组合恒等式的证明中，最容易犯的错误是——把 C(n,k) 当成连续函数来处理，忽略了 n 和 k 是整数的前提。比如将 C(n,k) = n!/(k!(n-k)!) 展开后，遇到 ∑ 符号就机械地套用微分积分公式，结果忘记检查求导后等式的整数性质是否一致。我建议你：无论用什么方法，每次令 x 等于某个值之前，先想清楚这个 x 在原定义中是否有意义。

2.7 本章小结与过渡

今天我们学习了组合数的三种定义视角、Pascal恒等式与杨辉三角、核心求和恒等式 ∑C(n,k) = 2^n、朱世杰恒等式和 Vandermonde 卷积。我们整理了一张常用的组合恒等式速查表，并讨论了四种证明组合恒等式的方法。

组合恒等式是组合数学的"代数语言"——掌握了它们，你不仅能解决组合问题，还能将看似无关的数学分支（代数、概率、数论）串联起来。下一章我们要讲二项式定理与多项式，你会发现今天学的恒等式很多都可以通过二项式展开来统一理解和证明。二项式定理就像一座桥梁，连通了组合数与多项式这两个世界。我们第三章见。
