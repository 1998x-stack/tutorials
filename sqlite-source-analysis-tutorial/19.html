第19章 内置函数实现

大家好，我是你们的讲师。上一章我们深入了查询规划器，这一章来聊一个更"接地气"的话题——内置函数。当你写 SELECT LOWER(name), COUNT(*) FROM users 时，这些函数到底是怎样实现的？答案是：它们通过一套统一的 函数注册与回调机制 实现，并且这套机制完全开放——你可以把自己写的 C 函数注册进去，让 SQLite 在 SQL 中直接调用。

内核视角： 标量函数、聚合函数和窗口函数，本质上都是"注册了回调的 FuncDef 对象"。理解了这个模型，你就能随心所欲地扩展 SQLite 的 SQL 表达力。

函数注册机制

一切从 sqlite3CreateFunc() 开始。它在 func.c 中定义，是内置函数和自定义函数的共同注册入口。注册后的函数被存储在 FuncDef 结构中：

typedef struct FuncDef FuncDef;
struct FuncDef {
  i8 nArg;                     /* 参数个数（-1 表示可变） */
  u8 funcFlags;                /* SQLITE_FUNC_* 标志位 */
  void *pUserData;             /* 用户自定义数据 */
  void (*xSFunc)(sqlite3_context*,int,sqlite3_value**); /* 标量回调 */
  void (*xFinalize)(sqlite3_context*);                  /* 聚合回调 */
};

函数查找的核心机制是哈希表。在 db->aFunc.a[] 数组中，SQLite 维护了按函数名散列的查找表。查找函数 sqlite3FindFunction() 实现"参数个数最佳匹配"策略——你调用的函数参数个数越匹配，优先级越高。如果系统中只有可变参数版本（nArg==-1），也会被匹配。

注册方式回调类型例子源码文件
sqlite3CreateFuncxSFunc标量函数 LOWERfunc.c
sqlite3CreateFunc + xStep/xFinal聚合函数COUNT, SUMfunc.c
sqlite3_window_link窗口函数row_numberwindow.c
💡 SQLite 的内置函数通过 sqlite3BuiltinFunctions() 懒加载注册——第一次使用时才添加到哈希表，避免了启动时的内存开销。

标量函数的实现模式

标量函数输入一行数据输出一个值。实现模式非常简单：注册一个 xFunc 回调，通过 sqlite3_value_*() 读取参数，通过 sqlite3_result_*() 输出结果。

来看 lower() 函数的实现：

static void lowerFunc(sqlite3_context *context, int argc, sqlite3_value **argv){
  const unsigned char *zIn = sqlite3_value_text(argv[0]);
  int n = sqlite3_value_bytes(argv[0]);
  char *zOut = sqlite3_malloc(n+1);   /* 申请输出缓冲区 */
  if( zOut==0 ){
    sqlite3_result_error_nomem(context);
    return;
  }
  for(int i=0; i<n; i++){
    zOut[i] = sqlite3Tolower(zIn[i]); /* 逐字符转小写 */
  }
  zOut[n] = '\0';
  sqlite3_result_text(context, zOut, n, sqlite3_free); /* 所有权转移 */
}

注意 sqlite3_result_text() 的最后一个参数是释放回调——传 sqlite3_free 表示把内存所有权移交给 SQLite，由它负责释放；传 SQLITE_TRANSIENT 则 SQLite 会自己拷贝一份。

聚合函数实现

聚合函数采用 xStep + xFinal 两阶段模式——xStep 每行调用一次累加状态，xFinal 在处理完毕后输出结果。状态通过 sqlite3_aggregate_context() 分配：

/* COUNT() 聚合的实现 */
typedef struct CountCtx CountCtx;
struct CountCtx { i64 n; };

static void countStep(sqlite3_context *context, int argc, sqlite3_value **argv){
  CountCtx *p = (CountCtx*)sqlite3_aggregate_context(context, sizeof(*p));
  if( p && (argc==0 || sqlite3_value_type(argv[0])!=SQLITE_NULL) ){
    p->n++;             /* COUNT(*) 无条件递增，COUNT(col) 跳过 NULL */
  }
}

static void countFinalize(sqlite3_context *context){
  CountCtx *p = (CountCtx*)sqlite3_aggregate_context(context, 0);
  sqlite3_result_int64(context, p ? p->n : 0);
}
⚠️ sqlite3_aggregate_context() 的第一个参数传大小来初始化聚合上下文，后续传 0 获取已分配的指针。务必在 xStep 中传入正确的大小初始化，否则返回 NULL。

Window Function 框架

窗口函数（3.25.0 引入）和聚合函数有很深的关系——窗口函数就是"带窗口的聚合函数"。同一个 xStep/xFinal 回调可以同时在 GROUP BY 聚合和窗口函数中使用，区别在于：普通聚合对全体行调用一次 xFinal，窗口函数为每一行调用一次 xFinal（只在窗口范围内有效）。

纯计算窗口函数如 row_number() 更简单——本质就是一个自增计数器：

static void row_numberStep(sqlite3_context *context, int NotUsed,
                            sqlite3_value **NotUsed2){
  sqlite3_int64 *p = (sqlite3_int64*)sqlite3_aggregate_context(context, sizeof(*p));
  if( p ) (*p)++;
}
static void row_numberFinalize(sqlite3_context *context){
  sqlite3_int64 *p = (sqlite3_int64*)sqlite3_aggregate_context(context, 0);
  sqlite3_result_int64(context, p ? *p : 0);
}

日期时间函数 (date.c)

日期时间函数全部位于 src/date.c 中，包括 datetime()、date()、time()、strftime()、julianday()、unixepoch() 等。所有函数共享同一个核心解析引擎 parseModifier()，通过"修饰符链"逐步转换时间字符串。

内部使用儒略日（Julian Day）作为中间表示，所有时间操作都转化为浮点数。计算两个日期的天数差只需一次浮点减法，无需处理闰年闰月。

/* date.c — 时间对象结构 */
typedef struct DateTime DateTime;
struct DateTime {
  sqlite3_int64 iJD;   /* 儒略日（毫秒精度） */
  int Y, M, D, h, m, s;
  int validYMD, validHMS, validJD;
  int tz;              /* 时区偏移（分钟） */
};
生产环境建议： 自定义函数注册时尽量使用 SQLITE_DETERMINISTIC 标志，告诉 SQLite 函数的返回值完全由参数决定。这样查询规划器可以在常量折叠优化中提前计算函数值，提升查询性能。

实战：编写自定义函数

假设我们要实现一个 CHAR_LEN(s) 函数，计算 UTF-8 字符串的"视觉长度"——忽略零宽字符和组合字符。这个流程代表了所有自定义函数的完整生命周期：编写 C 代码 → 编译动态库 → 在 SQLite 中加载 → 在 SQL 中使用。

#include <sqlite3ext.h>
SQLITE_EXTENSION_INIT1

static void charlenFunc(sqlite3_context *ctx, int argc, sqlite3_value **argv){
  const unsigned char *z = sqlite3_value_text(argv[0]);
  int n = sqlite3_value_bytes(argv[0]), charCount = 0, i = 0;
  while( i < n ){
    unsigned char c = z[i];
    int byteLen = (c < 0x80) ? 1 : ((c & 0xE0)==0xC0) ? 2 :
                  ((c & 0xF0)==0xE0) ? 3 : ((c & 0xF8)==0xF0) ? 4 : 0;
    if( byteLen==0 ){ i++; continue; }
    if( byteLen==3 ){ /* 跳过零宽字符 */ }
    charCount++; i += byteLen;
  }
  sqlite3_result_int(ctx, charCount);
}

int sqlite3_extension_init(sqlite3 *db, char **pzErrMsg,
                           const sqlite3_api_routines *pApi){
  SQLITE_EXTENSION_INIT2(pApi);
  sqlite3_create_function(db, "CHAR_LEN", 1,
    SQLITE_UTF8|SQLITE_DETERMINISTIC, 0, charlenFunc, 0, 0);
  return SQLITE_OK;
}

/* 编译：gcc -g -fPIC -shared charlen.c -o charlen.so */
/* 使用：.load ./charlen; SELECT CHAR_LEN('hello'); → 5 */
