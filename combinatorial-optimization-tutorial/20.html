开篇：社交网络上的"病毒营销"困境

大家好，我是你们的讲师。2015年，Facebook发表了一篇著名的论文，讲述他们如何利用影响力最大化（Influence Maximization）来提升用户活跃度。问题是这样的：你有100万个用户，想在社交网络上发起一个推广活动——给其中一小部分用户免费发放优惠券，希望通过他们的社交关系链传播给更多人。你要选哪K个人来开始？

直觉上，你应该选"最有影响力"的人——那些粉丝多、转发率高的人。但问题是，两个最有影响力的人可能有完全相同的社交圈子——你选了A就不用选B，因为A的影响已经把B的圈子覆盖了。这就是边际收益递减（Diminishing Returns）的核心思想——多选一个人带来的新增影响力，随着已选人数的增加而减少。

这为什么和组合优化有关？这个"选K个人使总影响力最大"的问题，和上一章中我们学到的拟阵秩函数有一个共同的关键性质：次模性（Submodularity）。次模性被很多研究者称为"离散世界的凸性"——它在组合优化中扮演的角色，就像凸性在连续优化中扮演的角色一样重要。只是次模性说的是"边际收益递减"，而不是"函数图像向下凸"。

好，让我们正式进入次模函数优化的世界。

次模性定义

次模性描述的是集合函数的"边际收益递减"性质。我们先来看它的正式定义，再通过例子来建立直觉。

边际收益递减

一个集合函数 f: 2ᴱ → ℝ 被称为次模函数（Submodular Function），如果对任意 A ⊆ B ⊆ E 和 x ∈ E \ B，有：

f(A ∪ {x}) - f(A) ≥ f(B ∪ {x}) - f(B)

换句话说：当你已经拥有更多元素时，新增一个元素带来的收益不会更大——往往是更小。这就是"边际收益递减"。

这个定义还有一个等价的全局形式，有时更方便：

对任意 A, B ⊆ E：
f(A) + f(B) ≥ f(A ∪ B) + f(A ∩ B)

这个形式揭示了次模性的对称美——两个集合的函数值之和不小于它们的并与交的函数值之和。

对比两种定义：边际收益递减的形式适合构建贪心算法时的直觉，而全局形式更适合理论分析。当你在论文中看到"by submodularity"时，80%的后续推导用的都是全局形式。我建议你把两个定义都记住，在不同场景灵活切换。

如果一个次模函数 f 还满足 f(A) ≤ f(B) 当 A ⊆ B，则称它是单调的（Monotone）。单调性在很多实际场景中自然成立——你永远不会因为覆盖了更多传感器而"减少"覆盖面积。

次模函数的例子

次模函数其实无处不在。一旦你学会了识别它，你会发现它比想象中普遍得多。

覆盖函数

这是最典型的次模函数。假设你部署传感器监测一个区域——每个传感器能覆盖一定的范围。你选了一组传感器，覆盖的总面积是 f(S) = |∪_{i∈S} Aᵢ|。如果两个传感器的覆盖范围有重叠，选第二个传感器时新增的覆盖面积就会变小——完美符合边际收益递减。

熵函数

在信息论中，一组随机变量的联合熵 H(X_S) 是次模的。当变量间存在相关性时，多观测一个变量带来的信息增益会减少——这正是次模性的体现。这个性质被用于特征选择（Feature Selection）：选出一组特征使得联合熵最大，即信息最丰富。

影响力函数

回到开篇的社交网络例子。在独立级联模型（Independent Cascade Model）下，一个种子集合 S 的期望传播范围（即最终会被影响的人数）是次模函数的——第一个种子用户可能带来100个新用户，但第K个种子带来的新用户可能只有20个，因为大部分人在前面已经被覆盖过了。

割容量函数

给定有向图 G = (V, E)，定义 f(S) 为从 S 到 V \ S 的割的容量。这个函数是次模的——而且和拟阵的秩函数一样，是最早被研究的次模函数之一。

函数类型 定义 单调？ 典型应用
覆盖函数 f(S) = |∪ᵢ∈S Aᵢ| 是 传感器布局、影响力最大化
熵函数 f(S) = H(X_S) 是 特征选择、实验设计
割函数 f(S) = cap(δ(S)) 否 图分割、聚类
拟阵秩函数 r(S) = max{|I| : I⊆S, I独立} 是 资源分配（见第19章）
设施选址 f(S) = Σⱼ maxᵢ∈S wᵢⱼ 是 仓库选址、聚类
一个练习：试着用次模性的边际收益递减定义去检查你日常工作中的目标函数。推荐系统的多样性（点击总和）是不是次模的？数据摘要中的信息覆盖是不是次模的？在大多数情况下，只要涉及"覆盖"或者"选择"而且伴随明显的边际递减效应，大概率就是次模的。这种"次模思维"本身就是一种很有价值的分析能力。

次模最大化与贪心

次模优化有两个方向：最大化和最小化。这两个方向的难度天差地别——最大化是NP难的，但有漂亮的近似保证；最小化反而是多项式时间可解的。

贪心算法的(1-1/e)近似保证

考虑单调次模函数受基数约束的最大化问题：

max f(S)  subject to |S| ≤ k

贪心算法很简单：

贪心次模最大化
S ← ∅
for i = 1 to k:
    x ← argmax_{x ∈ E \ S} f(S ∪ {x}) - f(S)
    S ← S ∪ {x}
return S

Nemhauser-Wolsey-Fisher定理（1978）：对于单调次模函数，这个贪心算法给出的解 S_g 满足：

f(S_g) ≥ (1 - 1/e) · f(S*)

其中 S* 是全局最优解，1 - 1/e ≈ 63%

也就是说，贪心算法至少能拿到最优值的63%。更令人惊叹的是，这个界是紧的——存在次模函数使得贪心算法恰好只能达到(1-1/e)倍的最优值，不可能改进。

一个工程师的踩坑故事：我曾在一家广告技术公司工作，一位同事用贪心算法选K个广告位来最大化曝光量。上线后表现不错，曝光量达到了预期。但竞争对手（当时不知道他们在用啥方法）同样的预算却获得了高出30%的转化率。后来才发现，我们的目标函数不是单调次模的——因为用户看过同一广告两次会产生厌烦（负边际收益）。贪心的63%近似保证只对单调次模函数成立，不适用于非单调的情况。这提醒我们：所有理论保证都有前提条件，使用时务必确认。

紧的例子

为什么是(1-1/e)而不是100%？考虑一个极端例子：k个候选元素，每个元素单独带来的价值是1，但所有元素一起带来的总价值只有k·(1 - (1-1/k)^k) ≈ k·(1 - 1/e)。贪心算法选第一个元素时选到最好的（价值1），选第二个时发现剩下的每个元素新增收益只有(1 - 1/k)，依次递减……最后的总和就是大约(1 - 1/e)·k，而最优解（知道每个元素的价值，不需要贪心地迭代计算）可以达到k。

这个紧的例子说明：次模性是"非线性"的，贪心无法"看透"高阶交互效应。它每次只选当前最好的，但可能错过了某些元素组合才能产生的协同增量。

次模最小化

次模最小化是完全不同的故事——它居然可以在多项式时间内精确求解！这是组合优化中极少数"非凸但在多项式时间内可精确求解"的例子之一。

Lovász扩展

1982年，Lovász做了一个关键的观察：任何次模函数 f 都可以被扩展为一个凸函数 f̂，这个扩展定义在超立方体 [0,1]^E 上。

Lovász扩展 f̂: [0,1]^E → ℝ

给定向量 x ∈ [0,1]^E，将坐标按降序排列 x₁ ≥ x₂ ≥ ... ≥ xₙ
令 Sⱼ = {元素₁, ..., 元素ⱼ}（前j个最大坐标对应的元素集）
则 f̂(x) = Σ_{j=1}^{n} xⱼ · (f(Sⱼ) - f(Sⱼ₋₁))
       = f(∅)·(1 - x₁) + Σ_{j=1}^{n} f(Sⱼ)·(xⱼ - xⱼ₊₁)

关键性质：f 是次模的 ⇔ f̂ 是凸函数
f 在整数点上的最小值 = f̂ 在 [0,1]^E 上的最小值

这就是"次模性 = 离散凸性"的真正含义——每个次模函数都有一个自然的凸扩展，离散最小化变成了连续凸最小化。

Ellipsoid方法 + 分离oracle

有了Lovász扩展，我们可以利用凸优化的工具来求解次模最小化。最经典的方法是椭球法（Ellipsoid Method）配合分离oracle（Separation Oracle）：

  • 目标：最小化 f(S) 等价于最小化凸函数 f̂(x) 在 [0,1]^E 上
  • 椭球法每次维护一个包含最优解的椭球
  • 分离oracle判断当前点是否在可行域内，如果不在，返回一个"切割超平面"
  • 对次模函数，分离oracle可以在多项式时间内实现（通过求解一个简单的次模最小化子问题）

因此，次模最小化可以在多项式时间内求解。虽然椭球法在实际中较慢，但它在理论上确认了问题的可解性。

最小范数点算法

在实际工程中，更常用的是最小范数点（Minimum Norm Point）算法。对次模函数 f，考虑它的基多面体（Base Polyhedron）：

B(f) = { x ∈ ℝ^E | x(S) ≤ f(S) for all S ⊆ E, x(E) = f(E) }

最小化 f(S) 等价于在 B(f) 上找一个"最小范数点"。

Fujishige-Wolfe算法（2006）通过在基多面体上迭代投影来寻找最小范数点，在实践中收敛很快，能处理百万级元素的问题。

关于椭球法和最小范数点算法的选择：如果你是做理论研究，椭球法更容易理解和分析；如果你要写代码解决实际问题，建议使用Fujishige-Wolfe最小范数点算法。Python的submodlib库中已经有高效实现，对于n=10,000规模的问题通常能在几秒内求解。

次模优化在ML中的应用

次模优化在机器学习中的应用是近年来最活跃的研究方向之一。这里给出几个典型的场景。

数据摘要

假设你有一个包含100万张图片的数据集，想从中选出100张最能"代表"整个数据集的图片。这个问题可以建模为次模最大化：定义 f(S) 为集合 S 覆盖的"视觉多样性"或"信息含量"——这是次模的。贪心算法给出63%的近似保证，实际中效果往往更好。

特征选择

在机器学习中，过多的特征会导致过拟合和训练时间过长。我们希望从D维特征中选出k个最"有用"的。定义 f(S) 为选中特征与目标变量之间的互信息 I(X_S; Y)——这是次模的。最大化互信息等价于选择最有预测力的特征子集。

主动学习

在主动学习中，我们有一个无标注数据集，希望选择最有信息量的样本让人工标注。选择策略可以建模为次模最大化：选出的样本集应该最大程度地降低模型的不确定性。常用的不确定性采样和多样性采样都可以统一到次模框架下。

影响力最大化

这已经是我们开篇的例子的正式解答了。Kempe-Kleinberg-Tardos在2003年的开创性论文中证明了：在标准的传播模型（独立级联模型和线性阈值模型）下，影响力函数是次模的。因此，贪心算法选K个初始种子，至少能达到最优影响力的63%。这篇论文是社交网络分析领域被引用最多的论文之一。

有意思的后续：Kempe等人的论文发表后，Microsoft Research开发了一个叫NETRANK的系统，利用次模贪心来选择最有影响力的K个用户。他们在Facebook的一个子图上测试，发现贪心算法选出的种子集比"度中心性最高"（即选粉丝最多的）策略高出40%的传播范围。这个差距的价值——在大规模营销中——意味着数百万美元。所以，次模优化不是在"理论"上更优，而是在"真金白银"上更优。
