第14章 统计信号处理基础

到目前为止，我们课程中处理的信号都有一个共同特点：它们是确定的。给你一个公式x[n] = cos(2π·f₀·n/f_s)，你就可以精确计算出任意时刻的值。这是确定性信号——过去和未来完全被数学规律所支配。

但现实世界中的绝大部分信号不是这样的。打开麦克风录一段环境音，你能用一个公式描述它吗？不能。你能精确预测下一秒的声压值吗？也不能。股市的收益率、无线信道的衰落、心电图的RR间期、雷达接收机的噪声基底——这些信号的本质特征不是"确定性"，而是随机性。我们需要一套新的工具来描述和处理它们。

这就是统计信号处理登场的时刻。统计信号处理不再试图精确预测信号的每一个样本值，而是转而描述信号的统计特性——均值、方差、相关函数、功率谱密度。这些统计量虽然不能告诉你下一瞬间的具体值，但能告诉你"平均来说"信号会怎么变化、它的能量分布在什么频段、以及两个信号之间是否存在关联。

💡 核心认知：统计信号处理是DSP从"确定论"到"概率论"的一次思维跃迁。确定论视角下，信号是一个函数；统计视角下，信号是一个随机过程的一次实现（a single realization of a stochastic process）。这个视角的转变，让我们能够处理那些无法用公式描述的真实信号——而且往往能得到比确定性方法更鲁棒的工程方案。

14.1 确定性信号 vs 随机信号

我们先明确区分这两个概念：

对比项 确定性信号 随机信号
定义 可用数学公式精确描述 无法精确描述，只能统计描述
未来可预测性 完全可预测 仅可概率预测
频谱 线状谱（离散频率分量） 连续谱（功率谱密度）
傅里叶变换 存在（绝对可积/可和） 通常不存在（能量无限）
典型例子 正弦波、阶跃信号 热噪声、语音、地震波

确定性信号的频谱是线状的——在特定频率上有能量，其他地方能量为零。随机信号的频谱是连续的——能量连续分布在频率轴上，我们称之为"功率谱密度"而非"频谱幅值"。

你可能会问：为什么不能用FFT直接分析随机信号？当然可以——但问题是对随机信号来说，不同时间段的FFT结果会不一样，而且即使你增加FFT点数，频谱也不会收敛到一个确定的形状（反而会更"毛躁"）。这就是为什么我们需要功率谱密度估计，而不是直接照搬确定性信号的FFT方法。

14.2 平稳性与各态历经性

在描述随机信号时，我们引入了随机过程的概念。一个随机过程X(t)（或X[n]）是一族随机变量的集合，每个时间点对应一个随机变量。当我们实际测量时，得到的是这个随机过程的一个样本函数（realization），也就是我们手里的那一段数据。

但这里有个实际问题：如果我们只有一个样本函数（一段数据），如何估计随机过程的统计特性？这就需要两个关键假设：

平稳性（Stationarity）：随机过程的统计特性不随时间平移而改变。严格平稳要求所有阶的统计量不变，但在工程中我们通常只要求宽平稳（Wide-Sense Stationary, WSS）：

  • 均值E[X[n]]是常数（不依赖于n）；
  • 自相关函数E[X[n]·X[n+τ]]只依赖于τ而不依赖于n。

各态历经性（Ergodicity）：这是工程中最重要的假设——时间平均等于集合平均。也就是说，用一段足够长的单次观测数据计算的时间平均，可以替代对所有可能实现的集合平均。

各态历经性假设下的均值估计：
μ_x ≈ (1/N) · Σ_{n=0}^{N-1} x[n]

自相关函数估计：
r_xx[τ] ≈ (1/N) · Σ_{n=0}^{N-1-τ} x[n] · x[n+τ]

各态历经性是一个"信则灵"的假设——绝大多数工程信号都被假定为各态历经的（否则你就没法用一段数据做任何统计估计了）。但要注意，并非所有平稳过程都是各态历经的。比如一个固定的直流信号加上随机相位正弦波——如果相位不是均匀分布，时间平均和集合平均就不相等。

🔧 工程直觉：在实际项目中，工程师很少会去严格检验一个信号是否各态历经。通常的做法是：默认假设信号是各态历经的，用数据做估计，然后通过交叉验证（比如把数据分成两段，分别计算统计量，看是否一致）来检验这个假设是否合理。如果两半的结果差别很大，那就不能看作各态历经信号。

14.3 自相关函数与互相关函数

自相关函数（Auto-Correlation Function, ACF）是描述信号"和自己在不同时间延迟下的相似程度"的函数。对于宽平稳随机过程，自相关函数定义为：

r_xx[τ] = E[X[n] · X[n+τ]]

自相关函数有以下重要性质：

  • 对称性：r_xx[-τ] = r_xx[τ]（实信号）；
  • 最大值在τ=0：r_xx[0] = E[X²[n]]（信号的平均功率）；
  • 边界行为：如果信号不含周期分量，r_xx[τ] → 0 当 |τ| → ∞（对于零均值信号）。

互相关函数描述了两个不同信号之间的相似程度：

r_xy[τ] = E[X[n] · Y[n+τ]]

互相关的工程应用非常广泛：

  • 时延估计（Time Delay Estimation）：在雷达和声纳中，发射信号和接收回波的互相关峰值位置对应目标距离。这是最经典的应用。
  • 系统辨识（System Identification）：用白噪声激励未知系统，计算输入和输出的互相关，可以得到系统的冲激响应估计。
  • 同步：通信中的帧同步——接收端用已知的前导序列（preamble）和接收信号做互相关，峰值位置就是帧起始位置。
⚠️ 踩坑实录：一次在做超声测距项目时，我们用互相关估计发射脉冲和回波之间的时延。在实验室环境下精度很好，但到了实际现场（工业水箱液位测量），互相关峰值变得很不稳定。排查发现：现场有多台超声波传感器同时工作，互相干扰。A传感器发射的信号经过水箱壁反射后，被B传感器接收——相当于B接收到了A的"寄生回波"。我们的算法对着两个传感器的混合信号做互相关，结果出现了多个虚假峰值。解决方案是采用正交编码：给每个传感器分配不同的扩频码序列，只有匹配的码序列才能产生高相关峰值。这其实就是CDMA通信中的基本思想。

14.4 功率谱密度（PSD）估计

功率谱密度是随机信号在频域的"名片"。对于确定性信号，我们有傅里叶变换X(f)；对于随机信号（能量无限），我们用PSD来描述单位频率上的功率分布。

根据Wiener-Khinchin定理，宽平稳随机过程的PSD和自相关函数互为傅里叶变换对：

S_xx(f) = Σ_{τ=-∞}^{∞} r_xx[τ] · e^{-j2πfτ}
r_xx[τ] = ∫_{-1/2}^{1/2} S_xx(f) · e^{j2πfτ} df

这个关系极其重要：它告诉我们时域的相关性和频域的功率分布是等价的。如果你知道信号的PSD，你就知道信号在什么频率上有"能量"；知道自相关函数，你就知道信号随时间变化的"记忆长度"。

工程上，PSD的估计有两种主流方法：

周期图法（Periodogram）：

# 直接对N点数据做FFT，取模平方，除以N
from scipy import signal
f, Pxx = signal.periodogram(x, fs=44100)
# Pxx 就是功率谱密度估计
# 注意：periodogram默认使用矩形窗，方差很大

周期图法的优点是计算简单，一个FFT完事。但它的方差极大——估计值的标准差和均值差不多大（即估计的"抖动"和信号本身一样大）。而且无论你增加N到多少，方差都不会减小！这是周期图法的根本缺陷。

Welch法（平均加窗周期图）：

# Welch法：把数据分段、加窗、计算周期图、取平均
f, Pxx = signal.welch(x, fs=44100, nperseg=1024, window='hann')
# nperseg 控制分段长度，越长频率分辨率越高，但分段数越少（方差越大）

Welch法将N点数据分成K段（有重叠，通常50%），每段计算周期图，然后取平均。因为对K个独立估计取平均，方差减小为周期图法的1/K。这是经典的分辨率-方差折中：分段越短、段数越多，方差越小但分辨率越差；分段越长、段数越少，分辨率越好但方差越大。

PSD方法 频率分辨率 方差 计算量 最佳用途
周期图法 高（~f_s/N） 高（不随N改善） 低（一次FFT） 快速查看、长记录
Welch法 中（~f_s/N_pseg） 低（可调） 中（K次FFT） 一般用途（首选）
多窗口法（Multitaper） 中高 低 中高 超低噪声场景（如天文）
参数化法（AR模型） 极高 模型依赖 中 短记录、高分辨率需求
📊 经验法则：做PSD估计时，永远从Welch法开始（nperseg取256-4096之间，重叠50%，Hanning窗）。如果频率分辨率不够，增加nperseg（牺牲方差）。如果方差太大，减少nperseg增加段数（牺牲分辨率）。当你调不动Welch法的时候，再考虑Multitaper或AR模型——对于绝大多数工程问题，Welch法都够用了。

14.5 白噪声与有色噪声

在统计信号处理中，噪声按PSD的形状分为两大类：

白噪声（White Noise）：

  • PSD在所有频率上是常数（S(f) = σ²）；
  • 自相关函数为r_xx[τ] = σ² · δ[τ]（仅在τ=0处有值）；
  • 理想白噪声在物理上不存在（需要无限功率），但带限白噪声（0~f_s/2内PSD平坦）是工程中常用的模型。

有色噪声（Colored Noise）：

  • PSD随频率变化，即"着色"了；
  • 常见的颜色：粉红噪声（1/f噪声，PSD∝1/f，自然界最普遍）、布朗噪声（PSD∝1/f²，随机游走）、蓝噪声（PSD∝f，较少见）；
  • 几乎所有实际电子器件的噪声都是有色噪声（至少在中低频段表现为1/f噪声）。
# Python生成白噪声和有色噪声示例
import numpy as np
from scipy import signal

# 白噪声
white_noise = np.random.randn(44100)

# 粉红噪声（通过滤波白噪声得到）
b, a = signal.butter(2, 0.01, btype='low', fs=44100)  # 近似
pink_noise = signal.lfilter(b, a, white_noise)

# 检查PSD
f, Pxx_white = signal.welch(white_noise, fs=44100)
f, Pxx_pink = signal.welch(pink_noise, fs=44100)
# 白噪声PSD基本平坦，粉红噪声PSD随频率增加而下降

识别噪声类型是工程中的重要技能。如果你的系统输出噪声在低频远高于高频，那么问题可能是1/f噪声或漂移；如果高频噪声异常突出，可能是时钟抖动或混叠问题。

14.6 短记录的PSD：为什么方差这么大？

这是一个经典的统计信号处理踩坑点——记录长度对PSD估计的影响。

想象你有一段只有256个样本的短记录，你想分析它的频率成分。直觉上你可能会说："那我做256点FFT不就行了？"周期图法确实可以给出结果，但它的方差几乎是"满格"的——估计值的标准差和均值在相同数量级。

为什么？因为周期图法本质上是用一次观测（一个随机过程的单次实现）来估计PSD。对于随机过程，一次观测的FFT也是一个随机变量，它的"抖动"不会随着FFT点数的增加而减小——增加FFT点数只会增加频率采样点的密度，但每个点上的估计方差不变。

Welch法如何解决这个问题：通过把数据分成多段并取平均，Welch法用"牺牲分辨率换方差降低"的策略解决了这个困境。对于256点数据，你可以在nperseg=64、重叠50%下得到约7段。7段平均后的方差约为周期图法的1/7——虽然分辨率降到了64点FFT的水平，但估计值至少"可看"了。

⚠️ 踩坑实录：多年前做一个振动监测项目，需要在嵌入式设备上实时计算振动信号的PSD。嵌入式DSP的内存非常有限，一次只能缓存256个样本（采样率1kHz，即0.256秒数据）。初版代码直接对256点做周期图，结果频谱图上"毛刺"丛生——0.256秒的数据每次都不相同，提取的"特征频率"在连续帧之间大幅跳变，导致频繁的误报警。后来改用Welch法（4段64点、50%重叠、Hanning窗），虽然频率分辨率从3.9Hz降到了15.6Hz，但PSD估计平滑了很多，误报率下降了90%以上。这就是"宁可分辨率差一点，也要估计稳定"的工程哲学。

14.7 本章小结

这一章我们做了以下几件事：

  • 对比了确定性信号和随机信号的根本差异，解释了为什么随机信号需要统计描述手段；
  • 介绍了平稳性和各态历经性的概念——工程中"用一段数据估计统计特性"的理论基础；
  • 讲解了自相关函数和互相关函数的定义、性质和关键工程应用（时延估计、系统辨识、帧同步）；
  • 介绍了Wiener-Khinchin定理（PSD和自相关函数互为傅里叶变换对），并详细对比了周期图法和Welch法两种PSD估计方案，给出了一个Welch法消除误报的真实案例；

  • 深入分析了短记录PSD估计的方差问题，解释了为什么Welch法的"平均策略"是工程首选。
