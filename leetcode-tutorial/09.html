四种遍历方式 — 递归与迭代实现

大家好，我是你们的讲师。上一章我们学习了二叉树的核心递归思维，这一章我们进入一个更具体也更关键的话题——树的遍历。如果说递归是二叉树面试的"内功心法"，那么遍历就是"招式套路"。四种遍历方式——前序、中序、后序、层序——每一种都在特定的题目中大放异彩。

先快速回顾一下三种深度优先遍历的定义（以递归版本为例）：

# 前序遍历：根 → 左 → 右
def preorder(root):
    if not root: return
    print(root.val)
    preorder(root.left)
    preorder(root.right)

# 中序遍历：左 → 根 → 右
def inorder(root):
    if not root: return
    inorder(root.left)
    print(root.val)
    inorder(root.right)

# 后序遍历：左 → 右 → 根
def postorder(root):
    if not root: return
    postorder(root.left)
    postorder(root.right)
    print(root.val)

递归版本的代码几乎一样，区别只在 print 放的位置。但递归有一个问题：当树很深时，递归栈会溢出。面试时，面试官大概率会追问："你能用迭代实现吗？"

对于迭代遍历，核心思想是用显式栈模拟递归的调用栈。其中前序遍历是最容易的——把根节点入栈，循环弹出，先压右子节点再压左子节点（这样左子节点先弹出）。中序遍历稍复杂，需要一个 cur 指针不断往左走并把沿途节点压栈。

def inorder_iterative(root):
    stack, res = [], []
    cur = root
    while cur or stack:
        while cur:
            stack.append(cur)
            cur = cur.left
        cur = stack.pop()
        res.append(cur.val)
        cur = cur.right
    return res

这段代码是面试手撕的必考模板。它的思路是："能往左就往左，不能往左了，就弹出一个节点，处理它，然后转向右子节点。"记住这个口诀，中序遍历的迭代实现就能信手拈来。

记忆技巧：后序遍历的迭代是最麻烦的。一个取巧的方式是用"根右左"的前序遍历，最后反转结果——这样就得到了"左右根"的后序。很多 LeetCode 高手都用这个技巧，它不是最"正宗"的解法，但面试时绝对够用。

BFS 应用

广度优先搜索（BFS）在二叉树中表现为层序遍历，使用队列实现。LeetCode 102——二叉树的层序遍历（Binary Tree Level Order Traversal）是最基本的模板题。

def level_order(root):
    if not root:
        return []
    res, queue = [], [root]
    while queue:
        level = []
        for _ in range(len(queue)):
            node = queue.pop(0)
            level.append(node.val)
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        res.append(level)
    return res

关键技巧：在每一层开始时，先记录当前队列的长度，这个长度就是当前层的节点数。然后弹出该数量的节点，把它们的子节点入队。这样就实现了按层划分。

锯齿形遍历与右视图

层序遍历的变体非常丰富。LeetCode 103——二叉树的锯齿形层序遍历，只需在层序遍历的基础上，按层数奇偶决定是否反转当前层列表。

LeetCode 199——二叉树的右视图（Binary Tree Right Side View），看似复杂，实际也是层序遍历的简单变体：在每一层中，只取最后一个节点的值。

def right_side_view(root):
    if not root:
        return []
    res, queue = [], [root]
    while queue:
        for i in range(len(queue)):
            node = queue.pop(0)
            if i == len(queue):  # 实际上是 i == len(queue_before_pop) - 1
                res.append(node.val)
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
    return res

不过上面的代码有一个小坑——len(queue) 在循环中变化。更安全的写法是事先用变量保存当前层的长度。这个问题我当年就掉进去过，调试了半天才发现队列长度在循环中被修改了。

常见错误：在使用 for _ in range(len(queue)) 时，Python 的 range 是在循环开始时计算好的——所以即使循环中 queue 的长度变了，range 也不会变。但如果用 for i in range(len(queue)) 并在循环中 pop(0)，下标 i 对应的元素就不是你想象的那个了。建议用 for _ in range(len(queue)) 这种不关心下标的写法。

从遍历序列构建二叉树

这是一个经典到不能再经典的考点：给你两种遍历序列，让你重建出二叉树。LeetCode 上对应两道题：105（前序 + 中序）和 106（后序 + 中序）。

从前序与中序遍历序列构建（105）

核心观察：前序遍历的第一个节点是根节点；在中序遍历中找到这个根节点，它左边的所有节点属于左子树，右边的所有节点属于右子树。然后递归。

def build_tree(preorder, inorder):
    if not preorder or not inorder:
        return None

    root_val = preorder[0]
    root = TreeNode(root_val)

    # 在中序遍历中找到根节点的位置
    mid = inorder.index(root_val)

    # 左子树的前序和中序
    root.left = build_tree(preorder[1:mid+1], inorder[:mid])
    # 右子树的前序和中序
    root.right = build_tree(preorder[mid+1:], inorder[mid+1:])

    return root

这个写法很直观，但每次 inorder.index(root_val) 需要 O(n) 时间，而且 preorder[1:mid+1] 这种切片操作会产生新列表。优化方案是使用哈希表预处理中序序列的值到下标映射，并在递归中传递索引范围而不是切片。

def build_tree_optimized(preorder, inorder):
    index_map = {val: i for i, val in enumerate(inorder)}

    def helper(pre_left, pre_right, in_left, in_right):
        if pre_left > pre_right:
            return None
        root_val = preorder[pre_left]
        root = TreeNode(root_val)
        in_index = index_map[root_val]
        left_size = in_index - in_left
        root.left = helper(pre_left + 1, pre_left + left_size, in_left, in_index - 1)
        root.right = helper(pre_left + left_size + 1, pre_right, in_index + 1, in_right)
        return root

    return helper(0, len(preorder) - 1, 0, len(inorder) - 1)

这个优化版的时间复杂度从 O(n²) 降到了 O(n)，是商业级代码应该有的质量。面试时，如果你能先写出简洁版，再主动提出优化成索引版，面试官会非常欣赏。

从后序与中序遍历构建（106）

思路完全对称：后序遍历的最后一个节点是根节点。在中序中找到它，划分左右子树，递归构建。注意递归时左右子树在后续遍历中的范围要对应好。

核心规律：无论是前序+中序还是后序+中序，"划分"的关键都在中序序列上。中序告诉了你哪些节点在左子树、哪些在右子树。另一个序列（前序或后序）告诉了你谁是根节点。必须有中序才能唯一确定一棵二叉树——只有前序+后序无法唯一重建二叉树（除非是满二叉树）。

序列化与反序列化

LeetCode 297——二叉树的序列化与反序列化（Serialize and Deserialize Binary Tree），是一道非常有意思的设计题。题目要求实现两个方法：serialize 把二叉树转成字符串，deserialize 把字符串还原为二叉树。序列化的格式没有限定，只要两个方法能配合工作即可。

层序序列化

最常见的方案是使用层序遍历，把空节点标记为 #：

def serialize(root):
    if not root:
        return "[]"
    res, queue = [], [root]
    while queue:
        node = queue.pop(0)
        if node:
            res.append(str(node.val))
            queue.append(node.left)
            queue.append(node.right)
        else:
            res.append("#")
    return "[" + ",".join(res) + "]"

def deserialize(data):
    if data == "[]":
        return None
    vals = data[1:-1].split(",")
    root = TreeNode(int(vals[0]))
    queue = [root]
    i = 1
    while queue and i < len(vals):
        node = queue.pop(0)
        if vals[i] != "#":
            node.left = TreeNode(int(vals[i]))
            queue.append(node.left)
        i += 1
        if i < len(vals) and vals[i] != "#":
            node.right = TreeNode(int(vals[i]))
            queue.append(node.right)
        i += 1
    return root

这里序列化和反序列化都用到了队列。序列化时，每个节点被访问一次，空节点用 # 标记。反序列化时，按照同样的顺序重建节点，用两个指针/索引来跟踪父子关系——一个指向当前要连接的父节点，一个指向待处理的子节点值。

N 叉树的序列化（LeetCode 428）思路类似，只是每个节点的子节点数量不固定，需要在序列化时记录子节点个数或者用特殊标记分隔。

踩坑与总结

坑 1：递归构建时的索引计算

在从序列重建二叉树时，索引 off-by-one 错误是最常见的 bug。强烈建议先画图：画一棵具体的树，写出它的前序、中序、后序遍历序列，然后手动模拟重建过程。把自己的手指当作指针，一步一步走——这比在脑子里空想要可靠得多。

坑 2：层序的空节点标记

层序遍历序列化时，最后一层如果全是 #，需不需要保留？答案是不需要——末尾连续的 # 是冗余的，去掉之后反序列化也能正确重建。不过 # 去掉后，反序列化时要注意提前结束的条件。

坑 3：递归和迭代的选择

在面试中，如果面试官只要求"实现前序遍历"，递归版本是最快最简洁的方案。但如果面试官追问"如果树有 10⁵ 个节点呢？"，你就需要立刻切换到迭代版本。我建议你对前序和中序的迭代遍历做到肌肉记忆——这几乎是每家大厂面试都会考的内容。

遍历方式 数据结构 递归/迭代 典型应用
前序 栈 递归简单，迭代中等 序列化、二叉树镜像
中序 栈 递归简单，迭代稍难 BST 验证、第 K 小
后序 栈 递归简单，迭代最难 删除节点、自底向上计算
层序 队列 迭代 BFS、右视图、锯齿遍历
构建树 递归 + HashMap 必须递归 105 / 106
序列化 队列/栈 迭代 297 / 428
本章心法：遍历二叉树就像走迷宫——前序是"先看入口再往里走"，中序是"走到最里面再回头看"，后序是"逛完再回来留个影"，层序是"一层一层扫荡"。每种方式对应一种解题视角。当你在 LeetCode 上遇到树题目时，先问自己：我应该用什么顺序访问节点？想清楚这个，代码就水到渠成了。
