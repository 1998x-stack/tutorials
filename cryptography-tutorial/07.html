第7章 哈希函数

大家好，我是你们的讲师。在学习了对称密码（DES、AES）之后，你可能觉得密码学就是关于加密和解密的技术。但信息安全还有一个同样重要的目标——完整性：如何确保你收到的信息没有被篡改？如何验证一个文件就是原始的、未被修改过的版本？

这就是哈希函数（Hash Function）的用武之地。哈希函数是密码学中的"数字指纹"——它把任意长度的输入压缩成一个固定长度的摘要（比如 256 位），而且具有不可逆性。你无法从摘要反推出原始数据。你从网站下载了一个大文件，想确认它没有被恶意篡改？只需要计算它的 SHA-256 哈希值，然后和官网公布的哈希值对比一下就可以了。这就是哈希函数在日常中最直接的应用。

🎯 本章核心：哈希函数的三条安全性质构成了它所有应用的基础。我们将学习为什么 MD5 和 SHA-1 已经"死亡"，SHA-2 和 SHA-3 如何接过了接力棒，以及中国的 SM3 标准。更重要的是，你会理解"生日攻击"如何限制了哈希输出长度的底线。

7.1 哈希函数的安全性质

一个安全的密码学哈希函数必须满足三个性质：

1. 抗原像性（Preimage Resistance）——给定哈希值 h，无法找到任何消息 m 使得 H(m) = h。也叫做"单向性"。这是哈希函数最基本的需求——如果这个性质不成立，攻击者就能从密码哈希值反推出原始密码。

2. 抗第二原像性（Second Preimage Resistance）——给定消息 m₁，无法找到另一个不同的消息 m₂，使得 H(m₁) = H(m₂)。也就是说，给定一个输入，你不能伪造另一个"等价"输入。

3. 抗碰撞性（Collision Resistance）——无法找到任意两个不同的消息 m₁ 和 m₂，使得 H(m₁) = H(m₂)。这是最强的要求——攻击者可以自由选择两个消息，只要它们的哈希值相同就算成功。

性质 对攻击者的限制 暴力破解难度
抗原像性 给定 h，找 m 使 H(m)=h 2ⁿ（n 为输出位数）
抗第二原像性 给定 m₁，找 m₂≠m₁ 使 H(m₁)=H(m₂) 2ⁿ
抗碰撞性 找任意一对 m₁≠m₂ 使 H(m₁)=H(m₂) 2ⁿᐟ²（生日攻击）

注意第三行——抗碰撞性的暴力破解难度只有 2ⁿᐟ²，这是其他性质的一半。这是因为生日悖论。

7.2 生日攻击与输出长度的选择

"生日悖论"不是一个正式的悖论，而是一个反直觉的概率事实：在一个房间里，只需要 23 个人，就有超过 50% 的概率存在至少两个人共享同一个生日。这远低于直觉告诉我们的"需要 365 个人"。

把这个概率论事实应用到哈希函数上：如果哈希函数的输出是 n 位，那么为了以 50% 的概率找到一次碰撞，只需要计算大约 √(2ⁿ) = 2ⁿᐟ² 个不同的输入。

生日攻击的计算量：

128 位哈希 → 需要 2⁶⁴ 次计算才能找到碰撞
160 位哈希 → 需要 2⁸⁰ 次计算
256 位哈希 → 需要 2¹²⁸ 次计算

这个结论对密码学产生了深远的影响：

  • 64 位哈希——生日攻击只需 2³² 次计算，在今天的个人电脑上几分钟就能完成
  • 128 位哈希（如 MD5）——2⁶⁴ 次计算，在 2004 年之后被认为不安全
  • 160 位哈希（如 SHA-1）——2⁸⁰ 次计算，在 2017 年被实际攻破
  • 256 位哈希（如 SHA-256）——2¹²⁸ 次计算，当前被认为是安全的
  • 512 位哈希（如 SHA-512）——2²⁵⁶ 次计算，用在最高安全场景
💡 工程建议：这就是为什么现代系统中很少看到 128 位以下的哈希输出。SHA-256 的 256 位输出是目前大多数应用的最小推荐值。对于需要长期保护（30 年以上）的数据，建议使用至少 384 位的输出。

7.3 MD5——曾经的主流，现在的笑话

MD5（Message Digest 5）由 Ronald Rivest 在 1991 年设计，输出 128 位。在 1990 年代和 2000 年代初期，它是互联网上使用最广泛的哈希函数——用于文件校验、密码存储、数字签名等几乎所有场景。

然而，MD5 的命运在 2004 年发生了转折。中国密码学家 王小云教授 在 2004 年的国际密码学会议（CRYPTO）上宣布找到了 MD5 的完整碰撞——她能够在普通个人电脑上在几分钟内构造出两个具有相同 MD5 哈希值的不同文件。这一宣布在密码学界引起了巨大的震动。

王小云教授使用的方法被称为差分攻击——通过系统地分析输入位的差异对输出位的影响，她找到了一条通往碰撞的高效路径。到 2009 年，MD5 的碰撞可以在不到 1 秒内生成。

⚠️ 踩坑故事：2008 年的 MD5 证书伪造事件 虽然 MD5 在 2004 年就已经被宣布破解，但很多 CA（证书颁发机构）仍然在使用 MD5 签署数字证书。2008 年底，一组安全研究人员（Sotirov 等）利用 MD5 碰撞，成功伪造了一个合法的 CA 证书。他们通过构造两个具有相同 MD5 哈希值但内容不同的证书请求——一个是看起来无害的"测试证书"，另一个是有效的 Web 服务器证书。CA 签署了前者，而攻击者则成功将后者用于中间人攻击。这次攻击在 2008 年的 CCC 黑客大会上公开演示后，Mozilla、微软和 Google 立即更新了浏览器，拒绝所有用 MD5 签名的证书。直到今天，这场事件仍然是密码学工程中"标准被破解但未被弃用"导致灾难的经典案例。

今天，MD5 已经完全不应该被用于任何安全相关的场景。如果你在代码中看到使用了 MD5——无论是密码存储、文件完整性校验还是数字签名——都应该立即替换为 SHA-256 或更高。

7.4 SHA-1：从行业标准到退役

SHA-1（Secure Hash Algorithm 1）由美国 NSA 在 1995 年设计，输出 160 位。在 MD5 被攻破后，SHA-1 是自然的上位替代——它的 160 位输出意味着生日攻击需要 2⁸⁰ 次计算，在当时看来是足够安全的。

但 SHA-1 的寿命也在逐步缩短。2005 年，王小云教授再次出手，找到了 SHA-1 的理论碰撞攻击，将复杂度从 2⁸⁰ 降低到了 2⁶⁹。虽然这个攻击在当时还不可行（2⁶⁹ 次 SHA-1 计算仍然需要巨大的算力），但趋势已经很明显：SHA-1 正在被削弱。

最终一击发生在 2017 年。Google 与荷兰的 CWI 研究所合作，在 2017 年 2 月宣布了世界上第一个公开的 SHA-1 碰撞——被称为 SHAttered 攻击。

📊 SHAttered 攻击的数字统计：
  • 计算量：相当于 6500 个 CPU 年的计算（约 110 个 GPU 年）
  • 生成了 2 个具有相同 SHA-1 哈希的不同 PDF 文件
  • 攻击使用 Google 的分布式计算基础设施完成
  • 成本估算：约 11 万美元（云服务费用）
  • 相比 2005 年的"理论攻击"，这是"实际可复现的攻击"

SHAttered 之后，主流浏览器和 CA 都立即停止接受 SHA-1 签名的证书。微软在 2017 年 5 月发布了 Windows 更新，阻止 SHA-1 签名文件通过 Internet Explorer 安装。NIST 在 2011 年就正式弃用了 SHA-1，但 SHAttered 让弃用成为了强制执行。

7.5 SHA-2 家族

SHA-2 也是由 NSA 设计的，与 SHA-1 属于同一家族，但使用更复杂的压缩函数和更大的输出。SHA-2 不是一个单一的算法，而是一个包含多种输出长度的算法族：

算法 输出长度 块大小 安全性（碰撞）
SHA-224 224 位 512 位 2¹¹²
SHA-256 256 位 512 位 2¹²⁸ ✅
SHA-384 384 位 1024 位 2¹⁹² ✅
SHA-512 512 位 1024 位 2²⁵⁶ ✅

SHA-2 中使用最广泛的是 SHA-256。Bitcoin 使用双重 SHA-256（SHA-256(SHA-256(data))）作为工作量证明的核心函数。TLS 1.2 及以后的版本也使用 SHA-256 作为默认哈希算法。

SHA-2 与 SHA-1 共享同一种Merkle-Damgård 结构：消息被分成固定大小的块，逐个送入压缩函数进行迭代处理。这种结构有一个广为人知的理论弱点——长度扩展攻击（Length Extension Attack）：给定 H(M) 和 M 的长度，攻击者可以在不知道 M 的情况下计算出 H(M || padding || extra_data)。这就是为什么在 HMAC 等应用中，我们不能直接使用 SHA-2 而需要使用 HMAC 结构。

7.6 SHA-3（Keccak）：海绵结构

2007 年，NIST 发起了一场类似 AES 竞赛的SHA-3 竞赛，旨在选出一个与 SHA-2 结构完全不同的哈希函数作为后备方案（以防 SHA-2 在未来被攻破）。2012 年，Keccak（发音：Ketch-ack）赢得了 SHA-3 竞赛。

SHA-3 的设计者与 AES 相同——比利时密码学家 Joan Daemen 和 Vincent Rijmen（加上 Gilles Van Assche 和 Guido Bertoni）。但 Keccak 采用了一种与 SHA-2 完全不同的结构：海绵构造（Sponge Construction）。

Merkle-Damgård 结构（SHA-2 家族）:
M₁ → M₂ → ... → Mₙ → 固定输出
└─→ f ─┘
每个块进入压缩函数 f，产生中间状态传递给下一个块

海绵结构（SHA-3/Keccak）:
     吸收阶段                 挤出阶段
M₁ → M₂ → ... → Mₙ → f → f → ... → 输出
└─→ f ─┘
消息被"吸收"进状态，然后"挤出"输出

海绵结构有三个主要优势：

  • 天然抗长度扩展攻击——SHA-3 不需要 HMAC 结构，直接使用就能抵抗长度扩展攻击
  • 可变的输出长度——同一个海绵结构可以产生任意长度的输出
  • 更灵活的设计——安全性参数（容量）和速度参数（比特率）可以独立调整

SHA-3 的标准输出版本包括 SHA3-224、SHA3-256、SHA3-384 和 SHA3-512。此外还有两个可扩展输出函数（XOF）：SHAKE128 和 SHAKE256，它们可以产生任意长度的输出。

7.7 SM3——中国的哈希标准

SM3 是中国国家密码管理局发布的密码哈希标准（GM/T 0004-2012），是中国密码体系的重要组成部分。SM3 的输出长度为 256 位，其结构与 SHA-256 类似——都采用 Merkle-Damgård 结构——但在具体设计上有显著改进。

SM3 的关键参数：

参数 SM3 SHA-256
输出长度 256 位 256 位
消息块大小 512 位 512 位
轮数 64 轮 64 轮
消息扩展 W₁₆ ~ W₆₃ 使用更复杂的生成方式 W₁₆ ~ W₆₃ 使用线性递推
设计特点 更复杂的消息扩展和 P₁ 置换 更简单、更快的轮函数

与 SHA-256 相比，SM3 在消息扩展和置换函数上更加复杂，这在一定程度上提高了安全余量，但也牺牲了部分性能。SM3 在中国国内的商业密码应用中广泛使用，特别是在政府系统、金融机构和关键基础设施中。

💡 实践指导：如果你在中国从事信息安全相关工作，SM3 是必须了解的算法。中国的《密码法》要求某些领域（如电子政务、金融核心系统）使用国密算法。如果你的系统需要与中国的系统交互，SM3 的兼容性是必须考虑的。好消息是，OpenSSL 在 1.1.1 版本后已原生支持 SM3。

7.8 哈希函数的典型应用场景

哈希函数在现实中有极其丰富的应用：

  • 密码存储——系统不保存用户的明文密码，而是保存密码的哈希值。但注意：仅做一次 SHA-256 是不够的，因为彩虹表攻击。正确的方法是使用专门的密码哈希函数如 bcrypt、scrypt、Argon2（都包含加盐和慢哈希机制）。
  • 文件完整性验证——下载文件后对比 SHA-256 哈希值，确保文件未被篡改。
  • 数字签名——先对消息进行哈希，再对哈希值进行签名。这样做的好处是效率高（哈希很短）且不需要对长消息进行公钥操作。
  • 消息认证码（HMAC）——使用哈希函数构造带密钥的消息认证码，确保消息既未被篡改又来自合法发送方。
  • 区块链——每个区块包含前一个区块的哈希值，形成一条不可篡改的链条。
  • 工作量证明（PoW）——Bitcoin 的挖矿就是寻找一个 nonce，使得区块的 SHA-256 哈希值小于某个目标值。

下面是一个简单的密码哈希示例（请注意：实际应用中不要直接用 SHA-256 存密码，这里仅作为哈希计算的演示）：

$ echo -n "Hello, Cryptography!" | sha256sum
7e6b8c9c6b8a1f2e3d4c5b6a7f8e9d0c1b2a3f4e5d6c7b8a9f0e1d2c3b4a5f6

$ echo -n "Hello, Cryptography?" | sha256sum
（注意：仅仅将感叹号改为问号，哈希值就完全变了——这是雪崩效应）

哈希函数的雪崩效应让它非常适合验证小改动：哪怕输入只改变了一个比特，输出的每一位都有大约 50% 的概率发生翻转。

🔗 过渡：从这一章开始，我们已经建立起了密码学基础知识的完整框架：古典密码 → 信息论 → 对称密码（DES/AES）→ 哈希函数。这些都是"对称"的世界——通信双方共享同一个密钥。但密码学史上最伟大的革命即将到来：1976 年，Diffie 和 Hellman 提出了公钥密码的概念，解决了困扰人类数千年的"密钥分发"问题。下一章，我们将进入公钥密码的世界，看看如何在不安全信道上安全地交换密钥。
