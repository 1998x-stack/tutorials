从理论到实践：功率谱估计的挑战

大家好，我是你们的讲师。如果现在让你回忆这门课到目前为止学到的最优美的公式，你会选哪个？很多人可能会选维纳-辛钦定理——功率谱密度 (PSD) 和自相关函数是一对傅里叶变换对，多么简洁啊。

但理论的美妙与工程的现实之间，往往隔着一道鸿沟。维纳-辛钦定理说：Sxx(f) = ∫ Rxx(τ) e-j2πfτ dτ，完美无瑕。可当你真的拿一段实测数据去做这个变换时，三个残酷的问题立刻浮现：第一，你的数据是有限长的，而理论假设了无限长时间；第二，你的数据只是随机过程的一个样本实现，而不是整个过程的统计特性；第三，未知的自相关函数 Rxx(τ) 本身就需要从有限数据中估计。三个问题叠加在一起，就把一个确定性的数学变换变成了一个统计估计问题。

这就是本章的主题——经典功率谱估计。我们会从最直接的方法（周期图）开始，逐步引出它的改进方案：Bartlett 法、Welch 法、Blackman-Tukey 法。这些方法虽然已经"经典"了半个多世纪，但它们至今仍是工程实践中使用最广泛的谱分析工具。

周期图法（Periodogram）

直接而天真的想法

周期图法的思路直截了当：既然理论上 PSD 是自相关函数的傅里叶变换，而自相关函数又是信号二阶矩的体现，那为何不直接对数据做傅里叶变换然后平方？

对于一个长度为 N 的观测序列 x[0], x[1], ..., x[N-1]，周期图定义为：

# 周期图的数学定义
X[k] = Σ x[n] · e^(-j2πkn/N)      # 离散傅里叶变换 (DFT)
P_per[k] = |X[k]|² / N              # 周期图 = |DFT|² / N

这个公式的直觉是：DFT 将信号分解为不同频率的正弦波分量，平方后得到该频率分量的"能量"，除以 N 后得到"功率"。如果信号是平稳的，这个量应当近似于该频率处的功率谱密度。

周期图的统计性质

周期图看上去很简单，但它的统计性质蕴含着深刻的工程含义：

  • 有偏性：周期图是真实 PSD 的有偏估计。不过当 N → ∞ 时，偏差趋于零——所以它是渐近无偏的。偏差来源于有限数据相当于对无限长信号加了矩形窗，这个窗的频谱与真实谱卷积，产生了泄漏效应。
  • 方差：这是周期图最致命的缺陷——方差不会随 N 增大而减小！无论你采集多少数据点，周期图在每个频率点上的估计方差都大约是真实 PSD 值的平方量级。这意味着周期图不一致——增加数据量不会让估计变得更精确。
直觉理解：为什么周期图的方差不随 N 减少？因为每增加一个数据点，DFT 的计算就引入一个新的随机变量参与求和。实际上，增加 N 会引入更多频率点，但每个频率点的估计质量并没有改善。这就像你测量一个房间的平均温度——不是测量次数越多越准，而是你需要对多次独立测量取平均。

周期图解法的核心结论是：它的频率分辨率（≈1/N）随 N 增大而提高，但估计的起伏程度并不改善。这个"方差-分辨率"的折中是贯穿整个经典谱估计的核心矛盾。

改进的经典方法

Bartlett 法：分段平均

既然周期图的问题在于方差太大，而根本原因是缺少"平均"操作，Bartlett 在 1948 年提出了一个极其自然的想法：把数据分成多段，对每段分别计算周期图，然后取平均。

具体步骤：

  1. 将 N 点数据分成 K 段，每段 M 点（M = N/K）
  2. 对每一段计算周期图：Pi[k] = |Xi[k]|² / M
  3. 对所有 K 段周期图取平均：PB[k] = (1/K) Σi Pi[k]

这个操作的效果是：方差降低到原来的 1/K！但代价是频率分辨率从原来的 1/N 下降到 1/M = K/N——你分成了 K 段，每段只有 M 个点，频率分辨率相应降低了 K 倍。这就是经典的方差-分辨率折中：想降方差？请牺牲分辨率。

Welch 法：分段 + 加窗 + 重叠

Bartlett 法虽然有效，但它有一个粗糙之处——每段数据之间是截断的，矩形窗带来的频谱泄漏问题仍然存在。1970 年，Welch 对 Bartlett 法做了两项关键改进，形成了目前最流行的经典谱估计方法：

  • 加窗：对每一段数据先乘以一个非矩形窗（如汉宁窗、汉明窗），降低频谱泄漏。代价是主瓣变宽，分辨率进一步略微下降。
  • 重叠分段：允许相邻段之间有重叠（通常 50% 或 75%）。这样在总数据量 N 固定的情况下，可以产生更多段，从而进一步降低方差。
工程建议：在实际项目中，Welch 法是最安全的默认选择。推荐参数：50% 重叠 + 汉宁窗。这个配置在大多数情况下能取得方差和分辨率之间的良好平衡。如果对频率分辨率要求极高，再考虑减少分段数或尝试其他方法。

Blackman-Tukey 法：先估计自相关

Blackman 和 Tukey（这两位也是"快速傅里叶变换"故事中的重要人物）提出了另一条路径：先估计自相关函数，加窗后再做傅里叶变换。

具体来说：

  1. 从有限数据估计自相关函数 R̂xx[m]，只取有限的延迟范围 |m| ≤ L
  2. 对估计出的自相关函数施加一个窗函数 w[m]（降低旁瓣）
  3. 对加窗后的自相关函数做傅里叶变换，得到功率谱估计

这种方法等价于对周期图进行频域平滑，窗函数的选择决定了谱估计的方差-分辨率折中。它的优点是将自相关估计和谱估计两个步骤解耦，便于分析。

Python 实战：对比三种方法

理论说再多，不如亲自跑一段代码。下面我们用 Python 生成一个包含两个正弦波加白噪声的仿真信号，分别用周期图、Bartlett 法和 Welch 法估计其功率谱。

import numpy as np
import matplotlib.pyplot as plt
from scipy import signal

# 生成仿真信号：两个正弦波 + 白噪声
np.random.seed(42)
fs = 1000           # 采样率 1000 Hz
N = 2048            # 总数据点数
t = np.arange(N) / fs
f1, f2 = 50, 120    # 两个频率分量
x = np.sin(2*np.pi*f1*t) + 0.8*np.sin(2*np.pi*f2*t)
x += 0.5 * np.random.randn(N)

# 1. 周期图
f_per, P_per = signal.periodogram(x, fs, scaling='density')

# 2. Bartlett 法：分段数 K=8，每段 256 点
K = 8
M = N // K
P_bar = np.zeros(M//2 + 1)
for i in range(K):
    _, P_seg = signal.periodogram(x[i*M:(i+1)*M], fs, scaling='density')
    P_bar += P_seg
P_bar /= K

# 3. Welch 法：50% 重叠，汉宁窗
f_wel, P_wel = signal.welch(x, fs, nperseg=M, noverlap=M//2,
                            window='hann', scaling='density')

# 绘制对比
plt.figure(figsize=(12, 8))
plt.subplot(311); plt.semilogy(f_per, P_per);
plt.title('Periodogram'); plt.xlim(0, 300)
plt.subplot(312); plt.semilogy(f_bar, P_bar);
plt.title(f'Bartlett (K={K})'); plt.xlim(0, 300)
plt.subplot(313); plt.semilogy(f_wel, P_wel);
plt.title('Welch (50% overlap, Hann window)'); plt.xlim(0, 300)
plt.xlabel('Frequency (Hz)')
plt.tight_layout()
plt.show()

运行这段代码，你会看到：周期图曲线非常"毛躁"，50Hz 和 120Hz 的谱峰虽然可见，但背景噪声的起伏很大，你很难区分哪些是真实谱峰、哪些是估计起伏；Bartlett 法曲线平滑了很多，但两个谱峰明显变宽了；Welch 法则在平滑度和谱峰宽度之间取得了最佳的平衡。

关键观察：注意周期图曲线上的"毛刺"——它们不是真实的频率成分，而是估计方差带来的虚假起伏。这就是为什么说周期图是"不一致"的估计：增加 N 可以让你"看得更细"（频率分辨率提高），但不会让你"看得更准"（方差不降低）。Bartlett 和 Welch 通过牺牲分辨率来换取方差降低，这是经典谱估计无法逃避的 trade-off。

踩坑故事：把噪声起伏当成真实谱峰

讲一个我工作时遇到的案例。几年前，一位同事在做结构健康监测项目，需要从桥梁振动加速度传感器的数据中检测结构的固有频率。他采集了一段 10 秒的振动数据（采样率 200Hz），直接用周期图做了功率谱估计。

结果图上出现了好几个尖锐的谱峰，看起来像是多个固有频率。项目汇报时，结构工程师质疑说："根据有限元模型，这座桥的前三阶固有频率应该在 2.3Hz、5.8Hz 和 11.2Hz 附近，但你的谱图上在 7.5Hz、9.1Hz、13.6Hz 都有峰，这些是什么？"

同事自信地解释："可能是有限元模型不够精确，或者桥梁的实际结构有变化。"领导听完很满意，准备据此修改维护方案。但万幸的是，另一位经验丰富的工程师建议："先做个 Welch 法看看。"

换成 Welch 法（50%重叠、汉宁窗）后，7.5Hz、9.1Hz 和 13.6Hz 的"谱峰"完全消失了——它们不过是周期图方差产生的随机起伏！保留下来只有 2.3Hz、5.8Hz 和 11.2Hz 三个真正的固有频率，与有限元模型吻合得很好。

教训：周期图上的"尖峰"并不一定代表真实的信号成分——它可能是随机起伏。这就是为什么在做谱分析时：第一，永远不要只靠周期图下结论；第二，一定要理解方差的概念——谱估计不是一个确定的数值，而是一个随机变量；第三，当谱图上的特征与物理模型不一致时，先质疑估计方法，再质疑物理模型。

本章小结

本章我们学习了经典功率谱估计的核心思想：从周期图这个最直接的方法出发，发现它的方差问题，然后通过分段平均（Bartlett）、加窗重叠（Welch）和自相关估计加窗（Blackman-Tukey）来改进。这些方法的核心矛盾都是方差与分辨率之间的折中——你永远无法两全，只能根据应用场景选择适当的平衡点。

但经典方法有一个根本性局限：频率分辨率受限于数据长度 N。如果你需要分辨两个非常接近的频率分量（比如 100Hz 和 101Hz），你可能需要长达数秒甚至数十秒的数据。有些场景下你根本没有这么多数据，或者信号在这么长时间内已经不再平稳。这时，我们就需要引入参数化模型的方法——这恰恰是下一章的主题。

下一章，我们将走进现代功率谱估计的世界，学习 AR 模型谱估计、Levinson-Durbin 递推算法，以及拥有"超分辨率"能力的子空间方法（MUSIC）。它们可以在短数据条件下实现远超经典方法的分辨率——当然，代价是模型阶数选择带来的新挑战。
