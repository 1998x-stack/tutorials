01 背包 — 二维 → 一维优化

大家好，我是你们的讲师。背包问题是动态规划最重要的分支之一，也是 LeetCode 面试中最高频的 DP 题型。背包问题的家族非常庞大，但万变不离其宗——核心都是"在容量限制下选择物品，最大化价值"。

我们先从最基础的 01 背包开始。所谓 01，就是每个物品要么选（1）要么不选（0），没有"选多次"这个选项。

首先看二维 DP 的标准写法：

def knapsack_01(weights, values, capacity):
    n = len(weights)
    # dp[i][c] 表示前 i 个物品在容量 c 下的最大价值
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        w, v = weights[i - 1], values[i - 1]
        for c in range(capacity + 1):
            if c < w:
                dp[i][c] = dp[i - 1][c]      # 装不下
            else:
                dp[i][c] = max(dp[i - 1][c],          # 不装
                               dp[i - 1][c - w] + v)  # 装
    return dp[n][capacity]

观察转移方程 dp[i][c] = max(dp[i-1][c], dp[i-1][c-w] + v)——dp[i][c] 只依赖 dp[i-1][...]，即上一行的值。这意味着我们可以把二维压缩成一维：

def knapsack_01_optimized(weights, values, capacity):
    n = len(weights)
    dp = [0] * (capacity + 1)

    for i in range(n):
        w, v = weights[i], values[i]
        for c in range(capacity, w - 1, -1):   # 逆序遍历！
            dp[c] = max(dp[c], dp[c - w] + v)

    return dp[capacity]

注意这里的关键：内层容量循环必须逆序。为什么？因为 dp[c] 依赖的是上一行的 dp[c] 和 dp[c-w]。如果正序遍历，dp[c-w] 已经被当前行的值覆盖了——一个物品就会被使用多次。逆序遍历保证了 dp[c-w] 还是上一行的旧值。

一维 01 背包口诀：外层循环物品，内层循环容量，容量逆序。这个"逆序"是 01 背包和完全背包的唯一区别，也是面试必考的知识点。

分割等和子集（LeetCode 416）

题目：给定一个只包含正整数的数组，判断能否将其分割成两个子集，使两个子集的元素和相等。这本质上是一个 01 背包问题——能否选出一些数使它们的和等于总和的一半。

状态定义：dp[c] 表示能否选出若干数使其和为 c。

def can_partition(nums):
    total = sum(nums)
    if total % 2 != 0:
        return False
    target = total // 2
    dp = [False] * (target + 1)
    dp[0] = True    # 和为 0 总是可以的

    for num in nums:
        for c in range(target, num - 1, -1):
            if dp[c - num]:
                dp[c] = True

    return dp[target]

目标和（LeetCode 494）

题目：给每个数字前面加上正号或负号，使表达式的值等于 target。这其实也是一个背包问题——把数字分成"加正号"和"加负号"两组，相当于求"加正号"组的总和等于特定值的方案数。

转换技巧：很多乍看不像背包的题目，其实可以转换成背包问题。通用的转换思路是：找到需要"选择"或"分配"的东西，找到容量限制，然后套入 01 背包或完全背包的框架。

完全背包 — 正序遍历容量

完全背包和 01 背包的唯一区别：每个物品可以无限次选取。而实现"无限次"的方法就是——内层容量循环正序遍历。

def knapsack_complete(weights, values, capacity):
    n = len(weights)
    dp = [0] * (capacity + 1)

    for i in range(n):
        w, v = weights[i], values[i]
        for c in range(w, capacity + 1):     # 正序遍历！
            dp[c] = max(dp[c], dp[c - w] + v)

    return dp[capacity]

正序遍历时，dp[c-w] 可能已经被当前物品更新过了——这正好允许了同一个物品被多次选取。这就是完全背包的"无限次"的奥秘。

零钱兑换（LeetCode 322）

题目：给定不同面额的硬币和总金额，求凑出该金额所需的最少硬币数量（每种硬币无限使用）。这是完全背包求最小值的经典题。

def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0

    for coin in coins:
        for a in range(coin, amount + 1):
            dp[a] = min(dp[a], dp[a - coin] + 1)

    return dp[amount] if dp[amount] != float('inf') else -1

零钱兑换 II（LeetCode 518）

同样的输入，但问的是凑成总金额的硬币组合数（注意是"组合"不是"排列"）。这里就涉及到了遍历顺序的微妙影响。

def change(amount, coins):
    dp = [0] * (amount + 1)
    dp[0] = 1

    for coin in coins:             # 先物品
        for a in range(coin, amount + 1):
            dp[a] += dp[a - coin]

    return dp[amount]

这个"先物品、正序容量"的嵌套顺序，保证了每个硬币的出现顺序是固定的——这产生的是组合数。如果调换内外循环的顺序（先容量再物品），那得到的就是排列数。

面试重点：面试官特别爱问的一个陷阱是——"零钱兑换 II 中，如果我把硬币循环放内层、金额循环放外层，结果会怎么样？"答案是：你会数出排列数而不是组合数，结果远大于正确答案。理解这个区别，就是理解了背包问题的精髓。

组合 vs 排列 — 遍历顺序决定结果

这个知识点太重要了，值得单独拿出来强调。在完全背包问题中，先物品后容量 → 组合，先容量后物品 → 排列。

为什么？因为当物品循环在外层时，每个物品在 dp 中出现的顺序是固定的——先处理的物品总是出现在后处理的物品之前。这就保证了 {coin1=1, coin2=2} 和 {coin1=2, coin2=1} 不会同时出现，因为 2 只能在 1 之后出现。

而当容量循环在外层时，每一轮容量都会尝试所有物品，因此不同的顺序会被计数多次。

组合总和 IV（LeetCode 377）

这道题虽然叫"组合总和"，但实际上问的是排列数——[1,1,2] 和 [1,2,1] 被视为不同的结果。所以需要用先容量后物品的遍历顺序：

def combination_sum4(nums, target):
    dp = [0] * (target + 1)
    dp[0] = 1

    for t in range(1, target + 1):     # 先容量
        for num in nums:               # 后物品
            if t >= num:
                dp[t] += dp[t - num]

    return dp[target]

看看，只是把内外循环换了个顺序，结果就从"组合数"变成了"排列数"。这就是背包问题最神奇的地方。

多维背包与变形

当背包的限制条件从一个维度扩展到多个维度时，就变成了多维背包。

一和零（LeetCode 474）

题目：给定一个字符串数组和两个容量 m（0 的个数上限）和 n（1 的个数上限），求最多能选多少个字符串。每个字符串相当于一个物品，消耗一定数量的 0 和 1。

def find_max_form(strs, m, n):
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for s in strs:
        zeros = s.count('0')
        ones = s.count('1')
        for i in range(m, zeros - 1, -1):
            for j in range(n, ones - 1, -1):
                dp[i][j] = max(dp[i][j], dp[i - zeros][j - ones] + 1)

    return dp[m][n]

二维容量 + 逆序遍历，就是 01 背包的多维版本。

踩坑与总结

坑 1：背包问题统一心法

很多同学觉得背包问题"题型多、记不住"。我的建议是：不要死记题型，要记住框架。背包问题的核心框架只有三个变量：物品 i、容量 c、价值 v。不同的题目只是在问法上做文章——"能否凑出"、"最多能选多少"、"最少需要几个"、"有几种凑法"——但这些都只是转移方程中 max/min/+/or 的替换。

坑 2：初始化傻傻分不清

背包问题初始化最容易出错的地方：

  • 求最大值： dp[0] = 0，其他 dp = 0 或 -inf？如果问题要求"恰好装满"，则 dp[0] = 0，其他 dp = -inf（表示不可达）。
  • 求最小值： dp[0] = 0，其他 dp = inf（如零钱兑换 322）。
  • 求方案数： dp[0] = 1（空集也是一种方案），其他 dp = 0。
  • 求是否存在： dp[0] = True，其他 dp = False（如分割等和子集 416）。

分享一个我的踩坑故事。几年前我准备面试时，刷 LeetCode 416 分割等和子集，我想当然地把 dp 初始化成全 True，然后把 dp[0] = True 写在后面。结果所有测试用例都返回 True——因为一个全 True 的 dp 数组，当然每个容量都"可达"。浪费了我一个小时才发现这个低级错误。从此以后，初始化是我写 DP 代码时第一件检查的事。

坑 3：忘记判断容量下限

在遍历容量时，for c in range(target, num - 1, -1) 中的 num - 1 很容易写错成 num 或 0。写错了不会报错——只是多了一些无用的循环，或者少了一些必要的循环。小数据量可能测不出来，但大数据量下要么 TLE 要么 WA。

本章心法：背包问题的核心就是三个维度——物品、容量、价值；两种类型——01 背包（逆序）和完全背包（正序）；两个顺序——组合（先物品）和排列（先容量）。掌握了这张表，LeetCode 上所有背包问题你都能快速归类并写出代码。

背包问题是 DP 中最标准化的分支——框架固定、变化有限。但 DP 的世界远不止背包这一种形态。下一章我们将进入区间 DP 与状态机，探索区间问题（最长回文子串）、股票买卖中的状态机模型、状态压缩 DP 和树形 DP。这些知识将带你从 DP 的入门阶段，迈向真正的进阶之路。下一章见！
