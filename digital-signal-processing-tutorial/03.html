第3章 离散时间系统

上一章我们认识了"信号"——那串按时间排列的数字。现在我们要问一个很自然的问题：如果你把这串数字丢给某个"处理单元"，它会输出什么？这个"处理单元"就是离散时间系统。

说得直白一点，离散时间系统就是一个输入-输出映射：你送进去一个序列 x[n]，它吐出来一个序列 y[n]。我们用算子 T{·} 来抽象的表示这个映射：y[n] = T{x[n]}。这个算子可以是任何东西——一个简单的差分方程，一个复杂的滤波算法，甚至是一个神经网络。在接下来的几章里，我们的核心任务就是搞清楚一个问题：看到 T{·}，如何判断系统的性质？如何计算输出？

3.1 系统的四大基本性质

一个离散时间系统可以有很多性质，但在DSP中，有四个性质是最核心的判断标准。其中前两个（线性和时不变性）尤其重要——因为只有满足这两条的系统才能用卷积来分析和设计。

性质 定义 数学表达
线性 满足齐次性和叠加性 T{a·x₁ + b·x₂} = a·T{x₁} + b·T{x₂}
时不变性 输入移位导致输出等量移位 若 y[n] = T{x[n]}, 则 T{x[n-k]} = y[n-k]
因果性 输出只取决于当前及过去的输入 y[n] 只依赖于 x[n], x[n-1], x[n-2], …
稳定性 有界输入产生有界输出（BIBO） 若 |x[n]| < ∞, 则 |y[n]| < ∞
💡 直觉记忆：线性保证你可以"分解问题"——把复杂输入拆成简单部分的叠加分别处理再求和；时不变保证你"一次学会，到处应用"——知道了系统对一个冲激的响应，就知道对任何时间位置的冲激的响应。

来看几个具体的例子。假设一个系统定义为 y[n] = 2x[n] + 3，这是什么性质？它不满足线性（因为加了一个常数3破坏了齐次性），但它是时不变的（输入移位导致输出等量移位），它是因果的（只依赖当前输入），也是稳定的（有界输入产生有界输出）。另一个系统 y[n] = n·x[n] 呢？它是线性的，但它是时变的——因为系数n依赖于时间索引。这种时变系统在自适应滤波（第15章）中非常常见。

3.2 冲激响应与卷积和

现在来到本章（甚至整个课程）最重要的一页。我们要推导一个极其强大且优美的结论：如果系统是线性且时不变的（LTI），那么给定冲激响应h[n]，系统对任何输入x[n]的响应都可以用一个公式计算——这个公式就是卷积和。

推导思路是这样的：

第一步：任何序列都可以表示为移位冲激的加权和。这是第2章的一个关键观察：

        x[n] = Σ_{k=-∞}^{∞} x[k]·δ[n-k]

        比如 x[n] = [3, 1, 2]（在n=0,1,2处）可以写成：
        x[n] = 3·δ[n] + 1·δ[n-1] + 2·δ[n-2]

第二步：将系统T{·}作用在等式两边。由线性（和的响应=响应的和）和时不变（δ[n-k]的响应=h[n-k]）：

        y[n] = T{x[n]} = T{ Σ_k x[k]·δ[n-k] }
                      = Σ_k x[k]·T{δ[n-k]}    （线性）
                      = Σ_k x[k]·h[n-k]       （时不变）

        这就是卷积和！

记作 y[n] = x[n] * h[n]，其中"*"表示卷积运算符（不是乘法！）。

⌨️ 记忆口诀：卷积就是"先翻转、再移位、再相乘累加"。具体步骤：把h[k]翻转为h[-k]，然后向右滑动n步（得到h[n-k]），和x[k]逐点相乘后对所有k求和。这就是"卷积"这个名字的由来——"卷"就是翻转，"积"就是乘积累加。

3.3 卷积的计算方法

理论推导很优美，但真正做题和写代码的时候怎么算？有三种常用方法。

方法一：图形法（手算利器）

假设 x[n] = [1, 2, 3]（n=0,1,2），h[n] = [1, 1]（n=0,1）。手算卷积：

        y[0] = x[0]·h[0]            = 1·1 = 1
        y[1] = x[0]·h[1] + x[1]·h[0] = 1·1 + 2·1 = 3
        y[2] = x[1]·h[1] + x[2]·h[0] = 2·1 + 3·1 = 5
        y[3] = x[2]·h[1]             = 3·1 = 3

        结果：y[n] = [1, 3, 5, 3]  (n=0,1,2,3)

注意一个非常实用的规律：两个序列卷积后的长度 = 两序列长度之和减1。L_x = 3，L_h = 2，L_y = 3+2-1 = 4。这个规律在后面的DFT和FFT章节中会用来检查线性卷积的正确性。

方法二：列表法（适合程序实现）

        以滑动窗口的方式理解：将x放在一个长条上，h翻转后从左向右滑动，
        重合部分的乘积累加即为该位置的输出。

        这本质上就是FIR滤波器的计算流程 —— 第9章会详细展开。

方法三：Python代码（工程实现）

        import numpy as np

        def convolve(x, h):
            Lx, Lh = len(x), len(h)
            Ly = Lx + Lh - 1
            y = np.zeros(Ly)
            for n in range(Ly):
                for k in range(Lx):
                    if 0 <= n - k < Lh:
                        y[n] += x[k] * h[n - k]
            return y

        x = np.array([1, 2, 3])
        h = np.array([1, 1])
        print(convolve(x, h))  # [1. 3. 5. 3.]

        # 跟numpy内置对比
        print(np.convolve(x, h))  # [1 3 5 3]

在实际DSP芯片上，卷积是用乘累加（MAC）指令单周期完成的——dsp芯片一个时钟周期就能完成一次乘法+累加。TI TMS320C64x系列可以在一个时钟周期内执行4次MAC操作，这就是为什么实时信号处理在嵌入式芯片上也能跑得动。

⚠️ 踩坑实录：曾经有一个做音频后处理的朋友，在实现一个简易混响效果时，手动写了一个卷积函数。他把输入信号和冲激响应（一个5000点的房间脉冲响应）做卷积，用于模拟房间回声。结果播放时发现延迟大到不可接受（500ms以上）。问题出在哪？他用了直接卷积（O(N²)级的复杂度），输入信号每秒48000点，冲激响应5000点，每次卷积需要做 48000×5000 = 2.4亿次乘加运算——CPU根本扛不住。后来改用FFT加速卷积（基于卷积定理，把时域卷积变成频域乘积），计算量直接降到O(N log N)，延迟降低到可忽略的几毫秒。这个故事告诉我们：卷积的直觉很重要，但工程实现中几乎从不直接用暴力卷积——都是FFT加速。我们在第8章FFT之后会专门讲这个技巧。

3.4 LTI系统的稳定性与因果性判据

对于一个LTI系统，冲激响应h[n]包含了系统的全部信息。系统的稳定性、因果性可以直接从h[n]判断：

  • BIBO稳定 ⇔ h[n]绝对可和： Σ_{n=-∞}^{∞} |h[n]| < ∞。换句话说，冲激响应必须是能量有限的衰减序列。如果h[n]是无穷长且不衰减的（比如h[n]=u[n]），那么在某些有界输入下输出会发散。
  • 因果 ⇔ h[n] = 0 for n < 0：系统不能"预知未来"。如果n<0时h[n]≠0，意味着冲激还没到就已经有响应了——这在实时系统中做不到。

这里有一个实际意义：我们设计的所有可实现的数字滤波器，h[n]必须满足n<0时为零（因果），且|h[n]|随n增加而衰减（稳定）。第9章和第10章设计滤波器时，这两条是必须满足的约束。

3.5 互联系统

在实际工程中，很少只有一个孤立的LTI系统。通常是把多个子系统以不同方式连接起来：

连接方式 等效冲激响应 应用场景
级联（串联） h[n] = h₁[n] * h₂[n] 多级滤波器串联实现高性能滤波
并联 h[n] = h₁[n] + h₂[n] 多频段均衡器（图形均衡器）
反馈连接 h[n] = h₁[n] * (δ[n] + h₂[n] + h₂[n]*h₂[n] + …) IIR滤波器（递归结构）

级联的交换律是卷积乘法的一个漂亮性质：h₁[n] * h₂[n] = h₂[n] * h₁[n]。这意味着级联滤波器的顺序不影响最终结果——当然这是在数学上，定点实现中不同的顺序会导致不同的量化误差，第20章会涉及。
