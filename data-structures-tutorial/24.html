数组与哈希

1. 两数之和 (LeetCode 1) — 面试"Hello World"

如果面试题也有"入门仪式"，那一定是两数之和。它简单、优雅、完美展示了"空间换时间"的核心思想。

为什么选哈希表？ 暴力法是O(n²)——两层循环枚举所有数对。哈希表的核心洞察是：当我们遍历到nums[i]时，不需要往后找，只需要问"target - nums[i] 之前出现过吗？" 哈希表的O(1)查找能力让这个查询瞬间完成。

def two_sum(nums, target):
    """LeetCode 1. 两数之和 — 哈希表解法 O(n)"""
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return [-1, -1]

"""
复杂度推导：
- 时间：O(n) —— 遍历一次数组，每次哈希表操作O(1)
- 空间：O(n) —— 哈希表中最多存储n个元素
- 能不能优化空间？如果数组已排序，可用双指针做到O(1)空间

口述模板：
"这道题最直观的解法是暴力枚举，时间复杂度O(n²)。
最优解是用哈希表，遍历数组时检查target - nums[i]是否已经在哈希表中。
如果在，说明找到了两个数；如果不在，把当前数加入哈希表。
这样做的时间复杂度是O(n)，空间复杂度是O(n)。"

易错点：
- 不能使用同一个元素两次：边遍历边加入哈希表就避免了这个问题
- 返回的是下标，不是值本身
"""

2. 三数之和 (LeetCode 15) — 从两数到三数的跨越

两数之和的升级版。难点不只是"多了一个数"，而是去重——如果数组里有重复元素，如何保证结果不包含重复的三元组？

def three_sum(nums):
    """LeetCode 15. 三数之和 — 排序+双指针 O(n²)"""
    nums.sort()
    res = []
    n = len(nums)
    for i in range(n - 2):
        if nums[i] > 0:          # 排序后第一个数>0，后面不可能和为0
            break
        if i > 0 and nums[i] == nums[i - 1]:  # 跳过重复的i
            continue
        left, right = i + 1, n - 1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            if s == 0:
                res.append([nums[i], nums[left], nums[right]])
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1
                left += 1
                right -= 1
            elif s < 0:
                left += 1
            else:
                right -= 1
    return res

"""
复杂度推导：
- 排序O(n log n) + 外层循环O(n) × 双指针O(n) = O(n²)
- 空间：O(1) 额外空间（排序可能用O(log n)栈空间），不算结果数组

为什么选排序+双指针？
- 排序让数组有序，有序才能用双指针
- 双指针把三层循环降成了两层循环
- 排序还天然帮助了去重

易错点：
- 三个地方需要去重：i, left, right
- 先移动指针再去重 vs 先去重再移动指针——要画图理解
"""
"两数之和→三数之和→四数之和" 是一条完整的思维链：
两数和(无序)→哈希表；两数和(有序)→双指针；三数和→固定一个+双指针；四数和→固定两个+双指针。这就是从一道题学到一类题的典型案例。

链表经典

3. 反转链表 (LeetCode 206) — 链表题的"太极拳"

上一章我们已经详细讲过反转链表的两种写法。这里补充一点：面试中优先写迭代版——空间O(1)比递归的O(n)更优，而且不容易栈溢出。但如果面试官追问"你能用递归写吗？"也要能写出来。

def reverse_list(head):
    """迭代反转 — 面试推荐写法"""
    prev = None
    curr = head
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    return prev

"""
复杂度：O(n) 时间，O(1) 空间
口述："三指针法，每次反转当前节点和上一个节点的指向关系。
"""

4. 环形链表 II (LeetCode 142) — 快慢指针 + 数学推理

这题在面试中经常作为"追问题"出现——不仅判断是否有环，还要找到环的入口。

def detect_cycle(head):
    """LeetCode 142. 环形链表 II — 快慢指针"""
    slow = fast = head
    has_cycle = False
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            has_cycle = True
            break
    if not has_cycle:
        return None

    # 数学推导：相遇后，一个指针从头走，一个从相遇点走
    # 再次相遇处即为环入口
    slow = head
    while slow != fast:
        slow = slow.next
        fast = fast.next
    return slow

"""
复杂度：O(n) 时间，O(1) 空间

口述模板：
"第一步用快慢指针判断是否有环。快指针每次走两步，慢指针每次走一步，
如果相遇说明有环。
第二步找环入口，一个指针从头开始，一个指针从相遇点开始，
每次都走一步，再次相遇的地方就是环入口。"

数学推导（简单说一下，不用背公式）：
设链表头到环入口距离为a，环入口到相遇点距离为b，相遇点到环入口为c。
慢指针走了a+b，快指针走了a+b+k(b+c)。因为快指针走的步数是慢指针的2倍，
所以2(a+b)=a+b+k(b+c)，得出a=(k-1)(b+c)+c。所以从头和相遇点同时出发，
一定在环入口相遇。

易错点：
- 快慢指针的初始化：都从头节点开始
- while循环条件：fast and fast.next 都要检查
"""

5. 有效括号 (LeetCode 20) — 栈的经典应用

def is_valid(s):
    """LeetCode 20. 有效括号 — 栈"""
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for ch in s:
        if ch in pairs:          # 右括号
            if not stack or stack[-1] != pairs[ch]:
                return False
            stack.pop()
        else:                    # 左括号
            stack.append(ch)
    return not stack             # 栈为空才合法

"""
复杂度：O(n) 时间，O(n) 空间
易错点：栈为空时遇到右括号、遍历结束后栈不为空
"""

6. 滑动窗口最大值 (LeetCode 239) — 单调队列

from collections import deque

def max_sliding_window(nums, k):
    """LeetCode 239. 滑动窗口最大值 — 单调递减双端队列"""
    dq = deque()  # 存储下标，对应值单调递减
    result = []

    for i, num in enumerate(nums):
        # 移除超出窗口左界的下标
        if dq and dq[0] <= i - k:
            dq.popleft()

        # 维护单调递减：移除队尾所有比当前元素小的
        while dq and nums[dq[-1]] <= num:
            dq.pop()

        dq.append(i)

        # 窗口形成后，队首就是最大值
        if i >= k - 1:
            result.append(nums[dq[0]])

    return result

"""
复杂度：O(n) —— 每个元素最多入队出队一次
空间：O(k) —— 队列最多存储k个元素

为什么选单调队列？
- 暴力法是O(nk)，每次窗口都要扫描k个元素
- 单调队列维护了"潜在的最大值候选"，每次窗口滑动只需O(1)
- 核心思想：如果新元素比队尾大，队尾的元素永远不可能是最大值了
"""

7. 二叉树中序遍历 (LeetCode 94) — 递归 + 迭代

def inorder_traversal(root):
    """递归版 — 最简单"""
    res = []
    def dfs(node):
        if not node:
            return
        dfs(node.left)
        res.append(node.val)
        dfs(node.right)
    dfs(root)
    return res

def inorder_traversal_iterative(root):
    """迭代版 — 面试加分项，手动模拟栈"""
    res = []
    stack = []
    curr = root
    while curr or stack:
        while curr:            # 一直往左走
            stack.append(curr)
            curr = curr.left
        curr = stack.pop()     # 弹出，访问
        res.append(curr.val)
        curr = curr.right      # 转向右子树
    return res

"""
复杂度：O(n) 时间，O(n) 空间
面试建议：先写递归版（快），再写迭代版（展示你理解栈的原理）
"""

8. 层序遍历 (LeetCode 102) — BFS模板

from collections import deque

def level_order(root):
    """LeetCode 102. 二叉树的层序遍历 — BFS"""
    if not root:
        return []
    res = []
    q = deque([root])
    while q:
        level_size = len(q)
        level = []
        for _ in range(level_size):
            node = q.popleft()
            level.append(node.val)
            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)
        res.append(level)
    return res

"""
这就是BFS的标准模板！可以套用到很多题目上：
- 二叉树的右视图(199)：取每层的最后一个元素
- 之字形遍历(103)：加一个方向标志
- 最大深度(104)：统计层数
"""

9. 岛屿数量 (LeetCode 200) — 三种解法，一种思维

这题是面试中的"多解法之王"——可以用DFS、BFS、并查集三种方法求解。面试官如果让你"换一种解法"，你能立刻切换。

def num_islands(grid):
    """LeetCode 200. 岛屿数量 — DFS解法"""
    if not grid:
        return 0
    rows, cols = len(grid), len(grid[0])
    count = 0

    def dfs(r, c):
        if r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] == '0':
            return
        grid[r][c] = '0'  # 淹没法：将访问过的陆地标记为水
        dfs(r + 1, c)
        dfs(r - 1, c)
        dfs(r, c + 1)
        dfs(r, c - 1)

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                count += 1
                dfs(r, c)  # 将整个岛屿淹没
    return count

"""
复杂度：O(m×n) 时间，O(min(m,n)) 空间（递归栈深度）
BFS解法：用队列代替递归栈，本质一样
并查集解法：每个陆地作为节点，相邻的陆地union，最后统计root数量

口述模板：
"遍历整个网格，遇到1就计数加一，然后用DFS把这个1相连的所有1都变成0，
相当于淹没了整个岛屿。继续遍历，下一个1就是新的岛屿。"
"""
这些题不是"背答案"用的，是"理解思维"用的。面试官大概率不会直接考原题——他会在原题上做变形。比如两数之和变成"三数之和最接近target"（LeetCode 16），反转链表变成"反转链表中的一段"（LeetCode 92）。所以，不要背代码，要理解每道题背后的为什么——为什么选这个数据结构？为什么这个复杂度？边界条件如何处理？

10. 爬楼梯 (LeetCode 70) — DP入门第一题

def climb_stairs(n):
    """LeetCode 70. 爬楼梯 — 斐波那契DP"""
    if n <= 2:
        return n
    a, b = 1, 2   # dp[1], dp[2]
    for _ in range(3, n + 1):
        a, b = b, a + b  # dp[i] = dp[i-1] + dp[i-2]
    return b

"""
复杂度：O(n) 时间，O(1) 空间
为什么用DP？因为爬i阶的方法数 = 爬i-1阶再爬1步 + 爬i-2阶再爬2步
这本质就是斐波那契数列，可以用滚动变量优化空间。
"""

11. 最长递增子序列 (LeetCode 300) — LIS经典

def length_of_lis(nums):
    """LeetCode 300. 最长递增子序列 — 耐心排序 + 二分"""
    tails = []  # tails[i] = 长度为i+1的子序列的最小末尾值
    for num in nums:
        # 二分查找tails中第一个 >= num 的位置
        left, right = 0, len(tails)
        while left < right:
            mid = (left + right) // 2
            if tails[mid] < num:
                left = mid + 1
            else:
                right = mid
        if left == len(tails):
            tails.append(num)
        else:
            tails[left] = num
    return len(tails)

"""
复杂度：O(n log n) 时间，O(n) 空间

为什么叫"耐心排序"？
tails数组维护了一个性质：它始终是递增的。
对于每个num，找到tails中第一个>=num的位置，替换掉它。
如果num比所有tails元素都大，就追加到末尾。
tails的长度就是LIS的长度。

这个解法不要求背，但要理解"为什么tails的长度就是LIS"——这是贪心的思想。
"""

12. 最长公共子序列 (LeetCode 1143) — 二维DP模板

def longest_common_subsequence(text1, text2):
    """LeetCode 1143. 最长公共子序列 — 二维DP"""
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[m][n]

"""
复杂度：O(m×n) 时间，O(m×n) 空间（可优化到O(n)）

DP定义：dp[i][j] = text1前i个字符和text2前j个字符的LCS长度
转移方程：
- 如果text1[i-1] == text2[j-1]：dp[i][j] = dp[i-1][j-1] + 1
- 否则：dp[i][j] = max(dp[i-1][j], dp[i][j-1])
"""

设计题

13-15. LRU缓存 (LeetCode 146) — 系统设计的"入门题"

这道题在面试中的地位极其特殊——它是纯算法题和系统设计题的桥梁。面试官通过这道题考察你能否设计一个"真实系统中的组件"。

为什么选哈希表+双向链表？

  • 我们需要O(1)的get——哈希表可以做到。
  • 我们需要O(1)的put——哈希表也可以。
  • 当缓存满时，需要淘汰最久未使用的元素——这需要维护一个"访问顺序"，双向链表正好可以做到：最近访问的移到头部，最久未使用的在尾部。
  • 为什么是双向链表不是单向？因为我们需要O(1)删除任意节点（在 put 已存在的key时需要把节点移到头部），单向链表做不到O(1)删除。
class ListNode:
    def __init__(self, key=0, value=0):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None

class LRUCache:
    """LeetCode 146. LRU缓存 — 哈希表+双向链表"""

    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}                 # key -> node
        # 哨兵节点，避免处理头尾边界
        self.head = ListNode()          # 虚拟头，head.next是真正的最少使用
        self.tail = ListNode()          # 虚拟尾，tail.prev是真正的最少使用
        self.head.next = self.tail
        self.tail.prev = self.head

    def _remove(self, node):
        """从链表中移除节点"""
        node.prev.next = node.next
        node.next.prev = node.prev

    def _add_to_head(self, node):
        """将节点添加到头部（最近使用）"""
        node.next = self.head.next
        node.prev = self.head
        self.head.next.prev = node
        self.head.next = node

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        node = self.cache[key]
        # 移到头部表示最近使用过
        self._remove(node)
        self._add_to_head(node)
        return node.value

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            # key已存在：更新值，移到头部
            node = self.cache[key]
            node.value = value
            self._remove(node)
            self._add_to_head(node)
        else:
            # key不存在：创建新节点
            if len(self.cache) >= self.capacity:
                # 淘汰最久未使用的（尾部节点）
                lru = self.tail.prev
                self._remove(lru)
                del self.cache[lru.key]
            node = ListNode(key, value)
            self.cache[key] = node
            self._add_to_head(node)

"""
复杂度分析：
- get: O(1) —— 哈希表查找 + 链表操作（均O(1)）
- put: O(1) —— 哈希表插入 + 链表操作（均O(1)）
- 空间：O(capacity)

口述模板：
"LRU缓存的核心是哈希表+双向链表。哈希表提供O(1)的key查找，
双向链表维护访问顺序——最近访问的节点在头部，最久未使用的在尾部。
get时把访问的节点移到头部，put时如果缓存已满就淘汰尾部节点。

为什么用双向链表？因为删除任意节点需要O(1)，这要求节点知道前后邻居。
为什么用哨兵节点？避免处理头尾为空的边界情况。

面试追问1：为什么不用单向链表？
答：单向链表删除节点需要找到前驱节点，需要O(n)遍历。

面试追问2：可以用OrderedDict实现吗？
答：Python的OrderedDict可以实现同样的功能，但面试时用双向链表实现
更能展示对底层原理的理解。

如何最大化这15道题的价值？
不要看完就过去。对每一道题，执行下面的步骤：
1. 自己写一遍代码，确保bug-free
2. 对着镜子口述复杂度分析
3. 想一个面试官可能追问的问题
4. 找到这道题的2-3道变体题
如果每道题都这样做一遍，这15道题的训练效果不亚于盲目刷100道。

本章小结

在这一章中，我们精讲了15道大厂面试高频题，覆盖了六大题型方向：

  • 数组与哈希——两数之和、三数之和：空间换时间和去重技巧
  • 链表经典——反转链表、环形链表II：递归/迭代和快慢指针
  • 栈与队列——有效括号、滑动窗口最大值：单调栈/队列思想
  • 树与图——中序遍历、层序遍历、岛屿数量：DFS/BFS/并查集
  • 动态规划——爬楼梯、LIS、LCS：一维/二维DP模板
  • 设计题——LRU缓存：哈希表+双向链表的经典组合

这些题目是面试的"基础武器"，但它们只是工具。真正决定你能不能在面试中脱颖而出的，是你能否在系统设计中灵活运用这些数据结构——这正是我们下一章的主题。

在第25章，我们将跳出LeetCode，看看在真实的系统设计中——短网址、实时排行榜、限流器、Feed流推送、即时通讯——数据结构和算法是如何成为系统架构的灵魂的。你会发现，那些在刷题中学到的数据结构，在真实的工程世界里有更广阔的应用场景。

我们第25章见。
