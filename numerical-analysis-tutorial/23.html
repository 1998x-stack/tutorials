第23章 矩阵特征值计算

23.1 特征值问题概述

大家好，我是你们的讲师。今天我们要聊一个非常重要的话题：矩阵特征值（Eigenvalue）。如果说前面几章我们一直在解决"解方程"的问题——解线性方程组、解非线性方程——那么从这一章开始，我们要问一个更深层的问题：矩阵到底"长什么样"？

特征值和特征向量就是回答这个问题的工具。对于一个方阵 A，如果存在一个非零向量 v 和一个标量 λ，满足：

Av = λv

那么 λ 就是 A 的特征值（eigenvalue），v 是对应的特征向量（eigenvector）。几何上说，特征向量是那些被 A 作用后方向不变、仅伸缩的特殊向量，伸缩倍数就是特征值。

你可能会问："我为什么要关心特征值？" 答案是：特征值无处不在。下面这张表展示了一些典型应用：

领域 应用 特征值/特征向量扮演的角色
结构力学 桥梁/建筑振动分析 特征值 = 固有频率的平方，特征向量 = 振型
数据科学 PCA 主成分分析 特征向量 = 主方向，特征值 = 方差大小
搜索引擎 Google PageRank PageRank 向量 = 转移矩阵的主特征向量
量子力学 薛定谔方程 特征值 = 能级，特征向量 = 波函数
控制系统 稳定性分析 系统矩阵特征值的实部决定系统是否稳定
图论 谱聚类 Laplacian 矩阵的特征向量用于降维聚类
流体力学 线性稳定性分析 Navier-Stokes 线性化算子的特征值决定流态稳定性

从这张表可以看出，特征值问题横跨了从理论物理到工业应用的各个角落。在数值分析的语境下，我们的任务是：给定一个矩阵 A，如何高效、稳定地计算它的特征值和特征向量？

这个问题比解线性方程组难得多。对于一个 n × n 矩阵，特征多项式是 n 次多项式——而根据 Abel-Ruffini 定理，五次及以上多项式没有通用的根式解。所以当 n ≥ 5 时，任何特征值算法本质上都是迭代的。这也正是这一章和下一章要探讨的核心内容。

一个重要区别：解线性方程组 Ax = b 和计算特征值 Av = λv 是两种不同难度的问题。前者本质上是 O(n³) 的直接法（Gauss 消去），而后者需要迭代法，且收敛速度取决于矩阵本身的谱性质。在工程实践中，求解 1000×1000 的线性方程组是"秒级"的事，但计算全部特征值可能需要几秒甚至更久。

23.2 Gerschgorin 圆盘定理

在实际计算特征值之前，我们先掌握一个强大的定性分析工具：Gerschgorin 圆盘定理（Gerschgorin Circle Theorem）。这个定理告诉我们：不需要求解特征多项式，仅通过矩阵的元素就能"画圈"确定特征值的大致位置。

定理表述：对于 n × n 矩阵 A，定义第 i 个 Gerschgorin 圆盘为：

D(aii, Ri) = {z ∈ ℂ : |z - aii| ≤ Ri}

其中 Ri = Σj ≠ i |aij|（即第 i 行非对角元素的绝对值之和）。那么 A 的所有特征值都位于这些圆盘的并集中：

λ(A) ⊆ ∪i=1n D(aii, Ri)

不仅如此，对于 AT 的圆盘（即按列求和），A 的特征值也同样位于它们的并集中。因此，特征值实际上位于行圆盘和列圆盘的交集内——这个信息往往更精确。

来看一个具体例子：

import numpy as np

A = np.array([
    [10,  1,  0],
    [ 1,  8,  1],
    [ 0,  1,  5]
], dtype=float)

# 行圆盘
n = A.shape[0]
for i in range(n):
    R_i = np.sum(np.abs(A[i, :])) - np.abs(A[i, i])
    print(f"D({A[i,i]:.0f}, {R_i:.0f})  →  λ ∈ [{A[i,i]-R_i:.0f}, {A[i,i]+R_i:.0f}]")

# 列圆盘
for j in range(n):
    C_j = np.sum(np.abs(A[:, j])) - np.abs(A[j, j])
    print(f"列D({A[j,j]:.0f}, {C_j:.0f})  →  λ ∈ [{A[j,j]-C_j:.0f}, {A[j,j]+C_j:.0f}]")

# 实际计算特征值用于验证
eigvals = np.linalg.eigvals(A)
print(f"\n实际特征值: {np.sort(eigvals)}")
# 输出: [ 4.899  7.653  10.448]

从结果可以看到，三个行圆盘分别是 [9, 11]、[6, 10]、[4, 6]。实际特征值 4.899、7.653、10.448 确实落在对应的圆盘内。更妙的是，三个圆盘是"不相交的"（各自独立），这意味着每个圆盘恰好包含一个特征值——这种结论在圆盘连通时就不一定成立了。

Gerschgorin 定理的实用价值：在工程中，当你拿到一个大规模矩阵时，Gerschgorin 圆盘可以帮你快速回答"特征值大概在什么范围"、"矩阵是否正定"（所有圆盘都在右半平面）或"谱半径大约是多少"等问题——这些判断完全不需要求解特征多项式，代价几乎为零。

Gerschgorin 定理还有一个重要的推论：如果 k 个圆盘连成一片，且与其他圆盘不连通，那么这片区域中恰好包含 k 个特征值。这个结论在理论上很有用，但在实践中要小心——圆盘是否"连通"依赖于圆盘的半径精度，而不一定是特征值实际分布的真实反映。

一个容易踩的坑：Gerschgorin 圆盘给出的是特征值所在范围，而不是精确位置。初学者常犯的错误是直接用圆盘中心（即对角元）作为特征值的近似。实际上，对角元 aii 和特征值之间可能相差很远——特别是当非对角元素的值很大时，圆盘半径很大，圆心几乎不提供任何有用信息。所以记住：圆盘定理是一个定性工具，不是定量工具。

23.3 幂法（Power Method）

现在我们来讨论第一个真正计算特征值的算法：幂法（Power Method）。它的目标很明确：找出矩阵的模最大的特征值（dominant eigenvalue）及其对应的特征向量。

算法的思想非常朴素：反复用矩阵 A 去乘一个向量，然后归一化。

xk+1 = A xk / ||A xk||

为什么这样会收敛到主特征向量？直观理解：将初始向量 x₀ 按特征向量基展开，每次乘以 A 相当于把每个分量乘上对应的特征值。经过足够多次迭代，模最大特征值对应的分量"增长最快"，最终主导整个向量。收敛速度取决于比值 |λ₂/λ₁|——这个比值越小，收敛越快。

收敛速度的关键：幂法的收敛速度是线性的，比例因子为 |λ₂/λ₁|。如果这个比值接近 1（比如 0.99），幂法会收敛得非常慢——可能需要成千上万次迭代。反之，如果 |λ₂/λ₁| ≤ 0.5，十几二十步就能得到满意的精度。

当我们得到特征向量的近似后，可以用Rayleigh 商（Rayleigh Quotient）来估计特征值：

λ ≈ (xT A x) / (xT x)

Rayleigh 商有一个非常好的性质：如果 x 的误差是 O(ε)，那么 Rayleigh 商给出的特征值误差是 O(ε²)——它自动"平方"了精度。

下面是带收敛监控的 Python 实现：

import numpy as np

def power_method(A, max_iter=1000, tol=1e-10):
    """幂法求主特征值和特征向量"""
    n = A.shape[0]
    x = np.random.randn(n)
    x = x / np.linalg.norm(x)

    for k in range(max_iter):
        x_new = A @ x
        eigenvalue = x @ x_new  # Rayleigh 商
        x_new = x_new / np.linalg.norm(x_new)

        # 收敛监控：相邻迭代特征向量差
        diff = np.linalg.norm(x_new - x)
        if diff < tol:
            print(f"收敛于第 {k+1} 次迭代")
            break
        x = x_new

    return eigenvalue, x

# 测试
np.random.seed(42)
A = np.diag([10, 3, 1]) + 0.1 * np.random.randn(3, 3)
A = (A + A.T) / 2  # 对称化

lam, v = power_method(A)
print(f"计算的主特征值: {lam:.6f}")
print(f"numpy 参考值: {np.linalg.eigvalsh(A)[-1]:.6f}")
print(f"|λ₂/λ₁| ≈ {np.linalg.eigvalsh(A)[-2] / lam:.4f}")
# 若收敛快，说明比值小；若慢，说明比值接近 1

幂法的局限性：幂法只能求模最大的那个特征值。如果最大的特征值和第二大的模相等（比如复数共轭对），幂法不会收敛。如果初始向量与主特征向量正交，幂法也永远不会"看到"它——虽然在浮点运算中舍入误差通常会把你从正交方向上"踢开"，但这是一个需要意识的隐患。

23.4 反幂法与原点平移

幂法只能求模最大的特征值。那如果我们想求模最小的特征值呢？答案就是——反幂法（Inverse Iteration）。

反幂法的思路非常巧妙：注意到如果 A 的特征值是 λ，那么 A-1 的特征值是 1/λ。因此，对 A-1 应用幂法，就会收敛到模最小的特征值（对应 A-1 的主特征值）。迭代格式为：

xk+1 = A-1 xk / ||A-1 xk||

在实际计算中，我们不需要显式求逆——每次迭代解一个线性方程组 A y = xk 即可，而且可以先做 LU 分解，每次迭代只做回代，效率很高。

更进一步，加上原点平移（Shift），我们可以收敛到任意指定的特征值附近：

对 (A - σI)-1 应用幂法 → 收敛到最接近 σ 的特征值

因为 (A - σI) 的特征值是 λ - σ，其倒数 1/(λ - σ) 模最大的那个，对应的就是最接近 σ 的 λ。

一个真实的桥梁故事：我曾经参与过一个桥梁结构的模态分析项目。桥梁的屈曲载荷（buckling load）是设计的核心指标——它对应着结构刚度矩阵的最小正特征值。初始设计用有限元算出来是 120 吨，安全系数 1.5，看起来不错。但团队的资深工程师坚持要用反幂法做一次"精确验算"，并且特意把平移量 σ 设在 80 吨附近来加速收敛。结果一算——最小特征值其实是 73 吨。原始设计的简化模型在用幂法时，由于最小特征值和次小特征值之间的间隔很小（|λ₃/λ₂| 接近 1），幂法收敛极慢，在有限迭代次数下给出的结果严重偏大。如果在那种情况下直接投产——好吧，想想都后怕。这个教训让我明白了一个道理：在工程中，你不必精确计算出每一个特征值，但你必须确保你的算法收敛到了正确的那个。

最后，还有一个非常漂亮的技术值得一提：Rayleigh 商迭代（Rayleigh Quotient Iteration, RQI）。它的想法很简单——每步都用当前的 Rayleigh 商作为平移量 σk = (xkT A xk) / (xkT xk)，然后求解 (A - σkI) y = xk。

RQI 的收敛速度是三次的（cubic convergence）——这意味着每步迭代的误差从 ε 降到 ε³！在正常情况下，3-4 步就能达到双精度浮点数的极限精度。代价就是每次迭代都要解一个不同的线性方程组（因为平移量变了），计算量大，但对于需要高精度的情况，RQI 是无可替代的选择。

算法选择指南：如果你的目标只是谱半径（比如 PageRank 的主特征值），幂法就够了；如果你需要特定区域的特征值（比如振动分析的几个最低模态），用带平移的反幂法；如果你需要全部特征值且有高精度要求，那就需要下一章要讲的 QR 方法了。选择正确的工具，效率和精度的差异可能是数量级的。

到这里，矩阵特征值计算的第一部分就结束了。这一章我们从特征值的定义和重要性出发，学习了 Gerschgorin 圆盘定理这个定性分析工具，然后掌握了幂法和反幂法这两类基础迭代算法。下一章，我们将进入特征值计算的"重型武器"——QR 方法，它是现代数值线性代数中求解全部特征值的事实标准。我们到时候见。
