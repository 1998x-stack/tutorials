开篇：有了模型，如何求解？

大家好，欢迎继续因子分析的学习。上一章我们建立了 FA 的理论模型——X = μ + LF + ε，理解了共同度、特殊方差这些概念。但一个核心问题还没有解决：给定观测数据 X，我们怎么估计出因子载荷矩阵 L 和特殊方差 Ψ？

更关键的是，即使估计出来了，初始解往往难以解释——每个变量在多个因子上都有中等载荷，"轮廓模糊"。这时候就需要因子旋转来"纯化"载荷模式，让结果一目了然。

本章就围绕这两个主题展开：因子估计（有哪些方法、各有什么优劣）和因子旋转（正交旋转、斜交旋转、什么时候用哪种）。学完本章，你就可以独立完成一个完整的因子分析——从数据出发，估计因子，旋转，直到得到可解释的结果。

主成分因子提取法

最简单的因子估计方法——你可能已经猜到了——基于 PCA。这个方法称为主成分因子提取法 (Principal Component Factor Analysis, PCF)。

它的思路非常直接：先对标准化数据做完整的 PCA，得到 p 个主成分，然后截取前 m 个作为初始因子解。载荷矩阵 L̂ 的估计就是前 m 个特征向量乘以对应特征值的平方根：

主成分因子提取法：

1. 计算相关矩阵 R（如果已标准化，R = 协方差矩阵）
2. 对 R 做特征值分解：R = Q Λ Qᵀ
3. 取前 m 个特征值 λ₁ ≥ λ₂ ≥ ... ≥ λₘ 和对应的特征向量 e₁, ..., eₘ
4. 因子载荷矩阵的估计：
   L̂ = [√λ₁ e₁, √λ₂ e₂, ..., √λₘ eₘ]
   即第 j 列的值为 √λⱼ × eⱼ
5. 特殊方差的估计：
   ψ̂ᵢ = 1 - Σⱼ₌₁ᵐ l̂ᵢⱼ² = 1 - ĥᵢ²
   （即未被前 m 个因子解释的方差）

这个方法简单、快速、数值稳定，而且不需要迭代。这也是为什么许多统计软件（如 SPSS）默认使用这种方法——它几乎永远能给出一个收敛的解。

但 PCF 有一个重要的理论缺陷：它假设特殊方差 Ψ = 0，即所有方差都是公共的——这与 FA 的基本假设相矛盾。实际上，PCF 更像是"PCA 在 FA 框架下的借用"，严格来说不是真正的因子分析。不过在实践中，当 p 不太大且共同度普遍较高时，PCF 的结果与更复杂的方法差异不大，是一个很好的快速探索工具。

何时用 PCF：当你需要快速了解数据中可能存在多少个因子，或者当更复杂的迭代方法不收敛时。但要记住：PCF 给出的载荷估计是有偏的（倾向于高估载荷），特别是当样本量小或共同度低时。正式的学术研究建议使用极大似然法。

极大似然因子估计

极大似然因子分析 (Maximum Likelihood Factor Analysis, MLFA) 是更统计严谨的方法。它假设数据服从多元正态分布，然后寻找使观测数据的似然函数最大化的参数值（L 和 Ψ）。

MLFA 的核心是迭代优化。给定因子数 m，算法通过以下步骤交替更新参数：

极大似然因子估计的迭代过程：

初始化：用 PCF 给出 L 和 Ψ 的初始值

重复直到收敛：
  步骤 A：固定 Ψ，更新 L
  对 Ψ^(-1/2) R Ψ^(-1/2) 做特征值分解
  取前 m 个特征值和特征向量，更新 L̂

  步骤 B：固定 L，更新 Ψ
  ψ̂ᵢ = 1 - Σⱼ₌₁ᵐ l̂ᵢⱼ²
  （确保 Ψ 是正定矩阵）

收敛后输出：L̂, Ψ̂

拟合优度检验（当 n 大且假设正态时有效）：
  零假设：m 个因子足够解释所有相关性
  检验统计量近似服从 χ² 分布
  如果 p 值 > 0.05，说明 m 个因子已足够

MLFA 的优点很明显：它提供了统计推断的基础——你可以检验"m 个因子是否足够"这个假设，可以计算标准误差，可以构建置信区间。这使得 MLFA 成为学术研究的标准方法。

但 MLFA 也有缺点：它需要迭代，可能不收敛（特别是当样本量小、共同度低、或初始值选择不当时）；它对正态性假设敏感——如果数据严重偏离正态，ML 估计可能有偏；它比 PCF 慢很多，特别是在 p 较大的时候。

特征主成分法 (PCF)极大似然法 (MLFA)
计算速度快速（非迭代）较慢（迭代）
正态假设不需要需要（近似）
统计检验无有（似然比检验）
收敛问题永不失败可能不收敛
小样本表现有偏但稳定可能不稳定
学术认可较低（探索性）较高（正式研究）

其他因子提取方法

除了 PCF 和 MLFA，还有一些其他方法针对特定场景设计：

主轴因子法 (Principal Axis Factoring, PAF)——这是 PCF 的改进版。不同之处在于，PCF 对相关矩阵 R 做特征值分解，而 PAF 对"约化相关矩阵"(reduced correlation matrix) R* 做特征值分解：R* = R - diag(Ψ̂)，即把对角线上的 1 替换为共同度的初始估计。然后迭代更新共同度，直到收敛。PAF 在心理测量学中非常流行，它对正态性的依赖比 MLFA 小。

Alpha 因子分析 (Alpha Factoring)——这种方法从"信度"的角度出发，最大化因子的 Cronbach's α 系数。它关注的是因子的"可推广性"(generalizability)，即抽取的因子在多大程度上能推广到变量总体中。这种方法的哲学是：你的 p 个变量是来自一个更大的变量总体中的样本——你希望找到能代表这个总体而非仅当前样本的因子。

映像因子分析 (Image Factoring)——基于 Guttman 的映像理论。它将每个变量分解为"映像"(image，可被其他变量线性预测的部分)和"反映像"(anti-image，不能被预测的部分)，然后对映像部分的协方差矩阵做因子分析。这种方法不依赖正态假设。

方法选择建议：对大部分实际项目来说，MLFA 是第一选择——因为它有统计检验的支持。如果 MLFA 不收敛，退回到 PAF。如果样本量极小（n < 100）或变量数很少（p < 10），用 PCF 做快速探索。Alpha 和映像因子分析比较小众，除非你明确知道它们的适用场景，否则不必优先考虑。

因子旋转的必要性

现在假设我们估计出了载荷矩阵 L̂。但问题是：这个解是不唯一的。

回忆因子模型 X = μ + LF + ε。如果我们在 L 右边乘一个正交矩阵 T（即 L* = LT），同时在 F 左边乘 Tᵀ（即 F* = TᵀF），那么 X = μ + L* F* + ε——模型完全不变！因为 (LT)(TᵀF) = LFF = LF。这意味着载荷矩阵有无穷多种等价的表达方式——它们产生完全相同的协方差矩阵 Σ = LLᵀ + Ψ。

这个"旋转自由度"既是麻烦也是机会。麻烦在于初始解不一定容易解释，机会在于我们可以选择一个最容易解释的旋转角度。这就是因子旋转 (factor rotation) 的动机。

旋转的目标是实现简单结构 (simple structure)——Thurstone 在 1947 年提出的理想：

  • 每个变量在尽可能少的因子上有高载荷（最好只在一个因子上高）
  • 每个因子在尽可能少的变量上有高载荷
  • 载荷模式中高值和低值两极分化，避免居中暧昧

简单结构使得因子解释变得清晰：这个变量归这个因子，那个变量归那个因子，一目了然。

正交旋转

正交旋转 (Orthogonal Rotation) 保持因子之间的不相关性——旋转后的因子仍然正交（不相关）。最常用的正交旋转是 Varimax（方差最大旋转）。

Varimax 旋转

Varimax 的目标是最大化每个因子上载荷平方的方差。直观地说，它试图让每个因子只有少数几个大载荷和许多接近零的小载荷——这就是"两极分化"。Kaiser 在 1958 年提出的标准化 Varimax 准则至今仍是使用最广泛的旋转方法。

# Python：Varimax 旋转（通过 factor_analyzer 包）
from factor_analyzer import Rotator

# 假设 L 是初始载荷矩阵 (p × m)
rotator = Rotator(method='varimax')
L_rotated = rotator.fit_transform(L)

# L_rotated 就是旋转后的载荷矩阵
# 此时每个变量通常只在少数因子上有高载荷

Quartimax 旋转

Quartimax 最小化每个变量在所有因子上载荷四次方的负值。它倾向于产生一个"一般因子"(general factor，几乎所有变量都在它上面有高载荷)加上几个小的特定因子。Quartimax 更适合存在一个强大的共同因子的场景，但在多数情况下不如 Varimax 受欢迎。

Equimax 旋转

Equimax 试图在 Varimax（简化因子）和 Quartimax（简化变量）之间取得平衡。它有一个参数可以调节两种目标的权重。在实践中，Equimax 的使用不如 Varimax 普遍。

正交旋转方法优化目标效果推荐场景
Varimax最大化载荷平方的列方差因子结构清晰，最常用通用场景，默认选择
Quartimax最小化载荷四次方的行方差产生一般因子存在强共同因子
EquimaxVarimax 与 Quartimax 的折中居中效果无明显偏好时试探

斜交旋转

正交旋转的约束"因子必须不相关"在实际中往往过于严格。心理学中的智力因子——"语言能力"和"数学能力"显然是相关的——有数学天赋的人通常语言也不差。如果强行迫使它们不相关，你可能会扭曲因子结构。

斜交旋转 (Oblique Rotation) 允许因子之间相关，这更符合现实，但也使模型稍微复杂了一些（因为现在有了因子相关矩阵需要解读）。

Promax 旋转

Promax 是应用最广泛的斜交旋转方法。它先做 Varimax 正交旋转，然后通过让载荷向更极端的方向"松弛"来允许因子相关。具体来说，它构造一个"目标矩阵"，其中每个载荷的 k 次幂（通常 k = 2, 3 或 4）保持符号不变——这相当于说"大载荷要更大，小载荷要更小"，然后找到一个斜交变换最接近这个目标。

Oblimin 直接旋转

Direct Oblimin 是另一种斜交旋转方法。它有一个参数 δ (delta) 控制因子之间允许的相关程度：δ ≤ 0 时因子之间较不相关（接近正交），δ 越大因子之间可以更相关。默认 δ = 0 通常是一个合理的起点。

# Python：Promax 旋转
from factor_analyzer import Rotator

# Promax 斜交旋转（k=4 是常见选择）
rotator = Rotator(method='promax', k=4)
L_rotated = rotator.fit_transform(L)

# 旋转后的结果包括：
# L_rotated: 旋转后的因子载荷矩阵
# phi_matrix: 因子相关矩阵（斜交旋转特有）
# structure_matrix: 因子结构矩阵（载荷 + 因子间相关）

在斜交旋转中，我们区分两种矩阵：因子模式矩阵 (pattern matrix)——即通常说的载荷矩阵 L，每个元素是因子对变量的唯一贡献（控制了其他因子后）；因子结构矩阵 (structure matrix)——变量与因子之间的简单相关系数。当因子不相关时，两个矩阵相同；当因子相关时，结构矩阵 = 模式矩阵 × 因子相关矩阵。

正交 vs 斜交的选择指南：如果你有理论依据认为因子之间应该不相关（比如在一个精心设计的量表中，各维度在概念上应该是正交的），用 Varimax。如果你不确定因子是否相关，或者你有理由相信它们相关（这在社会科学中几乎总是如此），先用 Promax 试试——如果因子之间的相关性很小（r < 0.3），你可以转而用 Varimax；如果相关性显著（r > 0.3），斜交旋转才真正必要。

踩坑故事：旋转太多因子，追逐噪声

几年前我审阅一篇管理学的论文，作者对一份 30 个题目的问卷做了因子分析。问卷设计时假设 5 个维度，但作者跑完平行分析和 scree 图后，决定提取 8 个因子——因为他发现第 6 到第 8 个因子的特征值刚好大于 1。他接着做了 Varimax 旋转，然后欣喜地发现这 8 个因子"都有清晰的含义"。

但我仔细一看，所谓"第 7 个因子"只有两个题目在它上面有高载荷，而且这两个题目的内容风马牛不相及——一个是"我享受团队合作"，另一个是"我愿意加班完成紧急任务"。这纯粹是过度提取——他提取的因子太多，以至于一些本质上属于"随机噪声"的成分被当成了因子。

教训：因子分析不是提取越多因子越好。过度提取 (over-extraction) 会导致你"发现"一些根本不存在、无法解释的"幻影因子"。一个判断标准：每个因子至少应该有 3 个高载荷变量（|载荷| > 0.4），少于这个数的因子很可能是噪声。另外，不要只依赖 Kaiser 规则（特征值 > 1）——在边界情况下使用平行分析会更可靠。记住：少即是多——4 个可解释的因子远好于 8 个无法解释的因子。

本章小结与衔接

本章我们学习了因子分析的两个核心操作：因子估计和因子旋转。估计方法方面，我们比较了 PCF（简单快速但有偏）和 MLFA（统计严谨但需迭代）以及 PAF 等其他方法，给出了不同场景下的选择建议。旋转方面，我们理解了旋转的必要性（载荷不唯一性），掌握了 Varimax 正交旋转和 Promax 斜交旋转，以及它们的适用场景。最后，一个关于"过度提取"的踩坑故事提醒我们：因子分析不是机械的操作，需要结合理论判断和经验积累。

因子载荷矩阵有了，因子也旋转得清晰可读了——但还有一个关键步骤没完成：我们想知道每个样本在每个因子上的得分是多少。下一章将专门讨论因子得分的估计方法（回归法和 Bartlett 法），以及因子的命名——如何给抽象的统计因子起一个有意义的名字。同时我们还会用一个完整的心理学量表案例，把因子分析的整个流程走一遍。
