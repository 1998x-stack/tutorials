8.1 最古老的ML算法，依然是工业界的中坚

线性回归诞生于200多年前（1805年，Legendre发表最小二乘法），但它至今仍是工业界使用最频繁的模型之一。为什么？三个字：可解释、快、稳。在很多场景（如信贷评分卡、房价评估），你需要的不是一个黑盒深度学习模型，而是一个系数有明确业务含义的线性模型。这一章我们彻底搞懂线性回归以及它的三个"强化版"——Ridge、Lasso、ElasticNet。

8.2 最小二乘法：线性回归的心脏

线性回归的目标是找到一组权重w，使得预测值Xw与真实值y之间的误差平方和最小：

min ||Xw − y||²

这个优化问题有闭式解（可以直接算出来，不需要梯度下降迭代）：

w = (X^T X)^(-1) X^T y

这就是著名的"正规方程"（Normal Equation）。当特征数量n > 样本数m时，X^T X不可逆，正规方程失效——这是线性回归的命门。

💡 面试经典问题："什么时候用正规方程，什么时候用梯度下降？" 正规方程适合特征数 < 10,000（矩阵求逆是O(n³)），不需要选学习率，但大数据下内存吃不消。梯度下降适合特征多的大数据集，需要调学习率，但是更灵活。

8.3 多项式回归：一条直线不够用的时候

如果数据有明显的曲线趋势怎么办？多项式回归本质上还是线性回归——不过先把特征做了非线性变换：

from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import Pipeline

# 二次多项式回归：把 [x] 变成 [1, x, x²]
poly_reg = Pipeline([
    ('poly', PolynomialFeatures(degree=2, include_bias=False)),
    ('linear', LinearRegression())
])
poly_reg.fit(X, y)

但注意：degree越高，模型越容易过拟合。一般情况下degree=2或3就够了。degree=100的多项式会疯狂地上下摆动去"穿过"每一个训练点，在训练集上误差为零，但在测试集上惨不忍睹。

8.4 正则化：防止过拟合的三兄弟

当特征很多、样本很少时，线性回归会变得不稳定——某个特征的系数可能是+1000，另一个是-998，两者几乎抵消，但对噪声极度敏感。正则化在损失函数里加入对权重的惩罚，让模型"老实一点"。

方法惩罚项效果适用场景
Ridge (L2)λ·Σw²所有系数收缩接近0但不为0特征都或多或少有用
Lasso (L1)λ·Σ|w|部分系数精确变为0（特征选择）只有少数特征真的重要
ElasticNetλ₁·Σ|w| + λ₂·Σw²L1+L2的组合，取两者之长特征多且相关度高时

8.5 L1 vs L2 的几何直觉

为什么Lasso能让系数精确为0而Ridge不能？看几何：

  • L2约束区域是圆形：等高线切到圆的边上，交点在坐标轴上概率极低 → 系数通常不为0
  • L1约束区域是菱形（有尖角）：等高线很容易切到菱形的角上，角在坐标轴上 → 系数精确为0

这个几何差异导致了一个重要的工程选择：如果你有1000个特征但怀疑只有20个真的有用，用Lasso——它能自动把没用的980个特征系数压成0，省去你做特征选择的麻烦。

8.6 正则化路径与λ选择

λ（正则化强度）是最关键的参数。λ=0就是普通线性回归，λ→∞所有系数→0。

import numpy as np
from sklearn.linear_model import Ridge, Lasso, ElasticNet, RidgeCV

# RidgeCV自动帮你选最好的λ
alphas = np.logspace(-3, 3, 50)
ridge_cv = RidgeCV(alphas=alphas, store_cv_values=True)
ridge_cv.fit(X_train, y_train)
print(f"最佳alpha: {ridge_cv.alpha_:.4f}")

# 比较三种正则化
models = {
    'Linear': LinearRegression(),
    'Ridge': Ridge(alpha=1.0),
    'Lasso': Lasso(alpha=0.01),
    'ElasticNet': ElasticNet(alpha=0.01, l1_ratio=0.5)
}

for name, model in models.items():
    model.fit(X_train, y_train)
    score = model.score(X_test, y_test)
    coef_nonzero = (model.coef_ if name != 'Linear' else model.named_steps
                    if hasattr(model, 'named_steps') else None)
    print(f"{name}: R²={score:.4f}")
⚠️ 使用Lasso前的关键步骤：Lasso对特征尺度非常敏感——不同特征的数量级差异会导致惩罚不均衡。使用Lasso前必须先用StandardScaler标准化。Ridge也有同样要求。很多人在标准化前就直接fit，结果发现Lasso效果还不如Ridge，以为是模型的问题，其实是预处理的问题。
🔑 一句话总结：大多数情况下用Ridge（稳定、效果好）；如果你怀疑很多特征是冗余的，用Lasso做自动特征选择；特征之间有强相关性时用ElasticNet（Lasso在高度相关特征中倾向于随机选一个，ElasticNet能同时保留相关特征）。
