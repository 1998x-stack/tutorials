单调栈思想

大家好。不知道你们在刷题时有没有遇到过这样一种感觉：明明题目就是在一维数组上找下一个更大的元素，也能想到用栈，但代码就是写不对——要么下标搞反了，要么循环条件弄混了。如果是这样，别担心，今天这一章就是为你准备的。我们将系统地讲解单调栈（Monotonic Stack）和单调队列（Monotonic Queue），它们是处理"下一个更大/更小"类问题和滑动窗口最值问题的利器。

先说单调栈的核心思想：维护一个栈，使栈内元素保持单调递增或单调递减。遍历数组时，每当新元素破坏了单调性，就不断弹出栈顶元素——而弹出的那一刻，往往就能确定答案。这个"弹出时定答案"的机制，是单调栈最精妙的设计。

我们用一张表直观对比两种单调栈的场景：

类型 栈内顺序 解决的问题 典型题
单调递增栈 栈底→栈顶 递增 下一个更小元素 每日温度（反向思维）
单调递减栈 栈底→栈顶 递减 下一个更大元素 496、739、42

面试中约 80% 的单调栈题目是"下一个更大元素"类的，所以单调递减栈是最常用的。接下来我们从最基础的模板题开始。

单调栈模板

下一个更大元素 I（LeetCode 496）

题目说：给定两个没有重复元素的数组 nums1 和 nums2，其中 nums1 是 nums2 的子集。找出 nums1 中每个元素在 nums2 中的下一个更大元素。所谓"下一个更大元素"是指该元素右边第一个比它大的数，不存在则返回 -1。

这是单调栈最经典的入门题。思路是：先遍历 nums2，用单调递减栈计算出每个元素的下一个更大元素，存入 HashMap。然后遍历 nums1 查表即可。

def next_greater_element(nums1, nums2):
            stack = []            # 单调递减栈（存元素值）
            next_greater = {}     # 元素 -> 下一个更大元素

            for num in nums2:
                # 如果当前元素比栈顶大，说明找到了栈顶的下一个更大元素
                while stack and num > stack[-1]:
                    smaller = stack.pop()
                    next_greater[smaller] = num
                stack.append(num)

            # 栈中剩余元素没有下一个更大元素
            for num in stack:
                next_greater[num] = -1

            return [next_greater[num] for num in nums1]

这段代码虽然短，但承载了单调栈的标准范式：while 栈不空 and 当前元素 > 栈顶: 弹出栈顶并记录答案; 然后将当前元素入栈。记住这个模板，你可以解决一大半单调栈题目。

小技巧：单调栈通常配合 HashMap 使用。栈用来处理数组，HashMap 用来记录结果——"栈负责算，Map 负责记"。

每日温度（LeetCode 739）

这道题把场景具象化了：给定一个气温列表，对每一天，要等多少天才能等到更高的气温。本质上就是"下一个更大元素"——但返回的不是值，而是下标差。

核心区别：栈里存的不是元素值，而是下标。因为我们需要用下标计算天数差。

def daily_temperatures(temperatures):
            n = len(temperatures)
            ans = [0] * n
            stack = []  # 存下标，从栈底到栈顶递减

            for i in range(n):
                # 当前温度比栈顶对应温度高
                while stack and temperatures[i] > temperatures[stack[-1]]:
                    prev_index = stack.pop()
                    ans[prev_index] = i - prev_index
                stack.append(i)

            return ans

这个写法几乎是 496 的翻版，唯一的区别是栈里存下标、答案记录天数差。理解了这一点，你就掌握了"下一个更大元素"系列的精髓。

通用框架：当我们遍历数组时，维护一个栈（存下标或值），确保栈内元素是单调的。当新元素破坏了单调性时，不断弹出栈顶并在弹出时记录答案。最后将当前元素入栈。

循环数组与接雨水

下一个更大元素 II（LeetCode 503）

本题是 496 的升级版——数组是循环的，意味着最后一个元素的下一个更大元素可能是第一个元素。如何处理循环？最常用的技巧是"数组拼接"：遍历两倍长度的数组，用 i % n 取模访问原始元素。

def next_greater_elements(nums):
            n = len(nums)
            ans = [-1] * n
            stack = []  # 存下标

            for i in range(2 * n):
                idx = i % n
                while stack and nums[idx] > nums[stack[-1]]:
                    prev = stack.pop()
                    ans[prev] = nums[idx]
                stack.append(idx)

            return ans

注意这里我们只遍历两轮，但实际上每个元素最多入栈一次、出栈一次，时间复杂度仍然是 O(n)。这个"取模遍历"的技巧非常实用，遇到环形数组问题应该第一时间想到它。

接雨水（LeetCode 42）

如果说"下一个更大元素"是单调栈的入门题，那么接雨水就是单调栈的毕业题——它是 LeetCode 上点赞数最高的题目之一，也是面试中的超级高频题。

题目很直观：给定 n 个非负整数表示每个宽度为 1 的柱子的高度，计算下雨后能接多少雨水。我们从单调栈的角度来理解：当栈顶元素被弹出时，它左右两边第一个比它高的柱子就是"碗沿"，两者之间的最低高度决定了能装多少水。

def trap(height):
            stack = []  # 单调递减栈，存下标
            water = 0

            for i in range(len(height)):
                while stack and height[i] > height[stack[-1]]:
                    top = stack.pop()  # 凹槽底部
                    if not stack:
                        break
                    left = stack[-1]   # 左边界
                    distance = i - left - 1
                    bounded_height = min(height[left], height[i]) - height[top]
                    water += distance * bounded_height
                stack.append(i)

            return water

这个解法的时间复杂度 O(n)，空间复杂度 O(n)。每次弹出栈顶时，它左右两边的高墙分别是当前元素 i 和新的栈顶 stack[-1]。水的容量就是 宽度 × 高度差。

踩坑提醒：接雨水的单调栈解法中，if not stack: break 这一行极易被遗漏。想象一下，当栈中只有一个元素且当前柱子更高时，弹出后栈为空，没有左边界——这时候就必须 break，否则会报 IndexError。我第一次写时就漏了这行，debug 了整整一小时。

单调队列

说完单调栈，我们来说说它的"近亲"——单调队列。单调队列的核心应用场景只有一个，但极其经典：固定窗口内的最值问题。

滑动窗口最大值（LeetCode 239）

给定一个数组和滑动窗口大小 k，找出所有滑动窗口中的最大值。这道题的难度是 Hard，但用单调队列可以做到 O(n) 时间。

思路是我们维护一个双端队列（deque），队首到队尾保持单调递减。队列里存下标。每来一个新元素：

  • 从队尾弹出所有小于等于当前元素的下标（保持单调性）
  • 将当前下标加入队尾
  • 如果队首下标已经滑出窗口范围，从队首弹出
  • 窗口形成后，队首元素就是当前窗口的最大值
from collections import deque

        def max_sliding_window(nums, k):
            dq = deque()  # 存下标，单调递减
            result = []

            for i in range(len(nums)):
                # 保持单调性：弹出队尾所有比 nums[i] 小的
                while dq and nums[dq[-1]] <= nums[i]:
                    dq.pop()
                dq.append(i)

                # 移除过期窗口的队首
                if dq[0] == i - k:
                    dq.popleft()

                # 窗口形成后记录结果
                if i >= k - 1:
                    result.append(nums[dq[0]])

            return result

这段代码的精妙之处在于：每个元素最多入队一次、出队一次，所以总复杂度是 O(n)。deque 的队首始终是当前窗口的最大值，队尾始终保持单调递减——这就是"单调队列"名字的由来。

单调栈 vs 单调队列：单调栈解决的是"下一个更大/更小"问题，用普通栈（list）实现；单调队列解决的是"滑动窗口最值"问题，用双端队列（deque）实现。前者是"全局"的，后者是"窗口"的——注意区分应用场景。

踩坑与总结

单调栈和单调队列的核心代码都很短，但细节极多。我总结几个高频踩坑点。

等于号的处理

单调栈可以是严格单调（不含相等元素）或非严格单调（允许相等）。在"下一个更大元素"中，我们通常用严格递减栈——即 while stack and nums[i] > nums[stack[-1]]，注意这里用的是 > 而不是 >=。因为如果相等，我们不认为它是"下一个更大"，所以保留栈顶。

但在某些题目中，比如"下一个大于或等于"的元素，就需要用 >=。务必根据题目语义判断，不要死记模板。

栈里存什么

如果能存值解决问题就存值（如 496），如果需要索引差就存下标（如 739、42）。我个人的建议是统一存下标——因为有了下标就一定能拿到值，反之则不行。存下标更通用。

接雨水双解法对比

接雨水除了单调栈，还有双指针法：左右各维护一个最大高度，每次移动较矮的那一侧。

解法 时间复杂度 空间复杂度 核心思想
单调栈 O(n) O(n) 按"行"计算，每次处理一个凹槽
双指针 O(n) O(1) 按"列"计算，每次处理一个位置
动态规划（前缀最大） O(n) O(n) 预处理左右最大高度再逐列累加

三种解法都是 O(n) 时间，但双指针空间最优。我个人面试时偏好双指针——代码短、空间省、不易出错。但单调栈解法体现了"数据结构的思维"，面试官有时专门想考察你能不能想到用栈来处理这种嵌套结构。

学习建议：单调栈的题目模式非常固定——80% 是"下一个更大元素"变体。先把 496、739、503 这三道刷熟，再攻克 42 和 84（柱状图中最大的矩形）。刷完这五道题，单调栈就基本毕业了。至于单调队列，239 是唯一需要熟练掌握的题目。
