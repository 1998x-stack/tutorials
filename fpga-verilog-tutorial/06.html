第6章 时序逻辑设计实战

大家好，我是你们的讲师。如果说组合逻辑是 FPGA 的"肌肉"，那么时序逻辑就是 FPGA 的"心跳"。每一个时钟周期，寄存器都在推动系统从一个状态稳定地进入下一个状态。这一章我们聚焦于最常用的三类时序电路：计数器、分频器和移位寄存器，再加上两个工程中反复出现的场景——流水灯和按键消抖。

这些电路看似简单，但每一个都有"正确"和"错误"的写法。写对了一劳永逸，写错了——轻则仿真不通过，重则板级调试三天三夜。

6.1 计数器

计数器是时序逻辑中最基础的构件。它的核心是一个加/减法器 + 一组寄存器，每个时钟沿更新结果。

6.1.1 二进制计数器（使能与清零的正确写法）

module binary_counter #(
    parameter WIDTH = 8
) (
    input  wire             clk,
    input  wire             rst_n,
    input  wire             en,      // 计数使能
    input  wire             clear,   // 同步清零
    output reg  [WIDTH-1:0] count
);

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            count <= 0;
        else if (clear)
            count <= 0;
        else if (en)
            count <= count + 1;
        // else: count 保持不变
    end

endmodule

注意这里的结构：异步复位优先级最高，其次是同步清零，然后是使能。三个条件互斥，每个时钟沿只会触发一种操作。这种"优先级链"的写法是工业界标准。

6.1.2 BCD 计数器

BCD（Binary-Coded Decimal）计数器每计到 9 就回 0，常用于数字显示或人机接口：

module bcd_counter (
    input  wire       clk, rst_n, en,
    output reg  [3:0] count,
    output reg        carry    // 每 10 个脉冲输出一个进位
);

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            count <= 0;
            carry <= 0;
        end else if (en) begin
            if (count == 9) begin
                count <= 0;
                carry <= 1;
            end else begin
                count <= count + 1;
                carry <= 0;
            end
        end else begin
            carry <= 0;
        end
    end

endmodule

6.1.3 饱和计数器

饱和计数器"计到最大值后不再增加"——常用于状态超时保护或 FIFO 指针：

// 饱和计数器——到达最大值后不再变化
always @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        sat_count <= 0;
    else if (clear)
        sat_count <= 0;
    else if (en & (sat_count < MAX_VAL))
        sat_count <= sat_count + 1;
end

比较 sat_count < MAX_VAL 是关键——不加这个比较，计数器会回卷到 0（模运算特性）。

6.2 分频器

分频器本质上就是计数器——通过计数到特定值来产生周期性信号。工程中的分频有偶数分频、奇数分频和分数分频三类。

6.2.1 偶数分频（最简单）

对一个 N 分频（N 为偶数），计数器计到 N/2 - 1 时翻转输出：

// 偶数分频：clk_out = clk_in / DIV
module even_divider #(
    parameter DIV = 4  // 必须是偶数
) (
    input  wire       clk_in,
    input  wire       rst_n,
    output reg        clk_out
);
    localparam CNT_WIDTH = $clog2(DIV/2);
    reg [CNT_WIDTH-1:0] cnt;

    always @(posedge clk_in or negedge rst_n) begin
        if (!rst_n) begin
            cnt     <= 0;
            clk_out <= 0;
        end else begin
            if (cnt == (DIV/2 - 1)) begin
                cnt     <= 0;
                clk_out <= ~clk_out;
            end else begin
                cnt <= cnt + 1;
            end
        end
    end
endmodule

6.2.2 奇数分频（50% 占空比）

奇数分频实现 50% 占空比比偶数分频复杂——需要用上升沿和下降沿分别计数，然后组合输出：

// 奇数分频（50% 占空比）：clk_out = clk_in / DIV
module odd_divider #(
    parameter DIV = 5  // 必须是奇数
) (
    input  wire       clk_in,
    input  wire       rst_n,
    output wire       clk_out
);
    localparam HALF = (DIV - 1) / 2;
    localparam CNT_WIDTH = $clog2(DIV);

    reg [CNT_WIDTH-1:0] cnt_pos, cnt_neg;
    reg                 clk_pos, clk_neg;

    // 上升沿计数
    always @(posedge clk_in or negedge rst_n) begin
        if (!rst_n) begin
            cnt_pos <= 0;
            clk_pos <= 0;
        end else begin
            if (cnt_pos == (DIV - 1)) begin
                cnt_pos <= 0;
            end else begin
                cnt_pos <= cnt_pos + 1;
            end

            if (cnt_pos < HALF)
                clk_pos <= 1;
            else
                clk_pos <= 0;
        end
    end

    // 下降沿计数
    always @(negedge clk_in or negedge rst_n) begin
        if (!rst_n) begin
            cnt_neg <= 0;
            clk_neg <= 0;
        end else begin
            if (cnt_neg == (DIV - 1)) begin
                cnt_neg <= 0;
            end else begin
                cnt_neg <= cnt_neg + 1;
            end

            if (cnt_neg < HALF)
                clk_neg <= 1;
            else
                clk_neg <= 0;
        end
    end

    // 组合两个半周期信号得到 50% 占空比
    assign clk_out = clk_pos | clk_neg;
endmodule
关于分频设计理念的转变：在传统的 MCU 思维中，我们会用分频器产生多个不同频率的时钟信号。但在现代 FPGA 设计中，推荐的做法是只用 PLL 产生需要的时钟频率，然后用时钟使能实现慢速逻辑。原因有三：第一，分频后的时钟没有经过全局时钟网络，skew 和抖动不可控；第二，多时钟域增加了 STA 和 CDC 分析的复杂度；第三，PLL 的输出时钟之间可以有已知的相位关系，利于时序约束。所以，本章的分频器代码主要用于教学和低频实验——在工程应用中，请优先使用 PLL + 时钟使能方案。

6.2.3 分数分频概述

分数分频（如 1.5 分频、2.5 分频）通过"吞脉冲"技术实现——周期性地插入一个"短周期"。实现方案通常使用双模分频器（Dual-Modulus Divider）：大部分时间用 N 分频，每隔 M 个周期用 N+1 分频一次，从而得到平均分频比 N + 1/M。这是 PLL 中 N/N+1 双模分频器的基本原理。在纯 RTL 中实现分数分频需要状态机 + 累加器，有一定复杂度，工程中通常用 PLL 或 MMCM 原语直接生成。

6.3 移位寄存器

6.3.1 四种基本形态

// SIPO：串行输入，并行输出（SPI 接收端常用）
module sipo_shiftreg #(parameter WIDTH = 8) (
    input  wire             clk, rst_n,
    input  wire             serial_in,   // 串行输入
    output wire [WIDTH-1:0] parallel_out // 并行输出
);
    reg [WIDTH-1:0] shift_reg;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            shift_reg <= 0;
        else
            shift_reg <= {shift_reg[WIDTH-2:0], serial_in};
    end

    assign parallel_out = shift_reg;
endmodule

6.3.2 LFSR：线性反馈移位寄存器

LFSR 用于生成伪随机序列（PRBS），在通信加扰、测试向量生成、白噪声模拟中广泛使用。8-bit LFSR 的实现：

// 8-bit LFSR（多项式 x^8 + x^6 + x^5 + x^4 + 1）
module lfsr_8bit (
    input  wire       clk, rst_n,
    input  wire       en,
    output wire [7:0] random_data
);

    reg [7:0] lfsr;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            lfsr <= 8'b00000001;  // 种子不能为全 0
        else if (en) begin
            // 抽头：bit 8(1) XOR bit 6(s^6) XOR bit 5(s^5) XOR bit 4(s^4)
            lfsr <= {lfsr[6:0],
                      lfsr[7] ^ lfsr[5] ^ lfsr[4] ^ lfsr[3]};
        end
    end

    assign random_data = lfsr;

endmodule

注意事项：

  • 种子不能为全 0——如果 LFSR 全 0，异或结果永远是 0，LFSR 会"卡死"。
  • 8-bit LFSR 的伪随机序列长度为 2^8 - 1 = 255 个周期（全 0 状态被排除）。
  • 抽头位置由本原多项式决定。Vivado 的 LFSR IP 核支持自动选择多项式。

6.4 流水灯与按键消抖

6.4.1 流水灯（LED Chaser）

流水灯是 FPGA 入门的基础实验——本质上是一个移位寄存器 + 使能定时器：

// 流水灯：LED 从左到右循环点亮
module led_chaser #(
    parameter CLK_FREQ = 50_000_000,  // 时钟频率 50 MHz
    parameter SCAN_MS  = 100          // 每 100ms 移动一次
) (
    input  wire       clk, rst_n,
    output reg  [7:0] led
);

    localparam TIMER_TOP = CLK_FREQ / (1000 / SCAN_MS);
    reg [25:0] timer;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            timer <= 0;
            led   <= 8'b00000001;
        end else begin
            if (timer == TIMER_TOP - 1) begin
                timer <= 0;
                led <= {led[6:0], led[7]};  // 循环左移
            end else begin
                timer <= timer + 1;
            end
        end
    end

endmodule

6.4.2 按键消抖

机械按键在按下和释放瞬间会产生 5-20ms 的抖动（物理弹跳）。FPGA 中用计数器实现硬件消抖——连续采样到稳定电平超过 10ms 才认为按键有效：

// 按键消抖模块
module debounce #(
    parameter CLK_FREQ  = 50_000_000,
    parameter DEBOUNCE_MS = 10     // 消抖时间窗口
) (
    input  wire       clk, rst_n,
    input  wire       key_in,      // 按键输入（假设低有效）
    output reg        key_out,     // 消抖后输出
    output reg        key_falling  // 下降沿检测（按键按下瞬间）
);

    localparam MAX_CNT = CLK_FREQ / (1000 / DEBOUNCE_MS);
    reg [25:0] cnt;
    reg        key_sync0, key_sync1;  // 两级同步

    // 同步（跨时钟域处理——虽然是同一时钟，但按键是异步输入）
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            key_sync0 <= 1;
            key_sync1 <= 1;
        end else begin
            key_sync0 <= key_in;
            key_sync1 <= key_sync0;
        end
    end

    // 消抖计数器
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            cnt     <= 0;
            key_out <= 1;
        end else begin
            if (key_sync1 !== key_out) begin
                // 输入状态与输出状态不一致——正在抖动或有效变化
                if (cnt == MAX_CNT - 1) begin
                    cnt     <= 0;
                    key_out <= ~key_out;  // 稳定超过阈值，更新输出
                end else begin
                    cnt <= cnt + 1;
                end
            end else begin
                cnt <= 0;  // 状态一致，复位计数器
            end
        end
    end

    // 边沿检测：检测 key_out 的下降沿（按键按下）
    reg key_delay;
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            key_delay <= 1;
        end else begin
            key_delay <= key_out;
        end
    end

    assign key_falling = ~key_out & key_delay;  // 下降沿检测

endmodule

这个消抖模块融合了同步器、计数器和边沿检测三个核心技术——是 FPGA 工程中典型的"三个电路组合成一个模块"的设计模式。

消抖中容易忽略的细节：第一级 key_sync0 的 D 输入可能因为按键是异步信号而出现亚稳态。两级寄存器的 MTBF 在 50MHz 下足以将亚稳态概率降低到忽略不计。但如果系统时钟超过 200MHz，建议用三级同步。另外，这里用了 !==（全等比较）来比较 key_sync1 和 key_out——因为 key_sync0 在同步过程中可能短暂出现 X 态，!== 会正确处理 X 状态。

6.4.3 边沿检测电路

边沿检测是每个 FPGA 项目都会用到的"标准工具"：

// 通用边沿检测模块
module edge_detector (
    input  wire       clk, rst_n,
    input  wire       signal_in,
    output reg        rising,     // 上升沿检测
    output reg        falling     // 下降沿检测
);

    reg signal_delay;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            signal_delay <= 0;
            rising  <= 0;
            falling <= 0;
        end else begin
            signal_delay <= signal_in;          // 延迟一拍
            rising  <= signal_in & ~signal_delay; // 上升沿
            falling <= ~signal_in & signal_delay; // 下降沿
        end
    end

endmodule

原理很简单：将输入延迟一个时钟周期，然后用当前值异或延迟值（与/非组合）得到边沿。这个电路在按键响应、通信协议握手、中断触发等场景中无处不在。

6.5 一个真实的踩坑故事：复位释放不当导致计数器从"某处"开始

2021 年我做一个高清视频时序发生器（用于 HDMI 输出），需要一个行计数器和一个帧计数器来生成视频同步信号。系统时钟 148.5MHz（1080p 标准），所有计数器在复位释放后开始计数。测试时发现：大约每 1000 帧中会有一帧图像的起始位置偏移了几个像素，导致屏幕出现一条细线。

用 ILA 抓了行计数值，发现复位释放后的第一个计数值不是预期的 0，而是一个随机数。问题出在复位释放方式：

// 有问题的复位分配
always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        h_cnt <= 0;
        // 其他寄存器也清零...
    end else begin
        h_cnt <= h_cnt + 1;
    end
end

代码看起来没有问题。但板级测试中，rst_n 由板上的电源监控芯片产生——它监视三路电源都稳定后释放复位。问题在于：复位释放信号到达 FPGA 的全局复位引脚时，与 148.5MHz 的时钟沿可能有皮秒级的不确定偏移。如果复位释放瞬间正好落在时钟建立/保持窗口内，部分寄存器的复位被"部分解除"——有些 FF 接收到复位信号，有些没有。

修复方案：

// 修复：用同步复位释放（异步复位，同步释放）
reg rst_sync1, rst_sync2;
always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        rst_sync1 <= 0;
        rst_sync2 <= 0;
    end else begin
        rst_sync1 <= 1;
        rst_sync2 <= rst_sync1;
    end
end

wire rst_sync_n = rst_sync2;  // 同步后的复位信号

// 使用同步后的复位
always @(posedge clk) begin
    if (!rst_sync_n) begin
        h_cnt <= 0;
    end else begin
        h_cnt <= h_cnt + 1;
    end
end

这就是经典的异步复位，同步释放（Asynchronous Assertion, Synchronous Deassertion）技术。复位生效是异步的（立即复位），但复位释放经过两级同步器，确保所有 FF 在同一个时钟沿退出复位。从此再没有出现过复位引起的计数错乱。
