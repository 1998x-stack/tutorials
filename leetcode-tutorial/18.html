堆：低调而强大的数据结构

同学们好。如果你回顾一下我们学过的数据结构，会发现一个有趣的现象：数组、链表、哈希表、树、图——它们的操作要么是 O(1)（哈希表），要么是 O(log n)（平衡树）。但有一个结构，它的插入和删除都是 O(log n)，却能解决一大堆看似无关的问题——它就是堆（Heap），也叫优先队列（Priority Queue）。

堆在 LeetCode 上的出场率非常高。Top K 问题用它、数据流中位数靠它、合并有序链表需要它、Dijkstra 最短路径的核心也是它。毫不夸张地说，堆是 LeetCode 上最重要的"辅助数据结构"之一——它很少单独作为考点，但几乎所有需要"动态维护最值"的场景，堆都是最优解。

本章路线：堆的本质与操作 → Top K 系列（347/215/692）→ 双堆技巧（295）→ 多路归并（23）→ Python heapq 踩坑与技巧。学完本章，你能用堆解决 LeetCode 上约 15% 的题目——这个比例相当高了。

堆的本质：完全二叉树

堆在逻辑上是一棵完全二叉树，在物理上用一个数组存储。对于下标为 i 的元素：左孩子是 2i + 1，右孩子是 2i + 2，父节点是 (i - 1) // 2。

  • 大根堆：每个节点的值 ≥ 子节点的值（堆顶是最大值）
  • 小根堆：每个节点的值 ≤ 子节点的值（堆顶是最小值）

核心操作

操作 时间复杂度 说明
push（插入） O(log n) 插入末尾，执行"上浮"（sift up）
pop（删除堆顶） O(log n) 堆顶与末尾交换，删除末尾，执行"下沉"（sift down）
peek（查看堆顶） O(1) 直接返回数组第一个元素
build（建堆） O(n) 从最后一个非叶子节点开始下沉

建堆的 O(n) 复杂度初学者常感困惑——n 个元素插入 n 次不是 O(n log n) 吗？没错，但如果从无序数组开始，从最后一个非叶子节点开始下沉调整，总比较次数是 O(n)（具体推导：高度从 1 到 log n 的节点分别需要最多 h 次交换，求和后收敛于 O(n)）。

# Python 的 heapq 模块 —— 默认小根堆
import heapq

nums = [3, 1, 4, 1, 5, 9, 2, 6]

heapq.heapify(nums)   # 建堆：O(n)，nums 变成 [1, 1, 2, 3, 5, 9, 4, 6]

heapq.heappush(nums, 0)  # 插入：O(log n)

smallest = heapq.heappop(nums)  # 弹出最小值：O(log n)
# smallest = 0, nums 恢复堆性质

# 查看堆顶（最小值）
top = nums[0]  # O(1)
Python 的 heapq 只有小根堆：如果想用大根堆，有两个常用技巧：(1) 存入负数，pop 时取负回来；(2) 对于自定义对象，定义 __lt__ 时反转比较逻辑。

Top K 系列：堆的看家本领

Top K 问题是堆的最经典应用——"找前 K 个最大/最小/最频繁的元素"。核心思路是维护一个大小为 K 的堆：求 Top K 大用小根堆（堆顶是当前第 K 大），求 Top K 小用大根堆。

前 K 个高频元素（LeetCode 347）

题目：给定一个整数数组，返回前 K 个高频元素。

def top_k_frequent(nums, k):
    from collections import Counter
    count = Counter(nums)

    # 小根堆，维护频率最高的 k 个元素
    heap = []
    for num, freq in count.items():
        heapq.heappush(heap, (freq, num))
        if len(heap) > k:
            heapq.heappop(heap)

    return [num for _, num in heap]

时间复杂度 O(n log k)，空间复杂度 O(n + k)。当 k 远小于 n 时，这比把全部元素排序（O(n log n)）快很多。

面试时被追问怎么办：如果面试官问你"能不能不用堆解决？"——可以告诉你，用快速选择（Quick Select）可以做到 O(n) 平均时间，但最坏情况是 O(n²)。一般场景下堆解法已经够好了，重点是把思路讲清楚。

数组中的第 K 个最大元素（LeetCode 215）

题目：在未排序数组中找第 K 大的元素。

def find_kth_largest(nums, k):
    # 维护大小为 k 的小根堆
    heap = nums[:k]
    heapq.heapify(heap)

    for num in nums[k:]:
        if num > heap[0]:
            heapq.heapreplace(heap, num)  # pop 后 push，一步到位

    return heap[0]  # 堆顶就是第 K 大

heapq.heapreplace 是一个很有用的函数——它先 pop 再 push，比分开调用高效（只需要一次 O(log n) 操作）。但注意：它要求堆非空，且 pop 在 push 之前。

前 K 个高频单词（LeetCode 692）

这道题多了排序规则：频率从高到低，频率相同按字母顺序。

def top_k_frequent(words, k):
    from collections import Counter
    count = Counter(words)

    # 自定义排序：频率升序，字母降序（因为用小根堆，所以要反转）
    heap = []
    for word, freq in count.items():
        heapq.heappush(heap, (-freq, word))  # 频率取负，模拟大根堆

    result = []
    for _ in range(k):
        result.append(heapq.heappop(heap)[1])

    return result

这里的技巧是 (-freq, word)——因为 Python 的 tuple 比较是逐个元素进行的，所以先比 -freq（取负后，频率越高反而越小，pop 时先出），如果频率相同再比单词的字母序。`heapq` 默认小根堆，所以取负是最常用的"假装大根堆"手段。

Top K 思维框架：看到"前 K 个最大/最小/最频繁"→ 维护一个大小为 K 的堆。求最大用小根堆（堆顶是门槛，比门槛大的才进来），求最小用大根堆（Python 中用负数模拟）。

数据流中位数：双堆技巧

这是堆的一个著名应用——Find Median from Data Stream（LeetCode 295）。题目要求设计一个数据结构，支持：添加一个数，以及查询当前所有数的中位数。

最朴素的想法是每次插入时排序（O(n log n)）或在有序数组中二分插入（O(n)），但都不够快。双堆技巧可以做到：插入 O(log n)，查询中位数 O(1)。

思路：用一个小根堆存较大的一半，一个大根堆存较小的一半。关键在于保持两个堆的大小平衡：最多差 1，且大根堆的堆顶 ≤ 小根堆的堆顶。

class MedianFinder:
    def __init__(self):
        self.small = []  # 大根堆（存较小的一半，用负数模拟）
        self.large = []  # 小根堆（存较大的一半）

    def addNum(self, num: int) -> None:
        # 先插入 small（大根堆），再平衡
        heapq.heappush(self.small, -num)

        # 保证 small 的每个元素 ≤ large 的每个元素
        if self.small and self.large and \
           (-self.small[0]) > self.large[0]:
            val = -heapq.heappop(self.small)
            heapq.heappush(self.large, val)

        # 保证大小平衡：len(small) >= len(large)
        if len(self.small) > len(self.large) + 1:
            val = -heapq.heappop(self.small)
            heapq.heappush(self.large, val)
        elif len(self.large) > len(self.small):
            val = heapq.heappop(self.large)
            heapq.heappush(self.small, -val)

    def findMedian(self) -> float:
        if len(self.small) > len(self.large):
            return -self.small[0]
        return (-self.small[0] + self.large[0]) / 2.0

这个代码看起来很繁琐，但核心逻辑就三步：插入 → 调整大小侧 → 调整平衡。面试时写这个算法要注意"大根堆用负数模拟"这个 trick，以及每一步之后都需要重新平衡。

双堆应用场景：除了中位数，所有需要"动态维护一个有序序列的某个分位数"的问题都可以用双堆。比如找数据流的 90% 分位数——调整左右堆的比例从 1:1 变成 9:1 即可。

关于双堆，我还有一个个人的教训。我第一次写 MedianFinder 时，添加数据后只顾着平衡大小，忘记了还需要保证"small 中所有元素 ≤ large 中的所有元素"这一约束。结果 small 的堆顶（最大元素）比 large 的堆顶（最小元素）还大，中位数算出来自然是错的。后面的平衡步骤建立在正确的前提上——但前提错了，一切都错。

合并K个有序链表/数组：多路归并

合并 K 个有序链表（LeetCode 23）是堆的另一个经典应用。题目：给定 K 个有序链表，将它们合并成一个有序链表。

最暴力的方法：把所有节点取出排序（O(N log N)），或者逐一合并（两两合并 K 次，共 O(KN)）。但用堆可以做到 O(N log K)：每次把 K 个链表的当前头节点放入小根堆，弹出最小的那个，再从被弹出的链表取下一个节点入堆。

def merge_k_lists(lists):
    heap = []
    # 把所有链表的头节点入堆
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))
            # 为什么存 i？因为 ListNode 不可比较，用 i 作为次级 key 避免比较 node

    dummy = ListNode(0)
    cur = dummy

    while heap:
        val, i, node = heapq.heappop(heap)
        cur.next = node
        cur = cur.next

        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next

这里有一个重要的细节：heapq 在比较 tuple 时，如果第一个元素相同，会比较第二个元素。但如果两个节点的 val 相同，又没有提供次级比较键，Python 会尝试比较 ListNode 对象——而 ListNode 没有实现 __lt__，会报错。所以加一个 i（链表索引）来打破平局。

自定义对象的堆排序：如果你想把自定义对象放入 heapq，有两个选择：(1) 在 tuple 中加一个不会重复的键（如索引或 id）作为次级比较；(2) 在自定义类中实现 __lt__ 方法。我个人推荐第一种——简单、无需修改类定义、不出错。

踩坑与总结

坑1：Python heapq 只有小根堆

这是 Python 面试中最常见的 "gotcha" 之一。很多在其他语言中用大根堆的习惯带到 Python 来直接用不了。解决方法：

  • 数字取负后放入：heapq.heappush(heap, -num)
  • 自定义对象用 (-priority, data) 的 tuple 形式
  • 或者在类中定义反转的 __lt__

坑2：堆中元素修改后不会自动调整

堆只保证在插入和删除时维持堆性质。如果你直接修改了堆中某个元素的值，堆结构不会被自动修复。这在 Dijkstra 算法中尤为重要——我们通过 push 新值 + 惰性删除来规避，而不是去修改堆中已有的元素。

坑3：heapq.heapreplace 和 heapq.heappushpop 的区别

heapreplace：先 pop 再 push。要求堆非空。

heappushpop：先 push 再 pop。不要求堆非空（至少 push 后肯定非空）。

在 Top K 问题中，如果新元素大于堆顶（当前第 K 大），你需要用新元素替换堆顶。选 heapreplace 还是 heappushpop？——如果新元素一定可以入堆，heapreplace 更高效（少一次 push 的 O(log n)）。但如果新元素还不如堆顶大，heappushpop 会先 push 再 pop，结果不变但多了一次操作。所以标准写法是先判断再操作。

问题 堆策略 时间复杂度
Top K 最大 小根堆，维护大小为 K O(n log K)
Top K 最小 大根堆（负数），维护大小为 K O(n log K)
数据流中位数 双堆：大根堆 + 小根堆 O(log n) 插入，O(1) 查询
合并 K 个有序序列 小根堆存 K 个头指针 O(N log K)
Dijkstra 最短路 小根堆存 (dist, node) O((V+E) log V)
任务调度 / CPU 调度 大根堆按优先级 O(log n) 每次调度
本章总结：堆的结构虽然简单（一个数组，一棵完全二叉树），但它的应用范围极广。它不像哈希表那样"显眼"——面试官很少直接问"请实现一个堆"，但几乎每一家大厂的面试题都有一道能用堆解决的问题。请记住两个最重要的范式：Top K 用小根堆维护"门槛"、中位数用双堆维护"平衡"。这两个模式在手，堆的应用题就难不倒你了。
