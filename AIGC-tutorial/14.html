第14章 视频扩散模型

大家好，我是你们的讲师。如果说 2022-2023 年是"文生图"的年代，那么 2024-2026 年就是"文生视频"的爆发期。OpenAI 的 Sora、字节跳动的 Seedance、腾讯的混元视频、HunyuanVideo、CogVideoX……一波又一波的文生视频模型将 AI 生成的内容从静态图像推向动态影像。

但视频生成远比图像生成困难。多了一个"时间维度"，意味着模型不仅要理解空间上的内容（是什么、在哪里），还要理解时间上的演化（接下来会发生什么）。这一章，我们将系统性地讲解视频扩散模型的架构设计、关键挑战、主流模型对比，以及训练和部署中的实战经验。

14.1 从图像到视频：核心挑战

将图像扩散模型扩展到视频领域，面临三个核心挑战：

挑战描述影响
帧间一致性相邻帧之间的物体位置、形状、颜色必须连贯变化，不能出现突变物体闪烁、消失/重生、颜色跳变
时序连贯性物体的运动轨迹在长时间范围内必须合理（物理规律）违背重力、运动卡顿、物体漂移
计算量激增视频有 T 帧，每帧都需要处理，计算量是图像的 T 倍训练动辄成千上万 GPU 天，推理显存爆炸

早期方法（2022-2023）的解决思路是：在预训练的 2D 图像扩散模型上，加入"时序层"（Temporal Layers）来建模帧间关系。这种"从 2D 扩展到 3D"的思路至今仍是主流范式之一。

💡 直觉理解：视频扩散模型可以看作是一个"三维扩散"问题——空间维度（H×W）加上时间维度（T）。三维卷积和时序注意力就是用来处理这个新增的时间维度的。从图像模型初始化视频模型的权重（"膨胀"技术）是训练视频扩散模型的标配做法。

14.2 时空架构设计

视频扩散模型的核心架构有两种主要路线：3D U-Net 和 3D DiT。

3D U-Net：时空卷积 + 时序注意力

这是 Video Diffusion Model（VDM）和早期 Stable Video Diffusion 使用的架构。它在标准 2D U-Net 的基础上，在每一层加入：

  • 3D 卷积：将 2D 卷积替换为 3D 卷积（核从 3×3 变为 3×3×3），同时处理空间和时间维度
  • 时序自注意力（Temporal Self-Attention）：在空间注意力之后，额外加一层作用于时间维度的自注意力——同一像素位置在不同帧之间做注意力
  • 帧间交叉注意力：将第一帧作为"参考帧"，其余帧通过交叉注意力与参考帧对齐
# 3D U-Net Block 简化结构（空间+时间）
def video_unet_block(x, t_emb, cond):
    # 空间处理（2D 卷积 + 空间自注意力）
    x = SpatialResBlock(x, t_emb)      # 3D Conv (kernel=1×3×3)
    x = SpatialSelfAttention(x)        # 同一帧内跨空间位置

    # 时序处理（时域卷积 + 时序自注意力）
    x = TemporalResBlock(x, t_emb)      # 3D Conv (kernel=3×1×1)
    x = TemporalSelfAttention(x)        # 同一位置跨帧

    # 条件注入（文本/图像）
    x = CrossAttention(x, cond)
    return x

3D DiT：视频扩散 Transformer

2024-2025 年的新模型（如 Sora、HunyuanVideo、CogVideoX）更倾向于使用 3D DiT 架构。它将视频切分为时空 patch（Spatial-Temporal Patches），然后用 Transformer 统一处理所有 token。

  • 3D patchify：将视频卷切成大小为 p_t × p_h × p_w 的时空块，展平为 token 序列
  • 3D RoPE：在位置编码中同时编码空间和时间位置信息
  • 级联架构：先低分辨率生成，再级联超分和时间插帧

14.4 常用数据集与评估标准

视频生成模型的训练和评估需要专门的数据集和指标：

常用数据集

  • WebVid-10M：约 1000 万条文本-视频对，从互联网抓取，是训练视频扩散模型最常用的数据集之一
  • HD-VILA-100M：1 亿条文本-视频对，分辨率更高，时长更长
  • Panda-70M：7000 万条高质量视频片段，由 InternVideo 团队整理
  • UCF-101 / Kinetics-600：学术基准数据集，用于动作识别和视频理解，也被用于生成评估

VBench 评估标准

VBench 是当前最权威的文生视频综合评估框架，包含 16 个维度的指标：

评估维度说明典型得分范围
Subject Consistency主体在帧间是否保持一致的外观0.90 - 0.99
Background Consistency背景是否连贯0.90 - 0.99
Temporal Flickering帧间的闪烁程度0.90 - 0.99
Motion Smoothness运动是否流畅自然0.80 - 0.99
Dynamic Degree视频的运动幅度0.30 - 0.80
Appearance Style视觉风格的匹配度0.15 - 0.30
Spatial Relationship空间关系的合理性0.50 - 0.90
Text-to-Video Alignment文本描述与视频内容的一致性0.50 - 0.85
Overall Impression综合人类评价0.50 - 0.90
💡 评估建议：VBench 的 16 个维度覆盖了视频质量的大部分方面，但要注意——自动化的 VBench 分数不完全等同于人类的主观观感。统计学研究发现，"Subject Consistency"和"Background Consistency"对人类的整体满意度影响最大，这两个维度应该重点关注。

14.5 视频生成的实际应用与限制

尽管视频生成在 2025-2026 年取得了惊人的进展，但它仍然面临一些本质性的限制：

  • 时长限制：即使是 Sora 也只能稳定生成 60 秒的视频。超过这个限度，帧间一致性会快速下降。这与扩散模型本身的设计有关——自回归地逐段生成长视频会累积误差。
  • 物理规律理解不足：视频模型经常违反常识物理——物体应该落下却飘在空中，倒水时水应该向下流却向上飞。这是因为训练数据中存在大量"不自然"的视频（广告特效、CG 动画等），模型没有真正理解物理规律。
  • 计算成本极高：训练一个 Sora 级别的视频模型需要数千张 H100 连续训练数周。推理生成一段 10 秒的 720p 视频也需要数分钟的高端 GPU 时间。
  • 文本理解能力弱：相比图像生成，视频生成中文本 prompt 的遵循度要差得多，特别是涉及"先后顺序"（"先 A 后 B"）和"因果关系"（"因为 A 所以 B"）的文本。

14.6 视频扩散训练中的显存管理

显存管理是视频扩散模型训练和推理中最头疼的问题之一。视频需要同时处理多个帧，显存占用比图像呈线性增长。

# 以 CogVideoX 训练为例的显存占用
# batch_size=1, num_frames=49, resolution=480×720

# 输入数据:
#   latent [1, 49, 16, 40, 60] → ~12 GB (fp16)
# 模型参数:
#   DiT 3B params → ~6 GB (fp16)
# 优化器状态:
#   AdamW + 梯度 → ~24 GB
# 中间激活值（最吃显存的部分）:
#   Transformer activation memory → ~20 GB（使用activation offloading可降到 ~8 GB）
#
# 总显存估算: 12 + 6 + 24 + 20 = ~62 GB（单卡不够！）
# 典型配置: 8×A100 80G，分别加载模型分片 + ZeRO-3 + activation offloading
⚠️ 真实的踩坑故事：我曾在 4×A100 80G 上尝试微调一个视频扩散模型，结果反复 OOM（Out of Memory）。排查时发现，我设置的帧数（49 帧 + 1 帧参考帧）正好让 activation memory 超过了单卡的承载能力。我尝试了各种方法：gradient checkpointing、activation offloading、降低 batch size 到 1……都不够。最后解决方法是：将帧数从 49 降到 25，同时使用 DeepSpeed ZeRO-3 将模型参数均衡分片到 4 张卡上。这个经历让我深刻认识到：视频扩散训练的显存优化没有银弹——你需要同时使用梯度检查点、ZeRO 分片、activation offloading 和精确的帧数选择，缺一不可。在开始训练之前，最好先用一个"显存预算计算器"预估不同配置的显存占用，而不是盲目尝试。

视频扩散模型的快速发展正在重塑内容创作行业——从短视频制作到电影特效，从广告生成到虚拟主播。虽然它还有诸多限制，但可以预见的是，随着 DiT 架构的成熟和算力成本的下降，高质量视频生成的门槛将越来越低。
