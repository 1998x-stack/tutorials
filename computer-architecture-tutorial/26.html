第26章 从零构建计算机

同学们好！今天我们要做一件非常酷的事情——从零开始，自己动手构建一台完整的计算机。不是纸上谈兵，而是真正地从最底层的NAND门开始，一层一层搭上去，直到跑起来一个高级语言程序。这听起来像是在做梦？Noam Nisan和Shimon Schocken在2005年就让它变成了现实。

Nand to Tetris：为什么这个项目如此重要

2005年，以色列魏茨曼科学研究所的两位教授——Noam Nisan和Shimon Schocken——出版了一本名为《The Elements of Computing Systems》的书，中文版译作《计算机系统要素》。这本书提出了一个雄心勃勃的前提：从NAND门开始，通过12个项目，构建一台完整、可运行的计算机系统——然后在上面运行Tetris（俄罗斯方块）。这就是著名的"Nand to Tetris"项目。

为什么这个项目对学习体系结构如此重要？因为在传统教学中，我们总是在"信任的抽象层"上工作——你写C代码时信任编译器，学指令集时信任处理器已经设计好了，学数字逻辑时信任门电路已经封装好了。每层之间都有"这里我们就不深究了"的断层。Nand to Tetris消除了所有断层：你亲手实现每一层，从NAND门到操作系统，每一步都没有黑盒。

截至2024年，这个课程已经被全球超过40万名学生选修，Coursera评分常年保持在4.9/5.0。它被MIT、Stanford、Princeton、UC Berkeley等顶级院校采纳为计算机系统入门课程的指定教材。Shimon Schocken教授在TED上的演讲"The Self-Sufficient Learner"也讲述了这门课背后的教育理念——让学生成为自给自足的学习者。

课程的核心理念可以概括为三句话：没有黑盒（No Black Boxes）、你自己构建一切（You Build Everything）、它真的能跑（It Really Runs）。课程的最终目标是在你自己的Hack计算机上运行Tetris——黑客马拉松精神拉满。

第1-5章：从NAND到计算机硬件

Nand to Tetris的前5个项目专注于硬件层面，从逻辑门一路构建到完整的计算机。我们来逐一走过这些项目。

Project 1：布尔逻辑（Boolean Logic）。一切从NAND门开始。为什么是NAND？因为NAND是通用的（functionally complete）——你可以用NAND门构造出任何其他逻辑门。你需要实现AND、OR、NOT、XOR、Mux（多路选择器）、DMux（多路分配器）等芯片。每个芯片的HDL描述只有几行，但你必须理解它们的真值表和行为。

Project 2：布尔算术（Boolean Arithmetic）。从HalfAdder（半加器）开始——两个输入（a, b），两个输出（sum, carry）。然后组合两个HalfAdder构成FullAdder（全加器）。再把16个FullAdder级联成16位加法器（Add16）。最后，用加法器加上一些逻辑门构建ALU（算术逻辑单元）——Hack ALU只用一个控制信号就能完成18种运算。

Project 3：时序逻辑（Sequential Logic）。时序逻辑引入了"状态"的概念——输出不仅取决于当前输入，还取决于历史状态。从DFF（D型触发器）开始，构建Bit（1位寄存器），然后构建Register（16位寄存器），再到RAM8、RAM64、RAM4K、RAM16K。同时还要构建Program Counter（程序计数器），这是CPU执行指令的核心。

Project 4：机器语言（Machine Language）。这是第一次"编程"——直接在Hack汇编中写程序。你需要编写乘法、I/O操作等程序。Hack的ISA非常简洁：两个16位寄存器（A和D），一个15位的地址寄存器（A寄存器的低15位），C指令格式为dest = comp ; jump。没有复杂寻址模式，没有多级流水线，但图灵完备。

Project 5：计算机体系结构（Computer Architecture）。最后一个硬件项目：把CPU（ALU + 寄存器组 + 控制逻辑）和Memory（RAM + ROM）连接起来，构成完整的Hack计算机。CPU的核心逻辑：从ROM取指令 → 解码 → ALU运算 → 写回寄存器/内存。你亲手搭建的计算机，在这一刻真正可以执行程序了。

关键洞见：从NAND门开始，一步一个脚印地构建出完整的计算机系统——这种从头到尾的理解，是任何教科书都无法给予的。当你亲眼看着自己用NAND搭出来的计算机在屏幕上显示出"Hell World"时，那种成就感无可替代。

第6-11章：从汇编器到编译器

硬件搭建完毕，现在进入软件层面。Nand to Tetris的后半段将你的Hack计算机从只有机器码的裸机，变成可以运行高级语言的完整系统。

Project 6：汇编器（Assembler）。你需要用一门高级语言（Java/Python/任何语言）编写一个汇编器，将Hack汇编代码翻译成机器码。这涉及到符号解析（symbol resolution）——标签（labels）和变量（variables）需要转换为具体的内存地址。汇编器让你第一次感受到：原来"代码"是这样变成"0和1"的。

Project 7-8：VM翻译器（VM Translator）。这两章引入了一个栈式虚拟机（Stack-based VM），类似于JVM的简化版。VM代码在虚拟的栈机上运行——操作数在栈上压入和弹出，不需要寄存器。你的任务是将VM指令翻译成Hack汇编。这种两层编译模型（高级语言 → VM → 汇编 → 机器码）是Java、C#等语言的核心架构。

Project 9：高级语言（High-Level Language）。Jack语言——一种简化的Java-like面向对象语言，包含类、方法、字段、数组、字符串、控制流等概念。你用Jack编写程序（包括最终目标：Tetris）。语法很简洁，但图灵完备，而且有完整的OOP支持。

Project 10-11：编译器（Compiler）。最宏大的项目：将Jack程序编译为VM代码。分为两个阶段：语法分析（Tokenizer + Parser，构建抽象语法树AST）和代码生成（Code Generator，将AST翻译为VM指令）。编译器的工作原理——词法分析、语法分析、符号表管理、递归下降解析——都是你在编译原理课上学到的经典技术，但这一次你亲手实现了它们。

完成这6个项目后，你的完整软件栈是：Jack应用程序 → Jack编译器 → VM翻译器 → Hack汇编器 → 机器码 → 在Hack计算机上运行。这是一个完整的、端到端的系统，每一步都是你亲手构建的。

项目构建内容使用工具核心概念
1布尔逻辑门（AND, OR, NOT, XOR等）HDL + 硬件模拟器功能完备性、真值表、多路选择
2布尔算术单元（加法器、ALU）HDL + 硬件模拟器补码运算、ALU设计
3时序逻辑（寄存器、RAM、PC）HDL + 硬件模拟器时钟边沿触发、状态保持
4机器语言编程汇编器 + CPU模拟器ISA、寻址、控制流
5完整计算机（CPU + 内存）HDL + 硬件模拟器冯诺依曼架构、数据通路
6汇编器（符号 → 机器码）高级语言符号解析、两遍扫描
7VM翻译器 I（基本指令）高级语言栈式虚拟机、内存段
8VM翻译器 II（控制流 + 函数调用）高级语言调用栈、作用域、递归
9高级语言编程（Jack）Jack编译器 + VM模拟器OOP、数组、字符串
10编译器 I（词法/语法分析）高级语言Tokenizer、Parser、AST
11编译器 II（代码生成）高级语言递归下降生成、符号表
12操作系统（内存管理、I/O驱动）Jack语言多任务、屏幕绘制、键盘输入

硬件描述语言与FPGA原型验证

Nand to Tetris中的硬件描述使用的是课程自研的HDL和硬件模拟器。但在真实的工业界和学术界，Verilog是硬件描述语言的标准。如果你未来想设计真正的芯片，Verilog是必须掌握的工具。

Verilog的核心概念相当简洁：module（模块——一切硬件的封装单元）、wire（连线——模块间的信号连接）、reg（寄存器——存储状态）、assign（连续赋值——组合逻辑）、always @(posedge clk)（时序逻辑——在时钟上升沿触发）。下面是一个典型的5级RISC-V流水线的Verilog描述示例，展示IF/ID/EX/MEM/WB各阶段：

// 5-stage RISC-V pipeline core (RV32I)
// Simplified structural description showing IF/ID/EX/MEM/WB stages

module riscv_pipeline (
    input  wire        clk,
    input  wire        rst_n,
    input  wire [31:0] instr,
    output wire [31:0] pc_fetch,
    output wire        mem_read,
    output wire        mem_write,
    output wire [31:0] mem_addr,
    output wire [31:0] mem_wdata,
    input  wire [31:0] mem_rdata
);

    // --- IF Stage: Instruction Fetch ---
    reg  [31:0] pc;
    wire [31:0] pc_next;
    wire [31:0] pc_plus_4;

    assign pc_plus_4 = pc + 32'd4;
    assign pc_fetch  = pc;

    // PC update with stall and flush support
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            pc <= 32'h8000_0000;
        else if (!stall_if)
            pc <= flush_if ? branch_target : pc_plus_4;
    end

    // IF/ID pipeline register
    reg  [31:0] if_id_pc;
    reg  [31:0] if_id_instr;
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            if_id_pc    <= 32'h0;
            if_id_instr <= 32'h0;
        end else if (!stall_if) begin
            if_id_pc    <= pc;
            if_id_instr <= flush_if ? 32'h0 : instr;
        end
    end

    // --- ID Stage: Instruction Decode ---
    reg  [31:0] id_pc;
    wire [6:0]  opcode;
    wire [4:0]  rd, rs1, rs2;
    wire [2:0]  funct3;
    wire [6:0]  funct7;

    assign opcode = if_id_instr[6:0];
    assign rd     = if_id_instr[11:7];
    assign funct3 = if_id_instr[14:12];
    assign rs1    = if_id_instr[19:15];
    assign rs2    = if_id_instr[24:20];
    assign funct7 = if_id_instr[31:25];

    // Control unit (combinational)
    reg        reg_write, mem_to_reg;
    reg        mem_read_id, mem_write_id;
    reg        alu_src;
    reg [1:0]  alu_op;
    reg        branch, jump;

    always @(*) begin
        // Defaults
        {reg_write, mem_to_reg, mem_read_id, mem_write_id,
         alu_src, branch, jump} = 7'b0;
        alu_op = 2'b00;
        case (opcode)
            7'b0110011: begin // R-type
                reg_write = 1'b1; alu_op = 2'b10;
            end
            7'b0000011: begin // I-type (load)
                reg_write = 1'b1; mem_to_reg = 1'b1;
                mem_read_id = 1'b1; alu_src = 1'b1;
            end
            7'b0100011: begin // S-type (store)
                mem_write_id = 1'b1; alu_src = 1'b1;
            end
            7'b1100011: begin // B-type (branch)
                branch = 1'b1; alu_op = 2'b01;
            end
        endcase
    end

    // Register file: 32 x 32-bit registers
    reg [31:0] regfile [0:31];
    wire [31:0] read_data1, read_data2;

    assign read_data1 = regfile[rs1];
    assign read_data2 = regfile[rs2];

    always @(posedge clk) begin
        if (reg_write && wb_rd != 5'b0)
            regfile[wb_rd] <= wb_result;
    end

    // ID/EX pipeline register
    reg  [31:0] id_ex_pc;
    reg  [31:0] id_ex_rs1_data, id_ex_rs2_data;
    reg  [4:0]  id_ex_rs1, id_ex_rs2, id_ex_rd;
    reg         id_ex_reg_write, id_ex_mem_to_reg;
    reg         id_ex_mem_read, id_ex_mem_write;
    reg         id_ex_alu_src;
    reg  [1:0]  id_ex_alu_op;
    reg         id_ex_branch;

    always @(posedge clk) begin
        if (!rst_n) begin
            id_ex_reg_write <= 1'b0;
            // ... (reset all control signals)
        end else if (!stall_id) begin
            id_ex_pc         <= id_pc;
            id_ex_rs1_data   <= read_data1;
            id_ex_rs2_data   <= read_data2;
            id_ex_rs1        <= rs1;
            id_ex_rs2        <= rs2;
            id_ex_rd         <= rd;
            id_ex_reg_write  <= reg_write;
            id_ex_mem_to_reg <= mem_to_reg;
            id_ex_mem_read   <= mem_read_id;
            id_ex_mem_write  <= mem_write_id;
            id_ex_alu_src    <= alu_src;
            id_ex_alu_op     <= alu_op;
            id_ex_branch     <= branch;
        end
    end

    // --- EX Stage: Execute ---
    wire [31:0] alu_in2;
    assign alu_in2 = id_ex_alu_src ? imm_ext : id_ex_rs2_data;

    // ALU (simplified)
    reg [31:0] alu_result;
    always @(*) begin
        case (alu_ctrl)
            3'b000: alu_result = alu_in1 + alu_in2;   // ADD
            3'b001: alu_result = alu_in1 - alu_in2;   // SUB
            3'b010: alu_result = alu_in1 & alu_in2;   // AND
            3'b011: alu_result = alu_in1 | alu_in2;   // OR
            3'b100: alu_result = alu_in1 & alu_in2;   // XOR (placeholder)
            default: alu_result = 32'h0;
        endcase
    end

    // EX/MEM pipeline register
    reg  [31:0] ex_mem_alu_result;
    reg  [31:0] ex_mem_store_data;
    reg  [4:0]  ex_mem_rd;
    reg         ex_mem_reg_write, ex_mem_mem_to_reg;
    reg         ex_mem_mem_read, ex_mem_mem_write;

    // --- MEM Stage: Memory Access ---
    assign mem_read  = ex_mem_mem_read;
    assign mem_write = ex_mem_mem_write;
    assign mem_addr  = ex_mem_alu_result;
    assign mem_wdata = ex_mem_store_data;

    // MEM/WB pipeline register
    reg  [31:0] mem_wb_alu_result;
    reg  [31:0] mem_wb_mem_data;
    reg  [4:0]  mem_wb_rd;
    reg         mem_wb_reg_write, mem_wb_mem_to_reg;

    // --- WB Stage: Write Back ---
    wire [4:0]  wb_rd;
    wire [31:0] wb_result;
    assign wb_rd    = mem_wb_rd;
    assign wb_result = mem_wb_mem_to_reg ? mem_wb_mem_data : mem_wb_alu_result;

    // Hazard detection unit (forwarding + stalling)
    // ... (forwarding muxes and stall logic omitted for brevity)

endmodule

上面这段代码描述了一个5级流水线RISC-V处理器的核心骨架。如果你在大学里做过计算机组成原理课设，可能见过类似的代码——但不同的是，现在你有机会把这段代码烧录到真正的硬件上运行。

FPGA（Field-Programmable Gate Array，现场可编程门阵列）就是用来干这个的。FPGA是一种可重新编程的硬件：它由大量可配置逻辑块（CLB/LUT）和可编程互连组成，你可以通过烧录比特流（bitstream）将FPGA配置成任意数字电路。现代高端FPGA如Xilinx Versal、Intel Agilex，已经集成了数百万个LUT、ARM处理器核心和AI引擎。

实用建议：在投片（tape-out）之前，先用FPGA验证你的处理器设计。这是一个行业标准流程——一个RTL bug在仿真阶段发现只需几小时修复，但如果到芯片回来才发现，修复成本可能高达数百万美元。FPGA原型验证是芯片设计流程中不可或缺的一环。

开源EDA工具链

在很长一段时间里，芯片设计工具被少数几家商业公司垄断——Synopsys、Cadence、Siemens EDA。一套完整的EDA工具 license 每年花费数十万美元，这在很大程度上阻碍了硬件开源运动的兴起。但近几年，情况发生了根本性变化。

Yosys是一个开源的RTL综合工具。它将Verilog RTL代码综合为门级网表（gate-level netlist）。Yosys支持Verilog-2005语法子集，能够处理工业级设计。它的综合流程包括：read_verilog → hierarchy → proc → flatten → opt → techmap → abc（使用ABC工具进行逻辑优化和工艺映射）。

OpenROAD是完整的开源物理设计工具链，功能覆盖从门级网表到GDSII的全部流程：floorplanning（布图规划）→ placement（标准单元布局）→ CTS（时钟树综合）→ routing（布线）→ signoff验证。

OpenLANE将Yosys、OpenROAD以及一系列其他开源工具整合为一个自动化的RTL-to-GDSII流程。只需一个命令，就能从Verilog代码生成可交付制造的版图：make mount && ./flow.tcl -design my_design。

SkyWater 130nm是第一个完全开源的工艺设计套件（PDK），由SkyWater Technology和Google合作发布。这意味着任何人都可以下载这个130nm工艺的完整设计规则、器件模型和标准单元库，并用于开源芯片设计。Google和Efabless还推出了免费的芯片流片计划（shuttle program）——开源芯片设计可以通过这个计划免费制造。

这里分享一个真实的故事：清华大学的一个本科生团队在2022年参加了Efabless的芯片设计挑战赛，他们用OpenLANE工具链设计了一颗基于RISC-V的SoC，集成了协处理器和AI加速功能。设计经过SkyWater 130nm工艺流片，芯片在回片后一次点亮。这些学生没有花费一分钱的EDA工具费用，就完成了从设计到流片的完整流程。这在前开源时代是完全不可想象的。

开源EDA的意义远不止省钱。它降低了芯片设计的门槛，让更多大学、初创公司和爱好者能够参与到芯片设计中来。正如Linux改变了软件世界，开源EDA正在改变硬件世界。

好了，现在你已经有了一台自己的计算机——不论是Nand to Tetris中的Hack计算机还是一个在FPGA上运行的RISC-V核。但一台计算机好不好用，不只看它能不能跑，更要看它跑得多快。下一章，我们将学习如何测量和优化计算机系统的性能。
