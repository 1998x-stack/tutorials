第十五章：探索与利用策略

同学们好！上一章我们讨论了连续动作空间算法——DDPG、TD3 和 SAC。这些算法能在机器人控制、自动驾驶等领域大显身手。但细心的你可能已经注意到一个潜藏的问题：智能体在未知环境中，到底应该选择已知的最佳动作，还是尝试可能更好的新动作？

这个问题在强化学习中被称为探索与利用的权衡（Exploration-Exploitation Dilemma），它是贯穿整个领域的一条主线。从最简单的多臂赌博机到最前沿的深度RL探案，所有算法都在以各自的方式回答这个问题。本章我们就来系统性地梳理这些探索策略，看看三十多年来研究者们是如何攻克这个难题的。

15.1 探索困境

要理解探索困境，我们从一个最简单的场景开始。

餐馆困境：假设你和朋友来到一条美食街，这里有10家餐馆，你打算来30次。每次你选择一家，吃完后得到一个"满意度评分"（1~10分）。你的目标是让30次的总满意度最大化。

问题来了：

  • 如果永远只去第一次去的那家——你可能会错过真正最好的餐馆。
  • 如果每次随机选一家——你永远无法充分利用已知的好餐馆。
  • 如果先去每一家试一次，然后一直去评分最高的一家——你只试了一次，万一那家那天只是运气好呢？

这就是探索困境。在强化学习的术语中，每个餐馆是一个臂（arm），每次选择是一个拉杆（pull），评分是奖励（reward）。这个纯碎的决策问题就是著名的多臂赌博机问题（Multi-Armed Bandit）。

形式化地说，我们有 K 个动作（臂），每个动作 a 有一个未知的期望奖励 Q*(a) = E[r | a]。在第 t 步，智能体选择动作 At，获得奖励 Rt。令最优动作为 a* = argmaxa Q*(a)。则期望遗憾（Expected Regret）定义为：

Regret(T) = T · Q*(a*) - Σt=1T E[ Rt ]

遗憾衡量的是"如果早知道最优动作，能多拿多少奖励"。最优策略就是最小化累积遗憾。注意这个"遗憾"不是情感上的后悔——它是一个严谨的数学量，用来量化探索不足或探索过度的代价。

为什么遗憾是比"总奖励"更好的指标？

假设两个算法在同一赌博机上的总奖励分别是 200 和 220。单看数字，220 更好。但如果这个赌博机的最优期望奖励是 10（所以 T×10 = 300），那么前者的遗憾是 100，后者是 80——差距显著缩小了。遗憾消除了任务难度的差异，让我们能比较算法在不同问题上的相对表现。更重要的是，理论家们关心遗憾的增长阶——是 O(log T) 还是 O(√T)？这决定了算法是否能"学习到最终"。

餐馆困境的直觉告诉我们：探索和利用是一对矛盾。探索多，利用的信息就不充分；利用多，可能错过更好的选择。好的探索策略需要在这两个极端之间找到平衡点。

15.2 ε-贪心及其变体

最简单的探索策略就是ε-贪心 (ε-greedy)：以概率 ε 随机选择一个动作（探索），以概率 1-ε 选择当前估计 Q 值最大的动作（利用）。

它的实现非常简洁：

import numpy as np

class EpsilonGreedyBandit:
    """ε-贪心多臂赌博机"""
    def __init__(self, n_arms, epsilon=0.1):
        self.n_arms = n_arms
        self.epsilon = epsilon          # 探索概率
        self.Q = np.zeros(n_arms)       # 动作价值估计
        self.N = np.zeros(n_arms)       # 每个动作被选的次数
        self.t = 0                      # 总步数

    def select_action(self):
        self.t += 1
        if np.random.random() < self.epsilon:
            return np.random.randint(self.n_arms)  # 探索
        else:
            return np.argmax(self.Q)               # 利用

    def update(self, arm, reward):
        self.N[arm] += 1
        # 增量式更新均值
        self.Q[arm] += (reward - self.Q[arm]) / self.N[arm]


# 使用衰减 ε
class DecayEpsilonGreedy(EpsilonGreedyBandit):
    """ε 随时间衰减的贪心策略"""
    def select_action(self):
        self.t += 1
        # ε = 1 / t，随时间自然衰减
        eps = 1.0 / self.t
        if np.random.random() < eps:
            return np.random.randint(self.n_arms)
        else:
            return np.argmax(self.Q)

ε-贪心的三个变体：

  • 固定 ε：最简单，ε=0.1 意味着永远有 10% 的随机探索。坏处是即使已经找到了最优动作，仍然在浪费 10% 的机会。累积遗憾是线性的 O(T)。
  • 衰减 ε：让 ε 随时间减小，例如 εt = 1/t。早期充分探索，后期集中利用。理论上遗憾可以收敛到 O(log T)，但衰减速度需要精心设计——太快了探索不足，太慢了浪费太多。
  • 乐观初始化 (Optimistic Initialization)：把 Q 的初始值设置得很高（例如全都设为 10，而实际奖励范围是 0~1）。这样一开始每个动作都被"高估"了，智能体会自然地先尝试每个动作来验证它们是否真的那么好。一旦某个动作的真实价值被揭示，它的估计值会回落到合理范围，其他未被充分尝试的动作仍然保持高估值，从而被主动探索。这种方法不需要单独的 ε 参数，非常优雅。

Boltzmann / Softmax 探索：

ε-贪心在探索时均匀随机选择所有非最优动作——它对"次优"和"很差"一视同仁。更好的做法是根据 Q 值的大小来决定探索哪些动作。Softmax 策略正是这么做的：

π(a | s) = exp( Q(s,a) / τ ) / Σa' exp( Q(s,a') / τ )

其中 τ 是温度参数 (temperature)。τ → 0 时策略退化为贪心（只选最优）；τ → ∞ 时策略退化为均匀随机。中等温度下，好的动作被选中的概率更高，差的动作也有机会被尝试——但概率更低。这种"按质探索"策略往往比 ε-greedy 的"盲目探索"更高效。

踩坑故事："永不衰减的 ε" 葬送了一个项目

我曾经带过一个研究生，他要在某个线上推荐系统中用 ε-greedy 做 A/B 测试优化。他设计得很漂亮：10 个候选策略（臂），用 ε=0.1 做探索。上线跑了两周，累计收益比基线还低 5%。他百思不得其解，来问我。

我让他拉一下每个臂被选中的次数分布。结果发现最优策略被选了约 1 万次，但次优和第三优的策略也被选了各 8000 多次——因为 10% 的探索在 14 天里均匀分布在 9 个次优臂上，每个都拿到了大量本不该属于它们的流量。

问题出在哪？固定 ε=0.1 意味着即使某臂已经被清晰地证明是次优的（采样了 5000 次），系统仍然以同等概率继续探索它。这就是固定 ε 的致命伤——它不区分"已经充分探索过的"和"还没怎么探索的"。正确的做法是使用衰减 ε，或者改用 UCB——UCB 会自动减少对已知次优动作的探索。

教训：固定 ε 只适合短时间运行或作为理论教学的例子。在真实系统中，务必使用衰减 ε 或更高级的探索策略。否则你探索的每一分成本，都在白白流失。

15.3 UCB 与汤普森采样

ε-greedy 有一个深层问题：它做出"探索/利用"决策时是完全盲目的——抛硬币决定探索，然后均匀随机选动作。它没有回答一个核心问题：哪个动作最值得探索？

置信上界（Upper Confidence Bound, UCB） 给出了一个漂亮的答案。UCB 的核心思想是：对每个动作维护一个"乐观估计"，选择具有最高上界的动作。公式如下：

At = argmaxa [ Qt(a) + c · √ ( ln t / Nt(a) ) ]

其中：

  • Qt(a) 是动作 a 当前的平均奖励估计（"利用部分"）
  • c · √(ln t / Nt(a)) 是置信区间宽度（"探索部分"）
  • Nt(a) 是动作 a 被选的次数，t 是总步数
  • c 是控制探索程度的超参数

这个公式的美妙之处在于：

  • 如果一个动作很少被选（Nt(a) 很小），不确定性大，UCB 鼓励探索它。
  • 如果一个动作已经被选了很多次（Nt(a) 很大），不确定性小，UCB 会降低它的"探索加分"。
  • 随着 t 增长，ln t 增长非常缓慢——这意味着探索量随总步数对数增长，恰好是最优的探索速率。

UCB 的理论保证： 可以证明 UCB 的累积遗憾是 O(log T)，即在 T 步后，遗憾的上界以对数方式增长。这是理论上最优的遗憾阶——不存在任何算法能达到比 O(log T) 更好的遗憾增长阶。相比之下，固定 ε 的 ε-greedy 的遗憾是 O(T)（线性增长），差距巨大。

汤普森采样（Thompson Sampling） 是另一种极为优雅的探索方法。它的思路完全不同：

  • 对每个动作 a，维护一个后验分布 p(Q(a) | 历史数据)。
  • 每一步从每个动作的后验分布中采样一个 Q 值。
  • 选择采样值最大的那个动作。

直观理解：如果某个动作的 Q 值不确定性很大（后验分布很宽），采样时它有可能"运气好"抽到一个很高的值，从而被选中。随着数据增多，后验分布变窄，采样值趋近真实值，探索自然减少。

在伯努利奖励（0/1 奖励）的情况下，使用 Beta 分布作为共轭先验，汤普森采样的实现极其简洁：

class ThompsonSamplingBandit:
    """汤普森采样：Beta-Bernoulli 模型"""
    def __init__(self, n_arms, alpha=1, beta=1):
        self.n_arms = n_arms
        # Beta 分布参数，alpha 是成功次数+1，beta 是失败次数+1
        self.alpha = [alpha] * n_arms
        self.beta  = [beta]  * n_arms

    def select_action(self):
        # 从每个臂的后验分布中采样
        samples = [np.random.beta(a, b)
                   for a, b in zip(self.alpha, self.beta)]
        return np.argmax(samples)

    def update(self, arm, reward):
        # 更新 Beta 分布参数
        if reward == 1:
            self.alpha[arm] += 1
        else:
            self.beta[arm] += 1

汤普森采样虽然在 1933 年就被提出来了，但在很长一段时间里被学术界忽视了，因为缺乏理论分析。直到 2010 年左右，研究者才证明它在许多场景下也是渐近最优的（遗憾 O(log T)），而且在实际任务中往往比 UCB 表现更好。2012 年 Chapman & Bell 在微软的在线广告系统中大规模部署了汤普森采样，据称提升了数百万美元的广告收入。

UCB vs 汤普森采样：直觉对比

UCB 像一个"理性乐观者"：它精确计算每个选择的不确定性上界，然后小心翼翼地探索——数学上美，但需要手动调参数 c。

汤普森采样像一个"概率决策者"：它不从公式推导，而是通过采样自然引入不确定性。它的优势在于：天然处理延迟反馈、容易扩展到复杂模型、没有额外的超参数。在深度学习时代，汤普森采样的"采样后贪心"范式更容易与神经网络结合——你只需要在 Q 网络的输出上增加不确定性估计，其余的框架完全不变。

15.4 深度 RL 中的探索

前面讨论的探索方法都假设环境是稳定的（stationary）或者变化缓慢的。但在深度强化学习中，智能体面对的往往是高维状态空间（如游戏画面）、极度稀疏的奖励（如 Montezuma's Revenge 中几千步才有一个正奖励），而且奖励信号本身也随着 Q 网络的更新而变化。这就对探索策略提出了全新的挑战。

Montezuma's Revenge 问题：

这是深度 RL 历史上最著名的"硬骨头"之一。在这个 Atari 游戏中，智能体控制一个探险家在充满陷阱、钥匙和门的迷宫中穿行。关键问题是：奖励极其稀疏——你可能需要执行数百步正确操作才能第一次拿到钥匙（正奖励），在此之前智能体只能得到零奖励。标准 DQN（使用 ε-greedy）在这个游戏上的得分是 0——它从未成功拿到过第一个正奖励，因此从未学到任何有价值的东西。

随机网络蒸馏（Random Network Distillation, RND）：

RND 由 OpenAI 在 2018 年提出，是解决稀疏奖励探索问题的里程碑式工作。它的思路非常巧妙：

  • 初始化一个固定的随机目标网络 f: 状态 → 特征向量。
  • 训练一个预测网络 f̂: 状态 → 特征向量，让它去预测目标网络的输出。
  • 对于常见的、已经被探索过的状态，预测网络能很好地拟合目标网络，预测误差小。
  • 对于新颖的、从未见过的状态，预测误差大——这个误差就被当作内在奖励（intrinsic reward）。

换句话说，RND 鼓励智能体去访问"预测网络搞不定"的状态——那些未知的、新颖的地方。内在奖励 + 外在奖励共同驱动智能体的行为：

rtotal = rextrinsic + β · || f(s) - f̂(s) ||2

OpenAI 用 RND 在 Montezuma's Revenge 上取得了破纪录的 10000+ 分，证明了这种探索方法的强大。

内在好奇心模块（Intrinsic Curiosity Module, ICM）：

ICM 由 DeepMind 在 2017 年提出，思路与 RND 类似但更加"有根据"：好奇心的来源是对环境的预测能力。ICM 包含两个组件：

  • 前向模型：给定状态 s 和动作 a，预测下一个状态的特征 φ(s')。
  • 逆向模型：给定状态 s 和 s'，预测中间执行的动作 a。

好奇心（内在奖励）等于前向模型的预测误差：如果智能体无法预测一个动作会带来什么结果，说明这个转换是"令人惊讶的"、"信息量大的"——它值得被探索。逆向模型的作用是学习一个对动作相关的信息敏感的状态特征表示，避免被"无关噪声"（如树叶晃动）吸引。

基于计数的探索（Count-Based Exploration）：

最原始的探索思路：记录每个状态被访问的次数 N(s)，然后给罕见状态额外奖励 r = 1 / √(N(s))。但状态空间是连续且高维的（比如一帧 Atari 画面），不可能精确计数每个像素组合。解决方法是用哈希编码（Hash Coding）：

  • 用一个哈希函数将高维状态映射到有限个哈希桶中。
  • 统计每个哈希桶被访问的次数。
  • 访问次数少的桶对应的状态获得高内在奖励。

这种方法简单粗暴，配合适当的哈希函数（如 SimHash、通过学习得到的编码器），在多个 Atari 游戏中取得了不错的探索效果。

深度 RL 探索的核心矛盾

传统探索方法（ε-greedy、UCB）面临一个根本困境："在不知道什么是'有趣的'情况下，如何选择探索方向？"

在低维离散动作空间中，均匀探索勉强可行。但在深度 RL 的高维状态空间中，均匀探索如同大海捞针——Montezuma's Revenge 的动作序列空间大到天文数字，随机漫步无法触及任何正奖励状态。

这就解释了为什么深度 RL 的探索需要状态层面的指导：不是"随机选动作"，而是"去那些你还没搞明白的地方"。RND 和 ICM 的本质都是在学习一个"新奇度函数"，让探索变得有方向。

15.5 探索策略对比

我们把本章讨论的探索策略放在一起做一个系统的比较：

策略 探索依据 计算开销 理论保证 深度RL兼容性 最佳使用场景
ε-greedy（固定） 随机 极低 O(T) 遗憾（线性） 简单兼容 基准测试、教学示例
ε-greedy（衰减） 随机 + 时间 极低 O(log T)（理论最优） 简单兼容 小规模问题、快速跑通基线
乐观初始化 高估初始值 极低 O(log T) 适合表格型 确定性系统、通信网络
Softmax/Boltzmann Q 值按比例 低 依赖 τ 调度 兼容（需调温） 离散动作、需要差异化探索
UCB 不确定性上界 低 O(log T)（强保证） 需扩展（不确定性估计） 在线推荐、广告投放
汤普森采样 后验采样 中 O(log T)（渐近最优） 需贝叶斯神经网络 在线广告、临床试验、延迟反馈
基于计数+哈希 状态新奇度 中 依赖哈希函数质量 好 Atari 游戏、高维离散状态
ICM（好奇心） 预测误差 高 无明显理论保证 天然兼容 稀疏奖励、机器人控制
RND 蒸馏误差 高 无明显理论保证 天然兼容 超稀疏奖励（如 Montezuma）

从表中我们可以看出一个清晰的规律：探索越有效，计算开销越大。这是一个正确的权衡方向——在最难的问题（超稀疏奖励）上，我们愿意用计算量换取好的探索策略。而在简单问题上，固定 ε-greedy 已经够用，不需要复杂的方法。

实践选择指南：

- 刚入门/快速验证 → ε-greedy 衰减，3 行代码搞定
- 在线系统（广告、推荐） → UCB 或汤普森采样，理论有保障
- Atari 游戏 → 基于计数 + 哈希，效果好实现简单
- 超稀疏奖励（如 Montezuma 类问题） → RND 或 ICM，专门为解决这个问题设计
- 真实机器人 → ICM，好奇心驱动机器人主动探索环境中的可交互物体

本章小结

  • 探索与利用的权衡是强化学习的核心问题，从多臂赌博机到深度 RL 无处不在。
  • 多臂赌博机是探索问题的最小化形式，完美隔离了"学习"和"决策"的纠缠。
  • 期望遗憾是衡量探索策略优劣的标准指标，最优算法的遗憾以对数方式增长 O(log T)。
  • ε-greedy 最简单，但固定 ε 导致线性遗憾；衰减 ε 和乐观初始化可以改善。
  • Softmax / Boltzmann 探索 按 Q 值比例探索，比 ε-greedy 的均匀探索更聪明。
  • UCB 通过置信上界自动权衡探索与利用，有最强的理论保证。
  • 汤普森采样 通过后验采样自然引入探索，实用中往往最优且没有超参数。
  • 深度 RL 的探索 需要状态层面的指导：RND、ICM、基于计数的哈希编码等方法通过内在奖励驱动智能体探索未知状态。
  • 探索不是孤立存在的——它需要与奖励设计（Reward Design）紧密配合。再好的探索策略，如果奖励信号本身设计不当，也无法学到正确的行为。下一章我们将讨论如何设计奖励函数，让智能体既能高效探索，又能学到我们真正期望的行为。

探索与利用是强化学习中一个哲学色彩很浓的话题。它不只是一个算法问题——它触及了"智能体如何在不完全信息下做决策"这个根本问题。从餐馆选择到 Montezuma 的宝藏，从在线广告到火星车导航，每一次决策都在回答同样的问题：你应该相信已知的，还是追求未知的？
