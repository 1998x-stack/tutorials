第4章 BERT开山论文精读(下)

大家好，我是你们的讲师。上一章我们把BERT论文的架构设计、MLM任务和训练配置聊透了。但BERT能成为"NLP的ImageNet"的另一个关键原因，在于它提供了一套统一的微调范式——任何NLU任务，只要把输入整理好塞进BERT，在输出端加一个简单的分类器，就能达到远超之前方法的效果。这种"one model for all"的简洁性，让NLP社区从"每个任务设计专用架构"的时代迈入了"预训练+微调"的新纪元。

我自己在2019年第一次用BERT做文本分类时，感触最深的是：原来一个好的预训练模型可以让你"躺赢"。用随机初始化的LSTM做分类，可能需要调一周的超参数才能到90%；换成BERT，直接跑3个epoch就能到94%。当时我就在想——这背后一定有深刻的设计逻辑。今天我们就来挖一挖这些逻辑。

4.1 NSP：下一句预测任务

除了MLM，BERT还设计了另一个预训练任务——Next Sentence Prediction（NSP）。NSP的动机很直观：很多下游任务（如QA、自然语言推理）需要模型理解两个句子之间的关系。MLM只关注词级别的理解，不足以建模句间关系。

4.1.1 数据构造

NSP从语料中构造二分类样本：

  • 正例（50%）： 从文档中选取连续的两个句子 A+B
  • 负例（50%）： 从不同文档中随机选取句子 A 和 B（或同一文档的非连续句）
# NSP训练样本构造示意
# 正例：来自同一文档的连续句子
Sentence A: "今天天气真不错。"
Sentence B: "我们去公园散步吧。"
Label: IsNext (1)

# 负例：来自不同文档
Sentence A: "今天天气真不错。"
Sentence B: "导数描述了函数在某一点的变化率。"
Label: NotNext (0)

NSP的任务就是让[CLS]位置的输出向量做二分类，判断B句是否为A句的下一句。

后来被证伪： 2019年RoBERTa论文通过实验表明，去掉NSP后模型在GLUE上的性能几乎没有下降，甚至略有提升。后续研究发现NSP任务设计存在缺陷——负样本来自不同文档，模型很可能只是"偷懒"地通过判断"主题是否一致"来完成分类，而不需要真正理解句子间的逻辑关系。这个发现直接推动了ALBERT用SOP替代NSP（后面章节详谈）。

4.2 统一微调范式

BERT最大的贡献之一，是证明了同一个预训练模型可以适配不同类型的下游任务，只需在输入输出层面做调整：

任务类型 输入构造 输出 新增参数
文本分类（单句） [CLS] 句子 [SEP] [CLS] → 线性分类器 分类权重 K×H
文本分类（句对） [CLS] A [SEP] B [SEP] [CLS] → 线性分类器 分类权重 K×H
序列标注（NER） [CLS] 句子 [SEP] 每个位置 → softmax 标签权重 K×H
抽取式QA [CLS] 问题 [SEP] 文章 [SEP] Start logits + End logits 开始/结束向量 2×H
语义相似度（STS） [CLS] A [SEP] B [SEP] [CLS] → 回归输出 回归权重 1×H 
# 抽取式QA的输出层（论文附录A.4）
# 对于段落中的每个位置i，计算其作为答案开始/结束的概率
Start_logits = S^T @ T_i     # S是可学习的开始向量，T_i是第i个token的BERT输出
End_logits = E^T @ T_i       # E是可学习的结束向量

# Softmax得到概率分布
Start_probs = softmax(Start_logits)
End_probs = softmax(End_logits)

# 预测的答案区间: (argmax_start, argmax_end)
pred_start = argmax(Start_probs)
pred_end = argmax(End_probs)
answer_tokens = passage_tokens[pred_start:pred_end+1]

统一性的力量： 在BERT之前，不同NLP任务使用根本不同的模型架构（序列标注用BiLSTM-CRF、分类用CNN/TextCNN、QA用带有指针网络的模型）。BERT证明了输入格式的变换已经足够让同一个Transformer模型处理所有任务——这是架构统一性的里程碑。

4.3 GLUE基准：11项任务全面SOTA

论文在GLUE（General Language Understanding Evaluation）基准的11项任务上进行了评测。结果堪称"碾压级"：

任务 类型 指标 之前最佳 BERT-Base BERT-Large
MNLI 自然语言推理（句对） 准确率 80.6% 84.5% 86.7%
QQP 语义相似度（句对） F1 66.1% 71.2% 72.1%
QNLI 问答推断（句对） 准确率 82.3% 88.4% 92.2%
SST-2 情感分类（单句） 准确率 94.8% 92.1% 95.5%
MRPC 释义检测（句对） F1 84.4% 84.8% 89.5%
STS-B 语义相似度（回归） Spearman 80.0% 85.4% 86.5%
CoLA 语法可接受性（单句） Matthews 45.2% 49.8% 60.6%
SQuAD v1.1 抽取式QA F1 84.9% 88.5% 92.2%
SQuAD v2.0 抽取式QA（含不可回答） F1 66.3% 71.4% 81.8%
SWAG 常识推理 准确率 60.4% 74.6% 81.3%

特别值得关注的是SQuAD v2.0——它增加了"问题不可回答"的样本，需要模型判断段落中是否包含答案。BERT-Large达到81.8% F1，比之前最好的方法高出15.5个百分点。这意味着BERT真正学会了理解答案与问题的语义关系，而不是仅仅做表面匹配。

4.4 消融实验

论文进行了多项消融实验来验证设计选择：

实验1：去掉NSP

在QNLI（句对任务）上，去掉NSP后准确率从88.4%降至85.8%。但在单句任务（如SST-2）上变化不大。这符合直觉——NSP主要贡献句间关系理解。

实验2：去掉双向（使用单向LM）

如果使用GPT式的从左到右单向注意力，即使在相同数据量下，所有任务性能都会大幅下降（MNLI降低约5.6个百分点）。这直接证明了双向预训练是BERT核心优势。

实验3：使用LSTM替代Transformer

即使保留双向条件，使用双向LSTM也比BERT差很多，说明Transformer的并行化和长距离建模能力不可替代。

学到一个教训： 好的消融实验不只是"证明我的方法有用"，更是告诉社区为什么有用。BERT的三个消融实验分别验证了：NSP→句间关系、双向注意力→上下文建模、Transformer→长距离依赖。这种清晰的实验设计是论文成为经典的原因之一。

4.5 mBERT：跨语言的BERT

BERT论文还发布了多语言版本mBERT（Multilingual BERT），共享词表覆盖104种语言。关键设计：

  • 共享一个WordPiece词表（约110k subwords）
  • 每一种语言的训练文本按语言名称标注（如 [unused0] 代表中文）
  • 在104种语言的Wikipedia上联合预训练
  • 不删除任何语言——即使语料很少的语言也包含在训练中

最令人惊讶的结果是零样本跨语言迁移的可行性：在一个语言上微调（如英文），直接在其他语言上推理（如中文）——不需要任何目标语言的标注数据！比如在英文SQuAD上微调后，直接在中文数据集上预测答案，mBERT能够跨语言泛化。这个发现催生了后续大量跨语言NLP研究。

跨语言迁移的直觉解释： 不同语言的token在共享嵌入空间中形成了"对齐"——中文的"猫"和英文的"cat"在语义空间中距离很近（即使它们在训练中从未作为翻译对出现），因为它们的上下文分布相似（都跟在"pet/furry/动物"等上下文中）。这就是分布语义假设在跨语言层面的体现。

4.6 本章小结

到此，BERT论文精读就全部完成了。下半部分的核心内容回顾：

  • NSP通过50%/50%的正负样本构造句间关系理解（后被RoBERTa质疑有效性）
  • 统一微调范式让所有任务共享同一个架构，只需改变输入输出
  • GLUE/ SQuAD/ SWAG全面SOTA，验证了双向预训练的巨大优势
  • 消融实验清晰揭示了每个设计选择的价值
  • mBERT打开了跨语言NLP的大门

读完了论文，下一章我们将深入到工程层面，看看BERT的输入表示是如何构造的——WordPiece分词、三层嵌入叠加、特殊Token的含义，以及中文BERT分词的特殊处理。从理论到工程，我们一步步落地。
