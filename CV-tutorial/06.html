主流框架概览 — PyTorch vs TensorFlow vs JAX

在前三章中，我们用纯NumPy手动实现了卷积、反向传播、梯度下降——这在理解原理上非常有用，但如果你在实际项目中这样做，你的同事一定会觉得你疯了。手动实现每一层的梯度计算不仅效率低下，而且极易出错——漏掉一个负号、弄错一个维度，调试起来堪比大海捞针。

深度学习框架（Deep Learning Framework）就是为了解决这个问题而生的。它提供了三个核心能力：自动求导（你只需要定义前向传播，梯度自动计算）、GPU加速（张量操作自动运行在GPU上）、模块化组件（卷积、池化、损失函数、优化器都是现成的）。这三个能力让你从繁琐的底层实现中解放出来，专注于架构设计和实验探索——这才是深度学习中真正有价值的部分。

目前主流的深度学习框架有三个：

PyTorch（由Meta/Facebook开发）是目前CV和NLP研究领域的绝对主流。它的设计哲学是"直观和灵活"——用Python原生的控制流结构定义网络，支持动态计算图（即每次前向传播时计算图都重新构建），调试体验极佳。PyTorch的生态系统也非常丰富：torchvision提供了大量CV预训练模型和数据集，HuggingFace的Transformers库深度集成PyTorch。如果你在2024年从头开始学深度学习，PyTorch几乎是不二之选。

TensorFlow（由Google开发）曾是2016-2019年间最流行的框架，拥有最多的生产部署案例。TensorFlow 2.x引入了Eager Execution（动态执行），大幅改善了易用性。Keras作为TF的高级API，简化了模型构建流程。TensorFlow Serving和TensorFlow Lite在模型部署方面有独特的优势。但在学术研究领域，PyTorch已经后来居上——2024年NeurIPS论文中PyTorch的使用率超过90%。

JAX（由Google Research开发）是最新的玩家。它的设计理念与前两者截然不同：JAX不是传统意义上的深度学习框架，而是一个"可微分编程"库。它提供函数式变换（grad、jit、vmap、pmap）来对纯Python函数做求导、JIT编译、向量化和并行化。JAX在科学计算和某些前沿研究方向（如基于强化学习的机器学习）中很受欢迎，但目前学习曲线较陡、社区生态不如PyTorch成熟。

💡 框架选择建议：本课程全程使用PyTorch。如果你是初学者，直接从PyTorch开始就好——它让你可以把精力集中在CV概念本身，而不是和框架作斗争。如果你已经在做生产部署，可以考虑用TensorFlow Serving——但这属于部署范畴，不影响学习。JAX适合有扎实函数式编程基础、追求极致性能的进阶用户。

张量操作基础 — Tensor创建、索引、形状变换、设备管理

PyTorch中的Tensor（张量）是NumPy ndarray的"GPU加强版"。它在数据结构上几乎和NumPy数组一样——支持多维数组、切片、广播——但多了两个关键能力：在GPU上运行和参与自动求导计算图。

创建Tensor的方式非常多样：从列表或NumPy数组创建、使用工厂函数（zeros、ones、randn、arange）、或从已有的tensor派生。数据的dtype（默认float32）和device（CPU vs GPU）是每个tensor的基本属性。

索引和切片的语法和NumPy完全一致——用方括号和冒号操作。你可以轻松取出某个通道、某个区域或某个batch的样本。

形状变换是日常使用最频繁的操作之一。view()改变tensor的形状而不改变内存布局（类似于NumPy的reshape），permute()重新排列维度顺序，unsqueeze()和squeeze()增加或删除长度为1的维度。在用CNN处理图像时，你经常需要在(B, C, H, W)和(B, H, W, C)之间转换——permute就是做这个的。

设备管理是另一个频繁出现的需求。PyTorch让你通过.to(device)在CPU和GPU之间移动tensor和模型。一个常见的模式是：在代码开头检查CUDA是否可用，设置device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')，然后用.to(device)将tensor和模型移到该设备上。

# PyTorch 张量操作入门
import torch

# 创建 Tensor 的多种方式
a = torch.tensor([[1, 2], [3, 4]])         # 从列表
b = torch.zeros(3, 4)                       # 全零矩阵
c = torch.randn(2, 3, 224, 224)             # 随机初始化 (模拟 batch 图像)
d = torch.ones(3, 3, dtype=torch.float32)   # 指定数据类型

print(f"a = {a}")
print(f"b 形状: {b.shape}")
print(f"c 形状: {c.shape}  (batch, channels, height, width)")

# 索引和切片
print(f"a 第一行: {a[0]}")
print(f"a 第一列: {a[:, 0]}")

# 形状变换
x = torch.randn(4, 3, 64, 64)   # (B, C, H, W)
print(f"\n原始形状: {x.shape}")
print(f"view 后:   {x.view(4, -1).shape}")     # 展平为 (4, 12288)
print(f"permute 后:{x.permute(0, 2, 3, 1).shape}")  # (B, H, W, C)

# 设备管理
if torch.cuda.is_available():
    device = torch.device('cuda')
    x_gpu = x.to(device)
    print(f"\nTensor 已移至 GPU: {x_gpu.device}")
else:
    print("\n无 GPU 可用, 当前使用 CPU")

自动求导机制 — autograd、计算图、梯度累积

在第4章中，我们手动实现了反向传播——你看，即使是个简单的两层网络，代码也已经相当繁琐。PyTorch的autograd模块彻底解放了你：只要将tensor的requires_grad=True，PyTorch就会自动构建计算图并计算梯度。

autograd的工作原理可以概括为：在前向传播过程中，PyTorch自动构建一个"计算图"（Computational Graph），这个图记录了从输入到输出的所有操作。当调用.backward()时，autograd从输出节点开始，沿着计算图反向传播梯度。

计算图默认是动态的——每次调用.backward()后图就被释放（除非设置retain_graph=True）。这也意味着梯度默认是累积的——每次.backward()的梯度会累加到.grad属性上，而不是替换。所以在每次优化步骤之前，需要手动清零梯度（optimizer.zero_grad()）。

在推理（inference）阶段，你应该使用torch.no_grad()上下文管理器来禁用自动求导——这不仅能节省大量内存（不需要构建计算图），还能加速计算。这是一个非常容易被初学者忽视的最佳实践。

# 自动求导示例
import torch

# 创建需要梯度的 tensor
x = torch.tensor([2.0, 3.0], requires_grad=True)
w = torch.tensor([1.0, -1.0], requires_grad=True)
b = torch.tensor(0.5, requires_grad=True)

# 前向计算: y = w1*x1 + w2*x2 + b
y = torch.dot(x, w) + b
print(f"前向输出: y = {y:.2f}")

# 反向传播: 计算 dy/dw1, dy/dw2, dy/db
y.backward()

print(f"\n梯度:")
print(f"  dy/dw1 = {w.grad[0]:.2f}  (等于 x1 = {x[0]})")
print(f"  dy/dw2 = {w.grad[1]:.2f}  (等于 x2 = {x[1]})")
print(f"  dy/db  = {b.grad[0]:.2f}  (等于 1)")

# 验证: y = w1*x1 + w2*x2 + b
# dy/dw1 = x1 = 2.0 ✓
# dy/db  = 1.0   ✓

# 推理时禁用自动求导
with torch.no_grad():
    y_infer = torch.dot(x, w) + b
    print(f"\n推理模式输出: {y_infer:.2f} (不追踪梯度)")

# 注意: 梯度是累加的!
for _ in range(3):
    y = torch.dot(x, w) + b
    y.backward()
print(f"\n不 reset 梯度, 调用 3 次 backward 后 dw1 = {w.grad[0]:.2f}")
print(f"(应该是原始梯度 {x[0]:.2f} 的 3 倍 = {x[0]*3:.2f})")
⚠️ 经典失误：忘记 optimizer.zero_grad() 是PyTorch初学者的头号bug。如果不清零，梯度会不断累积——一开始你可能发现loss下降很快（因为梯度方向被"平均"了），但越往后越不对劲（梯度震荡、不收敛）。我自己就因为这个bug浪费了整整两天，一直在怀疑模型结构有问题，直到打印出每一步的梯度才发现问题所在。解决办法很简单：在每步训练中确保 optimizer.zero_grad() 被调用。

构建神经网络 — nn.Module、nn.Sequential、自定义层

PyTorch将神经网络抽象为nn.Module——这是所有网络组件的基类。你的自定义网络只需要继承nn.Module，在__init__中定义各层，在forward中定义前向传播逻辑。

nn.Module最强大的特性是参数自动管理：当你把子模块（如nn.Conv2d、nn.Linear）赋值给self的属性时，PyTorch会自动跟踪它们的参数——你可以通过model.parameters()获取网络中所有可训练的参数，直接传给优化器。这意味着你不再需要像我们在第4章中那样手动维护一个参数字典。

nn.Sequential是一个容器，用于按顺序执行多个层。对于"A→B→C→D"这种简单的直线结构，nn.Sequential让你用最少的代码定义网络。但更复杂的结构（多分支、跳跃连接）需要在forward方法中手动控制。

自定义层也很简单：继承nn.Module，实现__init__和forward。如果自定义层包含可学习参数，使用nn.Parameter包装——这告诉PyTorch"这是我要训练的参数"。

# 用 PyTorch 构建 CNN
import torch
import torch.nn as nn

# 方式一: 使用 nn.Sequential (适合简单直连结构)
simple_cnn = nn.Sequential(
    nn.Conv2d(3, 16, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.Conv2d(16, 32, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.AdaptiveAvgPool2d((1, 1)),  # 全局平均池化
    nn.Flatten(),
    nn.Linear(32, 10)
)

print("Simple CNN:")
print(f"  参数量: {sum(p.numel() for p in simple_cnn.parameters()):,}")

# 方式二: 继承 nn.Module (适合复杂结构)
class CustomCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(),
            nn.MaxPool2d(2),
        )
        self.classifier = nn.Sequential(
            nn.AdaptiveAvgPool2d((1, 1)),
            nn.Flatten(),
            nn.Linear(64, 128),
            nn.ReLU(),
            nn.Dropout(0.5),
            nn.Linear(128, num_classes)
        )

    def forward(self, x):
        x = self.features(x)
        x = self.classifier(x)
        return x

model = CustomCNN(num_classes=10)
print(f"\nCustomCNN 参数量: {sum(p.numel() for p in model.parameters()):,}")

# 测试前向传播
dummy_input = torch.randn(4, 3, 32, 32)  # batch=4, RGB, 32×32
output = model(dummy_input)
print(f"输出形状: {output.shape}  (batch, num_classes)")

数据加载与预处理 — DataLoader、Dataset、transforms

在真正的训练中，数据加载是一个极其重要的环节——如果数据加载速度跟不上GPU的计算速度，GPU就会"挨饿"，浪费宝贵的计算资源。PyTorch的torch.utils.data模块提供了高效的解决方案。

Dataset是一个抽象类，你需要继承它并实现两个方法：__len__（返回数据集大小）和__getitem__（根据索引返回一个样本及其标签）。PyTorch已经为常见的数据集（如MNIST、CIFAR、ImageNet）提供了现成的实现，但当你处理自定义数据时，就需要自己写Dataset。

DataLoader在Dataset之上提供批量加载、自动打乱、多进程预读取等功能。关键参数包括：batch_size（每批样本数）、shuffle（是否打乱）、num_workers（用于数据加载的子进程数——设置正确可以大幅提升数据加载速度）。

transforms是数据预处理和增强的工具集。torchvision.transforms提供了丰富的图像变换：Resize（缩放）、ToTensor（将PIL图像或NumPy数组转为Tensor）、Normalize（按均值和标准差归一化）、RandomHorizontalFlip（随机水平翻转——一种数据增强方法）等。通常使用transforms.Compose将多个变换组合成pipeline。

# 数据加载完整示例 (使用 torchvision 的 CIFAR-10)
import torch
import torchvision
import torchvision.transforms as transforms

# 定义数据预处理
transform_train = transforms.Compose([
    transforms.RandomHorizontalFlip(),          # 数据增强: 随机翻转
    transforms.RandomCrop(32, padding=4),        # 数据增强: 随机裁剪
    transforms.ToTensor(),                       # PIL → Tensor
    transforms.Normalize(                        # 归一化到 N(0, 1)
        mean=[0.4914, 0.4822, 0.4465],
        std=[0.2470, 0.2435, 0.2616]
    ),
])

transform_test = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize(
        mean=[0.4914, 0.4822, 0.4465],
        std=[0.2470, 0.2435, 0.2616]
    ),
])

# 下载并加载 CIFAR-10 数据集
trainset = torchvision.datasets.CIFAR10(
    root='./data', train=True, download=True, transform=transform_train
)
testset = torchvision.datasets.CIFAR10(
    root='./data', train=False, download=True, transform=transform_test
)

# DataLoader: 批量、打乱、多进程
trainloader = torch.utils.data.DataLoader(
    trainset, batch_size=128, shuffle=True, num_workers=4
)
testloader = torch.utils.data.DataLoader(
    testset, batch_size=128, shuffle=False, num_workers=4
)

print(f"训练集大小: {len(trainset)}")
print(f"测试集大小: {len(testset)}")
print(f"每个 epoch 的 batch 数: {len(trainloader)}")

# 获取一个 batch 检查
images, labels = next(iter(trainloader))
print(f"\n一个 batch 的图像形状: {images.shape}")
print(f"一个 batch 的标签形状: {labels.shape}")
print(f"标签范围: {labels.min().item()} ~ {labels.max().item()}")

完整训练流程 — 模型定义→损失→优化→训练循环

现在我们把之前学到的所有PyTorch组件组合成一个完整的训练脚本。这是一个最小可行训练循环——包含了训练深度学习模型的所有核心步骤。理解这个循环的结构后，你在后续章节中只需要替换模型、数据、损失函数等组件即可。

# 完整训练流程 (PyTorch)
import torch
import torch.nn as nn
import torch.optim as optim

def train_one_epoch(model, trainloader, criterion, optimizer, device):
    """训练一个 epoch"""
    model.train()  # 设置为训练模式 (启用 Dropout/BatchNorm 训练行为)
    running_loss = 0.0
    correct = 0
    total = 0

    for inputs, labels in trainloader:
        inputs, labels = inputs.to(device), labels.to(device)

        # 1. 清零梯度 (非常重要!)
        optimizer.zero_grad()

        # 2. 前向传播
        outputs = model(inputs)
        loss = criterion(outputs, labels)

        # 3. 反向传播
        loss.backward()

        # 4. 参数更新
        optimizer.step()

        # 统计
        running_loss += loss.item()
        _, predicted = outputs.max(1)
        total += labels.size(0)
        correct += predicted.eq(labels).sum().item()

    epoch_loss = running_loss / len(trainloader)
    epoch_acc = 100. * correct / total
    return epoch_loss, epoch_acc

def evaluate(model, testloader, criterion, device):
    """评估模型"""
    model.eval()  # 设置为评估模式 (禁用 Dropout)
    running_loss = 0.0
    correct = 0
    total = 0

    with torch.no_grad():  # 推理模式: 不追踪梯度
        for inputs, labels in testloader:
            inputs, labels = inputs.to(device), labels.to(device)
            outputs = model(inputs)
            loss = criterion(outputs, labels)

            running_loss += loss.item()
            _, predicted = outputs.max(1)
            total += labels.size(0)
            correct += predicted.eq(labels).sum().item()

    epoch_loss = running_loss / len(testloader)
    epoch_acc = 100. * correct / total
    return epoch_loss, epoch_acc

# ========= 主程序 =========
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
print(f"使用设备: {device}")

# 1. 定义模型
model = CustomCNN(num_classes=10).to(device)  # 前面定义的 CustomCNN
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-3)
scheduler = optim.lr_scheduler.StepLR(optimizer, step_size=10, gamma=0.1)

# 2. 训练循环
num_epochs = 20
for epoch in range(1, num_epochs + 1):
    train_loss, train_acc = train_one_epoch(
        model, trainloader, criterion, optimizer, device
    )
    test_loss, test_acc = evaluate(model, testloader, criterion, device)
    scheduler.step()  # 调整学习率

    if epoch % 5 == 0 or epoch == 1:
        print(f"Epoch {epoch:2d} | "
              f"Train Loss: {train_loss:.4f} | Train Acc: {train_acc:.2f}% | "
              f"Test Acc: {test_acc:.2f}%")

print("\n训练完成!")
print(f"最终测试准确率: {test_acc:.2f}%")

# 3. 保存模型
# torch.save(model.state_dict(), 'cifar10_cnn.pth')
print("模型权重已保存到 cifar10_cnn.pth")
🛠️ 完整训练流程总结：深度学习框架的核心价值就是将下面这个循环标准化了——定义模型、加载数据、前向传播、计算损失、反向传播、更新参数、评估、保存。一旦你理解了PyTorch中的这个循环结构，后续所有CV模型的训练都只是在这个框架中替换不同的组件（模型、数据、损失函数、优化器）而已。这就是"框架"的意义——它提供了一个可靠的"支架"，让你可以专注于真正有创造性的工作：设计更好的模型、选择更好的训练策略。
