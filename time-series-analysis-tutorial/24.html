第24章 卷积与时序

大家好！上一章我们学习了RNN和LSTM，它们的设计天然适合序列数据。但RNN有一个根本性的局限：它是串行计算的——当前时间步必须等待前一步的计算结果，这使得训练速度慢、难以并行化。那么，有没有一种方法既能捕捉局部时间模式，又能充分利用GPU的并行计算能力呢？答案是肯定的——用卷积神经网络（Convolutional Neural Network, CNN）来处理时间序列。

你可能会问："卷积不是用来做图像识别的吗？"没错，CNN确实在计算机视觉领域大放异彩，但它的核心操作——卷积——本质上就是一个"模式匹配器"。在图像上，它匹配的是空间模式（边缘、纹理）；在时间序列上，它匹配的是时间模式（上升趋势、周期性波动、尖峰信号）。而且CNN可以并行计算，训练速度远快于RNN。

24.1 一维卷积：时序数据的模式匹配器

标准的CNN使用二维卷积核在图像上滑动，提取空间特征。对于时间序列，我们使用一维卷积核（1D Convolution）沿着时间轴滑动，提取局部时间模式。

一维卷积的工作原理非常直观：一个小窗口（比如长度为3或5的卷积核）在时间序列上滑动，每个位置计算窗口内数值与卷积核权重的点积。不同的卷积核可以检测不同的模式——有的检测上升趋势，有的检测尖峰，有的检测周期性波动。

1D CNN的关键参数：

  • Kernel Size：卷积核的长度，决定一次能看到多少个时间步。一般设为3-9。
  • Stride：滑动的步长。步长越大，输出序列越短。
  • Padding：在序列两端补零，控制输出长度。Same padding 保持长度不变。
  • Dilation：膨胀率，控制卷积核之间的间隔——这个参数在TCN中至关重要。
直观理解：可以把1D卷积理解为一系列"模板匹配器"。每个卷积核学习一种特定的局部时间模式（如"连续三天的上涨"或"突然的尖峰"），然后在整个序列上扫描，标记哪些位置出现了这种模式。

24.2 TCN：膨胀卷积与感受野

普通1D CNN有一个明显的局限：感受野（Receptive Field）太小。一个 kernel_size=3 的卷积核一次只能看到3个时间步。要看到更远的依赖，只能堆叠很多层，但这会增加参数量和计算量。

时序卷积网络（Temporal Convolutional Network, TCN）由 Bai 等人在2018年提出，它通过两个关键设计解决了这个问题：

膨胀卷积（Dilated Convolution）：卷积核的每个元素之间插入间隔，让卷积核在不增加参数的情况下"看到"更远的范围。例如，dilation=2 时，一个 kernel_size=3 的卷积核实际上覆盖了5个时间步；dilation=4 时覆盖9个时间步。通过指数级增大膨胀率（1, 2, 4, 8, ...），TCN可以用很少的层数获得极大的感受野。

残差连接（Residual Connection）：每个卷积块包含一个跳跃连接，让梯度直接流过，避免了深层网络的退化问题。这在时序预测中非常有效。

TCN的优势在于：它结合了CNN的并行计算能力和极大的感受野，在很多序列建模任务上表现超过了同等规模的LSTM。

24.3 WaveNet：从语音合成到时序预测

WaveNet 是由 DeepMind 在2016年提出的音频生成模型，最初用于合成逼真的人类语音。它使用了一种特殊的因果卷积（Causal Convolution），确保模型在预测当前时间步时不会"看到"未来的数据——这对时间序列预测来说是天生的优势，因为预测就必须遵守时间顺序。

因果卷积的原理很简单：在时间 t 的输出只依赖于 t 及之前的数据，绝不依赖 t+1 之后的数据。普通卷积在 padding 时可能会引入未来信息，但因果卷积通过特殊的 padding 方式严格保证了时间上的因果性。

这里有一个有趣的故事：WaveNet 的研发团队最初的目标是改善文本到语音（TTS）的合成质量，他们用大量的人类语音数据训练了一个巨大的膨胀因果卷积网络，生成的语音质量达到了当时的最优水平。但很快，研究者和工程师们发现，WaveNet 的架构本质上就是一个时序模型——因果卷积 + 膨胀卷积的组合对于任何序列数据都有效。于是，WaveNet 被迅速应用到音乐生成、语音识别，以及——时间序列预测。这正是一个"跨领域迁移"的经典案例：为一个领域（语音合成）设计的架构，在另一个看似无关的领域（时序预测）中同样大放异彩。

启发：时序预测领域有很多优秀的思想来自其他领域——WaveNet来自语音合成，Transformer来自机器翻译，Informer来自NLP的self-attention。保持广阔的视野，往往能发现意想不到的解决方案。

24.4 CNN-LSTM混合架构

既然CNN擅长提取局部模式（像"特征提取器"），而LSTM擅长捕捉长期依赖（像"序列记忆器"），那为什么不把两者结合起来呢？这就是CNN-LSTM混合架构的基本思想。

典型的CNN-LSTM架构包含两个阶段：首先，1D CNN在输入序列上滑动，提取局部特征（比如每天的模式、每周的形态），生成一组特征序列；然后，LSTM在这组特征序列上建模长期依赖关系。这种"先局部后全局"的设计在多个数据集上都取得了优于单一模型的效果。

CNN-LSTM的变体：

  • 并行CNN-LSTM：CNN和LSTM并行处理输入，然后融合输出。适合需要同时捕捉局部和全局模式的任务。
  • ConvLSTM：将LSTM内部的矩阵乘法替换为卷积操作，适合时空序列预测（如天气预报中的雷达回波外推）。

24.5 RNN vs CNN：如何选择？

面对时间序列任务，到底是选RNN还是CNN（尤其是TCN）？下面这个对比表可以帮助你做决定：

维度 RNN / LSTM CNN / TCN
并行计算 串行，不能并行 完全并行，训练快
感受野 理论上无限（通过隐藏状态） 受层数和膨胀率限制，但可灵活调节
梯度稳定性 有消失/爆炸风险（需门控机制） 残差连接天然缓解梯度问题
长程依赖 LSTM/GRU擅长（门控机制） TCN通过膨胀卷积也能覆盖长程
参数量 较大（尤其是LSTM四个门） 较小（权重共享）
内存消耗 需要存所有时间步的中间状态 训练时更省内存
适用场景 变长序列、Seq2Seq、需要灵活记忆 定长窗口、大规模数据、追求速度
注意：没有绝对"更好"的模型。TCN在多个基准测试中超过了LSTM，但在需要灵活记忆（如序列长度变化很大、需要重置状态）的场景中，LSTM仍然有不可替代的优势。建议在具体任务上都尝试一下，用交叉验证来选择。

24.6 PyTorch实现：1D CNN时序预测

下面是一个简单的1D CNN时间序列预测模型，展示了卷积在时序数据上的基本用法：

import torch
import torch.nn as nn

class CNN1D(nn.Module):
    def __init__(self, input_channels=1):
        super().__init__()
        self.conv1 = nn.Conv1d(
            input_channels, 32, kernel_size=3,
            padding=1
        )
        self.conv2 = nn.Conv1d(
            32, 64, kernel_size=3, padding=1
        )
        self.pool = nn.AdaptiveAvgPool1d(1)
        self.fc = nn.Linear(64, 1)

    def forward(self, x):
        # x: (batch, channels, seq_len)
        x = torch.relu(self.conv1(x))
        x = torch.relu(self.conv2(x))
        x = self.pool(x).squeeze(-1)  # (batch, 64)
        return self.fc(x)

# TCN残差块示意
class TCNBlock(nn.Module):
    def __init__(self, in_channels, out_channels,
                 kernel_size=3, dilation=1):
        super().__init__()
        padding = (kernel_size - 1) * dilation
        self.conv = nn.Conv1d(
            in_channels, out_channels, kernel_size,
            padding=padding, dilation=dilation
        )
        self.relu = nn.ReLU()
        self.residual = nn.Conv1d(
            in_channels, out_channels, 1
        ) if in_channels != out_channels else nn.Identity()

    def forward(self, x):
        out = self.relu(self.conv(x))
        # 因果卷积需要裁剪掉未来信息
        out = out[:, :, :-self.conv.padding[0]]
        return self.relu(out + self.residual(x))

24.7 小结

本章我们学习了CNN在时间序列中的应用。一维卷积作为局部模式匹配器，天然适合提取时序中的短程特征。TCN通过膨胀卷积和残差连接，在保持CNN并行计算优势的同时，大幅扩展了感受野。WaveNet的故事告诉我们，跨领域的思想迁移往往能带来惊喜。而CNN-LSTM混合架构则展示了"取长补短"的智慧——让CNN处理局部模式，LSTM处理长期依赖。

无论是RNN还是CNN，它们都通过"循环"或"滑动"的方式处理序列。但还有一种完全不同的思路——让序列中的每个元素互相"看"对方一眼，直接建模所有位置的关系。这就是下一章的主角：Transformer。
