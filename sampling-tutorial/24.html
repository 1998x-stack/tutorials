第24章 Python实现

大家好，我是你们的讲师。上一章我们看了 R 语言如何实现抽样分析，这一章我们来聊 Python。如果你问我到底学 R 还是学 Python？我的回答是：都学。R 在调查数据分析上有无可比拟的生态优势，但 Python 在通用数据处理和机器学习方面更强。在实际工作中，你很可能需要在两个工具之间来回切换。

坦白说，Python 在复杂调查数据分析方面没有像 R 的 survey 包那样成熟的一站式解决方案。但这不意味着 Python 做不了——只是需要多动动手。本章我会教你如何用 Python 的标准库和第三方库来实现各种抽样方法，并且正确地进行加权估计和方差估计。

Python的抽样生态

Python 没有像 R 的 survey 包那样的大一统包，但通过组合使用以下工具，完全可以实现同样的功能：

工具/库 功能 适用场景
random random.sample() / random.choices() 简单随机抽样
numpy.random np.random.choice() 带权重的随机抽样、数组操作
pandas df.sample()、groupby + sample 分层抽样、整群抽样、权重计算
scipy.stats 统计分布、假设检验 抽样分布、置信区间计算
sampling (PyPI) 各种概率抽样方法 专业的抽样操作（类似 R 的 sampling 包）

其中 pandas 是绝对核心。在 Python 中做抽样分析，90% 的工作量在 pandas 的数据操作上——分组、抽样、合并、计算权重，几乎全靠它。

小贴士：PyPI 上有一个 sampling 库（注意不是 scipy.sampling，是独立的 pip install sampling），它实现了许多 R 的 sampling 包中的方法，包括分层抽样、系统抽样、PPS 抽样等。不过社区活跃度不如 R 版本高，建议谨慎用于生产环境。

简单随机抽样（SRS）

Python 实现 SRS 最简单的方式是用 random.sample() 或 np.random.choice()：

import random
import numpy as np
import pandas as pd

# 构造总体
np.random.seed(42)
N = 10000
population = pd.DataFrame({
    'id': range(N),
    'income': np.random.normal(5000, 1500, N),
    'age': np.random.randint(18, 81, N)
})

# 方法1: random.sample
sample_indices = random.sample(range(N), 500)
srs_sample1 = population.iloc[sample_indices]

# 方法2: np.random.choice
sample_indices2 = np.random.choice(N, size=500, replace=False)
srs_sample2 = population.iloc[sample_indices2]

# 方法3: pandas 内置 sample
srs_sample3 = population.sample(n=500, random_state=42)

# 加权估计——手动计算
# SRS 权重 = N/n = 20
weight = N / 500
weighted_mean = (srs_sample3['income'] * weight).sum() / N
print(f"加权均值: {weighted_mean:.2f}")

# 更简单的做法：直接计算普通均值（SRS 中无偏）
simple_mean = srs_sample3['income'].mean()
print(f"简单均值: {simple_mean:.2f}")

SRS 是最简单的情况——由于每个个体被抽中的概率相等，权重均匀，所以加权均值和简单均值是相等的。但这只是特例，一旦涉及分层或不等概率，就必须正确使用权重。

分层抽样

分层抽样在 Python 中的实现思路是：先按分层变量分组，然后在每组内独立抽样：

# 构造分层变量
population['stratum'] = np.random.choice(
    ['东部', '中部', '西部'], N,
    p=[0.4, 0.3, 0.3]
)

# 比例分配：每层抽 5%
strat_sample = (
    population
    .groupby('stratum', group_keys=False)
    .apply(lambda x: x.sample(frac=0.05, random_state=42))
)

# 计算权重
# 每层的权重 = 层总体规模 / 层样本规模
stratum_counts = population['stratum'].value_counts()
stratum_sample_counts = strat_sample['stratum'].value_counts()

stratum_weights = stratum_counts / stratum_sample_counts
strat_sample['weight'] = strat_sample['stratum'].map(stratum_weights)

# 分层加权估计
weighted_mean = (
    strat_sample['income'] * strat_sample['weight']
).sum() / stratum_counts.sum()

# 也可以按层分别估计
strat_estimates = strat_sample.groupby('stratum').apply(
    lambda x: (x['income'] * x['weight']).sum() / stratum_counts[x.name]
)
print(strat_estimates)

注意这里我手动计算了权重。在分层比例分配下，每层的权重 W_h = N_h / n_h。如果是 Neyman 分配或最优分配，权重计算会更复杂一些，但思路是一样的：权重 = 总体单元数 / 样本单元数。

整群抽样

整群抽样的核心是"先选群，再展开"：

# 构造整群数据：50 个社区，每个社区 200 人
clusters = pd.DataFrame({
    'cluster_id': range(50),
    'cluster_size': 200
})

# 生成个体数据
individuals = []
for cid in range(50):
    for i in range(200):
        individuals.append({
            'cluster_id': cid,
            'person_id': i,
            'income': np.random.normal(5000 + cid * 20, 1500)
        })
population_cluster = pd.DataFrame(individuals)

# 第一阶段：随机抽取 10 个社区
selected_clusters = np.random.choice(50, size=10, replace=False)
cluster_sample = population_cluster[
    population_cluster['cluster_id'].isin(selected_clusters)
]

# 整群抽样的权重
# 每个个体被抽中的概率 = (10/50) * 1 = 0.2
# 权重 = 1/0.2 = 5
cluster_weight = 50 / 10
cluster_mean = cluster_sample['income'].mean() * cluster_weight
# 注意：整群抽样的均值估计需要先计算群均值，再取平均
cluster_means = cluster_sample.groupby('cluster_id')['income'].mean()
final_estimate = cluster_means.mean()
print(f"整群抽样估计的总体均值: {final_estimate:.2f}")
关键区别：整群抽样的方差估计不能简单用个体层面的标准误。正确的做法是计算群均值之间的方差——因为抽样单元是"群"而不是个体。在代码中，cluster_means.std() / np.sqrt(len(cluster_means)) 才是正确的标准误。

方差估计：Bootstrap 方法

在 Python 中做复杂抽样的方差估计，最灵活的方法是 Bootstrap（自助法）。它不受估计量形式的限制——均值、中位数、分位数、甚至是机器学习模型的 AUC，都能用 Bootstrap 估计方差：

def bootstrap_variance(data, weights, estimator_func, n_reps=500):
    """
    Bootstrap 方差估计

    Parameters:
    - data: DataFrame, 原始样本数据
    - weights: 样本权重
    - estimator_func: 估计量函数
    - n_reps: 重复次数
    """
    n = len(data)
    estimates = []

    for _ in range(n_reps):
        # 从原始样本中做有放回重抽样
        boot_indices = np.random.choice(n, size=n, replace=True)
        boot_data = data.iloc[boot_indices]
        boot_weights = weights.iloc[boot_indices]

        # 计算估计量
        est = estimator_func(boot_data, boot_weights)
        estimates.append(est)

    # Bootstrap 方差 = 重复估计量的样本方差
    return np.var(estimates, ddof=1)

# 示例：估计加权均值
def weighted_mean_est(data, weights):
    return (data['income'] * weights).sum() / weights.sum()

# 计算 Bootstrap 方差
boot_var = bootstrap_variance(
    strat_sample,
    strat_sample['weight'],
    weighted_mean_est
)
print(f"Bootstrap 方差: {boot_var:.4f}")
print(f"标准误: {np.sqrt(boot_var):.2f}")

Bootstrap 的好处是普适性强。但注意：对于复杂抽样设计，简单的 Bootstrap（从样本中做 i.i.d. 重抽样）会低估方差。更严谨的做法是使用分层 Bootstrap（按层分别重抽样）或整群 Bootstrap（以 PSU 为单位重抽样）。

完整示例：PPS 抽样与加权分析

我们来做一个更完整的例子——PPS（概率与规模成比例）抽样，这是多阶段调查中最常用的方法：

# 构造 PPS 抽样场景：先抽城市，再抽居民
cities = pd.DataFrame({
    'city_id': range(20),
    'population': np.random.randint(50000, 500000, 20)
})

# PPS 抽样：选择概率与城市人口规模成正比
total_pop = cities['population'].sum()
cities['prob'] = cities['population'] / total_pop
cities['cum_prob'] = cities['prob'].cumsum()

# 用系统抽样法实现 PPS 抽取 5 个城市
def pps_sample(df, n, size_col):
    """PPS 抽样"""
    df = df.copy()
    df['cum'] = df[size_col].cumsum()
    total = df[size_col].sum()
    step = total / n
    start = np.random.uniform(0, step)
    picks = start + np.arange(n) * step
    picks_idx = np.searchsorted(df['cum'], picks)
    return df.iloc[picks_idx]

selected_cities = pps_sample(cities, 5, 'population')

# 从每个选中的城市抽取居民
city_samples = []
for cid in selected_cities['city_id']:
    city_pop = cities[cities['city_id'] == cid]['population'].values[0]
    n_residents = 100  # 每个城市固定抽 100 人
    residents = pd.DataFrame({
        'city_id': cid,
        'income': np.random.normal(5000, 1500, n_residents)
    })
    # PPS 权重 = 1/(选中概率 * 第二阶段抽样概率)
    prob_city = city_pop / total_pop  # 5 个城市，所以实际是 5 * (city_pop/total_pop)，简化处理
    prob_resident = n_residents / city_pop
    residents['weight'] = 1 / (prob_city * prob_resident)
    city_samples.append(residents)

final = pd.concat(city_samples, ignore_index=True)
weighted_avg = (final['income'] * final['weight']).sum() / final['weight'].sum()
print(f"PPS 加权均值: {weighted_avg:.2f}")

真实踩坑故事：数据科学竞赛中的权重陷阱

几年前，某知名数据科学平台举办了一场关于"全国居民健康指数预测"的竞赛。主办方提供了一份包含 20 万条记录的调查数据，其中包含了年龄、收入、生活习惯等特征，目标是预测"健康风险评分"。

参赛者们的操作手法几乎一致：下载数据 → 划分训练集/测试集 → 特征工程 → XGBoost/LightGBM → 提交。线下交叉验证分数很高，大家都在庆祝自己轻松达到 0.92 的 AUC。

但线上分数公布时，几乎所有参赛者的分数都崩了——AUC 只有 0.65 左右。发生了什么？

问题出在权重上。这份数据来自复杂抽样设计——欠发达地区的居民被过度抽样（oversampled），发达地区则抽样不足。如果不使用权重来校正这种不等概率设计，模型就是在"拟合样本分布"，而不是"拟合总体分布"。线下 CV 之所以看起来很好，是因为 CV 也是在非加权数据上做的——自欺欺人而已。

教训：当你用调查数据训练机器学习模型时，必须考虑抽样权重。正确的做法是：在损失函数中引入权重，或者在训练过程中进行加权抽样（weighted sampling）。具体来说：

1. 对于 sklearn 模型，使用 sample_weight 参数
2. 对于 XGBoost，使用 weight 参数
3. 对于神经网络，可以在 DataLoader 中设置 WeightedRandomSampler

群里的那个竞赛后来有一位选手发现了这个秘密，做了加权处理之后，线上分数直接从 0.65 跃升到了 0.88。权重的重要性，由此可见一斑。
