23.1 让机器理解语言——NLP的前世今生

自然语言处理（NLP）是AI领域最激动人心的方向之一。从2013年Word2Vec让词向量成为可能，到2017年Transformer颠覆一切，再到2022年ChatGPT引爆全球——NLP在不到十年内经历了翻天覆地的变化。

NLP的终极挑战是：语言是离散的、组合的、歧义的、依赖上下文的。同一句话"我喜欢上了这门课"——在不同上下文中可以是"我喜欢/上了这门课"或"我喜欢上/了这门课"。人类靠常识消歧义，而机器只能靠统计模式。

23.2 第一步：把文字变成数字

模型吃的是数字，你给我的是文字。第一步必须是把文字变成向量。这个转化方式经历了三次重大变革：

方法原理问题
One-Hot / Bag of Words每个词是一个维度的0/1或计数维度过高（词表多大就多少维），无语义信息
Word2Vec / GloVe用周围词预测中心词，获得稠密低维向量一个词只有一个向量，无法处理多义词
Contextual Embedding一个词在不同上下文中获得不同向量需要预训练大模型，推理成本高

Word2Vec：词向量的"Hello World"

核心思想：一个词的含义由它周围的词决定（"You shall know a word by the company it keeps"，Firth, 1957）。两种训练方式：

  • CBOW：用上下文预测中心词（"我___机器学习" → "爱"）
  • Skip-gram：用中心词预测上下文（"爱" → "我", "机器学习"）

Word2Vec训练完后，你会发现：king - man + woman ≈ queen。这就是词向量最经典的语义算术——向量空间中的方向编码了语义关系。

23.3 Transformer：NLP的"工业革命"

RNN/LSTM的问题在第22章已经铺垫过：无法并行、长程依赖困难。Transformer用自注意力（Self-Attention）完全解决了这两个问题。

自注意力的直觉

对于输入序列中的每个词，Self-Attention计算它和序列中所有词（包括自己）的"相关性"，然后用这些相关性作为权重，对所有词的信息做加权平均。

比如句子"The animal didn't cross the street because it was too tired"——"it"应该和"animal"强相关，而不是"street"。Self-Attention自动学会了这种指代消解。

多头注意力（Multi-Head Attention）

Q（Query）、K（Key）、V（Value）三个矩阵是Transformer注意力机制的核心。每个"头"看不同的方面：一个头可能关注语法关系，另一个关注语义相似度，另一个关注位置信息。多个头合在一起，模型能从多个角度理解序列。

import torch
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=512, n_heads=8):
        super().__init__()
        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads

        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, Q, K, V, mask=None):
        batch_size = Q.size(0)

        # 1. 线性变换 + 分头
        Q = self.W_q(Q).view(batch_size, -1, self.n_heads, self.d_k).transpose(1,2)
        K = self.W_k(K).view(batch_size, -1, self.n_heads, self.d_k).transpose(1,2)
        V = self.W_v(V).view(batch_size, -1, self.n_heads, self.d_k).transpose(1,2)

        # 2. 计算注意力分数: softmax(QK^T / sqrt(d_k)) × V
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        attention = torch.softmax(scores, dim=-1)
        out = torch.matmul(attention, V)

        # 3. 合并多头 → 输出
        out = out.transpose(1, 2).contiguous().view(batch_size, -1, self.d_model)
        return self.W_o(out)
🔑 为什么要除以sqrt(d_k)？当d_k很大时，QK^T的点积值也会很大，导致softmax后的分布非常"尖锐"（接近one-hot），梯度几乎为零。除以sqrt(d_k)相当于做了缩放，让softmax保持在梯度有效的区域。这是Transformer设计中的一个关键细节——没有它，训练不稳定甚至发散。

23.4 BERT：预训练 + 微调的范式革命

2018年BERT的出现彻底改变了NLP的游戏规则。它的核心思想：先在海量无标注文本上预训练一个通用语言理解模型，然后在下游任务数据上微调。这个"预训练→微调"的范式成为了NLP的标配。

BERT的创新点：

  • 双向编码：传统语言模型只能从左到右（或从右到左）看上下文。BERT同时看两边——用Masked Language Model（MLM）任务，随机遮住一些词让模型预测
  • 下一句预测（NSP）：判断两句话是否是连续的。后来发现这个任务不那么重要（RoBERTa去掉了NSP效果反而更好）
from transformers import AutoTokenizer, AutoModelForSequenceClassification

# 加载预训练BERT做中文情感分析
model_name = "bert-base-chinese"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=2)

# 微调（只需几行代码）
texts = ["这部电影太棒了", "剧情无聊到想睡觉"]
labels = [1, 0]
inputs = tokenizer(texts, padding=True, truncation=True, return_tensors="pt")
outputs = model(**inputs, labels=torch.tensor(labels))
loss = outputs.loss
loss.backward()
💡 BERT vs GPT的本质区别：BERT是编码器（Encoder-only），看双向上下文，擅长"理解"任务（分类、NER、QA）。GPT是解码器（Decoder-only），只看左边上下文（自回归），擅长"生成"任务（续写、对话、翻译）。这就是为什么做文本分类用BERT，做聊天机器人用GPT——架构决定了能力边界。
