13.1 没有标签的时候怎么学？

之前我们学的所有算法都是监督学习——每个训练样本都有正确的标签。但现实中，大量数据是没有标签的。你怎么把100万用户分成几个有意义的群体？你怎么给一个没有目录的文档库建立分类体系？这就是聚类（Clustering）的用武之地——在没有标签的情况下，自动发现数据中的自然分组。

我在一家内容平台做过一个项目：有500万篇用户发布的文章，没有任何分类标签，需要自动归到几十个主题下。我一开始直接上K-Means，设K=50，结果发现"搞笑"类全是讲笑话的，"科技"类却混进了大量鸡汤文。后来才意识到：选什么聚类算法、K取多少、怎么评估聚类质量——这些选择对结果的影响远大于算法本身。

13.2 K-Means：最经典的聚类算法

K-Means的思想简单到可以用一句话描述：随机选K个中心点，把每个样本归到最近的中心，然后移动中心到它所属样本的平均位置，重复直到收敛。

from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler

# K-Means对尺度敏感，必须标准化
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

kmeans = KMeans(n_clusters=5, init='k-means++', n_init=10,
                max_iter=300, random_state=42)
kmeans.fit(X_scaled)

# 获取每个样本的簇标签
labels = kmeans.labels_
# 获取簇中心
centers = kmeans.cluster_centers_
# 对新样本预测簇
new_labels = kmeans.predict(X_new)

K-Means的三个致命弱点

  • 必须预设K：如果不知道应该聚成几类，就得试。肘部法则（画K vs Inertia曲线找"拐点"）和轮廓系数是常用方法
  • 假设球形簇：K-Means隐含假设每个簇是球形的（各方向方差相同）。如果数据是月牙形或环形，K-Means完全抓瞎
  • 对初始中心敏感：不同初始化可能得到完全不同的结果。k-means++初始化大大改善了这个问题

13.3 DBSCAN：不需要预设K的聚类

DBSCAN的核心思想：一个簇是一组"密度相连"的点。它只需两个参数：ε（邻域半径）和min_samples（成为核心点所需的最小邻居数）。它能发现任意形状的簇，而且能自动识别噪声点（不属于任何簇的离群点）。

特性K-MeansDBSCAN
需要预设簇数是（K）否（自动发现）
簇形状只能球形任意形状
噪声处理所有点都归入某个簇自动识别离群点
大数据集快 O(n·K·d)较慢 O(n·log n) 到 O(n²)
高维数据差（距离失真）非常差（密度概念在高维失效）
💡 DBSCAN参数调优技巧：ε 的选择可以用"k-distance图"——对每个点计算它到第k个最近邻的距离，排序后画图，拐点处就是合适的ε。min_samples通常设为特征维度的2倍。一个经验法则：min_samples ≥ 维度数 + 1。

13.4 层次聚类：建一棵"聚类树"

层次聚类不需要指定K，而是构建一个树状结构（树状图/Dendrogram）。有两种策略：

  • 自底向上（Agglomerative）：每个点是一个簇 → 合并最近的两个簇 → 重复 → 直到所有点在一个簇。最常用
  • 自顶向下（Divisive）：所有点在一个簇 → 分裂 → 重复

关键选择是"簇间距离"怎么定义：single-linkage（最近点距离，容易产生"链"）、complete-linkage（最远点距离，产生紧凑簇）、Ward's method（最小化簇内方差，最常用）。

13.5 聚类评估：这事比分类难得多

分类有准确率，回归有RMSE——聚类没有标准答案，评估是聚类最大的难点。

指标含义范围越高越好？
轮廓系数簇内紧密 + 簇间分离[-1, 1]越高越好（>0.5不错）
Davies-Bouldin Index簇内距离 / 簇间距离[0, ∞)越低越好
Calinski-Harabasz簇间方差 / 簇内方差[0, ∞)越高越好
Inertia (K-Means)每个点到其簇中心的距离平方和[0, ∞)越低越紧密，但不单调
⚠️ 关键提醒：这些内部评估指标只是衡量几何结构的好坏，和业务需求无关。如果业务上你需要的是"按用户消费水平分为3档"，即使DBSCAN自动发现了7个形状优美的簇也是错的。聚类评估的终极标准永远是：聚出来的分组在业务上是否有意义？

13.6 聚类实战流程

from sklearn.cluster import KMeans, DBSCAN, AgglomerativeClustering
from sklearn.metrics import silhouette_score

# 标准化
X_scaled = StandardScaler().fit_transform(X)

# 尝试K-Means，用肘部法则选K
inertias = []
K_range = range(1, 15)
for k in K_range:
    km = KMeans(n_clusters=k, init='k-means++', n_init=10, random_state=42)
    km.fit(X_scaled)
    inertias.append(km.inertia_)

# 画肘部图找拐点

# 用轮廓系数验证
for k in [3, 4, 5, 6]:
    km = KMeans(n_clusters=k, random_state=42)
    labels = km.fit_predict(X_scaled)
    sil = silhouette_score(X_scaled, labels)
    print(f"K={k}, 轮廓系数={sil:.4f}")

# DBSCAN（如果怀疑有奇怪形状的簇或需要识别离群点）
dbscan = DBSCAN(eps=0.5, min_samples=5)
labels_db = dbscan.fit_predict(X_scaled)
n_clusters = len(set(labels_db)) - (1 if -1 in labels_db else 0)
n_noise = list(labels_db).count(-1)
print(f"发现了{n_clusters}个簇, {n_noise}个噪声点")
