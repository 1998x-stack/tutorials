第十七章：训练稳定性与调参

同学们好！走到这里，我们已经学了DQN、DDPG、PPO、SAC等一系列强化学习算法。你可能已经发现了一个令人沮丧的事实：即使你把算法论文里的伪代码一字不差地实现出来，你的智能体也大概率不会收敛。更气人的是——同样的代码、同样的环境，别人跑出来状态爆炸，你跑出来纹丝不动。

这绝不是你一个人的问题。我刚读研那会儿，第一次复现DQN时在Atari Breakout上跑了整整三天，loss曲线稳得像一条直线，reward始终在1分上下徘徊。我检查了每一行代码，把网络架构改了七八遍，甚至怀疑是PyTorch的bug。最后发现是学习率设成了0.01——比论文推荐的0.00025大了四十倍。梯度在反向传播时直接NaN了。这就是我给你们讲的第一个"踩坑"故事：一个参数的失误，葬送了三天算力。

踩坑实录：某次我在HalfCheetah上跑SAC，reward始终上不去，调试了一周。最后发现reward scaling写错了——把reward乘以了10而不是除以10。这导致Q值爆炸，策略完全崩溃。从此以后，我每次实验前都会先打印一条transition的reward、Q值和梯度范数，确认数值范围正常再跑长实验。这个习惯帮我省下了至少几个月的无效算力。

本章我们就来系统地探讨训练稳定性与调参这个"RL工程师的必修课"。这可能是整本教程里最"不性感"但却最实用的一章——掌握它，你的实验时间可以减少一半以上。

17.1 RL为什么不稳定

要解决不稳定问题，首先需要理解不稳定的根源。强化学习的训练不稳定并非偶然，而是由其内在结构决定的。Sutton在2015年提出了著名的"致命三要素（Deadly Triad）"理论：当以下三个条件同时满足时，训练发散几乎是不可能避免的：

  • 函数逼近（Function Approximation）：我们用神经网络来近似价值函数或策略。神经网络的表达能力很强，但它会带来近似误差，而且不同状态之间的估计会相互干扰。
  • 自举（Bootstrapping）：我们用当前的估计值来更新目标值（如TD目标 r + γ Q(s', a')）。这意味着目标本身也在不断变化——你是在"追逐一个移动的靶子"。
  • 异策略学习（Off-Policy Learning）：我们用过去的数据来更新当前的策略。数据分布与当前策略的分布不一致，导致更新方向可能存在偏差。

几乎所有主流的深度RL算法（DQN、DDPG、SAC、TD3）都同时满足这三个条件，因此它们本质上都是不稳定的。即使像PPO这样的同策略算法，因为使用了函数逼近和自举，也依然存在稳定性问题。

除此之外，RL还面临两个独特的挑战：

  • 灾难性遗忘（Catastrophic Forgetting）：神经网络在更新过程中会"忘记"之前学到的知识。在监督学习中，数据分布是固定的，所以这个问题不太严重。但在RL中，新数据来自当前策略，如果策略突然改变，网络可能会忘记如何在之前的状态下做出好的决策。
  • 非平稳数据分布（Non-Stationary Data Distribution）：这是RL和监督学习最本质的区别。在监督学习中，训练集和测试集是独立同分布的。但在RL中，智能体策略的变化会改变它所见到的数据——策略变好了一些，它就遇到了新的状态，这些新状态下的数据分布和之前完全不同。这就像一个学生每次考试都在换科目，学习自然困难。
直观理解：想象你一边在跑步机上跑步，一边有人不断调整跑步机的速度和坡度。你既要保持平衡（策略优化），又要适应不断变化的环境（数据分布）。这就是RL训练的本质困境。

17.2 稳定性技巧

既然不稳定是RL的内在属性，那我们能做什么？好消息是，经过多年实践，社区积累了一套行之有效的稳定性技巧。下面列出最重要的几个：

梯度裁剪（Gradient Clipping）：这是最基础也是最有效的技巧。当梯度过大时，参数更新会"一步迈过头"，导致策略瞬间崩溃。梯度裁剪将梯度的范数限制在一个合理范围内（通常max_norm=0.5或1.0）。实践中几乎所有RL实现都应该加上梯度裁剪。

奖励归一化（Reward Normalization）：不同环境的奖励尺度可以相差巨大——CartPole的奖励大约是1/tick，而MuJoCo的奖励可能是几千。如果不做归一化，Q值的尺度就会爆炸。常用做法是维护一个running mean和running std，将奖励除以标准差（或者使用Reward Scaling，将奖励缩放到[-1, 1]区间）。

优势归一化（Advantage Normalization）：在Actor-Critic算法中，优势函数A(s, a)的值范围也需要注意。如果在PPO中不对优势做归一化，策略更新的步长就会忽大忽小。标准做法是在batch维度上对优势进行z-score归一化。

目标网络更新率（Target Network Update Rate）：对于使用目标网络的算法（DQN、DDPG、SAC），tau值（软更新系数）是一个关键参数。tau太小（如0.0001），目标网络变化太慢，学习速度受影响；tau太大（如0.01），目标网络变化太快，近似于没有目标网络。常用的tau值是0.005（SAC/DDPG）或周期性硬更新（DQN每1000步更新一次）。

经验回放缓冲区（Replay Buffer）：缓冲区大小对训练有显著影响。太小的缓冲区（如1000条）会导致样本多样性不足，模型过拟合近期经验；太大的缓冲区（如10^7条）会包含太多过时的数据，降低学习效率。经验法则：简单环境用10^4~10^5，复杂环境用10^5~10^6。采样策略也很关键——优先采样近期经验的策略有时效果更好。

观测预处理（Observation Preprocessing）：对于Atari游戏，标准的预处理流程是：将RGB图像缩放到84x84的灰度图，然后堆叠最近4帧作为状态输入。这4帧堆叠为智能体提供了运动信息（如球的方向和速度）。对于MuJoCo等连续控制任务，通常将观测归一化到零均值和单位方差。

17.3 学习率与优化器

学习率可能是RL中最敏感的超参数。选对学习率，你的算法就成功了80%。

优化器选择：目前Adam是RL领域的事实标准。相比SGD，Adam自带自适应学习率和动量，对超参数的敏感度更低。虽然有一些工作尝试使用SGD（声称泛化性能更好），但在RL这个"非平稳+小批量"的场景下，Adam的稳定性优势远大于其泛化劣势。RMSProp也偶有使用（主要在DQN的原始论文中），但Adam几乎是所有现代RL库的默认选择。

学习率调度（Learning Rate Scheduling）：

  • 常数学习率：最简单，但如果LR选得不好，要么收敛太慢，要么震荡不收敛。
  • 线性衰减（Linear Decay）：在训练过程中线性地将LR从初始值降到0。效果通常不错，需要指定总训练步数。
  • 余弦退火（Cosine Annealing）：按照余弦曲线降低LR，在训练后期可以有"重启"效果。在PPO中表现不错。

典型取值范围：

  • Actor网络（策略网络）：3e-4 到 1e-4。注意Actor的LR通常比Critic小。
  • Critic网络（价值网络）：1e-3 到 3e-4。Critic通常需要更高的学习率，因为它需要更快地拟合不断变化的Q值。
  • 如果actor和critic使用相同的LR（如PPO中共享网络），常用3e-4。

学习率与batch size的交互：一般来说，更大的batch size可以支持更大的学习率。这是因为大batch的梯度估计更准确，方差更小。但RL中batch size不能太大——太大的batch会降低更新频率，减少样本效率。通常的batch size范围是64~4096（同策略算法如PPO可以用更大的batch）。

黄金法则：如果你不确定从哪里开始，对于大多数RL算法，使用Adam优化器 + 3e-4的学习率 + 256的batch size。这个配置在大量环境和算法上都有不错的表现。从"安全起点"出发再逐个调整，比从"激进参数"出发去debug NaN要好得多。

17.4 调参方法论

调参不是玄学，而是一个系统性的实验过程。以下是经过多年实践验证的调参方法论：

第一步：从简单开始（Start Simple）。先用最简单的小网络、最简单的环境验证你的代码能不能学东西。比如在CartPole上测试DQN，在HalfCheetah上测试SAC——不要一上来就在Atari或Humanoid上跑。简单环境训练快，调试周期短。

第二步：用已知配置验证（Validate with Known Configs）。从论文作者提供的开源代码或权威实现（如Stable-Baselines3、rlkit）中获取推荐的超参数配置。先确保你的实现能在这些配置下重现论文的结果，再去做修改。如果你的实现连标准配置都跑不出合理结果，那一定不是参数的问题，而是代码的问题。

第三步：一次只改一个参数（Change One Parameter at a Time）。这是最基本的实验原则。同时修改多个参数，你永远不知道是哪个变化产生了效果。使用版本控制工具（如Weights & Biases、TensorBoard）来记录每次实验的超参数组合。

第四步：记录一切（Log Everything）。除了reward曲线，至少还要记录：loss曲线、梯度范数、Q值分布、explained variance、动作分布、更新幅度。很多时候，reward不动但loss在降，说明Q值在收敛但策略没有优化——这是learning rate mismatch的典型信号。

第五步：使用多个种子（Multiple Seeds）。RL实验结果方差极大——同一个超参数在seed=0时表现炸裂，在seed=42时可能完全不行。最少跑3个种子，建议跑5个。报告结果时用mean ± std，而不是只挑最好的那个seed。

强化学习假设检验（The RL Hypothesis Test）：当你认为"参数A比参数B更好"时，用5个不同的seed运行两组实验，然后做独立样本t检验或者Mann-Whitney U检验，确认差异在统计上显著。视觉判断不可靠——人类的模式识别能力太强，即使随机噪声也能看出趋势。

17.5 网络架构与初始化

网络架构和初始化方法对训练的稳定性和收敛速度有显著影响。虽然"网络越深越好"在CV和NLP中成立，但在RL中并非如此——过深的网络反而容易导致训练不稳定。

MLP架构（用于MuJoCo等连续控制任务）：最常见的配置是2~3层隐藏层，每层64~256个神经元。PPO论文使用(64, 64)的小网络，而SAC和TD3通常使用(256, 256)或(256, 256, 256)。经验是：不要太深，太深反而难训。

CNN架构（用于Atari等视觉任务）：标准的Nature DQN架构是3层卷积（32/64/64个filter，8/4/3的stride）后接512维的全连接层。输入是84x84的灰度图（4帧堆叠）。这个架构至今仍是大多数Atari任务的基准选择。

权重初始化（Weight Initialization）：

  • 正交初始化（Orthogonal Initialization）：对策略网络（policy network）的输出层效果最好。正交矩阵保证了前向传播和反向传播中梯度的范数不会剧烈变化，这对策略梯度的稳定性至关重要。在很多RL实现中（如Stable-Baselines3），所有的线性层默认使用正交初始化，增益(gain)根据激活函数选择。
  • Xavier/Glorot初始化：适用于大多数Critic网络和其他不使用ReLU的层。如果你的网络使用tanh激活函数，Xavier初始化和正交初始化效果接近。
  • Kaiming/He初始化：适用于使用ReLU激活函数的层。
  • 输出层初始化（Final Layer Initialization）：这是非常关键但容易被忽视的细节。策略网络输出的均值（mean）应该初始化为0附近的小值（如使用正交初始化且gain=0.01），这样初始策略接近均匀分布，不会在一开始就产生极端动作。价值网络的输出层也应该用小权重初始化，让初始Q值接近0。

层归一化 vs 批归一化：在RL中，层归一化（LayerNorm）通常比批归一化（BatchNorm）更适用。原因在于RL的batch size通常较小，且数据分布非平稳——BatchNorm依赖的running统计量在RL中反而会引入额外的偏差。LayerNorm对每个样本独立计算归一化统计量，不受batch size和数据分布变化的影响。很多现代RL实现（如DrQ、CURL）都使用LayerNorm作为默认选择。

下面是一个结合了上述技巧的稳定训练代码示例（基于SAC的Critic更新部分）：

import torch
import torch.nn as nn
import torch.nn.functional as F

class StableCritic(nn.Module):
    """一个稳定性优化的Critic网络"""
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super().__init__()
        # 使用正交初始化
        self.fc1 = nn.Linear(state_dim + action_dim, hidden_dim)
        self.ln1 = nn.LayerNorm(hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, hidden_dim)
        self.ln2 = nn.LayerNorm(hidden_dim)
        self.fc3 = nn.Linear(hidden_dim, 1)

        # 初始化：最后输出层用小权重
        self.fc3.weight.data.mul_(0.01)
        self.fc3.bias.data.zero_()

    def forward(self, state, action):
        x = torch.cat([state, action], dim=-1)
        x = self.fc1(x)
        x = self.ln1(x)
        x = F.relu(x)
        x = self.fc2(x)
        x = self.ln2(x)
        x = F.relu(x)
        return self.fc3(x)

def update_critic(critic, target_critic, actor, replay_buffer,
                  optimizer, gamma=0.99, tau=0.005,
                  grad_clip=1.0, device='cuda'):
    """一个带有梯度裁剪和软更新的Critic更新步骤"""
    # 从回放缓冲区采样
    states, actions, rewards, next_states, dones = replay_buffer.sample()

    # ---- 稳定性技巧1: 奖励归一化 ----
    rewards = (rewards - rewards.mean()) / (rewards.std() + 1e-8)

    # ---- 目标Q值计算 ----
    with torch.no_grad():
        next_actions, next_log_probs = actor.sample(next_states)
        target_q1, target_q2 = target_critic(next_states, next_actions)
        target_q = torch.min(target_q1, target_q2)
        target_value = rewards + gamma * (1 - dones) * target_q

    # ---- 当前Q值 ----
    current_q1, current_q2 = critic(states, actions)

    # ---- Critic损失 ----
    critic_loss = F.mse_loss(current_q1, target_value) + \
                  F.mse_loss(current_q2, target_value)

    # ---- 反向传播与梯度裁剪 ----
    optimizer.zero_grad()
    critic_loss.backward()

    # 稳定性技巧2: 梯度裁剪
    torch.nn.utils.clip_grad_norm_(critic.parameters(), grad_clip)

    optimizer.step()

    # ---- 稳定性技巧3: 软更新目标网络 ----
    for param, target_param in zip(critic.parameters(),
                                    target_critic.parameters()):
        target_param.data.copy_(tau * param.data +
                                (1 - tau) * target_param.data)

    return critic_loss.item()
代码解读：这段代码集成了本章讨论的多个稳定性技巧——LayerNorm确保各层输出分布稳定；小权重初始化防止初始Q值过大；奖励归一化控制目标值尺度；梯度裁剪防止单步更新过大；软更新保证目标网络平滑变化。每一步都有明确的目的，不是"玄学"。

最后，让我用一个真实的故事来结束本章。我的一位同学在复现SAC时，训练Ant-v2始终只能跑到2000分出头（论文报告是5000+）。他检查了所有超参数，和论文一模一样。最后他发现——Ant环境在v2版本中的termination条件是"身体高度低于阈值"，而这个条件在训练后期频繁触发，导致episode被截断。他关掉了early termination，reward立刻冲到了5500。这个故事告诉我们：有时候不是算法的问题，不是超参数的问题，而是对环境定义的误解。所以调参的第一步，永远是了解你的环境。

从下一章开始，我们将进入RL中更前沿的话题。首先登场的是Rainbow——它将DQN的六大改进融为一体，展示了如何通过系统性工程优化获得SOTA性能。再之后，我们会讨论安全强化学习（Safe RL），探讨如何在保证智能体行为安全的前提下进行策略优化——这在实际部署中至关重要，也是近年来RL研究最活跃的方向之一。
