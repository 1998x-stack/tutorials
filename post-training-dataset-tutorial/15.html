第15章 偏好对齐数据基础

大家好，我是你们的讲师。从这一章开始，我们正式进入课程最有意思的部分——偏好对齐数据。如果你之前接触过 SFT 数据，可能会觉得：模型不是已经在指令数据上微调过了吗？为什么还需要"偏好对齐"？

答案很简单：SFT 让模型学会了"怎么回答"，但没有让模型学会"什么回答更好"。

为什么 SFT 不够——模型需要知道「好答案」和「更好答案」的区别

想象一下你是一个老师，你教一个学生解题。SFT 阶段你做的事情是：给出一道题目和一个标准答案，让学生记住这个对应关系。经过大量这样的训练，学生能够比较准确地回答各种问题。

但问题是：如果两道不同的题目恰好引出同一个答案，或者同一个问题有多种合理的回答方式，学生怎么知道哪个回答更好？SFT 没有提供这个信号。SFT 的目标函数是最大化正确答案的似然，它并不区分一个"勉强及格"的答案和一个"完美"的答案。

这就是偏好对齐要解决的问题——在"正确"的基础上，引入"更好"的概念。偏好数据告诉模型：对于同一个 prompt，回答 A 比回答 B 好，请向 A 的方向靠近，同时远离 B。这种对比学习信号，让模型的输出质量从"能回答"跃升到"能回答好"。

一个经典的类比：SFT 像教学生写作文，给他看范文让他模仿；偏好对齐则像作文批改——不仅告诉他这篇作文好在哪里，还告诉他这篇不如另一篇的地方。后者带来的学习效率和质量上限，远超单纯的模仿。

偏好数据的数学基础 — Bradley-Terry 偏好模型

要理解偏好数据，先要理解偏好背后的人类决策模型。在统计学和经济学中，有一类成熟的模型用于描述人如何做"二选一"的判断。其中最经典的，就是 Bradley-Terry 模型。

Bradley-Terry 模型的核心假设是：每一个选项都有一个潜在的"强度值"或"质量分"。当人在两个选项之间做选择时，选择 A 而非 B 的概率，取决于 A 和 B 各自强度值的比值：

P(A 优于 B) = exp(score_A) / (exp(score_A) + exp(score_B))

在 RLHF 的语境下，我们将这个"强度值"理解为奖励函数 r(x, y) —— 给定 prompt x 和回答 y，模型认为这个回答有多"好"。于是：

P(y_c 优于 y_r | x) = exp(r(x, y_c)) / (exp(r(x, y_c)) + exp(r(x, y_r)))
                    = σ(r(x, y_c) − r(x, y_r))

其中 σ 是 sigmoid 函数。这个公式是整个偏好对齐理论的基石。它告诉我们：偏好数据本质上是在编码一个潜在的奖励函数——当我们记录"对于 prompt x，回答 y_c 比 y_r 好"时，我们实际上是在隐式地标注了一个不等式 r(x, y_c) > r(x, y_r)。

这个洞察引出了一个重要的实践结论：偏好数据的质量，取决于它能否真实反映奖励函数的大小关系。如果标注者随机或错误地选择了"更好"的回答，那这条数据实际上是在编码一个错误的奖励信号。

Bradley-Terry 模型的局限性：这个模型假设偏好是"可传递的"——如果 A 优于 B，B 优于 C，那么 A 必然优于 C。但人类偏好并不总是满足这个性质（想想"循环投票"悖论）。所以在实际标注中，偶尔会出现违反传递性的情况，这是正常的。关键是要保证大多数偏好对的相对关系是一致的。

偏好数据的标准格式

理解了数学基础，我们来看看偏好数据长什么样。现在业界最通用的偏好数据格式是一个三元组：

{
  "prompt": "请解释一下量子纠缠是什么？",
  "chosen": "量子纠缠是量子力学中的一种现象，两个或多个粒子...",
  "rejected": "量子纠缠就是两个粒子互相影响，很复杂的东西。",
  // 可选元数据
  "metadata": {
    "score_chosen": 4.5,
    "score_rejected": 2.0,
    "dimensions": {
      "helpfulness": {"chosen": 5, "rejected": 2},
      "harmlessness": {"chosen": 5, "rejected": 3},
      "truthfulness": {"chosen": 4, "rejected": 2}
    },
    "annotator_id": "annotator_0823",
    "source": "expert_human"
  }
}

核心字段：

  • prompt：输入指令或问题，通常是 SFT 阶段也会使用的格式
  • chosen：被标注者选为"更好"的回答
  • rejected：被标注者选为"较差"的回答

可选元数据字段：

  • 评分：chosen 和 rejected 各自的绝对评分，有助于后续分析
  • 多维度评分：从帮助性、无害性、真实性等维度分别打分
  • 标注者信息：标注者 ID，便于追溯和一致性分析
  • 数据来源：人工标注 / 模型生成 / A/B 测试等

为什么需要成对数据

你可能会问：为什么不直接让标注者对单个回答打分（比如 1-5 分），然后取高分回答作为训练目标？这样不是更简单吗？

这个问题触及了偏好数据设计的核心哲学。研究表明：人类在相对排序上比绝对打分要可靠得多。原因有几方面：

  • 参照系漂移：两个不同的标注者，一个对所有回答都很严格（平均打 2 分），另一个很宽松（平均打 4 分）。绝对分数无法跨标注者对比，但"选择 A 还是 B"的判断则相对稳定。
  • 天花板效应：当两个回答质量都很高时，标注者很难在绝对评分上区分它们（都是 4.5 分 vs 4.6 分），但要求标注者"选一个更好的"时，依然能做出有意义的判断。
  • 认知负荷：给一个回答打 1-5 分需要调用抽象的质量标准；而对比两个回答并判断哪个更好，更符合人类的本能决策方式。
2025 年的一篇有趣研究：有团队尝试了"绝对打分 + 成对排序"的混合标注策略——先让标注者给每个回答独立打分，再对有争议的样本做成对排序。结果发现，成对排序的一致性（跨标注者）比绝对打分高出约 20-30 个百分点。这说明相对判断确实是人类更擅长的任务。

当然，成对数据也有缺点——标注成本翻倍。每条偏好数据需要生成两个回答并进行对比标注，成本大约是单条 SFT 数据的 2-3 倍。这也是为什么开源偏好数据集通常比 SFT 数据集要小的原因之一。

偏好数据的标注质量标准

有了数据格式，下一个关键问题就是：什么样的偏好数据是高质量的？这需要一套量化的质量控制指标。

标注者一致性：Cohen's Kappa

Cohen's Kappa 是最常用的标注者间一致性指标。它的计算考虑了随机一致的概率，比简单的"一致率"更可靠：

κ = (P_observed − P_expected) / (1 − P_expected)

其中：
- P_observed = 标注者间实际一致的比例
- P_expected = 随机情况下标注者一致的概率

κ 值解读：
- κ < 0.20：一致性很差，数据不可用
- 0.20 ≤ κ < 0.40：一致性一般，需加强标注培训
- 0.40 ≤ κ < 0.60：中等一致性，可用但有噪声
- 0.60 ≤ κ < 0.80：高度一致，数据质量良好
- κ ≥ 0.80：几乎完美一致，非常可靠

在实际项目中，建议至少每个批次安排 10-20% 的样本由双标注者独立标注，以持续监控 Cohen's Kappa。如果 κ 值低于 0.4，说明标注标准可能不清晰或标注者培训不足，需要立即干预。

区分度要求

偏好数据的一个常见陷阱是：chosen 和 rejected 之间的差距太小。如果 chosen 只是"稍微好一点点"，模型从这条数据中学到的信号非常微弱。实践中，我们通常要求：

  • 核心质量维度上至少 2 分差距（以 5 分制计算）
  • 标注者需要明确说明选择理由，避免"两者都差不多，随便选一个"
  • 拒绝不明确样本：如果标注者认为两个回答质量相当、难以区分，应该将该样本标为"放弃"而不是勉强选择一个
一个常见的失败模式：有些团队为了提高数据产出速度，允许标注者在"不确定"时随便选择一个。结果训练出来的模型偏向性非常弱，效果甚至不如只用 SFT。这是因为模型从模糊的偏好对中学习到的信号接近随机噪声，反而干扰了 SFT 阶段学到的知识。记住——一条高质量的偏好数据胜过十条模糊的数据。

从偏好数据到对齐算法

不同类型的对齐算法，对偏好数据的使用方式截然不同。理解这些差异，有助于你根据手头的数据类型选择合适的训练策略：

方法 是否需要偏好对 是否需要奖励模型 数据格式要求 典型使用方式
SFT
（监督微调）		 ❌ 不需要 ❌ 不需要 prompt + answer 直接最大化 chosen 的似然，忽略对比信号
RM
（奖励模型）		 ✅ 需要成对 N/A（训练目标） prompt + chosen + rejected 训练一个独立的打分模型来预测偏好
DPO
（直接偏好优化）		 ✅ 需要成对 ❌ 不需要 prompt + chosen + rejected 直接通过偏好对优化策略模型，无需显式 RM
PPO
（近端策略优化）		 ✳️ 需要评分/奖励 ✅ 需要 prompt + answer + reward_score 先用偏好数据训练 RM，再用 RM 的奖励信号训练策略
KTO ❌ 不需要成对 ❌ 不需要 prompt + answer + is_desirable 仅需正/负反馈信号，无需成对对比
ORPO ✅ 需要成对 ❌ 不需要 prompt + chosen + rejected 在 SFT 训练中直接加入偏好对比损失

从上表可以看出，偏好对（chosen/rejected）是最通用的偏好数据格式。无论你后续选择 DPO、PPO+RHF 还是 ORPO，有成对数据在手都不会出错。这也是为什么全球主要的偏好数据集（如 HH-RLHF、UltraFeedback）都以成对格式发布。

如果你只有单一回答的评分数据（比如用户点赞/点踩），那 KTO 是你的最佳选择。如果你的数据既有评分又有成对对比，那你可以灵活选择几乎任何算法。关于这些算法对数据格式的具体要求，我们在第 17 章会深入讲解。

给初学者的建议：如果你是第一次构建偏好对齐管线，建议从 DPO + 成对数据开始。DPO 的训练流程最简单（不需要独立 RM，不需要对抗采样），并且成对数据的标注流程已经非常成熟。等你对偏好对齐有了手感，再尝试 PPO、KTO 等更复杂的方案。
