大家好，我是你们的讲师。

通信世界充满了噪声：手机信号穿过墙壁会衰减，光盘表面的划痕会破坏数据，深空探测器的信号穿越数百万公里后微弱得几乎无法辨认。当你在地球上接收来自火星探测器的照片时，信号强度可能只有发射时的十亿分之一——而在这段漫长的旅程中，宇宙射线、太阳风、热噪声都在不停地干扰它。

面对这些噪声，工程师们很早就意识到：发送原始数据是不行的。你需要一些"冗余"——在发送的信息中加入额外的结构，让接收方即使收到部分损坏的数据，也能恢复出原始信息。

但冗余加多了，传输效率就下降了。加少了，又不够纠错。这个"效率 vs. 可靠性"的权衡，是所有通信系统设计的核心矛盾。而信息论用一个非常优雅的数学结构——离散信道——来建模所有这些问题。

我在做卫星图像传输研究的时候，有一次实验发现接收端解码出来的图像总是有一些奇怪的条纹。排查了三天，发现是因为我没有正确建模信道中的"突发错误"（burst error）——实际信道中的错误往往是成串出现的，而我们用的是针对独立错误的纠错码。这个教训告诉我：没有"万能的"信道模型，选对模型比选对编码更重要。

信道的数学模型

信息论中的"信道"是一个非常抽象的概念：它只关心输入和输出的统计关系，不关心底层的物理机制。

离散无记忆信道（Discrete Memoryless Channel, DMC）的数学定义如下：

  • 输入字母表 X = {x₁, x₂, ..., xr}
  • 输出字母表 Y = {y₁, y₂, ..., ys}
  • 转移概率 p(y|x) — 给定输入x时收到输出y的概率
  • 无记忆性：p(y₁y₂...yₙ|x₁x₂...xₙ) = ∏ p(yi|xi)

转移概率可以用一个信道矩阵来表示——每一行对应一个输入，每一列对应一个输出，矩阵元素为 p(y|x)：

         输出: y₁   y₂  ...  y_s
输入 x₁  [ p(y₁|x₁) p(y₂|x₁) ... p(y_s|x₁) ]
     x₂  [ p(y₁|x₂) p(y₂|x₂) ... p(y_s|x₂) ]
     ... [   ...      ...       ...   ...   ]
     x_r  [ p(y₁|x_r) p(y₂|x_r) ... p(y_s|x_r) ]

每一行的和 = 1（给定输入，某个输出必然发生）

这里"无记忆"的假设非常关键。它意味着：信道在每一次使用时的行为是独立的，前一次传输不会影响后一次。这个假设在很多实际场景中近似成立（比如加性白高斯噪声信道），但在有衰落、干扰的情况下就不太成立了。

直觉理解：信道模型就像是快递公司的"服务质量承诺"。输入X是你寄出的包裹，输出Y是收件人实际收到的。转移概率p(y|x)告诉你：寄出"完好无损的玻璃杯"，收到"完好无损"的概率有多大？收到"碎成八瓣"的概率有多大？好的信道（比如快递专线）转移概率集中在y=x附近；差的信道（比如暴力运输）转移概率比较发散。

对称信道与BSC/BEC

在所有信道模型中，有两类最简单也最重要的——它们是理解复杂信道的基石。

二元对称信道（Binary Symmetric Channel, BSC）

BSC是最简单的信道模型：输入是{0,1}，输出是{0,1}。每个比特以概率p被翻转，以概率1-p被正确传输。

BSC(p) 的信道矩阵：

         输出: 0    1
输入 0  [ 1-p   p  ]
     1  [  p   1-p ]

对称性：0→1和1→0的错误概率相同（都是p）

p就是误比特率（Bit Error Rate, BER）。在工程中有个直观的对应：BSC模型用于描述很多实际系统的基础层，比如无线通信中解调器之后的输出。

二元删除信道（Binary Erasure Channel, BEC）

BEC的输入是{0,1}，但输出有三个可能性：{0, 1, ?}——其中"?"表示删除（erasure），即接收方知道这个比特没有被可靠接收。

BEC(ε) 的信道矩阵：

           输出: 0    ?    1
输入 0  [  1-ε   ε    0  ]
     1  [   0    ε   1-ε  ]

删除概率为ε，正确概率为1-ε
注意：0→1和1→0的概率都是0——BEC从不出错，只删除

BEC在实际中也有对应——比如在IP网络中，一个数据包要么完整到达，要么完全丢失（但你知道它丢了）。这就是一个删除信道。BEC虽然模型简单，但具有深刻的数学意义——许多信道编码定理的证明在BEC上最容易理解。

这两者的区别非常关键：在BSC中，你不知道哪些比特出错了；在BEC中，你知道哪些比特被删除了，但不知道删掉的是什么。后者的问题反而更容易处理——因为你至少知道"哪里出了问题"。

信道的级联

如果我们把两个信道串起来使用——比如信号先经过一个无线信道传到中继站，再经过另一个无线信道传到最终目的地——会发生什么？

对于三个随机变量 X → Y → Z（形成马尔可夫链），我们可以计算等效的信道转移概率：

等效信道矩阵 = 信道1矩阵 × 信道2矩阵

p(z|x) = ∑_y p(y|x) · p(z|y)

直觉：所有经过中间状态y的路径加起来。

这里一个非常重要的概念是数据处理不等式（Data Processing Inequality）：

如果 X → Y → Z 形成马尔可夫链，则 I(X; Z) ≤ I(X; Y)。

换句话说：对数据进行任何处理（哪怕是精心设计的信号处理），都不会增加其中包含的原始信息。你只能保持或减少信息量，绝对不会增加。这个不等式虽然看起来简单，但它的推论非常深远——它意味着任何信号处理系统的性能都有"天花板"。

数据处理不等式的威力：这个看似简单的结论，在机器学习、生物信息学、统计学中有无数的应用。它意味着：不论你的特征工程做得多么精细，从原始数据中提取的特征X→处理后的特征Z，Z携带的关于原始标签的信息一定不超过X携带的。这也是为什么"原始数据"在深度学习时代如此重要——端到端学习避免了手工特征提取带来的信息损失。

信道容量定义的直觉

现在来到了全章最核心的概念——信道容量。

对于一个信道（由转移概率 p(y|x) 定义），信道容量C定义为：

C = maxp(x) I(X; Y)

其中最大值遍历所有可能的输入分布p(x)。

为什么是max？因为我们可以选择如何编码——通过调整输入符号的使用频率，我们可以最大化信道每使用一次能"安全"传输的信息量。

为什么是I(X;Y)？因为互信息度量的是"通过观察Y，我们消除了关于X的多少不确定性"。这正是信道"传递"的信息量。

举个具体的例子——BSC(p=0.1)的信道容量是多少？

对于BSC，由对称性可知，最优输入分布是均匀分布：p(0)=p(1)=0.5

在均匀输入下：
  H(Y) = H(0.5) = 1 bit  （输出也是均匀的）
  H(Y|X) = H(p) = H(0.1) = 0.469 bit （给定输入，输出分布的熵）

所以：
  I(X;Y) = H(Y) - H(Y|X) = 1 - 0.469 = 0.531 bits/次信道使用

事实上，对于BSC(p)，信道容量 C = 1 - H(p) bits/次使用。

这个结果非常直观：一个误码率10%的信道，每次使用最多可以安全传输0.531比特的信息。不管你用多好的纠错码，这个上限不可突破。

我们通过Python来验证一下不同信道模型的互信息计算：

import numpy as np

def mutual_information(channel_matrix, p_x):
    """
    计算给定信道矩阵和输入分布下的互信息 I(X;Y)
    channel_matrix: shape (r, s)，第i行是 p(y|x_i)
    p_x: shape (r,)，输入分布
    """
    r, s = channel_matrix.shape
    # 联合分布 p(x,y)
    p_xy = np.zeros((r, s))
    for i in range(r):
        p_xy[i, :] = p_x[i] * channel_matrix[i, :]

    # 边缘分布 p(y)
    p_y = p_xy.sum(axis=0)

    # I(X;Y) = ∑∑ p(x,y) log(p(x,y)/(p(x)p(y)))
    I = 0.0
    for i in range(r):
        for j in range(s):
            if p_xy[i,j] > 1e-15:
                I += p_xy[i,j] * np.log2(
                    p_xy[i,j] / (p_x[i] * p_y[j])
                )
    return I

# BSC(p) 信道
def bsc_matrix(p):
    return np.array([[1-p, p],
                     [p, 1-p]])

print("BSC(p=0.1) 在不同输入分布下的互信息：")
for p0 in [0.1, 0.3, 0.5, 0.7, 0.9]:
    p_x = np.array([p0, 1-p0])
    I = mutual_information(bsc_matrix(0.1), p_x)
    print(f"  p(0)={p0:.1f}: I(X;Y) = {I:.4f} bits/use")

print("\n理论最大值（均匀分布时）: C = 1 - H(0.1) = 0.5310")

# 各种常见信道模型对比
print("\n常见信道模型的容量对比：")
print(f"BSC(p=0.01): C = {1 - -np.log2(0.01) - 0.99*np.log2(0.99):.4f}")
print(f"BSC(p=0.1):  C = {1 - -0.1*np.log2(0.1) - 0.9*np.log2(0.9):.4f}")
print(f"BEC(ε=0.1): C = {1-0.1:.4f} (= 1-ε)")
print(f"BEC(ε=0.5): C = {1-0.5:.4f} (= 1-ε)")

# 输出：
# BSC(p=0.1) 在不同输入分布下的互信息：
#   p(0)=0.1: I(X;Y) = 0.3499 bits/use
#   p(0)=0.3: I(X;Y) = 0.5026 bits/use
#   p(0)=0.5: I(X;Y) = 0.5310 bits/use  ← 最大值
#   p(0)=0.7: I(X;Y) = 0.5026 bits/use
#   p(0)=0.9: I(X;Y) = 0.3499 bits/use

看到了吗？当输入分布偏离均匀分布时，互信息下降了——这就是为什么信道容量定义中需要取最大值。我们"有权"选择最优的输入分布来最大化传输速率。

信道模型 输入 → 输出 参数 信道矩阵形式 容量公式
BSC {0,1} → {0,1} p = 误码率 [[1-p, p], [p, 1-p]] C = 1 - H(p)
BEC {0,1} → {0,?,1} ε = 删除概率 [[1-ε, ε, 0], [0, ε, 1-ε]] C = 1 - ε
Z信道 {0,1} → {0,1} q = 1→0的概率 [[1, 0], [q, 1-q]] 需数值计算
AWGN ℝ → ℝ SNR = 信噪比 连续输出，高斯噪声 C = ½ log₂(1+SNR)
信道容量的直觉：它是这个信道"每小时能安全通行多少辆车的上限"——不管你怎么设计车（编码），都不可能超过这个速率。香农告诉我们：只要你的传输速率低于容量，就可以做到任意低的错误概率。但高于容量时，错误率必然被某个正数所约束。

信道容量是信息论最著名的公式之一——它刻画了一个通信信道的"信息吞吐能力"上限。这个上限不依赖于任何具体的编码方案，只取决于信道本身的物理特性。

但这里有一个实际问题：对于一个复杂的信道（比如不对称的、输入输出字母表很大的），我们怎么计算C = max I(X;Y)？直接暴力搜索输入空间的维度太高了。这个问题曾经困扰信息论学者几十年——直到1972年Blahut和Arimoto各自独立发现了一种优雅的迭代算法。
