第3章 插值法基础

大家好，我是你们的讲师。今天我们来聊一个非常直观但又极其深刻的问题：手头只有一些离散的数据点，能不能猜出它们之间的值？

想象你是一个18世纪的探险家，在海上航行时每隔一小时记录一次船的位置。现在你想知道"半小时前船在哪里"，但又没有GPS——你就需要一种方法，根据已知的离散数据点来推测未知位置的值。这就是插值（interpolation）问题。

🔍 日常类比： 当你在Excel里选中两列数据点右键"添加趋势线"时，你就在做插值。当Photoshop在你放大图片时"补全"像素时，它也在做插值。插值是现代科学计算和计算机图形学的基石之一。

这一章我们会建立插值问题的数学框架，学完本章后，你将能回答：给定 n+1 个点，存在多少个多项式能完美穿过它们？答案只有一个，而且我们能用Lagrange方法显式地把它写出来。

3.1 插值问题的定义

给定 n+1 个互异的点 (x₀, y₀), (x₁, y₁), ..., (xₙ, yₙ)，其中 xᵢ 互不相同。我们要找一个多项式 p(x)，满足：

p(xᵢ) = yᵢ, i = 0, 1, ..., n

且 p(x) 的次数 ≤ n。

唯一性定理：满足上述条件的多项式存在且唯一。

证明思路非常简单：假设存在两个不同的 n 次多项式 p 和 q 都满足插值条件，那么它们的差 r(x) = p(x) − q(x) 是一个次数 ≤ n 的多项式，却有 n+1 个不同的根 x₀, x₁, ..., xₙ。根据代数基本定理，一个 n 次非零多项式最多有 n 个根，所以 r(x) 必须是零多项式，即 p(x) ≡ q(x)。

💡 理解唯一性的意义： 插值多项式是唯一的，但它的表达形式可以多种多样。Lagrange形式、Newton形式、以及后面会学的各种形式——它们只是同一只猫的不同包装。

3.2 Lagrange插值多项式

Lagrange插值的核心思想是构造一组"开关函数"——每个基函数 Lᵢ(x) 在自己对应的节点 xᵢ 上取值为1，在其他所有节点上取值为0。

Lagrange基函数定义为：

Lᵢ(x) = ∏ⱼ₌₀,ⱼ≠ᵢⁿ (x − xⱼ) / (xᵢ − xⱼ)

然后插值多项式就是它们的线性组合：

pₙ(x) = Σᵢ₌₀ⁿ yᵢ · Lᵢ(x)

实例：已知三点 (0, 1), (1, 3), (2, 7)，求二次插值多项式。

# Lagrange插值的Python实现
def lagrange_interp(x_points, y_points, x):
    """计算Lagrange插值多项式在x处的值"""
    n = len(x_points)
    result = 0.0
    for i in range(n):
        term = y_points[i]
        for j in range(n):
            if i != j:
                term *= (x - x_points[j]) / (x_points[i] - x_points[j])
        result += term
    return result

# 验证三点插值
x_pts = [0.0, 1.0, 2.0]
y_pts = [1.0, 3.0, 7.0]
for x in [0.0, 0.5, 1.0, 1.5, 2.0]:
    y = lagrange_interp(x_pts, y_pts, x)
    print(f"p({x:.1f}) = {y:.4f}")

# 输出：p(0.0)=1.0, p(0.5)=2.0, p(1.0)=3.0, p(1.5)=4.5, p(2.0)=7.0
# 恰好是 p(x) = x² + 1

3.3 插值余项与误差估计

现在问题是：如果被插值函数 f(x) 本身不是多项式，那么误差有多大？

设 f ∈ Cⁿ⁺¹[a, b]，pₙ(x) 是 f 在节点 x₀, ..., xₙ 上的 n 次插值多项式。对任意 x ∈ [a, b]，存在 ξ(x) ∈ (a, b) 使得：

f(x) − pₙ(x) = f⁽ⁿ⁺¹⁾(ξ) / (n+1)! · Πᵢ₌₀ⁿ (x − xᵢ)

这就是著名的插值余项定理。它的实际意义是：

  • 误差与 f⁽ⁿ⁺¹⁾ 的大小有关——函数越"不平滑"，误差越大。
  • 误差与 Π(x − xᵢ) 的大小有关——节点分布越合理，误差越小。
  • 增加节点并不一定减少误差——这为我们即将看到的Runge现象埋下了伏笔。

误差上界可以写成：

|f(x) − pₙ(x)| ≤ Mₙ₊₁ / (n+1)! · max|Π(x − xᵢ)|

其中 Mₙ₊₁ = max|f⁽ⁿ⁺¹⁾(x)| 在区间 [a, b] 上。

🔍 实用洞察： 如果 f 是 sin(x) 或 eˣ 这样的函数，它的高阶导数是有界的，那么随着 n 增加，误差确实会减小（因为 (n+1)! 在分母上飞速增长）。但如果 f 是 1/(1+25x²) 这样的函数——它的高阶导数会随着 n 的增长爆炸——增加节点反而会让误差变大。这就是著名的Runge现象。

3.4 Runge现象——高阶插值的黑暗面

1901年，德国数学家Carl Runge发现了一个反直觉的现象：对函数 f(x) = 1/(1 + 25x²) 在 [-1, 1] 上用等距节点做高次多项式插值，随着节点数增加，插值多项式在区间两端会出现剧烈的振荡，误差不但不减小反而趋于无穷。

import numpy as np

def runge(x):
    return 1.0 / (1.0 + 25.0 * x**2)

def lagrange_interp(x_points, y_points, x):
    n = len(x_points)
    result = 0.0
    for i in range(n):
        term = y_points[i]
        for j in range(n):
            if i != j:
                term *= (x - x_points[j]) / (x_points[i] - x_points[j])
        result += term
    return result

# 等距节点，n 从 5 到 15
for n in [5, 10, 15]:
    x_pts = np.linspace(-1, 1, n+1)
    y_pts = runge(x_pts)
    # 在靠近端点处测试误差
    x_test = 0.95
    true_val = runge(x_test)
    interp_val = lagrange_interp(x_pts, y_pts, x_test)
    err = abs(true_val - interp_val)
    print(f"n={n:2d}, p({x_test}) = {interp_val:.6f}, 真值 = {true_val:.6f}, 误差 = {err:.2e}")
⚠️ 踩坑故事： 某深度学习团队尝试用高次多项式拟合高维loss landscape上的采样点来做可视化，期望得到一个光滑的loss曲面。结果因为Runge现象，插值多项式在采样边界剧烈振荡，画出来的loss曲面完全失真，导致团队误判了优化路径。最后改用分段低次插值后，才得到了有意义的可视化结果。这个故事告诉我们：深度学习的"过拟合"和Runge现象本质上是同一个数学问题——高自由度模型在有限数据下的不稳定性。

Runge现象的原因可以从插值余项公式理解：等距节点下 Π(x − xᵢ) 在区间两端会变得非常大，而 Runge 函数的高阶导数在靠近端点处也急剧增长。两个大数相乘，误差自然爆炸。

解决方案有两种思路：一是换用Chebyshev节点（在端点处更密集），二是改用分段低次插值。这就是后两章要讲的内容。

方法 优点 缺点
高次多项式插值（等距节点） 整体光滑，理论优雅 Runge现象，对节点分布敏感
分段低次插值 稳定性好，无Runge现象 整体光滑性差（导数不连续）
Chebyshev节点插值 抑制Runge现象，收敛性好 节点位置由算法决定，不能随意选择
💡 思考题： Runge现象和深度学习中"过拟合"的本质联系是什么？两者都是模型自由度远大于数据有效信息量时产生的伪像。正则化（相当于限制多项式系数的大小）可以被理解为一种"Chebyshev节点式"的约束——通过引入偏差来抑制方差。

下一章，我们学习Newton插值法。你会看到，在不改变插值结果的前提下，Newton形式如何优雅地解决了"增加一个节点时全部重算"的问题——O(n²) vs O(n) 的差距，在实际工程中意味着天壤之别。
