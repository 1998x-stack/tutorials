开篇：一个让CFO差点辞职的LP解

大家好，我是你们的讲师。今天讲整数规划，我想用一个非常惨痛的真实故事开场。

2021年，一家私募基金找到我，说他们的投资组合优化模型"出了大问题"。他们的团队用线性规划（LP）来优化资产配置——给定 N 支股票，每支买多少股，使得在给定风险约束下的预期收益最大化。

"模型跑出来的结果非常好，"他们告诉我，"预期年化收益率28%，最大回撤控制在12%以内。我们按这个结果投了2个亿。"

我一看他们的模型就发现了一个致命问题：他们用了连续变量，但现实中股票必须以整手（100股为单位）购买。他们的 LP 解建议购买 137.4 手某支股票——但谁能买 0.4 手？

"你们没有加整数约束？"我问。

"加了整数约束求解器就跑不动了，所以我们把它改成了连续变量……然后把结果四舍五入到了整数。"

我深吸一口气。把 LP 解四舍五入到最近整数——这是新手最常见的错误之一。在这个案例中，四舍五入后的解违反了风险约束的幅度达到了15%，而且预期收益掉到了18%——"最优解"和"实际可用的解"之间有整整10个百分点的差距。

这个故事的核心教训是：LP ≠ ILP。整数规划不是线性规划的"简单扩展"——它是一个本质上更难的问题（NP-难 vs P）。用求解 LP 的思路去"近似"整数规划，往往得到令人失望的结果。但也正是因为这个难度，整数规划的工具和方法（割平面、分支定界、分支切割）才显得如此重要。

本章的定位：第12章是第三部分"整数规划与多面体理论"的起点。我们之前已经接触过具体问题的组合算法（网络流、匹配），但从现在开始，我们将建立一个更强大的通用框架——整数线性规划。这个框架可以表达几乎所有组合优化问题，但代价是求解难度大大增加。

整数规划的挑战

一个整数线性规划（Integer Linear Program, ILP）的标准形式是：

min  c^T x
s.t. Ax ≤ b
     x ≥ 0
     x ∈ ℤⁿ

对比LP 松弛（去掉整数约束 x ∈ ℤⁿ 改为 x ∈ ℝⁿ），两者有本质的区别：

  • LP 可以在多项式时间内求解（椭球法、内点法）
  • ILP 是 NP-难的（即使变量限制为 0-1 也是 NP-完全的）
  • LP 最优值 ≤ ILP 最优值（对于最小化问题），这个差值称为对偶间隙（Duality Gap）或整数间隙（Integrality Gap）

整数间隙（Integrality Gap）

对于给定的 ILP 问题，整数间隙定义为：

Integrality Gap = max_{instance I} ( ILP_opt(I) / LP_relax_opt(I) )
                 （对于最小化问题）

这个比值衡量了"LP 松弛给出的下界有多好"。如果比值接近1，说明 LP 松弛很强，用单纯形法得到的 LP 解几乎就是整数解。如果比值很大，说明 LP 松弛完全不能反映整数问题的难度。

问题类别 整数间隙 说明
二分图匹配 1 LP 松弛自动整数（全幺模矩阵）
网络流 1 约束矩阵是节点-边关联矩阵，全幺模
无容量设施选址 ∞ LP 松弛可能极其弱
TSP 2（MST 松弛） 需要大量有效不等式才能收紧
最大割 ≈ 2（三角不等式松弛） Goemans-Williamson 用 SDP 改善
四舍五入LP解：为什么它是个坏主意——LP 的最优解通常位于多面体的一个顶点（极点）上。当你四舍五入时，你离开了这个顶点。四舍五入后的点可能：① 不再是可行解（违反约束）；② 即使可行，也可能非常次优。正确的做法要么是使用整数规划专用解法（分支定界、割平面），要么是设计有理论保证的近似算法。绝对不要拍脑袋做四舍五入——这会让你的 CFO 血压飙升。

全幺模矩阵

有一些幸运的 ILP 问题，它们的 LP 松弛自动给出整数解。全幺模矩阵（Total Unimodularity, TU）是判断一个 LP 松弛是否"自动整数"的关键工具。

定义：一个整数矩阵 A 称为全幺模矩阵，如果 A 的每个方子阵的行列式属于 {0, +1, -1}。

这里的"每个方子阵"意味着：任意选取 k 行和 k 列（k=1,2,...,最小行列数）组成方阵，其行列式必须是 -1、0 或 1。特别地，每个矩阵元素本身必须是 -1、0 或 1（因为 1×1 子阵就是元素本身）。

关键定理：如果约束矩阵 A 是全幺模矩阵，且右端向量 b 是整数向量，那么线性规划 min{c^T x | Ax ≤ b, x ≥ 0} 的所有基可行解（极点）都是整数解。

这意味着：对于全幺模矩阵描述的问题，我们不需要整数约束——LP 求解器自动给出整数最优解！

常见的全幺模矩阵

  • 有向图的节点-边关联矩阵：每列（对应一条边）有一个 +1（起点）和一个 -1（终点），其余为0。这是网络流问题中约束矩阵的形式。
  • 二分图的节点-边关联矩阵（无向）：每列（对应一条边）在两个端点处为1。当图是二分图时，这个矩阵是全幺模矩阵（一般的无向图关联矩阵不是全幺模矩阵）。
  • 区间矩阵：每列中的1是连续的（entirely 1 property）。例如调度问题中的某些约束。
记住这个判断：当你遇到一个组合优化问题，它的约束矩阵是 TU，那么恭喜你——这个问题可以在多项式时间内通过求解一个 LP 来解决。很多经典问题（网络流、二分图匹配、运输问题）都满足这个性质。这正是为什么我们之前花了那么多章讲这些问题的专用算法——它们背后共享同样的 TU 结构。

TU矩阵的判定与应用

Ghouila-Houri判定条件

如何判断一个矩阵是否全幺模？一个实用的充要条件是Ghouila-Houri 定理：

矩阵 A 是全幺模矩阵 ⇔ 对每个行子集 R，存在将 R 分为 R₁ 和 R₂ 的划分，使得对每一列 j，|∑_{i∈R₁} a_{ij} - ∑_{i∈R₂} a_{ij}| ≤ 1。

这个条件看起来复杂，但它刻画了 TU 矩阵的核心性质——每一列的数字可以"平衡"到相差不超过1的程度。用这个条件可以验证很多常见矩阵是否为 TU。

应用举例：运输问题

运输问题（Transportation Problem）是 TU 的经典案例。有 m 个仓库（供应量 s_i）和 n 个零售店（需求量 d_j），从仓库 i 到零售店 j 的单位运输成本为 c_{ij}。求总运输成本最小的方案。

变量：x_{ij} = 从仓库 i 到零售店 j 的运输量

目标：min ∑_{i=1}^{m} ∑_{j=1}^{n} c_{ij} x_{ij}

约束：
  ∑_{j=1}^{n} x_{ij} = s_i    ∀ i ∈ [m]  （仓库供应约束）
  ∑_{i=1}^{m} x_{ij} = d_j    ∀ j ∈ [n]  （零售店需求约束）
  x_{ij} ≥ 0                   ∀ i, j

这个问题的约束矩阵是 TU 的（实际上它是一个网络流问题的特例）。因此即使去掉整数约束，LP 最优解也是整数——前提是供应量和需求量都是整数。这告诉我们：运输问题不需要整数规划，用标准 LP 求解器就行。

一个重要的"但是"：TU 性质依赖约束的形式。如果你改变了问题的表述方式——例如在运输问题中加入了"每个路径的最大运输量限制"——约束矩阵可能就不再是 TU 了。换句说，TU 不是一个"问题内在性质"，而是"某种建模方式的属性"。同一个问题，换个不同的建模方式，可能从 TU 变成非 TU，或者反过来。这提醒我们：建模方式的选择至关重要。

割平面法

对于约束矩阵不是 TU 的问题（大多数实际问题都是这样），我们不能指望 LP 松弛自动给出整数解。这时就需要割平面法（Cutting Plane Method）——通过添加新的线性不等式（称为"割"或"有效不等式"）来逐渐收紧 LP 松弛，直到 LP 最优解恰好是整数。

Gomory割

Gomory 割是 Ralph Gomory 在1958年提出的一种通用割平面方法。它的美妙之处在于：不依赖具体问题的结构——任何 ILP 都可以用 Gomory 割来求解。

Gomory 割的推导基于单纯形表。假设我们用单纯形法求解 LP 松弛，得到最优基解，但某个基本变量 x_i 的取值不是整数。设 x_i 对应的单纯形表格行为：

x_i + ∑_{j} a_{ij} x_j = b_i

其中 b_i 不是整数

将 a_{ij} 和 b_i 分解为整数部分和小数部分：a_{ij} = ⌊a_{ij}⌋ + f_{ij}（其中 0 ≤ f_{ij} < 1），b_i = ⌊b_i⌋ + f_{i0}（0 < f_{i0} < 1）。

Gomory 割就是：

∑_{j} f_{ij} x_j ≥ f_{i0}

或者等价地：
∑_{j} (-f_{ij}) x_j ≤ -f_{i0}

这个不等式满足两个关键性质：

  1. 有效性：所有整数可行解都满足这个不等式（即它不会切掉任何整数解）
  2. 切割性：当前 LP 最优解（非整数的）不满足这个不等式
Gomory 割的数学直觉：等式 x_i + ∑a_{ij}x_j = b_i 表示 x_i 和其他变量的关系。因为 x_i 必须是整数，所以等号右边的小数部分 f_{i0} 必须由左边的小数部分"抵消"。这个抵消条件自然导出了一个线性不等式。我个人认为，Gomory 割是"从代数推导中自然涌现的几何结构"的最佳范例之一。

迭代割平面法

割平面法的基本流程是：

  1. 求解 LP 松弛，得到最优解 x*
  2. 如果 x* 是整数，终止（已找到 ILP 最优解）
  3. 否则，生成一个割平面（如 Gomory 割），加入 LP
  4. 回到第1步

这个过程在有限步内终止（Gomory 在1960年证明了有限收敛性），但最坏情况可能需要指数多步。不过在实践中，Gomory 割加上分支定界（即分支切割法，下一章）的效果通常很好。

方法 优点 缺点
纯 Gomory 割平面 通用，不需要问题特定结构 收敛慢，数值稳定性差
纯分支定界 稳定，实现简单 搜索树可能爆炸
分支切割（割+分支） 结合两者优势 实现复杂
专用有效不等式 收敛极快 需要问题特定知识

Chvátal-Gomory闭包

割平面法的理论深度体现在Chvátal-Gomory 闭包（Chvátal-Gomory Closure）的概念中。

CG割的生成规则

给定一个多面体 P = {x | Ax ≤ b, x ≥ 0}，一个Chvátal-Gomory 不等式（简称 CG 割）的生成方式是：

  1. 取一组非负乘子 y，构造线性组合 y^T A x ≤ y^T b
  2. 由于 x ≥ 0（或 x 有整数约束），我们可以对系数进行舍入：左边下取整得到更紧的不等式
CG 割的构造：

设 y ≥ 0 为行向量的非负线性组合

1. y^T A x ≤ y^T b      （原约束的线性组合）
2. ⌊y^T A⌋ x ≤ y^T b    （因为 x ≥ 0，舍入不会松弛不等式）
   （注意：⌊y^T A⌋ 是对每列系数下取整，但每个元素 x_j ≥ 0）
3. ⌊y^T A⌋ x ≤ ⌊y^T b⌋ （因为左侧是整数，右侧可下取整）

所有由不同 y 生成的 CG 不等式的交集，称为多面体 P 的一次 CG 闭包。这个闭包仍然是一个多面体。

CG秩

从原多面体 P 出发，反复取 CG 闭包，我们得到一族逐渐收缩的多面体：

P = P₀ ⊇ P₁ ⊇ P₂ ⊇ ... ⊇ P_I

其中 P_I 是整数可行解的凸包。

CG秩（Chvátal Rank）是使得 P_k = P_I 的最小 k。对于某些问题（如二分图匹配），CG 秩为 1（一次 CG 闭包就得到整数凸包）。对于某些问题（如 TSP），CG 秩可以非常大。

理论中的优雅，实践中的无奈：虽然任何 ILP 的整数凸包都可以通过有限次 CG 闭包得到，但 CG 秩可以指数大。这意味着纯割平面法在最坏情况下的效率没有保证。这正是分支切割法（下一章）存在的理由——通过分支将问题分解为更小的子问题，在每个子问题中使用割平面来收紧松弛，两者协同工作，比任何单独的方法都强大。

1988年，Schrijver 证明了 TSP 的 CG 秩至少为 Ω(√n)，这意味着对 TSP 来说，纯割平面法需要指数多个 CG 层才能收敛。这解释了为什么现代 MIP 求解器不使用纯割平面法，而是结合了分支和切割。

本章小结与下章预告

在这一章中，我们学习了：

  • 整数规划为什么比 LP 难——整数间隙、四舍五入的陷阱、ILP 的 NP-难性；
  • 全幺模矩阵——方子阵行列式 ∈ {-1,0,1}，保证 LP 松弛的整数性，运输问题等经典案例；
  • TU 矩阵的判定——Ghouila-Houri 条件，及 TU 与具体建模方式的相关性；
  • 割平面法与 Gomory 割——从分数最优解推导出切除分数解但不切整数解的不等式；
  • Chvátal-Gomory 闭包——CG 不等式、CG 秩、有限终止性但指数下界。
