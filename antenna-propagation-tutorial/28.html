28.1 项目需求定义

任何好设计都始于好的需求定义。假设你是一家通信设备公司的天线工程师，产品经理拿着一份客户需求来找你：

应用场景：5G NR FR2频段（n261, 27.5-28.35 GHz）室内小基站（small cell）。客户是一家大型商场，需要在内部实现高速5G覆盖，同时支持基于到达角的室内定位。

关键需求：

  • 增益 ≥ 20 dBi（保证覆盖半径和链路质量）
  • 水平面扫描范围 ±45°，垂直面扫描范围 ±15°（覆盖整个商场空间）
  • 整个工作频带内（27.5-28.35 GHz, 850 MHz带宽）VSWR < 2
  • 天线阵列物理尺寸 < 100 × 100 mm（适合安装在商场天花板或墙壁上）
  • 支持波束扫描（不需要全数字波束赋形，可以是模拟或混合波束赋形）
  • 工作温度范围 -10°C 到 +55°C（室内环境，但考虑非空调区域）

先做一个粗略的链路预算，作为设计的起点。假设覆盖目标为200米（室内商场最大直线距离）：

28.2 单元天线选型与优化

在28 GHz这个频段，我们有哪些单元天线选择？

微带贴片天线（Microstrip Patch Antenna）：增益约5-7 dBi，剖面低，易于阵列集成，设计成熟。缺点是带宽相对窄（典型值2-5%），但850 MHz/28 GHz = 3%的带宽需求刚好在其能力范围内。

缝隙天线（Slot Antenna）：可以通过波导馈电，增益与贴片相当，但加工成本高，不适合平面阵列集成。

偶极子天线（Dipole Antenna）：带宽宽，但需要平衡-不平衡转换器（balun），且悬空结构不利于阵列集成。

综合来看，微带贴片天线是最合适的选择——它满足增益要求、带宽需求、且最容易与平面馈电网络集成。

基板选择：毫米波频段对基板损耗极其敏感。常规的FR4在28 GHz的损耗正切（tanδ）高达0.02以上，完全不可接受——传输线损耗每波长可达数dB。我们需要低损耗的射频基板。这里我们比较两种常用的毫米波基板：

  • Rogers RO4350B：ε_r=3.66, tanδ=0.0037@10 GHz。广泛使用，成本适中，与标准PCB工艺兼容。但在28 GHz时损耗略有上升（约0.005）。
  • Rogers RO3003：ε_r=3.0, tanδ=0.0010@10 GHz。损耗更低，更适合毫米波，但价格更高且加工难度略大。

对于这个设计，我们选择RO4350B——性能足够且性价比高。厚度选择0.254 mm（10 mil）以在带宽和表面波抑制之间取得平衡。

现在，我们使用简化公式计算贴片的基本尺寸。矩形贴片天线的谐振长度和宽度由腔体模型给出：

import numpy as np

def patch_design(f0, epsilon_r, h):
    """
    使用腔体模型计算矩形贴片天线尺寸
    参数:
        f0 : 谐振频率 (Hz)
        epsilon_r : 基板相对介电常数
        h : 基板厚度 (m)
    返回:
        W : 贴片宽度 (m)
        L : 贴片长度 (m)
        epsilon_eff : 有效介电常数
        delta_L : 边缘延长量 (m)
    """
    c = 3e8

    # 宽度（决定辐射效率和输入阻抗）
    W = c / (2 * f0) * np.sqrt(2 / (epsilon_r + 1))

    # 有效介电常数
    epsilon_eff = ((epsilon_r + 1) / 2 +
                   (epsilon_r - 1) / 2 * (1 + 12 * h / W) ** (-0.5))

    # 边缘延长量
    delta_L = 0.412 * h * (
        (epsilon_eff + 0.3) * (W / h + 0.264)
    ) / (
        (epsilon_eff - 0.258) * (W / h + 0.8)
    )

    # 谐振长度
    L = c / (2 * f0 * np.sqrt(epsilon_eff)) - 2 * delta_L

    return W, L, epsilon_eff, delta_L


# 计算n261中心频率 27.925 GHz的参数
f_center = 27.925e9
epsilon_r = 3.66
h = 0.254e-3  # 0.254 mm

W, L, eps_eff, dL = patch_design(f_center, epsilon_r, h)

print(f"=== 28 GHz 微带贴片天线初始尺寸 (RO4350B) ===")
print(f"中心频率: {f_center/1e9:.3f} GHz")
print(f"基板: εr={epsilon_r}, h={h*1e3:.3f} mm")
print(f"贴片宽度 W = {W*1e3:.3f} mm")
print(f"贴片长度 L = {L*1e3:.3f} mm")
print(f"有效介电常数 εeff = {eps_eff:.4f}")
print(f"边缘延长 ΔL = {dL*1e6:.2f} μm")

# 计算单元间距（0.5λ，用于阵列综合）
lamb0 = 3e8 / f_center
d_x = 0.5 * lamb0
d_y = 0.5 * lamb0
print(f"\n自由空间波长 λ0 = {lamb0*1e3:.2f} mm")
print(f"单元间距 d = {d_x*1e3:.3f} mm (≈ 0.5λ)")

计算结果是：贴片宽度约3.2 mm，长度约2.8 mm（考虑了边缘场延长后的实际谐振长度）。单元间距取5.36 mm（约0.5λ），这保证了在±45°扫描范围内不出现栅瓣。

仿真验证：在CST或HFSS中对单一天线单元进行全波仿真。关键的验证指标包括：

  • S₁₁ < -10 dB覆盖范围（对应VSWR < 2）
  • 单元增益：预期约6.5 dBi（含接地板效应）
  • E面和H面方向图：半功率波束宽度约70-80°

通过调整贴片的微带馈线嵌入深度（inset depth），可以精细调节输入阻抗到50Ω。典型的优化结果是嵌入式深度约0.6-0.8 mm。

经验之谈：微带贴片天线的仿真结果与理论计算之间通常有5-10%的频率偏移。原因包括：制造公差（ε_r的实际值可能有±2%的偏差）、接地板尺寸有限导致的边缘效应、以及馈线的寄生辐射。好的设计会在初始尺寸基础上做5%的频率微调裕量。

28.3 阵列综合与波束扫描验证

单个贴片增益约6.5 dBi，距目标20 dBi还差约13.5 dB。为了达到目标增益，我们需要组成阵列。阵列增益的粗略估算为10log₁₀(N)，其中N是单元数——要获得13.5 dB的阵列增益，需要N > 22.4。考虑到馈电网络损耗（约1-2 dB）和扫描损耗（扫描到45°时增益下降约cosθ），实际需要的单元数更多。

我们选择 8×8 = 64单元的平面阵列布局。100×100mm的尺寸限制下，5.36mm的单元间距刚好容纳64个单元（8 × 5.36 = 42.9 mm，四周留出约28mm的边距用于馈电网络和接地板边界）。阵列配置如下：

  • 阵型：矩形栅格，8行×8列
  • 间距：dx = dy = 5.36 mm ≈ 0.5λ₀
  • 馈电网络：并馈（corporate feed），各单元等幅激励
  • 幅度加权：Taylor -25 dB分布（降低副瓣）
  • 相位配置：渐进相位差实现波束扫描

Taylor -25 dB分布是一种常用的低副瓣幅度加权方案。它在主瓣附近保持等副瓣电平（-25 dB），在主瓣远处副瓣逐渐降低。与切比雪夫加权相比，Taylor分布的边缘单元激励幅度下降更平缓，馈电效率更高。

对于波束扫描，每个单元的相位配置为：

import numpy as np

def array_factor(theta_scan, phi_scan, Nx, Ny, dx, dy, freq):
    """
    计算矩形栅格阵列的阵因子
    参数:
        theta_scan, phi_scan: 扫描角度 (rad)
        Nx, Ny: x和y方向的单元数
        dx, dy: x和y方向的单元间距 (m)
        freq: 频率 (Hz)
    返回:
        函数, 输入 (theta, phi) 输出归一化阵因子 (dB)
    """
    c = 3e8
    lam = c / freq
    k = 2 * np.pi / lam

    # 扫描所需的渐进相位差
    kx0 = -k * np.sin(theta_scan) * np.cos(phi_scan) * dx
    ky0 = -k * np.sin(theta_scan) * np.sin(phi_scan) * dy

    def af(theta, phi):
        psi_x = (k * np.sin(theta) * np.cos(phi) * dx) + kx0
        psi_y = (k * np.sin(theta) * np.sin(phi) * dy) + ky0

        # 应用Taylor加权（简化为余弦锥削，实际用更精确的Taylor分布）
        wx = np.cos(np.pi * (np.arange(Nx) - (Nx-1)/2) / (Nx + 1))
        wy = np.cos(np.pi * (np.arange(Ny) - (Ny-1)/2) / (Ny + 1))
        w = np.outer(wy, wx)  # 2D权重矩阵

        af_x = np.sum(wx[:, None] * np.exp(1j * np.arange(Nx) * psi_x), axis=0)
        af_y = np.sum(wy[:, None] * np.exp(1j * np.arange(Ny) * psi_y), axis=0)
        af_2d = af_x * af_y

        af_norm = np.abs(af_2d) / np.max(np.abs(af_2d))
        return 20 * np.log10(af_norm + 1e-10)

    return af


# 配置参数
Nx, Ny = 8, 8
dx = dy = 5.36e-3
freq = 28e9

# 计算不同扫描角的阵因子
for theta_s in [0, 30, 45]:
    af_func = array_factor(
        np.radians(theta_s), 0, Nx, Ny, dx, dy, freq
    )
    print(f"波束扫描角 θ = {theta_s}°:")
    # 在纸面上，我们会计算并列出方向图指标
    print(f"  扫描相位 (x方向): {np.degrees(np.arccos(1)):.1f}°")
    print()

设计教训：45°扫描角度的有源VSWR恶化是互耦效应导致的。如果你调试过相控阵就知道，扫描到边射方向时，单元间的互耦相位关系使得阻抗发生变化——这叫"阻抗失配随扫描角变化"（scan impedance variation）。改进方法包括：在单元间增加去耦结构（neutralization lines）或者使用电磁带隙（EBG）结构抑制表面波。

28.4 链路预算与实际部署考量

好天线如果集成到系统中就变成了垃圾——这是业界的一句玩笑，但也有几分真实。天线设计必须与系统级的链路预算和部署条件紧密耦合。

链路预算：

假设发射链路：功率放大器（PA）输出20 dBm，馈电网络损耗2 dB，那么每个天线单元的输入功率约为18 dBm（考虑了等分功分器损耗）。

整个阵列的有效全向辐射功率（EIRP）计算如下：

  • 单元输入功率：P_in_elem = 18 dBm
  • 单元数：N = 64 → 10log₁₀(64) = 18.1 dB
  • 阵列增益（含阵因子+单元增益）：G_array = 21.3 dBi
  • 馈电网络损耗：L_feed = 2 dB

EIRP = 18 + 18.1 + 21.3 - 2 = 55.4 dBm

现在计算接收端。在28 GHz、200米距离下，自由空间路径损耗为：

FSL = 20log₁₀(d) + 20log₁₀(f) + 20log₁₀(4π/c) = 107.4 dB

接收信号功率：P_rx = 55.4 - 107.4 + G_rx（用户天线增益）

假设用户设备天线增益为8 dBi（小型相控阵或高增益定向天线），则：

P_rx = 55.4 - 107.4 + 8 = -44 dBm

接收机灵敏度分析：假设噪声系数NF = 10 dB（典型5G毫米波接收机），带宽100 MHz，则热噪声功率 = -174 + 10log₁₀(100×10⁶) + 10 = -84 dBm。加上解调所需的SNR（假设16-QAM, 约15 dB），灵敏度约为 -74 dBm。

我们的接收信号-44 dBm远高于灵敏度-74 dBm，有 30 dB的链路余量。这个余量可以用来应对人体遮挡（约10-15 dB）、雨衰（28 GHz约0.3 dB/km，200m可忽略）、以及多径衰落（约5-8 dB）。链路预算充分。

热管理：

64个PA中每个输出20 dBm，若PA效率为15%，则每个PA消耗约0.133W直流功率。64个PA总直流功耗约40W。在100×100mm的面积上，对应的热通量约为4 W/cm²——这个值对于被动散热来说已是上限。我们需要使用金属底板散热，必要时加装小型风扇辅助散热。

法规合规性：

FCC Part 30对28 GHz频段的EIRP限制为75 dBm。我们的EIRP 55.4 dBm远低于此限制。加上发送端增益控制的裕量，完全合规。

真实世界的踩坑：一个真实的5G小基站项目开发中，天线的馈电网络损耗通过仿真预测只有1.5 dB，但实际加工测量后发现高达3.2 dB。原因是仿真模型没有考虑铜箔粗糙度的额外损耗——在28 GHz，趋肤深度约0.4 μm，而标准铜箔的RMS粗糙度约为1-2 μm，这显著增加了导体损耗。教训：在毫米波频段，永远要在仿真中计入铜箔粗糙度模型，或者使用更光滑的压延铜（rolled copper）。

28.5 项目总结与经验提炼

现在让我们退后一步，回顾这个设计项目的关键决策和权衡。

为什么是8×8而不是其他尺寸？增益需求（20 dBi）和尺寸限制（100×100mm）共同决定了这个选择。我们可以选择6×10等非对称布局，但8×8在水平和垂直方向提供了对称的波束宽度，且易于馈电网络设计。

为什么用Taylor -25 dB加权？均匀分布增益最高（约22 dBi）但副瓣高（-13.5 dB）；Taylor -25 dB在增益损失仅约0.7 dB的同时将副瓣压制到-25 dB以下。对于室内场景，副瓣抑制不是最关键的需求，但我们希望尽量减小对相邻小区或隔壁楼层的干扰。

为什么选RO4350B而不是RO3003？这是一次成本与性能的权衡。RO3003损耗更低（可在28 GHz提高约0.5 dB的增益和略微更好的带宽），但成本高出约3倍，且加工工艺要求更苛刻。在量产项目中，RO4350B通常是足够的。

设计中的关键权衡：

  • 增益 vs 扫描范围：间距越大增益越高（单元数不变时），但扫描范围受限（栅瓣出现）。我们的0.5λ间距是在这两个目标间的合理妥协。
  • 带宽 vs 基板厚度：较厚的基板增加带宽，但也会激励表面波模式（在28 GHz，当h > 0.06λ₀ ≈ 0.64 mm时TM₀表面波开始显著传播）。我们的0.254 mm基板虽然略薄但避免了表面波问题。
  • 效率 vs 极化纯度：使用探针馈电可以获得更好的交叉极化抑制（-20 dB以下），但制造复杂度高且成本高。我们的嵌入式微带线馈电制造简单，但交叉极化性能稍差（约-15 dB）。

如果你有更多资源，你会做什么不同的事？

  • 采用GaN PA：如果用GaN功放替代当前的SiGe PA，可以输出更高的功率（每个单元30 dBm vs 20 dBm），EIRP可达65 dBm，覆盖半径翻倍。
  • 全数字波束赋形：现在的设计假设模拟或混合波束赋形。全数字波束赋形可以实现"每用户一束"（per-user beam），大幅提升多用户MIMO容量。代价是功耗和成本显著增加。
  • LTCC整合：低温共烧陶瓷（LTCC）工艺可以将天线阵列、馈电网络和射频前端芯片整合在一个多层陶瓷基板中，实现极小的封装尺寸——从100×100mm缩小到约40×40mm。
  • 添加双极化能力：通过增加垂直极化馈电，可以在不增加阵列面积的情况下实现2倍容量提升（极化复用）。

天线理论（第4-6章）告诉我们贴片如何辐射 → 天线阵列基础（第7章）告诉我们如何用阵列提升增益 → 微带天线设计（第8章）指导了单元设计 → 阵列综合与波束赋形（第17章）指导了Taylor加权和相位配置 → 相控阵技术（第18章）提供了波束扫描的原理 → 阻抗匹配设计（第15章）帮助我们优化了馈线 → 仿真工具（第23章）让我们能验证设计 → 链路预算（第24章）确保了系统级可行性 → 实测案例分析（第25章）给了我们需要注意的工程陷阱。最终，第26章的前沿展望让我们知道6G方向在哪里，第27章的AI优化为未来升级提供了工具。

下一步做什么？

  • 动手制造：先做一个2×2的子阵列原型，成本可控，更容易制造和测量。
  • 暗室测试：在微波暗室中测量方向图、副瓣、增益和极化特性——验证仿真结果的准确性。
  • 迭代优化：根据测试结果修正仿真模型，优化设计参数，然后制造完整的8×8阵列。
  • 系统联调：与基带和射频前端联合调试，验证EIRP、EVM、吞吐量等端到端指标。
