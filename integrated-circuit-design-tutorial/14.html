第14章 物理设计

大家好，我是你们的讲师。在之前的所有章节中，芯片一直存在于抽象的层面——RTL代码、门级网表、时序约束。但所有这些抽象的终点只有一个：变成一张可以送到晶圆厂去制造的光刻掩模版图（GDSII）。完成这个从逻辑到物理的转换过程的，就是物理设计（Physical Design）——一个也被称为"后端设计"或"布局布线"的阶段。

物理设计是芯片流片前的最后一站。在这里，你的每一个设计决策都直接影响良率和性能——放错一个宏单元可能导致整颗芯片的时序无法收敛，画错一条电源带可能导致芯片烧毁。物理设计工程师是芯片诞生的"助产士"——他们让逻辑设计在硅片上"活"起来。这一章，我们来走完这块芯片从网表到GDSII的最后一段路。

14.1 布局规划

Floorplan（布局规划）是物理设计的第一步——相当于城市规划师决定"住宅区放哪里、商业区放哪里、工业区放哪里"。一个差的floorplan会让后续的place和route怎么做都做不好。

Floorplan的主要任务：

  • 芯片面积估算：根据综合后的标准单元总面积、宏单元（SRAM/ROM/Analog IP）面积、以及布线资源开销，估算芯片所需的die面积。通常标准单元的总利用率在60-80%之间，剩下的面积预留给布线通道和decap电容。
  • I/O Pad规划：根据封装类型和信号流向，确定I/O pad的位置。高速信号靠近芯片边缘、模拟信号远离数字噪声源、供电pad均匀分布。
  • 宏单元/存储器放置：大块的SRAM和IP模块需要优先放置——它们的位置直接影响芯片的形状和面积利用率。宏单元通常放在芯片边缘或角落，中间区域留给标准单元。
  • Block Partitioning：对于分层设计（hierarchical design），将芯片划分为多个物理block，每个block独立做place和route，然后在顶层整合。分层设计可以大幅减少单次运行的复杂性，但block之间的接口规划难度较大。
  • 电源网格规划：在floorplan阶段就要画出主要的电源走线（power strap/ring/mesh），确保电流能够均匀地输送到芯片的每个角落。IR drop（电压降）的预算通常控制在VDD的3-5%以内。

一个好的floorplan可以为后续所有步骤节省数周的迭代时间。反之，一个拍脑袋决定的floorplan——比如把CPU核放在了芯片的最远端、把高速存储器放在了I/O pad的另一边——会在后续阶段造成无穷无尽的时序和布线问题。

14.2 电源规划

电源网络（Power Grid）是芯片的"血管"——它把电流从焊盘输送到每一个晶体管的源极。如果电源网络设计不当，芯片内部的电压降（IR Drop）可能导致逻辑门得不到足够的工作电压，从而引发功能失效。

典型的电源网络结构：

  • Power Ring（电源环）：围绕芯片或block外围的环形金属走线。焊盘通过bond wire或bump连接到power ring。
  • Power Strap（电源带）：跨过芯片内部的横向或纵向金属走线，连接power ring到芯片内部区域。
  • Power Mesh（电源网格）：由多层金属（M1-M顶）组成的纵横交叉的电源走线网络。顶层金属（厚金属）走主干，中间层金属做次干，底层金属（M1）连接到标准单元的VDD/VSS引脚。

在电源规划中需要特别关注电迁移（Electromigration, EM）——当金属导线中的电流密度过大时，电子会"撞击"金属原子，导致金属原子沿电流方向迁移，最终造成导线断裂或短路。EM是一个可靠性问题——芯片可能在出厂时是好的，但使用了几个月或几年后突然失效。EM约束限制了每段金属走线的最大允许电流密度。

Decap电容（去耦电容）的放置也是电源规划的重要部分——它们在芯片内部提供局部电荷存储，当逻辑门瞬态切换需要大量电流时，decap可以"就近"提供电荷，减少IR drop。通常芯片面积的10-20%会被decap电容占用。

14.3 布图与时钟树综合

标准单元布局（Placement）

在floorplan确定之后，需要把标准单元（与门、或门、触发器、MUX等）摆放到芯片内部。这分为两个阶段：

  • 全局布局（Global Placement）：粗略地确定每个标准单元应该放在哪个区域，不考虑详细的合法性（比如单元之间是否重叠）。
  • 详细布局（Detail Placement）：在全局布局的基础上进行微调，解决合法性问题——消除重叠、对齐row、确保没有单元超出边界。

布局优化的目标是：最小化所有连接的总线长（wirelength）、确保时序收敛、避免局部拥塞区域。

时钟树综合（Clock Tree Synthesis, CTS）

时钟信号需要被同时送到芯片上成千上万个触发器的时钟引脚——而且要在几乎相同的时间到达。CTS就是设计这个"时钟分发网络"的过程。

常见的时钟树拓扑：

  • H-Tree：递归的"H"形结构，每个分支对称，使信号到达每个叶节点的距离相等。适合规则阵列分布。
  • Fishbone（鱼骨形）：一条主干时钟线横穿芯片，两侧延伸出分支。适合长窄型芯片。
  • 平衡树（Balanced Tree）：通过插入buffer使所有分支的延迟相等——不管位置在哪里，延迟相同。

CTS的核心指标是时钟偏斜（Clock Skew）——时钟信号到达不同触发器的最大时间差。目标通常是将skew控制在时钟周期的5%以内。现代CTS工具还可以做useful-skew——有意地制造skew来改善关键路径的时序。

14.4 布线

所有标准单元和宏单元都摆放好了，但它们之间还是"孤岛"——需要通过金属互连线连接起来。这就是布线（Routing）的工作。

布线分三个阶段：

  • 全局布线（Global Routing）：把芯片划分为规则的网格（grid），为每条线分配经过哪些网格，但不决定具体的金属层和走线位置。
  • 轨道分配（Track Assignment）：把全局布线的结果映射到具体的金属走线轨道上。
  • 详细布线（Detail Routing）：完成每条线的具体金属走线——确定通孔（via）位置、金属宽度、走线间距，确保满足所有DRC规则。

布线阶段会遇到各种问题：

  • DRC Violation：金属间距不够、宽度不足、通孔堆叠违规等——需要反复修正。
  • 天线效应（Antenna Effect）：长金属线在等离子体刻蚀过程中会像"天线"一样收集电荷，可能击穿栅氧化层。解决方法是插入天线二极管（antenna diode）提供电荷泄放路径。
  • 拥塞（Congestion）：局部区域走线过密导致无法完成布线——通常需要回到placement阶段重新调整。
  • ECO布线：在芯片设计的最后阶段，由于时序ECO（小范围网表修改）导致的局部布线修改，需要在不影响已有布线的条件下完成。

14.5 寄生提取与签核

布线完成后，就可以进行物理设计的最后一步——签核（Signoff）。在把GDSII送到晶圆厂之前，需要通过一系列极其严格的检查。

寄生参数提取（Parasitic Extraction）：

布线完成后，每条金属线都有真实的电阻（R）、电容（C）和耦合电容（C_c）。这些值需要通过工具提取出来——常用的工具有Cadence QRC和Synopsys StarRC。提取精度分为2.5D和3D两种——3D提取最准确但最慢，通常只对关键路径使用。提取结果以SPEF（Standard Parasitic Exchange Format）格式输出，供STA工具做post-layout时序分析。

物理设计阶段 主要任务 常用EDA工具 输出文件
Floorplan 面积估算, I/O规划, 宏单元放置, 电源网格规划 Innovus, ICC2, Fusion Compiler .floorplan, .def
Power Planning 电源环/带/网格设计, decap放置, EM检查 Innovus, ICC2, RedHawk .def, .spf
Placement 全局布局, 详细布局, 拥塞优化 Innovus, ICC2 .def
CTS 时钟树综合, skew优化, useful-skew CTS (Innovus/ICC2), EZwave .def, .sdc
Routing 全局布线, 详细布线, DRC修复, 天线修复 Innovus NanoRoute, ICC2 .def

Physical Verification DRC, LVS, ERC, ANT Calibre, ICV .drc.rpt, .lvs.rpt

物理验证（Physical Verification）包括：

  • DRC（Design Rule Check）：检查版图是否符合晶圆厂的所有设计规则——最小间距、最小宽度、最小包围等。一颗28nm芯片的DRC规则有数千条，7nm芯片超过数万条。
  • LVS（Layout vs. Schematic）：检查版图中的电路连接是否和网表一致——连对了，没有短路，没有断路。
  • ERC（Electrical Rule Check）：检查电气规则——比如浮空的输入引脚、短路的电源地、不合理的扇出等。

所有检查通过后，将GDSII文件发送给晶圆厂——这就是流片（Tape-out）。此时你手里只有一个文件（可能几百MB到几GB），但它代表了数十亿个晶体管和数百亿条连接的全部信息。

以下是一个时序签核的检查清单示例：

# ============================================
# Timing Signoff 检查清单
# (供PrimeTime运行后确认)
# ============================================

# 1. 确认使用的corner
#  - SS (0.81V, 125°C, slow) -> setup check
#  - FF (0.99V, -40°C, fast) -> hold check
#  - TT (0.90V,  25°C, typ)  -> functional check

# 2. 确认约束完整性
report_analysis_coverage     # 确认100%路径被覆盖
check_timing                 # 确认无unconstrained path
report_clock                 # 确认所有时钟正确定义

# 3. 确认时序收敛
report_sworst_timing -max_paths 100  # 检查WNS/TNS
report_sworst_timing -min_paths 100  # 检查hold WNS/TNS

# 4. 确认CPPR（共同路径悲观消除）
set_analysis_mode -cppr both
report_timing -cppr -nworst 10

# 5. 确认OCV设置
report_derate                        # 确认derate因子正确
report_analysis_mode -mcm            # 确认MCMM设置

# 6. 确认时钟门控时序
report_clock_gating -check setup
report_clock_gating -check hold

# 7. 确认异步路径已正确处理
report_false_path                    # 确认没有遗漏
report_disable_timing                # 确认没有遗漏

# 8. 确认跨时钟域路径
report_clock_interaction -setup      # 确认CDC路径正确

# 9. 确认signoff网表正确性
report_annotated_parasitics          # 确认SPEF注释率>95%
report_annotation_status -type sdf   # 确认SDF回注正确
一个关于IR drop的真实故事：某团队设计了一颗用于边缘计算的AI加速芯片，采用16nm工艺。在物理设计阶段，IR drop仿真显示静态IR drop为VDD的2.5%（在3%预算内），动态IR drop峰值为VDD的6.8%（超过了5%的预算，但团队认为"只有一小块区域超了，应该没问题"）。芯片流片回来后，在bench测试中——当芯片运行最大功耗负载（所有MAC阵列同时计算）时——芯片在几秒钟后随机出现计算错误。定位后发现：IR drop最严重的区域恰好是芯片时序最紧张的路径所在区域。当大负载启动时，局部电压从0.9V跌到了0.78V，导致这条路径的setup全面违例——芯片没有物理损坏，但计算结果完全错误。更棘手的是，这个电压跌落在功能测试中很难复现，因为ATE测试向量不会产生峰值功耗。这也是为什么越来越多SoC在芯片内部集成在线电压监控器来实时检测IR drop。修复方案？在下一版芯片中把电源网格加粗、增加decap、并在布局时把高功耗的MAC阵列分散布置。而这一版已经流片的芯片——只能降频使用。

好了，物理设计的知识就讲到这里。从Floorplan到Signoff，我们走完了数字芯片后端设计的全流程。现在你理解了芯片设计从Spec到GDSII的全貌——这是一个从抽象到具体、从逻辑到物理的渐进过程。但我们的课程还没有结束。从下一章开始，我们将进入芯片设计的另一个宏大世界——模拟IC设计基础。如果你以为数字芯片设计就够复杂了，模拟电路会给你带来完全不同的视角。敬请期待。
