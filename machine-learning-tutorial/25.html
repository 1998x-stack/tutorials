25.1 第三种学习范式

回顾一下：监督学习需要标注数据（每个样本有正确答案），无监督学习从无标注数据中发现结构。强化学习（Reinforcement Learning, RL）是第三种范式——通过和环境的交互来学习。智能体（Agent）在环境中执行动作（Action），环境反馈奖励（Reward），Agent的目标是最大化长期累积奖励。

RL最震撼的成就：AlphaGo击败李世石（2016），OpenAI Five在Dota 2中击败世界冠军（2019），DeepMind用RL控制核聚变反应堆（2022）。RL解决的是"序列决策"问题——除了当下这一步，还要考虑长远后果。

25.2 MDP：强化学习的数学框架

强化学习问题被建模为马尔可夫决策过程（MDP），包含五个要素：

要素符号例子（迷宫游戏）
状态 StateS智能体在迷宫中的位置坐标
动作 ActionA上、下、左、右
奖励 RewardR到达终点+10，每走一步-0.1
状态转移P(s'|s,a)在(2,3)向上走→到达(2,4)
折扣因子 γγ∈[0,1]γ=0.9：近期奖励比远期更重要

马尔可夫性质的意思是：未来只取决于现在，和过去无关。给定当前状态，历史状态不会提供额外的有用信息。这是RL理论的基础假设（虽然现实中经常被违背，但足够好用）。

25.3 Q-Learning：值函数方法

Q值的定义：在状态s执行动作a后，未来能获得的期望累积奖励。如果知道每个(s,a)的Q值，最优策略就是：在每个状态选Q值最大的动作。

Q-Learning的核心是贝尔曼方程，它把"长期价值"分解为"即时奖励 + 未来价值"：

Q(s,a) = r + γ · max Q(s', a')

这看起来像一个递归定义（Q值包含Q值），但正是这个递归结构使得我们能用动态规划思路迭代求解。在表格场景下（状态和动作都是离散且有限的），Q-Learning能收敛到最优解。

import gymnasium as gym
import numpy as np

env = gym.make('FrozenLake-v1', is_slippery=False)
Q = np.zeros([env.observation_space.n, env.action_space.n])

# Q-Learning算法
alpha, gamma, epsilon = 0.1, 0.99, 0.1
for episode in range(10000):
    state, _ = env.reset()
    done = False
    while not done:
        # ε-greedy：10%概率随机探索，90%概率选最优
        if np.random.random() < epsilon:
            action = env.action_space.sample()
        else:
            action = np.argmax(Q[state])

        next_state, reward, done, _, _ = env.step(action)

        # 核心更新：Q-Learning
        Q[state][action] += alpha * (
            reward + gamma * np.max(Q[next_state]) - Q[state][action]
        )
        state = next_state

print(f"训练完成，初始状态的Q值: {Q[0]}")

25.4 DQN：深度学习 + Q-Learning

表格型Q-Learning的问题是：当状态空间很大（比如Atari游戏的屏幕像素），表格根本存不下。DQN用神经网络来近似Q函数，输入是状态，输出是每个动作的Q值。

DQN的两个关键创新：

  • 经验回放（Experience Replay）：把经验(s,a,r,s')存到缓冲区，随机采样训练——打破数据间的时序相关性
  • 目标网络（Target Network）：用两个网络：在线网络输出选动作的Q值，目标网络输出目标Q值。目标网络定期从在线网络复制权重。避免训练不稳定（"追自己的尾巴"）

25.5 Policy Gradient：直接优化策略

Q-Learning间接学习策略（先学Q值，再从Q值导出策略）。策略梯度方法直接学习策略函数π(a|s)——给定状态，输出每个动作的概率分布。

REINFORCE是最基础的策略梯度算法：执行一个完整的回合（episode），然后根据获得的总奖励调整策略——好的动作（导致高奖励）增加概率，差的动作减少概率。

PPO（Proximal Policy Optimization）是当前最流行的策略梯度算法。核心改进：限制每次更新的幅度（用clip函数），防止策略变化太剧烈导致训练崩溃。PPO是OpenAI在各种任务上的默认算法，也是RLHF（用RL微调语言模型）的基础。

🔑 Value-based vs Policy-based：值函数方法（Q-Learning, DQN）适合离散动作空间（左/右/跳），间接得到策略。策略梯度方法适合连续动作空间（方向盘角度、机械臂力矩），直接输出动作。现代RL的王者Actor-Critic结合两者：一个网络（Actor）输出策略，另一个（Critic）估计Q值辅助Actor训练。PPO、A3C、SAC都是Actor-Critic家族成员。

25.6 RL的工业应用与现实挑战

领域应用主要挑战
游戏AlphaGo, OpenAI Five算力投入巨大，可仿真但不保证迁移到真实世界
推荐系统长期用户engagement优化奖励延迟（用户可能几天后才转化），离线评估难
机器人抓取、行走、操纵Sim-to-Real gap，真实世界数据昂贵且危险
LLM对齐RLHF（用人类偏好微调模型）奖励模型可能被"钻空子"（reward hacking）
💡 学RL前必知：RL是ML所有分支中最"不成熟"的——它不稳定、调参难、对随机种子敏感、在不同任务上表现方差极大。如果你只是想做推荐或分类，用监督学习就够了。但如果你需要让AI在复杂环境中做序列决策（自动驾驶、游戏AI、机器人控制），RL是你最好的选择。而且RLHF（RL from Human Feedback）已经成为微调大语言模型的标准技术——这是RL在2020年代最重要的工业应用。
