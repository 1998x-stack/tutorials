第23章 音频信号处理

好，同学们，欢迎来到应用篇的第一站。如果你问我DSP哪一领域的"用户感知"最强，答案毫无疑问是音频。人耳是一个极其敏感的频谱分析仪——20Hz-20kHz的听觉范围、超过120dB的动态范围感知力，对谐波失真低至0.1%就能察觉的挑剔度——每一个数字都在告诉你：音频DSP做好了没人夸，做差了全世界都在骂。

我有个做TWS耳机的朋友跟我讲过一句名言："消费者可以接受音质"一般般"，但绝不可能接受通话有回声或者降噪有可闻的pumping噪声。"为什么？因为音频信号处理中的每一个算法——EQ、压缩器、AEC——都直接作用于人的听觉感知系统，任何artifact都会被大脑放大。本章我们就来拆解音频DSP中最核心的三个模块。

23.1 数字音频基础参数

在动手设计算法之前，先夯实一下音频信号的基本参数。这些数字你大概率已经见过无数次了，但真正理解它们背后"为什么是这些值"的人并不多。

参数 常见取值 典型场景 为什么是这个值
采样率 44.1 kHz CD音频、MP3 Nyquist定理：人耳上限20kHz × 2 + 抗混叠滤波器过渡带余量
采样率 48 kHz 电影、视频配音 与视频帧率（24/30fps）整数倍对齐，方便音视频同步
采样率 96 kHz 专业录音棚、Hi-Res 为超声波频段留余量、降低ADC群延时、减少预振铃
位深 16 bit CD、流媒体 动态范围 16×6.02 = 96.3 dB，足够覆盖人耳听觉范围
位深 24 bit 录音/混音 144 dB动态范围，留足headroom，避免混音过程中的数字削波

这里有一个极其容易混淆的概念：dB SPL vs dBFS。dB SPL（声压级）是物理世界的声学量，0 dB SPL约等于人耳能听到的最微弱声音（20 μPa），而130 dB SPL已经是痛阈。dB FS（Full Scale）则是数字域的量，0 dB FS代表ADC能表示的最大幅值（即所有bit全1），超过即削波。两者之间靠麦克风灵敏度（通常以dBV/Pa为单位）来桥接。举个例子：一个灵敏度为-26 dBV/Pa的MEMS麦克风，在94 dB SPL（1 Pa）声压下输出0.05 Vrms，换算到数字域约-26 dB FS——这意味着算法的headroom非常充足。

🔧 实践经验：在做音频处理链时，强烈建议在16bit的定点系统中保留6dB的数字headroom（即目标峰值不超过-6 dB FS）。否则任何滤波器的通带纹波叠加在一起都可能造成意外削波。

23.2 参量均衡器（PEQ）

均衡器恐怕是音频DSP中最容易被低估的模块了——外行觉得"不就是调个高音低音吗"，内行知道PEQ背后是满满的双二阶滤波器设计学问。

一个完整的参量均衡器通常包含三类滤波器：

  • 峰值滤波（Peaking）：在中心频率处提升或衰减指定幅度，两侧对称。典型公式：H(s) = (s² + (A/Q·ω₀)·s + ω₀²) / (s² + (1/(A·Q)·ω₀)·s + ω₀²)，其中A是增益幅度，Q是品质因子。
  • 搁架滤波（Shelf）：低频（Low Shelf）或高频（High Shelf）整体提升/衰减，过渡带平滑。常见于音箱的"响度补偿"。
  • 高低通滤波：具有指定阶数和Q值的截止型滤波器（Butterworth/Linkwitz-Riley等），用于分频或切除无用频段。

你可能会问：那么多参数，PEQ最核心的设计指标是什么？答案是Q值。Q值决定了滤波器的带宽——Q = ω₀ / BW。一个高Q（比如Q=10）的峰值滤波器会在中心频率周围形成一个很窄的"尖峰"，非常适合消除某个特定频率的共鸣（比如房间模式驻波）。而低Q（Q=0.7）则适合做宽范围的音色调整。

多数专业音频DSP芯片上，PEQ的典型实现是级联双二阶滤波器——即多个二阶节串联。以10段PEQ为例：

输入 → [Biquad_1] → [Biquad_2] → ... → [Biquad_10] → 输出
          ↑ 每个Biquad对应一段EQ参数 (fc, Q, Gain)

每个双二阶节的传递函数为：

       b₀ + b₁·z⁻¹ + b₂·z⁻²
H(z) = ──────────────────────
       a₀ + a₁·z⁻¹ + a₂·z⁻²

设计时先根据模拟域目标（fc, Q, Gain）通过双线性变换（Bilinear Transform，第10章讲过的预畸变别忘了！）算出系数{b₀,b₁,b₂,a₀,a₁,a₂}，然后选择具体实现结构。

⚠️ 踩坑：PEQ在低频高Q时的系数量化问题

这是我在一个车载DSP项目中踩得最深的坑。设计了一个50 Hz、Q=10、+12 dB的峰值滤波器用于补偿低音炮的频响凹陷，双线性变换后得到的系数在float32仿真中完美无缺。但在定点Q24格式上实现时，低频滤波器的极点非常靠近单位圆（因为中心频率远低于采样率），b₁和a₁系数接近-2.0且精度要求极高——24 bit定点量化后系数的微小误差直接导致滤波器不稳定，输出自激振荡（sustained oscillation）。

解决方案：查阅了RBJ Audio-EQ的Cookbook后，我发现问题出在双二阶结构的选取上。Direct Form I（DFI）对系数量化最敏感且动态范围损耗大；Direct Form II（DFII）节省一个状态变量但对中间变量的精度要求更高；转置Direct Form II（Transposed DFII）在数值稳定性上显著优于前两者，因为它将累加器置于滤波器前端，避免了中间变量被反复截断。改用Transposed DFII + 增加一个内部48 bit精度累加器后，问题彻底解决。后来的经验是：在定点实现中，低频高Q滤波器一律用Transposed DFII + 双精度累加，这是铁律。

23.3 动态范围处理

如果说EQ是音频信号的"空间整形"，那动态处理器就是音频的"时间整形"。常见的三件套是：

  • 压缩器（Compressor）：当信号超过阈值（Threshold）时，按比例（Ratio）减小增益。例如Threshold = -20 dB FS, Ratio = 4:1，意味着输入每上升4 dB，输出只上升1 dB。
  • 限幅器（Limiter）：实际上是Ratio无限大的压缩器，确保输出永远不会超过某个绝对上限（通常设为-0.1 dB FS防止削波）。
  • 噪声门（Noise Gate）：当信号低于阈值时，将增益降到极低（或静音）。用于去除安静段的本底噪声。

理解压缩器的核心在于两个时间参数：Attack和Release。Attack是检测到信号超过阈值后，增益开始减少的速度（通常1-50 ms）。Release是信号回落阈值以下后，增益恢复的速度（通常50 ms - 2 s）。这两个参数决定了压缩器的"性格"——快的Attack+Ratio可以当限幅器用，慢的Release会产生"pumping"效果（即呼吸感），在电子舞曲中反而是常用技巧。

压缩器的增益计算分为两种检测模式：

  • 峰值检测（Peak Detection）：直接取信号样本的绝对值，经过一个平滑滤波器后作为控制信号。响应快但容易随个别尖峰误触发。
  • RMS检测：取一段时间窗口（通常和Attack时间一致）的均方根值，更接近人耳感知的响度。专业压缩器几乎都用RMS检测或混合模式。
// 简化的 RMS 压缩器增益计算伪代码
for each sample n:
    // 1. 计算 RMS 值（窗口长度 W）
    x_rms[n] = sqrt(mean(x[n-W:n]²))

    // 2. 转换为 dB
    x_dB = 20 * log10(x_rms[n])

    // 3. 计算增益衰减（dB）
    if x_dB > threshold:
        gain_dB = (threshold - x_dB) + (x_dB - threshold) / ratio
    else:
        gain_dB = 0

    // 4. Attack/Release 平滑（一阶IIR）
    gain_smoothed[n] = alpha * gain_smoothed[n-1] + (1-alpha) * gain_dB
    // alpha 由 Attack/Release 时间常数决定

    // 5. 应用增益
    y[n] = x[n] * 10^(gain_smoothed[n] / 20)

23.4 声学回声消除（AEC）

这是音频通信中技术含量最高、也最令工程师头疼的模块。回声消除不是简单的"检测到远端说话就关掉近端麦克风"（那是半双工），真正的全双工AEC要做到：近端和远端同时说话时，麦克风信号中近端人声保留、远端扬声器声音被消除。

AEC的核心原理是自适应滤波（第15章的大显身手之地），流程如下：

  1. 参考信号获取：从DAC输出到扬声器的数字信号 x[n] 作为参考输入。
  2. 回声路径建模：自适应滤波器 ĥ[n] 模拟声音从扬声器到麦克风的传递路径（包括扬声器腔体、空气传播、麦克风振膜的联合响应）。
  3. 回声估计与减法：估计值 ŷ[n] = ĥ[n] * x[n]，从麦克风信号 d[n] 中减去，得到残差 e[n] = d[n] - ŷ[n]。
  4. 滤波器更新：用残差 e[n] 驱动NLMS自适应算法更新 ĥ[n] 的系数。

是不是听起来很简单？但实际工程中有三个魔鬼细节：

细节一：双讲检测（Double-Talk Detection，DTD）。当近端用户和远端用户同时说话时，麦克风信号 d[n] = 近端语音 + 回声 + 噪声。此时残差 e[n] 同时包含近端语音和自适应残差——如果用这个残差去更新滤波器，近端语音会作为"误差"被滤波器误学习，导致滤波器系数发散（即所谓"冰面效应"：近端说话时回声突然消失的恐怖感觉）。解决方案是在NLMS更新前加一个DTD判决门：检测到双讲就冻结滤波器系数更新，只做减法。

细节二：非线性处理（NLP）。自适应滤波器只能消除线性回声路径。但扬声器在低频段存在明显的谐波失真（THD可能有5-10%），这些非线性成分会被麦克风拾取但无法被线性滤波器消除。解决方案是在减法后加一个中心削波器（center clipper）或频谱衰减器，将低电平残余超标量消除——代价是小信号近端语音会有轻微的"空洞"感。

细节三：收敛速度 vs 滤波器长度的权衡。在室内环境下，回声路径的脉冲响应可能长达50-100 ms（取决于房间混响时间）。对于16 kHz采样率，这意味着滤波器需要800-1600个抽头。NLMS的收敛速度与滤波器长度成反比——抽头越多，收敛越慢。工程上常用子带自适应滤波（Subband AEC）来降低每路子带内的滤波器阶数，同时用不同步长加速收敛。

💡 认知提升：AEC是理论+工程的完美结合。第15章的自适应滤波器理论是这个系统的理论基础，但真正可用的AEC产品还需要DTD、NLP、残留回声抑制（RES）、舒适噪声注入（CNG）这一整套工程链。这就是为什么市面上好的AEC方案（如音效算法公司Dolby、QNX的AEC模块）都是经过10+年迭代的"黑匣子"。

23.5 实时音频链路的设计三角

把所有模块串起来之后，你需要面对一个终极问题：缓冲大小、延迟、计算量三者不可兼得。

设计参数 小缓冲（≈ 64 samples） 大缓冲（≈ 1024 samples）
算法延迟 约1.3 ms @ 48 kHz 约21 ms @ 48 kHz
FFT分辨率 低（750 Hz/bin @ 48k/64） 高（46.8 Hz/bin @ 48k/1024）
计算量 高（单位时间内处理更多帧） 低（帧间重叠少）
适用场景 通话AEC、监听返送 音乐播放、后期处理

记住一条经验法则：人类对音频延迟的感知阈值大约是10 ms。低于10 ms的往返延迟（RTT）大多数人感觉"即时"，10-20 ms musician能察觉，超过20 ms所有用户都能听到delay。这个认知决定了AEC必须在低延迟下运行——这就是为什么实时通信系统（WebRTC、Zoom）的音频模块几乎是DSP工程中最复杂的部分。

23.6 本章小结

这一章我们深入了音频DSP的三个核心领域：用PEQ做频域塑造、用动态处理器做时域控制、用AEC解决全双工通信的回声问题。我们特别强调了定点实现中DFI→DFII→转置的选择对数值稳定性的影响——这不是学院派的问题，而是上机器就会爆炸的真问题。

音频信号处理有一个迷人的特质：它的成功标准不是数学指标而是人的耳朵。你在示波器上看完美无瑕的波形，人耳可能觉得"不对味"；反之，你用算法做了大量不符合教科书审美的非线性处理，听众反而觉得"更好听"了。这种主观性和工程性的拉扯，正是音频DSP的魅力所在。

下一章我们将把视角从音频切换到通信领域，看看DSP在数字通信的基带处理中扮演着什么角色——从数字上下变频到符号同步、从载波恢复到信道均衡。你会发现很多原理是相通的，但通信系统多了一个新的维度：不确定性——你不知道信号什么时候来，也不知道信道长什么样。
