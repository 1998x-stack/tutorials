第13章 Neck与Head设计

大家好，我是你们的讲师。如果说backbone是眼睛，neck是神经通路，那么head就是做出决策的大脑。现代YOLO的性能提升，很大程度上归功于neck和head的精细化设计。它们将backbone提取的原始特征进一步加工、融合，最终输出检测结果。

本章我们深入探讨YOLO系列中neck和head的具体结构——从SPP到PANet，从耦合头到解耦头，看看这些"幕后英雄"是如何工作的。

13.1 Neck的角色：特征加工厂

Neck处于backbone和head之间，负责对backbone提取的多尺度特征进行融合和增强。它的核心目标有两个：

  • 多尺度融合： 将不同分辨率的特征图对齐并融合，让检测头在每个尺度上都获得丰富的信息
  • 感受野增强： 增大特征图中每个像素的视野范围，帮助检测头理解全局上下文

YOLO系列中，neck主要有两个组件：SPP/SPPF（空间金字塔池化）和PANet（路径聚合网络）。

13.2 SPP：空间金字塔池化

SPP（Spatial Pyramid Pooling）最初由何恺明在2015年提出，YOLOv4将其引入检测器作为neck的一部分。它的作用是：在单一特征图上提取多尺度的上下文信息。

SPP的核心操作是对输入特征图使用不同尺寸的池化核（如5×5、9×9、13×13），分别进行最大池化，然后将所有池化结果拼接在一起：

def spp_block(x, pool_sizes=[5, 9, 13]):
    """SPP多尺度池化"""
    input_channels = x.shape[1]

    # 1×1卷积压缩通道
    x = Conv2d(input_channels, input_channels // 2, 1)(x)

    # 多尺度池化分支
    branches = [x]  # 保留原始分支（等价于1×1池化）
    for size in pool_sizes:
        pool = MaxPool2d(kernel_size=size, stride=1, padding=size//2)(x)
        branches.append(pool)

    # 沿通道拼接所有分支
    out = torch.cat(branches, dim=1)
    return out
SPP为何有效？ 不同尺寸的池化核捕捉了不同范围的特征：小核关注局部细节（如物体边缘），大核关注全局结构（如物体整体形状）。SPP将这些信息浓缩到同一个特征图的不同通道中，让后续的检测头可以同时利用局部和全局信息做出判断。

13.3 SPPF：YOLOv5的快速变体

YOLOv5将SPP改进为SPPF（SPP Fast），大幅提升了计算效率。SPPF的核心思想是串行串联多个小池化核来等效替代多个不同尺寸的并行池化：

def sppf_block(x, pool_size=5):
    """SPPF：串行小池化核，等效于大池化核"""
    x = Conv2d(x, x.shape[1] // 2, 1)(x)  # 通道压缩

    # 串行池化：每个池化的输出作为下一个池化的输入
    y1 = MaxPool2d(pool_size, stride=1, padding=pool_size//2)(x)
    y2 = MaxPool2d(pool_size, stride=1, padding=pool_size//2)(y1)
    y3 = MaxPool2d(pool_size, stride=1, padding=pool_size//2)(y2)

    # 拼接原始 + 3个池化结果
    out = torch.cat([x, y1, y2, y3], dim=1)
    return out

与SPP相比，SPPF的计算量从O(k)减小到了O(log k)——通过三次5×5串行池化等效于一个13×13池化，但计算复杂度大幅降低。在YOLOv5s中，将SPP替换为SPPF后推理速度提升了约35%，而精度保持完全一致。

13.4 PANet：双向路径聚合

在第十章我们已经详细讨论了FPN和PANet的原理。这里我们从neck的角度重新审视PANet。在YOLOv4/v5/v8中，标准的neck结构是FPN + PANet的组合：

  • FPN（自顶向下）： 将高层的语义信息传播到低层，增强低层特征的语义判别力
  • PANet（自底向上）： 将低层的空间细节传播到高层，增强高层特征的定位精度

这个双向的信息高速公路确保了每个尺度上的特征图都是"信息完备"的。在YOLOv8的实现中，PANet路径上每经过一次融合，都会接一个C2f模块来进一步提取特征：

# YOLOv8 neck (简化)
class YOLOv8Neck(nn.Module):
    def forward(self, features):
        # features = [P3, P4, P5] 来自backbone

        # FPN 自顶向下
        f5 = self.c2f_top(P5)
        f4 = self.c2f_mid(concat(P4, upsample(f5)))
        f3 = self.c2f_low(concat(P3, upsample(f4)))

        # PANet 自底向上
        n3 = self.c2f_n3(f3)
        n4 = self.c2f_n4(concat(downsample(n3), f4))
        n5 = self.c2f_n5(concat(downsample(n4), f5))

        return [n3, n4, n5]  # 最终三个尺度的特征

13.5 解耦头（Decoupled Head）

在YOLOv8之前，YOLO的检测头是耦合的：同一个特征图同时预测分类和回归。例如YOLOv3的输出通道数为 3 × (5 + num_classes)——3个锚框，每个框预测5个回归参数加上C个类别概率。

YOLOv8引入了解耦头（Decoupled Head），将分类和回归任务分开处理：

# 解耦头结构
class DecoupledHead(nn.Module):
    def __init__(self, ch, num_classes, reg_max=16):
        # 分类分支
        self.cls_conv = nn.Sequential(
            Conv2d(ch, ch, 3),
            Conv2d(ch, num_classes, 1)  # 输出类别数
        )
        # 回归分支
        self.reg_conv = nn.Sequential(
            Conv2d(ch, ch, 3),
            Conv2d(ch, 4 * reg_max, 1)  # DFL: 4条边, 每条边reg_max个离散bin
        )
        # 与IoU相关的置信度分支
        self.iou_conv = Conv2d(ch, 1, 1)  # 预测IoU分数用于NMS

    def forward(self, x):
        cls_out = self.cls_conv(x)
        reg_out = self.reg_conv(x)
        iou_out = self.iou_conv(x)
        return cls_out, reg_out, iou_out
解耦头为什么更好？ 分类和回归任务关注的特征不同：分类需要判别"这是什么"，关注类别间的差异；回归需要精确"框出位置"，关注空间精准度。共享参数会导致两个任务互相干扰。解耦后，每个分支可以学习各自所需的特征，提升整体性能。在COCO上，解耦头相比耦合头带来了约2.4%的AP提升。

13.6 TaskAlignedAssigner：正样本分配器

有了解耦头之后，还需要一个机制来决定"哪些预测参与训练"。这就是TaskAlignedAssigner在YOLOv8中的作用。

传统的正样本分配策略只考虑IoU（谁的IoU高谁就是正样本），但这种方法忽视了分类质量。TaskAlignedAssigner综合考虑分类得分和IoU：

# TaskAlignedAssigner的正样本选择标准
def task_aligned_score(cls_score, iou, alpha=1.0, beta=6.0):
    """综合分类和回归质量的对齐分数"""
    # 分类得分的alpha次方 × IoU的beta次方
    return cls_score ** alpha * iou ** beta

# 对于每个ground truth，选择top-k个对齐分数最高的预测作为正样本
# 其余预测中，如果与任何gt的IoU < threshold，则为负样本
# IoU在threshold以上但未被选中的预测被忽略（不参与loss计算）

这种设计的精妙之处在于：它避免了一个"分类准但框不准"的预测被选为正样本（框不准会在大梯度），也避免了一个"框得准但分类差"的预测主导训练（分类错误会干扰类别学习）。只有两个任务都表现良好的预测，才有资格成为正样本。

踩坑实录：解耦头导致显存翻倍。 一次将YOLOv5的耦合头改为YOLOv8风格的解耦头后，发现batch size从64骤降到32才能塞进V100。原因是解耦头的分类和回归分支各自维护独立的卷积参数，每个分支都有独立的特征图，显存占用增加了近一倍。解决方法是给解耦头加上一个1×1通道压缩层，将neck输入的通道数从256压缩到128后再送入两个分支。最终效果是：显存占用恢复了，精度还提升了0.3%（通道压缩起到了正则化作用）。

13.7 辅助头与推理头

YOLOv9引入了"辅助推理头"（Auxiliary and Inference Heads）的概念：训练时用一个辅助头额外计算loss，推理时只用主推理头。辅助头的梯度可以引导backbone学习更好的中间层特征表示，而推理时不需要额外计算开销。

辅助头的结构与主头类似，但附加在backbone的中间层（如更浅层的特征图上）。它的loss权重通常设为主头loss的0.25倍，仅作为梯度路径的补充。

13.8 总结

Neck和Head是YOLO架构中"连接特征和决策"的关键环节。从SPP到SPPF的优化体现了工程效率的追求，从耦合头到解耦头的演变反映了对"任务分离"的深入理解，而TaskAlignedAssigner则代表了对"正负样本分配"的精细化思考。

有了以上这些组件的支撑，YOLO系列终于构建了一个完整、高效的检测管道。下一章，我们将探讨检测流程的最后一步——NMS与后处理，看看如何从众多预测框中筛选出最终的检测结果。
