损失函数——决定了模型"关心什么"

大家好，我是你们的讲师。今天我们来聊一个触及问题本质的话题——损失函数（Loss Function）。

你可能会想："损失函数不就是交叉熵或者MSE吗？框架自带的，有什么好调的？"这个想法很危险——损失函数定义了模型的"世界观"。你用交叉熵，模型关心的是"把每个类分对"；你用MSE，模型关心的是"输出值接近目标值"；你用Focal Loss，模型自动把更多注意力放在难分类的样本上。同一个模型、同一份数据，换一个损失函数，模型学到的"正确"可能是完全不同的东西。

我见过一个最夸张的例子：一个团队做文本分类，用交叉熵训练，模型F1=0.82。他们试了各种调参——学习率、优化器、网络结构——F1始终上不去。后来把交叉熵换成了Focal Loss（γ=2），什么都没改，F1=0.87。不是模型容量不够，不是数据不够——而是损失函数没有告诉模型应该关注什么。在那个数据集中，少量难分类的样本才是决定性能上限的关键。交叉熵把它们和大量简单样本同等对待，而Focal Loss让模型把精力聚焦到了该聚焦的地方。

交叉熵 vs MSE的梯度特性对比

先看两个最常用的损失函数——交叉熵（Cross Entropy）和均方误差（MSE）——在分类任务中的梯度行为差异。这个差异决定了为什么分类任务几乎不用MSE。

# 交叉熵配合Softmax的梯度特性
# 输出：softmax后的概率分布 p_i
# 目标：one-hot标签 y_i（正确类为1，其余为0）
# 梯度：∂L/∂z_i = p_i - y_i（对于正确类为 p_i - 1，错误类为 p_i）

# MSE配合Sigmoid的梯度特性
# 输出：sigmoid后的概率 σ(z)
# 目标：0或1
# 梯度：∂L/∂z = 2 * (σ(z) - y) * σ(z) * (1 - σ(z))
# 当σ(z)接近0或1时，σ(z)*(1-σ(z))接近0 → 梯度消失

交叉熵配合softmax的梯度是 (pred - target)——极其简洁。当预测正确率很高时，pred接近target，梯度小（已经学好了不需要大更新）；当预测错误时，pred和target差距大，梯度大（需要大幅修正）。这种自适应的梯度缩放正是分类任务所需要的。

而MSE配合sigmoid的梯度中包含 σ(z) * (1-σ(z)) 项——这是sigmoid的导数。当输出接近0或1（即分类很确定）时，梯度趋近于0。这意味着在训练早期还没学好时，梯度反而大；训练后期快稳定了，梯度反而小——这和我们的需求正好相反。交叉熵的梯度特性使得分类任务收敛更快、更稳定。

Focal Loss的巧妙设计

标准交叉熵对所有样本一视同仁——简单样本和难分类样本的loss权重相同。当数据存在大量简单负样本时（比如目标检测中一张图有10万个候选框，只有几个包含目标），简单的负样本的loss总和会淹没少数难分类正样本的信号。

直觉上你会想："那把负样本的权重降低不就行了？"但简单地对所有负样本降权是一种粗暴的做法——因为有些难分类的负样本其实也很有信息量（例如长得像目标的背景区域）。

Focal Loss的精妙之处就在于它不是静态地降权，而是动态地根据样本的学习难度来调整权重。它的公式在交叉熵基础上引入了一个调制因子：

import torch.nn.functional as F

def focal_loss(logits, targets, gamma=2.0, alpha=0.25):
    """
    Focal Loss for binary classification
    Args:
        gamma: 聚焦参数，控制易分类样本的权重衰减速度
        alpha: 平衡权重，处理正负样本数量不均衡
    """
    ce_loss = F.binary_cross_entropy_with_logits(
        logits, targets, reduction='none'
    )
    p = torch.sigmoid(logits)  # 预测概率
    # pt = p if y=1 else 1-p — 模型对正确分类的置信度
    pt = p * targets + (1 - p) * (1 - targets)
    # 调制因子 (1-pt)^gamma
    focal_weight = (1 - pt) ** gamma

    if alpha >= 0:
        alpha_t = alpha * targets + (1 - alpha) * (1 - targets)
        focal_weight = alpha_t * focal_weight

    return (focal_weight * ce_loss).mean()
    # γ=0 → 退化为标准交叉熵
    # γ=2 → 常用值。pt=0.9时权重=0.01，pt=0.5时权重=0.25

调制因子 (1-pt)^γ 的工作原理：当一个样本的预测置信度pt很高（模型已经能轻松分类）时，(1-pt)^γ接近于0，该样本的loss贡献被大幅降低。当一个样本置信度低（模型还分不对），(1-pt)^γ接近1，loss几乎不受影响。γ控制这个降权曲线的"陡峭程度"——γ=0退化为交叉熵，γ=2是最常用的值。

Focal Loss还有一个α参数处理正负样本数量不均衡。注意不要直观地把α理解成正样本权重——当γ=2时，大量简单负样本已经被调制因子大幅降权了，α主要用于精细调节。

γ的搜索策略：以10倍步长搜索。先试γ=0.5，效果好则降为γ=0.2、0.1；效果差则升为γ=2、5。不要默认使用γ=2——在轻度不平衡时可能有害（过度关注噪声样本）。γ越大，模型越"聚焦"于难分类样本，但也越容易过拟合于标注错误的样本。

Label Smoothing的校准原理

标准分类任务使用one-hot标签——正确类为1，其余为0。这看起来天经地义，但其实它有一个隐藏的问题：鼓励过度自信。为了让正确类的softmax输出接近1，模型会不断增大正确类的logit，使其趋于无穷大。这导致模型对所有预测都极度自信——即使在面对模糊样本时，也会输出0.999这样的概率。

过度自信带来的问题是：模型输出概率不再反映真实正确率。一个预测概率0.999的样本，实际正确率可能只有0.92——这种校准不良的模型在实际部署中非常危险。

Label Smoothing的做法很简单：将one-hot标签和均匀分布做加权平均。

# Label Smoothing —— PyTorch实现
def label_smoothing_loss(logits, targets, alpha=0.1):
    """alpha是平滑强度，常用0.1"""
    K = logits.size(-1)  # 类别数
    log_probs = F.log_softmax(logits, dim=-1)

    # 标准交叉熵（只对正确类算loss）
    ce_loss = F.nll_loss(log_probs, targets, reduction='none')

    # KL散度项：对所有类别的log_probs求平均
    kl_loss = -log_probs.mean(dim=-1)

    # 组合损失
    return (1 - alpha) * ce_loss + alpha * (kl_loss * K / (K - 1))

# 或者直接用PyTorch内置实现（>=1.10）
# loss_fn = nn.CrossEntropyLoss(label_smoothing=0.1)

假设类别数K=1000，α=0.1。原本正确类标签从1降为0.9 + 0.1/1000 = 0.9001，其他类从0升为0.1/1000 = 0.0001。这个微小的变化迫使模型不再追求"正确类logit趋近无穷"，而是满足于正确类和错误类之间"有界"的差距。

Label Smoothing的收益：更好的模型校准（输出概率更反映真实正确率）、对标注噪声更鲁棒、更容易与知识蒸馏配合（软标签更适合作为教师输出）。

什么时候跳过Label Smoothing？当训练数据标注绝对正确且无噪声时，平滑反而会损失信息。在二分类中平滑效果也有限（类别数太少，平滑后标签几乎没变化）。如果你只关注top-1准确率而不需要概率校准，Label Smoothing也未必有帮助——它的主要收益在改善置信度校准上。

多任务损失平衡

在多任务学习中，不同任务的损失函数量级不同——一个分类loss可能是0.1量级，一个回归loss可能是10量级。直接把它们加起来，大loss的任务会主导梯度，小loss的任务"被绑架"：

# 多任务损失的"梯度绑架"问题
# loss = loss_cls + loss_reg
# loss_cls ≈ 0.1，loss_reg ≈ 10.0
# 梯度中 ~99% 来自回归任务 → 分类任务几乎没学到

解决这个问题有三种常见方法：

1. 手动加权。手动设置每个loss的权重：loss = w1*loss_cls + w2*loss_reg。权重的取值通过验证集调优。简单直接，但需要多次实验。

2. 同方差不确定性加权。Kendall et al.（2018）提出让网络学习每个任务的不确定性参数σ，然后加权：loss = loss_1 / σ₁² + loss_2 / σ₂² + log(σ₁ * σ₂)。不确定性越高的任务权重越低——不用人工调权，自动学习。

3. GradNorm。动态调整权重，使得不同任务的梯度范数在同一个量级。比不确定性加权更复杂，但理论上更精确。

实践中，大多数场景使用手动加权就够了。先把各loss归一化到相近量级，然后在验证集上调优权重比例。

损失函数 适用场景 梯度特性
Cross Entropy + Softmax 多分类 梯度 = pred - target，简洁高效，自适应缩放
Binary Cross Entropy + Sigmoid 二分类、多标签分类 每个输出独立计算，互不干扰
MSE（L2 Loss） 回归 梯度 ∝ (pred - target)，大误差大梯度，对异常值敏感
MAE（L1 Loss） 回归（对异常值鲁棒） 梯度 = sign(pred - target)，恒为±1，不随误差大小变化
Focal Loss 极端不平衡分类、目标检测 动态降低易分类样本权重，聚焦难样本
Huber Loss 回归（异常值场景） 误差小→L2，误差大→L1，兼具两者优点
Contrastive Loss 对比学习、度量学习 拉近同类、推远异类，正负样本对训练
损失函数设计的第一原则：损失函数必须与业务目标对齐。你优化交叉熵，模型就最小化分类错误；你优化NDCG，模型就最优排序。如果离线loss下降但业务指标不动，大概率是你的loss函数没有"代理"好业务目标——这是损失函数设计中最常见的根本性错误。回到数据，看bad case，问自己："我希望模型在这里做什么？当前的loss函数在鼓励它做这个吗？"

总结一下，损失函数是模型的"北极星"——定义了优化方向。交叉熵配合softmax是分类任务的标准配置，梯度特性简洁高效。Focal Loss通过调制因子动态聚焦难分类样本，是处理极端不平衡的利器。Label Smoothing通过对标签做平滑软化，改善模型校准和多任务效果。多任务学习中需要注意各loss的量级平衡，避免梯度绑架。
