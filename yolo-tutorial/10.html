第10章 特征金字塔FPN

大家好，我是你们的讲师。在YOLOv3之前，小目标检测一直是YOLO的软肋。FPN（Feature Pyramid Network）的出现彻底改变了这一局面——它让浅层的细节信息和高层的语义信息在同一个特征图上相遇。这节课我们来深入理解这个让YOLO"眼观六路"的关键技术。

多尺度问题是目标检测的核心挑战之一。一幅图像中的物体，大小可能相差数百倍。如何让一个单一的网络同时检测到巨大的公交车和微小的交通标志？答案就是——特征金字塔。

10.1 多尺度挑战

在FPN出现之前，目标检测处理多尺度问题主要有三种方式：

  • 图像金字塔（Image Pyramid）： 将输入缩放到多个尺寸分别推理——精度高但计算量成倍增加，完全无法实时
  • 单尺度预测（Single-scale）： 只在网络最后一层特征图上做预测——如YOLOv1/v2，丢失了浅层的细节信息
  • 多尺度特征（Multi-scale）： 在不同深度的层上分别做预测——如SSD，但浅层特征缺乏语义信息

理想的情况是：既能利用浅层的高分辨率（精细定位），又能利用深层的强语义（准确分类）。这就是FPN要解决的问题。

直观理解： 如果把卷积网络比作一个"不断下采样的信息漏斗"，浅层特征告诉我们是"轮廓和边缘"，深层特征告诉我们是"这是什么物体"。FPN在这个漏斗上开了一扇"横向门"，让信息可以双向流动。

10.2 FPN的架构设计

FPN由Facebook AI Research在2017年提出，其核心由两个通路组成：

自底向上通路（Bottom-up Pathway）

这就是标准的骨干网络前向传播过程。以ResNet为例，每个stage的输出构成一个特征层级——C2, C3, C4, C5，其空间分辨率依次减半，但语义层次递增。

自顶向下通路（Top-down Pathway）

从最高层（最抽象）的特征开始，通过上采样（通常使用最近邻插值×2）将特征图放大，然后与对应的自底向上特征通过横向连接（1×1卷积）融合：

P5 = Conv1x1(C5)
P4 = Conv1x1(C4) + UpSample×2(P5)
P3 = Conv1x1(C3) + UpSample×2(P4)
P2 = Conv1x1(C2) + UpSample×2(P3)

每个融合后的特征图P_i再经过一个3×3卷积消除上采样的混叠效应，最终形成特征金字塔：P2, P3, P4, P5。每个层级都具有丰富的语义信息（来自高层）和精确的空间信息（来自低层）。

金字塔层级 空间尺寸（输入640×640） 对应下采样倍数 适合检测的物体
P2 / P3（大特征图） 80×80 / 40×40 8× / 16× 小物体（<32px）
P4（中等特征图） 20×20 32× 中等物体
P5（小特征图） 10×10 64× 大物体（>128px）

10.3 YOLOv3中的FPN实现

YOLOv3采用了FPN的思想，但做了简化：它只在三个尺度上做预测，分别是13×13、26×26和52×52（以416×416输入为例）。这三个尺度对应于下采样的32倍、16倍和8倍。

# YOLOv3 FPN的简化伪代码
def yolov3_fpn(x):
    # 自底向上：Darknet-53
    c3 = darknet_stage_3(x)   # 52×52
    c4 = darknet_stage_4(c3)  # 26×26
    c5 = darknet_stage_5(c4)  # 13×13

    # 自顶向下融合
    p5 = conv5(c5)            # 13×13 → 第一个预测头
    p5_up = upsample(p5)      # → 26×26

    p4 = concat(p5_up, c4)    # 26×26 横向连接
    p4 = conv5(p4)            # 26×26 → 第二个预测头
    p4_up = upsample(p4)      # → 52×52

    p3 = concat(p4_up, c3)    # 52×52 横向连接
    p3 = conv5(p3)            # 52×52 → 第三个预测头

    return [p3, p4, p5]       # 三个尺度的预测

YOLOv3的FPN虽然只有三个层级，但它已经带来了质的飞跃——在COCO数据集上，YOLOv3的AP_S（小物体AP）从YOLOv2的不到10%提升到了18%以上。

YOLOv3 FPN的关键设计： 每个预测头使用的锚框不同——大特征图（52×52）用最小的三个锚框检测小物体，小特征图（13×13）用最大的三个锚框检测大物体。这种"不同尺度对应不同锚框"的设计成为了后续所有多尺度YOLO的标准范式。

10.4 PANet：自底向上的增强

FPN虽然实现了语义信息的自上而下传递，但它缺少一个自底向上的通路来增强浅层的定位特征。YOLOv4引入了PANet（Path Aggregation Network）来解决这个问题。

PANet在FPN的基础上增加了一个自底向上的增强通路：

# PANet增强通路
N3 = conv3(P3)
N4 = conv3(concat(downsample(N3), P4))
N5 = conv3(concat(downsample(N4), P5))

简单来说，FPN让高层语义信息传递到低层；PANet再让低层的精确位置信息传递回高层。这种双向信息流确保了每个尺度的特征图同时具备精确的空间细节和丰富的语义信息。

YOLOv4中的PANet结构将AP提升了约2-3%。在PANet之后，YOLOv5/v8/v9等版本都沿用了"FPN+PAN"的双塔结构作为默认neck。

10.5 BiFPN：加权双向融合

EfficientDet提出的BiFPN（Bidirectional Feature Pyramid Network）对FPN做了进一步的优化：

  • 删除单输入节点： 在PANet中，只有一条输入边的节点贡献有限，BiFPN将其移除
  • 跨尺度连接： 在同一层级的输入和输出节点之间增加跳跃连接，融合更多特征
  • 加权特征融合： 不同输入特征的重要性不同，BiFPN使用可学习的权重来调节融合比例
# 加权特征融合
O = Σ (w_i * I_i) / (ε + Σ w_j)

其中 w_i 是可学习权重（通过ReLU确保非负），
ε 是防止除零的小常数（通常1e-4）
踩坑实录：FPN层数与算力的权衡。 我在一个交通监控项目中使用YOLOv5，为了追求小目标检测精度，将FPN从3层扩展到了5层（增加P2和P6）。结果mAP只提升了1.2%，但推理速度从2ms降到了3.8ms，几乎翻倍。原因是P2（160×160的特征图）的计算量太大了——它的分辨率是P5的256倍！在实际工程中，3层FPN通常是最优的性价比选择。

10.6 FPN如何改善小目标检测

很多同学会问：FPN为什么特别有助于小目标检测？原因有两点：

第一，大特征图保留了空间细节。 小目标在原始图像中只占据少量像素，经过多次下采样后，这些小目标的特征可能会完全消失（比如一个8×8像素的物体经过32倍下采样后只剩不到1个像素）。FPN在8×或16×下采样的特征图上做预测，保留了足够的分辨率来检测小物体。

第二，高层语义指导低层定位。 如果没有FPN，在浅层特征图上做预测虽然分辨率高，但特征语义不足——模型可能把多个零散的边缘误识别为物体。FPN给浅层特征"注入"了来自高层的语义信息，让模型能更准确地判断"这片细节到底是不是一个物体"。

10.7 总结

从FPN到PANet再到BiFPN，特征金字塔的设计一直在追求一个目标：让不同尺度的特征图都成为高质量的"检测基础"。FPN解决了"语义-空间"的矛盾，PANet增补了信息的双向流动，BiFPN则通过加权融合让模型自主决定特征的重要性。

在YOLO的后续版本中，FPN+PANet成为了事实上的标准neck结构，直到YOLOv8/v9仍然保持这种基础设计。理解了FPN，你就理解了YOLO检测头的"前半生"。下一章我们将探讨一个看似简单但威力巨大的主题——数据增强策略。
