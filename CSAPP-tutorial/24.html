第二十四章：经典Lab实战(I)——Data/Bomb/Attack

如果你去问任何一个 CSAPP 的过来人"这门课最难但也最值得做的是什么"，答案几乎一致：Lab。CSAPP 的七个 Lab 是卡内基梅隆大学计算机系几十年教学经验的结晶，每一个都经过精心设计——让你在"玩"的过程中把书本知识变成肌肉记忆。从这一章开始，我们用三章的篇幅逐一带你攻坚这七个 Lab。

本章聚焦前三个 Lab：Data Lab 让你用最基本的位运算实现所有整数运算；Bomb Lab 让你化身拆弹专家用 GDB 逆向你不知道的二进制程序；Attack Lab 让你站在黑客的角度理解缓冲区溢出的威力。这三个 Lab 覆盖了全书前七章的核心——数据表示、机器级代码、以及系统安全的脆弱面。

24.1 Data Lab——位运算极限挑战

Data Lab 是所有 Lab 中最"纯粹"的一个：你不用写任何复杂逻辑，只用一组受限的位运算符（~ & | ^ << >>）来实现各种整数和浮点数操作。规则极其严苛——比如实现 !（逻辑非）不能用条件分支，实现 absVal（绝对值）不能用乘法——目的就是逼你深刻理解补码的每一位。

来看几个典型 puzzles 的解法思路：

// Puzzle 1: bitXor —— 只用 ~ 和 & 实现异或
// 公式：x ^ y = (~x & y) | (x & ~y)
//          = ~(~(~x & y) & ~(x & ~y))  用德摩根律消去 |
int bitXor(int x, int y) {
    return ~(~(~x & y) & ~(x & ~y));
}

// Puzzle 2: tmin —— 返回最小的补码整数
// 对于 32 位系统，TMin = 0x80000000 = 1 << 31
int tmin(void) {
    return 1 << 31;
}

// Puzzle 3: isTmax —— 判断 x 是否为最大补码整数
// TMax = 0x7FFFFFFF，加 1 得 0x80000000（TMin）
// TMin + TMin = 0，而 TMax + 1 + TMax + 1 != 0
// 巧妙的性质：TMax + 1 == ~TMax
int isTmax(int x) {
    int plus1 = x + 1;
    return !(plus1 + plus1) && !!plus1;
}

在 Data Lab 中你会频繁用到下面这些位运算技巧：

技巧 表达式 用途
取最低有效位 x & 1 判断奇偶性
提取特定位 (x >> n) & mask 取从第 n 位开始的若干位
取符号位 (x >> 31) & 1 或 !!(x >> 31) 判断正负（补码最高位）
取反加一 ~x + 1 取补码的相反数（-x）
掩码生成 (1 << n) - 1 生成长度为 n 的全 1 掩码
符号扩展 (x << shift) >> shift 将低 n 位的数按符号扩展到全宽

浮点数部分更有意思——你需要在不使用浮点指令的前提下，手动实现浮点数的编码和解码，比如把 unsigned 解释成 float 的位模式。这迫使你理解 IEEE 754 的符号位、阶码（biased exponent）和尾数（fraction）三部分的真正含义。

Data Lab 通关心法：不要急着写代码。先在纸上列出真值表，用布尔代数推导出最简表达式。位运算的终极法宝是德摩根律 ~(~A & ~B) = A | B 和分配律 (A & B) | (A & C) = A & (B | C)。如果能用 5 个运算符搞定，就别用 6 个——这既是规则要求，也是对你逻辑压缩能力的考验。

24.2 Bomb Lab——GDB 拆弹全流程

Bomb Lab 的设定是：你面前有一个二进制炸弹文件，运行时它会提示你输入 6 个字符串（对应 6 个 phase）。输入对了就进入下一关，输入错了炸弹会"爆炸"并扣分。你的任务是用 GDB 逆向分析这个二进制文件，找出每个 phase 应该输入什么。

六个 phase 覆盖了不同的反汇编分析技巧：

  • Phase 1——字符串比较。反汇编后看到 strings_not_equal 调用，在 GDB 中用 x/s <地址> 查看目标字符串即可。
  • Phase 2——数组循环。需要输入 6 个数字，反汇编后看到循环结构和 read_six_numbers，推导数列递推关系。
  • Phase 3——switch 跳转表。输入两个整数，第一个作为 switch 的 index，第二个与跳转表中的目标值比较。
  • Phase 4——递归函数。输入两个整数，反汇编递归调用，理解递归函数的语义（通常是 Fibonacci 类函数）。
  • Phase 5——字符串变换。输入一个 6 字符的字符串，每个字符的低 4 位作为索引去查一个固定数组，变换后的结果必须等于目标字符串。
  • Phase 6——链表排序。最复杂的一关，需要输入 6 个数字，对应链表节点的排序顺序，需要理解链表的遍历和比较逻辑。

GDB 拆弹工作流：(1) 在 phase 入口设置断点 b phase_4；(2) 运行炸弹 run，输入一个测试字符串；(3) 用 layout asm 进入汇编视图；(4) 逐指令执行 stepi，观察寄存器变化 info registers；(5) 遇到 cmp 指令时查看比较值——x/s $rdi 看字符串、x/d $rsi 看整数；(6) 所有 phase 通过后，会看到"Congratulations!"的提示。熟练之后，一个下午拆完六个 phase 不是问题。

24.3 Attack Lab——缓冲区攻击之旅

Attack Lab 是 Bomb Lab 的精神续作。你面对一个存在缓冲区溢出漏洞的程序（getbuf 函数把输入读到一个很小的栈缓冲区中），目标是利用溢出控制程序执行流。Lab 分三个递进的 level：

  • Level 1: Code Injection（CI）——最简单的版本。溢出缓冲区，覆盖返回地址，让 getbuf 返回到 touch1 函数（而不是它原本应该返回的调用者）。你只需要用 xxd 或 printf 构造一个字节序列，把 touch1 的地址精确覆盖到返回地址的位置。
  • Level 2: CI + 参数传递——在 Level 1 的基础上，你还需要将 touch2 要求的参数（如 cookie）传入 %rdi 寄存器。方法：在缓冲区中注入一段 shellcode，shellcode 设置 %rdi 后跳转到 touch2。
  • Level 3: ROP（Return-Oriented Programming）——因为现代系统启用了 NX（No-Execute）保护，栈上的代码不能直接执行。你需要从已有的二进制代码中寻找以 ret 结尾的指令片段（gadget），把它们串联成一条"攻击链"。ROP 的终极形态是构建 Turing-complete 的计算能力——不需要注入一行新代码。
// Level 2 的典型 shellcode（AT&T 语法）
// 将 cookie 值（假设为 0x59b997fa）存入 %rdi，然后跳转 touch2
mov $0x59b997fa, %rdi
push $0x4017ec    // touch2 的地址
ret

法律与道德提醒：Attack Lab 的目的不是教你做黑客，而是让你理解缓冲区溢出的底层原理，从而在写代码时有意识地避免这类漏洞。在真实的系统上测试攻击代码是非法的——即使你只是"好奇"。把学到的知识用在保护系统上，而不是攻破它们。值得一提的是，Intel 和 AMD 的现代 CPU 已经支持 Control-flow Enforcement Technology（CET），硬件级别阻止 ROP 攻击，但这个 Lab 让你理解为什么需要这些保护机制。

Data Lab 锻炼了你的位运算功底，Bomb Lab 提升了你的逆向分析能力，Attack Lab 让你看清了栈的脆弱本质。这三个 Lab 完成以后，你已经具备了 C 语言底层编程的"铁人三项"基本功。下一章我们将进入系统编程的三座大山——Cache Lab 的缓存模拟器、Shell Lab 的进程管理、Malloc Lab 的内存分配器。
