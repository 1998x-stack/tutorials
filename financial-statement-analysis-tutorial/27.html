大家好，我是你们的讲师。想象一下这个场景：你面前摆着两份年报——A公司营收增长20%、净利润增长15%、ROE高达18%；B公司营收增长8%、净利润增长6%、ROE只有12%。光看这几个数字，大多数人的直觉是A公司更好。但如果我告诉你：A公司的F-Score只有3分（满分9分）、M-Score高达-1.8（表明高盈余管理风险）、Z-Score仅为1.4（濒临破产区），而B公司的三项评分分别是8分、-2.8和3.2——你现在还会选A吗？

这就是量化财务模型的魅力所在。它们不依赖主观判断，而是基于学术研究和大规模数据验证，用客观、可重复的公式给出一个"健康分数"。这类模型不是万能的，但它们能帮咱们避免很多最基本的错误——至少不会选到那些"看起来很好、实际上很危险"的公司。本章咱们将深入讲解三个最经典的量化模型：Piotroski F-Score、Beneish M-Score和Altman Z-Score，然后教你如何构建自己的多因子打分卡。

Piotroski F-Score实战

F-Score由芝加哥大学会计学教授Joseph Piotroski在2000年提出。它的思路非常朴素：用9个简单的财务指标来评估一家公司的"基本面是否在变好"，每个指标满足条件得1分，总分0-9分。分数越高，说明公司越健康。

这个模型在学术界的验证结果令人印象深刻：买入高F-Score（8-9分）的公司、做空低F-Score（0-1分）的公司，年均超额收益可以达到15%以上。更重要的是，F-Score在A股的有效性也得到了实证支持——沪深交易所的研究表明，2010-2025年间，高F-Score组合在A股的累计超额收益约为80%。

9项指标逐一解析

F-Score的9个指标分为三类：盈利能力（4项）、财务杠杆与流动性（3项）、运营效率（2项）。

类别 编号 指标 计算公式 得分条件
盈利能力 F1 ROA（总资产收益率） 净利润 / 总资产 ROA > 0
F2 CFO（经营现金流） 经营活动现金流净额 CFO > 0
F3 AROA（ROA变动） 本年度ROA - 上年度ROA AROA > 0
F4 ACCRUAL CFO - 净利润（应计项） CFO > 净利润（应计项为负）
财务杠杆/流动性 F5 ALEVER（杠杆变动） 本年度资产负债率 - 上年度资产负债率 ALEVER < 0
F6 ALIQUID（流动性变动） 本年度流动比率 - 上年度流动比率 ALIQUID > 0
F7 EQ_OFFER（股权融资） 是否发行新股（不含配股） 未发行新股
运营效率 F8 AMARGIN（毛利率变动） 本年度毛利率 - 上年度毛利率 AMARGIN > 0
F9 ATURN（资产周转率变动） 本年度周转率 - 上年度周转率 ATURN > 0 
F-Score + M-Score 组合筛选Python代码（伪代码框架）：

import pandas as pd
import numpy as np

def calculate_f_score(financial_data):
    """
    financial_data: DataFrame, 包含所需财务指标
    返回每只股票的F-Score得分
    """
    scores = pd.DataFrame(index=financial_data.index)
    scores['F1'] = (financial_data['ROA'] > 0).astype(int)
    scores['F2'] = (financial_data['CFO'] > 0).astype(int)
    scores['F3'] = (financial_data['ROA_change'] > 0).astype(int)
    scores['F4'] = (financial_data['ACCRUAL'] < 0).astype(int)  # ACCRUAL = CFO - 净利润
    scores['F5'] = (financial_data['LEVER_change'] < 0).astype(int)
    scores['F6'] = (financial_data['LIQUID_change'] > 0).astype(int)
    scores['F7'] = (~financial_data['issued_new_shares']).astype(int)
    scores['F8'] = (financial_data['MARGIN_change'] > 0).astype(int)
    scores['F9'] = (financial_data['TURN_change'] > 0).astype(int)

    scores['F_SCORE'] = scores.sum(axis=1)
    return scores['F_SCORE']

def calculate_m_score(financial_data):
    """
    计算Beneish M-Score
    8个变量加权求和
    """
    # M-Score = -4.84 + 0.92*(DSRI) + 0.528*(GMI)
    #          + 0.404*(AQI) + 0.892*(SGI)
    #          + 0.115*(DEPI) - 0.172*(SGAI)
    #          + 4.679*(ACCRUAL) - 0.327*(LEVI)
    #
    # DSRI: 应收账款/营收比率变动
    # GMI: 毛利率变动指数
    # AQI: 资产质量指数
    # SGI: 营收增长率
    # DEPI: 折旧率变动指数
    # SGAI: 销售管理费率变动
    # ACCRUAL: 总应计项/总资产
    # LEVI: 杠杆指数

    m_score = (-4.84
               + 0.92 * financial_data['DSRI']
               + 0.528 * financial_data['GMI']
               + 0.404 * financial_data['AQI']
               + 0.892 * financial_data['SGI']
               + 0.115 * financial_data['DEPI']
               - 0.172 * financial_data['SGAI']
               + 4.679 * financial_data['ACCRUAL_TA']
               - 0.327 * financial_data['LEVI'])
    return m_score

# 组合筛选策略
def combined_screening(financial_data):
    f_score = calculate_f_score(financial_data)
    m_score = calculate_m_score(financial_data)

    results = pd.DataFrame({
        'F_SCORE': f_score,
        'M_SCORE': m_score
    })

    # 筛选条件
    results['PASS'] = (
        (results['F_SCORE'] >= 7) &    # 基本面优秀
        (results['M_SCORE'] < -2.22)   # 低盈余管理风险
    )

    return results[results['PASS']].sort_values('F_SCORE', ascending=False)
A股使用建议：F-Score在A股使用时，F7（股权融资）这一项的适用性需要灵活处理。A股上市公司经常通过定向增发融资，但这不一定是坏信号——也可能是为了扩张产能。建议把F7的权重调低，或者在判断时结合公司的融资用途说明。另外，建议将F-Score阈值调整为A股特定版本：≥7分为"优秀"，4-6分"中等"，≤3分"警惕"。

Beneish M-Score实战

如果说F-Score是"好学生成绩单"，那M-Score就是"舞弊探测器"。M-Score由印第安纳大学M. Daniel Beneish教授在1999年提出，专门用于检测上市公司是否可能存在盈余管理甚至财务造假。

M-Score通过8个变量的加权组合，计算出一个分数。阈值是-2.22：M-Score大于-2.22（即小于绝对值），标记为"可能存在盈余管理"；M-Score小于-2.22，标记为"正常"。

重要说明：M-Score不是定罪工具。M-Score大于-2.22并不代表公司一定在造假，而是说"从财务数据的统计学特征来看，这家公司和历史上那些被证实造假的公司在数据模式上相似"。它提示你：这家公司值得深挖——不是直接卖出，而是要投入更多精力做尽调。

八个核心变量

M-Score的8个变量覆盖了财务造假的常见操作路径：

  • DSRI（应收账款/营收指数）：应收账款增速远高于营收增速，可能表明公司通过放宽信用政策来虚增收入。
  • GMI（毛利率指数）：毛利率连续下滑但营收增长，可能表明公司面临竞争压力，有造假动机。
  • AQI（资产质量指数）：无形资产和其他资产占比上升，可能表明公司将费用资本化来美饰利润。
  • SGI（营收增长率）：高速增长的公司面临更大的业绩压力，造假动机更强。
  • DEPI（折旧指数）：折旧率下降可能表明公司延长了资产使用寿命来降低费用。
  • SGAI（销售管理费用指数）：销售管理费用率上升，可能表明公司效率下降或存在隐性支出。
  • ACCRUAL（总应计项/总资产）：应计利润占比过高是造假的高危信号。
  • LEVI（杠杆指数）：负债率快速上升，公司为满足债务契约可能进行盈余管理。

在实际使用中，建议将M-Score和F-Score组合起来——F-Score选好公司，M-Score排除坏公司。比如：先筛选F-Score≥7的公司，再排除M-Score>-2.22的公司。这样能够同时保障"选到基本面好的"和"踩不到雷"。

Altman Z-Score破产预测

Z-Score由纽约大学Altman教授在1968年提出，是最早的财务预警模型之一。它通过5个财务比率的加权组合来预测企业未来2年内破产的概率。

Z值计算公式

原始Z-Score公式适用于制造业上市公司：

Z = 1.2 × X1 + 1.4 × X2 + 3.3 × X3 + 0.6 × X4 + 1.0 × X5

其中：
X1 = 营运资本 / 总资产           — 流动性水平
X2 = 留存收益 / 总资产           — 累积盈利能力
X3 = EBIT / 总资产                — 资产盈利能力
X4 = 股权市值 / 负债账面价值      — 资本结构稳定性
X5 = 营业收入 / 总资产            — 资产周转效率

阈值判断：
Z > 2.99     — 安全区（破产概率低）
1.81 < Z < 2.99 — 灰色区（需要进一步分析）
Z < 1.81     — 危险区（破产概率高）

A股适用性调整

Z-Score在中国的适用性需要做一些调整。研究发现，直接套用美国市场的阈值在中国市场存在偏差。一种被广泛使用的调整方式是使用中国市场特有的阈值：

  • 制造业企业：Z' = 0.73×X1 + 0.32×X2 + 1.25×X3 + 0.64×X4 + 0.82×X5（中国学者拟合版本），阈值1.2/2.6。
  • 非制造业企业：使用Z''替代公式（移除X5），避免高周转率的零售业公司被错误分类。
  • 时间窗口：Z-Score最好结合趋势使用——连续2年Z值下降比单一年份的低Z值更有预警意义。
踩坑故事——纯量化模型在2015年牛市的惨败：2015年上半年，一个做量化选股的团队严格按照F-Score + Z-Score筛选出了一批"高质量股票池"——基本面优秀、Z值安全、估值合理。结果在2015年3-6月，这个组合的收益率只有大盘涨幅的不到一半。而那些被他们排除的"高风险"股票（低F-Score、高M-Score的小盘概念股）反而涨得飞起。到了6月股灾之后，他们的组合确实跌得少（回撤20% vs 大盘回撤40%），但客户已经不买账了——因为前面"跑输大盘"已经把信任消耗光了。教训是什么？量化模型不管市场情绪。在极度亢奋的市场中，"好公司"不一定涨，"烂公司"不一定跌。量化模型给出了正确的"风险评估"，但无法预测"市场定价偏差会持续多久"。这也是为什么咱们一直强调：量化模型是分析工具，不是交易信号。最终的投资决策还要结合市场环境、资金面和投资者的风险偏好。

构建自己的多因子打分卡

三大经典模型各有侧重，但如果你想把它们的优点结合起来，最好的办法是构建自己的多因子打分卡。

一个实用的四维打分卡框架如下：

  • 盈利能力维度（30分）：ROE（10分）、毛利率趋势（10分）、净利率水平（10分）
  • 成长性维度（25分）：营收增速（10分）、净利润增速（10分）、经营性现金流增速（5分）
  • 财务健康维度（25分）：F-Score（10分）、M-Score排雷（10分）、Z-Score安全（5分）
  • 估值维度（20分）：PE分位数（7分）、PB分位数（7分）、股息率（6分）

总分100分，按分数区间判断投资价值：≥80分为"强烈关注"，60-79分为"可以关注"，40-59分为"观望"，<40分为"回避"。

进阶建议：打分卡的权重不是固定的——在不同市场环境下可以调整。牛市中可以适当调高成长性维度的权重（因为市场预期较高），熊市中可以调高财务健康维度的权重（因为安全第一）。另外，建议你在构建好打分卡之后，先用历史数据回测至少3年——看看你的阈值设置是否有效。

量化模型给了我们一个客观的标尺，但它不能代替人的判断。正如咱们在踩坑故事中看到的——纯靠模型就可能被市场情绪"教育"。最好的方式是量化模型+定性分析相结合：模型帮你锁定范围，人的判断帮你做最终决策。
