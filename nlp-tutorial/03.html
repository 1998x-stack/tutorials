第三章：语言模型基础

大家好！这一章我们来聊语言模型（Language Model, LM）。如果说词向量是NLP的"词汇表"，那么语言模型就是NLP的"语法规则"——它回答一个看似简单的问题：给定一段文本，它有多"像人话"？

我2015年第一次用N-gram做输入法联想时，模型训练出来效果奇差。排查半天发现训练语料是新闻文本，但测试用的是聊天记录——两个领域的语言分布完全不同。这个教训教会我：语言模型的好坏，很大程度上取决于训练数据是否匹配目标场景。

3.1 什么是语言模型

语言模型的数学定义很简洁：计算一个词序列出现的概率。比如P("今天天气真好")应该远大于P("真好天气今天")。用链式法则展开：

# 链式法则：把句子概率分解为条件概率的乘积
P(w1, w2, ..., wn) = P(w1) × P(w2|w1) × P(w3|w1,w2) × ... × P(wn|w1,...,wn-1)

# 例子：P("我 爱 NLP")
# = P("我") × P("爱"|"我") × P("NLP"|"我 爱")

问题是——历史越长，条件概率越难估计。"我爱自然语言处理"的前文有4个词，组合可能性爆炸。这就是语言模型要解决的核心问题：如何在计算可行的情况下近似这个概率。

3.2 N-gram语言模型

N-gram的核心思想是马尔可夫假设：当前词只依赖于前N-1个词，不看更远的历史。

模型假设例子参数量
Unigram (1-gram)词之间完全独立P("我爱NLP") ≈ P(我)P(爱)P(NLP)V
Bigram (2-gram)只看前1个词P(w3|w2)V²
Trigram (3-gram)只看前2个词P(w3|w1,w2)V³
4-gram只看前3个词P(w4|w1,w2,w3)V⁴
# Bigram 模型的计算
# P("我 爱 自然 语言 处理")
# = P(我|<s>) × P(爱|我) × P(自然|爱) × P(语言|自然) × P(处理|语言) × P(</s>|处理)

# 每一项 P(w_i|w_{i-1}) = count(w_{i-1}, w_i) / count(w_{i-1})
# 从语料中直接统计

N的选择是个经验活。N太小（bigram）抓不到足够的上下文，N太大（5-gram+）数据稀疏得没法用。实际中trigram是个不错的折中。

3.3 平滑：解决零概率问题

N-gram最大的痛：语料中没见过的组合，概率就是0——而语言永远有新组合。这需要一个操作叫平滑（Smoothing）。

⚠️ 踩坑记录：我2016年写过一个拼写纠错器，用trigram语言模型。测试时输入"the weather is nice"居然被标记为错误——因为语料里没有"weather is nice"这个trigram。原因就是没有做平滑。加上Kneser-Ney后问题立刻解决。

平滑方法核心思想效果
加一平滑(Laplace)所有计数+1简单但会把太多概率分给没见过的事件
Good-Turing用出现一次的事件的频率来估计未出现事件比Laplace好，但NLP中不常用
Kneser-Ney考虑词的"延续概率"而非频率⭐ 最常用，效果最好
回退(Backoff)N-gram没数据就用(N-1)-gram配合Kneser-Ney使用
# Kneser-Ney 平滑的核心直觉
# 不是"San Francisco"频率高所以好，而是"Francisco"几乎只跟在"San"后面
# P_KN(Francisco|San) 高 ≠ P_KN(Francisco|其他) 高
# 这让Kneser-Ney对高频但不灵活的搭配给低概率

3.4 困惑度：评估语言模型

困惑度（Perplexity, PPL）是语言模型最常用的评估指标。它的本质是：模型对测试集中每个词平均有多少种选择。Perplexity越低越好，说明模型预测越确定。

# Perplexity 公式
# PPL = exp( -1/N × Σ log P(w_i|context) )
#
# 直观理解：
# PPL=10  → 模型平均在10个词中猜测下一个
# PPL=100 → 模型平均在100个词中猜测下一个（更不确定）

# 计算示例
import math
def compute_ppl(probs):
    """probs: 每个token的预测概率列表"""
    log_sum = sum(math.log(p) for p in probs)
    return math.exp(-log_sum / len(probs))

# 如果10个词的预测概率都是0.5 → PPL=2（最优情况）
# 如果10个词的预测概率都是0.01 → PPL=100（很差）

核心要点：Perplexity只在同词汇表的模型之间比较有意义。用BPE的GPT-2 (PPL~35)和用WordPiece的BERT不能直接比PPL。另外，低PPL不等于好生成——PPL衡量的是概率校准，不是生成质量。

3.5 从N-gram到神经网络语言模型

N-gram最大的局限是什么？泛化能力。"我在北京吃了烤鸭"和"我在上海吃了小笼包"在N-gram看来是完全不同的trigram组合。但如果用词向量表示，模型可以学到"北京"和"上海"都是城市，行为类似。

2003年Bengio提出的神经网络语言模型（NNLM）是转折点——用一个简单的全连接网络，输入前N个词的词向量拼接，输出下一个词的概率。虽然今天看来很原始，但它建立了"用神经网络建模语言"的范式。

# 神经网络语言模型的三个层次
# 1. 输入层：前N-1个词的词向量拼接
# 2. 隐藏层：tanh(W × concat(embeddings) + b)
# 3. 输出层：softmax(V × h) → V维度（词汇表大小）的概率分布

# 相比N-gram的优势：
# - 词向量捕捉语义相似性
# - 参数量O(V*d) 而非 O(V^N)
# - 可以泛化到未见过的词组合

NNLM在当年有个致命瓶颈——最后的softmax要算整个词汇表（几万到几十万词），训练极慢。这个问题直到Word2Vec才被优雅地解决（下一章会讲）。
