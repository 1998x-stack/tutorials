第19章 RLVR与推理增强技术

大家好，我是你们的讲师。在上一章我们聊了PEFT，了解了如何用小成本微调大模型。但大家有没有想过一个问题：SFT教模型"答对"，但没教模型"想对"。你给模型一个数学题，它在SFT数据里见过类似题就能答对，换个没见过但需要推理的题就崩了。这就像你让学生把习题集的答案全背下来，考试时题目一换就傻眼。

2025年初，DeepSeek-R1的发布彻底改变了这个局面。它证明了一个激动人心的事实：推理能力是可以被"训练"出来的——而且不需要人工标注思考过程。取而代之的是一套全新的训练范式：RLVR（Reinforcement Learning with Verifiable Rewards）。这一章我们就来深入拆解这个范式背后的算法、实践和前沿进展。

19.1 RLVR的核心思想

RLVR的全称是Reinforcement Learning with Verifiable Rewards，翻译过来就是"可验证奖励的强化学习"。它的核心思路非常朴素：我不需要知道正确答案的推理过程是什么，我只需要能判断最终答案是对是错——然后让模型自己"悟"出正确的思考路径。

这和传统的RLHF有什么区别？在RLHF中，我们训练一个奖励模型（Reward Model）来给模型的输出打分。这个奖励模型需要大量人工标注的偏好数据来训练，而人工标注本身就有噪声、有偏差，还很昂贵。RLVR则完全绕开了奖励模型：reward signal直接来自任务本身的客观验证标准。比如数学题的答案是否正确、代码能不能通过测试用例、逻辑推理结论是否自洽。这些验证不需要人类判断，可以用程序自动完成。

维度 RLHF RLVR
奖励来源 人工训练的Reward Model 任务本身的客观验证
标注成本 高（需要大量偏好标注） 低（自动化验证）
适用场景 主观质量（帮助性、安全性） 客观正确性（数学、代码、逻辑）
奖励噪声 高（标注者偏差） 低（确定性验证）
可扩展性 受限于标注人力 高度可扩展

RLVR的关键洞察是：客观验证信号虽然"稀疏"（只有对或错，没有中间分数），但足够驱动模型学会复杂的推理行为。这其实是一个在强化学习中已经被验证过的思想——只要reward signal是真实的（哪怕很稀疏），通过足够多的探索，agent终将学会最优策略。说白了就是"题海战术"：模型做一万道数学题，做对的被奖励，做错的被惩罚，慢慢地它就自己学会了怎么推导正确答案。

核心洞察：RLVR最被低估的价值不是训练效率，而是数据飞轮。传统SFT每一条数据都需要人类写出"高质量的思考过程"，而RLVR只需要"最终答案的对错"。这意味着你可以在数学网站、编程题库、科学竞赛数据库里自动获取几乎无限量的训练信号。一个团队告诉我，他们用RLVR在3万道自动验证的数学题上训练后，模型在未见过题型上的推理准确率提升了37%。如果这3万条数据都要人工写推理过程，成本至少是百万级别。

19.2 GRPO算法深度

RLVR说起来简单，实际操作中有一个核心难题：怎么从稀疏的reward信号中高效地学习策略？传统PPO算法需要同时维护一个价值网络（Value Model）来估计baseline，这个价值网络的训练本身就需要大量数据和计算资源。DeepSeek团队提出了一种更优雅的解决方案——GRPO（Group Relative Policy Optimization）。

GRPO的核心创新是group normalization。它对同一个prompt采样多个response（比如一组8个），然后用这组response的reward均值作为baseline，而不是依赖一个独立的value network。这样一来，PPO中那个又大又难训的value model就被彻底干掉了。

# GRPO loss的简化示意
# 对于同一个prompt，采样一组response {o1, o2, ..., oG}
# 每个response的reward为 {r1, r2, ..., rG}

# 1. 计算group内的reward均值和标准差
mean_r = mean(r1, r2, ..., rG)
std_r  = std(r1, r2, ..., rG)

# 2. 对每个response计算advantage（归一化reward）
advantage_i = (ri - mean_r) / std_r

# 3. GRPO loss = group内所有response的PPO-clip loss之和
#    不需要value network！
for i in 1..G:
    ratio = pi_theta(oi|prompt) / pi_theta_old(oi|prompt)
    loss_i = -min(ratio * advantage_i,
                  clip(ratio, 1-eps, 1+eps) * advantage_i)

total_loss = sum(loss_i for i in 1..G) / G

GRPO的优势显而易见。第一，省掉value model省了一大笔显存和计算。在PPO中，value model和policy model几乎一样大，训练时需要同时加载两个模型。GRPO只需要一个policy model。第二，group normalization天然做了对比学习——模型会倾向于"做得比group平均水平好"，而不是"达到某个绝对标准"。这让训练更加稳定，不容易出现reward hacking。第三，每个prompt的多个采样天然探索了不同的推理路径，比PPO中单一的on-policy采样更高效。

实践建议：GRPO的group size是一个关键超参数。太小（比如2-4）的话baseline不够稳定，太大（比如64+）的话计算开销剧增。根据DeepSeek-R1论文和社区实践，8-16是最常用的区间。另外注意同一个batch内不同prompt的难度差异——如果一个prompt简单到所有response都答对、另一个难到全军覆没，group内的variance会很低，advantage也不够有区分度。建议在数据准备阶段就把题目按难度分层，每个group内尽量同质。

19.3 DeepSeek-R1训练管线

DeepSeek-R1是RLVR路线最标志性的工作。它的训练管线分为四个阶段，每个阶段的精心设计都值得仔细拆解。

第一阶段：Cold-Start Data（冷启动数据）。DeepSeek-R1-Zero直接用RL从零开始训练base model，结果虽然有效但输出格式混乱——模型会输出中英文混杂、推理链无结构化、甚至出现乱码。DeepSeek-R1的改进是在RL之前先做了一小步SFT：用几千条精心编写的"长推理链"样例做冷启动。这些样例不需要特别多（几千条量级），也不需要覆盖所有题型，只需要示范"合理的推理过程长什么样"。这个冷启动让后面的RL训练收敛快了3倍以上。

第二阶段：大规模RL训练（RLVR）。在冷启动之后，进入核心的GRPO训练。训练数据主要是数学、编程、逻辑推理等具有确定答案的题型。reward signal包括：答案正确性（rule-based verification）、格式合规性（是否在<think>标签内输出推理过程）、语言一致性（是否用同一种语言输出）。这个阶段是模型推理能力"质变"的阶段。

第三阶段：Rejection Sampling + SFT。RL训练到一定程度后，模型已经能产出高质量的推理轨迹了。此时DeepSeek用模型自身生成大量推理数据（rejection sampling），然后用这些数据做一轮SFT。这一步相当于"模型自我蒸馏"——把自己探索出的好路径固化下来。同时在这个阶段混入一些通用领域的SFT数据（写作、翻译、对话等），避免模型只在数理领域变强但其他能力退化。

第四阶段：通用对齐。最后，用传统的RLHF（偏好对齐）对模型进行通用对齐，确保模型在安全性、帮助性方面达到标准。注意这里又回到了RLHF，因为安全偏好不是"verifiable"的——你不能用程序判断一个回答是否安全。

阶段 方法 数据 目标
Cold Start SFT 几千条人工推理示例 建立推理输出格式
RLVR训练 GRPO 数学/代码/逻辑题 激发推理能力
Rejection Sampling SFT 模型自生成的推理数据 固化推理轨迹、扩展通用能力
通用对齐 RLHF 偏好数据 安全性和帮助性

踩坑案例：有个团队在复现R1方案时，跳过冷启动直接上RLVR。他们觉得"反正RL能让model自己学会，省掉写样例的麻烦"。结果训练了两周，模型输出全是诸如"让我们一步一步思考（首先，我们需要分析这个问题...（停顿）...好的，让我再检查一遍...）"这种冗余循环，推理链特别长但答案经常是错的——模型学会了"装模作样地思考"而不是真正的推理。这就是缺了冷启动的后果：模型在探索空间里迷失了方向，reward signal太稀疏导致它找不到正确的行为模式。几千条冷启动数据就是"路标"，引导模型往正确的方向探索。

19.4 "Aha Moment"现象

DeepSeek-R1的训练过程中出现了一个让研究者都感到惊讶的现象——"Aha Moment"。在RL训练的中后期，模型开始自发地展现出一些从未被明确训练过的行为：它会重新读题、会检查自己的计算、会在发现错误后回溯修正，甚至会在推理过程中对自己说"等等，让我再验证一下这个步骤"。

最神奇的是，这些行为在冷启动数据里并没有被显式示范过。冷启动数据只是展示了"一个完整的推理过程"，但没有教模型"检查错误"或"回溯修正"。这些行为完全是模型在RL训练中通过reward信号自发涌现出来的。举个例子，模型可能在推理中先算出x=5，然后代入原题发现矛盾，于是说"不对，如果x=5的话总和不等于42，让我重新算"，然后修正为x=7。这种行为没有人标注过，是模型自己"悟"出来的。

这种涌现现象背后的机制是：reward信号鼓励了"正确的最终答案"，而模型发现"自我纠正"和"验证检查"能提高正确率。在RL的探索过程中，那些偶然做了"自我检查"的轨迹获得了更高reward，于是这种行为被强化。这其实和人类学习的过程非常相似——一个好的老师不会手把手教每一步思考，而是通过结果反馈让学生自己摸索出高效的思考策略。

学术价值："Aha Moment"的存在证明了RLVR不仅仅是在训练"输出格式"，而是在激发模型的元认知能力（metacognition）——模型学会了监控自己的思考过程。这对AI安全也有深层含义：一个会"自我检查"的模型，在生成有害内容或错误信息之前更有可能"悬崖勒马"。DeepSeek-R1的作者在论文中写道："This is a fascinating property of RL that has not been observed in SFT-based reasoning."

在工程实践中，我们也可以主动设计一些"促进涌现"的奖励信号来实现Aha Moment。比如给"包含自我验证步骤"的推理过程额外加分，或者设计prompt template时预留<check>标签的位置。不过要注意尺度的把握——如果奖励系数过高，模型会变成"过度检查强迫症"：每做一步都要检查三次，推理链膨胀到数万token，反而降低了有效推理的密度。

19.5 Test-time Compute

训练出推理能力之后，下一个自然的问题就是：推理时能不能用更多的计算来换取更好的效果？这就是Test-time Compute（推理时计算扩展）的范畴。你可以把它理解为"考试时多花时间检查"——在推理阶段投入更多计算资源，让模型在多个候选中选择最佳答案，或者在更广的搜索空间中寻找正确路径。

Best-of-N采样是最简单有效的方法。对同一个prompt采样N个response，然后用验证器（verifier）选取reward最高的那个。这个方法的效果随着N的增大呈现对数增长——N从1到16的提升很明显，但从16到256的提升就放缓了。在DeepSeek-R1的设置中，Best-of-64相比单次采样在数学推理准确率上提升了大约8-12个百分点。

# Best-of-N with verifier
def best_of_n(prompt, model, verifier, n=16):
    candidates = []
    for i in range(n):
        response = model.generate(prompt, temperature=0.7)
        score = verifier(response)  # 验证器评分
        candidates.append((score, response))
    # 选择评分最高的response
    return max(candidates, key=lambda x: x[0])[1]

# Process Reward Model guided search
def prm_beam_search(prompt, model, prm, beam_width=4, max_steps=8):
    beams = [(prompt, 0.0)]
    for step in range(max_steps):
        candidates = []
        for seq, score in beams:
            # 扩展若干step的推理
            continuations = model.generate_continuations(seq, n=beam_width)
            for cont in continuations:
                step_score = prm.score(cont)  # PRM给每一步打分
                candidates.append((cont, score + step_score))
        # 保留top-k beam
        beams = sorted(candidates, key=lambda x: -x[1])[:beam_width]
    return beams[0][0]

Beam Search是另一个思路。不同于Best-of-N的独立采样，beam search在推理过程中做"宽优先搜索"——每一步保留K个最优的中间状态，每一步都做剪枝。它的优势在于探索更加定向集中，但实现复杂度更高，因为需要能在推理的中间步骤做截断和比较。

Process Reward Model（PRM）guided search是目前最前沿的方法。与传统的outcome reward model（只对最终答案打分）不同，PRM对推理的每一步都给出一个分数——"到目前为止的推理是合理的吗？"。这相当于在模型思考的每个环节都配了一个"督学"，一旦发现思考方向偏离就立刻纠正。OpenAI的o1系列和DeepSeek-R1都在这个方向上做了大量工作。不过PRM的训练本身就是个挑战——你需要标注推理链每一步的质量，这可比标注最终答案贵多了。

方法 计算开销 效果提升 实现难度
Best-of-N O(N) 中等（对数增长） 低
Beam Search O(K * steps) 较高 中等
PRM Guided Search O(steps * candidates) 高 高

工程实践：Test-time Compute有一个经典的tradeoff需要你根据场景做选择。如果推理延迟是核心约束（比如线上API服务），Best-of-N的并行采样可以让所有N个response同时生成，延迟几乎不变，只有verifier的额外计算。如果推理质量是核心约束（比如离线批量推理、科研分析），PRM guided search能带来更好的边际收益。我建议你的MVP先从Best-of-8或Best-of-16起步，上线后再根据质量瓶颈决定是否升级到更复杂的搜索策略。

RLVR和推理增强技术正在重塑整个LLM训练范式。以前我们关注的是"模型知道什么"，现在关注的是"模型怎么思考"。这个转变的意义不亚于从"统计语言模型"到"预训练-微调"的跃迁。下一章我们要聊第20章「指令微调与Agent能力」，从推理的"内部思考"走向Agent的"外部行动"——不仅要让模型会想，还要让它会用工具、会对话、会规划。这是通往AGI路上必不可少的一步，咱们下章见。
