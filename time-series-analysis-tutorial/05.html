第5章 指数平滑

大家好，我是你们的讲师。你有没有想过一个问题：如果需要预测明天的气温，你会怎么估？大部分人会说："看今天的温度，再稍微参考一下前几天的。"这种直觉其实就是指数平滑（Exponential Smoothing）的核心思想——越近的数据越重要，越远的数据权重越小。

指数平滑是时间序列预测中最古老也最经典的方法之一，诞生于20世纪50年代。尽管后来有了 ARIMA、深度学习这些更复杂的模型，指数平滑依然在大量实际场景中占据一席之地。原因无它——它极其简单、计算效率高、在适当的场景下预测效果不输复杂模型。从我个人的项目经验来看，在零售预测、库存管理这类场景中，一个调好参数的 Holt-Winters 模型经常能跑赢新手调的 LSTM。

说一个真实案例。2019年我参与了一个连锁超市的库存优化项目，每种商品需要预测未来4周的周销量。团队一开始就上了 Prophet、SARIMA 等"重武器"，折腾了一个月还没调好。后来我们用 Holt-Winters 指数平滑作为 baseline，只花了一周，预测精度已经达到业务要求的85%。最终上线的生产线，有一半商品用的就是指数平滑。这个故事告诉我们：不要看不起简单的模型。

5.1 从移动平均到指数平滑

在指数平滑出现之前，最朴素的预测方法是移动平均（Moving Average）：

预测值 = (Y_t + Y_{t-1} + ... + Y_{t-N+1}) / N

移动平均给所有观测值一样的权重，但直觉告诉我们——昨天的数据应该比30天前的数据更有参考价值。指数平滑把这个直觉变成了数学：

预测值 = α * 当前值 + (1 - α) * 上一个预测值

展开后你会发现，这实际上给历史数据分配了指数衰减的权重。α 越大，模型对最新数据越敏感；α 越小，模型越平滑。

面经精选：面试中经常被问到"指数平滑和移动平均的区别是什么？"标准答案有三点：(1) 指数平滑给近期数据更高权重，移动平均等权重；(2) 指数平滑需要存储的数据更少（只需上一个预测值和当前值）；(3) 移动平均的窗口大小是离散的整数选择，α 是连续的——调参更灵活。

5.2 三种指数平滑方法

简单指数平滑（Simple Exponential Smoothing / SES）

适用于没有趋势和季节性的数据。只有一个参数 α。

预测公式：Ŷ_{t+1} = α * Y_t + (1 - α) * Ŷ_t

对于 h 步预测：Ŷ_{t+h} = Ŷ_{t+1}（所有未来预测值相同，因为没趋势）

Holt 线性趋势法（Holt's Linear Trend）

增加了趋势成分，有两个参数：α（水平）和 β（趋势）。

水平方程：L_t = α * Y_t + (1 - α) * (L_{t-1} + T_{t-1})

趋势方程：T_t = β * (L_t - L_{t-1}) + (1 - β) * T_{t-1}

h 步预测：Ŷ_{t+h} = L_t + h * T_t

Holt-Winters 季节性方法

在 Holt 的基础上再增加季节成分，有三个参数：α（水平）、β（趋势）、γ（季节）。

这是指数平滑家族的旗舰版本，也是实际应用中最常用的。后面我们单独展开。

5.3 加法季节性与乘法季节性

Holt-Winters 有两种季节模式：

  • 加法季节性——季节波动幅度恒定。公式：Y_t = 水平 + 趋势 + 季节。适用于温度、湿度等自然量
  • 乘法季节性——季节波动幅度随水平变化。公式：Y_t = (水平 + 趋势) × 季节。适用于销售额、流量等经济量

怎么判断用哪一个？看数据图：如果季节波峰的绝对高度每年差不多，用加法；如果波峰高度随趋势增长而增长，用乘法。

方法参数趋势季节性典型应用
SESα无无白噪声过程、平稳序列
Holtα, β线性无缓慢增长的业务指标
Holt-Winters 加法α, β, γ线性恒定振幅气温、水位
Holt-Winters 乘法α, β, γ线性比例振幅零售额、网站流量

5.4 ETS 框架

ETS（Error-Trend-Seasonality）框架由 Hyndman 等人系统化提出，把指数平滑家族统一成了一个分类体系。ETS 用三个字母来描述一个模型：

  • 第一个字母：误差（Error）——加法（A）或乘法（M）
  • 第二个字母：趋势（Trend）——无（N）、加法（A）或衰减（Ad）
  • 第三个字母：季节（Seasonality）——无（N）、加法（A）或乘法（M）

举个例子：

  • ETS(A,N,N)——加法误差、无趋势、无季节。就是简单指数平滑。
  • ETS(A,A,A)——加法误差、加法趋势、加法季节。Holt-Winters 加法版本。
  • ETS(M,A,M)——乘法误差、加法趋势、乘法季节。Holt-Winters 乘法版本。

ETS 框架的一个巨大优势是：你可以通过 AIC 在不同 ETS 组合中自动选取最优模型，不需要人工一个个试。

5.5 Python 实现

import pandas as pd
import numpy as np
from statsmodels.tsa.holtwinters import ExponentialSmoothing, SimpleExpSmoothing
import matplotlib.pyplot as plt

# 生成模拟数据：趋势 + 季节性
t = np.arange(1, 145)  # 2年周数据
trend = 10 + 0.05 * t
seasonal = 5 * np.sin(2 * np.pi * t / 12)
data = trend + seasonal + np.random.normal(0, 1, 144)

# Holt-Winters 加法季节模型
model = ExponentialSmoothing(
    data,
    trend='add',
    seasonal='add',
    seasonal_periods=12
)
fit = model.fit()
print(fit.params)

# 预测未来 12 期
forecast = fit.forecast(12)

# 绘制
plt.plot(data, label='Actual')
plt.plot(fit.fittedvalues, label='Fitted')
plt.plot(range(len(data), len(data)+12), forecast, label='Forecast')
plt.legend()
plt.show()
参数初值警告：statsmodels 的 ExponentialSmoothing 默认使用启发式方法初始化参数，这大多数情况下工作良好。但对于某些数据（特别是季节性特别强或趋势突变的数据），初始值可能不合适，导致模型收敛到局部最优。解决方法是尝试不同的初始值：fit(initial_level=value)。另外，建议总是设置 fit(optimized=True) 让库自动优化参数，而不是手动指定 α、β、γ。

5.6 参数解读与业务含义

指数平滑的参数不只是数学工具，它们有直观的业务含义：

  • α 接近 1——模型几乎只相信最新观测值。如果你的业务变化极快（比如时尚零售），α 应该大
  • α 接近 0——模型平滑程度很高，几乎不受新数据影响。如果业务非常稳定（比如公共事业消费），α 可以小
  • β 接近 0——趋势基本不变。长期稳定的增长场景
  • γ 接近 0——季节模式基本固定。比如圣诞节效应每年相同

一般来说，α 在 0.1~0.3 之间效果较好。如果最优 α 超过 0.5，说明数据的信噪比很低，可能要考虑是否是数据质量的问题。

指数平滑优雅地解决了"给历史数据赋权"的问题，但它的局限性也很明确：它假设数据有稳定的趋势和季节模式，无法很好地处理突变、结构性断点。下一章我们将换一个视角，用回归的方法来建模时间序列，看看如何把时间作为解释变量纳入模型，同时也看看这会带来什么新的问题。
