第22章 多模态推理

vLLM 不只是文字模型——它完整支持图像、视频、音频等多种模态。2025-2026 年，多模态推理是最活跃的功能方向之一。

22.1 支持的多模态模型

模态代表模型V1 支持状态
图像（Vision）Qwen2.5-VL, Qwen3-VL, InternVL2, LLaVA, Pixtral, Phi-3-Vision🟢 完备
视频LLaVA-NeXT-Video, Qwen2-VL (video)🟢 完备
音频Whisper (encoder-decoder)🟢 完备
混合/通用Chameleon, Fuyu, PALIGEMMA🟢 完备

22.2 多模态推理流程

图像输入 (224×224 RGB)
    │
    ▼
┌──────────────┐
│ Vision Encoder│  ViT/CLIP/SigLIP
│ (ViT)        │  图像 → visual tokens (如 256 个)
└──────┬───────┘
       │ visual tokens
       ▼
┌──────────────────────────────────────────────┐
│ Prompt: [IMAGE] "这张图片里有什么？"           │
│ ↓ Embedding                                  │
│ [visual_tokens | text_tokens]                │
│ ↓                                            │
│ LLM Backbone (共享 Transformer)              │
│ ↓                                            │
│ "这张图片里有一只猫坐在沙发上。"                  │
└──────────────────────────────────────────────┘

22.3 使用示例

# 启动多模态模型
vllm serve Qwen/Qwen2.5-VL-7B-Instruct \
    --max-model-len 8192 \
    --limit-mm-per-prompt image=16

# Python 调用
from vllm import LLM, SamplingParams

llm = LLM("Qwen/Qwen2.5-VL-7B-Instruct")
sampling_params = SamplingParams(temperature=0.1, max_tokens=256)

outputs = llm.generate([{
    "prompt": "描述这张图片的内容。",
    "multi_modal_data": {"image": "photo.jpg"}
}])

# 多图 + 文本混合
outputs = llm.generate([{
    "prompt": "图1 和图2 有什么区别？比较两者的不同。",
    "multi_modal_data": {
        "image": ["img1.jpg", "img2.jpg"]
    }
}])

22.4 Vision Encoder CUDA Graphs（2026 新特性）

ViT 编码器（视觉模型的"眼睛"）传统上没有被 CUDA Graph 覆盖。V1 的 Vision Encoder CUDA Graphs 把 ViT 也纳入 graph capture：

指标之前之后
P99 延迟 (4×GPU)基准-84.9%
token budget 支持固定2048/4096/8192/13824 动态选择
Batch 调度FCFSGreedy bin-packing

22.5 LoRA 适配器

如果你有不同的任务需要同一个基座模型的不同适配：

# 启动时加载多个 LoRA
vllm serve meta-llama/Llama-3-8B \
    --enable-lora \
    --max-lora-rank 64 \
    --lora-modules \
        math-lora=/loras/math/ \
        code-lora=/loras/code/ \
        chat-lora=/loras/chat/

# API 请求时指定
curl http://localhost:8000/v1/chat/completions \
    -d '{
        "model": "math-lora",
        "messages": [{"role": "user", "content": "解方程 x²+3x-4=0"}]
    }'

vLLM 使用 Punica 算法实现高效的多 LoRA 批处理——Se segmented Gather 矩阵乘法让不同 LoRA 的请求在同一 batch 中高效处理。

当前局限：多模态模型 + LoRA 的组合支持有限（取决于模型架构）；推测解码 + LoRA 和 多模态+推测解码组合暂不支持。涉及这些场景前先在测试环境验证。
