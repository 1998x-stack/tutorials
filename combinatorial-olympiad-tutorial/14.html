第十四章：数学归纳法与加强命题

大家好，我是你们的讲师。数学归纳法可能是你在数学课上学到的第一个"高级"证明方法——但它也是最容易被低估的方法。很多同学觉得归纳法就是"n=1成立，假设n成立推n+1成立"，但真正在竞赛中需要用到归纳法时，往往发现原命题的归纳假设根本不够用。

比如这样一个问题："证明 1 + 1/2² + 1/3² + ... + 1/n² < 2。" 如果你假设 n 时不等式成立，想推 n+1 时也成立——你会发现 n+1 时左边比 n 时多了一项 1/(n+1)²，而右边还是 2。但归纳假设只告诉你 n 时左边 < 2，加上 1/(n+1)² 后你还怎么保证 < 2？归纳假设不够强！

这就是为什么我们需要加强命题——有时你必须先证明一个更强的结论，才能让归纳法顺利推进。

14.1 为什么需要"加强命题"

先看一个最经典的例子来说明问题。

例1：求证 1 + 1/2² + 1/3² + ... + 1/n² < 2。

尝试直接归纳：

  • n=1：1 < 2，成立。
  • 假设 n 时成立，即 1 + 1/2² + ... + 1/n² < 2。
  • 那么 n+1 时：1 + 1/2² + ... + 1/n² + 1/(n+1)² < 2 + 1/(n+1)² → 不能推出 < 2。

归纳失败了，因为增加了一个正项，不等式右端没有增加。

解决方案：加强命题

我们证明一个更强的结论：1 + 1/2² + 1/3² + ... + 1/n² ≤ 2 - 1/n

这个新的命题对原命题是"加强"——如果它成立，原命题自然成立（因为 2 - 1/n < 2）。

用归纳法证明加强后的命题：

  • n=1：1 ≤ 2 - 1 = 1，成立。
  • 假设 n 时成立：1 + 1/2² + ... + 1/n² ≤ 2 - 1/n。
  • n+1 时：左边 = (1 + ... + 1/n²) + 1/(n+1)² ≤ 2 - 1/n + 1/(n+1)²
  • 需要证明 2 - 1/n + 1/(n+1)² ≤ 2 - 1/(n+1)
  • 即证 -1/n + 1/(n+1)² ≤ -1/(n+1)
  • 即 1/(n+1) + 1/(n+1)² ≤ 1/n
  • 通分：左边 = (n+1+1)/(n+1)² = (n+2)/(n+1)²，右边 = 1/n
  • 交叉相乘：n(n+2) = n²+2n ≤ (n+1)² = n²+2n+1 ☑

加强命题归纳成立，原命题得证。

加强命题的精髓：
原命题：P(n)  不够强 → 归纳断裂
加强后：Q(n)  足够强 → 归纳顺畅
关系：Q(n) ⇒ P(n)  （加强后的命题推出原命题）

关键：如何找到合适的 Q(n)？
通常是在原命题中加入一个"余项"或"松弛量"，
让归纳步的推导能够闭合。

什么时候应该加强命题——以下几个信号值得警惕：

  1. 不等式归纳：右边是常数（如上面的 < 2），左边逐项增加。这种几乎必然需要加强。
  2. 递推关系需要更强的结构性假设：比如证明某种构造方案的存在性时，只假设"存在一组解"往往不够，需要假设"存在一组具有某种额外性质的解"。
  3. 多个变量同时出现：对 n 个变量成立的性质，推广到 n+1 个时，需要假设 n 个变量的某种性质是"对任意 n 个元素都成立"而不是"对具体的某 n 个元素成立"。

14.2 经典案例：均值不等式的归纳证明

均值不等式（AM-GM）是数学中最基本的不等式之一。它的归纳证明是"加强命题"和"Cauchy 向前向后归纳法"的经典范例。

均值不等式：对任意正数 a₁, a₂, ..., aₙ，有：

(a₁ + a₂ + ... + aₙ)/n ≥ ⁿ√(a₁·a₂·...·aₙ)

等号成立当且仅当所有 aᵢ 相等。

这个不等式通常用 Cauchy 的向前-向后归纳法（Forward-Backward Induction）证明。这个证明不需要加强命题本身，但它展示了一种精妙的归纳变形。

证明思路：

第一步（向后归纳）：证明如果对 n 成立，则对 n-1 也成立。

设均值不等式对 n 个正数成立。对于 n-1 个正数 a₁,...,aₙ₋₁，令 aₙ = (a₁+...+aₙ₋₁)/(n-1)（即前 n-1 个数的平均值）。由 n 情形的 AM-GM：

(a₁ + ... + aₙ₋₁ + A)/n ≥ (a₁·...·aₙ₋₁·A)^{1/n}

其中 A = (a₁+...+aₙ₋₁)/(n-1)。

左边 = (nA)/n = A。所以 A ≥ (a₁·...·aₙ₋₁·A)^{1/n}。

两边 n 次方：Aⁿ ≥ a₁·...·aₙ₋₁·A，所以 Aⁿ⁻¹ ≥ a₁·...·aₙ₋₁。

两边开 n-1 次方即得 A ≥ (a₁·...·aₙ₋₁)^{1/(n-1)}，这正是对 n-1 情形的 AM-GM。

第二步（向前归纳）：证明如果对 n 成立，则对 2n 也成立。

对于 2n 个正数 a₁,...,a₂ₙ，由 n 情形的 AM-GM：

(a₁ + ... + aₙ)/n ≥ (a₁...aₙ)^{1/n}
(aₙ₊₁ + ... + a₂ₙ)/n ≥ (aₙ₊₁...a₂ₙ)^{1/n}

令 A = (a₁+...+aₙ)/n，B = (aₙ₊₁+...+a₂ₙ)/n。那么 (A+B)/2 ≥ √(AB)（n=2 时的 AM-GM）。

而 (a₁+...+a₂ₙ)/(2n) = (A+B)/2 ≥ √(AB) ≥ √[(a₁...aₙ)^{1/n} · (aₙ₊₁...a₂ₙ)^{1/n}] = (a₁...a₂ₙ)^{1/(2n)}。

第三步：组合两步。

先对 n=2（显然成立），用第二步推出 n=4,8,16,... 成立（所有 2 的幂次）。

再用第一步从较大的 2ᵏ 倒退到所有小于 2ᵏ 的自然数。

由此 AM-GM 对任意正整数 n 成立。

向前-向后归纳法的模式：

  1. 证明基础情形（如 n=2）
  2. 证明"n → 2n"（向前跳，覆盖所有 2 的幂）
  3. 证明"n → n-1"（向后推到所有小于 2ᵏ 的数）

这个模式在无法从 n 直接推到 n+1 时特别有用——先跳到足够大的数，再退回来。

14.3 归纳法处理图论问题

图论是归纳法应用最频繁的领域之一。归纳变量通常选顶点数或边数。

14.3.1 对顶点数归纳

例2：证明任意 n 个顶点的树（连通无环图）恰有 n-1 条边。

对顶点数 n 归纳：

  • n=1：一个顶点，0 条边，成立。
  • 假设对 n 个顶点的树成立（有 n-1 条边）。考虑 n+1 个顶点的树 T。
  • 在 T 中取一个叶子（度数为 1 的顶点，树中一定存在），去掉这个叶子及其相连的边，得到 T'。
  • T' 仍然是树（因为去掉叶子不会产生环，且仍然连通），有 n 个顶点。
  • 由归纳假设，T' 有 n-1 条边。所以 T 有 (n-1) + 1 = n 条边。

证毕。这里的关键是"去掉一个叶子"——利用树的极端结构来简化问题。

14.3.2 对边数归纳

例3：证明任意连通图 G 至少包含一个生成树（包含所有顶点的极小连通子图）。

对边数 m 归纳：

  • 如果 G 没有环（本身就是树），那么 G 就是自己的生成树。
  • 如果 G 有环，去掉环中的任意一条边 e。G-e 仍然连通（因为环去掉一边不破坏连通性），边数减少 1。
  • 由归纳假设，G-e 有生成树 T。T 也是 G 的生成树。
  • 证毕。

这个归纳极其简单——但它推导出了一个重要的结论：任何连通图都有生成树，而且可以通过不断删环得到。

14.3.3 对顶点数归纳+加强命题

例4：证明任何 n 个顶点的竞赛图（complete directed graph，每对顶点间恰有一条有向边）包含一条哈密顿路径（经过所有顶点恰好一次的有向路径）。

对顶点数 n 归纳：

  • n=1,2 显然。
  • 假设对 n-1 个顶点的竞赛图成立。
  • 考虑 n 个顶点的竞赛图 T。去掉顶点 v，剩下 n-1 个顶点的竞赛图 T' 有哈密顿路径 P' = v₁ → v₂ → ... → vₙ₋₁。
  • 现在需要把 v 插回路径中。分情况讨论 v 与 P' 中各顶点的有向边方向：
  • 如果存在边 v → v₁，则在开头插入：v → v₁ → ... → vₙ₋₁。
  • 如果存在边 vₙ₋₁ → v，则在结尾插入：v₁ → ... → vₙ₋₁ → v。
  • 否则，v₁ → v 且 v → vₙ₋₁。那么在路径中找第一个满足 v → vᵢ+1 的点 vᵢ（这样的点一定存在，因为 vₙ₋₁ → v 但 v₁ → v，所以从 v₁ 到 vₙ₋₁ 必然有方向变化——在某个位置，箭头从"指向 v"变成"从 v 指出"）。
  • 在 vᵢ 和 vᵢ₊₁ 之间插入 v：... → vᵢ → v → vᵢ₊₁ → ...

这个证明中，归纳假设自然适用——不需要加强。但如果题目问的是"证明竞赛图包含哈密顿回路"（经过所有顶点回到起点），那就需要加强命题了（需要假设更强的结构性条件）。

图论中归纳法的使用指南：

对顶点数归纳：
1. 去掉一个顶点（和它的边）
2. 对剩下的图应用归纳假设
3. 把去掉的顶点加回来

对边数归纳：
1. 去掉一条边（或删环、合并顶点等）
2. 对剩下的图应用归纳假设
3. 处理移除边的影响

14.4 归纳构造的常见模式

在组合中，归纳法不仅用于证明，还可以用于构造——假设 n 的情形已构造好，然后将其"扩展"到 n+1 的情形。

例5：对任意 n ≥ 1，构造一个 n×n 的 0-1 矩阵，使得每一行、每一列都恰好有两个 1。

用归纳法构造：

  • n=2：矩阵 [[1,1],[1,1]]，每行每列各两个 1，成立。
  • 假设已构造了 n×n 矩阵 Aₙ 满足条件。来构造 Aₙ₊₁：
  • 取 Aₙ，在右下角加一行一列：
Aₙ₊₁ = [ Aₙ    0  ]
        [       1  ]
        [ 0 1   0  ]
        [ 1 0   1  ]

更规范地：
将 Aₙ 放在左上角。在新行的第 n 列放 1、第 n+1 列放 1。
在新列的第 n 行放 1、第 n+1 行放 1。其余为 0。

验证：原来的 n 行每行多了第 n+1 列的一个 0，所以每行还是两个 1（来自 Aₙ）。
原来的 n 列同理。新行在第 n 列和第 n+1 列有 1，共两个 1。新列同理。
所有条件满足！

这个归纳构造的思路是：把新的元素"接"在已有的结构上，同时保证所有条件仍然满足。在竞赛中，很多构造题的答案都可以用这种"n→n+1"的模式得到。

归纳构造的核心技巧：

当你需要构造一个规模为 n 的对象时，先假设你能构造一个规模为 n-1 的类似对象，然后思考：

  1. 附加法：在已有的结构上添加新元素，并适当调整
  2. 拆分法：把 n 拆成 n₁+n₂ 两个更小的规模，分别构造再组合
  3. 倍增法：用两个 n/2 规模的对象拼成一个 n 规模的对象

附加法最常用，但要注意添加新元素后不能破坏旧结构已经满足的条件。

14.5 反向归纳法

反向归纳法（Backward Induction）是一种特殊的归纳形式——它不是从 n 推到 n+1，而是从 n 推到 n-1。

反向归纳法的逻辑是：

  1. 证明命题对无穷多个 n 成立（通常是通过正向归纳或直接构造，证明对所有 2ᵏ 成立）
  2. 证明如果命题对 n 成立，则对 n-1 也成立
  3. 由 (1) 和 (2)，命题对所有正整数成立

我们已经在 AM-GM 的证明中看到了这种模式——先正向跳到 2 的幂，再反向退回到所有整数。

例6（Jensen 不等式的反向归纳证明）：

Jensen 不等式说，对于凸函数 f，有 f((x₁+...+xₙ)/n) ≤ (f(x₁)+...+f(xₙ))/n。

反向归纳证明：

  • 对 n=2，由凸函数定义直接成立。
  • 用 n=2 反复应用可证明对所有 2ᵏ 成立（正向归纳）。
  • 假设对 n 成立。令 y = (x₁+...+xₙ₋₁)/(n-1)，由 n 情形：
f((x₁+...+xₙ₋₁ + y)/n) ≤ (f(x₁)+...+f(xₙ₋₁)+f(y))/n

但 (x₁+...+xₙ₋₁ + y)/n = ((n-1)y + y)/n = y

所以 f(y) ≤ (f(x₁)+...+f(xₙ₋₁)+f(y))/n
两边乘以 n - f(y)：n·f(y) ≤ f(x₁)+...+f(xₙ₋₁)+f(y)
即 (n-1)f(y) ≤ f(x₁)+...+f(xₙ₋₁)
即 f(y) ≤ (f(x₁)+...+f(xₙ₋₁))/(n-1)
这正是对 n-1 情形的 Jensen 不等式。

这样就从 n 推到了 n-1，再由 2ᵏ 覆盖所有正整数。

什么时候用反向归纳？

  • 当从 n 到 n+1 的推导比较困难，但从 n 到 n-1 的推导很自然时
  • 当"加一个元素"会破坏结构（需要很多条件才能加），但"减一个元素"很容易时
  • 当你知道命题对所有 2 的幂次成立时（比如可以用倍增法证明）

14.6 归纳法的常见误区

我教了这么多年竞赛，总结了以下归纳法最常见的错误：

错误类型描述改正方法
归纳假设过弱只用"存在性"假设，但需要"对所有可能"都成立才能推下一步加强命题，把"存在"改为"任意"或"所有"
跳过太多从 k 推到 k+1 时，用到了 k-1 或更早的结果改为强归纳法（假设所有小于 n 的情形成立）
忽略基础情形只注重了递推步，但 n=1 时命题不成立多验证几个基础值
循环论证在推 n+1 时，不恰当地使用了 n+1 情形的结论检查每一步是否只用了归纳假设
归参数选择错误对 n 归纳但实际需要的是对 k 归纳选择合适的归纳变量（比如顶点数 vs 边数）

⚠️ 踩坑实录：有一次我在课堂上出了这样一道题："证明任意 n 条直线（无两条平行、无三线共点）将平面分成 n(n+1)/2 + 1 个区域。"

一个学生这样证：假设 n 条直线分成 R(n) 个区域。添加第 n+1 条直线，它与前面 n 条直线各交于一点，被分成 n+1 段，每段将一个原有区域一分为二，所以增加 n+1 个区域。所以 R(n+1) = R(n) + n+1。结合 R(1)=2，递推得到 R(n)=n(n+1)/2+1。

这个推导本身是对的，但他没有用数学归纳法的标准形式——他直接用了递推公式。严格地说，如果要用归纳法，应该写清楚：假设 R(n)=n(n+1)/2+1，然后证明 R(n+1)=(n+1)(n+2)/2+1。虽然只是形式上的差别，但在竞赛答卷中，清晰地写出归纳假设和推导步骤是得分的关键。很多学生省略了"由归纳假设"的字样，导致扣分。

另外还有一个学生更惨——他假设 n 条直线有 R(n) 个区域，然后想证明 n+1 条直线有 R(n)+n+1 个区域。但他用了"因为第 n+1 条线与前 n 条线交于 n 个点，所以新增 n+1 个区域"——这句话严格来说需要证明。他简单地用"显然"带过了，但评卷老师认为这一步缺少论证，扣了过程分。

教训：归纳法中的每一步都应该清晰地写出来——归纳假设、归纳步骤、每个等式的推导来源。不要偷懒，不要用"显然"跳过关键步骤。

14.7 本章小结与过渡

本章我们学习了数学归纳法在组合问题中的高级应用。核心要点是：当直接归纳遇到瓶颈时，加强命题是突破的关键——通过证明一个更强的结论来让归纳假设变得足够"有力"。我们还学习了向前-向后归纳法（Cauchy 经典技巧）、图论中的归纳技巧、归纳构造的一般模式，以及反向归纳法的应用场景。

归纳法与加强命题是连接"小规模成立"和"大规模成立"的桥梁。在组合竞赛中，它是最基础也最灵活的证明方法之一。有时候你可以直接归纳，有时候你需要先绕个弯（加强命题或双向归纳）——这些技巧都需要通过大量练习来内化。

至此，我们完成了"核心方法"部分的全部内容。从第八章的抽屉原理到第十四章的归纳法，我们已经建立了一个强大的组合方法论工具箱。下一章我们将进入第三部分——进阶技巧，首先学习的是概率方法，一种用概率论的语言来解决组合存在性问题的精妙方法。
