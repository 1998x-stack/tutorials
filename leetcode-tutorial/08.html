递归思维 — 二叉树问题的核心方法论

大家好，我是你们的讲师。从这一章开始，我们正式进入树的世界。如果说数组和链表是线性的战场，那么树就是非线性的天地——而二叉树，则是这个天地中最基础也最重要的结构。翻开 LeetCode 的 Tree 标签，你会发现从简单到困难，题目多达 200+ 道，而其中 90% 以上用到了递归。

为什么递归和二叉树这么"般配"？因为二叉树天然具有递归结构：一棵树由根节点、左子树和右子树组成，而每一棵子树本身又是一棵树。这种自相似性决定了递归是处理二叉树问题最自然的方式。当你面对一道二叉树题目时，第一步永远不是写代码，而是想清楚两件事：Base Case（递归终止条件）是什么？递推公式是什么？

拿二叉树的最大深度（LeetCode 104）来说，这几乎是最简单的二叉树递归题：如果节点为空，深度为 0；否则，深度 = max(左子树深度, 右子树深度) + 1。这就是最标准的 base case + 递推公式。

def max_depth(root):
    if not root:          # base case
        return 0
    left = max_depth(root.left)
    right = max_depth(root.right)
    return max(left, right) + 1  # 递推公式

这段代码虽然只有五行，但揭示了二叉树递归的核心模式：先处理当前节点，还是先处理子树？前者叫前序位置，后者叫后序位置。这个区别至关重要。

递归心法：写递归函数时，不要钻进递归的细节中去——你不必在脑子里模拟整个调用栈。你只需要相信两件事：1）base case 是正确的；2）递归调用返回的是正确结果。这种"信仰之跃"是每个递归高手的基本素质。

前序位置和后序位置的区别，打个比方：前序是"到办公室先打卡"，后序是"下班前再打卡"。如果我们要统计全公司的人数——在前序位置加一，和在后序位置加一，最终结果一样，但执行时机完全不同。对于某些需要"先知道子树信息才能处理当前节点"的问题（比如判断平衡二叉树、求直径），后序位置就是唯一的选择。

二叉搜索树性质

二叉搜索树（BST）是二叉树中最重要的特化类型。它的定义很简单：左子树所有节点值 < 根节点值 < 右子树所有节点值。但这个简单的定义衍生出了极其强大的性质。

中序遍历有序

BST 最重要的性质：中序遍历得到的是递增序列。这个性质被大量题目所利用。比如说，LeetCode 230——二叉搜索树中第K小的元素，最直接的思路就是中序遍历，取第 k 个元素即可。

def kth_smallest(root, k):
    stack = []
    cur = root
    while cur or stack:
        while cur:
            stack.append(cur)
            cur = cur.left
        cur = stack.pop()
        k -= 1
        if k == 0:
            return cur.val
        cur = cur.right

这个迭代版中序遍历的核心思想：先把所有左子节点压栈，然后依次弹出，每弹一个就去试探它的右子树。整个过程模拟了递归的调用栈，但省去了递归的开销。当 k 比较小时（比如 k=1 找最小值），这个解法可以在找到结果后立即返回，效率很高。

验证二叉搜索树

LeetCode 98——验证二叉搜索树（Validate Binary Search Tree），这是一道高频中的高频题，也是一道容易想简单但实际有陷阱的问题。

很多新手会写出这样的代码：对每个节点，检查左子节点 < 当前节点 < 右子节点。但这不对！BST 要求的是左子树所有节点都小于根节点，不仅仅是左子节点。看个反例：根节点 5，左子节点 3，左子节点的右子节点是 6——每个节点和它相邻子节点的关系都满足，但 6 > 5，它不该出现在左子树里。

正确的做法是给每个节点传入一个取值范围：

def is_valid_bst(root):
    def dfs(node, low, high):
        if not node:
            return True
        if node.val <= low or node.val >= high:
            return False
        return dfs(node.left, low, node.val) and dfs(node.right, node.val, high)

    return dfs(root, float('-inf'), float('inf'))

每个节点都有一个下限 low 和上限 high，左子树的上限收紧为当前节点值，右子树的下限收紧为当前节点值。这个技巧是 BST 题目的万能模板。

易错点：题目如果明确说"BST 中节点的值互不相同"，那用 < 和 > 就够了。但如果没说，记得考虑等于的情况——严格小于 / 严格大于还是允许等于？读题时一定要注意这一点。

最近公共祖先

最近公共祖先（Lowest Common Ancestor, LCA）是二叉树题目中的经典，也是面试中的常客。LeetCode 上有两道题：一道是普通二叉树的 LCA（236），一道是 BST 的 LCA（235）。解法思路完全不同。

二叉树的 LCA（236）

普通二叉树没有排序信息，我们只能通过后序遍历来查找。核心思想：在左右子树中分别寻找 p 和 q，如果左子树找到了一个，右子树找到了另一个，那么当前节点就是 LCA。

def lowest_common_ancestor(root, p, q):
    if not root or root == p or root == q:
        return root

    left = lowest_common_ancestor(root.left, p, q)
    right = lowest_common_ancestor(root.right, p, q)

    if left and right:   # p 和 q 分别在左右子树
        return root
    return left or right  # 其中一个为空，返回非空的那边

这个解法的精妙之处：base case 不仅处理了空节点，也处理了"当前节点就是 p 或 q"的情况——这意味着如果 p 是 q 的祖先，直接返回 p，无需继续寻找。

BST 的 LCA（235）

BST 有大小关系，所以可以利用值来导航：如果 root.val 大于 p 和 q 的值，说明 p 和 q 都在左子树；如果 root.val 小于 p 和 q 的值，说明在右子树；否则，当前节点就是 LCA（或者 p、q 中的一个就是 LCA）。

def lowest_common_ancestor_bst(root, p, q):
    while root:
        if root.val > p.val and root.val > q.val:
            root = root.left
        elif root.val < p.val and root.val < q.val:
            root = root.right
        else:
            return root

对比两道题：236 是通用的递归后序遍历，时间复杂度 O(n)；235 利用 BST 性质使用迭代，平均 O(log n)。这就是数据结构性质带来的性能提升。

路径相关问题

路径问题是二叉树的另一个大类。核心模式是：从根到叶子（或任意节点）的路径上收集/统计信息。

路径总和系列（112/113）

LeetCode 112——路径总和（Path Sum）：判断是否存在一条从根到叶子的路径，其节点值之和等于 targetSum。

def has_path_sum(root, target_sum):
    if not root:
        return False
    if not root.left and not root.right:  # 叶子节点
        return root.val == target_sum
    return has_path_sum(root.left, target_sum - root.val) or \
           has_path_sum(root.right, target_sum - root.val)

注意这里用了一个小技巧：不是累加路径和，而是用 targetSum 逐层减去当前节点值。到达叶子时，检查差值是否等于该叶子节点的值。这种"减法思路"比"加法思路"更简洁，不需要额外维护一个累加变量。

LeetCode 113——路径总和 II 则是 112 的升级版：不仅要判断是否存在，还要找出所有符合条件的路径。这引入了回溯的思想——在递归中维护一个路径列表，符合条件时加入结果集。

def path_sum(root, target_sum):
    res = []
    def dfs(node, remaining, path):
        if not node:
            return
        path.append(node.val)
        if not node.left and not node.right and remaining == node.val:
            res.append(path.copy())    # 注意深拷贝！
        dfs(node.left, remaining - node.val, path)
        dfs(node.right, remaining - node.val, path)
        path.pop()    # 回溯，撤销选择

    dfs(root, target_sum, [])
    return res
关键细节：path.copy() 这一步特别容易被忽略。Python 中列表是引用传递，如果你直接 res.append(path)，后续 path.pop() 会修改已经存入结果的对象。我见过不止一个同学在这个坑里徘徊了两个小时——包括当年的我自己。

来看一个相关的经典问题。二叉树中的最大路径和（LeetCode 124）是路径系列的天花板题目。它允许路径从任意节点开始到任意节点结束（不一定要经过根）。解法思路：对每个节点，计算"经过该节点的最大路径和"——通过后序递归获取左右子树的最大贡献值，然后组合。这道题在 LeetCode 上标记为 Hard，但掌握了后序递归的思维模式后，你就能理解为什么它实际上"Hard 在表面，Medium 在本质"。

踩坑与总结

二叉树递归的坑点非常集中，我总结三个最常见的：

坑 1：递归栈溢出

Python 默认递归深度大约是 1000 层。如果一棵二叉树是一条链（比如 1 → 2 → 3 → ... → n），递归深度就是 n。当 n > 1000 时，你会收获一个可爱的 RecursionError。解决方案：要么用迭代（栈模拟递归），要么在写递归前先检查树的形状，或者设置 sys.setrecursionlimit(10000)——但治标不治本。

坑 2：Null 判断的顺序

递归函数中，if not root: return ... 一定要放在函数最前面。有人喜欢在 root.left 和 root.right 之前分别判断——代码立马膨胀一倍。除非特别需要区分"左空右不空"和"左不空右空"的情况，否则统一在顶部处理 null 是最优雅的写法。

坑 3：BST 性质误用

并不是"中序遍历有序"就万事大吉了。比如 LeetCode 99——恢复二叉搜索树，虽然可以用中序遍历找到两个被交换的节点，但处理相邻交换和不相邻交换的逻辑完全不同。在写 BST 相关代码时，先搞清楚题目利用了 BST 的哪个性质，再动笔。

问题类型 核心思路 遍历方式 复杂度
最大深度 / 直径 后序收集子树信息 后序 O(n)
验证 BST 取值范围传递 前序 O(n)
BST 第 K 小 中序遍历有序 中序 O(n)
LCA（普通树） 后序查找 p/q 后序 O(n)
LCA（BST） 利用值导航 迭代 O(log n)
路径总和 减法 + 回溯 前序 + 回溯 O(n)
本章心法：二叉树递归的核心就两句话——"相信递归函数能正确完成它的任务"和"想清楚你是在前序、中序还是后序位置做处理"。如果你能做到这两点，二叉树 80% 的题目对你来说都只是"翻译题"——把思路翻译成代码而已。
