第20章 鲁棒调度

大家好，我是你们的讲师。上一章我们讨论了随机调度——用概率分布来刻画不确定性，在期望意义下寻找最优解。但现实中有一种更棘手的情况：你根本不知道分布长什么样。新投产的产线，历史数据不够建模；老产线换了新型号，加工时间变了；或者供应商的供应链变化了，到货时间不再是过去的规律了。当分布未知时，随机规划就像在没有地图的森林里走路——你算出来的"期望最优解"可能是错误的。

这就是鲁棒调度（Robust Scheduling）登场的时刻。鲁棒调度的哲学是：我不需要精确知道不确定性服从什么分布，但我建立一个不确定集（Uncertainty Set）来囊括所有可能发生的参数值，然后寻找一个"在最坏情况下仍然可行且尽量好"的解。通俗地说，就是"做最坏的打算，争取最好的结果"。

鲁棒优化（Robust Optimization, RO）的现代形式由 Ben-Tal、El Ghaoui 和 Nemirovski 等人在 2000 年代系统建立，在供应链、能源、金融领域获得了广泛应用。而分布鲁棒优化（Distributionally Robust Optimization, DRO）则是 2025-2026 年运筹学界最活跃的研究方向之一。这一章我们会从基础概念一直延伸到前沿方法。

鲁棒优化的核心思想 — 保护最坏情况

想象你是一个调度员，要为一个加工周期排班。你只知道每道工序的加工时间在 [10, 20] 分钟之间波动，但不知道分布。如果你采用随机规划，你需要假设一个分布（比如均匀分布），然后优化期望 makespan。但万一真实的分布是双峰的——大部分时间是 10 分钟，偶尔跳变到 20 分钟——你的"期望最优解"可能在 20 分钟的场景下彻底不可行。

鲁棒优化的做法是：定义一个不确定集 U，包含所有可能出现的参数值组合，然后求解下面这个"最小化最坏情况"的问题：

鲁棒对等问题（Robust Counterpart）的形式：

  min   max   f(x, ξ)
   x    ξ∈U

  s.t.  g(x, ξ) ≤ 0,  ∀ ξ ∈ U

约束 g(x, ξ) ≤ 0 必须在整个不确定集 U 上同时成立——这意味着调度方案必须对所有可能的不确定性实现都可行。目标函数 f(x, ξ) 要最小化最坏情况下的目标值。

常见的三种不确定集包括：

  • 盒式不确定集（Box Uncertainty Set）：ξ ∈ [μ - σ, μ + σ]，即每个独立参数在给定区间内任意取值。简单，但过于保守——它假设所有参数同时达到最坏值，这在现实中几乎不会发生。
  • 椭球不确定集（Ellipsoidal Uncertainty Set）：(ξ - μ)ᵀ Σ⁻¹ (ξ - μ) ≤ Γ。椭球集通过协方差结构描述了参数之间的相关性，且通常可以转化为二阶锥约束，求解方便。
  • 预算不确定集（Budgeted Uncertainty Set / Bertsimas-Sim）：{ ξ | |ξᵢ - μᵢ| ≤ σᵢ, Σ |ξᵢ - μᵢ| / σᵢ ≤ Γ }。Bertsimas 和 Sim 在 2004 年提出的关键创新——通过预算参数 Γ 控制最多有多少个参数会同时偏离其名义值。Γ = 0 对应确定性优化，Γ = n 对应极端保守的盒式集。调整 Γ 可以在鲁棒性和最优性之间灵活折中。

个人经验：Bertsimas-Sim 的预算不确定集是我个人在实际项目中最喜欢的选择。它有一个非常实用的特性：你可以通过调节 Γ 来直观地控制保守程度，而且它保持了问题的线性结构（不需要引入二阶锥约束）。对于一个有 100 道工序的调度问题，如果你设 Γ = 10，那就意味着你要求调度方案能同时容忍最多 10 道工序的加工时间同时偏离名义值——这个"最多10个"的业务直觉，业务方非常容易理解。

分布鲁棒优化(DRO) — 2025-2026最活跃的研究方向

经典的鲁棒优化要求约束在整个不确定集上成立——这个要求过于强硬，导致解常常过于保守。分布鲁棒优化（Distributionally Robust Optimization, DRO）提供了一种精巧的折中方案：假设真实的分布属于一个模糊集（Ambiguity Set）——由已知的部分信息（如均值、方差、支撑集）定义的分布集合——然后最小化最坏情况下的期望值。

DRO 位于随机规划和鲁棒优化之间：它比随机规划需要更少的分布假设，又比鲁棒优化更少保守。从 2020 年开始，DRO 在运筹学顶刊上的论文数量呈爆发式增长，到 2025-2026 年已经成为不确定性优化领域最活跃的子方向。

Wasserstein模糊集的构建

当前最流行的模糊集构造方式是Wasserstein 模糊集（Wasserstein Ambiguity Set）。它基于最优传输理论（Optimal Transport Theory）中的 Wasserstein 距离——度量两个概率分布之间的距离。模糊集定义为"以经验分布为中心、半径为 ε 的 Wasserstein 球"：

Wasserstein 模糊集的数学定义：

  B_ε(P̂_N) = { Q : W_p(Q, P̂_N) ≤ ε }

其中：
  - P̂_N  = (1/N) Σ δ_{ξᵢ}  — 基于 N 个样本点的经验分布
  - W_p(Q, P̂_N) — Q 与 P̂_N 之间的 p-Wasserstein 距离
  - ε — 模糊集半径（超参数），控制保护程度

Wasserstein DRO 的两个突出优势：

  • 从数据中学习： Wasserstein 球可以理解为"以数据为中心、向外扩展一个置信区域"。半径 ε 的理论选择取决于样本数量 N 和置信水平——当 N → ∞ 时，ε → 0，DRO 退化为随机规划。
  • 可处理性：对于很多常见的不确定性结构（如线性目标、凸约束），Wasserstein DRO 可以等价地转化为一个有限维的凸优化问题——在实际计算中，它往往只比对应的随机规划多一个额外的变量和一个不等式约束，求解代价几乎可以忽略。

2023-2025 年的研究进一步扩展了 Wasserstein DRO 在调度中的应用：Kuhn 等人证明了 Wasserstein DRO 与正则化经验风险最小化之间的等价关系；Esteban-Perez 和 Morales 将 Wasserstein DRO 应用于两阶段随机调度，在风电并网调度中比传统方法减少了 15% 的运行成本。

从历史数据中学习不确定性集合

DRO 的一个核心信念是：不确定性集合不应该由专家凭经验拍脑袋确定，而应该从历史数据中学习。2024-2026 年出现了一系列"数据驱动的不确定集学习"方法：

  • 假设检验方法：利用 Kolmogorov-Smirnov 检验或 χ² 检验来构造包含真实分布的概率为 (1-α) 的置信区域。
  • CVaR 方法：通过条件风险价值（Conditional Value at Risk）来定义模糊集，直接控制尾部风险。
  • 生成式方法：使用 GAN 或扩散模型从历史数据中学习加工时间的联合分布，然后基于生成模型的潜在空间构造模糊集。

这些方法共同指向一个趋势：未来的鲁棒调度系统不再需要运筹专家手动配置不确定集参数，而是能自动从产线历史数据中学习一个"裁剪合身"的不确定集。

自适应鲁棒优化(ARO) — 多阶段决策

经典鲁棒优化做一个"一次性决策"——在不确定性实现之前，把所有决策都定下来。但这在调度中往往不现实：你在周一排好了这周的生产计划，但当周三机器突然故障时，你会重新调整计划，而不是死守周一排的方案。也就是说，决策是分阶段做出的，后续决策可以依赖于前面阶段观察到的信息。

"here-and-now" vs "wait-and-see"

自适应鲁棒优化（Adaptive Robust Optimization, ARO）将这个信息结构显式地纳入了模型：

  • Here-and-now 决策：在不确定性实现之前就必须做出的决策。比如，你需要在知道明天的实际订单之前，确定明天的班次人数和物料采购量。
  • Wait-and-see 决策：可以在不确定性实现之后再做出的决策。比如，当实际订单到达后，再决定具体的加工排序。

在调度问题中，典型的分解是：工件分配是 here-and-now 决策，排产顺序是 wait-and-see 决策。这种两阶段（或多阶段）结构比单阶段 RO 更灵活，因为它允许调度方案根据实际观察到的不确定性进行调整。

决策规则近似

ARO 的求解比单阶段 RO 困难得多——实际上，多阶段自适应鲁棒优化一般是 NP-hard 的。实际工程中主要使用决策规则近似（Decision Rule Approximation）：

  • 线性决策规则（LDR）：假设 wait-and-see 决策是不确定性参数的线性函数。例如，x₂ = α + β·ξ。这看似简单，但 Ben-Tal 等人证明，对于线性约束和盒式不确定集，LDR 几乎能恢复全自适应问题的最优解。
  • 分段线性决策规则（PLDR）：比 LDR 更灵活，通过将不确定集划分成若干个区域，在每个区域上用不同的线性函数。
  • 非线性决策规则（如神经网络决策规则）：2025 年的最新工作探索了用神经网络参数化 wait-and-see 策略，然后用对抗训练来优化——这实际上已经和"鲁棒强化学习"交汇了。

反应式调度(重调度)

前面讨论的方法都在"排程阶段"做文章——在不确定性发生之前就设计一个鲁棒的方案。但无论你的方案多么鲁棒，总有一些扰动是无法提前预防的——比如一场突发的火灾疏散了整个车间。这时你需要的是反应式调度（Reactive Scheduling），也叫重调度（Rescheduling）。

何时触发：事件触发/周期触发/混合触发

重调度的第一个问题是：什么时候需要重调度？三种常见策略：

  • 事件触发（Event-Driven）：当某个特定事件发生时触发重调度——比如机器故障、紧急订单插入、关键质量异常。优点是反应快速，缺点是在高扰动环境中可能频繁触发，导致"排班抖动"——工人刚刚拿到新方案，还没来得及执行，又被新的重调度方案覆盖了。
  • 周期触发（Periodic）：固定时间间隔（如每 2 小时、每个班次开始时）进行重调度。优点是节奏稳定、执行效率高，缺点是可能错过两次重调度之间的关键事件。
  • 混合触发（Hybrid）：以周期重调度为基础，同时在发生严重扰动（如主要设备停机、质量事故）时触发额外的事件驱动重调度。这是工业界的主流做法。

如何修正：局部修复 vs 全局重优化

当决定重调度之后，第二个问题是：如何在尽量少打乱原方案的前提下修正调度？

  • 局部修复（Local Repair）：只调整受扰动影响的区域，保持原方案的其余部分不变。例如，当一台机器故障时，只把该机器上待加工的工序重新分配到其他同型机器上，不改变其他工序的排程。局部修复速度快（通常秒级完成），但修复后的方案可能次优——局部调整累积多了，整体调度会"跑偏"。
  • 全局重优化（Global Reoptimization）：以当前系统状态为起点，用优化算法从零开始计算从当前时刻到未来的完整调度方案。全局重优化质量更高，但计算时间长，且可能大幅改变原方案——让工人不知所措。

一个流行的折中方案是右移重调度（Right-Shift Rescheduling）：当发生延迟时，简单地将所有未加工工序向后平移，保持工序顺序不变。这既保留了原方案的结构（工人不用重新学），又用最低的计算代价修复了可行性。缺点是它没有利用多余的机器容量来追回延迟，可能导致累积延迟越来越大。

RL驱动的智能触发策略

传统的事件/周期混合触发有一个漏洞：触发阈值是手工设定的。阈值设得太低，频繁重调度；太高，错过关键事件。2025-2026 年的前沿工作尝试用强化学习（RL）来自动学习重调度触发策略——RL 智能体监控系统状态（机器利用率、队列长度、已延迟工件数等），在"触发重调度"和"维持原方案"之间做决策。

在一篇 2025 年的论文中（发表于 INFORMS Journal on Computing），作者在半导体制造 benchmark 上对比了 RL 驱动的混合触发和手工阈值触发，发现 RL 策略在保持相同水平 schedule stability（方案稳定性）的前提下，makespan 降低了 7.3%，而且不需要人工调参。

鲁棒不是越保守越好——一个真实的代价：2018 年，一家汽车零配件工厂引入了一个鲁棒排程系统。原来的确定性排程方案 makespan 是 100 小时，但因为加工时间波动，平均有 15% 的订单延期。运筹团队设计了一个"高鲁棒性"方案——将所有加工时间的不确定区间设为名义值的 ±30%，并用盒式不确定集（最保守的形式）来确保 100% 的约束满足。结果是：调度方案为应对最极端的情况，在每台机器前都塞进了巨大的安全时间，导致整体 makespan 飙升到了 140 小时，设备利用率从 90% 暴跌到 62%。更糟糕的是，实际加工中 "±30% 极端情况" 只发生了不到 0.5% 的时间——为了 0.5% 的概率，牺牲了 30% 的产能。真正的教训是：鲁棒不是越保守越好，过度鲁棒就是把黄金当废铁用。如果你选择 RO，一定要用交叉验证来选择不确定集的保守程度，而不是拍脑袋。

好，这一章我们从经典鲁棒优化出发，讨论了不确定集的三种构造方式、分布鲁棒优化（DRO）、自适应鲁棒优化（ARO），以及反应式调度/重调度的触发机制和修正策略。鲁棒调度给了我们在"分布未知"的场景下保护最坏情况的能力。但不管确定性、随机性还是鲁棒性，我们讨论的都是"单一目标"——最小化 makespan 或最小化成本。现实中，调度往往需要同时考虑多个相互冲突的目标。下一章多目标调度优化，我们将讨论如何用 Pareto 最优和非支配排序来应对这种"鱼和熊掌不可兼得"的困境。我们下一章见。
