第25章 模型评估与调优

大家好，我是你们的讲师。

模型训练完看到 mAP 0.85 就高兴太早了——真正有价值的分析来自 PR 曲线和混淆矩阵。你可能发现模型对某一类目标的 AP 只有 0.3，而这才是你需要关注的重点。我曾经做过一个交通标志检测项目，整体 mAP 达到了 0.91，看起来已经很不错了。但仔细看每个类别的 AP 才发现，"停止标志"的 AP 高达 0.98，而"限速标志"的 AP 只有 0.45。为什么？因为训练集中的限速标志样本太少，而且大都是远距离小目标。发现这个问题后，我针对性地补充了限速标志的数据和增广策略，最终将其 AP 提升到了 0.85，整体 mAP 也跃升到了 0.94。

这一章我们会深入讲解检测模型的评估指标、如何使用混淆矩阵和 PR 曲线来分析模型问题、以及系统性的调优方法论。目标就是：用数据说话，而不是凭感觉调参。

评估指标详解

如果你觉得评估指标就是"mAP越高越好"，那你还只掌握了皮毛。真正理解每个指标的含义，你才能准确判断模型的问题所在。

基础指标：Precision 和 Recall

先复习一下最基础的两个指标。所有检测指标都建立在以下四个统计量的基础上：

  • TP（True Positive）：预测框与真实框的 IOU > 阈值，且类别正确
  • FP（False Positive）：预测框与任何真实框的 IOU 都低于阈值，或类别错误（误报）
  • FN（False Negative）：真实框没有被任何预测框匹配到（漏检）
  • TN（True Negative）：在检测中基本不使用——因为背景"正确"的数量难以有意义地统计
Precision = TP / (TP + FP)  # 模型预测的所有框中，有多少是正确的
Recall    = TP / (TP + FN)  # 所有真实目标中，模型找出了多少

高 Precision 意味着模型的预测"可靠"——预测出来的大部分都是对的。高 Recall 意味着模型的预测"全面"——大部分真实目标都被找到了。两者通常是矛盾的：提高置信度阈值会提升 Precision 但降低 Recall，反之亦然。

AP 和 mAP

AP（Average Precision）是 PR 曲线下的面积。对于每个类别，按置信度从高到低排序预测框，计算每个阈值下的 Precision 和 Recall，然后绘制 PR 曲线并计算面积。

mAP（Mean AP）是所有类别 AP 的平均值。

在 COCO 评估体系中，有两个关键指标：

指标 定义 意义
mAP@0.5 IOU 阈值取 0.5 时的 mAP 传统指标，宽松评估，更关注"是否检测到"
mAP@0.5:0.95 IOU 阈值从 0.5 到 0.95，步长 0.05，共 10 个阈值的 mAP 平均值 严格指标，对定位精度敏感，是 COCO 的主要评估指标
AP_s 小目标（面积 < 32²）的 AP 小目标检测能力
AP_m 中目标（32² < 面积 < 96²）的 AP 中目标检测能力
AP_l 大目标（面积 > 96²）的 AP 大目标检测能力

在 Ultralytics 中验证模型后，你会看到类似这样的输出：

这些数字能告诉你很多信息。比如上面这个表中，"bike"类别的 mAP@0.5:0.95 只有 0.623，显著低于平均值 0.712——说明模型对自行车的检测精度不够，需要重点优化。

关键洞察：不要只看 mAP 平均值。每个类别的 AP 差异能帮你精准定位模型的薄弱环节。一般而言，当某个类别的 AP 明显低于平均值时，原因不外乎：①该类别的训练样本不足；②该类别的目标尺度/形态特殊，模型难以泛化；③该类别的标注质量较差。

混淆矩阵

混淆矩阵（Confusion Matrix）是最直观的错误分析工具。在 YOLO 中，混淆矩阵是一个 (N+1)×(N+1) 的矩阵，其中 N 是类别数，额外的一行和一列代表背景（background）。

观察混淆矩阵，你主要关注以下几点：

  • 对角线上的数值：越大越好，表示正确分类的比例
  • 非对角线上的数值：表示类别混淆——模型把 A 类误认为 B 类
  • 最后一列（background FP）：表示背景被误检为目标（误报）
  • 最后一行（background FN）：表示目标被漏检（漏报）
# 生成混淆矩阵
from ultralytics import YOLO

model = YOLO("runs/detect/exp/weights/best.pt")
results = model.val(data="coco128.yaml", plots=True)

通过分析混淆矩阵，你可以得到很多有价值的结论：

混淆矩阵中发现的问题 可能的解决方案
A 类和 B 类经常互相混淆 检查这两类的标注是否准确区分；考虑添加更多区分性特征（如引入更多该类别的训练数据）
背景 FP 很高（很多背景被误检） 增加包含复杂背景的负样本；提高置信度阈值；适当降低 NMS 的 IOU 阈值
背景 FN 很高（很多目标被漏检） 降低置信度阈值；检查小目标检测能力；检查数据增强是否过度模糊了目标

PR 曲线与 F1-Confidence 曲线

PR 曲线（Precision-Recall Curve）展示了 Precision 和 Recall 的权衡关系。曲线的形状越接近右上角（1.0, 1.0），模型性能越好。

F1-Confidence 曲线则展示了不同置信度阈值下的 F1 分数（Precision 和 Recall 的调和平均）。这个曲线能帮你选择最优的置信度阈值：

# F1 = 2 * (Precision * Recall) / (Precision + Recall)

# 最佳置信度阈值通常是 F1 曲线最高点对应的阈值
# Ultralytics 会在验证时自动输出：
# Optimal confidence threshold: 0.312
# Optimal F1 score: 0.854

在实际部署中，你可以根据业务需求选择不同的阈值：

  • 如果业务要求高精度、低误报（如质检），选择 F1 曲线右侧（高置信度）
  • 如果业务要求高召回、不漏检（如安防），选择 F1 曲线左侧（低置信度）
一个实用技巧：默认的置信度阈值是 0.25，但这并不一定是最优的。部署前，务必根据验证集的 F1-Confidence 曲线选择合适的置信度阈值。这个决策能直接提升你的上线效果，而且零成本。

错误分析：定位误差 vs 分类误差

当模型性能不达预期时，你需要诊断错误类型。经典的检测错误分析将错误分为以下几类：

  • 定位误差（Localization Error）：预测框和真实框的 IOU 在 0.1~0.5 之间——检测到了目标但位置不准
  • 分类误差（Classification Error）：框的位置对了但类别分错了
  • 背景误检（Background FP）：在背景区域产生了检测框
  • 重复检测（Duplicate Detection）：同一目标被多个框覆盖
def error_analysis(gt_boxes, pred_boxes, iou_thresh=0.5):
    """对预测结果进行错误类型分类"""
    errors = {"correct": 0, "localization": 0,
              "classification": 0, "background": 0, "duplicate": 0}

    for pred in pred_boxes:
        matched_gts = [g for g in gt_boxes if iou(pred, g) > iou_thresh]

        if not matched_gts:
            # 没有匹配到任何真实框
            # 检查是否有 IOU > 0.1 但 < 0.5 的真实框
            weak_matches = [g for g in gt_boxes
                           if 0.1 < iou(pred, g) < 0.5]
            if weak_matches:
                errors["localization"] += 1
            else:
                errors["background"] += 1
        elif any(pred.class_id == g.class_id for g in matched_gts):
            errors["correct"] += 1
        else:
            errors["classification"] += 1

    return errors

通过这个分析，你可以定位模型的瓶颈：

  • 定位误差多 → 改进回归损失（如 CIoU）、优化锚框设计
  • 分类误差多 → 增加训练数据、改善特征提取、调整类别权重
  • 背景误检多 → 增加负样本、提高置信度阈值、改进 NMS

超参数调优

手工调参就像"玄学"，正确的做法是使用系统化的调优策略。

网格搜索（Grid Search）

对几个关键超参数列举候选值，进行全组合搜索。适合小规模调优：

from ultralytics import YOLO
import itertools

# 定义搜索空间
param_grid = {
    "lr0": [0.001, 0.005, 0.01],
    "batch": [16, 32],
    "weight_decay": [0.0001, 0.0005],
}

keys, values = zip(*param_grid.items())
for combo in itertools.product(*values):
    params = dict(zip(keys, combo))
    model = YOLO("yolov8n.pt")
    results = model.train(
        data="dataset.yaml", epochs=50,
        **params, project="grid_search",
        name=f"lr{params['lr0']}_b{params['batch']}_wd{params['weight_decay']}"
    )

贝叶斯优化（Bayesian Optimization）

使用 Optuna 或 Hyperopt 进行更高效的搜索。贝叶斯优化会根据历史结果智能选择下一组参数，比网格搜索效率高得多：

import optuna
from ultralytics import YOLO

def objective(trial):
    # 定义搜索空间
    lr0 = trial.suggest_float("lr0", 1e-4, 1e-2, log=True)
    momentum = trial.suggest_float("momentum", 0.8, 0.98)
    weight_decay = trial.suggest_float("weight_decay", 1e-5, 1e-3, log=True)

    # 训练模型
    model = YOLO("yolov8n.pt")
    results = model.train(
        data="dataset.yaml", epochs=30,
        lr0=lr0, momentum=momentum, weight_decay=weight_decay,
        project="bayes_opt", name=f"trial_{trial.number}",
        verbose=False  # 减少日志输出
    )

    # 返回优化目标（mAP50-95）
    return results.results_dict["metrics/mAP50-95(B)"]

# 运行优化
study = optuna.create_study(direction="maximize")
study.optimize(objective, n_trials=20)

print("最佳参数:", study.best_params)
print("最佳 mAP:", study.best_value)
调参中的坑：调参时最常犯的错误是一次性改变太多参数。这样就无法知道哪个参数的改变带来了提升。正确的做法是"单一变量原则"——每次只改变一个参数，观察其对验证集指标的影响。另一个常见错误是在测试集上调参——你会无意中过拟合测试集。务必在验证集上做决策，只保留测试集做最终评估。

Bad Case 分析方法

系统性地分析坏例（bad case）是提升模型最有效的方法。我推荐的流程是：

  1. 收集坏例：运行验证集推理，筛选出所有 FP 和 FN 的样本
  2. 分类归档：将坏例按错误类型分类（定位、分类、背景、漏检）
  3. 根因分析：对每个类别找 3-5 个典型样本，分析"为什么错了"
  4. 制定修复方案：根据根因制定数据/模型改进方案
  5. 验证修复效果：跑新的训练，确认坏例是否改善，且没有引入新的坏例

模型改进检查清单：
  • 数据层面：各类别样本是否均衡？是否有标注错误？是否包含足够多的困难样本？
  • 模型层面：模型大小是否适合任务？骨干网络是否足够强大？特征金字塔是否适配了目标尺度？
  • 训练层面：学习率是否合适？是否过拟合/欠拟合？数据增强策略是否合理？
  • 后处理层面：置信度阈值是否最优？NMS 的 IOU 阈值是否需要调整？

实战：完整调优案例

假设你有一个行人检测任务，初始模型的 mAP@0.5:0.95 只有 0.45。按照我们的方法论，系统性地分析并优化：

步骤 分析 操作 mAP提升
1 混淆矩阵显示很多背景误检 增加 500 张不包含行人的背景负样本 +0.03
2 PR 曲线发现 Recall 偏低 降低置信度阈值从 0.25 到 0.15 +0.02
3 错误分析发现小目标定位误差大 调整输入分辨率到 1280；启用 4 倍下采样检测头 +0.06
4 分析坏例发现遮挡严重场景性能差 添加 Copy-Paste 增强和遮挡增强 +0.04
5 Bayes 调参优化超参数 Optuna 搜索 30 轮找到最优 lr / wd / momentum 组合 +0.03

最终模型从 mAP 0.45 提升到了 0.63。注意每一步提升量都不大，但通过系统性的分析，每步都在解决一个具体的问题，累积起来就是一个显著的飞跃。
