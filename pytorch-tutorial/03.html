第3章 张量运算与广播

大家好，我是你们的讲师。第2章我们学会了如何创建各种 Tensor——从全零到随机数，从 CPU 到 GPU。但 Tensor 真正的价值在于运算。你创建了一堆数据，最终目的还是对它们进行计算——加减乘除、矩阵乘法、求均值、找最大值……这一章我们就来系统地学习 Tensor 的各种运算。

这一章有一个非常重要的概念——广播（Broadcasting）。简单说，就是 PyTorch 允许不同形状的 Tensor 进行运算，自动把小的"扩展"成大的。这个机制让代码变得极其简洁，但也隐藏着很多陷阱。不理解广播，你写的 bug 可能就是隐形的——代码能跑，结果全错。

元素级运算（Element-wise Operations）

元素级运算是最基本的运算形式——两个形状相同的 Tensor，对应位置上的元素逐个计算。这可能是深度学习中发生最频繁的操作了。

import torch

a = torch.tensor([1, 2, 3])
b = torch.tensor([4, 5, 6])

# 基本算术运算
print(a + b)   # tensor([5, 7, 9])
print(a - b)   # tensor([-3, -3, -3])
print(a * b)   # tensor([4, 10, 18])  — 注意：这不是矩阵乘法
print(a / b)   # tensor([0.2500, 0.4000, 0.5000])
print(a ** 2)  # tensor([1, 4, 9])

# 标量运算（标量会自动广播）
print(a + 10)  # tensor([11, 12, 13])
print(a * 2)   # tensor([2, 4, 6])

# 比较运算
print(a > 1)   # tensor([False, True, True])
print(a == 2)  # tensor([False, True, False])

PyTorch 也提供了函数形式的操作，效果和运算符完全一样——选择哪种取决于你的编码风格：

# 运算符 vs 函数，两者等价
print(a + b)
print(torch.add(a, b))

print(a * b)
print(torch.mul(a, b))

# 还有一些实用的数学函数
print(torch.abs(torch.tensor([-1, -2, 3])))  # 绝对值
print(torch.sqrt(a))                          # 平方根
print(torch.exp(a))                           # 指数
print(torch.log(a.float()))                   # 自然对数
print(torch.clamp(a, min=2, max=3))           # 裁剪：[2,4,4] → [2,3,3]

# 三角函数
print(torch.sin(a.float()))                   # 正弦
print(torch.cos(a.float()))                   # 余弦

矩阵乘法：matmul vs mm vs @

矩阵乘法是神经网络的前向传播和反向传播中最核心的运算。y = Wx + b，这个简单的公式里就包含了矩阵乘法。

PyTorch 提供了三个做矩阵乘法的工具，它们之间有些细微的区别：

方法 别名 支持的维度 广播支持 推荐场景
torch.mm(a, b) - 仅 2D (矩阵) 否 明确的两矩阵相乘
torch.matmul(a, b) torch.mm 的高维版本 任意维度 是（高维广播） 批量矩阵乘法
a @ b matmul 的运算符重载 任意维度 是（同 matmul） 日常使用，最简洁 
# 2D 矩阵乘法
A = torch.randn(3, 4)
B = torch.randn(4, 5)

C1 = torch.mm(A, B)        # 经典写法
C2 = torch.matmul(A, B)    # 通用写法
C3 = A @ B                 # 最简洁写法（推荐日常使用）
print(C1.shape)            # torch.Size([3, 5])

# 批量矩阵乘法（batched matmul）
# 假设你有 8 个 3x4 矩阵和 8 个 4x5 矩阵，要做 8 次矩阵乘法
batch_A = torch.randn(8, 3, 4)
batch_B = torch.randn(8, 4, 5)
batch_C = batch_A @ batch_B
print(batch_C.shape)       # torch.Size([8, 3, 5])
日常建议：直接用 @ 运算符。它最简洁，而且支持所有维度——和 PyTorch 2.x 中 torch.compile 的兼容性也最好。只有在需要显式表明"这里是矩阵乘法"以增加代码可读性时，才用 torch.matmul()。

广播机制（Broadcasting）

这是本章最重要的概念。我见过太多初学者因为不理解广播机制而写出了看似正确、实则错误的代码。

广播机制的精髓是：当两个 Tensor 做运算时，PyTorch 会自动将较小的 Tensor 扩展到与较大的 Tensor 形状一致，然后再做元素级运算。你不需要手动复制数据。

广播规则很简单：从最后一个维度开始比较，每个维度要么相等，要么其中一个为 1，要么其中一个缺失（即维度数不够）。

先看一个具体的例子：

# 形状 [3, 1] 的矩阵
a = torch.tensor([[1], [2], [3]])
print(a.shape)  # torch.Size([3, 1])

# 形状 [1, 4] 的矩阵
b = torch.tensor([[10, 20, 30, 40]])
print(b.shape)  # torch.Size([1, 4])

# 两者相加时，广播规则自动扩展
c = a + b
print(c.shape)  # torch.Size([3, 4])
print(c)
# tensor([[11, 21, 31, 41],
#         [12, 22, 32, 42],
#         [13, 23, 33, 43]])

这个例子非常经典，我们来拆解一下广播的过程：

  • a 的形状是 [3, 1]，b 的形状是 [1, 4]
  • 从最后一个维度开始比较：第 1 维 a=1 vs b=4 → 不相等，但 a 的为 1，所以 a 沿这个维度扩展为 4
  • 第 0 维 a=3 vs b=1 → 不相等，但 b 的为 1，所以 b 沿这个维度扩展为 3
  • 最终两者都是 [3, 4]，元素级相加
广播速记口诀：
  1. 从后向前比维度
  2. 相等或有一为 1 —— 可以广播
  3. 都不满足 —— 报错
  4. 维度不够 —— 自动补 1

再看两个常见的误区：

# 形状不匹配导致广播失败
try:
    a = torch.randn(3, 2)
    b = torch.randn(3, 4)
    c = a + b  # 第1维: 2 vs 4，都不为1，无法广播
except RuntimeError as e:
    print(e)  # "The size of tensor a (2) must match the size of tensor b (4)..."

# 看似正确实则错误的广播
a = torch.randn(3, 4)
b = torch.randn(4,)
# 你想做的事情: a 的每一行减去 b
# 正确做法: b 的形状是 [4]，广播后变成 [1, 4] → [3, 4]
c = a - b  # 正确！形状 [3, 4] - [4] → [3, 4] - [1, 4] → [3, 4]
print(c.shape)  # torch.Size([3, 4])
广播陷阱：当你做 a - b 且 a 的形状是 [3, 4]，b 的形状是 [4] 时，结果是你预期的"每行减同一个向量"。但如果你不小心把 b 的形状变成了 [3, 1] 或 [3]，结果会完全不同——PyTorch 不会报错，只是默默地沿着你没想到的维度广播。这种 bug 是最难排查的，因为代码不报错，loss 也能降，但结果不对。

规约运算（Reduction Operations）

规约运算是指将一个维度上的多个元素"合并"成一个——求 sum、mean、max、min 等。理解"沿哪个维度操作"是关键。

x = torch.tensor([[1, 2, 3],
                   [4, 5, 6]])

# 全局规约
print(x.sum())            # tensor(21) — 所有元素求和
print(x.mean().float())   # tensor(3.5000) — 所有元素求均值
print(x.max())            # tensor(6)
print(x.min())            # tensor(1)
print(x.prod())           # tensor(720) — 所有元素求积

# 沿指定维度规约（dim 参数）
print(x.sum(dim=0))       # tensor([5, 7, 9])  — 沿第0维求和（压缩行，保留列）
print(x.sum(dim=1))       # tensor([6, 15])    — 沿第1维求和（压缩列，保留行）

# keepdim=True 保持维度
print(x.sum(dim=1, keepdim=True))
# tensor([[6],
#         [15]]) — 形状仍是 [2, 1]，保持维度

# argmax / argmin（返回索引）
print(x.argmax(dim=1))    # tensor([2, 2]) — 每行最大值的列索引
关于 keepdim 的小技巧：很多初学者不理解为什么要有 keepdim=True。它的作用是保持张量的维度结构，避免因为规约操作导致维度减少。比如你用 softmax 之后做 sum(dim=-1)，如果不加 keepdim，结果就从 [B, N] 变成了 [B]，后面再做广播时就会出现意外。建议在大多数情况下使用 keepdim=True。

Contiguous 与非 Contiguous 内存

这是一个稍微进阶但非常实用的话题。理解了它，你才能搞明白为什么有时候 .view() 会报错而 .reshape() 不会。

在 PyTorch 中，Tensor 的数据在底层是按一维数组存储的。访问数据有两种方式：

  • contiguous（连续内存）：数据在内存中是按行优先顺序连续存储的。这是默认方式，访问速度最快。
  • non-contiguous（非连续内存）：数据在内存中不是连续存放的——比如你对一个 Tensor 做了转置（transpose）或某些切片操作后，新 Tensor 的"逻辑形状"改变了，但底层数据没有重新排列。
x = torch.randn(2, 3)
print(x.is_contiguous())  # True — 默认是连续的

# 转置后变成非连续
x_t = x.transpose(0, 1)
print(x_t.shape)           # torch.Size([3, 2])
print(x_t.is_contiguous()) # False — 数据在内存中变成了跳着读的

# 切片也可能产生非连续的 Tensor
x_slice = x[:, 0:2]
print(x_slice.is_contiguous())  # 可能为 False

view() vs reshape() 的陷阱

下面讲一个几乎每个 PyTorch 初学者都会踩的坑。

先说一个真实的故事。2019 年，我的一个朋友在改写一个图像分类模型时，需要对特征图做维度变换。他先对张量做了 .transpose() 交换了两个维度，然后想用 .view() 把它展平。结果一运行就报错：

RuntimeError: view size is not compatible with input tensor's size and stride (at least one dimension spans across two contiguous subspaces)

他盯着这个错误看了半小时，去 Google 搜了一圈，在 Stack Overflow 上找到了答案：view() 要求 Tensor 在内存中必须是 contiguous 的，而 transpose() 之后的 Tensor 是非 contiguous 的，所以 view() 会报错。

修复方法有两种：

x = torch.randn(2, 3, 4)

# 先做转置
x_t = x.transpose(0, 1)  # 形状变成 [3, 2, 4]

# 方法1：先 .contiguous() 再 .view()
x_contig = x_t.contiguous()  # 强制重新排列内存，变成连续
result1 = x_contig.view(-1)  # 成功

# 方法2：直接用 .reshape()（推荐）
result2 = x_t.reshape(-1)    # reshape 会自动处理 contiguous 问题

那 view() 和 reshape() 到底有什么区别？

特性 view() reshape()
是否要求 contiguous 是（否则报错） 否（自动处理）
是否会复制数据 不会（始终共享内存） 可能（非 contiguous 时会复制）
性能 快（零拷贝） 可能慢（需要复制）
安全性 低（容易踩坑） 高（更宽容）
我个人的最佳实践：
  • 当你确定 Tensor 是 contiguous 的时候（比如刚创建、刚做了 .contiguous()），用 .view()，零拷贝、性能最好。
  • 当你不确定的时候，用 .reshape()，它会自动处理一切。
  • 千万不要在非 contiguous 的 Tensor 上调用 .view() 而不加 .contiguous()——这种 bug 非常难定位。

其他常用运算

还有一些运算在深度学习中非常常用，我们快速过一遍：

x = torch.randn(3, 4)

# 拼接与堆叠
a = torch.randn(3, 4)
b = torch.randn(3, 4)
cat_v = torch.cat([a, b], dim=0)    # 沿第0维拼接 → [6, 4]
cat_h = torch.cat([a, b], dim=1)    # 沿第1维拼接 → [3, 8]
stack = torch.stack([a, b], dim=0)  # 新增一个维度堆叠 → [2, 3, 4]

# 拆分与分割
split = torch.split(x, 2, dim=0)    # 沿第0维每份2个元素拆分
chunk = torch.chunk(x, 2, dim=0)    # 沿第0维均匀分成2份

# 压缩与解压缩
x_4d = torch.randn(1, 3, 1, 4)
squeezed = x_4d.squeeze()           # 移除所有大小为1的维度 → [3, 4]
unsqueezed = x.unsqueeze(0)         # 在位置0添加一个维度 → [1, 3, 4]

# 转置与置换
x_2d = torch.randn(3, 4)
t = x_2d.T                          # 转置 → [4, 3]（仅限2D）

x_3d = torch.randn(2, 3, 4)
permuted = x_3d.permute(2, 0, 1)    # 重新排列维度 → [4, 2, 3]
transposed = x_3d.transpose(0, 1)   # 交换两个维度 → [3, 2, 4]

# 扩张与重复
original = torch.tensor([[1, 2], [3, 4]])
expanded = original.expand(3, -1, -1)  # 沿第0维扩展到3（不复制数据）
repeated = original.repeat(3, 1, 1)    # 沿第0维复制3次（复制数据）
expand vs repeat：.expand() 不分配新内存（只是改变 view 的 stride），速度快但只能扩展维度为 1 的维度。而 .repeat() 会实际复制数据，占用更多显存，但没限制。在写广播操作时，优先考虑用 .expand() 而不是 .repeat() 来节省显存。

本章小结

这一章我们学习了张量运算的核心内容。下面是速查表：

运算类别 核心方法/运算符 关键提醒
元素级运算 + - * / **, torch.sqrt, torch.exp, torch.clamp 形状完全相同时最安全
矩阵乘法 @, torch.matmul, torch.mm 日常用 @，最简洁最通用
广播机制 自动应用 从后往前比维度，相等或有一为 1
规约运算 .sum(), .mean(), .max(), .argmax() 用 keepdim=True 避免维度塌缩
形状变换 .view(), .reshape(), .permute(), .transpose() 非 contiguous 时 view 会报错，用 reshape 更安全
拼接/拆分 torch.cat, torch.stack, torch.split cat 不增维度，stack 增维度

现在你已经能够熟练地创建和运算 Tensor 了——这相当于你已经学会了 PyTorch 的"基本语法"。但问题来了：在深度学习中，我们需要计算梯度来进行参数更新，怎么办呢？总不能再手写求导公式吧？

下一章我们就来揭晓答案——自动求导机制（Autograd），这是 PyTorch 最强大的底牌之一。掌握了它，你就能理解为什么 PyTorch 能自动帮你算出所有参数的梯度。下一章见！
