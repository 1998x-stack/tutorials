第27章 综合案例实战

大家好，我是你们的讲师。经历了 26 章的理论学习、工具实践和前沿探讨，今天终于到了"大练兵"的时刻。这一章我们将完整地走一遍抽样调查的全流程——从一个研究问题出发，经过抽样设计、实施、加权、分析，到最终产出报告。

这样的实战演练是最有价值的，因为在真实工作中没有任何项目会按照教科书完美展开。你会看到每一个阶段的权衡取舍、那些"想不到但一定会遇到"的问题、以及我们是怎么解决的。

案例背景：某城市居民健康调查

假设你现在是某市疾控中心的数据分析师。市政府计划开展一项"全市居民健康素养与慢性病状况调查"，要求你做方案设计并组织实施。市里给你 6 个月时间和 200 万预算。

本市是一个二线城市，常住人口约 800 万，下辖 12 个区县（6 个城区、6 个郊区），每个区县下辖若干街道/乡镇。居民的健康水平存在明显的区域差异——城区居民收入较高、医疗资源丰富；郊区居民相对贫困、健康意识较弱。

步骤1：定义研究问题

首先要把模糊的"健康调查"转化为可操作的研究目标：

  • 目标总体：本市所有 18 岁以上的常住居民（在本市居住满 6 个月以上）
  • 核心指标：健康素养达标率、高血压患病率、糖尿病患病率、BMI 分布、吸烟率、体检率
  • 估计要求：全市整体估计 + 各区县分别估计，核心指标的相对误差控制在 10% 以内
  • 分析目标：描述整体水平、比较不同人群差异、识别风险因素

关键决策：由于既要全市估计又要区县分别估计，样本量不能只满足全市精度，还要满足每个区县的精度需求——这对样本量提出了很高的要求。

步骤2：抽样设计

基于目标需求和预算约束，我们选择分层二阶段 PPS 抽样：

设计要素 选择 理由
分层变量 区县（12 个）+ 街道类型（城区/郊区） 各区县都需要独立估计，城区/郊区健康水平差异大
第一阶段 PSU 街道/乡镇 全市约 150 个街道/乡镇，有完整的名录框
第一阶段抽样方式 PPS（概率与人口规模成比例） 街道人口规模差异大，PPS 可提高效率
第二阶段 SSU 常住居民 从选中的街道中抽取居民
总样本量 6000 人 按 deff=2.0、预期应答率 75% 计算，确保各区县约 500 人
层内分配 比例分配（按人口规模） 兼顾全市估计和区县估计精度

样本量计算的具体过程：

# 样本量计算 (Python)
import math

# 参数设定
target_cv = 0.10       # 目标相对误差 10%
deff = 2.0             # 预期设计效应
response_rate = 0.75   # 预期应答率
N_districts = 12       # 区县数量
n_per_district = 500   # 每个区县至少 500 人

# 按简单随机抽样计算初始样本量（基于率的样本量公式）
# 假设最保守的情况：p = 0.5
z = 1.96               # 95% 置信水平
p = 0.5                # 预期比例
d = 0.10 * p           # 绝对容许误差 = 相对误差 * p

n_srs = (z**2 * p * (1-p)) / (d**2)
print(f"SRS 所需样本量: {n_srs:.0f}")

# 考虑设计效应和应答率
n_adjusted = n_srs * deff / response_rate
print(f"调整后 (deff={deff}, RR={response_rate}): {n_adjusted:.0f}")

# 同时满足各区县的估计需求
total_required = n_per_district * N_districts / response_rate
print(f"满足区县估计需求: {total_required:.0f}")

# 最终取较大值：6000 人左右
final_n = 6000
print(f"最终样本量: {final_n}")
print(f"平均每街道样本量: {final_n / 60:.0f} 人")  # 60 个 PSU

步骤3：实施

抽样设计的实施是整个项目中最容易被低估的环节。以下是我们的执行流程：

绘制抽样框

从市公安局获取全市 150 个街道/乡镇的常住人口数据（作为 PSU 抽样框）。从各街道的居委会/村委会获取辖区居民名册（作为 SSU 抽样框）。注意：在实际操作中，居民名册往往不完整、过期——这是最常见的抽样框问题。我们要求各街道在调查开始前更新名册。

抽取样本

按 PPS 方法从每个区县抽取 5 个街道/乡镇（共 60 个 PSU）。在每个被选中的街道中，按系统抽样方法从居民名册中抽取 100 个居民。

培训调查员

面向 60 个街道的调查员（每街道 2 人，共 120 人）开展为期 3 天的集中培训。内容包括：调查目的、入户规范、问卷填写、伦理要求、安全须知。培训后统一考核，不合格者补考。

实战小技巧：在培训阶段就设计好"模拟入户"环节——让调查员两两配对，一人扮演拒访者，另一人练习说服技巧。这个环节非常有效，能显著降低正式调查中的拒访率。我们在第一次培训时发现，很多调查员在面对"我没时间"的回应时就直接放弃了，经过模拟训练后，他们的二次接触成功率提高了约 30%。

步骤4：加权

数据收上来之后，第一步不是分析，而是加权。我们的加权方案分为三步：

# 加权计算示例 (Python / pandas)
import pandas as pd
import numpy as np

# 假设数据已经清理完毕
data = pd.read_csv('health_survey_clean.csv')

# ========== 第1步：基础权重 ==========
# 基础权重 = 1 / (PSU选中概率 * SSU选中概率)
# PSU选中概率（PPS）：(每街道抽选数 * 街道人口 / 区县人口)
# SSU选中概率：100 / 街道常住人口

data['psu_prob'] = (5 * data['street_pop']) / data['district_pop']
data['ssu_prob'] = 100 / data['street_pop']
data['base_weight'] = 1 / (data['psu_prob'] * data['ssu_prob'])

# ========== 第2步：无应答调整 ==========
# 按街道计算无应答调整因子
response_rates = data.groupby('street_id').apply(
    lambda x: x['responded'].sum() / len(x)
)
data = data.merge(
    response_rates.rename('rr').reset_index(),
    on='street_id'
)
data['adj_weight'] = data['base_weight'] / data['rr']

# ========== 第3步：事后分层 ==========
# 利用人口普查的年龄x性别分布进行校准
# 构造年龄-性别组
data['age_group'] = pd.cut(data['age'],
    bins=[18, 30, 45, 60, 100],
    labels=['18-29', '30-44', '45-59', '60+'])
data['post_stratum'] = data['gender'] + '_' + data['age_group']

# 人口总体各层人数（来自人口普查）
pop_totals = {
    '男_18-29': 680000, '女_18-29': 650000,
    '男_30-44': 750000, '女_30-44': 730000,
    '男_45-59': 720000, '女_45-59': 710000,
    '男_60+': 450000,  '女_60+': 510000
}

# 事后分层权重 = 总体层人数 / 加权样本层人数
for stratum, total in pop_totals.items():
    mask = data['post_stratum'] == stratum
    weighted_n = data.loc[mask, 'adj_weight'].sum()
    data.loc[mask, 'post_weight'] = data.loc[mask, 'adj_weight'] * (total / weighted_n)

# 最终权重
data['final_weight'] = data['post_weight']
权重检查：加权完成后要做"权重诊断"。检查权重的基本统计量——均值、标准差、最大值、最小值。如果最大权重是最小权重的 10 倍以上，说明权重的变异过大，需要考虑权重截尾（weight trimming）。通常的做法是将权重超过某个阈值（如均值的 3 倍）的值截断到该阈值。在我们的项目中，权重范围在 0.8 到 4.2 之间，变异在可接受范围内。

步骤5：分析

加权完成后，进入核心分析阶段。关键分析代码：

# 加权估计 (Python)
import numpy as np
import pandas as pd

# 加权患病率估计
def weighted_rate(data, var, weight_col='final_weight'):
    """计算加权率"""
    numerator = (data[var] * data[weight_col]).sum()
    denominator = data[weight_col].sum()
    return numerator / denominator

# 全市高血压患病率
htn_rate = weighted_rate(data, 'hypertension')
print(f"高血压加权患病率: {htn_rate:.1%}")

# 各区县加权患病率
dist_rates = data.groupby('district_id').apply(
    lambda x: weighted_rate(x, 'hypertension')
)
print("各区县患病率:")
print(dist_rates)

# Bootstrap 方差估计
np.random.seed(42)
B = 500
boot_estimates = []
for _ in range(B):
    boot_idx = np.random.choice(len(data), size=len(data), replace=True)
    boot_data = data.iloc[boot_idx]
    boot_est = weighted_rate(boot_data, 'hypertension')
    boot_estimates.append(boot_est)

boot_se = np.std(boot_estimates, ddof=1)
ci_lower = htn_rate - 1.96 * boot_se
ci_upper = htn_rate + 1.96 * boot_se

print(f"标准误 (Bootstrap): {boot_se:.3f}")
print(f"95% CI: [{ci_lower:.3f}, {ci_upper:.3f}]")

# 设计效应
deff_est = boot_se**2 / (htn_rate * (1 - htn_rate) / len(data))
print(f"设计效应 (DEff): {deff_est:.2f}")

步骤6：报告

最后一步是产出调查报告。除了具体的分析结果外，方法学部分必须包含：

  • 应答率：总体应答率 78.5%，各区县应答率 62.3%-89.1%
  • 加权方法：基础权重（PPS 倒数）→ 无应答调整（街道层面）→ 事后分层（年龄×性别）
  • 设计效应：核心指标的平均 deff = 1.89（范围 1.35-2.42）
  • 标准误估计方法：Bootstrap（500 次重复）
  • 局限性：郊区 2 个街道的应答率偏低（< 65%），该区域的估计值需谨慎解读

整个项目的时间线和关键决策点总结如下：

时间 阶段 关键决策 / 事件
第1个月 方案设计 确定分层二阶段 PPS 设计；与市政府确认预算
第2个月 抽样准备 收集各区县人口数据；更新抽样框；编制问卷
第3个月初 调查员培训 120 人集中培训 3 天；模拟入户训练
第3-4个月 数据收集 实时监控应答率；发现某街道应答率仅 45%，紧急增派督导员
第5个月 数据处理 数据清理、逻辑校验；加权计算；发现部分问卷关键变量缺失严重（>15%）
第5-6个月 分析撰写 加权估计、方差估计；报告撰写；结果汇报
真实项目中一定会遇到的突发状况：在第 3-4 个月的数据收集阶段，我们遇到了一个实际问题——郊区某街道发生了行政区划调整，原来的居民名册已经不适用。由于我们设置了实时应答率监控机制，这个问题在第一周就被发现了（应答率异常低）。我们立即启动了应急预案：联系新街道的办事处获取最新名册，重新抽取样本。如果等到数据收集完了才发现，这就是一个无法补救的抽样偏差。所以再次强调：质量控制必须是实时的，不能是事后追溯的。
