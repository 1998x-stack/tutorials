第二章：必备数学工具箱

大家好，我是你们的讲师。欢迎来到第二章！

上一章我们聊了强化学习的基本框架——智能体、环境、状态、动作、奖励，以及它是怎么跟监督学习和无监督学习分家过日子的。但我敢打赌，你心里肯定有个疑问："这东西听着挺酷，可真要我自己写算法，从哪儿下手？"

别急。在动手写代码之前，我们需要先准备好三样数学工具。它们就像三国里刘备的"三顾茅庐"——没有这三顾，请不出诸葛亮；没有这三样数学基础，后面所有的 RL 算法都很难真正理解透。这三样是：概率论、线性代数 和 梯度优化。

我知道你在想什么——"又是数学课？我当初学 RL 就是为了逃避数学的！" 请给我一个机会解释：我们不讲那种为了证明而证明的纯数学，我们讲的每一条公式，都会立刻告诉你它在 RL 里具体用在哪里。数学是地图，不是路障。

🎯 本章路线图： 概率论 → 线性代数 → 梯度下降 → 三者在 RL 中的交汇。每学一个概念，我们就问一次："这个在强化学习里对应什么？"

2.1 概率论基础——预测不确定的世界

强化学习的核心场景是：智能体在当前状态下做一个动作，环境返回下一个状态和一个奖励。但关键问题是——环境是有随机性的。你按同一个按钮，游戏里的反应可能每次都不一样。这就是概率论要管的事。

2.1.1 随机变量与概率分布

随机变量 (Random Variable) 就是一个取值不确定的变量。比如你掷一个骰子，结果 X 可以取 1 到 6，每个概率是 1/6。这个"每个值的概率分布"就是 Probability Distribution。

在 RL 里，最经典的概率分布出现在策略 (Policy) 中。策略 π(a|s) 本质上就是一个条件概率分布——给定状态 s，采取每个动作 a 的概率是多少。比如在迷宫里面，当智能体在岔路口时，策略可能给出 "向左走 70%，向右走 30%" 的概率分布。

💡 RL 关联： 概率论在 RL 中的核心应用——当你看到符号 P(s' | s, a) 时，它在说"在状态 s 执行动作 a 后，转移到状态 s' 的概率"。这就是环境模型 (Model) 的数学基础。这就像你女朋友说"如果你再打游戏到半夜，我生气的概率是 90%"——一个条件概率分布。

2.1.2 期望 (Expectation)

期望就是加权平均。如果考试成绩有 50% 概率是 90 分，50% 概率是 70 分，那期望就是 0.5×90 + 0.5×70 = 80 分。公式是：

E[X] = Σ p(x) · x     (离散情况)
E[X] = ∫ p(x) · x dx  (连续情况)

期望在 RL 中无处不在。为什么？因为 RL 的终极目标就是最大化期望累积奖励，而不是最大化某一次随机得到的奖励。你训练一个机器人走迷宫，跑一百次，有的跑完全程有的半路撞墙，我们关注的是"平均下来能拿多少奖励"。

用数学写出来就是：

目标函数: J(π) = E[ Σ γᵗ · rₜ ]   (t 从 0 到 T)

含义: 在策略 π 下，把所有时间步 t 的奖励 rₜ 乘以折扣因子 γᵗ，
     然后对整个轨迹的随机性取期望。

2.1.3 条件概率与贝叶斯思维

条件概率 P(A|B) 的意思是"已知 B 发生的情况下，A 的概率"。公式是 P(A|B) = P(A∩B) / P(B)。

RL 里的 Markov Property (马尔可夫性) 本质上就是条件概率的简化——"给定现在，未来与过去独立"。也就是说，P(sₜ₊₁ | sₜ, aₜ, sₜ₋₁, aₜ₋₁, ...) = P(sₜ₊₁ | sₜ, aₜ)。这个假设让 RL 问题变得可解。没有它，每个决策都需要回顾整个历史，计算量直接爆炸。

⚠️ 踩坑故事——期望的陷阱

我刚开始做 RL 项目的时候，写了一个 CartPole (倒立摆) 的 DQN 训练脚本。训练了 200 轮，我把"最近 10 轮的平均奖励"打印出来看——平均 195 分，接近满分 200。

我激动地直接部署到真实环境测试。结果小车一秒就倒了。

后来排查才发现：我犯了一个概率论级别的低级错误——我把评估时的 epsilon-greedy 的 epsilon 设成了 0.1，留了 10% 的随机探索。虽然平均期望是 195，但那一轮真实运行恰好遇到了那 10% 的随机动作，直接翻车。

教训：期望高不等于每次表现都好。尤其是 RL 评估时要关掉探索 (epsilon=0)，用确定性策略去测。记住：你关心的应该是 E[R] 的实际表现分布，而不仅仅是它的平均值。

2.2 线性代数核心——让计算变得高效

如果说概率论是 RL 的"语义层"，那线性代数就是 RL 的"高性能引擎"。当你的状态空间有几百上千个维度、动作有几十种时，不用矩阵运算，纯 for 循环能把你的电脑跑冒烟。

2.2.1 向量与状态表示

向量 (Vector) 就是一组数字排成一列。在 RL 里，状态几乎永远用向量表示。比如 Atari 游戏的一帧画面是 84×84 像素的灰度图，展平后就是 7056 维的向量。机器人关节的角度、速度也可以拼成一个向量：

状态向量 s = [关节角₁, 角速度₁, 关节角₂, 角速度₂, ..., 目标位置偏差]ᵀ

在代码里就是:
state = np.array([0.52, -0.13, 1.24, 0.07, 0.85])
维度 = len(state)  # 比如 5 维

2.2.2 矩阵与状态转移

矩阵 (Matrix) 是数字的二维表格。对于有限状态空间（比如网格世界有 100 个格子），状态转移可以用一个 转移矩阵 (Transition Matrix) 来描述。矩阵的第 i 行第 j 列表示从状态 i 到状态 j 的概率。

从\到 状态 A 状态 B 状态 C
状态 A 0.7 0.2 0.1
状态 B 0.3 0.5 0.2
状态 C 0.1 0.6 0.3

每一行的和都是 1.0——因为从某个状态出发，总得去往某个地方（包括原地不动）。这个矩阵在 Value Iteration (值迭代) 和 Policy Evaluation (策略评估) 中会被反复使用。

2.2.3 矩阵乘法——神经网络的核心操作

矩阵乘法 C = A × B 中的每个元素是 A 的第 i 行与 B 的第 j 列的点积。这听起来抽象，但在 RL 的神经网络中它的含义非常直观：

  • 输入层 → 隐藏层： 状态向量 s (1×n) × 权重矩阵 W₁ (n×h) + 偏置 b₁ = 隐藏特征 (1×h)
  • 隐藏层 → 输出层： 隐藏特征 (1×h) × 权重矩阵 W₂ (h×m) = 每个动作的 Q 值 (1×m)

一个最简单的 DQN 网络就是一个矩阵乘法链：

import numpy as np

def simple_q_network(state, W1, b1, W2, b2):
    # state: (4,)  — 比如 CartPole 的 4 维状态
    # W1:    (4, 128) — 第一层权重
    # b1:    (128,)   — 第一层偏置
    # W2:    (128, 2) — 第二层权重（2 个动作）
    # b2:    (2,)     — 第二层偏置

    hidden = np.maximum(0, state @ W1 + b1)  # ReLU 激活
    q_values = hidden @ W2 + b2               # 输出 Q 值
    return q_values

# 计算 Q 值
q = simple_q_network(np.array([0.1, -0.5, 0.3, 0.2]),
                     np.random.randn(4, 128),
                     np.zeros(128),
                     np.random.randn(128, 2),
                     np.zeros(2))
print(f"Q(s, a) = {q}")  # e.g. [0.42, -0.15]

你有没有发现？这个"矩阵乘法 + 非线性激活"的堆叠就是深度学习能做函数逼近 (Function Approximation) 的根本原因。而函数逼近恰是 RL 从"小规模表格方法"跨越到"大规模深度学习"的桥梁。

2.3 梯度下降与最优化——让智能体学会进步

有了概率论描述不确定性，有了线性代数做高效计算，怎么让智能体变强？这就是梯度下降登场的时候了。

2.3.1 梯度下降的直觉

想象你站在一座山上，周围大雾弥漫，你要走到山谷的最低点。你看不清整座山，但你能感觉到脚下哪个方向是下坡。梯度下降就是那个"感觉"的过程：

  1. 计算当前位置的 梯度 (Gradient)——即"最陡的上坡方向"
  2. 朝梯度的反方向走一小步——因为你要下山
  3. 重复，直到你走到（或接近）谷底
💡 关键类比： 梯度 ∇f 告诉你"往哪个方向函数值增长最快"。你取负号 -∇f 就是往下降最快的方向走。学习率 η 就是"一步迈多大"。

2.3.2 学习率 (Learning Rate)

学习率是梯度下降里最重要的超参数，没有之一。

学习率 特征 后果
过大 (如 1.0) 步子太大 在谷底附近震荡甚至发散，永远不收敛
过小 (如 1e-6) 步子太小 收敛极慢，训练到天荒地老
适中 (如 1e-3 ~ 1e-4) 稳步下降 在合理时间内收敛到较优解

在 RL 中，学习率的选择比监督学习更敏感。因为 RL 的 训练数据是非平稳的 (non-stationary)——智能体的策略一直在变，导致它看到的"数据分布"也在变。学习率太大，策略会剧烈震荡；学习率太小，策略更新太慢，探索了半天也学不到东西。

2.3.3 损失函数 (Loss Function) 在 RL 里的工作方式

在监督学习中，损失函数是预测值和真实值的差距，比如 MSE Loss = (ŷ - y)²。但在 RL 中，loss function 的构建更巧妙：

# DQN 中的损失函数示例
# Q_target = r + γ · max_a Q(s', a)   —— 贝尔曼方程告诉我们"目标值"是多少
# Q_predicted = Q(s, a)                —— 网络当前对 Q(s,a) 的估计

loss = MSE(Q_predicted, Q_target.detach())
# 注意 detach()：我们只让梯度流回 Q_predicted，不流回 Q_target

# 或者写成展开式：
# loss = (r + γ · max_a Q(s', a) - Q(s, a))²

背后的哲学是：我们并没有现成的"正确答案"来训练 RL 网络。我们用贝尔曼方程 (Bellman Equation) 自己造出一个目标值。这个"自己教自己"的思想叫 Temporal Difference Learning (时序差分学习)，我们会在第 7 章详细展开。

📌 重要结论： 无论 RL 算法看起来多复杂（PPO、SAC、TD3 等等），它们的核心都在做同一件事：定义一个损失函数，然后用梯度下降优化它。理解了这个，你就抓住了 90% 深度强化学习的本质。

2.4 数学通向强化学习的桥梁

现在我们把三样工具串起来，看看它们如何共同支撑起 RL 的大厦。

数学工具 RL 中的具体应用 典型公式 / 算法
概率论 策略 π(a|s)、状态转移 P(s'|s,a)、期望累积奖励 J(π) = E[Σ γᵗ rₜ]
线性代数 状态向量、Q-table、神经网络权重、矩阵形式的贝尔曼方程 V = R + γ P V
梯度下降 策略梯度、Q 网络训练、Actor-Critic 参数更新 θ ← θ + α · ∇J(θ)

让我们用一个具体链条把它们串起来：

  1. 概率论 定义了 RL 的目标——最大化期望累积奖励。因为有随机性，我们不能只看一次结果，必须看期望。
  2. 线性代数 让这个"期望累积奖励"可以被高效计算——状态是向量，转移是矩阵，值函数是向量，整个 Value Iteration 就是矩阵-向量运算。
  3. 梯度下降 让我们可以优化连续的、参数化的策略和价值函数——当状态空间太大无法用表格存储时，我们用神经网络 (矩阵乘法的堆叠) 做函数逼近，用梯度下降来更新网络参数。
💡 回头看第一章： 还记得第一章那个"Agent-Environment 循环图"吗？概率论描述了环境反馈的随机性，线性代数描述了状态和 Q 值的计算方式，梯度下降描述了 Agent 如何通过经验改进自己。这三个数学工具就是这个循环中每一环的数学引擎。

如果你在这一章的任何一个数学公式上卡住了，不用焦虑。知道每个公式在 RL 中有什么用，比能推导它更重要。我们后面的章节会反复用到这些概念，每次用的时候都会再回顾。重复是最好的老师。
