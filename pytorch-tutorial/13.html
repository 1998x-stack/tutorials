第13章 模型评估与可视化

大家好，我是你们的讲师。

"我的模型在测试集上达到了 99% 的准确率！"——当你听到有人这么说时，先不要急着鼓掌。事实上，99% 的准确率极大概率意味着你的评估方法有问题，或者数据集极其不平衡。我曾经见过一个做欺诈检测的团队，他们激动地报告取得了 99.2% 的准确率——但仔细一查，数据集中只有 0.1% 是欺诈样本，而模型学到的策略是"永远预测不是欺诈"。

这个故事说明：正确评估模型和训练模型同样重要。如果你用错误的指标评判模型，你可能会花几个月的时间去优化一个"看起来好、实际没用"的模型。这一章我们来系统学习：用什么指标、怎么算、怎么看，以及如何用 TensorBoard 让训练过程"可视化"。

13.1 分类任务的四大指标

先明确几个基本概念。对于二分类问题，模型的预测结果可以分为四类：

  • TP（True Positive）：预测为正类，实际为正类（预测正确）
  • TN（True Negative）：预测为负类，实际为负类（预测正确）
  • FP（False Positive）：预测为正类，实际为负类（假阳性，误报）
  • FN（False Negative）：预测为负类，实际为正类（假阴性，漏报）

基于这四个值，我们定义四个最常用的指标：

指标 公式 含义 适用场景
Accuracy（准确率） (TP + TN) / (TP + TN + FP + FN) 所有预测中正确预测的比例 类别均衡时常用
Precision（精确率） TP / (TP + FP) 你预测为正类的结果中，有多少是真的正类 关注"假阳性"危害的场景（如垃圾邮件检测）
Recall（召回率） TP / (TP + FN) 所有真正的正类中，你找出了多少 关注"漏报"危害的场景（如癌症筛查）
F1-score 2 × (P × R) / (P + R) Precision 和 Recall 的调和平均 类别不均衡时的综合指标

在 PyTorch 中从零计算这些指标：

def compute_metrics(y_true, y_pred, num_classes):
    """计算混淆矩阵和各项指标"""
    cm = torch.zeros(num_classes, num_classes, dtype=torch.int64)
    for t, p in zip(y_true, y_pred):
        cm[t, p] += 1

    # 从混淆矩阵计算指标
    tp = cm.diag()                                  # 每个类别的 TP
    fp = cm.sum(dim=0) - tp                         # 每个类别的 FP
    fn = cm.sum(dim=1) - tp                         # 每个类别的 FN
    tn = cm.sum() - tp - fp - fn                    # 每个类别的 TN

    accuracy = tp.sum() / cm.sum()
    precision = tp / (tp + fp + 1e-10)
    recall = tp / (tp + fn + 1e-10)
    f1 = 2 * precision * recall / (precision + recall + 1e-10)

    # Macro 平均（每个类别平等对待）
    macro_f1 = f1.mean().item()

    return {
        'accuracy': accuracy.item(),
        'precision_macro': precision.mean().item(),
        'recall_macro': recall.mean().item(),
        'macro_f1': macro_f1,
        'confusion_matrix': cm,
    }

13.2 什么时候 Accuracy 是"毒药"

前面那个"99% 准确率的欺诈检测模型"是一个经典案例。我们详细分析一下：

假设数据集有 100,000 条记录，其中 100 条是欺诈（正类），99,900 条不是欺诈（负类）。一个模型"永远预测不是欺诈"会得到：

  • TP = 0, TN = 99,900, FP = 0, FN = 100
  • Accuracy = 99,900 / 100,000 = 99.9%

看，99.9% 的准确率！但模型实际上毫无价值——它没有识别出任何一个欺诈案例。这个例子的教训是：当类别极度不均衡时，Accuracy 是误导性的指标。你应该关注 Precision、Recall 和 F1-score。

选择评估指标的经验法则：
  • 类别均衡（各类别占比接近）→ Accuracy
  • 类别不均衡 → F1-score（Macro-F1 或 Weighted-F1）
  • 假阳性成本高（如垃圾邮件）→ Precision
  • 假阴性成本高（如疾病筛查）→ Recall
  • 需要兼顾 Precision 和 Recall → F1-score

13.3 TensorBoard 可视化入门

TensorBoard 是 TensorFlow 推出的可视化工具，但 PyTorch 通过 torch.utils.tensorboard 模块原生支持它。安装方法非常简单：

pip install tensorboard

然后在训练代码中初始化 SummaryWriter：

from torch.utils.tensorboard import SummaryWriter
import datetime

# 创建日志目录（按时间命名，方便区分不同实验）
log_dir = f'runs/experiment_{datetime.datetime.now():%Y%m%d_%H%M%S}'
writer = SummaryWriter(log_dir)

# 在每个 epoch 结束时记录
for epoch in range(num_epochs):
    train_loss = train_one_epoch(...)
    val_loss, val_accuracy = validate(...)

    # 记录标量（loss 曲线）
    writer.add_scalar('Loss/train', train_loss, epoch)
    writer.add_scalar('Loss/val', val_loss, epoch)
    writer.add_scalar('Accuracy/val', val_accuracy, epoch)

    # 记录学习率变化
    current_lr = optimizer.param_groups[0]['lr']
    writer.add_scalar('LR', current_lr, epoch)

writer.close()

启动 TensorBoard 可视化：

tensorboard --logdir runs --port 6006
# 然后在浏览器打开 http://localhost:6006

13.4 监控梯度分布（调试利器）

TensorBoard 还有一个极其有用的功能——记录梯度直方图。梯度分布能告诉你模型是否出现了梯度消失或梯度爆炸：

def log_gradients(model, writer, step):
    """记录所有参数的梯度分布"""
    for name, param in model.named_parameters():
        if param.grad is not None:
            writer.add_histogram(
                f'gradients/{name}',
                param.grad,
                step
            )
            writer.add_histogram(
                f'weights/{name}',
                param.data,
                step
            )

# 在训练循环中每隔 N 步调用一次
if step % 100 == 0:
    log_gradients(model, writer, step)

怎么"读"梯度直方图？简单说：

  • 如果梯度集中在 0 附近且范围极小（如 1e-6）—— 梯度消失，网络停止学习
  • 如果梯度的值超过 100 甚至 1000—— 梯度爆炸，可能出现 NaN
  • 如果梯度分布是健康的（以 0 为中心的钟形曲线，范围在 1e-3 到 1 之间）—— 训练正常

13.5 从训练曲线诊断模型健康

学会看训练曲线是深度学习中最重要的技能之一。以下是一个快速诊断指南：

现象 训练 Loss 验证 Loss 诊断 解决方法
两者都不降 高原不动 高原不动 欠拟合（模型容量不足，或学习率太小） 增加模型深度/宽度，或提高学习率
训练降、验证不降 持续下降 先降后升 过拟合 增加正则化（Dropout/Weight Decay），或减少模型容量
两者差距大 很低 高 严重过拟合 数据增强，或使用迁移学习
曲线剧烈抖动 上下震荡 上下震荡 batch size 太小，或学习率太高 增大 batch size，或降低学习率
突然变成 NaN 突然跳到 NaN — 梯度爆炸 梯度裁剪，或降低学习率

13.6 模型比较的方法论

在实验研究中，你经常需要比较不同模型的性能。正确的比较方法是：

  • 使用相同的评价协议：所有模型都在相同的训练/验证/测试集划分下评估
  • 多次运行取统计量：至少运行 3-5 次（不同随机种子），报告均值 ± 标准差
  • 同时报告多个指标：不要只报 accuracy，要报 Precision、Recall、F1 的完整表格
  • 记录评估时的计算资源：模型大小（参数量）、推理时间、显存占用，这些在实际部署时很重要
实验跟踪最佳实践：不要手动记录实验结果！使用实验管理工具（如 MLflow、Weights & Biases、或者最简单的 Excel 表格）记录每个实验的超参数和指标。你会发现，当你有几十个实验时，没有系统性的记录，你根本不知道"哪个配置跑出了最好结果"。

这一章我们学习了模型评估和可视化：从混淆矩阵到四大分类指标，从 TensorBoard 的标量/梯度记录到训练曲线的健康诊断。评估不是训练完成后的"收尾工作"，而是训练过程中的导航仪——没有它，你不知道模型在往哪个方向走，也不知道是否需要调整方向。下一章我们学习迁移学习——如何站在"巨人"的肩膀上，用预训练模型快速解决自己的任务。
