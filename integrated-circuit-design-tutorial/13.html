第13章 低功耗设计技术

大家好，我是你们的讲师。在芯片设计的早期（0.18um及以上工艺时代），功耗几乎不是一个设计约束——你只需要把功能做对、时序做够，剩下的功耗问题交给封装和散热解决就行了。但到了先进工艺节点（28nm及以下），情况完全变了。功耗已经成为和时序、面积并列的"三大设计约束"之一——在很多场景中，功耗甚至是首要约束。

想想你的手机：如果芯片功耗高1W，意味着电池续航可能减少10-15%，或者手机需要更厚重的散热方案。再看看数据中心：一颗GPU芯片动辄300-700W，一个数据中心有数万颗GPU——电费就是天文数字。在AI时代，甚至出现了"电力不足"制约AI算力发展的局面。所以，低功耗设计不再是"加分项"，而是"必选项"。这一章我们来系统性地学习现代低功耗设计技术。

13.1 功耗来源分析

芯片的功耗由两部分组成：动态功耗（Dynamic Power）和静态功耗（Static Power）。

动态功耗是电路在工作时消耗的功率，包括两个部分：

  • 开关功耗（Switching Power）：每一次信号翻转时，对负载电容充放电消耗的能量。公式：P_sw = α · C · V² · f，其中α是翻转活动因子（activity factor），C是负载电容，V是电源电压，f是工作频率。这是动态功耗的主要来源。
  • 短路功耗（Short-Circuit Power）：在信号翻转瞬间，PMOS和NMOS同时导通，形成从VDD到VSS的短暂短路电流。通常占总动态功耗的10-20%。

静态功耗（又称漏电功耗 Leakage Power）是电路在静止状态下仍然消耗的功率，主要来源于：

  • 亚阈值漏电流（Subthreshold Leakage）：Vgs小于Vth时，晶体管并未完全关断——仍有微弱的漏电流。
  • 栅极漏电流（Gate Leakage）：栅氧化层极薄时（几个原子层厚度），电子通过量子隧穿直接穿过栅极。
  • 栅极感应漏电流（GIDL）、PN结漏电流等。

在180nm时代，漏电功耗几乎可以忽略。但在28nm时，漏电已占芯片总功耗的20-30%。到了7nm/5nm，漏电功耗可能占总功耗的40-50%——芯片什么都不做也在大量耗电。这就是为什么手机在待机模式下也需要复杂的电源管理。

不同功耗优化技术的典型效果对比：

技术 节省类型 典型节省幅度 设计复杂度 对性能影响
Clock Gating 动态功耗 20-40% 低（自动插入） 无
Operand Isolation 动态功耗 5-15% 中（需修改RTL） 微小
Multi-Vt Libraries 静态功耗 30-50% 低（综合工具自动选择） 可能有setup退化
Power Gating 静态功耗 80-99%（关断域内） 高（需UPF, isolation, retention） 唤醒延迟大
DVFS 动态+静态 30-60% 高（需PMIC, 软件协同） 性能动态调整
Adaptive Voltage Scaling 动态+静态 额外10-20% 高（片内电压检测） 无（保持性能）
Substrate Biasing 静态功耗 50-80% 较高（需三阱工艺） 可能影响Vth

13.2 Clock Gating

Clock Gating（时钟门控）是最基本、最广泛使用的低功耗技术。它的原理极其简单：当一个寄存器不需要更新数据时，把它时钟关掉。

在RTL中，你可能会写出这样的代码：

// 不推荐：时钟一直翻转
always @(posedge clk) begin
    if (en)
        data_q <= data_in;
end

在这段代码中，即使en为低，每个时钟周期data_q也会被"写回自身"——时钟一直在翻转，D触发器的内部节点每一次都在消耗动态功耗。综合工具可以自动识别这种模式，插入clock gating：

// 综合工具插入clock gating后的等效电路：
wire gated_clk = clk & en;
always @(posedge gated_clk) begin
    data_q <= data_in;
end

Clock gating的实现有两种方式：

  • AND-based：用AND门直接与使能信号（简单但有glitch风险）。
  • Latch-based：用透明低电平锁存器"锁住"使能信号，再与时钟AND。这种方法消除了glitch，是工业标准做法。

Clock Gating分为Fine-Grain（每个寄存器组单独gating）和Coarse-Grain（多个寄存器共享一个gating逻辑）。Coarse-grain节省更多功耗（共享的clock buffer更少），但gating效率低一些。

13.3 Power Gating

Clock gating只能减少动态功耗，但寄存器在时钟关掉时仍然在漏电。如果某个模块在较长时间内完全不工作（比如手机的视频编解码器在你听音乐时），最好的办法是——直接切断它的电源。这就是Power Gating。

Power Gating通过在模块的供电路径上插入休眠晶体管（Sleep Transistor）来实现：

  • Header Switch：在VDD和模块之间串联PMOS。当sleep信号为高时，PMOS关断，模块的VDD被切断。
  • Footer Switch：在VSS和模块之间串联NMOS。当sleep信号为高时，NMOS关断，模块的VSS被浮空。

Power Gating设计中的关键挑战：

  • Inrush Current（浪涌电流）：切断VDD时，模块输出节点的电容上还有电荷。恢复供电的一瞬间，这些电容同时充电，会产生巨大的浪涌电流——可能烧毁电源线或导致供电电压瞬间跌落。
  • State Retention：电源切断后，所有寄存器的状态丢失。如果模块需要"记住"断电前的状态，必须使用状态保持寄存器（State Retention Flip-Flop, SRFF）——一种在断电时用"always-on"的小电源域保存状态的特殊触发器。
  • Isolation Cells（隔离单元）：模块断电后，输出变为未知（X）。如果这个输出连接到仍然上电的模块，会传播X态甚至导致短路电流。需要在边界上插入隔离单元——在断电时将输出钳位到0或1。
  • Wake-up Sequence：恢复供电不是简单的"sleep=0"——需要先打开电源开关，等待电压稳定，然后再释放隔离，最后释放状态保持。这通常需要硬件状态机或PMIC（电源管理芯片）来控制。
真实的"烧板"故事：某团队在做一颗28nm IoT芯片时，设计了一个power gating域。在仿真中，所有断电和唤醒时序都通过了验证。但当芯片第一次上电测试时——刚进入唤醒模式的瞬间——听到了"啪"的一声，测试板的一块区域冒烟了。调查发现：power switch的size设计得太小了——为了节省面积，设计师用了最小尺寸的PMOS做header switch。唤醒时，模块内的数十万寄存器和组合逻辑节点同时充电，浪涌电流远超过sleep transistor能承受的限值，导致晶体管瞬间烧毁。修复方案是把sleep transistor的尺寸增大3倍——代价是面积增加了约5%。从这以后，该团队的设计规范中增加了一条：sleep transistor的总宽度必须根据inrush current仿真结果来确定，不能凭经验猜测。

13.4 多电压域与DVFS

芯片的不同模块对性能的要求不同——CPU核需要工作在最高电压以获得最高频率，而GPIO、定时器等外设可以在低电压下运行。使用不同的供电电压给不同模块——这就是多电压域（Multi-VDD）设计。

多电压域设计要求在电压域之间插入电平转换器（Level Shifter）——一种把信号从一个电压域转换到另一个电压域的单元。电平转换器需要双向支持（高→低、低→高），并且不能消耗太多功耗。

DVFS（Dynamic Voltage and Frequency Scaling）是移动芯片最关键的功耗管理技术之一。其核心思想是：根据当前工作负载，动态调整供电电压和工作频率。CPU在跑游戏时，电压拉到最高、频率提升到最大；CPU在待机时，电压降到最低、频率降到最小甚至暂停时钟。

DVFS的实现需要芯片与PMIC协同——芯片通过I2C或类似接口向PMIC发送"请把VDD_CORE从1.0V降到0.7V"的指令，PMIC调整输出电压，芯片在电压稳定后调整PLL频率。这整个过程可能只需要几十微秒。

AVS（Adaptive Voltage Scaling）是DVFS的进阶版——芯片内部有一个电压检测环（voltage monitor），实时测量芯片的"速度"，自动将供电电压调整到恰好满足性能需求的最低值。由于制造偏差的存在，同一批芯片中，快的芯片可以在更低电压下运行——AVS能榨出每一颗芯片的能效潜力。

13.5 UPF/CPF低功耗意图

随着低功耗设计技术越来越复杂，我们需要一种标准化的方式来描述"电源怎么连接、怎么关断、怎么恢复"。这就是UPF（Unified Power Format, IEEE 1801）和CPF（Common Power Format）的作用。

UPF文件描述了芯片的"功耗意图（power intent）"——它独立于RTL代码，定义了所有的电源域、电源网络、开关策略、隔离策略和状态保持策略。EDA工具（综合、物理设计、时序分析）读入UPF后，自动插入所需的所有低功耗单元。

一个典型的UPF片段：

# ============================================
# UPF低功耗意图描述 (IEEE 1801)
# ============================================

# 定义电源域
create_power_domain PD_CORE -include_scope {
    u_cpu_top
}
create_power_domain PD_IO -include_scope {
    u_uart u_spi u_gpio
}
create_power_domain PD_ALWAYS_ON -include_scope {
    u_wake_ctrl u_pmc
}

# 定义电源网络
create_supply_net VDD_CORE -domain PD_CORE
create_supply_net VDD_IO   -domain PD_IO
create_supply_net VDD_AON  -domain PD_ALWAYS_ON
create_supply_net VSS      -domain {PD_CORE PD_IO PD_ALWAYS_ON}

# 连接电源端口到外部引脚
connect_supply_net VDD_CORE -ports VDD_CORE_PAD
connect_supply_net VDD_IO   -ports VDD_IO_PAD
connect_supply_net VDD_AON  -ports VDD_AON_PAD
connect_supply_net VSS      -ports VSS_PAD

# 定义电源开关
create_power_switch SW_CORE \
    -domain PD_CORE \
    -input_supply_port {in VDD_AON} \
    -output_supply_port {out VDD_CORE_SW} \
    -control_port {sleep sleep_ctrl} \
    -on_state {core_on {in} {!sleep}} \
    -off_state {core_off {sleep}}

# 定义隔离策略
set_isolation ISO_CORE \
    -domain PD_CORE \
    -isolation_power_net VDD_AON \
    -isolation_ground_net VSS \
    -clamp_value 0 \
    -applies_to outputs

# 定义状态保持
set_retention RET_CORE \
    -domain PD_CORE \
    -retention_power_net VDD_AON \
    -save_signal {save_ret_n low} \
    -restore_signal {restore_ret_n low}

# 定义电源状态
add_port_state VDD_CORE_PAD -state {ON 1.0} -state {OFF 0.0}
add_port_state VDD_IO_PAD   -state {ON 1.8}
add_port_state VDD_AON_PAD  -state {ON 1.0}

create_pst pst_main -supplies {
    VDD_CORE_PAD VDD_IO_PAD VDD_AON_PAD
}
add_pst_state pst_full     -pst pst_main -state {ON ON ON}
add_pst_state pst_io_only  -pst pst_main -state {OFF ON ON}
add_pst_state pst_shutdown -pst pst_main -state {OFF OFF ON}

这个UPF定义了三个电源域：核心域（可关断）、IO域（常开）、Always-On域（常开）。核心域有Power Gating控制、隔离单元和状态保持。电源状态包括全速运行、IO-only待机和关机模式。

一个关键原则：低功耗设计不是在项目后期"加上的"，而是需要在RTL设计阶段就开始规划。如果你在RTL里把CPU核、GPU核、总线控制器、DMA、外设等所有的信号糊在一起混写，到了UPF阶段会非常痛苦——因为你需要在已经有大量连接关系的信号中去划分电源域边界。好的做法是：在架构设计阶段就确定哪些模块属于哪个电源域，物理上在顶层用明确的接口封装好跨域通信。电源域规划越晚，代价越大。
