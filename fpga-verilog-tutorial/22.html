第22章 高速串行接口设计

如果你打开一块现代 FPGA 开发板的原理图，会看到几组特殊的差分信号对——它们以 "GTP_XX_P/N" 或 "GXB_XX_P/N" 命名，连接到 SMA 连接器或 PCIe 金手指上。这些就是 FPGA 的高速串行收发器（SerDes）通道。单根信号线的速率从 1 Gbps 到 32 Gbps 不等，远高于传统的并行 LVDS 或 CMOS I/O。本章我们深入理解这些高速接口的工作原理，并学习如何在 FPGA 上实现常见的串行协议。

22.1 从并行到串行——为什么高速要用串行

并行总线（如传统的 32-bit PCI、EMIF、Utopia）曾经是高速传输的主流方案。但随着速率提升，并行总线遇到了一个根本性的物理限制——信号歪斜（SKEW）。32 位并行数据在 PCB 上走线长度不同，到达时间偏差增加；加上各条线路的介电常数差异、温度变化，导致接收端的采样窗口越来越小。当数据速率超过几百 MHz 时，并行总线的时序裕量几乎为零。

SerDes（Serializer / Deserializer）的解决方案简单而优雅：不再同时传输多条信号线，而是把数据串化成高速单比特流。只需要一对差分线（P/N），配合时钟恢复电路从数据流中提取时钟，完美消除了 SKEW 问题。代价是 SerDes 的模拟电路设计难度远高于并行 I/O——这正是 FPGA 厂商的核心竞争力之一。

22.2 SerDes 基本原理

一个典型的 SerDes 通道包含以下几个关键模块：

  • 8b/10b 编码：将每 8 位数据映射到 10 位码字。保证 DC 平衡（Equal number of 0s and 1s），提供足够的电平跳变用于时钟恢复。编码后的运行差异度（Running Disparity, RD）用于消除直流偏移。常见的替代方案还有 64b/66b（PCIe Gen3+、10GbE）和 128b/130b（PCIe Gen4+）。
  • 预加重（Pre-Emphasis）与去加重（De-Emphasis）：在发送端对信号的高频分量预加强，补偿传输线的高频衰减。
  • 接收均衡：接收端通过连续时间线性均衡器（CTLE）和判决反馈均衡器（DFE）恢复信号眼图。CTLE 补偿通道的频响，DFE 消除码间干扰（ISI）。

下面是一个简化版 8b/10b 编码器的 Verilog 实现，采用查找表方法：

// Simplified 8b/10b Encoder — Lookup Table Approach
// 3b/4b encoding for lower nibble, 5b/6b encoding for upper nibble
module enc_8b10b (
    input  wire       clk,
    input  wire       rst_n,
    input  wire [7:0] data_in,       // 8-bit input data
    input  wire       data_valid,
    output reg  [9:0] data_out,      // 10-bit encoded output
    output reg        out_valid
);
    reg rd;  // running disparity: 0 = negative, 1 = positive

    // 5b/6b encoding (simplified — 8 entries for illustration)
    function [5:0] encode_5b6b;
        input [4:0] symbol;
        input       cur_rd;
        case (symbol)
            5'b00000: encode_5b6b = cur_rd ? 6'b011000 : 6'b100111;
            5'b00001: encode_5b6b = cur_rd ? 6'b100010 : 6'b011101;
            5'b00010: encode_5b6b = cur_rd ? 6'b010010 : 6'b101101;
            5'b00011: encode_5b6b = 6'b110001;
            5'b00100: encode_5b6b = cur_rd ? 6'b001010 : 6'b110101;
            default:   encode_5b6b = {symbol[0], symbol[1], symbol[2],
                                      symbol[3], symbol[4], 1'b0};
        endcase
    endfunction

    // 3b/4b encoding (simplified)
    function [3:0] encode_3b4b;
        input [2:0] symbol;
        input       cur_rd;
        case (symbol)
            3'b000: encode_3b4b = cur_rd ? 4'b0100 : 4'b1011;
            3'b001: encode_3b4b = cur_rd ? 4'b1001 : 4'b1001;
            3'b010: encode_3b4b = cur_rd ? 4'b0101 : 4'b0101;
            3'b011: encode_3b4b = 4'b0011;
            default: encode_3b4b = 4'b0000;
        endcase
    endfunction

    always @(posedge clk) begin
        if (!rst_n) begin
            rd        <= 1'b0;
            data_out  <= 10'd0;
            out_valid <= 1'b0;
        end else begin
            out_valid <= data_valid;
            if (data_valid) begin
                // Upper 5 bits (5b/6b), Lower 3 bits (3b/4b)
                data_out[9:4] <= encode_5b6b(data_in[7:3], rd);
                // Update running disparity (simplified)
                rd <= rd ^ (^data_in[7:3]);  // XOR reduction for demo
                data_out[3:0] <= encode_3b4b(data_in[2:0], rd);
            end
        end
    end
endmodule
注意：上面的代码是教学简化版。真实的 8b/10b 编码需要完整的 5b/6b 和 3b/4b 映射表（各 32 和 8 个有效条目），外加严格的运行差异度（RD）更新规则。完整的编码器可以参考 Widmer 和 Franaszek 1983 年的原始论文或者 IBM 的专利文档。在 Xilinx 器件中，GTP/GTX 收发器已经内置了硬件 8b/10b 编码器——通常你不需要自己实现它。

22.3 FPGA 高速收发器：GTP / GTX / GTH / GTY

Xilinx 7 系列及以后的 FPGA 提供了四档 SerDes 收发器，按性能从低到高排列：

收发器最大线速率工艺节点典型器件典型协议
GTP3.75 Gbps28nmArtix-7, Zynq-7000PCIe Gen2, SATA 3G, 1GbE
GTX12.5 Gbps28nmKintex-7, Virtex-7PCIe Gen3, 10GbE, SATA 6G
GTH16.3 Gbps28nmKintex UltraScale, Virtex UltraScalePCIe Gen4, 25GbE, Interlaken
GTY32.75 Gbps16nmKintex UltraScale+, Virtex UltraScale+PCIe Gen5, 100GbE, 112G PAM4

Intel (Altera) 的对应层级是 GX（~17 Gbps）和 SX（~28 Gbps）。Lattice 的 ECP5 和 CertusPro-NX 也集成了 SerDes，速率在 5~10 Gbps 级别。选型时要记住：收发器的数量（通道数）和速率是决定 FPGA 成本的主要因素之一——如果你的设计只需要 1 Gbps 的连接，选择只有 GTP 的 Artix-7 比带 GTY 的 Kintex 经济得多。

22.4 PCIe 接口概述

PCI Express（PCIe）是目前最通用的高速串行互连标准。FPGA 端的典型实现采用 Xilinx 的 XDMA IP 核（DMA for PCI Express），它提供了完整的事务层（TLP）处理：

  • 物理层（PHY）：使用 FPGA 的 GTP/GTX 收发器，负责 8b/10b 或 128b/130b 编码、串并行转换、链路训练（Link Training）。
  • 数据链路层（Data Link Layer）：提供 Ack/Nak 重传机制，确保 TLP 的可靠传输。
  • 事务层（Transaction Layer）：处理 Memory Read/Write、Completion 等 TLP 包的封装与解析。XDMA 把这一层抽象成了 AXI4-Stream 或者 AXI4-Full 接口。

配置 XDMA 时需要在 Vivado 中指定：链路宽度（x1 / x2 / x4 / x8）、最大有效载荷（128~1024 字节）、以及 DMA 描述符队列的数量。对于典型的 PCIe Gen3 x4，理论带宽约为 3.938 GB/s（128b/130b 编码开销纳入后）。

22.5 MIPI DSI / CSI 在 FPGA 上的实现

MIPI（Mobile Industry Processor Interface）是移动设备中最常用的摄像头和显示器接口——DSI（显示）和 CSI（摄像头）。MIPI 在 FPGA 上的实现比 PCIe 更"接地气"，因为很多工业视觉和嵌入式项目需要把 MIPI 摄像头接到 FPGA 做图像处理。

接口协议每通道速率通道数编码方式FPGA 实现方式
PCIe Gen38 GT/sx1 / x4 / x8 / x16128b/130bGTP/GTX + XDMA IP
SATA 6G6 Gbps18b/10bGTP + SATA IP
USB 3.05 Gbps18b/10bGTP + USB3 IP 或外部 PHY
10GbE10.3125 Gbps1/464b/66bGTX/GTH + 10G MAC IP
MIPI D-PHY~1.5 Gbps1~4DDR 差分SelectIO + 电阻网络或专用 IP
JESD204B~12.5 Gbps1~88b/10bGTX/GTH + JESD204 IP

MIPI D-PHY 的电气层比较特殊：它使用DDR 双沿采样和LP（低功耗）/ HS（高速）双模工作。FPGA 的普通 I/O BANK 很难直接满足 MIPI 的 HS 模式下 1.5 Gbps 的时序要求。常见的解决方案有二：

  • 电阻网络方案（低成本）：用 FPGA 的 HP（High Performance）BANK + 外部电阻分压网络将 MIPI 差分信号转换为 LVDS 电平。适用于速率在 1 Gbps 以下的场景。
  • 专用 IP 方案（高可靠）：使用 Xilinx MIPI D-PHY IP（消耗专用的 I/O 资源和 SerDes 通道），支持完整的 CSI-2 / DSI 协议栈。
工程建议：如果你的项目需要多路高速串行接口同时工作（例如 4 路 MIPI CSI 输入 + PCIe 上传主机），一定要在选型阶段计算 SerDes 通道数预算。Artix-7 只有 4 个 GTP 通道，Kintex-7 有 16 个 GTX 通道，而 Kintex UltraScale+ 有多达 64 个 GTY 通道。选错了器件封装意味着额外的 BOM 成本和 PCB 层数。
踩坑——PCIe 链路降速：在一次为期两周的实验室环境测试中，我们的 FPGA 板在室温下 PCIe Gen3 x4 链接正常。但是用热风枪对板卡进行温循测试（0°C→85°C → 0°C 循环），PCIe 链接随机从 Gen3 降到 Gen1，带宽从 ~3.9 GB/s 骤降到 ~0.25 GB/s。用 Vivado 的 IBERT 工具测试后发现——参考时钟的 PPM（百万分之一偏差）在温度变化下达到了 200 PPM，远超过 PCIe Gen3 规范要求的 ±100 PPM（Gen1/Gen2 为 ±300 PPM，Gen3 更严格）。解决方案是换用温漂特性更好的晶振（从普通石英晶振换成了 TCXO，温漂 < 50 PPM），并在 PCB 布局上让参考时钟走线远离大功率器件。
