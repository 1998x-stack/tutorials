第13章 元启发式算法（上）

大家好，我是你们的讲师。上一章我们聊了精确求解方法——它们像狙击枪，精确但每颗子弹都要精心计算。而对于大规模调度问题，我们需要的更像是霰弹枪——不求一枪命中靶心，但求在可接受的时间内给出足够好的解。这就来到元启发式算法（Metaheuristics）的世界。

元启发式这个名字可能有点学术化，但它的核心理念非常接地气：模仿自然界的智慧。进化、退火、群体协作——这些大自然经过亿万年优化的机制，被我们"偷"来求解调度问题。今天上半场，我们先讲两个最经典的：遗传算法（GA）和模拟退火（SA）。

遗传算法 — 自然选择的优化之道

遗传算法（Genetic Algorithm, GA）由 John Holland 在 1975 年提出，灵感来自达尔文的自然选择理论。它的基本流程是：

  1. 初始化：随机生成一个种群（population），每个个体代表一个候选调度方案。
  2. 选择（Selection）：根据适应度（fitness，即 makespan 的倒数或某种评分）选择优秀的个体。
  3. 交叉（Crossover）：将两个父代个体的部分"基因"组合生成子代。
  4. 变异（Mutation）：对子代的某些基因做随机扰动，增加种群多样性。
  5. 替代（Replacement）：用新生成的子代替换种群中较差的个体。
  6. 重复步骤 2-5，直到满足终止条件（如达到最大代数或收敛）。

对于调度问题，最核心的设计决策是编码方式——如何把一个调度方案表示成"染色体"。

一、编码方式

  • 排列编码（Permutation Encoding）：最直观的方式。染色体是一个工件 ID 的排列，代表机器上的加工顺序。适用于单机和流水车间。
  • 优先列表编码（Priority List Encoding）：每个工件分配一个优先级数字，调度时按优先级高低依次安排。可用于更复杂的问题。
  • 基于操作的编码（Operation-Based Encoding）：针对 JSP 的经典方式。每个工件 ID 在染色体中出现 k 次（k 为该工件的操作数），第 t 次出现代表该工件的第 t 道工序。例如染色体 [1, 2, 1, 2] 表示：先做工件 1 的第一道工序，再做工件 2 的第一道工序，然后工件 1 的第二道工序，最后工件 2 的第二道工序。这种编码天然满足工艺约束（同一工件的操作顺序不会逆序）。
  • 随机键编码（Random Key Encoding）：每个工件对应一个 [0,1] 随机数，按随机数大小确定顺序。变异时改变随机数即可。优点是交叉后永远产生合法解。

二、交叉算子

对于排列编码，简单的单点交叉会破坏排列的合法性（数字重复或缺失），所以需要专门设计的交叉算子：

交叉算子 全称 工作原理 适用场景
PMX Partially Mapped Crossover 选择一段子串进行交换，用映射关系修复重复/缺失的基因 TSP、调度问题的排列编码
OX Order Crossover 保留一个父代的子串顺序，从另一个父代取缺失的基因 调度的排列编码（保持相对顺序重要时）
LOX Linear Order Crossover 类似 OX，但随机选择两个切割点 流水车间排列调度
POX Precedence Operation Crossover 专为 JSP 设计的交叉，分别从两个父代继承部分工件的全部操作 作业车间调度

三、变异算子

  • 交换变异（Swap）：随机选择两个位置，交换它们对应的工件。
  • 插入变异（Insertion）：随机选择一个工件，移动到另一个随机位置。
  • 逆序变异（Inversion）：反转一段子序列的顺序。

适应度函数通常定义为 1 / (makespan + 1)（加 1 防止除零），或者使用 (max_makespan - makespan) 的排名。选择策略常用的有：

  • 轮盘赌选择（Roulette Wheel Selection）：适应度高的个体被选中的概率大。
  • 锦标赛选择（Tournament Selection）：随机选 k 个个体，取其中最好的。
  • 精英保留（Elitism）：将每代最好的几个个体直接复制到下一代，防止最优解丢失。
遗传算法 — 编码/交叉/变异伪代码（JSP示例）

// 编码：基于操作的编码（Operation-Based）
// 例如 3个工件各2道工序 → 染色体长度6
// 父代1: [1, 2, 1, 3, 2, 3]
// 父代2: [2, 3, 1, 2, 3, 1]

// POX交叉（Precedence Operation Crossover）
Function POX_Crossover(parent1, parent2):
  随机选择一组工件 J_set（如 {1, 3}）
  子代1 ← 从 parent1 继承 J_set 中工件的所有操作（保持原位置）
  子代1 ← 从 parent2 继承剩余工件的操作（按原顺序填入空位）
  子代2 ← 对称操作
  return 子代1, 子代2

示例:
  J_set = {1, 3}
  子代1骨架: [1, _, 1, _, 3, _]  (来自parent1)
  从parent2取 {2} 的操作(保持顺序): [2, 2]
  填入空位 → [1, 2, 1, 2, 3, 3]

// 交换变异
Function Swap_Mutation(chromosome):
  pos1, pos2 ← random(0, len(chromosome)-1), random(0, len(chromosome)-1)
  swap(chromosome[pos1], chromosome[pos2])
  return chromosome

四、GA 参数调优

GA 的参数设置对性能影响很大，以下是一般参考值：

参数 典型范围 说明
种群大小 50–500 太小易早熟收敛，太大计算慢。一般取 100 起步
交叉率 0.6–0.95 控制产生子代的比例。太高可能破坏优良模式
变异率 0.01–0.1 太低会缺乏多样性，太高会让算法退化为随机搜索
精英数量 1–5 保留每代最好的几个个体，防止最优解丢失
终止代数 100–2000 或连续 N 代无改进时提前终止

一个极其真实的踩坑故事：我刚开始做调度优化时，花了大把时间调 GA 的交叉率、变异率，试图找到完美的参数组合。结果一位经验丰富的同事看了一眼我的代码说："你的编码方式是错的"。我当时很不服气——我用的是标准的排列编码啊。但他指出：在 JSP 中，简单的排列编码会破坏工艺约束——父代合法，但经过交叉后子代可能产生一个工件在前一道工序还没做完时后一道工序就被安排了的荒谬情况。换了基于操作的编码后，同样的参数设置，解的质量直接提升了 20%。从此我得到一个血泪教训：编码方式的选择比算法参数重要十倍。基因编好了，自然选择才有意义；编错了，达尔文来了也救不了。

模拟退火 — 物理退火的算法映射

模拟退火（Simulated Annealing, SA）的灵感来自冶金学中的退火过程：将金属加热到高温，然后缓慢冷却。高温时原子剧烈运动（允许大幅探索解空间），随着温度降低逐渐趋于稳定（收敛到最优解附近）。

SA 算法的核心是 Metropolis 准则：从当前解 x 生成一个新解 x'（在 x 的邻域中随机扰动），如果 f(x') < f(x)（更优），则接受 x'；如果 f(x') ≥ f(x)（更差），以概率 exp(-Δf / T) 接受它，其中 Δf = f(x') - f(x)，T 是当前温度。

这个"以一定概率接受差解"的机制是 SA 的灵魂——它让算法在早期能够跳出局部最优，随着温度降低，接受差解的概率逐渐趋近于零，算法最终收敛。

冷却策略

  • 指数冷却（Exponential Cooling）：T(t) = T₀ · αt，其中 α 通常在 0.8-0.99 之间。这是最常用的策略，降温速度前期快后期慢。
  • 对数冷却（Logarithmic Cooling）：T(t) = T₀ / log(t + 1)。理论上可以保证收敛到全局最优，但需要极多的迭代次数，实践中很少用。
  • 线性冷却（Linear Cooling）：T(t) = T₀ - β · t。简单直接，但需要精心选择 β。

SA 的收敛性理论告诉我们的结论是：只要降温速度足够慢（比如对数冷却），SA 以概率 1 收敛到全局最优。当然，在实际应用中，我们没有足够的计算资源去实现"足够慢"的降温，所以通常使用指数冷却配合精心调校的初始温度 T₀ 和终止温度 Tmin。

模拟退火 — 通用伪代码

算法 SimulatedAnnealing(初始解 x0, 初始温度 T0, 终止温度 Tmin, 冷却率 α)
  x ← x0
  best_x ← x0
  best_f ← f(x0)
  T ← T0

  while T > Tmin:
    for iter = 1 to L:  // L = 每个温度下的迭代次数
      x' ← 在 x 的邻域内随机生成一个新解
      Δf ← f(x') - f(x)

      if Δf < 0:  // 更优解
        x ← x'
        if f(x') < best_f:
          best_x ← x'
          best_f ← f(x')
      else:  // 更差解，以概率接受
        r ← random(0, 1)
        if r < exp(-Δf / T):
          x ← x'

    T ← α * T  // 降温

  return best_x, best_f

一个小建议：SA 的初始温度 T₀ 对性能影响很大。T₀ 太高 → 前期接受太多差解，收敛慢；T₀ 太低 → 算法退化为局部搜索（爬山法），容易陷入局部最优。一个实用的做法是：先随机采样一批解，计算它们的 f 值的标准差 σ，然后设 T₀ = k·σ，其中 k 取 5-10。这样初始的接受概率大约在 0.5-0.8 之间，是一个比较合理的起点。

这一章我们学习了 GA 和 SA 两种最经典的元启发式算法。GA 擅长并行探索和组合，SA 擅长跳出局部最优。下一章我们将继续介绍另外三种流行的元启发式——粒子群算法（PSO）、蚁群算法（ACO），以及最重要的：怎么在调度问题中选择和调优这些算法。我们下一章见。
