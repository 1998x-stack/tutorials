第08章 流水车间调度

大家好，我是你们的讲师。上一章讲单机调度时，你可能觉得"不过如此"——一台机器，排个序而已，能有多难？那么现在我们把难度提升一个台阶：假设不是一台机器，而是五台机器排成一条线，每个工件都要依次经过这五台机器，而且加工路线一模一样——这就是流水车间（Flow Shop）。现在问题还简单吗？

流水车间是制造业中最常见的生产形态。汽车装配线、电子产品组装线、食品加工线——几乎所有的流水线生产都属于 Flow Shop。这一章我们从一个特例开始——两台机器的流水车间——然后逐步推广到更现实的多机场景。

流水车间的定义与假设

用三参数表示法，流水车间写作 F / Cmax。它的核心特征是：所有工件的加工路线完全相同——每个工件都先上机器 1，再上机器 2，一直到最后机器 m。没有任何回溯，没有旁路，所有工件服服帖帖地走同一条路。

在经典的流水车间模型中，有几个基础假设：

  • 置换假设（Permutation）——在每台机器上，工件的加工顺序相同。也就是说，如果工件 A 在机器 1 上先于工件 B 被加工，那么在所有后续机器上，A 仍然先于 B。满足这个假设的称为置换流水车间（Permutation Flow Shop, F-perm / Cmax）。
  • 不可中断——一旦一个工件在某台机器上开始加工，就必须加工完成才能释放机器。
  • 无限缓冲区——每台机器前都有无限大的缓冲区，工件加工完后可以排队等待下一台机器。
  • 无准备时间——机器在不同工件之间切换不需要准备时间（或者准备时间已包含在加工时间中）。

这些假设虽然只是现实世界的近似，但它们使得问题数学上可处理，同时又能捕捉到流水车间调度的核心矛盾：如何安排加工顺序来最小化整个生产线的完工时间。

现实插曲：我曾参观过一家电子元件制造厂，它们的 SMT 贴片生产线就是经典的 Flow Shop——PCB 板依次经过锡膏印刷、贴片、回流焊、AOI 检测四台设备。车间主任告诉我，他们一直靠"先来先服务"排产，直到一次产能审计发现瓶颈设备的利用率只有 62%。换了一种排产顺序后，同样的设备和人员，产能提升了 23%。厂长当场感叹："不花一分钱，换个顺序就能多产四分之一，这钱赚得太轻松了。" 这让我深刻体会到，调度算法的价值往往被严重低估。

Johnson算法 — n/2/F/Cmax的精确解

在所有 Flow Shop 问题中，只有一种情况存在多项式时间的精确算法：两台机器的流水车间，记作 n / 2 / F / Cmax。这个算法由 S.M. Johnson 在 1954 年提出，至今仍然是调度理论中最优雅的结果之一。

Johnson 规则

规则本身极其简单：

  1. 对于每个工件 i，记它在机器 1 上的加工时间为 ai，在机器 2 上的加工时间为 bi。
  2. 如果 ai < bi，将工件 i 放在前面（按 ai 升序排列）。
  3. 如果 ai > bi，将工件 i 放在后面（按 bi 降序排列）。
  4. 如果 ai = bi，可放在任意一组。

合并后的序列就是最优加工顺序。为什么？核心证明思路是：对于两机 Flow Shop，最优解一定是置换调度（两台机器上的工件顺序相同），而 Johnson 规则通过交换论证（adjacent pairwise interchange）可以证明其最优性。简洁、优美、高效——这就是经典算法的魅力。

手工计算示例：5 工件 × 2 机器

假设我们有 5 个工件，加工时间如下：

工件 机器 1 (ai) 机器 2 (bi)
A52
B36
C84
D45
E63

第一步：判断每个工件的类型。

  • A: 5 > 2 → 后面组（bi 降序）
  • B: 3 < 6 → 前面组（ai 升序）
  • C: 8 > 4 → 后面组（bi 降序）
  • D: 4 < 5 → 前面组（ai 升序）
  • E: 6 > 3 → 后面组（bi 降序）

第二步：排序。

  • 前面组（ai 升序）：B(3), D(4) → 顺序 B → D
  • 后面组（bi 降序）：A(2), E(3), C(4) → 顺序 C → E → A

第三步：合并 → B → D → C → E → A

这个序列就是在 n/2/F/Cmax 下的最优加工顺序。我们验证一下 makespan：

Johnson算法 — 核心步骤
输入：工件集合 J，每工件在机器1的加工时间 a[i]、机器2的加工时间 b[i]
输出：最优加工顺序 π

1. 将工件分成两组：
   Group1 ← { i | a[i] ≤ b[i] }
   Group2 ← { i | a[i] > b[i] }

2. Group1 按 a[i] 升序排列
3. Group2 按 b[i] 降序排列

4. π ← Group1 + Group2
5. 计算 makespan: 模拟两台机器上的加工过程

验证示例 5×2：

顺序: B(3,6) → D(4,5) → C(8,4) → E(6,3) → A(5,2)

机器1完工时间:  B:3   D:7   C:15  E:21  A:26
机器2开始时间:  B:3   D:7   C:15  E:21  A:26
机器2完工时间:  B:9   D:14  C:19  E:24  A:28

Makespan = max(26, 28) = 28

如果尝试顺序 A→B→C→D→E，计算可得 makespan = 32
步骤 操作 说明
Step 1 判断 ai ≤ bi 或 ai > bi 分成 Group1（机器1时间短）和 Group2（机器2时间长）
Step 2 Group1 按 ai 升序 机器1时间短的尽量靠前，减少机器1空闲
Step 3 Group2 按 bi 降序 机器2时间长的尽量靠前，减少机器2等待
Step 4 拼接 Group1 + Group2 形成完整的最优序列
Step 5 计算 makespan 验证 模拟每台机器的完工时间

初学者经典误区：以为 Johnson 算法能直接推广到 3 台机器。这是流水车间调度中最大的"美丽的误会"。Johnson 算法只对 n/2/F/Cmax 成立！一旦机器数量 m ≥ 3（即使只有 3 台），问题就从 P 跳到了 NP-hard。有人试图把三台机器"合并"成两台（比如把机器1+2的加工时间求和当作新机器1），这种方法不保证最优，只是启发式。所以记住：Johnson 是两机的特例，不是多机的通则。

CDS法 — Johnson算法向多机的扩展

虽然 Johnson 算法不能直接推广到 m ≥ 3，但 Campbell, Dudek 和 Smith 在 1970 年提出了一个非常聪明的想法：将 m 台机器的 Flow Shop 问题转化为 m-1 个两机问题，每个两机问题用 Johnson 算法求解，最后取最好的一个结果。

具体做法是把原始 m 台机器按不同方式"合并"成两组：

  • 第 k 次迭代的机器 A：机器 1 到机器 k 的加工时间之和。
  • 第 k 次迭代的机器 B：机器 k+1 到机器 m 的加工时间之和。

其中 k = 1, 2, ..., m-1。每一次迭代都产生一个候选解，CDS 取其中 makespan 最小的那个。

CDS 算法虽然不能保证最优解（因为 m ≥ 3 时问题是 NP-hard 的），但它在实践中表现相当不错，尤其是当机器数不多时。它的计算复杂度是 O(m · n log n)，对于大规模问题非常友好。

NEH算法 — 目前最好的启发式之一

如果说 CDS 是"还不错"的启发式，那么 NEH 算法（Nawaz-Enscore-Ham, 1983）就是目前公认的 Flow Shop 启发式中效果最好的之一。它的核心思想是：加工时间越长的工件，越应该优先安排。

NEH 算法的步骤如下：

  1. 计算每个工件的总加工时间 Pi = Σj=1m pij，按 Pi 降序排列得到一个初始序列。
  2. 取出前两个工件，找到使 partial makespan 最小的顺序，作为当前部分序列。
  3. 依次取出下一个工件，尝试将它插入到当前部分序列的每一个可能位置，选择使 makespan 最小的位置。
  4. 重复步骤 3，直到所有工件都插入完毕。

NEH 算法的性能保证：虽然没有严格的近似比证明（因为问题是 NP-hard），但在大量 benchmark 测试中，NEH 的平均偏差通常在 3%-5% 以内，远好于 CDS 和随机排序。

NEH算法 — 伪代码

输入：n个工件，m台机器，加工时间矩阵 p[i][j]
输出：近优序列 π

1. for i = 1 to n:
       计算总加工时间 total[i] = Σ p[i][j]  (j = 1..m)

2. 按 total[i] 降序排列工件 → seq

3. π ← [seq[1], seq[2]]  // 取前两个工件
   选择使 makespan 最小的顺序（尝试两种排列）

4. for k = 3 to n:
       当前工件 cur = seq[k]
       best_cost = ∞
       for pos = 1 to len(π) + 1:
           将 cur 插入到 π 的 pos 位置 → 临时序列 π'
           计算 π' 的 makespan → cost
           if cost < best_cost:
               best_cost = cost
               best_pos = pos
       将 cur 插入到 π 的 best_pos 位置

5. return π

NEH 的计算复杂度为 O(n² · m) —— 因为每个新工件的插入需要尝试 O(n) 个位置，每个位置的 makespan 计算需要 O(m) —— 总共 n 个工件，所以是 O(n² · m)。对于 n=500、m=20 的问题，现代计算机只需要几秒钟。

一个真实的工程教训：我曾在一家汽车零部件厂帮他们做流水线排产。起初我们直接用了学术论文中的标准 NEH 算法，但发现它在实际生产中表现不稳定——有些排产方案的 makespan 虽然小，但某个机器上的负载极其不均衡，导致操作工不停地在机器之间跑来跑去。后来我们在 NEH 的插入阶段增加了一个负载均衡惩罚项，makespan 仅仅增加了 2%，但现场操作工的满意度大幅提升。这提醒我们：调度优化不仅是数学问题，更是人因工程问题。最优解≠最好执行。

最后，我们来对比一下流水车间调度中的几种方法：

方法 适用范围 最优性 时间复杂度
Johnson 算法 n/2/F/Cmax 精确最优 O(n log n)
CDS 启发式 n/m/F/Cmax (m ≥ 2) 近似解 (m-1次Johnson) O(m · n log n)
NEH 启发式 n/m/F/Cmax (m ≥ 2) 近似解（通常更优） O(n² · m)
分枝定界法 小规模 n/m/F/Cmax 精确最优 指数级

好的，到这里我们已经掌握了流水车间的基本模型和三种代表性算法——Johnson 的精确解、CDS 的启发式扩展、NEH 的贪心构造。下一章我们将进入一个更复杂、也更贴近现实的世界——作业车间调度（Job Shop）。在 Job Shop 中，每个工件都有自己的工艺路线，不再千篇一律。那才是调度领域真正的"硬核战场"。我们下一章见。
