第23章 数据获取与处理

大家好，我是你们的讲师。从这一章开始，我们正式进入量化分析的核心环节。前面几章我们讲了多因子模型、行业轮动——这些策略听起来都很美，但有一个前提：你得有高质量的数据。 没有数据，量化分析就是空中楼阁。回测做得再漂亮，数据如果错了，结果就是 garbage in, garbage out。

我自己刚开始做量化的时候，犯过一个特别低级的错误：从Tushare拉了日线数据就直接跑回测，跑了几年发现年化收益高达40%，兴奋得以为发现了圣杯。结果后来仔细一查——数据没有复权。 分红送股导致的价格跳空，让回测收益凭空多了十几个点。这个故事我们后面展开讲。先来说说数据从哪里来。

23.1 金融数据生态概览

想做量化，第一步是找到可靠的数据源。中国的金融数据市场大致可以分为三个梯队：

专业终端——Wind（万得）、Choice（东方财富）、iFinD（同花顺）。这类工具功能强大，数据质量高，但价格不菲。Wind一个终端年费在3-5万元，个人投资者很难承受。不过很多券商的机构业务部会向客户提供Wind权限，如果你有渠道可以利用一下。

开源Python工具——Tushare、BaoStock、AkShare。这三个是个人量化投资者最常用的数据源，完全免费或部分免费。其中Tushare需要注册获取token，BaoStock无需注册开箱即用但不支持分钟线，AkShare覆盖范围最广（股票、期货、外汇、宏观全都有）。

国外数据源——Yahoo Finance（通过yfinance库调用）、Quandl（已被Nasdaq收购）。如果你研究美股或全球市场，这些是标准选项。不过Yahoo Finance的数据质量参差不齐，Quandl的优质数据需要付费。

建议：新手从BaoStock入门（零门槛、高质量），进阶后用Tushare Pro获取更多维度的数据。想做全球策略就加上yfinance。不需要一上来就买Wind——等你跑通了完整的策略流程，再用Wind验证数据质量也不迟。

23.2 行情数据获取与复权处理

行情数据（日线、分钟线）是量化分析的基础。但有一个重要的细节很多人会忽略——复权。

复权的三种方式

股票的股价会因为分红、送股、配股等事件发生非交易性的跳空。如果不做处理，直接用原始价格做回测，结果会严重失真。

  • 前复权：调整历史价格，使价格连续。当前时点的价格就是最新价，历史价格按比例下调。这是回测中最常用的方式，因为你看到的历史价格是"如果把分红再投资，历史上应该是什么价格"。
  • 后复权：调整当前价格，使历史价格不变。当前价格被上调。这种方式适合看长期涨幅——贵州茅台后复权价格早已过万，能真实反映持有期的总回报。
  • 不复权：原始价格，不做任何调整。如果你做超短线交易（持股不超过几天），复权与否影响不大。但做中长期回测，千万不要用不复权数据。

用Tushare获取复权数据

下面是一个用Tushare Pro获取A股日线数据的示例，我们直接请求前复权数据：

import tushare as ts
import pandas as pd

# 设置token（需要在tushare.pro注册获取）
ts.set_token('your_token_here')
pro = ts.pro_api()

# 获取贵州茅台(600519.SH)的日线数据，前复权
df = pro.daily(
    ts_code='600519.SH',
    start_date='20200101',
    end_date='20250630',
    adj='qfq'  # 前复权
)

# 查看数据概览
print(df.head())
print(f"共 {len(df)} 个交易日")
print(f"日期范围: {df['trade_date'].min()} 到 {df['trade_date'].max()}")

# 如果需要同时获取复权因子，用于自定义计算
adj_factor = pro.adj_factor(ts_code='600519.SH')
print(adj_factor.head())

踩坑故事：忘复权的惨痛教训

2020年我刚入门量化的时候，从Tushare拉了一批日线数据，兴冲冲地跑了一个"低市盈率选股"策略的回测。结果年化收益率高达45%，最大回撤只有10%，完美得不像真的。我当时觉得"我的策略太牛逼了"。

后来在跟一个前辈交流的时候，他看了一眼我的代码，问我："你复权了吗？"我愣住了。回去一查——我拉的是不复权数据。分红送股导致的价格跳空，被系统当成了"大幅下跌"，然后"低市盈率"策略自动抄底，产生了大量虚假的低买高卖收益。

修正复权后，年化收益率从45%掉到了18%。虽然18%也不算差，但45%和18%之间差了一整个复权的认知差距。从那以后，我每次拉数据第一件事就是确认：复权方式对不对？这个习惯救了我不止一次。

23.3 财务数据的坑

财务数据比行情数据更复杂。这里面的"坑"如果没踩过，很难想象它有多深。

合并报表 vs 母公司报表

同一家公司会发布两种财务报表：合并报表（Consolidated）和母公司报表（Parent）。合并报表包含了母公司和所有控股子公司的财务数据，母公司报表只反映母公司本身的财务状况。在做投资分析时，应该使用合并报表。 因为母公司的营收和利润可能大量来自于子公司——只看母公司报表会严重低估公司的实际经营规模。

但有一个例外：很多财务比率（比如ROE）的计算口径不同，结果差异很大。同一个指标，用合并报表算和用母公司报表算，数值可以差好几个百分点。在使用任何金融数据库时，务必搞清楚它提供的财务数据是基于哪张报表。

会计准则差异

A股上市公司执行的是中国企业会计准则（CAS），与国际财务报告准则（IFRS）基本趋同但仍有差异。如果你同时研究A股和港股（尤其是AH两地上市公司），同一家公司按CAS和IFRS编制的财报数据可能会有明显差异——尤其是在资产减值、政府补助确认等项目上。

数据发布延迟

A股的财报披露有法定的时间要求：年报（年度报告）必须在次年4月底前披露；半年报（中期报告）在8月底前披露；季报（季度报告）在季度结束后1个月内披露。这意味着你看到的财务数据天然存在延迟——2025年4月30日你才能看到某公司2024年的年报，此时2025年已经过了三分之一。

在做回测时，必须考虑这个延迟。很多新手直接用历史财务数据做回测，但没考虑"数据发布的时间滞后"——实际上你在某个时间点根本拿不到这些数据。这会导致前视偏差（look-ahead bias），让回测结果虚高。

解决方法：做回测时，对财务数据做"滞后一期"（shift）处理。比如你在2025年5月1日回测选股，只能使用截至2024年年报及之前的数据。可以用Tushare的disclosure_date字段来精确匹配报表可用的时间点。

23.4 数据清洗实战

从API拉回来的原始数据，几乎从来不能直接用于策略开发。下面几个是最常见的问题：

缺失值处理——停牌的股票没有交易数据，新上市的股票历史数据不够长。常见方法：直接删除（如果缺失比例低）、用前一个交易日数据填充（fillna/ffill）、或者用行业平均值替代。

异常值检测——ST股票（特别处理）的价格行为异常，新股前几个交易日波动极大（注册制下前5天不设涨跌幅），涨跌停期间价格失真。这些都需要在数据预处理阶段剔除或标记。

数据对齐——不同频率的数据需要对齐时间戳。日线数据和财务数据的频率不同，分钟线和日线需要对齐。pandas的reindex()和align()方法可以解决大部分对齐问题。

import pandas as pd
import numpy as np

# 读取原始行情数据
df = pd.read_csv('stock_data.csv', parse_dates=['trade_date'])

# 1. 剔除ST股票（股票代码带ST标记）
df = df[~df['stock_name'].str.contains('ST|退市', na=False)]

# 2. 剔除新股上市前5天
df = df[df['days_listed'] >= 5]

# 3. 处理缺失值：向前填充
df = df.sort_values(['ts_code', 'trade_date'])
df['close'] = df.groupby('ts_code')['close'].ffill()

# 4. 异常值处理：剔除涨跌停日（价格失真）
df['pct_chg'] = df.groupby('ts_code')['close'].pct_change()
df = df[df['pct_chg'].abs() < 0.098]  # 保留涨幅小于9.8%的交易日

# 5. 时间戳对齐：对齐到月末（用于和财务数据合并）
monthly = df.set_index('trade_date').groupby('ts_code').resample('M').last()
monthly = monthly.reset_index()

print(f"清洗前：{len(df)} 行")
print(f"清洗后：{len(monthly)} 行")

23.5 构建自己的金融数据库

每次跑策略都要重新拉数据、清洗、处理——既浪费时间又容易出错。一个更好的做法是：建立自己的本地数据库。

推荐使用SQLite作为本地存储方案。SQLite是嵌入式数据库，不需要安装服务端，一个文件就是一个数据库，非常适合个人量化投资者。配合Python的sqlite3标准库，可以轻松实现数据的定期更新和查询。

数据更新的策略有两种：

  • 全量刷新：每次重新拉取全部数据，覆盖原有数据。简单粗暴，适合数据量小（比如只有日线数据）的场景。
  • 增量更新：只拉取新产生的数据（比如最近一个交易日的数据），追加到数据库中。效率高，适合数据量大或者有分钟线/高频数据的场景。

无论用哪种策略，都建议在数据入库后做一个自动化的数据质量检查：检查最近交易日的数据是否缺失、价格是否为负、成交量是否异常放大或缩小等。这些检查可以写成脚本每天运行一次，发现问题自动发送告警。

核心原则：数据是量化投资的原材料。原材料质量决定了最终产品的质量。花30%的时间在数据获取和清洗上，是一个量化研究员最值得的时间投入。别急着跑策略——先确保你的数据是对的。
