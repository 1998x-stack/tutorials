22.1 序列有自己的逻辑

文本、语音、股票价格、传感器读数——这些数据有一个共同点：顺序很重要。把"我爱你"三个字倒过来变成"你爱我"，意思完全不同。而CNN和全连接网络天生假设输入是"无序的"——它们看不见时间维度。

循环神经网络（RNN）就是为了处理序列数据而设计的。核心思想很简单：在处理序列的第t个元素时，RNN不仅看当前输入x_t，还"记住"之前处理过的一个状态h_{t-1}——这个"记忆"通过循环连接从一个时间步传到下一个。

22.2 从RNN到LSTM：解决"记忆"难题

基础RNN有一个致命问题：长程依赖。当序列很长时（比如一段500字的文本），前100个字的信息在反向传播中梯度逐渐消失，无法影响后续的决策。比如"我出生在法国...（中间400字）...所以我的母语是___"，模型需要记住开头"法国"这个线索来填空，但基本RNN做不到。

LSTM：门控机制

LSTM引入了三个"门"（Gate）来控制信息流：

  • 遗忘门：决定从旧记忆中丢弃什么信息。比如进入一个新段落时，上一段的信息可以忘掉
  • 输入门：决定什么新信息应该存入记忆。比如出现"法国"这个重要线索时，存入记忆
  • 输出门：决定基于当前的记忆输出什么。比如当前记忆有"法国"，输出"法语"的概率就大

GRU：LSTM的简化版

GRU把遗忘门和输入门合并成"更新门"，只有两个门。参数更少，训练更快，效果通常和LSTM相当。没有明确理由选LSTM而不是GRU——建议两个都试。

🔑 2026年的现实：虽然RNN/LSTM在理论上很重要，但在实际NLP任务中，Transformer已经完全取代了它们。Google在2017年发表了Attention is All You Need之后，序列建模的霸主地位就易主了。但理解LSTM仍然有价值——它让你理解"门控机制"和"长程记忆"的设计思想，这些思想在GRU和现代架构中得到了继承。

22.3 Seq2Seq与编码器-解码器架构

很多序列任务不是"输入一个序列，输出一个标签"，而是"输入一个序列，输出另一个序列"：机器翻译（中文→英文）、文本摘要（长文→短摘要）、语音识别（音频→文字）。这就是Seq2Seq架构的用武之地。

Seq2Seq由两部分组成：编码器（Encoder）把输入序列压缩成一个"上下文向量"，解码器（Decoder）从这个上下文向量出发，逐步生成输出序列。

Seq2Seq的致命弱点：编码器必须把整个输入序列压缩成一个固定大小的向量。当输入很长时，这个"瓶颈"会丢失大量信息。这就引出了下一节的核心概念——Attention。

22.4 Attention：让解码器"看到"整个输入

Attention机制的核心思想：解码器在生成每个输出词时，不只看最后一个编码器状态（上下文向量），而是动态关注输入序列中相关的部分。

比如翻译"I love machine learning"为"我爱机器学习"：生成"机器"时，模型关注"machine"；生成"学习"时，关注"learning"。Attention权重可视化出来非常直观——你可以看到模型在"看"哪里。

Attention的计算可以总结为三步：(1) 计算解码器当前状态和每个编码器状态的"相关性分数"；(2) Softmax归一化成注意力权重；(3) 用权重对编码器状态加权求和得到"上下文向量"。

import torch
import torch.nn as nn

class BahdanauAttention(nn.Module):
    def __init__(self, hidden_size):
        super().__init__()
        self.Wa = nn.Linear(hidden_size * 2, hidden_size)
        self.Va = nn.Linear(hidden_size, 1)

    def forward(self, decoder_hidden, encoder_outputs):
        # decoder_hidden: (batch, hidden)
        # encoder_outputs: (batch, seq_len, hidden)
        seq_len = encoder_outputs.size(1)
        decoder_hidden = decoder_hidden.unsqueeze(1).repeat(1, seq_len, 1)
        energy = torch.tanh(self.Wa(
            torch.cat((decoder_hidden, encoder_outputs), dim=2)
        ))
        attention_weights = torch.softmax(self.Va(energy).squeeze(2), dim=1)
        context = torch.bmm(attention_weights.unsqueeze(1), encoder_outputs)
        return context.squeeze(1), attention_weights
💡 Attention的直观理解：想象你在翻译一个长句子。你不需要记住整个句子——你在写每个词的时候，眼睛自然地去回看原文中相关的部分。Attention就是给模型装上了这个"回看"的能力。这也是为什么Transformer论文的标题是"Attention Is All You Need"——当你能随时回看任何位置，你就不需要RNN那个有限的"记忆"了。

22.5 从RNN到Transformer：时代的交接

2017年之后，Transformer以碾压之势取代了RNN/LSTM。为什么？三个核心优势：

  1. 并行化：RNN必须按顺序处理（t=1→t=2→t=3），Transformer可以同时处理整个序列
  2. 长程依赖：Attention机制直接建立任意两个位置之间的连接，不需要通过隐藏状态逐步传递
  3. 可扩展性：Transformer架构天然适合扩展——更大的模型、更多的数据、更好的效果

RNN/LSTM的时代正在过去，但它们的思想（循环、门控、编码器-解码器、Attention）已经成为了深度学习大厦的基石。理解了这些，你就能无缝过渡到第23章的Transformer和现代NLP。
