9.1 名字是"回归"，干的却是"分类"

逻辑回归（Logistic Regression）是机器学习命名的最大"骗局"之一——它名字里带"回归"，但它是做分类的。为什么叫这个名字？因为它的核心是在线性回归的输出上套了一个Sigmoid函数，把(-∞, +∞)的实数压缩到(0, 1)之间，变成一个概率。这个"用回归做分类"的思路影响了一代算法。

先讲个故事。我在银行做信用评分卡的时候，监管要求模型必须有"可解释性"——每拒绝一个贷款申请，你得能说出原因。深度神经网络做不到这一点，但逻辑回归可以。每个特征的系数就是它对最终评分的影响方向与大小。这就是为什么在金融、医疗这类强监管行业，逻辑回归至今没有被取代。

9.2 Sigmoid：从回归到概率的"魔法函数"

逻辑回归的核心操作：先算线性部分 z = w^T x + b，然后过一个Sigmoid：

P(y=1|x) = σ(z) = 1 / (1 + e^(-z))

Sigmoid有两个关键性质：(1) 输出在0到1之间，可以当作概率；(2) 在z=0附近接近线性，在两端饱和——这意味着当模型很确定时（z很大或很小），梯度几乎为零，学习停止。这也是为什么后来深度学习改用ReLU的原因之一。

9.3 交叉熵损失：专为概率输出设计

分类问题不能用MSE。为什么？因为MSE + Sigmoid会导致梯度消失——当预测非常错误时（比如实际是1，预测接近0），sigmoid的梯度几乎为0，导致模型更新极慢。交叉熵损失完美解决了这个问题：

L = -[y log(p) + (1-y) log(1-p)]

当实际y=1而预测p→0时，log(p)→-∞，损失→+∞——模型会被狠狠惩罚。而且交叉熵的梯度是p-y，预测和真实之间的差距直接驱动更新。

💡 直觉记忆：MSE适合输出是连续值的情况（回归），交叉熵适合输出是概率的情况（分类）。用MSE做分类有两个问题：(1) 梯度消失；(2) MSE对概率的惩罚是非凸的，可能陷入局部最优。所以分类问题老老实实用交叉熵。

9.4 从二分类到多分类

逻辑回归原生只支持二分类。做多分类有两种策略：

  • OvR（One-vs-Rest）：为每个类别训练一个二分类器，把这个类别当正例，其余全当负例。训练K个模型，取得分最高的那个。简单粗暴，类别多时计算量大
  • OvO（One-vs-One）：为每一对类别训练一个二分类器。K个类别要训练K(K-1)/2个模型。每个模型只关心两个类，但对噪声敏感
  • Softmax回归（Multinomial）：逻辑回归的直接推广，输出层有K个神经元，softmax归一化得到K个概率。一次训练一个模型。标准方案
from sklearn.linear_model import LogisticRegression

# 默认使用OvR（对于二分类自动退化）
lr_ovr = LogisticRegression(multi_class='ovr', max_iter=1000)
lr_ovr.fit(X_train, y_train)

# Multinomial使用Softmax，配合saga求解器
lr_softmax = LogisticRegression(multi_class='multinomial',
                                 solver='saga', max_iter=1000)
lr_softmax.fit(X_train, y_train)

# 获取每个类别的预测概率
probs = lr_softmax.predict_proba(X_test)  # shape: (n_samples, n_classes)

9.5 决策边界与阈值调优

逻辑回归输出的概率，默认≥0.5判为正类。但这个0.5不是金科玉律。

在癌症筛查场景中，你宁可让健康人多做一次检查（假阳性），也不愿意漏掉一个真正的患者（假阴性）。这时候应该降低阈值（比如降到0.3），让模型更容易预测"阳性"。在垃圾邮件过滤中，你宁可漏掉几封垃圾，也不愿意把重要邮件扔进垃圾箱。这时候应该提高阈值（比如升到0.7），让模型更保守。

如何选择最佳阈值？画Precision-Recall曲线或ROC曲线，根据业务需求选最合适的点。如果你的KPI是"最大化F1分数"，就在PR曲线上找F1最大的那个阈值。

9.6 概率校准

逻辑回归输出的"概率"真的能当概率用吗？默认情况下，Sigmoid输出的值确实可以被解释为概率（是well-calibrated的），但经过正则化后可能不够校准。如果你的业务需要准确的概率估计（如保险定价、广告竞价），建议做概率校准：

from sklearn.calibration import CalibratedClassifierCV

# Platt Scaling: 在原始模型输出上再训练一个Sigmoid校准器
calibrated = CalibratedClassifierCV(lr, method='sigmoid', cv=5)
calibrated.fit(X_train, y_train)
calibrated_probs = calibrated.predict_proba(X_test)
🔑 关键判断：如果你的问题只关心排序（比如推荐系统中的"这个比那个更相关"），不需要校准。如果你的业务需要解释"这个用户有73%的概率转化"，就需要校准。逻辑回归默认比随机森林和SVM更校准——这也是它在风险建模中受欢迎的原因之一。
