第二十章：离线强化学习

同学们好，欢迎来到第二十章。在之前的课程中，我们学过的所有强化学习算法都有一个共同的前提：agent 必须与环境在线交互，通过试错来收集数据、更新策略。无论是 Q-Learning、SAC 还是 PPO，都默认 agent 可以随时随地与环境互动。但在现实世界中，这个前提往往不成立。

想象一下：你是一个医疗 AI 工程师，想训练一个策略来优化病人的呼吸机参数设置。你能让 agent 在真实病人身上"试错"吗？绝对不能。或者你在做自动驾驶，你想让车在高速公路上通过随机探索来学习怎么开车——这同样离谱。这些场景需要的是一种全新的范式：离线强化学习（Offline Reinforcement Learning），也叫 Batch RL。它的核心设定是：给你一个固定的数据集，里面是别人已经收集好的 (s, a, r, s') 四元组，你要从这个数据集中直接学出一个策略，完全不再需要与环境交互。

20.1 在线 vs 离线：根本性的范式转变

我们先来理清这两个概念的本质区别。

在线强化学习（Online RL）中，agent 与环境的交互是一个持续循环的过程。Agent 在状态 s 下执行动作 a，环境返回下一个状态 s' 和奖励 r，agent 把这条经验存入回放缓冲区，然后从缓冲区采样来更新策略。注意：即使使用经验回放（如 DQN 和 SAC），数据也是由当前策略生成的，或者至少是最新版本的策略生成的。策略和数据之间存在着紧密的耦合关系。

离线强化学习（Offline RL）则截然不同。数据集 D = {(s_i, a_i, r_i, s'_i)} 是预先收集好的，可能来自于某个专家的演示、一个旧策略的日志、或者传感器记录。训练过程中 agent 完全不能和环境互动，只能从这个静态的数据集中反复学习。这听起来有点像监督学习——给一堆数据，学一个映射——但问题远没有那么简单。

为什么离线 RL 如此重要？因为它在以下场景中是唯一可行的选择：

  • 医疗： 从历史病历数据中学习最优治疗方案，不能在病人身上试错。
  • 自动驾驶： 从人类驾驶的日志数据中学习驾驶策略，不能在路上随机探索。
  • 机器人： 从人类演示数据中学习操作技能，避免每次训练都对硬件造成磨损。
  • 推荐系统： 从用户历史交互数据中学习推荐策略，避免用实时 A/B 测试去探索次优推荐。
直觉类比： Online RL 像是学游泳——跳进水里扑腾，喝几口水就会了。Offline RL 像是看游泳教学视频学游泳——你只能看别人的动作，然后自己琢磨。后者显然更难，但更安全，也适用于那些"不能下水"的场景。

离线 RL 面临的最大挑战，也是这一章的核心问题，叫做分布偏移（Distributional Shift）。我们接下来就深入分析这个问题。

20.2 分布偏移问题：为什么标准算法会失败

假设你有一个数据集 D，里面记录了状态 s 下的各种动作。当你用这个数据集来训练 Q 函数时，一个严重的问题出现了：你学到的策略 π 会选择那些在数据集中从未出现过的动作。

听起来不可思议？让我们一步一步推演：

标准的 Q-Learning 更新公式是 Q(s, a) ← Q(s, a) + α[r + γ maxa' Q(s', a') − Q(s, a)]。注意这里的 maxa' Q(s', a')——为了计算目标值，我们需要对 所有 可能的动作 a' 取最大值。但是数据集中可能只包含了少数几个动作。对于那些没有出现在数据集中的动作（out-of-distribution actions，简称 OOD actions），Q 函数没有任何训练信号，它的输出完全依赖于网络的外推（extrapolation）。

问题就出在这里。神经网络的外推是极其不可靠的。当在 OOD 动作上查询 Q 函数时，它往往给出一个异常大的正值——这就是所谓的外推误差（Extrapolation Error）。然后 max 操作会选中这个被高估的 OOD 动作，导致目标值被系统性抬高，误差通过 Bellman 更新不断自举放大。最终 Q 函数彻底崩掉，学出来的策略当然也是一团糟。

举个简单的例子：假设你在一个二维导航任务中收集了数据，agent 只在四个方向（上、下、左、右）上移动。数据集里干净整洁。现在你用 SAC 的离线版本去训练——SAC 的策略 π 会输出一个高斯分布，它可能采样出一个从未出现过的动作，比如"右上45度"。Q 网络从未见过这个动作，胡乱给了一个很高的估值。策略看到"哦，原来右上45度 Q 值这么高"，于是加大了这个方向的概率。下一次更新，Q 为了匹配策略的选择又进一步推高估值。这是一个灾难性的正反馈循环。

踩坑故事： 我在一次研究项目中试图用标准的 SAC 在离线数据上训练一个机器人开门策略。数据集是专家演示的，质量很高。我满怀信心地跑了训练，结果策略的回报曲线是这样的：前 10 轮一切正常，第 15 轮突然崩塌，第 20 轮之后完全无法恢复。我检查了 Q 值，发现从第 12 轮开始 Q 值开始异常飙升——从正常的 50 左右一路冲到 5000 多。这就是典型的外推误差失控。后来我才意识到，是策略开始采样到 OOD 动作后，Q 函数进入了一个"自欺欺人"的循环：Q 自我抬高 -> 策略相信 Q 并选择更极端的动作 -> Q 继续抬高。这个教训让我深刻理解了：在离线 RL 中，算法的失败不是缓慢退化，而是瞬间崩塌。解决办法就是接下来要讲的 CQL。
一句话总结： 离线 RL 的核心矛盾在于——策略想要探索更好的动作，但数据集之外的动作 Q 值不可信。所以离线 RL 算法的核心设计思路就是让策略"老实"一点，待在数据集的范围内。

20.3 保守Q学习（CQL）

CQL（Conservative Q-Learning）是离线 RL 领域最经典、最实用的算法之一，由 Sergey Levine 团队在 2020 年提出。它的核心思想极其简洁有力：压低 OOD 动作的 Q 值，同时抬升数据集中动作的 Q 值。

标准 Q-Learning 的损失函数是 TD 误差：

LTD = E(s,a,r,s')~D[(r + γ maxa' Q(s', a') − Q(s, a))2]

CQL 在这个基础上增加了一个正则化项：

LCQL = LTD + α (Es~D, a~μ[Q(s, a)] − Es,a~D[Q(s, a)])

其中 μ 是某个宽泛的动作分布（通常是均匀分布或当前策略的采样），α 是一个超参数。第一项 E[Q(s, a)] for a~μ 试图压低所有动作（包括 OOD 动作）的 Q 值，第二项 E[Q(s, a)] for (s,a)~D 试图抬升数据集中动作的 Q 值。这两项相互对抗，最终的效果是：对数据集中出现过的动作，Q 值被抬升到合理水平；对未出现过的动作，Q 值则被压低。策略在选择动作时自然就会倾向于数据集内的动作。

下面是 CQL 的 PyTorch 风格伪代码，展示了关键部分和标准 SAC 的区别：

# CQL 的 Q 函数更新（基于 SAC 框架）
def update_critic(batch, alpha, cql_alpha):
    s, a, r, s_prime, done = batch

    # ---------- 标准 TD 损失 ----------
    # 目标 Q 值（使用 target Q 网络）
    with torch.no_grad():
        a_prime = policy(s_prime)
        q_target = target_q(s_prime, a_prime)
        y = r + gamma * q_target * (~done)

    td_loss = F.mse_loss(q(s, a), y)

    # ---------- CQL 正则化项 ----------
    # 第1步: 从均匀分布采样动作，压低 Q 值
    rand_actions = uniform.sample([batch_size, action_dim],
                                   a_min, a_max)
    q_rand = q(s, rand_actions).logsumexp(dim=1).mean()

    # 第2步: 从当前策略采样动作，压低 Q 值
    current_actions = policy.sample(s)
    q_current = q(s, current_actions).logsumexp(dim=1).mean()

    # 第3步: 抬高数据集中动作的 Q 值
    q_data = q(s, a).mean()

    # CQL 损失：压低 OOD + 抬高 in-distribution
    cql_loss = cql_alpha * (q_rand + q_current - q_data)

    # 总损失
    total_loss = td_loss + cql_loss

    return total_loss
关于 cql_alpha 的选择： α 控制正则化的强度。α 太小 -> CQL 退化为标准 SAC，仍然会过估计 OOD 动作。α 太大 -> Q 值被过度压低，策略变得过于保守，几乎只做数据集中出现过的动作。实践中 α 通常在 0.5 ~ 5 之间调节。有些版本的 CQL 会使用自动调节的 α（像 SAC 自动调节温度系数那样），效果更加鲁棒。

CQL 的美妙之处在于它几乎不需要修改策略网络——只需要在 Q 函数的损失函数上加一项正则项。这使得 CQL 可以作为"插件"轻松集成到任何基于 Q 函数的离线 RL 算法中（如 SAC、DDPG、TD3 的离线版本）。

20.4 IQL 与 Decision Transformer：不同的路线

CQL 的思路是"让 Q 函数变得保守"。但这不是唯一的方法。这一节介绍两条完全不同的技术路线。

IQL（Implicit Q-Learning） 由 Hansen 等人在 2022 年提出。它的洞察非常巧妙：离线 RL 的困难来源之一是我们在计算 TD 目标时需要使用 maxa' Q(s', a')，而这个操作会查询 OOD 动作。IQL 的解决方案是：根本不去查询那些 OOD 动作。那没有 max 操作，怎么计算 TD 目标呢？IQL 使用了一种叫做 expectile regression 的统计技术。

简单解释：expectile regression 不是预测条件期望（conditional mean），而是预测条件分位数的某个推广。通过调节 expectile 参数 τ（通常在 0.7 ~ 0.9 之间），IQL 可以近似地估计 "V(s) = 数据集中的上分位数 Q 值"，而不需要显式地在 OOD 动作上取 max。这相当于隐式地做了一个"只使用数据集内好动作"的价值估计。IQL 在 D4RL 等标准离线 RL 基准上取得了非常强的结果，而且实现极其简洁——代码量只有 CQL 的 1/3。

Decision Transformer（决策 Transformer） 则走上了一条完全不同的道路。2021 年由 Chen 等人提出的这个算法，彻底抛弃了 Q 函数和 Bellman 方程，把强化学习重新定义为一个序列建模问题。

具体来说，Decision Transformer 的输入是一个长度为 K 的序列：

(&Rcirc;t−K+1, st−K+1, at−K+1, &Rcirc;t−K+2, st−K+2, at−K+2, ..., &Rcirc;t, st)

其中 &Rcirc;t 是待实现的回报（Return-to-Go），即从时间步 t 开始到回合结束的累计奖励。模型的任务是：给定当前的回报目标 &Rcirc;t 和状态 st，预测应该执行的动作 at。整个模型使用因果 Transformer（causal transformer）架构，用标准的自回归损失（交叉熵或 MSE）来训练。

使用 Decision Transformer 做离线 RL 的流程是：训练阶段，从数据集中采样序列片段，用 GPT 式的 next-token-prediction 来训练。推理阶段，设定一个期望的总回报（比如"我想拿到 500 分"），然后模型会自回归地生成动作序列来达到这个目标。

三种离线 RL 路线的对比：
  • CQL： 修改 Q 函数，加正则化项压 OOD 估值。基于值函数。是经典的 RL 思路。
  • IQL： 不去查询 OOD 动作，用 expectile regression 隐式估计最优值。介于值函数和无模型之间。
  • Decision Transformer： 彻底抛弃 RL 框架，把问题当作序列建模。基于 Transformer。需要大量的数据。

这三种方法各有优劣。CQL 最通用、最鲁棒，适合各种场景。IQL 实现简单、计算效率高，但在数据质量差时可能不够稳定。Decision Transformer 在大数据量下表现惊艳，但对数据量需求大，且对 Return-to-Go 的设定值敏感。

20.5 离线 RL 的实际应用

讲了这么多理论，离线 RL 到底在哪些地方真正落地了？我们来盘点几个关键领域。

推荐系统： 这是离线 RL 最成功的商业应用之一。推荐系统的核心问题是"在给定用户状态时，推荐什么物品能使长期用户留存最大化"。这天然是一个 RL 问题——但你不能拿真实用户来做在线探索（至少不能大规模地探索次优推荐）。离线 RL 可以用历史用户-物品交互日志来训练推荐策略，同时用保守策略（类似 CQL 的思路）来避免推荐那些从未被推荐过的物品。

机器人操作： 机器人领域积累了大量的演示数据——人类遥操作数据、已有的 MPC 控制器数据、仿真数据。这些数据通常质量参差不齐，但数量庞大。离线 RL 可以从这些混合数据中学习出一个统一的策略。Google 的 RT-1 和 RT-2 等机器人基础模型在底层就使用了离线 RL 的思想来从大规模机器人数据中学习。

医疗决策： 从电子病历中学习治疗策略是离线 RL 的经典应用场景。例如学习脓毒症患者的抗生素给药策略、呼吸机参数调节策略等。这些任务的共同点是：你不能在病人身上试错，但历史数据中蕴含了大量的治疗经验。

下面的表格总结了在线 RL、离线 RL 和基于模型的 RL（MBRL）三者的关键区别：

维度在线 RL离线 RL基于模型的 RL
数据来源当前策略自生成固定数据集，来源各异自生成 + 模型仿真
环境交互需要不需要部分需要（初始数据收集）
分布偏移不存在（数据与策略同步）核心挑战有模型误差导致的分布偏移
样本效率低（需要大量交互）高（复用已有数据）中高（用模型生成虚拟数据）
安全性低（探索可能有风险）高（完全离线训练）中（在模型内探索）
典型算法SAC, PPO, DQNCQL, IQL, DTDreamer, MuZero
最佳场景仿真环境、游戏医疗、自动驾驶、日志数据需要规划、数据效率优先
一个重要的提醒： 离线 RL 在实际部署时有一个几乎总是被忽视的问题——数据质量 > 数据数量。一个包含大量次优和随机行为的数据集，即使用再好的离线 RL 算法也很难学到好策略。相反，一个质量高但规模适中的专家演示数据集，配合合适的算法（如 IQL），往往能产出相当不错的策略。所以，如果你要做离线 RL 项目，请先把 80% 的精力花在数据收集和清洗上。

知识延伸：从离线到多智能体

到目前为止，我们的教程中讨论的场景都是单智能体——一个 agent 在一个环境中独自学习。离线 RL 的范式进一步缩小了交互范围：连环境都不需要了，只有一个静态的数据集。

但真实世界中的许多问题本质上都是多智能体系统：自动驾驶中多个车辆的协同、股票市场中多个交易者的博弈、电网调度中多个发电单元的协调。在多智能体场景下，每个 agent 面对的不仅仅是环境的动态变化，还有其他 agent 的策略也在不断变化。这带来了非平稳性（non-stationarity）问题——当我更新我的策略时，其他 agent 的策略可能也变了，导致我眼中的环境动态完全改变。

而离线多智能体 RL 则更加困难：数据集中包含了多个 agent 的交互记录，但这些 agent 的策略和协作方式可能与我们要训练的 agent 完全不同。如何从异质的多智能体交互数据中学习一个既能协作又能竞争的策略？这是当前非常活跃的研究前沿。

下一章我们将正式进入多智能体强化学习（Multi-Agent Reinforcement Learning, MARL）的世界。你会看到，当多个 agent 同时学习时，那些在单智能体场景下好用的算法（如 Q-Learning、策略梯度）会面临怎样的新挑战，以及 MARL 领域的经典算法（如 MADDPG、QMIX）是如何应对这些挑战的。

本章小结

  1. 离线强化学习 从固定的数据集中学习策略，不需要与环境交互。这在医疗、自动驾驶、机器人等高风险领域是必须的。
  2. 分布偏移 是离线 RL 的核心挑战。策略选择的 OOD 动作会导致 Q 函数外推误差，产生灾难性的正反馈循环。
  3. CQL 通过在 TD 损失上添加正则化项来压低 OOD 动作的 Q 值，同时抬升数据集中动作的 Q 值，实现简单、效果可靠。
  4. IQL 用 expectile regression 隐式估计最优 Q 值，完全不查询 OOD 动作；Decision Transformer 则彻底抛弃 Q 函数，将 RL 转化为序列建模问题。
  5. 离线 RL 在推荐系统、机器人、医疗等领域已有实际应用。数据质量始终比数据数量更重要。
