第3章 早期TTS：共振峰合成

大家好，我是你们的讲师。上一章我们打下了基础——语音如何从物理世界的连续声波变成计算机里的离散数字表示。现在，我们要正式开始TTS历史之旅了。第一站是1980年代，那是TTS技术从实验室走向实用化的起点。

想象一下，你身处1985年，你对着一个比冰箱还大的计算机说了一句话，而它用机器人般的声音回应了你——在个人电脑都还是新鲜事物的年代，这听起来就像科幻成真了。今天回顾这段历史，共振峰合成可能显得简陋，但你会发现，它的核心思想——源-滤波器模型——至今仍深刻地影响着TTS的设计哲学。

1980年代：语音合成的起点

在1980年代之前，语音合成更多的是机械的而非电子的。1770年，奥地利科学家冯·肯佩伦（von Kempelen）制造了一台可以用风箱和皮革腔体"说话"的机械装置。1939年，贝尔实验室展出了VODER（Voice Operating Demonstrator），一个需要受过专门训练的操作员用键盘和脚踏板实时控制的电子合成器——这是史上第一台电子语音合成器。

真正意义上的计算机语音合成，要到1980年代才成熟。这背后的关键推动力是源-滤波器模型的提出。

源-滤波器模型

源-滤波器模型（Source-Filter Model）是理解语音生成最经典的框架，由Gunnar Fant在1960年系统化提出。它的核心思想极其简洁：语音 = 声源 × 滤波器。

  • 声源（Source）：有两种类型。对于元音和浊辅音，声源是声带周期性振动产生的脉冲波（周期性声源），其基频F0决定音高。对于清辅音（如s、sh、f），声源是气流通过声道狭窄处产生的湍流噪声（噪声声源），没有周期性。
  • 滤波器（Filter）：声道（口腔、鼻腔、咽腔）的形状像一个由多个谐振腔串联而成的声学滤波器。不同的舌位、嘴型、软腭位置改变了谐振腔的形状，从而改变了滤波器的频率响应。这个滤波器的谐振峰值就是我们反复提到的共振峰（Formant）。

这个模型的优雅之处在于它将复杂的语音生成解耦成了两个独立的部分。改变声源（比如提高F0）只会改变音高，不会改变音色；改变滤波器（比如把舌头位置从/i/移到/a/）只会改变音色，不会影响基频。这种分离使控制变得极简单。

元音与共振峰的位置：不同的元音对应着不同的共振峰频率组合。以下是一些标准美式英语元音的F1和F2数值（单位Hz）：

元音示例单词F1 (Hz)F2 (Hz)感受
/i/bee2702290高前元音，舌头抬高靠前
/e/bet5301840中前元音
/a/father7301090低后元音，舌头压低靠后
/o/hot570840中后元音
/u/boot300870高后元音，舌头抬高靠后
这就是为什么语音学教科书里有个著名的"元音梯形图"——F1和F2的二维平面几乎完美地映射了舌位的高低前后。

Klatt合成器

在实际实现源-滤波器模型时，最著名的努力来自Dennis Klatt。这位MIT的科学家在1980年发表了Klattalk系统，它成为了后来所有共振峰合成器的基准。

Klatt合成器的核心是一个复杂的共振峰滤波器网络，采用级联（cascade）和并联（parallel）两种结构混合的方式。级联结构模拟声道中多个谐振腔的串联效应，适合合成元音；并联结构则独立控制每个共振峰的幅度，对合成辅音（特别是擦音和塞音爆发段）更灵活。

一个简化版的共振峰合成流程可以用伪代码表示如下：

# 共振峰合成的简化伪代码
        def synthesize_vowel(f0, f1, f2, f3, duration_ms):
            sr = 16000                      # 采样率
            n_samples = int(sr * duration_ms / 1000)

            # 1. 生成声门源：周期脉冲串（基频f0）
            glottal_source = impulse_train(f0, n_samples, sr)

            # 2. 设计声道滤波器：三个共振峰滤波器的级联
            def formant_filter(signal, f, bw):
                # 二阶谐振滤波器，中心频率f，带宽bw
                # 这里简化为理想的带通滤波器
                return bandpass(signal, f - bw/2, f + bw/2)

            # 3. 逐级滤波
            output = glottal_source
            output = formant_filter(output, f1, bw=80)   # 第一共振峰
            output = formant_filter(output, f2, bw=100)  # 第二共振峰
            output = formant_filter(output, f3, bw=120)  # 第三共振峰

            # 4. 辐射补偿：模拟嘴唇的声阻抗
            output = radiation_compensation(output)

            return output

        # 合成元音 /a/（如同"father"中的a）
        audio_a = synthesize_vowel(
            f0=120, f1=730, f2=1090, f3=2440, duration_ms=300
        )

当然，真实的Klatt合成器远比这个伪代码复杂——它需要处理声源切换（浊音/清音）、噪声源（摩擦音）、鼻化（鼻腔共振）、振幅包络、过渡音段（formant transitions，从一个音到另一个音的共振峰平滑移动）等各种细节。一个完整的发音规则（rule table）可能需要数百行参数定义。

有趣的是，Klatt合成器在当时的最高配置是DECtalk，它运行在一台DEC PDP-8小型计算机上。DECtalk将Klatt的算法固化在了一块专用硬件板上，通过串口与主机通信。这在1980年代绝对是一个工程奇迹。

DECtalk的传奇

DECtalk是共振峰合成时代最耀眼的明星。它于1984年由DEC（Digital Equipment Corporation）发布，以其流畅的英语语音合成能力，成为当时最受欢迎的TTS系统之一。但DECtalk最传奇的故事，与一位伟大的物理学家有关——史蒂芬·霍金（Stephen Hawking）。

霍金在1985年因肺炎接受了气管切开手术，从此失去了说话能力。在那之前，他还能用勉强可辨的语音与助手交流。手术后，他需要一个替代的通讯方式。最初，他的助手使用了一套由Words+公司开发的基于Apple II的系统，但霍金对合成的美国口音不太满意。

1988年，一位加州工程师为霍金的系统增加了DECtalk DTC01——DECtalk的PC版本。霍金立刻爱上了它平滑的英式发音（虽然DECtalk默认是美国口音，但可以通过参数调整）。从此，DECtalk的声音成为了霍金独一无二的身份标识。那句著名的"无论生活多么艰难，总有一些事情你可以做并且成功"——就是通过DECtalk传递到全世界的。

一个有趣的细节：霍金本可以选择更新、更自然的语音合成器，但他坚持使用DECtalk，即使DEC在1990年代衰落，这块硬件的维护和升级变得越来越困难。霍金曾在一次采访中说："我喜欢它独特的声音，它已经成了我的一部分。"他甚至拒绝了Intel为他定制的更先进的语音系统。直到霍金2018年去世，他在将近30年的时间里一直使用同一个DECtalk声音——这也让这个"机器人音"变得如此人性化，因为它承载了一个伟大灵魂的全部声音。

共振峰合成的优缺点

共振峰合成作为第一代TTS技术，它的特点和局限都非常鲜明：

优点：

  • 完全规则驱动：不需要任何录音数据，所有的参数都由规则和公式生成。这意味着它可以合成任何单词或句子——即使是字典里查不到的新词。
  • 高度可控制：每一个声学参数（F0、共振峰频率、带宽、幅度、噪声量、送气量）都可以独立控制。这为表达性合成（比如让声音听起来高兴或悲伤）提供了极大的灵活度。
  • 可理解性：虽然听起来"机器感"很强，但合成的语音通常非常清晰，在噪音环境下甚至比自然的语音更容易辨识。
  • 资源占用极低：不到几百KB的代码就可以运行，完全不需要存储语音数据。

缺点：

  • 难以自然：合成语音带有明显的"嗡嗡"电子感（buzziness），听起来永远不像真人。这是因为简化模型无法完美模拟声门波形的复杂性和声道的动态变化。
  • 规则编写工作量大：要合成自然流畅的语音，需要定义几万条发音规则和例外。英语有数不清的不规则拼法（"though" vs "through" vs "cough"），每条例外都需要手工处理。
  • 韵律僵硬：虽然可以控制参数，但要生成自然的语调和节奏变化极其困难。早期的合成语音读句子就像在念清单——每个词的音高、时长几乎一样。
初学者常见误解：很多人觉得共振峰合成"太老了，不值得学"。但事实上，理解源-滤波器模型对理解后面所有的TTS技术都至关重要。WaveNet虽然直接用神经网络生成波形，但它的架构本质上仍在学习源-滤波器的隐含表示。很多现代的神经声码器（如HiFi-GAN）仍然显式地使用F0作为条件输入。源-滤波器模型不是被"推翻"的，而是被"吸收"和"超越"的。

共振峰合成在1980年代到1990年代初是TTS的主流范式。但研究者们越来越清楚地认识到：依赖规则的合成系统有一条"自然度天花板"，因为人的语音实在太复杂、太细碎了——连音、弱化、口音、个人习惯，所有这些都无法用有限的规则覆盖。要突破这个天花板，需要换一种思路：不靠规则，靠数据。这就是我们下一章要讲的主题——拼接合成（Concatenative Synthesis）。
