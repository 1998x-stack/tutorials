第25章 调查方案设计

大家好，我是你们的讲师。前面 24 章我们一直在讲"方法"——各种抽样技术的原理、公式和代码实现。但从这一章开始，我们要切换到一个更宏观的视角：如何设计一个完整的调查方案。

你可能会觉得，不就是做个调查吗？问卷发出去、数据收回来、跑个分析就完事了。但实际中，一个调查项目的失败往往不是因为某个技术细节没做对，而是从一开始方案就没设计好。抽样框选错了、样本量不够、质量控制缺失——这些问题一旦在数据收集完成后被发现，几乎无法补救。

调查全流程

一个规范的抽样调查项目，通常包含以下八个阶段：

阶段 核心任务 输出物 常见陷阱
1. 定义研究问题 明确调查目标、目标总体、关键变量 研究方案书 目标定义模糊，导致后续设计偏差
2. 抽样设计 选择抽样方法、确定样本量、构建抽样框 抽样方案文档 抽样框覆盖不足（覆盖率偏差）
3. 问卷设计 编制问卷、预测试、信效度检验 正式问卷 问题措辞引导性过强
4. 数据收集 培训调查员、执行现场调查、监控应答率 原始数据 调查员操作不规范
5. 数据清理 逻辑校验、异常值处理、缺失值标识 清洁数据 未记录数据删减原因
6. 加权调整 基础权重、无应答调整、事后分层 加权数据 权重截尾处理不当
7. 统计分析 描述统计、推断统计、建模 分析结果 未考虑设计效应
8. 报告撰写 呈现结果、说明精度、讨论局限 调查报告 遗漏关键方法学信息

这个流程中，前三个阶段决定了调查的质量上限——后续所有工作都是在这个基础上进行的。如果抽样设计出了问题，再精细的加权和处理也无法完全补救。

抽样方案文档的核心要素

一个完整的抽样方案文档应该包含以下内容：

  • 目标总体（Target Population）：明确你要推断的对象是谁。例如"全国 18-65 岁常住居民"，要给出操作性定义。
  • 抽样框（Sampling Frame）：具体列出从中抽样的名单或地图。例如"全国 2846 个区县的社区居委会列表"。要说明抽样框的质量和潜在覆盖率问题。
  • 抽样方法（Sampling Method）：每一阶段的具体方法——分层变量是什么、PSU 是什么、是否采用 PPS 等。
  • 样本量（Sample Size）：总样本量和各层的分配。要说明计算依据（目标精度、预期 deff、预期应答率）。
  • 预期精度（Expected Precision）：在给定样本量下的预期标准误或置信区间宽度。
  • 质量控制（Quality Control）：现场操作规范、监督机制、数据审核流程。
文档规范建议：抽样方案文档需要像技术合同一样严谨。因为在实际调查中，执行团队和设计团队往往是不同的人。抽样方案如果写得模糊，执行阶段就会产生偏差。我在实践中要求每个抽样方案都要通过内部评审（peer review），重点关注抽样框的覆盖是否完整、样本量计算是否合理、以及 deff 的假设是否有依据。

质量控制：预防胜于补救

质量控制需要贯穿调查的始终。以下是几个关键控制点：

调查员培训

调查员是数据收集的"第一责任人"。培训内容应包括：抽样方案的理解、受访者接触技巧、问卷填写规范、伦理要求（知情同意、隐私保护）。培训结束后要有考核，不合格者不能上岗。

一个常见的操作是并行编码（parallel coding）——在调查初期，安排两位编码员独立对同一批问卷进行编码，然后比对一致性。早期发现编码问题可以及时纠正，避免批量错误。

应答率监控

应答率（Response Rate）是调查质量的核心指标之一。在数据收集过程中，需要实时监控各层各群的应答情况。如果某层的应答率明显偏低，需要及时采取措施——增加催访次数、调整调查时段、或者启动替补样本。

# 应答率监控的简易 Python 脚本
import pandas as pd
import numpy as np

def compute_response_rate(contact_log):
    """
    计算应答率及各阶段转化率

    根据 AAPOR 标准定义：
    RR1 = 完成访谈数 / (完成 + 拒访 + 无法联系 + 其他不合格)
    """
    attempts = len(contact_log)
    completed = (contact_log['status'] == 'completed').sum()
    refused = (contact_log['status'] == 'refused').sum()
    non_contact = (contact_log['status'] == 'no_contact').sum()
    ineligible = (contact_log['status'] == 'ineligible').sum()

    rr1 = completed / (completed + refused + non_contact + ineligible)
    contact_rate = (completed + refused) / (completed + refused + non_contact + ineligible)
    cooperation_rate = completed / (completed + refused)

    return {
        'RR1 (应答率)': rr1,
        'Contact Rate (接触率)': contact_rate,
        'Cooperation Rate (合作率)': cooperation_rate
    }

逻辑校验

数据收集完成后，需要进行系统性的逻辑校验。例如：年龄和婚姻状况是否矛盾（10 岁的人不可能已婚）、多选题的跳转逻辑是否正确、异常值是否合理。每一步校验的结果都应该记录，数据删减要有明确的规则和理由。

一个真实的教训：某省级政府开展了一次居民健康素养调查，采用分层多阶段抽样设计，涉及 24 个区县、10 万样本。项目组制定了完整的抽样方案，但质量控制环节被严重低估——预算中只有 2% 用于质控。调查员培训只用了半天，没有考核；数据收集期间没有实时监控应答率；逻辑校验被委托给第三方公司在两周内完成。

结果数据收上来后，分析团队发现某县的有效样本只有计划量的 40%，原因是该县的调查员在入户时遇到大量拒访后没有上报，而是自行用附近的便利样本替代。由于没有实时监控，这个问题直到数据清理阶段才被发现，而此时现场调查已经结束，无法补救。最终该县的估计值不得不依赖模型推断，精度大幅下降。

调查方案的检查清单

为了帮助大家在实际工作中不遗漏关键环节，我整理了一份调查方案的检查清单。每次启动调查项目前，逐项核对：

类别 检查项 完成状态
研究设计 研究目标是否有明确的、可操作的定义？ ☐
目标总体是否有清晰的时间、空间和特征限定？ ☐
关键变量是否有公认的定义和测量方式？ ☐
抽样设计 抽样框是否完整覆盖目标总体？有无覆盖不足或过覆盖？ ☐
抽样方法的选择是否有充分理由？ ☐
样本量计算是否考虑了设计效应和预期应答率？ ☐
是否有替补样本方案（如果抽到的单元无法调查）？ ☐
质量控制 调查员培训是否有标准化流程和考核？ ☐
数据收集期间是否有应答率实时监控机制？ ☐
是否有独立的质量审核团队（非执行团队兼任）？ ☐
分析与报告 是否已确定加权方案和方差估计方法？ ☐
是否计划报告应答率、设计效应和加权方法？ ☐
是否考虑了数据开放和可复现性（代码、数据文档）？ ☐

调查报告中必须报告的内容

一个好的调查报告，不仅要说"我们发现了什么"，还要说"我们是怎么发现的"。美国 AAPOR（民意研究协会）和联合国的统计部门都有明确的报告标准。以下是必须报告的内容：

  • 应答率（Response Rate）——按 AAPOR 标准定义计算，最好报告各阶段的应答率分解
  • 抽样方法——每一阶段的具体方法、分层变量、抽样框描述
  • 加权方法——基础权重的计算方式、无应答调整的方法、事后分层的变量和来源
  • 设计效应——关键估计量的 deff 值，或者平均设计效应
  • 标准误估计方法——Taylor 线性化法、BRR、Jackknife、Bootstrap 等
  • 缺失数据处理——缺失率、缺失机制假设、处理方法
行规铁律：不报告应答率的调查结果是不完整的。不报告设计效应的推断统计是有误导性的。不报告加权方法的分析是不可复现的。如果你要发表基于调查数据的论文，方法学部分的详细程度决定了审稿人是否会质疑你的结论。我见过太多被拒稿的论文，归根结底不是分析有问题，而是方法的报告不够透明。

这一章我们从调查的全流程出发，讨论了方案设计、质量控制和报告规范。你会发现，前面 24 章学到的每一种抽样方法，最终都要融入到这套流程中去。下一章我们将讨论一个非常前沿的话题——大数据时代的抽样。当数据量不再是瓶颈，抽样技术是否还有用武之地？带着这个问题进入第 26 章。
