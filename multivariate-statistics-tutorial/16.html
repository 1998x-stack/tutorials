开篇：当系统聚类遇上大数据

大家好，欢迎继续学习。上一章我们学习了系统聚类法——它很直观、能生成漂亮的谱系图、不需要预先指定簇的个数。但如果你手头的数据有一百万条，系统聚类法的 O(n²) 或 O(n³) 复杂度会让你等到天荒地老。更麻烦的是，系统聚类一旦合并就不能回头——前方遇到更好的分组方式，它也无法"纠正"。

这一章我们学习另一大类聚类方法——动态聚类法 (Partitional Clustering)。它的核心思想是：先把所有样本一次性分配到 K 个组中，然后通过反复迭代优化分配结果。这类方法速度快（O(n) 量级）、可扩展性好，是大规模聚类的主力军。其中最著名的算法，就是几乎无人不知的 K-means。

除了 K-means，我们还会讨论它的变体 K-medoids（更稳健）和一种完全不同的密度聚类方法 DBSCAN（能够识别任意形状的簇）。到本章结束时，你会对不同聚类方法的适用场景有一个清晰的判断力。

K-means算法

K-means 的思想极为简洁：给定 K 个簇的中心（均值点），每个样本被分配到离它最近的中心所在的簇；然后用簇内所有样本的均值更新中心点；重复这两个步骤直到收敛。

算法：K-means 聚类

输入：数据矩阵 X (n × p)，簇数 K
输出：每个样本的簇标签

1. 从数据中随机选择 K 个样本作为初始簇中心
   μ₁, μ₂, ..., μₖ
2. 重复以下步骤直到收敛：
   a. 分配步骤 (Assignment)：
      对每个样本 xᵢ，计算它与所有簇中心的距离
      将 xᵢ 分配到距离最近的簇：
      cᵢ = argmin_k ||xᵢ - μₖ||²
   b. 更新步骤 (Update)：
      对每个簇 k，用簇内所有样本的均值更新中心：
      μₖ = (1/|Cₖ|) Σ_{xᵢ∈Cₖ} xᵢ
3. 当簇分配不再变化（或变化小于阈值）时停止

K-means 的目标函数 (objective function) 是组内离差平方和（within-cluster sum of squares, WCSS），也叫 inertia：

# K-means 最小化的目标
J = Σ_{k=1}^{K} Σ_{xᵢ∈Cₖ} ||xᵢ - μₖ||²

# 这个目标函数衡量的是：
# 每个样本到其所属簇中心的距离平方之和
# K-means 的每次迭代都保证 J 不增（单调收敛）

注意 K-means 只能保证收敛到局部最优，而非全局最优。初始中心的选择对最终结果影响巨大。这也是为什么实际使用 K-means 时，我们通常要跑多次（比如 10-50 次）并选择目标函数最小的一次的结果。

K-means 的优缺点速览：优点是简单、快速（O(n·K·d·iter)）、适合大规模数据。缺点是必须预先指定 K、对异常值敏感、只能发现球形簇、对初始中心敏感。后面的几种方法正是为了克服这些缺点而设计的。

初始中心选择

初始中心对 K-means 的影响再怎么强调也不为过。看看几种常见的策略：

随机初始化

最简单的方法：从数据中随机选取 K 个样本作为初始中心。问题是——如果随机选的几个中心碰巧靠得很近，聚类结果就会很糟糕。实践中多跑几次（multiple starts），选目标函数最小的那次。

K-means++

这是目前最主流的初始化方法。它的思路很巧妙：第一个中心随机选，后续每个中心以概率正比于距离已有中心最近距离的平方来选择。这样选出的中心相互之间足够分散，避免了"中心扎堆"的问题。

算法：K-means++ 初始化

1. 从数据中均匀随机选取第一个中心 μ₁
2. 对于 k = 2 到 K：
   a. 对每个样本 xᵢ，计算 D(xᵢ) = min_{j<k} ||xᵢ - μⱼ||²
   b. 以概率 P(xᵢ) = D(xᵢ) / Σⱼ D(xⱼ) 选择下一个中心 μₖ
3. 用选出的 K 个中心运行标准 K-means

# 这样选出的初始中心：越是远离已有中心，越可能被选中

scikit-learn 的 K-means 默认使用 K-means++ 初始化，这也是为什么你直接用 sklearn 通常能得到不错的结果。

种子敏感性

即使有了 K-means++，不同的随机种子仍然可能导致不同的结果。所以最佳实践是：设置 random_seed，固定一组随机数运行一次 K-means，记录结果。换几个种子重复此过程，选择目标函数最小的结果。scikit-learn 中通过 n_init 参数控制重复次数，默认是 10。

确定K值

K-means 最大的痛点就是需要预先指定 K。如果事先不知道数据应该分成几类怎么办？以下是几种常用方法：

肘部法则 (Elbow Method)

这是最直观的方法。对不同的 K 值分别运行 K-means，计算 WCSS（组内离差平方和），然后绘制 K - WCSS 曲线。随着 K 增加，WCSS 单调递减。当曲线出现一个"拐点"（elbow）时，拐点对应的 K 就是比较合理的簇数。

为什么肘部会出现？因为增加 K 总是能使每个簇更紧凑（WCSS 更小），但当 K 超过数据真正的分组数后，继续增加 K 带来的 WCSS 下降幅度会明显变小——因为你在把已经足够紧凑的群体硬拆开来。

轮廓系数 (Silhouette Coefficient)

轮廓系数同时考虑了簇内紧致度和簇间分离度。它的计算步骤如下：

  • 对每个样本 xᵢ，计算 a(i) = 到同簇其他样本的平均距离（簇内平均距离）
  • 计算 b(i) = 到其他簇中样本的平均距离的最小值（最近邻簇的平均距离）
  • 样本 i 的轮廓系数: s(i) = (b(i) - a(i)) / max(a(i), b(i))
  • 整体轮廓系数是所有样本 s(i) 的平均值
# 轮廓系数的含义
# s(i) 的范围：[-1, 1]
# s(i) ≈ 1  → 样本被很好地聚类（远离其他簇）
# s(i) ≈ 0  → 样本位于两个簇的边界上
# s(i) < 0  → 样本可能被分错了簇

# 整体轮廓系数最大的 K 是最优选择

其他方法

方法原理适用场景
Gap统计量比较实际数据的 WCSS 与均匀分布数据的期望 WCSS 的差距理论严谨，计算量大
CH指数Calinski-Harabasz指数 = (BSS/(K-1)) / (WSS/(n-K))类内紧密、类间分离时值大
Davies-Bouldin指数簇内散射与簇间距离的比值之和值越小越好
实践建议：不要只依赖一个指标来确定 K。至少同时看肘部法则和轮廓系数——如果两者指向同一个 K 值，那就很有信心了。如果它们不一致（这很常见），就需要结合领域知识来做决定。另外要注意：K 的选择本质上是"分辨率"的选择——同一个数据集，你既可以从市场细分的角度分成 3 类（高/中/低价值），也可以从更精细的角度分成 8 类。没有绝对的"正确"K 值。

K-medoids (PAM)

K-medoids 是 K-means 的一个稳健变体。两者的核心区别在于："簇中心"的定义不同。

  • K-means 的中心是簇内样本的均值 (mean)——这个均值点可能不在原始数据中，而且均值对异常值非常敏感
  • K-medoids 的中心是簇内的一个实际样本 (medoid)——也就是簇中到其他所有样本距离之和最小的那个样本

最常用的 K-medoids 实现是 PAM 算法 (Partitioning Around Medoids)：

算法：PAM (K-medoids)

1. 随机选择 K 个样本作为初始 medoids
2. 将每个样本分配到离它最近的 medoid
3. 对每个 medoid m，考虑每个非 medoid 样本 o：
   - 交换 m 和 o，重新分配样本
   - 如果总损失降低，则执行交换
4. 重复步骤 3 直到不再有交换发生

# 计算量比 K-means 大（O(K(n-K)²)）
# 但结果更稳健，且可以使用任意距离度量

K-medoids 的优势：一是对异常点不敏感——因为中心是一个"真实存在"的样本，不会被极端值拉偏；二是可以用任意距离度量（而不仅仅是欧氏距离），比如可以用曼哈顿距离来处理城市距离问题，用Gower距离来处理混合类型数据。

什么时候用 K-medoids 代替 K-means？当数据中存在明显的异常值，或者你的业务需求要求"簇中心是一个真实的、可解释的样本"时（比如在推荐系统中，一个 medoid 用户就是你推荐策略的"典型用户画像"）。

DBSCAN

K-means 和 K-medoids 都属于"基于中心点"的聚类方法，它们有一个根本性局限：只能发现近似球形的簇。如果数据中有半月形、螺旋形或其他复杂形状的簇，这些方法会惨不忍睹。

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) 采用了完全不同的思路：用"密度"而不是"距离到中心"来定义簇。它的核心概念只有两个参数：

  • eps (ε)：邻域半径
  • minPts：一个点被视为"核心点"所需的最小邻居数（包括自身）

DBSCAN 将样本分为三类：

  • 核心点 (core point)：在其 ε 邻域内至少有 minPts 个点
  • 边界点 (border point)：在某个核心点的 ε 邻域内，但自身不是核心点
  • 噪声点 (noise point)：既不是核心点也不是边界点——相当于 DBSCAN 的异常值
算法：DBSCAN

输入：数据 X，参数 eps，minPts

1. 对每个点 p：
   - 找出其 eps 邻域内的所有点 N(p)
   - 如果 |N(p)| ≥ minPts，p 标记为核心点
2. 将所有相互可达（通过核心点链连接）的核心点归为同一个簇
3. 将边界点分配到它所属的核心点所在的簇
4. 噪声点不分配簇标签

# 关键性质：
# - 不需要预先指定簇数 K
# - 能发现任意形状的簇
# - 能识别噪声点
# - 对参数 eps 和 minPts 敏感

DBSCAN 在空间数据分析、地理信息系统 (GIS)、异常检测等领域非常受欢迎。但它也有短处：当数据密度变化很大时（不同区域的密度差异悬殊），很难找到一组统一的 eps 和 minPts 参数来适配所有区域。

聚类结果可视化

聚类分析的结果可视化通常分两步走：

第一步：降维投影。用 PCA 或 t-SNE 将高维数据降到二维，然后用散点图展示，每个点按聚类标签着色。这能给你一个直观的"簇的形状和分布"的视觉印象。但要注意：降维后的可视化会损失信息，看起来分离良好的簇在原始空间中不一定真的可分。

第二步：簇轮廓分析 (Cluster Profiles)。对每个簇，计算各维度的均值和标准差，然后用并行坐标图 (parallel coordinates) 或雷达图 (radar chart) 展示各簇的"特征画像"。这对于业务理解至关重要——比如市场细分中，你不仅要告诉老板"分成了 5 类"，还要说清楚"第 1 类是高消费低频次的'品质买家'，第 2 类是低消费高频次的'薅羊毛党'"。

可视化检查清单：
  1. PCA 投影图：看簇是否在二维空间中有明显分离
  2. t-SNE 投影图：看局部结构（注意 t-SNE 的距离没有意义）
  3. 并行坐标图：看各维度的簇间差异
  4. 簇的统计摘要表格：各簇大小、各维度均值

踩坑故事：球形假设的陷阱

多年前我在做用户行为模式分析的项目。数据是用户在 App 内的行为序列经过 PCA 降维后的二维数据——散点图看起来很漂亮，有两个明显分离的集群，中间还有一些零散的点。我二话不说跑了一个 K-means (K=2)，结果令人沮丧：K-means 把两个集群各切了一半，组合在一起，而不是把两个真实的集群分开。

问题出在哪里？仔细看散点图——两个集群的分布并不是球形的。其中一个像是拉长的香蕉形，另一个是紧凑的圆形。K-means 假设所有簇都是球形的，它强行把香蕉形分成了两半，把其中一半和圆形合并在一起，以达到"组内离差平方和最小"的目标。

教训：在使用 K-means 之前，一定要先可视化数据，检查簇的形状假设是否合理。如果你看到细长或弯曲的簇形状，K-means 无能为力，应该改用 DBSCAN。后来我改用 DBSCAN（eps=0.3, minPts=5），两个集群被完美地分开，中间的那些点也正确地被标记为噪声——它们其实是一批测试账号，本来就该被排除在分析之外。记住一句口诀："K-means 假设你的簇是球形，DBSCAN 不在乎你的簇是什么形状。"

本章小结与衔接

在这一章中，我们学习了三种动态聚类方法：K-means（快速、简单、必须预设 K 和球形假设），K-medoids（更稳健、可以处理异常值和任意距离度量），以及 DBSCAN（密度聚类、不需要预设 K、可以处理任意形状并识别噪声）。我们还讨论了确定 K 值的常用方法——肘部法则、轮廓系数、Gap 统计量和 CH 指数——以及聚类结果的可视化策略。

至此，关于聚类分析的内容就讲完了。我们用了两章的篇幅覆盖了聚类的主要方法论——从层次方法到划分方法再到密度方法。在接下来的两章中，我们将转入另一个核心主题：判别分析 (Discriminant Analysis)。注意区分：聚类是无监督的（不知道标签硬要找分组），而判别分析是有监督的——我们有带标签的训练数据，目标是从特征中学习分类规则。下一章我们从最基础的距离判别和贝叶斯判别开始。
