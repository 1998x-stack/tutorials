开篇：一座迷宫和一把剪刀

大家好，我是你们的讲师。这一章要讲的分支定界（Branch and Bound），是我在组合优化中最钟爱的算法之一。让我用一个亲身经历来开场。

2016年，我还在读硕士时，帮导师做一个"物流枢纽选址"的项目。问题描述：有50个候选仓库位置，需要在其中选5个建仓，使得覆盖所有配送需求点的总成本最小。我当时想到的解法是穷举——C(50,5) ≈ 211万种组合。不算太离谱，但问题是每次组合都要解一个线性规划来算成本，211万次 LP 求解，每次0.1秒也要……211万秒 ≈ 24天。

我觉得肯定有更聪明的方法。导师给了我一篇关于分支定界（Branch and Bound, B&B）的论文，说："试试这个。"

我花了一周实现了基本的 B&B 框架，结果让我震惊：同样的 50选5 问题，B&B 只探索了 2,847 个分支就找到了最优解——用时 47 秒。相比穷举的 24 天，效率提升了超过 40,000 倍。

这个结果让我第一次深刻体会到："智能枚举"的力量远远超过"暴力枚举"。分支定界的核心思想就这么简单：与其枚举所有可能，不如边枚举边判断——如果某个分支不可能产生比当前已知最优解更好的结果，就果断剪掉它，不再往下探索。

分支定界的直觉

想象你在走一座巨大的迷宫，你的目标是找到最短的出口路径。穷举法是把每条路都走到底再折返——如果迷宫有 100 个岔路口，你需要走 2¹⁰⁰ 条路。分支定界法呢？它会做这样一件事：每走到一个岔路口，先大致判断一下"这个方向走下去最少要走多远"。如果这个"最少距离"已经大于你目前找到的最短路径了，直接放弃这个方向。

这就是分支定界的精髓：用"估计"代替"走到底"。关键在于，这个"估计"必须是下界（Lower Bound）——真实的距离不可能比这个估计更短。这样你才能放心剪枝。

B&B 之所以如此重要，是因为它是几乎所有商业 MIP 求解器（Gurobi, CPLEX, SCIP）的核心引擎。当你用求解器求解一个整数规划问题时，它在幕后运行的就是 B&B 算法（加上各种增强技术）。

B&B 的宏观比喻：想象你在管理一个庞大的"搜索帝国"。帝国太大了，你不可能亲自去每个人的家。于是你派"特使"（bounding）去每个省快速评估：如果某个省的人均 GDP 上限都低于帝国的平均值，那整个省都可以不用考察了。这就是分支定界——用低成本的信息来排除大量无用的搜索空间。

B&B 三要素

分支定界由三个基本操作组成：分支（Branching）、定界（Bounding）、剪枝（Pruning）。

1. 分支（Branching）

分支是将当前问题的可行域划分为两个或多个子区域的过程。每个子区域对应一个子问题（Subproblem）。

在整数规划中，最常见的分支方式是选一个取值非整数的变量 xⱼ，创建两个子问题：

  • 左分支：添加约束 xⱼ ≤ ⌊xⱼ⌋
  • 右分支：添加约束 xⱼ ≥ ⌈xⱼ⌉

例如，在 LP 松弛解中 x₁ = 3.7，我们就分支为 x₁ ≤ 3 和 x₁ ≥ 4。这样保证了所有子问题的并集覆盖了原问题的全部可行整数解，且子问题之间不重叠。

2. 定界（Bounding）

定界是为每个子问题计算一个下界（Lower Bound, LB）（对于最小化问题）。这个下界告诉我们：这个子问题中的最优解至少是多少。

计算下界最常用的方法是：

  • LP 松弛：去掉整数约束，求解 LP。LP 的最优值一定是原问题最优值的下界（因为可行域扩大了）。
  • 组合松弛：用更简单的组合结构替代原问题。例如 TSP 的下界可以用最小生成树来估计（关于 MST 下界后面会详细讲）。
  • Lagrange 松弛：把复杂约束移到目标函数中，通过乘子惩罚违反约束的程度。

3. 剪枝（Pruning）

当某个子问题的下界 ≥ 当前全局上界（Global Upper Bound, 即目前已找到的最佳可行解的目标值）时，这个子问题可以被剪枝——因为即使在这个子问题中找到最好的解，也不可能优于我们已经找到的解。

剪枝有三种情况：

  • 基于界剪枝：LB ≥ 当前全局上界（最常用）
  • 不可行剪枝：子问题无可行解（如 LP 松弛无解）
  • 整数解剪枝：子问题恰好找到了整数可行解——更新全局上界，然后剪枝
一个常见误解：很多人以为 B&B 保证找到最优解是因为它"搜索得更聪明"。其实 B&B 在最坏情况下仍然需要检查所有分支——仍然是 O(2ⁿ)。它的实际效率完全取决于定界的质量：如果下界很紧（接近真实最优值），剪枝效率就高；如果下界很松（离真实值很远），几乎剪不掉任何分支，B&B 就退化成了穷举。所以 B&B 的"艺术"全在于怎么构造一个好的定界器（Bounder）。

下界的构造

我们来深入探讨三种常用的下界构造方法，这是 B&B 成功的"秘密武器"。

LP 松弛

对于整数线性规划（ILP），最简单的下界就是去掉整数约束，求解对应的 LP（LP Relaxation）。LP 的可行域包含了 ILP 的所有可行解，所以 LP 最优值 ≤ ILP 最优值（对于最小化问题）。LP 松弛的质量取决于整数约束的"紧密度"——有些问题的 LP 松弛解天然就是整数（如网络流问题），这意味着下界就是最优解；而另一些问题（如 TSP）的 LP 松弛解非常"松"，需要额外技术来加强。

// 分支定界算法框架（最小化问题）
function BranchAndBound(problem):
    初始化：全局上界 UB = ∞
    初始化：活动节点列表 = {problem}

    while 活动节点列表不为空:
        选择一个节点（子问题）进行探索
        求解该子问题的松弛问题（如 LP 松弛）
        LB = 松弛问题的最优值

        if LB ≥ UB:
            剪枝（不可产生更好的解）
        else if 松弛解是整数可行解:
            UB = min(UB, LB)
            剪枝（更新全局上界）
        else:
            选择一个分数变量 xⱼ
            创建两个子节点：
                左分支：xⱼ ≤ floor(xⱼ)
                右分支：xⱼ ≥ ceil(xⱼ)
            将子节点加入活动节点列表

    return UB

组合松弛：MST 下界用于 TSP

TSP 要求找一条经过所有城市恰好一次并回到起点的最短环。如果我们去掉"哈密顿环"的条件，改成"找一个连通所有城市且总权最小的结构"，那就变成了最小生成树（MST）问题。

对于 TSP，MST 的总权一定 ≤ TSP 的最短环总长。为什么？因为 TSP 的解（哈密顿环）删掉任意一条边就是一个生成树，所以 TSP 环的权是 ≥ MST 权的。因此，MST 的总权是 TSP 的一个下界。

更紧的组合松弛包括1-树（1-Tree）和最小权完美匹配。Christofides 算法（TSP 的 1.5-近似算法）就利用了完美匹配来构造下界。

Lagrange 松弛

Lagrange 松弛（Lagrangian Relaxation）是一种更高级的定界技术。对于带有"难处理"约束的问题，我们把这些约束移到目标函数中，通过 Lagrange 乘子 λ 来惩罚违反约束的程度：

原始问题：
min cᵀx
s.t. Ax ≤ b    ← 难约束
     x ∈ X      ← 简单约束（如整数约束）

Lagrange 松弛：
min cᵀx + λᵀ(Ax - b)   ← 把难约束"买"进目标
s.t. x ∈ X
λ ≥ 0  ← Lagrange 乘子（用户设定）

对于固定的 λ ≥ 0，Lagrange 问题的最优值是原问题的一个下界。
我们想要找"最好的"下界——即 max_{λ ≥ 0} L(λ)。
这叫做 Lagrange 对偶问题。
最优的 λ* 可以通过次梯度法（Subgradient Method）找到。

Lagrange 松弛的优势在于：去掉难约束后，剩下的子问题往往可以用简单高效的方法求解（如动态规划、网络流）。它的下界质量通常比 LP 松弛要好——在有些问题中甚至是最优的。

定界方法 计算代价 下界质量 适用场景
LP 松弛 低（多项式时间） 中等，取决于建模 通用，所有 ILP
组合松弛 很低（如 MST O(E log V)） 因问题而异 有已知组合结构的问题
Lagrange 松弛 中等（需选取 λ 迭代） 通常很好，可逼近最优 复杂约束去除后子问题可解
一个需要警惕的陷阱：不是所有的松弛都能给出有用的下界。我见过有人用"线性规划松弛"解一个有向图上的路径优化问题——结果 LP 松弛给出的下界是 0（因为所有分数值的组合可以"作弊"）。下界太松和没有下界一样糟糕——它让 B&B 退化成裸枚举。选择松弛方法时，需要在"计算速度"和"下界紧度"之间做权衡。实践中，Lagrange 松弛往往是精度和速度的最佳平衡。

搜索策略

B&B 中的"分支"生成了一个树形结构（搜索树），我们需要决定以什么顺序探索这棵树。不同的搜索策略对内存和求解速度有显著影响。

深度优先搜索（DFS）

策略：沿着一条分支一直往下走到底，然后回溯到最近的有未探索分支的节点。

优点：

  • 内存使用极小（只保存当前路径上的节点）。
  • 很快能找到一个可行解（到达叶子节点），从而建立初始全局上界。

缺点：

  • 可能陷入一个"不好的"分支中浪费大量时间。
  • 找到的上界质量可能不高，导致剪枝效率降低。

最佳优先搜索（Best-First Search）

策略：每次从活动节点中选择下界最小（对于最小化问题）的节点优先探索。

优点：

  • 理论上能最小化探索的节点总数。
  • 倾向于快速收敛到最优解。

缺点：

  • 需要维护一个优先队列，内存占用可能很大。
  • 可能很晚才找到第一个可行解（如果最优解在一个很深的分支中）。

广度优先搜索（BFS）

策略：一层一层地探索，同一层的节点按某种顺序处理。

在实际的 MIP 求解器中极少使用——它的内存消耗太大，且收敛速度很慢。

混合策略（实际求解器的做法）

实际求解器（Gurobi, CPLEX）使用的 B&B 不是任何一种纯策略，而是混合策略：

  1. 第一阶段：用 DFS 快速找到一个好的可行解，建立初始上界。
  2. 第二阶段：切换到最佳优先，用紧密的下界来高效剪枝。
  3. 节点选择：综合使用"最佳下界"和"最近节点回溯"的加权评分策略。
求解器内部的秘密：现代 MIP 求解器的 B&B 远比教科书描述的复杂。除了节点选择策略外，它们还会做：预求解（Presolve）简化问题、生成割平面（Cutting Planes）收紧下界、原始启发式（Primal Heuristics）快速找可行解、强分支（Strong Branching）选择更好的分支变量……所有这些技术的叠加，才让 Gurobi 能解决数百万变量的问题。但万变不离其宗——核心永远是 B&B 三要素。

Gap 与收敛

在 B&B 的求解过程中，有两个关键量始终在变化：

  • Primal Bound（又称 Upper Bound, UB）：当前找到的最佳可行解的目标值。这是"我们实际能做到的最好"。随着搜索的进行，UB 不断下降（对于最小化问题）。
  • Dual Bound（又称 Lower Bound, LB）：当前所有活动节点下界的最小值。这是"最优解至少是多少"的估计。随着搜索的进行，LB 逐渐上升。

当 UB = LB 时，B&B 搜索完成，找到了全局最优解。但实践中，求解器往往不等 UB = LB 就提前终止——通过 Optimality Gap 来判断：

Optimality Gap = (UB - LB) / UB × 100%

解读这个日志：

  • 开始时 Gap = 97.5%——上界（9000）很糟糕，下界（227）很松。
  • 随着搜索推进，Incumbent（当前最佳可行解，即 UB）从 9000 下降到 423.4567，而 BestBd（全局对偶界，即 LB）从 227 上升到 423.4567。
  • 当 Gap 下降到 0% 时，UB = LB，最优解确认。
  • 注意：Gap 下降到 0.1% 时（只差 0.1% 就能证明最优），求解器已经花了 15 分钟。而最后的 0.1% 花了 2 分钟——因为证明最优往往比找到最优解更难（需要将所有剩余节点的下界都提升到超过 UB）。

在实际应用中，很多用户会设置一个允许的 Gap 阈值（例如 MIPGap = 1%）。如果求解器能证明当前解与最优解相差不到 1%，就提前终止。这往往可以节省大量时间——因为 B&B 的最后几个百分点通常花在"证明最优性"上，而不是"找更好的解"上。

实践建议：如果你用 Gurobi 或 CPLEX 求解 MIP，建议：
(1) 设置适度的 MIPGap（如 1% 或 0.5%），不要要求严格的 0%——除非你确实需要最优解；
(2) 监控求解器日志中的 Gap 变化——如果 Gap 长时间停滞不降，说明定界器遇到了瓶颈，可能需要重新审视模型或添加割平面；
(3) 设置时间限制和节点限制作为"安全阀"，避免求解器无限运行。
