📡 第5章 多址技术

大家好，我是你们的讲师。想象一下这样的场景：一个演唱会场馆里，5万个观众同时想用手机发朋友圈、刷短视频、上传合影——但基站只有几个，频谱只有那么一段。基站如何区分这5万个用户？如何保证每个用户的数据不互相干扰？

这就是多址技术要解决的核心问题：让多个用户共享同一段频谱资源，同时保证各自的数据能被正确区分和接收。多址技术是移动通信从1G到5G每一代升级中都会"大做文章"的关键技术——它直接决定了系统的容量和效率。

5.1 多址技术的本质

多址接入的本质是一个资源划分问题。频谱资源是一个多维空间——频率、时间、码字、空间位置都是我们可以利用的"维度"。理想的多址方案应该让多个用户在同一时刻、同一频率上同时传输而互不干扰。但物理世界是有约束的——我们必须在这些维度之间做取舍。

打个比方：频谱就像一个"通信广场"，多址技术就是如何在广场上安排不同的人聊天而不吵到彼此。你可以给他们安排不同的房间（FDMA——分频）、不同的时段（TDMA——分时）、说不同的语言（CDMA——分码）、或者站在不同的角落（SDMA——空分）。

5.2 多址技术演进对比

每一代移动通信的核心标志之一就是其多址方式。下面的表清晰地展示了这个演进过程：

代际 多址方式 资源划分维度 典型系统 优点 局限
1G FDMA 频率 AMPS、TACS 实现简单 频谱利用率低
2G TDMA/FDMA 时间+频率 GSM、IS-136 数字语音质量好 数据速率低
3G CDMA 码字 WCDMA、CDMA2000 软容量、抗干扰强 码间干扰、呼吸效应
4G OFDMA 时间+频率 LTE/LTE-A 频谱效率高、灵活调度 PAPR较高
5G OFDMA（增强） 时间+频率+空间 5G NR 超灵活、Massive MIMO 调度复杂度高
6G（候选） NOMA / OFDMA+ 功率域+多维度 待定 更高过载接入 干扰消除复杂度高

从表中可以清晰地看到，多址技术的发展是一个在更多维度上"精细化"划分资源的过程。每一代都试图在更小的粒度上、更灵活地分配资源给用户。

5.3 CDMA：用"语言"区分用户

CDMA（码分多址）是3G时代的标志性技术。它的核心思想非常巧妙：所有用户同时在同一频率上发送数据，但每个用户使用一个独一无二的扩频码（Spreading Code）来"编码"自己的信号。接收端用相同的扩频码去"解码"，就能把属于该用户的信号从混合信号中"提取"出来。

下面是一个简化的CDMA扩频过程示意：

CDMA扩频原理（简化示例，假设扩频因子=8）：

用户A的数据比特:    1            0
用户A的扩频码:    [1 0 1 1 0 0 1 0]
编码后:
  比特"1" → 扩频码本身:   [1, 0, 1, 1, 0, 0, 1, 0]
  比特"0" → 扩频码取反:   [0, 1, 0, 0, 1, 1, 0, 1]

用户B的数据比特:    0            1
用户B的扩频码:    [1 1 0 1 0 1 0 0]
编码后:
  比特"0" → 扩频码取反:   [0, 0, 1, 0, 1, 0, 1, 1]
  比特"1" → 扩频码本身:   [1, 1, 0, 1, 0, 1, 0, 0]

空中传输的是两个用户编码后的序列叠加（相加）：
  时刻:         t1 t2 t3 t4 t5 t6 t7 t8
  A的信号:      [1, 0, 1, 1, 0, 0, 1, 0]
  B的信号:      [0, 0, 1, 0, 1, 0, 1, 1]
  叠加（空中信号）:[1, 0, 2, 1, 1, 0, 2, 1]

接收端解码用户A：
  收到信号:     [1, 0, 2, 1, 1, 0, 2, 1]
  乘以A的码:    [1,-1, 1, 1,-1,-1, 1,-1]（双极性化后）
  逐位相乘:     [1, 0, 2, 1,-1, 0, 2,-1]
  求和:          1+0+2+1+(-1)+0+2+(-1) = 4
  ➡ 结果>0，判为"1" ✓

（注意：这只是一个概念性示例，实际CDMA系统使用正交或
准正交码，且包含功率控制、Rake接收机等复杂机制。）

CDMA的"鸡尾酒会效应"：理解CDMA最简单的方式是类比鸡尾酒会。在一个嘈杂的酒会上，每个人都在同时说话（所有用户同时发送）。但如果你只听某个特定的人——你可以"锁定"他的声音特征（相当于扩频码），把他从其他人的喧闹声中"分离"出来。其他人虽然也在说话，但他们的话对你来说只是"背景噪声"。CDMA中，其他用户的信号对你来说就是"多用户干扰"（MAI，Multiple Access Interference）。

CDMA的一个显著特点是软容量——不像FDMA/TDMA有硬性的用户数上限，CDMA中增加一个用户只是让所有人的"背景噪声"稍微大一点。但当用户数过多时，干扰会大到系统崩溃——这就是所谓的"呼吸效应"。

5.4 OFDMA：4G/5G的基石

OFDMA（正交频分多址）是OFDM和多址技术的结合。它将整个宽带划分为大量正交的子载波，然后在时间-频率二维网格上为不同用户分配不同的资源块（RB，Resource Block）。

OFDMA相比CDMA的优势明显：

  • 频域调度：基站可以针对每个用户的信道质量（在不同子载波上的差异），把用户调度到信道条件最好的子载波上——这叫"频选调度增益"。
  • 正交性：不同用户的子载波之间是正交的，理论上不存在用户间干扰（不像CDMA有MAI）。
  • 灵活带宽：通过聚合不同数量的子载波，OFDMA可以支持1.4MHz到400MHz的信道带宽——这就是5G NR中"灵活频谱"的底层能力。
  • MIMO天然适配：OFDMA的频域结构天然适合MIMO信道估计和处理。

在5G NR中，OFDMA的调度粒度非常精细：

5G NR OFDMA资源调度（FDD案例）：

时域：一个帧(10ms) = 10个子帧
      一个子帧(1ms) = 2^μ 个时隙（μ取决于子载波间隔）
      一个时隙 = 14个OFDM符号

频域：一个资源块(RB) = 12个子载波
      子载波间隔可配置：15/30/60/120 kHz
      最大信道带宽：400 MHz（毫米波场景）

频选调度示例：
  用户A（靠近基站，LOS）：分配高MCS的资源块 → 256QAM
  用户B（小区边缘，NLOS）：分配低MCS的资源块 → QPSK
  用户C（高速移动）：分配子载波间隔大的资源块 → 30kHz SCS

基站每1ms（一个子帧）做一次资源分配决策——这就是
"动态调度"的核心，也是5G低时延的关键支撑。

工程视角：OFDMA的调度器是5G基站L2（MAC层）中最复杂的模块之一。它需要在每个TTI（传输时间间隔，最短0.125ms）内，为每个用户决定：分配多少RB、在哪个频域位置、使用什么MCS、几层MIMO传输——这是一个动态规划问题。华为的5G基站调度器据称经过了数万小时的优化迭代，才能在0.1ms级别做出接近最优的调度决策。

5.5 NOMA：6G的候选者

NOMA（非正交多址）是面向6G的一种候选多址技术。它的核心思路是"拥抱干扰"：允许不同用户在同一时频资源上非正交地叠加传输，然后在接收端通过SIC（串行干扰消除）来分离用户。

NOMA的核心是功率域复用：不同用户分配不同的发射功率——信道条件好的用户分配小功率（因为它的解码能力强），信道条件差的用户分配大功率（保证它能正常解调）。接收端通过SIC先解码大功率用户，减去其信号后再解码小功率用户。

NOMA相比OFDMA的优势：

  • 更高的频谱效率：特别是在小区边缘用户和中心用户共存的场景下
  • 更大的连接数：适用于mMTC（海量机器类通信）场景
  • 更低的调度开销：免调度传输（Grant-free transmission）

不过，NOMA的产业化还面临不少挑战：SIC接收机的复杂度较高、功率控制的精度要求苛刻、实际增益在真实信道下可能打折扣。截至2026年，3GPP Release 19中已将NOMA列为研究项目，业界预计它可能在6G标准中占有一席之地。

5.6 踩坑故事：CDMA的"呼吸效应"

这不是一个具体的单事故，而是CDMA系统一个广为人知的工程痛点——"呼吸效应"（Cell Breathing）。

2008年左右，某沿海城市部署了3G WCDMA网络。在白天正常时段，网络运行稳定。但到了晚高峰（晚上7-10点），市中心几个热门商圈的基站频繁出现一个现象：终端的发射功率被功控算法不断推高，但仍然无法建立连接；已经接入的用户通话质量急剧下降；基站的覆盖半径仿佛在"收缩"——原本在1公里外能正常通话的用户，忽然发现没有信号了，就像基站"喘了一口气"缩小了覆盖范围。

根因分析：CDMA是自干扰系统——每个用户都对其他用户造成干扰。晚高峰用户大量涌入，干扰水平急剧上升。为了保证已有用户的基本通话质量，基站通过功率控制指令迫使终端提高发射功率——但功率一高，干扰更大，形成"正反馈"恶化循环。最终基站的噪声抬升（RoT，Rise over Thermal）超过了设计门限，部分远端用户直接被"挤出"覆盖区。

解决方案包括：增加该区域基站密度（减小每个扇区的用户数）、优化小区间切换参数、在一些极端场景下限制同时接入的用户数量。到了4G OFDMA时代，因为用户间干扰被正交性消除了，"呼吸效应"基本成为历史——但这个故事时刻提醒我们：任何系统设计都要考虑极端负载场景，不能只看"典型工况"。

好了，这一章我们比较了FDMA、TDMA、CDMA、OFDMA和NOMA几种多址技术——从频谱划分的"物理分割"到码字的"数学分割"，再到功率域的"非正交叠加"，多址技术走过了从"硬隔离"到"软共享"的演进之路。下一章，我们将进入一个更偏向数学但极其重要的领域——信道编码与信息论基础。它将回答一个根本性问题：在有噪声的信道上，我们到底能多可靠地传输信息？
