第24章 训练流水线搭建

大家好，我是你们的讲师。欢迎回到YOLO教程。

第一次训练YOLO时，我遇到的最奇怪的Bug是loss突然变成NaN——查了两天才发现是学习率太大导致梯度爆炸。这种坑，你迟早也会遇到。训练一个检测模型看起来就是一行命令的事，但真正搭建一条稳定、可复现的训练流水线，里面藏着无数细节。

这一章，我们从环境配置讲起，带你走通完整的训练流程：安装Ultralytics、准备数据集YAML、执行训练命令、理解关键超参数的含义、使用监控工具追踪训练状态、处理常见训练问题。学完这一章，你就能自信地跑出自己的第一个YOLO模型了。

环境搭建

训练YOLO最主流的方式是使用 Ultralytics 官方包。它统一了 YOLOv5/v8/v9/v10/v11/v12 的训练和推理接口，一行命令就能切换模型版本。

# 安装 Ultralytics
pip install ultralytics

# 验证安装
yolo <TAB>  # 应该能看到自动补全提示

# 或者用 Python 验证
python -c "from ultralytics import YOLO; print('安装成功')"

Ultralytics 的核心依赖如下表所示：

依赖 版本要求（2026年推荐） 作用
Python 3.10 ~ 3.12 运行时环境
PyTorch 2.4+ 深度学习框架
torchvision 0.19+ 图像处理与数据增强
CUDA 12.4+（推荐12.8） GPU加速支持
onnxruntime-gpu 1.18+ ONNX导出与推理
opencv-python 4.9+ 图像读写与预处理
常见环境问题：
  • CUDA 版本不匹配：安装前务必用 nvidia-smi 查看驱动支持的 CUDA 版本，选择对应的 PyTorch 版本安装
  • 显存不足：如果报 CUDA out of memory，尝试降低 batch size 或使用更小的模型（yolov8n.pt 而不是 yolov8x.pt）
  • libGL 报错：OpenCV 在 Linux 纯净容器中缺少 libGL.so.1，运行 apt install libgl1-mesa-glx 解决

数据集 YAML 配置

在第23章我们已经看过 YAML 文件的基本结构。这里我们深入一下，看看 Ultralytics 怎么使用这个配置来组织训练流程。

# dataset.yaml
path: /home/user/datasets/coco128  # 数据集根目录
train: images/train                 # 训练集路径（相对于path）
val: images/val                     # 验证集路径
test: images/test                   # 测试集路径（可选）

# 类别定义
nc: 80                              # 类别总数
names:
  0: person
  1: bicycle
  2: car
  # ... 总共80个类别

YAML 文件中的路径可以是绝对路径也可以是相对于 path 的相对路径。训练时，Ultralytics 会自动拼接路径找到图像文件，然后根据图像文件名在对应的 labels 目录中查找标注文件。

例如训练集中的 images/train/0001.jpg，程序会自动寻找 labels/train/0001.txt。这个映射关系是自动完成的，你不需要在 YAML 中显式指定 labels 目录——Ultralytics 默认 labels 目录就是 images 目录的兄弟目录，只是将 images 替换为 labels。

训练命令详解

一切准备就绪后，训练就简单了——只需要一行命令：

# 基础训练命令
yolo train model=yolov8n.pt data=coco128.yaml epochs=100 imgsz=640 batch=16

这行命令的含义：

  • model=yolov8n.pt — 选择预训练模型权重文件。如果指定为 yolov8n.yaml 则从头训练（不加载预训练权重）
  • data=coco128.yaml — 数据集配置文件
  • epochs=100 — 训练轮数
  • imgsz=640 — 输入图像尺寸（训练时会自动 resize）
  • batch=16 — 批次大小（根据显存调整）

也可以在 Python 脚本中控制训练流程：

from ultralytics import YOLO

# 加载预训练模型
model = YOLO("yolov8n.pt")

# 开始训练
results = model.train(
    data="coco128.yaml",
    epochs=100,
    imgsz=640,
    batch=16,
    device="cuda:0",       # 指定 GPU
    workers=8,              # 数据加载线程数
    lr0=0.01,              # 初始学习率
    optimizer="AdamW",      # 优化器
    amp=True,               # 混合精度训练
    project="my_project",   # 项目名称
    name="experiment_01",   # 实验名称
)

# 训练完成后自动保存最佳权重到 runs/detect/experiment_01/weights/best.pt

关键超参数解读

理解超参数的含义，是你从"调参侠"进阶到"炼丹师"的必经之路。下面是我认为最重要的几个超参数：

参数 默认值 推荐范围 说明
lr0 0.01 0.001 ~ 0.1 初始学习率。过大导致梯度爆炸（NaN loss），过小收敛极慢
lrf 0.01 0.001 ~ 0.1 最终学习率 = lr0 * lrf（配合余弦退火调度）
momentum 0.937 0.8 ~ 0.98 SGD 动量，越大越平滑但可能震荡
weight_decay 0.0005 0 ~ 0.001 权重衰减（L2正则化）。防止过拟合
warmup_epochs 3 0 ~ 10 学习率预热轮数，从0逐渐上升到lr0，防止初期震荡
batch 16 取决于显存 批次大小。越大梯度越稳定，但显存需求线性增长
patience 50 10 ~ 100 早停耐心值：连续patience轮val loss不下降则停止
超参数调优经验法则：
  • 学习率是最重要的超参数，没有之一。如果只调一个参数，就调 lr0
  • batch size 翻倍时，学习率也应该适当增加（线性缩放法则）
  • 小模型（n/s）可以用稍大的学习率，大模型（l/x）需要更小的学习率
  • 如果 loss 震荡严重，增大 batch 或降低 lr
  • 对小型数据集（少于1000张），加大 weight_decay 和 dropout 防止过拟合

TensorBoard 集成

# 训练时启用 TensorBoard 日志
yolo train model=yolov8n.pt data=coco128.yaml project=my_project tensorboard=True

# 启动 TensorBoard 查看
tensorboard --logdir my_project

Wandb 集成

Weights & Biases 是业界使用最广的实验追踪平台。Ultralytics 原生支持：

# 安装 wandb 并登录
pip install wandb
wandb login

# 训练时自动同步到 wandb
yolo train model=yolov8n.pt data=coco128.yaml project=my_project

# 或显式指定
yolo train model=yolov8n.pt data=coco128.yaml project=my_project
             wandb=True wandb_project="yolo-training" wandb_name="exp-01"

在 Wandb 面板上你可以实时查看：loss 曲线、mAP 曲线、学习率变化、GPU 利用率、以及每个 epoch 结束后模型的验证集预测样例。

断点续训与微调

训练到一半意外中断是最令人崩溃的事情之一。好在 Ultralytics 默认就支持断点续训：

# 从上次保存的 checkpoint 继续训练
yolo train model=runs/detect/exp1/weights/last.pt data=coco128.yaml epochs=100

# 注意：这里的 epochs 是总轮数，不是剩余轮数
# 如果之前训了 30 轮，设 epochs=100 会再训 70 轮

微调（fine-tuning）是迁移学习最常用的技巧——在预训练权重基础上，针对你的特定数据集微调：

# 冻结 backbone 的前 10 层，只训练 head
from ultralytics import YOLO

model = YOLO("yolov8n.pt")

# 冻结指定层
for i, (name, param) in enumerate(model.model.named_parameters()):
    if i < 10:  # 冻结前10层
        param.requires_grad = False

results = model.train(data="my_dataset.yaml", epochs=50, lr0=0.001)
微调的最佳实践：如果你只有少量数据（100-500张），建议冻结整个 backbone，只微调检测头（head），并且使用较小的学习率（lr0=0.001）。如果你的数据与预训练数据集（COCO）差异很大（比如医学影像），则建议不冻结，从头开始训练 backbone，但使用中等学习率。

常见训练问题排查

这是我过去几年训练 YOLO 时遇到过的经典问题，我直接给你解决方案：

现象 原因 解决方案
Loss 为 NaN 学习率过大 / 梯度爆炸 降低 lr0（尝试 0.001），开启梯度裁剪 clip=1.0
Loss 不下降 学习率太小 / 数据问题 增大 lr0，检查标注是否正确，检查数据增强是否过强
mAP 一直为 0 类别ID不匹配 / 标注格式错误 用第23章的验证脚本检查标注，确保类别ID从0开始连续
显存溢出 batch 太大 / 图像太大 降低 batch、imgsz、或用 device=0 指定单卡
训练集 mAP 高但验证集低 过拟合 增加 weight_decay、减少 epochs、增加数据增强
GPU 利用率低（<50%） 数据加载瓶颈 增加 workers（8-16），检查存储IO速度，启用 prefetch
踩坑故事：有一次我训练一个工业缺陷检测模型，loss 一直在 2.0 附近震荡，mAP 徘徊在 0.1。我花了三天调整超参数、换模型结构，都没有改善。最后迫不得已检查了数据——发现标注人员把所有缺陷类别都标成了同一个 class_id（猜测是 CVAT 导出时配置错误）。修正标注后，同一组超参数直接跑到了 mAP 0.89。这个故事告诉我们：训练出问题，永远最先怀疑数据。

多 GPU 训练

当你数据量很大或者模型很大时，单卡训练太慢。Ultralytics 支持 DataParallel（单机多卡）和 DDP（分布式数据并行）：

# 单机多卡训练（DataParallel）
yolo train model=yolov8n.pt data=coco128.yaml device=0,1,2,3 batch=64

# DDP 训练（推荐）
yolo train model=yolov8n.pt data=coco128.yaml device=0,1,2,3 batch=64 ddp=True

多卡训练时 total batch size = batch × GPU 数量。例如设 batch=64 device=0,1,2,3，每张卡的批大小为 16。学习率也要相应地按线性缩放法则调整——批次翻几倍，学习率就翻几倍。
