第十一章：策略梯度方法

同学们好！从本章开始，我们将迈入强化学习中最具影响力的一个分支——策略梯度方法（Policy Gradient Methods）。前面几章我们花了大量篇幅讨论值函数方法：DQN学习Q值，然后用 ε-贪婪策略选择动作。这种"先学值，再导策略"的间接路线虽然有效，但有没有想过——我们能不能直接优化策略本身？

策略梯度方法给出的回答是：能，而且应该。它在许多场景下比值函数方法更自然、更强大，也是现代强化学习算法（PPO、SAC、DDPG等）的基石。

11.1 为什么直接优化策略

值函数方法（如DQN）走的是"曲线救国"路线：先估计Q值，再通过 ε-贪婪从Q值导出策略。这条路线存在几个根本性的问题：

  • 连续动作空间无能为力： 求 π(s) = argmaxa Q(s,a) 需要对所有动作计算Q值。在离散动作空间里这还好（遍历一遍就行），但在连续动作空间（比如机器人关节角度），你根本没法遍历。策略梯度直接在连续空间上输出概率分布，顺滑解决。
  • 确定性策略可能不是最优： ε-贪婪本质上是一个带探索噪声的确定性策略。但在某些随机环境中，最优策略本身就是随机的。比如在"剪刀石头布"中，任何确定性策略都会被对手利用；你只能以1/3的概率随机出拳才是最优。
  • 值函数的小误差可能引发策略的大变化： argmax 是一个离散决策——两个动作的Q值相差0.01和0.02，选的动作完全可能不同。Q值估计的微小噪声就可能导致策略剧烈抖动。

策略梯度方法直接对参数化策略 πθ(a|s) 进行优化，用梯度上升最大化期望累积奖励。它的目标函数很简单：

策略梯度核心思想：
定义目标 J(θ) = E[Σ γt Rt | πθ]，然后计算 ∇θJ(θ) 并沿着梯度方向更新 θ。

这就像你在爬山——每一步都朝当前最陡的方向走，最终到达山顶。只不过这里的"山"是参数空间，而"坡度"需要从与环境交互的经验中估计。

11.2 策略梯度定理

那么问题来了：J(θ) 涉及整个轨迹的期望，它的梯度怎么算？直接把期望求导是不行的——期望中的采样过程（智能体根据策略选择动作）本身依赖于参数 θ，而采样操作是不连续的。

这就是策略梯度定理（Policy Gradient Theorem）大显身手的地方。它给出了一个令人惊讶的简洁结果：

策略梯度定理：

∇θJ(θ) = Es~dπ, a~π [ ∇θ log πθ(a|s) · Qπ(s,a) ]

其中 dπ 是在策略 π 下的稳态分布。

这个公式为什么成立？关键在于一个叫做对数导数技巧（Log-Derivative Trick）的数学变换：

∇θ πθ(a|s) = πθ(a|s) · ∇θ log πθ(a|s)

这个等式来自对数求导法则：∇ log f = (∇ f) / f，两边乘以 f 即得。它巧妙地解决了"采样不可导"的问题——把对概率的梯度转化成了对数概率的期望，而期望可以通过采样来近似。

再乘上 Qπ(s,a) 就得到了梯度：朝着能增加高回报动作概率的方向更新参数。直观上理解：如果一个动作(s,a)能带来很多回报(Q很大)，我们就让 πθ(a|s) 变大（即增加这个动作被选中的概率）；反之就减小它。

理解策略梯度定理的直觉：

想象你在训练一个机器人投篮。每次投篮后，你根据结果调整姿势：投中了就记住这个姿势，下次更容易复制；没投中就调整姿势避免再犯。策略梯度定理做的正是这件事——它用 Qπ(s,a) 告诉每个动作"这次干得怎么样"，然后用 ∇ log π 指出"怎么调整参数能让这个动作更容易被选到"。两者相乘就是"以正确方向调整参数"。

11.3 REINFORCE算法

REINFORCE算法（Williams, 1992）是最简单、最经典的策略梯度算法。它的核心思想是：既然策略梯度定理需要 Qπ(s,a)，而我们没有真实的Q函数，那就用蒙特卡洛回报 Gt 来近似它。

REINFORCE的更新公式：

REINFORCE 算法伪代码：

对于每个回合:
    根据 πθ 生成完整轨迹 S0, A0, R1, S1, A1, ..., ST
    对每一步 t = 0, 1, ..., T-1:
        G = Σk=t+1T γk-t-1 Rk   (计算折扣回报)
        θ ← θ + α · G · ∇θ log πθ(At|St)
    结束
结束

每一步的意思就是："刚才做的那个动作，看在它最终拿到了G这么多回报的份上，把它的对数概率朝G的方向调整"。

在具体实现中，如果你用神经网络来表示策略，∇θ log πθ(At|St) 就是交叉熵损失函数对网络参数的梯度——你只需要在PyTorch中计算损失函数 -log π(a|s) * G，然后反向传播即可。

让我们看一个在CartPole环境上的简单实现片段：

import torch
import torch.nn as nn
import torch.optim as optim

class PolicyNetwork(nn.Module):
    def __init__(self, obs_dim, act_dim):
        super().__init__()
        self.fc = nn.Sequential(
            nn.Linear(obs_dim, 128),
            nn.ReLU(),
            nn.Linear(128, act_dim),
            nn.Softmax(dim=-1)
        )

    def forward(self, x):
        return self.fc(x)

# 训练循环核心部分
policy = PolicyNetwork(4, 2)
optimizer = optim.Adam(policy.parameters(), lr=0.01)

for episode in range(1000):
    log_probs, rewards = [], []
    state = env.reset()
    done = False
    while not done:
        probs = policy(torch.FloatTensor(state))
        dist = torch.distributions.Categorical(probs)
        action = dist.sample()
        log_probs.append(dist.log_prob(action))
        state, reward, done, _ = env.step(action.item())
        rewards.append(reward)

    # 计算折扣回报
    G = 0
    returns = []
    for r in reversed(rewards):
        G = r + 0.99 * G
        returns.insert(0, G)

    # 策略梯度更新
    loss = []
    for log_prob, G in zip(log_probs, returns):
        loss.append(-log_prob * G)  # 梯度上升 → 最小化负的
    optimizer.zero_grad()
    torch.cat(loss).sum().backward()
    optimizer.step()

这段代码展示了REINFORCE最纯粹的形式——收集整个回合的数据，计算每个时间步的累积回报G，然后用 -log_prob * G 作为损失进行梯度下降（因为PyTorch做的是梯度下降，所以取负号把梯度上升转过来）。

踩坑故事：G的方差之痛

在我刚开始学REINFORCE的时候，兴致勃勃地把上面的代码跑在CartPole上。第一次跑——收敛了！我心想"就这？策略梯度也不过如此嘛"。然后我换了一个稍微复杂点的环境（LunarLander），结果无论怎么调学习率，智能体就跟醉了似的，在着陆平台旁边疯狂抖动，就是学不会稳定的着陆策略。

我花了两天时间排查。一开始以为是网络结构问题，换了大网络——更抖了；换成小网络——根本不学。以为是学习率问题，从1e-2试到1e-5——没一个管用。

最后让我恍然大悟的是一个实验：我让算法在同一个环境下跑10次，每次的G值画出来——差距大到离谱！有一次G=-800，同参数下一次运行G=+50。这就是REINFORCE的"原罪"——方差爆炸。MC回报G的方差随轨迹长度指数增长，一次幸运的探索可能让梯度步长巨大，一次不幸的探索又让参数朝反方向猛冲。说白了，REINFORCE就像是喝醉了酒走钢丝——步子迈得太大，随时会掉下去。

教训：如果你在复杂环境中发现REINFORCE不收敛，不要先怀疑你的代码有bug（虽然也很有可能），先想想方差问题。下一节讲的基线技巧（Baseline）就是你最需要的解药。

11.4 基线技巧（Baseline）

为了缓解REINFORCE的高方差问题，一个非常优雅的方法被提了出来：减去基线（Baseline）。

核心思想：我们不直接用G作为动作好坏的度量，而是用G减去一个基线b(s)来评估：

∇J(θ) = E[ ∇log πθ(a|s) · (G - b(s)) ]

为什么这不会引入偏差？因为基线b(s)只依赖于状态s，不依赖于动作a：

E[ ∇log π(a|s) · b(s) ] = b(s) · E[ ∇log π(a|s) ] = b(s) · 0 = 0

最后那个E[∇log π(a|s)] = 0可以从概率归一化条件推导出来（Σ π(a|s) = 1 两边对 θ 求导即得）。所以减去基线后期望不变——无偏。

那么基线怎么选？理想的情况是选择能最大化方差缩减的基线。一个公认的好选择是状态价值函数 V(s)——它代表着"在这个状态下，平均能拿到多少回报"。

当我们用 G - V(s) 替代 G 时，这个差值实际上就是优势函数（Advantage Function）的估计：

A(s,a) = Q(s,a) - V(s)

它的含义是："在状态s下选择动作a，相比平均水平的优势"。正值表示这个动作比平均水平好，负值表示比平均水平差。这样，梯度更新就变成了"做比平均好的动作概率增加，比平均差的动作概率减小"——比原始的"回报越大越好"信号清晰得多。

让我们用一个表格来对比使用基线前后的梯度信号：

场景 原始G 基线 G - V(s) 梯度更新方向
好状态 + 好动作 高（如+50） 正（+20） 温和增加该动作概率
好状态 + 坏动作 高（如+50） 负（-30） 降低该动作概率
坏状态 + 好动作 低（如-20） 正（+10） 增加该动作概率
坏状态 + 坏动作 低（如-20） 负（-30） 温和降低该动作概率

注意第二行：没有基线时，即使智能体在好状态下选了坏动作，G仍然很高（因为状态本身就容易得高分），梯度会错误地鼓励这个坏动作。有了基线，我们才能看到"静态"——这个动作在这个状态下其实是差的选择。

11.5 REINFORCE的改进

综合前面的讨论，我们可以对REINFORCE做以下改进：

  1. 添加基线： 用另一个网络（或同网络的价值头）估计V(s)，用 G - V(s) 代替原始的G。
  2. 使用优势函数： A(s,a) = Q(s,a) - V(s) 提供了方差更低且含义更清晰的梯度信号。
  3. 归一化回报： 对整个batch的回报做标准化（减去均值除以标准差），相当于使用一个自适应的baseline，在实践中很有效。

改进后的代码核心部分：

class ActorCritic(nn.Module):
    def __init__(self, obs_dim, act_dim):
        super().__init__()
        self.fc = nn.Sequential(nn.Linear(obs_dim, 128), nn.ReLU())
        self.actor = nn.Linear(128, act_dim)   # 策略头
        self.critic = nn.Linear(128, 1)        # 价值头

    def forward(self, x):
        feat = self.fc(x)
        probs = torch.softmax(self.actor(feat), dim=-1)
        value = self.critic(feat)
        return probs, value

# 带基线的REINFORCE更新
probs, value = policy(state)
dist = Categorical(probs)
action = dist.sample()
log_prob = dist.log_prob(action)

# ... 收集完整回合数据后:
returns = compute_discounted_returns(rewards)
advantages = returns - values.detach()  # 基线: G - V(s)

actor_loss = -(log_probs * advantages).mean()
critic_loss = F.mse_loss(values, returns)
total_loss = actor_loss + critic_loss  # 联合优化

optimizer.zero_grad()
total_loss.backward()
optimizer.step()

这段代码中，我们用一个共享的特征提取网络同时输出策略和价值。actor_loss 用优势函数替代了原始回报，critic_loss 让价值网络学习预测真实的回报。两者联合优化——这正是走向Actor-Critic架构的第一步。

尽管如此，REINFORCE仍然面临几个棘手的问题：

  • 回合制更新： 必须等完整回合结束才更新，这在长回合或连续任务中不现实。
  • 利用率低： 每个经验只用一次就被丢弃，不像DQN那样有经验回放。
  • 方差依然较大： 虽然基线帮助很大，但MC回报本身的方差随回合长度增长的问题没有根本解决。

这些问题引出了强化学习中一个更大的框架——Actor-Critic方法。它用自举（Bootstrap）代替蒙特卡洛采样，用TD误差作为优势函数的近似，将方差降到更低的水平。我们说REINFORCE是"用整场比赛的比分来评价球员每个动作"，而Actor-Critic是"用教练的实时判断来指导球员下一步怎么走"。后者更及时、更稳定，也是现代策略梯度算法的标准范式。

本章小结

  • 策略梯度方法直接优化参数化策略 πθ(a|s)，克服了值函数方法在连续动作空间和随机最优策略上的局限。
  • 策略梯度定理给出了目标函数梯度的解析形式：∇J = E[ ∇log π · Qπ ]。
  • 对数导数技巧 ∇π = π ∇log π 是将梯度通过采样过程传播的关键。
  • REINFORCE 是最简单的策略梯度算法，用MC回报 Gt 近似 Qπ。实现简单但方差极大。
  • 基线技巧（Baseline）在保持无偏性的前提下降低方差，常用选择是 V(s)，对应优势函数 A(s,a) = Q(s,a) - V(s)。
  • REINFORCE的高方差和回合制更新的局限，驱动了从"蒙特卡洛"到"自举"的转变——这正是下一章Actor-Critic的主题。

策略梯度方法是强化学习中思想最深刻、应用最广泛的技术路线之一。从本章的REINFORCE出发，我们将在后续章节中看到它如何一步步进化为PPO、SAC等现代工业级算法。每一步进化都是为了解决一个问题：如何更快、更稳地找到最优策略。掌握好本章的基础，后面的路就顺畅多了！
