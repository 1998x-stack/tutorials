21.1 全连接层的"原罪"

为什么不能用全连接网络（MLP）处理图像？一张224×224的图片，每个像素作为一个输入特征，第一层如果有1000个神经元，权重矩阵就是 (224×224×3) × 1000 ≈ 1.5亿个参数。而且全连接层把像素当成孤立的数字，完全忽略了空间结构——相邻的像素组成了边缘和纹理，但全连接层"看不见"这种空间关系。

CNN通过两个核心操作解决了这个问题：卷积（Convolution）——用小的滑动窗口提取局部特征，参数共享；池化（Pooling）——降采样，减少计算量的同时增加平移不变性。

21.2 卷积操作：用"滤镜"扫描图像

想象你拿着一个小小的放大镜（比如3×3的矩阵），在图像上从左到右、从上到下滑动。每到一个位置，你把放大镜覆盖的像素和放大镜上的数值做逐元素乘积再求和——这就是一次卷积操作。

这个小"放大镜"就是卷积核（Kernel/Filter）。不同的卷积核能检测不同的特征：有的检测水平边缘，有的检测垂直边缘，有的检测纹理。在CNN中，这些卷积核的值不是人工设计的——它们是从数据中学习的。

概念解释典型值
kernel_size卷积核的空间尺寸3×3（最常用），5×5，7×7
stride每次滑动的步长1（细粒度），2（降采样）
padding在边缘补0以控制输出尺寸'same'（保持尺寸），'valid'（不补）
out_channels这一层有多少个卷积核64→128→256，逐层翻倍

21.3 经典架构演化史

架构年份核心创新为什么重要
LeNet-51998Conv→Pool→FC的经典范式证明了CNN在手写数字识别上的可行性
AlexNet2012ReLU + Dropout + GPU深度学习革命的引爆点，ImageNet误差率下降10%+
VGG2014只用3×3小卷积核堆深度证明了"深度是关键"，结构简洁优美
ResNet2015残差连接（跳过连接）解决了深层网络退化问题，最深到1000+层
EfficientNet2019用NAS搜索最优深度/宽度/分辨率比例同等精度下参数和计算量大幅减少

21.4 ResNet：残差连接的威力

ResNet的核心洞察非常反直觉：堆更多层之后，训练误差反而更高了（不是过拟合，是训练误差！）。这说明深层网络存在优化困难——梯度在反向传播中逐渐消失。ResNet的解决方案：加入"跳过连接"（Skip Connection），让梯度有一条"高速公路"直接流回浅层：

output = F(x) + x

这样网络只需要学习"残差"F(x) = output - x，而不是从头学习output。如果恒等映射已经足够好，F(x)可以很容易地被推到0。这个看似简单的改动让训练100+层的网络成为可能。

import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, in_channels, out_channels, stride=1):
        super().__init__()
        self.conv1 = nn.Conv2d(in_channels, out_channels, 3, stride, 1)
        self.bn1 = nn.BatchNorm2d(out_channels)
        self.conv2 = nn.Conv2d(out_channels, out_channels, 3, 1, 1)
        self.bn2 = nn.BatchNorm2d(out_channels)

        # 如果维度变化，shortcut需要1×1卷积调整维度
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels, 1, stride),
                nn.BatchNorm2d(out_channels)
            )

    def forward(self, x):
        out = nn.ReLU()(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out += self.shortcut(x)  # 残差连接
        return nn.ReLU()(out)

21.5 迁移学习：站在巨人的肩膀上

从头训练一个ResNet-50需要大量数据（ImageNet有120万张图）和算力（8×V100训练几天）。但绝大多数实际项目没有这么多资源。解决方案：迁移学习（Transfer Learning）——用别人在ImageNet上预训练好的权重做初始化，你只微调最后几层。

from torchvision.models import resnet50, ResNet50_Weights

# 加载在ImageNet上预训练的ResNet50
model = resnet50(weights=ResNet50_Weights.IMAGENET1K_V2)

# 冻结所有层（不需要梯度）
for param in model.parameters():
    param.requires_grad = False

# 替换最后一层为你自己的分类器
model.fc = nn.Sequential(
    nn.Linear(2048, 256),
    nn.ReLU(),
    nn.Dropout(0.5),
    nn.Linear(256, num_classes)  # 你的类别数
)

# 只训练fc层
optimizer = torch.optim.Adam(model.fc.parameters(), lr=1e-3)
💡 迁移学习最佳实践：(1) 如果你的数据量 < 1000，冻结所有预训练层，只训练最后分类层。(2) 数据量 1000-10000，解冻最后几层一起微调。(3) 数据量 > 10000，可以用预训练权重做初始化，全网络微调，但用较小的学习率（预训练部分的lr是新的全连接层的1/10）。数据增强在任何数据量下都有用。

21.6 数据增强：免费的"数据倍增术"

from torchvision import transforms

train_transforms = transforms.Compose([
    transforms.RandomResizedCrop(224),
    transforms.RandomHorizontalFlip(p=0.5),
    transforms.RandomRotation(15),
    transforms.ColorJitter(brightness=0.2, contrast=0.2),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406],
                         std=[0.229, 0.224, 0.225])
])

# 验证/测试时只做resize + normalize，不做随机增强
test_transforms = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406],
                         std=[0.229, 0.224, 0.225])
])
