第20章：计算机视觉经典数据集

你训练的图像分类模型在CIFAR-10上跑到了95%的准确率，你信心满满地把它部署到一个真实产品中。结果上线后，实际准确率掉到了60%——模型把厨房里的橘子识别成"台灯"，把沙发上的抱枕当成"猫"。问题出在哪里？大概率不是你的模型结构，而是你的训练数据跟真实世界的差距太大了。

计算机视觉数据集与NLP数据集有一个根本差异：图像的"语义空间"是连续的。同样的"猫"这个概念，可以对应千差万别的像素排列——不同品种、不同角度、不同光照、不同遮挡程度。而文本的语义空间相对离散，"猫"这个词在哪个文档里写都是那几个汉字。这就使得CV数据集的覆盖度和多样性变得极其重要。本章将系统梳理CV领域从分类到检测、分割再到生成式视觉数据的经典数据集家族。

CV数据集的金字塔结构：底层是图像分类数据集（ImageNet、CIFAR），中层是目标检测（COCO、Pascal VOC），上层是语义分割（ADE20K、Cityscapes），顶端是生成式图文数据（LAION-5B）。每一层对数据的标注密度和复杂性要求依次递增。

图像分类基石数据集

图像分类数据是CV世界的"普通话"——几乎所有视觉模型的预训练都从分类开始。下面三个数据集构成了CV分类任务的三大基石。

ImageNet-1K — 不可绕过的巅峰

ImageNet 自2012年AlexNet在此一鸣惊人以来，就一直是CV领域的"高考"。ImageNet-1K包含128万张训练图片，覆盖1000个物体类别，从"金毛寻回犬"到"三角钢琴"无所不包。几乎所有现代视觉模型（ResNet、ViT、ConvNeXt、EVA）都在ImageNet上预训练，然后迁移到下游任务。

from datasets import load_dataset

# ImageNet-1K（需要登录HF Hub授权）
# 注意：ImageNet不是完全免费的，需要接受使用条款
imagenet = load_dataset("imagenet-1k", split="train")
print(imagenet[0])
# {'image': ,
#  'label': 0}

# 查看类别名称
class_names = imagenet.features["label"].names
print(class_names[0])  # 'tench, Tinca tinca'
重要提醒：ImageNet的访问不是"即开即用"的。你需要在HF Hub上登录账户，接受ImageNet的使用条款（需要提供学术或机构邮箱）。如果你用 load_dataset("imagenet-1k") 报权限错误，先去HF Hub上点击"同意条款"的按钮。另外，ImageNet的版权问题一直有争议——2021年有人发现它包含大量的人脸照片，引发了隐私投诉。

CIFAR-10/100 — 快速迭代的好伙伴

需要快速验证一个模型思路？别去碰ImageNet——128万张图的训练要花几天。用 CIFAR 吧。CIFAR-10包含6万张32×32的小图（10类，每类6000张），CIFAR-100则是100类每类600张。图片虽小，但麻雀虽小五脏俱全：不同光照、不同角度、不同背景，对分类任务来说是个高效的"试验田"。

# CIFAR-10 — 10类，秒级加载
cifar10 = load_dataset("cifar10", split="train")
print(cifar10[0])
# {'img': ,
#  'label': 6}  # 6 = frog

# CIFAR-100 — 100类，每类样本更少
cifar100 = load_dataset("cifar100", split="train")
print(cifar100.features["fine_label"].names[:5])
# ['apple', 'aquarium_fish', 'baby', 'bear', 'beaver']

MNIST — CV世界的Hello World

说起MNIST，可能有人觉得它已经"过时"了。但不要小看这个只有28×28灰度手写数字的数据集。它最大的价值不是刷点——而是调试管线。当你的数据处理pipeline出了奇怪的问题，先用MNIST跑一遍，排除CV特定bug，再上ImageNet。

mnist = load_dataset("mnist", split="train")
print(mnist[0])
# {'image': ,
#  'label': 5}

# 用MNIST做pipeline的冒烟测试
def smoke_test(pipeline_fn):
    """确保处理pipeline不报错"""
    sample = mnist["train"][0]
    try:
        result = pipeline_fn(sample["image"])
        print(f"Pipeline OK: {type(result)}")
    except Exception as e:
        print(f"Pipeline FAILED: {e}")

下表对比了三大分类数据集的差异，方便你根据需求快速选择：

数据集 图片数 类别数 图片尺寸 加载耗时 最佳用途
MNIST 7万 10 28×28（灰度） 数秒 pipeline调试、教学
CIFAR-10/100 6万 10/100 32×32（彩色） 数十秒 原型验证、对比实验
ImageNet-1K 128万 1000 可变（平均469×387） 数分钟 预训练、迁移学习基座
实用建议：做科研或论文对比时，CIFAR上10个epoch看到趋势就可以去ImageNet验证了。但如果是做产品原型，直接上ImageNet预训练+微调——CIFAR上训出的模型在真实场景中基本不可用。

目标检测与实例分割

分类告诉模型"这是什么"，检测则要求模型回答"这是什么 + 它在哪"。数据复杂度一下子翻倍了——每个样本不再只有一个标签，而是一组边界框（bounding box）和类别标签的列表。

COCO 2017 — 检测界的ImageNet

COCO（Common Objects in Context）是目标检测和实例分割领域事实上的标准数据集。它包含12万张图片，80个物体类别，每张图片平均有7.7个实例标注。这个"平均7.7个"是关键——COCO的图片不是"一个物体在一张图中间"那种简单场景，而是真实、拥挤、多尺度的日常场景。

coco = load_dataset("coco", "2017", split="train")
print(coco[0].keys())
# dict_keys(['image', 'image_id', 'width', 'height',
#            'objects', 'annotations', ...])

# 边界框格式：[x, y, width, height]（左上角坐标 + 宽高）
# 类别ID（注意：COCO的类别ID从1开始，不是0）
print(coco[0]["objects"])
# {'id': [...], 'area': [...], 'bbox': [[x,y,w,h], ...],
#  'category': [1, 2, ...], 'iscrowd': [0, 0, ...]}
踩坑实录：某团队一开始在Pascal VOC上做目标检测，VOC只有20类、每张图平均2-3个标注框，模型很快就跑到了85% mAP。他们信心满满地把同一个模型迁移到COCO上，mAP直接掉到了55%。团队花了整整一周排查——不是模型的问题，是数据难度差距太大。COCO的图片里物体更小、遮挡更多、背景更杂乱。这正是"简单数据集上的高分是幻觉"的典型例子。如果目标是上线的产品级检测模型，请从COCO开始，不要在VOC上自欺欺人。

Pascal VOC — 轻量入门之选

Pascal VOC的标注格式与COCO不同——它使用 XML文件 存储每张图的标注信息，而不是COCO的JSON格式。VOC只有20类，约2万张图，适合快速验证检测模型或做教学实验。

voc = load_dataset("pascal_voc", split="train")
print(voc[0].keys())
# dict_keys(['image', 'image_id', 'objects'])
# objects中每个实例包含 name, pose, truncated, difficult, bndbox

# VOC的边界框格式：[xmin, ymin, xmax, ymax]
# 注意与COCO的 [x, y, w, h] 格式不同！
格式转换提示：COCO使用 [x, y, width, height] 格式，VOC使用 [xmin, ymin, xmax, ymax] 格式，YOLO则使用 [x_center, y_center, width, height]（归一化）。在切换数据集或模型框架时，边界框格式转换是一个极其常见的bug来源。建议写一个格式转换工具函数并单元测试。

语义分割

如果说检测标出了"每个物体的外框"，分割则要求标注出"每个像素属于什么类别"。这是标注成本最高的视觉任务之一，也是数据最稀缺的领域。

ADE20K — 场景理解的百科全书

ADE20K 由MIT CSAIL发布，包含2万张训练图片，标注了150个语义类别——从"天空""道路"到"沙发""台灯"，甚至包括"天花板上的裂缝"。这种细粒度使得ADE20K成为场景分割和语义理解的标准基准。

ade20k = load_dataset("ade20k", split="train")
print(ade20k[0].keys())
# dict_keys(['image', 'label', 'image_id'])

# label是分割掩码（segmentation mask），与image同尺寸
# 每个像素的整数代表该像素的类别ID
mask = ade20k[0]["label"]  # 
print(mask.size)  # 与原图相同尺寸

# 查看类别名称
class_names = ade20k.features["label"].names
print(len(class_names), class_names[:5])
# 150 ['wall', 'building', 'sky', 'floor', 'tree']

Cityscapes — 自动驾驶场景

Cityscapes 专注于城市街景分割，包含50个城市的5000张精细标注图片。它的独特之处在于密集标注——每张图有30个类别，包括道路、人行道、建筑、交通标志等自动驾驶关心的目标。

cityscapes = load_dataset("cityscapes", "semantic-segmentation", split="train")
print(cityscapes[0].keys())
# dict_keys(['image_id', 'image', 'label', 'instance_id'])
分割数据可视化技巧：分割掩码是单通道灰度图，直接看是黑的。可视化时，需要用类别颜色映射（color palette）将整数ID转成彩色图。ADE20K和Cityscapes都提供了标准调色板——dataset.features["label"].int2str() 可以帮你获取类名，但颜色映射需要从数据集的官方repo中获取。

生成式视觉数据

随着Stable Diffusion、DALL·E和Sora等生成式模型的崛起，一类全新的视觉数据集进入主流视野——图文对数据集。这些数据的结构非常简单："一张图片 + 一段文字描述"，但规模却是分类数据集的成百上千倍。

LAION-5B — 58亿图文对的巨兽

LAION-5B 是迄今为止最大的公开图文数据集，包含58亿个（图片, 文本）对，从Common Crawl中筛选而来。它有多大？如果全部下载，大约需要240TB的存储空间。好在我们可以用 streaming=True 来流式加载——这也是你第一次真正体会到streaming不是"锦上添花"，而是"不这么做根本没法用"。

# LAION-5B 用 streaming 加载（否则你的磁盘会哭）
laion5b = load_dataset("laion/laion5b", split="train", streaming=True)

# 取1000条做快速分析
for i, sample in enumerate(laion5b):
    if i >= 5: break
    print(sample["TEXT"][:50])  # 文本描述的前50字符
    # 注意字段名是大写的：TEXT, URL, HEIGHT, WIDTH 等

# 图片地址在 URL 字段中，需要单独下载
# datasets库不自动下载LAION的图片！
LAION注意事项：第一，LAION的图片没有嵌入在数据集中——你拿到的是URL列表，需要自己用 requests 或 img2dataset 下载图片，而且很多URL已经失效了。第二，LAION的数据清洗质量参差不齐——有研究指出其中包含大量低质量、不相关甚至有害的图文对。用之前一定要做质量过滤。第三，LAION-5B本身因为涉及CSAM问题曾在2023年底被短暂下架，恢复后加强了过滤。

COCO Captions — 人工标注的高质量描述

比起LAION的"野生"数据，COCO Captions 是人工标注的精品。每张COCO图片有5个独立的人工描述，涵盖不同的描述角度和措辞。这对于训练能够"理解而非记忆"的文生图模型非常关键。

coco_captions = load_dataset("coco_captions", split="train")
print(coco_captions[0])
# {'image': , 'sentences': {'raw': [...], 'tokenized': [...]}}
# 'sentences.raw' 包含5个描述字符串
生成式数据的关键指标：图文匹配度（image-text alignment）和描述多样性（caption diversity）。LAION的匹配度低但覆盖广，COCO Captions匹配度高但规模小。实际训练中常用的策略是：用LAION做大规模预训练，用COCO Captions做精调和对齐。这也是Stable Diffusion训练的基本配方。

CV数据集选型与加载实战

选CV数据集就像选相机镜头——没有最好的，只有最适合的。下表帮你快速定位：

你的目标 推荐数据集 加载标识 数据规模 提示
图像分类预训练 ImageNet-1K load_dataset("imagenet-1k") 128万图 需要HF授权
快速原型验证 CIFAR-10 load_dataset("cifar10") 6万图 秒级加载
目标检测入门 Pascal VOC load_dataset("pascal_voc") 2万图 简单，20类
目标检测/分割 COCO 2017 load_dataset("coco", "2017") 12万图 80类，真实场景
语义分割 ADE20K load_dataset("ade20k") 2万图 150类，室内外
自动驾驶分割 Cityscapes load_dataset("cityscapes") 5000图 精细标注
文生图训练 LAION-5B load_dataset("laion/laion5b") 58亿对 streaming必开

加载大图集时，一个关键的实践建议是：始终先用 streaming=True 做快速数据验证。ImageNet的128万张图如果全部下载到本地大约150GB——如果你的数据清洗脚本有bug，150GB的下载时间就白费了。

# 流式验证数据质量的推荐模式
dataset = load_dataset("imagenet-1k", split="train", streaming=True)

# 检查前100张图的尺寸分布
sizes = []
for i, sample in enumerate(dataset):
    if i >= 100: break
    sizes.append(sample["image"].size)

# 统计
from collections import Counter
widths = [s[0] for s in sizes]
print(f"Width range: {min(widths)}-{max(widths)}")
难度梯度建议：如果你的项目需要多个CV数据集，推荐按难度梯度使用——用ImageNet预训练（分类），用COCO做检测/分割的基座，用ADE20K做场景理解的扩展。每上升一层，数据标注的密度和任务的复杂度都显著增加。不要期望在CIFAR上训出的模型能在COCO上表现好——它们根本不是一个量级的挑战。
