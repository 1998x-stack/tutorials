当你的数据"头重脚轻"——类别不平衡的数学与对策

大家好，我是你们的讲师。今天我们来面对一个几乎所有实际AI项目都会遇到的问题——类别不平衡与长尾分布。

想象你在做欺诈检测。正常交易占99.9%，欺诈交易只占0.1%。你训练一个模型，什么都没学到，只是把所有交易都判为"正常"——恭喜你，准确率99.9%！但这样的模型完全没用。这就是类别不平衡最狡猾的地方：它会让标准的评估指标和训练过程都产生幻觉，让你以为模型做得很好，实际上它在"偷懒"。

类别不平衡问题不仅仅存在于二分类中。多分类的长尾问题同样严重——头部类别（几百到几千样本）主导训练过程，尾部类别（个位数样本）几乎学不到任何有用的决策边界。从医学影像的罕见病检测到NLP中的细粒度实体识别，从推荐系统的点击率预测到自动驾驶的罕见场景，类别不平衡无处不在。

先讲一个令人警醒的真实案例。某医疗AI公司开发了一个皮肤癌检测模型，训练数据中良性样本占95%，恶性样本占5%。模型在验证集上的准确率达到了94%，看起来非常不错。但当部署到真实诊所时，医生发现模型对恶性病变的召回率只有12%——它几乎把所有病变都判定为良性了。在医疗场景，这是一个致命的失败。根源是什么？就是类别不平衡没有被正确处理，而团队的评估指标选错了——他们一直在看准确率，忽略了少数类的召回率。

类别不平衡为什么伤害模型

类别不平衡的伤害机制可以用一个恶性循环来理解：

第一步：多数类样本数量远多于少数类，标准交叉熵损失的总和中，多数类的贡献占绝对主导。假设正负比是99:1，那每100个样本的loss中，有99个来自多数类。

第二步：梯度由loss决定，所以梯度主要来自多数类。模型参数朝着"把多数类分好"的方向更新，而"把少数类分好"的信号非常微弱。

第三步：迭代下去，模型学会了一个捷径——全部预测为多数类。因为这样做损失最小。少数类的决策边界崩塌了。

这个循环一旦形成，模型就彻底"躺平"了。更糟糕的是，准确率在这个场景下是一个极具欺骗性的指标——99%的准确率在欺诈检测中毫无意义。这正是为什么我们在第14章强调过：先确认评估可信度，再调参。

核心认知：类别不平衡不解决，模型优化的方向就从根本上偏了。标准交叉熵损失被大量多数类样本主导，少数类的梯度信号被淹没，导致模型对少数类的决策边界片面甚至完全坍塌。

Focal Loss的梯度分析

Focal Loss是解决极度不平衡最经典的损失函数，由Lin等人在2017年RetinaNet论文中提出。它的数学形式非常优雅，我们来深入分析它的梯度行为。

标准二分类交叉熵是：

CE(p, y=1) = -log(p)
CE(p, y=0) = -log(1-p)

Focal Loss引入调制因子：

FL(p, y=1) = -α(1-p)^γ log(p)
FL(p, y=0) = -(1-α)p^γ log(1-p)

理解Focal Loss的关键在于分析(1-p)^γ这个调制因子的梯度行为。

在标准交叉熵中，即使一个样本已经被模型很好地区分（比如p=0.9的正样本），它仍然贡献不小的梯度。这些"易分类样本"在数量庞大时，它们的总梯度仍然会淹没那些"难分类样本"（p=0.3的正样本）的信号。

Focal Loss通过(1-p)^γ来解决这个问题：

  • 当一个样本的预测概率p很高（模型已经能轻松分类）时，(1-p)^γ趋近于0，该样本的loss贡献被大幅降低。
  • 当一个样本的预测概率p很低（模型难以分类）时，(1-p)^γ接近1，loss几乎不受影响。
  • γ越大，这种"聚焦效应"越强。γ=0时退化为标准交叉熵。

Gamma参数的选择建议：

  • γ=0.5：轻微聚焦，轻度不平衡（1:10 ~ 1:50）
  • γ=1.0：中等聚焦，常见起始值
  • γ=2.0：强聚焦，论文默认值，中高度不平衡（1:100 ~ 1:1000）
  • γ=5.0：极强聚焦，极端不平衡（1:10000+）

搜索策略推荐以10倍步长进行——先试γ=0.5，效果好则降为γ=0.2、0.1；效果差则升为γ=2、5。注意Gamma并非越大越好——过大会导致模型过度关注噪声样本（那些"难"是因为标注错误而非本质难分的样本），损害泛化能力。

# PyTorch实现Focal Loss
import torch.nn.functional as F

def focal_loss(logits, targets, gamma=2.0, alpha=0.25):
    ce_loss = F.binary_cross_entropy_with_logits(
        logits, targets, reduction='none')
    p = torch.sigmoid(logits)
    pt = p * targets + (1 - p) * (1 - targets)
    focal_weight = (1 - pt) ** gamma
    if alpha >= 0:
        alpha_t = alpha * targets + (1 - alpha) * (1 - targets)
        focal_weight = alpha_t * focal_weight
    return (focal_weight * ce_loss).mean()

重采样 vs 重加权的统计偏差

处理类别不平衡有两个基本思路：在数据层面做文章（重采样）和在损失函数层面做文章（重加权）。两者都能缓解不平衡，但它们的统计偏差性质不同。

重采样（Resampling）分为过采样和欠采样：

  • 过采样：对少数类重复采样使其数量与多数类接近。但简单复制会导致对少数类过拟合——模型反复看到相同的少数类样本。过采样倍数一般不要超过10倍。
  • 欠采样：随机删除多数类样本。适用于数据量极大的场景（100万+），但会丢弃可能有用的多数类信息。不过如果多数类中有大量冗余样本，欠采样可能不影响甚至提升模型性能。
  • SMOTE：在少数类样本之间插值生成合成样本，避免简单复制。SMOTE在低维连续特征上效果较好，但在高维稀疏数据中可能生成无意义样本。

重加权（Reweighting）是在损失函数中给不同类别的样本分配不同的权重。形式上等价于对每个样本的loss乘上一个类别相关的系数。它的好处是不改变数据分布，不会引入重复样本的过拟合风险。

两者最核心的统计差异在于：重采样改变了训练数据的分布，重加权不改变。过采样改变了少数类在数据中的先验概率，这意味着训练分布和测试分布产生了偏移。而重加权只是在优化目标中调整了不同样本的重要性，训练数据分布保持不变。实践中可以这样选择：

优先级建议：轻度到中度不平衡（1:10 ~ 1:100）首选类别加权，简单稳定。中度到高度不平衡（1:100 ~ 1:1000）可以过采样+类别加权组合。极端不平衡（1:10000+）用Focal Loss + 过采样组合。欠采样保留在数据量极大且计算资源受限的场景。

类别加权的代码实现非常简单：

from sklearn.utils.class_weight import compute_class_weight
weights = compute_class_weight(
    'balanced', classes=np.unique(y), y=y)
# 权重 = n_samples / (n_classes * np.bincount(y))

评估指标的选择

在不平衡场景下，选错评估指标比选错模型更致命。因为指标决定了你判断"好"与"坏"的标准，如果标准错了，所有调参努力都白费。

准确率（Accuracy）是类别不平衡下最危险的指标。在99:1的数据上，模型全预测为多数类就能达到99%准确率，但F1=0。如果你用准确率做模型选择和调参，结果就是"最优模型是一个什么都不做的分类器"。

ROC曲线 vs PR曲线是另一个常见的误区。ROC曲线对类别不平衡不敏感——即使正样本比例很低，ROC曲线的形状变化不大。而PR曲线（Precision-Recall Curve）对不平衡更敏感，能更真实地反映模型在少数类上的表现。这是因为ROC的横轴是FPR（假阳性率），分母是全部负样本，在负样本极多时FPR的变化很小；而PR曲线的横轴是召回率，纵轴是精确率，两者都直接关注正类的预测质量。

各类不平衡处理方法的对比总结如下：

方法 原理 优点 缺点 适用场景
类别加权 增大少数类loss权重 简单稳定、不改变数据分布 对所有少数类一视同仁 轻度到中度不平衡
Focal Loss 动态降低易分类样本权重 自适应聚焦、对极端不平衡有效 Gamma需精细调参 极端不平衡、大量易分类负样本
过采样 重复少数类样本 简单直接 易过拟合、改变数据分布 少数类样本极少
SMOTE 插值生成合成样本 避免简单复制 高维稀疏数据效果差 低维连续特征
欠采样 删除多数类样本 适合大数据 丢失可能有用信息 数据量百万以上
阈值移动 调整决策阈值 无需重新训练 需要验证集找最优阈值 任何不平衡场景的补充

还有一个重要的实践原则：只在训练数据上做不平衡处理，测试集保持原始分布不变。如果你在过采样后的验证集上评估，指标会虚高。评估必须在保持原始分布的真实测试集上进行。多分类中的长尾问题也一样——按类别频率分层加权，对尾部类别做更强的增强，或者使用解耦训练策略（Decoupled Training）——先在全量数据上训练特征提取器，再用平衡数据重训分类头。
