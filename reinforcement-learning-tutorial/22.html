第二十二章：逆强化学习与模仿学习

同学们好，欢迎来到第二十二章。

讲到现在，我们一直在学同一个套路：定义奖励函数 → 智能体通过最大化奖励来学习策略。这个框架很强大，但它有一个致命的隐含前提——你得先有一个好用的奖励函数。可是在真实场景中，设计奖励函数往往比训练智能体本身还难。

举个简单的例子：你想让一辆车学会"良好地驾驶"。什么是好的驾驶？不偏离车道、保持安全距离、变道平顺、在红灯前停下——这些规则加起来几百条，而且互相冲突。你花两周写了一个奖励函数，上车一跑，车是学会了"原地画圈"——因为你的奖励函数忘了惩罚原地打转。这就是奖励工程（Reward Engineering）的噩梦。

那有没有办法绕过这个噩梦呢？有。既然我们很难定义"什么是好的驾驶"，但我们有大量人类驾驶数据——那我就从人类驾驶数据中直接学习，要么学策略本身（模仿学习），要么学人类在优化什么（逆强化学习）。这就是本章要讲的核心内容。

22.1 从演示中学习

想像一下这个场景：你是一个机器人工程师，想让一个机械臂学会拧瓶盖。传统做法是：写一个奖励函数，给"旋转变大角度"正奖励，给"抓握不稳"负惩罚，然后跑几百轮PPO。但人拧瓶盖的方式非常自然——抓、转、提——你能不能直接录一段人手拧瓶盖的视频，让机器人"看会"？

这就是从演示中学习（Learning from Demonstrations, LfD） 的核心思想。我们收集专家（通常是人类）完成任务时的轨迹数据 τ = (s₀, a₀, s₁, a₁, ..., s_T)，然后利用这些数据来训练智能体。根据"学什么"，可以分为两大类：

  • 模仿学习（Imitation Learning）：直接学习专家的策略 π(a|s)，学的是"怎么做"。
  • 逆强化学习（Inverse Reinforcement Learning, IRL）：从专家轨迹中反推出奖励函数 R(s,a)，学的是"为什么这么做"。

这两种思路各有优劣，下面我们逐一说清楚。

22.2 行为克隆（Behavioral Cloning）

行为克隆（BC）是模仿学习里最简单直接的方法。它的思路朴素得令人发指：把专家的 (状态, 动作) 对当作监督学习的训练数据，直接用最大似然估计来学习策略 π_θ(a|s)。

具体来说，给定专家轨迹数据集 D = {(s_i, a_i)}，BC的训练目标就是最小化交叉熵损失（对离散动作）或 MSE（对连续动作）：

# 离散动作空间的 Behavioral Cloning 训练伪代码
import torch
import torch.nn as nn

class BehavioralCloning:
    def __init__(self, policy_net, optimizer):
        self.policy = policy_net       # 神经网络策略: state -> action logits
        self.optimizer = optimizer
        self.loss_fn = nn.CrossEntropyLoss()

    def train_step(self, expert_states, expert_actions):
        # expert_states:  (batch_size, state_dim)
        # expert_actions: (batch_size,)   —— 每个状态的专家动作

        logits = self.policy(expert_states)          # (batch_size, num_actions)
        loss = self.loss_fn(logits, expert_actions)  # 交叉熵损失

        self.optimizer.zero_grad()
        loss.backward()
        self.optimizer.step()

        return loss.item()

    def act(self, state, deterministic=True):
        logits = self.policy(state.unsqueeze(0))
        if deterministic:
            return logits.argmax(dim=-1).item()
        else:
            return torch.distributions.Categorical(logits=logits).sample().item()

这段代码看起来和训练一个图像分类器几乎一模一样——训练时给 (状态, 标签) 对，推理时输出动作。BC 确实就是这么简单，在很多场景下效果也还不错，尤其是当专家数据覆盖了状态空间的大部分区域时。

踩坑故事——Behavioral Cloning 的"偏航"悲剧

我前几年帮一个自动驾驶创业公司做车道保持功能的初步验证。时间紧任务重，组里一个实习生花了三天录了 10 万帧人类驾驶数据（包括直线、弯道、变道等场景），用 Behavioral Cloning 训练了一个端到端的转向模型。在验证集上，准确率高达 95%。大家都很兴奋，直接上车实测。

结果车一上路，前 10 秒稳稳当当，第 11 秒遇到一个训练集中没出现过的小弯道，模型输出了一个略微偏大的转向角。纠正回来时已经偏了一点，进入了"没见过"的状态。再纠正，偏更多。不到 30 秒，车就冲上了路沿——幸好速度很慢，没有出事。

这就是 BC 最经典的复合错误（Compounding Error）问题。BC 训练时所有输入都来自专家轨迹，专家总是在车道正中央。但实际部署时，策略的一个微小错误会让它进入一个从未见过的状态，在这个状态上模型的预测完全不可靠，错误进一步放大，恶性循环。学术界称之为分布偏移（Distributional Shift）。

教训：BC 只适合专家数据能完美覆盖所有可能状态的场景——这在现实中几乎不存在。如果你必须用 BC，一定要配合 DAgger 或类似方法来收集 learner 轨迹上的专家标注。

DAgger（Dataset Aggregation）的改进思路非常巧妙：让 learner 策略在环境中跑，把遇到的"陌生"状态记录下来，然后去问专家在这些状态下应该怎么做，把这些新数据加入训练集重新训练。这样策略就能学会在自己会犯错的地方纠正回来。DAgger 在实践中效果显著，但代价是需要专家在线提供服务——对人来说成本很高。

22.3 逆强化学习原理

行为克隆有一个更深层的问题：它只复制了专家的动作，但没有理解专家为什么这么做。比如人类驾驶员在路口减速，是因为"看到黄灯"还是"感觉路面湿滑"？如果没有理解背后的意图，一旦环境稍有变化，克隆的策略就会失灵。

逆强化学习（Inverse RL）走的是另一条路：从专家的行为轨迹中推断出奖励函数。一旦我们有了正确的奖励函数，就可以用任何标准 RL 算法去学最优策略。甚至可以说，IRL 帮我们解决了 RL 里最头疼的问题——"如何设计一个好的奖励函数"。

数学上，我们的观察是专家轨迹 τ_expert，目标是找到一个奖励函数 R(s,a)，使得专家的策略在这个奖励函数下是最优的。这与标准 RL 正好相反：

  • 标准 RL：已知 R(s,a) → 求最优策略 π*
  • 逆 RL：已知专家轨迹 τ_expert（隐含着 π*） → 求 R(s,a)

但问题来了：这是一个典型的病态问题（Ill-posed Problem）。无数个不同的奖励函数都能解释同一组专家轨迹。比如，"看到红灯停车"和"看到红灯停车且奖励 +100"都是同一个行为，前者奖励函数稀疏后者稠密，但行为完全一样。更极端的例子：R(s,a) = 0 对所有 (s,a) 也"解释"了专家的所有行为（专家做什么都无所谓，反正奖励都是 0）。

为了解决这个问题，Ziebart 等人在 2008 年提出了最大熵逆强化学习（MaxEnt IRL）。核心思想是：在能解释专家行为的所有奖励函数中，选择那个使得轨迹分布熵最大的——即最不偏不倚、不做额外假设的那个。这个框架假设专家行为服从玻尔兹曼分布：

P(τ) ∝ exp( Σ R(s_t, a_t) )    —— 累积奖励越高的轨迹，概率越大
                                  —— 专家以"软最大化"的方式选择轨迹，而非绝对最优

MaxEnt IRL 优化目标:
  max R   L(R) = E_{τ ~ expert}[ Σ R(s_t, a_t) ] - log Z(R)

其中 Z(R) = Σ_τ exp( Σ R(s_t, a_t) ) 是配分函数，
第一项让专家轨迹的累积奖励尽量高，
第二项让其他轨迹的奖励不要太高——即保持随机性。

MaxEnt IRL 的美妙之处在于，它自动处理了"专家的行为不是绝对最优，而是接近最优"这一现实情况。而且它的梯度有直观的解释：沿着梯度更新时，奖励函数会推高专家轨迹上的奖励，压低非专家轨迹上的奖励——直到专家和其他轨迹的"期望特征计数"一致。

理解关键：IRL 解决的不是"怎么做好动作"，而是"专家到底在优化什么目标"。一旦你理解了目标（奖励函数），学策略就是标准 RL 的事了。这就像你不需要照搬棋王每一步的走法——如果你理解了他的战略意图，你可以用你自己的风格走出同样高水平的棋。

22.4 学徒学习（Apprenticeship Learning）

在 MaxEnt IRL 之前，Abbeel 和 Ng 在 2004 年提出了一个更直觉的方法——学徒学习（Apprenticeship Learning）。它的思想是：如果专家策略的"特征期望"和我学到的策略的"特征期望"一致，那我的策略就和专家一样好了。

所谓特征期望（Feature Expectation），就是对轨迹上每个状态的特征函数 φ(s) 求折扣累积和的期望：

μ(π) = E[ Σ γᵗ · φ(s_t) | π ]    —— 策略 π 下的特征期望向量

如果奖励函数是特征的线性组合 R(s) = w·φ(s)，那特征期望匹配就保证了值函数匹配。算法的核心是一个迭代过程：

  1. 初始化：从专家轨迹计算特征期望 μ_E。
  2. 在当前奖励 R_k(s) = w_k·φ(s) 下，用 RL 求出最优策略 π_k，计算其特征期望 μ_k。
  3. 如果 μ_k 和 μ_E 足够接近，停止。否则，调整权重 w_{k+1}，使专家和当前策略更可区分。
  4. 回到第 2 步。

这个算法像不像 SVM 的最大间隔分类？是的——Abbeel 和 Ng 的原始论文正是用最大间隔方法学到的权重 w 来区分专家和 learner 的轨迹。学徒学习是 IRL 领域的重要里程碑，而且直观易懂：你的行为和专家越像（特征期望越接近），你越好。

不过学徒学习假设奖励函数是特征的线性组合，这个假设在复杂环境中往往不够灵活。接下来的 GAIL 就完全抛弃了这个假设。

22.5 GAIL（Generative Adversarial Imitation Learning）

2016 年，Ho 和 Ermon 提出 GAIL，把生成对抗网络（GAN）引入模仿学习，效果惊艳。在 GAIL 中：

  • 生成器（Generator） = 策略 π_θ，负责生成轨迹。
  • 判别器（Discriminator） = D_ω(s,a)，负责区分"这是专家轨迹"还是"生成器轨迹"。

训练过程就是策略和判别器之间的博弈：

  • 判别器努力把专家轨迹判为 1，把生成器轨迹判为 0。
  • 生成器（策略）努力产生能骗过判别器的轨迹——即让判别器分不出是专家还是自己。

这个游戏的目标函数是：

判别器损失:  L_D = E_{τ~expert}[ log D(s,a) ] + E_{τ~learner}[ log(1 - D(s,a)) ]
策略（生成器）损失:  L_π = E_{τ~learner}[ -log D(s,a) ]    —— 等价于最大化判别器的"置信度"

训练流程伪代码:
for iteration in range(num_iters):
    # 1. 用当前策略采样一批轨迹
    trajectories = policy.sample(env, num_steps=2048)

    # 2. 更新判别器: 区分专家 vs 生成器
    expert_logP = discriminator.evaluate(expert_states, expert_actions)  # log D(τ_expert)
    learner_logP = discriminator.evaluate(learner_states, learner_actions)  # log D(τ_learner)
    disc_loss = -expert_logP.mean() - torch.log(1 - learner_logP.exp()).mean()
    disc_optimizer.step(disc_loss)

    # 3. 更新策略: 用判别器输出作为奖励信号 + PPO 更新
    # 奖励信号: r_t = -log(1 - D(s_t, a_t))  或  r_t = log D(s_t, a_t)
    rewards = -torch.log(1 - discriminator(learner_states, learner_actions).detach())
    ppo_update(policy, learner_states, learner_actions, rewards)

GAIL 的最大优势是样本效率高——不需要像 MaxEnt IRL 那样计算配分函数（那通常是 intractable 的），只需要学一个判别器来做二分类。而且 GAIL 不假设奖励函数的特定形式，判别器可以是一个深度神经网络，表达能力很强。

但 GAIL 也有代价：它需要大量的环境交互来采样 learner 轨迹。如果你只能离线学习（没有环境可以交互），那 GAIL 就不适用了——这时候你需要离线模仿学习（如 DemoDICE、ValueDICE 等），那已经是另一个研究方向了。

现在我们来总结对比一下这几种方法：

方法 学什么 需要环境交互 抗分布偏移 代表工作
行为克隆（BC） 直接学策略 π(a|s) 否 弱 Pomerleau (1989)
DAgger BC + 在线专家标注 是（但可问专家） 较强 Ross et al. (2011)
MaxEnt IRL 学奖励函数 R(s,a) 是（内层 RL） 强 Ziebart et al. (2008)
学徒学习 匹配特征期望 是（内层 RL） 强 Abbeel & Ng (2004)
GAIL 对抗式模仿 是（采轨迹） 强 Ho & Ermon (2016)

一句话总结：如果你有完美的环境模拟器和大量专家数据，GAIL 是目前最实用的选择。如果你只能离线学习，BC + DAgger 是最快的起点。如果你需要理解专家行为背后"为什么"，逆强化学习（IRL）是必经之路。

在实际应用中，这三种思路的边界其实在逐渐模糊。比如近年来有人把 BC 和 GAIL 混合使用——先用 BC 做预训练得到初始策略，再用 GAIL 做 fine-tune，效果往往比单独用其中一种好很多。

说到实际应用，下一章我们终于要动手了。我们将用 Python 的 Gymnasium 库来搭建实际的强化学习环境——从导入环境、观察状态空间、执行动作，到跑通一个完整的训练循环。逆强化学习虽然听起来抽象，但它的底层（策略学习、奖励函数、轨迹采样）全是标准 RL 的老朋友。弄懂了今天这些概念，下一章的动手练习会让你觉得格外顺手。
