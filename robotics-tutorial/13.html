第13章 PID与反馈控制

大家好，我是你们的讲师。之前我们学了运动学、动力学、轨迹生成——我们已经告诉机器人"去哪"以及"用什么力度去"。但现实不是完美的：摩擦力没被建模、电机力矩有噪声、连杆有弹性形变。这就是为什么我们需要反馈控制（Feedback Control）——用真实的传感器测量值来纠正模型误差。

在反馈控制的世界里，PID控制是绝对的主角。自1922年Minorsky提出PID控制律以来，一百多年过去了，它仍然是工业机器人中最广泛使用的控制算法。据统计，超过95%的工业控制回路使用PID或其变体。无论你是做四旋翼、机械臂还是四足机器人，PID都是你的"默认技能"。它的优雅之处在于：不需要模型，只需要误差。

13.1 PID的物理直觉

PID控制器由三项组成，每项都有非常直观的物理含义：

  • P（比例项）：相当于一个弹簧。当位置有误差时，P项产生一个与误差成正比的恢复力。误差越大，拉回来的力度越大。类比：弹簧的刚度（Stiffness）。
  • I（积分项）：累积过去的误差，消除稳态误差。如果有一个恒定的外部扰动（如重力），P项无法完全消除误差（需要无穷大的P才会零误差），I项通过积累过去的误差来补偿这个扰动。类比：一个不断拧紧的螺丝。
  • D（微分项）：根据误差的变化率来预测未来。当误差正在快速减小时，D项会提前"收力"防止超调。类比：阻尼器（Damper）。

数学形式非常简洁：

u(t) = Kp · e(t) + Ki · ∫e(τ)dτ + Kd · de(t)/dt

其中 e(t) = q_d(t) - q(t) 是位置误差

每一项的作用和副作用总结如下：

参数 物理类比 增大效果 副作用
Kp 弹簧刚度 响应更快，稳态误差减小 超调增大，可能振荡
Ki 持续拧紧的螺丝 消除稳态误差 积分饱和，响应变慢
Kd 阻尼器 抑制超调，提高稳定性 放大噪声，导数冲击

13.2 PID调参方法

调参是PID控制的"艺术"部分。以下是几种经典的调参方法：

Ziegler-Nichols法：一种经验性方法。先将Ki和Kd设为零，增大Kp直到系统进入等幅振荡（临界振荡），记录此时的Kp值为Ku，振荡周期为Tu。然后按经验公式设置：

P控制器:  Kp = 0.5 · Ku
PI控制器: Kp = 0.45 · Ku,  Ki = 1.2 · Kp / Tu
PID控制器: Kp = 0.6 · Ku,  Ki = 2 · Kp / Tu,  Kd = Kp · Tu / 8

手动调参启发式：

  1. 先调Kp：从小到大增加Kp，直到系统开始出现轻微振荡，然后回退20%。
  2. 加入少量Kd：Kd抑制超调，通常从Kp的1/10开始试。
  3. 最后加Ki：如果存在稳态误差，从很小的Ki开始（Kp的1/100），增加到误差消除为止。

频率域方法（进阶）：通过Bode图分析增益裕度（Gain Margin）和相位裕度（Phase Margin）。通常要求相位裕度在30°-60°之间，增益裕度在6-12dB之间。这种方法可以提供稳定性保证，但需要系统的频率响应数据。

实操建议：对于关节伺服控制，一个常见的经验起点是 Kp=100, Kd=10, Ki=1（具体取决于系统惯量和编码器分辨率）。在仿真中把这三组参数调到"临界稳定"再下调30%，然后上真机微调。注意：仿真中的最优参数在真机上通常会振荡——因为仿真没有建模所有的摩擦、间隙和弹性变形。

13.3 前馈与级联控制

纯PID控制有一个根本限制：它是一种"反应式"控制——只有出现了误差才会产生控制作用。对于高速、高动态的系统，这往往不够。前馈控制（Feedforward Control）提供了一种互补思路：利用模型预测你需要多少力矩，在误差出现之前就输出。

经典的"前馈+反馈"结构：

τ_total = τ_ff + τ_fb

τ_ff = M(q)·q̈_d + C(q,q̇)·q̇ + G(q)  # 逆动力学前馈
τ_fb = Kp·e + Kd·ė + Ki·∫e dt         # PID反馈

前馈处理了"已知"的部分（重力、科氏力、惯性力），反馈处理了"未知"的部分（摩擦、未建模动力学、外部扰动）。两者配合可以达到P项很小但跟踪精度仍然很高的效果——这对于柔顺控制尤其重要。

级联控制（Cascade Control）是工业伺服驱动器的标准架构，通常分三层：

  • 最内层：电流环（Current Loop），频率4-20kHz，控制电机电流（力矩）
  • 中间层：速度环（Velocity Loop），频率1-4kHz，控制关节速度
  • 最外层：位置环（Position Loop），频率100-1kHz，控制关节位置

级联控制遵循一个关键原则：内环的频率至少是外环的5-10倍。因为内环的响应速度决定了外环能看到"多干净"的执行器行为。如果电流环太慢，速度环看到的力矩响应就会有延迟，导致整个级联系统不稳定。

在ROS 2的 ros2_control 框架中，级联控制通过 JointLimits 和 PIDController 的配置来实现：

# ros2_control 配置片段
joint1:
    pid:
        p: 100
        i: 1
        d: 10
        i_clamp: 50   # 积分饱和限制
    feedforward:
        gravity_compensation: true

踩坑故事：谈到积分饱和（Integral Windup），我必须分享一个让我印象深刻的经历。在一次大型六轴机器人调试中，我们给一个关节设置了较大的Ki值来消除重力造成的稳态误差。当机器人运动到接近极限位置时，遇到了硬限位——电机堵转了。但由于Ki仍在累积误差，积分项在一个小时内涨到了输出上限的300%。当操作员把机器人移开限位后，积分项瞬间释放，输出了一个巨大的力矩，机器人猛地向反方向甩去，撞上了旁边的料架——幸运的是没有人受伤。从此之后，我们的系统中永远启用了anti-windup机制。最常见的anti-windup方法是：当输出饱和时，冻结积分项的累加（有条件积分）。还有更高级的"back-calculation"方法，在饱和时将积分项往回拉。无论用哪种，anti-windup绝对不是一个可选项，而是必须项。

13.4 PID常见问题

除了积分饱和，PID在实践中还有一些常见问题需要特别注意：

  • Derivative Kick：当期望位置发生阶跃变化时，微分项中的 de/dt 会瞬间产生一个很大的脉冲信号（因为误差突然从0跳到一个大值）。解决方法是在D项中只对测量值求导，不对期望值求导：u_d = Kd · (0 - dq/dt)，而不是 u_d = Kd · d(q_d - q)/dt。
  • 测量噪声放大：D项本质上是一个高通滤波器，会放大编码器的高频噪声。即使是很小的量化噪声，经过微分后也会变成明显的力矩波动。解决方法是在微分路径上加入一阶低通滤波器（通常截止频率设为50-200Hz）。
  • 输出饱和：控制输出受限于电机最大电压/电流。未处理饱和时会出现"控制失效"的问题。除了前面提到的anti-windup，还应该在输出端做限幅 u = clip(u, -u_max, u_max)。

13.5 实战：仿真中的PID

下面是使用MuJoCo仿真环境整定PID的完整示例（Pyhton + MuJoCo）：

import numpy as np
import mujoco

# 加载MuJoCo模型
model = mujoco.MjModel.from_xml_path("robot.xml")
data = mujoco.MjData(model)

# PID参数
Kp = 200.0
Ki = 5.0
Kd = 20.0

# 目标位置（关节空间）
q_des = np.array([0.5, -0.3, 0.8])

# 控制状态
integral_error = np.zeros(3)
prev_error = np.zeros(3)

# 仿真主循环
for t in range(10000):
    # 当前误差
    error = q_des - data.qpos[:3]

    # 积分项（带anti-windup）
    integral_error += error * model.opt.timestep
    integral_error = np.clip(integral_error, -5.0, 5.0)

    # 微分项（误差变化率）
    derivative = (error - prev_error) / model.opt.timestep

    # PID输出
    tau = Kp * error + Ki * integral_error + Kd * derivative
    tau = np.clip(tau, -100.0, 100.0)  # 输出限幅

    # 施加控制
    data.ctrl[:3] = tau

    # 步进仿真
    mujoco.mj_step(model, data)

    # 记录数据
    prev_error = error.copy()

# 调参对比：只试Kp=50（太软 → 跟踪慢）
#           Kp=200（合适 → 跟踪快但少许超调）
#           加Kd=20（消除超调）
#           加Ki=5（消除稳态误差）
print("PID控制仿真完成")

在实践中，你可以依次尝试以下三组参数，观察响应差异：

  • P-only：Kp=100, Ki=0, Kd=0——有稳态误差，响应较慢
  • PD：Kp=200, Ki=0, Kd=20——响应更快，无超调，但可能有微小静差
  • PID：Kp=200, Ki=5, Kd=20——稳态误差被消除，响应快且无超调

终极建议：在实际机器人项目中，我从不用"纯PID"做关节控制。标准做法是PD + 重力前馈（Gravity Compensation），或者更完整的"逆动力学前馈 + PID反馈"。前者用 τ = Kp·e + Kd·ė + G(q)，后者用完整的 τ = M·q̈_d + C·q̇ + G + Kp·e + Kd·ė + Ki·∫e。重力补偿就能消除大部分稳态误差，使Ki可以设得很小甚至为零——这从根本上避免了积分饱和的风险。

本章我们学习了PID控制的物理含义（弹簧-阻尼-积分器）、调参方法与Ziegler-Nichols经验公式、前馈与级联控制的工程架构、以及积分饱和和Derivative Kick等常见问题及解决方案。PID是机器人控制的"第一性原理"，在任何一本机器人学教科书中它都是核心内容。

PID够用但有限——当机器人需要与环境交互（如装配、打磨）、处理约束（如关节极限、力矩极限）、或者协调全身多个任务时，我们需要更强大的控制框架。下一章：先进控制方法——计算力矩控制、阻抗控制、MPC和全身控制。
