第十章：Transformer架构

大家好！这一章是整个NLP课程最重要的章节——没有之一。2017年Google那篇"Attention is All You Need"彻底改变了NLP的格局，不只是技术升级，而是范式革命。这篇论文的8位作者后来全都成了AI领域的明星（有些人创办了Character.AI、Cohere等公司），这本身就是传奇。

我2017年底第一次读这篇论文时，说实话，第一遍没完全看懂。反复读了三四遍，又手写了前向传播的每一步计算，才真正理解它的精妙。Transformer放弃了RNN的循环结构，完全靠Attention机制建模序列关系——它用并行计算替代了串行计算，用显式的位置编码弥补丢失的顺序信息。接下来的内容，我尽量用最直观的方式帮你理解。

10.1 为什么要抛弃RNN

RNN/LSTM有三个根本性的问题，Transformer一次性全部解决：

RNN的问题Transformer的解决方案
串行计算：t时刻必须等t-1完成——无法并行Self-Attention对所有位置同时计算——完全并行
长距离依赖：梯度要传很多步，即使LSTM也只能到~200步任意两个位置之间是O(1)的路径——直接连接
信息混合：位置越远，信息越容易被"冲淡"Attention权重显式建模所有位置对之间的关系

这三个问题导致RNN既慢效果又受限。而Transformer的解法极其优雅——让每个词直接和其他所有词"打招呼"，距离缩短到1步。

10.2 Self-Attention：自己看自己

这是Transformer最核心的模块。Self-Attention和之前Seq2Seq中的Attention本质相同，区别在于：Seq2Seq的Attention是解码器看编码器（跨序列），Self-Attention是序列自己看自己（同序列内）。

# Self-Attention 计算（Scaled Dot-Product Attention）
# 输入：序列 X = (seq_len, d_model) 的词向量矩阵

# Step 1: 线性变换得到 Q, K, V
# Q = X · W_Q    # Query: "我想查什么"
# K = X · W_K    # Key:   "我能提供什么匹配"
# V = X · W_V    # Value: "我实际携带什么信息"

# Step 2: 计算注意力分数矩阵（seq_len × seq_len）
# Scores = Q · K^T / √d_k
# 除以√d_k是为了防止点积太大导致softmax梯度消失

# Step 3: Softmax + 加权
# Attention(Q,K,V) = softmax(Scores) · V

# 输出维度与输入相同：(seq_len, d_model)
# 但每个位置的输出现在包含了和所有位置交互后的信息

核心洞察：Self-Attention让序列中每对词之间都建立直接连接。在"I went to the bank to deposit money"中，"bank"和"deposit money"的Attention权重会很高——模型通过Self-Attention自动学会了消歧。

10.3 Multi-Head Attention：多角度看问题

单头Attention的问题：每个词只能以一种方式关注其他词。Multi-Head Attention同时做多组不同的Q/K/V变换，让模型从多个角度捕捉不同类型的依赖关系。

# Multi-Head Attention
# head_i = Attention(X·W_Q^i, X·W_K^i, X·W_V^i)
# MultiHead = Concat(head_1, ..., head_h) · W_O

# 8个头各学什么？（经验观察）
# Head 1: 相邻词关系（局部语法）
# Head 3: 指代消解（"it" → "the cat"）
# Head 5: 主谓一致（长距离句法）
# Head 8: 语义相关（"eat" → "food"）
# 不同头各司其职，合起来覆盖了语言的多个层面

10.4 位置编码：给并行计算补上顺序

Self-Attention对所有位置一视同仁——词序信息完全丢失。"我爱你"和"你爱我"在Self-Attention看来没有任何区别。位置编码（Positional Encoding）就是为了补上这个缺口。

# 正弦位置编码
# PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
# PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

# pos: 位置索引（0, 1, 2, ...）
# i: 维度索引（0, 1, ..., d_model/2-1）
# 偶数维用sin，奇数维用cos

# 巧妙之处：
# PE(pos+k) 可以由 PE(pos) 的线性变换得到
# 所以模型可以学到相对位置关系
# sin/cos的值域在[-1,1]，和词向量量级一致

为什么不用可学习的位置向量？其实可以用（BERT用的就是可学习位置嵌入）。但正弦编码的优势是可以外推——训练时最长128位置的模型，测试时勉强能处理150位置的序列。可学习编码无法外推。

10.5 Transformer完整架构

把全部组件拼起来，Transformer的编码器层长这样：

# Transformer Encoder Layer
def transformer_encoder_layer(x):
    # 1. Multi-Head Self-Attention + 残差 + LayerNorm
    attn_out = MultiHeadAttention(x, x, x)
    x = LayerNorm(x + attn_out)       # Add & Norm

    # 2. Feed-Forward Network + 残差 + LayerNorm
    ffn_out = FFN(x)  # = ReLU(x·W1 + b1)·W2 + b2
    x = LayerNorm(x + ffn_out)        # Add & Norm

    return x

# FFN通常：d_model → 4×d_model → d_model（先扩张4倍再缩回）
# 整个编码器堆叠N层（BERT-base: N=12, d_model=768, heads=12）

解码器和编码器类似，但多了一个Cross-Attention（看编码器输出的Attention），并且Self-Attention加了因果掩码（causal mask）——不能看未来的词。

⚠️ 常见误解：LayerNorm放在残差连接之后（Post-LN），而非之前。这是原始论文的做法。但后来的研究发现Pre-LN（LayerNorm在前）训练更稳定，GPT-3用的就是Pre-LN。两种写法效果不同，实现时要确认你用的是哪种。

10.6 复杂度分析

操作复杂度瓶颈
Self-AttentionO(n²·d)序列太长时（n>2048），n²会爆炸
FFNO(n·d²)当d很大时（如d=12288），d²主导
总体O(n²·d + n·d²)取决于n和d的相对大小

Self-Attention的O(n²)是对长序列最大的挑战——这也是为什么处理长文档时需要Longformer、BigBird等稀疏Attention变体。但对大多数NLP任务（n<512），这个代价完全可接受。
