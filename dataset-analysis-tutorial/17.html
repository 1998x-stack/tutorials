第17章：数据质量评估与清洗

有一个经典案例值得每个AI从业者记住。某电商平台开发了一个情感分析模型，用于自动识别用户评论的情感倾向。上线前的测试中，模型在验证集上达到了95%的准确率——令人满意的数字。然而上线后，真实用户的准确率暴跌到不到70%。更重要的是，模型对负面评论的召回率几乎为零——用户写了差评，模型判断为好评。原因查明后令人哭笑不得：训练数据集的全部10万条标注，居然是由一位实习生单独完成的。这位实习生有一个强烈的个人倾向——他无法忍受"负面情绪"，于是潜意识里把所有模棱两可的评论都标注成了正面。训练集中有接近40%的"正面"样本实际上是负面或中性的。模型的"95%准确率"不过是在学习标注者的偏见，而不是学习语言的情感——因为验证集也是同一个人标注的。

这个故事说明了两个道理：第一，数据质量比数据数量重要得多；第二，数据质量的缺陷往往是系统性的——不会是少数样本标错，而是整个标注流程存在偏差。如果不在训练前做质量评估，你甚至不知道问题存在。

核心认知：数据质量评估不是"可选的锦上添花"，而是"必需的防错机制"。在投入算力训练之前，先花10%的时间和预算做数据质量审计，可以节省后续90%的调试时间。

数据质量维度框架

评估数据质量不能凭感觉。业界普遍采用5维度评估模型来系统化地衡量数据集质量：

维度 定义 检查方法 可接受标准
完整性（Completeness） 样本关键字段无缺失 统计各列的null/NaN/empty比率 关键字段缺失率 < 1%
准确性（Accuracy） 样本值和真实情况一致 抽样人工校验 + 交叉验证 标注准确率 ≥ 95%
一致性（Consistency） 同类型样本的标注逻辑一致 Inter-annotator agreement 或逻辑校验 Cohen's Kappa ≥ 0.8
时效性（Timeliness） 数据反映当前现实 检查样本的时间戳分布 数据采集时间在模型预期有效期以内
唯一性（Uniqueness） 无重复或近似重复样本 MinHash + LSH去重检测 重复率 < 5%

这五个维度不是孤立的。比如，一个数据集可能在"准确性"上很高（每个标注都经过专家审核），但"时效性"很差（数据是5年前采集的），导致模型在当下场景中表现不佳。在做质量评估时，应该为每个维度设定一个量化阈值，使用"木桶效应"思路——最差的那个维度决定了整体质量。

实践建议：在项目的README.md中记录数据集的5维度评分。这不仅帮助团队成员理解数据状况，在向审计或合规部门汇报时也是很好的凭证。比如："本数据集完整性0.98，准确性0.96，一致性0.92，时效性0.85，唯一性0.99"。

EDA 检查清单

探索性数据分析（Exploratory Data Analysis）是质量评估的第一步。在开始任何清洗工作之前，先通过统计指标"感受"一下数据的脾气。

缺失率统计

from datasets import load_dataset
import pandas as pd

ds = load_dataset("imdb", split="train")

# 抽样统计缺失率（全量扫描大数据集可能很慢）
sample = ds.select(range(10000))
df = sample.to_pandas()

missing_rates = df.isnull().mean()
print("缺失率统计：")
print(missing_rates[missing_rates > 0])

# 如果是文本字段，空字符串也算"缺失"
empty_rates = (df == "").mean()
print("\n空字符串比率：")
print(empty_rates[empty_rates > 0])

缺失率只是起点。缺失的模式可能比缺失本身更重要：是随机缺失（MAR）还是系统性缺失（MNAR）？比如在用户评论数据中，如果"年龄"字段的缺失集中在某个特定的用户群体中，那这就是一个系统偏差信号。

异常值检测

对于数值型特征，IQR（四分位距）方法是最通用的异常值检测手段：

import numpy as np

def detect_outliers_iqr(series):
    Q1 = series.quantile(0.25)
    Q3 = series.quantile(0.75)
    IQR = Q3 - Q1
    lower = Q1 - 1.5 * IQR
    upper = Q3 + 1.5 * IQR
    outliers = series[(series < lower) | (series > upper)]
    return outliers, lower, upper

# 检测文本长度的异常值
text_lengths = df["text"].str.len()
outliers, lo, hi = detect_outliers_iqr(text_lengths)
print(f"文本长度正常范围: [{lo:.0f}, {hi:.0f}]")
print(f"异常样本数量: {len(outliers)} ({len(outliers)/len(df)*100:.1f}%)")
IQR的局限性：IQR方法假设数据大致呈单峰分布。对于多模态数据（比如同时包含短文本和长文档的数据集），IQR会把中间峰值的边界误判为异常。这种情况下考虑使用 Isolation Forest 或 DBSCAN 做更灵活的距离检测。

标签分布与平衡性

分类任务的标签分布如果不平衡，模型会偏向多数类。检查分布很简单：

# 标签分布检查
label_counts = df["label"].value_counts(normalize=True)
print("标签分布比例：")
for label, ratio in label_counts.items():
    bars = "█" * int(ratio * 50)
    print(f"  {label}: {ratio:.1%} {bars}")

# 最小类别占比
min_ratio = label_counts.min()
if min_ratio < 0.1:
    print(f"⚠️ 严重不平衡: 最小类别仅占 {min_ratio:.1%}")
    print("建议: 使用过采样、欠采样或加权损失函数")

除了标签分布本身，还要检查标签和特征之间的相关性。比如在评论数据中，如果"标签=正面"的评论平均长度是"标签=负面"的2倍，说明存在标注偏差——标注者可能习惯把短评论标为负面。

重复样本检测

重复数据会隐性降低数据集的有效规模，甚至导致训练集和验证集之间泄漏。文本数据的去重可以用 datasets 结合 datasketch 的 MinHash：

from datasketch import MinHash, MinHashLSH

def compute_minhash(text, num_perm=128):
    m = MinHash(num_perm=num_perm)
    for word in text.split():
        m.update(word.encode('utf-8'))
    return m

# 对第一批1000条做近似去重检测
lsh = MinHashLSH(threshold=0.8, num_perm=128)
duplicates = []

for i, row in df.head(1000).iterrows():
    m = compute_minhash(row["text"])
    result = lsh.query(m)
    if result:
        duplicates.append((i, result))
    lsh.insert(f"doc_{i}", m)

print(f"发现 {len(duplicates)} 个近似重复样本")

数据画像（Data Profiling）

手动编写统计脚本做EDA虽然灵活，但在实践中，使用自动化的数据画像工具可以大大提升效率。这些工具会一次性生成一个包含所有统计指标的报告，让你能在一张页面中"读完"数据集的全部特征。

ydata-profiling / sweetviz 集成

将 datasets 和 ydata-profiling 结合使用：

import ydata_profiling as yp
import pandas as pd
from datasets import load_dataset

ds = load_dataset("imdb", split="train")
# 抽样足够多样本保证统计可靠
sample = ds.select(range(5000)).to_pandas()

profile = yp.ProfileReport(sample, title="IMDb Dataset Profile")
profile.to_file("imdb_profile.html")
# 打开 imdb_profile.html 即可查看完整报告

ydata-profiling 的报告包含了上面提到的所有指标：缺失率、异常值、分布图、相关性矩阵，甚至还有交互式图表。在一个项目中，我通过这个报告发现了一个之前从未注意到的数据泄露：训练集和测试集中有超过15%的文本是近似重复的，因为数据采集时没有做去重。这个发现直接改变了整个实验的有效性。

使用建议：对于超大数据集，不要对整个数据集做profiling。随机抽样5,000~10,000条即可获得统计显著的画像。如果数据分布不均匀（如长尾分布），考虑分层抽样以确保尾部也被覆盖。

清洗决策树

发现了问题，接下来怎么处理？针对每一种数据质量问题，有一个"决策树"来指导行动：

缺失值处理决策树

缺失模式 缺失率 推荐操作 适用场景
随机缺失（MAR） < 5% 直接删除缺失行 数据量充足的场景
随机缺失（MAR） 5% ~ 30% 均值/中位数/众数填充 数值特征，分布大致对称
系统性缺失（MNAR） 任意 新建"是否缺失"指示列 缺失本身可能是信号
高缺失率 > 70% 删除该特征列 该特征几乎不提供有用信息
关键字段缺失 任意 模型预测填充或人工回溯 如label缺失，不能简单删除
决策原则：永远优先考虑"标记"而非"抹除"。在修复缺失值之前，先创建一个布尔列记录哪些行曾经有缺失。这样模型可以自己决定是否学习缺失模式。你可以在后续建模中发现，缺失模式本身可能是一个强大的预测特征。

异常值处理策略

from scipy import stats

def handle_outliers(series, method="winsorize"):
    """异常值处理函数"""
    if method == "winsorize":
        # Winsorize: 将极端值缩放到边界值
        return stats.mstats.winsorize(series, limits=[0.01, 0.01])
    elif method == "truncate":
        # 截断: 直接删除异常样本
        outliers, lower, upper = detect_outliers_iqr(series)
        return series[(series >= lower) & (series <= upper)]
    elif method == "cap":
        # 封顶: 限幅到阈值
        _, lower, upper = detect_outliers_iqr(series)
        return series.clip(lower, upper)
    elif method == "keep":
        # 保留: 标记异常但不修改
        return series  # 但记录异常索引备用
    else:
        raise ValueError(f"未知方法: {method}")

重复数据处理

  • 精确重复：直接去重，保留第一条。使用 df.drop_duplicates()。
  • 近似重复：用MinHash + LSH检测后，手动审查或合并。在文本数据中，近似重复往往比精确重复更危险——因为它更难检测，但同样会导致数据泄漏。
  • 标签冲突重复（同样内容但不同标签）：保留投票一致的标签。如果只有少数样本有冲突，考虑删除这些样本以避免混淆模型。

标注质量审计

如果说数据清洗解决的是"数据本身的问题"，标注审计解决的则是"标注过程的问题"。很多AI项目失败，不是因为数据量不够，而是因为标注质量不够好。

标注一致性（Inter-Annotator Agreement）

当多人参与标注时，计算他们之间的一致性是一个基本要求。最常用的指标是 Cohen's Kappa（两人）或 Fleiss' Kappa（多人）：

from sklearn.metrics import cohen_kappa_score

# 假设有两个标注者对1000条样本做了独立标注
annotator_a = [1, 0, 1, 1, 0, 1, ...]  # 标注者A的结果
annotator_b = [1, 0, 0, 1, 0, 1, ...]  # 标注者B的结果

kappa = cohen_kappa_score(annotator_a, annotator_b)
print(f"Cohen's Kappa: {kappa:.3f}")

if kappa >= 0.8:
    print("✅ 标注一致性优秀")
elif kappa >= 0.6:
    print("⚠️ 标注一致性良好，但需要定期复核")
else:
    print("❌ 标注一致性不足，需要重新培训标注者或简化标注指南")
Kappa 的陷阱：Kappa 分数受类别分布影响极大。当某个类别的占比超过90%时，Kappa会被人为拉低（这是因为"偶然一致"的概率变高了）。遇到高度不平衡的数据时，考虑使用 Gwet's AC1 作为替代指标。

Confident Learning 去噪

传统方法依靠人工抽检来发现标注错误。但面对百万级数据集，抽检率不可能超过5%。Confident Learning（置信学习）提供了一种自动化的噪声标签检测方法。它的核心思路是：用一个模型在数据上训练，找出那些"模型很自信预测对了，但标注却是错的"的样本——这些样本极大概率是被标错的噪声。

# cleanlab 集成的 Confident Learning 使用示例
from cleanlab.classification import CleanLearning
from sklearn.linear_model import LogisticRegression

# 假设 X_train 是特征，labels 是原始标注
cl = CleanLearning(clf=LogisticRegression(max_iter=1000))
cl.fit(X_train, labels)

# 获取被标记为"可能标错"的样本索引
label_issues = cl.get_label_issues()
noisy_indices = label_issues[label_issues["is_label_issue"]].index
print(f"发现 {len(noisy_indices)} 个可疑的标注错误样本")

# 将这些样本交给人工复审
suspicious_samples = df.iloc[noisy_indices]
suspicious_samples.to_csv("review_candidates.csv", index=False)

Confident Learning 的一个实际应用案例是修正 ImageNet 中的标注错误——一个广泛引用的研究发现 ImageNet 验证集中大约有5.8%的标注是错误的，使用 Confident Learning 可以自动发现其中大部分。

标注审计流水线：在实际项目中，建议建立以下审计流程：①每个标注批次完成后，计算标注者之间的Kappa系数 → ②将全体数据的10%分配给双重标注（两人独立标注同一批） → ③对Kappa低于阈值的数据退回重标 → ④用Confident Learning筛选候选错误 → ⑤人工审核候选并修正。这套流程可以把标注错误率从典型的5-10%降低到1%以下。

系统偏差检测

回到本章开头的故事——实习生的系统偏差。检测系统偏差的方法是在EDA中关注特征与标签之间的不合理关联。比如：

  • 所有长度低于10个字的评论都被标为"负面"？→ 可能的偏差
  • 所有上午标注的样本准确率明显高于下午？→ 标注疲劳
  • 标注者A偏好标签1而标注者B偏好标签2？→ 个人偏差
  • 女性面孔在"友好"标签中的比例异常高？→ 社会偏差

系统偏差比随机噪声危害大得多，因为模型会学到这些偏差并在推理时放大。如果发现系统偏差，最有效的解决方案是回炉重标——纠正标注指南、重新培训标注者、做双重标注校验。

检测偏差的一个有用工具：在标注过程中随机插入"黄金样本"（已经由专家确认标签正确的样本）。如果标注者在黄金样本上的正确率低于95%，说明培训不足或标注指南不清晰，需要暂停流水线重新校准。

数据质量评估和清洗不是为了追求"完美数据"——在实践中，完美数据是不存在的。质量评估的真正目标，是理解你的数据有什么样的缺陷，以及这些缺陷对下游模型会产生多大的影响。带着对数据缺陷的认知去训练模型，你就能在面对模型效果不佳时，准确判断是"模型的问题"还是"数据的问题"，而不是像很多人一样对着GPU日志乱调超参数。
