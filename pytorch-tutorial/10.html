第10章 循环神经网络与注意力

大家好，我是你们的讲师。欢迎来到第十章。

在处理序列数据时，我们面临一个和图像完全不同的挑战：文本、音频、时间序列不像图像那样有固定的空间结构。一句话可以有 5 个词也可以有 50 个词，同一个意思可以用不同的语序表达。处理这种变长数据，我们需要一种能够"记住"历史信息的网络结构——这就是循环神经网络（RNN）的出发点。

从 2014 年到 2020 年左右，RNN 和它的变体 LSTM、GRU 几乎统治了所有的序列建模任务：机器翻译、语音识别、文本生成、情感分析。直到 2017 年 Transformer 和自注意力机制的横空出世，格局才被彻底改写。但即使在 2026 年的今天，RNN 在特定场景下仍然有不可替代的价值。而理解 RNN 的局限性，恰恰能帮你更好地理解注意力机制为什么有效。这一章我们先讲 RNN 和 LSTM，然后过渡到自注意力——你会发现，这两者解决的是同一个问题，只是思路不同。

10.1 RNN 面临的挑战：变长与长期依赖

CNN 在图像上表现优秀，但无法直接处理序列数据，原因有二：

  • 变长输入：CNN 的卷积核尺寸是固定的，输入图片必须 resize 到相同尺寸。但句子长度天然不同，你不能把每句话都"resize"到相同长度——会丢失信息。
  • 时间依赖：序列中的每个元素都和前后的元素相关。CNN 只能看到局部区域（感受野有限），无法捕捉"前面说过的内容和后面有联系"这种长距离依赖。

RNN 的设计思想很直接：每次处理序列中的一个元素，同时维护一个"隐状态（hidden state）"向量，这个向量像接力棒一样在时间步之间传递。每一步，隐状态都会根据当前输入和上一步的隐状态进行更新。

import torch.nn as nn

# 最简单的 RNN
rnn = nn.RNN(
    input_size=100,    # 每个时间步的输入维度（如词嵌入维度）
    hidden_size=128,   # 隐状态的维度
    num_layers=1,
    batch_first=True,  # 输入形状: [B, seq_len, input_size]
)

# 前向传播
output, h_n = rnn(x)
# output: [B, seq_len, 128]  每个时间步的输出
# h_n:    [1, B, 128]         最后一个时间步的隐状态

10.2 梯度消失与 LSTM

普通 RNN 有一个致命缺陷——梯度消失。想象你在看一本 500 页的小说，看到第 500 页时还想记住第 1 页的内容。在 RNN 中，第 1 步的信息要经过 499 次矩阵乘法才能传到第 500 步，每次乘法都会把梯度缩小一些。经过几百步的累积，梯度几乎降为零——模型根本学不会"长期依赖"。

这种效应在反向传播中尤为直观：

# 梯度消失的数学直觉
# 在每个时间步，梯度乘以 W_hh 的权重矩阵
# 经过 T 步后，梯度 ≈ 初始梯度 × (W_hh)^T
# 如果 W_hh 的特征值小于 1 (T 很大时)，梯度趋向于 0
# 如果 W_hh 的特征值大于 1，梯度爆炸（梯度裁剪能缓解但不能解决消失）

LSTM（Long Short-Term Memory）通过引入"门控机制"解决了这个问题。它不再简单地用矩阵乘法更新隐状态，而是设计了三个门来精确控制信息的流动：

  • 遗忘门（Forget Gate）：决定"过去的信息有哪些需要丢弃"。一个 sigmoid 层输出 0~1 的值，0 表示"完全忘记"，1 表示"完全保留"。
  • 输入门（Input Gate）：决定"当前输入中有哪些新信息需要存入记忆"。用 sigmoid 决定要更新的内容，用 tanh 生成候选值。
  • 输出门（Output Gate）：决定"当前记忆中有哪些需要输出到隐状态"。
lstm = nn.LSTM(
    input_size=100,
    hidden_size=128,
    num_layers=2,
    batch_first=True,
    dropout=0.3,      # 多层 LSTM 层间 dropout
)

# 输出
output, (h_n, c_n) = lstm(x)
# output: [B, seq_len, 128]
# h_n:    [2, B, 128]  各层的最后隐状态
# c_n:    [2, B, 128]  各层的最后单元状态（LSTM 独有）
LSTM 的三个门，一个简单的直觉：
  • 遗忘门：你在读一本推理小说，记到第 10 章时，第一章的细节已经不重要了，可以忘掉。
  • 输入门：第 11 章出现了一个关键线索，需要重点记住。
  • 输出门：有人问你这本书讲了什么，你从记忆中提取最关键的部分回答。
这种"选择性记忆"的机制，让 LSTM 可以轻松处理上百步的序列。

10.3 GRU：LSTM 的简洁版本

GRU（Gated Recurrent Unit）是 LSTM 的简化变体，它将遗忘门和输入门合并为一个"更新门"（Update Gate），并且去掉了单独的单元状态（cell state）。结果就是：GRU 的参数比 LSTM 少约 25%，但性能在很多任务上和 LSTM 不相上下。

gru = nn.GRU(
    input_size=100,
    hidden_size=128,
    num_layers=2,
    batch_first=True,
)

output, h_n = gru(x)
# output: [B, seq_len, 128]
# h_n: [2, B, 128]

选择建议：如果你的数据量不大，或者需要快速迭代，GRU 是更好的起点。如果数据量很大且你有足够的计算资源，LSTM 的额外参数可以带来更好的表现力。对于大多数实际项目，GRU 通常是最佳性价比选择。

10.4 2026 年的 RNN：还有用武之地吗？

在大模型时代，RNN 被很多人认为已经过时了。但事实上，RNN 在以下几个场景中依然生机勃勃：

场景 为什么 RNN 仍适合 典型应用
边缘设备推理 RNN 参数量小、计算量低，可以在 MCU 上运行 智能手表上的语音唤醒
实时信号处理 RNN 是流式处理的天然选择，不需要完整序列 ECG 心电异常检测、传感器数据
时间序列预测 中等长度的时序数据，RNN/LSTM 仍有竞争力 股票价格、气象数据、工业监测
小样本序列任务 数据量少时，RNN 比 Transformer 更不容易过拟合 少量标注的语音/文本数据

10.5 自注意力：更好的选择

好，现在我们来说说为什么对于大多数序列任务，你应该用自注意力而不是 RNN。

先讲一个故事。2017 年，我参与了一个翻译项目，需要把中文新闻翻译成英文。我们用了一个基于 LSTM 的 seq2seq 模型，输入编码器的是 100 个词以内的中文句子。翻译短句时效果很好，但一旦句子超过 30 个词，质量就开始下降。到 60 个词时，翻译结果已经是"词不对意"了。我们试了双向 LSTM、注意力增强的 LSTM、甚至堆叠了 4 层 LSTM 编码器——效果有提升，但 100 步的距离始终是个瓶颈。

直到 2018 年我们尝试了 Transformer——同样的训练数据，同样的计算资源，翻译质量直接提升了 10 个 BLEU 点。原因很简单：Transformer 的自注意力机制允许每个位置直接"看"到序列中的所有其他位置，而不是像 RNN 那样一步一步地传递信息。

自注意力（Self-Attention）的核心思想只有三个概念：

  • Query（查询）：当前这个位置"想找什么"
  • Key（键）：每个位置"有什么"
  • Value（值）：每个位置"能提供什么信息"

它的计算过程就像一个信息检索系统：每个位置生成 Q、K、V，然后计算每个位置的 Query 和所有位置的 Key 的相似度（点积），得到注意力权重，最后用权重加权求和所有位置的 Value。这样，每个位置的输出都集结了整个序列的信息，而且路径长度只有 1——无论序列多长，信息都可以一步到达。

# 自注意力的核心计算（简化版）
def scaled_dot_product_attention(Q, K, V):
    # Q, K, V: [B, seq_len, d_k]
    scores = torch.matmul(Q, K.transpose(-2, -1))  # [B, seq_len, seq_len]
    scores = scores / (K.size(-1) ** 0.5)            # 缩放

    # 注意力权重
    attn_weights = torch.softmax(scores, dim=-1)

    # 加权求和
    output = torch.matmul(attn_weights, V)
    return output, attn_weights

10.6 Transformer 编码器简介

一个简化的 Transformer Encoder 层包含两个子层：多头自注意力（Multi-Head Self-Attention）和 前馈网络（Feed-Forward Network, FFN），每个子层都有残差连接（Residual Connection）和层归一化（LayerNorm）。

from torch.nn import TransformerEncoder, TransformerEncoderLayer

# 一个 Transformer 编码器层
encoder_layer = nn.TransformerEncoderLayer(
    d_model=512,        # 嵌入维度
    nhead=8,            # 注意力头数
    dim_feedforward=2048,
    dropout=0.1,
    batch_first=True,
)

encoder = nn.TransformerEncoder(encoder_layer, num_layers=6)
# 输入: [B, seq_len, 512]
RNN 和自注意力的根本区别：
  • RNN 是顺序处理——每一步依赖前一步的结果，不能并行，呈 O(seq_len) 的计算复杂度。
  • 自注意力是全局访问——所有位置同时参与计算，可完全并行，呈 O(seq_len²) 的复杂度（这是 Transformer 的瓶颈，但在 seq_len < 2048 时通常可以接受）。
简单说：RNN 慢但省内存，Transformer 快但吃内存。这也是为什么长文本场景依然在用各种高效注意力变体的原因。

这一章我们从 RNN 的梯度消失问题出发，理解了 LSTM 和 GRU 的门控设计哲学，也看到了为什么自注意力在大多数情况下是更好的选择。序列数据处理是深度学习中最丰富、最具挑战性的领域之一，RNN 和注意力就是我们理解这个领域的两个支点。下一章我们来讨论一个非常实用的话题——模型保存与加载。在这个大模型时代，学会怎么把训练好的模型安全地存起来、正确地加载出来，是每个工程师的必备技能。
