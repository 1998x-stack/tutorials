大家好，我是你们的讲师。上一章咱们用Excel搭建了一个漂亮的财报分析框架——但不知道你有没有遇到这些问题：当你想分析10家公司5年的数据时，Excel变得巨慢；当你想做一个复杂的统计回归时，Excel做不了；当你想定期获取最新数据并自动更新报告时，Excel全靠手动操作。

这就是咱们今天要学Python的原因。Python不是为了取代Excel，而是为了填补Excel做不到的那些事。在专业投资机构里，Excel和Python是并存的：Excel用来做快速查看和临时分析，Python用来做批量处理、统计建模和自动化报告生成。两者相辅相成，缺一不可。

别担心你没有编程基础。咱们这章不是Python编程课，而是一堂"用Python做财报分析"的实战课。你不需要成为程序员，你只需要能看懂和修改代码就够了——这就跟会用Excel公式是一个道理。

Python财务分析工具链

做Python财报分析，你需要了解以下几个核心工具：

  • akshare：开源的A股数据接口库，免费、数据源多、更新快。支持获取A股个股的三大报表、财务指标、行情数据等。这是目前国内做A股分析最推荐的免费数据源。
  • tushare：老牌的A股数据接口库，数据质量高，但部分接口需要积分（付费）。如果你做深度研究且预算允许，tushare的财务数据质量会比akshare更稳定。
  • baostock：免费的A股数据接口，无需注册，但数据覆盖面和更新速度略逊于akshare。
  • yfinance：获取美股（如苹果、特斯拉）财务数据的利器，通过雅虎财经接口获取数据。做跨境比较分析时非常有用。
  • pandas：Python数据分析的核心库。处理表格数据、计算财务比率、数据清洗全靠它。相当于"Excel的Python版本"。
  • matplotlib + seaborn：画图库。做趋势图、柱状图、热力图、雷达图等财务图表。

数据获取实战

咱们用一个具体的例子来演示。假设我们要分析贵州茅台（600519.SH）近5年的财务状况。

首先安装必要的库（只需运行一次）：

比率计算与可视化

数据到手后，咱们就可以用pandas快速计算所有核心财务比率了。不同于Excel需要一个个写公式，Python可以批量计算：

# 创建一个比率分析DataFrame
ratios = pd.DataFrame(index=annual_reports['报告期'].unique())

# 计算毛利率
if '营业收入' in annual_data.columns and '营业成本' in annual_data.columns:
    ratios['毛利率'] = (
        (annual_data['营业收入'] - annual_data['营业成本'])
        / annual_data['营业收入'] * 100
    )

# 计算净利率
if '营业收入' in annual_data.columns and '净利润' in annual_data.columns:
    ratios['净利率'] = (
        annual_data['净利润'] / annual_data['营业收入'] * 100
    )

# 从资产负债表获取总资产和净资产数据
total_assets = balance_sheet_data['资产总计']
total_equity = balance_sheet_data['归属于母公司股东权益合计']

# 计算ROA和ROE
ratios['ROA'] = net_profit / total_assets * 100
ratios['ROE'] = net_profit / total_equity * 100

# 计算营收增长率
ratios['营收增长率'] = revenue.pct_change() * 100

# 打印结果
print(ratios.round(2))

构建自动化分析脚本

掌握了基础操作后，咱们可以更进一步——构建一个自动化分析脚本，输入股票代码，输出完整的分析报告。

这里提供一个概念框架：

def financial_analysis_report(stock_code, company_name):
    """
    输入股票代码，输出完整财报分析报告
    """
    # 1. 获取数据
    balance = get_balance_sheet(stock_code)
    income = get_income_stmt(stock_code)
    cashflow = get_cash_flow(stock_code)

    # 2. 数据清洗与比率计算
    ratios = calculate_all_ratios(balance, income, cashflow)

    # 3. 打分系统（Piotroski F-Score）
    f_score = calculate_piotroski_fscore(balance, income, cashflow)
    print(f"{company_name} Piotroski F-Score: {f_score}/9")

    # 4. 生成图表
    generate_charts(ratios, company_name)

    # 5. 导出Excel报告
    with pd.ExcelWriter(f'{company_name}_财报分析.xlsx') as writer:
        ratios.to_excel(writer, sheet_name='财务比率')
        balance.to_excel(writer, sheet_name='资产负债表')
        income.to_excel(writer, sheet_name='利润表')

    print(f"报告已生成：{company_name}_财报分析.xlsx")
    return ratios, f_score

其中Piotroski F-Score是一个经典的基本面评分模型，从盈利能力、杠杆/流动性、运营效率三个维度对公司打分（满分9分）。实现起来并不复杂，核心是判断9个条件是否满足：

def calculate_piotroski_fscore(balance, income, cashflow):
    """计算Piotroski F-Score (0-9分)"""
    score = 0

    # === 盈利能力 (4分) ===
    # 1. 资产收益率(ROA)为正
    if roa > 0: score += 1
    # 2. 经营活动现金流为正
    if operating_cf > 0: score += 1
    # 3. ROA同比改善
    if roa_change > 0: score += 1
    # 4. 经营现金流 > 净利润（盈利质量高）
    if operating_cf > net_income: score += 1

    # === 杠杆/流动性 (3分) ===
    # 5. 资产负债率同比下降
    if debt_ratio_change < 0: score += 1
    # 6. 流动比率同比上升
    if current_ratio_change > 0: score += 1
    # 7. 未增发新股（稀释度下降）
    if not issued_new_shares: score += 1

    # === 运营效率 (2分) ===
    # 8. 毛利率同比改善
    if gross_margin_change > 0: score += 1
    # 9. 资产周转率同比改善
    if asset_turnover_change > 0: score += 1

    return score

F-Score > 7表示公司基本面很强，< 4表示存在风险。这个模型虽然简单，但在学术研究中被验证长期有效——高F-Score组合的收益率显著跑赢市场。

踩坑故事——用错数据字段导致分析结果完全相反

2022年，有个量化团队在做A股基本面因子研究时，犯了一个非常"低级"但后果严重的错误。他们的Python脚本在计算各公司的市盈率（PE-TTM）时，使用了扣非净利润作为分母——但隔壁团队在验证时，用的是归母净利润。

结果发现：扣非净利润口径下筛选出的"低PE组合"年化收益率16%，而归母净利润口径下只有11%。两个团队争论了整整两周，谁也说服不了谁。最后复盘发现，问题的根源在于akshare的接口文档中，字段名净利润实际上对应的是"归属于母公司所有者的净利润"（归母净利润），而不是他们以为的"合并报表净利润"。

这件事的教训是什么？在A股财务分析中，"净利润"有至少三种不同的口径：

  • 合并报表净利润：集团整体利润，包含少数股东损益
  • 归母净利润：属于母公司股东的利润（A股估值最常用的口径）
  • 扣非净利润：扣除非经常性损益后的归母净利润（反映主业盈利能力）

用错口径，分析结论可能完全相反。特别是对于少数股东损益占比较大的公司（比如那些有大量非全资子公司的集团型企业），合并净利润和归母净利润之间可能相差30%以上。

血泪教训：用Python获取财经数据时，永远不要相信字段名。拿到数据后第一件事：打印列名，核对数据的单位和口径。最好的做法是在脚本中加一个"数据校验"步骤——比如从两个不同数据源获取同一数据做交叉比对（akshare vs tushare），或者将Python获取的数字与公司年报原文核对一个关键科目。这个步骤只需要5分钟，但能避免99%的数据口径错误。
