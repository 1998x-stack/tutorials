第25章 AI加速器与领域专用架构

同学们好！上一章我们聊了Chiplet——芯片在物理形态上的变革。今天我们要聊的是体系结构层面最激动人心的话题：AI加速器与领域专用架构（Domain-Specific Architecture, DSA）。如果说过去50年计算机体系结构的故事是"通用处理器一统天下"，那么现在这个故事正在被重写——当AI工作负载成为计算需求的主要驱动力时，"专用的、为特定领域设计的"正在取代"通用的、什么都能跑的"。这不是渐进式的改进，而是范式级的颠覆。

25.1 通用计算的终结？DSA的黄金时代

要理解为什么DSA突然这么火，我们需要回顾三个关键趋势：

第一，Denard Scaling终结了。晶体管的尺寸不再能等比缩小电压，导致每个晶体管的功耗密度不再下降。这意味着我们不能再通过简单地增加更多通用核心来获得性能提升——热墙已经撞上。

第二，AI工作负载爆发了。传统的CPU和GPU虽然在不断变强，但AI模型的需求增长更快。Transformer模型的计算量在2017-2024年间增长了10万倍——从GPT-1的约10^18 FLOPs到GPT-4的约2.1e25 FLOPs。通用处理器无法在合理的功耗范围内跟上这个速度。

第三，RISC-V的模块化扩展理念正在蔓延。当你可以为一个特定领域设计自定义指令扩展时，"为什么不干脆设计一个完全为该领域优化的处理器？"这个问题就变得非常自然了。

DSA（领域专用架构）的核心思想很简单：为特定问题域设计专门的硬件。代价是灵活性降低——DSA没法高效运行通用软件。但收益非常惊人：在目标领域内，DSA可以达到通用处理器10-1000倍的能效提升。典型的DSA例子包括：

  • Google TPU——ML推理/训练加速
  • NVIDIA Tensor Core——GPU中的矩阵乘法专用硬件
  • Apple Neural Engine——iPhone上的端侧模型推理
  • AWS Nitro——虚拟化卸载卡
  • Qualcomm Hexagon——数字信号处理/传感融合

这些DSA有一个共同特征：它们不是为了通用计算设计的，但它们在特定任务上表现得比其他任何处理器都好。这不是偶然，而是一种设计哲学的选择。

25.2 Google TPU：DSA的教科书案例

如果要用一个案例来理解DSA的精髓，Google TPU（Tensor Processing Unit）是完美的教材。

TPU v1（2015年）：Google看到了一个关键洞察：神经网络推理不需要训练那么高的精度和灵活性。推理可以用8位整数运算，不需要32位浮点；推理不需要分支预测和乱序执行等复杂功能；推理的数据流是高度规则的——矩阵乘以矩阵。基于这些洞察，TPU v1采用了一个极简但极致的设计：

  • Systolic Array（脉动阵列）——256 × 256 = 65536个MAC单元，以脉冲式的方式将数据流过阵列。不像GPU的SIMT那样需要复杂的调度逻辑（warp scheduler、scoreboard、execution mask），Systolic Array的数据流是固定的、可预测的，因此控制逻辑极简。
  • 24MB片上SRAM——设计者知道，推理的瓶颈不是计算，而是数据搬运。24MB的统一缓冲区可以容纳大量中间结果，避免频繁访问外部DRAM。
  • 最小控制逻辑——没有Cache coherence、没有分支预测、没有乱序执行——这些东西对于推理来说基本是浪费晶体管。

结果：28nm工艺，92 TOPS（Tera Operations Per Second），而功耗仅75W。作为对比，当时的顶级GPU在同样工艺下，要达到相似的推理吞吐量可能需要200-300W。这就是DSA的威力——不做多余的事情。

TPU v2（2017年）开始支持训练，引入了bfloat16浮点格式（brain floating point，16位指数8位，尾数7位）。bfloat16的设计思路同样体现了DSA的思维模式：AI训练需要比int8更高的精度，但不需要像FP32那么大的动态范围。bfloat16的动态范围和FP32相同（都是8位指数），但尾数精度较低。结果：在模型精度几乎无损失的情况下，节省了一半的存储和带宽。

TPU v4（2021年）引入了光学互连（Optical Circuit Switching），实现了3D Torus拓扑的4096芯片互连网络。光学互连带来的灵活性可以动态调整网络拓扑，这对于分布式训练来说是一个巨大的优势。

TPU v5 / v5p（2023年）进一步引入了稀疏加速——利用模型权重中的稀疏性，跳过零值的计算。v5p达到459 TFLOPS BF16 per chip，单Pod支持8960个芯片。TPU v5p的一个Pod的算力，已经超过了世界上99.99%的超算。

TPU的产品线演进完美展示了DSA的设计原则：不追求"XXX最快的处理器"这样的泛泛目标，而是专注于"XXX最快的AI训练/推理处理器"。这种聚焦让TPU每次迭代都在AI领域把通用处理器甩得更远。

25.3 数据流架构：超越冯·诺依曼

2025-2026年体系结构领域最轰动的消息是什么？英伟达以200亿美元收购Groq，英特尔以160亿美元收购SambaNova——两家CPU巨头在2025-2026年花了360亿美元押注数据流架构。这不是买一个产品，而是买一条技术路线。这个信号非常明确：数据流是后摩尔时代的答案之一。

英伟达收购Groq和英特尔收购SambaNova——两家CPU巨头在2025-2026年花了360亿美元押注数据流架构，这标志着体系结构领域的一次代际转变。数据流架构正在从学术研究走向大规模商业部署。

数据流（Dataflow）架构与传统的控制流（Control Flow）架构有本质区别：

  • 控制流（冯·诺依曼模型）：程序是一个指令序列，PC（程序计数器）告诉我们下一条指令在哪，如果发生分支则跳转。数据的产生和使用被指令流暗含。
  • 数据流：程序是一个DAG（有向无环图），节点是操作，边是数据依赖。一个节点只有当其所有输入数据"准备好"时才会执行。数据像水流一样在节点之间流动。没有PC，没有分支预测。

数据流架构的四个关键优势：

  • 极致并行——DAG中的任何无依赖节点都可以并行执行。编译器/调度器负责将尽可能多的节点映射到可用的计算单元上。
  • 超低延迟——不需要取指、译码、分支预测、重排序。数据到达即执行，整个流程是流水线式的。
  • 高效内存访问——数据不需要通过传统的内存层次结构（L1-L2-L3-DRAM），而是通过可配置的片上网络（NoC）或流式寄存器直接传输到计算单元。
  • 能效优势——因为没有投机执行带来的浪费，所有计算都是"有用的"。典型数据流芯片的能效比相同工艺的GPU高出2-5倍。

Groq LPU（Language Processing Unit）：Groq的设计最具代表性。他们的LPU采用完全确定性的数据流架构——每个操作精确占用固定的时钟周期，没有cache miss，没有分支误预测，没有内存延迟抖动。这意味着什么？一个推理请求在LPU上的延迟是完全可预测的。对于需要服务水平协议（SLA）的云服务来说，确定性延迟是一个巨大的卖点。Groq LPU在单芯片上达到超过500 TOPS的int8性能。

SambaNova RDU（Reconfigurable Dataflow Unit）：SambaNova走的是另一条路——可重构数据流。他们的RDU包含大量的可配置计算和存储单元，编译器根据模型结构生成数据流图，然后"烧录"到RDU上。这种可重构性让RDU可以灵活适应不同模型的演进，不需要硬件改动——只要重新"烧录"一次数据流图即可。

25.4 存内计算与近存计算

AI计算的另一个核心问题是"内存墙"（Memory Wall）——对于许多AI模型，数据搬运消耗的能量和延迟远超计算本身。研究表明，在一些大模型推理场景中，数据移动占总能耗的80%以上！

解决方案的两条路线：

近存计算（Near-Memory Computing）：把计算单元放在靠近内存的地方。三星的HBM-PIM（Processing-in-Memory）就是典型代表——在HBM（高带宽内存）的每个Bank中直接嵌入一个FP16 MAC单元。数据不需要从HBM传输到处理器核心，直接在内存内部完成计算。结果：特定工作负载下性能提升2倍，能耗降低70%。

存内计算（Processing-in-Memory, PIM）：更激进的方案——直接在存储阵列内部进行计算。d-Matrix的Raptor平台采用3D In-Memory Compute（3DIMC）技术，将计算逻辑直接堆叠在DRAM上。据称可以减少10倍的数据搬运能耗。UPMEM则是另一种路线——在标准DRAM芯片中嵌入通用的DPU（Data Processing Unit），可以运行轻量级的计算任务。

存内计算面临的核心挑战是：DRAM工艺和逻辑工艺的差异。DRAM工艺优化的是存储密度，而逻辑工艺优化的是晶体管速度和漏电。在DRAM工艺中做逻辑运算，性能和功耗都不如逻辑工艺。所以"存内"和"近存"是一个权衡——完全存内可以获得最好的数据移动效率，但计算性能和密度受限；完全近存可以拥有更好的计算性能，但需要更多的数据传输。

踩坑故事：我在评估一款存内计算芯片时，看到厂商数据上声称"10倍能效提升"。实现中，我们发现了一个猫腻：他们对比的"基线"是一个非常老的GPU架构，而不是同代产品。当我们换上同代GPU重新对比，能效提升从10倍降到了约1.8倍——仍然不错，但远非"革命性"。这个教训告诉我们：在DSA领域，厂商的"能效对比"经常使用"弱基线"，一定要自己用公平的基线重新跑一遍测试。

25.5 DSA设计的方法论

聊了这么多具体的DSA，让我们总结一下设计一个好DSA的方法论：

  1. 识别领域的关键运算——是矩阵乘法？（AI）是卷积？（CV）是FFT？（信号处理）是图遍历？（图数据库）定位到最关键的计算模式。
  2. 设计专用数据通路——对关键运算做极致优化。如果90%的计算是矩阵乘法，那么90%的芯片面积应该服务于矩阵乘法，而不是留给"可能用到的"其他功能。
  3. 匹配存储层次——理解数据流模式：是流式的（一个token进来，一个token出去）还是随机的（需要频繁随机访问）？流式场景用large SRAM buffer就够了；随机访问场景需要更大的DRAM带宽或更智能的预取策略。
  4. 保留灵活的控制——别把芯片做得太死。AI算法在快速演进，你的DSA如果只能跑2018年的模型，到2026年就没用了。在专用化和灵活性之间找到平衡——比如可编程的互联网络、可配置的数据路径。
  5. 与通用处理器协同——任何DSA都无法独立工作。需要一个通用处理器（x86、ARM或RISC-V）来跑操作系统、调度任务、处理"长尾操作"。TPU是PCIe卡，需要CPU主机；NPU是嵌入在SoC中的，也需要CPU协调。

DSA设计的光谱：

ASIC（固定功能）← FPGA ← CGRA（粗粒度可重构阵列）← GPU ← CPU（完全通用）

越往左，能效越高、性能密度越大，但灵活性越低。一个聪明的DSA设计是找到"恰到好处"的点——在这个点上，你满足了目标领域的95%的需求，但只牺牲了5%的灵活性。

为什么那么多AI芯片初创公司失败了？我参加过几个AI芯片初创公司的技术交流会，几乎所有的创始人都能画出漂亮的架构图、写出亮眼的TOPS数字。但绝大多数失败在了软件栈上。造芯片只是DSA成功的20%，80%的工作是让芯片"好用"——这包括编译器（把PyTorch/TensorFlow模型映射到芯片上）、运行时库、调试工具、profiler、算子库。一家公司可能花了3年和1亿美元造出了一块完美的芯片，然后又花了3年和1.5亿美元把软件栈补齐。大多数初创公司倒在了"等软件栈"的这段黑暗时光里。所以，如果你问我DSA设计的最大教训是什么，我的回答是：在画第一版RTL之前，先想想软件栈怎么搞。

// 简化的Systolic Array（Weight-Stationary）伪代码
// 矩阵乘法 C[M][N] += A[M][K] × B[K][N]
// Weight-Stationary: 权重 B 固定在PE中，激活 A 流过PE阵列

module systolic_array #(
    parameter M = 4,  // 行数
    parameter N = 4,  // 列数
    parameter K = 4   // 内积维度
) (
    input  logic         clk,
    input  logic         rst_n,
    input  logic [K-1:0] activations [M-1:0], // A矩阵的行
    input  logic [K-1:0] weights     [N-1:0], // B矩阵的列
    output logic [K-1:0] results     [M-1:0]  // C矩阵的结果
);

    // PE阵列: M × N
    // 每个PE包含一个MAC（乘累加单元）和权重的本地寄存器

    logic [K-1:0] a_pipe [M-1:0][N-1:0];  // 激活值在PE中向右传递
    logic [K-1:0] w_reg  [N-1:0];          // 权重在PE中静止不变
    logic [K-1:0] c_accum[M-1:0][N-1:0];  // 部分和累加器

    // 每个时钟周期：
    // 1. 激活值a从左侧进入PE阵列
    // 2. 每个PE执行本地MAC: c += a × w
    // 3. 激活值向右传递到下一个PE

    always_ff @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            for (int i = 0; i < M; i++) begin
                for (int j = 0; j < N; j++) begin
                    c_accum[i][j] <= 0;
                end
                results[i] <= 0;
            end
        end else begin
            // 第0列：加载新的激活值
            for (int i = 0; i < M; i++) begin
                a_pipe[i][0] <= activations[i];
            end

            // 激活值逐列传递（从左侧到右侧）
            for (int j = 1; j < N; j++) begin
                for (int i = 0; i < M; i++) begin
                    a_pipe[i][j] <= a_pipe[i][j-1];
                end
            end

            // 每个PE执行MAC操作
            for (int i = 0; i < M; i++) begin
                for (int j = 0; j < N; j++) begin
                    c_accum[i][j] <= c_accum[i][j] + a_pipe[i][j] * w_reg[j];
                end
            end

            // 经过K个周期后，从底部读出结果
            // 这里简化处理：假设在K周期后读最后一列
            if (/* K周期计数器完成 */) begin
                for (int i = 0; i < M; i++) begin
                    results[i] <= c_accum[i][N-1];
                end
            end
        end
    end

    // 权重加载：初始时从外部加载
    always_ff @(posedge clk) begin
        for (int j = 0; j < N; j++) begin
            if (/* 权重加载阶段 */)
                w_reg[j] <= weights[j];
        end
    end

endmodule

// Weight-Stationary的关键洞察：
// 权重在PE中是固定的，不需要移动
// 激活值在PE之间传递，从左侧流入，右侧流出
// 部分和在PE内部累加
// 数据流是规则的、可预测的——硬连接的互连取代了复杂的路由逻辑
// 这就是为什么Systolic Array能以极低功耗获得极高吞吐量

好了，同学们，我们用了整整一章来探讨AI加速器和DSA——从Google TPU的Systolic Array到Groq的确定性数据流架构，从存内计算到DSA设计的方法论。你会发现，所有这些创新的核心思想其实是一致的：少做无用功。不做通用处理器必须做的那些"备而不用"的事（分支预测、乱序执行、Cache Coherence……），把省下来的每一颗晶体管都投入到对目标领域最有价值的计算上去。
