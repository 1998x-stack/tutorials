前言：叠盘子的人生哲学

大家好，我是你们的讲师。想一个场景：厨房里你刚洗完一摞盘子，一个一个往上叠。当你需要用盘子的时候，你会从哪拿？最上面那个——也就是最后放上去的那个。这就是"后进先出"（Last In First Out，LIFO）的日常写照。

栈（stack）就是计算机世界里的这摞盘子。它可能是所有数据结构中最简单的一个——只有两种基本操作：push（压栈）和 pop（弹栈）。但就是这个简单的结构，在计算机系统中扮演着极其重要的角色：函数调用靠它、浏览器的后退按钮靠它、编译器的语法分析也靠它。

栈的概念与生活类比

栈的核心特性用四个字概括就是"后进先出"。除了叠盘子，生活中还有很多例子：

  • 浏览器的后退按钮：你访问了 A → B → C，点击后退时回到 B（C 先被弹出）。如果再点后退回到 A，再前进又会回到 B——但如果你在 C 后退到 B 时访问了一个新页面 D，那 C 就永远回不去了（栈顶被 D 替换了）
  • 文本编辑器中的撤销 (Ctrl+Z)：最近的一次修改被放到栈顶，撤销就是弹出栈顶操作
  • 函数调用栈：函数 A 调用函数 B，B 调用 C——C 返回后回到 B，B 返回后回到 A，完美的 LIFO 顺序

理解函数调用栈对后续的学习特别重要——当你学到递归和 DFS（深度优先搜索）时，它们本质上都是在隐式或显式地使用栈。

栈的实现

在 Python 中，用 list 模拟栈是最直接的方式：

# Python list 就是天然栈
stack = []
stack.append(1)   # push → [1]
stack.append(2)   # push → [1, 2]
stack.append(3)   # push → [1, 2, 3]
top = stack.pop() # pop → 3, stack = [1, 2]
print(stack[-1])  # peek 看一眼栈顶 → 2

当然，我们也可以封装一个更正式的 Stack 类：

class Stack:
    def __init__(self):
        self._data = []

    def push(self, val):
        self._data.append(val)

    def pop(self):
        if self.is_empty():
            raise IndexError("pop from empty stack")
        return self._data.pop()

    def peek(self):
        if self.is_empty():
            raise IndexError("peek from empty stack")
        return self._data[-1]

    def is_empty(self):
        return len(self._data) == 0

    def size(self):
        return len(self._data)

用 list 模拟栈，push 和 pop 都是均摊 O(1) 的——因为底层是动态数组，偶尔需要扩容，但大多数时候都是常数时间。如果你对 O(1) 有严格需求，也可以用 collections.deque（双端队列），它在两端都是 O(1) 的。

栈的常见操作及复杂度

操作 方法 时间复杂度 说明
入栈 push(x) O(1)* 均摊 O(1)，偶尔扩容
出栈 pop() O(1) 移除栈顶元素
查看栈顶 peek() / stack[-1] O(1) 不移除
判空 is_empty() O(1) 检查 len == 0
大小 size() O(1) 返回元素个数

括号匹配 (LeetCode 20)

括号匹配是栈的"Hello World"——它最直观地展示了栈为什么有用：

def is_valid(s: str) -> bool:
    stack = []
    # 配对映射表
    pairs = {")": "(", "]": "[", "}": "{"}

    for ch in s:
        if ch in pairs:                 # 右括号
            if not stack or stack[-1] != pairs[ch]:
                return False            # 不匹配
            stack.pop()                 # 匹配，弹出左括号
        else:                           # 左括号
            stack.append(ch)

    return not stack  # 栈为空说明全部匹配

这道题的思路非常清晰：遇到左括号就入栈，遇到右括号就跟栈顶匹配——匹配上了就弹出，没匹配上或者栈空了就报错。最后如果栈不为空（说明有左括号没被匹配），也是无效的。

这是一个模板，可以推广到很多类似的场景：比如 LeetCode 856（括号的分数）、LeetCode 394（字符串解码）。核心思想都是"利用栈来跟踪需要匹配的上下文"。

表达式求值

表达式求值是栈的另一个经典应用。计算机不像人一样能一眼看出运算符优先级——它需要一种无歧义的表示方法。

中缀转后缀（逆波兰表达式）

我们平时写的 3 + 4 * 5 是中缀表达式。计算机更喜欢后缀表达式（逆波兰表示法，RPN）：3 4 5 * +。转换算法使用两个栈——一个放操作数，一个放运算符：

def infix_to_postfix(expression):
    precedence = {"+": 1, "-": 1, "*": 2, "/": 2}
    output = []
    ops = []

    for token in expression.split():
        if token.isdigit():
            output.append(token)
        elif token in precedence:
            while (ops and ops[-1] != "("
                   and precedence[ops[-1]] >= precedence[token]):
                output.append(ops.pop())
            ops.append(token)
        elif token == "(":
            ops.append(token)
        elif token == ")":
            while ops and ops[-1] != "(":
                output.append(ops.pop())
            ops.pop()  # 弹出 ( 丢弃

    while ops:
        output.append(ops.pop())

    return " ".join(output)

print(infix_to_postfix("3 + 4 * 5"))  # 3 4 5 * +

后缀表达式求值

def eval_rpn(tokens):
    stack = []
    for token in tokens:
        if token in {"+", "-", "*", "/"}:
            b = stack.pop()
            a = stack.pop()
            if token == "+":
                stack.append(a + b)
            elif token == "-":
                stack.append(a - b)
            elif token == "*":
                stack.append(a * b)
            else:
                stack.append(int(a / b))  # 向零取整
        else:
            stack.append(int(token))
    return stack[0]

这个算法的时间复杂度是 O(n)，空间复杂度是 O(n)。面试中如果被问到表达式求值，通常不需要你完整实现中缀转后缀，直接用双栈法处理中缀表达式即可——一个栈放数字，一个栈放运算符。

单调栈 (Monotonic Stack)

单调栈是栈的一种进阶用法——栈内元素保持单调递增或单调递减。它是解决"下一个更大元素""上一个更小元素"这类问题的利器。

每日温度 (LeetCode 739)

题目：给定一个温度列表，对于每一天，输出还要等多少天才能等到更高的温度。如果没有，输出 0。

def daily_temperatures(temperatures):
    n = len(temperatures)
    result = [0] * n
    stack = []  # 单调递减栈，存索引

    for i, t in enumerate(temperatures):
        # 当前温度比栈顶对应的温度高 → 找到了答案
        while stack and t > temperatures[stack[-1]]:
            prev_index = stack.pop()
            result[prev_index] = i - prev_index
        stack.append(i)

    return result

理解这个算法的关键：栈中保存的是"还没有找到更高温度"的索引，且这些索引对应的温度是单调递减的。当一个新的高温出现时，它依次解决栈中所有比它低的温度。

柱状图中最大矩形 (LeetCode 84)

这是单调栈的进阶应用：

def largest_rectangle_area(heights):
    stack = [-1]  # 哨兵，方便计算宽度
    max_area = 0
    heights.append(0)  # 末尾哨兵，保证最后能触发计算

    for i, h in enumerate(heights):
        while stack and heights[stack[-1]] > h:
            height = heights[stack.pop()]
            width = i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)

    heights.pop()
    return max_area

这道题的难度在于你要理解"以每个柱形为高度的最大矩形"如何计算——左边找第一个比它矮的，右边找第一个比它矮的，宽度就是两者之间的距离。单调栈恰好能高效地找到左右两边的"第一个更矮"。这个思路如果你第一次见觉得很难，非常正常——我当初也花了整整一下午来理解。

💡 小技巧：在算法题中，如果你看到"最近的"、"下一个更大的"、"上一个更小的"这些关键词，立刻想到单调栈。具体来说，"下一个更大"用单调递减栈，"下一个更小"用单调递增栈。记住这个对应关系，能帮你快速定位解题方向。

本章小结

栈虽然简单，但它的应用范围极广——从括号匹配到表达式求值，从函数调用到深度优先搜索。特别是单调栈这个进阶技巧，它把看似复杂的问题（"下一个更大元素"）变成了一个 O(n) 就能解决的简单问题。

下一章我们将学习栈的"孪生兄弟"——队列。它和栈正好相反：先进先出。你会发现，这两种看似对立的存取顺序，各自在计算机科学中占据着不可替代的位置。
