第8章：STM32 CAN外设：bxCAN控制器介绍、CAN外设时钟配置、GPIO复用功能配置

好，咱们今天来聊聊STM32的CAN外设。很多初学者一看到芯片手册里那一大堆寄存器，头就大了。其实没那么复杂，你把它拆开来看，就三个核心问题：用什么硬件、给它什么时钟、怎么连到外面去。搞明白这三件事，CAN通信的底层就算打通了。

8.1 bxCAN控制器：ST的“黑盒子”

STM32里集成的CAN控制器，官方叫bxCAN。这个“bx”是“Basic Extended”的缩写，说白了就是支持标准帧和扩展帧。我个人习惯把它看作一个智能的“黑盒子”——你只管把数据丢进去，它自动帮你完成帧封装、位填充、CRC校验、仲裁这些脏活累活。

bxCAN有几个关键特性，我挑重要的说：

  • 支持CAN 2.0A和2.0B：标准帧（11位ID）和扩展帧（29位ID）都能处理。现在绝大多数车载网络都用扩展帧，但老一些的J1939设备可能还在用标准帧，兼容性你得心里有数。
  • 3个发送邮箱：可以缓存3个待发送的报文。发送优先级由硬件自动仲裁，你不需要操心。
  • 2个接收FIFO：每个FIFO深度为3个报文。也就是说，最多可以缓存6个接收报文而不丢失。我在项目里遇到过接收溢出导致丢帧的情况，后来发现是FIFO满了，而我的应用层还没来得及读走数据。嗯，这里要注意：接收中断服务程序一定要写得快。
  • 28个可配置的筛选器组：这是bxCAN最强大的功能之一。你可以设置筛选规则，只接收你关心的ID，其他报文一律过滤掉。这能大大减轻CPU的负担。

核心要点：bxCAN本质上是一个独立于CPU运行的硬件状态机。它有自己的时钟域，通过APB总线与CPU交互。你配置好寄存器后，它就开始自主工作。

8.2 CAN外设时钟配置：别让时钟“饿着”

任何外设要工作，首先得有时钟。CAN外设挂在APB1总线上（对于STM32F1/F4系列）。时钟源通常来自HSE（外部高速晶振）经过PLL倍频后的系统时钟，再经过APB1预分频器得到。

为什么我要强调这个？因为CAN通信的位时序（Bit Timing）完全依赖于这个时钟。如果时钟不准，你的波特率就会漂移，通信就会出错。我曾经在一个项目中，因为APB1时钟配置错了，导致CAN波特率从500kbps变成了480kbps，结果和ECU怎么都连不上，排查了整整一天……

配置时钟的步骤其实很简单：

  1. 使能CAN外设时钟：通过RCC寄存器，比如RCC_APB1PeriphClockCmd(RCC_APB1Periph_CAN1, ENABLE);
  2. 配置CAN的位时序参数：这包括BS1（时间段1）、BS2（时间段2）和SJW（同步跳转宽度）。这些参数决定了最终的波特率。

波特率的计算公式是：

BaudRate = CAN_Clock / ( (1 + BS1 + BS2) * Prescaler )

举个例子，假设APB1时钟是36MHz，你想要500kbps的波特率，通常可以这样配：

CAN_InitStructure.CAN_SJW = CAN_SJW_1tq;      // 同步跳转宽度为1个时间量子
CAN_InitStructure.CAN_BS1 = CAN_BS1_8tq;      // 时间段1为8个时间量子
CAN_InitStructure.CAN_BS2 = CAN_BS2_7tq;      // 时间段2为7个时间量子
CAN_InitStructure.CAN_Prescaler = 4;           // 预分频系数为4
// 计算：36MHz / ( (1 + 8 + 7) * 4 ) = 36MHz / 64 = 562.5kbps
// 嗯，这个值不太对，说明需要调整参数。

你看，实际计算时，需要反复调整BS1、BS2和Prescaler，让结果尽量接近目标值。我建议你使用ST官方提供的CAN波特率计算工具，或者直接查芯片手册里的推荐配置表，省时省力。

个人经验：对于500kbps的常用波特率，在36MHz APB1时钟下，我常用的配置是：Prescaler=4, BS1=9tq, BS2=8tq, SJW=1tq。这样算出来正好是500kbps。你可以直接拿去用。

8.3 GPIO复用功能配置：把信号引出来

时钟配好了，控制器也初始化了，但信号怎么从芯片引脚出去呢？这就涉及到GPIO复用功能。CAN的发送引脚（TX）和接收引脚（RX）必须连接到特定的GPIO引脚上。

以STM32F103为例，CAN1的引脚通常是：

  • CAN1_TX：PA12（复用推挽输出）
  • CAN1_RX：PA11（浮空输入或上拉输入）

配置GPIO的步骤，我一般这样写：

GPIO_InitTypeDef GPIO_InitStructure;

// 使能GPIOA时钟
RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);

// 配置CAN1_TX (PA12) 为复用推挽输出
GPIO_InitStructure.GPIO_Pin = GPIO_Pin_12;
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;   // 复用推挽输出
GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
GPIO_Init(GPIOA, &GPIO_InitStructure);

// 配置CAN1_RX (PA11) 为上拉输入
GPIO_InitStructure.GPIO_Pin = GPIO_Pin_11;
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;     // 上拉输入
GPIO_Init(GPIOA, &GPIO_InitStructure);

这里有几个坑，我踩过，你注意一下：

  • TX引脚必须配置为复用推挽输出，不能是普通推挽输出。否则CAN控制器无法控制引脚电平。
  • RX引脚建议配置为上拉输入。虽然浮空输入也能工作，但上拉可以保证在总线空闲时，RX引脚保持高电平，避免误触发。
  • 不同型号的STM32，CAN引脚可能不同。比如STM32F4系列，CAN1的引脚可能是PB8（RX）和PB9（TX）。一定要查数据手册的“Alternate function mapping”表格。

避坑指南：我曾经在一个项目中，因为偷懒没查手册，直接复制了F1的代码到F4上，结果CAN死活不通。后来发现F4的CAN1引脚映射到了PB8/PB9，而我配置的是PA11/PA12。所以，换芯片型号时，务必重新确认引脚复用功能。

8.4 完整的初始化流程

把上面三块内容串起来，一个完整的CAN初始化流程大概是这样的：

  1. 使能时钟：RCC使能CAN外设时钟和对应GPIO时钟。
  2. 配置GPIO：将TX和RX引脚设置为正确的复用模式。
  3. 配置CAN控制器：设置波特率、工作模式（正常/环回/静默）、筛选器等。
  4. 使能CAN：调用CAN_Cmd(CAN1, ENABLE);启动CAN外设。

代码示例（简化版）：

void CAN_Config(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;
    CAN_InitTypeDef CAN_InitStructure;

    // 1. 使能时钟
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_CAN1, ENABLE);

    // 2. 配置GPIO
    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_12;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_11;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    // 3. 配置CAN控制器
    CAN_DeInit(CAN1);
    CAN_StructInit(&CAN_InitStructure);
    CAN_InitStructure.CAN_TTCM = DISABLE;
    CAN_InitStructure.CAN_ABOM = ENABLE;   // 自动离线管理
    CAN_InitStructure.CAN_AWUM = ENABLE;   // 自动唤醒
    CAN_InitStructure.CAN_NART = DISABLE;  // 禁止自动重传
    CAN_InitStructure.CAN_RFLM = DISABLE;  // 接收FIFO锁定模式
    CAN_InitStructure.CAN_TXFP = DISABLE;  // 发送优先级由ID决定
    CAN_InitStructure.CAN_Mode = CAN_Mode_Normal; // 正常模式
    CAN_InitStructure.CAN_SJW = CAN_SJW_1tq;
    CAN_InitStructure.CAN_BS1 = CAN_BS1_9tq;
    CAN_InitStructure.CAN_BS2 = CAN_BS2_8tq;
    CAN_InitStructure.CAN_Prescaler = 4;
    CAN_Init(CAN1, &CAN_InitStructure);

    // 4. 使能CAN
    CAN_Cmd(CAN1, ENABLE);
}

你看，其实就这么几步。你想想看，只要时钟对了、引脚对了、参数对了，CAN通信的硬件基础就搭好了。剩下的就是发送和接收的逻辑了。

总结一下：bxCAN控制器是核心，时钟配置是基础，GPIO复用是桥梁。这三者缺一不可。我建议你在实际调试时，先用逻辑分析仪抓一下CAN_TX和CAN_RX引脚的波形，确认时钟和波特率是否正确。这是最快定位问题的方法。
