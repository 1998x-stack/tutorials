开篇：代码复制粘贴的痛苦

想象这个场景：你需要写一个函数，找出数组中的最大值。对i32写一个版本，对f64写一个版本，对String再写一个版本——三个函数除了类型不同，逻辑完全一样。这个场景我在初学Rust时亲身经历过。

我当时为一个统计库写了5个版本的max函数，分别处理i32、i64、f32、f64和u64。直到我的导师看到代码，只说了一句："你听说过泛型吗？"

泛型（Generics）是Rust乃至所有现代静态语言的核心能力——它让你编写适用于多种类型的代码，而无需为每个类型重复编写。Rust的Vec<T>、Option<T>、Result<T, E>都是泛型的产物。这一章我们就来揭开泛型的面纱。

泛型函数

泛型函数允许你在函数签名中使用类型参数，用尖括号<>声明：

fn largest<T>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list {
        if item > largest {   // 这段代码暂时无法编译
            largest = item;
        }
    }
    largest
}

上述代码中T就是一个类型参数——调用时可以用任何类型替换：

fn main() {
    let numbers = vec![34, 50, 25, 100, 65];
    let result = largest(&numbers);   // T 被推断为 i32
    println!("最大值是 {result}");

    let chars = vec!['y', 'm', 'a', 'q'];
    let result = largest(&chars);     // T 被推断为 char
    println!("最大值是 {result}");
}

不过上面的largest函数其实编译不过——因为T必须实现PartialOrd trait才能使用>运算符。这引出了泛型约束的概念，我们稍后讲到。

命名惯例：类型参数通常使用单个大写字母——T（Type）、E（Error）、K（Key）、V（Value）。在复杂场景中可以用更具描述性的名称，比如TItem或蛇形命名全称。

泛型结构体

结构体也可以泛型化——将类型参数应用到字段类型上：

// 单类型参数
struct Point<T> {
    x: T,
    y: T,
}

// 多类型参数
struct Pair<K, V> {
    key: K,
    value: V,
}

impl<T> Point<T> {
    fn x(&self) -> &T {
        &self.x
    }
}

// 特定类型的impl
impl Point<f64> {
    fn distance_from_origin(&self) -> f64 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}

注意最后一段代码：我们为Point<f64>单独实现了distance_from_origin方法，这个实现只对f64版本的Point有效。这就是泛型的强大之处——你可以为所有类型做统一实现，也可以为特定类型做特殊实现。

泛型约束：Trait Bound

回到之前的largest函数。为了让T支持>比较，我们需要约束T必须实现PartialOrd trait：

fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list {
        if item > largest {
            largest = item;
        }
    }
    largest
}

T: PartialOrd就是一个Trait Bound（trait约束），它告诉编译器："只有实现了PartialOrd的类型才能用作T"。

约束语法

语法 含义 示例
T: Trait 单一约束 T: Display
T: Trait1 + Trait2 多重约束（交集） T: Clone + Debug
where T: Trait where子句（约束较多时） fn f<T>() where T: Clone + Debug
T: 'a 生命周期约束 T: 'static

当约束很多时，where子句让代码更清晰：

fn complex<T, U>(t: T, u: U) -> String
where
    T: Display + Clone,
    U: Display + Debug,
{
    format!("t: {t}, u: {u:?}")
}
impl Trait语法：如果类型参数只在一个位置出现，可以用impl Trait简写：fn print(val: impl Display)等价于fn print<T: Display>(val: T)。但前者不能用于指定返回类型（因为调用者无法推断具体类型）。我们会在第12章Trait中详细展开。

性能与Monomorphization

很多人不愿意用泛型，是担心"泛型有运行时开销"。在Rust中，这个担心完全多余。Monomorphization（单态化）是Rust编译器处理泛型的方式——在编译时为每个具体类型生成独立的代码副本。

比如你写了：

let v1: Vec<i32> = vec![1, 2, 3];
let v2: Vec<String> = vec!["a".into(), "b".into()];

编译器实际上生成了两份Vec实现：一份用于i32，一份用于String。同样的，Option<i32>和Option<String>也是不同的类型，各有独立的内存布局。

零成本抽象的含义：Rust的泛型不会导致任何运行时开销（没有虚函数表查找、没有类型擦除的装箱）。但代价是编译时间增加和二进制体积增大（代码膨胀）。对于性能和体积敏感的场景（如嵌入式开发），可以通过trait object（第12章）来减少代码膨胀——但会引入动态分发的开销。在服务器端应用场景中，代码膨胀很少成为问题。

关联类型（Associated Types）

关联类型是trait内部定义的"占位类型"，它和泛型参数有些相似但有本质区别：

// 使用泛型参数
trait MyIterator<T> {
    fn next(&mut self) -> Option<T>;
}

// 使用关联类型
trait MyIterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}

区别在于：

  • 泛型参数：一个类型可以对同一个trait实现多次（不同泛型参数的实例化），例如impl MyIterator<i32> for Foo和impl MyIterator<String> for Foo
  • 关联类型：一个类型只能对定义了关联类型的trait实现一次，因为type Item是"该类型唯一的"

关联类型让代码更简洁，因为你不必在每次使用trait时都写上类型参数。标准库的Iterator trait就使用关联类型：

// 实现时只需指定一次Item类型
impl Iterator for MyCounter {
    type Item = u32;      // 关联类型
    fn next(&mut self) -> Option<Self::Item> {
        // ...
    }
}

// 使用时不用重复写类型
let sum: u32 = my_counter.sum();

如果用泛型参数版本的Iterator：let sum: u32 = my_counter.sum::<u32>()——更啰嗦。

什么时候用关联类型 vs 泛型参数？简单规则：如果trait的一个类型对于特定实现是"唯一的"（如一个迭代器只能产生一种类型的元素），使用关联类型。如果类型可以是多个（如转换函数From<T>，输入类型可以是多种），使用泛型参数。

本章小结

泛型是Rust类型系统的核心支柱之一。这一章我们学习了泛型函数、泛型结构体、trait约束的语法，了解了单态化带来的零成本抽象，以及关联类型与泛型参数的区别。

泛型单独使用只是热身。下一章我们要看的是Rust中最重要、最核心的抽象机制——Trait。它是对"行为"的类型化描述，是泛型约束的基础，也是Rust实现多态的方式。你可以把trait理解为Rust版的"接口"——但它远比接口更灵活、更强大。第12章，我们正式进入Trait的世界。
