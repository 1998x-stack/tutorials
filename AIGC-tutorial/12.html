第12章 采样策略与调度

大家好，我是你们的讲师。如果你用过 ComfyUI 或 Automatic1111 WebUI，你一定见过采样器下拉菜单中那一长串名字：Euler、Euler a、Heun、DPM++ 2M、DPM++ SDE、DDIM、UniPC……面对二十多种采样器，初学者往往会不知所措。

这一章，我们将从数学视角理解这个问题的本质：扩散模型的采样过程本质上是在解一个常微分方程（ODE）或随机微分方程（SDE）。不同的采样器只是用不同的数值方法去"离散化"求解这个微分方程。理解了这一点，你就能理性地选择采样器、步数和参数，而不是靠试错。

12.1 ODE 视角下的扩散逆过程

扩散模型的反向过程可以看作是一个"概率流 ODE"（Probability Flow ODE）的积分过程。DDIM 论文揭示了一个关键洞察：扩散模型的反向去噪过程，本质上是在求解一个从噪声分布到数据分布的确定性 ODE。

这个概率流 ODE 的形式如下：

dx / dt = f(t) · x - (1/2) · g(t)² · ∇_x log p_t(x)

其中 f(t) 和 g(t) 由噪声调度 β_t 决定
∇_x log p_t(x) 是"得分函数"（score function），由 ε_θ 近似: ∇_x log p_t(x) ≈ - ε_θ(x_t, t) / √(1 - ᾱ_t)

这个视角的革命性在于：它将"随机去噪"转化为了"确定性 ODE 求解"问题。一旦是 ODE，我们就可以使用各种高效的数值求解器——Euler 法、Runge-Kutta 法、多步法等等。

🌟 核心洞察：把采样看作 ODE 求解后，离散化步数和求解器精度就成了两个独立的维度。步数少但求解器精度高（如 DPMSolver++ 2M）可以比步数多但求解器简单（如 Euler）取得更好的效果。这就像用高阶 Runge-Kutta 方法可以比 Euler 方法用更少的步数达到同样的精度。

12.2 离散化求解器对比

主流的采样求解器可以分为几大类：

求解器类型推荐步数随机性特点
DDIM隐式欧拉20-100确定性经典、稳定、可插值
Euler一阶显式20-50确定性最简单的 ODE 求解器
Euler a一阶加噪声20-50随机性（祖先采样）Euler + 每一步加噪声
Heun二阶（改进欧拉）20-30确定性精度高于 Euler，但需要两倍计算
DPM++ 2M二阶多步10-30确定性效率高、推荐日常使用
DPM++ SDE二阶 + SDE10-30随机性细节更丰富，但稍微慢一些

DPM++ 系列求解器（特别是 DPM++ 2M 和 DPM++ SDE）是当前最主流的采样器。它们基于扩散模型的特殊结构设计了专门的 ODE 求解公式，比通用的 ODE 求解器（如 Euler、Heun）在相同步数下精度更高。

12.3 Ancestral Sampling：引入随机性的采样器

在 DPM++ 等确定性求解器中，每一步的更新是确定的——给定相同的初始噪声，生成结果完全相同。但有些采样器（名称中带 "a" 或 "SDE"）会在每一步采样中加入随机噪声。

Ancestral Sampling 的核心思想是：在每一步更新后，再加入一个小的随机扰动：

# Euler a 的简化流程（带祖先噪声）
for t in reversed(range(steps)):
    # 1. 预测噪声
    pred = ε_θ(x_t, t)
    # 2. 沿梯度方向走一步（确定性更新）
    x_mean = x_t + step_size(pred, t)
    # 3. 加入随机噪声（祖先采样）
    x_{t-1} = x_mean + σ_t · ε,  ε ∼ N(0, I)

加入随机噪声的效果是：

  • 更高的多样性：同一 prompt 和 seed 也可能产生不同的结果
  • 更好的细节：随机性有助于模型跳出"局部最优"，产生更丰富的纹理
  • 代价：某些场景下会引入不必要的噪声导致质量下降
💡 实用建议：选择确定性采样器（DPM++ 2M、DDIM）还是随机性采样器（Euler a、DPM++ SDE）取决于你的目标：如果你在批量生成需要可复现的结果（比如生产环境），用确定性采样器。如果你在做艺术探索希望更多的"意外惊喜"，用随机性采样器。

12.4 步数 vs 质量

步数和质量的关系是采样中最重要的权衡。我们可以用一张对比表来直观展示：

从表中可以总结出几个实用规则：

  • 速度优先（SDXL）：DPM++ 2M Karras, 15-20 步
  • 质量优先（SD 1.5）：DPM++ 2M Karras, 30-50 步
  • 极速（实时场景）：LCM 或 SDXL Turbo, 1-4 步
  • 艺术探索：Euler a 或 DPM++ SDE, 20-30 步（利用随机性产生多样性）

12.5 LCM：蒸馏扩散模型为几步生成

LCM（Latent Consistency Model）是 2023 年提出的一种"扩散模型蒸馏"方法，目标是将预训练的扩散模型蒸馏为只需要 1-4 步就能生成的"一致性模型"。

LCM 的核心思想是：训练一个网络 f_θ(x_t, t) 直接输出 x₀ 的估计（或者跳过所有中间步骤），使得从任意时间步 t 出发，一步就能跳回到 x₀。

训练 LCM 的损失函数：

L_CM = E[ λ(t) · d( f_θ(x_{t_{n+1}}, t_{n+1}), f_θ⁻(x̂_t^{φ}, t_n) ) ]

其中：
  - f_θ⁻ 是目标网络（EMA 更新）
  - x̂_t^{φ} 是使用数值求解器从 t_{n+1} 到 t_n 的一步估计
  - d(·) 是距离函数（通常使用 MSE 或 LPIPS）

LCM 的惊艳之处在于：

  • 超低延迟：1-4 步生成，比原始扩散模型快 250-1000 倍
  • 无需特殊硬件：蒸馏在普通 GPU 上即可完成，生成的 LCM LoRA 可以叠加在其他模型上
  • 兼容性：LCM LoRA 权重可以加载到绝大多数 SD 1.5 和 SDXL 模型上
⚠️ 真实的踩坑故事：有次我把 LCM 用在了一个微调的动漫风格模型上，结果生成的人物脸部出现了奇怪的"蜂窝状"纹理。起初我以为是模型出了问题，后来发现 LCM 在蒸馏过程中对"高频细节"的建模不充分——LCM 擅长捕捉全局结构但牺牲了局部纹理的多样性。解决方法是使用 LCM 生成一张"底图"，然后用 ControlNet Tile + 原始采样器做精细化的方式，或者使用更新的 TCD（Trajectory Consistency Distillation）方法。这个教训是：LCM 适合快速原型验证和实时交互场景，但追求最高质量时还是需要用全步数采样。
