开篇：一段200行变20行的神奇经历

有一次我需要从一个日志文件中提取所有错误行，计算每种错误类型出现的次数，然后按频率排序。刚学Rust的时候，我会这样写：先创建一个空的Vec，逐行读取，用if判断是否包含"ERROR"，手动解析出错误类型，插入HashMap，再转成Vec排序，最后打印——大概200行。

后来学会了闭包和迭代器，同样的逻辑变成了这样：

use std::{fs, collections::HashMap};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let content = fs::read_to_string("app.log")?;
    let mut counts: HashMap<String, u32> = HashMap::new();

    content
        .lines()
        .filter(|line| line.contains("ERROR"))
        .filter_map(|line| line.split_whitespace().nth(2))
        .for_each(|err_type| {
            *counts.entry(err_type.to_string()).or_insert(0) += 1;
        });

    let mut sorted: Vec<_> = counts.into_iter().collect();
    sorted.sort_by(|a, b| b.1.cmp(&a.1));

    for (err, count) in &sorted {
        println!("{err}: {count}");
    }
    Ok(())
}

不到30行，每个步骤的意图清晰可读。这就是闭包和迭代器的魅力——让你的代码从"描述怎么做"变成"描述要什么"。

闭包语法

闭包（Closure）是可以捕获其环境的匿名函数。与普通函数不同，闭包可以从定义它的作用域中"捕获"变量。

基本语法

// 完整形式：|参数列表| -> 返回类型 { 表达式 }
let add = |a: i32, b: i32| -> i32 { a + b };

// 类型推断（大多数情况下可省略）
let add = |a, b| a + b;

// 不带参数的闭包
let hello = || println!("hello");

// 调用闭包
println!("{}", add(2, 3));

闭包还有一个关键特征：类型推断。编译器会根据第一次调用时传入的参数类型来推断闭包的参数类型。如果你在同一个闭包上尝试传不同类型，编译器会报错。

捕获环境变量

let name = String::from("Alice");

// 闭包捕获了 name
let greet = || println!("你好，{name}");

greet();  // 输出：你好，Alice
// greet() 之后 name 仍然可用（闭包只用了不可变引用）
println!("{name}");

闭包捕获变量的方式取决于闭包体如何使用变量：

捕获方式 trait 行为 闭包外变量状态
不可变引用 Fn 只读取变量 闭包外仍然可用
可变引用 FnMut 修改变量 闭包外可用
获取所有权 FnOnce 消耗变量（移动所有权） 闭包外不可用 
let mut count = 0;

// Fn：不可变捕获
let print_count = || println!("{count}");

// FnMut：可变捕获
let increment = || count += 1;

// FnOnce：所有权捕获（使用 move 关键字）
let name = String::from("Bob");
let consume = move || {
    println!("{name}");  // name 的所有权被移入闭包
};  // 这里 name 不再可用
// println!("{name}");  // 编译错误！

Fn/FnMut/FnOnce：三种捕获方式

Rust中闭包实现了三个trait之一（或几个）：

  • FnOnce：所有闭包都实现了FnOnce，因为它可以被调用至少一次。如果闭包移动了捕获变量的所有权（使用move或隐式所有权捕获），它就只能被调用一次。
  • FnMut：不移动捕获变量，但可能修改它们。闭包可以被多次调用。
  • Fn：不移动也不修改捕获变量（只读）。闭包可以被多次调用，甚至可以同时共享。
move关键字什么时候用？当你需要把闭包传给另一个线程或存储到需要在当前作用域之外存活的地方时——比如thread::spawn(|| ...)，必须使用move强制所有权转移到闭包中。否则，闭包捕获的是引用，引用的数据可能在闭包执行前就被释放了。

函数参数可以通过泛型约束指定闭包的捕获方式：

fn call_once<F>(f: F) where F: FnOnce() {
    f();
}

fn call_mut<F>(mut f: F) where F: FnMut() {
    f();
    f();
}

fn call_fn<F>(f: F) where F: Fn() {
    f();
    f();
    f();  // Fn 闭包可以安全地调用多次
}

Iterator trait

迭代器是Rust函数式编程的另一大支柱。Iterator trait非常简洁：

pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}

只要实现next方法，你就拥有了一整套迭代器适配器（map、filter、fold等等）。

基础消费者

let v = vec![1, 2, 3, 4, 5];

// collect：将迭代器收集到集合中
let doubled: Vec<i32> = v.iter().map(|x| x * 2).collect();

// sum：求和
let sum: i32 = v.iter().sum();

// count：计数
let count = v.iter().count();

// fold：折叠
let product = v.iter().fold(1, |acc, x| acc * x);

迭代器适配器

迭代器适配器（Adapter）是返回新迭代器的方法，它们惰性求值——只有被消费者（如collect、sum、for循环）消耗时才会真正执行。

适配器 作用 示例
map 对每个元素应用变换 iter.map(|x| x + 1)
filter 保留满足条件的元素 iter.filter(|x| x > 0)
filter_map 过滤 + 变换二合一 iter.filter_map(|x| x.parse().ok())
flat_map 每个元素展开为迭代器然后拼接 iter.flat_map(|x| x.split(','))
take(n) 取前n个元素 iter.take(5)
skip(n) 跳过前n个元素 iter.skip(5)
enumerate 附带索引（返回(usize, T)） iter.enumerate()
zip 将两个迭代器配对 a.iter().zip(b.iter())
chain 连接两个迭代器 a.iter().chain(b.iter())
cloned 克隆每个元素 iter.cloned()
copied 复制每个元素（需Copy） iter.copied()
惰性求值的陷阱：迭代器适配器是惰性的——如果你写了iter.map(|x| expensive(x))但没有加.collect()或for循环，expensive操作永远不会执行。编译器甚至会给出warning: unused Map that must be used警告。所以请记住：适配器只是"描述"，消费者才真正"执行"。

链式调用的力量

真正体现Rust迭代器优雅的地方在于链式调用：

let data = vec![
    "1", "2", "abc", "3", "def", "4",
];

let result: Vec<i32> = data
    .iter()
    .filter_map(|s| s.parse::<i32>().ok())  // 跳过无法解析的
    .map(|n| n * 2)                          // 每个乘以2
    .filter(|n| n > 5)                       // 只保留大于5的
    .collect();                              // 收集为Vec

println!("{result:?}");  // [6, 8]

等效的命令式代码需要临时变量、循环、if语句——而迭代器链式风格把每个步骤清晰分离。

性能提示：你可能会担心链式调用产生大量中间分配。Rust的迭代器适配器是零成本抽象——每个适配器只是一个结构体，不分配堆内存。编译器会尽可能将整个链式调用内联展开为高效的循环代码。事实上，很多情况下迭代器代码的性能超过了手写的手工循环代码，因为编译器更容易对无分支的链式调用做优化。

自定义迭代器

你可以在自己的类型上实现Iterator trait：

struct Fibonacci {
    current: u64,
    next: u64,
}

impl Fibonacci {
    fn new() -> Self {
        Fibonacci { current: 0, next: 1 }
    }
}

impl Iterator for Fibonacci {
    type Item = u64;

    fn next(&mut self) -> Option<Self::Item> {
        let current = self.current;
        self.current = self.next;
        self.next = current + self.next;
        Some(current)  // 永远不停止（无限迭代器）
    }
}

fn main() {
    let fib: Vec<u64> = Fibonacci::new()
        .take(20)     // 取前20个
        .collect();

    println!("前20个斐波那契数: {fib:?}");
}

一旦实现了Iterator，你的类型就自动获得了map、filter、fold等几十种适配器方法——这是trait的"组合威力"的最好证明。

你还可以实现IntoIterator让你的自定义集合可以被for循环遍历：

struct MyCollection<T>(Vec<T>);

impl<T> IntoIterator for MyCollection<T> {
    type Item = T;
    type IntoIter = std::vec::IntoIter<T>;

    fn into_iter(self) -> Self::IntoIter {
        self.0.into_iter()
    }
}
三种迭代范式：.iter()产生&T（不可变引用），.iter_mut()产生&mut T（可变引用），.into_iter()产生T（获取所有权）。选择哪种取决于你是否需要在迭代后继续使用原集合。

本章小结

闭包和迭代器是Rust函数式编程的双引擎。闭包让你可以灵活地捕获环境并传递给高阶函数；迭代器提供了一套声明式的数据处理管道，既简洁又高效。我们学习了闭包的三种捕获方式（Fn/FnMut/FnOnce）、迭代器的核心适配器和消费者、以及如何实现自定义迭代器。
