引子：一个中文分词工具的血泪史

之前我在做一个中文分词工具时，需要从一个大字符串中提取子串。我随手写下了&s[0..n]，结果程序在某些特定输入上崩溃了——原因是中文字符是多字节的，我按字节索引把一个"好"字从中间切开了。在Python里这不会出问题（Python的切片是字符级别的），在Rust中这就是panic。

Rust的字符串切片不允许跨越字符边界——这不是bug，是feature。它强迫你正视"字符串是UTF-8编码的字节序列"这个事实。很多语言把这个问题藏了起来，代价就是性能损失和偶尔的奇怪行为。Rust选择让你直面它，但同时也提供了足够强大的工具来处理它。

这一章，我们来彻底搞懂Rust中的字符串——String和&str，以及更通用的切片概念。

字符串切片 &str

基本切片语法

切片（slice）是Rust中一个核心概念——它是对一个集合中一段连续元素的不可变引用。

let s = String::from("hello world");

let hello = &s[0..5];   // "hello"
let world = &s[6..11];  // "world"
let whole = &s[..];     // 整个字符串

切片语法使用..范围表达式：

  • 0..5 从索引0到5（不包含5）
  • 0..=5 从索引0到5（包含5）
  • ..5 从开头到索引5（等价于 0..5）
  • 5.. 从索引5到末尾
  • .. 整个序列

字符串切片的类型是&str（读作"string slice"）。注意它和String的区别：&str是一个引用，不拥有数据，只是对原始字符串中某一段的"视图"。

字符串字面量就是 &str

你一直在用的字符串字面量"hello"其实就是一个&str：

let s: &str = "hello world";  // 字符串字面量，类型就是 &str
// 它被硬编码在编译后的二进制文件中，静态存活
切片就是不可变引用：因为&str是对String中某段数据的引用，所以它遵循所有权的借用规则——你不能在拥有不可变切片的同时修改原始字符串。这保证了切片始终指向有效、不会被篡改的数据。

String类型

String是Rust标准库中堆分配的、可增长的、UTF-8编码的字符串类型。它拥有其数据的所有权。

创建String

// 从字符串字面量创建
let s1 = String::from("hello");
let s2 = "world".to_string();

// 空字符串
let mut s3 = String::new();

// 重复字符
let s4 = "ha".repeat(3);  // "hahaha"

追加与拼接

let mut s = String::from("hello");
s.push_str(", world");  // 追加字符串切片
s.push('!');            // 追加单个字符
println!("{s}");        // "hello, world!"

// 用 + 拼接（注意：+ 会拿走左侧String的所有权）
let s1 = String::from("Hello, ");
let s2 = String::from("world!");
let s3 = s1 + &s2;  // s1被move了，之后不能再使用
println!("{s3}");    // "Hello, world!"

// 用 format! 宏拼接（更灵活，不获取所有权）
let s1 = String::from("Hello");
let s2 = String::from("world");
let s3 = format!("{s1}, {s2}!");
println!("{s3}");    // "Hello, world!"
// s1、s2都还能用

+运算符去掉了左侧String的所有权，这初看很奇怪，但背后有设计考量：它避免了不必要的复制。如果你还想用原来的String，用format!宏吧。

&str vs String

这两个类型是Rust初学者最大的混淆点之一。让我们厘清：

维度 String &str
所有权拥有数据借用数据
可变性可增长、修改不可变（只读视图）
存储位置堆上分配字符串字面量在静态区，切片的引用指向堆/静态区
运行时开销分配、释放、扩容零额外开销
何时使用需要拥有或修改字符串时只需要读取字符串内容时

黄金法则：函数参数用&str而不是&String。原因是&str既可以接收字符串字面量，也可以接收&String（Rust会自动解引用转换）：

fn takes_str(s: &str) {
    println!("{s}");
}

let s = String::from("hello");
takes_str(&s);        // &String 自动转为 &str
takes_str("world");   // &str 直接传入

如果你用&String做参数，调用者必须传一个String的引用——字符串字面量传不进去。所以，记住：参数用&str，需要所有权和可变性时用String。

String和&str的转换：
  • &str → String：s.to_string() 或 String::from(s)
  • String → &str：&s 或 &s[..]（Rust自动解引用）

UTF-8与字符串索引

这是Rust字符串中最容易"踩坑"的区域。

为什么String不支持索引

let s = String::from("你好");
// let c = s[0];  // 编译错误！String不能通过索引访问

为什么？因为Rust的字符串是UTF-8编码的，一个"字符"可能占用1到4个字节。"你"在UTF-8中占3个字节，s[0]应该返回什么？第一个字节（不完整）？第一个Unicode字符（'你'）？还是第一个字素簇（视觉上的"字"）？

Rust选择了明确——不让字符串支持整数索引，迫使你思考你到底想要什么。

字节、字符与字素簇

Rust从三个层级来看待字符串：

let s = "你好🦀";

// 1. 字节（bytes）：UTF-8原始编码
for b in s.bytes() {
    print!("{b:x} ");  // e4 bd a0 e5 a5 bd f0 9f a6 80
}

// 2. Unicode字符（chars）
for c in s.chars() {
    print!("{c} ");  // 你 好 🦀
}

// 3. 字素簇（grapheme clusters）：用户感知的"字符"
// 需要外部crate: unicode-segmentation
// use unicode_segmentation::UnicodeSegmentation;
// for g in s.graphemes(true) {
//     print!("{g} ");
// }
UTF-8索引陷阱：如果你用&s[0..1]对一个中文或emoji字符串做切片，Rust会在运行时panic，因为它跨到了一个字符的中间字节。这和C语言不同——C语言根本不会管你，直接给你不完整的数据。Rust说"宁panic也不返回无效数据"，这是内存安全哲学的延伸。

字符串操作

Rust的标准库为字符串提供了丰富的操作方法。以下是最常用的：

方法 作用 示例
len()返回字节数（不是字符数！）"你好".len() == 6
chars()返回Unicode字符迭代器遍历每个char
bytes()返回字节迭代器遍历每个字节
contains()是否包含子串"hello".contains("ell")
replace()替换所有匹配"a b c".replace(" ", ",")
split()按分割符拆分"a,b,c".split(",")
trim()去除首尾空白" hi ".trim()
to_lowercase()转为小写"HELLO".to_lowercase()
starts_with()是否以某前缀开头"file.rs".starts_with("file")
find()查找子串位置"hello".find("ll")
is_empty()是否为空"".is_empty()

来看一个综合示例：处理CSV中的一行数据

fn process_line(line: &str) {
    let trimmed = line.trim();
    if trimmed.is_empty() {
        return;
    }
    let parts: Vec<&str> = trimmed.split(',').collect();
    for (i, part) in parts.iter().enumerate() {
        println!("字段{i}: '{part}'");
    }
}

fn main() {
    let data = String::from("  Alice,25,engineer  ");
    process_line(&data);
    // 输出:
    // 字段0: 'Alice'
    // 字段1: '25'
    // 字段2: 'engineer'  (注意trim去掉了两端的空格)
}
性能提示：split()、trim()等方法返回的是&str切片——它们不需要新分配堆内存。replace()和to_lowercase()则返回新的String，因为需要修改内容。当你做大量字符串处理时，尽量用切片操作（不分配）而非产生新String的操作（有分配开销）。

本章小结

这一章我们深入学习了Rust中的切片和字符串：&str作为字符串切片的不可变引用语义；String作为可变、拥有的字符串类型；两者之间的区别与转换规则；UTF-8编码下字符串分三个层级（字节、字符、字素簇）的视角；以及标准库提供的常用字符串操作方法。

字符串是每个程序几乎都会用到的数据，在Rust中它尤其特别——它既不是C的"字符数组"也不是Java的"包装对象"，而是一个精心设计的所有权模型实例。理解了它，你就掌握了Rust所有权哲学的一个重要应用场景。
