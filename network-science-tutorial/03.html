第3章 网络基本度量

大家好，欢迎回到网络科学课程！上一章我们学习了图论的基础知识，认识了顶点、边、邻接矩阵等基本概念。掌握了这套描述网络的"语言"之后，今天我们要学习如何用具体的数字指标来刻画一个网络的特征。

想象一下，你要向一个从未去过北京的人描述这座城市。你会怎么说？"北京很大""人口很多""交通很堵"——这些描述太模糊了。更准确的描述应该是："面积16410平方公里""常住人口2188万""工作日早晚高峰平均时速约23公里"。网络科学也是一样的道理：我们需要一组标准化的度量指标，来精确地描述和比较不同的网络。

本章我们将学习五个核心度量：度与度分布、路径与距离、聚集系数、网络密度与稀疏性，最后用一个综合实例来展示这些度量的实际意义。

度与度分布

度（Degree）是网络分析中最基本也最重要的概念。一个顶点的度就是与该顶点相连的边的数量，记作 kᵢ 或 deg(vᵢ)。在无向图中，它就是邻居的个数。在有向图中，我们区分为入度（in-degree）和出度（out-degree）。

度的概念虽然简单，但它在网络科学中扮演着多重角色。首先，度是对顶点"重要性"的初步度量——一个拥有更多连接的顶点往往在网络上具有更大的影响力。其次，整个网络的度分布（Degree Distribution）——即随机选择一个顶点其度数为 k 的概率 P(k)——是区分不同类型网络的关键特征。

度分布可以直观地通过直方图展示。我们很快就会看到，不同类型的网络表现出截然不同的度分布形态：

  • 泊松分布（Poisson Distribution）：大多数顶点的度数集中在平均值附近，两端很少。这是ER随机图模型的典型特征。在这种网络中，绝大多数节点的度数都差不多——比如在一个100人的随机社交圈里，大多数人都有大约6-8个朋友。
  • 幂律分布（Power-law Distribution）：少数顶点拥有极高的度，而大多数顶点的度很低。这是无标度网络的典型特征。例如，在Twitter上，一小部分名人拥有千万级粉丝，而绝大多数普通用户只有几十到几百个粉丝。

还有一个与度相关的概念叫度序列（Degree Sequence）：将图中所有顶点的度数按某种顺序排列（通常是从大到小）得到一个序列。度序列是网络分析中经常使用的原始数据。例如，通过比较两个网络的度序列，我们可以初步判断它们是否可能来自同一类生成机制。

关键洞察：度分布的形状决定了网络的几乎所有其他性质。一个网络的度分布是"胖尾"的（即存在很多高度数节点）还是"瘦尾"的（所有节点的度数差不多），决定了它的鲁棒性、传播速度、社团结构等等。这就是为什么我们在后续章节中会花大量篇幅讨论度分布——它是网络科学的"基因"。

路径与距离

上一章我们学习了路径的概念。在网络科学中，路径不仅仅是一个存在性问题——我们还关心路径的长度。

最短路径（Shortest Path），也叫测地路径（geodesic path），是两个顶点之间边数最少的路径。两个顶点 u 和 v 之间的最短路径长度记作 d(u, v)，也称为它们之间的距离（distance）。

基于最短路径，我们定义几个重要的网络级度量：

平均路径长度（Average Path Length）：网络中所有顶点对之间最短路径长度的平均值。这是衡量网络"紧凑度"的关键指标。Facebook在2016年的研究发现，其全球用户之间的平均路径长度仅为4.57——也就是说，任意两个Facebook用户之间平均只需要不到5次转发就能建立联系。

网络直径（Diameter）：所有最短路径长度中的最大值。直径刻画了网络在"最坏情况"下的连通效率。一个社交网络的直径意味着"最远的两个人"之间的距离。

偏心率（Eccentricity）：从一个顶点 v 出发到所有其他顶点的最短路径长度中的最大值。偏心率最小的顶点被称为网络的"中心"——它到所有其他点的距离都不太远。实际上，在网络基础设施选址中，偏心率是一个实用的参考指标。

值得一提的是，真实世界的大规模网络通常表现出一个令人惊讶的性质：尽管网络包含成千上万甚至数十亿个节点，但其直径却出奇地小。这就是我们将在第5章深入讨论的"小世界"现象。

聚集系数

你的社交网络中有一个很常见的现象：两个共同的朋友之间往往也是朋友。也就是说，如果A是B的朋友，A也是C的朋友，那么B和C也很有可能是朋友。这种"朋友的朋友也是朋友"的倾向，在学术上称为聚集性（Clustering），其量化指标就是聚集系数。

局部聚集系数（Local Clustering Coefficient）衡量一个顶点 v 的邻居之间也互相连接的程度。具体地，设顶点 v 有 kᵥ 个邻居，这些邻居之间最多可以有 kᵥ(kᵥ-1)/2 条边（完全连接时），实际存在的边数为 eᵥ，则局部聚集系数为：

C(v) = eᵥ / [kᵥ × (kᵥ-1) / 2]
# 如果邻居数 ≤ 1，定义 C(v) = 0

# 用NetworkX计算局部聚集系数
import networkx as nx

G = nx.Graph()
G.add_edges_from([(1,2), (1,3), (1,4), (2,3)])  # 顶点1有3个邻居

# 顶点1的邻居是2,3,4，它们之间只有边(2,3)
# C(1) = 1 / [3×(3-1)/2] = 1/3 ≈ 0.333
print(f"C(1) = {nx.clustering(G, 1):.3f}")

# 计算所有顶点的平均聚集系数
print(f"平均聚集系数 = {nx.average_clustering(G):.3f}")

全局聚集系数（Global Clustering Coefficient）是整个网络层面的聚集性度量。它有两种常见定义：一种是所有顶点的局部聚集系数的平均值；另一种是传递性（Transitivity），定义为网络中"闭合三角形"的数量除以"连通三元组"（即长度为2的路径）的数量。传递性的概念非常直观：在一个连通三元组中，第三边存在的概率有多大？

真实社交网络的聚集系数通常远高于随机网络。设想一下：在一个随机网络中，如果两个人都认识你，那么他们之间认识的概率只是整个网络中任意两人相识的概率——通常非常小。但在真实社交中，"共同朋友"效应使得这个概率大大升高。聚集系数就是量化这个效应的关键指标。

网络密度与稀疏性

网络密度（Density）定义为网络中实际存在的边数与可能存在的最大边数之比：

# 无向图
density = 2m / [n(n-1)]
# 有向图
density = m / [n(n-1)]

# NetworkX直接计算
d = nx.density(G)
print(f"密度 = {d:.4f}")

其中 m 是实际边数，n 是顶点数。密度值介于0到1之间。密度为1表示完全图（每对顶点之间都有边），密度为0表示没有任何边的空图。

真实世界的网络有一个普遍特征：稀疏性（Sparsity）。大多数真实网络的密度都非常低，且随着网络规模的增大而降低。为什么？因为人类大脑的处理能力有限——一个普通人不可能同时维持成千上万段高质量的社交关系。这就是著名的邓巴数（Dunbar's Number）假说：人类能够维持稳定社交关系的人数上限约为150人。类似地，一个网页不可能链接到所有其他网页，一个神经元也不可能连接到所有其他神经元。

这意味着：虽然网络的总边数 m 会随着网络规模 n 增长，但通常 m ∝ n（线性增长）而不是 m ∝ n²（完全图的二次增长）。因此密度 d ∝ 1/n，随着网络规模增大而趋近于零。这一性质对网络分析算法的设计有深远影响——因为绝大多数真实网络都是稀疏的，所以使用邻接表存储和基于局部搜索的算法往往比基于矩阵运算的方法更加高效。

网络度量综合实例

让我们通过一个具体的例子，把本章学到的所有度量串联起来。假设我们有三个网络：一个ER随机图、一个真实社交网络、一个电力网络。我们可以用这些度量来量化它们之间的差异。

import networkx as nx
import numpy as np

# 生成一个ER随机图：n=1000, p=0.01
G_er = nx.erdos_renyi_graph(1000, 0.01)

# 加载一个真实网络的替代（用BA模型模拟）
G_ba = nx.barabasi_albert_graph(1000, 3)

# 计算各项度量
def network_profile(G, name):
    n = G.number_of_nodes()
    m = G.number_of_edges()
    degrees = [d for _, d in G.degree()]
    print(f"=== {name} ===")
    print(f"节点数: {n}, 边数: {m}")
    print(f"密度: {nx.density(G):.6f}")
    print(f"平均度: {np.mean(degrees):.2f}")
    print(f"度方差: {np.var(degrees):.2f}")
    print(f"平均聚集系数: {nx.average_clustering(G):.4f}")
    # 对于大规模图，使用采样估算平均路径长度
    if n <= 2000:
        print(f"平均路径长度: {nx.average_shortest_path_length(G):.2f}")
        print(f"直径: {nx.diameter(G)}")
    print()

network_profile(G_er, "ER随机图")
network_profile(G_ba, "BA无标度网络")

运行这段代码，你会看到两个网络在各项度量上的显著差异：

  • ER图的聚集系数接近于连接概率 p = 0.01，而BA网络的聚集系数往往更高。
  • ER图的度方差很小（度分布集中），而BA网络的度方差很大（存在枢纽节点）。
  • 两个网络的平均路径长度都较短（小世界特性），但背后的机制不同。

这些度量让我们能够用统一的标准来比较性质迥异的网络。例如，我们可以分析一个真实世界的网络——比如某个城市的公共交通网络——计算它的密度、聚集系数、平均路径长度，然后与ER模型或BA模型对比。如果发现它的某些度量显著偏离模型预测，那可能意味着这个网络背后存在我们尚未理解的组织原理。

实战踩坑：在网络度量计算中，最容易忽视的一点是有向性与无向性的混淆。我曾在分析一个引文网络时，直接使用无向图的公式计算聚集系数——得到了一个看似合理的值。但后来发现，引文网络是有向的（论文A引用论文B ≠ 论文B引用论文A），正确的做法是使用有向图的聚集系数定义。这个错误导致我最初的结论完全不可靠。所以在动手计算之前，务必先确认你的数据类型是有向还是无向，这一点怎么强调都不过分。

通过本章的学习，我们掌握了一套描述网络特征的量化工具：度与度分布揭示了网络中连接的分布模式；路径和距离刻画了信息传递的效率；聚集系数反映了"抱团"的倾向；密度则告诉我们网络的稀疏程度。这些度量就像医生的听诊器——虽然简单，但能提供关于网络"健康状况"的第一手信息。
