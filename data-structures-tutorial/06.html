前言：先来后到的公平

大家好，我是你们的讲师。上一章我们学了栈——后进先出。这一章的队列（queue）是它的对立面：先进先出（First In First Out，FIFO）。

队列在生活中无处不在：食堂打饭排队——先到的人先打饭；打印机任务队列——先提交的先打印；消息队列——先发送的消息先被处理。队列体现的是"公平"——按到达顺序处理请求。在计算机系统中，这种"排队"的机制是保证系统稳定运行的核心手段。

队列的概念与生活类比

队列的操作也很简单：

  • enqueue（入队）：从队尾加入一个元素
  • dequeue（出队）：从队首移除一个元素
  • peek / front：看一眼队首元素，不移除

生活中有趣的例子：

  • 咖啡店的取餐台：你下单后拿到一个号，叫到你的时候去取——这就是队列。如果咖啡店里每个顾客都挤在柜台前（变成了栈），那场面就混乱了
  • 呼叫中心：客服按来电顺序接听电话——精确的 FIFO 顺序
  • 操作系统的 I/O 请求：磁盘按请求到达的顺序处理读写——当然，这只是一个简化的模型，实际的 I/O 调度要复杂得多

一个容易混淆的点：队列的"入队"和"出队"发生在不同的端。栈是同一端进出，队列是两端各负其责。这个差别看似微小，但带来的数学性质完全不同。

队列的实现

Python 中最推荐的队列实现是 collections.deque（双端队列）：

from collections import deque

# deque 就是高效的队列
queue = deque()
queue.append(1)       # 从右边入队 → deque([1])
queue.append(2)       # → deque([1, 2])
queue.append(3)       # → deque([1, 2, 3])
front = queue.popleft()  # 从左边出队 → 1, deque 变为 [2, 3]
print(front)  # 1

为什么不用 list 实现队列？因为 list.pop(0) 是 O(n) 的——移除第一个元素后，后面的所有元素都要往前挪一个位置。deque.popleft() 是 O(1) 的——它底层用双向链表实现，两端的操作都是常数时间。

循环队列 (Circular Queue)

循环队列是一个很巧妙的设计。想象一下，你用数组来实现队列：

# 普通数组队列的问题：
queue = [None] * 5
# front = 0, rear = 0
# 入队 a,b,c → [a, b, c, None, None], rear = 3
# 出队两次 → front = 2, queue = [a, b, c, None, None]
# 此时前面两个位置空了，但 rear 在 3，不能再插入了
# 这就是"假溢出"——有空间但用不了

循环队列解决了这个问题：当 rear 到达数组末尾时，它绕回到开头。

class CircularQueue:
    def __init__(self, k: int):
        self.data = [None] * k
        self.capacity = k
        self.front = 0
        self.rear = 0
        self.size = 0

    def enqueue(self, val: int) -> bool:
        if self.is_full():
            return False
        self.data[self.rear] = val
        self.rear = (self.rear + 1) % self.capacity
        self.size += 1
        return True

    def dequeue(self) -> bool:
        if self.is_empty():
            return False
        self.front = (self.front + 1) % self.capacity
        self.size -= 1
        return True

    def front_value(self) -> int:
        if self.is_empty():
            return -1
        return self.data[self.front]

    def rear_value(self) -> int:
        if self.is_empty():
            return -1
        return self.data[(self.rear - 1) % self.capacity]

    def is_empty(self) -> bool:
        return self.size == 0

    def is_full(self) -> bool:
        return self.size == self.capacity

关键点：rear 指向的是下一个要插入的位置，而不是最后一个元素的位置。图中画出来会比较清晰——front 和 rear 两个指针在环形数组上追逐，当它们相遇且队列非空时，队列就满了。这种设计在操作系统的环形缓冲区、生产者-消费者模型中都有广泛应用。

双端队列 (Deque)

双端队列（deque，读作"deck"）允许在两端进行插入和删除操作。它结合了栈和队列的能力：

from collections import deque

dq = deque()
dq.append(1)        # 右端入 → [1]
dq.appendleft(2)    # 左端入 → [2, 1]
dq.append(3)        # 右端入 → [2, 1, 3]
print(dq.pop())     # 右端出 → 3
print(dq.popleft()) # 左端出 → 2

双端队列在滑动窗口类问题中非常有用。以 LeetCode 239. 滑动窗口最大值为例，我们可以用双端队列来维护一个"候选最大值"的窗口：

def max_sliding_window(nums, k):
    from collections import deque
    dq = deque()  # 存索引，对应的值单调递减
    result = []

    for i, n in enumerate(nums):
        # 移除窗口外的元素
        while dq and dq[0] <= i - k:
            dq.popleft()
        # 维护单调递减：移除所有比 n 小的
        while dq and nums[dq[-1]] <= n:
            dq.pop()
        dq.append(i)

        # 窗口形成了，记录结果
        if i >= k - 1:
            result.append(nums[dq[0]])

    return result

这个算法的时间复杂度是 O(n)——每个元素最多入队一次、出队一次。如果不使用双端队列，暴力法对于每个窗口都要遍历 k 个元素找最大值，复杂度是 O(n·k)。

单调队列 vs 单调栈

看到这里，你可能会觉得单调队列和上一章的单调栈很像。它们确实有相似之处，但有一个关键区别：

特性 单调栈 单调队列
数据结构 栈（LIFO） 双端队列（两端操作）
核心操作 push / pop（同一端） push（右端）/ pop（左端或右端）
典型问题 下一个更大元素 滑动窗口最值
窗口范围 向左/向右找第一个更大/更小 固定长度滑动区间
淘汰机制 遇到更大/小的就把栈顶弹出 既淘汰窗口外的，也淘汰比当前小的

简单来说：单调栈关注的是"下一个"，单调队列关注的是"当前窗口内的"。两者维护单调性的思路一致，但队列多了一个"左端出窗口"的操作。

队列在 BFS 中的角色

队列在图论和树的遍历中有一个极其重要的应用——广度优先搜索（Breadth-First Search，BFS）。

from collections import deque

def bfs_tree(root):
    """树的层序遍历（BFS）"""
    if not root:
        return []
    result = []
    queue = deque([root])

    while queue:
        node = queue.popleft()
        result.append(node.val)
        if node.left:
            queue.append(node.left)
        if node.right:
            queue.append(node.right)

    return result

# 层序遍历的结果就是按层输出的：[1, 2, 3, 4, 5, ...]
# 这正是利用了 FIFO 的特性——先进树的节点先被访问

为什么 BFS 要用队列？因为 BFS 要求"先发现先探索"——先发现的节点要先展开它的子节点，这正好是 FIFO 的顺序。如果你用栈（DFS），就会变成"后进先出"——后发现的先展开，这就是深度优先搜索了。

BFS 和队列的关系将在第 8 章（树基础）和第 12 章（图基础）中进一步展开。现在你只需要记住：看到"层序遍历"、"最短路径"、"按层次展开"，就想到队列。

⚠️ 性能陷阱：list.pop(0) 是 O(n) 的
很多 Python 初学者会用 list 当作队列，写 queue = []，然后用 queue.append(x) 入队、queue.pop(0) 出队。这在数据量小的时候没问题，但一旦数据量达到几十万，性能就会急剧下降——因为每次 pop(0) 都要把后面的所有元素往前移动。相比之下，deque.popleft() 是 O(1) 的。这是一个非常常见的性能陷阱。记住一个简单的规则：如果你需要从左边弹出元素，就用 deque，不要用 list。

本章小结

队列体现了计算机科学中"公平"的一面——先进先出。从基础的循环队列（解决假溢出），到双端队列（两端操作），再到单调队列（滑动窗口最值），以及 BFS 中的核心角色——队列的变化形式远比看上去丰富。

下一章，我们将进入一个全新的领域——哈希表。它是"空间换时间"这个思想的极致体现，也是你在面试中遇到"O(n²) 变 O(n)"这类优化时最常用的武器。
