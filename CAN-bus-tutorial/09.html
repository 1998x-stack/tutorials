第9章 STM32 CAN初始化：CAN初始化步骤、位时间配置、过滤器配置、中断配置

好，咱们今天来聊聊STM32的CAN初始化。说实话，很多新手朋友一上来就急着发数据，结果发现CAN总线死活没反应。嗯，我当年也干过这事儿。后来才明白，初始化这一步要是没走稳，后面全是白搭。

我个人习惯把CAN初始化拆成四个大块：时钟与GPIO配置、CAN工作模式与位时间、过滤器设置、中断使能。咱们一个一个来啃。

9.1 初始化步骤：先打好地基

先说说整体流程。你想想看，CAN外设就像一台精密仪器，你得先给它通电、接好线、告诉它怎么工作，它才能干活。

具体步骤我总结了一下：

  1. 使能时钟：打开CAN外设时钟，还有它对应的GPIO时钟。别忘了，CAN用的是复用功能。
  2. 配置GPIO：把CAN_TX设为复用推挽输出，CAN_RX设为浮空输入。我习惯把这两个引脚的速度调到最高，省得出问题。
  3. 复位CAN外设：调用CAN_DeInit()，让CAN回到初始状态。这一步很多人会漏掉，但我觉得挺重要的。
  4. 设置工作模式：正常模式、环回模式、静默模式，根据需求选。调试阶段我一般先用环回模式。
  5. 配置位时间：也就是波特率。这个后面细说。
  6. 配置过滤器：决定哪些报文能进，哪些得挡在外面。
  7. 使能中断：如果你要用中断接收，这一步不能少。
  8. 启动CAN：调用CAN_Start()，让CAN进入工作状态。

你看，一共八步。我在项目中遇到过有人把启动CAN放在过滤器前面，结果过滤器没生效，数据乱飞。所以顺序很重要，别搞反了。

9.2 位时间配置：波特率是怎么算出来的？

位时间配置，说白了就是决定CAN通信的快慢。CAN总线的波特率不是随便设的，它跟你的时钟频率、分频系数、时间段都有关系。

STM32的CAN位时间由三部分组成：

  • SYNC_SEG：同步段，固定为1个时间量子（tq）。
  • BS1：时间段1，可以设1~16个tq。
  • BS2：时间段2，可以设1~8个tq。

波特率的计算公式是：

波特率 = CAN时钟频率 / (分频系数 × (1 + BS1 + BS2))

举个例子。假设CAN时钟是36MHz，我想跑到500kbps。那分频系数设多少？

先算一下：500k = 36M / (分频 × (1 + BS1 + BS2))。我习惯把BS1设4，BS2设3，这样加起来是1+4+3=8个tq。那么分频系数就是36M / (500k × 8) = 9。

嗯，这里要注意：采样点位置也很关键。采样点一般在BS1结束的位置，也就是(1+BS1)/(1+BS1+BS2)这个比例。我建议采样点设在75%~85%之间，太靠前或太靠后都容易误采样。

重要提示：位时间配置直接影响总线通信的稳定性。我曾经在一个项目中，因为BS1和BS2设得不合理，导致总线在高温下频繁出错。后来把BS1调大了一点，问题就解决了。

9.3 过滤器配置：只收你想收的

过滤器，说白了就是CAN的看门人。它决定哪些报文能进入你的接收缓冲区。STM32的CAN有28个过滤器组，每个组可以工作在两种模式下：

  • 标识符列表模式：只接收指定ID的报文。适合节点数量少的场景。
  • 标识符屏蔽模式：接收符合掩码规则的报文。适合需要过滤一段ID范围的场景。

我个人习惯用屏蔽模式，因为它更灵活。举个例子：

CAN_FilterInitStructure.CAN_FilterNumber = 0;          // 使用过滤器组0
CAN_FilterInitStructure.CAN_FilterMode = CAN_FilterMode_IdMask;  // 屏蔽模式
CAN_FilterInitStructure.CAN_FilterScale = CAN_FilterScale_32bit; // 32位宽度
CAN_FilterInitStructure.CAN_FilterIdHigh = 0x0000;     // 标准ID高16位
CAN_FilterInitStructure.CAN_FilterIdLow = 0x0000;      // 标准ID低16位
CAN_FilterInitStructure.CAN_FilterMaskIdHigh = 0x0000; // 掩码高16位
CAN_FilterInitStructure.CAN_FilterMaskIdLow = 0x0000;  // 掩码低16位
CAN_FilterInitStructure.CAN_FilterFIFOAssignment = CAN_FIFO0; // 关联到FIFO0
CAN_FilterInitStructure.CAN_FilterActivation = ENABLE; // 使能过滤器
CAN_FilterInit(&CAN_FilterInitStructure);

你看，掩码全为0表示接收所有报文。如果我只想收ID为0x123的报文，那把掩码设成全1，ID设成0x123就行了。

小技巧：调试阶段我建议先把过滤器设成接收所有报文，等通信正常了再慢慢收紧过滤规则。这样能快速定位问题是出在发送端还是接收端。

9.4 中断配置：别让CPU一直等着

中断配置，说白了就是让CAN在收到报文时主动通知CPU，而不是让CPU轮询。这样CPU就能腾出手干别的活儿。

STM32的CAN中断有好几种：

  • 接收中断：FIFO0或FIFO1收到报文时触发。
  • 发送中断：发送邮箱空时触发。
  • 错误中断：总线错误或状态变化时触发。
  • 唤醒中断：CAN从休眠状态唤醒时触发。

我一般只开接收中断和错误中断。发送中断用得少，因为发送通常是主动的，用轮询就够了。

配置中断的步骤：

  1. 在NVIC中使能CAN的中断通道。
  2. 设置中断优先级。我习惯把CAN中断优先级设高一点，毕竟通信实时性很重要。
  3. 在CAN初始化中使能对应的中断源。

代码示例：

NVIC_InitStructure.NVIC_IRQChannel = CAN1_RX0_IRQn;    // 接收中断通道
NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 0; // 抢占优先级
NVIC_InitStructure.NVIC_IRQChannelSubPriority = 0;       // 子优先级
NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE;
NVIC_Init(&NVIC_InitStructure);

CAN_ITConfig(CAN1, CAN_IT_FMP0, ENABLE); // 使能FIFO0消息挂起中断

然后在中断服务函数里读取报文：

void CAN1_RX0_IRQHandler(void)
{
    CanRxMsg RxMessage;
    CAN_Receive(CAN1, CAN_FIFO0, &RxMessage);
    // 处理接收到的报文
}

警告：中断服务函数里别干太多活儿。我曾经在中断里做复杂的报文解析，结果导致后续报文丢失。记住，中断里只做最必要的事，比如把数据拷贝到缓冲区，然后设置一个标志位，让主循环去处理。

9.5 完整初始化示例

最后，我把上面这些串起来，给一个完整的初始化函数。你直接拿去用，改改参数就行。

void CAN_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;
    CAN_InitTypeDef CAN_InitStructure;
    CAN_FilterInitTypeDef CAN_FilterInitStructure;
    NVIC_InitTypeDef NVIC_InitStructure;

    // 1. 使能时钟
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_CAN1, ENABLE);

    // 2. 配置GPIO
    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_8; // CAN1_TX
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
    GPIO_Init(GPIOB, &GPIO_InitStructure);

    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_9; // CAN1_RX
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;
    GPIO_Init(GPIOB, &GPIO_InitStructure);

    // 3. 复位CAN
    CAN_DeInit(CAN1);

    // 4. 配置位时间（500kbps，时钟36MHz）
    CAN_InitStructure.CAN_TTCM = DISABLE;
    CAN_InitStructure.CAN_ABOM = ENABLE;
    CAN_InitStructure.CAN_AWUM = DISABLE;
    CAN_InitStructure.CAN_NART = DISABLE;
    CAN_InitStructure.CAN_RFLM = DISABLE;
    CAN_InitStructure.CAN_TXFP = DISABLE;
    CAN_InitStructure.CAN_Mode = CAN_Mode_Normal;
    CAN_InitStructure.CAN_SJW = CAN_SJW_1tq;
    CAN_InitStructure.CAN_BS1 = CAN_BS1_4tq;
    CAN_InitStructure.CAN_BS2 = CAN_BS2_3tq;
    CAN_InitStructure.CAN_Prescaler = 9;
    CAN_Init(CAN1, &CAN_InitStructure);

    // 5. 配置过滤器（接收所有报文）
    CAN_FilterInitStructure.CAN_FilterNumber = 0;
    CAN_FilterInitStructure.CAN_FilterMode = CAN_FilterMode_IdMask;
    CAN_FilterInitStructure.CAN_FilterScale = CAN_FilterScale_32bit;
    CAN_FilterInitStructure.CAN_FilterIdHigh = 0x0000;
    CAN_FilterInitStructure.CAN_FilterIdLow = 0x0000;
    CAN_FilterInitStructure.CAN_FilterMaskIdHigh = 0x0000;
    CAN_FilterInitStructure.CAN_FilterMaskIdLow = 0x0000;
    CAN_FilterInitStructure.CAN_FilterFIFOAssignment = CAN_FIFO0;
    CAN_FilterInitStructure.CAN_FilterActivation = ENABLE;
    CAN_FilterInit(&CAN_FilterInitStructure);

    // 6. 配置中断
    NVIC_InitStructure.NVIC_IRQChannel = CAN1_RX0_IRQn;
    NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 0;
    NVIC_InitStructure.NVIC_IRQChannelSubPriority = 0;
    NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE;
    NVIC_Init(&NVIC_InitStructure);

    CAN_ITConfig(CAN1, CAN_IT_FMP0, ENABLE);

    // 7. 启动CAN
    CAN_Cmd(CAN1, ENABLE);
}

好了，这就是CAN初始化的全部内容。你想想看，其实没那么复杂，就是按部就班地把每个环节配好。我在项目中遇到过最坑的事，就是忘了使能GPIO时钟，结果CAN_TX死活拉不出波形。嗯，从那以后我每次初始化都会先检查时钟有没有开。
