区间 DP — 区间长度作为状态维度

大家好，我是你们的讲师。欢迎来到第十四章——区间 DP 与状态机。这两个概念看起来不相关，但它们共同代表了 DP 进阶的两个重要方向：一个是状态维度的扩展（从单点到区间），一个是状态内涵的丰富（从单一值到多状态）。

先看 区间 DP。区间 DP 的状态定义和线性 DP 不同——线性 DP 的 dp[i] 通常表示"前 i 个元素"或"以 i 结尾"，而区间 DP 的 dp[i][j] 表示"从 i 到 j 这个区间"的最优解。两者的遍历顺序也因此完全不同：区间 DP 的典型遍历方式是按区间长度从小到大枚举，先算长度为 1 的区间，再算长度为 2 的，依此类推。

# 区间 DP 的标准遍历模板
n = len(arr)
dp = [[0] * n for _ in range(n)]

# 先初始化长度为 1 的区间
for i in range(n):
    dp[i][i] = initial_value

# 按长度从小到大遍历
for length in range(2, n + 1):          # 区间长度
    for i in range(0, n - length + 1):   # 左端点
        j = i + length - 1               # 右端点
        # 状态转移，通常枚举分割点
        for k in range(i, j):
            dp[i][j] = max(dp[i][j], dp[i][k] + dp[k+1][j] + cost)

这个三层循环的模板适用于大多数区间 DP 题目。外层 length 确保"短区间先计算"，内层枚举分割点 k。

最长回文子串（LeetCode 5）

这是区间 DP 的经典入门题。题目虽然也可以用中心扩展法解决，但 DP 的思维对后续题目帮助很大。

状态定义：dp[i][j] 表示子串 s[i:j+1] 是否是回文串。

def longest_palindrome(s):
    n = len(s)
    if n < 2:
        return s

    dp = [[False] * n for _ in range(n)]
    start, max_len = 0, 1

    # 所有长度为 1 的子串都是回文
    for i in range(n):
        dp[i][i] = True

    # 按长度从小到大枚举
    for length in range(2, n + 1):
        for i in range(0, n - length + 1):
            j = i + length - 1
            if s[i] == s[j]:
                if length == 2:
                    dp[i][j] = True
                else:
                    dp[i][j] = dp[i + 1][j - 1]  # 去掉首尾后的子串是回文
            if dp[i][j] and length > max_len:
                start = i
                max_len = length

    return s[start:start + max_len]

转移方程的核心：s[i] == s[j] 且 s[i+1:j] 是回文，则 s[i:j+1] 也是回文。这个转移非常简单，但它的遍历顺序（先长度后左端点）是区间 DP 的标志性特征。

中心扩展 vs 区间 DP：LeetCode 5 的最优解是中心扩展法（O(n²) 时间，O(1) 空间），但区间 DP（O(n²) 时间，O(n²) 空间）更容易扩展到其他回文相关问题，比如最长回文子序列。在面试中，如果你先给出 DP 解法，然后提到"这道题在空间上还可以优化到 O(1)"，面试官会非常认可。

最长回文子序列（LeetCode 516）

这道题和 5 号题的区别在于：子序列不要求连续，所以 DP 定义和转移都有变化。

def longest_palindrome_subseq(s):
    n = len(s)
    dp = [[0] * n for _ in range(n)]

    for i in range(n):
        dp[i][i] = 1

    for length in range(2, n + 1):
        for i in range(0, n - length + 1):
            j = i + length - 1
            if s[i] == s[j]:
                dp[i][j] = dp[i + 1][j - 1] + 2
            else:
                dp[i][j] = max(dp[i + 1][j], dp[i][j - 1])

    return dp[0][n - 1]

注意区别：当两端字符不同时，子序列问题可以选择"舍弃左边"或"舍弃右边"，取最大值。而子串问题由于连续性限制——两端字符不同就不可能构成回文子串，所以直接标记 False。

股票买卖 — 状态机 DP

股票买卖系列是 LeetCode 上非常经典的状态机 DP题目。所谓状态机 DP，就是状态不仅有"第几天"，还有"当前持仓状态"——比如：持有还是不持有？这个状态维度把所有可能的决策都建模进来了。

买卖股票的最佳时机（LeetCode 121）

最简单的版本：只能买卖一次。这个问题用贪心就能解决——记录历史最低价，计算当前卖出的最大利润。但为了引出状态机 DP 的思路，我们也可以用 DP 来做：

def max_profit(prices):
    n = len(prices)
    # dp[i][0] = 第 i 天不持有股票的最大利润
    # dp[i][1] = 第 i 天持有股票的最大利润
    dp = [[0, 0] for _ in range(n)]
    dp[0][0] = 0
    dp[0][1] = -prices[0]  # 第 0 天买入

    for i in range(1, n):
        dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])  # 卖出
        dp[i][1] = max(dp[i-1][1], -prices[i])               # 买入（只能买卖一次）

    return dp[n-1][0]

这里 dp[i][1] = max(dp[i-1][1], -prices[i]) 中的 -prices[i] 为什么不是 dp[i-1][0] - prices[i]？因为只能买卖一次，买入之前不能有卖出记录——所以买入时的利润是 -prices[i]，不是之前交易赚的钱减去买入价。

含冷冻期（LeetCode 309）

当引入了冷冻期（卖出后第二天不能买入），状态就需要从两个变成三个：

  • dp[i][0]：不持有股票，且不处于冷冻期
  • dp[i][1]：持有股票
  • dp[i][2]：不持有股票，且处于冷冻期（当天刚卖出）
def max_profit_with_cooldown(prices):
    n = len(prices)
    if n == 0:
        return 0
    dp = [[0, 0, 0] for _ in range(n)]
    dp[0][0] = 0
    dp[0][1] = -prices[0]
    dp[0][2] = 0

    for i in range(1, n):
        dp[i][0] = max(dp[i-1][0], dp[i-1][2])          # 保持非冷冻 / 冷冻恢复
        dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])  # 保持持有 / 非冷冻时买入
        dp[i][2] = dp[i-1][1] + prices[i]               # 卖出，进入冷冻

    return max(dp[n-1][0], dp[n-1][2])

看到状态机 DP 的威力了吗？随着约束条件的变化，我们只需要在状态转移中做对应调整，框架本身就是稳定的。这就是"把逻辑编码到状态中"的思维方式。

状态机 DP 心法：画状态转移图！股票问题的状态转移图非常直观：持有 → 卖出 → 冷冻 → 非冷冻 → 买入 → 持有。在图中标注好每个状态转移的"代价"（利润变化），然后把图翻译成代码。我每次做股票题，第一件事都是画状态机图——画清楚了，代码自然就写出来了。

状态压缩 DP

状态压缩 DP 使用二进制位来表示集合状态。当元素个数不超过 20 时（因为 2²⁰ ≈ 10⁶），可以用一个整数的二进制位来表示"哪些元素已经使用过"。

旅行商问题（TSP）模板

TSP：从起点出发，访问所有城市恰好一次，最后回到起点，求最短路径。

# TSP 状态压缩 DP 模板
# dp[mask][i] = 已经访问过 mask 中的城市，当前在 i 城市的最短路径
# mask 的二进制第 k 位为 1 表示已访问城市 k

def tsp(dist):
    n = len(dist)
    dp = [[float('inf')] * n for _ in range(1 << n)]
    dp[1][0] = 0  # 从城市 0 出发，只访问了城市 0

    for mask in range(1 << n):
        for i in range(n):
            if not (mask >> i) & 1:   # i 不在 mask 中，跳过
                continue
            if dp[mask][i] == float('inf'):
                continue
            for j in range(n):
                if (mask >> j) & 1:   # j 已访问过，跳过
                    continue
                new_mask = mask | (1 << j)
                dp[new_mask][j] = min(dp[new_mask][j],
                                      dp[mask][i] + dist[i][j])

    # 回到起点
    full_mask = (1 << n) - 1
    return min(dp[full_mask][i] + dist[i][0] for i in range(1, n))

状态 mask 的巧妙之处在于：它用 int 的二进制位代替了布尔数组，既节省了空间，也让状态转移变得非常高效。这个技巧在竞赛编程中非常常见，LeetCode 的题目里如 "划分为 k 个相等的子集"（698）也可以用状态压缩。

使用限制：状态压缩 DP 的复杂度通常是 O(n² × 2ⁿ) 或 O(n × 2ⁿ)。当 n 超过 20 时，2ⁿ 会超过 10⁶，通常会导致 TLE 或 MLE。所以面试中遇到 n ≤ 16 或 n ≤ 20 的题目，可以立刻想到状态压缩。如果 n > 20，即使想用也大概率不行——这时要寻找其他解法。

树形 DP — 树上的状态转移

树形 DP 是 DP 在树结构上的自然延伸。当 DP 的状态在树的节点之间转移时（通常是后序遍历），就形成了树形 DP。

打家劫舍 III（LeetCode 337）

这道题是打家劫舍系列的第三部——不过从线性结构变成了二叉树结构。每个节点有两个状态：偷或不偷。

def rob(root):
    def dfs(node):
        if not node:
            return (0, 0)  # (不偷当前节点, 偷当前节点)

        left = dfs(node.left)
        right = dfs(node.right)

        # 不偷当前节点：左右子节点可以偷或不偷，取各自的最大值
        not_rob = max(left) + max(right)
        # 偷当前节点：左右子节点都不能偷
        rob_cur = node.val + left[0] + right[0]

        return (not_rob, rob_cur)

    return max(dfs(root))

返回值是一个二元组，代表"不偷 / 偷"当前节点时的最大金额。后序遍历保证了子节点的结果先计算出来，父节点基于子节点的结果做决策。这就是树形 DP 的标准模式。

二叉树中的最大路径和（LeetCode 124）

这道题在第八章提过，用树形 DP 的思路来理解更清晰。每个节点计算两种值：

  • 单边贡献：当前节点 + max(左子贡献, 右子贡献, 0)——这是对父节点可提供的"最大路径段"。
  • 全局最大：当前节点 + max(左子贡献, 0) + max(右子贡献, 0)——这是"以当前节点为拐点"的完整路径。
def max_path_sum(root):
    global_max = float('-inf')

    def dfs(node):
        nonlocal global_max
        if not node:
            return 0

        left = max(dfs(node.left), 0)    # 左子树贡献（负数则舍弃）
        right = max(dfs(node.right), 0)  # 右子树贡献（负数则舍弃）

        # 以当前节点为拐点的路径和
        global_max = max(global_max, node.val + left + right)

        # 返回给父节点的最大单边贡献
        return node.val + max(left, right)

    dfs(root)
    return global_max

这里的 max(..., 0) 是一个重要的细节——如果子树的贡献是负数，为什么不直接舍弃？因为负数贡献只会拉低总和，不如不选。这个技巧在最大子数组和（53）中也出现过。

DP 类型 状态定义特征 遍历顺序 典型题目 复杂度
区间 DP dp[i][j] 表示区间 [i, j] 按区间长度从小到大 5 / 516 O(n²)
状态机 DP dp[i][k] 多一个状态维度 线性遍历 121 / 309 / 714 O(n)
状态压缩 DP dp[mask][i] mask 表示集合 遍历所有 mask TSP / 698 O(n² × 2ⁿ)

踩坑与总结

坑 1：状态机状态定义混淆

股票系列的进阶题（如 188——最多交易 k 次）中，状态变得非常复杂：dp[i][k][0/1] 表示第 i 天、交易了 k 次、当前是否持有股票。很多人在写三层 DP 数组时搞混维度的顺序——是先写天数、交易次数还是持仓状态？我的建议：固定顺序为 "天数 → 状态 → 其他"，这是最不容易出错的写法。

还有一个常见的混淆点：交易次数什么时候增加？是买入时算一次还是卖出时算一次？不同题目的定义可能不同，一定要先确认这个细节再动笔。我在做 LeetCode 188 时，因为搞混了"买入时计次"和"卖出时计次"，debug 了整整一个小时。

坑 2：区间 DP 遍历顺序

区间 DP 的遍历顺序是"先枚举区间长度，再枚举左端点"。新手经常写成先枚举左端点再枚举右端点——这样会导致计算 dp[i][j] 时，区间 dp[i+1][j-1] 还没有计算出来，结果全错。

这里有一个我个人的小技巧：如果记不住遍历顺序，把区间 DP 看作在二维矩阵的"上三角"上按对角线遍历。画一个矩阵，从主对角线开始，逐条向东北方向移动——每次移动对应区间长度 + 1。这个几何直观比背诵"先长度后左端点"要牢固得多。

坑 3：树形 DP 的"非局部"变量

在 LeetCode 124 中，global_max 是一个非局部变量。Python 中需要用 nonlocal 关键字来声明。很多人在刷题时用 Python 的 nonlocal 或 self. 来声明全局变量，但最容易忘记的是在 DFS 前初始化——比如 global_max = float('-inf') 如果写成 0，当所有节点都是负数时，结果就错了。

本章心法：区间 DP、状态机 DP、状态压缩 DP、树形 DP——这四种模式代表了 DP 状态设计的四种不同思路。区间 DP 关注"范围"，状态机 DP 关注"阶段"，状态压缩关注"集合"，树形 DP 关注"拓扑"。掌握这四种模式，你就真正建立起了 DP 的系统认知框架。之后的 DP 题目对你而言，就不再是"如何定义状态"的问题，而是"这属于哪个模式"的归类问题。
