大家好，我是你们的讲师。今天的话题从一个日常现象开始。

你在咖啡店里，周围五六张桌子都有顾客在聊天。你的朋友坐在对面说话，你能听懂他——尽管旁边有人在讨论股市、后面有人在大声点单、背景里还有研磨咖啡机的轰鸣。你甚至能做到一件更神奇的事：当另一个人叫你的名字时，你会下意识地侧耳过去——"鸡尾酒会效应"从生理到心理都在发挥作用。

在通信世界里，我们也面临类似的问题：多个用户共享同一个"听觉环境"——即同一个物理信道。手机用户同时向基站发送数据（上行），基站同时向多个手机广播数据（下行），Wi-Fi 网络中多个设备共享一个频率。这和"一对一"通信的本质区别在于：干扰。A的信息对B来说是噪声，B的信息对A来说也是噪声。

网络信息论（Network Information Theory）就是研究在这种多用户、有干扰的场景下，信息传输的理论极限。这个领域由香农在1960年代奠基，但和"点对点"通信不同——网络信息论中相当多的基本问题至今仍未完全解决。这是一个充满了开放性问题的学科。

多址接入信道 (MAC)

多址接入信道（Multiple Access Channel, MAC）是最简单的多用户信道模型：多个发送者，一个接收者。典型的例子是5G网络的上行链路——几十部手机同时向一个基站发送数据。

对于两用户高斯 MAC：

Y = X₁ + X₂ + N,   N ~ N(0, σ²)
E[X₁²] ≤ P₁, E[X₂²] ≤ P₂

这里的容量域（Capacity Region）不再是单一的数值，而是一个二维区域——所有可达的速率对 (R₁, R₂) 的集合。对于两用户高斯 MAC，容量域是一个五边形：

  • R₁ ≤ ½ log₂(1 + P₁/σ²) —— 当用户2不通信时
  • R₂ ≤ ½ log₂(1 + P₂/σ²) —— 当用户1不通信时
  • R₁ + R₂ ≤ ½ log₂(1 + (P₁+P₂)/σ²) —— 总速率上限

这个五边形区域的关键含义是：两个用户可以同时以各自的全速通信吗？——不行。它们必须为彼此"让路"。总速率的上限不是独立速率上限的简单相加——两个用户共享功率可以带来"功率合并增益"（总功率 P₁+P₂ 比两个独立功率的信噪比之和更高效）。

容量域边界上的两个角点对应于连续干扰消除（Successive Interference Cancellation, SIC）的策略：接收端先解码一个用户的信号（把另一个用户视为噪声），然后减去该信号后再解码另一个。解码顺序不同，得到两个不同的角点。而五边形内的任意点可以通过时分（Time-Sharing）在角点之间做时间分割来达到。

用 Python 画出两用户 MAC 的容量域：

import numpy as np
import matplotlib.pyplot as plt

def mac_capacity_region(P1, P2, noise_var):
    """两用户高斯MAC容量域的信道参数"""
    # 独立速率上限
    R1_max = 0.5 * np.log2(1 + P1 / noise_var)
    R2_max = 0.5 * np.log2(1 + P2 / noise_var)
    R_sum  = 0.5 * np.log2(1 + (P1 + P2) / noise_var)

    # 五个顶点（顺时针）
    pts = [(0, 0),
           (R1_max, 0),
           (R1_max, R_sum - R1_max),
           (R_sum - R2_max, R2_max),
           (0, R2_max)]
    return pts, R_sum

P1, P2, sigma2 = 2.0, 1.5, 1.0
pts, R_sum = mac_capacity_region(P1, P2, sigma2)

print(f"用户1最大速率: {0.5*np.log2(1+P1/sigma2):.3f} bits/symbol")
print(f"用户2最大速率: {0.5*np.log2(1+P2/sigma2):.3f} bits/symbol")
print(f"总速率上限: {R_sum:.3f} bits/symbol")
print(f"时分可达速率: ")
for alpha in np.linspace(0, 1, 5):
    # 两个角点之间时分
    cp1 = (pts[1][0], pts[1][1])  # 先解用户1
    cp2 = (pts[3][0], pts[3][1])  # 先解用户2
    R1_td = alpha * cp1[0] + (1-alpha) * cp2[0]
    R2_td = alpha * cp1[1] + (1-alpha) * cp2[1]
    print(f"  α={alpha:.2f}: R1={R1_td:.3f}, R2={R2_td:.3f}")

广播信道 (BC)

与 MAC 相对，广播信道（Broadcast Channel, BC）是一个发送者、多个接收者。典型例子：5G下行链路——基站同时向多个手机发送数据。

广播信道的研究比 MAC 困难得多。原因在于：发送者只有一份发射信号，不同的接收者经历不同的"视角"（不同的信道条件）。最经典的模型是退化广播信道（Degraded Broadcast Channel）：

X → Y₁ → Y₂  (马尔可夫链)

Y₁ = X + N₁,   Y₂ = X + N₂ = Y₁ + Ñ

其中 N₁ ~ N(0, σ₁²), N₂ ~ N(0, σ₂²)，且 σ₂² > σ₁²（用户2的信道更差）。

退化广播容量的核心编码技术是叠加编码（Superposition Coding）：发送端用两种不同的"功率层"来发送两个不同的消息。公共消息（对两个用户都重要）用高功率编码，私密消息（只对"好"用户）在剩余功率上编码。"差"用户只能解出公共部分，而"好"用户可以先解公共部分，减去后再解私密部分。这和 MAC 中的 SIC 有异曲同工之妙。

日常类比：想象你在一个嘈杂的广场上喊话。你要同时告诉远处的人（信道差）和近处的人（信道好）。你会怎么做？你会先用一个"公共音量"把所有人都要听的关键信息喊出来。如果你还有精力，再用一个"较低音量"补充细节——这些细节只有近处的人能听清。这就是叠加编码的直觉。

值得注意的是，广播信道的容量域在一般（非退化）情况下仍然是一个开放问题。Cover 在1972年提出了退化广播信道的容量域，但非退化广播信道的完整容量域至今未知。

中继信道与干扰信道

更复杂的网络拓扑包括中继和干扰：

中继信道 (Relay Channel)

源 → 中继 → 目的。中继可以帮助改善通信质量——当源和目的之间的直接链路太差时。典型应用场景：蜂窝网络中的信号中继站、卫星通信、Mesh Wi-Fi 网络。

中继的基本策略有两种：

  • 解码转发（Decode-and-Forward, DF）：中继先完全解码收到的消息，再重新编码转发。虽然简单，但中继可能引入错误传播。
  • 压缩转发（Compress-and-Forward, CF）：中继不试图解码，而是将收到的信号做有损压缩后转发给目的端，让目的端自己去"拼图"。CF 在弱中继链路下通常优于 DF。

中继信道的容量域同样是一个开放问题，目前最好的已知结果是由 Cover 和 El Gamal 在1979年给出的上下界——但上下界之间存在间隙。

干扰信道 (Interference Channel)

这是网络信息论中最"臭名昭著"的问题。两对发送-接收对在同一信道上通信：发送者1想和接收者1通信，发送者2想和接收者2通信——但每个发送者的信号都干扰了另一个接收者。

开放问题：干扰信道的容量域——即使是最简单的双用户高斯干扰信道——经过了40多年的研究仍未被完全确定。这是信息论中最重要的开放问题之一。

对于干扰信道，已知的结果包括：

  • 强干扰：此时可以把干扰当作信号的一部分来解码（类似于把邻居的信号解码出来），容量域由 Han-Kobayashi 界描述
  • 弱干扰：此时把干扰当作噪声——但这不是最优的。Han-Kobayashi 方案（部分解码干扰）是目前已知最好的可达策略，但一般情况下的最优性仍未被证明
  • 干扰对齐（Interference Alignment）：Cadambe 和 Jafar 在2008年的一项突破性工作，展示了在干扰信道中自由度可以随用户数线性增长——但这需要无限维的信号空间
信道模型 发送端数 接收端数 容量结果 典型应用
多址接入信道 (MAC) ≥ 2 1 完全已知（五边形区域） 5G上行、Wi-Fi多用户
广播信道 (BC) 1 ≥ 2 退化情况已知，一般情况开放 5G下行、数字电视
中继信道 1 1（+中继） 上下界已知，有间隙 卫星通信、IoT中继
干扰信道 ≥ 2 ≥ 2 Han-Kobayashi界，一般情况开放 D2D通信、未授权频段
双向信道 2（各发各收） 2 完全已知 对讲机、全双工
X信道 2 2（交叉连接） 自由度已知，容量开放 多天线干扰管理

这张表说明了一个事实：一旦通信从"一对一"变成"多对多"，很多看似简单的问题就变成了数学难题。这就是为什么网络信息论至今仍是一个活跃的研究领域。

网络编码基础

传统的网络通信中，中间节点（路由器、交换机）扮演的角色很简单：存储转发——收到一个数据包，查路由表，转发出去。网络编码（Network Coding）颠覆了这个思路：中间节点可以对收到的数据包做代数运算（比如 XOR），然后将运算结果转发出去。

最经典的例子是蝴蝶网络（Butterfly Network）：

         S
        / \
       /   \
      A     B       S: 源（有a、b两份数据）
       \   /        A, B, C, D: 节点
        \ /         t₁: 要a和b
         C          t₂: 要a和b
        / \
       /   \
      t₁   t₂

传统路由：中继链路 C 一次只能传 a 或 b——接收端永远无法同时获得两份数据。如果 C 轮流传 a 和 b，需要两倍的时间。

网络编码的解法：C 将 a 和 b 做 XOR（c = a ⊕ b），然后广播这一个数据包。接收端 t₁ 收到 a（从另一条路径）和 c，计算出 b = a ⊕ c；t₂ 用同样的方法得到 a。这样，中继链路 C 只需要一次传输就同时服务了两个接收者！

这个思想的关键在于：传统路由把信息当作"原子"来搬运，而网络编码把信息当作"数据流"来混合——就像把两封信拆开、把其中的半封信混合在一起重新装封，收信人通过某些"密钥"（自己所知的部分信息）可以解码出完整内容。

网络编码的深层意义：传统路由只能达到"存储转发"的吞吐量上限——即最大流最小割定理所确定的容量。但网络编码告诉我们：通过允许中间节点编码，可以达到多播（multicast）的理论最大容量。换句话说，最大流最小割定理是路由的极限，而网络编码可以触及这个极限。

实际应用方面，网络编码已经走出了单纯的学术研究范畴：

  • P2P文件共享：BitTorrent 的一些变种使用网络编码来提高块调度效率。传统方法中，文件被分成很多块，每个节点要协调"哪些块我有、哪些块你需要"——在大量节点时这非常复杂。使用线性网络编码，节点可以发送"块线性组合"，接收节点只要收到足够多的独立组合就能解码出原始文件。
  • 卫星通信：利用网络编码的"自然"擦除纠正能力，可以减少丢包重传的开销。
  • 无线Mesh网络：COPE 协议是第一个在真实无线路由器上实现网络编码的系统，在实验环境中(通过编码机会)实现了数倍的吞吐量提升。

线性网络编码的数学基础是有限域上的线性代数——每个数据包被看作是有限域 GF(2^m) 上的一个向量，中间节点生成这些向量的线性组合。只要组合系数保持充分随机性，接收端收到足够多的独立方程后就能解出所有原始数据。

这一章我们进入了网络信息论的世界——多个用户、多种拓扑、充满开放问题。关键 takeaways：

  • MAC信道（多对一）：容量域是五边形，可用 SIC 达到角点
  • 广播信道（一对多）：叠加编码是关键，退化情况容量已知
  • 中继信道和干扰信道的基本问题至今仍是开放研究
  • 网络编码让中间节点从"搬运工"变成"计算者"——一个简单的 XOR 操作就能大幅提升多播容量
