第7章：映射与对应

大家好，我是你们的讲师。到此为止，我们学过了很多计数技巧——排列组合公式、二项式定理、容斥原理、隔板法、递推归纳。这些方法有一个共同点：它们都试图直接计算你要数的那个东西的数量。但这一章我们将学习一种完全不同的思路——不直接数你要的东西，而是把它变成另一个更容易数的东西。

这就是映射方法（对应法）。它的核心理念是：如果你能建立一个集合 A 到集合 B 的双射（一一对应），那么 |A| = |B|。所以，找一个 B 使得 |B| 很好算，然后再建立一个漂亮的对应，问题就迎刃而解了。这种方法在组合数学中被誉为"最优雅的证明方式"——它让最复杂的计数问题变得像变魔术一样简单。

7.1 映射方法的三个步骤

使用映射法解题的标准流程分为三步：

  1. 构造映射：定义集合 A（你要数的东西）到集合 B（你知道怎么数的东西）的一个映射 f: A → B。
  2. 证明单射：证明 f 是单射（injection）——即 A 中不同的元素映射到 B 中不同的元素。这保证 |A| ≤ |B|。
  3. 证明满射：证明 f 是满射（surjection）——即 B 中每个元素都有 A 中的原像。这保证 |A| ≥ |B|。

如果 f 既是单射又是满射，那么 f 就是双射（bijection），于是 |A| = |B|。在实际竞赛中，很多时候我们只需要证明存在一个双射，而不需要显式地构造"逆映射"——但需要确保两边大小相等这个结论是正确的。

🔥 一个直觉类比：想象你在数一个剧场里的观众人数（集合 A）。你不必一个一个数观众，而是可以数空座位（集合 B）。如果你能证明每个观众恰好坐在一个座位上（单射），且每个座位都有人坐（满射），那么观众数 = 座位数。"对应法"就是把困难的计数转化成了简单的计数——关键在于找到一个合适的"座位"。

7.2 用对应法转化计数问题

7.2.1 基本思想：化难为易

对应法的精髓是——把"难数的"对应成"易数的"。来看一个经典的例子：

例1： 从 1 到 2n 中选 n 个数，使得选出的数中不包含两个连续的正整数。有多少种选法？

直接分类讨论会非常复杂——因为"不包含连续数"的约束打破了我们熟悉的组合模式。但如果我们做一个巧妙的变换：

设选出的数为 a₁ < a₂ < ... < a_n（来自 {1,2,...,2n}）
令 b_i = a_i - (i-1)
则 b₁ < b₂ < ... < b_n，且 1 ≤ b_i ≤ n+1

为什么？因为 a_i ≥ 2i-1（最紧凑的情况是选 1,3,5,...,2n-1），
所以 b_i = a_i - (i-1) ≥ (2i-1) - (i-1) = i
并且 a_i ≤ 2n - (n-i) = n+i（最宽松的情况是选 n+1,n+2,...,2n）
所以 b_i = a_i - (i-1) ≤ (n+i) - (i-1) = n+1

这个变换建立了——"从 1 到 2n 中选 n 个不相邻的数"与"从 1 到 n+1 中选 n 个数（无额外限制）"之间的一一对应。而后者就是 C(n+1, n) = n+1。

所以答案就是 n+1。是不是很简洁？

💡 思考：这个变换的本质是什么？b_i = a_i - (i-1) 实际上在"压缩"选出的数之间的空隙。因为 a_i 之间至少差 2（不相邻），所以减掉 (i-1) 后，b_i 之间至少差 1（普通的不重复整数）。这样一来，"不相邻"的约束就被消去了。

7.2.2 将组合恒等式转化为双射

我们在第二章中提到的 Vandermonde 卷积恒等式，也可以用双射来理解：

C(m+n, k) = ∑_{i=0}^{k} C(m, i) × C(n, k-i)

双射证明： 设 A 为从 m+n 个元素中选 k 个的所有子集的集合，|A| = C(m+n, k)。设 B 为所有有序对 (S, T) 的集合，其中 S 是从前 m 个元素中选出的 i 元子集（i 任意），T 是从后 n 个元素中选出的 k-i 元子集——显然 |B| = ∑C(m,i)·C(n,k-i)。

现在建立双射：给定 A 中的一个子集 U（大小为 k），令 S = U ∩ {前 m 个元素}，T = U ∩ {后 n 个元素}。那么 (S, T) 属于 B。反之，给定 B 中的 (S, T)，令 U = S ∪ T，则 U 是 A 中的元素。这是一个自然的双射。

所以 |A| = |B|，即 C(m+n, k) = ∑C(m,i)·C(n,k-i)。

你看，一旦找到了正确的双射，恒等式的证明变得几乎是"平凡"的——这比代数推导漂亮得多。

7.3 Dyck路径与Catalan数

Catalan 数 C_n 的众多组合解释之间都可以建立双射，而其中最著名的就是 Dyck 路径。

7.3.1 括号匹配问题

n 对括号的合法匹配数等于 C_n。所谓"合法匹配"是指：从左到右扫描时，任何时候右括号的数量不超过左括号的数量，且最终左括号和右括号数量相等。

例如 n=3 时，合法匹配有 5 种：

((()))   (()())   (())()   ()(())   ()()()

7.3.2 山脉路径（Dyck路径）

Dyck 路径是从 (0,0) 到 (2n,0) 的路径，每一步只能向右上 ↗（记为 U，up）或向右下 ↘（记为 D，down），且路径始终不越过 x 轴（即纵坐标始终非负）。

括号匹配和 Dyck 路径之间存在一一对应：把 '(' 映射为 ↗（U），把 ')' 映射为 ↘（D）。合法括号序列的条件"任何时候右括号不超过左括号"正好对应"路径纵坐标非负"。最终左右括号数相等对应"纵坐标回到 0"。

所以 Dyck 路径数 = C_n

这个对应关系非常直观：

括号序列Dyck路径 (U/D)路径示意特点
((()))UUUDDD↗↗↗↘↘↘全部先升后降
(()())UUDUDD↗↗↘↗↘↘两次小山峰
(())()UUDDUD↗↗↘↘↗↘一个高峰后回平再起
()(())UDUUDD↗↘↗↗↘↘先小峰再高峰
()()()UDUDUD↗↘↗↘↗↘连续小波

Catalan 数 C_n = C(2n, n)/(n+1) 可以用"反射原理"从 Dyck 路径推导——这是下一节的内容。

7.4 折线法与反射原理

反射原理（Reflection Principle）是组合数学中一个非常优美的技巧，用于计算满足某些边界条件的路径数量。

7.4.1 从 (0,0) 到 (m,n) 不越过对角线的路径

考虑从 (0,0) 到 (a,b) 的格点路径，每一步向右或向上，总共有 C(a+b, a) 种可能。但如果要求路径永远不越过直线 y=x（即路径上的点恒满足 x ≥ y），满足条件的路径有多少？

这个问题的答案是：

当 a < b 时，不存在这样的路径（因为终点已经在对角线上方）
当 a ≥ b 时，数量为 C(a+b, a) - C(a+b, a+1)

特别地，当 a=b=n 时（即到 (n,n) 且不越过对角线）：
数量 = C(2n, n) - C(2n, n+1) = C(2n, n)/(n+1) = C_n

这正是 Catalan 数的另一种形式！

7.4.2 反射原理的直观

反射原理的核心是"坏路径的对称转化"：所有从 (0,0) 到 (a,b) 且触碰（越过）直线 y=x+1 的路径，通过"对第一次触碰点之后的路径做关于 y=x+1 的反射"，得到所有从 (-1,1) 到 (a,b) 的路径。而后者可以直接计算。

坏路径数 = C(a+b, a+1)
好路径数 = 总路径数 - 坏路径数 = C(a+b, a) - C(a+b, a+1)

7.4.3 Ballot问题（选举问题）

Ballot 问题是反射原理最经典的应用：

"在一次选举中，候选人 A 得到 a 票，候选人 B 得到 b 票，a > b。假设选票被随机计票，问在整个计票过程中 A 的票数始终严格多于 B 的票数的概率是多少？"

答案：P = (a - b) / (a + b)

证明：用折线法表示计票过程。横坐标表示已经统计的票数，纵坐标 A 票数与 B 票数的差。要求整个过程中纵坐标始终为正（A 始终领先）。这就是"不接触 x 轴"的路径数问题——用反射原理可得答案为 C(a+b-1, a-1) - C(a+b-1, a) = (a-b)/(a+b)·C(a+b, a)。

🔥 反射原理的精髓：反射原理让我们把"有边界的路径计数"转化为"无边界路径计数"——通过将那些触碰边界的路径进行对称反射，使它们变成另一个更容易计数的集合。这种方法不仅适用于路径计数，在随机游走、统计物理和博弈论中都有广泛应用。

7.5 实战中的踩坑故事

说一个我辅导学生时遇到的真实案例。有一次我出了一道题：

"有 8 个男生和 8 个女生站成一排，要求任何时刻，从左到右数，女生的数量不超过男生的数量（即始终男生不少于女生）。有多少种排法？"

一个学生立刻说："这就是 Catalan 数！C₈ = 1430。"

我说："你确定吗？Catalan 数的条件是——任何时刻前缀中 U 的数量 ≥ D 的数量。在你的问题中，把男生看作 U，女生看作 D，是不是一样？"

他说："对啊，所以就是 C₈。"

我又问："Catalan 数 C_n 对应的是 2n 个步长的 Dyck 路径——从 (0,0) 到 (2n,0)，始终在 x 轴上方。这里的男生和女生是 8 对 8，就是 2×8=16 个人，从 (0,0) 到 (16,0)，始终横坐标 ≥ 纵坐标，即前缀中 U ≥ D。所以是 C₈ = 1430。对吗？"

"对。"

"那我问你——如果第一个人是女生呢？"

"那就不符合条件了——因为第一个前缀中女生就比男生多了。"

"对。所以你是对的吗？"

"我……等等，Catalan 数是不是要求第一条边是 U（男生）？实际上，Dyck 路径确实必须从 U 开始——因为从原点出发，如果第一步是 D，纵坐标就变成 -1 了，不满足非负条件。所以 Catalan 数确实要求第一个是 U。所以 D₈ = 1430 就是正确答案。"

"好。现在我问你另一个问题——如果男生和女生是有区别的（即每个具体的人不同），结果会怎样？"

他愣住了。

⚠️ 踩坑解析：Catalan 数假设所有 U 步（男生）都是相同的，所有 D 步（女生）也是相同的。如果 8 个男生各有不同，8 个女生各有不同，那么总数应该是 C₈ × 8! × 8!。因为：

  • C₈ 种"模式"（U 和 D 的序列模式）
  • 在每个模式中，8 个不同的男生可以排列到 8 个 U 位置上：8! 种
  • 8 个不同的女生可以排列到 8 个 D 位置上：8! 种

所以正确答案是 C₈ × 8! × 8!，而不是仅仅 1430。1460 和 1430 × 40320 × 40320 之间差了近 16 亿倍！

这个学生的错误在于——他把 Catalan 数的"无区别元素"假设错误地应用到了"有区别元素"的问题中。在组合数学中，区分"相同的元素"和"不同的元素"永远是第一位的。

7.6 更多映射技巧

7.6.1 编码法

映射的另一种常见形式是"编码"——把要计数的对象用另一种形式表示出来。比如：

  • 二进制串编码子集：第 i 位为 1 表示第 i 个元素在子集中
  • 括号序列编码二叉树：用括号的嵌套结构表示二叉树的结构
  • 排列编码成 Lehmer 码：将一个排列映射为一个 n 位数，每位数字的范围不同

7.6.2 配对法

配对法的思想是：如果要证明两个集合大小相等，可以构造一个"配对规则"，说明每个 A 中的元素唯一对应一个 B 中的元素，反之亦然。例如：

"证明：n 元集合的奇子集个数等于偶子集个数。"

证明：固定一个元素 x。对于任意子集 S，如果 x∈S，则将其与 S\{x} 配对；如果 x∉S，则与 S∪{x} 配对。每一对恰好一个是奇子集，一个是偶子集。所以奇子集与偶子集一一对应，数量相等。

这个证明极其简洁，不需要任何公式——就是一对一的配对。

7.7 本章小结与过渡

今天我们学习了映射与对应法的三个步骤——构造映射、证明单射、证明满射。我们通过不相邻选数、Vandermonde 卷积、Dyck 路径与 Catalan 数、反射原理与 Ballot 问题等经典例子，展示了映射法在组合计数中的强大威力。我们也重点讨论了"有区别"和"无区别"元素区分的重要性——这个区分在使用映射法时尤其容易被忽视。

映射法是组合数学中最优美的思想之一——它完全绕过了复杂的代数运算，通过巧妙的构造直接揭示问题的本质。这也是为什么在 IMO 级别的组合题中，双射法往往是最受推崇的解法。
