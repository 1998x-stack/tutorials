第一章：数值分析概述

1.1 引言：为什么学数值分析

大家好，我是你们的讲师。在正式开始之前，我想先讲一个我亲身经历的"翻车"故事。

几年前，我参与了一个金融风控项目，团队用深度学习模型做信用评分。模型在训练集上表现完美——AUC 0.98，几乎没有误判。所有人都信心满满地准备上线。结果到了测试集，模型直接崩了：loss 在几个 epoch 后突然发散到 NaN，预测结果全是乱码。

团队排查了整整三天，试了调学习率、换激活函数、加正则化，全都没用。最后我多留了个心眼，打印出了协方差矩阵的条件数（condition number）。一看数值——10的14次方。这意味着特征矩阵几乎是奇异的，矩阵求逆时的误差被放大了十万亿倍。梯度在反向传播过程中已经被数值误差完全吞噬了。

根源是什么？是多个强相关的金融特征（收入、负债率、信用卡使用率）导致的近奇异矩阵。而我在数据预处理阶段，顺手做了一次标准化，完全没有检查特征之间的共线性。

关键洞察：这个错误和模型架构无关，和调参技巧无关，和 GPU 算力无关——它是一个纯粹的数值分析问题。在 2026 年的今天，我们身边充满了 PyTorch、TensorFlow、JAX 这样的自动微分框架，它们把底层的矩阵运算包装得天衣无缝。但这并不意味着你不需要理解背后发生了什么。恰恰相反——框架越方便，你越容易忽略那些"看不见的敌人"：舍入误差、截断误差、条件数、算法稳定性。它们平时安静地潜伏在浮点数的尾数里，一旦爆发，就能让你的模型从"完美"变成"废铁"。

这个故事的教训是：数值分析不是一门过时的数学课，它是每一个做计算的人——无论你是做深度学习、科学计算还是量化交易——都必须掌握的基本功。就像开车需要知道刹车和油门的区别一样，做计算需要知道你的数字从哪里来、可信度有多高、会在哪里失效。

好，现在你明白为什么学它了。下面我们正式进入正题。

1.2 什么是数值分析

数值分析（Numerical Analysis），简单说就是研究如何用计算机求解数学问题的学科。它的核心使命是：设计并分析能在有限精度计算机上可靠运行的数学算法。

数值分析的历史可以追溯到几百年前。17 世纪，Newton 发明了用迭代法求方程根的算法（Newton-Raphson 法），以及用均差计算多项式插值的方法。18 世纪，Euler 提出了 Euler 法来近似求解常微分方程，并发展了数值积分的思想。19 世纪，Gauss 贡献了最小二乘法、Gauss 消去法和数值求积公式——这些方法至今仍然是数值计算的基石。

但数值分析真正成为一门独立的学科，是在 20 世纪中叶计算机诞生以后。因为计算机带来了一种全新的挑战：你写的数学公式是连续的、精确的，但计算机只能处理离散的、有限精度的浮点数。这种"连续性"与"离散性"之间的鸿沟，就是数值分析要填平的。

数值分析的核心可以概括为三个关键词：

  • 逼近（Approximation）：用简单的函数（多项式、分段函数）去近似复杂的函数——因为计算机只会做加减乘除，不会算 sin(x) 或 e^x。
  • 迭代（Iteration）：用反复逼近的方式逐步收敛到精确解——因为大多数方程（比如 x = cos(x)）没有封闭形式的解析解。
  • 离散化（Discretization）：将连续的问题（积分、微分方程）转化为有限维的离散问题——因为计算机只能存有限个数字。

数值分析的思维方式：当你在纸上推导一个公式时，你活在"连续世界"——微积分、极限、无穷级数。当你把这个公式写到计算机里运行时，你进入了"离散世界"——浮点数、迭代次数、网格划分。数值分析告诉你如何在这两个世界之间架一座可靠的桥梁。

1.3 误差的来源与分类

误差（error）是数值分析最核心的概念。你可能会觉得："误差嘛，就是算出来的数和真值之间的差距。" 没错，但关键是——误差有不同来源，不同来源的误差需要不同的应对策略。

我们把误差的来源归纳为四大类：

  • 模型误差（Model Error）：你用的数学模型本身就不精确。比如用线性模型拟合非线性关系，用牛顿力学逼近相对论效应——这不是计算的问题，是建模的问题。
  • 观测误差（Observation Error）：输入数据本身就有测量误差。温度计的读数、传感器的输出、实验仪器的精度限制——"垃圾进，垃圾出"。
  • 截断误差（Truncation Error）：用有限项近似无穷过程产生的误差。比如用泰勒展开的前 5 项逼近 e^x，丢掉的高阶项就是截断误差。这是数值方法设计时主要考虑的量。
  • 舍入误差（Roundoff Error）：计算机浮点数有限精度造成的误差。比如 1/3 在计算机里存成 0.33333333...，尾数被截断的那部分就是舍入误差。

下面这张表把四类误差的方方面面做了对比：

误差类型 来源 能否消除 典型例子
模型误差 模型假设与现实的差距 不能完全消除，可降低 用线性回归拟合非线性数据
观测误差 测量工具的精度限制 可降低，不能消除 传感器噪声、数据采集错误
截断误差 用有限代替无穷 可降低（取更多项），不能消除 泰勒展开截断、数值微分
舍入误差 浮点数有限精度 理论上存在，无法消除 1/3 的二进制表示无限循环

对数值算法设计来说，截断误差和舍入误差是最关键的两个。它们之间存在一种"跷跷板效应"：你减小截断误差（比如增加泰勒展开的项数），计算量变大，舍入误差反而可能积累更多。找到一个最优的平衡点，是数值分析的艺术所在。

截断误差 vs 舍入误差的跷跷板：以数值微分为例，中心差分公式的截断误差是 O(h²)，看起来步长 h 越小越好。但当 h 太小（比如接近 10⁻⁸），两个非常接近的函数值相减，有效数字大量丢失，舍入误差反而主导了结果。这就是为什么数值微分不是简单地"h 越小越精确"——实际工程中有一个最优步长，通常是 h ≈ 10⁻⁵ 附近。

1.4 绝对误差、相对误差与有效数字

知道了误差的来源，下一步就是如何量化误差。数值分析中最常用的三个误差度量是：

  • 绝对误差（Absolute Error）：近似值与精确值的差的绝对值。公式：e = |x - x*|，其中 x 是精确值，x* 是近似值。
  • 相对误差（Relative Error）：绝对误差除以精确值的绝对值。公式：ε = |x - x*| / |x|。注意：当 x = 0 时相对误差无定义。
  • 有效数字（Significant Digits）：近似值中从第一个非零数字开始，能准确反映真值的位数。

下面用一个具体例子来对比这三个概念：

精确值 x 近似值 x* 绝对误差 相对误差 有效数字
π = 3.14159265... 3.14 ≈ 0.001593 ≈ 0.000507 (0.05%) 3 位
π = 3.14159265... 3.1416 ≈ 0.00000735 ≈ 0.00000234 (0.0002%) 5 位
1,000,000 999,996 4 0.000004 (0.0004%) 6 位
0.00123456 0.00123 0.00000456 ≈ 0.00369 (0.37%) 3 位

从这个表中你可以注意到几件有意思的事：

  • 绝对误差 4 在"1,000,000 vs 999,996"的例子中看起来很小，但相对误差只有 0.0004%，这个近似其实非常好。反之，绝对误差 0.001593 看起来不大，但相对误差 0.05% 在精密计算中可能不可接受。
  • 绝对误差没有告诉我们"这个近似有多好"——同样绝对误差 0.001，对于 π（约 3.14）来说相对误差是 0.03%，但对于 0.001 本身来说相对误差是 100%。所以科学计算中更看重相对误差。
  • 有效数字位数是相对误差的一种直观表达。一般来说，d 位有效数字意味着相对误差不超过 5 × 10⁻ᵈ。

工程经验：在数值分析中，永远优先关注相对误差，而非绝对误差。一个常见的陷阱是：当真实值非常接近零时，绝对误差很小，但相对误差可能大到离谱。例如，x = 1.0 × 10⁻¹⁰，x* = 1.1 × 10⁻¹⁰，绝对误差 1.0 × 10⁻¹¹ 看起来"很小"，但相对误差是 10%——这在很多场景下是不可接受的。

1.5 算法稳定性与收敛性

有了误差的概念，我们就可以讨论两个更高层次的议题：稳定性（Stability）和收敛性（Convergence）。

算法稳定性指的是：输入数据的微小扰动，是否会导致输出结果的巨大变化。一个稳定的算法对小的输入误差不敏感；一个不稳定的算法则会把一点点误差放大到面目全非的程度。

最经典的稳定性反例——也是每个数值分析初学者都应该记住的——就是"两个相近的数相减"（subtractive cancellation）。看下面这个 Python 例子：

import math

# 计算二次方程 ax^2 + bx + c = 0 的根
a, b, c = 1, 1000.001, 1

# 标准求根公式（可能发生灾难性抵消）
x1_naive = (-b + math.sqrt(b**2 - 4*a*c)) / (2*a)
x2_naive = (-b - math.sqrt(b**2 - 4*a*c)) / (2*a)

print(f"标准公式: x1 = {x1_naive:.15f}, x2 = {x2_naive:.15f}")

# 数值稳定的版本：利用 Vieta 定理
x2_stable = (-b - math.sqrt(b**2 - 4*a*c)) / (2*a)
x1_stable = c / (a * x2_stable)  # 利用 x1 * x2 = c/a

print(f"稳定版本: x1 = {x1_stable:.15f}, x2 = {x2_stable:.15f}")

# 真值（高精度参考）
x1_true = -0.001000001
x2_true = -999.999

print(f"\n真值参考: x1 = {x1_true:.15f}, x2 = {x2_true:.15f}")
print(f"标准公式 x1 的相对误差: {abs(x1_naive - x1_true)/abs(x1_true):.2e}")
print(f"稳定版本 x1 的相对误差: {abs(x1_stable - x1_true)/abs(x1_true):.2e}")

这段代码的运行结果会让你吃惊：标准求根公式算出的 x1 可能只有 1-2 位有效数字，而稳定版本的 x1 几乎与真值完全一致。原因就是 -b + sqrt(Δ) 中，b 和 sqrt(Δ) 的数值非常接近，相减时有效位数灾难性丢失。

关键教训：数学上等价的公式，在数值计算中可能天差地别。二次方程求根公式在数学上是perfectly correct的，但在计算机上，当判别式 sqrt(Δ) 和 -b 数值接近时，它会变成一个灾难。这就是为什么数值分析不是"把数学公式翻译成代码"——它要求在数学等价类中寻找数值稳定的那一个。

收敛性（Convergence）则关注的是：当我们用迭代法或逼近法时，随着计算步数增加或步长减小，结果是否越来越接近真值？一个算法如果收敛，它的误差应该随着计算量的增加而递减；如果不收敛，你算得再多也是白搭。

数值分析中的收敛性通常用阶（order）来衡量。如果一个算法的误差以 O(h^p) 的速度衰减（h 是步长），我们就说它是 p 阶收敛的。阶数越高，同样的步长缩小能带来更快的误差下降——但通常也意味着更高的计算成本。

稳定性与收敛性的关系：这是数值分析中一条重要的红线——稳定的算法才能收敛。一个不稳定的算法，即使截断误差在理论上趋于零，舍入误差的积累也会摧毁所有精度。回顾前面金融风控的故事：条件数 10¹⁴ 的矩阵，意味着求解线性方程组的算法在该问题上是不稳定的——无论你用什么求解器，输入数据再微小的不确定性（观测误差），都会被放大到足以淹没信号的程度。

好了，第一章到这里就结束了。这一章我们搭建了数值分析的知识框架：知道了它要解决什么问题，理解了误差的四种来源，学会了量化和衡量误差的方法，最后讨论了算法稳定性和收敛性这两个核心概念。
