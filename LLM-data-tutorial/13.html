数据污染问题

设想这样一个场景：你花了几百万美元训练了一个 LLM，在 MMLU 上取得了 85% 的惊人成绩，准备发 paper 庆祝。结果有人发现——你的训练数据里包含了 MMLU 的测试题目。模型不是"学会了推理"，而是"记住了答案"。

这听起来像是一个极端的假设，但现实比小说更离奇。GPT-3 论文中就已经讨论过这个问题——他们发现 Common Crawl 中包含了多个标准 NLP 评测集（如 SQuAD、MNLI）的测试数据。这些评测集本身就是 Web 页面（如 arXiv 论文），被 Crawl 抓取后混入了训练数据。

更令人震惊的是 Dolma 团队的发现。AI2 在构建 Dolma 语料库时做了系统的去污染分析，结果发现大量与 MMLU 高度重叠的样本。MMLU 的题目来自教科书和学术网站——而这些正是高质量数据过滤策略会"主动保留"的内容。换句话说，越努力筛选高质量数据，越可能不小心混入评测集内容。

这就是数据污染（data contamination）问题的核心矛盾：高质量数据过滤和评测集去污染，在目标上存在冲突。

n-gram 重叠检测

检测数据污染的标准方法是 n-gram 重叠检测。思路很简单：如果训练数据和评测数据共享了足够长的连续子串，就判定为"污染"。

最常用的设置是 13-gram 重叠检测（在字符级别）。为什么是 13？13 个字符大约相当于 2-3 个英文单词或 4-6 个中文字符。如果两段文本共享超过 13 个字符的完整子串，它们几乎不可能是"巧合"——必定是某种形式的复制或迁移。

也有团队使用更保守的 8-gram 设置（Dolma 就是 8-gram），这样可以捕获更多的潜在污染，但也会增加误报率。

def detect_contamination(train_doc: str, benchmark_data: list, n=13):
    """检测训练文档是否包含评测集数据的 n-gram 重叠

    Args:
        train_doc: 训练文档文本
        benchmark_data: 评测集样本列表（每个样本是 dict: {question, answer}）
        n: n-gram 长度

    Returns:
        (是否污染, 重叠的评测样本, 最长重叠 n-gram 长度)
    """
    def get_ngrams(text, n):
        return {text[i:i+n] for i in range(len(text) - n + 1)}

    train_ngrams = get_ngrams(train_doc, n)

    for sample in benchmark_data:
        benchmark_text = sample["question"] + " " + sample.get("answer", "")

        # 计算 n-gram 交集
        bench_ngrams = get_ngrams(benchmark_text, n)
        overlap = train_ngrams & bench_ngrams

        if len(overlap) >= 1:
            # 提取最长的重叠子串（用于人工审核）
            max_overlap_len = max(len(s) for s in overlap) + n - 1
            return True, sample, max_overlap_len

    return False, None, 0


# 对 Dolma 语料进行去污染检查
benchmarks = {
    "mmlu": load_mmlu(),
    "hellaswag": load_hellaswag(),
    "arc": load_arc_challenge(),
}

contamination_report = {}
for bench_name, samples in benchmarks.items():
    contaminated_count = 0
    for doc_id, doc_text in tqdm(dolma_docs.items()):
        is_contam, sample, overlap = detect_contamination(
            doc_text, samples, n=8
        )
        if is_contam:
            contaminated_count += 1
    contamination_report[bench_name] = contaminated_count

去污染的实际策略

检测到污染后，下一步是处理。行业内有三种主要策略：

策略一：基于子串匹配的严格去污染。如果一篇训练文档包含了完整的评测集问题（或者连续 50+ 字符的重叠），就认为该文档被污染，整个文档被移除。这是最激进的方法，也是 Dolma 的默认策略。

策略二：基于模糊匹配的宽松去污染。语义相似但不完全相同的变体——比如同一个问题被改写成了不同的措辞——也会被视作污染。这种检测更困难，通常需要 embedding 相似度检索（如将文档和评测集问题编码为向量，计算余弦相似度）。

策略三：部分删除（温和策略）。不删除整篇文档，只删除与评测集重叠的句子或段落。这可以保留文档中其他有价值的内容，但实现更复杂——需要精确定位重叠区域。

残酷的现实： 去污染是"防君子不防小人"。13-gram 或 8-gram 检测只能捕获"一字不差"的复制，对于"改写了措辞但意思一样"的污染毫无办法——而这恰恰是人类评测者最容易作弊的方式。模糊匹配虽然理论上更强大，但计算成本高好几个数量级。行业目前的共识是：不完美的去污染仍然是必须做的，基线级别的 13-gram 检测是底线。

去污染的代价

去污染不是没有代价的。当你删除一篇"被污染"的训练文档时，也在删除其中的非污染部分。Dolma 的分析显示，被标记为污染的文档中，平均只有 15-30% 的内容实际与评测集重叠，其余 70-85% 是正常内容。

另一个问题是过度去污染导致的安全感幻觉。如果模型是真的"学会"了某个领域知识而非"记住"了题目，那包含一些与评测集"看起来相似"的训练数据其实是合理的。比如，MMLU 的题目来自教科书，而训练数据中也包含教科书——这不是"作弊"，而是正常学习。但现有的 n-gram 方法无法区分"学到了知识"和"记住了答案"。

更有趣的是开卷 vs 闭卷的争论：在实际使用中，很多 LLM 被设计为配合搜索引擎或 RAG 系统工作，它们本来就应该具备"查找已有信息"的能力。对于这些场景，训练数据包含评测集内容的"污染"反而变成了"合理的预训练曝光"。

开源透明： The Pile 团队践行了开放科学的最佳实践——他们不仅开源了去污染工具（在 GitHub 上可以找到 pile_dedup 仓库），还公开了所有被污染文档的 ID 列表。这样其他研究者可以精确知道哪些数据被移除了，也可以自行决定是否要对某些"轻度污染"网开一面。这种透明度在去污染工作中至关重要，因为不同项目对"污染"的接受程度不同。

评测集去污染是现代 LLM 评估体系中不可或缺的一环。不做去污染的模型分数，就像没有对照组的实验——你永远不知道模型的真实能力是多少。好在 n-gram 检测实现简单、成本低，属于"低成本高回报"的数据工程步骤。

经过清洗、去重、质量过滤和去污染后，每一篇保留下来的训练数据都经过了层层考验。但还有一个关键决策没有做：这些来自不同源头的数据，应该按什么比例混合在一起？Web 数据、代码、书籍、论文、Wikipedia——每种来源贡献的知识类型不同，如何搭配才能让模型全面发展？这就是下一章的主题——多源数据混合配比。
