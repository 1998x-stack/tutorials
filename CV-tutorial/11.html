11.1 当NLP的"王牌"闯入CV

2017年，Google在论文"Attention Is All You Need"中提出了Transformer架构。它彻底改变了NLP的面貌——BERT、GPT系列模型都建立在Transformer之上。但直到2020年，CV领域仍然是以CNN为主导的。大多数CV研究者认为：Transformer虽然适合NLP的序列数据，但图像是二维的、具有空间结构的数据，CNN的归纳偏置（局部连接、平移不变性）是不可或缺的。

被打破。2020年10月，Google Brain团队在ICLR 2021上发表了Vision Transformer（ViT）。这篇论文做了一个极其大胆的尝试：把图像当作NLP中的句子来处理——把图像切成固定大小的patch，把每个patch线性投影成一个"token"，然后直接用标准的Transformer Encoder来处理这些token序列。

结果让所有人震惊：在不使用任何卷积操作的情况下，ViT在ImageNet上的Top-1准确率达到了88.55%，超过了当时最好的CNN（EfficientNet的88.35%）。更令人惊讶的是，在大规模数据集（如ImageNet-21K、JFT-300M）上预训练之后，ViT的迁移学习效果全面超越了CNN，而且随着数据规模的增大，ViT的性能仍在提升，而CNN已经饱和了。

2021年初，我在论文审稿中第一次看到ViT的预印本时，第一反应是"这不可能"。我认为Transformer缺少CNN的平移不变性和局部连接性，不可能在图像任务上做得和CNN一样好。后来我亲自在ImageNet上复现了ViT的实验，在JFT预训练的ViT-Huge确实达到了比我当时最好的ResNeSt模型高出将近2个百分点的精度。那次实验之后，我明白了：有时候我们以为的"先验知识"（比如卷积对图像是必要的），其实只是我们的"偏见"。如果有足够的数据，模型可以自己学会这些"归纳偏置"，而不需要我们通过架构设计硬塞给它。

11.2 Transformer基础回顾

在深入ViT之前，让我们快速回顾一下Transformer的核心组件。如果你已经熟悉Transformer，可以跳过本节；如果不熟悉，下面这张表给出了核心概念：

组件功能数学形式
自注意力（Self-Attention）计算序列中每个元素与其他元素的相关性Attention(Q,K,V) = softmax(QK^T / √d) · V
多头注意力（Multi-Head Attention）并行计算多个自注意力，捕捉不同的关系模式MultiHead(Q,K,V) = Concat(head_1,...,head_h) · W_O
位置编码（Position Encoding）为序列中的每个位置注入位置信息正弦/余弦函数 或 可学习的嵌入向量
前馈网络（FFN）每个位置的独立非线性变换FFN(x) = GELU(xW_1 + b_1) · W_2 + b_2
层归一化（Layer Normalization）对每个样本的特征维度做归一化LN(x) = (x - μ) / σ * γ + β

自注意力的核心是QKV机制：每个token被映射为三个向量——查询（Query）、键（Key）和值（Value）。Q与所有K做点积，经softmax得到注意力权重，再用这些权重加权求和所有的V。这个过程让每个token都能"关注"到序列中所有其他位置的token，形成全局感受野。

11.3 ViT的架构详解

ViT的整个流程可以分为三个步骤：Patch Embedding → Transformer Encoder → Classification Head。

11.3.1 Patch Embedding：图像 → Token序列

ViT的第一步是将图像转换成一系列token。对于一张224×224×3的输入图像，ViT将图像分割成固定大小的patch（通常是16×16）：

# Patch Embedding 过程（以 ViT-Base/16 为例）
# 输入: [1, 3, 224, 224]   (1张图像, RGB三通道, 224×224)
#
# Step 1: 将图像分割为 (224/16) × (224/16) = 14 × 14 = 196 个patch
# 每个patch的大小: 16 × 16 × 3 = 768 个像素值
#
# Step 2: 用可训练的线性投影将每个patch映射到 D 维向量
# Conv2D(16×16, stride=16) 或 nn.Linear(768, D)
#
# Step 3: 加上位置编码（可学习的），保留空间位置信息
#
# Step 4: 在序列开头添加一个特殊的 [CLS] token（用于分类）
# 最终序列长度: 196 (patches) + 1 ([CLS]) = 197
# 每个token的维度: D（Base版本 D=768）
#
# 输出: [1, 197, 768]  —— 一个由197个token组成的序列

# BERT风格的 [CLS] token 对应最终的分类输出，
# 其他patch token的对应位置被忽略（除非做分割等密集预测任务）

11.3.2 Transformer Encoder：标准结构

得到token序列后，ViT将其送入标准的Transformer Encoder（与BERT的编码器几乎一模一样）。每个Encoder Block包含：

# 一个ViT Encoder Block
def vit_block(x):
    # 1. 层归一化
    x_norm = LayerNorm(x)

    # 2. 多头自注意力（全局交互）
    attn_out = MultiHeadSelfAttention(x_norm)

    # 3. 残差连接
    x = x + attn_out

    # 4. 层归一化
    x_norm = LayerNorm(x)

    # 5. MLP（前馈网络）：两个全连接层，中间用GELU激活
    #   MLP扩展比为4（D → 4D → D）
    mlp_out = Linear(GELU(Linear(x_norm, D→4D)), 4D→D)

    # 6. 残差连接
    x = x + mlp_out

    return x

# ViT-Base/16 使用12个这样的Block堆叠

11.4 ViT vs CNN：全局vs局部

ViT和CNN最本质的区别在于感受野和归纳偏置：

  • CNN是局部操作的：卷积核只关注一个局部邻域（如3×3），通过层层堆叠逐步扩大感受野。CNN自带两个重要的归纳偏置——局部连接性（相邻像素有强相关性）和平移不变性（物体在图像中移动后依然能被识别）。
  • ViT是全局操作的：自注意力机制从第一层开始就让每个patch关注所有其他patch。ViT没有施加任何空间上的先验——它需要从数据中自己学会哪些空间关系是重要的。

这种差异带来的影响是双面的。好处是ViT可以捕捉远程依赖关系（long-range dependency）——CNN需要十几层才能关联到的两个远距离像素，ViT在第一层就能做到。坏处是ViT缺少CNN的"数据效率"——在没有足够数据的情况下，纯Transformer在中等规模数据集（如ImageNet-1K）上表现不如同等等级的CNN。这就是为什么ViT论文的核心发现之一是：ViT需要在大规模数据上预训练才能发挥优势。

🔄 ViT的"数据饥饿"问题：ViT在ImageNet-1K（120万张图像）上从头训练，比同尺寸的ResNet/BiT差3-4个百分点。但在JFT-300M（3亿张图像）上预训练后，ViT在ImageNet上的微调结果全面超越CNN。这说明：CNN的归纳偏置是一种"数据效率"的奢侈——当你数据足够多时，模型可以自己学会这些先验，而且做得比人为设计的偏置更好。这也是"Scaling Law"在视觉领域的体现：更大的模型 + 更多的数据 = 更好的性能，这个规律对Transformer特别明显。

11.5 ViT的意义与局限

11.5.1 革命性意义

ViT的发表彻底改变了CV的研究方向。它的意义至少体现在三个方面：

  • 证明了纯注意力机制可以超越CNN。在ViT之前，主流观点认为卷积操作是图像理解所必需的。ViT用实验证明了这不是——只要有足够的算力和数据，注意力机制可以做得更好。这打破了CNN在视觉领域的垄断地位。
  • 统一了视觉和语言的模型架构。ViT使用与NLP完全相同的Transformer架构，这意味着视觉模型和语言模型终于可以共享同一个架构。这为CLIP、GPT-4V等多模态模型铺平了道路——它们不再需要为视觉和语言分别设计不同的backbone。
  • 推动了视觉预训练范式的转变。ViT之后，大规模预训练在CV领域变得和NLP一样重要。从ViT到MAE（Masked Autoencoder）、CLIP、DINO，视觉领域的"预训练→微调"范式逐渐成熟，彻底改变了CV的研究方式。

11.5.2 主要局限

  • 计算复杂度高：自注意力的计算量随序列长度平方增长（O(N²)）。对于224×224的图像（196个patch）还好，但对于高分辨率图像或密集预测任务（需要小patch），计算量会失控。
  • 缺少空间层次性：CNN通过池化逐步降低分辨率、增加通道数，天然形成"多尺度特征金字塔"。ViT在patch级别上是均匀处理的，缺乏这种层次化结构。
  • 数据需求高：如前所述，在没有足够数据的情况下，ViT不如CNN。
  • 位置编码受限：ViT使用固定分辨率的位置编码，当输入图像分辨率改变时，需要插值位置编码，这可能导致性能下降。
⚠️ 踩坑故事：2022年我尝试将ViT应用于一个医学图像分析任务——病理切片图像（WSI，Whole Slide Image）。典型的WSI分辨率高达100000×100000像素。如果用ViT的16×16 patch，token数量是 (100000/16)² ≈ 3900万个——quadratic attention的内存消耗是O(N²)，连A100 80GB都扛不住。我不得不先用一个CNN将WSI分割成小区域提取特征，再把压缩后的特征送入ViT——也就是这种"CNN缩小→Transformer处理"的两阶段方案。这个经历让我深刻理解了ViT在密集预测和高分辨率任务上的局限。后来Swin Transformer通过窗口注意力（window attention）解决了部分问题，这是后话。

11.6 ViT的后续发展

ViT开启了视觉Transformer的研究热潮，随后涌现了大量改进工作：

  • DeiT（Data-efficient Image Transformers，2020）：Facebook提出的知识蒸馏方法，让ViT可以在ImageNet-1K上从零训练，无需JFT-300M。核心技巧是添加一个"蒸馏token"，让student模型从teacher CNN（如RegNet）学习CNN的归纳偏置。DeiT让ViT的实用门槛大幅降低。
  • Swin Transformer（2021，CVPR 2022最佳论文）：微软亚洲研究院提出的层次化Transformer。通过窗口注意力（在局部窗口内计算注意力）和移位窗口操作，Swin在保持全局建模能力的同时大幅降低了计算复杂度。它的层次化结构（patch merging）使得它天然适合检测和分割等密集预测任务。
  • PVT（Pyramid Vision Transformer，2021）：类似Swin的层次化Transformer，设计了一个金字塔结构的ViT，循序渐进地减少序列长度、增加通道数，适合作为检测和分割的backbone。
  • MAE（Masked Autoencoder，2021）：Kaiming He团队的ViT自监督学习方法，通过随机遮挡75%的patch并重建它们，让ViT学到高效的视觉表示。MAE表明ViT非常适合做自监督学习。

ViT的出现标志着CV领域进入了一个新的时代。它告诉我们：不要迷信任何特定的架构设计——卷积很好，注意力也很好，关键在于理解它们各自的优势和局限，在合适的场景下做出合适的选择。更重要的是，ViT为多模态大模型的崛起打下了基础——如果没有ViT将视觉和语言统一到同一个架构上，今天的GPT-4V、Gemini等视觉语言模型可能还需要走更长的路。

下一章，我们将回到CNN的世界，但这次是一个"现代化"的CNN。ConvNeXt向我们展示了一个有趣的视角：如果把CNN的各个组件按照Transformer的思路进行现代化改造，CNN可以达到和Transformer同等等级的性能。这个故事告诉我们：CNN和Transformer可能正在趋同。我们下一章见！
