从分类到检测的跨越——分类 vs 定位 vs 检测

在前面的章节中，我们花了大量篇幅讨论图像分类——给定一张图，告诉我们是猫还是狗。坦率地说，分类在某种程度上是一个"偷懒"的任务：你只需要回答"这是什么"，而完全不需要回答"它在哪"。但在真实世界中，"在哪"往往比"是什么"更重要。

让我讲一个亲身经历的教训。2018年，我给一家工厂做质检系统，目标是检测流水线上的螺丝是否安装到位。一开始我们按分类的思路做：在传送带正上方架一个摄像头，拍一张图，用ResNet分类"合格/不合格"。在实验室测试中准确率高达99.5%，老板当场拍板部署。结果上线第一天就出了大问题——螺丝明明安装到位，但系统连续报了十几个"不合格"。排查了半天才发现原因：传送带上的零件位置有微小偏移，导致螺丝没有出现在图像的正中央。我们的分类模型根本没学"螺丝在不在那个孔里"，它学的是"整张图的全局纹理模式"。零件一偏移，全局纹理变了，模型就懵了。这个教训让我深刻理解了：分类回答"是什么"，检测才回答"在哪里+是什么"。

让我们明确三者的区别：

任务输入输出典型问题
图像分类一张图像单个类别标签"这张图里是猫还是狗？"
图像定位一张图像单个边界框 + 类别标签"猫在哪？请框出来。"
目标检测一张图像多个边界框 + 各自的类别标签"图里有哪些物体？它们分别在什么位置？"

分类假设图中只有一个主要物体。定位假设图中只有一个目标物体，输出它的位置。检测则面对最复杂的情况：图中可能有任意数量的物体，需要全部找到并分类。你可以把检测理解为"分类 + 定位"的多次组合，但这里的挑战在于：你不知道图像中有多少个物体，它们的大小、形状、位置都不固定——这正是检测问题最核心的难点。

边界框表示

在讨论如何检测之前，首先要定义"怎么描述一个物体的位置"。最常用的方法就是边界框（Bounding Box）——用一个矩形框把目标物体圈起来。看似简单，但边界框的表示方式有多种，初学者往往在这里踩坑。

常见的矩形框格式

在CV领域，主要有四种边界框表示格式：

格式参数示例说明
xyxy(x1, y1, x2, y2)(50, 30, 200, 180)左上角和右下角坐标，直观但宽度/高度需计算
xywh(x, y, w, h)(50, 30, 150, 150)左上角坐标 + 宽高，标注工具和YOLO系列常用
cxcywh(cx, cy, w, h)(125, 105, 150, 150)中心点坐标 + 宽高，Faster R-CNN等检测框架偏好
xyxy_norm(x1, y1, x2, y2)(0.1, 0.05, 0.8, 0.9)归一化到[0,1]的坐标，与图像分辨率解耦

不同框架对格式的偏好不同，而格式转换是每个CV工程师的基本功。一个在xyxy和cxcywh之间搞反坐标的bug，可能让你浪费整整一个下午的调试时间。

# 边界框格式转换示例
def xyxy_to_cxcywh(bbox):
    """(x1, y1, x2, y2) → (cx, cy, w, h)"""
    x1, y1, x2, y2 = bbox
    cx = (x1 + x2) / 2
    cy = (y1 + y2) / 2
    w = x2 - x1
    h = y2 - y1
    return (cx, cy, w, h)

def cxcywh_to_xyxy(bbox):
    """(cx, cy, w, h) → (x1, y1, x2, y2)"""
    cx, cy, w, h = bbox
    x1 = cx - w / 2
    y1 = cy - h / 2
    x2 = cx + w / 2
    y2 = cy + h / 2
    return (x1, y1, x2, y2)

# 注意：所有坐标都应在同一坐标系下（像素或归一化）
# 混合使用不同坐标系是常见错误来源

标注格式与数据集规范

不同的目标检测数据集使用不同的标注格式。以最常用的两个数据集为例：PASCAL VOC用XML文件存储标注，每个物体保存为<bndbox>节点，使用xyxy格式；MS COCO则用JSON文件存储，每个标注包含bbox字段，使用[x, y, w, h]格式，且坐标是绝对像素值。这种不一致性在做数据集转换时尤其让人头疼——我在一个项目中被COCO和VOC的xyxy/xywh差异坑了整整两天，最后发现只是格式问题而非模型问题。

⚠️ 踩坑实录：COCO的bbox格式是(x, y, w, h)，其中(x, y)是框的左上角。而很多论文提到的"标准xyxy"是(x1, y1, x2, y2)。我在一个项目中把COCO的bbox直接当成xyxy传给NMS函数，结果mAP直接从0.75掉到0.12——所有框的位置都错了。排查了整整一天才发现是格式问题。切记：任何时候进行格式转换，都要写单元测试验证。

IoU与评估指标

有了边界框的定义，下一个问题自然产生：如何衡量一个预测框的好坏？换句话说，"框得准不准"怎么量化？

IoU（交并比）

IoU（Intersection over Union，交并比）是目标检测中最核心的度量指标。它计算预测框与真实框（Ground Truth）的交集面积与并集面积之比：

def compute_iou(box1, box2):
    """计算两个xyxy格式边界框的IoU"""
    # 交集区域
    x1 = max(box1[0], box2[0])
    y1 = max(box1[1], box2[1])
    x2 = min(box1[2], box2[2])
    y2 = min(box1[3], box2[3])

    inter_area = max(0, x2 - x1) * max(0, y2 - y1)
    if inter_area == 0:
        return 0.0

    # 各自面积
    area1 = (box1[2] - box1[0]) * (box1[3] - box1[1])
    area2 = (box2[2] - box2[0]) * (box2[3] - box2[1])

    # IoU = 交集 / 并集
    iou = inter_area / (area1 + area2 - inter_area)
    return iou

# 示例：两个完全重叠的正方形框
print(compute_iou([0, 0, 10, 10], [0, 0, 10, 10]))  # 1.0
# 完全不重叠
print(compute_iou([0, 0, 10, 10], [20, 20, 30, 30]))  # 0.0
# 部分重叠
print(compute_iou([0, 0, 10, 10], [5, 5, 15, 15]))  # 0.1428...

IoU的取值范围是[0, 1]。在实际应用中，通常认为IoU > 0.5的预测框是"可用"的，IoU > 0.7是"精准"的。但不要机械地套用这些阈值——在密集小物体检测场景中，0.5的IoU可能已经是非常好的结果了。

mAP（Mean Average Precision）

mAP是目标检测领域最权威的评估指标，但也是初学者最容易困惑的概念。理解mAP需要先理解三个子概念：

  • Precision（精确率）：预测为正的样本中，真正为正的比例。即"你框出的物体里，有多少是真的对了？"
  • Recall（召回率）：所有真实为正的样本中，被正确预测出来的比例。即"所有该框的物体，你框出了多少？"
  • AP（Average Precision）：在不同召回率水平下，对精确率取平均值。相当于Precision-Recall曲线下的面积。

计算AP时，首先要确定一个IoU阈值来判断"预测是否正确"。例如mAP@0.5表示以IoU=0.5作为判断阈值计算AP。mAP@0.5:0.95则更严格——在IoU从0.5到0.95（步长0.05）共10个阈值下分别计算AP，再取平均。这就是COCO数据集的主要评估指标，也被称为"标准mAP"。

指标含义特点
mAP@0.5IoU阈值=0.5时的mAP较宽松，适合评估"大致定位"能力
mAP@0.5:0.95IoU从0.5到0.95取10个阈值的mAP平均严格全面，COCO官方主要指标
AP_small小物体（面积<32²）上的AP专门评估小物体检测能力
AP_medium中等物体（32² < 面积 < 96²）上的AP评估中等大小物体
AP_large大物体（面积>96²）上的AP评估大物体检测
ARAverage Recall，平均召回率关注"漏检率"，每张图给定检测数量上限
💡 实操建议：在实际项目中，不要只看mAP这一个指标。如果mAP@0.5很高但mAP@0.5:0.95很低，说明你的模型只会"大概定位"而做不到"精准定位"——这在自动驾驶等场景中是不可接受的。反之如果mAP@0.5低但mAP@0.5:0.95相对不低，说明模型对已检出的物体定位精准（召回不高但精度高），这时应该优先提升召回率。

非极大值抑制（NMS）

模型在预测时，往往会对同一个物体输出多个重叠的预测框——它"不太确定"到底哪个框最准，所以给了好几个候选。我们需要一种机制来"合并同类项"，只保留最可信的那个框。这就是非极大值抑制（Non-Maximum Suppression, NMS）的由来。

NMS的原理

NMS的算法非常直观：

  1. 将所有预测框按置信度从高到低排序。
  2. 选择置信度最高的框，保留它，并移除所有与它IoU超过阈值（通常0.5或0.7）的其他框。
  3. 在剩余的框中重复步骤2，直到所有框都被处理过。
def nms(boxes, scores, iou_threshold=0.5):
    """
    非极大值抑制
    boxes: List of [x1, y1, x2, y2]
    scores: List of confidence scores
    """
    if len(boxes) == 0:
        return []

    # 按置信度降序排列
    indices = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)
    keep = []

    while indices:
        # 保留当前最高分的框
        i = indices[0]
        keep.append(i)

        # 计算其余框与当前框的IoU
        rest = indices[1:]
        to_remove = []
        for j in rest:
            iou = compute_iou(boxes[i], boxes[j])
            if iou > iou_threshold:
                to_remove.append(j)

        # 移除IoU过高的框
        indices = [idx for idx in rest if idx not in to_remove]

    return keep

阈值选择与Soft-NMS

NMS的IoU阈值需要精心调参。阈值太低（如0.3），两个挨得很近的真实物体可能被误删一个（比如人群中挨着的两个人）。阈值太高（如0.9），同一个物体可能保留多个冗余框，导致误检。在实际项目中，这个阈值需要根据你的场景反复调试。

Soft-NMS是NMS的一个优雅改进。它的核心思想是：不要粗暴地删除与高置信度框重叠的其他框，而是根据重叠程度降低它们的置信度。重叠越大的框，置信度衰减越严重。这样一来，两个高度重叠的真实物体不会因为IoU超过阈值就被"一刀切"地删除——它们的置信度虽然降低但不会被清零，只需要再设一个较低的置信度阈值来过滤即可。

🔧 工程经验：NMS是目标检测的"最后一公里"——模型再好，NMS参数没调好，最终结果也会很差。我在一个密集人群检测项目中发现，将NMS阈值从0.5调整到0.3，mAP从0.72降到了0.48，因为很多人挨得太近被误删了。反之，把阈值调到0.7，mAP也只提升到0.74，但假阳性翻了一倍。最终的"甜点"是0.55。所以记住：永远不要使用默认NMS参数而不经过验证。

滑动窗口与候选区域

在深度学习时代之前，目标检测的主流范式是滑动窗口（Sliding Window）：用一个固定大小的窗口，以固定的步长在图像上滑动，每个窗口位置提取特征进行分类。窗口大小不够怎么办？那就构建图像金字塔——把图像缩放成不同尺度，在每个尺度上分别滑动窗口。

这种方法的缺陷是显而易见的：

  • 计算量爆炸：假设一张800×600的图像，窗口大小64×64，步长8像素。单尺度就需要约(800-64)/8 × (600-64)/8 ≈ 8500个窗口。金字塔再加4个尺度，就是42500个窗口。每个窗口都要过一遍分类器——这在计算上是灾难性的。
  • 窗口尺寸僵化：固定窗口只能检测固定长宽比和大小的物体。要检测不同形状的物体，需要多个不同尺寸的窗口，进一步放大计算量。
  • 冗余严重：相邻窗口的内容高度重叠，提取的特征几乎一样，造成大量的重复计算。

正是这些局限催生了候选区域（Region Proposal）方法的出现。与其在整张图上穷举滑动，不如先用一种轻量级的方法找出"可能含有物体的区域"，然后只在这些区域上进行分类。这种方法大大减少了需要处理的窗口数量——从数万个降到一两千个。最具代表性的候选区域方法就是Selective Search，它是下一章主角R-CNN的重要组成部分。

目标检测发展脉络

了解目标检测的方法论谱系，有助于你理解后续四章的内容。从宏观上看，检测方法的发展经历了三次重要的范式转换：

两阶段 vs 单阶段

这是检测领域最经典的方法论分野：

  • 两阶段检测器（Two-Stage）：第一阶段生成候选区域（Region Proposal），第二阶段对每个候选区域进行分类和回归。代表：R-CNN系列（R-CNN → Fast R-CNN → Faster R-CNN → Mask R-CNN）。特点是精度高，但速度慢。
  • 单阶段检测器（One-Stage）：直接在图像上预测边界框和类别，没有独立的候选区域生成阶段。代表：YOLO系列、SSD、RetinaNet。特点是速度快，但早期精度不如两阶段方法。

这个区分在2016-2019年间非常鲜明，但随着RetinaNet（2017）引入Focal Loss、YOLOv4（2020）引入CSP结构和Mish激活函数，单阶段方法在精度上已经追平甚至超越了同期的两阶段方法。两者的界限也在逐渐模糊。

Anchor-Based vs Anchor-Free

这是检测领域的第二种重要分类方式：

  • Anchor-Based：预定义一组固定大小和长宽比的"锚框"（anchor boxes），模型预测的是每个锚框的偏移量。Faster R-CNN的RPN首次引入anchor机制，之后成为检测器的标准配置。
  • Anchor-Free：不依赖预定义的锚框，直接预测物体的关键点（如角点、中心点）或像素级位置。代表：CornerNet（2018）、FCOS（2019）、CenterNet（2019）。Anchor-Free方法简化了超参数设计（不再需要调整anchor的尺寸、数量），但在某些场景下召回率略逊于Anchor-Based方法。

DETR与端到端检测

2020年，Facebook AI团队提出的DETR（Detection Transformer）开启了两条新的路线：第一，将Transformer引入视觉检测，不再依赖CNN的anchor和NMS后处理；第二，实现了真正的端到端检测——直接用集合预测（set prediction）的方式输出所有物体，不需要任何人工设计的后处理组件。DETR的核心思想将在第18章详细展开，这里先留一个悬念。

方法族代表模型核心思想速度精度
传统方法DPM、HOG+SVM手工特征 + 滑动窗口慢低
两阶段Faster R-CNNRPN提候选 + RoI分类回归中高
单阶段YOLO、SSD密集采样 + 直接预测快中→高
Anchor-FreeFCOS、CenterNet点/关键点驱动快高
端到端DETR、Deformable DETRTransformer + 集合预测慢高

好了，目标检测的基础知识就铺垫到这里。你现在已经理解了检测要解决什么问题、如何用数学语言描述它、怎么评估检测的好坏、以及NMS这个必备的后处理工具。最重要的是，你心里有了检测方法发展的"路线图"。

从下一章开始，我们将沿着这条路线图，逐个深入最经典的检测模型。第一站就是所有现代检测器的鼻祖——R-CNN。它虽然在今天看来已经被更快的模型取代，但它的设计理念和思路深刻地影响了后续所有检测方法。我们下一章见！
