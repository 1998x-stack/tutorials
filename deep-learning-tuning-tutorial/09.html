权重衰减——比看起来更深刻的技巧

大家好，我是你们的讲师。今天我们来拆解一个看起来很简单、但背后藏着一个"数学陷阱"的技巧——权重衰减（Weight Decay）。

先讲一个真实的故事。某团队在做NLP模型微调时，怎么调都过拟合：训练loss降到0.02，验证loss却在0.5附近徘徊。他们试了各种办法——加dropout、减少模型层数、早停——效果都不明显。后来一个资深工程师看了一眼他们的优化器配置，说了一句："你们用的是Adam吧？weight decay设了多少？""默认的0.01。""改成AdamW试试。"

就这么改了优化器一行代码，什么都没变，验证loss降到了0.3。团队很兴奋，但工程师说了一句意味深长的话："你们以为只是在加正则化，实际上Adam的L2正则化对自适应学习率做了你不想让它做的事。AdamW就是把这件事纠正过来。"

这就是我们今天要讲的核心——权重衰减不仅仅是加一个惩罚项那么简单，它与优化器的交互方式决定了它到底有没有在"正确地"工作。

L1 vs L2正则化的几何直觉

在理解权重衰减的"坑"之前，先要理解它本身是什么。标准形式是在损失函数中加入参数惩罚项：

# L2正则化（岭回归）
loss = original_loss + 0.5 * λ * ||θ||²
# 梯度更新：θ = θ - lr * ∇original_loss - lr * λ * θ

# L1正则化（Lasso）
loss = original_loss + λ * |θ|
# 梯度更新：θ = θ - lr * ∇original_loss - lr * λ * sign(θ)

这两种正则化的几何直觉非常直观。假设你有一个二维参数空间，损失函数的等高线呈椭圆分布。L2正则化相当于在原点周围画了一个圆形约束——最优解被拉向原点，所有参数均匀缩小。而L1正则化画了一个菱形约束——菱形的尖角在坐标轴上，最优解更容易落在坐标轴上，使得部分参数变为0，产生稀疏解。

这就是为什么L2适合"所有特征都有用，但权重不要太大"的场景，而L1适合"大部分特征没用，我们要做特征选择"的场景。在深度学习中，L2（权重衰减）是绝对主流，因为深度学习模型通常所有参数都有信息量，我们只需要防止它们过大导致过拟合。

一个小技巧：在PyTorch中，bias和LayerNorm/BatchNorm的参数通常不施加weight decay。因为它们的数值范围本就不大，正则化它们没有意义。需要手动将参数分组——把normalization参数和bias单独放在一个weight_decay=0的参数组里。LLaMA、GPT等主流模型都采用这种做法。

AdamW修正了什么

现在到了最核心的部分：为什么Adam中的L2正则化不等价于权重衰减？

要理解这个问题，先看SGD中的情况。在SGD中，每一步更新是：

# SGD + L2正则化（等价于权重衰减）
θ = θ - lr * ∇original_loss - lr * λ * θ
  = θ * (1 - lr * λ) - lr * ∇original_loss  # 先衰减权重，再梯度更新

在SGD中，L2正则化相当于每次更新前先把权重缩小λ * lr的比例——这就是"权重衰减"字面意思。两者完全等价。

但在Adam中，情况完全不同。Adam的自适应学习率会根据参数的梯度大小来缩放每个参数的更新步长——梯度大的参数更新慢，梯度小的参数更新快。当L2正则项的梯度（λ*θ）也被Adam的自适应学习率缩放了，就会导致：大梯度参数（对应高频特征）的权重衰减被抑制，小梯度参数（对应稀疏特征）的权重衰减被放大。这根本不是我们想要的正则化效果。

AdamW（2019年，Loshchilov & Hutter）的修正方法极其简单但有效：把权重衰减从Adam的梯度更新中解耦出来，在Adam的自适应更新之后独立执行一个简单的权重缩小。

# AdamW vs Adam + L2 —— PyTorch实现对比

# ❌ 错误做法：Adam + L2正则化（耦合的）
# L2梯度被Adam的自适应LR缩放，导致不均匀正则化
optimizer = torch.optim.Adam(model.parameters(), lr=1e-4, weight_decay=0.01)
# weight_decay在这里实现为L2正则化，和Adam耦合

# ✅ 正确做法：AdamW（解耦的权重衰减）
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4, weight_decay=0.01)
# weight_decay在这里实现为真正的权重衰减，与梯度更新解耦

# 两者的内部差异如下：
# Adam + L2（耦合）:
#    grad = grad + weight_decay * param
#    m = beta1 * m + (1-beta1) * grad
#    v = beta2 * v + (1-beta2) * grad^2
#    param = param - lr * m / (sqrt(v) + eps)

# AdamW（解耦）:
#    grad = original_grad  # 不加入weight_decay项
#    m = beta1 * m + (1-beta1) * grad
#    v = beta2 * v + (1-beta2) * grad^2
#    param = param - lr * m / (sqrt(v) + eps)  # Adam更新
#    param = param - lr * weight_decay * param  # 解耦的权重衰减

实验结果非常明确：在所有被测试的任务上，AdamW一致优于Adam+L2。从Adam切换到AdamW通常能获得0.5-2%的提升，而且不需要额外调参。这就是为什么HuggingFace Transformers、LLaMA、GPT系列等主流模型全部使用AdamW而非Adam。

常见误区："Adam + L2正则化 = AdamW"是我们在实战中见到的最常见的误解。两个看起来"数学上等价"的东西，因为与优化器的耦合方式不同，产生了截然不同的正则化效果。如果你的项目还在用Adam + weight_decay，现在就可以切换到AdamW——只需要改一行优化器名字。

权重衰减与数据集规模的关系

权重衰减的最佳取值与数据量和模型容量高度相关，理解这个关系能帮你快速锁定合理的搜索范围。

数据量大 + 深网络 → 小weight decay。当你有海量数据时，过拟合不是主要矛盾，你更多需要模型容量来记忆复杂模式。此时weight decay设得太大反而会限制模型表达力。典型取值1e-4到1e-6。

数据量小 + 浅网络 → 大weight decay。小数据场景下过拟合风险高，需要更强的正则化约束。典型取值1e-3到1e-2。

大LR自带正则化效果。这是一个容易被人忽略的知识点：学习率越大，参数更新步长越大，权重的天然波动就越大——这种波动本身就有正则化效果。所以当你的LR较大时，weight decay可以相应地设小一点。

一个有效的调参策略是：从weight_decay=0.01开始，先观察loss曲线是否发散——如果训练早期loss不降反升，说明WD太大了，降到0.001甚至0.0001。如果模型稳定但验证集loss偏高（过拟合信号），则增大WD。

现代网络中的权重衰减实践

不同架构对weight decay的敏感度差异极大，这是调参中非常容易出现的问题。

Transformer类模型：极度敏感。Weight decay对Transformer不仅是正则化，还影响训练稳定性。设太大可能使模型完全不收敛（loss不降反升），设太小则过拟合。HuggingFace的默认值是0.01，这是推荐的起点。如果训练不稳定（loss发散），降到0.001甚至0.0001。如果过拟合，升到0.1。

LoRA微调：通常不调。LoRA本身已经是一个正则化很强的微调方法（低秩约束限制了模型容量），因此LoRA的weight decay通常保持在0.01不动。一般先调rank和alpha，最后再动weight decay。

ResNet类CNN：相对不敏感。CNN的参数共享特性和BN的正则化效果使得它对weight decay很鲁棒——从0.0001到0.1都能工作。推荐的起点是0.0001（搭配SGD momentum）。

DeepSeek固定使用0.1。这是一个有意思的数据点。DeepSeek在大模型预训练中固定weight_decay=0.1，无论其他配置如何变化都不动它。这说明在大规模预训练场景下，weight decay可以作为一个固定的"工程常数"而不是需要每次搜索的超参数。

场景 推荐Weight Decay 说明
CV ResNet + SGD 0.0001 ~ 0.005 不敏感，0.0001是标准起点
CV + AdamW 0.01 ~ 0.1 AdamW通常用比SGD更大的WD
LLM预训练 (DeepSeek) 0.1 固定常数，几乎不调
Transformer微调 0.01 ~ 0.1 从0.01开始，不稳定则降低
LoRA微调 0.01 通常不优先调
NLP AdamW (HuggingFace) 0.01 官方默认值
小数据微调 0.001 ~ 0.01 保守开始，防止损伤预训练
核心原则：先过拟合再正则化。先确保模型容量足够在训练集上过拟合，再通过weight decay缩小泛化差距。诊断依据始终是训练loss和验证loss的差值——差值大（训练loss远低于验证loss）说明需要更强的正则化；两边都高说明需要更多模型容量。

总结一下，权重衰减看起来只是一个简单的惩罚项，但与优化器的交互让它变得复杂而微妙。L2正则化在SGD中等价于权重衰减，但在Adam中却不等于——AdamW修正了这个问题，成为了现代训练的标准选择。不同模型架构对weight decay的敏感度差异很大，Transformer最敏感，CNN最鲁棒。调参时从0.01开始，根据loss曲线的稳定性和泛化差距来调节。

下一章我们讲Dropout：隐式集成学习的数学。如果说权重衰减是从参数范数的角度约束模型，那Dropout就是从网络结构的角度做正则化——而且它还有一个更深刻的身份：指数级子网络的集成。
