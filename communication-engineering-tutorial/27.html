📡 第27章 6G关键技术

大家好，我是你们的讲师。上一章我们聊了6G的愿景和标准化路线图——IMT-2030六大场景、3GPP Rel-20/21时间表、1 Tbps峰值速率和0.1 ms时延的"疯狂"KPI。这一章我们要深入技术层面，看看为了实现这些指标，通信工程师们正在攻关哪些核心技术。

6G的关键技术可以用一句话概括：通信不再只是通信。它要感知、要思考、要重构电磁环境、要用太赫兹级别的带宽。每一项技术单独拿出来都足以写一本专著，但我们的目标是在一章之内把它们的核心原理、优势、挑战和产业现状讲清楚，并且告诉你——哪些已经落地了，哪些还在纸面上，哪些是"看起来很美好但做起来很难"的坑。

27.1 ISAC：通感一体化

ISAC（Integrated Sensing and Communication）是6G最具标志性的技术之一，也是5G时代完全没有的概念。它的核心思想是：用同一套波形、同一套硬件、同一个频谱同时完成通信和雷达感知。

为什么需要这个技术？因为6G的应用场景中，通信和感知经常同时出现：无人机物流配送需要跟无人机通信（下发指令、接收视频）的同时检测无人机的位置和速度；自动驾驶汽车需要跟其他车辆通信的同时感知周围环境；智能工厂的机械臂需要接收控制信号的同时检测工件位置。如果通信和感知各自占用独立的频谱和硬件，频谱效率低、设备成本高、电磁兼容性差。ISAC的目标就是"一鸡多吃"。

技术原理：OFDM信号的雷达处理

OFDM（正交频分复用）是4G/5G/6G的基石波形。有趣的是，OFDM信号天然适合雷达感知。原理如下：

  • 测距：发射OFDM信号到目标，目标反射回来后，通过测量发射信号和接收信号之间的时间差（时延），乘以光速再除以2，就得到目标的距离。
  • 测速：目标移动会导致反射信号产生多普勒频移。通过检测OFDM符号之间的相位变化，可以计算目标的速度。
  • 成像：利用MIMO天线阵列的多个通道接收反射信号，通过波束赋形算法可以重构目标的方位角和俯仰角，从而形成三维感知图像。
% OFDM雷达感知测距/测速原理（伪代码）
% 参数配置
N_subcarriers = 1024;       % 子载波数
N_symbols = 128;             % OFDM符号数
delta_f = 30e3;              % 子载波间隔 30 kHz
fs = N_subcarriers * delta_f; % 采样率

% 发射OFDM信号（频域QPSK调制）
tx_data = randi([0 3], N_subcarriers, N_symbols);
tx_signal = qammod(tx_data, 4); % QPSK调制
tx_ofdm = ifft(tx_signal);       % IFFT得到时域OFDM信号

% 接收信号（假设目标距离R，径向速度v）
R = 50;                        % 目标距离 50米
v = 10;                        % 径向速度 10 m/s
delay = 2*R / 3e8;             % 时延（光速传播往返）
doppler = 2*v / (3e8/delta_f); % 多普勒频移

% 对接收信号做OFDM解调
rx_ofdm = rx_signal;           % 接收到的时域信号
rx_data = fft(rx_ofdm);        % FFT回到频域

% 信道估计 → 得到信道响应矩阵 H(f,t)
% 对频域索引做IFFT → 得到距离信息
range_profile = ifft(H, N_subcarriers, 1);
% 对时间索引做FFT → 得到速度信息
doppler_profile = fft(H, N_symbols, 2);

% 输出：距离-速度二维图
[peak_range, peak_doppler] = find(abs(range_doppler_map) == max(abs(range_doppler_map(:))));
estimated_range = peak_range * (3e8 / (N_subcarriers * delta_f));
estimated_velocity = peak_doppler * (3e8 / (2 * fs));

这个伪代码展示了ISAC最核心的数学原理——本质上，通信和感知共享了OFDM物理层的信号处理链。通信接收机在做信道估计时，估计的就是信道的时延-多普勒响应，而这恰好就是雷达感知所需要的"距离-速度二维图"。这就是ISAC能够"一鸡多吃"的根本原因。

ISAC的典型应用

  • 低空经济无人机探测：基站发射通信信号的同时感知周围空域的无人机位置，实现"通信+空域监管"一体化。
  • 智能工厂高精度定位：工厂内的5G/6G基站实时定位AGV小车、机械臂、工件的位置，精度可达厘米级。
  • 自动驾驶环境感知：路侧单元（RSU）通过ISAC波形同时与车辆通信和感知路上的行人、障碍物。

27.2 太赫兹通信（100-300 GHz）

太赫兹通信被认为是6G实现Tbps峰值速率的关键频谱资源。处于100 GHz到300 GHz的Sub-THz频段，能提供数十GHz的连续带宽——这是毫米波（最大带宽约2 GHz）无法比拟的。

优势与挑战

维度 优势 挑战
带宽 单载波带宽可达10-50 GHz，支持Tbps级速率 数模转换器（DAC/ADC）采样率受限，功耗极高
路径损耗 — 自由空间路径损耗随频率平方增长，100 GHz比28 GHz高约11 dB
大气吸收 — 氧气吸收峰在60 GHz/120 GHz附近，水蒸气在183 GHz附近有明显吸收
穿透能力 — 几乎无法穿透墙壁、人体、树叶，基本只能视距（LoS）传播
天线尺寸 高频段对应小波长，可以在小面积内集成大规模天线阵列 相位噪声影响增大，振荡器稳定度要求极高
器件 — 功放效率低（PA效率通常<10%），射频前端成本高昂

中国在太赫兹通信领域处于全球领先地位。紫金山实验室和东南大学团队在2024年实现了基于太赫兹通信的实时Tbps无线传输验证，这是目前世界上公开报道的最高速率通信演示之一。该演示采用250-300 GHz频段，使用超大规模MIMO和轨道角动量复用技术实现了超过1 Tbps的实时数据传输。

踩坑故事：2023年我们实验室搭建了一套300 GHz太赫兹通信原型系统，想做一个"穿墙传输"的演示。结果发现，一张普通A4纸就能把信号衰减掉将近一半——这还只是纸！换成混凝土墙，信号直接从-40 dBm掉到了噪声底限。这次踩坑给了我们一个深刻的教训：太赫兹通信注定是"视距"技术，不要把毫米波的传播经验照搬到太赫兹上。在系统设计中，必须考虑智能中继或RIS反射来解决遮挡问题。

27.3 RIS：可重构智能表面

RIS（Reconfigurable Intelligent Surface，可重构智能表面）是6G的另一个明星技术。它的理念非常巧妙：在墙壁、玻璃、广告牌等表面上部署大量无源/半无源的可调反射单元，通过调节每个单元的相位和幅度，让电磁波按照"期望的方向"反射。

想象一下：你在会议室里开会，基站信号被一堵厚墙挡住了。传统方案是加装一台中继器（有源放大），但中继器要供电、要维护、要配置。RIS的方案是在墙上贴一块"智能贴纸"——不需要供电（或者仅需极低功耗的偏置电压），通过改变表面数千个单元的电磁特性，把信号"绕"过障碍物反射到你的手机位置。这不就是"斗转星移"吗？

RIS vs. 传统中继

对比维度 RIS（智能超表面） 传统中继（Relay）
功耗 极低（mW级，仅需偏置电压） 较高（W级，需PA、RF链、基带处理）
硬件复杂度 低（仅可变电容/变阻二极管+控制芯片） 高（射频收发、基带处理、协议栈）
处理方式 被动反射/透射，不放大信号 主动接收-放大-转发（DF或AF模式）
时延 极低（微秒级，接近光速反射） 中（毫秒级，需解码再编码）
部署灵活性 高（贴附于现有表面，不占空间） 中（需架设独立设备，需供电和回传）
信噪比增益 波束赋形增益（无噪声放大） 放大信号的同时也放大了噪声
成本 低（预期大规模量产后单价$10-100） 较高（中继器$500-2000）

RIS在6G中的目标是通过优化部署，实现边缘覆盖率提升20%以上。尤其在太赫兹通信场景中，RIS几乎是"必需品"——因为太赫兹信号几乎无法穿透任何障碍物，只能靠反射。没有RIS辅助，太赫兹通信的覆盖范围将极其有限。

技术前沿：目前RIS的研究正在从"单比特相位调控"（只能调0°/180°）向"多比特高精度调控"（360°连续可调）发展。相位调控精度直接决定了波束成形的质量。此外，有源RIS（搭载微小功放单元）也在研究中，它能在反射的同时提供一定的信号增益，但功耗和成本会相应上升。这个方向是6G学术界和产业界最活跃的研究领域之一。

27.4 AI原生网络

6G一个根本性的设计理念是——AI不是"加"到网络上的，而是"长"在网络里的。这就是"AI原生网络"（AI-native Network）的含义。

AI for Air Interface

在物理层和MAC层，AI/ML的典型应用包括：

  • CSI压缩反馈：传统5G中，终端需要上报大量信道状态信息（CSI），占用大量上行资源。通过AI自编码器，可以将CSI压缩到极低维度再传输，接收端用Decoder恢复——压缩率可达传统方案的4-8倍。
  • 波束预测：基站可以基于历史波束测量数据，用LSTM或Transformer预测移动终端即将使用的波束方向，从而减少波束扫描开销。这在高频段（毫米波、太赫兹）尤其重要——波束越窄，扫描越耗时。
  • 信道估计增强：传统LS/LMMSE信道估计算法在导频不足时性能急剧下降。基于深度学习的信道估计算法可以利用信道在时延-多普勒域的稀疏性，即使在低导频密度下也能获得接近最优的估计性能。

联邦学习与分布式训练

6G网络的分布式节点（基站、终端）每天产生海量数据，但这些数据往往包含用户隐私信息，不能集中到云端训练。联邦学习（Federated Learning）提供了一种解决方案：各节点在本地训练模型，只上传模型参数而非原始数据，中心服务器聚合这些参数后下发更新。

在6G网络中，联邦学习可以应用于：无线资源管理（各基站本地优化后共享调度策略）、异常检测（各节点检测本地异常流量的模式）、用户体验预测（基于本地历史数据预测用户的移动轨迹和业务需求）。

网络内生AI编排

6G的网络架构中将有一个"AI编排器"（AI Orchestrator）的逻辑实体，它的职责是动态调度网络中的AI计算资源——什么时候在基站侧推理、什么时候卸载到MEC、什么时候用云端大模型、用什么精度的模型来满足时延/功耗/精度之间的权衡。这被称为"端-边-云协同AI推理"。

关键洞察：AI原生网络是6G区别于5G的一个根本特征。5G的网络切片引入的是"业务逻辑的灵活性"，而6G的AI原生引入的是"网络行为的自学习能力"。一个自学习的网络能够根据业务模式、信道条件、用户行为自动调整参数和策略——这才是"智能网络"的真正含义。但这里也有一个风险：AI的可解释性和鲁棒性在通信系统中至关重要。一个通信系统如果因为AI模型的误判而导致用户脱网，后果可能很严重。因此在标准化的过程中，AI模型的验证和测试流程将是讨论的焦点之一。

27.5 全双工通信（SBFD）

全双工通信（SBFD，Sub-band Full Duplex）是另一个即将在6G中落地的技术。在现有的5G中，FDD模式使用不同的频率发送和接收，TDD模式使用不同的时间发送和接收——两者都不能"同时同频"。

SBFD打破了这一限制：基站可以在同一个时间、同一个频带内同时发送和接收。这听起来很简单，但实际做起来非常困难，因为基站发送的信号功率（~46 dBm）远远大于接收到的终端信号（~-90 dBm），差了超过130 dB。如果不做自干扰消除，基站的自发射信号会把接收信号完全"淹没"。

SBFD的核心技术是自干扰消除（Self-Interference Cancellation），包括三个层面：

  • 被动消除：在天线设计上隔离发射和接收路径（如环形器、双极化天线、收发天线物理隔离）
  • 模拟消除：在射频前端用重构技术模拟自干扰信道，从接收信号中减去自干扰
  • 数字消除：在基带数字域用自适应滤波器进一步消除残余的自干扰

当前研究进展已经可以实现超过110 dB的自干扰消除能力——这对于SBFD在6G中应用是一个重要的里程碑。

27.6 关键挑战与"坑"

上面讲的技术听起来都很令人兴奋，但作为通信工程师，我们必须保持清醒——每一项6G关键技术都面临着严峻的工程挑战。下面是我总结的几个"大坑"：

  • RIS相位调控精度：理想情况下RIS单元需要360度连续可调、相位步进<1度。但实际实现的变容二极管和PIN二极管的调控精度有限，且单元之间的互耦效应会导致实际相位响应偏离设计值。很多论文中的"假设完美相位调控"在工程上是不现实的。
  • THz器件成本：目前100 GHz以上的功放、混频器、倍频器大多采用InP（磷化铟）工艺或SiGe BiCMOS工艺，成本远高于CMOS工艺的sub-6 GHz器件。到2030年能否实现"白菜价"量产，是一个很大的问号。
  • ISAC波形设计的矛盾：通信追求高频谱效率（高阶调制），感知追求优异的自相关特性（恒包络波形）。这两种需求在同一个波形上实现"双赢"并不容易——实际工程中往往需要一定程度的性能折中。
  • AI模型的可解释性：通信系统需要确定性保障——你不能说"这个信道估计算法95%的情况下工作得很好"。运营商要求的是99.999%的可靠性，而这需要AI模型在各种边界条件下都能稳定工作。AI推理的"黑盒"特性与通信系统的确定性要求之间存在根本性的矛盾。

踩坑提醒：RIS技术目前有一个很典型的"学术-产业脱节"问题。学术界很多RIS论文假设RIS单元可以独立地、精确地控制每个单元的相位，并且假设信道状态信息是完美已知的。但在实际系统中，获取RIS辅助链路的CSI非常困难，因为RIS本身没有射频收发能力（无源RIS），无法主动发送导频信号。这意味着基站需要依靠终端的反馈来间接估计RIS相关的信道——这在移动场景下几乎不可行。所以，无源RIS的CSI获取问题是当前RIS从论文走向商用最大的瓶颈之一。关注这个方向的研究者可以重点关注"免CSI的RIS配置方案"和"有源RIS的辅助信道估计"两个子方向——它们是最有可能突破工程瓶颈的路径。

6G的关键技术远不止今天讲的这些，但ISAC、太赫兹通信、RIS、AI原生网络和全双工这五项是公认的"主力技术"，每一项都在3GPP Rel-20/21中有对应的研究项目。下一章我们将做一次大收尾——把第1章到第28章的所有知识点串成一幅知识地图，然后设计一个6G低空经济通感融合系统的综合毕业项目，并在其中展示链路预算计算工具的实战用法。
