为什么需要求导法则

大家好，我是你们的讲师。上一章我们用定义法求了几个基本函数的导数——常数函数、幂函数、正弦函数。每一次求导都要算一个复杂的极限，相当费力。想象一下，如果我们要对一个复杂的表达式——比如 f(x) = sin(3x² + 2x) · e^(cos x) / ln(1 + x²)——用定义法求导，那将耗费多少时间？极限表达式可能需要三四行才能写完，其中还有大量令人头疼的代数变形。

但好消息是：我们不需要每次都从定义出发。求导法则就像数学中的"自动化流水线"——你把原材料（函数表达式）放进去，它自动帮你算出结果（导数）。你不必每次都回到"导数是什么"这个哲学问题，只需要娴熟地运用几条规则。

打个比方：你会做一道菜"西红柿炒鸡蛋"。如果把每一步都从最源头做起——种西红柿、养鸡、生火——那太不现实了。你会直接去菜市场买西红柿和鸡蛋，回家打开燃气灶。求导法则就是微积分世界里的"菜市场"和"燃气灶"。

本章我们做两件事：第一，背下基本初等函数的导数公式表（这是你的"乘法口诀表"）；第二，掌握四则运算法则和链式法则（这是你的"运算规则"）。两者结合，你就可以对任意初等函数求导。

初看这张表，你可能会被 15 个公式吓到。但实际上，真正需要独立记忆的核心公式只有三组：幂函数、指数/对数、三角/反三角。而且每一组内部都有规律可循——比如 sin 和 cos 的导数互为相反数关系，arcsin 和 arccos 的导数也互为相反数。

记忆技巧：用"关联记忆法"
对比记忆：sin 和 cos 的导数（一个变另一个，差一个负号），tan 和 cot 的导数（都是平方形式，一正一负），sec 和 csc 的导数（都是自身乘另一个三角函数）。

反三角函数的导数都有平方根或平方分母——因为它们的"祖先"是三角恒等式，本质上是利用隐函数求导得到的。

和差积商的求导法则

有了基本公式，还需要知道如何组合它们。下面四个法则是求导的"交通规则"：

运算 规则 记忆提示
和差 (u ± v)' = u' ± v' 导数可以分开算——最简单的法则
数乘 (Cu)' = C·u' 常数不动，只对函数求导
乘法 (uv)' = u'v + uv' 前导后不导 + 前不导后导
除法 (u/v)' = (u'v - uv') / v² 分母平方，分子是"上导下不导减上不导下导"

最容易记混的是乘法法则和除法法则。有个小窍门：乘法法则是对称的——u 和 v 的地位完全平等，两项相加；除法法则是不对称的——分子是 u'v - uv'，减号前面的 u' 对应的是分子 u 的导数。

例：求 f(x) = x² · e^x 的导数。

令 u = x², v = e^x
        u' = 2x, v' = e^x

        f'(x) = u'v + uv' = (2x)(e^x) + (x²)(e^x) = e^x (x² + 2x)

链式法则："洋葱剥皮法"

如果说只记住一个求导法则，那就是链式法则。它是微积分应用中最常用的工具，没有之一。

链式法则
如果 y = f(u)，u = g(x)，那么复合函数 y = f(g(x)) 的导数为：

dy/dx = (dy/du) · (du/dx) = f'(g(x)) · g'(x)

中文记忆：外层导 × 内层导

链式法则的本质是：复合函数的变化率是外层变化率 × 内层变化率的乘积。我用一个"剥洋葱"的比喻来帮助你理解：

想象一个洋葱，外层是 f，内层是 g。求复合函数的导数就像一层层剥开洋葱——先对外层求导（保持内层不动），再乘以内层的导数。如果有更多层嵌套，就一层层剥下去。

例：求 y = sin(x²) 的导数

        外层 f(u) = sin u   →  f'(u) = cos u
        内层 u = g(x) = x²  →  g'(x) = 2x

        y' = cos(x²) · 2x = 2x · cos(x²)

        ╔══════════════════════════════════════════╗
        ║  步骤拆解：                               ║
        ║  1. 识别外层函数：sin(□)，导数是 cos(□)     ║
        ║  2. 保持 □ 里面的内容不变：cos(x²)          ║
        ║  3. 再乘以 □ 内部对 x 的导数：2x           ║
        ║  4. 结果：cos(x²) · 2x                    ║
        ╚══════════════════════════════════════════╝
链式法则最常见的错误
我改过无数份作业，链式法则的错误是最经典的"送分题变送命题"。最常见的错误就是忘记乘内层导数。

比如求 sin(x²) 的导数，很多人直接写成 cos(x²)——把外层导了，但忘了乘内层 x² 的导数 2x。

另一个常见错误是混淆了幂次的位置：sin² x = (sin x)² 的导数是 2 sin x · cos x，应用了外层 x² 导数为 2x，再乘内层 sin x 的导数 cos x。

链式法则的多层嵌套

复合函数可以有很多层。链式法则的伟大之处在于：不管嵌套多少层，方法都是一样的——一层一层剥开。

例：求 f(x) = e^(sin(x²)) 的导数。

这个函数有三层嵌套：

  • 最外层：e^(□)，导数还是 e^(□)
  • 中间层：sin(□)，导数为 cos(□)
  • 最内层：x²，导数为 2x
f'(x) = e^(sin(x²))  ·  cos(x²)  ·  2x
         ↑                       ↑          ↑
       最外层导数             中间层导数    最内层导数

        检查：一共三层，每层求了一次导，三部分相乘。
        如果你计算出三层，却只有两项相乘，一定是漏了某一步。

这里有一个实用的自查方法：数括号。函数表达式中有几层括号嵌套，链式法则就需要几步相乘。上例中 e^( sin( x² ) ) 有两层括号（不算最外面的括号），加上 x² 本身的指数运算，一共有三层嵌套。

反函数的求导

如果 y = f(x) 有反函数 x = f^(-1)(y)，那么反函数的导数可以通过原函数的导数来表示：

反函数求导公式
如果 f 在点 x 处可导且 f'(x) ≠ 0，那么它的反函数在相应的点 y = f(x) 处也可导，且

(f^(-1))'(y) = 1 / f'(x)

用莱布尼茨记号更直观：

dx/dy = 1 / (dy/dx)

意思是：反函数的导数等于原函数导数的倒数。这在几何上很直观——如果原函数的切线很陡（导数大），那么反函数在对应点的切线就比较平缓（导数小），互为倒数。

这个公式可以用来推导反三角函数的导数。例如，已知 y = arcsin x 是 x = sin y 的反函数：

d(arcsin x)/dx = 1 / (d(sin y)/dy) = 1 / cos y

        因为 x = sin y，所以 cos y = √(1 - sin² y) = √(1 - x²)

        因此 d(arcsin x)/dx = 1 / √(1 - x²)

这就验证了我们之前导数公式表中的结果。

综合求导实战

现在让我们把本章学到的所有工具组合起来，做一道"大综合"题：

综合题：求 f(x) = (x³ · e^(2x)) / sin(x² + 1) 的导数。

步骤分析：

  1. 这是商的形式，先用除法法则
  2. 分子是 x³ · e^(2x)，需要乘法法则 + 链式法则
  3. 分母是 sin(x² + 1)，需要链式法则
令 u = x³ · e^(2x),  v = sin(x² + 1)

        u' = (x³)' · e^(2x) + x³ · (e^(2x))'     ← 乘法法则
            = 3x² · e^(2x) + x³ · e^(2x) · 2     ← 链式法则用于 e^(2x)
            = e^(2x) · (3x² + 2x³)

        v' = cos(x² + 1) · 2x                      ← 链式法则

        f'(x) = (u'v - uv') / v²
               = [e^(2x)(3x² + 2x³) · sin(x² + 1) - x³·e^(2x) · cos(x² + 1) · 2x]
                 / sin²(x² + 1)

看起来很长，但每一小步都是我们学过的基本法则。这就是求导的"流水线"——你不需要创造性的灵感，只需要按部就班地应用规则。

求导实战的两个心法
1. 先化简，后求导：在动手求导之前，先看看表达式能不能化简。比如分式能不能约分？三角函数能不能用恒等式简化？简化后的表达式往往求导更容易。
2. 分步写，不跳步：越复杂的函数，越要分步写清楚。很多同学出错不是因为不会，而是因为跳步导致符号或系数写错了。在每个环节写清楚"这里用了什么法则"，能大幅降低错误率。

本章我们完成了求导工具的全面武装：背下了基本初等函数的导数公式表，掌握了和差积商的四则求导法则，用"洋葱剥皮法"理解了链式法则，并学会了反函数的求导。有了这些工具，任何初等函数的导数对你来说都不在话下。
