第十六章：Cholesky分解与追赶法

大家好，我是你们的讲师。这一章我们聊两个"特化版"的线性方程组求解方法。它们不像上一章的通用 LU 分解那样适用于任意矩阵，但当条件满足时，它们的效率会让人拍案叫绝。

先讲一个让我印象深刻的亲身经历。几年前，我在做一个气象模拟项目。模型的核心是一个扩散方程（diffusion equation），用 Crank-Nicolson 隐式格式离散化之后，每一步都需要求解一个 10⁶ × 10⁶ 的线性系统。我第一次尝试时用了一个通用的稀疏 LU 求解器，结果一步需要大约 5 秒——而我们的模型需要跑 10000 个时间步，那就是接近 14 个小时。项目 deadline 迫在眉睫，这个速度完全不可接受。

后来我意识到，这个离散化后的矩阵不仅是对称正定的（SPD），而且是一个三对角矩阵（tridiagonal）。我换用了 Cholesky 分解的特化版本——追赶法（Thomas algorithm），结果每个时间步从 5 秒降到了 0.003 秒。整个模拟从 14 小时缩短到了 30 秒。项目经理以为我发明了什么新的数学方法，其实我只是用对了工具。

这个故事告诉我们：有时效率提升不在于更好的算法，而在于更匹配的算法。当一个方法利用了矩阵的全部结构信息时，它的效率可以碾压通用方法几个数量级。

16.1 对称正定矩阵

对称正定（Symmetric Positive Definite, SPD）矩阵在工程和科学计算中无处不在。一个矩阵 A 是对称正定的，如果它满足：

  • 对称：A = Aᵀ
  • 正定：对任意非零向量 x，有 xᵀAx > 0

SPD 矩阵出现在哪里？几乎每一个从连续问题离散化来的二次型优化或物理系统中：

领域 矩阵来源 为什么是 SPD
结构力学 刚度矩阵 K（有限元法） 应变能 xᵀKx 总是正的
统计学 协方差矩阵 方差 xᵀΣx 非负且通常正定
机器学习 Gram 矩阵 K_{ij}=φ(x_i)ᵀφ(x_j) 内积的 Gram 矩阵总是半正定
优化 Hessian 矩阵（凸函数） 凸函数的二阶导矩阵正定
偏微分方程 Laplace 算子离散化 能量范数诱导的正定性

判断一个矩阵是否 SPD 有以下几种方法：

  • 特征值法：所有特征值 > 0。但计算特征值本身就挺贵的。
  • Sylvester 准则：所有顺序主子式 > 0。手算小矩阵很实用。
  • Cholesky 尝试法：直接尝试做 Cholesky 分解。如果中途遇到负的或零的平方根，说明矩阵不是 SPD。这其实是实践中最好的方法——因为它基本不额外花钱。

数值实践：在浮点精度下，判断"严格正定"有时很微妙。一个特征值为 10⁻¹⁶ 的矩阵在数学上正定，但在计算机上已是半正定。实际工作中，如果 Cholesky 分解在数值上失败（对角元的平方根接近机器精度），就应当将矩阵视为半正定或病态，并采取正则化措施。

16.2 Cholesky分解（平方根法）

对于 SPD 矩阵 A，存在唯一的下三角矩阵 L（对角元为正数），使得：

A = LLᵀ

这就是 Cholesky 分解（又称平方根法）。它本质上是对称版本的 LU 分解——利用对称性将计算量和存储量都减半。

计算公式：

对 j = 1, 2, ..., n：

  • l_{jj} = √(a_{jj} - Σ_{k=1}^{j-1} l_{jk}²)
  • 对 i = j+1, ..., n：l_{ij} = (a_{ij} - Σ_{k=1}^{j-1} l_{ik} l_{jk}) / l_{jj}

关键特性：

  • 运算量：大约 n³/3 次浮点运算——这是普通 LU 分解（2n³/3）的一半。当 n = 1000 时，Cholesky 比 LU 节省了约 3.3 亿次运算。
  • 无需选主元：对于 SPD 矩阵，Cholesky 分解永远有 l_{jj} > 0（在精确算术中），因此不需要选主元。这是 SPD 矩阵最重要且最实用的特性。
  • 数值稳定性：Cholesky 分解是数值稳定的。误差分析表明，它不会显著放大输入误差。

用 3×3 的例子来演示：

A = [4   2  -2]
    [2   5   1]
    [-2  1   6]

第 1 列：
l₁₁ = √4 = 2
l₂₁ = 2 / 2 = 1
l₃₁ = -2 / 2 = -1

第 2 列：
l₂₂ = √(5 - 1²) = √4 = 2
l₃₂ = (1 - (-1)×1) / 2 = (1 + 1) / 2 = 1

第 3 列：
l₃₃ = √(6 - (-1)² - 1²) = √(6 - 1 - 1) = √4 = 2

结果：
L = [ 2   0   0]
    [ 1   2   0]
    [-1   1   2]

验证：LLᵀ = [4  2 -2; 2  5  1; -2  1  6] ✓

16.3 改进平方根法（LDLᵀ分解）

传统 Cholesky 分解需要计算平方根——这在浮点运算中稍微增加了成本，且当矩阵接近奇异时，l_{jj} 趋近于零，平方根会放大数值误差。

LDLᵀ 分解（改进平方根法）避免了平方根运算：

A = LDLᵀ

其中 L 是单位下三角矩阵（对角元全为 1），D = diag(d₁, d₂, ..., dₙ) 是对角矩阵（dⱼ > 0）。

算法步骤：

import numpy as np

def ldl_decomposition(A):
    """
    计算对称正定矩阵 A 的 LDLᵀ 分解（改进平方根法）。
    返回 L（单位下三角）和 d（对角元向量）。
    """
    n = A.shape[0]
    L = np.eye(n)
    d = np.zeros(n)

    for j in range(n):
        # 计算 d_j
        v = d[:j] * L[j, :j]  # 逐元素乘法
        d[j] = A[j, j] - np.dot(L[j, :j], v)

        # 计算 L[i, j] for i > j
        for i in range(j + 1, n):
            v = d[:j] * L[i, :j]
            L[i, j] = (A[i, j] - np.dot(L[i, :j], v)) / d[j]

    return L, d

# ---- 测试 ----
A = np.array([[4, 2, -2],
              [2, 5,  1],
              [-2, 1, 6]], dtype=float)

L, d = ldl_decomposition(A)
print("L 矩阵：")
print(np.round(L, 4))
print("d 对角元：", np.round(d, 4))

# 验证 A = LDLᵀ
reconstructed = L @ np.diag(d) @ L.T
print("\n重构误差 ||A - LDLᵀ||：",
      np.linalg.norm(A - reconstructed))

# 求解 Ax = b
b = np.array([8, 11, 5], dtype=float)

# 前向替换 Ly = b
y = np.linalg.solve(L, b)
# 缩放 Dz = y → z = y ./ d
z = y / d
# 回代 Lᵀx = z
x = np.linalg.solve(L.T, z)
print("解 x：", np.round(x, 6))
print("残差 ||Ax - b||：", np.linalg.norm(A @ x - b))

Cholesky vs LDLᵀ 对比：Cholesky 需要 n 次平方根运算，LDLᵀ 完全不需要。对于接近奇异的 SPD 矩阵，LDLᵀ 的数值稳定性更好，因为 dⱼ 可以变得非常小但不会出现负的平方根。此外，LDLᵀ 的 L 是对角元为 1 的单位下三角矩阵，在某些迭代精化（iterative refinement）场景中更方便。

16.4 追赶法（Thomas算法）

现在来看一个极端的特例——三对角矩阵。它只在主对角线和两条次对角线上有非零元素：

A = [b₁  c₁   0  ...  0  ]
    [a₂  b₂  c₂  ...  0  ]
    [0   a₃  b₃  ...  0  ]
    [⋮    ⋮   ⋮   ⋱   c_{n-1}]
    [0    0   0  aₙ   bₙ ]

Thomas 算法本质上是针对三对角矩阵的高斯消去法——它利用三对角结构的特殊性，将运算量从 O(n³) 降到了 O(n)。

算法过程：

前向 sweep（消去次对角线）：

  • 对 i = 2, ..., n：
  • m = aᵢ / b_{i-1}
  • bᵢ' = bᵢ - m · c_{i-1}
  • dᵢ' = dᵢ - m · d_{i-1}

反向 sweep（回代）：

  • xₙ = dₙ' / bₙ'
  • 对 i = n-1, ..., 1：xᵢ = (dᵢ' - cᵢ · x_{i+1}) / bᵢ'

效率解释：为什么 Thomas 算法是 O(n) 而不是 O(n³)？因为三对角矩阵的 LU 分解中，L 只有主对角线和下次对角线有非零元，U 只有主对角线和上次对角线有非零元。整个分解只有 O(n) 个元素需要计算，而不是稠密矩阵的 O(n²) 个。

Python 实现：

import numpy as np

def thomas_solve(a, b, c, d):
    """
    求解三对角线性系统 Ax = d。
    a: 下次对角线 (长度 n-1, a[0] 对应第 2 行)
    b: 主对角线 (长度 n)
    c: 上次对角线 (长度 n-1, c[-1] 对应第 n-1 行)
    d: 右端项 (长度 n)
    """
    n = len(b)
    # 复制数组以避免修改输入
    bp = b.copy()
    dp = d.copy()
    x = np.zeros(n)

    # 前向 sweep
    for i in range(1, n):
        m = a[i-1] / bp[i-1]
        bp[i] = bp[i] - m * c[i-1]
        dp[i] = dp[i] - m * dp[i-1]

    # 反向 sweep
    x[-1] = dp[-1] / bp[-1]
    for i in range(n-2, -1, -1):
        x[i] = (dp[i] - c[i] * x[i+1]) / bp[i]

    return x

# ---- 测试：100 万阶三对角系统 ----
n = 1_000_000

# 构造一个典型的三对角矩阵（来自 PDE 离散化）
# A = tridiag(-1, 2, -1)，这是一个 SPD 矩阵
a = -np.ones(n-1)   # 下次对角线
b = 2 * np.ones(n)   # 主对角线
c = -np.ones(n-1)   # 上次对角线

# 右端项：sin 函数
x_true = np.sin(np.linspace(0, np.pi, n))
d = np.zeros(n)
d[0] = b[0] * x_true[0] + c[0] * x_true[1]
for i in range(1, n-1):
    d[i] = a[i-1] * x_true[i-1] + b[i] * x_true[i] + c[i] * x_true[i+1]
d[-1] = a[-1] * x_true[-2] + b[-1] * x_true[-1]

import time
start = time.time()
x = thomas_solve(a, b, c, d)
elapsed = time.time() - start

print(f"n = {n:,}")
print(f"求解时间: {elapsed:.4f} 秒")
print(f"最大误差: {np.max(np.abs(x - x_true)):.2e}")
print(f"每秒处理的自由度: {n / elapsed:.0f}")

踩坑提醒：当年我第一次用 Thomas 算法时，偷懒没有处理 a[0] 和 c[-1] 这两个"悬空"的数组边界。Python 的负索引让 a[-1] 悄悄地取到了 a 的最后一个元素，c[0] 也取到了 c 的第一个元素——代码跑了，但结果是错的！花了整整半天调试才发现问题。所以三对角数组索引千万要小心：a 是下次对角线，长度 n-1，a[i] 对应矩阵的第 i+2 行、第 i+1 列。建议总是画一个 3×3 的例子对照着写代码。

16.5 追赶法的典型应用

三对角系统广泛出现在以下场景中：

应用场景 为什么出现三对角 矩阵规模
三次样条插值 连续性条件导出的线性系统 n 个插值点 → n-1 个区间
一维热方程（隐式 Euler） 空间二阶导数的中心差分 空间网格点数
一维对流-扩散方程 Crank-Nicolson 格式 空间网格点数
时间序列的 AR(2) 模型 Yule-Walker 方程 滞后阶数
期权定价（隐式二叉树） Black-Scholes 方程离散化 价格网格点数

结构即效率：从通用的 LU 分解（O(n³)）到 Cholesky 分解（O(n³/3)）再到 Thomas 算法（O(n)），我们看到一个清晰的模式——矩阵的结构信息越多，算法的效率就越高。三对角结构的 LU 分解只有两个非零的对角线需要计算，因此变成了 O(n)。在数值线性代数中，"认识你的矩阵结构"是选择算法的最重要前提。

回想开头的气象模拟故事：从 14 小时到 30 秒，靠的不是什么魔法——只是正确地识别了矩阵的三对角 SPD 结构，然后选择了 Thomas 算法。那个 10⁶ × 10⁶ 的三对角系统，在 Thomas 算法下只需要约 3 千万次浮点运算，现代 CPU 可以在毫秒级完成。
