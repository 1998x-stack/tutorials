第28章 综合项目：FPGA图像采集与处理系统

欢迎来到本教程的最后一章。前 27 章我们学习了 Verilog 语法、数字电路设计、仿真验证、总线协议、时序分析、HLS、AI 辅助开发——现在是把这一切串起来的时候了。本章我们要亲手打造一个完整的嵌入式视觉系统：通过摄像头采集实时视频，在 FPGA 中进行图像处理，然后显示在显示器上。

这不是一个"玩具项目"。这个系统中融合了通信接口（DVP/SCCB）、存储子系统（DDR3）、数据流架构（AXI4-Stream）、图像算法管道（定点卷积）、时序控制（VGA/HDMI）、跨时钟域同步——覆盖了本书全部核心知识点。

28.1 项目概述

系统的数据流是这样走的：

OV5640 摄像头 → DVP 8-bit 并行接口 → 帧缓存写入 DDR3（MIG IP 核）→ 图像处理流水线（灰度转换 + Sobel 边缘检测）→ 帧缓存读出 → VGA/HDMI 显示输出

对应的硬件平台推荐：Xilinx Artix-7 (XC7A35T) 或 Zynq-7010 开发板（带 HDMI 输出 + 至少 256 MB DDR3 + OV5640 摄像头接口）。如果使用其他平台，需要相应调整 MIG 配置和显示时序参数。

系统性能目标（640x480@60fps）：像素时钟 25.175 MHz，每帧 307,200 像素，每像素 16 位（RGB565），每秒 18.4 MB 数据吞吐。Sobel 处理流水线要求在 1 个像素时钟周期内完成 3×3 卷积计算。DDR3 读写带宽要求（读 + 写同时进行）至少 36.8 MB/s——对于 533 MHz DDR3 的 64-bit 接口来说，绰绰有余。

28.2 系统架构设计

顶层划分为四个功能模块，通过 AXI4-Stream 互联：

28.3 摄像头采集模块

OV5640 支持 DVP 8-bit 并行接口和 SCCB（类似 I2C）配置总线。上电后，我们需要通过 SCCB 写寄存器来配置输出格式为 RGB565、分辨率 640x480、帧率 60 fps。

寄存器地址值功能

0x30350x21PLL 分频配置
0x30360x46PLL 倍频配置
0x38200x40垂直翻转关闭
0x38210x00水平镜像关闭
0x43000x61输出格式 — RGB565
0x501F0x01ISP 格式选择 — RGB

采集模块的核心是 DVP 时序解码：VSYNC 上升沿表示一帧开始，HSYNC 高电平期间每个 PCLK 上升沿传输一个字节数据（两个 PCLK = 一个 RGB565 像素）。我们需要实例化一个行缓存（Line Buffer），暂存一行像素数据，组装成 AXI4-Stream 包发送。

28.4 图像处理流水线

图像处理分为三步：RGB565 → 灰度图 → Sobel 边缘检测 → 二值化输出。

RGB → 灰度：使用定点加权平均 Gray = (77*R + 150*G + 29*B) >> 8，转换为 8-bit 灰度值。权重来自 ITU-R BT.601 标准，用整数定点实现避免浮点运算。

Sobel 边缘检测：3×3 卷积窗口通过两个 Line Buffer + 一级 Shift Register 生成。Sobel 算子分为水平方向 Gx 和垂直方向 Gy，梯度幅值近似为 |Gx| + |Gy|（绝对值之和，避免开方运算）。

// Sobel 边缘检测模块 — 核心卷积逻辑
module sobel_edge_detect #(
    parameter DATA_WIDTH = 8
)(
    input  wire clk,
    input  wire rst_n,
    input  wire [DATA_WIDTH-1:0] pixel_in,    // 灰度像素输入
    input  wire pixel_valid,
    output reg  pixel_ready,
    output reg  edge_out,                      // 二值化边缘输出
    output reg  edge_valid,
    input  wire edge_ready
);

    // 行缓存 — 3 行数据同时可用
    // line_buf[0] = 当前行, line_buf[1] = 上一行, line_buf[2] = 上上行
    reg [DATA_WIDTH-1:0] line_buf [0:2][0:639];
    reg [DATA_WIDTH-1:0] p [0:2][0:2];         // 3x3 窗口

    integer row, col;
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            for (row = 0; row < 3; row++)
                for (col = 0; col < 3; col++)
                    p[row][col] <= 0;
        end else if (pixel_valid && pixel_ready) begin
            // 移位窗口
            p[0][0] <= pixel_in;                // 新像素进入窗口左上
            p[0][1] <= p[0][0];
            p[0][2] <= p[0][1];
            p[1][0] <= line_buf[1][0];          // 上一行对应位置
            p[1][1] <= p[1][0];
            p[1][2] <= p[1][1];
            p[2][0] <= line_buf[2][0];          // 上上行对应位置
            p[2][1] <= p[2][0];
            p[2][2] <= p[2][1];
        end
    end

    // Sobel 算子 — 梯度计算（组合逻辑）
    // Gx = (p[0][2]+2*p[1][2]+p[2][2]) - (p[0][0]+2*p[1][0]+p[2][0])
    // Gy = (p[2][0]+2*p[2][1]+p[2][2]) - (p[0][0]+2*p[0][1]+p[0][2])
    wire signed [10:0] Gx, Gy;
    assign Gx = (p[0][2] + {p[1][2], 1'b0} + p[2][2])
              - (p[0][0] + {p[1][0], 1'b0} + p[2][0]);
    assign Gy = (p[2][0] + {p[2][1], 1'b0} + p[2][2])
              - (p[0][0] + {p[0][1], 1'b0} + p[0][2]);

    // 梯度幅值近似：|Gx| + |Gy|
    wire [10:0] grad_mag;
    assign grad_mag = (Gx[10] ? (~Gx + 1'b1) : Gx)
                    + (Gy[10] ? (~Gy + 1'b1) : Gy);

    // 阈值二值化 — 输出 0/1 边缘标志
    localparam THRESHOLD = 11'd80;
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            edge_out <= 0;
        end else begin
            edge_out <= (grad_mag > THRESHOLD) ? 1'b1 : 1'b0;
        end
    end

endmodule

28.5 DDR3 帧缓存与乒乓缓冲

DDR3 帧缓存通过 MIG IP 核实现。配置要点：时钟频率 200 MHz（DDR3 等效 533 MHz）、数据宽度 32 位（4 bytes/cycle）、突发长度 8。我们使用 AXI4 接口与 MIG 连接。

乒乓缓冲策略：在 DDR3 中划分两个帧缓冲区（Frame Buffer A 和 B）。写入模块和读取模块各自指向不同的缓冲区——写模块写入 A 时，读模块从 B 读取；完成一帧后交换指针。这样避免了"写一半读"造成的画面撕裂（tearing）。

实现细节：指针交换由一个简单的标志寄存器控制。当 camera_capture 模块检测到 VSYNC 上升沿（一帧结束），写入当前帧计数加一，读指针自动指向刚刚写入完毕的帧。这个寄存器用两级同步器跨到像素时钟域和数据时钟域，确保指针切换安全。

28.6 VGA 显示时序

以 640x480@60Hz 为例，像素时钟为 25.175 MHz。VGA 时序分为行同步和场同步：

// VGA 时序发生器 — 640x480@60Hz
module vga_timing_gen #(
    parameter  H_VISIBLE  = 640,
    parameter  H_FP       = 16,    // Front Porch
    parameter  H_SYNC     = 96,    // Sync Pulse
    parameter  H_BP       = 48,    // Back Porch
    parameter  V_VISIBLE  = 480,
    parameter  V_FP       = 10,
    parameter  V_SYNC     = 2,
    parameter  V_BP       = 33
)(
    input  wire clk_25m,           // 25.175 MHz 像素时钟
    input  wire rst_n,
    output reg  hsync,
    output reg  vsync,
    output reg  de,                // Data Enable
    output reg  [9:0] hcount,      // 当前列 (0 ~ H_TOTAL-1)
    output reg  [9:0] vcount       // 当前行 (0 ~ V_TOTAL-1)
);

    localparam H_TOTAL = H_VISIBLE + H_FP + H_SYNC + H_BP; // 800
    localparam V_TOTAL = V_VISIBLE + V_FP + V_SYNC + V_BP; // 525

    // 行/场计数器
    always @(posedge clk_25m or negedge rst_n) begin
        if (!rst_n) begin
            hcount <= 0;
            vcount <= 0;
        end else begin
            if (hcount == H_TOTAL - 1) begin
                hcount <= 0;
                vcount <= (vcount == V_TOTAL - 1) ? 0 : vcount + 1;
            end else begin
                hcount <= hcount + 1;
            end
        end
    end

    // HSYNC — 在同步脉冲区间拉低
    always @(posedge clk_25m or negedge rst_n) begin
        if (!rst_n) hsync <= 1;
        else hsync <= ~((hcount >= H_VISIBLE + H_FP) &&
                        (hcount < H_VISIBLE + H_FP + H_SYNC));
    end

    // VSYNC — 在同步脉冲区间拉低
    always @(posedge clk_25m or negedge rst_n) begin
        if (!rst_n) vsync <= 1;
        else vsync <= ~((vcount >= V_VISIBLE + V_FP) &&
                        (vcount < V_VISIBLE + V_FP + V_SYNC));
    end

    // DE — 在有效显示区间为高
    always @(posedge clk_25m or negedge rst_n) begin
        if (!rst_n) de <= 0;
        else de <= (hcount < H_VISIBLE) && (vcount < V_VISIBLE);
    end

endmodule
关于 HDMI 替代方案：如果开发板有 HDMI 输出而不是 VGA，可以使用 TMDS 编码器（如 Sil 1136 或 ADV7511）将 RGB + HSYNC/VSYNC/DE 转换为 HDMI 差分信号。RGB 数据通道只需将上述 VGA 模块输出的 12-bit RGB（4:4:4）送入 TMDS 编码器即可。

28.7 集成与调试

在 Vivado Block Design 中，将 camera_capture、image_processor、vga_display 作为 RTL 模块添加，MIG IP 核通过 Block Design 的 AXI SmartConnect 连接到 frame_buffer 模块的 AXI Full Master 接口。

调试分三个阶段进行：

第一阶段：ILA 抓取 camera_capture 模块输出端的 AXI4-Stream 信号。验证 HSYNC/VSYNC 时序正确、每个帧的像素计数等于 640×480。如果摄像头配置失败，检查 SCCB（I2C）初始化时序——这是最常见的失败点。

第二阶段：ILA 抓取 DDR3 写入前后的数据。写入前（原始摄像头数据）和写入后（从 DDR3 读回）做一个像素级对比——确保 MIG 读写正确。常见问题是 AXI 地址对齐错误导致的数据错位。

第三阶段：ILA 抓取 Sobel 模块输出端的边缘二值化结果。输入一组已知模式的测试图像（棋盘格、条纹），观察边缘是否正确。调整 THRESHOLD 参数以适应不同光照条件。

踩坑故事——跨时钟域导致的撕裂帧：第一个版本中，我将摄像头 PCLK 域的输出直接连到了 MIG 的写接口（运行在 200 MHz 的 clk_200 域），中间没有加异步 FIFO。结果第一帧显示出来是撕裂的——上面三分之二是正确的画面，下面三分之一是上一帧的残留数据。原因很简单：写指针跨时钟域没有同步，DDR3 写控制器在摄像头域和 MIG 域之间按错误的地址持续写入。解决方案是在 camera_capture 和 MIG 写接口之间插入一个 AXI4-Stream 异步 FIFO，深度至少为一行像素的 2 倍（1280 个 16-bit 像素）。教训：只要接口跨越不同时钟源——哪怕看起来频率接近——也一定要用异步 FIFO 或双时钟域握手电路。

28.8 结语：一段学习之旅的终点，也是工程之路的起点

二十八章。从 1985 年 XC2064 的 64 个逻辑单元，到 2026 年用 AI 辅助生成 AXI4-Stream FIFO，再到今天 OV5640 + DDR3 + Sobel 边缘检测的完整系统——你走过了一条相当完整的 FPGA 工程师成长之路。

回顾一下我们学会的东西：Verilog 的可综合语法和仿真验证（第 3-8 章）、UART/SPI/I2C 通信协议在 FPGA 上的实现细节（第 9-10 章）、跨时钟域处理和时序收敛（第 13-14 章）、Vivado 和 Quartus 的工程实践（第 15-16 章）、AXI4 全线协议的深度理解（第 19 章）、软核和 RISC-V 硬核的集成与定制（第 20-21 章）、硬件调试方法论的体系化掌握（第 23 章）、开源工具链的独立使用能力（第 24 章）、HLS 的合理使用和 AI 辅助开发的协作模式（第 26-27 章）。

这些知识就像是工具箱里的不同螺丝刀——你不需要每次都用全部，但当特定的问题出现时，你知道该用哪一把。

下一步的建议：如果你希望在 FPGA 这条路上继续深入，我推荐三个阶段性的方向——
短期（1-3 个月）：把你的开发板上的外设全驱动一遍。摄像头、HDMI、以太网、SD 卡——每驱动一个新外设，都在加深你对"真实硬件是如何工作的"的理解。
中期（3-12 个月）：参与一个开源 FPGA 项目。OpenCPI、LiteX、FuseSoC 都是很好的起点。在开源项目里，你能看到真正的工程代码是如何组织的，也能接触到多人协作的代码评审流程。
长期（1-3 年）：选择一个细分方向深度耕耘——要么走向系统架构（SoC 设计、NoC、异构计算），要么走向算法加速（AI 推理引擎、视频编解码器、5G LDPC 编码），要么走向 EDA 工具链（Yosys 插件开发、布局布线算法优化）。

最后，记住 1.7 节的那个故事——仿真不是万能的，硬件调试需要一整套方法论。保持怀疑、保持实验、保持对"为什么这个波形长这样"的好奇心。一个成功的 FPGA 工程师，不是"什么都会写"的人，而是"什么都能 debug"的人。

感谢你陪伴我走完这 28 章。现在——关掉教程，打开 Vivado，开始搭建你自己的系统吧。
