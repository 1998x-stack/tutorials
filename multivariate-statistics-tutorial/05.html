数据不会自动变干净——预处理是真正的"脏活"

大家好，欢迎继续学习。如果你去问一位资深的统计分析师："多元分析中最花费时间的是哪个环节？"答案几乎不会是"建模型"或"跑算法"，而是数据预处理 (data preprocessing)。业界有个广为流传的说法：一个数据科学项目 80% 的时间花在数据清洗和准备上，只有 20% 花在实际分析和建模上。为什么？因为真实世界的数据从来不会乖乖地以整齐的矩阵形式躺在 CSV 文件中等你分析——它可能有缺失值、异常值、不统一的量纲、偏态分布、编码错误……

本章我们将学习：如何处理缺失值和异常值，如何标准化和变换数据使其符合分析要求，以及如何用可视化方法"探索"高维数据结构。这些技能虽然不是"高大上"的数学方法，但它们是区分"能用"和"好用"的分析师的关键——而跳过它们，也是绝大多数分析翻车的根源。

数据质量与缺失值处理

先来理解缺失值的"机制"。统计学家将缺失值分为三种类型：

类型名称含义例子
MCAR完全随机缺失缺失的概率与任何变量无关实验室仪器随机故障导致部分数据丢失
MAR随机缺失缺失的概率依赖于其他观测到的变量高收入人群更倾向于不报告收入；但"是否缺失"可以部分被职业、学历预测
MNAR非随机缺失缺失的概率依赖于缺失值本身的真实值病情严重的患者更可能漏掉复诊；你无法从已有数据推断缺失的原因

理解缺失机制很重要，因为它决定了你应该用什么方法处理：

  • 删除法 (deletion)：最简单直接。包括列表删除 (listwise deletion)——只要某个变量缺失就删掉整行；和成对删除 (pairwise deletion)——只在需要用到的变量上删除缺失。列表删除在 MCAR 下是无偏的，但会损失样本量。如果缺失率超过 5%，列表删除就可能浪费大量数据。
  • 均值/中位数插补 (mean/median imputation)：用该变量的均值或中位数填充缺失值。简单但会低估方差、扭曲变量间的相关性。只推荐在缺失率很低（< 5%）时使用。
  • 回归插补 (regression imputation)：用其他变量预测缺失值。比均值插补更精准，但会人为增强变量间的相关性。
  • 多重插补 (multiple imputation)：生成多组插补数据，分别分析后再合并结果。这是目前最推荐的方法，能合理反映插补的不确定性。R 的 mice 包和 Python 的 IterativeImputer (sklearn) 都实现了多重插补。
import pandas as pd
import numpy as np
from sklearn.experimental import enable_iterative_imputer
from sklearn.impute import IterativeImputer

# 模拟一个有缺失值的数据集
np.random.seed(42)
n, p = 100, 5
data = np.random.randn(n, p)
# 随机引入 10% 的缺失值
mask = np.random.random(data.shape) < 0.1
data[mask] = np.nan

# 多重插补
imputer = IterativeImputer(max_iter=10, random_state=42)
data_imputed = imputer.fit_transform(data)

# 比较插补前后的均值和协方差
print("原始（删除缺失后）均值:", np.nanmean(data, axis=0))
print("插补后均值:", np.mean(data_imputed, axis=0))
注意：处理缺失值没有"万能药"。如果你的数据是 MCAR，列表删除简单有效。如果是 MAR，多重插补是推荐方法。如果是 MNAR，你需要专门的敏感性分析——这不是一个简单的技术问题，而是需要结合领域知识来判断。一个常见的错误是：不管缺失机制，一律用均值插补——这样得到的"干净"数据集其实比原始数据更误导人。

异常值检测

异常值 (outliers) 是那些与大多数观测值"格格不入"的数据点。在多元分析中，一个点在单个变量上可能看起来正常，但在多变量联合空间中可能是异常的——这就是多元异常值检测比一元检测更复杂的原因。

一元异常值检测

  • 箱线图 (boxplot)：超过 Q1-1.5×IQR 或 Q3+1.5×IQR 的点被视为异常值。简单直观。
  • Z-score：|z| > 3 的观测值通常被视为异常值（正态分布下，超出 3σ 的概率约为 0.3%）。

多元异常值检测

  • 马氏距离 (Mahalanobis distance)：这是多元异常值检测的标配工具。D² = (x - x̄)ᵀS⁻¹(x - x̄) 服从近似的 χ²(p) 分布。结合 χ² 分布的分位数（如 97.5% 分位数）来判断。
  • 杠杆值 (leverage)：在线性回归中，杠杆值衡量每个样本对回归估计的影响程度。高杠杆点可能不是异常值（y 方向），但对模型参数影响很大。
from scipy.spatial.distance import mahalanobis
from scipy.stats import chi2

# 计算每个样本的马氏距离
def mahalanobis_distance(X):
    cov = np.cov(X, rowvar=False)
    inv_cov = np.linalg.inv(cov)
    mean = np.mean(X, axis=0)
    dist = [mahalanobis(x, mean, inv_cov) for x in X]
    return np.array(dist)

dist = mahalanobis_distance(data_imputed)
p_value = 1 - chi2.cdf(dist**2, df=data_imputed.shape[1])
outliers = np.where(p_value < 0.025)[0]
print(f"检测到 {len(outliers)} 个多元异常值")

数据标准化与变换

许多多元分析方法（PCA、聚类、判别分析）对变量的尺度极其敏感。一个以"万元"为单位的变量和一个以"元"为单位的变量，前者的方差天然是后者的 10⁸ 倍——如果不对数据进行标准化，计算结果会被量大纲的变量主导，结果毫无意义。

Z-score 标准化

z = (x - μ) / σ

将每个变量变换为均值为0、标准差为1的分布。这是最常用的标准化方法。

Min-Max 缩放

x' = (x - x_min) / (x_max - x_min)

将数据映射到 [0, 1] 区间。对异常值敏感，因为最值可能被异常值拉偏。

Box-Cox 变换

当数据呈偏态分布时，Box-Cox 变换可以使其近似正态：

x(λ) = { (x^λ - 1)/λ,  λ ≠ 0
        { ln(x),          λ = 0

λ 通过极大似然估计得到。注意 x 必须为正数。

散点图矩阵

散点图矩阵 (scatter plot matrix) 是最基础也最实用的多元可视化工具。它将 p 个变量两两之间的散点图排列成一个 p × p 的网格。每个单元格显示两个变量的关系，对角线通常放变量名或单变量分布图。

通过散点图矩阵，你可以快速获取以下信息：

  • 哪些变量之间呈线性关系？哪些是非线性关系？
  • 是否存在明显的异常值？
  • 不同分组的样本在散点图上是否可分离？
  • 是否存在严重的多重共线性？
import seaborn as sns
import matplotlib.pyplot as plt
from sklearn.datasets import load_iris

iris = load_iris()
df = pd.DataFrame(iris.data, columns=iris.feature_names)
df['species'] = iris.target_names[iris.target]

# 散点图矩阵（按品种着色）
sns.pairplot(df, hue='species', diag_kind='kde')
plt.tight_layout()
plt.show()
# 可清晰看到：Setosa 品种的花瓣长宽与其他两个品种明显分离

散点图矩阵的局限性也很明显：当变量数 p 很大时（比如 p > 10），网格会大到无法辨认——每个单元格太小了。这时候就需要其他可视化方法。

平行坐标图

平行坐标图 (parallel coordinates plot) 是专门为高维数据设计的可视化工具。它将每个变量表示为一条垂直的平行轴，每条观测数据对应一条连接各轴上的值的折线。通过观察折线的模式，可以发现数据的分组结构、变量间的相关性以及异常值。

例如，用鸢尾花数据画平行坐标图，你会看到 Setosa 品种在所有轴上都有相对较低的值（特别是花瓣长宽），而 Virginica 品种值较高——三条"彩带"在视觉上分离得相当好。

实用技巧：在制作平行坐标图时，务必将各变量标准化到同一尺度（如 [0,1] 或 Z-score），否则量级大的变量会主导视觉模式。同时，对折线进行透明度 (alpha) 调整，可以避免"过度绘制"——当样本量很大时，所有折线叠在一起就像一团乱麻，降低透明度可以揭示密度模式。Python 的 pandas.plotting.parallel_coordinates 可以直接生成平行坐标图。

相关性热图

相关性热图 (correlation heatmap) 是将相关矩阵以颜色矩阵的形式可视化。每个单元格的颜色深浅表示相关性的强弱，通常用红蓝渐变（红色表示正相关、蓝色表示负相关）。结合层次聚类 (hierarchical clustering) 对变量重新排序，可以揭示变量间的"块状"结构——哪些变量倾向于聚在一起形成一组。

import seaborn as sns

# 计算相关矩阵并画热图
corr_matrix = df.iloc[:, :4].corr()
sns.clustermap(corr_matrix, annot=True, cmap='RdBu_r',
               vmin=-1, vmax=1, figsize=(8, 6))
# annot=True 显示相关系数值
# clustermap 自动对行列进行层次聚类重排

相关性热图结合层次聚类的最大价值在于：它不需要任何数学建模就能揭示数据的"内在结构"。比如在金融数据分析中，不同行业股票的收益率相关矩阵经过聚类重排后，通常会呈现出清晰的行业板块结构——同行业的股票相关性高，不同行业间相关性低。这与我们直觉一致，但可视化能让你"一眼看到"这个结构。

踩坑故事：Normalization before PCA matters

这个故事值得反复讲——因为我在不同场合已经听过至少五个人讲述过几乎一模一样的教训。

一位数据科学家用 PCA 分析客户消费数据，包含 20 个变量：年消费金额（几千到几十万）、登录次数（几十到几百）、浏览时长（几分钟到几小时）、以及各种消费类别的比例（0 到 1 之间的小数）。她直接对原始数据做了 PCA，画出前两个主成分，发现第一主成分几乎完全由"年消费金额"这一个变量驱动，解释了数据总变异的 95%。她兴奋地告诉团队："客户的消费模式差不多就是金额这一个维度决定的！"

团队的另一位同事看了之后问了一句："你标准化了吗？"空气突然安静了。重新标准化后做 PCA，前两个主成分只解释了 45% 的变异，而且多个变量都有贡献——消费模式原来是非常多元化的，远不止"金额"一个维度。之前的"发现"完全是假象——只是年消费金额的量纲（数值范围几千到几十万）天然主导了协方差矩阵的特征值计算。

教训：做 PCA 前一定要标准化。除非你确信所有变量在同一量纲下（比如都是同一量表的得分、都是百分比等），并且你希望通过 PCA 保留变量的原始方差结构——但这种情况极为罕见。实操中我有一条铁律：做 PCA，先标准化，再开跑。这个习惯帮我避免过无数次类似的"假阳性发现"。

本章小结与衔接

数据预处理和可视化是多元分析中"看不见的功夫"——你不能在论文的结果部分写"我们做了数据预处理"，但跳过它等于在沙滩上建城堡。这一章我们讨论了：缺失值的三种机制及其处理方法（从列表删除到多重插补）、异常值检测的一元和多元方法（特别是马氏距离）、数据标准化和变换（Z-score 标准化、Box-Cox 变换）、以及散点图矩阵、平行坐标图和相关性热图三种可视化工具。最后通过 PCA 前忘了标准化的惨痛教训，再次强调了预处理的重要性。

现在你的数据已经干净、标准化、经过可视化探索了。下一步，我们将进入一个看似简单但非常关键的问题：如何度量两个样本（或两个变量）之间的"距离"或"相似度"？欧氏距离、马氏距离、余弦相似度、Jaccard 系数——它们在聚类、分类、降维等后续方法中是"距离度量"这个核心模块的不同选择，而选错了距离度量，再好的算法也跑不出有意义的结果。
