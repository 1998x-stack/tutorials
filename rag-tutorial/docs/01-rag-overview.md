# RAG 概述与演进路线

## 1. 什么是 RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将**外部知识检索**与**大语言模型生成**相结合的技术架构。它的核心思想是：在 LLM 生成答案之前，先从外部知识库中检索相关文档片段，将检索结果作为上下文注入提示词，让 LLM 基于这些"参考资料"生成答案。

```
传统 LLM：  用户提问 → LLM 直接生成 → 答案（可能幻觉）
RAG：       用户提问 → 检索知识库 → 获取相关文档 → LLM 基于文档生成 → 答案（有据可查）
```

## 2. 为什么需要 RAG

大语言模型存在三个根本性局限：

| 痛点 | 描述 | RAG 如何解决 |
|------|------|-------------|
| **知识截断** | LLM 训练数据有截止日期，无法回答新问题 | 外部知识库可实时更新 |
| **幻觉问题** | LLM 可能编造事实，产生虚假信息 | 答案受限于检索到的真实文档 |
| **黑箱不可解释** | 用户无法验证答案来源 | RAG 可提供内联引用和来源追溯 |

## 3. RAG 的发展三阶段

### 3.1 Naive RAG（2023 年初）
最原始的形式：Query → Retrieve → Generate。简单直接，但存在检索质量差、上下文噪声多、缺乏质量控制等问题。

### 3.2 Advanced RAG（2023-2024）
在 Naive RAG 基础上增加：
- **查询优化**：查询重写、HyDE、子查询分解
- **混合检索**：稀疏检索（BM25）+ 稠密检索（语义向量）
- **重排序**：Cross-Encoder 对检索结果二次排序
- **上下文压缩**：去除无关信息后再送入 LLM

### 3.3 Agentic RAG（2025-2026）
LLM 从被动生成者升级为主动编排者：
- 规划-执行-评估-重试循环
- 多工具调用（向量检索 + 数据库查询 + API 调用 + 知识图谱遍历）
- 自纠错与自适应策略

## 4. RAG vs 微调 vs 提示工程

| 维度 | RAG | 微调 | 提示工程 |
|------|-----|------|---------|
| 知识更新 | 实时 | 需重新训练 | 有限 |
| 幻觉控制 | 较好 | 一般 | 较差 |
| 实施成本 | 中等 | 高 | 低 |
| 适用场景 | 知识密集型问答 | 领域风格适配 | 简单任务 |
| 可解释性 | 高（有引用） | 低 | 低 |

> **结论**：三者并非互斥。生产环境中常见的组合是 RAG（知识基础）+ 微调（领域适配）+ 提示工程（格式控制）。

## 5. RAG 系统的四阶段架构

```
┌─────────────────────────────────────────────────────┐
│                   离线：索引构建                      │
│  原始文档 → 解析清洗 → 智能分块 → 向量化 → 索引存储    │
├─────────────────────────────────────────────────────┤
│                   在线：检索增强                      │
│  用户查询 → 查询优化 → 混合检索 → 重排序 → 上下文增强   │
├─────────────────────────────────────────────────────┤
│                   生成与验证                          │
│  Prompt 构建 → LLM 生成 → 引用溯源 → 事实校验          │
├─────────────────────────────────────────────────────┤
│                   评估与反馈                          │
│  指标采集 → 链路追踪 → Golden Dataset → 持续优化       │
└─────────────────────────────────────────────────────┘
```

## 6. 核心性能指标

- **检索质量**：Recall@K、MRR（Mean Reciprocal Rank）、nDCG
- **生成质量**：忠实度（Faithfulness）、答案相关性（Answer Relevance）
- **系统性能**：TTFT（首 Token 时间）、端到端延迟、吞吐量
- **可信度**：幻觉率、引用准确率
