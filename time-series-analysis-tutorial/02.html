第2章 时序的组成与分解

大家好，我是你们的讲师。上一章我们认识了什么是时间序列，也看到了天气预报、股票曲线这些生活中的例子。但你有没有想过——一条看起来杂乱无章的曲线，其实可以拆解成几个非常清晰的部分？

就像医生通过血常规检查把血液拆成红细胞、白细胞、血小板来诊断病情一样，时间序列分解（Time Series Decomposition）就是把一条序列"拆开"，看看里面到底有什么。这个操作重要到什么程度呢？我可以负责任地说：不做分解，后面所有建模都是在盲人摸象。

让我讲一个小故事。几年前我帮一家连锁便利店做销售预测。原始销售额曲线看起来就是一条上升的线，我们团队新人直接拿原始数据训练了一个ARIMA模型，结果预测效果极差。后来一分解才发现：数据里藏着强烈的每周季节性——周末销售额比工作日高出40%；还有春节前后一个月的脉冲效应。如果不把这些成分拆开，模型会把季节波动当成噪声，把趋势变化误判成随机波动。这就是分解的意义。

2.1 四大成分

经典的时间序列分解把一条序列分成四个部分：

  • Trend（趋势）——长期走向，比如一个国家的GDP几十年持续增长
  • Seasonal（季节）——固定周期内的规律波动，比如冰淇淋销量每夏天都涨
  • Cyclical（周期）——非固定长度的涨落，比如经济繁荣-衰退循环
  • Residual（残差）——剩下的随机噪声，不可解释的部分

Trend 和 Cyclical 的区别新手经常搞混。简单说：Seasonal 是固定周期（比如一天、一周、一年），Cyclical 是不固定周期的长期波动。商业里说的"景气循环"通常持续3-10年，你不可能拿一年数据就看出来。

2.2 加法模型与乘法模型

分解的形式主要有两种：

加法模型（Additive Model）：Y = T + S + C + R

当季节波动的幅度不随趋势变化时用加法。比如某产品月销量长期在1000左右波动，季节性波动幅度始终在正负200。

乘法模型（Multiplicative Model）：Y = T x S x C x R

当季节波动的幅度与趋势水平成正比时用乘法。航空公司乘客量就是典型例子——随着航空业整体增长，夏季旺季的增量也越来越大，从50年前的几千人变成现在的几十万人。

特征加法模型乘法模型
季节波动幅度恒定随趋势变化
适用场景温度、湿度等自然量销售额、流量等经济量
数学形式Y = T + S + RY = T x S x R
对数变换后—变为加法模型
面试高频题：怎么判断用加法还是乘法？看残差图——如果残差的波动幅度随时间变化，或者季节峰的高度逐年增加，乘法模型更合适。一个更简单的做法是：先取对数，乘法模型就变成了加法模型，直接用加法方法处理即可。

2.3 经典分解法

经典分解法（Classical Decomposition）是最传统的方法，思路非常直观：

  1. 用移动平均估计趋势成分 T
  2. 从原序列减去 T（加法）得到 S + R
  3. 对每个季节位置取平均得到季节成分 S
  4. 剩下的就是残差 R

但它有几个硬伤：移动平均会丢失头尾数据；对异常值极度敏感；而且假设季节成分每年不变。所以现在很少有人用它了。目前工业界的主力是下面这位。

2.4 STL 分解

STL（Seasonal and Trend decomposition using Loess）是当今最主流的分解方法，由 Cleveland 等人在1990年提出。它的核心思想是用局部加权回归（Loess）来拟合趋势和季节成分，而不再依赖简单的移动平均。

STL 相比经典分解的几个巨大优势：

  • 可以处理任何类型的季节性，包括随时间变化的季节模式
  • 对异常值非常鲁棒（提供了稳健迭代版本）
  • 不损失头尾数据
  • 只有两个关键参数需要调节：trend window 和 seasonal window

下面是一段使用 Python 进行 STL 分解的示例代码：

import pandas as pd
import numpy as np
from statsmodels.tsa.seasonal import STL
import matplotlib.pyplot as plt

# 生成模拟数据：趋势 + 季节性 + 噪声
t = np.arange(1, 365)
trend = 0.02 * t + 10
seasonal = 5 * np.sin(2 * np.pi * t / 30)
noise = np.random.normal(0, 1, 364)
y = trend + seasonal + noise

# STL 分解
stl = STL(y, period=30, trend=91, seasonal=13)
result = stl.fit()

# 提取各成分
trend_est = result.trend
seasonal_est = result.seasonal
resid_est = result.resid

# 可视化（实际使用时取消注释）
# result.plot()
# plt.show()

实际业务中，STL 的一个经典应用场景是零售业销售预测。拿我们之前说的便利店数据为例，STL 能很好地把每周的周末效应、月中的促销效应和长期增长趋势分离开来，让后面做预测的模型只需要关注残差部分即可。

2.5 分解后做什么

分解不是终点，而是起点。拿到各成分后，常见的处理流程是：

  • 去趋势——后面讲平稳性时会看到，很多模型要求数据是平稳的，去掉趋势是第一道工序
  • 去季节——季节性调整（Seasonal Adjustment），让不同月份的数据可比。各国统计局发布的经济数据几乎都是季节调整后的
  • 分析残差——如果残差中还有明显模式，说明分解不充分或还有未捕捉的成分
踩坑提醒：我见过有人把分解后的残差直接当成"纯随机噪声"来处理。永远记住：残差只是"在当前分解方法下无法解释的部分"，换一种分解方法，残差可能就变样了。不要过度解读残差，但也别完全忽视它。

2.6 实操建议

在 Python 中，statsmodels 库提供了 STL 和 seasonal_decompose 两种工具。前者更现代，推荐优先使用。R 语言的 stl() 函数也是工业级实现。

关于参数调优：trend 参数控制趋势的平滑程度，设置越大趋势越平滑，通常取周期长度的奇数倍；seasonal 参数控制季节成分是否随时间变化，设置很大（比如 seasonal=period）相当于假设季节模式固定。如果不确定，从默认值开始，然后观察残差中是否还有周期性——如果还有，说明季节参数需要调整。

最后，聊到分解我们就不得不提到后面章节的主角。分解让我们看清数据的结构，而下一章我们将迎来时间序列建模的基石——平稳性。只有平稳的数据，才能用统计模型来建模和预测。你会看到，如果一个序列不平稳，所有经典的统计推断都会失效。让我们继续前进。
