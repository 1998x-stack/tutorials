第二章：文本预处理

大家好，欢迎回到NLP课程。今天我们要聊的话题可能听起来不太"高级"——文本预处理。但我要事先声明：你在工业界遇到的一半以上NLP问题，根源都在预处理上。模型再强大，喂进去的是垃圾，出来的也是垃圾。这个道理我花了三年才真正领悟到。

我2017年做过一个情感分析项目，BERT微调后准确率死活上不了80%。调参调了两周，最后发现是训练数据里有大量未转义HTML实体（比如"&amp;"），把tokenizer的词汇表污染了。就这一个问题，修复后直接涨了6个点。所以，千万别小看预处理。

2.1 分词：NLP的第一道工序

分词（Tokenization）是把连续的文本切成有意义的单元。这里有一个中英文的巨大差异：

语言分词方式难点示例
英文按空格+标点切分缩写（don't→do n't）、连字符（state-of-the-art）"I love NLP" → ["I", "love", "NLP"]
中文需要专门分词器歧义切分（"南京市长江大桥"）、未登录词"我爱自然语言处理" → ["我", "爱", "自然语言", "处理"]
日文需要形态素分析无空格、多种文字混合"自然言語処理が好き" → 需要MeCab等工具

中文分词的经典难题——"南京市长江大桥"到底怎么切？是"南京市/长江大桥"还是"南京/市长/江大桥"？人类靠常识消歧，机器需要上下文建模。

# 中文分词示例：jieba
import jieba

text = "南京市长江大桥"
print(list(jieba.cut(text)))
# ['南京市', '长江大桥']

# 加一个"欢迎"就变了
text2 = "欢迎南京市长江大桥"
print(list(jieba.cut(text2)))
# 可能变成 ['欢迎', '南京', '市长', '江大桥'] —— 这就是歧义！

⚠️ 踩坑记录：我曾经用jieba默认词典做医疗文本分词，"非小细胞肺癌"被切成["非", "小", "细胞", "肺癌"]，直接把"非"当成了否定词。加上自定义医疗词典后变成了["非小细胞肺癌"]。领域词典对分词质量的影响比你想象的大得多。

2.2 子词切分：BPE与WordPiece

现代NLP系统几乎不用传统的"词"作为最小单元了。为什么？因为词汇表不可能覆盖所有词。英语有几十万词，加上派生词、新词、拼写错误，无限多。子词切分（Subword Tokenization）优雅地解决了这个问题。

BPE（Byte Pair Encoding）的核心思想很简单：从字符开始，反复合并出现频率最高的相邻字符对。

# BPE 算法流程（简化版）
# 初始词汇表：每个字符一个token
# 语料："low"出现5次，"lower"出现2次，"newest"出现6次

# Step 1: 统计相邻pair频率
# (l,o) → 7次，(o,w) → 7次，(e,w) → 6次...

# Step 2: 合并最高频pair (e,s) → "es"
# 词汇表新增 "es"

# Step 3: 继续合并 (es,t) → "est"
# ...重复N次直到达到目标词汇表大小

# 最终可以把"lower"切分为 ["low", "er"] —— 共享low和er的含义

WordPiece是BERT使用的算法，和BPE的区别在于合并策略：BPE选频率最高的pair，WordPiece选能使训练语料似然度增加最多的pair。实际效果差别不大，但WordPiece用##标记非首词片段：

# BERT tokenizer 输出
"unaffordable" → ["un", "##aff", "##ord", "##able"]
# ## 表示这个词片段不是词的开始

2.3 文本归一化

归一化（Normalization）是把不同形式的文本映射到统一表示。主要包括：

操作说明例子注意事项
小写化所有字符转小写"Apple" → "apple"可能丢失信息（Apple是公司还是水果？）
Unicode归一统一等价字符编码"café" NFC vs NFD用NFKC/NFKD可以处理全角半角
数字归一把数字替换为占位符"call 555-1234" → "call NUM"电话号码和金额可能需要保留
词干提取去掉词缀得到词干"running" → "run"Porter Stemmer会过切：university→univers
词形还原还原到词典中的基本形式"better" → "good"需要词性标注，比词干提取更准确也更慢

实用建议：对于中文NLP，最重要的归一化是全角转半角（"，"→","）、繁体转简体、以及emoji和特殊符号的处理。如果你在做社交媒体文本，emoji的处理策略直接影响情感分析结果——😊该保留还是转成文本"[smile]"？

2.4 停用词与文本清洗

停用词（Stop Words）是"的、了、是、the、a、is"这类高频但通常不携带语义信息的词。传统NLP（TF-IDF时代）去除停用词是标准操作，但到了深度学习时代，这个做法值得商榷。

我做过一个实验：在BERT上做句对相似度任务，保留停用词的版本比去除停用词的版本准确率高2%。原因？BERT的Self-Attention需要完整的上下文，去掉停用词破坏了句法结构。

# 传统做法：去掉停用词
text = "The quick brown fox jumps over the lazy dog"
# 去停用词后：["quick", "brown", "fox", "jumps", "lazy", "dog"]

# 问题：丢失了"over"表示的空间关系，"the"指示的定指信息

文本清洗的其他常见操作：

  • HTML标签去除：用BeautifulSoup或正则，注意保留<img>的alt文本
  • 多余空白归一：连续空格、换行统一成一个空格
  • 控制字符移除：\x00、\x1a等不可见字符会导致tokenizer崩溃
  • URL/邮箱脱敏：替换为占位符 [URL]、[EMAIL]

黄金法则：预处理没有"标配"，要根据下游任务和模型来决定。用BERT就别去停用词，用TF-IDF就一定要去。预处理策略应该是你实验设计的一部分，不是盲目的流水线。
