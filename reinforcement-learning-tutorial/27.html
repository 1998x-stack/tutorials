第二十七章：从零实现DQN

本章目标：用 PyTorch 和 Gymnasium 从零实现一个完整的 DQN 智能体，在 CartPole-v1 环境上训练并达到收敛。全程手写，不依赖任何强化学习框架。读完本章你将拥有一个可直接跑通的 DQN 项目模板。

27.1 项目结构

完整的 DQN 实现包含四个核心模块，我们按功能拆分为以下文件：

文件作用核心类/函数
model.py定义 Q 网络QNetwork
replay_buffer.py经验回放缓冲区ReplayBuffer
agent.pyDQN 智能体逻辑DQNAgent
train.py训练入口与循环train()

这种拆分的好处是：每一部分职责单一，便于调试和后续替换（例如换一个网络结构或改用优先经验回放时只需修改对应文件）。

27.2 Q 网络实现

对于 CartPole-v1，观测是 4 维连续向量（小车位置、速度、杆角度、角速度），动作空间是 2 个离散动作（左、右）。我们用一个三层 MLP 作为 Q 函数逼近器：

import torch
import torch.nn as nn

class QNetwork(nn.Module):
    def __init__(self, state_dim=4, hidden_dim=128, action_dim=2):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, action_dim),
        )

    def forward(self, x):
        # x shape: (batch, state_dim)
        return self.net(x)

代码非常简单。输入 state_dim=4，通过两个 128 维的隐藏层和 ReLU 激活，最终输出每个动作对应的 Q 值（维度 2）。这里没有用偏置的特殊设置，全连接层默认带 bias。

踩坑记录：我在第一次实现时把第二个隐藏层忘掉了——直接把第一个隐藏层的输出接到输出层，心想"两个隐藏层多此一举"。结果训练了 500 回合始终无法收敛，平均回报只有 40 左右。后来加回第二个隐藏层，不到 150 回合就解决了 CartPole。对于 DQN，表达能力不足是最容易忽视的坑，而对于 CartPole 这个看似简单的问题，一层隐藏层就是不够。记住：欠拟合比过拟合更常见。

27.3 经验回放缓冲区

经验回放是 DQN 稳定训练的基石。它存储智能体与环境交互产生的转移元组，训练时从中随机采样，打破数据的时间相关性。

import numpy as np
import random
from collections import deque

class ReplayBuffer:
    def __init__(self, capacity=100000):
        self.buffer = deque(maxlen=capacity)

    def push(self, state, action, reward, next_state, done):
        self.buffer.append((state, action, reward, next_state, done))

    def sample(self, batch_size):
        batch = random.sample(self.buffer, batch_size)
        states, actions, rewards, next_states, dones = zip(*batch)
        return (
            np.array(states, dtype=np.float32),
            np.array(actions, dtype=np.int64),
            np.array(rewards, dtype=np.float32),
            np.array(next_states, dtype=np.float32),
            np.array(dones, dtype=np.float32),
        )

    def __len__(self):
        return len(self.buffer)

这里用 deque 自动管理容量，超出的旧数据会被丢弃。sample() 返回 NumPy 数组，由训练循环统一转化为张量——这样 ReplayBuffer 不依赖 PyTorch，便于单元测试和复用。

设计选择：很多实现会把张量转换放在 ReplayBuffer 内部。我的建议是保持缓冲区与深度学习框架无关。原因是：当你迁移到 TensorFlow 或 JAX 时，经验回放的逻辑完全相同，无需重写。框架相关的操作只在训练循环中处理即可。

27.4 DQN 智能体与训练循环

下面给出完整的 DQNAgent 类和训练脚本。其中包含策略网络 q_net、目标网络 target_net、epsilon-greedy 策略和软/硬更新机制。

# agent.py
import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np

class DQNAgent:
    def __init__(self, state_dim, action_dim, lr=3e-4,
                 gamma=0.99, tau=0.005, device="cpu"):
        self.device = device
        self.action_dim = action_dim
        self.gamma = gamma
        self.tau = tau
        self.q_net = QNetwork(state_dim, 128, action_dim).to(device)
        self.target_net = QNetwork(state_dim, 128, action_dim).to(device)
        self.target_net.load_state_dict(self.q_net.state_dict())
        self.optimizer = optim.Adam(self.q_net.parameters(), lr=lr)
        self.loss_fn = nn.SmoothL1Loss()  # Huber loss

    def act(self, state, epsilon=0.0):
        if np.random.random() < epsilon:
            return np.random.randint(self.action_dim)
        state_t = torch.FloatTensor(state).unsqueeze(0).to(self.device)
        q_values = self.q_net(state_t)
        return q_values.argmax(dim=1).item()

    def update(self, batch):
        states, actions, rewards, next_states, dones = batch
        states = torch.FloatTensor(states).to(self.device)
        actions = torch.LongTensor(actions).unsqueeze(1).to(self.device)
        rewards = torch.FloatTensor(rewards).unsqueeze(1).to(self.device)
        next_states = torch.FloatTensor(next_states).to(self.device)
        dones = torch.FloatTensor(dones).unsqueeze(1).to(self.device)

        # 当前 Q 值
        q_current = self.q_net(states).gather(1, actions)

        # 目标 Q 值（目标网络，不反向传播）
        with torch.no_grad():
            max_next_q = self.target_net(next_states).max(dim=1, keepdim=True)[0]
            q_target = rewards + self.gamma * max_next_q * (1 - dones)

        loss = self.loss_fn(q_current, q_target)

        self.optimizer.zero_grad()
        loss.backward()
        self.optimizer.step()

        # 软更新目标网络
        for param, target_param in zip(self.q_net.parameters(),
                                       self.target_net.parameters()):
            target_param.data.copy_(
                self.tau * param.data + (1 - self.tau) * target_param.data
            )

        return loss.item()

训练循环是整个项目的主干。下面这个脚本大约 60 行，完成后即可在终端直接运行：

# train.py
import torch
import gymnasium as gym
from model import QNetwork
from replay_buffer import ReplayBuffer
from agent import DQNAgent

def train():
    env = gym.make("CartPole-v1")
    state_dim = env.observation_space.shape[0]   # 4
    action_dim = env.action_space.n              # 2

    agent = DQNAgent(state_dim, action_dim, device="cuda" if torch.cuda.is_available() else "cpu")
    buffer = ReplayBuffer(capacity=100000)
    batch_size = 64
    n_episodes = 500
    epsilon_start, epsilon_end, epsilon_decay = 1.0, 0.01, 0.995
    epsilon = epsilon_start
    total_steps = 0

    for episode in range(1, n_episodes + 1):
        state, _ = env.reset()
        episode_reward = 0

        while True:
            action = agent.act(state, epsilon)
            next_state, reward, terminated, truncated, _ = env.step(action)
            done = terminated or truncated
            buffer.push(state, action, reward, next_state, done)
            state = next_state
            episode_reward += reward

            if len(buffer) > batch_size:
                batch = buffer.sample(batch_size)
                agent.update(batch)

            total_steps += 1
            if done:
                break

        epsilon = max(epsilon_end, epsilon * epsilon_decay)

        if episode % 20 == 0:
            print(f"Episode {episode:>3d} | Reward {episode_reward:>4.0f} "
                  f"| Epsilon {epsilon:.3f} | Steps {total_steps}")

    env.close()
    torch.save(agent.q_net.state_dict(), "dqn_cartpole.pth")

if __name__ == "__main__":
    train()
第二个踩坑提醒：初始化时千万别忘了把目标网络的权重拷贝给策略网络！我第一次写时只初始化了 self.target_net = QNetwork(...) 就跳过了 load_state_dict 这一步，结果目标网络和策略网络从不同的随机初始化开始，TD 目标一直在漂，损失根本不下降。这个 bug 查了我整整两个小时——看一眼代码、拍一下脑袋、叹一口气，循环往复。请一定记住：目标网络在开始时必须与策略网络拥有完全相同的参数。

27.5 评估与改进

为了让训练更稳健，我们在每 20 个 episode 后运行一次评估：关闭 epsilon 探索（设 epsilon=0），用当前策略网络跑 5 个 episode 取平均回报。CartPole-v1 的"已解决"标准是过去 100 个 episode 的平均回报 ≥ 500（新版 Gym 中的阈值），但在实际中如果能稳定达到 195 以上（旧版阈值）就已经说明 Q 函数学到了不错的策略。

训练回合平均回报（ε=0 评估）说明
0-3020 ~ 50随机探索阶段，几乎每步都倒
30-8050 ~ 150开始学会平衡，偶尔能撑过 100 步
80-150150 ~ 400Q 值逐渐准确，策略明显改善
150-200450 ~ 500收敛，稳定解决 CartPole

这是典型的 CartPole 学习曲线。如果你的实现迟迟停在 100 以下，请检查以下三点：

  1. 学习率：Adam 推荐 1e-4 ~ 3e-4，过高会导致 Q 值发散。
  2. 批大小：32 ~ 128 均可，64 是最保险的选择。
  3. 目标网络更新频率：软更新 tau=0.005 每步微调，或硬更新每 100 步整权重复制。
扩展方向（下一阶段的学习路线）：一旦你跑通了基础的 DQN，强烈建议尝试以下改进。这些被学术界证明有效的技巧可以让你的智能体在更复杂的环境（如 Atari 游戏）上取得好成绩。

1. Double DQN：将目标 Q 值中的 max 替换为策略网络选动作、目标网络算值，消除 Q 值高估。
2. Dueling 架构：把 Q 网络拆分为状态价值流和优势流，在动作空间大时收敛更快。
3. 优先经验回放：按 TD 误差的大小决定采样权重，让智能体更频繁地从"意外"的转换中学习。
4. 多步回报：用 n 步 TD 目标替代单步 TD 目标，在偏差和方差之间取得更好的平衡。

当你把上述技巧组合起来，就得到了 Rainbow DQN——这几乎是在 Atari 游戏上做 DQN 研究的标配基线。我们也将在后续章节中逐一深入这些改进。

向 Atari 迁移的注意事项：CartPole 和 Atari 最大的区别是观测空间。Atari 的输入是 (84, 84, 4) 的堆叠灰度帧，你需要用 CNN 替换 MLP，并添加帧预处理（降采样、灰度化、裁剪得分栏）。训练时通常使用 FrameSkip=4，学习率降至 1e-4，batch_size 升至 32，replay buffer 容量扩大到 1,000,000。这些参数来自 Mnih 2015 年的 Nature 论文，至今仍是 Atari DQN 实验的默认配置。
