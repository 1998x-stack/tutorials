前缀和思想

大家好，我是你们的讲师。今天我们来聊一个精巧而实用的技巧——前缀和（Prefix Sum）与差分（Difference Array）。前缀和的核心思想非常朴素：预处理出一个前缀和数组，然后用 O(1) 时间回答任意区间和查询。这是"空间换时间"的又一个经典案例。

假设有一个数组 arr = [a₀, a₁, a₂, ..., aₙ₋₁]，定义 prefix[i] = a₀ + a₁ + ... + aᵢ₋₁（前 i 个元素的和）。那么区间 [l, r) 的和就等于 prefix[r] - prefix[l]。这就是前缀和的全部秘密——一个简单的减法，省去了每次遍历求和的 O(n) 时间。

差分数组则反过来：它擅长处理区间批量加减的问题。给定一个数组，如果我们要在区间 [l, r] 上统一加一个数 x，差分数组可以在 O(1) 时间内完成操作，最后通过前缀和还原原数组。

核心关系：前缀和与差分互为逆运算。前缀和 = 差分的前缀和 = 原数组；差分 = 原数组的差分。理解了这个互逆关系，你就掌握了这对技巧的精髓。

一维前缀和

区域和检索 — 数组不可变（LeetCode 303）

这是前缀和最基础的用法——初始化时预处理前缀和数组，每次查询直接返回结果。

class NumArray:
    def __init__(self, nums: list[int]):
        self.prefix = [0]
        for num in nums:
            self.prefix.append(self.prefix[-1] + num)

    def sumRange(self, left: int, right: int) -> int:
        return self.prefix[right + 1] - self.prefix[left]

这里我用的定义是 prefix[i] 表示前 i 个元素的和（即 nums[0..i-1] 的和）。这样 sumRange(left, right) = prefix[right+1] - prefix[left]——公式非常干净。注意 prefix 的长度是 n+1，prefix[0] = 0，这个前置 0 的约定让我们不用单独处理 left=0 的边界情况。

和为 K 的子数组（LeetCode 560）

这道题是前缀和的"经典题中的经典题"。给定一个整数数组，统计有多少个和为 K 的连续子数组。

def subarray_sum(nums: list[int], k: int) -> int:
    prefix_sum = 0
    count = 0
    # key: 前缀和, value: 出现次数
    prefix_map = {0: 1}

    for num in nums:
        prefix_sum += num
        # 如果 prefix_sum - k 之前出现过，说明有以当前位置结尾的和为 k 的子数组
        if prefix_sum - k in prefix_map:
            count += prefix_map[prefix_sum - k]
        # 记录当前前缀和
        prefix_map[prefix_sum] = prefix_map.get(prefix_sum, 0) + 1

    return count

这道题的思维跨度很大：我们不是先算好所有前缀和再查，而是边遍历边计算边查询，用一个 HashMap 记录之前出现过的前缀和及其次数。对于当前位置 i，prefix_sum - k 在之前出现的次数，就是"以 i 结尾的、和为 k 的子数组"的个数。

为什么 prefix_map = {0: 1}？因为前缀和为 0 的子数组（空数组）算一次。如果某个子数组从开始到 i 的和正好等于 k，那么 prefix_sum - k = 0，我们需要从 map 中找到这个 0。

注意陷阱：560 是"子数组求和"而不是"子数组最大/最小"——不能用滑动窗口！因为数组包含负数，窗口的单调性不成立。很多人在 560 上尝试滑动窗口然后 WA，这是典型的"pattern 误用"。

二维前缀和

二维区域和检索（LeetCode 304）

从一维扩展到二维：给定一个二维矩阵，计算任意矩形区域的和。二维前缀和的核心公式是容斥原理：

  • 预处理：prefix[i+1][j+1] = prefix[i][j+1] + prefix[i+1][j] - prefix[i][j] + matrix[i][j]
  • 查询：sum(row1, col1, row2, col2) = prefix[row2+1][col2+1] - prefix[row1][col2+1] - prefix[row2+1][col1] + prefix[row1][col1]
class NumMatrix:
    def __init__(self, matrix: list[list[int]]):
        m, n = len(matrix), len(matrix[0])
        self.prefix = [[0] * (n + 1) for _ in range(m + 1)]
        for i in range(m):
            for j in range(n):
                self.prefix[i + 1][j + 1] = (
                    self.prefix[i][j + 1]
                    + self.prefix[i + 1][j]
                    - self.prefix[i][j]
                    + matrix[i][j]
                )

    def sumRegion(self, row1: int, col1: int, row2: int, col2: int) -> int:
        return (
            self.prefix[row2 + 1][col2 + 1]
            - self.prefix[row1][col2 + 1]
            - self.prefix[row2 + 1][col1]
            + self.prefix[row1][col1]
        )

画个图理解：prefix[i+1][j+1] 是从原矩阵 (0,0) 到 (i,j) 这个矩形的和。查询时加上 (row1,col1) 到 (row2,col2) 的矩形需要做两次减法和一次加法——这是因为减多了中间重叠的部分。

矩阵区域和（LeetCode 1314）

给定一个 m×n 的矩阵和一个整数 k，返回一个矩阵，其中每个元素是原矩阵对应位置周围 k 步范围内的元素之和。这是 304 的"反向"应用——对每个位置算一次矩形和。

def matrix_block_sum(mat: list[list[int]], k: int) -> list[list[int]]:
    m, n = len(mat), len(mat[0])
    # 构造二维前缀和
    prefix = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m):
        for j in range(n):
            prefix[i + 1][j + 1] = (
                prefix[i][j + 1] + prefix[i + 1][j]
                - prefix[i][j] + mat[i][j]
            )

    ans = [[0] * n for _ in range(m)]
    for i in range(m):
        for j in range(n):
            r1, c1 = max(0, i - k), max(0, j - k)
            r2, c2 = min(m - 1, i + k), min(n - 1, j + k)
            ans[i][j] = (
                prefix[r2 + 1][c2 + 1]
                - prefix[r1][c2 + 1]
                - prefix[r2 + 1][c1]
                + prefix[r1][c1]
            )
    return ans

这里要注意边界 max(0, i-k) 和 min(m-1, i+k) 的处理——不能超出矩阵范围。二维前缀和的灵活之处就在于这些边界细节。

差分数组

前缀和适合"频繁查询区间和"的场景，而差分数组（Difference Array）适合"频繁进行区间增减、最后一次性查询"的场景。差分数组就是原数组的"导数"——对差分数组求前缀和就能还原原数组。

航班预订统计（LeetCode 1109）

这里有 n 个航班，每个预订记录 [i, j, k] 表示从 i 到 j（包含）每个航班预订了 k 个座位。返回每个航班的总预订数。

def corp_flight_bookings(bookings: list[list[int]], n: int) -> list[int]:
    diff = [0] * (n + 2)  # 多开一点空间防止越界
    for i, j, k in bookings:
        diff[i] += k
        diff[j + 1] -= k

    # 对差分数组求前缀和得到结果
    ans = [0] * n
    current = 0
    for i in range(n):
        current += diff[i + 1]
        ans[i] = current
    return ans

这里的技巧是：对于区间 [i, j] 加 k，我们只在 diff[i] += k（区间起点加 k）和 diff[j+1] -= k（区间终点之后减 k）。然后对 diff 求前缀和，就能得到每个位置被加的总数。这个操作的巧妙之处在于，我们把"区间操作"转化为了"端点操作"，复杂度从 O(n) 降到了 O(1)。

类似的题目还有：拼车（LeetCode 1094）——在 [i, j) 区间内载客，求是否超载。完全可以用差分数组统计每个位置的乘客数，然后检查有没有超过 capacity 的地方。

踩坑与总结

前缀和的坑集中在前缀和哈希化的边界条件上。我最惨的一次踩坑是在 LeetCode 560 上——我忘写了 prefix_map = {0: 1}，结果第一个元素就等于 k 的情况永远统计不到。

另一个经典问题是整数溢出。前缀和的值可能非常大（比如数组长度 10⁵、元素范围 10⁹，前缀和可以达到 10¹⁴ 以上）。Python 的 int 是任意精度的，所以不需要担心，但如果用 C++/Java 就要用 long。在 Python 中思考这个问题虽然不会 crash，但理解大型数据下数值范围对算法的影响仍然很重要。

差分数组的常见错误是越界。当区间终点 j 等于 n-1（即最后一个元素）时，diff[j+1] 对应的是 diff[n]。如果差分数组只开到 n，就会越界。所以差分数组一定要多开一个位置。

最后，关于什么时候用前缀和 vs 滑动窗口：如果数组全是非负数且你需要"找和为 K 的子数组"，滑动窗口是更好的选择（O(1) 空间）。如果有负数存在，滑动窗口的单调性被破坏，就必须用前缀和+HashMap。这个区别很微妙但很重要。

思维提升：前缀和和差分数组代表了两种对立的思维方式——"预处理"和"延迟计算"。前缀和是"提前算好，快速查询"；差分是"快速操作，最后统一计算"。很多系统设计中也有类似的权衡：Cache 是"提前算好"，日志是"延迟计算"。算法思维在工程中的渗透比你想的更深。
