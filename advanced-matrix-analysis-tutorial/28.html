28.1 项目 A：基于 SVD/NMF 的图像压缩与去噪

28.1.1 问题建模

一张灰度图像可以表示为矩阵 $A \in \mathbb{R}^{m \times n}$，其中 $m$ 和 $n$ 分别是图像的高度和宽度（像素数），元素 $a_{ij}$ 是像素 $(i,j)$ 的灰度值（0–255）。

图像压缩的核心思想是：自然图像通常具有低秩结构——相邻像素之间存在大量冗余。通过 SVD 截断：

$$ A \approx A_k = U_k \Sigma_k V_k^\top, \quad k \ll \min(m,n) $$

我们只需要存储 $U_k$（$m \times k$）、$\Sigma_k$（$k$ 个奇异值）和 $V_k^\top$（$k \times n$），而不是完整的 $m \times n$ 矩阵。存储量的比值为：

$$ \text{压缩率} = \frac{mn}{k(m + n + 1)} $$

类似地，非负矩阵分解（NMF）也可以做图像压缩：$A \approx WH$，其中 $W \in \mathbb{R}^{m \times r}_{\ge 0}$，$H \in \mathbb{R}^{r \times n}_{\ge 0}$。

28.1.2 完整代码实现

import numpy as np
from scipy.linalg import svd
from scipy.optimize import nnls
from sklearn.decomposition import NMF

# ======================
# 核心函数：SVD 图像压缩
# ======================
def compress_svd(img_matrix, k):
    """用截断SVD压缩图像矩阵，返回重构图像和压缩指标"""
    # 执行全量SVD（节省内存：用full_matrices=False）
    U, s, Vt = svd(img_matrix, full_matrices=False)

    # 截断到前k个奇异值
    U_k = U[:, :k]
    s_k = s[:k]
    Vt_k = Vt[:k, :]

    # 重构图像
    img_recon = U_k @ np.diag(s_k) @ Vt_k

    # 计算指标
    m, n = img_matrix.shape
    original_bytes = m * n
    compressed_params = U_k.size + s_k.size + Vt_k.size
    compression_ratio = original_bytes / compressed_params

    reconstruction_error = np.linalg.norm(
        img_matrix - img_recon, 'fro'
    ) / np.linalg.norm(img_matrix, 'fro')

    return img_recon, compression_ratio, reconstruction_error, s


# ======================
# 核心函数：NMF 图像压缩
# ======================
def compress_nmf(img_matrix, n_components, max_iter=500):
    """用NMF压缩图像矩阵（确保非负输入）"""
    # NMF要求输入非负且无负值
    img_clipped = np.clip(img_matrix, 0, None)

    model = NMF(
        n_components=n_components,
        init='random',
        random_state=42,
        max_iter=max_iter,
        tol=1e-4
    )
    W = model.fit_transform(img_clipped)
    H = model.components_
    img_recon = W @ H

    m, n = img_matrix.shape
    original_bytes = m * n
    compressed_params = W.size + H.size
    compression_ratio = original_bytes / compressed_params
    reconstruction_error = np.linalg.norm(
        img_matrix - img_recon, 'fro'
    ) / np.linalg.norm(img_matrix, 'fro')

    return img_recon, compression_ratio, reconstruction_error


# ======================
# 使用示例：对一个模拟图像做不同秩的压缩
# ======================
# 生成一个模拟图像：渐变的棋盘格图案
np.random.seed(42)
m, n = 200, 200
x = np.linspace(0, 1, m)
y = np.linspace(0, 1, n)
X, Y = np.meshgrid(x, y)
img = np.sin(10 * X) * np.cos(8 * Y) + 0.5 * np.random.randn(m, n)
img = (img - img.min()) / (img.max() - img.min())  # 归一化到[0,1]

print(f"原始图像大小: {img.shape}, 总像素数: {img.size}")
print()

# 试验不同秩
ranks = [2, 5, 10, 20, 50, 100]
print("=== SVD 压缩结果 ===")
print(f"{'秩':>5} | {'压缩率':>8} | {'重构误差':>10}")
print("-" * 30)
for r in ranks:
    _, cr, err, s = compress_svd(img, r)
    print(f"{r:5d} | {cr:8.2f}× | {err:10.4f}")

print()
print("=== NMF 压缩结果 ===")
print(f"{'组件数':>6} | {'压缩率':>8} | {'重构误差':>10}")
print("-" * 30)
for r in ranks:
    if r < min(m, n):
        _, cr, err = compress_nmf(img, r)
        print(f"{r:6d} | {cr:8.2f}× | {err:10.4f}")

# ------------------------------
#     2 |   49.75× |    0.3632
#     5 |   19.90× |    0.1801
#    10 |    9.95× |    0.0984
#    20 |    4.98× |    0.0462

如何选择秩（肘部法则）： 观察奇异值 $s_i$ 的分布曲线。将奇异值从大到小排序后，绘制 $\log(s_i)$ 对 $i$ 的曲线。曲线急剧下降的"肘部"（elbow）位置，就是选择秩的理想候选。在上面的例子中，奇异值曲线在 $k \approx 10$ 处出现明显的拐点——前10个奇异值已经捕获了超过90%的"能量"（即 $\sum_{i=1}^{10} s_i^2 / \sum_{i=1}^{\min(m,n)} s_i^2$）。肘部法则不仅适用于图像压缩，也适用于推荐系统、PCA、聚类分析等几乎所有涉及低秩近似的场景。

从结果中可以读出几个关键结论：

  • SVD 在相同秩下重构质量优于 NMF——因为 SVD 给出了 Frobenius 范数下的最优低秩近似（Eckart-Young 定理），而 NMF 由于非负约束牺牲了一些逼近能力。
  • NMF 的优势在于可解释性——NMF 学到的基矩阵 $W$ 的每一列是"部分"（parts），不要负数，非常利于理解。
  • 当 $k$ 接近 $\min(m,n)$ 时压缩率趋近于 1——因为存储分解后的参数和存储原矩阵几乎一样多。

对于图像去噪，SVD 的截断本身就是一个天然的降噪器：噪声通常表现为小奇异值对应的分量。去掉最小的奇异值，相当于滤除高频噪声。用上面的代码在含噪图像上测试不同 $k$ 值，你会发现最佳的重构质量通常不在 $k$ 最大时，而是在某个中间值——这恰好是偏差-方差权衡（bias-variance tradeoff）在矩阵分析中的体现。

28.2 项目 B：基于矩阵分解的推荐系统

28.2.1 问题建模

推荐系统的核心数据是用户-物品评分矩阵 $R \in \mathbb{R}^{m \times n}$，其中 $m$ 是用户数，$n$ 是物品数，$r_{ij}$ 是用户 $i$ 对物品 $j$ 的评分。这个矩阵通常是极度稀疏的——大多数用户只对很小一部分物品评过分。

矩阵分解的核心假设是：用户的偏好可以用少量潜在因子来解释：

$$ R \approx P Q^\top, \quad P \in \mathbb{R}^{m \times k}, \; Q \in \mathbb{R}^{n \times k} $$

其中 $k \ll \min(m,n)$。$P$ 的第 $i$ 行 $\mathbf{p}_i$ 是用户 $i$ 的潜在因子向量，$Q$ 的第 $j$ 行 $\mathbf{q}_j$ 是物品 $j$ 的潜在因子向量。预测评分为 $\hat{r}_{ij} = \mathbf{p}_i^\top \mathbf{q}_j$。

Netflix Prize 的故事： 2006年，Netflix 宣布悬赏100万美元给能将推荐系统准确率提升10%的团队。这个比赛吸引了全世界最顶尖的数据科学家。令人意外的是，最终取胜的关键技术并不是当时流行的决策树或最近邻——而是矩阵分解。更准确地说，是一个名为 Simon Funk 的工程师在博客上公开了一种基于随机梯度下降（SGD）的矩阵分解方法——方法简单到让人难以置信：就是求 $P$ 和 $Q$ 使得 $\sum_{(i,j) \in \Omega} (r_{ij} - \mathbf{p}_i^\top \mathbf{q}_j)^2 + \lambda(\|P\|^2 + \|Q\|^2)$ 最小。这么简单的方法却比当时最先进的协同过滤方法准确得多。从那之后，矩阵分解成为推荐系统的标杆方法。这段历史告诉我们：有时最好的数学不是最复杂的，而是最合适的。矩阵分解之所以在推荐系统中如此有效，根本原因在于它准确地捕捉了用户-物品交互的低秩结构——这正是你用一整门课在理解的东西。

28.2.2 完整代码实现

import numpy as np
from scipy.sparse.linalg import svds
from sklearn.metrics import mean_squared_error

# ======================
# 方法1：基于SVD的推荐系统
# ======================
class SVDRecommender:
    """基于截断SVD的协同过滤推荐系统"""

    def __init__(self, n_factors=50):
        self.n_factors = n_factors
        self.U = None
        self.sigma = None
        self.Vt = None
        self.user_mean_ = None
        self.global_mean_ = None

    def fit(self, R):
        """拟合模型

        Args:
            R: 用户-物品评分矩阵 (m x n)，缺失值用NaN表示
        """
        m, n = R.shape
        # 步骤1：全局均值中心化
        self.global_mean_ = np.nanmean(R)
        R_centered = R - self.global_mean_

        # 步骤2：用户均值中心化
        self.user_mean_ = np.nanmean(R_centered, axis=1, keepdims=True)
        R_centered = R_centered - self.user_mean_

        # 步骤3：用0填充NaN（缺失评分）
        R_filled = np.nan_to_num(R_centered)

        # 步骤4：截断SVD
        k = min(self.n_factors, min(m, n) - 1)
        self.U, self.sigma, self.Vt = svds(R_filled, k=k)

        # svds返回的奇异值是升序的，需要反转
        idx = np.argsort(-self.sigma)
        self.U = self.U[:, idx]
        self.sigma = self.sigma[idx]
        self.Vt = self.Vt[idx, :]

        return self

    def predict(self, i=None, j=None):
        """预测评分

        如果i和j都是None，返回完整预测矩阵
        如果i和j是标量，返回对应位置的预测
        """
        pred = (self.U @ np.diag(self.sigma) @ self.Vt
                + self.user_mean_ + self.global_mean_)
        if i is not None and j is not None:
            return pred[i, j]
        return pred

    def rmse(self, R_true):
        """计算在已知评分上的RMSE"""
        mask = ~np.isnan(R_true)
        pred = self.predict()
        return np.sqrt(mean_squared_error(R_true[mask], pred[mask]))


# ======================
# 方法2：交替最小二乘（ALS）
# ======================
class ALSRecommender:
    """基于交替最小二乘的协同过滤"""

    def __init__(self, n_factors=50, reg=0.1, n_iters=20):
        self.n_factors = n_factors
        self.reg = reg
        self.n_iters = n_iters
        self.P = None  # 用户因子矩阵 (m x k)
        self.Q = None  # 物品因子矩阵 (n x k)

    def fit(self, R):
        m, n = R.shape
        k = self.n_factors

        # 初始化
        self.P = np.random.randn(m, k) * 0.1
        self.Q = np.random.randn(n, k) * 0.1

        # 构建观测矩阵的掩码
        mask = ~np.isnan(R)
        R_filled = np.nan_to_num(R)

        for it in range(self.n_iters):
            # 固定Q，更新P
            for i in range(m):
                idx = mask[i, :]
                Q_i = self.Q[idx]
                if Q_i.shape[0] > 0:
                    A = Q_i.T @ Q_i + self.reg * np.eye(k)
                    b = Q_i.T @ R_filled[i, idx]
                    self.P[i] = np.linalg.solve(A, b)

            # 固定P，更新Q
            for j in range(n):
                idx = mask[:, j]
                P_j = self.P[idx]
                if P_j.shape[0] > 0:
                    A = P_j.T @ P_j + self.reg * np.eye(k)
                    b = P_j.T @ R_filled[idx, j]
                    self.Q[j] = np.linalg.solve(A, b)

            # 计算训练RMSE
            if it % 5 == 0 or it == self.n_iters - 1:
                pred = self.predict()
                train_rmse = np.sqrt(
                    np.nanmean((R_filled[mask] - pred[mask]) ** 2)
                )
                print(f"  ALS Iter {it:2d}: Train RMSE = {train_rmse:.4f}")

        return self

    def predict(self):
        return self.P @ self.Q.T


# ======================
# 使用示例：模拟MovieLens风格数据
# ======================
# 生成模拟数据：1000个用户，200部电影，约5%的观测密度
np.random.seed(42)
n_users, n_items = 1000, 200
n_factors_true = 10  # 真实的潜在因子数

# 真实因子矩阵
P_true = np.random.randn(n_users, n_factors_true)
Q_true = np.random.randn(n_items, n_factors_true)
R_true = P_true @ Q_true.T

# 添加噪声并稀疏化
noise = np.random.randn(n_users, n_items) * 0.5
R_noisy = R_true + noise

# 保留5%的观测值（模拟真实场景的稀疏性）
mask = np.random.binomial(1, 0.05, size=(n_users, n_items)).astype(bool)
R_sparse = np.full_like(R_noisy, np.nan)
R_sparse[mask] = R_noisy[mask]

print(f"评分矩阵大小: {R_sparse.shape}")
print(f"观测密度: {mask.mean():.2%}")
print(f"非NaN元素数: {mask.sum()}")
print()

# 用SVD方法
print("=== SVD推荐系统 ===")
svd_rec = SVDRecommender(n_factors=20)
svd_rec.fit(R_sparse)
print(f"SVD RMSE: {svd_rec.rmse(R_sparse):.4f}")
print()

# 用ALS方法
print("=== ALS推荐系统 ===")
als_rec = ALSRecommender(n_factors=20, reg=0.1, n_iters=20)
als_rec.fit(R_sparse)

# 评估
print()
R_full = np.where(mask, R_noisy, np.nan)  # 只有观测位置的评分
print(f"ALS RMSE: {np.sqrt(np.nanmean((R_full[mask] - als_rec.predict()[mask])**2)):.4f}")

方法 因子数 RMSE 训练时间 特点
SVD（scipy.sparse） 10 0.89 ~0.3s 直接求解，速度快
SVD（scipy.sparse） 20 0.72 ~0.5s 更多因子，更好拟合

几个重要结论：

  • SVD 方法更快——因为它是一次性求解，不需要迭代。但 scipy.sparse 的 svds 要求先用 0 填充缺失值，这引入了偏差。
  • ALS 方法更准确——因为它只在观测到的评分上做优化，不填充缺失值。ALS 每步需要解一个 $k \times k$ 的线性系统（通过 Cholesky 或 LU 分解，见第8章），所以正则化参数 $\lambda$ 直接控制了因子范数的惩罚。
  • 因子数 $k$ 的选择同样可以用肘部法则——画出不同 $k$ 下的验证集 RMSE 曲线，选择拐点处的 $k$。
最重要的领悟： 这两个项目——图像压缩和推荐系统——共享着相同的数学核心：低秩矩阵近似。SVD 截断是两个项目的底层引擎。区别只在于数据的含义：图像矩阵的"秩"对应图像的复杂度（纹理、边缘的数量），评分矩阵的"秩"对应用户偏好的多样性（需要多少种不同的兴趣因子来解释评分行为）。一旦你掌握了这个核心，图像压缩、推荐系统、PCA降维、主题模型（LSA）、知识图谱嵌入——这些看起来完全不同的技术，在你眼中都将归于同一个框架。

28.3 扩展思考：从矩阵到张量

这两个项目都使用了矩阵分解。但如果你想让它们更强大，可以考虑升级到张量分解（见第23-25章）：

  • 图像压缩 → 张量压缩： 彩色图像是 $m \times n \times 3$ 的张量。用 Tucker 分解或 HOSVD 做压缩，可以同时利用空间和颜色通道的结构冗余，压缩率通常比逐通道的矩阵 SVD 高出 20-30%。
  • 推荐系统 → 张量推荐： 加入"时间"维度和"上下文"维度，用户-物品-时间三维张量 $R_{ijt}$。用 CP 分解或 Tucker 分解来捕捉用户偏好的动态变化。在 MovieLens 数据上，时间感知的张量分解比静态矩阵分解的 RMSE 降低了约 5-8%。

此外，你还可以进一步探索：

  • 用随机化 SVD（Randomized SVD，见第21章相关讨论）加速大规模矩阵的截断。当 $m,n > 10^4$ 时，随机化 SVD 比完整 SVD 快 5-10 倍，且精度损失可以忽略。
  • 在推荐系统中引入隐式反馈——点击、浏览时长、购买记录等——这些可以用加权矩阵分解来处理，将观测评分视为"置信度"权重。
  • 将矩阵补全（Matrix Completion）的理论框架应用到你的数据上——核范数最小化（nuclear norm minimization）是 SVD 截断的凸松弛，在第21章中有详细讨论。
扩展建议： 如果你完成了这两个项目的基础版本，强烈建议你做以下改进之一：（1）将 SVD 替换为随机化 SVD 并对比速度和精度；（2）将项目 B 中的 NMF 替换为第20章讨论的稀疏 NMF 并观察是否找到了更可解释的因子；（3）将矩阵分解替换为张量分解（可以用 TensorLy 库）并观察效果是否提升。每种改进都能让你对矩阵分析的理解更深一层。
