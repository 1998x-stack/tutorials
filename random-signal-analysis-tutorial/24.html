经典方法的局限与参数化革命

大家好，我是你们的讲师。上一章我们学习了经典功率谱估计，最后我留了一个问题：如果数据长度有限，你如何分辨两个频率仅差 1Hz 的信号？

在经典方法的世界里，答案有些令人沮丧——你只能依靠增加数据长度 N 来提升频率分辨率。这个分辨率极限被称为瑞利限（Rayleigh limit），约为 Δf ≈ 1/N。如果你想分辨 100Hz 和 101Hz，采样率 1000Hz 时你需要至少 1000 个采样点——也就 1 秒的数据。如果信号只能稳定存在 0.5 秒呢？经典方法到极限了。

但这还不是全部问题。经典方法还隐式假设了数据窗外观测为零——这个"截断"操作在频域表现为频谱泄漏，弱信号会被强信号的旁瓣淹没。面对这些困境，信号处理领域在 1970-80 年代迎来了一场革命：参数化谱估计。它的核心思想是：不要直接对数据做傅里叶变换，而是先假设数据服从一个参数化模型（如 AR 模型），然后从数据中估计模型参数，最后用模型参数计算出一个"完美"的功率谱。本章就带你走进这个世界。

AR 模型谱估计

将信号建模为白噪声驱动的 AR 过程

AR 模型（自回归模型）的基本思想非常直观：当前的信号值可以由过去 p 个值的线性组合加上一个白噪声激励来预测：

# AR(p) 模型：p 阶自回归
x[n] = -a₁·x[n-1] - a₂·x[n-2] - ... - aₚ·x[n-p] + w[n]

其中 w[n] 是均值为零、方差为 σ² 的白噪声

这个模型的含义是：信号 x[n] 的"记忆"只延续到过去的 p 个时刻——超过 p 步的历史信息，全部通过这 p 个系数间接体现了。这正是"自回归"这个名字的由来——它在对自己做回归。

一旦我们估计出了 AR 系数 a₁, a₂, ..., aₚ 和白噪声方差 σ²，功率谱就可以通过一个简单的闭式公式直接计算：

# AR 谱估计公式
S_AR(f) = σ² / |1 + Σ aₖ · e^(-j2πfk)|²     (|f| ≤ 0.5)

# 离散频率形式
S_AR[k] = σ² / |1 + Σ aₖ · e^(-j2πkn/N)|²

对比一下经典周期图和 AR 谱的差异：周期图是对数据做傅里叶变换然后平方，方差大；AR 谱是对模型做傅里叶变换，方差小（因为模型参数是从全部数据中联合估计的）。代价是：如果信号不符合 AR 模型假设，AR 谱会给出错误的结果。

直觉理解：AR 模型谱估计可以看作是一种"外推"——经典方法默认窗外数据为零，导致频谱泄漏；而 AR 模型用估计出的系数"推断"出窗外数据的可能行为，从而等效地得到了更长的数据，提高了频率分辨率。这就好比给你看一段旋律的前几秒，AR 模型试图猜出后面的音符——猜对了就获得了超越数据长度的"分辨率"。

Yule-Walker 方程

如何从数据中估计 AR 系数？最经典的方法通过 Yule-Walker 方程来实现。它的思路是：将 AR 模型方程两边乘以 x[n-m] 并取期望，利用自相关函数的定义，得到一组线性方程：

# Yule-Walker 方程（矩阵形式）
[ R(0)   R(1)   ...  R(p-1) ] [ a₁ ]     [ R(1) ]
[ R(1)   R(0)   ...  R(p-2) ] [ a₂ ]     [ R(2) ]
[  ...    ...    ...   ...   ] [ ... ] = -[ ... ]
[ R(p-1) R(p-2) ...  R(0)  ] [ aₚ ]     [ R(p) ]

# 白噪声方差由下式给出：
σ² = R(0) + Σ aₖ · R(k)

其中 R(k) 是信号的自相关函数。在实际应用中，我们无法获得真实的 R(k)，只能用样本自相关估计 R̂(k) 代入。

这个方程组看起来平淡无奇，但它的矩阵结构非常特殊——系数矩阵是托普利兹矩阵（Toeplitz matrix，每条对角线上的元素相同），并且是正定对称的。这两个性质让我们可以用远比高斯消元高效的算法来求解。

Levinson-Durbin 递推

O(p²) 的高效求解

一般来说，解一组 p 个方程的线性系统需要 O(p³) 的计算量。但如果矩阵是托普利兹正定矩阵，Levinson-Durbin 算法可以将复杂度降到 O(p²)——对于几百阶的 AR 模型，这速度差距是巨大的。

算法的核心思想是递推：从 1 阶 AR 模型开始，逐步增加阶数，每一阶的参数都由前一阶的结果导出。具体来说：

# Levinson-Durbin 递推（伪代码）
# 输入：自相关 R[0], R[1], ..., R[p]
# 输出：AR 系数 a₁...aₚ, 反射系数 K₁...Kₚ, 误差 σ²

σ²₀ = R[0]                     # 0 阶预测误差
for m = 1 to p:
    # 计算第 m 阶的反射系数
    Kₘ = (R[m] + Σ aᵢ·R[m-i]) / σ²ₘ₋₁

    # 更新 AR 系数（aₘₘ = Kₘ）
    aₘₘ = Kₘ
    for i = 1 to m-1:
        aᵢ = aᵢ + Kₘ · aₘ₋ᵢ

    # 更新预测误差方差
    σ²ₘ = σ²ₘ₋₁ · (1 - Kₘ²)

这个递推有一个非常好的副产品——反射系数 Kₘ。反射系数可以在递推过程中用来判断 AR 模型的稳定性：所有反射系数的绝对值都必须小于 1（即 |Kₘ| < 1），等价于 AR 多项式的所有根都在单位圆内。如果递推过程中某个 |Kₘ| ≥ 1，说明数据可能不是平稳的，或者模型阶数选择有问题。

工程实用技巧：反射系数本身也携带了信号的物理信息。在地震勘探中，反射系数对应着不同地质层之间的声阻抗变化界面——地震信号处理中的"反褶积"操作，本质上就是估计反射系数序列。这是 Levinson-Durbin 算法在地球物理学中的重要应用。

Burg 算法的思路

Yule-Walker 方程 + Levinson-Durbin 递推虽然高效，但它有一个潜在问题：在估计自相关函数时使用了加窗操作（假设窗外数据为零），这会给低阶自相关估计引入偏差。Burg 算法巧妙地绕过了这个步骤——它直接利用前后向预测误差来估计反射系数，而不需要先估计自相关。

具体来说，Burg 算法选择反射系数 Kₘ 使得前向预测误差和后向预测误差的平均功率最小。这种方法的优点是：

  • 不需要显式估计自相关，避免了加窗偏差
  • 对短数据序列效果更好
  • 保证多项式根在单位圆内（保证稳定）

在实践中，Burg 法通常比 Yule-Walker 法给出更尖锐的谱峰，对短数据的谱估计效果更好。但它也有缺点：对于正弦信号，Burg 法可能出现谱线分裂现象——一个正弦分量被分解成两个相邻的虚假谱峰。

子空间方法：MUSIC 算法

如果说 AR 模型谱估计是参数化方法中的"主力选手"，那么子空间方法就是"特种部队"——它在特定条件下拥有令人惊叹的超分辨率能力。

信号子空间与噪声子空间

MUSIC（Multiple Signal Classification）算法的出发点非常聪明：对于一个由 p 个正弦波加噪声组成的信号，我们可以构造一个自相关矩阵，然后对它做特征分解。

假设信号由 p 个正弦波（频率 f₁, f₂, ..., fₚ）加白噪声组成。构造 M×M 的自相关矩阵 R（M > p），特征分解后：

# 特征分解自相关矩阵
R = Σ λᵢ · vᵢ · vᵢᴴ

# 特征值分布：
λ₁ ≥ λ₂ ≥ ... ≥ λₚ >> λₚ₊₁ ≈ ... ≈ λₘ = σ²

# 前 p 个大特征值 → 信号子空间
# 后 M-p 个小特征值 → 噪声子空间

大的特征值对应着信号，小的特征值对应着噪声。关键洞察：信号子空间和噪声子空间是正交的。这意味着，真实的信号频率对应的导向矢量与噪声子空间正交，而正交性意味着点积为零。

MUSIC 算法的"伪谱"（pseudospectrum）定义为：

# MUSIC 伪谱
P_MUSIC(f) = 1 / Σ |eᴴ(f) · vₖ|²     (k = p+1 ... M)

其中 e(f) = [1, e^(j2πf), e^(j2πf·2), ..., e^(j2πf·(M-1))]ᵀ
vₖ 是噪声子空间的特征向量

当 f 等于真实信号频率时，e(f) 落在信号子空间中，与噪声子空间正交，分母趋近于零，伪谱出现尖锐峰值。这个峰值尖锐到什么程度？理论上可以无限尖锐——这意味着 MUSIC 可以分辨出间距小于瑞利限的两个频率！这就是"超分辨率"的来源。

超分辨率：经典方法的瑞利限 ≈ 1/N，但 MUSIC 可以突破这个限制。举例来说，你用 64 个数据点可以分辨 100Hz 和 101Hz 吗？经典方法不行（需要 1000 点），但 MUSIC 可以！这是因为 MUSIC 利用的是信号的模型结构（正弦波加噪声），而非单纯依靠数据长度。当然，天下没有免费的午餐——MUSIC 需要你知道信号中正弦波的数量 p，而且对模型失配非常敏感。

Python 实战：AR 谱估计与 MUSIC

下面我们用 Python 实现 AR 谱估计（Burg 法）和 MUSIC 算法，对比它们与经典 Welch 法在短数据条件下分辨两个接近频率的能力。

import numpy as np
import matplotlib.pyplot as plt
from scipy import signal

# 生成短数据：两个相近频率的正弦波 + 白噪声
np.random.seed(42)
fs = 1000
N = 64                    # 仅 64 个点 !
t = np.arange(N) / fs
f1, f2 = 100, 108         # 仅差 8Hz，瑞利限 ≈ 1000/64 ≈ 15.6Hz
x = np.sin(2*np.pi*f1*t) + np.sin(2*np.pi*f2*t)
x += 0.3 * np.random.randn(N)

# 1. Welch 法（经典）
f_wel, P_wel = signal.welch(x, fs, nperseg=32, noverlap=16)

# 2. Burg 法 AR 谱估计
# 使用 spectrum 库中的 pburg（需 pip install spectrum）
from spectrum import pburg
p = 8  # AR 模型阶数
P_burg = pburg(x, p, NFFT=512)
P_burg.plot()

# 3. MUSIC 算法（简化实现）
from scipy.linalg import eigh
M = 12  # 自相关矩阵维度
R = np.zeros((M, M), dtype=complex)
for i in range(M):
    for j in range(M):
        R[i,j] = np.correlate(x, x, mode='full')[N-1 + (i-j)] / N
eigvals, eigvecs = eigh(R)

# 假设已知信号数 p_signal = 2
p_signal = 2
Vn = eigvecs[:, :M-p_signal]  # 噪声子空间

f_music = np.linspace(0, fs/2, 1000)
P_music = np.zeros_like(f_music)
for idx, f in enumerate(f_music):
    e = np.exp(2j*np.pi*f*np.arange(M)/fs)
    P_music[idx] = 1 / np.sum(np.abs(e.conj() @ Vn)**2)

# 绘制对比
plt.figure(figsize=(12, 6))
plt.subplot(121); plt.semilogy(f_wel, P_wel);
plt.title('Welch (classical)'); plt.xlim(50, 180)
plt.subplot(122); plt.plot(f_music, 10*np.log10(P_music));
plt.title('MUSIC (subspace)'); plt.xlim(50, 180)
plt.tight_layout()
plt.show()

运行这段代码，你会看到令人震撼的对比：Welch 法只看到一个宽宽的谱包络，完全无法分辨 100Hz 和 108Hz 两个频率；而 MUSIC 的伪谱在两个频率处出现了两条尖锐如针的谱峰，一目了然。这就是超分辨率的威力。

注意：MUSIC 伪谱的纵坐标不是功率值（所以叫"伪谱"），它只给出频率的位置信息，不包含各频率分量的真实功率大小。如果你还需要知道每个频率分量的幅度，可以在 MUSIC 定位频率后，再用最小二乘来估计幅度和相位。

踩坑故事：AR 阶数选择之痛

讲一个我在研究生期间踩过的坑。当时我在做雷达目标微多普勒分析——雷达波照射到旋转的风扇叶片上，回波中会携带叶片旋转产生的微多普勒调制。我们需要从回波中精确估计旋转频率，从而识别目标（是直升机还是无人机）。

我拿到了几百个脉冲的回波数据，长度很短（每个距离门只有 128 个脉冲），经典方法根本无法分辨微多普勒频率的细微变化。于是我选择了 AR 模型谱估计，心想："参数化方法，分辨率高，完美！"

第一次跑，AR 模型阶数我凭感觉选了 p=6。结果谱峰非常平滑，本来应该出现的二次谐波和三次谐波完全看不见——阶数太低，模型容量不够，把细微的谱峰都抹平了。于是我改成 p=30。这下可好，谱图上密密麻麻全是峰——除了真实的旋转频率和谐波，还有大量虚假峰。这是因为阶数过高，模型把噪声的随机起伏也"拟合"进去了，类似于机器学习中的过拟合。

最后我花了大量时间尝试不同的阶数选择准则：AIC（赤池信息准则）、BIC（贝叶斯信息准则）、最终预测误差（FPE）……不同的准则给出的最优阶数差异很大。最终我采用的方案是：用 AIC 选出一个候选阶数范围（6-12），然后在这个范围内用物理合理性作为判据——只看那些在物理上可能存在的频率成分对应的谱峰。

教训：AR 模型阶数选择是参数化谱估计中最关键的步骤——没有之一。阶数太低 → 平滑掉真实信号；阶数太高 → 引入虚假谱峰。建议：先用 AIC/BIC 准则给出参考范围，再结合物理知识（你知道信号应该有几个频率分量吗？）做最终决定。如果完全不知道信号结构，可以尝试从低阶逐步增加，观察谱峰是否稳定出现——稳定出现的峰更可能是真实的，随阶数跳来跳去的峰多半是假的。

本章小结

本章我们学习了现代功率谱估计的两种主要思路。参数化方法（AR 模型谱估计）通过假设信号符合自回归模型，用模型参数计算功率谱，在短数据条件下能够突破经典方法的瑞利限。子空间方法（MUSIC）更进一步，通过对自相关矩阵的特征分解分离信号和噪声子空间，实现了令人惊叹的超分辨率能力。

但请记住：每一个"超能力"都有代价。参数化方法需要你选择模型阶数，子空间方法需要你知道信号的数量，模型假设如果与真实信号不符，结果会严重失真。工程实践中的智慧在于：先用经典方法（Welch）得到稳健但分辨率有限的估计，再用现代方法在关键频率区域"放大观察"。

下一章，我们将进入一个全新的方向——自适应滤波。前三章的维纳滤波、卡尔曼滤波和本章的谱估计假设信号的统计特性是已知或固定的。但如果噪声环境在不断变化呢？自适应滤波能够在线调整滤波器系数，实时跟踪信号统计特性的变化。从手机上的回声消除到心电图中的胎儿心跳提取，自适应滤波无处不在。
