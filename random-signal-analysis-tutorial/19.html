开篇：从离散事件到随机脉冲的桥梁

大家好，我是你们的讲师。在前面几章中，我们讨论了高斯过程、窄带信号和马尔可夫链——这些随机过程都有一个共同特点：它们描述的是连续变化的物理量，比如电压的起伏、相位的变化。但生活中还有另一类随机现象，它们发生在离散的时间点上——顾客到达收银台、网络数据包到达路由器、光电探测器接收到光子。这类"事件到达"的随机性，需要用一种全新的工具来刻画。

本章要介绍的泊松过程，就是描述"随机到达"最经典的数学模型。它不仅是排队论和通信网络的基石，也是理解散弹噪声（shot noise）的关键——而散弹噪声正是电子器件中一种基本的噪声类型。从光通信中的光子计数到半导体器件中的电流起伏，泊松过程的身影无处不在。

泊松过程的定义与性质

计数过程：从"事件到达"说起

想象一个场景：你盯着一个网络监控仪表盘，观察数据包到达某个交换机的时间。你在 t=0 时刻开始计数，记 N(t) 为 [0, t] 时间内到达的数据包总数。N(t) 是一个随机变量——下一时刻它变成多少，取决于这段时间内到达了多少个包。{N(t), t ≥ 0} 这样的随机过程被称为计数过程（counting process），它的样本路径是一条从0开始、每次事件到达时跳变+1的阶梯函数。

泊松过程是在所有计数过程中最重要的一个。它其实是两条假设约束下的产物：

  • 独立增量 (independent increments)：任意两个不交叠的时间段内，事件到达的数量是相互独立的。换句话说，过去到达了多少个包，不影响未来一段时间内到达的包数。
  • 平稳增量 (stationary increments)：事件到达的数量只依赖于时间段的长度，而不依赖于其起始位置。也就是说，[1s, 3s] 和 [5s, 7s] 内的到达数分布相同。

有了这两条性质，加上"事件不会同时到达"这一常规假设，就可以推导出泊松过程的完整分布。

核心公式：事件计数与泊松分布

对于强度为 λ 的泊松过程（λ 表示单位时间内事件的平均到达率），在长度为 t 的时间段内恰好有 k 个事件到达的概率为：

P(N(t) = k) = (λt)^k · e^(-λt) / k! ,   k = 0, 1, 2, ...

这个公式你应该很眼熟——它就是泊松分布的概率质量函数。参数 λt 既是均值也是方差：E[N(t)] = Var[N(t)] = λt。泊松分布的一个有趣特点是均值等于方差——这在实际应用中是一个可以用来检验数据是否符合泊松假设的快速指标。

直观理解：λ 是"事件到达的速率"，单位是"个/秒"。如果 λ = 5，那么平均每秒钟有5个事件到达。但泊松过程的美妙之处在于，它不仅仅告诉你均值是5，还告诉你"一秒钟内恰好到达3个的概率是多少"——这在系统容量设计、缓冲器大小估算中至关重要。

从定义还可以推出泊松过程的联合分布：对任意时间点 0 ≤ t₁ < t₂ < ... < tₙ，增量 N(t₁)-N(0), N(t₂)-N(t₁), ..., N(tₙ)-N(tₙ₋₁) 是相互独立的泊松随机变量，参数分别为 λ(t₁-0), λ(t₂-t₁), ..., λ(tₙ-tₙ₋₁)。这个性质在仿真中非常有用——你无需模拟每一时刻的到达，只需按时间段生成独立泊松变量即可。

到达时间与时间间隔

第 k 次到达时间：Gamma 分布

现在我们换个视角：不再问"一段时间内到了多少"，而是问"第 k 个事件什么时候到"。记 Sₖ 为第 k 个事件到达的时刻，它满足 Sₖ = inf{t ≥ 0: N(t) = k}。

由于泊松过程的增量独立且平稳，第 k 次到达时间 Sₖ 服从Gamma分布（也称埃尔朗分布）：

Sₖ ~ Gamma(k, λ)

        其PDF为：
        f_{Sₖ}(t) = λ^k · t^{k-1} · e^(-λt) / (k-1)!,   t ≥ 0

特别地，当 k=1 时，S₁ 即第一次到达时间，它服从指数分布——这与下面要讲的到达间隔是一回事。

到达时间间隔：指数分布的无记忆性

如果说泊松过程是计数过程的"王冠"，那指数分布就是这颗王冠上的明珠。记 X₁ = S₁, X₂ = S₂ - S₁, ..., Xₙ = Sₙ - Sₙ₋₁ 为相邻事件之间的时间间隔。可以证明：

X₁, X₂, ..., Xₙ 独立同分布，均服从指数分布 Exp(λ)

        PDF: f_X(t) = λ · e^(-λt),   t ≥ 0
        CDF: F_X(t) = 1 - e^(-λt),   t ≥ 0
        均值: 1/λ,  方差: 1/λ²

指数分布有一个在随机过程中极其重要的性质——无记忆性（memoryless property）：

P(X > s + t | X > s) = P(X > t),   ∀ s, t ≥ 0

翻译成大白话："如果我已经等了 s 秒还没等到事件，那我还需要再等 t 秒以上的概率，和从头开始等 t 秒以上的概率是一样的。"——也就是说，历史等待时间对未来的等待没有任何信息。这正是泊松过程"独立增量"假设的直接后果。如果事件到达有"忙时"和"闲时"之分，那就不满足无记忆性了。

工程意义：指数分布的无记忆性让泊松过程成为一个"完美随机"的到达模型——它没有状态、没有记忆、没有任何可预测的模式。这既是它的优势（数学处理极其简洁），也是它的局限（现实中的到达往往有记忆，如网络流量的自相似性）。理解这一点至关重要：在什么场景下可以使用泊松模型，什么场景下不行。

泊松冲激序列

在随机信号分析中，我们经常关心的不是"事件到达的时间本身"，而是事件到达时产生的物理效应。一个典型的模型是：每个事件到达时刻 tᵢ 产生一个单位冲激 δ(t-tᵢ)。这样就构造了一个泊松冲激序列：

S(t) = Σᵢ δ(t - tᵢ),   求和遍及所有到达时刻

这个过程在数学上是一个广义随机过程——它的"值"定义在广义函数空间上，而不是普通的实数。但它的统计特性却有非常清晰的物理解释。

泊松冲激序列的统计特性

泊松冲激序列的均值函数为：

E[S(t)] = λ

也就是说，均值的功率谱密度为 λ·δ(ω)。而其交流部分（零均值化后）的功率谱密度在所有非零频率上为常数：

S_{SS}(ω) = λ,   ω ≠ 0

注意这个结论——泊松冲激序列的 PSD 在所有频率上平坦！这与我们在第10章学过的白噪声的定义完全一致。也就是说，泊松冲激序列（扣除直流分量后）是严格意义上的白噪声——尽管它的幅度分布不是高斯分布，但其功率谱是平坦的。

联系与辨析：白噪声的定义是关于功率谱的平坦性，而非幅度分布。高斯白噪声是"幅度服从高斯分布 + 功率谱平坦"，而泊松冲激序列是"幅度服从泊松冲激分布 + 功率谱平坦"。两者都是白噪声，但统计特性截然不同。这个区别在系统分析中很重要——通过线性系统后，高斯白噪声的输出仍然是高斯过程，而泊松白噪声的输出则不是。

散弹噪声

有了泊松冲激序列的基础，我们现在可以讨论一个在电子工程中极其重要的噪声模型——散弹噪声（shot noise）。

物理模型：电子发射的随机性

散弹噪声的物理起源可以追溯到真空中电子管的阴极发射，或半导体中载流子穿越势垒的过程。以光电探测器为例：光子随机到达探测器表面，每个光子以一定的量子效率激发一个电子。这些电子的到达时刻服从泊松过程，而每个电子在电路中产生一个电流脉冲 h(t)。因此，总的散弹噪声电流可以写为：

I(t) = Σᵢ Aᵢ · h(t - tᵢ)

其中 tᵢ 是第 i 个载流子的到达时刻（泊松过程），Aᵢ 是第 i 个载流子的随机幅度（可能不同），h(t) 是单个载流子通过器件时产生的脉冲响应（通常是一个宽度很窄的脉冲）。

当每个载流子的幅度 Aᵢ 独立同分布且与到达过程独立时，这个模型就是复合泊松过程驱动的散弹噪声。

坎贝尔定理：散弹噪声的均值与功率谱

坎贝尔定理（Campbell's theorem）给出了散弹噪声的一阶和二阶统计量的闭合表达式。假设到达率为 λ，单个脉冲响应为 h(t)，幅度均值为 E[A]，幅度二阶矩为 E[A²]：

散弹噪声电流的均值：
        E[I(t)] = λ · E[A] · ∫ h(t) dt

        散弹噪声电流的功率谱密度：
        S_{II}(ω) = λ · E[A²] · |H(ω)|²

        其中 H(ω) 是 h(t) 的傅里叶变换。
坎贝尔定理的物理含义：散弹噪声的功率谱与脉冲响应的幅度谱的平方成正比，与到达率 λ 成正比。这意味着——脉冲越窄（H(ω) 带宽越宽），散弹噪声的频带就越宽。对于理想化的 δ 脉冲（即 h(t)=A·δ(t)），功率谱在所有频率上为常数，这就是白散弹噪声。

应用场景

散弹噪声在电子工程中有广泛的应用：

  • 光电探测器噪声：光子到达的泊松过程 + 光电转换效率，决定了探测器的信噪比下限。在弱光条件下，散弹噪声是主要噪声源，它决定了光学通信和天文观测的理论极限。
  • 晶体管散弹噪声：BJT 中基极-发射极结的载流子穿越势垒产生的电流起伏，是晶体管最主要的噪声源之一。运放输入级的设计很大程度上就是在与散弹噪声博弈。
  • 散粒噪声极限：在精密测量中，"散粒噪声极限"是一个不可逾越的物理极限——它来源于电流的离散粒子性。例如，在 MEMS 加速度计中，检测电容的充电电流也存在散弹噪声，制约着传感器的灵敏度。

踩坑故事：网络流量中的泊松迷思

讲一个经典的工程判断失误。上世纪90年代，随着互联网的快速发展，网络工程师需要为路由器设计缓冲区和带宽分配策略。他们自然想到了用泊松过程来建模数据包的到达——毕竟电话网络中的呼叫到达就很好地符合泊松过程，而"互联网不也是通信网络吗？"

于是一大批网络设备基于泊松假设进行了设计。然而当这些设备部署到真实互联网上时，出现了严重的性能问题：缓冲器频繁溢出，丢包率远超预期。经过深入研究，研究者发现了一个颠覆认知的事实——网络流量不是泊松的，它是自相似的（self-similar）或具有长程依赖性（long-range dependent）。

什么意思呢？在泊松过程下，不同时间尺度上的流量特征是一致的——1秒内的计数分布和1分钟内的计数分布由同一个 λ 参数决定。但真实网络流量呈现出一种"尺度不变"的模式：放大看（毫秒级）是突发性的，缩小看（小时级）还是突发性的。一个文件传输会产生一连串密集到达的数据包，然后是一段静默期——这完全不符合泊松过程的"完全随机"假设。

深刻教训：泊松过程假设的是"完全随机"的到达，但很多真实系统中的事件并不是完全随机的——它们由人类行为、协议机制、物理过程驱动，呈现出聚类性和相关性。在建模时，一定要先验证数据是否符合泊松假设。一个简单的检查方法：计算样本的均值与方差，如果两者不相等（泊松分布要求相等），那就说明需要更复杂的模型（如自相似过程、马尔可夫调制的泊松过程等）。

本章小结与衔接

本章我们学习了描述随机到达的核心模型——泊松过程，从它的定义（独立增量 + 平稳增量）出发，导出了计数分布、到达间隔分布、到达时间分布等一系列结果。我们还引入了泊松冲激序列和散弹噪声的概念，看到了它们与白噪声的内在联系。最后，坎贝尔定理给了我们计算散弹噪声统计特性的工具箱。

从下一章起，我们将进入信号处理领域的另一个核心分支——估计理论。如果说泊松过程是"描述随机现象"的工具，那么估计理论就是"从随机观测中提取信息"的工具。给定一堆带噪数据，你如何尽可能精确地估计出你关心的参数？MMSE、MAP、ML 这些准则分别解决什么问题？这些内容，将是我们后续维纳滤波和卡尔曼滤波的基础。下章见。
