第26章 人形与四足机器人

大家好，我是你们的讲师。如果我在2023年告诉你，三年后人形机器人将在工厂流水线上与工人并肩作业，你可能会觉得我在说科幻。但2026年的今天，这已经成为现实——波士顿动力Atlas在Hyundai工厂搬运重型冲压件，宇树H2 Plus在NVIDIA Isaac GR00T平台上实现动态平衡，Figure 02在宝马工厂累计处理了超过90,000个零部件。足式机器人，这个曾经被视为"实验室工艺品"的方向，正在以超乎所有人预期的速度冲进产业前线。

为什么是足式？因为人类的世界是为双足和四足设计的——楼梯、门槛、碎石路、狭窄通道、旋转门——轮式机器人遇到这些场景就寸步难行，而足式机器人可以像生物一样自然地通过。这一章，我们将深入足式机器人的核心技术：从双足行走的ZMP理论到四足运动的MPC控制，从全身协调控制到2026年的人形量产时代。这将是我们整个课程中最"前沿"的一章。

人形机器人的历史与现状

人形机器人（Humanoid Robot）的历史可以追溯到1973年——日本早稻田大学加藤一郎教授团队研制出了WABOT-1，这是世界上第一台全尺寸人形机器人。WABOT-1拥有肢体控制系统、视觉系统和对话系统，能够行走（虽然每步需要45秒）、抓取物体并用日语进行简单对话。以今天的标准看，它笨拙得像一台会走路的冰箱，但它的架构——感知、决策、执行分层设计——影响了此后五十年的所有足式机器人研究。

真正的突破发生在本世纪。2013年，Boston Dynamics发布了Atlas的第一版（当时还是PETMAN项目的延续），展示了双足机器人在复杂地形上的动态行走能力。2016年的"Atlas开门"和"Atlas后空翻"视频让全世界看到了人形机器人的极限可能性。但你可能不知道的是，这些令人惊叹的演示背后是一次又一次的跌倒和修复——Atlas的早期版本几乎每次测试都会摔坏，而每次修复都需要数小时的机械拆装。我2019年参观MIT CSAIL时，一位研究员告诉我："Atlas每在视频里翻一个跟头，工程师就要在机房里熬三个通宵。"这就是足式机器人研发的真实面貌——风光背后是极低的鲁棒性和极高的维护成本。

2024年是人形机器人的转折点。Figure AI发布了Figure 01，展示了一台商业级人形机器人在仓库中自主完成箱体搬运的视频。同年，宇树科技推出了首款通用人形机器人H1，以9.9万美元的定价震惊业界——这个价格不到Atlas的十分之一。进入2026年，人形机器人已经进入量产探索阶段，形成了清晰的技术分水岭：以Atlas为代表的高性能液压派、以Optimus为代表的高效电机派、以H2 Plus为代表的性价比派。

双足行走原理

双足行走（Bipedal Locomotion）是人形机器人最基础也最困难的能力。与四足和轮式不同，双足在站立时是一个"倒立摆"——重心远高于支撑点，天然不稳定。人类婴儿需要大约一年才能学会独立行走，而让人形机器人走得稳、走得自然，研究人员花了整整五十年。

ZMP稳定性判据是双足行走的基石。ZMP（Zero Moment Point，零力矩点）由南斯拉夫学者Miroslav Vukobratović在1968年提出。它的核心思想很简单：机器人脚底与地面接触的区域称为"支撑多边形"（Support Polygon），只要合力矩为零的点（即ZMP）落在这个多边形内部，机器人就处于动态稳定状态。换句话说，ZMP必须始终在脚底范围内——一旦ZMP越界，机器人就会倾倒。

LIPM（Linear Inverted Pendulum Model，线性倒立摆模型）是ZMP理论最经典的实现方式。它将机器人简化为一个质量集中点（CoM，Center of Mass）加一条无质量腿，腿的末端在ZMP位置。LIPM的美妙之处在于，它在水平方向上的动力学方程是线性的——这意味着我们可以用解析方法计算CoM轨迹：

# LIPM动力学仿真：三步走过程
import numpy as np

class LIPM:
    def __init__(self, com_height=0.85, g=9.81):
        self.z_h = com_height  # CoM高度（假设恒定）
        self.g = g
        self.w = np.sqrt(g / com_height)  # 自然频率

    def compute_dcm(self, com, com_vel, zmp):
        """Divergent Component of Motion (DCM) 计算"""
        dcm = com + com_vel / self.w
        dcm_vel = self.w * (dcm - zmp)
        return dcm, dcm_vel

    def plan_footstep(self, goal_pos, current_dcm, step_time=0.4):
        """基于Capture Point的步态规划"""
        cp = current_dcm  # Capture Point = DCM
        target_cp = goal_pos
        # 计算所需的ZMP位置使得DCM收敛到目标
        zmp_desired = (cp - target_cp * np.exp(-self.w * step_time))
        zmp_desired /= (1 - np.exp(-self.w * step_time))
        return zmp_desired

    def simulate_step(self, com, com_vel, zmp, dt=0.001):
        """单步积分：使用LIPM动力学方程"""
        com_acc = (self.w ** 2) * (com - zmp)
        com_vel_new = com_vel + com_acc * dt
        com_new = com + com_vel * dt
        return com_new, com_vel_new

上面的代码实现了LIPM的核心逻辑。注意compute_dcm函数中的DCM（Divergent Component of Motion）——这是现代双足控制中的关键概念。DCM是CoM状态的一个线性组合，它描述了机器人的"发散程度"：如果DCM在ZMP前方，机器人就会向前走；如果DCM在ZMP上，机器人就会静止。实际控制中，我们通过规划DCM轨迹来生成稳定的CoM运动，再通过逆动力学求解关节力矩。

Push Recovery（推扰恢复）是双足行走中最令人着迷的话题。想象一下，你正在走路，有人从侧面推了你一把——你会本能地做出反应：小推力时调整脚踝力矩，中推力时跨出一步，大推力时连续跨步。人形机器人的推扰恢复也遵循同样的三阶段策略：第一步是踝关节力矩补偿（Ankle Strategy），利用脚底的力矩传感器检测扰动并施加反向力矩；如果扰动超出了踝关节的能力范围，第二步是髋关节策略（Hip Strategy），通过上身摆动调整CoM位置；当扰动足够大时，第三步是跨步策略（Step Strategy），重新规划落脚点。

工程技巧：在实际调试中，我建议先用仿真测试推扰恢复算法。在MuJoCo中给机器人施加不同方向和幅度的推力（从50N到500N），观察各阶段策略的触发阈值。记录每次失败的原因——大多数情况下，问题出在足底打滑模型的准确性上。真实地面摩擦系数的不确定性是推扰恢复从仿真到真机最大的鸿沟。

四足运动控制

四足机器人（Quadruped Robot）在稳定性上比双足有天然优势——四条腿提供了更大的支撑多边形，这使得四足机器人在2026年已经率先实现了商业落地。Boston Dynamics Spot在全球部署了超过2000台，用于石油天然气巡检、建筑工地监控和公共安全。宇树B2（企业级四足机器人）在教育科研和安防巡检领域占据了显著市场份额。

四足运动的核心是步态（Gait）规划。不同的步态在速度、稳定性和能耗之间做出权衡：

  • Walk（慢走）：始终有三条腿着地，一条腿摆动。稳定裕度最大，速度最慢（~0.5 m/s）。适用于复杂地形和精确导航。
  • Trot（小跑）：对角线两条腿同时摆动（左前+右后，然后右前+左后）。速度和稳定性的最佳平衡（~1.5 m/s）。MIT Cheetah和Spot的主力步态。
  • Pace（同侧步）：同侧两条腿同时摆动。速度快但有侧向摆动，适合平坦地面。
  • Gallop（飞奔）：所有腿交替腾空。最高速度（MIT Cheetah达到12.7 m/s），但对结构冲击极大。

现代四足控制的主流方法是MPC（Model Predictive Control，模型预测控制）。MIT Cheetah团队的开创性工作展示了一种基于MPC的全身控制架构：在每个控制周期（约3ms），MPC求解一个有限时域优化问题，计算机器人未来0.5秒内的最优地面反力和关节轨迹。这个优化问题的目标函数包含CoM轨迹跟踪误差、足端接触力平滑性和关节力矩约束：

# MPC-based locomotion 伪代码框架
class QuadrupedMPC:
    def __init__(self, horizon=20, dt=0.025):

        self.dt = dt            # 每步时间间隔

            'stance': [0.0, 0.5],    # 支撑相占空比50%
            'swing': [0.5, 1.0]      # 摆动相占空比50%
        }

    def setup_ocp(self, com_ref, foot_ref):
        """构建MPC优化问题（凸QP形式）"""

        """MPC单步控制循环"""
        for k in range(self.N):

            forces = self.qp_weights @ state
            # 预测下一时刻状态

        return forces[:, 0]

除了MPC，Raibert Heuristics（Raibert启发式规则）仍然是四足控制的重要补充。1986年，Marc Raibert（Boston Dynamics创始人）在他的经典著作《Legged Robots That Balance》中总结了三条经验规则：1) 前向速度通过身体倾斜角度控制；2) 机身高度通过腿的长度控制；3) 转向通过左右腿的差动触地控制。这些规则虽然简单，但在2026年的四足控制器中仍然作为"安全兜底"层存在——当MPC求解失败或超出约束时，Raibert规则提供退路。

踩坑实录：2024年我给一台宇树B2做四足控制器适配时，遇到了一个极其隐蔽的问题。MPC在仿真中工作完美，步态切换平滑，速度跟踪误差小于5%。但到了真机测试时，第一条腿刚离地机器人就侧翻了。排查三天后发现原因：仿真中的摩擦系数设为0.8（橡胶-混凝土），而实际测试地面是环氧地坪（摩擦系数0.3）。MPC计算出的接触力在低摩擦地面上超出了摩擦锥约束，导致足端打滑。教训：摩擦力是四足控制中最容易被低估的边界条件。永远以实际测量值为准。

全身控制（WBC）

全身控制（Whole-Body Control, WBC）解决的核心问题是：当机器人有几十个自由度（如H2 Plus的31个DOF），而任务目标远少于自由度数量时，如何协调所有关节共同完成任务？答案是建立一个任务层级（Task Hierarchy），按照优先级排列所有控制目标：

  • P1 - CoM平衡：保持重心在支撑多边形内，防止倾倒。这是最底层的安全约束。
  • P2 - 姿态控制：保持躯干水平或按需倾斜。为上层任务提供稳定的基座。
  • P3 - 末端操作：如机械臂末端跟随目标轨迹、足端落点控制。
  • P4 - 头部/视觉：保持目标在视野中心，为感知系统提供服务。
  • P5 - 次要目标：关节限位回避、奇异姿态回避、能耗优化等。

WBC的数学本质是一个多优先级QP（Quadratic Programming，二次规划）问题。每个层级的任务被转化为一个等式或不等式约束，通过零空间投影（Null-Space Projection）确保高优先级任务的解不受低优先级任务影响。具体来说：首先在完整自由度空间求解P1任务，得到解q1；然后将P2任务投影到P1的零空间中进行求解，这样P2的控制量不会影响P1的输出；依此类推。

实际工程中，WBC的计算需要在1-2ms内完成。常用的实时QP求解器包括：qpOASES（基于积极集法，适合中小规模问题）、OSQP（基于ADMM算法，适合大规模问题）、以及各大机器人公司自研的定制求解器。NVIDIA的Isaac Lab框架为WBC提供了GPU加速的全套工具链，使得31-DOF人形机器人的全身控制可以在Jetson Thor上以1kHz稳定运行。

学习建议：WBC是机器人控制中数学最密集的领域之一。如果你想深入理解，我建议按以下顺序阅读：先读《Modern Robotics》第3章关于任务空间控制的介绍，再看Luis Sentis的博士论文（2007年，斯坦福），最后跑一遍Pinocchio或RBDL中关于多优先级控制的开源示例。不要试图一次性理解所有数学——先从QP的几何直觉入手，再补矩阵论细节。

商业化挑战

尽管技术突破令人振奋，人形机器人的商业化仍然面临严峻挑战。我带你逐一审视这些"拦路虎"。

成本问题是最直接的障碍。目前一台商业级人形机器人的BOM成本在$10K到$150K+之间，核心成本集中在三个方面：高扭矩密度电机（占35%）、精密减速器（占25%）、算力平台（占15%）。只有当整机售价降至$10K以下，人形机器人才可能进入消费市场。以当前的成本下降曲线推算，这个时间点在2028-2030年左右。

可靠性（MTBF）是比成本更隐蔽的挑战。工业机器人的MTBF（Mean Time Between Failures）通常要求超过50000小时，而2026年人形机器人的典型MTBF仅为500-1000小时——差了两个数量级。主要故障点包括：关节电机过热、减速器磨损、线缆疲劳断裂、传感器漂移。每500小时一次的维护需求对商业部署来说是不可接受的。

安全认证方面，人形机器人面临一个尴尬的局面：现有的ISO 10218（工业机器人安全标准）和ISO 13482（服务机器人安全标准）都不完全适用于动态行走的人形机器人。当一台150公斤的机器人以1.5m/s的速度在工厂里行走时，如何保证人类工人的安全？2026年，ISO TC 299正在制定针对人形机器人的专项安全标准（预计2028年发布），在此之前，企业在部署时只能依赖更保守的安全策略——物理围栏、速度限制和力限制。

续航问题是人形机器人的阿喀琉斯之踵。以H2 Plus为例，搭载2.3kWh电池组，轻载行走续航约2小时，重载操作续航仅45分钟。相比之下，人类工人一个班次8小时。电池能量密度的提升速度（每年约5-8%）远不能支撑人形机器人全天候工作。换电站方案（如每台机器人配2-3块电池）是当前最务实的折中方案。

我的判断：我经常被问到"人形机器人什么时候能进家门"。我的看法是：2026-2028年是工业落地期——在结构化场景（工厂、仓库、实验室）中完成从"可展示"到"可用"的跨越；2028-2031年是商业服务期——在商场、酒店、医院等半结构化场景中出现；2032年以后才可能进入家庭。这个时间线取决于三个因素：成本下降速度、AI能力的通用性、以及安全法规的完善度。不要被媒体的乐观报道牵着走——保持理性，但保持信心。
