开篇：数据市场的"超市货架"

大家好，我是你们的讲师。上一章我们讨论了一个核心理念：数据质量决定模型能力的上限。那么，当我们要开始训练一个大模型时，第一个实际问题就摆在了面前：数据从哪里来？

早年间，做预训练相对简单——去 Common Crawl 下载一份快照，跑一遍清洗流水线，数据就准备好了。但到了 2025-2026 年的今天，事情已经完全不同了。行业里涌现出了数十个高质量的公开数据集，每个都有自己的定位、规模、清洗标准和许可证限制。选对数据集组合，训练能事半功倍；选错了，轻则白费算力，重则惹上官司。

打个比方：这就像去逛一个大型超市。货架上琳琅满目，每个品牌都有自己的定位——有的是"全天然有机食品"（如 RedPajama），有的是"量大管饱的经济装"（如 C4），还有的是"进口精品超市专供"（如 The Pile）。作为"大厨"，你需要了解每种食材的特性，才能搭配出一桌好菜。本章我们就来一一盘点这些"食材"。

2.1 RedPajama-Data-1T：LLaMA 的开源复现

2023 年，Together Computer 联合多个研究机构发布了 RedPajama-Data-1T，这是一个试图完整复现 LLaMA 训练数据配比的开源项目。LLaMA 论文中描述了其数据来源和配比，但实际数据并未公开。RedPajama 的使命就是：按照 LLaMA 论文的配方，做出一个开源版本的数据集。

RedPajama 的组成如下：

数据源 规模 来源说明 占比
CommonCrawl ~800B tokens 经过清洗的网页数据，5次 CC 快照 67%
C4 ~175B tokens 与 LLaMA 相同的 C4 子集 15%
GitHub ~75B tokens 开源代码（去重后） 4.5%
Wikipedia ~12B tokens 多语言 Wikipedia 数据 4.5%
Books ~50B tokens Project Gutenberg 等公版书籍 4.5%
ArXiv ~25B tokens 学术论文（LaTeX 源码） 2.5%
StackExchange ~20B tokens 问答社区内容 2.0%

这个数据集的最大贡献在于：它打破了"大公司专属"的数据配方壁垒。在 RedPajama 出现之前，只有 Google、Meta、OpenAI 这样的公司才知道它们的模型具体用了什么数据。RedPajama 让学术界和小团队也能用上接近工业级水准的预训练数据。

RedPajama 采用了 Apache 2.0 许可证，这意味着它可以被自由地用于商业用途。这在美国出口管制趋严的大背景下，对中文开发者尤其友好。

实用建议：RedPajama 是目前最推荐的"起步数据集"。如果你在训练一个 1B-7B 级别的模型，直接使用 RedPajama 作为主要数据源，再按需补充中文或垂直领域数据，是一个稳妥且高效的策略。

2.2 SlimPajama-627B：更精简，更高效

SlimPajama 是 RedPajama 的"瘦身版"。这个名字起得很贴切——它把 RedPajama 的 1.2T token 去重压缩到了 627B token，几乎砍掉了一半。

具体是怎么做到的？核心是两个字：去重。RedPajama 在构建过程中虽然也做了基础的去重，但并不彻底。SlimPajama 团队使用了更严格的去重策略，包括：

  • 精确去重：移除完全相同的文档
  • 模糊去重：基于 MinHash + LSH 的 n-gram 级别的近似去重
  • 跨数据集去重：不仅在一个数据源内去重，还跨 Wikipedia、Books、C4 等多个数据源去重

结果令人惊讶：去掉了近一半的数据后，模型性能并没有显著下降。事实上，在某些任务上，使用 SlimPajama 训练的模型甚至略优于使用完整 RedPajama 的版本。这说明被去掉的那 50% 的数据，大部分是冗余信息。

SlimPajama vs RedPajama 效果对比（基于 1.3B 参数模型）：
  ┌─────────────────────┬───────────┬───────────┬──────────┐
  │ Benchmark           │ RedPajama │ SlimPajama│ 差异     │
  ├─────────────────────┼───────────┼───────────┼──────────┤
  │ LAMBADA (ppl)       │   3.13    │   3.11    │ -0.02 ▼  │
  │ HellaSwag (acc)     │   53.4%   │   54.1%   │ +0.7% ▲  │
  │ ARC-Challenge (acc) │   28.6%   │   28.9%   │ +0.3% ▲  │
  │ PIQA (acc)          │   71.2%   │   71.5%   │ +0.3% ▲  │
  │ Training time       │   100%    │    53%    │ -47% ▼   │
  └─────────────────────┴───────────┴───────────┴──────────┘

这是一个典型的"少即是多"的案例。更少的训练数据意味着更短的训练时间、更低的成本，而模型效果不仅没有打折，反而略有提升。这背后的逻辑是：去重本质上提升了数据的信噪比——模型不再花容量去记忆重复的模式，而是专注于学习多样化的知识。

启示：不要被"大"数据集迷惑。SlimPajama 告诉我们，精心去重后的 627B token 比原始 1.2T token 更有价值。在数据工程中，"更少但更精"往往是更好的策略。

2.3 Dolma：全透明的 3T 语料库

2024 年，AI2（Allen Institute for AI）发布了 Dolma 数据集，作为其开源 OLMo 模型的训练语料。Dolma 的规模达到了 3T token，是目前公开的最大规模的预训练数据集之一。

Dolma 最突出的特点是 全透明。AI2 不仅公开了数据本身，还公开了：

  • 完整的数据处理代码（包括清洗、去重、过滤的每一个步骤）
  • 数据源的详细组成和配比
  • 每个处理步骤对数据规模的影响（从原始到最终的缩减比例）
  • 许可证审计报告——对每个子集的许可条款做了详细的合规评估

Dolma 的组成包括六个主要来源：Common Crawl（经过 AI2 自己的清洗流水线处理）、The Stack 代码数据、C4、Reddit 社交内容、Wikipedia 和书籍。其中 Common Crawl 部分占比约 50%，但经过清洗后实际上只贡献了约 1.5T token——从 38T 的原始 CC 数据压缩到了 1.5T，缩减了 96%。

学习建议：如果你想深入理解"数据清洗流水线到底该怎么做"，Dolma 的代码仓库是最好的教科书。它比很多论文都详细地记录了每一步的决策依据和效果。

2.4 The Pile：学术界的"黄金标准"

The Pile 由 EleutherAI 在 2021 年发布，是学术预训练研究的奠基性数据集。虽然它的规模（825 GiB）在今天看来不算最大，但它对学术界的影响力无人能及——GPT-NeoX、GPT-J、Pythia 等一系列著名开源模型都基于 The Pile 训练。

The Pile 的最大特点是包含了 22 个子集，覆盖了从学术论文到小说、从法律文书到医学文献的广泛内容：

类别 子集举例 规模 许可证
学术 ArXiv, PubMed Central, PubMed Abstracts ~200 GiB 混合（各论文单独授权）
互联网文本 CommonCrawl, OpenWebText2 ~230 GiB 公共领域
代码 GitHub (去重) ~95 GiB 混合
书籍 Project Gutenberg, Bibliotik ~120 GiB Bibliotik 有严格限制
对话 StackExchange, Ubuntu IRC, HackerNews ~80 GiB CC BY-SA
其他 Wikipedia, DM Mathematics, PhilPapers ~100 GiB 混合
重要提醒：The Pile 的许可证状态比较复杂。虽然数据集整体采用了 MIT 许可证发布，但其部分子集（尤其是 Bibliotik 的书籍数据）有严格的非商业使用限制。如果用于商业模型训练，务必仔细检查每个子集的许可条款。正是这个问题，推动了后来的 Dolma 和 RedPajama 项目更加注重许可证审计。

The Pile 的一个不太为人知的故事是：它最初的书籍数据来自一个名为 Bibliotik 的私人追踪器，这引发了持续的版权争议。2023 年后，越来越多的研究和项目开始避开这些有争议的数据源，转而使用公版书籍或明确授权的文本。这件事情给数据工程行业敲响了警钟——许可证合规不是法务部门的专属工作，而是数据工程师必须掌握的技能。

2.5 Falcon RefinedWeb：工业级的网页清洗

Falcon RefinedWeb 是由 TII（阿布扎比技术创新研究所）为其 Falcon 系列模型构建的数据集。它专注于一件事：把 Common Crawl 清洗到极致。Falcon RefinedWeb 最终产出约 900B token，但它的清洗流程才是真正的宝藏。

RefinedWeb 的清洗管线包含以下核心步骤：

  • URL 级过滤：屏蔽已知的成人内容、垃圾网站、占位符页面
  • 语言识别：fastText 语言分类器，保留高质量英语内容
  • 质量过滤：基于启发式规则（标点密度、单词长度分布、特殊字符比例等）
  • 精确去重：URL 去重 + 文档级 SHA-1 去重
  • 模糊去重：基于 MinHash 的近似去重

Falcon RefinedWeb 的开源实践 (Apache 2.0) 证明了纯网页数据经过精心清洗，可以达到接近 curated 数据的质量水平。这极大地降低了预训练数据获取的门槛——你不需要昂贵的书籍版权或学术数据库授权，只要有 CC 快照和一套好的清洗流程，就能产出高质量的基座数据。

2.6 C4 / mC4：经典虽老，廉颇善饭

C4（Colossal Clean Crawled Corpus）由 Google 在 2019 年随 T5 论文发布，是预训练数据工程史上的里程碑。虽然经过了这么多年，C4 仍然是学术界和工业界最广泛使用的基线数据集之一。

C4 的核心特点：

  • 约 800 GiB 的纯英文文本
  • 清洗规则明确且可复现：只保留以标点结尾的句子、移除页面包含"lorem ipsum"等占位文本、过滤掉包含 JavaScript 或源代码的页面
  • 基于 Common Crawl 2019 年的月度快照

mC4 是 C4 的多语言版本，涵盖了 100+ 种语言，总计约 40T token。其中中文部分约有 1.5T token——虽然质量参差不齐，但作为一个免费的、可以快速获取的中文多语言语料来源，mC4 仍然有其价值。

C4 的清洗规则（简版实现）：
  def is_valid_document(text):
      # 1. 必须包含至少 3 个句子（以句号/感叹号/问号结尾）
      if sentence_count(text) < 3: return False

      # 2. 标点符号占比在合理范围内
      if punctuation_ratio(text) < 0.1: return False

      # 3. 不包含"lorem ipsum"等占位文本
      if contains_placeholder(text): return False

      # 4. 不包含 JavaScript 或编程代码特征
      if looks_like_code(text): return False

      # 5. 单词长度分布合理
      if outlier_word_length(text): return False

      return True
现实权衡：C4 的清洗规则在今天看来有些"粗糙"，但它胜在简单、可复现、可扩展。很多工业级流水线都是在 C4 规则的基础上做增量改进。如果你刚开始构建数据流水线，从 C4 规则出发并逐步迭代，比一开始就设计复杂的 ML-based 过滤方案要务实得多。

2.7 数据集对比与选型建议

最后，我们来做一个全景式的对比，帮助你根据实际需求选择合适的数据集：

数据集 规模 多语言 许可证 清洗质量 最佳用途
RedPajama-1T 1.2T tokens 弱（以英文为主） Apache 2.0 ✅ ⭐⭐⭐⭐ 通用英文模型基座
SlimPajama 627B tokens 弱 Apache 2.0 ✅ ⭐⭐⭐⭐⭐ 追求训练效率
Dolma 3T tokens 弱 ImpACT ✅ ⭐⭐⭐⭐⭐ 大型模型训练底座
The Pile 825 GiB 弱 MIT（部分子集受限）⚠️ ⭐⭐⭐ 学术研究、基线比较
Falcon RefinedWeb ~900B tokens 弱 Apache 2.0 ✅ ⭐⭐⭐⭐⭐ 纯网页数据、清洗参考
C4 ~800 GiB 仅英文 ODC-BY ⭐⭐⭐ 基线实验、快速原型
mC4 ~40T tokens 100+ 语言 ODC-BY ⭐⭐ 多语言预训练基线

选择数据集的决策流程可以这样概括：

  • 起步阶段（1B-3B 参数）：直接使用 SlimPajama 或 RedPajama，快速验证训练流程
  • 中型模型（7B-13B 参数）：Dolma + 补充中文/领域数据
  • 大型模型（30B+ 参数）：需要自建清洗流水线，以 Raw Common Crawl 为主要源，参考 RefinedWeb 的流程
  • 学术研究：用 The Pile 作为基准方便与论文对比
实战提示：不要在起跑线上追求完美。我的建议是：先拿 C4 或 SlimPajama 跑通训练流程，再逐步替换和增加数据源。很多团队花费数月搭建"完美"的数据流水线，结果发现模型架构还在频繁调整——数据被改来改去，浪费了大量精力。数据工程需要迭代思维，不能贪求一步到位。

本章小结

本章我们走马观花地看了全球最主流的基座数据集。从 RedPajama 到 Dolma，从 The Pile 到 Falcon RefinedWeb，每个数据集都有自己的设计哲学和适用场景。核心要点是：没有"万能的"数据集，只有"最适合你当前阶段"的数据集。理解每个数据集的来源、清洗策略和许可证约束，是做出正确选型决策的基础。

但你可能注意到了：上述所有数据集都以英文为主体。中文预训练数据的情况要复杂得多——这正是我们下一章要深入探讨的主题。我是你们的讲师，我们第 3 章见。
