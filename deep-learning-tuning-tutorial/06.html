初始化——"一次惨痛的教训是acc只能到70%，改成Xavier后acc到98%"

大家好，我是你们的讲师。知乎上有一条很火的回答："一次惨痛的教训是用初始化cnn的参数，最后acc只能到70%多，仅仅改成xavier，acc可以到98%。"——这不是段子，这是真实发生的。初始化方法选错了，后面再怎么调LR、换优化器、加数据，都像是在用错了起跑线的赛道上比赛。

我曾经帮一个朋友调试一个图像分类模型。他的ResNet50在CIFAR-10上准确率卡在73%，调了两周没进展。我仔细看了他的代码，问题出在初始化上——他用的框架默认初始化是某一种均匀分布，但和激活函数不匹配。改成了He初始化，只跑了100个epoch，准确率直接跳到91%——没动任何其他超参数。

初始化就是训练的起跑线——站错了位置，跑得再快也到不了终点。

为什么要初始化

有两个最基本的问题，必须用好的初始化来解决：

问题一：对称性破坏。如果所有参数初始化为0，会发生什么？在多层网络中，同一层内的所有神经元会学到完全相同的特征。因为所有神经元接收相同的输入、计算相同的梯度、做相同的更新。无论网络多宽，都退化为一个神经元在干活。这就是为什么参数必须随机初始化——破坏对称性，让不同的神经元学到不同的特征。

但注意：bias永远初始化为0。bias的对称性不会造成问题，因为不同神经元的权重已经是随机的了。如果bias也随机，相当于给每个神经元加了一个随机偏移，没有意义还会增加优化难度。

问题二：梯度消失/爆炸的数学根源。考虑一个没有激活函数的深度线性网络：每一层输出 y = Wx，经过L层后，输出 y_L = W_L·W_{L-1}·...·W_1·x。如果权重矩阵的奇异值大于1，经过多层乘积后输出会指数级增长（梯度爆炸）；如果奇异值小于1，输出会指数级衰减到0（梯度消失）。

这个直觉同样适用于带激活函数的深度网络。如果权重的方差太大，激活值的方差会逐层膨胀，最终饱和（sigmoid/tanh）或爆炸（ReLU）。如果方差太小，激活值的方差逐层收缩，最终消失——深层网络的参数几乎不更新。好的初始化就是要在前向和反向传播中保持方差不膨胀也不收缩。

调试提示：如果你训练时发现loss纹丝不动，可以打印每一层的激活值方差。如果第一层的方差是1.0，第10层的方差已经变成了0.001——那就是初始化导致的梯度消失。反过来，方差从1.0膨胀到1000——那是梯度爆炸。好的初始化应该让各层的激活值方差基本一致。

Xavier初始化的数学推导

Xavier初始化（也称Glorot初始化）是2010年由Xavier Glorot和Yoshua Bengio提出的。它的核心思想是：前向传播时各层激活值的方差保持一致，反向传播时各层梯度的方差保持一致。

推导开始：假设输入x和权重W独立同分布，均值都是0。对线性层 y = Wx：

Var(y) = n_in · Var(W) · Var(x)

其中n_in是输入神经元的数量。要保持Var(y) = Var(x)（前向方差不变），需要：

Var(W) = 1 / n_in

反向传播同理，要保持梯度方差不变，需要：

Var(W) = 1 / n_out

但是n_in和n_out通常不相等。Xavier的解决方案是取两者的调和平均：

Var(W) = 2 / (n_in + n_out)

对应两种分布的实现：

  • 均匀分布：W ~ Uniform(-√(6/(n_in+n_out)), √(6/(n_in+n_out)))
  • 正态分布：W ~ N(0, √(2/(n_in+n_out)))

均匀分布的边界√(6/(n_in+n_out))来源于方差公式：均匀分布U(-a, a)的方差为a²/3，令a²/3 = 2/(n_in+n_out)，解得a = √(6/(n_in+n_out))。

Xavier初始化适用的条件：激活函数在0附近近似线性且关于0对称。满足这两个条件的典型激活函数是tanh，sigmoid近似（但sigmoid在0附近不对称，效果不如tanh好）。

He初始化的ReLU修正

ReLU激活函数的出现打破了Xavier初始化的假设。ReLU在负半轴输出恒为0——这导致经过ReLU后大约一半的方差被丢弃了。

推导如下：

  • Xavier假设Var(y) = n_in · Var(W) · Var(x)
  • 经过ReLU后：x' = max(0, y)。因为y均值0且对称，Var(x') ≈ (1/2) · Var(y)
  • 要保持Var(x') = Var(x)，需要补偿回这个1/2因子
  • 所以：Var(W) = 2 / n_in

这就是He初始化（Kaiming初始化）的核心公式。对比Xavier的 2/(n_in+n_out)，He初始化有两个不同：

  • 分子为2而非调和平均：因为ReLU砍掉一半方差，需要翻倍补偿
  • 分母只用n_in：He的推导选择只保证前向稳定

对于LeakyReLU（负斜率a=0.01），公式修正为：

Var(W) = 2 / ((1 + a²) · n_in)

两种分布实现：

  • 均匀分布：W ~ Uniform(-√(6/n_in), √(6/n_in))
  • 正态分布：W ~ N(0, √(2/n_in))

PyTorch的nn.Linear和nn.Conv2d默认使用Kaiming Uniform初始化（nonlinearity='leaky_relu'），会自动根据激活函数调整缩放因子。

关键对比：Xavier和He的选择完全由激活函数决定。tanh/sigmoid → Xavier（Glorot）。ReLU/LeakyReLU/PReLU → He（Kaiming）。选错了，轻则收敛变慢，重则完全不收敛。在知乎上有位答主分享的案例：用错初始化，acc从98%掉到70%。这28个百分点的差距，就是一行代码的区别。

不同网络层的初始化策略

不同的网络层类型需要不同的初始化策略。下面是详细的推荐方案：

层类型 推荐初始化方法 标准差/缩放 PyTorch API
全连接层（tanh/sigmoid） Xavier Uniform √(6/(n_in+n_out)) xavier_uniform_
全连接层（ReLU） He Normal √(2/n_in) kaiming_normal_
卷积层（ReLU） He Normal √(2/(k²·c_in)) kaiming_normal_
卷积层（tanh） Xavier Uniform √(6/(k²·c_in + k²·c_out)) xavier_uniform_
Embedding 截断正态分布 σ=0.02 trunc_normal_
RNN/LSTM 正交初始化（SVD） 正交矩阵 orthogonal_
LLM/大模型预训练 正态分布（固定标准差） σ=0.06（DeepSeek） normal_(std=0.06)
Bias（任何层） 零初始化 0 zeros_

上面的一些策略值得展开说明：

Embedding层用截断正态分布。截断正态分布防止了极端值出现（超出2σ的值被截断重采样），保证了词嵌入向量的初始分布既随机又不过于离散。标准差一般取0.02左右。在PyTorch中用法：nn.init.trunc_normal_(embedding.weight, std=0.02)。

RNN/LSTM用正交初始化。正交初始化通过对随机矩阵做SVD分解得到正交权重矩阵。正交矩阵满足 WᵀW = I，这意味着输入信号的范数在层间传递时保持不变——对缓解RNN的梯度消失/爆炸非常有效。PyTorch中：nn.init.orthogonal_(rnn.weight_hh_l0)。

大模型用固定标准差。DeepSeek的实验表明，在1e17~2e19 FLOPs的计算规模下，参数初始化的标准差固定为0.06，不需要随计算量调整。这个经验值在大模型社区中被广泛接受。

特殊情况：在使用预训练模型微调时，不要手动执行初始化。加载预训练权重后，所有参数已经具有了良好的初始状态。如果再执行初始化，相当于把预训练权重全扔了，会造成灾难性遗忘。但有一种特殊情况：新增的层（如分类头）需要初始化——通常用Xavier/He Uniform即可，bias初始化为0。

PyTorch初始化代码大全

下面是一份完整的PyTorch初始化参考代码，包含了所有主流方案的调用方式：

import torch
        import torch.nn as nn
        import torch.nn.init as init

        class MyModel(nn.Module):
            def __init__(self):
                super().__init__()
                self.conv1 = nn.Conv2d(3, 64, 3)
                self.fc1 = nn.Linear(1024, 512)
                self.embed = nn.Embedding(10000, 256)
                self.lstm = nn.LSTM(256, 128, num_layers=2)
                self.fc_out = nn.Linear(128, 10)

                self._init_weights()

            def _init_weights(self):
                # Conv层 + ReLU → He Normal
                init.kaiming_normal_(self.conv1.weight, mode='fan_in',
                                     nonlinearity='relu')
                init.zeros_(self.conv1.bias)

                # 全连接层 + ReLU → He Uniform
                init.kaiming_uniform_(self.fc1.weight, mode='fan_in',
                                      nonlinearity='relu')
                init.zeros_(self.fc1.bias)

                # Embedding层 → 截断正态分布
                init.trunc_normal_(self.embed.weight, std=0.02)

                # LSTM → 正交初始化（隐藏层权重）
                for name, param in self.lstm.named_parameters():
                    if 'weight_hh' in name:
                        init.orthogonal_(param)
                    elif 'weight_ih' in name:
                        init.kaiming_uniform_(param, nonlinearity='tanh')
                    elif 'bias' in name:
                        init.zeros_(param)

                # 输出层（tanh/sigmoid）→ Xavier
                init.xavier_uniform_(self.fc_out.weight)
                init.zeros_(self.fc_out.bias)

        # 或者用apply方法对模型统一初始化
        def init_weights(module):
            if isinstance(module, nn.Linear):
                if module.out_features == 10:  # 输出层用Xavier
                    init.xavier_uniform_(module.weight)
                else:
                    init.kaiming_normal_(module.weight, nonlinearity='relu')
                init.zeros_(module.bias)
            elif isinstance(module, nn.Conv2d):
                init.kaiming_normal_(module.weight, mode='fan_in',
                                     nonlinearity='relu')
                if module.bias is not None:
                    init.zeros_(module.bias)
            elif isinstance(module, nn.Embedding):
                init.trunc_normal_(module.weight, std=0.02)

        model = MyModel()
        model.apply(init_weights)  # 递归应用初始化函数
记住一个原则："初始化足够好，超参都不用调，直接接近最优解。初始化没选对，结果跟模型有bug一样，不忍直视。"——来自知乎高赞答主。当你发现模型训练困难、loss不降时，先检查初始化方法是否与激活函数匹配，再怀疑其他问题。

这一章我们理解了初始化为什么重要——破坏对称性、防止梯度消失/爆炸。我们推导了Xavier初始化的方差保持公式（Var(W) = 2/(n_in+n_out)）和He初始化对ReLU的修正（Var(W) = 2/n_in）。我们整理了不同网络层的初始化策略表——Conv+ReLU用He，全连接+tanh用Xavier，Embedding用截断正态，RNN用正交初始化，大模型用固定标准差0.06。你也看到了完整的PyTorch初始化代码。
