第十一章：预训练词向量

好，同学们，欢迎来到第十一章。在之前的章节中，我们学习了 Word2Vec、GloVe 这些经典的词向量，它们在很多任务上表现优异。但今天我要抛出一个问题：你觉得 "苹果" 这个词，在 "我爱吃苹果" 和 "苹果发布了新手机" 这两句话里，是同一个意思吗？

你当然知道它们不同——前一个是水果，后一个是品牌。但如果我们用一个静态的 Word2Vec 向量来表示 "苹果"，它就只能有一个固定的向量。无论上下文如何变化，它都是同一个向量，这就尴尬了。静态词向量的这个核心缺陷——无法处理一词多义——正是本章要讨论的起点。

💡 本章核心问题：静态词向量给了每个词一个"固定身份证"，但真实语言中，一个词在不同的句子里扮演着完全不同的角色。我们需要的是一种能根据上下文动态调整的词表示——这就是上下文相关的词向量（Contextual Embeddings）诞生的原因。

11.1 一词多义的困境：静态向量不够用了

让我们先花点时间，把问题彻底讲透。假设我们用 Word2Vec 训练了一个向量空间。"bank" 这个词在语料库里既出现在 "river bank"（河岸）中，也出现在 "money bank"（银行）中。Word2Vec 会怎么处理？它会取一个折中：最终学到的向量既有点像"河岸"又有点像"银行"，结果在任何一个具体语境中都"差点意思"。

你再想想中文里的 "意思" 这个词——"你是什么意思"、"小意思而已"、"有意思"、"这个字的意思"——这么多用法，静态向量一个向量全包了，他能包得过来么？当然不行。

这就是所谓的一词多义问题。在 2017 年之前，整个 NLP 社区实际上都在忍受这个局限。大家心知肚明 Word2Vec 不够好，但也没有更好的替代方案。当时也有一些补救尝试，比如训练多个 sense 的向量（每个词义一个向量），但这些方法要么需要人工标注词义，要么聚类效果不稳定，始终没能成为主流。

问题的根本在于：词的表示不应该是一个查表操作，而应该是一个函数，输入是词的上下文，输出是这个词在当前语境下的表示。这个思路听起来很自然，但它一直到 2017~2018 年才被真正实现。

11.2 ELMo 架构：让词向量"活"起来

2018 年，Peters 等人提出了 ELMo（Embeddings from Language Models），在 NLP 领域引发了一场不小的地震。ELMo 的核心思想非常直观：既然我们需要上下文感知的词表示，那就训练一个双向语言模型（biLM），然后用语言模型的中间层表示作为词向量。

ELMo 使用的是双向 LSTM，而不是单向的。为什么呢？因为一个词的意义既依赖于它左边的词，也依赖于它右边的词。比如在 "他打开了苹果___" 这句话里，我们需要看完后面的词才能确定 "苹果" 的意思——"他打开了苹果电脑" vs "他打开了苹果包装盒"。

ELMo 的双向语言模型由两部分组成：

  • 前向 LSTM：从左到右读取文本，预测下一个词。
  • 后向 LSTM：从右到左读取文本，预测上一个词。
  • 两个方向独立训练，但共享词嵌入层和 softmax 输出层。

更关键的是，ELMo 不只是用最后一层的输出，而是把每一层的表示都保留下来，然后学习一个加权组合。为什么呢？因为不同的层捕获了不同粒度的信息：底层更接近词法特征（这个词是什么词性？），中层更接近句法特征（这个词在句子中扮演什么角色？），高层更接近语义特征（这个词在这个语境下是什么意思？）。把多层特征融合起来，就能得到一个更丰富的表示。

🔍 小知识：ELMo 名字里的 "E" 代表 Embeddings，"L" 代表 Language，"Mo" 代表 Model。合起来就是 "从语言模型学到的嵌入"。ELMo 的论文标题是 "Deep contextualized word representations"，这个 "Deep contextualized" 准确地概括了其贡献——深层的、上下文相关的词表示。

ELMo 的完整结构

ELMo 的整个处理流程可以概括为以下几步：首先，输入的每个词通过一个字符 CNN 层得到初始的词嵌入——没错，它用的是字符级 CNN 而不是查找表，这能处理 OOV（未登录词）问题。然后，这个嵌入进入一个 J 层的双向 LSTM。每一层都会产生一个前向表示和一个后向表示。最后，对于每个词，我们得到总共 2J+1 个表示（1 个初始嵌入 + J 层 x 2 个方向）。

然后怎么办呢？ELMo 学一个 softmax 归一化的权重，把这些表示加权求和，得到一个最终的自定义表示。这个加权机制允许不同的下游任务选择不同的层组合——比如词性标注任务可能会给底层更大的权重，而语义推理任务可能会给高层更大的权重。

# ELMo 向量生成的简化示意（伪代码风格）
import torch
import torch.nn as nn

class ELMoBiLM(nn.Module):
    def __init__(self, vocab_size, char_dim=512, lstm_dim=4096, num_layers=2):
        super().__init__()
        # 字符级 CNN 生成初始词嵌入
        self.char_cnn = CharCNN(char_dim=char_dim)
        # 双向 LSTM 层
        self.forward_lstms = nn.ModuleList([
            nn.LSTM(char_dim, lstm_dim, batch_first=True)
            for _ in range(num_layers)
        ])
        self.backward_lstms = nn.ModuleList([
            nn.LSTM(char_dim, lstm_dim, batch_first=True)
            for _ in range(num_layers)
        ])
        # 任务特定的 softmax 权重（学习得到）
        self.softmax_weights = nn.Parameter(
            torch.ones(2 * num_layers + 1) / (2 * num_layers + 1)
        )

    def forward(self, x):
        # x: (batch, seq_len) 字符级别的输入
        token_emb = self.char_cnn(x)  # 初始嵌入层
        all_layers = [token_emb]

        fwd_input, bwd_input = token_emb, token_emb
        for fwd_lstm, bwd_lstm in zip(self.forward_lstms, self.backward_lstms):
            fwd_out, _ = fwd_lstm(fwd_input)
            bwd_out, _ = bwd_lstm(bwd_input)
            # 拼接前向和后向表示
            combined = torch.cat([fwd_out, bwd_out], dim=-1)
            all_layers.append(combined)
            fwd_input, bwd_input = fwd_out, bwd_out

        # 加权融合所有层
        stacked = torch.stack(all_layers, dim=0)  # (layers, batch, seq, dim)
        weights = torch.softmax(self.softmax_weights, dim=0)
        elmo_vec = (weights[:, None, None, None] * stacked).sum(dim=0)
        return elmo_vec

11.3 ELMo 如何改变了游戏规则

ELMo 在 2018 年发布时，横扫了 6 项 NLP 基准任务，包括问答、文本蕴含、语义角色标注、命名实体识别、共指消解和情感分析。它以一种极其优雅的方式证明了：在大规模无标注数据上预训练的语言模型，能够学到通用且强大的语言知识。

但 ELMo 的意义远不止于刷榜。它带来了一种全新的方法论：特征基方法（Feature-based Approach）。我们来看看它和传统方法的区别：

  • 传统做法：用 Word2Vec 初始化嵌入层，在整个任务训练过程中微调词向量。
  • ELMo 的做法：冻结预训练好的 biLM，把它的每一层表示作为额外的特征，拼接到下游模型的输入中。下游模型本身不修改 ELMo 的参数。

这意味着什么？意味着 NLP 开发者不再需要从零开始训练词向量，也不再需要拥有海量数据。他们只需要加载一个预训练好的 ELMo 模型，在具体任务上加一个简单的分类器，就能获得非常大的性能提升。这听起来是不是和现在的预训练-微调范式很像？没错，ELMo 就是那个"推倒第一块多米诺骨牌"的工作。

⚠️ 踩坑实录：ELMo 的无尽预热
我第一次用 ELMo 做命名实体识别时，遇到了一个诡异的 bug：模型前几个 batch 的 loss 几乎不下降。我反复检查代码、学习率、数据预处理，折腾了一整天都没找到原因。后来偶然读到一篇博客才恍然大悟——ELMo 的 biLSTM 有 4096 个隐藏单元，对于很多下游任务来说这个维度太大，需要一个降维投影层来过渡。我忘了加这个投影层，导致 LSTM 的输出维度远大于下游分类器的输入维度，训练信号在维度转换过程中被严重稀释。加上一个简单的线性投影后，问题立刻解决了。这个教训提醒我：在使用大规模预训练模型时，中间适配层的设计绝不是小细节，它对训练稳定性至关重要。

11.4 从静态到上下文：一次范式革命

让我们用一个表格来对比静态词向量和上下文词向量的核心差异，这样更直观：

对比维度 静态词向量（Word2Vec / GloVe） 上下文词向量（ELMo / BERT）
表示方式 一个词对应一个固定向量（查表） 一个词在不同的上下文中对应不同的向量（函数）
多义词处理 无法区分，取多义的折中 根据上下文动态调整，天然支持多义
训练方式 使用局部上下文窗口（CBOW / Skip-gram） 使用整个句子或段落（语言模型目标）
模型架构 浅层神经网络（通常 1~2 层） 深层神经网络（ELMo 2 层 biLSTM，BERT 12~24 层 Transformer）
OOV（未登录词） 无法处理，需回退到特殊标记 可通过字符级或子词级建模来处理
下游使用方式 作为嵌入层初始化，可微调 作为额外特征（ELMo）或全模型微调（BERT）
数据需求 可在中等规模语料上训练 需要超大语料（数亿到数十亿词）才能发挥威力

从静态到上下文，不仅仅是技术上的进步，更是一次范式革命。它改变了我们看待 NLP 任务的方式：不再是为每个任务设计专门的架构，而是先在大规模数据上训练一个通用的语言表示模型，再针对具体任务进行适配。这种 "Pretrain + Adapt" 的模式，至今仍是 NLP 的主流范式。

11.5 预训练之路：ULMFiT 作为先行者

说到预训练方法，我们不能不提 ULMFiT（Universal Language Model Fine-Tuning）。它由 Howard 和 Ruder 在 2018 年初提出，比 ELMo 稍早几个月。虽然 ELMo 在知名度上更胜一筹，但 ULMFiT 提出了一套完整的迁移学习流程，对后来的 BERT 和 GPT 影响深远。

ULMFiT 的核心洞察是：如果我们在一个大规模通用语料库（比如维基百科）上预训练一个语言模型，然后把这个模型在目标任务的数据上继续微调，效果会比从零开始训练好得多。这个想法现在看来理所当然，但在当时是一个重要的突破。

ULMFiT 贡献了三个关键技术：

  • 差异化微调（Discriminative Fine-tuning）：对不同的层使用不同的学习率。底层的语言知识更通用，用较小的学习率微调；顶层的知识更偏向具体任务，用较大的学习率调整。
  • 倾斜三角学习率（Slanted Triangular Learning Rate）：先线性增加学习率（预热阶段），再线性衰减。这个策略让模型在微调初期快速适应任务，然后稳步收敛。
  • 逐步解冻（Gradual Unfreezing）：从最后一层开始，逐层解冻参与训练，而不是一次性全解冻。这样能防止灾难性遗忘。
🔗 关系梳理：ELMo 和 ULMFiT 虽然都使用了预训练语言模型，但它们的使用方式截然不同。ELMo 把预训练模型当作一个特征提取器（参数固定，提取特征给下游模型用），而 ULMFiT 开创了微调范式（预训练模型的参数在下游任务上继续更新）。这两种思路——特征基（feature-based） vs 微调（fine-tuning）——在后续的工作中都有发扬光大：ELMo 的思路影响了早期的 BERT 应用方式，而 ULMFiT 的微调范式直接启发了 GPT 和 BERT 的完整微调流程。

如果用一个比喻来总结 ELMo 和 ULMFiT 的关系：ELMo 像一把瑞士军刀，你可以把它带在身边，随时用它自带的功能来辅助你做各种事情；而 ULMFiT 像一套乐高指南，教你怎么把已有的积木（预训练模型）改装成你需要的样子。

从 ELMo 到 BERT：更大的想象空间

ELMo 和 ULMFiT 的成功，让整个 NLP 社区看到了预训练的潜力。但 ELMo 有一个天然的局限：它用的是 LSTM，只能从左到右（和从右到左分别）建模，无法像我们人类阅读一样同时看到整个句子。这就像一个只能通过慢慢扫描来理解一幅画的人，而不是一眼看到全局。

这个局限在第 10 章介绍的 Transformer 面前，显得格外突出。Transformer 的自注意力机制可以一次性看到所有的词，没有顺序处理的瓶颈。如果能在 Transformer 上做预训练，那效果会怎样？

答案就是：BERT。BERT 正是用 Transformer 编码器替代了 ELMo 的 LSTM，并在预训练任务上进行了精心设计——Masked Language Model（随机遮住一部分词让模型去预测）和 Next Sentence Prediction（判断两个句子是否连续）。这两个任务结合起来，让模型学到了极其丰富的语言知识。

而 GPT 则走了另一条路：用 Transformer 的解码器部分做单向语言模型预训练，然后在各种下游任务上微调。这就是我们现在常说的"生成式预训练"路线的起点。

所以 ELMo 最大的贡献，或许不是它本身的性能有多好（当然也确实很好），而是它打开了一扇门——证明了大规模预训练语言模型是通往更好 NLP 系统的关键路径。下一章我们来重点看 BERT，它会告诉我们这条路可以走多远。
