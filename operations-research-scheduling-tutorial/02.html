第02章 线性规划基础

大家好，我是你们的讲师。上一章我们搭建了运筹学与调度理论的整体框架，提到了单纯形法作为运筹学的第一个里程碑。今天，我们就来深入这个里程碑的核心——线性规划（Linear Programming, LP）。

线性规划可能是整个运筹学领域应用最广、影响力最大的模型框架。从石油炼化厂的混合配比到航空公司机组排班，从投资组合优化到物流网络设计，线性规划几乎无处不在。美国《Industrial Engineering》杂志曾做过一项调查，在"对企业最有价值的运筹学技术"排名中，线性规划常年霸榜第一。为什么它这么厉害？因为很多现实问题都可以用线性关系来近似描述，而线性规划又恰好有高效且稳定的求解算法。

这一章我们会从建模三要素讲起，引入图解法建立几何直觉，最后深入单纯形法的思想流程。准备好了吗？我们开始。

线性规划的建模三要素 — 决策变量、目标函数、约束条件

任何一个线性规划问题，都可以拆解为三个基本成分。先记住这个口诀：变量是你可控的选择，目标是你想优化的方向，约束是你不可逾越的边界。

  • 决策变量（Decision Variables）：代表你可以控制的数量。比如生产多少件产品 A、分配多少资金到股票 B。
  • 目标函数（Objective Function）：你想要最大化或最小化的指标。利润最大化、成本最小化、时间最短化——都是线性函数。
  • 约束条件（Constraints）：限制决策变量取值的等式或不等式。资源上限、需求下限、技术系数——一旦违背，方案就不可行。

只看定义有些抽象，我们用一个经典的生产计划问题来完整走一遍建模流程。

生产计划问题建模实战

假设你是一家家具厂的厂长。工厂生产两种产品：桌子和椅子。

  • 每张桌子利润 80 元，每把椅子利润 50 元。
  • 生产一张桌子需要 4 工时木工和 2 工时油漆工。
  • 生产一把椅子需要 2 工时木工和 2 工时油漆工。
  • 工厂每天可用木工 60 工时，油漆工 40 工时。
  • 市场需求限制：每天椅子产量不超过桌子产量的 2 倍。

现在问题来了：每天应该各生产多少，才能让总利润最大？

建模过程如下：

决策变量：
  x₁ = 每天生产的桌子数量
  x₂ = 每天生产的椅子数量

目标函数（最大化利润）：
  max  Z = 80x₁ + 50x₂

约束条件：
  木工工时：  4x₁ + 2x₂ ≤ 60
  油漆工时：  2x₁ + 2x₂ ≤ 40
  市场限制：   x₂ ≤ 2x₁  ——即 x₂ - 2x₁ ≤ 0
  非负约束：   x₁ ≥ 0, x₂ ≥ 0

这就是一个标准的线性规划模型。三个要素齐活：x₁, x₂ 是决策变量，Z 是线性目标函数，四个不等式是线性约束条件。

标准型的三种转化技巧

在实际求解中，我们通常将各种形式的 LP 统一为标准型（Standard Form），以便套用统一的算法。标准型的格式是：

min  cᵀx
s.t. Ax = b
     x ≥ 0

即最小化、等式约束、变量非负。把实际模型转化为标准型，需要掌握三种基本转化技巧：

原始形式 转化方法 标准型等价形式
最大化目标 max cᵀx 目标函数取负号 min -cᵀx
不等式约束 Ax ≤ b 添加松弛变量 s ≥ 0 Ax + s = b
不等式约束 Ax ≥ b 添加剩余变量 s ≥ 0 Ax - s = b
变量 x 无符号限制 拆分为 x = x⁺ - x⁻ x⁺ ≥ 0, x⁻ ≥ 0

举个例子，我们上面的家具厂模型是最大化问题，约束都是 ≤ 类型。转化为标准型：

min  -80x₁ - 50x₂
s.t. 4x₁ + 2x₂ + s₁ = 60
     2x₁ + 2x₂ + s₂ = 40
     x₂ - 2x₁ + s₃ = 0
     x₁, x₂, s₁, s₂, s₃ ≥ 0

其中 s₁, s₂, s₃ 就是松弛变量（Slack Variables），分别代表木工剩余工时、油漆剩余工时、市场限制的松弛量。它们的值也携带了信息——比如如果 s₁ = 0，说明木工工时刚好用完，木工就是"瓶颈资源"。

松弛变量的商业含义：松弛变量的值直接告诉你哪些资源是瓶颈、哪些资源有富余。在工厂管理中，"识别瓶颈"是排产优化的第一步。如果一个约束的松弛变量长期为零，说明这个资源需要优先扩产能。

图解法：二维空间的几何直觉

对于只有两个决策变量的问题，我们可以用图解法来获得最直观的理解。虽然实际工程问题很少只有两个变量，但对二维情形的几何理解，是理解高维 LP 直觉的基石。

可行域、等值线、最优解

还是用家具厂的例子。我们在平面上画出四条约束的边界线：

  • 4x₁ + 2x₂ = 60 → x₂ = 30 - 2x₁
  • 2x₁ + 2x₂ = 40 → x₂ = 20 - x₁
  • x₂ - 2x₁ = 0 → x₂ = 2x₁
  • x₁ = 0, x₂ = 0

所有约束的交集围成一个凸多边形——这就是可行域（Feasible Region）。可行域内的每一个点都代表一个合法的生产方案。

目标函数 Z = 80x₁ + 50x₂ 在平面上是一组平行线（等值线）。我们不断向外推这条线——即增大 Z——直到它即将离开可行域的"最后一刻"。这个"最后一刻"接触的点，就是最优解。

在我们的例子中，最优解位于 4x₁ + 2x₂ = 60 和 2x₁ + 2x₂ = 40 两条线的交点：(10, 10)。最优利润 Z = 80×10 + 50×10 = 1300 元。

四种特殊情形

并不是每个 LP 都有唯一的有限最优解。你可能会遇到以下四种情形：

情形 图例特征 原因 实际含义
无界（Unbounded） 可行域朝某个方向无限延伸，目标值可无限增大 缺少某些必要的约束 模型漏掉了现实中的关键限制，如资源上限
无可行解（Infeasible） 可行域为空——约束条件互相矛盾 约束之间存在冲突 需求太大资源太少，或输入数据有误
无穷多最优解 等值线与某条约束边界平行且重合 目标函数与约束梯度成比例 多个方案达到相同最优值，可灵活选择
退化（Degeneracy） 多个约束交于同一点 冗余约束 单纯形法可能循环，需要防循环规则

踩坑实录：我在某制造企业做咨询时，甲方提供了一组"看起来很正常"的产能数据。建模后求解器直接报"infeasible"。我们花了整整两天排查，最后发现——某个零件的加工时间是 0.5 小时，但录入系统时写成了 5 小时，导致所有资源约束同时过紧。一个数字的错误让整个模型不可行。这个教训告诉我们：模型无解时，首先怀疑数据，而不是算法。

单纯形法的思想之旅

图解法虽然直观，但工业问题动辄成千上万个变量，不可能画图。我们需要一个算法——就是 单纯形法（Simplex Method）。

从顶点到顶点的迭代搜索

单纯形法的核心思想可以用一句话概括：最优解一定出现在可行域的顶点上。线性规划的一个基本定理表明：如果 LP 有有限最优解，则一定存在一个顶点是最优解。

所以，我们找到了顶点就找到了候选最优解。但顶点数量可能非常大——对 n 个变量 m 个约束的 LP，顶点数可以达到 C(n, m)。枚举所有顶点不现实。单纯形法的聪明之处在于：它从某个顶点出发，沿着能使目标函数改善的边跳到相邻的、更好的顶点。重复这一过程，直到无法再改善为止。

这就像爬山（确切地说是下山，因为标准型是 minimize）：你每迈一步都在下坡，直到走到一个不能再下降的谷底。

基变量与非基变量的直觉解释

在标准型 Ax = b, x ≥ 0 中，变量被分为两组：

  • 基变量（Basic Variables）：对应顶点处的"紧约束"——它们在值为零的边缘线上自由移动。在一个有 m 个等式约束的 LP 中，恰好有 m 个基变量。
  • 非基变量（Non-basic Variables）：在当前顶点上值为 0 的变量。

每次迭代，我们让一个非基变量进基（Entering）——增大它的值，离开顶点沿边移动；同时让一个基变量出基（Leaving）——它最先降到 0，变成新的非基变量。这个过程就是一次"换基"，对应了一次从顶点到相邻顶点的移动。

下面是一个简单版本的手工迭代表：

初始单纯形表（家具厂问题）：

   基变量 |  x₁   x₂   s₁   s₂   s₃   RHS
  ─────────┼─────────────────────────────
    s₁     |  4    2    1    0    0    60
    s₂     |  2    2    0    1    0    40
    s₃     | -2    1    0    0    1     0
  ─────────┼─────────────────────────────
    -Z     | 80   50    0    0    0     0   ← 检验数行


1. 选择进基变量：检验数最大正数对应的列 x₁（检验数=80）
2. 最小比值测试：min(60/4, 40/2) = min(15, 20) = 15 → s₁ 出基
3. 主元消去，得新表：

   基变量 |  x₁   x₂   s₁   s₂   s₃   RHS
  ─────────┼─────────────────────────────
    x₁     |  1   0.5  0.25  0    0    15
    s₂     |  0    1  -0.5   1    0    10
    s₃     |  0    2   0.5   0    1    30
  ─────────┼─────────────────────────────
    -Z     | 0   10   -20   0    0   -1200

4. 选择进基变量：检验数 10 → x₂
5. 最小比值：min(15/0.5, 10/1, 30/2) = min(30, 10, 15) = 10 → s₂ 出基
6. 主元消去，得最终表：

   基变量 |  x₁   x₂    s₁    s₂   s₃   RHS
  ─────────┼─────────────────────────────
    x₁     |  1    0    0.5  -0.5   0    10
    x₂     |  0    1   -0.5   0.5   0    10
    s₃     |  0    0    1.5  -1.0   1    10
  ─────────┼─────────────────────────────
    -Z     | 0    0   -15   -5    0   -1300

7. 所有检验数 ≤ 0，最优！
   解：x₁=10, x₂=10, Z=1300

如果你仔细看最终表的检验数行（-15, -5, 0），它们其实就是影子价格的相反数——关于影子价格，我们下一章会详细讲。

为什么单纯形法在理论上是指数复杂度，但在实践中极其高效？

Klee 和 Minty 在 1972 年构造了一类特殊 LP（Klee-Minty 立方体），让单纯形法需要访问所有 2ⁿ 个顶点才能找到最优解，证明单纯形法在最坏情况下是指数时间的。然而在实际应用中，单纯形法通常只需 O(m) 到 O(2m) 次迭代就能收敛——即迭代次数与约束数成正比。这是因为实际 LP 问题的数据结构通常是"良性的"，顶点不是指数级分布的。

1984 年 Narendra Karmarkar 提出了内点法（Interior Point Method），是第一个多项式时间的 LP 算法。现代求解器（如 Gurobi、CPLEX）通常同时集成单纯形法和内点法，根据问题特征自动选择最合适的算法。

好，线性规划的基础框架我们已经搭起来了。在下一章中，我们将沿着这条线继续深入——当你求得一个 LP 的最优解之后，对偶理论告诉你"这个解到底有多好"，灵敏度分析告诉你"如果数据变化了，会怎样"。这两个工具在真实的调度决策中至关重要。我们下一章见。
