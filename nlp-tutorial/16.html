第十六章：偏见与公平性

大家好！这一章我们要直面NLP领域最"政治敏感"但又无法回避的问题——模型偏见。技术从来不是中立的，NLP模型从数据中学到的不仅是语言知识，还有人间的偏见。忽视这个问题不只是不道德，在商业上也会有严重后果。

2018年有一篇影响深远的论文（Bolukbasi et al.），发现Word2Vec训练出的词向量中，man:computer_programmer :: woman:homemaker。这不是巧合——训练语料（Google News）本身就反映了社会偏见，而模型忠实地"学会"了这些偏见。更恐怖的是，这种偏见在大模型时代不但没有消失，反而以更隐蔽的方式被编码进了数十亿参数中。

16.1 偏见从哪来

偏见来源说明例子
训练数据偏见语料本身反映了社会历史偏见新闻中"CEO"更多关联男性代词
标注偏见标注者的主观判断引入偏见标注毒性言论时对某些方言/口吻更敏感
模型放大模型训练过程可能放大数据中的minor偏见生成时比原数据更偏激
部署偏见模型在某些群体上准确率显著更低语音识别对非标准口音准确率差
# 词向量中的性别偏见——一个经典实验
# 使用Word2Vec on Google News

# 类比1（性别中立 → 职业偏见）：
# man - woman ≈ computer_programmer - homemaker ✓（不应该！）

# 类比2（性别本质化）：
# father - mother ≈ doctor - nurse ✓（不应该！）

# 用几何方法测量偏见
def gender_bias(word, male="he", female="she"):
    gender_direction = vec(male) - vec(female)
    return np.dot(vec(word), gender_direction)
# 正值表示偏向男性方向，负值表示偏向女性方向

16.2 公平性指标

公平性不是单一的指标，而是一组相互制衡的目标：

  • 人口统计均等：不同群体的预测正率相等。P(ŷ=1|A=a) = P(ŷ=1|A=b)，A是敏感属性（性别、种族等）。代价——可能降低整体准确率。
  • 等化几率：不同群体的真正率和假正率相等。在正类和负类上都要平等，比人口均等更强。
  • 个体公平：相似的个体应得到相似的预测。问题——怎么定义"相似"？

⚠️ 现实困境：这些公平性指标之间可能是矛盾的。有数学证明（Kleinberg et al. 2016）：除非两个群体的base rate完全相等或模型完美，否则人口均等和等化几率不可能同时满足。公平性和准确率之间也存在trade-off——追求公平可能降低总体准确率。

16.3 偏见缓解策略

# 方法1：数据处理层（Data-level）
# - 平衡训练数据中不同群体的表示
# - 数据增强（反事实数据：交换性别代词生成对称样本）
# - 去偏标注：用更中性的标注指南

# 方法2：模型层（Model-level）
# - 对抗训练：让模型的主任务好，但性别分类器猜不出性别
# - 正则化：在loss中加入公平性约束项
# 损失 = 任务损失 + λ × 公平性损失

# 方法3：后处理层（Post-processing）
# - 校准不同群体的输出阈值
# - 对生成文本做后过滤

重要教训：我曾经做过一个简历筛选的NLP系统。训练数据中70%的"hire"标签给了男性简历——因为历史招聘数据就有性别偏差。直接用这个数据训练，模型雪上加霜地放大了偏见。最后我们用了反事实数据增强——对训练集中所有简历交换性别代词，再要求模型在两个版本上输出一致。公平性指标改善了40%，但准确率下降了3%。这个trade-off是否值得，需要业务方和合规方共同决定——技术团队不能替他们做这个判断。

16.4 偏见检测工具与责任

实践中，定期用标注好的评估集（如WinoBias、StereoSet）检测模型偏见。更重要的原则是：偏见问题没有纯技术方案。技术手段（去偏、正则化）只能缓解，不能根除。公平性是一个社会技术问题（socio-technical），需要技术团队、产品团队、法律合规团队和受影响社区的共同努力。
