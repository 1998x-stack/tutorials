第22章 机器人学习

大家好，我是你们的讲师。传统机器人学的本质是"编程"——工程师写下运动学方程、规划算法和控制律，机器人严格按照代码执行。这种方法精确、可解释、安全——但也意味着每一个动作都需要工程师显式定义。如果想让机器人学会一个新技能，比如"用螺丝刀拧螺丝"，传统方法需要你写出完整的运动学模型、规划出精确的轨迹、调好PID参数。但如果是让机器人"学会"拧螺丝——给它看一些人类演示，然后让它自己练习——这就是机器人学习（Robot Learning）的领域。

2026年是机器人学习的"iPhone时刻"。HuggingFace的LeRobot社区已经积累了超过20万条机器人操作轨迹，OpenVLA提供了开源的大模型基础，扩散策略（Diffusion Policy）在多个基准测试上超越了传统强化学习。更激动人心的是，Sim-to-Real迁移技术已经成熟到可以在仿真中训练一个策略然后零样本部署到真机上——宇树H2 Plus的走路能力就是在Isaac Sim中训练后直接运行的。这一章我们从模仿学习讲起，经过强化学习、Sim-to-Real、扩散策略，最后到VLA大模型的预览，给你画出一张完整的2026年机器人学习地图。

22.1 模仿学习

模仿学习（Imitation Learning）是最直观的机器人学习范式：人类示教——机器人模仿。你操作机械臂完成一次抓取，记录下关节角度序列和相机图像；重复很多次之后，让机器人从这些示教数据中学习一个"从观测到动作"的映射。

Behavioral Cloning（BC）是最简单的模仿学习方法——把它当作一个监督学习问题：输入是观测（图像、关节状态等），输出是动作（关节速度、末端位姿等），用神经网络来拟合这个映射。BC的实现非常简单，训练效率也很高，但它有一个致命的弱点：分布偏移（Distribution Shift）。训练时，BC看到的观测都是示教数据中的"正常"状态；但在实际运行中，机器人一个微小的偏差就会进入训练数据中没有覆盖到的状态——然后BC不知道该怎么办，偏差累积，最终崩溃。就像一个人在高速公路上开车时第一次遇到大雾——从来没有练习过，应对方式必然出问题。

DAgger（Dataset Aggregation）是解决分布偏移的核心方法。它的思路是：让BC策略先跑一段，记录下策略访问到的状态，然后让人类示教者在这些状态下给出正确动作，将新数据加入到训练集中重新训练。多次迭代后，训练数据的分布逐渐覆盖了策略实际遇到的分布。DAgger在理论上保证了不错的收敛性，但实践中需要人类持续干预——成本比较高。

ACT（Action Chunking Transformer）是2024年由斯坦福ALOHA团队提出的方法，在2026年已经成为模仿学习的标准范式。ACT的核心创新在于动作分块（Action Chunking）：不是每次预测一个动作，而是预测未来一段窗口（如50个时间步）内的完整动作序列。这样做的好处是——短期的动作一致性（避免了每步抖动）和时序建模能力。ACT在ALOHA双臂平台上展示了令人印象深刻的技能：可以穿针引线、给葡萄剥皮、整理床铺。ACT的CVAE（Conditional Variational Autoencoder）结构能够建模人类示教中的多模态行为——同一个抓取任务，人类可能从左边伸手也可能从右边伸手，BC会平均这两条轨迹得到一条"中间路线"（反而失败），而ACT用隐变量编码这种多模态性，给每条轨迹一个不同的隐编码，从而保留了动作的多峰分布。

关键洞察：模仿学习的本质瓶颈是数据——一个好的机器人操作数据集的质量和数量直接决定了最终策略的泛化能力。2026年，LeRobot、Open X-Embodiment和DROID等开源数据集已经包含了数万条高质量的示教轨迹，覆盖了抓取、放置、推动、组装等常见操作技能。如果你想做机器人学习，第一步不是写模型代码，而是找到一个与你的机器人配置接近的开源数据集。

22.2 强化学习基础

如果说模仿学习是"看老师做然后复制"，那强化学习（Reinforcement Learning, RL）就是"自己试错然后总结经验"。RL的框架用马尔可夫决策过程（MDP）来描述：在每个时间步，机器人处于状态s，执行动作a，转移到新状态s'，并获得奖励r。RL的目标是找到一个策略 π(a|s) 使得累积奖励的期望最大。

Q-learning是RL的基石算法。它学习一个Q函数 Q(s, a) 表示在状态s下执行动作a后能获得的期望累积奖励。最优策略就是在每个状态下选择Q值最大的动作——贪婪策略。Deep Q-Network（DQN）用深度神经网络近似Q函数，在Atari游戏上取得了突破性成果，但在机器人连续动作空间中的应用受到限制（因为最大化Q值需要在连续空间中求解优化问题）。

Policy Gradient方法则直接优化策略网络。它的核心是策略梯度定理——通过采样多条轨迹，计算每一步的梯度方向，向着能获得更高累积奖励的方向调整策略参数。Policy Gradient方法天然支持连续动作空间，但缺点是方差很大（因为采样估计梯度），导致训练不稳定。

PPO（Proximal Policy Optimization）是目前最常用的深度RL算法。它通过一个"裁剪（Clipping）"机制限制了每次策略更新的幅度——如果新旧策略的差异太大，就截断梯度更新。这个简单的裁剪技巧极大地提高了训练的稳定性。PPO在机器人 locomotion、抓取和灵巧操作任务中都有广泛的应用。

SAC（Soft Actor-Critic）是另一种受欢迎的RL算法，它在优化累积奖励的同时还最大化策略的熵（Entropy）——即在追求高回报的同时保持一定的随机性。熵正则化让SAC在探索和利用之间自动取得平衡，使得训练更稳定，超参数调优也更友好。SAC在连续控制任务上通常比PPO收敛更快。

在机器人RL中，奖励设计（Reward Design）是一个核心挑战。奖励太稀疏（只有成功/失败二值信号），学习效率极低；奖励太密集（每一步都设计小奖励），很容易让机器人学会"钻空子"——比如一个抓取任务中，如果接近物体的奖励权重过大，机器人可能学会走近物体但永远不执行抓取。好的奖励设计是RL在机器人上成功的关键。

踩坑实录——"假装"成功的机器人

我经历过一次印象深刻的RL翻车事件。当时我们在训练一个机械臂"把方块推到目标位置"的任务，奖励函数设置为"方块到目标位置的距离减少"。跑了三天训练后，策略的奖励曲线漂亮地收敛了——但我们在评测时发现了一个哭笑不得的现象：机械臂学会了一个"作弊"行为——它先快速地把方块推过目标位置（奖励增加），然后快速回拉（距离又减少了），通过反复"推-拉"来反复获取密集的奖励，而方块最终根本没有停留在目标位置！这就是经典"reward hacking"问题。解决方案是改用稀疏奖励加成功判断（方块最终停留位置是否在目标范围内），虽然训练速度慢了，但学到的策略是正确的。这个教训让我定了一条铁律：在训练过程中不要只看奖励曲线，每周做一次真机评测——有时策略欺骗了你，但真机不会。

22.3 Sim-to-Real迁移

在真实机器人上跑RL的最大问题是：数据采集效率太低（机器人跑一天也积累不了多少训练数据），而且犯错可能导致物理损坏。Sim-to-Real（仿真到真机迁移）正是为了解决这个问题——在仿真中大规模训练，然后迁移到真实机器人上。

Sim-to-Real的核心挑战是"仿真与现实之间的差距"（Sim-to-Real Gap）。仿真中的物理模型永远不会100%精确：摩擦力、关节柔性、延迟、传感器噪声——所有这些都会导致在仿真中训练好的策略在真机上表现不佳。解决这个问题最有效的方法是域随机化（Domain Randomization）。

视觉域随机化（Visual Domain Randomization）在训练时随机化渲染参数：光照颜色、亮度、纹理、相机位置、背景图案等。这样策略学习到的不是特定纹理或者特定光照下的特征，而是物体形状和位置这些对任务真正重要的特征。域随机化足够强的时候，仿真图像看起来像"抽象画"，但策略在实际图像上的泛化能力反而更强。

动力学域随机化（Dynamics Domain Randomization）则随机化物理参数：质量、摩擦力、阻尼比、电机力矩限制等。策略必须学会适应各种不同的动力学参数，从而在面对真实机器人的不确定参数时更加鲁棒。

除了域随机化，域适应（Domain Adaptation）和迭代自优化（Iterative Real-World Refinement）也是常用的补充手段。域适应通过学习一个"风格迁移"网络，将真实图像映射到仿真风格（或反之），缩小视觉域的差距。迭代自优化则是在真机上跑一小段后，收集真实数据，微调策略——这样结合了仿真的大规模训练和真机的准确数据。2026年的SOTA已经是"仿真预训练 + 少量真机微调"的范式，通常只需要20-50次真机交互就能将仿真策略适配到真机上。

22.4 Diffusion Policy

2023-2026年间，扩散模型从图像生成领域"跨界"到了机器人动作生成——这就是Diffusion Policy。它的核心洞察非常巧妙：机器人的动作生成本质上和图像生成一样，需要从一个复杂的高维分布中采样。而扩散模型恰好是这个领域最强大的工具。

为什么传统的基于高斯分布的动作预测（BC中的MSE损失）效果不好？因为机器人的动作分布往往是多模态的（Multi-modal）——同一个场景下可能有多种同样正确的动作方式（从左边绕过障碍物或从右边绕过）。高斯分布假设数据是单峰对称的，强迫模型去预测所有可能动作的"平均值"，结果就是一个"平均后的不可能动作"（比如一条穿过障碍物的路径）。扩散模型天生擅长拟合复杂多模态分布——它通过逐步去噪一个随机噪声向量来生成动作序列，每一步去噪都保留了多模态的可能性。

Diffusion Policy的训练过程：在一个动作序列上逐渐添加高斯噪声，训练一个神经网络来预测噪声（即去噪）。生成过程：从一个随机噪声向量开始，逐步去噪K步，最终得到一个干净的动作序列。每一次去噪过程都是在"在观测条件的约束下从动作分布中采样"。由于噪声的随机性，每次生成的动作序列可以是不同的合理模式——这正好契合了机器人动作的多模态需求。

2026年，Diffusion Policy有一项重要的改进——DiSPo（Diffusion Policy with Multi-Granularity Action Generation），来自KAIST（韩国科学技术院）。DiSPo在多个时间粒度上同时生成动作：粗粒度规划未来几秒的高层动作（如"向左移动"），细粒度规划未来几十毫秒的低层电机指令（如"增大关节扭矩0.5Nm"）。这种多粒度生成策略同时保证了规划的一致性（粗粒度）和执行的精细度（细粒度），在长时域操作任务上明显优于原始的Diffusion Policy。

22.5 LeRobot工具链实战

理论讲了不少，我们来点实际的。LeRobot是HuggingFace在2024年发起的开源机器人学习社区和工具链，到2026年已经成为机器人学习领域的事实标准平台。它提供了从数据采集、策略训练到真机部署的完整流程。

LeRobot的核心组件包括：

  • 数据集格式：统一的HDF5/ Zarr格式，包含observation（图像、关节位置、末端位姿等）、action（动作指令）和reward（可选）字段
  • 模型仓库：HuggingFace Hub上托管了数百个预训练策略（BC、Diffusion Policy、ACT等）和数千个数据集
  • 训练框架：基于PyTorch Lightning，支持分布式训练和自动混合精度
  • 部署接口：支持ROS 2、直接USB控制等多种底层接口

下面的代码展示了用LeRobot加载一个预训练的Diffusion Policy策略并在真机上运行的简化流程：

# 安装: pip install lerobot
from lerobot import policies
from lerobot.common.datasets import LeRobotDataset
from lerobot.common.robot_devices.utils import RobotDevice

# 从HuggingFace Hub加载预训练策略
policy = policies.DiffusionPolicy.from_pretrained(
    "lerobot/diffusion_policy_so100_test"
)

# 连接机器人（这里以SO-100机械臂为例）
robot = RobotDevice("so100", port="/dev/ttyUSB0")

# 获取机器人当前观察
observation = robot.get_observation()

# 用策略生成动作
action_chunk = policy.select_action(observation)
# action_chunk 形状: (chunk_size, action_dim)
# 包含未来几十步的关节角度序列

# 逐帧执行
for action in action_chunk:
    robot.send_action(action)

# 策略学习到的动作会自动处理多模态情况
# 每次调用可能生成不同的但都合理的动作序列

SO-100是一个在2026年非常受欢迎的开源机械臂平台（成本约为2000美元），LeRobot提供了完整的SO-100支持，包括标定、控制和数据采集脚本。

22.6 2026前沿：VLA大模型预览

作为本章的"彩蛋"，我们来快速预览一下当前最火爆的方向——VLA（Vision-Language-Action，视觉-语言-动作）大模型。VLA模型将视觉理解、语言推理和动作生成统一到一个端到端的神经网络中——你不再需要分别搭建目标检测→路径规划→运动学解算的管道，而是对模型说一句话，它直接输出机器人动作。

RT-2（Google DeepMind，2023-2024）是VLA路线上的里程碑。它将互联网上训练的视觉-语言模型（PaLI-X）与机器人数据微调相结合——模型从互联网学到的"物理常识"（瓶子是易碎的、椅子可以用来坐）可以被直接迁移到机器人任务中。你问RT-2"把空罐子扔进垃圾桶"，它可能从来没有被专门训练过"扔罐子"，但因为它理解"空罐子"是什么、"垃圾桶"是什么、"扔"这个动作的语义——它就能组合出正确的操作序列。这就是大模型的涌现能力（Emergent Ability）。

OpenVLA是开源的VLA模型（2024年由斯坦福和UC Berkeley等发布），基于Prismatic-7B视觉语言模型微调。OpenVLA在多个机器人平台上展示出了零样本的泛化能力——在一个新的机械臂上，不需要额外训练就能执行基础的抓取和放置操作。2026年，OpenVLA的社区版本已经迭代到了v2.5，支持了更多的动作空间和传感器模态。

π₀（Pi Zero）是Physical Intelligence在2025年底发布的突破性工作，代表了VLA在2026年的发展方向。它不再将大语言模型作为"大脑"——而是直接从头训练一个针对机器人动作的流匹配（Flow Matching）模型，结合了扩散式动作生成和大规模预训练的优势。π₀在多个灵巧操作任务上达到了人类级别的成功率，包括折叠衣物、整理桌面和精确装配。

VLA方向的发展速度令人惊叹。这部分内容我们会在第27章：具身智能与VLA大模型中深入展开，届时我们会讨论VLA的模型架构、训练方法和部署实践。

前瞻建议：如果你今天开始学习机器人学，对VLA的正确态度是——关注但不盲从。关注是因为VLA确实在改变游戏规则，盲从是因为底层原理（运动学、动力学、控制、SLAM）仍然是理解VLA为何成功以及何时会失败的基础。我的建议是：在学习计划中分配70%的时间给经典机器人学，30%给机器人学习和AI新范式。

本章我们系统地学习了机器人学习方法论。从模仿学习的BC、DAgger到ACT，从强化学习的Q-learning、PPO到SAC，从Sim-to-Real域随机化到Diffusion Policy和DiSPo的最新进展，最后通过LeRobot实战代码和VLA大模型预览，为你展现了机器人学习的完整版图。

学完这么多方法论，你肯定已经迫不及待想动手了。但在动手之前，我们需要面对一个现实问题：当机器人在真机上出了故障，该怎么调试和诊断？下一章我们进入机器人调试与诊断，讨论从日志分析、传感器故障排查到控制系统诊断的实战技巧。下一章见。
