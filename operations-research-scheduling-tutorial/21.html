第21章 多目标调度优化

大家好，我是你们的讲师。我们在前面 20 章里讨论的所有调度问题，在数学上都有一个共同点：一个明确的目标函数——最小化 makespan、最小化总加权完工时间、最小化最大延迟。但在真实世界里，一个调度方案的好坏从来不是一个单一数字能衡量的。

你试想一下自己是一个车间主任：你希望生产时间越短越好（makespan 小），但你也希望每个订单都能按时交货（延迟小）；你希望机器利用率越高越好，但你也希望能耗越低越好；你希望多发急单抢进度，但你又不想让老客户的长期订单被挤压太多次。这些目标之间存在根本性的冲突——缩短 makespan 往往意味着要开更多机器，能耗自然上升；优先赶急单，常规订单的延迟就会增加。鱼和熊掌不可兼得——这就是多目标调度的核心困境。

这一章我们系统地讨论多目标调度优化：从 Pareto 最优的概念，到经典的多目标方法，再到前沿的多目标元启发式算法。

调度的多目标本质 — makespan、延迟、能耗、成本不可兼得

先列举调度问题中常见的目标函数，以及它们之间的典型冲突关系：

目标 1 目标 2 冲突关系 实际含义
最小化 makespan 最小化总能耗 强冲突 想快就要多开机器/提高转速，能耗必然上升
最小化 makespan 最小化最大延迟 Lmax 弱冲突或一致 整体加快有助于减少延迟，但不一定能满足紧急订单
最小化总延迟 ΣTj 最小化能耗 中等冲突 为减少延迟需要加速或加急，增加能耗
最小化最大延迟 Lmax 最小化能耗 强冲突 最紧急的订单往往需要不惜能耗来赶工
最小化加权完工时间 最小化成本 弱冲突 高优先级（权重高）的工件可能占用昂贵设备
最大化机器利用率 最小化在制品库存 中等冲突 高利用率需要持续投料，推高 WIP
最小化碳排放 最小化 makespan 强冲突 低碳排需要降速节能，必然延长生产周期

这些冲突意味着：不存在一个调度方案能同时优化所有目标。我们只能寻找"折中"或"权衡"——而这正是多目标优化的意义所在。

Pareto最优的概念 — 非支配解与Pareto前沿

多目标优化中最核心的概念是Pareto 最优（Pareto Optimality），由意大利经济学家 Vilfredo Pareto 在 1906 年提出。定义如下：

  • 一个解 x 被称为支配（dominate）另一个解 y，当且仅当 x 在所有目标上不劣于 y，且至少在某个目标上严格优于 y。
  • 一个解 x* 被称为Pareto 最优解（或非支配解），如果不存在任何可行解能支配它。
  • 所有 Pareto 最优解在目标空间中的像称为Pareto 前沿（Pareto Front）。

用一个通俗的比喻：如果你是一个调度员，你找到了一个方案 A——makespan 是 100 小时，能耗是 5000 kWh。另一个方案 B——makespan 是 95 小时，能耗是 5200 kWh。那么 A 和 B 互相不支配——A 能耗更低，B 时间更短。它们都是 Pareto 最优解（假设没有其他目标更好的方案）。你选择哪一个，取决于业务方更看重时间还是能耗。

多目标优化的目标不是找到一个最优解，而是找到一组具有良好分布性和收敛性的 Pareto 前沿解集，让决策者从中选择最符合当前偏好的方案。

经典多目标方法

加权和法、ε-约束法

加权和法（Weighted Sum Method）是最直观的方法：给每个目标分配一个权重 wᵢ，将多目标问题转化为单目标问题：

min  w₁·f₁(x) + w₂·f₂(x) + ... + wₖ·fₖ(x)

其中 Σ wᵢ = 1, wᵢ ≥ 0

加权和法的优点是简单、求解快——你可以直接用单目标求解器。缺点是：对于非凸的 Pareto 前沿，加权和法无法找到"凹坑"内部的 Pareto 解。而且权重的选择本身就是一个难题——不同的权重比例对应不同的 Pareto 解，要得到均匀分布的 Pareto 前沿，需要反复尝试权重组合。

ε-约束法（ε-Constraint Method）的思路不同：选择其中一个目标作为主优化目标，将其他目标转化为约束：

min  f₁(x)
s.t. f₂(x) ≤ ε₂
     f₃(x) ≤ ε₃
     ...
     fₖ(x) ≤ εₖ

通过系统地调整 ε₂, ε₃, ..., εₖ 的取值，可以生成不同的 Pareto 解。ε-约束法的优点是可以处理非凸的 Pareto 前沿，而且对决策者更直观（"我把能耗限制在 5000 kWh 以内，最小化 makespan"）。缺点是 ε 的步长选择影响解的分布质量，而且如果 ε 选得太紧，问题可能变得不可行。

目标规划(Goal Programming)

目标规划（Goal Programming, GP）由 Charnes 和 Cooper 在 1961 年提出，特别适合"决策者对每个目标有一个期望值"的场景。比如："我希望 makespan 不超过 100 小时，能耗不超过 5000 kWh，最大延迟不超过 5 小时"。GP 的数学形式是：

min  Σ (dᵢ⁺ + dᵢ⁻)
s.t. fᵢ(x) + dᵢ⁻ - dᵢ⁺ = gᵢ,   ∀i
     dᵢ⁺, dᵢ⁻ ≥ 0

其中 gᵢ 是目标的期望值，dᵢ⁺ 和 dᵢ⁻ 分别表示正偏差和负偏差。GP 的优点是业务语义非常清晰——"我允许目标偏离期望值，但我希望偏离越小越好"。加权 GP 还可以对不同目标设不同的权重，反映优先级。

字典序优化(Lexicographic Optimization)

当目标有明显的优先级顺序时——"首先最小化 makespan，在 makespan 不劣化的前提下最小化能耗，再在两者都不劣化的前提下最小化最大延迟"——字典序优化（Lexicographic Optimization）是最自然的选择。

做法是：先优化第一优先级的目标，得到最优值 f₁*；然后加上约束 f₁(x) ≤ f₁*，优化第二优先级目标；依次类推。字典序优化的优点是顺序清晰、无需权重，缺点是前序优先级的目标会完全压制后面的目标——即使略微牺牲 0.1% 的 makespan 能换来 50% 的能耗降低，字典序优化也不会接受。

多目标元启发式

当调度问题的规模变大，精确的多目标方法（如基于 ε-约束的 MIP）不再可行。这时多目标元启发式成为主力。

NSGA-II的非支配排序与拥挤度距离

NSGA-II（Non-dominated Sorting Genetic Algorithm II）由 Deb 等人在 2002 年提出，是多目标进化算法中最具影响力的算法。其核心机制有两个：

非支配排序（Non-dominated Sorting）：将种群中的解分层——第一层是当前种群中所有非支配解（Pareto 前沿），第二层是去掉第一层后剩余解中的非支配解，以此类推。排序越靠前的解（层级越低），其适应度越高，被选入下一代的机会越大。

拥挤度距离（Crowding Distance）：在同一非支配层级中，解与相邻解在目标空间中的距离。拥挤度距离大的解周围"人烟稀少"，选择它们可以维持 Pareto 前沿的多样性，避免所有解挤在一起。

下面给出 NSGA-II 非支配排序的伪代码：

NSGA-II 非支配排序算法

输入：种群 P（N 个个体，每个个体有 k 个目标值）

输出：每个个体的非支配层级 rank

1. 对于每个个体 p ∈ P:
   - 初始化 S_p = ∅          // p 支配的个体集合
   - 初始化 n_p = 0           // 支配 p 的个体数量
   - 对于每个个体 q ∈ P, q ≠ p:
       如果 p 支配 q，则 S_p = S_p ∪ {q}
       如果 q 支配 p，则 n_p = n_p + 1
   - 如果 n_p == 0，则 p 属于第一前沿：
       rank(p) = 1, F₁ = F₁ ∪ {p}

2. i = 1
3. 当 F_i ≠ ∅:
   - Q = ∅
   - 对于每个 p ∈ F_i:
       对于每个 q ∈ S_p:
         n_q = n_q - 1
         如果 n_q == 0:
           rank(q) = i + 1
           Q = Q ∪ {q}
   - i = i + 1
   - F_i = Q

4. 返回所有个体的 rank

注：支配关系的定义 —
  p 支配 q 当且仅当:
    ∀j ∈ {1,...,k}: f_j(p) ≤ f_j(q) 且
    ∃j ∈ {1,...,k}: f_j(p) < f_j(q)

NSGA-II 的完整流程是：每一代先做非支配排序和拥挤度距离计算，然后用二元锦标赛选择（基于 rank 优先、拥挤度距离次之）产生新种群，再通过交叉和变异生成子代，最后合并父代和子代后截断选择 N 个个体进入下一代。经过足够多代进化后，种群的 Pareto 前沿会收敛到真实前沿附近。

MOEA/D的分解策略

MOEA/D（Multi-Objective Evolutionary Algorithm based on Decomposition）由 Zhang 和 Li 在 2007 年提出，思路与 NSGA-II 截然不同：它将多目标问题分解为若干个单目标子问题，每个子问题对应一组权重向量，然后同时优化所有子问题。

MOEA/D 的核心步骤：

  • 生成 N 组均匀分布的权重向量 (w₁, w₂, ..., wₖ)
  • 对每组权重，构造一个聚合函数（如加权 Tchebycheff 函数）
  • 每个子问题维护一个当前最优解
  • 子问题之间通过"邻居"关系共享信息——一个子问题的新解可以传递给其邻居子问题

MOEA/D 的优势在于：它天然地生成均匀分布的 Pareto 前沿（因为权重向量是均匀分布的），而且它的搜索效率通常比 NSGA-II 高，尤其是在高维多目标（MaOP，目标数 ≥ 4）的情况下。

多目标调度benchmark与评价指标

评价多目标算法的性能不是一个 trivial 问题——因为我们不知道"真实 Pareto 前沿"是什么。常用的评价指标包括：

  • IGD（Inverted Generational Distance）：衡量算法获得的 Pareto 前沿与参考前沿之间的"距离"。IGD 值越低越好，同时衡量收敛性和多样性。
  • HV（Hypervolume）：算法获得的 Pareto 前沿在目标空间中"覆盖"的体积。HV 值越高越好，是目前最广泛使用的指标。
  • Spread（Δ）：衡量 Pareto 前沿的分布均匀程度。Δ 越接近 0 越好。

调度领域常用的多目标 benchmark 包括：Taillard 的 flow shop 实例扩展（加入能耗目标）、多目标 JSP 实例（如 from DMU 数据集）和 FJSP 多目标实例。在 2025 年的一项大规模实验中，有人对比了 15 种多目标算法在 120 个调度实例上的表现，结论是：NSGA-II 仍然是"最稳妥的基线选择"，但 MOEA/D 和最新的 NSGA-III（处理高维目标）在特定问题上表现更好。

一个来自2026年 permutation flow shop 的重要发现：2026 年发表在《European Journal of Operational Research》上的一篇论文，用 40 年的 Taillard benchmark 实例做了一个系统性的实验：对于每个实例，先用单目标优化得到 makespan 最优解，然后用多目标优化得到 Pareto 前沿。结果发现，在 50% 以上的实例中，单目标最优的 makespan 解，在"能耗"和"最大延迟"两个目标上严重失衡——这些解的平均能耗比 Pareto 前沿上的折中解高出 30% 以上，而最大延迟高出 2-3 倍。换句话说，只优化一个目标而忽略其他目标，产生的结果可能在业务上完全不可接受。这再次证实了：调度优化必须是多目标的——不是锦上添花，而是基本要求。

好，这一章我们讨论了多目标调度的本质、Pareto 最优的概念、经典多目标方法和前沿的多目标元启发式算法。但不管是单目标还是多目标，我们前面讨论的所有调度问题都隐含了一个假设：资源是无限可用的——至少我们没有显式地限制同时使用资源的数量。但在真正的项目管理中，人员、设备、资金都是有限的。下一章 资源约束项目调度（RCPSP），我们将正式引入资源约束，讨论如何调度一个包含数十到数百个活动、每种资源都有严格上限的项目。我们下一章见。
