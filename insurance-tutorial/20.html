第20章 非寿险精算

2017年，我作为精算顾问参与了一家车险公司的费率改革项目。客户的总精算师对我说了这么一句话："寿险精算师是在平稳的湖面上划船——死亡率有生命表，利率有国债曲线，一切都可预测。我们非寿险精算师是在大海里冲浪——车险事故率随天气和路况波动，责任险的长尾可以拖十几年，巨灾来了整个损失分布都是颠覆性的。"他说的也许有点夸张，但确实点出了非寿险精算的核心挑战：不确定性更大，数据的"信号"更弱，对精算师的判断力要求更高。

如果你在寿险公司做精算，你可以用几十年积累的生命表和市场利率来定价；如果做非寿险精算，你的数据可能只有过去3-5年的理赔记录，而且这些记录受经济周期、天气模式、法律环境甚至社会行为的影响，随时可能"突变"。本章我们来看看非寿险精算的那些独特挑战和方法。

非寿险精算的特点

与寿险精算的差异

非寿险精算和寿险精算虽然共享"精算"这个前缀，但从方法论到实务操作，差异非常大：

维度 寿险精算 非寿险精算
保单期限 长期（10年-终身） 短期（通常1年）
损失不确定性 死亡时间确定，金额固定 是否发生和损失金额都不确定
数据基础 生命表（百万级样本，稳定） 理赔记录（样本量小，波动大）
利率敏感性 极高（长期负债） 较低（短期负债，但长尾业务除外）
准备金 净保费法为主的确定性方法 统计估计为主（链梯法、BF法等）
分保定价 基于生命表和利率 基于损失分布、信度理论、经验费率

非寿险精算还有一个寿险没有的"痛"——长尾责任。责任险、工伤险的理赔可能拖10年、20年才完全结案（比如石棉肺赔偿案，暴露后几十年才发病）。这意味着非寿险精算师在做IBNR（已发生未报案）准备金估计时，面对的不确定性远比寿险大。

损失分布复杂

非寿险的损失有两个层次的不确定性：会不会发生（频率分布）和发生了赔多少（金额分布）。这两个问题各自需要不同的概率分布来建模，之后再结合起来。

损失分布

赔付频率分布

赔付频率描述的是"每张保单在保险期间内会理赔几次"。常用的分布包括：

  • 泊松分布（Poisson Distribution）：是最基础的频率模型，假设事件独立发生，平均频率为λ。常用于车险事故次数的建模。但泊松分布有一个限制——方差等于均值，而实际数据往往方差大于均值（过度离散，Overdispersion）
  • 负二项分布（Negative Binomial Distribution）：解决了泊松的过度离散问题。通过引入一个额外的"离散参数"，让分布可以拟合方差显著大于均值的实际数据。在大多数非寿险频率建模中，负二项分布比泊松拟合得更好

赔付金额分布

赔付金额分布描述的是"一旦发生理赔，要赔多少钱"。实际数据的典型特征是右偏（Right-skewed）——大部分理赔金额很小，少数大额理赔占了总赔付的很大比例。常用分布包括：

分布 特征 适用场景
伽马分布（Gamma Distribution） 灵活的右偏分布，形态参数可调 一般财产险的损失金额
对数正态分布（Log-normal Distribution） 取对数后为正态，尾部较厚 医疗赔付、责任险小额赔付
帕累托分布（Pareto Distribution） 厚尾分布，极端值概率高 巨灾损失、责任险大额索赔
Burr分布 帕累托的推广形式，尾部更灵活 多场景的损失金额（综合拟合）

实际建模中，精算师通常用复合分布（Compound Distribution）——频率分布和金额分布的乘积——来描述总损失。S = X₁ + X₂ + ... + XN，其中N是理赔次数、Xᵢ是单次赔付金额。这就是著名的集体风险模型（Collective Risk Model）。

集体风险模型:
总损失 S = X₁ + X₂ + ... + XN

其中 N ~ 频率分布（如泊松或负二项）
      Xᵢ ~ 金额分布（如伽马或对数正态）
      N 和 Xᵢ 相互独立

期望总损失 E[S] = E[N] × E[X]

方差 Var(S) = E[N] × Var(X) + Var(N) × (E[X])²

费率厘定

纯保费法

纯保费法（Pure Premium Method）是最直接的非寿险定价方法。纯保费 = 预期总损失 / 承保风险单位数。再纯保费的基础上加上费用附加和利润附加，得到毛保费。

纯保费法的核心步骤：

1. 计算纯保费 PP = 期望损失频率 × 期望损失金额
2. 计算毛保费 GP = PP / (1 - 费用率 - 目标利润率)
3. 举例: PP = 200元, 费用率=30%, 利润率=5%
   GP = 200 / (1 - 0.30 - 0.05) = 200/0.65 ≈ 307.7元

损失率法

损失率法（Loss Ratio Method）主要用于费率调整——如果现有费率下实际损失率高于预期损失率，说明费率需要上调。调整因子 = (实际损失率 - 预期损失率) / 预期损失率。

实务区别：纯保费法是从零开始定一个新费率（"need basis"），损失率法是在旧费率基础上调整（"adjustment basis"）。纯保费法更基础，但需要更详细的损失数据；损失率法更容易操作，但不适用于没有历史费率的新业务。

分类费率与个体费率

非寿险的费率不是一个固定数，而是根据风险因子分层的：

  • 分类费率（Class Rating）：将保单按风险特征（车险中的车型、驾驶年龄、行驶区域；健康险的年龄、性别、吸烟状态等）分成若干类别，每个类别设定一个基础费率
  • 个体费率（Individual Rating / Experience Rating）：在分类费率基础上，根据保单自身的理赔经验进行上调或下调。如果你的车连续三年无事故，保费打折——这就是经验费率

IBNR准备金

链梯法

IBNR（已发生未报案，Incurred But Not Reported）准备金是非寿险精算中最有挑战性的工作之一。一个事故在保险期间内发生了，但被保险人可能几个月甚至几年后才报案——这段时间内的赔款负债需要精算师去估计。

链梯法（Chain-Ladder Method）是最经典的IBNR估计方法。它基于"发展三角形"（Run-off Triangle）——把理赔数据按"事故发生年"和"发展年"排列成一个三角矩阵，然后观察每个事故年的赔付在后续发展年中的演变模式，用这个模式外推未发展完全的赔付金额。

发展三角形示例（累计赔付金额，单位万元）:
事故年     发展年1    发展年2    发展年3    发展年4
2019        1,200      2,100      2,500      2,700
2020        1,350      2,300      2,750      ?
2021        1,500      2,550      ?
2022        1,700      ?

链梯法步骤:
1. 计算每列的发展因子: f2 = 2,100/1,200 = 1.75 (2019年)
                               2,300/1,350 = 1.70 (2020年)
                               2,550/1,500 = 1.70 (2021年)
                               → 平均 f2 ≈ 1.72

2. 同理计算 f3, f4...

3. 用发展因子外推补齐"?"处的金额:
   2020年最终赔付估计 = 2,750 × f4
   2021年最终赔付估计 = 2,550 × f3 × f4
   2022年最终赔付估计 = 1,700 × f2 × f3 × f4

4. IBNR = 最终赔付估计 - 已报案赔付

Bornhuetter-Ferguson法

链梯法的假设是所有事故年的发展模式相同，但如果某个事故年的业务量突然增长或下降，链梯法的估计会出现系统性偏差。Bornhuetter-Ferguson法（简称BF法）解决了这个问题——它结合了链梯法（基于实际数据）和先验期望（基于预期损失率），在两者之间做加权平均。

应用场景：BF法的典型场景是新产品线或高速增长期的IBNR估计。比如一家公司2023年的车险保费比2022年增长了50%，纯用链梯法会低估IBNR（因为2023事故年的发展模式还很不完整），而BF法通过引入预期损失率作为"锚"，能给出更稳健的估计。

准备金充足性测试

中国偿二代监管要求保险公司每季度做准备金充足性测试（Reserve Adequacy Testing）。测试的核心逻辑是：用最新的数据重新估计准备金，如果有缺口，需要立即补提。测试方法通常包括：

  • 用多种方法（链梯、BF、预期损失率法等）独立估算，取合理区间
  • 进行回溯测试——过去估计的准确性如何
  • 考虑贴现因素（如果准备金是贴现后的）——贴现率变化的影响

踩坑故事：一家中型财险公司在2018-2019年期间信用保证保险业务规模暴增，精算团队用链梯法估计IBNR时，没有发现一个严重问题——该业务的理赔报案平均延迟长达9个月（链条末端数据极少），但链梯法基于前几个月的"快赔"数据做了外推，严重低估了最终赔付。2020年该业务出现大量违约后，公司被迫一次性补提了超过10亿元的准备金，直接导致当年亏损。复盘时发现：如果当时至少用一个BF法做交叉验证，或者对不同延迟模式做敏感性分析，这个坑是可以避开的。教训是：永远不要只用一种方法估IBNR。

信度理论

完全信度与部分信度

信度理论（Credibility Theory）解决的是这样一个问题：某个投保人的理赔经验数据，到底在多大程度上"可信"？如果这个客户有10年的数据，他的经验当然可信；如果只有1年的数据，那就不能完全相信他的个人经验，而应该向大类平均水平靠拢（"收缩"）。

  • 完全信度（Full Credibility）：当数据量足够大时（通常用理赔次数阈值判断，如车险要求≥1,082次理赔），完全相信个人经验
  • 部分信度（Partial Credibility）：数据量不够时，用信度因子Z（0<Z<1）在个人经验和先验均值之间做加权平均

Buhlmann模型

Buhlmann模型是最经典的信度模型，它的核心思想是把信度因子Z表达为：

Bühlmann信度模型:
估计值 = Z × 个人经验 + (1 - Z) × 先验均值

信度因子 Z = N / (N + K)

其中:
- N = 个人的理赔次数（或风险单位数）
- K = 信度参数 = E[过程方差] / Var(风险均值)
- 过程方差 = 个体内部随机波动
- 风险均值方差 = 个体之间的真实差异

这个公式的直觉是：当个体差异远大于随机波动时（即"好的风险"和"差的风险"之间的差异很大），K值小，信度因子趋近1，我们应该相信个人经验。反之，如果随机波动远大于个体差异（大家看起来都差不多），我们就应该向先验均值回归。

经验费率

经验费率是信度理论最广泛的应用。在车险中，你的无赔款优待系数（NCD, No Claim Discount）就是信度理论的一个简单版本——连续无事故的年份越多，你的"个人经验"就越可信，保费折扣就越大。在更大规模的企业保险中，经验费率更是核心定价工具：一家工厂的火灾损失记录直接影响它的续保费率。

偿付能力测试中的精算

压力测试

非寿险精算师不仅要做定价和准备金，还要参与偿付能力管理。在偿二代框架下，公司需要评估在各种不利情景下的偿付能力状况。压力测试（Stress Testing）就是模拟极端但合理的情景——比如：

  • 巨灾损失情景：一次超强台风导致大量车险和财产险索赔同时发生
  • 投资市场暴跌：股票市场下跌30%，债券违约率上升
  • 赔付率恶化情景：车险赔付率从60%跳到75%

敏感性分析

敏感性分析（Sensitivity Analysis）比压力测试更精细——它不仅看"会不会死"，还要看"哪个变量对偿付能力的影响最大"。常见的敏感性指标包括：

核心偿付能力充足率的敏感性:
假设变化                       影响幅度
赔付率上升5个百分点             -15%
费用率上升2个百分点              -6%
投资收益率下降100bp             -10%
保费增长率下降10%               -3%
巨灾损失（百年一遇）             -25%

结论：赔付率和投资收益率是最敏感的驱动因子

关键洞察：非寿险精算和寿险精算正在融合。过去它们是两个完全不同的"部落"，各有各的方法论和工具。但在偿二代框架下，两者在偿付能力管理层面开始走向统一——无论寿险还是非寿险，最终都要用同一个语言（经济资本、风险偏好、压力测试）回答同一个问题："公司有没有足够的钱扛过坏年景？"这也是我们下一章要深入的话题。

好了，第二十章到这里。我们从非寿险精算的独特挑战开始，了解了损失分布的建模方法（频率和金额的"双分布"组合），学习了纯保费法和损失率法两种费率厘定路线，掌握了IBNR估计的链梯法和BF法，最后看了信度理论和偿付能力测试中的精算应用。非寿险精算是一个"脏活累活"——数据不完美、方法不完美、但必须在有限信息下做出决策。也正是这种挑战性，让它成为精算领域中最锻炼人的分支。
