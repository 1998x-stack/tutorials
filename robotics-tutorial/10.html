第10章 微分运动学与雅可比

大家好，我是你们的讲师。如果说FK是"机器人的位置"，IK是"位置的反向映射"，那么今天我们要讲的微分运动学（Differential Kinematics）就是"机器人的速度"。而雅可比矩阵（Jacobian Matrix）正是连接关节速度与末端速度的桥梁。

你可能已经在第9章接触过雅可比伪逆——那是数值IK的核心。但雅可比的意义远不止于IK求解。它告诉你：哪个关节对末端运动贡献最大？在什么构型下机器人会失去自由度？末端能承受多大的力？可以说，理解了雅可比，你就理解了机器人在"动"和"受力"两个维度上的全部行为。

10.1 雅可比矩阵的物理含义

雅可比矩阵 J(q) ∈ ℝ^(6×n) 定义了一种速度映射关系：

ν = J(q) · q̇

其中 ν = [v; ω] 是末端空间速度（线速度v + 角速度ω），
q̇ 是关节速度向量。

从几何上看，雅可比的每一列对应一个关节的"瞬时螺旋轴"（Instantaneous Screw Axis）。第i列 J[:,i] 表示：仅当第i个关节以单位速度运动时，末端产生的空间速度。这就是雅可比的物理含义——它把每个关节的运动"投影"到末端运动上。

对于旋转关节，第i列的表达式为：

J[:,i] = | zᵢ₋₁ × (pₑ - pᵢ₋₁) |
         |        zᵢ₋₁         |

其中 zᵢ₋₁ 是第i-1个坐标系中的Z轴方向（在世界坐标系中表示），pₑ 是末端位置，pᵢ₋₁ 是第i-1个关节的位置。这个公式非常直观——它本质上是"力矩 = 力臂 × 力"的刚体运动学版本。

import pinocchio as pin
import numpy as np

model = pin.buildModelFromUrdf("robot.urdf")
data = model.createData()

q = np.array([0.0, -0.5, 0.8, 0.0, 0.5, 0.0])

# 计算雅可比矩阵
pin.forwardKinematics(model, data, q)
pin.updateFramePlacements(model, data)
ee_id = model.getFrameId("tool0")

# 计算局部雅可比（body-fixed frame）
J = pin.computeFrameJacobian(model, data, q, ee_id, pin.ReferenceFrame.LOCAL)

print("雅可比矩阵形状:", J.shape)  # (6, nq)
print("线速度部分:\n", J[:3, :])
print("角速度部分:\n", J[3:, :])

10.2 几何雅可比 vs 分析雅可比

需要注意的是，雅可比有两种不同的定义方式：

  • 几何雅可比（Geometric Jacobian）：末端速度用 [v; ω] 表示，其中 ω 是瞬时角速度矢量。这是Pinocchio等物理引擎使用的形式。
  • 分析雅可比（Analytical Jacobian）：末端速度用姿态参数的时间导数表示（如欧拉角速率、四元数导数）。形式取决于姿态参数化方式。

两者的关系是：J_analytic = E(φ) · J_geometric，其中 E(φ) 是将角速度映射到姿态参数导数的变换矩阵。举一个具体的例子——如果用ZYX欧拉角 (α, β, γ) 表示姿态：

| α̇ |   | -sinβ·cosγ   cosγ   0 |   | ωₓ |
| β̇ | = |  sinβ·sinγ  -sinγ   0 | · | ωᵧ |
| γ̇ |   |     cosβ      0     1 |   | ω_z |

何时用哪种？如果你做的是控制（输出关节力矩），用几何雅可比，因为物理量是矢量。如果你做的是规划（输出末端轨迹），用分析雅可比通常更方便，因为姿态参数的时间导数更直观。不过大多数现代框架（Pinocchio、Kindr）都默认使用几何雅可比——减少姿态参数化带来的奇异性。

10.3 奇异性分析

雅可比矩阵的秩直接反映了机器人在当前构型下的运动能力。当雅可比不满秩（rank(J) < 6）时，机器人处于奇异位形，此时在某些方向上失去运动能力。

常见的奇异位形类型：

类型 物理含义 常见场景 检测方法
边界奇异 手臂完全伸直或折叠 肘关节全伸展 检查关节极限
腕部奇异 第4轴和第6轴共线 第5轴接近0° 检查sin(θ₅) ≈ 0
肩部奇异 腕部中心位于肩关节正上方 第1轴和第4轴共线 检查条件数

最常用的奇异性检测方法是计算雅可比的条件数（Condition Number）：

cond(J) = σ_max / σ_min

其中 σ_max 和 σ_min 是J的最大和最小奇异值。

当条件数趋近无穷大时，机器人处于奇异位形。实践中，通常设置一个阈值（如 cond > 1000）来判定接近奇异区域。

# 奇异性检测示例
def check_singularity(J, threshold=1000):
    _, s, _ = np.linalg.svd(J)
    cond_num = s[0] / s[-1]
    min_sv = s[-1]

    if cond_num > threshold:
        return True, cond_num, min_sv  # 接近奇异
    return False, cond_num, min_sv

# 测试不同构型
for test_name, q_test in [("中立位姿", np.zeros(6)),
                            ("接近奇异", np.array([0, 0, 0.5, 0, 0.01, 0]))]:
    pin.forwardKinematics(model, data, q_test)
    J_test = pin.computeFrameJacobian(model, data, q_test,
                                       ee_id, pin.ReferenceFrame.LOCAL)
    is_sing, cond, sv_min = check_singularity(J_test)
    print(f"{test_name}: cond={cond:.1f}, min_sv={sv_min:.4f}, "
          f"singular={is_sing}")

避坑指南：我在调试一个焊接机器人时，发现在某个特定焊接路径上机器人总是"卡顿"一下，然后继续运动。用SVD分析后发现，在那个点雅可比的最小奇异值降到了0.02（正常值约2.0），条件数突破了1000。根本原因是焊枪姿态要求导致第5轴几乎为零，进入了腕部奇异。解决方案很简单——在规划阶段绕过奇异区域，而不是指望控制器在奇异点附近"硬撑"。记住：奇异位形不是控制能解决的问题，它是路径规划阶段就应该回避的。

10.4 可操作度

Yoshikawa在1985年提出了可操作度（Manipulability）的概念，用一个标量衡量机器人在当前构型下的"运动灵活性"：

w = √(det(J · Jᵀ)) = σ₁ · σ₂ · ... · σₘ

可操作度 w 越大，机器人在各个方向上的运动能力越均衡。可操作度为零意味着处于奇异位形。将这个标量可视化，就得到了可操作度椭球（Manipulability Ellipsoid）——椭球的轴方向由奇异向量决定，轴长由奇异值决定。

可操作度还有一个有趣的"对偶性"：速度可操作度椭球和力可操作度椭球互为对偶。如果在某个方向上运动速度很快（长轴），那么在同一个方向上能输出的力就很小（短轴）。这就是著名的速度-力对偶原理。

直觉理解：想象你的手臂完全伸直去推一堵墙——你能输出很大的力（力椭球在这个方向上很长），但你的手在这个方向上几乎不能移动（速度椭球在这个方向上很短）。反之，当你的肘部弯曲90°时，手可以快速移动（速度椭球大），但推力很小。这个对偶性在设计机器人操作任务时至关重要——重载搬运需要"力优势"构型，精密装配需要"速度优势"构型。

10.5 力域中的雅可比

雅可比不仅在速度域中起作用，在力域中同样关键。利用虚功原理（Virtual Work Principle），我们可以建立关节力矩 τ 与末端力 F 之间的关系：

τ = Jᵀ · F

这个公式的物理意义极为深刻：雅可比的转置将末端笛卡尔力映射为关节力矩。这是力控制、阻抗控制、混合力位控制的数学基础。想象一下：如果你的机器人末端需要施加10N的力，通过 Jᵀ 映射到各个关节上，就得到了每个关节应该输出的力矩。不需要显式计算动力学——只需要雅可比。

# 力域实例：重力补偿
pin.computeJointJacobians(model, data, q)
pin.framesForwardKinematics(model, data, q)

# 假设末端受到 5N 的力沿 Z 方向
F_ee = np.array([0.0, 0.0, -5.0, 0.0, 0.0, 0.0])

# 映射到关节力矩
J = pin.computeFrameJacobian(model, data, q, ee_id, pin.ReferenceFrame.LOCAL)
tau_gravity = J.T @ F_ee

print("重力补偿关节力矩:", tau_gravity)

在实际工程中，当机器人需要与环境交互（如打磨、装配、人机协作）时，Jᵀ 映射是力控制的基础。结合力/力矩传感器，可以实现精确的力位混合控制——这也是第14章的主题之一。

本章我们学习了雅可比矩阵的物理意义——它是关节空间与任务空间之间速度和力的桥梁。我们探讨了几何雅可比与分析雅可比、奇异性的检测与规避、可操作度的概念，以及 τ = Jᵀ·F 的力映射关系。雅可比是运动学与动力学之间的"转接器"——有了它，我们才能讨论机器人到底需要多大的力才能动起来。
