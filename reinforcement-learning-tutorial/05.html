第五章：动态规划

大家好，我是你们的讲师。在前两章中，我们学习了马尔可夫决策过程（MDP）和贝尔曼方程——它们是理解强化学习问题的数学语言。从这一章开始，我们将真正动手"求解"MDP，也就是找出最优策略。

本章要讲的是动态规划（Dynamic Programming, DP）。你可能会觉得"动态规划"这个名字听起来像是算法竞赛或者最优化理论里的概念，没错——它确实是这些领域的基础方法。但有趣的是，强化学习中最早的、也是最优雅的一类求解算法，正是建立在动态规划的思想之上的。

不过这里有一个重要的前提，请大家先记住：经典动态规划要求我们完全知道环境的模型。也就是说，我们必须知道状态转移概率 P 和奖励函数 R。在后续章节中，我们会一步步去掉这个假设，但今天，我们先在这个"已知模型"的理想世界里打好基础。

5.1 动态规划的核心前提 —— 已知模型

在强化学习的语境下，动态规划指的是一类在已知 MDP 模型的前提下求解最优策略的方法。这里的"已知模型"具体指：

  • 状态转移概率：P(s' | s, a) —— 在状态 s 执行动作 a 后，转移到 s' 的概率。
  • 奖励函数：R(s, a, s') —— 每次转移获得的即时奖励。

有了这两样东西，我们实际上拥有了环境的完整描述。此时求解最优策略的任务，在术语上被称为"规划"（planning），以区别于后面章节中不依赖模型的"学习"（learning）。规划像是在手拿地图的情况下找最短路径；学习则是在没有地图的情况下，通过不断试错来摸索出路径。

关键区分：
规划（Planning）：模型已知 → 通过计算得到最优策略。
学习（Learning）：模型未知 → 通过与环境的交互和经验来逼近最优策略。
本章全部属于"规划"范畴。

动态规划在强化学习中的应用主要围绕两个核心问题：

  1. 预测（Prediction）：给定一个策略 π，计算它的价值函数 Vπ 或 Qπ。
  2. 控制（Control）：通过不断改进策略，找到最优策略 π* 和最优价值函数 V*。

这两个问题对应了接下来的几节内容：策略评估、策略改进、策略迭代和价值迭代。

5.2 策略评估（Policy Evaluation）

策略评估的任务是：给定一个策略 π，计算它的状态价值函数 Vπ。

还记得上一章介绍的贝尔曼期望方程吗？对任意 s ∈ S：

Vπ(s) = Σa π(a|s) Σs' P(s'|s,a) [R(s,a,s') + γ Vπ(s')]

这是一个方程组——|S| 个未知数、|S| 个方程。理论上我们可以直接求解这个线性方程组（想象一下，如果 |S| = 10⁶ 呢？那么就需要解一个 100 万 × 100 万的矩阵方程——这在计算上是不现实的）。

因此，动态规划采用迭代法：从一个初始猜测 V0 出发，反复应用贝尔曼期望方程作为更新规则，让 V 逐渐收敛到 Vπ。

Vk+1(s) ← Σa π(a|s) Σs' P(s'|s,a) [R(s,a,s') + γ Vk(s')]

这个更新被称为贝尔曼期望备份（Bellman Expectation Backup）。它把旧的 V 值"备份"成新的 V 值。可以证明，在 γ < 1 的条件下，这个迭代过程保证收敛到唯一的 Vπ。

一个曾让我困惑好几天的细节：
我在第一次实现策略评估时，犯了一个非常基础的错误——我把 V 数组的更新做成了"就地更新"（in-place），即用新值直接覆盖旧值：V[s] = sum(...)。但实际上，正确的做法是同时更新（synchronous update）：用上一轮的全部旧值计算出本轮的全部新值，然后一次性替换。如果你使用就地更新，算法仍然可能收敛，但收敛到的解可能不是期望的那个（取决于更新顺序）。对于强迫症来说，这个 bug 让我排查了整整两天。永远记住：做策略评估时，维护一个 V_new 数组，计算完所有状态后再整体赋值给 V_old。

网格世界示例：

让我们用一个 4×4 的网格世界来直观感受一下。假设智能体从左上角出发，目标是右下角（奖励 +1），其余每一步的奖励为 -0.1。动作有四个：上、下、左、右，每个动作以 0.8 的概率走向目标方向，0.1 的概率走向左右两边（随机转移）。

假设我们评估一个完全随机的策略（每个动作概率 0.25）。下面是在不同迭代轮次下部分状态的 V 值：

状态坐标 V0 V1 V2 V3 V10（几乎收敛）
(0,0) 起点 0.00 -0.10 -0.21 -0.34 -2.15
(0,1) 0.00 -0.10 -0.18 -0.28 -1.87
(2,2) 0.00 -0.10 -0.12 -0.15 -0.94
(3,3) 目标 0.00 1.00 0.90 0.91 0.92

可以看到，价值信息从目标状态开始"反向传播"，逐渐扩散到整个状态空间。这其实就是动态规划的核心直觉——利用后继状态的价值来更新当前状态的价值。

5.3 策略改进（Policy Improvement）

策略评估告诉我们一个策略有多好，但它没有告诉我们如何变得更好。这就是策略改进要做的事情。

假设我们已经用策略评估计算出了当前策略 π 的价值函数 Vπ。现在，对于某个状态 s，我们考虑一个关键问题：如果在状态 s 执行一个不同于 π(s) 的动作 a，然后之后仍然遵循 π，这样做会不会更好？

这个"不遵循 π 一次"的价值，就是动作价值 Qπ(s, a)：

Qπ(s,a) = Σs' P(s'|s,a) [R(s,a,s') + γ Vπ(s')]

如果对于某个状态 s，存在一个动作 a ≠ π(s) 使得 Qπ(s, a) > Vπ(s)，那么我们就找到了一个比 π 更好的选择。策略改进定理保证：通过在每一个状态贪婪地选择 Q 值最大的动作，我们构造出的新策略 π' 至少和 π 一样好，而且严格更好（如果存在至少一个状态可以改进）。

π'(s) = argmaxa Qπ(s,a)
        = argmaxa Σs' P(s'|s,a) [R(s,a,s') + γ Vπ(s')]

这个过程称为策略改进（Policy Improvement）。直觉上非常清晰：既然我们知道每个动作的长期价值，那当然每次选价值最高的那个动作。

策略改进定理的直观理解：
想象你每天都在同一条路线上班。策略评估是计算你当前路线的平均通勤时间。策略改进则是看看"如果我在某个路口拐另一条路，之后还是按原来的路线走，会不会更快？"如果发现了更快的方式，你就更新路线。重复这个过程，最终你会找到最优通勤路线。

5.4 策略迭代（Policy Iteration）

策略迭代把评估和改进组合成一个完整的控制算法：

  1. 初始化：随机选择一个策略 π0。
  2. 策略评估：计算当前策略 πi 的价值函数 Vπi（迭代至收敛）。
  3. 策略改进：基于 Vπi 构造新策略 πi+1，在每个状态选择贪婪动作。
  4. 如果 πi+1 = πi，停止；否则回到步骤 2。

这个过程就像爬山：每一步都爬到一个更高的点（更好的策略），直到到达山顶（最优策略）。因为有限 MDP 中只有有限个确定性策略，策略迭代保证在有限步内收敛到最优策略。

算法伪代码：

1. 初始化：对任意 s ∈ S, V(s) = 0, π(s) = 随机动作
2. 重复直到策略稳定：
   a. 策略评估：
      重复直到 V 收敛：
        Δ ← 0
        对于每个 s ∈ S：
          v ← V(s)
          V(s) ← Σa π(a|s) Σs' P(s'|s,a) [R + γ V(s')]
          Δ ← max(Δ, |v - V(s)|)
        如果 Δ < θ（阈值），跳出循环
   b. 策略改进：
      policy_stable ← True
      对于每个 s ∈ S：
        old_action ← π(s)
        π(s) ← argmaxa Σs' P(s'|s,a) [R + γ V(s')]
        如果 old_action ≠ π(s)，policy_stable ← False
   如果 policy_stable = True，停止；否则回到步骤 a

复杂度分析：策略迭代的每次外层循环包含一次完整的策略评估，而策略评估本身需要 O(|S|²|A|) 每次迭代（假设所有状态可相互转移）。如果策略评估需要 k 次迭代，策略迭代需要 m 次改进，则总复杂度为 O(m · k · |S|² · |A|)。在实践中最坏情况下 m 随 |S| 指数增长，但在大多数实际 MDP 中 m 通常很小。

5.5 价值迭代（Value Iteration）

策略迭代的一个明显问题是：每次策略改进前，我们都需要把策略评估做"完全收敛"。这有必要吗？

答案是否定的。价值迭代的核心思想是：只做一次策略评估的更新，就立即做策略改进。换句话说，我们把策略评估截断为一次扫描（one sweep）。

价值迭代的更新规则非常简洁：

V(s) ← maxa Σs' P(s'|s,a) [R(s,a,s') + γ V(s')]

注意这里的区别：策略评估用的是期望（对 π 加权平均），而价值迭代用的是最大化（取所有动作中的最大值）。实际上，价值迭代直接对应贝尔曼最优方程：

V*(s) = maxa Σs' P(s'|s,a) [R(s,a,s') + γ V*(s')]

Python 伪代码实现：

def value_iteration(P, R, gamma=0.99, theta=1e-6):
    """
    P: dict - P[s][a][s'] = 转移概率
    R: dict - R[s][a][s'] = 奖励
    gamma: 折扣因子
    theta: 收敛阈值
    """
    # 初始化价值函数
    V = {s: 0.0 for s in P.keys()}

    while True:
        delta = 0.0
        for s in P.keys():
            v_old = V[s]
            # 计算所有动作的 Q 值，取最大值
            max_q = float('-inf')
            for a in P[s].keys():
                q = 0.0
                for s_next, prob in P[s][a].items():
                    reward = R[s][a][s_next]
                    q += prob * (reward + gamma * V[s_next])
                max_q = max(max_q, q)
            # 更新 V[s] 为最大 Q 值
            V[s] = max_q
            delta = max(delta, abs(v_old - V[s]))
        if delta < theta:
            break

    # 从最优 V 中提取最优策略
    policy = {}
    for s in P.keys():
        best_a = None
        best_q = float('-inf')
        for a in P[s].keys():
            q = 0.0
            for s_next, prob in P[s][a].items():
                reward = R[s][a][s_next]
                q += prob * (reward + gamma * V[s_next])
            if q > best_q:
                best_q = q
                best_a = a
        policy[s] = best_a

    return V, policy

策略迭代 vs 价值迭代：

对比维度 策略迭代 价值迭代
更新对象 策略 π 和价值 V 仅价值 V
每次迭代的计算量 策略评估需多次扫描直到收敛（较慢） 只做一次扫描（较快）
收敛所需的迭代次数 通常较少（策略快速变好） 通常较多（需要更多轮次）
总计算时间 策略评估开销大，可能总时间较长 每轮快，但可能需要更多轮
适用场景 状态空间小，需要精确策略 状态空间大，每轮计算需要轻量
停止条件 策略不再变化 价值函数变化小于阈值

实际中如何选择？我的经验是：如果状态空间适中（如几万以下）且你对策略的精确性有要求，策略迭代通常更快收敛；如果状态空间很大，或者你对每轮的计算预算有限，价值迭代是更好的选择。还有一种中间方案——截断策略迭代（Truncated Policy Iteration），它在策略评估中只做固定次数的扫描（比如 3 次或 5 次），而不是完全收敛，在实践中往往效果很好。

关于"规划"的思考：
动态规划揭示了强化学习中一个深刻的洞察：最优解可以通过反复利用"后继状态的价值"来"回溯"得到。这个"自举"（bootstrapping）的思想贯穿了后续几乎所有的强化学习算法。当你理解了 DP，你就理解了 RL 中最核心的"价值传播"机制。第 7 章要学的时序差分学习，本质就是在模型未知的情况下做同样的事。

到目前为止，我们一直在强调动态规划需要已知的模型。但在现实世界的强化学习问题中，我们几乎从来不会事先知道 P 和 R。那么该怎么办？

答案在下一章。蒙特卡洛方法告诉我们：即使没有模型，只要让智能体与环境交互、收集经验样本，我们依然能够评估策略、寻找最优解。蒙特卡洛方法放弃了"自举"（bootstrapping），转而使用完整的 episode 回报来估计价值。它虽然效率可能不如 DP，但它迈出了"脱离模型"的第一步——这也是通向现代深度强化学习的起点。
