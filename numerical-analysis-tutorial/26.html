第26章 常微分方程数值解(二)

26.1 Runge-Kutta 方法的一般思想

大家好，我是你们的讲师。上一章我们学习了 Euler 法和改进 Euler 法。你可能会想：有没有一种系统性的方法，可以构造任意高阶的数值方法？答案是：有，这就是 Runge-Kutta（龙格-库塔）方法家族。

Runge-Kutta 方法的核心思想是：在一个步长区间 [tₙ, tₙ₊₁] 内，采样多个点的斜率并取加权平均，以更精确地逼近 y(t) 的真实变化。

一个典型的 s 级显式 Runge-Kutta 方法的形式是：

k₁ = h · f(tₙ, yₙ)
k₂ = h · f(tₙ + c₂h, yₙ + a₂₁k₁)
k₃ = h · f(tₙ + c₃h, yₙ + a₃₁k₁ + a₃₂k₂)
...
kₛ = h · f(tₙ + cₛh, yₙ + aₛ₁k₁ + aₛ₂k₂ + ... + aₛ,ₛ₋₁kₛ₋₁)
yₙ₊₁ = yₙ + b₁k₁ + b₂k₂ + ... + bₛkₛ

系数 (A, b, c) 通常被整理成 Butcher 表（Butcher Tableau）的形式：

c A
bT

更具体地：

0
c₂ a₂₁
c₃ a₃₁ a₃₂
⋮ ⋮ ⋮ ⋱
cₛ aₛ₁ aₛ₂ ... as,s-1
b₁ b₂ ... bₛ

系数必须满足一致性条件 Σbᵢ = 1 和 cᵢ = Σaij（对于 i = 2, ..., s）。更高阶的条件来自对 Taylor 展开的匹配，随着阶数的增加，条件数量迅速增长——这就是为什么构造高阶 RK 方法是一件很有技巧性的事。

Butcher 表的直观理解：表中的 c 表示在区间内的采样位置，A 的每一行是计算该采样点斜率时对各已知斜率的权重，b 是最终输出时的加权系数。如果你看 Heun 法的 Butcher 表：c₂ = 1, a₂₁ = 1, b = [1/2, 1/2]——正好对应"先用 Euler 预测，再平均两个斜率"。

26.2 经典四阶 Runge-Kutta（RK4）

在所有 Runge-Kutta 方法中，最著名、最常用、最"传奇"的就是经典四阶 Runge-Kutta 方法（RK4）。它的公式是：

k₁ = h · f(tₙ, yₙ)
k₂ = h · f(tₙ + h/2, yₙ + k₁/2)
k₃ = h · f(tₙ + h/2, yₙ + k₂/2)
k₄ = h · f(tₙ + h, yₙ + k₃)
yₙ₊₁ = yₙ + (k₁ + 2k₂ + 2k₃ + k₄) / 6

RK4 的局部截断误差是 O(h⁵)，全局误差 O(h⁴) ——所以叫"四阶"方法。它的权重 1:2:2:1 对应 Simpson 求积公式的系数，这不是巧合——RK4 本质上是用 Simpson 规则来近似积分 ∫f dt。

import numpy as np

def rk4(f, y0, t_span, h):
    """经典四阶 Runge-Kutta 方法"""
    t0, tf = t_span
    n_steps = int((tf - t0) / h)
    t = np.linspace(t0, tf, n_steps + 1)
    y = np.zeros(n_steps + 1)
    y[0] = y0

    for i in range(n_steps):
        k1 = h * f(t[i], y[i])
        k2 = h * f(t[i] + h/2, y[i] + k1/2)
        k3 = h * f(t[i] + h/2, y[i] + k2/2)
        k4 = h * f(t[i] + h, y[i] + k3)
        y[i+1] = y[i] + (k1 + 2*k2 + 2*k3 + k4) / 6

    return t, y

# 测试：谐振子 y'' = -y, 转化为一阶系统
# 令 u = [y, y']^T, u' = [u[1], -u[0]]^T
def harmonic_oscillator(t, u):
    return np.array([u[1], -u[0]])

u0 = np.array([1.0, 0.0])  # 初始位置 1, 初始速度 0
t_span = (0.0, 2*np.pi)
h = 0.01

t_rk4, u_rk4 = rk4(harmonic_oscillator, u0, t_span, h)
# 解析解: y(t) = cos(t)
err = np.max(np.abs(u_rk4[:, 0] - np.cos(t_rk4)))
print(f"RK4 最大误差 (h={h}): {err:.2e}")

# 对比 Euler 法
def euler_ode(f, y0, t_span, h):
    t0, tf = t_span
    n = int((tf - t0) / h) + 1
    t = np.linspace(t0, tf, n)
    y = np.zeros((n, len(y0)))
    y[0] = y0
    for i in range(n-1):
        y[i+1] = y[i] + h * f(t[i], y[i])
    return t, y

t_e, u_e = euler_ode(harmonic_oscillator, u0, t_span, h)
err_euler = np.max(np.abs(u_e[:, 0] - np.cos(t_e)))
print(f"Euler 最大误差 (h={h}): {err_euler:.2e}")

# 展示阶数验证
for h_test in [0.1, 0.05, 0.025, 0.0125]:
    _, u = rk4(harmonic_oscillator, u0, t_span, h_test)
    e = np.max(np.abs(u[:, 0] - np.cos(t := np.linspace(0, 2*np.pi, int(2*np.pi/h_test)+1))))
    print(f"h={h_test:.4f}: RK4 误差={e:.2e}, 比值={e/(h_test**4):.2f}")
    # 期望：误差 / h^4 接近常数，验证四阶收敛

NASA Apollo 的传奇故事：你知道吗？1969 年阿波罗 11 号登月任务中，宇航员在月球着陆的最后阶段，导航计算机执行的就是 Runge-Kutta 积分。当时的计算机——Apollo Guidance Computer——主频只有 2 MHz，内存只有 4 KB。但它成功地在实时环境下对飞船的运动方程做了数值积分，以每秒几十次的速度更新位置和速度估计。用的就是 RK4 方法。这个例子告诉我们：好的数值方法 + 有限的算力 = 可行的工程方案。今天你的手机拥有比阿波罗计算机强百万倍的计算能力——但那些 RK4 算法几乎没有任何改变。

26.3 自适应步长控制

在实际工程中，固定的步长 h 几乎总是一个坏主意——因为解的光滑性在不同时间可能完全不同。在解变化缓慢的区域，大步长就够；在解陡峭变化的区域，必须用小步长来捕捉细节。这就是自适应步长控制（Adaptive Step Size Control）的用武之地。

基本思想是：在每一步中，用两种不同阶数的方法计算 yₙ₊₁，用它们的差来估计局部误差，然后根据误差决定是否接受这一步以及下一步用什么步长。

Fehlberg 的 4(5) 对（RK45）：Fehlberg 在 1969 年设计了一组巧妙的系数——使用6 个 k 同时计算出四阶和五阶的结果（共享 k 值，只加权系数不同）。两者的差给出误差估计。这就是 MATLAB 早期版本中 ode45 的基础。

Dormand-Prince 5(4)（dopri5）：这是现代标准——默认就在 scipy.integrate.solve_ivp 中使用。和 Fehlberg 类似，但系数设计不同，使得五阶公式的误差系数更小，并额外提供密集输出（dense output）功能（可在任意点插值而不需要重新积分）。

自适应步长控制的一般策略（步长加倍/减半）：

  1. 用当前步长 h 计算一步，得到误差估计 ε。
  2. 如果 ε > tol（容差），拒绝这一步，将 h 减半，重试。
  3. 如果 ε 远小于 tol，接受这一步，并将下一步的 h 放大（但不超过 2 倍）。
  4. 如果 ε 在可接受范围内，接受这一步，保持 h 不变或微调。

自适应方法的"甜蜜点"：自适应步长不仅提高了可靠性，还节省了计算量。对于一个在大部分区域光滑、只有少数区域陡峭的 ODE，自适应方法自动在光滑区域用大步长、陡峭区域用小步长，比固定步长方法效率高数十倍甚至更多。实际经验表明，对于大多数非刚性问题，scipy.integrate.solve_ivp 的默认设置（RK45，容差 1e-3）就能给出满意的结果——远比手动调步长更省心。

26.4 稳定性分析

现在我们讨论最重要的理论问题之一——数值稳定性（Numerical Stability）。你可能觉得："既然方法精度是四阶，那肯定没问题。" 错。一个方法即使理论精度很高，如果数值不稳定，结果也是垃圾。

稳定性分析的标准方法是研究测试方程（Test Equation）：

y' = λy, λ ∈ ℂ, Re(λ) < 0

这个简单的线性方程的解 y(t) = y₀eλt 是单调衰减的。我们用数值方法求解它，然后看数值解是否也衰减（而不是振荡增长）。

定义稳定性函数（Stability Function）R(z) = yₙ₊₁/yₙ，其中 z = hλ。对于不同的方法，R(z) 不同：

方法 稳定性函数 R(z) 绝对稳定区域 实轴上的稳定区间
向前 Euler 1 + z 以 (-1, 0) 为圆心、半径 1 的圆 [-2, 0]
向后 Euler 1 / (1 - z) 整个复平面去掉圆心 (1, 0) 的圆 (-∞, 0]（A-稳定）
梯形法 (1 + z/2) / (1 - z/2) 整个左半平面 (-∞, 0]（A-稳定）
RK2（改进 Euler） 1 + z + z²/2 复杂形状 ≈ [-2, 0]
RK4 1 + z + z²/2 + z³/6 + z⁴/24 复杂形状 ≈ [-2.78, 0]

绝对稳定区域（Region of Absolute Stability）定义为满足 |R(z)| ≤ 1 的所有 z = hλ 的集合。对于给定的 λ，这个区域直接将步长 h 限制在某个范围内。如果 h 超出这个范围，数值解就会振荡发散——即使真实解是单调衰减的。

一个惨痛的稳定性教训：几年前，我帮一个团队调试一个化学反应网络模型。模型包含一个快速反应，时间常数 τ = 10⁻⁶ 秒（λ = -10⁶）。团队用 RK4 方法，步长 h = 10⁻⁴ 秒——看起来"远小于"反应时间尺度？错！|hλ| = 100 ≫ 2.78，远远超出 RK4 的稳定区域。结果数值解疯了一样振荡到 ±10²⁰，然后 NaN。更危险的是，团队在调试过程中尝试减小步长，逐渐从 10⁻⁴ 减到 10⁻⁵，仍在振荡。直到 h ≤ 2.78 × 10⁻⁶ 才稳定——这就是为什么稳定性分析不是理论游戏，而是工程生命线。

显式 vs 隐式的稳定性博弈：从这个表格可以看出一条关键分界线：所有显式方法的稳定区域都是有界的（在实轴上最多到 -2 附近），而隐式方法（向后 Euler、梯形法）的稳定区域覆盖整个左半平面——这个性质叫 A-稳定性（A-Stability）。这就是为什么对于刚性方程（Stiff Equations）（包含快变和慢变两种时间尺度的系统），必须使用隐式方法。这个主题将是我们第 27 章的核心内容。

让我们用代码来直观感受一下稳定性边界的影响：

import numpy as np

# 测试稳定性：y' = -100y, y(0) = 1
# 稳定条件：h ≤ 2/100 = 0.02 (Euler), h ≤ 2.78/100 = 0.0278 (RK4)
def stability_test(method, h, lam=-100.0, T=1.0, y0=1.0):
    """测试不同步长下的稳定性"""
    n_steps = int(T / h)
    y = y0
    for _ in range(n_steps):
        if method == 'euler':
            y = y + h * lam * y
        elif method == 'rk4':
            k1 = h * lam * y
            k2 = h * lam * (y + k1/2)
            k3 = h * lam * (y + k2/2)
            k4 = h * lam * (y + k3)
            y = y + (k1 + 2*k2 + 2*k3 + k4) / 6
    return y

lams = [-100, -10, -1]
hs = [0.001, 0.005, 0.01, 0.02, 0.03, 0.05]

print("Euler 法稳定性测试 (y' = λy, t=1):")
for lam in lams:
    for h in hs:
        y_approx = stability_test('euler', h, lam=lam)
        y_exact = np.exp(lam * 1.0)
        stable = "✅" if abs(y_approx) < 1e3 else "❌"
        print(f"  λ={lam:4d}, h={h:.3f}, |hλ|={abs(h*lam):5.2f}: "
              f"y={y_approx:.4f}, 稳定={stable}")

print("\nRK4 稳定性测试:")
for lam in lams:
    for h in hs:
        y_approx = stability_test('rk4', h, lam=lam)
        y_exact = np.exp(lam * 1.0)
        stable = "✅" if abs(y_approx) < 1e3 else "❌"
        print(f"  λ={lam:4d}, h={h:.3f}, |hλ|={abs(h*lam):5.2f}: "
              f"y={y_approx:.4f}, 稳定={stable}")

运行这个代码你会看到：当 |hλ| 超过稳定边界时，数值解会剧烈振荡，产生完全不合理的巨大数值。这种发散不是程序 bug，而是方法本身的数学性质决定的。

好，这一章的内容到这里就结束了。我们从 Runge-Kutta 方法的一般框架开始，学习了 RK4 这个"工作马"方法，了解了自适应步长控制的思想，最后深入分析了数值稳定性这个关键概念。你可能会觉得稳定性那一节有点"重"——但相信我，它是所有这些方法中最值得你记住的部分。下一章，我们将进入常微分方程数值解的最后一个主题：刚性方程与线性多步法。在那里，稳定性的重要性会更加凸显。到时候见。
