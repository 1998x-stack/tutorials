"记忆只有一分钟"的随机过程

大家好，我是你们的讲师。到目前为止，我们讨论的随机过程——高斯过程、白噪声、窄带信号——大多数在时间维度上具有"长程记忆"：协方差函数描述了不同时刻之间的相关关系，一个时刻的值往往与它之前的所有时刻都存在统计关联。但在现实世界中，有很多序列数据并不具备这种复杂的记忆结构。

考虑一个例子：你正在刷短视频。推荐算法根据你当前正在看的视频类型（健身、美食、宠物），预测你下一个想看的视频。它需要知道你一个小时前看的10个视频吗？不一定——事实上，你最可能的下一个选择，几乎完全取决于你当前正在看什么。这就是"无后效性"的直观体现：给定现在，未来与过去无关。这种"记忆只有一分钟"的随机过程，就是马尔可夫链（Markov chain）。

马尔可夫链是随机信号分析和机器学习中最重要的基石模型之一。它的应用范围极其广泛：从语音识别中的声学建模、自然语言处理中的N-gram语言模型，到通信中的信道编码译码、强化学习中的状态转移建模，再到Google的PageRank算法——这些大名鼎鼎的技术，核心都建立在一个看似简单的假设之上："下一时刻的状态只依赖于当前状态。"本章我们就来深入理解这个优雅而强大的模型。

无后效性：马尔可夫性质

最简单的依赖结构

严格定义：一个离散时间随机过程 {Xₙ, n = 0, 1, 2, ...} 满足马尔可夫性质（Markov property），如果对任意时间 n 和任意状态 i₀, i₁, ..., iₙ, i_{n+1}：

P(X_{n+1}=i_{n+1} | X₀=i₀, X₁=i₁, ..., Xₙ=iₙ) = P(X_{n+1}=i_{n+1} | Xₙ=iₙ)

"给定现在，未来与过去独立"——这就是马尔可夫性质的核心思想。这个看似简单的条件，实际上极大地简化了分析。它意味着整个过程的概率结构完全由两个要素确定：

  • 初始分布：π₀(i) = P(X₀ = i)，过程在时刻0处于各状态的概率
  • 转移概率：P_{i→j} = P(X_{n+1}=j | Xₙ=i)，从状态i转移到状态j的概率（假设时齐，即不依赖于n）

有了这两个要素，我们可以计算出任意时刻、任意状态序列的概率：

P(X₀=i₀, X₁=i₁, ..., Xₙ=iₙ) = π₀(i₀) · P_{i₀→i₁} · P_{i₁→i₂} · ... · P_{i_{n-1}→iₙ}

这就是马尔可夫链的"链式"结构的来源——每一步只依赖前一步，像链条一样一环扣一环。

直觉：马尔可夫性质不是"无记忆"——它是有记忆的，只是记忆只有"一步"。如果你把一个系统建模为马尔可夫链，你其实是在说："当前状态已经包含了预测未来所需的全部历史信息。"如果这个假设成立，那么任何更久远的历史都不会提供额外的预测能力。

何时可以使用马尔可夫假设？

马尔可夫性质在很多场景中自然成立：

  • 物理系统：一个粒子的位置和速度（状态）完全决定了下一刻的位置和速度（牛顿力学本质上是马尔可夫的）
  • 排队系统：队列当前的长度决定了下一时刻长度的概率分布
  • 随机游走：当前位置加上随机步长就是下一位置，不需要知道过去怎么走过来的
  • 有限状态机：下一状态只取决于当前状态和输入

当然，也有很多场景马尔可夫假设不成立——我们会在本章最后的踩坑故事中讨论这一点。

转移概率与C-K方程

一步转移概率矩阵

对于状态空间为 S = {1, 2, ..., M} 的马尔可夫链，我们将所有一步转移概率组织成一个 M×M 矩阵：

P = [P_{ij}]，其中 P_{ij} = P(X_{n+1}=j | Xₙ=i)

P 是一个随机矩阵（stochastic matrix）：
① 每个元素非负：P_{ij} ≥ 0
② 每行和为1：∑ⱼ P_{ij} = 1   （从状态i出发，总会到某个状态）

随机矩阵是马尔可夫链的"发动机"——给定当前状态的分布向量 πₙ，下一时刻的分布就是 π_{n+1} = πₙ · P。如果 π₀ 是初始分布，那么 n 步后的分布就是 πₙ = π₀ · Pⁿ。

n步转移概率与C-K方程

定义 n 步转移概率：P^{(n)}_{ij} = P(X_{m+n}=j | Xₘ=i)，即从状态 i 出发经过 n 步到达状态 j 的概率。关键结论：

P^{(n)} = Pⁿ，即 n 步转移概率矩阵等于一步转移概率矩阵的 n 次幂。

切普曼-科尔莫戈罗夫方程（Chapman-Kolmogorov, C-K方程）描述了多步转移的分解关系：

P^{(m+n)}_{ij} = Σₖ P^{(m)}_{ik} · P^{(n)}_{kj}

矩阵形式：P^{(m+n)} = P^{(m)} · P^{(n)} = P^{m+n}

这个方程的直观含义是：
从状态i出发经过 m+n 步到达状态j
= 先走 m 步到某个中间状态 k，再走 n 步到 j
= 对所有可能的中间状态 k 求和

C-K方程是马尔可夫链分析的核心工具。它告诉我们：长程的转移可以分解为短程转移的乘积。这一性质在计算稳态分布、分析收敛速度时至关重要。

学习技巧：可以在纸上画一个只有三个状态（比如天气：晴、阴、雨）的马尔可夫链，定义转移概率矩阵 P，然后手工计算 P² 和 P³。你会直观地看到：即使最初的不确定性很大，随着 n 增大，Pⁿ 的每一行会趋于相同——这就是遍历性！过不了一会儿，初始条件的影响就"被遗忘了"。

状态分类

马尔可夫链的状态可以根据其转移特性分为不同的类别。理解这些分类对于判断链的长期行为至关重要。

可达与互通

如果存在 n ≥ 0 使得 P^{(n)}_{ij} > 0，则称状态 j 从状态 i 可达（accessible），记作 i → j。如果 i → j 且 j → i 同时成立，则称状态 i 和 j 互通（communicate），记作 i ↔ j。互通是一种等价关系，它将状态空间划分为若干互通类（communicating classes）。

常返与暂态

设 f_{ii} 是从状态 i 出发，最终返回状态 i 的概率（不是一步，而是一生）：

  • 如果 f_{ii} = 1：状态 i 是常返（recurrent）的——迟早会回来，几乎必然
  • 如果 f_{ii} < 1：状态 i 是暂态（transient）的——有正的概率一去不复返

进一步，对于常返状态：

  • 如果平均返回时间 μᵢ = E[返回时间] < ∞：正常返（positive recurrent）
  • 如果 μᵢ = ∞：零常返（null recurrent）——迟早会回来，但平均要等无穷长时间

周期

状态 i 的周期定义为：d = gcd{ n ≥ 1 : P^{(n)}_{ii} > 0 }。如果 d = 1，则状态是非周期（aperiodic）的；如果 d > 1，则状态是周期（periodic）的，周期为 d。

例如，一个"晴-雨"交替的2状态链，从"晴"出发回到"晴"只能在偶数步（2,4,6,...），因此它的周期 d=2。

分类维度类别定义含义
返回概率常返f_{ii} = 1一定会回来，不会漂走
返回概率暂态f_{ii} < 1有正概率一去不回
返回时间正常返μᵢ < ∞平均返回时间有限
返回时间零常返μᵢ = ∞会回来但平均要等无穷久
返回步数非周期gcd = 1任何步数都可能返回
返回步数周期 dgcd = d > 1只在特定步数倍数返回

一个遍历链（ergodic chain）要求：① 所有状态互通（不可约）；② 所有状态正常返；③ 所有状态非周期。遍历链具有最"友好"的性质——它有唯一的平稳分布并且从任意初始分布出发都会收敛到它。

工程直觉：在有限状态空间中，所有状态要么是正常返的，要么是暂态的。不会出现零常返。而且如果链是不可约的（所有状态互通），则所有状态的性质相同：要么全部正常返，要么全部暂态，要么全部周期。这意味着对于有限状态马尔可夫链，状态分类相对简单——你只需要检查一个状态的性质，就能知道所有状态。

平稳分布与极限分布

平稳分布的定义

一个概率分布向量 π = (π₁, π₂, ..., π_M) 被称为马尔可夫链的平稳分布（stationary distribution），如果：

π = π · P

即 πⱼ = Σᵢ πᵢ · P_{ij}，对所有 j

同时满足：πᵢ ≥ 0，∑ᵢ πᵢ = 1

平稳分布的物理含义是：如果链在时刻 n 的分布是 π，那么它在所有后续时刻的分布仍然是 π——分布"平稳"了，不再随时间变化。注意，平稳分布描述的是分布的稳定性，而非样本路径的稳定性。

极限分布

对于不可约、非周期、正常返的马尔可夫链（即遍历链），无论初始分布 π₀ 是什么，当 n → ∞ 时，状态分布都会收敛到唯一的平稳分布 π：

lim_{n→∞} π₀ · Pⁿ = π

等价地：lim_{n→∞} P^{(n)}_{ij} = πⱼ   （对任意初始状态 i 成立）

这个性质称为"遗忘初始条件"（forgetting the initial condition）——经过足够多步之后，初始状态的影响完全消失，链"忘记"了它从哪里出发。收敛速度由转移矩阵的第二大特征值决定（特征值越接近1，收敛越慢）。

PageRank：马尔可夫链的杀手级应用

Google的PageRank算法是马尔可夫链最著名的应用之一。它的核心思想极其简洁：

把互联网看作一个马尔可夫链：每个网页是一个状态，网页上的超链接是转移概率。用户以均匀概率点击当前页面的任意一个超链接进入下一个页面。那么，这个马尔可夫链的平稳分布就是每个网页被访问的长期概率——也就是网页的"重要性"排名。

然而，互联网图并不是一个"好"的马尔可夫链：
① 存在"死胡同"（没有超链接的页面）→ 暂态
② 存在"蜘蛛网"（只有内部链接的页面群）→ 非遍历

PageRank的"魔法"在于：以概率 α（通常0.85）按超链接跳转，
以概率 1-α 随机跳转到任意页面（teleportation）。
这保证了链是不可约且非周期的，从而平稳分布存在且唯一。

PageRank求解：π = π · [(1-α)·H + α·(1/N)·11ᵀ]
其中 H 是超链接矩阵，α 是"随机跳转"概率。

换句话说，PageRank的核心就是求解一个马尔可夫链的平稳分布！这个想法天才般地利用了马尔可夫链的理论工具来解决网页排序的实际问题。每当想起这个故事，我都深感：纯粹的数学理论（马尔可夫链在20世纪初由A.A. Markov提出）经过七八十年后，可以在一个看似毫不相关的领域产生如此巨大的商业和技术价值。

思维拓展：PageRank的故事告诉我们，学好随机过程不只是为了考试。马尔可夫链的平稳分布理论在20世纪初属于"纯数学"，但到了21世纪，它支撑了整个搜索引擎的运作。类似的故事还有：维纳滤波（1940s）→ 现代通信（2020s）、卡尔曼滤波（1960s）→ GPS导航（2020s）、强化学习中的马尔可夫决策过程 → AlphaGo和自动驾驶。基础理论的回报期很长，但回报率极高。

马尔可夫链的工程应用

除了PageRank，马尔可夫链在工程中还有大量应用：

  • 隐马尔可夫模型（HMM）：扩展的马尔可夫链——状态不可观测，只能通过"发射概率"观测到输出。语音识别、手写识别、生物信息学（基因预测）的经典工具。
  • 排队论：服务台前的排队长度可以用马尔可夫链建模。M/M/1队列的队长就是一个生灭过程——特殊类型的马尔可夫链。
  • 自然语言处理：N-gram语言模型本质上是一个高阶马尔可夫链。"今天天气真____"——在3-gram模型中，预测"好"的概率只取决于"天气真"两个词。
  • 强化学习：马尔可夫决策过程（MDP）是强化学习的数学基础。智能体在环境中根据当前状态选择动作，环境根据转移概率进入下一状态。
  • 通信信道建模：Gilbert-Elliott信道模型用两状态的马尔可夫链（"好状态"和"坏状态"）来描述突发错误信道。
踩坑故事 — 马尔可夫假设在推荐系统中的失效

某电商平台的推荐算法团队尝试使用马尔可夫链来建模用户的购买行为序列。他们的思路是：根据用户当前浏览/购买的商品，预测用户下一步最可能购买的商品。团队构建了包含数万个商品节点的"推荐马尔可夫链"，转移概率从数亿条用户行为日志中统计得出。离线评估结果非常漂亮——预测准确率超过了当时线上A/B测试中的协同过滤算法。

然而，线上A/B测试的结果令人大跌眼镜：马尔可夫链推荐模型的点击率（CTR）和转化率都远低于离线指标的预期，甚至低于简单的"热门商品"推荐策略。团队百思不得其解，进行了大量分析后发现了问题的根源。

真实用户的购买行为并不满足马尔可夫性质！用户的当前选择不仅取决于"上一个购买的商品"，还取决于：① 一周前搜索过什么（长周期兴趣）；② 当前季节和促销活动（外部环境因素）；③ 社交媒体上的讨论趋势（群体影响）。马尔可夫链把所有这些"长程依赖"全部丢弃了，只保留了"上一步"的记忆。在离线评估中，由于训练和测试数据来自同一分布，马尔可夫假设的偏差被低估了；但在线上真实环境中，用户行为远比链式模型复杂。

更具体地说，马尔可夫链无法处理"周期性消费"（如每月买一次猫粮）和"跨类别关联"（如买了婴儿车后几个月开始买奶粉）。这些行为的时间跨度远超"一步"，但在马尔可夫模型中完全丢失。

教训：马尔可夫链是一个非常强大的工具，但它的核心假设——"给定现在，未来与过去独立"——在真实世界的数据中往往不成立。在使用马尔可夫模型之前，请务必先问自己几个问题：① 我的系统的"状态"定义是否足够丰富，以至于当前状态确实包含了预测所需的全部信息？② 是否存在长周期依赖（用户行为中的周/月/季度规律）？③ 是否存在外部因素（天气、促销、节假日）对状态转移的影响？如果这些问题的答案是不能保证，那么扩展状态空间（使用高阶马尔可夫链）或采用更复杂的序列模型（如RNN、Transformer）可能是更好的选择。

本章小结与衔接

本章我们学习了马尔可夫链——最优雅也最实用的随机过程模型之一。从马尔可夫性质（给定现在未来与过去独立）出发，我们定义了一步转移概率矩阵、C-K方程、状态分类（常返/暂态/正常返/零常返/周期/非周期）、以及平稳分布和极限分布。我们用一个极具启发性的例子——PageRank算法——展示了马尔可夫链理论如何在互联网时代产生巨大影响，也通过推荐系统的踩坑故事提醒你谨慎对待马尔可夫假设。
