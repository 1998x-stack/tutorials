KMP算法

大家好。字符串匹配是计算机科学中最古老也最基础的问题之一。你在编辑器中按 Ctrl+F 搜索、在日志文件中 grep 关键词、甚至在 DNA 序列中寻找基因片段——背后都是字符串匹配算法在起作用。在 LeetCode 上，字符串匹配相关的题目虽然不多，但 KMP 算法是面试的高频考点——不是因为题目多，而是因为它考察候选人理解复杂算法的能力。

KMP 算法由 Knuth、Morris 和 Pratt 三位大神在 1977 年提出。它的核心洞察非常深刻：当匹配失败时，我们已经知道文本串中已经匹配了哪些字符——这些信息不应该被丢弃。KMP 利用一个 next 数组（也叫前缀函数），在失配时把模式串向右滑动尽可能远的距离，从而避免文本串指针的回退。

next数组的构建

next 数组是 KMP 的灵魂。定义：next[i] 表示模式串 P[0..i] 中，最长的相等真前缀和真后缀的长度。有点绕？简单说：对于模式串的每个前缀，它自己最长的"头尾相同"的部分有多长。

以 "ABABAC" 为例：

i 子串 最长相等前后缀 next[i]
0 A 无（长度 0） 0
1 AB 无 0
2 ABA "A" 1
3 ABAB "AB" 2
4 ABABA "ABA" 3
5 ABABAC 无 0

构建 next 数组的代码非常精炼，是用两个指针（i 和 j）动态规划出来的：

def build_next(pattern):
            m = len(pattern)
            next_arr = [0] * m
            j = 0  # 当前匹配的前缀长度

            for i in range(1, m):
                # 失配时回退
                while j > 0 and pattern[i] != pattern[j]:
                    j = next_arr[j - 1]
                # 匹配则前进
                if pattern[i] == pattern[j]:
                    j += 1
                    next_arr[i] = j
                # 否则 next[i] 保持为 0
            return next_arr
理解关键：j 既是"已匹配的长度"，也是"下一个要比较的下标"。失配时 j = next[j-1] 是最难理解的一行——它的含义是"利用已经算好的 next 值来回退 j"。把这行想通了，KMP 就等于学会了。

实现 strStr()（LeetCode 28）

题目标题很直白：给定两个字符串 haystack 和 needle，返回 needle 在 haystack 中出现的第一个位置，不存在则返回 -1。用 KMP 实现：

def strStr(haystack: str, needle: str) -> int:
            if not needle:
                return 0

            n, m = len(haystack), len(needle)
            next_arr = build_next(needle)

            j = 0
            for i in range(n):
                while j > 0 and haystack[i] != needle[j]:
                    j = next_arr[j - 1]
                if haystack[i] == needle[j]:
                    j += 1
                if j == m:
                    return i - m + 1
            return -1

注意匹配部分的代码和 next 构建的代码几乎一样——这就是 KMP 的美妙之处：构建 next 的过程本质上就是模式串自己匹配自己。整个算法的时间复杂度是 O(n + m)，空间复杂度 O(m)。

一个真实的故事：我大学时第一次学 KMP，看了三遍教材都没看懂 next 数组是怎么构建的。后来一个学长说："你就记住，j 是前缀的长度，也是前缀的下一个下标。构建 next 就是用模式串自己匹配自己。"这句话直到我工作后才真正理解。如果你现在也没完全搞懂，没关系——把代码写几遍、跑几个例子，慢慢就通了。

Rabin-Karp算法

如果说 KMP 是从"避免重复比较"的角度优化，那么 Rabin-Karp 就是从"快速比较"的角度出发——它利用哈希来比较字符串。Rabin-Karp 的核心思想是：把模式串和文本串中的每个长度为 m 的子串都计算哈希值，然后比较哈希值是否相等。

问题来了——每次计算子串的哈希需要 O(m) 时间，那和暴力法没有区别。所以 Rabin-Karp 的关键技巧是滚动哈希（Rolling Hash）：利用前一个子串的哈希，通过常数时间的操作计算下一个子串的哈希。

def rabin_karp(text, pattern):
            n, m = len(text), len(pattern)
            if m > n:
                return -1

            base = 26      # 字符集大小
            mod = 10**9 + 7

            # 计算 pattern 的哈希值和最高位权重
            pattern_hash = 0
            window_hash = 0
            power = 1

            for i in range(m):
                pattern_hash = (pattern_hash * base + ord(pattern[i])) % mod
                window_hash = (window_hash * base + ord(text[i])) % mod
                if i < m - 1:
                    power = (power * base) % mod

            if window_hash == pattern_hash and text[:m] == pattern:
                return 0

            for i in range(1, n - m + 1):
                # 移除前一个字符，加入新字符
                window_hash = (window_hash - ord(text[i - 1]) * power) % mod
                window_hash = (window_hash * base + ord(text[i + m - 1])) % mod
                window_hash = (window_hash + mod) % mod

                if window_hash == pattern_hash:
                    if text[i:i + m] == pattern:
                        return i
            return -1

平均时间复杂度 O(n + m)，最坏情况 O(nm)（哈希冲突太多）。但实际应用中，只要选择好的哈希函数和模数，冲突概率极低，Rabin-Karp 表现很好。

Python 模运算的坑：window_hash - ord(text[i-1]) * power 在 Python 中可能变成负数！虽然 Python 的 % 能将负数取模为正数，但最好统一用 (x % mod + mod) % mod 来保证结果在 [0, mod-1] 范围内。这是一个很容易在生产环境中引发 bug 的细节。

字符串哈希

Rabin-Karp 用的是单次滚动哈希。更一般的字符串哈希（也叫前缀哈希）思想是：预处理出字符串所有前缀的哈希值，然后通过公式 O(1) 时间计算任意子串的哈希值。

定义前缀哈希数组 h[i] 为 s[0..i-1] 的哈希值，递推公式：

h[i] = (h[i-1] * base + ord(s[i-1])) % mod

子串 s[l..r]（左闭右开）的哈希值为：

hash(l, r) = (h[r] - h[l] * base^(r-l)) % mod
一句话总结：字符串哈希把 O(n) 的字符串比较降为 O(1) 的整数比较。代价是存在哈希冲突的可能性（虽然极小），但配合双哈希（两个不同 base 或 mod）基本可以忽略不计。

重复的DNA序列（LeetCode 187）

题目：所有 DNA 由 'A'、'C'、'G'、'T' 四种核苷酸组成。给定一个 DNA 字符串，找出所有出现不止一次的长度为 10 的子串。典型的"滚动哈希"应用。

def find_repeated_dna_sequences(s):
            n = len(s)
            if n <= 10:
                return []

            # 将 4 种字符映射为 2 位二进制数
            mapping = {'A': 0, 'C': 1, 'G': 2, 'T': 3}
            nums = [mapping[ch] for ch in s]

            seen = set()
            result = set()
            # 用 20 位整数表示长度为 10 的子串（每字符 2 位）
            hash_val = 0
            mask = (1 << 20) - 1  # 20 个 1

            for i in range(n):
                # 移入新字符
                hash_val = ((hash_val << 2) | nums[i]) & mask

                if i >= 9:
                    if hash_val in seen:
                        result.add(s[i-9:i+1])
                    else:
                        seen.add(hash_val)
            return list(result)

这里用了一个巧妙的方法：因为字符集只有 4 个，每个字符用 2 位二进制数表示，10 个字符正好用 20 位整数表示——不需要取模，就没有冲突！这是位运算在字符串哈希中的惊艳应用。

Z函数

Z 函数（也叫 Z-algorithm）是 KMP 之外另一种线性时间字符串匹配算法。定义 Z[i] 为字符串 s 从 i 开始的后缀和 s 整个字符串的最长公共前缀的长度。换句话说，Z[i] 告诉我们"从 i 开始的子串和开头有多像"。

def z_function(s):
            n = len(s)
            z = [0] * n
            l = r = 0

            for i in range(1, n):
                if i <= r:
                    z[i] = min(r - i + 1, z[i - l])
                while i + z[i] < n and s[z[i]] == s[i + z[i]]:
                    z[i] += 1
                if i + z[i] - 1 > r:
                    l, r = i, i + z[i] - 1
            return z

要利用 Z 函数做字符串匹配，只需要把模式串 P 和文本串 T 拼成一个新串 P + '$' + T，其中 $ 是一个不在任何字符串中出现的分隔符。然后计算这个新串的 Z 函数——Z 值等于 len(P) 的位置就是匹配到的位置。

KMP vs Z 函数：两者都是 O(n+m) 时间的线性匹配算法。KMP 的 next 数组侧重于"失配后跳到哪里"，Z 函数侧重于"每个后缀和开头的匹配长度"。实现上 Z 函数更简洁，但 KMP 更常见于面试题中。我的建议是：面试时优先用 KMP（面试官更熟悉），竞赛时用 Z 函数（代码短）。
算法 时间复杂度 空间复杂度 核心数据结构 适用场景
KMP O(n+m) O(m) next 数组 单模式串匹配，面试首选
Rabin-Karp 平均 O(n+m) O(1) 滚动哈希 多模式串匹配、子串查找
字符串哈希 O(n) 预处理 + O(1) 查询 O(n) 前缀哈希数组 大量子串比较的场景
Z 函数 O(n+m) O(n+m) Z 数组 竞赛、需要额外信息的场景

踩坑与总结

KMP 的 next 数组理解难点

我观察到大多数人在学习 KMP 时有两个主要的困惑点：

  • next 数组的语义：不同教材的 next 数组定义不同。有的定义为"前缀函数"，有的定义为"失配后跳转的位置"。我的建议是统一理解成"最长相等前后缀长度"，然后匹配时用 j = next[j-1]。编码时宁愿多写一行注释也不要用错语义。
  • while 循环的终止：构建 next 时的 while j > 0 and pattern[i] != pattern[j]——有人会问为什么用 while 而不是 if？因为可能一次回退还不够，需要多次回退直到找到匹配或 j=0。这是 KMP 的核心机制。

哈希冲突处理

Rabin-Karp 和字符串哈希都面临哈希冲突的问题。处理方式有两种：

  • 双重哈希（Double Hashing）：用两个不同的 base 或 mod 同时计算两个哈希值，两个哈希都相等才视为匹配。冲突概率降为 1/mod1 * 1/mod2。
  • 哈希值校验后二次确认：哈希相等后直接比较原字符串（如我们代码中 if text[i:i+m] == pattern）。这是最简单有效的方法——反正冲突极少，多一次字符串比较的成本可忽略。
面试建议：如果面试遇到字符串匹配，首选 KMP。面试官问 KMP 通常期望你能手写 next 数组的实现——这是硬功夫，不能投机。实在记不住 next 构建，也可以先暴力实现 O(nm) 版本，再讲清楚 KMP 的思路，面试官也会认可你的问题分析能力。
