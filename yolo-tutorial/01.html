第01章：目标检测概述

想象一下，自动驾驶汽车如何在高速行驶的瞬间辨别出前方的行人、车辆和障碍物？当一辆配备视觉感知系统的车辆在晚高峰的十字路口穿行，它的摄像头每秒捕获 30 帧以上的图像，而车载计算单元必须在几十毫秒内回答一个关键问题：画面中到底有什么，它们分别在什么位置。这个看似本能的能力，背后是一整套复杂的计算机视觉技术——而其中最核心的，正是目标检测（Object Detection）。

目标检测是计算机视觉领域最基础也最具挑战性的任务之一。它不像图像分类那样只需要回答"这张图里是什么"，也不像语义分割那样需要将每个像素归类。目标检测处于两者之间：它必须同时完成"分类"和"定位"两项任务——即找到图像中所有感兴趣的目标，并用一个矩形框（bounding box）标出每个目标的位置，同时给出每个目标的类别标签。用一句话概括：目标检测 = 分类 + 定位。

1.1 什么是目标检测

一个直观的对比可以帮助你理解目标检测的独特之处。图像分类（Image Classification）的输入是一张图片，输出是一个类别标签，比如"猫"或"狗"。分类模型只需要抓住图像中最显著的特征即可做出判断。但目标检测要困难得多：一张街景照片中可能同时有行人、自行车、汽车、交通标志和红绿灯，模型不仅要把这些东西全部找出来，还要精确地告诉系统每个目标所在的空间位置。

更棘手的是，图像中目标的数量是可变的——这张图里可能有 3 个人，下一张图里可能有 15 个人；目标的尺寸也可以从几十像素的小目标到几乎占满全图的大目标不等；目标之间可能存在严重遮挡，比如人群中的行人彼此交叠。这些因素使得目标检测成为一个比分类复杂得多的学习问题。

1.2 核心任务：分类、定位、检测与分割

为了更清晰地理解目标检测在整个计算机视觉任务谱系中的位置，我们不妨用一个表格来对比几项核心任务：

任务 输出形式 空间精度 典型模型
图像分类 单一类别标签 无空间信息 ResNet, ViT, EfficientNet
目标检测 多个边界框 + 类别标签 矩形框级 YOLO, Faster R-CNN, DETR
语义分割 每个像素一个类别标签 像素级 U-Net, DeepLab, SegFormer
实例分割 每个实例独立像素掩膜 像素级 + 实例区分 Mask R-CNN, YOLACT

从表中可以看出，目标检测在"空间精度"和"计算开销"之间取得了很好的平衡——它不需要像分割那样精细到像素，但提供了比分类丰富得多的结构化信息。这也是为什么目标检测成为工业界应用最广泛的视觉技术之一：在自动驾驶、安防、工业质检等场景中，知道"哪里有障碍物"往往比知道"每个像素属于什么"更高效实用。

1.3 评估指标：如何衡量检测器的好坏

有了检测任务的定义，下一个自然的问题是：怎么定量评估一个检测模型的好坏？目标检测的评估比分类复杂得多，因为我们要同时衡量"定位是否准确"和"分类是否正确"。这里需要引入几个关键概念。

IoU（Intersection over Union，交并比）

IoU 是衡量两个边界框重叠程度的指标，也是目标检测中最基础的几何度量。它的计算方式很简单：两个框的交集面积除以它们的并集面积。IoU 的取值在 0 到 1 之间，越接近 1 表示两个框重合得越好。通常在评估时，我们会设定一个 IoU 阈值（比如 0.5），只有当预测框与真实框的 IoU 超过该阈值时，才认为这个检测是"正确"的。

下面是用 Python 实现 IoU 计算的代码：

def compute_iou(box1, box2):
    """计算两个边界框的 IoU 值

    Args:
        box1, box2: (x1, y1, x2, y2) 格式的边界框
    Returns:
        IoU 值 (0 ~ 1)
    """
    x1_min, y1_min, x1_max, y1_max = box1
    x2_min, y2_min, x2_max, y2_max = box2

    # 交集区域
    inter_xmin = max(x1_min, x2_min)
    inter_ymin = max(y1_min, y2_min)
    inter_xmax = min(x1_max, x2_max)
    inter_ymax = min(y1_max, y2_max)

    inter_area = max(0, inter_xmax - inter_xmin) * max(0, inter_ymax - inter_ymin)

    # 各自的面积
    area1 = (x1_max - x1_min) * (y1_max - y1_min)
    area2 = (x2_max - x2_min) * (y2_max - y2_min)

    # 并集 = 面积和 - 交集
    union_area = area1 + area2 - inter_area

    return inter_area / union_area if union_area > 0 else 0


# 示例：两个部分重叠的边界框
box_a = (50, 50, 150, 150)
box_b = (100, 100, 200, 200)
print(f"IoU = {compute_iou(box_a, box_b):.3f}")
# 输出: IoU = 0.143

mAP（mean Average Precision，平均精度均值）

仅仅知道单个框的 IoU 还不够，我们需要一个全局指标来衡量模型在整个数据集上的表现。mAP 是目前目标检测领域最通用的评估指标。它的计算过程分为两步：首先对每个类别分别计算 Average Precision（AP），即不同置信度阈值下 Precision-Recall 曲线下的面积；然后对所有类别的 AP 取平均值得到 mAP。

常见的 mAP 变体包括 mAP@0.5（IoU 阈值设为 0.5 时的 mAP）和 mAP@0.5:0.95（从 0.5 到 0.95 每隔 0.05 计算一次 mAP 再取平均，COCO 数据集官方标准）。后者更加严格，更能反映模型的定位精度。

实战踩坑：我曾在一个项目中踩过 mAP 的坑——团队将模型从 VOC 数据集迁移到 COCO 数据集上评测，发现 mAP@0.5 从 85% 降到了 78%，大家一度以为模型退步了。排查了很久才发现，VOC 的官方标准是 mAP@0.5，而 COCO 采用的是 mAP@0.5:0.95。模型本身的检测能力并没有下降，只是评估标准变得更加严格了。这个教训让我深刻认识到：读懂评估指标的内涵，比跑出一个漂亮的数字更重要。

FPS（Frames Per Second，每秒处理帧数）

FPS 衡量模型的推理速度，对于实时应用（如自动驾驶、视频监控）至关重要。在学术界，FPS 通常在单张 GPU 上以批大小 1 测量；在生产环境中，还需要考虑预处理、后处理以及 I/O 开销。YOLO 系列的一大核心优势就是高 FPS——早期 YOLOv1 在 Titan X 上就能跑到 45 FPS，而同期 Faster R-CNN 仅能达到约 7 FPS。

1.4 双阶段与单阶段：两种范式之争

在 YOLO 出现之前，目标检测领域的主流范式是"双阶段检测"（Two-stage Detection），其代表是 R-CNN 家族。双阶段检测的思路很直观：第一步先找出可能包含物体的候选区域（Region Proposal），第二步再对每个候选区域做分类和边框精修。R-CNN 是最早的开创者，使用 Selective Search 算法生成候选框；Fast R-CNN 引入了 ROI Pooling 加速特征提取；Faster R-CNN 则进一步用 Region Proposal Network（RPN）取代了外部候选框生成算法，实现了端到端训练。

单阶段检测（One-stage Detection）则采取了完全不同的策略：直接在图像上均匀采样，一步到位预测出边界框和类别，省去了独立的候选区域生成步骤。YOLO 是单阶段范式的奠基者之一（同期还有 SSD），它将检测问题重新定义为一个回归问题——从输入图像直接映射到边界框坐标和类别概率。

两种范式的关键区别可以用下表概括：

对比维度 双阶段（Faster R-CNN） 单阶段（YOLO）
推理速度 较慢（~7 FPS on Titan X） 快（45+ FPS on Titan X）
高 IoU 精度 较高（mAP@0.5:0.95 占优） 早期偏低，v3 后逐渐追平
小目标检测 相对更好 早期弱项，FPN 后改善
训练复杂度 较高（多阶段训练） 低（单次端到端训练）
部署便利性 较复杂 简单，社区生态完善
关键认知：双阶段和单阶段并非"谁替代谁"的关系。Faster R-CNN 等双阶段检测器在某些高精度场景（如遥感图像分析、医学影像检测）中仍有不可替代的优势，而 YOLO 则凭借速度和部署便利性成为实时应用和工程落地的主流选择。最近几年的趋势是两者融合——YOLO 的后续版本吸收了 RPN、FPN 等双阶段范式的思想，而双阶段模型也变得越来越"端到端"。

1.5 经典数据集

目标检测的发展离不开高质量标注数据的推动。以下三个数据集在目标检测历史上具有里程碑意义：

  • PASCAL VOC（Visual Object Classes）：2005-2012 年举办的视觉识别挑战赛，涵盖 20 个类别（人、动物、交通工具、室内物品等），约 11,000 张图像。VOC 2012 多年来一直是目标检测的标准基准，mAP@0.5 即来源于此。
  • MS COCO（Common Objects in Context）：微软于 2014 年发布的大规模数据集，包含 80 个类别、约 200,000 张标注图像，每张图像平均有 7.7 个标注目标。COCO 的标注质量更高，且引入了更严格的 mAP@0.5:0.95 评估标准，是目前最主流的目标检测基准。
  • ImageNet Detection：ImageNet 大赛中的检测赛道（ILSVRC 2013-2017），包含 200 个类别，约 500,000 张图像。由于类别多、数据量大，对模型的泛化能力要求更高。
数据集 类别数 标注图像数 每个目标平均标注数 评估标准
PASCAL VOC 2012 20 ~11,500 ~2.3 mAP@0.5
MS COCO 2017 80 ~200,000 ~7.7 mAP@0.5:0.95
ImageNet DET 200 ~470,000 ~1.1 mAP@0.5
注意事项：不同数据集的标注标准和难度差异很大。例如，COCO 数据集中的目标普遍比 VOC 中的目标更小（COCO 中超过 40% 的目标面积不足图像面积的 1%），且场景更复杂。因此，一个在 VOC 上表现优异的模型迁移到 COCO 上时，mAP 通常会下降 10-15 个百分点。这是正常现象，不代表模型有问题——了解数据集的特性对于正确解读评测结果至关重要。

1.6 应用场景

目标检测技术的应用范围极为广泛，几乎渗透到了每一个需要"看懂图像"的行业。以下是几个最具代表性的领域：

  • 自动驾驶与智能交通：检测车辆、行人、交通标志、车道线、红绿灯等，是自动驾驶感知系统的核心技术栈。百度 Apollo、特斯拉 Autopilot 等系统都重度依赖目标检测模型。这里对实时性的要求极高——从摄像头捕获到检测结果输出通常需要在 30-50ms 内完成。
  • 安防监控：人流统计、异常行为检测、车辆轨迹追踪、人脸检测等。在智慧城市项目中，后端服务器通常需要同时对数十路甚至上百路视频流进行实时检测分析。
  • 工业视觉质检：在电子元器件、纺织品、食品等生产线上，用目标检测模型代替人工目检，检测产品表面的缺陷、划痕、异物等。这类场景的特点是目标小、背景单一、对误检率要求极高。
  • 医学影像分析：在 CT、X 光、病理切片等医学影像中检测病灶、肿瘤、器官边界。辅助医生进行诊断筛查，可以提高效率并降低漏诊率。
  • 遥感与地理信息：从卫星图像或无人机航拍图中检测建筑物、船只、车辆、农田等目标，用于城市规划、军事侦察、灾害评估等。
  • 新零售与智慧仓储：商品识别、货架巡检、库存盘点、自助结算等场景。亚马逊 Go 无人便利店的核心感知技术就是目标检测。

这些场景各自有不同的权衡侧重：自动驾驶追求实时性与召回率，工业质检追求极低误检率，遥感检测需要处理超大分辨率图像中的小目标。本课程后续章节会针对这些具体挑战一一展开讨论。

一句话总结：目标检测是计算机视觉从"理解图像"走向"与环境交互"的桥梁。它让机器不仅知道"看到了什么"，还知道"在哪里看到"——这看似简单的一步跨越，却开启了无数实际应用的大门。
