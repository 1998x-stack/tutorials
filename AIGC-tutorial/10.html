第10章 Stable Diffusion架构

大家好，我是你们的讲师。如果说 DDPM 是扩散模型的"理论基础"，DDIM 和 LDM 是"关键突破"，那么 Stable Diffusion（SD）就是让扩散模型"走向大众"的里程碑。2022 年 8 月，Stability AI 将 SD 开源后，整个 AIGC 生态被彻底点燃——设计师、开发者、艺术家、甚至普通用户都能在自己的电脑上运行文生图模型。

这一章，我们将解剖 SD 的完整架构。你将看到三个核心模块——VAE、U-Net、CLIP——如何协同工作，以及 SD 从 1.0 到 3.0 经历了怎样的技术演进。理解 SD 架构是下一阶段学习 ControlNet、LoRA、模型微调等高级技术的必要前提。

10.1 整体架构：三模块协同

Stable Diffusion 不是一个单一的神经网络，而是三个模块的协同系统：

模块功能输入输出
CLIP Text Encoder将文本 prompt 编码为 77×768 的条件特征文本（最多 77 tokens）文本嵌入序列 + pooled 向量
U-Net（含 Cross-Attention）在潜空间去噪，接收文本条件含噪潜变量 + 时间步 t + 文本嵌入预测的噪声 ε_θ
VAE Decoder将去噪后的潜变量解码为像素图像64×64×4 潜特征图512×512×3 RGB 图像

推理时的工作流程：文本 prompt → CLIP 编码 → 随机噪声 → U-Net 多步去噪（条件于文本） → VAE 解码 → 最终图像。

🌟 核心洞察：SD 的架构设计体现了"分而治之"的思想——CLIP 负责"理解语义"，U-Net 负责"生成内容"，VAE 负责"解码呈现"。每个模块各司其职，可以独立优化和替换。这种模块化设计是 SD 生态如此繁荣的根本原因。

10.2 VAE 模块：8 倍压缩的感知压缩器

LDM（第 9 章）的核心思想被 SD 直接继承：用 VAE 将 512×512×3 的图像压缩为 64×64×4 的潜表示，压缩比约为 48 倍（512×512×3 / 64×64×4 ≈ 48）。

VAE 在 SD 中的关键参数和特点：

  • 下采样因子 f = 8：输入 512 → 潜空间 64（经过 3 次 2× 下采样）
  • 潜变量通道数 = 4：每个 64×64 位置有 4 个通道的浮点数
  • KL 正则化：训练时施加小的 KL 惩罚，使潜变量接近标准高斯分布
  • 缩放因子：潜变量乘以一个常数（SD 1.x/2.x 使用 0.18215）以控制方差

VAE 的编码器和解码器都是卷积架构，编码器使用 stride=2 的卷积实现下采样，解码器使用转置卷积实现上采样。

# VAE Encoder 简化结构（通道数示意）
# 输入: [1, 3, 512, 512]
# Conv2D(3→128, 3×3, stride=1) → ResBlock(128→128)
# Conv2D(128→256, 3×3, stride=2) → ResBlock(256→256)    # 256×256
# Conv2D(256→512, 3×3, stride=2) → ResBlock(512→512)    # 128×128
# Conv2D(512→512, 3×3, stride=2) → ResBlock(512→512)    # 64×64
# Conv2D(512→8, 3×3) → split into mean + logvar (各4通道)
# 输出: [1, 4, 64, 64]（采样后）

10.3 U-Net 结构

U-Net 是 SD 中真正的"主力"，它负责在潜空间中执行多步去噪。SD 的 U-Net 结构遵循经典的"下采样-瓶颈-上采样"设计，并加入了残差连接和跳跃连接（Skip Connections）。

SD 1.x 的 U-Net 主要参数：

  • 总参数量：约 860M（占 SD 1.5 全部权重的 80%+）
  • 基础通道数：320（后续阶段乘 1, 2, 4, 4）
  • 下采样阶段：4 个阶段（64→32→16→8 分辨率）
  • 残差块（ResBlock）：GroupNorm + SiLU + Conv 的标准设计
  • 跳跃连接：下采样路径的每个阶段输出都通过跳跃连接与上采样路径对应阶段拼接

U-Net 的不同分辨率阶段对应不同的作用：低分辨率阶段（8×8）关注全局语义和高层概念，高分辨率阶段（64×64）关注纹理、光照和局部细节。Cross-Attention 层通常部署在低分辨率阶段，因为那里有更丰富的语义信息。

💡 直觉理解：U-Net 的对称结构类似于一个"编码器-解码器"——编码器逐步提取高层语义特征，解码器将这些语义特征逐步还原为精细的空间结构。跳跃连接的作用是让解码器能直接访问编码器各层的细粒度空间信息，避免信息在瓶颈处丢失。

10.4 Cross-Attention 机制

Cross-Attention 是 SD 实现"文生图"的核心机制。它将 CLIP 编码的文本特征注入到 U-Net 的中间层：

# Cross-Attention 的计算公式
Q = W_Q · φ(x_t)     # 来自 U-Net 中间层的特征图
K = W_K · τ( prompt )  # 来自 CLIP 的文本嵌入
V = W_V · τ( prompt )  # 来自 CLIP 的文本嵌入

Attention(Q, K, V) = softmax(Q · K^T / √d) · V

这里的 φ(x_t) 是 U-Net 的中间特征，τ(prompt) 是 CLIP Text Encoder 输出的 77×768 文本嵌入。Q（Query）来自图像侧，K（Key）和 V（Value）来自文本侧——这就是"Cross-Attention"名称的由来：注意力不再发生在图像特征内部（Self-Attention），而是跨越了图像和文本两个模态。

每个注意力头的输出会经过一个线性投影后与输入相加（残差连接），使得模型可以选择性地"关注"文本中对应位置的语义信息。例如，生成 "a red apple on a blue table" 时，不同区域的像素会通过 Cross-Attention 找到对应的语义 token。

10.5 SD 版本演进对比

从 2022 年 8 月的 SD 1.0 到 2024 年的 SD 3.0，SD 经历了多次重大迭代：

版本发布时间U-Net 参数量文本编码器训练数据核心改进
SD 1.4/1.52022.08860MCLIP ViT-L (77 tokens)LAION-5B奠定基础，生态爆发点
SD 2.12022.12865MOpenCLIP ViT-H (77 tokens)LAION-5B（过滤后）更好的 NSFW 过滤，提升分辨率鲁棒性
SDXL2023.072.6B双编码器：CLIP ViT-bigG + OpenCLIP ViT-bigG内部数据原生 1024×1024、双文本编码器、更好的构图
SDXL Turbo2023.112.6B同上同上对抗式蒸馏，1-4步生成
SD 3.02024.068B (MM-DiT)三编码器：CLIP + T5-XXL + CLIP内部数据MM-DiT 架构替代 U-Net，Flow Matching
🌟 核心洞察：SD 的演进路线揭示了几个趋势：模型越来越大（860M → 8B），文本理解能力越来越强（单 CLIP → 双 CLIP → CLIP + T5），架构从 U-Net 向 DiT 迁移。SD 3.0 的 MM-DiT 架构是自 SD 1.0 以来最重大的架构变革——我们将在第 13 章和第 19 章详细分析。

10.6 完整推理管线

将上述所有部分串联起来，SD 的一次完整推理流程如下：

1. 文本编码: text_emb = CLIP( prompt )           # → [1, 77, 768]
2. 初始噪声: latents = randn([1, 4, 64, 64])     # 潜空间高斯噪声
3. 循环去噪（DDIM scheduler, 50步）:
   for t in reversed(range(50)):
     noise_pred = UNet(latents, t, text_emb)      # → [1, 4, 64, 64]
     latents = scheduler.step(noise_pred, t, latents)
4. VAE 解码: image = VAE_Decoder(latents)         # → [1, 3, 512, 512]
5. 后处理: image = (image.clamp(-1,1) + 1) / 2   # 归一化到 [0,1]

在 Diffusers 库中，上述流程可以用不到 30 行代码实现——但这背后是 VAE、U-Net、CLIP、Scheduler 四个组件的精密协作。

⚠️ 真实的踩坑故事：在我第一次尝试将 SD 集成到生产服务时，遇到了一个非常诡异的问题：同样的 prompt 和 seed，在本地笔记本上和服务器上生成的图像不一样。排查了三天才发现——两个环境中的 Diffusers 版本不同，导致采样器对时间步的计算方式有细微差异。后来我学会了一条铁律：生产环境的依赖版本必须和开发环境完全锁定，特别是 diffusers、transformers 和 torch 这三件套。一个微小的版本差异可能导致生成结果完全不同，这在评测和 A/B 测试中是灾难性的。
