第13章 数值微分

大家好，我是你们的讲师。今天我们来聊一个表面上看起来很简单、但实际陷阱非常多的话题——数值微分。如果说数值积分是"用离散和近似积分"的稳健过程，那么数值微分就是"用离散差近似导数"的不稳定过程。"积分平滑噪声，微分放大噪声"——这是数值分析中一条非常重要的经验法则。

我想起一个亲身经历的故事。有个师兄做控制系统设计，传感器的输出信号有轻微的噪声。他直接用差分法计算信号的导数来做反馈控制，结果控制系统剧烈振荡。检查了很久才发现问题不在控制器，而在数值微分——差分运算把传感器的高频噪声放大了几十万倍。后来他加了一个低通滤波器再微分，系统就稳定了。这个教训在工程实践中太常见了，所以今天我们要重点讨论步长选择和误差平衡的问题。

13.1 差商逼近的基本方法

数值微分的起点是 Taylor 展开。给定 $f \in C^3$，在 $x$ 处展开：

f(x+h) = f(x) + h f'(x) + h²/2 f''(x) + h³/6 f'''(ξ₁)
f(x-h) = f(x) - h f'(x) + h²/2 f''(x) - h³/6 f'''(ξ₂)

由第一个式子得到前向差分（forward difference）：

f'(x) ≈ (f(x+h) - f(x)) / h + O(h)

由第二个式子得到后向差分（backward difference）：

f'(x) ≈ (f(x) - f(x-h)) / h + O(h)

将两个式子相减得到中心差分（central difference）：

f'(x) ≈ (f(x+h) - f(x-h)) / (2h) + O(h²)

中心差分的误差阶是 $O(h^2)$，比前向/后向差分高一阶——这是对称性的回报。

什么时候用哪种？在区间内部，优先用中心差分，精度高一阶。在区间端点（没有 $f(x-h)$ 可用时），用前向或后向差分。如果你需要二阶导数，中心差分为：$f''(x) \approx (f(x+h) - 2f(x) + f(x-h)) / h² + O(h²)$。

13.2 步长选择的误差平衡

这是数值微分中最关键也最容易被忽视的问题。总误差由两部分组成：

总误差 = 截断误差（truncation error）+ 舍入误差（roundoff error）

以中心差分为例：

  • 截断误差：$\approx M h^2 / 6$，其中 $M = \max|f'''(\xi)|$。$h$ 越小，截断误差越小。
  • 舍入误差：$\approx \varepsilon / h$，其中 $\varepsilon$ 是机器精度（float64 约 $2\times10^{-16}$）。$h$ 越小，舍入误差越大！

为什么会这样？因为当 $h$ 非常小时，$f(x+h)$ 和 $f(x-h)$ 几乎相等，两者的差包含大量的相消性损失（catastrophic cancellation）——有效数字位数急剧减少。

优化 $h$：使总误差最小，求导得最优步长 $h_{\text{opt}} \approx \sqrt[3]{3\varepsilon / M}$。对于 float64，大约在 $h \approx 10^{-5}$ 量级。

import numpy as np

def f(x):
    return np.sin(x)

def f_prime_exact(x):
    return np.cos(x)

x = 1.0
exact = f_prime_exact(x)

print("步长扫描 — 中心差分逼近 f'(x) = cos(x) 在 x=1:")
print(f"{'h':>12} {'approx':>20} {'error':>15}")
print("-" * 50)

for k in range(-1, -17, -1):
    h = 10.0 ** k
    approx = (f(x + h) - f(x - h)) / (2 * h)
    error = abs(approx - exact)
    print(f"{h:.0e} {approx:20.14f} {error:15.10e}")

# 找到最优 h
hs = 10.0 ** np.arange(-1, -17, -1, dtype=float)
errors = np.abs((f(x + hs) - f(x - hs)) / (2 * hs) - exact)
best_h = hs[np.argmin(errors)]
print(f"\n最优步长: h = {best_h:.0e}, 最小误差: {np.min(errors):.2e}")

血的教训：我当年教数值分析课时布置了一个作业——用数值微分计算 $f(x) = e^x$ 在 $x=1$ 处的导数。有学生取了 $h=10^{-16}$，心想"步长越小越精确"。结果误差大到 0.5！为什么？因为 $e^{1+10^{-16}}$ 和 $e^{1-10^{-16}}$ 在 float64 下几乎无法区分，它们的差几乎全是噪声。记住：数值微分中最优 $h$ 不是 0，大约是 $\sqrt[3]{\varepsilon_{\text{mach}}} \approx 6 \times 10^{-6}$。小于这个值，误差不降反升。

差分格式公式截断误差舍入误差最优 $h$
前向差分$(f(x+h)-f(x))/h$$O(h)$$O(\varepsilon/h)$$\sim \sqrt{\varepsilon}$
中心差分$(f(x+h)-f(x-h))/(2h)$$O(h^2)$$O(\varepsilon/h)$$\sim \sqrt[3]{\varepsilon}$
四阶中心差分$\frac{-f(x+2h)+8f(x+h)-8f(x-h)+f(x-2h)}{12h}$$O(h^4)$$O(\varepsilon/h)$$\sim \sqrt[5]{\varepsilon}$

13.3 复数步长法

有没有办法彻底避免相消性损失？有一个非常巧妙的技巧——复数步长法（complex step differentiation）。

不是取实数的 $x+h$，而是取 $x + ih$（纯虚数步长），然后利用 Cauchy-Riemann 方程：

f'(x) ≈ Im[f(x + i h)] / h + O(h²)

这里的关键是：我们计算 $f(x+ih)$ 和 $f(x)$ 的差时，用的是虚部，而虚部本身不涉及两个大数相减！因此没有相消性损失，$h$ 可以取到非常小（比如 $10^{-100}$），截断误差趋近于零。

NASA 的真实案例：NASA 在计算飞行器气动导数时广泛使用复数步长法。传统方法需要手动推导几十页的导数公式，而复数步长法只需要在代码中用复数类型，零阶和一阶导数一次性得到。对于复杂的 CFD 代码，这大幅减少了人工推导的工作量和出错概率。注意：你的函数必须支持复数运算（即满足解析性）。Python 的 numpy 函数大多天然支持复数输入。

import numpy as np

def complex_step_derivative(f, x, h=1e-100):
    """复数步长法计算导数"""
    return np.imag(f(x + 1j * h)) / h

# 测试
f = np.exp
x = 1.0
print(f"精确值:   {np.exp(1):.15f}")
print(f"复数步长: {complex_step_derivative(f, x):.15f}")
print(f"误差:     {abs(complex_step_derivative(f, x) - np.exp(1)):.2e}")
# 试试 h=1e-100 仍然精确！

13.4 Richardson 外推于数值微分

就像在 Romberg 积分中那样，Richardson 外推同样适用于数值微分。中心差分 $D(h)$ 的误差展开也只包含 $h$ 的偶次幂：

D(h) = f'(x) + c₁h² + c₂h⁴ + c₃h⁶ + ...

用步长 $h$ 和 $h/2$ 做外推，得到 $O(h^4)$ 的精度：

D_new = (4 · D(h/2) - D(h)) / 3

重复这个过程可以得到任意高阶的导数逼近公式——本质上就是构造不同系数的有限差分模板。

def richardson_diff(f, x, h=0.1, k=3):
    """用 Richardson 外推提高数值微分的精度"""
    # D[j] = 中心差分 with h/2^j
    D = np.zeros(k + 1)
    for j in range(k + 1):
        h_j = h / (2 ** j)
        D[j] = (f(x + h_j) - f(x - h_j)) / (2 * h_j)

    # Richardson 外推表
    R = np.zeros((k + 1, k + 1))
    R[:, 0] = D

    for j in range(1, k + 1):
        for i in range(j, k + 1):
            factor = 4 ** j
            R[i, j] = (factor * R[i, j-1] - R[i-1, j-1]) / (factor - 1)

    return R

# 测试
f = np.sin
x = 1.0
exact = np.cos(x)
R = richardson_diff(f, x, h=0.1, k=4)
print(f"精确导数: {exact:.12f}")
for j in range(5):
    print(f"R(4,{j}) = {R[4,j]:.12f}, 误差 = {abs(R[4,j]-exact):.2e}")

通用思想：Richardson 外推不仅可以用于数值积分和数值微分，它实际上是一个通用的"从低阶方法构造高阶方法"的框架。只要你有一个逼近方法其误差具有已知的渐近展开形式，就可以用 Richardson 外推消除前几项误差。这个思想在数值分析中无处不在。

本章我们学习了数值微分的基本方法、步长选择的折衷、复数步长法这个巧妙技巧，以及 Richardson 外推的应用。数值微分是一个看起来简单但实践起来到处都是陷阱的技术——"微分放大噪声"这条法则值得永远记住。下一章我们将回到积分的主题，深入探讨正交多项式的理论及其在数值积分中的应用。这是理解 Gauss 求积为什么能取得最优效果的理论基础。
