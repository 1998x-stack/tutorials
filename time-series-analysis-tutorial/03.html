第3章 平稳性

大家好，我是你们的讲师。你有没有想过一个问题：为什么我们能用过去的气温数据预测明天的天气，却不能用过去十天的股票价格预测明天的股价？答案隐藏在"平稳性"这个概念里。

平稳性（Stationarity）是时间序列建模最重要也最容易被忽视的前提条件。如果说时间序列分析是一座大厦，那平稳性就是地基。地基不牢，上面修的任何漂亮模型都是危房。我见过太多人在非平稳数据上跑回归、做预测，结果模型在训练集上表现惊艳，一上线就崩盘——不是模型不好，是数据本身不满足模型的基本假设。

讲个我亲身经历的教训。几年前我在做一个电力负荷预测项目，用 ARIMA 模型预测未来一周的用电量。原始数据有明显的逐年上升趋势（经济发展带动用电增长），我直接扔进模型，结果预测值一路向下飘，完全不对。后来发现 ARIMA 的 "I"（Integrated，差分） 就是用来处理非平稳性的，而我跳过了这个关键步骤。一阶差分后数据变平稳了，模型预测立刻正常。这个教训让我从此把平稳性检验放在任何建模流程的第一步。

3.1 什么是平稳性

平稳性分为两种：

严平稳（Strict Stationarity）：序列的联合分布在时间平移下不变。这个条件极其严格，实际中几乎不可能满足，更多是理论概念。

弱平稳（Weak Stationarity / Covariance Stationarity）：只需要满足三个条件：

  • 均值恒定：E[Y_t] = μ（不随时间变化）
  • 方差恒定：Var(Y_t) = σ²（波动程度不变）
  • 协方差只与时间间隔有关：Cov(Y_t, Y_{t-k}) = γ_k（与具体时刻无关）

我们平时说"检验平稳性"，指的都是弱平稳。一个白噪声序列是典型的平稳序列——它的均值始终为0，方差恒定，不同时间点的值彼此独立。而股价、GDP、气温（长期趋势下）通常是非平稳的。

3.2 为什么平稳性如此重要

原因其实很直观：如果数据的统计性质在变化，你拿过去的数据训练模型，怎么能指望它预测未来？

从数学角度看，大部分时间序列模型（AR、MA、ARMA）的理论推导都建立在一个假设上——序列是平稳的。如果不平稳，参数的估计量不再具有一致性，t统计量不再服从t分布，预测区间也会变得毫无意义。简单说：不平稳数据上做统计推断，结果全是废的。

直观理解：想象你在一个移动的火车上投篮。篮筐也在移动。如果你不考虑火车速度（趋势），你永远投不中。平稳性就是让"篮筐固定不动"，这样你才能瞄准。

3.3 单位根的直觉

理解非平稳性的核心概念是"单位根"（Unit Root）。考虑一个简单的 AR(1) 模型：

Y_t = φ * Y_{t-1} + ε_t
  • 如果 |φ| < 1，序列是平稳的——冲击会随时间衰减消失
  • 如果 |φ| > 1，序列是爆炸性的——现实中很少见
  • 如果 φ = 1，序列就是随机游走——非平稳，且冲击的影响永不消失

φ = 1 这个情形就叫"单位根过程"。它的特性是：今天的值 = 昨天的值 + 随机扰动。股价就是最典型的例子。这就是为什么你在股票软件上看到的价格曲线看起来"漫无目的"地漂移——因为它确实在漫无目的地漂移。

3.4 ADF 检验和 KPSS 检验

ADF 检验（Augmented Dickey-Fuller Test）

原假设 H₀：序列存在单位根（非平稳）。备择假设 H₁：序列平稳。

如果 p 值小于显著性水平（比如0.05），拒绝原假设，认为序列平稳。这是使用最广泛的单位根检验。

KPSS 检验（Kwiatkowski-Phillips-Schmidt-Shin Test）

原假设 H₀：序列平稳。备择假设 H₁：序列存在单位根。

注意，KPSS 的原假设和 ADF 正好相反。所以通常的做法是两个检验一起做：如果 ADF 说非平稳、KPSS 说平稳，那就是非平稳；反之亦然。如果两个都说平稳（或两个都说非平稳），需要结合图像进一步判断。

3.5 让数据平稳：差分与变换

发现数据非平稳怎么办？两个主流方法：

差分（Differencing）

Y'_t = Y_t - Y_{t-1}

一阶差分能让线性趋势消失。如果一阶差分后还不平稳，可以做二阶差分。但注意：不要过度差分。差分会丢失信息——每做一次差分，数据就少一个观测值，而且你改变了数据的"记忆结构"。实际经验中，绝大多数经济时间序列一阶差分就够了，极少需要二阶以上。

Box-Cox 变换

如果问题在于方差随时间变化（即异方差性），Box-Cox 变换可以稳定方差：

Y_t(λ) = (Y_t^λ - 1) / λ,  当 λ ≠ 0
Y_t(λ) = ln(Y_t),          当 λ = 0

λ = 0 是对数变换，λ = 0.5 是平方根变换。对数变换最常用，适用于乘法模型的序列（比如销售额、客流量）。

过度差分的陷阱：我有一个合作过的分析员，拿到序列后机械地反复差分直到 AD 检验通过为止。结果他用了三阶差分才通过检验。但他没意识到：过度差分会引入本来不存在的自相关结构，让模型变得极其复杂且不可解释。一个简单的经验法则：如果一阶差分后的序列看起来是平稳的，就不要再差分了。如果一阶后还不平稳，先试试 Box-Cox 变换再差分。实在不行再考虑二阶差分——但仍然要谨慎。

3.6 实操流程

在实际项目中，我通常按这个流程来处理平稳性：

  1. 画时序图，目测是否有趋势或季节性
  2. 做 ADF 检验和 KPSS 检验
  3. 如果不平稳：尝试 Box-Cox 变换稳定方差
  4. 如果还不平稳：一阶差分
  5. 再次检验
  6. 如果一阶还不够：考虑是否需要二阶差分（极少见）

下面是用 Python 进行 ADF 检验的代码：

from statsmodels.tsa.stattools import adfuller, kpss
import pandas as pd

# 假设 data 是一个 pandas Series
# ADF 检验
adf_result = adfuller(data, autolag='AIC')
print(f"ADF Statistic: {adf_result[0]:.4f}")
print(f"p-value: {adf_result[1]:.4f}")
print(f"Critical values: {adf_result[4]}")

# KPSS 检验
kpss_result = kpss(data, regression='c')
print(f"KPSS Statistic: {kpss_result[0]:.4f}")
print(f"p-value: {kpss_result[1]:.4f}")

# 差分
data_diff = data.diff().dropna()

# Box-Cox 变换
from scipy.stats import boxcox
data_boxcox, lambda_opt = boxcox(data.dropna())
print(f"Optimal Box-Cox lambda: {lambda_opt:.4f}")

最后，关于"什么时候可以不做平稳化"这个问题。如果只是做描述性分析和可视化，当然不需要。但如果要建模、预测、做统计推断——平稳性不是一个选项，而是一个前提条件。好消息是，一旦你掌握了平稳性的概念和工具，接下来要学的自相关分析就会水到渠成。下一章我们就来看看，平稳序列的内在结构是怎么通过自相关函数（ACF）来揭示的。
