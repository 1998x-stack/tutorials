第25章 Transformer与时序

大家好！前两章我们分别学习了RNN和CNN在时间序列中的应用。RNN通过循环结构传递信息，CNN通过卷积核提取局部模式。这两种方法有一个共同点：它们都依赖"局部"信息来逐步构建全局理解——RNN一步一步地传递状态，CNN一层一层地扩大感受野。

但如果我说：有一种架构可以让序列中的任意两个位置直接"对话"，不管它们相隔多远——你会不会觉得这很神奇？这就是Transformer的核心思想：自注意力机制（Self-Attention）。它彻底改变了自然语言处理，也正在深刻改变时间序列分析的面貌。

25.1 自注意力机制

自注意力的核心操作是：对于序列中的每个元素，计算它与其他所有元素的相关性（注意力权重），然后用这些权重对所有元素的值做加权求和。具体来说，每个输入通过三个不同的线性变换得到 Query（查询）、Key（键）和 Value（值）：

  • Query：当前元素"想找什么"
  • Key：每个元素"是什么"的标签
  • Value：每个元素"携带什么信息"

注意力权重通过 Query 和 Key 的点积计算，经过 softmax 归一化后，对 Value 做加权平均。这个操作可以并行计算所有时间步，不受序列长度的限制——这是RNN做不到的。

# 缩放点积注意力（Scaled Dot-Product Attention）
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    # Q, K, V: (batch, seq_len, d_model)
    scores = torch.matmul(Q, K.transpose(-2, -1))
    scores = scores / (K.size(-1) ** 0.5)  # 缩放
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
    attn_weights = F.softmax(scores, dim=-1)
    # (batch, seq_len, seq_len) @ (batch, seq_len, d_model)
    return torch.matmul(attn_weights, V), attn_weights
关键洞察：自注意力计算得到一个 (seq_len, seq_len) 的注意力矩阵。第 i 行第 j 列的值表示第 i 个时间步对第 j 个时间步的"关注程度"。这个矩阵是对序列中所有依赖关系的完整建模——远距离的依赖、季节性的周期、突变的跳变，全部一目了然。

25.2 Transformer的时序适配

标准的Transformer架构（Vaswani et al., 2017）包含编码器和解码器，每层由多头自注意力（Multi-Head Attention）和前馈网络组成。但将Transformer直接应用于时间序列有几个挑战：

位置编码（Positional Encoding）：自注意力本身没有顺序感——它对序列的排列是不变的（permutation invariant）。因此必须显式地加入位置信息。原始Transformer使用正弦/余弦函数编码，但在时序任务中，更常见的是使用可学习的位置编码（Learned Positional Encoding），或者直接对时间戳本身做嵌入（如小时、星期、月份）。

计算复杂度：标准自注意力的复杂度是 O(L²)，其中 L 是序列长度。对于长序列（比如一年365天的日数据），直接计算是不现实的。这激发了后面要讲的一系列高效变体。

时序分解：时间序列具有趋势、季节性和残差等成分。直接将原始序列喂给Transformer可能效果不佳，更好的做法是先在架构内嵌入分解模块，让模型自动分离不同成分。

25.3 关键架构：Informer

Informer（Zhou et al., 2021）是专为长序列时间序列预测（LSTF）设计的Transformer变体。它的核心创新是ProbSparse自注意力机制。

ProbSparse的直觉是：在标准的自注意力矩阵中，大多数注意力权重都非常小（接近零），真正重要的只有少数几个"头"（dominant attention pairs）。Informer使用KL散度来度量每个Query的"重要性"，只保留最重要的Top-k个Query参与注意力计算。这使复杂度从 O(L²) 降低到 O(L log L)。

此外，Informer还引入了自注意力蒸馏（Self-Attention Distilling）和生成式解码器（Generative Decoder），进一步提升了长序列预测的效率和精度。在2021年发布时，Informer在多个长序列数据集上大幅超越了当时的SOTA方法。

25.4 关键架构：Autoformer

Autoformer（Wu et al., 2021）走了一条不同的路——它用自相关机制（Auto-Correlation Mechanism）替代了自注意力。

自相关机制的灵感来源于信号处理中的自相关函数。它通过计算序列与其延迟版本之间的相似度，直接发现序列中的周期性依赖关系。相比自注意力，自相关机制有两个优势：一是复杂度低（O(L log L)），二是天然适合发现季节性模式。

Autoformer还将时序分解（Trend-Seasonality Decomposition）深度集成到了Transformer的每一层中，让模型能够逐步分离出趋势成分和季节成分。这个设计非常优雅——它把传统时间序列分析的思想（分解）和深度学习的表达能力结合在了一起。

25.5 关键架构：PatchTST

PatchTST（Nie et al., 2023）提出了一个简单但有效的思路：将时间序列分割成多个"补丁"（Patch），每个补丁包含连续的几个时间步（如16步），然后在这些补丁上应用Transformer。

Patch的思路有两个好处：第一，减少了序列长度——如果原始序列有1024步，每个patch包含16步，那么Transformer只需要处理64个patch，计算量大降；第二，每个patch本身就是一个局部语义单元（如"一周的模式"、"一个季节的片段"），比单个时间点更有意义。

PatchTST采用了通道独立（Channel Independent）的策略——在多变量序列中，每个变量单独处理，而不是把所有变量拼接到特征维度。这简化了模型设计，且在大多数数据集上效果更好。

25.6 关键架构：Temporal Fusion Transformer

Temporal Fusion Transformer（TFT, Lim et al., 2019）是Google为多步预测设计的Transformer变体，它的最大特色是可解释性。

TFT包含多个精心设计的模块：变量选择网络（Variable Selection Network）自动选择重要的输入变量；门控残差网络（GRN）提供灵活的跳过连接；而最重要的是，它的多头注意力层可以生成时间维度的注意力权重，告诉我们模型在做预测时"关注了历史中的哪些时间点"。

这种可解释性在实践中非常宝贵——业务方不仅想知道预测结果，还想知道"为什么这么预测"。TFT的注意力热力图可以直观地展示哪些历史时间点对当前预测最重要，极大增强了模型的可信度。

实用建议：如果你是第一次将Transformer用于时间序列，可以按这个顺序尝试：先试 PatchTST（简单高效），如果数据量很大且序列很长，试 Informer；如果数据有明显的季节性，试 Autoformer；如果业务方要求可解释性，TFT 是最佳选择。

25.7 2026年的视角：混合架构成为主流

到了2026年，纯粹的"单架构"方法越来越少见了。从2024年以来的趋势看，多模块混合架构（Hybrid Architecture）已经成为时序预测的主流。典型的混合设计包括：

  • CNN + Transformer：CNN先做下采样和局部特征提取，减少序列长度，再交给Transformer做全局建模。
  • CNN + LSTM + Attention：CNN提取局部模式，LSTM建模中等长度的依赖，注意力层做长程依赖的精准聚焦。这种"三段式"设计在很多工业场景中效果出色。
  • 多尺度融合：在不同时间尺度（日、周、月）上分别建模，再将各尺度的表示融合。

站在现在的角度看，RNN、CNN和Transformer并非"三选一"的关系，而是各有专长的组件。好的时间序列模型，往往是这些组件在合适的位置协同工作的产物。

下表总结了本章介绍的几种Transformer变体的关键特性，方便你在选型时参考：

25.9 小结

本章我们学习了Transformer在时间序列中的应用。从自注意力机制开始，了解了它如何让序列任意两个位置直接"对话"；然后学习了Informer、Autoformer、PatchTST和TFT四种专门为时序设计的Transformer变体，它们各有侧重；最后，我们从2026年的视角看到了混合架构成为主流的趋势。

到这里，深度学习的三个篇章就结束了。这三章（RNN、CNN、Transformer）共同构成了现代时序预测的深度学习基石。但深度学习的脚步从未停下——下一章，我们将探讨更前沿的方向：基础模型（Foundation Models）和扩散模型（Diffusion Models）在时间序列中的应用。这些模型正在重新定义"时序预测能达到什么程度"这个问题的答案。
