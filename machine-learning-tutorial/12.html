12.1 两个"简单但有效"的算法

这一章我们把两个经典算法放在一起讲。朴素贝叶斯和KNN的共同点是：思想极其简单，但特定场景下效果出奇好。它们是ML入门必学的"两座桥梁"——朴素贝叶斯连接概率论和分类，KNN连接距离度量和决策。

12.2 朴素贝叶斯：一个"错误"的假设，却出奇好用

朴素贝叶斯的核心就是贝叶斯定理（第3章学过）：

P(类别|特征) ∝ P(特征|类别) × P(类别)

"朴素"在哪里？它假设所有特征之间相互独立——给定类别后，每个特征的条件概率是独立的。这个假设在现实中几乎不成立（"身高"和"体重"显然相关），但神奇的是，朴素贝叶斯在文本分类中效果极好。原因？虽然概率估计可能不准确（因为独立性假设被违反），但分类排序往往是对的——而分类只需要排序。

三种变体

变体特征分布假设适用场景
Gaussian NB连续值，每个特征服从高斯分布通用连续特征（如鸢尾花分类）
Multinomial NB离散计数（如词频）文本分类、垃圾邮件过滤
Bernoulli NB二值特征（出现/不出现）短文本、词袋模型中使用"是否出现"
from sklearn.naive_bayes import GaussianNB, MultinomialNB
from sklearn.feature_extraction.text import CountVectorizer

# 垃圾邮件过滤：MultinomialNB + 词袋模型
texts = ["免费领取大奖", "明天开会讨论", "恭喜你中奖了", "项目进展汇报"]
labels = [1, 0, 1, 0]  # 1=垃圾, 0=正常

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(texts)

nb = MultinomialNB(alpha=1.0)  # alpha是拉普拉斯平滑参数
nb.fit(X, labels)
print(f"预测概率: {nb.predict_proba(vectorizer.transform(['免费百万大奖']))}")
💡 拉普拉斯平滑（α）：如果训练集中某词从未在垃圾邮件中出现过，它的概率是0。在朴素贝叶斯中，一个0概率会让整个乘积变成0。α参数给每个词计数加α，保证没有概率是0。α=1（默认）就是经典的拉普拉斯平滑。这个简单的技巧解决了"零概率灾难"。

12.3 KNN：连"训练"都没有的算法

K近邻算法（K-Nearest Neighbors）可能是所有ML算法中最"懒"的——它根本没有显式的训练过程。当需要预测一个新样本时，KNN在训练集中找离它最近的K个邻居，看这K个邻居多数属于哪个类，就判为哪个类。

KNN没有"训练"时间，但预测很慢——每次预测都要扫描整个训练集计算距离。这和大多数算法相反（训练慢预测快）。

三个关键选择

  1. K值：K太小 → 对噪声敏感，过拟合。K太大 → 邻居里混入其他类，边界变模糊。通常用交叉验证选K
  2. 距离度量：欧氏距离（默认，适合连续值）、曼哈顿距离（适合高维数据）、余弦相似度（适合文本/高维稀疏向量）
  3. 投票权重：uniform（每人一票）vs distance（距离近的权重更大）。后者通常更好
from sklearn.neighbors import KNeighborsClassifier
from sklearn.preprocessing import StandardScaler

# KNN对尺度极其敏感，不标准化等于让大尺度特征独裁
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

knn = KNeighborsClassifier(n_neighbors=5, weights='distance', metric='minkowski', p=2)
knn.fit(X_train_scaled, y_train)  # 实际上只是"记住"数据，不做计算

# 找最优K值
k_range = range(1, 31)
scores = []
for k in k_range:
    knn = KNeighborsClassifier(n_neighbors=k)
    score = cross_val_score(knn, X_train_scaled, y_train, cv=5, scoring='accuracy').mean()
    scores.append(score)
best_k = k_range[np.argmax(scores)]
print(f"最佳K值: {best_k}")

12.4 维度灾难：KNN的阿喀琉斯之踵

KNN最大的敌人不是噪声，而是高维度。直觉：在1维空间，大多数点都在附近；在100维空间，所有点都离得很远——最近邻和最远邻的距离差异变得微不足道，距离失去了意义。

如果特征维度d > 20，KNN的效果通常急剧下降。这时候需要降维（PCA等）或者换一个不受维度灾难影响的算法（如树模型或SVM）。

⚠️ 两个常见错误：(1) 在KNN之前不做标准化——特征的量纲差异直接毁了距离计算。(2) 特征维度过高还硬用KNN——1000维数据下KNN基本等于随机猜测。如果非要在这个数据上用KNN，先用PCA降到20维以内。还有K-D Tree和Ball Tree可以加速最近邻搜索，但治标不治本。

12.5 对比总结

特性朴素贝叶斯KNN
训练速度极快（只统计计数）极快（不训练）
预测速度快慢（需扫描全部数据）
特征尺度敏感不敏感（高斯NB自适应）极其敏感
线性可分是（线性决策边界）否（任意复杂边界）
最佳场景文本分类、实时预测小数据、低维度、推荐系统
