大家好！如果你一路跟着课程走到了这里，你会发现我们已经学了很多算法：从 A* 到 RRT*，从 DWA 到 ORCA。这些算法各有特色，但它们有一个共同点——都是"专用"算法，各自用一套特定的启发式规则或采样策略来求解路径规划问题。而今天我们要换一个视角，把整个路径规划问题放到数学优化（Mathematical Optimization）的框架下来思考。这种方法的好处是：一旦你有了一个统一的数学建模方式，你就可以利用各种现成的优化求解器来求解，而不用每次都从头发明一个新算法。

讲个我自己的经历。有次我在做一个 AGV 的轨迹规划，要求路径不仅要避开障碍物，还要尽可能平滑（毕竟运的是精密仪器，颠簸不得），同时还要满足加速度限制和转向角限制。我用 A* 算了一条路径，然后做后处理平滑——结果平滑后的路径穿过了障碍物，来回调了一个多月。后来我的师兄看了一眼说："你为什么不直接把它写成优化问题？" 他用三行数学公式就把我的问题定义清楚了，然后用一个开源的优化求解器一个小时就搞定了。那一刻我深刻理解了什么叫"数学的力量"。

路径规划作为优化问题

把路径规划写成优化问题，通常包含三个要素：

1. 决策变量（Decision Variables）——你需要确定的东西。在路径规划中，决策变量就是路径上的关键点（waypoints）或控制输入。比如：x₀, x₁, ..., x_N 表示路径上的 N+1 个位置点，u₀, u₁, ..., u_{N-1} 表示每个段上的控制量（加速度、转向角等）。

2. 目标函数（Objective Function / Cost Function）——你想最小化的东西。可以是路径总长度、总时间、能量消耗、路径平滑度（加速度平方和）、障碍物安全距离倒数的惩罚等。通常是一个加权和：

min J = w₁ · (路径长度) + w₂ · (平滑度) + w₃ · (障碍物惩罚)

3. 约束条件（Constraints）——你必须满足的条件。包括：

  • 运动学约束——车辆不能原地转向、最大转弯半径有限
  • 动力学约束——最大加速度、最大速度、刹车距离
  • 避障约束——路径上的每个点与障碍物的距离大于安全阈值
  • 边界约束——起点和终点的位置、姿态固定

形式化地写出来就是这样：

minimize    J(x, u) = Σ c(x_i, u_i)          // 目标函数：累计代价
subject to  x_{i+1} = f(x_i, u_i)            // 动力学模型
            x_0 = x_start                     // 初始条件
            x_N = x_goal                      // 终点条件
            g(x_i, u_i) ≤ 0                   // 不等式约束（避障、限速等）
            h(x_i, u_i) = 0                   // 等式约束（姿态要求等）

一旦你将路径规划写成了这样的形式，你就有了强大的工具箱可以利用——二次规划（QP）、序列二次规划（SQP）、内点法、梯度下降、甚至商用求解器（如 Gurobi、CPLEX）。这些求解器经过几十年的优化，比任何人手写的专用规划器都要高效和稳定。

MPC模型预测控制

模型预测控制（Model Predictive Control, MPC）是将优化方法用于路径规划/控制的最典型代表。它的核心流程是：

  1. 在每一时刻，利用当前状态和系统的动力学模型，优化求解一个固定时间窗口（比如未来 3 秒）内的最优控制序列
  2. 只执行这个序列中的第一步控制量
  3. 进入下一时刻，利用最新的传感器测量更新状态，重新求解优化问题
  4. 重复——这就是"滚动优化（Receding Horizon）"

MPC 最大的优势是可以显式地处理约束。不论是避障约束、速度限制、加速度限制，还是转向角限制，你都可以把它们写成不等式约束，然后求解器会保证输出结果满足所有约束。这比 A* + 后处理平滑的方法要优雅得多。

下面是一个最小化的 MPC 路径规划器实现。它使用梯度下降来求解一个简单的"无碰撞 + 平滑"轨迹优化问题：

import numpy as np

class SimpleMPCPlanner:
    """
    一个简化的 MPC 路径规划器。
    在二维平面上规划一条从 start 到 goal 的路径，
    避开已知的圆形障碍物，同时最小化路径长度和加速度。
    """
    def __init__(self, dt=0.1, horizon=20):
        self.dt = dt          # 时间步长
        self.N = horizon      # 规划 horizon（路径点数量）
        self.dim = 2          # 二维空间

    def cost(self, trajectory, obstacles):
        """
        计算一条轨迹的总代价。
        trajectory: shape (N+1, 2) 的路径点序列
        """
        total = 0.0
        # 1. 路径长度代价：相邻点之间的欧几里得距离和
        for i in range(len(trajectory) - 1):
            dist = np.linalg.norm(trajectory[i+1] - trajectory[i])
            total += 50.0 * dist

        # 2. 平滑代价：加速度（二阶差分）的平方和
        for i in range(1, len(trajectory) - 1):
            acc = (trajectory[i+1] - 2*trajectory[i] + trajectory[i-1]) / (self.dt**2)
            total += 100.0 * np.sum(acc**2)

        # 3. 避障代价：对每个障碍物施加高斯惩罚
        for obs in obstacles:
            center = np.array(obs[:2])
            radius = obs[2]
            for point in trajectory:
                dist = np.linalg.norm(point - center)
                if dist < radius:
                    # 进入障碍物内部：极大惩罚
                    total += 10000.0 * (radius - dist)**2
                elif dist < radius + 1.0:
                    # 靠近障碍物：软惩罚
                    total += 1000.0 * (1.0 - dist + radius)**2

        return total

    def plan(self, start, goal, obstacles, iterations=500, lr=0.1):
        """
        使用梯度下降（有限差分法计算梯度）优化轨迹。
        """
        # 初始化：在起点和终点之间线性插值
        traj = np.zeros((self.N + 1, self.dim))
        for i in range(self.N + 1):
            alpha = i / self.N
            traj[i] = (1 - alpha) * np.array(start) + alpha * np.array(goal)

        # 固定起点和终点不变
        best_traj = traj.copy()
        best_cost = self.cost(traj, obstacles)

        for it in range(iterations):
            # 计算数值梯度（对内部路径点）
            grad = np.zeros_like(traj)
            eps = 1e-4
            for i in range(1, self.N):  # 不优化起点和终点
                for d in range(self.dim):
                    traj[i, d] += eps
                    c_plus = self.cost(traj, obstacles)
                    traj[i, d] -= 2 * eps
                    c_minus = self.cost(traj, obstacles)
                    grad[i, d] = (c_plus - c_minus) / (2 * eps)
                    traj[i, d] += eps

            # 梯度下降更新
            traj[1:self.N] -= lr * grad[1:self.N]

            # 评估新轨迹
            current_cost = self.cost(traj, obstacles)
            if current_cost < best_cost:
                best_cost = current_cost
                best_traj = traj.copy()

            # 学习率衰减
            if it % 100 == 0:
                lr *= 0.5
                print(f"迭代 {it}: 代价 = {current_cost:.4f}")

        return best_traj


# ---------- 使用示例 ----------
if __name__ == "__main__":
    planner = SimpleMPCPlanner(dt=0.1, horizon=30)

    start = (0.0, 0.0)
    goal = (10.0, 10.0)
    # 障碍物定义为 [x, y, radius]
    obstacles = [(3.0, 3.0, 1.5), (6.0, 7.0, 1.2), (8.0, 4.0, 1.0)]

    trajectory = planner.plan(start, goal, obstacles)

    print(f"规划完成！路径包含 {len(trajectory)} 个路径点")
    print(f"起点: {trajectory[0]}, 终点: {trajectory[-1]}")

    # 你可以在这里绘制轨迹验证避障效果

这段代码展示了优化方法的精髓：你不需要告诉求解器"向左走还是向右走"，你只需要定义好代价函数和约束条件，求解器自己会找到最优的走法。代价函数中的权重（50.0 给路径长度、100.0 给平滑度、10000.0 给避障）决定了不同目标之间的权衡——调整这些权重就相当于"调教"规划器的行为。

当然，这个简化版的 MPC 使用了梯度下降和数值梯度，效率不高。真实的 MPC 通常会利用问题的结构（比如二次型代价、线性动力学）来使用更高效的求解器（如 OSQP、qpOASES），可以在毫秒级别求解。

梯度优化方法

除了 MPC，路径规划领域还有几个著名的基于梯度的轨迹优化方法，值得你了解：

CHOMP（Covariant Hamiltonian Optimization for Motion Planning）——由 CMU 的 Matt Zucker 等人于 2013 年提出。CHOMP 的核心思想是对"轨迹"这个整体进行优化，使用函数空间的梯度（而非参数空间的梯度），将平滑度代价和避障代价用协变梯度下降（covariant gradient descent）联合优化。它的标志性贡献是引入了"梯度场"的概念——障碍物产生的排斥力以梯度场的形式作用在轨迹上，推动轨迹远离障碍物，同时平滑项防止轨迹变得过于曲折。

STOMP（Stochastic Trajectory Optimization for Motion Planning）——由 Mrinal Kalakrishnan 等人提出。与 CHOMP 的梯度下降不同，STOMP 使用随机优化：它生成大量带噪声的轨迹候选，然后选择其中代价最低的。这种方法不需要梯度信息，因此可以处理不可微的约束条件（比如某些障碍物形状不光滑）。代价是计算量较大。

TrajOpt——由 John Schulman（OpenAI 联合创始人，TRPO/PPO 的作者）在其博士期间提出。TrajOpt 使用序列凸优化（Sequential Convex Optimization）逐次逼近的方式求解带约束的轨迹优化问题。它在每一步将非凸的避障约束线性化，然后求解一个凸优化子问题，收敛速度非常快。

下表对比了这几种方法的差异：

方法 优化方式 是否需要梯度 约束处理 收敛速度 典型应用
MPC 滚动时域优化 是（或转化为 QP） 显式约束（硬约束） 快（几毫秒到几十毫秒） 自动驾驶、无人机控制
CHOMP 协变梯度下降 是 软约束（惩罚函数） 中等 机械臂运动规划
STOMP 随机优化 否 软约束（可处理不可微约束） 慢（需要大量采样） 复杂障碍物环境
TrajOpt 序列凸优化 是（雅可比矩阵） 显式约束（线性化后为硬约束） 快（通常 < 1 秒） 高维机械臂、接触丰富的任务

ROS2 Nav2实战

如果你觉得这些优化方法的理论还是太抽象了，那我们来点实战的——ROS2 Nav2 是目前最流行的机器人导航框架，它提供了一套可插拔的路径规划系统，让你可以在实际机器人上验证不同的规划器。

Nav2 的核心架构中，有两个关键组件：

  • Global Planner——全局路径规划器，负责在已知地图上规划从起点到终点的全局路径。默认使用 NavFn（基于 A* 的栅格规划器），但可以替换为 Smac Planner（Hybrid-A*，考虑运动学约束）或 Planner 插件
  • Local Planner / Controller——局部路径规划器 / 控制器，负责在全局路径的引导下，在局部环境中进行平滑跟踪和避障。默认使用 DWB（Dynamic Window-based），也可以使用 MPPI（Model Predictive Path Integral，一种基于采样的 MPC 变体）
Nav2 小技巧：更换规划器只需改一行配置！在 Nav2 的配置文件中，全局规划器的选择通过 planner_plugins 参数指定。例如，从默认的 NavFn 切换到 Smac Planner（Hybrid-A*），你只需要将 planner_plugins: ['NavFn'] 改为 planner_plugins: ['SmacPlanner']，并在配置中设置对应的参数（如网格分辨率、搜索窗口大小）。这种"插件化"的设计使得 Nav2 成为了测试不同规划算法的理想平台——你可以在仿真中一键切换规划器，对比它们的表现而不用改任何代码。如果你想知道不同规划器在你自己构建的地图上的实际表现，直接用 ros2 launch nav2_bringup navigation_launch.py 启动 Nav2，然后用 Rviz2 上的 "2D Pose Estimate" 和 "2D Nav Goal" 进行可视化测试即可。

Nav2 的插件生态还在迅速壮大中。除了官方的 NavFn 和 Smac Planner，社区还贡献了多种规划器插件，包括基于 RRT 的、基于状态晶格（State Lattice）的、以及基于优化的规划器。如果你想在真实机器人上部署前面学到的各种算法，Nav2 是一个绝佳的起点——它帮你处理好了传感器融合、定位、地图管理、控制执行等底层细节，让你可以专注于规划算法的设计和调优。
