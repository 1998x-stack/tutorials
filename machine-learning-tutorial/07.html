7.1 99%准确率也可能是垃圾模型

先讲一个让我印象深刻的真实案例。一个同事兴冲冲地说他的信用卡欺诈检测模型准确率99.8%，我问他欺诈样本占比多少，他说大概0.2%。真相瞬间大白了：如果模型对所有交易都预测"正常"，准确率就是99.8%。这个模型什么都没学到，但指标看起来完美。这就是为什么选择合适的评估指标比模型本身更重要。

7.2 分类问题的评估指标

混淆矩阵：所有分类指标的母体

一切分类指标都来自混淆矩阵的四个数字：

预测为正预测为负
实际为正TP（真正例）FN（假负例，漏报）
实际为负FP（假正例，误报）TN（真负例）

这四个数字衍生出所有分类指标：

指标公式关注什么适用场景
Accuracy(TP+TN)/ALL整体正确率类别平衡时
PrecisionTP/(TP+FP)预测为正的里面有多少是对的不能误报：垃圾邮件检测
RecallTP/(TP+FN)实际为正的里面找到了多少不能漏报：癌症筛查
F1 Score2PR/(P+R)Precision和Recall的调和平均类别不平衡时的综合指标
AUC-ROCROC曲线下面积模型排序能力二分类通用
💡 怎么记P和R：Precision = "我说对的有多少是对的"（精准度），Recall = "该找出来的我找出来了多少"（召回率）。在搜索场景里，Precision是搜出来的结果里相关的占多少，Recall是相关的结果里搜出来了多少。宁可漏掉也不误报 → 高Precision；宁可误报也不漏掉 → 高Recall。

7.3 回归问题的评估指标

指标公式特点何时用
MSE平均(真实-预测)²对大误差惩罚极重损失函数（可导）
RMSE√MSE和原数值同单位，可读汇报给业务方
MAE平均|真实-预测|对异常值不敏感有离群值的时候
R²1-SS_res/SS_tot解释了多少方差通用评估，-∞到1
MAPE平均|(真实-预测)/真实|百分比误差和业务方沟通用
⚠️ R²的陷阱：R²不是"准确率百分比"。R²=0.8不是说模型80%准确，而是模型解释了目标变量80%的方差。而且R²会随着特征数量增加而"虚高"——哪怕你加的是随机噪声特征。所以要用调整后的R²（Adjusted R²），或者在测试集上看R²。

7.4 交叉验证：别被一次划分骗了

很多人做ML的流程是：固定一个train_test_split的random_state → 调参调到这个划分上效果最好 → 沾沾自喜。但实际上，你的"好结果"可能只是运气好——恰好碰到了一个对你模型友好的数据划分。

这就是为什么需要交叉验证（Cross-Validation）：

  • K-Fold CV：把数据分成K份，用K-1份训练、1份验证，轮流K次，取平均。K通常取5或10
  • Stratified K-Fold：在分类问题中保证每折的类别比例一致 —— 分类问题默认用这个
  • TimeSeriesSplit：对时序数据，保证训练集的时间在验证集之前 —— 时序预测专用
from sklearn.model_selection import cross_val_score, StratifiedKFold

# 分类问题：必须用StratifiedKFold确保每折类别比例一致
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
cv_scores = cross_val_score(model, X, y, cv=skf, scoring='f1')
print(f"F1: {cv_scores.mean():.4f} ± {cv_scores.std():.4f}")

# 如果std太大（比如 > mean的10%），说明模型对数据划分很敏感
# 这时候需要更多的数据，或者更强的正则化

7.5 偏差-方差权衡：ML永恒的主题

这是机器学习中最核心的概念之一，理解了它，你就理解了为什么简单模型和复杂模型各有各的用武之地。

高偏差 (Underfitting)高方差 (Overfitting)
表现训练集和测试集效果都差训练集极好，测试集差
原因模型太简单，学不到规律模型太复杂，记住了噪声
解决方法加特征、换复杂模型、减少正则化加数据、加正则化、换简单模型
典型模型线性回归拟合非线性数据深度决策树记住训练集每个点
🔑 核心洞察：总误差 = 偏差² + 方差 + 不可约误差。偏差和方差像跷跷板——降低一个往往升高另一个。最好的模型是在两者之间找到平衡点。交叉验证是诊断过拟合最直接的工具：训练集CV分数 >> 测试集CV分数 → 过拟合；两者都很低 → 欠拟合。

7.6 学习曲线：最被低估的诊断工具

画一条学习曲线（随着训练样本量增加，训练误差和验证误差的变化）能帮你快速诊断模型的问题：

from sklearn.model_selection import learning_curve

train_sizes, train_scores, test_scores = learning_curve(
    model, X, y, cv=5, scoring='r2',
    train_sizes=np.linspace(0.1, 1.0, 10)
)

train_mean = train_scores.mean(axis=1)
test_mean = test_scores.mean(axis=1)

plt.plot(train_sizes, train_mean, 'o-', label='训练集')
plt.plot(train_sizes, test_mean, 'o-', label='验证集')
plt.xlabel('训练样本数')
plt.ylabel('R²')
plt.legend()
plt.show()

解读：如果两条线收敛在一起且分数低 → 欠拟合（加模型复杂度）。如果两条线差距很大 → 过拟合（加数据或正则化）。如果两条线收敛且分数高 → 模型到位了。

7.7 模型选择实战：多模型对比

实际项目中，你应该并行比较多个模型，而不是一个一个试：

from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.svm import SVC

models = {
    'Logistic Regression': LogisticRegression(max_iter=1000),
    'Random Forest': RandomForestClassifier(n_estimators=100),
    'Gradient Boosting': GradientBoostingClassifier(n_estimators=100),
    'SVM': SVC(kernel='rbf')
}

results = {}
for name, model in models.items():
    cv_scores = cross_val_score(model, X_train, y_train,
                                cv=5, scoring='f1')
    results[name] = {'mean': cv_scores.mean(), 'std': cv_scores.std()}
    print(f"{name:20s}: F1 = {cv_scores.mean():.4f} ± {cv_scores.std():.4f}")

best_model_name = max(results, key=lambda k: results[k]['mean'])
print(f"\n🏆 最佳模型: {best_model_name}")
