第25章 可视化与仪表盘

大家好，我是你们的讲师。先说一个真实场景：你花了两周时间构建了一个复杂的量化因子模型，回测夏普比率1.8，最大回撤15%，各项指标都很漂亮。你信心满满地走进会议室，打开你的Jupyter Notebook，准备向投资委员会展示——结果对方看了一眼说："这堆彩色格子到底是什么？你做的东西能不能让我三秒钟看懂？"

这不是玩笑——在金融行业，沟通能力往往比分析能力更能决定你的影响力。而可视化和仪表盘，就是分析师最重要的沟通工具。一张好的图表可以抵一千个文字，一个好的仪表盘可以让你的分析成果被每天使用。本章将带你从零搭建金融可视化的完整技能栈：从K线图到交互式仪表盘，从静态图表到可部署的Web应用。

25.1 金融图表类型学

金融分析中有几十种可用的图表类型，但真正高频使用的只有少数几种。关键不在于会画多少种图，而在于知道在什么场景下用哪种图。

图表类型 金融用途 适用场景 替代方案
K线图（Candlestick） 展示OHLC（开高低收） 股票/期货/外汇日度行情 美国线（HLC）
收益分布直方图 展示收益率分布形态 评估风险、检查正态性 箱线图、小提琴图
相关性热力图 展示多资产间相关性 组合构建、风险分散 散点图矩阵
回撤曲线（Drawdown） 展示从峰值的回撤过程 评估策略风险特征 水下曲线（Underwater Plot）
瀑布图（Waterfall） 展示逐步变化过程 利润拆解、现金流拆解 堆积柱状图
面积图 展示累计收益/规模变化 净值曲线、资产规模 折线图
热力图（日历） 展示某指标在时间上的聚集 月度收益率模式、波动率日历 颜色编码表格

选择图表的黄金法则是：不要让读者花力气去理解你的图。如果你需要加一大段文字来解释这个图在说什么，那不是读者的问题，是你的图表选错了。

一个常见的直觉错误：饼图在金融报告中非常流行，但几乎总是错误的选择。人类的眼睛很难准确比较不同扇区的大小。如果你需要展示占比，用横向条形图替代——条形图可以按大小排序，读者一眼就能看到"谁最大、谁最小"。

25.2 Plotly交互式图表

Plotly已经成为Python交互式可视化的首选库。它的核心优势在于：一行代码就能生成可缩放、可悬停、可筛选的HTML图表，无需任何前端知识。

Plotly Express快速出图

Plotly Express是Plotly的高级API，语法极简：

import plotly.express as px
import pandas as pd

# 假设 df 包含多只股票的历史日收益率
# 计算相关性矩阵
corr_matrix = df.pivot_table(
    index="date", columns="stock", values="return"
).corr()

# 一行代码生成交互式热力图
fig = px.imshow(
    corr_matrix,
    text_auto=".2f",
    color_continuous_scale="RdBu_r",
    title="股票收益率相关性矩阵",
    width=800, height=700
)
fig.show()

交互式K线 + 成交量联动

专业的K线图需要同时显示价格和成交量，并且两者是联动的——缩放到某个时间段时，成交量的时间范围自动跟随：

import plotly.graph_objects as go
from plotly.subplots import make_subplots
import akshare as ak

# 获取数据
df = ak.stock_zh_a_hist(symbol="600519", period="daily",
                        start_date="20250101", end_date="20260630")
df.rename(columns={"日期": "date", "开盘": "open", "收盘": "close",
                   "最高": "high", "最低": "low", "成交量": "volume"},
          inplace=True)
df["date"] = pd.to_datetime(df["date"])

# 创建子图：上方K线 + 下方成交量
fig = make_subplots(
    rows=2, cols=1, shared_xaxes=True,
    vertical_spacing=0.05,
    row_heights=[0.7, 0.3]
)

# K线
fig.add_trace(go.Candlestick(
    x=df["date"], open=df["open"], high=df["high"],
    low=df["low"], close=df["close"],
    name="茅台"
), row=1, col=1)

# 成交量柱状图（用颜色区分涨跌）
colors = ["red" if c >= o else "green"
          for c, o in zip(df["close"], df["open"])]
fig.add_trace(go.Bar(
    x=df["date"], y=df["volume"], name="成交量",
    marker_color=colors, opacity=0.6
), row=2, col=1)

# 布局优化
fig.update_layout(
    title="贵州茅台 K线图 + 成交量",
    xaxis_rangeslider_visible=False,
    hovermode="x unified",
    height=800
)
fig.update_yaxes(title_text="价格", row=1, col=1)
fig.update_yaxes(title_text="成交量", row=2, col=1)

fig.show()

自定义Hover模板

Plotly的Hover信息完全可定制，你可以让鼠标悬停时显示任何你需要的信息：

fig.update_traces(
    hovertemplate="%{x}
" +
                  "开盘: %{open:.2f}
" +
                  "最高: %{high:.2f}
" +
                  "最低: %{low:.2f}
" +
                  "收盘: %{close:.2f}
" +
                  "涨跌幅: %{customdata[0]:.2%}
" +
                  ""
)
保存与分享：Plotly图表可以用 fig.write_html("chart.html") 保存为独立的HTML文件。这个文件不需要任何服务器——双击就能在浏览器打开，对方也不需要装Python。这是分享分析结果给不懂技术的同事或客户的最佳方式。

25.3 Power BI金融仪表盘

当分析成果需要团队持续使用、而不是一次性展示时，就需要一个真正的仪表盘（Dashboard）工具了。Power BI是2026年金融行业使用最广泛的企业级BI工具。

一个典型的金融仪表盘构建流程如下：

  1. 数据接入——从Excel、CSV、SQL Server或API导入数据。Power Query提供图形化的数据清洗界面，可以完成大部分ETL工作。
  2. DAX度量值——用DAX（Data Analysis Expressions）语言定义核心指标。例如 TotalReturn = SUMX(Returns, [Return] * [Weight])。DAX的强大之处在于它支持时间智能函数——TOTALYTD、SAMEPERIODLASTYEAR 等——这在金融分析中非常有用。
  3. 动态筛选——设置切片器（Slicer）让用户按时间、板块、资产类别等维度自由筛选。一个好的筛选器设计应该让用户在3次点击内找到他关心的数据。
  4. 自动刷新与发布——将仪表盘发布到Power BI Service，设置定时刷新（如每个交易日收盘后自动更新）。管理层打开手机Power BI App就能看到最新的组合收益。
踩坑经验：我曾经设计过一个"完美"的Power BI仪表盘——12个页面、50+指标、所有维度都可筛选。然后我把它交给了投委会主席。第二天他让助理转告我："我要的只是一页纸，上面有三个数字——今天赚了多少、今年赚了多少、最大回撤是多少。" 这个故事告诉我们：仪表盘的设计必须从用户的需求出发，而不是从数据的丰富度出发。先做一个"极简版"，用户说"再加一个XX指标"的时候再加，远比一开始就堆满指标要好。

25.4 Streamlit轻量金融App

Power BI适合企业级部署，但如果你只是想把自己的一堆Python分析脚本包装成一个可交互的Web应用，SharePoint和同事共享——Streamlit是最快的方式。

Streamlit是一个Python库，让你用纯Python代码（不需要HTML/CSS/JavaScript）构建数据应用。以下是一个简单的策略回测仪表盘：

import streamlit as st
import akshare as ak
import pandas as pd
import plotly.graph_objects as go

st.set_page_config(page_title="量化策略回测器", layout="wide")
st.title("📊 SMA交叉策略回测仪表盘")

# 侧边栏参数输入
with st.sidebar:
    st.header("策略参数")
    symbol = st.text_input("股票代码", value="600519")
    start_date = st.date_input("开始日期", pd.to_datetime("2024-01-01"))
    end_date = st.date_input("结束日期", pd.to_datetime("2026-06-30"))
    fast_ma = st.slider("快线周期", 5, 50, 5)
    slow_ma = st.slider("慢线周期", 20, 200, 20)
    run_btn = st.button("运行回测")

if run_btn:
    with st.spinner("正在获取数据并计算..."):
        # 获取数据
        df = ak.stock_zh_a_hist(
            symbol=symbol, period="daily",
            start_date=start_date.strftime("%Y%m%d"),
            end_date=end_date.strftime("%Y%m%d")
        )
        df.rename(columns={"日期": "date", "收盘": "close",
                           "开盘": "open", "最高": "high",
                           "最低": "low", "成交量": "volume"},
                  inplace=True)
        df["date"] = pd.to_datetime(df["date"])
        df.sort_values("date", inplace=True)

        # 计算均线
        df["SMA_fast"] = df["close"].rolling(window=fast_ma).mean()
        df["SMA_slow"] = df["close"].rolling(window=slow_ma).mean()

        # 生成信号
        df["signal"] = (df["SMA_fast"] > df["SMA_slow"]).astype(int)
        df["returns"] = df["close"].pct_change()
        df["strategy_ret"] = df["signal"].shift(1) * df["returns"]

        # 展示指标
        col1, col2, col3, col4 = st.columns(4)
        total_ret = (1 + df["strategy_ret"]).prod() - 1
        buy_hold = (1 + df["returns"]).prod() - 1
        sharpe = (df["strategy_ret"].mean() / df["strategy_ret"].std()
                  * (252 ** 0.5))
        max_dd = (df["strategy_ret"].cumsum() -
                  df["strategy_ret"].cumsum().cummax()).min()

        col1.metric("策略总收益", f"{total_ret:.2%}")
        col2.metric("买入持有", f"{buy_hold:.2%}")
        col3.metric("夏普比率", f"{sharpe:.2f}")
        col4.metric("最大回撤", f"{max_dd:.2%}")

        # K线图
        fig = go.Figure()
        fig.add_trace(go.Candlestick(
            x=df["date"], open=df["open"], high=df["high"],
            low=df["low"], close=df["close"], name="K线"
        ))
        fig.add_trace(go.Scatter(
            x=df["date"], y=df["SMA_fast"],
            line=dict(color="blue", width=1), name=f"SMA {fast_ma}"
        ))
        fig.add_trace(go.Scatter(
            x=df["date"], y=df["SMA_slow"],
            line=dict(color="red", width=1), name=f"SMA {slow_ma}"
        ))
        fig.update_layout(height=500, xaxis_rangeslider_visible=False)
        st.plotly_chart(fig, use_container_width=True)

        # 数据预览
        with st.expander("查看原始数据"):
            st.dataframe(df.tail(50))

st.markdown("---")
st.caption("⚠️ 回测结果仅供参考，不构成投资建议。过往表现不代表未来收益。")

将以上代码保存为 app.py，然后运行 streamlit run app.py，浏览器会自动打开一个可交互的Web页面。如果需要分享给同事，可以免费部署到 share.streamlit.io 或使用公司内部的Streamlit Server。

mplfinance：快速绘制专业K线图

虽然Plotly和Power BI功能强大，但有时候你需要的就是一张干净、出版质量的静态K线图放在你的报告或研报里——这时候mplfinance是最好的选择。它是matplotlib的金融扩展包，专为K线图优化：

import mplfinance as mpf
import akshare as ak
import pandas as pd

# 获取数据
df = ak.stock_zh_a_hist(symbol="600519", period="daily",
                        start_date="20250501", end_date="20260630")
df.rename(columns={"日期": "date", "开盘": "Open", "收盘": "Close",
                   "最高": "High", "最低": "Low", "成交量": "Volume"},
          inplace=True)
df["date"] = pd.to_datetime(df["date"])
df.set_index("date", inplace=True)
df.sort_index(inplace=True)

# 确保mplfinance要求的OHLCV列名
df = df[["Open", "High", "Low", "Close", "Volume"]]

# 设置均线
ma_short = df["Close"].rolling(5).mean()
ma_long = df["Close"].rolling(20).mean()

# 绘制K线 + 成交量 + 均线
mpf.plot(
    df,
    type="candle",
    volume=True,
    mav=(5, 20),
    style="yahoo",
    title="贵州茅台 K线图 (SMA 5/20)",
    ylabel="价格 (元)",
    ylabel_lower="成交量",
    figsize=(14, 8),
    savefig="maotai_kline.png"  # 保存为高清图片
)

这段代码用到的列名必须是大写的Open/High/Low/Close/Volume——这是mplfinance的约定。如果列名不对，它会报错或画出来的图是空的。我第一次用的时候就是在这个小细节上卡了半小时。

总结一下什么时候用什么工具：如果你只是自己做研究，Jupyter Notebook里用mplfinance或Plotly就够了；如果你需要向客户或管理层做定期汇报，用Power BI建立持续更新的仪表盘；如果你想把自己写的分析工具分享给团队用，用Streamlit包装成一个Web应用。这三种技能并不互斥——一个好的金融分析师应该至少掌握其中两种。
