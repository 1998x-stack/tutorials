第4章 数据来源与采集策略

大家好，我是你们的讲师。到上一章为止，我们已经知道了数据有哪些类型，也知道了它们应该以什么格式存储。现在有一个更实际的问题摆在面前：这些数据到底从哪来？

你可能觉得答案很简单——从 HuggingFace 下载开源数据集就行了。但如果后训练数据工程真的这么简单，每个团队都能训出 GPT-4 了。事实上，数据来源的选择直接影响着模型的天花板：用什么样的数据，训出什么样的模型。这一章我们来系统地分析四种主流数据来源，以及它们的适用场景。

4.1 四种数据来源全景

后训练数据的来源可以归结为四大类：

  • 人工标注——雇佣标注人员根据任务规范撰写指令-回答对
  • 大模型生成——使用 GPT-4、Claude 等顶级模型生成训练数据
  • 真实用户采集——从产品对话日志中收集用户与模型的交互数据
  • 合成数据流水线——通过自动化管线（包含数据增强、课程生成等流程）批量制造数据

这四类来源各有优劣势，直接关系到你的数据预算、时间周期和最终模型效果。

4.2 各来源的优劣对比

维度 人工标注 模型生成 真实用户采集 合成数据流水线
数据质量 ⭐⭐⭐⭐⭐ 最高（专家标注） ⭐⭐⭐⭐ 高（取决于生成模型） ⭐⭐⭐ 中等（噪音多） ⭐⭐⭐⭐ 可控（依赖管线路由）
获取成本 最高（$1-10/条） 中等（API 费用） 低（已有产品） 高（基础设施投入）
规模上限 低（数千~数万条/月） 高（百万级/天） 取决于产品流量 极高（自动化不限量）
隐私风险 低（可控环境） 低（无用户数据） 高（含用户信息） 低（无用户数据）
时效性 慢（培训-标注-审核周期长） 快（即时生成） 实时（但需清洗周期） 中等（管线开发耗时）
多样性 依赖标注规范设计 依赖 prompt 工程设计 真实分布，最自然 系统化控制，可调节

从这张表可以看出，没有哪个来源是完美的。关键是根据你的项目阶段和目标做出权衡。

项目阶段参考：在探索阶段（验证数据策略是否有效），优先使用模型生成数据，成本低、迭代快。在冲刺阶段（追求最佳模型效果），引入人工标注数据做精细打磨。在生产阶段（持续优化），建立真实用户采集和合成数据流水线的飞轮机制。

4.3 2025-2026 趋势：合成数据崛起

2025 年是合成数据的"爆发之年"。几个标志性事件值得关注：

PiKa 3 万条合成数据打败 1000 万条专有数据。这个我们在第 1 章已经提到过——PiKa 的核心方法论是：不是简单地用模型生成大量数据，而是设计了一个多阶段的合成数据管线，包含难度控制、多样性采样和质量过滤。数据管线本身就是核心竞争力。

SPIN 自我博弈。2024-2025 年，SPIN（Self-Play Fine-Tuning）方法引起了广泛关注。它的核心思想是让模型和自己"下棋"——当前模型生成训练数据，然后用这些数据训练下一版模型，如此循环迭代。每一轮模型都在数据和训练上双重进化，形成一个自我强化的正向循环。

自动课程生成。2025-2026 年的一个前沿方向是让模型自动生成难度递增的训练课程。比如，在数学推理训练中，管线先生成简单的加减法，然后根据模型的表现自动升级到代数、微积分。这种自适应课程生成，让合成数据的效率得到了数量级的提升。

# 一个简化的自动课程生成器伪代码
class AutoCurriculumGenerator:
    def __init__(self, base_model, difficulty_levels):
        self.model = base_model
        self.levels = difficulty_levels  # ["easy", "medium", "hard", "expert"]
        self.performance = {}

    def generate_batch(self, level, count=1000):
        """在指定难度级别上生成训练数据"""
        prompt = f"Generate {count} {level}-level math problems with step-by-step solutions"
        return self.call_llm(prompt)

    def evaluate_and_advance(self):
        """评估模型在当前级别的表现，决定是否升级"""
        score = self.evaluate_on_level(self.current_level)
        if score > 0.85 and self.current_level_idx < len(self.levels) - 1:
            self.current_level_idx += 1
            print(f"模型已达到 {self.levels[self.current_level_idx-1]} 级别 85% 准确率，"
                  f"升级到 {self.levels[self.current_level_idx]}")
        return score
关键洞察：合成数据在 2025-2026 年的爆发，不是因为"合成"本身有多神奇，而是因为数据质量控制的系统化——难度分层、多样性控制、质量过滤、自我验证。一套完善的合成数据流水线，可以将优质数据的产出效率提升 100 倍以上。这将是未来两年后训练数据工程的核心竞争力。

4.4 混合数据策略

现实中，几乎没有团队会只依赖单一数据来源。最佳实践是根据场景特征设计混合策略：

应用场景 数据来源组合建议 理由
通用对话模型 70% 模型生成 + 20% 真实用户采集 + 10% 人工标注 模型生成覆盖指令多样性，真实数据提供自然分布，人工标注确保核心质量
医疗领域模型 80% 人工标注（专家） + 20% 模型生成（需医生审核） 医疗场景容错率极低，必须专家参与；模型生成辅助扩大覆盖
代码助手模型 50% 合成数据 + 30% 开源代码-指令数据 + 20% 人工标注 代码数据可以自动化验证（单元测试），合成数据效率最高
安全对齐数据 60% 红队测试生成 + 40% 人工标注 红队攻击需要人类的创造力和对抗思维
Agent/工具调用 70% 仿真环境合成 + 30% 人工标注轨迹 仿真环境可以低成本生成大量轨迹数据

混合策略的关键在于配比的动态调整。前期（模型能力较弱时）可以用更多模型生成数据快速扩大覆盖；中后期（模型需要精细化提升时）逐步增加人工标注和真实数据的比例。

4.5 一个踩坑故事：未经清洗的 ShareGPT 数据

2024 年初，我听说了一个令人警醒的案例。某团队想要快速构建一个高质量的中文对话模型，他们知道人工标注成本高、周期长，所以决定走一条"捷径"——直接从 ShareGPT 抓取用户与 ChatGPT 的真实对话数据作为训练集。

ShareGPT 数据确实诱人——真实用户提问、ChatGPT 回答、百万级别的规模、免费获取。没有比这更"划算"的数据来源了。团队一口气下载了 50 万条对话，简单过滤后直接用于 SFT 训练。

结果模型上线后出现了严重问题——模型会在对话中突然说出冒犯性内容。原因很简单：ShareGPT 数据中混入了大量用户发起的恶意 prompt——种族歧视、性别偏见、仇恨言论——以及某些情况下 ChatGPT 对这些 prompt 的不当回应。模型在学习如何回答问题的同时，也学到了这些有害内容。

教训：真实用户采集的数据天然包含"长尾毒药"——绝大多数对话是良性的，但少量有害样本足以污染整个模型。在使用真实用户数据前，必须建立多层过滤机制：关键词过滤 + 分类器过滤 + 人工抽检。没有经过严格清洗的真实数据，宁可不用的好。记住——数据不是越多越好，有毒的数据只会让你的模型更糟糕。

这个教训也引出了我们的下一个话题——获得数据只是第一步，如何评估数据质量？如何确保你采集到的数据是真正有用的？让我们进入第 5 章：数据质量评估框架。
