第23章 数据标注与格式

大家好，我是你们的讲师。这一章我们来聊一个"苦活"——数据标注。

你可能花了80%的时间标注数据，却只花20%的时间训练模型。但恰恰是这80%的数据工作决定了最终模型的上限——算法救不了垃圾数据。我在做第一个检测项目时，花了两周精心调优模型结构，mAP却始终在0.5徘徊。后来排查才发现，标注数据里有三分之一的边界框中心点标错了位置。修正标注后，同样的模型直接跳到了mAP 0.75。那一刻我深刻明白了一件事：数据质量，才是模型性能的第一道门槛。

这一章我们会系统学习数据标注的方方面面：标注工具的选择、主流数据集格式、格式之间的互相转换、数据集的正确组织方式，以及如何对标注数据进行质量验证。准备好了吗？我们开始。

标注工具的选择

工欲善其事，必先利其器。选择一款趁手的标注工具，能节省大量时间。目前市面上主流的选择主要有三大类：桌面端、Web端和云端平台。

LabelImg — 轻量级桌面工具

LabelImg 是一个用 Python + Qt 编写的开源标注工具，支持 YOLO 和 Pascal VOC 格式。它最大的优点是轻量——安装简单、界面直观，适合小规模数据集的快速标注。

# 安装 LabelImg
pip install labelimg

# 启动
labelimg

打开后界面简洁：左侧是文件列表，中间是图像显示区，右侧是标注属性面板。标注操作就是"画框 + 选类别"两步：点击"Create RectBox"按钮，在图像上拖出一个矩形框，然后选择或输入该框的类别标签。完成一张图后按 Ctrl+S 保存。

使用小技巧：LabelImg 支持自动保存模式（View → Auto Save mode），强烈建议开启。另外，预先在 data/predefined_classes.txt 中定义好所有类别名称，可以避免标注时反复输入同样的类别名。

CVAT — Web端协作平台

CVAT（Computer Vision Annotation Tool）是 Intel 开源的 Web 端标注平台，功能远比 LabelImg 强大。它支持多种标注类型：矩形框、多边形、关键点、语义分割等。团队协作是 CVAT 最大的杀手锏——多个标注员可以同时工作，任务可以分配和追踪。

CVAT 的部署方式：

# 使用 Docker 部署 CVAT
git clone https://github.com/opencv/cvat
cd cvat
docker compose up -d

# 访问 http://localhost:8080 开始使用

CVAT 内置了 AI 辅助标注功能（基于 Detectron2 和 YOLO 模型），可以先用模型自动标注，人工再微调——这能节省 60-80% 的标注时间。

Roboflow — 云端一站式平台

Roboflow 是一个商业化的数据标注和管理平台，集标注、管理、预处理、版本控制于一体。上传数据后，你可以在浏览器中标注，然后一键导出为 COCO、YOLO、VOC 等任意格式。它还内置了丰富的数据增强和预处理功能。

下表对比了三种工具的优劣势：

特性 LabelImg CVAT Roboflow
部署方式 pip 安装 Docker 部署 / 托管版 SaaS 云服务
协作能力 无 强（多用户、任务分配） 强（团队协作、版本管理）
AI辅助标注 无 内置（YOLO/Detectron2） 内置（多种模型）
格式支持 YOLO / VOC COCO / YOLO / VOC / 自定义 30+ 格式
适合场景 个人项目、小数据 团队协作、中等规模 企业级、大规模数据管理
费用 免费 免费开源 / 企业版付费 免费额度有限，Pro版付费

主流数据集格式详解

在检测领域，有三种格式你必须掌握：YOLO TXT、COCO JSON 和 Pascal VOC XML。你对它们的关系可以这样理解：YOLO格式是最简洁的，COCO格式是最通用的，VOC格式是历史最悠久的。

YOLO TXT 格式

YOLO 使用的标注格式是最简洁的——每个图像对应一个同名的 .txt 文件，每行代表一个目标。每行的内容是：class_id x_center y_center width height。注意：坐标值都是相对于图像宽度和高度的归一化值（0~1）。

# images/train/0001.jpg 对应的标注文件 labels/train/0001.txt
0 0.5 0.5 0.8 0.6      # person，中心在图像正中间，框宽80%，高60%
1 0.2 0.3 0.1 0.15     # car，中心在(0.2, 0.3)，框宽10%，高15%
2 0.7 0.8 0.05 0.08    # bicycle，中心在(0.7, 0.8)，小目标
关键注意事项：
  • 类别ID从0开始，必须连续——不能出现类别0和2但没有1
  • 坐标是归一化的，所以宽高可能大于1（当目标超出图像边界时）
  • 如果一张图中没有目标，标注文件为空文件即可——但不能缺失
  • 图像和标注文件必须同名（仅扩展名不同）

COCO JSON 格式

COCO 格式是学术界最广泛使用的检测数据格式。它是一个 JSON 文件，包含了所有图像的标注信息。结构大致如下：

{
  "images": [
    {"id": 1, "file_name": "0001.jpg", "width": 640, "height": 480},
    {"id": 2, "file_name": "0002.jpg", "width": 640, "height": 480}
  ],
  "annotations": [
    {"id": 1, "image_id": 1, "category_id": 1,
     "bbox": [320, 240, 512, 288], "area": 147456.0,
     "iscrowd": 0, "segmentation": []},
    {"id": 2, "image_id": 1, "category_id": 2,
     "bbox": [128, 144, 64, 72], "area": 4608.0,
     "iscrowd": 0, "segmentation": []}
  ],
  "categories": [
    {"id": 1, "name": "person", "supercategory": "person"},
    {"id": 2, "name": "car", "supercategory": "vehicle"}
  ]
}

COCO格式的bbox是[x, y, width, height]，其中x、y是框左上角的像素坐标。注意这和YOLO格式的中心点表示完全不同，做格式转换时最容易出错的就是坐标表示方式的转换。

Pascal VOC XML 格式

VOC 格式使用 XML 文件存储标注，每个图像对应一个 XML 文件。虽然现在直接用的人少了，但很多数据集（比如各类竞赛数据）仍然以 VOC 格式分发。

<annotation>
    <folder>images</folder>
    <filename>0001.jpg</filename>
    <size>
        <width>640</width>
        <height>480</height>
        <depth>3</depth>
    </size>
    <object>
        <name>person</name>
        <pose>Unspecified</pose>
        <truncated>0</truncated>
        <difficult>0</difficult>
        <bndbox>
            <xmin>64</xmin>
            <ymin>120</ymin>
            <xmax>576</xmax>
            <ymax>408</ymax>
        </bndbox>
    </object>
</annotation>

三种格式的核心区别在于坐标表示方式。下面的表格一目了然：

格式 坐标表示 归一化 扩展名
YOLO 中心点 (cx, cy, w, h) 是（/w, /h） .txt
COCO 左上角 (x, y, w, h) 否（绝对像素） .json
VOC 左上右下 (xmin, ymin, xmax, ymax) 否（绝对像素） .xml

格式转换实战

在实际项目中，你几乎肯定需要做格式转换。比如从网上下载的 COCO 格式数据集，YOLO 训练时需要转成 YOLO 格式。我写过一个格式转换的脚本，核心逻辑如下：

import json
import os

def coco_to_yolo(coco_json_path, output_dir, img_dir):
    """将 COCO JSON 转换为 YOLO TXT 格式"""
    with open(coco_json_path) as f:
        coco = json.load(f)

    # 建立 category_id 到 YOLO class_id 的映射
    cat_id_map = {cat["id"]: idx for idx, cat in enumerate(coco["categories"])}

    # 建立 image_id 到文件名的映射
    img_map = {img["id"]: img for img in coco["images"]}

    for ann in coco["annotations"]:
        img_info = img_map[ann["image_id"]]
        img_w, img_h = img_info["width"], img_info["height"]

        # COCO: [x, y, w, h] → YOLO: [cx, cy, w, h]（归一化）
        x, y, w, h = ann["bbox"]
        cx = (x + w / 2) / img_w
        cy = (y + h / 2) / img_h
        nw = w / img_w
        nh = h / img_h

        cls_id = cat_id_map[ann["category_id"]]
        line = f"{cls_id} {cx:.6f} {cy:.6f} {nw:.6f} {nh:.6f}\n"

        # 写入对应图像的 .txt 文件
        base_name = os.path.splitext(img_info["file_name"])[0]
        label_path = os.path.join(output_dir, f"{base_name}.txt")
        with open(label_path, "a") as f:
            f.write(line)

反过来，从 YOLO 格式转 COCO 格式也是类似——只是坐标运算的逆过程。关键是把握住坐标表示方式的换算关系：

# COCO → YOLO
cx = (x + w/2) / img_w
cy = (y + h/2) / img_h
nw = w / img_w
nh = h / img_h

# YOLO → COCO
x = (cx - nw/2) * img_w
y = (cy - nh/2) * img_h
w = nw * img_w
h = nh * img_h
常见踩坑：在做 COCO → YOLO 转换时，最容易犯的错误是忘记了归一化。COCO 的坐标是绝对像素值，YOLO 需要归一化到 [0,1]。如果你忘了除以图像宽高，模型训练时会出现 AssertionError: bbox must be in [0, 1]。别问我怎么知道的——我在这上面浪费了整整一个下午。

数据集组织规范

一个标准的检测数据集，目录结构应该清晰明了。Ultralytics YOLO 推荐的目录结构如下：

dataset/
├── images/
│   ├── train/        # 训练集图像
│   ├── val/          # 验证集图像
│   └── test/         # 测试集图像（可选）
├── labels/
│   ├── train/        # 训练集标注
│   ├── val/          # 验证集标注
│   └── test/         # 测试集标注（可选）
└── data.yaml         # 数据集配置文件

其中 data.yaml 的内容格式：

path: /path/to/dataset  # 数据集根目录
train: images/train      # 训练集图像目录
val: images/val          # 验证集图像目录
test: images/test        # 测试集图像目录（可选）

# 类别信息
nc: 3                    # 类别数量
names: ['person', 'car', 'bicycle']  # 类别名称列表
分区比例建议：一般推荐 70% 训练、20% 验证、10% 测试。如果数据量很小（少于1000张），可以考虑 80% 训练、20% 验证，不用单独留测试集，用 K-Fold 交叉验证来评估。划分时务必保证每个分区的类别分布尽量一致——不要出现训练集有 10 类但验证集只有 5 类的情况。

数据验证与质量检查

标注完成之后、训练开始之前，一定要做数据验证。这一步很多人会跳过，但这是避免训练时踩坑最关键的一步。

一个完整的数据验证流水线包括以下检查项：

import os

def validate_dataset(label_dir, img_dir, class_names):
    errors = []
    label_files = os.listdir(label_dir)

    for lf in label_files:
        base = os.path.splitext(lf)[0]
        # 检查1: 每个标签文件是否有对应的图像文件
        img_exists = any(f.startswith(base) for f in os.listdir(img_dir))
        if not img_exists:
            errors.append(f"标签 {lf} 没有对应的图像文件")
            continue

        with open(os.path.join(label_dir, lf)) as f:
            for line_num, line in enumerate(f, 1):
                parts = line.strip().split()
                if len(parts) != 5:
                    errors.append(f"{lf}:{line_num} 格式错误，需要5个字段")
                    continue

                cls_id = int(parts[0])
                # 检查2: 类别ID是否越界
                if cls_id < 0 or cls_id >= len(class_names):
                    errors.append(f"{lf}:{line_num} 类别ID {cls_id} 越界")

                # 检查3: 坐标是否在 [0, 1] 范围内
                cx, cy, w, h = map(float, parts[1:])
                if not (0 <= cx <= 1 and 0 <= cy <= 1):
                    errors.append(f"{lf}:{line_num} 中心坐标超出[0,1]")
                if w <= 0 or h <= 0:
                    errors.append(f"{lf}:{line_num} 框宽高必须为正数")
                if w > 1 or h > 1:
                    errors.append(f"{lf}:{line_num} 框宽高超过图像尺寸")

    return errors

除了格式检查，还应该做以下统计分析：

检查项 意义 发现问题后怎么办
类别分布直方图 检查类别是否严重不平衡 补充少样本类别的数据，或使用类别加权损失
框的面积分布 了解目标尺度范围 根据尺度范围设置锚框大小
框的长宽比分布 了解目标形状特征 同样用于锚框设计参考
每张图的框数分布 了解图像复杂度 剔除异常值（如一张图有100+个框）
标注员一致性 多人标注时的偏差 计算 IOU，对分歧较大的样本进行复审

多人标注的质量控制

这是本章最想强调的一点——多人标注时，一致性是最大的挑战。

我曾经参与过一个智能安防项目，5个标注员标注了10万张图片。训练出来的模型在测试集上表现尚可，但一上线就疯狂误报。排查后发现：同样的"行人"目标，有的标注员用紧密包围框，有的习惯留一圈余量，有的把远处的模糊行人全部略过，有的则把所有疑似行人全部框出来。结果是模型学习到了标注员的"个人风格"，而不是行人的通用特征。

解决办法是：

  • 制定统一的标注规范文档：明确框的边界标准、遮挡目标的标注规则、模糊目标的处理方式
  • 标注前培训 + 考核：先让小团队标注100张同一批图像，计算标注员之间的 IOU 一致性，达标后才能正式标注
  • 定期抽样复审：每周随机抽5%的标注数据，由资深标注员复审，发现问题及时回退
  • 使用 CVAT 的质量分析功能：自动计算标注员与基准标注的偏差
一个实用的标规范：框的边界应该紧贴目标的外轮廓，但不能切到目标——想象用一个最小矩形去框住目标，四个边刚好碰到目标的最外侧像素。如果目标被遮挡，按照可见部分来框。如果目标只有不到20%可见，标为"难例"（difficult）或直接跳过。

自动化标注与主动学习

2025-2026年，数据标注领域最大的趋势是主动学习 + 模型辅助标注。不再需要人工标注所有数据，而是：

  1. 先用少量标注数据训练一个初版模型
  2. 用模型对未标注数据做推理
  3. 选出模型"最不确定"的样本（置信度低、预测分歧大）
  4. 只标注这些高价值样本
  5. 重复迭代

这个方法可以在保持模型性能的同时，减少60-80%的标注量。Ultralytics YOLO 从 v8.3 开始内置了主动学习支持，通过 yolo active-learning 子命令即可启动。
