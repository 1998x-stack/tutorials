第17章：因果推断基础

大家好，我是你们的讲师。欢迎来到可能是整个统计学课程中最重要的一章。你可能听过无数次"相关不等于因果"——但你真的理解这句话背后的深层含义吗？你知道为什么两个变量高度相关但不一定是因果关系吗？更重要的是——如果我们想要做出因果性的结论，到底需要什么条件？

因果推断在过去十年间经历了革命性的发展。2021年诺贝尔经济学奖授予了David Card、Joshua Angrist和Guido Imbens，表彰他们在因果关系分析方法论上的开创性贡献。与此同时，计算机科学家Judea Pearl的因果图模型（DAG）正在深刻改变整个数据科学领域。这一章，我们用最直观的方式，把这些思想的核心讲明白。

17.1 为什么相关不等于因果

网上流传着各种"看似因果实则相关"的搞笑例子：

  • 每年溺水死亡人数和尼古拉斯·凯奇出演电影数量正相关——但你不会说凯奇的电影导致了溺水
  • 冰淇淋销量和溺水率正相关——真正的原因是"夏天"这个混杂变量同时导致了两个现象
  • 有消防车越多的地方火灾损失越严重——但你不会说消防车造成了火灾

在统计学上，两个变量X和Y出现相关，有四种可能的解释：

解释 说明 举例
X → Y（因果） X导致Y 吸烟导致肺癌
Y → X（反向因果） Y导致X 抑郁症和低收入的循环
Z → X 且 Z → Y（混杂） 第三个变量同时导致两者 夏天同时导致冰淇淋销量和溺水
纯巧合 完全随机 凯奇电影和溺水

关键洞察：因果推断的核心任务不是找到"两个变量有相关"，而是排除其他所有可能的解释。随机对照试验（RCT）之所以被称为金标准，就是因为它通过随机化来消除混杂变量的影响。但在观察性研究中（你不能"随机"让人吸烟或不吸烟），我们必须用统计分析来逼近因果结论。

17.2 混杂变量：因果推断的最大敌人

混杂变量（Confounder）是既影响处理变量（你感兴趣的X）又影响结果变量（你想解释的Y）的第三个变量。如果不对混杂变量进行控制，你观察到的X和Y之间的相关关系就可能完全是假的。

让我给你讲一个经典的"踩坑"故事，这个故事至今仍然是因果推断领域最常被引用的警示——激素替代疗法（HRT）。

20世纪80~90年代，大量观察性研究发现：接受激素替代疗法的女性，其冠心病发病率显著低于未接受者。这些研究的样本量巨大、统计方法先进、p值极其显著。全世界的医生据此给更年期女性开了大量HRT处方——因为"科学证据"表明HRT可以预防心脏病。

然后来了Women's Health Initiative（WHI）——一个大型随机对照试验。结果震惊了整个医学界：接受HRT的女性不仅没有降低心脏病风险，反而增加了冠心病、中风和乳腺癌的风险！

为什么会这样？因为在观察性研究中，选择接受HRT的女性通常社会经济地位更高、更注重健康、更频繁体检——这些都是"健康行为"的一部分。这些混杂变量导致了HRT看似有益。当RCT通过随机化消除了这些混杂后，真相才水落石出。

用一个简单的DAG来可视化混杂：

# 用Python + networkx 展示混杂的结构
# 但这部分我们先用文本表示：
#
#   [社会经济地位] ← 混杂变量
#     ↗              ↘
#   [HRT使用] ←→?→ [心脏病风险]
#
# 在观察性研究中，社会经济地位同时影响
# HRT使用和心脏病风险，造成了虚假的相关。
# RCT通过随机分配HRT，切断了社会经济地位
# 到HRT使用的箭头，从而消除了混杂。

⚠️ 激素替代疗法（HRT）的教训：这是我最常用来警示学生的案例。在WHI结果公布之前，HRT的观察性证据看起来如此"确凿"——数十万患者、多年的追踪、百篇论文的正面结论。然而全错了。这个案例告诉我们两件事：(1) 观察性研究的混杂效应可能非常强大，再大的样本量也无法替代随机化；(2) 当观察性研究和RCT结论冲突时，务必相信后者。我建议每个做数据分析的人都把HRT的故事作为"谦卑心训练"的第一课。

17.3 DAG（有向无环图）：可视化因果假设

传统的统计学教育几乎不讲因果关系——只讲相关、回归、假设检验。但Judea Pearl（2021年的图灵奖得主）改变了这一切。他提出的有向无环图（Directed Acyclic Graph, DAG）为因果推断提供了清晰的数学语言。

DAG的三个基本元素：

  • 节点（Node）：变量——可以是处理变量、结果变量、混杂变量等
  • 箭头（Edge）：因果关系方向——从原因指向结果
  • 路径（Path）：通过箭头连接起来的节点序列

DAG中最重要的概念是d-分离（d-separation）和后门准则（Backdoor Criterion）：

  • 后门路径：从X到Y，有一条指向X的箭头的路径——这通常是混杂导致的虚假关联
  • 后门准则：如果你控制了所有后门路径上的变量（即"阻断了所有后门路径"），那么X和Y之间的相关就等于因果

举个具体的例子——教育年限（X）对收入（Y）的影响：

# DAG示意图：
#
# [能力] → [教育年限] → [收入]
#    ↘                    ↗
#        [工作经验]
#
# 后门路径： 能力 → 教育年限 和 能力 → 收入
# 这个路径上的混杂变量是"能力"
# 要估计教育对收入的因果效应，你需要控制"能力"
# 或者控制"能力"的代用变量（如IQ测试分数）

# Python中模拟"控制混杂" vs "不控制"的差异
import numpy as np
import pandas as pd
import statsmodels.api as sm

np.random.seed(42)
n = 1000

# 生成数据
ability = np.random.normal(100, 15, n)         # 能力（混杂变量）
education = 10 + 0.05 * ability + np.random.normal(0, 1.5, n)  # 教育年限
income = 20000 + 3000 * education + 500 * ability + np.random.normal(0, 10000, n)

# 不控制能力：看看教育对收入的"效应"
X1 = sm.add_constant(education)
model1 = sm.OLS(income, X1).fit()
print("不控制混杂变量时的教育效应：")
print(f"系数 = {model1.params[1]:.1f}")

# 控制能力后：真正的教育效应
X2 = sm.add_constant(pd.DataFrame({'education': education, 'ability': ability}))
model2 = sm.OLS(income, X2).fit()
print("\n控制混杂变量后的教育效应：")
print(f"教育系数 = {model2.params[1]:.1f}")
print(f"（真实效应为3000，因为true model中income受education的影响系数是3000）")

实际使用DAG的流程是：

  1. 根据领域知识画出你认为的因果结构（DAG）
  2. 识别出从X到Y的后门路径
  3. 确定需要控制哪些变量来阻断所有后门路径
  4. 在统计分析中控制这些变量
  5. 解释结果——现在X对Y的估计效应就是"因果效应"（假设你的DAG正确）

使用DAG的注意事项：(1) DAG必须是有向的（不能有双向箭头）和无环的（不能有反馈循环）——这意味着DAG不太适合建模有反馈系统的场景；(2) DAG的正确性完全取决于你的领域知识——如果你画错了图，因果结论就是错的；(3) 实践中我推荐用DAG作为"沟通工具"——当你和同事争论某个变量是否需要控制时，画一个DAG往往比堆砌统计术语有效得多。有两个不错的DAG绘制工具：dagitty.net 和 Python的 `dagitty` 包。

17.4 随机对照试验（RCT）：因果推断的金标准

如果有一种方法能让你"按下按钮"消除所有混杂变量——那就是RCT。RCT（Randomized Controlled Trial）的基本逻辑极其优雅：

  • 将参与者随机分配到处理组和对照组
  • 因为随机化，理论上处理组和对照组的所有特征都是可比的（不仅是你观察到的特征，还包括你没观察到的）
  • 两组之间观察到的结果差异，只能归因于处理本身

为什么随机化如此强大？因为它在DAG中切断了所有指向处理变量（X）的箭头。一旦X是随机分配的，就不再存在"X ← Z → Y"这样的后门路径——因为没有任何变量能影响X。

RCT虽然强大，但也有局限：

局限 说明 可能解决方案
伦理问题 你不能随机让人吸烟或暴露于有害物质 自然实验、工具变量
外部有效性 RCT的参与者可能不代表目标人群 实用性试验（pragmatic trial）
成本 大规模RCT非常昂贵 准实验设计
非依从性 被分配到处理组的人可能不服药 ITT分析、工具变量
时效性 结果需要长时间才能显现 替代终点

在互联网行业，A/B测试就是RCT的一种特殊形式。你随机分配用户到旧版（对照组）和新版（实验组），然后比较结果指标。这就是为什么A/B测试被认为是产品迭代的"黄金标准"——不是因为它完美，而是因为它比任何观察性分析都更接近因果推断。

17.5 Pearl的因果阶梯：关联、干预、反事实

Judea Pearl提出了一个深刻的框架来理解因果推断的三个层次——因果阶梯（Ladder of Causation）：

  1. 第1层：关联（Association）——"如果我看到X，我会看到Y吗？"
  2. 第2层：干预（Intervention）——"如果我改变X，Y会变化吗？"
  3. 第3层：反事实（Counterfactuals）——"如果当时我没有做X，Y会怎样？"

第1层：关联

这是所有传统统计学能到达的层次。你可以计算相关系数、做回归分析、做假设检验——所有这些都只是描述"看到的东西"之间的关系。但关联不等于因果。这是Pearl说的"统计学只能告诉你过去发生了什么，不能告诉你改变过去会造成什么不同"。

第2层：干预

这一层问的是："如果我们主动改变X，Y会如何？"区分关联和干预的关键在于你是在"观察"还是"行动"。在数据中，P(Y|X=x)（给定我看到X=x时Y的分布）和P(Y|do(X=x))（给定我强制设定X=x时Y的分布）可能是完全不同的。Pearl的do-算子（do-operator）就是用来形式化这种差别的。

比如：在数据中，P(溺水|冰淇淋销量高) 很高（关联关系），但P(溺水|do(冰淇淋销量高)) 不会有变化（因果效应）——因为卖更多冰淇淋不会导致溺水。

第3层：反事实

这是最高的因果推理层次：问的是"如果当时我做了不同的选择，结果会怎样？"反事实思维本质上是想象——在头脑中构建一个"如果……就……"的世界。你无法在观察性数据中直接看到反事实，因为反事实对应的是你没有观察到的那条路。

比如：你吃了头痛药，头痛消失了。你知道吃药导致了头痛消失。但反事实问题是："如果我没吃药，头还痛吗？"——你永远无法同时观察到这两条路径，你只能通过因果模型来推断。

因果阶梯的核心启示：大多数数据分析只处于第1层（关联）。如果要说"因果关系"，你必须有意识地提升到第2层（干预）或第3层（反事实）。这意味着要么你做了RCT/A/B测试（第2层），要么你通过DAG+领域知识建立了因果模型（第2-3层）。没有这些，任何"因果"的宣称本质上都是推测——即使它听起来再"统计显著"。

让我用一个具体的例子来展示这三个层次的差异：

# 模拟：教育（X）对收入（Y）的因果阶梯
import numpy as np

np.random.seed(42)
n = 10000

# 第1层：关联——我们看到的数据
education = np.random.normal(12, 3, n)  # 当前的教育年限
income_observed = 20000 + 3000 * education + np.random.normal(0, 15000, n)

# 相关性
correlation = np.corrcoef(education, income_observed)[0, 1]
print(f"第1层（关联）：相关系数 = {correlation:.3f}")

# 第2层：干预——我们主动设置教育年限
# 在反事实框架中，假设我们把所有人的教育都设为16年
# 这需要知道真正的因果结构
def simulate_intervention(edu_years):
    true_effect = 3000  # 每多一年教育，收入增加3000
    base_income = 20000
    return base_income + true_effect * edu_years + np.random.normal(0, 15000, n)

income_if_edu_12 = simulate_intervention(12)
income_if_edu_16 = simulate_intervention(16)

print(f"\n第2层（干预）：")
print(f"如果所有人受教育12年，平均收入 = {income_if_edu_12.mean():.0f}")
print(f"如果所有人受教育16年，平均收入 = {income_if_edu_16.mean():.0f}")
print(f"因果效应（多4年教育）= {income_if_edu_16.mean() - income_if_edu_12.mean():.0f}")

# 第3层：反事实——对同一个人，问"如果……会怎样？"
# 这是最复杂的层次，通常需要额外的假设
print(f"\n第3层（反事实）：")
print(f"对于某个受过12年教育的人（收入={income_if_edu_12[0]:.0f}）：")
print(f"  - 事实：受教育12年，收入{income_if_edu_12[0]:.0f}")
print(f"  - 反事实：如果TA受教育16年，收入会是多少？")
print(f"    推测：{simulate_intervention(16)[0]:.0f}")
print(f"    （注意：反事实推断需要可忽略性假设等条件）")

好，这一章我们深入讨论了因果推断的核心思想——从"相关不等于因果"开始，了解了混杂变量的危害（通过HRT这个血淋淋的案例），学习了DAG这个可视化工具，认识了RCT作为因果推断的金标准，最后站在Pearl的因果阶梯上俯瞰了关联、干预和反事实三个层次。下一章，我们将转向一个更"实际"但同样重要的话题——统计伦理与最佳实践：作为数据分析师和数据科学家，我们有哪些底线不能碰？什么样的分析是负责任的分析？
