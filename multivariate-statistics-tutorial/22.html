开篇：从距离重建地图

大家好，欢迎继续学习。设想一个场景：有人给了你一张表格，上面列着北京到上海的距离、上海到广州的距离、广州到深圳的距离、以及所有其他城市两两之间的距离，但唯独没有给你地图。你问："这些城市在地图上的相对位置是怎样的？"答案是——理论上，仅凭这些两两之间的距离数据，你就可以重建出城市间的相对位置地图。当然，你得到的是一个"相对坐标系"（不知道经纬度），但城市之间的拓扑关系和相对方位是完全确定的。

这就是多维标度法 (Multidimensional Scaling, MDS) 的核心能力：给定一组对象（城市、产品、品牌、基因等）之间的两两距离或相似度，MDS能在低维空间（通常是2维或3维）中重建这些对象的相对位置，使得重建后的距离尽可能接近原始距离。它可以说是"从距离矩阵到地图"的数学魔法。

MDS在市场研究（品牌感知图）、生态学（物种分布）、心理学（感知相似性）、地理信息系统等领域有着广泛的应用。这一章我们会从经典度量MDS出发，延伸到非度量MDS，并通过完整案例展示MDS的使用方法。

MDS的核心问题

MDS要解决的基本问题可以形式化地表述为：

已知：n 个对象之间的两两距离矩阵 D（n × n），其中 dᵢⱼ ≥ 0, dᵢᵢ = 0
目标：找到 n 个低维空间中的点 {x₁, x₂, ..., xₙ} ⊂ ℝᵏ（通常 k=2 或 k=3），
       使得点间欧氏距离 ||xᵢ - xⱼ|| 尽可能接近原始距离 dᵢⱼ

优化目标（应力函数 Stress）：
Stress = √( Σᵢⱼ (||xᵢ - xⱼ|| - dᵢⱼ)² / Σᵢⱼ dᵢⱼ² )

MDS的输入距离矩阵可以来自任何度量——欧氏距离、曼哈顿距离、相关系数转化成的距离（如 1 - r）、甚至主观感知的相似度评分。不同距离度量会得到不同的"地图"，因此选择适当的距离度量是MDS应用中的关键一步。

MDS类型输入数据目标典型场景
度量MDS (Classical MDS)定量距离精确重建距离关系地理距离、基因距离
非度量MDS (Non-metric MDS)序数相似度/距离保留排名/序次关系消费者感知、品牌评分
加权MDS带权重的距离给某些距离更高权重缺失数据、差异化精度

度量MDS (Classical MDS)

经典MDS（也称为Torgerson-Gower标度法）是MDS中最古老也最优雅的一种形式。它有一个漂亮的数学闭式解，不需要迭代优化。

经典MDS的核心步骤：

设 D 为 n×n 距离矩阵，元素 dᵢⱼ 为对象 i 与 j 之间的距离。

步骤1：计算平方距离矩阵
Δ = [dᵢⱼ²]

步骤2：双中心化 (double-centering)
B = -½ J Δ J
其中 J = I - (1/n)11ᵀ 是中心化矩阵
（J 的作用是将矩阵的行和列均值都变为零）

步骤3：对 B 进行特征值分解
B = V Λ Vᵀ，其中 Λ = diag(λ₁, λ₂, ..., λₙ)，λ₁ ≥ λ₂ ≥ ... ≥ λₙ

步骤4：取前 k 个最大特征值及其对应的特征向量
X = Vₖ Λₖ¹/²

X 是 n×k 的坐标矩阵，第 i 行就是对象 i 在 k 维空间中的坐标。

双中心化的目的是将距离矩阵转换为内积矩阵。一旦我们得到了对象间的内积矩阵B，对它做特征值分解就相当于PCA——事实上，当距离是欧氏距离时，经典MDS等价于PCA。我们在后面会详细展开这个关系。

一点历史：经典MDS的数学基础可以追溯到年轻的Schönberg在1935年的一篇论文，他研究了"什么时候距离矩阵可以被嵌入欧氏空间？"结论是：当且仅当双中心化后的矩阵B是半正定的（特征值都非负），距离矩阵就能被精确地嵌入欧氏空间。这个条件在心理学和感知研究中往往不满足——因为人的判断不一致——这也正是为什么我们需要非度量MDS。

非度量MDS

在很多实际场景中，我们并不拥有精确的距离数值。一个典型的例子是消费者感知研究：让消费者对10个品牌的两两相似度打分（1=非常相似，7=非常不同）。得到的分数是序数级的——"品牌A和B的相似度为2"并不意味着它们之间的差异是"品牌C和D的相似度为4"的一半，只能说A和B比C和D更相似。

非度量MDS (Non-metric MDS, NMDS) 就是为了处理这种情况而设计的。它只要求输入数据保持序次关系 (rank order)，而不要求数值精度。

NMDS的目标：
找到低维坐标，使得坐标间的距离 d̂ᵢⱼ 与原始不相似度 δᵢⱼ 的排序一致。

即：如果 δᵢⱼ < δₖₗ，则 d̂ᵢⱼ 应该 < d̂ₖₗ（尽可能）

应力函数 (Stress)：
Stress = √( Σᵢⱼ (d̂ᵢⱼ - f(δᵢⱼ))² / Σᵢⱼ d̂ᵢⱼ² )

其中 f 是单调递增的变换函数（通过等序回归 monotonic regression 估计），
将原始不相似度 δᵢⱼ 映射到"目标距离"。

NMDS使用迭代优化算法（如梯度下降或SMACOF算法）来最小化应力函数。它不保证全局最优解，因此实践中需要多次随机初始化并选择应力最小的结果。

谢泼德图 (Shepard Diagram)

Shepard图是评估NMDS拟合质量的标配工具。横轴是原始不相似度 δᵢⱼ，纵轴是MDS重构距离 d̂ᵢⱼ。如果点在图中沿对角线方向紧密聚集，说明拟合良好——重构距离与原始不相似度的序次关系一致。阶梯状的单调回归曲线清晰地展示了NMDS如何将序数输入映射到定量距离。

MDS结果的评估

如何判断MDS的结果是否可靠？主要有三个指标：

应力系数 (Stress)

Kruskal提出了Stress-1系数来衡量拟合质量。不同范围的Stress对应不同的拟合水平：

Stress值拟合质量
< 0.05优秀 (excellent)
0.05 - 0.10良好 (good)
0.10 - 0.15可接受 (fair)
0.15 - 0.20勉强 (poor)
> 0.20不可接受

R² (拟合优度)

重构距离与原始距离（或单调变换后的距离）之间的相关系数的平方。R² > 0.60 通常认为可以接受。

可解释性

和所有降维方法一样，最关键的评估标准是：你能给坐标轴或图上聚类一个有意义的解释吗？如果二维MDS图上横轴可以解释为"价格-品质"维度、纵轴可以解释为"传统-创新"维度，那么即使应力稍高，这个MDS结果也是有价值的。

MDS与PCA的关系

这个关系值得特别强调，因为它经常引起混淆：

经典MDS（使用欧氏距离）≡ PCA

具体来说：如果你先计算数据矩阵的欧氏距离矩阵，然后运行经典MDS，得到的结果与直接对数据做PCA得到的得分（主成分坐标）是完全等价的——最多差一个旋转或镜像。这是因为PCA本质上也是在欧氏空间中最大化方差，而经典MDS则在保留欧氏距离结构。

那为什么还要用MDS？原因有三：

第一，MDS不要求原始数据是数值矩阵。只要你能定义"两个对象之间的距离"（比如问卷中的感知相似度打分），MDS就能工作。PCA要求你有 n×p 的原始数据矩阵。

第二，MDS能处理任意距离度量。你可以用曼哈顿距离、相关系数距离、甚至自定义的领域特定距离。这比PCA灵活得多。

第三，非度量MDS能处理序数输入。这是PCA完全做不到的。

关键区别速记：PCA从"数据矩阵→协方差→特征向量"中寻找最大方差方向，MDS从"距离矩阵→双中心化→特征向量"中寻找最佳投影。当距离是欧氏距离时，两条路通向同一个终点。当距离不是欧氏距离时，MDS提供了一条PCA无法替代的路径。

实战案例：城市间距离的MDS重建

让我们用MDS做一个经典实验：用中国主要城市间的公路距离数据，重建这些城市的"地图"。

# R代码示例：中国城市距离MDS
cities <- c("北京", "上海", "广州", "成都", "武汉", "西安", "深圳", "杭州")

# 简化的距离矩阵（单位：公里，示意数据）
dist_matrix <- matrix(c(
  0, 1213, 1889, 1542, 1155, 934, 1947, 1279,
  1213, 0, 1197, 1660, 688, 1361, 1217, 167,
  1889, 1197, 0, 1248, 812, 1391, 91, 1070,
  1542, 1660, 1248, 0, 1002, 623, 1246, 1525,
  1155, 688, 812, 1002, 0, 701, 840, 665,
  934, 1361, 1391, 623, 701, 0, 1524, 1276,
  1947, 1217, 91, 1246, 840, 1524, 0, 1070,
  1279, 167, 1070, 1525, 665, 1276, 1070, 0
), nrow = 8, byrow = TRUE)

# 经典MDS
mds_result <- cmdscale(dist_matrix, k = 2)
plot(mds_result, type = "n", xlab = "MDS1", ylab = "MDS2")
text(mds_result, labels = cities, cex = 0.8)

运行这段代码，你会发现MDS重建的城市相对位置与真实地图惊人地一致：北京在北方，广州和深圳在南方沿海，成都和西安在西部。虽然MDS不知道经纬度、不知道"北方在哪"，仅凭距离信息就能恢复相对位置——这就是MDS的强大之处。

在品牌感知映射的语境下，MDS的使用方式类似：收集消费者对各品牌间相似度的评分，运行MDS得到品牌感知图，然后解读坐标轴的含义。你可能发现某些品牌在感知图上聚在一起（构成竞争集合），而坐标轴可能对应着"价格定位"和"品质感知"等营销学概念。

踩坑故事：不恰当的距离度量带来的误导

我曾见过一个团队用MDS做"竞争对手分析"，他们把市场份额差异视为"距离"，用这个"市场份额距离矩阵"跑了MDS，得到了一张竞争格局图。图上显示公司A和公司B很接近，团队据此解读："A和B是直接竞争对手。"但这个结论有问题：市场份额差异只是竞争的一个维度，A和B可能分别服务完全不同的人群（A做高端、B做低端），只是碰巧市场份额接近。

用MDS时，必须回答一个前置问题："距离"在这里到底意味着什么？如果是感知相似度（消费者觉得这两个品牌像不像），那没问题。如果是市场份额之差，那MDS重建的就不是"这些公司之间的关系"，而是"这些公司市场份额的排序"，两者完全不同。

教训：MDS忠实地反映你输入的距离结构。如果输入距离有意义，输出就有意义。如果输入距离本身就是有问题的度量，那么再漂亮的MDS图也只是"垃圾进，垃圾出"的华丽版本。在应用MDS之前，务必审视你的距离矩阵是如何构建的：数据来源可靠吗？度量是否对研究问题有意义？是否存在某个维度主导了距离计算的权重（类似于PCA中的量纲问题）？一个良好的实践是：在运行MDS之前，先对距离矩阵做Spearman相关聚类或矩阵可视化，初步了解距离的结构，再决定是否用MDS。记住，MDS是一个"探索"工具，不是"验证"工具。它的价值在于激发洞察，而不是证明假设。

本章小结与第Ⅳ部分总结

本章我们学习了多维标度法的核心原理和实战应用。我们从MDS的基本问题出发——如何从距离矩阵重建几何构型——然后分别介绍了经典度量MDS的闭式解（双中心化+特征值分解）和非度量MDS的迭代优化方法（序数输入+Shepard图）。我们讨论了MDS结果的评估标准（Stress和R²），厘清了MDS与PCA的关系——经典MDS+欧氏距离≡PCA。通过城市距离重建的案例，我们直观地感受到了MDS"从距离到地图"的能力。最后，我们通过一个关于距离度量的踩坑故事，提醒大家MDS输出的质量完全取决于输入距离矩阵的质量。

至此，第Ⅳ部分——"多元关系的深入探索"——就告一段落了。在这部分我们学习了典型相关分析（CCA）、对应分析（CA）和多维标度法（MDS）。这三类方法的共同点是：它们都旨在揭示数据中隐藏的结构关系——CCA揭示两组变量的关联，CA揭示分类变量的共现模式，MDS揭示对象的相对距离关系。它们不是互相竞争的，而是互补的——具体用哪个取决于你的数据类型和研究问题。
