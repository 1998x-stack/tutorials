第25章 动手：部署一个代币 — 从零到链上

学到这里，你已经了解了比特币的底层技术、交易的方方面面、甚至学会了如何通过链上数据做出更好的判断。但还有一个"成人礼"没有完成——亲手发一个币。

你是不是觉得"发币"是只有专业团队才能做的事？错。在以太坊上，任何人——包括现在的你——都可以在 20 分钟内部署一个属于自己的 ERC-20 代币。不需要服务器、不需要团队、不需要许可证。只需要一个浏览器插件、几行代码，和一点点测试网 ETH。今天，我们就来走完这整个过程。

免责声明：本章的教学目的是让你理解 ERC-20 代币的技术实现，绝不是鼓励你去"发个空气币割韭菜"。你有能力发币，但请你做一个正直的人。一个没有实际价值的代币 + 一堆吹牛宣传 + 拉高砸盘 = 犯罪。历史上因此坐牢的人不少。

25.1 准备工作

动手之前，我们需要安装和配置几样东西。整个过程大约需要 10 分钟。

第一步：安装 MetaMask

MetaMask 是最流行的以太坊浏览器钱包，以 Chrome 插件形式运行。前往 metamask.io 下载安装。创建钱包时，请务必记下你的助记词（seed phrase）并离线保存——如果你弄丢了，没有任何人可以帮助你恢复钱包。

第二步：切换到 Sepolia 测试网

在 MetaMask 中，默认的网络是"Ethereum Mainnet"。点击左上角的网络切换按钮，选择"Sepolia Testnet"。如果列表中没有 Sepolia，点击"Add Network"手动添加：

  • 网络名称：Sepolia Testnet
  • RPC URL：https://rpc.sepolia.org
  • Chain ID：11155111
  • 货币符号：SepoliaETH
  • 区块浏览器：https://sepolia.etherscan.io

第三步：获取测试网 ETH

部署合约需要支付 Gas 费——但在测试网上，Gas 用的是免费的水龙头（faucet）ETH。搜索"Sepolia faucet"可以找到多个水龙头服务。推荐 faucet.sepolia.dev 或 infura.io/faucet/sepolia。输入你的钱包地址，点击"Send Me ETH"，几秒钟后你的钱包里就会收到 0.5-1 个 SepoliaETH——足够部署几十次合约了。

注意 fake faucet 骗局：搜索"Sepolia faucet"时，前面几页可能混着钓鱼网站。它们会要求你"输入私钥"或"提供助记词"来验证身份。真 faucet 只需要你的钱包地址。任何时候、任何网站问你要私钥或助记词——它 100% 是骗子。记住：私钥=你钱包的所有权，给出去就没。

25.2 Remix IDE 入门

Remix IDE 是一个基于浏览器的 Solidity 智能合约开发环境。不需要安装、不需要配置，打开浏览器就能写合约、编译和部署。它是由以太坊基金会维护的官方工具。

访问 remix.ethereum.org，你会看到三个主要区域：

区域位置功能
文件管理器左侧面板创建、重命名、删除 Solidity 文件
代码编辑器中央面板编写智能合约代码，带语法高亮
编译/部署面板右侧面板编译合约、选择网络、部署交互

创建一个新文件：在左侧文件管理器中，点击"contracts"文件夹，右键选择"New File"，命名为 MyToken.sol。这就是我们要写入代币合约的地方。

Remix 的四个按钮功能：

  • Solidity Compiler：编译你的 Solidity 代码，检查语法错误
  • Deploy & Run Transactions：部署合约并与之交互
  • File Explorer：管理项目文件
  • Plugin Manager：添加额外的功能插件

25.3 编写 ERC-20 合约

现在到了最激动人心的部分——写一个真正的 ERC-20 代币合约。ERC-20 是以太坊上的代币标准，它定义了一组接口，任何符合该标准的代币都可以被钱包、交易所和 DeFi 协议识别和使用。

我们要用 OpenZeppelin 库——这是最广泛使用的、经过安全审计的智能合约库。OpenZeppelin 已经实现了 ERC-20 标准的全部接口，我们只需要继承它的合约，然后配置我们的代币参数即可。

在 Remix 中把以下代码完整复制到 MyToken.sol：

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
import "@openzeppelin/contracts/access/Ownable.sol";

/**
 * @title MyToken
 * @author 你自己
 * @notice 一个可铸造的 ERC-20 代币，仅供教学用途
 * @dev 部署者拥有代币的管理权限
 */
contract MyToken is ERC20, Ownable {

    // ── 代币基本信息 ──
    // 这些信息会在部署时设置
    string private _tokenName;
    string private _tokenSymbol;

    // ── 小数的位数 ──
    // 18 位小数是 ERC-20 最常用的（与 ETH 相同）
    uint8 private _decimals;

    /**
     * @notice 构造函数：部署时只会执行一次
     * @param name_ 代币名称（如 "Bitcoin Class"）
     * @param symbol_ 代币符号（如 "BTCX"）
     * @param initialSupply_ 初始总供应量（以"完整代币"为单位）
     * @param decimals_ 小数位数（通常为 18）
     */
    constructor(
        string memory name_,
        string memory symbol_,
        uint256 initialSupply_,
        uint8 decimals_
    )
        ERC20(name_, symbol_)
        Ownable(msg.sender)
    {
        _tokenName = name_;
        _tokenSymbol = symbol_;
        _decimals = decimals_;

        // 铸造初始供应量给合约部署者
        // initialSupply_ * 10**decimals_ 转换为"最小单位"（wei 类似的概念）
        _mint(msg.sender, initialSupply_ * 10 ** decimals_);
    }

    /**
     * @notice 铸造新代币（只有合约所有者可以调用）
     * @param to 接收代币的地址
     * @param amount 铸造数量（以最小为单位）
     */
    function mint(address to, uint256 amount) external onlyOwner {
        _mint(to, amount);
    }

    /**
     * @notice 销毁代币（只有合约所有者可以调用）
     * @param amount 销毁数量（以最小为单位）
     */
    function burn(uint256 amount) external onlyOwner {
        _burn(msg.sender, amount);
    }

    /**
     * @notice 返回小数位数
     * @return 小数的位数（如 18 表示可分割到 0.000000000000000001）
     */
    function decimals() public view virtual override returns (uint8) {
        return _decimals;
    }
}
代码逐行解读：
  • pragma solidity ^0.8.20 — 指定编译器版本，^ 表示 0.8.20 及以上（但小于 0.9.0）
  • import ... — 引入 OpenZeppelin 的 ERC-20 和 Ownable 实现
  • is ERC20, Ownable — 继承两个合约，获得它们的所有功能
  • constructor(...) — 部署时运行一次，设置代币参数并铸造初始供应量
  • _mint(msg.sender, ...) — 铸造代币给部署者（就是你自己）
  • onlyOwner — 修饰符，确保只有合约所有者能调用 mint 和 burn
  • override — 覆盖父合约的 decimals() 函数，因为 OpenZeppelin 的默认实现固定为 18

编译合约：

  1. 在右侧面板点击"Solidity Compiler"（第二个图标）
  2. 确保 Compiler 版本设置为 0.8.20+
  3. 点击"Compile MyToken.sol"
  4. 如果看到绿色的勾，说明编译成功。如果有红色错误，检查代码是否有拼写错误
遇到 "import 找不到" 错误？Remix 会自动从 npm 拉取 OpenZeppelin 合约，但前提是你需要确认 Remix 的"Solidity Compiler"面板中"Auto compile"已开启，并且 Remix 已经识别到了 @openzeppelin/contracts 的依赖。如果还是报错，可以尝试在 Remix 的"File Explorer"中右键点击"contracts"文件夹，选择"Flatten"后再试——或者直接在 Remix 的"File Explorer"中点开"Solidity" → "OpenZeppelin"来确认库已经加载。

25.4 部署到测试网

编译成功后，就是部署环节了。这是让你的代币"上链"的时刻。

Step 1：配置部署参数

在右侧面板切换到"Deploy & Run Transactions"（第三个图标）。确保以下设置：

  • Environment：选择"Injected Provider — MetaMask"（这会触发 MetaMask 弹出）
  • Contract：选择"MyToken"（如果有多个合约，确保选对了）
  • 展开"Deploy"区域，你会看到构造函数需要的四个参数：name_、symbol_、initialSupply_、decimals_

Step 2：输入代币参数

以我们的示例代币为例：

  • name_："My First Token"（你的代币名称）
  • symbol_："MFT"（交易市场上显示的符号）
  • initialSupply_：1000000（总供应 100 万个）
  • decimals_：18（标准的小数位数）

Step 3：确认 MetaMask 交易

点击"Transact"按钮，MetaMask 会弹出一个窗口：

  • 查看交易的 Gas 估算
  • 确认 Chain ID 显示的是 Sepolia（11155111）而不是 Mainnet
  • 点击"Confirm"
部署到主网的风险：有一次，一个新手开发者在 YouTube 教程中跟着 Remix 操作，忘了把网络从"Ethereum Mainnet"切换到测试网。结果部署一笔花了 0.5 ETH（约 1,700 美元）的 Gas 费——部署上主网了。这个人不是别人，就是我朋友。这个教训告诉我们：每次部署前，三！次！检！查！网络设置！ MetaMask 上显示的网络名称、Chain ID、蓝色主网标识——全部确认无误再点确认。

Step 4：查看交易状态

MetaMask 确认后，Remix 下方的日志区域会显示交易哈希（tx hash）。点击这个哈希，它会跳转到 Sepolia Etherscan。等待 10-30 秒（测试网出块速度慢一些），当交易状态变为"Success"时，你的合约就正式部署成功了！

记下 Etherscan 上显示的合约地址（Contract Address，以 0x 开头）。这个地址就是你的代币在链上的家。

25.5 与你的代币交互

代币部署成功了，但默认情况下 MetaMask 不会自动识别它。你需要手动添加。

将代币添加到 MetaMask：

  1. 打开 MetaMask
  2. 点击"Assets"标签页下方的"Import tokens"
  3. 在"Token Contract Address"中输入你的合约地址
  4. MetaMask 应该会自动识别符号（MFT）和小数位数（18）
  5. 点击"Add Custom Token" → "Import"

现在你的 MetaMask 中应该能看到余额了——1,000,000 MFT。恭喜你，你刚刚成为了一个百万富翁（在测试网上）。

转账给朋友：

  1. 复制你朋友的测试网钱包地址
  2. 在 MetaMask 中点击 MFT 代币 → "Send"
  3. 粘贴朋友的地址，输入 1000 MFT
  4. 点击"Next" → "Confirm"

在区块浏览器上验证：

在 Sepolia Etherscan 上搜索你的合约地址。你会看到：

  • 代币的总供应量（Total Supply）
  • 持有者列表（Holders）— 至少有你和你朋友两个地址
  • 所有转帐记录（在 ERC-20 Transfers 标签页下）
  • 如果有其他地址调用了 mint 或 burn，也会显示在 Event Logs 中
你的第一次代币部署检查清单：

[✅] 编写 Solidity 合约代码
[✅] 在 Remix 中成功编译
[✅] 部署到 Sepolia 测试网
[✅] 在 MetaMask 中看到代币余额
[✅] 成功发送一笔转账
[✅] 在 Etherscan 上看到交易记录
[✅] 分享了 MFT 给朋友（记得感谢你）
部署的成功感 vs 真实的责任：当你看到自己的代币出现在 Etherscan 上时，那种感觉真的很特别——一个由你的代码创造的、在去中心化网络上独立存在的资产。但请记住：测试网上的 100 万 MFT 和主网上的 100 万 MFT 之间只差一个"Switch Network"的距离。在主网上部署之前，请确保你的合约经过专业审计、有真正的价值支撑、并且团队对用户负责。2022 年 Luna 崩盘后，无数人失去了毕生积蓄——而它的创始人 Do Kwon 现在正在监狱里。

25.6 下一步学习路径

恭喜你完成了第一个代币的部署！这就像编程课上的"Hello World"——但之后的道路还很长。

如果你想继续深入学习智能合约开发，以下是推荐的进阶路径：

阶段内容推荐资源/工具
1. 本地开发从 Remix 迁移到本地开发环境Hardhat 或 Foundry（Anvil + Forge）
2. 测试框架为智能合约编写自动化测试Hardhat 内置测试 + Chai + Ethers.js
3. 前端集成用 React 连接钱包与合约wagmi + viem + RainbowKit / Web3Modal
4. 高级 SolidityGas 优化、代理模式、闪电贷、MEVOpenZeppelin 文档、Ethernaut CTF
5. 安全审计学习常见的智能合约漏洞SWC Registry、Immunefi、Damn Vulnerable DeFi
6. 主网部署评估 Gas 成本、合约验证、审计流程Etherscan 合约验证、DefiLlama
推荐的学习项目：
  • CryptoZombies（cryptozombies.io）— 游戏中学习 Solidity，非常适合新手后续进阶
  • Ethernaut（ethernaut.openzeppelin.com）— OpenZeppelin 出品的智能合约闯关游戏，每一关都是一个真实的安全漏洞
  • Speed Run Ethereum（speedrunethereum.com）— 从入门到部署 DApp 的系列挑战
  • Patrick Collins 的 Hardhat 教程（YouTube）— 全栈 Solidity 开发的最佳视频教程之一

从本章开始，你已经从一个"使用加密货币的人"变成了一个"构建加密货币的人"。这是一个非常重要的转变。下一章我们将讨论一个全球热议的话题——加密资产的监管与合规。毕竟，有能力发币是一回事，在法律框架内负责任地运用这种能力，又是另一回事。
