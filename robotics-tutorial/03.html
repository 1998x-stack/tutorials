第3章 微积分与优化基础

大家好，我是你们的讲师。上一章我们打下了线性代数的基础——那是描述机器人"在什么位置、朝向哪里"的语言。但机器人不是静态的雕塑，它要运动、要控制、要找到最优路径。这就引出了本章的核心主题：微积分和优化。微积分告诉我们"一个量怎么变化"，优化告诉我们在给定了变化规律后，"怎样才能做到最好"。

想象你在操控一条六轴机械臂去抓取一个移动的传送带上的零件。机械臂的每个关节角度随时间变化，末端的位置和速度是关节角度的函数——要精确跟踪运动目标，你需要知道"关节角度变一点点对末端位置的影响有多大"，这正是微分在机器人学中最直接的应用。而优化则体现在：在众多可能的运动轨迹中，找到一条最平滑、最省力、且不碰撞障碍物的那一条。没有微积分和优化，机器人就会像盲人摸象一样——只能瞎试。

3.1 导数与梯度

导数的核心思想非常简单：当一个函数的输入变化一点点时，输出变化多少。用符号来说，\(f'(x) = \lim_{\Delta x \to 0} \frac{f(x+\Delta x) - f(x)}{\Delta x}\)。但在机器人学中，我们几乎永远处理的是多变量函数——比如位置是关节角度的函数 \(\mathbf{p}(\mathbf{q})\)，其中 \(\mathbf{q} = (q_1, q_2, ..., q_6)\) 是六个关节角度。

对于多变量函数，我们把每个方向上的偏导数组合成一个向量，就得到了梯度（Gradient）：\(\nabla f = (\partial f/\partial x_1, \partial f/\partial x_2, ..., \partial f/\partial x_n)^T\)。梯度的方向是函数值增长最快的方向。在机器人路径优化中，如果我们想最小化末端执行器的轨迹误差，就沿着梯度的反方向走——这就是梯度下降（Gradient Descent）的精髓。

让我分享一段亲身经历。几年前，我在做一个双臂协作机器人项目，需要优化两条机械臂的协同轨迹——既要保证它们各自到达目标位置，又要避免互相碰撞，还要最小化总的关节运动量。我一开始试图手工调参，折腾了整整一周，结果两条臂总是在某个区域差点碰撞。后来我换成了梯度优化的方法——把约束写成代价函数，算梯度，然后迭代优化——一天就解决了问题。这件事让我明白了：当你觉得"手动调参"很爽时，其实是在逃避写优化代码。数学工具在大多数情况下比直觉更可靠。

关键洞察：机器人学中的很多问题，本质上都可以归结为"求某个量的梯度"——例如，运动学求位置对关节角的梯度（Jacobian矩阵），动力学求力矩对加速度的梯度，SLAM求位姿对观测误差的梯度。掌握了梯度这一概念，你就掌握了机器人学优化的钥匙。

3.2 雅可比矩阵

雅可比矩阵（Jacobian Matrix）是多变量函数的一阶偏导数矩阵，是梯度概念从标量函数到向量值函数的推广。对于一个将 \(n\) 维输入映射到 \(m\) 维输出的函数 \(\mathbf{f}: \mathbb{R}^n \to \mathbb{R}^m\)，它的Jacobian矩阵是：

J = [[∂f₁/∂x₁, ∂f₁/∂x₂, ..., ∂f₁/∂xₙ],
     [∂f₂/∂x₁, ∂f₂/∂x₂, ..., ∂f₂/∂xₙ],
     [   ...      ...          ...  ],
     [∂fₘ/∂x₁, ∂fₘ/∂x₂, ..., ∂fₘ/∂xₙ]]

# 在机器人学中，J 是 6×n 的矩阵（6=3个位置+3个姿态）
# 第(i,j)个元素：第i个末端运动分量对第j个关节角度的偏导数

Jacobian在机器人学中最直接的应用是速度运动学——它建立了关节速度 \(\dot{\mathbf{q}}\) 和末端空间速度 \(\mathbf{v}\) 之间的线性关系：\(\mathbf{v} = \mathbf{J}(\mathbf{q}) \dot{\mathbf{q}}\)。请注意，这是一个线性关系——虽然机器人的运动学本身是非线性的，但在速度层面，关系是线性的！这就是Jacobian的强大之处：它将一个非线性系统在局部线性化了。

有了这个关系，我们就可以计算逆向速度运动学：\(\dot{\mathbf{q}} = \mathbf{J}(\mathbf{q})^{-1} \mathbf{v}\)——给定末端速度，反推出关节应该以多快的速度运动。当然，这需要 \(\mathbf{J}\) 是方阵且可逆，对于六轴机械臂（6个关节，6个自由度），这就是标准情形。当 \(\mathbf{J}\) 不可逆时，机器人处于奇异位形——下一章我们会深入讨论。

知识点：Jacobian矩阵的零空间（Null Space）对应不改变末端位姿的关节运动——也就是"自运动"。冗余自由度机器人（如7轴KUKA iiwa）正是利用这一点来实现避障的同时保持末端不动。这也是为什么近年来人形机器人普遍采用7自由度手臂——多出来的一个自由度就是"操作裕量"。

3.3 凸优化入门

凸优化（Convex Optimization）是机器人学中最重要的优化框架。一个优化问题是凸的，如果它同时满足两个条件：(1) 目标函数是凸函数；(2) 可行域是凸集。凸优化为什么这么重要？一句话：凸优化问题的任何局部最优解都是全局最优解。这意味着你可以放心地使用各种迭代算法来求解，不用担心陷入局部最优。

在机器人学中，常见的凸优化问题包括：

  • 线性规划（LP）：目标函数和约束都是线性的。用于多机器人任务分配、资源调度。例如，给一个仓储机器人车队分配取货任务，使得总行驶距离最短——这是一个经典的指派问题（Assignment Problem），可以转化为LP求解。
  • 二次规划（QP）：目标函数是二次的，约束是线性的。这是机器人控制中最常见的优化形式。模型预测控制（MPC）的核心求解器就是QP——在每一步优化一个二次型代价函数（跟踪误差 + 控制量惩罚），同时满足关节限位、速度限制等线性约束。
  • 二阶锥规划（SOCP）：约束包含二阶锥形式。常用于鲁棒优化，比如考虑传感器噪声不确定性的轨迹规划。

判断一个问题是不是凸的，有两条实用经验：一是"两个点之间的连线段是否都在可行域内"（判断凸集），二是"函数的Hessian矩阵是否半正定"（判断凸函数）。当然，现实中的很多机器人问题是非凸的（比如考虑障碍物的轨迹规划），但处理非凸问题的标准做法是"先凸松弛，再局部求精"——你在下一章会看到这个思路的具体应用。

3.4 非线性最小二乘

非线性最小二乘（Nonlinear Least Squares）是机器人学中出场率最高的优化形式。它的标准形式是：\(\min_{\mathbf{x}} \sum_{i} \|\mathbf{r}_i(\mathbf{x})\|^2\)，其中 \(\mathbf{r}_i\) 是残差（residual），通常是观测值与模型预测值之间的差异。在机器人学中，几乎所有基于传感器数据的估计问题都可以归结为非线性最小二乘：SLAM中的Bundle Adjustment、手眼标定、IMU预积分、点云配准……

求解非线性最小二乘的两个核心算法是Gauss-Newton和Levenberg-Marquardt（LM）。

Gauss-Newton的思想是：在当前估计点 \(\mathbf{x}_k\) 处对残差做一阶泰勒展开，得到一个线性最小二乘子问题，求解后更新 \(\mathbf{x}\)，重复直到收敛。它的更新步长是 \(\Delta \mathbf{x} = -(\mathbf{J}^T \mathbf{J})^{-1} \mathbf{J}^T \mathbf{r}\)，其中 \(\mathbf{J}\) 是残差的Jacobian矩阵。

Levenberg-Marquardt是Gauss-Newton的改进版：它引入了一个阻尼项 \(\lambda \mathbf{I}\)，将更新公式变为 \(\Delta \mathbf{x} = -(\mathbf{J}^T \mathbf{J} + \lambda \mathbf{I})^{-1} \mathbf{J}^T \mathbf{r}\)。当 \(\lambda\) 很大时，LM退化为梯度下降（稳健但慢）；当 \(\lambda\) 很小时，LM接近Gauss-Newton（快但可能不稳）。LM通过自适应地调整 \(\lambda\)，在两者之间取得平衡。在SLAM的Bundle Adjustment中，LM几乎是事实上的标准求解器——g2o、Ceres Solver、GTSAM这些库的核心都是LM。

避坑指南：使用Gauss-Newton时一定要检查 \(\mathbf{J}^T \mathbf{J}\) 是否可逆。如果残差在某方向上的信息不足（即该方向上的观测为0），\(\mathbf{J}^T \mathbf{J}\) 会是奇异的，算法会发散。这就是为什么在机器人标定时，你需要设计"激励充分"的运动——让机器人在各个方向上都有足够的信息量，否则标定结果会非常不稳定。

3.5 优化方法对比

下表总结了本章涉及的三种主要优化方法，以便你在实际项目中选择合适的工具：

方法 收敛速度 稳定性 是否需要二阶信息 典型应用
梯度下降 / SGD 线性收敛（慢） 高（只要学习率合适） 否（仅一阶梯度） 深度学习训练、在线自适应控制
Gauss-Newton 二次收敛（快） 低（需要 \(\mathbf{J}^T\mathbf{J}\) 可逆） 否（用Jacobian近似Hessian） SLAM Bundle Adjustment、点云配准
Levenberg-Marquardt 介于两者之间 高（阻尼项保证下降） 否（近似Hessian + 阻尼） 标定、SLAM、非线性最小二乘通用求解

3.6 数值优化实战

最后，我们用Python实际演示一下梯度下降和Levenberg-Marquardt在机器人优化中的应用场景。假设我们要优化一个2自由度的机械臂，使其末端达到目标位置：

import numpy as np

# === 两连杆机械臂的正向运动学 ===
def forward_kinematics(q, l1=1.0, l2=0.8):
    """给定关节角度 q = [θ1, θ2]，返回末端位置 (x, y)"""
    x = l1 * np.cos(q[0]) + l2 * np.cos(q[0] + q[1])
    y = l1 * np.sin(q[0]) + l2 * np.sin(q[0] + q[1])
    return np.array([x, y])

# === 目标 ===
target = np.array([1.5, 0.5])

# === 代价函数：末端位置与目标之差的平方和 ===
def cost(q):
    err = forward_kinematics(q) - target
    return 0.5 * np.dot(err, err)   # 1/2 * ||err||^2

# === 解析Jacobian ===
def jacobian(q, l1=1.0, l2=0.8):
    """2自由度机械臂的Jacobian矩阵 (2×2)"""
    θ1, θ2 = q
    J11 = -l1 * np.sin(θ1) - l2 * np.sin(θ1 + θ2)
    J12 = -l2 * np.sin(θ1 + θ2)
    J21 =  l1 * np.cos(θ1) + l2 * np.cos(θ1 + θ2)
    J22 =  l2 * np.cos(θ1 + θ2)
    return np.array([[J11, J12],
                     [J21, J22]])

# === 梯度下降 ===
def gradient_descent(q_init, lr=0.1, n_iters=100):
    q = q_init.copy()
    history = [q.copy()]
    for i in range(n_iters):
        # 计算代价函数对 q 的梯度
        err = forward_kinematics(q) - target
        J = jacobian(q)
        grad = J.T @ err           # ∇cost = J^T * err
        q = q - lr * grad
        history.append(q.copy())
    return q, history

# === Levenberg-Marquardt ===
def levenberg_marquardt(q_init, lam=1.0, n_iters=20):
    q = q_init.copy()
    history = [q.copy()]
    for i in range(n_iters):
        err = forward_kinematics(q) - target
        J = jacobian(q)
        H = J.T @ J               # 近似Hessian
        g = J.T @ err             # 梯度
        # 阻尼求解 (H + λI) Δq = -g
        delta = np.linalg.solve(H + lam * np.eye(2), -g)
        q_new = q + delta
        # 自适应阻尼（简化版：如果代价上升，增大λ；否则减小λ）
        if cost(q_new) < cost(q):
            lam *= 0.8            # 成功：减小阻尼，更接近Gauss-Newton
            q = q_new
        else:
            lam *= 2.0            # 失败：增大阻尼，更接近梯度下降
        history.append(q.copy())
    return q, history

# === 测试 ===
q0 = np.array([0.5, 0.5])         # 初始猜测
q_gd, _ = gradient_descent(q0)
q_lm, _ = levenberg_marquardt(q0)

print(f"初始位置: {forward_kinematics(q0)}")
print(f"目标位置: {target}")
print(f"梯度下降结果: q={q_gd}, pos={forward_kinematics(q_gd)}")
print(f"LM结果: q={q_lm}, pos={forward_kinematics(q_lm)}")

这个例子虽然简单，但五脏俱全——正向运动学、代价函数、Jacobian矩阵、两种优化算法，构成了机器人运动优化的最小闭环。你可以试着调整初始位置、目标点或连杆长度，观察不同算法的行为差异。在实际项目中，你通常不会手写LM——而是使用Ceres Solver、g2o或SciPy的optimize模块——但理解底层原理对于调试和参数调优至关重要。

到这里，微积分与优化的基础就讲完了。我们从导数讲到Jacobian，从凸优化讲到Levenberg-Marquardt，最后用Python实现了完整的最小二乘优化流程。有了数学工具，我们来面对机器人学中最核心的问题——不确定性。在真实世界中，传感器有噪声，执行器有误差，我们需要概率论来量化和管理这种不确定性。下一章见。
