Trie前缀树

大家好。欢迎来到第20章。今天我们把两个看似不相关、但组合起来威力巨大的话题放在一起讲——Trie 前缀树和位运算。Trie 是处理字符串前缀匹配的高效数据结构，而位运算是操作二进制位的底层技巧。当 Trie 遇上异或（XOR），会产生令人惊叹的化学反应，我们会在最后一道题中看到这一点。

先聊聊 Trie。Trie 这个名字来自"Retrieval"的中间四个字母，发音是"try"。它的核心思想是利用字符串的公共前缀来节省空间和查询时间。每个节点代表一个字符，从根节点到某个节点的路径构成了一个字符串的前缀。

Trie 的典型应用场景：自动补全、拼写检查、IP 路由前缀匹配。在 LeetCode 中，Trie 主要考察实现能力和与 DFS、回溯的结合。

实现 Trie（LeetCode 208）

这道题要求实现 Trie 类的三个核心方法：insert、search 和 startsWith。这是 Trie 的模板题，务必熟练掌握。

class TrieNode:
            def __init__(self):
                self.children = {}
                self.is_end = False

        class Trie:
            def __init__(self):
                self.root = TrieNode()

            def insert(self, word: str) -> None:
                node = self.root
                for ch in word:
                    if ch not in node.children:
                        node.children[ch] = TrieNode()
                    node = node.children[ch]
                node.is_end = True

            def search(self, word: str) -> bool:
                node = self.root
                for ch in word:
                    if ch not in node.children:
                        return False
                    node = node.children[ch]
                return node.is_end

            def startsWith(self, prefix: str) -> bool:
                node = self.root
                for ch in prefix:
                    if ch not in node.children:
                        return False
                    node = node.children[ch]
                return True

这里我把 children 实现为字典，这样代码最简洁。如果字符集只有小写字母，也可以用一个长度为 26 的数组——后者在时间和空间上更高效，但代码稍长。面试时建议先用字典实现，有时间再优化为数组。

字典 vs 数组：字典实现方便扩展（可以处理任意字符），数组实现更快更省内存。如果确定字符集只有 'a'-'z'，用 [None] * 26 是最佳选择。Python 中还可以用 defaultdict 来简化代码。

单词搜索 II（LeetCode 212）

这是一道非常有代表性的Trie + 回溯综合题。给定一个字母矩阵和一个单词列表，找出所有在矩阵中能通过相邻字母拼出的单词。如果每个单词单独做 DFS，时间复杂度是 O(单词数 × 4^L)，在大数据量下不可接受。用 Trie 可以把所有单词组织成前缀树，在 DFS 过程中同时匹配所有单词——一旦当前路径不是任何单词的前缀，立即剪枝。

def find_words(board, words):
            # 构建 Trie
            root = {}
            for word in words:
                node = root
                for ch in word:
                    if ch not in node:
                        node[ch] = {}
                    node = node[ch]
                node['#'] = True  # 标记单词结尾

            m, n = len(board), len(board[0])
            result = set()
            directions = [(0,1), (0,-1), (1,0), (-1,0)]

            def dfs(i, j, node, path):
                if '#' in node:
                    result.add(path)
                if i < 0 or i >= m or j < 0 or j >= n:
                    return
                ch = board[i][j]
                if ch not in node:
                    return
                board[i][j] = '#'  # 标记已访问
                for di, dj in directions:
                    dfs(i + di, j + dj, node[ch], path + ch)
                board[i][j] = ch  # 恢复现场

            for i in range(m):
                for j in range(n):
                    dfs(i, j, root, '')
            return list(result)

这个解法的核心优化是：把 Trie 当作 DFS 的"向导"——每走一步都要检查当前前缀是否在 Trie 中，不存在就立即返回。这大幅减少了搜索空间。

异或的性质

聊完 Trie，我们进入位运算的世界。位运算操作的是二进制位，速度极快，是优化代码的终极武器。在所有位运算中，异或（XOR，^）是最常用也最有趣的一个。

异或的三个基本性质是解题的核心：

  • 交换律：a ^ b = b ^ a
  • 结合律：(a ^ b) ^ c = a ^ (b ^ c)
  • 自反性：a ^ a = 0——任何数和自己异或等于 0

另外还有一个推论非常重要：a ^ 0 = a 和 a ^ b ^ b = a。这意味着如果你希望对一个数做两次"异或同一值"的操作，它会回到自身——这是加密和校验的基础。

核心模型：异或是一个"无进位加法"。理解这一点对你做位运算题目很有帮助。1 ^ 1 = 0（1+1=2，二进制进位后本位为 0），1 ^ 0 = 1（1+0=1，无进位）。这个视角在理解"两数之和不用加法"时非常有用。

位运算技巧

只出现一次的数字（LeetCode 136）

题目：给定一个非空整数数组，除了某个元素只出现一次外，其余每个元素都出现两次。找出那个只出现一次的元素。要求线性时间、常数空间。

如果不限制空间，直接用 HashMap 计数。但有了异或，一行代码搞定：

def single_number(nums):
            result = 0
            for num in nums:
                result ^= num
            return result

原理：对所有数做异或，出现两次的数都会抵消变为 0（a ^ a = 0），最后剩下的就是那个只出现一次的数。

变体思路：这个技巧可以扩展到"一个数出现奇数次，其他出现偶数次"——同样用异或。如果题目变成"两个数只出现一次，其他出现两次"（LeetCode 260），就需要先全部异或得到两个目标数的异或结果，再根据某一位是否为 1 将数组分成两组分别异或。

只出现一次的数字 III（LeetCode 260）

这是 136 的进阶版：有两个数只出现一次，其他数都出现两次。我们需要找到这两个数。

def single_number_iii(nums):
            # 第一步：全部异或，得到 x ^ y
            xor_all = 0
            for num in nums:
                xor_all ^= num

            # 第二步：找到 xor_all 中任意一个 1 位
            # 这代表 x 和 y 在这一位上不同
            diff_bit = xor_all & (-xor_all)

            # 第三步：按这一位分组异或
            x = y = 0
            for num in nums:
                if num & diff_bit:
                    x ^= num
                else:
                    y ^= num
            return [x, y]

这里的 diff_bit = xor_all & (-xor_all) 是一个经典位运算技巧——取出最低位的 1。因为 -xor_all 在补码表示下等于 ~xor_all + 1，与原数相与就能得到最低位的 1。

比特位计数（LeetCode 338）

题目：给定一个整数 n，计算从 0 到 n 每个数的二进制表示中 1 的个数。要求 O(n) 时间。

直接思路：对每个数单独计算 bin(i).count('1')，但那是 O(n log n)。如何做到 O(n)？利用动态规划 + 位运算：

def count_bits(n):
            dp = [0] * (n + 1)
            for i in range(1, n + 1):
                # i >> 1 是去掉最低位，i & 1 是最低位是否为 1
                dp[i] = dp[i >> 1] + (i & 1)
            return dp

状态转移方程：dp[i] = dp[i >> 1] + (i & 1)。比如 dp[5]：5 的二进制是 101，5 >> 1 = 2（二进制 10，有 1 个 1），再加最低位 5 & 1 = 1，结果 dp[5] = dp[2] + 1 = 2。非常巧妙，而且只需 O(n) 时间和 O(1) 额外空间（不算答案数组的话）。

Trie与异或结合

数组中两个数的最大异或值（LeetCode 421）

到了本章的高潮部分——Trie 和异或的结合。题目：给定一个非空整数数组，找到两个数的最大异或结果，要求 O(n) 时间。

暴力法 O(n²) 显然会超时。优化的关键思路是：从高位到低位，贪心地确定每一位能否为 1。我们把所有数字的二进制表示插入到 Trie 中，然后对每个数，在 Trie 中寻找一个与它每一位尽量不同的数——因为异或想要结果大，就需要两数的对应位尽量不相同。

class TrieNode:
            def __init__(self):
                self.children = [None, None]  # 0 和 1

        class BinaryTrie:
            def __init__(self):
                self.root = TrieNode()

            def insert(self, num):
                node = self.root
                for i in range(31, -1, -1):
                    bit = (num >> i) & 1
                    if not node.children[bit]:
                        node.children[bit] = TrieNode()
                    node = node.children[bit]

            def find_max_xor(self, num):
                node = self.root
                xor = 0
                for i in range(31, -1, -1):
                    bit = (num >> i) & 1
                    # 贪心：尽量走相反的位
                    toggled = 1 - bit
                    if node.children[toggled]:
                        xor = (xor << 1) | 1
                        node = node.children[toggled]
                    else:
                        xor = (xor << 1) | 0
                        node = node.children[bit]
                return xor

        def find_maximum_xor(nums):
            trie = BinaryTrie()
            for num in nums:
                trie.insert(num)
            max_xor = 0
            for num in nums:
                max_xor = max(max_xor, trie.find_max_xor(num))
            return max_xor

这个解法的精妙之处在于：Trie 把数字按二进制位组织成树，每次查询都在 O(32) = O(1) 时间内完成——因为我们固定了 32 位整数。总时间复杂度 O(32n) = O(n)。

思维模型："两个数异或最大"等价于"在 Trie 中为每个数寻找相反位最多的数"。记住这个"贪心 + Trie"的范式，它也可以用来解决"最大与"或"最大或"的问题，只需要改造 Trie 的查找策略即可。

踩坑与总结

Trie 的内存消耗

Trie 虽然查询快，但内存消耗非常大——每个节点都需要存储指针/字典。一个包含 10 万个英文单词的 Trie 可能占用几十 MB 内存。在 Python 中，将 children 实现为长度为 26 的数组比字典更省内存（字典的哈希表开销很大）。如果遇到 MLE，可以考虑用 __slots__ 优化。

移位运算的优先级陷阱

Python 中，移位运算符的优先级低于加减法！这是一个非常容易踩坑的点：

# 错误写法
        bit = num >> i & 1   # 实际: num >> (i & 1)，完全不是想要的结果

        # 正确写法
        bit = (num >> i) & 1

类似地，1 << i + 1 会被解析为 1 << (i + 1) 而不是 (1 << i) + 1。我的建议是：但凡涉及移位，一律加括号，不要依赖优先级。

技巧 表达式 作用
取最低位 1 x & (-x) 树状数组 lowbit
判断奇偶 x & 1 等价于 x % 2
清空最低位 1 x & (x - 1) 循环移位计数
判断 2 的幂 x > 0 and x & (x - 1) == 0 经典面试题
交换两数 a ^= b; b ^= a; a ^= b 无临时变量交换
一个真实的踩坑案例：我曾经在面试时写 421 题，Trie 的实现没有用 __slots__，结果 LeetCode 上 50 个测试用例全部通过，但面试官追问"你能估算一下你的 Trie 用了多少内存吗？"我支支吾吾半天答不上来。然后他又追问"如果数字范围扩大到 64 位呢？"——这是 Trie 的一个短板，位数增加意味着深度增加，每一层都要存储节点指针。所以用 Trie 做位运算时，要确认数据范围是否可控。
