28.2 项目：电商用户购买预测系统

业务场景

某电商平台需要预测用户在未来7天内是否会下单购买。基于这个预测，运营团队会对"高购买意向"用户发放优惠券，对"流失风险"用户发送Push唤醒。这是一个典型的二分类问题，但涉及数据管道、特征工程、模型训练、可解释性、部署监控的完整链路。

28.3 第一步：需求分析与指标定义

在写任何代码之前，先明确业务目标和评价指标：

  • 业务目标：用最低的营销成本最大化用户转化率
  • ML问题：二分类——预测用户是否会在未来7天内购买
  • 主要指标：AUC-ROC（评估排序能力）
  • 业务指标：优惠券转化率（发了券后实际购买的比例）
  • 约束：推理延迟 < 100ms，支持1000 QPS
💡 项目开始前必问的三个问题：(1) 这个模型预测了什么业务动作会因此改变？如果预测结果不会被使用，模型就没有价值。(2) 效果怎么衡量？必须有一个可量化的成功标准。(3) 最坏情况是什么？模型如果完全不工作会造成多大损失？这决定了你需要多快的自动回滚。

28.5 第三步：模型训练与实验管理

import mlflow
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.model_selection import StratifiedKFold, cross_val_score

mlflow.set_experiment("purchase_prediction_v2")

with mlflow.start_run(run_name="gbdt_baseline"):
    # 记录数据版本
    mlflow.log_param("data_version", "2026-06-30")
    mlflow.log_param("train_samples", len(X_train))
    mlflow.log_param("positive_rate", y_train.mean())

    # 不平衡处理（Ch16）
    model = GradientBoostingClassifier(
        n_estimators=200, max_depth=5, learning_rate=0.05,
        subsample=0.8, min_samples_leaf=50
    )

    # 时序交叉验证（Ch7 模型评估）
    cv = StratifiedKFold(n_splits=5, shuffle=True)
    auc_scores = cross_val_score(model, X_train, y_train,
                                  cv=cv, scoring='roc_auc')
    mlflow.log_metric("cv_auc_mean", auc_scores.mean())
    mlflow.log_metric("cv_auc_std", auc_scores.std())

    model.fit(X_train, y_train)
    mlflow.sklearn.log_model(model, "model")
