大家好，上一章我们聊了可视化与调试——确保算法实现正确。但实现正确只是第一步。现在问题来了：你的 A* 实现比别人快吗？你的 RRT* 找到的路径质量如何？你改进了 DWA 的代价函数，到底是变好了还是变差了？要回答这些问题，你需要一个严格的基准测试（benchmark）体系。

2015 年我在某自动驾驶公司实习时，同事小明说他优化后的 A* 比原来的快了一倍。我问他怎么测的，他说："我找了一张地图跑了三次。"结果我替他做了 100 次测试后发现，他的"优化"版本平均只快了 8%，而且有 15% 的情况下新版本更慢。问题出在哪？单次运行的随机波动被当成了性能提升。从那以后我养成了一个习惯：任何性能声明，必须有完整的基准测试数据作为支撑，否则一律视为"感觉"而非"事实"。

评价指标体系

路径规划算法的"好"是多维度的。只用一个指标评价算法就像只用身高评价篮球运动员——可能是最不重要的一项被当成了全部。下面是我们在评价路径规划算法时最常用的指标体系：

指标 单位 含义 适用范围 注意事项
路径长度 米 / 栅格数 从起点到终点的路径总长度 所有算法 最短路径不一定是最优路径（需要考虑平滑度、安全性）
计算时间 毫秒（ms） 算法从调用到返回路径的总耗时 所有算法 注意区分单次 vs 平均、排除 Python 解释器开销
成功率 百分比（%） 在 N 次试验中找到可行路径的比例 采样算法为主 和地图难度强相关，报告成功率时必须附带地图信息
路径平滑度 弧度 / 曲率变化 路径中转弯的剧烈程度，通常用最大曲率或累积转弯角衡量 全局规划 + 后处理 对人类来说"平滑"的路径很难用单一指标量化
内存使用 MB 算法运行过程中占用的峰值内存 嵌入式系统、大规模地图 Python 的内存管理会掩盖真实使用量，C++ 实现更准确
扩展节点数 个 搜索过程中访问的节点/状态数量 图搜索算法 与计算时间强相关，但更稳定（不受 CPU 调度影响）
血的教训：不要只看路径长度！一个比 A* 短 5% 但慢 10 倍的算法在机器人上毫无意义。真实的机器人规划是一个在线问题——如果规划耗时 500 毫秒而机器人速度是 1m/s，这意味着机器人在这 0.5 秒内要盲开半米。在狭窄空间中这半米足以酿成碰撞事故。我见过不止一个团队把论文级别的规划器搬到机器人上，精度无可挑剔，但计算时间太长导致机器人撞墙。评价算法时，永远把实时性放在和最优性同等重要的位置。

测试场景设计

基准测试的第二个关键要素是测试场景。如果你只在一种地图上测试算法，你学到的只是这个算法在那一种地图上的表现——这个结论对其他地方几乎没有参考价值。我们需要一组多样化的、有代表性的测试场景：

五种标准地图类型

地图类型 描述 测试目的 典型尺寸
狭窄通道 两个大障碍物之间有一条仅容一车通过的窄缝 测试算法处理约束瓶颈的能力 100x100
稠密障碍 地图上密集分布随机大小和位置的障碍物 测试算法在高复杂度环境中的搜索效率 200x200
迷宫 标准迷宫结构，从入口到出口 测试算法的探索策略和避免死胡同的能力 100x100
开阔平原 几乎没有障碍物的空旷地图 测试算法的基线表现和路径质量 100x100
随机杂乱 随机生成的多边形/圆形障碍物 模拟真实场景的不可预测性 200x200

下面是一个生成标准测试地图的 Python 函数。我建议你将这批地图保存下来作为你的"算法测试套件"，每次新算法上线前都要在这套地图上跑一遍：

import numpy as np
from scipy.ndimage import binary_dilation

def generate_benchmark_maps(size=100, seed=42):
    """生成五种标准测试地图"""
    rng = np.random.RandomState(seed)
    maps = {}

    # 1. 狭窄通道
    m = np.zeros((size, size))
    m[:, size//2 - 2:size//2 + 2] = 1  # 竖墙
    # 在墙中间开一个小口
    m[size//2 - 3:size//2 + 3, size//2 - 2:size//2 + 2] = 0
    maps['narrow_passage'] = m

    # 2. 稠密障碍
    m = np.zeros((size, size))
    num_obstacles = 200
    for _ in range(num_obstacles):
        cx, cy = rng.randint(5, size-5, 2)
        r = rng.randint(3, 10)
        rr, cc = np.ogrid[:size, :size]
        obstacle = (rr - cy)**2 + (cc - cx)**2 <= r**2
        m[obstacle] = 1
    maps['cluttered'] = m

    # 3. 迷宫（递归回溯算法生成）
    m = np.ones((size, size))
    # 用简单的螺旋迷宫
    for i in range(size // 4):
        layer_start = 2 * i + 1
        layer_end = size - 2 * i - 1
        if layer_start >= layer_end:
            break
        m[layer_start:layer_end, layer_start] = 0  # 左竖
        m[layer_start, layer_start:layer_end] = 0  # 上横
        m[layer_start:layer_end, layer_end-1] = 0  # 右竖
        m[layer_end-1, layer_start:layer_end] = 0  # 下横
        # 开缺口
        m[layer_start, layer_start + 2] = 1
        m[layer_start + 2, layer_end - 1] = 1
    maps['maze'] = m

    # 4. 开阔平原
    maps['open'] = np.zeros((size, size))

    # 5. 随机杂乱
    m = np.zeros((size, size))
    for _ in range(50):
        cx, cy = rng.randint(10, size-10, 2)
        w, h = rng.randint(5, 20, 2)
        angle = rng.uniform(0, np.pi)
        # 粗略旋转矩形
        x, y = np.meshgrid(np.arange(size), np.arange(size))
        x_rot = (x - cx) * np.cos(angle) + (y - cy) * np.sin(angle)
        y_rot = -(x - cx) * np.sin(angle) + (y - cy) * np.cos(angle)
        rect = (np.abs(x_rot) < w/2) & (np.abs(y_rot) < h/2)
        m[rect] = 1
    # 用膨胀让障碍物略微扩大
    m = binary_dilation(m, iterations=1).astype(int)
    maps['random_clutter'] = m

    return maps

公平对比的原则

做基准测试最核心的诉求是公平。不公平的对比比不对比更糟糕——因为它给你错误的结论。以下是确保公平对比的几个黄金原则：

同一硬件环境

同一组对比实验必须在同一台机器上运行。不同代的 CPU（比如 i7-12700 vs i7-14700）对计算时间的影响可能达到 2 倍以上。如果实在需要在不同机器上跑，请使用"归一化"指标，比如把计算时间除以同机器上 Dijkstra 的运行时间作为相对比值。

同一时间预算

对于 anytime 算法（RRT*、PRM 等），算法在运行 10ms 时的表现和在运行 1000ms 时完全不同。对比时必须固定时间预算，或者绘制"时间-质量"曲线来全面对比。我常用的是在不同时间预算下分别跑 50 次，然后绘制路径长度随时间的衰减曲线。

统计显著性

单次测试毫无意义。路径规划算法（尤其是基于采样的算法）具有很强的随机性。至少跑 30-50 次试验，计算均值和标准差。用 t 检验或 Mann-Whitney U 检验来判断两个算法的差异是否统计显著。

经验法则：RRT 在相同条件下两次运行的计算时间差异可达 3-5 倍。如果你只跑一次就下结论，你的结论几乎肯定会被下一次运行推翻。对于采样算法，至少跑 50 次；对于确定性算法（A*、Dijkstra），至少跑 10 次（虽然结果确定，但系统负载会影响计时）。

基准测试脚本

下面是一个完整的基准测试框架。它将所有算法统一包装成相同的接口，自动在多种地图上运行、收集指标、生成报告：

import time
import json
import numpy as np
from dataclasses import dataclass, field
from typing import Callable, Dict, List, Optional
from pathlib import Path


@dataclass
class BenchmarkResult:
    """单次测试的结果"""
    algorithm: str
    map_name: str
    trial: int
    success: bool
    path_length: float
    computation_time_ms: float
    nodes_expanded: int
    max_memory_mb: float


@dataclass
class BenchmarkConfig:
    """基准测试配置"""
    algorithms: Dict[str, Callable]  # 名称 -> 规划函数
    maps: Dict[str, np.ndarray]      # 名称 -> 栅格地图
    start_goal_pairs: List[tuple]    # [(start, goal), ...]
    trials_per_config: int = 30      # 每种配置的试验次数
    time_budget_ms: float = 1000.0   # anytime 算法的时间预算
    seed: int = 42


def run_benchmark(config: BenchmarkConfig) -> List[BenchmarkResult]:
    """运行完整的基准测试"""
    results = []
    rng = np.random.RandomState(config.seed)

    for algo_name, planner_fn in config.algorithms.items():
        for map_name, grid in config.maps.items():
            for trial in range(config.trials_per_config):
                # 选择 start-goal 对（随机或固定）
                if config.start_goal_pairs:
                    sg = config.start_goal_pairs[
                        rng.randint(len(config.start_goal_pairs))]
                else:
                    sg = _pick_random_start_goal(grid, rng)

                start_time = time.perf_counter()
                try:
                    path, stats = planner_fn(
                        grid, sg[0], sg[1],
                        time_budget=config.time_budget_ms / 1000.0,
                    )
                    elapsed = (time.perf_counter() - start_time) * 1000  # ms
                    success = path is not None

                    result = BenchmarkResult(
                        algorithm=algo_name,
                        map_name=map_name,
                        trial=trial,
                        success=success,
                        path_length=sum(
                            np.sqrt((path[i][0]-path[i-1][0])**2 +
                                    (path[i][1]-path[i-1][1])**2)
                            for i in range(1, len(path))
                        ) if success else float('inf'),
                        computation_time_ms=elapsed,
                        nodes_expanded=stats.get('nodes_expanded', 0),
                        max_memory_mb=stats.get('max_memory', 0),
                    )
                except Exception as e:
                    result = BenchmarkResult(
                        algorithm=algo_name, map_name=map_name,
                        trial=trial, success=False,
                        path_length=float('inf'),
                        computation_time_ms=0,
                        nodes_expanded=0, max_memory_mb=0,
                    )
                results.append(result)

    return results


def summarize_results(results: List[BenchmarkResult]) -> Dict:
    """汇总基准测试结果，生成报告"""
    summary = {}
    for result in results:
        key = (result.algorithm, result.map_name)
        if key not in summary:
            summary[key] = {
                'success_rate': [],
                'path_lengths': [],
                'times': [],
                'nodes': [],
            }
        if result.success:
            summary[key]['success_rate'].append(1)
            summary[key]['path_lengths'].append(result.path_length)
            summary[key]['nodes'].append(result.nodes_expanded)
        else:
            summary[key]['success_rate'].append(0)
        summary[key]['times'].append(result.computation_time_ms)

    report = {}
    for (algo, mname), vals in summary.items():
        success_rate = np.mean(vals['success_rate'])
        report[f"{algo}@{mname}"] = {
            'success_rate': f"{success_rate*100:.1f}%",
            'path_length_mean': f"{np.mean(vals['path_lengths']):.2f}",
            'path_length_std': f"{np.std(vals['path_lengths']):.2f}",
            'time_ms_mean': f"{np.mean(vals['times']):.1f}",
            'time_ms_std': f"{np.std(vals['times']):.1f}",
            'nodes_mean': f"{np.mean(vals['nodes']):.0f}",
            'trials': len(vals['success_rate']),
        }
    return report


def _pick_random_start_goal(grid, rng):
    """在地图空白区域随机选取起点和终点"""
    free = np.argwhere(grid == 0)
    idx = rng.choice(len(free), 2, replace=False)
    return (tuple(free[idx[0]][::-1]), tuple(free[idx[1]][::-1]))


# ---- 使用示例 ----
if __name__ == '__main__':
    from planners import astar_planner, rrt_planner, dwa_planner

    maps = generate_benchmark_maps(size=100)

    config = BenchmarkConfig(
        algorithms={
            'A*': astar_planner,
            'RRT': rrt_planner,
        },
        maps=maps,
        start_goal_pairs=[((5, 5), (95, 95))],
        trials_per_config=30,
    )

    results = run_benchmark(config)
    report = summarize_results(results)
    print(json.dumps(report, indent=2))

关键洞察：世界上没有"最好的"路径规划算法，只有"最适合特定场景"的算法。基准测试的真正价值不是告诉你哪个算法赢了，而是告诉你每个算法在什么场景下能做什么、不能做什么。当你面对新的规划问题时，回过头查阅你的基准测试数据库，你就能迅速缩小候选范围。这就是基准测试的复利效应——用得越多，收益越大。

建立你自己的基准测试管道需要一些前期投入，但这是整个路径规划工具箱中回报率最高的投资之一。每次你改进了一个算法、或者设计了一个新的规划器，只需运行 python benchmark.py，几分钟后你就能看到它在所有场景下的完整表现。不再靠感觉、不再凭运气。下一章我们将把经过基准测试验证的算法从仿真环境搬到真实机器人上——这才是真正的考验。我们第 25 章见。
