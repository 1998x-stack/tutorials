如何说两个样本"像不像"？

大家好，欢迎继续学习。在多元分析中，有一个问题贯穿所有方法：如何量化两个样本（或两个变量）之间的"相似程度"？这个问题看似简单，但实际上不同场景下的答案截然不同。聚类分析要根据"距离"把样本分组；判别分析要根据"距离"把新样本归到最近的类别；多维标度法要尽量保留样本间的"距离"关系在低维空间里——你看，"距离"几乎出现在了所有多元方法的核心。

但"距离"不是一个非黑即白的概念。两个人在"年龄"上相差 10 岁和在"收入"上相差 10 万元，哪个"距离"更大？无法直接比较，因为量纲不同。如果有 50 个变量，"距离"又怎么算？如果变量是分类型的（如性别：男/女），"距离"又如何定义？这一章我们就来系统地梳理这个问题，为后续的聚类、判别、MDS 等方法打下基础。

距离度量概述

在数学上，一个函数 d(x, y) 要称为"距离度量" (distance metric)，必须满足四个性质：

性质含义公式
非负性距离不可能是负的d(x, y) ≥ 0
同一性距离为0当且仅当两个点相同d(x, y) = 0 ⇔ x = y
对称性A到B的距离等于B到A的距离d(x, y) = d(y, x)
三角不等式直接距离不超过经过第三点的路径d(x, z) ≤ d(x, y) + d(y, z)

在聚类分析中，并不是所有用到的"相似性度量"都满足这些性质——比如余弦相似度就不满足三角不等式，但它在文本分析中仍然极其有用。所以更宽泛地说，我们关心的是任何能合理度量样本间相似/相异程度的量。

欧氏距离与明氏距离

欧氏距离 (Euclidean distance) 是大家最熟悉的距离度量，就是初中几何中的"直线距离"：

d(x, y) = √[(x₁ - y₁)² + (x₂ - y₂)² + ... + (xₚ - yₚ)²]

等价于：d(x, y) = ||x - y||₂

欧氏距离的好处是直观、计算简单。但它在多元分析中有一个致命缺陷：它对所有变量一视同仁，没有考虑变量的相关性和尺度差异。比如身体健康数据包含"身高（厘米）"和"体重（公斤）"两个变量——用欧氏距离时，身高从 150 到 190 的差异范围（约 40），远大于体重从 50 到 90 的差异（约 40，但量纲不同已经不重要了），实际上如果再加上"体温（36-37℃）"，体温的差异几乎可以忽略不计——但实际上体温异常可能比身高差异更重要。

明氏距离 (Minkowski distance) 是欧氏距离的推广：

d(x, y) = (Σ|xᵢ - yᵢ|ᵐ)^{1/m}

m=1: 曼哈顿距离 (Manhattan distance) — 城市街区距离
m=2: 欧氏距离
m→∞: 切比雪夫距离 (Chebyshev distance) — 最大差异

曼哈顿距离对异常值不如欧氏距离敏感（因为不平方差异），在数据含有较多异常值时可能更稳健。切比雪夫距离则只关心"最差的那个维度"——在某些场景（如多维度量中的"短板效应"）下有意义。

马氏距离

前面我们多次提到了马氏距离 (Mahalanobis distance)，现在是时候正式定义它了。给定一个分布（均值向量 μ，协方差矩阵 Σ），点 x 到分布中心 μ 的马氏距离为：

D_M(x) = √[(x - μ)ᵀ Σ⁻¹ (x - μ)]

两个点之间的马氏距离：

D_M(x, y) = √[(x - y)ᵀ Σ⁻¹ (x - y)]

马氏距离相较于欧氏距离有两大优势：

  • 尺度不变性 (scale invariance)：马氏距离考虑了各变量的方差差异，自动做了"标准化"。你把一个变量从"米"换成"厘米"，马氏距离不变——欧氏距离会变。
  • 相关性考虑：马氏距离通过 Σ⁻¹ 来"去除"变量间的相关性。如果两个变量高度正相关，那么沿相关方向的距离会被"缩短"——因为在这个方向上同时变化是"正常"的变异性，不应视为大的差异。
几何直觉：想象二维平面上一个椭圆形的点云（变量正相关）。欧氏距离的等高线是圆形——它认为各个方向的距离同等重要。但马氏距离的等高线是椭圆——沿着点云"伸展"的方向（即相关方向），同样的欧氏距离对应更小的马氏距离；垂直于点云"狭窄"方向，同样的欧氏距离对应更大的马氏距离。换句话说，马氏距离把数据"白化"（whiten）成了圆形后再计算欧氏距离，这也等价于先做 PCA 再在"主成分空间"中算欧氏距离。

马氏距离的唯一缺点是：需要计算协方差矩阵的逆。当 p > n 时 Σ 不可逆，需要用伪逆或正则化方法。另外，如果数据不是多元正态的，马氏距离的"统计距离"解释力会下降。

余弦相似度与相关系数

余弦相似度 (cosine similarity) 衡量的是两个向量之间方向的相似程度，而不是长度的差异：

cos(x, y) = (x · y) / (||x|| · ||y||)

取值范围：[-1, 1]
1 表示方向完全相同
0 表示正交
-1 表示方向完全相反

在文本分析中，余弦相似度是标配工具——文档被表示为词频向量后，用欧氏距离度量"文档差异"效果很差，因为文档长度不同会导致词频向量的模差异巨大。但余弦相似度只看词的分布模式而不是文档长度，因此更合理。

皮尔逊相关系数 (Pearson correlation coefficient) 本质上就是中心化后的余弦相似度——把每个变量减去其均值后再计算余弦相似度：

r(x, y) = Σ(xᵢ - x̄)(yᵢ - ȳ) / √[Σ(xᵢ - x̄)² · Σ(yᵢ - ȳ)²]

所以，皮尔逊相关系数衡量的是两个变量线性相关的程度，它不受变量尺度（量纲）的影响。如果 y = a + bx（b > 0），那么 r = 1——斜率和截距不影响相关系数。

斯皮尔曼秩相关系数 (Spearman's rank correlation) 是皮尔逊相关系数的非参数版本：先将数据转换为秩，再计算皮尔逊相关系数。它不要求变量之间的关系是线性的，只要求单调关系。如果数据中有异常值或者关系是单调但非线性的（如指数增长），斯皮尔曼相关系数比皮尔逊更稳健。

度量类型范围适用场景
欧氏距离距离[0, ∞)连续变量，变量独立同尺度
马氏距离距离[0, ∞)连续变量，变量相关且尺度不同
余弦相似度相似度[-1, 1]文本、方向性数据（忽略模长）
皮尔逊相关系数相似度[-1, 1]线性关系的强度
斯皮尔曼秩相关相似度[-1, 1]单调关系（对异常值稳健）
Jaccard 系数相似度[0, 1]二元/分类变量

分类变量的距离

当变量是分类变量（如性别、颜色、地区）而不是连续的数值时，上述距离度量都不适用。我们需要专门的方法。

简单匹配系数 (Simple Matching Coefficient)

对于两个二元向量 x 和 y：

SMC = (同时为1的个数 + 同时为0的个数) / 总变量数

Jaccard 系数

Jaccard = 同时为1的个数 / (至少有一个为1的个数)

Jaccard 系数与 SMC 的区别在于它不考虑同时为0的情况。在文本分析中，两个文档同时不包含某个词的信息量很小——几乎所有词都不出现。这时候 Jaccard 系数更合适。

Gower 距离

Gower 距离是最灵活的方案之一——它可以同时处理数值变量和分类变量混合的数据集。对数值变量使用归一化后的曼哈顿距离，对分类变量使用"相同=0，不同=1"的匹配度量，然后对所有变量取平均。

实战经验：很多新手在做聚类时，面对混合类型数据（既有年龄、收入等连续变量，又有性别、教育水平等分类变量），会直接"扔掉"分类变量或者给它们随便编码成数字（比如男=1，女=2）然后算欧氏距离——这两种做法都不对！前者浪费了信息，后者人为地制造了"男和女的差距为1"这样的无意义数值距离。正确的做法是用 Gower 距离或先将分类变量做 one-hot 编码再用适当的距离度量。

距离矩阵与可视化

在多元分析中，我们经常需要计算所有样本两两之间的距离，得到一个 n × n 的距离矩阵 (distance matrix) D，其中 Dij = d(xᵢ, xⱼ)。

距离矩阵本身就是一个强大的分析工具：

  • 它可以直接输入到 层次聚类 (hierarchical clustering) 中——我们在第15章会用到
  • 它是 多维标度法 (MDS) 的唯一输入——第22章的核心
  • 对距离矩阵做热图可视化可以直观地看到数据的聚类结构
import numpy as np
from scipy.spatial.distance import pdist, squareform
import seaborn as sns
import matplotlib.pyplot as plt

# 生成示例数据
np.random.seed(42)
X = np.random.randn(50, 4)  # 50个样本，4个变量

# 计算欧氏距离矩阵
dist_matrix = squareform(pdist(X, metric='euclidean'))

# 热图可视化
sns.heatmap(dist_matrix, cmap='viridis')
plt.title("欧氏距离矩阵热图")
plt.show()

# 换成马氏距离
# 注意：pdist 的 mahalanobis 参数需要提供协方差逆矩阵
VI = np.linalg.inv(np.cov(X, rowvar=False))
dist_mahal = squareform(pdist(X, metric='mahalanobis', VI=VI))

热图的颜色越深（或越冷色）表示距离越近，越亮（或越暖色）表示距离越远。如果数据天然有聚类结构，你会看到沿对角线存在一些"暗色方块"——那是同一类内的样本互相距离近。

踩坑故事：欧氏距离在基因表达数据中的失败

几年前我被问到这样一个问题：为什么用某个基因表达数据做聚类分析，得到的聚类结果完全不符合生物学常识？我去看了看那位同事的数据和代码，发现他用的是欧氏距离。问题出在哪里呢？基因表达数据中，不同基因的表达水平差异极大——有的基因（如看家基因）表达量高达数万，有的只有几十。使用欧氏距离时，那些高表达基因完全主导了距离计算，低表达基因的差异被淹没在"噪声"中。实际上生物学家更关心的是基因表达的相对模式而非绝对水平——两个样本在一个低表达基因上的两倍差异可能比在一个高表达基因上的 10% 差异更有生物学意义。

解决方案是什么？用 相关系数距离：d = 1 - r（r 是皮尔逊相关系数）。相关系数只关心基因表达的模式是否相似（比如是否一起上调或下调），而不关心绝对表达水平。改用相关系数距离后，聚类结果立马就合理了。

教训：选择距离度量不是数学问题，而是领域问题。在使用任何距离度量前，先问自己三个问题：(1) 变量的量纲有意义吗？(2) 我关心的是"绝对差异"还是"相对模式"？(3) 变量中有没有分类变量需要特殊处理？这三个问题的答案直接决定了你应该用欧氏距离、马氏距离、余弦相似度还是 Jaccard 系数。记住：选对了距离，聚类就成功了一半。

本章小结与衔接

这一章我们从距离度量的数学定义出发，回顾了欧氏距离和明氏距离族（曼哈顿、切比雪夫），深入理解了马氏距离如何通过 Σ⁻¹ 实现尺度不变性和去相关（它是 PCA 白化的另一种视角），学习了余弦相似度和两种相关系数（皮尔逊用于线性关系、斯皮尔曼用于单调关系），针对分类变量介绍了 SMC、Jaccard 系数和 Gower 距离，最后展示了距离矩阵的计算与可视化。贯穿始终的一个核心观点是：距离度量的选择必须考虑数据的特性和分析的目标，没有放之四海而皆准的"最佳距离"。

有了距离度量这个工具，下一章我们将进入多元分析的第一个"正式方法"——多元回归基础。你可能已经接触过一元线性回归，但多元线性回归不仅仅是把多个自变量放进方程那么简单——它面临着多元共线性、变量选择、正则化等一系列新问题。而且，多元回归的矩阵形式 Y = Xβ + ε 与其最小二乘解 β̂ = (X'X)⁻¹X'Y，将是我们后续理解典型相关分析和判别分析的重要桥梁。
