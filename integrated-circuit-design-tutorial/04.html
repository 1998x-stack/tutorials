第4章 CMOS数字电路基础

大家好，我是你们的讲师。在前两章，我们经历了从半导体物理到MOS晶体管的"底层之旅"。现在我们终于可以向上迈出关键的一步——用MOSFET来构建数字电路。

这一章的内容，本质上回答了这个问题："0"和"1"在芯片内部到底是什么？它不是抽象的逻辑值，而是具体的电压：高电平（VDD）和低电平（GND）。CMOS电路用一对互补的NMOS和PMOS网络来可靠地产生和维持这两个电压状态。理解这个"互补"的哲学，是理解整个数字芯片设计的钥匙。

4.1 CMOS反相器

反相器（Inverter）是CMOS数字电路中最基本、也最重要的构建模块。它只有一个输入和一个输出——输出是输入的"取反"——但它的特性决定了整个芯片的速度、功耗和噪声容忍度。

CMOS反相器由两个晶体管组成：一个PMOS作为上拉网络（Pull-Up Network, PUN），连接VDD和输出；一个NMOS作为下拉网络（Pull-Down Network, PDN），连接GND和输出。

  • 输入为0（GND）：NMOS关断，PMOS导通，输出通过PMOS连接到VDD → 输出为1。
  • 输入为1（VDD）：PMOS关断，NMOS导通，输出通过NMOS连接到GND → 输出为0。

静态（输入不变）时，总有一个管子关断，从VDD到GND之间没有直流通路——这是CMOS电路零静态功耗（忽略漏电流）的根本原因。这个特性使得CMOS比NMOS逻辑或双极型逻辑（如TTL、ECL）具有压倒性的功耗优势，也是CMOS能成为数字IC绝对主流的原因。

电压传输特性（VTC）

反相器的输入电压从0扫到VDD，输出电压的变化曲线就是电压传输特性（Voltage Transfer Characteristic, VTC）。理想的反相器VTC是一个"阶跃函数"，但实际上它是一个平滑的"S"形曲线。

VTC上几个关键点：

  • 开关阈值（Switching Threshold, VM）：Vin = Vout的点。通常设计在VDD/2附近，使噪声容限对称。VM由PMOS和NMOS的驱动能力比决定——PMOS做宽一些可以上移VM。
  • 噪声容限（Noise Margins）：衡量电路对输入噪声的容忍能力。NML（低电平噪声容限）= VIL - VOL；NMH（高电平噪声容限）= VOH - VIH。简单说，只要输入噪声幅度不超过噪声容限，输出就不会错误翻转。

反相器的延迟

反相器的传播延迟tp定义为从输入变化50%到输出变化50%的时间，通常分为tpHL（高到低）和tpLH（低到高）。延迟由晶体管的导通电阻和负载电容（扇出门的输入电容＋互连线电容）决定——这就是RC延迟模型的核心思想。

关于"为什么CMOS赢了"的简洁解释：在1980年代，行业内存在多种逻辑家族——NMOS逻辑、TTL（双极型）、ECL（发射极耦合逻辑）、CMOS等。CMOS的晶体管数量多（互补对意味着每个门需要两倍的管子），但功耗优势太显著了。一个NMOS逻辑反相器在"0"状态会有持续的静态电流（因为上拉电阻一直通电），而CMOS只在翻转时消耗动态电流。在大规模VLSI中，功耗是决定性的——几百万个门的静态电流累积起来是不可接受的。所以CMOS赢了，而且赢得非常彻底。

4.2 组合逻辑门设计

基本逻辑门

把反相器的思路扩展一下：用并联的PMOS网络实现"或"逻辑，串联的NMOS网络实现"与"逻辑（反过来也一样），我们就可以构建任意组合逻辑功能。

  • NAND（与非）：两个NMOS串联＋两个PMOS并联。输入全为1时输出0，否则输出1。NAND是最"高效"的CMOS门——它的速度比NOR快，因为PMOS（空穴迁移率低）并联而不是串联。
  • NOR（或非）：两个NMOS并联＋两个PMOS串联。输入全为0时输出1，否则输出0。
  • XOR（异或）：需要更多的晶体管来实现。典型实现需要两级逻辑或传输门结构。
特性 NAND NOR
NMOS结构 串联（下拉路径） 并联（下拉路径）
PMOS结构 并联（上拉路径） 串联（上拉路径）
上拉速度 快（PMOS并联，等效电阻小） 慢（PMOS串联，等效电阻大）
下拉速度 慢（NMOS串联） 快（NMOS并联）
总体延迟（等尺寸） 更优 更差
面积效率 更高 更低（PMOS需要大尺寸补偿）
设计偏好 优先使用 尽量避免

复合门与逻辑努力

更复杂的逻辑功能（如AOI——与或非门、OAI——或与非门）可以通过适当组合串联和并联网络来实现。这时候逻辑努力（Logical Effort）方法就非常有用了。逻辑努力是一种快速估算门延迟的方法：

d = f + p = g · h + p

其中d是归一化延迟，g是逻辑努力（门的固有复杂度），h是电气努力（扇出比：负载电容/输入电容），p是本征延迟。通过逻辑努力，设计师可以快速找出最优的门级数和晶体管尺寸。

4.3 传输门与三态门

传输门

一个传输门（Transmission Gate, TG）由一个NMOS和一个PMOS并联组成，两条控制信号互补。当控制信号有效时，传输门导通，输入信号可以"无衰减地"传递到输出（NMOS传递强0，PMOS传递强1，互补组合后任意电平都能无损通过）。

传输门是实现多路选择器（MUX）、D锁存器、XOR门的高效方式。不过需要小心的是：传输门是"双向"的——源和漏是对称的，信号可以从任何一端流向另一端。这在某些设计中会引入意料之外的路径。

三态缓冲器

三态缓冲器（Tri-state Buffer）有三个输出状态：0、1和高阻（High-Z，即输出断开）。它通过一个使能信号EN来控制是否驱动输出。三态缓冲器在总线结构中至关重要——多个模块可以共享同一根总线，但在同一时刻只有一个模块驱动总线，其他模块把输出置为高阻态。

4.4 时序单元

组合逻辑门处理的是"当前输入决定当前输出"。要构建有"记忆"的电路——寄存器、计数器、状态机——我们需要时序单元。

锁存器 vs 触发器

  • 锁存器（Latch）：透明（transparent）——当使能信号有效时，输出跟随输入变化；使能无效时，输出保持。锁存器是电平触发的。
  • 触发器（Flip-Flop, FF）：边沿触发——只在时钟沿（上升沿或下降沿）采样输入，其他时间输出保持不变。最常见的实现是主从触发器（master-slave DFF），由两个D锁存器级联而成。

建立时间（Setup Time, tsu）和保持时间（Hold Time, th）是触发器最重要的时序参数：

  • 建立时间：数据必须在时钟沿到来之前提前保持稳定的最短时间。
  • 保持时间：数据必须在时钟沿到来之后继续保持稳定的最短时间。

触发器的这些参数决定了系统的最快时钟频率（建立时间约束）和最小路径延迟（保持时间约束）。静态时序分析（STA，第10章）的核心就是检查这些约束是否被满足。

下面是一个D触发器的Verilog门级描述：

// 主从D触发器（正沿触发）—— 门级Verilog描述
module dff_gate (
    input  wire clk,
    input  wire d,
    output wire q,
    output wire qn
);
    wire m, mn;      // 主级输出
    wire ck, ckn;    // 时钟反相

    // 时钟反相器
    not (ckn, clk);
    not (ck, ckn);   // ck = clk（缓冲后的正相时钟）

    // 主级传输门（clk=0时透明）
    // 输入d -> 主级节点m
    // 此处使用简化表示，实际门级网表会用CMOS传输门实现
    cmos_tg tg1 (.a(d), .b(m), .cn(ckn), .cp(ck));
    // 主级反馈保持（clk=1时反馈环路闭合）
    cmos_tg tg2 (.a(mn), .b(m), .cn(ck), .cp(ckn));
    // 主级反相器
    not (mn, m);

    // 从级传输门（clk=1时透明）
    cmos_tg tg3 (.a(m), .b(q), .cn(ck), .cp(ckn));
    // 从级反馈保持
    cmos_tg tg4 (.a(qn), .b(q), .cn(ckn), .cp(ck));
    // 从级反相器
    not (qn, q);
endmodule

亚稳态

当触发器的数据输入在时钟沿附近发生变化（违反setup/hold时间窗口）时，触发器可能进入一种亚稳态（Metastability）——输出既不是0也不是1，而是一个"中间电压"，并且可能在这个中间状态徘徊很长时间（理论上可以是无限长）才最终稳定到0或1。

亚稳态无法完全消除，只能通过同步器（通常由两个或三个级联的触发器组成）来降低发生的概率——这个概率就是MTBF（Mean Time Between Failures，平均故障间隔时间）。对于高可靠性设计（如汽车、航天），MTBF需要达到数百年甚至更长。

真实的亚稳态故障案例：2019年某公司的一款AI推理芯片在客户量产导入阶段，发现在某些特定数据模式下，芯片会随机地产生错误的推理结果，概率约百万分之一。团队花了六周才定位到问题：芯片内部有一个异步FIFO的读指针同步器只用了单级触发器做跨时钟域同步，MTBF在设计阶段被认为"足够大"。但在实际使用中，由于两个时钟域的频率关系恰好形成了一个接近整数比（119.9MHz vs 60MHz）的拍频，导致同步器的失效概率比预期高了约三个数量级。修复方案是多加一级同步触发器。这个故事的教训是：跨时钟域设计不能"靠感觉"，需要严谨地计算MTBF，特别是两个时钟频率接近整数比时，亚稳态概率会急剧升高。

4.5 延迟与功耗

传播延迟模型

CMOS门的延迟主要来源于对负载电容的充放电。一个简单的RC模型：

tp ≈ 0.69 · Req · CL

其中Req是晶体管的等效导通电阻，CL是负载电容（包含扇出门的栅电容、扩散电容和互连线电容）。

影响延迟的工程因素：

  • 晶体管尺寸：增大W/L减小导通电阻，但增加栅电容（增大前级的负载）。
  • 供电电压VDD：降低VDD增加延迟（因为驱动电流减小），但功耗以V2关系降低——这就是动态电压频率调整（DVFS）的基本原理。
  • 温度：温度升高，迁移率下降，延迟增大——典型芯片在高温下比低温下慢20-30%。
  • 工艺角：SS（Slow-Slow）工艺角下，延迟比TT（Typical-Typical）大30-50%。

功耗组成

CMOS电路的功耗由三部分组成：

  • 动态功耗 Pdyn = α · CL · VDD² · f —— 这是CMOS功耗的主要来源。α是翻转活动因子（activity factor，平均每个时钟周期翻转的概率），CL是负载电容，f是时钟频率。从这个公式可以看出，降低电压带来的功耗收益是"平方级"的，这也是为什么低功耗设计的第一选择永远是降低供电电压。
  • 短路功耗 Psc：在输入翻转过程中，PMOS和NMOS会短暂同时导通，形成从VDD到GND的瞬间短路电流。通常占总功耗的10-20%。
  • 静态功耗 Pstatic = Ileak · VDD —— 由亚阈值漏电、栅漏电等组成。在先进工艺（7nm以下），静态功耗占总功耗的比例显著上升，甚至可能在待机状态下超过动态功耗。

4.6 本章小结

这一章我们从最基础的CMOS反相器开始，一路学习了组合逻辑门、传输门、时序单元和延迟/功耗模型。你现在已经理解了：CMOS数字电路的本质是用一对互补的NMOS/PMOS网络在VDD和GND之间"切换"输出；0和1不仅仅是抽象的逻辑值，它们是具体的电压状态，而保持这些状态需要在功耗、速度和噪声容限之间做权衡。
