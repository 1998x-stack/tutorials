第22章 文生视频前沿

大家好，我是你们的讲师。如果说 2022-2023 年是文生图的爆发期，那么 2024-2026 年就是文生视频的爆发期。从 OpenAI 在 2024 年 2 月发布的 Sora 引爆全球热度，到国产混元视频、可灵 3.0 的商业化落地，再到 2026 年 Seedance 2.5 实现 4K 30 秒视频生成——文生视频的技术演进速度远超所有人的预期。

这一章我们将深入前沿：Sora 的核心技术是什么？开源视频模型生态发展到什么程度了？视频生成的评估指标有哪些？以及站在 2026 年中的今天，视频生成面临的终极挑战是什么。

22.1 Sora 的核心创新

OpenAI 在 2024 年 2 月发布的 Sora 是文生视频领域的里程碑事件。Sora 的核心技术栈可以拆解为三个关键组件：

组件技术方案作用
视频压缩网络（Video Compressor）VAE 的 3D 扩展，将原始视频压缩到潜空间降低时空维度，使 DiT 可以处理长视频
时空 Patch Embedding将视频表示为时空 patch 序列（spacetime patches）统一不同分辨率、时长、宽高比的视频表示
DiT 骨干网络Scalable Diffusion Transformer with AdaLN在潜空间执行去噪/流匹配过程

Sora 最核心的洞察是：把视频当作"一堆时空 patch 的序列"。具体来说，Sora 的 Video Compressor 将原始视频（如 1920×1080×30fps×10s≈ 6.2 亿像素）压缩成低维的潜空间表示，然后在这个潜空间中将视频切分成 3D patch（时间×高度×宽度）。每个 patch 被展平成一个 token，加上时间位置编码和空间位置编码后，送入 DiT 进行处理。

# Sora 的时空 patch embedding 概念示意
class SpacetimePatchEmbed(nn.Module):
    """将视频转换为时空 patch token 序列"""

    def __init__(self, patch_size=(2, 2, 2), in_channels=4, embed_dim=1024):
        # patch_size = (t_patch, h_patch, w_patch)
        # 在时空三个维度上做 patch 化
        self.proj = nn.Conv3d(
            in_channels, embed_dim,
            kernel_size=patch_size, stride=patch_size
        )

    def forward(self, x):
        # x: (B, C, T, H, W) 潜空间视频
        x = self.proj(x)                     # → (B, D, T', H', W')
        x = x.flatten(2).transpose(1, 2)     # → (B, N_tokens, D)
        # 加上时间和空间位置编码
        x = x + self.temporal_pos_embed(x)
        x = x + self.spatial_pos_embed(x)
        return x

这个设计的精妙之处在于：通过 patch 化，Sora 可以处理任意时长、任意分辨率、任意宽高比的视频——只需要调整 patch 数量即可。这也让 Sora 具备了"原生"的多分辨率泛化能力。

🌟 Sora 的"遗产"：虽然 OpenAI 在 2025 年关停了 Sora 的独立服务（将其能力并入 GPT 生态和 Sora 2 代中），但 Sora 的技术范式——DiT + 时空 patch + 视频压缩网络——已经成为了整个视频生成领域的事实标准。几乎所有后续出现的视频模型都在沿用这个范式。

22.2 开源视频模型生态

Sora 发布之后，开源社区迅速跟进，涌现了一批高质量的视频生成模型。以下是截至 2026 年 6 月的主要代表性模型：

模型研发团队基础架构最大时长分辨率开源状态
Open-Sora 2.0ColossalAIDiT + 3D VAE~16 秒720p全面开源
HunyuanVideo（混元视频）腾讯DiT + Flow Matching~30 秒1080p开源权重
CogVideoX智谱 AI3D VAE + Transformer~10 秒720p开源
Wan2.1（万相）阿里DiT + Flow Matching~15 秒1080p开源
Seedance 2.5字节跳动改进 DiT + 4K 引擎30 秒4K (3840×2160)API 可用
Kling 3.0（可灵）快手3D VAE + DiT~60 秒1080pAPI 可用

HunyuanVideo 是开源模型中关注度最高的之一。它在 2024 年 12 月开源了完整的模型权重和推理代码，使用了 FLUX 类似的 MM-DiT + Rectified Flow 架构，并扩展到了视频领域。混元视频的核心创新包括：

  • 3D VAE：将视频压缩到时空潜空间（同时降采样时间和空间维度），压缩率比逐帧处理高 3-5 倍。
  • Separate Spatial-Temporal Attention：先在空间维度做 Self-Attention，再在时间维度做 Self-Attention，而非联合的 3D 注意力——计算量从 O(n²m²) 降到 O(n² + m²)。
  • Prompt 增强：使用大语言模型对用户输入进行描述扩充，生成更丰富的视频描述。
💡 提示：如果你想自己动手尝试文生视频，我推荐从 HunyuanVideo 入手。它的开源程度最高（权重 + 推理代码 + 训练代码均可用），社区资料最丰富，且在单卡 A100 上就能跑（通过梯度检查点和时序注意力切分）。运行命令很简单：python sample.py --prompt "你的描述"。

22.3 Sora 2 与 sCM 实时生成

2025 年底，OpenAI 发布了 Sora 2，它基于 sCM（Simplified Consistency Matching） 架构实现了实时视频生成。sCM 的核心思想是将 Flow Matching 的 ODE 求解步骤缩减到 1-2 步——它学习的是 Flow Matching 轨迹的"一致性映射"：

# sCM 的核心思路
# 传统 Flow Matching 需要 N 步求解 ODE：
# z_{t+Δt} = z_t + v_θ(z_t, t) * Δt        # N=20~50 次
#
# sCM 学习的是"一步映射"：
# z_1 = f_θ(z_0)  # 直接从噪声映射到数据
#
# 训练方法：通过一致性蒸馏 + 对抗损失
# 1. 用预训练 Flow Matching 模型生成配对数据
# 2. 训练学生网络 f_θ 学习从噪声到数据的直接映射
# 3. 加入 GAN 损失提高单步生成质量

Sora 2 以此实现了 1-2 步的实时视频生成——输入 prompt 后不到一秒就能看到结果。但 sCM 也有代价：单步生成的质量仍然不如多步 ODE 求解的结果，尤其是在细节一致性和运动流畅度上。

22.4 视频评估指标

视频生成的评估比图像复杂得多——不仅要评估单帧质量，还要评估时序一致性、运动自然度和文本-视频对齐。

指标全称测量对象说明
FVDFréchet Video Distance视频分布差异基于 I3D 特征，衡量真实视频和生成视频的分布距离。类似图像中的 FID。
VBenchVideo Benchmark综合质量包含 16 个维度的评价体系（色彩、运动、时序一致性等）。目前最全面的自动化评测。
CLIP ScoreCLIP Similarity文本-视频对齐计算视频帧和 prompt 的 CLIP 相似度均值。
人类偏好Human Preference主观质量人工评分，通常以 A/B Test 或五点量表进行。最为可靠但成本最高。
Flow Warp Error光流一致性误差时序连续性用光流估计视频相邻帧的像素偏移，计算一致性误差。衡量"帧与帧之间是否平滑"。
🌟 实战建议：如果你在做视频模型训练，我建议的评估流程是：训练中用 FVD + CLIP Score 做快速迭代验证（每 1000 步计算一次），训练完成后用 VBench 做全面评估，最终用人类偏好做上线前的最终确认。光流一致性是一个"被低估"的指标——很多模型单帧看起来很漂亮，但播放起来画面抖动严重，Flow Warp Error 能暴露这个问题。

22.5 2026 年最新进展

2026 年视频生成领域的几个里程碑：

  • Seedance 2.5（字节跳动）：实现了原生 4K 分辨率、30 秒长度的视频生成，且支持视频编辑功能（局部替换、风格转换、时长伸缩）。其背后的 Video DiT 采用了 3D RoPE 位置编码，让模型能在不同分辨率之间泛化。
  • Gemini Omni（Google）：多模态理解+生成统一模型，可以"看一眼图片/视频就能理解内容并生成新的视频"。标志着理解与生成的进一步融合。
  • Sora 2 的 sCM 实时生成：1 步生成 720p 视频，虽然细节质量不如多步推理，但在交互式场景（如实时直播、游戏 NPC 生成）中已有惊艳表现。
  • 可灵 3.0：实现"文生视频 + 图生视频 + 视频编辑 + 视频延展"的全链路覆盖，单次生成最长 60 秒。

22.6 视频生成的挑战

尽管取得了巨大进展，视频生成仍然面临着几个重大挑战：

挑战具体表现当前进展
长时序一致性10 秒以上的视频中，人物衣着、背景物品可能发生变化HunyuanVideo 通过 3D VAE 有效降低了长视频的 token 数量，但 30 秒以上仍不稳定
物理规律理解物体掉落方向不对、流体动画不自然、刚体碰撞物理不真实部分模型（如 Kling 3.0）引入物理先验，但泛化到未见过场景仍困难
推理成本生成 10 秒 1080p 视频在 A100 上需要 3-10 分钟sCM 和一致性蒸馏将时间缩短到秒级，但质量有折损
运动轨迹控制难以精确控制视频中物体的移动路径类似图像的 ControlNet 方案（如 CameraCtrl、MotionCtrl）正在发展中
多模态对齐同时理解文字、画面、音频、字幕的联合生成Gemini Omni 和 Emu3 在做统一建模，但音频-视频同步仍是难点
⚠️ 真实踩坑——视频生成的"幻觉"问题：2025 年我参与了一个用文生视频做产品广告的项目。我们使用 HunyuanVideo 生成一段 15 秒的产品展示视频，prompt 写了"一个白色陶瓷杯放在木桌上，咖啡冒着热气"。生成的每一帧都非常精美，但播到第 8 秒时，杯子突然变成了玻璃材质，第 12 秒又变回陶瓷。这个"材质漂移"现象在长视频中非常普遍——模型没有"保持物体一致性"的约束机制，每一帧都是独立生成的（即使有时序注意力），积累的误差会导致物体外观的渐变形变。目前最有效的缓解方案是 Cross-Frame Attention：让每一帧的 Self-Attention 层也看到第一帧的特征，从而维持外观一致性。这个技巧在大多数开源模型中都已经内置，但如果你自己在训练模型，一定要加上。

视频生成是一个技术还在快速迭代的领域。如果说图像生成在 2023-2024 年已经达到了"实用级"质量，那么视频生成在 2026 年的今天正处于"从实验结果到产品质量"的关键跨越期。这个领域还有大量未解决的问题——每一个问题都是研究机会。
