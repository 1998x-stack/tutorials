第11章 复合求积与 Romberg 外推

大家好，我是你们的讲师。上一章我们学了梯形公式和 Simpson 公式。大家可能会想：这两个公式的误差项里都有 $(b-a)$ 的高次幂，如果区间很大，误差岂不是很大？确实如此。那怎么办呢？答案很朴素——分而治之。把大区间切成很多小区间，在每个小区间上用简单公式，这就是复合求积的思想。

说到这个我想起一个有趣的类比。你去测量一段弯曲山路的总长度，如果用一把直尺一次性量整个路段，误差会大得离谱。但如果你把路分成很多小段，每段用直尺量，把结果加起来，精度就高多了。这就是复合求积的直觉。有趣的是，我们还可以"精确估计"直尺测量产生的误差模式，然后通过外推计算出更准确的结果——这就是 Romberg 方法的核心。

11.1 复合求积公式

将区间 $[a,b]$ 均匀分为 $n$ 个子区间，步长 $h = (b-a)/n$，节点 $x_i = a + ih$，$i = 0,1,\dots,n$。

复合梯形公式：在每个子区间 $[x_i, x_{i+1}]$ 上应用梯形公式后求和：

T_n = h · [½ f(x_0) + f(x_1) + f(x_2) + ... + f(x_{n-1}) + ½ f(x_n)]

误差为 $O(h^2)$，即 $O(1/n^2)$。这意味着子区间数翻倍时，误差大约缩小为原来的 1/4。

复合 Simpson 公式：在每个子区间 $[x_i, x_{i+1}]$ 上应用 Simpson 公式（需要 $n$ 为偶数）：

S_n = h/3 · [f(x_0) + 4f(x_1) + 2f(x_2) + 4f(x_3) + ... + 2f(x_{n-2}) + 4f(x_{n-1}) + f(x_n)]

误差为 $O(h^4)$，即 $O(1/n^4)$。这是复合梯形的大幅改进。

方法步长 $h$误差阶每减半步长误差缩小适用场景
复合梯形$O(1/n)$$O(h^2)$$\times 1/4$低精度要求、光滑性一般的函数
复合 Simpson$O(1/n)$$O(h^4)$$\times 1/16$中等精度、被积函数光滑
复合 Boole$O(1/n)$$O(h^6)$$\times 1/64$高精度、被积函数非常光滑

11.2 步长选择与自适应策略

在实践中我们面临一个问题：$n$ 应该取多大？如果 $n$ 太小精度不够，$n$ 太大会浪费计算资源。

一个实用的方法是误差的事后估计：分别用 $n$ 和 $2n$ 个子区间计算积分值，比较二者的差异。如果差异小于设定容差，就认为结果收敛了。

def adaptive_trapezoidal(f, a, b, tol=1e-6, max_n=1_000_000):
    """自适应步长复合梯形公式"""
    n = 1
    T_old = (b - a) * (f(a) + f(b)) / 2

    for k in range(1, 30):  # 最多迭代 30 次
        n = 2 * n
        h = (b - a) / n

        # 只计算新增节点（之前奇偶位置的节点已经算过了）
        x = a + h
        sum_new = 0.0
        for i in range(1, n, 2):
            sum_new += f(a + i * h)

        T_new = T_old / 2 + h * sum_new

        if abs(T_new - T_old) < tol:
            print(f"收敛: n={n}, 积分值={T_new:.10f}")
            return T_new
        T_old = T_new

    print("警告：未收敛到指定容差")
    return T_old

自适应求积：上面的方法是对整个区间统一加密。更高效的做法是自适应求积——仅在误差大的区域加密。方法是：在每个子区间上分别用简单 Simpson 和复合 Simpson 计算，如果二者差异大，则递归地对半细分该子区间。这样在函数变化平缓的地方少算点，变化剧烈的地方多算点。Python 的 scipy.integrate.quad 就采用了类似的自适应策略。

11.3 Richardson 外推

Richardson 外推是数值分析中一个非常优雅的想法。它的核心观察是：对于复合梯形公式，误差具有 Euler-Maclaurin 展开形式：

T(h) = I + c₁h² + c₂h⁴ + c₃h⁶ + ...

注意！展开中只有 $h$ 的偶次幂，没有奇次幂。这意味着如果我们用步长 $h$ 和 $h/2$ 分别计算，然后做一个线性组合消去 $h^2$ 项，就可以得到 $O(h^4)$ 的精度：

T_new = (4 · T(h/2) - T(h)) / 3

这个组合恰好就是复合 Simpson 公式！也就是说，Simpson 公式可以理解为梯形公式的 Richardson 外推。

Euler-Maclaurin 展开的威力：对于周期函数在完整周期上的积分，Euler-Maclaurin 展开中的所有项都消失（因为端点处的导数相等），梯形公式的误差衰减比任何幂次都快——这就是为什么对于周期函数，最简单的复合梯形公式反而是最好的选择！这个结论在计算物理中有着重要的应用。

11.4 Romberg 算法

Richardson 外推可以重复进行。如果我们消去了 $h^2$ 项，剩下的展开以 $h^4$ 开头，我们可以再外推一次消去 $h^4$ 项，得到 $O(h^6)$ 的精度。依此类推——这就是 Romberg 算法。

Romberg 算法建立一个三角表格 $R_{k,j}$：

R_{k,0} = T_{2^k} (复合梯形，步长 h = (b-a)/2^k)
R_{k,j} = (4^j · R_{k,j-1} - R_{k-1,j-1}) / (4^j - 1),  j ≥ 1

Python 实现：

import numpy as np

def romberg(f, a, b, max_k=10):
    """Romberg 积分"""
    R = np.zeros((max_k + 1, max_k + 1))

    # 第 0 列：复合梯形序列
    R[0, 0] = (b - a) * (f(a) + f(b)) / 2

    for k in range(1, max_k + 1):
        n = 2 ** k
        h = (b - a) / n

        # 复合梯形递推（只需计算新增节点）
        sum_new = 0.0
        for i in range(1, n, 2):
            sum_new += f(a + i * h)
        R[k, 0] = 0.5 * R[k-1, 0] + h * sum_new

        # Richardson 外推
        for j in range(1, k + 1):
            factor = 4 ** j
            R[k, j] = (factor * R[k, j-1] - R[k-1, j-1]) / (factor - 1)

    return R

# 测试：∫_0^π sin(x) dx = 2.0
f = np.sin
R = romberg(f, 0, np.pi, max_k=6)

# 打印 Romberg 表格
print("Romberg 积分表 (∫_0^π sin(x) dx = 2.0):")
print(f"{'k':>3} {'h':>10} {'R(k,0)':>14} {'R(k,1)':>14} {'R(k,2)':>14} {'R(k,3)':>14}")
for k in range(7):
    h = np.pi / (2**k)
    row = [f"{k:3d}", f"{h:.6f}"]
    for j in range(k+1):
        row.append(f"{R[k,j]:.12f}")
    print(" ".join(row))

print(f"\n最佳估计 (R(6,6)): {R[6,6]:.15f}")
print(f"真实误差: {abs(R[6,6] - 2.0):.2e}")

Romberg 的局限：Romberg 方法要求被积函数足够光滑，这样 Euler-Maclaurin 展开才有效。如果被积函数有奇点或不连续，外推可能失效甚至使结果更差。我见过一位同学用 Romberg 积分计算 $\int_0^1 \sqrt{x} \, dx$——$f(x) = \sqrt{x}$ 在 $x=0$ 处的导数无穷大，Euler-Maclaurin 展开不成立，Romberg 表格的对角线根本不收敛！遇到这种情况，需要先做变量替换消除奇点，或者使用专门处理奇点的求积方法。

本章我们学习了复合求积这个"分而治之"的思想，以及如何通过 Richardson 外推从低精度公式构造高精度公式。Romberg 算法将这两者结合，自动生成了一个精度递升的求积序列。下一章我们将学习数值积分中的"天花板"——Gauss 求积公式，它将节点和权重都作为自由变量来选择，实现了用最少的函数求值次数达到最高的代数精度。敬请期待！
