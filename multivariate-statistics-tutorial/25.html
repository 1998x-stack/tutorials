一个真实的商业场景

大家好，欢迎来到第25章！前24章我们搭建了完整的理论框架和技术工具栈。现在是时候把它们串联起来了——从真实问题出发，完整地走一遍数据分析流程。

想象这样一个场景：你是一家消费电子产品公司的数据分析师。公司有一个积累了两年多的会员数据库，包含了大约1000名会员的50个维度的信息——从基本的人口统计信息（年龄、收入、教育程度），到购买行为（年购买频次、客单价、品类偏好），再到心理特征（品牌忠诚度、价格敏感度、创新接受度等态度量表评分）。CEO给你一个任务："帮我把我们的用户分成几个有意义的群体，让我们知道应该给谁推高端产品、给谁推性价比产品、给谁做忠诚度维护。"

这就是典型的市场细分 (market segmentation)任务。在这个案例中，我们将综合运用PCA、聚类分析、判别分析和典型相关分析四类方法，来解决同一个商业问题。

本案路线图：数据预处理 → PCA降维 → K-means聚类 → LDA判别验证 → 群体画像 → CCA深度探索 → 商业建议。每一步的输出是下一步的输入，形成完整的数据分析流水线。

数据预处理

现实世界的数据从来不是干净整洁的。在开始正式分析之前，我们需要先做数据清洗：

import pandas as pd
import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.impute import SimpleImputer

# 假设我们已经加载了数据
df = pd.read_csv("customer_survey.csv")
print(f"原始数据：{df.shape[0]}行 x {df.shape[1]}列")

# ---- 数据清洗四步走 ----

# Step 1: 缺失值处理
# 对数值变量用中位数填充，对分类变量用众数填充
num_cols = df.select_dtypes(include=[np.number]).columns
cat_cols = df.select_dtypes(include=['object']).columns

imputer_num = SimpleImputer(strategy='median')
df[num_cols] = imputer_num.fit_transform(df[num_cols])

imputer_cat = SimpleImputer(strategy='most_frequent')
df[cat_cols] = imputer_cat.fit_transform(df[cat_cols])

print(f"缺失值处理后：{df.isnull().sum().sum()} 个缺失值剩余")

# Step 2: 异常值检测（使用IQR方法）
Q1 = df[num_cols].quantile(0.25)
Q3 = df[num_cols].quantile(0.75)
IQR = Q3 - Q1
outlier_mask = ((df[num_cols] < (Q1 - 1.5 * IQR)) |
                (df[num_cols] > (Q3 + 1.5 * IQR))).any(axis=1)
print(f"检测到 {outlier_mask.sum()} 个含有异常值的样本")

# 移除异常值（也可以选择winsorize处理，这里简单删除）
df_clean = df[~outlier_mask].copy()
print(f"清洗后数据：{df_clean.shape[0]}行 x {df_clean.shape[1]}列")

# Step 3: 标准化
cols_for_analysis = [c for c in num_cols if c != 'customer_id']
scaler = StandardScaler()
X_scaled = scaler.fit_transform(df_clean[cols_for_analysis])

# Step 4: 相关性分析 — 初步了解变量结构
corr_matrix = pd.DataFrame(X_scaled, columns=cols_for_analysis).corr()
# 找出高度相关的变量对（|r| > 0.8）
high_corr = np.where(np.abs(corr_matrix) > 0.8)
high_corr_pairs = [(corr_matrix.index[i], corr_matrix.columns[j])
                   for i, j in zip(*high_corr) if i < j]
print(f"发现 {len(high_corr_pairs)} 对高度相关变量")
变量类别示例变量数量
人口统计年龄、收入、教育年限、家庭人数8
购买行为年购买频次、客单价、总消费额、回购率12
态度量表品牌忠诚度(1-7)、价格敏感度(1-7)、创新接受度(1-7)20
产品偏好对5类产品的偏好评分(1-5)5
渠道偏好线上/线下购买比例、App使用频率5

PCA降维

我们有50个原始变量，太多了。直接做聚类的话，50维空间中的距离计算会受到"维度诅咒"的影响，聚类结果可能不稳定。所以我们先用PCA做降维。

from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

# PCA降维，不限制主成分数量
pca = PCA()
X_pca = pca.fit_transform(X_scaled)

# 绘制碎石图，确定保留多少个主成分
ev_ratio = pca.explained_variance_ratio_
cumulative = np.cumsum(ev_ratio)

plt.figure(figsize=(10, 4))
plt.subplot(1, 2, 1)
plt.bar(range(1, len(ev_ratio)+1), ev_ratio)
plt.axhline(y=0.05, color='r', linestyle='--', alpha=0.5)
plt.xlabel('主成分')
plt.ylabel('解释方差比例')
plt.title('碎石图')

plt.subplot(1, 2, 2)
plt.plot(range(1, len(cumulative)+1), cumulative, 'bo-')
plt.axhline(y=0.80, color='r', linestyle='--', alpha=0.5)
plt.xlabel('主成分数量')
plt.ylabel('累计解释方差比例')
plt.title('累计方差图')
plt.show()

# 选择累计解释方差 > 80% 的主成分
n_components = np.argmax(cumulative > 0.80) + 1
print(f"保留 {n_components} 个主成分，累计解释 {cumulative[n_components-1]:.1%} 的方差")
# 这里通常得到8个主成分

# 用选定的主成分数量重新fit
pca_final = PCA(n_components=n_components)
X_pca_final = pca_final.fit_transform(X_scaled)

# 载荷分析：解释每个主成分的含义
loadings = pd.DataFrame(
    pca_final.components_.T,
    columns=[f'PC{i+1}' for i in range(n_components)],
    index=cols_for_analysis
)

# 找出每个主成分上载荷绝对值最大的5个变量
for i in range(n_components):
    top_vars = loadings.iloc[:, i].abs().sort_values(ascending=False).head(5)
    print(f"PC{i+1}: {', '.join(top_vars.index.tolist())}")

从载荷分析中，我们可以解读每个主成分的业务含义。例如，PC1可能主要由收入、客单价和品牌忠诚度驱动，我们可以将其解释为"购买力维度"；PC2可能主要由创新接受度和线上购买比例驱动，解释为"数字化接受度维度"。这种对主成分的业务解读，是连接数据分析和商业决策的关键桥梁。

聚类分群

降维之后，我们在主成分得分（PC scores）上做K-means聚类。之所以在主成分上做而不在原始变量上做，是因为主成分已经去除了噪声、浓缩了信息，聚类效果更稳健。

from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

# 用轮廓系数确定最佳K值
sil_scores = []
K_range = range(2, 10)
for k in K_range:
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    labels = km.fit_predict(X_pca_final)
    sil_scores.append(silhouette_score(X_pca_final, labels))

plt.plot(K_range, sil_scores, 'bo-')
plt.xlabel('K')
plt.ylabel('Silhouette Score')
plt.title('Silhouette Analysis for Optimal K')
plt.axvline(x=np.argmax(sil_scores) + 2, color='r', linestyle='--')
plt.show()

optimal_k = K_range[np.argmax(sil_scores)]
print(f"最佳K值: {optimal_k}")
# 通常得到3-5个细分群体，这里假设得到4个

# 用最佳K值做最终聚类
kmeans = KMeans(n_clusters=optimal_k, random_state=42, n_init=10)
cluster_labels = kmeans.fit_predict(X_pca_final)
df_clean['Segment'] = cluster_labels

# 可视化：在主成分空间中查看分群
colors = ['#4C72B0', '#DD8452', '#55A868', '#C44E52']
plt.figure(figsize=(10, 7))
for seg in range(optimal_k):
    mask = cluster_labels == seg
    plt.scatter(X_pca_final[mask, 0], X_pca_final[mask, 1],
                c=colors[seg], label=f'Segment {seg+1}', alpha=0.6,
                edgecolors='w', linewidth=0.5)
plt.xlabel('PC1')
plt.ylabel('PC2')
plt.legend()
plt.title('Customer Segments in PC Space')
plt.grid(alpha=0.3)
plt.show()

判别验证

聚类结果出来了——4个用户群体。但我们需要验证：这4个群体是真的有区别，还是聚类算法强行分出来的？用LDA（线性判别分析）在原始变量上做验证是一个好方法：以聚类标签为因变量，以原始变量为自变量，训练分类器，看分类准确率有多高。

from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.model_selection import cross_val_score
from sklearn.metrics import classification_report, confusion_matrix

# 以原始变量（非PCA降维后的）预测聚类标签
X_original = df_clean[cols_for_analysis].values
y_segment = df_clean['Segment'].values

# LDA验证
lda_valid = LinearDiscriminantAnalysis()
cv_scores = cross_val_score(lda_valid, X_original, y_segment, cv=5)

print(f"LDA交叉验证准确率: {cv_scores.mean():.2%} +/- {cv_scores.std():.2%}")
# 如果准确率 > 85%，说明聚类分群在原始变量空间中是可区分的

# 如果准确率很高（> 95%），说明群体间差异非常清晰
# 如果准确率较低（< 75%），说明聚类分群可能不够稳定，需要重新考虑

一般来说，如果交叉验证准确率超过85%，我们就可以比较有信心地说"这四个群体确实有统计学上的显著差异"。如果低于70%，就需要重新审视——可能是K值选得不对，或者数据本身的信噪比太低。

细分画像

验证通过后，最重要的步骤来了：给每个群体起名字、画画像。这一步需要你同时理解统计输出和业务背景。

# 计算各群体在关键变量上的均值
profile_cols = ['age', 'income', 'annual_spend', 'avg_order_value',
                'purchase_frequency', 'brand_loyalty', 'price_sensitivity',
                'innovation_readiness', 'online_ratio']
profile_df = df_clean.groupby('Segment')[profile_cols].mean()
print(profile_df.round(2))

# 雷达图比较各群体
from math import pi
categories = profile_cols[:6]  # 选取6个关键维度做雷达图
angles = [n / float(len(categories)) * 2 * pi for n in range(len(categories))]
angles += angles[:1]

fig, ax = plt.subplots(figsize=(8, 8), subplot_kw={'projection': 'polar'})
for seg in range(optimal_k):

CCA探索：群体特征与产品偏好的关联

最后，我们做个锦上添花的分析：用典型相关分析来探索"群体特征变量"和"产品偏好变量"两组变量之间的关联。

from sklearn.cross_decomposition import CCA

# X组：群体特征变量（收入、年龄、忠诚度等）
X_group = df_clean[['income', 'age', 'brand_loyalty',
                     'price_sensitivity', 'innovation_readiness']].values
# Y组：产品偏好变量（5类产品的评分）
Y_group = df_clean[[c for c in df_clean.columns
                    if c.startswith('pref_')]].values

# CCA
cca = CCA(n_components=2)
cca.fit(X_group, Y_group)
X_c, Y_c = cca.transform(X_group, Y_group)

# 典型相关系数
corr1 = np.corrcoef(X_c[:, 0], Y_c[:, 0])[0, 1]
corr2 = np.corrcoef(X_c[:, 1], Y_c[:, 1])[0, 1]
print(f"Canonical correlations: {corr1:.3f}, {corr2:.3f}")

# 解读：第一典型变量揭示的特征组合与产品偏好组合
print("X weights (特征->典型变量):")
print(pd.DataFrame(cca.x_weights_,
      index=['income','age','loyalty','price_sens','innovation'],
      columns=['CV1','CV2']))
print("Y weights (偏好->典型变量):")
print(pd.DataFrame(cca.y_weights_,
      index=[f'pref_{i}' for i in range(5)],
      columns=['CV1','CV2']))

CCA结果通常能揭示出一些有趣的模式。例如，第一对典型变量可能揭示了"高收入+高忠诚度"的群体倾向于偏好"高端产品线+品牌周边产品"，而"高创新接受度+年轻"的群体倾向于偏好"新品类+智能产品"。这些洞察可以直接转化为产品推荐策略。

从分析到商业建议

现在，所有的分析结果都摆在了面前。是时候把它们翻译成CEO能听懂的语言了：

1. 高价值忠诚用户（22%）：他们是公司的利润核心。营销策略应聚焦于维护——VIP专属服务、会员积分加码、新产品优先体验权。这个群体的流失成本最高，所以建议投入最多的资源来维护。

2. 性价比追求者（31%）：他们占比最大，但利润贡献一般。策略应该是提升——通过捆绑销售、满减优惠来提高客单价。同时避免在品牌广告上对这个群体过度投入，因为他们的驱动因素主要是价格。

3. 潮流尝鲜者（27%）：他们是最有潜力的增长群体。策略是激活——推送新品信息、限时折扣、社交媒体种草。他们是你未来高端用户的"预备役"。

4. 传统稳健型（20%）：他们对线上渠道接受度低，但线下客单价高。策略是转化——通过线下体验活动引导他们尝试线上服务，逐步建立数字信任。

关键洞见：不需要对每个群体做同样的事情。资源是有限的，把营销预算投到ROI最高的地方——对高价值用户做维护（降低流失率），对潮流用户做激活（提升消费频次），对性价比用户做提升（提高客单价），对传统用户做转化（降低服务成本）。好的市场细分，本质上是资源的最优配置。

本章小结与衔接

这一章我们完整地走了一个端到端的市场细分项目：从数据预处理开始，用PCA将50个变量降维到8个主成分，在主成分得分上做K-means聚类得到4个用户群体，用LDA验证了群体的可分性，通过群体画像给每个细分起了名字并制定了差异化的营销策略，最后用CCA探索了群体特征与产品偏好的关联。这不是一个虚构的案例——全球各地的数据分析师每天都在用这套流程解决类似的商业问题。

下一章，我们将转向一个完全不同的挑战：当数据的维度变得非常高（成千上万甚至上百万）时，我们之前学的方法会怎样失效？有哪些应对策略？我们会讨论维度诅咒、t-SNE、UMAP和Autoencoder等高维数据分析工具。
