回溯三部曲 — 路径 + 选择列表 + 撤销选择

大家好，我是你们的讲师。欢迎来到第十章——回溯算法。如果说二叉树递归是"简单的递归"，那么回溯就是"带状态的递归"。它解决的是这样一类问题：在多个决策步骤中，尝试每一种可能性，走不通就回头。典型的应用场景包括排列、组合、子集、N 皇后、数独等。

回溯算法有一个极其强大的通用框架，我称之为"回溯三部曲"：

def backtrack(路径, 选择列表):
    if 满足结束条件:
        结果集.add(路径)
        return

    for 选择 in 选择列表:
        做选择                    # 将选择加入路径
        backtrack(路径, 选择列表)  # 递归
        撤销选择                  # 从路径中移除选择

就是这样！排列、组合、子集、N皇后、数独——所有的回溯问题都可以秒套这个框架。区别只在于：选择列表的定义和剪枝条件。

框架力量：如果你看懂了上面的代码，你就掌握了回溯算法的"心法"。剩下的工作只是根据具体问题，定义清楚什么是"路径"、什么是"选择列表"、什么是"结束条件"。这就是为什么很多 LeetCode 题解区的大佬，写回溯题就像在填空题一样——框架固定，填逻辑就行。

排列 / 组合 / 子集

排列、组合、子集是回溯算法最经典的应用场景，也是 LeetCode 面试中出现频率极高的题目。

全排列（LeetCode 46）

给定一个不重复的数组，返回所有可能的排列。使用 used 数组标记已使用的元素：

def permute(nums):
    res = []
    used = [False] * len(nums)

    def backtrack(path):
        if len(path) == len(nums):
            res.append(path.copy())
            return
        for i in range(len(nums)):
            if used[i]:
                continue
            used[i] = True
            path.append(nums[i])
            backtrack(path)
            path.pop()
            used[i] = False

    backtrack([])
    return res

注意 path.copy()——Python 的列表是引用类型，如果你直接 res.append(path)，后续的 path.pop() 会把你已经加入结果的路径也改掉。这个坑我见过不下十个人踩进去。

组合（LeetCode 77）

组合与排列最大的区别是：组合不关心顺序，[1,2] 和 [2,1] 是同一个组合。因此我们需要用一个 start 参数来保证"只往后看，不往前看"：

def combine(n, k):
    res = []

    def backtrack(start, path):
        if len(path) == k:
            res.append(path.copy())
            return
        for i in range(start, n + 1):
            path.append(i)
            backtrack(i + 1, path)
            path.pop()

    backtrack(1, [])
    return res

start = i + 1 这一行就是"组合"的精髓——选择了 i 之后，后续只能从 i+1 开始选。这样 [1,2] 和 [2,1] 就不会同时出现。

子集（LeetCode 78）

子集问题和组合的区别在于：所有长度的组合都是子集。所以不需要等到长度达到 k 才加入结果，每一步的 path 都是合法子集。

def subsets(nums):
    res = []

    def backtrack(start, path):
        res.append(path.copy())   # 每一步都加入结果
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()

    backtrack(0, [])
    return res

去重技巧（40 / 47 / 90）

当输入数组包含重复元素时，题目会升级。核心技巧：先排序，然后在同一层递归中跳过重复值。

# 子集 II（LeetCode 90）——包含重复元素的子集
def subsets_with_dup(nums):
    nums.sort()
    res = []

    def backtrack(start, path):
        res.append(path.copy())
        for i in range(start, len(nums)):
            if i > start and nums[i] == nums[i - 1]:
                continue        # 同一层去重
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()

    backtrack(0, [])
    return res

去重的关键理解：i > start 表示"在同一层递归中，当前元素和前一个元素相同且前一个元素已经被尝试过了，跳过"。这个条件保证了同一层不出现重复选择，但不同层允许重复值出现（因为不同层意味着不同的位置/深度）。

去重心法：排序 + used 数组用于排列的去重（如 47），排序 + start 参数用于组合/子集的去重（如 40、90）。两者的区别在于——排列需要 used 数组记录全局使用状态，而组合/子集只需要 start 控制方向。

棋盘问题

棋盘类问题是回溯算法最漂亮的展示——搜索空间大、约束多、剪枝效果显著。

N 皇后（LeetCode 51）

N 皇后问题：在 N×N 棋盘上放置 N 个皇后，使得任意两个皇后不在同一行、同一列、同一对角线上。

def solve_n_queens(n):
    cols, diag1, diag2 = set(), set(), set()
    res = []
    board = [["."] * n for _ in range(n)]

    def backtrack(row):
        if row == n:
            res.append(["".join(r) for r in board])
            return
        for col in range(n):
            if col in cols or (row - col) in diag1 or (row + col) in diag2:
                continue
            # 做选择
            board[row][col] = "Q"
            cols.add(col)
            diag1.add(row - col)    # 主对角线：row - col 为常数
            diag2.add(row + col)    # 副对角线：row + col 为常数

            backtrack(row + 1)

            # 撤销选择
            board[row][col] = "."
            cols.remove(col)
            diag1.remove(row - col)
            diag2.remove(row + col)

    backtrack(0)
    return res

这道题的美妙之处在于对角线的表示：主对角线上 row - col 为常数，副对角线上 row + col 为常数。下次面试官问 N 皇后，你直接画出棋盘，写下这两个公式，面试官就知道你来真的了。

解数独（LeetCode 37）

数独是 N 皇后之后另一个经典的"填空"类回溯问题。解数独的约束更复杂：行、列、3×3 宫格内数字都不能重复。搜索时，遍历空格子的位置，依次尝试 1~9。

数独的一个常见优化是 MRV（Minimum Remaining Values）启发式——每次选择候选数字最少的格子开始填，这样可以大幅减少回溯次数。虽然 LeetCode 上不优化也能过，但竞赛中 MRV 几乎是标配。

潜在风险：N 皇后和数独的搜索空间都很大。N=8 时还好，但 N=20 的皇后问题不加剪枝的话，搜索树的大小是 20! ≈ 2.4×10¹⁸——这辈子都等不到结果。所以剪枝不是"优化"，而是必要条件。在 N 皇后中，用集合记录列和对角线让检查时间降到 O(1)；在数独中，预先计算每行/每列/每宫格的可选数字。

剪枝优化

剪枝是回溯算法性能的关键。我把剪枝策略分为三种：

策略 1：可行性剪枝

当前选择已经不可能得到合法解时，提前终止。比如 N 皇后中，cols 集合保证了两皇后不在同一列。

策略 2：最优性剪枝（分支限界）

当前路径即使走到终点，也不可能优于已有的最优解。这种策略在"最优解"类问题中非常有效，比如旅行商问题（TSP）。

策略 3：重复性剪枝（去重）

如前所述，排序后在同一层跳过重复元素。这种剪枝不改变算法的空间复杂度，但能避免结果集出现大量重复。

问题类型 代表题目 选择列表 剪枝方式 复杂度（结果数）
全排列 46 未使用的元素 used 数组 n!
组合 77 start 之后的元素 start 递增 C(n,k)
子集 78 start 之后的元素 start 递增 2ⁿ
去重排列 47 未使用的元素 used + 排序去重 n! / 重复数

踩坑与总结

坑 1：引用传值导致的路径污染

这是回溯算法中最常见、最隐蔽的 bug。Python 函数参数传递的是引用——你把 path 传给递归函数，递归里修改了 path，外层也会受影响。所以加入结果时一定要 path.copy()（或 path[:]、list(path)）。

我记忆最深的一次踩坑：写 LeetCode 46 全排列时，忘记深度拷贝路径，运行发现结果是一个空列表的列表。当时盯着代码看了十分钟，觉得"逻辑完全正确啊"。最终是给每个结果加了 print，才发现所有结果都是同一个对象的引用——path.pop() 执行完毕后，所有"已保存"的结果也被清空了。

坑 2：剪枝条件遗漏

去重剪枝中，if i > start and nums[i] == nums[i-1] 这个条件里，i > start 非常关键。新手容易写成 if i > 0——这会导致所有层级的递归都跳过和前一个元素相同的值，结果就是本该出现在不同位置的重复值也被剪掉了。

坑 3：回溯和递归的混淆

不是所有递归都是回溯。回溯的特点：有"撤销选择"这一步。如果你看到代码中有"路径记录 + 撤销"的模式，那就是回溯；如果只是简单的"分治后合并结果"，那是普通的递归分治。区别这两个概念，对你理解算法本质非常有帮助。

本章心法：回溯算法本质上是在遍历一棵"决策树"。树的每个节点是一个状态，每条边是一个选择。backtrack 函数就是在这个树上做 DFS。理解了"路径 + 选择列表 + 撤销选择"这个三部曲，再难的回溯题也只是这三行的变体。

回溯算法的魅力在于，它用极其统一的框架解决了看似毫无关联的问题——排列、组合、子集、N皇后、数独，全都是同一棵树的不同遍历方式。下一章我们将进入贪心算法——一个"每一步都选最好的，最终得到全局最优"的思维范式。贪心和回溯形成鲜明对比：回溯是"尝试所有可能"，贪心是"只在当下做最优选择"。我们将详细剖析什么时候能用贪心、什么时候不能。下一章见！
