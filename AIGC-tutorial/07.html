第7章 生成质量评估指标

大家好，我是你们的讲师。前六章我们走过了概率基础→VAE→GAN→自回归的完整路线，每种模型都有自己的优势和局限。现在你可能会问一个非常实际的问题："我怎么知道我训练的生成模型好不好？"

这个问题比看起来要复杂得多。生成模型的评估是一个持续的开放难题——不像分类任务那样有单一的准确率指标。"好"可以指"看起来逼真"、"覆盖了所有模式"、"跟文本描述匹配"、"用户喜欢"——甚至这些维度之间还在互相冲突。这一章我们来逐个拆解当前最主流的评估指标，它们的数学含义、使用场景和常见陷阱。

7.1 为什么需要多个评估指标

生成质量是多维度的，没有任何一个指标能覆盖所有方面：

维度含义代表指标可能被忽略的另一个维度
保真度（Fidelity）单张图像是否逼真FID, IS多样性——可能只生成好看的但不覆盖所有类别
多样性（Diversity）是否覆盖所有模式Recall, Coverage保真度——覆盖全了但有些图质量差
图文对齐生成内容是否符合文本描述CLIP Score, BLIP-VQA美学质量——图文匹配但构图难看
人类偏好用户觉得好不好ELO 评分, A/B 胜率一致性——人工评估噪声很大
效率生成速度、计算成本推理时间, 参数量质量——速度快但质量差也没用
🌟 核心洞察：在 AI 生成领域有一个著名的"不可能三角"——质量、多样性、速度三者无法兼得。BigGAN 的 truncation trick 就是牺牲多样性换取保真度的经典例子。作为研究人员或工程师，你需要根据具体场景决定"哪个维度更重要"。

7.2 Inception Score (IS)

Inception Score 是 2016 年由 Salimans 等人提出的最早广泛使用的生成质量指标之一。它基于预训练的 Inception v3 网络，核心思想是：

  • 好的生成图像应该"清晰可辨"：Inception 网络对生成的图像应该输出高置信度的类别预测（条件标签分布 P(y|x) 的熵小）
  • 好的生成模型应该覆盖所有类别：整体上的类别分布应该均匀（边际分布 P(y) 的熵大）
IS = exp(E_x[KL(P(y|x) || P(y))])

其中：
  P(y|x) —— 给定图像 x 时 Inception 预测的类别概率分布
  P(y)   —— 所有生成图像的平均类别分布

直观理解：
  - 每个样本都被 Inception 以高置信度分到某个类别（P(y|x) 很尖）
  - 但所有样本平均来看分布在各类别中（P(y) 很均匀）
  → IS 值越大越好

局限性（非常严重）：

  • IS 只评估"有多少类物体被识别出来"，不评估内容是否合理——生成一张对抗性噪声也能得到高 IS
  • 高度依赖 Inception 的类别体系（1000 个 ImageNet 类别），不适用于域外数据（医学图像、抽象艺术）
  • 对多样性不敏感——只生成 1000 个"完美样本"（每个类别各一个）也能得到很高的 IS
  • IS 不能检测模式坍塌——模型只生成 1 类也能得高分（如果那 1 类的质量足够好）
⚠️ 一个真实的踩坑故事：2019 年我在发表一篇 GAN 论文时，审稿人质疑我的 IS 测试不够公正。我意识到我在计算 IS 时用了生成器和 Inception 在同一个数据集上训练的——这导致了严重的 bias：Inception 对生成器擅长的那几类特别"自信"。 正确的做法是：IS 和 Inception 的训练数据必须完全隔离，且生成样本必须和 Inception 的训练数据属于同一个域。审稿人逼着我用不同训练的 Inception 重新算了所有 IS——分值下降了 30%，但更有说服力了。 这个教训是：自动评估指标到处都是坑，理解每个指标的假设是避免学术造假（和论文被拒）的前提。

7.3 Frechet Inception Distance (FID)

FID（Heusel et al., 2017）是目前使用最广泛的生成质量指标，也是学术界和工业界的"标配"。它比 IS 更全面，因为它同时比较了真实图像和生成图像的分布距离。

计算步骤：

  1. 用 Inception v3 的倒数第二层（池化层，2048 维向量）提取所有真实图像和生成图像的特征
  2. 假设特征服从多维高斯分布，计算两个分布的均值和协方差
  3. 计算两个多维高斯分布之间的 Frechet 距离（也叫 Wasserstein-2 距离）
FID = ||μ_r - μ_g||² + Tr(Σ_r + Σ_g - 2(Σ_r Σ_g)^{½})

其中：
  μ_r, Σ_r —— 真实图像特征的均值和协方差
  μ_g, Σ_g —— 生成图像特征的均值和协方差
  Tr —— 矩阵的迹（对角线元素和）

FID 越小越好。经验阈值：
  - FID < 10：生成质量优秀
  - FID 10-30：生成质量良好
  - FID > 50：生成质量较差

FID 的优缺点总结：

优点缺点
同时评估保真度和多样性（两个分布的距离）高斯分布假设可能不成立（特征分布可能多模态）
对模式坍塌比 IS 更敏感需要大量样本才能稳定计算（通常需要 10K+）
不依赖 ImageNet 类别体系同样受限于 Inception 特征（不适用于所有域）
和人类判断的相关性比 IS 好小样本下方差大，结果不可信
💡 FID 使用实践：如果你要比较两个模型的 FID，确保：（1）使用相同数量的样本（推荐 10,000-50,000 张）；（2）使用相同的预处理（特别是图像缩放和中心裁剪）；（3）计算多次取平均值，报告标准差。我的经验是，同一个模型跑 3 次 FID，两次之间可能有 0.5-1.0 的波动——小于这个差值的变化不一定是模型改进带来的。

7.4 CLIP Score

FID 和 IS 都只评估"图像本身的质量"，但无法回答"生成的图像是否匹配文本描述"这个关键问题。随着文生图模型成为主流（2022 年后），图文对齐度成了最受关注的评估维度之一。

CLIP Score 的计算非常简单：

CLIP Score = E[cosine_similarity(CLIP_text(prompt), CLIP_image(image))]

其中：
  - CLIP_text 将文本 prompt 映射到联合嵌入空间
  - CLIP_image 将图像映射到同一联合嵌入空间
  - cosine_similarity 度量两者的对齐程度

分数范围通常在 0-1（或 0-100，取决于具体实现），越高越好。

CLIP Score 的优缺点：

  • 优点：计算快、不依赖参考图像（只需要 prompt）、通用性强
  • 缺点：对视觉细节不敏感（CLIP 是粗粒度的）——生成的图可能 CLIP Score 很高但实际构图很差
  • 缺点：CLIP 偏好"过度一致性"——如果一张图里放了太多和 prompt 相关的东西，CLIP Score 反而更高，但这可能牺牲了构图美感

在实际评测中，CLIP Score 通常和 FID 一起使用——FID 保证"像真实图像"，CLIP Score 保证"跟 prompt 匹配"。这正是 Artificial Analysis、T2I-CompBench 等评测平台的常见做法。

7.5 人类偏好评估

尽管自动指标越来越成熟，最终决定生成模型好坏的是人类用户的感受。人类评估的主流方法有两种：

A/B 测试（Pairwise Comparison）：给人类标注员同时展示两张图像和对应的 prompt，让他们选择"哪张更符合 prompt"或"哪张质量更好"。这是最可靠的方法，但成本高、速度慢。

ELO 评分系统：源自国际象棋的评分系统，现在被广泛用于 AI 模型评测。Artificial Analysis 是文生图领域最知名的 ELO 排行平台，其工作方式是：

  • 用户投票："模型 A 和模型 B，哪个更好？"
  • 胜者从败者那里获得 ELO 分数
  • 差距大时输赢分差大，差距小时分差小
  • 新模型初始 ELO 为 1000 或 1200
🌟 核心洞察：ELO 评分无法直接告诉你"这个模型有多好"——它只告诉你"这个模型相对其他模型有多好"。当你在论文中看到"Model X 达到了 1200 ELO"时，一定要问清楚"这个 ELO 是在什么数据集、什么对手模型、多少投票下算出来的"。不同的 ELO 榜单之间不具有可比性。

ImageReward / Pick-a-Pic 等评分模型：2023 年后出现了基于人类偏好训练的打分模型——它们试图用神经网络模拟人类评估。虽然不如真实人类准确，但可以在研发迭代中做快速粗略的评估。截至 2026 年，这类模型已经相当成熟，但仍然无法完全取代真实的人类评估。

7.6 指标对比总结与使用场景指南

每个指标都有最适合的场景和最不该用的场景。这里给出一个实战指南：

场景首要指标辅助指标注意事项
比较两个文生图模型FID + CLIP Score人类 A/B 测试确保 prompt 集合覆盖广泛
训练过程中监控进度FID（每 1K 步算一次）生成样本可视化不要只看 FID，肉眼观察同样重要
检测模式坍塌FID + Recall 指标IS生成 10K 样本，统计类别分布
评估图像-文本对齐CLIP ScoreBLIP-VQA准备细粒度的 prompt（包含对象、属性、关系）
论文实验 SOTA 对比FID + 人类评估IS, Precision/Recall必须使用官方或广泛认可的评测代码
产品级模型选择ELO 评分 + 用户留存FID, CLIP Score最终指标是用户付费率和活跃度
⚠️ 第二个踩坑故事：2023 年我在做一个图像增强项目时，模型在 FID 指标上取得了显著提升（从 35 降到了 22），我非常兴奋，觉得终于找到了正确的方案。结果我把前后生成的图给同事做盲测——同事无一例外地选择了"FID 更差"的旧版本！原来我优化的是 FID 中的"全局分布匹配"，但过度的平滑处理让图像失去了锐利的边缘。这个教训让我形成了铁律：任何自动指标提升，都必须配合至少一次人工盲测来验证。指标永远比人眼更容易被"欺骗"。
