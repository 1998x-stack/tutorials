第12章 MTP多Token预测

大家好！今天我们来聊一个训练目标的创新——MTP（Multi-Token Prediction，多 Token 预测）。

传统的大语言模型训练全部基于一个范式：Next-Token Prediction。给定前 t 个 token，预测第 t+1 个 token。如此简单，但效果却出奇地好。然而，DeepSeek-V3 提出了一个尖锐的问题：一次只学一个 token，是不是太慢了？

想象一下你在教一个学生学习写作文。你指着一句话"今天天气真好，我们一起去。"问他下一句是什么。他猜对了，你教下一个字。但如果你能一次让他预测"去公园/去海边/去爬山"三个未来的词呢？他是不是能更快地理解句子的上下文和句法结构？MTP 的核心思想就是这样——让模型在每个位置同时预测多个未来的 token。

但这里有一个技术难题：如何在不破坏因果链的前提下，并行预测多个未来 token？DeepSeek-V3 的 MTP 设计给出了一个优雅的答案。

12.1 Next-Token Prediction的局限

标准的语言模型训练目标很简单：给定序列 (t_1, t_2, ..., t_T)，最大化每个位置预测下一个 token 的对数概率之和。每个 token 只提供 1 个训练信号。这意味着数据效率存在天花板——无论你的数据多么丰富，每个 token 位置你只有一次学习机会。

更微妙的问题是，NTP 可能过于关注局部的 token 级模式，而忽略了全局的语义结构。例如，模型可能学会了"the"后面通常是"cat"这样的低阶共现模式，但对于"这句话的结构应该是主谓宾"这样的高阶语法模式，NTP 提供的学习信号是间接的。

学界一直有一些改良尝试：

  • Electra 用判别式预训练（判断 token 是否被替换），提供更密集的训练信号
  • Masked LM（BERT 风格）双向预测，但无法用于自回归生成
  • Prefix LM 部分自回归 + 部分双向，但实现复杂

但这些方案要么不适合 decoder-only 架构，要么带来了额外的推理复杂度。MTP 是第一个在保持标准自回归架构不变的同时，大幅增加训练信号密度的方案。

一个有趣的类比：传统的 NTP 训练就像你在学围棋时，每次只告诉你下一步的最佳落子位置。而 MTP 就像告诉你接下来三步的最佳位置。你不仅能学会"下一步怎么走"，还能开始理解"三步棋的组合策略"。当你的对手（模型）在预测下一步时，他实际上已经"思考"了后面几步的可能性——这是直觉推理能力的一种形式。

12.2 MTP核心思想

MTP 的结构非常巧妙。它没有改变主模型的架构，而是在主模型的基础上附加了 D 个额外的 MTP Module（每个对应一个未来的预测深度）：

MTP 架构概览（D=2 时，预测 next 2 token）：

输入: [t1, t2, t3, t4, ...]

主模型 (Next-Token Prediction):
  预测:  t2 | t3 | t4 | t5 | ...  (标准 NTP)

MTP Module 1 (Next 2 Token):
  输入: h^0_i (主模型第 i 层隐藏状态) + Emb(t_{i+1})
  预测: t3 | t4 | t5 | t6 | ...

MTP Module 2 (Next 3 Token):
  输入: h^1_i (MTP1 第 i 层隐藏状态) + Emb(t_{i+2})
  预测: t4 | t5 | t6 | t7 | ...

关键设计要点：

  • 主模型和所有 MTP Module 共享 embedding 层和 output head
  • 每个 MTP Module 都由一个独立的 Transformer block 处理
  • 每个 MTP Module 的输入 = 上一层的隐藏状态 + 对应位置的 token embedding
  • 因果链完整保持——预测第 i+k 个 token 时，只能看到前 i+k-1 个 token

在 DeepSeek-V3 中，D 被设为 2（即预测 Next 2 Token），MTP Loss 的权重 λ 在前 10T token 设为 0.3，后 4.8T token 降为 0.1。

12.3 独立输出头设计

每个 MTP Module 的核心是一个轻量级的 Transformer block，其结构与主模型的 block 类似但规模小得多。具体来说：

class MTPModule(nn.Module):
    def __init__(self, d_model, d_ff, n_heads):
        super().__init__()
        self.linear_proj = nn.Linear(2 * d_model, d_model)  # 融合输入
        self.transformer_block = TransformerBlock(d_model, d_ff, n_heads)
        # output head 与主模型共享

    def forward(self, h_prev, token_emb):
        # h_prev: 上一层在位置 i 的隐藏状态 [batch, seq, d]
        # token_emb: 目标 token_{i+k} 的 embedding [batch, seq, d]
        concat_input = torch.cat([RMSNorm(h_prev), RMSNorm(token_emb)], dim=-1)
        h_proj = self.linear_proj(concat_input)  # [batch, seq, d]
        h_out = self.transformer_block(h_proj)   # [batch, seq, d]
        return h_out

每个 MTP Module 的 Transformer block 是独立训练的（参数不共享）。这使得不同的预测深度可以发展出不同的"预测策略"——MTP Module 1 可能更关注句法和短程依赖，MTP Module 2 可能更关注语义和长程依赖。

12.4 训练信号倍增

MTP 最直接的效果就是训练信号密度的大幅提升。对于每个 token 位置，原本只有 1 个监督信号（预测下一个 token），现在变成了 D+1 个（主模型的 NTP + D 个 MTP 额外预测）。

在 D=2 的情况下，训练信号增加了 200%。这意味着：

  • 更高的数据效率：同样的训练数据量下，模型学到更多
  • 更好的表示学习：隐藏状态不仅要支持下一个 token 的预测，还要支持下下个和下下下个 token 的预测，迫使表示更加丰富和鲁棒
  • 推理能力提升：预测未来的 token 需要一定程度的"推理"——模型不能只看局部，而要理解全局意图
直觉解释：NTP 训练相当于让一个篮球运动员只练习"投篮"这一个动作。MTP 则让运动员同时练习"投篮→抢篮板→传球"三个连续动作。虽然比赛时他一次只做一个动作，但他在做第一个动作时，已经在潜意识里为后续动作做好了准备。这有点类似于人类认知中的"思维预演"。

12.5 推理零开销

MTP 有一个非常吸引人的特性：推理时完全不需要 MTP Module。

在训练完成后，MTP Module 被直接丢弃。推理时只用主模型做标准的下一个 token 自回归生成。这意味着 MTP 带来了训练阶段的全部好处，但推理阶段不增加任何计算量或参数。

不过，DeepSeek-V3 还发现了一个额外的好处：MTP Module 可以用于辅助投机解码（Speculative Decoding）。具体来说，MTP Module 1 的预测可以作为"候选 token"，主模型只做验证（draft-then-verify）。由于 MTP Module 1 的接受率高达 85-90%，这种方案可以将实际解码速度提升约 1.8 倍。虽然 DeepSeek-V3 的主要目标不是加速推理，但这确实是一个"买一送一"的惊喜。

对比维度 标准 NTP MTP (D=2)
每位置训练信号数 1 3
推理时参数增加 0 0（MTP Module 已丢弃）
训练计算量增加 — ~15%（2 个轻量 MTP Module）
可加速投机解码 否 是（1.8× 潜在加速）
模型质量影响 基线 多项 benchmark 提升 0.5-1.5%

12.6 效果提升

MTP 到底带来了多少实际收益？DeepSeek-V3 论文中报告了几个关键发现：

  • 推理能力加强：在 MATH 等数学推理 benchmark 上，MTP 带来了约 1-2% 的提升。模型在处理需要多步推理的问题时表现更好，可能因为 MTP 让模型学会了"想一步看三步"
  • 代码生成提升：在 HumanEval 和 MBPP 等代码 benchmark 上也有约 0.5-1.5% 的提升。代码生成天然需要前瞻性规划——写函数之前要想到函数体、返回值等
  • 长文本理解改善：在需要理解长上下文依赖的任务中，MTP 训练出的模型表现更好，可能因为多步预测迫使模型保持更长距离的语义一致性
一个"略为反直觉"的发现：你可能觉得 MTP 在深度 D=3 或更高时效果更好。但在 DeepSeek-V3 的实验中，D=2 就已经达到了收益的饱和点。更深的预测（D=3）带来的额外收益很小，但训练开销显著增加。这背后的原因可能是：语言中的长程依赖已经通过主模型的 NTP 间接学习了，MTP 的主要价值在"近未来"预测——这与机器翻译等任务中"预测未来 2-3 个词"能显著提升质量的经验是一致的。

MTP 是 DeepSeek-V3 在训练目标层面的核心创新。它的优雅之处在于：在几乎不增加推理成本的前提下，通过更丰富的训练信号提升了模型质量。这种"降本增效"的思路贯穿了 DeepSeek 的所有技术决策。
