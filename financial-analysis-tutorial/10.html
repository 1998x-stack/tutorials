第10章 DCF绝对估值法

大家好，我是你们的讲师。上一章我们说到了"估值的世界观"——内在价值与市场价格的区别、安全边际的重要性。现在，我们要深入到最核心的估值技术：DCF（Discounted Cash Flow，现金流折现模型）。

如果说估值是金融分析的"皇冠"，那么DCF就是皇冠上的"明珠"。它不仅是理论基础最严密的估值方法，也是几乎所有其他估值方法的思想源泉。无论是PE还是PB，最终都可以追溯到DCF的逻辑——只不过做了一些简化和替代。理解DCF，你就理解了估值的第一性原理。

但坦率地说，DCF在实战中的使用频率并不像它在课本中的篇幅那么高。原因很简单：它对假设太敏感了。稍微改变一下增长率或折现率，估值结果可以相差一倍。但这恰恰不是DCF的缺陷，而是它的价值——强迫你思考"什么假设才能支撑现在的价格"。即使你不用DCF来给出最终目标价，也应该用它来检验你的直觉判断是否经得起推敲。

10.1 DCF核心逻辑

DCF的逻辑简单到一句话就可以概括：一家公司的价值，等于它未来能够产生的所有自由现金流的现值之和。

为什么是"自由现金流"而不是"利润"？因为利润是会计概念，里面包含了很多非现金项目（折旧、摊销、应收账款等），而现金才是公司真正用来还债、分红、再投资的真金白银。很多利润丰厚的公司可能因为现金流断裂而倒闭——这就是"纸面富贵"的陷阱。

为什么是"现值"？因为今天的100块比一年后的100块更值钱。你可以把今天的100块存银行赚利息，或者投资到其他地方获取回报。所以未来的现金流需要按一定的折现率"打折"到今天的价值。

DCF的基本公式如下：

企业价值 = Σ[FCFt / (1 + WACC)t] + 终值 / (1 + WACC)n

其中：

  • FCFt = 第t年的自由现金流
  • WACC = 加权平均资本成本（折现率）
  • 终值（Terminal Value）= 预测期之后所有年份的现金流价值总和
  • n = 预测期的年数（通常为5-10年）
核心洞见：DCF模型中的所有复杂性——增长率的假设、资本支出的预测、营运资本的变化——最终都服务于一个目的：估算自由现金流。而DCF的"脆弱性"也因此而来——预测期越长，不确定性越大；但预测期太短，终值的占比又太高。这是一个无法完美解决的权衡。

10.2 自由现金流预测

DCF建模的第一步是预测自由现金流。这里我们需要先区分两个概念：FCFF（公司自由现金流）和FCFE（股权自由现金流）。

项目 FCFF（公司自由现金流） FCFE（股权自由现金流）
流向 所有资本提供者（股东+债权人） 仅股东
计算公式 EBIT×(1-t) + D&A - CapEx - ΔWC FCFF - 利息×(1-t) + 净新增借款
折现率 WACC 股权成本（CAPM中的Ke）
估值结果 企业价值（EV） 股权价值（直接对应市值）
常用场景 资本结构稳定或变化可预测的公司 高杠杆或资本结构剧烈变化的公司

在实践中，FCFF更常用——因为它是从经营层面出发，不受融资决策影响，更纯粹地反映公司的赚钱能力。

预测自由现金流通常采用两阶段模型：

  • 第一阶段（预测期，5-10年）：逐年预测详细的FCFF。这需要你对公司的收入增长率、毛利率、营业费用、资本支出、折旧摊销、营运资本变化等做出具体假设。
  • 第二阶段（终值期，永续）：假设公司进入稳定增长阶段，用一个固定的永续增长率g来估算所有后续年份的现金流价值。

为什么通常只做5-10年的详细预测？因为任何人都不可能精确预测一家公司10年以后的经营细节。但5-10年的预测期又足够长，可以让公司从当前的快速增长过渡到稳定的成熟状态。

10.3 WACC计算

WACC（加权平均资本成本）是DCF模型中最敏感、最关键的参数。它代表公司为筹集资金所需支付的平均成本，也是投资者要求的必要回报率。WACC的计算公式如下：

WACC = E/V × Ke + D/V × Kd × (1 - t)

其中：

  • E/V = 股权价值占总资本的比重
  • D/V = 债务价值占总资本的比重
  • Ke = 股权成本（通常用CAPM模型估算）
  • Kd = 债务成本（公司的借款利率）
  • t = 企业所得税税率

股权成本Ke的计算使用CAPM（资本资产定价模型）：

Ke = Rf + β × (Rm - Rf)

三个核心输入：

  • Rf（无风险利率）——通常用10年期国债收益率。在中国，2026年约为2.5%左右。
  • β（贝塔系数）——衡量个股相对于市场的波动性。β=1表示与市场同涨同跌，β>1表示波动大于市场（更激进），β<1表示波动小于市场（更稳健）。
  • (Rm - Rf)（股权风险溢价）——投资者承担股权风险所要求的额外回报。A股市场历史上约为5-7%。
实操提示：WACC的每个参数都可以找到合理的参考值，但不建议过度"调参"来拟合你想要的估值结果。一个常见的错误是：为了得到"合理"的估值，故意选择极端低或极端高的参数。专业做法是：先凭客观数据确定基准WACC，然后在敏感性分析中考察WACC变动的影响。
踩坑警示：WACC估算中有一种非常隐蔽的错误——"循环引用"。在某些复杂的财务模型中，WACC中的股权权重依赖于股权市值，而股权市值又是DCF的输出结果。这形成了一个循环：你需要股权市值来算WACC，但你需要WACC来算股权市值。破解方法是：用当前实际市值确定资本结构权重，再用DCF看是否存在显著偏离。如果偏离很大，再迭代一次即可——千万不要在Excel里设成循环计算，那会让你得到一个"自洽但完全错误"的数字。

10.4 终值估算与敏感性分析

终值（Terminal Value）在DCF中占据着举足轻重的地位。对于预测期为5年的模型，终值通常占企业总价值的70-80%；即使预测期拉长到10年，终值也往往占50%以上。这意味着DCF模型的结论在很大程度上是由终值驱动的。

终值的估算最常用的是戈登增长模型（Gordon Growth Model）：

终值 = FCFn+1 / (WACC - g)

其中g是永续增长率。这里有一个关键原则：g不能超过宏观经济的长期增长速度。如果名义GDP增速是5%，你假设一家公司永续增长6%，那就意味着它最终会吃掉整个经济体——这在数学上不可能。合理的g通常取2-4%（中国市场中长期名义GDP增速的大致范围）。

以下是用Python实现DCF模型并进行敏感性分析的示例：

import numpy as np
import pandas as pd

def dcf_two_stage(fcf_list, wacc, terminal_growth, debt, cash):
    """
    两阶段DCF模型
    fcf_list: 预测期每年的自由现金流（列表）
    wacc: 加权平均资本成本
    terminal_growth: 永续增长率
    debt: 净债务（用于从企业价值计算股权价值）
    cash: 现金及等价物
    """
    # 计算预测期现金流现值
    pv_fcf = sum(fcf / (1 + wacc)**(i+1)
                 for i, fcf in enumerate(fcf_list))
    # 终值（假设第n年之后永续增长）
    n = len(fcf_list)
    terminal_fcf = fcf_list[-1] * (1 + terminal_growth)
    terminal_value = terminal_fcf / (wacc - terminal_growth)
    pv_terminal = terminal_value / (1 + wacc)**n

    # 企业价值 = 预测期现值 + 终值现值
    enterprise_value = pv_fcf + pv_terminal
    # 股权价值 = 企业价值 - 净债务 + 现金
    equity_value = enterprise_value - debt + cash

    return equity_value, enterprise_value, pv_fcf, pv_terminal

# 示例：假设某公司未来5年FCF预测（单位：亿元）
fcf_projections = [10, 11.5, 13.2, 15.0, 17.0]
wacc = 0.10        # 10%
terminal_growth = 0.03  # 3%
debt, cash = 30, 15

eq_val, ev_val, pv_fcf, pv_term = dcf_two_stage(
    fcf_projections, wacc, terminal_growth, debt, cash
)
print(f"企业价值: {ev_val:.1f}亿")
print(f"预测期现值占比: {pv_fcf/ev_val:.1%}")
print(f"终值占比: {pv_term/ev_val:.1%}")

# 敏感性分析：不同的WACC和终值增长率
wacc_range = np.arange(0.08, 0.13, 0.005)
growth_range = np.arange(0.02, 0.045, 0.005)
sensitivity = pd.DataFrame(
    [[dcf_two_stage(fcf_projections, w, g, debt, cash)[0]
      for w in wacc_range]
     for g in growth_range],
    index=[f"g={g:.1%}" for g in growth_range],
    columns=[f"WACC={w:.1%}" for w in wacc_range]
)
print("\n敏感性分析：股权价值（亿元）")
print(sensitivity.round(1))

敏感性分析的结果通常以矩阵形式呈现，这里是一个示例：

股权价值(亿) WACC=8.0% WACC=9.0% WACC=10.0% WACC=11.0% WACC=12.0%
g=2.0% 152.3 122.1 99.8 82.7 69.2
g=2.5% 168.5 132.7 107.2 88.0 73.1
g=3.0% 189.4 145.8 115.9 94.1 77.5
g=3.5% 217.6 162.4 126.4 101.2 82.5
核心洞见：从上表可以看出一个惊人的事实——当WACC从10%降到8%时，估值从116亿涨到189亿，涨幅超过60%；当g从2.5%提高到3.5%时，估值也能变化20-30%。DCF不是一台"输出真理"的计算器，而是一个"测试假设"的框架。使用DCF的正确方式不是相信一个数字，而是理解多个假设下的可能结果区间。

DCF是估值理论中最重要的基石，但它的局限性也很明显——模型越精细，因为输入的微小变化导致的结果波动就越大。实践中，机构分析师常常用DCF来做"交叉验证"而非唯一决策依据。在下一章中，我们将转向另一种在实际工作中更常用的方法——相对估值法，它更直观、更贴近市场，但也需要我们更加警惕市场的非理性。
