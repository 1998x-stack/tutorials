第17章 数据转换器基础

大家好，我是你们的讲师。我们生活在一个"模拟"的世界里——声音是连续的空气振动、温度是连续的热力学量、加速度是连续的力学量。但我们用"数字"的世界来处理信息——0和1、数字信号处理、冯诺依曼架构。将这两个世界连接起来的桥梁，就是数据转换器（Data Converters）——ADC（模数转换器）和DAC（数模转换器）。

从某种意义上说，数据转换器是整个信息时代的"大门"——如果没有它们，数字信号处理器和计算机根本无法感知物理世界。你的手机里有至少5个ADC（麦克风、摄像头、触摸屏、传感器、RF接收链路）和至少3个DAC（音频输出、RF发射链路、显示驱动）。一颗现代SoC芯片中，数据转换器往往是设计难度最大、对性能影响最关键的模块之一。有些ADC——比如高速高精度ADC——被称为"芯片设计皇冠上的明珠"。

17.1 ADC/DAC基本原理

数据转换器的理论基础是奈奎斯特-香农采样定理（Nyquist-Shannon Sampling Theorem）：如果一个信号的最高频率分量为f_max，那么以不低于2·f_max的采样率对该信号进行采样，就可以从采样值中无失真地重建原始信号。这个"2·f_max"称为奈奎斯特频率（Nyquist rate）。

如果输入信号的频率高于f_s/2（即奈奎斯特频率），那么它会"混叠"（alias）到低频带内——这就是混叠现象（Aliasing）。例如，以100kHz采样一个80kHz的信号，重建后的信号看起来像是20kHz的信号——因为它与一个假想的20kHz信号的采样点完全重合。为了避免混叠，ADC之前必须放置一个抗混叠滤波器（Anti-Aliasing Filter, AAF）——通常是一个低通滤波器——来衰减所有高于f_s/2的频率分量。

数据转换器的另一个核心概念是量化误差（Quantization Error）。一个N位的ADC只能识别2^N个不同的电平，因此任何模拟输入信号都会被"四舍五入"到离它最近的那个量化电平。这个舍入误差在-½LSB到+½LSB之间均匀分布（理想情况下）。量化误差的功率为LSB²/12，因此对于满量程正弦波输入，理想信噪比为：

SNRideal = 6.02·N + 1.76 dB

这就是"每位6.02dB"的经典公式。一个12位ADC的理想SNR是74dB，16位ADC是98dB——但在实际中，由于热噪声、抖动和失真等因素，实际SNR总会低于理论值。因此我们引入ENOB（有效位数，Effective Number of Bits）的概念：ENOB = (SNRmeasured - 1.76) / 6.02。如果实测SNR是70dB，那么ENOB = (70 - 1.76)/6.02 ≈ 11.3位——你看，实际只有11.3位的有效精度，虽然它标称是12位。

17.2 常见ADC架构

不同的应用对ADC的要求天差地别——有的需要极快的速度（如RF接收机），有的需要极高的精度（如高精度测量仪器），有的需要极低的功耗（如IoT传感器节点）。因此，业界发展出了多种ADC架构，每种都有其独特的权衡。

Flash ADC（快闪式ADC）——这是最快的ADC架构。它使用2^N-1个比较器，每个比较器对应一个不同的参考电压，输入信号同时连接到所有比较器，输出一个"温度计码"（thermometer code）。Flash ADC可以在一个时钟周期内完成一次转换，速度可达数十GHz。但代价也是巨大的——每增加1位分辨率，比较器数量翻倍。8位Flash需要255个比较器，10位需要1023个——面积和功耗是指数增长的。因此Flash ADC几乎只用于分辨率≤8位的场景，通常用作高速Pipeline ADC的子ADC。

SAR ADC（逐次逼近型ADC）——这是目前最流行的ADC架构。它的核心思想是"二分法"：用一个比较器和一个DAC，从MSB开始逐位比较，逐次逼近输入信号。N位的SAR ADC需要N个时钟周期完成一次转换。SAR ADC的主要优势是功耗极低（因为它使用动态逻辑，没有静态功耗），而且面积小、结构简单。现代SAR ADC在先进工艺下可以达到16位分辨率、数MS/s的采样率。它已经成为中低功耗ADC的主流选择。

Pipeline ADC（流水线型ADC）——介于Flash和SAR之间，兼顾速度和精度。Pipeline ADC由多级串联组成，每级包含一个低分辨率的子ADC（通常1.5-4位）和一个MDAC（乘法DAC）。每级处理完当前数据后，将残差放大后传给下一级。多级同时工作（流水线），因此可以同时实现高速度（数十至数百MS/s）和高分辨率（12-16位）。Pipeline ADC广泛用于通信基站、雷达接收机和视频采集系统。

Sigma-Delta ADC（Σ-Δ ADC）——基于过采样（oversampling）和噪声整形（noise shaping）技术。它以远高于奈奎斯特频率的采样率对输入信号进行采样，然后通过负反馈环路将量化噪声推到高频段，再用数字滤波器滤除带外噪声。Sigma-Delta ADC可以达到极高的分辨率（16-24位甚至32位），但转换速率较低（通常≤数MS/s），主要用于音频和精密测量应用。

ADC架构 分辨率 采样率 功耗 面积 延迟 典型应用
Flash ≤8位 >10GS/s 极高 极大 1个周期 超高速数据采集、示波器前端
SAR 8-16位 100k-10MS/s 极低 小 N个周期 IoT、传感器、电池供电设备
Pipeline 10-16位 10M-1GS/s 中-高 中 多周期 通信基站、雷达、视频采集

17.3 常见DAC架构

DAC的架构选择与ADC类似，也是速度、精度、功耗和面积之间的权衡。

R-2R电阻梯形网络（R-2R Ladder）——这是最经典的DAC结构，仅用两种阻值（R和2R）的电阻构成。每个数字位控制一个开关，将对应的电流贡献到输出端。R-2R DAC的优点是结构简单、单调性有保证（输出随数字码增加而准确增加），但电阻的匹配精度限制了分辨率——即使是激光修调的薄膜电阻，也只能做到约12-14位的精度。在IC中实现高精度R-2R需要占用相当大的面积。

电流舵DAC（Current-Steering DAC）——这是高速DAC的主流架构。每个数字位控制一个电流源是否被切换到输出端。电流舵DAC的速度极快——可以工作在多GHz的更新频率——因此广泛用于RF发射链路中的Tx DAC。实现方式有二进制加权（binary-weighted）和温度计码（thermometer-coded）两种。温度计码的单调性和毛刺（glitch）性能更好，但解码逻辑面积更大。现代高速DAC常使用分段结构——高位采用温度计码，低位采用二进制加权——来平衡性能和面积。

串型DAC（String DAC）——用一个电阻串产生2^N个不同电压，每个节点连接一个开关选择输出。结构简单且单调性好，但分辨率增加时电阻数量指数增长。8位需要256个电阻，16位需要65536个——显然不切实际。因此String DAC通常只用于低分辨率（≤8位）或用作更复杂DAC中的子模块。

Sigma-Delta DAC——与Sigma-Delta ADC类似，利用过采样和噪声整形将低位量化噪声推到带外，然后通过模拟低通滤波器重建干净的模拟信号。Sigma-Delta DAC的精度极高（16-24位），但更新速率受限于数字调制器的时钟频率。几乎所有高保真音频DAC（如ESS Sabre、AKM Velvet Sound系列）都采用Sigma-Delta架构。

17.4 ADC/DAC性能测试

设计完一个数据转换器之后，你怎么知道它到底好不好？你需要测量它的性能指标。这些指标分为静态和动态两大类。

静态指标：

  • INL（积分非线性，Integral Nonlinearity）——实际传输曲线与理想直线之间的最大偏差，以LSB为单位。INL决定了DC精度和失真性能。
  • DNL（差分非线性，Differential Nonlinearity）——实际步长与理想步长（1LSB）之间的偏差。DNL > 1LSB意味着"漏码"（missing code）——至少有一个数字码永远不会出现。DNL是判断ADC是否单调的关键指标。

动态指标：

  • SNR（信噪比）——信号功率与噪声功率之比（不包括谐波）。
  • SINAD（信纳比，Signal-to-Noise and Distortion Ratio）——包括噪声和谐波的信号功率与总失真功率之比。
  • SFDR（无杂散动态范围，Spurious-Free Dynamic Range）——信号功率与最大杂散（spur）功率之比。SFDR在通信系统中尤其关键——带外的大杂散可能干扰相邻信道。
  • THD（总谐波失真，Total Harmonic Distortion）——各次谐波功率之和与基波功率之比。

最常用的测试方法是用一个纯净的正弦波输入ADC（或测试DAC的输出），采集大量样本后进行FFT（快速傅里叶变换）分析。FFT频谱展示了输入信号的基波分量、谐波、噪声基底和杂散。从FFT中你可以计算SNR、SINAD、SFDR、THD和ENOB等所有动态指标。

为了保证FFT测试的准确性，需要满足相干采样（Coherent Sampling）条件：f_in / f_s = M / N_fft，其中M是奇数（或质数），N_fft是FFT的点数。相干采样确保信号的能量集中在单一FFT bin中，不会因频谱泄漏造成测量误差。

Python FFT分析代码 —— 评估ADC动态性能

import numpy as np
import matplotlib.pyplot as plt

def adc_fft_analysis(data, fs, fin, nfft=8192, full_scale=1.0):
    """
    data: ADC采样数据（归一化到0~1范围或有符号浮点数）
    fs: 采样率 (Hz)
    fin: 输入信号频率 (Hz)
    nfft: FFT点数
    full_scale: ADC满量程幅度
    """
    # 窗口函数（Hanning减少频谱泄漏）
    window = np.hanning(nfft)

    # 选取N点加窗
    segment = data[:nfft] * window

    # FFT
    spectrum = np.fft.fft(segment) / np.sum(window)
    mag = np.abs(spectrum[:nfft//2])

    # 转换为dBFS
    mag_dbfs = 20 * np.log10(mag / (full_scale/2))

    # 频率轴
    freq_axis = np.arange(nfft//2) * fs / nfft

    # 查找基波（输入频率附近的最大分量）
    bin_fin = int(fin * nfft / fs)
    search_range = 5
    fund_bin = np.argmax(mag[bin_fin-search_range:bin_fin+search_range+1]) + bin_fin - search_range
    fund_power = mag[fund_bin]**2

    # 计算SNR（排除基波、谐波和DC）
    harmonic_bins = []
    for h in range(2, 10):
        h_bin = int(h * fund_bin)
        if h_bin < nfft//2:
            harmonic_bins.extend(range(h_bin-2, h_bin+3))

    noise_bins = np.ones(nfft//2, dtype=bool)
    noise_bins[0] = False          # 排除DC
    noise_bins[max(0,fund_bin-2):fund_bin+3] = False  # 排除基波
    for hb in harmonic_bins:
        if 0 <= hb < nfft//2:
            noise_bins[hb] = False  # 排除谐波

    noise_power = np.sum(mag[noise_bins]**2)
    harm_power = np.sum(mag[harmonic_bins]**2)

    snr = 10 * np.log10(fund_power / noise_power)
    sinad = 10 * np.log10(fund_power / (noise_power + harm_power))
    sfdr = 10 * np.log10(fund_power / np.max(mag[~noise_bins][mag[~noise_bins]>0]**2))
    enob = (sinad - 1.76) / 6.02

    return {
        'SNR': snr, 'SINAD': sinad, 'SFDR': sfdr, 'ENOB': enob,
        'freq_axis': freq_axis, 'mag_dbfs': mag_dbfs
    }

17.5 数据转换器选型

在实际项目中选择数据转换器——不管是买IP还是自己设计——你需要回答三个问题：需要多快？需要多准？多少功耗是允许的？

一个有用的工具是FoM（Figure of Merit，优值）：

FoM = Power / (2^ENOB × f_s) （单位：pJ/conversion-step）

FoM将分辨率、速度和功耗统一为一个数字。现代先进工艺下的SAR ADC可以达到1-10 fJ/conversion-step（注意单位是fJ，不是pJ），Pipeline ADC在100-500 fJ/conversion-step范围，Flash ADC在1-10 pJ/conversion-step。如果你设计一个ADC，FoM是衡量你的设计水平的重要标尺。

具体到不同应用：

  • 音频（Audio）：需要高分辨率（16-24位），但速度很低（48-384kHz）。Sigma-Delta ADC/DAC是唯一合理的选择。
  • IoT传感器：需要极低功耗（µW级别），速度极低（几kS/s）。SAR ADC占主导地位。
  • 通信基带：需要中高分辨率（10-14位）和中高速度（数十至数百MS/s）。Pipeline ADC或高速SAR ADC是备选。
  • RF接收机：需要极高速度（>1GS/s），分辨率较低（6-12位）。Flash或TI（时间交织）SAR ADC。
  • 精密测量：需要极高分辨率（18-24位），低速。Sigma-Delta ADC或高精度SAR ADC。
真实的"踩坑"故事：2018年参与一个传感器产品项目，选用了某款标称16位的SAR ADC作为信号采集核心。芯片回来后，系统在室温下性能良好，但到了85°C高温测试时，信号链的总THD恶化了约10dB，产品无法通过规范。最初大家怀疑是前端运放的问题，花了两个月排查运放、更换多款运放，问题依旧。最终定位到根本原因——ADC的DNL规格在数据手册上写的是"典型值±0.5 LSB"，但只有温度范围从0°C到70°C时才能保证。85°C时DNL实际上坏到了±2 LSB，导致傅里叶变换中出现大量谐波。而我们的选型工程师只看了室温下的DNL典型值，没有注意到温度范围的限制。这次教训之后，团队的所有选型流程要求同时审查"保证的温度范围"和"全温度范围内的最差规格"，而不是室温典型值。一个小小的选型疏忽，导致产品延期了6个月。

17.6 本章小结

这一章我们了解了数据转换器的世界。我们从奈奎斯特-香农采样定理出发，理解了混叠和量化噪声的基础知识；我们熟悉了四种主流ADC架构——Flash、SAR、Pipeline和Sigma-Delta——它们的优劣势和适用场景；我们学习了DAC的四种常见架构——R-2R、电流舵、串型和Sigma-Delta；我们掌握了ADC/DAC的静态和动态性能测试方法——INL/DNL和FFT分析；最后，我们研究了如何为特定应用选择合适的数据转换器架构。

数据转换器是一个极其丰富且深奥的领域——一本教科书都写不完。但本章给了你一张"全景地图"，让你能理解每种架构的基本原理和权衡。下一章我们将把这些知识整合起来，进入一个更综合的领域——混合信号设计方法（Mixed-Signal Design Methodology）。我们将讨论模拟信号和数字信号在同一颗芯片上共存时遇到的各种挑战——衬底噪声耦合、版图匹配、数模混合仿真和签核流程。这是芯片设计中真正的"硬桥硬马"之处。
