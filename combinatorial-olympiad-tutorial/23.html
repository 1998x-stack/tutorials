第23章 操作与对策（一）

大家好，我是你们的讲师。今天开启一个全新的专题——操作与对策。这可能是整个组合数学领域里最"好玩"的部分了：你有一个局面，你可以做某些操作，对手也可以做某些操作，你要么想证明"怎么折腾也到不了某个状态"（不变量），要么想证明"我肯定能赢"（必胜策略），要么想证明"这操作早晚得停"（单调有界）。本质上，这些问题都共享一个分析框架。

很多同学一看到操作题就头大，觉得没套路。但实际上，操作题的套路比计数的套路还要清晰——你只需要学会问自己三个问题：第一，有没有什么东西是操作前后不变的？第二，有没有什么东西在朝一个方向单调变化？第三，如果是对策游戏，哪些局面是必胜的、哪些是必败的？把这三个问题想清楚，90%的操作题都能解开。

本章我们重点建立这个框架，从最简单的操作开始，一步步深入到经典的Nim游戏和SG函数。下一章我们会讨论批量操作、翻转类问题和构造性操作。好了，我们开始吧。

23.1 操作问题的统一分析框架

无论操作题的面貌如何千变万化，其核心结构都是一样的。我们可以把任何操作问题抽象为以下四步：

  1. 定义状态：问题的局面用一个数学对象表示（数组、数对、图、多项式……）。
  2. 定义操作：合法操作如何改变状态。
  3. 找不变量/单调量：操作前后什么保持不变？什么在单调变化？
  4. 得出结论：利用不变量或单调量证明"不可能"、"必然"或"必胜"。

这个框架看似简单，但几乎所有操作题都能装进去。我们来看一个具体的例子：

例1：在黑板上写下数字 1, 2, 3, …, 20。每次操作擦去两个数 a 和 b，然后写上 a+b+ab。经过若干次操作后，黑板上只剩一个数。问这个数是多少？

分析：

  • 状态：当前黑板上的所有数构成的集合。
  • 操作：擦去 a, b，写入 a+b+ab。
  • 找不变量：考虑 (a+1)(b+1) = ab + a + b + 1。也就是说，每次操作把 a+1 和 b+1 替换成了 (a+1)(b+1)。

那么操作前后所有数的 (值+1) 的乘积保持不变！初始时乘积为 2×3×4×…×21 = 21!。最终只剩一个数 x，那么 x+1 = 21!，所以 x = 21! - 1。

这个例子完美展示了"找不变量"的威力——你不需要模拟整个操作过程，只需要抓住那个操作前后不变的量，一步就推出了结果。

23.2 不变量证明"不可能"型操作题

不变量最经典的应用就是证明"不可能"——比如"能否通过若干次操作将状态A变为状态B"，如果A和B的不变量不同，那就是不可能的。

23.2.1 奇偶性不变量

最常见的也是最简单的不变量：奇偶性。如果每次操作都改变某个量的奇偶性，而初始状态和目标状态的奇偶性不同，那就永远达不到目标。

例2：一个 8×8 的棋盘，左上角和右下角的格子被去掉了。问能否用 2×1 的多米诺骨牌不重叠地覆盖剩余的 62 个格子？

很多人直觉觉得可以——62是偶数，2×1骨牌每次覆盖2格，看起来没毛病。但答案是：不能。

为什么？ 将棋盘按国际象棋棋盘的方式黑白染色。一个 2×1 骨牌无论怎么放，总是覆盖一个黑格和一个白格。所以如果能够完全覆盖，那么被覆盖的黑格数必须等于白格数。但去掉的两个角都是同色的（国际象棋棋盘对角同色），所以剩下的棋盘有 32 个黑格和 30 个白格（或反过来），黑白格数不等。因而不可能被完全覆盖。

这个著名的"残缺棋盘问题"是奇偶性不变量应用的经典案例。竞赛中类似的题目多到数不过来——只要看到操作涉及"配对"或"相邻交换"，第一时间想到染色和奇偶性。

23.2.2 模运算不变量

奇偶性不变量本质上就是模2不变量。更一般地，我们可以考虑模某个数的余数。来看一个升级版的例子：

例3：有 n 堆石子，每堆有若干颗。每次操作可以选择任意一堆，从中取出至少一颗石子。证明：无论怎样操作，石子总数的奇偶性和初始时相同。

等等，这个例子太简单了——每次拿走至少一颗，总数减少，奇偶性难道不是可能改变吗？实际上这里有个陷阱。题目没有说"只能拿一颗"，拿一颗会改变奇偶性，拿两颗会保持奇偶性不变。所以实际上总数奇偶性并不是不变量。那真正的不变量是什么？正确的分析需要重新定义状态。

这个例子告诉我们在找不变量时一定要小心：不要随便声称某个量是"不变量"，必须严格验证每次操作下它是否真的不变。

23.2.3 加权和不变量

有时候简单的奇偶性或模数不够用，需要给每个位置赋予不同的"权重"，构造加权和作为不变量。这是竞赛中最高级的不变量技巧之一。

关键洞察：加权和不变量的构造思路是这样的——设每个位置 i 有权重 w_i，当前状态下位置 i 的值是 x_i，那么构造 S = Σ w_i × x_i。然后检查每次操作下 S 是否不变，据此反推 w_i 应该满足的条件。

来看一个具体的例子：

例4（IMO 1994 预选题）：在一个 8×8 的棋盘上，每个格子中有一个整数。每次操作可以选择一个 2×2 的子方格，将其中四个数各加 1。问能否经过若干次操作使得所有格子中的数相等？

我们可以给格子 (i,j) 赋予权重 (-1)^{i+j}。每次操作选择的 2×2 子方格中，四个格子的权重分别为 +1, -1, -1, +1（取决于你从哪个位置开始标记），它们的和是 0。所以加权和 S = Σ (-1)^{i+j} × a_{ij} 是一个不变量。如果初始时这个不变量不为0，而最终要达到全相等的状态（此时 S = 0 或不是0取决于相等值的奇偶），就可以判断不可能。

23.3 单调有界量证明"必然终止"型操作题

有些操作题不要求证明"不可能"，而是要求证明"这个过程一定会终止"或"最多进行多少步"。这时我们的武器是单调有界量——找一个每次操作都严格增加（或减少）且存在上界（或下界）的量。

例5：有 n 个正整数排成一行。每次操作可以任意选择两个相邻的数，如果它们的最大公约数大于 1，就把它们合并为一个数（新数为两数之和）。证明：这个操作过程必然在有限步内终止。

分析：每次操作合并两个相邻的数，数的个数减少 1。而数的个数从 n 开始，最少到 1。所以最多经过 n-1 次操作就必然终止——这就是最简单的"单调量"论证。

但更常见的情况是：操作不减少状态空间的大小，但某个"势能函数"在单调变化。例如：

例6：黑板上写有若干个正实数。每次操作擦去两个数 a 和 b，写上 (a+b)/2。证明：经过有限次操作后，所有数都会变得相等。

分析思路：考虑最大数 M 和最小数 m。每次操作取两个数 (a,b) 替换为 (a+b)/2。如果 a 和 b 不相等，(a+b)/2 严格介于 a 和 b 之间。所以 M 不会增加，m 不会减少。而且只要还有不相等的数，M 和 m 的差就会严格减小——因为要么最大数被操作了（新数小于原最大数），要么最小数被操作了（新数大于原最小数）。但 M-m 这个量是非负的、单调递减的。不过要注意，它不一定能严格递减到0——如果操作的两个数恰好是同样大的，M-m 不改变。

这个例子留给读者思考：如何构造一个"每次操作都严格减小"的势能函数？提示：考虑所有数的方差或所有数的两两差的平方和。

23.4 游戏对策中的必胜/必败态

现在我们进入对策游戏的部分。这类问题中，两个玩家轮流操作，目标通常是"无法操作的一方输"（normal play）。

23.4.1 必胜态与必败态的递归定义

这是所有组合游戏理论的根基。一个游戏状态被称为：

  • 必败态（P-position）：当前玩家无法做出任何合法操作（即已经输了），或者在当前状态下，无论当前玩家怎么操作，对手都面临必胜态。
  • 必胜态（N-position）：存在至少一种合法操作，使得操作后对手面临必败态。

更简洁的递归定义是：

一个状态是必败态，当且仅当它没有合法操作，
或者所有合法操作都通向必胜态。

一个状态是必胜态，当且仅当存在至少一个合法操作通向必败态。

这个递归定义看起来简单，但在复杂的游戏中直接应用可能需要分析大量状态。我们来看一个最基础的游戏：

例7（取石子游戏，单堆版）：有一堆石子，共 n 颗。两人轮流取走 1 到 m 颗石子，取走最后一颗者胜。问哪些 n 是必胜态？

石子数 n 0 1 2 3 4 5 6 7
m=2 时的状态 必败 必胜 必胜 必败 必胜 必胜 必败 必胜

发现规律了吗？n 是 (m+1) 的倍数时是必败态，否则是必胜态。这是因为：如果 n = k(m+1)，对手取 t 颗（1≤t≤m），你可以取 (m+1-t) 颗，让对手每次都面对 (m+1) 的倍数。这是"对称策略"的经典应用。

23.4.2 Nim 游戏

Nim 游戏是组合游戏理论中最重要的模型。它的规则是：有若干堆石子，每堆任意多颗。每次操作可以选择任意一堆，从中取走至少一颗（可以全部取走）。无法操作者输。

Nim 游戏有一个优美的结论，被称为 Bouton 定理：

Bouton 定理：Nim 游戏中，一个局面是必败态当且仅当所有堆石子数的异或（XOR）等于 0。

用符号表示：设各堆石子数为 a₁, a₂, …, a_k，则局面为必败态 ⟺ a₁ ⊕ a₂ ⊕ … ⊕ a_k = 0。

这个定理的证明分两步：

  1. 如果当前异或和为 0，那么任何操作（改变某一堆）都会使异或和变为非 0。
  2. 如果当前异或和为非 0，一定存在一种操作（改变某一堆）使得异或和变为 0。

证明第二步需要一点技巧：设异或和为 S ≠ 0，找到 S 的最高位 1 所在位置，那至少有一堆石子在该位上也是 1。从这堆中取走适当数量的石子，就能让异或和变为 0。具体构造方法如下：

设 S = a₁ ⊕ a₂ ⊕ … ⊕ a_k ≠ 0。
设 S 的最高位是第 d 位。
存在某堆 a_i，其第 d 位为 1。
令 a_i' = a_i ⊕ S，则 a_i' < a_i（因为第 d 位从 1 变为 0）。
从第 i 堆中取走 (a_i - a_i') 颗石子，操作后新异或和为：
(a_i ⊕ S) ⊕ (⊕_{j≠i} a_j) = (⊕_j a_j) ⊕ S = S ⊕ S = 0。

这个定理的美妙之处在于：它将一个看似复杂的多人博弈简化为一个异或运算。你不需要模拟任何操作，只需要计算异或和就能知道当前局面谁必胜。

⚠️ 踩坑实录：有一年我带学生参加省队选拔，一道看似简单的Nim变种题把好多人坑了。题目说："有3堆石子，分别有 7、9、13 颗。每次可以从任意一堆中取走任意正整数颗（至少1），但不能从两堆或多堆中同时取。取走最后一颗者胜。"一个学生直接算了 7⊕9⊕13 = 7⊕9=14, 14⊕13=3 ≠ 0，直接说"先手必胜"。但题目中有一个隐藏条件：取走的石子数不能等于当前堆已取走的石子总数（意思是有一个全局计数器记录已取走的石子总数）。这个额外规则完全改变了游戏的性质，再直接用标准Nim的异或和结论就错了。

这个故事告诉我们：不要套公式！ 看到"取石子"就立刻套 Nim 的结论是非常危险的。在套用任何现成结论之前，先仔细确认游戏规则是否完全匹配。

23.5 SG 函数简介

Nim 游戏能直接用异或和解决，但大多数组合游戏并没有这么简单的形式。我们需要一个更一般的理论——这就是 Sprague-Grundy 理论（简称SG理论）。

SG 函数的核心思想是：每个游戏状态都对应一个非负整数（称为SG值），这个值刻画了该状态在游戏中的"战斗值"。SG 值的递归定义如下：

设当前状态为 S，它的所有后继状态（一步操作能到达的状态）的 SG 值
构成的集合为 {g₁, g₂, …, g_k}。那么状态 S 的 SG 值为：

g(S) = mex{ g₁, g₂, …, g_k }

其中 mex（minimum excluded）表示"最小的未出现的非负整数"。

特别地，没有后继状态的状态（终止状态）的 SG 值为 0。

SG 值的重要性质：

  • g(S) = 0 当且仅当 S 是必败态
  • g(S) > 0 当且仅当 S 是必胜态
  • 如果游戏由多个独立子游戏组成（比如多堆石子，每次只能在一堆上操作），总游戏的 SG 值等于各子游戏 SG 值的异或和

最后一条性质是 SG 理论的核心：游戏的和 = SG 值的异或。这也就是为什么 Nim 游戏的结论恰好是异或和为 0——因为 Nim 游戏中一堆石子的 SG 值就是它本身的石子数（可以验证），所以多堆 Nim 的 SG 值就是各堆石子数的异或和。

一点思考：SG 函数的神奇之处在于，它把一个复杂游戏分解成了若干个简单游戏的"异或和"。如果你能算出每个子游戏的 SG 值，那么整个游戏的胜负就一目了然。不过，计算 SG 值通常需要递归或动态规划，对于状态空间很大的游戏，这不是一件容易的事。竞赛中常见的 SG 函数题，状态空间通常不会太大，或者有明确的数学规律。

来看一个具体的例子，计算某个游戏的 SG 值：

例8（移动棋子游戏）：在一条 1×n 的格子上，一枚棋子初始在位置 1。每次可以向左或向右移动 1 格或 2 格，但不能移出边界（即位置必须在 1 到 n 之间）。当棋子到达位置 n 时游戏结束（到达 n 的玩家获胜）。求每个位置的 SG 值。

我们手动计算 n=6 的情况：

位置 i 1 2 3 4 5 6
可到达的位置 2,3 1,3,4 1,2,4,5 2,3,5,6 3,4,6 —
后继 SG 值 g(2),g(3) g(1),g(3),g(4) g(1),g(2),g(4),g(5) g(2),g(3),g(5),g(6) g(3),g(4),g(6) —
SG 值 g(i) ? ? ? ? ? 0

从右向左递推：g(6)=0（终止态）。g(5)=mex{g(3),g(4),0}，需要先知道 g(3),g(4)。继续递推就能全部算出来。这个过程留给读者完成，计算后你会发现这个游戏的 SG 值呈现周期性规律——这正是 SG 函数的美妙之处。

23.6 "状态图"与可达性分析

操作问题除了对弈类之外，还有一大类就是"状态转移"类问题：给定初始状态和一系列合法操作，问你能到达哪些状态？或者最少步数是多少？

这类问题本质上是在一个有向图上做搜索——每个状态是一个节点，每次操作是一条有向边。但你通常不需要真的建图，而是用不变量来刻画可达状态的共同特征。

例9：有三个容器，容量分别为 10L、7L、3L。初始时 10L 的容器装满水，其余为空。每次操作可以将一个容器中的水倒入另一个容器，直到倒空或另一个容器满。问能否量出 5L 水？

这是一个经典的"倒水问题"。它的状态可以用三个数 (a,b,c) 表示当前三个容器中的水量。每次操作相当于改变这些数。关键不变量是什么？注意到三个数的和始终为 10（没有水被倒掉），而且每个容器的水量都是整数。所以我们要找的是满足 a+b+c=10 且 0≤a≤10, 0≤b≤7, 0≤c≤3 的状态中，能否达到 a=5、b=5、c=0 等状态。进一步的约束来自"倒水"操作的性质——但这需要更深入的分析。

我倒水问题在学生中有一个很有意思的现象：很多人觉得倒水问题"没有技术含量"，靠瞎试就能试出来。但竞赛中的倒水问题通常要你证明"不能量出某个体积"——这时候就必须用不变量来证明了。

23.7 本章小结

今天我们建立了一个完整的操作题分析框架：定义状态 → 定义操作 → 找不变量/单调量 → 得出结论。我们学习了三种主要的操作题类型：

  • 不变量证明"不可能"：奇偶性不变量、模不变量、加权和不变量。核心是找到一个操作前后绝对不变的量，如果初始和目标状态这个量不同，就不可能。
  • 单调有界证明"必然终止"：构造一个每次操作都严格单调变化且有上下界的势能函数，从而证明过程必然有限步内结束。
  • 必胜/必败态分析：从递归定义出发，到 Nim 游戏的异或和结论，再到更一般的 SG 函数理论。

下一章我们将继续这个专题，讨论批量操作（如翻转问题、开关灯问题）和构造性操作问题——也就是需要你给出具体的操作序列来达到目标状态。这些题目在联赛中出现的频率非常高，我们到时候见！
