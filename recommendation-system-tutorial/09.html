第9章 Wide&Deep与DeepFM

上一章我们看到了模型从LR进化到FM再到FFM的过程。但FM有一个天花板——它只能捕获二阶特征交叉。即使训练再久、数据再多，FM也无法自动学习"用户年龄×物品类别×时间"这样的三阶交叉。

与此同时，深度学习正在整个计算机领域中势如破竹。2016年，Google发表了一篇极具影响力的论文——Wide & Deep Learning for Recommender Systems。这篇论文开创性地把一个线性模型和一个深度神经网络联合训练，开启了CTR预估模型的深度学习时代。

1. Wide&Deep：记忆与泛化的艺术

Wide&Deep这篇论文来自Google Play Store的推荐团队。当时他们面临一个典型的两难问题：

  • 记忆（Memorization）：用户下载了"计算器"之后，再推荐"科学计算器"——这是基于共现模式的记忆。Wide侧的LR模型擅长这个。
  • 泛化（Generalization）：用户下载了一批工具类App，推荐其他工具类App——这是基于embedding的泛化。Deep侧的DNN擅长这个。

Wide&Deep的做法非常直接：把这两个组件拼在一起联合训练。

Wide侧就是一个标准的逻辑回归，但输入不是原始特征，而是手工构造的交叉特征（cross product transformation）。比如用"user_installed_app=计算器 AND impression_app=科学计算器"这样的组合特征，让模型可以精确记忆特定的共现模式。

Deep侧是一个前馈神经网络，将分类特征映射为低维稠密embedding，然后通过若干隐藏层来学习特征之间的非线性交互。embedding的引入让模型可以泛化到未见过的特征组合。

最终的输出是两部分的加权和：

ŷ = σ(w_wide^T x_wide + w_deep^T a^{[l]} + b)

其中 a^{[l]} 是Deep侧最后一层隐藏层的输出。整个模型通过联合训练（Joint Training）同时优化Wide和Deep的参数。

维度 Wide 侧 Deep 侧
核心能力 记忆（Memorization） 泛化（Generalization）
数学模型 逻辑回归（LR） 多层神经网络（DNN）
输入特征 手工交叉特征 原始特征 + Embedding
擅长场景 高频共现模式、精确规则 稀疏特征组合、新模式发现
学习方式 特征级别的精确权重 embedding级别的语义泛化
优化器 FTRL（带L1正则） Adam（带L2正则）

Google的线上实验结果显示，Wide&Deep相比纯Wide模型，App下载率提升了约3.9%。这个数字在今天看来可能不大，但对于Google Play这种体量的产品，3.9%意味着千万级的新增下载量。

Wide&Deep最大的历史贡献不是它的AUC指标，而是它证明了：推荐系统可以使用深度神经网络，而且深度学习与传统线性模型可以共存、互补。

2. DeepFM：用FM替代Wide

Wide&Deep虽然有效，但有一个明显的工程痛点：Wide侧需要手工构造交叉特征。Google在论文中使用了"user_installed_app=计算器 AND impression_app=科学计算器"这类特征，但具体哪些特征需要交叉——这是手工活，而且不同业务场景差异巨大。

2017年，华为诺亚方舟实验室提出了DeepFM，核心思想极其直接：把Wide&Deep中的Wide替换成FM，这样模型就不需要手工构造交叉特征了。

DeepFM的架构包含三个部分：

  • FM Component：捕获一阶和二阶特征交叉（自动的，不需要手工特征工程）
  • Deep Component：捕获高阶特征交叉（DNN）
  • 共享Embedding层：FM和Deep两部分共享同一个embedding矩阵

输出由两部分相加：

ŷ = σ(ŷ_FM + ŷ_Deep)

这个设计优雅在什么地方？Wide&Deep的Wide侧需要你手工挑选哪些特征需要做交叉变换——这是一个经验密集型工作。而DeepFM的FM侧自动对所有特征做二阶交叉，完全不需要人工介入。

你知道吗？DeepFM是在中国推荐技术圈落地最深、最广的深度CTR模型之一。从字节跳动到美团，从快手到阿里巴巴，几乎所有主流互联网公司的推荐系统中都有DeepFM或其变体的身影。它之所以如此流行，是因为它在精度和工程复杂度之间取得了近乎完美的平衡——效果显著好于Wide&Deep，但实现难度和训练成本只增加了很少。

3. 共享Embedding层的设计哲学

DeepFM的一个重要设计决策是FM和Deep共享同一个embedding层。这个决策看似简单，却蕴含着深刻的工程哲学。

共享embedding的三大好处：

  • 参数效率：FM和Deep各维护一套embedding的话，参数量翻倍，内存需求也翻倍。在特征数量动辄千万级的工业场景中，这个差别非常可观。
  • 联合训练信号：一个样本的反向传播信号同时更新embedding——FM部分提供二阶交互的梯度，Deep部分提供高阶非线性的梯度。共享embedding接收了来自两方面的监督信号，学到的表示更加鲁棒。
  • 特征表示统一：如果FM和Deep各有一套embedding，上线时两个embedding矩阵需要分别维护和更新，存在版本不一致的风险。共享embedding彻底避免了这个问题。

那embedding维度怎么选？工业界常用的范围是8到64，而且通常是2的幂次（8, 16, 32, 64）。为什么是2的幂？一方面和GPU内存对齐有关，另一方面是一个经验法则——特征总量越大、每个特征的频次越低，维度就应该越小。

4. PyTorch代码实战

下面我们用PyTorch实现一个完整的DeepFM模型。这个实现虽然是教学版的，但在结构上完全遵循了工业DeepFM的标准设计。

import torch
import torch.nn as nn
import torch.nn.functional as F

class DeepFM(nn.Module):
    def __init__(self, feature_dims, embed_dim=16, hidden_dims=[256, 128, 64]):
        """
        feature_dims: dict, 每个field的特征数量
        embed_dim: embedding维度
        hidden_dims: DNN隐层维度列表
        """
        super().__init__()
        total_features = sum(feature_dims.values())

        # 共享Embedding层
        self.embedding = nn.Embedding(total_features, embed_dim, sparse=True)

        # 一阶权重 (Linear part)
        self.linear = nn.Embedding(total_features, 1, sparse=True)

        # Deep部分 (DNN)
        dnn_input = len(feature_dims) * embed_dim  # 所有field embedding concat
        dnn_layers = []
        in_dim = dnn_input
        for h in hidden_dims:
            dnn_layers.append(nn.Linear(in_dim, h))
            dnn_layers.append(nn.BatchNorm1d(h))
            dnn_layers.append(nn.ReLU())
            dnn_layers.append(nn.Dropout(0.3))
            in_dim = h
        self.dnn = nn.Sequential(*dnn_layers)
        self.dnn_output = nn.Linear(hidden_dims[-1], 1)

        self._init_weights()

    def _init_weights(self):
        for m in self.modules():
            if isinstance(m, nn.Linear):
                nn.init.xavier_normal_(m.weight)

    def forward(self, x):
        """
        x: (batch, num_fields) 每个元素是特征的全局索引
        """
        # Embedding lookup: (batch, num_fields, embed_dim)
        embeddings = self.embedding(x)

        # --- Linear (一阶) ---
        linear_out = self.linear(x).squeeze(-1)  # (batch, num_fields)
        linear_out = linear_out.sum(dim=1, keepdim=True)  # (batch, 1)

        # --- FM (二阶交叉) ---
        sum_embed = embeddings.sum(dim=1)             # (batch, embed_dim)
        sum_square = (embeddings ** 2).sum(dim=1)     # (batch, embed_dim)
        fm_out = 0.5 * (sum_embed ** 2 - sum_square).sum(dim=1, keepdim=True)

        # --- Deep ---
        batch_size = embeddings.size(0)
        dnn_input = embeddings.reshape(batch_size, -1)  # flatten
        dnn_out = self.dnn(dnn_input)
        deep_out = self.dnn_output(dnn_out)

        # --- 合并输出 ---
        out = linear_out + fm_out + deep_out
        return torch.sigmoid(out)

训练代码的关键部分——数据加载需要将每个field的特征映射为全局索引，然后使用Adam或FTRL优化器。对大规模稀疏特征，推荐使用 sparse=True 的Embedding层，配合SGD或FTRL优化器，可以极大减少内存占用。

工业小贴士：DeepFM的训练可以使用"梯度截断"（Gradient Clipping）来防止极端样本导致的梯度爆炸。同时，建议为不同频率的特征设置不同的学习率——出现次数少的特征用更大的学习率。Facebook在工程论文中提出的"learning rate warmup + sqrt decay"学习率策略也值得一试。

让我再分享一个真实的迁移故事。有一家公司的推荐团队，从Wide&Deep迁移到DeepFM后，原来的全职特征工程岗位工作量直接减少了约60%。团队负责人说："以前我们每周要花两天跑交叉特征实验，现在一个月可能才需要做一次。模型自己学交叉，大部分人工特征都变成了冗余。"

DeepFM为推荐系统建立了一个"基准线"。无论是学术论文还是工业实践，当人们提出一个新模型时，第一个要对比的baseline往往就是DeepFM。但DeepFM不是终点——它仍然使用DNN隐式地进行高阶交叉。下一章我们要讲的DCN和xDeepFM，将带我们进入显式高阶交叉的世界。
