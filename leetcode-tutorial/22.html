树状数组

大家好。如果说前缀和是"静态区间查询"的王者，那树状数组（Fenwick Tree / Binary Indexed Tree）就是"动态区间查询"的利器。它支持 O(log n) 时间内的单点更新和前缀和查询，代码极其简洁——只有几行——却蕴含着精妙的二进制思想。

树状数组的名字来源于它的形状：每个节点 tree[i] 管理一段长度为 lowbit(i) 的区间。其中 lowbit(i) = i & (-i)，它返回的是 i 的二进制表示中最低位的 1 所对应的值。

先看树状数组的核心操作：

class FenwickTree:
            def __init__(self, n):
                self.n = n
                self.bit = [0] * (n + 1)

            def lowbit(self, i):
                return i & (-i)

            def update(self, i, delta):
                """将下标 i 的值增加 delta（1-indexed）"""
                while i <= self.n:
                    self.bit[i] += delta
                    i += self.lowbit(i)

            def query(self, i):
                """查询前缀和 [1..i]"""
                s = 0
                while i > 0:
                    s += self.bit[i]
                    i -= self.lowbit(i)
                return s

            def range_sum(self, l, r):
                """区间和 [l..r]"""
                return self.query(r) - self.query(l - 1)

是不是感觉代码非常对称？update 是 i += lowbit(i) 向上爬，query 是 i -= lowbit(i) 向下走。两者的循环次数都是 O(log n)——因为每次操作至少跳过一个二进制位。

理解关键：树状数组中每个节点存储的是"从它往前 lowbit(i) 个元素的和"。比如 tree[6]（二进制 110），lowbit(6)=2，它存储的是 arr[5] + arr[6]。画一张树状图出来，对照 update 和 query 的路径走一遍，比看十行文字管用。

区域和检索（LeetCode 307）

这道题要求实现一个 NumArray 类，支持更新某个下标的值和查询区间和。这正是树状数组的经典应用。

class NumArray:
            def __init__(self, nums):
                self.nums = nums
                self.n = len(nums)
                self.bit = FenwickTree(self.n)
                for i, val in enumerate(nums):
                    self.bit.update(i + 1, val)

            def update(self, index: int, val: int) -> None:
                delta = val - self.nums[index]
                self.nums[index] = val
                self.bit.update(index + 1, delta)

            def sumRange(self, left: int, right: int) -> int:
                return self.bit.range_sum(left + 1, right + 1)

注意树状数组一般是 1-indexed 的，所以传入下标时要加 1。这个细节容易忘，写完后检查一遍。

逆序对（剑指 Offer 51）

统计数组中的逆序对数量——如果 i < j 且 nums[i] > nums[j]，则 (i, j) 是一个逆序对。这是一道综合运用离散化 + 树状数组的经典题目。

def reverse_pairs(nums):
            # 离散化：将数值映射到排名
            sorted_nums = sorted(set(nums))
            rank = {val: idx + 1 for idx, val in enumerate(sorted_nums)}

            n = len(sorted_nums)
            bit = FenwickTree(n)
            count = 0

            # 从右往左遍历
            for i in range(len(nums) - 1, -1, -1):
                r = rank[nums[i]]
                # 查询比当前数小的数的个数（已遍历过的）
                count += bit.query(r - 1)
                bit.update(r, 1)

            return count

思路：从右往左遍历，树状数组维护"已经遍历过的元素中，每个排名出现的次数"。对于当前元素，查询所有比它小的排名出现次数之和——这些就是能和它组成逆序对的右边元素。离散化的原因是树状数组的下标需要从 1 开始连续排列。

套路总结：逆序对 = 遍历 + 树状数组计数。"从右往左"是标准模式，当然从左往右也可以——那就等价于统计"左边比当前大的元素个数"，两者是等价的。

线段树基础

如果树状数组是"区间问题的瑞士军刀"，那线段树（Segment Tree）就是"区间问题的重型工程机械"。它比树状数组更强大——不仅支持区间查询和区间更新（配合懒标记），还能处理各种自定义的区间聚合操作（求和、最大值、最小值、最大子段和等）。

线段树的基本思想：把数组不断二分，递归地构建一棵二叉树。每个节点代表一个区间，存储该区间的聚合信息。叶子节点对应单个元素。

建树、查询、更新

我们以求区间和为例，实现线段树的三个基本操作：

class SegmentTree:
            def __init__(self, nums):
                self.n = len(nums)
                self.tree = [0] * (4 * self.n)
                self._build(nums, 1, 0, self.n - 1)

            def _build(self, nums, node, l, r):
                if l == r:
                    self.tree[node] = nums[l]
                    return
                mid = (l + r) // 2
                self._build(nums, node * 2, l, mid)
                self._build(nums, node * 2 + 1, mid + 1, r)
                self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]

            def _update(self, node, l, r, idx, val):
                if l == r:
                    self.tree[node] = val
                    return
                mid = (l + r) // 2
                if idx <= mid:
                    self._update(node * 2, l, mid, idx, val)
                else:
                    self._update(node * 2 + 1, mid + 1, r, idx, val)
                self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]

            def _query(self, node, l, r, ql, qr):
                if ql <= l and r <= qr:
                    return self.tree[node]
                mid = (l + r) // 2
                res = 0
                if ql <= mid:
                    res += self._query(node * 2, l, mid, ql, qr)
                if qr > mid:
                    res += self._query(node * 2 + 1, mid + 1, r, ql, qr)
                return res

注意这里用了递归实现。对于 LeetCode 题目来说，递归的线段树通常够用了。如果数据范围很大（比如 10⁶ 以上），可能需要用迭代式线段树或 zkw 线段树来避免递归栈溢出。

Python 递归深度问题：Python 默认递归深度限制是 1000。如果数组长度超过 1000，递归建树就可能报 RecursionError。可以用 sys.setrecursionlimit(10**6) 增加限制，或者改用迭代实现。这要记住——我在 LeetCode 上至少见过三次因为这个问题导致 Runtime Error 的提交。

区间修改 + 懒标记

树状数组做区间更新需要差分技巧，而线段树支持原生的区间更新——通过懒标记（Lazy Propagation）。懒标记的核心思想：更新一个区间时，不立即更新所有子节点，而是在完全覆盖的节点上打一个标记，等后续查询需要时才下推。

class SegmentTreeLazy:
            def __init__(self, nums):
                self.n = len(nums)
                self.tree = [0] * (4 * self.n)
                self.lazy = [0] * (4 * self.n)
                self._build(nums, 1, 0, self.n - 1)

            def _push(self, node, l, r):
                if self.lazy[node] != 0:
                    mid = (l + r) // 2
                    # 下推到左右孩子
                    self.tree[node * 2] += self.lazy[node] * (mid - l + 1)
                    self.lazy[node * 2] += self.lazy[node]
                    self.tree[node * 2 + 1] += self.lazy[node] * (r - mid)
                    self.lazy[node * 2 + 1] += self.lazy[node]
                    self.lazy[node] = 0

            def _range_add(self, node, l, r, ql, qr, val):
                if ql <= l and r <= qr:
                    self.tree[node] += val * (r - l + 1)
                    self.lazy[node] += val
                    return
                self._push(node, l, r)
                mid = (l + r) // 2
                if ql <= mid:
                    self._range_add(node * 2, l, mid, ql, qr, val)
                if qr > mid:
                    self._range_add(node * 2 + 1, mid + 1, r, ql, qr, val)
                self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]

            def _range_query(self, node, l, r, ql, qr):
                if ql <= l and r <= qr:
                    return self.tree[node]
                self._push(node, l, r)
                mid = (l + r) // 2
                res = 0
                if ql <= mid:
                    res += self._range_query(node * 2, l, mid, ql, qr)
                if qr > mid:
                    res += self._range_query(node * 2 + 1, mid + 1, r, ql, qr)
                return res

懒标记的关键就一句话：用到才下推，不用就不推。每个节点上的 lazy 值代表"这个区间的所有元素还要额外加多少"。查询或更新到子节点前，先把懒标记推下去，确保子节点的值是正确的。

离散化

很多区间题目的数据范围非常大（比如坐标范围 10⁹），但实际用到的坐标点却很少（比如只有 10⁵ 个）。这时候需要离散化（Coordinate Compression）——把稀疏的坐标映射到紧凑的连续整数上。

def discrete(points):
            """离散化坐标列表，返回映射后的列表和排名字典"""
            sorted_unique = sorted(set(points))
            rank = {val: idx for idx, val in enumerate(sorted_unique)}
            return [rank[p] for p in points], rank

离散化的核心就是排序 + 去重 + 哈希映射。看似简单，但要注意两点：一是排序前一定要去重，否则映射不是一一对应的；二是如果题目涉及区间或长度，要小心相邻坐标之间是否留有"空隙"。

什么时候用离散化？当题目说"数据范围很大（10⁹），但实际出现的数字很少（10⁵）"时，就考虑离散化。典型场景：天际线问题、我的日程安排、区间合并类。

我的日历 I（LeetCode 729）

题目：实现一个 MyCalendar 类，支持 book(start, end) 操作——如果新事件与已有事件不重叠则成功添加并返回 True，否则返回 False。

最简单的办法是用有序集合（如 sortedcontainers），但 LeetCode 不支持第三方库。用线段树 + 离散化是一种经典解法：

class MyCalendar:
            def __init__(self):
                # 因为只需检测是否重叠，可以用二分查找维护有序区间列表
                self.intervals = []

            def book(self, start: int, end: int) -> bool:
                # 二分查找插入位置
                import bisect
                i = bisect.bisect_left(self.intervals, (start, end))
                # 检查与前一个区间是否重叠
                if i > 0 and self.intervals[i-1][1] > start:
                    return False
                # 检查与后一个区间是否重叠
                if i < len(self.intervals) and self.intervals[i][0] < end:
                    return False
                self.intervals.insert(i, (start, end))
                return True

这道题用二分法比线段树更简洁。但如果你要处理更复杂的情况——比如区间加减、区间查询——线段树就是不可替代的了。

天际线问题（LeetCode 218）

这道 Hard 题综合了离散化、线段树（或扫描线）和排序。题目要求输出所有建筑物的轮廓线关键点。完整代码较长，这里给出核心思路：

  • 将所有建筑物的左右边界坐标收集起来，做离散化
  • 用线段树（最大值的区间更新）在每个建筑物的区间上更新最大高度
  • 最后遍历所有离散化后的坐标，用线段树查询每个点的高度，高度变化的位置就是关键点

当然，扫描线 + 最大堆是更常见的解法。但线段树 + 离散化提供了另一种视角——把天际线问题理解为"区间取最大值"操作——这在线段树的训练中非常有价值。

对比总结

特性 树状数组 线段树
代码量 很短（~10 行核心） 较长（~50 行核心）
时间复杂度 O(log n) 每次操作 O(log n) 每次操作
空间复杂度 O(n) O(4n) ~ O(n)
单点更新 + 前缀查询 ✅ 原生支持 ✅ 支持
区间更新 + 区间查询 ⚠️ 需差分技巧 ✅ 懒标记原生支持
非求和操作（max/min） ⚠️ 仅前缀 max ✅ 任意可结合操作
常数因子 极小 较大（递归开销）
易用性 容易 中等
选择建议：能用树状数组就不要用线段树。树状数组代码短、常数小、不易出错。只有当你需要区间更新（懒标记）或非求和类聚合操作时，才出动线段树。记住这个经验法则："树状数组是工具，线段树是武器"——日常用工具，大场面才出武器。

踩坑与总结

线段树的 4 倍空间

为什么线段树需要 4n 的空间？因为线段树是一棵完全二叉树（虽然不一定是满二叉树）。对于长度为 n 的数组，最坏情况下树的高度是 ceil(log2(n)) + 1，节点总数 < 4n。开 4 * n 是一个安全的经验值。但如果你的递归实现在访问子节点时不做边界检查，4n 是必须的——少一个都不行。

我见过有人在开数组时用了 2 * n 结果在 n=100 时能过、n=10000 时疯狂 IndexError——这种 bug 非常难找。我的建议是一律开 4n，不要省这点空间。

递归实现的性能瓶颈

Python 的递归线段树在 n = 10⁵、操作次数 q = 10⁵ 的情况下通常能过。但如果 n = 10⁶、q = 10⁶，递归的栈开销和函数调用开销会非常明显。此时应该考虑：

  • 使用迭代式线段树（zkw 线段树）
  • 优先用树状数组（如果题目场景适合）
  • 改用 PyPy 提交（递归优化更好）
真实踩坑经历：有一次我用 Python 写线段树做区间加 + 区间求和的题目，本地测试全过，交到 LeetCode 上显示 TLE。我查了半天发现每次递归都传了 l, r, ql, qr 四个参数，递归层数一深，Python 的函数调用开销直接爆炸。后来把递归改成了迭代，耗时从 2.8s 降到了 0.3s。所以如果你发现 Python 线段树卡常，先尝试把递归改为迭代。
