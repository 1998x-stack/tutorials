第14章 正交多项式与数值积分

14.1 三项递推公式

所有正交多项式家族都满足一个优美的三项递推关系，这是数值计算它们的核心工具：

φ_{k+1}(x) = (x - α_k) φ_k(x) - β_k φ_{k-1}(x)

其中：
α_k = (x φ_k, φ_k) / (φ_k, φ_k)
β_k = (φ_k, φ_k) / (φ_{k-1}, φ_{k-1})   (k ≥ 1)
β_0 = 0

为什么这个递推很重要？因为直接使用单项式基底（monomial basis）$1, x, x^2, \dots$ 来构造多项式会在数值上产生灾难性后果——Hilbert 矩阵是严重病态的！而三项递推在数值上非常稳定，是我们实际计算正交多项式的标准路径。四大经典正交多项式的递推系数如下：

名称区间权函数 $w(x)$$\alpha_k$$\beta_k$
Legendre $P_k$$[-1,1]$$1$$0$$k^2/(4k^2-1)$
Chebyshev $T_k$$[-1,1]$$1/\sqrt{1-x^2}$$0$$1/2$ ($k=1$), $1/4$ ($k\ge2$)
Laguerre $L_k$$[0,\infty)$$e^{-x}$$2k+1$$k^2$
Hermite $H_k$$(-\infty,\infty)$$e^{-x^2}$$0$$k/2$

拿 Legendre 多项式举例，标准化的三项递推是 $(k+1)P_{k+1}(x) = (2k+1)xP_k(x) - kP_{k-1}(x)$。下面是用三项递推计算 Legendre 多项式的 Python 代码：

import numpy as np

def legendre_vals(x, n):
    """用三项递推计算 P_0(x) 到 P_n(x)"""
    phi = np.zeros((n+1,) + np.shape(x))
    phi[0] = 1.0
    if n >= 1:
        phi[1] = x
    for k in range(1, n):
        phi[k+1] = ((2*k+1) * x * phi[k] - k * phi[k-1]) / (k+1)
    return phi

# 测试：在 x=0 处的 Legendre 多项式值
x = 0.0
vals = legendre_vals(x, 6)
print("Legendre 多项式在 x=0 处的值:")
for k in range(7):
    print(f"P_{k}(0) = {vals[k]:.6f}")
# 期望: P_0=1, P_1=0, P_2=-1/2, P_3=0, P_4=3/8, P_5=0, P_6=-5/16
永远不要用显式公式或 Rodrigues 公式直接计算正交多项式的值！三项递推在数值上稳定得多。如果你要计算 Legendre 多项式的零点（也就是 Gauss-Legendre 的节点），请直接用 NumPy 的 numpy.polynomial.legendre.leggauss——它底层使用三项递推构造 Jacobi 矩阵并求其特征值，这是目前最稳定的方法。

14.2 正交多项式的零点性质

正交多项式的零点具有以下优美性质——这些性质对 Gauss 求积节点选择至关重要：

  • 实根性：$n$ 次正交多项式的所有 $n$ 个零点都是实数。
  • 区间性：所有零点都位于正交区间的内部，不会落在端点上。
  • 单根性：所有零点都是单根（代数重数为 1）。
  • 交错性：$P_n(x)$ 和 $P_{n+1}(x)$ 的零点互相交错——$P_{n+1}$ 的每两个相邻零点之间恰好有一个 $P_n$ 的零点。
为什么零点性质重要？交错性保证了 Gauss 求积节点在区间内分布良好，不会聚在一起。而且零点在区间内的分布密度与权函数 $w(x)$ 有关——权函数大的地方节点更密集，权函数小的地方节点较稀疏。这相当于"自适应"地分配了计算资源！比如 Chebyshev 多项式的零点在端点附近更密集，正好补偿了 Chebyshev 权函数 $1/\sqrt{1-x^2}$ 在端点附近的奇异性。

14.3 Gram-Schmidt 正交化

如果我们需要对一组给定的基底（比如 $1, x, x^2, \dots$）做正交化处理，Gram-Schmidt 过程是最直接的方法。但这里的坑比想象的多。

经典 Gram-Schmidt (CGS)：对 $k=0,1,\dots,n$，依次将 $x^k$ 减去它在前 $k-1$ 个已正交向量上的投影。

修正 Gram-Schmidt (MGS)：每次减去一个投影分量后立即用差值更新后续计算，而不是一次性减去所有投影。

import numpy as np

def modified_gs(A):
    """修正 Gram-Schmidt 正交化"""
    m, n = A.shape
    Q = np.zeros_like(A)
    R = np.zeros((n, n))
    for k in range(n):
        Q[:, k] = A[:, k]
        for j in range(k):
            R[j, k] = np.dot(Q[:, j], Q[:, k])
            Q[:, k] -= R[j, k] * Q[:, j]
        R[k, k] = np.linalg.norm(Q[:, k])
        Q[:, k] /= R[k, k]
    return Q, R

def classical_gs(A):
    """经典 Gram-Schmidt (数值不稳定版本)"""
    m, n = A.shape
    Q = np.zeros_like(A)
    for k in range(n):
        Q[:, k] = A[:, k].copy()
        Qk = Q[:, k].copy()  # 保存原始向量
        for j in range(k):
            R_jk = np.dot(Q[:, j], Qk)
            Q[:, k] -= R_jk * Q[:, j]
        Q[:, k] /= np.linalg.norm(Q[:, k])
    return Q

# 构造一个近线性相关的矩阵
np.random.seed(42)
A = np.random.randn(100, 20)
# 让某些列近似线性相关
A[:, 10] = A[:, 5] + 1e-6 * np.random.randn(100)

Q_cgs = classical_gs(A)
Q_mgs, _ = modified_gs(A)

I = np.eye(20)
err_cgs = np.linalg.norm(Q_cgs.T @ Q_cgs - I)
err_mgs = np.linalg.norm(Q_mgs.T @ Q_mgs - I)
print(f"CGS 正交性误差: {err_cgs:.2e}")
print(f"MGS 正交性误差: {err_mgs:.2e}")

上面的代码会显示：CGS 的误差可能高达 $10^{-5}$ 甚至更大，而 MGS 的误差接近机器精度。这是因为 CGS 中，对向量 $a_k$ 减去所有投影时，数值舍入误差会累积——你在用已经损失了正交性的向量去做投影，误差被"固化"了。而 MGS 每减去一个分量就立即更新，避免了这种累积。

踩坑警告——经典 GS 的数值不稳定性：经典 Gram-Schmidt 在数值上非常不稳定！当基底接近线性相关时，CGS 计算出的向量可能会严重偏离正交。我亲眼见过有人用 CGS 做特征值计算，结果算出来的"正交"基底之间的内积高达 0.1——这哪叫正交！而修正 Gram-Schmidt 的稳定性好得多（虽然也不及 Householder QR）。在新代码中永远用修正 GS 而不是经典 GS；更推荐的做法是使用 Householder 反射或 Givens 旋转。

14.4 正交多项式与 Gauss 求积的联系

终于到了本章的高潮——回答一个基础性问题：为什么正交多项式的零点是 Gauss 求积的最优节点？

考虑 $n+1$ 个节点 $x_0, \dots, x_n$ 的求积公式。我们希望它对所有 $2n+1$ 次多项式精确成立。神奇的是，只要选节点为 $n+1$ 次正交多项式 $\phi_{n+1}$ 的零点，再选取适当的求积权重 $w_i$，这个公式就自动满足 $2n+1$ 次代数精度。证明如下：

给定任意 $2n+1$ 次多项式 $p(x)$，将其除以 $n+1$ 次正交多项式 $\phi_{n+1}(x)$，做带余除法：

p(x) = q(x) · φ_{n+1}(x) + r(x)

其中 $\deg(q) \le n$，$\deg(r) \le n$。两边用权函数 $w(x)$ 积分：

∫ p(x) w(x) dx = ∫ q(x) φ_{n+1}(x) w(x) dx + ∫ r(x) w(x) dx

关键来了：$\phi_{n+1}$ 与所有次数 $\le n$ 的多项式正交，所以右边第一项为零。因此：

∫ p(x) w(x) dx = ∫ r(x) w(x) dx

现在，在求积节点 $x_i = \phi_{n+1}$ 的零点处，$\phi_{n+1}(x_i) = 0$，所以 $p(x_i) = r(x_i)$。于是求积公式的数值结果为：

Σ w_i p(x_i) = Σ w_i r(x_i)

由于 $r(x)$ 次数 $\le n$，$n+1$ 个节点的插值型求积公式能精确积分 $r(x)$（只要权重 $w_i$ 取为插值权重）。因此公式对 $p(x)$ 也精确成立。证毕。

证明的优美之处：通过带余除法将高次多项式问题转化为低次问题。正交性"杀死"了 $\phi_{n+1}$ 部分，剩下部分恰好可以在求积节点处用插值精确计算。这就是 Gauss 求积能达到 $2n+1$ 次代数精度的根本原因——正交多项式的零点完美地将高次多项式简化为了低次多项式。回想第 12 章我们直接用了 Gauss-Legendre 节点，现在你明白这些节点为什么"神奇"了吧。

为了直观地验证这一点，我们做一个数值实验——对比 Gauss 公式对 $2n+1$ 次和 $2n+2$ 次多项式的表现：

import numpy as np
from numpy.polynomial.legendre import leggauss

def gauss_legendre_quad(f, a, b, n):
    """n 点 Gauss-Legendre 求积"""
    x, w = leggauss(n)
    half = (b - a) / 2
    mid = (a + b) / 2
    return half * np.sum(w * f(half * x + mid))

# 测试 Gauss 公式对 2n+1 次多项式的精确性
print("测试 2n+1 次多项式（应精确到机器精度）:")
for n in range(1, 8):
    degree = 2 * n + 1
    poly = np.polynomial.Polynomial(np.random.randn(degree + 1))
    poly_int = poly.integ()
    exact = poly_int(1) - poly_int(0)
    approx = gauss_legendre_quad(poly, 0, 1, n)
    print(f"n={n}, degree={degree}: 误差={abs(exact-approx):.2e}")

print("\n测试 2n+2 次多项式（不应精确）:")
for n in range(1, 5):
    degree = 2 * n + 2
    poly = np.polynomial.Polynomial(np.random.randn(degree + 1))
    poly_int = poly.integ()
    exact = poly_int(1) - poly_int(0)
    approx = gauss_legendre_quad(poly, 0, 1, n)
    print(f"n={n}, degree={degree}: 误差={abs(exact-approx):.2e}")

输出会清晰地显示：对 $2n+1$ 次多项式误差接近机器精度（$10^{-15}$ 量级），而对 $2n+2$ 次多项式误差显著不为零（通常在 $10^{-1}$ 到 $10^{-3}$ 之间）。这就是代数精度的边界——Gauss 公式完美地做到了它能做到的最好。

总结——正交多项式在数值分析中的三重角色：
  • 逼近论（第7-8章）：正交多项式是最佳平方逼近的理想基底，使 Gram 矩阵对角化，计算量从 $O(n^3)$ 降为 $O(n)$；
  • 求积理论（第12章及本章）：正交多项式的零点是 Gauss 求积的最优节点，达到 $2n+1$ 次代数精度；
  • 谱方法：正交多项式展开（如 Chebyshev 谱方法）是求解偏微分方程的高精度工具，误差随 $n$ 指数衰减（称为"谱收敛"），是计算流体力学的核心方法之一。

本章我们系统学习了正交多项式的三大支柱——三项递推、零点性质和 Gram-Schmidt 正交化，并给出了正交多项式与 Gauss 求积之间深刻联系的一个完整证明框架。从第 8 章的最佳平方逼近到第 12 章的 Gauss 求积，再到本章的理论基础，正交多项式像一条红线贯穿了整个数值分析。下一章我们将进入数值线性代数的领域——从 Gauss 消去法与 LU 分解开始，学习如何高效、稳定地求解线性方程组。
