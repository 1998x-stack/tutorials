第五章：词向量

大家好！这一章是NLP深度学习时代的"开山之作"——词向量（Word Embedding）。2013年Word2Vec的提出标志着NLP从统计时代进入神经网络时代。我第一次读到论文时，看到"king - man + woman ≈ queen"这个结果，脊背发凉——机器居然自己学到了语义关系。

先给大家一个概念定位：词向量是分布式表示思想的具体实现，用低维稠密向量来表示词，使得语义相似的词在向量空间中距离接近。它不是"存储"语义，而是通过训练让词汇的语义关系从向量几何中涌现出来。

5.1 Word2Vec：两大训练范式

Word2Vec（Mikolov et al. 2013）提出了两种训练架构，理解它们的区别非常重要：

Skip-gramCBOW
预测方向给定中心词 → 预测周围词给定周围词 → 预测中心词
训练速度较慢较快
低频词效果⭐ 更好较差
适用场景大规模语料、学术研究小规模语料、生产系统
数学直觉每个中心词"分发"信息给上下文上下文"投票"决定中心词
# Skip-gram with Negative Sampling 的训练过程
# 正样本：("king", "man") — 从语料中滑动窗口得到的真实词对
# 负样本：("king", "table") — 随机采样的假词对

# 训练目标：让正样本的相似度 > 负样本的相似度
# loss = -log(σ(v_center · v_positive)) - Σ log(σ(-v_center · v_negative))
# σ是sigmoid函数，把点积映射到(0,1)区间

Skip-gram对小语料的低频词效果更好，这也是为什么它在学术研究中更受欢迎——真实语料的词频分布是长尾的，大部分词都是低频词。

5.2 负采样与层次Softmax

Word2Vec在训练时面临一个计算瓶颈：原始softmax要对整个词汇表求概率，每次更新要算几十万次向量点积。Google团队提出了两个解决方案：

# 方法1：负采样 (Negative Sampling)
# 每次只更新正样本和K个负样本（K通常5~20）
# 复杂度从O(V)降到O(K)，V=词汇表大小

# 方法2：层次Softmax (Hierarchical Softmax)
# 用Huffman树组织词汇表，高频词在浅层
# 复杂度从O(V)降到O(log V)
# 对高频词更快，但对低频词需要走到深层

重要洞察：负采样之所以有效，不是因为近似本身有多精确，而是因为我们真正需要的是词向量，不是语言模型的概率。Word2Vec只是把语言模型训练当成了学习词向量的"借口"——这是它和NNLM最本质的区别。

5.3 词向量的神奇算术

词向量最著名的属性莫过于向量算术——这也是它"出圈"的原因：

# 经典例子
vec("king") - vec("man") + vec("woman") ≈ vec("queen")

# 更多例子
vec("Paris") - vec("France") + vec("Italy") ≈ vec("Rome")
vec("walking") - vec("walk") + vec("swim") ≈ vec("swimming")

# 这不是巧合！训练过程中，上下文相似的模式被迫在向量空间中形成几何关系
# "king"和"queen"的上下文只差在性别相关的词（he/she, his/her）
# 所以 king - man + woman 恰好逼近 queen

这个性质说明词向量不是简单地"记住"了语义，而是学到了语义的结构化关系。这种结构化是涌现出来的——训练时并没有显式地告诉模型要做向量算术。

5.4 GloVe：用全局共现矩阵

Word2Vec基于局部滑动窗口，每次只看一个窗口内的词对。GloVe（Global Vectors, Pennington et al. 2014）换了一个思路：先统计整个语料的词共现矩阵，再训练低维向量来"复现"这个矩阵。

# GloVe的训练目标
# X_{ij} = 词i和词j在语料中共现的次数
# 目标：找到向量 w_i 和 w_j 使得:
# w_i · w_j + b_i + b_j ≈ log(X_{ij})

# GloVe vs Word2Vec
# GloVe利用全局统计 + 局部上下文，数学更优雅
# Word2Vec完全靠随机梯度下降，工程上更简单
# 实际效果：在大语料上GloVe略好，中小语料Word2Vec更鲁棒

5.5 FastText：子词信息的力量

Word2Vec和GloVe都有一个共同盲点：把每个词当作原子单元。这意味着"run"和"running"在Word2Vec看来是完全独立训练的两个向量。FastText（Bojanowski et al. 2017）用子词（character n-gram）来解决这个问题：

# FastText 把每个词表示为字符n-gram向量的和
# "running" → [""]
# 其中<和>标记词边界

# 优势：
# 1. 未登录词（OOV）也能有向量——用字符n-gram拼出来
# 2. 拼写错误有容错性——"runnig"和"running"的n-gram有重叠
# 3. 词形变化（running/run/runs）共享子词，向量更接近

# 在形态丰富的语言（土耳其语、芬兰语）中，FastText远超Word2Vec

选型指南：做英文且语料充足 → GloVe；做中文或多语言 → FastText（中文用部首或笔画n-gram）；做学术研究 → Word2Vec Skip-gram；2026年的工程实践 → 直接用BERT/RoBERTa的上下文词向量，静态词向量只用于极端资源受限场景。

5.6 词向量的局限

说到局限，最大的问题就是一词多义。Word2Vec给"apple"只有一个向量——它到底是水果还是公司？答案只能是"看上下文"，但静态词向量没有上下文。

这个问题直接催生了ELMo和BERT（第11、12章）——让每个词的向量依赖它的上下文。可以说，整个预训练时代的起点，就是解决"apple到底怎么表示"这个看起来简单的问题。

好了，有了词向量这个利器，接下来的问题是：怎么处理整个句子？一串词向量怎么变成对整个句子的理解？这就是第6章RNN要解决的问题。明天见！
