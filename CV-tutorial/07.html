图像分类任务定义 — 问题形式化、类别、Top-1/Top-5准确率

图像分类（Image Classification）是计算机视觉最基础、最核心的任务。简单来说，它的目标是：给出一张输入图像，模型输出这张图像所属的类别标签。比如输入一张猫的照片，输出"猫"；输入一张交通信号灯的照片，输出"红绿灯"。看起来很简单，不是吗？但正是这个看似简单的任务，驱动了过去十年深度学习领域最重要的一系列突破。

让我们从数学上形式化这个问题：假设有C个类别（例如C=1000），模型f将输入图像x映射到一个C维的向量z = f(x)，其中z_i表示图像属于第i类的"得分"或logit。通常我们用Softmax函数将这些logits转换为概率：p_i = e^{z_i} / Σ_j e^{z_j}。最终的预测类别就是概率最大的那个：ŷ = argmax_i p_i。这就是"多分类"的标准范式。

评估分类模型的指标中最常用的两个是Top-1准确率和Top-5准确率。Top-1准确率就是模型预测概率最高的类别是否等于真实标签——这是最严格的评估方式。Top-5准确率则检查真实标签是否出现在模型预测概率最高的前5个类别中——这在类别数量非常大的场景（如ImageNet有1000个类别）中更为宽容，因为有时一个图像可能确实有多个合理的解释（例如一张"拉布拉多犬"的照片，模型预测为"金毛犬"虽然错了，但至少都在"犬"的范畴内）。在早期的ImageNet竞赛中，Top-5准确率是主要的评估指标——AlexNet在2012年将Top-5错误率从26%降到了15%，正是这一数字震撼了整个CV界。

讲一个个人经历：刚做分类项目时，我训练了一个10类分类器，在测试集上Top-1准确率只有82%。当时很沮丧——差一个百分点就85%了。但后来检查发现，很多"错误"其实是标注问题——比如一张模糊的、人类都难以区分的"猫vs狗"图像被错误标注了。在CV领域，这一点非常重要：模型"犯错"不一定真是模型的问题，而数据集本身也有不完美之处。

# Top-1 和 Top-5 准确率计算
import torch

def accuracy(output, target, topk=(1, 5)):
    """
    output: 模型输出, shape (batch, num_classes)
    target: 真实标签, shape (batch,)
    """
    maxk = max(topk)
    batch_size = target.size(0)

    # 获取 top-k 预测的索引
    _, pred = output.topk(maxk, 1, True, True)    # (batch, maxk)
    pred = pred.t()                                # (maxk, batch)
    correct = pred.eq(target.view(1, -1).expand_as(pred))

    results = []
    for k in topk:
        correct_k = correct[:k].reshape(-1).float().sum(0, keepdim=True)
        acc = correct_k.mul_(100.0 / batch_size)
        results.append(acc.item())

    return results

# 模拟: batch=4, 1000个类别
output = torch.randn(4, 1000)  # 模拟 ImageNet 输出
target = torch.randint(0, 1000, (4,))

top1, top5 = accuracy(output, target, topk=(1, 5))
print(f"Top-1 准确率: {top1:.2f}%")
print(f"Top-5 准确率: {top5:.2f}%")
print(f"Top-5 通常远高于 Top-1 (在 ImageNet 上 ~80% vs ~60%)")

经典数据集 — MNIST、CIFAR-10/100、ImageNet

图像分类领域的进步与经典数据集的发展密不可分。每个数据集都有自己的"定位"和"性格"，理解它们有助于你在不同研究或项目阶段做出合适的选择。

MNIST（1998年，由Yann LeCun等人发布）是CV界的"Hello World"。它包含70000张28×28的灰度手写数字图像（0-9共10类），其中60000张用于训练，10000张用于测试。MNIST的简单性让它成为测试新算法概念时的首选——如果在一个算法在MNIST上都表现不好，那它在更复杂的数据集上基本没戏。但反过来，在MNIST上表现好并不代表什么——今天的模型在MNIST上已经能达到99.7%以上的准确率，"MNIST已死"在学术界甚至是个梗。

CIFAR-10和CIFAR-100（2009年，由Alex Krizhevsky等人发布，没错就是AlexNet的那个Alex）是比MNIST更有挑战性的数据集。它们包含60000张32×32的彩色图像——CIFAR-10有10个类别（飞机、汽车、鸟、猫、鹿、狗、青蛙、马、船、卡车），每类6000张；CIFAR-100有100个类别，每类600张。32×32的分辨率虽然很小，但已经足够呈现物体的基本形状和纹理。CIFAR是快速验证CNN架构的黄金标准——几乎所有经典CNN架构都会报告CIFAR上的结果。当前最好的模型在CIFAR-10上能达到99%以上的准确率，CIFAR-100上约为95%。

ImageNet（2009年，由李飞飞团队发布）是推动深度学习革命的关键数据集。它拥有超过1400万张图像，涵盖21841个类别（最常用的子集ILSVRC包含1000个类别，约120万张训练图像）。ImageNet的规模之大、类别之丰富，让它成为衡量模型"真本事"的试金石。在ImageNet上取得突破，就意味着你的模型学到了真正的视觉知识，而不仅仅是记忆了一个小数据集的模式。2012年AlexNet在ImageNet上的突破性表现，直接开启了深度学习时代——这个历史意义我们再怎么强调也不为过。

ImageNet-21K是ImageNet的完整版本，包含约1400万张图像、21000多个类别。近年来，越来越多的预训练模型开始在ImageNet-21K上预训练，然后在ImageNet-1K或其他下游任务上微调。更大的类别空间让模型学到了更丰富的视觉概念，从而在迁移学习时表现更好。

数据加载与预处理Pipeline — 图像解码、Resize、Normalize、增强

一个工业级的数据加载pipeline需要考虑多个环节：从磁盘读取图像文件（图像解码）、调整尺寸（Resize）、归一化（Normalize）、数据增强（Data Augmentation）、批量化（Batching）和预读取（Prefetching）。任何一个环节有瓶颈，都会拖累整个训练速度。

图像解码是第一个环节。原始图像以JPEG、PNG等压缩格式存储在磁盘上。训练时，DataLoader的子进程读取图像文件、解码为像素数组。如果使用num_workers设置多个子进程，解码过程可以并行进行，避免阻塞GPU。在PyTorch中推荐使用torchvision.datasets.ImageFolder来加载自定义目录结构的图像数据集。

Resize将所有图像调整到相同的尺寸（如224×224），因为CNN要求固定尺寸的输入。Resize的方法有多种：最常用的是将短边缩放到目标尺寸然后中心裁剪（center crop），或者直接拉伸/压缩。对于长宽比差异很大的图像，直接拉伸会引入畸变，中心裁剪则更合理。

Normalize将像素值从[0, 255]缩放到均值为0、方差为1的分布。这可以加速收敛——如果输入像素值范围很广（0-255），而权重初始值在[-0.1, 0.1]之间，梯度可能会非常大或非常小，导致训练不稳定。标准做法是用数据集的全局均值和标准差做标准化。例如，ImageNet的标准化参数是mean=[0.485, 0.456, 0.406]和std=[0.229, 0.224, 0.225]。

数据增强是对训练图像施加随机变换（如翻转、旋转、裁剪、颜色抖动等），以生成"新的"训练样本。这能大幅提升模型泛化能力——一个做随机翻转的数据增强，相当于将训练数据量增加了一倍（因为每张图都有翻转和不翻转两个版本）。我们将在第23章中深入探讨这个话题。

训练循环实现 — 前向传播、损失计算、反向传播、参数更新

上一章我们已经给出了一个完整的训练循环代码。这里我们不重复全部代码，而是深入剖析训练循环中每个步骤的"为什么"——理解这些之后，你才能真正灵活地调整训练策略。

标准的训练循环包含5个步骤，形成了一次完整的"学习迭代"：

步骤1：清零梯度（optimizer.zero_grad()）。如果不做这一步，梯度会在不同batch间累积。这有时候是Bug，但有时候也是一种"梯度累积"技巧——当GPU显存有限、无法使用大batch size时，可以通过梯度累积来模拟大batch size的效果。

步骤2：前向传播（outputs = model(inputs)）。数据从输入层流过所有中间层，最终产生预测输出。这就是我们在第4章和第5章中详细讨论的内容。

步骤3：计算损失（loss = criterion(outputs, labels)）。把预测和真实标签交给损失函数，它告诉模型"你错得有多离谱"。交叉熵损失是最常用的分类损失。

步骤4：反向传播（loss.backward()）。PyTorch自动计算损失对于所有可训练参数的梯度。所有设置了requires_grad=True的参数的.grad属性会被更新。

步骤5：参数更新（optimizer.step()）。优化器根据梯度更新每个参数。不同的优化器（SGD、Adam等）使用不同的更新规则。

这些步骤之外，学习率调度（Learning Rate Scheduling）是一个经常被初学者忽视但极其重要的策略。常见的方法包括：StepLR（每N个epoch学习率乘以一个因子）、CosineAnnealing（学习率按余弦曲线衰减）、以及Warmup（先在头几个epoch从很小的学习率逐渐增加到目标学习率）。正确使用学习率调度往往能带来几个百分点的性能提升。

🔄 训练循环的心法：很多人把训练循环写成一段"一次性代码"——跑完就扔。但真正有经验的工程师会把它封装成可复用的模块。你会发现本章的训练函数和下一章、下下章的几乎一模一样——唯一变化的是模型和数据。这就是框架设计的力量：把不变的抽象出来，把变化的作为参数注入。我建议你从现在开始就维护一个自己的"train_utils.py"文件，把train_one_epoch、evaluate这些函数放进去。

评估指标 — Accuracy、Precision/Recall、F1、Confusion Matrix

准确率（Accuracy）虽然是分类任务最常用的指标，但它在某些场景下可能具有欺骗性。想象一个场景：一个医疗图像数据集里，99%的图像是"健康"的，只有1%是"患病"的。一个"永远预测健康"的傻瓜模型也能达到99%的准确率——但它完全没有任何实际价值。这就是所谓的"类别不平衡"问题。此时我们需要更精细的评估指标。

Precision（精确率）和Recall（召回率）从两个互补的角度评估模型在特定类别上的表现。精确率回答的是"模型预测为这一类别的样本中，有多大比例是真的属于这个类别的"；召回率回答的是"所有真正属于这一类别的样本中，模型找出来了多大比例"。高精确率意味着模型的判断"靠谱"（假阳性少），高召回率意味着模型"全面"（漏检少）。

F1 Score是精确率和召回率的调和平均数：F1 = 2 × (P × R) / (P + R)。它被视为精确率和召回率的一个折中指标——只有当二者都高时，F1才高。在类别不平衡场景中，F1比准确率更能反映模型的真实性能。

混淆矩阵（Confusion Matrix）是一个C×C的矩阵（C为类别数），其中位置(i, j)表示"真实类别为i但被模型预测为j"的样本数。对角线上的是正确预测的样本。混淆矩阵能让你一目了然地看到模型主要在哪些类别之间混淆——例如，模型可能经常把"狼"误识别为"哈士奇"，或者把"猫"误识别为"老虎"。这种信息对于后续改进模型非常有价值。

指标公式含义适用场景
Accuracy(TP+TN)/(TP+TN+FP+FN)整体预测正确比例类别均衡
PrecisionTP/(TP+FP)预测为正的样本中有多少是真正的正样本假阳性代价高（如垃圾邮件过滤）
RecallTP/(TP+FN)真正的正样本中有多少被找出来了假阴性代价高（如癌症筛查）
F1 Score2·P·R/(P+R)精确率和召回率的调和平均类别不平衡、追求综合性能
# 评估指标计算
import numpy as np
from sklearn.metrics import confusion_matrix, classification_report

def compute_metrics(y_true, y_pred, num_classes):
    """计算分类评估指标"""
    # 混淆矩阵
    cm = confusion_matrix(y_true, y_pred, labels=range(num_classes))

    # 每个类别的精确率、召回率、F1
    metrics = {}
    for c in range(num_classes):
        tp = cm[c, c]
        fp = cm[:, c].sum() - tp
        fn = cm[c, :].sum() - tp
        tn = cm.sum() - (tp + fp + fn)

        precision = tp / (tp + fp + 1e-10)
        recall    = tp / (tp + fn + 1e-10)
        f1        = 2 * precision * recall / (precision + recall + 1e-10)
        accuracy  = (tp + tn) / (tp + tn + fp + fn)

        metrics[chr(65+c)] = {  # A, B, C...
            'precision': precision,
            'recall': recall,
            'f1': f1,
            'accuracy': accuracy
        }

    return cm, metrics

# 模拟一个三分类问题的评估结果
y_true = np.array([0, 1, 2, 0, 1, 2, 0, 0, 1, 1])
y_pred = np.array([0, 1, 2, 0, 2, 2, 0, 1, 1, 0])

cm, metrics = compute_metrics(y_true, y_pred, 3)
total_acc = (y_true == y_pred).mean()

print(f"总体准确率: {total_acc:.2%}")
print(f"\n混淆矩阵:")
print(cm)
print(f"\n各类别指标:")
for cls, m in metrics.items():
    print(f"  类别 {cls}: Precision={m['precision']:.2f}, "
          f"Recall={m['recall']:.2f}, F1={m['f1']:.2f}")
⚠️ 踩坑记录：在做生产环境下的CV项目时，我遇到过一个案例：模型在测试集上的准确率是95%，但我们总觉得哪里不对。一画混淆矩阵才发现——模型把所有的"安全帽"都正确分类了，但把"安全背心"全部识别成了"安全帽"。原因是训练数据中"安全背心"的图片都只有半身照（上半身穿背心），而"安全帽"的图片都包含完整的上半身。模型学到的不是"安全帽vs安全背心"的区别，而是"上半身vs仅头部"的区别。这个教训告诉我们：混淆矩阵和数据分析，远比一个简单的准确率数字重要。

从原型到生产 — 实验管理、日志记录、Checkpoint

在学术研究或工业项目中，你不会只跑一次训练就结束。你会尝试不同的学习率、不同的网络深度、不同的数据增强策略——你需要一个系统来管理这些实验，记录每个实验的超参数和结果，以便对比分析。这就是"从原型到生产"的关键一步。

实验管理的核心是"可复现性"——每一个实验的代码、超参数、数据、随机种子都应该被完整记录。最简单的做法是为每个实验创建一个独立的目录，保存一份超参数配置文件（如YAML或JSON）和最终的模型权重。TensorBoard是训练过程可视化的标准工具——它记录每个epoch的loss和准确率曲线，让你直观地看到训练的进展。更现代化的选择是Weights & Biases（wandb），它提供了云端实验管理、超参数对比、团队协作等功能。

日志记录不容忽视。在训练日志中，至少应该包含：当前epoch/总epoch数、当前学习率、训练loss/准确率、验证loss/准确率、每个epoch的耗时。一个格式良好的日志是调试问题的最有力工具——当模型不收敛时，查看loss曲线的形状就能帮你判断是学习率太大、梯度爆炸、还是数据有问题。

Checkpoint（模型检查点）是训练过程中定期保存的模型状态。一份好的checkpoint至少包含：model_state_dict（模型参数）、optimizer_state_dict（优化器状态，让你能从断点恢复训练）、epoch（当前训练轮数）和best_val_acc（最优验证准确率）。记得保存最优模型（validation上表现最好的）和最新模型（方便断点续训）两份checkpoint。

# 实验管理模板
import os
import json
import torch

def save_checkpoint(state, filename='checkpoint.pth.tar'):
    """保存检查点"""
    torch.save(state, filename)

def load_checkpoint(model, optimizer=None, filename='checkpoint.pth.tar'):
    """加载检查点 (支持断点续训)"""
    checkpoint = torch.load(filename)
    model.load_state_dict(checkpoint['model_state_dict'])

    if optimizer is not None:
        optimizer.load_state_dict(checkpoint['optimizer_state_dict'])

    epoch = checkpoint.get('epoch', 0)
    best_acc = checkpoint.get('best_acc', 0.0)
    return epoch, best_acc
