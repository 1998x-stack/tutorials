第20章 小目标检测优化

大家好，我是你们的讲师。在遥感图像中检测车辆、在医疗影像中检测微小病灶、在无人机航拍中检测行人——这些场景里，目标可能只有十几个像素。小目标检测是YOLO从诞生之日起就存在的一个软肋。COCO数据集中，小目标（面积 < 32² 像素）的检测mAP至今仍不到大目标的一半。而且，几乎所有声称在小目标上取得突破的学术研究，到了真实场景中往往效果打折——这是一个理论说得多、实战做得少的领域。这一章我们不谈"绝对最优解"，而是给出一套在工程上真正有效的技术组合。

20.1 为什么YOLO在小目标上表现差

要解决小目标检测问题，首先要理解问题的根源。YOLO对小目标检测效果差的原因是多方面的：

  • 下采样过度：YOLO的输入图像经过5次下采样（步长2，共32倍），原始图像中16×16像素的小目标在特征图上只剩0.5×0.5个像素——这意味着小目标的信号几乎在深层特征图中消失了。这是最根本的原因。
  • 锚框不匹配（针对anchor-based版本）：YOLOv5的锚框聚类倾向于大中目标，小目标对应的锚框数量少且匹配率低。虽然YOLOv8使用anchor-free缓解了这个问题，但底层特征的缺乏依然是瓶颈。
  • 正样本稀疏：小目标在图像中的有效特征点少，导致参与训练的正样本数量远少于中大目标，模型对小目标的关注度不够。
  • NMS误杀：小目标的检测框小，相邻小目标框之间的IoU往往较高（即使它们独立存在），NMS阈值设置不当会直接清洗掉大量正确的小目标检测框。

认知纠正：很多人以为小目标检测的瓶颈在"模型"，但其实更大的瓶颈在"数据"和"标注"。COCO数据集中小目标的标注质量参差不齐——很多小目标根本没有被标注（称为"缺失标注"）。如果你的训练数据中小目标标注就不全，模型不可能学好。在实际项目中，先确保小目标的标注质量，再考虑算法优化，这个顺序不能颠倒。

20.2 多尺度训练

多尺度训练是提升小目标检测最直接、最有效的方法。核心思路很简单：提高输入分辨率，让小目标在特征图上占据更多像素。

YOLO原生支持多尺度训练——在训练过程中，每隔一定步数（默认10个batch）随机选择一个输入尺寸（从0.5×到1.5×基准尺寸）。但这个方法对显存要求较高，而且YOLO的默认设置对多尺度的利用并不充分。

针对小目标的优化做法是：将基准输入分辨率从640×640提升到1280×1280或1536×1536。YOLOv5和YOLOv8官方代码都支持大尺寸训练模式（通过--img 1280参数启用）。需要注意的是，分辨率翻倍意味着计算量翻四倍，需要根据显存限制调整batch size。

分辨率选择建议：不要盲目使用最高分辨率。我的经验法则是——目标的最小尺寸（像素）除以分辨率应该大于特征图的有效感受野步长（即 目标尺寸 / 输入分辨率 > 0.01）。例如，如果你的最小目标为10像素，使用640分辨率（10/640=0.0156）是足够的；但如果最小目标只有5像素，则需要1280分辨率（5/1280=0.0039，仍然太小）甚至更高。一个实用的测试流程是：先用640分辨率训练，用测试集检查小目标recall，如果低于30%，再逐步提升分辨率。

20.3 特征增强：注意力机制与小目标

第二种有效的方法是增强特征金字塔对小目标信息的捕捉能力。YOLO标准架构中的FPN/PANet在设计中大目标时效果很好，但小目标的特征经过多层传递后已经衰减严重。

更深更宽的FPN。在YOLOv8中，FPN的P3层（对应8倍下采样）是检测小目标的主要层级。一个直观的优化是增加P2层（对应4倍下采样）或更浅的特征图。YOLOv8中的--optimize模式如果启用P2输出，可以提升小目标3-5%的recall，代价是推理速度下降15-20%。

注意力增强的特征融合。在FPN的特征融合阶段引入通道注意力（如SE或ECA模块）可以增强小目标特征通道的权重。具体做法是在FPN的每个上采样/下采样融合节点后插入一个轻量级的注意力模块，让网络自动学习哪些通道对小目标更重要。

# 在FPN中插入注意力增强的特征融合
class AttentionFPN(nn.Module):
    """带通道注意力增强的FPN融合模块"""
    def __init__(self, channels):
        super().__init__()
        # 标准FPN融合：上采样 + 拼接 + 卷积
        self.upsample = nn.Upsample(scale_factor=2, mode='nearest')
        self.conv = Conv(channels * 2, channels, 1)

        # 通道注意力模块（SE-style）
        self.se = nn.Sequential(
            nn.AdaptiveAvgPool2d(1),
            nn.Conv2d(channels, channels // 4, 1),
            nn.ReLU(),
            nn.Conv2d(channels // 4, channels, 1),
            nn.Sigmoid(),
        )

    def forward(self, x_low, x_high):
        # x_low: 深层语义特征（低分辨率）
        # x_high: 浅层细节特征（高分辨率）
        x = torch.cat([self.upsample(x_low), x_high], dim=1)
        x = self.conv(x)

        # 通道注意力加权
        weight = self.se(x)
        x = x * weight + x  # 残差连接，保留原始信息

        return x

20.4 Tiling / SAHI：切片检测

当分辨率提升和特征增强都达到瓶颈后，切片检测（Tiling / SAHI）是最后的王牌。它的思路直观到令人惊讶：把大图切成多张小图，每张单独检测，再合并结果。

SAHI（Slicing Aided Hyper Inference）是专门为小目标检测设计的切片推理方法。它的流程是：将输入大图按一定重叠比例切分成多个子图（如640×640），对每个子图独立用YOLO推理，再用NMS合并所有子图的重叠检测结果。

SAHI的好处是：在不改变模型的情况下，将小目标的"有效分辨率"提升了切片数量的倍数。一个4000×4000的遥感图像切成49个640×640的子图（7×7网格），每个子图中的小目标像素占比被放大了约40倍。代价是推理次数增加了49倍——但这是一个可以并行加速的过程。

踩坑案例：我们团队在一个光伏板缺陷检测项目中使用SAHI。客户发来的图像分辨率是6000×4000，我们切成640×640的子图（重叠率20%）后分别检测。结果发现切边上的缺陷被重复检测，而跨切片的缺陷（被切成两半的）两边都无法完整检出。我们的解决方案有两个：重叠率从20%提高到50%（缓解了边界问题），以及后处理时对置信度低于阈值的相邻检测框做"合并"而非"抑制"。最终recall从单个完整图检测的21%提升到了74%。

# SAHI切片检测的简化实现
def sahi_infer(model, image, slice_size=640, overlap_ratio=0.2):
    """
    SAHI切片推理主流程
    model: YOLO模型
    image: 原始大图 (H, W, 3)
    """
    h, w = image.shape[:2]
    stride = int(slice_size * (1 - overlap_ratio))
    all_detections = []

    # Step 1: 切片
    for y in range(0, h, stride):
        for x in range(0, w, stride):
            # 裁剪子图（确保不超过边界）
            y1, y2 = y, min(y + slice_size, h)
            x1, x2 = x, min(x + slice_size, w)
            patch = image[y1:y2, x1:x2]

            # Step 2: 对每个子图推理
            results = model(patch)
            boxes = results.boxes.data.cpu().numpy()

            # Step 3: 将子图坐标映射回原图
            for box in boxes:
                box[0] += x1  # x1坐标偏移
                box[1] += y1  # y1坐标偏移
                box[2] += x1  # x2坐标偏移
                box[3] += y1  # y2坐标偏移
                all_detections.append(box)

    # Step 4: 全局NMS合并重复检测
    final_boxes = nms(np.array(all_detections), iou_threshold=0.5)
    return final_boxes

20.5 数据级优化策略

架构和推理层面的优化之后，数据层面的策略同样关键——而且往往是性价比最高的方向。

小目标过采样。在训练时，对包含小目标的图像提高采样权重。COCO数据集的图像——小目标分布极度不均，20%的图像包含了80%的小目标。通过设置一个"小目标密度"指标（每张图像中小目标占比），对高密度图像过采样2-3倍，可以大幅提升小目标的训练信号。

Copy-Paste数据增强。这是一种对小目标特别有效的增强方法：从一张图中随机复制小目标实例，粘贴到另一张图中的随机位置。需要注意：粘贴时避免覆盖已有目标，对粘贴目标的位置做合理的上下文放置（比如车不要贴到天上），同时对粘贴目标进行微小的缩放和旋转变换。YOLOv8内置的Mosaic增强对小目标有一定帮助，但Copy-Paste更直接——它解决了小目标"样本不足"的核心问题。

优化方法 实现难度 小目标mAP提升 对中大目标影响 推理开销 推荐指数
提升输入分辨率 低（修改参数） +5~10% 略有提升 高（计算量×4） ★★★★★
P2特征层输出 中（修改模型配置） +3~5% 基本无影响 中（15-20%） ★★★★
注意力特征增强 中（新增模块） +2~4% 略有提升 低（5-10%） ★★★★
SAHI切片推理 中（后处理逻辑） +10~30% 提升较小 高（×切片数） ★★★（特定场景）
Copy-Paste增强 中（数据加载修改） +3~8% 可能略有下降 无（训练时） ★★★★★

实战建议：不要一次性使用所有优化方法，而是按投入产出比排序实施。我的推荐顺序是：(1) 检查标注质量，补标小目标；(2) 提升到1280分辨率训练；(3) 添加Copy-Paste增强；(4) 启用P2特征输出或修改模型配置增加小目标检测头；(5) 如果以上还不够，再引入SAHI切片推理。前三步通常能解决80%的小目标问题，后两步是为极端场景准备的杀手锏。

20.6 小目标检测的评价指标

最后要特别强调一个容易被忽视的问题：如何正确评估小目标检测的效果？COCO的mAP@0.5:0.95将目标按面积分为S（small < 32²）、M（medium 32²-96²）、L（large > 96²）三类分别计算AP。但在很多工业场景中，你需要关注更细粒度的指标。

在自动驾驶场景中，远处行人可能只有20×50像素，而近处行人可能有200×500像素。只用S/M/L分组不足以反映真实需求。实战中建议自己定义更细的分组：极小目标（< 16²）、小目标（16²-48²）、中等目标（48²-144²）、大目标（> 144²）。这种四分组方式能更准确地反映模型在不同距离/尺度下的表现。

另外，很多小目标检测论文中报告的mAP提升，在置信度阈值>0.5时效果会大幅缩水——因为模型在低置信度区间小目标会"好"一些（大量低质量的检测框提高了召回率），但实际部署时这些低置信度框都会被过滤掉。评估时一定要以实际部署的置信度阈值来确认小目标的真正性能。

关键认知：小目标检测不是一个"找到一个万能方法就能解决"的问题。它更像是一个系统工程问题——数据质量、模型架构、训练策略、推理后处理，每一个环节都对最终结果有显著影响。而最有经验的工程师都知道，小目标检测中80%的收益来自数据（标注+增强），只有20%来自算法创新。所以当你面对小目标问题时，先盯着数据看三天，再动模型。

好了，第20章的内容就到这里。小目标检测是一个"看起来简单做起来难"的问题——绝大多数公开方案在特定数据集上效果很好，但换到真实场景就水土不服。这一章我们给出的方法经过了大量工程实践的检验：从多尺度训练到特征增强，从SAHI切片到数据级优化，每一招都对应一个具体的瓶颈。然而，检测得再好也只是"模型本身的性能"。要让YOLO真正服务于生产，我们需要把模型从训练环境"搬"到部署环境——从PyTorch到ONNX到TensorRT，每一步都有独特的挑战。第21章「部署与推理加速」，我们来揭开模型部署的全流程。下章见。
