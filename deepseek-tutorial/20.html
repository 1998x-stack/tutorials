第20章 R1-Zero → R1

2024 年秋天的一个深夜，DeepSeek 的训练集群上，一个实验正在悄悄地运行。研究员们在 Kick 了两天的 GRPO 训练后，打开日志查看——MATH 基准的分数停在了令人失望的 15.6%。这个结果让他们有些沮丧。

但别急——这不是最终结果，这只是第一次尝试。团队很快发现，问题出在基座模型本身。V3-Base 虽然在通用任务上很强，但它一开始并不知道"推理"是什么意思——它不会使用 CoT 标记，不知道要先思考再回答。于是团队做了一项在当时看来相当激进的实验：直接用纯强化学习训练模型学会推理，不依赖任何监督数据。结果，他们见证了一个令人震撼的现象——模型竟然自己"学会"了怎么思考。

20.1 R1-Zero 实验：纯 RL 无 SFT 数据

R1-Zero 的实验设计出奇地简单：

  • 基座模型：DeepSeek-V3-Base（未经 SFT 对齐的原始预训练模型）
  • 训练算法：GRPO（第 19 章详细介绍）
  • 奖励信号：纯规则奖励——只有准确性奖励（答案匹配）和格式奖励（格式标记）
  • 训练数据：纯数学和编程问题，每个问题有标准答案
  • 零 SFT 数据：没有任何人工标注的推理链数据

换句话说，R1-Zero 的整个训练流程中，模型从未见过任何"推理示例"。它只知道两件事：问题是什么，以及正确的答案是什么。至于怎么从问题推导出答案——全靠自己摸索。

为什么会有效？这就像让一个从没下过棋的人直接上场对弈，只告诉他"赢了得一分，输了扣一分"。他不会开局套路、不会中局战术，但通过反复试错，他会自己发现"把马跳到这个位置能形成威胁"——这就是 RL 的力量。模型在探索中发现了推理的策略。

训练持续了数千步。在前几百步中，模型的输出是混乱的——语言不连贯，推理链跳跃，常常答非所问。但到了 1000 步左右，一个有趣的现象开始出现：输出的结构变得越来越清晰，CoT 标记开始稳定地出现在输出中。

20.2 推理能力涌现

经过约 2000 步的 GRPO 训练后，R1-Zero 的表现让团队震惊了。

在 MATH-500 基准 上，模型从基线的约 40% 一路飙升到了 78% 以上。更重要的是，在训练日志中观察到了推理能力的涌现现象——随着训练持续，模型的平均响应长度（思考步数）同步增长，而不仅仅是准确率在提升。

观察 R1-Zero 的输出样例时，研究人员发现了一些让人激动的东西：

  • 自我反思（Self-reflection）——模型会在中途停下来检查自己的推理，比如："等等，我上一步的计算好像有问题，让我重新检查一遍……"
  • 纠错能力（Self-correction）——发现错误后，模型会主动回溯并修正推理路径："嗯，我不应该用这个公式，因为条件 A 不满足，改用公式 B……"
  • 多路径探索（Multi-path exploration）——模型会尝试多种解题方案："方法一：用余弦定理……方法二：用向量法……嗯，方法二更简洁，我们用它"

这些行为在训练数据中从未被教过——模型是完全通过 RL 的试错过程"自发地"学会了这些策略。

关键洞察：R1-Zero 实验证明了"推理能力可以是涌现的"——只要你给模型足够的探索空间和正确的奖励信号，它就能自己学会如何进行多步推理、自我纠错和策略选择。这打破了"推理必须通过监督数据来教"的传统认知。

最终，R1-Zero 在 MATH-500 上达到了 82% 的准确率，接近当时最先进的闭源模型。对于纯 RL、零 SFT 数据的结果来说，这个数字堪称惊艳。

20.3 R1-Zero 的问题

然而，R1-Zero 并不是一个可以上线使用的模型。它有两个致命问题：

问题一：可控性差

RL 训练让模型学会了推理，但也带来了一些"副作用"。因为没有对人类偏好进行对齐，R1-Zero 的输出在以下方面表现不佳：

  • 毒性（Toxicity）：在开放域对话中，模型的输出有 15% 的概率包含不当言论。这是因为在纯数学/编程数据上训练的模型，没有学习过"什么话不该说"。
  • 语言混乱：模型会在同一个输出中混合多种语言，或者出现语义不连贯的段落。
  • 过度思考：对于简单问题（比如"2+3=?"），模型也会写上一大段推理过程，显得非常不自然。

问题二：开放域对话偏离

R1-Zero 的训练数据全部都是数学和编程题。这让它变成了一个"偏科生"——数学能力顶尖，但在开放域对话中表现很差。用户问"推荐一部好看的电影"，它可能给你列出概率公式来分析用户偏好。这就尴尬了。

评估维度 R1-Zero 需求目标 差距
MATH-500 82% 85%+ 接近目标
毒性比例 15% < 5% ❌ 严重超标
多语言混合 经常出现 几乎不出现 ❌ 需要修复
开放域对话质量 差（偏科严重） 好 ❌ 需要大量改进

R1-Zero 证明了纯 RL 路线的可行性，但也暴露了它的局限性。要让模型既有强大的推理能力，又安全可控、适配开放域对话，还需要更多的"教育"。

20.4 冷启动 SFT

DeepSeek 解决 R1-Zero 问题的方法，是引入 冷启动 SFT（Cold-start SFT）——在 RL 训练之前，先用高质量的长链推理数据对模型做一轮监督微调。

冷启动数据集的构建方法如下：

  • 来源：从 R1-Zero 的 RL 训练过程中收集表现最好的输出（奖励得分最高的），筛选出推理链清晰、格式规范的样本
  • 人工精炼：请人类标注员对这些 RL 生成的最佳输出进行修正和润色，修正语言错误、补充缺失的推理步骤
  • 规模：约 数千条 高质量长链推理数据——这个规模远远小于传统 SFT 的数十万条级别
  • 覆盖范围：除数学/编程外，也加入了通用对话场景的数据，以确保模型在全领域的基础能力

冷启动数据的格式示例如下：

{
  "prompt": "解方程 3x + 7 = 22",
  "response": "<think>这是一个一元一次方程。
目标是求出 x 的值。
第一步，将常数项移到等号右边：
3x = 22 - 7
3x = 15
第二步，两边同时除以系数 3：
x = 15 / 3
x = 5
所以方程的解是 x = 5。
</think>
<answer>x = 5</answer>",
  "reward_score": 0.95
}

为什么需要冷启动 SFT？原因有两个：

第一，RL 初始化。如果模型在 RL 训练开始前就掌握了基本的推理格式和行为模式，RL 可以专注于优化推理质量，而不是同时学习"怎么输出"和"输出什么"两件事。这大大加速了 RL 收敛。

第二，安全性基础。在冷启动阶段就加入安全对齐数据，模型从一开始就学会"什么话不能说"，后续 RL 训练中不会偏离太远。

注意：冷启动 SFT 和 R1-Zero 的纯 RL 思路并不矛盾。R1-Zero 证明了"纯 RL 可行"，但实践中"SFT + RL"的组合方案更优。纯 RL 是方向验证，SFT + RL 是工程实现。这就像学开车——你可以自己摸索（纯 RL），但先跟教练学基本操作（SFT），再去路上练习（RL），效率要高得多。

20.5 多阶段强化学习

冷启动 SFT 之后，R1 的训练进入了多阶段 RL 环节：

第一阶段：推理强化

这一阶段的任务就是提升推理能力。训练数据以数学、编程等有确定性答案的任务为主，使用 GRPO + 规则奖励。训练步数约 200K 步，学习率逐渐衰减。

第二阶段：全领域 RL

在推理能力达标后，将训练数据扩展到全领域——包括通用对话、写作、翻译、创意生成等。奖励信号从规则奖励切换为模型奖励（训练好的 Reward Model），用 PPO + KL 惩罚来约束模型行为。

第三阶段：安全对齐

最后一个阶段专注于安全对齐。使用 RLHF 方法，奖励信号来自安全偏好 Reward Model。同时加入强 KL 惩罚，确保模型在开源对话中的安全性。经过这一阶段，毒性比例从 R1-Zero 的 15% 降至 3.2%，达到了可上线标准。

阶段 训练数据 算法 奖励信号 训练步数 KL 惩罚
冷启动 SFT 数千条推理 + 对话 SFT 交叉熵损失 少量 -
推理强化 数学 / 编程 / 逻辑 GRPO 规则奖励 200K β × KL
全领域 RL 全领域对话 PPO + KL 模型奖励 100K 强 KL
安全对齐 安全偏好数据 RLHF 安全 Reward 50K 强 KL
训练策略的演进：从 R1-Zero 到 R1，训练策略经历了从"野蛮生长"到"精心栽培"的转变。R1-Zero 是"放养"——给模型极大的自由度去探索推理，能涌现出自我反思等高级能力但可控性差。R1 是"引导式栽培"——先 SFT 打基础、再分阶段 RL 逐步优化、最后安全对齐收尾。两者结合，既有 R1-Zero 的推理能力涌现，又有 SFT 的稳定性和可控性。

20.6 蒸馏 800K 样本

R1 训练完成之后，DeepSeek 团队面临一个新问题：R1 本身是一个 671B 参数的 MoE 大模型，推理成本高，部署门槛高。不是每个团队都能用得起的。

解决方案是知识蒸馏（Knowledge Distillation）——用 R1 这个"老师"生成高质量的推理数据，用来训练更小的"学生"模型。

具体流程：

  • 用 R1 在大量推理任务上生成输出，收集约 800K 条 推理样本
  • 包括数学推理、编程推理、逻辑推理等多种类型
  • 过滤掉质量低（奖励得分低）的输出
  • 用这些数据对 1.5B 到 70B 的小模型进行 SFT 训练

20.7 蒸馏效果

蒸馏的结果令人瞩目。用 R1 数据训练的小模型，在推理任务上的表现甚至超越了一些同样大小的最先进模型：

模型 参数量 MATH-500 GSM8K HumanEval
Qwen-2.5-7B 7B 58.4% 72.1% 45.2%
R1-Distill-Qwen-7B 7B 81.2% 88.4% 63.8%
LLaMA-3.1-8B 8B 38.5% 55.0% 34.2%
R1-Distill-LLaMA-8B 8B 76.8% 84.1% 59.8%
GPT-4o-mini 未知 78.2% 85.3% 61.4%
R1-Distill-Qwen-32B 32B 89.3% 93.2% 74.1%
R1-Distill-LLaMA-70B 70B 91.8% 95.0% 78.9%

最令人惊讶的是，7B 的蒸馏模型在 MATH-500 上达到了 81.2%，甚至超过了 GPT-4o-mini。这意味着 一个可以在手机上运行的 7B 模型，其数学推理能力超过了 OpenAI 的官方小模型。这就是蒸馏的巨大威力——把大模型通过 RL 学会的推理能力，压缩到小模型中。

思考：为什么 R1 的蒸馏效果这么好？关键在于 R1 的推理过程包含了丰富的"思维链"——自我反思、多路径探索、纠错等。当小模型学习这些长链推理过程时，它不仅仅是"记住答案"，而是学会了一种"思考方法"。这比单纯用标准答案训练的效果要好得多。

R1 的故事告诉我们，强化学习可以让模型自发涌现推理能力，而多阶段训练策略和知识蒸馏则将这些能力固化和传播。DeepSeek 不仅在研发层面实现了突破，还让这些能力以极低的成本惠及了每一个开发者。
