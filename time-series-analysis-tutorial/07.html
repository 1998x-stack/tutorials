第7章 频谱分析入门

大家好，我是你们的讲师。这一章我们来做一个思维切换。前面六章我们一直在"时域"（Time Domain）里打转——观察值如何随时间变化，数据在时间轴上的自相关结构如何。现在我要请你戴上另一副眼镜，从"频域"（Frequency Domain）的角度来看时间序列。

先放下公式，说个故事。1822年，法国数学家 Joseph Fourier 在研究热传导方程时，做出了一个石破天惊的断言：任何周期函数都可以分解成一系列正弦波和余弦波的叠加。这个发现在当时被认为是荒谬的（包括拉格朗日在内的数学权威都持反对意见），但最终它成为了整个工程学的基石。今天，从手机信号处理到 Spotify 的音频编码，从心电图分析到地震检测——处处都是傅里叶分析的影子。

在时间序列分析中，频谱分析（Spectral Analysis）让我们能回答一个时域方法很难回答的问题：这条序列里隐藏着哪些周期？ 比如：太阳黑子活动真的是11年一个周期吗？零售数据除了每年圣诞节高峰，还有没有其他更短的周期？

7.1 时域 vs. 频域

理解这两个视角最简单的类比是音乐：

  • 时域视角——看乐谱，音符在时间轴上依次排列，你会看到旋律的起伏、节奏的变化
  • 频域视角——看频谱分析仪，告诉你当前这段音乐里有哪些频率成分、各频率的能量多大

一张时域图只能告诉你"信号随时间怎么变"，一张频谱图告诉你"信号的周期性由哪些频率构成"。两者是互补关系，不是替代关系。

7.2 傅里叶级数基础

傅里叶级数的核心公式其实非常直观：

Y(t) = a₀ + Σ [a_k * cos(2π * k * t / n) + b_k * sin(2π * k * t / n)]

其中：

  • a₀ 是常数项（均值）
  • k 是频率序号，k=1 对应基频（一个完整周期），k=2 对应两倍频，等等
  • a_k, b_k 是对应频率的振幅系数

简单理解：任何时间序列都可以看作若干个不同频率、不同振幅的"波"叠加而成。高频的波对应快速的波动（如每日噪声），低频的波对应缓慢的摆动（如经济周期）。

7.3 周期图（Periodogram）

周期图就是把时间序列从时域变换到频域后的可视化结果。横轴是频率（或周期），纵轴是功率谱密度（即该频率成分的能量大小）。

周期图中的峰值对应数据中显著存在的周期。比如：

  • 如果数据有年度季节性，周期图会在频率 = 1/12（月度数据）或 1/52（周数据）处显示出峰值
  • 如果数据有周季节性和年度季节性，两个频率处都会有峰值

周期图计算的核心是快速傅里叶变换（FFT, Fast Fourier Transform），在 Python 里只用一行代码就能完成。

7.4 谱泄漏与窗口函数

周期图在实际使用中有一个大问题——谱泄漏（Spectral Leakage）。由于我们只能观测有限时间长度的数据（相当于在无限长的真实信号上"截断"），这会导致频谱能量从真实频率"泄漏"到相邻频率上，表现为周期图上出现不应有的旁瓣。

解决方法是使用窗口函数（Window Function）。窗口函数的作用是平滑数据的两端，减少截断效应。常见的窗口包括：

  • Hann 窗——最常用，平衡了主瓣宽度和旁瓣抑制
  • Hamming 窗——Hann 的变体，旁瓣更低
  • Blackman 窗——对旁瓣抑制更强，但主瓣更宽
  • 矩形窗——相当于不用窗口，谱泄漏最严重
实践口诀：一般来说，先用 Hann 窗做周期图。如果发现频率定位不准，换 Hamming 窗。如果对振幅精度要求高，换 Blackman 窗。大多数情况下，Hann 窗已经足够好。

7.5 识别隐藏周期

频谱分析最迷人的应用之一是发现数据中隐藏的周期性。以下是一个真实世界的例子：

太阳黑子（Sunspots）数据从1700年就开始被记录。在时域上看，太阳黑子数量在几十年尺度上起起落落。用频谱分析会发现，它的主要周期大约是11年（准确来说是10.5~11年）。但进一步分析会发现，在周期图中除了11年的主峰，还有一个微弱的副峰——对应约100年的周期（Gleissberg cycle）。这个发现在纯时域中几乎是看不出来的。

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from scipy import signal

# 生成包含两个隐藏周期的模拟数据
t = np.linspace(0, 100, 1000)  # 100年，每年10个采样点
# 周期1: 10年周期（主周期）
# 周期2: 3年周期（次周期）
y = (2 * np.sin(2 * np.pi * t / 10) +
     0.8 * np.sin(2 * np.pi * t / 3) +
     np.random.normal(0, 0.3, 1000))

# 计算周期图
frequencies, power = signal.periodogram(y, fs=10)

# 转换为周期
periods = 1 / frequencies

# 找出前3个最强峰
top_idx = np.argsort(power)[-3:][::-1]
print("最强周期：", periods[top_idx[0]])  # 应该接近 10
print("次强周期：", periods[top_idx[1]])  # 应该接近 3
业务洞察案例：我在一个电商项目中用频谱分析分析访客流量数据。原始时序图只能看到每周周末有个小高峰。但周期图揭示了一个有趣的现象：除了7天周期外，还有一个约30天的强周期。后来一查，原来对应的是公司每月1号的"会员日"促销。这个周期在时域里被周末噪声掩盖了，但在频域中一目了然。我们就利用这个发现优化了库存补货策略。

7.6 实际使用中的注意事项

频谱分析虽然强大，但有几个常见陷阱：

  • 数据长度——周期图需要足够长的数据才能可靠地检测低频周期。要检测一个周期为 T 的信号，至少需要 5T ~ 10T 长度的数据
  • 趋势和季节性——在计算周期图之前，建议先用 STL 或其他方法去掉确定性的趋势和季节成分，否则它们会掩盖更细微的周期模式
  • 统计检验——周期图的峰值不一定都统计显著。Fisher's g 检验和 Bartlett's 检验可以用来判断峰值的显著性
  • 平稳性假设——频谱分析假设序列是平稳的（或至少是周期平稳的）。非平稳数据上的周期图解读要非常谨慎
踩坑实录：曾经有学生把一条有强线性趋势的经济序列直接做频谱分析，结果周期图在最低频处出现了一个巨大的峰——这根本不是周期，而是趋势被"伪装"成了无穷长周期的信号。这就像把马拉松运动员的跑步数据当成"非常慢的周期性振荡"来看，逻辑上没错，但毫无意义。务必先去除趋势再做频谱分析！

频谱分析为时间序列分析打开了一扇新的窗户。它让我们能够看到时域中不可见的周期性结构，也为后续要讲的滤波方法（比如 Hodrick-Prescott 滤波、带通滤波）奠定了基础。现在你已经熟悉了时域和频域两个视角，下一章我们将回到时域，学习时间序列建模最核心的工具之一——自回归模型（AR）。你会发现，AR 模型本质上就是用过去的值对当前值做回归，而自相关分析中学到的 ACF 和 PACF 正是帮你选择 AR 阶数的利器。
