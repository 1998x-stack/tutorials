第28章 综合实战项目

恭喜你走到这里！这是整门课程的最后一章。前面27章我们学习了推荐系统的理论、模型、工程实践和前沿趋势。现在，是时候把所有的知识串联起来，亲手做一个完整的推荐系统了。

这一章不是一个代码片段合集，而是一个完整的项目路线图。我强烈建议你关上网页，打开IDE，跟着这一章的步骤一步步实现。读完28章不等于学完了这门课——当你亲手把最后一个API调通、看到推荐结果返回的那一刻，你才算真正毕业。

1. 项目定义与数据集

我们选择MovieLens-1M数据集作为项目的数据基础。为什么选它？

  • 真实的用户-物品交互数据：包含6040个用户对约3706部电影的100万条评分记录。虽然评分数值在1-5之间，但我们可以将它转化为隐式反馈（有评分=正样本）。
  • 规模适中：100万条记录，在笔记本电脑上即可完成全流程训练和评估，不需要GPU。
  • 广泛研究：这个数据集被大量论文引用，你的实现可以和公开结果做对比。

项目范围：做物品推荐（Item Recommendation），而非评分预测（Rating Prediction）。也就是说，目标是给用户推荐他可能喜欢看的电影，而不是预测他会给某部电影打几分。我们要构建的推荐系统包含召回、排序和简单的在线服务三个模块。

评估标准：

  • 离线评估：HR@10（Hit Rate at 10）和 NDCG@10（Normalized Discounted Cumulative Gain at 10）
  • 模拟在线评估：基于历史回放的Replay Evaluation——对于测试集中的每个用户行为时间点，只使用该时间点之前的数据做推荐，看推荐结果是否包含了用户实际点击的物品。

2. 数据流水线搭建

第一步是构建一个干净、可复用的数据流水线。高质量的pipeline比精致的模型更重要——数据科学家们常说一句话："Garbage in, garbage out."

Step 1：数据探索

在开始建模之前，花时间理解你的数据。MovieLens-1M包含三个表：ratings（用户-电影-评分-时间戳）、users（用户特征：性别、年龄、职业、邮编）、movies（电影特征：片名、类型）。先做几个基本的分析：

  • 评分分布：大部分评分集中在3-5分，4分最多。这意味着如果我们将评分>3视为正样本、≤3视为负样本，正负比大约为2:1。
  • 用户活跃度分布：少数活跃用户贡献了大部分评分，大部分用户评分数量很少（长尾分布）。冷启动用户的处理需要特别关注。
  • 物品流行度分布：同样呈现长尾分布——少数热门电影获得了大部分评分，大量冷门电影只有零星评分。

Step 2：特征工程

基于探索结果，我们构建三类特征：

  • 用户特征：用户平均评分、用户评分数量、用户最偏好的电影类型（统计层面）、用户年龄分组、用户性别
  • 物品特征：电影平均评分、电影被评分次数、电影类型（multi-hot编码）、电影上映年份
  • 上下文特征：评分时间戳→小时、星期几（虽然用于训练，但在模拟serving时需要注意时效性）

Step 3：时间感知的划分

按照时间顺序划分训练/验证/测试集，确保没有未来数据泄露（No Future Data Leakage）。具体做法：将每个用户的所有交互按时间排序，前70%作为训练集，中间10%作为验证集，后20%作为测试集。这样可以模拟真实的在线场景——用过去预测未来。

Step 4：负采样

推荐系统面临的是一个"只有正样本"的问题——用户看了某部电影（正样本），但用户没看过的电影不一定是负样本（可能用户不知道、没时间看、或者真的不喜欢）。通用的做法是对每个正样本采样4个负样本（用户没交互过的物品），训练时让模型学会区分正负样本。

3. 召回模型实现

召回阶段我们实现一个经典的双塔模型（Two-Tower Model）。双塔模型的核心思想是将用户和物品分别映射到同一个向量空间中，通过内积计算相似度。

以下是完整的双塔模型训练代码：

# Two-Tower Model 完整训练循环 (PyTorch)
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, Dataset
import faiss
import numpy as np

# ========== 模型定义 ==========
class TwoTower(nn.Module):
    def __init__(self, user_feat_dim, item_feat_dim, embed_dim=64):
        super().__init__()
        # User Tower: user features → 256 → 128 → 64
        self.user_tower = nn.Sequential(
            nn.Linear(user_feat_dim, 256),
            nn.ReLU(),
            nn.BatchNorm1d(256),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.BatchNorm1d(128),
            nn.Linear(128, embed_dim),
        )
        # Item Tower: item features → 256 → 128 → 64
        self.item_tower = nn.Sequential(
            nn.Linear(item_feat_dim, 256),
            nn.ReLU(),
            nn.BatchNorm1d(256),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.BatchNorm1d(128),
            nn.Linear(128, embed_dim),
        )

    def forward(self, user_feats, item_feats):
        user_emb = self.user_tower(user_feats)
        item_emb = self.item_tower(item_feats)
        # 对embedding做L2归一化
        user_emb = nn.functional.normalize(user_emb, p=2, dim=1)
        item_emb = nn.functional.normalize(item_emb, p=2, dim=1)
        return user_emb, item_emb

# ========== 训练数据集 ==========
class PairDataset(Dataset):
    """每个样本 = (user_features, pos_item_features, neg_item_features)"""
    def __init__(self, user_feats, pos_item_feats, neg_item_feats):
        self.user_feats = torch.FloatTensor(user_feats)
        self.pos_item_feats = torch.FloatTensor(pos_item_feats)
        self.neg_item_feats = torch.FloatTensor(neg_item_feats)

    def __len__(self):
        return len(self.user_feats)

    def __getitem__(self, idx):
        return (self.user_feats[idx],
                self.pos_item_feats[idx],
                self.neg_item_feats[idx])

# ========== 训练循环 ==========
def train_two_tower(model, train_loader, epochs=10, lr=1e-3):
    optimizer = optim.Adam(model.parameters(), lr=lr)
    criterion = nn.TripletMarginLoss(margin=0.5)

    for epoch in range(epochs):
        total_loss = 0.0
        model.train()

        for batch_idx, (user_feats, pos_feats, neg_feats) in enumerate(train_loader):
            user_emb, pos_emb = model(user_feats, pos_feats)
            _, neg_emb = model(user_feats, neg_feats)

            # Triplet loss: anchor=user, positive=pos_item, negative=neg_item
            loss = criterion(user_emb, pos_emb, neg_emb)

            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

            total_loss += loss.item()

            if batch_idx % 100 == 0:
                print(f"Epoch {epoch+1}, Batch {batch_idx}: loss={loss.item():.4f}")

        avg_loss = total_loss / len(train_loader)
        print(f"Epoch {epoch+1} completed. Avg loss: {avg_loss:.4f}")

    return model

# ========== 构建Faiss索引 ==========
def build_faiss_index(model, all_item_feats, device='cpu'):
    model.eval()
    with torch.no_grad():
        item_feats_tensor = torch.FloatTensor(all_item_feats).to(device)
        _, item_embeddings = model(
            torch.zeros((1, item_feats_tensor.shape[1])).to(device),
            item_feats_tensor
        )
    item_embeddings = item_embeddings.cpu().numpy().astype('float32')

    # 构建Faiss IVF索引（比暴力搜索快100倍）
    dim = item_embeddings.shape[1]
    nlist = min(int(len(item_embeddings) ** 0.5), 100)
    quantizer = faiss.IndexFlatIP(dim)
    index = faiss.IndexIVFFlat(quantizer, dim, nlist,
                                faiss.METRIC_INNER_PRODUCT)
    index.train(item_embeddings)
    index.add(item_embeddings)
    return index

# ========== 召回评估 ==========
def recall_at_k(model, index, test_data, k=100):
    """计算召回率@K"""
    model.eval()
    hits = 0
    total = 0
    with torch.no_grad():
        for user_feats, target_items in test_data:
            user_emb = model.user_tower(
                torch.FloatTensor(user_feats).unsqueeze(0)
            )
            user_emb = nn.functional.normalize(user_emb, p=2, dim=1)
            user_emb = user_emb.cpu().numpy().astype('float32')

            # Faiss搜索最近的k个物品
            distances, indices = index.search(user_emb, k)
            if target_items[0] in indices[0]:
                hits += 1
            total += 1

    return hits / total if total > 0 else 0.0

训练完成后，你的召回模型的Recall@100应该在0.4-0.6之间（取决于特征设计和负采样策略）。这就意味着对于测试集中用户的真实交互物品，有40-60%的概率被召回模型排在前100个候选里。

4. 排序模型实现

召回阶段为我们产生了100个候选物品。现在需要用更精确的模型对这100个候选进行排序。我们实现DeepFM——一个兼顾低阶特征交叉（FM）和高阶特征交叉（Deep）的经典模型。

DeepFM的PyTorch实现要点：

  • EmbeddingLayer：将稀疏的类别特征（用户ID、电影ID、类型等）映射为稠密embedding。共享给FM部分和Deep部分。
  • FMLayer：计算所有特征embedding的两两内积之和，捕捉二阶特征交叉。
  • DeepLayer：将embedding拼接后送入MLP，捕捉高阶非线性交叉。
  • Output：FM部分和Deep部分的输出相加，经过sigmoid得到最终点击概率。

训练使用Binary Cross-Entropy Loss。在MovieLens上，DeepFM的AUC通常在0.75-0.85之间。排序模型训练完成后，我们将召回和排序串联起来评估NDCG@10。一个好的模型组合应该能达到NDCG@10 > 0.3。

5. 在线服务搭建

模型训练好了，怎么让它为真实用户服务？我们需要搭建一个轻量级的在线推理服务。

FastAPI推理服务：

使用FastAPI搭建一个HTTP API服务，暴露一个 /recommend 端点：

  • 请求：GET /recommend?user_id=123&top_k=10
  • 处理流程：根据user_id获取用户特征 → 双塔模型输出用户embedding → Faiss召回Top-100候选 → DeepFM对候选重新排序 → 返回Top-10推荐结果
  • 响应：JSON格式，包含movie_id、预测得分、以及可选的推荐理由

Docker化部署：

用Docker将服务打包，确保环境一致性。Dockerfile的内容很简单：基于官方的PyTorch镜像，复制模型文件和代码，安装依赖（fastapi、uvicorn、faiss-cpu、pytorch），启动服务。

模拟A/B测试：

使用Replay Evaluation来模拟A/B测试的效果。在测试集的每个时间点，将我们的推荐系统（召回+排序）与一个"热门推荐"基线做对比。具体做法：

  • 基线：推荐全局最热门的10部电影（按历史评分次数排序）
  • 实验组：双塔召回 + DeepFM排序的个性化推荐
  • 指标：HR@10（推荐列表中是否包含用户实际点击的电影）

在MovieLens-1M上，个性化模型相比热门基线通常能提升HR@10 20-40%。这个提升证明了个性化的价值——不是所有人都喜欢《肖申克的救赎》（虽然人人都说它好），有人更喜欢小众的独立电影。

实战建议：如果你在完成这个项目时卡在任何一步，可以按照以下顺序排查：1) 检查数据划分是否保证了时间顺序（最常见的问题）；2) 检查负采样策略，负样本不能和用户有交互，但也不全是随机负样本（考虑用"热门但用户没看过的"做hard negative）；3) 检查embedding的归一化设置在训练和推理时是否一致（不归一化会导致内积结果异常）。95%的"模型不收敛"问题都出在这三个地方。
