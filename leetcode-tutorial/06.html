滑动窗口思想

大家好，我是你们的讲师。今天我们要聊的滑动窗口（Sliding Window），可以看作是上一章双指针的"同向指针"的进阶版。滑动窗口专门用于解决连续子数组 / 子串问题——这是算法面试中出现频率最高的题型之一。

滑动窗口的核心理念是：维护一个窗口（连续区间），通过不断调整窗口的左右边界来搜索最优解。窗口整体从左向右移动，每个元素进入窗口一次、离开窗口一次，因此总时间复杂度是 O(n) 而不是 O(n²)。

根据窗口大小是否固定，滑动窗口分为三种类型：定长窗口、不定长窗口（求最长/最短）和恰好 K 型（容斥原理）。我们逐一来看。

核心洞察：滑动窗口的本质是避免重复计算。暴力法每次重新计算子数组，而滑动窗口利用了相邻子数组之间的重叠部分，只更新变化的两端。这种"增量计算"的思维在很多算法中都有体现。

定长窗口模板

定长窗口是滑动窗口中最简单的形式——窗口大小固定为 k，每次移动一步，更新窗口内的结果。

大小为 K 且平均值最大的子数组（LeetCode 643）

def find_max_average(nums: list[int], k: int) -> float:
    # 先计算第一个窗口的和
    window_sum = sum(nums[:k])
    max_sum = window_sum

    # 窗口向右滑动
    for i in range(k, len(nums)):
        window_sum += nums[i] - nums[i - k]
        max_sum = max(max_sum, window_sum)

    return max_sum / k

关键操作就是 window_sum += nums[i] - nums[i - k]——加进新元素，移除旧元素，避免了每次重新求和。这个"增量更新"的思想是滑动窗口效率的来源：不要重新造轮子，利用好已经算好的结果。

不定长窗口 — 求最长 / 最短

不定长窗口是最灵活也最常用的滑动窗口形式。窗口大小不固定，根据条件动态调整右边界（扩展）和左边界（收缩）。

无重复字符的最长子串（LeetCode 3）

这道题是滑动窗口的"名片"——几乎所有刷 LeetCode 的人都做过它。找出字符串中不包含重复字符的最长子串的长度。

def length_of_longest_substring(s: str) -> int:
    char_set = set()
    left = 0
    ans = 0
    for right in range(len(s)):
        # 如果 s[right] 已在窗口内，移动 left 直到移除重复
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1
        char_set.add(s[right])
        ans = max(ans, right - left + 1)
    return ans

这个模式非常典型：right 负责扩展窗口，while 循环负责在条件不满足时收缩窗口。窗口始终维护"不包含重复字符"这个不变量。

这里有一个关键细节：什么时候更新结果？对于"求最长"的问题，一般在收缩完窗口后（即窗口满足条件时）更新结果；对于"求最短"的问题，一般在窗口满足条件时（即收缩前）更新结果。

长度最小的子数组（LeetCode 209）

给定一个正整数数组和一个目标值 s，找出和 ≥ s 的长度最小的连续子数组。

def min_sub_array_len(target: int, nums: list[int]) -> int:
    left = 0
    window_sum = 0
    ans = float('inf')
    for right in range(len(nums)):
        window_sum += nums[right]  # 扩展窗口
        while window_sum >= target:
            ans = min(ans, right - left + 1)
            window_sum -= nums[left]
            left += 1
    return 0 if ans == float('inf') else ans

注意和第三题的区别：这道题是求最短，所以结果在 while 内更新（窗口满足条件时尝试缩小）；第三题是求最长，结果在 while 外更新（窗口修正到合法状态后记录）。

易错点：求最短时 while 循环内先更新结果再收缩，求最长时先收缩再更新结果。我见过很多人在 LeetCode 3 上把 while 写成 if——当窗口内有多个重复字符时，一个 if 只能移除一个！比如 "abca" 这个 case，right 移到第二个 a 时，如果不循环移除，left 只前进一次，窗口里仍然有 a，就会直接爆炸。

恰好 K 型 — 容斥原理

"恰好 K" 类问题是滑动窗口中比较 tricky 的一种。比如"子数组中恰好有 K 个奇数"、"子串中恰好有 K 种字符"。直接维护"恰好 K"的条件比较困难，一种巧妙的思路是使用容斥原理：将"恰好 K"转化为"最多 K"减去"最多 K-1"。

统计「优美子数组」（LeetCode 1248）

给你一个整数数组和一个整数 k，统计有多少个子数组恰好包含 k 个奇数。

def number_of_subarrays(nums: list[int], k: int) -> int:
    def at_most(k: int) -> int:
        if k < 0:
            return 0
        left = 0
        count = 0
        ans = 0
        for right in range(len(nums)):
            if nums[right] % 2 == 1:
                count += 1
            while count > k:
                if nums[left] % 2 == 1:
                    count -= 1
                left += 1
            # 此时窗口 [left, right] 中最多有 k 个奇数
            # 以 right 结尾的合法子数组个数 = right - left + 1
            ans += right - left + 1
        return ans

    return at_most(k) - at_most(k - 1)

这个解法中 at_most(k) 统计的是"最多有 k 个奇数"的子数组个数。为什么 at_most(k) - at_most(k - 1) = 恰好 k 个？因为所有"最多 k 个"的子数组集合中，包含了"最多 k-1 个"的子数组，差集就是"恰好 k 个"的子数组——这就是容斥原理。

而 at_most(k) 本身是"求最长"的变体——窗口维护"最多 k 个奇数"，当奇数个数超过 k 时收缩左边界。每一次迭代中，right - left + 1 就是以当前 right 结尾的、最多有 k 个奇数的子数组个数。

模式总结："恰好 K" = "最多 K" - "最多 K-1"。这个技巧不仅适用于滑动窗口，在很多统计类问题中都能用。只要你能实现"最多 K"的版本，"恰好 K"只是做一次减法而已。

踩坑与总结

滑动窗口最常见的坑有两个：忘记更新窗口状态和left 移动顺序错误。

先讲第一个。我有个朋友在写 LeetCode 209（最小子数组和）时，写出了这样的代码：

while window_sum >= target:
    ans = min(ans, right - left + 1)
    left += 1
    window_sum -= nums[left]  # 错误！left 已经变了！

这个 bug 非常隐蔽——left += 1 先执行，然后 window_sum -= nums[left] 减去的是新 left 位置的元素，而不是旧 left 位置的元素。正确的顺序是先更新数据，再移动指针：先 window_sum -= nums[left]，再 left += 1。这个顺序问题我至少见过 5 个同学犯过。

第二个坑是 while s[right] in char_set 写成了 if s[right] in char_set。当窗口内有多个重复字符时，if 只能移除一次，而实际上可能需要多次移除才能让窗口重新合法。例如字符串 "abcaa"——right 指向索引 4 的 a 时，窗口 [0,4] 里有三个 a，if 只移除索引 0 的 a，窗口里仍然有重复的 a。

第三个容易被忽略的问题：窗口状态的数据结构选择。对于字符计数用长度为 256 的数组最快；对于整数计数用 Counter 或 dict；对于"是否存在"用 set。选择合适的数据结构能让代码更简洁高效。

到此我们完成了滑动窗口的核心内容。滑动窗口的核心就是"左右指针协调推进，增量更新窗口状态"。下一章我们将学习前缀和与差分——如果说滑动窗口是从"结构"角度处理子数组，前缀和就是从"计算"角度：把区间求和问题从 O(n) 降到 O(1)。两种思路正好形成互补。我们下一章见！
