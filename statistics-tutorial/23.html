第23章：统计学工具链

大家好，我是你们的讲师。学到这里，大家脑子里已经装满了各种统计概念——从描述统计到假设检验，从回归分析到贝叶斯推断。但你可能会问：在实际工作中，我到底该用什么工具来动手分析呢？这恰恰就是本章要解决的问题。统计学工具链不仅仅是"选个软件"，它直接决定你的分析效率、结果可重复性，以及你和团队协作的方式。

23.1 Python vs R：世纪之争

每个刚入行的数据分析师都会面临这个灵魂拷问："我应该学 Python 还是 R？" 答案其实很简单——看你的战场在哪里。R 语言诞生于统计学家之手，天生就是做统计分析和可视化的，它的 tidyverse 生态让数据清洗和探索变得浑然天成。Python 则是通用编程语言，凭借 scipy.stats、statsmodels 和 scikit-learn 在机器学习与工程落地上占据绝对优势。

我的建议是：不必二选一，而是两个都要懂。你日常的主力可以是一个，但理解另一个的生态会让你在跨团队协作时游刃有余。下面这张对比表可以帮你快速定位：

任务Python 包R 包

讲师经验：在实际项目中，我通常用 Python 做数据清洗和特征工程（因为 pandas 实在太顺手了），然后用 R 的 ggplot2 做出版级的可视化，最后在 Jupyter 里整合所有结果。Python 和 R 之间的互联通过 rpy2 包就可以实现，完全不需要在两种语言之间痛苦地切换。

23.2 Python 统计核心：scipy.stats 实战

scipy.stats 是 Python 生态中最基础的统计库，它涵盖了几乎所有常见的概率分布和假设检验方法。让我们通过一个实际例子来感受它的用法。假设我们有两组用户的点击率数据，想要检验它们是否存在显著差异：

import numpy as np
import scipy.stats as stats

# 模拟两组用户的点击率数据（单位：百分比）
np.random.seed(42)
group_a = np.random.normal(loc=5.2, scale=1.5, size=200)  # 对照组
group_b = np.random.normal(loc=6.8, scale=1.7, size=180)  # 实验组

# 1. 描述统计
print("A组: mean={:.3f}, std={:.3f}, n={}".format(
    group_a.mean(), group_a.std(ddof=1), len(group_a)))
print("B组: mean={:.3f}, std={:.3f}, n={}".format(
    group_b.mean(), group_b.std(ddof=1), len(group_b)))

# 2. 正态性检验（Shapiro-Wilk）
shap_a = stats.shapiro(group_a)
shap_b = stats.shapiro(group_b)
print(f"A组正态性 p={shap_a.pvalue:.4f}, B组正态性 p={shap_b.pvalue:.4f}")

# 3. 方差齐性检验（Levene）
levene = stats.levene(group_a, group_b)
print(f"方差齐性 p={levene.pvalue:.4f}")

# 4. 独立样本 t 检验（双尾）
t_stat, p_val = stats.ttest_ind(group_a, group_b, equal_var=True)
print(f"t检验: t={t_stat:.4f}, p={p_val:.6f}")

# 5. 如果正态性不满足，使用 Mann-Whitney U 检验
u_stat, u_pval = stats.mannwhitneyu(group_a, group_b, alternative='two-sided')
print(f"Mann-Whitney U: U={u_stat:.1f}, p={u_pval:.6f}")

# 6. 效应量（Cohen's d）
pooled_std = np.sqrt(((len(group_a)-1)*group_a.std(ddof=1)**2 +
                       (len(group_b)-1)*group_b.std(ddof=1)**2) /
                      (len(group_a)+len(group_b)-2))
cohens_d = (group_b.mean() - group_a.mean()) / pooled_std
print(f"Cohen's d = {cohens_d:.3f}")

这段代码展现了一个标准的假设检验流程：先做描述统计，再检查数据前提条件（正态性和方差齐性），然后选择合适的检验方法，最后计算效应量。每一步都是我们在前几章中学过的知识，现在都用代码串起来了。

23.3 statsmodels：公式接口让回归更优雅

statsmodels 是 Python 中最接近 R 风格的统计建模库。它最大的亮点是提供了类似 R 的公式 API（formula API），让你可以用 y ~ x1 + x2 这样的语法来定义模型。这对于习惯 R 的用户来说简直是福音。

import pandas as pd
import statsmodels.api as sm
import statsmodels.formula.api as smf
from statsmodels.stats.anova import anova_lm
from sklearn.datasets import make_regression

# 生成模拟数据
np.random.seed(123)
n = 500
X = np.random.RandomState(42).randn(n, 3)
beta = np.array([2.5, -1.3, 0.8])
y = 5.0 + X @ beta + np.random.randn(n) * 1.2

df = pd.DataFrame(X, columns=['age', 'income', 'education'])
df['target'] = y

# 使用 statsmodels 公式 API 拟合多元线性回归
model = smf.ols('target ~ age + income + education', data=df)
result = model.fit()
print(result.summary())

# 方差分析表
anova_table = anova_lm(result)
print("\nANOVA 分解：")
print(anova_table)

# 预测与残差诊断
df['pred'] = result.fittedvalues
df['resid'] = result.resid

# Durbin-Watson 检验（残差自相关）
dw = sm.stats.durbin_watson(df['resid'])
print(f"\nDurbin-Watson 统计量: {dw:.4f}")

# Breusch-Pagan 异方差检验
bp_test = sm.stats.het_breuschpagan(df['resid'], result.model.exog)
print(f"Breusch-Pagan 异方差检验 p={bp_test[1]:.4f}")
小贴士：result.summary() 输出的表格几乎是论文级的——它一次性报告了回归系数、标准误、t值、p值、置信区间、R-squared、F检验和多种信息准则。相比 scikit-learn，statsmodels 更关注统计推断而非纯预测，是统计建模的首选。

23.5 Jupyter Notebook：交互式分析的核心

Jupyter Notebook 已经成为数据科学界的"通用语言"。它的单元格式交互让探索性分析变得无比顺畅——你可以在一个单元格里跑描述统计，下一个单元格画图，再下一个跑模型，所有中间结果都保存在内存中。配合 nbconvert，你可以把 notebook 一键导出为 HTML、PDF 甚至幻灯片。

高级用法上，我推荐大家使用 Jupyter Lab 而非传统的 Notebook。 Lab 提供了文件浏览器、多标签页、终端集成等 IDE 级别的功能，还能并排查看多个 notebook 和数据文件。

23.6 常用工作流：一份完整的统计项目流程

将前面学到的工具整合起来，一个标准的统计学分析项目通常遵循以下流程：

  1. 问题定义：明确分析目标和假设
  2. 数据获取：从数据库/CSV/API 加载数据
  3. 数据清洗：处理缺失值、异常值、类型转换
  4. 探索性分析（EDA）：描述统计 + 可视化 + 相关性分析
  5. 统计建模：根据问题选择回归/分类/聚类等方法
  6. 模型诊断：残差分析、共线性检测、交叉验证
  7. 结果报告：用 Jupyter + Quarto 生成可复现报告
  8. 部署（可选）：将模型封装为 API 或仪表盘
踩坑故事：包版本冲突导致结果不一致

我曾经帮一个团队排查一个诡异的问题——同样的数据、同样的统计方法，两个分析师跑出的 p 值居然不一样。查了两天才发现：一个用的是 scipy 1.7.0，另一个是 scipy 1.11.0。原来在 scipy 1.9 之后，ttest_ind 的默认参数 equal_var 的实现细节变了，当两组样本量差异较大时，Welch t 检验的自由度计算公式做了微调，导致 p 值在边界线附近出现了差异。

教训：永远在项目中包含 requirements.txt 或 environment.yml，并记录关键包的版本号。对于重要结论，最好在两种工具中互相验证一下结果。
