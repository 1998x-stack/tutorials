第14章 SFT数据去噪与去重实战

在第 13 章中我们花了大量精力来优化数据配比。但如果你的数据本身存在大量重复或噪声，再好的配比策略也救不了——这就像用优质的调料去烹饪腐烂的食材，结果可想而知。

"后训练数据，质量远重于数量"——这是我们在第 1 章就反复强调的原则。这一章，我们将把这个原则落实到工程实现上。你将学到一套完整的数据清洗方法论：从去重（精确去重、模糊去重、语义去重）到质量过滤（长度过滤、困惑度过滤、格式校验），以及如何用工业化工具（NeMo Curator）在大规模数据集上高效地完成这些操作。

后训练数据「质量远重于数量」的工程实现

在正式进入技术细节之前，我们先建立一个框架性的理解。后训练数据的质量治理可以分为三个层次：

  • 第一层：去重——移除数据集中重复或高度相似的样本。重复数据不仅浪费算力，更严重的是会导致模型对特定模式过拟合。
  • 第二层：过滤——根据预设规则和阈值移除低质量样本。这一步需要你定义什么是"低质量"——太短、太乱、语言不对、格式有误等。
  • 第三层：评分与排序——不是简单地丢弃"坏"数据，而是对数据质量进行打分，让模型在训练中更关注高质量样本。这是更精细化的质量治理。

好的消息是，这三个层次都有成熟的开源工具和实现方案，不需要从零开始造轮子。

去重技术栈

去重是数据清洗中最基础也最重要的环节。根据重复的"粒度"，去重技术可以分为三个层级：

精确去重

完全一样的字符串或经过简单归一化（去除空格、统一大小写）后完全一样的文本。这是最简单也最有效的方法——没有遗漏，也不会误伤。

import hashlib

def exact_dedup(dataset, text_key="instruction"):
    """精确去重：基于 MD5 哈希"""
    seen = set()
    deduped = []
    for item in dataset:
        text = item[text_key]
        text_hash = hashlib.md5(text.encode()).hexdigest()
        if text_hash not in seen:
            seen.add(text_hash)
            deduped.append(item)
    return deduped

# 使用示例
dataset = load_dataset("tatsu-lab/alpaca", split="train")
deduped = exact_dedup(dataset, text_key="instruction")
print(f"去重前: {len(dataset)} 条, 去重后: {len(deduped)} 条")
# 通常 Alpaca 去重后会减少 2000-3000 条（大约 5%）

MinHash + LSH 模糊去重

精确去重无法处理"几乎一样但略有不同"的情况。比如两条指令："请用 Python 写一个二分查找函数"和"用 Python 写一个二分查找函数，谢谢"——内容高度相似，但逐字比对并不相同。这时候就需要模糊去重。

MinHash + LSH（Locality-Sensitive Hashing）是模糊去重的工业标准方法。它的核心思想是：将文本转换成固定长度的"签名"（signature），然后通过比较签名来判断文本相似度。如果两个文本的 MinHash 签名高度重合，它们就很可能包含相似的语义内容。

from datasketch import MinHash, MinHashLSH

def minhash_dedup(dataset, threshold=0.8, text_key="instruction"):
    """
    基于 MinHash + LSH 的模糊去重
    threshold: Jaccard 相似度阈值，大于此值的被视为重复
    """
    lsh = MinHashLSH(threshold=threshold, num_perm=128)
    deduped = []

    for idx, item in enumerate(dataset):
        text = item[text_key]
        # 构建 MinHash
        m = MinHash(num_perm=128)
        for word in text.split():
            m.update(word.encode())

        # 检查是否与已有样本高度相似
        if not lsh.query(m):
            lsh.insert(f"doc_{idx}", m)
            deduped.append(item)

    return deduped

语义 Embedding 去重

MinHash 基于词级别的重叠来计算相似度，无法捕捉"用词不同但意思相同"的语义重复。比如"二元查找算法的 Python 实现"和"如何使用 Python 实现二分搜索"——用词完全不同，但说的是同一件事。这时候就需要语义去重。

语义去重使用 Sentence Transformer 等模型将文本转换为向量，然后基于向量间的余弦相似度来判断重复。这种方法计算量大（需要跑一次 embedding），但效果最好，能发现写法和用词完全不同但语义高度重复的样本。

去重策略的选择顺序：建议先做精确去重，再做 MinHash 模糊去重，最后根据需要做语义去重。精确去重先移除最明显的重复，减少后续模糊去重的计算量；MinHash 处理字面级别的近似重复；语义去重作为"收尾"处理更深层的语义重复。这种分层策略兼顾了效率和效果。

质量过滤流水线

去重之后，下一步是质量过滤。以下是一套经过验证的过滤流水线：

过滤步骤 检测内容 典型阈值 移除比例（参考）
长度过滤 指令或回答过短/过长 回答 < 5 字符 或 > 4096 字符 2-5%
语言检测 语言识别与混合语言检测 非目标语言比例 > 30% 1-3%
困惑度过滤 用一个小模型评估文本困惑度 PPL > 平均 + 2 * 标准差 3-5%
格式完整性 JSON 格式校验、字段完整性 数据不符合 Schema 1-2%
毒性/不当内容 有害内容检测 毒性分数 > 0.8 0.5-2% 
def quality_filtering_pipeline(dataset):
    """完整的数据过滤管线"""
    filtered = []

    for item in dataset:
        # 1. 长度过滤
        response = item.get("output", "")
        if len(response) < 5 or len(response) > 4096:
            continue

        # 2. 语言检测（使用 fasttext 的 lid.176 模型）
        instruction = item.get("instruction", "")
        lang = detect_language(instruction)  # 返回 ISO 语言代码
        if lang not in ("zh", "en", "ja", "ko"):  # 根据需求调整
            continue

        # 3. 困惑度过滤
        ppl = compute_perplexity(response)
        if ppl > PPL_THRESHOLD:
            continue

        # 4. 格式完整性检查
        if not validate_structure(item):
            continue

        filtered.append(item)

    return filtered
关键实践经验：质量过滤的"黄金法则是"不要过度过滤"——过于激进的过滤会把"意外之喜"也过滤掉。有些看起来"质量不高"的数据（比如带语法错误的用户提问），恰恰因为更接近真实用户场景而成为更有价值的训练数据。建议采取"保守过滤，逐步收窄"的策略：先用宽松的阈值过滤掉明显的低质量数据，评估训练效果，再根据效果逐步收紧阈值。

实战：用datasets库搭建去重管线

让我们把前面学到的方法整合成一个完整的实战例子。下面是用 HuggingFace datasets 库搭建的去重+过滤管线：

from datasets import load_dataset, Dataset
from datasketch import MinHash, MinHashLSH
import hashlib

class DataCleaningPipeline:
    """后训练数据清洗管线"""

    def __init__(self, dataset):
        self.dataset = dataset
        self.stats = {"initial": len(dataset)}

    def exact_dedup(self, text_keys=["instruction", "output"]):
        """精确去重"""
        seen = set()
        deduped = []
        for item in self.dataset:
            combined = "".join(item.get(k, "") for k in text_keys)
            h = hashlib.md5(combined.encode()).hexdigest()
            if h not in seen:
                seen.add(h)
                deduped.append(item)
        self.dataset = deduped
        self.stats["exact_dedup"] = len(deduped)
        return self

    def minhash_dedup(self, threshold=0.8, text_key="instruction"):
        """模糊去重"""
        lsh = MinHashLSH(threshold=threshold, num_perm=128)
        deduped = []
        for idx, item in enumerate(self.dataset):
            m = MinHash(num_perm=128)
            for word in item.get(text_key, "").split():
                m.update(word.encode())
            if not lsh.query(m):
                lsh.insert(f"d{idx}", m)
                deduped.append(item)
        self.dataset = deduped
        self.stats["minhash_dedup"] = len(deduped)
        return self

    def length_filter(self, min_len=10, max_len=4096, text_key="output"):
        """长度过滤"""
        self.dataset = [
            item for item in self.dataset
            if min_len <= len(item.get(text_key, "")) <= max_len
        ]
        self.stats["length_filter"] = len(self.dataset)
        return self

    def report(self):
        """输出清洗报告"""
        print("=" * 50)
        print("数据清洗报告")
        print("=" * 50)
        initial = self.stats["initial"]
        for stage, count in self.stats.items():
            if stage == "initial":
                print(f"初始数据量: {count}")
            else:
                removed = self.stats[list(self.stats.keys())[
                    list(self.stats.keys()).index(stage) - 1
                ]] - count
                print(f"{stage}: {count} (移除了 {removed} 条)")
        final = self.stats[list(self.stats.keys())[-1]]
        print(f"\n最终保留: {final} / {initial} ({final/initial*100:.1f}%)")
        return self

# 运行清洗管线
raw_dataset = load_dataset("json", data_files="my_sft_data.jsonl",
                           split="train")
pipeline = DataCleaningPipeline(raw_dataset)
(
    pipeline
    .exact_dedup()
    .minhash_dedup(threshold=0.85)
    .length_filter(min_len=10, max_len=3072)
    .report()
)
# 输出示例:
# 初始数据量: 150000
# 精确去重: 147200 (移除了 2800 条)
# minhash_dedup: 138368 (移除了 8832 条)
# length_filter: 135600 (移除了 2768 条)
# 最终保留: 135600 / 150000 (90.4%)

NeMo Curator简介：GPU加速的数据策管工具

当你的数据量达到数百万甚至上千万条时，在单机上用 Python 逐条处理就太慢了。这时候需要工业级的、支持分布式和 GPU 加速的数据策管工具。

NeMo Curator 是 NVIDIA 开发的 GPU 加速数据策管工具，专为大规模语言模型训练数据的准备而设计。它的核心特性包括：

  • GPU 加速去重：利用 GPU 的并行计算能力加速 MinHash 和 embedding 去重。在 8 卡 A100 的配置下，NeMo Curator 可以在 1.8 小时内完成 1.1T token 的去重处理——这相当于处理整个 C4 数据集的规模。
  • 内置质量打分模型：使用 FastText 分类器、困惑度模型、以及基于 BERT 的质量评分模型，对文本进行多维度质量评分。
  • 灵活的分布式架构：基于 Dask 和 RAPIDS，可以无缝扩展到多节点集群。
  • HuggingFace 集成：可以直接读取 datasets 格式的数据，也可以输出为兼容格式。
# NeMo Curator 去重代码示例（命令行）
# 对 JSONL 格式的 SFT 数据进行近似去重

nemo_curator dedup \
    --input_dir /data/my_sft_dataset/ \
    --output_dir /data/deduped_sft/ \
    --id_key "id" \
    --text_key "instruction" \
    --num_workers 32 \
    --cache_dir /cache/nemo_curator/ \
    --minhash_threshold 0.85

# 在 4 卡 A100 上处理 500 万条 SFT 数据
# 预计耗时: 15-20 分钟（vs 单机 CPU 方案 6-8 小时）

对于 SFT 数据规模（通常是数十万到数百万条），NeMo Curator 可能有些"杀鸡用牛刀"。不过一旦你的数据管线开始处理预训练级别的数据（数十亿 token），或者你需要频繁迭代数据清洗流程，NeMo Curator 的 GPU 加速能力会带来极大的效率提升。

工具选择的建议：根据数据规模选择工具。对于 < 10 万条 的数据，用 Python 脚本 + datasets 库就够了。对于 10 万 - 500 万条 的数据，建议使用 Spark 或 Dask 的分布式处理方案。对于 > 500 万条 的海量数据，NeMo Curator 的 GPU 加速方案可以带来 10-50 倍的速度提升。

总结一下这一章的核心：数据去重和过滤是 SFT 数据工程中"投入产出比最高"的环节。花一两天搭建清洗管线，通常能带来 5-15% 的模型能力提升——这个 ROI 远高于增加数据量或调超参数。永远记住：喂给模型的数据，宁可少，不可脏。
