第9章 逆向运动学

大家好，我是你们的讲师。上一章我们学了正向运动学——给定关节角度，算末端位姿。但实际机器人应用中，我们面临的往往是反过来的问题：末端要去这里，关节应该怎么转？这就是逆向运动学（Inverse Kinematics, IK）。

如果FK是一道"通顺的送分题"，那IK就是"考验智商的大题"。IK解的存在性、唯一性、数值稳定性——每一个都是值得深入研究的课题。打个比方：FK就像你站在起点直接走到终点（路径唯一），而IK就像你只知道终点在哪里，但要倒推出起点怎么走——而且可能有无数条路能到，或者根本没有路。这正是IK的魅力所在。

9.1 IK问题定义

IK问题形式化地说就是：给定末端执行器的期望位姿 T_d ∈ SE(3)，求关节角度 q ∈ ℝⁿ，使得 FK(q) = T_d。

与FK不同，IK天然具有几个难点：

  • 解的存在性：期望位姿必须在机器人的工作空间（Workspace）内。如果目标点太远或者姿态不可能达到，IK无解。
  • 解的多重性：同一个末端位姿可能对应多组关节角度。典型例子：六轴机器人的"肘上"和"肘下"配置——两种姿态都能让末端到达同一点。
  • 冗余性：对于七轴（7-DOF）机器人，IK有无穷多解，形成一个流形（manifold）。
  • 奇异位形：在某些关节构型附近，数值IK的收敛速度会急剧下降，甚至发散。

根据机器人结构和应用场景的不同，IK解法分为两大类：解析法和数值法。

特性 解析法（Analytic） 数值法（Numerical）
计算速度 极快（微秒级） 中等（毫秒级）
解完整性 可得到所有解 通常只找到一个解
通用性 需特殊结构（Pieper准则） 适用于任何机器人
对初值依赖 无 依赖初始猜测
典型代表 PUMA 560、IKFast KDL、TRAC-IK、Pinocchio+优化

9.2 解析法IK

解析法IK的核心思路是通过几何和代数推导，直接得到关节角度的闭式表达式。解析法的存在性由Pieper准则判定：对于六轴机器人，如果三个相邻关节的轴线交于一点（或三轴平行），则存在解析IK解。大多数工业六轴机器人的最后三轴（腕部）轴线交于一点，满足Pieper准则——这也是PUMA 560如此经典的原因。

以二连杆平面臂为例，几何法极为直观：

import numpy as np

def ik_2link(x, y, l1, l2):
    """平面2R机械臂的解析IK"""
    # 计算肘关节角度 θ₂（余弦定理）
    cos_theta2 = (x**2 + y**2 - l1**2 - l2**2) / (2 * l1 * l2)
    cos_theta2 = np.clip(cos_theta2, -1.0, 1.0)

    # "肘上"和"肘下"两个解
    theta2_up = np.arccos(cos_theta2)     # 肘下（elbow down）
    theta2_low = -np.arccos(cos_theta2)    # 肘上（elbow up）

    # 计算肩关节角度 θ₁
    phi = np.arctan2(y, x)
    psi = np.arctan2(l2 * np.sin(theta2_up), l1 + l2 * np.cos(theta2_up))
    theta1 = phi - psi

    return theta1, theta2_up

# 测试：末端在 (0.6, 0.3)，连杆长 0.4, 0.4
theta1, theta2 = ik_2link(0.6, 0.3, 0.4, 0.4)
print(f"θ₁={np.degrees(theta1):.1f}°, θ₂={np.degrees(theta2):.1f}°")

这个二连杆的IK虽然简单，但揭示了解析IK的核心思想：把问题分解为平面几何的子问题，每个子问题独立求解。对于6轴机器人，一般先将位置和姿态解耦——由前三轴定位置，后三轴定姿态。

工程要点：解析法虽然快，但推导过程容易出错。一个常见的做法是用符号计算工具（如SymPy）验证推导结果，然后再转成高效的C++实现。实际上，IKFast工具就是自动完成这个过程的——你给它一个URDF，它自动生成解析IK的C++代码。

9.3 数值法IK

当机器人不满足Pieper准则时（比如七轴冗余臂、非球形腕部），解析法就无能为力了。这时候我们转向数值法。数值法IK的核心是把IK转换成一个优化问题：

min_q   ||FK(q) - T_d||²

最经典的数值法是基于雅可比伪逆的迭代方法：

Δq = J⁺ · Δx

其中 J⁺ = Jᵀ(J·Jᵀ)⁻¹ 是雅可比矩阵的伪逆，
Δx 是当前末端位姿与目标位姿之间的误差。

但雅可比伪逆在奇异性附近会导致关节速度过大。改进版本是Damped Least Squares（DLS）：

Δq = Jᵀ(J·Jᵀ + λ²·I)⁻¹ · Δx

这里的 λ 是阻尼因子，在奇异性附近自动增大以限制关节速度。DLS的代价是牺牲了部分精度换来了数值稳定性。

踩坑故事：几年前我在调试一个六轴协作机器人的抓取任务时，遇到了一个非常诡异的问题——机器人偶尔会在同一个抓取点附近"抖一下"再抓。排查了一整天，最终发现问题出在IK求解器上：某些构型下，数值IK收敛到了一个"肘上解"，而视觉系统期望的是"肘下解"。两种IK配置的末端位姿完全一致，但肘上解的关节角度导致机器人与旁边的料架发生了碰撞。解决方案是在IK优化中加入关节极限约束和"偏好配置"惩罚项。记住：IK不仅要"能到"，还要"安全地到"。

9.4 IKFast与数值求解器

解析IK解算器中，IKFast是无可争议的标杆。IKFast是OpenRAVE的一部分，它分析URDF中的运动学链结构，自动生成解析IK的C++代码。生成的代码微秒级求解、零迭代、保证找到所有解。

在数值求解器方面，ROS生态中最常用的是：

  • KDL（Kinematics and Dynamics Library）：基于Newton-Raphson迭代，简单可靠，但在奇异性附近容易发散发散。
  • TRAC-IK：KDL的升级版，同时运行两个求解器（KDL + SQP优化），取收敛更快的结果。可靠性显著高于KDL，是目前ROS MoveIt的默认IK求解器。
  • Pinocchio + 数值优化：利用Pinocchio提供快速FK和雅可比计算，用Crocoddyl或类似的数值优化器求解IK。

TRAC-IK的典型用法：

# 在C++中使用TRAC-IK（ROS环境）
#include <trac_ik/trac_ik.hpp>

TRAC_IK::TRAC_IK ik_solver("base_link", "tool0", robot_urdf);
KDL::Chain chain;
KDL::JntArray joint_seed(6), joint_result(6);
KDL::Frame end_effector_pose;

// 设置初始猜测
joint_seed.data = Eigen::VectorXd::Zero(6);

// 设置目标位姿
end_effector_pose.p = KDL::Vector(0.5, 0.2, 0.3);
end_effector_pose.M = KDL::Rotation::Quaternion(0, 0, 0, 1);

// 求解IK
int rc = ik_solver.CartToJnt(joint_seed, end_effector_pose, joint_result);
if (rc >= 0) {
    std::cout << "IK success: " << joint_result.data.transpose() << std::endl;
}

9.5 Pinocchio IK实战

用Pinocchio配合SciPy优化器的纯Python IK实现，适合快速原型验证：

import pinocchio as pin
import numpy as np
from scipy.optimize import minimize

model = pin.buildModelFromUrdf("robot.urdf")
data = model.createData()
ee_frame = model.getFrameId("tool0")

def fk_error(q):
    """返回末端位姿误差向量"""
    pin.forwardKinematics(model, data, q)
    pin.updateFramePlacements(model, data)
    T_ee = data.oMf[ee_frame]
    T_target = pin.SE3(R_target, p_target)
    return pin.log(T_target.inverse() * T_ee).vector  # se(3) 误差

def ik_pinocchio(q_init, target_pos, target_rot):
    global p_target, R_target
    p_target = target_pos
    R_target = target_rot

    # 边界约束
    bounds = [(m.lower, m.upper) for m in model.joints[1:]]

    # 使用SciPy进行数值优化
    result = minimize(
        lambda q: np.linalg.norm(fk_error(q))**2,
        q_init,
        method='SLSQP',
        bounds=bounds,
        options={'ftol': 1e-6, 'maxiter': 200}
    )
    return result.x

# 使用
q0 = np.zeros(6)
pos_des = np.array([0.4, 0.1, 0.3])
rot_des = np.eye(3)
q_sol = ik_pinocchio(q0, pos_des, rot_des)
print("IK解:", q_sol)

注意上面代码中的 pin.log 函数——它将SE(3)上的误差映射到se(3)的切向量空间，这是一个漂亮的黎曼几何技巧。对于冗余机器人，还可以在优化目标中加入 ||q - q_preferred||² 来利用零空间（Null Space）选择偏好构型。

核心洞察：在实际工程项目中，我建议采用"解析+数值"混合策略：先用IKFast生成解析解作为初始猜测，再用数值优化器在小范围内精调。这样既保证了解的存在性和速度，又能处理解析解因关节极限或避障约束而不可用的情况。

本章我们学习了IK的数学定义、解析法与数值法的优劣对比、IKFast等求解器的选择，以及用Pinocchio实现数值IK的实战方法。IK是运动学中最具挑战性的问题——它不仅是数学求解，更是在工作空间、关节极限、避障约束和偏好构型之间寻找最佳平衡的艺术。
