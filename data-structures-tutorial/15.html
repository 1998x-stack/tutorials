开篇：一个递归的段子

网上流传着一个很有意思的段子："要理解递归，你首先需要理解递归。"这个段子本身就是一个递归——它在用自己定义自己。笑完之后你会发现，这恰恰是递归最核心的思想：一个函数调用它自己。

在排序算法那章我们快速提过递归，但从这一章开始，我们要认真地和它打交道了。为什么？因为从这一章开始的回溯、动态规划、贪心——甚至后面的树和图的相关算法——几乎都建立在递归思维之上。可以这样说：没学好递归，后面几章寸步难行。这一章我们会从最基础的递归讲起，然后扩展到分治思想，为后续的所有算法思想打好地基。

什么是递归？——"把大问题缩小成同结构的小问题"

递归（Recursion）的概念其实非常简单：一个函数直接或间接地调用它自身。但真正理解递归，你需要的不是这句定义，而是下面这个思维方式：

不要试图跟踪递归的全过程，而要相信递归函数能正确解决它的小号问题。

我们用两个最经典的例子来建立直觉。

阶乘：最直观的递归

n的阶乘（n!）定义为：n! = n × (n-1) × (n-2) × ... × 1。用数学公式可以写成：

n! = n × (n-1)!   （当 n > 0）
0! = 1              （当 n = 0）

看到没有？n! 的求值依赖于 (n-1)!，而 (n-1)! 的计算方式和 n! 完全一样——只是参数变小了。这就是一个完美的递归结构：

def factorial(n):
    # 终止条件
    if n == 0:
        return 1
    # 递推关系：n! = n * (n-1)!
    return n * factorial(n - 1)

斐波那契数列：双路递归

斐波那契数列的定义是：F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)。这个定义天然就是递归的：

def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

这段代码只有三行，但如果你是第一次接触递归，你可能完全看不懂它为什么能工作。别急，下一节我们就来拆解递归的三要素。

递归三要素

任何一个正确的递归函数，都必须满足三个条件。我把它们叫做递归三要素：终止条件、递推关系、返回值。

用一个故事来帮你记忆。"电影院的座位号"：

你去看电影，坐在第15排。你前面有个高个子挡着，看不清屏幕上的排数编号。你不想站起来，于是你问前面的人："请问这是第几排？"前面的人也不知道，他又问更前面的人……就这样一路问到第1排。第1排的人回头说："这是第1排。"然后信息开始往回传——第2排的人知道了自己是第2排，第3排知道了自己是第3排……最后第15排的你也知道了自己的排数。

这个故事的三个要素对应着递归的三要素：

三要素 电影院故事 代码对应
终止条件（Base Case） 第1排的人知道自己是第1排，不用再往前问了 定义最简单的情况，直接返回结果，不再递归
递推关系（Recurrence Relation） 第n排的人 = 第n-1排的人 + 1 将原问题转化为规模更小的子问题
返回值（Return Value） 每一排都把自己的结果往后传 子问题的结果传递回上一层，最终汇聚成原问题的答案
递归的核心信念：你不需要手动模拟递归的每一步。你只需要保证两件事——（1）终止条件是正确的；（2）递推关系是正确的。如果这两点成立，递归函数就一定是正确的。这就是"数学归纳法"在编程中的体现。

递归的执行过程——调用栈可视化

递归在执行时，每一层调用都会在调用栈（Call Stack）上分配一个新的栈帧（Stack Frame），保存当前函数的局部变量和返回地址。当最深层的递归返回时，栈帧依次弹出，结果逐层传回。

我们可以用阶乘 factorial(3) 的执行过程来看调用栈的变化：

# factorial(3) 的调用栈变化
# 1. 主程序调用 factorial(3)
#    栈: [factorial(3)]
# 2. factorial(3) 调用 factorial(2)
#    栈: [factorial(3), factorial(2)]
# 3. factorial(2) 调用 factorial(1)
#    栈: [factorial(3), factorial(2), factorial(1)]
# 4. factorial(1) 不满足递归条件，直接返回 1
#    栈: [factorial(3), factorial(2)]
# 5. factorial(2) 得到结果 1，返回 2*1=2
#    栈: [factorial(3)]
# 6. factorial(3) 得到结果 2，返回 3*2=6
#    栈: []
# 结果: 6

def factorial(n):
    print(f"  factorial({n}) 被调用，栈深度 = {n+1}")
    if n == 0:
        result = 1
        print(f"  factorial({n}) 返回 {result}")
        return result
    result = n * factorial(n - 1)
    print(f"  factorial({n}) 返回 {result}")
    return result

汉诺塔问题——递归的经典舞台

汉诺塔（Tower of Hanoi）是展现递归之美的最佳案例。有三根柱子A、B、C，A柱上有n个大小不同的圆盘，小的在上大的在下。目标是把所有圆盘从A移到C，每次只能移动一个，且大圆盘不能放在小圆盘上。

递归解法极其优雅：

def hanoi(n, source, target, auxiliary):
    """将 n 个圆盘从 source 移动到 target，借助 auxiliary"""
    if n == 1:
        print(f"移动圆盘 1: {source} -> {target}")
        return
    # 将 n-1 个圆盘从 source 移到 auxiliary
    hanoi(n - 1, source, auxiliary, target)
    # 将最大的圆盘从 source 移到 target
    print(f"移动圆盘 {n}: {source} -> {target}")
    # 将 n-1 个圆盘从 auxiliary 移到 target
    hanoi(n - 1, auxiliary, target, source)

# 测试：3个圆盘
hanoi(3, 'A', 'C', 'B')

输出结果展示了7步移动过程。你可能觉得这像魔法——但请相信递归三要素：终止条件是 n=1，递推关系是"先把n-1个移走→移最大的→再把n-1个移回来"，每一步都是对的，整体就是对的。

二分查找——递归版与迭代版

二分查找（Binary Search）是分治思想的第一个应用。在一个有序数组中查找目标值，每次取中间元素比较，把搜索范围缩小一半。

递归版二分查找

def binary_search_recursive(arr, left, right, target):
    """递归版二分查找，返回目标值的下标，未找到返回 -1"""
    if left > right:
        return -1  # 终止条件：搜索区间为空

    mid = left + (right - left) // 2  # 防止整数溢出

    if arr[mid] == target:
        return mid
    elif arr[mid] < target:
        return binary_search_recursive(arr, mid + 1, right, target)
    else:
        return binary_search_recursive(arr, left, mid - 1, target)

迭代版二分查找

def binary_search_iterative(arr, target):
    """迭代版二分查找"""
    left, right = 0, len(arr) - 1

    while left <= right:
        mid = left + (right - left) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return -1
递归 vs 迭代：递归版代码更"数学化"，可读性强；迭代版性能更好（没有函数调用开销），且不会栈溢出。在实际工程中，能用迭代就用迭代，递归更多是用于理解分治思想的训练工具。

二分查找的四种变体

真正的二分查找远不止查到一个值那么简单。面试中常考的是下面四种变体，它们也叫"二分查找边界"问题：

变体 目标 核心逻辑
标准查找 找任意一个等于 target 的位置 找到就返回，否则缩小范围
第一个等于 target 找最左边等于 target 的位置 找到后不返回，继续向左收缩 right=mid-1
最后一个等于 target 找最右边等于 target 的位置 找到后不返回，继续向右收缩 left=mid+1
插入位置 找第一个 >= target 的位置 返回 left（即插入点） 
def first_equals(arr, target):
    """第一个等于 target 的元素下标"""
    left, right = 0, len(arr) - 1
    result = -1
    while left <= right:
        mid = left + (right - left) // 2
        if arr[mid] == target:
            result = mid
            right = mid - 1  # 继续向左找
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return result


def insertion_position(arr, target):
    """找到插入 target 的位置（第一个 >= target 的下标）"""
    left, right = 0, len(arr)
    while left < right:
        mid = left + (right - left) // 2
        if arr[mid] < target:
            left = mid + 1
        else:
            right = mid
    return left

分治思想

分治（Divide and Conquer）是递归最强大的应用场景。它的核心思想就三步：

  1. 分（Divide）：将原问题分解为若干个规模更小、结构相同的子问题。
  2. 治（Conquer）：递归地解决每个子问题。当子问题足够小时直接解决。
  3. 合（Combine）：将子问题的解合并为原问题的解。

归并排序——分治的教科书案例

我们之前在排序算法一章学过归并排序，现在用分治的视角重新审视它：

def merge_sort(arr):
    """归并排序：分治思想的完美体现"""
    if len(arr) <= 1:
        return arr  # 终止条件：单个元素已经有序

    # 1. 分：将数组均匀分成两半
    mid = len(arr) // 2
    left_half = merge_sort(arr[:mid])
    right_half = merge_sort(arr[mid:])

    # 2. 合：将两个有序子数组合并为一个有序数组
    return merge(left_half, right_half)


def merge(left, right):
    """合并两个有序数组"""
    result = []
    i = j = 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    # 把剩余元素追加到结果中
    result.extend(left[i:])
    result.extend(right[j:])
    return result

归并排序的时间复杂度是 O(n log n)，空间复杂度是 O(n)。这个 O(n log n) 是怎么算出来的？这就引出了我们分析递归复杂度的利器——主定理（Master Theorem）。

主定理——快速计算递归复杂度

对于形如 T(n) = a·T(n/b) + f(n) 的递归式（其中 a ≥ 1, b > 1），主定理给出了三种情况：

情况 条件 复杂度 典型算法
情况一 f(n) = O(nlogb a - ε) T(n) = Θ(nlogb a) 二分查找 T(n) = T(n/2) + O(1)
情况二 f(n) = Θ(nlogb a) T(n) = Θ(nlogb a · log n) 归并排序 T(n) = 2T(n/2) + O(n)
情况三 f(n) = Ω(nlogb a + ε) T(n) = Θ(f(n)) 快速排序最坏 T(n) = T(n-1) + O(n)

用主定理算一下归并排序：T(n) = 2T(n/2) + O(n)，a=2, b=2，
logb a = log2 2 = 1，f(n) = O(n) = Θ(n1)，符合情况二，
因此 T(n) = Θ(n log n)。

主定理的记忆技巧：比较 nlogba 和 f(n) 谁的增长速度更快。前者大就是情况一，两者相当就是情况二，后者大就是情况三。如果忘记细节也没关系——面试中很少让你手算主定理，你只需要知道归并排序 O(n log n)、二分查找 O(log n) 就够了。

递归的注意事项

递归虽美，但用不好就是一把双刃剑。

问题一：栈溢出（Stack Overflow）

递归的每一层调用都要占用栈空间。Python 默认的递归深度限制是 1000 层。如果你的递归深度超过这个数，就会抛出 RecursionError。

踩坑故事——"递归写得很爽，直到生产环境 StackOverflow"

一位朋友在写一个树形结构的遍历函数时，使用了递归实现。本地测试数据只有几十层，跑得飞起。部署到生产环境后，有一棵树的深度达到了几万个节点——结果服务直接崩溃，错误信息是 RecursionError: maximum recursion depth exceeded。

他后来把递归改成了迭代（用栈模拟），问题解决。教训是：递归深度不可控时，一定要考虑栈溢出的风险。生产环境慎用递归，尤其是处理不确定规模的数据时。

问题二：重复计算

还记得上面那个三行的斐波那契函数吗？它有一个严重问题——重复计算。计算 fibonacci(5) 需要计算 fibonacci(4) 和 fibonacci(3)，而计算 fibonacci(4) 又需要 fibonacci(3) 和 fibonacci(2)——fibonacci(3) 被重复计算了两次！

这个递归展开后是一个二叉树，节点数呈指数级增长。fibonacci(n) 的时间复杂度是 O(2n)——n=50 时，你的电脑就要计算超过 1015 次，这辈子都算不完。

这就引出了我们下一章要学的核心思想之一——记忆化（Memoization）。把已经算过的结果存起来，避免重复计算。再往深处走，这就是动态规划（Dynamic Programming）的雏形了。

问题三：尾递归优化

尾递归（Tail Recursion）是指递归调用发生在函数的最后一步（即 return 语句中只有递归调用，没有其他操作）。某些编译器可以对尾递归进行优化——复用当前栈帧，避免栈增长。

# 非尾递归：返回后还要乘以 n
def factorial(n):
    if n == 0: return 1
    return n * factorial(n - 1)

# 尾递归：递归调用在最后一步
def factorial_tail(n, acc=1):
    if n == 0: return acc
    return factorial_tail(n - 1, n * acc)

可惜的是，Python 不支持尾递归优化。Guido van Rossum（Python之父）明确说过"尾递归优化不是Python的风格"。所以你在 Python 中写尾递归并不会有什么性能提升。但如果你用 C++ 或 Java（某些JVM实现支持），尾递归确实可以避免栈溢出。

递归使用决策树：
  • 递归深度可控（≤ 几百层）→ 可以用递归，代码简洁
  • 递归深度不可控 → 用栈模拟或迭代
  • 有大量子问题重叠 → 用记忆化递归或 DP（下一章的内容）
  • 平台支持尾递归优化 → 考虑尾递归写法
