24.1 链路预算的核心公式

大家好，我是你们的讲师。上一章我们学会了如何用仿真工具设计一副性能优良的天线。但一个完整的无线系统远不止一副天线——我们需要知道：发射天线辐射出去的功率，到底有多少能到达接收天线？这个问题的答案，就是链路预算（Link Budget）。

链路预算本质上是一张"加减法"大表——所有的增益都加上，所有的损耗都减去，最后看看接收端还能剩下多少信号功率。这条最完整的链路方程如下：

Prx = Ptx + Gtx - Ltx + Grx - Lrx - FSL - Latm - Lrain - Lpol - Lfade + Gcode

其中每一项都是dB单位。逐一解读：

  • Ptx：发射机输出功率（dBm或dBW）
  • Gtx：发射天线增益（dBi）
  • Ltx：发射端馈线损耗（dB）——从发射机到天线之间的电缆、连接器、滤波器损耗
  • Grx：接收天线增益（dBi）
  • Lrx：接收端馈线损耗（dB）
  • FSL：自由空间路径损耗（Free Space Loss, dB）——我们第11章的核心公式
  • Latm：大气吸收损耗（dB）——主要是氧气和水蒸气
  • Lrain：降雨衰减（dB）——对10 GHz以上频段影响显著
  • Lpol：极化失配损耗（dB）——发射和接收极化不对齐造成的损失
  • Lfade：衰落余量（dB）——应对多径衰落、闪烁等不可预测的信号波动
  • Gcode：编码增益（dB）——纠错编码带来的等效信噪比提升

为什么RF工程师离不开dB？因为无线链路中的信号功率可以从纳瓦级（-60 dBm）到千瓦级（+60 dBm），跨度达12个数量级。用线性单位会写到手抽筋；用dB只需要几十位数字。更重要的是：乘法变成了加法——整个链路预算就变成了一张加减法表格。简洁优雅。

几个常见单位的区分：

  • dBm：相对于1 mW的功率，0 dBm = 1 mW，+30 dBm = 1 W，+40 dBm = 10 W
  • dBW：相对于1 W的功率，0 dBW = 1 W，-30 dBW = 1 mW
  • dBi：天线增益相对于理想全向辐射器（各向同性源）
  • dBd：天线增益相对于半波偶极子。0 dBd = 2.15 dBi（因为半波偶极子本身的增益就是2.15 dBi）
一个真实的教训——dBW和dBm混淆的代价：2010年代初，一颗商业通信卫星在轨测试中暴露出上行链路余量严重不足——实际接收到的信号比预期低了整整30 dB。调查发现，工程设计阶段有人将地球站的发射功率标注为"60 dBW"，但链路预算中作为"60 dBm"参与了计算。60 dBW = 1,000,000 W，60 dBm = 1,000 W——差了一千倍，也就是30 dB！这颗卫星虽然仍在工作，但地面段不得不额外投资数百万美元升级更大的天线来弥补这个错误。据估算，这颗卫星在整个寿命期内因此损失了约2亿美元的潜在收入。这个故事告诉我们：单位标注和单位换算是RF工程中最基本也最致命的细节。

24.2 卫星通信链路预算

卫星通信是链路预算最典型的应用场景。从地面站到卫星、再从卫星回到地面，信号经历了两次空间传播、两次大气衰减和一次星上处理（或变频转发）。我们通常关心的是载波与噪声功率密度比 C/N₀，因为它直接决定了链路能否达到所需的误码率。

上行链路：

C/N₀up = EIRPearth - FSLup + G/Tsat + 228.6 - Latm,up

其中：

  • EIRPearth = Ptx + Gtx - Ltx——地球站的有效全向辐射功率
  • G/Tsat——卫星接收系统的品质因数（天线增益除以系统噪声温度，单位dB/K）
  • 228.6——玻尔兹曼常数k = 1.38×10⁻²³ J/K的dB形式（10·log₁₀(k) = -228.6）

下行链路：

C/N₀down = EIRPsat - FSLdown + G/Tearth + 228.6 - Latm,down

总C/N₀：上行和下行C/N₀按噪声功率叠加，与电阻并联的公式非常相似：

1/(C/N₀)total = 1/(C/N₀)up + 1/(C/N₀)down + 1/(C/N₀)IM

其中(C/N₀)IM是星上功放交调噪声分量——转发器中的非线性放大产生的互调产物。

G/T——接收系统品质因数：G/T是卫星通信系统中最核心的指标之一（注意是"除法"关系，不是减法——只是在对数坐标系中呈现为减法）。高的G/T意味着接收系统能在噪声中提取出更微弱的信号。G/T越高，链路余量越大，或者同等余量下可以降低发射功率。一颗典型的GEO通信卫星的G/T约为-5到+5 dB/K。大型地球站的G/T可以达到30-40 dB/K。

ITU降雨衰减模型：ITU-R P.618和P.837建议书提供了全球范围的降雨衰减预测方法。链路预算中需要指定一个可用度目标：99.5%可用度意味着一年中大约43.8小时的不可用时间；99.99%可用度意味着只有53分钟的不可用时间。要求的可用度越高，需要留出的降雨余量就越大。对于Ka波段（20/30 GHz）的卫星通信，最坏月份的降雨余量可能高达10-15 dB。

下面是一个完整的Python卫星链路预算计算器：

import numpy as np

def satellite_link_budget(
    f_up, f_down,          # 上行/下行频率 (Hz)
    distance,              # 星地距离 (m)
    P_tx_earth,            # 地球站发射功率 (dBW)
    G_earth_up,            # 地球站发射天线增益 (dBi)
    L_tx,                  # 发射馈线损耗 (dB)
    G_sat_rx,              # 卫星接收天线增益 (dBi)
    T_sat,                 # 卫星接收系统噪声温度 (K)
    P_sat,                 # 卫星转发器输出功率 (dBW)
    G_sat_tx,              # 卫星发射天线增益 (dBi)
    G_earth_rx,            # 地球站接收天线增益 (dBi)
    T_earth,               # 地球站接收系统噪声温度 (K)
    B,                     # 信号带宽 (Hz)
    L_rain_up=0,           # 上行降雨损耗 (dB)
    L_rain_down=0,         # 下行降雨损耗 (dB)
    L_atm_up=0.3,          # 上行大气损耗 (dB)
    L_atm_down=0.3         # 下行大气损耗 (dB)
):
    c = 3e8
    # 自由空间路径损耗
    FSL_up = 20*np.log10(distance) + 20*np.log10(f_up) \
             + 20*np.log10(4*np.pi/c)
    FSL_down = 20*np.log10(distance) + 20*np.log10(f_down) \
               + 20*np.log10(4*np.pi/c)

    # 上行 C/N0
    EIRP_earth = P_tx_earth + G_earth_up - L_tx
    G_T_sat = G_sat_rx - 10*np.log10(T_sat)
    CN0_up = EIRP_earth - FSL_up - L_atm_up - L_rain_up + G_T_sat + 228.6

    # 下行 C/N0
    EIRP_sat = P_sat + G_sat_tx
    G_T_earth = G_earth_rx - 10*np.log10(T_earth)
    CN0_down = EIRP_sat - FSL_down - L_atm_down - L_rain_down \
               + G_T_earth + 228.6

    # 总 C/N0 (忽略交调噪声)
    CN0_total = -10 * np.log10(
        10**(-CN0_up/10) + 10**(-CN0_down/10)
    )

    # 载噪比 C/N
    CN = CN0_total - 10*np.log10(B)

    return CN0_up, CN0_down, CN0_total, CN, EIRP_earth, EIRP_sat

# 示例：Ku波段GEO卫星
f_up, f_down = 14e9, 12e9          # Ku-band
distance = 38000e3                  # GEO
P_tx_earth, G_earth_up, L_tx = 10, 50, 2
G_sat_rx, T_sat = 30, 500
P_sat, G_sat_tx = 5, 35
G_earth_rx, T_earth = 45, 150
B = 36e6  # 36 MHz转发器

cn0_up, cn0_down, cn0_total, cn, eirp_e, eirp_s = \
    satellite_link_budget(f_up, f_down, distance,
                          P_tx_earth, G_earth_up, L_tx,
                          G_sat_rx, T_sat,
                          P_sat, G_sat_tx,
                          G_earth_rx, T_earth, B)

print(f"===== Ku波段卫星链路预算 =====")
print(f"地球站EIRP: {eirp_e:.1f} dBW")
print(f"卫星EIRP: {eirp_s:.1f} dBW")
print(f"上行C/N0: {cn0_up:.1f} dBHz")
print(f"下行C/N0: {cn0_down:.1f} dBHz")
print(f"总C/N0:   {cn0_total:.1f} dBHz")
print(f"C/N (36 MHz): {cn:.1f} dB")
print(f"QPSK所需C/N: ~10 dB - 余量 {cn - 10:.1f} dB")

24.3 5G NR链路预算

5G NR（New Radio）的链路预算与传统卫星通信有几个关键差异：首先，5G工作在远高于4G的频段（FR2频段直达24-52 GHz毫米波）；其次，5G使用大规模MIMO和波束赋形，天线增益随用户位置而变化；第三，5G的信道带宽极大（最大400 MHz），噪声功率相应增加。

5G链路预算的独特挑战：

  • 极高的路径损耗：28 GHz相比3.5 GHz，自由空间损耗高出18 dB（因为FSL ∝ f²）。
  • 降雨衰减：在28 GHz，中等降雨率（10 mm/h）的衰减可达5-10 dB/km。
  • 植被/树叶损耗：有树叶的树木在28 GHz可产生10-30 dB的额外衰减。
  • 人体遮挡损耗：手、头、身体对毫米波信号的遮挡可达20-30 dB。
  • 建筑物穿透损耗：现代节能玻璃对毫米波的衰减可达30-40 dB。

最大路径损耗（MPL）：MPL是链路预算的"输出"——对于给定的发射功率、天线配置和接收机灵敏度，能承受的最大路径损耗是多少。如果MPL大于实际路径损耗，链路就"闭合"了（link closes）。5G基站部署中，MPL决定了小区的覆盖半径。

波束赋形增益：毫米波5G基站通常使用64单元或更大的阵列天线。每个阵元后接移相器和衰减器，通过波束赋形（beamforming）将能量集中于用户方向。一个64单元的阵列可以提供约18 dBi的阵列增益（10·log₁₀(64) ≈ 18 dB）。这恰好可以补偿28 GHz相比3.5 GHz高出约18 dB的自由空间路径损耗——这也是5G毫米波系统能够实际工作的重要原因之一！

参数 FR1 (3.5 GHz) FR2 (28 GHz)
发射功率 (基站) 200 W (53 dBm) 10 W (40 dBm)
天线阵元数 (基站) 64 (8×8) 256 (16×16)
阵列增益 (基站) ~18 dBi ~24 dBi
信道带宽 100 MHz 400 MHz
接收机噪声系数 5 dB 8 dB
接收机灵敏度 (QPSK) -96 dBm -83 dBm
自由空间路径损耗 (100 m) 103.4 dB 121.4 dB
最大允许路径损耗 (MPL) ~158 dB ~138 dB
典型覆盖半径 (城区宏站) ~500 m ~150 m
5G链路预算的底层逻辑：毫米波（FR2）虽然每个独立链路的覆盖半径比FR1小得多，但结合密集部署（小站间距100-200米）和波束赋形，可以在热点区域提供数十Gbps的极高容量。FR1负责广域覆盖，FR2负责热点容量——两者互补，这就是5G的"双层网络"策略。

24.4 干扰分析与频率协调

链路预算告诉你：在没有干扰的情况下，信号能走多远。但现实世界中，众多无线系统共享有限的频谱资源——干扰往往成为真正的限制因素。一个设计良好的无线链路不仅要克服噪声，还要在干扰环境中可靠工作。

干扰类型：

  • 同频干扰（Co-Channel Interference）：两个系统使用相同频率，距离不够远。这是蜂窝网络中最主要的干扰来源——因此有了频率复用（frequency reuse）的概念
  • 邻频干扰（Adjacent Channel Interference）：相邻频道的信号功率泄漏到目标频段内。由发射机的邻道泄漏比（ACLR）和接收机的邻道选择性（ACS）共同决定
  • 谐波干扰（Harmonic Interference）：大功率发射机的二次或三次谐波落入其他系统的接收频段
  • 交调干扰（Intermodulation）：两个或多个强信号在非线性器件中混合，产生新的频率分量。我们将在25.1节中深挖PIM（无源交调）问题
  • 交叉极化干扰（Cross-Polarization Interference）：垂直极化信号泄漏到水平极化通道（或反之），由天线的交叉极化鉴别度（XPD）决定

C/I（载波干扰比）要求：不同调制方式对C/I的容忍度差异很大。QPSK大约需要10 dB的C/I来维持可接受的误码率，64QAM需要约22 dB，256QAM需要约28 dB，而1024QAM需要约33 dB。这就是为什么高阶调制只适用于干扰环境良好（通常是光纤回传的小站）的场景——在小区边缘，C/I低，系统自动降阶到QPSK以保证连接不中断。

工程实例：计算两个5G基站的最小同频隔离距离

假设两个5G基站都工作在3.5 GHz，使用64单元阵列（增益18 dBi），发射功率53 dBm。要求C/I ≥ 22 dB（64QAM）。接收机噪声基底为-96 dBm。当两个基站同时向各自的用户发送数据时，第一个用户的期望信号来自其服务基站（距离200 m），干扰信号来自邻站（距离待求）。

期望信号功率：Pdes = 53 + 18 - FSL(200m) + 18 ≈ 53 + 18 - 103.4 + 18 = -14.4 dBm
干扰容忍上限：Pint,max = Pdes - C/I = -14.4 - 22 = -36.4 dBm
由Pint,max反推FSL：FSL = 53 + 18 + 18 - (-36.4) = 125.4 dB
由FSL反推距离：d = 10^((125.4 - 32.45 - 20·log₁₀(3500))/20) ≈ 5,200 m

这意味着在3.5 GHz下，两个同频基站的间距至少需要5.2公里才能满足64QAM的C/I要求。这就是为什么蜂窝网络必须进行频率规划——同一频率不能在每个小区都使用，而是以一定模式复用。

频率协调：在国际层面，ITU-R（国际电信联盟无线电通信部门）负责全球频谱划分和协调规则。各国则有各自的频谱管理机构（如中国的工信部无线电管理局、美国的FCC）。对于部署在卫星地球站或射电天文台附近的无线系统，往往需要划定协调区——在协调区内，新系统的部署需要与现有用户协商，确保不会造成有害干扰。一个典型的协调区半径为数十到数百公里，具体取决于频率和功率。

24.5 系统噪声与灵敏度分析

链路预算的最终目标，是确保接收端的信噪比（SNR或C/N）高于解调所需的最低门限。而接收机自身的噪声特性，决定了这个门限的具体数值。

噪声系数级联（Friis公式）

NFtotal = NF₁ + (NF₂ - 1)/G₁ + (NF₃ - 1)/(G₁·G₂) + ...

注意，这个公式中的NF和G是线性比值（不是dB）。它揭示了接收链路设计中最重要的规律：第一级放大器（LNA）的噪声系数主导了整个链路的噪声性能。因为后级的噪声贡献都被第一级的增益"除"掉了。

工程含义：这就是为什么LNA必须尽可能靠近天线放置！任何位于LNA之前的器件——电缆、连接器、滤波器——都会直接贡献到总噪声系数中。如果天线和LNA之间有一根损耗3 dB的电缆，NF就直接增加了3 dB，且后续LNA无法补救。因此工程上常见的做法是：将LNA直接安装在塔顶的天线端口下方（Tower-Mounted Amplifier, TMA），而不是在机房里的机架上。

接收机灵敏度

Smin (dBm) = -174 + 10·log₁₀(B) + NF + SNRmin

其中-174 dBm/Hz是室温下（290 K）的热噪声功率谱密度（k·T₀）。举一个真实的例子：一个20 MHz带宽的LTE接收机，NF = 5 dB，解调所需SNR = 10 dB（QPSK），灵敏度为：

Smin = -174 + 10·log₁₀(20×10⁶) + 5 + 10 = -174 + 73 + 5 + 10 = -86 dBm

ADC的影响

模数转换器（ADC）是现代接收机数字化的关键环节。ADC会引入量化噪声——理论上，N位ADC的最大SQNR（信号量化噪声比）约为6.02N + 1.76 dB。一个12位ADC的SQNR约为74 dB。但实际中还需要考虑：

  • SFDR（无杂散动态范围）：ADC能分辨的最大信号与最大杂散分量之间的比值，典型值70-90 dB
  • 采样抖动（Sampling Jitter）：时钟的不稳定性会导致采样误差，在高频时尤其显著。对于1 GHz以上的信号，即使只有0.1 ps的均方根抖动，也会显著限制SNR

动态范围

接收机的动态范围定义为最大可接受输入信号与最小可检测信号之间的比值。上限通常由1 dB压缩点（P1dB）或三阶截点（IP3）决定。一个典型接收链路需要处理-100 dBm的微弱信号到-20 dBm的强信号（如附近基站的信号），动态范围需求高达80 dB。

完整接收链路计算演练：

假设一个5G毫米波接收链路：天线(24 dBi) → 波导(1 dB损耗) → LNA(增益20 dB, NF=2.5 dB) → 混频器(变频损耗7 dB, NF=7 dB) → 中频放大器(增益30 dB, NF=5 dB) → ADC。

import numpy as np

def friis_nf(nf_db_list, gain_db_list):
    """
    计算级联噪声系数 (Friis公式)
    nf_db_list: 各级噪声系数 (dB)
    gain_db_list: 各级增益 (dB)
    """
    nf_linear = [10**(nf/10) for nf in nf_db_list]
    gain_linear = [10**(g/10) for g in gain_db_list]

    nf_total_linear = nf_linear[0]
    cum_gain = 1.0
    for i in range(1, len(nf_linear)):
        cum_gain *= gain_linear[i-1]
        nf_total_linear += (nf_linear[i] - 1) / cum_gain

    return 10 * np.log10(nf_total_linear)

# 5G毫米波接收链路
nf_list = [2.5, 7, 5]  # LNA, 混频器, IF放大器
gain_list = [20, -7, 30]  # 增益/损耗

nf_total = friis_nf(nf_list, gain_list)
print(f"总噪声系数: {nf_total:.1f} dB")

# 接收机灵敏度 (400 MHz带宽, 64QAM需SNR=22 dB)
B = 400e6
SNR_min = 22
S_min = -174 + 10*np.log10(B) + nf_total + SNR_min
print(f"接收机灵敏度 (400 MHz, 64QAM): {S_min:.1f} dBm")

# P1dB级联 (简化估算)
P1dB_LNA = -10  # dBm
gain_LNA = 20   # dB
P1dB_IF = 10    # dBm
P1dB_total = P1dB_IF - (gain_LNA - 7)  # 混频器有7dB损耗
print(f"系统P1dB: {P1dB_total:.1f} dBm")

# 动态范围
DR = P1dB_total - S_min
print(f"无杂散动态范围: {DR:.1f} dB")
===== 5G毫米波接收链路分析 =====
总噪声系数: 3.2 dB
接收机灵敏度 (400 MHz, 64QAM): -174 + 86 + 3.2 + 22 = -62.8 dBm
系统P1dB: -47 dBm
无杂散动态范围: 15.8 dB

可以看到，这个接收链路的动态范围只有约16 dB——意味着它无法同时处理非常强和非常弱的信号。这正是毫米波系统面临的真实挑战：波束赋形的高增益使接收信号动态范围变窄，而宽带宽又抬高了噪声基底。实际系统中需要加入可调衰减器（Step Attenuator）来扩展动态范围。

设计口诀："前放低噪，后放高增，中间滤波。"——第一级LNA追求低噪声系数和适中增益（15-25 dB），后面的放大器追求高增益和线性度（高P1dB/IP3），混频器前需要镜像抑制滤波器。ADC则根据最坏情况下的信号功率来选择满量程范围。

链路预算和系统噪声分析帮助我们回答了"系统能工作吗"的问题——从发射功率到天线增益、从路径损耗到接收机灵敏度，每一步都是可以计算的。但现实世界不会完全按照计算书来运行：生产公差、装配偏差、环境变化、器件老化——这些不确定性让链路预算成为一张"有余量的预算表"而不是精确的预测。那么，当我们把理论计算和仿真模型带到实际测试中时，会发生什么？第25章我们将通过一系列真实的实测案例，看看测量结果与理论预测之间的差距，并从中学到对付这些偏差的实战经验。
