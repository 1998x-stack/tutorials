开篇：维纳做不到的，卡尔曼来做

大家好，我是你们的讲师。上一章我们学习了维纳滤波——一个在频域中优雅地解决了"从噪声中恢复信号"的问题。但维纳滤波有一个致命的软肋：它要求信号和噪声都是平稳的。在现实世界中，目标的运动是变速的、信道的衰落是时变的、心电信号的波形是随着呼吸在变化的——几乎没有什么重要的信号是真正平稳的。

卡尔曼滤波（Kalman filtering）的出现彻底改变了这一局面。1960年，鲁道夫·卡尔曼（Rudolf Kalman）发表了一篇题为"A New Approach to Linear Filtering and Prediction Problems"的论文，提出了一种完全在时域、基于状态空间、以递推方式工作的最优滤波算法。它不仅适用于非平稳信号，计算量还极其小（每次更新只需要几个矩阵运算），而且形式优雅到让人惊叹——五个方程，囊括了整个预测-修正循环的全部内容。如今，从阿波罗登月的导航计算机到你的手机GPS芯片，卡尔曼滤波几乎无处不在。

从维纳到卡尔曼

要理解卡尔曼滤波为什么"高级"，先要看清维纳滤波的局限：

  • 维纳滤波：频域方法，要求信号平稳，需要知道信号和噪声的全部过去和未来的统计特性（Wiener-Hopf 方程是无限时间积分），计算的是整个冲击响应 h(t)，然后一次性处理所有数据。形象地说，维纳滤波是"洗照片"——拍完整个胶卷后一次性显影。
  • 卡尔曼滤波：时域方法，不要求平稳，只依赖当前时刻的状态和观测，用递推方式逐次更新估计。形象地说，卡尔曼滤波是"看直播"——每一帧画面到达时马上处理，不需要等待整场比赛结束。

这个转变不仅仅是数学上的便利——它代表了信号处理思想的一次飞跃：从"批量处理"到"在线处理"。卡尔曼滤波不需要存储全部历史数据，只需要记住当前的状态估计和估计误差协方差，就可以在每次新的观测到来时更新估计。这种"递推"特性，使得卡尔曼滤波特别适合实时系统——而实时系统恰恰是大多数工程应用的真实运行方式。

历史趣闻：卡尔曼1960年发表论文时，他的思想并没有立刻被接受。当时NASA的工程师们正为阿波罗登月的导航问题头疼——阿波罗飞船的导航计算机内存只有 2K 字（约 4KB），根本装不下维纳滤波所需的数据。卡尔曼在访问NASA艾姆斯研究中心时与工程师 Stanley Schmidt 交流，后者立刻意识到卡尔曼滤波的递推特性完美适配了阿波罗的有限计算资源。此后，卡尔曼滤波成了阿波罗导航系统的一部分，并随着阿波罗的成功而一举成名。

状态空间模型

卡尔曼滤波的数学基础是状态空间模型（state-space model）。它用两个方程来描述一个动态系统：

状态方程：系统如何演变

x_k = F · x_{k-1} + w_{k-1}

其中：

  • x_k：k 时刻的系统状态向量。例如对于目标跟踪问题，x_k 可能包含目标的位置和速度：[p_x, p_y, v_x, v_y]^T。
  • F：状态转移矩阵，描述系统从 k-1 时刻到 k 时刻的动态变化。如果目标是匀速运动且采样时间为 Δt，则 F 是一个块对角矩阵，每个维度包含一个 [[1, Δt], [0, 1]] 的子块。
  • w_{k-1}：过程噪声，通常假设为零均值高斯白噪声，协方差矩阵为 Q。它刻画了"模型的不确定性"——即使 F 是完美的，实际运动也可能有随机扰动（如风对飞机的影响）。

观测方程：传感器如何测量

z_k = H · x_k + v_k

其中：

  • z_k：k 时刻的观测量。例如雷达测量的是目标距离和角度。
  • H：观测矩阵，将状态空间映射到测量空间。如果你的状态包含位置，而传感器直接测量位置，那么 H 就是一个简单的选择矩阵。
  • v_k：观测噪声，通常假设为零均值高斯白噪声，协方差矩阵为 R。它刻画了传感器的测量误差。
状态空间的哲学：状态空间模型最深刻的思想是——把"我们关心的变量"（状态）和"我们实际能测量到的变量"（观测）分开。你的状态是"目标在三维空间中的真实位置和速度"，但你只能得到一个有延迟、有噪声的雷达回波。卡尔曼滤波的工作就是：利用系统的物理规律（状态方程）和传感器的模型（观测方程），从观测中"反推"出状态的最优估计。

卡尔曼滤波的五个方程

卡尔曼滤波的全部内容可以浓缩为五个方程，分为"预测"和"更新"两个步骤：

预测步（Time Update）

1. 状态预测：
   x̂_{k|k-1} = F · x̂_{k-1|k-1}

2. 误差协方差预测：
   P_{k|k-1} = F · P_{k-1|k-1} · F^T + Q

预测步的直觉很简单：基于上一时刻的估计，用状态转移矩阵 F 推算当前时刻的状态。同时，因为预测是不完美的（过程噪声 Q 的存在），我们需要增大误差协方差 P 来反映预测的不确定性增加。

更新步（Measurement Update）

3. 卡尔曼增益：
   K_k = P_{k|k-1} · H^T · (H · P_{k|k-1} · H^T + R)^{-1}

4. 状态更新：
   x̂_{k|k} = x̂_{k|k-1} + K_k · (z_k - H · x̂_{k|k-1})

5. 误差协方差更新：
   P_{k|k} = (I - K_k · H) · P_{k|k-1}

更新步做的事情是：当新的观测 z_k 到达时，比较"实际观测值"和"根据预测算出的期望观测值 H·x̂_{k|k-1}"，两者的差就是新息（innovation）。然后用卡尔曼增益 K_k 做加权，修正状态估计。

卡尔曼滤波的运作流程可以总结为一个循环：

① 根据物理模型预测下一时刻的状态 → ② 当新观测到来时，计算"预测值"和"实际值"的差异 → ③ 按卡尔曼增益的比例修正状态 → ④ 更新误差协方差 → ⑤ 回到①。

每一次循环，卡尔曼增益 K_k 自动地完成了"我应该更相信模型（预测）还是更相信传感器（观测）"的权衡。预测误差协方差 P_{k|k-1} 大时，K 变大，更信任观测；观测噪声 R 大时，K 变小，更信任模型。

卡尔曼滤波的直觉

预测-修正循环

卡尔曼滤波的本质就是一个预测-修正循环（predict-correct cycle）。这个循环在日常生活中非常常见：

  • 你约了一个朋友吃饭，你知道他通常走路速度是 1.4 m/s（状态模型），上次见他是在地铁出口（上一状态），那么你可以"预测"他目前大约在什么地方（状态预测）。
  • 然后你给他打了个电话，他告诉你他路过了一家便利店——这就是"观测"。
  • 你根据预测的位置和观测到的位置（可能他说得不精确），折中一下得到一个更好的位置估计。如果他的步行速度很稳定（Q 小），你更相信预测；如果他的定位描述很准确（R 小），你更相信观测。

这个循环在卡尔曼滤波中以数学方式精确运行——每一步都有严谨的概率论依据：在线性高斯假设下，卡尔曼滤波给出的就是 MMSE 最优估计。

卡尔曼增益 K 的物理含义

卡尔曼增益 K_k 是卡尔曼滤波中最重要的单一量。它的大小决定了滤波器的行为：

如果 P_{k|k-1} >> R（模型很不确定，观测很可靠）：
   K_k → H^{-1}  →  完全相信观测，忽略预测

如果 P_{k|k-1} << R（模型很确定，观测噪声大）：
   K_k → 0  →  完全相信模型，忽略观测

更妙的是，卡尔曼增益会自适应地调整：在初始阶段，P 通常较大（因为我们不知道初始状态），K 也较大，滤波器会积极利用观测来收敛；随着滤波器的运行，P 逐渐减小，K 也逐渐变小，滤波器变得更加依赖模型预测。这种"从信任观测到信任模型"的平滑过渡，是卡尔曼滤波区别于固定增益滤波器的核心优势。

扩展卡尔曼滤波（EKF）简介

经典卡尔曼滤波假设系统的状态方程和观测方程都是线性的。但现实中的大多数系统都是非线性的——例如，飞机的空气动力学方程包含速度的平方项，雷达的距离-角度测量涉及三角函数。这些非线性关系无法用 F 和 H 矩阵精确描述。

扩展卡尔曼滤波（Extended Kalman Filter, EKF）的解决思路非常直接：把非线性函数在估计点处做一阶泰勒展开（线性化），然后用线性化的雅可比矩阵代替 F 和 H：

非线性状态方程：x_k = f(x_{k-1}) + w_{k-1}
        线性化：F_k = ∂f/∂x |_{x̂_{k-1|k-1}}

        非线性观测方程：z_k = h(x_k) + v_k
        线性化：H_k = ∂h/∂x |_{x̂_{k|k-1}}

EKF 的局限性是明显的：线性化只在一阶近似下有效。当非线性程度严重时（例如角度变化剧烈时），EKF 可能发散。这就引出了更先进的方法：

  • 无迹卡尔曼滤波（Unscented Kalman Filter, UKF）：用确定性采样的 sigma 点来传播非线性，不需要计算雅可比矩阵，对中等非线性系统更准确、更稳定。
  • 粒子滤波（Particle Filter）：用蒙特卡洛采样逼近任意分布，适用于强非线性和非高斯场景——但计算量也相应大得多。
选型建议：如果你的系统是线性的，用经典卡尔曼滤波（因为它是理论最优的）。如果非线性程度较弱（弱到一阶泰勒展开足够近似），用 EKF。如果非线性程度中等且状态维度不高（< 10），用 UKF。如果系统高度非线性或非高斯，用粒子滤波。永远不要在可以用简单方法的场景下用复杂方法——卡尔曼滤波已经够优雅了，不需要给它增加不必要的负担。

踩坑故事：阿波罗登月中 Q/R 矩阵的调参艺术

卡尔曼滤波历史上最著名的"踩坑"经历，正是它的成名战场——阿波罗登月计划。

阿波罗飞船的导航系统中，卡尔曼滤波被用来融合多个传感器的数据：惯性测量单元（IMU）、星体跟踪器、地面雷达的数据。系统设计者需要确定过程噪声协方差矩阵 Q 和观测噪声协方差矩阵 R 的值。Q 应该取多大？这反映了"对系统模型的信任程度"——如果你认为飞船的运动模型非常精确，Q 可以很小；反之，Q 应该大一些以容忍模型误差。

在早期的开发测试中，工程师们凭经验和直觉设定了 Q 和 R。结果在仿真中，卡尔曼滤波反复出现发散——估计误差不仅没有收敛，反而随着时间推移越来越大。他们花了大量时间排查代码、检查传感器模型，最后发现问题出在 Q 矩阵的设置上：Q 被设置得太小了，导致滤波器过度信任模型，当实际飞行轨迹与模型有微小偏差时，滤波器拒绝接受观测数据的"纠正"，最终与真实状态越走越远。

解决方案是什么呢？不是改变算法，而是改变参数。经过反复调试，工程师们将 Q 矩阵中的某些元素增大了一个数量级，滤波器立刻收敛了。后来人们发明了"自适应调 Q"技术——通过在线估计新息的统计特性来动态调整 Q 和 R，让滤波器自己"学会"合适的参数。

工程教训：卡尔曼滤波并不是一个"设置好就可以不用管"的算法。Q 和 R 矩阵的选择直接决定了滤波器的收敛性和稳定性。"Q 太小"会让滤波器变得"顽固"——拒绝接受观测证据，导致发散；"Q 太大"又会让滤波器变得"神经质"——对噪声过于敏感，估计结果剧烈抖动。在实际工程中，调参是卡尔曼滤波落地中最费时、最需要经验的部分。一个常用的经验法则是：先用离线数据做参数扫描（grid search），找到在测试集上表现最好的 Q/R 组合，然后在线上运行时加入自适应机制来微调。
