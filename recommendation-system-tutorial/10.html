第10章 DCN与xDeepFM

上一章的DeepFM看起来已经很完美了对吧？FM负责二阶交叉，DNN负责高阶交叉——分工明确，效果出色。但细心的你可能已经发现了一个问题：DNN真的在"负责任"地进行高阶交叉吗？

答案可能让你失望——DNN的高阶交叉是"隐式的"。我们把特征embedding拼在一起，丢进全连接层，经过几层非线性变换，你根本不知道模型学到了几阶交叉、交叉了哪些特征。就像一个黑盒子，你只知道它输入一堆特征、输出一个分数，但中间发生了什么——不清楚。

本章的两个主角——DCN（Deep & Cross Network）和xDeepFM——正是为了解决这个问题而生。

1. 为什么需要显式特征交叉

让我们把问题说得更具体一些。假设我们有三个特征：用户性别、物品类别、当前时间段。DeepFM中的DNN可能会学到"男性 × 电子产品 × 晚上"这个三阶交叉模式，但这个模式是隐含在DNN的参数空间里的，我们无法知道它具体学到了哪些交叉。

为什么这是个问题？有三个原因：

  • 不可控：你不知道模型在交叉什么特征，也就无法针对性地优化。如果某个业务上重要的交叉没有被模型捕获，你只能"加大DNN层数"来赌一把。
  • 浪费参数：DNN在做交叉时，所有特征都参与了所有层级的计算——即使某些特征之间完全没有交互的必要。这浪费了大量的参数在无意义的计算上。
  • 难以解释：当模型推荐出了问题（比如推荐了完全不相关的内容），你很难追溯到"是哪个特征交叉导致的"。

所以，学术界和工业界开始思考：能不能设计一种结构，让模型显式地、可控制地进行有限阶的特征交叉？

核心洞察：DeepFM中的FM只能做到二阶交叉，DNN虽然理论上可以做到任意阶交叉，但它是隐式的、不可控的。我们需要一个中间地带——一种能够显式地进行有限阶（比如三阶、四阶）特征交叉的模块。

2. DCN-V1/V2：交叉网络的演进

2017年，Google在Wide&Deep论文之后不久，又提出了Deep & Cross Network（DCN）。DCN的核心是一个Cross Network——一种专门用于显式特征交叉的网络结构。

Cross Network的每一层是这样计算的：

x_{l+1} = x_0 * (w^T x_l) + b + x_l

我们来拆解这个公式。x_0 是输入的embedding向量，x_l 是第 l 层的输出。每一层做的事情是：

  • 用 x_l 和一个可学习的权重向量 w 做内积，得到一个标量
  • 将这个标量与 x_0 相乘（广播乘法），得到"x_0 的某个方向的伸缩"
  • 加上偏置 b 和残差连接 x_l

这个结构的精妙之处在于每一层增加了一阶交叉的"度数"。第1层输出包含了 x_0 的一阶和二阶交叉（因为 x_0 * (w^T x_0) 涉及了二阶）。第2层在此基础上增加三阶，以此类推。所以一个有 L 层的Cross Network，最多可以捕获 L+1 阶的特征交叉。

来看一个简化的实现：

import torch
import torch.nn as nn

class CrossLayer(nn.Module):
    """单个Cross Network层"""
    def __init__(self, dim):
        super().__init__()
        self.w = nn.Parameter(torch.randn(dim))
        self.b = nn.Parameter(torch.zeros(dim))

    def forward(self, x_0, x_l):
        # x_{l+1} = x_0 * (w^T x_l) + b + x_l
        cross = (self.w @ x_l) * x_0 + self.b + x_l
        return cross

class CrossNetwork(nn.Module):
    """多层Cross Network"""
    def __init__(self, dim, num_layers=3):
        super().__init__()
        self.layers = nn.ModuleList([
            CrossLayer(dim) for _ in range(num_layers)
        ])

    def forward(self, x_0):
        x_l = x_0
        for layer in self.layers:
            x_l = layer(x_0, x_l)
        return x_l

DCN在2017年刚提出时使用的是一维向量 w（即每层只有一个标量做交叉）。这个版本后来被称为DCN-V1。2020年，Google发布了DCN-V2，核心改进是将 w 从向量升级为矩阵 W。

V2的公式变成了：

x_{l+1} = x_0 * (W x_l) + b + x_l

这个变化带来的影响是巨大的——V1的每层只有 O(d) 个参数（d 是embedding维度），而V2的每层有 O(d²) 个参数。参数量大了 d 倍，但表达能力也相应提升了。DCN-V2还引入了一个"低秩近似"的trick：将 W 分解为两个低秩矩阵的乘积，在保持大部分表达能力的同时控制参数量。

工业小贴士：DCN-V2的参数量巨大，直接使用可能对训练和推理造成压力。Google论文中的实验表明，使用 rank=16 或 rank=32 的低秩近似，可以在保留95%以上效果的同时，将参数量降低一个数量级。部署时建议从低秩版本开始，如果有余力再尝试完整版本。

3. xDeepFM的CIN网络

2018年，中科大和微软联合提出了xDeepFM（论文全称：xDeepFM: Combining Explicit and Implicit Feature Interactions for Recommender Systems）。xDeepFM引入了一个全新的结构——Compressed Interaction Network（CIN）。

CIN的设计思路和Cross Network完全不同。Cross Network是在"向量级别"做交叉——两个向量相乘得到一个标量。而CIN是在"元素级别"做交叉——embedding向量的每个维度分别进行交互，保留了更细粒度的信息。

CIN的核心操作可以理解为：

  • 输入是多组embedding（每个field对应一个embedding向量）
  • 每层通过"外积 + 卷积"的方式生成新的一组embedding
  • 每层生成的embedding经过sum pooling后拼接到输出中

更直观地说，CIN在做的是一种"特征维度的交互图"：第1层产生所有二阶的维度级交叉，第2层产生三阶的，以此类推。每一层的输出都直接拼入最终的特征向量，让后面的DNN和输出层可以访问所有阶的交叉信息。

xDeepFM的完整架构是：Embedding → [CIN + DNN] → 合并 → 输出。CIN提供显式的有限阶交叉，DNN提供隐式的高阶交叉，两者互补。

踩坑提醒：CIN的计算复杂度较高——每层的复杂度是 O(m²·d·H)（m是field数，d是embedding维度，H是隐藏单元数）。在field数量很大（>200）的场景下，CIN的训练速度会明显慢于DCN和DeepFM。我见过一个团队在300+field的数据上使用xDeepFM，训练时间比DeepFM多了5倍，而AUC提升只有0.1%。这种情况下，xDeepFM的投入产出比并不划算。

4. 模型选型实战指南

面对这么多模型——DeepFM、DCN-V2、xDeepFM、AutoInt等等——工程师和团队该如何选择？下面的表格可以帮助你做决策：

模型 交叉方式 最高交叉阶数 参数效率 训练速度 部署复杂度 最佳适用场景
DeepFM FM（二阶）+ DNN（隐式高阶） 二阶显式, 任意阶隐式 ★★★★★ ★★★★★ 低 通用基线, 大多数场景首选
DCN-V1 CrossNetwork (向量级) L+1阶显式 ★★★★ ★★★★★ 低 需要可控高阶交叉
DCN-V2 CrossNetwork (矩阵级) L+1阶显式 ★★（完整版）
★★★★（低秩版）		 ★★★ 中 数据量大、重视高阶交叉
xDeepFM CIN（元素级显式） L+1阶显式 ★★ ★★ 高 field数少、追求极致精度
AutoInt Multi-head Self-Attention 任意阶 ★★★ ★★★ 中 序列特征丰富、需要可解释性

实际建议：对大多数团队来说，DeepFM或DCN-V2（低秩版）是最保险的选择。xDeepFM的CIN虽然理论上更强大，但在实际工业场景中，它带来的边际收益通常不足以覆盖增加的计算和工程复杂度。我的原则是——如果你的DeepFM baseline经过充分调参后，AUC已经达到0.78，而工程师花两周时间上线xDeepFM可能只能提到0.785，那这两周时间不如花在特征工程或数据质量上。

回顾一下，我们从Wide&Deep走到了DeepFM，再到DCN和xDeepFM。你会发现一个清晰的趋势：模型越来越关注"如何让特征交叉变得更可控、更显式"。但到目前为止，我们讨论的所有模型都忽略了一个重要的问题——用户的兴趣是动态变化的。你的模型把用户今天的行为和上周的行为一视同仁地混合在一起，但显然，昨天点击的内容比一个月前点击的内容更能反映当前的兴趣。

下一章，我们将进入深度兴趣网络（DIN）的世界——它用注意力机制来解决"用户的静态embedding无法刻画动态兴趣"这个核心问题。
