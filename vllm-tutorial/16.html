第16章 Kubernetes 部署

单机部署够简单，但生产级大规模的推理服务还是要上 Kubernetes。vLLM 官方提供了 Production Stack——一套 Helm Chart，几分钟就能搭起一个完整的推理集群。

16.1 Production Stack 概述

vLLM Production Stack 是官方维护的 K8s 原生部署方案：

Production Stack 核心组件:
├── vLLM Engine Pods       ← 模型推理实例（每个 Pod 一个 vLLM 进程）
├── Router (可选)          ← 前缀感知路由，同 system prompt 去同一 Pod
├── LMCache               ← 跨 Pod 共享 KV Cache（减少冷启动）
├── Prometheus + Grafana  ← 可观测性（指标采集 + 仪表板）
└── Helm Chart            ← 一键部署/<升级/卸载

16.2 安装与配置

# 1. 添加 Helm 仓库
helm repo add vllm https://vllm-project.github.io/production-stack
helm repo update

# 2. 配置文件 values.yaml
cat > values.yaml << 'EOF'
servingEngineSpec:
  modelSpec:
    - name: "llama-8b"
      repository: "vllm/vllm-openai"
      tag: "latest"
      modelURL: "meta-llama/Llama-3-8B"
      replicaCount: 2          # 2 个实例
      requestGPU: 1
      args:
        - --enable-prefix-caching
        - --kv-cache-dtype fp8
        - --max-model-len 8192

routerSpec:
  enabled: true                # 开启前缀感知路由
  replicaCount: 1

metricsSpec:
  enabled: true                # 开启 Prometheus 指标
EOF

# 3. 部署
helm install vllm vllm/vllm-stack -f values.yaml

16.3 生产栈关键能力

能力作用启用方式
多模型管理同一集群服务不同模型（路由匹配）在 modelSpec 中配置多个模型
前缀感知路由相同前缀的请求→同一个 Pod，最大化 prefix cache 命中routerSpec.enabled: true
LMCache 集成KV cache 离载到 CPU 内存/SSD，减少冷启动配置 LMCache 后端
自动扩缩基于 GPU 用量或 QPS 的 HPA配置 autoscaling
滚动更新模型版本升级不中断服务K8s 原生 rolling update

前缀感知路由为什么重要：假设你有 3 个 Pod，随机路由请求。每个 Pod 的 prefix cache 都只有 1/3 的命中率。加了前缀感知路由后，同样 system prompt 的请求全部到同一个 Pod，缓存命中率从 33% → 90%+。这是 "用路由层换缓存效率" 的经典案例。

16.4 GPU 资源管理

# 每个 vLLM Pod 申请 GPU 资源
resources:
  limits:
    nvidia.com/gpu: 1  # 1 张 GPU

# 多 GPU 张量并行
args:
  - --tensor-parallel-size 2
resources:
  limits:
    nvidia.com/gpu: 2  # 要配 2 张

# 注意：TP 要求 GPU 在同一节点 + NVLINK，用 nodeSelector 绑定
nodeSelector:
  nvidia.com/gpu.product: NVIDIA-A100-SXM4-80GB

16.5 模型预热

Pod 冷启动时加载模型可能耗时 30s-2min（取决于模型大小和网络速度）。可以用 initContainer 预下载模型：

initContainers:
  - name: model-downloader
    image: huggingface/transformers:latest
    command:
      - huggingface-cli
      - download
      - meta-llama/Llama-3-8B
      - --local-dir /models
    volumeMounts:
      - name: model-storage
        mountPath: /models
    env:
      - name: HF_TOKEN
        valueFrom:
          secretKeyRef:
            name: hf-secret
            key: token

成本优化：如果模型常用（不是冷门模型），把模型下载到持久卷（PVC/云盘），多个 Pod 共享，避免每次重启重新下载。模型文件通常几十 GB，下载一次重用省时省流量。
