超参数在"缩放"——当模型从百万参数到千亿参数

大家好，我是你们的讲师。如果你曾经拿着一个在ResNet-50上工作得很好的超参数配置，直接套到ViT-Large上发现效果一塌糊涂——或者更夸张一点，把BERT-base的配置迁移到GPT-3上结果loss直接飞到天上——那你一定理解今天这章要讲的核心问题：超参数如何随模型规模缩放。

过去六年的深度学习史，几乎就是一张模型规模指数级扩大的图表。从2017年Transformer的6500万参数，到GPT-3的1750亿，再到2025年各种万亿参数MoE模型——模型规模增长了上万倍。但调参的方法并没有跟上这个增长速度。把一个小模型上调好的超参数直接搬到大模型上用是不行的——那就像把自行车的轮胎装到卡车上，物理规律完全不一样。

好在学术界已经在这方面取得了突破性进展。从OpenAI的Scaling Law到Microsoft的μP参数化，从DeepSeek的固定超参策略到2026年最新的深度缩放定律——我们正在逐步建立起一套"超参数缩放理论"。今天这章，我们要把这些前沿成果讲清楚，让你真正理解：当模型从小变大时，超参数到底应该怎么变。

Scaling Law的惊人发现

2019年，OpenAI发表了一篇里程碑式的论文——"Scaling Laws for Neural Language Models"。这篇论文做了一个简单而深刻的实验：固定网络架构，只改变模型参数量N、数据量D和计算量C，观察loss与这三者的关系。结论非常惊人：在给定计算预算C的条件下，最优模型大小和数据大小遵循幂律关系。

但对我们调参工程师来说，更有价值的发现是另一个结论：给定计算量C，大多数超参数的最优值是固定的——只有学习率（LR）和batch size需要随计算量变化。这意味着什么？你用80亿参数模型调好的dropout比例、weight decay系数、Adam的β值，可以直接迁移到下一代千亿参数模型上。不需要重新在超参数空间的每个维度上都做grid search——只需要重新调节LR和batch size就可以了。

2024年，DeepSeek团队在训练DeepSeek-V2时进一步量化了这个关系，提出了非常实用的幂律公式：

最优学习率 η_opt = 0.3118 × C^(-0.1250)
最优批大小 B_opt = 0.2920 × C^(0.3271)

这里的C是总计算量（以FLOPs计）。这两个公式揭示了一个直观的趋势：计算量越大，最优学习率越小，最优batch size越大。为什么？当计算量增大时，模型通常也更大，参数空间的曲率变得更复杂，需要更小的步长才能稳定下降。同时，更大的模型意味着更冗余的梯度估计，可以用更大的batch size来摊平噪声。

实用提示：如果你有一个在1B模型上调好的LR=3e-4，想迁移到10B模型上，根据幂律公式，新LR ≈ 3e-4 × 10^(-0.125) ≈ 3e-4 × 0.75 ≈ 2.25e-4。也就是说，大致要降25%。这不是精确值，但给出了正确的调整方向。

Scaling Law的伟大之处在于它把"感觉"变成了"公式"。以前我们凭经验说"大模型需要小学习率"—现在DeepSeek给了你一个具体幂次：-0.125。这个指数在不同架构上略有波动，但基本都在-0.1到-0.15之间，说明这是一个相当普适的规律。与之对应，batch size的指数+0.3271告诉我们在计算量翻十倍时batch size大约要翻2.1倍——远没有LR的调整幅度大，但方向是明确的。

μP（最大更新参数化）原理

Scaling Law说的是超参数随计算量的宏观缩放，但还有一个更底层的问题：当模型宽度（hidden size）趋于无穷时，训练还能正常进行吗？这个问题听起来很理论，但它有非常实际的后果。

2018年有一篇经典论文指出，在标准参数化下，当模型的宽度d→∞时，模型会进入一个叫lazy regime的状态——网络的权重几乎没有实质性变化，模型只是在初始化附近做线性逼近，根本没有学到深层特征。这就解释了为什么小模型上调好的参数直接迁移到大模型上经常失效——因为大模型进入了不同的训练动力学区域。

2021-2022年，Microsoft Research的Greg Yang团队提出了μP（Maximal Update Parameterization，最大更新参数化），从根本上解决了这个问题。μP的核心思想是：让每一层的更新量在宽度变化时保持稳定。

具体来说，μP做了三件事：

  • 初始化标准差按层宽缩放：输入嵌入层和输出层的std=1/sqrt(d)，隐藏层的std=1/sqrt(d)，其中d是当前层的宽度。确保前向传播的激活值幅度与宽度无关。
  • 学习率按层宽缩放：输出层的LR需要乘以一个因子（通常是d_model/d_out）。因为输出层的梯度幅度与宽度有关，需要单独放大以保证更新量不随宽度萎缩。
  • Attention中的softmax前加1/sqrt(d)缩放：防止点积结果随维度增长而爆炸，保持softmax输出的"锐度"与宽度无关。
理解μP为什么work：在标准参数化下，宽度变大→每层的输出幅度变大→梯度更新时参数的相对变化量变小→模型接近初始化→lazy regime。μP通过按宽度缩放初始化（保持激活值幅度恒定）和按宽度缩放LR（保持参数更新的相对大小恒定），保证了不同宽度下模型始终处于"有特征学习"的状态。简单说：μP让"宽度"这个维度从模型的行为中"消去"了。

μP最令人兴奋的性质是零样本超参数迁移：你可以在一个小代理模型（比如宽度=256）上把LR、初始化标准差、Adam betas等超参数调好，然后把这些超参数直接用到宽度=4096甚至更大的模型上——效果几乎不变。这在实践中节省了天文数字的调参成本。例如，如果你要在宽度=8192的模型上训练，而调一次全尺寸实验需要花费100万美元——你可以在宽度=256的代理模型上只花几千美元调好参数，然后零成本迁移。

深度缩放中的-3/2幂律（2026最新）

如果说宽度缩放的μP问题已经基本解决，那深度缩放（depth scaling）是2025-2026年才取得突破的新前沿。2026年初，来自多个实验室的独立研究共同发现了一个惊人的规律：最优学习率随有效深度L的缩放遵循-3/2幂律。

η* ∝ L^(-3/2)

这个规律在CNN、ResNet、ViT等多种架构上都得到了验证。它意味着：当你的网络深度加倍时，最优学习率大约要降到原来的2^(-3/2) ≈ 35%。也就是说，从12层ResNet到18层，LR要从1e-3降到约3.5e-4——这是一个非常大的调整幅度，远远超过Scaling Law中计算量缩放带来的LR变化。

为什么深度越深，LR需要越小？直观理解是：每个层只贡献梯度路径的一部分。深层网络的梯度信号要穿过更多的非线性层，传递到浅层时已经被多次"衰减"——或者在BatchNorm等机制下被放大。层数越多，这种传递的变异性越大，因此需要更小的步长来维持整体的稳定。数学上可以证明，在理想化的深层网络中，梯度的方差随深度呈L^(3/2)增长，因此学习率必须按L^(-3/2)缩小才能保持更新量的方差恒定。

重要的实战区别：深度缩放和宽度缩放的规律完全不同。宽度缩放用μP可以做到零样本迁移，但深度缩放目前还没有这么优雅的解决方案。-3/2幂律是一个经验规律，可以作为调参的起点，但不像μP那样有严格的数学保证。当你同时增加宽度和深度时，建议先处理宽度迁移（用μP），再根据深度缩放规律微调LR。

DeepSeek的固定超参策略

看到这里你可能在想：这些理论很好，但在实际训练千亿参数模型时，到底哪些参数需要调，哪些可以固定？DeepSeek团队给出了一个极其实用的答案。

在DeepSeek-V2和后续模型的训练中，他们采用了如下策略：

  • 固定的超参数：β₁=0.9, β₂=0.95, weight_decay=0.1, σ_init=0.06, grad_clip=1.0。这些参数在多次大规模训练中始终保持不变，因为实验证明它们对模型规模不敏感。
  • 随计算量变化的超参数（仅两个）：学习率η和batch size B，按照前面给出的幂律公式自动调节，不需要额外搜索。
  • 学习率调度：warmup 2000步，然后使用multistep scheduler——在消耗80%和90%的training tokens时各将LR降低一次（通常降为原来的10-20%）。这种粗粒度的调度策略简单有效。

这个策略的核心思想是"固定大多数，只调最关键的两个"。在参数量从7B到数百B的训练过程中，Adam优化器的β值、权重衰减系数、梯度裁剪阈值这些参数表现出极强的鲁棒性——一旦在某个规模上选好，就不需要随着模型变大而改变。这不仅节省了调参成本，更重要的是减少了训练过程中需要跟踪的变量数目，使得每一次规模扩张都更加可控。

从中学到什么：DeepSeek的做法告诉我们，好的调参策略不是"什么都要调"，而是知道"什么可以不调"。如果你正在训练一个中等规模的模型，也建议你把大部分超参数固定到一个合理值（如β₁=0.9, β₂=0.999, wd=0.01作为默认值），只专注于优化LR和batch size——这两者通常占据了90%以上的效果提升空间。

下面我们来对比一下μP和标准参数化的关键区别：

对比维度 标准参数化（SP） μP（最大更新参数化）
初始化标准差 各层独立设定，常用固定值如0.02 按层宽缩放：1/sqrt(d)
学习率设置 全局统一LR 逐层按宽度缩放LR
宽度→∞时行为 进入lazy regime，无特征学习 持续保持特征学习
超参数迁移性 宽度变化需重新调参 零样本迁移到任意宽度
理论保证 无 有严格数学证明（Tensor Program）
实现复杂度 简单，PyTorch默认行为 中等，需分参数组设置不同LR
实际应用 绝大多数开源项目 DeepSeek、GPT-4等前沿大模型

下面是一个μP初始化和LR缩放的简单代码示例，帮助你直观理解实现方式：

import torch
import math

def mup_param_groups(model, d_model, lr_base):
    """
    为模型的不同参数组设置μP缩放后的学习率
    - 嵌入/隐藏层: LR保持不变（宽度无关）
    - 输出层（lm_head）: LR * d_model / d_out
    """
    param_groups = []
    for name, param in model.named_parameters():
        if 'lm_head' in name or 'output' in name:
            # 输出层: LR放大，初始化std缩放
            lr = lr_base * d_model / param.shape[-1]
            torch.nn.init.normal_(param, std=1.0 / math.sqrt(d_model))
        elif 'embed' in name:
            # 嵌入层: 初始化std缩放，LR不变
            torch.nn.init.normal_(param, std=1.0 / math.sqrt(d_model))
            lr = lr_base
        else:
            # 隐藏层: 均按宽度缩放初始化
            if param.dim() > 1:
                fan_in = param.shape[0]
                torch.nn.init.normal_(param, std=1.0 / math.sqrt(fan_in))
            lr = lr_base
        param_groups.append({'params': param, 'lr': lr})
    return param_groups

# 在宽度=768的模型上调参
model = YourTransformer(d_model=768, d_vocab=32000)
groups = mup_param_groups(model, d_model=768, lr_base=3e-4)
optimizer = torch.optim.AdamW(groups)

# 零样本迁移到宽度=4096的模型
model_wide = YourTransformer(d_model=4096, d_vocab=32000)
groups_wide = mup_param_groups(model_wide, d_model=4096, lr_base=3e-4)
optimizer_wide = torch.optim.AdamW(groups_wide)
# 注意：lr_base保持3e-4不变，这就是μP的零样本迁移

在上面的代码中，关键点在于当模型宽度从768扩展到4096时，lr_base保持3e-4不变——这就是μP的零样本迁移效果。如果没有μP，扩展到4096后必须从头重新扫描LR，那可能需要数十次实验才能找到合适值。

本章小结：调参的"缩放观"

从Scaling Law的幂律公式，到μP的宽度缩放理论，再到深度缩放的-3/2幂律，最后到DeepSeek的固定超参策略——这些东西串起来，你会发现一个统一的画面：超参数不是孤立数值，而是与模型规模绑定的函数。学习率不是3e-4，而是η(C) = 0.3118·C^(-0.1250)。batch size不是256，而是B(C) = 0.2920·C^(0.3271)。当理解了这个，调参就不再是"试出魔法数字"，而是"理解数据背后的缩放规律"。
