回到1991：一篇论文的诞生

大家好，我是你们的讲师。上一章我们走马观花地浏览了MoE的35年发展史，提到了1991年这个原点。今天，我们要做一件在AI教程里很少有人做但非常值得做的事——翻开一篇1991年的原始论文，逐字逐句吃透它。

这篇论文叫 Adaptive Mixture of Local Experts，作者是 Robert Jacobs、Michael Jordan 和 Geoffrey Hinton，发表在 Neural Computation 期刊上。你没看错，就是那位后来的"深度学习之父"Geoffrey Hinton。三位作者在当时想解决一个在今天看来依然很前沿的问题：如何让多个神经网络分工协作，各自负责输入空间的一个局部区域？

让我讲个有趣的历史背景。1991年正是神经网络第二次寒冬的前夜。Rumelhart和McClelland的PDP（并行分布式处理）浪潮已经过去几年，BP算法虽然被重新发现，但训练效果远没有达到人们的期望。就在这种氛围下，Jacobs、Jordan和Hinton提出了一个反直觉的想法：与其训练一个巨大的、试图解决所有问题的网络，为什么不训练一群"小专家"，再让一个门控网络来决定谁的话值得听？这个想法在当年太超前了——以至于用当时的硬件几乎无法在有趣的问题上验证它。但这恰恰是MoE故事最迷人的部分：一个在1991年就提出的架构，要到2017年才真正迎来它的时代。

论文精读：门控网络与专家网络

让我们直接进入论文的技术内核。Adaptive Mixture of Local Experts 的核心架构包含两个核心组件，今天读来依然让人拍案叫绝：

  • 专家网络（Expert Networks）：一组简单的神经网络（1991年用的是单层线性网络或浅层MLP），每个专家负责输入空间的一个局部区域。每个专家都有自己的权重，独立处理输入。
  • 门控网络（Gating Network）：一个竞争性网络，输出对各个专家的"责任权重"（responsibilities），决定每个输入样本应该分配给哪些专家。

门控网络的核心是一个 softmax 函数，它将每个专家的得分转化成一个概率分布。如果你熟悉今天的分类神经网络，这看起来平平无奇——但在1991年，softmax 作为"门控机制"出现在神经网络中，是一件非常有创意的事。

论文中的门控输出可以用如下形式表达：

# 1991年Jacobs论文的MoE前向计算（Python化伪码）
import numpy as np

def moe_forward_jacobs(x, experts, gate):
    # x: 输入样本 [d,]
    # experts: 列表，每个元素是一个专家网络函数
    # gate: 门控网络，输入x输出E维logits

    # Step 1: 门控网络计算责任权重
    g_logits = gate(x)                        # [E,] 未归一化得分
    g = np.exp(g_logits) / np.sum(np.exp(g_logits))  # [E,] softmax

    # Step 2: 每个专家独立计算输出
    expert_outputs = [expert(x) for expert in experts]  # 列表长度E

    # Step 3: 加权组合（稠密混合！）
    output = sum(g[i] * expert_outputs[i] for i in range(len(experts)))

    # 注意：1991年的版本是"稠密"的，所有专家都被激活
    # 只不过权重g可能接近0——这是最早的soft MoE思想
    return output

注意看上面代码的第3步：所有专家的输出都参与加权求和，只是权重由门控网络决定。这与今天稀疏MoE的"只激活top-k个专家"完全不同。1991年的MoE是稠密混合，2017年之后的MoE是稀疏激活。这个区别我们后面会详细展开。

论文最精妙的设计在于训练方法：专家网络和门控网络不是分开训练的，而是通过端到端的梯度下降联合训练。门控网络通过增大"正确"专家的权重、减小"错误"专家的权重，来学习如何分配责任。而专家网络则通过加权后的误差信号来更新自己的参数——权重高的专家收到更强的训练信号，权重低的专家收到的信号很弱。这种机制自然导致了竞争性学习：每个专家都争取在门控网络那里获得更高的权重分配。

💡 理解关键： 1991年论文的联合训练机制是MoE的"灵魂"。门控网络充当了"任务调度员"的角色，它不加显式约束，让专家在竞争中自然分化。今天所有的MoE训练方法——无论是添加负载均衡损失还是引入辅助损失——本质上都是在"竞争分化"这个原始设计上做修正，防止竞争过度导致某些专家"饿死"。

竞争学习：谁赢了谁说话

1991年论文中最有趣的概念是 竞争学习（Competitive Learning）。三位作者敏锐地观察到：如果不对门控网络施加任何约束，多专家模型会自然地走向一种"赢家通吃"的局面——对于每个输入，只有一个专家获得绝大部分权重。这不是设计缺陷，而是有意为之的特征。

让我用一个通俗的比喻来解释：想象一个足球队，有10个球员（专家），教练（门控网络）根据对手的情况决定派谁上场。如果教练发现7号球员在防守任意球方面特别强，以后遇到任意球就只派他上。久而久之，其他球员的防守任意球能力就荒废了——反正教练也不会派他们在这个场景下上场。这听起来浪费，但从整体效率来看，每个球员专攻一个方向，团队的总体能力反而更强。

竞争学习的目标正是让专家"分化"——每个专家专注于输入空间的一个局部区域。论文中通过实验展示了这个现象：在一个简单的分类任务中，训练后每个专家确实只对特定区域的数据产生高响应，而门控网络学会了将不同的输入模式分配给不同的专家。

下表对比了1991年MoE与当时主流方法的异同：

方法 架构 训练方式 参数效率 计算开销
单一大网络 单一MLP 端到端BP 低（所有参数对所有样本更新） 低
独立集成（Ensemble） 多个独立网络 各自独立训练后投票 低（参数不共享、不协同） 高（全部激活）
1991 MoE（稠密混合） 多个专家 + 门控 端到端联合训练 高（专家自然分工） 中（全部激活但加权）
模块化网络（Modular） 子模块 + 仲裁器 分阶段训练 中 中

表中可以看到，1991年的稠密MoE在参数效率上已经比单一网络和集成方法有了明显优势。但是，它的"稠密"特性——所有专家都必须参与每次前向计算——成了后来规模化道路上的关键瓶颈。

🔑 关键洞察： 1991年论文第一个公式化的MoE架构引出了一个核心思想——"分而治之"（divide and conquer）。这个思想在AI领域并不新鲜，但Jacobs等人用神经网络的方式实现了它：让子网络（专家）通过竞争自动发现如何划分问题空间，远比手工设计规则更灵活。今天来看，这几乎是"自动化的模块化"的雏形。

说一个我自己的经验。第一次读这篇1991年论文时，我犯了一个大错误：我把它当成了过时的"历史文献"，草草翻过就跳去读2017年的Shazeer论文了。结果读2017年论文时，我对"门控网络为什么需要加噪声"、"专家为什么需要容量上限"这些问题完全没有直觉。后来回头仔细研读了1991年论文，才恍然大悟：2017年论文解决的每一个问题，都是1991年框架在规模化过程中暴露出来的短板。不读1991年论文，相当于看一部电影的第三部却没看过第一部——能看懂情节，但感受不到铺垫的精妙。

稠密MoE vs 稀疏MoE：从全激活到选择性激活

我们现在有了一个关键的认识：1991年的MoE是"稠密"的。每次前向时，所有专家都计算，门控网络只负责加权。这在只有几个专家时完全没问题。但想象一下，如果你有256个专家，每个专家是一个巨大的FFN（比如5000维隐藏层），每次前向都激活所有专家——计算量会爆炸。

这就是"从稠密到稀疏"的必然演进。稀疏MoE的核心变化只有一个：对于每个输入，只激活所有专家中的一小部分。在1991年的框架里做这个改动，只需要把"所有专家的加权求和"改成"top-k个专家的加权求和"：

# 稀疏MoE（2017版）vs 稠密MoE（1991版）
def sparse_moe_forward(x, experts, gate, k=2):
    # 与1991年版唯一的区别：稀疏化
    g_logits = gate(x)                        # [E,] 得分
    top_k_vals, top_k_idx = np.sort(g_logits)[-k:], np.argsort(g_logits)[-k:]

    # 只选中top-k个专家计算
    output = 0
    for i, expert_idx in enumerate(top_k_idx):
        # 对top-k权重再做一次softmax
        w = np.exp(g_logits[expert_idx]) / np.sum(np.exp(top_k_vals))
        output += w * experts[expert_idx](x)

    return output

这个改动看起来小，但意义深远：

  • 计算量从O(E)降到O(k)，其中E是专家总数，k是选择的专家数量（通常k=1或2，E可以是8到256甚至更多）。
  • 参数总量与计算量解耦：你可以拥有256个专家（海量参数），但每个token只计算2个专家（少量计算）。
  • 门控从"软加权"变成"硬选择"：门控网络不再给所有专家分配权重，而是需要做出"是/否"的二值选择。这让门控的训练变得更加困难——梯度无法直接通过离散选择传播。

从稠密到稀疏，MoE完成了从一个"优雅的统计模型"到"实用的工程架构"的蜕变。但正如我们已经看到的，这个转变花了整整26年——从1991年到2017年。在这26年里，MoE并非完全沉寂，它被用于一些统计学习和信号处理场景，但始终没有成为深度学习的主流。

⚠️ 常见误解： 有人以为"稀疏MoE"是2017年才提出的全新概念。事实上，稀疏化的思想在1991年论文中已有萌芽——Jacobs等人提到可以使用"硬门控"（hard gating）让每个输入只分配给一个专家。但限于当时的优化理论，他们没有深入探索这条路。2017年Shazeer团队的贡献不是"发明稀疏MoE"这个概念，而是在深度学习的框架下解决了稀疏门控的梯度传播问题，让它可以在大规模Transformer模型上工作。

这章的结尾，我想分享一个关于"经典论文阅读"的体会。很多做AI的朋友有一个倾向：只读近两三年的论文，觉得"老论文没有技术价值"。我认为这是一个很大的损失。1991年的这篇论文，整个篇幅只有12页，数学推导简洁优雅，直觉清晰——读完之后你会在脑海里形成一个关于MoE的"心智模型"，再读后续任何一篇MoE论文时，都能快速定位它"在解决什么问题"、"在改进哪个环节"。站在1991年的地基上，2017年的创新看起来不再神秘，2024年的创新也只是在同一个方向上的延续和深化。
