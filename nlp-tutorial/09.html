第九章：Attention机制

大家好！上一章我们讲Seq2Seq时发现了一个致命问题——编码器要把整个输入句子压缩成一个固定长度的向量。30个词的句子和300个词的句子都压缩成同一个维度，这显然不合理。Attention机制正是为解决这个"信息瓶颈"而生的，而且它的影响远超预期——可以说，2015年Bahdanau那篇论文是整个Transformer时代的伏笔。

我给学生讲Attention时最喜欢用这个类比：你在翻译一个长句时不会把整句记在脑子里再翻译，而是会不时回看原文的相关部分。Attention就是给模型这个"回看"的能力。这个直觉如此自然，以至于你一旦理解了，就会觉得没有Attention的Seq2Seq简直不可思议。

9.1 从信息瓶颈到Attention

Seq2Seq的基本架构中，编码器LSTM处理完整个输入序列后，只把最后一个隐藏状态传给解码器。这意味着：

# Seq2Seq 信息瓶颈
# 编码器：h_1, h_2, ..., h_T → 最终只有 h_T 被使用
# 解码器：只看 h_T 生成翻译
#
# 问题：无论输入多长，全部信息必须塞进一个 h_T 里
# 实验数据：句子长度超过20词，BLEU分数急剧下降

Attention的解决方案是：不只看最后一个隐藏状态，而是让解码器在每一步都能"回顾"编码器的所有隐藏状态，并学会自己决定关注哪些位置。

对比维度纯Seq2SeqSeq2Seq + Attention
解码器输入只有最后一个h_T所有h_1...h_T的加权和
信息量固定维度，随输入增长而稀释自适应，可以聚焦关键位置
长句效果差（BLEU随长度急剧下降）好（几乎不受长度影响）
可解释性无（黑盒）对齐权重可以可视化

9.2 Bahdanau Attention（Additive Attention）

Bahdanau et al. (2015) 第一次将Attention引入NMT（神经机器翻译）。它的计算流程：

# Bahdanau Attention 计算步骤

# Step 1: 计算对齐分数（score）
# e_{t,i} = v^T · tanh(W_a · [s_{t-1}; h_i])
# s_{t-1}：解码器t-1时刻的隐藏状态
# h_i：编码器第i时刻的隐藏状态
# W_a, v：可学习的参数

# Step 2: 归一化得到注意力权重
# α_{t,i} = softmax(e_{t,:})_i
# Σ_i α_{t,i} = 1，权重和为1

# Step 3: 加权求和得到上下文向量
# c_t = Σ_i α_{t,i} · h_i

# Step 4: 用上下文向量辅助解码
# s_t = LSTM(s_{t-1}, [y_{t-1}; c_t])

关键洞察：Attention不是让模型"直接看"输入词的向量，而是看编码器处理这些词时的隐藏状态。隐藏状态已经融入了上下文信息，所以Attention本质是在"有上下文理解的源语言表示"上做加权。

9.3 Luong Attention与乘性变体

Bahdanau提出Attention后不久，Luong et al. (2015) 提出了几个变体，最流行的是乘性Attention（dot-product）：

# 三种对齐分数计算方式
# 1. Additive (Bahdanau): e = v^T·tanh(W·[s; h])
# 2. Dot-product (Luong): e = s^T · h
# 3. General (Luong):     e = s^T · W · h

# Dot-product最简单高效——没有可学习参数
# General多一个W矩阵多了表达力
# Additive在维度不同时更灵活

# PyTorch 实现dot-product attention
def dot_attention(decoder_hidden, encoder_outputs):
    # decoder_hidden: (batch, hidden)
    # encoder_outputs: (batch, src_len, hidden)
    scores = torch.bmm(encoder_outputs, decoder_hidden.unsqueeze(-1)).squeeze(-1)
    attn_weights = torch.softmax(scores, dim=1)
    context = torch.bmm(attn_weights.unsqueeze(1), encoder_outputs).squeeze(1)
    return context, attn_weights

9.4 对齐权重的可视化魔法

Attention最迷人的特性之一是可解释性——对齐权重α矩阵天然展示了源语言和目标语言之间的词对应关系。这在机器翻译中尤其直观：

把α矩阵画成热力图，你会看到对角线式的亮条纹——说明模型学会了源语言和目标语言的词序对应。"the cat"对齐到"猫"，"I love you"对齐到"我爱你"。更妙的是，在翻译德语→英语时，动词在从句末尾的对齐模式也清晰可见。这种不需要显式监督的无监督对齐是Attention最令人惊叹的性质。

9.5 Attention的通用化理解

跳出Seq2Seq的语境，Attention可以用一个更通用的框架来理解：

# Attention的通用三要素
# Query (Q): "我想查什么？"     ——解码器的当前状态
# Key   (K): "每个位置的标签"   ——编码器的隐藏状态
# Value (V): "每个位置的内容"   ——编码器的隐藏状态（可以和K相同）

# Attention(Q, K, V) = softmax(QK^T / √d_k) × V

# 本质：用Query和Key的相似度作为权重，对Value做加权平均
# 这是一个"软寻址"操作——从"内存"中按相关性取信息

这个Q/K/V框架就是下一章Transformer的核心。Attention不再只是Seq2Seq的一个"插件"，而是成为了模型的基本构建块——"Attention is All You Need"。下一章，我们将见证NLP史上最大的一次范式变革。
