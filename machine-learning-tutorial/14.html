14.1 特征工程：ML中最被低估的技能

如果你去问Kaggle竞赛的大师"你成功的秘诀是什么"，十个人里有八个会回答"特征工程"。算法是现成的、调参有AutoML，但创造出好的特征——这是纯人脑的创造性工作。一个巧妙构造的特征有时能带来比换模型大得多的提升。

我曾在一个电商点击率预测项目中卡在AUC 0.71，试遍了各种模型都无法提升。后来一个同事提了个想法：把"用户最近3天点击次数"除以"用户最近30天点击次数"，创造了一个"近期活跃度"特征。就这一个特征，AUC直接跳到0.76。这就是特征工程的魔力。

14.2 PCA：最经典的降维方法

PCA（主成分分析）是数据降维的标配工具。核心思想：找到数据方差最大的方向（第一主成分），然后找与它正交的方差第二大方向（第二主成分），以此类推。

from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# PCA前必须标准化！
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 先看需要多少成分才能保留足够方差
pca_full = PCA()
pca_full.fit(X_scaled)

import matplotlib.pyplot as plt
explained_var = np.cumsum(pca_full.explained_variance_ratio_)
plt.plot(range(1, len(explained_var)+1), explained_var, 'o-')
plt.axhline(y=0.95, color='r', linestyle='--')
plt.xlabel('主成分数量')
plt.ylabel('累计解释方差比例')
plt.show()

# 选择保留95%方差的主成分数
n_components = np.argmax(explained_var >= 0.95) + 1
print(f"保留95%方差需要{n_components}个主成分")

# 降维
pca = PCA(n_components=n_components)
X_pca = pca.fit_transform(X_scaled)
⚠️ PCA的三个前提：(1) 数据必须标准化——PCA对方差敏感，大尺度特征会主导主成分方向。(2) PCA假设线性关系——如果数据在高维空间中是非线性流形（如瑞士卷），PCA会失效，需要用t-SNE或UMAP。(3) 主成分不可解释——PCA产生的新特征是原始特征的线性组合，通常难以赋予业务含义。如果需要可解释的降维，用特征选择而非PCA。

14.3 t-SNE：可视化神器

PCA找的是"全局方差最大"的方向，t-SNE找的是"保持邻居关系"的低维嵌入。结果：t-SNE在可视化上远优于PCA，但它不能用于特征转换（只能降维，不能transform新数据），且参数（perplexity）对结果影响很大。

特性PCAt-SNE
目的降维、去噪、特征压缩可视化、探索性分析
可transform新数据是否（需重新运行）
保持全局结构好差（簇间距无意义）
保持局部结构差好（邻居关系保留好）
计算速度快慢（O(n²)）
💡 t-SNE调参口诀：perplexity（困惑度）大致等于你觉得每个点应该有多少个"近邻"。典型范围5-50。大的perplexity产生更全局的结构，小的关注局部细节。如果你不确定，试几个值（5, 15, 30, 50），选视觉上最有信息量的那个。相同perplexity下多次运行结果不同是正常的（随机初始化）。

14.4 特征选择的三大流派

降维（PCA）是创造新特征，特征选择是从原有特征中挑出最重要的。三种方法各有优劣：

过滤法（Filter）

用统计指标（相关性、互信息、卡方检验）独立评估每个特征，选分数最高的。速度快，但忽略了特征之间的交互效应。适合初始粗筛。

包裹法（Wrapper）

用模型的效果来评估特征子集的质量。如递归特征消除（RFE）：全特征训练 → 删除最不重要的特征 → 重复。效果好但计算量大。RFECV可以自动选特征数。

嵌入法（Embedded）

特征选择作为模型训练的一部分。Lasso（L1正则化）让不重要的特征系数变为0；树模型提供feature_importances_。实际中最常用的方法。

from sklearn.feature_selection import SelectKBest, mutual_info_classif, RFECV
from sklearn.ensemble import RandomForestClassifier
from sklearn.linear_model import LassoCV

# 方法1：Filter - 互信息选前20个特征
selector = SelectKBest(mutual_info_classif, k=20)
X_filtered = selector.fit_transform(X, y)

# 方法2：Embedded - Lasso自动特征选择
lasso = LassoCV(cv=5)
lasso.fit(X, y)
selected = np.where(lasso.coef_ != 0)[0]
print(f"Lasso选出了{len(selected)}个特征")

# 方法3：Embedded - 树模型特征重要性
rf = RandomForestClassifier(n_estimators=100)
rf.fit(X, y)
importances = rf.feature_importances_
# 选重要度前80%的特征
threshold = np.percentile(importances, 20)
selected_rf = np.where(importances > threshold)[0]

14.5 目标编码与特征交叉

这可能是工业界最有用的两个特征工程技巧：

  • 目标编码（Target Encoding）：用类别变量的目标均值替代原始类别。比如"城市" → "该城市的平均房价"。效果极好，但容易过拟合——一定要用交叉验证方式做，不能直接在整个训练集上算均值
  • 特征交叉（Feature Crossing）：两个或多个特征的组合。比如"年龄"和"性别"交叉成"年轻男性"、"中年女性"等，让线性模型能捕捉非线性交互
🔑 核心原则：特征工程的精髓不是堆砌技巧，而是理解你的数据和业务。好的特征来自对问题域的深刻理解——你知道什么信息对预测有用，然后想办法把它量化为特征。这是机器学习中最需要"人"的一环。
