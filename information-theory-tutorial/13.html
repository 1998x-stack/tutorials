大家好，我是你们的讲师。

C = max I(X;Y) —— 这个看似简单的公式背后，藏着一个曾经困扰信息论学者几十年的计算难题。

问题出在哪里？I(X;Y) 是关于输入分布 p(x) 的函数，p(x) 是一个定义在 |X| 维概率单纯形上的向量。对于一个有100个输入符号的信道，max I(X;Y) 变成了一个在99维空间上的凹优化问题。你可以用解析方法求解一些简单的对称信道——但大多数"真实"信道没有闭式解。

1972年，Richard Blahut（当时还在麻省理工学院的林肯实验室）和Suguru Arimoto（日本东京大学的教授）各自独立地发现了一种迭代算法——现在被称为Blahut-Arimoto算法。这个算法的美妙之处在于：它将一个高维优化问题转化为了一个交替迭代的简单过程，每一步都保证提高I(X;Y)，直至收敛到全局最大值。

我第一次用这个算法计算一个非对称信道的容量时，看到p(x)的迭代过程——每一轮都在前一轮的基础上微调——就像看一个雕塑家逐步把一块粗糙的石头打磨成精美的雕像。这种"交替优化"的优雅，后来在机器学习领域（EM算法、变分推断）被反复"重新发现"。

容量计算的挑战

让我们先更清楚地理解为什么信道容量计算不是一个平凡的优化问题。

我们从定义出发：

C = max_{p(x)} I(X; Y)
  s.t.  ∑_x p(x) = 1, p(x) ≥ 0

其中 I(X;Y) = ∑_x ∑_y p(x) p(y|x) log_2( p(y|x) / p(y) )
         = H(Y) - H(Y|X)

注：p(y) = ∑_x p(x) p(y|x) 依赖于 p(x)

I(X;Y) 是 p(x) 的凹函数（concave function）。凹函数的最大化是一个"好"问题——局部最优就是全局最优。但实际操作中有两个困难：

  • 维度灾难：字母表大小|X|决定了优化空间的维度。对于|X|=1000的大信道，99.9%的可行域维度对搜索来说是一个天文数字。
  • 没有闭式解：除了少数对称信道，大多数信道没有解析解。

在Blahut和Arimoto之前，人们常用的方法是离散化搜索或者用通用凸优化工具——但效率都很低。Blahut-Arimoto算法的突破在于利用了信息论问题的特殊结构。

对称信道的容量

在讲Blahut-Arimoto之前，我们有必要先处理"软柿子"——对称信道。对于这类信道，问题有漂亮的闭式解。

定义：一个信道是对称的，如果信道矩阵的每一行都是其他行的排列，且每一列也都是其他列的排列（即行和列都是置换对称的）。

对于对称信道，容量由以下定理给出：

对称信道的容量由均匀输入分布达到，大小为：
C = log|Y| - H(信道的任意一行)

例1：BSC(p)

信道矩阵：[[1-p, p], [p, 1-p]]
|Y| = 2, 一行是 (1-p, p)
H(一行) = -p log p - (1-p) log(1-p) = H(p)
C = log 2 - H(p) = 1 - H(p) bits/use ✓

例2：二元删除信道 BEC(ε)

信道矩阵：[[1-ε, ε, 0], [0, ε, 1-ε]]
|Y| = 3, 一行是 (1-ε, ε, 0)
H(一行) = -(1-ε) log(1-ε) - ε log ε - 0·log 0
但等等！这给出了 C = log 3 - H(一行)，不是之前说的 1-ε。

为什么不一样？因为BEC是"弱对称"但不是"严格对称"的——
检查列：第1列是 (1-ε, 0)，第2列是 (ε, ε)，第3列是 (0, 1-ε)，
这三列不是彼此的排列。所以对称信道的定理不直接适用！

实际上BEC(ε)的容量是 C = 1 - ε，来自更一般的推导。

这个例子说明了一个重要的事情：不要盲目套用公式——你需要理解公式的适用条件。对称信道的定理只适用于"行和列都对称"的严格对称信道。

来看BSC容量随误码率p的变化：

p（误码率） C = 1 - H(p)（bits/use） 含义
0 1.000 完美信道，每秒可传1比特
0.01 0.919 1%误码只损失8%的容量
0.05 0.714 5%误码损失约29%
0.10 0.531 10%误码，容量刚好过半
0.20 0.278 20%误码，容量仅剩28%
0.50 0.000 完全随机，零容量

注意p=0.5时容量为零——这很合理：如果每传一个比特都有50%的概率被翻转，那信道输出和输入之间就没有任何相关性了，信息量为零。

Blahut-Arimoto算法

对于一般信道，我们需要Blahut-Arimoto算法。它的核心思想是交替优化：

信道容量有另一个等价表达式：

C = max_{p(x)} max_{q(x|y)} ∑_x ∑_y p(x) p(y|x) log(q(x|y)/p(x))

其中q(x|y)是一个"辅助分布"——它表示解码器对输入x的猜测，给定输出y。

这个表达式引入了一个"虚拟"的逆信道q(x|y)，我们可以通过交替优化p(x)和q(x|y)来逼近容量：

Blahut-Arimoto算法迭代步骤：

1. 初始化 p(x) 为均匀分布

2. 重复直到收敛：
   a. 计算 q(x|y) = p(x) p(y|x) / ∑_{x'} p(x') p(y|x')
      这就是给定y时x的后验概率

   b. 更新 p(x) = [ ∏_y q(x|y)^{p(y|x)} ] / 归一化常数
      或者等价地：
      p_new(x) = p(x) · exp( D(p(y|x) || q(x|y对应的p(y))) ) 归一化

3. 收敛后，C = I(X;Y)（用最终p(x)计算）

迭代的每一步都在增加I(X;Y)，且当算法收敛时，我们达到了全局最大值。证明的关键是：这个迭代对应于对I(X;Y)这个凹函数的"凸共轭"形式的交替最优化。

Python实现：

import numpy as np

def blahut_arimoto(channel_matrix, max_iter=1000, tol=1e-10):
    """
    Blahut-Arimoto算法计算信道容量

    参数:
        channel_matrix: shape (r, s), 第i行对应 p(y|x_i)
        max_iter: 最大迭代次数
        tol: 收敛阈值

    返回:
        capacity: 信道容量
        p_x: 最优输入分布
        history: 每次迭代的容量值
    """
    r, s = channel_matrix.shape
    # 初始化：均匀输入分布
    p_x = np.ones(r) / r
    history = []

    for iteration in range(max_iter):
        # 步骤a: 计算后验 q(x|y)
        # p(x,y) = p(x) * p(y|x)
        p_xy = p_x[:, np.newaxis] * channel_matrix
        # p(y) = ∑_x p(x,y)
        p_y = p_xy.sum(axis=0)
        # q(x|y) = p(x,y) / p(y), 注意处理 p_y=0 的情况
        q_given_y = np.zeros((r, s))
        for j in range(s):
            if p_y[j] > 1e-15:
                q_given_y[:, j] = p_xy[:, j] / p_y[j]
            else:
                q_given_y[:, j] = p_x  # 默认值

        # 步骤b: 更新 p(x)
        # 使用 exp(∑_y p(y|x) log q(x|y)) 然后归一化
        log_q = np.log(q_given_y + 1e-15)
        p_x_new = np.exp(np.sum(channel_matrix * log_q, axis=1))
        p_x_new /= p_x_new.sum()

        # 计算当前容量
        # I(X;Y) = ∑_x ∑_y p(x)p(y|x) log(q(x|y)/p(x))
        I = 0.0
        for i in range(r):
            for j in range(s):
                if p_xy[i,j] > 1e-15:
                    I += p_xy[i,j] * np.log2(q_given_y[i,j] / p_x[i])
        history.append(I)

        # 检查收敛
        if iteration > 0 and abs(I - history[-2]) < tol:
            break

        p_x = p_x_new

    return I, p_x, history

# 测试：非对称信道
# 一个 3×2 的非对称信道
P = np.array([
    [0.9, 0.1],
    [0.2, 0.8],
    [0.5, 0.5],
])

C, p_opt, hist = blahut_arimoto(P)
print("信道矩阵:")
print(P)
print(f"\n信道容量: {C:.6f} bits/use")
print(f"最优输入分布: p={p_opt.round(4)}")

# 验证对称信道：BSC
print("\n=== 验证 BSC(p=0.1) ===")
bsc = np.array([[0.9, 0.1], [0.1, 0.9]])
C_bsc, p_bsc, _ = blahut_arimoto(bsc)
print(f"BSC(0.1)容量: {C_bsc:.6f} (理论值: {1 - -0.1*np.log2(0.1) - 0.9*np.log2(0.9):.6f})")
print(f"最优输入: {p_bsc.round(4)} (理论值: [0.5, 0.5])")

# 输出示例：
# 信道容量: 0.459148 bits/use
# 最优输入分布: p=[0.5982 0.4018 0.0000]
# 注意：第三个符号p几乎为零，说明这个输入符号没用

注意输出中的有趣现象：第三个输入符号在最优输入分布下概率几乎为零！这意味着这个符号"不好"——对于信道容量没有贡献。在实际系统中，这告诉我们：你不需要使用所有可用的输入符号，只要使用那些"好的"就行。

Blahut-Arimoto算法特性：虽然收敛是线性的（速度不快），但对于小字母表（|X|,|Y| ≤ 几十个）来说，实用且稳定。对于大规模信道，可以使用更先进的凸优化方法（如牛顿法、内点法）。但这个算法的最大价值在于：它把"最大互信息"这个抽象概念，变成了一个可以实际运行的迭代过程。

容量在工程实践中的意义

信道容量不只是信息论教科书中的理论概念——它是通信工程师设计系统时的"北极星"。

在日常通信中，我们关心的指标是频谱效率（Spectral Efficiency）：每Hz带宽每秒钟能传输多少比特。对于加性白高斯噪声（AWGN）信道，Shannon给出了著名的公式：

C = B · log₂(1 + SNR)

其中 B 是带宽（Hz），SNR 是信噪比
这个公式有时称为 Shannon-Hartley 定理。

这个公式的含义是：如果你想增加容量，要么加带宽（B），要么提信噪比（SNR）。增加带宽是线性的好处，提高信噪比是log的好处——所以对大多数系统来说，增加带宽比提高发射功率更有效。但带宽是稀缺资源，这就是为什么频谱拍卖总是天价。

现代编码技术（LDPC码、Turbo码、Polar码）已经可以在距离Shannon极限0.1dB的范围内工作。这里有一个著名的故事：

故事：1993年，Turbo码在IEEE国际通信会议上发布时，全场震惊。在此之前，人们认为"实际编码"距离Shannon极限至少有3-4dB。Turbo码只离极限0.5dB——这是信道编码领域几十年努力的重大突破。此后，LDPC码（1963年由Gallager发明但被"遗忘"）被重新发现，Polar码（2009年由Erdal Arikan提出）在5G标准中得到了应用。如果说Shannon告诉了我们"极限在哪"，那么Turbo码、LDPC码和Polar码告诉我们"如何到达那里"。

2020年，5G NR标准选择LDPC码作为数据信道编码、Polar码作为控制信道编码——这标志着逼近Shannon极限的编码技术已经从实验室走向了数十亿人的口袋。

了解了信道容量的概念和计算方法，下一个自然的问题是：知道了容量C，我们是否真的能以接近C的速率进行可靠的通信？比如传输速率R = 0.8C，我们能做到99.999%的传输可靠吗？

Shannon第二定理——也就是信道编码定理——给出了一个肯定的答案。但这个定理的证明用了一个非常"奢侈"的想法：随机编码。Shannon没有给出具体的编码构造——他只是证明了"好的编码一定存在"。从"存在性证明"到"实用构造"，通信工程师们花了半个多世纪。
