栈与队列的底层实现

大家好，我是你们的讲师。今天我们来聊聊栈（Stack）和队列（Queue）——可能是整个数据结构体系中最"亲民"的两种结构。但千万别小看它们，栈和队列是很多复杂算法的基础构件：递归调用栈、BFS 队列、单调栈、消息队列……这些概念追根溯源都回到这两个简单的结构。

先来区分它们：栈是后进先出（LIFO, Last In First Out），就像一叠盘子，你只能从顶部取和放；队列是先进先出（FIFO, First In First Out），就像排队买奶茶，先到先得。

具体实现上，栈既可以用数组也可以用链表。数组栈的优点是缓存友好、访问快；链式栈的优点是动态扩容没有摊销成本。Python 中的 list 天然就是数组栈——append() 对应 push，pop() 对应 pop，时间复杂度均为 O(1) 均摊。队列则常用 collections.deque，它是双端队列，两端操作都是 O(1)。

小贴士：Python 中用 list 模拟队列（pop(0)）是 O(n) 操作，因为需要移动所有元素。一定要用 deque 来做队列操作，否则大数据量下直接超时。

我们来看一道经典的队列应用题——设计循环队列（LeetCode 622）。循环队列使用固定长度的数组和头尾指针，避免动态扩容开销，非常适合嵌入式或高性能场景。

class MyCircularQueue:
    def __init__(self, k: int):
        self.queue = [0] * k
        self.head = 0
        self.tail = 0
        self.size = 0
        self.capacity = k

    def enQueue(self, value: int) -> bool:
        if self.isFull():
            return False
        self.queue[self.tail] = value
        self.tail = (self.tail + 1) % self.capacity
        self.size += 1
        return True

    def deQueue(self) -> bool:
        if self.isEmpty():
            return False
        self.head = (self.head + 1) % self.capacity
        self.size -= 1
        return True

    def Front(self) -> int:
        return -1 if self.isEmpty() else self.queue[self.head]

    def Rear(self) -> int:
        return -1 if self.isEmpty() else self.queue[(self.tail - 1) % self.capacity]

    def isEmpty(self) -> bool:
        return self.size == 0

    def isFull(self) -> bool:
        return self.size == self.capacity

注意 (self.tail - 1) % self.capacity 这个取模操作——因为 tail 指向的是下一个空位，所以队尾元素实际上是 tail 的前一个位置。取模保证了指针在数组范围内循环移动。

括号匹配与表达式计算

栈在括号匹配问题中的应用堪称经典。这类问题的本质是：遇到左括号就入栈，遇到右括号就检查栈顶是否匹配。

有效的括号（LeetCode 20）

这是栈的"Hello World"，但它的变体可以非常复杂（比如加通配符、多层嵌套）。

def is_valid(s: str) -> bool:
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for ch in s:
        if ch in pairs:
            if not stack or stack.pop() != pairs[ch]:
                return False
        else:
            stack.append(ch)
    return not stack

这段代码容易出错的地方是：当遇到右括号时，如果栈为空就直接返回 False。有人会写成 stack.pop() != pairs[ch] and not stack——顺序错了！先检查栈是否为空再 pop，这是必须遵守的安全原则。

逆波兰表达式求值（LeetCode 150）

逆波兰表达式（Reverse Polish Notation）把运算符放在操作数后面，计算机求值时只需要一个栈：遇到数字入栈，遇到运算符弹出两个数字计算后再入栈。

def eval_rpn(tokens: list[str]) -> int:
    stack = []
    for t in tokens:
        if t in '+-*/':
            b = stack.pop()
            a = stack.pop()
            if t == '+':
                stack.append(a + b)
            elif t == '-':
                stack.append(a - b)
            elif t == '*':
                stack.append(a * b)
            else:  # t == '/'
                stack.append(int(a / b))  # 向零截断
        else:
            stack.append(int(t))
    return stack[0]

注意 Python 的 / 除法默认是浮点数除法，需要用 int() 做向零截断。另外操作数的顺序是 a 是左边的数，b 是右边的数——减法和除法时顺序不能搞反！

关键认知：逆波兰表达式是栈的完美匹配——运算符栈天然解决了运算符优先级问题。如果你需要自己实现中缀表达式求值（比如实现一个计算器），第一步就是把中缀表达式转为逆波兰表达式（Shunting Yard 算法）。

单调栈进阶

单调栈（Monotonic Stack）是栈的进阶应用——栈中的元素保持单调递增或递减。它的作用是快速找到数组中每个元素左边或右边第一个比它大/小的元素。

下一个更大元素（LeetCode 496 / 739）

先来看 496：给定两个数组 nums1 和 nums2，找出 nums1 中每个元素在 nums2 中的下一个更大元素。解法是用单调递减栈预处理 nums2。

再来看 739 每日温度：给定每日温度列表，返回每个位置需要等多少天才能等到更高的温度。这是单调栈最经典的场景。

def daily_temperatures(temperatures: list[int]) -> list[int]:
    n = len(temperatures)
    ans = [0] * n
    stack = []  # 存下标，温度单调递减
    for i in range(n):
        while stack and temperatures[i] > temperatures[stack[-1]]:
            idx = stack.pop()
            ans[idx] = i - idx
        stack.append(i)
    return ans

核心逻辑：当新温度高于栈顶温度时，说明栈顶元素的下一个更高温度出现了，弹出并计算结果。栈中始终保持递减——这就是单调栈的维护原则。

柱状图中最大的矩形（LeetCode 84）

这是一道 Hard 题，也是单调栈的集大成之作。给定 n 个非负整数表示柱子的高度，求能勾勒出的最大矩形面积。

def largest_rectangle_area(heights: list[int]) -> int:
    stack = []
    max_area = 0
    # 在原数组末尾加一个 0，确保最后把栈清空
    heights = heights + [0]
    for i, h in enumerate(heights):
        while stack and h < heights[stack[-1]]:
            height = heights[stack.pop()]
            left = stack[-1] if stack else -1
            width = i - left - 1
            max_area = max(max_area, height * width)
        stack.append(i)
    return max_area

这个解法的精妙之处在于：当遇到更矮的柱子时，栈中比它高的柱子就无法向右扩展了，我们计算以每根柱子为高的最大面积。栈中存的是下标，高度通过 heights[stack[-1]] 获取。

易错点：单调栈中存的是下标而不是值！很多人一上来写 stack.append(heights[i])，然后发现无法计算宽度——因为宽度需要两个下标之差。记住：栈存下标，通过数组取值。

队列的特殊应用

栈和队列虽然底层结构不同，但可以通过某种方式互相模拟。

用栈实现队列（LeetCode 232）

用两个栈来实现一个队列：入队栈（in_stack）和出队栈（out_stack）。入队时压入 in_stack；出队时如果 out_stack 为空，将 in_stack 中所有元素弹出并压入 out_stack，然后从 out_stack 弹出。这样就实现了 FIFO。

class MyQueue:
    def __init__(self):
        self.in_stack = []
        self.out_stack = []

    def push(self, x: int) -> None:
        self.in_stack.append(x)

    def pop(self) -> int:
        self._move()
        return self.out_stack.pop()

    def peek(self) -> int:
        self._move()
        return self.out_stack[-1]

    def empty(self) -> bool:
        return not self.in_stack and not self.out_stack

    def _move(self):
        if not self.out_stack:
            while self.in_stack:
                self.out_stack.append(self.in_stack.pop())

时间复杂度分析：每个元素最多入栈两次（一次入 in_stack，一次入 out_stack），出栈两次，均摊后每个操作是 O(1)。

用队列实现栈（LeetCode 225）

反过来用队列实现栈稍微麻烦一些——因为队列是 FIFO，而栈是 LIFO。一种方法是用两个队列，另一种更简洁的方法是用一个队列：每次 push 时，把新元素放到队尾，然后把前面的元素依次出队再入队，这样新元素就"浮"到了队首。

class MyStack:
    def __init__(self):
        self.q = deque()

    def push(self, x: int) -> None:
        self.q.append(x)
        # 把前 n-1 个元素移到后面去
        for _ in range(len(self.q) - 1):
            self.q.append(self.q.popleft())

    def pop(self) -> int:
        return self.q.popleft()

    def top(self) -> int:
        return self.q[0]

    def empty(self) -> bool:
        return not self.q
思维拓展：232 和 225 这两道题看起来像是"为了做题而做题"，但它们实际上教会了你一个重要的抽象能力——一种数据结构的特性可以通过另一种结构组合来实现。这种抽象思维在系统设计中无处不在（比如用消息队列实现异步解耦）。

踩坑与总结

说一个我自己的单调栈踩坑故事。有一次写 LeetCode 84（柱状图最大矩形），我自以为理解了单调栈，直接开写，结果一提交就 WA（Wrong Answer）。debug 了半个小时才发现问题：我没有在 heights 末尾加 0。不加 0 的话，如果最后柱子是递增的，栈就永远不会被清空，最后一个柱子的面积就永远算不出来。这种"哨兵技巧"（Sentinel）在单调栈问题中至关重要——它确保所有元素最终都能出栈计算面积。

另一个常见问题是栈溢出。Python 的 list 作为栈理论上只受内存限制，但在递归场景下（比如系统调用栈），Python 默认递归深度限制是 1000。如果你用递归做深度优先搜索，遇到大规模数据就可能 RecursionError。这时候要手动设置 sys.setrecursionlimit(1000000)，或者直接改用显式的栈来实现。

典型应用 括号匹配、函数调用、回溯 BFS、缓存、任务调度

面试要点：栈和队列本身不难，但面试官通常把它们和具体场景结合来考——"用栈实现队列"考的是你能否跳出 LIFO 思维定势，"柱状图最大矩形"考的是你能否识别出单调栈的适用场景。掌握它们的最好方法是理解每个模式背后的"为什么要用栈/队列"，而不是死记代码模板。
