第18章 评测体系

2024 年底，DeepSeek-V3 的技术报告发布后，AI 社区出现了两种截然不同的声音。一边惊叹于它 $5.6M 的训练成本，另一边则紧盯着那张评测对比表，寻找它不如 GPT-4o 的地方。"在 GPQA 上还差 GPT-4o 两个点"、"在 LiveCodeBench 上只领先 0.3%"——人们以显微镜般的精度审视着每一行数字。

这种热烈的讨论本身说明了一个事实：大模型的能力已经无法靠"感觉"来衡量了。 几年前，大家还在问"这个模型能用吗"，现在问的是"你的 MMLU 多少分，我的 GPQA 多少分"。一个成熟的评测体系，是大模型领域从"蛮荒时代"走向"标准化时代"的标志。

这一章，我们系统地了解大模型评测的主要基准，以及 DeepSeek 系列在各个基准上的表现。

18.1 大模型评测概述

大模型评测通常分为四大类，每一类评测模型的不同维度：

评测类别 代表基准 评测内容
知识评测 MMLU, MMLU-Pro, C-Eval 模型对各学科知识的掌握程度
推理评测 GPQA, ARC, BBH, HellaSwag 模型的逻辑推理、科学分析能力
代码评测 HumanEval, LiveCodeBench, MBPP 代码生成、调试、算法实现能力
数学评测 MATH-500, GSM8K, AIME 数学计算、问题求解、定理证明

此外还有对齐评测（如 MT-Bench, AlpacaEval 评估模型的对话质量）、安全评测（评估模型的安全性和偏见）、长文本评测（如 RULER, Needle-in-a-Haystack）等等。一个完整的大模型评测体系需要覆盖所有这些维度。

评测的"内卷"：随着模型能力的整体提升，许多早期基准已经出现了"天花板效应"——顶级模型的得分超过了 95% 甚至 99%，无法区分模型之间的能力差异。因此，每隔一段时间就会出现更难的新基准（例如从 MMLU 到 MMLU-Pro），以保持对前沿模型的区分度。

18.2 MMLU 与 MMLU-Pro

MMLU（Massive Multitask Language Understanding） 是大模型评测中最经典、引用最广泛的基准之一。它覆盖了 57 个学科，从高中数学到美国历史、从医学到法学，横跨人文、社科、理工三大领域。每个学科包含数百道四选一或五选一选择题，评估模型在零样本或少样本设定下的知识广度。

MMLU 的设计理念是：一个真正智能的模型应该像接受了通识教育的大学生一样，在广泛学科上都具备基本的知识水平。 它不像某些专项基准那样只测一个能力维度，而是考验模型的"综合素质"。

DeepSeek-V3 在 MMLU 上取得了 88.5 分，超越了 GPT-4o 的 87.2 分。这个成绩的意义在于：V3 是一个 671B 总参数、37B 激活参数的 MoE 模型，但在知识广度上击败了参数更多（传闻 GPT-4 达 1.8T）的 dense 模型。MoE 的"专家分工"机制使得模型可以在相同的计算成本下"知道更多"，这正是 V3 在知识类评测中表现出色的原因。

MMLU-Pro 是 MMLU 的困难版本。它在 MMLU 的基础上做了两个改动：一是增加了更难的题目，二是将选项从 4 个扩展到 10 个，大幅降低了"蒙对"的概率。MMLU-Pro 的难度显著高于 MMLU，顶尖模型在 MMLU-Pro 上的得分通常在 60-75 之间。DeepSeek-V3 在 MMLU-Pro 上取得了 75.9 分，同样超越了 GPT-4o 的 73.7 分。

MMLU 的意义不止于分数：DeepSeek 团队在开发过程中发现，MMLU 分数可以作为模型"知识密度"的有效指标。当他们在消融实验中尝试不同的 MoE 配置时，MMLU 分数的变化趋势与模型在其他所有基准上的平均表现高度相关（相关系数 > 0.85）。这让 MMLU 成为了他们架构探索阶段的首选"代理指标"。

18.3 GPQA

GPQA（Google-Proof Q&A） 是一个以"难"著称的评测基准。它的题目由各领域的博士研究生编写，涉及物理学、化学、生物学等专业领域的深入推理。每一道题都经过严格筛选，确保"即使 Google 搜索也无法直接找到答案"——你需要在理解专业概念的基础上进行推理。

GPQA 的难度有多高？作为一个参考，人类博士研究生在 GPQA 上的得分约为 80%，而本科毕业生约为 40-50%。这意味着 GPQA 能够有效区分"普通 AI"和"专家级 AI"——这正是它作为新一代评测基准的价值所在。

DeepSeek-V3 在 GPQA 上取得了 59.1 分（Diamond 子集），这个成绩略低于 GPT-4o 的 62.4 分，但超过了 Claude 3.5 Sonnet 的 58.7 分。考虑到 GPT-4o 的训练成本远高于 V3，这个成绩已经很出色了。而且，在后续的 DeepSeek-R1 中，通过强化学习训练推理能力的模型在 GPQA 上进一步提升到了 71.5 分——首次超越了 GPT-4o。

GPQA 的得分差异揭示了一个有趣的规律：知识类评测（如 MMLU）主要依赖模型"记住了多少"，而推理类评测（如 GPQA）更依赖模型"想得有多深"。 DeepSeek-V3 在 MMLU 上的优异表现说明它的"记忆容量"足够大，而在 GPQA 上与 GPT-4o 的微小差距说明它的"推理深度"仍有提升空间——这正是后续 R1 模型重点攻克的方向。

18.4 HumanEval 与 LiveCodeBench

代码生成是大模型最受欢迎的应用场景之一。评测模型的代码能力，业界主要使用以下两个基准。

HumanEval

HumanEval 是 OpenAI 在 2021 年提出的代码生成基准，包含 164 道手写的 Python 编程问题。每道题包含函数签名、文档字符串和多个测试用例，模型需要生成完整的函数实现。评测指标是 pass@k——在 k 次采样中，至少有一次通过所有测试用例的比例。

DeepSeek-V3 在 HumanEval 上的 pass@1 得分为 82.6，接近 GPT-4o 的 86.2。但代码类评测的一个重要特点是：模型的分数高度依赖于正确的 few-shot prompting 和 post-processing。DeepSeek 团队发现，通过优化 prompt 中的代码风格示例和输出格式约束，V3 的 HumanEval 分数可以进一步提升到 90+ 的水平。

LiveCodeBench

LiveCodeBench 是 2024 年推出的一个动态代码评测基准，它的独特之处在于题目来自最新的编程竞赛（如 Codeforces、LeetCode 等），并且持续更新。这意味着它避免了"数据泄露"问题——模型无法在训练数据中看到评测题目。

DeepSeek-V3 在 LiveCodeBench 上取得了 39.2 分，是所有开源模型中的最高分，甚至略高于 GPT-4o（38.9 分）。对于编程竞赛级别的题目，V3 展现了极强的代码推理能力。这得益于 DeepSeek 在训练数据中加入了大量的代码数据——他们在 Pre-training 阶段使用了约 2T tokens 的代码数据（占总数据的约 15%），而且专门针对代码的格式和结构做了优化。

开源的胜利：在 LiveCodeBench 的排行上，DeepSeek-V3 是唯一进入前五的开源模型，其他四名（GPT-4o, Claude 3.5, Gemini 1.5 Pro）均为闭源模型。这证明了开源社区在代码生成领域完全有能力与闭源巨头一较高下。

18.5 MATH-500 与 GSM8K

数学推理被认为是"大模型智能的试金石"。一个模型如果连小学数学题都算不对，很难让人相信它能处理复杂的高阶推理。

GSM8K（Grade School Math 8K） 是最广泛使用的数学推理基准之一，包含 8,500 道小学数学应用题。题目虽然"小学水平"，但对模型的推理链条要求很高——模型需要理解自然语言描述的数学情境，将其转化为数学表达式，并逐步计算出结果。GSM8K 的评测方式通常使用 chain-of-thought（思维链）prompting，让模型在给出最终答案前先展示推理步骤。

DeepSeek-V3 在 GSM8K 上的得分为 94.2，超过了 GPT-4o 的 92.0。如果使用 8-shot CoT 方式，分数可以进一步提升到 96.5。这个成绩说明 V3 在基础数学推理方面已经达到了人类优秀小学生的水平。

MATH-500 是 MATH 数据集的一个精选子集，包含 500 道高中至大学低年级水平的数学竞赛题，涵盖代数、几何、概率、数论等多个分支。相比 GSM8K，MATH 的难度显著更高，很多题目需要多步推理和非标准的解题技巧。

DeepSeek-V3 在 MATH-500 上的得分为 90.2，而 GPT-4o 为 88.9。这个成绩说明 V3 不仅会算小学算术，还能处理一定复杂度的数学竞赛题。

# 数学评测的典型 prompt
# (GSM8K 示例)
#
# 问题: 小明有 12 个苹果，他给了小红三分之一的苹果，
# 然后又给了小刚 2 个，请问小明还剩下几个苹果？
#
# 推理步骤:
# 1. 小明给了小红 12 × 1/3 = 4 个苹果
# 2. 小明又给了小刚 2 个苹果
# 3. 小明一共给出 4 + 2 = 6 个苹果
# 4. 小明还剩 12 - 6 = 6 个苹果
#
# 答案: 6
CoT 的重要性：研究表明，使用 Chain-of-Thought（思维链）prompting 可以将 GSM8K 的得分提升 15-20 个百分点。DeepSeek 在训练时专门加入了推理链数据，确保模型能够像人类一样"逐步思考"而非"直接猜答案"。这也是 V3 在数学推理上超越 GPT-4o 的重要原因。

18.6 综合对比表

最后，我们来看一张 DeepSeek 全系列模型在主要基准上的综合对比表。这张表可以清晰地展示 DeepSeek 从 V2 到 V3 再到 R1 的能力演进。

基准 DeepSeek-V2 DeepSeek-V3 DeepSeek-R1 GPT-4o Claude 3.5 Sonnet
MMLU 78.5 88.5 90.8 87.2 88.3
MMLU-Pro — 75.9 — 73.7 —
GPQA Diamond 42.8 59.1 71.5 62.4 58.7
HumanEval 72.6 82.6 — 86.2 84.9
LiveCodeBench — 39.2 — 38.9 34.4
GSM8K 85.6 94.2 96.3 92.0 93.1
MATH-500 72.3 90.2 95.9 88.9 89.7
从中可以得出的几条重要结论：
1. V3 → R1 的推理飞跃：在 GPQA 和 MATH-500 等推理密集型基准上，R1 相比 V3 的提升非常大（GPQA 从 59.1 到 71.5，MATH-500 从 90.2 到 95.9），这体现了强化学习训练对推理能力的显著增强。
2. V2 → V3 的全方位提升：从 V2 到 V3，几乎所有基准都提升了 10 分以上，这得益于模型规模的扩大（236B → 671B）和数据量的增加。
3. 开源模型的竞争力：DeepSeek-V3 在多数据标上已经与 GPT-4o、Claude 3.5 等顶级闭源模型持平甚至超越，证明了开源路线的可行性。
评测的局限性：基准分数虽然重要，但并不能完全反映模型在实际场景中的表现。一个在 MMLU 上得高分的模型，在实际对话中可能显得呆板和教条；一个在 HumanEval 上满分通过的模型，在复杂的工程项目中可能完全不会用。评测基准是很好的参照系，但永远不能替代实际体验——特别是一个模型在你自己任务上的体验。
