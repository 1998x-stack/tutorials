第1章 课程导论与FPGA发展史

大家好，欢迎来到《FPGA/Verilog 嵌入式开发 · 从入门到精通》。我是你们的讲师。如果你是一名嵌入式工程师，每天和 STM32、ARM Cortex-M 打交道，用 C 语言在 while(1) 循环里写业务逻辑——那么你即将进入一个思维方式完全不同的世界。在这个世界里，没有"顺序执行"，所有模块同时运行；没有"变量存在内存里"，每根信号都是一条物理连线。这就是 FPGA 的世界。

本章我们先站高一点，看清 FPGA 从哪里来、在整个计算版图中处于什么位置、以及为什么 2026 年的今天，FPGA 开发正在经历一场由 AI 驱动的范式变革。

1.1 从可编程逻辑的起源讲起

故事要从 1970 年代讲起。当时的数字电路设计靠的是 74 系列 TTL 逻辑芯片——与门、或门、触发器，工程师在面包板上插芯片、飞线，设计一个复杂电路就要堆几十上百颗芯片。一旦设计需要修改，就得重新布线，甚至重新画 PCB。这就是所谓的"固定功能硬件"时代。

1975 年，Signetics 公司推出了第一代可编程逻辑器件——PAL（Programmable Array Logic，可编程阵列逻辑）。PAL 内部是"与阵列可编程 + 或阵列固定"的结构，用户通过熔丝编程决定逻辑功能。随后出现的 GAL（Generic Array Logic） 引入了电可擦除技术，允许反复编程。1980 年代的 CPLD（Complex PLD） 则将多个 PAL-like 宏单元集成到一颗芯片上，通过可编程互连矩阵连接。

但真正的革命发生在 1985 年——Ross Freeman 和他在 Xilinx 的团队推出了世界上第一款 FPGA：XC2064。

FPGA 的核心创新：Freeman 的革命性想法是——与其让用户编程"逻辑功能"（像 PAL 那样），不如让用户编程"逻辑单元之间的连接"。这就是 FPGA 得名的原因——Field-Programmable Gate Array，现场可编程门阵列。门阵列的架构是固定的（LUT + Flip-Flop + 互连），但用户通过配置 SRAM 单元来定义每个 LUT 的真值表和互连开关的状态，从而实现任意数字电路。

1.2 FPGA 的架构演进：从 LUT-4 到异构 SoC

第一款 FPGA XC2064 只有 64 个逻辑单元，采用 2.5μm 工艺，每个 LUT 只有 3 输入。将近 40 年后的今天，一块高端 FPGA 拥有数百万逻辑单元、数千个 DSP slices、高速 SerDes、嵌入式 ARM 核，甚至 AI 推理引擎。让我们看一下这个演进历程：

可以看到，FPGA 已经从单纯的"逻辑胶合"角色，演变为集成了处理器、DSP、AI 引擎、高速 I/O 的异构计算平台。

1.4 FPGA vs MCU vs GPU vs ASIC：FPGA 到底"牛"在哪？

很多从嵌入式转 FPGA 的工程师会问：FPGA 和单片机到底有什么区别？它和 GPU 比谁更快？为什么不直接做 ASIC？下面这张表讲清楚：

核心思维差异——MCU 人最容易犯的错：MCU 上写 LED = 1; delay(1000); LED = 0;，两行代码顺序执行。FPGA 上没有"delay"的概念——你需要实例化一个计数器，在时钟沿检测计数值达到目标时翻转 LED。这不是"写程序"，而是"描述一个在时钟驱动下自动运行的硬件电路"。本教程会反复强调这一点，直到它成为你的肌肉记忆。

1.5 2026 年 FPGA 行业的三大变革

FPGA 行业历来变化缓慢——从 LUT 阵列到 HLS 的普及花了将近二十年。但在 2025-2026 年，三股力量正在加速重塑这个行业：

变革一：AI 驱动的设计自动化。用自然语言描述需求，由大语言模型（LLM）直接生成可综合的 Verilog 代码，已经进入了实用阶段。像 SynthPilot 这样的 MCP 服务器让 LLM 可以直接操控 Vivado 的 414 个工具命令。一个 AXI4-Lite 从接口的完整 Verilog 模块，AI 数秒内就能生成，骨架正确率达 90% 以上。虽然时序敏感场景（跨时钟域同步等）仍需人工介入，但"自然语言→比特流"的路径已经清晰可见。

变革二：开源工具链成熟化。2026 年 5 月，AMD 宣布 Vivado 2026.1 取消 Linux 免费版授权。这促使大量开发者转向开源方案：Yosys（综合）+ nextpnr（布局布线）+ openXC7（Xilinx 7 系列数据库）+ openFPGALoader（比特流烧录）。对于 iCE40 和 ECP5 系列的 Lattice FPGA，开源工具链已经做到了"生产可用"级别。国产 GOWIN FPGA 的开源支持（Apicula 项目）也在快速发展。

变革三：HLS + RTL 混合设计成为主流。业界已经达成共识——算法内核（图像滤波、矩阵乘法、FFT、AI 推理）用 HLS 写（C/C++/SystemC），接口/协议/控制（AXI 总线、PCIe TLP 解码、DDR 控制器、系统级时序）用手写 RTL。两者各司其职，在同一工程中混合使用。这不是"谁替代谁"的问题，而是"什么时候用什么"的问题。

1.7 一个真实的踩坑故事

2015 年我刚接触 FPGA 时，用 Verilog 写了一个 UART 发送器。仿真波形完美——起始位、8 位数据、停止位，时序和教科书一模一样。信心满满地下载到开发板，接上串口调试助手……全是乱码。

我检查了三天。重写了三遍代码。最后发现是波特率发生器在复位释放后的第一个时钟沿状态不确定——仿真时复位信号干净利落地从 0 跳到 1，而真实电路中的复位释放是一个异步事件，可能与时钟沿有皮秒级的偏移（也就是我们后面会详细讲的"复位释放时序违例"）。仿真通过了，因为仿真器假设世界是理想的。

这个教训让我刻骨铭心——仿真不是万能的，硬件调试需要一整套方法论。这也是为什么本教程在每个关键知识点上都会加入"踩坑"故事和调试建议，而不仅仅是罗列语法和原理。
