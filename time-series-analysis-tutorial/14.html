第14章 预测与评估

大家好！经过前面几章的学习和建模实践，我们终于到了最激动人心的一步——用模型做预测，并回答那个终极问题："这个模型到底准不准？"

预测是时间序列分析的"产品交付"环节。不管你的模型理论多么完美、诊断多么漂亮，如果预测结果不准，在业务眼中就是废的。这一章我们聚焦两件事：怎么做预测，以及怎么判断预测好不好。

14.1 条件期望预测：ARIMA 如何"看到"未来

ARIMA 模型的预测基于一个优雅的思想——条件期望（Conditional Expectation）。给定到今天为止的所有历史信息，我们对未来某个时刻 Y_{t+h} 的最佳预测就是 E[Y_{t+h} | Y₁, Y₂, ..., Y_t]，也就是"在当前已知信息下，Y_{t+h} 的期望值"。

为什么是"条件期望"？因为对于 ARIMA 模型，条件期望恰好就是最小均方误差预测（Minimum Mean Square Error Forecast）。没有其他预测函数能产生更小的平均误差平方。

对于 ARIMA 模型，预测可以一步步递推：

一步预测（h=1）：Ŷ_{t+1|t} = c + φ₁Y_t + φ₂Y_{t-1} + ... + θ₁ε_t + θ₂ε_{t-1} + ...

注意所有右侧变量在时刻 t 都是已知的（Y 的历史值和 ε 的估计值），所以可以直接计算。

两步预测（h=2）：Ŷ_{t+2|t} = c + φ₁Ŷ_{t+1|t} + φ₂Y_t + ... + θ₂ε_t + ...

这里 Y_{t+1} 用预测值代替，ε_{t+1} 用 0 代替（因为 ε 是白噪声，对未来不可预测）。多步预测时，MA 部分会逐渐"耗尽"——超过 q 步后 MA 项全部为 0。

关键性质：对于平稳 ARMA 模型，随着预测步数 h 增加，预测值会收敛到序列的无条件均值。预测方差则从 σ²（一步预测误差方差）逐渐增大到序列的总体方差。这意味着：预测越远，你越不确定，所以预测会"回归"到平均水平。

14.2 预测误差方差与置信区间

点预测（Point Forecast）只有一半的信息，另一半是"这个预测有多不确定"。这就是预测区间（Prediction Interval）的作用。

对于 ARIMA 模型，在正态误差假设下，h 步预测的 95% 预测区间为：

Ŷ_{t+h|t} ± 1.96 × √(Var(e_{t+h|t}))

其中 Var(e_{t+h|t}) 是 h 步预测误差的方差，它随着 h 增加而增加。预测区间的宽度反映了预测的不确定性：

  • h=1 时区间最窄（只包含一步预测误差 σ²）
  • h 增大时区间变宽（因为多步预测误差是各步冲击的累积）
  • 对平稳 ARMA，区间最终收敛到一个固定宽度
  • 对 ARIMA（有差分），区间会随着 h 线性或指数级变宽
重要：如果 ARIMA 模型包含差分（d ≥ 1），预测区间会随着 h 趋于无穷而无限变宽。这完全合理——因为你无法无限精确地预测一个随机游走的未来位置。相反，如果数据是平稳的（d=0），预测区间会收敛到一个固定宽度。两种情形都是对的，只是数据的性质不同。

14.3 预测评估指标

如何量化一个预测的准确度？最常用的指标有以下几个：

指标 公式 特点
MAE mean(|Y_t - Ŷ_t|) 直观，单位与原始数据一致
RMSE sqrt(mean((Y_t - Ŷ_t)²)) 对大误差惩罚更重
MAPE mean(|Y_t - Ŷ_t| / Y_t) × 100% 无量纲，方便跨序列比较
MASE MAE / MAE_naive 相对于朴素基准的比较

MAE（Mean Absolute Error）是最直观的指标——平均每个预测偏差了多少。单位就是 Y 的单位，易懂但无法跨不同量纲的序列比较。

RMSE（Root Mean Square Error）由于平方操作，它对大误差的惩罚比 MAE 更重。如果你的业务场景中"大偏差"的代价远大于"小偏差"（比如库存缺货比库存过剩严重得多），RMSE 更合适。

MAPE（Mean Absolute Percentage Error）将误差转化为百分比，方便跨不同量级的序列比较。但 MAPE 有一个致命的缺陷——当实际值 Y_t 接近零时，MAPE 会趋向无穷大。

MAPE 翻车现场：有一次我在评估一个销量预测模型，大部分产品的 MAPE 在 15% 左右，但某个产品的 MAPE 高达 5800%。检查后发现：这个产品有几个月销量为零（缺货），公式 |0 - 预测值| / 0 产生了无穷大的百分比误差。更隐蔽的问题是，另一个产品的实际销量是 0.5 和 1.5 之间波动，MAPE 也在 50%~150% 之间——不是因为预测差，而是因为分母太小。所以请不要在间歇性需求数据上使用 MAPE！改用 MASE 或 MAE。

MASE（Mean Absolute Scaled Error）由 Rob Hyndman 提出，它把 MAE 除以一个"朴素预测"（naive forecast）的 MAE。朴素预测就是 Ŷ_t = Y_{t-1}（用上一期的值作为预测）。如果 MASE < 1，说明你的模型比朴素预测好；如果 MASE > 1，说明你不如简单用上一期值来预测。MASE 解决了 MAPE 的分母为零问题，也支持跨序列比较。

14.4 预测包络与 backtesting

预测包络（Forecast Envelope）是把不同步长的预测误差画在一起的工具。它的做法是：

  1. 在训练集上拟合模型
  2. 预测未来 h 期
  3. 当新的观测到来时，记录预测误差
  4. 滚动窗口重复上述过程，累积每个步长上的误差分布

通过预测包络，你可以回答："这个模型在 3 个月后的预测，通常偏差多大？"而不是笼统地说"平均误差 10%"。

Backtesting（回测）是评估预测模型最可靠的方法之一。它的基本思路是模拟历史——每次只用"到当前时刻为止"的数据做预测，然后与真实值比较：

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from statsmodels.tsa.arima.model import ARIMA
from sklearn.metrics import mean_absolute_error, mean_squared_error

# 生成模拟数据
np.random.seed(42)
n = 300
Y = np.zeros(n)
eps = np.random.normal(0, 1, n)
for t in range(1, n):
    Y[t] = 0.7*Y[t-1] + eps[t] + 0.3*eps[t-1]

# 回测：滚动预测
train_size = 200
test_size = n - train_size

predictions = []
actuals = Y[train_size:]

for i in range(test_size):
    # 每次使用到当前为止的所有数据
    train_data = Y[:train_size + i]

    # 拟合模型
    model = ARIMA(train_data, order=(1, 0, 1), trend='c')
    result = model.fit()

    # 预测下一步
    forecast = result.forecast(steps=1)
    predictions.append(forecast.iloc[0])

# 评估
predictions = np.array(predictions)
mae = mean_absolute_error(actuals, predictions)
rmse = np.sqrt(mean_squared_error(actuals, predictions))

# 也计算朴素预测的 MAE 作为基准
naive_pred = Y[train_size-1:-1]
mae_naive = mean_absolute_error(actuals, naive_pred)
mase = mae / mae_naive

print(f"MAE:  {mae:.3f}")
print(f"RMSE: {rmse:.3f}")
print(f"MASE: {mase:.3f} (基准朴素 MAE={mae_naive:.3f})")

# 画预测 vs 真实
plt.figure(figsize=(12, 5))
plt.plot(np.arange(train_size, n), actuals, 'b-', label='真实值')
plt.plot(np.arange(train_size, n), predictions, 'r--', label='预测值')
plt.legend()
plt.title('回测结果：滚动一步预测')
plt.show()

14.5 预测的"黑暗面"

作为一个负责任的"老师"，我必须告诉你们：预测不是魔法。即使你严格按照所有步骤做了，预测仍然可能在以下情况失效：

  • 结构突变（Structural Break）：如果数据生成过程在预测期内发生了变化（比如疫情爆发、政策调整），基于历史数据的模型必然失效。这被称为"卢卡斯批判"（Lucas Critique）在时间序列中的体现。
  • 预测越远，不确定性越大：这不是模型问题，而是信息问题——未来本来就是不确定的。一个"敢给很窄的预测区间"的模型往往不是好模型，而是过度自信的模型。
  • 组合预测可能优于单一模型：研究反复表明，多个模型的预测取加权平均（Model Averaging / Ensemble）往往比任何一个单一模型更稳定。这就是为什么在大型预测竞赛中，冠军方案永远是多个模型的组合。
ARMA 家族系列总结：从第 8 章到第 14 章，我们走完了一个完整的旅程——从最基础的 AR 模型开始，了解了 MA 模型的冲击捕捉能力，学会了如何用 ARMA 两者结合，扩展出差分处理趋势（ARIMA），加入季节性成分（SARIMA），最后掌握了完整的建模诊断流程和预测评估方法。这 7 章的内容构成了经典时间序列分析的核心骨架。当你今后面对任何一个时间序列预测问题时，这套方法论都将是你的第一个武器。当然，经典方法不是终点——第 15 章我们将进入波动率建模（ARCH/GARCH），处理金融时间序列中特有的"方差时变"问题。
