前言：最简单的结构，最不简单的技巧

大家好，我是你们的讲师。数组和字符串看起来是"入门级"的内容，但如果你去看 LeetCode 的热题榜，排在前面的很大一部分都是数组题。为什么？因为数组虽然简单，但围绕它可以衍生出大量精妙的算法技巧——前缀和、双指针、滑动窗口——这些技巧放到其他数据结构上依然成立。

字符串本质上就是一维字符数组，所以很多数组技巧在字符串上同样适用。但同时，字符串有自己独特的性质——不可变性、编码问题、模式匹配——这些也值得单独拿出来讲。今天我们就来系统地梳理数组和字符串中的核心技巧。

数组的内存模型

数组之所以能实现 O(1) 随机访问，根本原因在于连续内存分配。当你声明一个数组 arr = [10, 20, 30, 40] 时，计算机会在内存中分配一块连续的存储空间。每个元素占据固定大小的字节，那么 arr[i] 的地址就是：

arr[i] 的地址 = 数组起始地址 + i × 每个元素的大小

这是一条简单的乘法指令——不需要遍历，不需要比较，直接算出来。所以不管数组有 10 个元素还是 10 亿个元素，arr[5000000] 的访问时间是一样的。

但硬币的另一面是：插入和删除是昂贵的。如果你要在数组中间插入一个元素，必须把后面的所有元素依次往后移，平均时间复杂度是 O(n)。这决定了数组的适用场景——读多写少。

动态数组：Python list 的底层实现

Python 的 list 本质上就是动态数组。和静态数组不同，它不要求你在创建时指定大小——当空间不够时，它会自动扩容。

import sys

arr = []
for i in range(20):
    arr.append(i)
    print(f"len={len(arr):2d}, capacity={sys.getsizeof(arr):4d} bytes")
# 你会看到 getsizeof 的值并不是均匀增长的
# 它在一定范围内不变，然后突然跳变——那就是扩容发生了

关于扩容策略，这里有一个有意思的话题：不同的语言使用的扩容因子不同。Python 早期用的是 1.125 倍（约 12.5% 的增长），CPython 3.7+ 做了调整但核心逻辑不变。相比之下，C++ 的 std::vector 通常用 2 倍，Java 的 ArrayList 用 1.5 倍。

为什么有这些差异？扩容因子大（比如 2 倍），扩容次数少，但每次浪费的内存多；扩容因子小（比如 1.125 倍），内存利用率高，但扩容频繁，拷贝开销大。这又是一个典型的时间-空间权衡问题。

前缀和技巧

前缀和（prefix sum）是数组问题中最基础也最实用的技巧之一。它的思想很简单：预处理出一个前缀和数组，然后区间和就能在 O(1) 时间内求出。

def prefix_sum(arr):
    n = len(arr)
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + arr[i]
    return pref

# 求 arr[l:r] 的区间和 (左闭右开)
def range_sum(pref, l, r):
    return pref[r] - pref[l]

arr = [1, 2, 3, 4, 5]
pref = prefix_sum(arr)
print(range_sum(pref, 1, 4))  # 2 + 3 + 4 = 9

LeetCode 560. 和为 K 的子数组 是前缀和的经典变体。题目要求找出数组中所有和为 k 的连续子数组的个数。暴力法是 O(n²)，用前缀和 + 哈希表可以优化到 O(n)：

from collections import defaultdict

def subarray_sum(nums, k):
    pref = 0
    count = 0
    sum_map = defaultdict(int)
    sum_map[0] = 1   # 前缀和为 0 出现了一次

    for num in nums:
        pref += num
        # 如果存在 pref - k，说明找到了和为 k 的子数组
        count += sum_map[pref - k]
        sum_map[pref] += 1

    return count

这个解法在面试中出现频率极高——它把 O(n²) 的问题变成了 O(n)，充分展示了"空间换时间"的思想。

双指针技巧

双指针（two pointers）是数组问题的另一大神器。常用的双指针模式有两种：

对撞指针

两个指针分别从数组两端向中间移动，典型例子是 LeetCode 167. 两数之和 II — 输入有序数组：

def two_sum(numbers, target):
    left, right = 0, len(numbers) - 1
    while left < right:
        s = numbers[left] + numbers[right]
        if s == target:
            return [left + 1, right + 1]
        elif s < target:
            left += 1
        else:
            right -= 1
    return [-1, -1]

三数之和 (LeetCode 15)

两数之和的升级版——找三个数使它们的和为 0：

def three_sum(nums):
    nums.sort()
    res = []
    n = len(nums)
    for i in range(n - 2):
        # 跳过重复的 i
        if i > 0 and nums[i] == nums[i - 1]:
            continue
        left, right = i + 1, n - 1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            if s == 0:
                res.append([nums[i], nums[left], nums[right]])
                # 跳过重复
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1
                left += 1
                right -= 1
            elif s < 0:
                left += 1
            else:
                right -= 1
    return res

三数之和的难点不在于双指针本身，而在于去重逻辑。很多人在面试中会忘记跳过重复元素，导致结果中出现重复的三元组。这是一个需要细心的地方。

滑动窗口

滑动窗口（sliding window）可以看作双指针的一种特殊形式——两个指针之间的区间就是"窗口"，窗口在数组上滑动。它特别适合解决连续子数组/子串问题。

LeetCode 3. 无重复字符的最长子串 是滑动窗口的教科书题目：

def length_of_longest_substring(s):
    window = set()
    left = 0
    max_len = 0

    for right in range(len(s)):
        # 如果 s[right] 重复了，移动左指针直到不重复
        while s[right] in window:
            window.remove(s[left])
            left += 1
        window.add(s[right])
        max_len = max(max_len, right - left + 1)

    return max_len

这道题的模板可以泛化到很多滑动窗口问题：

# 滑动窗口通用模板
left = 0
for right in range(len(arr)):
    # 将 arr[right] 加入窗口
    # 更新窗口状态
    while 窗口不满足条件:
        # 将 arr[left] 移出窗口
        left += 1
    # 此时窗口满足条件，更新答案
    # answer = max(answer, right - left + 1)

这个模板看着简单，但实际用起来要小心：什么时候用 while 收缩，什么时候用 if 收缩；窗口的合法性判断条件是什么——这些需要根据具体题目来调整。

字符串基础

字符串在 Python 中是不可变的——每次拼接、替换都会创建新的字符串对象。这在面试中是一个常见的陷阱：

s = ""
for i in range(10000):
    s += str(i)   # 每次创建新字符串，O(n²)！

# 正确做法：用列表收集，最后 join
parts = []
for i in range(10000):
    parts.append(str(i))
s = "".join(parts)   # O(n)

关于字符串匹配，有一个大名鼎鼎的算法叫 KMP (Knuth-Morris-Pratt)，它能在 O(n+m) 时间内完成模式匹配。KMP 是一个非常精巧的算法，但我们不会在这一章展开，因为它的思想和实现都需要更多铺垫。我们将在第 21 章字典树 (Trie) 中一并讨论字符串匹配的各类算法。

数组 vs 链表：操作复杂度对比

操作 数组 链表 胜出方
随机访问 O(1) O(n) 数组
头部插入 O(n) O(1) 链表
尾部插入 O(1)* O(1)** 平手
中间插入 O(n) O(1)*** 链表
按值查找 O(n) O(n) 平手
内存占用 紧凑，无额外开销 每个元素需额外存指针 数组
缓存友好性 高（连续内存） 低（离散存储） 数组

* 动态数组均摊 O(1)；** 需维护尾指针；*** 假设已持有插入位置的节点引用

⚠️ 常见陷阱：数组索引越界
数组的 off-by-one 错误（差一错误）是新手最常犯的错误之一。比如想遍历数组的最后 k 个元素，很多人会写 for i in range(n - k, n)，但如果 k 可能为 0，这就变成了 range(n, n)——空循环，没问题。但如果 k 大于 n，就会得到负索引！在 Python 中负索引不报错，但结果完全不是你想要的。一个安全的写法是 for i in range(max(0, n - k), n)。永远记住：边界检查不是可选操作，是必须操作。

本章小结

数组是最基础的数据结构，但它衍生出的技巧——前缀和、双指针、滑动窗口——构成了算法面试的半壁江山。掌握这些技巧的关键不是背模板，而是理解它们背后的思想：前缀和用预处理换查询速度，双指针利用有序性减少遍历次数，滑动窗口把嵌套循环变成单层循环。

下一章，我们将学习一种和数组"对立"的数据结构——链表。数组的强项（随机访问）是链表的弱项，数组的弱项（插入删除）却是链表的强项。理解了这种对立互补的关系，你就能在合适的场景选择合适的结构。
