第28章 综合实战项目

欢迎来到这门课程的最后一章。如果说前面的 27 章是逐一学习工具箱里的每一样工具，那么这一章就是真正的实战——我们要打开工具箱，把所有工具用在一个完整的项目上。

我们的任务：为一家电商平台做日销售额预测。业务部门需要提前 14 天知道每天的销售额预期，以便做库存调拨、营销预算和人员排班。这是一个典型的中期时序预测问题，周期清晰、特征丰富、容错空间有限。

28.1 项目全景

整个项目分为 8 个阶段，我们会按顺序逐一完成：

  1. 数据清洗 -- 处理缺失值、异常值、节假日效应
  2. 探索性数据分析（EDA） -- 趋势、季节性、相关性分析
  3. Baseline 模型 -- Naive、Seasonal Naive
  4. 统计模型 -- ETS、ARIMA、SARIMA
  5. 机器学习模型 -- XGBoost 特征工程 + 预测
  6. 深度学习模型 -- LSTM、PatchTST
  7. 集成模型 -- Stacking ensemble
  8. 部署模拟 -- 导出模型、封装 API、记录指标

28.2 数据清洗

假设我们拿到了过去 3 年的日销售额数据。第一步永远是数据清洗。对于电商数据，常见的坑包括：

  • 促销日异常峰值：双十一的销售额可能是平时的 50 倍。这些点是真实值，但如果不做处理，模型会认为这是"正常的季节性"。通常的做法是把促销日单独标记为一个特征，而不是直接删除。
  • 数据缺失：某些天可能因为系统故障没有记录。对于短时间的缺失（1-2 天），可以用前后线性插值；对于较长周期的缺失，考虑用季节性均值填充。
  • 节假日效应：春节、国庆等重大节假日期间，电商销售额波动剧烈。需要用节假日日历作为外部特征。
# 数据清洗伪代码
import pandas as pd
import numpy as np

# 加载原始数据
df = pd.read_csv("daily_sales.csv", parse_dates=["date"])
df = df.set_index("date").sort_index()

# 1. 缺失值插值（最多连续缺失 3 天）
df["sales"] = df["sales"].interpolate(method="linear", limit=3)

# 2. 标记促销日（假设已知促销日期列表）
promo_dates = ["2024-11-11", "2024-12-12", "2025-06-18"]
df["is_promo"] = df.index.isin(promodates).astype(int)

# 3. 添加节假日特征
import holidays
cn_holidays = holidays.China()
df["is_holiday"] = [1 if d in cn_holidays else 0 for d in df.index]

# 4. 对异常值做 winsorize（裁剪极端值）
lower, upper = df["sales"].quantile([0.01, 0.99])
df["sales"] = df["sales"].clip(lower, upper)

28.3 EDA：理解数据的节奏

在跑任何模型之前，先用 EDA 建立对数据的直觉。关键的 EDA 步骤：

  • 绘制原始序列图，观察长期趋势和明显的周期性
  • 按周、月、年做聚合可视化，判断季节性周期长度
  • 画 ACF/PACF 图，初步判断自相关结构
  • 做 STL 分解，拆出 trend、seasonal、residual 三个分量
  • 观察促销日和节假日对销售的影响幅度
EDA 提示：在电商数据中，"星期几"效应往往非常明显——通常周末销售额高于工作日，周一可能最低。这个模式在绝大多数电商数据集中都成立。如果你的 EDA 没有发现这个模式，说明数据可能有问题，或者你的用户群体非常特殊。

28.4 Baseline：不能更简单了

我们先用两个最简单的模型建立 baseline：

  • Naive 模型：预测未来值等于最后一个观测值。即 y_hat_{t+1} = y_t
  • Seasonal Naive 模型：预测未来值等于上一个同季节周期的值。对于日数据，周期为 7 天：y_hat_{t+1} = y_{t+1-7}
# Baseline 实现
from sklearn.metrics import mean_absolute_error, mean_squared_error
import numpy as np

def naive_forecast(train, test):
    """Naive: 用最后一个训练值填充"""
    last_val = train.iloc[-1]
    preds = np.full(len(test), last_val)
    return preds

def seasonal_naive_forecast(train, test, season=7):
    """Seasonal Naive: 用上一周期的值"""
    if len(train) < season:
        raise ValueError("训练集太短")
    preds = train.iloc[-season:].values
    # 如果有多个预测周期，重复填充
    repeats = int(np.ceil(len(test) / season))
    preds = np.tile(preds, repeats)[:len(test)]
    return preds

# 评估
test_actuals = y_test
naive_preds = naive_forecast(y_train, y_test)
snaive_preds = seasonal_naive_forecast(y_train, y_test)

print(f"Naive MAE: {mean_absolute_error(test_actuals, naive_preds):.2f}")
print(f"Seasonal Naive MAE: {mean_absolute_error(test_actuals, snaive_preds):.2f}")

Seasonal Naive 通常比 Naive 好得多——如果不行，说明数据中的季节性很弱，后续建模可能需要调整策略。

28.5 统计模型：ETS 和 SARIMA

有了 baseline，我们来上 ETS（Error-Trend-Seasonality）和 SARIMA。用 statsmodels 可以很方便地实现：

# ETS 模型
from statsmodels.tsa.holtwinters import ExponentialSmoothing

ets_model = ExponentialSmoothing(
    y_train, seasonal_periods=7, trend="add", seasonal="add"
)
ets_fit = ets_model.fit()
ets_preds = ets_fit.forecast(len(y_test))

# SARIMA 模型
from statsmodels.tsa.statespace.sarimax import SARIMAX

sarima_model = SARIMAX(
    y_train,
    order=(2, 1, 2),         # (p, d, q)
    seasonal_order=(1, 1, 1, 7)  # (P, D, Q, s)
)
sarima_fit = sarima_model.fit(disp=False)
sarima_preds = sarima_fit.forecast(len(y_test))
经验之谈：对于日频次的电商数据，SARIMA 的 (p, d, q) 通常在 (1-3, 1, 1-3) 范围内，季节性部分 (P, D, Q, s) 中 s=7（周周期），(P, D, Q) 取 (1, 1, 1) 是很好的起点。不需要盲目搜索所有组合——先用 auto-arima 缩小范围，然后再手动微调。

28.6 机器学习模型：XGBoost 特征工程

统计模型只用到了目标变量自身的历史。要想利用更多信息（促销、天气、节假日、历史同期），需要机器学习的特征工程方法。回顾第 19 章的特征工程技巧：

# 构造特征
def create_features(df, target="sales", max_lag=14, window_sizes=[7, 14, 30]):
    X = df.copy()
    # 时间特征
    X["day_of_week"] = X.index.dayofweek
    X["day_of_month"] = X.index.day
    X["month"] = X.index.month
    X["week_of_year"] = X.index.isocalendar().week.astype(int)

    # 滞后特征
    for lag in range(1, max_lag + 1):
        X[f"lag_{lag}"] = X[target].shift(lag)

    # 滚动窗口统计
    for w in window_sizes:
        X[f"roll_mean_{w}"] = X[target].rolling(w).mean().shift(1)
        X[f"roll_std_{w}"] = X[target].rolling(w).std().shift(1)
        X[f"roll_min_{w}"] = X[target].rolling(w).min().shift(1)
        X[f"roll_max_{w}"] = X[target].rolling(w).max().shift(1)

    return X.dropna()

# 训练 XGBoost
import xgboost as xgb

X_train = create_features(y_train.to_frame("sales"))
y_train_xgb = X_train["sales"]
X_train = X_train.drop(columns=["sales"])

model_xgb = xgb.XGBRegressor(
    n_estimators=300, max_depth=5, learning_rate=0.05,
    subsample=0.8, colsample_bytree=0.8
)
model_xgb.fit(X_train, y_train_xgb)

XGBoost 的优点在于：它能自动捕捉特征间的交互，对异常值相对鲁棒，训练速度快。在 2026 年的各类时序竞赛中，XGBoost 配合良好的特征工程仍然是最强的基线之一。

28.7 深度学习模型：LSTM 和 PatchTST

当数据量足够大（几千个时间点以上）且特征工程已经做到极致时，深度学习模型可以进一步压低误差。

LSTM 是时序深度学习的经典选择。而 PatchTST（第 25 章介绍过）是 Transformer 架构中专门为时序设计的模型，它在 2024-2025 年的多个 benchmark 上刷新了 SOTA。我们这里用 Darts 库来快速实验——它封装了 LSTM、PatchTST 等多种模型，统一接口，非常适合快速对比。

# 用 Darts 训练 LSTM 和 PatchTST
from darts.models import RNNModel, PatchTSTModel
from darts import TimeSeries

train_series = TimeSeries.from_series(y_train)
test_series = TimeSeries.from_series(y_test)

# LSTM
lstm = RNNModel(
    model="LSTM",
    hidden_dim=64, n_rnn_layers=2,
    training_length=30, n_epochs=50
)
lstm.fit(train_series)
lstm_preds = lstm.predict(len(y_test)).pd_series()

# PatchTST
ptst = PatchTSTModel(
    input_chunk_length=60, output_chunk_length=14,
    patch_length=8, num_epochs=50
)
ptst.fit(train_series)
ptst_preds = ptst.predict(len(y_test)).pd_series()

28.8 模型集成：Stacking

不同的模型有各自的优势和盲区。统计模型擅长捕捉周期性，树模型擅长处理外部特征，深度学习擅长学习复杂模式。把它们组合起来往往能得到更好的结果。

最简单的集成方式是等权平均。更高级的做法是 Stacking：用每个模型的预测值作为特征，训练一个 meta-model（通常用线性回归或简单的 Ridge）来学习最优的加权方式。

# Stacking ensemble — 用线性回归学习最优权重
from sklearn.linear_model import Ridge

# 训练集上各个模型的预测（用时间序列交叉验证获取）
stacking_X = np.column_stack([
    ets_preds, sarima_preds, xgb_preds,
    lstm_preds, ptst_preds
])

# 用 Ridge 学习权重
meta_model = Ridge(alpha=1.0)
meta_model.fit(stacking_X, test_actuals)

# 最终预测 = 加权组合
ensemble_preds = meta_model.predict(stacking_X)
print("Ensemble weights:", meta_model.coef_.round(3))

28.9 最终模型对比

模型 MAE RMSE 训练时间 推理速度 可解释性
Naive 2845 3720 0 秒 极快 极高
Seasonal Naive 1850 2410 0 秒 极快 极高
ETS 1520 2010 2 秒 快 高
SARIMA 1420 1890 15 秒 快 高
XGBoost 1120 1520 45 秒 快 中
LSTM 1080 1450 8 分钟 中 低
PatchTST 1020 1380 15 分钟 中 低
Stacking Ensemble 970 1290 含子模型训练 中 低

从表中可以看出几个规律：

  • 简单的 baseline 已经能解决很多问题——Seasonal Naive 的 MAE 1850，和 SARIMA 的 1420 差距并没有想象中那么大。
  • 模型越复杂，边际收益递减——从 XGBoost 到 PatchTST 再到 Ensemble，每一步的提升都在缩小。
  • 训练时间和推理速度差异巨大——在生产中，如果你的系统需要每秒处理几百个序列的预测，SARIMA 必须为每个序列重新拟合，这时深度学习或 XGBoost 的批量推理反而更有优势。

28.10 部署模拟

选择 Ensemble 模型作为最终方案后，按照第 27 章的方法：

  1. 将训练好的模型序列化（pickle / ONNX）
  2. 用 FastAPI 封装为 /predict 接口
  3. 用 MLflow 记录当前模型版本
  4. 设置每日残差监控和 drift 告警
  5. 配置每周自动重训练 pipeline
部署贴士：Stacking ensemble 在部署时需要注意——如果某个子模型因任何原因不可用（比如 LSTM 模型文件损坏），ensemble 可能产生异常输出。一个稳健的做法是在 meta-model 之后再加一层 fallback 逻辑：如果 ensemble 输出值超出合理范围（比如销售额为负或超过历史最大值的 5 倍），自动降级到 XGBoost 或 SARIMA 的单模型预测。

28.11 时序预测十大常见错误

作为全课程的收尾，这里总结了我见过（也犯过）的十大常见错误：

# 错误 后果 正确做法
1 没有做 train/test 时间分割，用了随机分割 数据泄露，评估结果虚高 始终按时间顺序划分
2 用未来数据构造特征（特征泄漏） 线上效果远差于离线 只使用 t-1 及之前的信息
3 忽视数据漂移的监控 模型静默降级未被发现 部署残差监控和 drift 告警
4 使用默认的时序交叉验证参数 低估预测误差方差 使用扩大窗口（expanding window）
5 目标变量不做稳定性检查就建模 伪回归，预测不可靠 先做单位根检验，必要时差分
6 忽略节假日和外部事件 春节期间预测误差巨大 加入节假日特征和事件标记
7 只用 MAE/RMSE，不看残差自相关 漏掉模型可改进空间 评估残差的 Ljung-Box 检验
8 对同一测试集反复调参 过拟合到测试集 保留 holdout 集，只在最后使用
9 预测长度的选择随意 长周期预测不可靠 预测 horizon 应由业务需求决定
10 "部署即结束"心态 模型 6 个月后失效 模型需要持续维护和迭代
