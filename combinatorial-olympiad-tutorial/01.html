第一章：计数原理与排列

大家好，我是你们的讲师。今天正式开始学习组合数学——这门学科的核心问题概括起来就一句话："有多少种方法？"。听起来简单，但别小看它。从安排一场比赛的赛程，到计算密码的破解难度，再到分析基因序列的排列方式，组合数学几乎无处不在。

很多同学觉得组合数学就是"套公式"，背下 P(n,k) 和 C(n,k) 的公式就能做题了。但参加过竞赛的都知道，这是最大的误解。真正的难点不在公式本身，而在于判断什么时候用加法、什么时候用乘法、什么时候要排除重复。今天这一章，我们就从最根基的计数原理开始，把这些"直觉"真正建立起来。

1.1 加法原理与乘法原理

这两个原理看起来简单到不值一提，但它们是一切组合计数的地基。如果你只能记住组合数学中的两件事，那就是这两个原理。

1.1.1 加法原理（分类计数）

加法原理：如果完成一件事有 k 类互斥的方式，第 i 类方式中有 ni 种具体方法，且这些方法互不重叠，那么完成这件事总共有 n1 + n2 + ... + nk 种方法。

关键在"分类"二字——各类之间必须没有交集。用集合的语言说，就是各集合的并集大小为各集合大小之和，当且仅当这些集合两两不交。

举个简单的例子：从北京到上海，可以坐高铁（有30趟），可以坐飞机（有15班），也可以坐大巴（有10趟）。那么从北京到上海的总选择数是 30+15+10 = 55 种。

但在竞赛题中，加法原理的"分类"往往藏得更深。来看一个经典问题：

例1：用数字 1、2、3、4、5 可以组成多少个小于 1000 的正整数？（数字不允许重复使用）

分析：小于1000的正整数可以是一位数、两位数或三位数。这就构成了一个自然的分类框架：

  • 一位数：从5个数字中选1个，有5种
  • 两位数：十位5种选择 × 个位4种选择 = 20种
  • 三位数：百位5种 × 十位4种 × 个位3种 = 60种

总数 = 5 + 20 + 60 = 85 个。

分类时最容易犯的错误就是分类不互斥——类与类之间有重叠，导致重复计数。后面我们会专门讨论这个问题。

1.1.2 乘法原理（分步计数）

乘法原理：如果完成一件事需要 k 个步骤，第 i 个步骤有 ni 种方法，且各步骤之间相互独立、互不影响，那么完成这件事总共有 n1 × n2 × ... × nk 种方法。

关键在"分步"——每一步的选择不能受之前步骤的影响而改变数量（至少不能减少到零）。这也是乘法原理在实际应用中最大的陷阱。

还是先举简单的：食堂有3种主食、4种副食、2种汤。点一份套餐（主食+副食+汤）有多少种搭配？3×4×2 = 24 种。

乘法原理的威力在竞赛题中会变得非常惊人。来看一个需要巧妙分步的例子：

例2（染色问题）：用4种不同的颜色给下图中的A、B、C、D四个区域染色，要求相邻区域颜色不同。有多少种染法？

假设四个区域的相邻关系为：A与B、C相邻；B与A、C、D相邻；C与A、B、D相邻；D与B、C相邻。

解法：按 A → B → D → C 的顺序染色：

  • A有4种选择
  • B与A相邻，不能与A同色，有3种选择
  • D与B相邻，不能与B同色，有3种选择
  • C与A、B、D都相邻——需要分情况讨论。

这告诉我们：乘法原理中的"步骤顺序"不是随便定的。步骤选得好，问题就简化；步骤选得不好，就需要分类讨论。在竞赛中，选择合适的"分步顺序"本身就是一项重要技能。

1.1.3 加法 vs 乘法：如何区分？

这是初学者最常问的问题。有一个简单的判断标准：

特征 加法原理 乘法原理
逻辑关系 "或者"——任选一类即可 "并且"——每步都要做
关键词 分类、互斥、不同情况 分步、依次、连续步骤
运算 加法 乘法
检验标准 各类之间会"重叠"吗？ 各步之间会"干扰"吗？
典型场景 满足某条件的数有多少 排列、计数路径

如果你还是不确定，问自己一个问题：做这件事的时候，我是"选择其中一条路走"（加法），还是"一步一步走完所有路"（乘法）？

⚠️ 踩坑实录：有一年我带竞赛集训队，出了一个题："从A地到B地有3条路，从B地到C地有4条路，从A地直接到C地有2条路。从A到C有多少种走法？"结果超过一半的学生写了 3×4×2 = 24。错在哪里？因为"从A直接到C"和"从A经B到C"是两类路线，应该用加法：3×4 + 2 = 14。学生把"加法场景"当成了"乘法场景"——这是一个非常经典的加法乘法混淆。永远记住：先判断关系，再选择运算。

1.2 排列数公式的推导

理解了加法原理和乘法原理，排列数的公式就水到渠成了。

1.2.1 从 n 个不同元素中取 k 个的排列

排列数 P(n,k)（也记作 A(n,k) 或 nPk）表示从 n 个不同的元素中取出 k 个（k ≤ n），按顺序排成一列，所有不同排法的总数。

推导过程非常简单——就是乘法原理的一次应用：

  • 选第1个位置：有 n 种选择
  • 选第2个位置：剩下 n-1 种选择
  • 选第3个位置：剩下 n-2 种选择
  • ……
  • 选第 k 个位置：剩下 n-(k-1) = n-k+1 种选择

因此：

P(n,k) = n × (n-1) × (n-2) × ... × (n-k+1)

用阶乘表示为：

P(n,k) = n! / (n-k)!

这里 n!（读作"n的阶乘"）表示从1乘到n：n! = 1×2×3×...×n。并规定 0! = 1，这个规定在后面会看到它的合理性。

当 k = n 时，就是全排列：

P(n,n) = n × (n-1) × ... × 2 × 1 = n!

n 个人排成一排，有 n! 种排法。这个结论虽然简单，但 n! 的增长速度非常惊人：10! = 3,628,800，15! 就已经超过 1.3 万亿了。这就是为什么说"暴力枚举"在组合问题中几乎永远不可行。

1.2.2 排列数的几个重要变形

竞赛中常常不会直接给你 n 和 k 让你套公式，而是需要你灵活变形。以下三个变形最常出现：

变形一：P(n,k) = n × P(n-1, k-1)

这个式子的直观理解是：先从 n 个中选一个放在第一个位置（n 种方法），然后从剩下的 n-1 个中选 k-1 个排列在后续位置（P(n-1,k-1) 种方法）。这个递推关系在证明组合恒等式时非常有用。

变形二：P(n,k) = (n-k+1) × P(n, k-1)

先排列 k-1 个，再在最后一位从剩下的 n-(k-1) 个中选一个。

变形三：P(n,k) = n! / (n-k)!

这个最简洁的形式也最便于代数推导。很多竞赛题最终归结为处理阶乘的比值。

关键洞察：排列数的本质是"有序选取"——顺序不同就算不同的结果。"排列"和后面要讲的"组合"的唯一区别就是：排列考虑顺序，组合不考虑顺序。如果你在做题时犹豫"要不要考虑顺序"，问自己一句话：交换两个元素，结果是不同的吗？如果是，用排列数；如果不是，用组合数——那是第二章的内容。

1.3 全排列与阶乘

全排列 n! 是排列中最特殊也最常见的情形。我们通过几个例子来感受它的应用。

1.3.1 基本全排列

例3：将7本不同的书放在书架上，有多少种放法？

很简单，就是 7! = 5040 种。

例4：7本不同的书，其中3本是数学书、2本是物理书、2本是化学书。要求同科目的书必须相邻，有多少种放法？

这需要两步：先把3类科目看作3个"大块"，3个大块全排列有 3! = 6 种；然后在每个大块内部，数学书之间有 3! = 6 种排列，物理书之间有 2! = 2 种，化学书之间有 2! = 2 种。总数为 6 × 6 × 2 × 2 = 144 种。

1.3.2 有重复元素的排列

上面的例子中所有书都是不同的。但如果书的类型相同（比如3本数学书是一样的），情况就不同了。

一般来说，如果 n 个元素中，有 n1 个相同、n2 个相同、……、nk 个相同（n1+n2+...+nk = n），那么这 n 个元素的全排列数为：

n! / (n₁! × n₂! × ... × nₖ!)

道理很简单：如果所有元素都不相同，有 n! 种排列。但其中有 n1! 种排列实际上是把那 n1 个相同元素互换产生的——这些在问题中应视为同一种排列，所以要除以 n1!。

例5：用字母 A、A、B、B、C 可以组成多少个不同的5位字符串？

总数 = 5! / (2! × 2! × 1!) = 120 / (2×2) = 30 个。

一点思考：有重复元素的排列公式 n! / (n₁! × n₂! × ...) 实际上就是多项式系数的一种特殊情况。等你学到第三章的二项式定理时会发现，这些公式之间有着深刻的联系。组合数学的美妙之处就在于——各个知识点像一张网，互相印证、彼此关联。

1.4 圆排列

前面讨论的所有排列都是"线排列"——元素排成一条直线。但竞赛中还经常遇到"圆排列"的问题：元素排成一个圆环，旋转后相同的视为同一种排列。

n 个不同元素的圆排列数为 (n-1)!

为什么？想象一下，把 n 个人安排在圆形餐桌旁。我们把圆形餐桌的某个位置"固定"为参考点（比如正对门的位置），然后对剩下的 n-1 个人进行线排列——这样就能消除旋转带来的重复。因为将圆旋转一个位置后，每个人相对于参考点的位置就变了，所以需要固定一个参考点。

更严谨的推导：n 个元素的线排列有 n! 种。每种圆排列对应 n 种线排列（因为从 n 个不同的"缺口"处剪开会得到 n 种不同的线性序列）。所以圆排列数为 n! / n = (n-1)!。

例6：8位同学围坐一圈，有多少种坐法？

如果只考虑相对位置（旋转视为相同），有 (8-1)! = 5040 种。

如果还考虑镜子反射（翻转后视为相同），即"手环排列"，则还需要除以 2，得到 5040/2 = 2520 种。

类型 公式 示例（n=5） 说明
线排列 n! 120 排成一排
圆排列（旋转等同） (n-1)! 24 围坐圆桌
手环排列（旋转+翻转等同） (n-1)! / 2 12 项链、手环

⚠️ 常见错误：题目说"围坐一圈"，很多学生想都不想就除以 n。但请仔细看题——如果椅子是不同的（比如有编号），那就不需要除以 n，因为旋转后坐在不同椅子上的情况被视为不同的。反之，如果椅子没有编号，所有人都一模一样的圆桌，那才需要用 (n-1)!。我的一个学生曾在联赛中因此丢了15分——题目里就多了"编号"两个字，他硬是没看见。

1.5 常见计数陷阱

学习组合数学的过程中，"踩坑"本身就是最好的学习方式。下面这两个陷阱几乎每个竞赛生都栽过。

1.5.1 重复计数

重复计数是最常见的错误，它发生在你用不同方式去数同一组对象，然后不自觉地相加了。来看一个经典案例：

错误例：从6个人中选一个正组长、一个副组长和一个普通组员，有多少种选法？

有学生这么算：先选组长（6种），再选副组长（5种），再选组员（4种），总共 6×5×4 = 120 种。然后想，如果用排列数公式呢？P(6,3) = 6×5×4 = 120。看起来一致，好像没错。

但等一下——组长的角色和副组长的角色是不同的，而组员和组长也是不同的（谁不想当组长呢）。所以这里实际上是从6个人中选3个人并分别赋予3种不同的角色，确实就是排列数。没问题。

但如果题目改成"选3个人组成一个小组，其中1人担任组长"，这时我们选组长（6种），选剩下2人作为普通组员（C(5,2) 或 5×4/2 种）。如果用乘法 6 × C(5,2) = 60，或者你先选3人再选组长 C(6,3) × 3 = 60，都是对的。但有人会写成 6×5×4 = 120——这里的重复在于，当组长是A、组员是B和C时，和组长是A、组员是C和B实际上是同一回事，但6×5×4把它们算成了不同情况。

检验方法：用两种不同的方法去数同一个东西，如果结果不一致，说明至少其中一种方法有重复或遗漏。

1.5.2 遗漏计数

遗漏计数和重复计数相反——你把一些情况漏掉了。这通常源于分类不完整。

例7（一个真实的踩坑故事）：有一次我在给集训队出模拟题，题目是这样的："从1到100中选两个不同的数，使得它们的和是3的倍数，有多少种选法？"

一个平时成绩不错的同学这样分析：两个数的和是3的倍数，要么两个数都是3的倍数，要么一个模3余1一个模3余2。1到100中：3的倍数有33个，模3余1的有34个（1,4,...,100），模3余2的有33个（2,5,...,98）。

他的答案：C(33,2) + 34×33 = 528 + 1122 = 1650 种。

看起来合理对吧？但答案是错的。他漏掉了什么？

他的遗漏：他自己也检查了两遍没找到问题。我提示他："两个数都是3的倍数——你还记得100除以3余几吗？"他一拍脑门——100除以3余1，所以模3余1的有34个，他确实写了34。但C(33,2)中的33是3的倍数的个数，他手算33时完全正确，问题是——他分类时只列出了"两个都是3的倍数"和"一个余1一个余2"，他认为"一个余0一个余0"已经覆盖了第一种情况。但是，两个数如果都是模3余0（即3的倍数），和也是3的倍数，对吗？没问题。他选的33个中取2个，C(33,2)也没错。

那错在哪里？我让他再读一遍题："从1到100中选两个不同的数"。他的分类完全正确，但他写的C(33,2)中的33——100以内3的倍数有33个吗？3×33=99，所以1到100中3的倍数确实有33个：3,6,...,99。但这是整数除法，他的C(33,2)没错。

最终我发现——他没错！是我这个"出题人"搞错了。但这个故事告诉我们一个宝贵的教训：当你的答案和标准答案不一致时，先用另一种方法重新计算一遍，不要急于认定自己错了。在竞赛考场上，如果你有时间，用两种独立的方法计算同一个结果——如果一致，正确率可以接近100%。

1.5.3 一个系统性的防错方法

在多年的教学和竞赛实践中，我总结了一套防错的"三步检查法"：

  1. 第一步：确定是加法还是乘法。 先问自己——这些情况是互斥的吗？这些步骤是独立的吗？
  2. 第二步：用另一种方法重新计算。 如果有两种不同的"分类"或"分步"方式都能得到结果，分别算，对比是否一致。不一致则必有重复或遗漏。
  3. 第三步：用小规模验证。 如果 n=3 或 n=4 的情况可以直接枚举出来，先手动枚举验证公式是否正确，再推广到一般情况。

这个"三步法"不能保证你100%不出错，但可以筛掉80%以上的低级错误。

1.6 本章小结与过渡

今天我们从最基础的加法原理和乘法原理出发，一步步推导出了排列数的公式，并讨论了全排列、有重复排列、圆排列这些重要的变体。最后我们通过几个"踩坑"故事，了解了计数中最常见的两类错误——重复计数和遗漏计数。

这章的内容是整个组合计数的地基。后面的每一章都会在此基础上展开：下一章我们要讲组合数与恒等式——排列数考虑顺序，组合数不考虑顺序。你将会看到，理解了排列之后，组合数只是一个"除以重复"的概念，而且组合恒等式的世界远比看起来要丰富得多。准备好了吗？我们第二章见。
