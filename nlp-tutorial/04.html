第四章：文本表示

大家好！这一章我们来解决NLP中的一个根本问题：怎么把文本变成计算机能理解的数字？计算机只认数字，而文本是离散符号——这个"翻译"过程就是文本表示（Text Representation）。

我面试过不少候选人，问"TF-IDF和Word2Vec有什么区别"，很多人答不上来本质。其实这个问题是理解NLP技术演进的绝佳透镜：TF-IDF把文档表示为向量，Word2Vec把词表示为向量。一个是文档级，一个是词级。向量表示的质量，直接决定了下游模型的天花板。

4.1 词袋模型（Bag of Words）

词袋模型是最简单的文本表示方法。把文档看作一个装满词的"袋子"，忽略词序，只统计每个词出现的次数：

# 词袋模型示例
from sklearn.feature_extraction.text import CountVectorizer

corpus = [
    "我爱自然语言处理",
    "自然语言处理很有趣",
    "深度学习改变自然语言处理"
]

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(corpus)
print(vectorizer.get_feature_names_out())
# ['改变', '有趣', '深度', '爱', '自然语言处理', '学习']

print(X.toarray())
# 文档1: [0, 0, 0, 1, 1, 0]  → "爱"×1, "自然语言处理"×1
# 文档2: [0, 1, 0, 0, 1, 0]  → "有趣"×1, "自然语言处理"×1
# 文档3: [1, 0, 1, 0, 1, 1]  → "改变"×1, "深度"×1, "自然语言处理"×1, "学习"×1

词袋模型的优缺点非常明显：

优点缺点
简单、快速、可解释丢失了词序信息
配合朴素贝叶斯效果好高频词主导，低频词被淹没
适合主题分类等任务稀疏向量（大部分维度是0），维度=词汇表大小
作为baseline非常靠谱无法捕捉语义相似性

实战建议：做任何文本分类任务，先用词袋模型+朴素贝叶斯跑一个baseline。5分钟就能出结果，给你一个后续优化的参照线。别一上来就BERT，浪费GPU还搞不清提升来自哪里。

4.2 TF-IDF：给词加权

词袋模型的最大问题是：所有词一视同仁。但"的"和"深度学习"的重要性显然不同。TF-IDF（Term Frequency - Inverse Document Frequency）给每个词加权，让在当前文档中出现多、但在其他文档中出现少的词获得高分。

# TF-IDF 计算公式
# TF(t,d) = 词t在文档d中出现的次数 / 文档d的总词数
# IDF(t) = log(文档总数 / 包含词t的文档数) + 1
# TF-IDF(t,d) = TF(t,d) × IDF(t)

# 直觉：
# "的" → TF很高，但IDF很低（几乎所有文档都有） → TF-IDF低 ✓
# "BERT" → TF在本文档高，IDF高（只有少数文档出现） → TF-IDF高 ✓

from sklearn.feature_extraction.text import TfidfVectorizer

tfidf = TfidfVectorizer(max_features=5000)  # 限制特征维度
X = tfidf.fit_transform(corpus)
# 现在每篇文档的向量中，每个维度不再是计数，而是TF-IDF权重

TF-IDF在信息检索、关键词提取、文档相似度等任务上至今是强有力的baseline。即使到了大模型时代，Elasticsearch的BM25算法（TF-IDF的改进版）仍然是搜索引擎的标配。

4.3 One-Hot编码与稀疏表示的困境

有了词袋模型，每篇文档就是一个向量。但这个向量有几个致命问题：

# One-Hot 表示
# 词汇表 V = {"我":0, "爱":1, "NLP":2, "你":3}
# "爱" → [0, 1, 0, 0]
# "NLP" → [0, 0, 1, 0]

# 问题1：维度灾难
# 词汇表10万 → 每个词/文档是10万维向量，99.99%是0

# 问题2：语义鸿沟
cosine_similarity("猫", "狗")  # → 0.0 （One-Hot下任何两个不同词都正交）
cosine_similarity("猫", "电脑") # → 0.0 （和上一对一样，但语义上"猫"和"狗"更接近）

核心问题：One-Hot表示的"猫"和"狗"之间的距离，跟"猫"和"电脑"是一样的。这完全丢失了语义相似性。解决这个问题，正是下一章词向量（Word Embedding）的核心动机——把高维稀疏的One-Hot向量，映射到低维稠密的语义空间。

4.4 从稀疏到稠密：分布式表示的思想

分布式表示（Distributed Representation）的思想来自语言学中的"分布假说"：一个词的含义由它周围的词决定。"You shall know a word by the company it keeps"（Firth, 1957）——这句话写于近70年前，但到今天依然是NLP最核心的哲学。

在计算机里，分布式表示就是一个稠密的低维向量（通常50-300维），每个维度不直接对应某个具体语义，而是所有维度共同编码了词的语义。我常跟学生打比方：One-Hot是一个大房间里给你分配一把专门的椅子，分布式表示是一群演员合力演一个角色——单个演员不是这个角色，但合在一起就是了。

# One-Hot (稀疏) → Embedding (稠密)
# "king"   → [0, 0, 1, 0, 0, ...] (50000维)
# "king"   → [0.23, -0.17, 0.45, ..., 0.02] (300维)

# 稠密向量的优势：
# 1. 低维：300 << 50000
# 2. 语义邻近：cos("king","queen") ≈ 0.8, cos("king","table") ≈ 0.1
# 3. 可计算：king - man + woman ≈ queen

分布式表示是现代NLP的根基。下一章，我们就来详细展开词向量的学习和应用——这也是深度学习在NLP领域的第一个重大突破。
