第2章 图论基础

大家好，欢迎回到网络科学课程！上一章我们了解了网络科学的基本概念和发展历史，从欧拉的七桥问题到BA模型的无标度发现。今天，我们将深入网络科学的数学根基——图论（Graph Theory）。

如果你觉得"图论"听起来很抽象，不用担心。事实上，图论的思想框架与我们的日常直觉非常接近：我们把事物抽象为"点"，把事物之间的关系抽象为"线"。一张社交关系图、一张地铁线路图、一张蛋白质相互作用图——它们本质上都是"图"。掌握了图论的语言，你就拥有了描述任何网络系统的数学工具。

从哥尼斯堡七桥到现代图论

我们在第一章提到过1736年欧拉解决哥尼斯堡七桥问题的故事。但这个故事中有一个经常被忽略的关键点：欧拉在证明过程中所做的抽象（abstraction）本身，就是图论最核心的思想。

让我们复盘一下欧拉的思路。哥尼斯堡的居民关心的实际问题是：能否从某处出发，恰好走过七座桥各一次，回到起点？这个问题涉及大量细节——桥的长度、宽度、材质，河流的流速，步行时间等等。欧拉的第一个洞见是：这些细节都不重要。唯一重要的是：四块陆地（两个河岸和两个岛屿）之间如何通过桥梁相互连接。

于是，欧拉画出了历史上第一张"图"：四个点代表四块陆地，七条线代表七座桥。这个模型忽略了所有物理细节，只保留了连接关系。这个看似简单的动作，开启了数学的一个全新分支。

核心洞察：图论的本质就是"抛开细节，保留结构"。无论你研究的是社交网络、交通网络还是神经网络，图论的抽象方法都是一样的：找到系统中的"节点"和"边"，然后分析它们的连接模式。这种能力本身，就是网络科学最重要的思维方式。

欧拉的结论——当且仅当图中所有顶点的度数都是偶数时，存在经过每条边恰好一次并回到起点的回路（即欧拉回路）——至今仍是图论中的基本定理。而这条定理的应用远超桥的问题本身：DNA测序中的片段拼接、邮递员的路线规划、电路板的走线优化——这些现代问题都可以追溯到欧拉在1736年做出的那个抽象。

图的基本概念

现在，让我们正式定义图论中的基本术语。这些术语看似简单，但它们是后续所有分析的基础。

顶点（Vertex/Node）是图中的基本单元，通常用圆点表示。在社交网络中，顶点代表一个人；在万维网中，顶点代表一个网页。在数学表示中，顶点集合通常记为 V = {v₁, v₂, ..., vₙ}，其中 n = |V| 是顶点数。

边（Edge/Link）是两个顶点之间的连接，通常用线段表示。边的集合记为 E，其中每条边是顶点的一个无序对（无向图）或有序对（有向图）。边的数量记为 m = |E|。

度（Degree）是一个顶点的邻居数量，即与该顶点相连的边的数量。度为0的顶点称为孤立点（isolated vertex）。在有向图中，度分为入度（in-degree，指向该顶点的边数）和出度（out-degree，从该顶点出发的边数）。

此外，还有几个重要概念需要区分：

  • 简单图（Simple Graph）：没有平行边（两点之间最多一条边）且没有自环（顶点不与自身相连）的图。绝大多数网络科学研究都基于简单图。
  • 多重图（Multigraph）：允许两个顶点之间存在多条边（平行边）。例如，两个城市之间可能有多条航班线路，这就是多重边的实例。
  • 自环（Loop/Self-loop）：从一个顶点出发连接到自身的边。在某些网络中（如引文网络中的自引），自环是有意义的。

下图总结了这些基本概念的关系：

概念 英文 含义 示例
顶点 Vertex/Node 系统中的个体 社交网络中的用户
边 Edge/Link 个体间的关系 Facebook中的好友关系
度 Degree 顶点的邻居数量 你有多少微信好友
路径 Path 顶点序列，相邻顶点间有边 A→B→C的转发链
子图 Subgraph 原图的一部分 一个班级内部的社交关系

图的分类

根据边的不同属性，图可以划分为多种类型。理解这些分类有助于我们选择合适的分析工具。

无向图与有向图

无向图（Undirected Graph）中的边没有方向，顶点 u 和 v 之间的边表示一种对称关系。例如，Facebook的好友关系——如果你是我的好友，我必然也是你的好友。微信的"相互关注"也是无向关系。

有向图（Directed Graph / Digraph）中的边有方向，通常用箭头表示。例如，Twitter的关注关系——你可以关注一个人，但对方不一定关注你。网页的超链接、论文的引用、电话的通话记录——这些都是有向关系，因为它们天然具有方向性。

有意思的是，在实际的网络分析中，我们经常需要将无向图和有向图结合起来思考。例如，微博的"关注"关系是有向的，但如果我们只看"互关"关系，就形成了一个无向子图。这种视角转换在分析中非常重要。

加权图与无权图

在很多实际问题中，边不仅仅是"存在"或"不存在"，而是具有某种强度或权重。加权图（Weighted Graph）为每条边赋予一个数值，称为权重（weight）。例如，在航空网络中，权重可以是航班距离、飞行时间或票价；在合作网络中，权重可以是两位学者共同发表论文的数量。

加权网络的分析比无权网络更加丰富但也更加复杂。例如，计算最短路径时，我们不能只看"经过了几步"（hop count），而是要累加边的权重。这种情况下，一条路径如果经过的步数更多但总权重更小，反而可能是"最优"路径。

二分图

二分图（Bipartite Graph）是一类非常重要的特殊图。它的顶点集可以划分为两个不相交的子集 U 和 V，使得所有的边都连接一个 U 中的顶点和一个 V 中的顶点——即，边不能出现在同一子集内部。

现实世界中，二分图无处不在：

  • 用户与商品的购买关系（用户集合 × 商品集合）
  • 演员与电影的参演关系（演员集合 × 电影集合）
  • 论文与作者的作者关系（论文集合 × 作者集合）
  • 患者与症状的临床表现（患者集合 × 症状集合）

很多看似不是二分图的网络，实际上可以"投影"为二分图来分析。例如，推荐系统中常用的协同过滤算法，本质上就是在用户—商品的二分图上寻找相似用户或相似商品。

实用技巧：在处理真实网络数据时，首要任务就是确认你的数据适合用哪种图来表示。一个常见的"坑"是：把本质上是二分图的数据直接当作普通图来处理，导致丢失了重要的结构信息。例如，如果你买了某本书，而你的朋友也买了同一本书——这个"共同购买"关系是通过"书"这个中间节点间接连接的，不直接代表你和朋友之间的关系。正确的做法是先构建用户—商品二分图，必要时再进行单模投影（one-mode projection）。

图的表示方法

在计算机中表示图有三种主要方法：邻接矩阵、邻接表和边列表。选择合适的表示方法对算法的效率有显著影响。

邻接矩阵（Adjacency Matrix）

对于一个有 n 个顶点的图，邻接矩阵是一个 n × n 的方阵 A。对于无权无向图，如果顶点 i 和顶点 j 之间有边，则 A[i][j] = A[j][i] = 1，否则为0。对于有向图，A[i][j] = 1 表示有一条从 i 指向 j 的边。加权图的邻接矩阵则存储相应的权重值。

邻接矩阵的优点是直观且适合矩阵运算（如计算 A² 可得到长度为2的路径数），但缺点是存储空间为 O(n²)，对于稀疏的大规模网络来说非常浪费。

邻接表（Adjacency List）

邻接表为每个顶点维护一个列表，列表中存储该顶点的所有邻居。无向图中每条边会被存储两次（在两端各出现一次）。邻接表的存储空间为 O(n + m)，其中 n 是顶点数，m 是边数。对于真实世界的稀疏网络（通常 m ≪ n²），邻接表是最高效的表示方式。

边列表（Edge List）

边列表就是简单的 (u, v) 对的集合，有时还包含权重 (u, v, w)。它占用的空间同样为 O(m)，但查询某个顶点的所有邻居时需要遍历整个列表，效率较低。边列表常用于数据存储和交换格式。

表示方法 存储空间 查询邻居 判断边是否存在 适用场景
邻接矩阵 O(n²) O(n) O(1) 稠密图、矩阵运算
邻接表 O(n+m) O(deg(v)) O(deg(v)) 稀疏图、大规模网络
边列表 O(m) O(m) O(m) 数据存储、文件交换 
# Python中使用NetworkX创建和查看图
import networkx as nx

# 创建一个无向图
G = nx.Graph()
G.add_edge("你", "朋友A")
G.add_edge("你", "朋友B")
G.add_edge("朋友A", "朋友C")

# 查看邻接表
print(dict(G.adjacency()))
# 输出: {'你': {'朋友A': {}, '朋友B': {}}, '朋友A': {'你': {}, '朋友C': {}}, ...}

# 查看邻接矩阵（作为numpy数组）
A = nx.adjacency_matrix(G).todense()
print(A)

特殊图结构

在研究网络时，我们会反复遇到几种特殊的图结构。了解它们的性质能为后续分析奠定基础。

树（Tree）

树是连通且无环（acyclic）的无向图。树的关键性质是：一个有 n 个顶点的树恰好有 n-1 条边。树中的任意两个顶点之间有且仅有一条路径。树是图论中最基础的结构之一。计算机网络中的生成树协议（Spanning Tree Protocol）、决策树算法、文件系统的目录结构——这些都是树的实际应用。

完全图（Complete Graph）

完全图 Kₙ 是指 n 个顶点之间两两相连的图。完全图的边数为 n(n-1)/2（无向）或 n(n-1)（有向）。完全图是"最稠密"的图——所有可能的连接都存在。虽然完全图在现实中很少直接出现（因为几乎不可能让所有人都认识所有人），但它常被用作理论分析的基准。例如，"网络密度"（density）的定义就是实际边数与完全图边数的比值。

平面图（Planar Graph）

平面图是可以画在平面上且边之间互不相交（在非顶点处不交叉）的图。电路板布线、地铁线路图、地图的行政区域图——这些都是平面图的实例。著名的四色定理（任何平面图的地图可以用四种颜色染色，使得相邻区域颜色不同）就是图论中最重要的成果之一。

路径与连通性

一张图摆在我们面前，最自然的问题之一就是：这些顶点之间是如何连接的？一个顶点能否"到达"另一个顶点？这就是路径与连通性的问题。

路径（Path）是一个顶点序列 (v₀, v₁, ..., vₖ)，其中每一对相邻顶点之间都有边相连。路径的长度通常用经过的边数来衡量（即路径中顶点数减1）。简单路径（Simple Path）不重复经过任何顶点。

环（Cycle）是一条起点和终点相同的路径，且除了起点和终点外不重复经过任何顶点。环在网络中具有重要意义——例如，社交网络中的"三角形"（三个两两相连的朋友）就是一个长度为3的环。

连通分量（Connected Component）是图中的一个最大连通子图：其中的任意两个顶点之间都有至少一条路径相连，且无法再加入任何其他顶点而不破坏连通性。

对于一个有向图，连通性的概念更加丰富：

  • 弱连通（Weakly Connected）：如果将所有的有向边替换为无向边后，图是连通的。
  • 强连通（Strongly Connected）：对于任意两个顶点 u 和 v，既有从 u 到 v 的路径，也有从 v 到 u 的路径。

有向图的连通性分析在实际中非常有用。例如，在Twitter的社交网络中，弱连通分量可以帮助我们发现"话题圈"——那些虽然不一定相互关注，但通过转发链条连接在一起的人群。而强连通分量则揭示了"互粉群体"——那些相互关注、信息在其中能够双向流动的紧密社区。

踩坑经验：我曾经在一个电商用户行为分析项目中，直接把用户间的购买关系当作无向图来分析连通性。结果发现整个网络几乎都是一个大连通分量——"所有人都在一个连通分量里"，看起来毫无结构。后来才意识到问题：用户间的"同购关系"本质上是二分图（用户×商品）的投影，确实会把几乎所有用户连在一起。正确的做法是分析商品之间的共现网络或用户特征的相似度网络。这个教训让我深刻认识到：图的类型选择直接决定了分析结果的意义。

理解路径与连通性后，我们就有了量化网络特性的基础工具。两个顶点之间的距离（最短路径的长度）、网络的直径（所有最短路径中的最大值）——这些概念将是我们下一章的重点内容。

通过本章的学习，我们掌握了图论的基本语言：从顶点和边出发，区分了有向与无向、加权与无权、简单图与多重图，学习了邻接矩阵、邻接表等表示方法，并认识了树、完全图、平面图等特殊结构。最重要的是，我们理解了路径与连通性——这个贯穿整个网络科学的核心概念。
