第5章 Hermite插值与分段插值

大家好，我是你们的讲师。今天我们来处理一个更贴近真实工程场景的插值问题。

想象你要设计一条过山车的轨道。轨道不仅要经过某些给定的点，在连接处还必须平滑——不能有"折角"，否则过山车经过时乘客的脖子会断掉。这意味着你不仅需要控制位置（函数值），还需要控制方向（导数值）。这就是Hermite插值要解决的问题。

但故事不止于此。Hermite插值的全局版本同样有Runge现象的困扰。于是工程师们想出了一个聪明的办法：与其用一个高次多项式去拟合整个区间，不如把区间切成很多小段，每段用低次多项式插值，在连接处保证一定的光滑性。这就是分段插值的思路——它奠定了现代计算机图形学和CAD系统的数学基础。

5.1 带导数条件的插值

Hermite插值问题可以这样描述：给定节点 x₀, x₁, ..., xₙ 上的函数值 f(xᵢ) 和一阶导数值 f'(xᵢ)，求一个多项式 H(x)，满足：

  • H(xᵢ) = f(xᵢ), i = 0, 1, ..., n
  • H'(xᵢ) = f'(xᵢ), i = 0, 1, ..., n
  • H(x) 的次数 ≤ 2n+1

共 2n+2 个条件，唯一确定一个 2n+1 次多项式。这就是Hermite插值的基本定理。

💡 直观理解： 每个节点提供两个条件（值和斜率），就像用两根手指按住一根弹簧——位置和方向都被固定了。n+1 个节点就有 2n+2 个条件，自然确定一个 2n+1 次多项式。

5.2 Hermite插值多项式的构造

Hermite插值最优雅的构造方法是通过均差的极限形式来实现。回忆上一章我们说均差趋近于导数：当两个节点无限靠近时，一阶均差趋近于导数。

利用这个性质，我们只需要把每个节点在节点序列中重复两次，然后使用Newton插值公式——但要小心处理重复节点处的均差：

import numpy as np

def hermite_interp(x_points, f_vals, fp_vals, x):
    """
    Hermite插值：使用重复节点的Newton均差
    x_points: 节点数组 (长度为 n)
    f_vals: 函数值 (长度为 n)
    fp_vals: 导数值 (长度为 n)
    x: 求值点
    """
    n = len(x_points)
    # 构造重复节点序列：每个节点重复两次
    z = np.zeros(2*n)
    for i in range(n):
        z[2*i] = x_points[i]
        z[2*i+1] = x_points[i]

    # 构造函数值表（第一列）
    F = np.zeros((2*n, 2*n))
    for i in range(n):
        F[2*i, 0] = f_vals[i]
        F[2*i+1, 0] = f_vals[i]

    # 处理重复节点的一阶均差——用导数值
    for i in range(n):
        F[2*i+1, 1] = fp_vals[i]
        if i > 0:
            # 非重复节点的正常一阶均差
            F[2*i, 1] = (F[2*i, 0] - F[2*i-1, 0]) / (z[2*i] - z[2*i-1])

    # 计算高阶均差
    for j in range(2, 2*n):
        for i in range(2*n - j):
            F[i, j] = (F[i+1, j-1] - F[i, j-1]) / (z[i+j] - z[i])

    # 用Newton形式求值
    result = F[0, 0]
    term = 1.0
    for i in range(1, 2*n):
        term *= (x - z[i-1])
        result += F[0, i] * term
    return result

# 示例：f(x)=sin(x), 节点 x=0, π/2, π
x_pts = np.array([0.0, np.pi/2, np.pi])
f_vals = np.sin(x_pts)
fp_vals = np.cos(x_pts)

x_test = np.pi/4
true_val = np.sin(x_test)
hermite_val = hermite_interp(x_pts, f_vals, fp_vals, x_test)
print(f"Hermite H({x_test:.4f}) = {hermite_val:.6f}")
print(f"真值 sin({x_test:.4f}) = {true_val:.6f}")
print(f"误差 = {abs(true_val - hermite_val):.2e}")
🔍 关键洞察： 均差在节点重合时退化为导数除以阶乘——这是微积分基本定理在离散世界的投影。通过"重复节点"这个技巧，Hermite插值被统一到了Newton插值框架下，不需要任何新的数学理论。

5.3 分段低次插值

Hermite插值虽然强大，但全局高次Hermite多项式同样有Runge现象的困扰。在实际工程中，分段低次插值才是主流方案。

基本思想：将区间 [a, b] 划分为若干子区间，在每个子区间上用低次多项式（线性、二次或三次）进行插值，保证整体连续性。

分段线性插值是最简单的形式——相邻节点间用直线连接。它在节点处C⁰连续（函数值连续），但导数不连续。"折线"视觉效果不光滑，但永远不会振荡。

分段三次Hermite插值（PCHIP）在每个子区间上用三次多项式，在节点处保证C¹连续（函数值和一阶导数都连续）。它在光滑性和稳定性之间取得了很好的平衡。

💡 数学细节： PCHIP的全称是Piecewise Cubic Hermite Interpolating Polynomial。它的误差界是 O(h⁴)，其中 h 是子区间的最大长度。这意味着把区间剖分加密一倍，误差大约缩小到原来的1/16。

5.4 实际工程应用

为什么游戏引擎和CAD软件都大量使用分段插值而不是全局高次插值？原因有三：

  • 局部性：移动一个控制点只影响附近几个子区间，不影响全局。这使交互式设计成为可能——设计师拖动一个点，只有附近的曲线跟着变化。
  • 稳定性：分段低次插值彻底避免了Runge现象，高次多项式的大范围振荡不会出现。
  • 计算效率：每个子区间只需要解一个小规模的线性系统（甚至不需要解系统），计算代价极低。
方法 整体光滑阶数 Runge现象 局部性 计算量 典型应用
全局高次Lagrange C∞ 严重 无 O(n²) 理论分析
分段线性插值 C⁰ 无 强 O(1) 数据可视化
分段三次Hermite (PCHIP) C¹ 无 强 O(1) CAD/CAM, 动画
三次样条 C² 无 中等 O(n) 字体渲染, 轨迹规划
⚠️ 踩坑故事： 某动画工作室的早期角色骨骼动画系统使用了全局高次多项式来插值关键帧。当动画师添加第20个关键帧时，角色的手臂在关键帧之间出现了诡异的"波浪形"抖动——Runge现象在动画中的体现。调试了整整两周后，团队换用了分段三次Hermite插值，问题立即消失。从此他们的技术文档里多了一条铁律："任何时候都不要用超过5次的全局多项式来做插值。"

在Python中，可以直接使用SciPy的PCHIP实现：

from scipy.interpolate import PchipInterpolator
import numpy as np

# 生成数据点
x = np.array([0, 1, 2, 3, 4, 5])
y = np.array([0, 2, 1, 4, 3, 5])

# 创建PCHIP插值器
pchip = PchipInterpolator(x, y)

# 在细密网格上求值
x_fine = np.linspace(0, 5, 100)
y_fine = pchip(x_fine)

# 对比全局三次多项式插值
poly_coeffs = np.polyfit(x, y, 5)
y_poly = np.polyval(poly_coeffs, x_fine)

# 在端点附近测试稳定性
print("PCHIP 在端点几乎不会振荡，而高次多项式会。")
🔍 为什么不是更高阶？ 你可能想问：既然分段三次能做到C¹，为什么不做分段四次实现C²？实际上分段三次样条（下章内容）就能做到C²，而且它使用的仍然是三次多项式——光滑性不是通过提高多项式次数，而是通过全局约束条件来实现的。这告诉我们一个重要的数值分析哲学：很多时候，限制自由度比增加自由度更有效。

下一章，我们进入三次样条插值的世界——它在PCHIP的基础上更进一步，通过求解一个三对角方程组来保证整体C²光滑。这是数值分析中"全局约束"思想的经典范例。
