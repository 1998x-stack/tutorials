第二十一章：多智能体强化学习

同学们好，欢迎来到第二十一章。从这一章开始，我们的视野要从"单个智能体独自学习"拓展到"多个智能体同时学习、互相影响"的世界。前面二十章我们都在讨论一个智能体面对一个环境——它做决策，环境反馈奖励，它更新策略，如此循环。这个框架虽然强大，但它有一个隐含假设：环境是平稳的。也就是说，环境的转移概率不会随着时间变化。

但现实世界不是这样的。想象一下：你是一个自动驾驶汽车，周围还有几十辆也在学习驾驶的汽车。你的决策会影响其他车的决策，其他车的决策反过来又会影响你的决策。每个人都在"学习"，每个人都在"适应"——这就是多智能体强化学习（Multi-Agent Reinforcement Learning, MARL）要解决的问题。

这一章的内容量不小，我们会先理解MARL的核心挑战，然后学习当今最主流的两个范式——CTDE和值分解方法，再聊聊自博弈这个在竞技场景中大放异彩的技术，最后看看几个标志性的应用。准备好了吗？咱们开始。

先讲一个"踩坑"故事：几年前我带学生做一个多机器人协作搬运的项目。我们用标准的DQN分别训练每个机器人——结果让人崩溃。第一次训练，两个机器人配合得天衣无缝，完美把箱子搬到了目标点。第二次训练（同样的代码！），两个机器人直接撞在一起打转，箱子丢在半路。第三次，一个机器人全程不动，另一个自己完成了全部工作。同一个代码，三次不同的结果。后来我们才意识到问题所在：每个机器人的策略都在变，A的策略变化意味着B的环境发生了变化，B的变化又反过来影响A。整个系统是一个"移动的目标"，DQN那条平稳性假设被完全打破了。这个"踩坑"经历，刚好引出了MARL最核心的挑战——非平稳性。

21.1 MARL的核心挑战

单智能体强化学习建立在马尔可夫决策过程（MDP）之上，其核心假设是环境转移概率 P(s' | s, a) 是固定的。但在多智能体场景中，每个智能体 i 的转移不仅取决于它自己的动作，还取决于所有其他智能体的动作：

P(s' | s, a_1, a_2, ..., a_n)

而其他智能体也在学习，它们的策略 π_j 在变化。这意味着对智能体 i 来说，环境每时每刻都在变——因为对手/队友的策略变了。非平稳性（Non-stationarity） 是MARL的第一大挑战，它直接让标准的DQN、PPO等方法失效。

MARL的第二大挑战是可扩展性。每个智能体的动作空间大小是|A|，n个智能体的联合动作空间就是|A|^n。当n=10、|A|=5时，联合动作空间就有近1000万种组合。穷举是不可能的。

第三大挑战是信用分配（Credit Assignment）：团队赢了，谁的贡献最大？这个问题在单智能体场景中就不简单（是第3步的车还是第15步的炮？），在多智能体场景中成倍复杂。

根据智能体之间的关系，MARL问题可以分为三类：

类别 奖励结构 核心目标 典型例子
完全合作 所有智能体共享同一个奖励 r(s, a_1, ..., a_n) 最大化团队总回报 多机器人协作搬运
完全竞争 零和博弈：r_1 = -r_2 最大化自身收益（最小化对方收益） 围棋、象棋、格斗游戏
混合场景 一般和博弈：每个智能体有自己的奖励 个体理性与群体理性的权衡 自动驾驶、经济博弈

理解要点：合作场景的核心难题是如何让多个智能体协同，避免"搭便车"行为；竞争场景的核心难题是如何对抗一个不断进化的对手；混合场景则两者兼具，是最难也最接近现实的一类。

21.2 集中训练分散执行（CTDE）

面对非平稳性挑战，研究者们提出了一种优雅的思路——集中训练分散执行（Centralized Training with Decentralized Execution, CTDE）。这个想法概括起来就是：训练的时候让大家"共享视野"，执行的时候每个人"各管各的"。

具体来说：训练阶段，智能体可以访问全局信息（所有智能体的观测、动作甚至奖励），从而做出更准确的梯度更新；执行阶段，每个智能体只能基于自己的局部观测来做决策，不依赖任何通信。这个范式之所以强大，是因为它避开了"执行时需要通信"这个工程瓶颈，同时利用训练阶段的信息优势缓解非平稳性问题。

CTDE最经典的算法是MADDPG（Multi-Agent Deep Deterministic Policy Gradient）。MADDPG是DDPG在多智能体场景的扩展，核心贡献是"集中式评论家，分布式演员"：

  • 演员（Actor）：每个智能体有自己的策略网络 π_i(o_i)，只依赖自己的观测 o_i，在分布式执行时做决策。
  • 评论家（Critic）：每个智能体有一个集中式Q函数 Q_i(s, a_1, ..., a_n)，以全局状态和所有智能体的动作为输入，评估当前联合动作的好坏。

MADDPG的评论家更新公式为：

L(θ_i^Q) = E[ (Q_i(s, a_1, ..., a_n) - (r_i + γ · Q_i'(s', a_1', ..., a_n')))^2 ]

其中 a_j' 来自目标策略网络 π_j'。注意，即使智能体 i 只能控制自己的动作，它的Q函数也知道了所有其他智能体的动作。这就缓解了非平稳性问题——因为评论家知道了"别人在做什么"，就可以更准确地判断当前状态值。

CTDE的关键洞察：评论家需要知道的"上下文"比演员多得多。评论家可以奢侈地使用全局信息，因为它的参数只在训练时更新，不需要在部署时使用。这是一种"花训练时的计算，换取执行时的稳健性"的trade-off。

21.3 值分解方法

在完全合作场景中，所有智能体共享一个团队奖励。这时候我们面对的问题是：如何把团队的总Q值分配给每个智能体？值分解方法（Value Decomposition）就是回答这个问题的技术家族。

VDN（Value Decomposition Networks） 是2017年提出的最早的方法，想法极其简单：团队的总Q值等于每个智能体Q值的和。

Q_tot(τ, a) = Σ_i Q_i(τ_i, a_i)

其中 τ_i 是智能体 i 的历史轨迹。VDN假设各智能体的贡献是可加性分离的——这显然太强了。很多任务中，智能体之间的相互作用不是简单的加法关系。

QMIX 是VDN的升级版，2018年发表在ICML上，至今仍是合作MARL的标杆算法。QMIX的核心创新是：团队总Q值由混合网络（Mixing Network）计算，这个网络以所有智能体的个体Q值为输入，以全局状态s为条件，输出Q_tot。更重要的是，QMIX施加了一个单调性约束：

∂Q_tot / ∂Q_i ≥ 0, ∀i

这意味着任何一个智能体的Q_i增大，团队总Q_tot要么增大要么不变，但不会减小。这个约束保证了团队最优和个体最优方向一致，同时比VDN的加法分离灵活得多——混合网络可以是任意复杂的单调函数（通常用绝对值激活来保证正权重，或者用神经网络加约束）。

下面是一个简化的QMIX混合网络伪代码：

class QMIX:
    def __init__(self, n_agents, state_dim, hidden_dim):
        # 每个智能体一个DRQN（深度循环Q网络）
        self.agents = [DRQN(state_dim, hidden_dim) for _ in range(n_agents)]
        # 混合网络：智能体数量必须固定
        self.hypernet = HyperNetwork(state_dim, n_agents)

    def forward(self, obs_list, state):
        # 每个智能体用自己的DRQN计算个体Q值
        q_vals = []
        for i, obs in enumerate(obs_list):
            q_i = self.agents[i](obs)          # Q_i(τ_i, a_i)
            q_vals.append(q_i)

        # 混合网络以全局状态为条件，保证正权重
        weights, bias = self.hypernet(state)    # 所有权重>0
        q_tot = weighted_sum(q_vals, weights) + bias
        return q_tot, q_vals

    # 训练时用Q_tot计算TD误差，梯度同时更新混合网络和个体DRQN

QMIX通过单调性约束，可以表达比VDN更丰富的信用分配关系。例如：两个搬运机器人，一个推左、一个推右，单独任何一个都推不动箱子，必须同时用力才能移动。VDN学不会这种"互补关系"（因为每个Q_i单独看都是0），而QMIX的混合网络可以捕捉到这种非线性协作。

技术洞察：QMIX的单调性约束是"结构化的先验知识"——它告诉模型智能体之间是协作关系而非对抗关系。如果你在非合作场景强行用QMIX，单调性反而会变成限制。这也说明了一个重要原则：选择算法时一定要匹配问题的结构。

21.4 自博弈（Self-Play）

在完全竞争场景中，一个优雅的想法横空出世：让智能体和自己的副本对抗训练。这就是自博弈（Self-Play）。

自博弈的思想听起来简单：你找一个对手，对手就是你自己之前的某个版本。你赢不了"过去的自己"，那你就进步；你进步之后，"新的自己"再去挑战更老的自己。这个循环自动形成了一个课程学习（Curriculum Learning）——对手的难度总是和你当前的水平匹配。

自博弈最辉煌的成就包括：

  • AlphaGo / AlphaZero：完全通过自博弈从零开始学习围棋、国际象棋和将棋，不使用任何人类棋谱。AlphaZero仅用24小时的自博弈训练就超越了所有之前的棋类AI。
  • OpenAI Five：在Dota 2中击败世界冠军OG战队。训练过程中使用了自博弈+大规模分布式训练，每天积累180年的人类游戏经验。
  • AlphaStar：在StarCraft II中达到大师段位，使用了自博弈+联盟训练（League Training）的组合策略。

但自博弈不是万能的。它有一个著名的问题——策略循环（Policy Cycles）。想象三个策略A、B、C，A能稳定打败B，B能稳定打败C，但C又能打败A。自博弈的策略只在一条"链"上迭代，可能永远跳不出这个循环。现实中，不同的策略风格之间确实存在这种"石头剪刀布"式的循环关系。

解决方案是种群训练（Population-Based Training）：不只训练一个策略，而是维护一个策略"种群"。每个智能体从种群中随机选取对手进行训练，而不是只和自己的副本对练。这样即使某个策略陷入了循环，种群中的其他策略可以提供多样性，帮助跳出循环。AlphaStar的联盟训练就是一个典型——维护了一个包含不同打法的"联盟"，定期淘汰弱策略、引入新策略。

踩坑延续：回到开头那个机器人搬运的故事。我们后来的解决方案就是用CTDE框架——训练时每个机器人的网络可以看到全局状态，执行时只用自己的局部观测。这就像两个搬运工在训练时可以用对讲机协调（CTDE的集中训练），但实际干活时各自凭经验配合（分散执行）。训练稳定性大幅提升，两个机器人终于学会了稳定协作，而不是时好时坏。这个教训让我明白了：MARL不是"单智能体RL+n个副本"，它有自己独特的理论框架和工程挑战。

21.5 MARL标志性应用

MARL虽然学术上充满挑战，但它已经产出了一系列震撼世界的应用。我们来看看几个标志性的项目：

AlphaStar（DeepMind, 2019）：在即时战略游戏StarCraft II中达到大师级以上水平，超过了99.8%的人类玩家。AlphaStar使用了深度神经网络+Transformer架构，结合自博弈和联盟训练。MARL在其中的角色至关重要——多个智能体分别控制不同的单位（工兵、军队、侦查单位），需要实现微观操作层面的协同作战。

OpenAI Five（OpenAI, 2019）：在Dota 2这一极其复杂的MOBA游戏中击败世界冠军OG战队。五个英雄由五个独立的LSTM策略网络控制，通过自博弈和PPO算法训练。峰值训练使用了256台GPU和128,000个CPU核——这告诉我们MARL中计算资源的重要性。

Hide-and-Seek（OpenAI, 2019）：一个被严重低估的里程碑。OpenAI在一组捉迷藏环境中训练智能体，观察到了完全自发的工具使用行为——寻找方块的智能体学会了用障碍物搭建掩体，而寻找它们的智能体学会了用斜坡翻越障碍物。更精彩的是，随着训练进行，策略之间出现了连续的"军备竞赛"：一方找到对策，另一方再进化。这个实验完美展示了MARL中涌现（Emergence）的威力。

SMAC（StarCraft Multi-Agent Challenge）：这是MARL社区最常用的基准测试平台，由WhiRL实验室开发。SMAC提供了多种对战场景，覆盖了同构和异构智能体、不同队伍大小的设置。QMIX、VDN、MADDPG等算法都在SMAC上进行了系统的比较，是入坑MARL实验研究的首选平台。

从这些应用中，我们可以总结出一个重要规律：在MARL中，规模和计算量直接决定性能的上限。AlphaStar用了大量TPU，OpenAI Five用了超大规模分布式训练。这不是说小规模MARL实验没有价值——相反，理解算法在玩具环境中的行为是拓展到大规模系统的基础。但它提醒我们，不要把在小环境中的成功过度推广到复杂场景。

本章思考：MARL可能是强化学习中最"迷人"也最"恼人"的子领域。迷人之处在于它最接近真实世界——多个智能体共同演化、互相影响；恼人之处在于训练不稳定、复现困难、计算开销巨大。但正是这种挑战性，让MARL成为学术界和工业界最活跃的研究方向之一。从多机器人协作到自动驾驶车队，从金融市场模拟到社交网络推荐——理解和控制多智能体系统，是我们通向通用人工智能道路上绕不开的一环。

下一章我们将进入一个与MARL密切相关但视角完全不同的领域——逆强化学习（Inverse Reinforcement Learning, IRL）和模仿学习。如果说MARL关注的是"多个智能体如何同时学习"，那么逆强化学习关注的是"我们如何从人类专家的示范中推断出奖励函数"。当你不清楚奖励函数怎么设计时，就让专家做给你看——这个思路和MARL中的模仿学习、对手建模有着千丝万缕的联系。让我们在第22章继续探索。
