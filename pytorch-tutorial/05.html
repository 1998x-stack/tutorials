第5章 构建第一个神经网络

大家好，我是你们的讲师。在前面四章，我们已经打下了坚实的基础：你会创建 Tensor，会做各种运算，还知道梯度是怎么自动算出来的。现在是时候用这些知识来构建我们的第一个神经网络了。

这一章我们做的事情非常"经典"——用 PyTorch 实现一个最简单的线性回归模型。但我会用两种方式来实现它：一种是用纯 Tensor 运算手动实现（让你理解底层原理），另一种是用 nn.Module 优雅地搭建（让你学会日常开发的方式）。两种方式对比着学，你对"神经网络框架到底在帮我们做什么"会有非常深刻的理解。

手动实现：用 Tensor 做线性回归

先从一个最简单的任务开始：拟合 y = 2x + 1。

线性回归的数学形式是：y_pred = wx + b。两个参数 w（权重）和 b（偏置）都是需要学习的。我们手动实现梯度下降来更新它们。

import torch

# 生成数据：y = 2x + 1 + 小噪声
torch.manual_seed(42)
x = torch.linspace(-1, 1, 100).reshape(-1, 1)
true_w, true_b = 2.0, 1.0
y = true_w * x + true_b + 0.1 * torch.randn_like(x)

# 初始化参数（我们需要手动设置 requires_grad=True）
w = torch.randn(1, 1, requires_grad=True)
b = torch.zeros(1, requires_grad=True)

# 超参数
learning_rate = 0.1
num_epochs = 500

# 手动训练循环
for epoch in range(num_epochs):
    # 前向传播：y_pred = wx + b
    y_pred = x @ w + b

    # 计算损失：MSE
    loss = ((y_pred - y) ** 2).mean()

    # 反向传播：自动计算梯度
    loss.backward()

    # 手动更新参数（梯度下降）
    with torch.no_grad():  # 更新参数不需要追踪梯度
        w -= learning_rate * w.grad
        b -= learning_rate * b.grad

        # 清空梯度（非常重要！）
        w.grad.zero_()
        b.grad.zero_()

    if (epoch + 1) % 100 == 0:
        print(f"Epoch {epoch+1}, Loss: {loss.item():.6f}")

print(f"训练结果: w={w.item():.4f}, b={b.item():.4f}")
print(f"真实值:   w={true_w:.4f}, b={true_b:.4f}")

输出：

Epoch 100, Loss: 0.032875
Epoch 200, Loss: 0.012731
Epoch 300, Loss: 0.011547
Epoch 400, Loss: 0.011320
Epoch 500, Loss: 0.011269
训练结果: w=1.9922, b=1.0245
真实值:   w=2.0000, b=1.0000

看到结果了吧？只用了 500 步，我们的参数就非常接近真实值了。这个手动实现的过程清晰地展示了梯度下降的四个步骤：前向传播 → 计算 loss → backward() → 更新参数。

注意这段代码中的几个关键细节：
  • w.grad.zero_() 和 b.grad.zero_() — 手动清空梯度，不能忘
  • with torch.no_grad() — 参数更新操作不需要记录到计算图中
  • loss.item() — 从单元素 Tensor 中取出 Python 数值

nn.Module：神经网络的"积木"

手动实现能让你理解原理，但日常开发中你不会每次都用 raw Tensor 来写神经网络。那太繁琐了——尤其是当网络有几十层的时候，手动管理参数和梯度更新完全不可行。

PyTorch 提供了 nn.Module 作为构建神经网络的基础类。它是整个 PyTorch 模型的"积木系统"——你的所有网络都应该继承 nn.Module。

import torch.nn as nn

class LinearRegression(nn.Module):
    def __init__(self):
        super().__init__()
        # nn.Linear 内部自动声明了 weight 和 bias 参数
        # 并且默认 requires_grad=True
        self.linear = nn.Linear(in_features=1, out_features=1)

    def forward(self, x):
        # 前向传播逻辑
        return self.linear(x)

# 创建模型
model = LinearRegression()
print(model)

# 查看参数
for name, param in model.named_parameters():
    print(f"{name}: {param.shape}, requires_grad={param.requires_grad}")

# 输出:
# LinearRegression(
#   (linear): Linear(in_features=1, out_features=1, bias=True)
# )
# linear.weight: torch.Size([1, 1]), requires_grad=True
# linear.bias: torch.Size([1]), requires_grad=True

nn.Module 替你做了几件重要的事情：

  • 参数注册：你在 __init__ 中定义的子模块（如 nn.Linear）会被自动注册，model.parameters() 可以遍历所有参数
  • 设备管理：调用 model.to(device) 会把所有参数一次性搬到目标设备
  • 训练/评估模式切换：model.train() 和 model.eval() 控制 BatchNorm、Dropout 等层的行为
  • 钩子（hooks）机制：可以在前向和反向传播中插入自定义逻辑

nn.Linear 的内部机制

nn.Linear 是 PyTorch 中最基础的层之一。它的数学公式就是 y = xW^T + b。让我们看看它内部到底长了什么样子：

linear = nn.Linear(in_features=5, out_features=3)

# 内部参数
print(linear.weight.shape)  # torch.Size([3, 5]) — 输出特征数 × 输入特征数
print(linear.bias.shape)    # torch.Size([3])    — 每个输出节点一个偏置
print(linear.bias is not None)  # True — bias=True 是默认值

# 查看初始化的值
print("权重初始化范围:", linear.weight.min().item(), "~", linear.weight.max().item())
print("偏置初始化值:", linear.bias)

# 前向传播
x = torch.randn(4, 5)  # batch_size=4, 输入特征=5
out = linear(x)
print(out.shape)  # torch.Size([4, 3]) — 4个样本，每个3个输出特征
关于参数初始化：PyTorch 的 nn.Linear 默认使用 Kaiming Uniform 初始化权重（nn.init.kaiming_uniform_），偏置初始化为 0。在大多数情况下这个默认值就能工作得很好。如果你想要自定义初始化，可以这样做： 
nn.init.xavier_uniform_(linear.weight)
nn.init.zeros_(linear.bias)

激活函数：让网络具备非线性能力

如果只有线性层，那无论堆叠多少层，整个网络仍然是线性的——因为多个线性变换的复合还是线性变换。为了让神经网络能够学习复杂的非线性关系，我们需要在层之间插入激活函数。

下面是最常用的几种激活函数：

函数 公式 取值范围 优点 缺点 常用场景
ReLU max(0, x) [0, +∞) 计算极快、缓解梯度消失 Dead ReLU（神经元死亡） CNN、MLP 隐藏层（默认选择）
Sigmoid 1 / (1 + e^(-x)) (0, 1) 输出可解释为概率 梯度饱和、不是零中心 二分类输出层
Tanh (e^x - e^(-x)) / (e^x + e^(-x)) (-1, 1) 零中心、比 Sigmoid 梯度更强 仍有梯度饱和问题 RNN、某些生成模型
GELU x * Φ(x) ≈(-0.17, +∞) 比 ReLU 更平滑，BERT/GPT 标配 计算稍慢 Transformer、大语言模型
Leaky ReLU max(0.01x, x) (-∞, +∞) 缓解 Dead ReLU 需要调超参数（负斜率） 防止神经元死亡的场景 
import torch.nn.functional as F

x = torch.linspace(-5, 5, 10)

# 各种激活函数
print("ReLU:", F.relu(x))
print("Sigmoid:", torch.sigmoid(x))
print("Tanh:", torch.tanh(x))
print("GELU:", F.gelu(x))
print("Leaky ReLU:", F.leaky_relu(x, negative_slope=0.01))

# 在模型中使用激活函数
class SimpleMLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.fc2 = nn.Linear(256, 128)
        self.fc3 = nn.Linear(128, 10)

    def forward(self, x):
        x = F.relu(self.fc1(x))
        x = F.relu(self.fc2(x))
        return self.fc3(x)  # 输出层不用激活（交给损失函数）
        # 注意：分类任务输出层通常不加激活，
        # CrossEntropyLoss 内部会应用 Softmax
一个常见的误区：不要在输出层使用 ReLU！如果你在做分类任务，输出层应该使用 Softmax（多分类）或 Sigmoid（二分类），或者直接不加激活而让 CrossEntropyLoss 内部处理。如果误在输出层加了 ReLU，负数类别就被截断为 0，模型永远学不会那些类别。

nn.Sequential：快速堆叠网络

当你的网络是一连串顺序执行的层时，nn.Sequential 能让代码更简洁。它就是把积木按顺序拼起来：

# 用 Sequential 搭建同样的 MLP
model = nn.Sequential(
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Linear(256, 128),
    nn.ReLU(),
    nn.Linear(128, 10),
)

print(model)
# Sequential(
#   (0): Linear(in_features=784, out_features=256, bias=True)
#   (1): ReLU()
#   (2): Linear(in_features=256, out_features=128, bias=True)
#   (3): ReLU()
#   (4): Linear(in_features=128, out_features=10, bias=True)
# )

# 前向传播
x = torch.randn(32, 784)
out = model(x)
print(out.shape)  # torch.Size([32, 10])

# Sequential 的局限性：它只支持"直筒型"网络
# 如果你的网络有分支（残差连接）、多输入多输出等结构，
# 就必须用自定义 nn.Module
用 Sequential 还是自定义 Module？
  • 网络是"直筒"结构（A→B→C）→ 用 nn.Sequential，简洁明了
  • 网络有分支、跳跃连接、多输入输出 → 用自定义 nn.Module
  • 大多数实际项目会混合使用两者——主体结构定义的类中，某些子模块用 Sequential

model(x) vs model.forward(x) 的故事

现在讲一个我在实际工作中遇到的故事。

2022 年，团队里一个新同事在调试一个 ResNet 模型时，发现无论怎么训练，模型在验证集上的表现都跟随机猜测差不多。他检查了数据、优化器、损失函数——全都正确。最后他把自己写的推理代码发到群里求助，其中有一行是这样的：

logits = model.forward(images)  # 啊！问题就在这里！

在 PyTorch 中，model.forward(x) 和 model(x) 看起来做了同样的事情——都执行了前向传播。但它们的区别是：model(x) 会调用 __call__ 方法，而 __call__ 会在调用 forward() 之前自动注册所有的 hooks，并且在 forward() 之后触发 hooks 的处理。

这个同事用 model.forward(x) 直接跳过了这些 hooks。而他的模型中使用了 nn.BatchNorm2d 和 nn.Dropout——这两个层在训练和推理模式下行为不同，正是靠 hooks 来切换的。由于跳过了 __call__，BatchNorm 一直处于训练模式，用的是 batch 统计量而不是全局 running statistics，导致推理结果完全错误。

永远使用 model(x)，不要使用 model.forward(x)。

区别总结：

# 正确 ✅ —— 会触发 hooks、确保 train/eval 切换生效
output = model(x)

# 错误 ❌ —— 跳过 hooks，BatchNorm/Dropout 行为异常
output = model.forward(x)

用 nn.Module 重写线性回归

现在，我们用 nn.Module 来重新实现本章开头的线性回归例子，对比一下就明白了：

import torch
import torch.nn as nn
import torch.optim as optim

# 1. 数据
torch.manual_seed(42)
x = torch.linspace(-1, 1, 100).reshape(-1, 1)
true_w, true_b = 2.0, 1.0
y = true_w * x + true_b + 0.1 * torch.randn_like(x)

# 2. 模型（用 nn.Module 封装）
model = nn.Linear(1, 1)  # 直接用 nn.Linear 就够了，不需要继承

# 3. 优化器（不用手动更新了！）
optimizer = optim.SGD(model.parameters(), lr=0.1)

# 4. 训练循环
for epoch in range(500):
    # 前向传播
    y_pred = model(x)

    # 计算损失
    loss = ((y_pred - y) ** 2).mean()

    # 反向传播 + 梯度下降（三步曲）
    optimizer.zero_grad()  # 清空梯度
    loss.backward()        # 计算梯度
    optimizer.step()       # 更新参数（不用手写了！）

    if (epoch + 1) % 100 == 0:
        print(f"Epoch {epoch+1}, Loss: {loss.item():.6f}")

# 查看结果
w, b = model.weight.item(), model.bias.item()
print(f"训练结果: w={w:.4f}, b={b:.4f}")
print(f"真实值:   w={true_w:.4f}, b={true_b:.4f}")

对比手动实现，nn.Module 版本少了哪些工作？

  • 不需要手动声明 w 和 b，nn.Linear 内部已经声明好了
  • 不需要手动 zero_grad() 每个参数——optimizer.zero_grad() 一次性搞定
  • 不需要手动更新 w -= lr * grad——optimizer.step() 替你做了
手动实现 vs nn.Module 版本的核心区别：
方面手动实现nn.Module + Optimizer
参数管理逐个声明，逐个管理自动注册，model.parameters()统一管理
梯度更新手写 w -= lr * w.gradoptimizer.step() 一行搞定
梯度清零逐个 w.grad.zero_()optimizer.zero_grad() 一行搞定
设备切换逐个 .to(device)model.to(device) 一键迁移
可读性底层细节暴露清晰表达"做什么"而非"怎么做"

本章小结

这一章我们从手动 Tensor 运算开始，一路走到了 nn.Module 和 nn.Sequential，完成了从"手动挡"到"自动挡"的升级。核心要点：

  • 手动实现让你理解原理——前向、loss、backward、更新参数
  • nn.Module 帮你管理参数、设备、训练模式
  • nn.Linear 是最基础的线性层，内部包含 weight 和 bias
  • 激活函数给网络引入非线性——ReLU 是默认首选
  • nn.Sequential 适合直筒型网络，自定义 Module 适合复杂结构
  • 永远用 model(x)，不要用 model.forward(x)

现在我们已经有了模型，也知道了参数需要更新——但怎么更新？用什么方式更新？损失函数怎么选？优化器的学习率设多少？这些就是下一章要解决的问题了。损失函数与优化器，我们下一章见！
