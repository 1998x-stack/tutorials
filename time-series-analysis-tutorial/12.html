第12章 季节性ARIMA (SARIMA)

大家好！前一章我们学习了 ARIMA 模型，它通过差分解决了趋势非平稳问题。但如果数据还有另一个常见的非平稳来源——季节性，ARIMA 就力不从心了。

想一想这些场景：电商销售额在每年双十一暴涨，冰淇淋销量每夏季达到顶峰，电力消耗在每天傍晚出现高峰……这些都是季节性模式。季节性的特点是：它会在固定间隔的时间点上重复出现。处理季节性，我们需要一个更强大的工具——季节性 ARIMA，简称 SARIMA。

12.1 季节性带来的挑战

季节性对时间序列建模意味着什么？假设你有每月的气温数据，周期 s=12（12 个月一个轮回）。如果你用普通的 ARIMA(1,1,1) 来拟合，会发生两件事：

  • ACF 和 PACF 在季节性滞后（如滞后 12、24、36）上出现显著高峰
  • 残差在季节性滞后上仍然有强烈的自相关，说明模型没有捕捉到季节性
  • 预测结果会忽视季节模式——比如预测 7 月的气温和 1 月一样低，明显荒唐

ARIMA 模型假设数据只存在"短程依赖"（recent lags matter），而季节性本质上是"远程依赖"（lags at multiples of s also matter）。要处理这种远程依赖，我们需要在 ARIMA 框架中增加专门的季节性成分。

关键洞察：季节性 ARIMA 不是对 ARIMA 的替代，而是对 ARIMA 的扩展。它相当于在 ARIMA 的基础上，"并行地"增加了一套针对季节性滞后的 ARMA 结构。

12.2 SARIMA(p,d,q)(P,D,Q)s 的含义

SARIMA 的完整记号是 SARIMA(p,d,q)(P,D,Q)s，看起来参数很多，但分解后并不复杂：

参数 含义
p, d, q 非季节性部分的 ARIMA 阶数（和之前一样）
P 季节性自回归阶数（如 P=1 表示 Y_{t-s} 对 Y_t 有线性影响）
D 季节性差分阶数（如 D=1 表示做 Δ_s Y_t = Y_t - Y_{t-s}）
Q 季节性移动平均阶数（如 Q=1 表示 ε_{t-s} 对 Y_t 有影响）
s 季节周期长度（月度数据 s=12，季度数据 s=4，周数据 s=7）

完整模型（不写常数项）为：

Φ_p(L) Φ_P(L^s) (1-L)^d (1-L^s)^D Y_t = Θ_q(L) Θ_Q(L^s) ε_t

其中 Φ_P(L^s) = 1 - Φ₁L^s - Φ₂L^{2s} - ... - Φ_PL^{Ps} 处理季节性自回归，Θ_Q(L^s) 处理季节性移动平均。注意大写字母 Φ 和 Θ 表示季节性参数，与非季节性的小写 φ 和 θ 区分。

理解季节性差分：非季节性差分 ΔY_t = Y_t - Y_{t-1} 移除的是"相邻时刻的趋势"；季节性差分 Δ_s Y_t = Y_t - Y_{t-s} 移除的是"周期之间的趋势"。比如对月度数据做 Δ₁₂Y_t = Y_t - Y_{t-12}，得到的是"今年 7 月相比去年 7 月的变化量"。如果这个变化量是平稳的，那 D=1 就够了。

12.3 季节性 ACF 与 PACF 模式

判断 SARIMA 的参数和非季节性 ARIMA 一样，要借助 ACF 和 PACF。只是现在需要同时看"短滞后"和"季节性滞后"。

假设数据是月度周期（s=12），ACF 和 PACF 的解读需要分两层：

  • 非季节性部分：观察滞后 1~11 的 ACF/PACF 模式，这决定了 p 和 q
  • 季节性部分：观察滞后 12、24、36 的 ACF/PACF 模式，这决定了 P 和 Q

季节性 ACF/PACF 的判断规则和非季节性是平行的：

季节性成分 季节性滞后上的 ACF 季节性滞后上的 PACF
SAR(P) 拖尾（在 s, 2s, 3s 上逐渐衰减） 在滞后 Ps 后截尾
SMA(Q) 在滞后 Qs 后截尾 拖尾（在 s, 2s, 3s 上逐渐衰减）
踩坑故事：我接手的一个零售预测项目中，同事用 SARIMA(1,1,1)(1,1,1)₁₂ 拟合了所有 500 个 SKU 的月销量数据，结果有一半的 SKU 预测结果极差。调试后发现，很多 SKU 的 s 根本不是 12——有些快消品有 4 周的补货周期（s=4），有些有 13 周的季度周期（s=13）。更糟糕的是，季节性差分的阶数 D 被统一设为 1，但某些 SKU 的季节性非常稳定，根本不需要季节性差分。教训是：不要盲目套用默认参数，一定要逐个检查每个序列的 ACF/PACF 特征。

12.4 季节性成分与非季节性成分的耦合

SARIMA 的一个微妙之处是季节性部分和非季节性部分会"耦合"（coupling）。它们不是完全独立的两个模型简单地相加，而是会通过数据相互影响。

举个具体的例子。假设你拟合了 SARIMA(1,0,0)(1,0,0)₁₂ 模型：

Y_t = c + φ₁Y_{t-1} + Φ₁Y_{t-12} - φ₁Φ₁Y_{t-13} + ε_t

注意看！这个模型在滞后 13 处有一个交叉项 -φ₁Φ₁Y_{t-13}。这是多项式乘法 (1-φ₁L)(1-Φ₁L¹²)Y_t 展开的结果。这个交叉项是自动出现的，它意味着季节性模型隐含地包含了一些"混合滞后"的效应。

这种现象在实际中意味着什么？如果你发现一个 SARIMA(1,0,0)(1,0,0)₁₂ 模型在滞后 13 处有显著的自相关残留，不要急着增加额外的参数——先检查是不是交叉项的系数估计出了问题。

实用建议：SARIMA 的参数空间很大（p,d,q,P,D,Q,s 七个维度），手动搜索非常耗时。建议先用 auto.arima 或 pmdarima 的 auto_arima 函数设置 seasonal=True 进行自动搜索，再根据残差诊断微调。大多数情况下，D ≤ 1，P ≤ 2，Q ≤ 2 就足够了。

12.5 Python 实操：SARIMA 建模

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from statsmodels.tsa.arima.model import ARIMA
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf

# 生成带季节性的模拟数据（月度周期 s=12）
np.random.seed(42)
n = 300  # 25 年
t = np.arange(n)
trend = 0.1 * t
seasonal = 5 * np.sin(2 * np.pi * t / 12)
noise = np.random.normal(0, 1, n)
Y = trend + seasonal + noise

# 区分趋势和季节性
fig, (ax1, ax2) = plt.subplots(2, 1, figsize=(12, 6))
ax1.plot(Y)
ax1.set_title('原始数据（趋势 + 季节性 + 噪声）')

# 对季节性差分后的序列画 ACF/PACF
Y_seas_diff = pd.Series(Y).diff(12).dropna()
plot_acf(Y_seas_diff, lags=30, ax=ax2)
plt.show()

# 拟合 SARIMA(0,1,1)(0,1,1)12
# 注意：在 ARIMA 类中，seasonal_order 参数指定季节性部分
model = ARIMA(
    Y,
    order=(0, 1, 1),                # 非季节性: ARIMA(0,1,1)
    seasonal_order=(0, 1, 1, 12),    # 季节性: (0,1,1) 周期 12
    trend='c'
)
result = model.fit()
print(result.summary())

# 预测未来 24 个月
forecast = result.forecast(steps=24)
plt.figure(figsize=(12, 4))
plt.plot(Y, label='历史值')
plt.plot(np.arange(n, n+24), forecast, 'r--', label='预测值')
plt.legend()
plt.title('SARIMA 预测 - 未来 24 个月')
plt.show()

SARIMA 模型是目前为止我们学习的最完整的"经典时间序列模型"。它能够同时处理趋势非平稳（通过 d）和季节性非平稳（通过 D），是 ARIMA 家族中最强大的成员。但模型参数越多，选择越复杂——到底哪个 (p,d,q)(P,D,Q)s 组合才是最优的？这需要系统的建模实践和诊断方法。下一章我们将进入 Box-Jenkins 流程的核心环节，学习如何做模型诊断、检验假设、规避常见的陷阱。
