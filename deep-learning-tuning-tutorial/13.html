数据增强——"免费"的数据扩增

大家好，我是你们的讲师。今天我们来聊一个回报率最高的调参技巧——数据增强（Data Augmentation）。

我见过一个项目，团队花了两周调参——调学习率、换优化器、加正则化——全都试遍了，验证集准确率从72%卡在73%不动。后来他们做了件事：在数据预处理中加了一行代码——transforms.RandomHorizontalFlip(p=0.5)。一行代码，准确率从73%跳到了77%。

这就是数据增强的魔力。它不是通过改变模型或训练方式，而是通过改变模型看到什么来提升泛化。一张猫的图片，水平翻转后仍然是猫——模型学会了"猫的样子与朝向无关"。一个句子，删除一个无关词语后语义不变——模型学会了"核心语义与修饰词无关"。数据增强的本质，就是在告诉模型："这些变换不重要，你别去关心它们。"

数据增强作为隐式正则化

从信息论的角度来看，数据增强等价于向模型注入先验知识。每当你对数据施加一个保持标签语义不变的变换，你就在告诉模型："这个变换不改变答案，所以与你的决策边界无关。"

  • 水平翻转 → 告诉CV模型"物体的左右朝向不影响类别"
  • 随机裁剪 → 告诉模型"物体的精确位置不影响类别"
  • 色彩抖动 → 告诉模型"光照和颜色变化不影响类别"
  • 同义词替换 → 告诉NLP模型"具体用词不影响语义"

这些先验知识被编码到训练数据中，模型被迫学到对这些变换不变的表征。这和Weight Decay、Dropout一样是正则化——但它的效果通常更好，因为它不是在"约束模型"，而是在"丰富数据"。

用一句话总结：数据增强是回报率最高的正则化手段。先上数据增强，再考虑加Dropout、Weight Decay。

CV基础增强配置（TorchVision）：适合大多数视觉任务的起点——RandomResizedCrop(scale=(0.8, 1.0))、RandomHorizontalFlip(p=0.5)、ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2)。这三者组合通常能带来明显的泛化提升。但注意：增强强度本身也是超参数，scale下界从0.5到0.9搜索，总有一个最适合你的数据集的平衡点。

Mixup的流形混合原理

传统数据增强（翻转、裁剪）是在单张图片内部做变换。而Mixup做了一件更激进的事：在两张图片之间做插值。

# Mixup —— 在输入空间和标签空间同时做线性插值
import numpy as np

def mixup(x1, y1, x2, y2, alpha=1.0):
    """
    x1, x2: 两张输入图片
    y1, y2: 对应的one-hot标签
    alpha: Beta分布的参数，控制混合强度
    """
    lam = np.random.beta(alpha, alpha)  # 从Beta分布采样混合系数

    # 输入混合
    x_mixed = lam * x1 + (1 - lam) * x2
    # 标签混合
    y_mixed = lam * y1 + (1 - lam) * y2

    return x_mixed, y_mixed

# α=1.0 → lam在[0,1]均匀分布（平均混合）
# α=0.5 → lam集中在0和1附近（弱混合）
# α=2.0 → lam集中在0.5附近（强混合）

Mixup的直观理解：想象两张图片——一张猫和一张狗。Mixup把它们按7:3混合后，标签也是"70%猫+30%狗"。模型学到的是在猫和狗之间的"连续空间"上做决策，而不是在离散的类别边界上。这个连续空间使得决策边界更加平滑，模型对对抗性扰动更加鲁棒。

从流形学习的角度看，数据在高维空间中分布在一个低维流形上。Mixup相当于在流形上的两个采样点之间做插值，填充了训练数据覆盖不到的"间隙"。这迫使模型学习更平滑的流形表征，而不是过拟合于稀疏的离散样本点。

为什么Mixup能提升对抗鲁棒性和校准？因为模型在训练时看到了"模糊"的样本——70%猫+30%狗。当推理时遇到一个略微像猫又略微像狗的样本，模型不会武断地给出"猫=0.999，狗=0.001"的极端置信度，而是输出更温和的概率分布。对抗样本利用的正是模型在决策边界附近的过度自信——Mixup直接在这些边界上训练，压缩了过度自信的空间。

CutMix vs Mixup

Mixup有一个潜在的问题：在输入空间做线性插值会破坏图像的局部性——猫的像素和狗的像素在空间的同一位置被平均，产生"鬼影"效果。模型不得不学会处理这种不自然的混合模式。

CutMix采取了不同的混合方式：不是像素级混合，而是区域级替换——从一张图中裁剪一个矩形区域，粘贴到另一张图的对应位置。标签同样按区域面积比例混合。

# CutMix —— 区域替换而非像素混合
def cutmix(x1, y1, x2, y2, alpha=1.0):
    lam = np.random.beta(alpha, alpha)
    # 从x1中随机裁剪一个矩形区域
    bbx1, bby1, bbx2, bby2 = rand_bbox(x1.size(), lam)
    # 用x2的对应区域替换
    x1[:, bbx1:bbx2, bby1:bby2] = x2[:, bbx1:bbx2, bby1:bby2]
    # 标签按面积比例混合
    lam = 1 - ((bbx2 - bbx1) * (bby2 - bby1) / (x1.size(-2) * x1.size(-1)))
    return x1, lam * y1 + (1 - lam) * y2

CutMix保留了原始图像的局部结构——你在图中看到的是一个清晰的猫，只是某个区域被替换成了狗的部分。这迫使模型学习组合性特征：一个对象由多个局部部分组成，即使部分区域被遮挡，仍能通过剩余部分识别出物体。

Mixup和CutMix虽然实现方式不同，但核心思想一致：迫使模型在插值/混合的数据上学习平滑的决策边界。哪个更好？取决于任务。在ImageNet分类上，CutMix通常略优于Mixup。在实践中，很多团队把两者结合起来使用。

增强强度与模型容量的匹配

数据增强不是越强越好。这里有一个重要的匹配原则：增强强度必须与模型容量相匹配。

小模型 + 强增强 → 欠拟合。小模型的表达力有限，遇到太多变换后的数据会"应付不过来"——它连原始数据的模式都没学会，更别提对变换后的数据也做正确分类了。这种情况下，应该使用较弱的增强或更保守的增强参数。

大模型 + 弱增强 → 浪费容量。大模型（如ViT、大Transformer）的表达力远超小模型，如果增强太弱，模型很快就"背下了"训练集——强大的容量配合有限的数据=过拟合。ViT通常需要比CNN更强或更多的增强（如RandAugment、MixUp、CutMix）才能发挥其容量优势。

AutoAugment / RandAugment是解决这个问题的自动化方案。AutoAugment通过强化学习搜索最优增强策略，但计算成本很高（需要在验证集上评估策略）。RandAugment简化了这一点：随机从一组增强操作中选择N个，每个操作的强度为M。N和M成为两个可搜索的超参数——简单了很多，效果却几乎一样好。

增强策略 核心思想 适用场景 强度调参
几何变换（翻转/裁剪/旋转） 改变空间位置，标签不变 CV几乎必用 裁剪scale、旋转角度
色彩抖动（ColorJitter） 改变亮度/对比度/饱和度 CV光照多样的场景 抖动幅度
Mixup 像素级线性插值+标签混合 分类、表示学习 α参数（混合强度）
CutMix 区域替换+标签按面积混合 图像分类、检测 α参数
RandAugment 自动搜索增强组合 ViT、大模型 操作数N + 强度M
同义词替换 用同义词替换文本中的词 NLP文本分类 替换比例
回译（Back-Translation） 翻译到中间语言再译回 NLP语义理解 质量最高但计算成本高
增强的底线：标签语义不变。这是数据增强最根本的约束条件。旋转90°以上会使"直立的人"变成"躺着的人"——标签语义变了。对OCR任务垂直翻转文字——文字不再可读，标签无意义。对医学影像过强的色彩抖动——病灶特征可能被抹掉。每次设计或选择增强策略时，都问自己一句话："经过这个变换后，人还能正确标注这个样本吗？"如果答案是否定的，增强过头了。

总结一下，数据增强通过告诉模型"什么是你不需要关心的"来提升泛化，是回报率最高的正则化手段。Mixup通过线性插值迫使模型学习平滑决策边界，CutMix通过区域替换保留了局部信息。增强强度必须与模型容量匹配——小模型弱增强，大模型强增强。增强不是越多越好，底线是保持标签语义不变。
