大家好，我是你们的讲师。

如果你训练过深度学习模型，你一定用过 cross-entropy loss。它是图像分类、文本分类、机器翻译、语音识别——几乎所有分类任务——的默认损失函数。但你是否想过一个问题：为什么分类问题用交叉熵而不是均方误差（MSE）？

这不是一个无聊的理论问题。2018年，一位实习生在公司跑了一个实验：把某个模型的损失函数从 cross-entropy 换成 MSE，结果模型完全无法收敛。他花了两天排查代码，最后才意识到是损失函数的问题。在深度学习中，"为什么这个损失函数有效"不是书斋里的学问——它直接关系到你的模型能不能训出来。

背后的信息论解释不仅优美，而且能指导工程实践。这一章我们就来揭开 cross-entropy loss 的神秘面纱。

从最大似然到交叉熵

深度学习中"选择损失函数"这个问题，从统计学的角度看，本质上是选择概率模型和估计方法的问题。而最大似然估计（MLE）是最自然的起点。

假设我们有一个分类模型，给定输入 x，输出在 K 个类别上的概率分布 p(y|x; θ)。对于训练集 {(x_i, y_i)}，MLE 的目标是选择 θ 最大化所有样本的似然乘积：

max θ: ∏_{i=1}^N p(y_i|x_i; θ)

取负对数（log-likelihood的负数，因为优化算法通常做最小化）：

min θ: -∑_{i=1}^N log p(y_i|x_i; θ)

现在来看交叉熵的定义。Cross-entropy H(P, Q) = -∑ P(x) log Q(x)。如果我们把真实标签分布记作 P_true（对于单个样本，one-hot 编码），把模型预测记作 P_model，那么：

H(P_true, P_model) = -∑ P_true(x) log P_model(x)

对于 one-hot 标签，P_true 在正确类别 c 处为 1，其余为 0。所以交叉熵简化为：

H(P_true, P_model) = -log P_model(类别 c)

这不就是负对数似然吗？完全正确。

关键洞察：对于 one-hot 标签，最小化交叉熵损失 = 最大化对数似然。二者数学上完全等价。这就是为什么交叉熵在分类问题中是"正确"的损失函数——它直接对应统计学中最大似然估计的原则。

这引出了一个深刻的观点：损失函数不是随意选的。如果你的模型输出的是概率分布（softmax输出），那么 cross-entropy 是 MLE 的自然结果。用 MSE 拟合概率分布，在统计学上没有对应的解释——它不是任何一个合理概率模型的 MLE。

事实上，如果你把分类问题当作回归来做（用 MSE），你实际上是在假设输出是高斯分布的均值——但分类标签是离散的类别，不是连续值。模型和数据的"概率假设"不匹配，就像用英文语法去写中文句子一样别扭。

Softmax + Cross-Entropy：天作之合

如果我们只从 MLE 的角度理解交叉熵，已经足够有说服力了。但交叉熵和 softmax 之间还有一个更深层的数学"默契"——它们组合起来的梯度形式极其简洁优美。

先回顾 softmax 函数。对于 logits 向量 z = [z_1, ..., z_K]，softmax 将其转换为概率分布：

p_i = exp(z_i) / ∑_{j=1}^K exp(z_j)

交叉熵损失 L = -∑_{j=1}^K y_j log(p_j)，其中 y 是 one-hot 标签（y_c = 1，其余为 0）。

现在来算梯度 ∂L/∂z_i。经过优美的链式法则推导：

∂L/∂z_i = p_i - y_i

就这么简单。梯度 = 预测概率 - 真实概率。当预测正确时（p_i ≈ y_i），梯度接近零，模型不再更新。这个梯度形式意味着：模型会自动调节"关注度"——它对当前预测不好的样本给予更大的更新。

现在对比 MSE + sigmoid 的情况。MSE 的梯度包含 σ(z)·(1-σ(z)) 项（sigmoid 的导数），当 σ(z) 接近 0 或 1 时导数趋近于 0，造成梯度饱和——即使预测完全错了，梯度也几乎为零，模型学不动。这就是那个实习生遇到的坑：用 MSE 训练分类器，模型卡住不动了。

用 Python 感受一下这个差异：

import numpy as np

def softmax(x):
    exp_x = np.exp(x - np.max(x))
    return exp_x / exp_x.sum()

def cross_entropy_grad(logits, y_true):
    """返回 softmax+交叉熵的梯度 ∂L/∂z_i = p_i - y_i"""
    p = softmax(logits)
    return p - y_true

def mse_grad(logits, y_true):
    """MSE+sigmoid梯度，注意sigmoid导数导致的饱和"""
    p = 1 / (1 + np.exp(-logits))  # sigmoid
    grad = 2 * (p - y_true) * p * (1 - p)
    return grad

# 模拟一个3分类问题
logits = np.array([2.0, 5.0, -1.0])
y_true_onehot = np.array([0.0, 1.0, 0.0])  # 正确类别是第2类

print(f"Logits: {logits}")
print(f"Softmax输出: {softmax(logits).round(3)}")
print(f"CE梯度: {cross_entropy_grad(logits, y_true_onehot).round(4)}")
print(f"MSE梯度: {mse_grad(logits, y_true_onehot).round(4)}")
# CE梯度: [ 0.0468 -0.0468  0.    ]
# MSE梯度: [ 0.0035 -0.0067  0.    ]

# 现在模拟预测完全错误的情况：正确类别得分很低
logits_bad = np.array([5.0, -5.0, 0.0])
y_bad = np.array([1.0, 0.0, 0.0])

print(f"\n严重错误时:")
p_bad = softmax(logits_bad)
print(f"Softmax输出: {p_bad.round(4)}")
print(f"CE梯度: {cross_entropy_grad(logits_bad, y_bad).round(4)}")
print(f"MSE梯度: {mse_grad(logits_bad, y_bad).round(4)}")
# CE梯度在错误分类上很大，MSE梯度被sigmoid导数严重压制

这个实验清楚地展示了：交叉熵梯度在任何情况下都不会饱和，而 MSE 的梯度在 sigmoid 饱和区接近于零。这就是为什么在分类问题中，交叉熵总是优于 MSE。

Label Smoothing的信息论视角

在真实应用中，纯粹的 one-hot 标签有一个问题：它鼓励模型输出极端置信度。模型变得"过度自信"——对训练样本给出 p ≈ 1 的预测，这常常导致过拟合和泛化能力下降。

Label smoothing（标签平滑）的解决方法是：不再用 one-hot 作为真实分布，而是用"软化"后的标签：

y'_i = (1 - ε) * y_i + ε / K

其中 ε 是一个小常数（通常 0.1），K 是类别数。比如，原来 one-hot 是 [1, 0, 0]，平滑后变成 [0.933, 0.033, 0.033]。

从信息论的视角看，label smoothing 在做什么？

One-hot 标签的熵是 0——完全没有不确定性。在信息论中，熵为 0 的分布是"最确定"的分布，但也是"理论上最不可能在无限数据中自然出现"的分布。Label smoothing 将目标分布的熵从 0 提升到一个正数——实际上是在告诉模型："不要对训练标签过于确信，保留一些不确定性。"

更精确地说：label smoothing 等价于在目标分布中加入了一个均匀先验。均匀分布在所有离散分布中熵最大——它是"最不确定"的分布。通过混合 one-hot 和均匀分布，我们实际上是在做最大熵正则化。

信息论视角：Label smoothing 强迫模型学习到的特征散布在更多维度上，而不是"把所有赌注押在一个维度"。它有效提高了模型的校准度（calibration）——模型输出的概率值更接近真实的正确率。这就像一个明智的投资者会分散风险，而不是把所有钱投在一只股票上。

这个技术最早在 Google 的 Inception v2 论文中提出，现在已经成了几乎所有现代分类模型的标配。BERT、ResNet、EfficientNet——全都在用。它简单到只有一行代码的改变，却能稳定提升 0.5-1 个百分点的准确率。

但注意：label smoothing 的代价是什么？它会让"蒸馏"（knowledge distillation）变得困难——因为教师模型的 soft label 变得更软了，学生模型从中学到的信息变少了。这个世界上没有免费的午餐。

VAE与KL正则化

跨过分类任务，我们来看看另一个深度学习中的重要角色——变分自编码器（VAE）——和信息论的深刻联系。

VAE 的损失函数由两部分组成：

L = -E_{q(z|x)}[log p(x|z)] + D_KL(q(z|x) || p(z))
  • 第一项是重构损失（reconstruction loss）——解码器从隐变量 z 恢复原始数据 x 的能力
  • 第二项是KL 散度——编码器的后验分布 q(z|x) 与先验分布 p(z) 之间的距离

从信息论的角度看，KL 项在做一件非常优雅的事：它在约束"从 x 传递到 z 的信息量"。回想第 5 章学过的内容——D_KL(q(z|x) || p(z)) 衡量的是，用 p(z) 近似 q(z|x) 时损失了多少信息。当 KL 散度为 0 时，q(z|x) 完全等于 p(z)——模型根本没有使用 x 的任何信息！

反过来，如果 KL 项太大，模型就是在把 x 中的大量信息编码到 z 中——但在标准 VAE 中，p(z) 通常是一个标准正态分布，它没有能力"携带"太多复杂信息。所以 KL 项天然地限制信道的容量。

这就引出了一个有趣的权衡：

  • 重构误差大 = 信息丢失太多（z 没有充分捕捉 x 的特征）
  • KL 散度太小 = 没有压缩（z 几乎就是 x 本身，没有学到有意义的表示）

这正是率失真权衡（第 19 章）的活生生的例子！VAE 本质上是一个率失真系统：z 是编码后的表示，重构误差是"失真"，KL 散度是"率"——编码所需的额外 bit 数。

β-VAE 把这个权衡推到了极致：它给 KL 项加一个权重 β：

L_β = -E_{q(z|x)}[log p(x|z)] + β · D_KL(q(z|x) || p(z))

当 β > 1 时，模型更强调压缩——隐变量 z 必须更高效地编码信息。有趣的是，β-VAE 在 β 适当时能学到解耦表示（disentangled representation）：z 的不同维度自动对应 x 的不同独立因子（比如在生成人脸时，一个维度控制肤色，另一个控制表情，第三个控制角度）。这是信息论指导神经网络设计的一个漂亮案例。

我们用一个表格来总结本章涉及的各种损失函数：

损失函数 公式 梯度特点 典型应用
均方误差（MSE） ||y - ŷ||² 线性，但+sigmoid会饱和 回归问题
交叉熵（CE） -∑ y_i log p_i p_i - y_i，永不饱和 分类问题（默认）
KL 散度 ∑ y_i log(y_i/p_i) 不对称，对 q=0 敏感 生成模型、正则化
Focal Loss -(1-p_t)^γ log p_t 降低易分类样本权重 目标检测（类别不平衡）
对比损失（InfoNCE） -log[exp(sim)/∑exp(sim)] 推拉正负样本对 对比学习（SimCLR）
工程经验：交叉熵损失的梯度 p_i - y_i 有一个漂亮的解释：它只是"你预测的概率"减去"真实的概率"——当预测正确时梯度为零，模型不再更新。这使得交叉熵在分类任务中既高效又稳定。如果你在训分类模型时遇到了 loss 不下降的问题，第一步就是确认你用的是交叉熵而不是 MSE。

这一章我们从最大似然估计出发，推导出了交叉熵损失函数，看到了它为什么是分类问题的"天然选择"。Softmax 和交叉熵的组合给出了一个简单优美的梯度形式，避免了 MSE 的梯度饱和问题。Label smoothing 从信息论角度可以被理解为最大熵正则化，而 VAE 中的 KL 项则是率失真理论在深度学习中的直接应用。

如果交叉熵是神经网络的"燃料"，那么信息瓶颈就是解释神经网络如何"压缩信息"的理论框架。下一章我们来看一个自 1999 年提出以来持续引起争议的观点：神经网络学习的过程，本质上是一个信息压缩过程。
