第21章 非线性方程求根(二)

大家好，我是你们的讲师。上一章我们学习了二分法和不动点迭代法，它们分别走"稳"和"巧"两个路线。但在实际工程中，当效率成为关键瓶颈时，我们一般会祭出非线性方程求根的"王牌武器"——Newton法（也叫Newton-Raphson法）。

有人说Newton法就像一个天才型选手：当他状态好时（初值选得好、导数值正常），他以令人瞠目结舌的速度直捣黄龙；但一旦状态不佳（初值不理想、导数接近零），他可能会以同样惊人的速度崩溃。理解Newton法的优点和陷阱，是数值分析工程师的分水岭。

21.1 Newton法（切线法）

Newton法的思想非常朴素：用线性近似代替非线性函数。给定当前近似值xk，我们在这一点上用切线来近似f(x)：

f(x) ≈ f(xk) + f'(xk)(x - xk)

令这个近似等于零，解出x作为下一个近似值：

xk+1 = xk - f(xk) / f'(xk)

这就是Newton法的迭代公式。每步需要计算一次函数值和一次导数值。实现起来极其简单：

def newton(f, fprime, x0, tol=1e-12, max_iter=100):
    """
    Newton-Raphson 求根法
    f: 目标函数
    fprime: 导数函数
    x0: 初始猜测
    """
    x = x0
    for k in range(max_iter):
        fx = f(x)
        fpx = fprime(x)

        if abs(fpx) < 1e-15:
            raise ValueError("导数接近于零，Newton法失效")

        x_new = x - fx / fpx

        if abs(x_new - x) < tol:
            return x_new, k + 1

        x = x_new

    raise RuntimeError(f"超过最大迭代次数 {max_iter}")

Newton法收敛速度演示

考虑求解f(x) = x² - 2 = 0（即求√2）。取x₀ = 2：

  • 第0步：x₀ = 2，误差 ≈ 5.86×10⁻¹
  • 第1步：x₁ = 1.5，误差 ≈ 8.58×10⁻²
  • 第2步：x₂ = 1.4166667，误差 ≈ 2.45×10⁻³
  • 第3步：x₃ = 1.4142157，误差 ≈ 2.12×10⁻⁶
  • 第4步：x₄ = 1.41421356，误差 ≈ 1.59×10⁻¹²

注意观察误差的变化：0.586 → 0.086 → 0.0025 → 2.1×10⁻⁶ → 1.6×10⁻¹²。每一次误差大致是上一次的平方！这就是二次收敛的魅力。

21.2 Newton法的几何意义与收敛性

从几何上看，Newton法就是在当前点xk处作曲线的切线，切线与x轴的交点就是xk+1。这也就是为什么Newton法又被称为切线法。

为什么Newton法收敛这么快？我们来分析误差的递推关系。设x*是精确根，ek = xk - x*。将f(x*)在xk处Taylor展开：

0 = f(x*) = f(xk) + f'(xk)(x* - xk) + ½f''(ξ)(x* - xk)²

代入Newton迭代公式xk+1 = xk - f(xk)/f'(xk)，经过简单代数运算得到：

ek+1 = -[f''(ξ) / (2f'(xk))] · ek²

这正是二次收敛的数学原因：误差的平方项主导了下一步的误差。如果f'(x*) ≠ 0且f''(x*)存在，则|ek+1| ≤ C·|ek|²，其中C = max|f''|/(2·min|f'|)。

Newton-Kantorovich定理给出了Newton法的收敛条件：在适当的假设下（f可微、f' Lipschitz连续、初始点上f'可逆），只要初始残差足够小，Newton法就收敛。该定理不要求根是单根，是一个非常强的理论保证。

21.3 Newton法的陷阱

虽然Newton法极其强大，但实际使用中有不少"坑"。我大学时第一次用Newton法解f(x) = arctan(x) = 0（根在x=0），选了x₀ = 1.5，结果迭代到约x ≈ 1.39时，下一跳竟然跳到了x ≈ -1.58，然后又弹回来——形成了2-cycle振荡！

踩坑故事：Newton法遇到鞍点

几年前我参与一个深度学习项目的优化工作。有人异想天开想用Newton法代替SGD来训练一个小型网络，认为二次收敛速度能大幅加速训练。结果损失函数完全不下降，反而卡住了。排查后发现原因：在初始化参数所在的区域，Hessian矩阵的某些特征值接近零（即出现了"鞍点"），导致Newton步长|∇²f⁻¹∇f|变得非常大，直接把参数"弹飞"到损失更大的区域。这正是Newton法在深度学习中没有成为主流的原因之一——处理鞍点时，反曲率方向会让优化器"误以为"那里是下降方向。这个故事告诉我们：算法选择要匹配问题结构，盲目追求高收敛阶可能适得其反。

Newton法常见的失败模式包括：

场景原因表现解决方法
初始猜测太差线性近似在远离根处完全失效发散到无穷或振荡先用二分法或全局搜索得到好初值
f'(xk) ≈ 0切线接近水平，xk+1被抛向远方单步跳到一个巨大值使用阻尼Newton法（加步长因子λ）
重根情况f'(x*) = 0，二次收敛降为线性收敛极慢改用修正Newton法（乘以因子m，m为重根重数）
周期振荡函数图像恰好在两个点之间形成往复永远不会收敛换初始猜测或换方法

针对导数接近零的问题，一个实用的改进是阻尼Newton法：

xk+1 = xk - λ · f(xk)/f'(xk)，0 < λ ≤ 1

其中λ通过线性搜索(line search)确定，比如Armijo准则保证函数值的充分下降。这提高了Newton法的全局收敛性。

21.4 弦截法与抛物线法

Newton法需要提供解析导数f'(x)，但很多时候导数难以计算或者计算成本太高。这时，我们可以用数值差商代替导数。

弦截法(Secant Method)用最近两次的函数值来近似导数：

f'(xk) ≈ (f(xk) - f(xk-1)) / (xk - xk-1)

xk+1 = xk - f(xk) · (xk - xk-1) / (f(xk) - f(xk-1))

几何上讲，弦截法用经过(xk-1, f(xk-1))和(xk, f(xk))两点的割线与x轴的交点作为下一个近似。有趣的是，虽然弦截法只需要计算函数值不需要导数，它的收敛阶仍然达到p ≈ 1.618（黄金分割比！）。这比线性收敛好得多，但又不及Newton法的二次收敛。

def secant(f, x0, x1, tol=1e-12, max_iter=100):
    x_prev = x0
    x_curr = x1
    f_prev = f(x_prev)
    f_curr = f(x_curr)

    for k in range(max_iter):
        if abs(f_curr - f_prev) < 1e-15:
            raise ValueError("差商接近零，弦截法失效")

        x_next = x_curr - f_curr * (x_curr - x_prev) / (f_curr - f_prev)

        if abs(x_next - x_curr) < tol:
            return x_next, k + 1

        x_prev, x_curr = x_curr, x_next
        f_prev, f_curr = f_curr, f(x_next)

    return x_curr, max_iter

Müller法(抛物线法)更进一步：它用过三个点(xk-2, xk-1, xk)的二次抛物线来逼近函数，然后求出抛物线与x轴的交点作为下一个近似。优点是可以处理复根——因为二次方程总有两个根（可能是复数），Müller法能自动选择离当前点较近的那个。这对于在实轴上迭代时追上复数根很有用。

实际工程中的建议

在实际项目中，我通常的推荐策略是一个"组合拳"：先用二分法或网格搜索得到一个粗略的根区间，然后再用Newton法（如果导数容易计算）或弦截法（如果导数不好算）进行精细迭代。如果需要高可靠性（比如金融定价引擎），我还会加上"保护逻辑"：如果Newton法的步长超过合理范围，自动降级为二分法走一步。这种"混合策略"既能保证收敛，又能利用高阶收敛的速度优势。

好了，让我们总结一下本章的核心内容。Newton法是数值求根的"效率之王"，但需要小心处理其陷阱（初始猜测差、导数接近零、周期振荡）。当导数不便计算时，弦截法和Müller法提供了良好的折中。在实际使用中，"先粗后精、混合策略"往往是工程中最可靠的选择。
