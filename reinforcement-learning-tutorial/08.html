第八章：值函数逼近

同学们好！从第一章到现在，我们讨论的所有算法都有一个共同的前提：用一个表格（Table）来存储每个状态或每个状态-动作对的价值。在上一章的悬崖行走问题中，状态无非就是网格的几十个位置，表格完全够用。但现实世界中的强化学习问题，状态空间往往大到让人绝望。

这就是本章要解决的问题——当状态空间太大、甚至连续时，我们怎么表示价值函数？答案是值函数逼近（Value Function Approximation）。

8.1 维度诅咒：为什么表格不再适用

先来看几个数字：

  • 围棋的棋盘是19×19，每个位置有落黑子、落白子、空三种状态，理论状态数约为 3361 ≈ 10172 种。这比可观测宇宙中的原子总数（约 1080）还要多出 1092 个数量级。
  • Atari 2600 游戏的屏幕是 210×160 像素，每个像素 128 种颜色，理论状态数约为 12833600。这个数字大到已经失去了实际意义。
  • 即使是一个简单的机器人控制任务，关节角度和角速度是连续值，每个维度取 100 个离散值，6 个自由度就有 10012 = 1024 个状态。

这种现象被称为"维度诅咒"（Curse of Dimensionality）——随着状态空间维度的增加，所需的样本数呈指数级增长。用表格方法，每个状态都是独立的，彼此之间没有任何信息共享。访问过一个状态，学到的知识无法泛化到未访问过的相邻状态。

解决办法是：用一个参数化函数来逼近真实的价值函数。我们不再记录每个状态的值，而是学习一组参数 w，使得 V(s; w) ≈ Vπ(s)。这样，当智能体访问了一个状态后，参数更新会影响到所有"相似"状态的估值——这就是泛化（Generalization）的力量。

直觉类比： 表格方法就像一本电话黄页，每个条目都是一个独立的人名和号码。函数逼近方法就像一个线性回归模型——你不需要记住每个人的身高，只需要知道"身高 ≈ 0.6 × 父亲身高 + 0.4 × 母亲身高"这个公式，就能对没见过的家庭做出合理预测。

8.2 线性值函数逼近

线性值函数逼近是最简单、最经典的参数化方法。它的形式是：

V(s) ≈ wT φ(s) = Σi=1d wi φi(s)

这里 φ(s) = [φ1(s), φ2(s), ..., φd(s)]T 是状态 s 的特征向量（Feature Vector），w 是待学习的权重向量。线性逼近的更新规则可以直接从梯度下降推导出来。

回顾 TD(0) 的更新公式：V(s) ← V(s) + α[R + γ V(s') - V(s)]。在函数逼近的视角下，我们要更新的是参数 w 而不是直接更新 V(s)：

w ← w + α [R + γ V(s'; w) - V(s; w)] ∇w V(s; w)

对于线性逼近 ∇w V(s; w) = φ(s)，因此：
w ← w + α [R + γ wTφ(s') - wTφ(s)] φ(s)

这个更新规则非常优雅——TD 误差乘以特征向量，再以学习率 α 调整权重。它告诉我们：哪个特征对当前状态的预测贡献大，谁的权重就被调整得多。

特征构造方法

线性逼近的效果高度依赖于特征工程。以下是几种经典的特征构造方法：

方法 描述 优点 缺点
多项式特征 将原始状态的分量及其乘积作为特征，如 φ(s) = [1, s1, s2, s1s2, s12, s22] 实现简单，能捕捉非线性关系 维度随阶数指数增长
Tile Coding（瓦片编码） 将连续状态空间用多个偏移的网格覆盖，每个网格是一个"瓦片"，状态落入的瓦片集合构成特征 天然适合泛化，计算高效，可控制分辨率 需要手动设计网格偏移和粒度
傅里叶基（Fourier Basis） 使用余弦函数作为基函数，φi(s) = cos(π i · s) 理论上能逼近任意平滑函数，高频细节可控 对非平滑函数表现不佳
径向基函数（RBF） 使用高斯核作为特征，φi(s) = exp(-||s - ci||2 / 2σ2) 局部响应，光滑逼近 中心点选择困难，计算成本高

其中 Tile Coding 是强化学习中最常用的线性特征构造方法之一。它的核心思想很简单：把连续空间划分成多个交错的网格，每个状态会同时激活多个"瓦片"。因为网格之间有偏移，所以即使两个状态在同一个网格中落在不同的瓦片组合里，也能被区分开来。

线性方法的优势在于：简单、稳定、理论性质好。在适当的条件下（如学习率递减），线性半梯度 TD 方法被证明是收敛的。但它的局限性也很明显：表达能力有限——特征需要人工设计，无法自动学习有用的表示。

8.3 非线性值函数逼近

既然线性方法受限于特征工程，那我们自然想到：能不能让模型自己学习特征？答案是用神经网络作为值函数逼近器。

在非线性逼近中，价值函数 V(s; w) 由一个神经网络表示，w 是所有层的权重。梯度下降的更新规则在形式上与线性方法完全相同：

w ← w + α [R + γ V(s'; w) - V(s; w)] ∇w V(s; w)

区别在于 ∇w V(s; w) 现在是神经网络反向传播计算出的梯度，而不是简单的特征向量 φ(s)。神经网络的多个隐藏层可以自动学习出层次化的特征表示——底层识别简单模式，高层组合出抽象概念。

但事情并没有这么简单。当我们用神经网络做值函数逼近时，会撞上 Sutton 所说的"致命三要素"（Deadly Triad）：

致命三要素：当以下三个条件同时出现时，算法可能发散！

① 函数逼近（Function Approximation） — 用一个参数化函数来近似价值函数。
② 自举（Bootstrapping） — 用当前估值作为更新目标的一部分（如 TD 方法）。
③ 异策略学习（Off-Policy Learning） — 学习的目标策略与行为策略不同。

三者中的任意两个搭配通常都是安全的，但当三者凑齐——函数逼近 + 自举 + 异策略——算法就可能出现发散或不收敛。

一个简单的反例是 Baird's Counterexample（Baird 1995）：一个 7 状态的 MDP，使用线性值函数逼近 + TD(0) + 异策略。即使学习率很小，权重也会震荡并趋于无穷大。原因在于自举和异策略的组合使得贝尔曼算子不再是压缩映射，函数逼近器放大而不是缩小了误差。

这告诉我们：不能盲目地把神经网络塞进 TD 更新中。后面我们会看到 DQN 是如何通过经验回放（Experience Replay）和目标网络（Target Network）来驯服这些不稳定性的。

踩坑故事：从"完美收敛"到"梯度爆炸"

我有个学生，在 CartPole 环境上用一个小型神经网络做 Q-Learning。一开始一切正常，V 值稳步下降，策略越来越好。他觉得大功告成，就去吃了个午饭。

回来一看——loss 变成了 NaN，所有权重都是 inf。他检查了学习率（0.001，没问题）、网络结构（两层 64 神经元，没问题）、激活函数（ReLU，没问题）。百思不得其解。

后来我们一行一行地排查，发现了一个隐蔽的 bug：他在计算 TD 目标时使用了同一个网络既计算当前 Q 值又计算目标 Q 值。也就是说，每一步更新中，目标值 R + γ max Q(s', a') 里的 Q(s', a') 也在被当前梯度影响。等价于"自己评估自己的分数"，形成了正反馈循环：目标不断被拉高，网络权重跟着狂飙，最后爆炸。

教训：用神经网络做值函数逼近时，一定要把目标网络和在线网络分开——要么每隔若干步才同步一次目标网络（DQN 的做法），要么锁定目标网络的梯度（stop-gradient）。千万不要让目标值跟着当前更新步一起抖动！这个坑我见过至少五个同学踩进去，你现在知道了，就别再踩了。

8.4 半梯度方法（Semi-gradient Methods）

细心的同学可能已经注意到了——在上一节的更新公式中，我们计算梯度时只对 V(s; w) 求导，而没有对目标值 R + γ V(s'; w) 求导。也就是说，我们把目标值当成了一个常数。

这是有意为之的。如果我们也对目标值中的 w 求导，更新公式就变成了"全梯度方法"（Full Gradient Method）：

全梯度 TD:
w ← w + α [R + γ V(s'; w) - V(s; w)] [∇wV(s; w) - γ ∇wV(s'; w)]
半梯度 TD:
w ← w + α [R + γ V(s'; w) - V(s; w)] ∇wV(s; w)

半梯度方法忽略了 ∇wV(s'; w) 这一项，也就是"假装目标值中的参数 w 不会变化"。这样做有两个后果：

  • 优点：训练更稳定，计算更简单。线性半梯度 TD 在一定条件下被证明是收敛的。
  • 缺点：更新是有偏的——我们没有在最小化一个真正的目标函数。半梯度方法不下降任何函数的梯度（它不是一个真正的梯度下降）。

为什么在大多数实际应用中我们仍然使用半梯度？原因很简单：全梯度方法在实际中通常表现更差。公式中多出的 -γ∇wV(s'; w) 项会使更新方向变得不稳定，尤其是在函数逼近的表达能力较强时（如神经网络）。在 Deep Q-Network 中，目标网络的使用本质上也是为了减轻全梯度方法的不稳定性——通过冻结目标网络的参数，我们实际上是在强制使用半梯度更新。

半梯度方法不仅适用于 TD(0)，也适用于 SARSA 和 Q-Learning。以半梯度 SARSA 为例：

Q(s,a; w) ← Q(s,a; w) + α [R + γ Q(s',a'; w) - Q(s,a; w)] ∇wQ(s,a; w)

这里的 a' 是下一个状态中实际采取的动作（同策略），梯度只对 Q(s,a; w) 计算。这就是半梯度的精髓——在效率和稳定性之间做权衡。

8.5 特征工程实战

虽然神经网络可以自动学习特征，但在实际项目中，特征工程仍然是最重要的环节之一。好的特征能让最简单的线性方法达到接近神经网络的性能，而坏的特征能让最深的网络也学不到东西。

以下是我在实战中总结的几个原则：

归一化（Normalization）

RL 中的状态变量往往量纲不同——位置可能是 0~100，速度可能是 -5~5，角度可能是 -π~π。如果不做归一化，量纲大的特征会主导梯度更新，导致训练不稳定。

推荐的做法是将每个维度缩放到 [0, 1] 或 [-1, 1]：

# 状态归一化示例
def normalize_state(state, low, high):
    # low, high 是每个维度的最小值和最大值
    return (state - low) / (high - low + 1e-8)

One-Hot 编码与 Tile Coding

对于离散特征（如游戏中的物体类型、地图上的地形），使用 one-hot 编码。对于连续特征，Tile Coding 是 RL 中最经典、最实用的编码方式之一：

import numpy as np

class TileCoder:
    """简化版 Tile Coding 实现"""
    def __init__(self, dims, tiles, tiling_offset=4):
        self.dims = dims            # 状态空间维度
        self.tiles = tiles          # 每个维度的瓦片数量
        self.offset = tiling_offset # 网格偏移量
        self.num_tilings = 2 ** dims  # 网格层数
        self.table_size = self.num_tilings * (tiles ** dims)

    def get_features(self, state):
        """将连续状态映射到特征索引"""
        indices = []
        for tiling in range(self.num_tilings):
            tiling_offset = tiling * self.offset
            tile_indices = []
            for dim in range(self.dims):
                scaled = (state[dim] + tiling_offset) * self.tiles
                index = int(np.floor(scaled))
                tile_indices.append(index)
            # 将多维索引展平为一维
            flat_index = tiling * (self.tiles ** self.dims)
            for dim in range(self.dims):
                flat_index += tile_indices[dim] * (self.tiles ** dim)
            indices.append(flat_index)
        return indices  # 返回激活的瓦片索引列表

特征复杂度的权衡

在设计特征时，你会面临一个永恒的权衡：

特征策略 优点 风险 适用场景
简单特征 + 线性方法 训练快，可解释性强，理论保证好 欠拟合，无法处理复杂关系 状态维度低，特征明确的控制任务
复杂特征 + 线性方法 表达能力提升，训练仍较快 特征维度爆炸，过拟合 需要平衡性能与训练速度
原始输入 + 神经网络 自动特征学习，最强表达能力 训练慢，不稳定，需要大量数据 高维输入（图像、语音），数据充足

一个实用的建议是：先从最简单的特征和最快速的线性方法开始，建立一个性能基线。如果不够好，逐步增加特征复杂性或切换到神经网络。不要一上来就用 100 层 ResNet 做 CartPole——这就像用火箭筒打蚊子。

本章小结

  • 维度诅咒使得表格方法在大规模问题中不可行，必须使用函数逼近来泛化到未访问的状态。
  • 线性值函数逼近 V(s) ≈ wTφ(s) 是最简单的方法，更新规则为 w ← w + α δ φ(s)。常用特征构造方法包括 Tile Coding、多项式特征、傅里叶基等。
  • 非线性值函数逼近使用神经网络自动学习特征表示，但面临"致命三要素"（函数逼近 + 自举 + 异策略）带来的发散风险。
  • 半梯度方法通过忽略目标值对参数的依赖来获得稳定性，是实际应用中的标准做法。
  • 特征工程在 RL 中仍然至关重要——归一化、合适的编码方式、以及特征复杂度的谨慎权衡，往往比盲目增加网络层数更有效。

值函数逼近为我们打开了通往大规模强化学习的大门。下一章，我们将学习一种完全不同的方法——策略梯度（Policy Gradient），它不通过值函数来间接决定动作，而是直接优化策略本身。再往后，就是深度强化学习的重头戏——DQN（Deep Q-Network），它将深度神经网络与 Q-Learning 结合，在 Atari 游戏上创造了令人瞩目的成就。拭目以待！
