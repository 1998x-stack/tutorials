26.1 从监督到自监督 — 标注数据瓶颈与范式转变

大家好，欢迎来到第26章。在前面25章中，我们学习的几乎所有方法都依赖一个共同的前提：大量高质量的标注数据。ImageNet的1400万张人工标注图片、COCO的250万标注实例、Cityscapes的精细像素标注——这些数据集成就了深度学习，但它们的标注成本高得惊人。一张图像分割的像素级标注需要30-90秒，一个中等规模的数据集标注费用轻松达到数十万美元。

我亲身经历过这个痛苦。2020年在一家医疗AI公司，我们要做一个眼底病变检测系统。眼科医生标注一张OCT图像需要15分钟——因为病变区域微小到只有几十个像素。我们凑了3位医生标注了两个月，花了40万人民币，才拿到5000张标注图像。这点数据对于训练一个深度模型来说杯水车薪。更致命的是，换一种型号的OCT设备采集的图像，模型性能直接掉一半。我们迫切地需要一个方案：能不能不用这么多标注数据，也能学到好的视觉表征？

这就是自监督学习（Self-Supervised Learning, SSL）要解决的问题。它的核心理念极其优雅：从数据本身构造监督信号。不需要人工标注，让模型从图像自身的结构、关系或变换中学习。比如给模型看"同一张图像的两个不同裁剪版本"，让模型学会它们属于同一张图——这天然就是一个监督信号，而且是免费的。

自监督学习的崛起是CV领域近五年最重要的范式转变之一。它使得我们可以利用互联网上海量的无标注图像进行预训练，然后在具体任务上用很少的标注数据做微调。Google的Noisy Student、Facebook的DINO、OpenAI的CLIP、Meta的SAM——这些前沿模型的核心共性就是：大规模的、无监督的预训练 + 轻量级的任务适配。本课程在第1章中提到的"基础模型时代"，其技术内核正是自监督学习。这一章，我们就深入这个激动人心的主题。

26.2 对比学习 — SimCLR与MoCo

如果说自监督学习有一座最耀眼的里程碑，那一定是对比学习（Contrastive Learning）。它的核心思想可以用一句话概括：拉近相似的样本，推开不相似的样本。在自监督场景中，"相似的样本"指的是同一张图像经过不同数据增强后的两个视图，"不相似的样本"则来自不同图像。

2020年，Google Research提出的SimCLR（A Simple Framework for Contrastive Learning of Visual Representations）是这一方向的里程碑。它的流程极其简洁：对每张输入图像，随机应用两种不同的数据增强（随机裁剪、颜色抖动、高斯模糊等），得到两个增强视图；然后用同一个编码器（通常是ResNet）提取特征；最后用一个对比损失函数——NT-Xent Loss（Normalized Temperature-scaled Cross Entropy Loss）——最大化正样本对的相似度，最小化负样本对的相似度。

方法核心创新负样本策略编码器ImageNet Top-1
SimCLR v1简单直接的数据增强+对比Batch内所有其他样本ResNet-5069.3%
SimCLR v2更大的网络+蒸馏Batch内所有其他样本ResNet-152 (3x)79.8%

SimCLR的一个关键限制是：负样本的数量严重依赖batch size。如果你只有4张GPU，batch size设不了多大，负样本太少，训练效果就大打折扣。这也是为什么原始SimCLR用了多达4096的batch size和128块TPU。对于绝大多数研究者和工程师来说，这个资源门槛过于奢侈。

就在SimCLR发布的同一年，何恺明团队提出的MoCo（Momentum Contrast）用一种优雅的方式解决了这个问题。MoCo的核心创新有两个：动态字典（Queue）和动量编码器（Momentum Encoder）。简单来说：MoCo维护一个队列，里面存放着历史batch编码过的特征，作为负样本池。这个队列可以轻松容纳几万个负样本，而且不占用额外的显存。动量编码器则通过指数移动平均（EMA）的方式更新，确保队列中的特征保持一致性。

# SimCLR 对比损失的核心实现（PyTorch概念代码）
        def nt_xent_loss(z_i, z_j, temperature=0.5):
            # z_i, z_j: (N, D) 两路增强视图的特征
            # N: batch size, D: 特征维度
            # 拼接所有特征
            z = torch.cat([z_i, z_j], dim=0)  # (2N, D)

            # 计算所有pair之间的余弦相似度矩阵
            z_norm = F.normalize(z, dim=1)
            sim_matrix = torch.mm(z_norm, z_norm.T) / temperature  # (2N, 2N)

            # 构造标签: 正样本是 (i, i+N) 和 (i+N, i)
            labels = torch.arange(N).to(z.device)
            labels = torch.cat([labels, labels], dim=0)

            # 屏蔽自身（对角线）
            mask = ~torch.eye(2 * N, dtype=torch.bool).to(z.device)

            # 计算交叉熵损失
            loss = F.cross_entropy(sim_matrix[mask].view(2*N, -1), labels)

            return loss

在实际工程中，MoCo的队列机制有一个精妙但容易被忽视的细节：动量系数的设置。如果动量系数太大（接近1），队列中的特征更新太慢，无法适应编码器的快速变化；如果太小，特征一致性被破坏，损失无法收敛。我们在一次项目中直接套用了MoCo v2的默认参数（动量=0.999），但在小batch size（64）下训练了一个ViT-Small，结果损失曲线反复震荡。后来将动量调到0.995，并将学习率从0.03降到0.01，才稳定下来。

💡 实践要点：对比学习的batch size和动量系数需要联动调整。小batch size时建议降低动量系数（0.99-0.995），让编码器更新更快速；同时减小温度参数（如从0.2降到0.1），增加对难负样本的惩罚。这些调参经验在论文中很少提到，但在实际项目中至关重要。

26.3 自蒸馏方法 — DINO与iBot

对比学习需要精心设计负样本，而2021年提出的DINO（DIstillation with NO labels）则走上了另一条路：自蒸馏（Self-Distillation）。DINO的思想来自知识蒸馏——让一个"教师"网络引导一个"学生"网络学习，但这里的教师和学生共享相同的架构，且教师通过EMA的方式从学生更新。

DINO最令人震撼的发现是：自监督训练的ViT模型，竟然自发地学会了语义分割的能力。当研究者可视化DINO预训练的ViT的自注意力图时，他们发现注意力头自动聚焦到了物体的完整轮廓——比如一张图片中的动物、人物甚至物体的精确边界。这种能力完全没有用分割标注训练过，完全是自监督学习带来的"涌现"性质。

后续的iBot（image BERT pre-training）进一步结合了自蒸馏和掩码图像建模。它的思路是：对图像patch进行随机掩码，让模型从可见patch重建被掩码patch的特征。iBot在ImageNet上的线性探测准确率达到了82.3%，而且在下游任务（尤其是密集预测任务如分割、检测）上的迁移表现显著优于对比学习方法。

🔮 涌现现象：为什么自监督ViT能学到语义分割能力？一个直观的解释是：为了准确地对比两个视图（或预测被掩码的部分），模型必须理解图像的语义结构——而物体的边界和区域正是这种结构的自然体现。DINO的注意力可视化揭示了：不需要任何分割标注，模型自己就学会了"什么是物体"。这个发现直接启发了后来SAM的设计思路。

26.4 掩码重构方法 — MAE

如果说DINO是在探索"不需要负样本"的路径，那么MAE（Masked Autoencoders）则走了一条更极致的路——直接回归到自编码器的传统。MAE由何恺明团队在2021年底提出，其核心思路非常直接：随机遮盖图像中的大部分patch，然后让模型从可见patch重建被遮盖的部分。你可能觉得这听起来像BERT的视觉版本——没错，MAE确实是将掩码语言建模的思路移植到了视觉领域。

但MAE有一个重要的创新：75%的超高掩码率。在NLP中，BERT的掩码率是15%；而在MAE中，四分之三的图像patch被遮盖掉了。为什么这么高？何恺明在论文中的解释非常值得一提：图像和语言的信息密度不同。语言几乎每个词都携带语义信息，而图像中存在大量的空间冗余——天空的相邻像素非常相似，草地的大片区域纹理一致。因此，只有去掉足够多的信息，模型才被迫去学习真正的语义理解，而不是简单的像素插值。

MAE的另一个关键设计是非对称编解码器架构：编码器只处理可见patch（仅25%的patch），大大降低计算量；解码器处理全部patch（包括掩码token），负责像素重建。这种不对称设计使得MAE可以训练非常大的编码器。在实验中，只训练可见patch的编码器比全部patch送入编码器快了3.5倍以上，显存占用减少了近一半。

# MAE 核心流程的概念实现
        def mae_forward(images, encoder, decoder, mask_ratio=0.75):
            # images: (B, 3, H, W)
            # 1. 将图像分割成patch并嵌入
            patches = patchify(images)  # (B, N, D)  N=H//p * W//p

            # 2. 随机生成掩码（75%被掩码）
            N = patches.shape[1]
            num_keep = int(N * (1 - mask_ratio))
            noise = torch.rand(B, N)
            ids_shuffle = torch.argsort(noise, dim=1)
            ids_keep = ids_shuffle[:, :num_keep]
            ids_mask = ids_shuffle[:, num_keep:]

            # 3. 编码器只处理可见patch
            x_keep = gather(patches, ids_keep)
            encoded = encoder(x_keep)  # 只编码25%的patch

            # 4. 解码器处理全部patch（可见+掩码token）
            x_full = scatter(encoded, ids_keep) + mask_tokens(ids_mask)
            decoded = decoder(x_full)  # 重建所有patch

            # 5. 只在掩码patch上计算MSE损失
            loss = MSE(decoded[ids_mask], patches[ids_mask])

            return loss

我自己的一个实操经验：2022年我们用MAE在遥感图像数据集上做预训练。卫星图像的特殊性在于——云层遮挡非常常见，而且被遮挡区域可能占据图像的50%以上。标准的MAE在75%掩码率下效果并不好，因为可见patch太少，卫星图像的地物结构无法被有效捕捉。我们将掩码率调到50%，并加入了地理空间特有的数据增强（多光谱通道裁剪、地理坐标旋转），最终在土地利用分类任务上的准确率提升了4.7%。这个经历告诉我们：自监督方法的成功高度依赖对数据特点的理解和适配，照搬标准配置往往不是最优解。

⚠️ 踩坑提醒：MAE的大掩码率设计是为了减少图像的空间冗余，但这并不意味着对所有场景都适用。对于纹理密集的图像（如遥感、病理、显微图像），适当降低掩码率（如50-60%）往往效果更好。另外，MAE的解码器在训练后通常被丢弃，只保留编码器用于下游任务——不要忘记这一步，否则推理效率会大打折扣。

26.5 视觉-语言对比学习 — CLIP

如果说前面讨论的方法都还局限于"纯视觉"的自监督学习，那么CLIP（Contrastive Language-Image Pre-training）则打开了一扇全新的大门——用自然语言作为监督信号来训练视觉模型。

2021年OpenAI推出的CLIP，核心思路极其简洁：用4亿张互联网图文对训练两个编码器（一个图像编码器、一个文本编码器），通过对比学习让配对的图像-文本特征相互靠近，不配对的相互远离。训练完成后，CLIP展现出了一个惊人的能力——零样本分类：你不需要对新数据集的类别做任何训练，只需要把类别的文本描述（如"a photo of a cat"）送入文本编码器，然后将测试图像送入图像编码器，计算图像特征与所有类别文本特征的相似度，取最匹配的类别作为预测结果。

能力说明典型应用
零样本分类无需训练数据，直接用文本类别推理新类别快速分类、开放集识别
多模态检索文本搜图像、图像搜文本图文搜索、电商匹配
视觉特征提取作为通用视觉骨干用于下游任务目标检测、分割、图像描述
图像-文本对齐理解图像和文本的语义对应关系图文一致性判断、内容审核

在CV领域，CLIP的意义怎么强调都不为过。它证明了一个重要的范式：语言作为监督信号，可以替代大量的人工标注。互联网上已经存在海量的图-文配对数据——新闻图片、电商商品图、社交媒体的图文帖——这些"免费的标注"远比人工标注来得丰富和多样。这也是为什么CLIP训练集4亿图文对的规模远超任何人工标注数据集。

但CLIP也有其局限性。我们在一个工业缺陷检测项目中使用CLIP做零样本分类——产品缺陷的文本描述（如"light scratch on metal surface"）过于细微，CLIP的文本编码器难以精确捕捉这种细粒度差异。此外，CLIP对图像-文本配对的"组合性"理解有限，比如它很难区分"a dog chasing a cat"和"a cat chasing a dog"这类语义对立但视觉元素相同的描述。这些问题后来催生了E-CLIP、SigLIP等改进工作，也推动着视觉-语言模型的持续进化。

💡 直觉：CLIP告诉我们一个深刻的道理——有效的视觉表征不仅来自图像本身，更来自图像与语言的对齐。视觉和语言是人类认知的两大支柱，它们的结合可能才是通向通用视觉理解的正确路径。

26.6 通用分割基础模型 — SAM

2023年，Meta AI发布的Segment Anything Model（SAM）将自监督学习和基础模型的概念推向了新的高度。SAM是一个通用的图像分割模型，能够分割图像中的任何物体——是的，任何物体——通过简单的提示（prompt），如点选、框选或自动全图分割。

SAM的三个关键组成部分构成了它强大的能力：

  • 任务：可提示分割（Promptable Segmentation）——将分割重新定义为一个提示驱动的任务。给定任何提示（前景点、背景点、边界框、粗略掩码），模型输出对应的分割掩码。如果提示不明确（比如只点了一个点但物体边界有歧义），模型输出多个候选掩码。
  • 数据：数据引擎（Data Engine）——SAM使用了超过10亿个掩码进行训练，这是历史上最大的分割数据集。但更有创新性的是它的数据引擎流程：先人工标注少量掩码训练初始模型，然后用模型自动生成大量掩码，人工修正后再训练更强版本，如此迭代。这个"模型辅助标注"的飞轮是SAM规模化数据的关键。
  • 架构：SAM由一个图像编码器（基于MAE预训练的ViT）、一个提示编码器和一个轻量级的掩码解码器组成。图像编码器对每张图只运行一次，之后可以配合无限的提示进行交互——这种设计使得SAM在交互式分割场景中极其高效。

SAM的发布在CV社区引起了巨大反响。传统上，分割是一个高度专业化的任务——你需要为不同的物体类型、不同的图像域（医学、遥感、自然图像）训练不同的模型。SAM用一个统一的架构解决了所有分割问题。它的出现标志着CV正在从"专用模型"时代走向"基础模型+提示工程"的新范式。

在一次自动化标注项目中，我们用SAM辅助人工标注：标注员只需要在物体上点一个点，SAM就能生成精准的轮廓掩码，标注员只需确认或微调。传统标注一张图像的物体需要3分钟，用SAM辅助后压缩到20秒以内。效率提升了9倍。这就是基础模型改变工程实践的典型例子。

# SAM推理的核心流程（简化概念代码）
        import torch
        from segment_anything import sam_model_registry, SamPredictor

        # 加载预训练SAM模型
        sam = sam_model_registry["vit_h"](checkpoint="sam_vit_h_4b8939.pth")
        predictor = SamPredictor(sam)

        # 输入图像
        image = cv2.imread("street_scene.jpg")
        predictor.set_image(image)

        # 输入提示：在行人中心点一个点
        input_point = np.array([[450, 320]])
        input_label = np.array([1])  # 1=前景点

        # 生成掩码（SAM输出多个候选掩码处理歧义）
        masks, scores, logits = predictor.predict(
            point_coords=input_point,
            point_labels=input_label,
            multimask_output=True  # 输出多个候选
        )

        # 取置信度最高的掩码
        best_mask = masks[scores.argmax()]

        # 掩码覆盖了行人的完整轮廓
        # 整个过程只需要一个点和一次前向传播

回看这一章，从对比学习的"推拉"哲学（SimCLR、MoCo），到自蒸馏的师生共舞（DINO、iBot），到掩码重建的极简主义（MAE），再到语言引导的范式革命（CLIP）和通用分割的里程碑（SAM）——自监督学习的底层线索清晰可见：CV正在从"需要人工告诉模型什么是重要的"转向"让模型自己发现什么是重要的"。这一转变的深远影响才刚刚开始显现。

那么，当视觉模型学会了从无标注数据中理解世界，下一步自然是让它们与其他模态——语言、声音、甚至3D信息——交互。下一章，我们将进入多模态与生成模型的世界，探索视觉与语言融合的前沿进展。敬请期待！
