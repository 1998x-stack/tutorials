第9章：filter() 与数据筛选

做数据和淘金很像：你挖到的矿石（原始数据）通常含金量很低，需要用水流冲洗掉大量泥沙，才能得到闪闪发光的金粒。filter() 就是你的淘金筛子——它让你保留想要的，剔除不想要的。

在 datasets 库中，filter 是 map 的亲密搭档。map 负责"变换"，filter 负责"筛选"。两者组合起来，你可以构建出任意复杂度的数据清洗管线。但 filter 比 map 更危险——因为错误的筛选可能会在你不经意间破坏数据分布，而分布偏移的代价往往要到模型训练结束后才会暴露。这一章我们既要学会怎么用 filter，更要学会"谨慎地"用 filter。

核心认知：filter 不只是"扔掉坏数据"，它本质上是在重新定义你的数据分布。每次调用 filter，你都应该问自己：被扔掉的那部分数据和保留的部分，在分布上有什么系统性的差异？这个差异会不会影响模型的泛化能力？

filter() 方法基础

filter 的函数签名非常简洁——比 map 简单得多：

from datasets import Dataset

def filter(
    self,
    function,              # 返回 bool 值的函数
    with_indices=False,    # 是否传入索引
    input_schema=None,     # 输入格式约束
    keep_in_memory=False,  # 结果保留在内存
    load_from_cache_file=True,  # 是否使用缓存
    cache_file_name=None,  # 自定义缓存文件
    num_proc=None,         # 多进程数量
    fn_kwargs=None,        # 额外参数
    desc=None,             # 进度条描述
):

核心逻辑极其直观：对每个样本执行 function(example)，如果返回 True 则保留该样本，返回 False 则丢弃。

from datasets import load_dataset

dataset = load_dataset("imdb", split="train")
print(f"原始样本数: {len(dataset)}")  # 25000

# 最简单的 filter：保留 label=1（正面评价）
positive = dataset.filter(lambda x: x["label"] == 1)
print(f"正面评价数: {len(positive)}")  # 12500（正好一半）

# filter 也支持 num_proc 加速
long_reviews = dataset.filter(
    lambda x: len(x["text"]) > 500,
    num_proc=4
)
print(f"长评论数: {len(long_reviews)}")

filter 和 map 一样，支持 with_indices=True，此时函数接收两个参数：样本和索引。这在基于位置筛选场景中很实用：

# 保留索引为偶数的样本（模拟降采样）
even_samples = dataset.filter(
    lambda example, idx: idx % 2 == 0,
    with_indices=True
)

批量 filter 模式

与 map 不同，filter 的 batched 参数不是直接支持的。但你可以通过 map 结合条件筛选来实现类似的效果：先用 map 生成一个布尔列，再用 filter 保留对应样本：

# 两步法实现"批量 filter"
def compute_keep_flag(batch):
    # 批处理计算是否保留
    batch["_keep"] = [
        len(text) > 100 and "spoiler" not in text.lower()
        for text in batch["text"]
    ]
    return batch

# 第一步：map 生成保留标记
dataset = dataset.map(compute_keep_flag, batched=True)
# 第二步：filter 按标记筛选
dataset = dataset.filter(lambda x: x["_keep"])
# 第三步：清理标记列
dataset = dataset.remove_columns(["_keep"])
小技巧：两步法虽然看起来繁琐，但最大的好处是你可以先跑 map 生成标记列，检查标记的正确性（比如统计分布），确认无误后再执行 filter。这比在 filter 函数里直接写复杂逻辑更容易调试。

多条件组合筛选

实际项目中的筛选条件很少只有一个。你需要同时满足多个条件，或者满足任意一个条件即可。Python 的逻辑运算符就是你的组合工具：

from datasets import load_dataset

dataset = load_dataset("imdb", split="train")

# AND 条件：长文本 + 正面评价
def is_quality_review(example):
    return (
        len(example["text"]) >= 200          # 有足够内容量
        and example["label"] == 1             # 正面评价
        and "spoiler" not in example["text"].lower()  # 无剧透
    )

quality = dataset.filter(is_quality_review)
print(f"质量评价数: {len(quality)} / {len(dataset)}")

# OR 条件：要么很短要么很长
def is_extreme_length(example):
    return len(example["text"]) < 100 or len(example["text"]) > 2000

extreme = dataset.filter(is_extreme_length)

链式 filter 也是常用的写法——每个 filter 只负责一个条件，代码更清晰：

# 链式 filter：每个条件独立
pipeline = (
    dataset
    .filter(lambda x: len(x["text"]) >= 200)      # 长度下限
    .filter(lambda x: x["label"] == 1)             # 正面
    .filter(lambda x: "movie" in x["text"].lower()) # 包含特定词
)

按列值范围过滤

对于数值型的特征列，你可以构建区间过滤器。这在处理表格类数据集时尤其常见：

# 假设有一个包含评分的数据集
# dataset = load_dataset("your_dataset")
# 保留评分在 3.5 到 5.0 之间的样本
# filtered = dataset.filter(
#     lambda x: 3.5 <= x["rating"] <= 5.0
# )
实战：清洗 IMDB 里少于 50 字的评价

IMDB 数据集中的一些评论只有一句话甚至几个词，比如 "Great movie!" 或 "Terrible."。这些样本携带的信息量极少，对训练分类器几乎没有帮助，反而会增加噪声。

from datasets import load_dataset
import re

dataset = load_dataset("imdb", split="train")

def has_meaningful_content(example):
    text = example["text"]
    # 去掉 HTML 标签后计算实际字数
    clean = re.sub(r'<[^>]+>', '', text)
    words = clean.split()
    # 保留至少 50 个单词的评论
    return len(words) >= 50

# 看看过滤掉了多少
original_count = len(dataset)
filtered = dataset.filter(has_meaningful_content)
removed = original_count - len(filtered)
print(f"原始: {original_count}, 过滤后: {len(filtered)}, 移除: {removed} ({removed/original_count*100:.1f}%)")

索引级操作：select / take

filter 是按"条件"选择数据，而 select 和 take 是按"位置"选择数据。它们是 filter 的有效补充，在某些场景下更直接：

from datasets import load_dataset

dataset = load_dataset("imdb", split="train")

# select：按索引列表选择
subset = dataset.select([0, 5, 10, 100, 1000])
print(subset[0])  # 第 0 条
print(len(subset))  # 5

# take：取前 N 条
first_10 = dataset.take(10)

# 随机采样（不重复）
import random
random.seed(42)
rand_indices = random.sample(range(len(dataset)), 1000)
random_subset = dataset.select(rand_indices)

select 还有一个隐藏技能——它可以用来做"反向筛选"：先选出不要的索引，然后取差集：

# 假设我们要去掉前 1000 条
total = len(dataset)
keep_indices = list(range(1000, total))
subset = dataset.select(keep_indices)
print(f"去掉前1000条后: {len(subset)}")
性能提示：select 和 take 不会触发 map 式的缓存。它们直接操作 Arrow 索引，本质上只是"视图"，不复制数据。因此无论选择多少条，select 都是 O(1) 时间——这只是指针操作。但需要注意，select 返回的 Dataset 对象上的 filter 操作仍然会重新扫描整个数据集（取决于实现细节）。

select 与 filter 的互补关系

操作 选择依据 返回类型 典型场景
filter() 条件函数（True/False） Dataset（行过滤） 质量过滤、标签筛选、长度控制
select() 显式索引列表 Dataset（视图） 按需取子集、随机采样、反向排除
take() 前 N 条 Dataset（视图） 快速查看样本、小批量调试
shuffle() + select() 随机化后取前 N Dataset 随机采样固定大小子集

数据标注质量筛选

在监督学习中，标注错误是灾难性的。单个错误标注可能会影响模型对整类样本的决策边界。filter 可以用来做标注质量的初步筛查：

# 假设你的数据集有多个标注者的标注结果
# 结构：{"text": "...", "label_a": 1, "label_b": 2, "label_c": 1}
# 你需要排除那些标注者之间不一致的样本

def check_annotator_agreement(example):
    labels = [example["label_a"], example["label_b"], example["label_c"]]
    # 至少有 2/3 一致
    from collections import Counter
    most_common = Counter(labels).most_common(1)[0]
    # 最共同的标注至少要占 2/3
    return most_common[1] >= 2

筛选策略与数据偏倚

现在要讲 filter 最危险的一面。过度筛选举手之间就会破坏你的数据分布。来看一个真实的教训。

踩坑故事：某 NLP 团队在做文本分类任务时，为了控制输入长度，filter 掉了所有超过 512 token 的文本。逻辑看起来无懈可击——模型最长上下文就是 512，超长的文本反正也要被截断。但他们没注意到的是：长文本和短文本在语义分布上是有偏的。技术文档、学术论文、法律合同普遍较长（>512 token），而社交媒体帖子、产品评论普遍较短。过滤掉长文本后，训练集里几乎没有了"正式文体"的样本。结果模型在评测集上表现不错，但上线后面对企业用户的正式文档时完全无法工作——模型从来没在训练中见过这种长度的文本。团队花了三周才定位到问题：不是模型架构的问题，是数据筛选把长文本分布整个砍掉了。

筛选前后分布对比

每次执行 filter 后，都应该检查筛选前后的分布变化：

from datasets import load_dataset
import numpy as np

dataset = load_dataset("imdb", split="train")

# 筛选前：计算文本长度分布
lengths_before = np.array([len(x["text"]) for x in dataset])
print(f"筛选前: 均值={lengths_before.mean():.0f}, 中位数={np.median(lengths_before):.0f}, "
      f"标准差={lengths_before.std():.0f}")

# 做筛选：长度在 200-2000 之间
filtered = dataset.filter(
    lambda x: 200 <= len(x["text"]) <= 2000
)

# 筛选后：再次检查分布
lengths_after = np.array([len(x["text"]) for x in filtered])
print(f"筛选后: 均值={lengths_after.mean():.0f}, 中位数={np.median(lengths_after):.0f}, "
      f"标准差={lengths_after.std():.0f}")

print(f"移除了 {(1 - len(filtered)/len(dataset))*100:.1f}% 的数据")

保留筛选日志

任何生产级的 filter 操作都应该记录日志：谁在什么时候执行了什么条件，过滤掉了多少样本，分布发生了什么变化。这在排查模型效果回退时至关重要：

import json
import datetime

class FilterLogger:
    def __init__(self):
        self.logs = []

    def apply_filter(self, dataset, filter_fn, name, **kwargs):
        before = len(dataset)
        result = dataset.filter(filter_fn, **kwargs)
        after = len(result)
        entry = {
            "timestamp": datetime.datetime.now().isoformat(),
            "name": name,
            "before": before,
            "after": after,
            "removed": before - after,
            "removed_pct": round((before - after) / before * 100, 2),
        }
        self.logs.append(entry)
        print(f"[{name}] 移除 {entry['removed']} 条 ({entry['removed_pct']}%)")
        return result

    def save(self, path="filter_log.json"):
        with open(path, "w") as f:
            json.dump(self.logs, f, indent=2, ensure_ascii=False)
最佳实践：构建数据管线时，把每条 filter 操作当作一个"设计决策"来对待——记录它、度量它、审计它。一个好的数据团队和一般的团队之间的区别，往往就在于是否系统性地追踪了数据的每一次"删减"。
