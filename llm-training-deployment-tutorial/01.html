第1章 大模型训练全景图

大家好，我是你们的讲师。今天我们来聊一个既让人兴奋又让人头疼的话题——大模型训练。说它兴奋，是因为我们正处在AI历史上最激动人心的时代：从GPT横扫全球到DeepSeek震惊业界，大模型的能力每半年就上一个台阶。说它头疼，是因为这个领域的知识体系庞大得像一座迷宫：预训练、SFT、RLHF、MoE、量化、推理优化……每一个词背后都是一整套工程学问。

我见过太多团队踩进同一个坑：GPU集群到位了，框架选好了，数据集也攒了几TB，结果训练了三个星期，loss就是不降。有人发现是tokenizer没训练好，有人发现数据里有30%的噪声，还有人发现并行策略选错了，GPU利用率不到10%。这些问题的根因只有一个——对训练全链路缺乏整体认知。所以这门课的第一章，我们先不碰代码，上来画一张"全景图"，让你脑子里建立起整个训练-部署流程的地图。

1.1 从GPT到DeepSeek：大模型演进简史

要理解"训练"这件事的本质，先得知道模型是怎么"长"成今天这样的。我把关键里程碑梳理一下。

GPT时代：Scaling Law的验证之路。2018年，OpenAI发布GPT-1，参数量仅1.17亿，放在今天连"小模型"都算不上。但它证明了generative pre-training的可行性——先用无监督数据预训练，再用有监督数据微调。2019年的GPT-2把参数量推到15亿，展示出令人震惊的文本连贯性。2020年的GPT-3直接干到1750亿参数，"in-context learning"能力横空出世——你给几个例子它就能完成任务，无需更新权重。

开源生态崛起。2023年Meta发布LLaMA系列，用更小的模型（7B/13B/70B）达到了接近GPT-3.5的效果，关键在于训练数据的质量和配比。LLaMA催生了整个开源微调生态（Alpaca、Vicuna等），让"训练自己的模型"从巨头的专利变成了中等团队也能触碰的目标。

DeepSeek与中国力量的崛起。2024年底到2025年，DeepSeek-V3以MoE（Mixture of Experts）架构实现了惊人的训练效率——总参数量671B，但每个token只激活37B参数，训练成本仅为同级别dense模型的十分之一。2025年初的DeepSeek-R1进一步引入了"推理增强"范式，通过RLVR（Reinforcement Learning with Verifiable Rewards）让模型学会长链思考。这条技术路线彻底改变了行业对"训练"的理解。

年份 模型 参数量 关键创新
2018 GPT-1 117M Generative Pre-training
2019 GPT-2 1.5B 零样本泛化能力
2020 GPT-3 175B In-context Learning
2022 ChatGPT (GPT-3.5) 175B RLHF对齐训练
2023 GPT-4 1.8T (传闻) 多模态、推理能力
2023 LLaMA系列 7B-70B 开源、数据质量驱动
2024 DeepSeek-V3 671B (MoE) MoE架构、极致训练效率
2025 DeepSeek-R1 671B (MoE) RLVR推理增强

看这张表你会发现一个规律：每一次能力跃升，背后都是训练技术的突破——要么是数据规模扩大（GPT-2→GPT-3），要么是对齐方法的革新（RLHF），要么是架构的优化（MoE）。这正是我们这门课要深入拆解的东西。

1.2 预训练→后训练→部署：三阶段范式

今天的大模型训练，几乎所有的技术栈都围绕一个三阶段范式展开。不管你用的是Llama还是Qwen，不管你在做中文模型还是代码模型，这个框架都适用。

第一阶段：预训练（Pre-training）。这是最"烧钱"的阶段。模型在数万亿token的文本上做next token prediction——给你前文，预测下一个词。这一阶段产出的叫base model（基座模型）。Base model的能力就像一张白纸上有了基本轮廓：它懂语法、懂常识、懂知识，但不会"对话"——你问它问题，它可能会继续续写而不是回答。

第二阶段：后训练（Post-training）。这又分为两个子阶段。首先是SFT（Supervised Fine-Tuning），用高质量的指令-回答数据让模型学会遵循指令。然后是偏好对齐（RLHF或DPO），让模型学会输出符合人类偏好的回答——比如更有帮助、更诚实、更安全。产出的是aligned model（对齐模型），也就是你平时用到的Chat模型。

第三阶段：部署（Deployment）。训练完了，模型得"上车"才能产生价值。这里涉及量化（Quantization，把FP16转成INT4以降低显存）、推理优化（KV cache、Flash Attention）、服务架构（vLLM、TensorRT-LLM）、以及端侧部署。部署阶段的优化甚至会反过来影响训练阶段的决策——比如训练时就要考虑推理时的效率。

阶段 输入 输出 目标 典型成本
预训练 万亿级原始文本 Base Model 知识获取、语言能力 数百万~数亿美元
后训练 指令数据、偏好数据 Aligned Model 指令遵循、安全对齐 数万~数十万美元
部署 Aligned Model 推理API/端侧模型 高效推理、降低成本 可变（取决于规模）

核心认知：很多团队只关注"训练"这一个环节，忽视了部署优化的反作用。我见过一个团队花300万美元训了一个70B模型，结果上生产时发现推理延迟太高，不得不重新蒸馏成8B，精度掉了不少。如果你一开始就想好了部署场景，在训练阶段就可以为蒸馏、量化做铺垫——比如在SFT阶段就用知识蒸馏的loss。

1.3 为什么从零训练一个模型？

在GPT-4和Claude能力如此强大的今天，你可能会问：为什么不直接调用API，非要自己训练？这个问题问得好，实际上90%的场景确实不需要从零训练。但剩下的10%，恰恰是这门课的核心受众。

场景一：领域特化。通用模型在医疗、法律、金融等高度专业领域表现不佳。比如让GPT-4读一份中国法院判决书，它可能分不清"裁定"和"判决"的程序区别。领域特化模型需要注入大量专业语料，数据和训练策略都得定制。

场景二：数据隐私。某些行业（银行、保险、政务）的数据不能出域。你不能把客户保单数据传到OpenAI的服务器上做fine-tuning。唯一的办法是在自己的GPU集群上、在自己的VPC内完成全部训练。

场景三：成本控制。SaaS API调用在规模化后非常昂贵。如果你每天有上亿次请求，自建模型虽然在训练阶段烧钱，但推理阶段的边际成本远低于API调用费。一年省下来的钱够买几块H100了。

说个真实案例。去年有个创业团队找到我，他们想做一个法律咨询AI。团队很年轻，上来就说"我们要从零训一个70B的模型"，说这话时眼神里闪着光。我问他们三个问题：你的语料库多大？数据清洗管道搭好了吗？你做过了小规模实验吗？一个都答不上来。结果他们直接上了512张A100跑了两个月，花了300多万，训出来的模型连基础的法律条文检索都做不好。拆解原因：数据里有大量重复的法条解读文章，模型overfit在了少数高频模式上，泛化能力极差。

踩坑案例：这个团队犯了三个典型错误。第一，没有先做小规模实验验证数据质量（花几百美元跑个小模型就能暴露问题）。第二，没有建立数据质量评估流程，导致"垃圾进垃圾出"。第三，训练过程中只盯着loss下降而没看下游任务指标。300万美元买的教训——训练不是跑通了就万事大吉，全链路的质量控制才是关键。

是否从零训练，我建议你用这个判断标准：SFT和alignment解决不了的问题，才考虑pre-training或mid-training。先用API调优（prompt engineering），然后是fine-tuning（PEFT或full SFT），再然后是mid-training（领域继续预训练），最后才是从头pre-training。每一步成本差一个数量级，别一上来就王炸。

1.4 2026年技术全景

截至2026年中，大模型训练领域有几个不可忽视的趋势，我们在后续章节会逐一展开，这里先做个鸟瞰。

MoE成为主流架构。稠密模型（Dense Model）正在被稀疏MoE架构取代。DeepSeek-V3的成功证明了MoE可以在相同FLOPs预算下获得显著更好的质量。2026年几乎所有新发布的大模型都采用了MoE，这给训练和部署带来了新挑战：负载均衡、Expert容量规划、通信模式优化。

合成数据深度渗透训练链路。高质量人工标注数据越来越贵（一段数学推理标注可能要几十美元），而合成数据（Synthetic Data）已经成为预训练、SFT、RLHF各阶段的核心数据来源。如何控制合成数据的质量、防止模型坍缩（Model Collapse），是2026年的热点课题。

推理增强训练（RLVR / Process Reward Model）。DeepSeek-R1在2025年引爆了"训练侧推理增强"的方向。传统的SFT只教模型"怎么答"，而RLVR教模型"怎么想"。通过verifiable reward（比如数学答案正确与否、代码能否通过测试）来训练模型的思考过程，这一范式正在从数学/代码扩展到通用领域。

端侧部署迎来爆发。高通、联发科、Apple都在SoC中集成了NPU，端侧跑7B模型已经不是幻想。但端侧部署意味着模型必须足够小（量化到INT4甚至INT2），同时又不能太笨。训练阶段就考虑"被蒸馏""被量化"的模型（Quantization-Aware Training），而非训完再压缩，是2026年的主流做法。

1.5 本课程学习路线图

这门课一共28章，分成六个部分。下面是每部分的内容定位和你完成后能掌握的能力。

第一部分：训练基础（第1-4章）。全景图、数据工程、Tokenizer、Transformer架构。学完你能说清楚"训练一个模型需要准备好什么"。

第二部分：分布式训练核心（第5-11章）。分布式基础、优化器、混合精度、3D并行、MoE、训练稳定性、Scaling Laws。学完你能搭起一个大规模训练集群并跑通训练流程。

第三部分：数据与持续训练（第12-14章）。长上下文、合成数据、Mid-training。学完你能在基座模型基础上注入领域知识。

第四部分：后训练对齐（第15-20章）。SFT、RLHF、DPO、PEFT、RLVR、Agent训练。学完你能把base model变成好用的assistant model。

第五部分：高级训练技术（第21-22章）。多模态训练、模型合并与蒸馏。学完你能处理视觉-语言多模态场景，以及用蒸馏技术做模型瘦身。

第六部分：部署与工程实践（第23-28章）。量化、推理优化、服务引擎、端侧部署、集群运维、完整实战项目。学完你能把模型部署到生产环境并持续迭代。

学习建议：如果你是初学者，建议按顺序从第1章学到第28章。如果你已经有一些训练经验，可以重点看第8-11章（大规模训练）和第23-25章（推理部署）。每章都配有真实案例和踩坑经验，看的时候多想想"这个问题我遇到过吗"。
