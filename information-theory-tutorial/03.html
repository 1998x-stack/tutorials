大家好，我是你们的讲师。今天我们进入信息论最核心的概念——自信息（self-information）和熵（entropy）。

你有没有想过——天气预报说"明天会下雨"和"明天太阳会升起"，哪句话包含更多信息？直觉告诉你，前者更"有信息量"，因为后者太显然了。如果天气预报每天都告诉你"明天太阳会升起"，你很快就会忽略它——这条消息没有提供新信息。

这就是自信息的直觉：一个事件越不可能发生，它发生时带给我们的信息量就越大。"北京明天会下雨"的概率如果是0.1，那么当它真的发生，你获得的"惊讶值"就比较大。"太阳明天会升起"的概率是接近1的，所以不管播报多少遍，它也带不来多少新信息。

但这里有一个关键问题：我们谈论的是单个事件的信息量。那如果我们想要知道整个随机变量的平均信息量呢？那就要取所有可能事件信息量的期望——这就是熵。

自信息量：量化惊讶

自信息量（self-information）定义为：

I(x) = -log p(x)

其中 p(x) 是事件 x 发生的概率。

这个定义有几个关键特性：

  • 单调递减：概率越小，信息量越大。p → 0 时 I → ∞；p = 1 时 I = 0。
  • 非负性：log p(x) ≤ 0，所以 I(x) ≥ 0。
  • 可加性：对于独立事件，I(x,y) = -log p(x)p(y) = -log p(x) - log p(y) = I(x) + I(y)。两个独立事件的总信息量等于各自信息量之和。

对数的底数决定了信息量的单位：

  • 以2为底 → 单位是bit（比特）——最常用
  • 以自然对数e为底 → 单位是nat（奈特）——在理论推导中常用
  • 以10为底 → 单位是hartley（哈特利）——偶尔出现

整个教程中，除非特别说明，我们默认用log₂。这意味着所有信息量都以bit为单位。

举个具体的例子：抛一枚公平硬币，p(正) = 0.5，那么 I(正) = -log₂(0.5) = 1 bit。抛一枚严重偏倚的硬币，p(正) = 0.99，那么 I(正) = -log₂(0.99) ≈ 0.014 bit——正面朝上几乎毫无悬念，所以信息量接近于0。但如果那1%的反面发生了，I(反) = -log₂(0.01) ≈ 6.64 bit——一个罕见事件带来大量信息。

这就是"黑天鹅事件"的信息论解释：罕见事件的信息量极大。当一只黑天鹅出现时，你获得的信息远大于看到一只白天鹅——这就是为什么"意料之外"的消息总是最引人注目。

熵：不确定性的平均度量

自信息量衡量的是一个具体事件的信息量。但如果我想知道一个随机变量整体上有多"不确定"呢？那就需要将所有可能事件的自信息量取期望——这就是熵。

H(X) = E[-log p(X)] = -∑_{x∈X} p(x) log p(x)

熵 H(X) 是随机变量 X 不确定性的平均度量。它是信源的平均信息量，也是无损压缩的理论下限。

一个特殊但非常重要的情况是二元熵函数（binary entropy function）。当随机变量只有两个取值（正/反、是/否、0/1）时，设 p 为其中一个取值的概率，则：

H₂(p) = -p log₂ p - (1-p) log₂ (1-p)

这个函数在 p = 0.5 时取最大值 1 bit，在 p = 0 或 p = 1 时取最小值 0。它的形状是对称的、单峰的——就像一个倒扣的碗。如果你画出来，会发现当 p 趋近0或1时，熵趋近0——因为结果几乎是确定的，不确定性很小。当 p = 0.5 时，熵最大——你完全不确定结果。

熵的物理起源也非常有趣。这个词来自克劳修斯（Clausius）在热力学中定义的"熵"——它是系统混乱程度的度量。香农从数学家冯·诺依曼那里得到了建议："就叫它熵吧——一方面因为热力学已经有了同样的公式，另一方面人们不知道熵是什么，所以你在辩论中会有优势。"这个轶事虽然后来被查证可能不准确，但物理熵和信息熵确实共享了同一套数学结构。

联合熵与条件熵

熵的概念可以自然地推广到多个随机变量。

联合熵 H(X, Y) 衡量一对随机变量 (X, Y) 的联合不确定性：

H(X, Y) = -∑_{x∈X} ∑_{y∈Y} p(x,y) log p(x,y)

直观上，H(X, Y) 是对 (X, Y) 联合分布的不确定性度量。如果 X 和 Y 独立，那么 H(X, Y) = H(X) + H(Y)。

条件熵 H(Y|X) 衡量的是"已知 X 之后，Y 还剩多少不确定性"：

H(Y|X) = ∑_{x∈X} p(x) · H(Y|X=x) = -∑_{x∈X} ∑_{y∈Y} p(x,y) log p(y|x)

条件熵最重要的性质是链式法则：

H(X, Y) = H(X) + H(Y|X) = H(Y) + H(X|Y)

这条公式的直观意义：联合不确定性 = 先知道X的不确定性 + 知道X后Y还剩的不确定性。这完美呼应了我们在第1章讲的"信息减少不确定性"——知道了X之后，Y的不确定性从 H(Y) 降低到了 H(Y|X)，减少的量就是互信息 I(X;Y)，我们下一章会讲。

链式法则对任意数量的变量都适用：

H(X₁, X₂, ..., Xₙ) = H(X₁) + H(X₂|X₁) + H(X₃|X₁,X₂) + ... + H(Xₙ|X₁,...,Xₙ₋₁)

这个公式在信源编码中至关重要——它告诉我们，我们可以逐个符号地编码，每一步都利用前面符号的信息来减少不确定性。

熵的计算实例

理论说够了，我们做几个具体的计算来感受熵：

例1：公平硬币

p(正) = 0.5, p(反) = 0.5

H = -0.5 log₂ 0.5 - 0.5 log₂ 0.5 = 0.5 + 0.5 = 1 bit

例2：偏倚硬币（p=0.9）

p(正) = 0.9, p(反) = 0.1

H = -0.9 log₂ 0.9 - 0.1 log₂ 0.1 = 0.137 + 0.332 = 0.469 bit

熵从1降到了0.469——不确定性减少了，因为结果更可预测了。

例3：天气预报——三个状态

假设城市天气有三种状态：晴天(p=0.6)、多云(p=0.3)、雨天(p=0.1)

H = -0.6 log₂ 0.6 - 0.3 log₂ 0.3 - 0.1 log₂ 0.1

H = 0.442 + 0.521 + 0.332 = 1.295 bits

如果是均匀分布（每个概率1/3），H = log₂ 3 ≈ 1.585 bits。实际的1.295低于最大值，说明分布不均匀减少了不确定性。

下表是不同概率分布的熵值对照：

分布 概率 熵 (bits) 直觉
公平硬币 p=0.5 1.000 完全不确定
偏倚硬币 p=0.9 0.469 比较确定
严重偏倚硬币 p=0.99 0.081 几乎确定
公平6面骰子 均匀1/6 2.585 log₂ 6
k符号均匀 均匀1/k log₂ k 最大熵

Python实现熵的计算：

import numpy as np

def entropy(probs, base=2):
    """
    计算离散概率分布的熵。

    参数:
        probs: 概率数组，需满足 sum(probs) = 1
        base: 对数底数，默认2（单位bit）

    返回:
        熵值
    """
    probs = np.array(probs)
    # 只对非零概率计算，因为 0*log(0) = 0
    non_zero = probs > 0
    return -np.sum(probs[non_zero] * np.log(probs[non_zero])) / np.log(base)

# 测试不同分布
distributions = {
    "公平硬币": [0.5, 0.5],
    "偏倚硬币": [0.9, 0.1],
    "严重偏倚": [0.99, 0.01],
    "公平骰子": [1/6] * 6,
    "天气预报": [0.6, 0.3, 0.1],
}

for name, probs in distributions.items():
    h = entropy(probs)
    print(f"{name}: H = {h:.3f} bits")

# 输出：
# 公平硬币: H = 1.000 bits
# 偏倚硬币: H = 0.469 bits
# 严重偏倚: H = 0.081 bits
# 公平骰子: H = 2.585 bits
# 天气预报: H = 1.295 bits
常见误区：熵是随机变量的属性，不是某个具体事件的属性！初学者常犯的错误：说"这个事件有高熵"——应该说"这个分布有高熵"。一个事件可以用自信息量I(x)来描述，但熵是整个分布的平均不确定性。就像你不能说"今天温度是15°C所以气候是温带的"——一个数据点不能代表整个分布。

熵在机器学习中也有直接的应用。例如，在决策树算法中，每个节点的分裂准则是信息增益（Information Gain），即父节点熵减去子节点加权平均熵。算法本质上是：在每一步选择一个特征，使得条件熵 H(目标|特征) 最小，从而最大限度地减少对目标的不确定性。

这一章我们学习了信息论中最重要的概念：自信息量 I(x) = -log p(x) 量化了单个事件的"惊讶程度"，而熵 H(X) = -∑ p(x)log p(x) 量化了整个分布的平均不确定性。我们还学习了联合熵和条件熵，它们描述的是多变量场景下的不确定性。

关键 takeaways：

  • 自信息量 I(x) 与概率负对数成正比——事件越罕见，信息量越大
  • 熵 H(X) 是自信息量的期望，是分布不确定性的平均度量
  • 二元熵函数 H₂(p) 在 p=0.5 时取最大值1 bit
  • 链式法则 H(X,Y) = H(X) + H(Y|X) 连接了联合熵和条件熵

熵告诉我们一个变量"有多不确定"，但它不能告诉我们两个变量之间的共享信息——那是互信息的任务。
