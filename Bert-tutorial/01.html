第1章 NLP预训练发展简史

大家好，我是你们的讲师。在深入BERT之前，我们必须先回答一个根本性问题：为什么预训练会成为NLP的主流范式？这一章，我们将穿越NLP预训练的技术编年史，从2013年的word2vec到2018年BERT横空出世，看清每一次范式跃迁背后的驱动力。

如果你在2016年进入NLP领域，你会看到一个完全不同的世界——每个任务都需要从头训练模型，标注数据是唯一的生产资料。而今天，一个预训练BERT-base模型加几十行微调代码就能在大多数任务上达到85%+的baseline。这五年间发生了什么？

1.1 前预训练时代：词的离散表示

在深度学习介入NLP之前，词被表示为one-hot向量——一个维度等于词表大小的稀疏向量，其中只有一个位置为1。虽然简单，但存在两个致命缺陷：维度灾难（词表20万则向量20万维）和语义鸿沟（"猫"与"狗"的距离和"猫"与"计算机"的距离完全相同）。

2013年，Google的Tomas Mikolov团队发表了word2vec，这是NLP历史上最被低估的革命之一。word2vec的核心思想看似简单——用一个浅层神经网络预测词的上下文（CBOW）或用上下文预测中心词（Skip-gram），但训练结束后，隐藏层的权重矩阵就成了每个词的稠密向量表示。

关键洞察：word2vec证明了语言中存在可被无监督学习捕获的几何语义结构——"国王 - 男人 + 女人 ≈ 女王"这个经典类比并不是魔法，而是向量空间中方向的线性可加性。这是"预训练"思想在NLP中的第一次成功。

1.2 静态词向量的黄金时代

word2vec之后，一系列静态词向量方法相继涌现，它们在思想上共享同一个范式：用一个固定向量表示每个词，与上下文无关。

方法年份核心创新训练信号
word2vec2013CBOW/Skip-gram架构局部窗口共现
GloVe2014全局词共现矩阵分解全局共现统计
FastText2016子词n-gram信息字符级形态特征

这些方法各有千秋。GloVe（Stanford, 2014）通过分解词-词共现矩阵来学习向量，同时利用了全局统计信息和局部上下文；FastText（Facebook, 2016）更进一步把每个词拆成字符n-gram，解决了OOV（未登录词）问题。

但所有这些方法共享一个根本性局限：一个词只有一种表示。无论上下文是什么，"bank"这个词永远是同一个向量——不管它指的是河岸还是银行。这就是我们所说的"静态"词向量问题。

实用建议：在2026年的今天，静态词向量并未完全过时。如果你的任务数据量极小（< 1000样本），词表固定且领域特定（如工业故障代码分类），FastText仍然是比BERT更轻量的合理选择。

1.3 ELMo：第一缕动态曙光

2018年初，Allen AI发布了ELMo（Embeddings from Language Models），这是NLP预训练范式的分水岭。ELMo的核心理念是：词向量不应是静态的，而应是整个句子的函数。

具体做法：用一个双向LSTM作为语言模型，前向LSTM从左到右阅读文本，反向LSTM从右到左阅读文本，两者独立训练后拼接。对于一个输入句子，ELMo取每一层LSTM的隐藏状态，加权求和作为每个token的上下文化表示。

# ELMo 的核心计算（简化版）
# 对于句子中的每个token t：
h_forward  = ForwardLSTM(tokens[:t+1])     # 从左到右
h_backward = BackwardLSTM(tokens[t:])      # 从右到左
ELMo(t)    = γ * (w0*x + w1*h₁ + w2*h₂)   # 层加权融合

ELMo在六项NLP任务上刷新了SOTA，精度提升2-5个百分点。它证明了两个关键事实：(1) 大规模无监督语言模型预训练是有用的；(2) 深层网络的不同层捕获了不同层次的语言特征（底层=句法，顶层=语义）。

常见误解：很多人以为ELMo就是"双向LSTM"，但ELMo的前向和反向LSTM是独立训练、独立运行的——它们并非真正的双向上下文建模。前向LSTM看不到后面的词，反向LSTM看不到前面的词，只是最后把两个方向的结果拼接了起来。真正的双向建模要等到BERT。

1.4 GPT-1：自回归预训练的探索

2018年6月，OpenAI发布了GPT-1（Generative Pre-Training）。与ELMo使用LSTM不同，GPT-1采用了Transformer的Decoder部分作为模型骨架，用单向（从左到右）的语言模型作为预训练任务：预测每一个token时，只能看到它前面的词。

GPT-1的工作流程：先在BooksCorpus（7000本未出版书籍）上做无监督预训练，再为具体下游任务添加任务相关的输出层做有监督微调。这与后来BERT的"预训练+微调"范式已经非常接近。

但GPT-1有明显的方向性缺陷——单向语言模型无法利用待预测词右侧的上下文信息。在某些需要完整句子理解的任务上（如句子对判断、实体关系抽取），这会导致性能上限。不过，这个"缺陷"后来被OpenAI变成了优势——正是单向的特性让GPT系列在生成任务中大放异彩。

1.5 2018年秋天：BERT降临

2018年10月，Google发布了BERT论文（arXiv:1810.04805）。这篇仅16页的论文带来了三个根本性创新：

  1. Masked Language Model (MLM)：随机"遮盖"输入的15% token，用整个句子的上下文来预测被遮盖的词——实现了完全的双向上下文建模，这是ELMo和GPT都做不到的。
  2. Next Sentence Prediction (NSP)：输入一对句子，判断第二句是否真的是第一句的后继——捕捉句子之间的语义关系，这对QA和NLI至关重要。
  3. 统一微调框架：同一套预训练模型 + 一行输出层改动 = 适配所有NLP任务。GLUE基准集11项任务中，BERT全面超越当时所有方法，首次将GLUE分数从72.8直接拉到80.5（绝对提升7.7%）。
维度ELMo (2018.02)GPT-1 (2018.06)BERT (2018.10)
模型骨架双向LSTMTransformer DecoderTransformer Encoder
预训练任务双向LM（独立）单向LMMLM + NSP
上下文方向伪双向（拼接）单向（左→右）真正双向

GLUE分数~70~72~80.5
为什么BERT双向性如此关键？考虑这句话："我昨天去了__，买了一些苹果。"——要预测空白处的词（"超市"），右侧的"买苹果"这个信息与左侧的"去了"同样重要。单向模型无法利用右侧信息，而BERT的MLM可以同时看到两边。这种能力在中文等高语境语言中尤其关键。

1.6 预训练范式的哲学转变

回顾整个NLP预训练简史，我们看到的不只是技术的进步，更是一种范式级别的认识论转变：

  • 2013-2017：特征工程时代——词向量是"特征"，输入下游模型前先查表。模型仍然需要大量任务特定架构设计和标注数据。
  • 2018：预训练+微调时代（BERT开启）——模型不再是白纸，而是已经"读过"海量文本的语言专家。下游任务只需告诉它"你现在要做什么"，它自己知道怎么理解文本。
  • 2020-2025：极致效率时代——LoRA、蒸馏、量化让BERT从GPU奢侈品变成CPU可运行的日常工具。1.91MB的EI-BERT已经部署在支付宝8.4M日活设备上。

理解这段历史，你才能真正理解本书后续27章每项技术的动机——RoBERTa为什么删NSP？ELECTRA为什么要用判别式？ALBERT为什么要跨层参数共享？每一个设计决策背后，都是对前一代方法不足的精准回应。

学习提示：在阅读后续章节时，始终保持这个"问题驱动"的视角——每读到一项新技术，先问自己"它解决了前一代什么问题？"这样你学到的将不只是"BERT的变体有哪些"，而是"为什么NLP社区要这样演进预训练技术"——后者才是面试和工程中的核心竞争力。
