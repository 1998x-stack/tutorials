第17章 CAN总线通信速率：标准速率、速率与距离关系、采样点设置

各位同学，今天咱们聊聊CAN总线的通信速率。说实话，这个知识点看着简单，但坑特别多。我见过不少项目，硬件设计没问题，代码也没毛病，就是跑不起来——最后发现，全是速率和采样点设置惹的祸。

17.1 标准速率：125kbps、250kbps、500kbps、1Mbps

CAN总线支持哪些速率？嗯，标准里其实没强制规定死。但行业里形成了几个约定俗成的速率：125kbps、250kbps、500kbps、1Mbps。你想想看，为什么是这几个数？

说白了，它们跟晶振频率、分频系数有关。比如一个8MHz的晶振，经过合适的预分频和位时间配置，就能整出这些整数倍的速率来。我个人习惯，能用500kbps就别用250kbps，除非距离实在拉得太长。

标准速率 典型应用场景 位时间（μs）
125 kbps 工业控制、楼宇自动化 8.0
250 kbps 汽车车身控制、诊断系统 4.0
500 kbps 动力系统、底盘控制 2.0
1 Mbps 高速数据采集、测试系统 1.0
我的小建议： 做产品选型时，优先选500kbps。这个速率在大多数MCU上都能稳定实现，而且兼容性最好。1Mbps虽然快，但距离一超过10米就容易出问题。

17.2 速率与距离的关系

这里有个核心概念：速率越高，通信距离越短。为什么会这样？

CAN总线用的是差分信号，信号在线上传输会有衰减和延迟。速率高了，位时间就短。比如1Mbps时，一个位只有1微秒。如果总线太长，信号反射回来还没稳定，下一个位就来了——这不就乱套了吗？

我在项目中遇到过一件事：客户要求用1Mbps跑50米总线。我当时就摇头，这不可能。后来实测，30米就开始丢帧了。最后妥协到250kbps，稳稳当当跑了80米。

通信速率 最大理论距离 实际推荐距离
1 Mbps 40 m 10 - 20 m
500 kbps 100 m 50 - 80 m
250 kbps 200 m 100 - 150 m
125 kbps 500 m 200 - 400 m
注意： 上表是理论值。实际项目中，线缆质量、节点数量、终端电阻精度都会影响实际距离。我曾经在一条旧电缆上跑250kbps，30米就出错了——换了好线，直接干到150米。

17.3 采样点设置

采样点，说白了就是控制器在什么时候去读总线上的电平。这个点设得不对，再好的硬件也白搭。

CAN的位时间分成四段：同步段（Sync_Seg）、传播段（Prop_Seg）、相位缓冲段1（Phase_Seg1）、相位缓冲段2（Phase_Seg2）。采样点就在Phase_Seg1和Phase_Seg2之间。

我建议采样点设置在75% - 87.5%之间。为什么是这个范围？因为太早了，信号还没稳定；太晚了，留给下一个位的同步时间不够。

// 以500kbps为例，假设系统时钟为8MHz
// 位时间 = 16个TQ（Time Quantum）
// 采样点 = (Sync_Seg + Prop_Seg + Phase_Seg1) / 总TQ

// 推荐配置：
// Sync_Seg = 1 TQ
// Prop_Seg = 3 TQ
// Phase_Seg1 = 8 TQ
// Phase_Seg2 = 4 TQ
// 采样点 = (1 + 3 + 8) / 16 = 75%

// 另一种常用配置：
// Sync_Seg = 1 TQ
// Prop_Seg = 2 TQ
// Phase_Seg1 = 7 TQ
// Phase_Seg2 = 6 TQ
// 采样点 = (1 + 2 + 7) / 16 = 62.5%  ← 这个偏低，不推荐
核心要点： 采样点设置要遵循「三点原则」：
  • 不能太靠前（信号未稳定）
  • 不能太靠后（同步裕量不足）
  • 最好在75% - 87.5%之间

17.4 实际项目中的避坑指南

嗯，这里我总结几个实战经验，你们记一下：

  • 不同速率混用？不行。 一条CAN总线上所有节点必须用同一个速率。我曾经见过有人想用125kbps和500kbps混跑，结果总线直接瘫痪。
  • 采样点要统一。 不光速率要一样，采样点设置也要一样。不同MCU的默认采样点可能不同，比如STM32默认是75%，有些芯片默认是62.5%。不统一的话，长距离通信容易出错。
  • 距离不够？降速。 如果总线长度超过推荐值，别硬撑。降一档速率，往往比加中继器更靠谱。
  • 用示波器看波形。 我每次调试CAN，第一件事就是拿示波器看CAN_H和CAN_L的差分波形。看看上升沿、下降沿是否陡峭，有没有振铃。这个习惯帮我省了不少排查时间。
一个小技巧： 如果你不确定采样点设多少合适，可以用CAN分析仪自带的采样点计算工具。输入晶振频率和目标速率，它会自动推荐一组参数。但记住，工具只是参考，最终还是要实测验证。

好了，这一章的内容就这些。速率、距离、采样点，这三个参数是CAN总线通信的「铁三角」。任何一个设错了，通信都跑不稳。下一章咱们聊聊CAN的帧结构，那个更精彩。
