归一化层——深度学习最重要的"基础设施"

大家好，我是你们的讲师。如果让我投票选一个"过去十年对深度学习影响最大的技术"，我会选归一化层。不是Adam，不是Transformer，不是ResNet——而是归一化。因为归一化让以上所有技术都变得可训练了。

一个细节可能颠覆你的认知：原始的Transformer论文（Attention Is All You Need）使用了Post-LN（后归一化）结构，训练时需要非常精细的warmup策略。后来的改进——Pre-LN（前归一化）——让Transformer可以无需warmup直接训练，而且训练更稳定。同样一个架构，仅仅改变了归一化的位置，训练难度就天差地别。

这就是归一化的威力——它决定了你的模型是否能稳定收敛。

归一化的本质

归一化层的核心操作很简单：对中间激活值做标准化——减去均值除以标准差，再做一个可学习的仿射变换（乘以γ加上β）。公式就是一个标准化：

x̂ = (x - μ) / √(σ² + ε)
        y = γ · x̂ + β

其中μ和σ²是统计量，ε是防止除零的小常数，γ和β是可学习参数（恢复表征能力）。不同归一化方法的根本区别在于——μ和σ²在哪个维度上计算。

关于归一化为什么有效，有两个解释：

传统解释：内部协变量偏移（ICS）理论。ICS假说认为，在深层网络中，每层输入的分布在训练过程中不断变化（因为前一层参数在更新），这种分布偏移让后续层需要不断适应新的分布，拖慢了训练。归一化强制每层输入保持均值为0、方差为1的分布，消除了ICS。

现代解释：优化景观平滑理论。近年来的研究表明，ICS的解释并不完全准确。归一化真正的作用是平滑损失曲面——加了归一化后，损失函数的Lipschitz常数（梯度变化的剧烈程度）显著减小，梯度更加稳定。这使得更大的学习率可以被安全使用，从而加速收敛。这个解释也得到了大量实验支持。

无论是哪种解释，结论是一致的：归一化层让训练更稳定，允许更大的LR，加速收敛。

归一化 vs 初始化：初始化和归一化是解决"分布问题"的两层防线。初始化确保第一个step的激活值的分布合理。归一化确保整个训练过程中每层激活值的分布持续合理。初始化是起点，归一化是过程。两者互补，不能相互替代。用Kaiming的话说："He初始化 + BN是'2015-2020年间CNN训练的事实标准'。"

Batch Normalization深度解析

Batch Normalization（BN）是2015年由Google提出、开归一化之先河的方法。它在batch维度上计算均值和方差——对每个特征通道，在当前mini-batch的所有样本上计算统计量。

BN的核心公式：

对卷积层：μ_c = mean(N, H, W 维度上的 x_c)   # 跨batch、宽、高
        对全连接层：μ = mean(N 维度上的 x)          # 仅跨batch

BN在CNN中已经成为标配——几乎每个现代CNN架构（ResNet、DenseNet、EfficientNet）都内置了BN。Conv → BN → ReLU 是CNN的标准构建块。

训练 vs 推理的行为不一致：这是BN最需要注意的问题。

  • 训练时：BN使用当前mini-batch的均值和方差做标准化。梯度流经归一化层，统计量随batch变化，产生"正则化噪声"。
  • 推理时：BN使用训练期间积累的running mean和running var（滑动平均），而不是当前batch的统计量。因为推理时batch size可能很小（甚至为1），直接算当前batch的统计量是不准确的。

你必须确保在推理时调用 model.eval() 切换到BN的推理模式，否则单样本推理结果会是错误的（相当于BN退化为恒等映射，因为batch size=1时方差为0）。

SyncBN与分布式训练：在多卡训练中，每张卡只看到自己的那部分数据。默认情况下，各卡独立计算BN统计量——统计量只基于该卡上的小batch，而不是全局batch。当小batch很小时（如每卡只塞4张图），BN统计量的噪声会很大。SyncBN（同步BN）跨卡同步均值和方差，得到更准确的统计估计。PyTorch中使用 nn.SyncBatchNorm 替代 nn.BatchNorm2d。

BN与Dropout的方差偏移冲突：这是一个微妙但重要的问题。BN的正则化效果和Dropout的正则化效果可能相互干扰。具体来说：

  • 训练时，Dropout随机置零部分神经元——改变了激活值的分布
  • BN在训练时对每个mini-batch计算统计量——Dropout导致的分布变化被纳入BN的统计估计
  • 推理时，Dropout关闭——但BN使用训练时的running statistics——统计量和推理时的激活值分布不匹配

解决方案：如果同时使用BN和Dropout，Dropout应该放在BN之后。而且CNN中使用BN后，全连接层上的Dropout可省略或降为0.2-0.3，因为BN本身已经提供了正则化。

小batch场景下BN会退化。当batch size < 8时，BN的统计量噪声过大，训练不稳定。此时应该：用GroupNorm替代BN，或者用SyncBN跨卡同步，或者用LayerNorm（如果任务适合）。"小batch用BN"是新手最常见的一个错误。

Layer Normalization为何主导Transformer

Layer Normalization（LN）于2016年提出，作为BN在序列模型上的替代方案。LN在特征维度上对每个样本独立做标准化——完全不依赖batch维度：

对每个样本的每个位置：μ = mean(all features at this position)
        σ² = var(all features at this position)

LN和BN的关键差异：

  • BN：跨样本做归一化（对特征c，在所有N个样本上算mean/std）
  • LN：在样本内做归一化（对样本n，在所有C个特征上算mean/std）

LN之所以在Transformer中不可替代，有三个核心原因：

第一，变长序列。NLP中不同样本的序列长度不同。BN在batch维度上做归一化时，不同长度的序列padding不同，batch统计量不稳定。LN对每个样本独立做归一化，完全不受padding影响。

第二，逐token推理。Transformer推理时是一个token一个token生成的（自回归模式）。BN需要batch统计量——batch size=1时BN退化。LN每token独立计算，不受影响。

第三，训练推理完全一致。LN的训练和推理行为完全一致——没有running statistics的维护，没有模式切换的烦恼。你不需要在eval()和train()之间切换LN的行为。

Pre-LN vs Post-LN：这是Transformer中一个重要的设计选择：

  • Post-LN（原始Transformer）：LN放在残差连接之后。即：SubLayer → 残差相加 → LN。原始的Transformer使用Post-LN，训练需要精细的warmup。
  • Pre-LN（现代标准）：LN放在每个子层之前。即：LN → SubLayer → 残差相加。Pre-LN训练更稳定，不需要warmup也能收敛，是GPT、BERT、Llama等现代Transformer的主流配置。

此外，在大模型领域，RMS Norm正在逐渐替代完整的LayerNorm。RMS Norm去掉均值中心化，只做均方根缩放：RMS(x) = √(mean(x²))。计算更快（约15-30%提速），且实验发现在Transformer上效果几乎一致。Llama和GPT-NeoX都使用了RMS Norm。

Transformer训练的核心配置：Pre-LN + RMS Norm + AdamW + Warmup + Cosine Decay 已经是大模型训练的标配。其中Pre-LN让训练更稳定，RMS Norm提速不降质，AdamW提供自适应学习率，Warmup+Cosine Decay提供学习率调度。这个配置几乎被所有现代大语言模型采用（GPT、Llama、Mistral、DeepSeek等）。

Group Normalization及其他归一化

除了BN和LN之外，还有一些重要的归一化方法值得一提：

Group Normalization（GN）——小batch的救星。GN由FAIR的吴育昕和何恺明在2018年提出。它将通道分为若干组（group），每组内计算均值和方差。GN不依赖batch维度——batch size=1时也能正常工作。GN特别适合检测/分割等batch size很小的任务（每卡只能塞1-2张高分辨率图片）。

GN的核心操作：把输入reshape成 N × G × (C/G) × H × W，然后在组内通道（即C/G维）上做归一化。G是超参数，默认G=32（如果C<32，G=C）。

Instance Normalization（IN）——风格迁移专用。IN在每个样本的每个通道上独立做归一化。它的特性是：去掉样本间的对比度信息，保留风格信息——这正是风格迁移任务需要的。

归一化方法选型速查表

选择哪种归一化完全取决于你的任务架构和训练设置。下面是一个综合对比表：

方法 归一化维度 是否依赖batch 训练/推理一致性 小batch效果 推荐场景 常用位置
BatchNorm（BN） N × H × W（跨样本） 是 不一致（需running stats） 差（batch<8退化） CNN分类/检测 Conv后、激活前
LayerNorm（LN） C × H × W（样本内） 否 完全一致 好 Transformer/RNN/RL 子层前（Pre-LN）
InstanceNorm（IN） H × W（单通道） 否 完全一致 好 风格迁移/图像生成 Conv后、激活前
GroupNorm（GN） G组内通道×H×W 否 完全一致 好 小batch检测/分割 Conv后、激活前

一个简单的选型决策规则：

  • CNN + 大batch（≥16） → BatchNorm（标准配置）
  • CNN + 小batch（≤8） → GroupNorm（GN/G=32）
  • Transformer / RNN / NLP → LayerNorm（Pre-LN）
  • 风格迁移 / 图像生成 → InstanceNorm
  • 多卡训练 + BN → 必须用SyncBN跨卡同步
重要提示：有些模型的默认配置可能会混合使用多种归一化。例如，ViT（Vision Transformer）虽然本质上是Transformer，但仍可能在某些位置使用BN（如patch embedding后的linear projection）。如果你不确定用什么归一化，先看你的模型继承哪个架构的血统——Attention血统走LN，Conv血统走BN/GN。
