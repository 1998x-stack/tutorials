第17章 密码学数论基础

坦白说，这一章可能是整个课程中最"数学"的一章。但我想先给你一个定心丸：你不需要成为数论专家才能理解公钥密码学。如果你能理解加减乘除、乘方和模运算——实际上你已经具备了90%的密码学数论基础。

为什么数论在密码学中如此重要？因为公钥密码的核心都是单向函数（one-way function）——正向计算很容易，逆向计算极其困难。而这种"易-难"不对称性，几乎全部建立在一系列数论性质之上：大数分解的困难、离散对数求解的困难、椭圆曲线上标量乘法的不可逆性。如果你不了解这些数学基础，你使用RSA和ECC就像魔术师不了解自己的戏法——虽然能变，但永远不知道什么时候会穿帮。

本章的目标很明确：让你在读完以后能够用自己的话解释RSA为什么安全、DH密钥交换为什么成立、ECC和RSA在数学上有什么本质区别。

17.1 模运算基础

模运算（Modular Arithmetic）是密码学中最常用的数学工具。你可能已经在上小学时无意中接触过它——时钟的加法就是模12运算：如果现在是9点，过5个小时是几点？9 + 5 = 14 ≡ 2 (mod 12)。

正式定义很简单：给定整数 a 和正整数 n，a mod n 表示 a 除以 n 的余数。关键性质包括：

  • 加法：(a + b) mod n = [(a mod n) + (b mod n)] mod n
  • 乘法：(a × b) mod n = [(a mod n) × (b mod n)] mod n
  • 指数：a^b mod n = (a mod n)^b mod n

这些性质之所以重要，是因为它们让大数计算变得可行。在RSA中，我们要计算像 123456789^65537 mod 987654321123456789 这样的值。如果不利用模运算的分配律，中间结果会大到无法存储。有了模运算，我们可以在每一步都取模，让中间结果保持在一个可控的范围内。

模逆元是另一个关键概念。a 在模 n 下的逆元（记作 a^(-1) mod n）是一个整数 x，使得 a × x ≡ 1 (mod n)。逆元存在的条件是 gcd(a, n) = 1（即a和n互质）。这正是RSA密钥生成的基础——私钥 d 就是 e 在模 φ(n) 下的逆元。

🔢 数学速记：模逆元就像除法在模世界中的等价。在实数中，a的倒数是1/a，满足 a × 1/a = 1。在模世界中，a的逆元是那个"乘以a得到1 mod n"的数。RSA之所以能工作，本质上就是因为 m^(e × d) ≡ m (mod n) 这个等式——加密再解密，恢复原文。

17.2 费马小定理与欧拉定理

这两个定理是公钥密码学的擎天柱。先看费马小定理：

费马小定理：如果 p 是质数，且 gcd(a, p) = 1，则 a^(p-1) ≡ 1 (mod p)。

这个定理看似简单，但它是RSA正确性证明的基石。更一般的形式是欧拉定理：

欧拉定理：如果 gcd(a, n) = 1，则 a^φ(n) ≡ 1 (mod n)，其中 φ(n) 是欧拉函数，表示小于n且与n互质的正整数的个数。

对于质数 p，φ(p) = p - 1。对于两个不同质数的乘积 n = p × q，φ(n) = (p-1)(q-1)。这个性质就是RSA的核心：

RSA 密钥生成:
1. 选择两个大质数 p 和 q
2. 计算 n = p × q, φ(n) = (p-1)(q-1)
3. 选择公钥指数 e（通常取 65537），满足 gcd(e, φ(n)) = 1
4. 计算私钥 d = e^(-1) mod φ(n)  （即 e × d ≡ 1 mod φ(n)）

加密: c = m^e mod n
解密: m = c^d mod n = m^(e×d) mod n = m^(1 + k·φ(n)) mod n = m × (m^φ(n))^k mod n = m × 1^k mod n = m mod n

注意最后那步正是欧拉定理的妙用！只要 m 和 n 互质（概率极高），m^φ(n) ≡ 1 (mod n)，所以 m^(1 + kφ(n)) ≡ m (mod n)。这就是RSA能正确解密的原因。

💡 欧拉函数的小窍门：φ(n) 的值取决于 n 的质因数分解。如果 n = p × q（两个不同质数），φ(n) = (p-1)(q-1)。但如果有人能高效分解 n（即找出 p 和 q），那么 φ(n) 就暴露了，进而 d 也被暴露了。这就是"RSA的安全性等价于大整数分解的困难性"的数学根源。

17.3 中国剩余定理

中国剩余定理（CRT）是密码学实现中最重要的优化工具。它解决的是这样一类问题：

已知 x ≡ a₁ (mod m₁), x ≡ a₂ (mod m₂), ..., x ≡ a_k (mod m_k)，其中各模数两两互质。求 x 在模 M = m₁ × m₂ × ... × m_k 下的唯一解。

CRT有两条非常实际的应用：

应用一：加速RSA解密

RSA解密涉及计算 m = c^d mod n，其中 n = p × q。直接用这个公式需要处理一个2048位的模指数运算。利用CRT，我们可以：

用 CRT 加速 RSA 解密:
分解: m_p = c^(d mod (p-1)) mod p
      m_q = c^(d mod (q-1)) mod q

合并（CRT）: 从 m_p 和 m_q 恢复 m mod n

效果: 将 2048 位模指数 → 两个 1024 位模指数
速度提升约 4 倍

应用二：秘密共享（Shamir的秘密共享方案）

这是在多项式插值框架下的应用——将秘密分解为多个"份额"，任意 t 个份额可以重构秘密，少于 t 个则不能。

CRT在密码学中的出现频率比你想象的高得多。几乎所有的RSA实现都使用CRT加速——OpenSSL、BoringSSL、Java的BigInteger库等。但CRT实现也引入了著名的Bellcore攻击（1996年）：如果在CRT计算中某个模数下的计算发生了错误（比如通过物理注入干扰），攻击者可以直接根据错误结果分解出 p 和 q。这也是为什么现代RSA实现必须做"CRT结果校验"。

17.4 素数测试

RSA需要大质数，但怎么判断一个随机的几百位数字是不是质数？注意，这比"分解一个大数"容易得多（这是单向函数的关键）。

测试方法 确定性？ 速度 实际用途
试除法 是 极慢 小质数筛除（用前1000个质数筛掉明显非质数）
费马素性检验 否（有伪质数） 快 已被淘汰（存在Carmichael数使其失效）
Miller-Rabin 概率性（可调精度） 快 工业界首选
Baillie-PSW 至今无已知反例 中等 可靠的"实际确定性"方案
AKS 是（2002年发现） 较慢 理论意义大，实际中很少使用

Miller-Rabin测试是实际中最常用的方法。它的思想是：不断随机选取底数 a，检查 a 是否"见证"了 n 的合数性。如果对于 k 个不同的 a，n 都通过了测试，那么 n 是合数的概率不超过 (1/4)^k。取 k = 64，这个概率就低到了 2^(-128)——对于所有实际工程来说，已经足够了。

顺带一提，AKS算法（2002年由Agrawal、Kayal、Saxena发现）在理论上非常重要——它是第一个被证明的多项式时间确定性素性测试算法。三位印度学者在IIT Kanpur的网站上发表了论文，让全世界震动。不过实际应用中，它的O(log^6 n)复杂度还是不如Miller-Rabin实用。

17.5 离散对数问题

离散对数问题（DLP）是公钥密码学的第二个支柱。它的形式是：

给定质数 p、生成元 g 和值 h = g^x mod p，求 x。

正向计算（已知x求h）很容易——一次模指数运算。逆向计算（已知h求x）被广泛认为在计算上是困难的——目前最好的算法（数域筛法）需要次指数时间。这种不对称构成了Diffie-Hellman密钥交换和DSA数字签名的基础。

DLP还有一个重要变体：椭圆曲线离散对数问题（ECDLP）。在椭圆曲线群上，正向运算是标量乘法 Q = k × P，逆向运算是从Q和P求k。ECDLP被认为比相同规模的有限域DLP更困难——这就是为什么ECC可以用更短的密钥（256 bits）达到RSA-3072的安全级别。

📊 安全强度等价对照表：
AES-128 ≈ ECC-256 ≈ RSA-3072 ≈ 128 bits 安全强度
AES-192 ≈ ECC-384 ≈ RSA-7680 ≈ 192 bits 安全强度
AES-256 ≈ ECC-521 ≈ RSA-15360 ≈ 256 bits 安全强度
从这张表可以清晰看到：ECC在安全密度上碾压RSA——256位的椭圆曲线密钥和三万多位RSA密钥强度相当。

这个分解用到了数域筛法（NFS），消耗了大约2700核心年的计算资源，历时数月。相比之前RSA-240（795 bits）的分解（用了约1000核心年），RSA-250的难度提升了近三倍。

你应该问一个关键问题：这是不是意味着RSA-2048也不再安全？

答案是否定的。RSA-250只有829 bits，而RSA-2048是2048 bits。数域筛法的复杂度随bits增长呈次指数增长——具体来说，是 L[1/3, (64/9)^(1/3)] 这个看起来很吓人的函数。粗略估算：分解RSA-1024需要的计算量是RSA-250的 数十亿倍。而以目前的计算能力，分解一个1024位的RSA模数仍然是不现实的（虽然国家级对手也许已经接近这个能力了）。这也是为什么当前标准推荐至少使用 RSA-2048。

⚠️ 历史教训：密码学的"安全bit数"标准一直在上调。1999年，RSA-512（155位十进制）被分解；2009年，RSA-768（232位十进制）被分解；2022年，RSA-250（250位十进制）被分解。虽然RSA-2048当前无疑是安全的，但从趋势上看，2048位的安全余量可能在本世纪中叶受到威胁。这也是NIST推动后量子密码标准化的深层原因。
