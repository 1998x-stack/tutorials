第10章 ARMA模型

大家好！前面两章我们分别学习了 AR 模型和 MA 模型，就像学了两套独立的拳法。但现实世界的数据往往需要两种拳法组合使用——既有"惯性"（AR），又有"冲击的余波"（MA）。ARMA 模型就是把这两套拳法融合在一起，成为时间序列建模中最经典、最实用的工具之一。

你可以这样理解：AR 部分捕捉的是序列的"长期记忆"——上个月的销量高，这个月很可能也高。MA 部分捕捉的是"短期震荡"——上个月突发的促销活动造成的销量波动。两者结合，才能全面刻画一个时间序列的生成机制。

10.1 ARMA(p,q) 的定义

ARMA(p,q) 模型将 AR(p) 和 MA(q) 合并：

Y_t = c + φ₁Y_{t-1} + φ₂Y_{t-2} + ... + φ_pY_{t-p} + ε_t + θ₁ε_{t-1} + θ₂ε_{t-2} + ... + θ_qε_{t-q}

用滞后算子 L 可以写得更紧凑：

Φ(L) Y_t = c + Θ(L) ε_t

其中 Φ(L) = 1 - φ₁L - φ₂L² - ... - φ_pL^p，Θ(L) = 1 + θ₁L + θ₂L² + ... + θ_qL^q。

ARMA(p,q) 的平稳性完全由 AR 部分决定——Φ(L) 的特征根必须在单位圆之外。MA 部分的可逆性由 Θ(L) 决定。两个条件互不干扰，这是 ARMA 建模中非常重要的独立性。

关键点：ARMA 的平稳性条件只依赖于 AR 部分！如果你看到 Φ(L) 在单位圆上有根，整个 ARMA 就是非平稳的。这告诉我们，在建模之前必须先检验数据的平稳性——如果数据有单位根，需要先差分（第11章）再建模。

10.2 ACF 与 PACF 的识别指南

ARMA 模型的识别比纯 AR 或纯 MA 要复杂一些，因为它的 ACF 和 PACF 都不会截尾，而是呈现"拖尾"（tail off）模式。这对新手来说是最容易困惑的地方。

模型 ACF PACF
AR(p) 拖尾（指数或正弦衰减） 滞后 p 后截尾
MA(q) 滞后 q 后截尾 拖尾（指数或正弦衰减）
ARMA(p,q) 拖尾（从滞后 q 开始衰减） 拖尾（从滞后 p 开始衰减）

当 ACF 和 PACF 都拖尾时，我们通常从低阶 ARMA 开始尝试，然后通过信息准则（AIC/BIC）来选择最优模型。经验法则是：大多数经济金融时间序列可以用 ARMA(1,1) 或 ARMA(2,2) 很好地描述。

误区案例：有一个工程师在做传感器数据分析时，看到 ACF 和 PACF 都缓慢衰减，认为这是某种"高阶混合模型"，直接尝试了 ARMA(10,10)。模型有 20 个参数，看起来拟合优度很高，但预测效果一塌糊涂。后来发现数据实际上是非平稳的——ACF/PACF 的缓慢衰减不是拖尾而是单位根！正确的做法是先检查平稳性，做差分，再用低阶 ARMA 建模。这个故事告诉我们：不要急着加参数，先回头检查基本假设。

10.3 Box-Jenkins 方法论概述

Box-Jenkins 方法论是 ARMA 建模的完整框架，由 George Box 和 Gwilym Jenkins 在 1970 年代提出。它包含四个迭代步骤：

第一步：识别（Identification）。通过 ACF 和 PACF 图初步判断 p 和 q 的范围，同时进行平稳性检验。如果数据非平稳，先做差分变换。

第二步：估计（Estimation）。用最大似然估计（MLE）或条件最小二乘（CLS）来估计模型参数。这一步统计软件会自动完成。

第三步：诊断（Diagnosis）。检查残差是否为白噪声。如果残差仍有自相关，说明模型欠拟合，需要调整 p 或 q；如果参数不显著，考虑简化模型。

第四步：预测（Forecasting）。用最终选定的模型进行预测，并计算预测区间。

这四个步骤不是一次通过的——如果在第三步发现问题，需要回到第一步重新识别，形成一个迭代循环。

Box-Jenkins箴言："All models are wrong, but some are useful." 这句话是 George Box 的名言。不要追求"完美的"ARMA 模型——你永远找不到真正的数据生成过程，只要找到一个"足够好"的、能用于预测的模型就够了。

10.4 节俭原则

前面提到了节俭原则（Parsimony Principle），这里展开说明一下。在选择 ARMA 的阶数时，为什么我们倾向于用更少的参数？

  • 估计稳定性：参数越多，协方差矩阵的估计越不稳定，预测方差越大
  • 过拟合风险：高阶模型可能拟合了噪声而非常号，样本外预测会惨不忍睹
  • AR-MA 互换性：一个 AR(1) 可以写成 MA(∞)，一个 MA(1) 可以写成 AR(∞)。所以低阶混合模型常常能代替高阶单一模型

举个例子：ARMA(1,1) 有 2 个参数（φ₁ 和 θ₁），但它可以近似一个 AR(∞) 或 MA(∞)。而 AR(10) 需要 10 个参数才能达到类似效果。这就是节俭原则的威力——用最少的参数捕捉最多的动态特征。

模型选择时，我们常用 AIC（Akaike Information Criterion）和 BIC（Bayesian Information Criterion）来平衡拟合优度和模型复杂度：

AIC = -2ln(L) + 2k

BIC = -2ln(L) + k ln(T)

其中 L 是似然函数值，k 是参数个数，T 是样本量。AIC 和 BIC 都惩罚参数数量，但 BIC 的惩罚更重（尤其在大样本下）。通常我们在有限候选模型中选 AIC 或 BIC 最小的那个。

10.5 Python 实操：ARMA 模型选择与拟合

下面我们用 statsmodels 来演示 ARMA 模型的拟合并通过 AIC 选择最优阶数：

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from statsmodels.tsa.arima.model import ARIMA
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf
import warnings
warnings.filterwarnings('ignore')

# 生成 ARMA(1,1) 数据
np.random.seed(42)
n = 500
c, phi1, theta1 = 2.0, 0.7, 0.4

Y = np.zeros(n)
eps = np.random.normal(0, 1, n)
Y[0] = c / (1 - phi1)  # 从无条件均值开始
for t in range(1, n):
    Y[t] = c + phi1*Y[t-1] + eps[t] + theta1*eps[t-1]

# 画 ACF 和 PACF
fig, (ax1, ax2) = plt.subplots(2, 1, figsize=(10, 6))
plot_acf(Y, lags=20, ax=ax1)
plot_pacf(Y, lags=20, ax=ax2, method='ywm')
plt.show()

# 通过 AIC 选择最优 (p,q)
results = []
for p in range(0, 4):
    for q in range(0, 4):
        try:
            model = ARIMA(Y, order=(p, 0, q), trend='c')
            result = model.fit()
            results.append({
                'p': p, 'q': q,
                'AIC': result.aic,
                'BIC': result.bic
            })
        except:
            continue

df_results = pd.DataFrame(results)
print(df_results.sort_values('AIC'))

# 拟合选定的最优模型
best_p = int(df_results.loc[df_results['AIC'].idxmin(), 'p'])
best_q = int(df_results.loc[df_results['AIC'].idxmin(), 'q'])
best_model = ARIMA(Y, order=(best_p, 0, best_q), trend='c')
best_result = best_model.fit()
print(best_result.summary())

运行这段代码，你会看到 AIC 表中最优的通常是 p=1, q=1 或非常接近的搭配。这验证了低阶 ARMA 模型恢复数据生成过程的能力。

ARMA 模型是目前为止我们学到的最强大的时间序列建模工具。它把 AR 的"记忆"和 MA 的"冲击调整"结合在一起，用极少的参数就能描述丰富的数据动态。但 ARMA 有一个前提——数据必须是平稳的。现实中大多数原始数据并非如此。下一章要讲的差分和 ARIMA 模型，就是解决这个问题的关键：先通过差分把非平稳数据变平稳，再用 ARMA 建模。
