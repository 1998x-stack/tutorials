第19章 MATLAB/Python DSP仿真

讲到这里，我们花了整整18章把DSP的理论基础全部铺开——从离散时间信号到采样定理，从Z变换到FFT，从FIR/IIR滤波器到自适应算法，从谱估计到时频分析。我知道你内心可能在呐喊："老师，讲得好，但我怎么用这些东西？！"

好，从这一章开始，我们进入实现技术模块。第19-22章的目标非常明确：教会你如何把之前学到的DSP算法，在仿真环境和真实硬件上跑起来。本章先聚焦仿真——用Python和MATLAB搭建DSP算法的快速原型环境。

为什么仿真如此重要？因为没有仿真就直接上硬件，约等于蒙眼射箭。我在职业生涯中犯过的最昂贵的错误之一，就是在一款DSP芯片上直接写了300行定点FIR滤波器，烧进去之后发现输出全是噪声。排查了整整两天，最后发现是系数换了Q格式之后没做归一化——这种问题如果在MATLAB里先仿真一遍，10分钟就能找到。

19.2 信号生成与可视化

仿真的第一步是"制造"信号。在Python里生成各种测试信号是再简单不过的事情。我们来看几个最常用的模式。

import numpy as np
import matplotlib.pyplot as plt
from scipy import signal

# 参数设置
fs = 1000          # 采样率 1kHz
T = 1.0            # 时长 1秒
t = np.linspace(0, T, int(fs*T), endpoint=False)

# 1. 正弦信号
f0 = 50
x_sin = np.sin(2 * np.pi * f0 * t)

# 2. 线性扫频信号 (chirp)
f_start, f_end = 10, 200
x_chirp = signal.chirp(t, f_start, T, f_end, method='linear')

# 3. 白噪声
np.random.seed(42)
x_noise = np.random.randn(len(t))

# 4. 正弦 + 噪声 (实测信号模拟)
snr_db = 10     # 信噪比 10dB
x_meas = x_sin + np.sqrt(np.var(x_sin) / (10**(snr_db/10))) * x_noise

# 时域绘图
plt.figure(figsize=(10, 4))
plt.plot(t[:200], x_sin[:200], label='50Hz sine')
plt.plot(t[:200], x_meas[:200], label='Noisy (SNR=10dB)', alpha=0.7)
plt.xlabel('Time [s]'); plt.ylabel('Amplitude')
plt.legend(); plt.grid(True); plt.show()

# 频域绘图
X = np.fft.fft(x_meas)
freqs = np.fft.fftfreq(len(x_meas), 1/fs)
plt.figure(figsize=(10, 4))
plt.plot(freqs[:len(freqs)//2], 20*np.log10(np.abs(X[:len(X)//2]) + 1e-10))
plt.xlabel('Frequency [Hz]'); plt.ylabel('Magnitude [dB]')
plt.grid(True); plt.show()

这段代码演示了一个非常标准的仿真工作流：生成信号 → 加入噪声模拟真实环境 → 时域观察 → 频谱分析。那个 np.sqrt(np.var(x_sin) / (10**(snr_db/10))) 是控制信噪比的精确公式——很多初学者直接用 sigma=0.1 之类的魔数，这种做法在写Demo时可以，但做正式仿真时一定要用SNR控制。

19.3 scipy.signal 核心函数实战

scipy.signal 是整个DSP仿真生态的中枢。掌握了它，你就掌握了数字滤波器的"乐高积木"。我们来逐一过一遍最核心的函数。

频率响应分析：freqz

from scipy.signal import freqz, firwin, butter, lfilter

# FIR低通滤波器设计
b = firwin(31, 0.2)  # 31阶，归一化截止频率0.2
w, h = freqz(b, worN=8192)

plt.figure(figsize=(8, 4))
plt.subplot(121)
plt.plot(w/np.pi, 20*np.log10(np.abs(h) + 1e-10))
plt.ylim(-80, 5); plt.grid(True)
plt.title('Magnitude Response (dB)')
plt.xlabel('Normalized Frequency (×π rad/sample)')

plt.subplot(122)
angles = np.unwrap(np.angle(h))
plt.plot(w/np.pi, angles)
plt.grid(True); plt.title('Phase Response')
plt.xlabel('Normalized Frequency (×π rad/sample)')
plt.tight_layout(); plt.show()

freqz 返回的 w 是归一化频率（单位是 π rad/sample），h 是复数频率响应。记住：归一化频率1.0对应fs/2，即Nyquist频率。如果你设计的滤波器截止频率是200Hz而采样率是1000Hz，那归一化频率就是 200/(1000/2) = 0.4。

FIR滤波器设计：firls / remez

firls 使用最小二乘法设计FIR滤波器，remez 使用Parks-McClellan算法（等波纹法）。两者都支持任意形状的频率响应。

from scipy.signal import firls, remez

# 最小二乘法：多频带滤波器
bands = [0, 0.2, 0.25, 0.45, 0.5, 1.0]   # 频带边界
desired = [1, 1, 0, 0, 1, 1]             # 对应的理想增益
b_ls = firls(63, bands, desired)

# Parks-McClellan等波纹法（结果更优，但需要更多迭代）
b_pm = remez(63, bands, [1, 0, 1], fs=2)
w_ls, h_ls = freqz(b_ls)
w_pm, h_pm = freqz(b_pm)

plt.plot(w_ls/np.pi, 20*np.log10(np.abs(h_ls)), label='FIRLS')
plt.plot(w_pm/np.pi, 20*np.log10(np.abs(h_pm)), '--', label='REMEZ')
plt.ylim(-60, 5); plt.grid(True); plt.legend()
plt.title('FIRLS vs REMEZ: 多频带滤波器')
plt.xlabel('Normalized Frequency'); plt.show()

IIR滤波器设计：butter / cheby1 / ellip

函数 滤波器类型 特点 适用场景
butter 巴特沃斯 通带最平坦，滚降最慢 音频均衡、信号调理
cheby1 切比雪夫I型 通带有纹波，滚降更快 需要陡峭过渡带的场合
cheby2 切比雪夫II型 阻带有纹波，通带平坦 阻带衰减要求高的应用
ellip 椭圆 通带阻带都有纹波，滚降最陡 阶数受限时追求最陡过渡带 
from scipy.signal import butter, cheby1, ellip, lfilter

# 4阶巴特沃斯低通滤波器
b, a = butter(4, 0.2, btype='low')
y_butter = lfilter(b, a, x_noise)

# 4阶切比雪夫I型低通（通带纹波1dB）
b1, a1 = cheby1(4, 1, 0.2, btype='low')
y_cheby = lfilter(b1, a1, x_noise)

# 4阶椭圆低通（通带纹波1dB，阻带衰减40dB）
b2, a2 = ellip(4, 1, 40, 0.2, btype='low')
y_ellip = lfilter(b2, a2, x_noise)

时域滤波：lfilter vs filtfilt

有一件极其重要的事情你必须知道：lfilter 和 filtfilt 虽然都能做滤波，但行为完全不同。

  • lfilter：标准递推滤波（因果滤波）。输出依赖于过去的输入和输出，有固定的群延迟。适用于实时处理。
  • filtfilt：零相位滤波（非因果）。把信号正向滤波一次，再反向滤波一次，相位延迟抵消为零。但代价是——有效滤波器阶数翻倍。
⚠️ 踩坑实录：filtfilt不是"免费"的零相位 我曾经在一个语音降噪项目里用 filtfilt 处理离线音频，效果非常完美——相位失真为零，波形对齐得天衣无缝。然后我兴高采烈地把同样的滤波器系数移植到嵌入式DSP的实时处理链路上，结果一听，声音完全不对——相位畸变严重。原因？嵌入式端只能用因果滤波（lfilter），不能"看到未来的样本"去做反向滤波。filtfilt的零相位是用非因果性换来的——它在滤波前翻转了信号，这意味着它需要完整的信号序列，无法用于流式实时处理。请记住：离线仿真用filtfilt，实时系统用lfilter。用filtfilt验证过的算法转到实时平台时，相位指标一定要重新评估。

19.4 MATLAB Signal Processing Toolbox 速查对照

很多老派DSP工程师（包括我）最早都是用MATLAB入门的。虽然Python已经后来居上，但MATLAB的Signal Processing Toolbox依然在某些领域（通信系统级仿真、Simulink建模）不可替代。下面是常用函数的对照表。

MATLAB函数 Python/SciPy等价 功能
fir1 firwin 窗函数法FIR设计
firls firls（同名） 最小二乘FIR设计
firpm remez Parks-McClellan最优FIR
butter butter（同名） 巴特沃斯滤波器设计
cheby1 cheby1（同名） 切比雪夫I型
ellip ellip（同名） 椭圆滤波器
freqz freqz（同名） 数字频率响应
filter lfilter 因果数字滤波
filtfilt filtfilt（同名） 零相位滤波（非因果）
fft np.fft.fft 快速傅里叶变换
spectrogram signal.spectrogram 短时傅里叶变换时频谱

有一个差异需要注意：MATLAB的滤波器系数约定和Python不同。MATLAB用 [b, a] 表示 H(z) = (b0 + b1 z⁻¹ + ...)/(1 + a1 z⁻¹ + ...)，而SciPy的 lfilter 也是这个约定，所以直接搬系数是兼容的。但MATLAB的 freqz(b, a, N) 默认返回的角频率单位是 rad/sample（0到π），而SciPy的 freqz 默认返回归一化频率（0到1.0，对应0到π）。把MATLAB代码转Python时，这个归一化因子是最容易踩的坑。

19.5 三级验证方法学：浮点→定点→硬件

仿真不只是"写代码画图"那么简单。在工业级DSP开发中，有一个标准的三级验证流程，我称之为"三级A/B对比"。

  1. Level 1 — 浮点参考模型（Python/MATLAB）： 用双精度浮点实现算法。这是"黄金标准"（golden reference），所有后续实现都要与之对比。在这一级，你验证的是算法正确性。
  2. Level 2 — 定点模型（仿真级）： 在Python中模拟定点运算，包括量化、舍入、饱和。用 np.int16 / np.int32 模拟DSP芯片的数值行为。在这一级，你验证的是算法在有限精度下的鲁棒性。
  3. Level 3 — 硬件输出（实测）： 把代码烧到真实的DSP芯片或FPGA上，采集输出信号，与Level 1/2的结果做 bit-true 对比。在这一级，你验证的是实现正确性。
💡 核心认知：Level 2是整套流程中最容易被跳过的环节。很多团队直接从Level 1跳到Level 3，结果在硬件上发现了量化噪声、极限环、溢出等问题，然后返回Level 1改算法。这个来回一次至少一周。我强烈建议：任何定点DSP项目，必须在仿真环境中先做bit-true定点仿真。下一章我们会深入讨论定点实现的细节。 
# 三级验证的Python骨架
def level1_float(x, b, a):
    """双精度浮点参考模型"""
    from scipy.signal import lfilter
    return lfilter(b, a, x)

def level2_fixed(x, b_q, a_q, nbits=16):
    """定点模拟模型：输入输出均为Q格式整数"""
    # 模拟定点MAC累加
    acc_bits = 2 * nbits  # 累加器位宽
    y = np.zeros_like(x, dtype=np.int64)
    for n in range(len(x)):
        acc = 0
        for k in range(len(b_q)):
            if n >= k:
                acc += b_q[k] * x[n - k]
        for k in range(1, len(a_q)):
            if n >= k:
                acc -= a_q[k] * y[n - k]
        # 饱和截位到nbits
        max_val = (1 << (nbits - 1)) - 1
        acc = np.clip(acc >> (acc_bits - nbits), -max_val-1, max_val)
        y[n] = int(acc)
    return y
