第16章 多元时间序列：向量自回归模型 (VAR)

到目前为止，我们所有的模型都只在处理一个时间序列：预测某只股票的价格、某个城市的温度、某产品的销量。但在现实世界中，事物是相互联系的。GDP 的变化会影响失业率，失业率的变化会影响消费，消费的变化又会反过来影响 GDP。如果只用单变量模型逐一建模，我们就丢失了变量之间的反馈效应和同时动态。

向量自回归模型（Vector Autoregression，简称 VAR）正是为了解决这个问题而生的。它将单变量 AR 模型的思想推广到 k 维——每个变量都用自己的滞后项和其他所有变量的滞后项来回归。这样，整个系统的联合动态就装进了一个模型里。

本章核心目标：理解 VAR 模型的估计、定阶和解释方法，掌握 Granger 因果检验和脉冲响应分析的实践技能。

16.1 VAR 模型的结构

考虑两个变量 y₁ 和 y₂ 的情况。一个 VAR(1) 模型写成矩阵形式就是：

[y₁_t]   [c₁]   [a₁₁  a₁₂] [y₁_{t-1}]   [ε₁_t]
[y₂_t] = [c₂] + [a₂₁  a₂₂]·[y₂_{t-1}] + [ε₂_t]

更简洁地:
y_t = c + A·y_{t-1} + ε_t

关键的区别在于：每个方程都包含了对方变量的滞后项。a₁₂ 衡量的是 y₂ 滞后一期对今天 y₁ 的影响；a₂₁ 则反之。如果 a₁₂ ≠ 0，我们就说 y₂ "Granger 引起" y₁。

推广到 VAR(p) 就是：

y_t = c + A₁·y_{t-1} + A₂·y_{t-2} + ... + A_p·y_{t-p} + ε_t

其中每个 A_i 都是 k×k 的系数矩阵，一共需要估计 k + p·k² 个参数。可以看到，当变量数和滞后阶数增加时，参数数量增长得非常快。

经验法则：VAR 模型适合 3-8 个变量的中等规模系统。变量太多会导致参数爆炸——比如 10 个变量、VAR(4) 就有 10 + 4×100 = 410 个参数要估计。

16.2 定阶与估计

VAR 的滞后阶数 p 通常通过信息准则选择。最常用的是 AIC（赤池信息准则）和 BIC（贝叶斯信息准则）：

import pandas as pd
import yfinance as yf
from statsmodels.tsa.api import VAR

# 准备数据：GDP增速、利率、通胀率
data = yf.download(['GDP', '^TNX', 'CPI'], period='10y')
# 实际使用中应从 FRED 等经济数据库获取

# 假设 df 是包含三列的数据框
model = VAR(df)
# 自动选择滞后阶数
lag_order = model.select_order(maxlags=12)
print(lag_order.summary())

# 拟合 VAR(2)
results = model.fit(2)
print(results.summary())

需要注意：VAR 要求所有变量都是平稳的。如果变量不平稳，需要先做差分，或者考虑后面章节要讲的协整方法。

准则偏好说明
AIC倾向于选更多滞后预测导向，偏差小方差大
BIC倾向于选更少滞后一致性导向，大样本下选真实阶数
HQIC居中Hannan-Quinn 准则，介于二者之间

16.3 Granger 因果检验

Granger 因果性检验回答的问题是：变量 x 的过去值是否有助于预测变量 y 的未来值？如果能，我们就说 x "Granger 引起" y。

⚠ 极其重要的误解：Granger 因果关系≠真正的因果关系！它只能说明 x 在预测意义上先于 y，不能证明 x 导致了 y。著名的例子是"黄油产量 Granger 引起 GDP 增长"——两者同步上涨只是因为经济在增长，黄油产量并没有推动经济。

检验的原假设是：x 的所有滞后项系数联合为零（即 x 不 Granger 引起 y）。使用 F 统计量或卡方统计量进行检验：

# Granger 因果检验
from statsmodels.tsa.stattools import grangercausalitytests

# 检验利率是否 Granger 引起 GDP 增速
gc_result = grangercausalitytests(df[['GDP_growth', 'interest_rate']], maxlag=4)

# VAR 模型内置的因果检验
causality_dict = results.test_causality('GDP', 'interest_rate', kind='f')
print(causality_dict)

16.4 脉冲响应函数

VAR 模型的系数矩阵包含大量信息，但直接解读系数很困难——因为变量之间存在反馈效应，某个冲击会在系统内反复传导。脉冲响应函数（Impulse Response Function, IRF）是解决这个问题的利器。

IRF 追踪的是：给某个变量一个单位的标准差冲击，未来各期内所有变量会如何响应？

# 计算并绘制脉冲响应
irf = results.irf(periods=20)  # 追踪20期
irf.plot(orth=True)  # 使用正交化脉冲响应

# 查看具体数值
irf_table = irf.to_dataframe()
print(irf_table.head())

一个典型的 IRF 图包含 k×k 个子图。例如在 GDP-利率-通胀三变量系统中，(2,1) 位置的图展示的是：给 GDP 一个正向冲击后，利率在接下来 20 个季度中的反应路径。通常我们会看到 GDP 冲击先推高利率（央行加息抑制通胀），然后利率升高又逐渐压制 GDP，形成完整的反馈循环。

解读技巧：关注脉冲响应的形状而不是具体数值——冲击是逐渐衰减还是震荡？是短期反转还是长期持久？这些模式比系数大小更有经济意义。

16.5 方差分解

方差分解回答的是另一个问题：y 的预测误差中，有多大比例是由自身冲击引起的，有多大比例是由其他变量的冲击引起的？

# 预测误差方差分解
fevd = results.fevd(periods=10)
fevd.plot()

# 查看具体数值
print(fevd.summary())

如果 GDP 的预测误差在 10 期后仍有 80% 来自自身、20% 来自利率冲击，说明 GDP 的波动主要由内生动力驱动。反之，如果利率冲击能解释 50% 以上的 GDP 波动，那么货币政策对实体经济的影响就非常显著。

16.6 真实故事：GDP-利率-通胀的三角关系

2010 年代，经济学家利用 VAR 模型分析了中国宏观经济的三元系统——GDP 增速、银行间利率和 CPI 通胀率。数据覆盖 2000-2015 年，这是一个经典的 VAR 应用案例。

研究的关键发现包括：

  • Granger 因果：利率显著 Granger 引起通胀，但通胀对利率的 Granger 因果关系较弱——说明当时的货币政策对价格有引导作用，但价格对货币政策的反馈不够灵敏
  • 脉冲响应：一个百分点的利率提升，会在 4 个季度后使通胀下降约 0.3 个百分点，并在第 6 个季度达到最大效果
  • 方差分解：GDP 增速的波动中有约 35% 可由利率冲击解释，说明货币政策是经济周期的重要驱动力

这个案例生动展示了 VAR 模型如何帮助经济学家定量刻画宏观变量之间的互动关系。虽然不能断言因果关系，但它在预测和政策的"what-if"分析中提供了不可替代的价值。

要点总结：VAR 将单变量 AR 扩展到多变量系统，Granger 因果检验和脉冲响应是解读 VAR 的两大工具，方差分解量化各变量的相对重要性。

16.7 本章小结

本章我们学习了：

  • VAR 模型用所有变量的滞后项同时预测多个时间序列
  • 信息准则（AIC/BIC/HQIC）用于选择最优滞后阶数
  • Granger 因果检验衡量预测能力，但不是真正的因果推断
  • 脉冲响应函数追踪单一冲击在整个系统中的传播路径
  • 方差分解量化各变量对预测误差的贡献比例

VAR 模型有一个关键前提：所有变量必须是平稳的。但如果非平稳变量之间存在长期均衡关系呢？那就不能简单差分——我们需要协整这个更精巧的工具。下一章我们将学习协整与误差修正模型 (VECM)，它同时捕捉变量的短期动态和长期均衡。
