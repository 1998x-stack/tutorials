第2章 线性代数基础

大家好，我是你们的讲师。在开始这一章之前，我想让你做一个小实验：伸出一只手，指尖指向天花板，然后慢慢将手臂旋转——你刚刚做的事情，在数学上就是一个旋转变换，而描述这个变换的工具，正是线性代数。在机器人学中，从机械臂末端执行器的位置解算，到LiDAR点云的配准，再到SLAM中的图优化，线性代数无处不在。毫不夸张地说，线性代数就是机器人学的"普通话"——不会这门语言，你连机器人的基本动作都描述不清楚。

我见过太多初学机器人的同学，在刚接触到旋转矩阵、齐次坐标这些概念时就打了退堂鼓，觉得"我是做算法的，数学差不多就行了"。结果到了写逆运动学代码时，发现自己连Jacobian矩阵的维度都搞不清楚，更别说理解奇异值分解在其中的作用了。这一章，我会带着你把这些概念一个一个啃下来，而且——我保证——会很直观。

2.1 向量与向量空间

在机器人学中，向量（Vector）是最基本也是最重要的数学对象。机械臂末端的位置是一个三维向量 \((x, y, z)\)，末端执行器的线速度是一个三维向量 \((v_x, v_y, v_z)\)，施加在关节上的力同样可以用向量表示。

向量的两个核心运算是点积（Dot Product）和叉积（Cross Product）。点积 \(\mathbf{a} \cdot \mathbf{b} = \|\mathbf{a}\|\|\mathbf{b}\|\cos\theta\) 衡量了两个向量的对齐程度——当你用点积判断机器人是否正对着目标物体时，实际上就是在用这个几何意义。叉积 \(\mathbf{a} \times \mathbf{b}\) 则生成一个垂直于 \(\mathbf{a}\) 和 \(\mathbf{b}\) 的向量，其模长等于两个向量围成的平行四边形面积。叉积在机器人学中的一个重要应用是计算力矩：\(\boldsymbol{\tau} = \mathbf{r} \times \mathbf{F}\)。

直觉理解：点积回答"两个方向有多接近"，叉积回答"两个方向围成的面积有多大"。在机器人控制中，点积用于投影（比如将力投影到某个方向上），叉积用于生成垂直方向（比如从力和力臂计算力矩）。记住这两个直觉，你就能理解机器人学中90%的向量运算。

再往下是向量空间（Vector Space）的概念。机器人末端在三维空间中的所有可能位置构成一个 \(\mathbb{R}^3\) 空间，所有可能的姿态构成 \(\text{SO}(3)\) 群（三维旋转群）。这些集合之所以被称为"空间"，是因为它们满足一些公理——加法和数乘封闭、存在零向量和逆向量等等。你不需要死记这些公理，只需要记住一句话：凡是可以做"加法"和"数乘"的东西，都可以看作向量。

这里分享一个我当年踩过的坑。在做机械臂的轨迹插值时，我需要计算两个姿态之间的"中间姿态"。我当时想当然地对两个旋转矩阵做了线性插值 \((1-t)\mathbf{R}_1 + t\mathbf{R}_2\)，结果发现插出来的矩阵根本不是旋转矩阵（它的行列式不等于1）。后来才意识到旋转矩阵不构成向量空间——它们构成的是李群 \(\text{SO}(3)\)，加法不封闭，需要用特殊方法插值（比如Slerp，我们第5章会讲）。这个教训告诉我：时刻留意你操作的对象属于什么空间。

2.2 矩阵与线性变换

矩阵是线性代数中最重要的运算工具。在机器人学中，矩阵最基本的功能是描述线性变换——旋转、缩放、剪切、投影，这些都可以用矩阵来表示。

一个三维旋转矩阵 \(\mathbf{R} \in \mathbb{R}^{3 \times 3}\) 满足 \(\mathbf{R}^T \mathbf{R} = \mathbf{I}\) 且 \(\det(\mathbf{R}) = 1\)，所有这样的矩阵构成特殊正交群 \(\text{SO}(3)\)。当我们说"把机械臂末端绕z轴旋转30度"，在数学上就是用一个旋转矩阵左乘以末端的位置向量。

矩阵乘法 \(\mathbf{C} = \mathbf{A} \mathbf{B}\) 是"变换的复合"——先做变换 \(\mathbf{B}\)，再做变换 \(\mathbf{A}\)。这一点在运动学链中至关重要：一个六轴机械臂从基座到末端的变换，就是六个关节变换矩阵的乘积。每乘一个矩阵，坐标系就沿着运动链"走"了一个关节。数学上极其简洁，物理上极其直观。

# 机器人运动链中的矩阵复合
T_base_to_end = T_base_to_j1 @ T_j1_to_j2 @ T_j2_to_j3 @ T_j3_to_j4 @ T_j4_to_j5 @ T_j5_to_end

# 其中每个 T 是一个 4×4 齐次变换矩阵（我们第5章会详细介绍）
# 矩阵乘法顺序：从左到右依次应用变换
# 物理含义：从基座坐标系一步步"走到"末端坐标系

值得一提的是，矩阵乘法不满足交换律——\(\mathbf{A}\mathbf{B} \neq \mathbf{B}\mathbf{A}\) 在大多数情况下成立。这对应到物理世界就是"旋转的顺序影响最终结果"：先绕x轴转90度再绕y轴转90度，和先绕y轴再绕x轴，最终姿态完全不同。很多新手在调机器人姿态时遇到"明明发了正确角度，末端位姿却不对"，多半就是旋转顺序搞错了。

2.3 特征值与特征分解

如果说矩阵描述了一个变换"做了什么"，那么特征值和特征向量描述了变换"在哪个方向上做了什么"——对于矩阵 \(\mathbf{A}\)，如果存在非零向量 \(\mathbf{v}\) 和标量 \(\lambda\) 使得 \(\mathbf{A}\mathbf{v} = \lambda \mathbf{v}\)，那么 \(\mathbf{v}\) 就是特征向量，\(\lambda\) 就是对应的特征值。向量 \(\mathbf{v}\) 在经过变换后只被拉伸/压缩了 \(\lambda\) 倍，方向不变。

在机器人学中，特征分解最经典的应用是协方差矩阵的主成分分析（PCA）。假设你有一堆LiDAR点云数据，每个点都是三维坐标 \((x, y, z)\)。这些点的协方差矩阵 \(\mathbf{\Sigma} \in \mathbb{R}^{3 \times 3}\) 的特征向量指向数据变化最大的方向，特征值则告诉你在那个方向上的方差有多大。这在点云配准的ICP算法中用于判断对应关系，也在SLAM的数据关联中用于衡量不确定性。

知识点：协方差矩阵 \(\mathbf{\Sigma}\) 是实对称半正定矩阵，这意味着它总有实数特征值，且特征向量互相正交。这个性质使得特征分解在数值上非常稳定——这也是为什么机器人学中的不确定性建模广泛使用高斯分布（其协方差矩阵天然具有这些良性质）。

特征值还告诉我们一个变换的"条件数"（condition number），即最大特征值与最小特征值的比值。条件数很大的矩阵是病态的（ill-conditioned），在数值计算中会导致严重的精度损失。在逆向运动学中，当机械臂接近奇异位形时，Jacobian矩阵的条件数会趋近无穷大——这是判断奇异位形的重要数值指标。

2.4 奇异值分解（SVD）

如果只允许我选择一种矩阵分解来理解机器人学，我一定会选奇异值分解（Singular Value Decomposition）。对于任意矩阵 \(\mathbf{A} \in \mathbb{R}^{m \times n}\)，SVD将其分解为：

A = U @ S @ V^T

# 其中：
# U ∈ R^(m×m) —— 左奇异向量，列是 A A^T 的特征向量
# S ∈ R^(m×n) —— 奇异值对角矩阵，对角线元素 σ1 ≥ σ2 ≥ ... ≥ σr > 0
# V ∈ R^(n×n) —— 右奇异向量，列是 A^T A 的特征向量

SVD在机器人学中有三个极其重要的应用。

第一，求解线性最小二乘问题。当你需要求解 \(\mathbf{A}\mathbf{x} \approx \mathbf{b}\) 且方程数多于未知数时（比如用多个传感器观测来估计机器人的位姿），SVD给出数值最稳定的解 \(\mathbf{x} = \mathbf{V} \mathbf{S}^{+} \mathbf{U}^T \mathbf{b}\)，其中 \(\mathbf{S}^{+}\) 是伪逆。

第二，逆向运动学中的奇异值分析。Jacobian矩阵 \(\mathbf{J} \in \mathbb{R}^{6 \times n}\)（\(n\) 是关节数）的SVD可以告诉我们：在当前位置上，机器人最容易朝哪个方向运动（最大奇异值对应的方向），最不容易朝哪个方向运动（最小奇异值对应的方向）。当最小奇异值接近于零时，说明机器人处于或接近奇异位形。

第三，点云配准中的ICP算法。在Iterative Closest Point算法中，求解两组点云之间的最优旋转可以用 \(\mathbf{H} = \sum \mathbf{p}_i \mathbf{q}_i^T\) 的SVD来实现：\(\mathbf{R} = \mathbf{V} \mathbf{U}^T\)。这个解法优美、稳定、一阶最优。

我曾在一个移动机器人项目中遇到过这样的问题：AGV的激光定位在通过狭窄通道时会突然跳变数米。排查了两个月，最终发现是ICP配准中的SVD求解时，当点云近似共面（比如通道两侧的墙壁平行），最小奇异值变得非常小，导致配准解不稳定。解决方案是在SVD中加入阻尼项（类似Levenberg-Marquardt的思路），人为地限定了最小奇异值的下限。这个教训价值两个月的时间和大量熬夜——所以当你听到"SVD很稳定"时，记得加上一句"在非退化条件下"。

2.6 实战：Python/NumPy线性代数

光说不练假把式。我们来看一段用NumPy实现机器人学中常见线性代数运算的代码。建议你在自己的电脑上跟着敲一遍——亲手运行一次比读十遍都管用。

import numpy as np

# === 1. 向量运算 ===
p = np.array([0.5, 0.3, 0.8])      # 末端位置向量
v = np.array([0.1, -0.2, 0.05])    # 速度向量

dot = np.dot(p, v)                  # 点积：p 在 v 方向上的投影
cross = np.cross(p, v)              # 叉积：垂直于 p 和 v 的向量

print(f"点积 = {dot:.4f}")
print(f"叉积 = {cross}")

# === 2. 旋转矩阵与复合变换 ===
def rot_z(theta):
    """绕z轴旋转 theta 弧度的旋转矩阵"""
    c, s = np.cos(theta), np.sin(theta)
    return np.array([[c, -s, 0],
                     [s,  c, 0],
                     [0,  0, 1]])

def rot_y(theta):
    """绕y轴旋转 theta 弧度的旋转矩阵"""
    c, s = np.cos(theta), np.sin(theta)
    return np.array([[c,  0, s],
                     [0,  1, 0],
                     [-s, 0, c]])

# 先绕z转30度，再绕y转45度
theta_z = np.deg2rad(30)
theta_y = np.deg2rad(45)
R = rot_y(theta_y) @ rot_z(theta_z)  # 注意：先应用的变换写在右边

p_rotated = R @ p
print(f"旋转后的位置 = {p_rotated}")

# === 3. SVD 与伪逆 ===
# 构造一个接近奇异的矩阵
A = np.array([[1, 2],
              [2, 4]])  # 第二行是第一行的2倍，秩为1

U, S, Vt = np.linalg.svd(A)
print(f"奇异值 = {S}")
print(f"条件数 = {S[0] / S[-1]:.2f}")  # 接近无穷大

# 伪逆求解最小二乘
A_pinv = np.linalg.pinv(A)
x_ls = A_pinv @ np.array([1, 2])
print(f"最小二乘解 = {x_ls}")

# === 4. 特征分解（PCA 示例） ===
# 生成3D点云
points = np.random.randn(100, 3)
# 施加各向异性缩放
points = points @ np.diag([3, 1, 0.5])
# 计算协方差矩阵并进行特征分解
cov = np.cov(points.T)
eigvals, eigvecs = np.linalg.eigh(cov)
print(f"特征值 = {eigvals}")
print(f"主方向（最大特征值对应特征向量） = {eigvecs[:, -1]}")

这段代码虽然简单，但已经覆盖了机器人学中80%的线性代数操作。你可以把它当作一个"数学工具箱"来使用——每当你在后续章节遇到矩阵运算时，回来翻一翻这段代码。

避坑指南：在使用NumPy进行机器人学计算时，有几个常见陷阱需要注意。(1) 矩阵乘法用 @ 运算符，而不是 *（* 是逐元素乘法）。(2) 创建旋转矩阵时要注意角度单位——NumPy的三角函数默认使用弧度，而很多机器人API（如ROS 2）也使用弧度，但可视化工具可能使用度数，务必统一单位。(3) 涉及到姿态插值时不要在旋转矩阵上做线性插值——结果的矩阵行列式不会保持为1，正确的做法是用四元数Slerp或对数映射。

好了，线性代数的基础知识就讲到这里。我们从向量讲到了SVD，从理论讲到了NumPy实战。这些内容看起来有些抽象，但请相信——一旦你进入后续章节，你会发现这些概念会在每一个代码层面反复出现。掌握了线性代数，下一步我们需要微积分和优化工具来理解机器人的运动与最优控制。下一章见。
