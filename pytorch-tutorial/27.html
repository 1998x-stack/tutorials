第27章 实战项目：图像分类全流程

大家好，我是你们的讲师。如果你已经学完了前面 26 章的内容，那理论层面的储备已经非常充足了。但深度学习是一项实践性极强的技能——纸上谈兵永远不够。这一章我们要做一个完整的实战项目，把前面学到的所有知识串联起来。

我们的项目是：在 CIFAR-100 数据集上构建一个图像分类器。CIFAR-100 包含 100 个类别，每类 600 张 32x32 的小图片。虽然不算大规模，但它足够让我们体验一个完整的数据科学工作流——从数据分析到模型部署，走完所有环节。

让我先分享一个真实的故事。我有一个学生，在学完前面 25 章后自己做了一个图像分类项目。第一次跑的时候准确率只有 45%，比随机猜测（1%）好一些，但离可用还差得远。后来他回来找我说"老师，我是不是不适合做深度学习？"我帮他看了一遍代码，发现问题很多：数据增强太弱、没有用预训练模型、学习率设置不合理、跑了 10 个 epoch 就停了。我带着他一步一步调整，从数据预处理到训练策略，最终准确率达到了 87%。他说了一句话我印象很深："原来不是我不行，是我还没掌握正确的方法。"这一章，我就把这些正确的方法全部教给你。

好，让我们开始这个项目。

Step 1：数据探索与预处理

首先，我们用 torchvision 加载 CIFAR-100 数据集，并进行初步的探索性分析。

import torch
import torchvision
import torchvision.transforms.v2 as transforms
import matplotlib.pyplot as plt
import numpy as np
from collections import Counter

# 加载原始数据集（不进行增强，只做 ToTensor）
raw_dataset = torchvision.datasets.CIFAR100(
    root="./data", train=True, download=True
)

# 查看类别
print("类别数量:", len(raw_dataset.classes))
print("前10个类别:", raw_dataset.classes[:10])

# 检查类别分布
labels = [label for _, label in raw_dataset]
class_counts = Counter(labels)
print("最少样本类:", min(class_counts.values()))
print("最多样本类:", max(class_counts.values()))

# 展示一些样本
fig, axes = plt.subplots(4, 8, figsize=(12, 6))
for i, ax in enumerate(axes.flat):
    img, label = raw_dataset[i]
    ax.imshow(img)
    ax.set_title(raw_dataset.classes[label])
    ax.axis("off")
plt.tight_layout()
plt.savefig("data_samples.png")
为什么先做数据探索？很多初学者拿到数据集就直接开始训练，这是大忌。检查类别分布可以帮助你发现类别不平衡问题；查看样本图片可以让你了解数据的难度（CIFAR-100 的 32x32 分辨率非常低，人眼都不一定能分辨所有类别）。花 10 分钟做数据探索，能帮你节省后面 10 小时的调试时间。

CIFAR-100 的类别分布是均衡的（每类 500 张训练，100 张测试），所以我们不需要做重采样。接下来我们定义数据预处理流水线，使用 transforms.v2（PyTorch 2.x 推荐的 API）：

# 训练集数据增强
train_transforms = transforms.Compose([
    transforms.RandomCrop(32, padding=4),
    transforms.RandomHorizontalFlip(),
    transforms.RandomRotation(15),
    transforms.ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2),
    transforms.ToImage(),
    transforms.ToDtype(torch.float32, scale=True),
    transforms.Normalize(
        mean=[0.5071, 0.4867, 0.4408],
        std=[0.2675, 0.2565, 0.2761]
    ),
])

# 测试集只做标准化
test_transforms = transforms.Compose([
    transforms.ToImage(),
    transforms.ToDtype(torch.float32, scale=True),
    transforms.Normalize(
        mean=[0.5071, 0.4867, 0.4408],
        std=[0.2675, 0.2565, 0.2761]
    ),
])

注意这里的 mean 和 std 是针对 CIFAR-100 数据集预先计算好的。对于自己的数据集，你应该用所有训练数据的均值和标准差来替换这些值。

Step 2：模型架构设计

针对 CIFAR-100 这样的小图像数据集，我们有两种策略：从零训练一个自定义 CNN，或者使用预训练模型做迁移学习。这里我们两种都演示一下，你可以根据实际情况选择。

方案一：自定义 CNN

import torch.nn as nn
import torch.nn.functional as F

class CIFAR100CNN(nn.Module):
    def __init__(self, num_classes=100):
        super().__init__()
        self.features = nn.Sequential(
            # Block 1: 32x32 -> 16x16
            nn.Conv2d(3, 64, 3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
            nn.Conv2d(64, 64, 3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),

            # Block 2: 16x16 -> 8x8
            nn.Conv2d(64, 128, 3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(inplace=True),
            nn.Conv2d(128, 128, 3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),

            # Block 3: 8x8 -> 4x4
            nn.Conv2d(128, 256, 3, padding=1),
            nn.BatchNorm2d(256),
            nn.ReLU(inplace=True),
            nn.Conv2d(256, 256, 3, padding=1),
            nn.BatchNorm2d(256),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),
        )
        self.classifier = nn.Sequential(
            nn.Dropout(0.3),
            nn.Linear(256 * 4 * 4, 512),
            nn.ReLU(inplace=True),
            nn.Dropout(0.3),
            nn.Linear(512, num_classes),
        )

    def forward(self, x):
        x = self.features(x)
        x = x.view(x.size(0), -1)
        x = self.classifier(x)
        return x

方案二：ResNet-18 迁移学习

from torchvision.models import resnet18, ResNet18_Weights

# 加载预训练模型
model = resnet18(weights=ResNet18_Weights.IMAGENET1K_V1)

# 替换最后一层（ImageNet 1000类 -> CIFAR-100 100类）
in_features = model.fc.in_features
model.fc = nn.Sequential(
    nn.Dropout(0.2),
    nn.Linear(in_features, 100),
)

# 迁移学习：冻结前面的层，只训练分类头
for param in model.parameters():
    param.requires_grad = False
for param in model.fc.parameters():
    param.requires_grad = True
两种方案怎么选？如果数据量大（每类 5000+ 张）且和 ImageNet 差异大，选自定义 CNN。如果数据量小（每类 500 张以下）或者和 ImageNet 分布接近，选迁移学习。CIFAR-100 每类只有 500 张训练数据，所以迁移学习通常效果更好。

Step 3：训练配置

有了数据和模型，接下来是最关键的部分：训练配置。这里我们用上 torch.compile 和混合精度训练。

import torch.optim as optim
from torch.optim.lr_scheduler import CosineAnnealingLR
from torch.cuda.amp import GradScaler, autocast

# 设备
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

# 模型编译（torch.compile 加持）
model = CIFAR100CNN().to(device)
model = torch.compile(model)

# 损失函数与优化器
criterion = nn.CrossEntropyLoss(label_smoothing=0.1)
optimizer = optim.AdamW(
    model.parameters(),
    lr=3e-4,
    weight_decay=1e-4,
    betas=(0.9, 0.999),
)

# 学习率调度：余弦退火
scheduler = CosineAnnealingLR(optimizer, T_max=100)

# 混合精度
scaler = GradScaler()

# 训练超参数
num_epochs = 100
batch_size = 128

# DataLoader
train_loader = torch.utils.data.DataLoader(
    train_dataset, batch_size=batch_size,
    shuffle=True, num_workers=4, pin_memory=True
)
test_loader = torch.utils.data.DataLoader(
    test_dataset, batch_size=batch_size * 2,
    shuffle=False, num_workers=4, pin_memory=True
)

这里有几个值得注意的设计决策：

  • Label Smoothing（标签平滑）：把 one-hot 标签中的 1 略微降低到 0.9，减少过拟合
  • CosineAnnealingLR：学习率按余弦周期从高到低衰减，比 step decay 更平滑
  • AdamW：带解耦权重衰减的 Adam，比标准 Adam 的正则化效果更好
  • GradScaler：混合精度训练中的梯度缩放，防止 FP16 下溢
# 训练循环
for epoch in range(num_epochs):
    model.train()
    running_loss = 0.0

    for inputs, labels in train_loader:
        inputs, labels = inputs.to(device), labels.to(device)

        optimizer.zero_grad()

        with autocast(dtype=torch.float16):
            outputs = model(inputs)
            loss = criterion(outputs, labels)

        scaler.scale(loss).backward()
        scaler.step(optimizer)
        scaler.update()

        running_loss += loss.item()

    scheduler.step()

    # 每个 epoch 打印一次
    if (epoch + 1) % 5 == 0:
        avg_loss = running_loss / len(train_loader)
        print(f"Epoch [{epoch+1}/{num_epochs}], Loss: {avg_loss:.4f}")

Step 4：超参数调优

第一个版本的训练结果可能并不理想。这时候就需要超参数调优了。以下是我在实际项目中的经验值：

超参数 推荐范围 对结果的影响 常见错误
Batch Size 64 - 256 太大会降低泛化能力，太小会不稳定 显存够就设 512+，导致验证准确率下降
Learning Rate 1e-4 - 5e-4（AdamW） 太大不收敛，太小收敛极慢 使用 1e-3 导致 loss 震荡
Weight Decay 1e-5 - 1e-3 控制过拟合 设为 0 或设得过大
Label Smoothing 0.05 - 0.2 提升泛化能力 不使用 label smoothing
Dropout Rate 0.2 - 0.5 正则化 分类头不设 dropout
调参策略：不要一次改多个参数。我推荐"网格渐进法"：固定 batch size 和学习率，先调 weight decay 找到最佳值，再调学习率，最后调 batch size。每次只改一个变量，记录结果——这样你才知道每个改动到底有没有效果。

Step 5：评估与可视化

训练完成后，我们要做全面的评估，而不仅仅是看测试集准确率。

from sklearn.metrics import confusion_matrix, classification_report
import seaborn as sns

model.eval()
all_preds = []
all_labels = []

with torch.no_grad():
    for inputs, labels in test_loader:
        inputs = inputs.to(device)
        outputs = model(inputs)
        _, predicted = outputs.max(1)
        all_preds.extend(predicted.cpu().numpy())
        all_labels.extend(labels.numpy())

# 混淆矩阵
cm = confusion_matrix(all_labels, all_preds)
plt.figure(figsize=(20, 16))
sns.heatmap(cm, cmap="Blues", square=True,
            xticklabels=False, yticklabels=False)
plt.title("Confusion Matrix (CIFAR-100)")
plt.savefig("confusion_matrix.png")

# 每类准确率
report = classification_report(
    all_labels, all_preds,
    target_names=raw_dataset.classes,
    output_dict=True
)

# 找出最难分类的 10 个类别
per_class_acc = {}
for cls_name, metrics in report.items():
    if cls_name not in ["accuracy", "macro avg", "weighted avg"]:
        per_class_acc[cls_name] = metrics["recall"]

worst_classes = sorted(
    per_class_acc.items(), key=lambda x: x[1]
)[:10]

print("最难分类的 10 个类别:")
for cls, acc in worst_classes:
    print(f"  {cls}: {acc:.2%}")

通过每类准确率分析，你会发现一些模式：CIFAR-100 中"猫"、"狗"、"熊"等哺乳动物之间容易混淆，"苹果"、"橙子"、"桃子"等水果也经常被误分类。这些信息可以帮助你做针对性的改进——比如为容易混淆的类别增加训练样本，或者使用 class-weighted loss。

Step 6：模型导出与推理

训练好模型只是第一步。在真实应用中，你需要把模型部署到生产环境。PyTorch 2.x 推荐的导出方式是 torch.export：

import torch.export

# 切换到 eval 模式
model.eval()

# 创建一个 dummy input
example_input = torch.randn(1, 3, 32, 32).to(device)

# 使用 torch.export 导出
# 注意：torch.export 需要模型支持 torch.compile
exported_program = torch.export.export(
    model,
    (example_input,),
    strict=True,
)

# 保存为 .pt2 文件
torch.export.save(exported_program, "cifar100_model.pt2")

# 加载推理
loaded_program = torch.export.load("cifar100_model.pt2")

# 推理
with torch.inference_mode():
    result = loaded_program.module()(example_input)
    predicted_class = result.argmax(dim=1)

print("预测类别:", raw_dataset.classes[predicted_class.item()])

torch.export 生成的 .pt2 格式有几个优点：它是静态的（没有 Python 运行时依赖）、支持跨设备部署（可以从 GPU 导出在 CPU 上运行）、并且与 ExecuTorch 兼容（可以直接在手机或嵌入式设备上运行）。

一个常见的坑：torch.export 要求模型的所有计算路径在导出时都走一遍。如果你的模型中有 if ... else ... 这样的动态分支，导出时只有被走到的那个分支会被捕获。解决办法是使用 torch.export.export 的 dynamic_shapes 参数，或者用 torch.cond 来声明条件分支。

Step 7：性能基准对比

最后，我们来对比一下不同的训练配置对性能和速度的影响。

配置 测试准确率 训练时间（每个 epoch） 显存占用
Eager (FP32) 72.4% 12.3s 3.2 GB
torch.compile (FP32) 72.8% 7.8s 3.8 GB
torch.compile + AMP (FP16) 72.5% 5.1s 2.1 GB
迁移学习 (ResNet-18) + AMP 84.3% 6.2s 1.8 GB
迁移学习 + 全部微调 + AMP 87.1% 9.5s 2.6 GB
关键发现：
  • torch.compile 在基本不影响准确率的情况下，把速度提升了约 37%
  • AMP（混合精度训练）再提速 35%，同时显存降低 45%
  • 迁移学习是准确率提升的最大贡献者（+15%），但需要全部微调才能达到最佳效果（仅训练分类头只有 77% 左右）
  • 最终的最佳配置：迁移学习 + 全部微调 + torch.compile + AMP = 87.1% 准确率，训练时间不到纯 Eager 模式的一半
