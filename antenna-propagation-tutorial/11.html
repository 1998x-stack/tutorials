11.1 Friis传输方程的推导

大家好，我是你们的讲师。从今天开始，我们离开天线本身，进入电波传播的世界。你可能设计了一个完美匹配的天线、一个增益极高的阵列——但如果电波到达不了接收端，一切努力都是白费。所以，理解电波在空间中如何传播，与理解天线本身同样重要。

问题：已知发射天线输入功率P_t，发射天线增益G_t，接收天线增益G_r，两天线相距d，工作波长λ——求接收天线输出端的功率P_r。

这是每个无线通信工程师每天都要面对的核心问题。答案就是Friis传输方程（Friis transmission equation），由丹麦裔美国工程师Harald T. Friis在1946年提出。

我们来逐步推导：

Step 1：发射天线产生的功率密度

各向同性（isotropic）辐射时，在距离d处的球面上，功率密度为：

S_iso = P_t / (4πd²) (W/m²)

但发射天线的增益G_t意味着它把能量集中到了某个方向。所以在该方向上的实际功率密度为：

S = P_t · G_t / (4πd²)

Step 2：接收天线的有效口径

接收天线的有效口径（effective aperture）A_e与增益的关系为：

A_e = G_r · λ² / (4π)

Step 3：接收功率

接收到的功率等于入射功率密度乘以接收天线的有效口径：

P_r = S · A_e = [P_t · G_t / (4πd²)] · [G_r · λ² / (4π)]

整理即得：

Pr / Pt = Gt Gr (λ / 4πd)²

这个公式就是Friis传输方程。它Beautiful地简洁——仅用五个参数就描述了自由空间中的链路传输。

在工程中，更常用的是dB形式。取10log₁₀：

Pr(dBm) = Pt(dBm) + Gt(dBi) + Gr(dBi) - 20log₁₀(4πd/λ)

物理直觉：Friis公式中的(λ/4πd)²项意味着：距离每增加一倍，接收功率下降6 dB（即原来的1/4）。这是因为功率分布在更大的球面上，且接收天线有效口径也在缩小（在高频时尤为显著）。所以高频通信损耗更大——这是5G毫米波基站密度远高于4G的根本原因。

11.2 自由空间路径损耗

在Friis公式中，我们定义自由空间路径损耗（Free Space Path Loss, FSPL）：

FSPL(dB) = 20log₁₀(4πd/λ) = 20log₁₀(4π) + 20log₁₀(d) + 20log₁₀(f) - 20log₁₀(c)

代入c=3×10⁸ m/s，简化得到工程中最常用的形式：

FSPL(dB) = 32.45 + 20log₁₀(fMHz) + 20log₁₀(dkm)

其中f_MHz是以MHz为单位的频率，d_km是以km为单位的距离。这个形式极其方便——不需要换算单位，直接把数字代入即可。

两个关键洞察：

  • 距离翻倍 → 路径损耗增加6 dB
  • 频率翻倍 → 路径损耗增加6 dB

第二个洞察有着深远的影响：为什么5G的覆盖范围比4G小得多？因为5G工作在更高的频段（3.5 GHz vs 2.1 GHz），仅自由空间路径损耗就差了约4.4 dB。到了毫米波（28 GHz vs 3.5 GHz），这个差距扩大到18 dB——意味着在同等发射功率下，毫米波的覆盖半径只有sub-6 GHz的约1/8！

下面是Python实现的链路预算计算函数：

import numpy as np

def fspl(d, f):
    """
    计算自由空间路径损耗 (dB)
    参数:
        d : 距离 (m)
        f : 频率 (Hz)
    返回:
        L : 路径损耗 (dB)
    """
    c = 3e8
    L = 20 * np.log10(d) + 20 * np.log10(f) + 20 * np.log10(4 * np.pi / c)
    return L

def link_budget(Pt_dBm, Gt_dBi, Gr_dBi, d, f, L_extra=0):
    """
    完整链路预算计算
    参数:
        Pt_dBm : 发射功率 (dBm)
        Gt_dBi : 发射天线增益 (dBi)
        Gr_dBi : 接收天线增益 (dBi)
        d      : 距离 (m)
        f      : 频率 (Hz)
        L_extra: 额外损耗 (dB)
    返回:
        Pr_dBm : 接收功率 (dBm)
        FSPL   : 自由空间路径损耗 (dB)
    """
    FSPL = fspl(d, f)
    Pr_dBm = Pt_dBm + Gt_dBi + Gr_dBi - FSPL - L_extra
    return Pr_dBm, FSPL

# 对比三个场景
scenarios = [
    ("FM广播",   100e3,   1.0e8,  10, 6, 0),
    ("4G基站",   40000,   2.1e9,  18, 18, 3),
    ("5G毫米波", 100,     28e9,   23, 23, 5),
]

print("场景      | 距离(m) | 频率   | FSPL(dB) | Pr(dBm)")
print("-" * 55)
for name, d, f, Gt, Gr, Le in scenarios:
    Pr, F = link_budget(30, Gt, Gr, d, f, Le)
    freq_str = f"{f/1e6:.0f} MHz" if f < 1e9 else f"{f/1e9:.1f} GHz"
    print(f"{name:8s} | {d:6.0f} | {freq_str:8s} | {F:7.1f}  | {Pr:7.1f}")
# 输出示例:
# 场景      | 距离(m) | 频率   | FSPL(dB) | Pr(dBm)
# --------------------------------------------------------
# FM广播    | 100000  | 100 MHz |  92.4    | -46.4
# 4G基站    |  40000  | 2.1 GHz | 130.9    | -67.9
# 5G毫米波  |    100  | 28 GHz  | 101.4    | -30.4

11.3 接收功率与电场强度的换算

在实际工程中，我们有时更需要知道的不是接收功率而是场强（field strength）——特别是在EMC测试和无线电监测中。场强与功率密度的关系是：

S = E² / η₀

其中η₀ = 120π ≈ 377 Ω是自由空间波阻抗。联立功率密度与接收功率的关系：

S = P_r / A_e = P_r · 4π / (G_r λ²)

于是可以得到场强与接收功率的换算公式：

E = √(η₀ · P_r · 4π / (G_r λ²)) (V/m)

在EMC测试中，常用dBμV/m表示场强。如果一个测量的接收功率是P_r(dBm)，天线的增益是G_r(dBi)，天线因子（antenna factor）AF = E/P_r可以查表得到。

天线因子：天线制造商通常会提供天线的"天线因子"（AF, 单位dB/m），定义为E-field与接收电压的比值。一旦你知道AF，只要用频谱仪测出接收电平，就能直接换算成场强——这是EMC工程师的日常操作。天线因子通常是通过标准场强校准法测得的。

下表列出了典型通信系统的接收灵敏度：

通信系统 带宽 典型灵敏度 (dBm) 所需SNR (dB) 噪声系数 (dB)
FM广播（模拟） 200 kHz -100 30 5
GSM（2G） 200 kHz -102 9 6
LTE（4G） 20 MHz -98 (参考) 1.5 (EVM需求) 5
WiFi 6 (802.11ax) 20 MHz -82 (256-QAM) 25 5
5G NR 100 MHz -93 (参考) 3 (EVM需求) 6
GPS L1 2 MHz -130 - (扩频增益30 dB) 3
注意灵敏度陷阱：上表中的灵敏度只是参考值。实际接收机的灵敏度取决于噪声系数、解调要求的SNR、以及实现余量（implementation margin）。特别是GPS，热噪声本底在2 MHz带宽下约为-111 dBm，但GPS信号本身低于噪声本底约20 dB——这就是为什么我们说GPS信号是"埋在噪声里的"。接收机通过扩频增益（processing gain）把信号解调出来。

11.4 视距传播极限与有效地球半径

在自由空间模型中，我们假设天线之间没有障碍物。但在实际地球上，最根本的障碍是——地球是圆的。即使天线之间没有任何建筑物或山脉，地球曲率本身也会挡住电波的直线传播路径。

对于高度为h的天线，地球曲率形成的几何地平线距离为：

d(km) ≈ √(2Rh) ≈ 3.57√hm

其中R=6371 km是地球半径，h_m是天线高度（单位m）。当发射和接收天线都有一定高度时，最大视距距离为：

dmax(km) ≈ 3.57(√h₁ + √h₂)

举个例子：一座300 m高的电视发射塔，加上10 m高的接收天线，最大视距约为3.57×(√300 + √10) ≈ 73 km。

但实际的大气并不是均匀的——对流层的折射率随高度递减，使电波路径朝地球方向弯曲。也就是说，电波实际上可以传播得比几何视线更远。为了简化问题，工程中用有效地球半径模型（4/3 Earth radius model）：

Reff = (4/3) R ≈ 8495 km

这被称为"标准大气折射"条件。用R_eff代替R重新计算视距：

dmax(km) ≈ 4.12(√h₁ + √h₂)

故事：为什么电视塔要建那么高？著名的东京晴空塔（Tokyo Skytree）高634 m——它在2012年取代了333 m的东京塔，成为世界最高的电视塔。为什么非建这么高？每增加1米高度，就能多覆盖约4 km的范围（4.12 × 1/√h 的近似增量）。在东京这样人口密集、高楼林立的城市，天线高度直接决定了可覆盖的户数——这背后是巨大的商业利益。高塔的代价也很惊人：晴空塔的建设成本约650亿日元（约6亿美元），但覆盖了东京圈约3900万人口。

11.5 菲涅尔区的概念与清除准则

即使天线之间"看到"了对方（有视距路径），也不意味着信号传输就是自由空间传播。实际上，电波传播路径上有一个椭球形的区域——称为菲涅尔区（Fresnel zones）——对信号传播起到了关键作用。

菲涅尔区是从发射点到接收点的椭球面，其边界满足：从Tx到椭球面再到Rx的路径长度比Tx-Rx直达路径长n·λ/2。n=1称为第一菲涅尔区（first Fresnel zone），n=2是第二菲涅尔区，以此类推。

前几个菲涅尔区对信号贡献的相位交替变化：第一菲涅尔区贡献同相（加强），第二菲涅尔区贡献反相（减弱），第三又加强……总体而言，约70%的传输功率来自第一菲涅尔区。

第一菲涅尔区的半径计算公式为：

r₁ = √(λ · d₁ · d₂ / d)

其中d₁和d₂分别是障碍物到Tx和Rx的距离，d = d₁ + d₂是总路径长度。当障碍物位于中点时，r₁取得最大值：

r₁,max = √(λ · d) / 2

工程清除准则：为了保证传播接近自由空间条件，障碍物必须至少清除第一菲涅尔区的60%。也就是说，如果天线之间有个建筑物，它的顶部必须低于Tx-Rx连线的垂直距离至少0.6×r₁。如果清除不满足，就会产生额外的绕射损耗。

实用口诀：对于典型的微波链路（6-40 GHz），第一菲涅尔区半径随距离和频率的变化大致是：d(km)和f(GHz)用公式r₁(m) = 8.66√(d/f)来估算。例如10 km、15 GHz的链路，中点处r₁≈22 m。这意味着你需要清出22 m×60%≈13 m的净空高度。

当障碍物侵入第一菲涅尔区时，会引入额外的绕射损耗（diffraction loss）。如果障碍物刚好与视线相切（即0%清除），损耗约为6 dB。如果障碍物阻塞到视线以下，损耗迅速增加。具体的绕射损耗计算我们将在第13章详细讲解。

下面是一个计算菲涅尔区清除条件和预测绕射损耗的小工具：

import numpy as np

def fresnel_zone_radius(d1, d2, f):
    """
    计算第一菲涅尔区半径
    参数:
        d1, d2 : 距离 (m)
        f      : 频率 (Hz)
    返回:
        r1     : 第一菲涅尔区半径 (m)
    """
    c = 3e8
    lam = c / f
    d = d1 + d2
    r1 = np.sqrt(lam * d1 * d2 / d)
    return r1

def clearance_loss(h_obstacle, h_tx, h_rx, d1, d2, f):
    """
    估算障碍物引起的额外损耗
    参数:
        h_obstacle: 障碍物高度 (m)
        h_tx, h_rx: 天线高度 (m)
        d1, d2    : 障碍物到两端距离 (m)
        f         : 频率 (Hz)
    返回:
        loss      : 绕射损耗 (dB)
        clearance : 清除量 (占第一菲涅尔区半径的百分比)
    """
    # 视线在障碍物处的高度 (线性插值)
    d = d1 + d2
    h_los = h_tx + (h_rx - h_tx) * d1 / d
    # 障碍物侵入量
    h_clear = h_los - h_obstacle
    r1 = fresnel_zone_radius(d1, d2, f)
    clearance_pct = h_clear / r1 * 100

    # 近似绕射损耗 (刃峰绕射简化模型)
    if clearance_pct >= 60:
        loss = 0  # 自由空间传播
    elif clearance_pct >= 0:
        loss = 6 * (1 - clearance_pct / 60)
    else:
        # 障碍物突出视线: 使用刃峰绕射近似
        v = abs(h_clear) * np.sqrt(2 * d / (lam * d1 * d2))
        if v > 1:
            loss = 13 + 20 * np.log10(v)
        else:
            loss = 6 + 12 * v

    return loss, clearance_pct

# 示例
f = 15e9   # 15 GHz
d = 10000  # 10 km
d1 = 5000  # 障碍物在中间
d2 = 5000
h_tx = 50  # 铁塔高50 m
h_rx = 20
h_obs = 35 # 障碍物高35 m

loss, clr = clearance_loss(h_obs, h_tx, h_rx, d1, d2, f)
print(f"频率: {f/1e9:.1f} GHz")
print(f"障碍物处视线高度: {h_tx + (h_rx-h_tx)*d1/d:.1f} m")
print(f"清除量: {clr:.1f}%")
print(f"绕射损耗: {loss:.1f} dB")
# 输出示例:
# 频率: 15.0 GHz
# 障碍物处视线高度: 35.0 m
# 清除量: 0.0%
# 绕射损耗: 6.0 dB
实际工程中的坑：很多工程师在设计点对点微波链路时只考虑了自由空间路径损耗，忽略了菲涅尔区清除问题。结果安装调试时发现信号比预算少了10-20 dB——罪魁祸首往往是障碍物的绕射损耗。更糟的是，树木生长会逐渐侵入菲涅尔区：一条链路验收时100%清除，三年后树木长高，清除降到40%，信号开始出现频繁中断。所以设计微波链路时，一定要留出足够的净空余量，并考虑植被生长和建筑规划的动态变化。

现在我们已经理解了自由空间传播，但现实世界的地面从来都不是平坦的。下一章我们将在Friis公式的基础上引入地面反射——你会发现，"双射线模型"下接收功率随距离的衰减从1/d²变成了1/d⁴，更糟糕的是信号强度会在空间中出现剧烈的起伏振荡：这就是为什么你的WiFi信号隔了几个房间就时强时弱，而不仅仅是"越来越弱"。
