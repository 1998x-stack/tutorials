第四章：贝尔曼方程

同学们好！欢迎来到第四章。前两章我们认识了马尔可夫决策过程（MDP），也聊了策略迭代和价值迭代的基本思路。但有一个关键问题我们一直没深究：状态的价值到底怎么定义？怎么计算？

这一章的主角——贝尔曼方程（Bellman Equations）——就是回答这个问题的。它由数学家理查德·贝尔曼（Richard Bellman）在1950年代提出，是整个强化学习的数学基石。可以说，不理解贝尔曼方程，就不算真正理解强化学习。

4.1 价值函数的定义

在深入方程之前，我们先明确一个核心概念："价值"到底是什么？

直觉上，一个状态的价值，就是从这个状态开始，智能体未来能获得的全部累积奖励。但"未来"是无限的，所以我们引入了一个关键思想：折扣（Discounting）。

定义状态价值函数 Vπ(s) 为：

Vπ(s) = Eπ[ Gt | St = s ] = Eπ[ Rt+1 + γ Rt+2 + γ2 Rt+3 + ... | St = s ]

其中 γ ∈ [0,1] 是折扣因子（discount factor）。γ 越接近 1，智能体越"远视"；γ 越接近 0，智能体越"短视"。

为什么需要折扣因子？三个原因：1）数学上保证无限时间步的累积奖励收敛；2）模拟真实世界——今天的奖励比明天的更有价值；3）减少未来的不确定性。实际应用中，0.9 到 0.99 是最常见的取值区间。

类似地，我们定义动作价值函数（也称为 Q 函数）：

Qπ(s, a) = Eπ[ Gt | St = s, At = a ]

V 和 Q 之间有一个直观的关系：一个状态的价值，等于该状态下所有可能的动作的 Q 值的加权平均（按策略 π 的概率）：

Vπ(s) = Σa π(a|s) · Qπ(s, a)

这句话的意思是：你站在一个岔路口，每一条路的价值都不一样，而你最终能获得的期望价值，就是你选择各条路的概率乘以各条路的价值之和。

我们来直观对比一下 V 和 Q：

对比维度V(s) 状态价值函数Q(s, a) 动作价值函数
输入状态 s状态 s 和动作 a
含义处于状态 s 有多"好"在状态 s 执行动作 a 有多"好"
决策使用需要知道策略 π 才能选出动作直接比较不同动作的 Q 值即可
算法适用策略评估（如 DP）无模型方法（如 Q-Learning）
依赖策略V(s) 隐式依赖 πQ(s, a) 也隐式依赖 π

4.2 贝尔曼期望方程

现在到了最核心的部分。贝尔曼方程的精髓只有一句话：一个状态的价值，等于即时奖励加上下一状态的折扣价值。这就是"递归"（recursion）的思想。

我们从定义出发推导：

Vπ(s) = Eπ[ Rt+1 + γ Rt+2 + γ2 Rt+3 + ... | St = s ]
= Eπ[ Rt+1 + γ ( Rt+2 + γ Rt+3 + ... ) | St = s ]
= Eπ[ Rt+1 + γ Vπ(St+1) | St = s ]

看明白了吗？从 t+1 开始往后的累积折扣奖励，恰好就是 Vπ(St+1) 的定义！所以价值函数自己"递归"到了自己——这就是贝尔曼方程。

展开期望，写成更具体的形式就是：

Vπ(s) = Σa π(a|s) Σs' P(s' | s, a) [ R(s, a, s') + γ Vπ(s') ]

这个式子从左往右读：

  1. 在状态 s，按策略 π 选择动作 a（概率 π(a|s)）；
  2. 执行 a 后，环境以概率 P(s'|s, a) 转移到下一状态 s'，同时给出奖励 R(s, a, s')；
  3. 对于每一个可能的 s'，我们加上后续价值 γ Vπ(s')；
  4. 最后对所有 a 和 s' 求期望（加权平均）。

对应的，Q 函数的贝尔曼期望方程为：

Qπ(s, a) = Σs' P(s' | s, a) [ R(s, a, s') + γ Σa' π(a'|s') Qπ(s', a') ]
直观理解：贝尔曼方程本质上是一种"自洽性条件"（consistency condition）。它说：你对当前状态的价值估计，必须和你对未来状态的价值估计保持一致。如果 V(s) 和 V(s') 不满足这个等式，那你的价值估计就是不一致的——至少有一个是错的。

4.3 贝尔曼最优方程

上面我们讨论的是给定策略 π 下的价值函数。但如果我们想知道最优策略对应的价值呢？换句话说，最好的状态价值是多少？

这时，我们把策略的选择从"按概率加权平均"换成"选最好的那个动作"：

V*(s) = maxa Σs' P(s' | s, a) [ R(s, a, s') + γ V*(s') ]
Q*(s, a) = Σs' P(s' | s, a) [ R(s, a, s') + γ maxa' Q*(s', a') ]

注意这里的 max 操作——这是最优方程和期望方程最根本的区别。期望方程是线性的，因为它是加权平均；而最优方程是非线性的，因为 max 是一个非线性算子。

这个非线性的 max 带来了一个重要的后果：贝尔曼最优方程没有闭式解。你不能像解线性方程组那样一步到位算出 V*，必须通过迭代逼近。这正是后续几乎所有强化学习算法（Q-Learning、DQN 等）存在的原因——它们都在用各种方式逼近这个最优方程的解。

一个重要的区分：很多初学者会把"最优贝尔曼方程"和"贝尔曼期望方程"混为一谈。记住：期望方程描述"给定策略下的价值"，而最优方程描述"最优策略下的价值"。期望方程是线性的，可以精确求解；最优方程是非线性的，只能迭代逼近。

4.4 备份图（Backup Diagrams）

贝尔曼方程有一个非常直观的可视化方式——备份图。虽然这里没有实际画图，但我用文字给你描述一下：

状态价值函数的备份图：

想象一棵倒置的树。根节点是一个空心圆圈，代表当前状态 s。从 s 分出若干分支（动作 a），每个分支上标记了策略概率 π(a|s)。每个动作分支再连接到环境——环境以概率 P(s'|s, a) 将你送到下一状态 s'（用实心圆圈表示）。每个 s' 又带有它的价值 V(s')。

备份（backup）这个词很形象：你可以把 V(s') 的"信息"沿着树向上"备份"到 s，再加上中间经过的奖励 R，就得到了 V(s)。整个贝尔曼方程就是在描述这个"从叶子到根"的信息传递过程。

踩坑故事：我刚学贝尔曼方程时，犯过一个经典错误——我以为 V(s) 只依赖于 V(s')，没有意识到它其实依赖于 V(s')，而 V(s') 又依赖于 V(s'')，形成了一条无限链条。我试图手动展开递归去"计算"价值，结果算了 3 层就放弃了，觉得这玩意儿根本不可计算。后来才明白：贝尔曼方程的价值不在于展开递归，而在于把无限递归变成了一个可以迭代求解的固定点问题。每一次迭代都是对价值的一次"修正"，最终它会收敛到正确的值。如果你也曾经想手动展开递归，别担心——大家都踩过这个坑。

想一下，这不就是"自洽性"吗？最初的 V(s) 是一个猜测，经过备份之后得到一个新的 V(s)。如果猜测和备份后的值不一致，说明猜测需要调整。反复调整，直到一致——这就是后面所有算法的核心思想。

4.5 从方程到算法

贝尔曼方程是数学，但我们的目标是写出能跑的程序。方程怎么变成算法？答案是：把等号变成赋值。

贝尔曼期望方程说：

V(s) = Σa π(a|s) Σs' P(s'|s, a) [ R + γ V(s') ]

但一开始我们不知道右边的 V(s') 是多少——我们都是猜的。所以算法就是：拿当前的猜测去算右边，然后把算出来的值作为新的 V(s)：

# 贝尔曼更新伪代码：策略评估
# V 是价值函数的当前估计（初始化为 0）
# π 是待评估的策略
# MDP 已知（即 P 和 R 已知）
# θ 是收敛阈值

repeat:
    Δ ← 0
    for each state s in S:
        v ← V[s]
        sum ← 0
        for each action a in A:
            action_sum ← 0
            for each next state s' in S:
                prob ← P(s' | s, a)
                reward ← R(s, a, s')
                action_sum += prob * (reward + γ * V[s'])
            sum += π(a|s) * action_sum
        V[s] ← sum
        Δ ← max(Δ, |v - V[s]|)
until Δ < θ

这段代码的核心逻辑就是：用贝尔曼期望方程作为更新规则，反复把右边算出的值赋给左边。每一次更新都让 V 离真实值更近一步。这个过程叫做策略评估（Policy Evaluation），也叫预测问题（Prediction Problem）。

这个更新式的关键就是贝尔曼误差（Bellman Error）：

δ(s) = | Vnew(s) - Vold(s) |

当所有状态的 δ(s) 都接近 0 时，我们就说 V 收敛到了该策略下的真实价值函数。这个误差也可以作为损失函数来训练神经网络——这正是后续 DQN 的核心思想。

预告一下：下一章我们将看到，动态规划就是直接用上面的"全宽度"（full-width）备份来计算价值。而蒙特卡洛方法（第6章）是用采样代替对所有轨迹求和。时序差分学习（第7章）更进一步，只用一步转移就完成备份——它们都是在不同的"深度"和"宽度"上做同一个贝尔曼方程的近似求解。

本章小结

  • 状态价值函数 V(s) 和 动作价值函数 Q(s,a) 分别衡量"处于一个状态"和"在状态中执行一个动作"的长期期望回报。
  • 贝尔曼期望方程把 V(s) 表达为即时奖励和下一状态价值的递归和，是线性方程。
  • 贝尔曼最优方程把期望替换为 max 操作，是非线性方程，没有闭式解。
  • 备份图直观展示了价值信息如何从后继状态传递回当前状态。
  • 从方程到算法：将等式转化为迭代更新规则，用贝尔曼误差来监控收敛。

掌握了贝尔曼方程，你就拿到了强化学习的"钥匙"。下一章，我们将用动态规划来实际求解贝尔曼方程——看看经典的"全宽度备份"是如何运作的。
