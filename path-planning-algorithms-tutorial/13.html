大家好！前面两章我们学习了 RRT 和 RRT*，它们属于"单次查询"（single-query）方法——从头开始构建树来回答一次查询。但想象一个场景：一个仓库机器人在同一个地图上一天要执行几万次路径规划。每次重新建树太浪费了——如果能预先建好一张"路网"，然后在这个网上跑查询，那该多高效？

这就是 PRM（Probabilistic Roadmap，概率路线图）的核心思想。PRM 由 Lydia Kavraki 等人于 1996 年提出，是"多次查询"（multi-query）采样方法的代表。它花一次离线时间构建覆盖自由空间的路线图，之后就可以在上面高速地回答任意起点到终点的查询。

PRM 的两阶段哲学

PRM 将路径规划分为两个阶段，泾渭分明：

  • 学习阶段（Learning Phase）——在自由空间中随机采样大量点，连接附近的点形成一张图（路线图/roadmap）。这个阶段可以离线完成。
  • 查询阶段（Query Phase）——给定一个起点和终点查询，将起点和终点连接到路线图上，然后在图上运行 A* 或 Dijkstra 搜索。

这种分离带来了一个强大的优势：学习阶段的计算代价可以被无数次查询分摊。对于仓库机器人这种每天做几万次规划的静态环境来说，PRM 几乎是完美的选择。

构建路线图

路线图构建的步骤：

  1. 采样——在自由空间（C_free）中随机采样 N 个配置点。每个点必须是无碰撞的。
  2. 连接——对于每个点，找到它的 k 个最近邻（或用半径 r 内的邻域），尝试在两个点之间建立边。边必须经过碰撞检测——如果边穿过了障碍物，则这条边不加入路线图。
  3. 连通性检查——最终形成的图应该是一个连通的图（或至少包含一个连通的主要组件）。通常我们使用 k=10 或 k=15 作为默认值。

关键的选择参数：

  • 采样点数 N——越多覆盖率越高，但构建时间和内存占用也越大。经验法则：每 100 个自由空间"体积单位"采样 10-50 个点
  • 近邻数 k——连接越多，图越连通，但碰撞检测计算量也越大。典型值 k = 10~20
  • 采样策略——均匀采样是最简单的，但在复杂环境中可以用更聪明的策略（见下面的 tip）
PRM 的阿喀琉斯之踵：窄通道（Narrow Passages）——PRM 最大的弱点就是窄通道。想象一个门缝一样狭窄的通道：随机采样点落在通道中的概率极低（因为通道占自由空间的体积比例很小）。如果路线图中没有点覆盖到通道区域，那么通道两侧的子图就无法连通——PRM 就会错误地认为没有可行路径。这个问题被称为"窄通道问题"，是 PRM 及所有均匀采样方法面临的根本挑战。解决方法包括：桥梁采样（Bridge Sampling）、高斯采样（Gaussian Sampling），或在检测到窄通道时进行局部密集采样。
进阶技巧：高斯采样——一种有效解决窄通道问题的方法是"高斯采样"（Gaussian sampling）。不是均匀地采样整个空间，而是先在空间随机取一个点，然后在它附近取另一个点。如果两个点中恰好一个在障碍物中、一个在自由空间中，那么在自由空间中的那个点就被"推"到了障碍物边界附近。重复这个过程，你得到的就是密集分布在障碍物边界周围的采样点——而这些边界区域恰恰是窄通道最容易出现的地方。在窄通道场景中，高斯采样可以将路线图的连通率从 20% 提升到 90% 以上。

查询路线图

有了路线图后，回答一次路径查询只需要三步：

  1. 把起点 q_start 连接到路线图中最近的 k 个节点上（经过碰撞检测）
  2. 把终点 q_goal 同样连接到路线图上
  3. 在路线图上运行 A* 或 Dijkstra 找到从起点到终点的最短路径

这个过程的计算量很小，因为路线图上的节点数远少于原始空间中的栅格数。A* 在路线图上的搜索几乎瞬间完成。

手写 PRM

import numpy as np
from scipy.spatial import KDTree
import heapq

class PRM:
    def __init__(self, obstacle_check, bounds, n_samples=500, k_neighbors=10):
        self.obstacle_check = obstacle_check
        self.bounds = bounds
        self.n_samples = n_samples
        self.k_neighbors = k_neighbors
        self.nodes = []
        self.edges = {}  # node_idx → list of (neighbor_idx, cost)

    def build_roadmap(self):
        """学习阶段：构建路线图"""
        # Step 1: 采样
        i = 0
        while len(self.nodes) < self.n_samples:
            sample = np.array([np.random.uniform(low, high)
                               for low, high in self.bounds])
            if self.obstacle_check(sample, sample):  # 仅检查点本身是否无碰撞
                self.nodes.append(sample)
                i = 0
            else:
                i += 1
                if i > 1000:  # 预防死循环（环境可能被障碍物完全填满）
                    break

        if len(self.nodes) < 2:
            return  # 采样点太少，无法构建

        # Step 2: k-近邻连接
        tree = KDTree(self.nodes)
        for idx, node in enumerate(self.nodes):
            dists, indices = tree.query(node, k=min(self.k_neighbors + 1, len(self.nodes)))
            for j, dist in zip(indices, dists):
                if j == idx:
                    continue
                # 碰撞检测：检查边的每个插值点
                if self._edge_clear(self.nodes[idx], self.nodes[j]):
                    self.edges.setdefault(idx, []).append((j, dist))
                    self.edges.setdefault(j, []).append((idx, dist))

    def _edge_clear(self, a, b):
        """检查边 a-b 是否无碰撞"""
        steps = max(int(np.linalg.norm(b - a) / 2.0), 5)
        for t in np.linspace(0, 1, steps):
            point = a + t * (b - a)
            if not self.obstacle_check(point, point):
                return False
        return True

    def query(self, start, goal):
        """查询阶段：在路线图上搜索路径"""
        if not self.nodes:
            return None
        tree = KDTree(self.nodes)

        goal_idx = len(self.nodes) + 1

PRM vs RRT 家族

维度 PRM RRT / RRT*
查询类型 多次查询（multi-query） 单次查询（single-query）
构建开销 高（建图阶段） 低（随搜索逐步构建）
单次查询开销 极低（只跑 A*） 高（每次重新建树）
适合场景 静态环境、高频查询 动态环境、单次规划
高维 可行（但窄通道问题恶化） 可行（Voronoi 偏置缓解）
路径质量 依赖于路线图密度 RRT* 渐进最优

一个常见的混合方案是"PRM + RRT*"：用 PRM 的风格预采样构建路线图，然后在路线上运行 RRT* 来进一步优化路径。这种混合利用了 PRM 的离线效率优势和 RRT* 的渐进最优性。

PRM 是一类极其实用的算法——它理解了"一次投资，多次受益"的工程哲学。与 RRT 家族偏向探索不同，PRM 偏向覆盖。在静态已知环境中，PRM 配合适当的采样策略（如高斯采样处理窄通道）几乎是无可争议的王者。
