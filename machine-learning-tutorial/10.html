10.1 如果你的模型只需要一个if-else？

决策树是最接近人类思考方式的机器学习算法。你去医院看病，医生问你"发烧吗？"→ 是 → "咳嗽吗？"→ 是 → "查个血常规"→ 白细胞高 → "开抗生素"。这就是一个决策树。它不同于线性回归的"数学运算"，而是用一系列if-else规则来做决策——直观、可解释、无需特征缩放。

10.2 决策树的生长：分裂的艺术

决策树的训练是一个贪心的递归分裂过程：在每个节点，选一个特征和阈值，把数据分成"最纯"的两组。核心问题是：什么叫"纯"？

纯度度量公式直觉适用任务
基尼系数 (Gini)随机抽两个样本，它们类别不同的概率分类（CART默认）
信息增益 (Entropy)分裂前后不确定性的减少量分类（ID3/C4.5）
均方误差 (MSE)节点内目标值的方差回归（CART回归树）

Gini和Entropy在实际效果上差别很小，但Gini计算更快（没有log运算），所以sklearn默认用Gini。如果你调这两者换来换去发现效果差不多，不用惊讶——确实差不多。

10.3 剪枝：不让树"记住"训练集

一个未经约束的决策树会一直生长到每个叶子节点只包含一个样本——这在训练集上准确率100%，但测试集上可能很差。这就是经典的过拟合。解决方案：

  • 预剪枝（Pre-pruning）：在生长过程中就限制。max_depth（最大深度）、min_samples_split（节点最少样本数才分裂）、min_samples_leaf（叶节点最少样本数）。用得最多
  • 后剪枝（Post-pruning）：先让树充分生长，再从下往上剪掉不重要的分支。如代价复杂度剪枝（ccp_alpha）
from sklearn.tree import DecisionTreeClassifier, plot_tree

# 不加约束的树 → 过拟合的教科书
tree_overfit = DecisionTreeClassifier(random_state=42)
tree_overfit.fit(X_train, y_train)
print(f"训练集: {tree_overfit.score(X_train, y_train):.4f}")  # 可能1.000
print(f"测试集: {tree_overfit.score(X_test, y_test):.4f}")    # 可能0.75

# 加约束的树 → 泛化更好
tree_regular = DecisionTreeClassifier(
    max_depth=5,
    min_samples_split=20,
    min_samples_leaf=10,
    random_state=42
)
tree_regular.fit(X_train, y_train)
print(f"训练集: {tree_regular.score(X_train, y_train):.4f}")  # 0.88
print(f"测试集: {tree_regular.score(X_test, y_test):.4f}")    # 0.86
⚠️ 决策树的致命弱点：树对训练数据的微小变化极度敏感。改一个样本，整棵树的拓扑结构可能都变了。这就是决策树被称为"不稳定学习器"的原因。单个决策树在实际中很少直接使用——我们通常用它的"升级版"：随机森林和梯度提升树。

10.4 随机森林：众人拾柴火焰高

如果你问一个人意见，他可能犯糊涂。如果你问一百个人然后投票，正确率会大幅提升——这就是集成学习（Ensemble Learning）的思想。随机森林是Bagging（Bootstrap Aggregating）的代表作，训练多棵决策树，每棵树看数据的不同子集，最后投票：

  1. 从原始数据中有放回地采样N个样本（Bootstrap）
  2. 随机选择一部分特征（而不是全部特征）
  3. 用这个样本+特征子集训练一棵深度不受限的决策树
  4. 重复1-3步几百次（n_estimators棵树）
  5. 分类：多数投票；回归：取平均值
from sklearn.ensemble import RandomForestClassifier

rf = RandomForestClassifier(
    n_estimators=200,       # 树的数量：越多越稳定，但收益递减
    max_depth=15,            # 每棵树的最大深度
    min_samples_split=5,    # 叶子分裂最少样本数
    max_features='sqrt',    # 每次分裂考虑 sqrt(n_features) 个特征
    bootstrap=True,          # 启用有放回采样
    oob_score=True,          # 启用袋外估计
    n_jobs=-1,               # 并行
    random_state=42
)
rf.fit(X_train, y_train)
print(f"OOB Score: {rf.oob_score_:.4f}")  # 无需验证集的"免费"评估

10.5 OOB误差：免费的验证集

这是随机森林独有的福利。由于每棵树只用了一部分样本（Bootstrap sampling平均用到约63.2%的样本），剩下36.8%的样本没有被看到——这些就是"袋外"（Out-of-Bag, OOB）样本。用这些OOB样本评估模型，等于免费获得了一个验证集，不需要额外划分数据！

OOB Score通常略低于交叉验证的分数（因为只用一部分数据），但它是无偏估计，在数据量有限时尤其有价值。

10.6 特征重要性：随机森林的"隐藏技能"

除了预测，随机森林还能告诉你每个特征有多重要。这是企业应用中非常有价值的功能——不是黑盒预测，而是能解释"为什么"。重要性通过两种方式计算：

  • Gini Importance（默认）：该特征在所有树中用于分裂时，总共减少了多少不纯度
  • Permutation Importance：随机打乱该特征的值，看模型效果下降多少。下降越多越重要。更可靠但更慢
🔑 随机森林 vs GBDT 何时选谁？随机森林是"并行训练多棵树 → 投票"，GBDT是"串行训练 → 每棵树修正前一棵的残差"。RF对超参数不敏感（默认参数就能得到不错效果），抗过拟合能力强，适合快速baseline和特征重要度分析。GBDT（XGBoost/LightGBM/CatBoost）效果通常更好但需要更多调参。一句话：先跑RF做基线，再上GBDT冲性能。
特性随机森林GBDT (XGBoost/LightGBM)
训练方式并行（各树独立）串行（后一棵依赖前一棵）
过拟合倾向低（天然正则化）高（需要调参控制）
默认参数效果好一般（需要调参）
训练速度快（可并行）慢（串行）
竞赛表现中上顶级（表格数据的王者）

树模型的世界非常广阔——这一章我们讲了决策树和随机森林，但真正的"性能猛兽"GBDT系列（XGBoost、LightGBM、CatBoost）会在后续章节中结合实战深入讲解。下一章，我们进入优雅的SVM——一个把分类问题变成几何问题的算法。
