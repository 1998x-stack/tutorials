第14章 NMS与后处理

大家好，我是你们的讲师。NMS这个看似简单的小步骤，却常常成为部署时最容易被忽视的性能瓶颈。你可能调试了半天模型精度，最后发现问题出在后处理的参数设置上。从原始的贪心NMS到Soft-NMS再到DIoU-NMS，每一次改进都在解决"如何去掉冗余框"这个问题。

目标检测模型在推理时，会输出大量冗余的预测框——一个物体可能被几十个框同时命中。后处理的任务就是：从这些候选框中筛选出最终的、最精确的检测结果。

14.1 NMS的基本算法

NMS（Non-Maximum Suppression，非极大值抑制）是最经典的后处理方法。它的核心思想很简单：对于同一类别的重叠框，只保留得分最高的那个。

标准NMS的算法流程：

def nms(boxes, scores, iou_threshold=0.5):
    """Non-Maximum Suppression"""
    # boxes: (N, 4) 格式为 [x1, y1, x2, y2]
    # scores: (N,) 每个框的得分

    # 按得分降序排序
    indices = np.argsort(scores)[::-1]
    keep = []

    while len(indices) > 0:
        # 保留当前得分最高的框
        i = indices[0]
        keep.append(i)

        # 计算其余框与当前框的IoU
        ious = compute_iou(boxes[i], boxes[indices[1:]])

        # 保留IoU小于阈值的框（抑制高重叠的框）
        mask = ious < iou_threshold
        indices = indices[1:][mask]

    return keep

这个算法的时间复杂度是O(N²)，其中N是预测框的数量。对于大部分场景，N=几千到几万个，NMS的开销并不大。但在某些特殊场景——比如密集人群检测，一个图像中可能有数千个目标，NMS的时间就会显著增加。

NMS变体 核心机制 抑制策略 适用场景
贪心NMS IoU > 阈值 → 直接丢弃 硬抑制 稀疏场景
Soft-NMS IoU 越高 → 得分衰减越大 软抑制（线性/高斯） 密集人群、遮挡场景
DIoU-NMS 中心点距离 + IoU 联合判据 硬抑制（改进判据） 遮挡严重场景
Weighted NMS 多个框加权融合为一个 融合替代抑制 高精度需求场景
NMS-free（v10） 训练时消除冗余预测 无NMS 实时性极高场景

14.2 Soft-NMS

标准NMS有一个严重的问题：如果两个真实物体靠得非常近，它们的检测框会有很高的IoU。标准NMS会将得分较低的那个直接丢弃，导致漏检。

Soft-NMS的解决方案是：不要直接删除，而是降低得分。具体来说，当两个框的IoU很高时，不是将低分框丢弃，而是将其得分乘以一个衰减系数：

def soft_nms(boxes, scores, sigma=0.5, score_threshold=0.001):
    """Soft-NMS: 高斯衰减"""
    indices = np.argsort(scores)[::-1]
    keep = []

    while len(indices) > 0:
        i = indices[0]
        keep.append(i)

        # 计算其余框与当前框的IoU
        ious = compute_iou(boxes[i], boxes[indices[1:]])

        # 高斯衰减：得分 *= exp(-IoU² / σ)
        scores[indices[1:]] *= np.exp(-ious ** 2 / sigma)

        # 移除得分低于阈值的框
        mask = scores[indices[1:]] > score_threshold
        indices = indices[1:][mask]

    return keep
Soft-NMS的实际效果： 在COCO数据集上，Soft-NMS相比标准NMS可以提升0.5-1.0%的AP，特别是在AP_S（小物体）和AP_M（中物体）指标上提升更明显。更重要的是，Soft-NMS几乎不需要额外的计算开销——它只是修改了抑制策略，没有改变算法结构。对于大多数场景，高斯Soft-NMS（σ=0.5）是一个安全且有效的默认选择。

14.3 DIoU-NMS

DIoU-NMS是YOLOv4中引入的一种改进NMS。它的核心思想改变了抑制判据：标准NMS使用IoU来决定是否抑制，而DIoU-NMS使用DIoU度量作为判据。

DIoU度量的计算公式我们已经不陌生了：

DIoU = IoU - ρ²(center1, center2) / c²

其中 ρ 是两个中心点的欧氏距离，
c 是同时包含两个框的最小外接矩形的对角线长度。

DIoU-NMS的抑制判据为：

if DIoU(M, B_i) >= iou_threshold:
    抑制 B_i
else:
    保留 B_i

当两个框的中心点距离较远时，即使它们的IoU很大，DIoU-NMS也不会抑制它们。这对于密集场景特别有效——想象一下停在一起的自行车，每个自行车的框之间IoU很高，但中心点距离是不同的，DIoU-NMS会正确保留它们。

什么时候用DIoU-NMS？ 如果你的检测场景中存在大量密集排列的同类物体（如停车场中的车辆、货架上的商品、人群中的行人），DIoU-NMS通常比标准NMS好。在YOLOv4的实验中，DIoU-NMS相比普通NMS在COCO上AP提升了约0.8%，AP_S提升了约1.3%。

14.4 加权框聚类

上述NMS变体的共同问题是：它们只选择了一个框，抛弃了其他所有框。但直觉告诉我们，多个高质量的预测框应该能"综合"出一个更好的结果。加权框聚类（Weighted Box Clustering, WBC）正是基于这个思路。

def weighted_box_clustering(boxes, scores, iou_threshold=0.5):
    """通过加权平均融合多个框"""
    # 1. 按得分排序
    order = np.argsort(scores)[::-1]
    boxes = boxes[order]
    scores = scores[order]

    keep = []
    used = set()

    for i in range(len(boxes)):
        if i in used:
            continue

        cluster = [i]
        for j in range(i + 1, len(boxes)):
            if j in used:
                continue
            iou = compute_iou(boxes[i], boxes[j])
            if iou > iou_threshold:
                cluster.append(j)
                used.add(j)

        # 对当前簇进行加权融合
        cluster_boxes = boxes[cluster]   # (k, 4)
        cluster_scores = scores[cluster] # (k,)

        # 加权平均：得分作为权重
        weights = cluster_scores / cluster_scores.sum()
        fused_box = (cluster_boxes * weights[:, None]).sum(axis=0)

        keep.append(fused_box)

    return np.array(keep)

WBC的优势在于：它生成的边界框比任何单个预测框都更精确，因为它综合了多个高分的预测。缺点是计算量略大，且需要额外处理得分过低的情况。

14.5 YOLOv10的NMS-Free训练

YOLOv10迈出了一大步：它完全去掉NMS。这个看似激进的决策背后，是一个精妙的设计——一致双分配（Consistent Dual Assignments）。

YOLOv10在训练时使用两套标签分配策略：

  • 一对多分配： 一个ground truth匹配多个预测框（与标准YOLO相同），用于提供丰富的梯度信号
  • 一对一分配： 一个ground truth只匹配一个预测框（匈牙利算法），让每个物体只输出一个预测

这种双分配策略，让模型在训练时既获得了足够的监督信号（一对多），又学会了"一个物体只输出一个最佳框"的行为模式（一对一）。推理时只需要使用一对一的分支，每个物体只产出一个预测，自然无需NMS。

NMS-free设计带来了显著的优势：推理pipeline大幅简化，延迟降低了10-20%（NMS虽然不贵，但在某些场景下仍占推理时间的5-10%）。更重要的是，模型的输出是确定性的一对一映射，不会出现"一个物体多个框"的歧义。

踩坑实录：NMS的IoU阈值调优。 在一个工业缺陷检测项目中，我使用YOLOv5默认的NMS IOU阈值0.5，结果发现某些小缺陷被重复检测了5-6次，而某些真正的缺陷却被抑制了。后来通过逐个图片分析发现：缺陷的宽高比非常极端（1:10的细长裂纹），导致裂纹之间的IoU计算很特殊——位置相近但中心偏移大的裂纹IoU反而小。最终将IoU阈值从0.5调整到0.3，并启用了Soft-NMS（σ=0.3），检测精度从0.68提升到了0.82。NMS参数调优是最容易被忽视的优化手段之一。

14.7 总结

NMS和后处理是目标检测推理管线的"最后一公里"。从标准的贪心NMS到Soft-NMS、DIoU-NMS、加权聚类，再到YOLOv10的NMS-free，每一次改进都在回答同一个问题：如何在保留正确预测的同时去除冗余。值得记住的是，后处理的参数（IoU阈值、置信度阈值、NMS方法）是模型部署调试中最容易被忽略、却最容易出效果的优化方向。
