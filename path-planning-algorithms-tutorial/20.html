大家好！上一章我们聊了动态环境中的重规划，学会了如何在变化的世界中持续找到可行路径。但还有一些场景是普通路径规划解决不了的——比如机器人巡检时需要依次查看 20 个仪表盘、无人机需要飞过一片区域的每一个角落进行测绘、快递员需要在一趟行程中配送 50 个包裹。这些问题的共同特征是什么？不再只有一个终点，而是有多个目标点需要依次访问。这就是多目标路径规划（Multi-Goal Path Planning）。

在深入之前，我先讲一个有趣的亲身经历。2019 年我参与了一个仓库盘点机器人的项目——机器人需要在夜间自动巡视仓库，扫描每个货架上的 RFID 标签来盘点库存。仓库有 200 个货架，分布在 5000 平米的场地上。我们一开始天真地给机器人规划了一条"蛇形走位"路线：从最左边第一排走到头，拐到第二排再走回来，像割草一样。结果机器人的电池只能撑 2 小时，但蛇形路线慢悠悠要走 4 小时。问题在哪？因为我们没有优化访问顺序——有些货架明明在附近，我们却让它绕了一大圈才去访问。从那以后我明白了一个道理：在多目标路径规划中，决定先访问谁、后访问谁，往往比路径怎么走更重要。

多目标问题的本质

多目标路径规划在最简单的情况下可以被描述为：给定一个起点、一个终点和 N 个需要经过的中间目标点，找到一条经过所有这些点的最短路径。这听起来是不是有点耳熟？没错，这本质上就是旅行商问题（Traveling Salesman Problem, TSP）。

TSP 的问题是：给定 N 个城市以及两两之间的距离，找到一条经过所有城市且每个城市只经过一次的最短回路。而我们的多目标路径规划稍有不同——它有明确的起点和终点（不一定相同），而且"经过"不一定要求精确到点，只要路径覆盖了目标区域即可。

但它们的核心挑战是一样的：确定最优的访问顺序。如果你有 3 个目标点，那么可能的访问顺序有 3! = 6 种。如果有 10 个点，就有 10! ≈ 362 万种。如果有 20 个点，就有 2.43 × 10¹⁸ 种——即使使用最快的超级计算机暴力枚举也是不可能的。这就是所谓NP-hard问题的威力。

根据不同的场景需求，多目标规划还可以细分为：

  • 覆盖路径规划（Coverage Path Planning）——机器人需要经过区域内每一点（如扫地机器人、植保无人机），目标不是路径最短，而是覆盖率最高、重复率最低。
  • 巡检路径规划（Inspection Path Planning）——需要经过一系列指定的检查点（如电力线巡检、管道检测），目标是最短路径 + 覆盖所有检查点。
  • 配送路径规划（Delivery Route Planning）——在一个城市范围内配送多个包裹，目标是在满足时间窗口约束下的总距离最短。

TSP与路径规划

TSP 是一个非常古老且优美的问题，最早可以追溯到 19 世纪 30 年代。经过近两个世纪的研究，人们对它已经有了非常深的理解：

  • 精确解法——分支定界（Branch and Bound）、整数线性规划（ILP）、动态规划（Held-Karp 算法，O(n²·2ⁿ)）。对于 n ≤ 20 的问题，精确解法可以在合理时间内求解。
  • 近似算法——Christofides 算法（保证解不超过最优解的 1.5 倍）、最小生成树法（2 倍近似比）。对大规模问题有用。
  • 启发式算法——最近邻、2-opt、3-opt 局部搜索、模拟退火、遗传算法。在实践中使用最广泛。
  • 学习型方法——基于注意力机制（Attention）的 Pointer Networks、GCN-based 方法。前沿研究热点。

在路径规划的语境下，多目标规划通常是一个双层优化问题：外层决定访问顺序（这是一个 TSP），内层计算两点之间的最优路径（这是传统的路径规划）。两层耦合起来，使得问题比纯 TSP 更复杂——因为内层的路径代价不是固定的直线距离，而是取决于具体的地形和障碍物分布。

例如，虽然目标 A 和目标 B 的直线距离只有 10 米，但因为中间有一堵墙，实际路径长度可能是 30 米。这就意味着我们不能简单地用欧几里得距离作为 TSP 的代价矩阵，而必须先对所有目标点对运行一次路径规划，得到精确的距离矩阵。

最近邻启发式

在所有求解方法中，最近邻启发式（Nearest Neighbor Heuristic）是最简单、最容易实现的方法。它的逻辑非常直白：

  1. 从起点出发
  2. 在所有未访问的目标点中，选择离当前位置最近的那个
  3. 移动到该点，标记为已访问
  4. 重复步骤 2-3，直到所有目标点都被访问
  5. 如果终点与起点不同，最后移动到终点

这个算法的时间复杂度是 O(n²)，其中的 n 是目标点的数量。对于 50 个目标点，只需要 2500 次距离计算，在毫秒级别就能完成。

但它的缺点是显而易见的——你永远在"贪心"地选择最近的，但最终的总路径长度可能远非最优。数学上已经证明，最近邻启发式在最坏情况下的解可以劣于最优解约 25%。不过在实践中，对于随机分布的目标点，它的效果通常还不错——只需要最优解的 1.1 到 1.25 倍。

遗传算法求解

当目标点数超过 20 且你对路径质量有较高要求时，最近邻可能不够用了。这时候遗传算法（Genetic Algorithm, GA）是一个很不错的选择。

遗传算法模拟了自然选择的过程。在我们的多目标路径规划场景中：

  • 染色体（Chromosome）——一个候选的访问顺序，比如 [3, 1, 4, 0, 2] 表示先访问目标点 3，然后 1，然后 4……
  • 适应度（Fitness）——这条路径的总长度（适应度越高 = 路径越短）
  • 选择（Selection）——适应度高的染色体被选中的概率更大
  • 交叉（Crossover）——将两条父染色体的序列片段组合成新的子代
  • 变异（Mutation）——随机交换染色体中的两个位置

下面是使用遗传算法求解多目标路径规划的完整 Python 实现：

import random
import math

# ---------- 问题设定 ----------
goal_points = [(2, 3), (7, 5), (1, 8), (9, 2), (4, 6),
               (6, 9), (3, 1), (8, 4), (5, 7), (0, 5)]
start = (0, 0)
end = (10, 10)
N = len(goal_points)

# 两点间的路径代价（简化为欧几里得距离，实际中应该是路径规划结果）
def distance(a, b):
    return math.sqrt((a[0]-b[0])**2 + (a[1]-b[1])**2)

# 预先计算所有点对之间的距离矩阵
points = [start] + goal_points + [end]
dist = [[distance(points[i], points[j])
         for j in range(N+2)] for i in range(N+2)]

# ---------- 适应度函数 ----------
def fitness(route):
    """计算总路径长度：start -> route[0] -> ... -> route[-1] -> end"""
    total = dist[0][route[0] + 1]  # start到第一个目标
    for i in range(len(route) - 1):
        total += dist[route[i] + 1][route[i+1] + 1]
    total += dist[route[-1] + 1][N + 1]  # 最后一个目标到end
    return 1.0 / total   # 取倒数，使"越短越好"变成"越大越好"

# ---------- 初始化种群 ----------
POP_SIZE = 100
population = [random.sample(range(N), N) for _ in range(POP_SIZE)]

# ---------- 交叉操作：PMX (Partially Mapped Crossover) ----------
def pmx(parent1, parent2):
    size = len(parent1)
    child = [None] * size
    # 随机选择交叉区间
    cx1, cx2 = sorted(random.sample(range(size), 2))
    # 复制区间
    child[cx1:cx2] = parent1[cx1:cx2]
    # 映射剩余位置
    for i in range(cx1, cx2):
        if parent2[i] not in child:
            pos = i
            while cx1 <= pos < cx2:
                pos = parent2.index(parent1[pos])
            child[pos] = parent2[i]
    for i in range(size):
        if child[i] is None:
            child[i] = parent2[i]
    return child

# ---------- 变异操作：随机交换两个位置 ----------
def mutate(route, rate=0.1):
    if random.random() < rate:
        i, j = random.sample(range(len(route)), 2)
        route[i], route[j] = route[j], route[i]
    return route

# ---------- 主循环 ----------
GENERATIONS = 500
ELITE_SIZE = 5

for gen in range(GENERATIONS):
    # 计算适应度
    scored = [(fitness(route), route) for route in population]
    scored.sort(key=lambda x: x[0], reverse=True)

    # 精英保留
    new_pop = [route for _, route in scored[:ELITE_SIZE]]

    # 轮盘赌选择 + 生成子代
    weights = [s[0] for s in scored]
    while len(new_pop) < POP_SIZE:
        p1 = random.choices(population, weights=weights, k=1)[0]
        p2 = random.choices(population, weights=weights, k=1)[0]
        child = pmx(p1, p2)
        child = mutate(child)
        new_pop.append(child)

    population = new_pop

# ---------- 结果 ----------
best_route = max(population, key=fitness)
best_fitness = fitness(best_route)
total_distance = 1.0 / best_fitness

print(f"最优访问顺序: {best_route}")
print(f"总路径长度: {total_distance:.2f}")
print(f"访问路线: start -> ", end="")
for idx in best_route:
    print(f"g{idx}({goal_points[idx]}) -> ", end="")
print("end")

这段代码的核心在于 PMX 交叉操作——它保证了子代继承了父代中一段完整的访问序列，同时通过映射关系避免了重复目标。经过 500 代进化后，GA 通常能找到非常接近最优解的方案。

真实案例：亚马逊配送无人机的路径规划——亚马逊的 Prime Air 无人机一次起飞需要配送 50 多个包裹，覆盖一个社区的数十个投递点。如果按"先到先得"的顺序投递，一趟行程可能要飞 30 公里。而使用遗传算法优化访问顺序后，距离可以压缩到 15 公里以内——直接让单次配送的能耗减少一半，电池刚刚够用。据亚马逊公开的信息，GA 的适应度计算中还嵌入了风场模型、禁飞区规避和电池余量约束，比我们这里的简化版本复杂得多，但核心思想完全一致：顺序决定一切。

多目标路径规划把 TSP 和传统路径规划结合在了一起，是一个非常实用却又充满挑战的领域。当然，在真实世界中，机器人往往不是单独工作的——当多个机器人在同一空间内同时规划路径时，问题会变得更加复杂。下一章我们就来讨论多机器人协调问题，看看如何在共享空间中让多个机器人在不撞上彼此的情况下高效完成任务。
