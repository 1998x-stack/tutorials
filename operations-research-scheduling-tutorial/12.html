第12章 精确求解方法

大家好，我是你们的讲师。从第 8 章到第 11 章，我们一直在和各种启发式算法打交道——Johnson、CDS、NEH、LPT…… 这些算法快、实用，但有一个共同的"软肋"：它们不保证最优。那么问题来了：当我们需要一个真正的最优解时，该怎么办？

这就引出了今天的主角——精确求解方法（Exact Methods）。这些方法能保证找到全局最优解，或者证明一个解离最优有多远。它们通常比启发式慢得多（毕竟 NP-hard 问题在理论上就不存在多项式时间的精确算法），但它们提供了最优性的确定性——在工业合规、成本核算等场景中，这种确定性往往至关重要。

分枝定界法 — 精确求解的核心框架

分枝定界法（Branch and Bound, B&B）是精确求解组合优化问题最基本的框架。我们在第 10 章已经见过了它在 JSP 中的应用，现在我们来系统地拆解它的四个核心组件。

分支（Branching）—— 如何划分解空间

分支的目的是将原始问题的解空间分解为若干互不相交的子空间。以整数规划为例，假设一个整数变量 x 在上一层的解中取值 x* = 3.7（非整数），则我们可以创建两个子问题：

  • 左分支：添加约束 x ≤ 3
  • 右分支：添加约束 x ≥ 4

在调度问题中，分支策略可以是选择一对操作并决定谁先谁后（析取弧定向），也可以是选择下一个待安排的工件。

定界（Bounding）—— 松弛问题的下界计算

定界是 B&B 的性能关键。下界越紧（越接近真实最优值），剪枝效果越好。常用的定界方法包括：

  • LP 松弛：将整数约束松弛为连续约束，求解线性规划得到下界。
  • 拉格朗日松弛：将"困难"约束吸收到目标函数中，得到比 LP 松弛更紧的下界。
  • 组合下界：利用问题的结构特征，直接计算简单的下界（如机器负载下界、路径下界）。

搜索（Search）—— 遍历树的方式

  • Best-First Search：总是展开下界最小的节点。优点是能最快找到最优解；缺点是内存消耗大（需要维护优先队列）。
  • Depth-First Search：沿着一条路径走到叶子节点。优点是内存消耗小（只需要维护当前路径）；缺点是可能在一个坏分支上浪费大量时间。
  • 有限差异搜索（Limited Discrepancy Search）：假设启发式指引的方向大部分是正确的，只允许与启发式建议有最多 k 次偏离。这种策略在小规模树中非常高效。

剪枝（Pruning）—— 三种剪枝条件

  1. 下界剪枝：当前节点的下界 ≥ 当前全局上界。这是最重要的剪枝。
  2. 不可行剪枝：当前子问题无可行解（如约束冲突导致空集）。
  3. 占优剪枝：存在另一个节点，其产生的解集包含当前节点的解集且下界更优。
分枝定界法 — 通用伪代码

算法 BranchAndBound(问题实例 P)
  输入：优化问题 P（minimization）
  输出：最优解 x* 和目标值 f*

  初始化:
    ub ← ∞  // 全局上界（可行解的目标值）
    x* ← NULL  // 全局最优解
    活动节点列表 ← {P}

  while 活动节点列表非空:
    选择一个节点 N（根据搜索策略）
    从活动节点列表中移除 N

    // 定界
    lb ← 计算下界(N)
    if lb ≥ ub:
      continue  // 剪枝：不可能更好

    // 判断是否可求解
    if 可以求解 N（所有变量已固定）:
      计算 N 的精确解 f(N)
      if f(N) < ub:
        ub ← f(N)
        x* ← N 的解
      continue

    // 分支
    选择分支变量/约束
    创建子节点 N₁, N₂, ..., Nₖ

    for each 子节点 N_child:
      if 不可行(N_child):
        continue  // 剪枝
      lb_child ← 计算下界(N_child)
      if lb_child < ub:
        将 N_child 加入活动节点列表

  return x*, ub

分支切割法 — 在B&B中动态添加割平面

分支切割法（Branch and Cut）是 B&B 的进阶版。它在 B&B 的每个节点上做两件事：(1) 求解 LP 松弛；(2) 如果解是分数解，则添加割平面（cutting plane）来切割掉分数解，使可行域缩小，从而让下界更紧。

分支切割的流程是：

  1. 在一个节点上求解 LP 松弛。
  2. 如果最优解已经是整数解 → 更新上界。
  3. 如果最优解是分数解 → 尝试找到被当前 LP 松弛"忽略"的约束（即割平面），例如：
    • Gomory 割：从单纯形表中自动生成的有效不等式。
    • kl 树不等式：针对 Traveling Salesman Problem（TSP）的经典割平面。
    • 子集和不等式：针对背包问题的割平面。
  4. 如果找到了有效的割平面，添加到 LP 中重新求解 → 重复步骤 2-3。
  5. 如果无法继续添加割平面但解仍然是分数 → 分支。

Branch and Cut 是现代商用求解器（如 Gurobi、CPLEX）的核心技术。它能在大多数中等规模的整数规划问题上在可接受时间内找到最优解。

一个小插曲：1990 年代，CPLEX 的创始人 Bob Bixby 曾说过一句广为流传的话："Mixed Integer Programming 在过去 10 年的速度提升，大部分要归功于 Branch and Cut 的技术进步，而不是计算机硬件的提升。" 根据统计，从 1991 年到 2005 年，MIP 求解器的平均加速达到了惊人的 200,000 倍——其中硬件贡献了约 1,000 倍，算法贡献了约 200 倍。这个数据很好地说明了：好的算法比快的硬件更重要。

列生成 — 适用于变量极多的场景

有些调度和优化问题的特点是：变量数量极其庞大，多到无法在内存中显式存储。例如，在航空机组排班中，每个"可行的排班方案"都是一个变量——一家航空公司数万名空乘人员的可行排班方案数量可能是天文数字。

这时候就需要列生成（Column Generation / CG）。它的核心思想是：不需要一次性考虑所有变量，而是只用一个受限的主问题（Restricted Master Problem, RMP）管理当前"活跃"的变量，然后通过一个定价子问题（Pricing Subproblem）持续地生成新的"有潜力改进解"的变量。

列生成建立在 Dantzig-Wolfe 分解的基础之上。它将原始问题分解为：

  • 主问题（Master Problem）：负责选择哪些"模式"或"路径"被使用，满足全局约束（如覆盖约束）。
  • 子问题（Subproblem / Pricing Problem）：负责生成新的候选模式，这些模式能改善主问题的目标值。

列生成在调度问题中的应用非常广泛。一个经典的例子是裁切料问题（Cutting Stock Problem）转化到调度的类比：在调度中，我们可以把"机器上的一个加工序列"看作一个"模式"，列生成可以动态地生成新的可行序列来改善整个调度方案。

列生成算法框架

输入：问题的主结构（全局约束）和子结构（局部约束）
输出：最优解（可能需要对最后的小数解做取整）

初始化:
  RMP ← 用一组初始列（变量）构建受限主问题
  // 通常使用启发式或人工方式生成初始可行解

迭代:
  while true:
    1. 求解当前 RMP 的 LP 松弛 → 得到对偶变量 π
    2. 用对偶变量 π 构建定价子问题
    3. 求解定价子问题:
       - 找到"检验数"（reduced cost）为负的列
       - 检验数 = 该列的成本 - Σ(对偶变量 × 约束系数)
    4. if 找到检验数为负的新列:
       将新列加入 RMP，返回步骤 1
    5. else:
       当前 LP 松弛已最优 → break

取整:
  如果最终解是分数解，使用启发式取整或进入分支定界
  （完整算法称为 Branch and Price）

列生成与分枝定界结合后的完整方法称为分支定价法（Branch and Price），它是处理超大规模组合优化问题的最强工具之一。

下面我们来对比三种精确方法的特性：

方法 核心思想 适用场景 优势 局限
分枝定界（B&B） 系统性枚举 + 剪枝 中小规模整数规划 实现简单，框架通用 对大规模问题剪枝效率下降
分支切割（Branch and Cut） B&B + 割平面收紧松弛 中等规模整数规划 下界更紧，收敛更快 割平面挖掘需要领域知识
列生成（Column Generation） 动态变量生成 + 定价 变量极多的大规模问题 可处理指数级变量 需要精心设计定价子问题

一个很重要的认知：很多初学者会觉得"精确方法计算太慢，工业界不实用"。这种看法在某种程度上是对的——如果你追求 NP-hard 问题的大规模精确最优解，确实很难。但精确方法还有一个重要的"副产品"价值：下界计算。即使你最终用的是启发式算法，知道精确下界能帮你评估你的启发式离最优有多远。如果你的启发式偏差 5%，而下界显示它离最优只有 2%——那你就知道改进空间不大了。反之，如果偏差 30%，你就知道还有很大的优化余地。没有下界，你只是在黑暗中摸索。所以，请永远不要忽视精确方法——即使你最终不用它来求解。

好的，这一章我们介绍了精确求解的三大支柱：分枝定界是骨架，分支切割让松弛更紧，列生成让变量爆炸不再是噩梦。下一章我们将进入另一个精彩的领域——元启发式算法（Metaheuristics）。如果说精确方法是"追求确定性"的君子，那么元启发式就是"追求效率"的侠客。遗传算法、模拟退火、粒子群、蚁群……这些名字听起来就很有趣。我们下一章见。
