第13章 多语言BERT与跨语言迁移

大家好，我是你们的讲师。假设你接到了一个任务：要为50种语言构建一个文本分类系统，但其中48种语言只有一个句子的标注数据，而英语有10万条。你怎么办？一个一个语言去标注数据？还是找一个能"一劳永逸"的方案？

这就是多语言BERT和跨语言迁移要解决的核心问题。用一句话概括：训练一个模型理解所有语言，然后在一个语言上微调，让它在所有语言上都有效——听起来像魔法，但背后其实有一套严谨的工作原理。

13.1 mBERT：104语言的共享之旅

Google在发布BERT后不久就推出了多语言版本——mBERT (Multilingual BERT)。它的架构和BERT-Base完全一样——12层Transformer、768维隐藏层、12个注意力头——唯一的不同是词表：mBERT使用了一个包含104种语言的共享WordPiece词表，大小约11万token。

mBERT的训练数据是104种语言的Wikipedia页面，采样策略让人津津乐道——指数平滑加权(exponential smoothing)：对每种语言的数据量取0.7次幂作为采样概率。这样既不会让小语种被英语完全淹没，也不会让大语种的数据浪费太多。

一个令人惊讶的事实： mBERT并没有在训练中做任何显式的跨语言对齐——没有翻译对，没有跨语言对比学习，没有代码切换数据。它仅仅是用一个共享的词表和编码器在多种语言上做MLM训练，就自动学习到了跨语言的表示对齐。这说明"共享参数 + 共享subword"本身就足以让模型发现不同语言之间的共性。

13.2 XLM-R：RoBERTa的多语言版

mBERT的效果虽然不错，但它有两个明显的局限：使用Wikipedia数据（规模太小），以及BERT的训练配方不够优化（前面已经讲过了）。Facebook AI在2019年底推出了XLM-R (XLM-RoBERTa)，将RoBERTa的训练配方应用到了多语言场景。

维度mBERTXLM-R
架构BERT-Base (12层, 768d, 12头)RoBERTa-Base/Large
词表大小~110k (WordPiece)~250k (SentencePiece)
训练数据104种语言的Wikipedia100种语言的CommonCrawl (CC-100)
数据规模~13GB (总量)~2TB (总量)
训练任务MLM + NSPMLM (无NSP)
掩码策略静态掩码动态掩码
XNLI (零样本)65.479.2

XLM-R使用250k大小的SentencePiece词表，远大于mBERT的110k，这意味它能更细粒度地处理各种语言的文本。同时，使用CC-100语料（CommonCrawl中过滤出的100种语言数据，总量2TB）代替Wikipedia，数据量提升了两个数量级。

13.3 跨语言零样本迁移机制

跨语言零样本迁移为什么能工作？核心有三个因素：

  1. 共享词表：不同语言的文本通过共享的subword词表被映射到同一个嵌入空间。比如英语的"computer"和德语的"Computer"可能共享相近的subword分解。
  2. 共享Encoder参数：所有语言的文本都通过同一个Transformer编码器处理。模型被迫学习一种"语言无关"的表示，因为编码器不知道当前输入来自哪种语言。
  3. MLM任务的跨语言信号：在MLM训练中，模型需要根据上下文预测被mask的subword。虽然不同语言的上下文不同，但共享的subword提供了跨语言的"锚点"。
from transformers import AutoTokenizer, AutoModelForSequenceClassification

# 加载XLM-R模型和分词器
model_name = "xlm-roberta-base"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=2)

# 在英文数据上微调
english_texts = ["This movie is great!", "I hated this film."]
english_labels = [1, 0]

# ... 在英文数据上做标准微调 ...

# 微调后，在德文上做零样本推理
german_text = "Dieser Film ist großartig!"
inputs = tokenizer(german_text, return_tensors="pt")

with torch.no_grad():
    outputs = model(**inputs)
    prediction = torch.argmax(outputs.logits, dim=-1)
    print(f"德文零样本预测: {prediction.item()}")  # 应该也是正面(1)

13.4 XLM：跨语言预训练的先行者

在mBERT和XLM-R之间，还有一个重要的模型——XLM (Cross-lingual Language Model)，由Facebook AI在2019年1月提出。XLM引入了两个对跨语言预训练至关重要的技术：

  • 翻译语言模型(TLM, Translation Language Model)：在翻译对的平行语料上做MLM——给定一个句子的源语言和翻译，同时mask两种语言中的部分token，模型需要利用一种语言的上下文预测另一种语言的mask位置。这强制了跨语言的对齐。
  • 因果语言模型(CLM)：从左到右的AR语言模型（类似GPT），但跨语言版本的CLM通过在句子间插入语言嵌入(language embedding)来区分不同语言。
实践建议： TLM虽然效果好，但它需要双语平行语料，这在很多低资源语言对上是稀缺的。如果你的场景有足够的平行数据，在XLM-R的基础上继续用TLM目标做增量训练（continued pretraining），往往能获得显著的跨语言迁移提升。

13.5 代码切换数据增强

一个简单但非常有效的跨语言微调技巧是代码切换(Code-Switching)：在微调时将源语言的某些词随机替换成目标语言对应词，或者在句子中插入目标语的subword。这样做让模型在微调阶段就暴露在"混合语言"的环境下，提升了跨语言的泛化能力。

import random

def code_switching(text, source_lang="en", target_lang_dict=None, swap_prob=0.15):
    """
    简单的代码切换数据增强
    将源语言文本中的词按一定概率替换为目标语言词

    参数:
        text: 源语言文本
        target_lang_dict: {源语言词: 目标语言词}的映射字典
        swap_prob: 每个词被替换的概率
    """
    if target_lang_dict is None:
        return text

    words = text.split()
    augmented_words = []

    for word in words:
        # 检查这个词是否有对应的目标语言翻译
        target_word = target_lang_dict.get(word.lower(), None)
        if target_word and random.random() < swap_prob:
            augmented_words.append(target_word)
        else:
            augmented_words.append(word)

    return " ".join(augmented_words)

# 示例：中英代码切换
en_text = "I love this movie because it is very interesting"
en_to_zh = {
    "love": "喜欢",
    "movie": "电影",
    "very": "非常",
    "interesting": "有趣"
}

augmented = code_switching(en_text, target_lang_dict=en_to_zh, swap_prob=0.3)
print(f"原始: {en_text}")
print(f"增强: {augmented}")
# 输出可能: "I love this movie because it is 非常 interesting"
踩坑实录： 代码切换不是"词越多越好"。我曾试过把替换概率设到0.5，结果生成了一堆"中英夹杂"的混乱句子，模型反而困惑了。经验是：替换概率控制在0.1-0.3之间，且优先替换功能性强的词（动词、名词、形容词），不要替换连词、冠词等语法词。

13.6 低资源语言微调策略

当面对低资源语言（只有几十到几百条标注数据）时，以下策略可以最大化跨语言迁移的效果：

策略方法适用场景效果
高→低迁移先在高资源语言上微调，再在低资源语言上做少量微调低资源语言有少量标注⭐⭐⭐⭐⭐
零样本推理直接使用高资源语言微调后的模型在低资源语言上推理低资源语言无标注数据⭐⭐⭐
多任务微调在多个语言上同时微调，共享编码器参数多种低资源语言都有少量标注⭐⭐⭐⭐
语言适配器为每种语言添加轻量级adapter模块，仅训练adapter需要频繁新增语言⭐⭐⭐⭐
数据增强使用回译(back-translation)或代码切换扩充低资源语言数据有双语词典或翻译模型⭐⭐⭐

13.7 中文跨语言BERT：ERNIE-M与InfoXLM

对于中文社区来说，跨语言BERT有一些特殊的版本值得关注：

  • ERNIE-M (百度, 2021)：在XLM的基础上引入了"跨语言MLM"和"反事实训练"，特别优化了中英文对的跨语言表示。ERNIE-M在中文NLP任务上的跨语言迁移效果显著优于mBERT和XLM-R。
  • InfoXLM (微软, 2021)：将XLM-R与对比学习结合，使用互信息最大化(MI-based)作为训练目标。InfoXLM-base在XNLI上比XLM-R-base提升了约2个百分点。
  • XLM-ALIGN (Meta, 2022)：在mC4数据上使用对比学习对齐多语言表示，支持100+语言，在多种跨语言任务上达到SOTA。
选型建议（2025年视角）： 对于大多数跨语言场景，XLM-R仍然是最稳妥的选择——它生态成熟、HuggingFace支持完善、社区资源丰富。如果对中英对译场景有特别高的要求，可以试试ERNIE-M。如果你有充足的计算资源和双语平行语料，可以考虑在XLM-R基础上做TLM继续预训练——效果往往好于直接更换模型。

13.8 一个完整的跨语言微调示例

from transformers import (
    AutoTokenizer,
    AutoModelForSequenceClassification,
    TrainingArguments,
    Trainer
)

# 加载XLM-R
model_name = "xlm-roberta-base"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=3)

# 中文训练数据
train_texts_zh = ["这个产品很好用", "质量太差了", "一般般吧"]
train_labels = [2, 0, 1]  # 正面, 负面, 中性

# 英文训练数据（通过翻译或代码切换增强）
train_texts_en = ["This product works great", "Terrible quality", "It's okay"]
train_texts = train_texts_zh + train_texts_en
train_labels = train_labels + train_labels  # 同样的标签

# Tokenize
encodings = tokenizer(train_texts, truncation=True, padding=True, max_length=128)

class MultilingualDataset(torch.utils.data.Dataset):
    def __init__(self, encodings, labels):
        self.encodings = encodings
        self.labels = labels
    def __getitem__(self, idx):
        item = {k: torch.tensor(v[idx]) for k, v in self.encodings.items()}
        item["labels"] = torch.tensor(self.labels[idx])
        return item
    def __len__(self):
        return len(self.labels)

dataset = MultilingualDataset(encodings, train_labels)

# 训练参数
training_args = TrainingArguments(
    output_dir="./results",
    learning_rate=2e-5,
    per_device_train_batch_size=8,
    num_train_epochs=5,
    weight_decay=0.01,
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset,
)

trainer.train()

# 零样本迁移：在日语上推理
ja_text = "この製品は素晴らしいです"
inputs = tokenizer(ja_text, return_tensors="pt")
outputs = model(**inputs)
pred = torch.argmax(outputs.logits, dim=-1)
print(f"日语零样本预测: {pred.item()}")  # 预期: 2 (正面)

13.9 本章小结

多语言BERT让"一次训练，多种语言受益"从梦想变成了现实。mBERT证明了共享subword + 共享编码器的魔力，XLM-R将这一范式发挥到了极致，而TLM、代码切换、语言适配器等技术则为特定场景提供了更精细的解决方案。
