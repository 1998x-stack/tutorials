数据混合为什么是一门科学

大家好，我是你们的讲师。上章我们把数据去重和过滤做到了极致，得到了一堆干干净净、高高质量的文档。现在问题来了：这些文档来自网页、书籍、代码、论文、百科——我们应该按什么比例把它们混在一起训练？

你可能会想：多简单的事，同样比例不就完了？但事实上，这是大模型训练中最微妙也最容易被低估的决策之一。Llama-3 团队在技术报告中透露，他们在数据混合比例上花费了数月时间进行小规模实验；DeepSeek 的技术报告用了整整一节来讨论数据配比；而据传闻，某次顶级开源模型的失败就是因为代码数据配比失控了 0.3%。

为什么数据混合如此敏感？因为不同的数据来源承载着完全不同的知识分布和语言模式。网页数据提供了广泛的常识和表达方式；代码数据训练了逻辑推理和结构化思维；书籍数据带来了长文本理解和叙事能力。混合比例不仅决定了模型"知道什么"，更决定了模型"如何思考"。

核心洞察： "数据混合本质上是一个多目标优化问题——你在不同能力维度（推理、编码、知识、对话）之间分配有限的数据预算。你无法在所有维度上同时达到最优，所以混合策略本质上是关于你最看重什么能力的价值判断。"

经典混合策略

数据混合的研究经历了从"启发式"到"数据驱动"的演变。最早的做法是纯凭经验——互联网数据 80%、书籍 10%、代码 10% 这样拍脑袋。但实践证明，这种"一刀切"的做法远非最优。

DoReMi：域重加权

DoReMi (Xie et al., 2023) 是数据混合领域的第一篇系统性工作。它的核心思想是：用一个小型的代理模型（proxy model）来评估不同数据域对训练损失的贡献，然后自动调整各域的采样权重，使得所有域的损失下降尽量均衡。如果一个域的损失下降得太慢，就给它更高的采样权重；反之则降低权重。

这种方法的美妙之处在于：它不需要人类的先验判断，完全由数据驱动。DoReMi 在 GLaM 数据集上的实验显示，经过重加权的数据混合可以使模型在多个下游任务上取得一致提升，且没有明显短板。

RegMix：回归混合

RegMix (Liu et al., 2024) 则采取了另一种思路：训练几十个不同数据配比的小型模型，用它们的性能表现来拟合一个回归模型，从而预测最优配比。具体来说，RegMix 首先生成一批候选配比（通过拉丁超立方采样），训练对应的代理模型，然后在验证集上评估每个模型在不同任务上的表现。最后，用回归模型建模"数据配比 → 任务表现"的映射关系，反推最优配比。

RegMix 实验发现，最优配比对模型规模并不敏感——在 125M 参数的小模型上找到的最优配比，可以直接迁移到 1.5B 甚至 7B 的大模型上。这意味着你只需要花很少的钱做小规模实验，就能指导大规模训练的数据配比。

BiMix：双向混合

BiMix (Zhou et al., 2024) 提出了一个更细腻的视角：混合不仅发生在数据域之间，也发生在数据质量之间。BiMix 将数据同时按"域"和"质量分位数"两个维度分层，然后优化一个二维的采样矩阵。实验表明，即使在同一域内，不同质量层次的数据也应该有不同的采样策略——高质量数据需要被过采样，但过采样倍数随训练步数逐步衰减。

GEM：几何熵混合

如果说上述方法是经验主义的，那么 GEM（Geometric Entropy Mixing，几何熵混合） 则是 2026 年 ICML 上出现的一个更具理论优雅性的方案。

GEM 的核心洞察是：数据域之间的关系可以用几何结构来刻画。每个数据域可以视为高维空间中的一个点，域之间的"距离"由它们的文本分布差异决定。最优混合比例就是在这个几何空间中找到一个点，使得各个方向上的"信息熵"达到均衡。

数学上，GEM 将数据混合问题形式化为在概率单纯形（probability simplex）上的优化：

# GEM：几何熵混合——简化伪代码

import numpy as np
from scipy.optimize import minimize

def geometric_influence_score(domain_embeddings, target_domain):
    """
    计算每个域对目标域的几何影响分数

    domain_embeddings: dict, {domain_name: embedding_vector}
    target_domain: str, 目标域的嵌入
    """
    target_vec = domain_embeddings[target_domain]
    scores = {}
    for name, vec in domain_embeddings.items():
        if name == target_domain:
            continue
        # 余弦相似度加权空间距离
        cos_sim = np.dot(vec, target_vec) / (
            np.linalg.norm(vec) * np.linalg.norm(target_vec)
        )
        # 几何熵权重：相似域获得更高权重
        geo_entropy = np.exp(cos_sim) / (
            np.sum(np.exp(np.dot(domain_embeddings.values(), target_vec)))
        )
        scores[name] = geo_entropy
    return scores

def gem_optimal_mix(domain_embeddings, lambda_reg=0.1):
    """寻找几何熵最优混合比例"""
    num_domains = len(domain_embeddings)
    domains = list(domain_embeddings.keys())

    def objective(weights):
        # 加权平均嵌入
        weighted_avg = sum(
            w * domain_embeddings[d] for w, d in zip(weights, domains)
        )
        # 最大化与所有域的几何熵
        entropy = 0
        for d in domains:
            cos_sim = np.dot(domain_embeddings[d], weighted_avg) / (
                np.linalg.norm(domain_embeddings[d]) *
                np.linalg.norm(weighted_avg) + 1e-8
            )
            entropy += np.exp(cos_sim)
        # 加入L2正则防止过拟合到一个域
        reg = lambda_reg * np.sum(np.square(weights))
        return -entropy + reg

    # 约束：权重和为1，每个权重>=0
    constraints = ({'type': 'eq', 'fun': lambda w: np.sum(w) - 1})
    bounds = [(0, 1) for _ in range(num_domains)]
    init = np.ones(num_domains) / num_domains

    result = minimize(objective, init, method='SLSQP',
                      bounds=bounds, constraints=constraints)
    return {d: w for d, w in zip(domains, result.x)}

GEM 在多个模型规模上的实验表明，使用几何熵最优混合配比，相比均匀混合和 DoReMi 重加权，在代码生成、数学推理和知识问答三个维度上同时取得了 3-8% 的提升。更重要的是，GEM 给出的配比具有良好的理论可解释性——你可以直接看到每个域在"几何空间"中的相对位置如何影响它的最优权重。

实践建议： 如果你在做一个新的预训练项目，建议的混合策略探索顺序是：(1) 先用 RegMix 做小规模实验找到粗粒度的最优配比；(2) 用 GEM 进行理论验证和微调；(3) 在训练过程中用 DoReMi 风格的动态重加权做实时修正。三层叠加往往能产生最好的效果。

域选择与持续预训练

数据混合的另一个重要场景是持续预训练（continual pre-training, CPT）。当你在通用模型的基础上注入特定领域的知识时，混合策略的考虑因素会发生变化。

CMR Scaling Law 与 D-CPT Law

CMR Scaling Law (Chen et al., 2024) 发现，在持续预训练中，域数据的配比与通用数据的配比之间存在一种"权衡关系"：域内数据的比例过高会导致灾难性遗忘，比例过低则域知识注入不足。这个权衡由模型大小、域数据量和通用数据量共同决定。

D-CPT Law (Domain-specific Continual Pre-Training Law, Yang et al., 2025) 进一步量化了这个关系：

# D-CPT Law 的简化公式（用于确定域数据配比）

def d_cpt_law(model_size, domain_data_size, general_data_size):
    """
    计算持续预训练中的最优域数据比例

    Args:
        model_size: 模型参数量（B）
        domain_data_size: 域数据 token 数（B）
        general_data_size: 通用数据 token 数（B）

    Returns:
        optimal_domain_ratio: 最优域数据采样比例
        expected_forgetting: 预期遗忘率
    """
    # 经验参数（来自论文）
    alpha = 0.42   # 尺度指数
    beta = 0.31    # 遗忘敏感度
    gamma = 0.19   # 交互系数

    total_data = domain_data_size + general_data_size
    domain_ratio = domain_data_size / total_data

    # 最优域比例估计
    optimal_ratio = alpha * (model_size ** (-0.15)) * (
        1 + gamma * np.log1p(domain_data_size / general_data_size)
    )

    # 预测遗忘率
    forgetting = beta * (domain_ratio ** 0.7) * (
        model_size ** (-0.25)
    )

    return min(max(optimal_ratio, 0.01), 0.5), forgetting

CoLoR-Filter

CoLoR-Filter (Kumar et al., 2024) 提出了一个优雅的思路：与其手动决定选哪些域的数据，不如让"条件重要性"来指引。CoLoR-Filter 计算每个样本对目标域任务的条件贡献，只保留那些对目标域正向贡献的样本，无论它们来自哪个域。这实际上是"域选择"的终极形式——你不再以域为单位选择，而是以样本为单位。

数据混合的实战配方

理论说了这么多，让我们看看业界实际是怎么做的。以下是几个公开的混合配方：

模型网页代码书籍/长文论文/学术多语言其他
Llama-3~50%~25%~15%~5%~5%—
DeepSeek-V2~40%~30%~10%~10%~8%数学数据 ~2%
Qwen-2.5~45%~20%~12%~8%~12%—
Mistral-Large~55%~15%~15%~5%~10%—
CodeLlama~20%~60%~10%~5%~3%—
Phi-3 (代码专用)~10%~70%~10%~5%~3%合成数据 ~2%

这些公开配方有几个值得注意的共同点：

  • 代码比例显著上升：相比 GPT-3 时代的 5% 代码，现在主流模型普遍使用 20-30% 的代码数据
  • 书籍比例在下降：从 GPT-3 的 25% 降到了 10-15%，因为合成数据和高质量网页数据可以部分替代书籍的长文本理解功能
  • 多语言数据比例取决于目标：如果目标主要是英文，多语言 5-10% 就够了；如果需要强多语言能力，可能需要 30-50%
  • 数学/推理数据的特殊地位：像 DeepSeek 这样数学能力突出的模型，会专门为数学数据分配独立预算
踩坑实录： 某团队在复现 Llama-3 的数据配比时，发现他们的模型在法律评测上完全崩盘——F1 分数比预期低了 20 个点。经过数周的排查，问题出在一个细节上：他们复现时把"网页数据"中的法律相关网页比例从 2.1% 调整到了 2.9%。这 0.8% 的差异听起来微不足道，但它占用了本来分配给通用常识数据的训练预算。结果是法律能力看似提升了（因为法律数据更多了），但模型对于"常识 → 法律"的知识迁移能力大幅下降。最终解决方案是恢复原始配比，并单独做一个法律领域的 CPT 阶段。0.8% 的配比变化就足以引发蝴蝶效应——数据混合的敏感性超乎你的想象。

这个案例给我们的启示是深刻的：数据混合不是加法，而是乘法。当你增加一个域的比例，你不仅在增加这个域的数据量，更在减少其他所有域的数据量。更微妙的是，不同域的交互效应往往是非线性的——有时候减少某个域的数据反而会提升它对应的评测结果，因为通用能力的提升间接增强了该域的迁移学习效果。

好的，数据混合法则和域选择为我们解决了"用什么数据"的问题。但还有一大类数据我们还没讨论——那些自然界中不存在、由模型生成的合成数据。下一章，我们就来聊 合成数据生成与知识蒸馏，看看如何让模型成为自己的老师。
