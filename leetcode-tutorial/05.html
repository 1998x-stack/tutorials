二分查找的本质

大家好，我是你们的讲师。今天我们要讲的是算法界的"效率之王"——二分查找（Binary Search）。说它是效率之王，是因为它能把一个 O(n) 的线性搜索问题在 O(log n) 时间内解决。当 n = 10⁹ 时，线性查找需要 10 亿步，二分查找只需要 30 步——这个差距是降维打击级别的。

但别看二分查找代码短，它是最容易出错的算法之一。有统计说 90% 的程序员第一次写二分查找都有 bug。Bug 率如此之高的原因在于：边界条件的选择（左闭右闭 vs 左闭右开）、mid 的计算、死循环的避免……这些都容不得半点模糊。

二分查找有一个前提：数据必须是单调有序的。或者更广义地说，搜索空间必须具有"两段性"——存在一个分界点，分界点的一侧满足某条件，另一侧不满足。掌握了这个"两段性"思维，你就能把二分查找用到很多非传统场景中。

核心思想：二分查找不是"在有序数组中找某个数"，而是"在满足单调性的搜索空间中寻找边界"。每次迭代将搜索空间缩小一半，直到找到目标或确定目标不存在。

边界模板精讲

二分查找的模板有很多，但核心差异只有一个：区间的开闭约定。我给大家总结两种主流写法：左闭右闭 [left, right] 和左闭右开 [left, right)。

模板一：左闭右闭 [left, right]

def binary_search(nums: list[int], target: int) -> int:
    left, right = 0, len(nums) - 1  # 注意：right 是最后一个有效下标
    while left <= right:            # 终止条件：left > right
        mid = left + (right - left) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

模板二：左闭右开 [left, right)

def binary_search(nums: list[int], target: int) -> int:
    left, right = 0, len(nums)  # 注意：right 是越界下标
    while left < right:         # 终止条件：left == right
        mid = left + (right - left) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid
    return -1

两种模板的差异对照：

  • 闭区间模板的 right = len - 1，循环条件 left <= right，收缩时 right = mid - 1
  • 开区间模板的 right = len，循环条件 left < right，收缩时 right = mid
我的建议：选择一种你最能理解的模板，然后一直用下去。我个人推荐左闭右开模板——因为它的 left = mid + 1 和 right = mid 对应了"排除 mid"的逻辑，不容易出现死循环。而且 left < right 的终止语义是"搜索空间为空"，非常直观。

寻找左边界和右边界（LeetCode 34）

在排序数组中查找元素的第一个和最后一个位置，这个题考察的就是二分边界的精确控制。

def search_range(nums: list[int], target: int) -> list[int]:
    def find_first():
        left, right = 0, len(nums)
        while left < right:
            mid = left + (right - left) // 2
            if nums[mid] >= target:
                right = mid
            else:
                left = mid + 1
        return left

    def find_last():
        left, right = 0, len(nums)
        while left < right:
            mid = left + (right - left) // 2
            if nums[mid] > target:
                right = mid
            else:
                left = mid + 1
        return left - 1

    first = find_first()
    if first == len(nums) or nums[first] != target:
        return [-1, -1]
    return [first, find_last()]

注意两处微妙的差异：找左边界用 >=，找右边界用 >。为什么呢？因为我们要找到第一个大于等于 target 的位置和最后一个小于等于 target 的位置。这两者之间的差异就是一个等号。

旋转排序数组

旋转排序数组是指一个有序数组在某个未知点发生了旋转，比如 [4,5,6,7,0,1,2]。虽然整体不是有序的，但从任意位置看，左右两侧分别是有序的——我们仍然可以用二分查找。

搜索旋转排序数组（LeetCode 33）

def search(nums: list[int], target: int) -> int:
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] == target:
            return mid
        # 左半段有序
        if nums[left] <= nums[mid]:
            if nums[left] <= target < nums[mid]:
                right = mid - 1
            else:
                left = mid + 1
        # 右半段有序
        else:
            if nums[mid] < target <= nums[right]:
                left = mid + 1
            else:
                right = mid - 1
    return -1

思路是：先判断 mid 落在有序半段还是旋转半段，然后根据 target 是否在有序半段内决定搜索方向。判断有序的条件是 nums[left] <= nums[mid]——注意这里是 <= 不是 <，因为当 left == mid 时也算有序。

寻找旋转排序数组中的最小值（LeetCode 153）

def find_min(nums: list[int]) -> int:
    left, right = 0, len(nums) - 1
    while left < right:
        mid = left + (right - left) // 2
        if nums[mid] > nums[right]:
            left = mid + 1
        else:
            right = mid
    return nums[left]

这里利用了旋转数组的特性：最小值将数组分成两段递增序列。如果 nums[mid] > nums[right]，说明最小值在右半段；否则在左半段（含 mid 本身）。

易错点：旋转数组的二分模板和普通二分不一样！不要死记硬背，要理解判断条件。核心是：通过比较 nums[mid] 和 nums[right]（或 nums[left]）来确定哪半段有序。

非典型二分 — 搜索空间二分

这是二分查找最强大、也最容易被忽略的用法——不是在数组上二分，而是在答案空间上二分。只要你能验证某个猜测值是否符合要求，并且验证函数是单调的，就能用二分来搜索答案。

爱吃香蕉的珂珂（LeetCode 875）

珂珂每小时吃 K 根香蕉，每堆香蕉如果少于 K 根就吃完并等待下一小时。求在 H 小时内吃完的最小 K。

def min_eating_speed(piles: list[int], h: int) -> int:
    def can_finish(k: int) -> bool:
        hours = 0
        for p in piles:
            hours += (p + k - 1) // k
        return hours <= h

    left, right = 1, max(piles)
    while left < right:
        mid = left + (right - left) // 2
        if can_finish(mid):
            right = mid
        else:
            left = mid + 1
    return left

K 的搜索范围是 [1, max(piles)]，验证函数 can_finish(k) 是单调的——K 越大越容易完成。这就是"两段性"：存在一个分界 K₀，小于 K₀ 的吃不完，大于等于 K₀ 的能吃完。二分查找找的就是这个分界点。

分割数组的最大值（LeetCode 410）

给定一个数组，将其分成 m 个连续子数组，使得各子数组和的最大值最小。这个题是"搜索空间二分"的进阶版。

def split_array(nums: list[int], m: int) -> int:
    def can_split(max_sum: int) -> bool:
        count = 1
        current = 0
        for num in nums:
            current += num
            if current > max_sum:
                count += 1
                current = num
        return count <= m

    left, right = max(nums), sum(nums)
    while left < right:
        mid = left + (right - left) // 2
        if can_split(mid):
            right = mid
        else:
            left = mid + 1
    return left

验证函数 can_split(max_sum) 判断能否用不超过 m 个子数组覆盖所有数，且每个子数组的和不超过 max_sum。搜索范围从 max(nums)（最小可能的子数组和）到 sum(nums)（最大可能的子数组和）。

模式识别：当题目出现"最大值最小化"或"最小值最大化"字样时，80% 的可能是二分答案。回答这种问题时，面试官期待的并不是你直接给出公式，而是你展现出"先确定搜索范围 → 再定义验证函数 → 最后二分"的解题框架。

踩坑与总结

二分查找的坑，我踩过最大的一个就是死循环。有一次在左闭右开模板中，我写的是：

while left < right:
    mid = left + (right - left) // 2
    if condition(mid):
        left = mid    # 错误！
    else:
        right = mid

当 left 和 right 相邻时（比如 left=3, right=4），mid = 3 + (4-3)//2 = 3，如果 condition(3) 为真，left = mid = 3，left 没变！于是循环永远进行下去。死循环的根本原因是 left 更新时没有排除 mid。安全的做法是 left = mid + 1。

另一个经典错误是 mid 溢出。在 C++/Java 中 (left + right) // 2 在 left 和 right 都很大时会溢出。Python 不会溢出，但养成 left + (right - left) // 2 的写法是好习惯——它不仅仅是为了防溢出，这个写法本身就是一种"安全编程"的意识体现。

模板类型 区间表示 循环条件 left 更新 right 更新 适用场景
标准查找 [left, right] left <= right mid + 1 mid - 1 精确查找 target
找左边界 [left, right) left < right mid + 1 mid 第一个 >= target
找右边界 [left, right) left < right mid + 1 mid 第一个 > target
搜索空间二分 [left, right) left < right mid + 1 mid 答案空间单调
防坑口诀：二分查找不过三句话——① 确定搜索空间和单调性；② 选好模板，循环不变量明确；③ 验证边界（数组长度为 0、1、2 三种情况）。每次写完二分查找，拿这三个 case 跑一遍，90% 的 bug 都能提前暴露。

二分查找的威力远不止于"在有序数组里找数"。当你把"搜索答案空间"的思维内化之后，你会发现很多看似和二分无关的题目也能用二分解决。下一章我们将学习滑动窗口——双指针技巧的进阶版，专门解决连续子数组和子串问题。如果说二分是"砍半搜索"，那滑动窗口就是"线性推进"，两种思路互为补充。我们下一章见！
