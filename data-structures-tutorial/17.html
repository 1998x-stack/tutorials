开篇：一个面试失败的故事

我认识一位朋友，LeetCode 刷了 200 多题，链表、树、回溯、排序都烂熟于心。他去面一家大厂时信心满满。前两轮面试轻松过关。第三轮，面试官出了一道题——"打家劫舍"（LeetCode 198）。

他一看题目，心想："这不就是隔一个取一个吗？奇数位和偶数位比大小！"他洋洋洒洒写了一个贪心解法，提交上去。面试官看了三秒钟，说："你看看这个例子：[2, 1, 1, 2]。你的贪心选哪个？"他算了一下——奇数位 2+1=3，偶数位 1+2=3，贪心返回 3。但正确答案是选择第 1 个和第 4 个，得到 4。"隔一个取一个"的直觉错了，因为可以隔两个连续跳过。

最终他止步于这一轮。事后他说，"如果当时我能用动态规划来思考这道题，只要定义 dp[i] = 前 i 个房子能偷到的最大值，状态转移方程 dp[i] = max(dp[i-1], dp[i-2] + nums[i]) 就出来了。可惜我连 DP 的状态定义都没想过。"

我跳过了 DP 去面试，结果三道题两道是 DP 变种。这句话我听到过无数次。动态规划几乎是大厂算法面试的"必考点"——不是因为 DP 多"高级"，而是因为 DP 能考察候选人的抽象建模能力：你能不能把一个看似复杂的问题，转化成可以用状态递推解决的数学模型。

DP 的核心思想——最优子结构 + 重叠子问题

动态规划（Dynamic Programming，简称 DP）的核心思想可以浓缩成两句话：

  • 最优子结构（Optimal Substructure）——原问题的最优解包含子问题的最优解。你走过的每一步都是局部最优的，合起来就是全局最优。
  • 重叠子问题（Overlapping Subproblems）——不同的子问题会重复出现。DP 通过"记住"已计算的结果来避免重复劳动。

听起来抽象？我们用斐波那契数列的三种写法，把 DP 的本质讲得清清楚楚。

斐波那契——从递归到 DP 的渐进之路

第 15 章我们写了递归版的斐波那契：

# 写法一：纯递归（O(2^n)）——指数级爆炸
def fib_naive(n):
    if n <= 1:
        return n
    return fib_naive(n - 1) + fib_naive(n - 2)

# n=50 时这辈子都算不完
# print(fib_naive(50))  # 千万别试！

问题出在哪里？递归树中充满了重复计算——fib(3) 被 `fib(5)` 和 `fib(4)` 各算了一次，fib(2) 被重复计算了好几次。我们能不能把算过的结果存起来？

# 写法二：记忆化递归 / 自顶向下 DP（O(n)）
# 也叫 Memoization（注意没有 r）
def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]

# n=100 瞬间出结果
print(fib_memo(100))  # 354224848179261915075

这就是自顶向下（Top-down）的 DP——从原问题出发，递归求解子问题，记录已算过的结果。但递归仍然有栈溢出的风险。我们能不能不用递归，直接从最基础的问题开始递推？

# 写法三：递推 / 自底向上 DP（O(n)，迭代，无栈溢出风险）
def fib_dp(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]

print(fib_dp(100))

# 空间优化版：只用两个变量，O(1) 空间
def fib_optimized(n):
    if n <= 1:
        return n
    prev2, prev1 = 0, 1
    for _ in range(2, n + 1):
        cur = prev1 + prev2
        prev2, prev1 = prev1, cur
    return prev1

这三种写法的演变，恰是 DP 思想的完整学习路径：暴力递归 → 记忆化递归 → 递推 DP。

DP 的本质：从斐波那契的例子可以看出，DP 并没有什么神秘的——它就是"暴力递归 + 备忘录"。你把这个公式记住，DP 的一半恐惧就消失了。另一半恐惧来自于"状态定义"，这是我们接下来要攻克的目标。

DP 解题五步法

做 DP 题有固定的思考框架。我把这个框架总结为DP 解题五步法：

  1. 定义状态（State Definition）——dp[i] 或 dp[i][j] 表示什么？这是 DP 最重要的一步，也是最难的一步。
  2. 状态转移方程（Transition）——当前状态如何从之前的状态推导出来？这是 DP 的核心公式。
  3. 初始化（Initialization）——最基本的几个状态值是多少？
  4. 遍历顺序（Iteration Order）——从前往后？从后往前？从上到下？遍历顺序要保证计算当前状态时，依赖的状态已经被计算过了。
  5. 返回结果（Return）——最终答案是 dp[n] 还是 dp[m][n] 中的某个值？

接下来，我们拿五步法去套一个个经典问题，熟能生巧。

经典一维 DP

爬楼梯（LeetCode 70）

问题：爬 n 阶楼梯，每次可以爬 1 或 2 阶，有多少种不同的方法爬到楼顶？

# 五步法：
# 1. 定义状态：dp[i] = 爬到第 i 阶的方法数
# 2. 状态转移：dp[i] = dp[i-1] + dp[i-2]
#    到 i 阶有两种方式：从 i-1 爬 1 阶，或从 i-2 爬 2 阶
# 3. 初始化：dp[0]=1（不动也是一种方法），dp[1]=1
# 4. 遍历顺序：i 从 2 到 n
# 5. 返回结果：dp[n]

def climb_stairs(n):
    if n <= 1:
        return 1
    dp = [0] * (n + 1)
    dp[0], dp[1] = 1, 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]

print(climb_stairs(5))  # 8

有没有发现，爬楼梯和斐波那契的状态转移方程一模一样？不同的只是初始值。很多 DP 问题本质上是同一个数学结构的不同变体。

打家劫舍（LeetCode 198）

问题：一排房子，不能偷相邻的两间，求能偷到的最大金额。

def rob(nums):
    """打家劫舍"""
    if not nums:
        return 0
    if len(nums) == 1:
        return nums[0]

    # dp[i] = 前 i 个房子能偷到的最大金额
    dp = [0] * len(nums)
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])

    for i in range(2, len(nums)):
        # 偷 i：dp[i-2] + nums[i]
        # 不偷 i：dp[i-1]
        dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])

    return dp[-1]

# 空间优化版
def rob_optimized(nums):
    prev2, prev1 = 0, 0
    for num in nums:
        cur = max(prev1, prev2 + num)
        prev2, prev1 = prev1, cur
    return prev1

print(rob([2, 1, 1, 2]))  # 4，不是 3

这就是开篇故事中那道题的答案。状态转移方程 dp[i] = max(dp[i-1], dp[i-2] + nums[i]) 的意思是：要么不偷当前房子（沿用前 i-1 间的最优解），要么偷当前房子（前 i-2 间的最优解加上当前金额），取两者中的最大值。

最大子数组和 & Kadane 算法（LeetCode 53）

问题：给定整数数组，找出和最大的连续子数组。

def max_subarray(nums):
    """Kadane 算法：O(n) 时间，O(1) 空间"""
    # dp[i] = 以 nums[i] 结尾的最大子数组和
    cur_max = global_max = nums[0]

    for i in range(1, len(nums)):
        # 要么从当前元素重新开始，要么和前面的子数组连起来
        cur_max = max(nums[i], cur_max + nums[i])
        global_max = max(global_max, cur_max)

    return global_max

print(max_subarray([-2, 1, -3, 4, -1, 2, 1, -5, 4]))  # 6

Kadane 算法的精妙之处在于：状态定义是"以当前位置结尾的最大子数组和"。这个"结尾"的思路让问题变成了一个一维递推。你能说 Kadane 算法是 DP 还是贪心？它既是 DP（有递推关系），也体现了贪心的思想（每次只保留当前最优）。很多优秀的算法正是多种思想的融合。

背包问题

背包问题是 DP 入门的第二道坎，但也是面试最高频的 DP 题型之一。

01背包问题

问题：有 N 个物品，每个物品有重量 w[i] 和价值 v[i]。有一个容量为 C 的背包，求能装入的最大总价值。每个物品最多选一次。

def knap_01(weights, values, capacity):
    """01背包：二维 DP"""
    n = len(weights)
    # dp[i][c] = 前 i 个物品中选，容量为 c 时能得到的最大价值
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for c in range(1, capacity + 1):
            if weights[i - 1] > c:
                # 物品太重，放不下
                dp[i][c] = dp[i - 1][c]
            else:
                # 装或不装，取较大值
                dp[i][c] = max(dp[i - 1][c],
                               dp[i - 1][c - weights[i - 1]] + values[i - 1])

    return dp[n][capacity]


# 一维空间优化版（面试最常用）
def knap_01_optimized(weights, values, capacity):
    """01背包：一维 DP，逆序遍历容量"""
    n = len(weights)
    dp = [0] * (capacity + 1)

    for i in range(n):
        # 关键：逆序遍历容量，保证每个物品只被选一次
        for c in range(capacity, weights[i] - 1, -1):
            dp[c] = max(dp[c], dp[c - weights[i]] + values[i])

    return dp[capacity]

# 测试
weights = [2, 3, 4, 5]
values = [3, 4, 5, 6]
capacity = 8
print(knap_01(weights, values, capacity))  # 10（选物品1+物品2+物品4）
为什么 01 背包要逆序？因为一维 DP 中，dp[c - weights[i]] 如果正序遍历，可能已经被当前物品更新过了，导致一个物品被多次选中（变成完全背包）。逆序遍历保证了 dp[c - weights[i]] 还是上一个物品的状态，每个物品只选一次。

完全背包——零钱兑换（LeetCode 322）

问题：给定不同面额的硬币 coins 和一个总金额 amount，求凑成总金额所需的最少硬币数（每种硬币可以用无限次）。

def coin_change(coins, amount):
    """零钱兑换：完全背包问题"""
    # dp[i] = 凑成金额 i 所需的最少硬币数
    # 初始化为一个很大的数
    INF = float('inf')
    dp = [INF] * (amount + 1)
    dp[0] = 0  # 金额 0 不需要硬币

    for coin in coins:
        for i in range(coin, amount + 1):
            # 正序遍历：允许同一个硬币多次使用
            if dp[i - coin] != INF:
                dp[i] = min(dp[i], dp[i - coin] + 1)

    return dp[amount] if dp[amount] != INF else -1

# 测试
print(coin_change([1, 2, 5], 11))  # 3 (5+5+1)
print(coin_change([2], 3))         # -1（无法凑出）

注意完全背包和 01 背包的唯一区别——完全背包正序遍历容量，因为每个物品可以选无限次；01 背包逆序遍历容量，因为每个物品只能选一次。

经典二维 DP（上）

不同路径（LeetCode 62）

问题：一个 m×n 的网格，机器人从左上角走到右下角，每次只能向右或向下走，有多少条不同路径？

def unique_paths(m, n):
    """不同路径：二维 DP"""
    # dp[i][j] = 到达 (i,j) 的路径数
    dp = [[0] * n for _ in range(m)]

    # 第一行和第一列都只有一条路径
    for i in range(m):
        dp[i][0] = 1
    for j in range(n):
        dp[0][j] = 1

    for i in range(1, m):
        for j in range(1, n):
            # 到达 (i,j) 只能从上方或左方来
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1]

    return dp[m - 1][n - 1]

print(unique_paths(3, 7))  # 28

最小路径和（LeetCode 64）

def min_path_sum(grid):
    """最小路径和"""
    m, n = len(grid), len(grid[0])
    dp = [[0] * n for _ in range(m)]

    dp[0][0] = grid[0][0]

    # 初始化第一行和第一列
    for i in range(1, m):
        dp[i][0] = dp[i - 1][0] + grid[i][0]
    for j in range(1, n):
        dp[0][j] = dp[0][j - 1] + grid[0][j]

    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j]

    return dp[m - 1][n - 1]

# 一维空间优化
def min_path_sum_optimized(grid):
    m, n = len(grid), len(grid[0])
    dp = [0] * n
    dp[0] = grid[0][0]
    for j in range(1, n):
        dp[j] = dp[j - 1] + grid[0][j]

    for i in range(1, m):
        dp[0] += grid[i][0]
        for j in range(1, n):
            dp[j] = min(dp[j], dp[j - 1]) + grid[i][j]

    return dp[-1]

这两道题是二维 DP 的"开胃菜"，它们的共同特点是最优子结构非常直观——到达当前位置的路径，只取决于左边和上边的状态。

进阶 DP（下）

读者可以跳过本节，进阶 DP 在面试中出现的频率相对较低。但如果你要冲击顶级大厂，或者想增强自己的算法功底，下面的三个问题是非常好的训练材料。理解 LCS 和编辑距离对后续学习系统设计也很有帮助。

最长公共子序列 LCS（LeetCode 1143）

问题：给定两个字符串 text1 和 text2，返回它们的最长公共子序列的长度。

def longest_common_subsequence(text1, text2):
    """LCS：最长公共子序列"""
    m, n = len(text1), len(text2)
    # dp[i][j] = text1[:i] 和 text2[:j] 的 LCS 长度
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]

print(longest_common_subsequence("abcde", "ace"))  # 3 ("ace")

LCS 的核心思想：如果两个字符串的最后一个字符相等，那么这个字符一定在 LCS 中；否则，LCS 要么在 text1 去掉最后一个字符中，要么在 text2 去掉最后一个字符中。

最长递增子序列 LIS（LeetCode 300）

def length_of_lis(nums):
    """LIS：最长递增子序列（O(n²) DP）"""
    if not nums:
        return 0

    # dp[i] = 以 nums[i] 结尾的最长递增子序列长度
    dp = [1] * len(nums)

    for i in range(len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)

    return max(dp)


# O(n log n) 优化版：贪心 + 二分查找
import bisect

def length_of_lis_optimized(nums):
    """LIS 的 O(n log n) 解法"""
    tails = []  # tails[i] = 长度为 i+1 的递增子序列的最小末尾值
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    return len(tails)

print(length_of_lis([10, 9, 2, 5, 3, 7, 101, 18]))  # 4 ([2,3,7,101])

编辑距离（LeetCode 72）

问题：给定两个单词 word1 和 word2，返回将 word1 转换成 word2 所需的最少操作数。支持三种操作：插入一个字符、删除一个字符、替换一个字符。

def min_distance(word1, word2):
    """编辑距离：二维 DP 的经典问题"""
    m, n = len(word1), len(word2)
    # dp[i][j] = word1[:i] 转换成 word2[:j] 的最小编辑距离
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # 初始化：从空串到 word2 需要插入 n 个字符
    for j in range(n + 1):
        dp[0][j] = j
    # 初始化：从 word1 到空串需要删除 m 个字符
    for i in range(m + 1):
        dp[i][0] = i

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]  # 字符相同，无需编辑
            else:
                dp[i][j] = min(
                    dp[i - 1][j] + 1,      # 删除 word1[i-1]
                    dp[i][j - 1] + 1,      # 在 word1 中插入 word2[j-1]
                    dp[i - 1][j - 1] + 1   # 替换 word1[i-1] 为 word2[j-1]
                )

    return dp[m][n]

print(min_distance("horse", "ros"))  # 3

编辑距离是 NLP 领域的基础算法——拼写纠错、DNA 序列比对都用到了它。能写出编辑距离的 DP 解法，说明你对二维 DP 的掌握已经达到了一定的水平。

区间 DP 与状态压缩

最长回文子串（LeetCode 5）

问题：给定一个字符串 s，找到 s 中最长的回文子串。

解法一：中心扩展法（不是 DP，但非常高效 O(n²)）

def longest_palindrome_expand(s):
    """中心扩展法：以每个字符和每两个字符为中心向两边扩展"""
    def expand(left, right):
        while left >= 0 and right < len(s) and s[left] == s[right]:
            left -= 1
            right += 1
        return s[left + 1:right]

    result = ""
    for i in range(len(s)):
        # 奇数长度回文（中心是一个字符）
        odd = expand(i, i)
        # 偶数长度回文（中心是两个字符之间）
        even = expand(i, i + 1)
        result = max(result, odd, even, key=len)
    return result

解法二：区间 DP

def longest_palindrome_dp(s):
    """区间 DP：dp[i][j] = s[i:j+1] 是否是回文"""
    n = len(s)
    dp = [[False] * n for _ in range(n)]
    result = ""

    # 长度为 1 的子串都是回文
    for i in range(n):
        dp[i][i] = True
        result = s[i]

    # 按子串长度从小到大遍历
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            if s[i] == s[j]:
                if length == 2 or dp[i + 1][j - 1]:
                    dp[i][j] = True
                    if length > len(result):
                        result = s[i:j + 1]

    return result

区间 DP 的遍历顺序很特别——不是按行或列，而是按区间长度从小到大遍历。因为计算长区间时，依赖的短区间结果必须已经准备好了。

状态压缩 DP 入门

状态压缩 DP（Bitmask DP）是当状态数量不多（通常 n ≤ 20）时，用一个整数的二进制位来表示状态集合的 DP 技巧。

经典的例子：给定一组任务和一组工人，每个工人能完成某些任务，求最少需要多少工人才能完成所有任务。

# 状态压缩 DP 的经典例题：旅行商问题（TSP）的简化版
# 假设有 4 个城市，已知任意两个城市之间的距离
# 求从城市 0 出发走完所有城市并回到城市 0 的最短路径

def tsp(dist):
    """TSP 状态压缩 DP"""
    n = len(dist)
    # dp[mask][i] = 已经访问了 mask 中的城市，当前在 i 城市时的最短路径
    # mask 是一个二进制数，第 k 位为 1 表示已访问城市 k
    dp = [[float('inf')] * n for _ in range(1 << n)]

    dp[1][0] = 0  # 从城市 0 出发，只访问了城市 0

    for mask in range(1 << n):
        for last in range(n):
            if not (mask & (1 << last)):
                continue
            if dp[mask][last] == float('inf'):
                continue
            # 尝试访问下一个未访问的城市
            for nxt in range(n):
                if mask & (1 << nxt):
                    continue
                new_mask = mask | (1 << nxt)
                dp[new_mask][nxt] = min(dp[new_mask][nxt],
                                        dp[mask][last] + dist[last][nxt])

    # 最后回到城市 0
    full_mask = (1 << n) - 1
    result = min(dp[full_mask][i] + dist[i][0] for i in range(n))
    return result


dist = [
    [0, 10, 15, 20],
    [10, 0, 35, 25],
    [15, 35, 0, 30],
    [20, 25, 30, 0]
]
print(tsp(dist))  # 80

状态压缩 DP 是 DP 里比较"高级"的部分，面试中出现的频率不高（除非你在面量化交易或算法竞赛相关的岗位）。但理解它的核心思想——用一个整数的二进制位表示状态集合——对拓宽解题思路非常有帮助。

常见 DP 问题速查表

问题 状态定义 状态转移方程 时间
爬楼梯 dp[i] = 到第 i 阶的方法数 dp[i] = dp[i-1] + dp[i-2] O(n)
打家劫舍 dp[i] = 前 i 间最大金额 dp[i] = max(dp[i-1], dp[i-2]+nums[i]) O(n)
最大子数组和 dp[i] = 以 i 结尾的最大和 dp[i] = max(nums[i], dp[i-1]+nums[i]) O(n)
01背包 dp[i][c] = 前 i 物品容量 c 最大价值 dp[i][c] = max(dp[i-1][c], dp[i-1][c-w[i]]+v[i]) O(N·C)
完全背包 dp[i] = 凑成金额 i 的最少硬币数 dp[i] = min(dp[i], dp[i-coin]+1) O(N·C)
不同路径 dp[i][j] = 到达 (i,j) 的路径数 dp[i][j] = dp[i-1][j] + dp[i][j-1] O(m·n)
LCS dp[i][j] = text1[:i] 和 text2[:j] 的 LCS 长度 相同: dp[i-1][j-1]+1; 不同: max(dp[i-1][j], dp[i][j-1]) O(m·n)
编辑距离 dp[i][j] = word1[:i] 到 word2[:j] 的编辑距离 相同: dp[i-1][j-1]; 不同: min(删除,插入,替换)+1 O(m·n)
DP 学习心法：刚开始学 DP，你会觉得每道题都像新的一样——因为状态定义方式千变万化。但刷到一定数量（大约 30-50 道 DP 题）后，你会发现万变不离其宗：一维 DP 就是线性递推，二维 DP 就是表格填数，背包问题就是遍历顺序的巧用。不要被"状态定义"吓到——每道题先问自己一个问题："如果用一个一维数组，每个位置代表什么？"

本章小结

  • DP 核心——最优子结构 + 重叠子问题，本质是"暴力递归 + 备忘录"。
  • DP 五步法——定义状态 → 状态转移 → 初始化 → 遍历顺序 → 返回结果。
  • 经典一维 DP——爬楼梯、打家劫舍、Kadane 算法（最大子数组和）。
  • 背包问题——01 背包逆序遍历、完全背包正序遍历，零钱兑换是经典变体。
  • 二维 DP——不同路径、最小路径和、进阶的 LCS/LIS/编辑距离。
  • 区间 DP 与状态压缩——最长回文子串的两种解法，旅行商问题的位掩码表示。

DP 是算法面试中最重要的思想之一。如果你现在还不能完全消化所有内容（特别是进阶 DP），不要灰心——这完全正常。DP 是需要反复练习才能掌握的技能，没有人能一次学会。先把基础一维 DP 和背包问题吃透，进阶内容可以等你有了更多信心再回来看。
