第11章 中文SFT数据集体系

前几章我们介绍了 FLAN Collection、OpenHermes 2.5、UltraChat 等经典英文数据集。但如果你的目标是训练一个中文场景下的模型——无论是服务于中国用户的产品，还是需要对中文内容有深度理解的应用——你会发现直接使用这些英文数据集远远不够。原因不仅仅是语言不同，还涉及到文化语境、中文特有的表达习惯，以及中国市场的合规要求。

这一章，我们将系统性地介绍中文 SFT 数据集的代表性成果：Firefly、BELLE、UltraChat-zh 和 Alpaca-GPT4-zh。读完这一章，你会理解中文 SFT 数据的特点、各个数据集的定位，以及如何搭配使用它们达到最佳效果。

中文后训练数据的独特挑战

在做中文后训练数据时，有三个区别于英文数据的特殊挑战：

语言特性方面，中文 NLP 与英文 NLP 有着本质区别。中文没有明显的词边界（分词本身就是一个大问题），语法结构更加灵活，并且在简繁体、口语书面语之间有巨大的差异。一个好的中文 SFT 数据集需要覆盖这些语言变体。

文化内涵方面，很多英文指令数据集中的话题——比如美国政治、西方流行文化、基督教宗教概念——在中文场景下相关性很低。反过来，中文用户关心的"公务员考试"、"房价走势"、"四大名著"、"中医养生"等话题在英文数据集中几乎不存在。因此，直接翻译英文数据集是不够的，必须针对中文场景做原创性的数据设计。

安全合规方面，中国对生成式 AI 有明确的内容管理要求。模型需要遵守中国的法律法规和社会主义核心价值观，在回答政治话题、历史话题、社会热点时必须高度谨慎。这意味着中文 SFT 数据需要专门的安全对齐数据，我们会在第 18 章深入讨论这一点。

一个常见的误区："我的基座模型本身就是双语预训练的，所以直接用英文 SFT 数据微调也能处理中文。"这在简单场景下可能成立，但模型的中文对话能力和文化适应性会非常有限。典型表现是：模型能理解中文问题，但回答的句式生硬、用词不当，有时还会混入英文。要培养真正流利的中文对话能力，高质量的中文 SFT 数据是不可替代的。

Firefly（中文百万级商用语料）

Firefly（流萤） 是目前中文开源社区最受欢迎的 SFT 数据集之一，由华南理工大学团队开发维护。它的核心亮点十分突出：

  • 110 万条高质量中文指令数据，覆盖 28+ 种任务类型
  • 任务范围包括：文本分类、信息抽取、阅读理解、文本生成、创意写作、代码生成、数学推理、翻译、改写、摘要等
  • Apache 2.0 许可证，完全商用友好，没有附加限制

Firefly 的数据构建方式是"人工设计模板 + 模型生成"的混合策略。团队针对每种任务类型设计了细致的指令模板（比如信息抽取任务有 30+ 种不同的指令表述），然后使用 GPT-3.5/4 批量生成回答。这种方式的优势在于：任务覆盖更系统，指令表述的多样性更高，数据分布更加可控。

Firefly 在中文社区的广泛使用已经验证了它的有效性。很多国产开源模型（如 Qwen 系列早期版本的内测阶段）都参考了 Firefly 的数据设计思路，甚至直接使用 Firefly 作为 SFT 基线数据。

Firefly 的数据格式也很清晰：

{
  "id": "firefly_28001",
  "kind": "信息抽取",
  "instruction": "请从以下文本中提取所有的人名、地名和组织机构名：\n\n昨日，阿里巴巴创始人马云在杭州出席了浙商总会会议，浙江省省长王浩发表了讲话。",
  "output": "人名：马云、王浩\n地名：杭州\n组织机构名：阿里巴巴、浙商总会"
}

BELLE系列（中文指令微调开山）

BELLE（BE Large Language Model Evaluation） 是 2023 年由中国科学技术信息研究所和华为联合发布的中文指令微调数据集。它是中文 SFT 领域的先驱工作之一。

BELLE 系列包含多个版本：

版本 数据量 生成方式 特点
BELLE 50 万精选版 50 万条 Self-Instruct + GPT-3.5 精挑细选，质量最高
BELLE 200 万全量版 200 万条 Self-Instruct + GPT-3.5 覆盖面更广，但噪声更多
BELLE 多轮对话版 100 万轮 GPT-3.5 生成 + 人工校验 专门针对多轮对话场景

BELLE 的独特价值在于它的基线对照意义。作为中文 SFT 的"元老级"数据集，BELLE 系列在不同版本之间形成了天然的对照实验——你可以训练"精选 50 万"和"全量 200 万"两个版本，观察数据规模和质量之间的 trade-off。在很多团队的实践中，50 万精选版的效果往往不输甚至超过 200 万全量版，这再次印证了 LIMA "少即是多"的结论。

BELLE 的历史意义：在 BELLE 出现之前（2023 年初），中文 SFT 数据集几乎是一片空白。团队面临的选择只有两个：要么自己从零开始标注数据（成本极高），要么直接翻译英文数据集（质量堪忧）。BELLE 通过 Self-Instruct 方法，首次证明了用 GPT-3.5 生成高质量中文指令数据的可行性。它的方法论——种子指令设计、生成过滤、质量抽检——至今仍是中文 SFT 数据构建的标准化流程。

UltraChat-zh（中文多轮对话标杆）

在第 9 章我们介绍了多轮对话数据集 UltraChat（英文版）。它的中文版 UltraChat-zh 采用完全相同的数据生成策略，但指令和对话内容全部为中文。

  • 10 万轮中文多轮对话（约 70 万条消息）
  • 由 GPT-4 全中文生成，场景覆盖日常对话、知识问答、中文写作、古诗词理解、成语解释等中国用户高度相关的话题
  • 对话自然流畅，没有明显的"翻译腔"——因为数据是直接用中文生成的，而不是从英文翻译过来的

UltraChat-zh 的一个突出优点是它的中文对话的自然度。在评估中，很多 evaluator 表示 UltraChat-zh 的对话"读起来像是两个中文母语者在交流"。这种自然度对于训练出流畅的模型对话能力非常重要——如果训练数据本身就有"翻译味"，模型的输出也会显得不自然。

UltraChat-zh 的格式与英文版一致，但中文场景特有的对话模式被很好地保留了。比如包含"你好—你叫什么名字？—我叫小爱，很高兴认识你！"这类典型的中文打招呼模式，以及"我想了解一下——麻烦你说详细一点——好的我明白了，谢谢！"这类渐进探索式的用户行为。

Alpaca-GPT4-zh

最后要介绍的是 Alpaca-GPT4-zh——一个看似简单但实际上非常实用的中文数据集。它做的事情只有一件：把 Alpaca 的 52K 指令数据用 GPT-4 重写了中文版本，并做了一些质量优化。

  • 基于原始 Alpaca 的 52K 英文指令，翻译为中文指令
  • 用 GPT-4 重新生成回答（不是简单翻译英文回答）
  • 经过一轮人工质量抽检和格式清洗

这个数据集的使用场景非常明确——快速验证和基线对照。当你开始一个新的中文 SFT 实验时，先用 Alpaca-GPT4-zh 训练一个 baseline 模型，验证训练框架、评估流程和硬件配置是否正常，然后再扩展到更大的数据集。它的轻量级（52K）让每轮实验只需几十分钟，非常适合快速迭代。

许可证注意事项：Alpaca-GPT4-zh 基于 Alpaca 衍生而来，继承了 CC BY-NC 4.0 许可证。如果你用于商业项目，需要注意这一点。Firefly（Apache 2.0）是更好的商用选择。

中文数据集混合使用策略

知道了各个数据集的特色之后，关键问题来了：如何搭配使用它们才能达到最好的效果？

基于行业实践和公开经验，一个比较成熟的中文 SFT 混合策略如下：

数据来源 推荐比例 作用
Firefly（全量） 40-50% 主力数据，提供广泛的中文任务覆盖
UltraChat-zh 15-20% 提供自然的中文多轮对话能力
BELLE 50 万精选 10-15% 补充指令多样性，增加基线参考
英文 SFT 数据（如 OpenHermes 2.5） 10-15% 保持双语能力，防止灾难性遗忘英文
Alpaca-GPT4-zh 5% 基线验证 + 少量高质补充
领域特定数据 5-10% 根据项目需要灵活添加

这个配比背后的逻辑是：以 Firefly 作为"骨架"提供最广泛的中文任务覆盖，用 UltraChat-zh 注入多轮对话的自然感，用 BELLE 增加指令表述的多样性，同时保留适量的英文数据防止模型在英文能力上退化，最后根据项目需求添加领域数据。这套策略在很多国内团队的项目中被验证为"稳妥但有效的起点"。

# 一个简化的中文 SFT 数据混合示例
from datasets import load_dataset, concatenate_datasets, Dataset

# 加载各数据集（streaming 模式节省内存）
firefly = load_dataset("YeungNLP/firefly-train-1.1M",
                       split="train", streaming=True)
ultrachat_zh = load_dataset("HuggingFaceH4/ultrachat_200k",
                            split="train_sft", streaming=True)
belle = load_dataset("BelleGroup/train_0.5M_CN",
                     split="train", streaming=True)

# 按比例采样
def sample_ratio(ds, ratio, total):
    """从数据集中采指定比例的数据"""
    n = int(total * ratio)
    return list(ds.take(n))

total_samples = 50000  # 以 5 万条为例
mixed_data = []
mixed_data += sample_ratio(firefly, 0.45, total_samples)
mixed_data += sample_ratio(ultrachat_zh, 0.20, total_samples)
mixed_data += sample_ratio(belle, 0.15, total_samples)

print(f"混合后数据总量: {len(mixed_data)} 条")
print(f"Firefly: {len(mixed_data)*0.45:.0f}")
print(f"UltraChat-zh: {len(mixed_data)*0.20:.0f}")
print(f"BELLE: {len(mixed_data)*0.15:.0f}")

至此，我们已经覆盖了中文 SFT 数据体系的主要脉络。总结一下：Firefly 是商用友好的中文多任务首选，BELLE 提供了经典的基线参考，UltraChat-zh 补充了多轮对话的自然度，Alpaca-GPT4-zh 适用于快速验证。理解了这些数据集的特性，你就可以根据项目需要灵活搭配了。

不过，通用能力无论中文还是英文都只是基础。在很多实际场景中，我们需要模型成为特定领域的专家——程序员需要代码助手，医生需要医学问答，分析师需要金融数据解读。下一章，我们将进入垂直领域 SFT 数据的世界，看看如何通过领域数据将通用模型变成某一领域的专家。
