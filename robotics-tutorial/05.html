第5章 刚体运动与空间描述

大家好，我是你们的讲师。从这一章开始，我们正式进入机器人学的核心地带。前面的三章数学基础是"兵器"——向量、矩阵、导数、概率——而本章开始，我们开始"打仗"了。机器人的本质是一个在三维空间中运动的物体，而描述它在"哪里"（位置）、"朝向哪"（姿态）、以及"如何从坐标系A变换到坐标系B"就是本章要解决的问题。

你可能觉得"位置和朝向"好像很直观——不就是三维坐标加上几个角度吗？但一旦到了机器人领域，问题就变得棘手了。举个简单的例子：你告诉机械臂"末端移动到桌子上的那个杯子上方"，但"桌子上方"是在相机坐标系下描述的，而机械臂只知道自己的关节角度——它需要知道相机坐标系到基座坐标系的变换。这个坐标系转换就是刚体运动描述要解决的核心问题。

5.1 位姿：位置+姿态

在三维空间中，一个刚体的位姿（Pose）由两部分组成：位置（Position）和姿态（Orientation）。位置用三维平移向量 \(\mathbf{p} \in \mathbb{R}^3\) 表示；姿态用旋转矩阵 \(\mathbf{R} \in \text{SO}(3)\) 表示。所以一个位姿通常写作 \((\mathbf{p}, \mathbf{R})\) 或者用一个4×4齐次变换矩阵来表示。

"刚体（Rigid Body）"是一个重要假设——物体在外力作用下不会变形，所有点之间的相对距离保持不变。在实际机器人中，这个假设不严格成立（连杆有柔性，关节有弹性），但在大多数情况下，它足以满足我们分析和控制的需求。如果你的应用需要处理柔性变形——比如软体机器人或高精度加工——就需要更复杂的模型，但那是机器人学的高级话题了。

位姿描述的核心困难在于：位置和姿态是不同性质的量。位置可以简单地做加法和缩放（它是一个向量空间的元素），而姿态不能——两个旋转矩阵相加的结果通常不是一个有效的旋转矩阵。这就是为什么我们需要专门为姿态设计数学工具。

关键认知：机器人学中最常见的错误之一，就是试图把位置和姿态"加权平均"。比如评估两个机器人轨迹的差异时，新手可能会简单地对位置误差和角度误差做加权求和——但位置误差的单位是米，角度误差的单位是弧度，两者性质完全不同。正确的做法是在 \(\text{SE}(3)\) 上定义测度（如Lie代数上的范数），或者至少物理意义地选择合适的权重。

5.2 旋转的三种表示

表示三维旋转有多种方式，各有优劣。作为参考框架，我们用一个例子来对比：假设一个物体先绕x轴转30度，再绕y轴转45度，最后绕z轴转60度。

旋转矩阵（Rotation Matrix）：\(\mathbf{R} \in \mathbb{R}^{3 \times 3}\)，满足 \(\mathbf{R}^T\mathbf{R} = \mathbf{I}\) 且 \(\det(\mathbf{R}) = 1\)。优点是复合变换就是矩阵乘法，非常直观；缺点是有9个参数但只有3个自由度（冗余），数值迭代时容易偏离正交约束。

欧拉角（Euler Angles）：用三个角度表示旋转，常见的有 ZYX（roll-pitch-yaw）和 ZYZ 等。优点是少（3个参数）、直观（每个角度的物理意义明确）；缺点是存在万向锁（Gimbal Lock）——当 pitch = ±90度时，roll和yaw的转轴重合，失去一个自由度。

四元数（Quaternion）：用4个参数 \(q = (w, x, y, z)\) 表示旋转，其中 \(w^2 + x^2 + y^2 + z^2 = 1\)。优点是紧凑、无奇异性、易于插值（Slerp）；缺点是不够直观。

让我讲一个关于Gimbal Lock的真实故事。阿波罗11号登月任务的导航系统中使用了欧拉角来表示姿态。在任务的关键阶段，导航计算机报告了一个"Gimbal Lock"警告——飞船的惯性测量单元的三个万向节中，有两个变得共面了。幸运的是，阿波罗的软件有退出机制，宇航员没有手动控制，任务继续了。但如果发生在更早的测试飞行中，后果不堪设想。这就是为什么现代航天器几乎全部使用四元数进行姿态表示——你的机器人也应该如此。

知识点：一个常见误区是认为万向锁只发生在物理万向节上。实际上，任何使用欧拉角表示姿态的系统（包括软件中的数学表示）都会遇到万向锁。你可以在任何3D引擎中验证：让pitch=89.9度，然后轻微调整roll——你会发现yaw也在剧烈变化。这不是bug，是欧拉角内在的奇异性。

5.3 齐次变换矩阵

齐次变换矩阵（Homogeneous Transformation Matrix） \(\mathbf{T} \in \text{SE}(3)\) 是机器人学中描述坐标系间相对位姿的通用工具：

T = [[R, p],     # 4×4 矩阵
     [0, 1]]

# R ∈ SO(3) —— 旋转部分（3×3）
# p ∈ R^3   —— 平移部分（3×1）
# [0, 1]    —— 最后一行 [0, 0, 0, 1]

# 应用：将向量从坐标系B变换到坐标系A
# p_A = R_AB * p_B + t_AB        （向量形式）
# p_A = T_AB * p_B                （齐次坐标形式，p_B = [x,y,z,1]^T）

齐次变换有两个关键性质：

第一，复合性质：\(\mathbf{T}_{AC} = \mathbf{T}_{AB} \mathbf{T}_{BC}\)——从A到C的变换等于"先到B，再到C"两个变换的矩阵乘积。这使得运动学链的构建变得极其简洁。

第二，逆变换：\(\mathbf{T}_{AB}^{-1} = \begin{bmatrix} \mathbf{R}^T & -\mathbf{R}^T \mathbf{p} \\ \mathbf{0} & 1 \end{bmatrix}\)。给定从A到B的变换，求反向变换只需要转置旋转部分并重新计算平移部分即可。

我用一个简单的比喻来帮助记忆：齐次变换矩阵就像"坐标系的GPS导航"——它告诉你从当前坐标系如何"走到"目标坐标系。复合变换相当于在导航中设置中途点，逆变换相当于"原路返回"。

5.5 四元数深入

四元数看起来"反直觉"，但它有深刻的数学结构。一个单位四元数 \(q = (w, x, y, z)\) 等价于旋转轴 \(\mathbf{v} = (x, y, z) / \|(x, y, z)\|\) 和旋转角 \(\theta = 2 \arccos(w)\)。也就是说，四元数其实是"绕轴旋转"的紧凑编码。

四元数最强大的地方是球面线性插值（Slerp）——在两个姿态之间进行平滑插值：

def slerp(q1, q2, t):
    """
    球面线性插值：q1 和 q2 是单位四元数，t ∈ [0, 1]
        t=0 → q1, t=1 → q2
    """
    # 计算夹角
    cos_theta = np.dot(q1, q2)
    # 如果夹角为负，取最短路径
    if cos_theta < 0:
        q2 = -q2
        cos_theta = -cos_theta
    # 如果夹角很小，退化为线性插值
    if cos_theta > 0.995:
        result = q1 * (1 - t) + q2 * t
        return result / np.linalg.norm(result)
    # 标准 Slerp
    theta = np.arccos(cos_theta)
    sin_theta = np.sin(theta)
    return (np.sin((1 - t) * theta) / sin_theta) * q1 + \
           (np.sin(t * theta) / sin_theta) * q2

为什么四元数插值比欧拉角插值好？考虑一个相机从姿态A (roll=0°, pitch=0°, yaw=0°) 到姿态B (roll=0°, pitch=180°, yaw=0°)——欧拉角线性插值会在中间经过一系列的剧烈翻滚，而四元数Slerp给出的是从正上方翻转到正下方的最短路径。在工业机器人轨迹规划中，用错插值方式可能导致机器人走一个奇怪的路径并撞到周围设备。

5.6 DH参数法

Denavit-Hartenberg（DH）参数法是机器人运动学建模的标准工具。它用4个参数来描述相邻两个关节坐标系之间的变换，这4个参数分别是：

  • \(a_{i-1}\)（link length）：沿着 \(x_{i-1}\) 轴，从 \(z_{i-1}\) 到 \(z_i\) 的距离
  • \(\alpha_{i-1}\)（link twist）：绕 \(x_{i-1}\) 轴，从 \(z_{i-1}\) 到 \(z_i\) 的角度
  • \(d_i\)（link offset）：沿着 \(z_i\) 轴，从 \(x_{i-1}\) 到 \(x_i\) 的距离
  • \(\theta_i\)（joint angle）：绕 \(z_i\) 轴，从 \(x_{i-1}\) 到 \(x_i\) 的角度（对于旋转关节，这是变量）

从关节 \(i-1\) 到关节 \(i\) 的齐次变换矩阵就是这四个参数的函数：

T_i = Rot(x, α_{i-1}) @ Trans(x, a_{i-1}) @ Rot(z, θ_i) @ Trans(z, d_i)

# 展开为4×4矩阵：
# T_i = [[cosθ_i,    -sinθ_i,      0,          a_{i-1}],
#        [sinθ_i cosα_{i-1},  cosθ_i cosα_{i-1},  -sinα_{i-1},  -d_i sinα_{i-1}],
#        [sinθ_i sinα_{i-1},  cosθ_i sinα_{i-1},   cosα_{i-1},   d_i cosα_{i-1}],
#        [0,         0,          0,          1]]

一个六轴机械臂的正向运动学就是6个DH变换矩阵的乘积：\(\mathbf{T}_{base}^{end} = \mathbf{T}_1 \mathbf{T}_2 \mathbf{T}_3 \mathbf{T}_4 \mathbf{T}_5 \mathbf{T}_6\)。这个计算看起来复杂，但在计算机中只是6次4×4矩阵乘法——现代处理器可以在微秒级别完成。

避坑指南：DH参数有至少三种约定（标准DH、改进DH、Craig的DH），不同教材和软件库使用不同的约定。当你从论文中复制DH参数时，一定要注意它用的是哪种约定——搞混了会导致运动学计算完全错误。ROS 2的robot_description中使用的urdf文件使用的是改进DH（Modified DH）约定。在我的项目里，有一次从论文中抄DH参数做仿真，就是由于约定不同导致末端位姿差了十万八千里，排查了整整两天才发现问题。

5.7 实战：用Python计算位姿变换

下面我们用NumPy实现旋转矩阵、四元数转换和DH变换链的构建：

import numpy as np

# === 1. 旋转矩阵构造 ===
def rot_x(theta):
    c, s = np.cos(theta), np.sin(theta)
    return np.array([[1, 0, 0],
                     [0, c, -s],
                     [0, s, c]])

def rot_y(theta):
    c, s = np.cos(theta), np.sin(theta)
    return np.array([[c, 0, s],
                     [0, 1, 0],
                     [-s, 0, c]])

def rot_z(theta):
    c, s = np.cos(theta), np.sin(theta)
    return np.array([[c, -s, 0],
                     [s, c, 0],
                     [0, 0, 1]])

# === 2. 欧拉角（ZYX）→ 旋转矩阵 ===
def euler_to_matrix(roll, pitch, yaw):
    """ZYX欧拉角 → 旋转矩阵 R = Rz(yaw) @ Ry(pitch) @ Rx(roll)"""
    return rot_z(yaw) @ rot_y(pitch) @ rot_x(roll)

# === 3. 四元数 ↔ 旋转矩阵 ===
def quat_to_matrix(q):
    """单位四元数 (w, x, y, z) → 旋转矩阵 (3×3)"""
    w, x, y, z = q
    return np.array([[1 - 2*y*y - 2*z*z, 2*x*y - 2*w*z,     2*x*z + 2*w*y],
                     [2*x*y + 2*w*z,     1 - 2*x*x - 2*z*z, 2*y*z - 2*w*x],
                     [2*x*z - 2*w*y,     2*y*z + 2*w*x,     1 - 2*x*x - 2*y*y]])

def matrix_to_quat(R):
    """旋转矩阵 → 单位四元数"""
    tr = np.trace(R)
    if tr > 0:
        S = np.sqrt(tr + 1.0) * 2
        w = 0.25 * S
        x = (R[2, 1] - R[1, 2]) / S
        y = (R[0, 2] - R[2, 0]) / S
        z = (R[1, 0] - R[0, 1]) / S
    # 处理其他情况（trace ≤ 0）
    else:
        # 简化为取最大对角元对应情况
        max_diag = np.argmax([R[0,0], R[1,1], R[2,2]])
        if max_diag == 0:
            w, x, y, z = (R[2,1] - R[1,2])/4, 0.25, (R[0,1]+R[1,0])/4, (R[0,2]+R[2,0])/4
        elif max_diag == 1:
            w, x, y, z = (R[0,2] - R[2,0])/4, (R[0,1]+R[1,0])/4, 0.25, (R[1,2]+R[2,1])/4
        else:
            w, x, y, z = (R[1,0] - R[0,1])/4, (R[0,2]+R[2,0])/4, (R[1,2]+R[2,1])/4, 0.25
    return np.array([w, x, y, z])

# === 4. DH变换链 ===
def dh_transform(a, alpha, d, theta):
    """改进DH参数的变换矩阵"""
    ct, st = np.cos(theta), np.sin(theta)
    ca, sa = np.cos(alpha), np.sin(alpha)
    return np.array([[ct,    -st,    0,      a],
                     [st*ca,  ct*ca, -sa,  -d*sa],
                     [st*sa,  ct*sa,  ca,   d*ca],
                     [0,      0,      0,     1]])

# === 示例：一个2-DOF机械臂的DH链 ===
l1, l2 = 1.0, 0.8
q1, q2 = np.deg2rad(30), np.deg2rad(45)

# DH参数: [a, alpha, d, theta]
dh_params = [
    [0,    0,      0, q1],    # joint 1: 绕z1旋转，连杆长度l1=0（基座到关节1）
    [l1,   0,      0, q2],    # joint 2: 绕z2旋转，连杆长度a1=l1
    [l2,   0,      0, 0 ],    # joint 3: 末端，固定
]

T = np.eye(4)
for params in dh_params:
    T = T @ dh_transform(*params)

print("末端位姿（4×4齐次变换矩阵）:")
print(T)
print(f"末端位置: {T[:3, 3]}")
print(f"末端旋转矩阵:\n{T[:3, :3]}")
print(f"末端四元数: {matrix_to_quat(T[:3, :3])}")
