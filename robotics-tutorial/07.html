第7章 传感器与执行器

大家好，我是你们的讲师。到上一章为止，我们一直在"纸上谈兵"——讨论数学、运动学、机构学，这些都是理论工具。而从这一章开始，我们开始接触机器人系统中真正的"物理层"。传感器是机器人的感官——它让机器人知道自己在哪、周围有什么、手上拿了什么东西。执行器是机器人的肌肉——它让机器人能够真正地与环境交互、做功、移动。

如果你去翻一台2026年的协作机器人的BOM表（物料清单），你会发现传感器和执行器的成本占到了整台机器人成本的60%以上。相比之下，控制器的成本在持续下降（一个树莓派级别的单板计算机已经可以跑通全套控制算法），但物理器件的精度和可靠性却是用实实在在的硬件成本堆出来的。做一个好的机器人工程师，不懂硬件是不行的。

7.1 内部传感器

内部传感器（Proprioceptive Sensors）测量机器人自身状态——关节位置、速度、力矩、本体加速度和角速度。它们是机器人"本体感觉"的来源。

编码器（Encoder）是最基础也最重要的内部传感器。光学编码器通过光栅盘上的透光/遮光条纹来测量角度位置。增量式编码器输出脉冲数（A/B两相，通过相位差判断方向），需要上电后归零（homing）。绝对式编码器直接输出绝对角度值，上电即知位置，不需要归零过程——这在工业机器人中非常重要，因为大型机械臂的归零过程本身就耗时数分钟且有碰撞风险。2026年，磁性编码器（基于磁阻效应MR或巨磁阻GMR）正在快速普及，虽然分辨率略低于光学编码器（通常17-19位 vs 光学编码器的20-23位），但其抗污能力、抗振动能力和成本都显著优于光学方案。

IMU（Inertial Measurement Unit）集成了加速度计和陀螺仪，是现代机器人不可或缺的传感器。加速度计测量三轴线加速度，陀螺仪测量三轴角速度。一个消费级IMU（如BMI088）的价格在10美元左右，零偏稳定性约 \(10^\circ/h\)；而工业级IMU（如ADIS16470）价格在100美元以上，零偏稳定性可达 \(1^\circ/h\) 甚至更好。在足式机器人中，IMU是平衡控制的核心——机器人每毫秒读取一次IMU数据，估算当前的倾斜角度，然后调整关节力矩来维持平衡。没有IMU，双足机器人连站都站不起来。

关节扭矩传感器直接测量关节输出力矩，是实现力控（Force Control）的关键。KUKA iiwa和Franka Emika Panda这些协作机器人之所以能够实现精准的力位混合控制，就是因为在每一个关节内置了高精度扭矩传感器（精度通常为额定扭矩的0.5%）。2026年，基于SAW（声表面波）的无线扭矩传感器开始出现在高端机器人中——它不需要电池，通过射频供电，特别适合在对布线和重量有严格限制的人形机器人手指关节中使用。

7.2 外部感知传感器

外部感知传感器（Exteroceptive Sensors）测量机器人与环境的关系——障碍物在哪、目标物体的形状是什么、地面是否平整。

相机（Camera）是最丰富的外部传感器。RGB相机提供颜色信息，用于物体识别和人体姿态估计。深度相机（如Intel RealSense、Microsoft Kinect）通过结构光或ToF技术同时提供RGB和深度信息，是服务机器人的标配。2026年的一个重要趋势是事件相机（Event Camera）的普及——它不像传统相机那样以固定帧率输出完整图像，而是只输出像素亮度发生变化的"事件"，延迟低至微秒级别、动态范围高达120dB以上。在高速机器人抓取（如乒乓球回球）和快速无人机避障中，事件相机正在取代传统相机。

LiDAR（Light Detection And Ranging）通过发射激光脉冲并测量反射时间来计算距离。2D LiDAR（单线扫描）广泛用于AGV/AMR的SLAM和避障。3D LiDAR（多线/固态）提供密集的点云数据，是人形机器人导航和环境感知的核心传感器。2026年，固态LiDAR（OPA/Flash LiDAR）的成本已经降至200美元以下（如Livox MID-70），使得3D LiDAR从高端工业应用进入了服务机器人和商业机器人领域。

触觉传感器（Tactile Sensor）在过去十年取得了突破性进展。2026年，GelSight和类似的高分辨率触觉传感器已经实现了商业化——通过相机拍摄弹性体的形变来获取接触区域的精细力分布（空间分辨率可达几十微米）。更重要的是，全身触觉皮肤（Whole-body Tactile Skin）正在从实验室走向产业化。人形机器人公司Figure AI和1X Technologies都在2025-2026年间展示了覆盖全身的触觉皮肤原型——机器人不再需要"看着"自己抓东西，而是像人一样通过触觉反馈来调整抓取力。这是一项革命性的进步，因为仅仅靠视觉引导的抓取在面对透明物体、反光物体或遮挡时都会失效，而触觉不需要"看见"。

趋势洞察：2026年传感器领域的一个核心趋势是"外感内感一体化"——IMU和相机融合（Visual-Inertial Odometry, VIO）已经成了标配，而最新的发展方向是LiDAR-IMU-相机三模态紧耦合。另一个趋势是"触觉普惠化"——随着触觉传感器成本的下降（从2020年的数千美元降至2026年的数百美元），触觉正在从实验室走向规模化应用。我们会在第21章机器人视觉和第22章机器人学习中继续讨论这个话题。

7.3 电机与驱动

执行器是机器人的"肌肉"，而电机是最核心的执行器。机器人电机的选型直接决定了机器人的性能上限——负载能力、运动速度、精度和能耗。

直流有刷电机（DC Brushed Motor）是传统的方案，结构简单、控制容易（只需调节电压），但碳刷磨损限制了寿命和高速性能。今天主要用于廉价玩具机器人或低负载服务机器人。

无刷直流电机（BLDC）在2026年已经成为工业机器人和人形机器人的主流选择。BLDC没有碳刷，通过电子换相器（ESC）驱动，寿命长、效率高（90%以上）、功率密度大。FOC（Field-Oriented Control）是BLDC的驱动灵魂——它通过Clark变换和Park变换实现精确的力矩控制。一个典型的FOC控制环路以20kHz运行，每50微秒完成一次电流采样、坐标变换和PWM更新。

步进电机（Stepper Motor）在开环位置控制中表现出色——不需要编码器，即可通过脉冲数精确控制角度位置。但步进电机在低速时有共振问题，在高速时力矩急剧下降。主要用于3D打印机、小型机械臂等对成本敏感、对高速不敏感的场景。

伺服电机（Servo Motor）通常指"电机+编码器+驱动器"的集成单元，是工业机器人的标准配置。伺服驱动器内部运行着位置环、速度环和电流环三个PID控制回路（我们第13章会详细讨论PID）。高端伺服系统（如安川SGM7系列、汇川MS1系列）可以做到4000rpm以上转速、22位以上编码器分辨率、1.5kHz以上的速度环带宽。

2026年执行器领域最受关注的技术是直驱电机（Direct Drive Motor）和准直驱执行器（Quasi-Direct Drive, QDD）。传统机器人在电机和关节之间使用高减速比减速器（50:1到200:1），这会引入摩擦和背隙。QDD使用低减速比（6:1到10:1），让电机直接驱动关节——牺牲了部分峰值力矩，但获得了极好的反向驱动性（backdrivability），即外力可以轻松推动关节。这对人形机器人和协作机器人至关重要——如果电机被撞到，低阻抗的QDD可以避免伤害人类和损坏自己。宇树科技的H2 Plus人形机器人就在其腿部关节中使用了QDD方案。

7.4 传动机构

电机输出的高速低扭矩运动需要通过传动机构转化为机器人关节所需的低速高扭矩运动。传动机构的设计直接影响机器人的精度、刚度、寿命和成本。

谐波减速器（Harmonic Drive）是工业机器人关节的标配。它通过柔性轴承的弹性变形实现高减速比（30:1到160:1）、零背隙（zero backlash）、结构紧凑（同轴）、重量轻。谐波减速器的缺点是柔性轴承有疲劳寿命限制（约8,000-20,000小时满负载运行），以及输出刚度相对较低。常见品牌有日本的Harmonic Drive Systems和中国的绿的谐波。

RV减速器（Rotary Vector）是谐波减速器的高刚度替代方案。RV减速器使用摆线针轮传动，具有极高的扭转刚度和过载能力，但结构更复杂、成本更高。大负载工业机器人（20kg以上）几乎全部使用RV减速器。日本的纳博特斯克（Nabtesco）在RV减速器领域占据主导地位。

行星齿轮减速器（Planetary Gearbox）结构简单、效率高、成本低，但存在背隙（通常3-10 arcmin）。广泛应用于对精度要求不高、对成本和效率要求较高的场景，如AGV的驱动轮和简单抓取机械臂。

带传动（Belt Drive）和丝杠（Ball Screw）在直角坐标系机器人（Gantry）和直线运动系统中常见。带传动适合长距离、高速运动但力矩较小的场景（如3D打印机的XY轴）。滚珠丝杠则将旋转运动转换为直线运动，效率高（可达90%以上）、精度高，是机床和精密定位平台的标配。

知识点：有一个重要概念叫做"传动比的选择原则"——在满足力矩需求的前提下，尽量选择最小的减速比。为什么？因为减速比越高，反射到电机端的负载惯量越小，系统的响应速度越快，但同时也意味着外力更难反驱（backdrivability差）。对于协作机器人，选择适中的减速比是一个权衡——力矩够用即可，不要过度放大。

7.6 传感器选型实战

最后，分享一个我亲身经历的传感器选型教训。2019年，我在开发一台精密装配机器人，任务是将直径0.5mm的微小型轴承压入轴孔中，装配公差仅±5微米。我们选用了电机自带的17位增量式编码器（分辨率约0.003°），理论上结合减速比计算出的末端精度可以达到3微米——"看起来够用"。

结果在测试中，机器人的重复定位精度只有±15微米，远达不到要求。排查了很久，发现问题的根源不在编码器分辨率本身，而在于增量式编码器的温度漂移——在工作2小时后，电机发热导致编码器信号发生相位偏移，等效于位置误差累积。我们换成了绝对式编码器（不受温度影响）后，重复定位精度立刻提升到了±3微米，满足了工艺要求。

这个教训的核心是：传感器选型不能只看理论分辨率，还要看实际工作条件下的稳定性、漂移和抗干扰能力。以下是我总结的选型三步法：

# 机器人传感器选型检查清单

# Step 1: 确定精度要求
#   任务要求末端精度 → 反推关节角度精度
#   例: 末端位置精度 ±0.05mm, 臂长0.5m, 减速比100
#   关节角度精度要求 = ±0.05 / (500 * sin≈1) * 100 ≈ ±0.01 rad ≈ ±0.6°
#   编码器分辨率需求 > 12位 (0.088°) → 安全起见选择16位以上

# Step 2: 考虑环境因素
# - 温度范围: 编码器工作温度是否覆盖机器人工作温度?
# - 抗振动: 编码器是否能在机器人运动振动下稳定工作?
# - 防护等级: 是否需防水防尘? (焊接场景需防飞溅)
# - 电磁兼容: 电机高电流切换会不会干扰传感器信号?

# Step 3: 系统级权衡
# - 精度: 高精度传感器更贵、更大、更重, 够用即可
# - 频率: 控制循环频率决定了传感器最小延迟要求
# - 成本: 关节传感器 × 关节数量 = 巨大成本, 合理取舍
# - 接口: 是否兼容主控板的通信协议? (EtherCAT, CAN, SPI, I²C?)

# 示例: 六轴协作机器人传感器选型
# 关节编码器: 每个关节 × 19位磁性绝对值编码器
# 关节扭矩: 仅关键关节(腕部) × 高精度扭矩传感器
# IMU: 本体1个 × 工业级6轴IMU
# 相机: 1个 × RGB-D深度相机(RealSense D435i)
# 力传感器: 末端1个 × 六维力传感器(ATI Mini45)

这个检查清单虽然简单，但如果你在做机器人项目时能花30分钟过一遍，绝对能省下后面数周甚至数月的排查时间。记住：在硬件上贪便宜，最终会在开发时间上还回来。

避坑指南：传感器选型中最容易被忽视的是"时间同步"问题。当你使用多个传感器（相机、LiDAR、IMU）时，每个传感器的采样时间和延迟可能完全不同。如果不对这些时间戳做对齐，融合结果可能比不使用传感器更差。例如，一个延迟50ms的视觉观测和延迟1ms的IMU观测融合时，如果不做时间补偿，相当于用"过去50ms的信息来修正当前状态"——这在快速运动中会引入严重的误差。使用硬件同步信号（如PPS或EtherCAT的分布式时钟）来对齐各传感器的时间戳是标准的工程实践。

好的，传感器和执行器的基础知识就讲到这里。我们从内部传感器（编码器、IMU、扭矩传感器）、外部传感器（相机、LiDAR、触觉）、电机（BLDC、步进、伺服）、传动机构（谐波、RV、行星齿轮）一路讲下来，最后分享了选型实战经验和检查清单。硬件准备就绪，Part 2我们将进入机器人学的核心引擎——运动学。下一章我们从最基础的开始：正向运动学。下一章见。
