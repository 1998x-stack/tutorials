第2章 以太网的演化史

大家好，我是你们的讲师。上一章我们聊了以太网的出生——1973年Bob Metcalfe在Xerox PARC画出的那张草图，2.94 Mbps的速率，"以太"这个诗意的名字。但如果以太网仅仅停留在那个阶段，它早就和Token Ring、FDDI一样成为博物馆里的展品了。以太网之所以能统治局域网四十多年，靠的是它惊人的进化能力。

从粗笨的同轴电缆到轻巧的双绞线，从共享总线的"吵嚷"时代到交换机的精确转发，从10 Mbps到800 Gbps——每一次进化都不仅仅是速度的提升，更是设计范式的转变。本章我们就沿着这条时间线，看看以太网是如何一次次"脱胎换骨"的。

2.1 物理介质的革命：从粗缆到双绞线

1980年代初期的以太网，用的是10BASE5——一根直径接近1厘米、又粗又硬的同轴电缆，官方名字叫"粗缆"（Thick Ethernet）。为什么叫"粗缆"？因为它的直径相当于你的小拇指，外面包裹着厚厚的绝缘层，最大段长可达500米。安装10BASE5需要一种特殊的设备："吸血鬼接头"（Vampire Tap）——你需要在电缆上钻一个孔，然后把一个带针脚的接头"咬"进去，让针脚接触到电缆的内导体。这个名字非常形象：就像吸血鬼咬住猎物一样，一旦咬住就不再松口。一块10BASE5网卡加上一个AUI（Attachment Unit Interface）收发器，总成本在当年大约相当于一个普通工程师半个月的工资。每次看到老照片里那些笨重的黄色电缆，我都会感慨：技术的进步，本质上就是让同样的事情变得更便宜、更容易。

1985年，10BASE2（细缆以太网，Thin Ethernet）出现了。它使用更细的RG-58同轴电缆，直径只有约0.5厘米，最大段长185米——"2"就代表大约200米（取整）。安装方式也简单了很多：不再需要昂贵的吸血鬼接头，而是用BNC T型接头直接把计算机串联起来。这种"菊花链"拓扑一度非常流行，尤其是在小型办公室和实验室里。但10BASE2也有它的痛点：任何一个连接点松动了，或者某台机器的T型接头坏了，整个网段上的所有机器都会断网——这就是总线拓扑的"一坏全坏"问题。1990年代初我在学校机房维护时，最害怕的就是整个网段突然掉线，然后一根一根检查BNC接头——往往发现是某个角落里的接头被清洁工拖地时碰松了。

真正的革命发生在1990年。10BASE-T（Twisted Pair，双绞线）标准（IEEE 802.3i）被正式批准。它使用两对无屏蔽双绞线（UTP），用RJ-45接头连接，拓扑从总线型变成了星型——每台设备通过一根独立的网线连接到中心的集线器（Hub）。这听起来没什么了不起，但它的影响是深远的：

  • 故障隔离：一根网线坏了只影响一台设备，不会再出现"一坏全坏"的噩梦。
  • 布线简化：双绞线比同轴电缆便宜得多，也更容易弯曲和走线。
  • 为交换机做好了准备：星型拓扑天然适合用交换机替代集线器。

一个踩坑故事：2005年我帮一家小公司做网络升级，老工程师坚持要用10BASE2细缆，说"我那套线缆用了十年都没问题"。结果我们测了测，发现他的"十年没问题"其实是因为——公司的业务流量非常低，偶尔传几个Excel文件。当我们把网络升级到10BASE-T并接入互联网后，每周都有莫名其妙的断网事件，最后发现是墙内埋的细缆有一处绝缘层破损，导致信号反射和CRC校验错误。同轴电缆在低频下可以容忍一定的信号质量问题，但一旦数据量增大、信号频率提高，那些"藏"着的问题就会全部暴露出来。

2.2 从集线器到交换机的架构升级

在10BASE-T的时代，中心设备是集线器（Hub）。集线器的工作原理非常简单：从一个端口收到的信号，原封不动地复制到所有其他端口上。这意味着所有端口共享10 Mbps的带宽——这就好比一群人在一个房间里开会，任何时候只有一个人能说话，其他人必须听着。如果同时有两个人说话（碰撞），所有人都会听到噪音，然后等待随机时间再重新发言。

1990年，一家名叫Kalpana的初创公司做出了一个改变历史的发明——以太网交换机（Ethernet Switch）。Kalpana的EtherSwitch是世界上第一款商用以太网交换机。它的核心思想在今天看来理所当然：交换机不再是"广播"收到的信号，而是根据数据帧的目的MAC地址，只把它转发给需要的那一个端口。这意味着多个端口可以同时通信——A端口发数据给B端口的同时，C端口可以发数据给D端口，互不干扰。

这个改进的效果是惊人的。一个8口的集线器，总带宽只有10 Mbps（所有端口共享）。而一个8口的交换机，理论总带宽可以达到8×10 = 80 Mbps（每个端口独享）。更重要的是，交换机的出现让冲突域缩小到了每个端口上——换句话说，交换机彻底消除了端口之间的冲突。这个转变的意义我们会在第7章深入讨论。

1994年，Cisco收购了Kalpana，交换技术从此进入了主流。到了1990年代末期，交换机的价格已经从最初的数千美元降到了几百美元，集线器迅速退出了历史舞台。今天，你几乎已经找不到还在使用集线器的网络环境了——这也是为什么很多年轻工程师可能根本没见过集线器长什么样。

2.3 速率跃迁：十年十倍的增长

以太网的速率演进是理解其生命力的关键线索。下面这张表大致按照时间顺序，展示了以太网速率跃迁的完整图景：

速率 标准 发布年份 介质 关键创新
2.94 Mbps 实验性以太网 1973 同轴电缆 Alto计算机系统时钟分频
10 Mbps Ethernet II / 802.3-1985 1980/1985 同轴电缆/双绞线 DIX联盟标准化，CSMA/CD
100 Mbps Fast Ethernet (802.3u) 1995 Cat5 UTP/光纤 保持CSMA/CD，缩短位时间
1 Gbps GigE (802.3z/ab) 1998/1999 光纤/Cat5e UTP 载波扩展+帧突发，全双工为主
10 Gbps 10GigE (802.3ae) 2002 光纤/CX4 彻底放弃CSMA/CD，仅全双工
40/100 Gbps 802.3ba 2010 多模光纤/铜缆 多通道并行传输
200/400 Gbps 802.3bs 2017 单模光纤 PAM4调制，前向纠错(FEC)
800 Gbps / 1.6 Tbps 802.3dj 2024+ 单模光纤 224 Gbps/通道，C2C/LRO

在Linux系统中，你可以用 ethtool 命令查看当前网卡支持的速率和已经协商好的连接速度：

# ethtool eth0
Settings for eth0:
    Supported ports: [ TP ]
    Supported link modes:   10baseT/Half 10baseT/Full
                            100baseT/Half 100baseT/Full
                            1000baseT/Full
    Speed: 1000Mb/s
    Duplex: Full
    Auto-negotiation: on
    Link detected: yes

这张表里有几个值得注意的关键转折点。第一个是1995年的Fast Ethernet（802.3u）——当时Token Ring的支持者认为以太网永远不可能超过10 Mbps，因为CSMA/CD的碰撞检测机制在高速度下无法工作。但IEEE 802.3u工作组用一个巧妙的方法解决了这个问题：保持最小帧长（64字节）不变，把位时间从100 ns缩短到10 ns，同时把最大网络直径从2500米缩小到200米。这样碰撞检测机制依然有效，只是覆盖范围变小了。在大多数实际场景中，200米足以覆盖一个楼层内的所有设备。

第二个转折点是Gigabit Ethernet（802.3z/ab）——千兆以太网在半双工模式下引入了一种叫做"载波扩展"（Carrier Extension）和"帧突发"（Frame Bursting）的技术，以解决高速下的最小帧长问题。但实际上，千兆以太网的大部分部署都采用了全双工模式，彻底避免了碰撞问题。从此以后，CSMA/CD基本上成了以太网的历史包袱——后续的10 Gbps及以上标准全部只支持全双工模式。

第三个转折点是802.3ba（40/100 Gbps）——这是以太网标准第一次出现了两个速率同时被定义。背后的驱动因素是数据中心的需求：服务器接入需要40 Gbps，而核心汇聚需要100 Gbps。从这一代开始，多通道并行（如4×25 Gbps或10×10 Gbps）成为了提升速率的常用手段。

最近一代802.3dj（800 Gbps / 1.6 Tbps）使用的是PAM4调制和224 Gbps通道技术。PAM4（4级脉冲幅度调制）每个符号传输2比特，而不是传统NRZ的1比特，通过更复杂的调制方式来突破信号速率的上限。这些技术我们会在第27章深入讲解。

2.4 以太网为什么能"吃掉"所有对手

回顾这四十多年的历史，你会发现一个有意思的现象：每一种曾经与以太网竞争的局域网技术——Token Ring、FDDI、ARCnet、LocalTalk、100VG-AnyLAN——全都消失了。以太网用"吞噬"的方式吸收了所有对手的市场，这背后有几个深层原因：

第一，成本。以太网的规模效应是任何竞争对手都无法比拟的。到1990年代中期，全世界的以太网网卡年出货量已经达到数千万片，而Token Ring网卡的年出货量只有几十万片。规模的差距直接体现在成本上：1995年一块以太网网卡的价格大约是15美元，而一块Token Ring网卡是400美元。当你的网络规模超过100台设备时，使用Token Ring的网络设备成本是以太网的数倍，这是一个企业级客户无论如何都无法接受的价格差距。

第二，生态。以太网的成功不是一家公司的成功，而是一个产业生态的成功。从芯片制造商（Intel、Broadcom、Marvell）到设备厂商（Cisco、Juniper、Huawei）到软件开发商——整个产业链都在围绕以太网进行创新。而Token Ring高度依赖IBM一家公司，FDDI依赖DEC和Cabletron等少数厂商。单一厂商的技术路线一旦停滞，整个生态就会衰落。1998年IBM停止开发Token Ring产品时，这个技术实际上就已经被判了死刑。

第三，向后兼容。这是一个经常被低估但极其重要的因素。你的千兆以太网网卡插到百兆交换机上仍然能工作——通过Auto-Negotiation（自动协商）机制降速到100 Mbps。你的10 Mbps集线器可以与100 Mbps交换机共存（虽然只能以10 Mbps速率通信）。这种"向下兼容"能力意味着企业可以渐进式升级网络，而不需要一次性替换全部设备。相比之下，Token Ring从4 Mbps升级到16 Mbps需要更换网卡和MAU，而从16 Mbps到100 Mbps的HSTR（High-Speed Token Ring）更是彻底失败——几乎没有厂商愿意支持一个即将消亡的标准。

第四，持续创新。IEEE 802.3工作组并非一成不变地维护着"祖传"标准。他们不断吸收新技术：自动协商、全双工、流控、链路聚合、VLAN、PoE、时间同步……每一次创新都让以太网变得更加适应新的场景。用我们行业里的一句话来说："以太网不是一个标准，而是一个持续演化的标准家族。"

工程师的启示：以太网的故事告诉我们，技术竞争不仅仅是"谁更快"的问题。在很多情况下，"够用 + 便宜 + 开放"的组合，比"更快 + 更贵 + 封闭"更容易获胜。如果你正在做技术选型，不妨问自己一个问题："我更在乎的是绝对性能，还是生态兼容性和总拥有成本？"

好了，以上就是以太网四十多年的进化简史。从粗缆到双绞线，从集线器到交换机，从2.94 Mbps到1.6 Tbps——以太网的每一步进化都伴随着"为什么不行"的质疑和"其实可以"的回答。下一章（第3章：OSI模型与以太网的位置），我们来聊聊以太网在OSI七层模型中到底处于什么位置，以及物理层和数据链路层是如何分工配合的。这是一个理解以太网设计哲学的关键视角，不要错过。
