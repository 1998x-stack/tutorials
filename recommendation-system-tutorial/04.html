第4章 协同过滤：推荐的基石

大家好，我是你们的讲师。前几章我们打下了基础——了解了推荐系统的全貌、它的发展史以及评估体系。从这一章开始，我们正式进入算法的世界。第一个要讲的就是推荐系统最经典、最朴素、也最有影响力的算法——协同过滤（Collaborative Filtering, CF）。

协同过滤的思想可以用八个字概括："人以群分，物以类聚"。就是这么简单的直觉，支撑了整个推荐系统行业从0到1的起步。

1. 协同过滤的核心思想

想象你去一个陌生的城市出差，晚上想找个饭馆吃饭。你问一个同事："附近有什么好吃的？"同事说："我不太清楚，但老王口味和你很像——你们上次出差都喜欢那家川菜馆。我问问老王。"这就是协同过滤的核心：借助相似人群的经验来为你做决策。

从数学角度看，协同过滤利用的是用户-物品交互矩阵（User-Item Interaction Matrix）中的"共现模式"：用户A和用户B都喜欢物品1、2、3，那么用户A喜欢的物品4也很可能被用户B喜欢。

协同过滤主要分为两大类：

  • 基于内存的协同过滤（Memory-based CF）：直接用用户或物品之间的相似度做预测，包括UserCF和ItemCF
  • 基于模型的协同过滤（Model-based CF）：通过机器学习模型学习用户和物品的隐式表示，矩阵分解就是最典型的代表（我们下一章会讲）

2. UserCF：基于用户的协同过滤

2.1 算法原理

UserCF的基本流程分三步：

  1. 计算用户间的相似度：用余弦相似度或皮尔逊相关系数，计算目标用户与其他所有用户的相似度
  2. 找到最相似的K个邻居：选择与目标用户最相似的K个用户
  3. 预测物品评分：用邻居用户对某个物品的评分（加权平均）来预测目标用户对该物品的评分

预测公式如下：

# 预测用户 u 对物品 i 的评分
# sim(u, v) 是用户 u 和用户 v 的相似度
# r_v,i 是用户 v 对物品 i 的评分
# r_u_bar 是用户 u 的平均评分，r_v_bar 是用户 v 的平均评分

pred(u, i) = r_u_bar + Σ_v sim(u, v) * (r_v,i - r_v_bar) / Σ_v |sim(u, v)|

为什么要用平均评分做修正？因为有些用户习惯于给高分（比如永远5星），有些用户给分很严格（3分就算好评了）。减去平均评分可以消除这种"评分偏差"。

2.2 一个简单的例子

假设我们有一个用户-电影评分矩阵（评分范围1-5，0表示未评分）：

用户 《泰坦尼克号》 《阿凡达》 《盗梦空间》 《星际穿越》
用户A 5 4 5 ?
用户B 4 5 3 5
用户C 2 3 1 4
用户D 5 4 4 ?

我们要预测用户A对《星际穿越》的评分。直观上看，用户D和A的口味最相似（都给《泰坦尼克号》5分、《阿凡达》4分，《盗梦空间》A给了5分D给了4分也很接近）。UserCF会计算用户A与其他所有用户的相似度，选中用户D作为最相似的用户（可能还有用户B），然后用他们各自对《星际穿越》的评分来预测。

2.3 UserCF的优缺点

  • 优点：直觉简单，易于实现，推荐结果有社交解释性（"和你相似的用户也喜欢这个"）
  • 缺点：用户数增长时计算量爆炸（用户相似度矩阵是O(n²)的），对新用户和新物品都存在冷启动问题，推荐质量高度依赖于用户间共有的评分数据（稀疏性）

3. ItemCF：基于物品的协同过滤

ItemCF的思想和UserCF相反——不找"相似的人"，而是找"相似的物品"。"购买此商品的客户也购买了……"就是ItemCF最经典的应用。

3.1 算法原理

ItemCF的基本流程：

  1. 计算物品间的相似度：如果两个物品被同一群用户喜欢，它们就是相似的。注意这里不是基于物品的内容属性（颜色、价格、类别），而是基于用户的"共现行为"
  2. 为用户推荐与他过去喜欢的物品相似的物品

物品i和物品j的相似度计算公式（最常见的是余弦相似度的变体）：

# sim(i, j) = 喜欢物品i和物品j的用户数 / sqrt(喜欢物品i的用户数 * 喜欢物品j的用户数)
# 这个公式对热门物品做了惩罚（分母中的sqrt部分）

3.2 为什么ItemCF比UserCF更适合电商

Amazon在2003年的论文中给出了非常精彩的分析：

  • 稳定性：电商场景中用户数量远大于商品数量，而且商品变化比用户偏好变化慢。物品间的相似度可以离线计算，每天甚至每周更新一次即可——而用户相似度需要实时计算（因为用户行为每秒都在变）
  • 可解释性："因为你买了iPhone，所以推荐iPhone保护壳"比"因为和你相似的人也喜欢这个，所以推荐这个"更直接、更有说服力
  • 推荐质量：ItemCF的推荐结果更集中，更适合"买完这个还会买什么"的场景

3.3 一个ItemCF的例子

回到上面的评分矩阵。如果用ItemCF为喜欢《泰坦尼克号》的用户推荐电影，我们会计算《泰坦尼克号》与其他电影的用户共现相似度：

  • 《泰坦尼克号》与《阿凡达》：用户A、B、D都同时喜欢这两部，相似度很高
  • 《泰坦尼克号》与《盗梦空间》：用户A、D同时喜欢，用户C都不喜欢，相似度中等
  • 《泰坦尼克号》与《星际穿越》：数据太少，难以判断

所以给喜欢《泰坦尼克号》的用户，ItemCF会优先推荐《阿凡达》，其次是《盗梦空间》。

UserCF vs ItemCF 选型指南：

  • 用户数 >> 物品数（如电商）：用 ItemCF，物品相似度计算成本低且稳定
  • 物品数 >> 用户数（如新闻门户）：用 UserCF，因为新闻更新太快，ItemCF的物品相似度缓存秒变无效
  • 重视个性化体验：UserCF 的推荐结果更多样、更"意外"；ItemCF 的推荐更保守、更集中
  • 重视可解释性：ItemCF 的解释"因为你喜欢X，所以推荐Y"更有说服力

4. 相似度计算公式详解

相似度计算是协同过滤的核心组件。不同的相似度公式适用于不同的数据场景：

相似度指标 公式 适用场景 优势 劣势
余弦相似度（Cosine） cos(u,v) = (u·v) / (||u||·||v||) 评分数据、隐式反馈（0/1） 计算简单，不受坐标平移影响 不考虑评分尺度偏差
皮尔逊相关系数（Pearson） pearson(u,v) = cos(u-ū, v-v̄) 评分数据（不同用户的评分尺度不同） 消除评分偏差（用户A全给5星也没关系） 需要足够的共同评分项，对稀疏数据不稳定
Jaccard系数 J(A,B) = |A∩B| / |A∪B| 隐式反馈（只看有无，不看打分） 简单，适合二值数据 完全不考虑评分尺度，丢失信息量

5. 工业实践中CF的死穴

协同过滤虽然经典，但在工业规模下有很多致命问题。作为从业者，你必须对这些问题有清醒的认识。

5.1 冷启动（Cold Start）

CF完全依赖用户的历史交互数据。当一个新用户进入系统（没有任何行为）或者一个新商品上线（没有任何曝光），CF完全无能为力。这是CF最根本的局限，也是工业推荐中必须用"内容推荐"（Content-based）或"热度推荐"（Popularity-based）来补齐的原因。

5.2 稀疏性（Sparsity）

即使系统中有大量用户和物品，单个用户交互过的物品占比通常极小。在淘宝场景中，一个用户可能浏览过几百个商品，但淘宝商品总量以亿计——数据稀疏度远低于0.01%。在这种极端稀疏下，用户间"共同评分"几乎不存在，UserCF基本失效。

5.3 热门偏差（Popularity Bias）与哈利波特效应

ItemCF有一个臭名昭著的问题——"哈利波特效应"。因为《哈利波特》是超级畅销书，几乎每个买书的人都买过它。在ItemCF中，"买过书A的人也买了《哈利波特》"总是成立的，导致《哈利波特》出现在几乎所有人的推荐里。结果就是推荐列表缺乏多样性，用户看到的内容越来越趋同。这是整个推荐系统领域长期研究的热门偏差问题的一个缩影。

真实故事：一个电商平台的ItemCF崩盘事件

2018年左右，我听说某东南亚电商平台遇到了一个经典问题：他们的推荐系统重度依赖ItemCF，初期效果不错。但随着平台从几万SKU扩张到百万级别，数据稀疏度急剧恶化，ItemCF的覆盖率越来越低。更糟糕的是，他们发现推荐结果被少数爆款商品"霸占"了——因为热门商品和几乎所有其他商品的共现度都很高，ItemCF天然倾向于推荐爆款。平台的很多长尾商品从未被推荐到用户面前。他们最终的解决方案是：用ItemCF做召回之一（不是唯一的召回），同时引入基于内容的召回和基于图的召回，再通过重排阶段控制多样性。这也是工业推荐系统的常态——没有一个算法能包打天下，多路召回才是真相。

6. 用numpy实现UserCF和ItemCF

下面是一个极简的numpy实现，帮助你理解CF的核心计算逻辑：

import numpy as np

class UserCF:
    def __init__(self, n_neighbors=2):
        self.n_neighbors = n_neighbors

    def fit(self, rating_matrix):
        """rating_matrix: shape (n_users, n_items), 0 means unknown"""
        self.rating_matrix = rating_matrix.copy()
        self.n_users, self.n_items = rating_matrix.shape
        # 计算用户相似度矩阵 (cosine)
        norm = np.linalg.norm(rating_matrix, axis=1, keepdims=True)
        norm[norm == 0] = 1
        norm_matrix = rating_matrix / norm
        self.user_sim = norm_matrix @ norm_matrix.T

    def predict(self, user_id, item_id):
        """预测用户user_id对物品item_id的评分"""
        if item_id not in range(self.n_items):
            return 0
        # 找到与目标用户最相似的K个邻居
        sims = self.user_sim[user_id]
        neighbors = np.argsort(sims)[::-1][1:self.n_neighbors + 1]
        neighbor_sims = sims[neighbors]
        neighbor_ratings = self.rating_matrix[neighbors, item_id]
        mask = neighbor_ratings > 0  # 只考虑有评分的邻居
        if mask.sum() == 0:
            return 0
        return np.dot(neighbor_sims[mask], neighbor_ratings[mask]) / neighbor_sims[mask].sum()

class ItemCF:
    def __init__(self, n_neighbors=2):
        self.n_neighbors = n_neighbors

    def fit(self, rating_matrix):
        self.rating_matrix = rating_matrix.copy()
        self.n_users, self.n_items = rating_matrix.shape
        # 计算物品相似度矩阵
        norm = np.linalg.norm(rating_matrix, axis=0, keepdims=True)
        norm[norm == 0] = 1
        norm_matrix = rating_matrix / norm
        self.item_sim = norm_matrix.T @ norm_matrix

    def recommend(self, user_id, top_k=3):
        """为用户user_id推荐top_k个物品"""
        user_ratings = self.rating_matrix[user_id]
        scores = np.zeros(self.n_items)
        for i in range(self.n_items):
            if user_ratings[i] > 0:
                scores += self.item_sim[i] * user_ratings[i]
        # 排除已评分的物品
        scores[user_ratings > 0] = -1
        return np.argsort(scores)[::-1][:top_k]

# 测试
ratings = np.array([
    [5, 4, 5, 0],
    [4, 5, 3, 5],
    [2, 3, 1, 4],
    [5, 4, 4, 0],
])

ucf = UserCF(n_neighbors=2)
ucf.fit(ratings)
print(f"UserCF: 用户A对《星际穿越》的预测评分 = {ucf.predict(0, 3):.2f}")

icf = ItemCF(n_neighbors=2)
icf.fit(ratings)
print(f"ItemCF: 给用户A推荐 = {icf.recommend(0, top_k=3)}")

协同过滤是一个"上了年纪"但依然经典的算法。它的核心价值在于提出了一个极其重要的思想：用户的行为数据本身就蕴含着结构——我们可以通过"人群的智慧"来做推荐，而不需要理解物品的内容。正是这个思想，启发了后续模型可以"自动学习"用户和物品的表示——这就是我们下一章要讲的矩阵分解与隐语义模型。矩阵分解可以看作协同过滤的"深度学习前身"，它用隐向量的方式解决了CF的稀疏性问题。
