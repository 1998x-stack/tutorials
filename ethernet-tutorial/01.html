第1章 以太网的起源与精神

大家好，我是你们的讲师。今天我们要聊的是一个你可能每天都在用、却很少去思考它来历的技术——Ethernet（以太网）。

你可能会说："以太网？不就是插根网线、连上路由器就能上网吗？有什么好讲的？"

嗯，如果你这么想，那今天这一章可能会让你大吃一惊。以太网的故事，远比"插网线上网"这四个字要精彩得多。它是一部关于技术选择、商业联盟、开放标准如何击败封闭体系的史诗。而这一切，要从1973年说起——那一年，互联网还叫ARPANET。

1.1 为什么需要以太网

把时间拨回1973年。那一年，第一台手机还没诞生（摩托罗拉要等到10年后才推出DynaTAC），个人计算机还只是极客圈子里的玩具。如果你想在同一个办公室里把两台计算机连起来交换数据，你的选择极其有限：要么用磁带把数据倒腾过去（俗称"SneakerNet"——穿着球鞋跑一趟），要么通过电话线用调制解调器以300 bps的龟速拨号连接。300 bps是什么概念？传输一份今天的普通网页（约2 MB），你需要连续不断传输将近15个小时。

你可能会问："不是有ARPANET吗？"

没错，ARPANET确实存在，但那是广域网——连接的是美国各地的大学和军方研究机构。想接入ARPANET？你得先通过美国国防部高级研究计划局（ARPA）的审批，还需要专门的接口消息处理器（IMP），一台设备的价格相当于今天的数十万美元。对于普通公司或实验室来说，这完全是天方夜谭。ARPANET解决的是"远距离互联"的问题，但"近距离互联"——也就是一台计算机和隔壁另一台计算机之间的通信——反而是一片空白。

更有意思的是，当时世界上还没有"个人计算机"这个品类。1973年的典型场景是这样的：一栋大楼里有一台昂贵的大型机（mainframe），几十个用户各自坐在"哑终端"前，通过RS-232串行线连接到这台大型机上。所有的计算能力集中在一个点，终端只是"键盘+屏幕"的组合。这种模式叫做集中式计算。它的缺陷很明显：成本高（一台大型机几十万美元）、单点故障（大型机一挂，所有人罢工）、扩展难（想加一个用户就得拉一根串行线）。

但如果反过来想呢？如果让每一台计算机都拥有自己的处理能力，然后通过网络把它们连接起来互相通信——这不就是后来的"PC + 局域网"模式吗？这个在今天看来理所当然的想法，在1973年可是革命性的。当时的人们面临的核心问题是：能不能有一种技术，能在短距离内（比如一栋办公楼或一个园区），以低成本、高可靠性的方式，把几十台甚至上百台计算机连接起来？这就引出了局域网（Local Area Network, LAN）的概念。但在1973年，局域网的世界还是一片"蛮荒之地"。

1.2 ARPANET之前的局域网世界

在以太网诞生前后，市场上涌现过各种各样的局域网技术。用今天的眼光回溯，那真是一个"百家争鸣"的年代。我们来认识几位当时的"参赛选手"：

技术 推出时间 拓扑结构 传输速率 介质访问方式 最终结局
Ethernet 1973/1980 总线 2.94 Mbps → 10 Mbps CSMA/CD 胜出，一统天下
Token Ring 1984 环型 4 → 16 Mbps 令牌传递 2000年后消亡
ARCnet 1977 星型/总线 2.5 Mbps 令牌传递 1990年代消亡
FDDI 1988 双环 100 Mbps 令牌传递 2000年代消亡
LocalTalk 1985 总线 230.4 kbps CSMA/CA 1990年代消亡

看到这张表，你可能已经注意到一个有趣的现象：除了以太网之外，其他技术几乎都采用了令牌传递（Token Passing）的访问方式。这种方式的优点是确定性——不会发生碰撞，网络负载高时性能依然稳定。但代价是什么？复杂、昂贵、扩展性差。Token Ring的网卡和集线器（MAU）比以太网的同级别设备贵好几倍，而且不同厂商的设备之间常常存在兼容性问题。FDDI虽然达到了100 Mbps的速率（在1988年这是个惊人的数字），但它使用光纤和双环结构，部署成本堪比买一台小汽车。ARCnet是最早的商业化局域网之一，但它只有2.5 Mbps，而且市场主要局限在工业自动化领域。

反观以太网，它走的是另一条路——简单、开放、允许碰撞、通过"退避重试"来解决冲突。这在当时被很多"正统"的网络专家嗤之以鼻："什么？你的网络居然允许数据包互相撞车？这太不'工程'了！"但事实证明，有时候"足够好"比"完美"更有力量。

说到这里，我想起一个有意思的插曲。1990年代初期，IBM的Token Ring在技术上明显优于当时还停留在"冲突检测"阶段的以太网。IBM的销售团队经常拿"以太网会崩溃"作为卖点去游说企业客户。但到了1990年代后期，情况发生了戏剧性的逆转——以太网交换机的价格急剧下降，而100 Mbps的Fast Ethernet横空出世，直接把Token Ring的16 Mbps上限按在地上摩擦。从那时起，Token Ring的市场份额就像坐滑梯一样直线下降，2000年后彻底退出了历史舞台。

1.3 Xerox PARC的"摇篮"

要讲以太网诞生的故事，必须先讲一个地方——Xerox PARC（施乐帕克研究中心）。

PARC位于加利福尼亚州的帕洛阿尔托，成立于1970年。这个地方有多牛？随便列举几个它的发明你就明白了：图形用户界面（GUI）、激光打印机、鼠标、面向对象编程、Smalltalk语言、WYSIWYG文字处理……毫不夸张地说，PARC几乎以一己之力奠定了现代个人计算机的基石。硅谷流传着一句老话："PARC是那个创造了未来、却没能从中赚到钱的地方。"

1972年夏天，一个名叫Bob Metcalfe（罗伯特·梅特卡夫）的年轻人来到了PARC实习。Metcalfe的学术背景很有意思：他先在MIT读本科，然后去哈佛读博士。在攻读博士期间，他参与了对ALOHAnet的研究——这是夏威夷大学在1970年代建立的一个无线电网络，用于在夏威夷群岛之间传输数据。ALOHAnet采用了一种"先发后听"的策略：终端想发就发，如果发现冲突了就等一会儿再重试。这个看似简单粗暴的思路，后来成了以太网CSMA/CD的灵感来源。

这里我想展开讲讲ALOHAnet，因为它的设计哲学对以太网的影响实在太大。ALOHAnet的设计者Norman Abramson面临的问题是：夏威夷群岛之间铺设电缆的成本太高，只能用无线电通信。但在无线电环境中，多个终端同时发送必然产生碰撞。Abramson的解决方案非常大胆：不去避免碰撞，而是允许碰撞发生，然后让发送方自己检测并重试。这个"乱来"的方案被当时很多通信专家批评为"不靠谱"，但Abramson用数学证明了一个反直觉的结论：在这种"随机接入"模式下，信道利用率理论上可以达到18%——虽然不算高，但考虑到这是在根本不协调发送方的情况下做到的，已经是非常了不起的成绩。

1973年春天，正式入职PARC的Metcalfe面临一个实际问题：PARC已经拥有了世界上第一批个人计算机——Xerox Alto（是的，在IBM PC出现之前好几年）。这些Alto计算机需要互相通信，还需要共享当时极其昂贵的激光打印机。Metcalfe需要为这些设备设计一个局域网。他敏锐地意识到，ALOHAnet的随机接入思想可以改进后应用于有线网络——在有线环境中，载波侦听（先听听线上有没有信号再发）可以大幅提高效率。

1973年5月22日，Metcalfe写了一份改变计算机历史的备忘录。这份备忘录的标题是：

Ethernet: A Local Area Network
-- Data Link Layer and
   Physical Layer Specifications

by Robert M. Metcalfe
Xerox Palo Alto Research Center
May 22, 1973

趣闻：这份宝贵的原始备忘录至今仍然保存在Palo Alto的档案中。如果你有机会去计算机历史博物馆（Computer History Museum），说不定能见到它的复印件——上面还有Metcalfe的手写批注，非常有意思。另外，你可能注意到了"Ethernet"这个拼写——在最早的备忘录中，Metcalfe写的是"Ethernet"，后来才变为标准的"Ethernet"。

有趣的是，Metcalfe的博士论文最初被哈佛否决了——评审委员会认为"缺乏理论深度"。如果当时哈佛通过了，以太网可能就不会诞生了（因为Metcalfe可能就离开PARC去别处找工作了）。后来Metcalfe在加入PARC后继续完善了论文，这才从哈佛拿到了博士学位。这个故事告诉我们：有时候，学术界的"不通过"恰恰是创新的起点。

还有一个小细节：Metcalfe在PARC的同事David Boggs也是以太网的重要贡献者。实际上，Boggs实现了以太网的第一个硬件接口——一块连接Alto计算机和同轴电缆的电路板。两人在1976年共同发表了论文《Ethernet: Distributed Packet Switching for Local Computer Networks》，这篇论文后来成了计算机通信领域被引用最多的文献之一。

1.4 第一张以太网草图

Metcalfe最初设计的以太网是什么样的？让我们来看看技术细节。

第一张以太网的网络拓扑是总线型（Bus Topology）——所有的计算机都连接到一根共享的同轴电缆上。这根电缆就是"以太"的物理化身。数据传输速率为2.94 Mbps——这个奇怪的数字来源于Xerox Alto计算机的系统时钟频率。Alto的CPU时钟约为5.88 MHz，Metcalfe取了一半作为网络接口的时钟，于是就得到了2.94 Mbps这个"奇葩"速率。

它的结构可以用下面的ASCII示意图来表示：

+----------+  +----------+  +----------+  +----------+
| Station  |  | Station  |  | Station  |  | File     |
| A (Alto) |  | B (Alto) |  | C (Alto) |  | Server   |
+----+-----+  +----+-----+  +----+-----+  +----+-----+
     |             |             |             |
=====+=============+=============+=============+=====  Coaxial Cable
     |             |             |             |          (共享介质)
     |             |             |             |
  Terminator(50Ω)                         Terminator(50Ω)

这根同轴电缆就是所有设备共享的通信介质。任何一台计算机要发送数据，就把信号放到电缆上，所有其他计算机都能收到——这就是"广播"的本质。接收方检查数据帧中的目的地址：如果是给自己的，就收下；如果不是，就丢弃。发送前先侦听电缆上是否有信号（载波侦听），空闲时才发送；发送过程中持续监听，一旦发现冲突就立即停止，并等待一段随机时间后重试——这就是CSMA/CD（载波侦听多路访问/碰撞检测）的基本原理。关于CSMA/CD的详细机制，我们会在第6章专门展开。

那么"以太"这个名字是怎么来的？Metcalfe取这个名字是为了致敬19世纪物理学中假想的"光以太（Luminiferous Ether）"——当时物理学家认为电磁波是在一种看不见的介质"以太"中传播的（直到迈克尔逊-莫雷实验否定了这一假说）。Metcalfe借用这个比喻，把同轴电缆比喻成了一种"电磁以太"，数据信号就像光波一样在这个"以太"中传播。名字取得这么有诗意，难怪能流传五十年。实际上，据说Metcalfe最初想用的名字是"Alto Aloha Network"（Alto版的ALOHAnet），但后来觉得"Ethernet"更酷——不得不说，这是一个非常正确的审美判断。

1.5 DEC-Intel-Xerox联盟与开放标准

到了1979年，以太网面临一个关键转折点。Xerox的高层终于意识到以太网的商业价值，但光靠施乐一家公司是不够的——任何网络技术的价值都取决于它的网络效应：用的人越多，它就越有价值。如果以太网变成施乐的专有技术，其他公司要么不愿意采用（担心被供应商锁定），要么需要支付高额授权费——这两种情况都会限制以太网的普及。

于是，一个历史性的联盟成立了：DIX——DEC（数字设备公司）、Intel（英特尔）、Xerox（施乐）三家巨头联手，共同推动以太网的标准化。这三家公司的分工非常巧妙：DEC有丰富的网络系统经验，Intel有芯片制造能力，Xerox拥有核心专利。更重要的是，这三家公司的市场覆盖互补，几乎涵盖了计算机行业的各个角落。

1980年，DIX联盟发布了一份名为《The Ethernet: A Local Area Network》的规范，这就是著名的"蓝皮书"（Blue Book），也被称为Ethernet 1.0。它将速率从2.94 Mbps提升到了10 Mbps，并详细定义了物理层和数据链路层的规范。1982年，DIX发布了修订版Ethernet 2.0（也称为Ethernet II），这个版本的帧格式一直沿用至今——是的，你在Wireshark里抓到的大多数数据包，使用的还是四十多年前定义的帧格式。

1983年，IEEE 802.3工作组正式将以太网采纳为国际标准。虽然IEEE 802.3和Ethernet II在帧格式上有一个微小的差异（EtherType字段 vs. 长度字段），但这并不影响以太网的普及。实际上，IEEE后来在802.3x-1997修订中允许了两种格式共存，通过一个巧妙的方式区分：如果这个字段的值大于1536（0x0600），就解释为EtherType；否则解释为长度。这个细节我们会在第4章帧格式解析中详细讨论。

关键历史节点：如果Xerox当时选择闭门造车，把以太网作为独家技术垄断起来，它很可能像很多PARC的其他伟大发明一样（比如GUI和鼠标），最终被历史遗忘。DIX联盟选择了一条截然不同的道路——开放标准。任何公司都可以以极低的费用（最初是"一次性支付1美元"）获得以太网规范的授权。这个决策，是以太网最终战胜Token Ring等竞争对手的决定性因素。

说到Token Ring，我想起一个亲身经历的故事。2010年我刚入行时，公司的一位老工程师给我看他曾经维护过的一个Token Ring网络——一个大型金融机构的交易系统。IBM的Token Ring技术在当时确实很优秀：确定性的延迟、没有碰撞、16 Mbps的速率在1980年代非常可观。但这个网络的维护成本高得惊人：专用的IBM MAU（类似于集线器）、价格数倍于以太网网卡的Token Ring适配器、还需要专业的IBM工程师来配置。当以太网交换机的价格在1990年代急剧下降、100 Mbps Fast Ethernet问世时，这家金融机构毫不犹豫地抛弃了Token Ring，全面转向了以太网。那位老工程师跟我说了一句话，我至今记忆犹新："技术上的优势，抵不过生态上的劣势。"

1.6 以太网的哲学

以太网为什么能笑到最后？我认为有四个核心原因：

第一，开放。DIX联盟从一开始就选择将以太网作为开放标准发布，任何人只需要花1美元（实话说，这个费用更多是象征性的）就能获得规范。相比之下，Token Ring被IBM牢牢控制，FDDI虽由ANSI标准但实现成本高昂，ARCnet则是Datapoint公司的专有技术。开放带来了竞争，竞争带来了降价，降价带来了普及——这是一个完美的正向循环。到1990年代中期，一块以太网网卡的价格已经跌到了20美元以下，而Token Ring网卡还在200美元以上。价格的差距最终决定了市场的选择。

第二，简单。以太网的设计哲学是"足够简单，刚好能用"。CSMA/CD协议的核心逻辑可以用几句话讲清楚：先听后说（载波侦听），边说边听（碰撞检测），冲突了就等一会儿再说（二进制指数退避）。而Token Ring的令牌管理和环维护逻辑要复杂得多——活动监视器（Active Monitor）选举、邻居发现、环恢复等一系列协议，运行起来就像一台精密的瑞士钟表。精密的代价就是昂贵和脆弱。简单的设计意味着更便宜的硬件、更容易的调试、更低的入门门槛。

第三，持续进化。以太网不是一成不变的。它从2.94 Mbps走到10 Mbps，再到100 Mbps（Fast Ethernet）、1 Gbps（Gigabit Ethernet）、10 Gbps、40 Gbps、100 Gbps、400 Gbps，一直到今天的800 Gbps甚至1.6 Tbps。每次升级都保持了向后兼容——你的千兆网卡插到百兆交换机上仍然能用，虽然速率会协商到100 Mbps。这种"不破不立"的进化能力，是以太网四十多年不衰的关键。Token Ring则没有这种运气：IBM试图将Token Ring从16 Mbps升级到100 Mbps时遇到了巨大的技术困难（HSTR：High-Speed Token Ring），等到终于搞定的时候，市场早已被Fast Ethernet占领。

第四，"rough consensus and running code"。这是互联网工程任务组（IETF）的座右铭，但同样适用于以太网。当Token Ring的支持者还在争论"CSMA/CD在理论上不如令牌传递优雅"的时候，以太网的工程师们已经让成百上千台计算机在同一个以太网上跑起来了。"能跑"压倒"好看"——这是工程领域永恒的真理。以太网的故事告诉我们：在真实世界中，一个能用的、便宜的、扩展性好的方案，永远比一个"理论上完美"但昂贵复杂的方案更有生命力。

一个常见的误解：很多初学者会把"以太网"和"互联网"混为一谈。其实以太网是链路层的技术，负责在同一个局域网内传输数据帧；而互联网是网络层的互联体系，通过IP协议把不同的网络连接起来。简单来说，以太网是你的"本地道路"，互联网是连接各城市的"高速公路网"。没有以太网这样的局域网技术，互联网也就失去了连接的"最后一公里"。

回顾以太网五十多年的历史，它的故事远非一个技术标准的更替那么简单。它告诉我们：在技术领域，赢得市场的往往不是最初最先进、最优雅的方案，而是那个最开放、最简单、最能适应变化的方案。如果你正在做一个技术选择，不妨问问自己："这个方案十年后还能活得好吗？"以太网的答案是：能。
