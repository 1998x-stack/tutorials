第17章 以太网供电（PoE）

想象这样一个场景：你是一个企业的IT运维工程师，要为办公区部署50台IP电话和30个无线AP。按照常规思路，你需要做两件事——拉网线传数据、拉电源线供电。但问题来了：IP电话和AP通常安装在吊顶天花板上或墙壁高处，附近根本没有电源插座。为每一个设备单独拉一根220V电源线？成本高、布线丑、电工还要和弱电施工协调工期。

PoE（Power over Ethernet，以太网供电）的出现彻底解决了这个问题。一根标准的Cat5e/Cat6网线，同时传输数据和电力——下行设备不需要单独的电源适配器，不需要附近有电源插座，只需要一根网线连接到PoE交换机。

这听起来像"黑魔法"——数据和电流在同一条铜线上并行传输，互相不干扰。但它的原理其实非常朴素。这一章我们就来深入探讨PoE的工作原理、标准演进、供电协商过程，以及在实际部署中你会遇到的那些"坑"。

17.1 PoE的基本原理

以太网使用的双绞线（Twisted Pair）中有4对线。在10BASE-T和100BASE-TX中，实际数据传输只使用了其中的2对（1-2和3-6），另外2对（4-5和7-8）是空闲的。PoE最早的思路就是在空闲线对上传输直流电——这简单直接，互不干扰。

到了1000BASE-T（千兆以太网），情况发生了变化——千兆以太网需要全部4对线来传输数据。但PoE并没有因此被淘汰，因为IEEE想出了一个高明的方案：幻象供电（Phantom Powering）。它利用的是双绞线的"共模"特性——在一对线的两根导线之间传送的是差分数据信号，在两根导线同时施加相对于地的相同电压（共模电压），数据信号不受影响。也就是说，数据和直流电在同一对线上"叠加"传输。

具体来说，PSE（Power Sourcing Equipment，供电设备）通过两种方式向PD（Powered Device，受电设备）供电：

Alternative A（模式A）：通过数据线对（1-2和3-6）传输电力，使用幻象供电方式。直流电流在每对线的两根导线中以共模方式流动——两根导线之间的电压差（即数据信号）保持不变。

Alternative B（模式B）：通过空闲线对（4-5和7-8）传输电力，不涉及数据线对。这种方式只适用于不需要使用全部4对线传输数据的场景（如百兆以太网）。

一个容易混淆的点：很多人以为PoE只能在以太网上使用，其实PoE的核心标准（IEEE 802.3af/at/bt）定义的是通过以太网线缆供电的方法，它依赖的是线缆的物理特性，而不是以太网协议本身。只是在实际应用中，PoE几乎总是和以太网设备配合使用，所以"PoE = 以太网设备无需单独供电"这个印象深入人心。

17.2 供电过程：四步协商

PoE供电不是简单粗暴地"直接送电"。在PSE向PD送电之前，必须经过一个四步协商过程，确保安全：

第一步：检测（Detection）。PSE在端口上施加一个2.8V到10V之间的低压探测信号（电流限制在5mA以内）。如果检测到终端呈现一个特定范围内的电阻值（典型的特征电阻是19kΩ到26.5kΩ，实际标准中要求PoE PD在25kΩ附近），PSE就确认连接的是一个合法的PoE PD设备。如果检测到的电阻值超出范围（比如短路、开路、或接入了不支持PoE的非PD设备），PSE就不会继续供电——这是为了防止对非PoE设备造成损坏。

第二步：分级（Classification）。确认是合法的PD后，PSE升高电压到15-20V，观察PD的电流消耗来确定PD需要的功率等级。PD通过拉取不同的电流值来"告诉"PSE自己属于哪个class。标准802.3af定义了5个class（0-4），其中Class 0是默认值（表示PD需要15.4W以内的最大功率）。如果PSE没有足够功率分配给某个PD，它可以拒绝供电。

第三步：供电（Power On）。PSE将输出电压切换到44V-57V之间的工作电压，开始为PD提供直流电力。在整个供电过程中，PSE持续监控电流和电压：如果检测到电流过小（低于5mA且持续超过300ms），PSE认为PD已物理断开连接，便切断供电并回到第一步（检测状态）。如果检测到过流或短路，PSE也会立即切断供电。

第四步：断电（Disconnect）。当PD被拔下或关机时，PSE检测到线路电流下降到低于维持电流阈值（通常5-10mA）并持续一段时间，PSE就会切断电源并重新开始检测。

PoE供电协商时序：

        PSE端口电压
        │
        │  57V ───────────────────────────────────┐
        │        ┌─────────── 工作电压范围 ────────┤
        │  44V ──┴──────────────────────────────────┤
        │                     ┌──── 供电 ───────────┤
        │  20V ──────────────┤                     │
        │                     └─ 分级 ─────────────┘
        │  10V ──────────────────────┐
        │            ┌─── 检测 ──────┤
        │  2.8V ─────┘               │
        │  0V ───────────────────────┴─────────────────────→ 时间
        │
        └─── 检测 → 分级 → 供电 → (持续监控) → 断电

17.3 PoE标准四代演进

PoE技术自2003年标准化以来经历了四代演进，每代的供电能力都有显著提升：

标准 规范名称 发布时间 PSE每端口功率 PD可用功率 线对使用 典型应用
802.3af PoE 2003 15.4 W 12.95 W 2对 IP电话、低端AP
802.3at PoE+ 2009 30 W 25.5 W 2对 Wi-Fi AP、PTZ摄像头
802.3bt Type 3 PoE++ / 4PPoE 2018 60 W 51 W 4对 LED照明、数字标牌
802.3bt Type 4 PoE++ / 4PPoE 2018 90 W 71.3 W 4对 大屏一体机、笔记本电脑

注意到一个细节了吗？PSE每端口的功率和PD实际能用的功率之间存在差值——这个差值就是在线缆上损耗的功率。以802.3af为例，PSE输出15.4W，线缆损耗约2.45W（取决于线缆长度和质量），PD实际可用12.95W。这个损耗在100米的标准网线长度下，约合0.5-1W每10米的压降。

从802.3bt开始，标准引入了4对线同时供电（4PPoE）——所有4对双绞线都用来供电，每对线上的电流只有原来的一半，线损显著降低。这是实现60W和90W高功率供电的关键技术。

17.4 LLDP电源管理与智能协商

基础的分级（Classification）只能告诉PSE设备的功率等级（class 0-8），但这个等级是粗粒度的。802.3at和802.3bt引入了基于LLDP（Link Layer Discovery Protocol，链路层发现协议）的精细电源管理机制。

通过LLDP的电力管理TLV（Type-Length-Value），PD可以精确地告诉PSE它当前实际需要多少功率（精确到0.1W），而不是仅仅报告一个class等级。PSE可以根据总功率预算动态调整每个端口的供电策略：如果某个PD不需要那么多功率，PSE可以把余量分配给其他PD。

LLDP电力管理TLV的典型交互过程：

1. PSE发送LLDP报文，宣告自己的PoE能力（支持的功率级别、供电模式等）
2. PD回复LLDP报文，包含所需功率（如 "我需要 18.5W"）
3. PSE检查功率预算：
   - 如果总功率足够：确认供电
   - 如果总功率不足：可以发送"拒绝"或"降级"建议
4. PD可以根据PSE的回复调整功耗（比如关闭USB端口以降低功耗）
5. 当PD的功耗需求动态变化时（如AP的射频功率调节），
   通过LLDP重新协商

LLDP的实用价值：在部署高密度的PoE网络时，LLDP电力管理可以让交换机在总功率有限的情况下实现"削峰填谷"。例如一台PoE交换机总功率预算为740W，下挂24个802.11ax AP。每个AP的峰值功耗为25W，24个同时峰值需要600W——足够了。但如果在夜间所有AP切换到最大射频功率做背景扫描，功耗可能上升。通过LLDP，交换机可以实时掌握每个AP的当前功耗，合理分配预算，避免过载导致整机断电保护。

17.5 PoE的典型应用场景

PoE的应用早已超越了最初的IP电话。今天，它几乎无处不在：

IP电话（VoIP Phone）——PoE最经典的应用。一部IP电话通常只需要5-8W功率，802.3af完全足够。电话还内置了"直通端口"，让PC可以串接在电话后面，共用一根网线。

无线AP（Access Point）。Wi-Fi 4/5 AP通常需要10-15W，Wi-Fi 6 AP需要20-30W（因为更多的射频链和MU-MIMO），802.3at是基本要求。Wi-Fi 6E/Wi-Fi 7 AP由于增加了6GHz频段，功耗可能达到40W以上，需要802.3bt支持。

安防摄像头。固定摄像头通常8-12W即可，但PTZ（云台变焦）摄像头需要驱动电机，功耗可达20-30W。夜间开启红外LED补光时，功耗会额外增加5-10W。

新兴应用。PoE LED照明（每个灯具15-30W）、数字标牌（40-60W）、POS收银终端（10-20W）、甚至通过PoE供电的瘦客户机/笔记本电脑（802.3bt Type 4达到90W后，小型笔记本电脑的充电问题也解决了）。

17.6 一个真实的"踩坑"故事：夜间重启之谜

2019年，我为一个园区部署了约200个海康威视的PoE安防摄像头。交换机使用的是支持802.3at的PoE+交换机，每端口30W。摄像头的标称功耗是12W（红外关闭）到22W（红外开启），按理说完全在预算之内。

部署后的第一个月一切正常。但从第二个月开始，监控中心频繁报出"摄像头离线"告警，时间集中在晚上7点到11点之间——摄像头每隔30-60分钟就会掉线一次，1-2分钟后自动恢复。更诡异的是，这个问题只在冬天出现，夏天没有。

排查过程：

第一步怀疑是摄像头固件问题——升级固件后问题依旧。第二步怀疑是交换机PoE供电模块故障——更换交换机后问题依旧。第三步测量电压——在摄像头端的网线接口测量，发现供电电压只有39V，而PSE的输出端是52V。一台摄像头红外开启时电流约为0.5A，100米网线的总电阻约为10-12Ω（取决于线缆质量和温度），压降高达5-6V。到了冬天，铜缆的电阻随温度降低而减小——看起来应该是好事，但问题是园区冬天开暖气，吊顶内温度偏高，线缆温度反而升高，电阻增加，压降更大。

最终我们定位到的根因是：网线质量不达标。施工方使用的是Cat5e线缆，但材质不是纯铜（CCA——铜包铝，Copper-Clad Aluminum），电阻比标准纯铜网线高出约40%。加上部分摄像头的布线距离接近100米极限，红外开启时电流增大，综合压降导致摄像头端的电压跌到了36V以下——摄像头的PoE受电模块工作在欠压保护状态，自动关机。关机后摄像头功耗归零，电压恢复，摄像头重新上电启动——然后重复这一循环。

解决方案是：将布线距离超过70米的摄像头全部更换为纯铜Cat6网线，并在部分关键位置使用了PoE中继器（PoE Extender）。这之后故障彻底消失。

血泪教训：PoE的"100米距离限制"不是绝对的——在实际工程中，超过70米就很可能出现供电不稳定的问题，尤其是在使用CCA线缆时。务必使用纯铜（BCM——Bare Copper）线缆，Cat6或以上。省钱在网线上，最后花的钱一定更多。
