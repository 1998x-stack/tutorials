第12章 定位技术

大家好，我是你们的讲师。假设你在一个大型购物中心里丢了孩子——"他在哪？"这个问题比"他还好吗"更紧迫。在WSN中也是一样：传感器节点采集到了"温度32°C、湿度65%"，但如果不知道这个数据是在哪个位置采集的，这组数据的信息价值就大打折扣。在大多数WSN应用中，"感知数据 + 位置信息 = 有价值的信息"。

上一章我们解决了"什么时间"的问题（时间同步），本章要回答"什么地方"的问题——定位技术。与GPS不同，WSN定位往往没有昂贵的卫星接收器、没有充足的电力、没有清晰的天空视野。WSN的定位需要在资源受限的条件下，通过无线信号的特性和节点间的协作来估计位置。

本章核心问题： WSN中定位为什么这么难？为什么RSSI看起来很美但在实际部署中经常"翻车"？测距定位和免测距定位各自的优劣势是什么？学完本章，你将掌握WSN定位技术的完整方法论。

WSN定位问题定义

WSN定位问题的标准设定是这样的：网络中有两类节点——锚节点（Anchor/R-B节点）和未知节点（Unknown Node）。锚节点通过GPS或人工部署知道自己的精确坐标，未知节点则不知道自己在哪里。定位的目标就是：利用锚节点的已知位置信息和节点间的通信测量，估算所有未知节点的坐标。

从技术路线上，WSN定位分为两大类：

  • 基于测距（Range-based）：先测量节点间的距离或角度，再用几何方法计算位置
  • 免测距（Range-free）：不直接测量距离，通过网络的连通性等拓扑信息推断位置

前者的精度高但需要额外硬件或复杂的信号处理，后者成本低但精度有限。这是WSN定位中最基本的权衡。

基于RSSI的测距定位

RSSI（Received Signal Strength Indicator）测距是最"廉价"的测距方法——几乎所有无线收发芯片都提供RSSI读数，不需要任何额外硬件。它的原理很简单：信号强度随距离衰减，测出接收信号强度就能反推出距离。

常用的路径损耗模型：

PL(d) = PL(d₀) + 10 × n × log₁₀(d/d₀) + Xσ

其中PL(d)是距离d处的路径损耗（dB），n是路径损耗指数（自由空间=2，室内视距≈1.6-1.8，室内非视距≈4-6），Xσ是零均值高斯随机变量（代表阴影衰落），d₀是参考距离（通常取1m）。

测出三个或以上锚节点到未知节点的距离后，使用三边测量（Trilateration）计算未知节点的坐标：

// 最小二乘法三边测量求解
// 已知：三个锚节点坐标 (x1,y1), (x2,y2), (x3,y3)
// 已知：未知节点到三个锚节点的距离估计 d1, d2, d3
// 求：未知节点坐标 (x, y)

// 构建线性方程组（将第一个方程减去后面的方程，消去二次项）
// 对于 i = 2, 3:
//   (xi - x1)*x + (yi - y1)*y = 0.5*(d1^2 - di^2 + xi^2 + yi^2 - x1^2 - y1^2)

function trilateration(anchors, distances):
    // anchors = [{x, y}, ...]  至少3个锚节点
    // distances = [d1, d2, d3, ...]

    n = len(anchors)
    // 构造最小二乘矩阵 A 和 b
    A = matrix(n-1, 2)
    b = vector(n-1)

    for i = 1 to n-1:
        A[i][0] = anchors[i].x - anchors[0].x
        A[i][1] = anchors[i].y - anchors[0].y
        b[i] = 0.5 * (
            distances[0]^2 - distances[i]^2
            + anchors[i].x^2 + anchors[i].y^2
            - anchors[0].x^2 - anchors[0].y^2
        )

    // 最小二乘解: x = (A^T * A)^{-1} * A^T * b
    A_t = transpose(A)
    x = inverse(A_t * A) * A_t * b
    return (x[0], x[1])  // 未知节点的估计坐标
RSSI的最大敌人——多径衰落： RSSI测距在实验室环境中精度可以达到2-3米，但在真实部署中，由于多径衰落、阴影效应、天线方向性等因素，RSSI的波动可以轻松达到10-15dB，对应的距离误差可能超过10米。在一个有金属货架、电机运转的仓库中，RSSI测距的可靠性可能让你怀疑人生。后面我会分享一个具体的案例。

TOA/TDOA测距

TOA（Time of Arrival）：通过测量信号从发送方到接收方的传播时间来计算距离。距离 = 传播时间 × 信号速度。TOA需要精确测量信号的出发时间和到达时间——这意味着发送方和接收方必须精确同步（通常需要纳秒级的同步精度才能达到米级定位精度）。

TDOA（Time Difference of Arrival）：不要求发送方和接收方同步，而是测量同一个信号到达不同接收节点的时间差。通过三个以上锚节点测量的TDOA值，可以构建双曲线方程组求解位置。TDOA在GPS和蜂窝网络定位中广泛使用。

TOA/TDOA的精度远高于RSSI（可以达到厘米级），但代价是对硬件和同步的要求极高。在WSN中，TOA/TDOA通常需要超声波或UWB（超宽带）等特殊硬件的支持。

AOA定位

AOA（Angle of Arrival）通过天线阵列测量信号到达的角度来确定方向。如果两个锚节点都能测到未知节点信号的到达角，两条方向线的交点就是未知节点的位置。

AOA定位的优势是：只需要2个锚节点就能确定二维位置（而三边测量需要3个）。但它的硬件要求高——需要天线阵列或多路接收通道，而且角度测量精度受多径效应影响显著。在WSN中，由于节点体积和成本的限制，AOA的应用相对较少。不过，随着蓝牙5.1和BLE AoA/AoD技术的推广，AOA在室内定位中正在获得越来越多的关注。

免测距定位

免测距定位不直接测量距离或角度，而是利用网络的连通性信息来估计位置。虽然精度不如测距方法，但不需要额外硬件，适合大规模、低成本部署的WSN。

DV-Hop

DV-Hop的算法分为三步：

第一步：每个锚节点广播一个消息（包含自己的坐标和跳数计数器，初始为0），消息在网络中泛洪扩散。每个节点维护到每个锚节点的最小跳数。

第二步：每个锚节点计算"每跳平均距离"，等于该锚到其他锚的欧氏距离之和除以跳数之和。然后广播这个校正值。

第三步：未知节点收到校正值后，用"每跳平均距离 × 到锚节点的跳数"估算到各锚节点的距离，然后用三边测量法计算位置。

DV-Hop的精度通常在33%的通信半径左右，在均匀部署的网络中表现尚可，但在节点分布不均匀时误差显著增大。

APIT（Approximate Point-in-Triangulation Test）

APIT的原理是：未知节点判断自己是否在三个锚节点组成的三角形内部，通过多个三角形的包含关系将可能的位置范围缩小到某个区域。APIT使用一种称为"PIT测试"的近似方法（通过模拟移动邻居节点来判断节点是否在三角形内部），不需要实际的节点移动。

APIT的优点是实现简单、对锚节点数量不敏感，但PIT测试在节点密度较低时准确率下降，容易出现"误判"。

前沿：蓝牙6.0 Channel Sounding

2024年9月，蓝牙特别兴趣小组（SIG）发布了蓝牙6.0核心规范，其中一个最引人注目的新功能就是Channel Sounding——一种基于相位测距（Phase-based Ranging）的高精度距离测量技术。

Channel Sounding的原理是：在两个蓝牙设备之间通过多个信道上交换信号，测量信号在多个频率上的相位差，从而精确估计信号传播时间并换算出距离。由于相位测量比RSSI对多径效应更不敏感，Channel Sounding在理想条件下可以实现约10cm的测距精度——这比传统的RSSI蓝牙定位（3-10米）提高了整整一个数量级。

对于WSN来说，这意味着未来的传感器节点可能只需要集成一颗普通的蓝牙芯片（功耗极低），就能实现接近UWB的定位精度。苹果的"Find My"网络和Google的"Find My Device"网络都已经在转向类似的技术方向。可以预见，蓝牙6.0 Channel Sounding将在未来几年内显著改变WSN定位的性价比格局。

一个真实的"踩坑"故事

2019年，我为某大型仓储物流中心设计了一套基于RSSI的叉车定位系统。仓库面积约2万平方米，货架高度达到12米，我们在仓库顶部部署了80个蓝牙信标（锚节点），在每辆叉车上安装了一个接收器。按照实验室测试，RSSI三边测量的定位精度应该能达到3-5米，足够判断叉车在哪个巷道。

然而实际部署后，定位误差大到令人崩溃——最大误差超过30米，叉车的轨迹显示它不时"穿墙"甚至"飞"到仓库外面去了。经过两周的排查，罪魁祸首找到了：金属货架对2.4GHz信号的多径反射造成了严重的RSSI波动。叉车经过不同巷道时，由于金属货架的角度和距离不同，RSSI读数可以相差20dB以上——即使叉车停在同一个位置不动，RSSI的读数也会因为周边反射体的小变化而剧烈抖动。

最终的解决方案是"三管齐下"：第一，我们在离线阶段采集了整个仓库的RSSI指纹地图（fingerprinting），把定位模式从三边测量改为指纹匹配；第二，引入了卡尔曼滤波对位置估计做平滑处理，限制位置变化的物理合理性（叉车不可能每秒钟移动30米）；第三，在叉车上加装了低成本的惯性测量单元（IMU），在RSSI信号不可靠时用航位推算做补充。整改后，定位精度稳定在5米以内，满足了"知道叉车在哪个巷道"的业务需求。

经验总结： RSSI指纹地图的精度取决于参考点的密度和采集质量。在仓库场景中，我们每2米采集一个参考点，每个点采集30秒的RSSI均值。虽然部署阶段的工作量很大，但这几乎是唯一能让RSSI定位在复杂室内环境中"可用"的方法。

好了，本章我们学习了WSN定位的各种技术——从廉价的RSSI到高精度的UWB TDOA，从免测距的DV-Hop到前沿的蓝牙Channel Sounding。定位让每个数据有了"空间坐标"，但面对海量的、可能高度冗余的传感器数据，我们还需要一个关键技术来"化繁为简"——那就是下一章要讲的数据融合。数据融合技术将告诉你如何从多个不完美的传感器读数中提炼出最准确的信息，同时大幅减少需要传输的数据量。
