开篇：从"有没有信号"到"做出决策"

大家好，我是你们的讲师。前25章我们花了大量篇幅讨论如何描述随机信号、如何估计它的参数、如何从噪声中滤出有用信号。但有一个根本问题我们还没正面回答：你凭什么说"这里有信号"？

想象你是一名雷达操作员，盯着屏幕上闪烁的光点。屏幕上有一个微弱的回波——它是一架敌方飞行器，还是只是一群飞鸟？你的判断决定了是否要发射导弹。这就是信号检测问题的核心：在不确定性中做出二元决策。本章将系统介绍假设检验的数学框架、Neyman-Pearson准则、匹配滤波器以及ROC曲线，这些工具构成了几乎所有现代检测系统——从雷达、声纳到通信接收机、医学诊断——的理论基石。

检测问题的数学表述

二元假设框架

信号检测最基本的形式是二元假设检验。我们有两个互斥的假设：

  • H₀（零假设）：观测信号中只有噪声，没有目标信号。例如雷达接收到的纯噪声。
  • H₁（备择假设）：观测信号中包含目标信号加噪声。例如雷达接收到目标回波加噪声。

数学上表述为：

H₀: Y = W          （只有噪声）
H₁: Y = S + W      （信号 + 噪声）

其中 Y 是我们观测到的数据，S 是未知的有用信号，W 是随机噪声。我们的任务是基于观测 Y，选择一个决策 D₀（判定 H₀ 成立）或 D₁（判定 H₁ 成立）。

两类错误：虚警与漏检

任何检测系统都无法完美无缺。我们要面对两类错误：

实际 H₀ 为真实际 H₁ 为真
判定 D₀✅ 正确拒绝❌ 漏检 (Miss)
判定 D₁❌ 虚警 (False Alarm)✅ 正确检测

定义两个关键概率指标：

  • 虚警概率 Pfa = P(D₁ | H₀)——"本来没信号，你非说它有"。在雷达中，这意味着把一只鸟当成了敌机。
  • 漏检概率 Pm = P(D₀ | H₁)——"信号明明在，你却说没有"。在雷达中，这意味着真的敌机飞过而你毫无察觉。

显然我们希望两个错误都越小越好。但它们是跷跷板的两端——降低虚警率往往会增加漏检率，反之亦然。如何找到最佳平衡点？这就是Neyman-Pearson准则要解决的核心问题。

Neyman-Pearson准则

固定虚警，最大化检测概率

Neyman-Pearson（以下简称NP）准则的思想极其务实：给定一个可容忍的虚警概率上限 α，设计一个检测器使得检测概率 Pd = 1 - Pm 最大化。

为什么是固定虚警而不是固定漏检？因为在大多数工程场景中，虚警的代价更容易评估。例如雷达操作员可以容忍每100次扫描中误报1次目标（Pfa = 0.01），但无法容忍漏掉真实目标。所以虚警率通常是系统设计时确定的"硬约束"，在这个约束下我们尽可能多地捕获真实目标。

似然比检验

NP准则给出一个极其优雅的检测器结构——似然比检验：

Λ(Y) = P(Y | H₁) / P(Y | H₀)

如果 Λ(Y) > γ，则判 H₁（有信号）
如果 Λ(Y) ≤ γ，则判 H₀（无信号）

其中 Λ(Y) 是似然比——在两种假设下观测到 Y 的概率密度之比。γ 是检测阈值，由目标虚警概率 Pfa 决定。具体地，阈值 γ 满足：

Pfa = ∫{Y: Λ(Y) > γ} P(Y | H₀) dY = α

NP定理告诉我们：在所有可能的检测器中，似然比检验是在给定虚警率下检测概率最高的——没有比它更好的了。这是一个强有力的结论：你不需要发明新的检测结构，只需要计算似然比、选好阈值。

直观理解：似然比 Λ(Y) 就是"数据 Y 在信号存在下的可能性"除以"数据 Y 在信号不存在下的可能性"。如果这个比值很大，说明数据更支持"信号存在"的假设，我们就判 H₁。就像在法庭上，控方证据（似然比）超过了"排除合理怀疑"的门槛（阈值），才能定罪。

如何确定阈值

阈值 γ 由虚警概率 Pfa 确定，通常需要知道在 H₀ 下 Λ(Y) 的概率分布。在 AWGN 这类简单场景中，Λ(Y) 是观测 Y 的单调函数（例如匹配滤波器的输出），阈值可以解析表达。在更复杂的模型中，往往需要通过蒙特卡洛仿真来确定阈值。

AWGN中的信号检测

已知信号：匹配滤波器

当噪声是加性高斯白噪声（AWGN）、信号波形完全已知时，最优检测器就是匹配滤波器：

h(t) = s(T - t)

匹配滤波器的冲激响应是信号 s(t) 的时间反转。为什么这个结构最优？因为它最大化输出信噪比。具体来说，在采样时刻 t = T，匹配滤波器的输出信噪比达到最大值：

SNRmax = 2E / N₀

其中 E 是信号能量，N₀ 是噪声功率谱密度。这个结果与信号的具体形状无关——无论你用正弦波还是方波，只要能量相同，匹配滤波器能达到的信噪比上限就是一样的。

在 NP 框架下，匹配滤波器的输出与阈值比较：如果输出超过阈值，判 H₁；否则判 H₀。由于 AWGN 是高斯分布，匹配滤波器输出也是高斯随机变量，检测概率和虚警概率都有闭式表达式：

  • Pfa = Q(γ / σ)，其中 Q 是高斯分布的右尾函数
  • Pd = Q(γ / σ - √(2E/N₀))

未知相位信号：非相干检测

实际工程中，我们往往不知道信号的精确相位——例如通信接收机不知道载波相位，雷达不知道目标反射信号的相位。这时无法使用匹配滤波器（因为它需要信号波形完全已知），我们需要非相干检测。

标准做法是使用包络检波器：对接收信号做带通滤波后取包络（幅度），然后将包络与阈值比较。包络在 H₀ 下服从瑞利分布，在 H₁ 下服从莱斯分布。检测概率和虚警概率可以通过 Marcum Q-函数计算。

非相干检测的代价是：与相干检测（匹配滤波器）相比，大约有 1-3 dB 的性能损失——换句话说，要达到同样的检测概率，非相干检测需要多一倍左右的信号能量。这是"不知道相位"必须付出的代价。

ROC曲线

性能评估的标准工具

接收机工作特性（Receiver Operating Characteristic，简称ROC）曲线，是评价检测器性能的金标准。它描绘了检测概率 Pd 随虚警概率 Pfa 变化的曲线，参数是信噪比。

理想检测器的 ROC 曲线会紧贴左上角——在极低虚警率下达到极高检测概率。而无用的检测器（完全随机猜测）的 ROC 曲线就是对角线 Pd = Pfa。一个好的检测器位于两者之间。

AUC：一个数字衡量全局性能

ROC 曲线下的面积（Area Under the Curve，简称 AUC）是一个方便的标量指标：AUC = 1 是完美检测器，AUC = 0.5 是随机猜测。AUC 的优势在于它不依赖于具体阈值选择，反映了检测器的全局 discriminability。在医学诊断、机器学习分类等场景中，AUC 是最常用的性能评价指标之一。

检测场景信号知识最优检测器关键参数
确知信号波形完全已知匹配滤波器（相干检测）信号能量 E，噪声谱密度 N₀
随机相位波形已知，相位均匀分布包络检波器（非相干检测）SNR 损失约 1-3 dB
随机幅度波形已知，幅度衰落平方律检波器衰落深度、分集阶数
随机频率频率未知滤波器组 + 最大值选择频率搜索范围、通道数

Python实现：匹配滤波器检测与ROC曲线

import numpy as np
import matplotlib.pyplot as plt
from scipy import signal

# 参数设置
fs = 1000  # 采样率 (Hz)
T = 1.0    # 信号持续时间 (s)
t = np.linspace(0, T, int(fs*T), endpoint=False)
f0 = 50    # 正弦信号频率 (Hz)
s = np.sin(2 * np.pi * f0 * t)  # 已知信号
E = np.sum(s**2) / fs           # 信号能量

# 匹配滤波器（信号的时间反转）
h = s[::-1]

def simulate_detection(SNR_dB, N_trials=10000):
    """在给定 SNR 下仿真检测性能"""
    SNR_linear = 10**(SNR_dB/10)
    sigma = np.sqrt(E / SNR_linear)  # 噪声标准差

    # 生成 H0（纯噪声）和 H1（信号+噪声）的匹配滤波器输出
    y_H0 = np.zeros(N_trials)
    y_H1 = np.zeros(N_trials)

    for i in range(N_trials):
        noise = np.random.normal(0, sigma, len(t))
        y_H0[i] = np.abs(np.convolve(noise, h, mode='same'))[len(t)//2]

        noise2 = np.random.normal(0, sigma, len(t))
        y_H1[i] = np.abs(np.convolve(s + noise2, h, mode='same'))[len(t)//2]

    # 遍历不同阈值，生成 ROC 曲线
    thresholds = np.linspace(0, max(y_H1.max(), y_H0.max()), 200)
    Pfa = np.array([np.mean(y_H0 > th) for th in thresholds])
    Pd  = np.array([np.mean(y_H1 > th) for th in thresholds])

    return Pfa, Pd

# 计算不同 SNR 下的 ROC 曲线
SNRs = [-10, -5, 0, 5, 10]
plt.figure(figsize=(8, 6))
for SNR in SNRs:
    Pfa, Pd = simulate_detection(SNR)
    plt.plot(Pfa, Pd, label=f'SNR = {SNR} dB')

plt.plot([0, 1], [0, 1], 'k--', label='随机猜测')
plt.xscale('log')
plt.xlabel('虚警概率 Pfa')
plt.ylabel('检测概率 Pd')
plt.title('匹配滤波器 ROC 曲线')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()

踩坑故事：固定阈值在非平稳噪声下的惨痛教训

讲一个我在某通信设备公司听来的真实案例。某团队设计了一个基于匹配滤波器的信号检测模块，在实验室测试中表现完美——SNR = 0 dB 时检测概率超过 99%，虚警率低于 0.1%。团队信心满满地部署到了外场测试。

结果外场的第一天就出了大问题：系统的虚警率飙升至 30% 以上，几乎每分钟都在误报！团队百思不得其解——算法和在实验室跑的是同一套代码，没有任何改动。

经过一周的排查，问题终于水落石出：系统使用了固定的检测阈值。在实验室里，噪声是 AWGN，功率稳定，固定阈值完全够用。但外场环境的噪声是非平稳的——附近工厂的大型电机启停、路过的车辆、甚至天气变化都会导致噪声功率在几分钟内波动 ±6 dB。当噪声功率变大时，固定阈值对应的虚警率呈指数级增长。

教训：在真实工程中，噪声几乎从来不是平稳的。必须使用自适应阈值——根据实时估计的噪声功率动态调整检测门限。常见的做法包括：使用恒虚警率（CFAR，Constant False Alarm Rate）检测器，通过滑窗估计局部噪声功率来自适应调整阈值；或者用噪声功率归一化匹配滤波器的输出。永远不要假设噪声是平稳的，除非你有充分的物理理由。

这个案例揭示了一个深刻的工程真理：一个在仿真中完美的算法，放在真实环境中可能脆弱不堪。算法的鲁棒性往往不来自更精巧的数学，而来自对真实环境物理特性的敬畏。

本章小结

本章我们系统学习了信号检测与假设检验的核心框架。我们从二元假设的数学表述出发，理解了虚警和漏检两类错误的权衡关系；然后介绍了 NP 准则及其给出的最优检测器结构——似然比检验；接着深入 AWGN 环境，分别讨论了已知信号的匹配滤波器和未知相位的非相干检测；最后通过 ROC 曲线学习了检测性能的评估方法，并用一个"固定阈值"的踩坑故事警示了非平稳噪声的工程陷阱。
