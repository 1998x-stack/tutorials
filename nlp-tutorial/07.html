第七章：LSTM与GRU

大家好！上一章我们发现了RNN的阿喀琉斯之踵——梯度消失让模型记不住太久之前的信息。这一章的主角LSTM（长短期记忆网络）就是为此而生的。它用一套精巧的"门控"机制，让信息可以选择性地保留或遗忘，从而在几十甚至上百个时间步之前的信息都能回传梯度。

我第一次读LSTM论文的时候，被四个门的公式绕晕了。后来有个同事给我讲了一个比喻，瞬间通透：把LSTM想象成一个流水线工人，他面前有一条传送带（细胞状态），他手里有三个开关——什么时候把新零件放上去（输入门），什么时候把旧零件拿下来（遗忘门），什么时候把正在加工的零件展示给别人看（输出门）。记住这个比喻，后面的公式就好理解了。

7.1 门控机制的核心思想

RNN的问题是隐藏状态h既要"记忆"又要"输出"——两件事混在一起。LSTM的解决方案是引入一个独立的细胞状态C（cell state），专门负责承载长期记忆，隐藏状态h则负责输出。C像一条传送带贯穿所有时间步，信息可以在上面几乎无损耗地传播。

# LSTM 核心公式（简化理解）
# f_t = sigmoid(W_f · [h_{t-1}, x_t] + b_f)   ← 遗忘门：哪些旧信息该丢掉？
# i_t = sigmoid(W_i · [h_{t-1}, x_t] + b_i)   ← 输入门：哪些新信息该记住？
# o_t = sigmoid(W_o · [h_{t-1}, x_t] + b_o)   ← 输出门：哪些信息该展示出来？
# C̃_t  = tanh(W_c · [h_{t-1}, x_t] + b_c)     ← 候选新信息
# C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t            ← 更新细胞状态（记忆）
# h_t = o_t ⊙ tanh(C_t)                      ← 输出（筛选过的记忆）

# ⊙ 表示逐元素乘法（element-wise multiply）

注意所有门的值都在(0,1)之间（因为用了sigmoid）——它们不是"开/关"的二值开关，而是可以在0到1之间连续调节的"旋钮"。

直观理解：为什么C的梯度不容易消失？看 C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t，如果遗忘门f_t≈1、输入门i_t≈0，C_t≈C_{t-1}——梯度通过这条路径就是恒等映射，几乎不衰减。这就是LSTM能处理长序列的数学秘密。

7.2 细胞状态：LSTM的高速公路

细胞状态C是LSTM最精妙的设计。它只在上面做加法和乘法（逐元素），没有经过tanh或矩阵乘法——这意味着梯度在C上往回传时，衰减极小。

用一张表对比RNN和LSTM的信息流：

特性RNNLSTM
记忆机制h同时负责记忆和输出C负责长期记忆，h负责短期输出
信息更新h_t = tanh(W·[h_{t-1},x_t])，完全替换C_t = f_t·C_{t-1} + i_t·C̃_t，选择性更新
梯度路径每步都经过tanh导数→衰减遗忘门=1时梯度恒等传播→不衰减
有效记忆长度~20步~100-200步
参数量2个权重矩阵4个权重矩阵（每个门一个）

7.3 GRU：减法的艺术

LSTM效果好但参数多（4个门）。GRU（Gated Recurrent Unit, Cho et al. 2014）是一个更精简的设计：只保留两个门，把细胞状态C和隐藏状态h合二为一。

# GRU 核心公式（更简洁！）
# z_t = sigmoid(W_z · [h_{t-1}, x_t])   ← 更新门：保留多少旧信息？
# r_t = sigmoid(W_r · [h_{t-1}, x_t])   ← 重置门：忽略多少旧信息？
# h̃_t = tanh(W · [r_t ⊙ h_{t-1}, x_t])  ← 候选新状态
# h_t = (1-z_t) ⊙ h_{t-1} + z_t ⊙ h̃_t  ← 新老混合

# 对比：GRU的 (1-z_t) 相当于LSTM的遗忘门
# GRU用2个门实现了和LSTM类似的效果

GRU vs LSTM的选型是个经典问题：

维度LSTMGRU
门的数量3（遗忘+输入+输出）2（重置+更新）
参数量更多（~4×hidden²）更少（~3×hidden²）
训练速度较慢较快
表现大语料微优中小语料相当甚至更好
适用场景需要精细记忆控制数据较少、需要快速迭代

我的经验：不确定用哪个？先用GRU。训练快、参数少、调参容易。如果GRU效果不理想再换LSTM。我在十几个项目中总结出的规律是——数据量小时GRU有优势（不容易过拟合），数据量大时LSTM有优势（更强的表达力）。差别通常不大（<2%），所以开发效率更重要。

7.4 多层RNN与残差连接

单层LSTM已经很强，但要捕捉更抽象的语义模式，需要堆叠多层（Stacked LSTM）。浅层学语法特征（词性、短语结构），深层学语义特征（情感、意图）。实践中2-4层最常见：

# PyTorch 多层LSTM
lstm = nn.LSTM(input_size=300,
               hidden_size=256,
               num_layers=3,       # 堆叠3层
               dropout=0.3,        # 层间dropout防止过拟合
               batch_first=True,
               bidirectional=True) # 双向→每个方向3层

# 输出维度: (batch, seq_len, 256*2) → 双向拼接

⚠️ 常见坑：多层LSTM训练时经常出现loss震荡。我曾调了两天发现是学习率太大（0.001对3层LSTM太高），改成0.0001并加梯度裁剪（max_norm=5）立刻就稳了。另一个坑——忘记设置batch_first=True，在batch和seq维度之间debug了半天。

7.5 梯度流：为什么LSTM比RNN好那么多

从梯度流的角度看，RNN和LSTM的区别就像土路和高速公路：

# RNN: h_t = tanh(W·h_{t-1} + U·x_t)
# 梯度回传时每步乘 W 和 tanh' → 指数衰减

# LSTM: C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t
# 梯度回传时，C路径上只乘 f_t（sigmoid输出）
# 如果f_t≈1，梯度几乎无损传递

# 但注意：如果f_t≈0（遗忘），LSTM也会丢失信息
# 这就是为什么要用b_f初始化——让初始遗忘门偏向1（保留）
