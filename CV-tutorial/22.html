语义分割的演进

在之前的四章里，我们分别学习了分割基础、U-Net和Mask R-CNN。你可能注意到了：U-Net擅长编码-解码结构但在多尺度处理上有限，Mask R-CNN擅长实例分割但不处理背景语义。那么，有没有方法能够在像素级别对整张图像的每一个部分做精细分类，同时处理复杂的多尺度场景？这正是语义分割前沿方法所解决的课题。

语义分割的演进可以从2015年的FCN（Fully Convolutional Network）说起。FCN的核心贡献是证明了可以将分类网络（如VGG、AlexNet）最后的全连接层替换为卷积层，从而输出与输入同分辨率的逐像素预测。但FCN有一个硬伤：由于分类网络中的多次下采样，输出特征图的空间分辨率严重降低（原图的1/32），即使通过上采样恢复，边缘细节仍然粗糙。这个缺陷驱动了后续一系列改进。

DeepLab系列（v1/v2/v3/v3+，2016-2019）是Google团队对此问题的系统性解决方案。DeepLab v1和v2引入了空洞卷积（Atrous/Dilated Convolution）——在标准卷积核的像素之间插入空洞（即填零），在不增加参数量的情况下指数级扩大感受野。DeepLab v2进一步提出了ASPP（Atrous Spatial Pyramid Pooling，空洞空间金字塔池化），使用多个不同空洞率的空洞卷积并行提取多尺度特征，然后将它们融合。DeepLab v3改进了ASPP结构（加入了全局平均池化分支并去掉了CRF后处理），而DeepLab v3+则引入了编码-解码结构来恢复更精细的边界。

# 空洞卷积 —— 用一个感受野更大的"网格状"卷积核
import torch.nn as nn

# 普通 3×3 卷积，dilation=1（默认），感受野 = 3×3
conv_standard = nn.Conv2d(256, 256, 3, padding=1, dilation=1)

# 空洞卷积，dilation=2：卷积核像素之间隔1个空位
# 感受野扩大到 5×5，但参数量仍然是 3×3 = 9个
conv_dilated = nn.Conv2d(256, 256, 3, padding=2, dilation=2)

# 空洞卷积，dilation=4：感受野扩大到 9×9
conv_large_dilated = nn.Conv2d(256, 256, 3, padding=4, dilation=4)

# DeepLab 的 ASPP 模块 —— 多空洞率并行
class ASPP(nn.Module):
    def __init__(self, in_channels=2048, out_channels=256):
        super().__init__()
        self.branch1 = nn.Conv2d(in_channels, out_channels, 1)  # 1×1 分支
        self.branch2 = nn.Conv2d(in_channels, out_channels, 3, padding=6, dilation=6)
        self.branch3 = nn.Conv2d(in_channels, out_channels, 3, padding=12, dilation=12)
        self.branch4 = nn.Conv2d(in_channels, out_channels, 3, padding=18, dilation=18)
        # 全局平均池化分支：捕获全局上下文
        self.global_avg = nn.Sequential(
            nn.AdaptiveAvgPool2d(1),
            nn.Conv2d(in_channels, out_channels, 1)
        )

    def forward(self, x):
        b1 = self.branch1(x)
        b2 = self.branch2(x)
        b3 = self.branch3(x)
        b4 = self.branch4(x)
        b5 = F.interpolate(self.global_avg(x), size=x.shape[2:], mode='bilinear')
        return torch.cat([b1, b2, b3, b4, b5], dim=1)

编码-解码结构的深化

在FCN和DeepLab的启发下，研究者们意识到语义分割需要两个能力：提取多尺度语义和保留精细空间细节。"编码-解码"范式成为了主流方向，涌现了SegNet、PSPNet等一系列经典工作。

SegNet（2016）的思路和U-Net类似，但解码器更"轻量"——它使用编码器中最大池化层记录的池化索引（pooling indices）来做上采样，而不是学习转置卷积。这样解码器不需要学习额外的参数，但代价是灵活性降低。

PSPNet（Pyramid Scene Parsing Network，2017）提出了另一种思路：不靠编码-解码的对称性来恢复细节，而是通过金字塔池化模块（Pyramid Pooling Module, PPM）在编码器顶部捕获不同尺度的全局上下文。PPM将特征图分别划分为1×1、2×2、3×3、6×6的网格，对每个网格做全局平均池化，然后上采样回原尺寸后拼接。这样，模型同时获取了从全局到局部的多层次信息。PSPNet在ImageNet场景解析排行榜上曾一度位居榜首，它的设计也被后来许多模型借鉴。

Transformer时代的语义分割

2020年Vision Transformer（ViT）的成功自然地被移植到了分割领域。相比于CNN，Transformer的自注意力机制天然适合捕获长距离依赖——这正是语义分割所需要的。在CNN中，两个相距很远的像素需要通过层层卷积叠加才能建立关联，而Transformer的一层自注意力就可以让所有位置的像素直接交互。

SETR（SEgmentation TRansformer，2021）是最早尝试将纯Transformer用于语义分割的工作之一。它直接使用ViT作为编码器，然后在上采样阶段使用多层级联的CNN层恢复到全分辨率。SETR虽然展示了Transformer在分割上的潜力，但计算开销很大，且由于ViT缺乏层级化设计（始终保持单分辨率），对多尺度信息的建模不够自然。

SegFormer（NeurIPS 2021）在SETR的基础上做了重要改进，提出了分层Transformer编码器（Hierarchical Transformer Encoder）。和ViT的单分辨率不同，SegFormer的编码器像ResNet一样逐层下采样，生成多尺度的特征金字塔，再通过一个轻量级的MLP解码器融合各层特征得到最终分割结果。SegFormer的设计简洁高效——它不需要位置编码、不需要复杂的解码器设计，在ADE20K上达到了51.8的mIoU，同时速度比DeepLab v3+快数倍。

MaskFormer（NeurIPS 2021）提出了一个更有颠覆性的思路：将所有的分割任务统一为"掩码分类"问题。无论语义分割、实例分割还是全景分割，都可以分解为三件事——预测一组二进制掩码、预测每个掩码的类别标签、将掩码分配给图像中的像素。这个统一视角让MaskFormer能够用同一个架构处理所有分割任务，仅需在推理时根据不同任务的目标做后处理。

💡 从"像素分类"到"掩码分类"的范式转变：传统语义分割（如DeepLab）的范式是"每个像素做分类"——对图像中的每个像素，预测一个类别。MaskFormer的范式是"预测一组掩码+标签"——先预测整个图像中的所有可能物体/区域掩码，再给每个掩码分配一个类别。前者的输出是H×W的类别图，后者的输出是N个掩码×标签对。这个转变看似简单，却统一了语义分割和实例分割两个原来完全不同的任务。

全景分割——语义分割+实例分割的统一

你可能会想：既然语义分割处理一切背景像素（stuff），实例分割处理 countable objects（things），为什么不把它们放在一个统一的框架里？这正是全景分割（Panoptic Segmentation）的核心理念。

全景分割由Alexander Kirillov等人在2018年的论文Panoptic Segmentation（CVPR 2019）中正式提出。目标的定义很简单：对于图像中的每个像素，分配一个标签（label, instance_id）元组。对于stuff类，instance_id被忽略（或设为0）；对于things类，相同的instance_id表示属于同一个物体实例。这样，全景分割就完整覆盖了图像中的所有像素，不留死角。

实现全景分割的经典方法是Panoptic FPN（2019）。它在Mask R-CNN的FPN（特征金字塔）骨干上同时添加了一个语义分割头和一个实例分割头。语义头处理stuff类，实例头处理things类，最后通过一个融合模块将两者的输出合并为全景分割结果。Panoptic FPN简单有效，成为了后续全景分割工作的基线。

全景分割的评价指标是PQ（Panoptic Quality，全景质量），由识别质量（SQ，Segmentation Quality）和检测质量（RQ，Recognition Quality）两部分组成：

分割基础模型：SAM与Mask2Former

2023年，Meta AI开源的Segment Anything Model（SAM）彻底改变了分割领域的格局。SAM是一个"分割基础模型"——它不是在特定数据集上训练的专用模型，而是在超过10亿个掩码（覆盖1100万张图像）上训练的一个通用分割模型。SAM的核心理念是：用一个模型、一套权重，通过不同的提示（点、框、文本）分割任何图像中的任何物体。

SAM的架构包含三个部分：图像编码器（一个经过MAE预训练的ViT，将图像编码为特征嵌入）、提示编码器（将用户输入的提示转换为嵌入向量）和掩码解码器（一个轻量级的Transformer解码器，根据图像嵌入和提示嵌入预测掩码）。这种设计让SAM具备了惊人的泛化能力——即使在医学图像、卫星图像、显微图像等从未见过的领域，SAM也能给出合理的分割结果。

Mask2Former（CVPR 2022）在MaskFormer的基础上引入了Transformer解码器的掩码注意力（Masked Attention）机制，只让每个查询（object query）关注其预测区域的内部像素，大幅提升了训练效率和分割精度。Mask2Former用单一的架构在语义分割、实例分割和全景分割三个任务上都取得了当时的最佳结果，真正实现了"一次训练、三面全能"。

🔮 分割技术的未来趋势：从FCN到SAM，分割技术走过了一个完整的循环。最早的方法是"通用"的（FCN在任意图像上做逐像素预测），后来分化为语义、实例、全景三个方向各自深入发展。现在，MaskFormer/Mask2Former和SAM又走回了"通用"的路线——用统一的架构覆盖所有分割任务。这种"分化→统一"的螺旋式上升是技术发展的常态。预计未来的分割技术将沿着两条路径演进：一是基础模型的通用能力继续增强（更大数据、更强提示机制），二是面向特定领域的轻量化高效模型（移动端、实时部署）。

分割技术的应用

分割技术已经渗透到了视觉应用的方方面面。在此总结几个主要方向：

医学影像是分割最早也最深入的应用领域。从U-Net时代的细胞分割到SAM时代的通用医学图像分割，分割技术被用于CT/MRI中的器官自动标注、病理切片中的细胞核检测、眼底图像中的血管分割等。分割的精度直接影响着疾病诊断的可靠性——一个边界错误几毫米就可能改变肿瘤的TNM分期。

自动驾驶中，分割是感知系统的核心组件之一。语义分割用于可行驶区域识别、车道线检测、行人轮廓提取；全景分割则提供对整个街景的完整像素级理解——地面、建筑、车辆、行人、交通标志等——这些信息输入到规划和控制模块，指导车辆做出驾驶决策。

遥感与地理信息中，分割被用于高分辨率卫星图像和航拍图像的分析：建筑物轮廓提取、农田地块分割、森林覆盖面积统计、水体变化检测。在这些应用中，分割模型的效率和精度直接决定了地理信息更新的成本和周期。

图像编辑与创意设计是普通用户最能切身感受到的分割应用。从Photoshop的"快速选择"工具到手机相册的"人像模式"背景虚化，再到视频会议中的虚拟背景替换——这些功能的背后都离不开图像分割技术。SAM的出现让这类应用的交互方式发生了根本改变：用户可以像"聊天"一样用点选和框选的方式和图像交互，实时分割出想要的任何物体。

⚠️ 分割模型选型建议：面对众多的分割模型，如何选择适合自己项目的方案？这里给出一个简洁的决策流程——如果你想做像素级分类但不区分实例（比如识别道路、天空、建筑），首选DeepLab v3+或SegFormer；如果你需要区分物体个体且有充足算力（比如检测街景中的每辆车），首选Mask R-CNN或Cascade Mask R-CNN；如果你想在单一框架中同时覆盖things和stuff（比如完整的场景解析），使用Mask2Former；如果你的需求是通用的、没有特定训练数据、或者需要交互式分割，直接上SAM——它已经成为了分割领域的"default choice"。
