卷积运算详解 — 互相关、卷积核、特征图

在上一章中，我们看到全连接网络将输入的每个像素作为一个独立的特征维度来处理——对于一个224×224×3的彩色图像，这意味着第一层就有超过15万个输入。这不仅导致参数量爆炸，更重要的是完全忽略了图像的空间结构：一个像素和它相邻像素之间的关系，在全连接层中没有任何体现。

卷积神经网络（Convolutional Neural Network, CNN）正是为了解决这个问题而设计的。它的核心——卷积运算——灵感来源于生物学。Hubel和Wiesel在1962年的经典实验中发现了猫的视觉皮层中存在"简单细胞"和"复杂细胞"：简单细胞只对特定方向的光条有响应，复杂细胞则对方向敏感但对位置不敏感。卷积层就相当于一个可学习的简单细胞集合——每个卷积核专门检测图像中的某种局部模式（如边缘、纹理、角点）。

卷积运算本质上是一种"滑动窗口"操作：一个小的二维矩阵（称为卷积核或滤波器）在输入图像上从左到右、从上到下地滑动，在每个位置计算核与对应图像区域的逐元素乘积并求和。这个求和结果构成了输出特征图（Feature Map）中对应位置的值。值得注意的是，在深度学习的术语中，"卷积"实际上更准确地应该被称为互相关（Cross-Correlation）——因为我们对卷积核进行了翻转。不过由于卷积核的参数是可学习的，翻转与否没有实质性区别，所以大家一直在用"卷积"这个称呼。

每个卷积核学习识别一种特定的局部模式：水平边缘、垂直边缘、斜线、斑点、纹理……多个卷积核叠加在一起，就像给图像做了多组不同"滤镜"的处理。网络浅层的卷积核学习简单的模式（边缘、颜色），深层的卷积核组合浅层特征学习更复杂的模式（眼睛、轮子、窗户）。这种层次化的特征学习是CNN最强大的特性——它不需要任何人告诉它"什么是有用的特征"，而是从数据中自动学习。

# 手动实现 2D 卷积操作 (理解原理)
import numpy as np

def conv2d_manual(image, kernel, stride=1, padding=0):
    """手动实现 2D 卷积 (单通道)"""
    # 对输入做 padding
    if padding > 0:
        image = np.pad(image, padding, mode='constant')

    H, W = image.shape
    kH, kW = kernel.shape

    # 计算输出尺寸
    out_H = (H - kH) // stride + 1
    out_W = (W - kW) // stride + 1
    output = np.zeros((out_H, out_W))

    # 滑动窗口计算
    for i in range(out_H):
        for j in range(out_W):
            h_start = i * stride
            w_start = j * stride
            output[i, j] = np.sum(
                image[h_start:h_start+kH, w_start:w_start+kW] * kernel
            )

    return output

# 示例: 用 Sobel 边缘检测核做卷积
sobel_x = np.array([[-1, 0, 1],
                    [-2, 0, 2],
                    [-1, 0, 1]])

# 创建一个模拟的边缘图像
image = np.zeros((10, 10))
image[:, 5:] = 1.0  # 垂直边缘

result = conv2d_manual(image, sobel_x)
print("输入图像 (10×10, 垂直边缘):")
print(image)
print("\n卷积后的特征图 (8×8):")
print(np.round(result, 1))
print("\n可见: 卷积核在边缘位置产生了强响应!")

卷积层的关键参数 — 卷积核大小、步长、填充、输入输出通道

设计卷积层时，有四个关键参数决定了它的行为和输出尺寸。这可以说是CNN中最基本的"算术题"——每一个CV工程师都必须熟练到条件反射的程度。

卷积核大小（Kernel Size）决定了每个卷积核感受野的范围。最常用的是3×3核——它提供足够大的感受野来捕捉局部模式，同时保持了参数效率。5×5和7×7核可以覆盖更大范围，但参数更多、计算量更大。1×1卷积核看起来很奇怪——它只关注单个像素，但在跨通道信息融合中非常有用（我们会在后续章节中展开）。经验法则：多个小卷积核的堆叠（如两个3×3）比单个大卷积核（如一个5×5）效果更好、参数更少、非线性更强。

步长（Stride）是卷积核每次滑动的像素数。步长为1时输出特征图与输入大小相近；步长为2时输出尺寸减半。步长大于1的卷积天然起到了下采样（downsampling）的作用，相当于在特征提取的同时降低了空间分辨率——这在深层CNN中是一个关键的设计模式。

填充（Padding）是在输入图像的边缘补一圈（或多圈）零值像素。填充的主要目的是控制输出特征图的空间尺寸。如果不做填充，每次卷积都会使输出缩小（kernel_size - 1）个像素，多次卷积后图像会变得非常小。通常在希望输出尺寸与输入相同时使用"same"填充：补padding = (kernel_size - 1) // 2个像素的零。

输入输出通道是理解卷积层最关键的维度。输入通道数等于上一层的输出通道数（对第一层来说就是图像的通道数，如RGB为3）。每个卷积核有和输入通道数相同数量的"切片"，一个卷积核在所有输入通道上同时操作，产生输出特征图的一个通道。因此输出通道数等于这一层卷积核的个数。如果有K个输出通道、C个输入通道、H×W的卷积核大小，那这一层的参数量就是C × K × H × W（加上偏置）。

参数含义常见值对输出的影响
Kernel Size卷积核的空间尺寸1, 3, 5, 7越大感受野越大，参数越多
Stride滑动步长1, 2越大输出尺寸缩小越快
Padding边缘补零0, 1, 'same'控制输出尺寸衰减
Channels输入/输出通道数64, 128, 256, 512决定了层的表达能力
# 卷积层输出尺寸计算公式
# output_size = (input_size + 2*padding - kernel_size) // stride + 1

# 当 input=224, kernel=3, padding=1, stride=1:
# output = (224 + 2*1 - 3) // 1 + 1 = 224  (尺寸不变)

# 当 input=224, kernel=3, padding=1, stride=2:
# output = (224 + 2*1 - 3) // 2 + 1 = 112  (尺寸减半)

def conv_output_size(input_size, kernel_size, padding=0, stride=1):
    return (input_size + 2*padding - kernel_size) // stride + 1

print("常用配置的输出尺寸:")
for s in [1, 2]:
    out = conv_output_size(224, 3, 1, s)
    print(f"  224 → 3×3 conv, stride={s}: {out}")

# 参数量计算
def conv_params(in_channels, out_channels, kernel_size):
    return in_channels * out_channels * kernel_size * kernel_size

print(f"\n参数量对比:")
print(f"  3×3 conv, 3→64: {conv_params(3, 64, 3):,}")
print(f"  5×5 conv, 3→64: {conv_params(3, 64, 5):,}")
print(f"  3×3 conv×2, 3→64: {conv_params(3, 64, 3) + conv_params(64, 64, 3):,}")
print(f"(两个3×3的参数量 < 一个5×5, 但感受野相同!)")

池化层 — 最大池化、平均池化、全局平均池化

池化层（Pooling Layer）是CNN中另一个关键的组件。它的作用是下采样——降低特征图的空间尺寸，从而减少参数量、扩大感受野、并且提供微小平移不变性。池化是"暴力但有效"的：它在每个小窗口内只保留一个统计量（最大值或平均值），丢弃其他所有信息。

最大池化（Max Pooling）是最常用的池化方式。它在2×2或3×3的窗口内取最大值作为输出，通常配合步长等于窗口大小的设置（如2×2窗口，步长为2）。这样特征图的宽高各减半，但保留了每个区域内最显著的特征响应。最大池化的效果是：只关心"这个区域内有没有出现有用的特征"，不关心特征的确切位置。这种"位置容忍度"对于分类任务来说非常重要——我们希望无论猫出现在图像的左边还是右边，模型都能识别出它是一只猫。

平均池化（Average Pooling）在窗口内取平均值而不是最大值。它保留了区域内所有特征的总体强度信息，但没有最大池化的"突出显著特征"的特性。早期CNN（如LeNet-5）中使用平均池化，但现代CNN几乎都使用最大池化，因为实践中发现最大池化的效果普遍更好。不过平均池化在某个特定场景下有一个非常重要的应用——全局平均池化。

全局平均池化（Global Average Pooling, GAP）将整个特征图的每个通道压缩成一个标量——即对每个通道的所有位置取平均值。GAP通常放在卷积层的最后、全连接分类层之前。它取代了早期CNN中使用的展平（Flatten）操作+大型全连接层。GAP的优点是：消除了全连接层的大量参数（因为GAP的输出通道数等于最后的特征图通道数，直接连接分类器），同时让模型对输入图像的尺寸变化更加鲁棒。

# 池化层实现
import numpy as np

def max_pool2d(input, pool_size=2, stride=2):
    """最大池化"""
    batch, channels, H, W = input.shape
    out_H = (H - pool_size) // stride + 1
    out_W = (W - pool_size) // stride + 1
    output = np.zeros((batch, channels, out_H, out_W))

    for i in range(out_H):
        for j in range(out_W):
            h_start = i * stride
            w_start = j * stride
            output[:, :, i, j] = np.max(
                input[:, :, h_start:h_start+pool_size, w_start:w_start+pool_size],
                axis=(2, 3)
            )
    return output

def global_avg_pool(input):
    """全局平均池化: 将每个通道压缩为一个标量"""
    return np.mean(input, axis=(2, 3))  # shape: (batch, channels)

# 示例
x = np.random.randn(1, 64, 28, 28)  # (batch, channels, H, W)
x_pooled = max_pool2d(x, 2, 2)      # → (1, 64, 14, 14)
x_gap = global_avg_pool(x_pooled)    # → (1, 64)
print(f"输入形状:     {x.shape}")
print(f"最大池化后:   {x_pooled.shape}")
print(f"全局平均池化: {x_gap.shape}")

感受野与参数共享 — 为什么CNN适合图像

CNN之所以在图像处理中如此成功，可以归结为两个核心设计原则：局部连接和参数共享。这两个原则共同构成了CNN的"归纳偏置"——它告诉模型：图像中的信息具有空间局部性和平移不变性。

感受野（Receptive Field）指的是输出特征图上的一个点对应到输入图像上的区域大小。在全连接网络中，每个输出节点都连接到所有输入节点，因此感受野就是整个图像。在CNN中，每个输出节点只连接到一个局部区域——这个区域的大小就是感受野。通过堆叠多个卷积层，深层特征图中的每个点可以获得更大的感受野。例如：堆叠两个3×3卷积（步长为1）可以产生5×5的感受野，三个3×3卷积可以产生7×7的感受野。这就是"用小的核实现大的感受野"的经典技巧。

参数共享是CNN另一个革命性的思想：同一层的同一个卷积核在整个图像上重复使用。也就是说，无论图像中哪个位置出现了"水平边缘"，都是由同一个卷积核来检测。这有两层含义：第一，参数量大幅减少——一个3×3的卷积核只有9个参数（加上偏置），不管输入图像是28×28还是1024×1024，卷积核的参数数量不变。第二，平移不变性——一个模式无论出现在图像的左上角还是右下角，都能被同一个卷积核检测到。

相比之下，全连接层没有参数共享，每个输出节点有自己的权重。对于224×224的彩色图像，一个全连接层从输入到256个隐藏神经元就需要224×224×3×256 ≈ 4300万个参数——这比很多完整的CNN模型还多。这就是为什么全连接层只在网络最后、特征图尺寸已经很小时才被使用。

🎯 为什么CNN适合图像？设想一个场景：你在两张不同照片的左上角都看到了一个猫耳朵。在全连接网络中，左上角的猫耳朵和右下角的猫耳朵由不同的权重处理——它需要分别学习"左上角的猫耳朵"和"右下角的猫耳朵"两个不同的概念。在CNN中，由于参数共享，同一个卷积核负责整个图像上猫耳朵的检测——只要学会了猫耳朵长什么样，无论它出现在哪里都能被识别。参数共享让CNN天生具有平移不变性，这正是图像处理最需要的性质。

经典CNN结构 — LeNet-5、基本组件组合模式

理解了卷积层和池化层的原理之后，我们可以看看它们是如何组合成一个完整网络的。第一个成功的CNN应用——LeNet-5——由Yann LeCun在1998年提出，用于手写数字识别。它简洁优雅地展示了CNN的基本设计模式，这个模式影响了之后所有的CNN架构。

LeNet-5的网络结构如下：输入是32×32的灰度图像（手写数字）。第一层是6个5×5卷积核（步长1，无填充），输出6个28×28的特征图。接着是2×2的平均池化（步长2），输出变成6个14×14。第二层卷积用16个5×5卷积核，输出16个10×10的特征图，再经过一个2×2平均池化得到16个5×5。然后展平为400维向量，接两个全连接层（120→84→10），最后通过Softmax输出10个数字类别的概率。

LeNet-5的模式可以概括为：卷积 → 激活 → 池化 → (重复) → 展平 → 全连接 → 输出。这个"卷积提取特征 + 池化降低分辨率 + 全连接做分类"的三段式结构，成为后续所有CNN的通用模板。

CNN可视化 — 特征图可视化、感受野计算

"神经网络是黑盒"——这句话你肯定听过。但事实上，对于CNN来说，我们有很多工具可以"打开"这个黑盒。特征图可视化是最直观的一种方法，它能让你看到CNN每一层到底学到了什么。

特征图可视化非常简单：取一个中间层的输出，将其中的每个通道单独显示为一张图像。结果往往非常有趣——浅层（如conv1）的特征图看起来像是边缘检测器和颜色滤波器的集合，你能看到水平边缘、垂直边缘、颜色块等。中层（如conv3、conv4）的特征图开始出现纹理和局部图案——如"条纹"、"网格"、"圆点"等。深层（如conv5及之后）的特征图则高度抽象——它们不再对应人类可理解的视觉模式，而是对特定物体类别或物体部件的响应。这种从具体到抽象的层次化特征学习，是CNN最深刻的性质之一。

感受野计算在设计网络时非常重要——如果你想让深层特征图中的每个点"看到"输入图像上的大范围信息，需要确保感受野足够大。感受野的计算公式是递推的：RF_l = RF_{l-1} + (kernel_size - 1) × stride_product，其中stride_product是之前所有层的步长乘积。对于LeNet-5，最后一个卷积层（conv2）的感受野是5 + (5-1) × 2 = 28——即它能看到输入图像中28×28的区域（相对于32×32的输入，几乎覆盖了整张图）。

# 感受野计算
def receptive_field(layers):
    """
    layers: 列表, 每个元素是 (kernel_size, stride)
    返回每层结束后的感受野大小
    """
    rf = 1
    stride_product = 1
    rfs = [rf]

    for k, s in layers:
        rf = rf + (k - 1) * stride_product
        stride_product *= s
        rfs.append(rf)

    return rfs

# LeNet-5 的感受野计算
lenet_layers = [(5, 1), (2, 2), (5, 1), (2, 2)]  # conv1, pool1, conv2, pool2
rfs = receptive_field(lenet_layers)
print("LeNet-5 各层感受野:")
names = ["输入", "Conv1后", "Pool1后", "Conv2后", "Pool2后"]
for name, rf in zip(names, rfs):
    print(f"  {name}: {rf}×{rf}")

# 对比: 堆叠两个 3×3 vs 一个 5×5
stacked_3x3 = [(3, 1), (3, 1)]  # 两个 3×3 conv
single_5x5  = [(5, 1)]          # 一个 5×5 conv
print(f"\n两个 3×3 的感受野: {receptive_field(stacked_3x3)[-1]}")
print(f"一个 5×5 的感受野: {receptive_field(single_5x5)[-1]}")
print(f"感受野相同, 但两个 3×3 参数量更少且非线性更强!")
⚠️ 可视化误区：特征图可视化虽然直观，但要注意不要过度解读。看到某个通道对"眼睛"有响应，不代表这个通道就是专门检测眼睛的——CNN的特征是分布式表示的，同一个概念通常由多个通道共同编码。另外，深层特征图对人类肉眼来说可能看起来全是噪声，但这不代表它们没有信息——只是信息的编码方式不是人类视觉能直接理解的。这就是为什么我们还需要Grad-CAM等更高级的可视化工具（在第14章中会详细介绍）。
