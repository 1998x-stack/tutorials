第十章：DQN改进家族

同学们好，我是你们的讲师。上一章我们学会了 DQN——第一次用深度神经网络来拟合 Q 函数，让强化学习能够处理 Atari 游戏这样的高维视觉输入。但大家有没有发现，DQN 虽然革命性，但问题也不少：Q 值总是估计得偏高？在有些状态下动作选择对结果影响不大，但网络还是得为每个动作单独学习？经验回放池里绝大部分样本都是"废话"？

这一章我们就要解决这些问题。我会介绍四种最重要的 DQN 改进——Double DQN、Dueling DQN、优先经验回放（Prioritized Experience Replay），以及将它们融合在一起的 Rainbow。学完这一章，你会理解为什么说"一个简单的小改动有时比一个全新的算法更有价值"。

10.1 Double DQN：解决过估计问题

我们先从一个困扰 Q-Learning 多年的顽疾说起——过估计（overestimation）。

问题出在 Q-Learning 的 max 操作上。标准的 Q-Learning 目标值是这样计算的：

y = r + γ maxa' Q(s', a')

这里 max 操作同时做了两件事：它用同一个网络既选择了最佳动作（argmax），又评估了这个动作的价值（max）。问题在于 Q 函数的估计是有噪声的——如果某个动作被高估了，max 操作会"选中"这个被高估的动作，然后用这个偏高的值作为目标，导致 Q 值系统性地向正方向偏移。这就像考试时让学生自己出题自己打分，分数自然会虚高。

更严重的是，这种过估计不是均匀的。在动作空间大的环境中，随机噪声导致的最大值会随着动作数增加而增大。一开始可能只是微小的偏差，但通过 Bellman 更新的自举（bootstrapping）机制，偏差会在一轮又一轮的迭代中不断被放大，最终导致 learning 彻底失效。

直觉理解： 假设你在 10 个 Q 值估计中找最大值，每个都是真实值 0 加上均值为零的随机噪声。噪声最大的那个被选中的概率很高——你选中的不是"真实的 Q 最大"，而是"噪声最大的那个"。这在数学上被称为最大化偏差（maximization bias）。

Double DQN 的解决办法简洁而优雅——解耦选择与评估。它用在线网络（online network）来选择动作，用目标网络（target network）来评估该动作的价值。公式变为：

y = r + γ Qtarget(s', argmaxa' Qonline(s', a'))

Hado van Hasselt 在 2015 年的论文中证明，这种解耦能有效消除过估计。实现上只需要改动几行代码——是典型的"小改动大收益"：

import torch
import torch.nn.functional as F

# 标准 DQN 的目标值（过估计）
def compute_target_dqn(reward, next_state, done, gamma, target_net):
    q_next = target_net(next_state)             # [batch, n_actions]
    max_q_next = q_next.max(dim=1)[0]             # max 同时负责选择和评估
    return reward + gamma * max_q_next * (~done)

# Double DQN 的目标值（解耦）
def compute_target_ddqn(reward, next_state, done, gamma,
                          online_net, target_net):
    q_next_online = online_net(next_state)          # 在线网络：选择动作
    best_actions = q_next_online.argmax(dim=1)       # 选择：argmax
    q_next_target = target_net(next_state)          # 目标网络：评估价值
    selected = q_next_target.gather(1, best_actions.unsqueeze(1))
    return reward + gamma * selected.squeeze() * (~done)

看到区别了吗？标准 DQN 用了两个 max（一个是隐式的 argmax，一个是显式的 max），而 Double DQN 将这两个操作分配到两个不同的网络上。经验表明，在 Atari 游戏中，Double DQN 能显著提高最终得分和稳定性，尤其是在训练早期过估计最严重的时候。

10.2 Dueling DQN：状态价值与动作优势的分离

接下来这个改进的切入点更为深刻。Dueling DQN 的作者提出了一个问题：Q(s, a) = r + γV(s') 告诉我们 Q 值包含了状态价值和在当前状态下执行某个动作的"加成"。那我们为什么不把这两部分显式地建模出来呢？

Dueling DQN 的核心思想是将 Q 网络拆分为两个流（stream）：

  • 状态值流（Value Stream）： 输出 V(s)，表示状态 s 本身的价值，与具体动作无关。
  • 优势流（Advantage Stream）： 输出 A(s, a)，表示在状态 s 下选择动作 a 相对于平均水平的优势。

最后将两者合并得到 Q 值：

Q(s, a) = V(s) + A(s, a)

但这里有一个可辨识性（identifiability）问题：给定 Q，我们可以解出无数组 (V, A) 的组合。标准做法是强制让优势函数在最优动作上的值为零，等价于让 A(s, a) 减去其最大值：

Q(s, a) = V(s) + A(s, a) − maxa' A(s, a')

实际操作中，用减去均值代替减去最大值效果往往更好，因为这样更稳定：

Q(s, a) = V(s) + A(s, a) − (1/|A|) ∑a' A(s, a')

从网络架构来看，Dueling DQN 的特征提取部分（卷积层或全连接层）是共享的，之后才分叉为两个流。这种设计带来的好处非常直观：在一些状态下，动作选择本身并不重要，agent 可以专注于学习状态的价值。比如在 Atari 的某些游戏中，屏幕上有大量"无关紧要"的画面——agent 只需要学到"当前状态不错"就够了，不需要为每个动作精确计算 Q 值差异。

架构示意： 输入 -> 卷积特征提取 -> [全连接 -> V(s)] 与 [全连接 -> A(s, a)] 并联 -> Q = V + (A - mean(A))
踩坑实录： 我第一次实现 Dueling DQN 时犯了一个非常隐蔽的错误。我当时直接用了 Q = V + A 的公式，没有加任何约束。结果训练出来的网络完全不管动作——所有动作的 Q 值几乎一模一样！损失函数虽然下降得很快，但策略完全没学到东西。问题出在哪里？没有约束的 V 和 A 是可以互相"抵消"的：网络发现可以偷偷把 V 学成 Q 的平均值，让 A 在零附近波动，而不需要真正去建模动作之间的差异。加上 max 约束后问题瞬间解决，策略也立刻学会了区分好动作和坏动作。这个坑让我深刻理解了：在你的网络结构中加入归纳偏置（inductive bias）不只是锦上添花，有时是决定成败的关键。

10.3 优先经验回放（Prioritized Experience Replay）

经验回放（Experience Replay）是 DQN 的关键组件之一，但它有一个明显的缺陷：均匀采样假设所有样本同等重要。然而现实是，agent 在环境中收集到的数据大部分都是"平凡"的——比如在 Atari 游戏中，大部分时间画面没什么变化，agent 只是在漫无目的地移动。这些平凡的样本对学习的贡献很小。

什么样的样本有价值？TD 误差大的样本。TD 误差 |δ| = |y − Q(s, a)| 越大，说明当前 Q 函数的预测与目标值之间的差距越大，也就是 Q 网络在这个样本上"错得最离谱"。这些样本最能推动学习。

优先经验回放（Prioritized Experience Replay, PER）的核心思路就是：给每个样本一个优先级 pi = |δi| + ε（ε 是一个很小的正数，保证优先级不为零），采样概率正比于优先级。

但这里有一个陷阱。引入优先级后，采样分布被改变了，这意味着我们估计的期望值是有偏的。就像民意调查如果只采访"意见强烈"的人，结果就不代表整体民意。解决方法是引入重要性采样权重（Importance Sampling weights）来修正偏差：

wi = (1/N · 1/P(i))β

其中 β 在训练过程中从 0 逐渐增加到 1。训练初期允许有偏的优先采样来加速学习，训练后期逐渐修正偏差保证收敛质量。

实现技巧： PER 通常使用 Sum Tree（求和树）来高效实现优先级采样。Sum Tree 是一种二叉树，叶子节点存储每个样本的优先级，父节点存储子节点之和。采样时生成一个随机数，沿着树往下"二分查找"即可。复杂度从 O(N) 降到了 O(log N)。如果你的优先级更新较频繁（每条样本更新一次 Sum Tree），用分段线性分段（Segmented Tree）会更加高效。

经验表明，PER 是 DQN 改进中效果最明显的技术之一，单独使用就能大幅提升 Atari 游戏的得分和训练速度。

10.4 Rainbow：集大成者

前面我们介绍了三个改进。但学术界对 DQN 的改进远不止这些。DeepMind 在 2017 年的一篇论文中回顾了整个 DQN 改进家族，他们总结了六项比较重要的改进，并做了一个经典实验——将这些改进逐步叠加到一个 agent 中：

  1. Double DQN — 解决过估计
  2. Dueling DQN — 分离 V 和 A
  3. Prioritized Experience Replay — 聚焦重要样本
  4. Noisy Nets — 在参数上添加可学习的噪声，替代 ε-greedy 探索
  5. Distributional RL（C51） — 不再预测 Q 值的期望，而是预测整个回报分布
  6. Multi-step returns — 用 n 步回报替代单步 TD 目标，平衡偏差与方差

他们将这个结合了所有改进的 agent 命名为 Rainbow（彩虹）。实验结果令人振奋：Rainbow 的性能远超任何单个改进，甚至在很多 Atari 游戏上超过了人类水平。更重要的是，他们做了详尽的消融实验（ablation study）——每次移除一个组件，观察性能下降了多少：

算法中位数得分（标准化的 Atari 57 游戏）解读
Rainbow（完整版）153%六项改进协同工作，效果最好
Rainbow − 多步回报125%移除后下降最明显，多步回报贡献最大
Rainbow − Prioritized Replay123%PER 的贡献也很大
Rainbow − Distributional RL118%分布 RL 提供更丰富的信号
Rainbow − Noisy Nets109%探索策略的改进也重要
Rainbow − Dueling105%Dueling 的增益相对较小但仍有效
Rainbow − Double DQN103%Double DQN 的增益在组合中最小

这张表告诉我们两件重要的事情：第一，"整体大于部分之和"——各项改进之间存在协同效应，不是简单叠加。第二，消融实验至关重要——没有它，我们永远不知道每个组件到底贡献了多少。

关于 Noisy Nets： 这是一个很聪明的想法。传统的 ε-greedy 在参数空间中是"盲目的"随机探索。Noisy Nets 在神经网络的权重上添加高斯噪声，让噪声的均值和方差也能通过梯度下降来学习。这样 agent 可以自动学会在需要探索的状态下加大噪声、在需要利用的状态下减小噪声。这是一种自适应探索策略，比 ε-greedy 更灵活。
要不要用 Rainbow？ 在 2024 年的今天，如果你要做离散动作空间的 DRL 项目，Rainbow 仍然是不错的起点。但对于连续控制任务，作者建议直接使用 SAC 或 TD3。Rainbow 的意义更多在于它展示了一个方法论：系统地分析问题、解耦改进点、用消融实验验证每个假设——这种思路在任何 ML 项目中都适用。

知识延伸：从值函数到策略的呼唤

到这里，我们已经走完了值函数方法的几乎全部重要路线：从表格 Q-Learning 到 DQN，再到 DQN 的各种改进。值函数方法非常强大，但它有一个根本性的局限——它只能处理离散动作空间。在 Dueling DQN 中，网络的输出维度是 |A|，这要求动作空间不仅离散而且不能太大。

那连续动作怎么办？比如机器人关节控制中，动作是 7 维连续向量；自动驾驶中，方向盘角度是连续值。值函数方法在这里遇到了天花板——我们不可能为每个连续的动作值都输出一个 Q。

解决方案是换一种思路：不学 Q，直接学策略 π(a|s)。这就是下一章要讲的策略梯度方法（Policy Gradient Methods）。策略梯度直接将策略参数化，沿着提升期望回报的方向更新参数。它天然支持连续动作空间，而且还能学到随机策略——这在某些部分可观测的环境中至关重要。

当然，策略梯度也有自己的问题：高方差、收敛慢、样本效率低。所以再后来就有了 Actor-Critic——结合两者的优点，用值函数来降低策略梯度的方差。这就是第 12 章的内容了。

各位同学，请记住这一章的关键：DQN 改进家族展示了"简单的洞见 + 精确的实现 = 巨大的提升"。这些改进思路（解耦选择与评估、分解 Q 值、优先级采样、组合改进）远比具体的算法公式更值得你记住。下一章，我们将踏入一个全新的世界——策略梯度。那里没有 max 操作，没有 Bellman 最优方程，有的只是直接对策略的梯度上升。敬请期待！

本章小结

  1. Double DQN 通过解耦动作选择（在线网络）和动作评估（目标网络），消除了 Q-Learning 的系统性过估计问题。改动极小，效果显著。
  2. Dueling DQN 将 Q 值分解为状态价值 V(s) 和动作优势 A(s, a)，使网络能独立地学习"状态好坏"，在动作不影响结果时尤为高效。
  3. 优先经验回放（PER） 根据 TD 误差衡量样本重要性，聚焦于"难样本"，配合重要性采样权重修正分布偏移。
  4. Rainbow 融合了六项改进，展示了"系统性的组合"比"寻找单一魔法改进"更具价值。消融实验是理解每个组件贡献的关键工具。
  5. 值函数方法在处理连续动作空间时遇到根本性瓶颈，这促使我们转向策略梯度方法——下一章的主题。
