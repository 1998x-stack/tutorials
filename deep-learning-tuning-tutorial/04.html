学习率——深度学习最重要的超参数，没有之一

大家好，我是你们的讲师。如果让我在所有超参数中只能调一个，我会毫不犹豫地选学习率（Learning Rate）。知乎上有一段广为流传的金句："优先调 learning rate！优先调 learning rate！优先调 learning rate！"——重要的话说三遍。这不是夸张，而是无数踩坑换来的经验。

真实的故事：一位算法工程师接手了一个推荐模型优化任务，跑了一周，AUC纹丝不动。他尝试了改网络结构、加特征、调正则化——全都没用。我过去看了一眼，发现他的学习率设成了1e-6。AdamW的默认LR是3e-4左右，他的LR小了300倍。把LR改到3e-4，AUC在6小时内涨了2个点。一周的无效工作，源头只有一个超参数没设对。这不是个例，我在不同项目里见过至少五次类似的情况。

学习率不对，其他全白搭。

LR为什么是第一优先级

学习率的几何意义非常直观——它是参数更新时的步长。回到第二章的"蒙眼下山"比喻：学习率就是迈出的步长。

  • LR太小（比如1e-6）：你小心翼翼地挪动脚步，每步只有毫米级。理论上最终也能到谷底，但需要走几百万步——训练时间会让人崩溃。更糟糕的是，如果模型很深，小LR的更新信号可能在反向传播中就被吸收了（尤其是没有归一化层的网络），导致靠近输入层的参数几乎不动。
  • LR太大（比如10）：你以百米冲刺的步伐下山。一步可能从半山腰跨到对面山坡上，loss不降反升。更极端的，一步冲出参数空间的有效区域，loss直接炸到NaN。
  • LR合适：每一步都稳步朝谷底前进，loss平稳快速地下降。

在凸优化理论中，对于L-Lipschitz连续的损失函数，存在一个理论最优LR的上界：lr ≤ 1/L，其中L是损失函数的Lipschitz常数（梯度变化的最大速率）。超过这个上界，梯度下降就可能发散。这给了我们一个重要的直觉：LR的上限由损失曲面的"崎岖程度"决定——曲面越崎岖（L越大），LR就必须越小。

在实践中，调参的第一件事永远是先用LR Range Test确定合理范围，然后再调其他参数。这已经成为所有调参高手的肌肉记忆。

实战口诀："LR太小：loss几乎不动，中间层特征有水波纹状噪点。LR太大：loss震荡发散，甚至直接NaN。LR合适：loss平稳下降，曲线平滑。需要降低LR：loss一路降下来后不再下降——手动减半LR继续。"——来自Captain Jack的知乎回答

Warmup的数学原理

Warmup（预热）是指在训练初期从极小学习率逐渐上升到目标学习率的策略。它现在是训练的标准组件，尤其是在Transformer上，已经是必须加上的操作了。

Warmup为什么必要？有四个层面的原因：

第一，参数初始化不理想。训练刚开始时，参数还是随机的，远离最优解。此时损失曲面极度扭曲，梯度的方向和大小都不准确。如果直接上大LR，第一步就可能把参数踢到未知区域，然后梯度爆炸或者产生NaN。小LR先走几步，等参数稍微稳定了再放大——类似刚学自行车时先慢慢骑，熟练了再加速。

第二，Adam的二阶矩估计需要"热身"。Adam初始化时 m₀=0, v₀=0。训练的前几步，vₜ 非常小（因为累积的平方梯度还不够）。如果不加偏差修正（bias correction），√v̂ₜ 会非常小，导致 lr/√v̂ₜ 非常大，等效LR远高于设定值，让训练初期更新过大。Warmup给了vₜ足够的时间建立准确的估计。注意Adam的偏差修正项 1-β₂ᵗ 可以缓解这个问题，但对短训练（<1000步）仍然不够——这也是为什么短训练也要加warmup。

第三，BatchNorm的统计量需要稳定。训练刚开始时，BN的running mean和running var还没有建立。如果直接上大LR，参数快速变化会导致BN的统计量和当前激活值严重不匹配，形成恶性循环。Warmup让参数缓慢变化，给BN足够的时间跟踪分布。

第四，Attention矩阵初始不均衡。这是Transformer特有的问题。Attention机制在初始化时，Q和K的点积结果方差很大，导致softmax输出接近one-hot（非常尖锐的注意力分布）。如果直接上大LR，注意力分布会更加极端，梯度更新集中在少数token上。Warmup让注意力分布先"软"下来，再逐步训练。

Warmup步数的经验法则：模型越复杂warmup越长，数据越大warmup越短。BERT默认用总步数的1/10。LLM预训练通常采用固定2000步的warmup。

Cosine退火为什么有效

Cosine Annealing（余弦退火）是当前最流行的学习率调度策略之一。它的公式很简单：

lr(t) = lr_min + 0.5 × (lr_max - lr_min) × (1 + cos(π × t / T))

其中 t 是当前步数，T 是总步数。LR从 lr_max 开始，沿着半个余弦曲线平滑下降到 lr_min。最通用的配置是 lr_min=0。

Cosine退火为什么有效？因为它完美地解决了"探索-利用"（exploration-exploitation）的平衡：

  • 训练初期（大LR）：大LR让参数大步探索损失曲面，快速跳过不好的区域。这相当于"走出丛林"阶段——不追求精细，只求快速找到好区域。
  • 训练中期（中LR）：LR逐渐减小，参数开始在好区域周围精细化搜索。这个阶段损失下降最快。
  • 训练后期（小LR）：LR很小，参数在极小值附近精细收敛，不会因为步长太大而跳出好区域。

Cosine退火与阶梯衰减（Step Decay）的关键区别：Cosine的LR是连续平滑下降的，而Step Decay是断崖式下跌。平滑下降的好处是每次LR变化都不大，训练始终稳定。Step Decay每次断崖式降LR后，loss会短暂上升（因为参数需要适应新的学习率），然后才继续下降。

Cosine退火还有一种增强版——带热重启的余弦退火（SGDR）。它在一个周期结束后突然把LR跳回初始值，模拟重新开始训练。这种周期性重启可以帮助参数跳出当前的局部极小值，寻找更好的解。而且每次重启后收敛到不同的局部极小值，可以用多个checkpoint做模型集成。

关于warmup + cosine的组合用法：业界最通用的训练模板是"AdamW + Linear Warmup + Cosine Decay"。Warmup阶段线性增长到目标LR，之后cosine平滑下降到0。这个配置在各种任务上表现稳定，可以作为任何新任务的起点。

另一点：cosine退火的初始值（lr_max）应该用LR Range Test来确定，而不是随便设一个经验值。这是很多人的盲区——调度策略再好，初始LR选错了也白搭。

LR Range Test的操作与原理

学习率范围测试（LR Range Test / Learning Rate Finder）是Leslie N. Smith在2015年提出的方法，现在已经成为确定初始LR的标准手段。核心思想简单得令人发指：让LR从一个极小值开始，在若干batch内指数增长到极大值，观察loss曲线，然后从曲线中找到loss下降最快的区间。

具体操作流程：

  1. 准备一小部分训练数据——通常1-2个epoch就足够了，不需要完整数据集
  2. 初始化优化器——设置初始LR为极小值，比如1e-7
  3. 对每个mini-batch：执行前向+反向+更新，然后按指数增长增大LR
  4. 记录每个点的LR和loss值
  5. 绘制LR（对数坐标） vs Loss曲线

解读loss曲线的三个区间：

  • 区间1（LR过小）：loss几乎不变或下降极慢。模型几乎学不到任何东西。
  • 区间2（LR合适）：loss以最快速度下降，曲线斜率最大。这是目标区间。
  • 区间3（LR过大）：loss震荡或开始反弹。训练已不稳定。

推荐的初始LR在区间2内——通常是区间2中部，或者取区间2起点到终点中间更保守的值。一个常用的经验规则：取loss曲线斜率最大的点对应的LR，或者取区间2起点的LR。

下面是完整的PyTorch代码实现：

import torch

        def lr_range_test(model, train_loader, optimizer, loss_fn,
                          lr_min=1e-7, lr_max=10, num_iter=100):
            """LR范围测试：指数增长扫描LR，观察loss曲线"""
            lrs, losses = [], []
            lr = lr_min
            gamma = (lr_max / lr_min) ** (1.0 / num_iter)

            model.train()
            for i, (inputs, targets) in enumerate(train_loader):
                if i >= num_iter:
                    break

                # 设置当前LR（支持多param_group的场景）
                for param_group in optimizer.param_groups:
                    param_group['lr'] = lr

                # 前向 + 反向
                optimizer.zero_grad()
                outputs = model(inputs)
                loss = loss_fn(outputs, targets)
                loss.backward()
                optimizer.step()

                lrs.append(lr)
                losses.append(loss.item())
                lr *= gamma  # 指数增长

            return lrs, losses

        # 使用示例
        model = YourModel()
        optimizer = torch.optim.AdamW(model.parameters(), lr=1e-7)  # 初始极小LR
        lrs, losses = lr_range_test(model, train_loader, optimizer, nn.CrossEntropyLoss())

        # 找到loss下降最快的点
        # Simple heuristic: find the LR where loss decreases fastest
        losses = torch.tensor(losses)
        # 平滑处理（推荐指数移动平均）
        smoothed = [losses[0]]
        alpha = 0.1
        for loss in losses[1:]:
            smoothed.append(alpha * loss + (1 - alpha) * smoothed[-1])

        # 找下降最快的区间
        gradients = torch.tensor(smoothed).diff()
        best_idx = gradients.argmin().item()  # 负梯度最大（下降最快）
        print(f"建议初始LR: {lrs[best_idx]:.6f}")

        # 更保守：取最快下降点1/10
        conservative_lr = lrs[best_idx] / 10
        print(f"保守建议LR: {conservative_lr:.6f}")
LR Range Test的常见陷阱：（1）不要在完整数据集上跑——1-2个epoch就足够了，跑多了浪费算力，而且结果没有显著改善。（2）Range Test给出的是"loss下降最快的LR"，不是训练全周期的最优起点——考虑到warmup等因素，通常取的初始LR比Range Test建议值偏小。（3）对LLM等超大规模模型，Range Test的参考价值有限——训练数据分布太广，单次扫描偏差大，更推荐使用Scaling Law的预测或业界已验证的范围。

下面是针对不同模型类型的LR推荐范围汇总。注意这是初始LR范围，最终使用的LR需要配合调度策略（warmup + cosine decay等）：

ResNet（ImageNet） SGD+Momentum 1e-3 ~ 1e-1 0.1 经典的Kaiming初始化搭配
ResNet（CIFAR-10） SGD+Momentum 1e-2 ~ 1.0 0.1 数据集小LR可以更大
YOLO/检测模型 SGD+Momentum 1e-3 ~ 1e-2 0.01 取决于batch size

这个表格凝聚了大量实践经验。我的建议是：拿不到一个任务的LR范围时，先从表格中的"典型值"开始，然后用LR Range Test快速验证，再微调。

Warmup + Cosine退火的完整实现

下面是一个在生产环境中广泛使用的WarmupCosineScheduler实现。它支持线性warmup + cosine退火，是最通用的训练配置：

import math
        import torch
        from torch.optim.lr_scheduler import _LRScheduler

        class WarmupCosineScheduler(_LRScheduler):
            """Linear warmup + cosine decay scheduler.

            Args:
                optimizer: 优化器
                warmup_steps: warmup步数
                total_steps: 总训练步数
                min_lr: 最低LR（cosine退火的目标值）
                last_epoch: 上一轮epoch
            """
            def __init__(self, optimizer, warmup_steps, total_steps,
                         min_lr=0, last_epoch=-1):
                self.warmup_steps = warmup_steps
                self.total_steps = total_steps
                self.min_lr = min_lr
                super().__init__(optimizer, last_epoch)

            def get_lr(self):
                step = self.last_epoch + 1  # 当前步数

                if step <= self.warmup_steps:
                    # 阶段一：线性warmup
                    scale = step / self.warmup_steps
                    return [base_lr * scale for base_lr in self.base_lrs]
                else:
                    # 阶段二：cosine退火
                    progress = (step - self.warmup_steps) / \
                               (self.total_steps - self.warmup_steps)
                    progress = min(1.0, max(0.0, progress))
                    cosine_decay = 0.5 * (1.0 + math.cos(math.pi * progress))
                    return [(self.min_lr + (base_lr - self.min_lr) * cosine_decay)
                            for base_lr in self.base_lrs]

        # 使用示例
        model = YourModel()
        optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4)
        total_steps = 10000
        warmup_steps = int(total_steps * 0.1)  # 10% warmup

        scheduler = WarmupCosineScheduler(
            optimizer=optimizer,
            warmup_steps=warmup_steps,
            total_steps=total_steps,
            min_lr=0
        )

        # 训练循环中
        for step in range(total_steps):
            # ... 正常前向 + 反向 ...
            optimizer.step()
            scheduler.step()  # 更新LR

这个调度器的效果是：前 warmup_steps 步从0线性增长到目标LR，然后沿cosine曲线平滑下降到min_lr。它已经是当前绝大多数深度学习任务的默认配置。
