第18章：总线与I/O系统

同学们好！前面17章一直围绕处理器和内存打转——处理器架构、流水线、缓存、虚拟内存。但是，一台计算机不能只算数——它得从用户那里接受输入，得把结果展示出来，得读写硬盘，得连接网络。这些与外部世界的交互，就是I/O系统（Input/Output System）的职责。而连接一切的核心，就是总线（Bus）。这一章，我们要从底层的总线仲裁讲到最新的CXL缓存一致性互连——你会发现，总线技术在最近十年里经历了一场静悄悄的革命。

总线基础

总线是什么？用最简单的话说：总线是计算机各部件之间的共享通信通路。处理器、内存、I/O设备都连接在总线上，通过总线传输地址、数据和控制信号。

总线按工作方式分为两类：同步总线（Synchronous Bus）包含时钟信号，所有操作在时钟边沿上同步进行。优点是简单、快速；缺点是所有设备必须以相同时钟频率工作，系统的灵活性受限。早期的处理器内存总线（如Intel的FSB——Front Side Bus）就是同步的。

异步总线（Asynchronous Bus）使用握手协议，没有统一的时钟。发送方发出请求，接收方用"应答"信号确认——类似两个人在电话里说"听到了吗？""听到了，你说。"优点是设备可以有不同的速度，易于升级。但握手协议的开销使得性能不如同步总线高。

按结构层次，总线分为：

  • 处理器-内存总线：短、高速、通常与处理器紧耦合，设计上往往是私有的（proprietary）。
  • I/O总线：标准化、可扩展、支持多种设备。PCIe就是这一类。
  • 背板总线（Backplane）：连接整个系统，在服务器刀片架构中常见。

一个完整的总线事务包括三个阶段：仲裁（谁获得总线使用权）→ 地址（发目标地址/设备ID）→ 数据（传输数据）。

总线仲裁

多个设备可能同时想使用总线——谁来拿控制权？这就是仲裁（Arbitration）的责任。主要有几种方案：

菊花链（Daisy Chain）仲裁：总线授权信号从一个设备传到下一个设备。离仲裁器最近的设备优先级最高，离得远的设备可能永远拿不到总线（叫"饥饿"——starvation）。优点是接线简单、扩展容易；缺点是不公平。经典的SCSI总线就用这种方式。

集中式并行仲裁：每个设备有单独的总线请求（Request）线和总线授权（Grant）线连接到中央仲裁器。仲裁器根据优先级策略（固定优先级或轮转Round-Robin）选择谁获得总线。优点是公平性好，缺点是接线复杂，可扩展性受限。

自选择（Self-Selection）：每个设备有一个唯一的优先级编号。想用总线的设备把自己的编号放到总线上，然后检查是否"赢"了——如果自己放的编号比总线上的高（意味着有更高优先级的设备也在请求），就让步。PCI总线使用这种方法。

冲突检测：类似CSMA/CD（以太网）。设备直接发送，如果检测到冲突就退避重试。在系统总线上使用较少，主要用于网络场景。

现代PCIe的最大变化是：**不再有共享总线，也没有传统意义上的"仲裁"**。PCIe使用点对点串行链路加交换（Switch）架构，每个设备都有自己的独占通道。后续我再细讲。

DMA：让数据自己移动

在早期的计算机系统中，CPU必须亲自参与每一次I/O数据传输。

  • 轮询（Polling）：CPU不停检测设备状态寄存器——"数据来了吗？来了吗？来了吗？"——整个过程CPU被占用，不能做有用的事情。
  • 中断驱动（Interrupt-Driven）：设备准备好后通过中断通知CPU。CPU在中断处理中亲自把数据从设备拷贝到内存（或反向）。每次中断都有保存/恢复上下文的开销，高吞吐量场景下CPU还是受不了。

DMA（Direct Memory Access，直接内存访问）解决了这个问题：CPU设置好传输参数（源地址、目标地址、传输大小），然后DMA控制器接管，直接在设备和内存之间搬运数据，完成后发送一个中断通知CPU。CPU在DMA传输期间可以继续执行其他任务。

// DMA控制器的简化配置流程（C伪代码）

// 1. CPU配置DMA控制器
dma_ctrl->src_addr  = (uint32_t)device_data_register; // 从设备读
dma_ctrl->dst_addr  = (uint32_t)memory_buffer;         // 写入内存
dma_ctrl->count     = 4096;                            // 传输4KB
dma_ctrl->control   = DMA_READ | DMA_INTERRUPT_ENABLE; // 启动

// 2. DMA控制器自动传输
while (dma_ctrl->count > 0) {
    bus_read(device_data_register, &temp);   // 从设备读取
    bus_write(memory_buffer++, temp);         // 写入内存
    dma_ctrl->count--;
}

// 3. 传输完成后发送中断
if (dma_ctrl->control & DMA_INTERRUPT_ENABLE) {
    interrupt_cpu(IRQ_DMA_DONE);
}

// CPU看到buffer中已有4096字节数据，直接使用

DMA有两种工作模式：突发模式（Burst Mode）——DMA控制器持续占用总线，快速传输整个块，然后释放。适合大量数据传输，但可能导致CPU长时间无法访问主存。周期窃取模式（Cycle-Stealing Mode）——DMA每次只传输一个word或几个word，然后释放总线。DMA控制器"窃取"CPU的一个总线周期来传输数据，两者交替访问。对CPU干扰较小，但传输速度较慢。

现代高端DMA还支持Scatter-Gather：用一个描述符链表描述多个不连续的源/目标缓冲区，DMA控制器依次处理每个描述符。对于网络协议栈中常见的非连续缓冲区（sk_buff结构），一次DMA就可以完成所有数据搬运，省掉了数据拷贝的CPU开销。

PCIe：现代总线的标准

从1992年的PCI到2003年的PCI Express（PCIe），总线领域经历了一次根本性的变革。PCI是共享并行总线——所有设备共用32位或64位宽的数据线，工作频率从33MHz到66MHz。随着设备增多、频率升高，信号同步和反射问题变得极其困难。

PCIe则彻底转向了点对点串行链路（Point-to-Point Serial Links）。每个设备通过专用通道（Lane）连接到一个Switch，不再共享带宽。关键的转变包括：

  • 串行差分信号：每对数据用两根线（差分对），使用8b/10b编码（PCIe 1.0/2.0）或128b/130b编码（PCIe 3.0+）。串行信号频率可以远高于并行信号，因为不再需要保持所有数据线的同步。
  • 链路聚合：x1、x4、x8、x16通道捆绑。PCIe 5.0每个通道32 GT/s（Giga-transfers per second），x16通道总带宽约64 GB/s。
  • 分层协议：事务层（Transaction Layer）→ 数据链路层（Data Link Layer）→ 物理层（Physical Layer）。设备通过"能力结构"（Capability Structure）声明自身支持的功能。

PCIe 6.0是一个重大飞跃：它使用PAM-4信号编码（每符号2比特，而非传统的NRZ每符号1比特），达到64 GT/s。同时引入了前向纠错（Forward Error Correction, FEC）来处理PAM-4更高的信号噪声。x16通道提供约128 GB/s的双向带宽。PCIe 6.0还引入了Flit-based编码（固定大小的流量单元），进一步降低延迟。

CXL：缓存一致性互连

现在的问题是：PCIe虽然带宽很高，但它本质上还是I/O总线——设备不能直接看到和缓存主存（至少不能以缓存一致的方式）。这对加速器（GPU、FPGA、智能网卡）来说是个巨大的限制：每次访问内存都要通过驱动做DMA，通信延迟很高。

CXL（Compute Express Link，计算快速链接）应运而生。它在PCIe物理层的基础上，增加了三个关键协议：

  • CXL.io：基于PCIe的I/O协议，用于设备发现、配置、DMA。
  • CXL.cache：允许加速器访问主机的缓存层次并保持一致性。加速器可以缓存主存数据，当CPU修改了数据时，加速器的缓存也会收到失效消息。
  • CXL.mem：允许加速器访问主机内存（或访问连接的内存扩展设备），以类似本地内存的延迟和语义。这使得"内存池化"（Memory Pooling）成为可能。

为什么CXL是革命性的？想象一个数据中心：每个服务器节点有固定数量的内存（比如256GB）。有时某台服务器的应用需要更多内存但附近节点有空闲。传统方案无法共享，只能为每个节点配置过量内存（浪费成本）。CXL支持内存扩展——把内存做成独立的CXL设备，多个主机共享同一个内存池。CXL 3.0更进一步，支持跨多个主机的内存共享和一致性。这被称为分解式计算（Disaggregated Computing），被认为是未来数据中心的方向。

CXL 1.0/2.0（2019年）主要实现了单跳的一致性（即一个主机连接一个加速器）。CXL 3.0（2022年）实现了多级交换和Fabrics——多个主机可以同时访问共享内存池，并保持一致性。CXL 3.0的带宽基于PCIe 6.0（64 GT/s），延迟目标在亚微秒级别。

我给大家讲个实际的故事。2023年我参与了一个AI训练集群的选型评估。传统的方案是每个GPU节点配2TB内存——但大模型训练时，数据集的准备和预处理需要大量内存，峰值时不够用，平时又浪费。而通过CXL连接的内存扩展器（Memory Expander），我们可以在不同节点之间灵活分配内存——训练节点可以"借用"空闲节点的内存来做数据预处理。虽然这个技术当时还在早期阶段，但CXL的潜力已经非常明显。我相信未来三五年，CXL将像PCIe替代PCI一样，彻底改变服务器的内存架构。
