第26章 自回归统一模型

大家好，我是你们的讲师。如果你在过去两年里关注过生成模型的进展，你应该注意到一个有趣的现象：当所有人都在谈论扩散模型时，自回归路线悄无声息地杀了一个回马枪。2025 年，智源研究院在 Nature 上发表了 Emu3——一个用"预测下一个 token"统一了图像、视频、文本生成与理解的模型。紧接着 2026 年，OmniGen-AR 把这个范式的边界推到了"任意模态到任意模态"。

这一章，我们来看为什么"next-token prediction"这个最早出现在语言模型中的朴素思想，突然具备了挑战扩散模型统治地位的能力。这不是历史的倒退，而是范式的升维。

26.1 自回归路线复兴的背景

扩散模型在 2022-2025 年间毫无疑问是图像和视频生成的主导范式。但它的局限性也日益明显：

  • 速度瓶颈：即使使用了 DDIM、DPM-Solver 等加速采样器，扩散模型仍然需要数十步迭代。实时生成（如视频通话背景替换、交互式 AI 创作）对延迟的要求极其苛刻。
  • 架构割裂：文本生成用 Transformer（自回归），图像生成用 U-Net 或 DiT（扩散），视频生成又需要额外的时序模块——每种模态各有一套专门的模型架构和训练流程。
  • "理解"与"生成"的分离：GPT-4o 这种多模态模型虽然能理解图像，但无法生成图像；Stable Diffusion 能生成图像，但无法理解其中的语义内容。这种割裂意味着额外的工程复杂度和推理成本。

而自回归路线提供了一张反方向的"药方"：如果所有模态都能被离散化为同一套 token 序列，那一个 Transformer 模型就可以同时完成理解和生成——因为"理解"本质上也是预测缺失的 token。

🌟 核心洞察："预测下一个 token"这个范式如此强大，是因为它不需要为每种模态设计专门的损失函数或训练目标。文本的 next-token prediction、图像的 next-patch prediction、视频的 next-frame prediction——它们在数学形式上完全可以统一。这就是自回归统一路线的根基。

26.2 Emu3：智源研究院的 Nature 级工作

Emu3 的核心贡献可以用一句话概括：把图像和视频也变成"token 序列"，然后用一个标准的因果 Transformer（causal Transformer）做 next-token prediction。这个"原来如此"的简单想法，之所以等到 2025 年才实现，关键在于图像和视频的离散化（tokenization）一直没有找到足够好的方案。

Emu3 的技术方案分解为三步：

  • 第一步：视觉 tokenization。使用改进的 VQ-GAN（Vector Quantized GAN）将图像切分为 16×16 的 patch，每个 patch 映射到离散 codebook 中的一个 token。对于 256×256 的图像，这产生了 16×16 = 256 个离散 token。
  • 第二步：视频 tokenization。在图像 tokenization 的基础上引入时间维度，将视频帧序列也离散化为 token 序列，视频帧之间的时序关系通过自回归建模自然捕获——不需要额外的时序注意力模块。
  • 第三步：统一预训练。将文本 token、图像 token、视频 token 混合在一个序列中，训练一个标准的 causal Transformer 做 next-token prediction。训练完成后，模型既能做文本续写（理解），也能做图像生成和视频生成。
💡 技术细节：Emu3 使用的 VQ-GAN codebook 大小为 65536（2^16），每个 token 对应 16 bit 的信息。这意味着 256×256 的图像被编码为 256 个离散整数，压缩率惊人。相比之下，Stable Diffusion 的 VAE 将图像压缩到潜空间的连续特征图，两者在对齐语言模型 token space 方面，离散化方案自然更优——语言本身的 token 也是离散的整数 ID。

26.3 "预测下一个 token"统一生成与理解

Emu3 最令人兴奋的地方不是生成质量超过了哪个扩散模型，而是它证明了理解和生成可以被同一个模型、同一个损失函数完成。

考虑一个简单的例子：输入序列是 ["一张", "猫", "的", "照片", "<img>", "tok₁", "tok₂", ..., "tok₂₅₆", "</img>"]。模型在训练时，如果看到文本前缀就预测下一个图像 token——这是"文生图"；如果看到图像前缀就预测下一个文本 token——这是"图像描述"；如果遮住中间的图像 token，让模型根据上下文预测——这就是"图像修复"。

同样的 Transformer，不同的任务只取决于你如何排列和掩码 token 序列。这正是"统一"二字的含义。

范式输入预测目标对应任务
文生图文本 token 序列图像 token 序列文生图、文生视频
图生文图像 token 序列文本 token 序列图像描述、VQA
图文混合部分图文 token剩余 token多模态理解
视频预测前 N 帧 token后 M 帧 token视频续帧、视频预测
⚠️ 一个真实的踩坑故事：我曾在 2025 年初尝试复现 Emu3 的视觉 tokenization 部分。结果卡了整整两周——生成的图像总是出现明显的"棋盘格伪影"（checkerboard artifacts）。深入排查后发现，问题出在 VQ-GAN 的 codebook collapse（大部分 codebook entry 未被使用）。解决方案是引入 codebook 的重新初始化机制（codebook reset）和承诺损失（commitment loss）的权重调优。这个经历让我深刻认识到：自回归统一模型的"概念"很美，但让视觉 tokenizer 稳定工作，本身就是一篇论文的工作量。

26.4 OmniGen-AR：任意到任意模态生成

如果说 Emu3 证明了"统一是可能的"，那么 2026 年的 OmniGen-AR 证明了"统一是可以实战的"。OmniGen-AR 的全称是 Omni-modal Generative Auto-Regressive model，它将自回归统一范式的输入输出模态从"文本+图像+视频"扩展到了文本、图像、视频、音频、3D、代码六大模态。

OmniGen-AR 的关键技术创新：

  • 混合粒度 tokenization：不同模态的信息密度差异巨大——音频需要细粒度 token（如 50Hz 帧率），图像可以使用粗粒度 token（如 16×16 patch）。OmniGen-AR 为每种模态设计了独立的 tokenizer，但共享同一套 Transformer 的 embedding 空间。
  • 模态路由注意力（Modality-Routed Attention）：在 Transformer 的自注意力层中引入模态感知的路由机制，让来自同模态的 token 之间具有更高的注意力权重——这显著提升了多模态混合序列的生成质量。
  • 渐进式训练策略：先单模态预训练，再双模态对齐，最后统一微调。这种 curriculum learning 策略避免了多模态训练中的"模态竞争"问题（即模型偏向某种模态而忽略其他）。
# OmniGen-AR 伪代码：多模态统一训练循环示意
for batch in dataloader:
    # 1. 混合采样：从各模态数据集中按比例采样
    text_tokens = text_tokenizer(batch["text"])       # [B, L_t]
    image_tokens = img_tokenizer(batch["image"])       # [B, L_i]
    audio_tokens = audio_tokenizer(batch["audio"])     # [B, L_a]

    # 2. 拼接为统一序列（带模态标识符）
    unified_seq = concat_with_modality_ids(
        [text_tokens, image_tokens, audio_tokens],
        modality_ids=[0, 1, 2]  # 0=text, 1=image, 2=audio
    )

    # 3. 模态路由自注意力
    output = modality_routed_transformer(unified_seq)

    # 4. 损失：所有 token 的交叉熵（next-token prediction）
    loss = cross_entropy(output[:, :-1], unified_seq[:, 1:])
    loss.backward()

26.5 离散化方法：VQ-VAE / VQ-GAN

自回归统一模型的基石是离散 tokenization——把连续信号（图像像素、视频帧、音频波形）转换为离散整数序列。目前主流的方法是 VQ-VAE 和 VQ-GAN：

方法提出年份编码器量化方式感知损失GAN 判别器
VQ-VAE2017CNN/ResNet最近邻查找无无
VQ-VAE-22019层级式 CNN层级最近邻无无
VQ-GAN2021CNN/Transformer最近邻查找有 (LPIPS)有
ViT-VQGAN2022ViT最近邻查找有 (LPIPS)有
FSQ (免编码本)2024任意四舍五入量化可选可选

从表中可以看到一个趋势：量化方法本身越来越简单（FSQ 甚至完全去掉了可学习的 codebook），但生成质量却越来越高。这背后的逻辑是：随着 Transformer backbone 规模的增大，它有能力自己"消化"粗糙量化带来的信息损失——关键在于 codebook size 和 embedding 维度的匹配。

💡 实践建议：如果你要在自己的项目中使用 VQ tokenizer，建议从 VQ-GAN 开始。它的 GAN 判别器损失能显著提升重建图像的锐度。codebook size 建议设为 16384（2^14）或 65536（2^16），embedding 维度设为 256 或 512。codebook loss 的权重 β 通常设为 0.25——如果生成的图像出现模糊，尝试增大到 0.5；如果 codebook 使用率低（很多 entry 从未被选中），尝试降低 commitment loss 权重。

26.6 自回归 vs 扩散的对比与融合趋势

站在 2026 年中这个时间节点回望，自回归和扩散两条路线的竞争与融合已经非常清晰：

对比维度自回归模型扩散模型
生成速度慢（串行逐 token）中（数十步去噪，可加速）
推理时延随序列长度线性增长固定步数，与分辨率弱相关
统一多模态能力天然支持（同一 token 空间）需要额外设计（每模态独立架构）
高分辨率生成困难（长序列推理成本高）成熟（潜空间扩散）
训练稳定性高（标准交叉熵）中（需要噪声调度技巧）
数学似然可精确计算只能近似（ELBO）
编辑/控制灵活性低（序列难以局部修改）高（可通过 latent 操作精确控制）

目前可以看到的融合趋势包括：

  • 扩散 + 自回归：MAR（Masked Auto-Regressive）模型在扩散的去噪过程中引入自回归的先验信息
  • 离散扩散：D3PM、MaskGIT 等模型将扩散过程定义在离散 token 空间上，结合了两者的优点
  • 混合架构：在前几轮使用扩散做"粗粒度生成"，后几轮使用自回归做"细粒度精修"
🌟 核心洞察：不要纠结"哪个路线最终会赢"。实际工程中，Google 的 Gemini 和 OpenAI 的 GPT-5 都在探索"自回归理解 + 扩散生成"的混合架构——理解用自回归（因为需要长程依赖），生成用扩散（因为质量和控制更重要）。两条路线的融合才是真正的大趋势。

总结来说，自回归统一模型代表了一个极具吸引力的愿景：一个模型，一个损失函数，统一所有模态的理解与生成。虽然目前在高分辨率生成和推理效率上仍落后于扩散模型，但它的统一性优势是不可忽视的。下一章，我们将从另一个方向探讨实时生成——既然自回归太慢，扩散几步太多，那有没有办法把生成压缩到"一步"完成？
