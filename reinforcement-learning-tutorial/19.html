第十九章：基于模型的强化学习

同学们好！从本章开始，我们将进入一个全新的范式——基于模型的强化学习（Model-Based Reinforcement Learning, 简称 MBRL）。在前面十几章中，从 DQN 到 PPO 再到 SAC，所有算法都属于"免模型"（Model-Free）方法。免模型方法直接从交互数据中学习策略或值函数，不需要对环境显式建模。这种方法简单而强大，但它有一个致命的短板：样本效率极低。

19.1 模型的价值

不妨先做一个直观的对比。一个典型的免模型算法（比如 PPO 或 SAC）在一个 MuJoCo 机器人 locomotion 任务上，通常需要 106 到 107 个时间步才能学到像样的策略。而人类只需要几十次尝试就能大致掌握一个运动技能。为什么差距如此之大？因为免模型方法每一步交互都在"试错"——它没有利用一个最自然的能力：在头脑中模拟。

基于模型的方法正是为了弥补这个鸿沟而设计的。它的核心思想可以分为两步：

  • 学习模型：从交互数据中学习环境的动力学，即 \(s' \approx f(s, a)\) 和奖励函数 \(r \approx R(s, a)\)。
  • 利用模型做规划：用学到的模型"想象"未来的轨迹，基于想象的预演来选择动作。

可以想象一个国际象棋棋手。免模型的方法就像一个只通过下完整棋局来学习的棋手——他下了十万盘棋，每一盘都从头走到尾，从胜负结果中慢慢领悟哪些走法是好棋。而基于模型的方法则像一个可以在脑海中推演的棋手——他可以思考"如果我走马吃兵，对手可能会怎么应对，然后我再如何应对……"，在想象中探索成千上万种可能性，而实际一盘棋都不用下完。

这就是模型的巨大价值：把昂贵的实际交互转化为廉价的计算。样本效率的差距可以达到一到两个数量级——基于模型的方法只需 102 到 104 个真实 episodes 即可学会许多任务。

然而，模型也是一把双刃剑。一个核心挑战是 模型误差随规划步数累积。你想象的前两步可能很准，到第五步就完全偏离了真实轨迹。职业棋手能算到十几步之后，但初学者算三步就全错了。你学习的模型越不准，"想象力"就越靠不住。

对比维度 免模型（Model-Free） 基于模型（Model-Based）
样本效率 低（106~107 步） 高（102~104 步）
计算开销 低（只需策略/值网络前向） 高（需为模型推理 + 规划分配算力）
泛化到新任务 差（策略对新环境无适应能力） 好（模型可迁移，规划可调整）
理论收敛保证 在表格型 MDP 下有保证 模型误差导致保证较弱
实现复杂度 较低（端到端训练） 较高（模型 + 规划两套系统）
典型算法 DQN, PPO, SAC, TD3 Dyna-Q, MBPO, Dreamer, PlanNet

19.2 Dyna-Q 架构

Dyna-Q 由强化学习之父 Sutton 在 1990 年提出，是基于模型强化学习中最简单优雅的算法。它的思想可以用一个公式概括：

Dyna-Q = Q-Learning + 模型学习 + 规划

具体来说，Dyna-Q 在标准 Q-Learning 的基础上维护三个组件：

  • Q 函数 \(Q(s, a)\)：与 Q-Learning 完全相同，估计状态-动作值。
  • 环境模型 \(\hat{P}(s' \mid s, a), \hat{R}(s, a)\)：从真实交互数据中学习。最简单的实现就是记录每个 (s, a) 对应的 (s', r)，变成一个查表模型。
  • 规划过程：每次从模型中采样一批之前经历过的状态-动作对，用模型生成的模拟数据更新 Q 值。
算法：Dyna-Q（表格版本）

初始化 Q(s, a) = 0 对所有 s, a
初始化模型 Model(s, a) = (s', r) 对所有 s, a
初始化回放缓冲区 B

对于每个回合（episode）:
    初始化状态 s
    对回合中的每一步:
        a ← 由 Q(s, ·) 通过 ε-贪心选择
        执行 a，观察 r 和 s'
        用 (s, a, r, s') 更新 Q：Q(s, a) ← Q(s, a) + α[r + γ maxa' Q(s', a') - Q(s, a)]
        更新模型：Model(s, a) ← (s', r)
        重复 n 次（规划循环）:
            从已访问过的状态-动作中随机采样 (s_plan, a_plan)
            从模型获取 (s'_plan, r_plan) = Model(s_plan, a_plan)
            更新 Q：Q(s_plan, a_plan) ← Q(s_plan, a_plan) + α[r_plan + γ maxa' Q(s'_plan, a') - Q(s_plan, a_plan)]
        s ← s'
    直到 s 为终止状态

Dyna-Q 有一个重要的超参数 n-planning-steps（规划步数），即每一步真实交互后，用模型做了多少次模拟 Q 更新。n = 0 就是普通的 Q-Learning，纯"实践"不"思考"；n 越大，算力花在"思考"上的比例越高，样本效率通常也越高。但要注意：如果模型不准，思考得越多反而学得越差——这回到我们刚才说的模型误差问题。

19.3 学习动力学模型

在复杂任务中，查表模型显然不够用——状态空间是连续的，我们不可能为每一个见过的 (s, a) 记录一个对应的 s'。现代基于模型的方法使用 神经网络 来学习动力学模型。

确定性模型 vs 概率性模型

最直接的做法是训练一个神经网络 \(f_\theta(s, a) \to s'\)，通过最小化预测误差来学习：

确定性模型损失：L(θ) = E[ || fθ(s, a) - s' ||22 ]

这称为确定性模型——给定相同的 (s, a)，它永远预测同一个 s'。但现实世界的状态转移往往是随机的：同一个动作可能导致多种不同的结果。概率模型则输出一个条件概率分布 \(\hat{P}_\theta(s' \mid s, a)\)，通常建模为对角高斯分布——网络输出均值 \(\mu_\theta(s, a)\) 和方差 \(\sigma_\theta(s, a)\)，通过最大化对数似然来训练：

概率模型损失：L(θ) = -E[ log N( s' ; μθ(s, a), σθ(s, a) ) ]

集成模型与不确定性估计

然而，无论是确定性模型还是概率模型，都面临一个根本问题：模型在未见过的状态上会犯错。而强化学习的策略恰恰倾向于把智能体推向它没见过的状态——于是模型的错误被策略"利用"了。这个问题叫 model exploitation（模型利用），是基于模型强化学习中最致命的陷阱。

踩坑故事：模型利用的惨痛教训

我有个学生做一个四足机器人行走项目。他训练了一个神经网络动力学模型，在验证集上的预测误差只有 2%，非常漂亮。然后他用这个模型做规划，让机器人在"想象"中学习走路。最初的几千步模拟学得非常顺利——机器人走出了漂亮的步态，速度也越来越快。

他兴高采烈地把策略部署到真实机器人上。结果机器人刚一迈腿就摔了个四脚朝天。

问题出在哪里？验证集误差只衡量了模型在已有数据覆盖的区域上的表现。但强化学习策略在优化过程中，会主动去寻找那些模型预测"奖励特别高"的状态-动作——而这些往往恰好是训练数据没有覆盖、模型在胡乱"脑补"的区域。在想象中，模型"觉得"某个奇怪的动作能让机器人跳起来，于是策略拼命往这个方向优化，但真实世界中这个动作只会让机器人摔倒。

教训： 低验证误差 ≠ 好模型！解决方案是用模型集成——训练多个动力学模型，用它们预测的分歧程度来检测不确定性。当模型之间"意见不一致"时，就说明模型在该区域不可靠，不要盲目相信模型的想象。另外，限制想象轨迹的长度（只做短 rollout）也能有效防止误差累积。

解决 model exploitation 的关键武器是 模型集成（Ensemble）。我们训练 K 个独立的动力学模型 \(\{ f_{\theta_1}, f_{\theta_2}, ..., f_{\theta_K} \}\)，每个模型在数据的不同子集上初始化并训练。当预测下一个状态时，K 个模型的 分歧（disagreement） 就反映了模型在这个区域的不确定性：

  • 高度不确定的区域：模型之间的预测差异很大，说明该区域数据不足。
  • 充分探索过的区域：所有模型给出相似的结果，说明模型可信。

智能体利用这种不确定性来避免"盲目自信"——当模型集成分歧很大时，应该更多地依赖真实交互，而不是模型的想象。

19.4 现代基于模型的方法

MBPO：Model-Based Policy Optimization

MBPO 由 Janner 等人在 2019 年提出。它的核心洞见非常直观：模型生成的"短"轨迹是可信的，长轨迹不可信。因为误差会随步数指数级累积，但前几步通常还是靠谱的。因此，MBPO 不使用模型生成长轨迹，而是只生成长度为 H 的短分支（通常 H = 1~5 步），然后将这些短轨迹与真实轨迹混合，输入一个免模型算法（作者选用 SAC）进行训练。

具体流程：

  1. 维护一个模型集成（通常 7 个模型，每个是概率神经网络）。
  2. 从真实数据缓冲区采样一批初始状态。
  3. 用模型从这些初始状态开始 rollout H 步，将生成的模拟数据加入模型缓冲区。
  4. SAC 从混合缓冲区（真实数据 + 模拟数据）中采样训练。
  5. 定期用最新收集的真实数据重新训练模型。

MBPO 是"用模型做数据增强"这一思路的典范——它不试图让模型完美拟合环境，而是聪明地利用模型生成可信的短片段。

Dreamer：Dreaming with Model-Based RL

Dreamer 由 Hafner 等人在 2019-2020 年提出，是 Google DeepMind 的代表作之一。它的突破在于：把"想象"完全放在 潜空间（latent space） 中进行，从而大幅降低想象的维度。

Dreamer 学习三个组件：

  • 表示模型（Representation Model）：\(z_t \sim q(z_t \mid z_{t-1}, a_{t-1}, o_t)\)，将高维观测编码为低维潜状态。
  • 转移模型（Transition Model）：\(\hat{z}_t \sim p(\hat{z}_t \mid z_{t-1}, a_{t-1})\)，在潜空间中预测下一状态。
  • 奖励模型（Reward Model）：\(\hat{r}_t \sim p(\hat{r}_t \mid z_t)\)，从潜状态预测奖励。

训练完成后，Dreamer 完全在潜空间中进行"梦境" rollout：从真实编码的潜状态出发，利用转移模型和奖励模型生成想象中的轨迹，然后在想象中学习策略。由于潜空间通常比原始观测空间（如图像像素）小得多，Dreamer 每步真实交互可以"梦"出成千上万条想象轨迹，极大提升了样本效率。

PlanNet：Planning with Learned Models

PlanNet 走的是一条不同的路线：它专注于用学到的模型做 模型预测控制（MPC, Model Predictive Control）。每一时间步，PlanNet 在模型内部模拟多条未来轨迹（例如随机采样大量的动作序列，或使用交叉熵方法 CEM 优化），从中选出累计奖励最高的动作序列，然后只执行序列中的第一个动作。下一时间步，用最新的真实观测重新规划——这种"滚动时域"的方式天然地避免了模型误差的长期累积，因为每次规划都从真实的当前状态出发，且规划长度通常很短。

老师划重点： MBPO、Dreamer 和 PlanNet 代表了三种不同的"模型使用哲学"——MBPO 把模型当作数据增强工具，Dreamer 在潜空间中做梦并学策略，PlanNet 始终用 MPC 在线规划而不学策略。在实际任务中如何选择？如果你的任务是高维观测（如图像），Dreamer 的潜空间方法最合适；如果状态空间不大但动力学复杂，MBPO 的短 rollout + SAC 方案最稳健；如果任务需要快速适应且计算资源允许在线规划，PlanNet 的 MPC 方式最灵活。

19.5 基于模型 vs 免模型的抉择

在实际项目中如何选择？这不是一个非此即彼的问题，而是一道权衡题。

优先选择基于模型的情况：

  • 样本极其有限：真实环境昂贵或危险（机器人硬件磨损、医疗决策、自动驾驶路测）。每一秒钟的交互都意味着真金白银或潜在风险。
  • 需要快速适应：环境经常变化（如机器人换了负载、地形改变），模型可以快速重学或微调，而免模型方法需要重新采样海量数据。
  • 环境动力学相对简单：低维状态空间、平滑的动力学、可预测的过渡——这样的环境用神经网络模型很容易拟合。

坚持用免模型的情况：

  • 有高效的模拟器：像游戏、MuJoCo 这类环境可以并行化运行，样本不是瓶颈。
  • 动力学极其复杂：流体、软体、接触丰富的操作——这种环境建模本身就极其困难，模型误差大得让规划毫无意义。
  • 任务已被免模型方法很好地解决：如果 PPO 或 SAC 已经能跑出不错的结果，引入模型的复杂度反而增加维护成本。

但在最新的研究中，混合方法（Hybrid Approaches）正成为主流趋势：用模型做短期规划（如 MPC 规划几步），用免模型 RL 学一个策略提供长期引导；或者像 MBPO 那样用模型做数据增强，再用免模型方法在增强数据上训练。这种"两条腿走路"的思路往往能兼顾样本效率和最终性能。

关注数据效率的终极问题：

基于模型的方法通过"学习+想象"来提升样本效率，但它仍然需要与环境交互来收集数据训练模型。这就引出了一个更深刻的问题：如果我们连与环境交互的机会都没有——只有一个固定的、事先收集好的数据集，我们能从中学习最优策略吗？

这正是 离线强化学习（Offline RL） 要解决的问题。离线 RL 不从环境中采集新数据，而是像监督学习一样，从一个静态数据集中学习最优策略。它抛弃了"试错"的循环，这既是它的力量来源（不再需要与环境交互），也是它的根本挑战（没有试错意味着无法纠正数据中的错误）。离线 RL 可以被看作是一个"只有模型数据、没有真实交互"的极限场景——下一章我们将深入探讨这个问题。

本章小结

  • 基于模型的强化学习通过学习环境动力学模型，将昂贵的环境交互转化为廉价的"想象中的"规划，大幅提升样本效率。
  • Dyna-Q 是最简单的模型-based 架构，在 Q-Learning 基础上加入模型学习和规划循环，通过 n-planning-steps 参数调节"思考"的强度。
  • 学习动力学模型的核心挑战是模型误差累积和 model exploitation。解决方案包括概率模型（捕捉随机性）和模型集成（用分歧估计不确定性）。
  • 现代方法百花齐放：MBPO 用短 rollout + SAC 做数据增强，Dreamer 在潜空间中做梦并学策略，PlanNet 用 MPC 在线规划。
  • 混合方法结合模型-based 和 model-free 各自的优势，是当前实际应用的主流趋势。
  • 当交互完全不可行时，问题退化为离线强化学习——一个更具挑战性的范式，也是我们下一章的主题。
