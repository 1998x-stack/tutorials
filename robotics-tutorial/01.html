第1章 机器人学概述

大家好，我是你们的讲师。欢迎来到《机器人学·从入门到精通》。在2026年的今天，机器人行业正经历一场前所未有的变革——人形机器人开始进入工厂产线，VLA大模型让机器人第一次真正"理解"了人类语言，Sim-to-Real技术让仿真训练的机器人直接在真机上跑了起来。如果你选择在这一年踏入机器人学的大门，那真是挑对了时候。

在开始之前，我想先问你一个问题：你心目中"机器人"是什么？是像Atlas那样翻跟头的双足机器人？还是工厂里日夜不停挥舞的机械臂？或者，是一台帮你扫地、拖地的家用机器人？答案是——以上都是，但又不完全是。机器人学的魅力正在于此：它不止是一种机器，更是一整套让机器自主感知环境、做出决策、执行动作的学问。

1.1 什么是机器人学

机器人学（Robotics）的定义可以概括为一个闭环：感知（Sense）→ 规划（Plan）→ 行动（Act）。机器人通过传感器获取环境信息，经过算法处理后做出决策，最后通过执行器完成物理动作。这个循环不断重复，就构成了机器人的自主行为。

"Robot"这个词的来历很有意思。1920年，捷克作家卡雷尔·恰佩克（Karel Capek）在他的科幻戏剧《R.U.R.》（Rossum's Universal Robots）中首次创造了这个词，源自捷克语"robota"，意为"强制劳动"或"苦工"。恰佩克恐怕想不到，一百多年后，"robot"已经成为一门横跨机械、电子、控制、计算机和人工智能的交叉学科。

核心认知：机器人学不是单一学科，而是五大学科的深度融合——机械结构提供身体，电子电路提供神经，控制理论提供反射，计算机提供大脑，AI提供智慧。缺了任何一环，机器人都无法真正"活"起来。

在现代工业语境下，一个完整的机器人系统通常包含以下层次：

  • 机械层：关节、连杆、减速器、末端执行器——决定了机器人能做什么动作
  • 电气层：伺服电机、编码器、驱动器、通信总线——决定了动作的精度和响应速度
  • 控制层：运动学解算、动力学模型、力位混合控制——决定了动作的质量
  • 感知层：IMU、激光雷达、深度相机、六维力传感器——让机器人知道"自己在哪里"和"周围有什么"
  • 决策层：任务规划器、运动规划器、AI大模型——决定"下一步做什么"

1.2 机器人五千年简史

你可能觉得机器人是近百年才有的东西，但人类对"自动机器"的追求可以追溯到几千年前。我带你快速穿越一下这段有趣的历史。

古代自动机时代：公元前350年，古希腊数学家阿齐塔斯（Archytas）制造了一只蒸汽驱动的机械鸽子，被认为是历史上第一个自动机器。公元3世纪，中国古代科学家马钧发明了指南车——一套纯机械的差分齿轮系统，无论车子怎么转弯，车上小人的手臂始终指向南方。达芬奇在1495年设计了一台"机械骑士"，能够坐起、挥臂、转动头部，虽然从未被建造，但设计手稿展示了对人体运动学的深刻理解。

工业机器人诞生：真正的转折点发生在1961年。乔治·德沃尔（George Devol）和约瑟夫·恩格伯格（Joseph Engelberger）合作发明了Unimate——世界上第一台工业机器人。Unimate在通用汽车的新泽西工厂正式服役，负责从压铸机中取出炽热的金属零件并堆叠起来。我要讲一个小故事：当时通用汽车的工人对这台"铁家伙"充满了敌意，有个工人甚至偷偷在Unimate的控制器里塞了一块三明治，试图让它"罢工"。这件事后来成了机器人史上的著名趣闻，但也说明了一个道理——新技术总是伴随着误解和恐惧，真正让它被接受的，是它实实在在地减少了工伤、提高了效率。

智能机器人萌芽：1966年到1972年间，斯坦福研究所（SRI）的Shakey机器人成为了第一个能够"推理"自己行为的移动机器人。Shakey看起来就像一个套在计算机上的金属盒子，但它可以规划路径、推开障碍物、理解简单的自然语言指令。同一时期，斯坦福大学的Stanford Cart在1979年成功实现了自主导航——它花5个小时穿过了一个堆满椅子的房间，以大约1米/小时的速度缓慢前行。今天看来很可笑，但在当时，这已经是突破性的成就。

我把关键里程碑整理成了表格，方便你对照：

年代 机器人 意义

1.3 机器人的四大分类

机器人的种类繁多，为了便于研究，我们通常按应用场景分为四大类。每一类在设计理念、技术栈和评价指标上都有显著差异。

工业机器人（Industrial Robots）——精度就是生命。以六轴机械臂、SCARA、AGV/AMR为代表，核心指标是重复定位精度（通常±0.02mm到±0.1mm）、速度和可靠性。FANUC、ABB、KUKA是这一领域的传统巨头。2026年，协作机器人（Cobot）正在快速普及，优傲（Universal Robots）的e系列产品已经能够与工人安全共线作业。

服务机器人（Service Robots）——交互是第一要务。扫地机器人、送餐机器人、引导接待机器人等都属于这一类。它们不需要微米级的精度，但需要强大的自主导航、避障和人机交互能力。2026年的服务机器人几乎标配了大语言模型（LLM），可以流畅地进行多轮对话。

特种机器人（Specialty Robots）——能人所不能。包括水下机器人（ROV/AUV）、太空机器人（如NASA的Robonaut）、排爆机器人、搜救机器人等。它们被设计用于人类无法或不宜直接操作的极端环境，对密封、抗辐射、耐高温高压有特殊要求。

人形与四足机器人（Humanoid & Quadruped）——通用平台的终极形态。2026年被称为"人形机器人量产元年"：宇树科技的H2 Plus（搭载NVIDIA Isaac GR00T平台）在Computex 2026上大放异彩，波士顿动力的Atlas正式进入商业部署阶段，Figure 02在宝马工厂累计处理了超过90,000个零部件。四足机器人方面，波士顿动力Spot已经在石油、电力和建筑巡检领域站稳了脚跟。

以下是四大类机器人的核心特征对比：

类别 代表产品 核心指标 技术难点 典型售价（2026）
工业 FANUC CRX、ABB IRB 重复精度±0.02mm 运动学标定、振动抑制 $20K~$100K
服务 优地配送、云鲸扫地 导航成功率>99.9% 动态环境避障、语音交互 $500~$30K
特种 OceanOne、Robonaut 环境耐受等级 密封设计、远程遥操作 $100K~$10M

1.4 感知-决策-执行闭环

无论哪种机器人，其核心运作模式都可以归纳为"感知-决策-执行"（Sense-Plan-Act）闭环。我用一个简单的伪代码来描述这个循环：

while robot_is_running:
    # === 感知阶段 ===
    raw_data = read_sensors()           # 读取激光雷达/相机/IMU等
    state = estimate_state(raw_data)    # 状态估计（滤波、SLAM）
    env_map = build_map(state)          # 环境建模（栅格地图/TSDF）

    # === 决策阶段 ===
    task = parse_task_goal()            # 任务解析（如"取螺丝刀"）
    path = plan_path(state, env_map)    # 路径规划（A*、RRT）
    trajectory = generate_trajectory(path)  # 轨迹生成（时间参数化）

    # === 执行阶段 ===
    joint_cmds = inverse_kinematics(trajectory)  # 逆运动学解算
    torques = compute_control(joint_cmds)         # 控制律计算（PID、MPC）
    apply_torques(torques)                        # 驱动机器人运动

    # 闭环反馈：下一次迭代用新的传感器数据修正
    loop_rate.sleep()  # 控制循环频率（典型1kHz）

这个循环的频率决定了机器人的响应能力。工业机器人通常运行在1kHz（即每1ms完成一次感知-决策-执行循环），而人形机器人的平衡控制则需要4kHz甚至更高。作为对比，人类小脑处理运动指令的延迟大约是100ms——机器人比人快了100倍。

避坑指南：很多新手在设计控制循环时，只关注了"功能正确"而忽略了"时序确定"。我曾经见过一个移动机器人项目，SLAM线程和规划线程之间的数据传递用的是共享内存加互斥锁，结果在高负载下控制循环偶尔被阻塞长达50ms——机器人就会"愣"一下然后撞墙。记住：机器人系统里，确定性的延迟比不正确的数据更致命。我们会在第14章详细讨论实时控制策略。

1.5 2026机器人学新范式

如果你在2020年问我"机器人学该怎么学"，我会告诉你从数学、运动学、动力学、控制理论一步步来。但在2026年的今天，这个答案已经被改写了。三个新范式正在重塑整个领域。

第一，VLA大模型打破模块化管线。传统的Sense-Plan-Act是一个串行管线，每个模块独立设计、独立调试。而2026年的VLA模型（如Google的RT-2、NVIDIA的Isaac GR00T）将视觉感知、语言理解和动作生成统一到一个端到端的神经网络中。你只需要告诉机器人"把这颗螺丝拧到面板上"，VLA模型就会直接输出关节角度序列，不需要显式的运动学建模和路径规划。这不是替代传统方法，而是为传统方法增加了一个新的"上层接口"。

第二，Sim-to-Real让仿真成为第一公民。过去仿真只是"锦上添花"，现在的流程已经变成了：在Isaac Sim或MuJoCo中大规模训练策略 → 应用域随机化（Domain Randomization） → 零样本迁移到真机。2026年，宇树H2 Plus的走路策略就是在Isaac Sim中训练了相当于200年实际时间的虚拟经验后，直接部署到硬件上的，第一次行走就成功了。

第三，具身智能（Embodied Intelligence）成为主流。机器人不再"被编程"，而是"自主学习"。HuggingFace的LeRobot社区已经积累了数千种机器人操作数据集，OpenVLA提供了开源的视觉-语言-动作模型。这意味着，你不需要从零开始写运动学代码——但你必须理解运动学原理，才能知道预训练模型在什么情况下会失败。

关键洞察：在2026年学习机器人学，我给你一个建议——"双脚走路"：一只脚踩在经典理论（运动学、动力学、控制、SLAM）上，这是你的安全网；另一只脚踩在AI新范式（VLA、Sim-to-Real、模仿学习）上，这是你的加速器。两条腿缺一不可。
