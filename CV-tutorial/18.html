两阶段 vs 单阶段——精度 vs 速度的经典权衡

2015年是目标检测的"奇迹之年"。上半年，Ross Girshick在Fast R-CNN中解决了重复计算问题；下半年，两个独立的研究组从完全不同的方向同时突破了检测的速度瓶颈：一个是Ren Shaoqing等人的Faster R-CNN（RPN+Roi Pooling的两阶段路线），另一个是Joseph Redmon等人的YOLO（You Only Look Once，单阶段路线）。这两条路线从诞生之初就带着鲜明的哲学分歧。

让我用一个形象的比喻来理解两者的区别。想象你是一个侦探，要在人群中找出所有小偷：两阶段方法就像"先拿望远镜扫一遍，圈出几个可疑人物，再逐个盘问核实"。第一阶段（RPN）快速筛选候选项，保召回但不求精确——宁可抓错也不能放过；第二阶段（分类头）仔细鉴别，降误检但提升精度。而单阶段方法就像"直接在人群中大喊一声——谁是小偷给我站出来"——一步到位，但早期难免误伤好人。

维度两阶段（Faster R-CNN为代表）单阶段（YOLO/SSD为代表）
核心流程候选区域提议 → 分类回归密集采样 → 直接预测
速度中（10-20 FPS on VGG16）快（30-150+ FPS）
精度高（尤其小物体）早期低，后期（YOLOv4+）追平
小物体检测更强（RoI Pooling精确定位）偏弱（特征图分辨率限制）
训练难度较高（多任务、交替训练）较低（单一损失函数）
正负样本平衡自然平衡（RPN筛选后）极端不平衡（背景远多于前景）

这张表反映了2015-2019年间的典型状况。但值得注意的是，两阶段和单阶段的精度差距在持续缩小。2017年RetinaNet的提出是一个转折点——它通过Focal Loss解决了单阶段检测的正负样本不平衡问题，首次在单阶段框架上达到了与两阶段方法相当的精度。2020年的YOLOv4和2021年的YOLOv5更是在精度上全面超越了同期的Faster R-CNN变体。到2025年的今天，"两阶段精度高、单阶段速度快"这句老话已经不再适用了。

🔍 深层分析：两阶段和单阶段的本质区别不在于"有没有RPN"，而在于正负样本的处理策略。两阶段方法通过RPN的初步筛选，将候选区域的数量从几万降到几百，同时将正负样本比例从1:1000+改善到大约1:3——分类器处理的是一个相对平衡的问题。单阶段方法直接在原始采样上做预测，面临极端不平衡问题（一张图中负样本区域远多于正样本区域）。Focal Loss的成功揭示了：只要解决了样本不平衡问题，单阶段方法完全可以达到与两阶段相当的精度。

YOLO系列核心思想

YOLO（You Only Look Once）的核心理念可以浓缩为一句话：将目标检测重新定义为回归问题。它不再把检测拆分为"候选区域+分类"两个步骤，而是用一个单一的神经网络直接从图像像素回归到边界框坐标和类别概率。

网格划分与预测

YOLO将输入图像划分为S×S的网格（在YOLOv1中是7×7）。每个网格单元负责检测中心点落在该网格内的物体。每个网格预测B个边界框（YOLOv1中B=2），每个框包含5个参数：(x, y, w, h, confidence)。同时，每个网格还要预测C个类别的条件概率。因此，YOLOv1的输出是一个S×S×(B*5 + C)的张量——在VOC数据集（20类）上就是7×7×30。

# YOLOv1的输出张量结构（S=7, B=2, C=20）
# 形状：(7, 7, 30)
# 对每个网格单元 (i, j)：
#   output[i, j, 0:5]   → 第一个边界框 (x, y, w, h, confidence)
#   output[i, j, 5:10]  → 第二个边界框 (x, y, w, h, confidence)
#   output[i, j, 10:30] → 20个类别的条件概率

# 最终检测得分：
# score(c, box) = confidence(box) × class_probability(c | grid)
#
# 其中confidence定义为：
# confidence = Pr(Object) × IoU(pred, truth)
# 如果网格内没有物体 → confidence应为0
# 如果有物体         → confidence应为预测框与真实框的IoU

YOLO的设计哲学是"简单粗暴"的：整个网络就是一个卷积神经网络，结构上类似GoogLeNet（24个卷积层 + 2个全连接层），没有任何RPN、RoI Pooling之类的中介组件。输入一张图，前向传播一次，直接得到所有检测结果。

YOLO的优缺点

YOLO的最大优势是速度。YOLOv1在Titan X GPU上可以达到45 FPS，轻量版Fast YOLO甚至能达到155 FPS。但它的代价也很明显：

  • 网格大小限制：7×7的网格太粗了，每个网格只能检测中心点落在其中的物体。两个小物体挨得很近时，如果它们的中心落在同一个网格中，YOLO只能检测其中一个（因为每个网格最多预测2个框）。
  • 尺度泛化差：YOLO对训练数据中不常见的长宽比物体检测效果不佳——因为边界框的预测是直接从全连接层学出来的，没有锚框的先验约束。
  • 定位精度不足：尤其是小物体的定位误差较大。YOLOv1的定位误差是Faster R-CNN的2-3倍。

后来的YOLOv2/v3/v4/v5/v7/v8通过引入锚框、多尺度预测、FPN、CSP结构、Mish激活函数、自注意力机制等不断改进，解决了上述大部分问题。到YOLOv8，它已经成为工业界最广泛使用的检测框架之一。

💡 YOLO社区的一个有趣现象：YOLO家族的版本号远比论文数量"丰富"。Joseph Redmon在YOLOv3之后因担忧军事应用而退出了CV研究，之后YOLO的发展由社区接管——出现了YOLOv4（Alexey Bochkovskiy）、YOLOv5/v8（Ultralytics、Glenn Jocher）、YOLOv6（美团）、YOLOv7（Wang Chien-Yao）等多个独立分支。每个分支都有自己的特色，但核心思想都继承自最初的YOLO。这让YOLO成了检测领域中最"热闹"的家族，但也给初学者带来了选择困难。

SSD核心思想

在YOLO发布后不到四个月，Wei Liu等人提出了SSD（Single Shot MultiBox Detector）。SSD的设计目标很明确：保留YOLO的单阶段速度优势，同时通过多尺度特征图检测来弥补YOLO在小物体上的精度短板。

多尺度特征图检测

SSD的核心创新是：在不同分辨率的特征图上分别进行检测。低层特征图分辨率高，感受野小，适合检测小物体；高层特征图分辨率低，感受野大，适合检测大物体。SSD从VGG16的conv4_3开始，在后面又接了多个逐渐变小的卷积层（conv7, conv8_2, conv9_2, conv10_2, conv11_2），在这6个不同尺度的特征图上都设置检测器。

特征图层特征图尺寸每个位置默认框数适合检测的物体尺度
conv4_338×384小物体
conv7 (fc7)19×196中等偏小
conv8_210×106中等
conv9_25×56中等偏大
conv10_23×34大物体
conv11_21×14特大物体

默认框（Default Boxes，即锚框）在每个特征图层上的尺度和长宽比根据该层的感受野大小来设计。SSD共生成8732个默认框，覆盖了不同的尺度和长宽比。

SSD vs YOLO

SSD和YOLO虽然都是单阶段方法，但设计哲学有明显差异：

  • 框的预测方式：YOLOv1没有锚框，直接回归绝对坐标；SSD基于锚框回归偏移量，借鉴了Faster R-CNN的做法。
  • 多尺度策略：YOLOv1仅用最后一个特征层做检测；SSD在6个不同尺度的特征图上分别检测，极大改善了小物体检测。
  • 正负样本策略：SSD使用hard negative mining（保持正负样本比例1:3），YOLO没有显式的正负样本平衡策略。
  • 性能对比（VOC 2007）：SSD300（74.3% mAP）比YOLOv1（63.4% mAP）高约11个点，同时速度相当（46 FPS vs 45 FPS）。

这说明了一个重要道理：单阶段不等于"不用锚框"——SSD和后来的YOLOv2都从Faster R-CNN的anchor机制中汲取了营养，锚框设计已经成为检测方法的"通用语言"。

DETR与Transformer检测

2020年，Facebook AI团队在ECCV上发表了DETR（Detection Transformer），以一种完全不同的方式重新思考了目标检测。DETR的核心洞察是：目标检测本质上是一个集合预测问题，而不是回归+分类问题。它提出了一个颠覆性的问题：我们真的需要锚框、NMS、RoI Pooling这些精心设计的组件吗？能否用一个纯粹的端到端框架解决检测？

DETR的工作原理

DETR的架构由三部分组成：CNN骨干（提取特征）→ Transformer编码器-解码器（建模全局关系）→ FFN（预测边界框和类别）。它的核心创新在于：

  • Object Queries：可学习的N个查询向量（通常N=100），作为Transformer解码器的输入。每个query负责预测一个物体。训练过程中，这些query会"学会"关注图像中不同位置和不同类别的物体。
  • 匈牙利匹配损失（Hungarian Loss）：DETR将预测结果与真实标注做二分图最优匹配（用匈牙利算法），然后计算匹配后的损失。这使得DETR不需要NMS——因为匈牙利匹配本身就已经解决了"哪个预测对应哪个真实框"的分配问题。
  • 移除所有手工组件：不需要锚框设计、不需要NMS阈值调参、不需要RoI Pooling。DETR的设计简单到了极致。
# DETR的推理流程（伪代码）
def detr_inference(image, model, num_queries=100):
    # 1. CNN骨干提取特征
    features = cnn_backbone(image)  # 降采样32倍

    # 2. Transformer编码器（捕获全局上下文）
    memory = transformer_encoder(features + positional_encoding)

    # 3. Transformer解码器 + Object Queries
    #    100个可学习的query向量，每个负责"寻找"一个物体
    queries = learnable_queries  # shape: (100, d_model)
    decoder_output = transformer_decoder(memory, queries)

    # 4. 预测头
    logits = class_head(decoder_output)   # (100, num_classes)
    boxes = bbox_head(decoder_output)     # (100, 4) — 归一化坐标

    # 5. 不需要NMS！
    # 匈牙利匹配已经在训练时教会了queries互不重叠地检测不同物体
    return logits, boxes

DETR的优劣势

DETR的优雅性令人赞叹，但它也有自己的问题：

  • 训练收敛慢：DETR需要500个epoch才能收敛（Faster R-CNN通常只需要36个epoch）。Transformer的注意力机制需要较长时间才能学会"每个query关注不同位置"。
  • 小物体检测差：Transformer的全局注意力对小物体的细节不够敏感。后来的Deformable DETR通过可变形注意力机制解决了这个问题。
  • 推理速度慢：DETR的推理速度大约在10-15 FPS，远不如YOLO系列。
  • 虽然不用NMS，但训练中匈牙利的计算成本较高。
⚠️ 部署踩坑：我在DETR刚开源时就尝试将其用于工业零件的缺陷检测。虽然DETR不需要调NMS参数很省心，但它的100个query意味着每张图固定输出100个预测——其中大部分是"no object"。在推理时需要设置一个置信度阈值来过滤空预测，而这个阈值的敏感度远高于NMS阈值：低一点就一大堆假阳性，高一点就漏检。DETR只是把"NMS阈值调参"换成了"置信度阈值调参"——并非完全没有超参数。

检测模型选型指南

面对如此多的检测方法，实际项目中该如何选择？这里给出一个基于场景的选型建议：

场景推荐模型原因
实时视频分析（30+ FPS）YOLOv8/v11, RT-DETR速度优先，精度可接受
高精度工业质检（小缺陷）Faster R-CNN + FPN小物体检测能力强，可定制化程度高
移动端/嵌入式YOLOv8-nano, NanoDet模型体积小，推理速度快
医学影像Faster R-CNN / RetinaNet精度优先，且通常无需实时
通用场景（精度速度均衡）YOLOv8/v11, RT-DETR近年来最好的精度-速度权衡
学术研究/快速原型DETR / Deformable DETR简洁干净的框架，易于修改扩展
密集小物体（如细胞、人群）Faster R-CNN + FPN + 多尺度训练两阶段方法的精确定位优势

需要强调的是：没有"最好"的检测器，只有"最适合"你的检测器。选择模型时，不仅要看公开benchmark上的mAP，还要考虑你的部署环境（GPU/CPU/边缘设备）、实时性要求、标注数据量、以及团队对特定框架的熟悉程度——后者在实际工程中往往是最重要的因素。

两阶段与单阶段的融合趋势

前面我们花了很多篇幅讨论两阶段和单阶段的差异，但最新的研究趋势告诉我们：这个区分正在变得模糊。最先进的检测器已经从两派的做法中各取所长：

  • YOLO系列中的"两阶段化"：YOLOv3引入了类似FPN的多尺度预测；YOLOv4引入了CSP结构和PANet（路径聚合网络），本质上是"让信息在多个尺度的特征之间流动"——这与两阶段方法中的FPN异曲同工。YOLOv5/v8进一步引入了Task-Specific Decoupling Head（解耦头），将分类和回归分支分开——这其实在模仿两阶段方法的"分工明确"。
  • Faster R-CNN中的"单阶段化"：RetinaNet基本去掉了RPN的第二阶段，直接在RPN级别的密集采样上用Focal Loss做分类——这本质上就是一个单阶段检测器。后来的Sparse R-CNN更进一步，用可学习的proposal代替RPN的密集锚框，大大缩小了两阶段和单阶段在"速度"上的差距。
  • Transformer的统一：DETR和它的后继者（Deformable DETR、DINO、RT-DETR）提供了一个全新的框架，既不是严格意义上的两阶段也不是单阶段。它们证明了：注意力机制可以替代RPN+NMS+RoI Pooling的所有功能。RT-DETR甚至实现了比YOLO更快的推理速度，同时保持了匹配甚至更高的精度。
🔮 未来展望：检测方法的未来很可能不再是"两阶段vs单阶段"的争论，而是"端到端检测"的全面胜利——DETR家族的query-based范式展示了一条清晰的道路。但卷积时代的遗产（锚框、NMS、FPN）不会完全消失，它们以各种形式融入了新的框架。理解这些经典方法的设计思路，就是理解检测领域最本质的智慧——如何在"在哪找"和"是什么"之间优雅地分配计算资源。

至此，我们关于目标检测的四章内容就全部结束了。你从检测的基础概念开始，一路走过了R-CNN的开山之作、Faster R-CNN的区域提议革命、以及单阶段方法的百花齐放。你应该已经掌握了目标检测的核心知识体系——从边界框表示、IoU、mAP这些基础，到两阶段/单阶段/端到端三大流派的核心思想。
