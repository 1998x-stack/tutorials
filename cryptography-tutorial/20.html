第20章 同态加密

想象这样一个场景：你把自己最重要的文件锁进了一个保险箱，然后把保险箱交给一家公司，让他们帮你"处理"这些文件——计算其中某些数据的平均值、搜索特定关键词、甚至生成一份完整的分析报告。关键问题是：这家公司全程无法打开保险箱，看不到文件内容，但却能完美地完成所有操作。这听起来像魔法吗？这就是同态加密要解决的问题——对加密数据进行计算，而无需解密。

同态加密（Homomorphic Encryption, HE）被誉为密码学界的"圣杯"之一。如果完全同态加密能够高效实现，它将彻底改变我们对数据所有权的认知——你可以把计算任务外包给云服务器，而服务器在完全不接触原始数据的情况下完成计算。这在云计算、医疗数据分析和隐私保护机器学习中有着不可替代的价值。

20.1 部分同态加密

在讨论完全同态加密之前，我们需要先理解一个更简单的概念：部分同态加密。有些加密方案天生就具有某种代数结构，允许对密文执行特定的运算。

乘法同态：RSA

你可能没有意识到，我们在第 9 章学过的 RSA 算法实际上就具备部分同态性。回忆 RSA 的加密过程：E(m) = mᵉ mod N。现在观察两个密文相乘会发生什么：

E(m₁) · E(m₂) = (m₁ᵉ · m₂ᵉ) mod N
               = (m₁ · m₂)ᵉ mod N
               = E(m₁ · m₂)

也就是说，对两个 RSA 密文做乘法，等价于对对应明文做乘法后再加密。RSA 是乘法同态的。但 RSA 对加法不是同态的——你不能通过操作密文来得到两个明文的加和。

加法同态：Paillier

1999 年，Pascal Paillier 提出了一种对加法同态的公钥密码方案。Paillier 加密的密文相乘对应的是明文的加法：

// Paillier 加密: E(m) = gᵐ · rⁿ mod n²
// 其中 n = p·q, g 是生成元, r 是随机数
//
// 同态性质:
// E(m₁) · E(m₂) = E(m₁ + m₂ mod n)
// E(m)ᵏ = E(k · m)

这一性质在电子投票系统中非常有用：投票者每人加密自己的投票（0 或 1），计票员只需将所有密文相乘，就能得到总票数的密文，再解密即可。整个过程不暴露任何单个选民的投票倾向。

方案 同态类型 运算 安全性假设 典型用途
RSA 乘法同态 E(m₁)·E(m₂) = E(m₁·m₂) RSA 假设 密钥交换（非安全用于同态）
ElGamal 乘法同态 E(m₁)·E(m₂) = E(m₁·m₂) DDH 假设 密文投票系统
Paillier 加法同态 E(m₁)·E(m₂) = E(m₁+m₂) 合数剩余假设 电子投票、隐私支付
Goldwasser-Micali 异或同态 E(b₁)·E(b₂) = E(b₁⊕b₂) 二次剩余假设 比特加密

20.2 从部分同态到全同态

部分同态加密只能支持一种类型的运算（加法或乘法）。但现实中我们需要的是同时支持加法和乘法的方案——因为任何计算都可以用加法和乘法组合来表达。这就引出了两个中间概念：

Somewhat Homomorphic Encryption（SHE，适度同态加密）：可以同时支持加法和乘法，但有次数限制。比如你可以做 10 次加法和 3 次乘法，之后噪声积累得太多，解密就会失败。这是为什么会出现"噪声"问题——同态加密方案中，每次运算都会在密文中引入微小的噪声，乘法产生的噪声比加法增长快得多。

层次型同态加密（Leveled HE）：预先指定一个最大乘法深度，然后选择足够大的参数来保证在该深度内的任意计算都能正确解密。这是 SHE 到 FHE 之间的实用折中方案。

💡 噪声问题：大多数同态加密方案都在密文中嵌入了一个小噪声来保证安全性。解密成功的条件是噪声小于某个阈值。加法会使噪声线性增长，而乘法会使噪声以乘积方式暴增。如果噪声超过阈值，解密就会得到无意义的结果。这就好比你在一个精确的天平上加砝码——每次操作都会造成微小的误差，积累到一定程度天平就不再可信了。

20.3 Craig Gentry 的革命性突破

在 2009 年之前，全同态加密被认为是一个遥不可及的目标。虽然科学家们尝试了 30 多年，但始终没有人能构造出同时支持加法和乘法且没有运算次数限制的方案。直到 2009 年，一个名叫 Craig Gentry 的 IBM 研究员在他的博士论文中做出了划时代的贡献——第一个全同态加密（Fully Homomorphic Encryption, FHE）方案。

Gentry 的核心思想令人拍案叫绝。他提出：既然噪声是限制运算次数的根源，那能不能在噪声积累到危险水平之前就"清除"它？但问题是——要清除噪声，你需要解密；而解密需要密钥，你不想把密钥给服务器。Gentry 的解决方案是一种称为 Bootstrapping（自举） 的技术：

// Bootstrapping 的核心思想
//
// 给定一个密文 c（噪声已接近阈值），
// 以及加密后的私钥 Enc(sk)：
//
// 1. 将解密电路本身作为同态运算来执行
// 2. 在密文上计算 Dec(c, sk)，
//    但 sk 被替换为 Enc(sk)
// 3. 结果是解密后的明文 m 的新的加密密文 c'
// 4. 关键：c' 的噪声被"重置"为一个更小的值！
//
// 这就像在脏数据上执行一次"加密的"清洗操作，
// 而清洗工自己都不知道数据内容。

用更直观的方式理解：想象你有一张沾满污渍的照片，你想清洗它，但你不能直接看照片内容。Bootstrapping 就是让你戴上一副特殊的"加密手套"，可以在不看照片的情况下完成清洗操作——而且洗完后照片变得更清晰了。

🔑 Gentry 的重大意义：2009 年 Gentry 的博士论文《A Fully Homomorphic Encryption Scheme》被公认为 21 世纪最重要的密码学贡献之一。它证明了全同态加密是理论上可行的。虽然最初的方案远达不到实用的效率（一个比特的运算需要 30 分钟！），但它打开了通向密码学新世界的大门。Gentry 也因此获得了 2010 年的 ACM 博士论文奖和 2022 年的突破奖（数学/计算机科学领域）。

20.4 现代 FHE 方案

Gentry 启发了后续一大批 FHE 方案的诞生。当前最主流的方案包括：

方案 提出年份 基于 核心运算 特点
BGV 2011 LWE/RLWE 整数多项式环 层次型，支持模切换（Modulus Switching）
BFV 2012 RLWE 整数多项式环 比例编码，适合整数运算
CKKS 2017 RLWE 浮点数近似运算 支持浮点数和定点数，适合 ML
TFHE 2016 LWE 布尔电路 极快的 bootstrapping（~0.1s）

CKKS 方案尤为重要，因为它支持在加密数据上执行近似浮点运算——这对机器学习来说是致命的特性。在 CKKS 中，你可以把多个值编码到一个多项式里，利用 SIMD（单指令多数据）风格执行并行运算。这意味着你可以同时加密一批数据，在密文上完成矩阵乘法、神经网络推理等操作。

20.5 同态加密的应用与挑战

经过十几年的发展，同态加密已经从纯理论走向了初步的工程实践。但它的道路仍然充满挑战：

  • 隐私保护的机器学习：医疗机构可以在加密的 MRI 图像上运行 AI 诊断模型，获得诊断结果的同时保护患者隐私。例如，iDASH 隐私保护竞赛中已经出现了使用 CKKS 进行基因组数据分析和疾病预测的方案。
  • 加密数据库查询：企业可以将加密的客户数据存储在云端，然后对密文数据库执行搜索和聚合操作。服务器看不到任何原始记录，但仍然能返回正确的查询结果。
  • 金融隐私计算：多家银行可以在互相不暴露客户信息的前提下，通过 HE 计算联合风控模型，检测跨行的洗钱行为。
⚠️ 性能挑战：这是同态加密面临的最大障碍。与明文计算相比，FHE 的开销通常是几个数量级。一个在明文上只需几毫秒的运算，在 FHE 上可能需要几分钟甚至几小时。具体而言：
  • 密文膨胀：一个 64 位整数的密文可能占用几千字节
  • 计算慢：多项式环上的乘法需要 NTT（数论变换），比明文乘法的开销高出 10³ - 10⁶ 倍
  • Bootstrapping 昂贵：虽然 TFHE 把 bootstrapping 降到了亚秒级，但这仍然是 FHE 最耗时的操作
如果效率不是你的首要考量——比如离线批量处理——那 FHE 已经可以用了。但如果你想做实时推理，还需要等待硬件加速和算法优化的进一步突破。
💡 性能提升路径：近年来，FHE 的性能正在快速改善。硬件加速方面，Intel 的 HEXL 库利用 AVX-512 指令集加速多项式运算；FPGA 上的 FHE 加速器也已达到吞吐量的百倍提升。算法方面，自举技术的改进（如 GINX 和 LMK+ 自举变体）使得 bootstrapping 速度不断提升。虽然离"透明"还有距离，但趋势明显是向好的。

同态加密让我们做到了信息安全领域最违反直觉的事情之一：在不看数据的情况下处理数据。从 Gentry 2009 年的理论突破到今天初步可用的工程库（Microsoft SEAL、IBM HElib、Intel HEXL），这条路走了十五年。尽管性能瓶颈仍然存在，但随着硬件加速和算法优化的不断进步，FHE 正在慢慢走向实用化。下一章，我们将学习另一种"多方协作"的隐私计算范式——安全多方计算，它解决的是另一个根本问题：多个互不信任的参与方如何共同计算一个函数，且各自不泄露自己的输入。
