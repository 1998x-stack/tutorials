大家好！过去三章我们深入学习了 RRT、RRT* 和 PRM 这三大采样方法。现在是时候把它们——以及它们的变体——放在同一个竞技场上进行公平对决了。这是"核心算法篇"的收官之战，也是本书最重要的方法论总结之一。

理解采样方法之间的优劣取舍，比死记硬背任何一个算法都更重要。因为在实际工程中，你很少会说"我要跑 PRM"——而是会说"我需要一个能在 2 秒内找到路径、容忍路径不是最优但至少不太离谱、且能处理 6 维 C-space 的方法"——然后你才能推算出该用什么。

统一评价框架

公平比较的前提是统一标准。在我们的对比实验中，所有算法共享以下设置：

  • 相同地图——三张标准测试地图（窄通道、密集障碍物、开阔空间），每张地图 200x200
  • 相同时间预算——每张地图允许 10 秒规划时间，记录不同时刻的最优路径代价
  • 相同指标——首次找到路径的时间、最终路径代价、50 次运行的成功率、展开节点数
  • 相同实现——所有算法的碰撞检测、最近邻搜索（KDTree）、steer 函数共享同一套底层实现

这里要特别强调"相同底层实现"的重要性。很多论文声称"我们的算法比 A* 快 100 倍"——仔细一看，是用 Python 的 A* 和 C++ 的 PRM 做比较。这种不公平的比较在路径规划领域屡见不鲜，我们务必避免。

基准测试脚本

import time
import numpy as np
# 假设上述算法的实现都在各自模块中
from rrt import RRT
from rrt_star import RRTStar
from informed_rrt_star import InformedRRTStar
from prm import PRM
from bit_star import BITStar

def benchmark(maps, algorithms, n_runs=10, time_budget=10.0):
    """对多张地图运行多种算法，收集指标"""
    results = {}

    for map_name, (obstacle_check, bounds, start, goal) in maps.items():
        results[map_name] = {}
        for algo_name, AlgoClass in algorithms.items():
            run_data = []
            for _ in range(n_runs):
                planner = AlgoClass(start, goal, obstacle_check, bounds)
                start_time = time.time()
                path = None
                best_cost = float('inf')
                # 持续搜索直到时间耗尽或成功
                while time.time() - start_time < time_budget:
                    path = planner.plan()
                    if path is not None:
                        cost = sum(np.linalg.norm(path[i+1] - path[i])
                                   for i in range(len(path) - 1))
                        if cost < best_cost:
                            best_cost = cost
                    # 继续改进（对于 RRT* 等 Anytime 方法）
                    if not hasattr(planner, 'improve'):
                        break

                elapsed = time.time() - start_time
                run_data.append({
                    'success': path is not None,
                    'cost': best_cost if best_cost != float('inf') else None,
                    'time': elapsed,
                    'nodes': len(planner.nodes) if hasattr(planner, 'nodes') else None,
                })
            results[map_name][algo_name] = run_data

    # 汇总统计
    for map_name in results:
        print(f"\n=== {map_name} ===")
        for algo_name in results[map_name]:
            costs = [r['cost'] for r in results[map_name][algo_name] if r['success']]
            successes = sum(1 for r in results[map_name][algo_name] if r['success'])
            avg_cost = np.mean(costs) if costs else float('inf')
            print(f"  {algo_name}: success={successes}/{n_runs}, "
                  f"avg_cost={avg_cost:.2f}")

if __name__ == '__main__':

选型决策树

清晰选型决策流程：

Step 1：环境是静态还是动态？
  → 动态：使用 RRT 或 RRT-Connect（单次查询，不需要构建全局图）
  → 静态：进入 Step 2

Step 2：需要多次查询还是单次查询？
  → 多次查询（如仓库机器人每天规划万次）：使用 PRM
  → 单次查询（如机械臂抓取一次）：进入 Step 3

Step 3：需要最优路径吗？
  → 需要最优（最优性非妥协）：进入 Step 4
  → 不需要最优（"能走就行"）：使用 RRT 或 RRT-Connect（最快）

Step 4：对收敛速度有要求吗？
  → 时间充裕（秒级）：使用 RRT*
  → 时间紧张（毫秒级）：使用 Informed RRT*（椭球加速）
  → 需要同时保证多次查询最优：使用 BIT*（批处理采样+启发式搜索）

终极建议：如果不确定，先跑 RRT-Connect。它实现简单、速度快、覆盖面广。然后用 Informed RRT* 提升路径质量。只有在你确认 "环境完全静态且查询频繁" 时，才投入 PRM 的建图成本。

混合策略——超越单个算法

在真实系统中，最好的方法往往不是"选择"一个算法，而是"组合"多个算法。下面是一些经过验证的混合方案：

RRT + 轨迹优化

先用 RRT 快速找到一条可行路径（虽然粗糙），然后用轨迹优化方法（如 CHOMP 或梯度下降）对路径进行平滑。这种"粗糙初解 + 数值优化"的组合在机械臂规划中非常流行。RRT 负责提供拓扑正确的路径（穿越正确的障碍物间隙），优化负责在局部修正路径的平滑度。

PRM + RRT* 混合

在 PRM 构建的路线图上，每个"边"并不是一个简单连接——而是一个 RRT* 子段。这样路线图既有了 PRM 的多查询能力，又有 RRT* 的路径优化能力。代价是实现复杂度和计算开销都显著增加。

基于学习的采样

近年来兴起的方法：用神经网络学习采样分布。传统 RRT 用均匀采样，但实际环境中的自由空间分布远非均匀。通过学习历史成功路径的分布模式，训练一个条件变分自编码器（CVAE）或扩散模型来生成更有"针对性"的采样点。这类方法在特定场景下可以比均匀采样快 5-10 倍，但泛化到新环境仍是挑战。
