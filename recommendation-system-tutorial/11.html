第11章 DIN：深度兴趣网络

看到这里，你可能会产生一个感觉：DeepFM、DCN、xDeepFM……这些模型的核心都在做同一件事——特征交叉。它们把用户特征、物品特征、上下文特征糅合在一起，通过各种精巧的结构让特征之间产生交互。

但这里有一个被严重忽略的问题。假设一个用户的历史行为是：昨天点击了"iPhone 15手机壳"，上周看了"MacBook Pro评测"，一个月前搜索了"AirPods 3"。现在我们要评估他对两个候选物品的点击概率——一个是"透明手机壳"，一个是"机械键盘"。DeepFM会怎么做？它把用户的所有历史行为揉成一个固定的embedding向量，然后用这个同一的向量去和两个候选物品做交叉。

这合理吗？显然不合理。用户对"手机壳"的兴趣应该主导对"透明手机壳"的预测，而对"手机"和"电脑"的兴趣和"机械键盘"的关系不大。但传统模型不区分这些——它们用一个"平均脸"去应对所有候选物品。

本章的DIN（Deep Interest Network）正是要解决这个问题。

1. 用户兴趣不是静态向量

传统推荐模型的用户表示方式可以概括为：将用户的所有行为特征（点击过的物品ID、浏览过的类别、搜索过的关键词等）通过embedding和pooling，压缩成一个固定长度的向量。这个向量代表了用户的"全局兴趣"。

但问题在于：用户的兴趣是多维度的、与候选物品相关的。你对"手机配件"的兴趣不应该影响对你推荐"游戏外设"的决策。

阿里巴巴在发表DIN论文时分享了一个极其生动的例子。在淘宝的推荐场景中，他们发现传统模型（如DeepFM）有强烈的"同类别过度推荐"倾向——一个用户点击了某款连衣裙，模型就会疯狂推荐所有连衣裙。为什么？因为用户embedding被所有行为"平均化"了，而连衣裙行为在embedding中留下的痕迹会"传染"到所有推荐候选上。

真实故事：阿里巴巴的发现

DIN团队在分析淘宝用户行为数据时发现了一个有趣的模式：一位女性用户在过去一周点击了"泳衣""防晒霜""旅行箱"和"婴儿奶粉"。传统模型会将她压缩成一个"泛泛的母婴/旅行"向量，在推荐时对所有候选物品使用同一个表示。但仔细想想——她对"泳衣"的兴趣和"婴儿奶粉"很可能来自完全不同的生活场景（给自己 vs 给孩子）。DIN的核心贡献就是让模型针对"泳衣"的候选，主要从"泳衣"相关的历史行为中构建用户表示；针对"奶粉"的候选，则主要从"奶粉"相关的历史行为中构建用户表示。

这个发现直接引出了DIN的核心思想：用户的表示应该是自适应的（adaptive），应该依赖于当前候选物品。

2. DIN的核心：自适应兴趣表示

DIN的解决方案是引入注意力机制（Attention Mechanism）。对于每一个候选物品，DIN会在用户的历史行为序列上计算注意力权重——哪些历史行为和当前候选更相关，就给更高的权重。

具体来说，DIN的用户兴趣表达计算如下：

给定用户的历史行为序列 {b_1, b_2, ..., b_n} 和候选物品向量 e_c：

  • 对每个行为 b_i，计算其与候选物品 e_c 的相关性得分：a_i = Attention(b_i, e_c)
  • 用 a_i 对行为序列做加权求和：v_u = Σ a_i · b_i
  • 这个 v_u 就是针对候选物品 e_c 的"自适应用户兴趣表示"

注意力得分的计算方式是DIN的精华之一。它使用一个MLP来计算相关性，而不是简单的点积：

a(b_i, e_c) = MLP([e_i, e_c, e_i ⊙ e_c, e_i - e_c])

注意输入包含了四个部分：行为embedding e_i、候选embedding e_c、逐元素乘积 e_i ⊙ e_c、逐元素差 e_i - e_c。为什么要这么复杂？因为点积只能建模"e_i 和 e_c 的方向一致性"，而MLP的输入中包含了 e_i 和 e_c 的多种交互模式——模型可以学到"e_i 的哪些维度大、哪些维度小"与候选物品的关系。

下面是DIN注意力单元的PyTorch实现：

import torch
import torch.nn as nn
import torch.nn.functional as F

class AttentionUnit(nn.Module):
    """DIN的自适应注意力单元"""
    def __init__(self, embed_dim, hidden_units=[80, 40]):
        super().__init__()
        layers = []
        in_dim = 4 * embed_dim  # concat([e_i, e_c, e_i*e_c, e_i-e_c])
        for h in hidden_units:
            layers.append(nn.Linear(in_dim, h))
            layers.append(nn.ReLU())
            layers.append(nn.BatchNorm1d(h))
            in_dim = h
        layers.append(nn.Linear(in_dim, 1))
        self.mlp = nn.Sequential(*layers)

    def forward(self, behavior_embeds, candidate_embeds):
        """
        behavior_embeds: (batch, seq_len, embed_dim)
        candidate_embeds: (batch, 1, embed_dim) 广播到所有行为
        """
        # 扩展候选到序列长度
        cand = candidate_embeds.expand_as(behavior_embeds)

        # 四种交互模式的拼接
        concat_input = torch.cat([
            behavior_embeds,
            cand,
            behavior_embeds * cand,          # 逐元素乘积
            behavior_embeds - cand            # 逐元素差
        ], dim=-1)  # (batch, seq_len, 4*embed_dim)

        # 通过MLP得到注意力得分
        # 先 reshape 为 (batch*seq_len, 4*embed_dim)
        batch, seq_len, dim = concat_input.shape
        scores = self.mlp(concat_input.reshape(-1, dim))
        scores = scores.reshape(batch, seq_len, 1)

        # Softmax归一化
        weights = F.softmax(scores, dim=1)
        return weights

整个DIN模型的结构是：Embedding层 → Attention Pooling → 拼接其他特征 → DNN → 输出。注意力加权后的用户行为向量、用户profile特征、物品特征、上下文特征拼接在一起，送入后续的DNN做最终预测。

3. Mini-batch Aware正则化

DIN面临一个巨大的工程挑战：用户行为序列中的物品ID空间非常大（淘宝的商品SKU以亿计）。标准的L2正则化需要对所有参数的平方求和——这在亿级embedding上几乎不可行。

DIN的解决方案是Mini-batch Aware Regularization。核心思想很简单：在每次mini-batch更新时，只对当前batch中出现过的特征对应的参数施加正则化。

数学上，对于特征 j，其正则化项乘以一个指示变量 I_j——如果特征 j 在当前batch中出现，I_j=1，否则 I_j=0。这样，每个batch只需要正则化一小部分参数。

这个trick的效果非常显著：

  • 在淘宝的数据集上，全量L2正则化需要遍历数十亿参数，而mini-batch aware正则化每次只处理数万个参数
  • 训练速度提升了不止一个数量级
  • 同时保留了正则化的核心效果——防止过拟合

工业小贴士：Mini-batch Aware正则化的思想可以推广到几乎所有使用大规模embedding的推荐模型中。如果你的模型训练时embedding部分占了绝大多数参数，但样本是高度稀疏的，强烈建议实现这个正则化策略。具体实现时，维护一个"当前batch的特征集合"，然后只对这个集合中的embedding施加正则化。

4. 从论文到工业落地

DIN论文公布了在阿里巴巴展示广告场景的线上A/B测试结果：CTR提升了10%，RPM（千次展示收入）提升了3.8%。这在广告系统中是一个非常显著的提升，既提升了用户体验（更高的CTR），也提升了平台收入（更高的RPM）。

但DIN的落地并不总是一帆风顺。有几件事需要特别注意：

  • 行为序列长度：DIN的注意力计算复杂度是 O(seq_len · embed_dim)。对于超长序列（几百甚至上千），计算量和内存占用会急剧上升。工业实践中通常截断到50~200的长度，或者使用"采样"策略——从完整序列中随机采样一个子集。
  • 在线serving：DIN的inference需要在请求时计算用户行为序列中每个行为与候选物品的注意力分数。这意味着无法像双塔模型那样预计算用户向量。阿里巴巴的实践是将注意力计算优化到GPU上，单次推理时间控制在10ms以内。
  • 长尾行为：对于行为很少的新用户，注意力机制退化为平均池化——因为没有足够的行为来产生有意义的权重差异。这种情况下，DIN的效果和传统模型差不多。

DIN的行业影响：DIN论文发表于2018年KDD，至今仍是推荐系统领域引用量最高的论文之一。它的历史地位在于——它证明了注意力机制（Attention）不仅仅可以在NLP中做机器翻译，也可以在推荐系统中取得工业级的巨大收益。自此之后，attention-based recommendation model成为了推荐系统研究的主流方向之一。

不过，DIN有一个潜在的盲区：它虽然能捕捉"哪些历史行为和当前候选相关"，但它无法捕捉"用户的兴趣是如何随时间演化的"。你今天买手机壳，下周买手机，下个月买耳机——这是一个"手机生态"的兴趣演化路径。DIN把行为序列当作无序集合（虽然有attention，但位置信息没有被有效利用）。下一章的DIEN将在此基础上引入序列建模，让模型不仅能"看相关"还能"看演化"。
