从导航App的"最短路径"说起

想象你每天早上打开地图App，输入公司地址。App在几秒内给你规划出一条"最优"路线——耗时最短、不堵车、没有红绿灯太多。你有没有想过，它在背后做了什么？

地图App把道路网建模成一个带权有向图：路口是顶点，道路是有向边，边的权重是通行时间（考虑实时路况）。而"规划最短路径"就是在这个巨大的图上运行最短路径算法。

图算法是计算机科学中最具实用价值的算法群之一。从地图导航到社交网络推荐，从项目管理到网络路由——图算法在支撑着我们日常使用的几乎每一个数字服务。这一章我们将学习四个最重要的图算法家族：最短路径、拓扑排序、最小生成树和关键路径。

最短路径

Dijkstra算法——贪心+堆优化

Dijkstra算法是解决单源最短路径问题（从一个起点到所有其他点的最短路径）最著名的算法。它的核心思想是贪心：每次从未处理的顶点中选择"距离起点最近"的那个，然后更新它的邻居。

不加堆优化的Dijkstra复杂度是 O(n²)，适合稠密图。加上优先队列（最小堆）优化后，复杂度降为 O((n + m) log n)，适合稀疏图。

import heapq
from collections import defaultdict

def dijkstra(graph, start):
    """
    Dijkstra最短路径算法（堆优化版）
    graph: 邻接表，{u: [(v, weight), ...]}
    start: 起点
    返回: dist 字典（起点到各点的最短距离）
    """
    dist = defaultdict(lambda: float('inf'))
    dist[start] = 0
    pq = [(0, start)]  # (距离, 顶点)
    visited = set()

    while pq:
        d, u = heapq.heappop(pq)
        if u in visited:
            continue
        visited.add(u)

        for v, w in graph[u]:
            if v not in visited:
                new_dist = d + w
                if new_dist < dist[v]:
                    dist[v] = new_dist
                    heapq.heappush(pq, (new_dist, v))

    return dist

# 使用示例
# 图结构：
#   A ---(4)--> B
#   A ---(2)--> C
#   B ---(1)--> C
#   B ---(5)--> D
#   C ---(8)--> D

graph = {
    'A': [('B', 4), ('C', 2)],
    'B': [('C', 1), ('D', 5)],
    'C': [('D', 8)],
    'D': [],
}

dist = dijkstra(graph, 'A')
# dist = {'A': 0, 'C': 2, 'B': 4, 'D': 7}
# 最短路径 A→D: A→C→D? 不对，A→B→C→D 的权重是 4+1+8=13
#          A→B→D 是 4+5=9
#          A→C→D 是 2+8=10
# 等等，A→B是4，A→C是2，B→C是1，所以A→B→C是5 > 2，所以A到C最短是2
# A→B: 4, A→C: 2, B→D: 5, C→D: 8
# A到D：A→C→D = 10, A→B→D = 9 → 最短是9
# 但等一下：A→(C)→B→D? C到B没有边
# 所以A→B→D = 4+5=9, A→C→D = 2+8=10
# 最短A→D = 9: A→B→D
一个让我后悔莫及的错误：我曾经认为Dijkstra算法可以处理"所有"图的最短路径问题。直到有一天，我在一个金融系统的交易路由场景中使用了Dijkstra，结果得到了错误的答案。原因是交易路由中存在"手续费返还"——相当于负权边。Dijkstra算法基于"每次选当前最近的节点"的贪心策略，一旦遇到负权边，这个贪心就失效了——因为一个看起来"远"的节点可能通过负权边变成"近"的。那次故障让我深刻记住了：Dijkstra不能处理负权边！需要处理负权边时，用Bellman-Ford算法。

Bellman-Ford算法——处理负权边

Bellman-Ford算法的思想很朴素：对所有边进行 n-1 轮"松弛操作"。每轮遍历所有边，尝试用当前已知的最短路径更新相邻顶点的距离。

def bellman_ford(edges, n, start):
    """
    edges: [(u, v, weight), ...]
    n: 顶点数量
    start: 起点
    返回: dist 数组，如果存在负权环则返回 None
    """
    dist = [float('inf')] * n
    dist[start] = 0

    # 第1到n-1轮松弛
    for _ in range(n - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                updated = True
        if not updated:
            break  # 提前结束

    # 检测负权环：再做一轮，如果还有更新说明有负权环
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            return None  # 存在负权环

    return dist

复杂度 O(n × m) — 比Dijkstra慢，但能处理负权边。适用于顶点数较少但可能有负权边的场景。

多源最短路径——Floyd-Warshall

Dijkstra和Bellman-Ford都是"单源"的——从一个起点到所有点。如果要求所有顶点对之间的最短路径，一个非常优雅的做法是Floyd-Warshall算法。

Floyd-Warshall的核心是一个动态规划思想：dist[k][i][j] 表示"只允许使用前 k 个顶点作为中间节点时，i 到 j 的最短距离"。递推公式：

dist[k][i][j] = min(dist[k-1][i][j], dist[k-1][i][k] + dist[k-1][k][j])

# 简化后可以直接在原数组上进行：
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

用三层循环实现，极其简洁：

def floyd_warshall(n, graph):
    """
    n: 顶点数量
    graph: n×n 邻接矩阵，graph[i][j] = 权重（没有边则为inf，对角线为0）
    返回: n×n 最短距离矩阵
    """
    dist = [row[:] for row in graph]

    for k in range(n):      # 中间节点
        for i in range(n):   # 起点
            for j in range(n):  # 终点
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]

    return dist

你没看错——只有5行代码。但它的时间复杂度是 O(n³)，空间复杂度 O(n²)。适用于顶点数不太多（n ≤ 几百）且需要"所有点对"最短路径的场景。

最短路径算法对比：

算法 类型 时间复杂度 空间复杂度 负权边 适用场景
Dijkstra（无堆优化） 单源 O(n²) O(n) 不能 稠密图，无负权
Dijkstra（堆优化） 单源 O((n+m) log n) O(n+m) 不能 稀疏图，无负权
Bellman-Ford 单源 O(n×m) O(n) 能 有负权边，检测负权环
Floyd-Warshall 多源 O(n³) O(n²) 能（无负权环） 顶点数小，求所有点对

拓扑排序

拓扑排序（Topological Sort）是对有向无环图（DAG）的顶点进行线性排序，使得对于每一条边 u→v，u 在排序中出现在 v 之前。拓扑排序不是唯一的——一个DAG可能有多种合法的拓扑排序。

典型的应用场景：课程先修关系。比如"数据结构"课程要求在"算法设计"之前修读，那么"数据结构"就必须排在"算法设计"前面。

Kahn算法（BFS + 入度表）

from collections import deque

def topological_sort_kahn(graph):
    """
    Kahn算法（BFS+入度表）
    graph: 邻接表，{u: [v, ...]}
    返回: 拓扑排序列表，如果存在环则返回空列表
    """
    # 计算入度
    in_degree = defaultdict(int)
    for u in graph:
        in_degree[u]  # 确保所有顶点都在字典里
    for u in graph:
        for v in graph[u]:
            in_degree[v] += 1

    # 入度为0的顶点入队
    queue = deque([u for u in in_degree if in_degree[u] == 0])
    result = []

    while queue:
        u = queue.popleft()
        result.append(u)
        for v in graph[u]:
            in_degree[v] -= 1
            if in_degree[v] == 0:
                queue.append(v)

    # 如果result长度不等于顶点数，说明存在环
    return result if len(result) == len(graph) else []

# 课程表问题 LeetCode 207
def can_finish(num_courses, prerequisites):
    """判断是否能完成所有课程"""
    graph = defaultdict(list)
    for course, prereq in prerequisites:
        graph[prereq].append(course)

    result = topological_sort_kahn(graph)
    return len(result) == num_courses

DFS后序遍历法

另一种获得拓扑排序的方法是对DAG做DFS后序遍历：

def topological_sort_dfs(graph):
    """DFS后序法获取拓扑排序"""
    visited = set()
    result = []

    def dfs(u):
        visited.add(u)
        for v in graph[u]:
            if v not in visited:
                dfs(v)
        result.append(u)  # 后序加入

    for u in graph:
        if u not in visited:
            dfs(u)

    return result[::-1]  # 反转

两种方法的时间复杂度都是 O(n + m)。Kahn算法更容易检测环（通过结果长度判断），DFS后序法更简洁。

最小生成树

假设你是一家电信公司的网络工程师，需要在 n 个城市之间铺设光缆。你知道了任意两个城市之间铺设光缆的成本。你希望用总成本最低的方式把所有城市连通起来——这就是最小生成树（Minimum Spanning Tree, MST）问题。

Prim算法（贪心+堆）

Prim算法从一个顶点出发，每次选择一条"连接已选择顶点集合和未选择顶点集合的最小权重边"，将边的另一端加入集合。

def prim_mst(graph, start=0):
    """
    Prim最小生成树算法（堆优化版）
    graph: 邻接表，{u: [(v, weight), ...]}
    返回: (总权重, 边列表)
    """
    visited = {start}
    # 存储所有从visited到未visited的边
    heap = []
    for v, w in graph[start]:
        heapq.heappush(heap, (w, start, v))

    total_weight = 0
    mst_edges = []

    while heap and len(visited) < len(graph):
        w, u, v = heapq.heappop(heap)
        if v in visited:
            continue
        visited.add(v)
        total_weight += w
        mst_edges.append((u, v, w))
        for next_v, next_w in graph[v]:
            if next_v not in visited:
                heapq.heappush(heap, (next_w, v, next_v))

    return total_weight, mst_edges

Kruskal算法（并查集）

Kruskal算法更直观：先把所有边按权重排序，然后从小到大依次选择边，只要这条边不会形成环（用并查集判断）。

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py:
            return False
        if self.rank[px] < self.rank[py]:
            self.parent[px] = py
        elif self.rank[px] > self.rank[py]:
            self.parent[py] = px
        else:
            self.parent[py] = px
            self.rank[px] += 1
        return True

def kruskal_mst(n, edges):
    """
    Kruskal最小生成树算法
    n: 顶点数
    edges: [(u, v, weight), ...]
    返回: (总权重, 边列表)
    """
    edges = sorted(edges, key=lambda x: x[2])
    uf = UnionFind(n)
    total_weight = 0
    mst_edges = []

    for u, v, w in edges:
        if uf.union(u, v):
            total_weight += w
            mst_edges.append((u, v, w))
            if len(mst_edges) == n - 1:
                break

    return total_weight, mst_edges

Prim vs Kruskal 对比：

特性 Prim算法 Kruskal算法
核心思想 从顶点出发，逐步扩展 从边出发，从小到大选择
数据结构 堆（优先队列） 并查集
时间复杂度 O((n+m) log n) O(m log m)（主要是排序）
适合场景 稠密图（m ≈ n²） 稀疏图（m << n²）
实现难度 中等 简单（如果有并查集库）

关键路径——项目管理中的图算法

关键路径法（Critical Path Method, CPM）是项目管理领域的经典算法。它将项目分解为一系列"活动"（任务），每个活动有持续时间，活动之间有依赖关系。通过计算最早开始时间和最晚开始时间，找到哪些活动是"关键的"——它们如果延迟，整个项目就会延迟。

AOE网（Activity on Edge）将活动表示为边，顶点表示事件（活动完成的里程碑）。关键路径就是AOE网中从起点到终点的最长路径（决定了项目的最短工期）。

def critical_path(vertices, edges):
    """
    关键路径算法
    vertices: 顶点列表
    edges: [(u, v, duration), ...] 表示活动从u到v，耗时duration
    返回: (最短工期, 关键活动列表)
    """
    # 构建图和入度
    graph = defaultdict(list)
    in_degree = defaultdict(int)
    for u, v, d in edges:
        graph[u].append((v, d))
        in_degree[v] += 1
        in_degree[u]  # 确保所有顶点在字典里

    # 拓扑排序（Kahn算法）
    from collections import deque
    queue = deque([v for v in vertices if in_degree[v] == 0])
    topo = []
    while queue:
        u = queue.popleft()
        topo.append(u)
        for v, d in graph[u]:
            in_degree[v] -= 1
            if in_degree[v] == 0:
                queue.append(v)

    # 计算最早开始时间（正向）
    earliest = {v: 0 for v in vertices}
    for u in topo:
        for v, d in graph[u]:
            if earliest[u] + d > earliest[v]:
                earliest[v] = earliest[u] + d

    # 计算最晚开始时间（反向）
    latest = {v: float('inf') for v in vertices}
    # 终点的最晚时间 = 最早时间
    for v in vertices:
        if not [e for e in edges if e[1] == v]:  # 没有出边的是终点
            latest[v] = earliest[v]

    for u in reversed(topo):
        for v, d in graph[u]:
            if latest[v] - d < latest[u]:
                latest[u] = latest[v] - d

    # 关键活动：最早时间 == 最晚时间的活动
    critical_edges = []
    for u, v, d in edges:
        if earliest[u] == latest[u] and earliest[v] == latest[v]:
            if latest[v] - earliest[u] == d:
                critical_edges.append((u, v, d))

    return earliest[topo[-1]], critical_edges

关键路径分析在项目管理软件（如Microsoft Project、Jira高级路线图）中被广泛使用——自动识别哪些任务有"浮动时间"（可以延迟而不影响总体进度），哪些任务是"关键"的（一刻也不能拖）。

图算法总结：这四个算法家族覆盖了图论中最重要的应用场景—— 最短路径（导航、路由）、拓扑排序（依赖管理、构建系统）、 最小生成树（网络设计、聚类分析）、关键路径（项目管理、调度）。 掌握它们，你就能解决绝大多数"图相关"的面试题和工程问题。
