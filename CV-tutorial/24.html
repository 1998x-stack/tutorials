什么是迁移学习

想象一下：你花了三年时间学会了弹钢琴，现在想学电子琴。你需要从头开始学吗？当然不需要——指法、乐理、节奏感这些基础能力完全可以迁移过去，你只需要适应电子琴特有的音色和触键差异就够了。这就是迁移学习（Transfer Learning）的核心逻辑：将在一个任务上学到的知识迁移到另一个相关任务上，避免从零开始。

2017年我接手过一个细胞分类项目，任务是从显微图像中区分正常细胞和病变细胞。团队只有500张标注图像——这点数据训练任何CNN都不可能收敛。情急之下我试着加载了ImageNet上预训练的ResNet-50，冻结了前40层，只在最后加了一个二分类头，用这500张图微调了不到10分钟。结果呢？测试集准确率91%。这个结果至今让我印象深刻——ResNet-50在ImageNet上学习到的"边缘检测"、"纹理识别"、"形状感知"等通用视觉能力，对分析细胞形态几乎完全适用。人类细胞和ImageNet里的狗、猫、汽车长得完全不一样，但底层的视觉特征——轮廓、纹理、颜色分布——却是相通的。

这就是迁移学习在小数据集上也能成功的根本原因：深度神经网络的前几层学到的都是通用的视觉基础特征（边缘、角点、纹理基元），这些特征与具体任务无关。你只需要在顶层重新训练任务相关的特征组合。因为AlexNet在2012年的成功，很大程度上也依赖于在ImageNet上学习到的通用特征——这在当时被称为"ImageNet特征"且被广泛用于各类视觉任务。

💡 关键理解：神经网络的学习具有层次性。浅层学到的是Gabor滤波器类似的边缘和纹理检测器，中间层学到的是形状和部件级的特征，只有最后几层才学到的才是"猫的脸"或"汽车轮子"这种任务相关的高层语义。你冻结前几层，保留的正是那些通用视觉能力。

预训练权重的力量

预训练权重的本质是什么？就是在海量数据上经过充分优化的模型参数初始值。相对于随机初始化（Xavier或Kaiming初始化），预训练权重给模型提供了一个极好的起点——模型已经知道如何"看"图像了，只需要针对你的特定任务做微调即可。

ImageNet预训练（ImageNet Pretraining）是CV领域最经典、应用最广泛的预训练范式。它之所以有效，是因为ImageNet拥有1400万张图像、21841个类别，覆盖了极其丰富的视觉概念。模型在ImageNet上预训练后，学到了令人惊讶的通用视觉表征。下表展示了不同预训练数据规模对下游任务性能的影响：

预训练数据图像数量下游任务（分类）检测AP分割mIoU
随机初始化（无预训练）065.2%42.134.5
ImageNet-1K（标准）140万82.7%53.645.2
ImageNet-21K（大规模）1400万85.1%56.347.8
自监督预训练（DINO / MAE）140万83.5%54.846.1

上表的数据告诉我们几个重要结论：第一，无论预训练数据是什么，只要有预训练，性能就显著优于随机初始化；第二，更大规模的预训练数据（ImageNet-21K）带来了进一步的提升；第三，自监督预训练（不需要标注）的效果已经接近监督预训练——这预示着"预训练"可能不再需要昂贵的标注数据。

💡 实用指导：在开始任何CV项目时，先尝试加载预训练权重已经成为行业标准做法。即便你的数据集很大（比如10万张），从预训练权重开始微调也比从头训练收敛更快、最终性能更好。除非你的数据集达到百万级且与ImageNet分布差异极大，否则没有理由"从零训练"。

微调策略

加载预训练权重只是第一步，如何微调（Fine-tuning）才是决定最终性能的关键。不同的微调策略适用于不同的场景，选择不当可能导致灾难性遗忘、过拟合或训练效率低下。

全量微调（Full Fine-tuning）

全量微调是最直接的方法：加载预训练权重后，用目标数据集训练所有层的参数。这适用于目标数据集足够大（每个类别>1000张）的情况。优点是模型能完全适应新任务，缺点是计算成本高，且在小数据集上容易过拟合。

冻结Backbone（Feature Extractor）

当目标数据集非常小（每个类别几十张）时，最好的策略是冻结预训练模型的主干网络（backbone），只训练新添加的分类头或检测头。这种做法将预训练模型视为一个特征提取器（feature extractor），只利用它提取通用特征，然后学习如何组合这些特征来做新任务的决策。训练速度极快，不容易过拟合，是"小数据+大模型"场景下的标配。

渐进解冻（Gradual Unfreezing）

渐进解冻是介于"全量微调"和"冻结Backbone"之间的折中策略。具体做法：先只训练分类头，收敛后解冻最后几层继续训练，然后一步步解冻更多层。这种方法的好处是：先让分类头适应新任务的特征空间，再逐步调整底层的预训练特征。这有点像"先学会走路再跑步"——如果一开始就让所有层同时更新，分类头（尚未训练好）和backbone（预训练好的初始条件）之间的梯度信号会互相干扰。

差分学习率（Discriminative Learning Rates）

差分学习率是微调中最实用的技巧之一。它的核心思想是：不同层应该使用不同的学习率——靠近输入的层使用较小的学习率（因为它们学习的是通用特征，不需要大的更新），靠近输出的层使用较大的学习率（因为它们需要快速适应新任务）。

import torch.nn as nn
import torch.optim as optim
from torchvision.models import resnet50

model = resnet50(pretrained=True)

# 将模型参数分成两组
# 1. Backbone: 使用较低的学习率
# 2. 分类头（新添加的层）: 使用较高的学习率
num_ftrs = model.fc.in_features
model.fc = nn.Linear(num_ftrs, 2)  # 替换为二分类头

params_to_update = [
    {'params': list(model.parameters())[:-2], 'lr': 1e-5},    # backbone: 低学习率
    {'params': model.fc.parameters(), 'lr': 1e-3},            # 分类头: 高学习率
]

optimizer = optim.AdamW(params_to_update, weight_decay=1e-4)
# 注意：这里不能给optimizer传全局lr，因为已经在分组中指定

# 也可以使用学习率调度器（如余弦退火）
scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=50)
⚠️ 踩坑提醒：差分学习率中backbone的学习率不能过高。我见过有人将全量微调时统一设成1e-3，然后发现模型在验证集上的损失先下降后急剧上升——这是灾难性遗忘（catastrophic forgetting）的典型表现：预训练知识在过大的梯度更新中被完全覆盖，新数据不够模型重新学习。Backbone学习率一般建议从1e-5或1e-6开始试。

常见的迁移学习误区

迁移学习虽然强大，但在实际使用中存在几个常见误区，我几乎在每次团队code review中都会遇到：

误区一：预训练数据分布与目标任务差异过大。预训练在ImageNet上的模型擅长处理自然场景图像（物体在画面中心、光照均匀、背景清晰）。但如果你在医学CT图像上直接使用ImageNet预训练权重，效果可能会很差——CT图像是单通道灰度图、像素值代表组织密度而非光照反射、空间结构与自然图像完全不同。解决方案：可以在ImageNet预训练基础上，先在一个中等规模的特定领域数据集上继续进行预训练（domain-specific pre-training），然后再微调到具体任务。

误区二：学习率设置错误。最常见的错误是将微调的学习率设置得和从头训练一样高（如1e-3）。预训练权重已经是一个不错的局部最优解，过大的学习率会让参数直接跳出这个最优区域。正确的做法是：微调的学习率通常比从头训练小一个数量级——建议从1e-4（全量微调）或1e-5（只微调高层）开始探索。

误区三：过早停止训练。微调只需要很少的epoch就能收敛（通常10-20个epoch足够了）。但"不需要很多epoch"不等于"训练1个epoch就能好"。很多人在第一个epoch验证损失下降不明显时就放弃了——请记住，预训练模型在目标数据分布上可能需要几个epoch的"适应期"。

场景推荐策略学习率范围Epoch数
目标数据很小（<500张）冻结Backbone + 仅训练分类头1e-3 ~ 1e-4（分类头）20-50
目标数据中等（500~5000张）渐进解冻或差分学习率1e-5（backbone）+ 1e-4（头）20-40
目标数据较大（>5000张）全量微调1e-4（权重衰减1e-4）30-100
领域差异很大（如医学/遥感）先领域预训练，再微调领域预训练1e-4，微调1e-550-100 + 20-30

领域适配与Domain Shift

迁移学习的最大挑战之一是域偏移（Domain Shift）：源域（预训练数据的分布）和目标域（你的任务数据的分布）之间存在差异。域偏移的程度决定了迁移学习的难度。

域偏移可以分为几个层次：低层次偏移包括光照、颜色、分辨率等底层视觉差异——这类偏移通常可以通过数据增强和标准化来解决；中层次偏移包括物体的纹理、形状、空间布局差异——比如自然图像中的物体通常出现在画面中央，而航拍图像中的物体分布在整个画面；高层次偏移包括类别体系的差异——ImageNet上有"猫"和"狗"的类别，但你的医学任务中只有"肿瘤"和"正常组织"——此时顶层语义完全不同。

从自然图像到医学影像、遥感图像或工业检测图像，域偏移尤为显著。以下是一些针对特定领域的适配策略：

  • 医学影像：通常使用ImageNet预训练加载前几层的权重（只加载卷积层，不加载全连接层），然后在大规模医学数据集（如CheXpert、RadImageNet）上继续预训练。一个常见技巧是将RGB的预训练权重通过平均或重复扩展到单通道或多光谱输入。
  • 遥感图像：遥感图像通常是正射（从上往下拍），具有特殊的空间结构。建议在遥感专用的预训练数据集（如RGB遥感数据集、Million-AID）上继续训练。旋转增强（全角度）是必须的。
  • 工业检测：工业图像通常背景单一、光照可控。域偏移的主要来源是不同产线、不同相机型号导致的色彩和分辨率差异。建议使用颜色增强+风格迁移来弥合差距。
💡 域适配技巧：一个简单有效的trick：如果目标域和源域的分布差距较大，可以在训练初期只更新Batch Normalization层的running mean和running variance，而不更新卷积核参数。这是因为BN层的统计量直接反映了数据分布，让模型先用目标数据"校正"BN层的统计信息，再微调卷积核，往往能获得更稳定的训练过程。

参数高效微调

随着模型越来越大（ViT-Large有3亿参数，ViT-Huge有6亿参数），全量微调的计算成本越来越高。近年来，参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）成为NLP和CV领域共同的研究热点。PEFT的核心目标是：在保持预训练权重不变的前提下，只训练/插入少量额外参数，就能达到接近全量微调的性能。

Linear Probing

Linear Probing是最简单的PEFT方法：完全冻结backbone的所有参数，只在顶端训练一个线性分类器（单层全连接层）。这在特征提取器足够强的场景下（如CLIP、DINOv2），效果出奇地好。CLIP的zero-shot分类能力本质上就是一种text-driven的linear probing。

Adapter

Adapter在网络的每一层中插入一个小型的瓶颈（bottleneck）模块：先将特征维度压缩（降维），再通过一个非线性激活，最后恢复原始维度。训练时只更新这些adapter模块的参数，预训练参数全部冻结。Adapter的参数量通常只有原模型的1-5%，但性能可以达到全量微调的95%以上。

LoRA（Low-Rank Adaptation）

LoRA最初在NLP的大语言模型微调中提出，但很快被引入视觉Transformer的微调。它的核心思想是：对预训练权重的增量更新进行低秩分解。对于权重矩阵W ∈ R^(d×k)，LoRA将其更新量ΔW分解为两个低秩矩阵A和B的乘积：ΔW = BA，其中B ∈ R^(d×r)，A ∈ R^(r×k)，r远小于min(d,k)。训练时只更新A和B，W冻结不变。这在视觉Transformer的微调中表现出色——特别是当你有多个下游任务时，每个任务只需要保存一个很小的LoRA权重文件（几MB），而不是完整的模型副本（几百MB）。

# LoRA在ViT上的简化实现（核心逻辑）
class LoRALayer(nn.Module):
    def __init__(self, in_dim, out_dim, rank=4, alpha=1.0):
        super().__init__()
        # 原始权重冻结
        self.weight = nn.Parameter(torch.randn(out_dim, in_dim), requires_grad=False)
        # LoRA低秩矩阵
        self.lora_A = nn.Parameter(torch.randn(rank, in_dim) * 0.01)
        self.lora_B = nn.Parameter(torch.zeros(out_dim, rank))
        self.alpha = alpha
        self.scaling = alpha / rank

    def forward(self, x):
        # 原始前向 + LoRA增量
        base_out = F.linear(x, self.weight)
        lora_out = (x @ self.lora_A.T) @ self.lora_B.T * self.scaling
        return base_out + lora_out

🔬 前沿趋势：2024年后的研究发现，对CLIP、DINOv2、SAM这类视觉基础模型来说，Linear Probing结合精心设计的数据增强，效果竟然接近甚至超过了全量微调。这表明模型在预训练阶段学到的表征已经足够强大——也许未来"微调"会越来越少，取而代之的是"更好的预训练+简单的适配"。

迁移学习和模型微调是CV工程中最实用的技术之一。一个经过充分预训练的模型配合正确的微调策略，能让小团队在有限的数据和算力下也能训练出高性能的视觉模型。但在实际生产环境中，训练出一个好模型只是第一步——如何把它部署到生产环境、放到边缘设备上高效运行，是同样重要的工程挑战。下一章我们将进入模型优化与部署的世界。
