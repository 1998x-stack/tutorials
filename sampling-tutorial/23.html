第23章 R语言实现

大家好，我是你们的讲师。前22章我们把抽样的理论知识梳理了一遍，从简单随机抽样到非概率抽样校正，涵盖了抽样技术的方方面面。但从今天开始，我们要换个角度——动手。理论学得再好，不会用代码实现，那终究是纸上谈兵。

R语言在统计调查领域有着无可撼动的地位。原因很简单：R是统计学家为统计学家设计的语言。它不仅有最全面的抽样方法实现，更有以 survey 包为代表的复杂调查数据分析生态。可以毫不夸张地说，如果你要做政府统计级别的抽样调查工作，R是首选工具——没有之一。本章我就带大家用R把之前学过的各种抽样方法一一落地。

R的抽样生态

R的抽样相关包非常丰富，我挑四个最核心的来介绍：

包名 作者 核心功能 适用场景
survey Thomas Lumley 复杂调查数据设计、加权估计、方差估计 分析已收集的复杂调查数据
sampling Yves Tillé & Alina Matei 各种概率抽样方法的选择与实现 从总体中抽取样本
surveyplanning LVNS 样本量计算、精度评估、设计效应估计 调查方案的计划阶段
PracTools Richard Valliant 抽样设计所需的各种计算工具 教学与实用抽样设计

这四者各有侧重。sampling包主要解决"怎么抽"的问题——从总体中选出一组样本；survey包解决"抽完了怎么分析"的问题——考虑复杂设计下的加权估计和方差估计；surveyplanning和PracTools则偏向设计阶段的规划计算。

工作流建议：在设计阶段用 surveyplanning 或 PracTools 计算样本量 → 用 sampling 包的函数执行抽样 → 用 survey 包的 svydesign() 声明设计 → 用 svytotal() / svymean() 等函数进行估计。三步走，清晰明了。

简单随机抽样（SRS）

先来最简单的。R内置的 sample() 函数本身就是做简单随机抽样的：

# 假设总体有 10000 个个体
N <- 10000
population <- data.frame(
    id = 1:N,
    income = rnorm(N, mean = 5000, sd = 1500),
    age = sample(18:80, N, replace = TRUE)
)

# 从总体中抽取 500 个样本（不放回）
set.seed(2024)
sample_indices <- sample(1:N, size = 500, replace = FALSE)
srs_sample <- population[sample_indices, ]

# 用 survey 包进行设计声明和估计
library(survey)
srs_design <- svydesign(id = ~1, data = srs_sample)

# 加权估计——SRS 中权重 = N/n = 20
svymean(~income, design = srs_design)
svytotal(~income, design = srs_design)

svydesign(id = ~1) 是 survey 包的核心入口。id = ~1 表示观察单元之间相互独立（即没有聚类）。对于 SRS 来说，每个样本的权重就是 N/n，survey 包会自动计算。你不需要手动操作权重。

分层抽样（Stratified Sampling）

分层抽样在 R 中可以通过 sampling 包的 strata() 函数来实现：

library(sampling)

# 给总体加上分层变量
population$stratum <- sample(c("东部", "中部", "西部"),
                             N, replace = TRUE, prob = c(0.4, 0.3, 0.3))

# 按层进行比例分配抽样（每层抽 10%）
set.seed(2024)
strat_sample <- strata(
    data = population,
    stratanames = "stratum",   # 分层变量
    size = c(400, 300, 300),   # 各层样本量
    method = "srswor"          # 不放回简单随机抽样
)

# strata() 返回的是索引，需要用它来取出实际数据
strat_data <- getdata(population, strat_sample)

# 用 survey 包声明分层设计
strat_design <- svydesign(
    id = ~1,
    strata = ~stratum,
    data = strat_data,
    fpc = rep(c(4000, 3000, 3000), c(400, 300, 300))
)

# 注意 fpc（有限总体校正）参数给出了每层的总体规模
svymean(~income, design = strat_design)

strata() 函数的 size 参数指定每层抽多少样本，你可以按比例分配，也可以按 Neyman 分配来优化精度。method = "srswor" 是简单随机不放回；如果想用系统抽样，可以改成 method = "systematic"。

整群抽样与多阶段抽样

对于整群抽样和多阶段抽样，核心在于处理好 svydesign() 的 id 参数。每一个阶段对应一个 id 层级：

# 构造一个二阶段抽样数据：先抽学校，再抽学生
# 假设总体有 200 所学校，每校约 50 名学生
library(dplyr)

# 第一阶段：抽取 20 所学校
schools <- data.frame(
    school_id = 1:200,
    school_size = round(runif(200, 200, 800))
)
set.seed(42)
selected_schools <- sample(1:200, 20)

# 第二阶段：从每所选校抽取 10 名学生
student_list <- do.call(rbind, lapply(selected_schools, function(sid) {
    n_students <- 50  # 简化处理，每校 50 人
    data.frame(
        school_id = sid,
        student_id = 1:n_students,
        score = rnorm(n_students, mean = 75, sd = 10)
    )
}))

# 从每校抽 10 名学生
final_sample <- student_list %>%
    group_by(school_id) %>%
    slice_sample(n = 10)

# 声明二阶段设计
two_stage_design <- svydesign(
    id = ~ school_id + student_id,  # 第一阶段 PSU, 第二阶段 SSU
    data = final_sample
)

svymean(~score, design = two_stage_design)
deff(two_stage_design)  # 计算设计效应

注意 id = ~ school_id + student_id：第一项是初级抽样单元（PSU），第二项是次级抽样单元（SSU）。survey 包会根据这个层级结构自动计算正确的标准误。

权重处理与设计效应

survey 包的一大优势是它会自动处理权重。当你用 svydesign() 声明设计后，svytotal()、svymean()、svyratio() 等估计函数都会自动使用设计权重。你不需要手动在 data.frame 里维护一列权重。

如果你需要按子组（subgroup）分别估计设计效应，可以用 svyby()：

# 按地区分别估计平均收入，同时计算设计效应
svyby(~income, ~stratum, design = strat_design,
      FUN = svymean, deff = TRUE)

# 结果示例：
#   stratum    income     SE   DEff
#   东部    5123.45  85.23  1.12
#   中部    4876.32  92.15  1.08
#   西部    4654.78 112.47  1.21

deff = TRUE 参数会输出每个子组的设计效应。这是一个非常好的实践——在你的调查报告中，设计效应是必须报告的信息之一。

小贴士：survey 包的 svrepdesign() 函数可以创建重抽样设计（bootstrap、jackknife、BRR），用于复杂统计量的方差估计。如果你的估计量不是简单的均值或总数（比如中位数、Gini 系数），建议使用重抽样方法。用 as.svrepdesign() 可以将已有的 svydesign 对象转换为重抽样设计对象。

真实踩坑故事：t.test 的陷阱

我要给你讲一个我在实际工作中遇到的案例。几年前，一位医学研究者找到我，他做了一项全国性的居民健康调查，采用的是分层多阶段抽样设计——标准的复杂调查设计。他收集了约 5000 份有效问卷，数据分析阶段他熟练地打开了 R，然后……

他用 t.test() 来比较两组人群的血压均值差异。

乍一看没问题对不对？两组比较，t 检验，天经地义。但问题在于：t.test() 假设样本是独立同分布的（i.i.d.），即每个观察之间相互独立，且来自同一个分布。而他的数据来自复杂抽样设计——同一 PSU 内的受访者在统计上是不独立的（因为他们在同一个社区被抽中），而且不同层的抽样概率不同，样本是有权重的。

当他用 t.test() 跑完分析后，得到了一个 p 值 = 0.02，宣称"两组差异显著"。我让他改用 svyttest()（survey 包中专门用于复杂调查数据的 t 检验函数），结果 p 值变成了 0.13——差异根本不显著。

为什么 t.test 会给出假阳性的结果？ 因为复杂抽样中的设计效应（Design Effect）会让标准误变大。t.test 没有考虑聚类带来的方差膨胀，低估了标准误，从而高估了统计显著性。这个惨痛的教训告诉我们一个铁律：复杂调查数据必须用 survey 包的特殊函数来做统计推断——当你用 svydesign 声明了设计之后，一律使用 svyglm()、svyttest()、svychisq() 等函数，绝不能用基础 R 的统计函数。

完整示例：从抽样到分析的流水线

最后，我给大家一个完整的端到端示例，把这一章的内容串起来：

# =============================================
# 完整抽样分析流水线 (R)
# =============================================
library(survey)
library(sampling)
library(surveyplanning)

# 1. 设计阶段：计算样本量
# 假设目标精度: 相对误差 5%, 设计效应 deff = 1.5
# 用 surveyplanning 包
n_req <- n.ps(
    sigma2 = 1500^2,    # 总体方差
    cv = 0.05,          # 目标变异系数
    deff = 1.5,         # 预估设计效应
    N = 100000,         # 总体规模
    R = 0.8             # 预期应答率
)
print(paste("所需样本量:", n_req))

# 2. 分层抽取样本
set.seed(123)
population$stratum <- sample(LETTERS[1:4], N, replace = TRUE)
strata_sizes <- table(population$stratum)
sample_sizes <- round(strata_sizes * 0.05)  # 每层 5%

selected <- strata(
    population, "stratum",
    size = sample_sizes,
    method = "srswor"
)
sample_data <- getdata(population, selected)

# 3. 声明调查设计
design <- svydesign(
    id = ~1,
    strata = ~stratum,
    data = sample_data,
    fpc = rep(strata_sizes, sample_sizes)
)

# 4. 加权估计
total_income <- svytotal(~income, design)
mean_income <- svymean(~income, design)
cv_income <- cv(mean_income)

print(total_income)
print(mean_income)
print(paste("变异系数:", round(cv_income * 100, 2), "%"))

# 5. 子组分析 & 设计效应
svyby(~income, ~stratum, design, svymean, deff = TRUE)

# 6. 回归建模
model <- svyglm(income ~ age + stratum, design = design)
summary(model)

这一套流程涵盖了从设计阶段的样本量规划，到抽样执行，再到加权估计和建模分析的全过程。熟练之后，你可以把它封装成自己的分析模板。
