大家好，我是你们的讲师。今天我们要一起踏上一段旅程——从零开始，亲手构建路径规划算法。你可能会问：什么是路径规划？为什么它如此重要？如果你曾用过手机上的地图导航，或者看过机器人自动在仓库中穿梭取货，那么你已经见过路径规划在真实世界中的身影。但在这背后，究竟藏着怎样的数学原理和算法设计？这正是本课程要回答的问题。

在开始之前，我想先讲一个小故事。2003年，一家名为 Kiva Systems 的初创公司做了一个大胆的决定：让几百个"橙色小机器人"在仓库地板上跑来跑去，把货架搬到工人面前。当时所有人都觉得这是天方夜谭——这些机器人怎么知道去哪里、走哪条路、怎么避开彼此？十年后，亚马逊以 7.75 亿美元收购了 Kiva，这套路径规划系统每年为亚马逊节省了数十亿美元运营成本。今天，全球超过 20 万个 Kiva 风格的机器人每天在执行数十亿次路径规划。这就是路径规划的力量——它看不见摸不着，却在无声地驱动着现代世界的运转。

什么是路径规划？

最简洁的定义是：路径规划（Path Planning）是在给定的环境中，找到一条从起点到终点的、无碰撞的连续路径。听起来很简单，对吧？但让我们拆解一下这句话里的每一个要素。

环境（Environment）——这是机器人或智能体所在的"世界"。它可以是连续的地形、离散的栅格地图、甚至是抽象的状态空间。在自动驾驶场景中，环境就是道路、建筑物、行人构成的复杂动态场景。在游戏中，环境就是虚拟地图的几何结构。

起点（Start）与终点（Goal）——这两者定义了路径规划问题的边界。有时候我们清楚知道终点在哪（比如导航到具体地址），有时候我们只知道一个目标区域（比如"飞到那片区域上空侦察"），还有时候目标是"以特定姿态到达"，这被称为位姿（pose）目标。

无碰撞（Collision-free）——这是路径规划中最核心的约束。路径不能穿过障碍物。听起来理所当然，但在高维空间中判断碰撞本身就是一个复杂问题。比如一个六轴机械臂在狭窄空间中运动，你需要确保它的每一个关节、每一个连杆都不与环境中的任何物体发生干涉。

让我们用几个日常例子来感受一下：

  • GPS 导航——从家到公司，地图应用计算出一条避开拥堵、距离合理的行车路线。这是最常见的路径规划。
  • 游戏 NPC 找路——在《星际争霸》中，一个机枪兵需要从基地走到前线，绕过地形和敌方单位。游戏 AI 每帧都在进行成千上万次路径规划。
  • 仓库机器人取货——Kiva 机器人在数十万平米的仓库中穿梭，选择最优路线到达指定货架，再搬运到打包站。它需要在几百个同伴间实时避让。
  • 手机芯片布线——EDA（电子设计自动化）工具在指甲盖大小的芯片上为数十亿晶体管规划连线路径，这是最精密、约束最苛刻的路径规划场景之一。
核心要点：路径规划的三个要素——起点、终点、环境——构成了本课程所有算法的基础框架。无论算法多么复杂，归根结底都在问同一个问题："我如何从这里到达那里，且不撞上任何东西？"

路径规划 vs 运动规划 vs 轨迹规划

在进入算法之前，我们需要澄清三个经常被混用的概念。很多初学者在这里踩坑——开会时跟同事说"我在做路径规划"，结果发现对方理解的"路径规划"和你说的根本不是一回事。

概念 英文 输出 考虑因素 典型问题
路径规划 Path Planning 几何路径（一系列位置点） 障碍物、环境几何 "走哪条路不会被卡住？"
运动规划 Motion Planning 运动序列（含姿态、运动学约束） 障碍物 + 机器人运动学 "这辆车能拐过这个弯吗？"
轨迹规划 Trajectory Planning 时变轨迹（位置、速度、加速度随时间变化） 障碍物 + 运动学 + 动力学 + 时间 "如何在 3 秒内平滑经过弯道？"

打个比方：假设你要去厨房倒杯水。

  • 路径规划决定："绕过沙发，穿过走廊，进入厨房"——这是空间中的一条几何路径。
  • 运动规划决定："先转身避开茶几，再小碎步穿过狭窄的过道"——考虑了你的身体尺寸和转身能力。
  • 轨迹规划决定："前两步慢走避免水洒出，第三步快走跟上节奏"——加入了时间维度和物理约束。
小提示：在本课程中，我们将从最基础的路径规划开始，逐步引入运动学约束和动力学约束，最终让你能够独立搭建一个完整的轨迹规划系统。理解这三个层次的区别，对你后续选择正确的方法至关重要。

路径规划用在哪里？

路径规划的应用广度可能远超你的想象。我们来看几个代表性领域：

领域 具体应用 代表案例 算法需求特点
自动驾驶 L4 级自动驾驶车辆在全球道路网中自主行驶 Waymo 在凤凰城、Tesla FSD 在城市街道 实时性极高（毫秒级）、处理动态障碍物、遵守交规
仓储物流 数百台机器人同时在地面网格上搬运货架 Amazon Robotics (原 Kiva)、Geek+ 多机器人协调、避免死锁、高吞吐量
游戏 AI 游戏中非玩家角色（NPC）在场景中智能移动 《星际争霸》寻路、《黑神话：悟空》Boss 追击逻辑 海量实体同时规划、内存效率优先、允许次优解
芯片 EDA 在芯片版图上为互连线规划路径 Cadence、Synopsys 的自动布线工具 约束极其复杂、可牺牲时间换最优性
无人机航迹 无人机在三维空间中避开建筑、实现全覆盖巡检 大疆智能飞行、农业植保无人机 三维空间、能量最优、风扰鲁棒性
医疗手术 手术机器人在人体内规划器械路径避开关键组织 达芬奇手术机器人术前规划 极高安全要求、精度达到亚毫米级

在这些应用中，自动驾驶和仓储机器人对路径规划算法的需求最为典型。以 Amazon 的 Kiva 系统为例：一个 Kiva 机器人平均每天行走超过 20 公里，在 20 万平米的仓库中与数百个同伴协同作业。它的路径规划器需要在几十毫秒内完成一次规划，且要保证全局无死锁。这背后是图搜索、交通管制、资源调度等一系列算法的精密配合。2012 年亚马逊收购 Kiva 后，仅通过优化路径规划就将仓库运营效率提升了 3-5 倍——这正是算法带来的真实价值。

算法分类全景图

路径规划领域经过几十年发展，已经形成了多个不同的方法论流派。理解这张"全景图"有助于你在面对具体问题时快速确定应该采用哪类算法。下表将主要算法按方法论进行分类：

类别 代表性算法 核心思想 适用场景
基于图搜索 Dijkstra, A*, D* Lite, JPS 将环境离散化为图结构，利用搜索策略在有向图中找到最短路径 已知静态环境、栅格地图、需要最优路径
基于采样 RRT, RRT*, PRM 通过随机采样构建路线图或搜索树，在高维空间中高效探索 高维空间、连续环境、机械臂规划
基于优化 MPC, 梯度优化, CHOMP 将路径规划建模为优化问题，通过数值优化方法求解 带动力学约束、需要轨迹平滑
基于学习 深度强化学习、模仿学习、GNN-based 利用神经网络从数据中学习规划策略，端到端或辅助决策 复杂动态环境、需要自适应、先验知识丰富
局部规划 DWA, 人工势场法, TEB 仅规划短时间窗口内的局部运动，快速响应环境变化 动态障碍物密集、实时避障

值得注意的是，这五类方法并非互斥的。在实际系统中，它们往往被组合使用。经典的自动驾驶规划栈就是一个"全局路径规划 + 局部路径规划"的分层架构：上层用 A* 或类似算法做全局导航，下层用 DWA 或 MPC 做局部避障。

几乎所有路径规划算法都可以用一个统一的框架来理解：

function PathPlanner(start, goal, environment):
    // 1. 建模：将环境转化为算法可处理的表示
    representation = buildRepresentation(environment)

    // 2. 初始化：起点加入候选集
    frontier = {start}
    visited = {}

    // 3. 迭代搜索：不断扩展直到抵达终点
    while frontier is not empty:
        current = selectMostPromising(frontier)
        if current == goal:
            return reconstructPath(current)
        frontier.remove(current)
        visited.add(current)
        for neighbor in getNeighbors(current, representation):
            if neighbor in visited:
                continue
            updateCost(neighbor, current)
            frontier.add(neighbor)

    // 4. 搜索失败：不存在可行路径
    return None

这个骨架贯穿了 Dijkstra、A*、RRT 等绝大多数经典算法。不同算法的本质差异在于：selectMostPromising 的选择策略（是贪心、是启发式、还是随机采样）、representation 的建模方式（是栅格、是图、还是采样点集）、以及 updateCost 的代价定义（是距离、是时间、还是能量）。理解了这一点，你就能举一反三地掌握整个算法家族。

本课程学习路线

本课程共 28 章，分为 6 个部分，带你从零基础成长为能够独立设计和实现路径规划系统的开发者：

  • 第一部分（第 1-5 章）：基础概念——构型空间、图论、栅格地图、启发式搜索入门。打好理论基础。
  • 第二部分（第 6-14 章）：经典算法——Dijkstra、A* 及其变体、D* Lite、JPS、RRT、RRT*、PRM，以及采样方法对比。这是课程的核心内容。
  • 第三部分（第 15-18 章）：路径质量与局部规划——路径平滑、DWA、人工势场法、运动学约束与曲线。让你的路径不仅"能走"，而且"走得稳"。
  • 第四部分（第 19-21 章）：高级场景——动态环境重规划、多目标路径规划、多机器人协调。面对真实世界的复杂性。
  • 第五部分（第 22-25 章）：实战工具——优化模型方法、可视化与调试、性能基准测试、仿真到实车部署。把算法变成可运行的代码。
  • 第六部分（第 26-28 章）：前沿与项目——学习型路径规划、前沿趋势、综合项目。放眼未来，融会贯通。

到课程结束时，你将能够：

  • 从零实现 A*、RRT、DWA 等核心算法
  • 理解不同算法在最优性、完备性、实时性上的取舍
  • 根据实际场景（已知 vs 未知、静态 vs 动态、2D vs 高维）选择正确的算法
  • 搭建一个完整的导航栈，并在仿真环境中完成部署
学习建议：路径规划是一个动手远重于理论的领域。每学完一个算法，请在仿真环境（推荐 Python + matplotlib 快速验证，后用 C++ 做性能优化）中亲手实现一遍。光看不练，三个月后留在你脑子里的只剩下"A* 好像是用什么启发式搜索"。相信我，这是过来人的血泪教训。

下一章我们将正式进入路径规划的数学基础——构型空间（Configuration Space / C-space）。这是理解所有路径规划算法之前必须掌握的核心概念。通俗地说，构型空间是把"机器人在地图中的位置"抽象成一个数学空间，让"寻找无碰撞路径"这个看似复杂的问题，转化为"在无障碍的高维空间中找一条连续曲线"——而后者要容易处理得多。我们第二章见。
