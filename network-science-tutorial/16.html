第16章 链路预测

你有没有在网购时被推荐过"你可能认识的人"？或者在Facebook、LinkedIn上收到过好友推荐通知？这些推荐背后有一套通用的算法逻辑：你还没有和某人连接，但根据现有的网络结构特征，系统判定你们之间的"边"是高度可能存在的。这就是链路预测（link prediction）——网络科学中一个极其实用又充满魅力的研究课题。

上一章我们讨论了网络如何被破坏。这一章恰恰相反：我们要研究网络如何"生长"。链路预测试图回答两个密切相关的问题：在当前网络中，哪些节点之间"应该"有边但被遗漏了（missing links）？在未来，哪些节点之间最有可能产生新的边（future links）？

链路预测问题

链路预测的形式化定义很简单：给定一个无向网络 G = (V, E)，在时间 t 观察到的边集为 E_t，我们的目标是对所有目前不存在的节点对 (u, v) ∉ E_t，计算一个连接可能性得分 score(u, v)，并按分数从高到低排序。分数越高，表示该节点对之间存在边的概率越大。

这个问题看似简单，却有着极其广泛的应用场景：

  • 社交网络中的好友推荐——Facebook在2012年宣称其好友推荐算法为平台贡献了约50%的新关系建立。
  • 生物网络重构——蛋白质相互作用网络常常因为实验成本的限制而存在大量缺失的边，链路预测可以帮助科学家优先验证最有可能的候选交互。
  • 推荐系统——用户-商品二分图上的链路预测本质上就是协同过滤推荐。
  • 犯罪网络分析——在犯罪嫌疑人之间的关系网络中，预测尚未发现的关联有助于警方锁定调查方向。

链路预测方法可以大致分为三类：基于相似性的方法、基于路径的方法和基于概率模型的方法。我们逐一来看。

基于相似性的方法

这类方法的核心思想很简单：如果两个节点在结构上"相似"，它们之间就更可能产生连接。"相似性"的具体定义有很多种，我们介绍几个最经典也最常用的指标。

共同邻居（Common Neighbors, CN）

这是最直观的指标：两个节点共享的邻居越多，它们就越可能连接。用数学语言表达很简单：

score(u, v) = |Γ(u) ∩ Γ(v)|

其中 Γ(u) 表示节点 u 的邻居集合。这个指标的背后直觉是，"朋友的朋友更可能成为朋友"。在社交网络分析中，这一现象也被称为"三元闭包"（triadic closure）：如果 A 认识 B，A 也认识 C，那么 B 和 C 之间产生连接的可能性就很高。

Jaccard系数

共同邻居有一个明显的缺陷：它没有考虑节点的度数。想象一下，一个万人迷网红和一个普通人，他们有一两个共同朋友，这个共同朋友的数量可能只是随机事件。Jaccard系数通过对共同邻居数量做归一化来解决这个问题：

score(u, v) = |Γ(u) ∩ Γ(v)| / |Γ(u) ∪ Γ(v)|

这个值的范围在0到1之间，越接近1表示两个节点的邻居集合越接近。

Adamic-Adar指数（AA）

AA指数引入了另一个重要的洞察：共同邻居中，那些"小众"的共同邻居比"大众"的共同邻居更有预测价值。如果你们俩的共同朋友是那种加了几千个好友的"社交达人"，这个共同关系的参考意义不大；但如果你们俩的共同朋友是一个只有5个好友的社恐，那么这个共同关系的权重就应该更高。

在形式上，AA指数为每个共同邻居分配一个权重 1/log(k)，其中 k 是该邻居的度数：

score(u, v) = Σ_{w ∈ Γ(u) ∩ Γ(v)} 1 / log|Γ(w)|

Resource Allocation（RA）

RA指数来自物理学的资源分配视角。想象节点 u 有一些"资源"需要传递给节点 v，通过中间节点 w 作为传递者。如果 w 有很多邻居，那么资源就会被稀释，最终到达 v 的资源就会很少。RA指数的公式与AA很相似，但权重更激进：

score(u, v) = Σ_{w ∈ Γ(u) ∩ Γ(v)} 1 / |Γ(w)|

指标 公式 特点 适用场景
CN |Γ(u) ∩ Γ(v)| 最简单，但偏高估高度节点 小规模网络
Jaccard |Γ(u) ∩ Γ(v)| / |Γ(u) ∪ Γ(v)| 归一化，避免度数偏差 度分布差异大的网络
AA Σ 1/log(k) 惩罚大众邻居，奖赏小众邻居 社交网络（推荐好友）
RA Σ 1/k 对高度数邻居惩罚最强 生物网络（预测PPI） 
# 在NetworkX中实现基于相似性的链路预测
import networkx as nx
import pandas as pd

# 创建一个小型社交网络
G = nx.karate_club_graph()

# 使用NetworkX内置的预测函数
pred_cn = list(nx.common_neighbors(G, 0, 4))
print(f"节点0和4的共同邻居: {pred_cn}")

# 对所有未连接节点对计算AA指数
def aa_score(G, u, v):
    """Adamic-Adar指数"""
    if G.has_edge(u, v) or u == v:
        return -float('inf')
    score = 0.0
    for w in nx.common_neighbors(G, u, v):
        score += 1.0 / np.log(G.degree(w))
    return score

import numpy as np
scores = []
nodes = list(G.nodes())
for i, u in enumerate(nodes):
    for v in nodes[i+1:]:
        if not G.has_edge(u, v):
            scores.append((u, v, aa_score(G, u, v)))

# 按分数排序，分数越高越可能连接
scores.sort(key=lambda x: x[2], reverse=True)
print("最有可能产生连接的Top-5节点对：")
for u, v, s in scores[:5]:
    print(f"  ({u}, {v}) -> AA分数: {s:.4f}")

基于路径的方法

相似性指标只考虑了直接邻居的信息，但有时候节点之间的连接需要"隔代"观察。这时候就需要基于路径的方法登场了。

Katz指数

Katz指数可以说是所有链路预测指标的"祖师爷"。它考虑了节点之间的所有路径，但给较短路径赋予更高的权重：

score(u, v) = Σ_{l=1}^{∞} β^l · |paths_{u,v}^{l}|

其中 |paths_{u,v}^{l}| 是连接 u 和 v 的长度为 l 的路径数量，β 是衰减因子（通常很小，比如0.005）。由于所有路径的总和可以通过矩阵运算高效计算：S = (I - βA)^{-1} - I，其中 A 是邻接矩阵。

关键洞察：Katz指数是"你能走多远"的全局度量。当 β 很小（接近0）时，Katz指数近似于共同邻居的数量（因为长路径的贡献几乎为零）。随着 β 增大，更长路径的信息也被纳入考量。这种从局部到全局的调节能力使得Katz成为链路预测领域的一个经典基准方法。

Local Path指数（LP）

Katz指数要考虑"所有长度"的路径，在大规模网络上计算开销很大。Local Path指数做了折中：只考虑长度为2和3的路径：

score(u, v) = A² + ε · A³

其中 A² 实际上就是共同邻居的数量（A²_{uv} = |Γ(u) ∩ Γ(v)|），ε 是长度为3的路径的权重参数。实践表明，LP指数在预测精度上与Katz相当，但计算效率高得多。

Random Walk with Restart（RWR）

RWR模拟一个随机游走者在网络中的漫游过程：从节点 u 出发，每一步以概率 c 回到起点，以概率 1-c 随机走向一个邻居。达到稳态后，停留在各节点的概率向量就反映了节点 u 到其他节点的"接近程度"。这种方法的优点是可以捕获网络中的全局拓扑信息，而且天然适用于有向图。

概率模型

相比基于相似性的启发式方法，概率模型从另一个角度切入——它们假设网络是由某个潜在的概率模型生成的，链路预测的任务就是推断这个模型并预测缺失的边。

随机分块模型（Stochastic Block Model, SBM）是其中最具代表性的一种。SBM假设网络中的节点属于若干个"块"（社团），节点之间的连接概率只取决于它们所属的块。如果两个节点属于同一个块，它们连接的概率较高；如果属于不同的块，连接概率较低。通过给定观察到的边集，我们可以使用EM算法或MCMC方法来推断每个节点最可能属于哪个块，然后计算节点对之间的连接概率。

层次结构模型（Hierarchical Structure Model）则提供了另一种视角。它假设网络存在一种树状的层次结构——无论是社团的嵌套还是其他组织形式。通过构建一棵分层树（dendrogram），我们可以计算任意两个节点之间的连接概率。这种方法在具有明显层次组织的网络中表现优异，比如蛋白质相互作用网络和超链接网络。

这些概率模型虽然在理论上更优雅，但计算成本通常也更高。实践中，人们往往在不同的场景下选择不同的方法：在需要快速计算的在线推荐系统中使用相似性方法，在追求精度的学术研究中尝试概率模型。

实操建议：如果在实际项目中需要使用链路预测，我建议你从最简单的CN指标开始，建立评估基准。然后逐步尝试AA、Katz等更复杂的指标，看哪个在你的数据集上表现最好。不要让"最优方法"的追求拖延了项目的落地——在大多数场景下，简单的相似性方法经过合理调参已经能达到可用的效果。

评估指标

有了各种预测方法，我们怎么知道哪个方法更好呢？常见的链路预测实验设置如下：将现有边集 E 随机分成训练集 E_train 和测试集 E_test（通常70%-30%或90%-10%的比例），在训练集上构建模型，预测测试集中的那些边是否出现。同时，还需要从未存在的节点对中随机选取同样数量的"负样本"。

链路预测领域最常用的评估指标有三个：

AUC（Area Under ROC Curve）衡量的是整体排序质量。它的含义是：从测试集中随机选取一条边，其得分高于从不存在边中随机选取一条边的概率。AUC越大越好，0.5表示随机猜测，1.0表示完美排序。

精确率（Precision）关注的是Top预测的效果：在得分最高的L个预测中，有多少是真正出现在测试集中的？定义 top-L 预测集合为测试集边所占的比例。这个指标在推荐系统中尤其重要——毕竟用户只会注意到Top-10推荐。

命中率（Hit Rate）考察的是测试集中的边是否出现在Top-L预测中。与精确率的区别在于，精确率考察的是"预测的前L个中对的比率"，而命中率考察的是"测试集中的有多少被预测到了"。两者侧重点不同，往往需要结合使用。

此外还需要注意交叉验证。由于网络的非独立同分布特性（边之间存在依赖关系），标准的k-fold交叉验证在链路预测中需要谨慎使用。一种常见的做法是"按时间划分"——如果数据带着时间戳，就应该用旧数据预测新边，而不是随机划分。这更贴近链路预测的"预测"本质。

应用案例

最后，我们看看链路预测在实际中的几个代表性应用。

推荐系统。淘宝的商品推荐有一个经典的链路预测视角：用户和商品构成一个二分图，用户-商品交互（点击、购买、收藏）是图中的边。链路预测要解决的问题是，从"尚未产生交互"的用户-商品对中找出最可能产生交互的那些。这正是协同过滤算法的数学本质。事实上，Amazon最早的"购买此商品的用户也购买了……"推荐功能，可以看作共同邻居指标在二分图上的直接应用。

生物网络重建。蛋白质相互作用网络的实验验证非常昂贵（一个相互作用的验证可能需要数千美元），但现有的已知相互作用只占全部可能相互作用的一小部分。链路预测可以帮助生物学家确定哪些实验最值得做。2010年的一项研究利用链路预测方法对人类蛋白质相互作用网络进行分析，预测出的Top候选中有超过60%在后续实验中被验证为真实相互作用——这个命中率远远高于随机筛选。

社交网络增长预测。Facebook、微博等社交平台的好友推荐本质上就是链路预测。通过结合网络结构特征（共同朋友、活跃社群等）和用户属性特征（学校、公司、地理位置等），推荐算法可以预测哪些人之间最可能建立新的朋友关系。特别是在新用户注册的"冷启动"阶段，合理的链路预测可以显著提高用户留存率——一个刚注册的用户如果能被快速推荐到真实社交圈子，ta留下的概率会高得多。

链路预测是一门"向网络要关系"的艺术。它告诉我们，看似随机的连接背后隐藏着可预测的规律。从相似性方法到概率模型，从简单的共同邻居到复杂的随机游走，链路预测的方法论体系既具有数学深度，又不缺乏实用性。
