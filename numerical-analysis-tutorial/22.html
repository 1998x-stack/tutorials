第22章 非线性方程组Newton法

大家好，我是你们的讲师。今天我们终于要讲一个真正在工程一线"大放异彩"的方法——非线性方程组的Newton法。

为什么说"真正"呢？因为单变量的非线性方程求根虽然常见，但在实际工程中，更多的场景是求解系统。比如：电路仿真中的节点电压由成百上千个非线性方程决定；化工过程模拟中的反应平衡条件是一个大型非线性系统；有限元分析中的接触问题的求解也是非线性方程组。可以说，"非线性"才是工程世界的常态，"线性"只是一个美好的近似。

22.1 多变量Newton法

我们从单变量推广到多变量。设F: ℝⁿ → ℝⁿ是一个向量值函数：

F(x) = [f₁(x₁, x₂, ..., xₙ), f₂(x₁, x₂, ..., xₙ), ..., fₙ(x₁, x₂, ..., xₙ)]T

我们要求解F(x) = 0。和一维情况一样，我们对F进行Taylor展开，取线性部分：

F(x) ≈ F(xk) + JF(xk) · (x - xk)

其中JF是F的Jacobian矩阵（雅可比矩阵），它是一个n×n矩阵，其(i,j)元素为∂fᵢ/∂xⱼ。令上述近似为零，得到多变量Newton迭代公式：

xk+1 = xk - JF(xk)⁻¹ · F(xk)

在实际计算中，我们不会显式求逆矩阵，而是求解线性方程组：

JF(xk) · s = -F(xk) ⇒ xk+1 = xk + s

这里s = -J⁻¹F称为Newton步。和一维情况一样，在适当的条件下（F足够光滑、J(x*)非奇异、初始猜测足够好），多变量Newton法也保持二次收敛。

多变量Newton法的几何解释

在一维情况下，我们"沿着切线走到x轴"。在多维情况下，我们在当前点用一个线性映射JF(xk)近似F，然后找到使这个线性映射为零的点。从流形的角度看，这是在每一步用切空间上的线性化来逼近原函数的零流形。这种"以直代曲"的思路是整个数值分析中最深刻、最强大的思想之一。

22.2 Jacobian矩阵的计算

多变量Newton法每步需要做两件事：计算Jacobian矩阵J(xk)，以及求解一个n×n线性系统J·s = -F。计算J有两种主要方式。

解析Jacobian：当F的表达式已知且导数可求时，手动推导或借助符号计算（如SymPy）生成解析表达式。这是最精确的方式，收敛性最好。但缺点是对于复杂系统，偏导数的推导极易出错。

有限差分近似：当解析Jacobian不可得时，用差商近似：

∂fᵢ/∂xⱼ ≈ (fᵢ(x + h·eⱼ) - fᵢ(x)) / h

其中eⱼ是第j个单位向量。这种方法需要额外计算n次函数值（如果采用前向差分），精度受h的选择影响，通常选择h = √εmachine · max(1, |xⱼ|)。

import numpy as np

def multivariate_newton(F, J, x0, tol=1e-12, max_iter=100):
    """
    多变量Newton法求解 F(x) = 0
    F: 函数句柄，输入x输出n维向量
    J: Jacobian函数句柄，输入x输出n×n矩阵。如果为None，使用有限差分
    """
    x = np.array(x0, dtype=float)
    n = len(x)

    for k in range(max_iter):
        Fx = F(x)
        if np.linalg.norm(Fx) < tol:
            return x, k + 1

        if J is not None:
            Jx = J(x)
        else:
            # 有限差分近似Jacobian
            h = 1e-8
            Jx = np.zeros((n, n))
            for j in range(n):
                x_plus = x.copy()
                x_plus[j] += h
                Jx[:, j] = (F(x_plus) - Fx) / h

        # 求解线性系统 J s = -F
        s = np.linalg.solve(Jx, -Fx)

        # 阻尼Newton法：自适应步长
        alpha = 1.0
        while np.linalg.norm(F(x + alpha * s)) >= np.linalg.norm(Fx) and alpha > 1e-10:
            alpha *= 0.5
        x = x + alpha * s

        if np.linalg.norm(s) < tol:
            return x, k + 1

    raise RuntimeError(f"超过最大迭代次数 {max_iter}")


# 示例：求解方程组
# x^2 + y^2 = 4
# x^2 - y^2 = 1
def F_example(x):
    return np.array([
        x[0]**2 + x[1]**2 - 4,
        x[0]**2 - x[1]**2 - 1
    ])

def J_example(x):
    return np.array([
        [2*x[0], 2*x[1]],
        [2*x[0], -2*x[1]]
    ])

踩坑故事：漏掉Jacobian非对角项造成的灾难

我在做化学反应网络模拟时，需要求解几十个非线性方程组成的系统。一开始图省事，我认为"各个变量之间耦合很弱"，把Jacobian的非对角元素全部设为零，只保留对角线上的偏导数。结果迭代完全不收敛——残差不但不减小，反而一路上升到无穷大。我花了两天时间调试，以为代码有bug，直到重新审视问题才发现：反应网络中各组分浓度之间通过质量作用定律存在强耦合，忽略这些耦合项相当于完全抛弃了系统结构信息。这个故事提醒我们：Jacobian矩阵中每个非零元素都携带了变量之间相互作用的信息，在构建Jacobian时绝不能走捷径。正确的做法是先用自动微分或符号计算工具生成精确的Jacobian，然后再根据稀疏结构做优化。

22.3 拟Newton法(Broyden)

多变量Newton法虽然收敛快，但每步的计算成本很高：计算Jacobian需要O(n²)次偏导数（或n次函数评估），求解线性系统需要O(n³)次运算。当n很大时（比如n > 1000），这个代价变得难以承受。

Broyden方法（拟Newton法的一种）巧妙地解决了这个问题。它的想法是：不每一步都重新计算Jacobian，而是用一个低秩更新来近似Jacobian的变化：

Jk+1 = Jk + (y - Jks) · sT / (sTs)

其中s = xk+1 - xk，y = F(xk+1) - F(xk)。这个更新的含义是：使近似Jacobian满足割线条件Jk+1·s = y，并且在所有满足割线条件的矩阵中，Jk+1是改变最小的那个（在Frobenius范数意义下）。

Broyden方法的效率

更棒的是，我们可以用Sherman-Morrison公式直接更新Jacobian的逆：

Hk+1 = Hk + (s - Hky) · sTHk / (sTHky)

其中Hk = Jk⁻¹的近似。这样一来，每步只需要O(n²)的向量更新操作——比Newton法的O(n³)快一个数量级！

def broyden(F, x0, tol=1e-12, max_iter=200):
    """
    Broyden拟Newton法求解 F(x) = 0
    使用Sherman-Morrison公式更新Jacobian的逆
    """
    x = np.array(x0, dtype=float)
    n = len(x)
    Fx = F(x)

    # 初始Jacobian用有限差分或单位矩阵
    H = np.eye(n)  # 近似 J^{-1}

    for k in range(max_iter):
        if np.linalg.norm(Fx) < tol:
            return x, k + 1

        s = -H @ Fx        # Newton步
        x_new = x + s
        F_new = F(x_new)
        y = F_new - Fx

        # Sherman-Morrison更新 H = H + (s - H y) s^T H / (s^T H y)
        denom = s @ (H @ y)
        if abs(denom) > 1e-15:
            H = H + np.outer(s - H @ y, s @ H) / denom

        x = x_new
        Fx = F_new

    return x, max_iter

故事：CFD求解器中的Broyden法

我参与过一个计算流体力学(CFD)项目，求解Navier-Stokes方程的离散化系统，自由度约5万。使用标准Newton法时，每步构建和分解Jacobian矩阵需要约30秒，总共需要约15步收敛——总耗时约7.5分钟。对于稳态计算这还可以接受，但设计参数扫描需要跑几百个工况，时间就变得无法忍受了。换用Broyden方法后：初始几步用有限差分构建精确Jacobian（花费2次函数评估），之后只做低秩更新。每步的时间从30秒降至约0.5秒（主要是函数评估的时间），收敛在约25步内完成。总耗时从7.5分钟降至约20秒——节省了90%以上的Jacobian计算时间！

22.4 实际求解策略

在现实中，非线性方程组常常高度病态、变量尺度差异巨大、或者初始猜测难以获得。这就需要一些更高级的策略。

延拓法（Homotopy Method）：将一个容易求解的问题逐渐变形为目标问题。构造一条路径：

H(x, t) = (1-t)F₀(x) + tF(x) = 0，t ∈ [0, 1]

其中F₀(x) = x - a（或另一个有显式解的问题）。从t=0开始，沿着参数t = 0 → 1逐步追踪解曲线。每一步t的变化量Δ都很小，所以当前解可以作为下一步Newton法的好初值。延拓法特别适合那些Newton法对初值极其敏感的问题。

全局化策略：为Newton法添加全局收敛保障。

  • 线搜索(Line Search)：在Newton方向s上，根据Armijo准则或Wolfe条件选择步长α，保证函数值（或残差范数）充分下降。
  • 信赖域(Trust Region)：在当前点周围定义一个"信赖区域"，在该区域内极小化二次模型。如果模型预测与实际下降一致性好，扩大区域；否则缩小。这比线搜索更鲁棒，能处理Jacobian奇异的情况。

工具箱推荐：SciPy的root函数

在实际工程中，我很少自己手写多变量Newton法，更常用的策略是利用SciPy的scipy.optimize.root函数。它提供了多种方法选项：'hybr'（改进的Powell混合法，类似MINPACK的hybrd，默认方法），'lm'（Levenberg-Marquardt），'broyden1'/'broyden2'等。对于大部分实际工程问题，默认的'hybr'方法就非常可靠。但如果需要调整求解策略，理解这些方法背后的原理（即我们本章讲的内容）至关重要——没有对原理的深入理解，你连参数都不知道该怎么调。

最后，我想强调一个重要的观点：理解问题结构远重要于选择求解器。同样一个非线性方程组，如果变量之间存在天然的层次结构（比如有些变量是"快变量"，有些是"慢变量"），采用分块Newton法或Gauss-Seidel型迭代往往比直接对所有变量同时Newton迭代更有效。这也是为什么实际工程中，数值方法永远需要与领域知识相结合。

到此为止，关于非线性方程（组）求根的讨论就结束了。我们从一维的二分法、不动点迭代讲到了多维的Newton法和Broyden法，从线性收敛讲到了二次收敛，从简单实用讲到了大规模高效——希望这一路走来，大家对"如何让计算机求解非线性问题"有了系统和深入的理解。
