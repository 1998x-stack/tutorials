第17章 MPNet：掩码语言模型与排列语言模型的统一框架

同学们好。前面两章我们分别学习了ELECTRA的判别式预训练和XLNet的排列语言模型。你可能会想：BERT的MLM和XLNet的PLM能否合二为一，各取所长？这正是微软研究院在arXiv:2004.09297（Song et al.，2020）中提出的MPNet（Masked and Permuted Language Network）所做的核心贡献。MPNet不是简单的拼接，而是从数学上统一了两个框架，在检索和语义匹配任务上展现了极强实力。

核心思想：对被mask的位置用排列语言建模（利用XLNet的优点），对未被mask的位置保留BERT的双向上下文建模——一个模型兼具两者的优势。

17.1 BERT和XLNet各自的缺陷

要理解MPNet为什么重要，我们需要先看清BERT和XLNet各自的"心病"：

模型 优势 不足
BERT (MLM) 双向上下文，依赖关系建模充分 预训练中有[MASK]，微调中没有；被mask的token之间相互独立
XLNet (PLM) 无[MASK]标记，自回归预测自然，适合生成 只对被预测的token建模依赖，未预测的token位置之间缺乏双向建模

BERT的MLM假设被mask的token之间相互独立（给定上下文下，预测x₂时不知道x₃被mask成了什么），而XLNet的PLM只关注预测位置的token依赖，对被预测token以外的位置忽略了两两之间的交互关系。这两者都不完美。

17.2 MPNet的统一方案：掩码+排列

MPNet的改进思路非常清晰：

  1. 保留BERT的双向建模：对于输入序列中未被mask的位置，使用BERT-style的双向注意力，让所有可见位置互相看到（包括非mask位置之间、非mask位置看mask位置）。
  2. 引入XLNet的排列：对于被mask的位置，采用排列语言建模的方式，让它们在自回归框架下按照随机顺序（通过注意力掩码控制）依次被预测。
  3. 位置补偿机制：在排列过程中，每个token的原始位置信息保持不变，确保预测时能准确知道每个token在原始序列中的位置。
形象理解：想象你在拼拼图。MPNet的做法是：已拼好的部分（未被mask的位置）你可以自由地双向观看；对于空缺的部分（被mask的位置），你按照一个随机顺序一块一块地拼——每拼一块时你可以看到已拼好的部分和之前已拼好的空缺位置。

17.3 位置补偿机制详解

位置补偿是MPNet的一个关键设计。为什么需要它呢？

在XLNet中，当排列顺序改变时，每个token在排列中的"相对位置"改变了，但"原始位置"信息丢失了。举个例子，原始句子是"[我] [爱] [你]"，如果排列是[3, 1, 2]，那么预测"你"时，"你"的位置编码可能是"位置1"（排列中的第一个），但它在原文中的实际位置是"位置3"。

MPNet通过一个位置补偿项，在排列建模中同时保留了：

  • 原始位置编码：token在原始序列中的绝对位置（告诉模型这个token在原文中的位置）。
  • 排列位置信息：token在当前排列中的预测顺序（告诉模型当前要预测哪个位置）。

两者结合，模型既知道"我在预测原文的第3个词"，又知道"当前排列顺序中它排第1位"。这比XLNet中只使用排列位置信息更加精确。

效果对比：在MS MARCO段落检索任务上，MPNet比同等规模的BERT和XLNet高出2-3个点的MRR。在语义相似度任务（STS-B）上，MPNet也优于两个基线。这说明"位置补偿"对理解语义细节至关重要。 
# MPNet 位置补偿机制示意
def mpnet_position_encoding(original_pos, perm_order):
    """
    original_pos: 原始位置编码 [batch, seq_len, d_model]
    perm_order: 当前排列顺序 [batch, seq_len]
    """
    batch, seq_len, d_model = original_pos.shape

    # 1. 原始位置编码（BERT风格）
    pos_original = sinusoidal_position_encoding(original_pos)

    # 2. 排列后的位置编码（XLNet风格）
    # 根据排列顺序重新组织位置编码
    perm_pos = torch.gather(pos_original, 1,
                            perm_order.unsqueeze(-1).expand(-1, -1, d_model))

    # 3. 位置补偿：将原始位置信息融合到排列注意力中
    # 对非mask位置：使用原始位置编码（双向建模）
    # 对mask位置：使用排列位置编码 + 原始位置偏移补偿
    pos_compensated = torch.where(
        is_masked.unsqueeze(-1),
        perm_pos + positional_offset(original_pos, perm_order),
        pos_original
    )

    return pos_compensated

17.4 MPNet vs BERT vs XLNet：三方大战

对比维度 BERT XLNet MPNet
核心机制 去噪自编码（MLM） 排列自回归（PLM） 掩码+排列（统一框架）
双向上下文 全部位置 仅通过排列间接实现 未mask位置完全双向，mask位置排列双向
[MASK]标记 有 无 仅用于标记被预测的位置（不参与模型输入）
位置编码 绝对位置编码 相对位置编码 原始位置+排列位置补偿
训练效率 基准 较慢（双流注意力+排列采样） 接近BERT（比XLNet快）
GLUE (Base) 82.2 85.0 ~85.5
语义匹配（STS-B） 84.5 85.5 86.8（表现突出）
检索（MS MARCO） MRR 26.5 MRR 27.8 MRR 29.0（优势明显）
踩坑实录：有次我把MPNet作为教师模型做知识蒸馏，结果学生模型在分类任务上表现还行，在检索任务上却差了一大截。排查发现是我蒸馏时只传了最后一层的logits，没有蒸馏中间层的"位置补偿"特征。MPNet的位置补偿机制需要在中间层也加以约束，蒸馏效果才会好。后来加上中间层蒸馏后，学生模型的检索性能提升了5个点。

17.5 MPNet的适用场景

基于MPNet的特性，下面这些场景尤其适合使用MPNet：

  • 语义搜索（Semantic Search）：MPNet在段落检索上的优势是三个模型中最突出的，如果你在做信息检索相关的任务，MPNet是一个非常强有力的基线。
  • 文本匹配（Text Matching）：语义相似度计算需要精细捕捉词序和位置信息，MPNet的位置补偿机制正好派上用场。
  • 知识蒸馏的教师模型：MPNet优越的表示能力使其成为理想的"教师"，尤其在为检索模型做蒸馏时优势明显。
  • 需要双向上下文+自回归输出的混合场景——不过这种场景比较少见。

17.6 MPNet的局限

当然，MPNet也并非无懈可击：

  • 模型复杂度较高：同时实现了掩码建模和排列建模，代码实现和调试难度高于单一的BERT或XLNet。
  • 在标准NLU任务上的提升有限：在GLUE榜单上，MPNet相较于ELECTRA和XLNet的提升并不显著（约0.5个点），主要优势体现在检索和匹配任务上。
  • 社区生态不如BERT丰富：官方开源的MPNet预训练模型数量和后续微调实践远不如BERT丰富，在特定领域的预训练模型（如生物医疗、法律）几乎没有。

总的来说，MPNet在技术上是一个非常漂亮的"缝合"工作，它揭示了MLM和PLM本质上可以在一个统一的数学框架下共存。如果你做的是检索或语义匹配方向，MPNet值得一试。下一章我们将学习ALBERT——它从一个完全不同的角度（参数高效）对BERT进行了极致优化。

本章小结：MPNet通过融合BERT的掩码建模和XLNet的排列建模，在统一框架下兼得两者的优势。位置补偿机制是其核心创新，使得模型在检索和语义匹配任务上表现尤为突出。对于语义搜索和文本匹配方向的实践者，MPNet是一个值得认真对待的选项。
