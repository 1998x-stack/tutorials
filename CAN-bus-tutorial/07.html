第7章：CAN收发器——信号世界的翻译官

各位同学，今天我们来聊聊CAN收发器。说实话，很多初学者觉得收发器就是个电平转换芯片，没什么好学的。但我在汽车电子干了这么多年，可以负责任地告诉你：收发器选不好，整个CAN网络都可能瘫痪。

CAN收发器，说白了就是连接CAN控制器和物理总线的桥梁。控制器输出的是逻辑电平（0或1），而总线上跑的是差分信号。收发器就是负责这两者之间的翻译工作。

7.1 收发器在CAN系统中的位置

先看一张典型的CAN节点框图：

┌─────────────────────────────────┐
│          MCU/微控制器            │
│  ┌──────────┐    ┌──────────┐   │
│  │ CAN控制器 │←──→│  SPI/I2C │   │
│  │ (内置)    │    │ 等外设   │   │
│  └─────┬────┘    └──────────┘   │
│        │ TXD/RXD                 │
└────────┼────────────────────────┘
         │
    ┌────┴────┐
    │ CAN收发器│  ← 我们今天的主角
    │ TJA1050 │
    └────┬────┘
         │ CANH/CANL
    ┌────┴────┐
    │ 物理总线 │
    │ 双绞线   │
    └─────────┘

收发器有四个关键引脚：

  • TXD：发送数据输入（来自控制器）
  • RXD：接收数据输出（给控制器）
  • CANH：CAN高电平总线
  • CANL：CAN低电平总线

嗯，这里要注意：TXD和RXD是逻辑电平（通常是3.3V或5V），而CANH/CANL是差分信号。收发器就是在这两种信号之间来回转换。

7.2 经典收发器：TJA1050

TJA1050是NXP（原飞利浦）的经典产品，也是我最早接触的CAN收发器之一。记得2008年做第一个车载项目时，用的就是它。到现在，这颗芯片依然活跃在很多设计中。

主要特性：

参数 典型值 说明
工作电压 5V 单电源供电
总线电压 CANH: 3.5V, CANL: 1.5V 显性位时
速率 最高1Mbps 实际常用250k/500k
工作温度 -40℃ ~ +125℃ 车规级
待机电流 ＜5μA 低功耗模式

引脚功能：

  • Pin1 TXD：发送数据输入。低电平驱动显性位，高电平驱动隐性位
  • Pin2 GND：接地
  • Pin3 VCC：5V电源
  • Pin4 RXD：接收数据输出。总线显性时输出低电平
  • Pin5 VREF：参考电压输出（1/2 VCC），用于共模偏置
  • Pin6 CANL：CAN低电平总线
  • Pin7 CANH：CAN高电平总线
  • Pin8 S：静音模式控制（高电平进入静音）
我的小技巧： 在设计PCB时，我习惯在TXD和RXD引脚上各加一个100Ω的串联电阻。这能有效抑制信号反射，特别是在长走线的情况下。有一次调试一个干扰严重的项目，就是靠这两个电阻搞定的。

7.3 另一款常用收发器：SN65HVD230

TI的SN65HVD230是3.3V系统的首选。现在很多MCU都是3.3V供电，如果用TJA1050还得加电平转换，而HVD230可以直接对接。

与TJA1050的主要区别：

参数 TJA1050 SN65HVD230
供电电压 5V 3.3V
逻辑电平 5V TTL 3.3V CMOS
总线电压 CANH: 3.5V, CANL: 1.5V CANH: 2.4V, CANL: 0.9V
待机模式 通过S引脚控制 通过RS引脚控制
斜率控制 无 有（通过Rs引脚）

HVD230的Rs引脚很有意思。接不同电阻可以控制上升/下降斜率：

  • Rs接地：高速模式（斜率最快）
  • Rs接10kΩ到GND：中等斜率
  • Rs接100kΩ到GND：低斜率（适合低速、抗干扰）

我个人建议：如果总线长度超过10米，或者环境干扰大，可以把斜率调低一点。虽然速率会受影响，但可靠性提升明显。

7.4 电气特性详解

收发器的电气特性直接决定了通信的可靠性。我挑几个关键参数说说：

1. 差分输出电压

显性位时，CANH - CANL 的电压差：

  • TJA1050：约2V（CANH=3.5V, CANL=1.5V）
  • SN65HVD230：约1.5V（CANH=2.4V, CANL=0.9V）

隐性位时，CANH和CANL都偏置到2.5V（或1.2V），电压差接近0V。

2. 共模电压范围

CAN标准要求收发器能承受-2V到+7V的共模电压。为什么？因为总线两端的地电位可能有差异。我曾经在一条50米长的总线上测到过3V的地电位差，如果收发器共模范围不够，通信直接就挂了。

3. 输入阈值

接收器的差分输入阈值：

  • 显性位识别：差分电压 > 0.9V
  • 隐性位识别：差分电压 < 0.5V
  • 中间区域（0.5V~0.9V）：不确定区，应避免
避坑指南： 我曾经遇到过一个项目，总线上的显性位差分电压只有0.8V。理论上应该能识别，但实际测试时偶尔会丢帧。后来发现是收发器供电电压偏低，导致驱动能力不足。所以，设计时一定要留足余量，显性位差分电压最好做到1.5V以上。

7.5 共模信号与差模信号

这是很多初学者容易混淆的概念。我尽量用大白话讲清楚。

差模信号： 就是CANH和CANL之间的电压差。这是我们真正要传输的信号。显性位时差2V，隐性位时差0V。接收器就是靠检测这个差值来还原数据的。

共模信号： 是CANH和CANL相对于GND的平均电压。比如CANH=3.5V, CANL=1.5V，共模电压就是(3.5+1.5)/2=2.5V。

为什么CAN要用差分信号？说白了就是为了抗共模干扰。你想想看，如果外部电磁干扰同时耦合到CANH和CANL上，两根线上的干扰幅度几乎相同。差模信号（CANH-CANL）基本不受影响。这就是差分传输的魔力。

实际波形示例：

示波器看到的CANH和CANL波形（显性位+隐性位）：

电压
│
5V ┤
   │     ┌───┐         ┌───┐
3.5V┤─────┤   ├─────────┤   ├──── CANH
   │     │   │         │   │
2.5V┤─────┤   ├─────────┤   ├──── 共模电平
   │     │   │         │   │
1.5V┤─────┤   ├─────────┤   ├──── CANL
   │     └───┘         └───┘
0V ┤
   └────────────────────────────── 时间
       显性   隐性    显性

注意看：显性位时，CANH被拉到3.5V，CANL被拉到1.5V，差分电压2V。隐性位时，两根线都回到2.5V，差分电压0V。

7.6 收发器选型建议

根据我多年的经验，选型时考虑这几点：

  1. 供电电压：5V系统用TJA1050，3.3V系统用SN65HVD230
  2. 速率要求：普通应用500kbps以内，两款都够用
  3. 功耗要求：需要低功耗待机的，选带待机模式的型号
  4. EMC性能：TJA1050的EMC表现更好，适合恶劣环境
  5. 温度范围：车规必须选-40℃~125℃的

核心要点总结：

  • 收发器是控制器和总线之间的桥梁
  • TJA1050是5V系统的经典选择，SN65HVD230适合3.3V系统
  • 差分信号是CAN抗干扰的关键
  • 共模电压范围决定了系统的鲁棒性
  • 选型时留足余量，特别是差分电压和共模范围
