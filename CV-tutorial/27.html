27.1 扩散模型原理 — DDPM到Stable Diffusion

如果你在过去两年关注过AI图像生成，你一定听说过"扩散模型"（Diffusion Models）这个术语。从DALL-E 2到Stable Diffusion，从Midjourney到Sora——这些令人惊叹的图像和视频生成工具的核心引擎都是扩散模型。这是一个从物理热力学中获得启发的生成范式，它的出现彻底取代了GAN在图像生成领域的统治地位。

故事要从2015年说起。当时Jascha Sohl-Dickstein等人提出了扩散模型的基础框架，但当时的生成效果远不如GAN。直到2020年，Ho等人的里程碑论文DDPM（Denoising Diffusion Probabilistic Models）才真正展示了扩散模型的潜力。DDPM的核心思想可以这样理解：前向过程——不断地在图像上加高斯噪声，直到图像完全变成随机噪声；反向过程——训练一个神经网络从随机噪声中一步步去噪，还原出原始图像。

DDPM的前向过程是一个马尔可夫链：每一步给数据添加少量噪声，经过足够多的步数（DDPM用了1000步），图像完全变成标准高斯噪声。反向过程则是学习一个参数化的去噪网络，预测每一步的噪声，逐步还原图像。DDPM的损失函数形式极其简洁：最小化预测噪声和实际噪声之间的MSE。

💡 直觉理解：可以把扩散模型想象成一个"反向的雕塑过程"。前向过程就像把一个完美的雕塑逐渐风化成一堆碎石；反向过程就是学习如何把碎石一步步拼回雕塑。模型不是直接预测最终的雕塑，而是每一步只学习"把这堆碎石稍微修整一下"——这让问题变得简单了很多。

然而DDPM的一个巨大问题是：生成速度太慢。1000步的迭代采样意味着生成一张图需要数秒甚至数十秒的推理时间。2021年，Song等人提出的DDIM（Denoising Diffusion Implicit Models）将采样步数减少到50-100步。更重要的突破来自Rombach等人的Latent Diffusion Model（LDM），也就是后来的Stable Diffusion——它不是在原始像素空间做扩散，而是在VAE压缩后的隐空间（Latent Space）中做扩散。图像被压缩到原来大小的1/48（从512×512×3压缩到64×64×4），扩散过程在低维隐空间中运行，计算量大幅降低，同时生成质量不减反增。

        def ddpm_training(images, model, noise_scheduler):
            # 对每张图像随机采样一个时间步 t
            t = torch.randint(0, T, (images.shape[0],))  # T=1000

            # 生成对应时间步的随机噪声
            noise = torch.randn_like(images)

            # 前向加噪：x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1 - alpha_bar_t) * epsilon
            noisy_images = noise_scheduler.add_noise(images, noise, t)

            # 模型预测噪声（U-Net架构）
            predicted_noise = model(noisy_images, t)

            # 损失函数：预测噪声与实际噪声的MSE
            loss = F.mse_loss(predicted_noise, noise)

            return loss

        # 采样（推理）过程
        def ddpm_sampling(model, noise_scheduler, shape):
            # 从纯噪声开始
            x_t = torch.randn(shape)

            # 从 T 到 1 逐步去噪
            for t in reversed(range(T)):
                predicted_noise = model(x_t, t)
                x_t = noise_scheduler.step(predicted_noise, t, x_t)

            return x_t  # 生成的图像

我在2023年尝试用Stable Diffusion做一个工业质检的数据增强项目。我们的检测数据集只有2000张良品和300张不良品，训练一个YOLOv8模型严重过拟合。我尝试用ControlNet + SD生成不同形态的不良品样本——通过边缘图控制生成物体的形状，通过深度图控制空间布局。最终生成了3000张合成不良品图像，将模型的mAP从0.43提升到了0.67。这个项目让我深刻体会到：扩散模型不仅是"好玩"的生成工具，它们正在成为解决实际数据稀缺问题的利器。

27.2 视觉语言模型（VLM）— BLIP、LLaVA、GPT-4V

扩散模型让我们可以"从文生图"，那么反过来呢？视觉语言模型（Vision-Language Model, VLM）的目标是让AI能够"看图说话"——理解图像内容并用自然语言表达出来，甚至与人类进行视觉对话。

早期的VLM可以分为两大类：基于双塔架构的（如CLIP、ALIGN）和基于融合编码器的（如UNITER、ViLBERT）。双塔模型擅长对比/检索任务但缺乏细粒度的视觉理解；融合模型理解能力强但计算量大。BLIP（Bootstrapping Language-Image Pre-training）在2022年巧妙地结合了两种架构的优势，并引入了一个重要的技术创新——字幕滤波（Caption Filtering）：利用互联网嘈杂的图文数据训练后，再生成高质量的字幕来过滤或增强训练数据，形成一个自我改进的循环。

2023年，LLaVA（Large Language and Vision Assistant）的开源发布标志着VLM进入了一个新的阶段。LLaVA的核心思路极其优雅：将一个预训练的视觉编码器（CLIP的ViT）通过一个简单的线性投影层连接到LLM（LLaMA/Vicuna）。视觉编码器提取图像特征，投影层将视觉特征映射到LLM的embedding空间，然后LLM基于视觉和文本的双重输入进行理解和推理。这个简洁的设计使得LLaVA具备了强大的多模态对话能力——你可以给它一张图片，然后问它"这张图里的人在做什么？""背景中的建筑是什么风格的？"

GPT-4V（2023）和后来的GPT-4o（2024）则代表了商业VLM的巅峰。它们不仅能理解图像内容，还能进行复杂的推理——比如看一张手绘的架构图然后指出设计中的缺陷、阅读照片中的菜单并推荐菜品、甚至通过摄像头实时指导你组装家具。这些能力的背后是比开源VLM大得多的模型规模和更精细的多模态对齐训练。

🔮 趋势：VLM的发展正在模糊CV和NLP的边界。当模型可以理解并对图像进行复杂推理时，"视觉"就不再是一个独立的任务，而是LLM感知世界的一种方式。这种趋势越往后走越明显——2024-2025年的多模态大模型几乎都采用了"一个统一的Transformer + 视觉tokens"的架构。

27.3 图像生成经典模型 — GAN与Diffusion

在扩散模型崛起之前，GAN（Generative Adversarial Network）是图像生成的绝对王者。理解GAN为什么被扩散模型取代，以及各自的优势，对于理解图像生成技术的演进至关重要。

GAN由Ian Goodfellow于2014年提出，核心思想是一个"造假者"（生成器G）和一个"鉴定师"（判别器D）的博弈：G从随机噪声生成假图像，D判断图像是真还是假。两者在对抗中共同进步——G越来越擅长以假乱真，D越来越火眼金睛。这个过程被称为对抗训练（Adversarial Training）。

StyleGAN系列（2019-2021）是GAN技术集大成者。StyleGAN的核心创新包括：映射网络（Mapping Network）将随机噪声映射到解耦的隐空间、自适应实例归一化（AdaIN）来控制风格、以及渐进式增长训练。StyleGAN生成的逼真人脸曾经达到了"真假难辨"的水平。但它也有两个致命弱点：模式坍塌（生成器只学会生成小部分模式，缺失多样性）和训练不稳定（G和D之间的平衡极其脆弱，稍有不慎训练就崩溃）。

27.4 AIGC的视觉应用 — ControlNet与图像编辑

如果说扩散模型提供了"原料"，那么AIGC（AI Generated Content）领域的各种工具和方法就是"加工工艺"。这里最值得一提的就是ControlNet——它让文生图从"全凭运气"变成了"精准可控"。

ControlNet由张吕敏（Lvmin Zhang）在2023年初提出。它解决的痛点非常直观：用Stable Diffusion生成图像时，你只能通过文本描述来控制结果——"a red car on a mountain road"——但你无法精确控制车的姿态、位置、大小。ControlNet的方法是在扩散模型中额外引入一个条件控制通道：你可以提供一张边缘图（Canny edge）、深度图、姿态骨骼图、甚至是另一张图像的语义分割图，ControlNet会将这些条件注入到扩散过程的不同层中，精确引导生成结果的几何结构和空间布局。

⚠️ 踩坑提醒：ControlNet的权重需要根据不同的控制类型选择——Canny控制对边缘敏感但容易生成"描边填色"的僵硬效果；深度控制能保持空间结构但可能丢失纹理细节；姿态控制对人物姿态非常精确但有时会扭曲背景。实践中最常用的组合是"边缘+深度"双ControlNet，兼顾几何精度和自然度。但这也意味着推理时间翻倍——需要在质量和速度之间做权衡。

另一个重要的AIGC方向是图像编辑。InstructPix2Pix（2023）允许你通过自然语言指令编辑图像——"把背景改成日落""让这个人戴一顶帽子""把地上的雪去掉"。它的训练数据是通过组合GPT-3生成的编辑指令和由另一个扩散模型生成的编辑前/后图像对来构造的。DreamBooth和LoRA等技术则让你能用少量图像（3-5张）对一个扩散模型进行个性化微调，使其学会生成特定对象（比如你的宠物狗）在不同场景下的图像。

27.5 3D视觉前沿 — NeRF与3D Gaussian Splatting

我们前面讨论的都还停留在2D图像。那么CV的下一个前沿在哪里？3D视觉无疑是重要的方向之一。这里必须要提到两个改变游戏规则的技术：NeRF和3D Gaussian Splatting。

NeRF（Neural Radiance Fields）由Mildenhall等人于2020年提出。它的核心思想是用一个MLP神经网络隐式地表示一个3D场景——输入一个3D空间点和观察方向(x, y, z, theta, phi)，网络输出该点的密度和颜色。通过从多个视角拍摄的一组2D图像，NeRF可以学习到完整的3D场景表示，然后从任意新视角渲染出逼真的新视图。NeRF的效果令人惊叹——新视角渲染的细节丰富度远超传统的3D重建方法。

但NeRF有两个主要痛点：训练速度慢（一个场景需要数小时到数十小时）和推理速度慢（渲染一帧需要数秒）。后续的改进工作如Instant NGP（Instant Neural Graphics Primitives）通过哈希编码和多分辨率特征网格将训练时间缩短到秒级。而2023年的3D Gaussian Splatting则完全换了一种思路——不再用隐式神经表示，而是使用数百万个显式的3D高斯椭球体来直接建模场景。这种方法在保持NeRF级渲染质量的同时，实现了实时（30+ FPS）的新视角渲染。我在2024年初尝试用3D Gaussian Splatting重建一个历史建筑的外观，用手机拍摄了50张照片，训练只用了15分钟，渲染效果媲美专业3D扫描。这项技术正在被广泛用于数字孪生、VR/AR、电影特效等领域。

💡 前瞻：3D Gaussian Splatting的一个重要趋势是与生成模型的结合。2024年已经有工作尝试用扩散模型直接从单张图片生成3D Gaussian表示（如MVDream、DreamGaussian），这预示着一个"从单张照片直接生成可交互3D场景"的未来并不遥远。

27.6 CV的未完成挑战

经过前面五个小节的介绍，你可能会觉得CV已经发展得非常成熟了——图像分类超过人类、生成效果逼真、模型通用化趋势明显。但作为一个负责任的老师，我想在这个章节的最后，坦诚地告诉你CV领域还有哪些未完成的挑战。这些挑战既是这个领域的"硬骨头"，也是你未来可以深入钻研的方向。

  • 鲁棒性（Robustness）：对抗样本的存在——对人眼不可感知的像素扰动就能让最先进的分类器出错——揭示了深度学习模型的根本脆弱性。在自动驾驶、医疗影像等安全关键场景中，鲁棒性问题还没有被真正解决。
  • 可解释性（Interpretability）：即使是最先进的ViT模型，我们仍然无法完全理解其内部决策过程。一位医生如果使用AI辅助诊断，他需要知道"为什么模型认为这片区域是病变？"——目前的可视化方法（GradCAM、注意力图）只能提供粗略的解释，远远不够。
  • 少样本学习（Few-Shot Learning）：人类可以从一两张照片中就学会识别一个新物体，而CV模型仍然需要数十到数百张标注图像才能达到可用的精度。基础模型（如CLIP、SAM）显著改善了这个差距，但没有完全消除。
  • 具身智能（Embodied Intelligence）：目前的CV系统大多是"被动观察者"——它们看图像、做判断，但不与环境互动。真正的智能应该能够像人类一样在三维空间中移动、操作物体、与环境交互。具身AI要求CV不仅理解"看到什么"，还要理解"我如何与它交互"。
  • 长尾分布（Long-Tail Distribution）：现实世界的数据分布是严重不均衡的——常见类别占据了大多数样本，而大量罕见类别只有极少样本。大多数CV模型在长尾分布上的表现远差于在均衡数据集上的表现。
  • 持续学习（Continual Learning）：人类可以持续学习新知识而不会忘记旧知识。CV模型则面临"灾难性遗忘"的问题——在学会识别新的10类物体后，之前学会的100类物体的识别准确率会显著下降。

这张清单并不是让人灰心，反而应该让人振奋——因为它意味着CV领域仍有大量值得投入的开放问题。每一代研究者都会推动这些问题向前一步。你完全有可能在某个方向上做出你的贡献。

好了，第27章到这里就结束了。从扩散模型的精妙原理到VLM的多模态理解，从AIGC的创意应用到3D视觉的前沿突破，再到CV领域尚未解决的挑战——你看到了一个蓬勃发展但远未成熟的技术画卷。下一章也是本课程的最后一章，我们将把所有28章的知识串联起来，进行一个综合实战项目——从需求分析到数据准备，从模型训练到部署上线，完整体验一个CV项目的全流程。
