第04章 整数规划与建模

大家好，我是你们的讲师。前两章我们深入了线性规划的世界——连续变量的优化。但现实世界中，很多决策不是连续的：你不可能建"2.7 条"装配线，也不可能派"0.3 个"工人去完成一项任务。这些决策本质上是离散的，需要整数规划（Integer Programming, IP）。

如果把线性规划比作"在连续的光滑曲面上滑行"，那整数规划就是"在离散的阶梯上跳跃"——看似只多了一个"整数"限制，求解难度却天差地别。这一章我们先从整数规划的分类开始，重点掌握 0-1 变量的建模技巧，然后拆解几个经典模型，最后理解为什么整数规划比线性规划难那么多。

整数规划的三种类型

根据整数变量的范围和分布，整数规划分为三大类：

  • 纯整数规划（Pure Integer Programming）：所有决策变量都要求取整数值。例如："安排多少辆卡车执行运输任务"——卡车数量必须是整数。如果有 n 个变量，就是 min cᵀx, s.t. Ax ≥ b, x ∈ ℤⁿ⁺。
  • 混合整数规划（Mixed Integer Linear Programming, MILP）：部分变量要求整数，部分可以是连续值。这是工业界最常见的类型。例如：是否新建工厂（0-1 决策）+ 工厂投产后生产多少产品（连续决策）。
  • 0-1 规划（Binary Integer Programming）：也叫二元规划，变量只能取 0 或 1。特别适合表示"选/不选"、"做/不做"的决策。背包问题、设施选址、指派问题都属于这一类。

MILP 是工业界最常用的模型框架——据 Gurobi 的用户统计，超过 70% 的生产环境优化模型都是 MILP。

0-1 变量的建模技巧 — 逻辑约束的数学表达

0-1 变量的真正威力不在于它本身的值，而在于它作为逻辑开关的能力。通过组合多个 0-1 变量和约束，我们可以表达几乎任何业务逻辑。

固定费用问题

假设你要决定是否开启一条生产线。开启需要一次性准备费用（set-up cost），比如设备调试费 5000 元。如果生产，还产生单位可变成本。可变成本是连续的，但固定费用只有"开启"才发生。

变量：
  x = 生产数量（连续，≥ 0）
  y = 是否开启生产线（0-1 变量）

目标函数：
  min  可变成本·x + 5000·y

约束：
  x ≤ M·y     ← 关键约束！
  x ≥ 0, y ∈ {0, 1}

这里的 M 被称为 Big-M，是一个足够大的常数（大于 x 所有可能取值的上界）。当 y = 0 时，x ≤ 0 → 强制 x = 0（不生产）。当 y = 1 时，x ≤ M → x 可以在 0 到产能上限之间自由取值。这样，通过一个 0-1 变量和 Big-M 约束，我们就精确刻画了"固定费用"行为。

常用逻辑约束的 MILP 建模

业务逻辑 数学表达式 MILP 建模
至少选一个（OR） x₁ + x₂ ≥ 1 x₁ + x₂ ≥ 1
最多选一个 x₁ + x₂ ≤ 1 x₁ + x₂ ≤ 1
只能选一个（XOR） x₁ + x₂ = 1 x₁ + x₂ = 1
若 A 则 B（if-then） z = 1 → x ≤ 10 x ≤ 10 + M·(1 - z)
若 A 则非 B z = 1 → x ≥ 5 x ≥ 5 - M·(1 - z)
二者择一约束 要么 f(x) ≤ 0 要么 g(x) ≤ 0 f(x) ≤ M·y, g(x) ≤ M·(1 - y)
有固定成本的连续活动 x > 0 ⇒ y = 1 x ≤ M·y, x ≥ 0

看到上面的第二行了吗？"最多选一个"这个约束在指派问题中非常常见——比如每台机器最多分配给一个工件。第四行的"if-then"几乎能表达任何条件逻辑，是 MILP 建模的瑞士军刀。

Big-M 法的选择原则

Big-M 的取值非常有讲究。M 太小会错误地切掉可行解（使本应有的选择变为不可能），太大则会导致数值不稳定——求解器在浮点运算中难以区分 M 和 M+1，导致收敛缓慢甚至错误。

选择 Big-M 的最佳实践：取尽可能小但足够大的值。比如，如果 x 的上界是 1000，就让 M = 1000，而不是 M = 10⁶。

Big-M 法建模示例：固定费用问题

假设：生产产品 A 需要一次性调试费 2000 元，
      单位利润 30 元，最大产能 500 件。
      是否生产？生产多少？

决策变量：
  x = 生产数量（整数或连续）
  y = 是否生产（0-1）

目标：max  30x - 2000y

约束：x ≤ 500·y         ← Big-M = 500
      x ≥ 0
      y ∈ {0, 1}

分析：
  - 若 y=0：x ≤ 0 → 不生产，利润=0
  - 若 y=1：x ≤ 500，固定成本 2000 被计入
  - 盈亏平衡点：30x ≥ 2000 → x ≥ 66.7
  - 所以只有当 x ≥ 67 时生产才划算

注意：如果 M 取 10⁷，求解器在处理这个约束时需要
在非常宽的范围内做 branching，效率将大幅下降。

Big-M 踩坑故事：某次我给一家钢铁企业做排程优化，模型中用到了 Big-M。当时图省事，对几个产能上限不明确的地方都设了 M = 999999。结果求解器跑了 2 小时都不收敛。排查发现，有两个 Big-M 约束之间的交互导致 LP 松弛的 gap 极大，分支定界树爆炸到了上百万个节点。把 M 从 999999 改为精确的上界 5000 之后，求解时间从 2 小时+ 降到了 11 秒。那次之后，我在任何 Big-M 建模中都坚持"能算多精确就算多精确"原则。

经典整数规划模型

背包问题（Knapsack Problem）

经典的"装包决策"：有 n 件物品，每件有重量 wᵢ 和价值 vᵢ，背包容量为 W。如何选择物品使总价值最大且不超过容量？

决策变量：xᵢ ∈ {0, 1}，1 表示选择物品 i
目标：max  Σ vᵢ·xᵢ
约束：Σ wᵢ·xᵢ ≤ W

看似简单，但它是 NP-Hard 问题。一个物品的取舍就能让求解时间天差地别。

集合覆盖问题（Set Covering Problem）

假设你需要覆盖城市里的所有消防需求点。有若干个候选消防站位置，每个站能覆盖一定范围内的需求点。最少建几个站？

变量：yⱼ ∈ {0, 1} — 是否在第 j 个候选位置建站
目标：min  Σ yⱼ
约束：Σ aᵢⱼ·yⱼ ≥ 1  — 每个需求点 i 至少被一个站覆盖
      其中 aᵢⱼ=1 表示位置 j 能覆盖需求点 i

集合覆盖的变体在调度中也大量出现——比如"需要多少辆应急维修车才能保证 30 分钟内到达任何故障点"。

旅行商问题（TSP）

这可能是最著名的组合优化问题：一个销售员要访问 n 个城市，每个城市去一次并返回起点，求最短的环游路线。TSP 的 MILP 建模用到了 uᵢ 子回路消除约束（Miller-Tucker-Zemlin 约束）。

变量：
  xᵢⱼ ∈ {0, 1} — 是否从城市 i 直接到城市 j
  uᵢ ≥ 0       — 城市 i 在路径中的序号（辅助变量）

目标：min  Σ cᵢⱼ·xᵢⱼ

约束：
  Σⱼ xᵢⱼ = 1    ∀i（每个城市出发一次）
  Σᵢ xᵢⱼ = 1    ∀j（每个城市到达一次）
  uᵢ - uⱼ + n·xᵢⱼ ≤ n-1  ∀i,j（MTZ 子回路消除）
  xᵢⱼ ∈ {0, 1}, uᵢ ≥ 0

没有中間的 MTZ 约束，解可能就是多个独立的小环——你去了上海飞北京、北京飞上海，成都飞重庆、重庆飞成都，但实际根本没有连接南北。MTZ 约束精确地防止了这种"子回路"。

线性松弛与整数 gap

为什么整数规划比线性规划难那么多？关键在于线性松弛（Linear Relaxation）——把整数约束去掉，把 IP 变成 LP。这个 LP 的解给出了 IP 最优值的下界（最大化问题中），但两者之间可能存在一个巨大的差距——这就是整数 gap（Integrality Gap）。

考虑一个极端的例子：

max  x₂
s.t. -x₁ + x₂ ≤ 0
      x₁ + x₂ ≤ 2
      x₁, x₂ ∈ {0, 1}

线性松弛（去掉整数约束）：
  s.t. 同上，但 x₁, x₂ ∈ [0, 1]
  LP 最优：x₁ = 1, x₂ = 1, Z = 1.0

整数最优：x₁ = 1, x₂ = 1, Z = 1.0  → gap = 0

但如果修改目标为 max x₁ + x₂：
  LP 最优：x₁ = 1, x₂ = 1, Z = 2.0
  整数最优：x₁ = 1, x₂ = 1, Z = 2.0  → 仍然是 0

更 tricky 的例子：
max  x₁
       -x₁ + x₂ ≤ 0
       x₁ + x₂ ≤ 1
       x₁, x₂ ≥ 0, 整数

LP 松弛最优：x₁=0.5, x₂=0.5, Z=0.5
整数最优：x₁=0, x₂=0, Z=0
gap = 0.5 / 0 = ∞  !!

整数 gap 越大，说明整数约束的"威力"越大——也就是说，连续的松弛解与离散的真实解差异巨大。MILP 难就难在：你没法简单地四舍五入。把 x₁ = 0.5 舍入到 1 可能违反约束，舍入到 0 可能目标值很差。

求解 MILP 的核心策略：分支定界（Branch and Bound）。它的思想是：先求解 LP 松弛，如果解恰好是整数，完美结束；如果某个变量应该是整数但不是，就"分支"——强制它 ≤ floor(值) 或 ≥ ceil(值)，递归求解两个子问题。通过比较上界和下界来剪枝。当代求解器还加入了切割平面（Cutting Planes）来收紧松弛，大幅加速求解过程。

整数规划是调度建模的核心工具。几乎所有调度问题的 MILP 模型都包含 0-1 变量——"工件 i 是否在工件 j 之前加工"、"任务 k 是否分配给机器 m"。掌握这些建模技巧，你就能把绝大多数工业调度问题转化为求解器可识别的语言。
