第22章 图神经网络入门

同学们好！上一章我们学习了网络嵌入，它通过随机游走为每个节点学习一个固定向量。但DeepWalk和Node2Vec有一个本质局限：它们只是"查表式"的嵌入——每个节点有自己独立的向量，且训练时看不到节点的属性特征。如果有一个新节点加入网络，你必须重新训练整个模型。图神经网络（Graph Neural Network, GNN）的出现彻底改变了这一局面，它让网络上的表示学习真正进入了"深度学习时代"。

为什么需要图神经网络

回顾上一章的浅层嵌入方法，它们虽然有效，但存在三个核心痛点。

痛点一：无法泛化到未见节点（Transductive vs Inductive）。DeepWalk 和 Node2Vec 是直推式的（transductive）：它们在训练时"看到"了所有节点，并且为每个节点分配了一个嵌入向量。如果有一个新节点加入网络，你需要重新运行随机游走并重新训练整个模型。这在大规模动态网络中是一个严重的瓶颈——想想看，一个社交网络每天有数百万新用户注册，每次都重新训练是不现实的。

痛点二：忽略了节点属性。在很多实际场景中，节点本身携带丰富的信息：论文有摘要和关键词、用户有个人资料和发布的内容、蛋白质有序列和结构特征。浅层嵌入方法完全忽略了这些属性信息——它们只考虑图的拓扑结构。而 GNN 可以天然地融合拓扑信息和节点属性。

痛点三：缺乏层次化的特征学习。在图像处理中，CNN通过堆叠卷积层来自动学习从边缘到纹理再到物体的层次化特征。浅层网络嵌入没有这种层次化结构——它们只是一次性的"查表"映射。GNN 通过多层消息传递，可以逐层聚合越来越远距离的信息，形成层次化的表示。

核心洞察：如果要把浅层嵌入和GNN做一个类比的话，浅层嵌入就像是"词袋模型"（每个词有独立向量但看不到上下文），而GNN就像是"BERT"（每个词的表示由其上下文动态决定）。GNN的突破在于：节点的表示不是独立的，而是通过图结构从邻居节点"聚合"而来的。

消息传递框架

绝大多数 GNN 模型都可以统一到一个框架下——消息传递（Message Passing）。这个框架的核心思想非常直观：每个节点通过与其邻居交换信息来更新自己的表示。

对于一个节点 v，第 k 层消息传递可以分为两步：

  1. 聚合（Aggregate）：将邻居节点在第 k-1 层的表示收集起来，通过某种聚合函数（求和、平均、最大值等）合并成一个"消息"。
  2. 更新（Combine/Update）：将节点自身的表示与聚合得到的消息结合起来，通过一个可学习的函数（通常是全连接层+激活函数）得到第 k 层的表示。

数学上可以写成：

av(k) = AGGREGATE(k)( { hu(k-1) : u in N(v) } )

hv(k) = COMBINE(k)( hv(k-1), av(k) )

其中 hv(k) 是节点 v 在第 k 层的表示，N(v) 是 v 的邻居集合。

置换不变性（Permutation Invariance）

聚合函数有一个重要的数学要求：它必须对节点邻居的顺序不敏感。因为图本身没有"顺序"的概念——你列举邻居的时候，无论按什么顺序，聚合结果都应该一样。满足这个性质的函数包括 sum、mean、max 等，但不包括 concat 或按位置的加权和。这是 GNN 与 CNN/RRN 的一个根本区别——CNN 依赖空间顺序，GNN 则必须对无序数据操作。

GCN——图卷积网络

GCN（Graph Convolutional Network）是图神经网络领域最具影响力的模型之一，由Thomas Kipf和Max Welling在2017年提出。它可以说是"每个学GNN的人都要读的第一篇论文"。

谱域 vs 空域

在进入 GCN 的细节之前，需要简单了解两种不同的设计思路：

谱域方法（Spectral）从图信号处理的角度出发，利用图的拉普拉斯矩阵的特征分解来定义卷积操作。这种方法的优点是数学优美，但缺点是：(1) 需要计算拉普拉斯矩阵的特征分解，计算复杂度为 O(N³)；(2) 学到的卷积核是图结构相关的，无法在不同结构的图之间迁移。

空域方法（Spatial）则是直接在图的邻居关系上定义聚合操作——这正是前面提到的消息传递框架。GCN 本质上是一种空域方法（尽管最初的推导是从谱域切入的），它的每一层只是对邻居节点的特征进行加权求和和变换。

Kipf & Welling GCN

Kipf和Welling提出的GCN的每一层可以表示为：

H(l+1) = sigma( D-1/2 A' D-1/2 H(l) W(l) )

其中：

  • A' = A + I 是添加了自环的邻接矩阵（让节点在聚合时包含自身的信息）
  • D 是 A' 的度矩阵
  • D-1/2 A' D-1/2 是归一化的邻接矩阵（防止高阶节点的表示值过大）
  • H(l) 是第 l 层的所有节点的特征矩阵
  • W(l) 是第 l 层的可学习权重矩阵
  • sigma 是激活函数（常用ReLU）

这个公式看起来有点吓人，但它本质上是做了三件简单的事情：求和邻居特征、归一化、过线性层。

# GCN层的简化实现（PyTorch风格）
import torch
import torch.nn as nn
import torch.nn.functional as F

class GCNLayer(nn.Module):
    def __init__(self, in_features, out_features):
        super().__init__()
        self.W = nn.Linear(in_features, out_features)

    def forward(self, A_hat, X):
        # A_hat: 归一化邻接矩阵 (N x N)
        # X: 节点特征矩阵 (N x in_features)
        # 消息聚合: A_hat @ X
        agg = torch.mm(A_hat, X)  # 对每个节点，加权求和邻居特征
        # 线性变换 + 激活
        return F.relu(self.W(agg))

# 构建一个2层GCN
class GCN(nn.Module):
    def __init__(self, n_features, hidden_dim, n_classes):
        super().__init__()
        self.layer1 = GCNLayer(n_features, hidden_dim)
        self.layer2 = GCNLayer(hidden_dim, n_classes)

    def forward(self, A_hat, X):
        h = self.layer1(A_hat, X)
        h = self.layer2(A_hat, h)
        return F.log_softmax(h, dim=1)

Kipf 和 Welling 在论文中展示了：仅仅使用一个2层的GCN，在Cora、Citeseer和Pubmed三个引文网络数据集上，半监督节点分类的准确率就大幅超过了当时的所有方法。更令人印象深刻的是，这个模型只有几十行代码（他们在GitHub上放了一个非常简洁的TensorFlow实现），却带来了质的飞跃。

为什么GCN这么有效？一个直观的理解是：GCN的每一层都在做"平滑"操作——它将邻居的特征进行加权平均，使得相邻节点的表示趋于相似。多层GCN则允许信息从更远的节点传播过来。在引文网络中，这种现象非常合理：一篇论文的类别（如"深度学习"或"数据库"）通常可以通过它引用的论文的类别来推断。这正是GCN能够"传递"标签信息做出准确分类的原因。

GAT——图注意力网络

GCN的一个隐含假设是：所有邻居对中心节点的影响是相同的（聚合时按度归一化权重）。但这个假设在多数场景下不成立。在你的社交网络中，有些朋友的意见对你影响更大，另一些则不那么重要。GAT（Graph Attention Network，Velickovic等，2018年）让模型自己学习每个邻居的重要性。

注意力机制

GAT的核心创新是在聚合之前，先计算一个注意力系数 alpha_{uv}，表示节点 u 对节点 v 的重要性。具体计算方式如下：

  1. 将节点 u 和节点 v 的表示拼接起来：concat(h_u, h_v)
  2. 通过一个可学习的向量 a 做点积，得到一个标量分数
  3. 用 LeakyReLU 激活后，通过 softmax 在所有邻居上归一化

这种"自注意力"机制使得模型能够为不同的邻居分配不同的权重。例如，在引文网络中，一篇论文可能更关注同领域的引用，而给跨领域的引用较低的权重——注意力机制可以自动学习这种偏好，而不需要人为指定。

多头注意力（Multi-Head Attention）

为了增加模型的表达能力，GAT引入了多头注意力：同时计算 K 组独立的注意力系数和变换，然后将 K 个结果拼接（或平均）起来。这种做法在Transformer中已经被验证非常有效，它的效果类似于集成学习——每个注意力头可以关注不同的"关系维度"。

# GAT注意力机制的简化实现
class GATLayer(nn.Module):
    def __init__(self, in_features, out_features, dropout=0.6, alpha=0.2):
        super().__init__()
        self.W = nn.Linear(in_features, out_features)
        self.a = nn.Parameter(torch.zeros(1, 2 * out_features))
        self.dropout = nn.Dropout(dropout)
        self.leakyrelu = nn.LeakyReLU(alpha)

    def forward(self, h, adj):
        # h: 节点特征 (N x in_features)
        Wh = self.W(h)  # (N x out_features)

        # 计算所有节点对的注意力分数
        Wh1 = Wh @ self.a[:, :self.out_features].T  # (N x 1)
        Wh2 = Wh @ self.a[:, self.out_features:].T  # (N x 1)
        e = self.leakyrelu(Wh1 + Wh2.T)  # (N x N) 未归一化注意力

        # 掩码：只保留有边的节点对
        attention = torch.where(adj > 0, e, -1e10 * torch.ones_like(e))
        attention = F.softmax(attention, dim=1)
        attention = self.dropout(attention)

        # 加权聚合邻居特征
        h_prime = attention @ Wh
        return h_prime

GAT的一个实用优势是：注意力系数提供了可解释性。你可以可视化哪些邻居对中心节点的决策影响最大，这在某些需要解释的领域（如药物发现）中非常宝贵。

GraphSAGE——归纳式学习

GCN和GAT都是直推式的——它们在训练时假设测试时也会看到相同的节点集合。但真实世界的图是动态的——新用户加入、新论文发表、新商品上架。GraphSAGE（Hamilton等，2017年）专门解决了这个问题，它将GNN从直推式（transductive）推广到了归纳式（inductive）。

邻居采样（Neighborhood Sampling）

GraphSAGE的关键创新在于：它不是用所有的邻居来聚合消息，而是每层随机采样固定数量的邻居。这个设计有两大好处：

  • 计算可扩展性：即使在度非常高的节点上，每层的计算量也是固定的（比如固定采3个邻居），可以轻松扩展到数十亿节点的图。
  • 归纳能力：模型学习的是"如何聚合邻居的特征"，而不是"为特定节点学一个向量"。当新节点加入时，只需要获取它的邻居和特征，就可以通过训练好的聚合函数计算它的嵌入。

三种聚合器

GraphSAGE提出了三种不同的聚合函数，每一种对应不同的"邻居消息组合方式"：

聚合器 操作 特点 适用场景
Mean 对邻居向量求平均 对称、简单、高效 大多数场景的默认选择
LSTM 按随机顺序输入LSTM 表达能力更强，但不对称 邻居有序列属性的情况
Pooling 每个邻居过MLP后取最大 对称、鲁棒、能捕捉关键邻居 需要关注最显著邻居时
选型经验：根据我的实践经验，对于大多数中小规模图（节点数万以内），GCN和GAT通常优于GraphSAGE，因为它们利用了全部邻居信息。但对于大规模图（百万级以上），GraphSAGE的采样策略使其成为唯一可行的选择。另外，如果你需要处理不断变化的数据（如实时推荐），GraphSAGE的归纳能力是无可替代的。

PyTorch Geometric入门

了解了理论基础后，让我们来动手实践。目前最流行的GNN框架有两个：PyTorch Geometric（PyG）和DGL（Deep Graph Library）。接下来我们用 PyG 来构建一个简单的 GNN 模型。

安装与基本概念

PyTorch Geometric 的安装非常方便（假设你已经安装了 PyTorch）：

pip install torch_geometric

在 PyG 中，图数据用 Data 对象表示，核心属性包括：

  • x：节点特征矩阵 (N x F)
  • edge_index：边的索引 (2 x E)，COO格式
  • y：节点标签（监督学习时使用）
  • train_mask / val_mask / test_mask：划分训练集、验证集和测试集

实战：节点分类

下面我们用一个完整的例子来训练一个GNN模型，在Cora数据集上做节点分类：

import torch
import torch.nn.functional as F
from torch_geometric.nn import GCNConv
from torch_geometric.datasets import Planetoid

# 1. 加载Cora数据集
dataset = Planetoid(root='/tmp/Cora', name='Cora')
data = dataset[0]
print(f"图信息: {data}")
print(f"节点数: {data.num_nodes}, 边数: {data.num_edges}")
print(f"特征维度: {data.num_features}, 类别数: {dataset.num_classes}")

# 2. 定义2层GCN模型
class GCN(torch.nn.Module):
    def __init__(self, in_channels, hidden_channels, out_channels):
        super().__init__()
        self.conv1 = GCNConv(in_channels, hidden_channels)
        self.conv2 = GCNConv(hidden_channels, out_channels)

    def forward(self, x, edge_index):
        x = self.conv1(x, edge_index)
        x = F.relu(x)
        x = F.dropout(x, p=0.5, training=self.training)
        x = self.conv2(x, edge_index)
        return F.log_softmax(x, dim=1)

# 3. 训练
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
model = GCN(dataset.num_features, 16, dataset.num_classes).to(device)
data = data.to(device)
optimizer = torch.optim.Adam(model.parameters(), lr=0.01, weight_decay=5e-4)

model.train()
for epoch in range(200):
    optimizer.zero_grad()
    out = model(data.x, data.edge_index)
    loss = F.nll_loss(out[data.train_mask], data.y[data.train_mask])
    loss.backward()
    optimizer.step()

    # 每10个epoch输出验证准确率
    if epoch % 10 == 0:
        model.eval()
        pred = model(data.x, data.edge_index).argmax(dim=1)
        acc = (pred[data.val_mask] == data.y[data.val_mask]).float().mean()
        print(f'Epoch {epoch:3d}, Loss: {loss:.4f}, Val Acc: {acc:.4f}')
        model.train()

# 4. 测试
model.eval()
pred = model(data.x, data.edge_index).argmax(dim=1)
test_acc = (pred[data.test_mask] == data.y[data.test_mask]).float().mean()
print(f'测试集准确率: {test_acc:.4f}')

这个只有几十行代码的模型，在 Cora 数据集上通常能达到 80% 以上的测试准确率。你只需要替换不同的数据集和修改几行网络结构，就可以应用到各种实际问题上。

新手常见问题：如果你刚开始用PyG，有一个最常犯的错误——edge_index必须是LongTensor类型（即索引为整数）。如果你从其他格式导入数据时不小心把边表转成了浮点数，PyG会静默地"接受"但内部计算会出错。建议在构建Data对象后，用data.validate()检查一下数据格式是否合法。
