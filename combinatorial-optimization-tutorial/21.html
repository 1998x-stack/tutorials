开篇：美国队长和灭霸的"最优解"之争

大家好，我是你们的讲师。在开始这一章之前，我想先讲一个看似和算法无关的故事。

在《复仇者联盟3》中，灭霸消灭了宇宙一半的人口——他用了"最优解"：用一个响指，随机消灭一半。从他的目标函数来看，这个"解"是完美的。但从我们的角度看，这个解当然是完全不可接受的——因为我们有更多的约束条件（道德、人性、生命权），而灭霸的模型忽略了它们。

这个例子引出了我们这一章的核心思想：什么时候"接近最优"就是够好了？

在上一章中，我们看到次模最大化可以用贪心得到一个(1-1/e)的近似。这听起来像是一个"退而求其次"的方案。但事实上，对于绝大多数NP难的组合优化问题，连次优近似都是我们用精巧的理论设计出来的——而不是随便瞎猜就能得到的。近似算法的任务是：放弃"精确最优"的执念，转而追求有理论保证的"足够好"，且保证能在多项式时间内运行。

正式定义：对于一个最小化问题，算法A是 α-近似的，如果对任意实例，A(I) ≤ α · OPT(I)。其中 α ≥ 1 称为近似比（Approximation Ratio）。对于最大化问题，α ≤ 1，且 A(I) ≥ α · OPT(I)。我们总是希望α尽量接近1。

好，让我们正式开始学习近似算法的艺术。

近似算法设计范式

近似算法不是靠运气的——它有一些经典设计范式。掌握了这些范式，你就能在面对一个新问题时有章可循。

五大设计范式总览

范式 核心思想 经典案例 典型近似比
贪心 每步选当前最优，不回溯 集合覆盖、最大割 ln n、1/2
LP舍入 解LP松弛，把分数解舍入为整数解 顶点覆盖、背包 2、FPTAS
原始对偶 同时构造原问题和对偶问题的解 顶点覆盖、设施选址 2、1.861
局部搜索 从解出发，在邻域中迭代改进 最大割、k-中位数 1/2、3+ε
随机化舍入 用概率方法把分数解随机舍入为整数 最大可满足性(Max-SAT) 3/4

贪心我们已经在拟阵和次模优化中深入讨论过了。LP舍入和原始对偶依赖于第4-5章中学习的线性规划理论。局部搜索是元启发式（第25章）的基础。而随机化舍入——用概率来求解确定性问题——是近似算法中最具创意的一支。

直觉理解

这些范式其实都有一个人性化的类比：

  • 贪心 = 买房子时，先看最喜欢的，再看第二喜欢的……只进不退。
  • LP舍入 = 先算个概数（LP松弛），然后"四舍五入"成整数。
  • 原始对偶 = 买东西时同时看价格标签（原问题）和钱包余额（对偶问题），边看边调整。
  • 局部搜索 = 爬山——不断往高处走，直到四周都比自己低为止。
  • 随机化舍入 = 抛硬币做决策，但硬币的偏向由LP解决定。

现在让我们深入几个最经典的近似算法，看看这些范式是如何在实际问题中应用的。

经典近似算法

Christofides算法——TSP的1.5近似

旅行商问题（TSP）是组合优化中最著名的问题。它描述起来极其简单——找一条经过所有城市恰好一次的最短回路——但它的精确求解是NP难的。在一般的度量空间（满足三角不等式）中，Christofides（1976）给出了一个优雅的1.5近似算法。

Christofides算法（度量TSP的1.5近似）

输入：完全图 G = (V, E)，边权满足三角不等式
输出：哈密顿回路

1. 计算 G 的最小生成树 T
2. 令 O = V 中在 T 上度数为奇数的顶点集合
     （度数为奇数的顶点个数一定是偶数——证明：握手引理）
3. 在 O 上计算最小权完美匹配 M
4. 将 T 和 M 的边合并，得到欧拉图 H = (V, T ∪ M)
5. 在 H 中找欧拉回路
6. 在欧拉回路中跳过重复顶点（捷径），得到哈密顿回路

为什么这是1.5近似？

  • MST的权重 ≤ OPT（任何哈密顿回路去掉一条边就是生成树）
  • 最小完美匹配的权重 ≤ OPT / 2（奇数度集合上的最优TSP子回路经过O的顶点，可以分解为两个完美匹配）
  • 总权重 ≤ MST + 匹配 ≤ OPT + OPT/2 = 1.5 OPT
深度理解：这个算法让我着迷的地方在于：它用两个"亚结构"——生成树和完美匹配——组合出了一个全局解。生成树保证了"所有城市被连起来了"，而完美匹配修正了生成树中的"奇数度"问题。它既不是贪心也不是LP，而是一种独特的"组合拼装"思路。1985年至今，虽然很多人尝试过改进，TSP的近似比仍然停在这个1.5——这也说明这个算法有多精巧。

最小权顶点覆盖——2近似

顶点覆盖问题：找一组顶点，使得每条边至少有一个端点被选中。最小化选中的顶点数。

这个问题的2近似算法非常简单到令人难以置信：

近似顶点覆盖（2-近似）

C ← ∅
while 存在未被覆盖的边 (u, v):
    C ← C ∪ {u, v}
    从图中移除 u, v 以及所有与 u, v 关联的边
return C

这个算法每次选一条边，把它的两个端点都加入覆盖集。因为最优解对每条"被选的边"至少要选其中一个端点，因此|C| ≤ 2 · OPT。

有意思的是，这也可以看作一个极大匹配算法——每次选一条边加入匹配，然后把两个端点的所有边都删掉。这个匹配的每条边在最优解中至少需要一个顶点来覆盖，而我们"浪费"了另一个顶点，因此近似比是2。

一个真实的坑：我在2019年辅导一个团队做视频监控的摄像头布局优化时，他们天真地把这个2近似算法直接套用在了非标准的"顶点覆盖"上——他们的"顶点"是行道树路口，"边"是道路路段。结果发现近似比远大于2。为什么？因为他们的图是平面图（城市道路网络），而我们的近似比分析假设了一般图——对平面图，更好的近似是可能的（Baker技术），但他们用了最粗糙的通用算法。这个教训：近似比是"最坏情况"的分析，如果你的数据有特殊结构，你可以做得更好——也要做得更好。

集合覆盖——贪心的ln n近似

集合覆盖（Set Cover）：给定一个全集U和它的若干子集，选择最少的子集使得它们的并覆盖U。这是一个比顶点覆盖更一般的问题。

贪心算法：每次选能覆盖最多"未被覆盖元素"的子集。

贪心集合覆盖

U' ← U  # 待覆盖的元素
S ← ∅  # 已选子集
while U' ≠ ∅:
    选子集 C ∈ {子集族} 使得 |C ∩ U'| 最大
    S ← S ∪ {C}
    U' ← U' \ C
return S

近似比分析：这个贪心算法给出 H(d) ≈ ln n + 1 的近似比，其中d是最大子集的大小。而且这个界是紧的——存在实例使得贪心恰好达到ln n倍的OPT。

值得一提的是，这个ln n近似也是你能做到的最好结果——Feige在1998年证明了：除非P=NP，否则没有算法能给出 (1-ε)ln n 的近似。

近似方案

PTAS vs FPTAS

有些问题可以做到"任意接近最优"。如果一个算法族对于任意固定的 ε > 0，能在多项式时间内给出 (1+ε) 近似（对最小化问题），就称为多项式时间近似方案（PTAS, Polynomial-Time Approximation Scheme）。如果运行时间还是输入大小和 1/ε 的多项式，则称为完全多项式时间近似方案（FPTAS, Fully Polynomial-Time Approximation Scheme）。

方案 运行时间 例子
FPTAS poly(n, 1/ε) 背包问题
PTAS O(n^{f(1/ε)}) 欧几里得TSP
EPTAS O(f(1/ε) · poly(n)) 平面图上的问题
无PTAS 除非P=NP 集合覆盖、最大团

注意区别：FPTAS的运行时间对 1/ε 是多项式，而PTAS可以是任意函数——比如 n^{1/ε}。当 ε = 0.001 时，FPTAS还行（如 O(n²/ε) = 1000n²），但PTAS的 n^{1000} 就不可行。

背包问题的FPTAS

背包问题是"最简单"的NP难问题，也是FPTAS的教科书级范例。

核心思路是缩放+DP（Dynamic Programming by Profit Rounding）：

背包FPTAS（Ibarra-Kim, 1975）

输入：N个物品，重量 wᵢ，价值 vᵢ，容量 C，近似参数 ε
输出：(1-ε) 近似的最大价值

1. 令 V = maxᵢ vᵢ
2. 设置缩放因子 θ = ε·V / N
3. 对每个物品，定义缩放后的价值 v̂ᵢ = ⌊vᵢ / θ⌋
4. 用经典DP求解缩放后的问题：
     DP[i][p] = 达到总缩放价值p所需的最小重量
5. 找出满足 DP[N][p] ≤ C 的最大 p
6. 原问题的近似解价值 ≈ p·θ

这个算法的时间复杂度是 O(N²/ε)，绝对是一个FPTAS。其中的关键是缩放——我们把价值"压缩"了，使得DP的状态空间从 O(N·V) 缩小到 O(N·N/ε) = O(N²/ε)。

平面图上的Baker技术

对于平面图上的问题（如独立集、支配集），Baker在1994年提出了一个通用的PTAS设计技术：将平面图分解为k个"层"（layers），对每层进行最优求解，再通过"删除一个层"来打破环路，实现 (1+1/k) 的近似。

这个技术的巧妙之处在于：平面图的"平面性"使得图的树宽（Treewidth）可以被控制，从而在固定宽度的子图上可以在多项式时间内精确求解。

一个哲学问题：为什么有些问题有FPTAS（背包），有些只能有PTAS（平面图独立集），而有些连PTAS都没有（集合覆盖、TSP的一般形式）？答案藏在问题的"结构"里。背包问题是fptas的因为价值可以被"缩放"而不丢失太多信息；TSP没有PTAS因为即使"近似"的距离矩阵也需要精确保存几何信息。这个观察也暗示：在设计近似算法之前，先要理解你的问题属于哪一类——不同类型需要完全不同的工具。

不可近似性

不是所有问题都可以很好地近似。有些问题不但无法精确求解，甚至连"接近求解"都是NP难的。

PCP定理的直觉

PCP（Probabilistically Checkable Proof）定理是理论计算机科学中最深刻的成果之一。它说的是：NP问题的解可以用一种"概率可验证"的方式编码，验证者只需随机查看证明的极少几个比特，就能以高概率判断证明的正确性。

这个定理的推论之一是：很多优化问题的近似版本也是NP难的。换句话说：如果你能找到某个问题的好的近似算法，你就等于证明了P=NP。

直观理解PCP定理：想象你在批改3000份试卷，按照PCP定理，你只需要随机挑出3道题检查，只要这3道题的回答一致且正确，整份试卷就大概率是正确的。这个"局部检查≈全局正确"的不可思议的性质，是PCP定理的核心。而这个性质在近似算法中的镜像就是："局部近似≈全局近似"可能是错的——有些问题的解即使局部上与最优相差很小，全局上也可能差得很远。

Gap技术与不可近似性

Gap技术是证明不可近似性的标准方法。思路是：如果存在一个问题的α近似算法，那么就能用它来区分"最优值≤L"和"最优值≥α·L"这两种情况——这等价于解决一个NP难的判定问题。

利用这个技术，研究人员证明了：

  • 最大独立集（Maximum Independent Set）：没有 n^{1-ε} 近似算法（除非P=NP）。这是极强的不可近似性——近似比随着问题规模增大而劣化！
  • 最大团（Maximum Clique）：与最大独立集一样，同样有 n^{1-ε} 的不可近似性。
  • 集合覆盖：没有 (1-ε)ln n 近似算法（除非P=NP）。这就是为什么贪心算法的ln n近似已经是"最优"的了。

TSP的一般情形

值得一提的是，一般的TSP（不满足三角不等式）是无法近似的——对任意α ≥ 1，不存在α近似算法（除非P=NP）。原因很简单：你可以把哈密顿回路的存在性归约到TSP近似问题上。

这也解释了为什么Christofides算法要求"三角不等式"——那个1.5近似是对度量TSP而言的，不是对一般TSP的。

近似算法的现状

截至2020年代，近似算法领域已经有了一个相当完整的图景。这里给一个快速总结。

哪些问题有好的近似比

  • 顶点覆盖：2近似（组合），1.999...近似（用更强的方法，但进展非常缓慢）
  • 度量TSP：1.5近似（Christofides，1976），近年来有极小改进到1.5 - ε（2020）
  • Steiner树：1.39近似（Byrka等，2010），这个领域进展更为显著
  • 设施选址：1.488近似（Li，2011），组合优化中近似比"军备竞赛"的经典案例
  • 最大割：0.878近似（Goemans-Williamson, 1995），使用半定规划松弛

哪些问题没有

  • 最大独立集：几乎不可近似
  • 最大团：几乎不可近似
  • 一般TSP：完全不可近似
  • 图着色：不可近似因子为 n^{1-ε}

Unique Games Conjecture的影响

Unique Games Conjecture（UGC）是Khot在2005年提出的一个猜想。如果它成立，则许多优化问题的"最优近似比"将被精确确定：

  • 最大割：0.878正好就是最优（Goemans-Williamson的算法已经是最优的了）
  • 顶点覆盖：2正好就是最优
  • 满足程度最高的约束满足问题：近似比的精确界限由Unique Games的"几何"结构决定

UGC被广泛认为是近似算法理论的"大统一假设"。如果它为真，那么过去二十年的近似算法研究就不再是零散的"个案"，而是一个宏大图景的各块拼图。2020年，Khot因为这个猜想获得了Nesetril奖——即使它尚未被证明，它的深远影响已经被学界公认。

给非理论研究者的建议：如果你主要做工程应用，不需要理解PCP定理或UGC的细节。但你需要记住的是：不要指望所有问题都有好的近似算法。如果你的问题对应的是"近似困难的"问题（如最大独立集），就不要在近似算法上浪费时间——直接考虑启发式、分支定界、或问题结构的特殊利用。知难而退也是一种智慧。
