第19章 时序特征工程

大家好。在前面的章节中，我们学习了状态空间模型和卡尔曼滤波——它们属于经典的时序建模方法。但从这一章开始，我们要进入一个更广阔的天地：用机器学习做时序预测。

你可能会问：我学了那么多 ARIMA、GARCH，为什么还要用 ML？原因很简单——真实世界的时序数据往往充满非线性关系、复杂的交互作用和大量外生变量。传统的参数化模型在面对这些场景时很快就会捉襟见肘。而机器学习模型，尤其是树模型和神经网络，恰恰擅长处理这类问题。

但把时间序列塞进 ML 模型并不是直接扔进去就行。时序数据的结构是顺序依赖的，你不能像处理普通表格数据那样随便取一行就训练。这一章的核心就是时序特征工程——如何把一条时间序列安全地转换成 ML 模型能吃的"表格"。如果你这一步做错了，后面的模型再花哨也是白搭。

19.1 从时序到表格：核心思路

机器学习模型（XGBoost、Random Forest、神经网络等）通常期待输入是一个矩阵，每一行是一个样本，每一列是一个特征。但时间序列是一条先后有序的序列。怎么办呢？答案是滑窗变换：用过去一段窗口内的数据构造特征，预测未来的目标值。

假设我们有一个每日销量序列 [y1, y2, ..., yt]。我们可以构造这样一个表格：

行特征: y(t-2)特征: y(t-1)目标: y(t)
1y1y2y3
2y2y3y4
3y3y4y5

每一行对应一个时间点，特征是用历史值构造的滞后特征。这样就把时序问题转化成了监督学习问题。

19.2 滞后特征的选择

最基本的特征就是滞后值（lag features）：lag_1, lag_2, ..., lag_k。但选择多少阶滞后不是拍脑袋决定的，我们有一些工具辅助判断。

偏自相关函数是你在第 4 章学过的老朋友。PACF 图中，显著超出置信区间的滞后阶数就是潜在的候选特征。此外，领域知识同样重要。比如预测电商销量，过去 7 天的滞后几乎总是比过去 6 天更有用——因为周周期性。

一个实用的建议：先选一个较大的滞后集合（比如过去 30 天），然后用树模型的特征重要性或 Lasso 做特征选择，让数据自己决定哪些滞后有用。不要手动筛选太多，模型会比你想的更善于处理冗余特征。

小贴士：在构造滞后特征时，确保训练集和验证集之间没有数据交叉。也就是说，验证集中第 t 行的特征只能来自 t 时刻之前的数据。这一点看似简单，但在实践中非常容易出错。

19.3 滚动窗口统计量

仅用原始滞后值还不够。真实时序中，局部趋势和波动模式往往比精确的过去值更有信息量。滚动窗口统计量可以捕捉这些模式。

常用的滚动特征包括：

  • 滚动均值（rolling mean）：过去 N 期平均值，反映局部趋势
  • 滚动标准差：过去 N 期波动率，反映局部不确定性
  • 滚动最小值/最大值：过去 N 期的区间边界
  • 滚动分位数：如滚动中位数、25%/75% 分位数，对异常值更稳健
  • 滚动斜率：过去 N 期线性回归的斜率，反映局部变化速度
import pandas as pd
import numpy as np

def create_rolling_features(series, window=7):
    """为时序数据生成滚动窗口特征"""
    df = pd.DataFrame({'y': series})
    df['rolling_mean'] = df['y'].rolling(window).mean()
    df['rolling_std'] = df['y'].rolling(window).std()
    df['rolling_min'] = df['y'].rolling(window).min()
    df['rolling_max'] = df['y'].rolling(window).max()
    df['rolling_median'] = df['y'].rolling(window).median()
    df['rolling_range'] = df['rolling_max'] - df['rolling_min']
    return df
踩坑警告：数据泄露！这是时序特征工程中最常见、也最隐蔽的错误。假设你想用过去 7 天的均值预测第 t 天的值。如果你直接对整个序列调用 rolling(7).mean()，然后和原始数据并在一起划分训练集和测试集——你已经在用"未来"信息了。正确的做法是：在训练集内，每个时间点的滚动特征只能包含该点之前的窗口数据。pandas 的 shift() 是你的朋友：series.rolling(7).mean().shift(1) 才能保证不泄露当前时间点的信息。

我曾经见过一个团队在 Kaggle 竞赛中提交了本地验证 0.98 R² 的模型，结果公开榜只有 0.2。排查了一整天，发现是他们在构造滚动均值时忘了 shift(1)——每个样本都"看到"了当天的真实值。这个教训告诉我们，时序特征工程中，数据泄露的代价是毁灭性的，而且往往要到最后阶段才会暴露。

19.4 日历特征

时间本身携带了大量信息。日历特征能帮助模型捕捉人类社会的周期性行为：

  • 小时 / 星期几 / 月份：最基础的周期性编码
  • 是否周末 / 是否节假日：对于零售、交通等领域至关重要
  • 一年中的第几天 / 第几周：捕捉季节性变化
  • 月初 / 月末 / 季末：财务报表、工资发放等行为模式
  • 特殊事件标记：双十一、黑色星期五、疫情封控等一次性事件

对于节假日特征，你可以使用各语言的开放节假日库（Python 的 holidays 包或 workalendar），也可以手动标记。更高级的做法是为每个节假日添加一个"节前 N 天"和"节后 N 天"的窗口特征——很多消费行为在节前达到峰值，节后迅速回落。

关键洞察：日历特征对树模型特别有效。树模型无法"理解"时间连续性，但如果你把月份编码为 1-12，树模型可以学到"6-8 月销量高"这样的规则。但需要注意：月份和星期几这类循环特征（12 月之后是 1 月，周日之后是周一），最好同时保留数值编码和正弦/余弦编码，让模型有机会学到循环模式。

19.5 傅里叶项编码季节性

当我们拥有较长时间序列时，用离散的月份编码可能无法捕捉复杂的多周期模式。这时候 傅里叶项就派上了用场。

傅里叶特征通过正弦和余弦函数对不同频率的周期进行编码。对于一个长度为 T 的序列，你可以添加如下特征：

import numpy as np

def add_fourier_features(df, period, max_k):
    """
    为 DataFrame 添加傅里叶特征
    period: 周期长度（如每周 7 天，每年 365 天）
    max_k: 使用的正弦/余弦对数量
    """
    t = np.arange(len(df))
    for k in range(1, max_k + 1):
        df[f'sin_{period}_{k}'] = np.sin(2 * np.pi * k * t / period)
        df[f'cos_{period}_{k}'] = np.cos(2 * np.pi * k * t / period)
    return df

实际使用时，你可以同时为多个周期添加傅里叶项。例如对日数据：周期为 7（周周期）用 3 对正余弦，周期为 365.25（年周期）用 5-10 对。这些特征能够让线性模型也具备捕捉复杂周期性模式的能力。许多成熟的预测系统（包括 Prophet，我们将在第 21 章学到）内部就是靠傅里叶项来处理季节性的。

19.6 缺失值处理

时间序列的缺失值比普通表格数据更棘手，因为你不能简单删除一行——那会打乱时间顺序。

常见的处理方法包括：

方法适用场景注意事项
前向填充（ffill）缺失间隔较短，值变化缓慢会引入滞后偏差
线性插值趋势较平滑的场景在跳跃点效果差
季节性插值有强季节性的数据用前后周期的同位置值填充
模型预测填充缺失段较长计算成本高，有引入模型偏差的风险
直接删除缺失比例极小（<1%）仅适用于非连续的短缺失

我的经验是：对大多数场景，前向填充加一个"是否被填充"的指示特征是一个简单而高效的 baseline。这个标记列告诉模型"这个值是真实观测还是填充的"，让模型自己决定是否信任它。

19.7 异常值检测

时序异常值可能是传感器故障、数据录入错误，也可能是一个真实的事件信号（比如某天销量暴增是因为促销）。在特征工程阶段，我们至少应该识别它们，而不是盲目处理。

两种常用的检测方法：

Z-score 方法：计算每个点与均值的差值（以标准差为单位）。通常 |Z| > 3 被认为是异常。但这个方法假设数据接近正态分布，且对极值敏感（因为均值和标准差本身会被异常值污染）。

IQR 方法：用中位数和四分位距，更加稳健。计算 Q1 - 1.5*IQR 和 Q3 + 1.5*IQR 之外的值为异常。但 IQR 方法不考虑时序的局部性——序列整体的正常值可能在局部看是异常。

更推荐的时序异常检测是滚动窗口版本：用过去 N 个点的均值和标准差计算局部 Z-score，这样能感知局部波动。这也是为什么前面我们强调构造滚动标准差特征——它不仅对预测本身有用，还能帮你做异常检测。

def detect_anomalies(series, window=30, threshold=3):
    """基于滚动窗口的 Z-score 异常检测"""
    rolling_mean = series.rolling(window).mean()
    rolling_std = series.rolling(window).std()
    z_scores = (series - rolling_mean) / rolling_std
    return z_scores.abs() > threshold

19.8 特征工程检查清单

结束之前，给大家一份实用清单。每次做时序 ML 项目时，可以逐项检查：

  • 所有滚动特征都加了 shift(1) 吗？
  • 训练集和验证集在时间上没有交叉吗？
  • 滞后特征的阶数有理论或统计依据吗？
  • 日历特征覆盖了所有已知的周期性模式吗？
  • 缺失值处理方式是合理的还是"偷懒"的？
  • 异常值在预测目标中被正确处理了吗？（而不是简单删除）
