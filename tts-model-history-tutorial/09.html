第9章 Tacotron系列：端到端声学模型

大家好，我是你们的讲师。上一章我们讲了WaveNet，它可以直接生成原始波形，但需要以声学特征作为条件。换句话说，WaveNet自己不会"读文本"——它需要别人先把文本转成声学特征。那谁来干这个活呢？Tacotron系列模型就是答案。

如果把TTS系统比作一个人，那Tacotron就是负责"理解文本并决定怎么发音"的大脑，而WaveNet是负责把想法变成声音的嘴巴。两者结合，才构成了完整的TTS系统。

2017年3月，Google发表了Tacotron论文，三个月后的12月又发布了Tacotron 2。这一系列工作不仅在技术上实现了巨大的飞跃，更重要的是，它让TTS第一次"听起来像真人了"。

"读出来"：从字符到频谱的端到端学习

Tacotron的核心哲学是端到端——输入字符（或者音素），直接输出梅尔频谱。不需要人工设计的语言学特征，不需要复杂的对齐规则，一切都在神经网络中自动学习。

这个想法在当时是非常激进的。之前的声学模型（HMM、DNN参数合成）都需要一个独立的时长模型来预测每个音素持续多久，再通过一个复杂的上下文处理流程来生成声学参数。前端文本分析更是需要大量语言学知识：分词、词性标注、韵律边界预测、重音预测……每个环节都可能引入错误。

Tacotron用一套Encoder-Attention-Decoder架构端到端解决了这个问题——我们从第1章回顾这个经典流程：

# Tacotron v1 的整体流程

# 1. Encoder：将字符序列映射到隐藏表示
text_chars = "你好世界"
encoder_output = encoder(text_chars)  # [T, hidden_dim]

# 2. Attention：在每个解码步，从编码器输出中"关注"相关信息
#    content-based attention + location-sensitive attention
for t in range(max_mel_steps):
    context = attention(decoder_state, encoder_output)
    mel_frame = decoder_step(context, prev_frame)

# 3. CBHG + PostNet：细化频谱
mel_spectrogram = cbhg(mel_frames)
mel_spectrogram = postnet(mel_spectrogram) + mel_spectrogram

# 4. 用 Griffin-Lim 从 Mel 重建波形
waveform = griffin_lim(mel_spectrogram)

Tacotron v1：初露锋芒

Tacotron v1（2017年3月）提出了一套优雅的架构：

  • Encoder：输入字符经过一个嵌入层（Character Embedding）后，通过一个CBHG模块（由1D卷积+Highway网络+双向GRU组成），提取文本的上下文表示。
  • Attention：基于内容的注意力（Content-based Attention），在每个解码步决定"当前应该关注输入文本的哪个位置"。
  • Decoder：一个自回归的GRU，每次预测多个梅尔帧（Reduction Factor，比如 r=2或3），加速推理并增强稳定性。
  • PostNet & CBHG：对解码器输出的粗糙频谱进行细化和平滑。
  • Vocoder：Griffin-Lim声码器——一种无需训练的相位重建算法，质量有限但胜在简单。

Tacotron v1取得了MOS 3.82的成绩。在当时，这已经相当不错了——毕竟它直接"读"字符就生成了频谱，不需要任何手工特征。但Griffin-Lim声码器是明显的瓶颈：它重建的波形带有明显的"嘶嘶声"和"金属感"。

有趣的细节：Tacotron 采用了 "Reduction Factor"（缩减因子）的设计——解码器每次前向传播生成 r 帧梅尔频谱（而不是1帧）。直觉上这应该会让质量变差（减少了生成的时间分辨率），但实际上它让注意力更容易学习，反而提升了稳定性。这个"看似会降低质量、实则提升质量"的设计，在深度学习研究中经常出现——永远不要凭直觉判断，要动手实验。

Location-Sensitive Attention

Tacotron v1有一个严重问题：注意力会"走神"。具体来说，模型在生成长句时可能跳过某些词（attention没有覆盖到），也可能重复生成某个词（attention在同一位置来回徘徊）。

这个问题在Tacotron 2中得到了针对性改进。Location-Sensitive Attention（位置敏感注意力）在注意力分数的计算中引入了一个额外的累加权重项：

# 位置敏感注意力（Location-Sensitive Attention）
# 相比普通注意力多了"位置累积"特征

def location_sensitive_attention(query, keys, prev_attention, cum_attention):
    """
    query: 解码器当前状态
    keys: 编码器输出 [T, dim]
    prev_attention: 上一步的注意力权重
    cum_attention: 到目前为止的累积注意力权重
    """
    # 将前一步注意力和累积注意力做卷积，提取位置特征
    location_feat = conv1d(prev_attention) + conv1d(cum_attention)

    # 能量 = 内容匹配 + 位置匹配
    energy = score_content(query, keys) + score_location(location_feat, keys)

    # softmax归一化
    attention_weights = softmax(energy)

    # 更新累积权重
    cum_attention = cum_attention + attention_weights

    return attention_weights, cum_attention

这个设计的作用是：当注意力权重在一个位置累积到一定程度后，location特征会"推动"注意力向前移动，防止原地打转。同时，累积权重也保证了注意力最终会覆盖整个输入序列，减少漏词的风险。

Attention机制类比：可以把注意力想象成一个人在朗读文本时的手指——手指从左向右移动，每读到一处就停留一会。普通注意力相当于"凭记忆看下面该读哪"，有时会退回读过的词或跳行；而位置敏感注意力相当于"手指一直在纸上从左向右移动"，每读完一个词就朝前移一点，很少出错。

Tacotron 2：全面进化

2017年12月，Google发布了Tacotron 2。它不是一个全新的架构，而是对Tacotron v1的全面优化，加上最重要的改进——用WaveNet声码器替代Griffin-Lim。

Tacotron 2的主要变化：

模块 Tacotron v1 Tacotron 2
Encoder CBHG（卷积+Highway+GRU） 3层Conv1D + 双向LSTM
Attention Content-based Attention Location-Sensitive Attention
Decoder GRU + Reduction Factor 双层LSTM，Reduction Factor=1（逐帧生成）
PostNet CBHG模块 5层Conv1D残差网络
Stop Token 无（固定帧数） 线性层+logistic sigmoid，预测"是否结束"
Vocoder Griffin-Lim（无训练） WaveNet（有条件）
MOS评分 3.82 4.53

除了用LSTM替换GRU、取消Reduction Factor等细节优化，Tacotron 2最重要的两个设计是：

1. 停止符预测（Stop Token Prediction）

在Tacotron v1中，解码器固定生成N帧梅尔频谱——如果N太小，句子没说完就被截断；N太大，解码器在结束后胡乱生成无意义内容。Tacotron 2引入了一个简单的停止符分类器：在每帧输出时，预测"这个位置是否应该结束"。当停止符的概率超过0.5时，解码停止。这让Tacotron 2能够自适应地处理任意长度的句子。

2. WaveNet声码器替代Griffin-Lim

这是质变的关键。Griffin-Lim虽然不需要训练，但重建质量差。WaveNet以Tacotron 2输出的梅尔频谱为条件，生成原始波形。两个各自很强的模型组合在一起，产生了1+1>2的效果。

MOS 4.53的震撼

Tacotron 2的MOS评分4.53，是人类录音4.58的99%——差距仅0.05分。这是在TTS史上前所未有的里程碑。

历史时刻：据Google研究员回忆，在Tacotron 2的盲听测试中，有几位听评者打分时完全分不清哪些是机器合成、哪些是真人录音。其中一位听评者甚至留言说："你们的录音质量很好，但请如实告诉我，这里面到底有没有机器合成的样本？"——当得知其中有大约一半是机器合成的时，他非常惊讶。这是TTS历史上第一次发生"研究者自己也分不清"的情况。

Tacotron 2还做了消融实验来验证每个模块的贡献。结果很有意思：把WaveNet换成Griffin-Lim，MOS从4.53降到4.08。去掉Location-Sensitive Attention，长句合成出现重复/漏词。去掉PostNet，频谱细节变差。每个组件都有不可替代的作用。

Tacotron 2的工程局限

虽然Tacotron 2在听感上接近完美，但它有严重的工程问题：

  • 推理速度慢：自回归解码每帧都要做Attention+Decoder+PostNet，接着WaveNet又要逐采样点生成。整个系统的RTF在0.2-0.5左右——生成1秒音频需要0.2-0.5秒，勉强可用于实时场景，但远谈不上"轻快"。
  • 注意力偶尔崩溃：位置敏感注意力虽然在大多数情况下工作良好，但在处理超长句（>100个字符）或罕见的标点组合时，仍然会出现注意力发散的问题。
  • 不擅长长句：实验表明，句子越长，Tacotron 2的合成质量下降越快。超过15个词的句子，明显出现韵律平淡、语速不均匀等问题。
  • 缺乏可控性：你无法控制语速、无法调节情感、无法指定重音——模型学会了什么风格就只能生成什么风格。这在工业应用中是一个大问题。
踩坑实录：在一个中文TTS项目中，我们用Tacotron 2训练了一个中文女声音色。短句（<10个字）效果非常好，几乎以假乱真。但一到长句场景（比如朗读整段新闻），问题就暴露了：20字以上的句子经常出现"吞字"现象——明明输入是"今天的最高气温达到了38摄氏度"，合成出来是"今天的达到了38摄氏度"。更麻烦的是，这种错误是随机的——同样的句子合成两次，可能一次对一次错。这在产品中是不可接受的，因为用户遇到一次"吞字"就会对整个系统失去信任。后来我们不得不加入规则后处理，用拼接合成的结果来"兜底"。这个经历让我深刻感受到：学术界衡量模型用的是平均分，但工业界关心的是最差情况下的稳定性。

但无论如何，Tacotron 2的历史地位是不可动摇的。它第一次证明了神经网络TTS可以在听感上追平真人录音，激发了整个行业对端到端TTS的巨大热情。在接下来的几年里，Tacotron 2成为了几乎所有TTS研究的"基线模型"——任何新方法都要先和Tacotron 2对比。

然而，Tacotron 2的"自回归"特性带来的效率问题，很快就催生了一个全新的研究方向——非自回归并行合成。但在那之前，我们先看看声码器领域发生了什么——因为WaveNet虽然效果好，但实在太慢了。下一章，我们会看到一群"追求速度"的研究者是如何把声码器的速度提升了成千上万倍、同时还能保持接近WaveNet的质量。
