第17章 卷积码与Viterbi译码

大家好，我是你们的讲师。1967年，一位叫Andrew Viterbi的年轻教授正在加州大学洛杉矶分校（UCLA）教书。他研究的是一个看起来非常具体的问题：给定一个有噪声的信道和一种特殊的编码，怎么找到最可能被发送的比特序列？

他提出的解法后来被称为Viterbi算法——一种基于动态规划的序列译码方法。这个算法不仅成为数字通信的基石，还深刻地影响了其他领域：语音识别中的"HMM解码"、生物信息学中的"序列比对"、甚至自然语言处理中的"序列标注"——背后都跑着Viterbi算法。

而Viterbi本人后来与朋友共同创立了一家叫Qualcomm（高通）的公司。没错，你手机里的基带芯片，其核心技术之一就是Viterbi算法。

要理解Viterbi算法，我们首先要理解卷积码（convolutional code）——这是一种不把数据分成独立块、而是像流水一样连续处理的编码方式。

卷积码的编码器结构

与分组码不同，卷积码的编码器是一个有限状态机：由一组移位寄存器和模2加法器（异或门）构成。每输入1比特（或k比特），就输出n比特——输出不仅依赖当前输入，还依赖移位寄存器中保存的历史状态。

几个关键参数：

  • 约束长度K：编码器存储的状态位数（加上当前输入）。K越大，码的记忆越长，纠错能力越强。
  • 码率R = k/n：每输入k比特，输出n比特。
  • 生成多项式：每个输出比特与寄存器连接的抽头模式。

来看一个最经典的例子：码率1/2、约束长度K=3的卷积编码器。

# 卷积码编码器示例：码率 1/2，K=3
# 生成多项式: g0 = 7 (111, 八进制), g1 = 5 (101, 八进制)

class ConvEncoder:
    def __init__(self):
        self.state = 0  # 2位寄存器，初始为0

    def encode_bit(self, bit):
        """输入1比特，输出2比特"""
        # 当前状态 + 新输入的比特确定输出
        # state的位: [s1, s0]，s1是最旧的位

        # 输出0: s1 ^ s0 ^ bit  (生成多项式 111)
        out0 = self.state >> 1 ^ self.state & 1 ^ bit
        # 输出1: s1 ^ bit        (生成多项式 101)
        out1 = self.state >> 1 ^ bit

        # 更新状态: 右移1位，新输入进入最高位
        self.state = ((self.state << 1) | bit) & 0x3

        return (out0, out1)

    def encode(self, bits):
        """编码整个比特序列"""
        output = []
        for b in bits:
            output.extend(self.encode_bit(b))
        # 末尾刷新：输入2个0使状态归零
        for _ in range(2):
            output.extend(self.encode_bit(0))
        return output

# 测试
encoder = ConvEncoder()
test_bits = [1, 0, 1, 1]
encoded = encoder.encode(test_bits)
print(f"输入比特: {test_bits}")
print(f"编码输出: {encoded}")
print(f"输出长度: {len(encoded)} (输入4比特 + 2比特刷新，码率约1/2)")

网格图与状态转移

卷积码可以用网格图（trellis diagram）来可视化——这是理解译码的关键工具。

对于K=3的编码器，状态由寄存器中的2个比特决定，共有4种状态：00、01、10、11。每个输入比特（0或1）都导致一次状态转移，并产生一组输出比特。

网格图展开如下：

  • 每一列代表一个时间步
  • 每一行代表一个状态
  • 每个时间步的每个状态都有两条出边（输入0和输入1），也有两条入边
  • 每条边上标注了输入比特和对应的输出比特

举个例子：从状态00出发，输入0 → 输出00，状态保持00。输入1 → 输出11，状态变为10。从状态01出发，输入0 → 输出10，状态变为00。输入1 → 输出01，状态变为10。

随着时间步增加，网格"展开"成一个重复的结构。译码的任务就是：在网格上找到一条路径，使得产生的输出序列与接收到的信号最匹配。

Viterbi算法：最大似然序列译码

Viterbi算法的核心思想极其精妙：在网格的每一列，对于每个状态，只保留一条最优（距离最小）的路径。因为未来的路径选择只依赖当前状态，不依赖如何到达这个状态——这正是动态规划的无后效性（Bellman最优性原理）的完美例子。

算法的具体步骤：

  1. 初始化：从全零状态开始，路径度量为0，其他状态为∞。
  2. 扩展：对于每个时间步，从所有活跃状态出发，沿所有可能的转移到达下一状态，计算新的路径度量（原有度量 + 当前分支的汉明距离）。
  3. 竞争（Add-Compare-Select, ACS）：如果多个路径到达同一状态，只保留度量最小的那个（幸存路径）。
  4. 回溯：处理完所有数据后，从度量最小的终态开始回溯，恢复出最可能的输入序列。

让我们用Python实现一个完整的Viterbi译码器：

import numpy as np

class ViterbiDecoder:
    """用于码率1/2、K=3卷积码的Viterbi译码器"""

    def __init__(self):
        # 4个状态: 00, 01, 10, 11
        self.num_states = 4

        # 状态转移: (当前状态, 输入比特) -> (下一状态, 输出比特对)
        # 生成多项式: g0=111, g1=101
        self.next_state = np.zeros((4, 2), dtype=int)
        self.output = np.zeros((4, 2, 2), dtype=int)

        for s in range(4):
            for inp in [0, 1]:
                # 计算输出
                s1 = (s >> 1) & 1
                s0 = s & 1
                out0 = s1 ^ s0 ^ inp
                out1 = s1 ^ inp
                # 下一状态
                ns = ((s << 1) | inp) & 0x3
                self.next_state[s, inp] = ns
                self.output[s, inp] = [out0, out1]

    def decode(self, received):
        """接收到的软/硬判决比特序列，返回译码结果"""
        T = len(received) // 2  # 时间步数（每步2个输出比特）

        # 路径度量
        path_metric = np.full((T + 1, self.num_states), np.inf)
        path_metric[0, 0] = 0

        # 回溯表: trace[t][s] = 到达状态s的前一状态和输入比特
        trace = np.zeros((T, self.num_states, 2), dtype=int)

        # 前向传播
        for t in range(T):
            out0 = received[2 * t]
            out1 = received[2 * t + 1]

            for s in range(self.num_states):
                if path_metric[t, s] == np.inf:
                    continue
                for inp in [0, 1]:
                    ns = self.next_state[s, inp]
                    o0, o1 = self.output[s, inp]
                    # 汉明距离
                    dist = (out0 ^ o0) + (out1 ^ o1)
                    new_metric = path_metric[t, s] + dist

                    if new_metric < path_metric[t + 1, ns]:
                        path_metric[t + 1, ns] = new_metric
                        trace[t, ns] = [s, inp]

        # 回溯
        best_state = np.argmin(path_metric[T, :])
        decoded = []
        for t in range(T - 1, -1, -1):
            s, inp = trace[t, best_state]
            decoded.insert(0, inp)
            best_state = s

        return decoded

# 测试
encoder = ConvEncoder()
original = [1, 0, 1, 1, 0, 0, 1, 0]
encoded = encoder.encode(original)

# 模拟信道噪声（翻转1个比特）
received = encoded.copy()
received[3] ^= 1  # 在第4个输出比特引入错误
print(f"原始:    {original}")
print(f"编码:    {encoded}")
print(f"接收(有噪声): {received}")

decoder = ViterbiDecoder()
decoded = decoder.decode(received)
print(f"译码:    {decoded}")
print(f"正确:    {decoded == original}")

这个算法的巧妙之处在于：对于网格上的最短路径问题，它不需要枚举所有2n条可能路径——每条路径对应一个完整的比特序列——而只需要在每个时间步为每个状态保留一条幸存路径。复杂度从指数级降为O(T · 2K-1)，其中T是序列长度，K是约束长度。

深入理解：Viterbi算法是最早使用动态规划的实际应用之一。它的精妙之处在于利用了这样一个事实：对于有限状态马尔可夫过程，最优路径的子路径也一定是最优的。这让我们可以用"逐层剪枝"替代"全局搜索"。今天，Viterbi算法是每一个通信工程专业学生的必修内容，但其思想在语音识别、生物信息学中同样大放异彩。

卷积码的经典应用

卷积码+ Viterbi译码在过去的半个世纪中主导了无数通信系统。

系统 码率 约束长度K 生成多项式（八进制） 说明
NASA Voyager 1/2 7 g0 = 171, g1 = 133 深空通信的经典，后续加了Reed-Solomon级联
GSM 1/2 5 g0 = 23, g1 = 33 2G移动通信语音信道
IEEE 802.11a (WiFi) 1/2 ~ 3/4 7 g0 = 171, g1 = 133 通过打孔（puncturing）获取高码率
DVB-S (卫星TV) 1/2 ~ 7/8 7 g0 = 171, g1 = 133 数字卫星电视标准
IS-95 (CDMA) 1/2 ~ 1/3 9 多种组合 高通主导的3G前身

注意到没有？NASA Voyager和现代的WiFi使用了完全相同的生成多项式（171和133，八进制）！1977年发射的Voyager探测器上使用的卷积码方案，至今仍在WiFi芯片中默默运行——这大概是最长寿的编码标准之一了。

Voyager任务是一个值得一提的故事：当旅行者号飞越木星和土星时，距离地球数十亿公里，信号功率衰减到令人难以置信的水平（大约10-16瓦）。在这种条件下，没有强大的纠错码，我们根本不可能收到那些壮丽的木星大红斑照片。卷积码与Viterbi译码提供了约5~6dB的编码增益——这相当于将发射功率提升3~4倍的效果。

小结与预告

卷积码打破了分组码的"块处理"限制，用有限状态机的视角来看待编码。而Viterbi算法则用动态规划的思想，在网格上高效地找到了最可能的发送序列。这一对组合在过去50年中经受住了无数实际系统的考验。

从下一章开始，我们将切换视角——纠错码保护了数据的完整性，但如果你想保护的不是完整性，而是机密性呢？信息论同样给出了优雅的答案。Shannon在1949年发表的另一篇开创性论文中，用熵的概念定义了完善保密性，直接奠定了现代密码学的信息论基础。请进入第18章：信息论与安全。
