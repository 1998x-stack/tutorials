第17章 策略回测体系

大家好，我是你们的讲师。我有一个朋友，2022年的时候兴冲冲地给我看了一个他"发明"的量化策略：在A股市场，每月初买入上月跌幅最大的10只股票，持有一个月后卖出。他在Excel上手工"回测"了三个月的数据，发现这个策略每个月都能跑赢沪深300超过5%。"你觉得怎么样？"他眼睛发亮地问我。

我没有直接回答，而是让他把回测时间拉长到三年。一个周末之后，他沮丧地告诉我：回测结果是个年化亏损8%的垃圾策略。为什么？因为他"回测"的那三个月恰好是2022年4月到6月——A股刚从大跌中反弹，超跌反弹策略在那个特定窗口确实有效，但把时间拉长一看，均值回归的幅度根本不足以覆盖选到"烂股票"的风险。

回测（Backtesting）是用历史数据验证交易策略表现的过程，它几乎是每个量化策略上线前的必经之路。但正如这个例子所揭示的——回测做不对，等于在垃圾堆里找黄金。本章将系统性地介绍回测的方法论、核心指标和常见陷阱，帮你建立一套严谨的回测体系。

17.1 回测框架选型

做回测的第一步是选择一个合适的回测框架。Python生态下有多个优秀的回测库，各有侧重和取舍。下面这张表格帮你快速对比：

框架 执行速度 功能复杂度 资产类型支持 适用场景
vectorbt 极快（向量化） 中低 股票、ETF、加密货币 快速原型验证、大规模参数扫描
backtrader 中等（事件驱动） 高 股票、期货、期权、外汇 精细化策略开发、复杂交易逻辑
vnpy 中等（C++核心加速） 极高（全栈） 期货、CTP接口、股票 实盘交易、期货CTA策略
zipline 中等 高 美股为主 学术研究、美股策略原型

对于个人投资者和入门者，我强烈推荐从vectorbt开始。它的向量化计算让回测速度极快——一次回测可能需要的事件驱动框架要跑几十秒，vectorbt可能一秒就完成了。而且它的API设计非常Pythonic，几个小时就能上手。

选型建议：别在一开始就追求"功能最全"的框架。回测的核心目的是"快速验证想法并发现错误"，而不是"写一个能跑在实盘服务器上的系统"——那是后面的事。从简单的框架开始，等需求复杂了再迁移也不迟。

下面是用vectorbt做一个简单双均线策略回测的示例：

import vectorbt as vbt
import akshare as ak
import pandas as pd

# 获取沪深300指数日线数据
df = ak.stock_zh_index_daily(symbol="sh000300")
price = df['close']

# 计算5日均线和20日均线
fast_ma = price.rolling(5).mean()
slow_ma = price.rolling(20).mean()

# 生成信号：快线上穿慢线时买入，下穿时卖出
entries = fast_ma > slow_ma
exits = fast_ma <= slow_ma

# 运行回测
portfolio = vbt.Portfolio.from_signals(price, entries, exits,
                                        init_cash=1000000,
                                        fees=0.0003)  # 万分之三佣金

# 输出绩效指标
print(f"累计收益率: {portfolio.total_return():.2%}")
print(f"年化收益率: {portfolio.annualized_return():.2%}")
print(f"最大回撤: {portfolio.max_drawdown():.2%}")
print(f"夏普比率: {portfolio.sharpe_ratio():.2f}")
print(f"交易次数: {portfolio.trades.count()}")

# 可视化净值曲线
portfolio.plot().show()

这个策略当然非常基础，但它的框架是完整的：数据获取 → 信号生成 → 回测执行 → 绩效分析。你在实盘中见到的绝大多数策略，无论是简单的均线策略还是复杂的多因子模型，都遵循这个逻辑链条。

17.2 核心绩效指标

回测完成后，面对一堆输出数据，我们需要用标准化的指标来评估策略的"好坏"。以下是最关键的几项：

指标 公式 解读 注意事项
年化收益率 (1+R)^(252/N) - 1 策略的年度复合增长率 回测期越短越不可靠
最大回撤 max(Peak - Valley) / Peak 历史最高点到最低点的跌幅 衡量策略的"痛苦程度"
夏普比率 (Rp - Rf) / σp 风险调整后收益 A股通常 > 1 就算优秀
胜率 盈利交易次数 / 总交易次数 交易判断的准确率 单看胜率没有意义，需结合盈亏比
盈亏比 平均盈利 / 平均亏损 盈利时赚多少 vs 亏损时亏多少 趋势策略通常胜率低但盈亏比高
卡玛比率 (Rp - Rf) / MDD 每单位最大回撤的回报 比夏普更"接地气"的风险指标

一个经常被忽略的关键点是：以上所有指标都只是"样本内"（in-sample）的表现，并不代表"样本外"（out-of-sample）的表现。换句话说，你用过去5年数据回测得到的夏普比率2.0，到了未来可能变成0.5甚至负数。

核心洞见：我判断一个策略是否"值得认真对待"，通常会看三件事：① 策略逻辑是否有经济直觉支撑（而不是纯数据挖掘）；② 样本外表现是否显著优于随机；③ 最差情况下的回撤是否在可接受范围内。回测数据只是门槛，过了门槛之后，逻辑和鲁棒性比完美的数字重要得多。

17.3 回测的致命陷阱

如果说回测是一座金矿，那陷阱比金子多得多。下面这几条是我自己在实战中踩过或亲眼见过的"回测杀手"：

1. 前视偏差（Look-Ahead Bias）——这是回测中最常见也最危险的错误。指的是在回测中使用了"在当时还无法获得"的信息。例如，在计算均线时使用了当天的收盘价来生成当天的买卖信号——但实际交易中，你只能在收盘后才能用这个收盘价来计算信号和下单。

2. 幸存者偏差（Survivorship Bias）——回测时只使用了当前还在交易的股票，忽略了那些已经退市或被ST的股票。这会导致回测结果系统性偏高，因为存活下来的股票本身就有"好股票"的偏误。有人估计，忽略幸存者偏差可能使回测的年化收益高估3%-5%。

3. 过拟合（Overfitting）——这是最隐蔽的陷阱。当你反复调整策略参数（移动平均线的天数、RSI的阈值、止损的百分比等）以追求过去数据上的最优表现时，你实际上是在"拟合噪音"——把随机的波动当成了规律。一个过拟合的策略在样本内看起来完美无缺，但一到真实数据上就会原形毕露。

4. 忽略滑点和交易成本——回测中假设每次都能以收盘价成交，市场有无限的流动性。但在现实中，大单会推动价格、极速行情会有滑点、高频交易会吃掉价差。对于一个日频交易、每月买卖几次的策略，忽略万分之几的佣金和冲击成本影响不大；但对于高频策略，交易成本可能就是盈利和亏损的全部区别。

踩坑警示：我见过一个真实案例。2020年，某量化团队精心开发了一个CTA策略，在5年的历史数据上回测年化收益35%，最大回撤仅12%，夏普比率2.3。团队信心满满地开始实盘，结果6个月后亏损了18%。事后复盘发现：回测时他们使用了"昨日收盘价"作为成交价，但实际上他们的策略判断是在盘中形成的，真正成交时价格早已偏离了收盘价好几个tick——再加上期货的滑点和手续费，一个看起来稳赚的策略实际上稳亏。这个教训的代价是几百万。

5. 数据窥探偏差（Data Snooping Bias）——你在同一个数据集上测试了100个策略，总会有那么几个看起来效果不错。这完全是统计上的"多重比较"问题。纠正方法包括：使用独立的样本外数据验证、交叉验证、或者至少对多重比较进行统计修正。

6. 未来信息泄漏——这具体表现为：回测时你可能无意中使用了未来的财务数据（如年报发布后的数据来生成信号，但信号本该基于三个季度前的数据），或者使用了未来的宏观数据修正（如GDP修订值而不是初值）。

自查清单：每次你拿到一个"惊艳"的回测结果时，先跑一遍下面的checklist：① 回测中是否用了未来数据？② 是否考虑了所有交易成本（佣金+印花税+滑点）？③ 是否做了样本外测试（至少把数据切成训练集和测试集）？④ 策略逻辑是否有经济解释？⑤ 如果把参数微调一下，结果是否仍然稳健？这五点过完，大多数"好策略"都会露出破绽。

一个完美回测曲线的惨痛教训

让我再讲一个真实的故事。2017年，某知名私募在路演时展示了一个"多因子选股策略"的回测曲线——从2012年到2017年，净值曲线几乎以45度角稳步上升，只在2015年股灾期间出现了一个不到10%的小回撤，然后迅速创新高。年化收益41%，最大回撤9.6%，夏普比率3.1。投资人看到这个曲线，几乎是抢着把钱投进去。

结果呢？2018年，这个策略亏损了32%，同期沪深300只跌了25%。更讽刺的是，这个"阿尔法策略"在2018年的每一波下跌中都跌得比市场更惨——说明它非但没有选股能力，反而系统性地选中了更差的股票。

事后复盘发现了问题的根源：该策略的回测包含了60多个因子，团队用了长达五年的历史数据反复调试后，"恰好"选出了在2012-2017年间表现最好的一组因子权重。但这组权重完全是在"倒后镜里开车"——那些在2012-2017年有效的因子（如小市值、高弹性），在2018年的市场环境下变成了"毒药"。这就是过拟合的经典症状：一个完美的回测曲线，往往是过拟合的最有力证据。

真正的教训是：回测不是用来"证明一个策略好"的，而是用来"发现一个策略哪里有问题的"。如果你的回测结果完美到没有瑕疵，那它大概率是有问题的。

17.4 从回测到实盘

当你有了一个通过了基础检验的回测结果，下一步就是把它推向实盘。但实盘和回测之间的鸿沟，远比大多数初学者想象的大。下面是一个推荐的渐进式上线路径：

第一步：Paper Trading（模拟交易）——在模拟环境中运行策略至少1-3个月。模拟交易虽然不需要真金白银，但能暴露你在回测中忽略的问题：数据源的实时性、信号的计算延迟、API的稳定性等。

第二步：小资金验证——用总资金的1%-5%进行真金实银的交易。这个阶段的目标不是赚钱，而是验证"回测中的收益/风险特征是否能在实盘中复现"。特别是要注意实盘成交价格与回测价格之间的差异——如果滑点超过了预期，说明策略的流动性假设是错的。

第三步：逐步放量——如果小资金验证3-6个月后表现稳定（与回测大致相符），可以逐步增加资金量。注意"逐步"——每次加量后都要观察效果，因为更大的资金量意味着更大的市场冲击。

第四步：监控与预警——即使策略上了全仓位，持续监控也必不可少。设定明确的预警阈值：当日回撤超过X%、连续X周跑输基准、策略的核心参数发生漂移——任何一条触发都值得重新评估策略是否仍然有效。

核心洞见：永远不要——我再重复一遍——永远不要在一开始就用全仓资金上实盘。无论你的回测做得多么细致、逻辑多么自洽，市场的复杂性决定了回测和实盘之间必然存在差距。这个差距可能很小（万分之几的滑点差异），也可能很大（策略逻辑本身已经失效）。渐进式上线是对自己和资金最好的保护。
