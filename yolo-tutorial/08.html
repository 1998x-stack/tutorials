第8章 锚框机制与聚类

大家好，我是你们的讲师。锚框是YOLO系列中最微妙的设计之一——选好了模型事半功倍，选错了训练都难以收敛。你可能不知道，YOLOv3的9个锚框尺度过经历了怎样的网格搜索。从YOLOv2引入锚框开始，这个小小的先验设计就深刻影响了整个目标检测领域的发展方向。

本章我们将深入剖析锚框（anchor box）的设计思想、聚类生成方法、赋值策略，以及从有锚到无锚的演进历程。理解锚框，是理解YOLO检测头设计的基石。

8.1 为什么需要锚框

在YOLOv1中，每个网格单元直接预测边界框的绝对坐标（x, y, w, h）。这种做法有两个明显的问题：第一，直接预测绝对坐标让梯度的范围非常不稳定，小物体的偏移和大物体的偏移尺度差异巨大；第二，没有任何先验知识来约束输出框的形状，模型需要从头学习所有可能的宽高比。

锚框（也称为先验框，priors）的核心思想是：在训练之前，我们就定义好一组预设的边界框形状（宽度和高度），让模型在预测时只需学习相对于这些预设框的偏移量，而非直接预测绝对坐标。这大大降低了学习的难度。

锚框的优势： 将边界框预测分解为"在哪附近"（锚框中心）和"是什么形状"（锚框尺寸），使得模型可以专注于微调而非全局搜索。

8.2 锚框的设计原理

每个锚框由以下参数定义：中心坐标（相对于网格单元）、宽度和高度（相对于整幅图像的比例），以及与之关联的形状先验（aspect ratio）。在YOLOv2及之后的版本中，模型预测的是偏移量 t_x, t_y, t_w, t_h，而非直接坐标。

参数 预测值 解码公式 含义
中心x t_x b_x = σ(t_x) + c_x 相对网格单元的偏移（sigmoid约束）
中心y t_y b_y = σ(t_y) + c_y 相对网格单元的偏移（sigmoid约束）
宽度 t_w b_w = p_w * e^(t_w) 相对锚框宽度的缩放
高度 t_h b_h = p_h * e^(t_h) 相对锚框高度的缩放
置信度 t_o obj = σ(t_o) 该框包含物体的概率

其中 σ(t) 表示sigmoid函数，c_x, c_y 是网格单元的左上角坐标，p_w, p_h 是锚框的先验宽高。通过sigmoid将中心偏移限制在0-1之间，确保每个锚框只能预测其所在网格单元附近的物体。

8.3 K-means聚类生成锚框

锚框既然是"先验"知识，那么如何获得一组好的先验呢？YOLOv2提出了一种优雅的方法：在训练数据集上使用K-means聚类，从所有标注框的尺寸中自动学习出最具代表性的宽高组合。

这里有一个关键细节：传统的K-means使用欧氏距离作为度量，但在锚框聚类中，欧氏距离会导致大框产生更大的误差，从而主导聚类结果。YOLOv2的作者创新地使用了IoU距离作为度量指标：

距离度量 d(box, centroid) = 1 - IoU(box, centroid)

这种距离度量只关心框之间的重叠程度，与框的绝对大小无关。对于每个聚类的中心 (w_i, h_i)，K-means迭代地调整它们，使得所有框到其最近中心点的IoU距离之和最小化。

8.4 锚框赋值策略

有了锚框之后，下一个问题是：训练时如何将ground truth分配给具体的锚框？这个步骤称为锚框赋值（anchor assignment）。

YOLOv3的赋值策略

YOLOv3采用了一个相对简单的策略：对于每个ground truth框，计算它与该特征图上所有锚框的IoU，选择IoU最大的那个锚框作为正样本。同时，如果某个锚框与任何ground truth的IoU都低于阈值（通常是0.5），则它被标记为负样本（背景）。

TaskAlignedAssigner（YOLOv8）

从YOLOv8开始，赋值策略演进为更精细的任务对齐分配器（TaskAlignedAssigner）。它不再单纯依赖IoU，而是综合分类得分和回归质量来选择正样本：

alignment_score = classification_score^α × IoU^β

其中α和β是超参数（通常α=1, β=6）。对于一个ground truth框，选择alignment score最高的top-k个锚框作为正样本。这种策略确保了分类和回归任务之间的对齐——一个框既要"分得准"也要"框得正"才能被选为正样本。

8.5 从有锚到无锚

锚框机制虽然有效，但也带来了几个问题：

  • 超参数敏感： 锚框的数量和尺寸需要精心设计，不同数据集差异很大
  • 正负样本不均衡： 大量锚框被标记为负样本，导致训练效率低下
  • 推理开销： 需要解码锚框偏移量，增加了后处理复杂度
  • 对小物体不友好： 固定锚框难以覆盖极端尺寸的物体

因此，YOLOv8和YOLOv10向锚框自由（anchor-free）方向演进。YOLOv8中，每个位置只预测一个框（不再有多个锚框），直接将特征图上的每个点视为潜在物体的中心。分类头判断该点属于哪个类别，回归头直接预测从该点到边界框四条边的距离：

左边界距离 l = x - x1
上边界距离 t = y - y1
右边界距离 r = x2 - x
下边界距离 b = y2 - y

这种anchor-free设计的优势在于：消除了锚框超参数调优的痛苦，简化了模型结构，同时也保持了与有锚版本相当的精度。

注意： anchor-free并不意味着完全抛弃了先验知识。实际上，YOLOv8的anchor-free设计仍然依赖于特征金字塔的多尺度结构——不同尺度的特征图天然地对应了不同大小的物体，这本身就是一种隐式的先验。

8.6 实战：K-means聚类锚框

下面是一个在自定义数据集上通过K-means聚类生成锚框的Python示例：

import numpy as np
from sklearn.cluster import KMeans

def iou_distance(box, centroids):
    """计算box与centroids之间的IoU距离"""
    # box: (w, h), centroids: (k, 2)
    w1, h1 = box
    ious = []
    for cw, ch in centroids:
        inter_w = min(w1, cw)
        inter_h = min(h1, ch)
        inter = inter_w * inter_h
        union = w1 * h1 + cw * ch - inter
        iou = inter / (union + 1e-8)
        ious.append(iou)
    return np.array(ious)

def kmeans_anchors(bboxes, k=9, iterations=300):
    """对数据集bboxes进行K-means聚类生成锚框"""
    # bboxes: (N, 2) 每行为(w, h)，已归一化到[0,1]
    # 随机初始化k个中心
    centroids = bboxes[np.random.choice(len(bboxes), k, replace=False)]
    for _ in range(iterations):
        # 计算每个框到每个中心的IoU距离
        distances = np.array([
            1 - iou_distance(b, centroids) for b in bboxes
        ])
        # 分配到最近的中心
        labels = np.argmin(distances, axis=1)
        # 更新中心（取该类的中位数）
        new_centroids = np.array([
            np.median(bboxes[labels == i], axis=0) for i in range(k)
        ])
        if np.allclose(centroids, new_centroids, atol=1e-4):
            break
        centroids = new_centroids
    # 按面积排序
    areas = centroids[:, 0] * centroids[:, 1]
    sorted_idx = np.argsort(areas)
    return centroids[sorted_idx]
关键实践建议：
  • 聚类前将所有标注框的宽高归一化到[0,1]区间（除以图像宽高）
  • 使用中位数而非均值来更新中心，对异常值更鲁棒
  • 最终锚框按面积从小到大排序，小锚框分配给大特征图
  • K值的选择：一般9个锚框（3尺度×3个/尺度）在COCO上表现好；简单数据集可以用6个

8.7 总结与展望

锚框机制从YOLOv2引入至今，经历了从单尺度5框到三尺度9框、再到YOLOv8/v10的anchor-free的巨大转变。这一演进反映了目标检测领域对"先验知识"理解的变化：早期我们依赖明确的手工先验（固定宽高比），后来学会了从数据中自动学习（K-means聚类），现在则通过多尺度特征图隐式地编码物体的尺寸信息。

理解锚框的来龙去脉，对于掌握YOLO系列的设计哲学至关重要。下一章，我们将深入另一个决定检测精度的核心组件——损失函数，看看从YOLOv1到YOLOv10，损失函数经历了怎样的变迁。
