第十五章：NLP评估指标

大家好！前面我们学了一大堆模型，但一个根本问题还没回答：怎么判断模型好不好？这一章我们就来系统讨论NLP的评估。这件事比很多人想的复杂——自动指标和人的感觉之间经常有鸿沟，而选错指标可能导致你优化了一个完全错误的方向。

我2019年做过一个对话生成项目。团队用BLEU评估，优化了两个月，BLEU从0.15涨到0.22。上线后用户反馈"回复不像人说的话"。后来换成人工评估才发现——模型学会了说安全的废话（"嗯嗯""好的呀""是这样呢"），BLEU高是因为这些短回复正好在reference里出现。血的教训：指标驱动行为，选错指标就优化错了目标。

15.1 分类任务的评估指标

分类是NLP最常见的任务形式。别被"准确率95%"这种单一数字骗了——它可能在多数类上95%，少数类上0%。

# 混淆矩阵 (Confusion Matrix)
#               预测正    预测负
# 实际正        TP        FN
# 实际负        FP        TN

# Accuracy  = (TP+TN) / (TP+TN+FP+FN)
# Precision = TP / (TP+FP)   # 预测为正中实际正的占比
# Recall    = TP / (TP+FN)   # 实际正中预测出的占比
# F1        = 2·P·R / (P+R)   # P和R的调和平均

# 例子：100个样本，5个正类（罕见病诊断）
# 全预测负 → Accuracy=95%，但Recall=0%
# 这个模型"95%准确率"，但一个病人都没诊断出来——完全没用
指标关注什么适合场景不适合场景
Accuracy整体对错类别平衡类别不平衡（欺诈检测、罕见病）
Precision预测正例中多少是对的不能有误报（垃圾邮件过滤）需要找出所有正例
Recall实际正例中多少被找出不能有漏报（疾病筛查）可以承受一些误报
F1P和R的平衡P和R都重要对P和R有不同偏好时

⚠️ 常见坑：面试时很多人说"F1是准确率和召回率的调和平均"——错！F1是Precision和Recall的调和平均，跟Accuracy没关系。这种基础概念面试答错直接就挂了。

15.2 生成任务的评估：BLEU与ROUGE

分类有明确的标签，评估简单。生成任务（翻译、摘要）的评估就复杂得多——同样的意思有无数种表达方式。

# BLEU (Bilingual Evaluation Understudy)
# 核心思想：比较生成文本和参考文本的n-gram重叠度
# BLEU-1: unigram precision
# BLEU-4: 1-gram到4-gram的加权几何平均（最常用）

# 计算步骤：
# 1. 统计生成文本中每个n-gram出现的次数
# 2. 对每个n-gram，用它在参考文本中的最大出现次数截断（避免重复加分）
# 3. 计算修正后的n-gram precision
# 4. 加长度惩罚（BP）：生成太短要扣分

from nltk.translate.bleu_score import sentence_bleu
ref = [["the", "cat", "is", "on", "the", "mat"]]
hyp = ["the", "cat", "sat", "on", "the", "mat"]
score = sentence_bleu(ref, hyp, weights=(0.25, 0.25, 0.25, 0.25))
# weights均匀 = BLEU-4

ROUGE（Recall-Oriented Understudy for Gisting Evaluation）主要用于摘要评估。和BLEU的最大区别——BLEU侧重精确率（生成内容有多少在参考中），ROUGE侧重召回率（参考内容有多少被生成出来）。

指标侧重主要用途局限
BLEUPrecision机器翻译不关心召回、对同义词不敏感
ROUGE-1Unigram Recall摘要忽略词序
ROUGE-2Bigram Recall摘要（考虑词序）对改写敏感
ROUGE-L最长公共子序列摘要（考虑结构）计算较慢

15.3 自动指标的局限性

BLEU和ROUGE的最大问题是完全基于表面形式（n-gram重叠），对语义无感知。"The cat sat on the mat"和"The feline rested on the rug"意思几乎一样，但BLEU得分可能为零。这个问题在大模型时代变得更突出——大模型生成的内容往往"换了一种说法"，自动指标完全检测不到。

2026年的实践：评估生成质量时，自动指标只能做初筛。真正的判断要靠：1）GPT-4作为评判者（LLM-as-judge），2）人工评估。用GPT-4打分和人工评分的相关性在0.7-0.85之间，比BLEU的0.3好得多。但也要注意LLM-as-judge可能有位置偏见（偏好第一个候选）和长度偏见。

15.4 什么时候需要人工评估

以下几种情况必须做人工评估：1）任务目标是"像人"（对话、创意写作），2）自动指标和业务指标不一致，3）上线前的最终检查。人工评估的成本很高——标注员要培训、标注一致性（IAA）要计算、样本量要足够。但它是检验NLP系统真实质量的唯一金标准。

好了，评估指标给了我们衡量模型好坏的尺子。但模型好坏不仅看指标——还要看公平性和偏见。下一章我们聊一个更敏感但绝对不可回避的话题。
