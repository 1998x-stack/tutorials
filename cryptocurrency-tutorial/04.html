第4章 区块链是什么 — 不可篡改的分布式账本

大家好，我是你们的讲师。前面两章我们分别讲了密码学基础和比特币白皮书——一个是微观工具箱，一个是宏观蓝图。现在到了把它们串联起来的时刻：区块链（blockchain）——这个你听过无数遍却可能从未真正理解的词。

如果你去问街上的路人"区块链是什么"，大概率会得到"是一种分布式的、去中心化的、不可篡改的技术"这类教科书般的答案。这没错，但太空洞。这一章，我们会打开一个真实的比特币区块，看看里面到底有什么、每部分干什么、这些零件怎么拼成一个不可篡改的账本。

课前提醒：这一章会涉及一些技术细节，但请记住——你不需要记住每个字段的精确字节数。你需要的是理解"为什么这样设计"。一旦理解了这个"为什么"，区块链的神秘感就消失了。它只是一套设计精良的数据结构。

4.1 区块的解剖

比特币的每个区块（block）包含两部分：区块头（block header）和交易列表（transactions）。区块头只有80字节，但包含了所有关键信息。

让我们解剖一个真实的区块（以区块#800000为例）：

字段大小说明真实数据（节选）
版本号 (Version)4 字节区块协议的版本，指示支持的规则0x20400000
前区块哈希 (Prev Block Hash)32 字节指向父区块的链接（区块链的"链"）000000...a3b2
Merkle根 (Merkle Root)32 字节所有交易的哈希树的根d7e8...9f1c
时间戳 (Timestamp)4 字节矿工开始挖这个区块的Unix时间2024-01-15 14:32:18
难度目标 (Bits)4 字节压缩后的挖矿难度目标值0x17040b2e
随机数 (Nonce)4 字节矿工不断修改以寻找有效哈希3128456790
交易计数 (Tx Count)1-9 字节 (可变)区块中包含的交易数量2,512 笔交易

区块体的核心就是这笔交易列表——包括一个Coinbase交易（矿工给自己发的新币奖励 + 手续费）和所有被打包的用户交易。

区块结构（概念图）：

┌─────────────────────────────────────────────────┐
│                    区块                          │
│  ┌───────────────────────────────────────────┐  │
│  │               区块头 (80 bytes)             │  │
│  │  ┌──────┬──────────┬──────────┬────────┐  │  │
│  │  │版本号 │ 前哈希    │ Merkle根 │ 时间戳 │  │  │
│  │  ├──────┼──────────┼──────────┼────────┤  │  │
│  │  │难度目 │ Nonce    │          │        │  │  │
│  │  │标     │          │          │        │  │  │
│  │  └──────┴──────────┴──────────┴────────┘  │  │
│  └───────────────────────────────────────────┘  │
│                                                  │
│  ┌───────────────────────────────────────────┐  │
│  │              交易列表                       │  │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐  │  │
│  │  │ Coinbase │ │ Tx #1    │ │ Tx #2    │  │  │
│  │  │ 交易     │ │          │ │    ...   │  │  │
│  │  └──────────┘ └──────────┘ └──────────┘  │  │
│  └───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
补充知识：Nonce是4字节，最大值约42亿。在比特币早期，Nonce循环一遍就够找到有效哈希了。而到2024年，全网算力已经到了惊人的级别，Nonce从0跑到42亿可能连一个哈希都挖不到。矿工不得不通过修改Coinbase交易中的ExtraNonce字段（变相延长Nonce空间）来继续寻找有效区块。

4.2 链式结构与防篡改原理

区块链的核心创新是"链式结构"——每个区块通过前区块哈希字段引用前一个区块。这种设计有一个美妙的防篡改特性：

假设有人想修改区块链历史中的某个交易。他需要：

  • 修改目标区块中的交易数据
  • 重新计算该区块的Merkle根——因为交易变了
  • 重新计算该区块的哈希——因为Merkle根变了
  • 找到满足难度目标的新Nonce——因为哈希变了
  • 修改下一个区块中的"前区块哈希"字段——因为前一个区块的哈希变了
  • 重新计算下一个区块的哈希并找到新Nonce
  • 以此类推，修改后面所有区块直到最新

这需要的力量是多少？我们做个计算：

防篡改的数学壁垒（2026年数据）：

全网算力 ≈ 600 EH/s（exahashes 每秒）
          = 600,000,000,000,000,000,000 次哈希/秒

篡改一个 6 次确认的区块 ≈ 需要重算 6 个区块
难度 ≈ 找 1 个区块需要全网挖 10 分钟
篡改 6 个区块 ≈ 6 × 10 分钟 = 1 小时的全网算力
              ≈ 600 EH × 3600 秒 = 2.16 × 10^24 次哈希

这个算力的电费成本 ≈ $300,000 - $500,000
且需要控制全网 51% 以上的算力才能"赶上"诚实的链

结论：超过 6 次确认的区块，实际上不可篡改。
一种常见的误解："区块链上的数据是加密的，所以别人看不懂。"这是错的！区块链上的交易数据大部分是明文的。比如转账地址、金额都是公开可见的。密码学保护的是所有权（只有私钥持有者能花费），而不是隐私（所有人都能看到交易内容）。这正是为什么比特币被称为"伪匿名"（pseudonymous）而非"匿名"（anonymous）。

4.3 Merkle树

Merkle树（默克尔树）是区块结构中非常精妙的设计，由计算机科学家Ralph Merkle在1979年发明。它的存在解决了一个实际问题：如何在不下载整个区块的情况下，验证一笔交易确实在区块中？

想象一下：你在手机上用轻钱包（SPV钱包）运行比特币。你肯定不想下载整个区块链（超过600GB）——手机根本存不下。但你同时需要验证交易是否被确认了。Merkle树就是解决方案。

Merkle树的构造方式：

Merkle树构造过程：

                          Top Hash
                       (Merkle Root)
                     ┌──────┴──────┐
                     │             │
              Hash AB             Hash CD
            ┌────┴────┐       ┌────┴────┐
            │         │       │         │
         Hash A     Hash B  Hash C    Hash D
           │          │       │         │
        Tx A        Tx B    Tx C      Tx D

构造步骤：
1. 对每笔交易计算哈希：Hash A = SHA256(Tx A)
2. 相邻哈希配对并再次哈希：
   Hash AB = SHA256(Hash A || Hash B)
3. 递归向上，直到只剩一个哈希（Merkle Root）

验证交易 C 是否在区块中：
   SPV钱包收到：Tx C + Hash D + Hash AB
   自己计算：Hash C = SHA256(Tx C)
   自己计算：Hash CD = SHA256(Hash C || Hash D)
   自己计算：Top Hash = SHA256(Hash AB || Hash CD)
   对比 Top Hash 是否等于区块头中的 Merkle Root

   只需提供 3 个哈希（而不是全部 4 笔交易）就能验证！
   如果有 1000 笔交易，只需要 log2(1000) ≈ 10 个哈希

Merkle Proof（默克尔证明）让轻客户端可以用极小的数据量验证交易。这个机制对于"可扩展性"至关重要——正是一千万用户都能用手机验证比特币交易的基础。

发明故事的启发：Merkle在1979年提交他的专利时，给这个数据结构的描述是"一种基于哈希的数字签名系统"。他当时可能没想到——40多年后，这个结构成为了一万亿美元资产的安全基础。这就是基础研究的力量：你不知道今天的数学发现会在几十年后变成什么。

4.4 创世区块

2009年1月3日，中本聪挖出了比特币的创世区块（Genesis Block），即区块高度0。这是区块链世界的"第一天"。

创世区块有几个有趣的特征：

  • 不可花费的产出：创世区块的50 BTC奖励是永远无法花费的——Coinbase交易的输出脚本是一个自定义的脚本，不是标准的P2PKH。中本聪可能是有意为之，也可能是一开始设计中的"特殊处理"
  • 硬编码的哈希：前区块哈希被设为0——因为没有前一个区块
  • 隐藏的消息：Coinbase交易中嵌入了当天《泰晤士报》的标题："The Times 03/Jan/2009 Chancellor on brink of second bailout for banks"
  • 非标准时间戳：创世区块的时间戳是2009-01-03 18:15:05 UTC
  • 难度：创世区块的难度是1——在当时，普通的CPU就能挖矿

创世区块的Coinbase地址是1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa。有趣的是，这个地址后来收到了来自全球各地网友的捐赠——有些人把比特币发送到这个"最古老的地址"以示敬意。截至2026年，这个地址仍然持有大约66 BTC。

你可以自己验证：在任意区块链浏览器（如mempool.space或blockchair.com）中搜索创世区块的哈希 000000000019d6689c085ae165831e934ff763ae46a2a6c172b3f1b60a8ce26f。看到原文中嵌入的泰晤士报标题，你会切身感受到——这在当时是一个宣言。

4.5 区块链不等于比特币

很多人把"区块链"和"比特币"混为一谈。实际上，区块链是一个更通用的数据结构，比特币只是它的第一个应用场景。

区块链的本质：一个只追加的、分布式的时间戳数据库（append-only, distributed, timestamped database）。任何需要"按时间顺序记录不可篡改历史"的场景都可以应用区块链。

后来出现的各种区块链平台，各有不同的取舍：

平台共识机制设计目标典型应用
比特币 (Bitcoin)PoW去中心化数字现金价值存储、支付
以太坊 (Ethereum)PoS (原PoW)全球计算机（智能合约）DeFi、NFT、DAO
SolanaPoH + PoS高吞吐量公链高频交易、游戏
Hyperledger Fabric允许节点共识企业级联盟链供应链、政务
区分两个概念："区块链"是一种技术，"比特币"是建立在区块链上的一个应用。就像"互联网"是一种网络，"电子邮件"是互联网上的一个应用。但比特币的区块链是最安全、最去中心化、运行时间最长的区块链——这也是为什么它的资产价值最高。

本章我们打开了区块链的黑盒，看到了区块头的80字节数据结构、链式链接的防篡改机制、Merkle树的优雅设计、以及创世区块的历史意义。但区块链本身只是一个"记录载体"——它需要一种机制来决定"谁有权添加下一个区块"以及"大家都同意哪条链是正确的"。这就是我们下一章要讲的内容：去中心化共识。
