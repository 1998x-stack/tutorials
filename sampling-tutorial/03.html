第3章 简单随机抽样

大家好，我是你们的讲师。上一章我们建立了抽样的"通用语言"——总体、参数、统计量、误差、置信区间。现在，我们终于要学习第一个具体的抽样方法了。

简单随机抽样（Simple Random Sampling, SRS）是所有概率抽样方法中最基础、最直观的一个。说它"简单"，是因为它的原理极其直白：从总体中等概率、独立地抽取个体。但你千万不要因为它"简单"就看不起它——事实上，几乎其他所有抽样方法（分层、整群、系统、PPS等）的设计思想，都是在SRS的基础上针对某些缺陷进行改进的结果。不理解SRS，就无法理解后续的任何方法。

在这一章中，我们还会介绍一个二战期间的传奇案例——"德国坦克问题"，看看盟军统计学家如何用SRS的思维，仅凭缴获坦克上的序列号就估算出了德军坦克的月产量，准确度远超情报部门的传统手段。

SRS的基本原理

核心思想

SRS的核心思想可以用一句话概括：从包含N个个体的总体中，随机抽取n个个体，使得每个可能的样本组合都有相同的概率被抽中。这个简单的原则赋予了SRS两个关键性质：

  • 无偏性：样本均值是总体均值的无偏估计——长期平均等于真值。
  • 可计算性：抽样误差可以用公式精确计算——你知道了样本量，就知道了误差范围。

有放回 vs 不放回

SRS有两种实现方式，它们的差异虽然微妙，但在理论上很重要：

维度 SRSWR（有放回） SRSWOR（不放回）
抽取方式 每次抽取后放回总体，下次仍可再次抽中 抽取后不放回，每个个体最多被抽一次
样本独立性 每次抽取相互独立 抽取不独立（后续抽取受前面结果影响）
样本均值的方差 Var(x̄) = σ²/n Var(x̄) = (σ²/n)·(N-n)/(N-1)
FPC因子 无（等同于总体无限） 有，(N-n)/(N-1) 即有限总体校正
效率 方差更大（效率更低） 方差更小（效率更高）
实际使用 极少（除了bootstrap重抽样） 绝大多数场景

两者方差之间就差了一个因子，即有限总体校正（Finite Population Correction, FPC）：

FPC = (N - n) / (N - 1)

当n相对于N很小时（比如抽样比例不到5%），FPC ≈ 1，SRSWOR和SRSWR几乎没有区别。但当抽样比例较大时（比如抽了总体的30%），FPC的修正效应就很明显了——不放回抽样的方差要小得多。这也很直观：不放回抽样中，你不可能把同一个个体抽两次，所以样本携带的信息量更大。

实际应用中的FPC：在大多数市场调研和民意调查中，总体N通常很大（百万级），样本n通常只有1000左右，抽样比不到0.1%，所以FPC几乎等于1，可以忽略。但在企业客户调研（客户总数可能只有几千）、学校调查、工厂质量抽检等场景中，抽样比可能达到10%甚至更高，这时必须使用FPC。我有一个做企业调研的朋友就踩过这个坑——客户总数只有800家，他抽了200家，算标准误时忘了用FPC，结果把置信区间报宽了，差点被客户质疑专业性。

估计量及其性质

均值的估计

在SRSWOR下，总体均值μ的估计就是样本均值x̄：

x̄ = (1/n) Σyᵢ

它的方差（即估计量的方差，反映了估计的波动性）为：

Var(x̄) = (σ²/n) × (N-n)/(N-1)

由于总体方差σ²通常未知，我们用样本方差s²来代替：

s² = Σ(yᵢ - x̄)² / (n-1)

于是标准误（Standard Error）的估计值为：

SE(x̄) = s / √n × √(N-n)/(N-1)

比例的估计

当目标变量是分类变量（如"支持/不支持"、"买/不买"）时，我们关心的是总体比例π。样本比例 p̂ = x/n 是π的无偏估计，其中x为样本中具有该特征的个体数。其方差为：

Var(p̂) = [π(1-π)/n] × (N-n)/(N-1)

由于π未知，用p̂代替：

SE(p̂) = √[p̂(1-p̂)/(n-1)] × √(N-n)/(N-1)

当n足够大时（通常要求np̂ ≥ 5 且 n(1-p̂) ≥ 5），p̂的抽样分布近似正态，我们可以构造置信区间：

p̂ ± zα/2 × SE(p̂)

一个直觉：比例的方差公式中 p̂(1-p̂) 这个项在 p̂=0.5 时取最大值 0.25。这意味着当比例接近50%时，抽样误差最大；当比例接近0%或100%时，抽样误差最小。这就是为什么你在民调中经常看到"抽样误差±3%"——他们假设了最保守的情况（p̂=0.5）来估算误差上限。如果实际比例是10%或90%，真实的抽样误差其实比±3%要小。

样本量公式

这是实际工作中最常遇到的问题：给定所需的精度，需要多大的样本量？

对于比例估计，在SRS下（忽略FPC时），样本量的计算公式为：

n₀ = z²pq / d²

其中：

  • z：标准正态分布的临界值（95%置信度下取1.96，通常近似为2）
  • p：预估的比例（最保守取0.5）
  • q：1 - p
  • d：允许的边际误差（Margin of Error），如±0.03表示3%

举个例子：假设我们要调查某市居民对垃圾分类政策的支持率，希望误差不超过±3%，置信度95%。用最保守的p=0.5：

n₀ = 1.96² × 0.5 × 0.5 / 0.03² ≈ 1067

如果考虑到有限总体校正，需要对n₀做调整：

n = n₀ / [1 + (n₀ - 1)/N]

当N很大时，n ≈ n₀；当N较小时（比如全市只有5000户），n会显著小于n₀。

def sample_size_srs(mar_error, conf_level=0.95, p=0.5, N=None):
    """
    计算SRS所需样本量
    mar_error: 允许的边际误差（如0.03）
    conf_level: 置信度
    p: 预估比例（默认0.5最保守）
    N: 总体大小（None表示忽略FPC）
    """
    z = {0.90: 1.645, 0.95: 1.96, 0.99: 2.576}[conf_level]
    n0 = z**2 * p * (1-p) / mar_error**2
    if N is not None:
        n = n0 / (1 + (n0 - 1) / N)
    else:
        n = n0
    return int(round(n))

print(f"d=±3%, p=0.5: n0 = {sample_size_srs(0.03)}")       # 约1067
print(f"d=±5%, p=0.5: n0 = {sample_size_srs(0.05)}")       # 约385
print(f"d=±3%, N=5000: n = {sample_size_srs(0.03, N=5000)}")  # 约880
print(f"d=±3%, N=10000: n = {sample_size_srs(0.03, N=10000)}") # 约969
样本量的"反直觉"事实：从上面的计算你可以看出，当N从5000变成10000时，所需样本量只增加了不到100。事实上，一旦N超过某个阈值（大约是20倍n₀），FPC的影响就微乎其微了。这意味着：给1000万人的全国做调查，和给100万人的全省做调查，所需要的样本量几乎一样多！关键在于绝对样本量（absolute sample size），而不是抽样比。很多刚入行的调研人员不理解这一点，总以为"全国调查需要几万个样本才够"——其实1000个精心抽取的样本已经能提供±3%的精度了。

历史案例：德国坦克问题

第二次世界大战期间，盟军面临一个关键难题：德军到底每个月生产多少辆坦克？这是决定战略部署的重要情报。情报部门通过间谍、审讯战俘、截获通讯等方式给出了各种估算，但不同情报来源之间的矛盾很大。

1941到1942年间，盟军缴获了一些德军坦克——上面有完整的序列号。盟军统计学家意识到，德国人很可能是在坦克出厂时按顺序编号的，因此缴获坦克的序列号就是来自{1, 2, ..., N}的一个简单随机样本（假设缴获过程近似随机），其中N就是德军的坦克总产量。

基于这个思路，统计学家给出了一个至今仍被津津乐道的估计量。设缴获的n辆坦克序列号为 x₁, x₂, ..., xₙ，其中的最大值为M，则坦克总数N的估计为：

N̂ = M + (M - n) / n = M × (1 + 1/n) - 1

这个估计量的直觉是这样的：如果坦克是从1到N按顺序编号的，那么缴获坦克中的最大值M应该"接近"N，但通常会略小一些。平均而言，M与N之间的差距大约为 (N - M) / n，因此我们在这个差距上做一个修正。

通过这种方法——后来被称为"德国坦克问题"（German Tank Problem）——统计学家估计出1942年德军每月生产约246辆坦克，与实际生产数据（约245辆）惊人地接近。而同期情报部门通过传统手段估算的数字是每月约1400辆，高估了近6倍。

这个案例清晰展示了概率抽样的力量：仅凭缴获的少量样本，加上正确的推导，就可以得到比大量情报汇总更准确的估计。

def estimate_tank_total(serial_numbers):
    """
    德国坦克问题：基于序列号估计总数N
    serial_numbers: 缴获坦克的序列号列表
    """
    n = len(serial_numbers)
    M = max(serial_numbers)
    return M + (M - n) / n - 1

# 假设缴获了5辆坦克，序列号为[42, 78, 103, 157, 201]
captured = [42, 78, 103, 157, 201]
est = estimate_tank_total(captured)
print(f"估计坦克总数: {est:.0f}")
# 真实值如果为N=250左右，这个估计相当不错
重要前提：德国坦克问题的方法之所以有效，依赖于一个关键假设——序列号是连续的且缴获过程近似于随机抽样。如果德国人不是顺序编号（比如用了随机编号），或者缴获的坦克不是随机样本（比如总是缴获早期型号），这个方法就会失效。这提醒我们：任何统计推断的有效性都建立在前提假设之上——假设错了，结论就不可靠。我见过不少人在数据分析中盲目套用公式，从不检查前提条件，这是非常危险的。

SRS的优缺点

在正式进入更复杂的方法之前，有必要直面SRS的局限性：

优点：

  • 原理简单、操作直观、容易理解
  • 无偏估计，数学性质好
  • 方差估计有明确的公式
  • 不需要任何辅助信息

缺点：

  • 需要完整的抽样框——当总体很大或没有现成名录时难以实现
  • 如果总体高度异质，SRS的方差较大——需要较大的样本量才能达到所需精度
  • 如果总体在地理上分散，SRS的实地调查成本极高（比如全国范围抽1000户，可能分布在800个不同的村庄）
  • 对子总体（Subpopulation）的估计精度差——如果你还想分析男性vs女性的差异，SRS对每个子总体的样本量无法控制

正是为了克服这些缺点，统计学家们开发了分层抽样、整群抽样、系统抽样等方法。在接下来的几章中，我们将逐一学习这些方法，并看看它们各自在什么场景下比SRS更高效。

本章小结

简单随机抽样是所有概率抽样的基石。我们学习了：有放回和不放回两种方式的区别、FPC因子的作用、均值和比例的估计公式、样本量计算、以及"德国坦克问题"这个经典应用。SRS虽然在实际大规模调查中很少独立使用，但它是理解其他一切抽样方法的起点。
