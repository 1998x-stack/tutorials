第24章 图神经网络与调度

大家好，我是你们的讲师。上一章我们讨论了强化学习调度，核心是把调度问题建模为 MDP，然后训练一个智能体来做出序贯决策。但你有没有想过一个问题：我们用全连接网络或 CNN 来处理调度状态，真的合理吗？调度问题的状态天然是图结构——工件之间有工艺顺序约束，机器之间有加工依赖关系，工件和机器之间又有分配关系。强制把这么丰富的结构信息拍扁成一个固定长度的向量，其实是信息损失很大的操作。

这一章我们介绍的工具，正是为了解决这个问题而生的——图神经网络（Graph Neural Network, GNN）。GNN 直接在图结构数据上操作，通过消息传递（Message Passing）机制来聚合邻居节点的信息，生成节点级别的嵌入表示。2017 年之后，GNN 在分子性质预测、推荐系统、交通流量预测等领域大放异彩；而在调度领域，GNN 的出现让"端到端的调度学习"真正变得可行——你不再需要手工设计状态特征，GNN 可以从调度实例的图结构中自动提取有用信息。

调度实例天然是图 — 析取图、工序图

在调度理论中，析取图（Disjunctive Graph）是描述 Job Shop 调度问题的标准工具。让我们从图的角度重新理解调度问题。

一个 Job Shop 实例可以表示为一个有向图 G = (V, C ∪ D)，其中：

  • V 是节点集合，每个节点代表一道工序（operation）。加上两个虚拟节点：源点（source）和汇点（sink），分别表示开始和结束。
  • C 是合取弧（conjunctive arcs），连接同一工件内的连续工序，方向固定，表示工艺顺序约束——比如工件的第 2 道工序必须在第 1 道工序完成后才能开始。
  • D 是析取弧（disjunctive arcs），连接在同一台机器上加工的不同工序，方向未定——调度决策本质上就是为每条析取弧确定一个方向。

换句话说，求解 Job Shop 调度问题 = 为析取弧定向，使得得到的无环有向图中从源点到汇点的最长路径（即 makespan）最短。这个视角把调度问题变成了一个图上的优化问题——也就自然地为 GNN 的入场铺平了道路。

类似地，Flow Shop 可以看作析取弧结构更规整的特例；并行机问题的图结构更简单——每个工件只有一个节点，机器之间通过一个共享资源节点连接。但在所有这些变体中，图结构天然编码了调度问题的约束信息，这是 GNN 能够起作用的核心原因。

核心认知：传统方法需要人类专家将图结构信息手工编码到特征向量中（如"工件剩余工序数"、"机器负载率"等）。而 GNN 直接在原始图上操作，让模型自己学习哪些结构特征对决策重要。你可以把 GNN 理解为自动特征工程师——它从图结构中提取的信息，往往比手工设计的特征更丰富、更有效。

GNN基础：消息传递与图嵌入

GNN 的核心机制是消息传递（Message Passing）。每一层中，每个节点会收到来自其邻居节点的信息，聚合这些信息后更新自己的嵌入表示。经过若干层之后，每个节点的表示就包含了 k 跳邻域的结构信息。

Graph Convolution, Graph Attention, Graph Isomorphism Network

调度问题中最常用的三种 GNN 架构：

  • Graph Convolutional Network（GCN）—— 最简单的消息传递方式，对邻居消息取均值（或加权均值），然后通过一个线性变换+非线性激活更新节点表示。GCN 计算高效，但在调度问题中表达能力有限，特别是对图同构的区分能力不足。
  • Graph Attention Network（GAT）—— 引入注意力机制，让模型学到的不是固定的平均权重，而是根据不同节点对之间的相关性动态计算权重。在调度中，GAT 可以学会"关注那些在关键路径上的节点"，效果通常优于 GCN。
  • Graph Isomorphism Network（GIN）—— 理论上表达能力最强的 GNN 架构之一，其消息聚合函数使用 MLP + sum 的组合，可以达到 Weisfeiler-Lehman 图同构测试的表达能力。对于需要精细区分不同调度实例的任务，GIN 往往是更好的选择。

这三种架构在调度场景中的选择，本质上是一个表达能力与计算效率的权衡。GCN 最快但表达能力最弱，GIN 表达能力最强但训练更慢，GAT 是折中方案。根据我们的经验，对于大多数调度问题，GAT 提供了最好的性价比。

节点嵌入到策略输出的端到端架构

GNN 在调度中的标准用法是：输入调度实例的图表示（节点特征 + 边特征），经过若干层消息传递后，得到每个节点的嵌入向量。然后，这些嵌入向量被送入一个决策头部（decision head）——通常是一个 MLP 或 attention 层——输出每个动作的价值或概率。

一个典型的端到端调度模型架构如下：

# 端到端 GNN 调度模型架构（伪代码）
class GNNScheduler(nn.Module):
    def __init__(self, node_dim, hidden_dim, num_layers=3):
        super().__init__()
        self.layers = nn.ModuleList([
            GATConv(node_dim if i == 0 else hidden_dim, hidden_dim)
            for i in range(num_layers)
        ])
        self.encoder = nn.Linear(node_dim, hidden_dim)  # 初始嵌入
        self.decoder = nn.Linear(hidden_dim, 1)         # 输出优先级分数

    def forward(self, x, edge_index):
        # x: 节点特征矩阵 [num_nodes, node_dim]
        # edge_index: 边索引 [2, num_edges]
        h = self.encoder(x)
        for conv in self.layers:
            h = conv(h, edge_index)
            h = F.relu(h)
        # 输出每个工序的优先级分数
        priorities = self.decoder(h).squeeze(-1)
        return priorities
        # 在实际调度中，按优先级从高到低选择下一个加工的工序

这个架构的实现非常简洁——不到 20 行代码就定义了一个完整的调度策略网络。当然，在实际训练中你还需要将其嵌入到 RL 的训练循环中，并处理各种调度约束。

GNN在调度中的应用框架

有了 GNN 作为特征提取器，学术界已经提出了多种将 GNN 与调度决策结合的方式。

Graph2Sequence：GNN提取特征→Decoder生成调度序列

Graph2Sequence 框架借鉴了 Seq2Seq 的思想。GNN 作为 Encoder 将调度实例的图结构编码为节点嵌入，然后一个解码器逐步输出调度序列——每一步决定下一个调度的工序是什么。这个框架的优势是端到端学习，不需要手工设计的决策规则；缺点是解码过程是自回归的，推理速度较慢，且误差会逐级累积。

端到端策略映射：输入图→输出优先值→贪心构造

这是目前工业界更常用的框架。GNN 网络接收当前调度状态（部分调度的析取图），输出每个待调度工序的优先级分数，然后使用贪心策略选择分数最高的工序进行加工。每调度一步后，环境更新状态，GNN 重新计算优先级分数——如此循环直到所有工序调度完毕。

这种框架的优点是简单、高效，而且可以与 RL 训练无缝配合——奖励信号直接通过策略梯度反传到 GNN 的参数中。这也是目前主流论文中采用最多的方案。

前沿动态：2024-2025 年，GNN+调度的一个重要突破来自隐式可行约束学习。传统方法需要将所有约束显式编码到算法中，但某些约束在数学上难以形式化（如操作工的偏好、上下游缓冲区的隐性规则）。研究者发现，通过在 GNN 之上堆叠决策树和回归模型，可以在训练数据中隐式学习这些约束，生成的调度方案在可行性上显著优于纯 GNN 方案。2026 年，针对 FJSP（柔性作业车间调度）的 GNN 方法已经能在 50×20 规模上达到 MILP 求解器 1 小时求解质量的平均 95%——但推理时间仅需 0.2 秒。

下面我们用 PyTorch Geometric 构建一个简化版的调度析取图处理示例：

import torch
import torch.nn.functional as F
from torch_geometric.data import Data
from torch_geometric.nn import GATConv

# 构建一个简单的调度析取图
# 假设我们有 3 个工件，每个 2 道工序，共 6 个节点
num_nodes = 6
# 节点特征：每道工序的加工时间 + 机器 ID
node_features = torch.tensor([
    [5.0, 0],   # Job 0, Op 0, Machine 0
    [3.0, 1],   # Job 0, Op 1, Machine 1
    [4.0, 1],   # Job 1, Op 0, Machine 1
    [6.0, 0],   # Job 1, Op 1, Machine 0
    [2.0, 0],   # Job 2, Op 0, Machine 0
    [7.0, 1],   # Job 2, Op 1, Machine 1
], dtype=torch.float)

# 合取弧（conjunctive edges）：同一工件内工序的先后顺序
# (0→1), (2→3), (4→5)
conjunctive = torch.tensor([[0, 2, 4], [1, 3, 5]], dtype=torch.long)

# 析取弧（disjunctive edges）：同一机器上不同工序之间的连接
# 机器 0 上的工序：0, 3, 4 之间全连接
# 机器 1 上的工序：1, 2, 5 之间全连接
# 这里为了简化只列出部分边
disjunctive = torch.tensor([
    [0, 0, 3, 3, 4, 4],
    [3, 4, 0, 4, 0, 3]
], dtype=torch.long)

edge_index = torch.cat([conjunctive, disjunctive], dim=1)

# 构建 PyG Data 对象
data = Data(x=node_features, edge_index=edge_index)

# 定义两层的 GAT 模型
class GATScheduler(torch.nn.Module):
    def __init__(self, in_dim=2, hidden_dim=32, out_dim=16):
        super().__init__()
        self.conv1 = GATConv(in_dim, hidden_dim, heads=4)
        self.conv2 = GATConv(hidden_dim * 4, out_dim, heads=1)
        self.out = torch.nn.Linear(out_dim, 1)

    def forward(self, data):
        x, edge_index = data.x, data.edge_index
        x = self.conv1(x, edge_index)
        x = F.elu(x)
        x = self.conv2(x, edge_index)
        x = F.elu(x)
        # 输出每个工序的优先级分数
        priority = self.out(x).squeeze(-1)
        return F.softmax(priority, dim=0)

model = GATScheduler()
priority_scores = model(data)
print("工序优先级分数:", priority_scores)
# 分数最高的工序将优先被调度

这段代码虽然简单，但它包含了 GNN 调度模型的核心骨架。在实际项目中，你需要加入编码器来处理异构节点类型、动态更新图结构（每调度一步就从图中移除已完成的工序）、以及在训练时与 RL 环境对接。

下面是 GNN 与调度问题的映射关系表，帮助你快速理解不同调度问题适合哪种 GNN 方案：

调度问题 图结构 节点含义 边类型 推荐GNN
单机调度 无向完全图（工件间全连接） 每个工件 相似度边 GAT
Flow Shop 有向无环图（DAG） 每道工序 合取弧（固定方向） GCN / GAT
Job Shop 析取图 每道工序 合取弧 + 析取弧 GAT（异构边类型）
FJSP（柔性作业车间） 二分图（工件-机器） 工序节点 + 机器节点 工件→机器（可选分配） Heterogeneous GNN
并行机调度 完全二分图 工件 + 机器 工件-机器分配 GAT / GIN
RCPSP（资源约束项目调度） 有向图 + 资源二分图 任务 + 资源 前置依赖 + 资源需求 Heterogeneous GNN

2025-2026前沿

在 2025-2026 年间，GNN 在调度领域的发展可以用"从能用到好用"来形容。几个值得关注的方向：

学习隐式可行约束。传统 GNN 调度器只知道显式编码的约束（工艺顺序、机器容量等），但实际生产中有大量"潜规则"——比如"尽量不要在夜班安排需要质检员配合的工序"、"某台旧机床加工精密度高的工件时需要额外冷却时间"。这些知识很难手工编码，但可以通过历史数据让 GNN 隐式学习。2025 年的一篇 ICML 论文提出了 GNN+决策树+回归的堆叠模型，在隐式约束学习上取得了显著进步。

FJSP 上的突破。柔性作业车间调度（FJSP）因为同时需要分配机器和排序工序，一直被认为是调度领域最困难的问题之一。2026 年初，多个团队独立提出了基于 Heterogeneous GNN 的 FJSP 求解方案，在公共基准上超越了所有传统元启发式方法，GAP 从平均 8% 降到了 3% 以内。

并行机和 RCPSP 上的推广。最开始 GNN 调度主要聚焦在 Job Shop 上，但 2025 年以来，GNN 方法成功推广到了并行机调度和资源约束项目调度（RCPSP）。特别是 2026 年 DRL+GNN 结合的方法在 RCPSP 上达到了与 CP-SAT 相当的解质量，但推理时间缩短了 100 倍以上。

GNN 泛化性的重要进展：长期以来，GNN 调度方法最大的痛点是"训练在小实例，部署在大实例"时性能断崖式下降。2026 年，这一难题终于取得了重要突破。通过结合元学习（MAML）和可扩展注意力机制（Scalable Attention），研究者在 20 节点的小实例上训练的 GNN 调度器，在 200 节点的大实例上仍然保持了 93% 以上的性能——这对工业落地意义重大，因为工厂实际问题的规模可能在训练数据的 10 倍以上。
