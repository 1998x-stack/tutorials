第16章 风险管理体系

大家好，我是你们的讲师。上一章我们讲了现代投资组合理论，学了如何用数学方法构建"最优组合"。但有一个问题被很多人忽略了：你的组合在95%的时间里可能运行得很好，但那5%的极端行情，可能会让你失去一切。

投资圈有一句经典的话："你赚到的每一分钱，都是你风险管理能力的变现。" 听起来不像巴菲特的名言那么激动人心，但这是实话。在A股市场，一次股灾就能抹掉你过去三五年的全部收益。2015年股灾、2016年熔断、2018年单边下跌、2024年初的流动性危机——每一轮极端行情都在提醒我们：不掌握风险管理就冲进市场，跟不带降落伞跳伞没什么区别。这一章我们就来系统性地建立自己的风险管理工具箱。

16.1 风险不是亏损，是不确定性

系统性风险 vs 非系统性风险

金融学把风险分为两大类：

系统性风险（Systematic Risk） —— 也叫市场风险或不可分散风险。它是整个市场共同面临的风险，比如经济衰退、利率变化、地缘政治冲突。这类风险无法通过分散化消除，你只能通过调整仓位来管理它。

非系统性风险（Idiosyncratic Risk） —— 也叫个体风险或可分散风险。它是某只股票或某个行业特有的风险，比如公司管理层丑闻、产品安全事故、行业监管变化。这类风险可以通过分散化大幅降低——持有一个足够分散的组合后，一只股票的坏消息对整体的冲击会小很多。

风险管理的目标不是消除风险

这是最关键的一个认知：风险管理的目标不是消除风险，而是把你暴露在"你愿意承担且有相应回报"的风险中。 如果你完全消除了风险，那你也同时消除了收益。投资本质上是在选择——你选择承担什么样的风险，来换取什么样的回报。风险管理要做的是：确保你承担的风险是你理解且有能力承受的，而不是盲目地承担自己一无所知的风险。

认知：真正的风险不是波动本身，而是你在最不该卖出的时候被迫卖出。如果你能在市场暴跌时保持仓位不动甚至加仓，那波动对你来说就不是风险，而是机会。风险管理体系的终极目标，就是让你在任何市场环境下都能"活得下去"。

16.2 VaR与CVaR

VaR（Value at Risk）

VaR是风险管理中最常用的指标，它回答的问题很简单："在给定置信水平下，你的组合在未来一段时间内最多可能亏多少钱？"

比如一个组合的日度95% VaR是100万元，意思是：在正常的市场条件下，有95%的概率，你一天的最大亏损不会超过100万元。或者说，每20个交易日里，大约有1天的亏损会超过100万元。

VaR的三种计算方法

1. 历史模拟法：直接使用过去N天的历史收益率排序，取第5%分位的收益率。优点是不需要对收益分布做任何假设，缺点是如果过去没有发生过极端事件，VaR就会低估风险。

2. 参数法（方差-协方差法）：假设收益率服从正态分布，VaR = μ - z×σ。优点是计算简单，缺点是金融数据通常有"厚尾"特征（极端值比正态分布更频繁），导致VaR低估尾部风险。

3. 蒙特卡洛模拟：对收益率分布进行随机抽样模拟，生成成千上万种可能的情景，再取分位数。优点是灵活，可以加入各种假设；缺点是计算量大，而且结果依赖输入假设的合理性。

CVaR（Conditional VaR）

VaR有一个致命缺陷：它只告诉你"在95%的情况下最多亏多少"，但不告诉你"一旦超过这个门槛，平均会亏多少"。CVaR（也叫Expected Shortfall）弥补了这个缺陷——它计算的是超越VaR门槛之后的平均损失。

举个例子：某组合95% VaR是100万，CVaR是150万。意思是：在最差的5%情景中，平均每笔损失是150万。CVaR比VaR更能反映尾部风险，在巴塞尔协议III之后的银行业风险管理中，CVaR已经取代VaR成为核心指标。

VaR与CVaR计算示例（Python风格伪代码）：

        import numpy as np

        # 假设我们有1000个交易日的历史收益率
        returns = np.array([...])  # 日收益率数组

        # 历史模拟法计算95% VaR
        var_95 = np.percentile(returns, 5)  # 第5%分位数的收益率
        # 如果var_95 = -2.3%，意味着95% VaR = 2.3%

        # 计算CVaR（95%）
        cvar_95 = returns[returns <= var_95].mean()
        # 取所有低于VaR的收益率的平均值

        # 参数法（假设正态分布）
        mu = returns.mean()
        sigma = returns.std()
        var_95_param = mu - 1.645 * sigma  # 95%置信度的z值=1.645
        var_99_param = mu - 2.326 * sigma  # 99%置信度的z值=2.326

        # 组合VaR计算（两只股票）
        # VaR_p = sqrt(VaR1^2 + VaR2^2 + 2*rho*VaR1*VaR2)
        var_1 = 100000  # 股票1的VaR
        var_2 = 80000   # 股票2的VaR
        rho = 0.5       # 相关系数
        var_p = np.sqrt(var_1**2 + var_2**2 + 2*rho*var_1*var_2)

VaR的局限

VaR最大的问题是尾部风险。正态分布假设下，-4σ以上的事件几乎是"不可能"的（概率约0.003%）。但在真实的金融市场中，"黑天鹅"事件的频率远高于正态分布的预测。仅2008年全球金融危机、2015年A股股灾、2020年新冠暴跌、2024年初A股流动性危机——这些"几个sigma的事件"每隔几年就来一次。单纯依靠VaR做风控，会在你最需要它的时候失效。

16.3 最大回撤：投资者最痛的数字

如果说VaR是一个"冷冰冰的数字"，那最大回撤（Maximum Drawdown, MDD）就是每个投资者心里最真实的痛感。最大回撤衡量的是：从历史最高点跌下来的最大幅度。

为什么回撤如此重要？因为亏损具有不对称性——亏50%之后需要涨100%才能回本。回撤越大，回本需要的涨幅就以几何级数上升：

回撤幅度 回本所需涨幅 体会
-10% +11.1% 还能接受
-20% +25.0% 有点难受了
-30% +42.9% 开始怀疑人生
-40% +66.7% 非常绝望
-50% +100.0% 想回本？翻倍再说
-60% +150.0% 基本告别回本
-80% +400.0% 已卒

各风险指标的综合对比：

风险指标 衡量什么 优点 缺点
波动率（σ） 收益率的离散程度 标准指标，计算简单 对上行和下行波动一视同仁
VaR（95%） 最大可能损失（给定概率） 直观易懂，行业标准 低估尾部风险，不告知极端损失
CVaR 尾部损失均值 比VaR更稳健，关注极端情景 计算复杂，需要更多数据
最大回撤 历史最高点到最低点的跌幅 最贴近投资者的真实体验 向后看，不代表未来
下行波动率 只考虑负收益的波动 更聚焦风险本质 不如波动率通用

16.4 仓位管理三原则

凯利公式

仓位管理是风险管理中最具实操性的一环。给多少仓位出手，直接决定了你的长期收益率曲线。这方面最经典的工具是凯利公式（Kelly Criterion），它在赌博场景中被发明，后被引入投资领域。

凯利公式：f* = (bp - q) / b

        其中：
        f* = 最优投资比例（占总资金的比例）
        b = 赔率（盈亏比）
        p = 获胜概率
        q = 失败概率 = 1 - p

        例1：某交易策略胜率60%(p=0.6)，盈亏比2:1(b=2)
        f* = (2x0.6 - 0.4) / 2 = 0.8/2 = 0.4
        -> 最优仓位为总资金的40%

        例2：胜率55%，盈亏比1.5:1
        f* = (1.5x0.55 - 0.45) / 1.5 = 0.375/1.5 = 0.25
        -> 最优仓位为总资金的25%

        注意：凯利公式非常敏感，实践中通常使用"半凯利"
        （将f*除以2）来降低波动，保留安全边际。

凯利公式告诉我们一个重要原则：胜率和赔率共同决定了最合适的仓位。 高胜率高赔率的交易可以重仓，但现实中这样的机会很少。对于大多数不太确定的投资机会，凯利公式给出的仓位建议比你的直觉要小得多。

等权重 vs 波动率倒数加权

构建组合时，最常见的两种仓位分配方式是：

  • 等权重（Equal Weight）：每只股票分配相同比例的仓位。简单、不用预测、自动平衡（卖掉涨多的买入跌多的）。实证研究表明，1/N等权重组合在大多数情况下跑赢了复杂的优化模型。
  • 波动率倒数加权（Volatility Inverse Weighting）：低波动的股票配更多仓位，高波动的配更少。核心思想是"风险预算"——每只股票对组合贡献的风险应该大致相等。对于波动率差异很大的资产（比如股票+债券），这种方法效果很好。

集中 vs 分散的权衡

仓位管理的另一个维度是集中与分散。巴菲特说"分散化是对无知的保护"，但这不等于你应该全仓一只股票。关键是要找到一个平衡点：

  • 如果你对某只股票有极强的信心和深入的研究，可以给较高仓位（比如10%-20%）
  • 如果你只是"感觉不错"，那单只股票的仓位最好不要超过5%
  • 即使最看好的股票，单一仓位不超过总资金的25%——这是很多成熟投资者的铁律

实用原则：以凯利公式为锚，以最大单票仓位上限为约束，以波动率倒数加权为分配框架，构建一个既不过度保守也不过度激进的仓位管理体系。记住：仓位的目标是让亏损可控，而不是让收益最大化。

16.5 止损策略设计

止损的四种类型

1. 固定比例止损：买入时设定一个固定跌幅作为止损线，比如-10%或-15%。最简单、最机械、但也是最有效的方法。关键在于——设了就一定要执行，不能因为"感觉要涨了"就撤销止损。

2. 移动止损（Trailing Stop）：随着股价上涨上调止损位。比如设定20%的移动止损：股价涨到100时止损位是80；涨到120时止损位变成96；跌到110时止损位依然是96。这样可以保护既得利润，同时给股价波动留出空间。

3. 时间止损：如果买入后一定时间内没有达到预期，就离场。比如"买入后2周内如果涨幅不超过5%就卖出"。时间止损的逻辑是：机会成本也是成本。如果市场证明了你的判断没有兑现，那就认错走人。

4. 基本面止损：不是根据价格止损，而是根据基本面变化止损。比如你买入的逻辑是公司新产品放量，但后来发现新品销量远不及预期——这时候不管股价涨跌都应该卖出。基本面止损是最难的，因为它需要持续跟踪和独立判断。

止损的纪律比点位更重要

止损策略最大的敌人不是"设错了点位"，而是"设了却不执行"。心理学研究发现，投资者在面对浮亏时，大脑中与恐惧相关的区域被激活，导致他们倾向于"再等等"——结果越等越深。止损不是一种技术，它是一种纪律。

踩坑故事：不止损的灾难性后果

2015年，我有一个朋友在杠杆牛市的高点入市。他买入了一只创业板股票，成本价约80元。股价跌到70时，他想"这只是回调"；跌到60时，他想"已经跌了25%了，不可能再跌了"，不仅不止损，反而借钱加仓摊低成本；跌到40时，券商开始催缴保证金，他不想割肉，又从信用卡套现追加保证金。

最终的结果呢？连续跌停后，他的账户被强制平仓。80万本金加40万杠杆，最终只拿回来不到5万。更让人心痛的是——那只股票后来确实涨回来了，但跟他已经没有关系了。因为他不是在80元买入的，他是在80、60、40、20元一路"抄底"买入的，然后被杠杆和强制平仓彻底摧毁了。

这个故事的教训很深刻：不止损是你亏钱的开始，加杠杆死扛是你爆仓的原因。 如果他在-15%的时候止损出局，后面所有的痛苦都不会发生。每一次你没有执行止损，都在为下一次更大的亏损埋下伏笔。

小结与预习

这一章我们完整梳理了风险管理体系：从系统性风险和非系统性风险的区分，到VaR/CVaR的计算方法，到最大回撤的惨痛现实，再到凯利公式指导下的仓位管理，最后讨论了四种止损策略。核心就一句话：投资的第一要务不是赚钱，是活着。

但你可能已经注意到一个有趣的现象——很多风险管理上的"应该做"和投资者的"实际做"之间存在着巨大的差距。我们明明知道应该止损，却老是扛单；明明知道应该分散，却总想全仓押注。这是为什么？下一章我们就进入行为金融学，探讨投资者心理中那些非理性的角落——你的大脑是如何在这条投资之路上"坑"你的。
