为什么用Python做多元分析

大家好，欢迎继续学习！上一章我们用R把五大多元分析方法完整地跑了一遍。这一章轮到Python了。如果你问我：R和Python做统计，到底选哪个？我的回答是——两个都要会。不是和稀泥，而是这两个语言在数据科学生态中的定位确实不同：R是统计学家写的，Python是工程师写的。R的语法更贴近统计表达（比如用 ~ 表示公式），Python的API更统一（fit/predict/transform三部曲）。在实际工作中，你很可能上午用R做探索性数据分析，下午用Python把模型部署到生产环境。

Python做多元分析的核心生态由四个库组成：scikit-learn（通用机器学习，包含PCA、KMeans、LDA）、statsmodels（统计推断，包含MANOVA、因子分析）、factor_analyzer（专门做因子分析，由Educational Testing Service的团队维护）、matplotlib/seaborn（可视化）。下面我们逐一上手。

库名用途核心函数/类特点
scikit-learn通用ML & 多元分析PCA, KMeans, LDA, CCA统一API，文档最全
statsmodels统计推断MANOVA, factor analysis输出p值、置信区间等统计量
factor_analyzer因子分析专用FactorAnalyzerKMO检验、Bartlett检验、多种旋转
matplotlib/seaborn可视化pairplot, heatmap, clustermap与pandas无缝集成

PCA in Python

scikit-learn的 PCA 类是对R的 prcomp() 的Python等价物，底层同样使用SVD分解。用法非常简洁：

import numpy as np
import pandas as pd
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt
import seaborn as sns

# 加载数据：用seaborn自带的iris数据集
df = sns.load_dataset("iris")
X = df.iloc[:, :4].values
y = df.iloc[:, 4].values

# 标准化（PCA对量纲敏感）
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# PCA降维
pca = PCA()
X_pca = pca.fit_transform(X_scaled)

# 解释方差比例
print("Explained variance ratio:", pca.explained_variance_ratio_)
# [0.7296 0.2285 0.0367 0.0052]

# 累计解释方差
print("Cumulative:", np.cumsum(pca.explained_variance_ratio_))
# [0.7296 0.9581 0.9948 1.0000]

# 碎石图
plt.figure(figsize=(8, 5))
plt.bar(range(1, 5), pca.explained_variance_ratio_)
plt.step(range(1, 5), np.cumsum(pca.explained_variance_ratio_), where='mid')
plt.xlabel('Principal Component')
plt.ylabel('Explained Variance Ratio')
plt.show()

# 载荷矩阵
loadings = pd.DataFrame(
    pca.components_.T,
    columns=['PC1', 'PC2', 'PC3', 'PC4'],
    index=df.columns[:4]
)
print(loadings)

# biplot
def biplot(score, coeff, labels=None):
    xs = score[:, 0]
    ys = score[:, 1]
    n = coeff.shape[0]
    plt.scatter(xs, ys, c=y.astype('category').cat.codes, alpha=0.6)
    for i in range(n):
        plt.arrow(0, 0, coeff[i, 0]*3, coeff[i, 1]*3,
                  color='r', alpha=0.8)
        if labels is None:
            plt.text(coeff[i, 0]*3.2, coeff[i, 1]*3.2,
                     f"Var{i+1}", color='r')
        else:
            plt.text(coeff[i, 0]*3.2, coeff[i, 1]*3.2,
                     labels[i], color='r')
    plt.xlabel("PC1")
    plt.ylabel("PC2")
    plt.grid(alpha=0.3)

biplot(X_pca, pca.components_.T, labels=df.columns[:4])
plt.show()

这里有一个很容易被忽略的细节：PCA() 默认不标准化数据（不像R的 prcomp(scale.=TRUE)），你必须手动调用 StandardScaler。初学者最容易犯的错误就是直接把原始数据丢进PCA，然后发现第一主成分完全由量纲最大的变量主导——我们在第1章踩过的坑，这里又出现了。

最佳实践：在scikit-learn中，永远记得做三步：StandardScaler().fit_transform(X) → PCA().fit_transform(X_scaled)。你也可以用 Pipeline 把这两步串起来，避免遗漏。

因子分析 in Python

Python的标准库中没有一个像R的 psych 那样功能全面的因子分析包。最接近的是 factor_analyzer 包，由ETS（美国教育考试服务中心）的团队维护。它提供了KMO检验、Bartlett球形检验、多种旋转方法和因子得分计算。

# pip install factor_analyzer
from factor_analyzer import FactorAnalyzer
from factor_analyzer.factor_analyzer import calculate_kmo, calculate_bartlett_sphericity

# 计算KMO值
kmo_all, kmo_model = calculate_kmo(X_scaled)
print(f"KMO: {kmo_model:.3f}")

# Bartlett球形检验
chi2, p_value = calculate_bartlett_sphericity(X_scaled)
print(f"Bartlett chi2: {chi2:.2f}, p-value: {p_value:.4f}")

# 因子分析：提取2个因子，使用方差最大旋转
fa = FactorAnalyzer(n_factors=2, rotation="varimax")
fa.fit(X_scaled)

# 因子载荷
loadings = fa.loadings_
loadings_df = pd.DataFrame(
    loadings,
    index=df.columns[:4],
    columns=['Factor1', 'Factor2']
)
print(loadings_df)

# 共同度（communality）
communality = fa.get_communalities()
print("Communalities:", communality)

# 因子得分
scores = fa.transform(X_scaled)
print(scores[:5, :])

factor_analyzer包与R的 psych::fa() 的一个重要区别是：它默认使用主成分法（principal factor method）而不是最大似然法（ML）。如果你需要ML估计，需要设置参数 method="ml"。另外，factor_analyzer不自动计算标准误和置信区间——如果需要做严格的统计推断（比如报告因子载荷的标准误），建议还是回到R。

聚类分析 in Python

scikit-learn的聚类模块非常全面，涵盖了KMeans、层次聚类、DBSCAN、Gaussian Mixture Models等十几种算法。API高度统一：fit_predict() 一步到位。

from sklearn.cluster import KMeans, AgglomerativeClustering, DBSCAN
from sklearn.metrics import silhouette_score
import scipy.cluster.hierarchy as sch

# === K-means ===
kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
kmeans_labels = kmeans.fit_predict(X_scaled)

# 轮廓系数
sil = silhouette_score(X_scaled, kmeans_labels)
print(f"Silhouette Score: {sil:.3f}")

# 肘部法则找最佳K
inertias = []
silhouettes = []
K_range = range(2, 9)
for k in K_range:
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X_scaled)
    inertias.append(km.inertia_)
    silhouettes.append(silhouette_score(X_scaled, km.labels_))

plt.figure(figsize=(12, 4))
plt.subplot(1, 2, 1)
plt.plot(K_range, inertias, 'bo-')
plt.xlabel('K')
plt.ylabel('Inertia')
plt.title('Elbow Method')

plt.subplot(1, 2, 2)
plt.plot(K_range, silhouettes, 'ro-')
plt.xlabel('K')
plt.ylabel('Silhouette Score')
plt.title('Silhouette Analysis')
plt.show()

# === 层次聚类 ===
Z = sch.linkage(X_scaled, method='ward')
plt.figure(figsize=(10, 6))
sch.dendrogram(Z, labels=y.tolist(), leaf_rotation=90)
plt.title("Hierarchical Clustering Dendrogram")
plt.show()

# === DBSCAN（基于密度的聚类，不需要预设簇数） ===
dbscan = DBSCAN(eps=0.5, min_samples=5)
dbscan_labels = dbscan.fit_predict(X_scaled)
print(f"DBSCAN found {len(set(dbscan_labels))} clusters")
# DBSCAN会自动识别噪声点（标签为-1）

# === 可视化聚类结果 ===
from sklearn.decomposition import PCA as PCA_vis
pca_2d = PCA_vis(n_components=2)
X_2d = pca_2d.fit_transform(X_scaled)

plt.figure(figsize=(12, 4))
plt.subplot(1, 3, 1)
plt.scatter(X_2d[:, 0], X_2d[:, 1], c=kmeans_labels, cmap='viridis')
plt.title('K-means')

plt.subplot(1, 3, 2)
plt.scatter(X_2d[:, 0], X_2d[:, 1], c=dbscan_labels, cmap='viridis')
plt.title('DBSCAN')

plt.subplot(1, 3, 3)
plt.scatter(X_2d[:, 0], X_2d[:, 1], c=y.astype('category').cat.codes, cmap='viridis')
plt.title('Ground Truth')
plt.show()

层次聚类的树状图可视化在Python中比R稍微麻烦一点——你需要用 scipy.cluster.hierarchy 的 dendrogram 函数来绘制。不过它的灵活性更高，可以交互式地裁剪树状图的特定分支、给叶子节点着不同的颜色。

判别分析 in Python

scikit-learn的 discriminant_analysis 模块提供了LDA和QDA的完整实现。用法和其他scikit-learn估计器完全一致。

from sklearn.discriminant_analysis import (
    LinearDiscriminantAnalysis,
    QuadraticDiscriminantAnalysis
)
from sklearn.model_selection import cross_val_score, train_test_split
from sklearn.metrics import classification_report, confusion_matrix

# 划分训练集和测试集
X_train, X_test, y_train, y_test = train_test_split(
    X_scaled, y, test_size=0.3, random_state=42
)

# LDA
lda = LinearDiscriminantAnalysis()
lda.fit(X_train, y_train)
y_pred_lda = lda.predict(X_test)

print("LDA Accuracy:", (y_pred_lda == y_test).mean())
print(classification_report(y_test, y_pred_lda))

# LDA的混淆矩阵
cm = confusion_matrix(y_test, y_pred_lda)
sns.heatmap(cm, annot=True, fmt='d', cmap='Blues',
            xticklabels=lda.classes_,
            yticklabels=lda.classes_)
plt.title("LDA Confusion Matrix")
plt.show()

# 5折交叉验证
cv_scores = cross_val_score(lda, X_scaled, y, cv=5)
print(f"LDA CV Accuracy: {cv_scores.mean():.3f} +/- {cv_scores.std():.3f}")

# QDA（二次判别分析，不假设协方差矩阵相等）
qda = QuadraticDiscriminantAnalysis()
qda.fit(X_train, y_train)
y_pred_qda = qda.predict(X_test)
print(f"QDA Accuracy: {(y_pred_qda == y_test).mean():.3f}")

# LDA的降维投影
X_lda = lda.transform(X_scaled)
print(f"LDA降维后维度: {X_lda.shape[1]}")
# 对于3分类问题，最多降维到2维

LDA和QDA的选择取决于你对协方差矩阵的假设：LDA假设所有类别的协方差矩阵相同（共享协方差矩阵），QDA允许每个类别有自己的协方差矩阵。QDA更灵活但需要更多的参数估计（每个类别一个p×p协方差矩阵），在样本量不足时容易过拟合。一个经验法则：如果样本量充足（每个类别的样本数远大于变量数），QDA可能更好；否则，LDA更稳健。

CCA in Python

scikit-learn的 cross_decomposition 模块提供了一个 CCA 类，但它的实现是PLS（偏最小二乘）框架下的CCA，与传统CCA在细节上略有不同。

from sklearn.cross_decomposition import CCA

# 构造两组变量
X = X_scaled[:, :2]  # 花萼相关变量
Y = X_scaled[:, 2:]  # 花瓣相关变量

# CCA
cca = CCA(n_components=2)
cca.fit(X, Y)

# 投影后的典型变量得分
X_c, Y_c = cca.transform(X, Y)

# 计算典型相关系数
correlations = [
    np.corrcoef(X_c[:, i], Y_c[:, i])[0, 1]
    for i in range(2)
]
print("Canonical correlations:", correlations)
# [0.9406, 0.7771] — 和R的CCA::cc()结果一致

# X组和Y组的典型权重
print("X weights:", cca.x_weights_)
print("Y weights:", cca.y_weights_)

# 可视化第一对典型变量
plt.figure(figsize=(6, 6))
plt.scatter(X_c[:, 0], Y_c[:, 0], c=y.astype('category').cat.codes, alpha=0.7)
plt.xlabel('First Canonical Variable (X set)')
plt.ylabel('First Canonical Variable (Y set)')
plt.title(f'CCA: r = {correlations[0]:.3f}')
plt.plot([-4, 4], [-4, 4], 'r--', alpha=0.5)
plt.grid(alpha=0.3)
plt.show()

scikit-learn的CCA不提供显著性检验（不像R的CCP包里的 p.asym()），也不计算冗余指数。如果需要这些，你可以手动计算Wilks' Lambda，或者使用 statsmodels 中的 CanonicalCorrelation（在statsmodels 0.14+版本中引入）。另一种选择是使用 cca_zoo 包——这是一个专门的CCA库，支持稀疏CCA、核CCA、深度CCA等变体。

数据可视化

Python的可视化生态由matplotlib、seaborn和plotly三足鼎立。对于多元分析，seaborn尤其强大：

# 散点图矩阵（pairplot）— 多元分析中最常用的EDA工具
sns.pairplot(df, hue="species", diag_kind="kde")
plt.show()

# 相关系数热力图
plt.figure(figsize=(8, 6))
corr_matrix = df.iloc[:, :4].corr()
sns.heatmap(corr_matrix, annot=True, fmt='.2f',
            cmap='RdBu_r', center=0,
            square=True, linewidths=0.5)
plt.title("Correlation Heatmap")
plt.show()

# 聚类热力图（clustermap）— 同时展示层次聚类和热力图
sns.clustermap(df.iloc[:, :4], z_score=0,
               cmap='vlag', method='ward',
               col_cluster=True, row_cluster=True)
plt.show()

# 箱线图 — 比较不同组在各变量上的分布
fig, axes = plt.subplots(2, 2, figsize=(12, 8))
for idx, col in enumerate(df.columns[:4]):
    sns.boxplot(data=df, x='species', y=col, ax=axes[idx//2, idx%2])
plt.tight_layout()
plt.show()

seaborn的 clustermap 是我们做"数据探索"阶段的瑞士军刀——它同时展示了两件事：行和列的层次聚类树状图（告诉你哪些样本/变量相似）和颜色编码的数值矩阵（告诉你具体的相关强度和方向）。一张图就能对数据的整体结构有个直觉认识。

踩坑故事：sklearn PCA vs factor_analyzer 结果不一致

有一次我在做一个消费者偏好的因子分析项目，先用scikit-learn的PCA做了一次"快速预览"——"嗯，前5个主成分解释了72%的方差，差不多了"。然后我用factor_analyzer提取5个因子，结果发现factor_analyzer报告的"累计方差解释"只有58%。差了14个百分点！我一度怀疑factor_analyzer有bug。

后来仔细想明白了：PCA提取的是主成分，它直接对总方差进行分解，每个主成分都贡献一部分方差；而factor_analyzer提取的是因子，它只分解变量之间的共享方差（communality），把每个变量的独有方差（uniqueness）排除在外了。所以因子分析的总方差解释天然会比PCA低——这不是bug，是两种方法的哲学差异：PCA解释的是全部方差，因子分析只解释共同方差。

教训：永远不要直接比较PCA和因子分析的"方差解释比例"——这两个概念的计算分母不同。PCA的分母是总方差，因子分析的分母是共同方差。如果你看到一篇论文说"PCA解释了80%的方差，因子分析解释了70%"，这不说明PCA更好，两者本来就不是同一个尺度的指标。
