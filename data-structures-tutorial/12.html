从"六度分隔"说起

你听说过"六度分隔（Six Degrees of Separation）"理论吗？这个理论认为，世界上任意两个人之间最多通过六个中间人就能建立起联系。你的朋友的朋友的朋友的朋友的朋友的朋友——可能和地球上任何一个人相连。

这个理论的核心是什么？是图（Graph）。每个人是一个"顶点"（Vertex），人与人之间的认识关系是一条"边"（Edge）。整个社交网络就是一张巨大的图。

图是计算机科学中最具表达能力的数据结构。树实际上是图的一种特例（没有环的连通图），链表是图的一种更特例（每个节点最多只有一个邻居）。当你需要表达任意两个实体之间的任意关系时——用图就对了。

从地图导航（路口=顶点，道路=边）到社交推荐（用户=顶点，关注=边），从网页排名（页面=顶点，超链接=边）到电路设计（元件引脚=顶点，导线=边）——图无处不在。

图的概念

让我们正式定义图以及相关的核心术语：

一个图（Graph） G = (V, E) 由两个集合组成：

  • V（Vertex）：顶点的有限集合。
  • E（Edge）：边的有限集合，每条边是顶点对 (u, v)。
术语 英文 含义
有向图 Directed Graph / Digraph 边有方向，如 (u→v) 和 (v→u) 是两条不同的边。Twitter关注关系是有向图——你关注了对方，对方不一定关注你。
无向图 Undirected Graph 边没有方向，(u, v) 和 (v, u) 是同一条边。Facebook好友关系是无向图——如果A是B的好友，B也一定是A的好友。
带权图 Weighted Graph 每条边有一个权重（cost/length）。地图导航中，边权可以是距离或时间。
度 Degree 无向图中，一个顶点的邻居个数。有向图中分为入度（In-degree）和出度（Out-degree）。
路径 Path 从一个顶点到另一个顶点经过的顶点序列。路径上顶点不重复称为"简单路径"。
环 Cycle 起点和终点相同的路径。没有环的图称为"无环图"（Acyclic Graph）。
连通分量 Connected Component 无向图中，互相可达的最大顶点子集。一个图可以有多个连通分量。
强连通分量 Strongly Connected Component 有向图中，任意两个顶点互相可达的最大子图。
森林 Forest 没有环的图（无向无环图）。一棵树是一个连通的森林。
一个记忆技巧：树是特殊的图——树是"连通的无环图"。如果你学会了图的所有概念和应用，树就自动被你"降维打击"了。反过来看，树中的很多算法（DFS、BFS）可以直接推广到图上。

图的存储方式

在代码中表示图，最常用的两种方式是邻接矩阵和邻接表。

邻接矩阵（Adjacency Matrix）

使用一个 n×n 的二维数组来存储图，其中 matrix[i][j] 表示顶点 i 到顶点 j 是否存在边（或边的权重）。

# 无向图的邻接矩阵
#    0 --- 1
#    |     |
#    3 --- 2
# 顶点数 n = 4
n = 4
matrix = [
    [0, 1, 0, 1],  # 顶点0到0(无),1(有),2(无),3(有)
    [1, 0, 1, 0],  # 顶点1到0(有),1(无),2(有),3(无)
    [0, 1, 0, 1],  # 顶点2...
    [1, 0, 1, 0],  # 顶点3...
]

def has_edge(matrix, i, j):
    return matrix[i][j] != 0  # O(1) 查边

def get_neighbors(matrix, i):
    n = len(matrix)
    neighbors = []
    for j in range(n):
        if matrix[i][j] != 0:
            neighbors.append(j)
    return neighbors  # O(n) 遍历所有顶点

邻接表（Adjacency List）

为每个顶点维护一个列表，存储它的所有邻居。对于稀疏图（边的数量远小于 n²），邻接表能省大量空间。

# 用邻接表表示同一张图
#    0 --- 1
#    |     |
#    3 --- 2
graph = {
    0: [1, 3],
    1: [0, 2],
    2: [1, 3],
    3: [0, 2],
}

def has_edge_adj(graph, i, j):
    return j in graph[i]  # O(degree(i))，最坏 O(n)

def get_neighbors_adj(graph, i):
    return graph[i]  # O(degree(i))

# 有向带权图的邻接表
# 0 ---(5)--> 1
# 0 ---(3)--> 2
# 1 ---(2)--> 2
weighted_graph = {
    0: [(1, 5), (2, 3)],
    1: [(2, 2)],
    2: [],
}

对比总结

特性 邻接矩阵 邻接表
空间复杂度 O(n²) O(n + m)（m 为边的数量）
查边 (u, v) O(1) O(degree(u))
遍历所有邻居 O(n) O(degree(u))
添加边 O(1) O(1)
删除边 O(1) O(degree(u))
适合场景 稠密图（m ≈ n²） 稀疏图（m << n²）
算法偏好 Floyd-Warshall（需要O(1)查边） DFS/BFS/Dijkstra（需要遍历邻居）

在面试和实际工程中，邻接表是绝对的主流——绝大多数图是稀疏的。当你写 graph[u].append(v) 时，你使用的就是邻接表。

DFS遍历

深度优先搜索（Depth-First Search, DFS）是图最基础的遍历方式之一。它的思想是"一条道走到黑"——从起点出发，沿着一个方向一直走，走到尽头再回溯。

DFS可以用递归实现，也可以用显式栈实现。我们来看两种写法：

def dfs_recursive(graph, node, visited=None):
    """递归DFS遍历"""
    if visited is None:
        visited = set()
    visited.add(node)
    print(node, end=' ')  # 访问节点
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited)
    return visited

def dfs_iterative(graph, start):
    """栈实现DFS遍历"""
    visited = set()
    stack = [start]
    while stack:
        node = stack.pop()
        if node not in visited:
            visited.add(node)
            print(node, end=' ')
            # 逆序入栈，保持和递归一致的访问顺序
            for neighbor in reversed(graph[node]):
                if neighbor not in visited:
                    stack.append(neighbor)
    return visited

经典题目：岛屿数量（LeetCode 200）

这是DFS最经典的面试题之一。给定一个二维网格（'1'表示陆地，'0'表示水），计算岛屿（相连的'1'形成的连通区域）的数量。每个格子只和上下左右四个方向相连。

def num_islands(grid):
    """DFS解法：遇到'1'就把它所在的整个岛屿'淹没'"""
    if not grid:
        return 0

    m, n = len(grid), len(grid[0])
    count = 0

    def dfs(i, j):
        # 越界或遇到水，返回
        if i < 0 or i >= m or j < 0 or j >= n or grid[i][j] == '0':
            return
        grid[i][j] = '0'  # 标记为已访问（沉岛）
        # 递归遍历上下左右
        dfs(i - 1, j)
        dfs(i + 1, j)
        dfs(i, j - 1)
        dfs(i, j + 1)

    for i in range(m):
        for j in range(n):
            if grid[i][j] == '1':
                count += 1
                dfs(i, j)  # 淹掉整座岛

    return count

这个解法被称为"沉岛算法"——遇到一个岛就把整个岛淹掉（标记为'0'），防止重复计数。代码简洁、思路清晰，是面试中的满分答案。

BFS遍历

广度优先搜索（Breadth-First Search, BFS）的思想是"层层推进"——从起点出发，先访问所有距离为1的节点，再访问所有距离为2的节点……

BFS天然使用队列（Queue）实现：

from collections import deque

def bfs(graph, start):
    """队列实现BFS遍历"""
    visited = {start}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        print(node, end=' ')  # 访问节点
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return visited

为什么BFS天然适合求最短路径？

因为BFS是"按层"遍历的。在无权图中（所有边的权重为1），BFS第一次访问到目标节点时，走的路径一定是最短的（边数最少的）——因为BFS确保先访问距离为d的节点，再访问距离为d+1的节点。

用BFS求解无权图的最短路径：

def shortest_path_bfs(graph, start, end):
    """BFS求无权图的最短路径（返回路径上的节点列表）"""
    if start == end:
        return [start]
    visited = {start}
    queue = deque([[start]])  # 队列中存储路径
    while queue:
        path = queue.popleft()
        node = path[-1]
        for neighbor in graph[node]:
            if neighbor == end:
                return path + [neighbor]
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(path + [neighbor])
    return None  # 没有路径

对比DFS和BFS：

特性 DFS BFS
数据结构 栈（或递归栈） 队列
空间复杂度 O(h)，h为深度 O(w)，w为最大宽度（通常更大）
最短路径 不能保证 保证（无权图）
连通分量 可以 可以
拓扑排序 可以（后序遍历） 可以（Kahn算法）
典型例题 岛屿数量、排列组合 最短路径、单词接龙

连通分量与环检测

无向图的连通分量

一个图可能不是"完整"的——可能有若干个互相不连通的部分。每个内部连通的部分称为一个连通分量。计算连通分量的数量，就是对每个未被访问的节点执行一次DFS/BFS：

def count_components(graph):
    """计算无向图中的连通分量数量"""
    visited = set()
    count = 0

    def dfs(node):
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                dfs(neighbor)

    for node in graph:
        if node not in visited:
            count += 1
            dfs(node)

    return count

这道题的变体出现在各种面试题中——比如"省份数量"（LeetCode 547），本质上就是连通分量计数。

有向图的环检测（三色标记法）

有向图中的环检测比无向图复杂一些。一种经典的思路是三色标记法：

  • WHITE（白色）：未访问。
  • GRAY（灰色）：正在访问中（在当前DFS路径上）。
  • BLACK（黑色）：已访问完毕（所有子孙都已处理）。

如果在DFS过程中遇到了一个"灰色"节点，说明存在环——因为这意味着从当前节点出发有一条路径回到了它自己。

WHITE, GRAY, BLACK = 0, 1, 2

def has_cycle(graph):
    """检测有向图是否有环，三色标记法"""
    color = {node: WHITE for node in graph}

    def dfs(node):
        if color[node] == GRAY:   # 遇到灰色节点 → 有环
            return True
        if color[node] == BLACK:  # 已经处理过，无需重复
            return False

        color[node] = GRAY        # 标记为正在访问
        for neighbor in graph[node]:
            if dfs(neighbor):
                return True
        color[node] = BLACK       # 标记为已处理完毕
        return False

    for node in graph:
        if color[node] == WHITE:
            if dfs(node):
                return True
    return False

"课程表"问题（LeetCode 207）就是环检测的标准应用——课程之间的先修关系构成了一个有向图，如果图中存在环，说明无法完成所有课程。

Python递归深度警告：如果图的顶点数很大（比如超过1000个），递归DFS会触发 Python 的递归深度限制（默认 1000）。在处理大型图时，优先使用栈实现DFS或队列实现BFS。或者，通过 sys.setrecursionlimit(10000) 提高限制，但请确保你知道自己在做什么。
