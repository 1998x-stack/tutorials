第18章 网络优化与故障排查

大家好，我是你们的讲师。从第15章到第17章，我们走完了一座基站从"零件"到"开通"的全过程。但通信工程不是"装好走人"的工作——基站上线的那一天，恰恰是网优工程师工作的开始。一座基站在出厂时性能参数都是最优的，但一旦部署到真实的电磁环境中，覆盖空洞、同频干扰、邻区漏配、负载不均——问题一个接一个地冒出来。

网络优化（网优）的核心哲学是：没有最优的网络，只有不断趋近最优的迭代。无线信号在空中传播，受地形、天气、建筑、人口密度、用户行为等无数变量影响。今天调好的参数，明天可能因为隔壁商场开了一家新的LED广告大屏而恶化。一个优秀的网优工程师，既要有系统性的分析思维，也要有在路测车里啃面包的吃苦精神。

18.1 网优流程：闭环方法论

网络优化是一个"测量-分析-调整-验证"的闭环流程：

  1. 数据采集：通过路测（DT）、定点测试（CQT）、网管KPI统计、用户投诉等方式收集网络性能数据。
  2. KPI分析：对比基准指标，识别劣化项。常用的KPI包括无线接通率、掉线率、切换成功率、RRC建立成功率、E-RAB建立成功率等。
  3. 问题定位：根据KPI劣化特征和路测数据，确定问题的根因——是覆盖问题、干扰问题还是参数配置问题？
  4. 优化调整：针对定位到的问题，调整天线参数（下倾角、方位角）、功率参数、邻区关系、切换门限等。
  5. 效果验证：调整后进行复测，确认指标改善。如果未达到目标，回到步骤2继续迭代。

18.2 DT路测与CQT定点测试

路测（Drive Test, DT）是网优最"接地气"的工作——把测试设备装到车上，在目标区域开车测试，记录沿途的无线信号质量。CQT（Call Quality Test, 定点测试）则是在特定点位进行固定位置的测试。

以下是5G路测中最核心的几个覆盖和速率指标：

指标 全称 优秀 良好 临界
RSRP 参考信号接收功率 ≥ -95 dBm -105 ~ -95 dBm < -105 dBm
SINR 信号与干扰加噪声比 ≥ 20 dB 10 ~ 20 dB < 10 dB
下行速率 用户面下行吞吐量 ≥ 800 Mbps 400 ~ 800 Mbps < 400 Mbps
上行速率 用户面上行吞吐量 ≥ 100 Mbps 50 ~ 100 Mbps < 50 Mbps

重要提示：路测时要注意测试手机和车辆天线的位置。测试手机应固定在车顶天线或车窗吸盘天线上，车内测试会因车体屏蔽导致RSRP偏低5 ~ 10 dB。测试路线应尽可能覆盖目标区域内所有道路，尤其是用户投诉集中的区域。好的路测规划是网优成功的一半——路测路径覆盖率不够，数据分析就会失真。

18.3 覆盖优化

覆盖是网络性能的基石。覆盖不行，其他的优化都是空中楼阁。

弱覆盖

弱覆盖是指目标区域内RSRP低于-105 dBm的区域。处理手段包括：

  • 加站：在弱覆盖区域新建基站或加装补盲微站——成本最高但效果最彻底。
  • 抬升发射功率：在不超过设备规格和法规限制的前提下，适当提高小区最大发射功率。
  • 调小下倾角：减小天线的电子下倾角或机械下倾角，让信号覆盖更远。
  • 更换高增益天线：将原有天线更换为增益更高的天线型号。

越区覆盖

越区覆盖是指本小区的信号"飘"到了远处另一个小区的覆盖范围内，造成干扰和乒乓切换。解决方案是加大下倾角或者降低功率。

覆盖空洞

覆盖空洞是指被建筑物、山体等阻挡物遮挡形成的"信号死区"。常见于城市峡谷（高楼之间的街道）和隧道内部。室分系统（DAS）或数字室分（pRRU）是解决室内覆盖空洞的主要手段。

18.4 干扰优化

干扰是移动通信"永远的敌人"。在5G时代，干扰问题甚至比覆盖问题更加棘手，因为Massive MIMO和密集组网使得干扰场景更加复杂。

系统内干扰

系统内干扰来自同一个NR网络中的其他小区：

  • PCI冲突：两个相邻小区配置了相同的PCI，导致UE无法区分。排查方法是通过网管输出PCI分配表，检查邻区列表中是否有重复PCI。
  • PCI MOD3干扰：5G NR中，PCI对3取模的结果决定了SSB在频域的位置。相邻小区的PCI若MOD3值相同，SSB会在相同RE上发送，产生严重干扰。规划时需保证邻区间PCI MOD3不同。
  • 同频干扰：相邻小区使用相同频点，在小区边缘区域信号交叠产生干扰。可通过ICIC（小区间干扰协调）或eICIC/FeICIC技术缓解。

系统间干扰

来自其他通信系统或工业设备的干扰：

  • 谐波干扰：广播台、电视台的发射谐波落入通信频段。通常在站点勘察阶段就应该排查规避。
  • 杂散干扰：邻频系统的发射机杂散辐射落入本系统接收频段。需要在发射端加装滤波器。

18.5 接入优化

接入成功率（RRC Setup Success Rate）是网优最关注的KPI之一。如果UE连不上基站，后面的所有业务都无从谈起。

接入问题常见原因：

  • RACH前导功率不足：UE在初始接入时发送的随机接入前导（Preamble）功率不够，基站收不到。解决方案是提高P0（标称接收功率）参数。
  • 前导格式选择：对于覆盖范围较大的小区（如郊区宏站），应使用前导格式较长的配置（Format 1/2/3），以提升基站的检测概率。城区小站可以用Format 0/B4。
  • PRACH频域偏移冲突：邻区间PRACH配置冲突导致前导相互干扰。

18.6 切换优化

切换是移动通信"移动性"的核心体现。切换失败通常表现为话音断续、视频卡顿甚至掉话。

5G NR中最常用的切换事件是A3事件和A5事件：

/* A3事件配置示例 — 切换门限与迟滞 */
A3 offset:        +3 dB   /* 邻区比本区好3dB即触发测量上报 */
Hysteresis:       1 dB    /* 防乒乓切换的迟滞量 */
TimeToTrigger:    320 ms  /* 触发后等待320ms再上报，避免波动误触发 */

/* A5事件配置示例 — 覆盖边缘切换控制 */
Thresh1 (本区):   -110 dBm  /* 本区信号低于此门限 */
Thresh2 (邻区):   -105 dBm  /* 邻区信号高于此门限 */
/* 两个条件同时满足时触发切换 */
  • A3事件：邻小区信号强度比本小区高出一个门限值（Offset + Hysteresis），触发切换测量报告。典型配置为Offset = 3 dB，Hysteresis = 1 dB。
  • A5事件：本小区信号低于门限1（Thresh1），且邻小区信号高于门限2（Thresh2），触发切换。用于覆盖边缘的切换控制。

切换问题中最常见的根因是邻区漏配——UE发现了一个信号很好的小区，但本基站没有将该小区配置为邻区，导致UE无法切换过去，信号越来越差直到掉线。

踩坑故事——PCI MOD3冲突导致速率暴跌：2022年我在华南某城市参与一个5G网络的KPI优化项目。某区域连续一周SINR只有5 ~ 8 dB，下行速率仅200 ~ 300 Mbps，远低于同城其他区域的800 Mbps+。用户投诉量激增。我们带着频谱仪和路测设备到现场排查，发现该区域三个宏站呈三角形分布，彼此的覆盖边缘充分交叠。查看PCI配置后发现问题：三个站的6个小区中，有4个小区的PCI对3取模的结果都是"2"。这意味着相邻小区的SSB在完全相同的RE上发送，产生了严重的同时隙干扰——6个扇区中有4个在互相"掐架"。解决方案是重规划PCI，确保相邻小区的PCI MOD3值各不相同。重新配置后复测，SINR从5 dB提升到22 dB，下行速率从280 Mbps飙升到920 Mbps——整整提升了3倍多。PCI规划不是小事，MOD3/"MOD6"/"MOD30"这几层约束在5G密集组网中尤为重要。

18.7 常见告警处理

故障排查是网优的另一半工作。以下是基站运行中常见的告警及处理思路：

告警类型 可能原因 排查步骤
光模块告警 光模块故障、光纤损坏、Tx/Rx接反 ①检查光功率 → ②清洁光纤端面 → ③更换光模块验证
VSWR告警 馈线接头松动、天线故障、馈线进水 ①用Site Master测驻波比 → ②检查接头密封 → ③分段排查
时钟失锁 GPS/北斗天线遮挡、接收机故障 ①检查GPS天线天空视野 → ②查看卫星捕获数量 → ③更换GPS接收机
传输闪断 光缆受损、传输设备故障、SFP光模块劣化 ①ping测试确认连通性 → ②检查光模块收发光功率 → ③逐段排查传输链路
小区不可用 基带板故障、CPRI/eCPRI链路中断、License过期 ①检查BBU单板状态 → ②检查前传链路 → ③确认License有效期

18.8 仪表使用

网优工程师的"武器"不只是笔记本电脑。以下几个仪表是必备的：

  • Site Master（天馈线测试仪）：用于测量馈线和天线系统的驻波比（VSWR）和回波损耗。通过测量反射信号的能量，判断天馈系统是否存在接头松动、馈线破损或进水等物理问题。驻波比超过1.5:1就需要排查。
  • 频谱仪：用于分析无线电频谱环境，查找干扰源。在5G频段上扫频，可以看到哪些频率被占用、底噪有多高。底噪抬升超过-105 dBm/100kHz的区域通常存在干扰。
  • 路测软件：如鼎利、TEMS、Nemo等，配合扫频仪和测试终端，实时记录RSRP、SINR、上下行速率、切换事件等数据。

网优工程师自白：做网优有一个"三不"原则——不只看后台参数、不只看路测截图、不只听用户描述。后台KPI好看不代表用户体验好，路测截图完美不代表用户家里就行。真正的网优工作，是坐在路测车里晒着太阳、盯着屏幕上跳动的RSRP数值、脑子里同时在分析"这附近的三个小区谁在干扰谁"——这是一门"体力活+脑力活"相结合的职业。但当你把一个区域的SINR从8 dB优化到22 dB，当投诉用户打电话来说"信号好了很多"——那种成就感，什么代码都换不来。
