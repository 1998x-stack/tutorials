第11章 滤波器结构

前面的两章，我们分别学会了如何设计FIR和IIR滤波器的系数。但得到了一组b和a系数之后，事情并没有结束——甚至可以说，真正精彩的工程问题才刚刚开始。

想象一个具体的场景：你在MATLAB中用双精度浮点设计了一个8阶椭圆低通滤波器，频率响应完美。然后你把系数下载到一颗16位定点DSP芯片上，结果滤波器的输出和仿真结果差了十万八千里。你没有改任何系数，没有改任何算法——只是把"精度"从64位降到了16位。为什么结果就完全不一样了？

答案藏在滤波器结构中。同样的系统函数H(z)，用不同的信号流图（结构）去实现，在无限精度下结果是完全等价的，但在有限精度（定点数、有限字长）下，不同结构对量化误差、乘法舍入噪声和稳定性有着天壤之别的影响。

💡 核心认知：滤波器结构是连接"理论设计"和"工程实现"的桥梁。同样一组系数，选对结构可以让你在16位定点DSP上跑出接近浮点的性能；选错结构，即使是32位定点也可能出现极限环振荡或极点漂移到单位圆外。这一章的内容，对于你将来的嵌入式DSP开发至关重要。

11.1 FIR结构：直接型与转置

FIR滤波器的差分方程非常简单：

y[n] = b₀·x[n] + b₁·x[n-1] + b₂·x[n-2] + ... + b_{N-1}·x[n-N+1]

这就是"延迟线→乘系数→累加"的结构，也叫直接型或抽头延迟线。信号流图可以画成一个横向排列的链：每个延迟单元的输出乘以对应的滤波器系数b_k，然后全部加起来。

直接型还有一个神奇的变体叫转置结构（Transposed Form）。转置规则很简单：把信号流图中所有箭头反向、把分叉点变成汇合点、把汇合点变成分叉点。转置后的结构在数学上完全等价，但在硬件实现上有一个巨大的好处——不需要输入数据的移位寄存器，只需要输出端的累加器。对FPGA实现来说，转置结构通常能达到更高的时钟频率（因为关键路径更短）。

# 直接型FIR实现（C语言风格）
for (int n = 0; n < N; n++) {
    y[n] = 0;
    for (int k = 0; k < L; k++) {
        y[n] += b[k] * x[n - k];
    }
}

# 转置型FIR实现（乘法结果直接累加到输出端）
for (int n = 0; n < N; n++) {
    for (int k = 0; k < L; k++) {
        z[k] = b[k] * x[n] + z[k+1];  // 流水线结构
    }
    y[n] = z[0];
}

11.2 IIR直接I型与直接II型

IIR滤波器的差分方程比FIR多了反馈项：

y[n] = Σ b_k·x[n-k] - Σ a_k·y[n-k]

直接I型是最"暴力"的实现：直接把分子部分（前向通路）和分母部分（反馈通路）分别用延迟线实现，然后串联。它需要2N个延迟单元（N为滤波器阶数）。

直接II型（也叫典范型）更聪明——它利用了一个事实：前向和反馈通路的延迟线可以共享。因为x[n]经过前向系数后的结果和y[n]经过反馈系数的结果是在同一个节点相加的，我们可以先把输入通过反馈部分（分母），再把中间结果通过前向部分（分子），这样延迟单元只需要N个，节省了一半。

# 直接II型实现（典范型）
w[n] = x[n] - a₁·w[n-1] - a₂·w[n-2] - ... - a_N·w[n-N]
y[n] = b₀·w[n] + b₁·w[n-1] + ... + b_N·w[n-N]

直接II型在存储空间有限的嵌入式系统中非常有用——少N个延迟单元意味着少N个RAM字和少N次内存访问。但要注意，直接型（无论I型还是II型）对系数量化误差比较敏感，尤其是高阶滤波器。

⚠️ 踩坑实录：某次我在一个工业变频器项目中，需要在TI TMS320F28335 DSP上实现一个8阶Butterworth带通滤波器。直接用了MATLAB生成的直接II型系数，结果在电机启动瞬间滤波器输出产生了持续的自激振荡，差点导致过流保护跳闸。后来分析发现：高阶直接型的极点对系数量化极其敏感——MATLAB中双精度的极点位置在定点化后发生了偏移，其中一对共轭极点恰好移到了单位圆外！解决方法很简单：把8阶滤波器分解为4个二阶节（Biquad）级联，每个二阶节的极点对数量化不敏感，问题立刻消失。

11.3 级联型（二阶节级联）

级联型（Cascade Form）将高阶H(z)分解为若干个二阶节（Biquad）的乘积。一个二阶节的系统函数是：

H_k(z) = (b₀k + b₁k·z^{-1} + b₂k·z^{-2}) / (1 + a₁k·z^{-1} + a₂k·z^{-2})

整个高阶滤波器就是这些二阶节的串联：H(z) = H₁(z) · H₂(z) · ... · H_M(z)。

为什么二阶节级联是工程首选？原因有以下几点：

  • 量化灵敏度最低。每个二阶节的极点都是独立的，系数量化只影响该节的极点位置，不会"传播"到其他节。对比直接型，一个系数量化会影响所有极点。
  • 易于调试。你可以单独测试每个二阶节的频率响应，逐个排查问题。
  • 数值稳定性最好。只要保证每个二阶节的极点都在单位圆内，整个系统就一定稳定。
  • 可复用。所有二阶节可以用同一段代码实现，只是系数不同。在实时音频处理中，这种"Biquad链"是标准架构。

在Python中，scipy.signal.sosfreqz就是基于二阶节级联的频率响应计算函数。把高阶IIR转换为二阶节级联也非常简单：

from scipy import signal

# 设计一个6阶Chebyshev低通
b, a = signal.cheby1(6, 1, 1000, btype='low', fs=44100)

# 转换为二阶节级联（SOS格式）
sos = signal.tf2sos(b, a)
# sos 是一个 (3, 6) 的矩阵，每行是一个二阶节的 [b0,b1,b2, a0,a1,a2]

# 直接用SOS格式滤波
y = signal.sosfilt(sos, x)

11.4 并联型与格型结构

除了级联，并联型是另一种分解方式：将H(z)展开为部分分式之和——每个分式对应一个一阶或二阶节，所有分式并联后求和：

H(z) = C + Σ H_k(z)    （所有节的输入相同，输出相加）

并联型的优点：每个二阶节完全不互相影响，一个节的量化误差不会传递到其他节（对比级联型，误差会逐级放大）。但并联型实现起来比级联型稍微复杂一些，因为需要做部分分式展开。

格型结构（Lattice Structure）是一种更"高级"的结构，最早起源于线性预测（LPC）领域。它的特点是：

  • 用反射系数（reflection coefficients）而不是直接系数来表征滤波器；
  • 对系数量化非常不敏感（比级联型还好）；
  • 容易判断稳定性（|k_i| < 1 对所有i）；
  • 格型结构在自适应滤波器（第15章）中特别有用。

格型结构在普通工程应用中不太常见，但在语音编码（LPC）、通道均衡和自适应滤波中是标配。如果你未来做语音编解码（如Opus、Speex编解码器），你会在源码中频繁看到格型结构的实现。

11.5 系数量化效应与极限环

这是本章最有"工程味"的章节，也是你在课本中不容易学到的知识。

系数量化效应：当你把用MATLAB双精度设计的滤波器系数（比如a₂= -1.6180339887...）截断为16位定点数（比如a₂_Q15 = -1.6180），滤波器的实际零极点位置会偏离设计值。对高阶直接型IIR来说，稍微的极点偏移可能导致：

  • 截止频率偏移（最轻微的问题）；
  • 通带纹波增大（中等严重）；
  • 极点漂移到单位圆外导致不稳定（致命问题）。

极限环（Limit Cycle）现象：这是定点IIR滤波器最诡异的特性之一。当输入变为零（或小到一定程度）后，由于量化（舍入/截断）的非线性效应，滤波器的输出可能不会归零，而是持续在一个小幅度的周期性振荡中——这就是极限环。在音频应用中，极限环表现为"死区"中的可闻噪声。

⚠️ 踩坑故事：团队在做一个16位定点DSP上的双二阶均衡器——很简单的一个低频衰减滤波器。经过严格的仿真验证后，产品进入了试产阶段。但在批量测试中，有大约2%的样机在无信号输入时出现了"嗡嗡"的低频噪声。一开始怀疑是电源噪声耦合，查了三天PCB布局无果。最后用示波器直接量DSP输出——稳定的60Hz正弦波！即便输入悬空，滤波器输出仍然保持了一个约-50dBu的振荡。这就是典型的极限环：有限字长导致的非线性振荡。解决方案有两个：一是给滤波器系数增加额外的量化保护位（从16位升级到24位内部精度），二是改用格型结构（它对极限环有天然的抑制能力）。最终我们选择了把内部累加精度提升到32位，解决了这个问题。

11.6 结构选型实战指南

不同类型的项目需要不同的滤波器结构选择。以下是一份实用的选型参考：

场景 推荐结构 原因
浮点DSP/CPU上的IIR 二阶节级联（Biquad链） 最稳定、最易调试、足够高效
16位定点DSP上的IIR 二阶节级联 + 双精度累加 抗量化、抗极限环
FPGA上的FIR 转置直接型 流水线结构关键路径最短
窄带高Q值滤波器 格型/梯格型 极点密集时量化不敏感
语音LPC分析 格型 保证稳定性、反射系数物理意义明确
自适应滤波器 格型或转置 系数更新方便、收敛性好
