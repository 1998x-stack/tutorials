📡 第7章 蜂窝网络架构演进

大家好，我是你们的讲师。前面六章我们一直在讲"传输"层面的技术——信号、调制、信道、多址、编码。从这一章开始，我们的视角将从"信号如何传输"上升到"网络如何组织"。如果说前六章是在讲"通信的物理层技术"，那么从本章开始的第二部分就是在讲"把这些技术组装成一个可以商用运营的网络系统"。

想象一下：一个运营商要在一个城市部署移动通信网络。它需要决定在哪里建基站、基站之间如何协作、用户在不同基站之间移动时如何切换不掉线、语音通话和数据业务如何路由——这些问题的答案，就写在每一代移动通信的网络架构里。

7.1 蜂窝概念：频率复用与小区分裂

在蜂窝概念提出之前，移动通信系统用的是"大功率单基站"模式——一个城市中心架一个大功率发射塔，覆盖整个城区。这种方式有两个致命问题：一是容量极其有限——一个基站只能支持几十个并发用户；二是频谱效率极低——不能复用频率。

蜂窝概念的突破性在于：不再用一个大功率基站覆盖全城，而是用大量小功率基站构成"六边形蜂窝"网络。每个基站覆盖一小片区域（一个小区），相邻小区使用不同的频率，相隔足够远的小区则可以复用相同的频率。这就是频率复用（Frequency Reuse）思想。

频率复用模式（N=7的经典复用）：
        频点F1
       /   \
     F2     F3
    /  \   /  \
   F4   F5    F6
    \  /   \  /
     F7     F1  ← F1在相隔两个小区后复用
      \    /
        ...

复用因子N=7的含义：每7个小区为一组使用全部可用频谱
每个小区只能使用 1/7 的频谱资源
但通过空间复用，整体频谱效率大幅提升

现代5G中，频率复用因子几乎是1（全网同频），
依靠Massive MIMO和波束赋形来管理同频干扰。

工程亮点：蜂窝系统的频率复用方案有一个从"硬复用"到"软复用"的演进趋势。2G时代复用因子7-12（频率严格错开），3G时代复用因子1（CDMA天然软容量），4G/5G也是复用因子1（利用ICIC干扰协调和波束赋形控制干扰）。复用因子越低，频谱效率越高，但干扰管理越复杂。

7.2 2G GSM：电路交换的黄金时代

GSM（全球移动通信系统）是2G时代的代表，也是第一个真正意义上的全球统一数字移动通信标准。它的网络架构由三部分组成：

  • BTS（基站收发台）：负责无线信号的收发，挂在铁塔上的就是它。BTS处理物理层的调制解调和时隙管理。
  • BSC（基站控制器）：管理多个BTS，负责切换决策、频率管理、功率控制。BSC是GSM中"大脑"级别的网元。
  • MSC（移动交换中心）：电路交换的核心，负责呼叫建立、路由、计费。MSC与PSTN（公共交换电话网）相连，实现手机和固定电话的互通。

GSM的核心设计思想是电路交换（CS，Circuit Switching）——通话期间为每个用户预留一条专用的通信链路，即使不说话资源也被占用着。这对语音通话来说很合适（实时性要求高），但对数据业务来说效率极低。GSM后来通过GPRS和EDGE引入了分组交换能力，但本质上是在电路交换骨架上打的"补丁"。

7.3 3G UMTS：分组交换的起步

3G UMTS（通用移动通信系统）的架构演进主要体现在两点：

  • NodeB + RNC：BTS升级为NodeB（基站），BSC升级为RNC（无线网络控制器）。RNC负责无线资源管理和移动性管理，功能比BSC更强大。
  • 核心网分组化：3G核心网引入了SGSN（服务GPRS支持节点）和GGSN（网关GPRS支持节点），真正意义上支持了分组交换（PS，Packet Switching）数据业务——手机上网从"按分钟计费"变成了"按流量计费"。

但3G的架构仍然有一个明显的"历史包袱"：它保留了电路交换域（CS Domain）来处理语音通话（通过MSC），同时增加了分组交换域（PS Domain）来处理数据。这种"双核心"架构在运营商建网和运维中带来了不小的复杂度。

关键转折：3G时代是移动通信从"电话网络"向"数据网络"转型的关键时期。2007年iPhone发布后，移动数据流量开始爆炸式增长——2008年到2012年间全球移动数据流量增长了约30倍。3G架构中的分组交换能力让运营商能应对这一浪潮，但与生俱来的CS/PS双核心设计让业界清楚地意识到：下一代网络必须彻底拥抱全IP。

7.4 4G LTE：全IP扁平化革命

4G LTE（长期演进）是一次彻底的架构革命。它去掉了RNC这个中间节点，实现了"扁平化"架构：

  • eNodeB（演进型NodeB）：比3G的NodeB强大得多，集成了原RNC的全部无线资源管理功能。基站之间通过X2接口直接通信（无需经过RNC转发），切换时延大幅降低。
  • EPC（演进分组核心网）：彻底抛弃电路交换，全IP分组核心网。核心网元包括MME（移动性管理实体）、S-GW（服务网关）、P-GW（分组数据网络网关）。
  • 全IP：从终端到核心网的所有接口都基于IP协议。语音通话不再走CS域，而是通过VoLTE（基于IMS的IP语音）在分组域上承载。

4G LTE架构的一个关键指标是用户面时延从3G的50-100ms降到了10-20ms——这直接催生了移动直播、视频通话、手机游戏等实时互动应用的爆发。

7.5 5G架构概览：gNB与5GC

5G架构在4G扁平化的基础上更进一步。它引入了服务化架构（SBA，Service-Based Architecture）和CU/DU/AAU三级拆分：

  • gNB（下一代NodeB）：拆分为CU（中央单元）和DU（分布单元）——CU处理RRC和PDCP层（非实时部分），DU处理RLC、MAC和部分PHY层（实时部分）。AAU（有源天线单元）包含射频和天线部分。
  • 5GC（5G核心网）：采用SBA架构，核心网功能被拆分为多个独立的网络功能（NF），如AMF（接入和移动性管理功能）、SMF（会话管理功能）、UPF（用户面功能）。NF之间通过HTTP/2 API通信。
  • SA vs NSA：5G有两种组网模式——这是中国运营商在实际部署中必须面对的选择。
对比维度 SA（独立组网） NSA（非独立组网）
核心网 独立的5G核心网（5GC） 依赖4G EPC核心网
控制面锚点 5G基站（gNB） 4G基站（eNodeB）+ LTE核心网
用户面路径 终端 ↔ gNB ↔ 5GC 终端 ↔ gNB/eNodeB ↔ EPC
5G能力 完全释放（uRLLC、网络切片） 部分释放（主要是eMBB增强带宽）
部署速度 慢（需新建5GC） 快（复用4G核心网）
中国现状（2026年） 已全面转向SA 早期建设模式，逐步退网

中国从2020年下半年的5G二期建设开始就全面转向了SA模式。截至2026年，中国三大运营商的5G SA网络已实现全国地级市以上覆盖。SA的优势是真正能发挥5G的全部能力——网络切片、uRLLC、MEC边缘计算——而这些是NSA无法支持的。

7.6 频谱与代际的演变

每一代移动通信使用的频段都不同——体现了低频覆盖好、高频容量大这个永恒的矛盾：

代际 典型频段 频率范围 覆盖能力 主要用途
2G GSM 900/1800 MHz 低频 极好 语音通话、短信
3G WCDMA 2100 MHz 中频 好 语音+低速数据
4G LTE 1800/2600 MHz 中高频 较好 移动宽带数据
5G FR1 700/2600/3500/4900 MHz 中高频 中等（依赖密集组网） eMBB、uRLLC、mMTC
5G FR2 24-52 GHz（毫米波） 极高 差（需极密集部署） 热点增强、固定无线接入
6G（展望） 7-24 GHz + 100-300 GHz 极高+太赫兹 很差（需超密集组网） 全息通信、通感一体化

从中可以清晰地看到频率使用的规律：低频段（<1GHz）覆盖好、穿透强，但带宽有限——因此用于广覆盖（中国广电的700MHz "黄金频段"正是为此）；高频段（>24GHz）带宽极大，但覆盖差——适用于热点区域和固定接入。

7.7 中国运营商网络现状

截至2026年中，中国5G网络的格局如下：

  • 中国移动：以2.6GHz（n41）为覆盖主力，4.9GHz（n79）用于热点补充。移动的5G基站数量超过210万座，是中国最大的5G运营商。移动的策略是"低频（700MHz广电共享）+中频（2.6GHz）"双频组网，兼顾覆盖和容量。
  • 中国电信 + 中国联通：共建共享3.5GHz（n78）频段，截至2026年累计开通5G共享基站超过120万座。通过共享节省了超过2000亿元的建设和运营成本。两家企业在3.5GHz频段上各分100MHz，通过载波聚合实现了更大的峰值速率。
  • 中国广电：手握700MHz"黄金频段"，与中国移动共建共享，主打农村和偏远地区的5G覆盖。700MHz的覆盖能力是2.6GHz的3-4倍——一个700MHz基站的覆盖半径约3-5公里，而2.6GHz基站约1-2公里。

行业趋势：2025-2026年是5G-Advanced（5G-A，即3GPP Release 18/19）的商用元年。5G-A在5G基础上增强了三大能力：上行增强（从1Gbps到10Gbps）、通感一体化（基站同时实现通信和雷达感知）、低空经济支持（无人机管理）。到2026年底，中国一线城市的核心商圈将完成5G-A升级。

7.8 踩坑故事：NSA组网下LTE锚点导致5G终端无法接入

2020年，某运营商在5G NSA商用初期遇到了一个让人头疼的问题：在某个市中心区域，用户换了5G手机、办了5G套餐，但手机却始终无法接入5G网络——明明显示"5G"图标，但测速只有4G的水平。

排查过程：

  • 第一步：检查5G NR信号——RSRP正常（-78dBm），说明5G基站信号没问题
  • 第二步：检查NSA组网配置——发现该区域的"锚点策略"配置有误

NSA组网中，5G终端必须先接入4G LTE基站（作为锚点），然后通过"双连接"（EN-DC，E-UTRAN New Radio Dual Connectivity）的方式叠加5G NR载波。这意味着4G锚点的性能直接决定了5G能否正常接入。

问题出在该区域的4G LTE基站在规划时被列入了"清频退网"计划——4G基站配置了部分频段供5G NR使用，但锚点策略没有同步更新。终端搜索5G时找到了5G NR信号，但锚点的4G LTE信号强度不足（锚点4G频段被部分回收用于5G），导致EN-DC建立失败，终端回退到纯4G模式——显示"5G"图标但实际跑的是4G的速率。

解决方案是在该区域重新规划锚点频段，确保4G LTE锚点基站有足够的覆盖能力来支撑NSA的双连接建立流程。同时，运营商加速了该区域向SA组网的升级——SA模式下，终端直接接入5G核心网，不再需要LTE锚点，这个问题自然消失。

这个案例的深层教训是：NSA虽然被设计为"从4G到5G的平滑过渡方案"，但实际上它的运维复杂度远超预期——4G和5G之间的锚点依赖关系引入了大量的互操作和参数配置问题。这也是为什么中国运营商在NSA试商用不到一年后就全面转向了SA——长痛不如短痛。
