第7章 离散傅里叶变换 (DFT)

在上一章的最后，我们留下了一个关键问题：DTFT虽然优美，但它的频谱X(e^{jω})是ω的连续函数——换句话说，计算机无法直接计算DTFT。我们需要一个"离散的、有限的、可编程的"傅里叶变换，这就是离散傅里叶变换（DFT）出场的时候。

如果说DTFT是一整条连续的光谱线，那么DFT就是在这条线上均匀地"打点采样"。这一采样看似简单，却引发了一系列重要的工程问题——频谱泄露、频率分辨率、循环卷积…… 这些都是每一个DSP工程师必须吃透的概念。

💡 一句话总结DFT的价值：DFT是DSP领域唯一一个在计算机上可实际计算的傅里叶变换。没有DFT，就没有频谱分析，就没有今天的数字通信、音频编码、雷达信号处理——甚至连你手机上的"均衡器"特效都不会存在。

7.1 DFT的定义：从DTFT采样说起

给定一个长度为N的有限长序列 x[0], x[1], ..., x[N-1]，它的DTFT是一条连续曲线：

        X(e^{jω}) = Σ_{n=0}^{N-1} x[n] · e^{-jωn},  ω ∈ [-π, π]

        现在，我们在ω轴上均匀取N个点进行采样：
        ω_k = 2πk / N,  k = 0, 1, ..., N-1

        这些采样点就是DFT的输出：
        X[k] = X(e^{jω})|_{ω=2πk/N} = Σ_{n=0}^{N-1} x[n] · e^{-j2πkn/N}

        这就是DFT的标准定义！

换句话说，DFT就是DTFT在 ω_k = 2πk/N 这N个频率点上的采样值。

DFT还可以写成矩阵形式，这在理解DFT的计算和性质时非常有帮助：

        ┌        ┐     ┌                    ┐ ┌      ┐
        │ X[0]  │     │ 1     1     ...  1 │ │ x[0] │
        │ X[1]  │     │ 1    W_N   ...  W_N^{N-1}│ │ x[1] │
        │ ...   │  =  │ ...  ...   ... ... │ │ ...  │
        │X[N-1] │     │1 W_N^{N-1}... W_N^{(N-1)²}│ │x[N-1]│
        └        ┘     └                    ┘ └      ┘

        其中 W_N = e^{-j2π/N}，称为旋转因子（twiddle factor）。

DFT矩阵的每一行对应一个频率"检测器"——它把输入序列和不同频率的复指数做内积。这和你用手机App看音频频谱时看到的"频谱柱状图"是同一回事：每一根柱子就是一个X[k]的幅值。

7.2 DFT的重要性质

DFT继承了DTFT的大部分性质，但因为频域被采样和截断，有一些独特的特性需要注意：

性质 时域 频域 备注
线性 a·x₁[n] + b·x₂[n] a·X₁[k] + b·X₂[k] 长度需一致（或补零对齐）
循环移位 x[(n - m) mod N] W_N^{km} · X[k] 注意是"循环"移位，不是线性移位
对偶性 X[n] N·x[(-k) mod N] 时频对称的体现
循环卷积 x[n] Ⓝ h[n] X[k] · H[k] DFT域乘积对应循环卷积
Parseval Σ |x[n]|² (1/N) Σ |X[k]|² 能量守恒

这里要特别强调循环移位和循环卷积。因为DFT默认信号是周期的（长度为N的序列被隐式周期延拓），所以移位操作会在边界处"绕回"。这和我们熟悉的线性移位不同。

7.3 循环卷积 vs 线性卷积：补零的艺术

这是DFT中最容易混淆但又最重要的概念。

在第3章我们学会了线性卷积：y_linear[n] = x[n] * h[n]。在第7章我们知道了DFT的卷积定理：DFT{x[n] Ⓝ h[n]} = X[k]·H[k]。但问题是，"Ⓝ"是循环卷积，不是线性卷积"*"。

什么时候循环卷积等于线性卷积？答案：当N ≥ L_x + L_h - 1 时，对x和h补零到长度N再做循环卷积，结果等于线性卷积。

        Python 示例：用DFT实现线性卷积

        import numpy as np

        def dft_convolution(x, h):
            Lx, Lh = len(x), len(h)
            N = Lx + Lh - 1  # 保证不混叠的最小长度

            # 补零到N
            x_pad = np.pad(x, (0, N - Lx))
            h_pad = np.pad(h, (0, N - Lh))

            # DFT → 相乘 → IDFT
            X = np.fft.fft(x_pad)
            H = np.fft.fft(h_pad)
            y = np.fft.ifft(X * H)

            return np.real(y)

        x = np.array([1, 2, 3])
        h = np.array([1, 1])
        print(dft_convolution(x, h))  # [1. 3. 5. 3.]

        # 和直接卷积对比
        print(np.convolve(x, h))      # [1 3 5 3]
⌨️ 工程意义：时域直接卷积的复杂度是O(L_x·L_h)，而DFT+乘积+IDFT的复杂度是O(N log N)（通过FFT实现）。当L_x和L_h都很大时（比如L_x=10000, L_h=1000），FFT加速的DFT卷积可以比时域快几十到几百倍。这就是第8章FFT的核心价值之一。

如果不补零直接做N点循环卷积会怎样？循环卷积的结果中，尾部会"卷绕"到头部——线性卷积中本该在尾部的大值，出现在了序列的开头。这是初学者最容易犯的错误：直接用N点FFT做"卷积"，得到的是循环卷积，不是你要的线性卷积。

7.4 频谱泄露与加窗

现实世界中没有无限长的信号——我们总是截取一段有限长的数据来做DFT。这个"截取"操作实际上是在原始信号上施加了一个矩形窗：

        x_windowed[n] = x_original[n] · w_R[n]

        其中 w_R[n] = 1, 0 ≤ n ≤ N-1 （矩形窗）

        根据DTFT的调制性质，时域相乘=频域卷积：
        X_windowed(e^{jω}) = X_original(e^{jω}) ⨂ W_R(e^{jω})

        矩形窗的频谱（Dirichlet核）有显著旁瓣！
        导致的结果就是：信号中本应在单根谱线上的能量"泄露"到了整个频带。

频谱泄漏（spectral leakage）是DFT分析中最常见的困惑来源。比如一个纯净的1kHz正弦信号，如果你截取的长度不是周期整数倍，DFT得到的频谱峰值会变"胖"且变"矮"——能量从主瓣泄漏到了旁瓣。

⚠️ 踩坑实录——频谱泄露引起的"幽灵信号"： 曾经帮一个做振动监测的朋友排查问题。他们在旋转机械上安装加速度传感器，采样率10kHz，采集1000个点做DFT分析。正常情况下应该只有基频（50Hz）及其倍频的谱线。但他们发现在37.6Hz处有一根莫名其妙的小谱线。团队花了两周时间排查——怀疑是轴承故障、齿轮磨损、甚至传感器松动。 最后发现，37.6Hz根本不是故障信号——它是基频（50Hz）由于截断长度不是50Hz的整数倍（1000/10000 × 50 = 5个周期，但5正好是整数？等等——1000点/10kHz = 0.1s，50Hz的周期是0.02s，恰好5个整周期），看起来周期数刚好是整数，但实际上因为FFT的频率分辨率是Δf = 10kHz/1000 = 10Hz，50Hz的谱线落在k=5的位置，这本来是好的。但他们实际分析时在1000点数据前做了一个"去直流"处理，随后数据被截断——这个截断引入了额外的频谱泄漏，导致50Hz的能量泄漏到了旁边的频率bin。 解决方案很简单：先加一个汉宁窗（Hanning window）再计算DFT。汉宁窗的旁瓣比矩形窗低得多（-32dB vs -13dB），泄漏被显著抑制，"幽灵信号"消失了。

常用窗函数的特性对比如下：

窗函数 主瓣宽度 旁瓣峰值 衰减速率 最佳用途
矩形窗 4π/N -13 dB -6 dB/oct 瞬态分析（精确提取短时事件）
汉宁窗 8π/N -32 dB -18 dB/oct 通用频谱分析（最常用）
海明窗 8π/N -43 dB -6 dB/oct 接近汉宁但旁瓣更低
布莱克曼窗 12π/N -58 dB -18 dB/oct 需要极低旁瓣的场景
凯泽窗 可调 可调 可调 最灵活，通过β参数调节

选择窗函数是"主瓣宽度"和"旁瓣抑制"之间的权衡——主瓣越宽，频率分辨率越低（两个相近的频率分量更难区分）；旁瓣越低，频谱泄漏越少。汉宁窗是"万一不知道用什么就用它"的安全选择。

7.5 频率分辨率：物理 vs 计算

这里要澄清一个在网络上传播非常广的误解。假设我们采集了N个样本，采样率是fₛ：

  • 物理分辨率（物理限界）：Δf = fₛ / N。这是DFT谱线上相邻两个频率点之间的间隔。要提高物理分辨率，要么降低采样率（但可能违反采样定理），要么增加采样点数N（延长观测时间T₀ = N·Tₛ = N/fₛ）。
  • 计算分辨率（补零内插）：你可以对N点数据补零到M点（M > N），再做M点DFT。频谱"看起来"更精细了，但实际的信息量没有增加——这只是对原始N点DTFT的曲线做了更密集的采样，并没有分辨出两个距离小于Δf的频率峰。
💡 一个经典的误解澄清："我采集了100个点，然后补了900个零，做了1000点DFT，频谱是不是就精细了10倍？"——错！补零确实让频域采样更密（计算分辨率提升），但这只是对原始100点DTFT曲线的内插，两个频率峰之间的分辨能力还是由原始N=100决定的（Δf = fₛ / 100）。好比你把一张低像素照片用插值放大10倍——图片变大了但细节没有增加。

所以DFT频率分辨率的核心公式就是：

        Δf = fₛ / N

        要提高分辨率（即减小Δf）：
        1. 增加N（延长观测时间）——唯一真正有效的方法
        2. 降低fₛ（但受采样定理约束——不能低于2f_max）

        补零只用于：
        - 让频谱看起来更平滑
        - 解决DFT长度必须为2的幂次的要求（FFT的约束）
        - 精确确定谱峰位置（通过内插）

        永远不要为了"提高分辨率"而补零。

7.6 本章小结

  • 理解了DFT的定义——它是DTFT在频域的等间隔采样：X[k] = X(e^{jω})|_{ω=2πk/N}；
  • 掌握了DFT的矩阵表示和旋转因子W_N = e^{-j2π/N}；
  • 区分了循环卷积和线性卷积，学会了用补零法通过DFT实现线性卷积：N ≥ L_x + L_h - 1；
  • 通过振动监测案例，理解了频谱泄露的机理和加窗的必要性，掌握了常用窗函数的特性对比；
  • 澄清了物理分辨率（fₛ/N）和计算分辨率（补零内插）的区别——补零不增加真实信息。

DFT是DSP领域"理论与实践"的完美交汇点。但N点DFT的直接计算需要N²次复数乘法和加法——对于N=1024来说就是约100万次运算，这在实时系统中是不可接受的。好在，下一章要讲的快速傅里叶变换（FFT）把复杂度降到了O(N log N)——N=1024时只需要约1万次运算，快了整整100倍。这个突破在1965年彻底改变了数字信号处理的面貌。
