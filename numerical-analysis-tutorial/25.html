第25章 常微分方程数值解(一)

25.1 ODE 初值问题

大家好，我是你们的讲师。从这一章开始，我们进入数值分析最贴近"真实世界"的领域——常微分方程数值解（Numerical Solution of ODEs）。

让我先问一个问题：你觉得世界上有多少物理过程可以用一个封闭形式的解析函数来描述？答案是——极少。大多数微分方程，从最简单的 y' = y² 开始，就已经没有解析解了。弹道轨迹、化学反应动力学、卫星轨道、流行病传播、期权定价——这些实际问题几乎全部依赖数值方法来求解。

我们考虑一阶常微分方程的初值问题（Initial Value Problem, IVP）：

y'(t) = f(t, y(t)), y(t₀) = y₀

目标是找到 y(t) 在 t > t₀ 上的值。所有的数值方法都遵循同一个基本策略——逐步推进（step-by-step marching）：

tₙ = t₀ + n·h, yₙ ≈ y(tₙ), n = 0, 1, 2, ...

其中 h 是步长（step size）。算法设计的关键问题就是：给定 (tₙ, yₙ)，如何计算 yₙ₊₁？

下面这个表格对比了各种方法的基本思路：

方法 公式形式 类型 精度阶数 每步计算量
向前 Euler yₙ₊₁ = yₙ + h·f(tₙ, yₙ) 显式 1 1 次 f 求值
向后 Euler yₙ₊₁ = yₙ + h·f(tₙ₊₁, yₙ₊₁) 隐式 1 求解方程
梯形法 yₙ₊₁ = yₙ + h/2·[f(tₙ,yₙ)+f(tₙ₊₁,yₙ₊₁)] 隐式 2 求解方程
改进 Euler (Heun) 预估 + 校正 显式 2 2 次 f 求值
经典 RK4 四级四阶 显式 4 4 次 f 求值

这一章我们重点讨论 Euler 法和改进 Euler 法。下一章再进入更强大的 Runge-Kutta 方法。

"显式"与"隐式"的直觉理解：显式方法就像"跟着导航开车"——你只看当前的位置和前方道路，就能算出下一步去哪。隐式方法则像"倒车入库"——你需要的下一步位置出现在公式两边，必须解一个方程才能找到它。隐式方法每步更贵，但通常更稳定（我们会在第 27 章讨论这一点）。

25.2 Euler 法

Euler 法是所有 ODE 数值方法中最简单的一个——简单到它几乎可以当作"定义"来理解。

向前 Euler 法（Forward Euler）的推导非常直观。回忆导数的定义：

y'(tₙ) ≈ (y(tₙ₊₁) - y(tₙ)) / h

代入 y'(tₙ) = f(tₙ, y(tₙ))，就得到：

yₙ₊₁ = yₙ + h·f(tₙ, yₙ)

几何上，这个方法用 tₙ 处的切线来近似 y(t) 整个区间 [tₙ, tₙ₊₁] 上的变化。显然，如果 y(t) 在这个区间内是弯曲的（即曲率大），这个近似就会很差——需要用更小的步长或更高阶的方法。

向后 Euler 法（Backward Euler）则使用 tₙ₊₁ 处的导数：

yₙ₊₁ = yₙ + h·f(tₙ₊₁, yₙ₊₁)

这是一个隐式公式——yₙ₊₁ 出现在等式两边。如果 f 关于 y 是线性的，比如 f(t, y) = λy，那么可以显式解出 yₙ₊₁ = yₙ / (1 - hλ)。如果 f 是非线性的，则需要在每一步用 Newton 法求解方程。

几何直觉 vs 数值现实：向前 Euler 在几何上很好理解（沿着切线走一步），但它有一个致命弱点：条件稳定性。考虑 y' = -100y（快速衰减过程），如果步长 h > 0.02，向前 Euler 的解会振荡发散——即使真实解是单调衰减的。这就是为什么"直觉上正确"的方法，在数值上可能完全不可用。后面我们在第 26 章会详细分析稳定性。

25.3 局部截断误差与整体误差

在 Euler 法中，我们在每一步丢弃了 Taylor 展开的高阶项。这就引入了误差，而且是两类不同类型的误差。

局部截断误差（Local Truncation Error, LTE）：假设 yₙ = y(tₙ) 完全精确，单步计算引入的误差。对于向前 Euler：

LTE = y(tₙ₊₁) - [y(tₙ) + h·f(tₙ, y(tₙ))]

= y(tₙ) + h·y'(tₙ) + h²/2·y''(ξ) - y(tₙ) - h·y'(tₙ)

= h²/2·y''(ξ) = O(h²)

所以 Euler 法的 LTE 是 O(h²)，方法是一阶的（阶数 p 满足 LTE = O(hp+1)）。

整体误差（Global Error）：从 t₀ 到 t 的总误差。假设我们需要走过 T/h 步，每步引入 O(h²) 的 LTE，那么整体误差的粗略估计为：

全局误差 ≈ (T/h) × O(h²) = O(h)

这就是为什么我们说 Euler 法是一阶方法（p=1）：LTE 是 O(h²)，全局误差是 O(h)。

更精确地说，对于一般的 p 阶方法：

  • 局部截断误差：LTE = O(hp+1)
  • 整体误差：全局 = O(hp)

这个关系非常关键——它意味着如果你想提高精度，减小步长带来的收益是有限的（线性地减少）。这就是为什么工程中很少直接用 Euler 法，而更倾向使用高阶方法的原因。

一个真实的"步长选择"踩坑故事：我曾经参与一个化学动力学模拟的项目，模型包含一系列快速反应，时间常数在微秒量级。团队里的新手直接用向前 Euler 法，选了一个"看起来合理"的步长 h = 0.001 秒。结果模拟结果完全混乱——浓度变成了负数，然后又爆炸到无穷大。原因就是 h 远大于系统的最小时间常数，违反了稳定性条件。更坑的是，程序没有报错，默默地产生了"看似合理"的振荡数据。我们花了两周时间才意识到模拟结果完全是数值伪影。这就是数值分析课上反复强调的"永远不要相信未经稳定性验证的数值解"。

25.4 改进 Euler 法与预估-校正

既然 Euler 法只有一阶精度，那如何改进？一个自然的想法是：区间 [tₙ, tₙ₊₁] 上的平均变化率，比单用端点变化率更准确。

梯形法（Trapezoidal Rule）正是基于这个思想：

yₙ₊₁ = yₙ + h/2 · [f(tₙ, yₙ) + f(tₙ₊₁, yₙ₊₁)]

梯形法使用左右两个端点的导数的平均值，它的 LTE 是 O(h³)——二阶方法（p=2）。但它是隐式的，需要求解方程。

Heun 法（改进 Euler 法）把梯形法"显式化"：先用向前 Euler 做一个预估（predictor），再用梯形公式做校正（corrector）。

import numpy as np
import matplotlib.pyplot as plt

def forward_euler(f, y0, t_span, h):
    """向前 Euler 法"""
    t0, tf = t_span
    n_steps = int((tf - t0) / h)
    t = np.linspace(t0, tf, n_steps + 1)
    y = np.zeros(n_steps + 1)
    y[0] = y0
    for i in range(n_steps):
        y[i+1] = y[i] + h * f(t[i], y[i])
    return t, y

def heun_method(f, y0, t_span, h):
    """Heun 法（改进 Euler）"""
    t0, tf = t_span
    n_steps = int((tf - t0) / h)
    t = np.linspace(t0, tf, n_steps + 1)
    y = np.zeros(n_steps + 1)
    y[0] = y0
    for i in range(n_steps):
        k1 = f(t[i], y[i])
        # 预估步
        y_pred = y[i] + h * k1
        # 校正步
        k2 = f(t[i+1], y_pred)
        y[i+1] = y[i] + h/2 * (k1 + k2)
    return t, y

# 测试：y' = -y, y(0) = 1 → 解析解 y(t) = e^{-t}
def f(t, y):
    return -y

y0 = 1.0
t_span = (0.0, 5.0)
h = 0.1

t_euler, y_euler = forward_euler(f, y0, t_span, h)
t_heun, y_heun = heun_method(f, y0, t_span, h)
t_exact = np.linspace(0, 5, 100)
y_exact = np.exp(-t_exact)

# 计算在 t=5 处的误差
idx = -1
err_euler = abs(y_euler[idx] - np.exp(-t_euler[idx]))
err_heun = abs(y_heun[idx] - np.exp(-t_heun[idx]))
print(f"h = {h}")
print(f"Euler 法在 t=5 的误差: {err_euler:.6e}")
print(f"Heun 法在 t=5 的误差: {err_heun:.6e}")

# 验证全局误差阶数：h 减半看误差变化
for h_test in [0.1, 0.05, 0.025]:
    _, y_e = forward_euler(f, y0, t_span, h_test)
    _, y_h = heun_method(f, y0, t_span, h_test)
    e_err = abs(y_e[-1] - np.exp(-5.0))
    h_err = abs(y_h[-1] - np.exp(-5.0))
    print(f"h={h_test:.3f}: Euler 误差={e_err:.2e}, Heun 误差={h_err:.2e}")
    # 期望：Euler 误差随 h 线性减小，Heun 随 h² 减小

预估算法的实现细节：Heun 法中，预估步提供 yₙ₊₁ 的一个"粗糙近似"作为校正步中 f(tₙ₊₁, yₙ₊₁) 的输入。如果一次校正不够，可以迭代多次——但通常一次就足够了，因为校正步已经将全局精度提升到了二阶。实际上，每步计算两次 f 的 Heun 法，精度远高于每步计算一次 f 的 Euler 法，而计算量只翻了一倍——这是性价比极高的升级。

运行上面代码你会发现，在 h = 0.1 时，Euler 法的误差大约是 10⁻² 量级，而 Heun 法的误差大约是 10⁻⁴ 量级——差距接近两个数量级。

如果你把 h 减半，Euler 的误差大约会减半（O(h)），而 Heun 的误差大约会变成四分之一（O(h²)）。这就是阶数差异在实际中的体现——对一阶方法，要想精度提高 10 倍，步长需要缩小 10 倍（意味着 10 倍的计算量）；对二阶方法，步长只需缩小 sqrt(10) ≈ 3.16 倍。

方法选择的经济学：在工程中，选择哪种方法本质上是一个经济决策。一阶方法每步便宜，但达到给定精度需要的步数太多。高阶方法每步更贵，但达到同样精度需要的步数少得多。对于容忍误差 10⁻⁴ 的应用，你可能愿意用 Euler 法配合小步长；但对于要求 10⁻⁸ 精度的应用（比如轨道力学），高阶方法是唯一现实的选择。下一章我们学习的 RK4 就是精度和计算量的一个"甜蜜点"。

这一章我们从 ODE 初值问题的基本概念出发，学习了 Euler 法和改进 Euler 法——它们是理解更高级方法的基石。但你可能已经注意到一个重要问题：Euler 法只有一阶精度，Heun 法也才二阶。在实际工程中，最常用的"主力"方法是四阶 Runge-Kutta 方法（RK4）。下一章，我们将深入 Runge-Kutta 方法的大家族，学习 RK4、自适应步长控制和稳定性分析。到时候见。
