第2章 语音的数字表示

大家好，我是你们的讲师。上一章我们看到了TTS技术的全景地图，从1980年代的共振峰合成一路讲到2025年的零样本克隆。但在深入每一种合成技术之前，我们有一个绕不开的基础问题要解决——语音在计算机里到底怎么存、怎么表示？

想象一下这个场景：你想用电脑录制一段语音"你好，世界"。你点击录音按钮，麦克风捕捉到了声波，然后——这个过程之后发生了什么？声波是如何变成硬盘上那一串0和1的？在TTS系统里，计算机又需要怎么理解这些数字信号，才能最终合成出自然的人声？

这一章，我们就来揭开语音数字化的面纱。这些知识不仅对理解TTS不可或缺，也是整个语音技术领域的基石。

为什么语音需要数字化？

声音本质上是一种机械波——物体振动引起空气分子的疏密变化，传播到人的耳朵里，推动鼓膜振动，最终被大脑感知为"声音"。在物理世界中，声波是连续的（模拟信号），但计算机只能处理离散的（数字信号）。因此，我们必须完成一次从连续到离散的转换。

这个转换过程通常包含两个步骤：采样（Sampling）和量化（Quantization）。合在一起，就是大家熟悉的PCM编码（Pulse Code Modulation，脉冲编码调制）。

一个直观的类比：采样就像用网格纸描一条曲线——网格越密（采样率越高），描出来的曲线就越接近原图。量化则是限制你只能用有限种颜色的笔来画——颜色种类越多（量化位数越高），画面层次就越丰富。

波形与采样率

PCM编码的原理非常简单直接：每隔一个固定的时间间隔，测量一次声波在当前时刻的振幅值，然后将这个测量值记录为一个整数。这个固定的时间间隔决定了采样率（Sample Rate），而记录数值的精度决定了量化位数（Bit Depth）。

这里有一个关键问题：采样率要设多高才够？答案来自著名的奈奎斯特采样定理（Nyquist Sampling Theorem）：为了无失真地重建一个信号，采样率必须至少是信号最高频率的两倍。换句话说，如果你想记录最高频率为 f 的声音，采样率至少要达到 2f。

这个定理直接影响着实际应用中的采样率选择：

  • 8kHz（电话语音）：可表示最高频率4kHz，刚好覆盖人话音的主要能量范围。这是传统电话网络的标准，也是为什么电话里的声音听起来"闷"——因为4kHz以上的高频信息全被切掉了。
  • 16kHz（宽带语音）：可表示最高频率8kHz，是目前TTS系统最常用的采样率。它保留了足够的语音细节，同时数据量适中。
  • 22.05kHz（半CD音质）：可表示最高频率约11kHz，常用于音乐预览或FM广播质量的音频。
  • 44.1kHz（CD音质）：可表示最高频率22.05kHz，覆盖了人耳的可听范围（20Hz-20kHz）。这是音乐CD的标准。

量化位数通常为16位（即每个采样点用2字节表示，可表示65536个级别的振幅），这是CD和大多数音频文件的标准。更高的24位或32位浮点常用于专业录音和后期处理，但对于TTS来说，16位已经绰绰有余。

做一个简单的计算：一个16kHz、16位、单声道的语音信号，每秒的数据量为 16000 × 2 = 32000 字节，约31.25 KB/s。一小时这样的录音大约是112.5 MB。

从波形到频谱：短时傅里叶变换

原始的PCM波形虽然是语音的完整表示，但它有一个根本问题：太"细"了。PCM告诉我们每个时刻的振幅是多少，但这个信息对于高阶的语音处理来说太底层。我们需要的是从另一个角度观察语音——频率的角度。

这就引出了傅里叶变换（Fourier Transform）。傅里叶变换告诉我们一个信号由哪些频率成分组成。但语音信号是非平稳的——它的频率特性随时间变化。一段语音中可能前半段是元音（低频能量集中），后半段是摩擦音（高频能量集中）。用全局的傅里叶变换会把这些混在一起，什么都看不清。

解决办法是短时傅里叶变换（STFT，Short-Time Fourier Transform）。核心思想是：假设在非常短的时间窗内（通常20-50毫秒），语音信号是平稳的，然后对这个短片段做傅里叶变换。把这个窗口滑动到整个语音上，就能得到频率随时间的变化。

这里有一个重要的细节：窗函数。直接截取一段信号等价于乘了一个矩形窗，这会在频域引入严重的旁瓣泄漏（spectral leakage）。为了减轻这个问题，通常使用平滑的窗函数，如汉明窗（Hamming Window）或汉宁窗（Hann Window）。它们的共同特点是中间高、两边低，让截取片段的两端平滑过渡到零。

STFT的结果通常可视化为语谱图（Spectrogram）——横轴是时间，纵轴是频率，每个点的颜色深浅代表该频率在该时刻的能量强度。语谱图是语音研究者最重要的可视化工具，有人形容它为"语音的指纹"。经验丰富的语音学家仅凭看语谱图就能判断出发的是什么音、说话人的性别甚至情绪状态。

Mel尺度与人耳听觉

人类的听觉系统有一个奇妙的特点：它不是线性地感知频率的。人耳对低频变化更敏感，对高频变化则相对迟钝。举个例子，从100Hz到200Hz（翻倍）听起来是"提高了很多"，但从10000Hz到10100Hz（同样的绝对差值）你几乎感觉不到变化。

这个特性被Stevens、Volkmann和Newman在1937年用心理声学实验量化成了Mel尺度。Mel是"melody"的缩写，意在表示这是符合人耳音高感知的尺度。从频率 f（Hz）到 Mel 频率 m 的转换公式如下：

m = 2595 * log10(1 + f / 700)

把线性频率映射到Mel尺度后，我们就可以设计一组Mel滤波器组：在Mel尺度上等间隔分布的三角滤波器，映射回线性频率时，低频区域的滤波器间隔小、带宽窄，高频区域的滤波器间隔大、带宽宽。这与人类听觉的感知特性完全一致。

有了Mel滤波器组，提取MFCC（Mel频率倒谱系数，Mel-Frequency Cepstral Coefficients）就顺理成章了：

  1. 对语音信号分帧、加窗（通常帧长25ms，帧移10ms）
  2. 对每一帧做STFT，得到频谱
  3. 将频谱能量乘上Mel滤波器组，得到Mel频谱
  4. 对Mel频谱取对数（模拟人耳对响度的对数感知）
  5. 对Log-Mel频谱做离散余弦变换（DCT），得到MFCC

MFCC在很长时间里是语音领域的标配特征——不仅用于TTS，也广泛用于语音识别和说话人识别。但在TTS领域，近几年的趋势是用原始Mel频谱（不做最后一步DCT）来代替MFCC，因为深度神经网络可以直接学习到频谱中的结构信息，不需要DCT去去相关。

下面是用Python和librosa库提取Mel频谱的代码：

import librosa
        import numpy as np

        # 加载音频文件，重采样到16kHz
        audio, sr = librosa.load('speech.wav', sr=16000)

        # 提取Mel频谱
        # n_fft=1024: FFT窗口大小
        # hop_length=256: 帧移（对应16ms@16kHz）
        # n_mels=80: 80维Mel滤波器组
        mel_spec = librosa.feature.melspectrogram(
            y=audio, sr=sr,
            n_fft=1024,
            hop_length=256,
            n_mels=80,
            fmin=0,
            fmax=8000
        )

        # 转换为对数尺度
        log_mel_spec = librosa.power_to_db(mel_spec)

        # 输出形状: (80, T)
        # 80是Mel维度, T是时间帧数
        print(f"Mel频谱形状: {log_mel_spec.shape}")
        # 例如5秒的语音 -> 形状 (80, 313)
        # 每帧覆盖10ms，5秒约5000ms/16ms ≈ 313帧

基频F0与谐波结构

除了频谱包络所代表的声道形状信息，语音中另一个关键特征是基频（Fundamental Frequency，简称F0）。基频对应声带振动的频率，它直接决定了我们感知到的"音高"——男性说话时声带厚而长，振动频率低（约80-180Hz）；女性声带薄而短，振动频率高（约180-400Hz）。

基频在语音中的物理表现是一系列的谐波：如果基频是200Hz，那么频谱上会在200Hz、400Hz、600Hz、800Hz……处看到能量尖峰。这些谐波的包络由声道滤波决定，形成共振峰结构。

在TTS中，基频是一个至关重要但又难以完美建模的参数：

  • 韵律与情感：基频的升降变化（语调）承载了大量的语用信息。"他在家吗？"的末尾调升高，"他在家。"的末尾调下降。同一个句子用不同的F0曲线说出来，可能表达完全不同的态度（平静vs兴奋vs沮丧）。
  • F0的连续性问题：在浊音段（元音、鼻音等），声带振动，F0有明确的数值；在清音段（s、sh、f等摩擦音），声带不振动，F0不存在或为零。F0还经常出现倍频或半倍频的检测错误（octave jump），给建模带来很大挑战。
  • F0与说话人身份：基频是区分不同说话人最直接的特征之一。在多说话人TTS中，F0的建模直接影响语音克隆的相似度。

语音特征的工程选择

不同的TTS系统对语音特征有不同的偏好。以下表格对比了几种常见的音频表示方式：

表示方式 维度（典型值） 信息量 是否适合作为TTS目标 典型应用
原始PCM波形 1维（16kHz = 16000点/秒） 完整信息，包含相位 少数端到端模型（WaveNet） WaveNet、VALL-E
线性幅度谱 513维（fft=1024时的正频部分） 较完整，含高频细节 部分声码器 WORLD声码器
Mel频谱（Log-Mel） 80维（当前最主流选择） 压缩了高频信息，保留了感知关键信息 ✅ 绝对主流 Tacotron、FastSpeech、VITS
MFCC 13-40维 高度压缩，丢失了部分频谱细节 曾是主流，现已较少使用 传统HMM系统、语音识别

在实践中，当前TTS系统最主流的选择是：80维对数Mel频谱（声学模型输出）+ 1维对数F0（单独预测）+ 清/浊音标志（voice/unvoiced flag）。以16kHz采样率、帧移10ms（160点）为例，每秒钟产生100帧，每帧81维（80维Mel + 1维F0标志），也就是每秒8100个浮点数。相比直接用PCM波形（16000个16位整数），这是一个巨大的压缩。

一个有趣的历史细节：在2010年代初期，很多TTS研究者坚信MFCC是最好的语音特征，因为DCT去相关后更适合用对角协方差的高斯模型来建模（当时HMM是主力）。但深度神经网络普及后，人们发现让网络学习原始Mel频谱反而更好——DCT带来的信息损失比相关性"干扰"更严重。技术的进步有时就是这么讽刺：你费尽心思设计了一个"优化"步骤，结果后来被证明是多余的。
