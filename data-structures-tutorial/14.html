排序——算法世界的"基本功"

如果你问一个程序员："你学习的第一种算法是什么？"十有八九的答案是"排序"。排序不仅是算法入门的"Hello World"，更是计算机科学中研究最深入、应用最广泛的问题之一。

为什么排序如此重要？因为有序数据比无序数据更容易处理。想象你有一本没有按字母排序的电话本——要找"张三"，你得翻遍整本本子。有了排序，二分查找、双指针、去重、归并这些高效操作才成为可能。

更关键的是——排序是许多算法和数据结构的"前置依赖"：

  • 二分查找：要求数组已排序。
  • 双指针（两数之和、三数之和）：通常需要先排序。
  • 数据去重：排序后重复元素相邻，一次遍历即可。
  • 区间合并：需要按起点排序。
  • Kruskal算法：需要按边权重排序。

掌握排序算法，意味着你同时掌握了递归、分治、堆操作、比较器等基础编程技能。这一章我们不会局限于"排序算法怎么实现"，而是深入到"为什么这个排序快，那个排序慢""什么时候用这个，什么时候用那个"。

基础排序——O(n²)三人组

冒泡排序（Bubble Sort）

冒泡排序是最直观的排序：每次比较相邻的两个元素，如果顺序不对就交换。一趟下来，最大的元素"冒"到了末尾。重复 n-1 趟。

def bubble_sort(arr):
    """冒泡排序，O(n²)"""
    n = len(arr)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:  # 优化：如果没有交换，说明已经有序
            break
    return arr

选择排序（Selection Sort）

选择排序也很直观：每趟从未排序部分找到最小值，放到已排序部分的末尾。

def selection_sort(arr):
    """选择排序，O(n²)"""
    n = len(arr)
    for i in range(n - 1):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr

插入排序（Insertion Sort）

插入排序像"打扑克牌"——每次从桌上摸一张牌，插入到手中已排序牌组的正确位置。

def insertion_sort(arr):
    """插入排序，O(n²)"""
    n = len(arr)
    for i in range(1, n):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr

你可能心想："O(n²)？这么慢，还有学的必要吗？" 答案是——有的。插入排序在两种情况下非常有用：

  1. 数据基本有序时：插入排序的时间复杂度接近 O(n)。
  2. 数据量很小时（如 n ≤ 50）：常数因子小，实际运行速度可能比O(n log n)的排序还快。

正因为如此，很多高效的排序算法（如Timsort——Python和Java内置的排序算法）在数据量小时会"退化"到插入排序。

面试小贴士：当面试官问"你什么时候会用O(n²)的排序？"不要只说"永远不会"。可以回答："在数据量很小或者数据基本有序时，插入排序是实际最快的选择。很多工业级排序库对于小规模子数组都是用插入排序完成的。"

高效排序——O(n log n)三剑客

归并排序（Merge Sort）——分治+稳定

归并排序是分治思想的经典应用：先把数组分成两半，分别排序，然后合并两个有序数组。

def merge_sort(arr):
    """归并排序，O(n log n)，稳定"""
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    # 合并两个有序数组
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:  # <= 保证稳定性
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

归并排序的特点：

  • 稳定排序：相等元素的相对顺序不变。
  • 空间复杂度 O(n)：需要额外数组来合并（这是它的主要缺点）。
  • 对链表排序友好：链表无法随机访问，但归并排序只需要顺序访问。
  • 外部排序：当数据量大到无法全部装入内存时，归并排序是标准选择。

快速排序（Quick Sort）——分区+不稳定

快速排序也用了分治思想，但与归并的不同之处在于：它先"分区"（partition），再递归处理左右两部分。核心是选择一个枢轴（pivot）元素，把小于pivot的放左边，大于pivot的放右边。

def quick_sort(arr, low=0, high=None):
    """快速排序，平均 O(n log n)，最坏 O(n²)，不稳定"""
    if high is None:
        high = len(arr) - 1

    if low < high:
        pivot_idx = partition(arr, low, high)
        quick_sort(arr, low, pivot_idx - 1)
        quick_sort(arr, pivot_idx + 1, high)

    return arr

def partition(arr, low, high):
    """Lomuto分区方案"""
    pivot = arr[high]  # 选最后一个元素作为pivot
    i = low - 1        # i 指向"小于pivot的区域"的末尾
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1

# 更高效的 Hoare 分区方案
def partition_hoare(arr, low, high):
    pivot = arr[(low + high) // 2]
    i, j = low - 1, high + 1
    while True:
        i += 1
        while arr[i] < pivot:
            i += 1
        j -= 1
        while arr[j] > pivot:
            j -= 1
        if i >= j:
            return j
        arr[i], arr[j] = arr[j], arr[i]

堆排序（Heap Sort）——原地排序

堆排序我们在第11章已经详细学过，这里简单复用一下它的思想：建堆 → 不断取堆顶。

def heap_sort(arr):
    """堆排序，O(n log n)，原地排序，不稳定"""
    n = len(arr)

    # 建最大堆
    for i in range(n // 2 - 1, -1, -1):
        _heapify(arr, n, i)

    # 逐个取出堆顶
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]
        _heapify(arr, i, 0)

    return arr

def _heapify(arr, n, i):
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2
    if left < n and arr[left] > arr[largest]:
        largest = left
    if right < n and arr[right] > arr[largest]:
        largest = right
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        _heapify(arr, n, largest)

为什么快排实际最快？

面试中有一个经典问题："既然三者都是 O(n log n)，为什么快速排序是实际中最快的排序算法？"

原因有三：

  1. 缓存友好性：快排访问内存是顺序的（从左到右扫描数组），CPU缓存命中率极高。堆排序访问内存是跳跃的（访问堆中的父节点和子节点，它们在数组中的位置可能相距很远），缓存不命中率更高。归并排序需要额外数组，在大数据量时有内存复制的开销。
  2. 常数因子小：快排的内层循环非常简单——就是比较和交换两个元素。而堆排序有更多的操作（多次比较和交换）。
  3. 原地排序：快排和堆排序都是原地排序（空间 O(log n) 是递归栈），而归并排序需要 O(n) 额外空间。

但注意——快排有一个"阿喀琉斯之踵"：最坏时间复杂度 O(n²)。当pivot每次选到最大或最小元素时（比如数组已经有序），快排退化为 O(n²)。解决方案：随机选择pivot 或 三值取中法（取首、中、尾三个元素的中位数作为pivot），这两种方法使最坏情况几乎不可能发生。

快排 vs 归并的工程选择：大多数语言的通用排序库实际上使用混合排序——Python的Timsort是归并+插入的混合体，C++的std::sort是快排+堆+插入的混合体（称为introsort）。没有"最好"的排序算法，只有最适合场景的排序算法。

线性时间排序

你可能听说过一个"定理"：基于比较的排序算法的时间复杂度下限是 O(n log n)。这是真的——任何基于两两比较的排序算法，在最坏情况下至少需要 n log n 次比较。

但是，如果我们不依赖于比较，而是利用数据本身的特殊性质，就可以突破 O(n log n) 的下限。线性时间排序（Linear-time Sort）就是为此而生。

计数排序（Counting Sort）

适用场景：数据范围（最大值 - 最小值）不大，且知道数据范围。

def counting_sort(arr):
    """计数排序，O(n + k)，k是数据范围"""
    if not arr:
        return arr
    max_val, min_val = max(arr), min(arr)
    range_size = max_val - min_val + 1
    count = [0] * range_size

    # 计数
    for num in arr:
        count[num - min_val] += 1

    # 输出
    result = []
    for i in range(range_size):
        result.extend([i + min_val] * count[i])

    return result

桶排序（Bucket Sort）

适用场景：数据在某个范围内均匀分布（如[0,1)范围内的浮点数）。

def bucket_sort(arr, bucket_size=5):
    """桶排序，平均 O(n + k)"""
    if not arr:
        return arr

    min_val, max_val = min(arr), max(arr)
    bucket_count = (max_val - min_val) // bucket_size + 1
    buckets = [[] for _ in range(bucket_count)]

    # 分配到桶中
    for num in arr:
        idx = (num - min_val) // bucket_size
        buckets[idx].append(num)

    # 对每个桶排序（可以用插入排序）
    result = []
    for bucket in buckets:
        result.extend(insertion_sort(bucket))

    return result

基数排序（Radix Sort）

适用场景：整数排序，按位（个位、十位、百位……）依次排序。每一轮使用稳定的计数排序。

def radix_sort(arr):
    """基数排序，O(d × (n + k))，d是位数"""
    if not arr:
        return arr

    max_val = max(arr)
    exp = 1  # 当前位数：1=个位, 10=十位, 100=百位...

    while max_val // exp > 0:
        counting_sort_by_digit(arr, exp)
        exp *= 10

    return arr

def counting_sort_by_digit(arr, exp):
    """按指定位数进行计数排序（稳定）"""
    n = len(arr)
    output = [0] * n
    count = [0] * 10  # 0-9

    # 统计当前位的出现次数
    for num in arr:
        digit = (num // exp) % 10
        count[digit] += 1

    # 前缀和，用于确定位置
    for i in range(1, 10):
        count[i] += count[i - 1]

    # 倒序输出，保证稳定性
    for i in range(n - 1, -1, -1):
        digit = (arr[i] // exp) % 10
        count[digit] -= 1
        output[count[digit]] = arr[i]

    for i in range(n):
        arr[i] = output[i]

线性时间排序不是万能的——它们要么需要额外空间，要么只适用于特定类型的数据（整数、小范围、均匀分布）。但在合适的场景下，它们比 O(n log n) 的排序快得多。

排序算法对比总结

终于到了"全家福"时间。这是面试中最常被问到的表格——建议背下来。

算法 平均 最坏 最好 空间 稳定 适用场景
冒泡排序 O(n²) O(n²) O(n) O(1) ✔ 教学示例，几乎不用
选择排序 O(n²) O(n²) O(n²) O(1) ✘ 教学示例，交换次数最少
插入排序 O(n²) O(n²) O(n) O(1) ✔ 基本有序、数据量小
归并排序 O(n log n) O(n log n) O(n log n) O(n) ✔ 链表、外部排序、需要稳定
快速排序 O(n log n) O(n²) O(n log n) O(log n) ✘ 通用排序（配合随机pivot）
堆排序 O(n log n) O(n log n) O(n log n) O(1) ✘ 需要原地排序、嵌入式环境
计数排序 O(n + k) O(n + k) O(n + k) O(k) ✔ 范围小的整数
桶排序 O(n + k) O(n²) O(n + k) O(n) ✔ 均匀分布数据
基数排序 O(d·(n+k)) O(d·(n+k)) O(d·(n+k)) O(n + k) ✔ 整数、定长字符串
核心思想：没有"最好"的排序算法，只有最适合场景的排序算法。如果你需要的是通用排序，用快排（或语言内置的混合排序）。如果你需要稳定排序，用归并。如果你数据量特别大且对空间敏感，用堆排序。如果你的数据有特殊性质（小范围整数、几乎有序），利用这些性质选择线性时间排序或插入排序。

最后给你一个实用建议——在几乎所有实际场景中，使用语言内置的排序函数（Python的 sorted()、Java的 Arrays.sort()、C++的 std::sort()）。这些内置函数都是经过高度优化的，内部使用了混合策略，比你手写的任何单一排序算法都要快。但理解它们背后的原理——那就是另一回事了。知其然，更要知其所以然。
