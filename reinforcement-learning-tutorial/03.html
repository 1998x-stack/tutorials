第三章：马尔可夫决策过程

大家好，我是你们的讲师。上一章我们聊了强化学习的基本框架——agent 和环境不断交互，agent 采取动作，环境反馈状态与奖励。但你有没有想过：这个交互过程到底能不能用数学严格地描述？agent 在每一步到底需要知道多少历史信息才能做出好的决策？

这一章我们就来回答这些问题。我们将引入强化学习中最核心的数学模型——马尔可夫决策过程（Markov Decision Process，MDP）。可以说，95% 以上的强化学习问题最终都会被建模成一个 MDP。理解了 MDP，你就掌握了强化学习的"通用语言"。

3.1 MDP 的形式化定义

一个马尔可夫决策过程由五个元素构成，通常记作一个五元组 (S, A, P, R, γ)。让我们一个一个来看。

符号名称含义
S状态空间
State Space		所有可能状态的集合。可以是离散的（如网格的每个格子）或连续的（如机器人关节角度）。
A动作空间
Action Space		所有可能动作的集合。同样可以是离散的或连续的。
P状态转移概率
Transition Probability		P(s' | s, a) 表示在状态 s 执行动作 a 后，转移到 s' 的概率。
R奖励函数
Reward Function		R(s, a) 表示在状态 s 执行动作 a 后获得的即时奖励（有时也写作 R(s, a, s')）。
γ折扣因子
Discount Factor		γ ∈ [0, 1)，衡量未来奖励相对于当前奖励的重要程度。

MDP 的交互流程可以这样描述：在每一个时间步 t，agent 观察到当前状态 st ∈ S，然后从动作空间中选择一个动作 at ∈ A。环境根据转移概率 P(st+1 | st, at) 产生下一个状态 st+1，同时返回一个即时奖励 rt = R(st, at)。这个过程不断重复，直到达到终止状态（如果任务有终止条件的话）。

为什么 γ 必须小于 1？ 这里有一个非常微妙但至关重要的原因。在 infinite-horizon（无限期）设定下，agent 会无限地交互下去。如果 γ = 1，那么累计奖励 Gt = rt + rt+1 + rt+2 + … 就是一个无穷级数，很可能发散到无穷大，导致我们无法比较不同策略的好坏。而当 γ ∈ [0, 1) 时，这是一个收敛的几何级数，累计奖励有界，策略比较就有了坚实的数学基础。这一点在后续推导 Bellman Equation 时尤为关键。

3.2 马尔可夫性质

MDP 名字中的"马尔可夫"来源于一个关键假设——马尔可夫性质（Markov Property）：

P(st+1 | st, at) = P(st+1 | st, at, st-1, at-1, …, s0, a0)

翻译成人话就是：未来只依赖于现在，与过去无关。当前的 state 已经包含了做决策所需的全部信息。你不需要记住过去的历史——当前的局面就已经是"充分的统计量"。

举个具体的例子。下国际象棋时，如果你只告诉我"当前棋盘上每个棋子的位置"，这其实就已经包含了做出最优走法所需的全部信息。我不需要知道"白方上一手是怎么走的"或者"黑方在第三步是不是犹豫了很久"，因为这些历史信息不会改变当前局面的最优策略。换句话说，棋盘的当前配置本身就满足了马尔可夫性质。

踩坑实录：我刚接触 RL 时犯过一个典型的错误。当时我在做一个"自动驾驶小车"的实验，state 只用了小车当前的位置和速度。结果训练出来的策略表现极差，小车总是在同一个地方打转。排查了好久才发现问题：小车的发动机有一个延迟效应——油门踩下去后要过 0.2 秒才有反应。单靠当前的速度完全无法判断发动机的"真实状态"。解决办法是 在 state 中加入最近 3 步的动作历史，让 state 重新满足马尔可夫性质。这个教训让我记住了一句话："如果你的 state 不是 Markovian，那你的 RL 代码写得再好也没用。"

什么时候马尔可夫性质会不成立？简单说，当你的 state 定义不够完整时。比如：

  • 部分可观测环境（POMDP）—— agent 只能看到环境的一部分信息。
  • 有隐藏状态的环境——如上面提到的发动机延迟、或者游戏中有迷雾遮挡。
  • state 的表示粒度不够——比如用 10 个格子表示一个连续空间，丢失了精细位置信息。

在这些情况下，我们需要引入更复杂的模型（如 POMDP、RNN 策略、或者使用 frame stacking），但这超出了本章的范围。好消息是：大量实际 RL 问题可以被巧妙地设计成满足马尔可夫性质，这正是 MDP 如此强大的原因。

3.3 转移概率与奖励函数

转移概率 P(s' | s, a) 描述了环境的"物理规则"：从状态 s 出发执行动作 a 后，世界会变成什么样子。这里有两种情况：

确定性转移（Deterministic Transition）： 执行某个动作后，下一状态是唯一确定的。例如在 GridWorld 中，你按"上"键，只要上面不是墙，就一定会移动到上面的格子。

随机性转移（Stochastic Transition）： 执行同一个动作，可能得到多个不同的结果。例如在机器人控制中，由于电机噪声，你发出"前进"指令，机器人可能稍微偏左或偏右。Slip probability 在这种场景中非常常见。

奖励函数 R(s, a) 定义了什么行为是"好的"。注意奖励函数和转移概率是环境的一部分——agent 不能修改它们，只能去适应它们。这一点容易被初学者混淆：agent 学习策略，而奖励函数是由设计者定义的。

当我们说"期望奖励"时，指的是：

R(s, a) = E[ rt | st = s, at = a ]

在随机性环境中，即使 (s, a) 相同，实际获得的即时奖励也可能不同。公式中的期望就是在这些随机性上取平均。

3.4 用 MDP 建模现实问题：GridWorld 实例

没有什么比一个具体的例子更能说明问题了。我们来看经典的 GridWorld（网格世界）问题。

场景描述： 一个 5×5 的网格，agent 从左上角 (0,0) 出发，目标是到达右下角 (4,4) 的宝藏格子。某些格子是障碍物（墙），agent 不能穿过。每走一步消耗 1 点能量（reward = -1），到达宝藏获得 +50。

这个简单的问题可以完美映射到 MDP 的五元组：

MDP 元素GridWorld 中的对应
S所有非障碍的格子坐标 (x, y)，共约 23 个状态
A{上, 下, 左, 右} 四个动作
P若 s' 不是墙，P(s'|s,a)=1；否则 agent 停留在原地
R到达宝藏 +50，普通步 -1，撞墙 -1（并留在原地）
γ0.9，鼓励尽快到达目标

下面是这个 GridWorld 环境的 Python 伪代码实现。注意这只是一个教学示例——实际代码会更精细，但核心逻辑就在这里：

import numpy as np

class GridWorldMDP:
    def __init__(self, size=5, gamma=0.9):
        self.size = size          # 网格大小 5×5
        self.gamma = gamma        # 折扣因子
        self.goal = (4, 4)        # 宝藏位置

        # 状态空间：所有非障碍的格子
        self.states = [(i, j) for i in range(size)
                          for j in range(size)
                          if (i, j) not in self.walls]

        # 动作空间：四个方向
        self.actions = {'up': (-1, 0), 'down': (1, 0),
                   'left': (0, -1), 'right': (0, 1)}

    def transition(self, s, a):
        """转移概率 P(s' | s, a) —— 此处为确定性"""
        i, j = s
        di, dj = self.actions[a]
        s_next = (i + di, j + dj)

        # 撞墙或出界则留在原地
        if s_next not in self.states:
            s_next = s
        return s_next

    def reward(self, s, a):
        """奖励函数 R(s, a)"""
        s_next = self.transition(s, a)
        if s_next == self.goal:
            return 50         # 找到宝藏！
        else:
            return -1          # 每走一步消耗能量

    def step(self, s, a):
        s_next = self.transition(s, a)
        r = self.reward(s, a)
        done = (s_next == self.goal)
        return s_next, r, done
💡 设计技巧： 在 GridWorld 中，每一步给 -1 的 reward 实际上是在编码"尽快到达目标"的目标。如果不用 -1 而用 0，那么 agent 可能绕远路也没关系。这就是 reward shaping（奖励塑形）的基本思想——通过设计奖励函数来引导 agent 的行为。但要注意，过度 shaping 可能导致 agent 钻空子，学出设计者预期之外的行为（reward hacking），这也是 RL 实践中一个非常有趣的现象。

在这个 MDP 的基础上，一个最优策略（optimal policy）会告诉 agent：在每个格子应该往哪个方向走，才能最大化累积折扣奖励。对于这个简单的 GridWorld，最优策略几乎就是"朝着宝藏方向的最短路径"。但在更复杂的问题中，策略可能非常反直觉——这正是 RL 算法的用武之地。

3.5 策略与值函数初步

有了 MDP 的定义，我们就可以精确地描述 agent 的行为和"好坏了"。这里引入两个关键概念：策略（Policy）和值函数（Value Function）。

策略 π(a | s) 定义了 agent 在状态 s 下采取动作 a 的概率。策略有两种形式：

  • 确定性策略（Deterministic Policy）： π(a | s) = 1 对于某个 a，其余动作为 0。即"在状态 s 下永远执行动作 a"。
  • 随机性策略（Stochastic Policy）： π(a | s) 是一个概率分布。在同一个状态下，每次可能选择不同的动作。

初学者可能会问：为什么要用随机策略？确定性策略不是更直接吗？原因有二：一是环境中存在不确定性时，随机策略通常更鲁棒；二是随机策略天然具备探索能力——这一点我们讲到 exploration-exploitation tradeoff 时再深入展开。

状态值函数 Vπ(s) 表示在策略 π 下，从状态 s 出发所能获得的期望累积折扣奖励：

Vπ(s) = Eπ [ ∑k=0∞ γk rt+k | st = s ]

动作值函数 Qπ(s, a) 则更进一步：它表示在状态 s 先执行动作 a，之后遵循策略 π，所能获得的期望累积折扣奖励：

Qπ(s, a) = Eπ [ ∑k=0∞ γk rt+k | st = s, at = a ]

这两个"值函数"是强化学习的核心。如果你只能掌握两个概念，那就是 policy 和 value function。几乎所有 RL 算法都可以归为三类：基于值函数的（value-based，学 Q 函数）、基于策略的（policy-based，直接学 π）、以及二者结合的 actor-critic。

直观理解： 想象你是一个围棋棋手。V(s) 问的是"当前棋盘这个局面，我最终赢的概率是多少"；Q(s, a) 问的是"在当前棋盘下，如果我落子在某个具体位置，之后我赢的概率是多少"。策略 π(a|s) 则是"在当前局面下，我选择每个落子位置的概率"。

这里我们只是初步认识这两个概念。下一章我们将深入探讨它们之间最深刻的关系——贝尔曼方程（Bellman Equation）。这个方程将 V 和 Q 与 MDP 的 P、R 联系在一起，构成了几乎所有后续 RL 算法的理论基础。

本章小结

让我们回顾一下这一章的核心要点：

  1. MDP (S, A, P, R, γ) 是强化学习的标准数学框架。几乎所有 RL 问题都可以（或可以被设计成）MDP。
  2. 马尔可夫性质 保证了"当前状态包含所有必要信息"。如果你的 state 设计不满足这个性质，RL 算法很可能失败。
  3. 转移概率和奖励函数 定义了环境的动力学。前者描述"世界如何变化"，后者描述"什么行为是好的"。
  4. 通过 GridWorld 的例子，我们看到了一个具体问题如何映射到 MDP 的形式化框架中。
  5. 策略 π 和 值函数 Vπ、Qπ 是描述 agent 行为和评估 agent 表现的核心工具。

下一章，我们将进入强化学习最优雅的数学结果之一——贝尔曼方程。这个方程揭示了 V、Q 与 P、R 之间的递归关系，它将为后续所有算法（从动态规划到深度 Q 网络）提供统一的数学基础。敬请期待！
