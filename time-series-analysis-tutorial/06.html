第6章 时间序列回归

大家好，我是你们的讲师。如果你学过计量经济学或机器学习，你一定对线性回归不陌生。Y = β₀ + β₁X₁ + ... + ε——这个公式太经典了。但把回归直接用在时间序列上，会有一个巨大的陷阱，甚至可以让你得出完全错误的结论。

想象一下：你把中国过去20年的GDP和某个随机生成的随机游走序列做回归，结果发现 p 值小于 0.001，R² 高达 0.85。这是不是说明这个随机变量能"解释"中国经济增长？当然不是。这就是著名的"伪回归"（Spurious Regression）问题——两个毫无关系的非平稳序列，仅仅因为都随时间增长，就会表现出虚假的相关性。

1993年《经济学杂志》上有一篇经典文章，模拟了完全独立的随机游走变量之间的回归，结果在5%的显著性水平下有超过70%的情况都"显著"地拒绝了原假设。换句话说，用非平稳数据做回归，显著性检验毫无意义。

我自己就踩过这个坑。早年做宏观因子分析时，我把 CPI 和 M2 货币供应量直接回归，得到了一个漂亮的 R²，兴奋地拿给导师看。导师只看了一眼说："把两个序列的差分画出来看看。"我照做，结果差分后的 R² 几乎是 0。那一刻我才真正理解了伪回归的含义。

6.1 OLS 回归在时间序列中的应用

也不是说时间序列不能用回归。在满足一定条件的情况下，回归在时间序列中非常好用。最常见的两种场景是：

1. 确定性趋势模型

Y_t = β₀ + β₁ * t + ε_t

把时间 t 作为解释变量，拟合线性趋势。这种方法适合描述有稳定增长的业务指标。比如某APP的日活用户数可以用时间趋势来近似。

2. 带季节性虚拟变量

Y_t = β₀ + β₁ * t + γ₂ * D₂ + γ₃ * D₃ + γ₄ * D₄ + ε_t

其中 D₂、D₃、D₄ 是季度虚拟变量（以 Q1 为基准），这样可以同时捕捉趋势和季节效应。这种方法在经济数据中非常常用，比如用回归做季度调整。

6.2 伪回归的根源

伪回归发生的根本原因是：当两个序列都是非平稳时，OLS 估计量不再具有一致性，t 统计量和 F 统计量的分布完全偏离了标准分布。简单说——你查 t 分布表得到的 p 值是错的。

伪回归的几个典型特征：

  • R² 异常高（通常 > 0.8）
  • Durbin-Watson 统计量很低（< 0.5）
  • 回归系数的 t 统计量都很显著
  • 但残差有明显的自相关
快速识别伪回归：如果回归的 R² 大于 Durbin-Watson 统计量，你很可能遇到了伪回归！这个经验法则叫"Granger-Newbold 准则"，在很多实证论文中被引用。

6.3 Durbin-Watson 检验

Durbin-Watson 检验（简称 DW 检验）专门用来检测回归残差的一阶自相关：

DW = Σ(ε_t - ε_{t-1})² / Σ(ε_t)²

DW 值的范围是 0 到 4：

  • DW ≈ 2——残差无自相关（理想状态）
  • DW < 2——正自相关（残差中还有结构）
  • DW > 2——负自相关（较少见）
  • DW 接近 0——严重正自相关，伪回归嫌疑极大

每个 DW 值都有一个对应的 p 值范围，实际使用中查表即得。在 Python 中可以直接调用 statsmodels 计算。

6.4 Cochrane-Orcutt 修正

如果发现残差存在自相关，最简单的修正方法是 Cochrane-Orcutt 估计。它的思路分两步：

  1. 先用 OLS 估计回归，计算残差，然后估计残差的一阶自相关系数 ρ
  2. 用 ρ 对原始变量做"准差分"变换：Y*_t = Y_t - ρ * Y_{t-1}，X*_t = X_t - ρ * X_{t-1}
  3. 在变换后的数据上重新做 OLS

这个方法的直觉是：先把残差中的自相关"剥离"掉，再用干净的数据做回归。

import numpy as np
import pandas as pd
from statsmodels.regression.linear_model import OLS
from statsmodels.tsa.stattools import acf

# 假设 y 和 x 是两个 Series
# 第一步：OLS 回归
model = OLS(y, x)
result = model.fit()
residuals = result.resid

# 第二步：估计残差的一阶自相关系数
rho = acf(residuals, nlags=1)[1]

# 第三步：准差分变换
y_star = y[1:] - rho * y[:-1].values
x_star = x[1:] - rho * x[:-1].values

# 第四步：重新回归
model_corrected = OLS(y_star, x_star)
result_corrected = model_corrected.fit()
print(result_corrected.summary())

6.5 在回归中包含趋势和季节

一种更简单的避免伪回归的策略是：在回归中显式包含趋势项和季节项，从而使残差变得平稳。

成分实现方式示例公式
线性趋势添加时间索引项 tY = β₀ + β₁t + ...
二次趋势添加 t²Y = β₀ + β₁t + β₂t² + ...
季节虚拟变量为 m 个季节创建 m-1 个 0/1 变量Y = ... + γ₂D₂ + γ₃D₃ + γ₄D₄
傅里叶项用 sin/cos 捕捉平滑季节Y = ... + Σ(α sin + β cos)

傅里叶项（Fourier terms）对季节性特别长的场景很有效。比如预测每日数据，如果用虚拟变量需要 364 个，而用傅里叶项可能 5~10 对 sin/cos 就能捕捉到年度模式。这在后面的 Prophet 模型中会大量用到。

实践中的忠告：在时间序列回归中，永远记住三步曲：

  1. 回归前：检验每个变量的平稳性（ADF + KPSS）
  2. 回归后：检查残差是否白噪声（DW 检验 + Ljung-Box）
  3. 如果不满足：要么差分，要么加入趋势项，要么用 Cochrane-Orcutt 修正

这三步走完，你的回归结果才有可信度。

6.6 什么时候回归优于其他方法

时间序列回归在以下场景特别有用：

  • 你需要解释而不是仅仅预测——回归系数有明确的经济含义
  • 有多个外生变量（external regressors）——比如用天气、节假日、促销活动来解释销售额
  • 需要做政策评估或因果推断——比如"广告投入增加10%带来多少销售额提升"

但如果你的目标纯粹是预测，而且数据中主要的结构是自相关和季节性，后面要学的 ARIMA 模型通常比回归方法更高效。

总结一下：时间序列回归是一把双刃剑——用好了可以解释变量之间的关系，用不好就会被伪回归欺骗。关键就是记住三条：平稳性检查、残差诊断、适当修正。下一章我们将彻底转换视角，从时域跨入频域，用频谱分析来发现时间序列中隐藏的周期性信号。你会看到，同样的数据在"频率"这个视角下，可以呈现出完全不同的面貌。
