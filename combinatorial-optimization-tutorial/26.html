开篇：从"算"到"学"的范式转变

大家好，我是你们的讲师。在前面的25章中，我们花了大量篇幅学习各种算法——分支定界、切割平面、列生成、模拟退火……这些东西有一个共同点：它们都是在"算"——给定一个问题实例，算法从零开始搜索、枚举、迭代，最后给出答案。这个范式的隐含假设是：每一次求解都是独立的，上一个实例的经验对下一个实例毫无帮助。

但让我们停下来想一想真实世界的场景。京东物流每天要调度数万个配送订单。这些订单的分布虽然每天略有不同，但远非完全随机——城市的路网结构是固定的，订单密度的高峰区域是相对稳定的。如果你每天用Gurobi从头求解每一个调度问题，你实际上是在反复解决一系列结构高度相似的MIP问题。这难道不是巨大的浪费吗？

2020年，Google Research发表了一篇令人印象深刻的论文——《Attention, Learn to Solve Routing Problems!》。作者训练了一个基于注意力机制的神经网络，在100节点的TSP实例上达到了接近最优的解质量，但推理时间只有传统求解器的千分之一。这篇论文点燃了神经组合优化（Neural Combinatorial Optimization, NCO）这个方向的热情。三年前我在一个运筹学会议上遇到一位老教授，他听完这个报告后感叹："我做了40年的TSP算法，没想到有一天会让神经网络来教我。"

核心洞察：神经组合优化的基本思想不是让神经网络"搜索"解空间，而是让神经网络学习解空间的结构。当同一分布下的问题被反复求解时，神经网络能从大量训练数据中提取出问题实例之间的共性模式，从而实现"看一眼就知道大概怎么走"的直觉推理。这和AlphaGo学习下棋而不是穷举博弈树，是同一个道理。

在本章中，我们将深入探讨神经组合优化的三个核心问题：如何用神经网络表示组合优化问题的解？如何训练这样的网络？以及——最重要的——当前最前沿的2026年技术能做到什么程度？

为什么用神经网络解组合优化

在花时间学习具体技术之前，我们需要先回答一个根本问题：为什么神经网络解决组合优化问题是有意义的？这不是显而易见的——毕竟前25章我们学了那么多"传统"方法，而且效果已经很好了。

传统方法的瓶颈

传统精确算法和启发式方法的共同瓶颈在于：每次求解都是冷启动。对于一个新的TSP实例，分支定界从根节点开始搜索，模拟退火从随机解开始迭代——过去求解的一万个TSP实例对当前实例没有任何帮助。这就好比一个围棋选手每次下棋都重新学习规则，而不是依靠积累的棋感。

当然，求解器确实有一些"热启动"机制（如MIP start），但那是针对同一个问题实例的修改版本，而非从同分布的不同实例中学习。

神经网络的优势

神经网络带来的是模式识别驱动的瞬时推理。在推理阶段，一个训练好的网络可以在毫秒级别给出一个解（或者一个高质量的解起点），因为前向传播只是矩阵乘法。其计算复杂度与问题规模近似线性或二次关系，而非传统算法的指数或超多项式关系。

下表总结了传统方法与神经方法的对比：

维度 传统精确/启发式算法 神经组合优化
每次求解 冷启动，从零开始 利用预训练知识，瞬时推理
推理速度 秒到小时级（取决于规模） 毫秒到秒级
解质量保证 最优/有界gap（精确算法） 无保证，依赖训练分布
泛化能力 对任意实例适用 仅在训练分布内表现好
计算资源 求解时消耗CPU 训练时消耗GPU，推理时轻量
适合场景 通用求解、一次性问题 重复求解同分布实例

神经组合优化并不是要"取代"传统方法，而是在传统方法不擅长的场景中提供补充——特别是那些需要实时响应且问题实例来自同一数据分布的应用。比如外卖骑手实时派单、自动驾驶中的路径重规划、在线广告的实时竞价优化等。

一个重要的界限：请不要对NCO抱有不切实际的期望。截至2026年，没有任何一个NCO方法能在通用性上挑战Gurobi或CPLEX。神经方法擅长的是"特定分布下的快速近似解"，而传统求解器擅长的是"任意实例下的最优/有界解"。聪明的工程师会把两者结合起来——先用神经网络给出一个热启动解，再用传统求解器精调。

一个类比：直觉 vs 逻辑

我用一个类比来帮助大家理解神经组合优化的定位。人类解决复杂决策问题时，通常使用两种思维模式：系统1（快思考）和系统2（慢思考）。系统1是直觉式的、快速的、自动化的——比如你一眼就能认出朋友的脸。系统2是分析式的、缓慢的、需要主动推理的——比如解一道微积分题。

传统算法是纯粹的系统2——每一步都精确计算。神经网络提供的是系统1——通过大量经验积累形成的"直觉"。最好的决策者是两个系统协同工作：先依靠直觉快速缩小搜索范围，再用逻辑推理精细验证。神经组合优化的最终目标，就是为组合优化问题构建这样一个"系统1"。

Pointer Network与Attention Model

理解了"为什么"之后，我们来学习"怎么做"。神经组合优化最早期的突破来自一个优雅的架构设计——让神经网络学会"指向"输入序列中的元素，从而自然地输出一个排列。

Pointer Network的起源

2015年，Vinyals等人在NIPS上发表了Pointer Networks。传统的Seq2Seq模型（如机器翻译中使用的）有一个根本局限：输出词汇表是固定的。但在组合优化中，输出必须从输入中选择——比如TSP中，每一步选择的"下一个城市"必须是输入中给定的城市之一，输出词汇表随问题规模变化。

Pointer Network的核心创新是：解码器不再从一个固定词汇表中选择输出，而是通过注意力机制"指向"输入序列中的某个位置。具体来说：

编码器：LSTM处理输入序列（城市坐标）→ 隐状态 h₁,...,hₙ
解码器：LSTM生成解码状态 sₜ
注意力：uⱼᵗ = vᵀ tanh(W₁hⱼ + W₂sₜ)  (j = 1,...,n)
指向：p(yₜ | y₁,...,yₜ₋₁, X) = softmax(uᵗ)

这里最关键的是：softmax的输出维度等于输入序列的长度n，所以模型可以"指向"任意一个输入位置。而且由于注意力权重依赖于当前解码状态，模型可以学会考虑已经访问过的城市，避免重复。

Pointer Network在20节点的TSP上取得了接近最优的结果。但它的局限也很明显——编码器使用LSTM，难以捕捉城市之间的空间关系；而且它没有使用自注意力机制来建模所有城市之间的成对关系。

Attention Model：Transformer登场

2020年，Wouter Kool等人的Attention, Learn to Solve Routing Problems!论文将Pointer Network升级为了Attention Model (AM)。核心改进是用Transformer编码器替代了LSTM编码器：

输入：N个城市的坐标 (xᵢ, yᵢ)
嵌入层：Linear(2 → 128) + 位置编码

Transformer编码器（3层×8头注意力）：
    MHA(多头注意力) → Skip-Connect → LayerNorm → FF → Skip-Connect → LayerNorm
    → 输出节点嵌入 h₁,...,hₙ

解码器（单步展开）：
    上下文嵌入 = mean(h₁,...,hₙ) + h_first + h_last(已访问)
    对于每一步 t：
        用注意力机制计算当前context与所有未访问城市的相关性
        softmax得到选择概率 → 采样或贪心选择下一个城市

这个架构带来的提升是显著的。在100节点TSP上，Attention Model的求解质量（约7.8%的gap）远优于Pointer Network。更重要的是，Transformer的自注意力机制天然适合建模组合优化问题——每个城市要"关注"所有其他城市的信息才能做出好的决策，而这正是多头注意力最擅长的。

一个设计直觉：为什么Transformer比LSTM更适合组合优化？因为组合优化的解是一个全局决策——选择第t个城市时，你需要理解所有城市之间的空间关系，而不仅是已访问城市的信息。LSTM的序列性天然限制了信息流动的方向，而Transformer的自注意力让所有节点之间的信息可以在单层内直接交互。这就好比LSTM是逐户传话，Transformer是开全体大会——信息传递的效率完全不同。

TSP的序列到序列建模

在Attention Model的框架中，TSP被建模为一个条件序列生成问题：给定N个城市的坐标，生成一个城市的排列（即访问顺序），使得总长度最小。这个视角的精妙之处在于，它将一个NP-hard的组合优化问题转化为了一个有监督或无监督的序列学习问题——而后者的工具在过去十年中已经非常成熟。

但这并不意味着问题变简单了。序列生成和真正的组合优化之间存在一个根本差异：在TSP中，两个不同排列可能对应完全相同的环（只是起点不同或方向相反），但序列模型会把它们当作完全不同的输出。这是NCO面临的一个基础性挑战，目前仍然没有完美的解决方案。

强化学习训练范式

有了网络架构，下一个问题是：如何训练它？这里有一个重要的观察：对于TSP等组合优化问题，我们通常没有"正确答案"（最优解）作为监督信号。即使有，用有监督学习去拟合最优解也是有问题的——最优解通常是唯一的，但网络的输出是一个概率分布，直接拟合会抹杀探索的可能性。

REINFORCE with Baseline

Attention Model的训练使用了策略梯度方法（Policy Gradient），具体来说是REINFORCE算法。核心思想很直接：让网络自己生成解，然后根据解的质量（旅行总长度）调整网络参数——好的被鼓励，差的被惩罚。

REINFORCE的训练目标可以写作：

∇J(θ) = E_{π ~ p_θ(·|s)} [ (L(π) - b(s)) · ∇log p_θ(π|s) ]

其中：
  π = 网络生成的解（城市排列）
  L(π) = 解π的总旅行长度
  b(s) = baseline（当前策略的平均解质量）
  p_θ(π|s) = 网络生成解π的概率

关键点：如果当前解优于baseline（L(π) < b(s)），
梯度方向会推动网络参数增加生成这类解的概率。

baseline的作用是减小梯度估计的方差。如果没有baseline，即使是一个非常好的解，如果L(π)是正数（所有解的长度都是正的），梯度方向也会被统一偏移。baseline相当于"居中"了奖励信号——我们关心的是"这个解比平均好多少"，而不是它的绝对值。

Rollout Baseline与Self-Critical Training

Attention Model中使用的是一种被称为rollout baseline的技巧：维护一个"老版本"的策略网络作为baseline，定期用当前策略替换它。具体做法是：

  • 每个训练batch，当前策略采样生成多个解
  • 用老策略（rollout baseline）对同一批实例贪心生成解
  • 优化信号 = 当前策略的解长度 - 老策略的解长度
  • 每若干epoch，如果当前策略显著优于老策略，更新baseline

这个方法的优点是baseline的质量随着训练自动提升，不需要额外训练一个critic网络。但缺点是计算量大——每步需要两次前向传播（当前策略和baseline）。

后来出现的自批评训练（Self-Critical Training）是一个更优雅的替代方案：使用同一策略网络，但用贪心解码的输出作为baseline，用采样解码的输出作为训练样本。这样只需一次前向传播，而且baseline是实时更新的。

实践建议：如果你的计算资源有限，优先尝试self-critical training。它比rollout baseline收敛速度略慢，但每步计算量减半，且实现更简单。在我们的实验中，self-critical在100节点TSP上达到了与rollout baseline几乎相同的最终解质量。

训练信号的设计艺术

NCO训练中最容易被忽视但至关重要的，是奖励/损失函数的设计。对于TSP，损失是路径长度——这很直接。但对于有约束的问题（如带时间窗的VRP），奖励函数需要同时考虑目标值和约束违反程度。常见的做法是将约束违反作为惩罚项加入奖励：

R(π) = -[C(π) + λ · V(π)]

其中 C(π) 是原始目标（如距离），V(π) 是约束违反惩罚，
λ 是惩罚系数。λ的大小决定了"满足约束"相对于"优化目标"的重要程度。

这里有一个经典的踩坑经验：惩罚系数λ的设置非常敏感。设得太小，网络学会了无视约束；设得太大，网络变得极度保守，不敢探索有风险但可能更好的解。我见过一个团队花了两周时间调λ——最后发现用一个动态的、随训练进程逐渐增大的λ调度策略效果最好。这个技巧和强化学习中的课程学习（Curriculum Learning）有异曲同工之妙。

GNN+传统方法融合

2024-2026年，神经组合优化的最前沿方向从"端到端求解"转向了神经引导的传统方法（Neural-Guided Traditional Methods）。这个思路更务实：用神经网络来辅助传统算法，而不是取代它们。

GNN学习分支策略（PlanB&B）

分支定界算法的核心是分支策略——选择哪个变量先分支。传统上，最常用的策略是"最伪布尔值"（Pseudo-Cost）或"强分支"（Strong Branching）。但强分支虽然效果好，代价极高（需要求解大量候选子问题的松弛LP）。

2024年的PlanB&B方法提出：用图神经网络（GNN）来模仿强分支策略。GNN的输入是当前节点的LP松弛信息（变量值、对偶乘子、约束系数矩阵等），输出是每个变量被选为分支变量的得分。通过behavioral cloning（行为克隆），GNN学会了在极低的计算成本下逼近强分支的决策质量。

在实际测试中，PlanB&B将分支决策的计算时间降低了80%，同时保持了与强分支几乎相同的节点数量。这意味着分支定界可以在相同时间内探索更多节点——或者换句话说，在更短时间内达到最优性证明。

一个需要警惕的问题：GNN分支策略面临分布偏移（distribution shift）的挑战。训练数据来自某些特定问题实例，但当应用到完全不同类型的问题实例时，GNN的决策质量可能急剧下降。这和深度学习中的域适应（domain adaptation）问题是同一个逻辑。所以在实际使用中，最好为每类问题单独训练一个分支策略网络，而不是期待一个"万能"的分支网络。

RL-SPH：100%可行率的原始启发式

2025年的RL-SPH（Reinforcement Learning-based Search for Primal Heuristics）方法是另一个令人印象深刻的融合方向。它的目标是用强化学习训练一个策略网络，在分支定界的搜索过程中快速生成原始可行解。

这个方法的独特之处在于它明确保证了100%的可行率——网络生成的解总是满足所有约束。这是通过一个"可行投影层"实现的：网络的输出被映射到最近的可行解。在超过300个MIPLIB实例的测试中，RL-SPH在平均求解时间上比大邻域搜索（LNS）启发式提升了35%。

RL-SPH的成功证明了：即使神经网络没有完美的最优解，只要能稳定提供可行的初始解，就能显著加速求解器的原始-对偶收敛过程。

神经局部搜索（Neuro-PLS）

2026年的Neuro-PLS（神经并行走廊局部搜索）将GNN与局部搜索结合。它的想法是：用GNN评估当前解的每个邻域移动（neighborhood move）的潜力，而不是盲目地探索所有邻域解。

在标准局部搜索中，每一步需要评估O(n²)个可能的交换操作（在TSP中是2-opt交换）。Neuro-PLS的GNN可以将候选移动的数量降低到O(k)（k ≪ n²），因为网络学会了快速过滤掉那些"明显不值得尝试"的移动。在车辆路径问题（VRP）上，Neuro-PLS在相同时间内找到了比标准局部搜索好3-5%的解。

2026最新突破

2026年是神经组合优化领域令人振奋的一年。几个突破性成果将NCO的边界推到了之前难以想象的地方。

渐进训练：从小规模到大规模

NCO长期面临的一个致命问题是规模泛化——在20节点TSP上训练的网络，在100节点上几乎完全失效。这是因为注意力模式随规模变化很大。2026年初，一种称为渐进训练（Progressive Training）的方案取得了突破。

方法很简单但有效：

  • 阶段1（小规模监督预训练）：在50-100节点的小规模TSP实例上，用有监督学习（以Concorde求解的最优解为标签）训练一个基础模型。这一阶段确保网络学到的"基础模式"是正确的。
  • 阶段2（大规模强化学习微调）：在1000-10000节点的大规模实例上，用REINFORCE做强化学习微调。由于基础模型已经具备基本能力，RL只需要调整网络来适应更大规模的注意力范围。

实验结果表明，渐进训练方法在10000节点的TSP实例上达到了约12%的最优性gap——这听起来不高，但考虑到之前最好的NCO方法在1000节点以上就完全失效了，这是一个巨大的进步。而且推理时间从传统算法的数小时降低到了秒级。

启示：渐进训练的成功说明了一个重要道理——NCO的规模泛化问题不是"不可解决的"，而是"需要正确的训练策略"。先学好小规模下的基本模式，再逐步扩展到大规模，这和人类学习复杂技能的方式惊人地一致：先学加减法，再学微积分。

Decision Transformer离线NCO

2026年的另一项重大突破来自Decision Transformer的离线学习范式。将组合优化的求解过程视为一个序列决策问题：每一步（选哪个节点/哪个变量）对应一个token，整个求解轨迹（从初始状态到最终解）对应一个token序列。

Decision Transformer将这一轨迹建模为条件序列生成：给定目标（例如"找到一个总长度小于100的路径"），模型生成一个完整的求解轨迹。关键是，训练数据来源于专家求解轨迹——用Concorde或Gurobi等工具求解决策树中记录的搜索路径，作为监督信号。

实验表明，在中等规模问题（100-500节点）上，Decision Transformer NCO模型超越了其训练数据中的教师启发式——这意味着它学到了比人类设计的启发式规则更好的策略。这在NCO领域是一个里程碑式的结果，因为它证明神经网络可以超越其训练数据中使用的传统方法，而不是简单地模仿它们。

LLM辅助泛化：零样本到达100000节点

最激动人心的成果来自大语言模型（LLM）辅助的NCO泛化。2026年的测试时投影学习（Test-Time Projection Learning）方法将基础模型与大语言模型的推理能力结合：

  • 基础模型：在1000节点规模上预训练的GNN策略网络
  • LLM分析器：在推理时，LLM分析当前实例的全局结构（用自然语言描述TSP实例的特征，如"城市分布呈现东南-西北条带状，其中东南方向聚类密度较高"），然后输出一个投影参数（projection parameter），将当前实例调整为基础模型可处理的格式
  • 迭代精化：基础模型的输出再反馈给LLM进行循环精化

这种方法在100000节点规模的问题上，仅需3-5次LLM调用循环即可达到有竞争力的解质量。虽然计算开销仍然很高（LLM推理较慢），但这个结果证明了推理时自适应是克服NCO规模泛化问题的一个有效方向。

展望：LLM+NCO的交叉是一个刚刚起步但潜力巨大的方向。目前LLM主要被用作"元控制器"——分析问题结构并协调多个子模型。随着LLM推理成本的下降和推理速度的提升，这种混合架构可能在2-3年内成为工业级NCO系统的主流范式。

本章小结

在这一章中，我们学习了神经组合优化从2015年Pointer Network到2026年LLM辅助泛化的完整演进路径：

  • 神经网络解决组合优化的核心动机——从重复求解相同分布的问题实例中学习模式，实现瞬时推理；
  • Pointer Network到Attention Model的架构演进——从LSTM到Transformer，从固定词汇表到动态指向；
  • REINFORCE with baseline的强化学习训练范式——rollout baseline与self-critical training的对比；
  • 2024-2026年GNN与传统方法的四大融合方向——PlanB&B、RL-SPH、Neuro-PLS、Learn2Cut；
  • 2026年的三项前沿突破——渐进训练达到10000节点、Decision Transformer超越教师启发式、LLM辅助泛化达到100000节点。

神经组合优化是一个发展极快的领域，几乎每个月都有新的突破。但请记住，NCO并不"取代"传统方法，而是与之互补。在第27章中，我们将从神经网络的视角转向硬件加速的视角——看看GPU加速、Ising机和量子计算如何从另一个维度突破组合优化的计算瓶颈。你会看到，异构计算（CPU+GPU+量子）的协同可能是未来十年组合优化最激动人心的发展方向。
