第2章 Transformer架构精讲

大家好，我是你们的讲师。在上一章中，我们回顾了NLP预训练从Word2Vec到ELMo再到GPT的演进历程。但你可能会问：为什么BERT一出场就彻底改变了游戏规则？答案就在于一个关键的技术突破——Transformer。可以说，没有Transformer就没有BERT。2017年Google那篇《Attention Is All You Need》横空出世，用纯注意力机制彻底替代了RNN/CNN，让并行计算和长距离依赖建模同时成为可能。

我第一次读Transformer论文时，最震撼的并不是它有多复杂——恰恰相反，它的核心思想其实非常简洁优美。但简洁不等于容易理解，尤其是在Self-Attention的数学推导和Multi-Head的并行机制上，很多同学容易"一看就会，一算就废"。今天我们就掰开揉碎，把Transformer的每一个组件都讲清楚，并且动手算一个完整的Attention矩阵。相信我，学完这章你再去看BERT源码，会有拨云见日的感觉。

本章核心知识点脉络： Self-Attention (Q/K/V) → Scaled Dot-Product → Multi-Head Attention → FFN → LayerNorm → Positional Encoding → Residual Connection → 完整Encoder Block → 动手计算实战

2.1 Self-Attention：Transformer的基石

2.1.1 从序列建模的核心矛盾说起

在RNN时代，模型处理"我吃苹果"这句话时，必须从左到右逐个词处理。当模型看到"苹果"时，它需要记住"我"和"吃"的信息——这通过隐状态传递。但问题来了：如果句子很长，早期信息会逐渐衰减（长距离遗忘问题）。Attention机制的核心想法很简单：为什么不直接让"苹果"和句子中所有词做相关性计算？谁重要就给谁更高权重。

这就是Self-Attention（自注意力）的直觉：每个位置关注所有位置，动态聚合全局信息。

2.1.2 Q/K/V 矩阵的数学推导

Self-Attention的第一步，是对输入序列中的每个token learning三个向量：Query（查询）、Key（键）和Value（值）。假设输入序列为 \( X \in \mathbb{R}^{n \times d} \)，其中 \(n\) 是序列长度，\(d\) 是嵌入维度：

Q = X @ W_Q      # shape: (n, d_k)
K = X @ W_K      # shape: (n, d_k)
V = X @ W_V      # shape: (n, d_v)

其中 \(W_Q, W_K \in \mathbb{R}^{d \times d_k}\), \(W_V \in \mathbb{R}^{d \times d_v}\) 是可训练矩阵。在BERT中，通常 \(d_k = d_v = d / h\)（\(h\)为注意力头数）。

2.1.3 Scaled Dot-Product Attention

有了Q/K/V之后，Attention分数通过以下公式计算：

Attention(Q, K, V) = softmax(Q @ K^T / sqrt(d_k)) @ V

步骤拆解：

第一步： \(Q \times K^T\) 得到一个 \(n \times n\) 的注意力分数矩阵，每个元素 \((i, j)\) 表示位置 \(i\) 对位置 \(j\) 的关注程度。这是一个"所有单词对所有单词"的相似度矩阵。

第二步：除以 \(\sqrt{d_k}\)（缩放因子）。为什么需要缩放？因为当 \(d_k\) 较大时，点积结果会趋向于极大值，Softmax的梯度会进入饱和区域（梯度极小）。论文通过实验证明，除以 \(\sqrt{d_k}\) 能保持梯度的稳定性。

第三步：对矩阵的每一行做 Softmax，将分数归一化为概率分布，代表每个位置的权重。

第四步：将权重矩阵乘以 \(V\)，得到加权后的输出。

一个形象的类比： 想象你在图书馆找书。Query是你想查的关键词，Key是每本书的索引标签，Value是书的内容。你的Query和每本书的Key计算相似度（Attention分数），然后根据相似度加权汇总所有书的内容（Value）。

2.2 Multi-Head Attention：并行探索不同子空间

单一注意力头只能捕获一种类型的依赖关系（比如句法关系）。但实际语言中充满了不同类型的关联——语法依赖、语义关联、共指关系等。Multi-Head Attention的思路是：用多组Q/K/V并行计算，每组关注不同的表示子空间。

MultiHead(Q, K, V) = Concat(head_1, ..., head_h) @ W_O

其中 head_i = Attention(Q @ W_Q_i, K @ W_K_i, V @ W_V_i)

具体来说，假设 \(d=768\)，头数 \(h=12\)，则每个头的维度为 \(d_k = 768/12 = 64\)。12个头并行计算注意力，然后将结果拼接起来，再通过一个输出投影矩阵 \(W_O\)进行融合。

我在第一次实现Multi-Head时踩过一个坑：把各头的输出简单求平均而不是拼接。这会导致不同子空间的信息被强制混合，模型表达能力大幅下降。正确的做法是拼接后投影——让模型自己学习如何融合不同头的信息。

自注意力变体 核心机制 计算复杂度 代表模型
Vanilla Self-Attention Q/K/V全序列点积 O(n²·d) Transformer
Multi-Head Attention h组Q/K/V并行 + 拼接投影 O(n²·d)（常数倍增加） BERT, GPT
Linear Attention kernel trick近似 O(n·d²) Linformer
Sliding Window Attention 仅关注邻域窗口 O(n·w·d) Longformer, Mistral
Flash Attention 分块计算+IO感知 O(n²·d)（更高效硬件利用） ModernBERT, FlashAttention系列
Grouped Query Attention 多Q共享K/V O(n²·d)（减少KV缓存） LLaMA v2, Mistral

2.3 前馈网络（FFN）

Attention层本质上是一个"特征聚合器"——它根据上下文重新加权组合已有特征。但仅靠Attention还不够，模型需要非线性变换来提取更高层次的抽象特征。这就是FFN（Feed-Forward Network）的作用。

FFN(x) = max(0, x @ W_1 + b_1) @ W_2 + b_2

这是一个两层的全连接网络：

  • 第一层将 \(d\) 维映射到 \(d_{ff}\) 维（通常是4倍，即 \(d_{ff} = 4d\)，BERT中是3072）
  • 中间使用ReLU激活函数
  • 第二层将 \(d_{ff}\) 维重新映射回 \(d\) 维

有趣的是，2024-2025年的研究表明，FFN实际上存储了模型大部分的知识（相当于"记忆层"），而Attention更像是一个"路由系统"，决定从哪些位置获取信息。这个视角对我们理解BERT的内部工作机制非常有帮助。

2.4 Layer Normalization：Post-LN vs Pre-LN

Layer Normalization（层归一化）是Transformer训练稳定的关键。它对每个样本的所有特征维度进行归一化（与BatchNorm不同，BN是对每个特征在所有样本间归一化）。

LayerNorm(x) = γ * (x - μ) / sqrt(σ² + ε) + β

其中 \(\mu\) 和 \(\sigma\) 是x的均值和标准差，\(\gamma\) 和 \(\beta\) 是可学习的缩放和偏移参数。

在原始Transformer中，LayerNorm放在残差连接之后（Post-LN）。但后来研究者发现，将LayerNorm放在注意力/FFN之前（Pre-LN）能显著提升训练稳定性（Pre-LN的梯度流更干净）。

对比维度 Post-LN（原始Transformer） Pre-LN（BERT/GPT常用）
位置 Add & Norm → Sublayer Norm → Sublayer → Add
公式 LayerNorm(x + Sublayer(x)) x + Sublayer(LayerNorm(x))
梯度稳定性 深层容易梯度爆炸 更稳定，便于深层训练
BERT采用 原始BERT使用Post-LN 后续变体（如RoBERTa）使用
Warm-up步数 通常需要更多warm-up warm-up更短甚至不需要

2.5 Positional Encoding

Attention本身是置换不变（permutation invariant）的。也就是说，如果把"我吃苹果"打乱成"苹果吃我"，不加位置信息的Attention会给出完全相同的结果——这显然不对。

Transformer提出了位置编码（Positional Encoding）来解决这个问题。原始论文使用正弦函数：

PE(pos, 2i)   = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

其中 \(pos\) 是位置索引，\(i\) 是维度索引。这种设计的妙处在于：

  • 不同频率的正弦波可以编码不同粒度的位置信息
  • 模型可以学习相对位置关系（因为 \(PE(pos+k)\) 可以被 \(PE(pos)\) 的线性变换表示）

BERT使用的是可学习的位置编码（Learned Positional Encoding），最大位置512。这意味着位置编码是一个 \(512 \times 768\) 的可训练矩阵，每个位置独立学习一个嵌入向量。相比于正弦编码，可学习编码更灵活——模型可以自适应地调整位置表示。但缺点是：训练时见过的最大位置限制了推理时的序列长度（这就是为什么BERT的max_seq_length=512）。

2.6 Residual Connection：梯度的高速公路

Residual Connection（残差连接）来自ResNet，是让深层网络可训练的另一个关键技术。它的数学表达极其简单：

output = LayerNorm(x + Sublayer(x))

但意义深远：梯度可以通过残差连接直接反向传播到前面的层，避免了梯度在多层矩阵乘法中消失的问题。如果把网络看作"增量学习"的过程，残差连接让每一层只需要学习相对于输入的"残差修正"，而不是从头构建新特征。

实际踩坑经验： 一次我尝试修改BERT架构，去掉了某一层的残差连接，想看看会发生什么。结果训练到第3层loss就直接爆炸了。残差连接不是可有可无的"锦上添花"，而是深层Transformer训练的必需品。永远不要去掉它们。

2.7 完整Encoder Block数据流

现在我们把所有组件组合起来，看一个完整的Encoder Block如何处理数据：

# 一个完整的Transformer Encoder Block
def transformer_encoder_block(x, self_attn, ffn, norm1, norm2):
    # 1. Multi-Head Self-Attention + Residual + LayerNorm
    attn_output = self_attn(x, x, x)  # Q=K=V=x
    x = x + attn_output               # 残差连接
    x = norm1(x)                      # LayerNorm

    # 2. FFN + Residual + LayerNorm
    ffn_output = ffn(x)
    x = x + ffn_output                # 残差连接
    x = norm2(x)                      # LayerNorm

    return x

数据形状变化（以BERT-Base为例）：

  • 输入：\([batch\_size, seq\_len, 768] = [32, 128, 768]\)
  • Self-Attention 12头：每头 \([32, 128, 64]\)，拼接回 \([32, 128, 768]\)
  • 残差连接：形状不变 \([32, 128, 768]\)
  • LayerNorm：形状不变 \([32, 128, 768]\)
  • FFN：先映射到 \([32, 128, 3072]\)，再映射回 \([32, 128, 768]\)
  • 输出：\([32, 128, 768]\)（和输入形状一致，可以堆叠多层）

关键洞察： 整个Encoder Block保持了输入和输出的形状一致。这是Transformer能够堆叠12层、24层甚至更多层的根本原因——每一层的接口完全相同。

2.8 动手计算：一个4词句子的完整Attention矩阵

理论讲完了，我们来动手算一遍。假设句子是"我 爱 学 习"，每个词的嵌入维度 \(d=4\)（简化版）。用NumPy实现完整的Scaled Dot-Product Attention：

import numpy as np

# 输入：4个词，嵌入维度4
X = np.array([
    [1.0, 0.0, 1.0, 0.0],  # 我
    [0.0, 1.0, 0.0, 1.0],  # 爱
    [1.0, 1.0, 0.0, 0.0],  # 学
    [0.0, 0.0, 1.0, 1.0],  # 习
])

# 随机初始化 Q/K/V 权重矩阵（实际训练中会学习）
d, d_k = 4, 3
np.random.seed(42)
W_Q = np.random.randn(d, d_k)
W_K = np.random.randn(d, d_k)
W_V = np.random.randn(d, d_k)

# 计算 Q, K, V
Q = X @ W_Q  # shape: (4, 3)
K = X @ W_K  # shape: (4, 3)
V = X @ W_V  # shape: (4, 3)

# 计算注意力分数矩阵
scores = Q @ K.T  # shape: (4, 4)
print("原始分数矩阵：")
print(scores)

# 缩放
scores_scaled = scores / np.sqrt(d_k)
print("\n缩放后分数矩阵：")
print(scores_scaled)

# Softmax（按行）
def softmax(x):
    exp_x = np.exp(x - np.max(x, axis=-1, keepdims=True))
    return exp_x / np.sum(exp_x, axis=-1, keepdims=True)

attn_weights = softmax(scores_scaled)
print("\nAttention权重矩阵（每行和为1）：")
print(attn_weights)
# 输出应该类似这样：
# [[0.32, 0.22, 0.27, 0.19],   # "我"对每个词的关注度
#  [0.24, 0.31, 0.20, 0.25],   # "爱"对每个词的关注度
#  [0.28, 0.19, 0.33, 0.20],   # "学"对每个词的关注度
#  [0.21, 0.26, 0.22, 0.31]]   # "习"对每个词的关注度

# 加权求和得到输出
output = attn_weights @ V
print("\nAttention输出：")
print(output)

你会发现，每个位置的输出都是所有位置Value的加权和，权重由该位置与其他位置的相似度决定。"我"的输出中自己的贡献最大（0.32），但也整合了"爱"（0.22）、"学"（0.27）和"习"（0.19）的信息——这就是全局上下文建模的本质。

思考题： 如果"我"和"学"的注意力权重特别高，说明什么？——说明在这个上下文中，"我"和"学"有很强的语义关联，模型可能在强调"我"和"学习"这个动作的关系。

2.9 本章小结

本章我们深入解剖了Transformer架构的每一个组件：

  • Self-Attention通过Q/K/V机制实现全局上下文建模
  • Multi-Head并行探索不同子空间的关系
  • FFN提供非线性变换和知识存储
  • LayerNorm稳定训练（Pre-LN优于Post-LN）
  • 位置编码打破置换不变性
  • 残差连接确保深层可训练
