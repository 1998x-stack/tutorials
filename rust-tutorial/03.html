引子：那个被花括号"坑"了的Python程序员

我带过一个实习生，他Python写得非常溜。第一次写Rust时，他写了一个这样的函数：

fn add(a: i32, b: i32) -> i32 {
    a + b;
}

然后他发现这个函数一直返回()而不是a + b的结果。他盯着屏幕看了十分钟，狂敲键盘加return——才注意到原来是多了一个分号。分号在Rust里不是可有可无的装饰品，它决定了表达式和语句之间的天壤之别。

这个故事的教训是：Rust是表达式导向的语言，理解表达式 vs 语句是学好函数和控制流的第一步。

函数定义

fn关键字与参数

用fn关键字定义一个函数：

fn greet(name: &str) {
    println!("你好，{name}！");
}

fn main() {
    greet("小明");
}

每个参数都必须显式标注类型——Rust不会推断函数参数的类型。这是有意为之的设计：函数签名是API的一部分，类型标注让调用者一眼就知道该传什么。

返回值

函数返回值用->指定类型。有两种返回方式：

// 方式1：用return关键字（推荐在提前返回时使用）
fn early_return(x: i32) -> i32 {
    if x < 0 {
        return 0;
    }
    x * 2
}

// 方式2：用表达式（最后一个表达式的值就是返回值）
fn square(x: i32) -> i32 {
    x * x  // 没有分号！这就是返回值
}
表达式 vs 语句——Rust程序员必须分清的两个概念
  • 表达式（Expression）：产生一个值。比如 x + 1、if true { 5 } else { 6 }。表达式没有分号。
  • 语句（Statement）：执行一个动作但不产生值。比如 let y = 5;、println!("hi");。语句有分号。

这个区分看似简单，但初学Rust时最容易翻车——几乎所有"期望类型A却得到类型B"的编译错误，根源都在于不小心在表达式末尾加了分号。

看一个经典错误：

fn add(x: i32, y: i32) -> i32 {
    x + y;  // 错误！分号让表达式变成了语句，函数实际返回()
}
// 编译器报错：expected `i32`, found `()`

if / else

if的条件必须是bool类型——Rust不会像C语言那样自动把非零值当作true：

let number = 3;

if number < 5 {
    println!("小于5");
} else if number == 5 {
    println!("等于5");
} else {
    println!("大于5");
}

// 错误的写法：
if number {     // 编译错误！number是i32，不是bool
    // ...
}

if是表达式

Rust的if是表达式，这意味着你可以把它用在let绑定的右侧：

let condition = true;
let value = if condition { 42 } else { 0 };
println!("{value}");  // 42

这里的if分支必须返回相同类型的值——否则编译会报错：

let value = if condition { 42 } else { "hello" };
// 编译错误：if and else have incompatible types
代码风格：把if作为表达式赋值给变量时，建议保持if和else分支的代码行数在2行以内。如果逻辑太复杂，拆成独立的函数会更清晰。没有人喜欢看一个占据了20行的"表达式"。

循环

Rust提供了三种循环：loop、while、for。它们各有各的用武之地。

循环类型 适用场景 能否返回 示例
loop 无条件循环，直到手动break 可以（break带值） 服务器事件循环、重试机制
while 条件循环，每次前检查条件 不能 轮询状态、等待条件满足
for 遍历迭代器 不能 遍历数组/范围/集合

loop

loop无限循环，直到遇到break。它有一个独特的能力——break可以带出返回值：

let mut counter = 0;
let result = loop {
    counter += 1;
    if counter == 10 {
        break counter * 2;  // 返回20
    }
};
println!("{result}");  // 20

这个特性在你需要"先循环计算，然后把结果赋值给变量"时非常有用，避免了在外面再声明一个可变变量。

while

let mut n = 5;
while n > 0 {
    println!("{n}");
    n -= 1;
}
// 输出: 5 4 3 2 1

for in

这是Rust中最常用、最推荐的循环方式：

let arr = [10, 20, 30, 40];
for element in arr {
    println!("{element}");
}

// 范围语法（Range），左闭右开
for i in 0..5 {
    print!("{i} ");  // 0 1 2 3 4
}

// 包含右端点的范围
for i in 0..=5 {
    print!("{i} ");  // 0 1 2 3 4 5
}
一个真实的生产事故：我曾在一个网络服务中看到这样的代码——用while循环配合索引遍历一个Vec来查找数据。当数据量从几千涨到几百万时，这段代码成了性能瓶颈。原因很简单：while循环每次迭代都要做边界检查，而且无法利用CPU的向量化指令。改成for element in collection后，Rust的迭代器可以自动优化，性能提升了一个数量级。在Rust中，for通常是你的首选——它既是表达力的胜利，也是性能的胜利。

match基础

match是Rust中极其强大的控制流操作符。你可以把它想象成"switch的超级进化版"：

let number = 3;

match number {
    1 => println!("一"),
    2 => println!("二"),
    3 => println!("三"),
    _ => println!("其他数字"),  // _ 通配符匹配所有剩余情况
}

几个要点：

  • match的每个分支由模式 => 代码组成
  • 所有可能的情况必须穷尽（exhaustive）——编译器会检查你是否覆盖了所有可能的值
  • _是通配符模式，匹配一切剩余情况，类似其他语言中的default
  • match也是表达式，可以返回值
// match作为表达式
let grade = 85;
let level = match grade {
    90..=100 => "优秀",
    80..=89  => "良好",
    70..=79  => "中等",
    60..=69  => "及格",
    _        => "不及格",
};
println!("{level}");  // 良好

注意到这里用了..=范围模式，它是match中非常有用的语法糖。第7章我们会深入学习match的各种高级模式。

函数作为一等公民

Rust中函数是一等公民——你可以把函数赋值给变量，作为参数传递，甚至作为返回值返回。

函数指针（Function Pointer）

fn add_one(x: i32) -> i32 {
    x + 1
}

fn do_twice(f: fn(i32) -> i32, x: i32) -> i32 {
    f(x) + f(x)
}

fn main() {
    let result = do_twice(add_one, 5);
    println!("{result}");  // 12
}

fn(i32) -> i32是函数指针的类型签名，这告诉Rust：这个参数是一个函数，接受i32参数，返回i32。

函数作为参数的实际应用

一个更贴近真实场景的例子——写一个数组处理函数，把转换逻辑交给调用者决定：

fn map_arr(arr: &[i32], f: fn(i32) -> i32) -> Vec<i32> {
    arr.iter().map(|&x| f(x)).collect()
}

fn double(x: i32) -> i32 { x * 2 }
fn square(x: i32) -> i32 { x * x }

fn main() {
    let nums = [1, 2, 3, 4];
    let doubled = map_arr(&nums, double);
    println!("{:?}", doubled);  // [2, 4, 6, 8]

    let squared = map_arr(&nums, square);
    println!("{:?}", squared);  // [1, 4, 9, 16]
}

这种模式在真实项目中非常常见——标准库中很多方法（如map、filter、fold）都接受函数或闭包作为参数。第14章我们会深入闭包和迭代器，那时你会发现函数式编程在Rust中是多么自然。

趣味知识：Rust中函数指针fn(T) -> R和闭包Fn(T) -> R是两个不同的概念。函数指针实现了所有三种闭包trait（Fn、FnMut、FnOnce），因此任何需要闭包的地方你都可以传一个函数指针——但反过来不行。第14章会展开讲。
