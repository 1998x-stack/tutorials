10.1 深度网络的"天花板"

回想我们在上一章讨论的问题：VGG展示了深度带来的力量，但同时也遇到了瓶颈。2015年初，整个CV社区有一个悬而未决的问题：为什么更深的网络反而更难训练？

你可能会直觉地认为：更多的层意味着更强的表示能力，所以应该效果更好。但实验给出了完全相反的结论。Kaiming He和他的微软研究院团队做了一个对比实验：在CIFAR-10上训练一个20层的CNN和一个56层的CNN。按理说，56层的网络应该至少不比20层的差——毕竟20层的解空间是56层解空间的一个子集（56层的网络可以先退化成20层的等效网络，再加36个恒等层）。然而实验结果是：56层网络的训练误差和测试误差都明显高于20层网络。

这就是著名的退化问题（degradation problem）——当网络深度增加到一定程度后，网络的准确率会饱和并迅速下降。注意，这不是过拟合（训练误差也在升高），也不是梯度消失（后面我们会看到梯度消失确实被缓解了，但退化的现象仍然存在）。它是一个更本质的优化难题：深度网络难以通过随机梯度下降直接优化。

2015年秋天，我亲自做过这个对比实验来验证。我在自己组的服务器上分别训练了一个20层和一个50层的卷积网络——没有残差连接。50层网络的训练Loss降到某个值后就再也下不去了，而20层的还在继续下降。我当时盯着tensorboard的Loss曲线看了很久，那种感觉就像你明明给了模型更多的"参数"和"自由度"，它反而做得更差了——真的很反直觉。

10.2 残差学习的核心思想

Kaiming He团队提出的解决方案极其优雅。他们问了一个极其深刻的问题：如果深度网络难以直接学习一个复杂的映射H(x)，那么让它去学习另一个函数F(x)会不会更容易？

具体来说，假设我们希望网络学习的底层映射是H(x)。我们不直接让网络拟合H(x)，而是将网络设计为学习残差函数：

# 残差学习的核心公式
#
# 令 F(x) 为网络层需要学习的残差映射
# 目标映射 H(x) 被重写为：
#
#     H(x) = F(x) + x
#
# 则网络实际要学习的是：
#
#     F(x) = H(x) - x
#
# 这就是"残差"名称的由来——我们学习的是目标与输入的差，
# 而不是目标本身。

# 残差块的数学形式（一个包含两层卷积的残差块）：
# y = F(x, {W_i}) + x
#
# 其中：
#   F = W_2 * σ(W_1 * x)   （σ 是ReLU激活函数）
#   x 通过"跳跃连接"直接加到输出上

# 当需要改变维度时（例如通道数变化）：
# y = F(x, {W_i}) + W_s * x
# 其中 W_s 是一个1×1卷积，用于对齐维度

为什么学习残差比学习原始映射更容易？直觉上，如果我们想让一个恒等映射（identity mapping）是最优的，那么学习F(x)=0比学习H(x)=x要容易得多——因为神经网络中的权重初始值通常接近0，所以F(x)本身就接近0，加上x后自然接近恒等映射。相反，如果直接学习H(x)=x，网络需要学会"所有权重等于1、偏置等于0"这种精确的配置，这对随机初始化来说非常困难。

💡 直观理解：可以把残差学习想象成"delta learning"——你不需要从零开始预测一个值，只需要预测它对某个基准（输入x）的修正量。这就好比让你猜一个陌生人的年龄，你很难一次猜准。但如果告诉你这个人大概30岁左右，你只需要猜"偏大几岁"或"偏小几岁"——修正量通常比绝对值小得多，也更容易学习。

10.3 残差块的实现细节

ResNet中的基本构建模块是残差块（Residual Block）。根据网络深度的不同，使用了两种不同类型的残差块：

10.3.1 BasicBlock（用于ResNet-18/34）

BasicBlock包含两个3×3卷积层，保持通道数不变。适用于较浅的ResNet变体。

# BasicBlock：两个 3×3 卷积 + 跳跃连接
# 输入: [Batch, C, H, W]
#    ↓
# Conv2d(3×3, C→C) → BN → ReLU
#    ↓
# Conv2d(3×3, C→C) → BN
#    ↓
# + (与输入x做逐元素相加)
#    ↓
# ReLU
# 输出: [Batch, C, H, W]

10.3.2 Bottleneck（用于ResNet-50/101/152）

对于更深的网络，BasicBlock的计算量太大。Bottleneck用1×1卷积来"压缩-计算-扩张"：

# Bottleneck：1×1降维 → 3×3 → 1×1升维
# 输入: [Batch, 256, H, W]
#    ↓
# Conv2d(1×1, 256→64) → BN → ReLU   # 压缩通道数
#    ↓
# Conv2d(3×3, 64→64) → BN → ReLU    # 核心计算（低维空间）
#    ↓
# Conv2d(1×1, 64→256) → BN           # 恢复通道数
#    ↓
# + (与输入x做逐元素相加)
#    ↓
# ReLU
# 输出: [Batch, 256, H, W]
#
# 计算量对比（相同输入输出通道256）：
#   BasicBlock: 3×3×256×256 × 2 = 1,179,648 次乘法
#   Bottleneck: 1×1×256×64 + 3×3×64×64 + 1×1×64×256 = 69,632 次乘法
# 节省了约 94% 的计算量！

Bottleneck的设计巧妙地利用了"信息瓶颈"的概念：先用1×1卷积将高维特征压缩到低维（通常压缩4倍），在低维空间做3×3卷积（计算量大幅降低），再用1×1卷积恢复到高维。这种"宽→窄→宽"的设计在后续的许多网络架构中被反复使用。

10.4 ResNet系列变体

ResNet家族包含多个深度版本，常见的配置如下：

变体层数残差块类型参数量ImageNet Top-1错误率
ResNet-1818BasicBlock11.7M~29.4%
ResNet-3434BasicBlock21.8M~27.7%
ResNet-5050Bottleneck25.6M~23.9%
ResNet-101101Bottleneck44.7M~22.6%
ResNet-152152Bottleneck60.3M~22.0%

有趣的一点是：ResNet-50和ResNet-34相比，层数增加了50%，参数量只增加了不到20%（25.6M vs 21.8M），这得益于Bottleneck设计的高效率。而且ResNet-152的参数量（60.3M）不到VGG16的138M的一半，但分类精度远远超出——这充分展示了残差连接+瓶颈设计的威力。

10.5 预激活ResNet（Pre-activation ResNet）

2016年，Kaiming He团队在原始ResNet的基础上做了进一步的改进——预激活ResNet。核心变化是：将BN和ReLU从卷积层的后面移到前面。

原始残差块（Post-activation）：

# 原始ResNet块：Conv → BN → ReLU → Conv → BN → + → ReLU
#                                       ↑______|
# 问题：最后一个ReLU在加法之后，会阻挡信息的直接传递

预激活残差块（Pre-activation）：

# 预激活ResNet块：BN → ReLU → Conv → BN → ReLU → Conv → +
#                                                ↑______|
# 好处：跳跃连接上没有任何非线性操作，
# 梯度可以无损地流过所有跳跃连接

预激活的网络在使用1000层以上的超深网络时表现出更好的训练稳定性。它还让BN的作用更充分——因为BN被放在激活之前，归一化效果更直接。虽然在标准ResNet-50/101上预激活带来的提升不大，但在1000层以上的极端深度下，这个改进是至关重要的。

10.6 ResNet的革命性影响

ResNet的贡献怎么强调都不过分。在CVPR 2016上，ResNet毫无争议地获得了最佳论文奖（Best Paper Award）。但它的影响远远超越了那篇论文本身：

  • 解决了梯度消失：残差连接为梯度的反向传播提供了一条"高速公路"——梯度可以通过跳跃连接直接从深层流向浅层，不受中间层的非线性变换影响。这使得100层、甚至1000层的网络可以稳定训练。
  • 成为通用组件：残差连接很快从图像分类扩散到了几乎所有深度学习架构——目标检测（Faster R-CNN的backbone变成了ResNet）、语义分割（DeepLab的backbone变了ResNet）、甚至NLP领域（Transformer中的Add & Norm本质上也是一种残差连接）。
  • 推动了更深架构的探索：在ResNet之后，DenseNet、ResNeXt、ResNeSt等一系列基于残差连接的改进架构相继涌现。残差连接成为了CNN设计的"基本操作"，就像加法一样自然。
  • 工业落地的主力：即使是在ViT已经非常成熟的2024年，ResNet-50仍然是工业界最常用的基准backbone之一。它的平衡性——精度尚可、速度够快、显存占用适中——使它成为许多实际产品中的默认选择。
🔗 一个简单的思想实验：想象没有残差连接的深度学习世界。没有ResNet，就不会有152层的网络；没有152层的网络，ImageNet分类很难突破人类水平；没有ImageNet上的突破，后续的检测、分割任务可能还会被困在手工特征时代。更重要的是，没有残差连接，Transformer中的"Add & Norm"层可能就不会诞生——而如果不理解残差连接在Transformer中的作用，你甚至无法理解为什么20层的Transformer可以稳定训练。可以说，整个现代深度学习的架构设计，都建立在残差连接这个基础之上。
⚠️ 踩坑故事：有一年我在做行人重识别（Person ReID）项目时，尝试自己设计一个baseline网络。我在ResNet-50的基础上增加了一些自适应的注意力模块。训练时我发现Loss下降得非常慢，而且最终的精度还比不上不加注意力模块的原始ResNet。我排查了两周，最终发现了一个让人哭笑不得的问题：我的注意力模块中某个分支忘记了添加残差连接。仅仅少了那么一个"+ x"，整个训练就崩了。这个故事告诉我们：在深层网络中，残差连接不是"锦上添花"——它是"必备条件"。少了它，深层网络根本没法正常工作。

至此，我们走完了CNN架构设计的三大里程碑：AlexNet（证明可行）→ VGG（极简极致）→ ResNet（突破瓶颈）。从下一章开始，我们将进入一个全新的范式——Transformer取代了CNN成为视觉领域的主角。ViT（Vision Transformer）将告诉我们：卷积不是唯一的选择，注意力机制可能才是通往通用视觉理解的关键。我们下一章见！
