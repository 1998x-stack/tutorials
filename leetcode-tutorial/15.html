图论：连接一切的数据结构

各位同学，欢迎来到图论的世界。如果说前面的数据结构（数组、链表、树）都是在"线性"或"层次"结构中操作数据，那么图（Graph）就是最能描述真实世界的结构——社交网络的好友关系、地图上的道路连接、互联网的路由拓扑，无一不是图。在 LeetCode 面试中，图论题目出现频率极高（尤其是大厂），而且通常是区分"能否通过"的关键题型。

很多同学一听到"图论"就发怵，觉得它复杂、抽象、算法多。其实图论题在面试中的考察点非常集中：图的遍历（DFS/BFS） 以及 拓扑排序、最短路径、并查集 这几种范式。本章我们先打好最核心的基础——图的表示和遍历，为后续几章的进阶算法做铺垫。

本章路线：图的两种表示方式 → DFS与BFS遍历 → 网格图中的DFS范式（岛屿问题） → BFS的应用 → 二分图判定。学完本章，你能解决 LeetCode 上约 60% 的图论入门题。

图的表示：邻接矩阵 vs 邻接表

在写任何图算法之前，第一件事就是把图"存"下来。LeetCode 上图的输入形式五花八门——有的是边列表，有的是邻接表，有的干脆给你一个二维网格。但在我们内部实现中，通常需要把输入转换成统一的表示。

邻接矩阵（Adjacency Matrix）

用一个 n × n 的二维布尔数组/整型数组来存储：matrix[i][j] = 1 表示 i 到 j 有边。无向图则矩阵对称。

# 邻接矩阵：n 个节点，edges 是边列表
def build_adj_matrix(n, edges):
    matrix = [[0] * n for _ in range(n)]
    for u, v in edges:
        matrix[u][v] = 1
        matrix[v][u] = 1  # 无向图
    return matrix

邻接表（Adjacency List）

用一个长度为 n 的列表，adj[i] 是一个列表，存储所有与 i 相邻的节点。这是最常用的表示法。

# 邻接表：n 个节点，edges 是边列表
def build_adj_list(n, edges):
    adj = [[] for _ in range(n)]
    for u, v in edges:
        adj[u].append(v)
        adj[v].append(u)  # 无向图
    return adj
特性 邻接矩阵 邻接表
空间复杂度 O(V²) O(V + E)
查两点是否相邻 O(1) O(degree)
遍历所有邻居 O(V) O(degree)
适用场景 稠密图（E ≈ V²） 稀疏图（绝大多数情况）
Floyd算法 必需 不方便
面试建议：99% 的 LeetCode 图论题用邻接表。除非题目明确是稠密图或要用到 Floyd 算法，否则默认走邻接表路线。节省空间且遍历邻居快。

图的DFS遍历：岛屿问题系列

说到图上的 DFS，最典型的莫过于 LeetCode 上的"岛屿问题"系列。这类问题的输入不是一个显式的 adj 列表，而是一个 m × n 的二维网格（Grid）。你把每个格子看作一个节点，相邻的四个方向就是边——这就是隐式图（Implicit Graph）。

岛屿数量（LeetCode 200）

题目：给定一个由 '1'（陆地）和 '0'（水域）组成的二维网格，计算岛屿数量。一个岛屿由相邻的陆地组成（上下左右四个方向）。

这就是网格 DFS 的"Hello World"。核心模板如下：

def num_islands(grid):
    if not grid:
        return 0
    m, n = len(grid), len(grid[0])
    count = 0

    def dfs(i, j):
        # 越界或遇到水/已访问，直接返回
        if i < 0 or i >= m or j < 0 or j >= n or grid[i][j] != '1':
            return
        # 标记已访问（淹掉陆地）
        grid[i][j] = '0'
        # 递归四个方向
        dfs(i+1, j)
        dfs(i-1, j)
        dfs(i, j+1)
        dfs(i, j-1)

    for i in range(m):
        for j in range(n):
            if grid[i][j] == '1':
                count += 1
                dfs(i, j)
    return count

这个模板的核心要点就三个：边界检查 → 访问标记 → 四方向递归。其中"访问标记"这一步极其关键——grid[i][j] = '0' 把陆地"淹掉"，避免重复访问。如果不做这步，DFS 会无限递归，栈溢出没商量。

血的教训：我曾帮一位同学 debug 他的岛屿问题代码。他用了 visited = set() 来记录访问过的格子，但忘记把 (i, j) 加入集合就调用了 dfs——同样的格子被反复入栈，直接爆栈。记住：访问标记必须在递归调用之前设置，而不是之后！

变体一：不同的岛屿数量（LeetCode 694）

这道题是岛屿数量的进阶版——需要统计"形状不同"的岛屿。怎么做？关键思路是：记录 DFS 遍历的路径序列。每个岛屿从起点开始 DFS，把相对坐标的移动序列（上、下、左、右）记下来，用这个序列作为岛屿的"指纹"。

def num_distinct_islands(grid):
    m, n = len(grid), len(grid[0])
    shapes = set()

    def dfs(i, j, di, dj, path):
        if i < 0 or i >= m or j < 0 or j >= n or grid[i][j] != 1:
            return
        grid[i][j] = 0
        path.append((di, dj))  # 记录相对坐标
        for dx, dy in [(1,0), (-1,0), (0,1), (0,-1)]:
            dfs(i+dx, j+dy, di+dx, dj+dy, path)
        # 注意：回溯时也要记录"返回"信号
        path.append((-di, -dj))

    for i in range(m):
        for j in range(n):
            if grid[i][j] == 1:
                path = []
                dfs(i, j, 0, 0, path)
                shapes.add(tuple(path))
    return len(shapes)

这里的技巧是 path.append((-di, -dj))——回溯时也记录一个反向信号。为什么？因为如果不这样做，两个形状相同但遍历顺序不同的岛屿会产生不同的路径。加了这个"回溯标记"后，路径序列就唯一了。

变体二：统计子岛屿（LeetCode 1905）

给定两个网格 grid1 和 grid2，如果 grid2 中的一个岛屿完全被 grid1 中的某个岛屿包含，则称它为"子岛屿"。这题的关键是：先遍历 grid2 的每个岛屿，在 DFS 过程中同时检查 grid1 的对应格子是否也是陆地。如果某个格子在 grid1 中是水，则标记该岛屿不是子岛屿。

def count_sub_islands(grid1, grid2):
    m, n = len(grid1), len(grid1[0])
    is_sub = True

    def dfs(i, j):
        nonlocal is_sub
        if i < 0 or i >= m or j < 0 or j >= n or grid2[i][j] != 1:
            return
        grid2[i][j] = 0
        if grid1[i][j] != 1:
            is_sub = False  # grid2的这块陆地在grid1中是水
        for dx, dy in [(1,0), (-1,0), (0,1), (0,-1)]:
            dfs(i+dx, j+dy)

    count = 0
    for i in range(m):
        for j in range(n):
            if grid2[i][j] == 1:
                is_sub = True
                dfs(i, j)
                if is_sub:
                    count += 1
    return count
岛屿问题总结：三题的核心都是"DFS + 访问标记"的四方向遍历模板。不同点只在于 DFS 过程中收集的信息不同——数量、形状、或与另一个网格的对应关系。掌握这个模板，所有网格 DFS 题目一通百通。

图的BFS遍历

BFS 的核心优势是最短路径——在无权图中，BFS 第一次访问到目标节点时走过的路径就是最短的。LeetCode 上最常见的 BFS 应用场景有两种：网格中的"感染扩散"和单词/状态的"最短转换"。

腐烂的橘子（LeetCode 994）

题目：网格中每个格子可能是新鲜橘子(1)、腐烂橘子(2)或空(0)。每分钟腐烂橘子会感染上下左右相邻的新鲜橘子。求全部腐烂所需的最短分钟数。

典型的"多源 BFS"问题——所有初始的腐烂橘子同时开始扩散。

def oranges_rotting(grid):
    m, n = len(grid), len(grid[0])
    queue = []
    fresh = 0

    # 初始化：所有腐烂橘子入队，统计新鲜橘子数
    for i in range(m):
        for j in range(n):
            if grid[i][j] == 2:
                queue.append((i, j))
            elif grid[i][j] == 1:
                fresh += 1

    if fresh == 0:
        return 0

    minutes = -1
    while queue:
        minutes += 1
        # 逐层处理
        for _ in range(len(queue)):
            i, j = queue.pop(0)
            for dx, dy in [(1,0), (-1,0), (0,1), (0,-1)]:
                ni, nj = i + dx, j + dy
                if 0 <= ni < m and 0 <= nj < n and grid[ni][nj] == 1:
                    grid[ni][nj] = 2
                    fresh -= 1
                    queue.append((ni, nj))

    return minutes if fresh == 0 else -1

注意这里的关键点：for _ in range(len(queue)) 是分层 BFS 的经典写法。它确保每次循环处理的是同一"分钟"内感染的橘子，这样 minutes 才能正确计数。

BFS vs DFS 选择指南：如果问题问的是"最短步数""最少时间""最早感染"等带"最"字的路径问题，第一反应应该是 BFS。DFS 适合"是否存在路径""有多少种路径"这类连通性/计数问题。

单词接龙（LeetCode 127）

题目：给定一个开始单词、一个结束单词和一个单词列表，每次改变一个字母，找从 beginWord 到 endWord 的最短转换序列长度。

这道题的巧妙之处在于把"单词"看作节点，"相差一个字母的单词对"看作边。构造图之后，就是标准的 BFS 找最短路径。

def ladder_length(beginWord, endWord, wordList):
    word_set = set(wordList)
    if endWord not in word_set:
        return 0

    queue = [(beginWord, 1)]  # (当前单词, 当前长度)
    visited = {beginWord}

    while queue:
        word, length = queue.pop(0)
        if word == endWord:
            return length

        # 尝试改变每个位置上的每个字母
        for i in range(len(word)):
            for ch in 'abcdefghijklmnopqrstuvwxyz':
                if ch == word[i]:
                    continue
                next_word = word[:i] + ch + word[i+1:]
                if next_word in word_set and next_word not in visited:
                    visited.add(next_word)
                    queue.append((next_word, length + 1))

    return 0

这个解法的时间复杂度是 O(26 × L × N)（L 为单词长度，N 为单词数）。LeetCode 上能过，但如果你追求极致性能，可以用双向 BFS——从 beginWord 和 endWord 同时开始搜索，当两个方向的 visited 集合相交时，路径就找到了。双向 BFS 能把搜索空间从 b^d 降到 2 × b^(d/2)，提升非常明显。

二分图判定：染色法

有一个经典面试题——判断二分图（LeetCode 785）。所谓二分图，就是可以把所有节点分成两个集合，使得每条边的两个端点分别属于不同集合。用人话说就是：用两种颜色给图上色，确保相邻节点颜色不同。

def is_bipartite(graph):
    n = len(graph)
    color = [-1] * n  # -1:未染色, 0/1:两种颜色

    def bfs(start):
        queue = [start]
        color[start] = 0
        while queue:
            node = queue.pop(0)
            for neighbor in graph[node]:
                if color[neighbor] == -1:
                    color[neighbor] = 1 - color[node]
                    queue.append(neighbor)
                elif color[neighbor] == color[node]:
                    return False
        return True

    for i in range(n):
        if color[i] == -1:
            if not bfs(i):
                return False
    return True

为什么要遍历所有节点？因为图可能是不连通的。我们需要对每个连通分量分别染色。只要有一个连通分量不能二染色，整个图就不是二分图。

二分图应用：二部图匹配、任务分配、课程安排中的冲突检测。另外，如果一个图是二分图，当且仅当图中没有奇环（长度为奇数的环）。这是二分图判定的充要条件，面试时偶尔会问到。

踩坑与总结

图论入门阶段的坑非常多，我挑几个最常见的来说。

坑1：访问标记不更新导致死循环

这是图论题中最常见的 bug，没有之一。在 DFS 或 BFS 中，访问标记必须在"入队/入栈"时设置，而不是"出队/出栈"时。如果在出队时才标记已访问，同一个节点可能被多个邻居加入队列多次，轻则性能下降，重则栈溢出。

# 错误写法：出队时才标记
def bfs_wrong(graph, start):
    queue = [start]
    while queue:
        node = queue.pop(0)
        visited.add(node)  # 这里才标记？太晚了！
        for nb in graph[node]:
            if nb not in visited:
                queue.append(nb)

# 正确写法：入队时就标记
def bfs_correct(graph, start):
    queue = [start]
    visited = {start}
    while queue:
        node = queue.pop(0)
        for nb in graph[node]:
            if nb not in visited:
                visited.add(nb)
                queue.append(nb)

坑2：邻接表构建失误

LeetCode 上很多图论题的输入是 edges = [[0,1],[1,2],[2,3]] 这种边列表。新手很容易写出 adj = [[]] * n——这在 Python 中会创建 n 个指向同一个列表的引用！修改一个会影响所有。正确的写法是 adj = [[] for _ in range(n)]。

自查清单：写图论题之前，先问自己三个问题——(1) 是有向图还是无向图？(2) 节点编号从 0 开始还是从 1 开始？(3) 图中有没有孤立的节点？这些问题看似基础，但答错一个代码就全盘皆输。

坑3：递归深度限制

Python 默认递归深度是 1000。如果图很大（比如 500×500 的网格），DFS 递归可能超过这个限制。解决方案有三个：用 sys.setrecursionlimit(10000) 加大限制、改用迭代式 DFS（显式栈）、或者直接用 BFS。

场景 推荐遍历方式 理由
网格连通分量（小图） DFS 递归 代码简洁直观
网格连通分量（大图） BFS 或迭代 DFS 避免递归栈溢出
最短路径/步数 BFS 天然保证最短性
二分图判定 BFS/DFS 均可 染色法两种都支持
拓扑排序 BFS (Kahn) 入度表法更直观

到此为止，图论的基础知识就铺垫完了。我们从图的两种表示（邻接矩阵 vs 邻接表）出发，掌握了 DFS 和 BFS 两种遍历方式在网格图和抽象图上的应用，最后学习了二分图判定的染色法。下一章我们将进入图论的两大核心应用——拓扑排序与并查集，它们是解决课程安排、任务调度、动态连通性等问题的利器。我们下一章见！
