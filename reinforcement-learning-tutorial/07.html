第七章：时序差分学习

同学们好！今天我们来聊聊强化学习中一个划时代的思想——时序差分学习（Temporal Difference Learning, TD）。如果说蒙特卡洛方法教会了智能体"从经验中总结教训"，动态规划教会了它"站在未来的肩膀上回望当下"，那么TD方法则做了一件更聪明的事——它让二者合二为一。

7.1 TD学习的思想起源

想象一下：你每天从家开车到公司，心里会默默估计"大概要35分钟"。结果今天开到半路遇到了堵车，实际到公司花了50分钟。请问，在什么时候你会意识到自己的估计有偏差？——显然，不是到了公司的那一刻，而是在你看到堵车的那一刻，你就已经知道"哦，今天要晚了"。

这个直觉正是TD学习的核心。它不需要等到最终结果出现，就能利用当前的局部信息对估计进行修正。

这个故事其实有一个经典的心理学实验背景。20世纪50年代，心理学家Richardson和Kloper研究动物的条件反射：给狗一个铃声（状态St），然后给食物（奖励Rt+1），再等一段时间给第二个铃声（状态St+1）。他们发现，狗在听到第一个铃声后就会开始分泌唾液——这说明动物的大脑在实时地根据"当前状态与下一个状态的差异"来调整预测，而不是等到实验全部结束才做总结。

这个"差异"就是强化学习中最核心的公式之一——TD误差（TD Error）：

TD误差定义：
δt = Rt+1 + γ V(St+1) - V(St)

这个式子看似简单，但它包含了两个深刻的思想：

  • Rt+1 + γ V(St+1) 是当前对"真实价值"的一个估计——用动态规划的方式，拿一步实际奖励加上后续状态的价值（bootstrap，自举）。
  • V(St) 是之前对这个状态的估值。
  • 两者的差值就是"预测误差"——如果为正，说明实际情况比预期好，我们要上调估值；如果为负，说明实际情况比预期差，我们要下调估值。

可以把这个过程理解为"边走边学"。蒙特卡洛方法是"期末考完再复盘"，动态规划是"靠课本推导"（需要完整的环境模型），而TD方法是每做一道题就对一对答案——效率高，反馈快。

7.2 TD(0) 预测

TD(0)是最简单的TD预测算法，它的更新公式非常直观：

TD(0) 价值更新公式:
V(s) ← V(s) + α [ R + γ V(s') - V(s) ]

其中 α 是学习率，控制每次更新的步长。方括号里的内容正是我们刚刚提到的TD误差。

让我们用一个生活中的例子来理解。假设你每天上班的通勤时间是这样的：

时刻事件TD(0)更新过程
t=0出门（状态S0），估计全程30min初始值V(S0)=30
t=1到地铁站（状态S1），花了5minV(S0)←V(S0)+α[5+V(S1)-V(S0)]
t=2上地铁（状态S2），又花了3minV(S1)←V(S1)+α[3+V(S2)-V(S1)]
.........

关键区别在这里：

  • 蒙特卡洛：等你到公司，发现总共用了50分钟，然后回过头来调整"出门"这个状态的估值。
  • TD(0)：你刚到地铁站，发现这段路花了5分钟，再结合地铁站这个状态的估值，立刻就可以更新"出门"状态的估值。

这就是"即时自举（Bootstrap）"的含义——用当前估计的价值去更新另一个价值估计，就像抓住自己的鞋带把自己提起来。听起来有点"左脚踩右脚"的意思，但数学上这被证明是收敛的，而且方差远低于蒙特卡洛方法。

从偏差-方差的角度看：MC方法无偏但方差大（因为需要整条轨迹的累积回报，受随机性影响大），TD(0)有偏但方差小（因为只用了一步真实奖励，其余靠估计值的自举）。在实际应用中，TD(0)通常比MC收敛快得多。

7.3 SARSA：同策略TD控制

学会了预测（估计价值函数），我们自然要问：能不能用TD来做控制（找到最优策略）？答案是肯定的！

首先登场的是SARSA算法。它的名字来源于更新所需的五元组：(St, At, Rt+1, St+1, At+1)——取首字母正好组成"SARSA"。

SARSA的Q值更新公式：

Q(s,a) ← Q(s,a) + α [ r + γ Q(s',a') - Q(s,a) ]

注意这里的 a' 是当前策略在 s' 状态下选择的动作。也就是说，SARSA使用同策略（on-policy）的方式学习——它评估和改进的是同一个正在执行的策略。

动作选择通常采用 ε-贪婪策略（ε-greedy）：以 1-ε 的概率选择当前Q值最高的动作（利用），以 ε 的概率随机探索。这个 ε 通常在训练过程中逐渐衰减，实现从探索到利用的过渡。

SARSA算法的伪代码如下：

算法：SARSA（同策略TD控制）

初始化 Q(s,a) 任意值，对于所有 s∈S, a∈A(s)
对于每个回合（episode）:
    初始化状态 S
    使用 ε-贪婪策略从 Q 选择动作 A
    对回合中的每一步:
        执行动作 A，观察奖励 R 和下一个状态 S'
        使用 ε-贪婪策略从 Q 选择动作 A'（在状态 S' 下）
        Q(S,A) ← Q(S,A) + α [ R + γ Q(S',A') - Q(S,A) ]
        S ← S', A ← A'
    直到 S 为终止状态

你可以看到，SARSA在每一步更新中都用到了实际执行的下一步动作A'。这也是它被称为"同策略"的原因——你学习的策略就是你在执行的策略，没有二心。

7.4 Q-Learning：异策略TD控制

接下来是强化学习中最具影响力的算法之一——Q-Learning。它由Chris Watkins在1989年提出，是异策略（off-policy）TD控制的代表。

Q-Learning的更新公式：

Q(s,a) ← Q(s,a) + α [ r + γ maxa' Q(s',a') - Q(s,a) ]

看到和SARSA的差别了吗？区别就在 maxa' Q(s',a') 上。Q-Learning在更新目标中使用了下一个状态所有可能动作中最大的Q值，而不是SARSA中实际选中的那个动作的Q值。

这个 max 操作符是革命性的——它意味着Q-Learning直接学习最优动作价值函数 Q*，而不依赖于当前执行的策略是什么。即使你正在用一个随机策略来探索环境，Q-Learning的更新目标仍然是"假设后面每一步都选最优动作"会得到什么结果。这就是"异策略"的含义：策略评估和目标策略可以不一样。

老师划重点： SARSA和Q-Learning的区别可以这样记忆——SARSA是"保守型选手"，它问："按照我现在的策略，下一步会怎样？"；Q-Learning是"理想主义者"，它问："不管我现在怎么做，如果后面每一步都选最好的，会怎样？"

实际应用中，SARSA在训练过程中会更安全（因为它学习的是带探索的策略），而Q-Learning学到的更优但可能更冒险。
踩坑故事：悬崖行走的教训

我的一个学生曾经在一个"悬崖行走"（Cliff Walking）环境中犯过这样一个错误——他实现了Q-Learning，但收敛速度特别慢，而且智能体总在悬崖边上徘徊。他检查了好几遍代码，觉得更新公式没问题。

后来发现，问题出在探索策略和更新策略混用了。他在更新Q值时用了Q-Learning的公式（max），但在计算TD目标中的Q(s',a')时，却错误地传入了当前ε-贪婪策略选出的动作Q值，而不是max。这就变成了"用SARSA的核，穿Q-Learning的马甲"，两边不讨好。

教训：实现时一定要盯住更新公式里的那个小细节——Q-Learning的 max 是你通往最优策略的"快捷键"，别丢了它；SARSA选中哪个动作就用哪个动作的Q值，别画蛇添足。去翻开你的代码，拿公式逐字对一下，很多时候bug就藏在一个 max 的缺失里。

悬崖行走（Cliff Walking）是理解两者差异的经典场景。一个网格世界，智能体要从起点走到终点，边缘是悬崖。SARSA由于考虑到了探索策略可能带来的风险（ε-贪婪会随机往悬崖边走），所以会学出一条远离悬崖的"安全路径"。而Q-Learning学的是最优策略，它会沿着悬崖边缘走（因为这是最短路径），但在训练过程中可能会因为探索而频繁掉下悬崖——所以实际表现反而是SARSA在训练期间的总奖励更高。这是"同策略 vs 异策略"的一个非常生动的体现。

7.5 TD(λ) 与资格迹

既然TD(0)只用一步自举，MC用完整轨迹，那有没有"介于两者之间"的方法？当然有——TD(λ)和资格迹（Eligibility Traces）。

核心思想是：一步TD太短，MC太长，λ步刚刚好。

前向视角（Forward View）：λ-回报

考虑一个 n 步回报：

  • 1步回报：Gt:t+1 = Rt+1 + γ Vt(St+1)
  • 2步回报：Gt:t+2 = Rt+1 + γ Rt+2 + γ² V(St+2)
  • ...
  • ∞步回报：Gt = Rt+1 + γ Rt+2 + γ² Rt+3 + ... (即MC回报)

TD(λ) 将所有这些 n 步回报按权重 (1-λ) λn-1 组合起来，得到 λ-回报：

Gtλ = (1-λ) Σn=1∞ λn-1 Gt:t+n

当 λ=0 时，权重全部给到1步回报，退化为TD(0)。当 λ=1 时，权重平均分配，趋近于MC方法。

后向视角（Backward View）：资格迹

资格迹是实际实现中使用的机制。它为每个状态维护一个"资格值" e(s)，表示这个状态"应该为当前误差负多大责任"：

资格迹更新：
e(s) ← γλ e(s) + 1  (如果 s 是当前状态)
e(s) ← γλ e(s)      (其他状态)

TD(λ) 价值更新：
V(s) ← V(s) + α δ e(s)    ∀s

这里的直观理解是：资格迹就像"记忆的痕迹"。当你踩到一个坑，一开始记忆很清晰（e很大），但随着时间推移，记忆逐渐淡化（e衰减）。λ控制衰减速度——λ越大，记忆越持久，越接近MC的"全记住"方式。

7.6 三大阵营总结

到了这里，我们已经学过了三种强化学习的核心方法：动态规划（DP）、蒙特卡洛（MC）和时序差分（TD）。让我们用一个表格来清晰地对比它们：

对比维度 动态规划（DP） 蒙特卡洛（MC） 时序差分（TD）
是否需要环境模型 是（需要转移概率） 否（从经验学习） 否（从经验学习）
是否自举（Bootstrap） 是 否（等待完整回报） 是
是否采样（Sampling） 否（全量期望） 是（采样完整轨迹） 是（采样单步）
更新时机 全量更新 回合结束后 每步更新
偏差（Bias） 无偏（精确计算） 无偏 有偏（自举引入偏差）
方差（Variance） 无 高（累积随机性） 低（只用一步）
适用场景 环境模型已知、状态空间小 回合制任务、需要无偏估计 通用、连续任务、在线学习
收敛速度 取决于模型精度 慢（需完整回合） 快（即时更新）

从这张表中可以看到，TD方法巧妙地结合了DP的"自举"思想和MC的"采样"思想，同时去掉了两者的主要缺点——既不需要环境模型（DP的痛点），也不需要等到回合结束（MC的痛点）。这也是为什么TD成为现代强化学习中最重要的基石之一。

当然，天下没有免费的午餐。TD的代价就是引入了偏差——因为用估计值去更新估计值，初始阶段难免有偏。但好消息是，在合适的条件下（如学习率递减且满足Robbins-Monro条件），TD方法被证明是收敛的。

本章小结

  • 时序差分（TD）是蒙特卡洛思想和动态规划思想的结合，核心在于使用自举（bootstrap）和采样（sampling）。
  • TD误差 δ = R + γ V(s') - V(s) 是一切TD方法的起点。
  • TD(0) 是最简单的TD预测算法，每步更新，即时学习。
  • SARSA 是同策略TD控制算法，使用五元组 (s,a,r,s',a') 更新。
  • Q-Learning 是异策略TD控制算法，使用 max 操作直接学习最优Q函数。
  • TD(λ) 通过资格迹在TD(0)和MC之间架起桥梁，λ=0时退化为TD，λ=1时趋近MC。
  • DP、MC、TD三者在是否需要模型、是否自举、是否采样三个维度上各有不同，理解它们的区别是构建强化学习直觉的关键。

掌握了时序差分学习，你已经拥有了强化学习中最核心的算法武器。从下一章开始，我们将进入一个新的世界——当状态空间太大，无法用表格存储时，我们如何用函数逼近（Function Approximation）来解决问题。深度强化学习的序幕即将拉开！
