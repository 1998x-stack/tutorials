第21章 单对以太网(SPE)

大家好，欢迎来到第21章。上一章我们聊了汽车以太网——在汽车里用单对非屏蔽双绞线跑以太网。你可能在想：既然汽车可以，那工业现场、楼宇自控、甚至偏远传感器为什么不能用？答案是：当然可以，而且这正是单对以太网（Single Pair Ethernet, SPE）正在做的事情。

SPE可以说是近年来以太网物理层最激动人心的创新之一。它把以太网从"四对线"的束缚中解放出来，让一根双绞线同时承载数据和电力，还能传输到1公里之外。这一下子打开了无数新场景的大门——从油田的压力传感器到智能楼宇的温度控制器，从矿山的安全监测到农业的土壤湿度监测。本章我们就来深入探索SPE的方方面面。

21.1 什么是SPE

传统以太网从10BASE-T开始就使用4对双绞线（或者说至少2对：100BASE-TX使用2对，1000BASE-T使用4对）。4对线的连接器（RJ45）体积不算小，线缆也比较粗重。在数据中心和办公环境中，这不是大问题。但在工业现场、汽车、传感器网络中，情况就完全不同了：

  • 空间：一个工业传感器外壳可能只有拇指大小，装不下一个RJ45插座。
  • 重量和成本：4对以太网线缆每米的重量是SPE线缆的3-4倍，整个工厂数万米的布线累计下来，差异非常大。
  • 连接器可靠性：标准RJ45的卡扣在振动环境下容易松脱，工业场景需要更可靠的锁定机制。
  • 传输距离：传统以太网100米就不行了，但很多工业场景需要几百米到一公里的连接距离。

单对以太网（SPE）顾名思义，只用一对双绞线传输数据。它最革命性的特点有两个：第一，只用一根线传数据和供电（PoDL，数据线供电）；第二，10BASE-T1L可以传输长达1公里。这让它成为替代传统RS-485、RS-232、HART等串行总线的理想选择——用一套统一的IP网络替代这些各不相同的专有总线。

21.2 10BASE-T1L (802.3cg)：远距离SPE

10BASE-T1L（IEEE 802.3cg，2019年发布）是SPE家族中第一个获得广泛关注的标准。"1L"中的L代表Long Reach——长距离。它的技术参数如下：

  • 速率：10 Mbps（全双工）
  • 传输距离：最长1公里（通过单对平衡电缆）
  • 编码方式：4B3B + PAM3，配合DME（Differential Manchester Encoding）边沿编码
  • 线路电压：2.4V的线路信号摆幅，针对长距离传输优化
  • 电源：支持PoDL（Power over Data Line），可选供电

10BASE-T1L的10 Mbps速率在现代IT工程师看来似乎"太慢了"——我们习惯了千兆甚至万兆。但在过程自动化（process automation）领域，10 Mbps已经是飞跃式的进步。原有的HART协议只有1.2 kbps，RS-485通常也只有几十到几百kbps，Modbus RTU的速度更慢。10 Mbps意味着可以传输更丰富的数据内容——除了简单的传感器读数，还可以传输诊断信息、配置参数、固件升级包等。

10BASE-T1L的1公里传输距离也是非常关键的指标。在化工厂、炼油厂、水处理厂等大型设施中，传感器到控制柜之间的距离经常超过100米（标准以太网的极限）。以前需要中继器或者光纤，现在一根SPE线缆就能解决问题。

下面是一个10BASE-T1L的典型系统框图：

    [控制室]                           [现场传感器]
  +------------+                    +--------------+
  | SPE交换机   | ---- 单对双绞线 --- | 10BASE-T1L   |
  | (10BASE-T1L)|    (最长1公里)     | 传感器接口   |
  +------------+   +------------+   +--------------+
       |           | PoDL电源  |        |
       |           | (可选)    |        |
  RJ45端口          +------------+    传感器探头
   (传统以太网)                          |
  转SPE介质转换器                   温度/压力/流量

技术细节：10BASE-T1L使用了一种聪明的编码策略来平衡带宽和距离的关系。它采用4B3B编码（4位数据→3位符号）+ PAM3调制，将信号带宽控制在较低水平，从而获得更长的传输距离。实际上，10BASE-T1L的信号在1公里处的衰减可能超过20 dB，因此PHY芯片需要具备很强的均衡能力来恢复信号。

21.3 100BASE-T1L：中速SPE

除了10BASE-T1L之外，IEEE 802.3dg工作组正在标准化100BASE-T1L——在1公里距离上提供100 Mbps的速率。这比10BASE-T1L又进了一大步，其设计挑战也更大：更高的速率意味着更宽的信号带宽，在长距离传输中信号衰减更严重。

100BASE-T1L目前使用的技术方案包括PAM4编码和更强的链路均衡器（Link Equalization），类似于1000BASE-T1在汽车以太网中采用的技术。它和10BASE-T1L一样支持PoDL，单对线缆同时传数据+供电。

虽然100BASE-T1L的标准尚未最终定稿（截至2026年），但它已经引起了工业界的高度关注，特别是那些需要"同时传输多个视频流"或"汇聚多个传感器数据流"的工业场景。

21.4 PoDL：数据线供电

PoDL（Power over Data Line）是SPE生态系统中的关键组成部分。它和传统的PoE（Power over Ethernet）有相似的目标——通过数据线缆供电——但它们的实现方式和应用场景有很大区别。

下面这张表对比了PoDL和PoE的主要差异：

特性 PoE (802.3af/at/bt) PoDL (802.3bu)
线对数 4对或2对 1对
供电方式 通过空闲线对或幻象供电 通过单对线中心抽头
最大功率 15.4W (af) / 30W (at) / 90W (bt) 0.5W-50W (分10个等级)
供电距离 100米 最长1公里 (取决于功率等级)
连接器 RJ45 单对连接器 (M8/M12/通用端子)
典型场景 IP摄像头、无线AP、IoT网关 工业传感器、执行器、现场仪表

PoDL把功率划分为不同的等级（Class），从Class 0（约0.5W）到Class 10（约50W），以适应不同功耗的设备。一个简单的温度传感器只需要Class 0（0.5W足够），而一个带加热功能的流量计可能需要Class 6以上。供电端（Power Sourcing Equipment, PSE）通过检测受电设备（Powered Device, PD）的签名电阻来判断应该提供多大的功率，这个过程和PoE非常相似。

21.5 SPE的典型应用场景

SPE的应用版图非常广阔，我们来逐一看看：

工业物联网（IIoT）：工厂里的数千个传感器、执行器、电机驱动器如果用传统4对以太网连接，线缆成本和布放难度都太高。SPE用一根细线就能把每个设备接入IP网络，而且可以通过PoDL供电，彻底告别了"数据一根线、电源一根线"的双线缆模式。更重要的是，它们可以通过标准的IP协议（MQTT、OPC UA、Modbus TCP）与云端通信，实现真正的"从传感器到云端"的统一连接。

楼宇自动化：传统的楼宇自控网络使用BACnet MS/TP（基于RS-485）或Modbus RTU来连接温度传感器、灯光控制器、风机盘管等设备。这些总线需要独立布线，传输速率慢（通常几十kbps），而且不同厂商的设备兼容性经常出问题。用SPE替代后，可以改造成标准的BACnet/IP或Modbus TCP，速率提升到10-100 Mbps，而且可以直接接入标准IT网络。

过程自动化：炼油厂、化工厂、电力行业的现场仪表（压力变送器、温度变送器、流量计等）通常使用4-20 mA模拟信号或HART协议（4-20 mA上叠加1.2 kbps的数字信号）。这些技术已经存在了几十年，带宽极低、诊断能力弱。10BASE-T1L的1公里传输距离和10 Mbps速率，刚好满足过程自动化的需求，而且能提供远比4-20 mA和HART丰富的数据内容。

铁路和交通：铁路沿线的信号设备、轨道传感器、道岔控制器分布在数公里的线路上。SPE的单对线+长距离传输特性，使得铁路通信系统可以用一根线串联多个设备，大幅降低线缆成本和施工难度。

真实案例：某煤矿安全监测项目需要在地下300米深处的采掘工作面部署瓦斯浓度、温度、粉尘浓度等多种传感器。传统方案使用RS-485总线连接所有传感器，每个传感器还需要单独的供电线缆。施工中遇到了两个问题：一是RS-485的通信距离限制（不加中继器大约1200米），矿道长度远远超出；二是供电线缆和信号线缆的"双重布线"导致施工量翻倍。项目团队在2022年改用10BASE-T1L方案：一根4 mm直径的单对铠装电缆同时传输数据和供电，通过PoDL为每个传感器提供约2W的功率。单个SPE网段覆盖了1.5公里的矿道（使用了中继器），施工成本比RS-485方案降低了约60%，而且网络调试速度大大加快——因为所有设备直接分配IP地址，不再需要配置Modbus设备ID和波特率。项目负责人评价说："最大的变化是运维简单了。以前RS-485出问题，你要排查波特率配置、终端电阻、屏蔽接地……现在就跟家里修WiFi一样，ping一下就知道哪里断了。"

这个案例很有代表性——SPE的核心价值不仅仅是"技术升级"，更是将IT的运维范式引入了OT领域。当温度传感器拥有自己的IP地址时，它的管理方式就从一个"现场仪表"变成了一个"网络设备"。

21.6 SPE的生态与连接器

SPE的推广还需要解决一个"最后一厘米"的问题：连接器。RJ45是为4对线设计的，体积太大，不适合SPE。工业领域需要一个小型化、高可靠性、支持PoDL的单对线连接器标准。

IEC 63171系列标准定义了SPE的几种连接器规格：

  • IEC 63171-1：M8/M12圆形连接器，工业场景常见，带螺纹锁紧，抗振动
  • IEC 63171-2：小型推拉式连接器，类似RJ45的小型化版本
  • IEC 63171-6：带屏蔽的推拉式连接器，用于要求EMC保护的场景

这些连接器都支持PoDL，并且有防插错设计——不会把SPE连接器插到传统的RJ45插座里（反过来也不会）。随着更多设备厂商开始支持SPE接口，连接器的成本正在快速下降。
