大家好！之前近十章的课程，我们一直在一个假设下工作：环境被离散化为图/栅格结构。这适用于 2D 平面导航，但当你面对一个七轴机械臂——它的构型空间是 7 维的——栅格化就彻底失效了。一个 100x100 的 2D 栅格有 1 万个格子，但一个每维 100 等分的 7 维栅格却有 100 万亿个节点！

这就是 RRT（Rapidly-exploring Random Tree）登场的时刻。RRT 由 Steven LaValle 于 1998 年提出，是采样方法（sampling-based methods）的奠基之作。它不离散化空间，而是通过随机采样逐步探索连续空间，在高维路径规划中几乎成了事实标准。

为什么要采样？

先澄清一个根本性问题：为什么我们要放弃"精确求解"而转向"随机采样"？

路径规划问题在最一般形式下是 PSPACE-hard 的——意味着精确求解的计算复杂度随着维度呈指数级增长，这是"维度诅咒"（curse of dimensionality）的核心体现。对于高维空间，我们别无选择，只能用近似方法。

采样方法的哲学是：与其精确地搜索整个空间，不如随机地探索，逐渐覆盖自由空间。这听起来很粗糙，但有一个重要的理论保证：概率完备性（probabilistic completeness）——当采样次数趋于无穷时，算法找到路径的概率趋于 1。换句话说，只要给足够多的时间，采样方法几乎肯定能找到路径。

RRT 核心流程

RRT 的基本流程极其直观，只有四个重复步骤：

  1. Sample（采样）——在自由空间中随机选一个点 x_rand
  2. Nearest（最近邻）——在已有树中找到离 x_rand 最近的节点 x_near
  3. Steer（伸展）——从 x_near 向 x_rand 方向走一步（步长 delta），得到新节点 x_new
  4. CollisionCheck（碰撞检测）——检查从 x_near 到 x_new 的边是否无碰撞，是则加入树中

重复以上步骤，直到树伸展到目标附近或达到最大迭代次数。这就是 RRT 的全部。简洁到令人难以置信——却又异常有效。

让我们用一个比喻来理解：想象你在一个黑暗的大房间里找出口。你每走一步就朝随机方向扔一个石子——石子落地的方向就是你的下一步方向。你不需要完整的房间地图，只需要确认走的这一段路上没有障碍物。这就是 RRT 的工作原理。

手写 RRT

我们来实现一个完整的 RRT。注意这里的最近邻搜索我们用 scipy.spatial.KDTree 来加速——如果没有 KDTree，暴力搜索在大规模树上的复杂度是 O(n²)，很快会成为瓶颈。

import numpy as np
from scipy.spatial import KDTree

class RRT:
    def __init__(self, start, goal, obstacle_check, bounds,
                 max_iter=1000, step_size=5.0, goal_tolerance=5.0):
        self.start = np.array(start)
        self.goal = np.array(goal)
        self.obstacle_check = obstacle_check  # 碰撞检测回调
        self.bounds = bounds  # [(xmin, xmax), (ymin, ymax), ...]
        self.max_iter = max_iter
        self.step_size = step_size
        self.goal_tolerance = goal_tolerance
        self.nodes = [self.start]
        self.parents = {0: -1}  # 节点的父节点索引

    def sample(self):
        """随机采样一个点"""
        if np.random.random() < 0.05:  # 5% goal bias
            return self.goal
        return np.array([np.random.uniform(low, high)
                         for low, high in self.bounds])

    def steer(self, from_node, to_node):
        """从 from_node 向 to_node 伸展 step_size 距离"""
        direction = to_node - from_node
        dist = np.linalg.norm(direction)
        if dist < self.step_size:
            return to_node.copy()
        return from_node + (direction / dist) * self.step_size

    def plan(self):
        tree = KDTree([self.start])
        for i in range(self.max_iter):
            x_rand = self.sample()
            _, idx = tree.query(x_rand)
            x_near = self.nodes[idx]
            x_new = self.steer(x_near, x_rand)
            if self.obstacle_check(x_near, x_new):
                self.nodes.append(x_new)
                tree = KDTree(self.nodes)  # 重建 KDTree
                self.parents[len(self.nodes) - 1] = idx

                if np.linalg.norm(x_new - self.goal) < self.goal_tolerance:
                    return self._reconstruct_path(len(self.nodes) - 1)
        return None  # 未找到路径

    def _reconstruct_path(self, idx):
        path = []
        while idx != -1:
            path.append(self.nodes[idx])
            idx = self.parents[idx]
        return path[::-1]
小技巧：5% Goal Bias——上面代码中你注意到了吗？sample() 有 5% 的概率直接返回目标点。这个技巧叫做"目标偏置"（goal bias），是 RRT 最实用、最有效的改进之一。纯粹的随机探索可能会在目标附近徘徊很久才发现终点——每次直接采样目标会让树有意地朝目标方向生长。5% 是一个经验值，太小了效果不明显，太大了会导致树被"拉变形"。一个小改动，规划时间可能缩短 3-5 倍。我第一次加了这个偏置后，眼睁睁看着仿真中的机械臂从 "在目标点四周转圈圈" 变成了 "直扑目标"，感觉像是给机器人装上了导航雷达。

RRT 的特性与局限

RRT 有许多引人注目的特性，也有不容忽视的局限：

Voronoi 偏置

RRT 有一个"隐性"的探索优势：由于采样均匀分布，树中未被充分探索的区域在 Voronoi 图中占据更大的面积，因此有更高的概率被选中生长。这个性质让 RRT 天然地倾向于探索未开发区域——这就是它名字中"Rapidly-exploring"的含义。你不需要显式地"鼓励探索"，算法自然就会去做。

与 A* 的对比

维度 A* RRT
空间类型 离散图/栅格 连续空间
2D 搜索效率 优秀（有最优性保证） 一般（路径粗糙）
高维（5D+）搜索 不可能（维度诅咒） 可行（核心优势）
最优性 保证最优 不保证（路径通常很差）
完备性 分辨率完备 概率完备
运动学约束 难以集成 天然支持（修改 steer）

RRT 最大的局限是路径质量——它找到的路径往往是锯齿状的、充满不必要拐弯的"丑路径"。这是因为每次采样都是随机的，树生长方向缺乏"全局视野"。你会在下一章看到 RRT* 如何利用重新布线来解决这个问题。

说到 RRT*，这正是我们下一章的主题。RRT* 保持了 RRT 的所有优点（高维搜索能力、概率完备性），但额外保证了渐进最优性——随着采样增加，路径会逐步趋近最优。它的核心创新只有一个操作，叫做"Rewire"（重新布线），但这个操作的价值怎么强调都不过分。我们下一章见。
