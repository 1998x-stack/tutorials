第8章 最佳平方逼近

大家好，我是你们的讲师。今天我们要聊一个非常有意思的话题——最佳平方逼近。大家还记得上一章我们讲的函数逼近基本理论吗？我们说插值像是在函数上"踩点"，通过有限个点把函数钉住。但有个现实问题：如果数据本身就有噪声，或者我们需要的不是局部精确匹配而是整体趋势逼近，插值就不太适合了。

我给大家讲个故事。好多年前我参与过一个气象数据分析项目，需要从离散的观测站数据重建温度场的连续分布。如果用拉格朗日插值，结果简直惨不忍睹——插值多项式在两个观测站之间疯狂振荡，出现了离谱的负绝对温度！这当然不是物理现实，而是插值的过拟合问题。后来我们换成了最佳平方逼近，结果就稳定多了。整体误差虽然每个点都不完美，但全局来看是最优的。这就是最佳平方逼近的魅力所在。

8.1 最佳平方逼近问题的提法

我们先给出严格的数学定义。设 $f \in C[a,b]$ 是我们要逼近的目标函数，我们想找一个次数不超过 $n$ 的多项式 $p_n(x)$，使得它在加权 $L^2$ 范数意义下与 $f$ 的误差最小：

最小化 ||f - p_n||_2 = sqrt(∫_a^b [f(x) - p_n(x)]^2 w(x) dx)

其中 $w(x) > 0$ 是权函数，用来控制不同区域的重要性。这个问题和插值的区别非常本质：

对比维度插值最佳平方逼近
匹配方式在节点处严格相等整体平方误差最小
对噪声敏感度高度敏感（过拟合）鲁棒（平滑效应）
节点数需要 $n+1$ 个节点确定 $n$ 次多项式任意多个数据点
解的存在性唯一（节点互异时）唯一（权函数正定）
主要应用精确数据重构噪声数据拟合

核心 insight：插值是"点对点"的精确匹配，最佳平方逼近是"全局"的能量最小化。前者适合确定性数据，后者适合带噪声的观测数据。在实际工程中，90% 的逼近问题其实更适合用平方逼近而非插值。

8.2 法方程方法

怎么求解这个问题呢？最直接的方法是选定一组基底函数 $\{\varphi_0(x), \varphi_1(x), \dots, \varphi_n(x)\}$，将逼近多项式表示为它们的线性组合：

p_n(x) = c_0 φ_0(x) + c_1 φ_1(x) + ... + c_n φ_n(x)

代入最小化问题，对系数 $c_k$ 求偏导并令其为零，就得到所谓的法方程（normal equations）：

G c = b

其中：
G_{ij} = ∫_a^b φ_i(x) φ_j(x) w(x) dx   (Gram 矩阵)
b_i    = ∫_a^b f(x) φ_i(x) w(x) dx
c      = [c_0, c_1, ..., c_n]^T

这里的 Gram 矩阵 $G$ 是对称正定的，所以法方程有唯一解。但问题来了——如果用最简单的单项式基底 $\varphi_k(x) = x^k$，Gram 矩阵会是什么样？

踩坑警告！用单项式基底 $1, x, x^2, \dots, x^n$ 做最佳平方逼近，Gram 矩阵是 Hilbert 矩阵的变种：$G_{ij} = (b^{i+j+1} - a^{i+j+1})/(i+j+1)$。Hilbert 矩阵的条件数随 $n$ 指数增长！$n=10$ 时条件数已经超过 $10^{13}$，法方程几乎不可解。我当年第一次做这个实验时，用单精度浮点数直接求解，结果得到的系数完全错误——这就是基底选择的重要性。

8.3 用正交多项式作最佳平方逼近

既然单项式基底会导致病态的 Gram 矩阵，我们自然想到：能不能选一组基底使得 Gram 矩阵是对角矩阵？答案是肯定的——用正交多项式！

如果 $\{\varphi_k\}$ 是关于权函数 $w(x)$ 正交的，即：

∫_a^b φ_i(x) φ_j(x) w(x) dx = 0,  i ≠ j
∫_a^b φ_i(x) φ_j(x) w(x) dx = ||φ_i||^2,  i = j

那么 Gram 矩阵就变成了对角矩阵！系数可以直接计算：

c_k = (f, φ_k) / (φ_k, φ_k) = ∫_a^b f(x) φ_k(x) w(x) dx / ∫_a^b φ_k(x)² w(x) dx

这太漂亮了——不需要解线性方程组，每个系数独立计算！

最常见的例子就是在 $[-1,1]$ 上用 Legendre 多项式作最佳平方逼近。Legendre 多项式 $P_k(x)$ 关于权函数 $w(x)=1$ 正交。这就是经典的 Legendre 级数展开：

f(x) ≈ Σ_{k=0}^n c_k P_k(x),  c_k = (2k+1)/2 ∫_{-1}^1 f(x) P_k(x) dx

下面是一个 Python 代码对比，展示用单项式基底和 Legendre 多项式基底逼近 $f(x) = e^x$ 在 $[-1,1]$ 上的效果：

import numpy as np
from numpy.polynomial import Legendre
from numpy.linalg import solve, cond

def monomial_approximation(f, n, a=-1, b=1):
    """用单项式基底做最佳平方逼近"""
    # 构造 Gram 矩阵 (Hilbert-like)
    G = np.zeros((n+1, n+1))
    b_vec = np.zeros(n+1)
    for i in range(n+1):
        for j in range(n+1):
            G[i,j] = (b**(i+j+1) - a**(i+j+1)) / (i+j+1)
        # 右端项 ∫ f(x) x^i dx
        # 用数值积分近似
        xs, ws = np.polynomial.legendre.leggauss(100)
        half = (b-a)/2
        mid = (a+b)/2
        b_vec[i] = half * np.sum(ws * f(half*xs + mid) * (half*xs + mid)**i)
    print(f"n={n}: cond(G) = {cond(G):.2e}")
    c = solve(G, b_vec)
    return lambda x: sum(c[i]*x**i for i in range(n+1))

def legendre_approximation(f, n, a=-1, b=1):
    """用 Legendre 多项式基底做最佳平方逼近"""
    # 将 f 从 [a,b] 映射到 [-1,1]
    half = (b-a)/2
    mid = (a+b)/2
    f_mapped = lambda t: f(half*t + mid)

    # 计算 Legendre 系数
    series = Legendre.fit(f_mapped, n+1, [-1,1])
    # 提取 Legendre 系数
    c = series.coef
    print(f"n={n}: using Legendre basis, no linear system needed!")
    return lambda x: series(x)

# 测试 f(x) = e^x
f = np.exp
for n in [5, 10, 15]:
    p_mono = monomial_approximation(f, n)
    p_leg = legendre_approximation(f, n)
    xs = np.linspace(-1, 1, 1000)
    err_mono = np.max(np.abs(p_mono(xs) - f(xs)))
    err_leg = np.max(np.abs(p_leg(xs) - f(xs)))
    print(f"  Monomial max error: {err_mono:.2e}, Legendre max error: {err_leg:.2e}")

实验观察：运行上述代码你会发现，当 $n=10$ 时单项式基底的 Gram 矩阵条件数已经达到 $10^{14}$ 量级，解出的系数完全不可信。而 Legendre 多项式基底根本不需要解线性方程组，直接积分得到系数，精度高得多。这就是为什么在实际计算中我们几乎总是用正交多项式基底。

8.4 Fourier 级数与平方逼近

最佳平方逼近还有一个大家非常熟悉的特例——Fourier 级数。当区间选为 $[-\pi, \pi]$，权函数 $w(x)=1$，基底选为三角函数系 $\{1, \cos kx, \sin kx\}$ 时，我们实际上就在做 $L^2$ 意义下的最佳平方逼近。

Fourier 级数的截断：

S_N(x) = a_0/2 + Σ_{k=1}^N (a_k cos(kx) + b_k sin(kx))

正是 $f(x)$ 在 $N$ 阶三角多项式空间中的最佳平方逼近。这一点很多学信号处理的同学可能没有意识到——Fourier 级数本质上就是在做最佳平方逼近！

有趣的联系：正交多项式展开是 Fourier 分析在一般函数空间中的推广。Legendre 级数、Chebyshev 级数、Fourier 级数，它们都是同一个框架下的不同实例——选取不同的正交基底来做最佳平方逼近。

说到 Fourier 级数就不得不提Gibbs 现象。我读书时有个同学做信号处理的课程设计，用 Fourier 级数逼近方波信号。他取了 100 项展开，心想这总该够精确了吧？结果发现在间断点附近始终存在约 9% 的过冲，而且增加项数并不会让过冲消失，只会让它越来越靠近间断点。

Gibbs 现象：用 Fourier 级数逼近有间断点的函数时，在间断点附近会出现持续的振荡和过冲。即使 $N \to \infty$，过冲幅度也不会消失（约 8.95%）。这意味着用平方逼近来逼近非光滑函数时，虽然在 $L^2$ 意义下收敛，但逐点逼近效果可能很差。这个坑在做信号滤波时尤其常见——别以为取更多 Fourier 项就能消除 Gibbs 现象，你需要的是用不同的基底（比如小波）或者加窗处理。

从本章的讨论我们可以看出，最佳平方逼近是一个非常强大的工具，它的核心在于基底的选择。用正交多项式基底可以避免病态的线性方程组，而不同的正交多项式适用于不同的区间和权函数。下一章我们将进入最佳平方逼近在实际应用中最常见的形式——最小二乘曲线拟合，解决超定方程组的数据拟合问题。大家准备好了吗？
