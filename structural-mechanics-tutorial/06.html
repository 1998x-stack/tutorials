开篇：不让任何一种"武器"闲置

大家好，我是你们的讲师。今天的内容很有意思——我们要讲组合结构（Composite/Combined Structure）。

到现在为止，我们学了几种"纯种"结构：梁（以受弯为主）、桁架（杆件只受轴力）、拱（以受压为主）、刚架（梁柱刚接，承受弯剪轴）。但在实际工程中，结构工程师很少会"只用一种结构形式"。就像一位厨师不会只用一种调料一样，聪明的结构工程师会根据受力特点，在结构的不同部位采用不同的结构形式——这就组合结构。

我来讲一个真实的案例。上海浦东国际机场的T1航站楼，建成于1999年，它的屋盖结构是一种非常典型的组合结构——张弦梁（Beam String Structure）。什么是张弦梁？简单来说，就是"一根拱形的梁 + 下面的拉索 + 中间连接它们的撑杆"。梁承受弯矩和压力，索承受拉力，撑杆把两者连在一起。这种结构既利用了梁的刚度来抵抗局部荷载，又利用了索的高抗拉强度来提供整体刚度——两种材料、两种结构形式的完美结合，实现了72米的无柱大空间。

这个案例告诉我们一个道理：没有一种结构形式是万能的。组合结构的艺术，就是让每种结构形式做它最擅长的事。

组合结构的概念与分类

所谓组合结构，是指由两种或两种以上的不同受力特征的结构单元组合而成的结构体系。在结构力学课程中，我们主要讨论的是杆件组合结构——即由梁式杆（受弯构件）和二力杆（轴向拉压构件）组合而成的结构。

常见的组合结构类型包括：

  • 梁-桁组合（Beam-Truss Composite）——用桁架杆件"加固"梁，形成加劲梁。典型应用：桥梁中常见的桁架加劲梁，用桁架的斜腹杆来承受剪力，使主梁的剪力大幅减小。
  • 拱-梁组合（Arch-Beam Composite）——拱和梁通过撑杆或吊杆连接，共同受力。典型应用：系杆拱桥（tied-arch bridge）、下承式拱桥。
  • 索-梁组合（Cable-Beam Composite）——斜拉桥就是最典型的例子：主梁承受弯矩，斜拉索承受拉力，两者通过桥塔连接。
  • 多层混合结构——在建筑结构中，下部采用刚架、上部采用桁架的组合方式也很常见。例如，体育馆的底部看台采用混凝土框架，顶部屋盖采用钢桁架。
组合类型 梁式杆作用 二力杆作用 典型工程
梁-桁组合 承受整体弯矩 承受剪力（斜腹杆） 桁架加劲梁桥
拱-梁组合 承受局部荷载弯矩 拱圈受压、吊杆受拉 系杆拱桥
索-梁组合 承受桥面弯矩 斜拉索受拉 斜拉桥
刚架-桁架组合 下部框架受力 上部桁架屋盖 大跨体育馆

组合结构的受力分析步骤

组合结构分析的核心挑战是：要正确处理"梁式杆"和"二力杆"之间的连接关系。在计算简图中，梁式杆用"粗线"表示（能承受弯矩），二力杆用"细线"表示（两端铰接，只受轴力）。

通用的分析步骤如下：

  1. 识别构件类型——把结构中的每根杆件区分为"梁式杆"（受弯）和"二力杆"（只受轴力）。判断依据是：两端铰接且杆间无横向荷载的杆件为二力杆；否则为梁式杆。
  2. 确定分析顺序——与多跨静定梁类似，先分析附属部分，后分析基本部分。
  3. 计算支座反力——对整体列平衡方程。
  4. 取隔离体——在梁式杆与二力杆的连接处切开，露出未知力。
  5. 先求二力杆内力——二力杆内力仅用结点法或截面法即可求出，然后将求得的力反向作用于梁式杆。
  6. 再分析梁式杆——此时梁式杆已承受所有已知力（外荷载+二力杆传来的力），按梁的常规方法绘制内力图。
核心原则：组合结构分析的"黄金法则"是——二力杆的轴力总是可以通过平衡方程直接求出，不需要考虑梁式杆的弯曲变形。因为静定结构的反力和内力只由平衡方程决定，与变形无关。这就意味着，你可以先把所有二力杆的内力求出来（作为"已知荷载"），再分析梁式杆的内力。这个"分步走"的策略可以避免联立方程的麻烦。

梁-桁混合结构内力计算

梁-桁混合结构是组合结构中最常见的一种。它的"梁部分"主要承受弯矩，"桁架部分"（斜腹杆+竖杆）主要承受剪力。

让我们通过一个具体的示例来说明：

示例：一榀加劲钢梁的分析

        荷载：均布荷载 q = 10 kN/m
        跨度：l = 12 m
        结构形式：工字钢梁 + 三角形桁架加劲

        计算简图：
        q = 10kN/m
        ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓
        ────────────────────────    ← 主梁（梁式杆）
              /\    /\    /\
             /  \  /  \  /  \
            /    \/    \/    \      ← 腹杆（二力杆）
           /______\/______\/
        △                           △
        支座                      支座

        分析步骤：

        第1步：求支座反力
        R_A = R_B = q·l/2 = 60 kN

        第2步：识别二力杆
        所有斜腹杆和竖杆均为二力杆（两端铰接）。
        主梁为梁式杆。

        第3步：取结点法求腹杆内力
        从支座结点开始，逐次求解各腹杆的轴力。

        第4步：将腹杆轴力反向作用于主梁结点
        此时主梁上的荷载 = 原外荷载 q + 腹杆传上来的集中力

        第5步：绘制主梁的弯矩图和剪力图
        此时主梁的弯矩和剪力已经远小于"无加劲"的情况。

这个例子展示了加劲梁的工作原理：不加劲的简支梁最大弯矩为 ql²/8 = 180 kN·m。加上桁架加劲后，主梁的跨中最大弯矩可能降低到原来的1/3到1/2。换句话说，加劲桁架承担了大部分荷载产生的弯矩和剪力，让主梁"轻松"了很多。

一个小技巧：在做组合结构分析时，建议你把所有二力杆的轴力标注在结构简图上（受拉标"+"，受压标"-"）。这样，当你在分析梁式杆时，可以一目了然地看到每个结点处有哪些"额外"的集中力作用。

分析中容易犯的错误

组合结构分析中，初学者最容易犯以下几个错误：

  • 把梁式杆误判为二力杆——如果一根杆件中间有横向荷载（哪怕是一根小檩条传来的力），它就不是"纯"二力杆，必须按梁式杆分析。
  • 忽略了结点处的力平衡——在二力杆和梁式杆的连接结点处，必须满足力的平衡条件。初学者常常在"转交"荷载时漏掉某个力分量。
  • 搞错了二力杆的受力方向——二力杆的内力沿着杆轴线方向。在隔离体上画出时，方向必须正确，否则平衡方程会出错。
踩坑故事：我的一位研究生同学在做毕业设计时，分析了一个飞机库的屋盖结构——一个大型的拱-桁架组合结构。他用有限元软件分析后，发现某些弦杆的内力异常大，反复检查模型也没发现问题。后来他的导师看了一眼，说了一句："你把撑杆和主弦杆的连接做成了刚接，但实际它是铰接的。"就这么一个细节——在模型中把铰接做成了刚接——导致撑杆中出现了不应该有的弯矩，整个内力分布全错了。他花了整整两周才找到这个错误。这个教训是：在组合结构中，"结点类型的正确设定"比"计算的精度"重要得多。结点错了，算得再准也是白算。

从组合结构到结构优化

从某种意义上说，组合结构是"结构优化"思想的自然产物。如果你仔细观察大自然中的"结构"——比如鸟类的骨骼——你会发现它们也是组合结构：致密的外壳（类似梁式杆）承受弯曲，内部的蜂窝状结构（类似桁架杆件）提供抗压强度，两者配合得天衣无缝。

学会分析组合结构之后，你可以开始思考一个更深层次的问题：如何主动设计结构形式来优化力的传递路径？这就是结构拓扑优化（Topology Optimization）的萌芽思想——不满足于"给定的结构怎么算"，而是思考"什么结构形式最好"。我们在第26章（AI赋能结构分析）中会进一步展开这个话题。

一个有意思的类比：组合结构就像一个团队——梁式杆是"全能型选手"（能抗弯、抗剪、抗压、抗拉），二力杆是"专精型选手"（只做轴向拉压一件事但效率极高）。好的团队配置，就是让全能型选手做决策（承受整体弯矩），让专精型选手做执行（承受局部剪力或拉力）。这种"各司其职"的思想，不仅在结构设计中成立，在组织管理和系统设计中也同样成立。
