第17章 基站开通与数据配置

大家好，我是你们的讲师。前两章我们完成了基站硬件安装和工程部署——设备上架了，线缆接通了，地线接好了。现在是时候按下电源键，让这座基站"活"过来了。基站开通调测，是通信工程中最有"仪式感"的环节。从你插上电、看到BBU绿色指示灯亮起的那一刻起，一座新的基站就诞生了。

但是，"亮灯"不等于"通网"。要让基站正式为UE提供服务，需要配置海量的数据——从全局ID到IP地址，从无线频点到邻区关系，每一条参数背后都对应着协议栈中的某个行为。这个环节是通信工程中"出错率最高"的阶段之一，也是最考验工程师系统理解能力的地方。如果把基站比作一台电脑，安装部署是"组装硬件"，数据配置就是"装操作系统和驱动程序"。

17.1 开局流程概览

一座基站从"出厂状态"到"入网服务"，大体经过以下五个步骤：

  1. 设备上电：接通-48V DC电源，观察单板启动指示灯状态。正常情况下一台BBU在上电后3 ~ 5分钟内所有单板指示灯应变为绿色常亮。
  2. 版本加载：通过网管或本地调试口加载基站软件版本（通常是一个包含完整协议栈的软件包，大小在几百MB到几个GB不等）。
  3. 数据配置：配置全局参数、传输参数、无线参数和业务参数——这是本章的重点。
  4. 联调测试：与核心网进行SCTP偶联测试、NG接口注册、PDU会话建立测试。
  5. 入网：确认所有指标正常后，将基站由"调测态"切换为"服务态"，正式放号入网。

17.2 全局数据配置

全局数据是基站的"身份证"，定义了基站在网络中的唯一身份和归属关系。

参数 含义 典型配置示例 注意事项
gNB ID gNodeB全局标识，全网唯一 123456 与邻站和核心网侧预先协商好
Cell ID 小区标识，站内唯一 0 / 1 / 2（三扇区） Cell ID = gNB ID × 256 + Local Cell ID
PLMN 公共陆地移动网络号，MCC+MNC 460-00（中国移动）/ 460-01（中国联通） MCC 460 = 中国，MNC由工信部分配
TAC 跟踪区码，用于UE位置管理 25801 同一TA下的所有gNB的TAC必须一致
gNB Name 基站名称（便于运维识别） SZ-Nanshan-SGCC-5G-001 建议统一命名规则：城市-区域-站点-制式-编号

17.3 传输配置

基站的核心网侧通信依赖IP传输网。传输配置的复杂度不亚于无线配置，而且一旦出错，网管都连不上基站。

IP地址规划

每个基站至少需要配置以下IP地址：

  • 带内管理IP：用于网管系统（OMC）管理基站，要求路由可达。
  • 带外管理IP：通过独立的带外网口连接维护终端，在带内网络不可用时使用。
  • N2接口IP：gNB与AMF之间的信令面IP，承载NGAP协议。
  • N3接口IP：gNB与UPF之间的用户面IP，承载GTP-U隧道。

SCTP偶联配置

5G核心网的信令面基于SCTP（Stream Control Transmission Protocol）传输。SCTP偶联需要在gNB和AMF之间双向配置。常见的配置错误包括SCTP端口号不匹配和本地/对端IP地址写反。

/* gNB传输配置示例 — 华为MA5600 CLI 风格 */
!
interface GigabitEthernet0/1/0
 description TO_CORE_NETWORK
 ip address 192.168.100.2 255.255.255.0
 vlan dot1q 100
!
interface LoopBack0
 description OAM_MANAGEMENT_IP
 ip address 10.10.10.1 255.255.255.255
!
sctp peer AMF-1
 local-ip 192.168.100.2
 local-port 38412
 peer-ip 192.168.100.10
 peer-port 38412
 sctp-profile default
!
ng-interface
 amf peer AMF-1
 local sctp-association AMF-1
!

PS：SCTP偶联是基站入网的第一步。如果SCTP起不来，NG接口就建立不了，UE注册更是无从谈起。开站调测时，判断的核心是看SCTP关联状态是否为"Established"。在命令行输入 disp sctp association 后看到Status=OK，才能继续往下配。不确认SCTP就往下配无线参数，相当于地基没打好就往上盖楼。

17.4 无线参数配置

无线参数决定了基站空口的物理行为，也是5G NR配置中最核心的部分。

  • 频点（ARFCN）：5G NR的绝对频点号，由3GPP 38.104定义。例如n78频段（3.5 GHz）的中心频点ARFCN通常配置为627264。不同的ARFCN对应不同的实际频率。
  • 信道带宽：5G NR支持5/10/15/20/25/30/40/50/60/80/100 MHz带宽。常见的宏站配置为100 MHz。
  • PCI（物理小区标识）：物理层小区识别号，取值范围0 ~ 1007。PCI规划是网规的核心工作之一——同频同PCI会导致UE无法区分小区。
  • TDD时隙配比：5G NR TDD模式需要配置上下行时隙比例。典型的配置为"7D3S2U"（7个下行时隙、3个特殊时隙、2个上行时隙），上下行资源比约为3:1。
/* 5G NR小区基本参数配置示例 — XML格式 */
<NRCellConfig>
    <gNBId>123456</gNBId>
    <cellLocalId>0</cellLocalId>
    <cellName>SZ-Nanshan-SGCC-5G-001_Cell0</cellName>

    <!-- 频域配置 -->
    <arfcn>627264</arfcn>          <!-- n78 中心频点 -->
    <bandWidth>100</bandWidth>      <!-- MHz -->
    <subCarrierSpacing>30</subCarrierSpacing> <!-- kHz -->

    <!-- 时域配置 -->
    <tddPattern>7D3S2U</tddPattern>

    <!-- 小区标识 -->
    <pci>101</pci>                  <!-- 物理小区ID -->
    <tac>25801</tac>                <!-- 跟踪区码 -->
    <plmnList>
        <plmn>460-00</plmn>         <!-- 中国移动 -->
    </plmnList>

    <!-- 功率配置 -->
    <maxTxPower>46</maxTxPower>     <!-- dBm -->
    <p0Nominal>-90</p0Nominal>      <!-- 标称接收功率 -->
</NRCellConfig>

17.5 邻区配置

一个基站不能孤立存在。邻区关系定义了UE在本小区信号变差时应该尝试切换的小区。邻区配置遗漏或错误，是导致掉话和切换失败的常见原因。

在5G NSA（非独立组网）场景下，5G基站（gNB）需要与LTE基站（eNB）配置邻区关系，因为控制面信令由LTE承载。而在SA（独立组网）场景下，只需要NR邻区即可。

常用的邻区配置核查项

  • 本小区与邻小区是否互相添加了邻区关系（单向邻区会导致切换只能去不能回）
  • 邻区PCI是否与本小区或其他邻区冲突
  • 邻区的频点（ARFCN）是否配置正确
  • 邻区的PLMN是否与目标小区一致

17.6 业务开通与单站验证

数据配置完成后，不能直接说"行了"。需要通过一系列测试验证基站是否正常工作：

验证项目 测试方法 通过标准
NG接口建立 检查gNB与AMF的NG Setup流程 NG Setup Response 成功
小区状态 检查小区是否进入"Available"状态 Cell State = Activated
Ping测试 UE附着后ping核心网网关 时延 ≤ 30 ms，丢包率 = 0%
FTP上传/下载 UE连接FTP服务器进行文件传输 下行速率 ≥ 800 Mbps，上行 ≥ 100 Mbps
接入成功率 测试手机连续发起100次附着/去附着 成功率 ≥ 99%
切换测试 UE在相邻小区之间移动，验证切换是否成功 切换成功率 ≥ 98%，切换时延 ≤ 50 ms

踩坑故事——TAC配置错误的排查噩梦：2021年某运营商在华北某城市批量开通5G SA站点。一个批次的12个新站全部开通后，现场测试发现：UE在部分新站覆盖区域下可以正常注册，但在某几个特定站点覆盖区域内注册成功率极低，表现为"UE发完REGISTRATION REQUEST后收不到REGISTRATION ACCEPT"。网管后台查看，基站一切正常——小区状态Activated，NG接口建立成功，SCTP偶联正常。最后抓取信令跟踪发现：这些问题基站的TAC配置成了25802，而核心网AMF侧的TAC配置为25801。核心网收到UE的注册请求后，发现TAC不匹配，按照3GPP规范，AMF直接拒绝了该请求。修改TAC为25801后，UE立即注册成功。整个排查过程用了整整2天——原因就是配置模板中一个数字的差异。数据配置的每一个字段都有意义，模板复用时务必逐项复核。

开局配置Checklist：以下是一份常用的基站开局核查清单，建议打印出来逐项打勾：
✅ 全局参数（gNB ID / Cell ID / PLMN / TAC）
✅ 传输参数（OAM IP / N2 IP / N3 IP / VLAN / SCTP）
✅ 无线参数（ARFCN / BW / PCI / 时隙配比 / 功率）
✅ 邻区关系（NR邻区 / LTE邻区 / 双向配置）
✅ NG接口状态（SCTP Established / NG Setup Success）
✅ 小区状态（Cell Activated / SSB正常发射）
✅ 业务验证（Ping / FTP / 接入成功率 / 切换测试）
✅ 告警检查（无Major/Critical告警）

基站成功入网只是第一步。一个基站开好之后，它的"工作"才刚刚开始——周围电磁环境在变，用户负载在变，天气季节在变。如何让基站持续提供最佳性能？下一章我们进入网络优化与故障排查——从网优工程师的视角，解决覆盖、干扰、接入、切换等实际问题。
