第2章 数字逻辑基础

同学们好，欢迎回到计算机体系结构课堂。在上一章中，我们俯瞰了计算机体系结构的全貌——从冯·诺依曼结构到现代处理器的五大部件。今天，我们要潜入最底层，看看支撑这一切的基石究竟是什么。

你可能觉得"数字逻辑"听起来像是电子工程的内容，跟计算机体系结构关系不大。但请记住一个事实：你写的每一行高级语言代码，最终都会变成数十亿次晶体管开关动作。你发送的每一个网络请求，底层都是逻辑门在毫秒级别内做出数亿次判断。理解了数字逻辑，你就理解了计算机的"母语"。

2.1 布尔代数与逻辑门

1854年，英国数学家乔治·布尔（George Boole）出版了一本名为《思维定律的研究》的著作，提出了一套用代数符号表示逻辑推理的系统。他大概不会想到，一百年后，他的理论会成为整个数字世界的基石。

布尔代数只有两个值：真（True，记作1）和假（False，记作0）。基本操作有三种：与（AND）、或（OR）、非（NOT）。从这三种基本操作，可以派生出更多有用的逻辑门：与非（NAND）、或非（NOR）、异或（XOR）。

下面是最基本的七种逻辑门的真值表（假设两个输入A和B）：

A B AND OR NAND NOR XOR NOT A
00001101
01011011
10011010
11110000

这里有一个令人惊讶的事实：只用NAND门，就可以实现AND、OR、NOT、NOR、XOR等所有逻辑门。不信？试试看：把NAND的两个输入短接（A和B连在一起），它就变成了一个NOT门；在NAND后面再接一个NAND做NOT，就得到了AND门。NOR门也有同样的性质。这就是所谓的"功能完备性"（functional completeness）。

💡 核心要点：NAND门是数字世界的"万能积木"——用它可以实现任何布尔函数。正因为如此，现代芯片中NAND门的用量远超其他门类型，工艺优化也最成熟。实际上，你的CPU几乎全是用NAND门搭起来的。

在实际的CMOS工艺中，NAND和NOR比AND和OR更容易实现（需要的晶体管更少），这也是为什么芯片设计者偏爱NAND的原因之一。当你听到某款芯片有"XX亿个晶体管"时，其中大部分都是组成NAND门的晶体管。

2.2 组合逻辑电路

把基本逻辑门组合起来，就可以实现有用的功能。组合逻辑的特点是：输出只取决于当前的输入，与历史状态无关。没有记忆能力。

多路选择器（Multiplexer, MUX）：根据选择信号，从多个输入中选出一个送到输出。2-to-1 MUX需要1个选择位，4-to-1 MUX需要2个选择位。MUX在现代处理器中无处不在——寄存器堆的读取端口、ALU的操作数选择、指令译码逻辑都离不开它。

译码器（Decoder）：将n位输入编码为2^n位独热编码输出。例如，3-to-8译码器将3位二进制数（000到111）转换为8个输出中的一个为高电平。译码器在指令译码阶段至关重要——操作码需要被译码后才能产生控制信号。

编码器（Encoder）：译码器的逆过程，将2^n位输入压缩为n位输出。优先级编码器（priority encoder）在中断控制器中用于识别最高优先级的中断源。

加法器（Adder）：这是算术逻辑单元（ALU）的核心。先从最简单的开始：

  • 半加器（Half Adder）：计算两个1位二进制数的和。输出和（Sum）与进位（Carry）。Sum = A XOR B, Carry = A AND B。
  • 全加器（Full Adder）：比半加器多一个进位输入（Carry-in），可以级联。全加器的逻辑表达式：Sum = A XOR B XOR Cin, Cout = (A AND B) OR (Cin AND (A XOR B))。
  • 行波进位加法器（Ripple-Carry Adder）：将N个全加器串联起来，每个全加器的进位输出接到下一个全加器的进位输入。

一个4位行波进位加法器的结构如下：最低位（bit 0）的两个输入A0、B0和进位输入C0（通常为0）进入第一个全加器，产生S0和进位C1；C1与A1、B1进入第二个全加器，产生S1和C2；以此类推，直到产生S3和最终的进位输出C4。这个结构很直观，但有一个严重问题：进位必须从最低位逐级"行波"到最高位，对于32位加法器，在最坏情况下需要等待32级门延迟。这也是为什么现代处理器使用更快的进位结构（如超前进位加法器）。

虽然简单，但行波进位加法器完美体现了"用基础门构造复杂功能"这个核心思想。从AND/OR/NOT门到全加器，从全加器到ALU，这种层层抽象正是计算机体系结构的精髓。

2.3 时序逻辑电路

组合逻辑没有记忆，但计算机需要记住东西——当前运行到哪条指令、某个变量的值是多少、中断是否发生。这就需要时序逻辑电路登场了。

时序逻辑的关键：引入"状态"。输出不仅取决于当前输入，还取决于之前的历史状态。状态是靠反馈环路实现的——把输出引回输入。

SR锁存器（SR Latch）：用两个NOR门（或NAND门）交叉耦合构成。S（Set）置位输出为1，R（Reset）复位输出为0。当S=R=0时，保持当前状态。这是最基础的存储单元，每比特只需要两个门。

D锁存器（D Latch）：在SR锁存器基础上加入使能信号（Enable）。当Enable=1时，输出跟随输入D；当Enable=0时，保持状态。D锁存器是电平触发的。

D触发器（D Flip-Flop）：边沿触发。只在时钟信号的上升沿（或下降沿）采样输入，其他时间保持。标准实现是"主从结构"：两个D锁存器串联，第一个在时钟高电平透明，第二个在时钟低电平透明，组合起来只在时钟边沿采样。

下面是用Verilog描述的D触发器，这在数字芯片设计中是最常用的基本单元：

module d_flip_flop (
    input wire clk,      // 时钟信号
    input wire rst_n,    // 异步复位（低有效）
    input wire d,        // 数据输入
    output reg q         // 数据输出
);
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            q <= 1'b0;   // 复位时输出0
        else
            q <= d;       // 时钟上升沿采样
    end
endmodule

在时序电路中，有两个关键时序参数：

  • Setup Time（建立时间）：数据必须在时钟边沿到来之前保持稳定的最短时间。
  • Hold Time（保持时间）：数据必须在时钟边沿到来之后保持稳定的最短时间。

如果这两个约束得不到满足，触发器就会进入亚稳态（Metastability）——输出既不是确定的0也不是确定的1，而是在两者之间振荡。亚稳态不会持续太久（通常几纳秒内会稳定到0或1），但在这段不确定时间内，下游电路读取到的值可能是错的。

⚠️ 实战踩坑：亚稳态引发的"幽灵故障"

几年前我在调试一块FPGA板卡时遇到了一个令人抓狂的问题：板卡在实验室运行一切正常，但部署到客户机房后，每隔几小时就会随机地出现一次数据错误。我们用示波器抓了整整两天，最终发现是跨时钟域的同步问题——一个异步输入的信号没有做两级同步处理，导致触发器偶尔进入亚稳态，把错误的数据传播到了后续逻辑中。修复方法仅仅是在信号路径上插入两个额外的D触发器做同步（双级同步器），成本为零，但排查用了三周。这个教训让我深刻理解了setup/hold time约束的重要性。

2.4 寄存器与有限状态机

寄存器（Register）：多个D触发器共享同一个时钟信号，组成一个多位存储单元。一个32位寄存器就是32个D触发器并排，每个存储1比特。RISC-V处理器有32个通用寄存器（x0到x31），每个32位宽，就是32×32=1024个触发器。

移位寄存器（Shift Register）：将多个D触发器串联，每个的输出接到下一个的输入。在时钟驱动下，数据逐位移动。移位寄存器在串行通信接口（如UART、SPI）中用于并串/串并转换，也在乘法器中用于实现shift-and-add算法。

有限状态机（Finite State Machine, FSM）：由状态寄存器和组合逻辑组成。状态寄存器保存当前状态，组合逻辑根据当前状态和输入计算下一个状态和输出。FSM是处理器控制单元的核心。

两种FSM类型：

  • Moore型：输出只取决于当前状态。优点是输出在时钟边沿同步变化，没有毛刺。缺点是可能需要更多状态。
  • Mealy型：输出取决于当前状态和输入。通常需要的状态数更少，但输出可能因为输入的变化而产生毛刺。

来看一个交通灯控制器的例子。假设一个简单的十字路口，只有一条主干道和一条支路。主干道默认绿灯，当支路有车等待（传感器输入为1）时，切换为支路绿灯。这个FSM有四个状态：主干道绿灯、主干道黄灯、支路绿灯、支路黄灯。传感器输入和定时器超时信号共同驱动状态迁移。这样一个状态机，用几十个逻辑门和几个触发器就能实现，但它确实能无故障运行数十年——这正是数字逻辑的可靠之美。

2.5 从门电路到处理器的桥梁

在本章中，我们从最底层的逻辑门开始，一步步构建了组合逻辑和时序逻辑的基本构件。现在让我们把这些构件映射到CPU的各个部分：

  • ALU：由加法器、移位器、比较器等组合逻辑构成，加上多路选择器选择运算类型。完全由组合逻辑实现。
  • 寄存器堆（Register File）：由寄存器和译码器构成。读端口使用译码器选择要读取的寄存器，写端口使用译码器选择要写入的寄存器。
  • 程序计数器（PC）：是一个寄存器（加增量逻辑），每个时钟周期更新为下一条指令的地址。
  • 控制单元（Control Unit）：是一个有限状态机（或者在现代处理器中，是一个更复杂的微程序控制器），根据指令操作码产生所有控制信号。
  • 数据通路（Datapath）：寄存器、ALU、多路选择器、存储器的互连网络，加上驱动它们的控制信号。

这一切都建立在第一章提到的冯·诺依曼结构框架内，用本章学习的门级构件填充了细节。下一章，我们将探讨计算机如何表示和操作数据——从整数到浮点数，从二进制补码到IEEE 754标准。理解了数据表示，你才能真正理解为什么你的程序有时会给出"错误"的答案（提示：不是bug，是浮点精度问题），以及为什么-1的二进制表示是32个1。
