第11章 差分与ARIMA

大家好！前几章我们一直在讲"平稳序列"的建模。但现实中的数据有几个是平稳的？股票价格一路飙升，GDP 逐年增长，气温随着季节起伏——这些序列几乎都不平稳。那难道这些数据就不能用 ARMA 模型了吗？

答案是否定的。Box 和 Jenkins 的厉害之处在于他们发现：很多非平稳序列经过一个简单的操作——差分——就能变成平稳的。这个把 ARMA 扩展到非平稳领域的产物，就是 ARIMA 模型（Autoregressive Integrated Moving Average） 。"I"（Integrated）代表的就是差分这一步。

11.1 差分：几何直觉

差分的概念极其简单：

一阶差分：ΔY_t = Y_t - Y_{t-1}

就是把相邻两个点的差值作为新的序列。它衡量的是"变化量"而不是"水平值"。

为什么差分能让序列变平稳？想象一下你在开车：

  • Y_t 是行驶的总里程——随着时间的推移，这个数字只会增加，显然非平稳
  • ΔY_t 是每小时的行驶距离（即时速）——只要你的驾驶习惯相对稳定，这个序列就是平稳的（没有长期趋势，围绕某个均值波动）

大多数经济金融序列都类似：价格水平非平稳，但价格的变化率（收益率）通常是平稳的。GDP 水平非平稳，但 GDP 增长率是平稳的。

核心思想：ARIMA 做的不是对"数据本身"建模，而是对"数据的变化"建模。差分就像一把手术刀，切掉趋势，留下平稳的核心信息。

11.2 ARIMA(p,d,q) 的定义

ARIMA(p,d,q) 模型的定义分三步：

  1. 对 Y_t 做 d 次差分，得到平稳序列 W_t = Δ^d Y_t
  2. 对 W_t 拟合 ARMA(p,q) 模型
  3. 将差分操作"反转"得到对原始 Y_t 的预测

用滞后算子表达：

Φ(L) (1-L)^d Y_t = c + Θ(L) ε_t

大多数情况下 d=1 就足够了。对经济数据，d 几乎不会超过 2。

d 值 操作 适用场景
d=0 无差分 原始数据已经平稳
d=1 一阶差分 线性趋势（股价、GDP、销量）
d=2 二阶差分 曲率趋势（加速度变化），极少使用

11.3 单位根：为什么差分有效

要理解差分为什么有效，需要回到第8章的特征根概念。

一个 AR(1) 过程 Y_t = φY_{t-1} + ε_t，当 φ=1 时就是随机游走（Random Walk）。这个过程的特征方程 1 - φL = 0 的根是 L = 1/φ = 1，正好在单位圆上。这就是"单位根"（Unit Root）这个名称的由来。

当 φ=1 时，序列的方差随时间线性增长，任何冲击都不会衰减——今天的价格是昨天的价格加上一个随机增量，昨天的价格又是前天的价格加上随机增量……这种"累积"效应正是非平稳的来源。

对随机游走做一阶差分：ΔY_t = Y_t - Y_{t-1} = ε_t。结果完美地变成了白噪声！这就是差分的神奇之处——把单位根从特征方程中移除。

单位根检验：在决定 d 值之前，应该先做单位根检验。最常用的是 ADF 检验（Augmented Dickey-Fuller test）。原假设是"序列存在单位根"。如果 p 值小于 0.05，拒绝原假设，认为数据平稳。否则需要差分。但要注意：ADF 检验的检验力有限（尤其是样本量小的时候），不要盲目依赖。

11.4 过度差分：一个常见的悲剧

差分不是越多越好。过度差分（Overdifferencing）是一个初学者最容易犯的错误之一。

假设一个序列已经是平稳的（d=0 足够），但你硬是做了差分（d=1）。这会产生什么后果？原来序列 Y_t 是一个平稳 ARMA 过程，差分后 W_t = Y_t - Y_{t-1} 的方差会变大，而且会在移动平均部分引入一个不可逆的 MA 成分。

具体来说，如果 Y_t 是 ARMA(p,q)，那么 ΔY_t 是 ARMA(p, q+1)，但 MA 部分会有一个单位根——这就是过度差分的信号。在实际中，过度差分会让你的模型参数估计不稳定，预测区间异常宽大。

踩坑故事：一位量化分析师在处理一个宏观数据集时，看到序列有明显的上升趋势，想都没想就做了两次差分（d=2）。结果模型的 AIC 确实比 d=1 时更好——因为过度差分后 MA 部分吸收了一部分信号，看似拟合得更好。但样本外预测时，模型完全崩溃了：预测值上下剧烈震荡，区间宽度是 d=1 时模型的 3 倍。事后检查发现，其实 d=1 就足够了，过度差分把数据的协方差结构扭曲了。教训是：d 不要超过 2，而且一定要对比不同 d 值的预测效果，不能只看样本内拟合。

11.5 ARIMA 和 ARMA 的关系

一个有趣的视角：ARIMA 其实可以看作 ARMA 对"累积数据"的建模。反过来，如果你对 ARIMA 的原始序列求差分，就变成了 ARMA。

用数学表达：

  • 如果 Y_t ~ ARIMA(p,d,q)，那么 Δ^d Y_t ~ ARMA(p,q)
  • 如果 Y_t ~ ARMA(p,q)，那么 ΣY_t（累计和） ~ ARIMA(p,1,q)

所以 ARIMA 和 ARMA 之间就差了一个求和/差分操作。这也是为什么"ARIMA"中的"I"代表"Integrated"（积分）——你可以把差分理解为"微分"，把逆向操作（从差分恢复到原始值）理解为"积分"。

11.6 auto.arima：自动化的艺术

手动选择 p、d、q 可能很耗时。在 R 语言中有一个著名的函数 auto.arima（来自 forecast 包），它会自动搜索最优参数。Python 的 pmdarima 库提供了类似的功能。

auto.arima 的基本策略是：

  1. 先用单位根检验确定 d（如 KPSS 检验 + ADF 检验的联合决策）
  2. 在 d 固定的情况下，搜索 p 和 q 的组合（通常限制 p ≤ 5, q ≤ 5）
  3. 使用 AICc（小样本修正的 AIC）作为选择标准
  4. 考虑是否包含常数项（c）
import numpy as np
import pmdarima as pm
import matplotlib.pyplot as plt

# 生成一个需要差分的非平稳数据
np.random.seed(42)
n = 200
Y = np.cumsum(np.random.normal(0.1, 1, n))  # 随机游走带漂移

# 用 auto.arima 自动选择参数
model = pm.auto_arima(
    Y,
    start_p=0, max_p=5,
    start_q=0, max_q=5,
    start_d=0, max_d=2,
    seasonal=False,
    trace=True,          # 打印搜索过程
    error_action='ignore',
    suppress_warnings=True,
    stepwise=True,       # 使用逐步搜索（更快）
    information_criterion='aic'
)

print(model.summary())
print(f"自动选择的阶数: ARIMA{model.order}")

# 拟合后的残差诊断
residuals = model.resid()
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4))
ax1.plot(residuals)
ax1.set_title('残差序列')
from statsmodels.graphics.tsaplots import plot_acf
plot_acf(residuals, lags=20, ax=ax2)
plt.show()
注意：auto.arima 是个好工具，但它不是万能的。它会选择统计上"最优"的模型，但统计最优不等于业务最优。在实际项目中，建议把 auto.arima 的结果作为起点，再结合业务理解和残差诊断进行调整。机器学习中的"没有免费午餐定理"同样适用于这里。

ARIMA 模型把 ARMA 的应用范围从平稳数据扩展到了非平稳数据，这是时间序列分析史上的一大飞跃。但 ARIMA 有一个盲区——它只能处理趋势非平稳，面对季节性非平稳就力不从心了。比如说空调销量数据，既有一年比一年增长的趋势（需要 d=1），又有冬夏周期波动的季节性（需要额外的季节性处理）。下一章我们将引入季节性 ARIMA（SARIMA），专门解决这个问题。
