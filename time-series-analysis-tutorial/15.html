第15章 波动率建模：ARCH与GARCH

想象一下你是一名量化交易员，每天盯着股票收益率曲线。大部分时间收益率在 ±1% 之间温和波动，但突然某天波动骤增到 ±5%，然后接下来几天持续剧烈震荡，之后又慢慢归于平静。这种"大波动扎堆出现"的现象，就是金融时间序列最核心的特征之一——波动率聚集（Volatility Clustering）。

传统 ARIMA 模型假设方差恒定，这在金融数据面前完全行不通。波动率本身就是一个需要建模的对象。本章就来讲解如何用 ARCH 和 GARCH 模型捕捉金融时间序列的"脾气"。

本章核心目标：掌握波动率建模的基本框架，能够用 GARCH 类模型量化风险，并计算 Value-at-Risk (VaR)。

15.1 金融时间序列的"怪脾气"

为什么金融数据不能用普通时序模型处理？三个关键特征决定了它的特殊性：

特征含义直观表现
厚尾分布 (Fat Tails)极端值出现概率远高于正态分布"黑天鹅"事件频发
波动率聚集大波动后跟大波动，小波动后跟小波动震荡期与平静期交替出现
杠杆效应 (Leverage Effect)负收益对波动率的提升大于正收益暴跌后市场格外脆弱

GARCH 类模型正是为处理这些特征而设计的。它不直接预测价格涨跌，而是预测波动率本身——也就是收益率的条件方差。

15.2 从 ARCH 到 GARCH

1982年，经济学家 Robert Engle 提出了 ARCH（自回归条件异方差）模型，开创性地将波动率建模为过去残差平方的函数。模型的核心思想非常直观：如果昨天的收益波动很大，那么今天的条件方差也应该偏大。

ARCH(q) 模型的形式为：

收益率方程: r_t = μ + ε_t,   ε_t ~ N(0, σ²_t)
方差方程:   σ²_t = ω + α₁·ε²_{t-1} + α₂·ε²_{t-2} + ... + α_q·ε²_{t-q}

其中 σ²_t 是条件方差（今天的波动率），它等于过去 q 期残差平方的加权平均。参数 ω 是长期平均方差，α 系数衡量新信息对波动率的冲击强度。

但 ARCH 模型有个实际痛点：如果要捕捉长期的波动记忆，就需要很多阶数 (q 很大)，参数过多导致估计不稳定。1986年，Engle 的学生 Tim Bollerslev 拓展出了 GARCH（广义自回归条件异方差）模型，在方程中加入了条件方差自身的滞后项：

GARCH(1,1) 方差方程:
σ²_t = ω + α·ε²_{t-1} + β·σ²_{t-1}

这个简洁的公式仅用三个参数就描述了一个无限记忆的波动过程。α (ARCH项) 代表新信息冲击的强度，β (GARCH项) 代表波动的持久性。当α+β接近1时，波动率冲击非常持久——这在真实金融数据中极为常见。

实务经验：GARCH(1,1) 是金融实践中使用频率最高的波动率模型。统计显著且α+β ≈ 1 的 GARCH(1,1) 通常比任何高阶 GARCH(p,q) 表现都好。

15.3 用 Python 拟合 GARCH 模型

以下代码展示了如何用 arch 库对股票收益率拟合 GARCH(1,1) 模型并做波动率预测：

import yfinance as yf
import pandas as pd
from arch import arch_model
import matplotlib.pyplot as plt

# 下载标普500数据
sp500 = yf.download('SPY', start='2015-01-01', end='2024-12-31')
returns = sp500['Adj Close'].pct_change().dropna() * 100

# 拟合 GARCH(1,1) 模型
model = arch_model(returns, vol='Garch', p=1, q=1, dist='normal')
result = model.fit(disp='off')

print(result.summary())

# 预测未来10天的波动率
forecasts = result.forecast(horizon=10)
predicted_vol = forecasts.variance.iloc[-1] ** 0.5
print("预测波动率 (标准差%):")
print(predicted_vol)

# 绘制条件波动率
result.conditional_volatility.plot(figsize=(10, 4))
plt.title('SPY 条件波动率 (GARCH(1,1))')
plt.show()

运行结果中的 alpha[1] 和 beta[1] 系数之和通常在 0.97-0.99 之间，印证了波动率的强持久性。

15.4 GARCH 家族的扩展

标准 GARCH 假设好消息和坏消息对波动率的影响是对称的。但真实市场并非如此——杠杆效应意味着负收益带来的波动更大。为此，研究者开发了多种扩展模型：

模型全称核心特点
EGARCH指数 GARCH允许不对称效应，杠杆系数为负时表示坏消息冲击更大；无需约束参数为正
GJR-GARCHGlosten-Jagannathan-Runkle GARCH引入哑变量区分正负冲击，直接量化杠杆效应
TGARCHThreshold GARCH类似 GJR，用阈值区分正负残差
APARCHAsymmetric Power ARCH在方差方程中引入幂变换，灵活性最高

在大多数股票指数数据中，EGARCH 和 GJR-GARCH 比标准 GARCH 拟合更好，因为它们的杠杆项通常高度显著。

# EGARCH 模型示例
egarch_model = arch_model(returns, vol='EGARCH', p=1, q=1)
egarch_result = egarch_model.fit(disp='off')
print(egarch_result.summary())

# 关注 leverage 系数——显著为负说明杠杆效应存在

15.5 波动率预测与 VaR 计算

波动率预测的直接应用是风险管理。最常见的指标是 Value-at-Risk (VaR)，即在给定置信水平下，某资产在持有期内的最大可能损失。

基于 GARCH 的 VaR 计算公式为：

VaR(95%) = μ + σ_t × z_0.05

其中 σ_t 是 GARCH 预测的标准差，z_0.05 = -1.645 (正态分布5%分位数)

以下代码展示了回测 VaR 的方法：

# 基于 GARCH 的 VaR 计算
var_95 = result.conditional_volatility * (-1.645)
var_99 = result.conditional_volatility * (-2.326)

# 检验 VaR 的命中率
hits_95 = (returns < var_95).mean()
print(f"95% VaR 命中率: {hits_95:.4f} (期望 0.05)")

# 绘制回测结果
plt.figure(figsize=(12, 5))
plt.plot(returns.index, returns, alpha=0.6, label='收益率')
plt.plot(var_95.index, var_95, label='95% VaR', color='red')
plt.legend()
plt.show()

一个好的 VaR 模型，其实际亏损超过 VaR 边界的比例应该接近理论值（5% 或 1%）。如果偏差过大，说明模型需要调整。

⚠ 重要提醒：VaR 不是完美指标。当市场发生极端暴跌时，VaR 往往低估实际风险。建议结合压力测试和情景分析一同使用。

15.6 真实的教训：LTCM 与忽视的波动率

1998年，长期资本管理公司（LTCM）的崩溃是金融史上最经典的波动率风险管理失败案例。LTCM 由诺贝尔经济学奖得主 Myron Scholes 和 Robert Merton 联合创立，团队堪称"梦之队"。

LTCM 的核心策略是"收敛套利"——发现价格偏离的债券，押注它们最终会回归合理价差。他们的模型假设市场波动率维持在历史平均水平附近。然而 1998 年 8 月，俄罗斯债务违约引发全球恐慌，市场波动率在数天内飙升到历史均值的 5 倍以上。

如果 LTCM 使用 GARCH 模型动态监测波动率，他们会发现：早在危机爆发前，模型估计的条件波动率已经开始显著攀升。GARCH 模型的"波动率记忆"特性能够捕捉到这种渐进的紧张状态，远比基于历史均值的静态估计灵敏。遗憾的是，LTCM 过度依赖理论模型，忽视了波动率的动态变化，最终在短短四个月内损失 46 亿美元并导致系统性风险爆发。

要点总结：波动率是时变的，且具有聚集性和持久性。用 GARCH 类模型动态建模波动率，是现代金融风险管理的基石。忽视波动率的动态特征，代价可能是灾难性的。

15.7 本章小结

本章我们学习了波动率建模的核心思想：

  • 金融时间序列的典型特征（厚尾、波动聚集、杠杆效应）
  • ARCH 模型用滞后残差平方预测条件方差
  • GARCH(1,1) 以极简的三参数形式捕捉波动率的长期记忆
  • EGARCH 和 GJR-GARCH 处理杠杆效应——负收益对波动的影响更大
  • 基于 GARCH 的波动率预测可用于 VaR 计算和风险管理

GARCH 模型处理的是单一时间序列的波动率。但在现实经济系统中，多个变量之间相互影响——GDP 变动会影响利率，利率又反作用于 GDP。下一章我们将走进多元时间序列的世界，学习向量自回归模型 (VAR)，它让我们能够同时建模多个交互的时间序列。
