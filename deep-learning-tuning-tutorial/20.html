NLP微调——从预训练到任务适配的艺术

大家好，我是你们的讲师。今天我们来深入探讨NLP模型微调的原理。如果你觉得微调就是"加载预训练模型然后跑几个epoch"，那你可能会遇到很多麻烦——loss爆炸、不收敛、收敛了但效果还不如从零训练。这些问题背后的原因往往不是模型结构有问题，而是微调策略本身需要精细设计。

NLP领域的预训练-微调范式，是深度学习工业化的一个里程碑。一个在数十亿token上预训练好的BERT或GPT模型，可以只用几千条标注数据就在特定任务上达到令人惊讶的效果。但这种威力是有前提的——你对微调的每一个细节都要理解到位。

让我分享一个真实的故事。某团队在做文本分类微调时，直接把BERT-base的学习率设成了1e-4（这是他们之前做CNN迁移学习时的习惯），结果训练loss在第一步就从0.7跳到了8.3，而且完全降不下来。他们花了三天排查数据问题、模型代码问题，最后发现只是学习率设大了10倍。改了学习率到2e-5，模型正常收敛了。像这样的故事，在NLP微调中每天都在发生。

微调为什么需要更低的LR

预训练权重已经包含了极其丰富的语言知识——语法、语义、世界知识。你拿到的不是一个随机初始化的模型，而是一个已经"学会说话"的模型。微调的目标是轻微调整这些知识，让它们适配你的特定任务，而不是重新学习语言本身。

如果学习率太大，会发生灾难性遗忘（Catastrophic Forgetting）——一个大的梯度更新步会"洗掉"预训练阶段学到的语言知识。模型可能会在目标任务上快速拟合训练集，但它的泛化能力和语言理解能力大幅下降。这和CNN微调有本质区别：CNN的低层特征（边缘、纹理）相对通用，可以用稍大的学习率（1e-4到1e-3）；而BERT的自注意力机制对参数扰动比CNN敏感得多，过大的更新步长会迅速破坏已经学好的注意力模式。

那多大的学习率是合适的？在NLP领域有一条铁律：BERT类模型的学习率在1e-5量级。具体来说：

  • 文本分类（情感分析、主题分类）：2e-5是常见起点
  • 文本匹配（语义相似度、NLI）：1e-5 ~ 3e-5，不宜过大
  • 序列标注（NER、POS）：2e-5 ~ 5e-5
  • 阅读理解：3e-5 ~ 5e-5
  • 生成任务（BART、T5微调）：3e-5 ~ 5e-5

wiki中的经验总结很直接："lr，最重要的参数，一般nlp bert类模型在1e-5级别附近，warmup，衰减。"建议用对数刻度搜索：尝试[1e-5, 2e-5, 3e-5, 5e-5]，对比验证集损失变化。如果看到训练损失在第一步就跳到10+，基本就是学习率过大了。

常见错误：学习率从1e-4开始是BERT微调最常见的错误。BERT微调的合理范围很少超过5e-5。将CNN迁移学习的经验直接搬到NLP微调上，几乎必然导致损失爆炸或不收敛。

子词分词的统计基础

子词分词（Subword Tokenization）是现代NLP模型的基石。你可能每天都在用tokenizer，但你真的理解它的统计原理吗？

子词分词的核心矛盾是：词级分词导致词表过大且无法处理未登录词，字符级分词又丢失语义信息且序列过长。子词分词在两者之间取得平衡——常见词保留为完整单元，罕见词拆分为已知的子词片段。

三种主流算法的对比：

算法 合并标准 需要预分词 代表模型
BPE 频率最高 是 GPT、RoBERTa、BART
WordPiece 似然提升最大 是 BERT、DistilBERT
SentencePiece BPE或Unigram LM 否（Unicode序列） T5、LLaMA、Gemma

BPE（Byte Pair Encoding）最初是数据压缩算法。从字符级开始，反复合并出现频率最高的相邻token对，直到达到设定的词表大小。WordPiece与BPE类似但合并标准不同——它选择能使语言模型似然增加最多的token对，而非最频繁的。SentencePiece则将输入视为Unicode字符序列，不需要预分词，因此更加语言无关。

这里有一个极其重要的实践要点：tokenizer不一致是微调第一坑。你微调一个BERT模型，必须使用和预训练时完全相同的tokenizer配置。不同模型的分词器不能互换使用——BERT的WordPiece词表和GPT的BPE词表完全不同，用错分词器会产出无意义的token ID序列。微调时修改词表（比如增删token）会破坏预训练权重，导致严重冷启动问题。

词表大小的选择也有讲究：中型模型（100M~1B参数）通常用32K，大型模型用50K~64K，多语言模型需要更大的100K~250K。更小的词表意味着更短的序列但更大的embedding矩阵，需要在显存和表示力之间权衡。

文本数据增强的特殊性

NLP的数据增强和CV有本质不同。你不能像旋转图片一样旋转一段文字——那样会破坏语义。NLP的数据增强需要在保持语义的前提下引入多样性。

EDA（Easy Data Augmentation）是NLP数据增强最经典的基准方法，包含四件套：

  1. 同义词替换（SR）：随机选择n个非停用词，用同义词替换。这是最基础也最安全的方法。
  2. 随机插入（RI）：随机选择一个词，在其同义词中随机选一个插入到句中随机位置。
  3. 随机交换（RS）：随机选择两个词并交换位置。
  4. 随机删除（RD）：以概率p随机删除每个词。

EDA在小数据集上效果显著。想象你只有500条标注数据，EDA可以将有效训练数据量扩充2-3倍。但需要注意增强比例——增强太多会引入噪声，通常建议每条原始样本生成1-2条增强样本即可。

回译（Back Translation）是另一种强力方法：将中文翻译成英文，再翻译回中文。因为翻译过程会引入措辞变化但保留语义，回译生成的数据质量通常高于同义词替换。代价是需要额外的翻译模型，但效果往往值得。

除了显式的数据增强，NLP还有一个"免费的正则化"手段——对抗训练。对抗训练（FGM/PGD）的核心思想是在Embedding层添加微小的对抗扰动，让模型对输入的小变化不敏感。wiki中评价："对于nlp任务，采用embedding扰动的方式有奇效。"

# FGM 对抗训练（PyTorch风格）
for batch in dataloader:
    loss = model(batch)
    loss.backward()
    # 保存原始embedding参数
    embedding = model.bert.embeddings.word_embeddings
    emb_backup = embedding.weight.data.clone()
    # 计算扰动
    grad = embedding.weight.grad.data
    norm = grad.norm()
    if norm != 0:
        epsilon = 1.0
        embedding.weight.data += epsilon * grad / norm
        # 再用扰动后的embedding计算一次loss
        loss_adv = model(batch)
        loss_adv.backward()
    # 恢复原始embedding
    embedding.weight.data = emb_backup

这段代码展示的是Fast Gradient Method（FGM）——在Embedding上沿着梯度方向加一个小扰动，然后用扰动后的Embedding重新算一次loss。代码几乎为零成本，但能显著提升模型鲁棒性。PGD（Projected Gradient Descent）是多步版本的FGM，效果更好但计算成本也更高。

实战建议：对抗训练可以和微调天然结合。在微调BERT时开启FGM对抗训练，通常在分类和NER任务上能稳定提升0.5-1个点，而且"实现成本低，有奇效"。

BERT微调实操指南

最后，我们来看BERT微调的几个实操要点，这些是区分"会用"和"会调"的关键。

第一，Slanted Triangular LR Scheduler。这是BERT微调中最有效的学习率调度策略。它的设计原理是"快速升温+缓慢下降"：训练初期用非常小的学习率（约1/10的目标LR）预热几个epoch，让优化器状态（Adam的一阶/二阶动量）稳定下来，然后快速升温到目标LR，再以余弦或线性方式缓慢衰减。这个形状像一个不对称的三角形——陡升缓降。warmup步数建议占总步数的3%~10%，简单任务取低值，复杂任务取高值。Transformer系列模型基本必须开启warmup，wiki指出："lr_scheduler的warmup步数设置：一般遵循模型复杂，步数多点；数据集大，步数少点。"

第二，Discriminative Fine-tuning（分层学习率）。不同层对微调的需求是不同的：底层学到的是通用语言特征（语法、词法），需要稳定的学习率小调；顶层学到的是任务相关特征，需要更大的学习率适配。所以一个合理做法是：底层学习率小、顶层学习率大，中间层线性插值。比如BERT-base有12层，你可以设底层学习率为1e-5、顶层为3e-5，中间层等比插值。

第三，长文本的截断策略。BERT的最大输入长度是512 token。如果你的文本超过512，简单截断前512会丢失关键信息。更好的做法是：

  • 头尾截断：保留文档的前128和后384个token，很多关键信息出现在开头和结尾。
  • 滑动窗口：将长文本切分为多个512的窗口，分别预测再聚合。
  • 分块+Pooling：将文本分成多个chunk，分别过BERT后做mean pooling聚合。
  • 换Longformer：如果长文本是核心场景，考虑使用Longformer（支持4096+长度），wiki指出"对于长文本来说longformer的效果高于各种bert变体"。

还有一个常被忽视的细节：分布式训练时的学习率缩放。分布式训练时batch size成倍增大，应相应降低学习率（线性缩放法则：lr_new = lr_base × sqrt(batch_new / batch_base)）。如果单卡batch size=16用2e-5，4卡分布式时batch size变成64，学习率调整为2e-5 × sqrt(64/16) ≈ 3e-5。
