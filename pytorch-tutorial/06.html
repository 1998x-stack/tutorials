第6章 损失函数与优化器

大家好，我是你们的讲师。上一章我们学会了构建神经网络——无论是手动实现还是用 nn.Module 封装。但模型搭建起来之后，我们面临两个关键问题：

第一，怎么衡量模型的好坏？模型预测的结果跟真实标签差距有多大？这就是损失函数（Loss Function）要做的事情。

第二，怎么让模型变得更好？有了差距的度量之后，如何调整模型的参数来缩小这个差距？这就是优化器（Optimizer）的职责。

如果把训练模型比作射箭：损失函数告诉你箭离靶心有多远，优化器则告诉你下一箭该往哪个方向调整。这一章我们就来透彻地理解这两个核心组件。

损失函数：模型好坏的度量尺

损失函数（也叫代价函数）是一个数学函数，它接收模型的预测值和真实标签，输出一个标量——数值越小说明预测越准，数值越大说明误差越大。

PyTorch 在 torch.nn 模块中提供了丰富的损失函数。下面是三个最常用的：

损失函数 适用场景 数学形式（简化） 输出范围 注意事项
MSELoss 回归任务（预测连续值） mean((y_pred - y_true)^2) [0, +∞) 对大误差敏感（平方放大），易受异常值影响
CrossEntropyLoss 多分类任务 内部结合了 LogSoftmax + NLLLoss [0, +∞) 输入是 logits，标签是 class index（不是 one-hot）
BCELoss 二分类任务 -mean(y*log(p) + (1-y)*log(1-p)) [0, +∞) 输入必须是概率（0~1），前面需要加 Sigmoid

还有其他一些值得了解的损失函数：

  • L1Loss（MAE）—— 回归任务，对异常值更鲁棒
  • BCEWithLogitsLoss —— 更稳定的二分类损失（内部组合了 Sigmoid + BCELoss）
  • KLDivLoss —— KL 散度，用于知识蒸馏、变分推断
  • MarginRankingLoss —— 排序学习
这里讲一个我亲眼见过的经典 bug：

2018 年，我们团队的一个项目在做图像分类。一个同学在准备标签时做了 one-hot 编码（[[0,1,0], [1,0,0], ...]），然后直接喂给了 CrossEntropyLoss。结果 loss 不断下降，但准确率始终在 chance level 徘徊。他困惑了整整两天。

问题出在哪？CrossEntropyLoss 期望的标签是类别索引（tensor([1, 0, ...])），而不是 one-hot 向量。如果传入 one-hot，PyTorch 会把它当作一个 3 维的类别分布来解读——逐元素去算 loss——结果完全错误但又不会报错。

正确的做法：

# ❌ 错误：传入 one-hot 编码
target_onehot = torch.eye(10)[labels]  # [B, 10]
loss = F.cross_entropy(logits, target_onehot)  # 不报错但结果全错！

# ✅ 正确：传入类别索引
loss = F.cross_entropy(logits, labels)  # labels shape [B]，值是 0~9
import torch
import torch.nn as nn

# MSELoss — 回归
mse = nn.MSELoss()
y_pred = torch.tensor([2.5, 0.0, 2.0])
y_true = torch.tensor([3.0, 0.5, 2.0])
print(f"MSE Loss: {mse(y_pred, y_true):.4f}")  # 比较温和的惩罚

# CrossEntropyLoss — 多分类（最常用！）
ce = nn.CrossEntropyLoss()
logits = torch.tensor([[1.0, 2.0, 0.5],     # 模型原始输出（logits）
                        [0.1, 3.0, 1.0]])
labels = torch.tensor([1, 1])                 # 正确类别索引
print(f"CrossEntropy Loss: {ce(logits, labels):.4f}")  # 小值说明分类好

# BCELoss — 二分类（需要先过 Sigmoid）
bce = nn.BCELoss()
probs = torch.sigmoid(torch.tensor([1.5, -0.5, 2.0]))
binary_labels = torch.tensor([1.0, 0.0, 1.0])
print(f"BCE Loss: {bce(probs, binary_labels):.4f}")

# 推荐替代：BCEWithLogitsLoss（更稳定，内部做 Sigmoid）
bce_logits = nn.BCEWithLogitsLoss()
logits_binary = torch.tensor([1.5, -0.5, 2.0])
print(f"BCEWithLogits Loss: {bce_logits(logits_binary, binary_labels):.4f}")
推荐使用 BCEWithLogitsLoss 而不是 BCELoss：前者内部做了数值稳定的 log-sum-exp 技巧，比"手动 Sigmoid + BCELoss"更精确、更稳定。同理，如果你在做多分类且想自定义 softmax 之前的操作，可以用 nn.CrossEntropyLoss（它内部就是 LogSoftmax + NLLLoss）。

梯度下降：从山顶走到山谷

现在有了损失函数，我们需要通过调整模型参数来使损失函数的值最小化。这个过程就是优化（Optimization），最核心的方法是梯度下降（Gradient Descent）。

想象你站在一座山的山坡上，浓雾弥漫，看不见整座山的地形。但你知道当前位置的坡度（梯度）。梯度下降的策略很简单：沿着最陡的下坡方向走一小步。不断重复，最终你会走到山谷的最低点。

数学上，参数更新的公式为：θ_new = θ_old - lr * ∇θ(Loss)

其中 lr（学习率）决定了每一步的大小——步子太小走得太慢，步子太大可能直接跨过山谷跳到对面山坡上。

SGD 优化器

随机梯度下降（SGD）是最经典的优化器。虽然名字叫"随机"，但在 PyTorch 中，SGD 默认使用的是 mini-batch 梯度下降——每个 batch 计算一次梯度并更新。

import torch.optim as optim

# 基本 SGD
optimizer = optim.SGD(model.parameters(), lr=0.01)

# 带 Momentum 的 SGD（更常用的版本）
optimizer = optim.SGD(model.parameters(), lr=0.01, momentum=0.9)

# 带 weight_decay（L2 正则化）的 SGD
optimizer = optim.SGD(model.parameters(), lr=0.01, momentum=0.9, weight_decay=1e-4)

Momentum 是什么？想象一个球从山坡滚下来——它不仅受当前梯度的影响，还保持着之前的"惯性"。momentum=0.9 表示保留前一步更新方向的 90%。这能有效抑制梯度震荡，加速收敛。

Weight Decay 是什么？它是 L2 正则化——在损失函数中隐式地加入 λ * ||θ||² 项，惩罚过大的参数值，防止过拟合。

Adam 优化器

Adam（Adaptive Moment Estimation）是目前最流行的优化器之一。它结合了 Momentum（动量）和 RMSProp（自适应学习率）的优点：

  • 自适应学习率：每个参数有自己的学习率——频繁更新的参数学习率自动降低，稀疏更新的参数学习率自动升高
  • 含动量：同 SGD Momentum
  • 偏差校正：在训练初期，动量估计有偏，Adam 会自动校正
# 标准 Adam
optimizer = optim.Adam(model.parameters(), lr=1e-3)

# Adam 的超参数（通常不需要动，但了解一下）
optimizer = optim.Adam(
    model.parameters(),
    lr=1e-3,        # 学习率（Adam 默认推荐 1e-3）
    betas=(0.9, 0.999),  # 动量衰减系数和平方梯度衰减系数
    eps=1e-8,       # 数值稳定性
    weight_decay=0  # L2 正则化（Adam 中建议用 AdamW 代替）
)

# AdamW —— 修正了 weight_decay 实现方式的版本（推荐！）
optimizer = optim.AdamW(model.parameters(), lr=1e-3, weight_decay=1e-4)
SGD vs Adam 怎么选？
场景推荐优化器理由
CV 领域（CNN）SGD + MomentumSGD 的泛化能力往往更好，收敛到的解更平坦
NLP / TransformerAdam / AdamW自适应学习率对不同词频的 Embedding 友好
新手 / 快速实验Adam对学习率不敏感，基本开箱即用
大模型预训练AdamW修正的 weight_decay 在大规模训练中效果更好
强化学习Adam非平稳目标，自适应学习率帮助大

优化三件套：zero_grad → backward → step

我们已经多次提到这个"铁三角"了，但这一章我们要把它的底层原理彻底讲清楚：

for batch in dataloader:
    # 第1步：清空梯度
    optimizer.zero_grad()
    # 🔍 这步在做什么？遍历所有参数，把 param.grad 设为 None（或零）
    # 为什么需要？因为梯度是累积的（第4章讲过）

    # 第2步：前向 + 计算损失
    y_pred = model(x)
    loss = loss_fn(y_pred, y)

    # 第3步：反向传播
    loss.backward()
    # 🔍 这步在做什么？从 loss 出发反向遍历计算图
    # 计算每个参数的梯度，存入 param.grad

    # 第4步：更新参数
    optimizer.step()
    # 🔍 这步在做什么？遍历每个参数，执行 θ -= lr * grad
    # 不同的优化器（SGD/Adam）在这一步的公式不同
关于 zero_grad 的性能优化：optimizer.zero_grad() 默认把梯度设为 0。在 PyTorch 2.x 中，推荐使用 optimizer.zero_grad(set_to_none=True)，它把梯度设为 None 而不是 0——这样内存分配更高效。如果你的代码中对 .grad 有非 None 的依赖判断，注意这个变化。

下面是一个完整的优化循环示例（用我们上一章的线性回归模型）：

import torch
import torch.nn as nn
import torch.optim as optim

# 数据
torch.manual_seed(42)
x = torch.linspace(-1, 1, 100).reshape(-1, 1)
y = 2 * x + 1 + 0.1 * torch.randn_like(x)

# 模型 + 损失 + 优化器
model = nn.Linear(1, 1)
loss_fn = nn.MSELoss()
optimizer = optim.SGD(model.parameters(), lr=0.1)

# 训练
for epoch in range(200):
    # 前向
    y_pred = model(x)
    loss = loss_fn(y_pred, y)

    # 优化三件套
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    if (epoch + 1) % 40 == 0:
        print(f"Epoch {epoch+1}, Loss: {loss.item():.6f}")

print(f"训练结果: w={model.weight.item():.4f}, b={model.bias.item():.4f}")
# 接近 w=2.0, b=1.0

输出：

Epoch 40, Loss: 0.017204
Epoch 80, Loss: 0.011569
Epoch 120, Loss: 0.011321
Epoch 160, Loss: 0.011283
Epoch 200, Loss: 0.011269
训练结果: w=1.9893, b=1.0295

学习率调参的故事

讲一个真实的故事。2020 年，我指导的一个同学在训练一个图像分类模型时，loss 从一开始就是 NaN。他检查了数据、模型，都没发现问题。最后发现是学习率设成了 10.0 而非 0.01——这个模型只有几层，lr=10 意味着参数更新幅度太大，直接跳出了函数的有效定义域，数值爆炸了。

反过来，另一个同学把学习率设成了 1e-7，loss 下降得像乌龟爬——训练了 100 个 epoch 后 loss 从 2.3 降到了 2.2，照这个速度需要训练一万个 epoch 才能收敛。

学习率的"黄金法则"：
  • 如果 loss 变成 NaN → 学习率太大，减小 10 倍试试
  • 如果 loss 下降太慢 → 学习率太小，增大 10 倍试试
  • 学习率"调优区间"通常在 [1e-4, 1e-2] 之间（对 Adam 来说 1e-3 是默认好选择）
  • 可以用 lr_finder（学习率范围测试）来快速找到合适的初始学习率
  • 建议配合学习率调度器（下章讲）使用——先大后小

优化器超参数速查

这里总结一下两个最常用优化器的关键超参数：

优化器 超参数 默认值 调参建议
SGD lr 0.001（需手动调） 常见范围 [1e-4, 1e-1]，依任务规模而定
momentum 0 经典值 0.9，一般设为 0.9 或 0.99
weight_decay 0 GPU 友好，更快的速度
Adam / AdamW lr 0.001 通常不用动，非常鲁棒。需要调时从 1e-4~3e-3 试
betas (0.9, 0.999) 极少需要调整。对于稀疏梯度可以增大 beta1
weight_decay 0 AdamW 建议设为 1e-4~1e-2，Adam 中慎用

本章小结

这一章我们学习了训练神经网络的两个核心组件：

  • 损失函数：衡量模型预测与真实标签的差距。回归用 MSE，多分类用 CrossEntropyLoss，二分类用 BCEWithLogitsLoss。
  • 优化器：根据梯度调整参数。SGD 经典通用，Adam 则更省心。
  • 优化三件套：zero_grad() → backward() → step()，顺序固定不能乱。
  • 学习率是最关键的超参数——太大 NaN，太小蜗牛爬。
