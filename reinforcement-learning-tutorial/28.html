第二十八章：综合实战：Atari智能体

28.1 项目概览

欢迎来到本课程的最后一章！在前面的二十七章里，我们从马尔可夫决策过程出发，一路走到策略梯度、PPO、SAC甚至RLHF。今天，我们要把所学的一切汇聚到一个经典项目上——训练一个深度Q网络（DQN）智能体，让它在Atari游戏（Pong或Breakout）中达到超人水平。

为什么要选Atari作为毕业项目？因为Atari是强化学习的"ImageNet时刻"。2013年DeepMind首次用DQN在Atari上超越人类水平，开启了深度强化学习的时代。Atari环境兼具多样性、视觉丰富性和明确的奖励信号——从乒乓球到打砖块，每一帧画面都蕴含着需要智能体去理解的结构。在这个项目中，你将亲手搭建一条从原始像素到游戏智能体的完整训练管线。

项目目标： 实现一个端到端的DQN训练管线，包含环境包装、CNN网络、经验回放与评估模块。最终得到一个能在Pong中稳定击败内置AI的智能体。

28.2 环境配置

首先安装必要的依赖。Atari环境在Gymnasium生态中使用ale-py作为底层模拟器。安装指令如下：

pip install gymnasium[atari] gymnasium[accept-rom-license] ale-py

环境包装是整个训练流程的关键环节。原始Atari模拟器输出的画面尺寸为210×160，帧率每秒60帧，直接送入神经网络既不经济也不稳定。我们需要一套精心设计的包装器栈：

  • NoopResetEnv：在回合开始时随机执行若干次无操作，增加初始状态的多样性，避免过拟合固定的起始画面。
  • MaxAndSkipEnv：每4帧取一次动作，返回这4帧中每通道的最大值。这既降低了时间分辨率，又解决了帧闪烁问题。
  • EpisodicLifeEnv：将游戏中的每条命视为一个episode，但只有真正Game Over才算终结。这能大幅增加训练样本的多样性。
  • FireResetEnv：很多Atari游戏需要按"开火"键才能开始，这个包装器自动处理这件事。
  • WarpFrame：将画面缩放为84×84灰度图，极大减少输入维度。
  • FrameStack：将连续4帧堆叠成一个状态，让网络能够感知运动信息。没有这个步骤，智能体连球往哪个方向飞都判断不了。
import gymnasium as gym
from ale_py import ALEInterface
from gymnasium.wrappers import (
    NoopResetEnv, MaxAndSkipEnv, EpisodicLifeEnv,
    FireResetEnv, WarpFrame, FrameStack
)

def make_atari_env(env_name="PongNoFrameskip-v4"):
    env = gym.make(env_name, render_mode="rgb_array")
    env = NoopResetEnv(env, noop_max=30)
    env = MaxAndSkipEnv(env, skip=4)
    env = EpisodicLifeEnv(env)
    if "FIRE" in env.unwrapped.get_action_meanings():
        env = FireResetEnv(env)
    env = WarpFrame(env, width=84, height=84, grayscale=True)
    env = FrameStack(env, num_stack=4)
    return env
踩坑实录： 我第一次训练Atari时，忘了加FrameStack包装器。结果训练了两天，智能体的得分始终在随机水平徘徊。我盯着屏幕看了半天，才意识到：只用单帧画面，网络根本看不见球的速度和方向。就像让一个人闭着一只眼睛接乒乓球一样——不是不努力，是信息不够。加上FrameStack之后，仅仅几万步，得分曲线就开始抬头了。这个教训让我记住：在任何涉及运动的环境里，时间信息比空间信息更重要。

28.3 网络架构

Atari DQN的CNN架构经过大量实验验证，已经成为事实标准。其设计哲学是：用较深的卷积层逐步提取空间特征，最后通过全连接层映射到各个动作的Q值。具体的层结构如下：

import torch
import torch.nn as nn
import torch.nn.functional as F

class AtariDQN(nn.Module):
    def __init__(self, n_actions):
        super().__init__()
        self.conv1 = nn.Conv2d(4, 32, kernel_size=8, stride=4)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=4, stride=2)
        self.conv3 = nn.Conv2d(64, 64, kernel_size=3, stride=1)
        self.fc1 = nn.Linear(64 * 7 * 7, 512)
        self.fc2 = nn.Linear(512, n_actions)

    def forward(self, x):
        x = x / 255.0  # 归一化到 [0,1]
        x = F.relu(self.conv1(x))
        x = F.relu(self.conv2(x))
        x = F.relu(self.conv3(x))
        x = x.view(x.size(0), -1)
        x = F.relu(self.fc1(x))
        return self.fc2(x)

输入维度为 (batch, 4, 84, 84)，其中4是堆叠帧数，84是缩放后的画面尺寸。第三层卷积输出是64通道、7×7的空间尺寸，展平后得到 64×7×7 = 3136 维向量，依次经过512维隐藏层后输出动作数目的Q值。这个结构在Pong（6个动作）和Breakout（4个动作）上都有出色的表现。

28.4 完整训练流程

训练超参数直接沿用Nature DQN论文的配置。下表列出了最关键的超参数及其取值依据：

超参数取值说明
replay buffer size1,000,000容纳最近100万条经验，足够覆盖多轮游戏的内容
learning rate2.5 × 10⁻⁴Adam优化器下的稳定学习率，比RMSprop略低
batch size32从经验池中采样的批量大小，平衡效率与稳定性
target network update每10,000步目标网络冻结更新的间隔，防止自举偏差放大
ε 起始 / 终止1.0 → 0.1探索率在100万步内线性退火，之后固定为0.1
train frequency每4步不是每步都训练，而是每隔4个环境步训练一次
total steps10,000,000总训练步数，在单GPU上约需1~2天

训练脚本的核心循环如下。注意其中经验回放的采样、目标网络的同步和探索率的退火三个关键机制：

def train_dqn(env, n_actions, total_steps=10_000_000):
    online_net = AtariDQN(n_actions).to(device)
    target_net = AtariDQN(n_actions).to(device)
    target_net.load_state_dict(online_net.state_dict())
    optimizer = torch.optim.Adam(online_net.parameters(), lr=2.5e-4)
    replay = ReplayBuffer(capacity=1_000_000)

    state, _ = env.reset()
    eps = 1.0
    for step in range(1, total_steps + 1):
        # epsilon-greedy 动作选择
        if random.random() < eps:
            action = env.action_space.sample()
        else:
            with torch.no_grad():
                q = online_net(state.unsqueeze(0).to(device))
                action = q.argmax().item()

        next_state, reward, terminated, truncated, _ = env.step(action)
        replay.push(state, action, reward, terminated, next_state)

        # 训练
        if step % 4 == 0 and len(replay) >= 32:
            batch = replay.sample(32)
            loss = compute_dqn_loss(online_net, target_net, batch, gamma=0.99)
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

        # 同步目标网络
        if step % 10_000 == 0:
            target_net.load_state_dict(online_net.state_dict())

        # 退火探索率
        eps = max(0.1, 1.0 - step / 1_000_000)

        state = next_state
        if terminated or truncated:
            state, _ = env.reset()

        if step % 100_000 == 0:
            print(f"Step {step}, eps={eps:.3f}")
注意： 完整训练需要10M~50M步。在单张RTX 3090上运行Pong需要大约24小时。建议先用1M步快速验证损失函数是否正确下降，再启动长时间训练。提前关掉训练是最大的浪费——一定要先做小规模验证。

28.5 结果分析与课程总结

训练完成后，最重要的环节是评估。不要只看最终得分——建议从三个维度观察：

  • 多episode平均回报：冻结网络，在不加探索的情况下运行至少100个episode，计算平均得分。Pong的得分范围是[-21, +21]，人类水平大约在+9左右，优秀的DQN可以达到+18以上。
  • Q值追踪：记录训练过程中每个状态的最大Q值，如果Q值持续增长但实际回报没有改善，说明网络产生了过度估计。这时可以考虑加入Double DQN。
  • 视频回放：用gymnasium.wrappers.RecordVideo记录几个完整的episode，直观地看智能体的行为。很多时候，看视频比看曲线更能发现问题。

一条健康的训练曲线应该是：前期收益徘徊在最低水平（智能体还在随机试探），中期有一个陡峭的上升段（Q网络开始学到有用的策略），后期逐渐收敛到稳定水平。如果曲线剧烈震荡，检查学习率是否过高或buffer size是否太小。

扩展阅读： Rainbow DQN将在本章的DQN基础上融合Double DQN、Dueling DQN、优先经验回放、多步学习、分布式Q学习和噪声网络六大改进。如果你想让Atari智能体更上一层楼，Rainbow是最好的起点。

课程结语

到这里，我们二十八章的强化学习之旅就告一段落了。回过头看，我们从最基础的马尔可夫决策过程开始，理解了贝尔曼方程如何描述最优决策的本质；然后一步步探索了无模型方法——蒙特卡洛、时序差分、SARSA、Q学习；接着进入深度学习的时代，看到了DQN如何把强化学习与卷积神经网络结合在一起，让智能体学会从像素中理解世界；再后来，我们拥抱了策略梯度、Actor-Critic、PPO、SAC，理解了为什么它们在某些场景下比值函数方法更强大；最后，我们触及了模仿学习、逆强化学习、世界模型、多智能体系统和RLHF这些前沿领域。

强化学习不是一个"搭好环境、调好参数、按下按钮"就能出成果的领域。它需要你对算法原理有深刻理解，需要你有耐心观察训练曲线，需要你在模型不断失败的时候仍然相信"下一次就能学到"。这正是它的魅力所在——当你在深夜看到损失函数终于开始下降，当智能体第一次打出制胜一击，当平均得分曲线冲破人类基线的那一刻，所有的等待和调试都是值得的。

给每一位同学的话： 算法和数学是工具，真正驱动进步的，是你提出问题、设计实验、从失败中学习的能力。不要害怕在GitHub上读别人的代码，不要害怕自己的模型不收敛。强化学习的每一个"bug"都是一次深入理解算法的机会。拿起代码，去训练你的第一个Atari智能体吧——游戏的胜负只是结果，真正重要的是你在过程中学到的东西。祝你在强化学习的道路上一路顺风！
