第5章 Verilog HDL入门

大家好，我是你们的讲师。之前几章我们从物理一路学到了电路——半导体物理 → MOS晶体管 → CMOS数字电路。现在我们面临一个重要的"范式转换"：从画电路图到写代码。

你可能已经跃跃欲试——"终于到了可以写代码的部分了！"确实，对于很多同学来说，Verilog是他们进入IC设计的"第一道门"。但我要提醒你：Verilog不像C语言那样是"顺序执行"的，它是"并行执行"的硬件描述语言。这个思维转变是初学者遇到的最大障碍。我们从这一章开始，就要帮你跨过这道坎。

5.1 Verilog语言基础

历史与定位

Verilog诞生于1984年，由Gateway Design Automation公司开发。1990年，Cadence收购了Gateway，随后将Verilog推向开放标准。1995年，IEEE发布了Verilog的第一个正式标准——IEEE 1364-1995。后续的版本（2001、2005）增加了大量新特性，而SystemVerilog（IEEE 1800）则在Verilog的基础上增加了面向对象编程、断言（assertion）、覆盖率收集等高级验证特性。

Verilog的最大竞争对手是VHDL（VHSIC Hardware Description Language，1980年代由美国国防部资助开发）。两者在本质上能做的事情是相同的，但哲学不同：

  • Verilog：更接近C语言，语法灵活、简洁，适合RTL级数字设计。在北美和中国大陆的IC设计公司中占主导地位。
  • VHDL：更接近Ada或Pascal，语法严谨、强类型，适合大型复杂系统的行为级建模。在欧洲和航空航天/军工领域仍有广泛应用。

总的来说，如果你在中国或北美的数字IC设计公司工作，大概率用Verilog/SystemVerilog；如果你是做FPGA开发（尤其是Xilinx/AMD生态），两种语言都有广泛用户。

模块结构

Verilog的基本设计单元是模块（module）。一个模块可以描述一个门、一个加法器、一个CPU、甚至整个SoC——这就是硬件描述的"层次化"哲学。

// 一个完整的Verilog模块示例：4位计数器 + Testbench

// ---- 设计模块：4位同步计数器 ----
module counter_4bit (
    input  wire       clk,      // 时钟
    input  wire       rst_n,    // 异步复位，低电平有效
    input  wire       en,       // 计数使能
    output reg  [3:0] count     // 4位计数值
);

    // 时序逻辑：在时钟上升沿更新计数值
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            count <= 4'd0;           // 非阻塞赋值（时序逻辑）
        else if (en)
            count <= count + 1'b1;   // 加1计数
        else
            count <= count;          // 保持
    end

endmodule

// ---- 数据类型的说明 ----
// wire  : 线网类型，用于组合逻辑连接（类似于"导线"）
// reg   : 寄存器类型，用于always块中被赋值的变量（不一定映射为"寄存器"，在组合逻辑中可映射为wire）
// 在Verilog中，模块的输入必须是wire，输出可以是wire或reg

// ---- Testbench（测试平台） ----
module tb_counter;

    // 声明信号
    reg        clk;
    reg        rst_n;
    reg        en;
    wire [3:0] count;

    // 实例化被测试模块
    counter_4bit u_dut (
        .clk    (clk),
        .rst_n  (rst_n),
        .en     (en),
        .count  (count)
    );

    // 生成时钟：周期为10ns (100MHz)
    initial begin
        clk = 0;
        forever #5 clk = ~clk;   // 每5ns翻转一次
    end

    // 测试激励
    initial begin
        // 初始化
        rst_n = 0;
        en    = 0;

        // 释放复位
        #20 rst_n = 1;

        // 使能计数
        #20 en = 1;

        // 运行100ns
        #100;

        // 暂停计数
        en = 0;

        // 再观察50ns
        #50;

        // 结束仿真
        #10 $finish;
    end

    // 监控输出
    initial begin
        $monitor("time=%0t  clk=%b  rst_n=%b  en=%b  count=%d",
                 $time, clk, rst_n, en, count);
    end

    // 生成波形文件（供GTKWave等查看）
    initial begin
        $dumpfile("counter.vcd");
        $dumpvars(0, tb_counter);
    end

endmodule

5.2 模块与端口

Verilog的模块（module）定义以module关键字开始，以endmodule结束。模块的端口（port）定义了与外部环境的接口：

  • input：输入端口，内部读取，外部驱动。
  • output：输出端口，内部驱动，外部读取。
  • inout：双向端口（如三态总线），需要特殊处理。

模块实例化是将一个模块"嵌入"到另一个模块中——这是实现层次化设计的核心机制。实例化时有两种端口连接方式：按端口位置连接（不推荐，容易出错）和按端口名称连接（推荐）。

参数传递使得模块可以"定制化"——例如一个N位计数器可以通过parameter参数来指定宽度：

// 带参数的模块
module counter #(
    parameter WIDTH = 8
) (
    input  wire             clk,
    input  wire             rst_n,
    output reg  [WIDTH-1:0] count
);
    // ... 实现
endmodule

// 实例化时指定不同的宽度
counter #(.WIDTH(4)) u_cnt4 ( ... );  // 4位计数器
counter #(.WIDTH(8)) u_cnt8 ( ... );  // 8位计数器

5.3 组合逻辑描述

在Verilog中，有两种主要方式描述组合逻辑：

assign语句（连续赋值）

连续赋值用于描述简单的组合逻辑——赋值语句右侧的信号变化时，左侧立即更新：

assign sum = a ^ b;        // 异或
assign carry = a & b;      // 与
assign y = s ? a : b;      // 2选1多路选择器
assign {cout, sum} = a + b + cin;  // 拼接操作

always块组合逻辑

对于更复杂的组合逻辑，使用always @(*)或always @(敏感信号列表)块。注意：在组合逻辑的always块中，必须使用阻塞赋值（=）：

// 组合逻辑always块 — 使用阻塞赋值
always @(*) begin
    case (sel)
        2'b00: out = a;
        2'b01: out = b;
        2'b10: out = c;
        2'b11: out = d;
        default: out = 'bx;  // 无关项
    endcase
end

阻塞赋值 vs 非阻塞赋值

这是Verilog初学者的头号陷阱——总结一下：

特性 阻塞赋值 = 非阻塞赋值 <=
执行方式 顺序执行（语句顺序决定结果） 并行执行（所有RHS在同一时刻计算，然后同时更新LHS）
使用场景 组合逻辑 always @(*) 时序逻辑 always @(posedge clk)
典型用法 计算中间值、组合逻辑功能 寄存器更新、移位寄存器、计数器
模拟行为 立即更新，可能产生竞争 在同一时间步结束时统一更新
综合结果 组合逻辑 边沿触发的时序逻辑（触发器）
经典踩坑——"史上最常见的Verilog bug"：几乎每一个Verilog初学者都会踩到这个坑。想象你写了一个移位寄存器，在always @(posedge clk)块中错误地使用了阻塞赋值：a = b; b = a;——你期望的结果是a得到b的值、b得到a的值（交换），但实际上，由于阻塞赋值的顺序执行特性，a = b先执行（a已经变成了b），然后b = a执行（b变成了b，而不是原来的a）——两个寄存器都变成了b的值！正确的写法应该是用a <= b; b <= a;（用非阻塞赋值实现并行更新）。这个bug在仿真中非常痛苦——因为"逻辑看起来对，但行为不对"。记住这个黄金法则：时序逻辑用非阻塞赋值，组合逻辑用阻塞赋值，不要在同一个always块中混用两者。

5.4 时序逻辑描述

时序逻辑的核心是always @(posedge clk)或always @(negedge clk)——所有时序逻辑都使用非阻塞赋值（<=）：

// D触发器
always @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        q <= 1'b0;
    else
        q <= d;
end

// 移位寄存器（4级）
always @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        {q3, q2, q1, q0} <= 4'b0;
    else
        {q3, q2, q1, q0} <= {q2, q1, q0, din};
end

// 同步复位 vs 异步复位
// 同步复位：复位只在时钟沿生效
always @(posedge clk) begin
    if (rst_sync)     // 同步复位
        q <= 1'b0;
    else
        q <= d;
end

// 异步复位：复位信号立即生效（不依赖时钟）
always @(posedge clk or negedge rst_n) begin
    if (!rst_n)       // 异步复位
        q <= 1'b0;
    else
        q <= d;
end

异步复位的优势是复位响应快（不需要等时钟沿），但缺点是复位信号需要在时钟域内做异步复位同步释放处理，否则可能导致亚稳态。同步复位没有亚稳态问题，但需要时钟工作才能复位——如果时钟停了，就复不了位。

5.5 Testbench编写

写RTL代码只是芯片设计工作的一部分——甚至不是最大的一部分。验证工程师的数量往往是设计师的1.5-2倍，而验证的第一步就是写testbench（测试平台）。

一个基本的testbench结构：

  1. 实例化被测试模块（DUT, Device Under Test）
  2. 生成时钟和复位信号
  3. 施加激励（测试向量）
  4. 检查输出（人工查看波形或自动比对）
  5. 结束仿真

常用的仿真系统任务：

  • $display：在仿真控制台输出文本（类似C的printf）
  • $monitor：持续监控信号变化，有变化自动输出
  • $time：返回当前仿真时间
  • $finish：结束仿真
  • $dumpvars / $dumpfile：生成VCD波形文件供查看

更高级的验证方法学——如UVM（Universal Verification Methodology）、约束随机验证、功能覆盖率驱动验证——我们会在第11章（功能验证方法学）中详细讨论。

5.6 本章小结

这一章我们完成了从"画电路"到"写电路"的思维转换。你学习了Verilog的基本语法、模块与端口的定义方式、组合逻辑和时序逻辑的描述方法——包括阻塞与非阻塞赋值这个经典陷阱——以及如何编写一个简单的testbench来验证你的设计。

你现在应该可以写一个简单的模块并用仿真来验证它了。但Verilog只是"描述工具"——真正的芯片设计远不止写RTL。下一章，我们将走上一条完全不同的路线：从物理层面理解芯片是如何制造的。是的，我们在"最高层"（Verilog）短暂停留后，要再次回到"最底层"——去看看你写的那些RTL代码最终是如何在晶圆厂里变成真正的芯片的。芯片制造工艺。
