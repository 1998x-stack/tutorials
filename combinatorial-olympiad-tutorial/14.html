第十四章：数学归纳法与加强命题

大家好，我是你们的讲师。数学归纳法可能是各位最早接触的"高级"证明方法——但也是最容易被低估的。很多同学觉得归纳法就是"n=1成立，假设n成立推出n+1成立"的套公式游戏。但真正到了竞赛中，你往往会发现：原命题的归纳假设根本不够用。

来看一个经典困境："证明 1 + 1/2² + 1/3² + ... + 1/n² < 2。" 如果你假设n时不等式成立，想推n+1时也成立——你会发现n+1时左边多了一项 1/(n+1)²，但右边仍然是2。归纳假设告诉你n时左边 < 2，但加上一个正项后，你怎么保证仍然 < 2？

这就是归纳法最核心的"痛点"：归纳假设太弱，推不动下一步。解决方案听起来很反直觉——加强命题：你先证明一个更强的结论，虽然听起来更难，但更强的假设恰好让归纳法能顺利推进。

14.1 为什么需要"加强命题"

继续上面的例子。

例1：求证 1 + 1/2² + 1/3² + ... + 1/n² < 2。

尝试直接归纳：

n=1：1 < 2，成立。
假设 n 时成立：1 + 1/2² + ... + 1/n² < 2。

那么 n+1 时：
左边 = (1 + ... + 1/n²) + 1/(n+1)²
     < 2 + 1/(n+1)²
     > 2  ???  无法推出 < 2 !

归纳失败了——因为右侧的常数2没有给递推留出余量。

解决方案：加强命题。我们来证明一个更强的结论：

加强命题：1 + 1/2² + 1/3² + ... + 1/n² ≤ 2 - 1/n

这个命题比原命题"更强"——如果它成立，原命题自然成立（因为 2 - 1/n < 2）。而且它在右侧留出了一个"余量" 1/n，使得递推步能闭合。

用归纳法证明加强后的命题：

n=1：1 ≤ 2 - 1 = 1，成立（等号成立）。

假设 n 时成立：
1 + 1/2² + ... + 1/n² ≤ 2 - 1/n

n+1 时：
左边 ≤ (2 - 1/n) + 1/(n+1)²

需要证明：2 - 1/n + 1/(n+1)² ≤ 2 - 1/(n+1)
即证：-1/n + 1/(n+1)² ≤ -1/(n+1)
即：1/(n+1) + 1/(n+1)² ≤ 1/n

通分左边：(n+1+1)/(n+1)² = (n+2)/(n+1)²
交叉相乘：n(n+2) = n²+2n ≤ (n+1)² = n²+2n+1  ☑ 显然成立

所以加强命题归纳成立，原命题得证。

这个例子展示了加强命题的精髓：在右侧添加一个"消失项" 1/n，这个项在归纳步中被消耗掉，使递推闭合。

什么时候应该加强命题——以下信号值得警惕：

  1. 不等式归纳：右侧是常数（如 < 2），左侧逐项增加。几乎必然需要加强。
  2. 递推需要更强的结构性假设：只假设"存在一组解"往往不够，需要假设"存在一组具有某种额外性质的解"。
  3. 多个变量的约束：对n个变量成立的性质推广到n+1个时，需要假设性质"对任意n个元素"成立而不是"对某具体n个元素"。

14.2 经典案例：用加强命题证明均值不等式

均值不等式（AM-GM）是数学中最基本的不等式之一。它的经典证明——Cauchy的向前-向后归纳法（Forward-Backward Induction）——是加强命题和归纳变形的完美展示。

均值不等式：对任意正数 a₁, a₂, ..., aₙ，有 (a₁+...+aₙ)/n ≥ ⁿ√(a₁·...·aₙ)，等号当且仅当所有aᵢ相等。

这个证明不需要写"加强命题"四个字，但它的策略本质就是加强：先证明对所有2的幂成立（正向跳步），再证明可以倒退到更小的数（反向退步）。

第一步（向后归纳）：若对n成立，则对n-1也成立

设均值不等式对 n 个正数成立。对于 n-1 个正数 a₁,...,aₙ₋₁，
令 aₙ = (a₁+...+aₙ₋₁)/(n-1)（即前 n-1 个数的平均值，记为 A）。

由 n 情形的 AM-GM：
(a₁ + ... + aₙ₋₁ + A)/n ≥ (a₁·...·aₙ₋₁·A)^{1/n}

左边 = (nA)/n = A。

所以 A ≥ (a₁·...·aₙ₋₁·A)^{1/n}
两边 n 次方：Aⁿ ≥ a₁·...·aₙ₋₁·A
约去 A（正数）：Aⁿ⁻¹ ≥ a₁·...·aₙ₋₁
两边开 n-1 次方：A ≥ (a₁·...·aₙ₋₁)^{1/(n-1)}

这正是对 n-1 个数的 AM-GM！

第二步（向前归纳）：若对n成立，则对2n也成立

对于 2n 个正数 a₁,...,a₂ₙ，由 n 情形的 AM-GM：
(a₁+...+aₙ)/n ≥ (a₁...aₙ)^{1/n}
(aₙ₊₁+...+a₂ₙ)/n ≥ (aₙ₊₁...a₂ₙ)^{1/n}

令 A = (a₁+...+aₙ)/n，B = (aₙ₊₁+...+a₂ₙ)/n。
由 n=2 时的 AM-GM：(A+B)/2 ≥ √(AB)。

所以 (a₁+...+a₂ₙ)/(2n) = (A+B)/2 ≥ √(AB)
≥ √[(a₁...aₙ)^{1/n} · (aₙ₊₁...a₂ₙ)^{1/n}]
= (a₁...a₂ₙ)^{1/(2n)}

这正是对 2n 个数的 AM-GM！

第三步：组合两步

1. n=2 时直接验证成立。
2. 由"向前归纳"：n=2 → 4 → 8 → 16 → ... 即所有 2ᵏ 成立。
3. 由"向后归纳"：从任意大于目标 n 的 2ᵏ 开始，不断退回到 n。

因此 AM-GM 对任意正整数 n 成立。证毕。

向前-向后归纳法的适用场景：

  1. 从 n 到 n+1 的推导困难，但 n 到 2n 的推导简单
  2. 从 n 到 n-1 的推导也简单（通过巧妙赋值）
  3. 结论对 2 的幂次容易证明（如均值不等式对 n=2 显然成立）

这个方法特别适合对称性不等式的证明。

14.3 归纳法处理图论问题

图论是归纳法应用最频繁的领域之一。归纳变量通常选顶点数或边数，两种选择各有千秋。

14.3.1 对顶点数归纳

例2：证明任意n个顶点的树（连通无环图）恰有n-1条边。

对顶点数 n 归纳：
n=1：一个顶点，0 条边，成立。

假设对 n 个顶点的树成立（有 n-1 条边）。
考虑 n+1 个顶点的树 T。
在 T 中取一个叶子（度数为 1 的顶点——树中一定存在这样的顶点）。
去掉这个叶子及其相连的边，得到 T'。
T' 仍然是树（去掉叶子不会产生环，且仍连通），有 n 个顶点。
由归纳假设，T' 有 n-1 条边。
所以 T 有 (n-1) + 1 = n 条边。证毕。

这里的关键是"去掉一个叶子"——利用树的极端结构（一定有叶子）来化归问题。这本质上也是极端原理在图论归纳中的应用。

14.3.2 对边数归纳

例3：证明任意连通图G至少包含一个生成树（包含所有顶点的极小连通子图）。

对边数 m 归纳：
如果 G 没有环（本身就是树），那么 G 就是自己的生成树。成立。

如果 G 有环，去掉环中的任意一条边 e。
G-e 仍然连通（因为环上去掉一边不破坏连通性），边数减少 1。
由归纳假设，G-e 有生成树 T。
T 也是 G 的生成树（因为它包含 G 的所有顶点且连通）。
证毕。

这个归纳极其简单——但它推导出了一个重要的结论：任何连通图都有生成树，而且可以通过不断删环得到。在竞赛中，"生成树的存在性"是很多图论问题的基础。

14.3.3 对顶点数归纳 + 加强命题

例4：证明任何n个顶点的竞赛图（每对顶点间恰有一条有向边）包含一条哈密顿路径（经过所有顶点恰好一次的有向路径）。

对顶点数 n 归纳：
n=1,2 显然成立。

假设对 n-1 个顶点的竞赛图成立。
考虑 n 个顶点的竞赛图 T。去掉顶点 v，剩下 n-1 个顶点的竞赛图 T'，
由归纳假设 T' 有哈密顿路径 P' = v₁ → v₂ → ... → vₙ₋₁。

现在把 v 插回路径中：
- 如果存在边 v → v₁，则在开头插入：v → v₁ → ... → vₙ₋₁
- 如果存在边 vₙ₋₁ → v，则在结尾插入
- 否则 v₁ → v 且 v → vₙ₋₁，则路径中必然存在相邻顶点 vᵢ, vᵢ₊₁
  使得 vᵢ → v 且 v → vᵢ₊₁（因为箭头方向从"指向v"变成了"从v指出"，
  一定在某处发生了反转）。在 vᵢ 和 vᵢ₊₁ 之间插入 v。

三种情况都构造出了 n 个顶点的哈密顿路径。证毕。

这个证明不需要加强命题——原命题的归纳假设刚好够用。但如果题目改为"证明竞赛图有哈密顿回路"（经过所有顶点回到起点），原命题就不够用了——需要先证明"竞赛图是强连通的"这个更强的结论。

图论中归纳法的使用指南：

对顶点数归纳：
1. 去掉一个顶点（和它的边）
2. 对剩下的图应用归纳假设
3. 把去掉的顶点加回来（需要分析新顶点与原图的关系）

对边数归纳：
1. 去掉一条边（或删环、合并顶点等）
2. 对剩下的图应用归纳假设
3. 分析移除边的影响

选择原则：如果"加一个顶点"比"加一条边"容易控制，用顶点归纳；
反之用边归纳。大多数问题用顶点归纳更自然。

14.4 归纳构造的常见模式

在组合中，归纳法不仅用于证明，也用于构造——假设 n 的情形已构造好，然后将其"扩展"到 n+1 的情形。

例5：对任意 n ≥ 2，构造一个 n×n 的 0-1 矩阵，使得每一行、每一列都恰好有两个 1。

用归纳法构造：

n=2：矩阵 [[1,1],[1,1]]，每行每列各两个 1，成立。

假设已构造了 n×n 矩阵 Aₙ 满足条件。来构造 Aₙ₊₁：

取 Aₙ 放在左上角，剩下加一行一列。
在新行的第 n 列放 1、第 n+1 列放 1。
在新列的第 n 行放 1、第 n+1 行放 1。
其余位置补 0。

验证：
- 前 n-1 行：每行在 Aₙ 中有两个 1，新加的一列全 0 → 仍有两个 1
- 第 n 行：在 Aₙ 中有两个 1，新加的第 n+1 列有 1 → 变成三个 1！
  这违反条件了。需要调整。

修正：让第 n 行在 Aₙ 中只有一个 1，新加一列后再加一个 1，
总共两个 1。这需要 Aₙ 的构造也满足"第 n 行恰有一个 1"的额外条件。
——这就是"加强命题"在构造中的应用！

加强后的归纳假设：存在 n×n 的 0-1 矩阵，每行每列都恰有两个 1，
且最后一行和最后一列各恰有一个 1（位于交叉点）。

这样一来，在扩展时就能保持所有条件。
这个例子告诉我们：有时候为了构造成功，你需要对构造对象
施加比题目要求更强的"归纳假设"。

归纳构造的核心技巧：

  1. 附加法：在已有的结构上添加新元素，并适当调整——最常用
  2. 拆分法：把n拆成n₁+n₂两个更小的规模，分别构造再组合
  3. 倍增法：用两个n/2规模的对象拼成一个n规模的对象

附加法最常用，但要注意添加新元素后不能破坏旧结构已经满足的条件——这就需要"加强命题"来预留调整空间。

14.5 反向归纳法

反向归纳法（Backward Induction）是一种特殊的归纳形式——它不是从n推到n+1，而是从n推到n-1。

反向归纳法的逻辑：

  1. 证明命题对无穷多个n成立（通常通过对2ᵏ正向归纳或直接验证）
  2. 证明如果命题对n成立，则对n-1也成立
  3. 由(1)和(2)，命题对所有正整数成立

我们已经在 AM-GM 的证明中看到了这种模式。下面是另一个经典例子。

例6（Jensen不等式的反向归纳证明）：

Jensen 不等式：对于凸函数 f，有
f((x₁+...+xₙ)/n) ≤ (f(x₁)+...+f(xₙ))/n

证明（反向归纳法）：
n=2 时由凸函数的定义直接成立。

正向归纳：由 n=2 反复应用可证 n=4,8,16,... 成立。
（因为 2n 的情形可以通过分成两组各 n 个，应用 n=2 的 Jensen 得到）

反向归纳：假设对 n 成立。令 y = (x₁+...+xₙ₋₁)/(n-1)。
由 n 情形的 Jensen：
f((x₁+...+xₙ₋₁ + y)/n) ≤ (f(x₁)+...+f(xₙ₋₁)+f(y))/n

但 (x₁+...+xₙ₋₁ + y)/n = ((n-1)y + y)/n = y

所以 f(y) ≤ (f(x₁)+...+f(xₙ₋₁)+f(y))/n
两边乘以 n：n·f(y) ≤ f(x₁)+...+f(xₙ₋₁)+f(y)
移项：(n-1)f(y) ≤ f(x₁)+...+f(xₙ₋₁)
即 f(y) ≤ (f(x₁)+...+f(xₙ₋₁))/(n-1)

这正是对 n-1 个数的 Jensen 不等式！

什么时候用反向归纳？

  • 从 n 到 n+1 的推导困难，但从 n 到 n-1 的推导很自然时
  • "加一个元素"会破坏结构，但"减一个元素"很容易时
  • 已知命题对所有 2 的幂次成立，需要推广到所有正整数时

14.6 归纳法的常见误区

我教了这么多年竞赛，总结了以下归纳法最常见的错误。每个都来自真实的学生作业：

错误类型具体表现改正方法
归纳假设过弱只用"存在性"假设但需要"对所有可能"成立加强命题，把"存在"改为"任意"或"所有"
跳跃过大从k推到k+1时用到了k-1或更早的结果却没说明改为强归纳法（假设所有小于n的情形成立）
忽略基础情形递推步完美但n=1时命题不成立多验证几个基础值（特别是n=1,2）
循环论证推n+1时不恰当地使用了n+1情形的结论检查每一步是否只用了≤n的假设
归参数选错需要对顶点数归纳却对边数归纳根据问题结构选合适的归纳变量

⚠️ 踩坑实录：有一次我在课堂上出了这样一道题："证明任意 n 条直线（无两条平行、无三线共点）将平面分成 n(n+1)/2 + 1 个区域。"

一个学生这样证：假设 n 条直线分成 R(n) 个区域。添加第 n+1 条直线，它与前面 n 条直线各交于一点，被分成 n+1 段，每段将一个原有区域一分为二，所以增加 n+1 个区域。所以 R(n+1) = R(n) + n+1。结合 R(1)=2，得 R(n)=n(n+1)/2+1。

这个推导本身是对的，但他没有用数学归纳法的标准形式——他直接用递推公式求解。严格来说，如果用归纳法，应该写清楚：假设 R(n)=n(n+1)/2+1，在n+1条直线时，由"第n+1条线被n个交点分成n+1段，每段增加一个区域"可得 R(n+1)=R(n)+n+1=n(n+1)/2+1+n+1=(n+1)(n+2)/2+1。在竞赛答卷中，清晰地写出归纳假设和推导步骤是得分的关键。很多学生省略了"由归纳假设"的字样，导致扣分。

还有一个学生的错误更致命——他试图用直接归纳证明"1 + 1/2 + 1/3 + ... + 1/n 不是整数"。他的归纳假设是"1+...+1/n不是整数"，然后试图证明"加1/(n+1)后也不是整数"。但两个非整数之和可能是整数！这就是归纳假设太弱——应该加强为"1+...+1/n 的分母是某个特定数"或"写成既约分数后分子是奇数"等更强的条件。

教训：归纳法中的每一步都要"可证"。如果你发现推不动，不要硬推——停下来想想是不是假设太弱了。

14.7 本章小结与过渡

本章我们学习了数学归纳法在组合问题中的高级应用。核心要点是：当直接归纳遇到瓶颈时，加强命题是突破的关键——通过证明一个更强的结论来让归纳假设变得足够"有力"。我们还学习了向前-向后归纳法（Cauchy的经典技巧）、图论中的顶点/边归纳策略、归纳构造的常见模式（附加、拆分、倍增），以及反向归纳法的应用场景。

归纳法与加强命题是连接"小规模成立"和"大规模成立"的桥梁。在组合竞赛中，它是最基础也是最灵活的证明方法之一。有时候你可以直接归纳，有时候需要先绕个弯（加强命题或双向往复）——这些技巧都需要通过大量练习来内化。

至此，我们完成了"核心方法"部分的全部内容。从第八章的抽屉原理到本章的归纳法，我们已经建立了一个强大的组合方法论工具箱。下一章我们将进入第三部分——进阶技巧，首先学习的是概率方法，一种用概率论的语言来解决组合存在性问题的精妙方法。
