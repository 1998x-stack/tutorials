第23章 循环神经网络

大家好！欢迎来到深度学习三部曲的第一章。在前面的课程中，我们学习了ARIMA、GARCH、Prophet等经典方法，它们各有千秋，但有一个共同的局限：需要人工设计特征和模型结构。从本章开始的三章，我们将进入"让数据自己说话"的世界——深度学习。而这一切的起点，就是循环神经网络（Recurrent Neural Network, RNN）。

为什么RNN对时间序列如此重要？因为时间序列本质上是序列数据——每个观测值之间有时间顺序，而RNN恰恰是为这种结构设计的。在RNN出现之前，处理序列数据主要靠固定长度的滑动窗口和手工特征，模型无法充分利用序列中的长程依赖关系。RNN通过维护一个"记忆"状态，让网络能自动学习时间模式。从2015年到2020年，几乎所有时间序列预测竞赛的冠军都用到了RNN或其变体——它的影响力可见一斑。

23.1 RNN的核心思想

RNN的核心理念很简单：让网络"记住"过去的信息。在每一时间步 t，RNN不仅接收当前输入 xt，还接收上一个时刻的隐藏状态 ht-1，两者共同决定当前输出和新的隐藏状态：

h_t = tanh(W_ih · x_t + b_ih + W_hh · h_{t-1} + b_hh)

如果我们将RNN在时间上"展开"（unroll），它会看起来像一个深层前馈网络，但每一层共享相同的权重矩阵 Wih 和 Whh。这种权重共享机制使得RNN能够处理可变长度的序列——不管序列多长，使用的都是同一套参数，只是循环次数不同。

这个设计非常优雅，但在实践中，训练RNN遇到了一个巨大的障碍，而这个障碍是在一个硕士生的毕业论文中被首次揭示的。

关键概念：RNN的"记忆"通过隐藏状态 ht 传递。理想情况下，ht 应该包含截至 t 时刻的所有历史信息。但在实践中，梯度消失问题让RNN的"记忆"非常短暂，一般不超过5-10步。

23.2 梯度消失问题——一个硕士生的发现

1991年，一位名叫 Sepp Hochreiter 的德国硕士生在慕尼黑工业大学撰写他的毕业论文。他在研究RNN的训练过程时，发现了一个令人沮丧的现象：当序列长度超过5-10步时，RNN几乎无法学到任何长期依赖关系。他深入分析了反向传播的数学过程，发现误差信号在时间轴上每反向传播一步，都会乘以权重矩阵的雅可比矩阵。如果这个矩阵的谱范数小于1，梯度就会指数级衰减到零——这就是著名的梯度消失问题（Vanishing Gradient Problem）。

Hochreiter 的导师是 Jürgen Schmidhuber，后者立刻意识到这个发现的重大意义。但当时整个学术界对深度学习还不太关注，这篇毕业论文的发现直到1998年才被正式发表。Hochreiter 后来回忆说，他反复用实验验证了这一现象，发现RNN确实"记不住"超过5步的信息。他甚至一度怀疑：序列建模这条路是否真的走得通？

有趣的是，梯度消失和梯度爆炸就像序列学习的一体两面，同时存在于RNN的训练过程中：

问题 现象 原因 后果
梯度消失 梯度趋近于零 雅可比矩阵谱范数 < 1 无法学习长期依赖关系
梯度爆炸 梯度指数级增长 雅可比矩阵谱范数 > 1 参数更新不稳定、训练发散

梯度爆炸可以通过梯度裁剪来缓解（后面会详细介绍），但梯度消失才是RNN最致命的先天弱点。Hochreiter 的这个发现直接催生了后来改变深度学习格局的解决方案——长短期记忆网络。

23.3 LSTM——引入门控机制

长短期记忆网络（Long Short-Term Memory, LSTM）由 Hochreiter 和 Schmidhuber 在1997年提出。它的核心创新是引入了细胞状态（Cell State）和三个门控机制。

细胞状态 ct 就像一条传送带，贯穿整个序列处理过程。信息在这条传送带上流动，门控机制有选择地增删信息。这种设计的精妙之处在于：梯度可以沿着细胞状态"无损"地反向传播——因为细胞状态的更新主要是加法运算（加法和按元素乘），而不是非线性函数的复合。这从根本上缓解了困扰RNN多年的梯度消失问题。

LSTM的三个门控各自承担不同角色：

  • 遗忘门（Forget Gate）：决定从细胞状态中丢弃哪些旧信息。sigmoid输出0到1之间的值，0表示"完全丢弃"，1表示"完全保留"。
  • 输入门（Input Gate）：决定将哪些新信息存入细胞状态。输入门和候选细胞状态共同决定更新内容。
  • 输出门（Output Gate）：基于当前的细胞状态，决定输出哪些信息到隐藏状态。
# LSTM核心计算（简化伪代码）
f_t = sigmoid(W_f · [h_{t-1}, x_t] + b_f)   # 遗忘门
i_t = sigmoid(W_i · [h_{t-1}, x_t] + b_i)   # 输入门
c̃_t = tanh(W_c · [h_{t-1}, x_t] + b_c)      # 候选细胞状态
c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t             # 更新细胞状态
o_t = sigmoid(W_o · [h_{t-1}, x_t] + b_o)   # 输出门
h_t = o_t ⊙ tanh(c_t)                       # 更新隐藏状态
记忆技巧：把LSTM想象成一个智能档案管理员。遗忘门决定扔掉哪些旧档案，输入门决定存入哪些新档案，输出门决定展示哪些档案给外界。细胞状态就是档案柜，信息在其中长期保存不受干扰。

23.4 GRU——更简洁的门控

门控循环单元（Gated Recurrent Unit, GRU）由 Cho 等人在2014年提出，是LSTM的简化版本。GRU将三个门合并为两个：更新门（Update Gate）和重置门（Reset Gate）。

相比LSTM，GRU没有独立的细胞状态，而是直接将隐藏状态作为信息的载体。更新门控制"保留多少旧信息"，重置门控制"忽略多少旧信息"。GRU参数更少（少了输出门），训练更快，在数据量不大时往往表现与LSTM相当甚至更好。选择建议：数据量大、序列很长时优先LSTM；数据量中等、追求训练速度时优先GRU。

23.5 双向与堆叠LSTM

在实际应用中，我们很少使用单层单向的LSTM，通常会进行以下扩展：

双向LSTM（Bidirectional LSTM）：同时从正向和反向读取序列，每个时间步的隐藏状态由两个方向的状态拼接而成。在异常检测、分类等任务中效果显著，因为当前时刻的判断可以受益于"未来的上下文"。但注意：在预测未来值时，双向LSTM会引入未来信息，因此更适合分类/填补任务而非纯预测。

堆叠LSTM（Stacked LSTM）：将多个LSTM层垂直堆叠，下层的隐藏状态作为上层的输入。底层捕捉短期、局部模式，高层捕捉长期、全局模式。通常2-3层效果最佳，层数过多反而容易过拟合且训练成本飙升。

23.6 Seq2Seq架构

序列到序列（Sequence-to-Sequence, Seq2Seq）架构是RNN最重要的应用范式之一。它由编码器（Encoder）和解码器（Decoder）两部分组成：编码器读取输入序列并将其压缩为一个固定维度的上下文向量，解码器再从这个上下文向量出发，逐步生成输出序列。

Seq2Seq最早用于机器翻译（如英语→法语的序列转换），后被广泛引入时间序列的多步预测。在预测场景中，编码器读取历史窗口，解码器逐步生成未来值。这种"自回归生成"的范式极大提升了多步预测的精度，也为后续的Transformer奠定了基础。

23.7 范式转变：从滑动窗口到自回归生成

深度学习的引入不仅是模型变了，整个处理序列的思维方式也发生了根本转变：

  • 滑动窗口法（传统统计方法）：用固定长度的历史预测下一步，窗口外的信息完全丢弃。简单高效，但信息利用率低。
  • 序列标注（深度学习引入）：每个时间步都有输出，可以逐点预测。适合异常检测、逐点分类等任务。
  • 自回归生成（Seq2Seq / Transformer）：模型逐步"生成"未来序列，每一步的预测值作为下一步的输入。适合中长期多步预测。

这三种范式并非替代关系，而是适用不同场景。理解它们的区别有助于我们针对具体问题选择合适的方法。

23.8 训练技巧

训练RNN/LSTM有几个关键技巧，新手往往容易忽略：

Teacher Forcing（教师强制）：在Seq2Seq训练时，解码器每一步的输入使用真实值（而不是上一步的预测值）作为输入。这极大加速了训练收敛。但要注意：推理时没有真实值可用，所以训练时应当适当加入"自由运行"（free running）的样本，否则会出现训练-推理不匹配的问题（exposure bias）。

梯度裁剪（Gradient Clipping）：将梯度的L2范数限制在一个阈值内（通常设为1.0或5.0），防止梯度爆炸。这个技巧简单但极其有效，几乎是训练RNN的标配。

截断BPTT（Truncated Backpropagation Through Time）：将长序列截断为固定长度（如64或128步），在每个子序列上独立进行反向传播。这既控制了显存消耗，又保留了足够的上下文信息。

23.9 PyTorch实现：LSTM预测管道

下面是一个完整的LSTM时间序列预测示例，包含数据准备、模型定义、训练和预测：

import torch
import torch.nn as nn
import numpy as np

# 1. 生成示例数据
def create_sequences(data, seq_len=24, pred_len=1):
    xs, ys = [], []
    for i in range(len(data) - seq_len - pred_len + 1):
        xs.append(data[i:i+seq_len])
        ys.append(data[i+seq_len:i+seq_len+pred_len])
    return np.array(xs), np.array(ys)

# 2. 定义LSTM模型
class LSTMForecast(nn.Module):
    def __init__(self, input_size=1, hidden_size=64,
                 num_layers=2):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size, hidden_size, num_layers,
            batch_first=True, dropout=0.2
        )
        self.fc = nn.Linear(hidden_size, 1)

    def forward(self, x):
        # x: (batch, seq_len, input_size)
        out, (h_n, c_n) = self.lstm(x)
        return self.fc(out[:, -1, :])  # 取最后一步输出

# 3. 训练循环
model = LSTMForecast()
criterion = nn.MSELoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)

for epoch in range(200):
    for batch_x, batch_y in dataloader:
        pred = model(batch_x)
        loss = criterion(pred, batch_y)
        optimizer.zero_grad()
        loss.backward()
        # 梯度裁剪——防止梯度爆炸
        torch.nn.utils.clip_grad_norm_(
            model.parameters(), max_norm=1.0
        )
        optimizer.step()

# 4. 预测
model.eval()
with torch.no_grad():
    forecast = model(test_x)
    print(forecast.shape)  # (n_samples, 1)
常见踩坑：使用LSTM时要注意数据缩放！RNN对输入数据的尺度非常敏感，一定要先做标准化或归一化。此外，batch_first=True 这个参数经常被忽略——默认是 (seq_len, batch, features)，设成 True 后才是 (batch, seq_len, features)。新手经常在这里搞混维度，导致形状不匹配的错误。

23.10 小结

本章我们学习了循环神经网络及其变体。从Hochreiter意外发现的梯度消失问题开始，到LSTM和GRU的门控机制，再到Seq2Seq架构和关键训练技巧，可以看到深度学习为时间序列建模带来了全新的视角和强大的工具。

但RNN并非完美——它的串行计算结构限制了训练速度（当前时间步必须等前一步计算完），对超长序列仍显吃力。下一章，我们将从一个看似无关的角度切入：用卷积神经网络来处理时间序列。你可能会觉得奇怪——卷积不是做图像识别的吗？它怎么处理时序数据呢？下一章我们揭晓答案。
