第6章：递推与归纳法

大家好，我是你们的讲师。到目前为止，我们学的所有计数方法都遵循一个模式：直接写出答案的表达式——C(n,k)、2^n、n!、D_n 等等。但有些问题太复杂了，没有办法写出简洁的封闭公式。或者说，有公式但推导起来很困难。这时候我们需要另一种思路：不说"答案是多少"，而说"答案和更小的问题有什么关系"。

这就是递推法的核心思想。递推（recurrence）和归纳（induction）是组合数学中最重要的一对搭档——递推负责"建立关系"，归纳负责"证明结论"。今天我们就把这对搭档的用法彻底讲透。

6.1 递推关系的建立方法

递推法分为三个步骤：

  1. 找规律：观察问题，看看小规模情况的结果能否帮助推算大规模情况
  2. 建立递推：找到 a_n 和 a_{n-1}, a_{n-2}, ... 之间的关系
  3. 解递推：求解递推关系，得到 a_n 的通项公式（或直接计算所需项）

这三个步骤中，第二步是最核心也是最困难的——它需要你找到"从 n-1 到 n"的过渡方式。下面我们通过几个经典问题来训练这个能力。

6.1.1 一个简单的例子：上楼梯问题

先说一个经典问题："一个人爬 n 级台阶，每次可以跨 1 级或 2 级。有多少种不同的爬法？"

设 a_n 为爬 n 级台阶的方法数。

要到达第 n 级，最后一步要么是从第 n-1 级跨 1 级上来，要么是从第 n-2 级跨 2 级上来。所以：

a_n = a_{n-1} + a_{n-2}

初始值：a₁ = 1（一级台阶只有一种走法：跨1级），a₂ = 2（两种走法：1+1 或 2）。

这就是 Fibonacci 数列（但起始值略有不同）：a_n = F_{n+1}（Fibonacci 数）。

n12345678910
a_n123581321345589

这个递推的巧妙之处在于：我们把"到达第 n 级"这个事件，按照最后一步的动作进行了分类。分类的基本原则是"互斥且完备"——这两类既不重叠（最后一步不能同时是跨1级和跨2级），又能覆盖所有情况。这个分类-递推的思路是组合递推的基石。

6.2 Fibonacci 数列的组合应用

Fibonacci 数列在组合数学中无处不在，几乎每隔几道竞赛题就会碰到一次。下面是几个最常见的组合模型：

6.2.1 骨牌覆盖问题

"用 1×2 的骨牌覆盖一个 2×n 的棋盘，有多少种不同的覆盖方法？"

答案是 Fibonacci 数列。设 f(n) 为覆盖方法数。考虑最左边的一列：

  • 如果竖着放一块骨牌，剩下的 2×(n-1) 有 f(n-1) 种覆盖方法
  • 如果横着放两块骨牌（占据左边两列），剩下的 2×(n-2) 有 f(n-2) 种覆盖方法

所以 f(n) = f(n-1) + f(n-2)，且 f(1)=1, f(2)=2。这就是我们在上楼梯问题中看到的同一个数列。

6.2.2 不含连续 1 的二进制串

"有多少个长度为 n 的二进制串，使得没有两个 1 是相邻的？"

设 g(n) 为总数。考虑第一位：

  • 如果第一位是 0，则剩下的 n-1 位可以任意（只要自身不违反规则），有 g(n-1) 种
  • 如果第一位是 1，则第二位必须是 0，剩下的 n-2 位有 g(n-2) 种

所以 g(n) = g(n-1) + g(n-2)，g(1)=2, g(2)=3。这仍然是 Fibonacci 数列（只是偏移了一下）。

💡 模式识别：什么时候该想到 Fibonacci？当一个问题可以分解成"假设第一步选了某种方案，剩下的问题规模减小"的时候，如果选择方案有恰好两种（如跨1或跨2、竖放或横放、第一位是0或1），且两种方案分别将问题规模减少 1 和 2，那么递推关系就是 Fibonacci 型：a_n = a_{n-1} + a_{n-2}。

6.3 特征方程解线性递推

找到了递推关系之后，我们通常希望得到一个通项公式——这样我们就能直接计算第 n 项的值，而不需要从第一项开始一项项地递推。对于线性齐次递推（形如 a_n = c₁a_{n-1} + c₂a_{n-2} + ... + c_k a_{n-k}），我们有标准解法——特征方程法。

6.3.1 一阶线性递推

a_n = p·a_{n-1} + q

解法：
1. 找到不动点：a = pa + q → a = q/(1-p)（如果 p≠1）
2. 令 b_n = a_n - a，则 b_n = p·b_{n-1}
3. 所以 b_n = p^{n-1}·b₁
4. 代回得 a_n = a + p^{n-1}(a₁ - a)

6.3.2 二阶线性齐次递推

a_n = p·a_{n-1} + q·a_{n-2}

特征方程：x² - px - q = 0
设两根为 r₁, r₂。则通解为：
- 若 r₁ ≠ r₂：a_n = A·r₁ⁿ + B·r₂ⁿ
- 若 r₁ = r₂ = r：a_n = (A + Bn)·rⁿ

其中 A, B 由初始条件 a₁, a₂ 确定。

以 Fibonacci 数列 F_n = F_{n-1} + F_{n-2} 为例：

特征方程：x² - x - 1 = 0
两根：r₁ = (1+√5)/2 ≈ 1.618（黄金分割比φ）
     r₂ = (1-√5)/2 ≈ -0.618（-1/φ）

设 F_n = A·φⁿ + B·(-1/φ)ⁿ
由 F₁=1, F₂=1 解得：
A = 1/√5, B = -1/√5

所以 F_n = (φⁿ - (-1/φ)ⁿ) / √5

这就是著名的比内公式（Binet's formula）。

🔥 关键洞察：特征方程法的本质是——把递推问题转化为代数方程求解。递推关系决定了数列的"局部规则"，特征方程则揭示了数列的"全局形态"。这就像微分方程和特征根的关系一样。掌握了特征方程法，你就能解几乎所有常见的线性递推。

6.4 归纳法在组合证明中的强化技巧

建立了递推关系之后，我们经常需要用数学归纳法来证明某些结论。但竞赛中常常遇到这样的情况：直接归纳不够用，必须加强命题。

6.4.1 为什么需要"加强命题"？

假设我们要证明某个命题 P(n) 对一切 n 成立。归纳法的标准步骤是：证明 P(1) 成立，然后假设 P(n-1) 成立，推导 P(n) 成立。但有时候，从 P(n-1) 推导 P(n) 的过程中，我们需要一个比 P(n-1) 更强的条件——这时候就要修改命题本身，把它变得更强。

举一个典型的例子：

"证明：1 + 1/2² + 1/3² + ... + 1/n² < 2 - 1/n"

有人可能会想，先用归纳法证明它小于 2。但你能从"小于 2"推出"小于 2"吗？不能——归纳步骤中需要的是一个更紧的上界才能继续缩。所以我们要把命题加强为：

1 + 1/2² + ... + 1/n² < 2 - 1/n

验证 n=1：左边=1，右边=2-1=1，不成立（1 < 1 不真）！

再加强：1 + 1/2² + ... + 1/n² ≤ 2 - 1/n
n=1：1 ≤ 1，成立。
归纳步骤：假设 n=k 成立，则 n=k+1 时：
左边 ≤ 2 - 1/k + 1/(k+1)²
我们要证明 ≤ 2 - 1/(k+1)
即证 2 - 1/k + 1/(k+1)² ≤ 2 - 1/(k+1)
化简得 1/(k+1)² ≤ 1/k - 1/(k+1) = 1/(k(k+1))
即 1/(k+1) ≤ 1/k，显然成立。
所以加强后的命题得证。

这里的关键是：我们加强了一点点（从"严格小于"改成"小于等于"），就让归纳步骤能走通了。这个技巧在竞赛中极其重要。

6.4.2 归纳法在组合问题中的典型应用

组合竞赛题中，归纳法和递推几乎总是同时出现：

例： 证明任何 2ⁿ × 2ⁿ 的棋盘去掉一个格子后，可以用 L 形三格骨牌（"L形"覆盖三个相连的格子）完全覆盖。

这个问题的归纳证明非常漂亮：

  • 基础情况：n=1，2×2 棋盘去掉一个格子后刚好是一个 L 形，能被覆盖。
  • 归纳步骤：将 2ⁿ⁺¹ × 2ⁿ⁺¹ 棋盘分成 4 个 2ⁿ × 2ⁿ 的子棋盘。缺口在其中一个子棋盘中。在另外三个子棋盘"相邻的角"上放一个 L 形骨牌——这样每个子棋盘都恰好缺了一个格子（三个是人为缺的，一个是原来就缺的）。由归纳假设，每个子棋盘都可被覆盖。总覆盖方案成立。

这里的关键技巧是——用一块骨牌在中心"创造"出三个缺口，使每个子棋盘都变成"缺一格"的标准形式。这个"创造缺口"的思路在竞赛中经常用到。

6.5 经典递推：Catalan数、Bell数

6.5.1 Catalan数

Catalan 数 C_n 是组合数学中出现频率最高的数列之一。它的递推公式是：

C₀ = 1，C_{n+1} = ∑_{i=0}^{n} C_i · C_{n-i}

Catalan 数的前几项：1, 1, 2, 5, 14, 42, 132, 429, 1430, 4862, ...

Catalan 数有封闭公式：

C_n = C(2n, n) / (n+1) = C(2n, n) - C(2n, n+1)

Catalan 数对应的组合问题至少有几十种，以下是竞赛中最常见的几种：

编号问题答案
1n 对括号的合法匹配数C_n
2n+2 边形的三角剖分数C_n
3n 个节点的不同二叉树的个数C_n
4从 (0,0) 到 (n,n) 不越过对角线的路径数C_n
5n 个数的不同出栈序列数C_n
6圆周上 2n 个点用 n 条不相交弦连接的方法数C_n

所有这些问题的递推关系都源自同一个结构——某种"首尾分离"的操作。以括号匹配为例：设一对括号 (A) 中 A 是合法括号序列。A 可以分成两部分：外面是一对括号，里面是另一个合法序列。这就是 Catalan 递推的来源。

6.5.2 Bell数

Bell 数 B_n 表示 n 个不同元素的集合的所有划分（partition）的个数。它的递推是：

B_{n+1} = ∑_{k=0}^{n} C(n,k) · B_k

证明思路：考虑元素 n+1 所在的等价类。设这个类的大小为 k+1（即除了 n+1 之外还有 k 个元素），那么：

  • 从 n 个元素中选 k 个和 n+1 分在一起：C(n,k) 种选法
  • 剩下的 n-k 个元素构成剩下的划分：B_{n-k} 种

对 k 从 0 到 n 求和即得递推公式。

Bell 数的前几项：1, 1, 2, 5, 15, 52, 203, 877, 4140, ...

💡 递推的三种风格：比较一下我们遇到过的三个经典递推：

  • Fibonacci：a_n = a_{n-1} + a_{n-2}（二阶齐次）——相邻项的关系
  • Catalan：C_{n+1} = ∑C_i·C_{n-i}（卷积型）——"分治"式的自相似结构
  • Bell：B_{n+1} = ∑C(n,k)B_k（混合型）——包含组合系数的递推

掌握了这三种递推风格，你就掌握了竞赛中 80% 以上的递推问题。

6.6 踩坑故事——递推边界的重要性

说一个我亲身经历的"血案"。有一年我在模拟赛中出了一道递推题：

"用 1×2 和 1×3 的骨牌覆盖 1×n 的棋盘，有多少种不同的覆盖方法？"

一个学生很快给出了递推关系：a_n = a_{n-1} + a_{n-2} + a_{n-3}（因为最后可以放 1×1、1×2 或 1×3 的骨牌——等等，题目中只有 1×2 和 1×3，没有 1×1 啊）。

我让学生解释。他说："最后放 1×2 的骨牌，就是 a_{n-2}；最后放 1×3 的骨牌，就是 a_{n-3}。但是，如果剩下 1 格，就放不了了——所以他的递推应该是 a_n = a_{n-2} + a_{n-3}，而不是 a_n = a_{n-1} + a_{n-2} + a_{n-3}。"

他说完就修正了答案。但我接着问："那么 a₁ 是多少？a₂ 呢？a₃ 呢？"

  • a₁ = 0（1×1 无法用 1×2 或 1×3 覆盖）
  • a₂ = 1（一块 1×2）
  • a₃ = 1（一块 1×3）
  • a₄ = a₂ + a₁ = 1 + 0 = 1
  • a₅ = a₃ + a₂ = 1 + 1 = 2
  • a₆ = a₄ + a₃ = 1 + 1 = 2

"现在来验证一下——a₆ 真的有 2 种覆盖方法吗？"

学生动手枚举了一下，发现 a₆ 其实有 3 种方法：

  1. 三块 1×2
  2. 两块 1×3
  3. 一块 1×3 + 一块 1×2 + 一块 1×3（不对——这个排列里 1×3 和 1×2 交错，无法用"最后一步"法直接得到）

⚠️ 问题在哪里？ 在 a₆ 的递推推导中，我们假设"最后一步"只有两种可能——最后放 1×2 或最后放 1×3。但实际覆盖中，"最后"指的是覆盖的末端。如果先放一个 1×3，再放一个 1×2，再放一个 1×3，末端是一个 1×3——从递推角度看，它属于 a_{n-3} 的情况。但是，在 a_{n-3} 中，我们假设的是前面 n-3 格已经覆盖好，最后放一块 1×3。对于"1×3 + 1×2 + 1×3"的覆盖，去掉最后一块 1×3 后前面是"1×3 + 1×2"，长度为 3。a₃ 有多少种覆盖？a₃ = 1（一块 1×3）。所以 a₆ 应该包含 a₃ 中的那一种。但是，"1×3 + 1×2"这个覆盖真的在 a₃ 中吗？

问题暴露了！"1×3 + 1×2"的长度是 5，不是 3。去掉最后一块 1×3 后，剩下"1×3 + 1×2"长度是 5。而我之前计算 a₅ = 2，确实包含了"1×3 + 1×2"这种覆盖。所以 a₆ = a₄ + a₃ 时，a₃ 贡献的是"在覆盖好 3 格后加一块 1×3"——也就是"1×3 + 1×3"。a₄ 贡献的是"在覆盖好 4 格后加一块 1×2"——但 a₄ = 1，所以"1×2 + 1×2 + 1×2"这是一种。"1×3 + 1×2 + 1×3"不在 a₄ 中（因为去掉最后的 1×2，前面是 1×3 + 1×3 = 6...

我已经把自己绕晕了。这个故事的真正教训是：

递推的边界条件需要大量验证。 在这个问题中，正确的递推确实是 a_n = a_{n-2} + a_{n-3}，但初始条件需要给到 a₁, a₂, a₃, a₄, a₅，并且通过手动枚举来验证前几项。我当时在考场上因为"自觉递推正确"就没有验证前几项，结果丢了大分。从此以后，不管递推关系看起来多么显然，我都一定会验证至少前 3 项。

6.7 本章小结与过渡

今天我们学习了递推关系的建立方法、Fibonacci 数列的组合应用、特征方程解线性递推、归纳法中的加强命题技巧，以及 Catalan 数和 Bell 数这两个经典递推数列。最重要的是，我们再次强调了递推边界条件的验证——这是竞赛中最容易忽视也最容易丢分的地方。

递推法和归纳法的核心思想是"化大为小"、"以小见大"——这和映射法的思想有异曲同工之妙。下一章我们要学习映射与对应，这是一种完全不同的计数思路：不直接数你要数的东西，而是建立一个双射，把它变成另一个更容易计数的东西。你会发现，递推和映射常常在同一道题中配合使用，互为补充。我们第七章见。
