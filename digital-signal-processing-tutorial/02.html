第2章 离散时间信号

想象这样一个场景：你的手机正在用麦克风录制一段语音，然后把它压缩成MP3发送出去。在这个过程中，麦克风输出的连续电压波形被ADC每隔一小段时间"快照"一次，变成一串数字——比如 [0.12, 0.35, 0.58, 0.71, ...]。这串数字，就是我们这一章的主角：离散时间信号。

说白了，离散时间信号就是一个按时间顺序排列的数字序列。但别被"离散时间"这个听起来很高端的名字吓到——它本质上就是你中学数学里的数列。只不过在DSP的世界里，我们用它来表示真实世界的信号。这一章，我们就从最基础的东西开始：怎么表示它，有哪些基本类型，怎么对它做运算，以及怎么衡量它的"大小"。

2.1 数学表示：x[n]的秘密

在DSP里，我们约定用 x[n] 来表示离散时间信号。这里有几个细节需要注意：

  • 中括号 [·] vs 小括号 (·)：在信号处理领域，x[n] 表示离散时间信号（n是整数），而 x(t) 表示连续时间信号（t是实数）。这个符号上的区别是领域惯例，就像程序员看到 list[i] 知道是在取数组的第i个元素一样。
  • n是整数索引：n的取值范围通常是所有整数（正负无穷），但在实际系统中我们只处理有限长的信号片段，比如n=0到N-1。
  • 采样周期Tₛ的暗含：虽然我们写的是 x[n]，但实际上它对应的是连续信号在t = n·Tₛ时刻的采样值。采样周期Tₛ = 1/fₛ，fₛ是采样率。比如fₛ = 44.1kHz（CD音质），Tₛ ≈ 22.7μs，那么x[0]对应0时刻，x[1]对应22.7μs时刻，以此类推。
⌨️ 命名的艺术：为什么不用 x(n) 而用 x[n]？因为连续信号的圆括号暗示了自变量可以取任意实数值，而方括号强调了自变量只能是整数——这是一种视觉化的类型约束。在实际的C代码里你会写成 double x[N]; 然后通过 x[i] 索引。没错，数学符号和代码写法高度一致。

离散时间信号有三种常见的表示方式：

表示方式 示例 适用场景
列表/图形表示 x[0] = 1, x[1] = 2, x[2] = 3, … 演示和手算
解析公式表示 x[n] = aⁿ·u[n] 理论分析和推导
向量/数组表示 x = [1, 2, 3, 4, 5] 编程实现

2.2 基本序列：DSP的"原子"构件

就像自然界的物质由原子组成一样，复杂的数字信号也可以由几种基本序列"拼装"而成。熟练掌握这几种基本序列，后续学卷积、Z变换、DFT都会轻松很多。

单位冲激序列 δ[n]——DSP里最重要、最"纯粹"的信号：

         δ[n] = { 1,  n = 0
                 { 0,  n ≠ 0

        图形表示：

        δ[n]     ● (1)
        │
        ─┼───●───●───●───●───▶ n
         -2 -1  0  1  2  3

单位冲激的妙用在于：任意序列都可以表示为不同位置、不同幅度的冲激之和。这就是后面卷积公式的直觉基础。

单位阶跃序列 u[n]——"开关"信号：

         u[n] = { 1,  n ≥ 0
                 { 0,  n < 0

        它和δ[n]的关系：δ[n] = u[n] - u[n-1]   (一阶差分)
                          u[n] = Σ_{k=-∞}^{n} δ[k]  (累加)

指数序列——最有表现力的序列：

         x[n] = aⁿ·u[n],  a为实数或复数

        当 |a| < 1 时序列衰减 → 收敛
        当 |a| > 1 时序列增长 → 发散
        当 a = 1 时就是u[n]本身

指数序列为什么重要？因为它是线性时不变系统（下一章的主角）的特征函数——系统对指数序列的响应就是指数序列乘以一个复数因子。这个概念贯穿整个DSP。

正弦序列——最符合直觉的周期信号：

         x[n] = A·cos(ω₀n + φ)

        关键：ω₀是数字角频率，单位是弧度/采样点
        模拟角频率Ω₀和数字角频率ω₀的关系：ω₀ = Ω₀·Tₛ = 2πf₀/fₛ

注意一个重要的细节：离散正弦序列的周期性条件比连续情况严格得多。连续正弦 cos(Ω₀t) 永远是周期的（周期T=2π/Ω₀），但离散正弦 cos(ω₀n) 只有当 ω₀/2π 是有理数时才是周期的。如果 ω₀=1（弧度/采样点），ω₀/2π=1/(2π) 是无理数，这个序列永远不会重复——这相当于你在模拟频率上永远采不到一个完整的周期，是一个很反直觉但又非常重要的现象。

💡 直观理解：采样就是"拍照"，快门速度（采样率）决定了你能否捕捉到信号的完整周期。如果你拍得太慢，相片里的运动会显得不自然——这就是混叠的起点。我们第4章会深入讲。

2.3 序列的基本运算

有了基本序列，接下来需要知道怎么"操作"它们。这六种运算是后续算法的基础：

运算 公式 直观理解
移位 y[n] = x[n - k] k>0右移（延迟），k<0左移（超前）
翻转 y[n] = x[-n] 以n=0为中心反转，类似"回放磁带"
尺度变换 y[n] = x[kn]（抽取）或 y[n] = x[n/k]（插值） 改变时基——快放或慢放
相加 y[n] = x₁[n] + x₂[n] 逐点相加，混合信号
相乘 y[n] = x₁[n] · x₂[n] 逐点相乘，加窗调制
累加 y[n] = Σ_{k=-∞}^{n} x[k] 离散版本积分

这里特别提一下"尺度变换"（也叫抽取和插值）。我见过不少初学者混淆了"尺度变换"和"改变幅值"——x[2n] 是每隔一个采样点取一个（抽取），而 2·x[n] 是幅值加倍。前者改变的是时间轴密度，后者改变的是幅值。这两个概念在多速率信号处理（第12-13章）中会极其重要。

2.4 信号的能量与功率

现在我们知道序列是什么了，能做哪些运算了。下一个问题是：怎么衡量一个信号的"大小"？

在DSP中，我们通过能量和功率来量化信号：

        信号能量：  E = Σ_{n=-∞}^{∞} |x[n]|²

        平均功率：  P = lim_{N→∞} 1/(2N+1) · Σ_{n=-N}^{N} |x[n]|²

根据这两个定义，信号被分为三类：

  • 能量信号：有限能量（E < ∞）。例如有限长序列、衰减指数序列。这类信号在实际系统中最常见——因为任何实际采集到的信号都是有限长的。
  • 功率信号：无限能量但有限平均功率。例如周期序列（正弦波）、阶跃序列。试想一个永不停止的正弦波——它的能量是无穷大（因为一直在振荡），但平均功率是有限的。
  • 既不是能量也不是功率信号：比如增长指数序列 x[n] = 2ⁿ·u[n]，能量和功率都发散的信号在实际系统中不存在（它会撑爆ADC）。
⚠️ 踩坑实录：很多年前我在做语音端点检测时，写了一个能量检测器来区分"说话"和"静音"。结果在安静的办公室里表现很好，但一到嘈杂的咖啡厅就频繁误触发。排查了三天才发现是直流偏置的问题——ADC在无信号输入时输出不是0，而是一个小的偏置电压（比如+2.5mV）。这个偏置导致静音的"能量"远大于理论值。解决方案是在计算能量之前加一个直流去除（DC removal）滤波器，其实就是做 y[n] = x[n] - x[n-1] 这样一个简单的高通处理。从那以后，我在所有涉及能量计算的工程中都会先问自己一句："我的信号被'抬升'了吗？"

在工程实现中，信号能量和功率的计算通常是基于有限长帧（frame）而不是无限累加。比如在语音处理中，我们取20ms为一帧（在16kHz采样率下就是320个采样点），计算这一帧的能量：

        // 实际工程中的帧能量计算 (Python风格)
        import numpy as np

        def frame_energy(x, frame_size=320):
            # x: 输入信号, frame_size: 帧长
            energy = np.sum(x**2) / frame_size  # 归一化
            return energy

这种基于帧的能量计算是语音活动检测（VAD）、自动增益控制（AGC）、降噪算法的底层基础。没有能量和功率的概念，这些上层算法就如同空中楼阁。

2.5 本章小结

这一章我们完成了从"连续世界"到"数字世界"的第一步跨越：

  • 学会了用 x[n] 表示离散时间信号，理解了n、Tₛ和采样率的含义；
  • 认识了四种基本序列：δ[n]（冲激）、u[n]（阶跃）、指数序列和正弦序列；
  • 掌握了序列的六种基本运算：移位、翻转、尺度变换、相加、相乘、累加；
  • 学会了用能量和功率来量化信号，并了解了它们在工程帧处理中的实际应用。
