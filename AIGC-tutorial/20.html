第20章 Flow Matching整流流

大家好，我是你们的讲师。在上一章中，我们看到了 FLUX 和 SD3 使用 Rectified Flow 替代了传统的扩散去噪过程。但 Rectified Flow 并非凭空出现的——它是 Flow Matching（流匹配） 这个大框架中的一个特例。而 Flow Matching 本身，又是 连续归一化流（Continuous Normalizing Flow, CNF） 的一种高效训练方法。

这一章我们将站在理论的高度，把第 8 章的 DDPM 扩散理论、第 10 章的 Stable Diffusion 和第 19 章的 FLUX/SD3 串起来。你会发现，这些看似截然不同的模型，本质上都可以用 Flow Matching 的统一语言来描述。理解了这个框架，你就掌握了整个扩散生成模型家族的核心密码。

20.1 从扩散到 Flow

2015 年，Sohl-Dickstein 等人首次将非平衡热力学中的扩散过程引入生成模型。2020 年的 DDPM 则是这一思想的集大成者：前向过程逐渐给数据加噪直到变成纯高斯噪声，逆向过程通过学习去噪来生成样本。

但如果我们把扩散过程放在 连续时间 的视角下看，它的本质是一个 随机微分方程（SDE）：

# 扩散过程的连续时间形式（SDE）
# 前向过程：
# dx = f(x, t)dt + g(t)dw
#
# 逆向过程（通过 Fokker-Planck 方程可推出 ODE 形式）：
# dx = [f(x, t) - 1/2 * g(t)^2 * ∇_x log p_t(x)] dt
#
# 这个确定性 ODE 就是概率流 ODE（Probability Flow ODE）
# 它将噪声分布映射到数据分布

这个概率流 ODE 告诉我们：即使原本是一个随机过程，我们也可以用确定性轨迹把噪声变换成数据。这也暗示了另一种可能性——与其通过扩散加噪再去噪这个间接途径，为什么我们不直接设计一条从噪声到数据的 直线轨迹 呢？这正是 Flow Matching 的出发点。

🌟 核心洞察：扩散模型 → 概率流 ODE → Flow Matching，这是一条连续的认知链。DDPM 和 Flow Matching 的区别不在于"哪个更好"，而在于"DDPM 选择了一条弯的路，Flow Matching 选择了一条直的路"。弯路的优点是理论推导更成熟，直路的优点是推理效率更高。

20.2 连续归一化流（CNF）基础

连续归一化流（CNF）的核心思想很简单：用一个关于时间 t 的常微分方程（ODE）来定义一个可逆的变换，将初始分布 p_0（通常是高斯噪声）映射到目标分布 p_1（数据分布）。

具体来说，CNF 定义了一个速度场（Vector Field）v_θ(x, t)，它告诉我们在时间 t、位置 x 处，"粒子"应该朝哪个方向运动：

# CNF 的 ODE 定义
#
# dz/dt = v_θ(z_t, t)      # 速度场驱动状态变化
# z_0 ~ p_0 = N(0, I)      # 初始分布：标准高斯噪声
# z_1 ~ p_1 ≈ p_data       # 最终分布：数据分布
#
# 生成过程：给定 z_0，求解 ODE 到 t=1 得到 z_1
# 似然计算：通过 instantaneous change of variables 公式

CNF 的优势是概念优雅——一个连续的确定性变换就能实现分布转换。但实际的训练非常困难，因为我们需要知道每个时刻的速度场 v_θ 应该输什么值。传统的 CNF 训练需要求解昂贵的 ODE 前向过程，这使得它在 2018-2022 年间无法大规模应用。

20.3 Rectified Flow

Rectified Flow 是 Flow Matching 家族中最具实际影响力的成员。它的思想极其简洁：

给定一对数据点 (x_0, x_1)，其中 x_0 ∼ N(0, I)（噪声），x_1 ∼ p_data（真实数据），我们定义线性插值路径：

x_t = (1 - t) * x_0 + t * x_1,   t ∈ [0, 1]

沿着这条路径，速度（即 x_t 对 t 的导数）是常数：

d(x_t)/dt = x_1 - x_0

训练目标就是让神经网络预测这个速度：

L = E_{x_0, x_1, t} [ || v_θ(x_t, t) - (x_1 - x_0) ||^2 ]
💡 直觉理解：想象你站在位置 x_0（噪声），想去位置 x_1（图像）。线性插值路径就是"从起点直线走到终点"。Rectified Flow 就是训练一个模型，告诉它"无论你当前在哪里、在哪个时间点，你都应该沿着 x_1 - x_0 的方向走"。推理时，模型沿着自己预测的方向一步一步走向 x_1。

更精彩的是"整流"（Rectify）的递归过程：训练完第一轮模型后，我们可以用模型生成的样本替换真实数据，重新配对后再次训练——这个过程能让轨迹变得更直。理论上，重复这个"Reflow"过程无限次，轨迹会变成完美的直线，此时一步就能完成生成。

20.4 Flow Matching 损失函数

Flow Matching 的完整理论框架比 Rectified Flow 更一般化。设 p_t(x) 是时间 t 时的概率密度，我们希望学习一个速度场 v_θ(x, t) 来使概率密度沿着 ODE 演化。Flow Matching 的目标函数是：

# 无条件 Flow Matching 损失
L_FM = E_{t ∼ U[0,1], x ∼ p_t(x)} [ || v_θ(x, t) - u_t(x) ||^2 ]

# 其中 u_t(x) 是"真实"速度场，但我们不知道它的闭式解

# 条件 Flow Matching 损失（关键创新！）
L_CFM = E_{t, x_1, x ∼ p_t(x|x_1)} [ || v_θ(x, t) - u_t(x|x_1) ||^2 ]

# 这里 u_t(x|x_1) 是给定终点的条件速度场
# 即 u_t(x|x_1) = (x_1 - x_0) / (1 - t) 对于线性路径

条件 Flow Matching 的伟大之处在于：通过引入条件信息，我们将一个无法直接计算的"全局速度场"问题，转化成了一个容易计算的"局部条件速度场"问题。而这个转化不改变最优解——条件 Flow Matching 和无条件 Flow Matching 共享同一个最优解 v_θ*。

换句话说，训练时只需要关注每一对 (x_0, x_1) 的局部路径，最终的 v_θ 学会了整个分布上的最优速度场——这就是为什么它能生成高质量的样本。

方法损失函数形式路径类型特点
DDPM|| ε_θ(x_t, t) - ε ||²扩散加噪路径弯曲、随机
Rectified Flow|| v_θ(x_t, t) - (x_1 - x_0) ||²线性插值路径直线、确定

让我们从几个关键维度系统比较 DDPM 和 Flow Matching：

从实践中来看，Flow Matching 在少步数场景下的优势非常显著：用 4 步推理时，Rectified Flow 生成的人脸还能保持五官清晰，而 DDPM 用 4 步采样基本是模糊一片。但在 50 步以上时，两者质量相当。

⚠️ 一个真实踩坑故事：2024 年底，我尝试将一个 DDPM 训练的模型用 Rectified Flow 的调度器来做推理——我以为"反正都是去噪，调度器应该通用"。结果输出一片混乱。原因是 DDPM 的噪声调度和 Rectified Flow 的路径定义完全不一样：DDPM 的噪声方差随时间变化（有 ᾱ_t 调度），而 Rectified Flow 的路径是简单的线性插值。两者的时间步语义也不匹配——DDPM 的 t 表示"加噪程度"，Rectified Flow 的 t 表示"从噪声到数据的进度百分比"。调度器和模型必须配套使用，混用等于乱来。

20.6 2026 年的现状

截至 2026 年中，Flow Matching 已经不算"新"技术了——它已经成为图像和视频生成模型的事实标准训练范式。回顾一下关键时间线：

  • 2022 年：Flow Matching 论文发表（Lipman et al.），Rectified Flow（Liu et al.），Stochastic Interpolants（Albergo et al.）同一时期提出。
  • 2023 年：理论验证阶段——小规模实验证明 Flow Matching 可以生成高质量图像，但社区关注度仍远低于 DDPM。
  • 2024 年：SD3 和 FLUX 采用 Rectified Flow 作为训练目标——直接导致整个文生图领域的范式转移。
  • 2025 年：几乎所有新发布的扩散类模型都默认使用 Flow Matching。国产模型（如可灵、混元视频）也全面转向 Flow Matching 路线。
  • 2026 年：Flow Matching 的变体（如 sCM、Consistency Flow）在实时生成领域取得突破。现在"Flow Matching 还是 DDPM"已经不是一个问题了——答案是"看你要几步推理：少步数选 Flow，多步数两可"。
🌟 总结：Flow Matching 统一了扩散模型、归一化流和得分匹配三个领域。如果你只用一句话来概括它的意义——"把弯弯曲曲的扩散路径掰直了，用更少的步数到达终点。"下一章，我们将把这些理论工具投入实战——学习如何使用 ControlNet、LoRA 和 IP-Adapter 来精准控制生成的内容。
